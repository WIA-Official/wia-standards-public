Chapter 4: Protocol Specification

4.1 Overview

This chapter defines the real-time communication protocols and safety interlocks that ensure reliable, safe operation of rehabilitation exoskeleton systems. The WIA protocol operates at multiple levels, from low-level motor control to high-level therapy management.

Safety Safety Monitor 1000 Hz Continuous safety checking
Transport UDP/TCP - Network communication

4.3 Real-time Control Protocol

4.3.1 Control Loop Timing

Critical Timing Requirements

  • Control Frequency: Minimum 100 Hz, recommended 200 Hz
  • Maximum Latency: 10 ms from sensor reading to motor command
  • Jitter: Less than 1 ms variation in control cycle time
  • Watchdog Timeout: 50 ms (trigger safety stop if exceeded)

4.4 Safety Interlocks

4.4.1 Hierarchical Safety System

Level Response Time Action Override
Hardware E-Stop < 1 ms Cut motor power None
Watchdog Timer < 50 ms Controlled stop None
Limit Switch < 5 ms Stop motion, reverse Therapist
Torque Limit < 10 ms Reduce torque Therapist
Software Limit < 10 ms Decelerate to stop Therapist

4.4.2 Emergency Stop Protocol

E-Stop Sequence

  1. Detection (0 ms): E-stop button pressed or safety condition triggered
  2. Signal (0-1 ms): Hardware interrupt to all motor controllers
  3. Power Cut (1-5 ms): Motor power relays open, brakes engage
  4. Verification (5-10 ms): Confirm all motors stopped
  5. Logging (10-100 ms): Record event with full system state
  6. Notification (100-500 ms): Alert operator and safety personnel

4.4.3 Safety Conditions

Condition Threshold Response
Joint Angle Limit Beyond safe range Stop motion, reverse direction
Velocity Limit > 200 deg/s Reduce speed, controlled stop
Torque Overload > 150% rated Immediate reduction to safe level
Motor Temperature > 80°C Reduce load, increase cooling time
Battery Voltage < 20V Warning, prepare for shutdown
Communication Loss > 50 ms Watchdog timeout, controlled stop
Fall Detection IMU threshold Increase stability, alert operator

4.6 Assistance Control Protocols

4.6.1 Impedance Control

Torque is calculated based on virtual spring-damper parameters:

τ = K(θ₀ - θ) + B(0 - θ̇)

Where:
- τ = motor torque
- K = virtual stiffness (Nm/rad)
- θ₀ = equilibrium angle
- θ = current angle
- B = virtual damping (Nm·s/rad)
- θ̇ = current angular velocity
Assistance Level Stiffness (Nm/rad) Damping (Nm·s/rad)
Minimal (0-20%) 0-10 0.5-2
Light (20-40%) 10-30 2-5
Moderate (40-60%) 30-50 5-10
High (60-80%) 50-80 10-15
Maximum (80-100%) 80-120 15-20

4.6.2 Adaptive Assistance Algorithm

if (RMSE > threshold_high):
    assistance = min(current + adaptation_rate, max_assistance)
elif (RMSE < threshold_low):
    assistance = max(current - adaptation_rate/2, min_assistance)
else:
    assistance = current  # Maintain current level

Where RMSE is root-mean-square error between target and actual gait pattern.

4.8 Fault Detection and Recovery

4.8.1 Fault Categories

Fault Type Detection Method Recovery Action
Sensor Failure Data validation, range checking Switch to redundant sensor or safe mode
Communication Loss Watchdog timeout Controlled stop, attempt reconnection
Motor Overload Current monitoring Reduce torque, increase cooling
Battery Depletion Voltage monitoring Gradual shutdown, notify operator
Software Exception Exception handling Log error, restart affected module

4.8.2 Graceful Degradation

Degradation Hierarchy

  1. Full Operation: All systems nominal
  2. Reduced Performance: Limited torque or speed, but functional
  3. Safe Mode: Minimal assistance, safety functions only
  4. Passive Mode: Motors disabled, mechanical support only
  5. Emergency Stop: Complete shutdown

4.10 Protocol Versioning

Semantic Versioning

The WIA Exoskeleton Protocol follows semantic versioning (MAJOR.MINOR.PATCH):

  • MAJOR: Incompatible API changes
  • MINOR: Backward-compatible functionality additions
  • PATCH: Backward-compatible bug fixes

Current Version: 1.0.0
