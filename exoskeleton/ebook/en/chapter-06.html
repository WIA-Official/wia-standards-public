Chapter 6: Security & Privacy

6.1 Overview

Patient data protection and device security are critical for rehabilitation exoskeleton systems. This chapter outlines comprehensive security requirements following HIPAA, GDPR, and medical device cybersecurity standards.

6.2 Data Classification

Data Type Classification Protection Level
Patient Identity (PHI) Highly Sensitive Maximum encryption, strict access control
Session Biomechanics Sensitive Encryption, role-based access
Device Telemetry Internal Encryption, authenticated access
Aggregate Statistics De-identified Standard encryption

6.3 Encryption Requirements

6.3.1 Data at Rest

  • Algorithm: AES-256-GCM
  • Key Management: Hardware Security Module (HSM) or cloud KMS
  • Key Rotation: Every 90 days
  • Scope: All patient data, session recordings, device logs

6.3.2 Data in Transit

  • TLS 1.3 for all API communications
  • Certificate pinning for mobile applications
  • Perfect forward secrecy enabled
  • Mutual TLS for device-to-server communication

6.5 Device Security

6.5.1 Secure Boot

Boot Security Chain

  1. Hardware Root of Trust: TPM or secure element validates bootloader
  2. Signed Bootloader: Cryptographically signed by manufacturer
  3. Kernel Verification: Bootloader verifies OS kernel signature
  4. Application Verification: Only signed applications execute

6.5.2 Firmware Security

Security Feature Implementation
Code Signing All firmware signed with manufacturer private key
Secure Updates Encrypted channel, signature verification before install
Rollback Protection Version counter prevents downgrade attacks
Memory Protection DEP, ASLR, stack canaries enabled

6.6 Network Security

6.6.1 Network Segmentation

Medical Device Network (VLAN 100)
├── Exoskeleton Devices (192.168.100.0/24)
│   └── Isolated from general network
├── Therapy Control Station (192.168.100.128/25)
│   └── Firewalled connection to EHR
└── Data Server (192.168.100.200/29)
    └── One-way data flow to warehouse

6.6.2 Firewall Rules

Source Destination Port Protocol
Exoskeleton Control Station 8443 HTTPS (mutual TLS)
Control Station EHR Server 443 HTTPS
Data Server Warehouse 5432 PostgreSQL (TLS)

6.8 Privacy by Design

6.8.1 Data Minimization

Collect and retain only data necessary for:

  • Patient therapy and safety
  • Clinical decision support
  • Regulatory compliance
  • Quality improvement
