# 제4장: 데이터 형식 사양

## 원주민 문자 데이터의 표준화

이 장에서는 원주민 문자 디지털화에 필요한 모든 데이터 형식을 정의합니다. 문자 인코딩, 폰트 형식, 키보드 정의, 그리고 메타데이터 스키마를 다룹니다.

## 문자 인코딩 스키마

### 유니코드 제안 템플릿

```json
{
  "proposal": {
    "script": {
      "name": "Santali (Ol Chiki)",
      "iso15924": "Olck",
      "direction": "left-to-right",
      "type": "alphabet"
    },
    "codePoints": {
      "requestedRange": "U+1C50-U+1C7F",
      "characters": 48,
      "breakdown": {
        "letters": 30,
        "digits": 10,
        "punctuation": 5,
        "signs": 3
      }
    },
    "evidence": {
      "historicalDocuments": 15,
      "modernUsage": "교육, 출판, 디지털",
      "speakers": 7000000,
      "institutions": ["정부", "학교", "미디어"]
    }
  }
}
```

### 문자 속성 정의

각 문자에 대해 다음 속성을 정의합니다:

```json
{
  "character": {
    "codePoint": "U+1C50",
    "name": "OL CHIKI LETTER LA",
    "generalCategory": "Lo",
    "script": "Olck",
    "properties": {
      "alphabetic": true,
      "lowercase": false,
      "uppercase": false
    },
    "glyph": {
      "reference": "olck-la.svg",
      "variants": ["italic", "handwritten"]
    },
    "phonetic": {
      "ipa": "/la/",
      "romanization": "la"
    }
  }
}
```

### 사설 사용 영역 (PUA) 가이드

유니코드 등재 전 임시 사용:

| 영역 | 범위 | 용도 |
|------|------|------|
| BMP PUA | U+E000-U+F8FF | 일반 문자 |
| Plane 15 | U+F0000-U+FFFFD | 확장 문자 |
| Plane 16 | U+100000-U+10FFFD | 특수 용도 |

**PUA 관리 정책**:
```json
{
  "pua_allocation": {
    "prefix": "WIA",
    "registry": "https://registry.wia-lang.org/pua",
    "allocation_rules": [
      "중앙 레지스트리에 등록 필수",
      "유니코드 등재 후 마이그레이션 계획",
      "충돌 방지를 위한 범위 예약"
    ]
  }
}
```

## 폰트 형식 요구사항

### 필수 폰트 테이블

| 테이블 | 필수 | 설명 |
|--------|------|------|
| `cmap` | ✓ | 문자-글리프 매핑 |
| `glyf`/`CFF` | ✓ | 글리프 데이터 |
| `head` | ✓ | 폰트 헤더 |
| `hhea` | ✓ | 수평 메트릭 헤더 |
| `hmtx` | ✓ | 수평 메트릭 |
| `maxp` | ✓ | 최대 프로필 |
| `name` | ✓ | 이름 테이블 |
| `OS/2` | ✓ | OS/2 및 Windows 메트릭 |
| `post` | ✓ | PostScript 정보 |
| `GSUB` | 조건부 | 글리프 대체 |
| `GPOS` | 조건부 | 글리프 위치 |

### OpenType 기능 요구사항

**복합 문자용 GSUB 기능**:

```
필수 GSUB 기능:
├── ccmp: 문자 구성
├── locl: 지역화된 형태
└── 문자별 추가
    ├── 아랍계: init, medi, fina, isol
    ├── 인도계: akhn, rphf, blwf, half, pstf
    └── 연결 문자: liga, clig
```

**GPOS 기능**:
```
GPOS 기능:
├── mark: 마크 to 베이스
├── mkmk: 마크 to 마크
└── kern: 커닝
```

## 파일 형식 규격

### 폰트 파일

| 형식 | 확장자 | 용도 |
|------|--------|------|
| OpenType/TT | .ttf | 데스크톱 |
| OpenType/CFF | .otf | 데스크톱/인쇄 |
| WOFF2 | .woff2 | 웹 |
| Variable | .ttf/.woff2 | 반응형 |

### 키보드 파일

| 형식 | 확장자 | 플랫폼 |
|------|--------|--------|
| LDML | .xml | 크로스플랫폼 |
| KLC | .klc | Windows |
| Bundle | .bundle | macOS |
| JSON | .json | WIA 표준 |

### 데이터 교환

| 형식 | 용도 |
|------|------|
| JSON | API 통신 |
| JSON-LD | 연결 데이터 |
| CSV | 대량 데이터 |
| XML | 레거시 시스템 |

## 핵심 요약

**주요 내용:**

1. **문자 인코딩**: 유니코드 제안 템플릿, 문자 속성 정의, PUA 관리 정책
