# 제6장: 배포 및 동기화 프로토콜

## 폰트와 키보드의 효율적 배포

이 장에서는 원주민 문자 자원(폰트, 키보드)의 배포와 동기화를 위한 프로토콜을 다룹니다.

## 폰트 배포 프로토콜

### CDN 기반 배포

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│   원본 저장소  │────>│     CDN     │────>│   클라이언트  │
│   (Origin)   │     │   (Edge)    │     │   (Client)  │
└─────────────┘     └─────────────┘     └─────────────┘
```

**CDN 캐시 정책**:
```http
Cache-Control: public, max-age=31536000, immutable
ETag: "v2.005-abc123"
```

### 점진적 폰트 로딩

```javascript
// CSS Font Loading API 사용
const font = new FontFace(
  'Noto Sans Ol Chiki',
  'url(/fonts/NotoSansOlChiki.woff2)',
  { unicodeRange: 'U+1C50-1C7F' }
);

await font.load();
document.fonts.add(font);

// 폴백 처리
document.documentElement.classList.add('fonts-loaded');
```

### 서브셋 폰트

대용량 폰트의 서브셋 제공:

```http
GET /fonts/{id}/subset?chars=U+1C50-U+1C6A

Response:
Content-Type: font/woff2
X-Subset-Chars: U+1C50-U+1C6A
X-Original-Size: 45KB
X-Subset-Size: 12KB
```

## 키보드 동기화 프로토콜

### 버전 관리

```json
{
  "keyboard": {
    "id": "olck-standard",
    "version": "1.2.0",
    "checksum": "sha256:abc123",
    "updated": "2025-01-01T00:00:00Z"
  }
}
```

### 델타 업데이트

```http
GET /keyboards/{id}/delta?from=1.1.0&to=1.2.0

Response:
{
  "delta": {
    "added": [...],
    "modified": [...],
    "removed": [...]
  },
  "patchSize": "2KB"
}
```

### 오프라인 지원

```javascript
// Service Worker 캐싱
self.addEventListener('install', event => {
  event.waitUntil(
    caches.open('wia-keyboards-v1').then(cache => {
      return cache.addAll([
        '/keyboards/olck-standard.json',
        '/fonts/NotoSansOlChiki.woff2'
      ]);
    })
  );
});
```

## 실시간 동기화

### WebSocket 프로토콜

```
클라이언트                          서버
    │                               │
    │──── 연결 ────────────────────>│
    │                               │
    │<─── 구독 확인 ────────────────│
    │                               │
    │<─── 폰트 업데이트 알림 ─────────│
    │                               │
    │──── 다운로드 요청 ─────────────>│
    │                               │
    │<─── 폰트 데이터 ──────────────│
```

### 메시지 형식

```json
{
  "type": "font.updated",
  "data": {
    "fontId": "noto-sans-ol-chiki",
    "version": "2.006",
    "changes": ["added-glyphs", "kerning-fix"]
  }
}
```

## 검증 프로토콜

### 폰트 무결성 검증

```json
{
  "verification": {
    "checksum": {
      "algorithm": "sha256",
      "value": "abc123..."
    },
    "signature": {
      "algorithm": "ed25519",
      "publicKey": "...",
      "value": "..."
    }
  }
}
```

### 키보드 검증

```bash
wia-script verify-keyboard olck-standard.json
# Output:
# ✓ Schema valid
# ✓ Characters in Unicode range
# ✓ No duplicate mappings
# ✓ Signature verified
```

## 오프라인 우선 아키텍처

### IndexedDB 저장

```javascript
const db = await openDB('wia-script', 1, {
  upgrade(db) {
    db.createObjectStore('fonts', { keyPath: 'id' });
    db.createObjectStore('keyboards', { keyPath: 'id' });
  }
});

// 저장
await db.put('fonts', fontData);

// 로드
const font = await db.get('fonts', 'noto-sans-ol-chiki');
```

### 동기화 전략

```
온라인 상태:
├── 서버에서 최신 버전 확인
├── 변경 있으면 다운로드
└── 로컬 DB 업데이트

오프라인 상태:
├── 로컬 DB에서 로드
├── 캐시된 리소스 사용
└── 온라인 복귀 시 동기화
```

## 보안 프로토콜

### 폰트 서명

```
폰트 서명 프로세스:
1. 폰트 해시 생성 (SHA-256)
2. 개인키로 서명
3. 서명을 폰트 메타데이터에 포함
4. 클라이언트가 공개키로 검증
```

### 전송 보안

```
TLS 1.3 필수
├── 인증서 피닝 (선택)
├── HSTS 헤더
└── CSP 헤더 (폰트 소스 제한)
```

## 핵심 요약

1. **폰트 배포**: CDN 기반, 점진적 로딩, 서브셋 지원

3. **실시간 동기화**: WebSocket 알림, 자동 업데이트
4. **오프라인 우선**: IndexedDB, Service Worker 캐싱
5. **보안**: 서명 검증, TLS 필수
