안전한 자격증명 발급

난민 문서를 위한 암호화 기반, 분산 식별자 및 안전한 워크플로우

안전한 자격증명 발급 소개

자격증명 발급은 권한 있는 기관이 개인에게 자격증명을 생성하고 전달하는 과정입니다. 난민 자격증명의 경우, 이 과정은 안전하면서도 실용적이어야 합니다 - 사기를 방지하고 자격증명 무결성을 보장할 만큼 안전하면서도 난민이 등록되는 도전적인 환경에서 작동할 수 있을 만큼 실용적이어야 합니다. WIA 표준은 암호화 요구 사항, 운영 보안 및 워크플로우 설계를 다루는 포괄적인 발급 프레임워크를 명시합니다.

현대 자격증명 발급은 수십 년의 암호화 연구 및 표준 개발을 기반으로 합니다. 디지털 서명은 위조하거나 탐지 없이 수정할 수 없는 자격증명을 가능하게 합니다. 분산 식별자는 단일 기관에 의존하지 않는 신원 앵커를 가능하게 합니다. 검증 가능한 자격증명은 클레임을 표현하고 검증하기 위한 표준화된 형식을 제공합니다. WIA 표준은 이러한 기술을 인도주의적 맥락에 최적화된 일관된 프레임워크로 통합합니다.

암호화 기반

암호화는 자격증명 보안의 수학적 기반을 제공합니다. 디지털 서명은 자격증명이 권한 있는 기관에 의해 발급되었고 수정되지 않았음을 증명합니다. 해시 함수는 자격증명 내용의 고유한 지문을 생성합니다. 공개 키 인프라는 서명 및 검증에 필요한 키를 관리합니다. WIA 표준은 난민 자격증명에 적절한 보안 수준을 제공하는 암호화 알고리즘 및 매개변수를 명시합니다.

디지털 서명

디지털 서명은 자격증명 보안의 초석입니다. 발급자가 자격증명에 서명하면 자격증명 내용을 발급자의 개인 키에 바인딩하는 수학적 증명을 생성합니다. 발급자의 공개 키를 가진 누구나 이 증명을 검증하여 발급자의 신원과 자격증명의 무결성을 확인할 수 있습니다. 서명 후 자격증명에 대한 모든 수정은 서명을 무효화하여 변조를 감지할 수 있게 합니다. 

// 지원되는 서명 알고리즘
{
  "signatureAlgorithms": {
    "recommended": {
      "algorithm": "Ed25519",
      "keySize": 256,
      "signatureSize": 64,
      "performance": "우수",
      "specification": "RFC 8032"
    },
    "alternative": {
      "algorithm": "ES256",
      "curve": "P-256",
      "keySize": 256,
      "signatureSize": 64,
      "specification": "RFC 7518"
    },
    "legacy": {
      "algorithm": "RS256",
      "keySize": "최소 2048, 권장 4096",
      "signatureSize": "256-512 바이트",
      "specification": "RFC 7518"
    }
  }
}

키 관리

암호화 키는 자격증명 보안의 기반입니다. 개인 키는 극도의 주의를 기울여 보호해야 합니다 - 발급자의 개인 키가 손상되면 공격자가 합법적인 것과 구별할 수 없는 사기성 자격증명을 생성할 수 있습니다. WIA 표준은 키 생성, 저장, 사용, 교체 및 파기를 다루는 키 관리 요구 사항을 명시합니다.

하드웨어 보안 모듈(HSM)은 고가치 발급자 키에 대한 가장 강력한 보호를 제공합니다. HSM은 내부에서 키를 생성하고 소프트웨어에 개인 키를 노출하지 않는 변조 방지 장치입니다. 모든 서명 작업은 HSM 내에서 발생하여 손상된 서버라도 키를 추출할 수 없도록 보장합니다. WIA 표준은 IAL2 이상의 모든 프로덕션 자격증명 발급 시스템에 대해 HSM 지원 키 저장을 요구합니다.

키 관리 요구 사항

요구 사항 IAL1-2 IAL3-4
키 생성 암호학적으로 안전한 RNG HSM 생성
키 저장 암호화, 접근 제어 HSM 지원
키 사용 애플리케이션 레벨 제어 HSM 정책 시행
키 교체 최소 연간 반기 또는 손상 시
키 백업 암호화된 오프라인 백업 HSM 키 세레모니

분산 식별자 (DIDs)

분산 식별자는 검증 가능한 분산 디지털 신원을 위해 특별히 설계된 새로운 유형의 식별자를 제공합니다. 정부나 기업이 발급하는 전통적인 식별자와 달리, DID는 어떤 조직의 허가 없이 누구나 생성할 수 있으면서도 암호학적으로 검증 가능합니다. 이 속성은 전통적인 신원 발급자에 대한 접근권이 없을 수 있는 난민 인구에게 필수적입니다.

DID 구조

DID는 주체를 식별하고 공개 키 및 기타 메타데이터를 포함하는 DID 문서로 확인될 수 있는 URI입니다. DID 형식은 표준 구조를 따릅니다: 스킴 식별자("did"), 메서드 식별자(DID가 작동하는 방식 지정), 메서드별 식별자(해당 메서드 내의 실제 식별자). 

// DID 구조 예시
did:wia:refugee:b94d27b9934d3e08a52e52d7da7dabfac484efe37a5380ee9088f7ace2efcde9

┌─────────┬────────────┬─────────────────────────────────────────────────────────┐
│  did:   │    wia:    │    refugee:b94d27b9934d3e08...                          │
├─────────┼────────────┼─────────────────────────────────────────────────────────┤
│ 스킴    │  메서드    │  메서드별 식별자                                        │
│         │            │  (초기 등록 데이터의 암호화 해시)                        │
└─────────┴────────────┴─────────────────────────────────────────────────────────┘

DID 문서

모든 DID는 DID 문서 - 해당 신원과 연결된 공개 키, 인증 방법 및 서비스 엔드포인트를 포함하는 JSON-LD 문서 - 로 확인됩니다. DID 문서는 검증자가 자격증명을 확인하고 신원 보유자와 통신하는 데 필요한 정보를 조회할 수 있게 합니다. 

// DID 문서 예시
{
  "@context": [
    "https://www.w3.org/ns/did/v1",
    "https://wia.org/ns/refugee-credential/v1"
  ],
  "id": "did:wia:refugee:b94d27b9934d3e08...",
  "controller": "did:wia:refugee:b94d27b9934d3e08...",
  "verificationMethod": [{
    "id": "did:wia:refugee:b94d27b9934d3e08...#key-1",
    "type": "Ed25519VerificationKey2020",
    "controller": "did:wia:refugee:b94d27b9934d3e08...",
    "publicKeyMultibase": "z6MkhaXgBZDvotDkL5257faiztiGiC2QtKLGpbnnEGta2doK"
  }],
  "authentication": [
    "did:wia:refugee:b94d27b9934d3e08...#key-1"
  ],
  "service": [{
    "id": "did:wia:refugee:b94d27b9934d3e08...#credential-service",
    "type": "CredentialService",
    "serviceEndpoint": "https://api.wia.org/credentials/v1"
  }]
}

검증 가능한 자격증명 아키텍처

검증 가능한 자격증명(VC)은 암호학적으로 안전하고 프라이버시를 존중하며 기계가 검증할 수 있는 방식으로 자격증명을 표현하기 위한 W3C 표준입니다. VC는 주체에 대한 클레임, 자격증명 자체에 대한 메타데이터, 암호화 증명을 포함합니다. WIA 표준은 난민 자격증명에 특화된 확장과 함께 VC 데이터 모델을 기반으로 합니다.

자격증명 구조

모든 검증 가능한 자격증명은 여러 필수 요소를 포함합니다: 컨텍스트(사용된 용어 정의), 유형(자격증명 분류), 발급자(누가 만들었는지), 발급 날짜, 자격증명 주체(누구에 대한 것인지), 증명(암호화 서명). 선택적 요소에는 만료 날짜, 자격증명 상태(취소 확인용) 및 추가 메타데이터가 포함됩니다.

발급 워크플로우

자격증명 발급은 자격증명 생성 전에 적절한 확인을 보장하고 보유자에게 안전하게 전달하는 구조화된 워크플로우를 따릅니다. WIA 표준은 다른 자격증명 유형 및 보증 수준에 대한 발급 워크플로우를 정의하며, 모든 워크플로우에 공통 요소가 있습니다.

1

등록 요청

자격증명 주체가 등록을 위해 자신을 제시하여 프로세스를 시작합니다. 생체 인식 캡처가 신원 앵커를 확립합니다. 기본 인적 정보가 수집됩니다. 기존 문서가 검토됩니다.

2

신원 확인

요청된 보증 수준에 적합한 확인 절차가 완료됩니다. 생체 인식 중복 제거가 기존 등록을 확인합니다. 해당되는 경우 문서 확인. 필요에 따라 증인 증명 또는 인터뷰.

3

DID 생성

아직 등록되지 않은 경우 주체에 대해 DID가 생성됩니다. 키 쌍이 보안 환경에서 생성됩니다. DID 문서가 생성되고 게시됩니다. 주체가 DID와 초기 키를 받습니다.

4

자격증명 생성

적절한 스키마에 따라 자격증명이 구성됩니다. 확인된 정보에서 클레임이 채워집니다. 만료 및 상태 엔드포인트를 포함한 메타데이터가 추가됩니다. 발급자가 자격증명에 서명합니다.

5

자격증명 전달

서명된 자격증명이 보유자에게 전달됩니다. 전달은 모바일 지갑, 임베디드 칩이 있는 물리적 카드 또는 보안 클라우드 저장소로 가능합니다. 보유자가 수신 및 자격증명 제시 능력을 확인합니다.

자격증명 저장 및 지갑

발급되면 자격증명은 안전하게 저장되고 제시를 위해 접근 가능해야 합니다. 디지털 지갑은 보유자가 자격증명을 관리하고, 무단 접근으로부터 보호하면서 검증자에게 편리한 제시를 가능하게 하는 인터페이스를 제공합니다. WIA 표준은 보안과 상호운용성을 보장하는 지갑 요구 사항을 명시합니다.

지갑 보안 요구 사항

  • 키 보호: 가능한 경우 보안 엔클레이브 또는 TEE에 개인 키 저장
  • 접근 제어: 자격증명 접근에 생체 인식 또는 PIN 인증 필요
  • 암호화: 저장된 모든 자격증명은 저장 시 암호화
  • 백업: 키 복구 옵션과 함께 암호화된 백업 기능
  • 취소: 장치 분실 시 원격으로 지갑 접근 취소 가능
  • 감사 로그: 보유자 검토를 위한 모든 자격증명 제시 기록

취소 및 생명주기 관리

자격증명은 다양한 이유로 자연 만료 전에 취소해야 할 수 있습니다: 보유자의 상태가 변경될 수 있고, 자격증명이 오류로 발급되었을 수 있으며, 보유자가 자격증명 분실 또는 도난을 신고할 수 있습니다. WIA 표준은 프라이버시 우려에 대비하여 적시 상태 업데이트의 필요성을 균형 맞추는 자격증명 취소 메커니즘을 제공합니다.

장 요약

안전한 자격증명 발급은 운영 보안 관행과 암호화 기술의 통합이 필요합니다. 디지털 서명(권장 Ed25519)은 자격증명 무결성과 발급자 진정성을 보장합니다. HSM 지원 키 관리는 프로덕션 발급 시스템에 필수입니다. 분산 식별자는 중앙 기관 의존성 없이 신원을 가능하게 합니다. 검증 가능한 자격증명은 난민 맥락에 대한 WIA 확장과 함께 W3C 표준을 따릅니다.

핵심 요점

  • 디지털 서명(권장 Ed25519)이 자격증명 무결성과 발급자 진정성을 보장
  • 프로덕션 발급 시스템에 HSM 지원 키 관리 필수
  • 분산 식별자가 중앙 기관 의존성 없이 신원 가능
  • 검증 가능한 자격증명이 난민 맥락에 대한 WIA 확장과 함께 W3C 표준 준수
  • 발급 워크플로우가 보증 수준 요구 사항에 확인 강도를 맞춤
  • 자격증명 지갑이 오프라인 저장 및 안전한 제시 지원 필수
  • 취소 메커니즘이 프라이버시를 보호하면서 적시 상태 업데이트 가능
