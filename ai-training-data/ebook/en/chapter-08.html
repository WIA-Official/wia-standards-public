Chapter 8: Privacy-Preserving Training Data

Protecting Privacy While Enabling AI Innovation

Privacy Challenges in AI Training Data

Training data often contains sensitive personal information (PII), protected health information (PHI), financial records, or confidential business data. Protecting privacy while maintaining data utility is essential for ethical AI development aligned with 弘益人間 principles. WIA-AI-007 provides comprehensive frameworks for privacy-preserving data management.

Privacy Threats and Risks

Re-identification Attacks

Combining anonymized data with external information sources to identify individuals. Famous example: Netflix Prize dataset re-identification.

Membership Inference

Determining whether a specific individual's data was in the training set by analyzing model behavior.

Model Inversion

Reconstructing training data samples from model parameters or predictions.

Attribute Inference

Inferring sensitive attributes not explicitly in the model from other available features.

Data Anonymization Techniques

De-identification

Remove or mask direct identifiers like names, addresses, phone numbers, email addresses, and social security numbers.

from wia_ai_007 import DataAnonymizer

anonymizer = DataAnonymizer()

# Remove PII fields
anonymized_data = anonymizer.remove_pii(
    dataset="medical_records.parquet",
    pii_fields=["patient_name", "ssn", "address", "phone", "email"],
    replacement_strategy="pseudonym"
)

# Verify PII removal
verification = anonymizer.verify_anonymization(
    dataset=anonymized_data,
    pii_detection_threshold=0.99
)

if verification.pii_detected:
    print("⚠️ PII still detected:", verification.detected_fields)

# 弘益人間 - Protecting individual privacy for collective benefit

K-Anonymity

Ensure each record is indistinguishable from at least k-1 other records with respect to quasi-identifiers.

# Apply k-anonymity
k_anon_data = anonymizer.k_anonymity(
    dataset="patient_data.parquet",
    quasi_identifiers=["age", "zip_code", "gender"],
    k=5,  # Each combination appears ≥5 times
    generalization_hierarchy="hierarchies.json"
)

# Verify k-anonymity
assert k_anon_data.min_group_size() >= 5

L-Diversity

Extends k-anonymity by requiring each equivalence class to have at least l well-represented values for sensitive attributes.

T-Closeness

Ensures the distribution of sensitive attributes in each equivalence class is close to the overall distribution.

Differential Privacy

Mathematical framework providing provable privacy guarantees by adding calibrated noise to data or query results.

ε-Differential Privacy

A randomized algorithm M satisfies ε-differential privacy if for all datasets D1 and D2 differing by one record, and all possible outputs S:

P(M(D1) ∈ S) ≤ e^ε × P(M(D2) ∈ S)

from wia_ai_007 import DifferentialPrivacy

dp = DifferentialPrivacy(epsilon=1.0)  # Privacy budget

# Add noise to aggregated statistics
noisy_mean = dp.add_laplace_noise(
    true_value=statistics.mean(ages),
    sensitivity=1.0
)

# Private data release
private_dataset = dp.privatize_dataset(
    dataset="sensitive_data.parquet",
    epsilon=1.0,
    delta=1e-5,
    method="laplace_mechanism"
)

# Track privacy budget consumption
print(f"Remaining privacy budget: {dp.remaining_budget()}")

Federated Learning

Train models across decentralized data sources without centralizing data.

from wia_ai_007 import FederatedLearning

# Initialize federated learning
fed_learning = FederatedLearning(
    num_clients=10,
    aggregation_method="federated_averaging"
)

# Clients train locally on their private data
for round in range(100):
    # Each client trains on local data
    local_updates = []
    for client in fed_learning.clients:
        update = client.train_local(
            epochs=5,
            data=client.local_data
        )
        local_updates.append(update)

    # Server aggregates updates (no raw data shared)
    global_model = fed_learning.aggregate(local_updates)

    # Distribute updated model to clients
    fed_learning.broadcast_model(global_model)

# 弘益人間 - Collaborative learning while protecting privacy

Secure Aggregation

Cryptographically secure aggregation ensuring the server only learns the aggregate update, not individual contributions.

Secure Multi-Party Computation (SMPC)

Enable multiple parties to jointly compute functions over their private inputs without revealing inputs to each other.

from wia_ai_007 import SecureComputation

# Three hospitals want to train jointly without sharing data
smpc = SecureComputation(parties=["hospital_A", "hospital_B", "hospital_C"])

# Compute joint statistics securely
joint_mean = smpc.secure_mean(
    party_data={
        "hospital_A": hospital_A_ages,
        "hospital_B": hospital_B_ages,
        "hospital_C": hospital_C_ages
    }
)

# No party learns others' individual data
# Only aggregate result is revealed

Homomorphic Encryption

Perform computations on encrypted data without decryption. Results remain encrypted until decrypted by the key holder.

Partially Homomorphic Encryption (PHE)

Supports either addition or multiplication on encrypted data.

Fully Homomorphic Encryption (FHE)

Supports arbitrary computations on encrypted data, but computationally expensive.

Synthetic Data Generation

Generate artificial datasets that preserve statistical properties of real data while containing no actual personal information.

from wia_ai_007 import SyntheticDataGenerator

generator = SyntheticDataGenerator()

# Train generative model on real data
generator.fit(
    real_data="sensitive_medical_data.parquet",
    privacy_guarantee="differential_privacy",
    epsilon=1.0
)

# Generate synthetic data
synthetic_data = generator.generate(
    num_samples=10000,
    preserve_correlations=True
)

# Validate privacy and utility
privacy_score = generator.measure_privacy(synthetic_data, real_data)
utility_score = generator.measure_utility(synthetic_data, real_data)

print(f"Privacy score: {privacy_score:.3f}")
print(f"Utility score: {utility_score:.3f}")

Generative Models for Privacy

  • GANs: Generative Adversarial Networks for realistic synthetic data
  • VAEs: Variational Autoencoders for controlled generation
  • Diffusion Models: State-of-the-art generation with privacy controls
  • Copulas: Preserve statistical dependencies

Privacy-Utility Trade-offs

Strong privacy protections often reduce data utility. WIA-AI-007 recommends:

  • Define minimum utility requirements for the task
  • Set appropriate privacy parameters (ε, k, l) based on sensitivity
  • Use privacy-enhancing techniques that minimize utility loss
  • Document trade-off decisions transparently
  • Validate that privacy-protected data still enables effective models

Regulatory Compliance

GDPR (General Data Protection Regulation)

  • Right to erasure ("right to be forgotten")
  • Data minimization principles
  • Purpose limitation
  • Privacy by design and by default

HIPAA (Health Insurance Portability and Accountability Act)

  • Protection of PHI (Protected Health Information)
  • Safe harbor de-identification
  • Expert determination method
  • Business Associate Agreements

CCPA (California Consumer Privacy Act)

  • Consumer rights to know, delete, and opt-out
  • Data sale disclosure
  • Non-discrimination for privacy choices

Access Controls and Audit Logging

Implement strong access controls and comprehensive audit trails:

  • Role-based access control (RBAC)
  • Data access logging and monitoring
  • Encryption at rest and in transit
  • Multi-factor authentication for sensitive data
  • Regular access audits and reviews

Privacy-Preserving Machine Learning

DP-SGD (Differentially Private Stochastic Gradient Descent)

Train models with formal differential privacy guarantees by clipping gradients and adding calibrated noise during training.

from wia_ai_007 import PrivateTraining

private_trainer = PrivateTraining(
    epsilon=8.0,
    delta=1e-5,
    max_grad_norm=1.0
)

model = private_trainer.train(
    model=neural_network,
    training_data=sensitive_data,
    epochs=50,
    batch_size=256
)

print(f"Model trained with (ε={private_trainer.epsilon}, δ={private_trainer.delta})-DP")

Private Aggregation of Teacher Ensembles (PATE)

Train student model on unlabeled public data using noisy consensus of teacher models trained on private partitions.

Data Retention and Deletion

Implement policies for data lifecycle management:

  • Define retention periods based on legal and business requirements
  • Automate data deletion after retention period expires
  • Support user-requested deletion (GDPR right to erasure)
  • Maintain deletion logs for compliance audits
  • Securely wipe deleted data (not just logical deletion)

Best Practices for Privacy

  • Privacy by design: Integrate privacy from the start, not as an afterthought
  • Minimize collection: Collect only necessary data
  • Layered protection: Combine multiple privacy techniques
  • Regular assessment: Conduct periodic privacy impact assessments
  • Transparency: Document privacy measures clearly
  • User consent: Obtain informed consent for data usage
  • Incident response: Prepare for potential breaches
  • Continuous monitoring: Monitor for privacy violations

Chapter Summary

Privacy-preserving techniques enable AI development while protecting sensitive information. WIA-AI-007 provides frameworks for privacy impact assessment, regulatory compliance (GDPR, HIPAA, CCPA), and privacy-utility trade-off management. Privacy-preserving machine learning techniques like DP-SGD and PATE enable training with formal privacy guarantees. These practices embody 弘益人間 by ensuring AI benefits humanity while respecting individual privacy rights.
