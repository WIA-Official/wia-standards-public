Building Fair and Ethical AI Systems

Understanding Bias in Training Data

Bias in training data leads to AI systems that perpetuate and amplify discrimination, unfairness, and societal inequities. Addressing bias is fundamental to the philosophy of 弘益人間 (Benefit All Humanity)—AI must serve all people fairly and equitably. WIA-AI-007 provides comprehensive frameworks for detecting, measuring, and mitigating bias throughout the data lifecycle.

Types of Bias

Representation Bias

Occurs when some groups are under-represented or over-represented in the training data. Example: A medical dataset with 90% data from one demographic group leads to poor performance on others.

Measurement Bias

Arises from systematic errors in how features are captured or labeled. Example: Pulse oximeters that are less accurate for darker skin tones create biased training labels.

Historical Bias

Reflects past societal biases and inequalities embedded in historical data. Example: Hiring data reflecting historical gender imbalances in certain professions.

Aggregation Bias

Results from using a single model for diverse groups with different relationships between features and outcomes. Example: A diabetes risk model that performs differently across ethnic groups.

Evaluation Bias

Occurs when test/validation data doesn't represent deployment diversity. Models appear fair on biased test sets but fail in practice.

Label Bias

Systematic errors or stereotypes in labeling. Example: Biased annotators consistently mislabeling certain demographics.

Bias Detection Frameworks

from wia_ai_007 import BiasDetector

detector = BiasDetector()

# Analyze representation across protected attributes
representation_analysis = detector.analyze_representation(
    dataset="medical_dataset.parquet",
    protected_attributes=["gender", "race", "age_group"],
    report_format="detailed"
)

print(f"Gender distribution: {representation_analysis['gender']}")
print(f"Race distribution: {representation_analysis['race']}")
print(f"Bias score: {representation_analysis['overall_bias_score']}")

# Detect label bias
label_bias = detector.analyze_label_bias(
    dataset="medical_dataset.parquet",
    labels="diagnosis",
    protected_attributes=["race", "gender"]
)

if label_bias.significant:
    print("⚠️ Significant label bias detected")
    print(f"Affected groups: {label_bias.affected_groups}")

Fairness Metrics

Demographic Parity

Equal positive prediction rates across groups: P(Ŷ=1 | A=a) = P(Ŷ=1 | A=b)

Equal Opportunity

Equal true positive rates across groups: P(Ŷ=1 | Y=1, A=a) = P(Ŷ=1 | Y=1, A=b)

Equalized Odds

Equal true positive and false positive rates across groups.

Predictive Parity

Equal precision across groups: P(Y=1 | Ŷ=1, A=a) = P(Y=1 | Ŷ=1, A=b)

from wia_ai_007 import FairnessMetrics

fairness = FairnessMetrics()

# Calculate multiple fairness metrics
metrics = fairness.calculate(
    dataset="predictions.parquet",
    protected_attribute="race",
    label="actual_outcome",
    prediction="predicted_outcome",
    metrics=["demographic_parity", "equal_opportunity", "equalized_odds"]
)

# Generate fairness report
report = fairness.generate_report(
    metrics=metrics,
    threshold=0.8,  # 80% fairness threshold
    output="fairness_report.html"
)

print(f"Demographic parity: {metrics['demographic_parity']:.3f}")
print(f"Passes fairness threshold: {report.passes}")

Intersectional Bias Analysis

Bias often manifests at the intersection of multiple protected attributes (e.g., Black women, elderly Asian men). Intersectional analysis reveals biases missed by single-attribute analysis.

# Intersectional bias detection
intersectional_analysis = detector.analyze_intersectional(
    dataset="medical_dataset.parquet",
    protected_attributes=["race", "gender", "age_group"],
    min_group_size=100
)

# Identify most affected intersections
for intersection in intersectional_analysis.top_disparities:
    print(f"{intersection.groups}: {intersection.disparity_ratio:.2f}")

Bias Mitigation Strategies

Pre-Processing Mitigation

  • Resampling: Oversample underrepresented groups, undersample overrepresented
  • Reweighting: Assign weights to balance group representation
  • Synthetic data generation: Generate synthetic samples for underrepresented groups
  • Fairness-aware feature engineering: Remove or transform biased features
from wia_ai_007 import BiasMitigator

mitigator = BiasMitigator()

# Rebalance dataset
balanced_dataset = mitigator.rebalance(
    dataset="medical_dataset.parquet",
    protected_attribute="race",
    method="SMOTE",  # Synthetic oversampling
    target_distribution="uniform"
)

# Reweight samples for fairness
weighted_dataset = mitigator.reweight(
    dataset="medical_dataset.parquet",
    protected_attribute="gender",
    fairness_constraint="demographic_parity"
)

In-Processing Mitigation

Modify learning algorithms to optimize for fairness during training:

  • Adversarial debiasing
  • Prejudice remover regularization
  • Fairness-constrained optimization

Post-Processing Mitigation

Adjust model predictions to satisfy fairness constraints:

  • Threshold optimization per group
  • Calibrated equalized odds
  • Reject option classification

Counterfactual Fairness

A prediction is counterfactually fair if it remains the same in the actual world and in a counterfactual world where the individual belonged to a different demographic group.

Fairness-Accuracy Trade-offs

Improving fairness sometimes reduces overall accuracy. WIA-AI-007 recommends:

  • Define minimum acceptable accuracy per group
  • Set fairness thresholds based on application criticality
  • Document trade-off decisions transparently
  • Involve stakeholders in fairness-accuracy balance decisions

Continuous Bias Monitoring

# Set up continuous monitoring
from wia_ai_007 import BiasMonitor

monitor = BiasMonitor(
    dataset_id="medical-predictions",
    protected_attributes=["race", "gender"],
    fairness_metrics=["demographic_parity", "equal_opportunity"]
)

# Define alerts for fairness violations
monitor.add_alert(
    metric="demographic_parity",
    threshold=0.8,
    action="notify_team",
    recipients=["fairness-team@example.com"]
)

# Generate periodic fairness reports
monitor.schedule_reports(
    frequency="weekly",
    format="html",
    recipients=["stakeholders@example.com"]
)

monitor.start()
