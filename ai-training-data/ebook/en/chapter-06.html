Chapter 6: Data Augmentation Techniques

Expanding Datasets While Preserving Integrity

The Role of Data Augmentation

Data augmentation artificially expands training datasets by creating modified versions of existing samples. This improves model robustness, reduces overfitting, and enables learning from limited data. Properly designed augmentation preserves label validity while introducing meaningful variation, embodying the 弘益人間 principle of maximizing value from available resources.

Image Augmentation Techniques

Geometric Transformations

  • Rotation: Rotate images by various angles (-30° to +30°)
  • Flipping: Horizontal and vertical flips
  • Scaling: Zoom in/out (80%-120% of original size)
  • Translation: Shift images horizontally/vertically
  • Shearing: Slant images along axes
  • Perspective transforms: Simulate different viewing angles
from wia_ai_007 import ImageAugmentor

augmentor = ImageAugmentor()

# Define augmentation pipeline
pipeline = augmentor.create_pipeline([
    {"operation": "rotation", "range": [-15, 15]},
    {"operation": "horizontal_flip", "probability": 0.5},
    {"operation": "zoom", "range": [0.9, 1.1]},
    {"operation": "brightness", "range": [0.8, 1.2]},
    {"operation": "contrast", "range": [0.9, 1.1]}
])

# Apply to dataset
augmented_data = augmentor.augment(
    original_dataset=medical_images,
    pipeline=pipeline,
    augmentation_factor=3,  # 3x data size
    preserve_originals=True
)

Color and Intensity Transformations

  • Brightness adjustment: Vary image brightness
  • Contrast adjustment: Modify contrast levels
  • Saturation: Change color saturation
  • Hue rotation: Shift color hues
  • Grayscale conversion: Convert to grayscale
  • Gaussian noise: Add random noise
  • Blur: Apply Gaussian or motion blur

Advanced Image Augmentation

  • Cutout/Random erasing: Mask random rectangular regions
  • Mixup: Blend two images and their labels
  • CutMix: Cut and paste image patches
  • AutoAugment: Learn optimal augmentation policies
  • RandAugment: Random augmentation with consistent strength

Text Augmentation Techniques

Token-Level Augmentation

  • Synonym replacement: Replace words with synonyms
  • Random insertion: Insert random synonyms
  • Random deletion: Remove random words
  • Random swap: Swap word positions

Semantic Augmentation

  • Back-translation: Translate to another language and back
  • Paraphrasing: Generate semantic equivalents
  • Contextual word substitution: Use language models for replacements
from wia_ai_007 import TextAugmentor

text_augmentor = TextAugmentor()

# Synonym-based augmentation
augmented_texts = text_augmentor.augment(
    texts=training_texts,
    methods=["synonym_replacement", "random_insertion"],
    augmentation_factor=2,
    preserve_labels=True
)

# Back-translation augmentation
back_translated = text_augmentor.back_translate(
    texts=training_texts,
    intermediate_language="fr",
    model="opus-mt"
)

Audio Augmentation Techniques

  • Time stretching: Change speed without affecting pitch
  • Pitch shifting: Change pitch without affecting tempo
  • Add noise: Inject background noise
  • Volume adjustment: Scale amplitude
  • Time shifting: Shift audio in time
  • Room simulation: Add reverb/echo effects

Tabular Data Augmentation

  • SMOTE: Synthetic Minority Over-sampling Technique
  • ADASYN: Adaptive Synthetic Sampling
  • Gaussian noise injection: Add noise to numerical features
  • Feature mixing: Mix features from similar samples
from wia_ai_007 import TabularAugmentor

tab_augmentor = TabularAugmentor()

# SMOTE for imbalanced classification
augmented_data = tab_augmentor.smote(
    X=features,
    y=labels,
    sampling_strategy="minority",
    k_neighbors=5
)

# Gaussian noise augmentation
noisy_data = tab_augmentor.add_gaussian_noise(
    data=features,
    noise_level=0.05,
    features=["age", "blood_pressure", "glucose"]
)

Synthetic Data Generation

Generative Models

  • GANs: Generative Adversarial Networks for realistic synthetic samples
  • VAEs: Variational Autoencoders for controlled generation
  • Diffusion Models: State-of-the-art image generation
  • GPT/LLM: Large language models for text generation

Simulation-Based Generation

Use simulators to generate labeled synthetic data for domains like robotics, autonomous vehicles, and physics-based scenarios.

Domain-Specific Augmentation

Medical Imaging

  • Preserve anatomical validity
  • Avoid unrealistic transformations
  • Consider imaging modality constraints (CT, MRI, X-ray)
  • Maintain pathology characteristics

Autonomous Vehicles

  • Weather condition simulation (rain, fog, snow)
  • Lighting variations (day, night, shadows)
  • Sensor noise simulation
  • Object occlusion scenarios

Augmentation Best Practices

  • Preserve label validity: Ensure augmentations don't change labels
  • Domain appropriateness: Use augmentations relevant to deployment conditions
  • Balanced application: Apply augmentation proportionally across classes
  • Validation set integrity: Never augment validation/test sets
  • Track augmentation: Record which samples are augmented and how
  • Quality control: Review augmented samples for artifacts
  • Performance monitoring: Verify augmentation improves results

Online vs Offline Augmentation

Offline (Pre-computed)

  • Generate augmented samples beforehand
  • Store augmented data permanently
  • Faster training but requires more storage
  • Good for expensive augmentation operations

Online (On-the-fly)

  • Generate augmentations during training
  • No additional storage required
  • Infinite variations possible
  • Slight training overhead

Augmentation Policy Optimization

Automatically discover optimal augmentation strategies:

from wia_ai_007 import AutoAugment

auto_aug = AutoAugment()

# Search for optimal policy
optimal_policy = auto_aug.search(
    training_data=train_set,
    validation_data=val_set,
    search_space="medical_imaging",
    search_iterations=100
)

# Apply discovered policy
augmented_data = auto_aug.apply_policy(
    data=train_set,
    policy=optimal_policy
)

Validation and Quality Control

Ensure augmented data maintains quality:

  • Visual/manual inspection of samples
  • Statistical comparison with original distribution
  • Downstream task performance validation
  • Expert review for domain-critical applications
  • Artifact detection automated checks

Ethical Considerations

Augmentation must be applied responsibly:

  • Bias amplification: Ensure augmentation doesn't amplify existing biases
  • Realistic constraints: Don't create unrealistic scenarios
  • Privacy preservation: Synthetic data should not reveal private information
  • Transparency: Document augmentation methods clearly
  • Validation: Verify augmented models work in real conditions

Chapter Summary

Data augmentation expands training datasets through transformations that preserve label validity while introducing beneficial variation. Techniques vary by modality: geometric and color transformations for images, token substitution and paraphrasing for text, time/pitch shifts for audio, and SMOTE for tabular data. Synthetic generation via GANs, VAEs, and simulators provides additional augmentation capabilities.

WIA-AI-007 provides standardized metadata for tracking augmentations, best practices for domain-specific applications, and guidelines for ethical augmentation. Proper augmentation improves model robustness and generalization, embodying 弘益人間 by maximizing the value of available training data.
