2.1 기본 구문

모든 WIA-INTENT 프로그램은 의도 선언으로 구성됩니다. 기본 구조는 다음과 같습니다: 

// 의도 선언
intent <IntentName> {
  // 주어진 것 (입력)
  given: <inputs>

  // 원하는 것 (출력/목표)
  want: <desired_outcome>

  // 솔루션에 대한 제약
  constraints {
    <constraint_name>: <constraint_value>
  }

  // 신뢰도/확실성 요구 사항
  certainty: <0.0 - 1.0>

  // 의도를 충족할 수 없는 경우 수행할 작업
  fallback: <fallback_action>

  // 솔루션이 진화할 수 있는 방법
  evolve: <evolution_rules>
}

주요 구성 요소

  • given: 입력 데이터 및 컨텍스트
  • want: 원하는 결과 또는 목표
  • constraints: 요구 사항 및 경계
  • certainty: 필요한 신뢰 수준 (0.0-1.0)
  • fallback: 의도를 충족할 수 없을 때의 동작
  • evolve: 학습 및 개선 규칙

2.2 데이터 타입

2.2.1 기본 타입

WIA-INTENT는 표준 기본형과 고유한 확률적 타입을 지원합니다:

// 스칼라 타입
number: 42, 3.14, -17
text: "hello world"
truth: true, false, maybe(0.7)  // 확률적 진리값!
nothing: void

// 시간 타입
moment: @2025-12-16T10:30:00Z
duration: 5.minutes, 2.hours, 1.day

2.2.2 확률적 타입

WIA-INTENT의 고유한 기능: 타입 시스템에 내장된 불확실성 및 확률 분포에 대한 기본 지원. 

// 불확실성을 언어 자체에서 표현
probability: 0.85
distribution: normal(mean: 100, std: 15)
range: 10..20 with_confidence 0.9

// Maybe 타입 (확률적)
maybe<number>: 42 with_probability 0.95

확률적 타입이 중요한 이유

실제 데이터는 불확실합니다. 날씨 예측, 사용자 행동, 시장 트렌드— 모두 확률을 포함합니다. WIA-INTENT는 불확실성을 일급 개념으로 만들어 프로그램이 신뢰 수준에 대해 명시적으로 추론할 수 있도록 합니다.

2.2.3 복합 타입

// 컬렉션
list<T>: [1, 2, 3]
set<T>: {1, 2, 3}
map<K, V>: {"a": 1, "b": 2}

// 구조
entity Person {

2.3 의도 선언

간단한 의도

intent MakeCoffee {
  want: hot_coffee
  constraints {
    temperature: 85..95 celsius
    time: <= 5.minutes
  }
}

입력 및 출력이 있는 의도

intent ProcessData {
  given: raw_data, transformation_rules
  want: processed_data

  constraints {
    accuracy: >= 0.95
    time: <= 1.second
  }

  certainty: >= 0.9

  fallback {
    if certainty < 0.9:
      return partial_results with warning
  }
}

2.4 제약 정의

제약은 솔루션이 작동해야 하는 경계를 정의합니다. 전통적인 어설션보다 더 유연합니다. 

constraints {
  // 시간 제약
  time: <= 5.minutes
  deadline: @2025-12-16T18:00:00Z

  // 자원 제약
  memory: <= 1.gigabyte
  cost: <= 100.dollars
  energy: minimize

  // 품질 제약
  accuracy: >= 0.95
  latency: <= 100.milliseconds

  // 윤리 제약
  privacy: respect
  fairness: ensure
  transparency: required

  // 사용자 정의 제약
  custom("no_profanity"): true
}
제약 유형 목적 예제
시간 시간 및 마감일 제한 time: <= 5.minutes
자원 메모리, 비용, 에너지 경계 memory: <= 1.gigabyte
품질 정확도 및 성능 accuracy: >= 0.95
윤리 프라이버시, 공정성, 투명성 privacy: respect
사용자 정의 도메인별 규칙 custom("rule_name"): true

2.5 확실성 값

확실성은 의도의 결과에 필요한 신뢰 수준을 나타냅니다. 0.0 (완전한 불확실성)과 1.0 (절대적 확실성) 사이의 값입니다. 

intent PredictWeather {
  given: location, date
  want: weather_forecast

  certainty: >= 0.8

  fallback {
    if certainty < 0.8:
      return uncertain_forecast with warning

    if impossible:
      ask_human("날씨 예측 실패. 어떻게 할까요?")
  }
}

확실성 수준 (가이드)

확실성 해석 사용 사례
0.99+ 거의 절대적 확실성 안전 중요 시스템
0.95-0.99 매우 높은 신뢰도 금융 거래
0.80-0.95 높은 신뢰도 의료 진단 지원
0.60-0.80 중간 신뢰도 추천, 예측
<0.60 낮은 신뢰도 탐색적 분석

중요: 불확실성에 대한 정직함

WIA-INTENT는 시스템이 모르는 것에 대해 정직하도록 권장합니다. 낮은 확실성은 실패가 아닙니다—의사 결정을 위한 귀중한 정보입니다.

2.6 의도 구성

순차적 의도

intent MakeCoffee {
  sequence {
    1. GrindBeans { want: ground_coffee }
    2. HeatWater { want: hot_water, temp: 92..96 celsius }
    3. Brew { given: ground_coffee, hot_water; want: coffee }
  }
}

병렬 의도

intent PrepareBreakfast {
  parallel {
    MakeCoffee { want: coffee }
    ToastBread { want: toast }
    FryEggs { want: eggs, style: sunny_side_up }
  }
  then: Serve { given: coffee, toast, eggs }
}

조건부 의도

intent GetToWork {
  given: current_location, work_location, weather, time

  choose {
    when weather.is_rainy and time.is_rush_hour:
      UseSubway { want: arrive_at work_location }

    when weather.is_nice and time.is_flexible:
      WalkOrBike { want: arrive_at work_location, prefer: scenic }

    default:
      UseCar { want: arrive_at work_location }
  }
}
