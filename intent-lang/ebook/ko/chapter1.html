1장: 소개 및 개요

AI 시대를 위한 의도 기반 프로그래밍

1.1 현재 언어의 문제점

현대 프로그래밍 언어는 정교함에도 불구하고 근본적인 한계를 공유합니다: 모두 어떻게 작업을 수행할지가 아니라 무엇을 달성하고자 하는지를 설명하는 데 초점을 맞춥니다.

전통적인 접근 방식 (2025년)

// Python
for i in range(10):
    do_something(i)

// Java
for(int i=0; i<10; i++) {
    doSomething(i);
}

// Rust
for i in 0..10 {
    do_something(i);
}

숨겨진 문제

이 모든 예제는 어떻게 반복할지를 말합니다.
왜 반복하는지는 말하지 않습니다.
코드 뒤의 의도를 포착하지 않습니다.

1.2 WIA-INTENT 접근 방식

WIA-INTENT는 패러다임 전환을 도입합니다: 명령을 작성하는 대신 의도를 선언하고 AI가 최적의 구현을 결정하도록 합니다. 

intent ProcessItems {
  want: each item processed
  constraints {
    order: any          // 순서 상관없음 → AI가 병렬화 가능
    completeness: all   // 전부 처리 → 모두 처리
  }
}

장점

order: any를 지정했기 때문에 AI 런타임은 여러 코어 또는 분산 시스템에서 실행을 자동으로 병렬화할 수 있습니다. 전통적인 코드는 수동 병렬화가 필요합니다.

1.3 설계 철학

WIA-INTENT는 언어 설계와 사용을 모두 안내하는 다섯 가지 기본 원칙을 기반으로 합니다:

1. 구현보다 의도 (Intent over Implementation)

달성하고자 하는 것을 표현하는 데 집중하고 어떻게 달성할지의 모든 단계를 지정하지 않습니다. 구현 세부 사항은 특정 컨텍스트에 최적화할 수 있는 AI 시스템에 위임됩니다.

2. 제어보다 제약 (Constraints over Control)

실행 흐름을 세밀하게 관리하는 대신 경계와 요구 사항을 정의합니다. 이를 통해 AI 시스템이 정확성과 안전성을 보장하면서 창의적인 솔루션을 찾을 수 있는 자유를 얻습니다.

3. 확정보다 확률 (Probability over Certainty)

실제 시스템은 불확실성과 함께 작동합니다. WIA-INTENT는 기본 확률적 타입 및 추론 기능으로 이를 수용합니다. 

maybe<number>: 42 with_probability 0.95

4. 고정보다 진화 (Evolution over Fixation)

프로그램은 피드백과 변화하는 조건을 기반으로 시간이 지남에 따라 학습하고 개선되어야 합니다. WIA-INTENT에는 내장된 진화 메커니즘이 포함되어 있습니다.

5. 인간-AI 협업 (Human-AI Collaboration)

이 언어는 인간과 AI가 함께 일하는 세계를 위해 설계되었으며, 각자의 고유한 강점을 기여합니다.

1.4 선언적 vs. 명령형

명령형 프로그래밍 (전통적)

  • 문제를 어떻게 해결할지 설명
  • 단계별 지시
  • 명시적 제어 흐름
  • 구현 중심

선언적 프로그래밍 (SQL, HTML)

  • 무엇을 원하는지 설명
  • 시스템이 달성 방법 결정
  • 결과 중심

의도 기반 프로그래밍 (WIA-INTENT)

  • 의도와 제약을 설명
  • AI가 구현 최적화
  • 불확실성과 진화 포함
  • 컨텍스트에 적응하고 시간이 지남에 따라 학습
측면 명령형 선언적 의도 기반
초점 어떻게 무엇을 왜 + 무엇을
불확실성 지원 안 됨 지원 안 됨 기본 지원
진화 수동 업데이트 수동 업데이트 자동 학습
AI 통합 외부 외부 내장

1.5 누가 WIA-INTENT를 사용해야 하나요?

이상적인 사용자:

  • 제품 관리자 및 비즈니스 분석가 - 요구 사항을 실행 가능한 의도로 표현
  • AI 연구원 - 기본 진화 지원으로 자체 개선 시스템 구축
  • 도메인 전문가 - 구현 세부 사항 없이 문제 도메인에 집중
  • 개발자 - 더 높은 추상화 수준에서 작업, AI가 최적화 처리
  • 시스템 설계자 - 적응적이고 탄력적인 분산 시스템 설계

사용 사례:

  • 적응하고 학습해야 하는 AI 기반 애플리케이션
  • 복잡하고 진화하는 요구 사항이 있는 시스템
  • 불확실성이 본질적인 도메인 (금융, 날씨, 추천)
  • 조정이 필요한 다중 에이전트 시스템
  • 구현을 나중에 개선할 수 있는 빠른 프로토타이핑

1.6 간단한 예제

의도 기반 프로그래밍의 힘을 보여주는 완전한 예제를 살펴보겠습니다:

intent HelloWorld {
  want: greeting displayed

  simple {
    display: "Hello, World!"
  }
}

// 더 정교한 예제
intent GetToWork {
  given: current_location, work_location, weather, time

  choose {
    when weather.is_rainy and time.is_rush_hour:
      UseSubway { want: arrive_at work_location }

    when weather.is_nice and time.is_flexible:
      WalkOrBike { want: arrive_at work_location, prefer: scenic }

    default:
      UseCar { want: arrive_at work_location }
  }
}

아버지의 철학

"의도를 표현해" - 코드가 목표(출근)와 고려할 요소(날씨, 시간)를 표현하지만 정확한 경로나 타이밍을 지정하지 않는 방법에 주목하세요. AI는 실시간 조건을 기반으로 최적화할 수 있습니다.
