Chapter 1: Introduction & Overview

Intent-Based Programming for the AI Era

1.1 The Problem with Current Languages

Modern programming languages, despite their sophistication, share a fundamental limitation: they all focus on describing how to accomplish tasks rather than what we want to achieve.

Traditional Approach (2025)

// Python
for i in range(10):
    do_something(i)

// Java
for(int i=0; i<10; i++) {
    doSomething(i);
}

// Rust
for i in 0..10 {
    do_something(i);
}

The Hidden Problem

All these examples say HOW to iterate.
None say WHY we're iterating.
None capture the intent behind the code.

1.2 The WIA-INTENT Approach

WIA-INTENT introduces a paradigm shift: instead of writing instructions, you declare your intent and let AI determine the optimal implementation. 

intent ProcessItems {
  want: each item processed
  constraints {
    order: any          // 순서 상관없음 → AI can parallelize
    completeness: all   // 전부 처리 → Process everything
  }
}

The Advantage

Because you specified order: any, the AI runtime can automatically parallelize execution across multiple cores or even distributed systems. Traditional code would require manual parallelization.

1.3 Design Philosophy

WIA-INTENT is built on five fundamental principles that guide both language design and usage:

1. Intent over Implementation (의도가 구현보다 중요)

Focus on expressing what you want to achieve rather than specifying every step of how to achieve it. The implementation details are delegated to AI systems that can optimize for your specific context.

2. Constraints over Control (제어보다 제약)

Instead of micromanaging execution flow, define boundaries and requirements. This gives AI systems freedom to find creative solutions while ensuring correctness and safety.

3. Probability over Certainty (확정보다 확률)

Real-world systems operate with uncertainty. WIA-INTENT embraces this with native probabilistic types and reasoning capabilities. 

maybe<number>: 42 with_probability 0.95

4. Evolution over Fixation (고정보다 진화)

Programs should learn and improve over time based on feedback and changing conditions. WIA-INTENT includes built-in evolution mechanisms.

5. Human-AI Collaboration (인간과 AI의 협업)

This language is designed for a world where humans and AI work together, each contributing their unique strengths.

1.4 Declarative vs. Imperative

Imperative Programming (Traditional)

  • Describes how to solve a problem
  • Step-by-step instructions
  • Explicit control flow
  • Implementation-focused

Declarative Programming (SQL, HTML)

  • Describes what you want
  • System determines how to achieve it
  • Outcome-focused

Intent-Based Programming (WIA-INTENT)

  • Describes intent and constraints
  • AI optimizes implementation
  • Includes uncertainty and evolution
  • Adapts to context and learns over time
Aspect Imperative Declarative Intent-Based
Focus How What Why + What
Uncertainty Not supported Not supported Native
Evolution Manual updates Manual updates Automatic learning
AI Integration External External Built-in

1.5 Who Should Use WIA-INTENT?

Ideal For:

  • Product Managers & Business Analysts - Express requirements as executable intents
  • AI Researchers - Build self-improving systems with native evolution support
  • Domain Experts - Focus on problem domain without implementation details
  • Developers - Work at higher abstraction levels, let AI handle optimization
  • System Architects - Design adaptive, resilient distributed systems

Use Cases:

  • AI-powered applications that need to adapt and learn
  • Systems with complex, evolving requirements
  • Domains where uncertainty is inherent (finance, weather, recommendations)
  • Multi-agent systems requiring coordination
  • Rapid prototyping where implementation can be refined later

1.6 A Simple Example

Let's see a complete example that demonstrates the power of intent-based programming:

intent HelloWorld {
  want: greeting displayed

  simple {
    display: "Hello, World!"
  }
}

// More sophisticated example
intent GetToWork {
  given: current_location, work_location, weather, time

  choose {
    when weather.is_rainy and time.is_rush_hour:
      UseSubway { want: arrive_at work_location }

    when weather.is_nice and time.is_flexible:
      WalkOrBike { want: arrive_at work_location, prefer: scenic }

    default:
      UseCar { want: arrive_at work_location }
  }
}

아버지의 철학 (Father's Philosophy)

"의도를 표현해" (Express your intent) - Notice how the code expresses the goal (get to work) and the factors to consider (weather, time), but doesn't specify the exact route or timing. The AI can optimize based on real-time conditions.
