Chapter 2: Data Schema & Type System

Intent Declarations, Constraints, and Certainty Values

2.1 Basic Syntax

Every WIA-INTENT program is built from intent declarations. Here's the fundamental structure: 

// Intent declaration
intent <IntentName> {
  // What is given (inputs)
  given: <inputs>

  // What is wanted (outputs/goals)
  want: <desired_outcome>

  // Constraints on the solution
  constraints {
    <constraint_name>: <constraint_value>
  }

  // Confidence/certainty requirements
  certainty: <0.0 - 1.0>

  // What to do if intent cannot be fulfilled
  fallback: <fallback_action>

  // How the solution can evolve
  evolve: <evolution_rules>
}

Key Components

  • given: Input data and context
  • want: Desired outcome or goal
  • constraints: Requirements and boundaries
  • certainty: Required confidence level (0.0-1.0)
  • fallback: Behavior when intent cannot be fulfilled
  • evolve: Rules for learning and improvement

2.2 Data Types

2.2.1 Primitive Types

WIA-INTENT supports standard primitives plus unique probabilistic types:

// Scalar types
number: 42, 3.14, -17
text: "hello world"
truth: true, false, maybe(0.7)  // 확률적 진리값!
nothing: void

// Temporal types
moment: @2025-12-16T10:30:00Z
duration: 5.minutes, 2.hours, 1.day

2.2.2 Probabilistic Types (확률적 타입)

A unique feature of WIA-INTENT: native support for uncertainty and probability distributions built into the type system. 

// 불확실성을 언어 자체에서 표현
probability: 0.85
distribution: normal(mean: 100, std: 15)
range: 10..20 with_confidence 0.9

// Maybe type (확률적)
maybe<number>: 42 with_probability 0.95

Why Probabilistic Types Matter

Real-world data is uncertain. Weather predictions, user behavior, market trends— all involve probability. WIA-INTENT makes uncertainty a first-class concept, allowing programs to reason about confidence levels explicitly.

2.2.3 Composite Types

// Collections
list<T>: [1, 2, 3]

map<K, V>: {"a": 1, "b": 2}

// Structures

2.3 Intent Declarations

Simple Intent

intent MakeCoffee {
  want: hot_coffee
  constraints {
    temperature: 85..95 celsius
    time: <= 5.minutes
  }
}

Intent with Inputs and Outputs

intent ProcessData {
  given: raw_data, transformation_rules
  want: processed_data

  constraints {
    accuracy: >= 0.95
    time: <= 1.second
  }

  certainty: >= 0.9

  fallback {
    if certainty < 0.9:
      return partial_results with warning
  }
}

2.4 Constraint Definitions

Constraints define the boundaries within which solutions must operate. They're more flexible than traditional assertions. 

constraints {
  // 시간 제약 (Time constraints)
  time: <= 5.minutes
  deadline: @2025-12-16T18:00:00Z

  // 자원 제약 (Resource constraints)
  memory: <= 1.gigabyte
  cost: <= 100.dollars
  energy: minimize

  // 품질 제약 (Quality constraints)
  accuracy: >= 0.95
  latency: <= 100.milliseconds

  // 윤리 제약 (Ethics constraints)
  privacy: respect
  fairness: ensure
  transparency: required

  // 사용자 정의 제약 (Custom constraints)
  custom("no_profanity"): true
}
Constraint Type Purpose Example
Temporal Time and deadline limits time: <= 5.minutes
Resource Memory, cost, energy bounds memory: <= 1.gigabyte
Quality Accuracy and performance accuracy: >= 0.95
Ethical Privacy, fairness, transparency privacy: respect
Custom Domain-specific rules custom("rule_name"): true

2.5 Certainty Values (확실성)

Certainty represents the confidence level required for an intent's outcome. It's a value between 0.0 (complete uncertainty) and 1.0 (absolute certainty). 

intent PredictWeather {
  given: location, date
  want: weather_forecast

  certainty: >= 0.8

  fallback {
    if certainty < 0.8:
      return uncertain_forecast with warning

    if impossible:
      ask_human("날씨 예측 실패. 어떻게 할까요?")
  }
}

Certainty Levels (Guide)

Certainty Interpretation Use Case
0.99+ Near-absolute certainty Safety-critical systems
0.95-0.99 Very high confidence Financial transactions
0.80-0.95 High confidence Medical diagnosis support
0.60-0.80 Moderate confidence Recommendations, predictions
<0.60 Low confidence Exploratory analysis

Important: Honesty About Uncertainty

WIA-INTENT encourages systems to be honest about what they don't know. Low certainty is not a failure—it's valuable information for decision-making.

2.6 Intent Composition

Sequential Intents

intent MakeCoffee {
  sequence {
    1. GrindBeans { want: ground_coffee }
    2. HeatWater { want: hot_water, temp: 92..96 celsius }
    3. Brew { given: ground_coffee, hot_water; want: coffee }
  }
}

Parallel Intents

intent PrepareBreakfast {
  parallel {
    MakeCoffee { want: coffee }
    ToastBread { want: toast }
    FryEggs { want: eggs, style: sunny_side_up }
  }
  then: Serve { given: coffee, toast, eggs }
}

Conditional Intents

intent GetToWork {
  given: current_location, work_location, weather, time

  choose {
    when weather.is_rainy and time.is_rush_hour:
      UseSubway { want: arrive_at work_location }

    when weather.is_nice and time.is_flexible:
      WalkOrBike { want: arrive_at work_location, prefer: scenic }

    default:
      UseCar { want: arrive_at work_location }
  }
}

2.7 Entity Definitions

Define structured data types for your domain:

entity User {
  id: uuid
  name: text
  email: text
  preferences: map<text, any>
  satisfaction: distribution  // Probabilistic!
}

entity Product {
  id: uuid
  name: text
  price: number
  availability: maybe<number>  // Might be uncertain
  reviews: list<Review>
}

entity Review {
  rating: 1..5
  sentiment: distribution
  text: text
  helpful_votes: number
}
