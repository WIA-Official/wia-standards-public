Chapter 08

WIA-SLR-001 Standard Specification

Complete technical specification, APIs, data formats, and certification requirements

Standard Overview

The WIA-SLR-001 standard provides a comprehensive framework for sea level rise monitoring, impact assessment, adaptation planning, and implementation. This chapter presents the complete technical specification, including data format requirements, API interfaces, compliance criteria, and certification processes.

The standard is designed to be implementable across diverse contexts, from national meteorological services to local planning agencies, while maintaining the consistency and quality necessary for scientific credibility and operational reliability. It emphasizes interoperability, transparency, and alignment with existing international standards and scientific best practices.

Standard Philosophy: 弘益人間 (Hongik Ingan)

The WIA-SLR-001 standard is grounded in the philosophy of 弘益人間 - "Benefit All Humanity." This principle requires that the standard be accessible to all, prioritize protection of the most vulnerable, and support equitable adaptation worldwide. Technical requirements are designed to be achievable by developing countries while maintaining quality standards necessary for effective action.

Normative Requirements

The following sections specify mandatory (SHALL), recommended (SHOULD), and optional (MAY) requirements for WIA-SLR-001 compliance. Organizations seeking certification must demonstrate compliance with all mandatory requirements and a specified proportion of recommended requirements based on certification level.

SLR-001-M01 REQUIRED

Sea Level Data Reference Frame

All sea level measurements and projections SHALL be expressed relative to the International Terrestrial Reference Frame (ITRF) or a clearly documented local reference with transformation parameters to ITRF. The reference frame version and epoch SHALL be explicitly documented.

Requirements
  • Reference frame SHALL be ITRF2014 or later version
  • Epoch date SHALL be explicitly stated (e.g., 2020.0)
  • Transformation parameters to ITRF SHALL be provided if local reference used
  • Vertical land motion corrections SHALL be documented and traceable
SLR-001-M02 REQUIRED

Projection Scenario Documentation

Sea level projections SHALL include explicit documentation of emission scenarios, probability levels, and the scientific basis for projections. Multiple scenarios spanning the range of plausible outcomes SHALL be provided.

Requirements
  • Projections SHALL reference IPCC SSP scenarios or equivalent
  • Probability/confidence levels SHALL be explicitly stated
  • At minimum, low (SSP1-2.6), medium (SSP2-4.5), and high (SSP5-8.5) scenarios required
  • High-end/low-likelihood scenarios SHOULD be included for risk-averse planning
SLR-001-M03 REQUIRED

Uncertainty Quantification

All sea level data products SHALL include quantified uncertainty estimates. Uncertainty SHALL be expressed as standard uncertainty or confidence intervals at specified probability levels.

Requirements
  • Measurement uncertainty SHALL be traceable to national/international standards
  • Projection uncertainty SHALL include contributions from emission scenarios, climate models, and ice sheet processes
  • Confidence intervals SHALL be provided at 5-95% or 17-83% levels
  • Uncertainty methodology SHALL be documented following GUM principles
SLR-001-R01 RECOMMENDED

Real-Time Data Availability

Tide gauge and monitoring station data SHOULD be made available in near-real-time (within 1 hour of observation) through standard data services. Historical data SHOULD be available through documented APIs.

Recommendations
  • Data SHOULD be available via OGC-compliant services (WMS, WFS, SOS)
  • RESTful API SHOULD be provided for programmatic access
  • Data format SHOULD follow CF conventions for NetCDF files
  • Data quality flags SHOULD be included with all observations

Data Format Specifications

The WIA-SLR-001 standard specifies data formats for interoperable exchange of sea level information. The primary format is JSON for API responses and configuration, with NetCDF recommended for gridded data products.

Sea Level Observation Data Format

// WIA-SLR-001 Sea Level Observation Format { "@context": "https://wia.org/slr/context/v1", "@type": "SeaLevelObservation", "id": "urn:wia:slr:obs:2024:12345", "station": { "id": "GLOSS-001", "name": "Honolulu, Hawaii", "location": { "latitude": 21.3069, "longitude": -157.8583, "elevation": 0.0 }, "referenceFrame": "ITRF2020" }, "timestamp": "2024-06-15T12:00:00Z", "seaLevel": { "value": 0.523, "unit": "m", "datum": "MSL1983-2001", "uncertainty": 0.005, "qualityFlag": "good" }, "meteorological": { "pressure": 1013.25, "windSpeed": 5.2, "windDirection": 270 } }

API Specification

The WIA-SLR-001 standard defines a RESTful API for accessing sea level data and services. Implementations SHOULD provide this API for programmatic access to observations, projections, and assessment tools.

Core API Endpoints

GET /api/v1/observations/{station_id}
Retrieve sea level observations for a specific station. Supports query parameters for time range, averaging period, and data quality filtering.
GET /api/v1/projections/{location}
Retrieve sea level projections for a location. Returns projections for all scenarios and time horizons. Location specified as lat,lon or place name.
GET /api/v1/stations
List available monitoring stations with metadata including location, data availability, and quality metrics.
POST /api/v1/assessment/vulnerability
Submit data for vulnerability assessment. Returns standardized vulnerability scores and risk ratings based on provided exposure and sensitivity data.
GET /api/v1/extremes/{station_id}/return-levels
Retrieve extreme sea level return level estimates incorporating sea level rise projections. Returns current and future return levels for specified return periods.
Parameter Type Description
start_date ISO 8601 date Start of query time range
end_date ISO 8601 date End of query time range
scenario string SSP scenario (SSP1-1.9, SSP1-2.6, SSP2-4.5, SSP3-7.0, SSP5-8.5)
confidence string Confidence level for projections (likely, very_likely, p5_p95)
format string Response format (json, csv, netcdf)

Assessment Framework

The WIA-SLR-001 standard specifies a standardized framework for sea level rise impact and vulnerability assessment. This framework ensures consistency and comparability across assessments while accommodating local context and data availability.

Vulnerability Index Components

Component Weight Indicators
Physical Exposure 30% Elevation, distance to coast, flood frequency, erosion rate
Socioeconomic Sensitivity 25% Population density, poverty rate, age structure, building stock
Infrastructure Criticality 20% Critical facilities, transport networks, utilities
Ecological Sensitivity 15% Ecosystem type, habitat connectivity, biodiversity value
Adaptive Capacity 10% Institutional capacity, financial resources, technical capability

Customization Guidance

While the standard specifies default weights, implementing organizations MAY adjust weights to reflect local priorities and conditions. Any deviation from default weights SHALL be documented with justification. Total weights SHALL sum to 100%.

Certification Levels

The WIA-SLR-001 certification program recognizes organizations that demonstrate compliance with the standard at various levels. Certification is awarded based on independent assessment of compliance with mandatory and recommended requirements.

🥉

Bronze

All mandatory requirements met. Basic compliance demonstrating commitment to standardized practices.

🥈

Silver

All mandatory + 50% recommended requirements. Enhanced capability with strong data quality practices.

🥇

Gold

All mandatory + 80% recommended requirements. Advanced implementation with comprehensive capabilities.

💎

Platinum

Full compliance + innovation contribution. Leadership in advancing sea level rise science and practice.

Certification Process

  • Application: Submit application with self-assessment documentation
  • Documentation Review: WIA assessors review submitted documentation
  • Technical Audit: On-site or remote audit of systems and processes
  • Data Quality Assessment: Evaluation of data products against standard requirements
  • Certification Decision: Award of certification level based on findings
  • Surveillance: Annual surveillance audits to maintain certification
  • Recertification: Full recertification every 3 years

Implementation Guidance

Organizations implementing WIA-SLR-001 should follow a phased approach, beginning with core requirements and progressively expanding capabilities. The following guidance supports successful implementation.

Phase 1: Foundation (Months 1-6)

  • Establish reference frame and datum documentation
  • Implement basic data quality control procedures
  • Document existing data sources and their characteristics
  • Train staff on standard requirements and terminology

Phase 2: Core Implementation (Months 7-12)

  • Implement compliant data formats and APIs
  • Establish uncertainty quantification procedures
  • Deploy projection data with scenario documentation
  • Implement assessment framework components

Phase 3: Enhancement (Months 13-24)

  • Achieve real-time data availability targets
  • Expand coverage and capabilities
  • Implement advanced analytics and services
  • Pursue certification at target level

Support Resources

WIA provides implementation support including template documentation, reference implementations, training materials, and technical assistance. Contact the WIA Secretariat for information on available resources and capacity building programs.

Conclusion

The WIA-SLR-001 standard provides a comprehensive framework for addressing sea level rise through standardized monitoring, assessment, and adaptation practices. By establishing common requirements for data quality, interoperability, and methodology, the standard enables effective international cooperation and evidence-based decision-making.

Implementation of this standard contributes to the global goal of protecting coastal communities and ecosystems from sea level rise impacts. Through certification, organizations demonstrate their commitment to best practices and their contribution to the shared mission of 弘益人間 - benefiting all humanity.

As sea level rise accelerates and our scientific understanding continues to advance, the WIA-SLR-001 standard will be periodically updated to reflect new knowledge and capabilities. Organizations are encouraged to engage with the WIA standards development process to contribute to these ongoing improvements.
