4장: 프로토콜 명세

4.1 의식 전송 프로토콜 (CTP)

의식 전송 프로토콜은 무결성과 진정성을 유지하면서 디지털 의식 데이터가 시스템 간에 안전하게 전송되는 방법을 정의합니다.

중요: 모든 의식 전송은 윤리 준수를 위해 인증, 암호화 및 로깅되어야 합니다.

프로토콜 계층

계층 기능 프로토콜
애플리케이션 의식 데이터 형식 WIA-CDF (의식 데이터 형식)
검증 신원 및 동의 검증 WIA-ICV (신원 동의 검증)
보안 암호화 및 서명 TLS 1.3 + 양자 내성
전송 신뢰성 있는 전달 gRPC / WebSocket

4.2 메시지 형식

Protocol Buffer - CTP 메시지
syntax = "proto3";

package wia.afterlife.ctp;

message ConsciousnessTransfer {
  // 헤더
  string transfer_id = 1;
  string source_system = 2;
  string destination_system = 3;
  int64 timestamp = 4;

  // 검증
  ConsentProof consent_proof = 5;
  IdentityAttestation identity = 6;

  // 페이로드
  ConsciousnessPayload payload = 7;

  // 무결성
  bytes signature = 8;
  string hash_algorithm = 9;
  bytes payload_hash = 10;
}

message ConsentProof {
  string consent_id = 1;
  string consent_type = 2;
  bytes consent_signature = 3;
  repeated string witness_keys = 4;
  int64 expiry_timestamp = 5;
}

enum ConsciousnessLevel {
  STATIC_MEMORIAL = 0;
  INTERACTIVE_BOT = 1;
  DIGITAL_AVATAR = 2;
  CONSCIOUSNESS_TRANSFER = 3;
}

4.3 전송 상태 머신

상태 전이
전송_상태:
  ┌─────────────┐
  │   대기 중    │
  └──────┬──────┘
         │ 동의_확인됨
         ▼
  ┌─────────────┐
  │   확인됨    │
  └──────┬──────┘
         │ 전송_시작됨
         ▼
  ┌─────────────┐
  │   전송 중   │──────┐
  └──────┬──────┘      │ 오류
         │ 완료        ▼
         ▼        ┌─────────┐
  ┌─────────────┐ │  실패   │
  │   검증 중   │ └─────────┘
  └──────┬──────┘
         │ 검증됨
         ▼
  ┌─────────────┐
  │   완료      │
  └─────────────┘

롤백_트리거:
  - consent_revoked → 즉시_종료
  - integrity_failure → 소스로_롤백
  - timeout (30분) → 재시도_또는_실패
