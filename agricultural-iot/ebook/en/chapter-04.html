Chapter 4

Data Analytics

This chapter explores advanced analytics for agricultural IoT data, including yield prediction models, pest and disease detection algorithms, irrigation optimization, and decision support systems. We cover machine learning approaches, real-time processing pipelines, and practical implementation strategies for extracting actionable insights from farm sensor data.

1. Agricultural Data Characteristics

1.1 Time-Series Nature of Farm Data

Agricultural IoT generates massive time-series datasets—continuous streams of measurements from thousands of sensors over multiple growing seasons. Unlike transactional data, farm telemetry exhibits strong temporal patterns: diurnal cycles in temperature and solar radiation, seasonal variations in soil moisture and crop development, and multi-year trends driven by climate change and soil health evolution.

Effective analytics must account for temporal dependencies, seasonal decomposition, and the lag between interventions and observable outcomes. A fertilizer application may take weeks to manifest in crop growth metrics. Irrigation decisions made today affect soil moisture tomorrow but influence yield months later. This temporal complexity requires sophisticated modeling approaches beyond simple correlation analysis.

1.2 Spatial Heterogeneity

Agricultural fields exhibit significant spatial variability in soil properties, topography, drainage, and microclimate. Analytics must consider geospatial relationships—nearby sensors are correlated, but this correlation decays with distance. Geostatistical techniques like kriging interpolate between sparse sensor locations to create continuous field maps. Spatial clustering identifies management zones with similar characteristics.

// TypeScript: Time-Series Data Processing Pipeline interface SensorTimeSeries { sensorId: string; location: GeoCoordinates; measurements: Measurement[]; samplingInterval: number; // seconds } interface Measurement { timestamp: Date; value: number; quality: 'good' | 'questionable' | 'bad'; } class TimeSeriesAnalyzer { async processRawData( raw: SensorTimeSeries ): Promise { // Step 1: Quality control - remove outliers, flag suspicious readings const filtered = await this.qualityControl(raw.measurements); // Step 2: Gap filling - interpolate missing values const complete = await this.fillGaps(filtered, raw.samplingInterval); // Step 3: Smoothing - reduce noise while preserving trends const smoothed = await this.applyMovingAverage(complete, { window: 6 }); // Step 4: Decomposition - separate trend, seasonal, residual const decomposed = await this.seasonalDecomposition(smoothed); return { sensorId: raw.sensorId, location: raw.location, original: raw.measurements, processed: smoothed, trend: decomposed.trend, seasonal: decomposed.seasonal, residual: decomposed.residual }; } private async qualityControl( measurements: Measurement[] ): Promise { const mean = measurements.reduce((sum, m) => sum + m.value, 0) / measurements.length; const stdDev = Math.sqrt( measurements.reduce((sum, m) => sum + Math.pow(m.value - mean, 2), 0) / measurements.length ); // Flag outliers beyond 3 standard deviations return measurements.map(m => ({ ...m, quality: Math.abs(m.value - mean) > 3 * stdDev ? 'questionable' : m.quality })).filter(m => m.quality !== 'bad'); } private async fillGaps( measurements: Measurement[], expectedInterval: number ): Promise { const complete: Measurement[] = []; for (let i = 0; i < measurements.length - 1; i++) { complete.push(measurements[i]); const gap = measurements[i + 1].timestamp.getTime() - measurements[i].timestamp.getTime(); if (gap > expectedInterval * 1000 * 1.5) { // Gap detected - linear interpolation const steps = Math.floor(gap / (expectedInterval * 1000)); for (let j = 1; j < steps; j++) { const interpolated: Measurement = { timestamp: new Date( measurements[i].timestamp.getTime() + j * expectedInterval * 1000 ), value: measurements[i].value + (measurements[i + 1].value - measurements[i].value) * (j / steps), quality: 'questionable' }; complete.push(interpolated); } } } complete.push(measurements[measurements.length - 1]); return complete; } private async seasonalDecomposition( data: Measurement[] ): Promise { const period = 24; // Daily seasonality for hourly data // Calculate seasonal component using moving average const seasonal = this.extractSeasonalPattern(data, period); // Detrend to get trend component const trend = this.extractTrend(data, period); // Residual = Original - Trend - Seasonal const residual = data.map((m, i) => ({ timestamp: m.timestamp, value: m.value - trend[i].value - seasonal[i % period] })); return { trend, seasonal, residual }; } async detectAnomalies( series: ProcessedTimeSeries ): Promise { const anomalies: Anomaly[] = []; // Statistical anomaly detection on residuals const residuals = series.residual.map(r => r.value); const mean = residuals.reduce((a, b) => a + b, 0) / residuals.length; const stdDev = Math.sqrt( residuals.reduce((sum, r) => sum + Math.pow(r - mean, 2), 0) / residuals.length ); series.residual.forEach((r, i) => { const zScore = Math.abs((r.value - mean) / stdDev); if (zScore > 3) { anomalies.push({ timestamp: r.timestamp, severity: zScore > 4 ? 'high' : 'medium', description: `Unexpected ${series.sensorId} reading: ${series.processed[i].value}`, possibleCauses: this.identifyCauses(series.sensorId, r.value) }); } }); return anomalies; } }

2. Yield Prediction Models

2.1 Crop Growth Modeling

Yield prediction integrates multiple data streams—weather, soil conditions, crop development indicators, and historical yield records—to forecast harvest outcomes weeks or months in advance. Accurate predictions enable proactive decision-making: adjusting irrigation, applying supplemental nutrition, or arranging harvest logistics and storage capacity.

Two main approaches dominate yield prediction: process-based models that simulate crop physiology and growth dynamics, and data-driven machine learning models that identify patterns in historical data. Process-based models like DSSAT and APSIM incorporate scientific understanding of photosynthesis, transpiration, and nutrient uptake. They generalize well but require extensive parameterization. Machine learning approaches (random forests, neural networks, gradient boosting) excel when abundant historical data is available but may fail under novel conditions outside training distribution.

// TypeScript: Yield Prediction System interface CropGrowthData { fieldId: string; cropType: string; plantingDate: Date; weatherHistory: WeatherObservation[]; soilData: SoilProfile; ndviTimeSeries: { date: Date; ndvi: number }[]; managementPractices: ManagementEvent[]; } interface YieldPrediction { fieldId: string; predictedYield: number; // kg/hectare confidence: { lower: number; upper: number; interval: number; // percentage (e.g., 95%) }; keyFactors: FactorInfluence[]; predictionDate: Date; harvestDate: Date; } class YieldPredictionModel { private model: MachineLearningModel; private historicalData: HistoricalYield[]; async predictYield( data: CropGrowthData ): Promise { // Extract features from multi-source data const features = await this.extractFeatures(data); // Generate prediction using ensemble model const prediction = await this.model.predict(features); // Calculate confidence intervals using quantile regression const confidence = await this.estimateConfidence(features); // SHAP analysis for feature importance const influences = await this.analyzeFeatureImportance(features); return { fieldId: data.fieldId, predictedYield: prediction.value, confidence: { lower: prediction.value - confidence.lowerBound, upper: prediction.value + confidence.upperBound, interval: 95 }, keyFactors: influences, predictionDate: new Date(), harvestDate: this.estimateHarvestDate(data) }; } private async extractFeatures( data: CropGrowthData ): Promise { const daysAfterPlanting = Math.floor( (Date.now() - data.plantingDate.getTime()) / (1000 * 60 * 60 * 24) ); // Weather-based features const gdd = this.calculateGDD(data.weatherHistory, data.cropType); const cumulativeRainfall = data.weatherHistory.reduce( (sum, w) => sum + w.rainfall, 0 ); const avgTemp = data.weatherHistory.reduce( (sum, w) => sum + w.temperature, 0 ) / data.weatherHistory.length; // NDVI-based features const currentNDVI = data.ndviTimeSeries[data.ndviTimeSeries.length - 1]?.ndvi || 0; const peakNDVI = Math.max(...data.ndviTimeSeries.map(n => n.ndvi)); const ndviTrend = this.calculateNDVITrend(data.ndviTimeSeries); // Soil features const soilWaterCapacity = this.calculateWHC(data.soilData); const soilFertility = this.assessSoilFertility(data.soilData); return { daysAfterPlanting, gdd, cumulativeRainfall, avgTemp, currentNDVI, peakNDVI, ndviTrend, soilWaterCapacity, soilFertility, irrigationCount: data.managementPractices.filter( p => p.type === 'irrigation' ).length, fertilizerApplications: data.managementPractices.filter( p => p.type === 'fertilization' ).length }; } private calculateGDD( weather: WeatherObservation[], crop: string ): number { // Growing Degree Days calculation const baseTemp = this.getBaseTemperature(crop); const maxTemp = this.getMaxTemperature(crop); return weather.reduce((gdd, day) => { const tmax = Math.min(day.temperature + 5, maxTemp); // Approximate daily max const tmin = Math.max(day.temperature - 5, baseTemp); // Approximate daily min const tavg = (tmax + tmin) / 2; const dailyGDD = Math.max(0, Math.min(tavg - baseTemp, maxTemp - baseTemp)); return gdd + dailyGDD; }, 0); } private async analyzeFeatureImportance( features: FeatureVector ): Promise { // SHAP (SHapley Additive exPlanations) values const shapValues = await this.model.explainPrediction(features); return Object.entries(shapValues) .map(([feature, impact]) => ({ factor: this.humanizeFeatureName(feature), influence: impact as number, direction: (impact as number) > 0 ? 'positive' : 'negative', description: this.getFactorDescription(feature, features[feature]) })) .sort((a, b) => Math.abs(b.influence) - Math.abs(a.influence)) .slice(0, 5); // Top 5 factors } async trainModel(historicalData: HistoricalYield[]): Promise { // Prepare training data const X: FeatureVector[] = []; const y: number[] = []; for (const record of historicalData) { const features = await this.extractFeatures(record.growthData); X.push(features); y.push(record.actualYield); } // Train ensemble model (Random Forest + Gradient Boosting) const rfModel = await this.trainRandomForest(X, y); const gbModel = await this.trainGradientBoosting(X, y); // Combine predictions with weighted average this.model = { predict: async (features: FeatureVector) => { const rf = await rfModel.predict(features); const gb = await gbModel.predict(features); return { value: 0.6 * rf + 0.4 * gb }; }, explainPrediction: async (features: FeatureVector) => { return await rfModel.featureImportance(features); } }; } }

3. Pest and Disease Detection

3.1 Early Warning Systems

Timely detection of pest infestations and plant diseases is critical for minimizing crop damage. Traditional scouting methods are labor-intensive and often detect problems after significant damage has occurred. IoT-based early warning systems integrate environmental monitoring, image analysis, and predictive models to identify risks before visual symptoms appear.

Disease development depends on environmental conditions—temperature, humidity, leaf wetness—that favor pathogen growth. By continuously monitoring these parameters and comparing them against disease models, systems can alert farmers when conditions become conducive to specific diseases, enabling preventive fungicide application before infection occurs.

// TypeScript: Pest and Disease Detection System interface DiseasePredictionModel { disease: string; crop: string; favorableConditions: EnvironmentalConditions; incubationPeriod: number; // days } interface EnvironmentalConditions { temperatureRange: { min: number; max: number }; humidityMin: number; leafWetnessHours: number; rainfall?: number; } class PestDiseaseEarlyWarning { private models: Map; private imageClassifier: DeepLearningModel; async assessDiseaseRisk( fieldId: string, crop: string ): Promise { // Get relevant disease models for this crop const relevantModels = Array.from(this.models.values()).filter( m => m.crop === crop ); const risks: DiseaseRisk[] = []; for (const model of relevantModels) { const risk = await this.evaluateModel(fieldId, model); if (risk.level !== 'none') { risks.push(risk); } } return { fieldId, assessmentDate: new Date(), risks: risks.sort((a, b) => this.riskScore(b) - this.riskScore(a)), recommendations: this.generateRecommendations(risks) }; } private async evaluateModel( fieldId: string, model: DiseasePredictionModel ): Promise { // Get recent weather data const weather = await this.getWeatherHistory(fieldId, { days: 7 }); // Check environmental conditions const conditions = this.analyzeConditions(weather, model.favorableConditions); // Calculate risk score let score = 0; let metConditions = 0; if (conditions.temperatureMatch) { score += 30; metConditions++; } if (conditions.humidityMatch) { score += 25; metConditions++; } if (conditions.leafWetnessMatch) { score += 35; metConditions++; } if (conditions.rainfallMatch) { score += 10; metConditions++; } const level = score >= 80 ? 'high' : score >= 50 ? 'medium' : score >= 25 ? 'low' : 'none'; return { disease: model.disease, level, score, conditionsMet: metConditions, totalConditions: 4, projectedOnset: this.projectOnset(model, conditions), confidence: this.calculateConfidence(conditions) }; } async analyzeFieldImages( images: FieldImage[] ): Promise { const results: ImageAnalysisResult[] = []; for (const image of images) { // Preprocess image const processed = await this.preprocessImage(image); // Run through CNN classifier const predictions = await this.imageClassifier.classify(processed); // Detect diseases/pests const detections = predictions.filter(p => p.confidence > 0.7); if (detections.length > 0) { results.push({ imageId: image.id, location: image.gps, timestamp: image.timestamp, detections: detections.map(d => ({ class: d.class, confidence: d.confidence, boundingBox: d.bbox, severity: this.assessSeverity(d), recommendation: this.getDetectionRecommendation(d.class) })) }); } } return results; } private async preprocessImage(image: FieldImage): Promise { // Resize to model input size const resized = await this.resize(image.data, { width: 224, height: 224 }); // Normalize pixel values const normalized = this.normalize(resized, { mean: [0.485, 0.456, 0.406], std: [0.229, 0.224, 0.225] }); // Convert to tensor return this.toTensor(normalized); } async detectPestTraps( trapData: PestTrapReading[] ): Promise { // Analyze trap counts over time const timeline = trapData.sort((a, b) => a.timestamp.getTime() - b.timestamp.getTime() ); const trend = this.calculateTrend(timeline.map(t => t.count)); // Economic threshold analysis const threshold = this.getEconomicThreshold(trapData[0].pest); const currentPressure = timeline[timeline.length - 1].count; return { pest: trapData[0].pest, currentCount: currentPressure, economicThreshold: threshold, status: currentPressure >= threshold ? 'action_required' : currentPressure >= threshold * 0.7 ? 'monitor_closely' : 'acceptable', trend: trend > 0 ? 'increasing' : trend < 0 ? 'decreasing' : 'stable', projectedThresholdDate: this.projectThresholdCrossing( timeline, threshold, trend ), recommendation: currentPressure >= threshold ? 'Apply insecticide within 48 hours' : 'Continue monitoring trap counts daily' }; } }

4. Irrigation Optimization

4.1 Smart Irrigation Algorithms

Irrigation optimization balances crop water requirements, soil water availability, weather forecasts, and water costs to minimize input while maintaining optimal crop development. Advanced systems integrate real-time soil moisture data, evapotranspiration calculations, weather predictions, and crop growth models to prescribe precise irrigation schedules.

Model predictive control (MPC) approaches optimize irrigation over a forecast horizon, considering predicted rainfall, temperature, and crop water demand. If substantial rain is forecasted within 48 hours, the system may defer irrigation even if soil moisture is slightly below optimal. Conversely, anticipating a hot, dry spell, the system may irrigate preemptively to fill soil water reserves.

Analytics Pipeline Architecture

Modern agricultural analytics employ edge-cloud hybrid architectures. Edge devices (field gateways) perform real-time processing for time-critical decisions (irrigation activation, pest alerts). Cloud platforms handle computationally intensive tasks (yield prediction, seasonal trend analysis) and store historical data. This architecture balances latency, bandwidth, and computational requirements while ensuring system reliability even during connectivity outages.

4.2 Water Balance Modeling

Soil water balance tracks inputs (rainfall, irrigation) and outputs (evapotranspiration, drainage) to maintain optimal soil moisture. The fundamental equation: ΔS = P + I - ET - D - R, where ΔS is change in soil moisture storage, P is precipitation, I is irrigation, ET is evapotranspiration, D is drainage, and R is runoff. Continuous monitoring of each component enables precise water management.

// TypeScript: Smart Irrigation Optimizer interface IrrigationZone { zoneId: string; area: number; // hectares crop: string; soilType: string; sensors: string[]; // sensor IDs irrigationCapacity: number; // mm/hour } interface IrrigationDecision { zoneId: string; action: 'irrigate' | 'defer' | 'none'; duration: number; // minutes timing: Date; volume: number; // cubic meters rationale: string; expectedSoilMoistureIncrease: number; // percentage } class SmartIrrigationOptimizer { async optimizeIrrigation( zone: IrrigationZone, forecast: WeatherForecast ): Promise { // Get current soil moisture const soilMoisture = await this.getCurrentMoisture(zone); // Calculate crop water requirement const cropWaterNeed = await this.calculateCropWaterRequirement(zone); // Estimate evapotranspiration over forecast period const forecastET = await this.forecastEvapotranspiration( zone, forecast ); // Check rainfall forecast const forecastRainfall = forecast.precipitation.reduce( (sum, p) => sum + p.amount, 0 ); // Calculate water deficit/surplus const currentDeficit = cropWaterNeed.optimal - soilMoisture.current; const futureDeficit = currentDeficit + forecastET - forecastRainfall; // Decision logic if (currentDeficit > cropWaterNeed.critical) { // Immediate irrigation needed return this.planImmediateIrrigation(zone, currentDeficit); } else if (futureDeficit > cropWaterNeed.critical && forecastRainfall < 5) { // Preemptive irrigation before stress develops return this.planPreemptiveIrrigation(zone, futureDeficit, forecast); } else if (forecastRainfall > 15) { // Defer irrigation, rain expected return { zoneId: zone.zoneId, action: 'defer', duration: 0, timing: new Date(), volume: 0, rationale: `Forecast rainfall ${forecastRainfall}mm sufficient to meet crop needs`, expectedSoilMoistureIncrease: forecastRainfall * 0.8 // 80% infiltration }; } else { // No irrigation needed return { zoneId: zone.zoneId, action: 'none', duration: 0, timing: new Date(), volume: 0, rationale: 'Soil moisture within optimal range', expectedSoilMoistureIncrease: 0 }; } } private async calculateCropWaterRequirement( zone: IrrigationZone ): Promise { const cropCoefficient = this.getCropCoefficient( zone.crop, await this.getCropStage(zone) ); const et0 = await this.getReferenceET(zone); const etc = et0 * cropCoefficient; // Crop ET const mad = this.getMAD(zone.crop); // Management Allowed Depletion const whc = this.getWHC(zone.soilType); // Water Holding Capacity return { optimal: whc * (1 - mad), critical: whc * (1 - mad - 0.2), dailyUse: etc }; } private async planImmediateIrrigation( zone: IrrigationZone, deficit: number ): Promise { // Calculate irrigation depth needed const depth = deficit * 1.1; // Add 10% for distribution uniformity // Convert to time based on system capacity const duration = (depth / zone.irrigationCapacity) * 60; // minutes // Calculate volume const volume = zone.area * 10000 * (depth / 1000); // cubic meters return { zoneId: zone.zoneId, action: 'irrigate', duration: Math.round(duration), timing: new Date(), volume: Math.round(volume), rationale: `Soil moisture ${deficit}mm below optimal, immediate irrigation required`, expectedSoilMoistureIncrease: deficit }; } async modelPredictiveControl( zones: IrrigationZone[], horizon: number // days ): Promise { // MPC: Optimize irrigation over forecast horizon const forecast = await this.getExtendedForecast(horizon); const schedule: IrrigationEvent[] = []; for (const zone of zones) { // Simulate water balance over horizon let soilMoisture = (await this.getCurrentMoisture(zone)).current; const requirement = await this.calculateCropWaterRequirement(zone); for (let day = 0; day < horizon; day++) { const dailyET = await this.forecastEvapotranspiration( zone, forecast.days[day] ); const rainfall = forecast.days[day].precipitation.reduce( (sum, p) => sum + p.amount, 0 ); soilMoisture = soilMoisture - dailyET + rainfall * 0.8; // If moisture falls below optimal, schedule irrigation if (soilMoisture < requirement.optimal) { const irrigationAmount = requirement.optimal - soilMoisture; schedule.push({ zoneId: zone.zoneId, date: this.addDays(new Date(), day), amount: irrigationAmount, reason: 'MPC optimization' }); soilMoisture = requirement.optimal; } } } return { startDate: new Date(), endDate: this.addDays(new Date(), horizon), events: schedule, totalWaterUsed: schedule.reduce((sum, e) => sum + e.amount, 0), estimatedSavings: this.calculateSavings(schedule) }; } }

5. Decision Support Systems

5.1 Integrated Farm Management Platforms

Decision support systems aggregate data from multiple sources—IoT sensors, weather services, satellite imagery, market prices, historical records—to provide comprehensive operational recommendations. These platforms present actionable insights through intuitive dashboards, mobile alerts, and automated reports, translating complex analytics into practical guidance farmers can act upon.

5.2 Prescriptive Analytics

Moving beyond predictive analytics that forecast what will happen, prescriptive analytics recommend what actions to take. Optimization algorithms consider multiple objectives—maximizing yield, minimizing input costs, reducing environmental impact—subject to constraints like water availability, labor capacity, and equipment limitations. These multi-objective optimizations provide decision-makers with Pareto-optimal solutions representing different trade-offs between competing goals.

Key Takeaways

  • Agricultural data analytics must address temporal dependencies, seasonal patterns, and spatial heterogeneity inherent in farm data
  • Yield prediction combines process-based crop models with machine learning, achieving accuracies of 85-95% using ensemble approaches
  • Pest and disease detection integrates environmental monitoring, image recognition (85-95% accuracy), and predictive models for early warning
  • Smart irrigation optimization uses model predictive control, balancing current soil moisture, weather forecasts, and crop requirements to achieve 20-30% water savings
  • Edge-cloud hybrid architectures enable real-time analytics for time-critical decisions while leveraging cloud for intensive computations
  • Prescriptive analytics provide actionable recommendations optimized for multiple objectives (yield, cost, sustainability) subject to farm constraints

Practical Implementation Checklist

  • Manifest integrity — every published manifest is signed via Sigstore DSSE. Tampering is detectable without trusting an intermediary. Updates are issued as new manifests; the prior manifest is preserved with a supersede marker.
  • Retention — keep signed evidence for at least seven years to satisfy ISO/IEC 17065:2012 audit expectations. Some regulators require longer retention; in that case the deployment policy MUST extend the window rather than relying on the defaults.
  • Versioning — follow Semantic Versioning 2.0.0. Major version bumps require at least a 90-day overlap with the prior major version on every WIA-published reference implementation.
  • Negative-result reporting — a skipped indicator without recorded justification is treated as a failed indicator at audit time.
  • Key rotation — manifest signing keys are rotated per the deployment policy. The previous key is preserved in a published key history so that auditors can verify historical manifests offline.

Reference Topologies

Implementations declare their operational topology in the manifest: single-tenant edge (one runtime per organization), multi-tenant gateway (shared runtime with header-based isolation), federated mesh (multiple runtimes peer to one another and publish manifests to a directory service), or air-gapped batch (signed evidence package transported out-of-band). Each topology ships with reference test vectors so that implementations can validate their deployment matches the canonical pattern.
