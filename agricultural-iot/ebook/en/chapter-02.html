Chapter 2

Sensors & Devices

This chapter provides comprehensive coverage of agricultural sensors and IoT devices, including soil monitoring systems, weather stations, crop health sensors, and livestock tracking technologies. We explore hardware selection criteria, deployment strategies, calibration procedures, and integration patterns for building robust sensor networks.

1. Soil Monitoring Sensors

1.1 Soil Moisture Sensors

Soil moisture sensors are fundamental to precision irrigation management. These sensors measure volumetric water content in soil, enabling farmers to optimize irrigation schedules based on actual plant water needs rather than fixed schedules or subjective assessment.

Several sensing technologies are available, each with distinct characteristics:

  • Capacitance Sensors: Measure dielectric constant of soil, which correlates with water content. Low cost, low power, suitable for continuous monitoring
  • Time Domain Reflectometry (TDR): Measures electromagnetic wave travel time in soil. High accuracy, expensive, research-grade applications
  • Frequency Domain Reflectometry (FDR): Balance between cost and accuracy, widely used in commercial agriculture
  • Tensiometers: Measure soil water tension (suction). Direct measurement of water availability to plants
// TypeScript: Soil Moisture Sensor Interface interface SoilMoistureSensor { id: string; location: GeoCoordinates; depth: number; // cm type: 'capacitance' | 'TDR' | 'FDR' | 'tensiometer'; calibration: CalibrationData; } interface MoistureReading { sensorId: string; timestamp: Date; volumetricWaterContent: number; // percentage temperature: number; // celsius electricalConductivity?: number; // dS/m batteryLevel: number; // percentage } class SoilMoistureMonitor { private sensors: Map; private readings: TimeSeriesDB; async collectReading(sensorId: string): Promise { const sensor = this.sensors.get(sensorId); if (!sensor) throw new Error(`Sensor ${sensorId} not found`); const rawData = await this.readSensorData(sensorId); const calibrated = this.applySoilCalibration(rawData, sensor.calibration); const reading: MoistureReading = { sensorId, timestamp: new Date(), volumetricWaterContent: calibrated.vwc, temperature: rawData.temperature, electricalConductivity: rawData.ec, batteryLevel: rawData.battery }; await this.readings.insert(reading); return reading; } private applySoilCalibration( raw: RawSensorData, calibration: CalibrationData ): CalibratedData { // Topp equation for VWC from dielectric constant const dielectric = raw.rawValue; const vwc = -0.053 + 0.0292 * dielectric - 0.00055 * Math.pow(dielectric, 2) + 0.0000043 * Math.pow(dielectric, 3); // Apply soil-specific calibration const adjusted = calibration.slope * vwc + calibration.intercept; return { vwc: Math.max(0, Math.min(100, adjusted * 100)), confidence: this.calculateConfidence(raw, calibration) }; } async analyzeIrrigationNeed( fieldId: string ): Promise { const sensors = this.getSensorsByField(fieldId); const readings = await Promise.all( sensors.map(s => this.getLatestReading(s.id)) ); const avgMoisture = readings.reduce((sum, r) => sum + r.volumetricWaterContent, 0) / readings.length; const crop = await this.getCropInfo(fieldId); const threshold = this.getIrrigationThreshold(crop); if (avgMoisture < threshold.critical) { return { action: 'irrigate_immediately', priority: 'high', estimatedVolume: this.calculateIrrigationVolume( fieldId, threshold.target - avgMoisture ), affectedZones: this.identifyDryZones(readings, threshold) }; } else if (avgMoisture < threshold.optimal) { return { action: 'schedule_irrigation', priority: 'medium', recommendedTime: this.predictIrrigationTiming(readings, crop), estimatedVolume: this.calculateIrrigationVolume( fieldId, threshold.target - avgMoisture ) }; } else { return { action: 'no_irrigation_needed', priority: 'low', nextCheckTime: new Date(Date.now() + 4 * 3600 * 1000) }; } } }

1.2 Soil Nutrient Sensors

Soil nutrient sensors measure levels of key nutrients—primarily nitrogen (N), phosphorus (P), and potassium (K)— as well as pH and electrical conductivity. Real-time nutrient monitoring enables variable rate fertilization, reducing input costs and environmental impact while maintaining optimal crop nutrition.

Modern nutrient sensors employ ion-selective electrodes, optical spectroscopy, or electrochemical methods. While laboratory soil testing remains the gold standard for comprehensive nutrient analysis, in-situ sensors provide continuous monitoring capabilities that support dynamic management decisions.

2. Weather Monitoring Stations

2.1 Microclimate Sensors

Agricultural weather stations collect localized meteorological data critical for irrigation scheduling, disease prediction, frost protection, and growing degree day calculations. Unlike regional weather forecasts, on-farm stations capture microclimate variations that significantly affect crop development and pest dynamics.

A comprehensive weather station typically includes:

  • Temperature and Humidity: Air temperature, relative humidity, dew point calculation
  • Precipitation: Rainfall amount and intensity, critical for irrigation decisions
  • Wind: Speed and direction, affects disease spread and spray application timing
  • Solar Radiation: Photosynthetically active radiation (PAR) for growth modeling
  • Barometric Pressure: Weather pattern prediction
  • Leaf Wetness: Disease risk assessment for foliar pathogens
// TypeScript: Weather Station Implementation interface WeatherStation { stationId: string; location: GeoCoordinates; elevation: number; sensors: WeatherSensorSuite; dataLogger: DataLogger; connectivity: NetworkInterface; } interface WeatherSensorSuite { temperatureHumidity: THSensor; rainfall: RainGauge; windSpeedDirection: Anemometer; solarRadiation: Pyranometer; barometer: BarometricSensor; leafWetness?: LeafWetnessSensor; } interface WeatherObservation { stationId: string; timestamp: Date; temperature: number; // celsius humidity: number; // percentage dewPoint: number; // celsius rainfall: number; // mm rainfallIntensity: number; // mm/hr windSpeed: number; // m/s windDirection: number; // degrees solarRadiation: number; // W/m² pressure: number; // hPa leafWetness?: number; // minutes in last hour } class WeatherMonitoringSystem { private stations: Map; private observations: TimeSeriesDB; async recordObservation( stationId: string ): Promise { const station = this.stations.get(stationId); if (!station) throw new Error(`Station ${stationId} not found`); const sensors = station.sensors; const temp = await sensors.temperatureHumidity.readTemperature(); const humidity = await sensors.temperatureHumidity.readHumidity(); const dewPoint = this.calculateDewPoint(temp, humidity); const observation: WeatherObservation = { stationId, timestamp: new Date(), temperature: temp, humidity, dewPoint, rainfall: await sensors.rainfall.readAccumulation(), rainfallIntensity: await sensors.rainfall.readIntensity(), windSpeed: await sensors.windSpeedDirection.readSpeed(), windDirection: await sensors.windSpeedDirection.readDirection(), solarRadiation: await sensors.solarRadiation.read(), pressure: await sensors.barometer.read(), leafWetness: sensors.leafWetness ? await sensors.leafWetness.read() : undefined }; await this.observations.insert(observation); await this.analyzeConditions(observation); return observation; } private calculateDewPoint(temp: number, humidity: number): number { const a = 17.27; const b = 237.7; const alpha = ((a * temp) / (b + temp)) + Math.log(humidity / 100); return (b * alpha) / (a - alpha); } async predictDiseasePressure( fieldId: string, crop: CropType ): Promise { const station = this.getNearestStation(fieldId); const observations = await this.getRecentObservations( station.stationId, { hours: 48 } ); const leafWetnessHours = observations.reduce((total, obs) => total + (obs.leafWetness || 0), 0) / 60; const avgTemp = observations.reduce((sum, obs) => sum + obs.temperature, 0) / observations.length; const avgHumidity = observations.reduce((sum, obs) => sum + obs.humidity, 0) / observations.length; // Disease model example: Late Blight for potatoes if (crop === 'potato') { if (avgTemp >= 10 && avgTemp <= 25 && avgHumidity > 90 && leafWetnessHours > 10) { return { disease: 'late_blight', riskLevel: 'high', recommendation: 'Apply fungicide within 24 hours', confidence: 0.85, favorableConditions: { temperature: true, humidity: true, leafWetness: true } }; } } return { disease: null, riskLevel: 'low', recommendation: 'Continue monitoring', confidence: 0.75 }; } async calculateEvapotranspiration( stationId: string, date: Date ): Promise { // FAO Penman-Monteith equation for reference ET const obs = await this.getDailyObservation(stationId, date); const station = this.stations.get(stationId); const T = obs.temperature; const RH = obs.humidity; const u2 = obs.windSpeed; // at 2m height const Rs = obs.solarRadiation; const elevation = station!.elevation; // Atmospheric pressure const P = 101.3 * Math.pow((293 - 0.0065 * elevation) / 293, 5.26); // Saturation vapor pressure const es = 0.6108 * Math.exp(17.27 * T / (T + 237.3)); // Actual vapor pressure const ea = es * RH / 100; // Slope of saturation vapor pressure curve const delta = 4098 * es / Math.pow(T + 237.3, 2); // Psychrometric constant const gamma = 0.000665 * P; // Net radiation (simplified) const Rn = 0.77 * Rs - 0.3; // MJ/m²/day // Soil heat flux (negligible for daily calculations) const G = 0; // Reference evapotranspiration (mm/day) const ET0 = (0.408 * delta * (Rn - G) + gamma * (900 / (T + 273)) * u2 * (es - ea)) / (delta + gamma * (1 + 0.34 * u2)); return ET0; } }

3. Crop Monitoring Sensors

3.1 Multispectral and Hyperspectral Imaging

Optical sensors capture crop reflectance in specific wavelength bands to assess plant health, stress, and development stage. The most common metric is the Normalized Difference Vegetation Index (NDVI), which correlates with chlorophyll content, biomass, and overall crop vigor.

These sensors can be deployed on ground-based platforms, UAVs, or satellites. Ground-based sensors provide continuous monitoring of specific plots, while aerial platforms enable whole-field mapping. Integration of multi-scale imagery creates comprehensive crop health monitoring systems.

3.2 Plant Health Sensors

Advanced crop monitoring includes thermal imaging to detect water stress before visible symptoms appear, fluorescence sensors to measure photosynthetic efficiency, and acoustic sensors to detect insect infestations inside plant stems. These technologies enable early intervention, preventing minor issues from becoming major yield losses.

Deployment Best Practice

For soil moisture sensors, deploy at multiple depths (15cm, 30cm, 60cm) to capture the full root zone profile. Install at representative locations within each management zone, avoiding field edges, old fence lines, or areas with atypical soil conditions. Minimum density: one sensor set per 5-10 hectares for uniform fields, higher density for variable terrain.

4. Livestock Tracking and Monitoring

4.1 Animal Wearables

Livestock IoT devices track animal location, activity, health metrics, and behavior patterns. Collar-mounted sensors monitor GPS position, accelerometer data for activity classification, rumination time, body temperature, and heart rate. These devices enable precision livestock farming, improving animal welfare while optimizing production efficiency.

4.2 Health and Reproduction Monitoring

Early detection of health issues and optimal estrus detection significantly impact livestock operations economics. Accelerometer-based activity monitoring detects behavioral changes associated with illness or heat cycles. Temperature sensors identify fever or hypothermia. Rumination sensors track digestive health, with decreased rumination often being an early indicator of metabolic disorders or infectious disease.

} return analysis; } private analyzeHealthTrends( current: AnimalHealthData, historical: AnimalHealthData[] ): HealthAnalysis { const alerts: Alert[] = []; // Temperature deviation const avgTemp = historical.reduce((sum, d) => sum + d.vitalSigns.bodyTemperature, 0) / historical.length; if (current.vitalSigns.bodyTemperature > avgTemp + 1.0) { alerts.push({ type: 'fever', severity: 'high', message: `Elevated temperature: ${current.vitalSigns.bodyTemperature}°C`, animalId: current.animalId, timestamp: current.timestamp }); } // Activity decrease (potential illness) const avgSteps = historical.reduce((sum, d) => sum + d.activity.steps, 0) / historical.length; if (current.activity.steps < avgSteps * 0.5) { alerts.push({ type: 'low_activity', severity: 'medium', message: `Activity 50% below normal: ${current.activity.steps} steps`, animalId: current.animalId, timestamp: current.timestamp }); } // Rumination decrease (digestive issues) const avgRumination = historical.reduce((sum, d) => sum + d.activity.ruminating, 0) / historical.length; if (current.activity.ruminating < avgRumination * 0.7) { alerts.push({ type: 'reduced_rumination', severity: 'medium', message: `Rumination time reduced by 30%`, animalId: current.animalId, timestamp: current.timestamp }); } // Estrus detection (increased activity) if (current.activity.steps > avgSteps * 2.0 && current.activity.ruminating < avgRumination * 0.8) { alerts.push({ type: 'potential_estrus', severity: 'info', message: 'Behavior pattern consistent with estrus', animalId: current.animalId, timestamp: current.timestamp }); } return { animalId: current.animalId, healthScore: this.calculateHealthScore(current, historical), alerts, trends: this.calculateTrends(historical), recommendations: this.generateRecommendations(alerts) }; } async trackHerdLocation(): Promise { const devices = Array.from(this.wearables.values()); const locations = await Promise.all( devices.map(d => this.getLatestLocation(d.deviceId)) ); // Cluster analysis for grazing patterns const clusters = this.performSpatialClustering(locations); return { timestamp: new Date(), totalAnimals: devices.length, clusters: clusters.map(c => ({ centroid: c.center, animalCount: c.members.length, area: c.coverageArea, density: c.members.length / c.coverageArea })), virtualFenceViolations: this.checkVirtualFences(locations) }; } }

5. Sensor Deployment and Maintenance

5.1 Installation Best Practices

Proper sensor installation is critical for data quality and system longevity. Soil sensors should be installed when soil is at field capacity to ensure good soil-sensor contact. Weather stations require open locations away from trees, buildings, or other obstructions that create microclimates. Regular calibration checks and maintenance prevent sensor drift and ensure continued accuracy.

5.2 Power Management

Battery-powered sensors must balance data collection frequency with power consumption. Duty cycling—powering sensors only during measurements—extends battery life. Solar panels can provide sustainable power for always-on applications. Low-power wide-area networks (LPWAN) minimize communication energy consumption.

Key Takeaways

  • Soil moisture sensors enable precision irrigation, with capacitance sensors offering best cost-accuracy balance for most applications
  • Weather stations provide microclimate data critical for irrigation scheduling, disease prediction, and evapotranspiration calculations
  • Multispectral imaging and NDVI monitoring detect crop stress before visual symptoms, enabling early intervention
  • Livestock wearables track activity, rumination, and temperature to detect health issues and estrus with 85%+ accuracy
  • Proper sensor placement, calibration, and maintenance are as important as sensor selection for data quality
  • Power management through duty cycling and solar charging enables multi-year deployments without battery replacement

Practical Implementation Checklist

  • Manifest integrity — every published manifest is signed via Sigstore DSSE. Tampering is detectable without trusting an intermediary. Updates are issued as new manifests; the prior manifest is preserved with a supersede marker.
  • Retention — keep signed evidence for at least seven years to satisfy ISO/IEC 17065:2012 audit expectations. Some regulators require longer retention; in that case the deployment policy MUST extend the window rather than relying on the defaults.
  • Versioning — follow Semantic Versioning 2.0.0. Major version bumps require at least a 90-day overlap with the prior major version on every WIA-published reference implementation.
  • Negative-result reporting — a skipped indicator without recorded justification is treated as a failed indicator at audit time.
  • Key rotation — manifest signing keys are rotated per the deployment policy. The previous key is preserved in a published key history so that auditors can verify historical manifests offline.

Reference Topologies

Implementations declare their operational topology in the manifest: single-tenant edge (one runtime per organization), multi-tenant gateway (shared runtime with header-based isolation), federated mesh (multiple runtimes peer to one another and publish manifests to a directory service), or air-gapped batch (signed evidence package transported out-of-band). Each topology ships with reference test vectors so that implementations can validate their deployment matches the canonical pattern.
