1. 디지털 소멸권의 법적 토대

디지털 소멸권은 전 세계적으로 다양한 법적 형태로 구현되고 있습니다. 유럽연합의 GDPR(General Data Protection Regulation)과 한국의 개인정보보호법은 각각 독특한 접근 방식으로 개인의 데이터 삭제 권리를 보장하고 있습니다. 이 장에서는 두 법률 체계를 심층 비교하고, 실무에서 마주하는 법적 쟁점을 다룹니다.

2. GDPR Article 17: Right to Erasure

2.1 GDPR의 탄생 배경

2018년 5월 25일, 유럽연합(EU)은 개인정보보호 역사상 가장 강력한 법률인 GDPR을 시행했습니다. 이 법은 1995년의 EU Data Protection Directive를 대체하며, 디지털 시대에 맞는 포괄적인 개인정보 보호 체계를 구축했습니다.

GDPR은 EU 내 5억 명 이상의 시민을 대상으로 하며, EU 시민의 데이터를 처리하는 전 세계 모든 기업에 적용됩니다. 이는 한국 기업이 유럽 고객을 대상으로 서비스를 제공할 경우에도 GDPR을 준수해야 함을 의미합니다.

2.2 Article 17의 전문

⚖️ GDPR Article 17 - Right to Erasure ('Right to be Forgotten')

제1항: 정보주체는 다음 각 호의 어느 하나에 해당하는 경우, 부당한 지체 없이 자신에 관한 개인정보의 삭제를 관리자에게 요청할 권리를 가지며, 관리자는 부당한 지체 없이 개인정보를 삭제할 의무를 진다:

  1. (a) 개인정보가 수집 또는 처리 목적과 관련하여 더 이상 필요하지 않은 경우
  2. (b) 정보주체가 동의를 철회한 경우 (Article 6(1)(a) 또는 Article 9(2)(a))
  3. (c) 정보주체가 Article 21(1)에 따라 처리에 반대하는 경우
  4. (d) 개인정보가 불법적으로 처리된 경우
  5. (e) 법적 의무 준수를 위해 개인정보가 삭제되어야 하는 경우
  6. (f) 정보사회 서비스 제공 관련하여 아동의 개인정보가 수집된 경우 (Article 8(1))

2.3 GDPR의 핵심 원칙

원칙 내용 실무 적용
적법성 (Lawfulness) 개인정보 처리는 합법적 근거가 있어야 함 동의, 계약, 법적 의무 등 명확한 근거 확보
목적 제한 (Purpose Limitation) 수집 목적 외 사용 금지 명시적 목적 달성 후 삭제 의무
데이터 최소화 (Data Minimisation) 필요 최소한의 정보만 수집 불필요한 데이터는 즉시 삭제
정확성 (Accuracy) 개인정보는 정확하고 최신이어야 함 부정확한 정보는 수정 또는 삭제
보관 제한 (Storage Limitation) 필요 기간만 보관 보관 기간 경과 시 자동 삭제
무결성과 기밀성 (Integrity & Confidentiality) 적절한 보안 조치 안전한 삭제 프로세스 구현

2.4 삭제 요청의 예외 사항

GDPR Article 17(3)는 다음의 경우 삭제 의무가 적용되지 않음을 명시합니다:

  • (a) 표현 및 정보의 자유 행사: 언론 보도, 학술 연구, 예술적 표현
  • (b) 법적 의무 준수: EU 또는 회원국 법률에 따른 보존 의무
  • (c) 공공 이익: 공중보건, 공공안전, 역사적 기록 보존
  • (d) 과학적/역사적/통계적 연구: 공익을 위한 연구 목적
  • (e) 법적 청구의 제기, 행사, 방어: 소송 관련 증거 보존

2.5 GDPR 위반 시 제재

⚠️ GDPR 벌금 체계

1단계 위반 (최대 1,000만 유로 또는 전년도 전 세계 매출의 2% 중 높은 금액):

  • 프로세서의 의무 위반 (Article 28-36, 42, 43)
  • 인증 기관의 의무 위반
  • 감독 기관의 조사 불응

2단계 위반 (최대 2,000만 유로 또는 전년도 전 세계 매출의 4% 중 높은 금액):

  • 기본 원칙 위반 (Article 5-11)
  • 정보주체의 권리 침해 (Article 12-22, 포함 삭제권)
  • 제3국 전송 규정 위반 (Article 44-49)

실제 벌금 사례:

  • Amazon (2021): 7억 4,600만 유로 - 동의 없는 개인정보 처리
  • Google Ireland (2022): 9,000만 유로 - 삭제권 미보장
  • Meta Ireland (2023): 12억 유로 - 미국으로의 불법 데이터 전송

3. 한국 개인정보보호법 제21조

3.1 개인정보보호법의 역사

한국의 개인정보보호법은 2011년 9월 30일 제정되어 2011년 3월 29일부터 시행되었습니다. 이는 공공과 민간 부문을 모두 포괄하는 일반법으로, GDPR보다 7년 앞서 시행되었습니다. 2020년 2월 데이터 3법(개인정보보호법, 정보통신망법, 신용정보법) 개정으로 개인정보 보호 체계가 더욱 강화되었습니다.

3.2 제21조 전문

⚖️ 개인정보보호법 제21조 (개인정보의 파기)

제1항: 개인정보처리자는 보유기간의 경과, 개인정보의 처리 목적 달성 등 그 개인정보가 불필요하게 되었을 때에는 지체 없이 그 개인정보를 파기하여야 한다. 다만, 다른 법령에 따라 보존하여야 하는 경우에는 그러하지 아니하다.

제2항: 개인정보처리자가 제1항에 따라 개인정보를 파기할 때에는 복구 또는 재생되지 아니하도록 조치하여야 한다.

제3항: 개인정보처리자가 제1항 단서에 따라 개인정보를 파기하지 아니하고 보존하여야 하는 경우에는 해당 개인정보 또는 개인정보파일을 다른 개인정보와 분리하여서 저장·관리하여야 한다.

제4항: 개인정보의 파기방법 및 절차 등에 필요한 사항은 대통령령으로 정한다.

3.3 시행령상 파기 방법

형태 파기 방법 기술적 요구사항
전자파일 - 복원 불가능한 방법으로 영구 삭제
- 로우레벨 포맷 또는 덮어쓰기		 미 국방부 DoD 5220.22-M 표준 또는 동등 이상
종이문서 - 파쇄 또는 소각
- 전문 파기업체 이용 가능		 복원 불가능하도록 세절 (2mm x 15mm 이하 권장)
저장매체 - 물리적 파쇄
- 자기장을 이용한 소거 (Degaussing)		 물리적 파괴 또는 전자기 소거 장비 사용
백업데이터 - 정기적인 백업 데이터 파기
- 보존기간 내 백업도 암호화		 백업 시스템에서도 동일한 파기 기준 적용

3.4 개인정보보호위원회의 역할

개인정보보호위원회는 2020년 8월 국무총리 소속에서 대통령 직속 중앙행정기관으로 격상되었습니다. 주요 권한은 다음과 같습니다:

  • 조사 및 감독: 개인정보 처리 실태 조사 및 시정 명령
  • 과징금 부과: 위반 행위에 대한 과징금 및 과태료 부과
  • 가이드라인 제공: 업종별, 분야별 개인정보 보호 가이드 제시
  • 분쟁 조정: 개인정보 침해로 인한 분쟁 조정
  • 국제 협력: GDPR 적정성 평가 대응 등 국제 협력

3.5 법 위반 시 제재

⚠️ 개인정보보호법 벌칙 규정

형사 처벌 (제71조, 제72조, 제73조):

  • 5년 이하 징역 또는 5,000만원 이하 벌금: 부정한 방법으로 개인정보 취득, 제3자 제공
  • 3년 이하 징역 또는 3,000만원 이하 벌금: 안전성 확보조치 미이행으로 인한 개인정보 유출
  • 2년 이하 징역 또는 2,000만원 이하 벌금: 파기의무 위반 등

과징금 (제34조의2):

  • 매출액의 100분의 3 이하 (최대 한도)
  • 매출액 산정이 곤란한 경우: 5억원 이하

과태료 (제75조):

  • 5,000만원 이하: 파기의무 위반, 안전조치 미이행 등
  • 3,000만원 이하: 동의 절차 위반 등

4. GDPR vs 한국법: 상세 비교

4.1 체계적 차이

구분 GDPR 한국 개인정보보호법
제정 시기 2018년 5월 시행 2011년 9월 시행 (2020년 개정)
적용 범위 EU 시민 데이터 처리 모든 기업 (역외 적용) 한국 내 개인정보 처리자 (일부 역외 적용)
감독 기관 각국 DPA (Data Protection Authority) 개인정보보호위원회 (단일 기관)
권리 명칭 Right to Erasure (삭제권) 개인정보 파기 의무 / 삭제 요구권
법적 성격 정보주체의 능동적 권리 처리자의 의무 + 정보주체의 권리
최대 벌금 매출의 4% 또는 2,000만 유로 매출의 3% (과징금) + 형사처벌

4.2 삭제 사유 비교

GDPR Article 17(1) 한국법 제21조 차이점

불법 처리 (위법행위 시 형사처벌) ⚠️ 별도 구조

4.3 예외 사유 비교

// GDPR 예외 사유 (Article 17(3))
const gdprExceptions = [
  "표현 및 정보의 자유",
  "법적 의무 준수",
  "공공 이익 (보건, 안전)",
  "과학/역사/통계 연구",
  "법적 청구 행사/방어"
];

// 한국법 예외 사유 (제21조 제1항 단서 + 제58조)
const koreaExceptions = [
  "다른 법령에 따른 보존 의무",
  "전자상거래법: 계약/청약철회 기록 (5년)",
  "통신비밀보호법: 통신사실확인자료 (12개월)",
  "국세기본법: 장부 (5년)",
  "상법: 회계장부 (10년)"
];

// 실무 적용 시 고려사항
function checkErasureException(request, userLocation) {
  if (userLocation === 'EU') {
    return evaluateGDPRExceptions(request);
  } else if (userLocation === 'KR') {
    return evaluateKoreaExceptions(request);
  }
  // 가장 엄격한 기준 적용
  return evaluateBoth(request);
}

5. 실무 적용: 법적 준수 전략

5.1 글로벌 서비스 제공 시 전략

🌍 지역별 법률 준수 전략

Option 1: 최고 수준 통일 (Highest Standard Approach)

  • ✅ 장점: 단일 프로세스로 모든 지역 준수 가능
  • ❌ 단점: 가장 엄격한 기준 적용으로 비용 증가
  • 📌 추천: 중소 규모 서비스, 리소스 제한적일 때

Option 2: 지역별 맞춤 (Localized Approach)

  • ✅ 장점: 각 지역 법률에 최적화된 효율적 운영
  • ❌ 단점: 복잡한 다중 프로세스 관리 필요
  • 📌 추천: 대규모 글로벌 서비스, 충분한 법무팀

Option 3: 하이브리드 (Hybrid Approach)

  • ✅ 장점: 핵심은 통일, 세부사항은 지역별 맞춤
  • ❌ 단점: 초기 설계 복잡도 높음
  • 📌 추천: WIA-DIGITAL_ERASURE 표준 권장 방식

5.2 삭제 요청 처리 프로세스

// WIA-DIGITAL_ERASURE 표준 처리 흐름
async function processErasureRequest(request) {
  // 1. 신원 확인
  const verified = await verifyIdentity(request.userId);
  if (!verified) {
    return { status: 'REJECTED', reason: 'IDENTITY_VERIFICATION_FAILED' };
  }

  // 2. 관할 법률 판단
  const jurisdiction = determineJurisdiction(request.userId);
  const applicableLaw = (jurisdiction === 'EU') ? 'GDPR' :
                        (jurisdiction === 'KR') ? 'PIPA' : 'OTHER';

  // 3. 예외 사유 검토
  const exceptions = await checkLegalExceptions(request, applicableLaw);
  if (exceptions.length > 0) {
    return {
      status: 'PARTIAL',
      reason: 'LEGAL_EXCEPTIONS_APPLY',
      exceptions: exceptions,
      deletableData: exceptions.deletableScope
    };
  }

  // 4. 삭제 범위 결정
  const scope = await determineDeletionScope(request);

  // 5. 삭제 실행
  const results = await executeErasure(scope);

  // 6. 검증
  const verification = await verifyErasure(scope);

  // 7. 로그 기록 (최소한)
  await logErasureAction({
    requestId: request.id,
    timestamp: new Date(),
    userId: hashUserId(request.userId), // 해시 처리
    scope: scope,
    result: results.status
  });

  // 8. 사용자 통지
  await notifyUser(request.userId, {
    status: 'COMPLETED',
    deletedData: scope,
    completionDate: new Date(),
    verificationCode: verification.code
  });

  return {
    status: 'SUCCESS',
    verificationCode: verification.code
  };
}

7. 정리: 이 장에서 배운 내용

🎯 핵심 요점 정리

  • GDPR Article 17: 정보주체 중심의 능동적 삭제권 보장
  • 한국법 제21조: 처리자 의무 중심의 파기 의무 규정
  • 공통점: 목적 달성 후 삭제, 법적 예외 인정, 강력한 제재
  • 차이점: 법적 구조, 벌금 체계, 예외 사유의 구체성
  • 글로벌 전략: 최고 수준 통일 또는 지역별 맞춤 선택
  • 실무 핵심: 신원 확인, 관할 판단, 예외 검토, 검증 가능한 삭제
  • 판례의 중요성: 법 조문만으로 부족, 판례 통해 구체적 기준 파악 필수
  • WIA 표준의 역할: 서로 다른 법체계 간 상호운용성 제공
  • 弘益人間: 법률 준수를 통한 개인의 권리 보호와 사회적 신뢰 구축
