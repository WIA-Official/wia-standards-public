WIA-DIGITAL_ERASURE API Standards

1. API 인터페이스 개요

WIA-DIGITAL_ERASURE API는 개인정보 삭제 요청을 표준화된 방식으로 처리하기 위한 RESTful API 인터페이스입니다. 이 표준은 네이버, 카카오, 구글 등 주요 플랫폼이 채택할 수 있도록 설계되었으며, 개인정보보호법 제21조(개인정보의 파기)를 기술적으로 구현합니다.

📌 학습 목표

  • WIA-DIGITAL_ERASURE API의 핵심 엔드포인트 이해
  • 삭제 요청 생명주기(Request Lifecycle) 관리
  • 인증 및 권한 부여 메커니즘
  • 네이버/카카오 등 국내 플랫폼 적용 사례
  • GDPR Article 17 준수를 위한 API 설계

2. API 아키텍처

WIA-DIGITAL_ERASURE API는 3-Layer 아키텍처를 기반으로 합니다:

계층 역할 기술 스택 책임
Presentation Layer API Gateway REST API, OAuth 2.0 요청 검증, 인증, 라우팅
Business Logic Layer Erasure Orchestrator Node.js, TypeScript 삭제 로직, 상태 관리, 검증
Data Layer Storage Manager PostgreSQL, Redis 데이터 영구 삭제, 로그 보관

4. 인증 및 권한 부여

4.1 OAuth 2.0 인증 흐름

WIA-DIGITAL_ERASURE API는 OAuth 2.0 표준을 따르며, 다음과 같은 인증 방식을 지원합니다:

인증 방식 사용 시나리오 보안 수준 국내 지원
Authorization Code 웹 애플리케이션 ⭐⭐⭐⭐⭐ 네이버, 카카오
PKCE Extension 모바일 앱 ⭐⭐⭐⭐⭐ 전체 지원
Device Code 스마트 TV, IoT ⭐⭐⭐⭐ 제한적
Public Key Certificate 정부/금융 서비스 ⭐⭐⭐⭐⭐ 행정안전부 인증

4.2 본인 인증 레벨

/**
 * 본인 인증 레벨 정의
 * @compliance 전자서명법, 개인정보보호법
 */

enum VerificationLevel {
  LEVEL_1 = 1,  // 이메일/SMS 인증
  LEVEL_2 = 2,  // 2단계 인증 (MFA)
  LEVEL_3 = 3,  // 공인인증서 (구)
  LEVEL_4 = 4   // 금융인증서 / 간편인증
}

// 삭제 범위에 따른 필요 인증 레벨
const requiredVerificationLevel = (scope: ErasureScope): VerificationLevel => {
  if (scope.includes('FINANCIAL_DATA')) {
    return VerificationLevel.LEVEL_4;  // 금융 데이터는 최고 등급 필요
  }

  if (scope.includes('SENSITIVE_PERSONAL_INFO')) {
    return VerificationLevel.LEVEL_3;  // 민감정보는 공인인증
  }

  if (scope.includes('POSTS') || scope.includes('COMMENTS')) {
    return VerificationLevel.LEVEL_2;  // 일반 콘텐츠는 2단계 인증
  }

  return VerificationLevel.LEVEL_1;  // 기본 데이터는 이메일 인증
};

// 네이버 적용 사례
const naverVerification = {
  email: VerificationLevel.LEVEL_1,
  sms: VerificationLevel.LEVEL_1,
  naverAuth: VerificationLevel.LEVEL_2,  // 네이버 OTP
  certificate: VerificationLevel.LEVEL_4  // 금융인증서
};

// 카카오 적용 사례
const kakaoVerification = {
  kakaoTalk: VerificationLevel.LEVEL_2,   // 카카오톡 본인인증
  kakaoPass: VerificationLevel.LEVEL_4,   // 카카오페이 인증
  biometric: VerificationLevel.LEVEL_3    // 생체인증
};

6. 삭제 요청 생명주기

🔄 Erasure Request Lifecycle

1. CREATED (생성됨)
   ↓ 본인 인증 대기
2. PENDING_VERIFICATION (인증 대기)
   ↓ OTP/인증서 확인
3. VERIFIED (인증 완료)
   ↓ 삭제 예약
4. SCHEDULED (삭제 예정)
   ↓ 삭제 시작
5. IN_PROGRESS (진행 중)
   ├─→ PRIMARY_DB_DELETION (주 데이터베이스 삭제)
   ├─→ BACKUP_DELETION (백업 삭제)
   ├─→ CACHE_INVALIDATION (캐시 무효화)
   ├─→ SEARCH_INDEX_REMOVAL (검색 인덱스 제거)
   └─→ THIRD_PARTY_NOTIFICATION (제3자 통지)
   ↓ 검증
6. VERIFICATION (검증 중)
   ↓ 완료 확인
7. COMPLETED (완료)

// 예외 상태
- FAILED (실패): 시스템 오류
- CANCELLED (취소됨): 사용자 취소
- EXPIRED (만료됨): 인증 시간 초과
- BLOCKED (차단됨): 법적 보존 의무

7. 국내 플랫폼 적용 사례

7.1 네이버 통합 예제

/**
 * 네이버 플랫폼 WIA-DIGITAL_ERASURE 적용
 * @platform Naver
 * @compliance 개인정보보호법 제21조
 */

import { WIAErasureClient } from '@wia/digital-erasure';

const client = new WIAErasureClient({
  platform: 'NAVER',
  apiKey: process.env.NAVER_API_KEY,
  endpoint: 'https://api.naver.com/wia/erasure/v1'
});

// 네이버 계정 삭제 요청
async function deleteNaverAccount(userId: string) {
  try {
    // 1. 삭제 요청 생성
    const request = await client.createErasureRequest({
      userId: userId,
      userIdType: 'NAVER_ID',
      scope: {
        dataCategories: [
          'PROFILE',          // 프로필
          'NAVER_BLOG',       // 블로그
          'NAVER_CAFE',       // 카페
          'NAVER_MAIL',       // 메일
          'NAVER_CLOUD',      // 클라우드
          'NAVER_PAY',        // 네이버페이
          'LINE',             // 라인 (연동 계정)
          'SHOPPING_HISTORY'  // 쇼핑 내역
        ],
        includeBackups: true,
        includeThirdParty: true  // 제휴사 포함
      },
      legalBasis: {
        regulation: 'PIPA_ARTICLE_21',
        reason: 'WITHDRAWAL_OF_CONSENT'
      },
      verification: {
        method: 'NAVER_OTP'  // 네이버 OTP
      }
    });

    console.log(`삭제 요청 생성: ${request.requestId}`);
    console.log(`인증 방법: ${request.verification.method}`);

    // 2. 인증 진행 (네이버 OTP)
    const otpCode = await getNaverOTP();  // 사용자로부터 OTP 입력 받기

    const verified = await client.verifyRequest(request.requestId, {
      method: 'NAVER_OTP',
      value: otpCode
    });

    if (!verified.success) {
      throw new Error('인증 실패');
    }

    // 3. 진행 상황 모니터링
    const interval = setInterval(async () => {
      const status = await client.getStatus(request.requestId);

      console.log(`진행률: ${status.progress.percentage}%`);
      console.log(`현재 단계: ${status.progress.currentPhase}`);

      if (status.status === 'COMPLETED') {
        clearInterval(interval);
        console.log('✅ 네이버 계정 삭제 완료');

        // 4. 삭제 완료 보고서
        const report = await client.getCompletionReport(request.requestId);
        console.log(`삭제된 데이터: ${report.deletedData.totalSize}`);
        console.log(`처리 시간: ${report.duration}`);
      }

      if (status.status === 'FAILED') {
        clearInterval(interval);
        console.error('❌ 삭제 실패:', status.error);
      }
    }, 5000);  // 5초마다 확인

  } catch (error) {
    console.error('네이버 계정 삭제 오류:', error);
    throw error;
  }
}

// 실행
deleteNaverAccount('user@naver.com');

  • ☑️ SQL Injection 방어: Prepared Statement
  • ☑️ 감사 로그: 모든 API 호출 기록 (최소 3년 보관)
  • ☑️ GDPR Article 32: 기술적·관리적 보호조치
  • ☑️ 개인정보보호법 제29조: 안전성 확보조치

9.1 통합 테스트 시나리오

/**
 * WIA-DIGITAL_ERASURE API 통합 테스트
 * @test-framework Jest + Supertest
 */

import { describe, it, expect } from '@jest/globals';
import { WIAErasureClient } from '@wia/digital-erasure';

describe('WIA-DIGITAL_ERASURE API Integration Test', () => {

  it('정상적인 삭제 요청 생성', async () => {
    const response = await client.createErasureRequest({
      userId: 'test@example.com',
      scope: { dataCategories: ['PROFILE'] },
      verification: { method: 'EMAIL_OTP' }
    });

    expect(response.success).toBe(true);
    expect(response.requestId).toMatch(/^ER-\d{4}-\d{4}-\d{6}$/);
    expect(response.status).toBe('PENDING_VERIFICATION');
  });

  it('중복 요청 시 409 Conflict', async () => {
    await expect(
      client.createErasureRequest({ userId: 'test@example.com' })
    ).rejects.toThrow('ER-4091');
  });

  it('법적 보존 기간 중인 데이터 삭제 시도', async () => {
    const response = await client.createErasureRequest({
      userId: 'user-with-legal-hold@example.com',
      scope: { dataCategories: ['FINANCIAL_DATA'] }
    });

    expect(response.success).toBe(false);
    expect(response.error.code).toBe('ER-4231');
    expect(response.error.type).toBe('LEGAL_HOLD');
  });

  it('10일 이내 삭제 완료 (SLA 준수)', async () => {
    const request = await client.createErasureRequest({
      userId: 'test-sla@example.com'
    });

    // 최대 10일 대기 (개인정보보호법 준수)
    const startTime = Date.now();
    await client.waitForCompletion(request.requestId, { timeout: 10 * 24 * 60 * 60 * 1000 });
    const duration = Date.now() - startTime;

    expect(duration).toBeLessThan(10 * 24 * 60 * 60 * 1000);  // 10일
  });
});

  • API 아키텍처: Presentation, Business Logic, Data Layer 3계층 구조
