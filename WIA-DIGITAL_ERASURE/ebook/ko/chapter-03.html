1. 개인정보 파기 의무의 법적 구조

한국 개인정보보호법 제21조는 단순한 삭제권을 넘어, 개인정보처리자의 적극적인 파기 의무를 규정하고 있습니다. 이는 EU GDPR의 정보주체 중심 접근과는 다른, 처리자 의무 중심의 독특한 구조입니다. 이 장에서는 제21조의 실무 적용을 위한 세부 가이드라인과 구체적인 파기 방법을 다룹니다.

📌 학습 목표

  • 개인정보보호법 시행령 및 시행규칙 이해
  • 개인정보보호위원회 가이드라인 숙지
  • 법정 보존기간 및 예외 사유 정확한 파악
  • 기술적 파기 방법의 구체적 요구사항 학습
  • 개인정보 처리방침 작성법 실습
  • 개인정보 영향평가(PIA) 수행 방법 이해

2. 개인정보보호법 시행령 제16조: 파기 방법

2.1 전자파일 형태의 파기

⚖️ 시행령 제16조 제1항

개인정보처리자는 전자적 파일 형태로 저장된 개인정보를 파기하고자 하는 때에는 다음 각 호의 어느 하나의 조치를 하여야 한다:

  1. 복원이 불가능한 방법으로 영구 삭제
  2. 저장매체에 대한 물리적 파괴 또는 자기장을 이용한 소거
  3. 전문 파기 업체를 통한 안전한 파기 및 파기증명서 수령

2.2 기술적 파기 방법 상세

파기 방법 기술 표준 적용 대상 보안 수준
DoD 5220.22-M 3회 덮어쓰기 (0xFF, 0x00, 난수) 일반 HDD ⭐⭐⭐
Gutmann Method 35회 덮어쓰기 고보안 HDD ⭐⭐⭐⭐⭐
ATA Secure Erase 펌웨어 레벨 완전 삭제 SSD, HDD ⭐⭐⭐⭐
Degaussing 자기장 소거 (10,000+ Gauss) 자기 저장매체 ⭐⭐⭐⭐⭐
Physical Destruction 물리적 파쇄/소각 모든 매체 ⭐⭐⭐⭐⭐
Cryptographic Erasure 암호화 키 파기 암호화된 데이터 ⭐⭐⭐⭐

2.3 파기 구현 코드 예시

// WIA-DIGITAL_ERASURE 표준 파기 모듈
class SecureDataEraser {
  constructor(config) {
    this.method = config.method || 'DOD_5220_22_M';
    this.verificationEnabled = config.verification || true;
    this.logRetention = config.logRetention || 180; // 일
  }

  async eraseFile(filePath, options = {}) {
    const startTime = Date.now();
    const fileInfo = await this.getFileInfo(filePath);

    try {
      // 1. 파일 존재 확인
      if (!await this.fileExists(filePath)) {
        throw new Error('File not found');
      }

      // 2. 파기 방법 선택
      let result;
      switch (this.method) {
        case 'DOD_5220_22_M':
          result = await this.dod5220(filePath);
          break;
        case 'GUTMANN':
          result = await this.gutmann(filePath);
          break;
        case 'CRYPTO_ERASE':
          result = await this.cryptoErase(filePath);
          break;
        default:
          throw new Error('Unknown erasure method');
      }

      // 3. 검증
      if (this.verificationEnabled) {
        const verified = await this.verifyErasure(filePath);
        if (!verified) {
          throw new Error('Erasure verification failed');
        }
      }

      // 4. 로그 기록
      await this.logErasure({
        fileHash: fileInfo.hash,
        fileSize: fileInfo.size,
        method: this.method,
        timestamp: new Date(),
        duration: Date.now() - startTime,
        verified: this.verificationEnabled,
        status: 'SUCCESS'
      });

      return {
        status: 'SUCCESS',
        fileHash: fileInfo.hash,
        method: this.method,
        timestamp: new Date(),
        verificationCode: result.verificationCode
      };

    } catch (error) {
      await this.logErasure({
        fileHash: fileInfo.hash,
        error: error.message,
        status: 'FAILED'
      });
      throw error;
    }
  }

  // DoD 5220.22-M: 3회 덮어쓰기
  async dod5220(filePath) {
    const passes = [
      0xFF, // 첫 번째: 모두 1
      0x00, // 두 번째: 모두 0
      () => Math.random() * 0xFF // 세 번째: 랜덤
    ];

    for (let i = 0; i < passes.length; i++) {
      await this.overwriteFile(filePath, passes[i]);
    }

    // 파일 메타데이터 삭제
    await this.removeMetadata(filePath);

    // 파일명 난수화 후 삭제
    const randomName = this.generateRandomName();
    await this.rename(filePath, randomName);
    await this.delete(randomName);

    return {
      verificationCode: this.generateVerificationCode()
    };
  }

  // 암호화 키 파기를 통한 논리적 삭제
  async cryptoErase(filePath) {
    // 파일이 암호화되어 있는 경우
    const keyId = await this.getEncryptionKeyId(filePath);

    // 암호화 키 파기
    await this.destroyEncryptionKey(keyId);

    // 키 파기 후 파일은 복호화 불가능
    return {
      verificationCode: this.generateVerificationCode(),
      keyDestructionTimestamp: new Date()
    };
  }
}

5. 개인정보 처리방침 작성

5.1 필수 포함 사항

// 개인정보 처리방침 파기 관련 필수 명시사항

제7조 (개인정보의 파기)

① 회사는 개인정보 보유기간의 경과, 처리목적 달성 등 개인정보가
   불필요하게 되었을 때에는 지체 없이 해당 개인정보를 파기합니다.

② 정보주체로부터 동의받은 개인정보 보유기간이 경과하거나 처리목적이
   달성되었음에도 불구하고 다른 법령에 따라 개인정보를 계속 보존하여야
   하는 경우에는, 해당 개인정보를 별도의 데이터베이스(DB)로 옮기거나
   보관장소를 달리하여 보존합니다.

③ 개인정보 파기의 절차 및 방법은 다음과 같습니다.

   1. 파기절차
      - 회사는 파기 사유가 발생한 개인정보를 선정하고,
        개인정보 보호책임자의 승인을 받아 개인정보를 파기합니다.
      - 법령에 따라 보존해야 하는 정보는 별도 DB로 이관하여
        내부 방침 및 기타 관련 법령에 따라 일정기간 저장된 후 파기됩니다.

   2. 파기방법
      - 전자적 파일 형태: 복구 및 재생이 불가능한 기술적 방법을
        사용하여 완전히 삭제
      - 종이 문서: 분쇄기로 분쇄하거나 소각을 통하여 파기

④ 보유기간 및 파기 시점
   - 회원정보: 회원 탈퇴 시 즉시 파기
   - 계약/청약철회 기록: 5년 (전자상거래법)
   - 대금결제/재화공급 기록: 5년 (전자상거래법)
   - 소비자 불만/분쟁처리 기록: 3년 (전자상거래법)
   - 접속 로그 기록: 3개월 (통신비밀보호법)

6. 개인정보 영향평가 (Privacy Impact Assessment)

6.1 PIA 수행 대상

⚠️ 영향평가 의무 대상 (개인정보보호법 제33조)

  1. 5만 명 이상의 정보주체에 관한 민감정보 또는 고유식별정보의 처리
  2. 100만 명 이상의 정보주체에 관한 개인정보 파일 구축·운용
  3. 법령에서 구체적으로 평가 대상을 규정한 경우

* 위 기준에 해당하는 개인정보 파일을 운용하는 경우, 구축·운용 전 또는 변경 전에 영향평가 수행 의무

  • 복구 불가 처리: 3회 덮어쓰기 (DoD 5220.22-M) 기준 적용
  • 파기 확인: 사용자에게 탈퇴 완료 및 파기 확인 이메일 발송

7.2 카카오의 개인정보 파기 프로세스

✅ 카카오 서비스 탈퇴 및 정보 파기

  • 통합 파기: 카카오계정 탈퇴 시 모든 연결 서비스 정보 일괄 파기

8. 실무 체크리스트

✅ 한국 기업을 위한 개인정보 파기 체크리스트

A. 정책 및 문서화

  • ☐ 개인정보 처리방침에 파기 절차 명시
  • ☐ 내부 파기 지침 문서화
  • ☐ 보유기간별 개인정보 목록 작성
  • ☐ 법정 보존 정보 목록 및 근거 명시

B. 기술적 조치

  • ☐ 자동 파기 시스템 구축
  • ☐ 복구 불가능한 파기 방법 적용 (DoD 5220.22-M 이상)
  • ☐ 백업 시스템 파기 프로세스 구현
  • ☐ 분리 보관 시스템 구축 (법정 보존 정보)
  • ☐ 파기 검증 프로세스 구현

C. 관리적 조치

  • ☐ 개인정보 보호책임자 지정
  • ☐ 파기 승인 절차 수립
  • ☐ 파기 이력 기록 및 보관
  • ☐ 정기적인 파기 현황 점검 (분기별)
  • ☐ 임직원 교육 실시 (연 1회 이상)

D. 제3자 관리

  • ☐ 위탁업체 파기 의무 계약서 명시
  • ☐ 제3자 제공 데이터 파기 요청 프로세스
  • ☐ 파기 전문업체 이용 시 파기증명서 수령
