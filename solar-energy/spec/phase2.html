☀️ WIA-ENE-005: Phase 2 Specification

Integration - Grid Connectivity and System Integration

1. Overview

Phase 2 builds upon Phase 1 foundations to enable comprehensive grid integration, battery storage interfacing, and advanced monitoring capabilities. This phase focuses on bidirectional power flow, energy management, and system-level optimization.

2. Grid Integration Requirements

2.1 Advanced Grid Support Functions

Phase 2 systems must provide configurable grid support services:

Function Requirement Response Time Accuracy
Reactive Power Control ±60% of rated power <1 second ±5% of setpoint
Power Factor Control 0.85 leading to 0.85 lagging <2 seconds ±0.01
Frequency-Watt Programmable droop curve <2 seconds ±2% of setpoint
Volt-VAR Programmable curve <5 seconds ±5% of setpoint
Ramp Rate Control 1-100% per minute Continuous ±10% of setpoint

2.2 Voltage Ride-Through

Systems must remain connected during voltage and frequency excursions:

  • Low voltage ride-through: 50% voltage for up to 1 second
  • High voltage ride-through: 110% voltage for up to 1 second
  • Frequency range: 57-61 Hz (60Hz systems), 47-51 Hz (50Hz systems)

3.2 Operating Modes

Phase 2 systems must support multiple operating modes:

  • Self-Consumption: Maximize use of solar generation, minimize grid import
  • Time-of-Use: Charge during low-rate periods, discharge during high-rate periods
  • Backup: Maintain battery charge for outage support
  • Grid Services: Provide frequency regulation, demand response, peak shaving
  • Manual: User-defined charging/discharging schedule

4. Enhanced Monitoring and Data

4.1 Real-Time Data Streaming

Systems must provide high-frequency data via MQTT or WebSocket:

Parameter Update Rate Protocol
Instantaneous Power 1 Hz MQTT/WebSocket
Voltage/Current 1 Hz MQTT/WebSocket
Energy Totals 0.1 Hz (10 sec) MQTT/WebSocket
Status/Alarms Event-driven MQTT/WebSocket

4.2 Historical Data Requirements

  • 1-minute averages: 30 days retention
  • 15-minute averages: 1 year retention
  • Daily totals: Indefinite retention
  • Event log: 1 year retention minimum

5.2 Dynamic Setpoint Control

Support remote setpoint adjustments via API:

  • Active power limit (curtailment)
  • Reactive power setpoint
  • Power factor target
  • Battery charge/discharge commands
  • Operating mode selection

6. Communication Security

6.1 Encryption Requirements

  • TLS 1.2 or higher for all IP-based communication
  • Certificate-based authentication for API access
  • AES-256 encryption for stored credentials
  • Regular security updates and patch management

6.2 Access Control

  • Role-based access control (RBAC) for user management
  • Multi-factor authentication for administrative access
  • Audit logging of all configuration changes
  • IP whitelisting for critical control functions
