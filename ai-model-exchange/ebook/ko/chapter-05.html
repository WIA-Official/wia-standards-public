5.1 모델 최적화가 중요한 이유

모델 최적화는 허용 가능한 정확도를 유지하면서 크기를 줄이고, 추론 속도를 향상시키며, 에너지 소비를 줄입니다. 이는 메모리와 컴퓨팅이 제한된 스마트폰, 임베디드 시스템, 엣지 디바이스 등 리소스가 제한된 디바이스에 배포하는 데 중요합니다.

최적화 목표

  • 모델 크기 축소: 100MB 단위에서 한 자릿수 MB로
  • 지연 시간 개선: 실시간 애플리케이션을 위한 더 빠른 추론
  • 전력 소비 감소: 모바일 및 엣지 디바이스에 필수적
  • 정확도 유지: 최적화로 인한 저하 최소화

5.2 양자화 기초

양자화는 모델 가중치와 활성화의 정밀도를 부동 소수점(FP32)에서 더 낮은 정밀도 형식(FP16, INT8, INT4)으로 줄입니다. 이는 모델 크기를 극적으로 줄이고 특수 정수 산술 유닛이 있는 하드웨어에서 추론 속도를 향상시킵니다.

양자화 유형

유형 정밀도 크기 축소 속도 향상 정확도 손실
FP32 (기준선) 32비트 부동소수점 1x 1x 0%
FP16 16비트 부동소수점 2x 1.5-2x < 0.1%
INT8 동적 8비트 정수 4x 2-4x 0.5-2%
INT8 정적 8비트 정수 4x 3-4x 1-3%
INT4 4비트 정수 8x 4-8x 3-5%

5.3 학습 후 양자화

학습 후 양자화(PTQ)는 재학습 없이 이미 학습된 모델에 양자화를 적용합니다.

PyTorch 동적 양자화

import torch
import torch.quantization
import torchvision.models as models

# 사전 학습된 모델 로드
model = models.resnet18(pretrained=True)
model.eval()

# 동적 양자화 (가중치만)
quantized_model = torch.quantization.quantize_dynamic(
    model,
    {torch.nn.Linear, torch.nn.Conv2d},  # 양자화할 레이어
    dtype=torch.qint8
)

# 크기 비교
def get_model_size(model):
    torch.save(model.state_dict(), "temp.pth")
    size_mb = os.path.getsize("temp.pth") / (1024 * 1024)
    os.remove("temp.pth")
    return size_mb

original_size = get_model_size(model)
quantized_size = get_model_size(quantized_model)

print(f"원본 모델: {original_size:.2f} MB")
print(f"양자화된 모델: {quantized_size:.2f} MB")
print(f"압축 비율: {original_size / quantized_size:.2f}x")

5.4 양자화 인식 학습

양자화 인식 학습(QAT)은 학습 중 양자화를 시뮬레이션하여 모델이 적응하고 양자화 후 더 높은 정확도를 유지할 수 있게 합니다.

PyTorch QAT

import torch
import torch.quantization as quantization

# 모델 정의
model = models.resnet18(pretrained=True)

# QAT 설정 지정
model.train()
model.qconfig = quantization.get_default_qat_qconfig('fbgemm')

# QAT 준비
model_prepared = quantization.prepare_qat(model)

# QAT로 학습 루프
optimizer = torch.optim.SGD(model_prepared.parameters(), lr=0.001)
criterion = torch.nn.CrossEntropyLoss()

for epoch in range(num_epochs):
    for data, target in train_loader:
        optimizer.zero_grad()
        output = model_prepared(data)
        loss = criterion(output, target)
        loss.backward()
        optimizer.step()

    # 몇 epoch 후 옵저버 비활성화 및 배치 정규화 고정
    if epoch > 3:
        model_prepared.apply(torch.quantization.disable_observer)
    if epoch > 2:
        model_prepared.apply(torch.nn.intrinsic.qat.freeze_bn_stats)

# 양자화된 모델로 변환
model_prepared.eval()
model_quantized = quantization.convert(model_prepared)

5.5 프루닝

프루닝은 네트워크에서 중요하지 않은 가중치를 제거하여 더 작고 빠른 희소 모델을 생성합니다.

크기 기반 프루닝

import torch
import torch.nn.utils.prune as prune

# 모델 로드
model = models.resnet18(pretrained=True)

# Conv2d 가중치의 30%를 크기로 프루닝
for name, module in model.named_modules():
    if isinstance(module, torch.nn.Conv2d):
        prune.l1_unstructured(module, name='weight', amount=0.3)

# 프루닝을 영구적으로 만들기
for name, module in model.named_modules():
    if isinstance(module, torch.nn.Conv2d):
        prune.remove(module, 'weight')

# 희소성 확인
def check_sparsity(model):
    total_params = 0
    zero_params = 0
    for param in model.parameters():
        total_params += param.numel()
        zero_params += (param == 0).sum().item()

    sparsity = 100.0 * zero_params / total_params
    return sparsity

sparsity = check_sparsity(model)
print(f"모델 희소성: {sparsity:.2f}%")

구조적 프루닝

# 전체 채널/필터 프루닝 (하드웨어 친화적)
prune.ln_structured(

    name='weight',
    amount=0.5,  # 필터의 50% 프루닝
    n=2,         # L2 노름
    dim=0        # 출력 채널 차원으로 프루닝
)

5.6 지식 증류

지식 증류는 더 큰 "교사" 모델을 모방하도록 더 작은 "학생" 모델을 학습시켜 학생을 처음부터 학습시키는 것보다 더 나은 정확도를 달성합니다. 

import torch
import torch.nn as nn
import torch.nn.functional as F

class DistillationLoss(nn.Module):
    def __init__(self, temperature=3.0, alpha=0.5):
        super().__init__()
        self.temperature = temperature
        self.alpha = alpha
        self.ce_loss = nn.CrossEntropyLoss()

    def forward(self, student_logits, teacher_logits, targets):
        # 교사의 소프트 타겟
        soft_loss = F.kl_div(
            F.log_softmax(student_logits / self.temperature, dim=1),
            F.softmax(teacher_logits / self.temperature, dim=1),
            reduction='batchmean'
        ) * (self.temperature ** 2)

        # 하드 타겟 (실제 정답)
        hard_loss = self.ce_loss(student_logits, targets)

        # 결합된 손실
        return self.alpha * soft_loss + (1 - self.alpha) * hard_loss

# 교사 모델 (큰 모델)
teacher = models.resnet50(pretrained=True)
teacher.eval()

# 학생 모델 (작은 모델)
student = models.resnet18(pretrained=False)

# 학습 루프
distill_loss_fn = DistillationLoss(temperature=3.0, alpha=0.7)
optimizer = torch.optim.Adam(student.parameters(), lr=0.001)

for epoch in range(num_epochs):
    for data, targets in train_loader:
        # 교사 예측 (그래디언트 없음)
        with torch.no_grad():
            teacher_logits = teacher(data)

        # 학생 예측
        student_logits = student(data)

        # 증류 손실
        loss = distill_loss_fn(student_logits, teacher_logits, targets)

        optimizer.zero_grad()
        loss.backward()
        optimizer.step()

print("지식 증류 완료")

5.9 최적화 영향 측정

벤치마크 스위트

import time
import numpy as np

def benchmark_model(model, input_shape, num_iterations=100, warmup=10):
    """
    모델 추론 성능 벤치마크
    """
    # 워밍업
    dummy_input = torch.randn(*input_shape)
    for _ in range(warmup):
        _ = model(dummy_input)

    # 벤치마크
    torch.cuda.synchronize() if torch.cuda.is_available() else None
    start_time = time.time()

    for _ in range(num_iterations):
        _ = model(dummy_input)

    torch.cuda.synchronize() if torch.cuda.is_available() else None
    end_time = time.time()

    # 메트릭 계산
    total_time = end_time - start_time
    avg_latency = (total_time / num_iterations) * 1000  # ms
    throughput = num_iterations / total_time  # fps

    return {
        'avg_latency_ms': avg_latency,
        'throughput_fps': throughput,
        'total_time_s': total_time
    }

# 원본 vs 최적화 비교
results_original = benchmark_model(model_original, (1, 3, 224, 224))
results_optimized = benchmark_model(model_optimized, (1, 3, 224, 224))

print("원본 모델:")
print(f"  지연 시간: {results_original['avg_latency_ms']:.2f} ms")
print(f"  처리량: {results_original['throughput_fps']:.2f} FPS")

print("\n최적화된 모델:")
print(f"  지연 시간: {results_optimized['avg_latency_ms']:.2f} ms")
print(f"  처리량: {results_optimized['throughput_fps']:.2f} FPS")

speedup = results_original['avg_latency_ms'] / results_optimized['avg_latency_ms']
print(f"\n속도 향상: {speedup:.2f}x")

요약

  • 모델 최적화는 크기를 줄이고, 속도를 향상시키며, 전력 소비를 낮춥니다
  • 양자화는 FP32를 더 낮은 정밀도(FP16, INT8, INT4)로 변환하여 2-8배 압축
  • 학습 후 양자화(PTQ)는 재학습 없이 학습된 모델에 적용됩니다
  • 양자화 인식 학습(QAT)은 학습 중 양자화를 시뮬레이션하여 더 높은 정확도 유지
  • 프루닝은 중요하지 않은 가중치를 제거하여 희소 모델 생성
  • 지식 증류는 더 작은 모델이 더 큰 모델을 모방하도록 학습
  • 그래프 최적화(융합, 상수 폴딩)로 추론 성능 향상
  • 하드웨어별 최적화(TensorRT, CoreML)로 추가 속도 향상
  • 항상 벤치마크를 수행하고 정확도-성능 트레이드오프를 측정하세요
  • 배포 대상과 요구사항에 따라 최적화를 선택하세요
