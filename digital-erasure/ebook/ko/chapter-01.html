잊힐 권리

1.1 Introduction to the 잊힐 권리

The 잊힐 권리 (RTBF), also known as the "right to erasure," is a fundamental privacy right enshrined in Article 17 of the General Data Protection Regulation (GDPR). This right empowers individuals to request the deletion of their personal data under specific circumstances, providing them with greater control over their digital identity and personal information.

The WIA 디지털 삭제 표준 provides a comprehensive framework for implementing RTBF compliance, ensuring that organizations can effectively honor erasure requests while maintaining data integrity, legal compliance, and operational efficiency.

1.1.1 Legal Foundation

The right to erasure is grounded in several key legal principles:

  • Personal data is no longer necessary for the purposes for which it was collected
  • The data subject withdraws consent and there is no other legal ground for processing
  • The data subject objects to processing and there are no overriding legitimate grounds
  • Personal data has been unlawfully processed
  • Personal data must be erased to comply with a legal obligation
  • Personal data was collected in relation to the offer of information society services to children

1.6 Implementation Example: RTBF Request Handler

The following Python implementation demonstrates a comprehensive RTBF request handler:

import asyncio
import uuid
from datetime import datetime, timedelta
from typing import List, Dict, Optional
from enum import Enum
import hashlib
import json

class RequestPhase(Enum):
    SUBMITTED = "SUBMITTED"
    IDENTITY_VERIFICATION = "IDENTITY_VERIFICATION"
    LEGAL_ASSESSMENT = "LEGAL_ASSESSMENT"
    DATA_MAPPING = "DATA_MAPPING"
    THIRD_PARTY_NOTIFICATION = "THIRD_PARTY_NOTIFICATION"
    ERASURE_EXECUTION = "ERASURE_EXECUTION"
    VERIFICATION = "VERIFICATION"
    COMPLETED = "COMPLETED"
    REJECTED = "REJECTED"

class DataCategory(Enum):
    PROFILE_DATA = "PROFILE_DATA"
    AUTHENTICATION_DATA = "AUTHENTICATION_DATA"
    FINANCIAL_DATA = "FINANCIAL_DATA"
    HEALTH_DATA = "HEALTH_DATA"
    BEHAVIORAL_DATA = "BEHAVIORAL_DATA"
    USER_CONTENT = "USER_CONTENT"
    BIOMETRIC_DATA = "BIOMETRIC_DATA"
    COMMUNICATION_RECORDS = "COMMUNICATION_RECORDS"

class RTBFRequestHandler:
    def __init__(self, database_client, notification_service, audit_logger):
        self.db = database_client
        self.notifier = notification_service
        self.audit = audit_logger
        self.verification_timeout = 72  # hours

    async def submit_request(self, user_id: str, email: str,
                            data_categories: List[DataCategory],
                            legal_grounds: str) -> Dict:
        """Submit a new 잊힐 권리 request"""
        request_id = str(uuid.uuid4())
        submission_time = datetime.utcnow()

        request_data = {
            "requestId": request_id,
            "userId": user_id,
            "requestType": "FULL_ERASURE",
            "submissionTimestamp": submission_time.isoformat(),
            "requesterInfo": {
                "email": email,
                "verificationMethod": "EMAIL_OTP",
                "verificationStatus": "PENDING"
            },
            "dataCategories": [cat.value for cat in data_categories],
            "legalGrounds": legal_grounds,
            "processingStatus": {
                "currentPhase": RequestPhase.SUBMITTED.value,
                "phaseStartTime": submission_time.isoformat(),
                "estimatedCompletionTime": (
                    submission_time + timedelta(days=30)
                ).isoformat(),
                "completionPercentage": 0
            },
            "auditTrail": [{
                "timestamp": submission_time.isoformat(),
                "action": "REQUEST_SUBMITTED",
                "performedBy": user_id,
                "details": f"RTBF request submitted for {len(data_categories)} data categories",
                "systemId": "RTBF_HANDLER_v1"
            }]
        }

        await self.db.store_request(request_id, request_data)
        await self.send_verification_email(email, request_id)
        await self.audit.log_event("RTBF_REQUEST_SUBMITTED", request_id, user_id)

        return {
            "requestId": request_id,
            "status": "SUBMITTED",
            "message": "Verification email sent. Please verify your identity within 72 hours.",
            "verificationDeadline": (
                submission_time + timedelta(hours=self.verification_timeout)
            ).isoformat()
        }

    async def verify_identity(self, request_id: str,
                             verification_token: str) -> bool:
        """Verify user identity for RTBF request"""
        request = await self.db.get_request(request_id)

        if not request:
            raise ValueError("Request not found")

        stored_token_hash = request["requesterInfo"].get("verificationTokenHash")
        provided_token_hash = hashlib.sha256(
            verification_token.encode()
        ).hexdigest()

        if stored_token_hash == provided_token_hash:
            request["requesterInfo"]["verificationStatus"] = "VERIFIED"
            request["processingStatus"]["currentPhase"] = (
                RequestPhase.LEGAL_ASSESSMENT.value
            )
            request["processingStatus"]["completionPercentage"] = 15

            await self.db.update_request(request_id, request)
            await self.audit.log_event("IDENTITY_VERIFIED", request_id,
                                      request["userId"])

            # Automatically trigger legal assessment
            await self.perform_legal_assessment(request_id)

            return True

        return False

    async def perform_legal_assessment(self, request_id: str) -> Dict:
        """Assess legal validity of RTBF request"""
        request = await self.db.get_request(request_id)
        exceptions = []

        # Check for legal obligations that prevent erasure
        financial_data_retention = await self.check_financial_retention(
            request["userId"]
        )
        if financial_data_retention:
            exceptions.append({
                "exceptionType": "LEGAL_OBLIGATION",
                "description": "Tax law requires 7-year retention of financial records",
                "retentionPeriod": "7 years",
                "autoExpirationDate": financial_data_retention["expirationDate"]
            })

        # Check for ongoing legal claims
        active_litigation = await self.check_active_litigation(
            request["userId"]
        )
        if active_litigation:
            exceptions.append({
                "exceptionType": "DEFENSE_OF_LEGAL_CLAIMS",
                "description": f"Active litigation case: {active_litigation['caseId']}",
                "retentionPeriod": "Until case resolution",
                "autoExpirationDate": None
            })

        request["exceptions"] = exceptions
        request["processingStatus"]["currentPhase"] = (
            RequestPhase.DATA_MAPPING.value
        )
        request["processingStatus"]["completionPercentage"] = 30

        await self.db.update_request(request_id, request)
        await self.audit.log_event("LEGAL_ASSESSMENT_COMPLETED",
                                   request_id, request["userId"])

        return {
            "assessmentComplete": True,
            "exceptionsFound": len(exceptions),
            "canProceed": True,
            "exceptions": exceptions
        }

    async def execute_erasure(self, request_id: str) -> Dict:
        """Execute data erasure across all systems"""
        request = await self.db.get_request(request_id)
        user_id = request["userId"]
        data_categories = request["dataCategories"]

        erasure_results = {
            "totalSystems": 0,
            "successfulErasures": 0,
            "failedErasures": 0,
            "systemResults": []
        }

        # Execute erasure for each data category across relevant systems
        for category in data_categories:
            systems = await self.get_systems_for_category(category)

            for system in systems:
                erasure_results["totalSystems"] += 1

                try:
                    await self.erase_from_system(system, user_id, category)
                    erasure_results["successfulErasures"] += 1
                    erasure_results["systemResults"].append({
                        "system": system,
                        "category": category,
                        "status": "SUCCESS",
                        "timestamp": datetime.utcnow().isoformat()
                    })
                except Exception as e:
                    erasure_results["failedErasures"] += 1
                    erasure_results["systemResults"].append({
                        "system": system,
                        "category": category,
                        "status": "FAILED",
                        "error": str(e),
                        "timestamp": datetime.utcnow().isoformat()
                    })

        request["processingStatus"]["currentPhase"] = (
            RequestPhase.VERIFICATION.value
        )
        request["processingStatus"]["completionPercentage"] = 85
        request["erasureResults"] = erasure_results

        await self.db.update_request(request_id, request)
        await self.audit.log_event("ERASURE_EXECUTED", request_id, user_id)

        return erasure_results

# Example usage
async def main():

        user_id="user_12345",

1.7 Third-Party Notification Service

Organizations must notify all third-party data processors and recipients when executing RTBF requests:

import aiohttp
import asyncio
from typing import List, Dict
from datetime import datetime

class ThirdPartyNotificationService:
    def __init__(self, timeout: int = 30):
        self.timeout = aiohttp.ClientTimeout(total=timeout)
        self.retry_attempts = 3
        self.retry_delay = 5  # seconds

    async def notify_all_processors(self, user_id: str,
                                    request_id: str,
                                    processors: List[Dict]) -> List[Dict]:
        """Send erasure notifications to all data processors"""
        notification_results = []

        async with aiohttp.ClientSession(timeout=self.timeout) as session:
            tasks = [
                self.notify_processor(session, processor, user_id, request_id)
                for processor in processors
            ]
            results = await asyncio.gather(*tasks, return_exceptions=True)

            for processor, result in zip(processors, results):
                if isinstance(result, Exception):
                    notification_results.append({
                        "processorName": processor["name"],
                        "processorId": processor["id"],
                        "notificationSent": False,
                        "notificationTimestamp": datetime.utcnow().isoformat(),
                        "acknowledgmentReceived": False,
                        "erasureConfirmed": False,
                        "error": str(result)
                    })
                else:
                    notification_results.append(result)

        return notification_results

    async def notify_processor(self, session: aiohttp.ClientSession,
                               processor: Dict, user_id: str,
                               request_id: str) -> Dict:
        """Notify individual processor of erasure request"""
        notification_payload = {
            "requestId": request_id,
            "userId": user_id,
            "action": "ERASE_USER_DATA",
            "deadline": (
                datetime.utcnow() + timedelta(days=7)
            ).isoformat(),
            "verificationRequired": True,
            "callbackUrl": f"https://api.example.com/rtbf/callback/{request_id}"
        }

        for attempt in range(self.retry_attempts):
            try:
                async with session.post(
                    processor["erasureEndpoint"],
                    json=notification_payload,
                    headers={
                        "Authorization": f"Bearer {processor['apiKey']}",
                        "Content-Type": "application/json"
                    }
                ) as response:
                    response.raise_for_status()
                    response_data = await response.json()

                    return {
                        "processorName": processor["name"],
                        "processorId": processor["id"],
                        "notificationSent": True,
                        "notificationTimestamp": datetime.utcnow().isoformat(),
                        "acknowledgmentReceived": True,
                        "erasureConfirmed": response_data.get("confirmed", False),
                        "processorRequestId": response_data.get("requestId")
                    }

            except aiohttp.ClientError as e:
                if attempt == self.retry_attempts - 1:
                    raise
                await asyncio.sleep(self.retry_delay * (attempt + 1))

        raise Exception(f"Failed to notify {processor['name']} after {self.retry_attempts} attempts")

    async def verify_processor_erasure(self, processor: Dict,
                                      processor_request_id: str) -> bool:
        """Verify that processor has completed erasure"""
        async with aiohttp.ClientSession(timeout=self.timeout) as session:
            async with session.get(
                f"{processor['verificationEndpoint']}/{processor_request_id}",
                headers={"Authorization": f"Bearer {processor['apiKey']}"}
            ) as response:
                response.raise_for_status()
                data = await response.json()
                return data.get("erasureComplete", False)

1.8 Erasure Verification and Audit

Comprehensive verification ensures complete erasure and regulatory compliance:

import asyncio
from typing import List, Dict, Set
from datetime import datetime
import hashlib

class ErasureVerificationService:
    def __init__(self, database_clients: Dict, search_indices: List):
        self.db_clients = database_clients
        self.search_indices = search_indices
        self.verification_methods = [
            self.verify_primary_database,
            self.verify_backup_systems,
            self.verify_cache_systems,
            self.verify_search_indices,
            self.verify_logs,
            self.verify_file_storage,
            self.verify_cdn_content
        ]

    async def verify_complete_erasure(self, user_id: str,
                                     request_id: str) -> Dict:
        """Perform comprehensive erasure verification"""
        verification_report = {
            "verificationTimestamp": datetime.utcnow().isoformat(),
            "verificationMethod": "AUTOMATED_SCAN",
            "dataSystemsChecked": [],
            "erasureConfirmed": True,
            "failedVerifications": [],
            "verificationDetails": []
        }

        # Execute all verification methods
        for method in self.verification_methods:
            system_name = method.__name__.replace("verify_", "")
            verification_report["dataSystemsChecked"].append(system_name)

            try:
                result = await method(user_id)

                if not result["erasureComplete"]:
                    verification_report["erasureConfirmed"] = False
                    verification_report["failedVerifications"].append({
                        "system": system_name,
                        "remainingData": result.get("remainingData", []),
                        "details": result.get("details")
                    })

                verification_report["verificationDetails"].append({
                    "system": system_name,
                    "status": "PASS" if result["erasureComplete"] else "FAIL",
                    "recordsFound": result.get("recordsFound", 0),
                    "timestamp": datetime.utcnow().isoformat()
                })

            except Exception as e:
                verification_report["erasureConfirmed"] = False
                verification_report["failedVerifications"].append({
                    "system": system_name,
                    "error": str(e)
                })

        # Generate cryptographic proof of verification
        if verification_report["erasureConfirmed"]:
            verification_report["verificationCertificateId"] = (
                self.generate_verification_certificate(
                    user_id, request_id, verification_report
                )
            )

        return verification_report

    async def verify_primary_database(self, user_id: str) -> Dict:
        """Verify erasure from primary database"""
        remaining_records = []

        for db_name, client in self.db_clients.items():
            query = f"SELECT * FROM users WHERE user_id = '{user_id}'"
            result = await client.execute(query)

            if result and len(result) > 0:
                remaining_records.append({
                    "database": db_name,
                    "table": "users",
                    "recordCount": len(result)
                })

        return {
            "erasureComplete": len(remaining_records) == 0,
            "remainingData": remaining_records,
            "recordsFound": sum(r["recordCount"] for r in remaining_records)
        }

    async def verify_backup_systems(self, user_id: str) -> Dict:
        """Verify erasure from backup systems"""
        # Check backup systems (implementation depends on backup strategy)
        backup_check_passed = True  # Placeholder

        return {
            "erasureComplete": backup_check_passed,
            "details": "Backup verification completed"
        }

    async def verify_search_indices(self, user_id: str) -> Dict:
        """Verify erasure from search indices"""
        found_in_indices = []

        for index in self.search_indices:
            results = await index.search(f"user_id:{user_id}")

            if results and len(results) > 0:
                found_in_indices.append({
                    "index": index.name,
                    "documentCount": len(results)
                })

        return {
            "erasureComplete": len(found_in_indices) == 0,
            "remainingData": found_in_indices,
            "recordsFound": sum(i["documentCount"] for i in found_in_indices)
        }

    def generate_verification_certificate(self, user_id: str,
                                         request_id: str,
                                         verification_report: Dict) -> str:
        """Generate cryptographic certificate for verified erasure"""
        certificate_data = {
            "userId": user_id,
            "requestId": request_id,
            "verificationTimestamp": verification_report["verificationTimestamp"],
            "systemsChecked": verification_report["dataSystemsChecked"],
            "verificationMethod": verification_report["verificationMethod"]
        }

        # Create cryptographic hash as certificate ID
        certificate_string = json.dumps(certificate_data, sort_keys=True)
        certificate_hash = hashlib.sha256(certificate_string.encode()).hexdigest()

Organizations implementing RTBF compliance must meet the following certification requirements:

Technical Requirements (Items 1-10)

  • Automated RTBF request intake and processing system
  • Multi-factor authentication for request verification
  • Comprehensive data mapping across all systems and databases
  • Cryptographic erasure capabilities for sensitive data
  • Real-time audit trail generation and immutable logging
  • Third-party notification API with delivery confirmation
  • Automated verification scanning across all data stores
  • Backup and archive system integration for complete erasure
  • Search index and cache purging mechanisms
  • Certificate generation system for verified erasures

Process Requirements (Items 11-15)

  • User notification system at each processing phase

  • Third-party processor registry with contact information
  • Data retention policy documentation by category

1.10 Exceptions and Special Cases

While the right to be forgotten is a fundamental privacy right, certain exceptions apply under specific circumstances. Organizations must carefully evaluate each request against these legal exceptions:

Important: All exception claims must be thoroughly documented with specific legal citations and business justification. Improper use of exceptions can result in regulatory penalties and loss of user trust.

1.10.1 Freedom of Expression and Information

Data may be retained when necessary for exercising the right of freedom of expression and information, particularly in journalistic, academic, artistic, or literary contexts.

1.10.2 Legal Compliance

Data must be retained when required by EU or member state law, such as tax records, financial transaction data, or employment records mandated by labor regulations.

1.10.3 Public Interest

Data processing may continue when necessary for public health purposes, historical research, statistical purposes, or other tasks carried out in the public interest.

1.10.4 Legal Claims

Data relevant to the establishment, exercise, or defense of legal claims must be retained until the claim is resolved, including applicable statute of limitations periods.
