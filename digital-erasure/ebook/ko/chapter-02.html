2.1 Introduction to 데이터 삭제 프로토콜

Data deletion protocols define the technical and procedural standards for permanently removing personal data from information systems. Effective deletion goes beyond simple record removal and requires comprehensive strategies to ensure data is irretrievable across all storage media, backup systems, and distributed architectures.

The WIA 디지털 삭제 표준 provides a multi-layered approach to data deletion, incorporating cryptographic erasure, physical destruction, logical deletion, and verification mechanisms to ensure complete and compliant data removal.

2.1.1 Deletion Methods Overview

Organizations must implement multiple deletion methods based on data sensitivity, storage medium, and regulatory requirements. The following table provides a comprehensive comparison of deletion approaches:

Deletion Method Effectiveness Use Cases Recovery Risk Performance Impact
Soft Delete (Flag-based) Low Temporary deletion, grace periods High - Easily recoverable Minimal
Hard Delete (Immediate removal) Medium Standard production deletions Medium - May persist in backups Low
Cryptographic Erasure Very High Encrypted data, rapid deletion Very Low - Key destruction prevents recovery Minimal
Overwriting (Single pass) High Magnetic media, file systems Low - Forensic recovery difficult Medium
Overwriting (Multiple passes) Very High High-security requirements Very Low - Near impossible recovery High
Physical Destruction Absolute Hardware decommissioning None - Media destroyed N/A - Hardware loss
Degaussing Very High Magnetic storage media Very Low - Magnetic patterns erased N/A - Media unusable
Block-level Erasure High SSD, flash storage Low - Controller-level deletion Medium

Selecting the appropriate deletion protocol requires careful consideration of data classification, storage infrastructure, and compliance requirements:

2.5 Implementation: Cryptographic Erasure Service

Cryptographic erasure provides the most efficient method for large-scale data deletion by destroying encryption keys:

import asyncio
from typing import List, Dict, Optional
from datetime import datetime
import hashlib
import secrets
from cryptography.hazmat.primitives.ciphers import Cipher, algorithms, modes
from cryptography.hazmat.backends import default_backend
from cryptography.hazmat.primitives import hashes
from cryptography.hazmat.primitives.kdf.pbkdf2 import PBKDF2HMAC

class CryptographicErasureService:
    """
    Implements cryptographic erasure by managing encryption keys
    and ensuring their secure destruction
    """

    def __init__(self, key_management_service, audit_logger):
        self.kms = key_management_service
        self.audit = audit_logger
        self.algorithm = "AES-256-GCM"

    async def execute_crypto_erasure(self, user_id: str,
                                    deletion_id: str,
                                    data_locations: List[Dict]) -> Dict:
        """
        Execute cryptographic erasure by destroying all encryption keys
        associated with user data
        """
        erasure_report = {
            "deletionId": deletion_id,
            "userId": user_id,
            "deletionMethod": "CRYPTOGRAPHIC_ERASURE",
            "initiationTimestamp": datetime.utcnow().isoformat(),
            "keysDestroyed": [],
            "dataLocations": [],
            "verificationProof": None
        }

        try:
            # Step 1: Identify all encryption keys for user data
            encryption_keys = await self.kms.get_user_keys(user_id)

            self.audit.log_event(
                "CRYPTO_ERASURE_INITIATED",
                deletion_id,
                f"Found {len(encryption_keys)} encryption keys for user {user_id}"
            )

            # Step 2: Verify all data is encrypted
            unencrypted_data = await self.verify_data_encryption(
                user_id,
                data_locations
            )

            if unencrypted_data:
                raise Exception(
                    f"Found {len(unencrypted_data)} unencrypted data locations. "
                    "Cryptographic erasure requires all data to be encrypted."
                )

            # Step 3: Destroy each encryption key
            for key_info in encryption_keys:
                destruction_result = await self.destroy_encryption_key(
                    key_info["keyId"],
                    deletion_id
                )

                erasure_report["keysDestroyed"].append({
                    "keyId": key_info["keyId"],
                    "algorithm": key_info["algorithm"],
                    "destructionTimestamp": destruction_result["timestamp"],
                    "destructionMethod": destruction_result["method"],
                    "proof": destruction_result["proof"]
                })

                self.audit.log_event(
                    "ENCRYPTION_KEY_DESTROYED",
                    deletion_id,
                    f"Key {key_info['keyId']} destroyed successfully"
                )

            # Step 4: Update data location metadata
            for location in data_locations:
                await self.mark_data_as_inaccessible(
                    location["systemId"],
                    location["dataPath"],
                    deletion_id
                )

                erasure_report["dataLocations"].append({
                    "systemId": location["systemId"],
                    "dataPath": location["dataPath"],
                    "status": "INACCESSIBLE",
                    "timestamp": datetime.utcnow().isoformat()
                })

            # Step 5: Generate cryptographic proof of erasure
            erasure_report["verificationProof"] = (
                self.generate_erasure_proof(erasure_report)
            )

            erasure_report["executionStatus"] = "COMPLETED"
            erasure_report["completionTimestamp"] = datetime.utcnow().isoformat()

            self.audit.log_event(
                "CRYPTO_ERASURE_COMPLETED",
                deletion_id,
                f"Successfully destroyed {len(encryption_keys)} keys"
            )

        except Exception as e:
            erasure_report["executionStatus"] = "FAILED"
            erasure_report["errorMessage"] = str(e)
            erasure_report["failureTimestamp"] = datetime.utcnow().isoformat()

            self.audit.log_event(
                "CRYPTO_ERASURE_FAILED",
                deletion_id,
                f"Error: {str(e)}"
            )

        return erasure_report

    async def destroy_encryption_key(self, key_id: str,
                                    deletion_id: str) -> Dict:
        """
        Securely destroy an encryption key and generate proof
        """
        # Retrieve key metadata before destruction
        key_metadata = await self.kms.get_key_metadata(key_id)

        # Perform secure overwrite of key material
        destruction_timestamp = datetime.utcnow()

        # Method 1: Overwrite key with random data
        random_data = secrets.token_bytes(32)  # 256 bits
        await self.kms.overwrite_key(key_id, random_data)

        # Method 2: Mark key as destroyed in KMS
        await self.kms.mark_key_destroyed(key_id, deletion_id)

        # Method 3: Remove key from all active caches
        await self.kms.purge_key_from_caches(key_id)

        # Generate proof of destruction
        proof_data = {
            "keyId": key_id,
            "destructionTimestamp": destruction_timestamp.isoformat(),
            "deletionId": deletion_id,
            "previousKeyHash": key_metadata["keyHash"],
            "destructionMethod": "SECURE_OVERWRITE"
        }

        proof_hash = hashlib.sha256(
            str(proof_data).encode()
        ).hexdigest()

        return {
            "keyId": key_id,
            "timestamp": destruction_timestamp.isoformat(),
            "method": "SECURE_OVERWRITE",
            "proof": proof_hash
        }

    async def verify_data_encryption(self, user_id: str,
                                    data_locations: List[Dict]) -> List[Dict]:
        """
        Verify that all user data is properly encrypted
        """
        unencrypted_data = []

        for location in data_locations:
            encryption_status = await self.check_encryption_status(
                location["systemId"],
                location["dataPath"]
            )

            if not encryption_status["encrypted"]:
                unencrypted_data.append({
                    "systemId": location["systemId"],
                    "dataPath": location["dataPath"],
                    "reason": encryption_status["reason"]
                })

        return unencrypted_data

    def generate_erasure_proof(self, erasure_report: Dict) -> str:
        """
        Generate cryptographic proof of successful erasure
        """
        proof_components = {
            "deletionId": erasure_report["deletionId"],
            "userId": erasure_report["userId"],
            "keysDestroyed": len(erasure_report["keysDestroyed"]),
            "timestamp": erasure_report.get("completionTimestamp"),
            "keyHashes": [
                k["proof"] for k in erasure_report["keysDestroyed"]
            ]
        }

        # Create composite hash as proof
        proof_string = str(proof_components)
        proof_hash = hashlib.sha256(proof_string.encode()).hexdigest()

        return f"CRYPTO-ERASURE-{proof_hash[:16].upper()}"

2.6 Implementation: Multi-Pass Overwrite Service

For scenarios requiring physical data destruction on storage media, multi-pass overwriting ensures data irretrievability:

import asyncio
import os
import hashlib
from typing import List, Dict, Optional
from datetime import datetime
import random

class MultiPassOverwriteService:
    """
    Implements DoD 5220.22-M and other multi-pass overwrite standards
    for secure data destruction on physical media
    """

    def __init__(self, audit_logger):
        self.audit = audit_logger
        self.standards = {
            "DOD_3_PASS": 3,
            "DOD_7_PASS": 7,
            "GUTMANN_35_PASS": 35,
            "NIST_CLEAR": 1,
            "NIST_PURGE": 3
        }

    async def execute_overwrite(self, file_path: str,
                               standard: str = "DOD_7_PASS",
                               deletion_id: str = None) -> Dict:
        """
        Execute multi-pass overwrite on specified file or directory
        """
        if not os.path.exists(file_path):
            raise FileNotFoundError(f"Path not found: {file_path}")

        num_passes = self.standards.get(standard, 7)

        overwrite_report = {
            "deletionId": deletion_id,
            "filePath": file_path,
            "overwriteStandard": standard,
            "numberOfPasses": num_passes,
            "initiationTimestamp": datetime.utcnow().isoformat(),
            "passResults": []
        }

        try:
            # Get file size for progress tracking
            file_size = os.path.getsize(file_path)

            self.audit.log_event(
                "OVERWRITE_INITIATED",
                deletion_id,
                f"Starting {num_passes}-pass overwrite of {file_path} ({file_size} bytes)"
            )

            # Execute each overwrite pass
            for pass_number in range(1, num_passes + 1):
                pass_result = await self.execute_single_pass(
                    file_path,
                    pass_number,
                    num_passes,
                    file_size
                )

                overwrite_report["passResults"].append(pass_result)

                self.audit.log_event(
                    "OVERWRITE_PASS_COMPLETED",
                    deletion_id,
                    f"Pass {pass_number}/{num_passes} completed for {file_path}"
                )

            # Final deletion of file
            os.remove(file_path)

            overwrite_report["executionStatus"] = "COMPLETED"
            overwrite_report["completionTimestamp"] = datetime.utcnow().isoformat()
            overwrite_report["fileRemoved"] = True

            # Generate verification hash
            overwrite_report["verificationHash"] = self.generate_verification_hash(
                overwrite_report
            )

            self.audit.log_event(
                "OVERWRITE_COMPLETED",
                deletion_id,
                f"Successfully overwrote and deleted {file_path}"
            )

        except Exception as e:
            overwrite_report["executionStatus"] = "FAILED"
            overwrite_report["errorMessage"] = str(e)
            overwrite_report["failureTimestamp"] = datetime.utcnow().isoformat()

            self.audit.log_event(
                "OVERWRITE_FAILED",
                deletion_id,
                f"Error overwriting {file_path}: {str(e)}"
            )

        return overwrite_report

    async def execute_single_pass(self, file_path: str, pass_number: int,
                                 total_passes: int, file_size: int) -> Dict:
        """
        Execute a single overwrite pass using specified pattern
        """
        pass_start = datetime.utcnow()

        # Determine overwrite pattern based on pass number
        pattern = self.get_overwrite_pattern(pass_number, total_passes)

        # Open file in binary write mode
        with open(file_path, 'r+b') as f:
            bytes_written = 0
            chunk_size = 1024 * 1024  # 1MB chunks

            while bytes_written < file_size:
                chunk = self.generate_pattern_data(
                    pattern,
                    min(chunk_size, file_size - bytes_written)
                )

                f.write(chunk)
                bytes_written += len(chunk)

            # Force write to physical media
            f.flush()
            os.fsync(f.fileno())

        pass_duration = (datetime.utcnow() - pass_start).total_seconds()

        return {
            "passNumber": pass_number,
            "pattern": pattern,
            "bytesWritten": bytes_written,
            "duration": pass_duration,
            "timestamp": datetime.utcnow().isoformat()
        }

    def get_overwrite_pattern(self, pass_number: int,
                             total_passes: int) -> str:
        """
        Determine overwrite pattern based on DoD 5220.22-M standard
        """
        if total_passes == 3:  # DOD 3-pass
            patterns = ["ZEROS", "ONES", "RANDOM"]
            return patterns[pass_number - 1]
        elif total_passes == 7:  # DOD 7-pass
            patterns = ["RANDOM", "ZEROS", "ONES", "RANDOM",
                       "ZEROS", "ONES", "RANDOM"]
            return patterns[pass_number - 1]
        else:
            return "RANDOM"

    def generate_pattern_data(self, pattern: str, size: int) -> bytes:
        """
        Generate overwrite data based on pattern type
        """
        if pattern == "ZEROS":
            return b'\x00' * size
        elif pattern == "ONES":
            return b'\xFF' * size
        elif pattern == "RANDOM":
            return os.urandom(size)
        else:
            return os.urandom(size)

    def generate_verification_hash(self, overwrite_report: Dict) -> str:
        """
        Generate verification hash for overwrite operation
        """
        hash_data = {
            "deletionId": overwrite_report["deletionId"],
            "filePath": overwrite_report["filePath"],
            "passes": overwrite_report["numberOfPasses"],
            "completionTimestamp": overwrite_report["completionTimestamp"]
        }

        hash_string = str(hash_data)
        return hashlib.sha256(hash_string.encode()).hexdigest()

2.7 Backup System Scrubbing

Comprehensive data deletion requires scrubbing data from backup systems and archives:

import asyncio
from typing import List, Dict, Optional
from datetime import datetime, timedelta

class BackupScrubService:
    """
    Manages deletion of user data from backup systems and archives
    """

    def __init__(self, backup_manager, audit_logger):
        self.backup_mgr = backup_manager
        self.audit = audit_logger

    async def scrub_user_from_backups(self, user_id: str,
                                      deletion_id: str,
                                      retention_cutoff: Optional[datetime] = None
                                      ) -> Dict:
        """
        Remove user data from all backup systems
        """
        scrub_report = {
            "deletionId": deletion_id,
            "userId": user_id,
            "initiationTimestamp": datetime.utcnow().isoformat(),
            "backupsProcessed": [],
            "totalBackupsScanned": 0,
            "totalBackupsModified": 0,
            "totalRecordsRemoved": 0
        }

        try:
            # Get list of all backup sets
            backup_sets = await self.backup_mgr.list_backup_sets()

            scrub_report["totalBackupsScanned"] = len(backup_sets)

            self.audit.log_event(
                "BACKUP_SCRUB_INITIATED",
                deletion_id,
                f"Scanning {len(backup_sets)} backup sets for user {user_id}"
            )

            # Process each backup set
            for backup_set in backup_sets:
                # Skip backups older than retention cutoff if specified
                if retention_cutoff and backup_set["timestamp"] < retention_cutoff:
                    continue

                backup_result = await self.scrub_backup_set(
                    backup_set,
                    user_id,
                    deletion_id
                )

                if backup_result["recordsRemoved"] > 0:
                    scrub_report["totalBackupsModified"] += 1
                    scrub_report["totalRecordsRemoved"] += backup_result["recordsRemoved"]

                scrub_report["backupsProcessed"].append(backup_result)

            scrub_report["executionStatus"] = "COMPLETED"
            scrub_report["completionTimestamp"] = datetime.utcnow().isoformat()

            self.audit.log_event(
                "BACKUP_SCRUB_COMPLETED",
                deletion_id,
                f"Scrubbed {scrub_report['totalRecordsRemoved']} records from "
                f"{scrub_report['totalBackupsModified']} backup sets"
            )

        except Exception as e:
            scrub_report["executionStatus"] = "FAILED"
            scrub_report["errorMessage"] = str(e)

            self.audit.log_event(
                "BACKUP_SCRUB_FAILED",
                deletion_id,
                f"Error: {str(e)}"
            )

        return scrub_report

    async def scrub_backup_set(self, backup_set: Dict, user_id: str,
                              deletion_id: str) -> Dict:
        """
        Scrub user data from a single backup set
        """
        result = {
            "backupId": backup_set["id"],
            "backupTimestamp": backup_set["timestamp"],
            "recordsRemoved": 0,
            "scrubTimestamp": datetime.utcnow().isoformat()
        }

        # Check if backup contains user data
        contains_user_data = await self.backup_mgr.check_user_in_backup(
            backup_set["id"],
            user_id
        )

        if not contains_user_data:
            result["action"] = "SKIPPED"
            return result

        # Remove user data from backup
        records_removed = await self.backup_mgr.remove_user_from_backup(
            backup_set["id"],
            user_id
        )

        result["recordsRemoved"] = records_removed
        result["action"] = "SCRUBBED"

        return result

2.8 Data Deletion Certification Requirements

Organizations must meet comprehensive certification requirements for data deletion protocols:

Technical Implementation (Items 1-8)

  • Support for multiple deletion methods (soft, hard, cryptographic, overwrite)
  • Cryptographic erasure capability with key destruction verification
  • Multi-pass overwrite implementation (DOD 5220.22-M or equivalent)
  • Backup system integration for complete data scrubbing
  • Log file scrubbing to remove personal data from audit trails
  • Cache and temporary storage purging mechanisms
  • Search index and CDN content removal capabilities
  • Automated verification scanning post-deletion

Process Requirements (Items 9-15)

  • Documented deletion protocol selection criteria by data classification
  • Scheduled deletion capability for retention policy compliance
  • Grace period implementation for accidental deletion recovery
  • Rollback procedures for failed deletion operations
  • Emergency deletion capability for security incidents
  • Batch deletion support for large-scale erasure operations
  • Cascading deletion for related data across systems

Verification Requirements (Items 16-20)

  • Automated post-deletion verification across all systems
  • Cryptographic proof generation for verified deletions
  • Third-party audit support and independent verification
  • Forensic analysis capability for deletion confirmation
  • Deletion certificate generation with tamper-proof signatures

Compliance Standards (Items 21-25)

  • NIST SP 800-88 compliance for media sanitization
  • DoD 5220.22-M standard implementation for overwriting
  • GDPR Article 17 technical requirements compliance
  • HIPAA § 164.530(c) secure disposal requirements
  • PCI DSS 3.1 secure deletion and media destruction standards

2.9 Best Practices for Data Deletion

Implementing effective data deletion protocols requires adherence to industry best practices:

  • Classify data according to sensitivity and apply appropriate deletion methods
  • Implement "delete by default" architectures using cryptographic erasure where possible
  • Maintain comprehensive data inventories with deletion method specifications
  • Test deletion procedures regularly through simulation and verification
  • Document all deletion operations with immutable audit trails
  • Implement automated verification to confirm deletion completeness
  • Consider data dependencies and cascading effects before deletion
  • Provide user notifications before irreversible deletion operations
  • Maintain deletion certificates for compliance and audit purposes
  • Regular review and update of deletion protocols based on new threats and standards

2.10 Summary

Data deletion protocols form the technical foundation for digital erasure compliance. Organizations must implement a comprehensive suite of deletion methods, from soft deletes for temporary scenarios to cryptographic erasure and physical destruction for permanent removal. The WIA 디지털 삭제 표준 provides detailed specifications for each deletion approach, ensuring organizations can select and implement appropriate protocols based on data sensitivity, storage infrastructure, and regulatory requirements.

By following these protocols and meeting certification requirements, organizations can ensure data is permanently and verifiably removed from all systems, providing users with confidence in their privacy rights while maintaining compliance with global data protection regulations.
