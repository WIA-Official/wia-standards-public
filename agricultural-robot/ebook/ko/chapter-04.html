弘益人間 (홍익인간) - 널리 인간을 이롭게 하라

4.1 데이터 형식 표준화의 중요성

1단계 데이터 형식 표준화는 WIA-ROB-007의 기초입니다. 모든 농업 로봇이 동일한 데이터 구조, 단위, 정밀도를 사용하면 서로 다른 제조사의 로봇 간에도 데이터를 원활하게 교환하고 통합 분석할 수 있습니다.

데이터 형식의 일관성은 상호운용성의 첫 걸음입니다. 아무리 우수한 API나 통신 프로토콜을 사용해도, 데이터 자체가 표준화되지 않으면 의미 있는 정보 교환이 불가능합니다.

4.2 JSON 기반 데이터 구조

WIA-ROB-007은 데이터 교환 형식으로 JSON (JavaScript Object Notation)을 사용합니다. JSON은 가볍고, 사람이 읽기 쉬우며, 대부분의 프로그래밍 언어에서 지원되는 검증된 형식입니다.

4.2.1 기본 메시지 구조

모든 WIA-ROB-007 데이터 메시지는 다음 기본 구조를 따릅니다. 

{
    "version": "1.0",
    "message_id": "msg-20250115-143000-001",
    "timestamp": "2025-01-15T14:30:00.000Z",
    "robot_id": "ROB-FARM-001",
    "farm_id": "FARM-KR-SEOUL-001",
    "message_type": "telemetry | command | event | response",
    "payload": {
        // 메시지 타입별 실제 데이터
    }
}

4.2.2 필수 필드 설명

필드명 타입 설명 예시
version string 표준 버전 "1.0"
message_id string 메시지 고유 식별자 "msg-20250115-143000-001"
timestamp ISO 8601 UTC 기준 시각 "2025-01-15T14:30:00.000Z"
robot_id string 로봇 고유 식별자 "ROB-FARM-001"
farm_id string 농장 고유 식별자 "FARM-KR-SEOUL-001"
message_type enum 메시지 유형 "telemetry"

4.6 표준 단위 시스템

일관된 측정 단위 사용은 데이터 통합에 필수적입니다. WIA-ROB-007은 SI 단위계를 기본으로 하되, 농업에서 통용되는 단위도 명시적으로 지정합니다.

측정 항목 표준 단위 필드명 접미사 예시

4.7 데이터 정밀도와 품질

정확한 데이터는 올바른 의사결정의 기반입니다. WIA-ROB-007은 데이터 정밀도와 품질에 대한 명확한 지침을 제공합니다.

4.7.1 정밀도 요구사항

데이터 유형 정밀도 예시

무게 소수점 2자리 375.50

4.9 데이터 검증 스키마

데이터 무결성을 보장하기 위해 JSON Schema를 사용한 검증을 권장합니다. 

{
    "$schema": "http://json-schema.org/draft-07/schema#",
    "title": "WIA-ROB-007 Soil Sensor Data",
    "type": "object",
    "required": ["version", "message_id", "timestamp", "robot_id", "payload"],
    "properties": {
        "version": {
            "type": "string",
            "pattern": "^[0-9]+\\.[0-9]+$"
        },
        "message_id": {
            "type": "string"
        },
        "timestamp": {
            "type": "string",
            "format": "date-time"
        },
        "robot_id": {
            "type": "string"
        },
        "payload": {
            "type": "object",
            "required": ["sensor_type", "measurements"],
            "properties": {
                "sensor_type": {
                    "type": "string",
                    "enum": ["soil", "weather", "crop_vision"]
                },
                "measurements": {
                    "type": "object",
                    "properties": {
                        "temperature_celsius": {
                            "type": "number",
                            "minimum": -50,
                            "maximum": 100
                        },
                        "moisture_percent": {
                            "type": "number",
                            "minimum": 0,
                            "maximum": 100
                        }
                    }
                }
            }
        }
    }
}

4.10 데이터 압축 및 전송 최적화

대량의 데이터를 효율적으로 전송하기 위한 최적화 방법도 표준에 포함됩니다.

4.10.1 배치 전송

{
    "version": "1.0",
    "message_id": "msg-batch-20250115-143000-001",
    "timestamp": "2025-01-15T14:30:00.000Z",
    "robot_id": "ROB-FARM-001",
    "farm_id": "FARM-KR-SEOUL-001",
    "message_type": "telemetry_batch",
    "payload": {
        "batch_info": {
            "count": 100,
            "start_time": "2025-01-15T14:00:00.000Z",
            "end_time": "2025-01-15T14:30:00.000Z"
        },
        "measurements": [
            {
                "timestamp": "2025-01-15T14:00:00.000Z",
                "temperature_celsius": 22.5,
                "moisture_percent": 65.0
            },
            // ... 98개 더
        ]
    }
}

데이터 최적화 전략

  • 압축: gzip 또는 brotli를 사용한 JSON 압축
  • 배치 전송: 여러 측정값을 묶어서 전송
  • 델타 인코딩: 변경된 값만 전송
  • 샘플링: 중요하지 않은 데이터는 샘플링하여 전송
  • 로컬 집계: 로봇에서 통계 처리 후 요약 데이터 전송

핵심 내용 요약

  • WIA-ROB-007은 JSON을 기본 데이터 형식으로 사용
  • 모든 메시지는 version, message_id, timestamp, robot_id 등 필수 필드 포함
  • 센서 데이터는 토양, 기상, 작물 등 유형별로 표준 스키마 정의
  • SI 단위계 기반의 표준 단위 시스템 사용
  • 필드명 접미사로 단위를 명시 (예: temperature_celsius)
  • 정밀도 요구사항과 품질 메타데이터로 데이터 신뢰성 보장
  • 에러 및 예외 상황도 표준화된 형식으로 보고
  • JSON Schema를 통한 데이터 검증 권장
  • 배치 전송, 압축 등으로 전송 효율성 최적화
