제6장: 3단계: 통신 프로토콜

弘益人間 (홍익인간) - 널리 인간을 이롭게 하라

6.1 통신 프로토콜의 필요성

농업 로봇은 실시간으로 데이터를 전송하고 명령을 받아야 합니다. HTTP 기반 REST API는 요청-응답 방식이므로 실시간 양방향 통신에는 적합하지 않습니다. WIA-ROB-007의 3단계는 실시간 통신을 위한 프로토콜을 정의합니다.

농장 환경은 네트워크 연결이 불안정할 수 있으며, 수십 대의 로봇이 동시에 통신할 수 있습니다. 따라서 경량화되고 신뢰성 있으며 확장 가능한 프로토콜이 필요합니다.

6.2 MQTT 프로토콜

WIA-ROB-007은 MQTT (Message Queuing Telemetry Transport)를 기본 통신 프로토콜로 채택합니다. MQTT는 IoT 환경에 최적화된 경량 프로토콜로, 다음과 같은 장점이 있습니다.

MQTT의 장점

  • 경량성: 헤더 크기가 작아 대역폭 사용 최소화
  • 신뢰성: QoS(Quality of Service) 레벨로 메시지 전달 보장
  • Pub/Sub 모델: 느슨한 결합으로 확장성 향상
  • 지속 연결: Keep-alive로 연결 상태 유지
  • Last Will: 연결 끊김 시 자동 알림
  • 저전력: 배터리 구동 로봇에 적합

6.2.1 MQTT 아키텍처

MQTT는 발행-구독(Publish-Subscribe) 패턴을 사용합니다. 로봇(클라이언트)은 브로커(Broker)를 통해 메시지를 주고받습니다. 

┌─────────────┐         ┌─────────────┐         ┌─────────────┐
│  Robot A    │────────▶│   MQTT      │◀────────│  Robot B    │
│ (Publisher) │         │  Broker     │         │(Subscriber) │
└─────────────┘         └─────────────┘         └─────────────┘
      │                       │                       │
      │                       │                       │
      └───────────────────────┴───────────────────────┘
                    Farm Management System

6.3 토픽 구조

WIA-ROB-007은 계층적 토픽 구조를 정의하여 메시지를 체계적으로 관리합니다.

6.3.1 표준 토픽 계층

wia-rob-007/{farm_id}/{category}/{robot_id}/{data_type}

예시:
wia-rob-007/FARM-001/robots/ROB-001/status
wia-rob-007/FARM-001/robots/ROB-001/telemetry
wia-rob-007/FARM-001/robots/ROB-001/command
wia-rob-007/FARM-001/farm/weather
wia-rob-007/FARM-001/system/alerts

6.3.2 주요 토픽 카테고리

카테고리 토픽 패턴 설명 방향
로봇 상태 robots/{robot_id}/status 로봇 상태 정보 Robot → Server
센서 데이터 robots/{robot_id}/telemetry 센서 측정값 Robot → Server
명령 robots/{robot_id}/command 제어 명령 Server → Robot
응답 robots/{robot_id}/response 명령 응답 Robot → Server
이벤트 robots/{robot_id}/events 특별 이벤트 Robot → Server
농장 정보 farm/{farm_id}/weather 농장 전체 데이터 Broadcast
조정 coordination/{area_id} 로봇 간 조정 Bidirectional
시스템 system/{type} 시스템 메시지 Broadcast

6.4 QoS (Quality of Service) 레벨

MQTT는 3가지 QoS 레벨을 제공하여 메시지 전달 보장 수준을 선택할 수 있습니다.

QoS 이름 보장 수준 사용 예
0 At most once 최대 1회 전달 (손실 가능) 빈번한 센서 데이터
1 At least once 최소 1회 전달 (중복 가능) 중요 상태 정보
2 Exactly once 정확히 1회 전달 제어 명령, 작업 지시

6.4.1 WIA-ROB-007 QoS 권장 사항

  • QoS 0: 센서 텔레메트리 (1초 이하 간격)
  • QoS 1: 로봇 상태, 이벤트 알림
  • QoS 2: 제어 명령, 작업 시작/정지, 긴급 정지

6.6 Retained Messages와 Last Will

6.6.1 Retained Messages

Retained 플래그를 사용하면 새로 구독하는 클라이언트가 최신 상태를 즉시 받을 수 있습니다. 

// 로봇 상태는 Retained로 발행
Topic: wia-rob-007/FARM-001/robots/ROB-001/status
Retained: true
Payload: { "operational_state": "working", ... }

// 새 클라이언트가 구독하면 즉시 마지막 상태 수신

6.6.2 Last Will and Testament (LWT)

로봇 연결이 예기치 않게 끊어졌을 때 자동으로 알림을 보냅니다. 

// MQTT 연결 시 LWT 설정
Will Topic: wia-rob-007/FARM-001/robots/ROB-001/status
Will QoS: 1
Will Retain: true
Will Payload:
{
    "version": "1.0",
    "timestamp": "2025-01-15T14:30:00.000Z",
    "robot_id": "ROB-FARM-001",
    "operational_state": "disconnected",
    "reason": "connection_lost"
}

6.7 WebSocket 프로토콜

웹 기반 대시보드나 모바일 앱을 위해 WIA-ROB-007은 WebSocket도 지원합니다. WebSocket은 브라우저에서 쉽게 사용할 수 있는 실시간 양방향 통신 프로토콜입니다.

// 구독 메시지

    "type": "subscribe",

        "robots/ROB-001/status",

6.8 보안 및 암호화

모든 통신은 암호화되어야 하며, 인증된 클라이언트만 접속할 수 있어야 합니다.

6.8.1 TLS/SSL 암호화

  • MQTT over TLS: MQTTS (포트 8883) 사용 필수
  • WebSocket Secure: WSS 프로토콜 사용
  • TLS 버전: TLS 1.2 이상 (TLS 1.3 권장)
  • 인증서: CA 서명 인증서 또는 자체 서명 인증서

6.8.2 클라이언트 인증

인증 방법 설명 보안 수준
Username/Password 사용자명과 비밀번호 기본
Client Certificate X.509 클라이언트 인증서 높음
Token-based JWT 토큰 중간

6.8.3 MQTT 연결 보안 설정

// Python paho-mqtt 예제
import paho.mqtt.client as mqtt
import ssl

client = mqtt.Client(client_id="ROB-FARM-001")

# TLS 설정
client.tls_set(
    ca_certs="/path/to/ca.crt",
    certfile="/path/to/client.crt",
    keyfile="/path/to/client.key",
    tls_version=ssl.PROTOCOL_TLSv1_2
)

# 인증
client.username_pw_set("robot001", "secure_password")

# 연결
client.connect("mqtt.farm.example.com", 8883, 60)

6.11 에러 처리 및 재연결

네트워크 장애 시 자동으로 재연결하고 메시지를 재전송해야 합니다.

6.11.1 재연결 전략

// 지수 백오프(Exponential Backoff) 재연결
def reconnect_with_backoff():
    retry_count = 0
    max_retries = 10
    base_delay = 1  # 1초

    while retry_count < max_retries:
        try:
            client.reconnect()
            return True
        except:
            delay = min(base_delay * (2 ** retry_count), 300)  # 최대 5분
            time.sleep(delay)
            retry_count += 1

    return False

6.11.2 메시지 버퍼링

// 연결 끊김 시 메시지를 로컬에 저장
// 재연결 후 순서대로 전송

class BufferedMQTTClient:
    def __init__(self):
        self.buffer = []
        self.max_buffer_size = 1000

    def publish_with_buffer(self, topic, payload, qos):
        try:
            self.client.publish(topic, payload, qos)
        except:
            if len(self.buffer) < self.max_buffer_size:
                self.buffer.append((topic, payload, qos))

    def flush_buffer(self):
        while self.buffer:
            topic, payload, qos = self.buffer.pop(0)
            self.client.publish(topic, payload, qos)

핵심 내용 요약

  • WIA-ROB-007은 MQTT를 기본 실시간 통신 프로토콜로 사용
  • 계층적 토픽 구조: wia-rob-007/{farm_id}/{category}/{robot_id}/{type}
  • QoS 0/1/2를 용도에 따라 선택하여 신뢰성과 효율성 균형
  • Retained 메시지로 최신 상태 유지, LWT로 연결 끊김 감지
  • WebSocket도 지원하여 웹/모바일 앱 통합 용이
  • TLS/SSL 암호화 필수, 클라이언트 인증으로 보안 보장
  • 페이로드 압축과 배치 전송으로 대역폭 최적화
  • Keep-Alive와 하트비트로 연결 상태 모니터링
  • 지수 백오프 재연결과 메시지 버퍼링으로 안정성 확보
