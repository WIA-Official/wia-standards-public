제2장: P2P 아키텍처

"홍익인간: 분산 네트워크는 모든 사람이 자신만의 호스트가 될 수 있게 합니다."

2.1 개인 웹 서버

내 PC가 서버
============

기존 서버:
  - 데이터센터에서 24/7 운영
  - 고가의 하드웨어
  - 회사가 관리
  - 당신이 임대료 지불

WIA-HOME 서버:
  - 내 PC/노트북에서 실행
  - 최소한의 자원 사용
  - 내가 모든 것 통제
  - 비용 없음


서버 구성요소
=============

1. HTTP 서버
   - 웹 페이지 제공
   - 요청 처리
   - 저자원 최적화

2. P2P 노드
   - 메시 네트워크 연결
   - NAT 트래버설 처리
   - 발견 가능하게 함

3. 콘텐츠 매니저
   - 페이지 저장
   - 미디어 관리
   - 캐싱 처리

4. 보안 레이어
   - TLS 암호화
   - DDoS 방어
   - 방화벽


자원 사용량
===========

최소 요구사항:
  CPU:     활성 시 5-10%
  메모리:  128-256 MB
  저장공간: 100 MB + 내 콘텐츠
  네트워크: 1 Mbps 업로드

오래된 노트북도 가능합니다!


절전 모드
=========

방문자가 없을 때:
  - 서버가 절전 모드 진입
  - 거의 자원 사용 안 함
  - 요청 들어오면 깨어남
  - 방문자에게는 자연스러움

에너지 효율적인 호스팅.

2.2 P2P 메시 네트워크

네트워크 토폴로지
=================

      [피어 A]--------[피어 B]
        /  \            /  \
       /    \          /    \
      /      \        /      \
 [피어 C]----[내 PC]----[피어 D]
      \      /        \      /
       \    /          \    /
        \  /            \  /
      [피어 E]--------[피어 F]
              \      /
               \    /
             [릴레이 노드]


중앙 서버 없음
==============

기존:
  모든 트래픽 -> 중앙 서버 -> 사용자
  서버 다운 = 웹사이트 다운

WIA-HOME:
  트래픽이 네트워크에 분산
  내 PC 꺼져도 = 캐시된 콘텐츠 제공
  네트워크가 복원력 있음


피어 발견
=========

1. 부트스트랩
   - 알려진 피어에 연결
   - 피어 목록 받기
   - 네트워크 참여

2. DHT (분산 해시 테이블)
   - 도메인 매핑 저장
   - 중앙 DNS 불필요
   - 장애에 복원력 있음

3. 가십 프로토콜
   - 피어 정보 공유
   - 상태 모니터링
   - 부하 분산

2.3 NAT 트래버설

NAT 문제
========

대부분의 가정 네트워크는 NAT 사용:
  - 내부 IP: 192.168.1.x
  - 외부 IP: 단일 공인 IP
  - 들어오는 연결 차단됨

NAT를 뚫어야 합니다.


NAT 유형
========

Open (10%):
  [OK] 직접 연결 가능
  [OK] 특별한 처리 불필요

Full Cone (30%):
  [OK] 포트 포워딩 작동
  [OK] 예측 가능한 포트
  [i] 약간의 설정 필요

Restricted (40%):
  [i] 홀 펀칭 필요
  [i] 조율 필요
  [OK] 보통 작동

Symmetric (20%):
  [!] 가장 제한적
  [!] 연결마다 다른 포트
  [!] 릴레이 필요


트래버설 기술
=============

1. STUN (세션 트래버설)
   - 외부 IP/포트 발견
   - 대부분의 NAT에서 작동
   - 낮은 오버헤드

2. TURN (릴레이)
   - 릴레이 서버 사용
   - 모든 NAT에서 작동
   - 더 높은 지연시간

3. ICE (대화형 연결)
   - 모든 방법 시도
   - 최적 옵션 선택
   - 자동 폴백


WIA 구현
========

자동 감지:
  1. NAT 유형 감지
  2. 직접 연결 시도
  3. 홀 펀칭 시도
  4. 릴레이로 폴백

사용자가 보는 것:
  "사이트가 라이브입니다!"
  (모든 복잡성 숨김)

2.4 WIA DNS

도메인 시스템
=============

기존 DNS:
  - 중앙화된 서버
  - 연간 갱신 비용
  - 압류/차단 가능
  - 등록기관에 종속

WIA DNS:
  - 네트워크에 분산
  - 영원히 무료
  - 검열 저항성
  - 내가 통제


도메인 형식
===========

yourname.wia.home
  - 개인 사이트
  - 포트폴리오
  - 블로그

yourname.wia.shop
  - 전자상거래
  - 비즈니스 사이트
  - 상품 카탈로그

yourname.wia.blog
  - 블로그 중심
  - 콘텐츠 사이트
  - 출판물

yourname.wia.page
  - 단일 페이지
  - 랜딩 페이지
  - 프로젝트 페이지


등록
====

도메인 등록:
  1. 이름 선택
  2. 키쌍 생성
  3. 등록 서명
  4. 네트워크에 브로드캐스트

소유권:
  - 암호화 키로 증명
  - 중앙 기관 없음
  - 빼앗길 수 없음


커스텀 도메인 (선택)
====================

이미 도메인이 있다면?
  - DNS를 WIA 네트워크로 지정
  - 두 주소 모두 작동
  - 점진적 마이그레이션 가능

2.5 콘텐츠 배포

CDN 없는 CDN
============

기존 CDN:
  - 전 세계 엣지 서버
  - 고비용 (월 5-50만원)
  - 제공업체 종속

WIA P2P 배포:
  - 피어가 인기 콘텐츠 캐시
  - 자동 배포
  - 무료이고 탈중앙화


작동 방식
=========

1. 첫 번째 요청
   방문자 -> 내 PC -> 콘텐츠

2. 이후 요청 (같은 지역)
   방문자 -> 근처 피어 (캐시됨) -> 콘텐츠

3. 내가 오프라인일 때
   방문자 -> 여러 피어 (캐시됨) -> 콘텐츠


캐시 전략
=========

인기 콘텐츠 (핫):
  - 많은 피어에 캐시됨
  - 빠른 전달
  - 높은 가용성

비인기 콘텐츠 (콜드):
  - 원본에서 직접
  - 온디맨드 캐싱
  - 여전히 접근 가능


이점
====

[OK] 빠른 전달 (근처 피어)
[OK] 높은 가용성 (중복성)
[OK] 무료 (CDN 비용 없음)
[OK] 자동 확장
[OK] 내가 자는 동안에도 작동

장 요약

  • [OK] 개인 서버가 최소 자원으로 내 PC에서 실행
  • [OK] P2P 메시 네트워크가 중앙 장애 지점 제거
  • [OK] NAT 트래버설이 네트워크 복잡성 자동 처리
  • [OK] WIA DNS가 무료, 검열 저항성 도메인 제공
  • [OK] P2P 콘텐츠 배포가 비싼 CDN 대체
