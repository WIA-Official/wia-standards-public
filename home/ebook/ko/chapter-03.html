제3장: 사이트 생성

"홍익인간: 웹사이트 만들기는 원하는 것을 설명하는 것만큼 쉬워야 합니다."

3.1 클론 및 커스터마이징

벤치마킹 모드
=============

마음에 드는 웹사이트가 있나요?
내 브랜딩으로 클론하세요.

프로세스:
  1. 경쟁사 URL 입력
  2. AI가 구조 분석
  3. 내 정보로 클론
  4. 색상/콘텐츠 커스터마이징
  5. 런칭!


사이트 분석
===========

WIA-HOME이 분석하는 것:

구조:
  - 페이지 (홈, 소개, 연락처 등)
  - 네비게이션 패턴
  - 레이아웃 유형

기능:
  - 쇼핑몰 기능
  - 블로그 기능
  - 문의 폼
  - 예약 시스템

기술:
  - 사용된 프레임워크
  - CMS 종류
  - 호스팅 제공업체

성능:
  - 로드 시간
  - 페이지 크기
  - 최적화 점수


분석 예시
=========

URL: https://유명빵집.com

구조:
  페이지: 8개 감지됨
  - 홈, 소개, 메뉴, 갤러리
  - 문의, 블로그, 주문, FAQ
  레이아웃: 히어로 있는 싱글 컬럼
  네비게이션: 수평 메뉴

기능:
  [OK] 쇼핑몰 (상품 카탈로그)
  [OK] 문의 폼
  [OK] SNS 링크
  [ ] 예약 시스템
  [ ] 뉴스레터

기술:
  프레임워크: React
  호스팅: AWS (~월 5만원)

추천:
  [i] 예약 기능 추가
  [i] 이미지 최적화 (60% 절감)
  [i] SEO용 블로그 추가

3.2 의도 기반 생성

아버지 (INTENT) 연동
====================

원하는 것만 설명하세요.
아버지가 집을 설계합니다.

자연어 입력:
  "나는 동네 빵집을 운영해.
   빵 종류랑 가격 보여주고
   예약 받을 수 있게 해줘.
   인스타그램 연동해줘."

아버지가 생성:
  - 히어로가 있는 홈페이지
  - 상품 카탈로그
  - 예약 시스템
  - 인스타그램 피드 위젯
  - 문의 페이지


프롬프트 예시
=============

빵집:
  "나는 동네 빵집이야.
   빵이랑 케이크 보여줘.
   카카오페이로 온라인 주문.
   영업시간이랑 위치 보여줘."

프리랜서:
  "나는 UI/UX 디자이너야.
   포트폴리오 10개 프로젝트.
   자기소개 섹션.
   견적 문의 폼.
   다크모드, 모던한 느낌."

미용실:
  "강남 미용실.
   작업 포트폴리오 보여줘.
   온라인 예약 시스템.
   스태프 소개.
   인스타그램 링크."


작동 방식
=========

1. 의도 파싱
   - 사업 유형 추출
   - 필요한 기능 식별
   - 스타일 선호도 이해

2. 템플릿 선택
   - 업종에 맞춤
   - 기능에 맞게 조정

3. 콘텐츠 생성
   - 페이지 구조 생성
   - 플레이스홀더 콘텐츠 추가
   - 스타일 적용

4. 커스터마이징
   - 사용자 정보 삽입
   - 색상 적용
   - 레이아웃 최적화

5. 검토 및 런칭
   - 사이트 미리보기
   - 조정하기
   - 라이브

3.3 업종별 템플릿

미리 만들어진 템플릿
====================

식당:
  기능: 메뉴, 예약, 배달, 리뷰
  연동: 배민, 요기요, 쿠팡이츠

카페:
  기능: 메뉴, 위치, 인스타그램 피드
  연동: 인스타그램, 네이버 플레이스

빵집:
  기능: 상품, 주문, 결제, 갤러리
  연동: 카카오페이, 네이버페이

미용실:
  기능: 포트폴리오, 예약, 가격표, 스태프
  연동: 네이버 예약, 카카오 헤어샵

병원:
  기능: 진료과, 의료진, 예약
  연동: 네이버 플레이스, 카카오맵

학원:
  기능: 강좌, 시간표, 수강신청
  연동: 네이버 학원

사진작가:
  기능: 포트폴리오, 가격, 예약, 블로그
  연동: 인스타그램

디자이너:
  기능: 포트폴리오, 서비스, 견적 폼
  연동: 비핸스, 드리블


템플릿 선택
===========

템플릿 선택:
  1. 업종 선택
  2. 로케일 선택 (ko-KR, en-US 등)
  3. 기능 선택
  4. 브랜딩 커스터마이징
  5. 런칭

3.4 기존 사이트 마이그레이션

지원 플랫폼
===========

WordPress:
  - 포스트와 페이지
  - 미디어 라이브러리
  - 댓글
  - 상품 (WooCommerce)

Wix:
  - 내보내기 파일 임포트
  - 페이지 구조
  - 이미지와 콘텐츠

네이버 블로그:
  - 모든 포스트
  - 이미지
  - 카테고리

티스토리:
  - 블로그 포스트
  - 첨부파일
  - 댓글


마이그레이션 프로세스
=====================

1. 소스 연결
   - 자격증명 입력 또는
   - 내보내기 파일 업로드

2. 콘텐츠 선택
   - 임포트할 페이지
   - 임포트할 포스트
   - 임포트할 미디어
   - 상품 (해당 시)

3. 콘텐츠 매핑
   - 새 구조에 할당
   - 카테고리 매칭
   - 리다이렉트 처리

4. 임포트
   - 콘텐츠 전송
   - 미디어 다운로드
   - 링크 보존

5. 검토
   - 모든 콘텐츠 확인
   - 문제 수정
   - 필요시 업데이트

3.5 퀵 스타트

5분 웹사이트
============

바쁜 사업주를 위해:

const mySite = await WIAHome.quickStart({
  benchmark: 'https://경쟁사.com',
  myInfo: {
    name: '행복한 빵집',
    phone: '010-1234-5678',
    address: '서울 강남구',
    hours: '09:00-21:00'
  }
});

await mySite.start();
// 라이브: happy-bakery.wia.shop


퀵 스타트가 하는 일
===================

1. 벤치마크 URL 분석 (30초)
2. 구조 클론 (30초)
3. 내 브랜딩 적용 (30초)
4. 내 정보 삽입 (30초)
5. 성능 최적화 (1분)
6. 서버 시작 (30초)

총: ~5분


퀵 스타트 결과
==============

생성됨:

  [OK] 소개 페이지

최적화:
