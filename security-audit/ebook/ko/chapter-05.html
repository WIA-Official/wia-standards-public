제5장: 2단계 - API 인터페이스

RESTful API 설계 및 구현

5.1 API 개요

WIA-SEC-017은 감사 이벤트 관리를 위한 RESTful API를 정의합니다:

카테고리엔드포인트목적
이벤트/events이벤트 제출 및 조회
체인/chain체인 검증 및 상태
검색/search고급 검색
규정 준수/compliance규정 준수 보고
알림/alerts보안 알림

기본 URL

https://audit.example.com/wia-audit/v1

5.2 이벤트 API

POST /events - 이벤트 제출

POST /wia-audit/v1/events
Content-Type: application/json
Authorization: Bearer <token>

{
  "events": [
    {
      "timestamp": "2025-01-15T09:30:45.123Z",
      "actor": {
        "type": "user",
        "id": "user_123",
        "email": "user@example.com"
      },
      "action": {
        "type": "DATA_ACCESS",
        "operation": "READ",
        "resource": "/api/customers/456"
      },
      "outcome": {
        "status": "SUCCESS"
      }
    }
  ]
}

// 응답
{
  "accepted": 1,
  "rejected": 0,
  "events": [
    {
      "eventId": "evt_01HQMXK9J2N4P5R6S7T8V9W0",
      "sequence": 12345,
      "status": "accepted"

}

5.4 검색 API

POST /search - 고급 검색

POST /wia-audit/v1/search
{
  "query": {
    "timeRange": {
      "from": "2025-01-01T00:00:00Z",
      "to": "2025-01-31T23:59:59Z"
    },
    "filters": [
      {"field": "actor.email", "operator": "equals", "value": "admin@example.com"},
      {"field": "action.type", "operator": "in", "value": ["DATA_ACCESS", "AUTHENTICATION"]},
      {"field": "outcome.status", "operator": "equals", "value": "FAILURE"}
    ],
    "sort": [
      {"field": "timestamp", "order": "desc"}
    ]
  },
  "pagination": {
    "limit": 50,
    "offset": 0
  }
}

// 응답
{
  "results": [...],
  "total": 150,
  "aggregations": {
    "byActionType": {
      "DATA_ACCESS": 120,
      "AUTHENTICATION": 30
    }
  }
}

검색 연산자

연산자설명예시
equals정확히 일치status = "SUCCESS"
not_equals불일치status != "SUCCESS"
in목록에 포함type in ["A", "B"]
contains포함email contains "@example"
starts_with시작ip starts_with "192.168"
range범위statusCode range [400, 499]

5.7 인증 및 권한

인증 방법

방법사용 사례헤더
Bearer Token사용자 세션Authorization: Bearer <jwt>
API Key서비스 통합X-API-Key: <key>
mTLS서비스 간 통신클라이언트 인증서

권한 범위

  • audit:write - 이벤트 제출
  • audit:read - 이벤트 조회
  • audit:search - 고급 검색
  • audit:verify - 체인 검증
  • audit:admin - 관리 작업

5.8 장 요약

핵심 내용

  • RESTful API로 감사 이벤트 관리
  • 이벤트 API: 제출, 조회, 목록
  • 체인 API: 헤드, 검증, 스팟 체크
  • 검색 API: 다양한 연산자 지원
  • 규정 준수 API: 상태 및 보고서
  • 알림 API: 보안 이벤트 관리
