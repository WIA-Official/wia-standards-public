7.3 클라우드 플랫폼 통합

AWS 통합

// CloudTrail → WIA-SEC-017
{
  "connector": "aws-cloudtrail",
  "config": {
    "region": "ap-northeast-2",
    "trailArn": "arn:aws:cloudtrail:ap-northeast-2:123456789:trail/main",
    "roleArn": "arn:aws:iam::123456789:role/WIAAuditReader",
    "eventTypes": ["ConsoleLogin", "AssumeRole", "CreateUser"]
  }
}

Kubernetes 통합

# Kubernetes 감사 웹훅 구성
apiVersion: v1
kind: Config
clusters:
- name: wia-audit
  cluster:
    server: https://audit.company.com/wia-audit/v1/k8s-webhook
    certificate-authority: /etc/wia/ca.crt
users:
- name: wia-audit
  user:
    client-certificate: /etc/wia/client.crt
    client-key: /etc/wia/client.key
contexts:
- name: default
  context:
    cluster: wia-audit
    user: wia-audit

7.6 SDK 통합

JavaScript/TypeScript

import { AuditClient } from '@wia/sec-017-audit';

const audit = new AuditClient({
  endpoint: 'https://audit.company.com',
  apiKey: process.env.AUDIT_API_KEY,
  compliance: ['SOC2', 'ISO27001']
});

// 인증 이벤트 로깅
await audit.logAuthentication('user@example.com', true, {
  ip: req.ip,
  userAgent: req.headers['user-agent'],
  mfa: true
});

// 데이터 액세스 로깅
await audit.logDataAccess(
  'user@example.com',
  '/api/customers/123',
  'READ',
  'CONFIDENTIAL'
);

Python

from wia_audit import AuditClient

audit = AuditClient(
    endpoint='https://audit.company.com',
    api_key=os.environ['AUDIT_API_KEY']
)

# 이벤트 로깅
audit.log_event(
    actor={'type': 'user', 'id': 'user_123'},
    action={'type': 'DATA_ACCESS', 'operation': 'READ'},
    resource='/api/customers/456',
    outcome='SUCCESS'
)

7.7 장 요약

핵심 내용

  • 커넥터 프레임워크로 기존 도구와 통합
  • 주요 SIEM, 클라우드, 데이터베이스용 사전 구축 커넥터
  • 네이티브 Kubernetes 감사 로그 통합
  • 고가용성은 최소 3개 노드 필요
  • 운영을 위한 포괄적 모니터링 메트릭
  • 배포 체크리스트로 프로덕션 준비 보장
