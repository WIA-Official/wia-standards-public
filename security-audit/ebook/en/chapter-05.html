Chapter 5: Phase 2 - API Interfaces

RESTful APIs for Audit Operations

5.1 API Design Philosophy

WIA-SEC-017 APIs follow RESTful principles with security and performance as primary concerns:

  • Resource-oriented: URLs represent audit resources
  • JSON-first: All payloads use JSON format
  • Secure by default: TLS required, authentication mandatory
  • Idempotent: Safe retry of failed requests

5.2 Event Submission API

Submit Single Event

POST /wia-audit/v1/events
Authorization: Bearer {api_key}
Content-Type: application/json

{
  "type": "AUTHENTICATION",
  "actor": {
    "id": "urn:wia:identity:user:john.doe",
    "type": "USER"
  },
  "action": "LOGIN",
  "outcome": "SUCCESS",
  "metadata": { "ip": "192.168.1.100", "mfa": true }
}

Response: 201 Created
{
  "id": "urn:wia:audit:evt:550e8400-e29b-41d4",
  "sequence": 12345,
  "hash": "d4e5f6a7b8c9d0..."
}

Batch Submission

POST /wia-audit/v1/events/batch
Authorization: Bearer {api_key}
Content-Type: application/json

{
  "events": [
    { "type": "DATA_ACCESS", "action": "READ", ... },
    { "type": "DATA_ACCESS", "action": "READ", ... }
  ]
}

Response: 201 Created
{
  "accepted": 2,
  "rejected": 0,
  "events": [...]
}

5.3 Query API

EndpointMethodDescription
/eventsGETList events with filters
/events/{id}GETGet single event
/events/searchPOSTAdvanced search
/events/exportPOSTExport to file
/chain/verifyPOSTVerify chain integrity

Query Examples

# List recent authentication failures
GET /wia-audit/v1/events?type=AUTHENTICATION&outcome=FAILURE&limit=100

# Search by actor
POST /wia-audit/v1/events/search
{
  "query": {
    "actor.id": "urn:wia:identity:user:john.doe",
    "timestamp": { "$gte": "2025-01-01T00:00:00Z" }
  },
  "sort": { "timestamp": "desc" },
  "limit": 50
}

5.4 Real-Time Streaming

# WebSocket connection for real-time events
ws://audit.company.com/wia-audit/v1/stream

# Subscribe to specific event types
{
  "action": "subscribe",
  "filters": {
    "types": ["AUTHENTICATION", "SECURITY"],
    "outcome": "FAILURE"
  }
}

# Incoming events
{
  "type": "event",
  "data": {
    "id": "urn:wia:audit:evt:...",
    "type": "AUTHENTICATION",
    "action": "LOGIN",
    "outcome": "FAILURE",
    ...
  }
}

5.5 Compliance Reports API

# Generate SOC 2 compliance report
POST /wia-audit/v1/reports
{
  "type": "SOC2",
  "period": {
    "start": "2025-01-01",
    "end": "2025-03-31"
  },
  "format": "PDF"
}

Response:
{
  "reportId": "rpt-12345",
  "status": "GENERATING",
  "estimatedCompletion": "2025-01-07T11:00:00Z"
}
