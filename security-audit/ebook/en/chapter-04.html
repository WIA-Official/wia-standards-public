Chapter 4: Phase 1 - Data Formats

Standardized JSON Schemas for Audit Events

4.1 Data Model Overview

WIA-SEC-017 defines standardized JSON schemas for all audit-related data, ensuring interoperability across systems and vendors.

Core Entities:
┌─────────────────────────────────────────────────────────────┐
│  AUDIT EVENT                                                 │
│  The primary unit of audit data                              │
│  └── id, timestamp, type, actor, action, resource, outcome  │
├─────────────────────────────────────────────────────────────┤
│  ACTOR                                                       │
│  Who performed the action                                    │
│  └── id, type (user/service/device), attributes             │
├─────────────────────────────────────────────────────────────┤
│  RESOURCE                                                    │
│  What was acted upon                                         │
│  └── id, type, classification, owner                        │
├─────────────────────────────────────────────────────────────┤
│  CHAIN ENTRY                                                 │
│  Tamper-proof wrapper                                        │
│  └── sequence, previousHash, hash, signature                │
└─────────────────────────────────────────────────────────────┘

4.2 Audit Event Schema

{
  "$schema": "https://json-schema.org/draft/2020-12/schema",
  "$id": "https://wiastandards.com/sec-017/audit-event-v1.0.json",
  "title": "WIA-SEC-017 Audit Event",
  "type": "object",
  "required": ["id", "timestamp", "type", "actor", "action", "outcome"],
  "properties": {
    "id": {
      "type": "string",
      "format": "uri",
      "description": "Unique event identifier"
    },
    "timestamp": {
      "type": "string",
      "format": "date-time",
      "description": "ISO 8601 timestamp with timezone"
    },
    "type": {
      "type": "string",
      "enum": ["AUTHENTICATION", "AUTHORIZATION", "DATA_ACCESS",
               "CONFIGURATION", "SECURITY", "ADMINISTRATIVE"]
    },
    "actor": { "$ref": "#/definitions/actor" },
    "action": { "type": "string" },
    "resource": { "$ref": "#/definitions/resource" },
    "outcome": {
      "type": "string",
      "enum": ["SUCCESS", "FAILURE", "PARTIAL", "UNKNOWN"]
    },
    "metadata": { "type": "object" }
  }
}

4.3 Event Types

TypeDescriptionExamples
AUTHENTICATIONIdentity verification eventsLogin, logout, MFA challenge
AUTHORIZATIONAccess control decisionsPermission grant, denial, escalation
DATA_ACCESSData read/write operationsView record, export data, delete
CONFIGURATIONSystem setting changesPolicy update, role modification
SECURITYSecurity-specific eventsFirewall block, malware detected
ADMINISTRATIVEAdmin operationsUser creation, system restart

4.4 Chain Entry Structure

Every audit event is wrapped in a chain entry for tamper-proofing:

{
  "sequence": 12345,
  "event": {
    "id": "urn:wia:audit:evt:550e8400-e29b-41d4-a716",
    "timestamp": "2025-01-07T10:30:00.123Z",
    "type": "AUTHENTICATION",
    "actor": {
      "id": "urn:wia:identity:user:john.doe",
      "type": "USER",
      "attributes": { "email": "john.doe@example.com" }
    },
    "action": "LOGIN",
    "outcome": "SUCCESS",
    "metadata": { "ip": "192.168.1.100", "mfa": true }
  },
  "previousHash": "a3f2b1c4d5e6f7...",
  "hash": "d4e5f6a7b8c9d0...",
  "signature": "RSA-SHA256:abcd1234..."
}

4.6 Chapter Summary

Key Takeaways

  • WIA-SEC-017 defines JSON schemas for all audit data
  • Core entities: Event, Actor, Resource, Chain Entry
  • Six event types cover all security-relevant activities
  • Chain entries provide cryptographic tamper-proofing
  • URN-based identifiers ensure global uniqueness
