6.3 Chain Verification Protocol

The chain verification protocol ensures audit trail integrity:

Chain Verification Flow:
┌──────────────┐                        ┌───────────────┐
│   Verifier   │                        │  Audit Server │
└──────┬───────┘                        └───────┬───────┘
       │                                        │
       │  1. GET /chain/head                    │
       │───────────────────────────────────────►│
       │◄──────────────────────────────────────┤
       │     { sequence: 12345, hash: "..." }   │
       │                                        │
       │  2. GET /chain/verify?from=1&to=12345  │
       │───────────────────────────────────────►│
       │◄──────────────────────────────────────┤
       │     { valid: true, checked: 12345 }    │
       │                                        │
       │  3. POST /chain/audit (spot check)     │
       │───────────────────────────────────────►│
       │◄──────────────────────────────────────┤
       │     { verified: true, samples: [...] } │

Verification Algorithm

function verifyChain(entries) {
  for (let i = 1; i < entries.length; i++) {
    const prev = entries[i - 1];
    const curr = entries[i];

    // Verify hash chain
    const expectedHash = sha256(
      JSON.stringify(curr.event) + prev.hash
    );

    if (curr.hash !== expectedHash) {
      return { valid: false, brokenAt: curr.sequence };
    }

    // Verify signature
    if (!verifySignature(curr.hash, curr.signature)) {
      return { valid: false, invalidSignature: curr.sequence };
    }
  }
  return { valid: true };
}

6.4 Security Requirements

TLS Configuration

RequirementSpecification
Minimum VersionTLS 1.2 (TLS 1.3 recommended)
Cipher SuitesAEAD only (AES-GCM, ChaCha20-Poly1305)
Key ExchangeECDHE with P-256, P-384, or X25519
CertificatesRSA 2048+ or ECDSA P-256+

Authentication

  • API Keys: For automated systems and services
  • OAuth 2.0: For user-facing applications
  • mTLS: For inter-service communication
  • HMAC Signing: For webhook callbacks
