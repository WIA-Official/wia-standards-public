Chapter 04

Feature Engineering and Model Training

4.1 The features that actually predict yield

Yield prediction lives or dies on the quality of the input feature set. Across hundreds of empirical studies the same families dominate: cumulative growing-degree days (GDD), soil moisture at the root-zone depth, leaf-area index (LAI) at canopy closure, the timing of phenological transitions (planting, flowering, grain fill, maturity), and recent precipitation anomalies relative to a 10-year normal. Adding more features past this core set yields rapidly diminishing returns; many production models use fewer than 20 inputs total.

The standard's Phase 1 envelope reflects this empirical hierarchy. Required fields cover GDD, soil moisture, and the four phenological transition timestamps. Optional fields cover satellite-derived NDVI / EVI, field-station weather observations, and management interventions (irrigation, fertiliser, pesticide). A producing model declares which optional fields it consumes via the feature_set_id attribute so a consumer can decide whether the prediction is comparable across regions.

4.2 Train / validation / test discipline

The standard requires a strict temporal split for any model claiming yield-forecast conformance: the training set must end at least 30 days before the validation set begins, the validation set must end at least 30 days before the test set begins, and the test set must contain at least one full growing season. Random k-fold splits leak future information into past predictions and are explicitly forbidden for conformance models.

For a regional model serving multiple climates, a leave-one-region-out cross-validation is the canonical generalisation test. The standard requires that this be reported alongside the in-region test score; a model that scores well in-region and poorly out-of-region is honest about its scope, while a model that reports only in-region scores is implicitly claiming generalisation it has not earned.

4.3 Calibration over raw accuracy

A point prediction without a calibrated uncertainty interval is rarely actionable. A farmer choosing whether to invest in late-season irrigation needs to know not just the predicted yield but the probability that the yield falls below the break-even point. The standard requires that yield-forecast envelopes carry a calibrated 80% prediction interval and that the model's calibration be re-verified at the end of each season against actual measured yields.

4.4 Feature provenance and the audit trail

Each feature in a Phase 1 envelope carries a provenance pointer describing the upstream measurement chain. Soil moisture from a calibrated TDR probe, satellite NDVI from a specific Sentinel-2 pass, weather from a specific WMO-class station — each is named. This lets a consumer evaluating a model's claims trace any feature back to its source measurement and, if the source is later found faulty, retract or recompute the affected predictions.
