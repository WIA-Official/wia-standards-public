제5장: 통신 프로토콜

弘益人間 (홍익인간) - 널리 인간을 이롭게 하라
안전하고 효율적인 통신 프로토콜을 통해 전 세계 언어 보존 시스템이 원활하게 연결되고, 데이터가 안전하게 전송되며, 모든 연구자가 평등하게 접근할 수 있도록 합니다.

1. 통신 프로토콜 개요

WIA-LANG-001 통신 프로토콜은 분산된 언어 보존 시스템 간의 안전하고 효율적인 데이터 교환을 가능하게 합니다. 전 세계 수천 개의 언어 보존 프로젝트가 서로 다른 환경에서 운영되므로, 표준화된 통신 프로토콜은 상호운용성을 보장하는 핵심 요소입니다.

1.1 프로토콜 스택 구조

WIA-LANG 프로토콜 스택은 OSI 모델을 기반으로 다음과 같은 계층으로 구성됩니다:

계층 프로토콜 역할 예시
응용 계층 WIA-LANG Protocol 언어 데이터 교환 어휘 동기화, 메타데이터 전송
표현 계층 JSON/MessagePack 데이터 직렬화 JSON 인코딩/디코딩
세션 계층 WebSocket/gRPC 연결 관리 지속적 연결, 양방향 통신
전송 계층 TLS 1.3 보안 전송 암호화, 인증
네트워크 계층 TCP/IP, HTTP/3 패킷 라우팅 데이터 전송

2. HTTP/HTTPS 기반 통신

3. WebSocket 실시간 통신

실시간 언어 데이터 동기화와 협업을 위해 WIA-LANG은 WebSocket 프로토콜을 지원합니다. 이를 통해 여러 연구자가 동시에 같은 언어 데이터를 편집하고, 변경사항을 즉시 공유할 수 있습니다.

4. gRPC 고성능 통신

대용량 언어 데이터 전송과 마이크로서비스 간 통신을 위해 WIA-LANG은 gRPC를 지원합니다. gRPC는 HTTP/2 기반의 고성능 RPC 프레임워크로, Protocol Buffers를 사용하여 효율적인 직렬화를 제공합니다.

4.1 Protocol Buffers 정의

// wia_lang.proto
syntax = "proto3";

package wia.lang.v1;

// 언어 서비스 정의
service LanguageService {
  // 언어 목록 조회
  rpc ListLanguages(ListLanguagesRequest) returns (ListLanguagesResponse);

  // 특정 언어 조회
  rpc GetLanguage(GetLanguageRequest) returns (Language);

  // 어휘 목록 조회 (스트리밍)
  rpc StreamLexicon(StreamLexiconRequest) returns (stream Word);

  // 오디오 업로드 (클라이언트 스트리밍)
  rpc UploadAudio(stream AudioChunk) returns (AudioUploadResponse);

  // 실시간 동기화 (양방향 스트리밍)
  rpc SyncLanguageData(stream SyncRequest) returns (stream SyncResponse);
}

// 메시지 정의
message Language {
  string id = 1;
  string code = 2;
  string name = 3;
  string english_name = 4;
  EndangermentLevel endangerment_level = 5;
  int32 speaker_count = 6;
  Region region = 7;
  string philosophy = 8; // "弘益人間"
}

enum EndangermentLevel {
  SAFE = 0;
  VULNERABLE = 1;
  ENDANGERED = 2;
  CRITICALLY_ENDANGERED = 3;
  EXTINCT = 4;
}

message Region {
  string country = 1;
  string province = 2;
  Coordinates coordinates = 3;
}

message Coordinates {
  double latitude = 1;
  double longitude = 2;
}

message Word {
  string id = 1;
  string lemma = 2;
  string translation = 3;
  string part_of_speech = 4;
  Pronunciation pronunciation = 5;
}

message Pronunciation {
  string ipa = 1;
  string audio_url = 2;
}

message ListLanguagesRequest {
  int32 page = 1;
  int32 limit = 2;
  EndangermentLevel endangerment_level = 3;
  string region = 4;
}

message ListLanguagesResponse {
  repeated Language languages = 1;
  Pagination pagination = 2;
  string philosophy = 3; // "弘益人間"
}

message Pagination {
  int32 current_page = 1;
  int32 total_pages = 2;
  int32 total_items = 3;
}

6. 보안 프로토콜

언어 데이터의 무결성과 기밀성을 보장하기 위해 WIA-LANG은 다층 보안 프로토콜을 적용합니다.

6.1 TLS 1.3 암호화

TLS 1.3 핸드셰이크

  1. ClientHello: 클라이언트가 지원하는 암호화 방식 전송
  2. ServerHello: 서버가 선택한 암호화 방식 응답
  3. 키 교환: Diffie-Hellman 키 교환으로 세션 키 생성
  4. 인증: 서버 인증서 검증 (선택적 클라이언트 인증)
  5. 암호화 통신: 모든 후속 데이터 암호화
// Node.js TLS 설정 예제
const https = require('https');
const fs = require('fs');

const options = {
  key: fs.readFileSync('private-key.pem'),
  cert: fs.readFileSync('certificate.pem'),
  ca: fs.readFileSync('ca-cert.pem'),

  // TLS 1.3만 허용
  minVersion: 'TLSv1.3',
  maxVersion: 'TLSv1.3',

  // 안전한 암호화 스위트
  ciphers: [
    'TLS_AES_256_GCM_SHA384',
    'TLS_CHACHA20_POLY1305_SHA256',
    'TLS_AES_128_GCM_SHA256'
  ].join(':'),

  // Forward Secrecy 활성화
  honorCipherOrder: true,

  // 클라이언트 인증 요구 (선택적)
  requestCert: false,
  rejectUnauthorized: true
};

const server = https.createServer(options, (req, res) => {
  res.writeHead(200, {
    'Content-Type': 'application/json',
    'X-WIA-Philosophy': '弘益人間'
  });
  res.end(JSON.stringify({ status: 'secure' }));
});

server.listen(443, () => {
  console.log('WIA-LANG 보안 서버 실행 중 (TLS 1.3)');
});

6.2 데이터 무결성 검증

모든 전송 데이터는 체크섬을 통해 무결성을 검증합니다. 

// SHA-256 체크섬 계산 및 검증
const crypto = require('crypto');

function calculateChecksum(data) {
  return crypto
    .createHash('sha256')
    .update(JSON.stringify(data))
    .digest('hex');
}

// 데이터 전송 시 체크섬 포함
const languageData = {
  id: 'lang-jje-001',
  name: '제주어',
  philosophy: '弘익人間'
};

const checksum = calculateChecksum(languageData);

const payload = {
  data: languageData,
  checksum: checksum,
  algorithm: 'sha256'
};

// 수신 측 검증
function verifyChecksum(payload) {
  const receivedChecksum = payload.checksum;
  const calculatedChecksum = calculateChecksum(payload.data);

  if (receivedChecksum !== calculatedChecksum) {
    throw new Error('데이터 무결성 검증 실패!');
  }

  console.log('✓ 체크섬 검증 성공');
  return payload.data;
}

try {
  const verifiedData = verifyChecksum(payload);
  console.log('검증된 데이터:', verifiedData);
} catch (error) {
  console.error('오류:', error.message);
}

7. 오류 복구 및 재시도 메커니즘

네트워크 불안정성을 고려하여 WIA-LANG은 지능적인 재시도 메커니즘을 제공합니다.

7.1 지수 백오프 (Exponential Backoff)

// 지수 백오프 재시도 구현
async function fetchWithRetry(url, options = {}, maxRetries = 5) {
  let lastError;

  for (let attempt = 0; attempt < maxRetries; attempt++) {
    try {
      console.log(`시도 ${attempt + 1}/${maxRetries}`);

      const response = await fetch(url, {
        ...options,
        headers: {
          ...options.headers,
          'X-WIA-Retry-Attempt': attempt.toString(),
          'X-WIA-Philosophy': '弘益人間'
        }
      });

      if (response.ok) {
        return await response.json();
      }

      // 429 (Too Many Requests) 처리
      if (response.status === 429) {
        const retryAfter = response.headers.get('Retry-After');
        const waitTime = retryAfter ? parseInt(retryAfter) * 1000 : calculateBackoff(attempt);

        console.log(`Rate limit 도달. ${waitTime}ms 대기 중...`);
        await sleep(waitTime);
        continue;
      }

      // 5xx 서버 오류는 재시도
      if (response.status >= 500) {
        throw new Error(`서버 오류: ${response.status}`);
      }

      // 4xx 클라이언트 오류는 재시도 안 함
      throw new Error(`클라이언트 오류: ${response.status}`);

    } catch (error) {
      lastError = error;
      console.error(`오류 발생:`, error.message);

      if (attempt < maxRetries - 1) {
        const backoffTime = calculateBackoff(attempt);
        console.log(`${backoffTime}ms 후 재시도...`);
        await sleep(backoffTime);
      }
    }
  }

  throw new Error(`최대 재시도 횟수 초과: ${lastError.message}`);
}

function calculateBackoff(attempt) {
  // 지수 백오프: 2^attempt * 1000ms + 랜덤 지터
  const exponentialTime = Math.pow(2, attempt) * 1000;
  const jitter = Math.random() * 1000;
  return Math.min(exponentialTime + jitter, 60000); // 최대 60초
}

function sleep(ms) {
  return new Promise(resolve => setTimeout(resolve, ms));
}

// 사용 예제
try {
  const data = await fetchWithRetry('https://api.wia.org/v1/languages', {
    method: 'GET',
    headers: {
      'Authorization': 'Bearer token123'
    }
  });

  console.log('데이터 조회 성공:', data);
} catch (error) {
  console.error('최종 실패:', error.message);
}

7.2 Circuit Breaker 패턴

// Circuit Breaker 구현
class CircuitBreaker {
  constructor(threshold = 5, timeout = 60000) {
    this.failureCount = 0;
    this.threshold = threshold;
    this.timeout = timeout;
    this.state = 'CLOSED'; // CLOSED, OPEN, HALF_OPEN
    this.nextAttempt = Date.now();
  }

  async execute(fn) {
    if (this.state === 'OPEN') {
      if (Date.now() < this.nextAttempt) {
        throw new Error('Circuit breaker OPEN 상태 - 서비스 일시 중단');
      }
      this.state = 'HALF_OPEN';
    }

    try {
      const result = await fn();
      this.onSuccess();
      return result;
    } catch (error) {
      this.onFailure();
      throw error;
    }
  }

  onSuccess() {
    this.failureCount = 0;
    this.state = 'CLOSED';
    console.log('✓ Circuit breaker: CLOSED');
  }

  onFailure() {
    this.failureCount++;
    console.log(`✗ Circuit breaker: 실패 ${this.failureCount}/${this.threshold}`);

    if (this.failureCount >= this.threshold) {
      this.state = 'OPEN';
      this.nextAttempt = Date.now() + this.timeout;
      console.log(`⚠ Circuit breaker: OPEN (${this.timeout}ms 동안)`);
    }
  }

  getState() {
    return this.state;
  }
}

// 사용 예제
const breaker = new CircuitBreaker(3, 30000);

async function callAPI() {
  return breaker.execute(async () => {
    const response = await fetch('https://api.wia.org/v1/languages');
    if (!response.ok) throw new Error('API 오류');
    return response.json();
  });
}

// 여러 번 호출 시도
for (let i = 0; i < 10; i++) {
  try {
    const data = await callAPI();
    console.log('성공:', data.meta?.philosophy); // "弘益人間"
  } catch (error) {
    console.error('실패:', error.message);
  }
  await sleep(1000);
}

  signalServer: 'wss://signal.wia.org',
  philosophy: '弘益人間'
});

// 시그널 서버 연결
await p2p.connect();

// 다른 연구자와 P2P 연결
const peer = await p2p.connectToPeer('peer-linguist-123');

// 언어 데이터 직접 전송
peer.send({
  type: 'language_data',
  languageId: 'lang-jje-001',
  data: {
    wordId: 'word-jje-2501',
    lemma: '고망',
    audio: [오디오 바이너리 데이터]
  }
});

// 데이터 수신
peer.on('data', (message) => {
  console.log('P2P 데이터 수신:', message);

  if (message.type === 'language_data') {
    // 로컬 데이터베이스에 저장
    saveLanguageData(message.data);
  }
});

console.log('弘益人間 - P2P로 언어 데이터 공유 중');

장 요약

  1. 다층 프로토콜 스택: WIA-LANG-001은 OSI 모델을 기반으로 응용 계층부터 네트워크 계층까지 표준화된 프로토콜 스택을 정의하여 언어 보존 시스템 간 안정적인 통신을 보장합니다.
  2. HTTPS 기본 통신: TLS 1.3 이상의 암호화를 필수로 하며, 커스텀 HTTP 헤더를 통해 요청 추적, 버전 관리, 속도 제한 등을 효과적으로 수행합니다.
  3. 실시간 WebSocket 지원: 양방향 실시간 통신을 통해 전 세계 연구자들이 동시에 언어 데이터를 편집하고 동기화할 수 있는 협업 환경을 제공합니다.
  4. 고성능 gRPC: Protocol Buffers 기반의 gRPC를 통해 대용량 언어 데이터를 효율적으로 전송하며, 스트리밍 API로 실시간 데이터 처리를 지원합니다.
  5. 지능적 오류 복구: 지수 백오프 재시도와 Circuit Breaker 패턴을 통해 네트워크 불안정성에 대응하고, 시스템 안정성을 높입니다.
  6. P2P 분산 통신: WebRTC 기반 P2P 연결로 중앙 서버 의존도를 낮추고, 弘益人間 철학에 따라 분산된 언어 보존 생태계를 구축합니다.
