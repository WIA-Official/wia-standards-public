제4장: API 인터페이스

弘益人間 (홍익인간) - 널리 인간을 이롭게 하라
개방형 API를 통해 전 세계 개발자와 연구자가 언어 보존 시스템을 자유롭게 구축하고 공유할 수 있도록 합니다. WIA-LANG-001 API는 기술적 장벽을 낮추고 협력을 촉진하는 도구입니다.

1. WIA-LANG API 개요

WIA-LANG-001 API는 RESTful 아키텍처를 기반으로 하는 개방형 언어 보존 인터페이스입니다. 이 API를 통해 개발자는 멸종위기 언어 데이터에 접근하고, 새로운 데이터를 추가하며, 언어 분석 도구를 통합할 수 있습니다. 모든 엔드포인트는 HTTPS를 통해 보호되며, OAuth 2.0 인증을 지원합니다.

1.1 API 설계 원칙

  • RESTful 설계: 직관적인 URL 구조와 HTTP 메서드 활용
  • 버전 관리: URL 경로에 버전 포함 (예: /v1/languages)
  • JSON 응답: 모든 응답은 JSON 형식으로 반환
  • 페이지네이션: 대용량 데이터 효율적 처리
  • 오류 처리: 명확한 HTTP 상태 코드와 오류 메시지
  • CORS 지원: 크로스 오리진 요청 허용
  • 속도 제한: 공정한 리소스 사용을 위한 Rate Limiting

2. 인증 및 권한 관리

WIA-LANG API는 OAuth 2.0 프로토콜을 사용하여 안전한 인증을 제공합니다. 弘益人間 정신에 따라 읽기 API는 무료로 공개되며, 쓰기 작업은 인증이 필요합니다.

2.1 API 키 발급 및 사용

// API 키 발급 (개발자 포털에서 발급)
const WIA_API_KEY = 'wia_lang_key_abc123def456';
const WIA_API_SECRET = 'secret_xyz789';

// API 클라이언트 초기화
import { WIALangClient } from '@wia/lang-client';

const client = new WIALangClient({
  apiKey: WIA_API_KEY,
  apiSecret: WIA_API_SECRET,
  baseUrl: 'https://api.wia.org/v1',
  philosophy: '弘益人間'
});

// OAuth 2.0 인증 토큰 발급
const authResponse = await client.authenticate();
console.log('Access Token:', authResponse.accessToken);
console.log('Expires In:', authResponse.expiresIn);

// 인증된 요청 예제
const languages = await client.getLanguages({
  endangermentLevel: 'critically_endangered',
  limit: 10
});

5. 에러 처리 및 상태 코드

WIA-LANG API는 표준 HTTP 상태 코드를 사용하여 요청의 성공 또는 실패를 나타냅니다.

상태 코드 설명 예시 상황
200 OK 요청 성공 데이터 조회 성공
201 Created 리소스 생성 성공 새 어휘 항목 추가
400 Bad Request 잘못된 요청 필수 필드 누락
401 Unauthorized 인증 실패 유효하지 않은 API 키
403 Forbidden 권한 없음 관리자 전용 작업
404 Not Found 리소스 없음 존재하지 않는 언어 ID
429 Too Many Requests 요청 한도 초과 Rate Limit 도달
500 Internal Server Error 서버 오류 예상치 못한 서버 문제

6. SDK 및 클라이언트 라이브러리

WIA-LANG API는 다양한 프로그래밍 언어로 작성된 공식 SDK를 제공합니다.

6.1 JavaScript/TypeScript SDK

// 설치
npm install @wia/lang-client

// 사용 예제
import { WIALangClient, EndangermentLevel } from '@wia/lang-client';

const client = new WIALangClient({
  apiKey: process.env.WIA_API_KEY,
  philosophy: '弘益人間'
});

// 타입 안전 언어 조회
const languages = await client.languages.list({
  endangermentLevel: EndangermentLevel.CriticallyEndangered,
  region: 'Asia',
  hasAudio: true,
  page: 1,
  limit: 20
});

console.log(`발견된 언어: ${languages.data.length}`);

for (const lang of languages.data) {
  console.log(`${lang.name} (${lang.code}): ${lang.speakerCount}명`);

  // 어휘 조회
  const lexicon = await client.lexicon.list(lang.id, {
    limit: 10
  });

  console.log(`  어휘 항목: ${lexicon.pagination.totalItems}개`);
}

// 새 어휘 추가
const newWord = await client.lexicon.create('lang-jje-001', {
  lemma: '바당',
  translation: '바다',
  englishTranslation: 'sea',
  partOfSpeech: 'noun',
  pronunciation: {
    ipa: 'padaŋ'
  }
});

console.log(`새 단어 추가됨: ${newWord.id}`);

print(f"발견된 언어: {len(languages.data)}")

# 오디오 업로드

            'age': 68,
            'gender': 'female',
            'native_speaker': True
        }
    )

7. 웹훅 및 실시간 알림

WIA-LANG API는 웹훅을 통해 실시간 이벤트 알림을 제공합니다.

7.1 웹훅 이벤트 유형

  • language.created - 새 언어 추가됨
  • word.created - 새 어휘 추가됨
  • word.approved - 어휘 검토 승인됨
  • audio.uploaded - 오디오 업로드 완료
  • language.status_changed - 멸종위기 등급 변경

장 요약

  1. RESTful API 아키텍처: WIA-LANG-001 API는 직관적인 RESTful 설계를 채택하여 개발자가 쉽게 통합할 수 있으며, JSON 형식의 명확한 요청/응답 구조를 제공합니다.
  2. OAuth 2.0 인증 시스템: 안전한 OAuth 2.0 프로토콜을 통한 인증을 제공하며, 弘益人間 정신에 따라 읽기 API는 공개하고 쓰기 작업만 인증을 요구합니다.
  3. 포괄적인 엔드포인트: 언어 조회, 어휘 관리, 오디오 업로드, 텍스트 분석, 통계 조회 등 언어 보존에 필요한 모든 기능을 API로 제공합니다.
  4. 다중 언어 SDK 지원: JavaScript/TypeScript, Python을 포함한 주요 프로그래밍 언어의 공식 SDK를 제공하여 개발 생산성을 높입니다.
  5. 명확한 에러 처리: 표준 HTTP 상태 코드와 상세한 에러 메시지를 통해 디버깅을 용이하게 하고, 개발자 경험을 향상시킵니다.
  6. 실시간 협업 지원: 웹훅과 실시간 알림 시스템을 통해 전 세계 연구자와 개발자가 협력하여 언어 데이터를 공동으로 보존할 수 있는 환경을 제공합니다.
