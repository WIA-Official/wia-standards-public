# 제4장: Phase 1 - 데이터 형식 사양

## 언어 데이터의 표준화된 구조

Phase 1은 WIA-LANG-001 표준의 기초를 형성합니다. 언어 보존에 필요한 모든 데이터 유형에 대한 표준화된 형식을 정의하여, 다양한 시스템과 도구 간의 상호운용성을 보장합니다.

## 핵심 데이터 구조

### 언어 메타데이터 스키마

모든 언어 데이터의 기본이 되는 메타데이터 구조입니다:

```json
{
  "standard": "WIA-LANG-001",
  "version": "1.0",
  "language": {
    "name": {
      "native": "アイヌ イタㇰ",
      "english": "Ainu",
      "korean": "아이누어"
    },
    "iso639-3": "ain",
    "glottolog": "ainu1240",
    "family": {
      "name": "Language Isolate",
      "branch": null
    },
    "endangerment": {
      "status": "critically_endangered",
      "unescoLevel": 4,
      "lastAssessed": "2024-06-15"
    },
    "speakers": {
      "native": {
        "count": 10,
        "uncertainty": "high",
        "notes": "대부분 80세 이상"
      },
```

### 필수 필드와 선택 필드

| 필드 경로 | 필수 | 타입 | 설명 |
|----------|------|------|------|
| `standard` | ✓ | string | 표준 식별자 |
| `version` | ✓ | string | 표준 버전 |
| `language.name.english` | ✓ | string | 영문 언어명 |
| `language.iso639-3` | ✓ | string | ISO 639-3 코드 |
| `language.endangerment.status` | ✓ | enum | 위기 상태 |
| `language.speakers.native` | ✓ | object | 원어민 화자 정보 |
| `language.name.native` | | string | 자국어 표기 |
| `language.glottolog` | | string | Glottolog 코드 |
| `language.family` | | object | 어족 정보 |
| `language.regions` | | array | 사용 지역 |

## 텍스트 데이터 형식

### 인코딩 요구사항

모든 텍스트 데이터는 다음 요구사항을 충족해야 합니다:

| 항목 | 요구사항 | 비고 |
|------|----------|------|
| 문자 인코딩 | UTF-8 | 필수 |
| 정규화 | NFC | Unicode 정규화 |
| BOM | 사용 금지 | Byte Order Mark 제외 |
| 줄바꿈 | LF | Unix 스타일 |

### 텍스트 타입별 형식

**원문 텍스트**:
```json
{
  "text": {
    "id": "txt-ain-001",
    "language": "ain",
    "content": "ウコイキ オルンオシキ イタㇰ アン",
    "script": {
      "name": "Katakana",

      "direction": "ltr"
    },
    "orthography": {
      "system": "standard",
      "version": "1999"
    }
  }
}
```

**음성 전사 (IPA)**:
```json
{
  "phonetic": {
    "system": "IPA",
    "broad": "/ukoiki orunoski itak an/",
    "narrow": "[ɯkoiki oɾɯnoski itak an]",
```


### ELAN/Praat 호환성

WIA-LANG 형식은 주요 전사 도구와 상호 변환이 가능합니다:

```
WIA-LANG ←→ ELAN (.eaf)
WIA-LANG ←→ Praat (.TextGrid)
WIA-LANG ←→ Transcriber (.trs)
WIA-LANG ←→ CLAN (.cha)
```

변환 예시:
```bash
# ELAN에서 WIA-LANG으로 변환
wia-lang convert --from elan --to wia recording.eaf

# WIA-LANG에서 Praat으로 변환
wia-lang convert --from wia --to praat recording.json
```

## 파일 명명 규칙

### 기본 형식

```
{언어코드}_{유형}_{날짜}_{순번}.{확장자}

예시:
ain_recording_20250101_001.flac
ain_transcription_20250101_001.json

```

### 유형 코드

| 코드 | 의미 | 파일 형식 |
|------|------|----------|
| `recording` | 오디오 녹음 | FLAC, WAV, MP3 |

## 규정 준수

### 필수 준수 사항

모든 WIA-LANG-001 구현은 다음을 준수해야 합니다:

1. **UTF-8 인코딩**: 모든 텍스트 데이터
2. **ISO 8601 타임스탬프**: 모든 날짜/시간
3. **RFC 3986 URI**: 모든 참조
4. **JSON Schema 검증**: 모든 JSON 데이터

### 검증 도구

```bash
# 데이터 검증
wia-lang validate --schema language-metadata data.json

# 오디오 형식 검증
wia-lang validate --type audio recording.flac

# 전체 프로젝트 검증
wia-lang validate --project ./my-language-project/
```

## 핵심 요약

**주요 내용:**

1. **언어 메타데이터**: ISO 639-3 코드, UNESCO 위기 등급, 화자 정보를 포함한 표준화된 스키마

2. **오디오/비디오 형식**: 아카이빙용 무손실 형식(FLAC, ProRes)과 배포용 압축 형식의 이중 구조

3. **텍스트 형식**: UTF-8 인코딩 필수, IPA 음성 전사, 라이프치히 글로싱 규칙 적용

4. **전사 형식**: 시간 정렬 구조, ELAN/Praat 등 기존 도구와 상호 변환 지원

5. **저장 요구사항**: 최소 3개 복제, 지리적 분산, AES-256 암호화, 정기적 무결성 검증
