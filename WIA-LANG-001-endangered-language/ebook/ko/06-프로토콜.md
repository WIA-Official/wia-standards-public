# 제6장: Phase 3 - 통신 프로토콜

## 실시간 연결과 데이터 스트리밍

Phase 3는 언어 보존 시스템 간의 실시간 통신을 위한 프로토콜을 정의합니다. 현장 녹음의 실시간 전송, 협업 전사 작업, 분산 아카이브 간 동기화 등 다양한 실시간 시나리오를 지원합니다.

## 프로토콜 아키텍처

### 계층 구조

```
┌─────────────────────────────────────────────────────────────┐
│                     애플리케이션 계층                          │
│  (언어 보존 메시지: 녹음, 전사, 메타데이터)                      │
├─────────────────────────────────────────────────────────────┤
│                      세션 계층                               │
│  (연결 관리, 인증, 세션 상태)                                  │
├─────────────────────────────────────────────────────────────┤
│                      전송 계층                               │
│  (HTTP/2, WebSocket, gRPC)                                 │
├─────────────────────────────────────────────────────────────┤
│                      보안 계층                               │
│  (TLS 1.3)                                                 │
└─────────────────────────────────────────────────────────────┘
```

### 지원 전송 프로토콜

| 프로토콜 | 용도 | 특징 |
|----------|------|------|
| HTTP/2 | REST API | 멀티플렉싱, 헤더 압축 |
| WebSocket | 실시간 양방향 | 저지연, 지속 연결 |
| gRPC | 서비스 간 통신 | 고성능, 스트리밍 |
| Server-Sent Events | 서버→클라이언트 | 단방향 푸시 |

### gRPC 서비스 정의

```protobuf
syntax = "proto3";

package wia.lang.v1;

service LanguageService {
  // 단일 요청-응답
  rpc GetLanguage(GetLanguageRequest) returns (Language);

  // 서버 스트리밍
  rpc StreamRecordings(StreamRequest) returns (stream Recording);

  // 클라이언트 스트리밍
  rpc UploadAudioChunks(stream AudioChunk) returns (UploadResponse);

  // 양방향 스트리밍
  rpc CollaborativeTranscription(stream TranscriptionUpdate)
      returns (stream TranscriptionEvent);
}

message AudioChunk {
  string session_id = 1;
  bytes data = 2;
  int64 timestamp = 3;
  bool is_final = 4;
}

message Recording {
  string id = 1;
  string language = 2;
  float duration = 3;
  bytes audio_data = 4;
  map<string, string> metadata = 5;
}
```

## 메시지 형식

### 기본 메시지 구조

모든 프로토콜 메시지는 다음 구조를 따릅니다:

```json
{
  "version": "1.0",
  "messageId": "msg-uuid-v4",
  "timestamp": "2025-01-01T10:00:00.000Z",
  "type": "MESSAGE_TYPE",
  "action": "ACTION_NAME",
  "payload": {
    // 메시지 타입별 데이터
  },
  "metadata": {
    "correlationId": "optional-correlation-id",
    "replyTo": "optional-reply-channel"
  }
}
```

### 메시지 타입

| 타입 | 설명 | 방향 |
|------|------|------|
| `request` | 요청 메시지 | 클라이언트→서버 |
| `response` | 응답 메시지 | 서버→클라이언트 |
| `event` | 이벤트 알림 | 서버→클라이언트 |
| `stream_data` | 스트림 데이터 | 양방향 |
| `stream_end` | 스트림 종료 | 양방향 |
| `error` | 오류 메시지 | 양방향 |
| `ping` / `pong` | 연결 확인 | 양방향 |

## 연결 관리

### 연결 상태

```
┌─────────────┐
│ Disconnected│
└──────┬──────┘
       │ connect()
       ▼
┌─────────────┐    auth success    ┌─────────────┐
│ Connecting  │ ─────────────────> │ Connected   │
└──────┬──────┘                    └──────┬──────┘
       │                                  │
       │ auth failed                      │ disconnect/error
       ▼                                  ▼
┌─────────────┐                    ┌─────────────┐
│   Error     │                    │ Reconnecting│
└─────────────┘                    └──────┬──────┘
                                          │
                                          │ max attempts
                                          ▼
                                   ┌─────────────┐
                                   │ Disconnected│
                                   └─────────────┘
```

### 재연결 전략

```json
{
  "reconnection": {
    "enabled": true,
    "strategy": "exponential_backoff",
    "initialDelay": 1000,
    "maxDelay": 30000,
    "multiplier": 2,
    "maxAttempts": 10,
    "jitter": 0.1
  }
}
```

**재연결 알고리즘**:
```python
def calculate_delay(attempt, config):
    base_delay = config['initialDelay'] * (config['multiplier'] ** attempt)
    delay = min(base_delay, config['maxDelay'])
    jitter = delay * config['jitter'] * random.uniform(-1, 1)
    return delay + jitter

# 시도 1: 1초 ± 0.1초
# 시도 2: 2초 ± 0.2초
# 시도 3: 4초 ± 0.4초
# ...
# 시도 5+: 30초 ± 3초 (최대)
```

## 데이터 압축

### 압축 옵션

| 알고리즘 | 압축률 | 속도 | 용도 |
|----------|--------|------|------|
| gzip | 높음 | 중간 | 일반 데이터 |
| br (Brotli) | 최고 | 느림 | 정적 콘텐츠 |
| zstd | 높음 | 빠름 | 실시간 스트리밍 |
| lz4 | 중간 | 최고 | 저지연 요구 |

### 압축 협상

```http
// 클라이언트 요청
Accept-Encoding: zstd, br, gzip

// 서버 응답
Content-Encoding: zstd
```

WebSocket 압축:
```
Sec-WebSocket-Extensions: permessage-deflate; client_max_window_bits=15
```

### 오류 유형

| 코드 | 유형 | 설명 | 복구 방법 |
|------|------|------|----------|
| `E001` | 연결 손실 | 네트워크 중단 | 자동 재연결 |
| `E002` | 인증 만료 | 토큰 만료 | 토큰 갱신 후 재연결 |

| `E005` | 형식 오류 | 잘못된 메시지 형식 | 메시지 폐기 |

### 오류 처리 흐름

```python
async def handle_connection():
    while True:
        try:
            message = await receive_message()
            await process_message(message)
        except ConnectionLost:
            await reconnect_with_backoff()
            await resume_session()
        except AuthExpired:
            await refresh_token()
            await reconnect()
        except MessageFormatError as e:
            log_error(e)
            # 계속 진행
        except Exception as e:
            log_error(e)
            raise
```
