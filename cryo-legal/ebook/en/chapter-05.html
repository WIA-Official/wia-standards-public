5.3 Recognition Protocols
--------------------------------------------------------------------------------

Recognition protocols establish how one jurisdiction acknowledges the legal
status granted by another.

[ID] Recognition Levels:

    RecognitionLevels {
      FULL: {
        definition:   "Complete recognition without additional process"
        requirements: "Treaty membership + proper documentation"
        effect:       "Status treated as if granted locally"
        examples:     "Between ICLST full members"
      }

      PARTIAL: {
        definition:   "Recognition with limitations"
        requirements: "Bilateral agreement or judicial decision"
        effect:       "Core status recognized, some rights limited"
        examples:     "Between treaty member and associate"
      }

      PENDING: {
        definition:   "Recognition under consideration"
        requirements: "Application filed, awaiting determination"
        effect:       "Provisional recognition may be granted"
        timeline:     "Typically 30-90 days"
      }

      NOT_RECOGNIZED: {
        definition:   "No recognition of foreign status"
        requirements: "No treaty or bilateral agreement"
        effect:       "Must seek local status determination"
        remedy:       "File for local preservation status"
      }
    }

    RecognitionProcess {

5.4 Conflict Resolution
--------------------------------------------------------------------------------

When jurisdictions disagree on status or recognition, conflict resolution
mechanisms apply.

[ID] Conflict Types:

    ConflictTypes {
      statusConflict: {
        description: "Different jurisdictions assign different status"
        example:     "Country A: PRESERVED_STATUS; Country B: PERMANENT_DEATH"
        resolution:  "Primary jurisdiction status prevails"
      }

      guardianConflict: {
        description: "Different guardians appointed in different jurisdictions"
        example:     "Country A appoints Guardian X; Country B appoints Guardian Y"
        resolution:  "Primary jurisdiction appointment prevails"
      }

      assetConflict: {
        description: "Dispute over asset control"
        example:     "Guardian vs. local heir over local assets"
        resolution:  "Local asset rules + international coordination"
      }

      procedureConflict: {
        description: "Different procedural requirements"
        example:     "Country A requires court hearing; Country B automatic"
        resolution:  "Apply receiving jurisdiction procedures"
      }
    }
