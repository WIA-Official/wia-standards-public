3.1 Death Certificate Processing
--------------------------------------------------------------------------------

The death certificate is the foundational document that initiates the legal
status transition from LIVING to LEGALLY_DECEASED.

[ID] Death Certificate Structure:

    DeathCertificate {
      certificateId:    "DC-2075-KR-000001234"
      jurisdiction:     "KR"
      issuingAuthority: "Seoul Metropolitan Government"

      decedentInfo: {
        fullName:       "Kim Preservation"
        dateOfBirth:    "1985-03-15"
        placeOfBirth:   "Seoul, South Korea"
        identification: "National ID: 850315-1234567"
      }

      deathInfo: {
        dateOfDeath:    "2075-06-15"
        timeOfDeath:    "14:30:00"
        placeOfDeath:   "Seoul National University Hospital"
        causeOfDeath: {
          immediate:    "Cardiac arrest"
          underlying:   "Coronary artery disease"
          contributing: ["Hypertension", "Diabetes mellitus"]
        }
      }

      certification: {
        certifyingPhysician: "Dr. Park Medical"
        licenseNumber:       "MD-KR-12345"
        certificationDate:   "2075-06-15"
      }

      preservationAnnotation: {
        intendedForPreservation: true
        preservationProvider:    "WIA Cryopreservation Facility"
        consentVerified:         true
        consentId:               "CON-2075-KR-001234"
      }
    }

[ID] Certificate Processing Workflow:

    DeathCertificateWorkflow {
      step1: {
        name:     "Medical Certification"
        actor:    "Attending physician"
        actions: [
          "Confirm death using standard criteria",
          "Document time and cause of death",
          "Check for preservation consent"
        ]
        timeline: "Immediate upon death"
      }

      step2: {
        name:     "Preservation Notification"
        actor:    "Medical facility"
        actions: [
          "Notify designated preservation provider",
          "Verify consent documentation",
          "Initiate preservation protocol"
        ]
        timeline: "Within 1 hour"
      }

      step3: {
        name:     "Certificate Issuance"
        actor:    "Registrar"
        actions: [
          "Review medical certification",
          "Issue death certificate",
          "Add preservation annotation"
        ]
        timeline: "Within 24 hours"
      }

      step4: {
        name:     "Status Transition Filing"
        actor:    "Legal representative"
        actions: [
          "File for preservation status",
          "Submit supporting documents",
          "Initiate court proceedings"
        ]
        timeline: "Within 72 hours"
      }
    }

[!] Critical: Death certificates for cryopreserved individuals must include
    the preservation annotation to ensure proper status transition processing.

3.2 Legal Death vs. Biological State
--------------------------------------------------------------------------------

CRYO-LEGAL distinguishes between legal death (a legal status) and biological
state (a physical condition).

[ID] Conceptual Framework:

    DeathConceptFramework {
      legalDeath: {
        definition:  "Official declaration ending legal personhood"
        basis:       "Medical certification + legal registration"
        consequences: [
          "Estate opening",
          "Marriage termination",
          "Contract termination",
          "Voting rights cessation"
        ]
        reversibility: "Possible through court restoration"
      }

      biologicalState: {
        categories: {
          living:    "Active biological functions"
          deceased:  "Cessation of biological functions"
          preserved: "Biological processes halted but potentially
                      reversible"
        }
      }

      relationship: {
        traditionalView:  "Legal death = biological death (irreversible)"
        cryoLegalView:    "Legal death may precede biological finality"
        implication:      "New legal category needed for preservation"
      }
    }

[ID] Preservation Status Rationale:

    PreservationStatusRationale {
      problem: {
        statement: "Traditional death declaration assumes irreversibility"
        conflict:  "Cryopreservation introduces potential reversibility"
      }

      solution: {
        approach: "Create intermediate legal status"
        status:   "PRESERVED_STATUS"
        effect:   "Acknowledges death declaration while preserving
                   potential for restoration"
      }

      legalEffect: {
        deathCertificate: "Remains valid but marked as potentially
                           reversible"
        estatePlanning:   "Held in suspension rather than distribution"
        personalRights:   "Suspended rather than terminated"
      }
    }

[i] Note: The legal fiction of PRESERVED_STATUS allows the legal system
    to accommodate the reality that biological death may not be permanent
    for cryopreserved individuals.

3.3 Multi-Jurisdiction Considerations
--------------------------------------------------------------------------------

Death declarations often involve multiple jurisdictions, requiring careful
coordination.

[ID] Jurisdictional Issues:

    JurisdictionalConsiderations {
      deathLocation: {
        description: "Where death occurred"
        relevance:   "Primary jurisdiction for death certificate"
        examples: [
          "Hospital in home country",
          "Foreign country during travel",
          "International waters"
        ]
      }

      residenceJurisdiction: {
        description: "Where individual was legally resident"
        relevance:   "Estate and status implications"
      }

      citizenshipJurisdiction: {
        description: "Country of citizenship"

3.4 Special Death Classifications
--------------------------------------------------------------------------------

Certain circumstances require special handling of death declarations.

[ID] Special Cases:

    SpecialDeathClassifications {
      presumedDeath: {
        definition:  "Death presumed without body"
        relevance:   "Cannot apply to cryopreservation"
        handling:    "Standard presumed death rules"
      }

      violentDeath: {
        definition:  "Death by accident, homicide, or suicide"
        relevance:   "May require autopsy before preservation"
        handling: {
          autopsy:    "May be required by law"
          timeline:   "Preservation delay possible"
          consent:    "Legal representative consent"
        }
      }

      medicallyAssistedDeath: {
        definition:  "Death through legal euthanasia/assisted dying"
        relevance:   "Planned death with preservation intent"
        handling: {
          timing:     "Preservation can be optimally timed"
          consent:    "Enhanced consent requirements"
          documentation: "Additional certifications"
        }
      }

      terminalSedation: {
        definition:  "Death following palliative sedation"
        relevance:   "Similar to medically assisted death"
        handling:    "Coordinated preservation protocol"
      }

      suddenDeath: {
        definition:  "Unexpected death requiring rapid response"
        relevance:   "Time-critical preservation"
        handling: {
          standbyTeam: "Pre-arranged if possible"
          emergencyProtocol: "Standard emergency response"
          documentation: "Retrospective consent verification"
        }
      }
    }

[ID] Forensic Considerations:

    ForensicConsiderations {
      autopsyRequirements: {
        mandatory: [
          "Suspected homicide",
          "Unattended death",
          "Death during police custody",
          "Death of unknown cause"
        ]

        waivable: [
          "Natural death with known cause",
          "Death under medical care",
          "Death with clear medical history"
        ]
      }

      preservationCompatibility: {
        fullAutopsy:    "May compromise preservation quality"
        limitedAutopsy: "Possible with coordination"
        noAutopsy:      "Optimal for preservation"

        negotiation: {
          approach: "Work with medical examiner"
          goals:    ["Minimize invasive procedures",
                     "Preserve critical tissues",
                     "Document for legal purposes"]
        }
      }

      documentation: {
        forensicReport:     "Required if investigation conducted"
        chainOfCustody:     "Maintained throughout"
        evidencePreservation: "May require tissue samples"
      }
    }

[!] Warning: In cases requiring forensic investigation, preservation may be
    delayed. Pre-planning and advance directives can help minimize delays.
