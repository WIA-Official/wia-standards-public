5.1 다중 관할권 프레임워크
--------------------------------------------------------------------------------

냉동보존은 종종 여러 법적 관할권을 포함하여 조정과 인정을 위한
종합적인 프레임워크가 필요합니다.

[ID] 관할권 프레임워크:

    JurisdictionalFramework {
      primaryJurisdiction: {
        definition: "지위에 대한 주요 권한을 가진 관할권"
        determination: {
          rules: [
            "보존 시설 위치 (주요 요소)",
            "개인의 최종 주소지",
            "시민권",
            "보존 지위를 발급한 법원"
          ]
          conflicts: "국제 프로토콜로 해결"
        }
        authority: [
          "지위 결정",
          "후견인 선임",
          "회복 절차",
          "분쟁 해결"
        ]
      }

      secondaryJurisdictions: {
        definition: "정당한 이해관계를 가진 다른 관할권"
        examples: [
          "사망 발생 관할권",
          "시민권 관할권",
          "자산 소재 관할권",
          "가족 거주 관할권"
        ]
        role: [
          "주요 관할권 결정 인정",
          "현지 문서 등록",
          "자산 관리 협력",
          "가족법 문제"
        ]
      }
    }

[ID] 관할권 유형:

    +------------------------------------------------------------------+
    | 유형                 | 권한                                      |
    +------------------------------------------------------------------+
    | 보존                 | 지위 관리, 후견인 감독                     |
    | 시민권               | 영사 서비스, 여권 문제                     |
    | 자산 소재            | 현지 재산/금융 문제                        |
    | 가족 거주            | 가족법, 자녀 양육권                        |
    | 사망 장소            | 사망증명서, 초기 절차                      |
    +------------------------------------------------------------------+


--------------------------------------------------------------------------------
5.2 국제 조약
--------------------------------------------------------------------------------

국제 조약은 냉동보존 법적 지위의 국경 간 인정을 위한 프레임워크를
제공합니다.

[ID] 주요 조약 프레임워크:

    InternationalTreaties {
      ICLST: {
        name:     "국제 냉동보존 법적 지위 조약"
        adopted:  2055
        members:  47
        purpose:  "냉동보존 법적 지위 인정을 위한 통일 표준 수립"

        keyProvisions: {
          mutualRecognition: {
            article: 3
            content: "회원국은 다른 회원국이 부여한
                     보존 지위를 인정"
          }

          documentStandards: {
            article: 5
            content: "표준화된 문서 형식 및 인증 요건"
          }

          guardianRecognition: {
            article: 7
            content: "후견인 권한이 회원국 국경을 넘어 인정"
          }

          restorationCooperation: {
            article: 12
            content: "회복 절차에서의 협력"
          }

          disputeResolution: {
            article: 15
            content: "국가 간 분쟁을 위한 중재"
          }
        }
      }

      bilateralAgreements: {
        purpose:  "특정 국가에 대해 조약 보완"
        examples: [
          "한미 냉동보존 협력 협정",
          "EU-스위스 상호 인정 프로토콜",
          "호주-싱가포르 보존 지위 협정"
        ]
        contents: [
          "상세한 인정 절차",
          "문서 요건",
          "연락 창구 지정",
          "긴급 프로토콜"
        ]
      }
    }

[ID] 조약 회원 혜택:

    TreatyMembershipBenefits {
      fullMembers: {
        recognition:    "자동 지위 인정"
        documents:      "간소화된 인증"
        guardian:       "권한 자동 인정"
        cooperation:    "완전한 사법 협력"
      }

      associateMembers: {
        recognition:    "등록으로 인정"
        documents:      "표준 아포스티유 필요"
        guardian:       "현지 확인 필요할 수 있음"
        cooperation:    "제한적 사법 협력"
      }

      nonMembers: {
        recognition:    "사례별 기준"
        documents:      "전체 공증 필요"
        guardian:       "현지 선임 필요할 수 있음"
        cooperation:    "외교 채널을 통해"
      }
    }


--------------------------------------------------------------------------------
5.3 인정 프로토콜
--------------------------------------------------------------------------------

인정 프로토콜은 하나의 관할권이 다른 관할권에서 부여한 법적 지위를
인정하는 방법을 수립합니다.

[ID] 인정 수준:

    RecognitionLevels {
      FULL: {
        definition:   "추가 절차 없이 완전한 인정"
        requirements: "조약 회원 + 적절한 문서화"
        effect:       "지위가 현지 부여와 같이 취급"
        examples:     "ICLST 정회원 간"
      }

      PARTIAL: {
        definition:   "제한이 있는 인정"
        requirements: "양자 협정 또는 사법 결정"
        effect:       "핵심 지위 인정, 일부 권리 제한"
        examples:     "조약 회원과 준회원 간"
      }

      PENDING: {
        definition:   "검토 중인 인정"
        requirements: "신청 제출, 결정 대기 중"
        effect:       "잠정적 인정 부여 가능"
        timeline:     "일반적으로 30-90일"
      }

      NOT_RECOGNIZED: {
        definition:   "외국 지위 인정 없음"
        requirements: "조약 또는 양자 협정 없음"
        effect:       "현지 지위 결정 신청 필요"
        remedy:       "현지 보존 지위 신청"
      }
    }

[ID] 인정 절차:

    RecognitionProcess {
      automaticRecognition: {
        applicability: "ICLST 정회원 간"
        process: [
          "인증된 문서 수령",
          "조약 회원 확인",
          "현지 지위 등록",
          "관련 기관 통지"
        ]
        timeline: "7-14일"
        documents: [
          "보존 선언서 (아포스티유)",
          "후견인 선임서 (아포스티유)",
          "신원 확인",
          "조약 회원 증명서"
        ]
      }

      registrationRecognition: {
        applicability: "ICLST와 준회원 간"
        process: [
          "인정 신청서 제출",
          "인증된 문서 제출",
          "법원 심사",
          "등록 명령 발급"
        ]
        timeline: "30-60일"
        additionalRequirements: [
          "현지 법적 대리",
          "문서 번역",
          "신청 비용"
        ]
      }

      judicialRecognition: {
        applicability: "비조약 국가"
        process: [
          "인정 청원 제출",
          "모든 문서 제출 (완전 공증)",
          "법원 심리",
          "사법 결정"
        ]
        timeline: "60-180일"
        considerations: [
          "공공 정책 검토",
          "상호주의 분석",
          "절차적 적절성 검토"
        ]
      }
    }

5.4 분쟁 해결
--------------------------------------------------------------------------------

관할권이 지위 또는 인정에 대해 불일치할 때 분쟁 해결 메커니즘이
적용됩니다.

[ID] 분쟁 유형:

    ConflictTypes {
      statusConflict: {
        description: "다른 관할권이 다른 지위 부여"
        example:     "A국: PRESERVED_STATUS; B국: PERMANENT_DEATH"
        resolution:  "주요 관할권 지위 우선"
      }

      guardianConflict: {
        description: "다른 관할권에서 다른 후견인 선임"
        example:     "A국이 후견인 X 선임; B국이 후견인 Y 선임"
        resolution:  "주요 관할권 선임 우선"
      }

      assetConflict: {
        description: "자산 관리 분쟁"
        example:     "후견인 vs 현지 상속인 간 현지 자산"
        resolution:  "현지 자산 규칙 + 국제 조정"
      }

      procedureConflict: {
        description: "다른 절차적 요건"
        example:     "A국은 법원 심리 필요; B국은 자동"
        resolution:  "수령 관할권 절차 적용"
      }
    }

[ID] 해결 메커니즘:

    ConflictResolutionMechanisms {
      treatyArbitration: {
        applicability: "ICLST 회원 간 분쟁"
        forum:         "ICLST 분쟁 해결 패널"
        process: [
          "분쟁 제기",
          "서면 제출",
          "구두 심리 (필요시)",
          "패널 결정"
        ]
        binding:       "예, 회원국에 대해"
        timeline:      "90-180일"
      }

      diplomaticResolution: {
        applicability: "비조약 국가 분쟁"
        forum:         "외교 채널"
        process: [
          "정부 간 통신",
          "협상",
          "상호 합의"
        ]
        binding:       "합의에 따라"
      }

      judicialResolution: {
        applicability: "개별 사례"
        forum:         "관련 관할권 법원"
        process: [
          "적절한 법원에서 소송",
          "국제사법 적용",
          "판결"
        ]
        enforcement:   "인정 및 집행 절차를 통해"
      }

      mediation: {
        applicability: "모든 분쟁"
        forum:         "합의된 조정인 또는 조정 서비스"
        process: [
          "조정 합의",
          "조정 세션",
          "합의서"
        ]
        binding:       "합의가 성립되고 공식화된 경우"
      }
    }

[ID] 준거법 규칙:

    ChoiceOfLawRules {
      statusDetermination: {
        rule:     "보존 위치 법"
        rationale: "지속적인 지위와 가장 밀접하게 연결"
      }

      guardianAuthority: {
        rule:     "후견인 선임 법"
        rationale: "선임 관할권이 범위 설정"
      }

      assetMatters: {
        realProperty: "위치 법 (lex situs)"
        personalProperty: "보존 관할권 법"
        financialAssets: "계좌 위치 법"
      }

      familyMatters: {
        rule:     "가족 거주지 법"
        exception: "보존 관할권의 보존 지위"

      contracts: {
        rule:     "선택된 법 또는 가장 밀접한 연결 법"

      }
    }
