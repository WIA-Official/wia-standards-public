--------------------------------------------------------------------------------
3.1 사망증명서 처리
--------------------------------------------------------------------------------

사망증명서는 LIVING에서 LEGALLY_DECEASED로의 법적 지위 전환을
시작하는 기본 문서입니다.

[ID] 사망증명서 구조:

    DeathCertificate {
      certificateId:    "DC-2075-KR-000001234"
      jurisdiction:     "KR"
      issuingAuthority: "서울특별시"

      decedentInfo: {
        fullName:       "김보존"
        dateOfBirth:    "1985-03-15"
        placeOfBirth:   "대한민국 서울"
        identification: "주민등록번호: 850315-1234567"
      }

      deathInfo: {
        dateOfDeath:    "2075-06-15"
        timeOfDeath:    "14:30:00"
        placeOfDeath:   "서울대학교병원"
        causeOfDeath: {
          immediate:    "심정지"
          underlying:   "관상동맥질환"
          contributing: ["고혈압", "당뇨병"]
        }
      }

      certification: {
        certifyingPhysician: "박의료 박사"
        licenseNumber:       "MD-KR-12345"
        certificationDate:   "2075-06-15"
      }

      preservationAnnotation: {
        intendedForPreservation: true
        preservationProvider:    "WIA 냉동보존 시설"
        consentVerified:         true
        consentId:               "CON-2075-KR-001234"
      }
    }

[ID] 증명서 처리 워크플로우:

    DeathCertificateWorkflow {
      step1: {
        name:     "의료 증명"
        actor:    "담당 의사"
        actions: [
          "표준 기준으로 사망 확인",
          "사망 시간과 원인 문서화",
          "보존 동의 확인"
        ]
        timeline: "사망 즉시"
      }

      step2: {
        name:     "보존 통지"
        actor:    "의료 시설"
        actions: [
          "지정된 보존 제공자에게 통지",
          "동의 문서 확인",
          "보존 프로토콜 시작"
        ]
        timeline: "1시간 이내"
      }

      step3: {
        name:     "증명서 발급"
        actor:    "등록관"
        actions: [
          "의료 증명 검토",
          "사망증명서 발급",
          "보존 주석 추가"
        ]
        timeline: "24시간 이내"
      }

      step4: {
        name:     "지위 전환 신청"
        actor:    "법적 대리인"
        actions: [
          "보존 지위 신청",
          "지원 문서 제출",
          "법원 절차 시작"
        ]
        timeline: "72시간 이내"
      }
    }

[!] 중요: 냉동보존 대상자의 사망증명서는 적절한 지위 전환
    처리를 보장하기 위해 보존 주석을 포함해야 합니다.


--------------------------------------------------------------------------------
3.2 법적 사망 vs 생물학적 상태
--------------------------------------------------------------------------------

CRYO-LEGAL은 법적 사망(법적 지위)과 생물학적 상태(신체적 상태)를
구분합니다.

[ID] 개념적 프레임워크:

    DeathConceptFramework {
      legalDeath: {
        definition:  "법적 인격을 종료하는 공식 선언"
        basis:       "의료 증명 + 법적 등록"
        consequences: [
          "유산 개시",
          "혼인 종료",
          "계약 종료",
          "투표권 중단"
        ]
        reversibility: "법원 회복을 통해 가능"
      }

      biologicalState: {
        categories: {
          living:    "활성 생물학적 기능"
          deceased:  "생물학적 기능 중단"
          preserved: "생물학적 과정 중단되었으나 잠재적으로
                      가역적"
        }
      }

      relationship: {
        traditionalView:  "법적 사망 = 생물학적 사망 (비가역적)"
        cryoLegalView:    "법적 사망이 생물학적 최종성에 선행할 수 있음"
        implication:      "보존을 위한 새로운 법적 범주 필요"
      }
    }

[ID] 보존 지위 근거:

    PreservationStatusRationale {
      problem: {
        statement: "전통적 사망 선언은 비가역성을 가정"
        conflict:  "냉동보존이 잠재적 가역성 도입"
      }

      solution: {
        approach: "중간 법적 지위 생성"
        status:   "PRESERVED_STATUS"
        effect:   "회복 가능성을 보존하면서 사망 선언 인정"
      }

      legalEffect: {
        deathCertificate: "유효하나 잠재적으로 가역적으로 표시"
        estatePlanning:   "분배 대신 정지 보유"
        personalRights:   "종료 대신 정지"
      }
    }

[i] 참고: PRESERVED_STATUS의 법적 의제는 법률 시스템이 냉동보존
    대상자의 생물학적 사망이 영구적이지 않을 수 있다는 현실을
    수용할 수 있게 합니다.


--------------------------------------------------------------------------------
3.3 다중 관할권 고려사항
--------------------------------------------------------------------------------

사망 선언은 종종 여러 관할권을 포함하여 신중한 조정이 필요합니다.

[ID] 관할권 문제:

    JurisdictionalConsiderations {
      deathLocation: {
        description: "사망이 발생한 장소"
        relevance:   "사망증명서의 주요 관할권"
        examples: [
          "본국의 병원",
          "여행 중 외국",
          "공해상"
        ]
      }

      residenceJurisdiction: {
        description: "개인이 법적으로 거주한 곳"
        relevance:   "유산 및 지위 함의"
      }

      citizenshipJurisdiction: {
        description: "시민권 국가"
        relevance:   "영사 개입, 여권 취소"
      }

      preservationLocation: {
        description: "보존이 발생하는 곳"
        relevance:   "지속적인 지위 관리"
      }
    }

[ID] 국경 간 처리:

    CrossBorderProcessing {
      scenario1: {
        name:        "해외에서 사망, 해외에서 보존"
        deathCert:   "외국 관할권 발급"
        recognition: "본국 관할권이 인정해야 함"
        status:      "보존 위치에서 관리"
        documents:   "아포스티유 필요"
      }

      scenario2: {
        name:        "해외에서 사망, 보존을 위해 본국 송환"
        deathCert:   "외국 관할권 발급"
        transport:   "특별 허가 필요"
        preservation: "본국 관할권"
        status:      "본국 관할권에서 관리"
      }

      scenario3: {
        name:        "본국에서 사망, 해외에서 보존"
        deathCert:   "본국 관할권 발급"
        transport:   "수출 허가 필요"
        preservation: "외국 관할권"
        status:      "조정 필요"
      }

      requiredDocuments: [
        "사망증명서 (아포스티유)",
        "운송 허가",
        "영사 증명",
        "보존 동의서 (번역)",
        "보관 이전 문서"
      ]
    }

[ID] 인정 요건:

    +------------------------------------------------------------------+
    | 문서 유형               | 필요한 인증                              |
    +------------------------------------------------------------------+
    | 사망증명서              | 아포스티유 + 공인 번역                    |
    | 보존 선언서             | 아포스티유 + 공인 번역                    |
    | 법원 명령               | 아포스티유 + 필요시 집행 인가             |
    | 후견인 선임             | 아포스티유 + 인정 절차                    |
    +------------------------------------------------------------------+


--------------------------------------------------------------------------------
3.4 특수 사망 분류
--------------------------------------------------------------------------------

특정 상황은 사망 선언의 특수 처리가 필요합니다.

[ID] 특수 사례:

    SpecialDeathClassifications {
      presumedDeath: {
        definition:  "시신 없이 사망 추정"
        relevance:   "냉동보존에 적용 불가"
        handling:    "표준 추정 사망 규칙"
      }

      violentDeath: {
        definition:  "사고, 살인 또는 자살에 의한 사망"
        relevance:   "보존 전 부검 필요할 수 있음"
        handling: {
          autopsy:    "법에 의해 필요할 수 있음"
          timeline:   "보존 지연 가능"
          consent:    "법적 대리인 동의"
        }
      }

      medicallyAssistedDeath: {
        definition:  "합법적 안락사/조력사를 통한 사망"
        relevance:   "보존 의도가 있는 계획된 사망"
        handling: {
          timing:     "보존을 최적으로 타이밍 가능"
          consent:    "강화된 동의 요건"
          documentation: "추가 증명"
        }
      }

      terminalSedation: {
        definition:  "완화적 진정 후 사망"
        relevance:   "의료 조력사와 유사"
        handling:    "조정된 보존 프로토콜"
      }

      suddenDeath: {
        definition:  "빠른 대응이 필요한 예기치 못한 사망"
        relevance:   "시간 중요한 보존"
        handling: {
          standbyTeam: "가능하면 사전 준비"
          emergencyProtocol: "표준 응급 대응"
          documentation: "소급적 동의 확인"
        }
      }
    }

[ID] 법의학적 고려사항:

    ForensicConsiderations {
      autopsyRequirements: {
        mandatory: [
          "살인 의심",
          "무인 사망",
          "경찰 구금 중 사망",
          "원인 불명 사망"
        ]

        waivable: [
          "알려진 원인의 자연사",
          "의료 관리 하 사망",
          "명확한 병력이 있는 사망"
        ]
      }

      preservationCompatibility: {
        fullAutopsy:    "보존 품질 손상 가능"
        limitedAutopsy: "조정으로 가능"
        noAutopsy:      "보존에 최적"

        negotiation: {
          approach: "검시관과 협력"
          goals:    ["침습적 시술 최소화",
                     "중요 조직 보존",
                     "법적 목적 문서화"]
        }
      }

      documentation: {
        forensicReport:     "조사 수행 시 필요"
        chainOfCustody:     "전 과정 유지"
        evidencePreservation: "조직 샘플 필요할 수 있음"
      }
    }

[!] 경고: 법의학적 조사가 필요한 경우 보존이 지연될 수 있습니다.
    사전 계획과 사전 의료지시서가 지연을 최소화하는 데 도움이 됩니다.
