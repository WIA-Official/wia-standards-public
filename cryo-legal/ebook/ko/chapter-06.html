6.1 회복 워크플로우
--------------------------------------------------------------------------------

법적 회복은 성공적으로 소생된 개인에게 완전한 법적 인격을 회복하는
공식 절차입니다.

[ID] 회복 단계 개요:

    RestorationWorkflow {
      stage1: {
        name:     "INITIATED"
        trigger:  "소생 절차 시작"
        actions: [
          "법원에 소생 시작 통지",
          "후견인 확인",
          "법률 고문 참여",
          "회복 파일 개설"
        ]
        duration: "소생 시작 즉시"
      }

      stage2: {
        name:     "IDENTITY_VERIFICATION"
        trigger:  "의료적 안정화 달성"
        actions: [
          "생물학적 신원 확인",
          "CRYO-IDENTITY 프로토콜 실행",
          "문서 신원 확인",
          "가능한 경우 증인 진술"
        ]
        duration: "1-7일"
      }

      stage3: {
        name:     "MEDICAL_CLEARANCE"
        trigger:  "신원 확인됨"
        actions: [
          "완전한 의료 평가",
          "인지 평가",
          "능력 평가",
          "의료 보고서 준비"
        ]
        duration: "7-30일"
      }

      stage4: {
        name:     "LEGAL_REVIEW"
        trigger:  "의료 허가 획득"
        actions: [
          "모든 법적 문서 검토",
          "동의 준수 확인",
          "분쟁 또는 청구 평가",
          "법원 청원 준비"
        ]
        duration: "7-14일"
      }

      stage5: {
        name:     "COURT_PETITION"
        trigger:  "법적 검토 완료"
        actions: [
          "회복 청원 제출",
          "이해관계자에게 통지 송달",
          "지원 문서 제출",
          "심리 일정 요청"
        ]
        duration: "14-30일"
      }

      stage6: {
        name:     "COURT_HEARING"
        trigger:  "청원 제출됨"
        actions: [
          "신원 증거 제시",
          "의료 허가 제시",
          "이의 처리",
          "사법 결정"
        ]
        duration: "1-2일 (심리)"
      }

      stage7: {
        name:     "RESTORATION_GRANTED"
        trigger:  "법원이 회복 승인"
        actions: [
          "법원이 회복 명령 발급",
          "사망증명서 대체됨",
          "새 지위 등록",
          "통지 발송"
        ]
        duration: "명령 즉시"
      }

      stage8: {
        name:     "RIGHTS_RESTORATION"
        trigger:  "회복 명령 발급됨"
        actions: [
          "시민권 회복",
          "정부 기록 갱신",
          "새 신분증 발급",
          "신탁에서 자산 이전"
        ]
        duration: "30-90일"
      }

      stage9: {
        name:     "COMPLETED"
        trigger:  "모든 권리 회복됨"
        actions: [
          "최종 지위: LEGALLY_RESTORED",
          "후견인 권한 종료",
          "신탁 해산",
          "완전한 법적 능력 확인"
        ]
        duration: "최종 상태"
      }
    }

[ID] 워크플로우 시각화:

    INITIATED
        |
        v
    IDENTITY_VERIFICATION -----> (신원 분쟁: 일시 중지)
        |
        v
    MEDICAL_CLEARANCE ---------> (무능력: 수정된 회복)
        |
        v
    LEGAL_REVIEW --------------> (청구: 진행 전 해결)
        |
        v
    COURT_PETITION
        |
        v
    COURT_HEARING -------------> (기각: 항소 가능)
        |
        v
    RESTORATION_GRANTED
        |
        v
    RIGHTS_RESTORATION
        |
        v
    COMPLETED


--------------------------------------------------------------------------------
6.2 신원 확인 요건
--------------------------------------------------------------------------------

신원 확인은 소생된 개인이 보존된 개인과 동일인임을 보장하는 데
필수적입니다.

[ID] 신원 확인 프레임워크:

    IdentityVerificationRequirements {
      biologicalVerification: {
        mandatory: [
          "보존 전 샘플과 DNA 일치",
          "생체 인식 확인 (가능한 경우 지문)",
          "신체적 특성 비교"
        ]
        standard:    "99.99% 신뢰도 필요"
        authority:   "CRYO-IDENTITY 인증 실험실"
        documentation: "신원 확인 증명서"
      }

      documentaryVerification: {
        mandatory: [
          "원본 신분증 비교",
          "보존 동의 문서 확인",
          "사망증명서 확인"
        ]
        supporting: [
          "역사적 기록",
          "사진",
          "증인 진술"
        ]
      }

      cognitiveVerification: {
        purpose:     "의식 연속성 확인"
        methods: [
          "기억 확인 (개인 역사)",
          "지식 확인 (보존 전 사실)",
          "성격 평가"
        ]
        limitations: "일부 기억 손실 예상 및 허용"
        standard:    "완벽이 아닌 실질적 연속성"
      }
    }

[ID] 확인 절차:

    VerificationProcess {
      step1: {
        name:     "샘플 수집"
        action:   "소생된 개인에서 생물학적 샘플 수집"
        samples:  ["혈액", "타액", "필요시 기타"]
        chainOfCustody: "엄격한 문서화 필요"
      }

      step2: {
        name:     "실험실 분석"
        action:   "보존 전 샘플과 비교"
        methods:  ["DNA 시퀀싱", "지문 분석"]
        timeline: "24-72시간"
      }

      step3: {
        name:     "문서 검토"
        action:   "모든 신분증 검토"
        comparison: "보존 전후 문서"
        verification: "문서 인증"
      }

      step4: {
        name:     "인지 평가"
        action:   "기억과 인지 평가"
        assessor: "자격 있는 신경심리학자"
        documentation: "상세한 평가 보고서"
      }

      step5: {
        name:     "증명서 발급"
        action:   "신원 확인 증명서 발급"
        authority: "CRYO-IDENTITY 인증 기관"
        validity: "회복 절차에 유효"
      }
    }

[!] 중요: 신원 분쟁은 회복 진행 전에 해결되어야 합니다.
    분쟁이 발생하면 법원이 독립 전문가를 선임할 수 있습니다.


--------------------------------------------------------------------------------
6.3 법원 절차
--------------------------------------------------------------------------------

회복은 법적 인격을 공식적으로 회복하기 위한 사법 절차가 필요합니다.

[ID] 법원 절차 구조:

    CourtProceedings {
      jurisdiction: {
        proper: "보존 지위를 발급한 법원 또는
                현재 거주지 법원"
        type:   "가정법원, 상속법원 또는 지정된
                회복 법원"
      }

      petition: {
        petitioner: "소생된 개인 (변호사를 통해) 또는
                    개인을 대신한 후견인"
        contents: [
          "개인 식별",
          "보존 이력",
          "성공적 소생 증거",
          "신원 확인 증명서",
          "의료 허가",
          "회복 명령 요청"
        ]
        filingFee: "현지 법원 규칙에 따라"
      }

      notice: {
        requiredRecipients: [
          "전 배우자 (있는 경우)",
          "자녀 또는 상속인",
          "후견인",
          "수탁자",
          "정부 기관",
          "잠재적 청구가 있는 모든 사람"
        ]
        method:    "개인 송달 또는 공고"
        timeline:  "심리 최소 30일 전"
      }

      hearing: {
        format:    "공식 법원 심리"
        evidence: [
          "신원 확인 증언",
          "의료 전문가 증언",
          "개인의 증언 (가능한 경우)",
          "문서 증거"
        ]
        objections: "이해관계자가 이의 제기 가능"
        standard:  "명확하고 설득력 있는 증거"
      }

      order: {
        contents: [
          "회복 선언",
          "사망증명서 대체",
          "정부 기관 지시",
          "후견인/수탁자 권한 처분",
          "조건 또는 제한"
        ]
        effect:    "발급 즉시"
        appeal:    "표준 항소 절차"
      }
    }

[ID] 절차의 일반적 문제:

    ProceedingIssues {
      identityDispute: {
        nature:     "신원 확인에 대한 이의"
        resolution: "추가 검사, 전문가 증언"
        burden:     "이의 제기자가 신원 반증 책임"
      }

      capacityIssue: {
        nature:     "개인이 완전한 정신 능력 결여"
        resolution: "지속적 후견인이 있는 수정된 회복"
        options:    ["완전 회복", "부분 회복",
                     "연기된 회복"]
      }

      creditorClaims: {
        nature:     "보존 전 채무"
        resolution: "회복 명령에서 처리"
        principle:  "채무는 일반적으로 존속하나 제한
                     적용 가능"
      }

      familyDisputes: {
        nature:     "전 배우자, 상속인 이의"
        resolution: "법원 결정"
        considerations: ["배우자 재혼",
                         "이미 이루어진 유산 분배",
                         "자녀 양육권 변경"]
      }

      criminalMatters: {
        nature:     "보존 전 형사 문제"
        resolution: "형사 법원에 회부"
        principle:  "회복이 형사 책임을
                     소멸시키지 않음"
      }
    }

6.4 권리 회복
--------------------------------------------------------------------------------

회복 명령이 발급된 후 특정 권리는 다양한 절차를 통해 개별적으로
회복되어야 합니다.

[ID] 권리 회복 범주:

    RightsRestoration {
      civilRights: {
        voting: {
          process:   "선거 기관에 재등록"
          documents: "회복 명령 + 신분증"
          timeline:  "등록 마감에 따라"
        }

        citizenship: {
          process:   "시민권 기관에 통지"
          documents: "회복 명령"
          result:    "시민권 확인/회복"
        }

        drivingLicense: {
          process:   "면허 기관에 신청"
          requirements: "의료 허가 + 시험 필요할 수 있음"
        }

        professionalLicenses: {
          process:   "면허 기관에 신청"
          requirements: "재자격 필요할 수 있음"
          consideration: "경과 시간, 표준 변경"
        }
      }

      familyRights: {
        marriageStatus: {
          ifSpouseAlive: {
            options: ["혼인 회복 (상호 동의)",
                      "공식 해소",
                      "중간 재혼 인정"]
            courtDetermination: "분쟁 사례에 필요"
          }
          ifSpouseDeceased: {
            status: "회복 시 홀아비/과부"
          }
          ifSpouseRemarried: {
            status:     "원래 혼인 종료"
            recognition: "이후 혼인 유효"
          }
        }

        parentalRights: {
          minorChildren: {
            process: "양육권 법원 검토"
            consideration: "자녀의 최선의 이익"
            options: ["완전 회복", "공동 양육권",
                      "방문권", "회복 없음"]
          }
          adultChildren: {
            status: "부모-자녀 관계 회복"
            inheritance: "상호 상속권"
          }
        }
      }

      propertyRights: {
        heldInTrust: {
          process:   "신탁 해산 및 이전"
          timeline:  "신탁 조건 및 법원 명령에 따라"
          taxes:     "세금 영향 있을 수 있음"
        }

        distributedToHeirs: {
          principle: "일반적으로 회복 불가"
          exceptions: ["사기", "악의의 분배"]
          alternative: "유산에 대한 청구"
        }

        realProperty: {
          ifRetained: "신탁/후견인에서 소유권 이전"
          ifSold:     "가능한 경우 수익금 추적"
        }
      }

      newIdentityDocuments: {
        required: [
          "새 출생증명서 주석",
          "새 정부 신분증",
          "새 여권",
          "새 주민등록번호 (또는 재활성화)",
          "전문 자격증"
        ]
        timeline: "일반적으로 30-90일"
      }
    }


--------------------------------------------------------------------------------
장 요약
--------------------------------------------------------------------------------

핵심 개념:
  [OK] 회복 워크플로우는 9개의 정의된 단계 포함
  [OK] 신원 확인은 생물학적, 문서적, 인지적 평가 필요
  [OK] 법원 절차는 구조화된 청원과 심리 절차 따름
  [OK] 명확하고 설득력 있는 증거 기준 적용
  [OK] 권리는 특정 절차를 통해 개별적으로 회복
  [OK] 가족 권리는 복잡한 법원 결정 필요할 수 있음
  [OK] 회복 후 지원이 전환 지원
