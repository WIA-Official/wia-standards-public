2.1 지위 유형 및 정의
--------------------------------------------------------------------------------

CRYO-LEGAL은 각각 특정 법적 함의와 요건을 가진 6가지 구별되는
법적 지위 범주를 정의합니다.

[ID] 지위 유형 정의:

    LegalStatusTypes {
      LIVING: {
        code:        "LIV"
        description: "자연인의 표준 법적 지위"
        capacity:    "완전한 법적 능력"
        rights:      "모든 민사, 정치, 재산권"
        obligations: "모든 법적 의무"

        characteristics: {
          canContract:     true
          canVote:         true
          canOwnProperty:  true
          canMarry:        true
          taxLiability:    true
          criminalCapacity: true
        }
      }

      LEGALLY_DECEASED: {
        code:        "DEC"
        description: "사망이 법적으로 선언됨"
        capacity:    "없음 (표준 사망)"
        duration:    "일반적으로 짧음 - 보존 지위까지"

        consequences: {
          estateOpens:        true
          marriageTerminated: true
          contractsTerminated: "조건에 따라"
          votingRights:       false
        }

        note: "이것은 과도적 상태입니다. 냉동보존 대상자의 경우,
               영구적 유산 분배가 아닌 PRESERVED_STATUS로 이어집니다."
      }

      PRESERVED_STATUS: {
        code:        "PRE"
        description: "냉동보존 중 특수 지위"
        capacity:    "정지됨, 후견인을 통해 행사"
        duration:    "보존 기간"

        characteristics: {
          personalRights:    "정지됨"
          propertyRights:    "신탁 또는 후견인 보유"
          privacy:           "보호됨"
          dignityProtection: "보장됨"
          revivalRight:      "기본권"
        }

        protections: [
          "원인 없이 영구 사망 선언 불가",
          "신체 무결성 보호",
          "사전 의료지시서 존중",
          "후견인 감독 필요"
        ]
      }

      REVIVAL_PENDING: {
        code:        "REV"
        description: "소생 절차 시작됨"
        capacity:    "완전한 능력으로 전환 중"
        duration:    "소생 및 회복 절차 기간"

        characteristics: {
          medicalStatus:      "적극적 소생 중"
          legalStatus:        "회복 절차 시작됨"
          guardianAuthority:  "회복까지 계속"
          anticipatoryRights: "완전 회복 준비"
        }
      }

      LEGALLY_RESTORED: {
        code:        "RES"
        description: "법원 명령으로 완전한 법적 인격 회복"
        capacity:    "완전한 법적 능력"
        rights:      "모든 권리 회복"

        effects: {

          propertyRights:     "회복됨"
          criminalRecord:     "검토 및 갱신"
        }
      }

      PERMANENT_DEATH: {
        code:        "PER"
        description: "최종적, 비가역적 사망"
        capacity:    "없음"
        finality:    "절대적"

        triggers: [
          "소생 불가능 판단",
          "보존 시설 장애",
          "명시적 종료 요청 존중",

--------------------------------------------------------------------------------
2.2 상태 전환 모델
--------------------------------------------------------------------------------

법적 지위 전환은 적절한 권한을 가지고 정의된 경로를 따라야 합니다.

[ID] 상태 전환 다이어그램:

    LIVING ---------> LEGALLY_DECEASED
                            |
                            v
                      PRESERVED_STATUS ---------> PERMANENT_DEATH
                            |
                            v
                      REVIVAL_PENDING -----------> PERMANENT_DEATH
                       /    |
                      /     v
        (복귀)       /  LEGALLY_RESTORED
    PRESERVED_STATUS

[ID] 유효한 전환:

    TransitionRules {
      fromLiving: {
        validTargets: ["LEGALLY_DECEASED"]
        requirements: {
          deathCertificate: "필수"
          medicalConfirmation: "필수"
          authority: "의료 전문가 + 등록관"
        }
      }

      fromLegallyDeceased: {
        validTargets: ["PRESERVED_STATUS"]
        requirements: {
          preservationInitiated: "필수"
          preservationDeclaration: "필수"
          courtOrder: "필수 또는 관할권에 따라 자동"
          authority: "법원 또는 지정 기관"
        }
      }

      fromPreservedStatus: {
        validTargets: ["REVIVAL_PENDING", "PERMANENT_DEATH"]

        toRevivalPending: {
          requirements: {
            revivalInitiation: "의료 확인"
            courtNotification: "필수"
            guardianConsent: "필수"
          }
        }

        toPermanentDeath: {
          requirements: {
            cause: "보존 실패 또는 종료 요청"
            courtOrder: "필수"
            medicalCertification: "필수"
          }
        }
      }

      fromRevivalPending: {
        validTargets: ["LEGALLY_RESTORED", "PRESERVED_STATUS", "PERMANENT_DEATH"]

        toLegallyRestored: {
          requirements: {
            revivalSuccess: "의료 확인"
            identityVerification: "CRYO-IDENTITY 확인"
            courtRestoration: "법원 명령 필수"
          }
        }

        toPreservedStatus: {
          requirements: {
            revivalPaused: "의료 결정"
            courtNotification: "필수"
          }
        }

        toPermanentDeath: {
          requirements: {
            revivalFailed: "의료 결정"
            courtOrder: "필수"
          }
        }
      }

      fromLegallyRestored: {
        validTargets: []
        note: "성공적 소생의 최종 상태"
      }

      fromPermanentDeath: {
        validTargets: []
        note: "최종 상태 - 절대적으로 비가역적"
      }
    }

[!] 중요: 지위 전환은 감사되며 기록되면 불변합니다.
    모든 전환은 적절한 권한과 문서화가 필요합니다.


--------------------------------------------------------------------------------
2.3 지위별 법적 권리
--------------------------------------------------------------------------------

각 지위 범주는 특정 권리와 제한을 수반합니다.

[ID] 권리 매트릭스:

    +------------------------------------------------------------------+
    | 권리 범주           | LIV | DEC | PRE | REV | RES | PER          |
    +------------------------------------------------------------------+
    | 개인의 자유         | 예  | 아니오| 정지 | 정지 | 예  | 아니오     |
    | 재산 소유           | 예  | 유산 | 신탁 | 신탁 | 예  | 유산       |
    | 계약 능력           | 예  | 아니오| 후견 | 후견 | 예  | 아니오     |
    | 투표권              | 예  | 아니오| 아니오| 아니오| 예 | 아니오    |
    | 혼인 상태           | 예  | 종료 | 정지 | 정지 | 검토 | 종료      |
    | 친권                | 예  | 종료 | 정지 | 정지 | 검토 | 종료      |
    | 사생활 보호         | 예  | 제한 | 예  | 예  | 예  | 아니오      |
    | 존엄성 보호         | 예  | 예  | 예  | 예  | 예  | 예          |
    | 신체 자율권         | 예  | 제한 | 후견 | 의료 | 예  | 아니오     |
    | 소생권              | 해당없음 | 해당없음 | 예 | 예 | 해당없음 | 해당없음 |
    +------------------------------------------------------------------+

    범례:
      예   = 완전한 권리
      아니오 = 권리 없음
      정지 = 정지됨 (재개 가능)
      신탁 = 신탁으로 보유
      후견 = 후견인을 통해 행사
      유산 = 유산 규칙 적용
      제한 = 제한적 보호
      검토 = 법원에 의해 검토/회복
      의료 = 의료 권한

[ID] 지위별 세부 권리:

    PreservedStatusRights {
      fundamental: {
        revival: {
          description: "가능할 때 소생 시도 권리"
          protection:  "타인에 의해 포기될 수 없음"
          exception:   "유효한 사전 의료지시서"
        }

        dignity: {
          description: "존중과 존엄으로 대우받을 권리"
          protection:  "보존 기간 내내 적용"
        }

        privacy: {
          description: "개인 정보 보호"
          protection:  "제한적 공개 규칙"
        }

        bodyIntegrity: {
          description: "신체 무결성 권리"
          protection:  "무단 시술 불가"
          guardian:    "필요한 의료 결정 수행"
        }
      }

      protectedInterests: {
        reputation: {
          description: "명예훼손 보호"
          duration:    "보존 중 계속"
          enforcement: "후견인이 조치 가능"
        }

        intellectualProperty: {
          description: "지적재산권 계속"
          management:  "수탁자 또는 후견인 관리"
          duration:    "표준 IP 기간 적용"
        }

        advanceDirectives: {
          description: "사망 전 의사 존중"
          scope:       "소생 결정, 돌봄 선호"
          enforcement: "후견인이 따라야 함"
        }
      }
    }


--------------------------------------------------------------------------------
2.4 문서 요건
--------------------------------------------------------------------------------

각 지위는 특정 문서를 필요로 합니다.

[ID] 지위별 필수 문서:

    DocumentationRequirements {
      LIVING: {
        standard: ["정부 발급 신분증", "출생증명서"]
        cryoRelated: ["사전 의료지시서 (선택)", "동의 문서"]
      }

      LEGALLY_DECEASED: {
        required: [
          "사망증명서",
          "사망의 의료 증명",
          "사망자 신원 확인"
        ]
        timeline: "사망 즉시"
      }

      PRESERVED_STATUS: {
        required: [
          "사망증명서 (원본)",
          "보존 선언서",
          "보존 지위 법원 명령",
          "시설 보관 문서",
          "후견인 선임 명령"
        ]
        timeline: "보존 시작 후 30일 이내"
      }

      REVIVAL_PENDING: {
        required: [
          "소생 시작 증명서",
          "의료 상태 보고서",
          "법원 통지",
          "후견인 확인서"
        ]
        timeline: "소생 시작 시"
      }

      LEGALLY_RESTORED: {
        required: [
          "법원 회복 명령",
          "새로운 신분증",
          "의료 허가",
          "신원 확인 증명서"
        ]
        timeline: "성공적 회복 시"
      }

      PERMANENT_DEATH: {
        required: [
          "영구 사망 선언",
          "법원 명령 (해당시)",
          "의료 증명",
          "유산 최종화 문서"
        ]
        timeline: "결정 시"
      }
    }

[ID] 문서 표준:

    DocumentStandards {
      format: {
        encoding:      "UTF-8"
        signature:     "디지털 + 필요시 물리적"
        authentication: "공증 + 국제용 아포스티유"
        storage:       "안전, 이중화, 변조 방지"
      }

      retention: {
        preservedStatus:   "보존 기간 + 100년"
        restoredStatus:    "생애 + 100년"
        permanentDeath:    "선언 후 100년"
      }

      accessibility: {
        guardian:          "전체 접근"
        legalCounsel:      "권한 있는 전체 접근"
        family:            "정책에 따른 제한적 접근"
        public:            "기본 지위만"
      }
    }

장 요약
--------------------------------------------------------------------------------

핵심 개념:
  [OK] 6가지 법적 지위 범주: LIVING, DECEASED, PRESERVED, REVIVAL,
       RESTORED, PERMANENT_DEATH
  [OK] 지위 전환은 권한을 가지고 정의된 경로를 따름
  [OK] 각 지위는 특정 권리와 제한을 가짐
  [OK] 권리는 보존 중 정지되지만 소멸되지 않음
  [OK] 각 지위에 종합적인 문서화 필요
  [OK] 문서 보존은 개별 지위 변경 이후로도 연장
