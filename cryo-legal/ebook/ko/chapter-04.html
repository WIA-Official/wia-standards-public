4.1 보존 선언
--------------------------------------------------------------------------------

보존 선언은 냉동보존 대상자에 대해 PRESERVED_STATUS를 공식적으로
수립합니다.

4.2 법적 지위 정지
--------------------------------------------------------------------------------

PRESERVED_STATUS가 부여되면 사망의 특정 법적 결과는 실행되지 않고
정지됩니다.

[ID] 정지 vs 실행 효과:

    StatusSuspensionEffects {
      suspended: {
        estateDistribution: {
          normalEffect:    "상속인에게 자산 분배"
          suspendedEffect: "자산 신탁으로 보유"
          management:      "후견인 또는 수탁자 관리"
          duration:        "회복 또는 영구 사망까지"
        }

        marriageTermination: {
          normalEffect:    "사망 시 혼인 종료"
          suspendedEffect: "혼인 상태 정지"
          spouseRights:    "종료 청원 또는 정지 상태 유지 가능"
          duration:        "법원 결정까지"
        }

        parentalRights: {
          normalEffect:    "사망 시 권리 종료"
          suspendedEffect: "권리 정지"
          childGuardianship: "자녀를 위한 별도 후견인 선임"
          duration:        "회복 또는 영구 사망까지"
        }

        contracts: {
          normalEffect:    "많은 계약이 사망 시 종료"
          suspendedEffect: "가능한 경우 이행 정지"
          management:      "후견인이 관리 또는 종료 가능"
          exceptions:      "일부 계약은 명시적 종료"
        }
      }

      executed: {
        voterRegistration: {
          effect:          "유권자 명부에서 삭제"
          restoration:     "회복 시 재등록 필요"
        }

        publicOffice: {
          effect:          "공직 공석"
          restoration:     "재선출/재임명 필요"
        }

        professionalLicenses: {
          effect:          "정지 또는 만료"
          restoration:     "재자격 필요할 수 있음"
        }
      }
    }

4.3 보존 중 권리
--------------------------------------------------------------------------------

보존된 개인은 보존 중에도 특정 기본권을 유지합니다.

[ID] 유지되는 권리:

    PreservationRights {
      fundamentalRights: {
        rightToRevival: {
          description: "가능할 때 소생 시도 권리"
          source:      "기본권으로 인정"
          protection:  "타인에 의해 포기될 수 없음"
          limitations: "기술적 가능성에 따름"
        }

        dignityRight: {
          description: "존엄하게 대우받을 권리"
          source:      "인간 존엄성 원칙"
          protection:  "보존 기간 내내 적용"
          obligations: ["존중하는 취급",
                       "사생활 보호",
                       "착취 금지"]
        }

        privacyRight: {
          description: "개인 정보 보호"

      protectedInterests: {
        reputation: {
          description: "명예훼손 보호"

        intellectualProperty: {
          description: "지적재산권 계속"

4.4 지위 모니터링 및 갱신
--------------------------------------------------------------------------------

보존 지위는 지속적인 모니터링과 정기적 갱신이 필요합니다.

[ID] 모니터링 요건:

    StatusMonitoring {
      periodicReporting: {
        quarterly: {
          reporter:   "보존 시설"
          contents: [
            "보관 상태 확인",
            "보안 상태",
            "사건 또는 우려사항"
          ]
          recipient:  "후견인 + 법원 (연간 요약)"
        }

        annual: {
          reporter:   "후견인"
          contents: [
            "재정 회계",
            "보존 상태 검토",
            "신탁 관리 보고",
            "법적 절차"
          ]
          recipient:  "감독 법원"
        }

        asNeeded: {
          reporter:   "모든 당사자"
          triggers: [
            "상황의 중대한 변화",
            "시설 문제",
            "후견인 우려",
            "제3자 분쟁"
          ]
          recipient:  "법원"
        }
      }

      facilityInspections: {
        routine:    "규제 기관에 의해 연간"
        special:    "민원 또는 사건 시"
        results:    "후견인 및 법원과 공유"
      }
    }

[ID] 지위 갱신:

    StatusUpdateEvents {
      facilityTransfer: {
        trigger:     "시설 폐쇄 또는 개인 이전"
        process: [
          "법원 승인 필요",
          "이전 프로토콜 실행",
          "기록 갱신",
          "새 시설 등록"
        ]
        documentation: "이전 증명서 발급"
      }

      guardianChange: {
        trigger:     "후견인 사임, 해임 또는 사망"
        process: [
          "법원 청원 제출",
          "새 후견인 지명",
          "법원 승인",
          "권한 이전"
        ]
        documentation: "새 후견 명령"
      }

      jurisdictionChange: {
        trigger:     "다른 관할권으로 이전"
        process: [
          "새 관할권에서 인정",
          "지위 등록",
          "후견인 선임 인정",
          "문서 아포스티유"
        ]
        documentation: "새 관할권 등록"
      }

      revivalInitiation: {
        trigger:     "의료팀이 소생 시작"
        process: [
          "법원에 즉시 통지",
          "후견인 확인",
          "지위를 REVIVAL_PENDING으로 변경",
          "회복 절차 시작"
        ]
        documentation: "소생 시작 증명서"
      }
    }
