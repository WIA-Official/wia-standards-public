# 1장: 합성생물학 레지스트리 시스템 소개

## 개요

생물학적 부품 레지스트리는 합성생물학이 진정한 엔지니어링 분야로 기능할 수 있도록 하는 기초 인프라입니다. 이 장에서는 레지스트리가 무엇인지, 왜 중요한지, 어떻게 구조화되어 있는지, 그리고 더 넓은 합성생물학 생태계에서의 역할을 탐구합니다.

## 1.1 생물학적 부품 레지스트리란 무엇인가?

### 정의 및 목적

생물학적 부품 레지스트리는 표준화된 생물학적 구성 요소에 대한 정보를 카탈로그화, 문서화 및 배포하는 구조화된 저장소입니다. 종종 "부품", "장치" 또는 "시스템"이라고 하는 이러한 구성 요소는 개별 유전자와 프로모터에서 완전한 유전 회로, 심지어 전체 생물체에 이르기까지 다양합니다.

**레지스트리의 핵심 기능**:

1. **카탈로그화**: 고유 식별자로 생물학적 부품 구성
2. **문서화**: 서열, 특성화 데이터 및 사용 정보 저장
3. **배포**: 부품 및 관련 정보에 대한 접근 제공
4. **표준화**: 공통 형식 및 사양 시행
5. **버전 관리**: 부품의 변경 및 업데이트 추적
6. **품질 보증**: 부품이 지정된 기준을 충족하는지 검증
7. **발견**: 사용 가능한 부품의 검색 및 비교 가능

### 저장되는 정보 유형

생물학적 레지스트리는 여러 계층의 정보를 포함합니다:

**서열 정보**:
- DNA 서열 (프로모터, 코딩 서열, 종결자)
- RNA 서열 (리보스위치, 앱타머, 가이드)
- 단백질 서열 (효소, 구조 단백질, 조절자)
- 완전한 플라스미드 서열
- 게놈 세그먼트

**기능적 특성화**:
- 다양한 조건에서의 발현 수준
- 결합 친화도 및 동역학 매개변수
- 숙주 생물체에 대한 성장 영향
- 다른 부품과의 호환성
- 환경 반응

**메타데이터**:
- 생성자/기여자 정보
- 생성 날짜 및 마지막 수정
- 기원 생물체
- 테스트된 숙주 생물체
- 문헌 참조
- 부품 계보 및 파생 이력

**실용 정보**:
- 물리적 샘플 가용성
- 클로닝 벡터 및 저항성 마커
- 성장 및 유지 프로토콜
- 알려진 문제 또는 실패 모드
- 사용자 경험 및 피드백

### 다른 엔지니어링 레지스트리와의 비교

생물학적 부품 레지스트리는 다른 엔지니어링 분야의 구성 요소 저장소와 유사한 기능을 수행합니다:

**전자공학**:
- 부품 카탈로그 (저항기, 커패시터, IC)
- 전기적 특성이 있는 데이터시트
- 애플리케이션 노트 및 참조 설계
- 예: Digi-Key, Mouser, Octopart

**소프트웨어**:
- 패키지 저장소 (npm, PyPI, Maven)
- 버전 관리 시스템 (GitHub, GitLab)
- API 문서
- 예: GitHub, npm 레지스트리

**기계공학**:
- CAD 부품 라이브러리 (McMaster-Carr)
- 재료 사양
- 제조 표준
- 예: McMaster-Carr, Grainger

**생물학적 레지스트리의 고유한 측면**:

전자 또는 소프트웨어 구성 요소와 달리 생물학적 부품은:
- 많은 상호작용이 있는 복잡한 세포 맥락에서 작동
- 시간이 지남에 따라 돌연변이 및 진화할 수 있음
- 환경 조건에 크게 의존
- 고유한 가변성과 노이즈 존재
- 안전 또는 보안 문제 제기 가능
- 광범위한 규제 대상

이러한 복잡성은 생물학적 레지스트리를 더 어렵게 만들지만 분야 발전에 더욱 중요하게 만듭니다.

## 1.2 역사적 발전

### 초기: 레지스트리 이전 시대 (2000년 이전)

공식화된 레지스트리 이전에는 생물학적 지식이 다음에 분산되어 있었습니다:
- 개별 실험실 냉동고 및 플라스미드 컬렉션
- 문헌 설명 (종종 불완전)
- 비공식 공유 네트워크
- 상업 공급업체 (제한된 선택)

**이 접근 방식의 문제점**:
- 관련 부품을 찾기 어려움
- 일관성 없는 문서
- 낮은 재현성
- 실험실 간 중복 작업
- 새로운 연구자에 대한 높은 진입 장벽

### BioBricks의 탄생 (2003-2006)

2003년에 Drew Endy, Tom Knight 등이 설립한 BioBricks Foundation은 혁명적인 개념을 도입했습니다: 표준화되고 상호교환 가능한 생물학적 부품.

**주요 혁신**:
- 일관된 제한 부위를 사용한 표준 조립 방법
- 부품 카탈로그화를 위한 레지스트리 프레임워크
- 소프트웨어에서 차용한 오픈소스 철학
- 법적 프레임워크 (BioBricks Public Agreement)

"BioBrick"이라는 용어는 레고 블록과 유사하게 표준화된 생물학적 부품의 동의어가 되었습니다.

### iGEM 및 레지스트리 성장 (2004-현재)

2004년 MIT에서 시작된 International Genetically Engineered Machine (iGEM) 대회는 생물학적 부품 레지스트리의 폭발적인 성장을 주도했습니다:

**타임라인**:
- **2004년**: 5개 팀으로 첫 iGEM, ~100개 부품 기여
- **2006년**: Registry of Standard Biological Parts 온라인 (parts.igem.org)
- **2008년**: 84개 팀, 레지스트리 1,000개 부품 초과
- **2012년**: 190개 이상 팀, 레지스트리 10,000개 부품 초과
- **2016년**: iGEM Foundation 설립, 레지스트리 국제화
- **2020년**: 20,000개 이상 부품, 50개 이상 국가 참가자
- **2025년**: 여러 상호 연결된 레지스트리를 갖춘 성숙한 생태계

### 병행 발전

iGEM이 성장하는 동안 다른 레지스트리 시스템도 등장했습니다:

**Addgene** (2004):
- MIT에서 설립되어 플라스미드 공유 간소화
- 과학자 간 플라스미드 배포
- 현재 6,000개 이상 실험실의 100,000개 이상 플라스미드 호스팅
- CRISPR 구조물, 바이러스 벡터 등 포함

**JBEI Public Registry** (2010):
- Joint BioEnergy Institute의 기여
- 바이오 연료 생산 경로에 중점
- 고품질, 잘 특성화된 부품
- 설계 도구와 통합

**SynBioHub** (2017):
- 분산 레지스트리 인프라
- SBOL 데이터 표준 기반
- 기관이 자체 레지스트리 호스팅 가능
- 레지스트리 인스턴스 간 상호운용성

**상업 플랫폼**:
- Benchling (2012): 클라우드 기반 생물학 R&D 플랫폼
- Ginkgo Bioworks (2008): 생물체 엔지니어링 플랫폼
- Zymergen (2013): 머신러닝 기반 바이오 제조 (현재 전환)

### 표준의 진화

레지스트리 개발과 병행하여 데이터 표준도 진화했습니다:

**SBOL (Synthetic Biology Open Language)**:
- 버전 1.0: 2011
- 버전 2.0: 2015 (기능 설명 추가)
- 버전 3.0: 2020 (포괄적인 재설계)
- 유전 회로 다이어그램을 위한 시각적 글리프

**GenBank 형식**:
- 1988년부터 NCBI의 표준
- 서열 제출을 위한 범용
- 풍부한 주석 기능
- 문헌 (PubMed)과 통합

**FASTA 형식**:
- 간단한 서열 형식 (1985)
- 널리 지원됨
- 쉽게 파싱 및 생성
- 제한된 메타데이터 용량

## 1.3 레지스트리 아키텍처 및 구성 요소

### 핵심 구성 요소

현대 생물학적 레지스트리는 여러 상호 연결된 시스템으로 구성됩니다:

#### 1. 데이터베이스 계층

데이터베이스는 모든 레지스트리 정보를 저장합니다:

**서열 데이터베이스**:
- 1차 서열 데이터 (DNA, RNA, 단백질)
- 주석 (특징, 결합 부위, 도메인)
- 벡터 백본 및 조립 표준 마커

**메타데이터 데이터베이스**:
- 부품 설명 및 기능 정보
- 저작자 및 출처
- 타임스탬프 및 버전 기록
- 문헌 및 기타 데이터베이스에 대한 상호 참조

**특성화 데이터베이스**:
- 실험 측정
- 성장 곡선 및 발현 수준
- 형광, 흡광도 및 기타 분석 데이터
- 환경 조건 기록

**사용자 데이터베이스**:
- 사용자 계정 및 권한
- 기여 이력
- 사용 추적 및 분석

**전형적인 스키마 예제**:

```sql
CREATE TABLE biological_parts (
    part_id VARCHAR(50) PRIMARY KEY,
    part_name VARCHAR(200),
    part_type ENUM('promoter', 'rbs', 'cds', 'terminator', 'plasmid', 'composite'),
    sequence TEXT,
    length INT,
    description TEXT,
    creator_id INT,
    created_date TIMESTAMP,
    modified_date TIMESTAMP,
    status ENUM('available', 'discontinued', 'planning'),
    biosafety_level INT,
    FOREIGN KEY (creator_id) REFERENCES users(user_id)
);

CREATE TABLE characterization_data (
    data_id INT PRIMARY KEY AUTO_INCREMENT,
    part_id VARCHAR(50),
    measurement_type VARCHAR(100),
    value DECIMAL(15,6),
    units VARCHAR(50),
    conditions TEXT,
    measurement_date TIMESTAMP,
    FOREIGN KEY (part_id) REFERENCES biological_parts(part_id)
);

CREATE TABLE part_relationships (
    relationship_id INT PRIMARY KEY AUTO_INCREMENT,
    parent_part_id VARCHAR(50),
    child_part_id VARCHAR(50),
    relationship_type ENUM('contains', 'derived_from', 'similar_to', 'precedes'),
    FOREIGN KEY (parent_part_id) REFERENCES biological_parts(part_id),
    FOREIGN KEY (child_part_id) REFERENCES biological_parts(part_id)
);
```

#### 2. 웹 인터페이스

다음을 제공하는 사용자 대상 포털:

**검색 및 탐색**:
- 설명에 걸친 전체 텍스트 검색
- 부품 유형, 생물체, 기능별 필터링
- 여러 기준으로 고급 검색
- 패싯 네비게이션

**부품 페이지**:
- 상세한 부품 정보 및 문서
- 주석이 있는 서열 뷰어
- 특성화 데이터 및 플롯
- 사용 예제 및 프로토콜
- 다운로드 옵션 (FASTA, GenBank, SBOL)

**제출 포털**:
- 부품 제출을 위한 웹 양식
- 서열 업로드 및 검증
- 메타데이터 입력 및 편집
- 일괄 제출 도구

**사용자 대시보드**:
- 개인 컬렉션 관리
- 기여 추적
- 사용 통계 보기
- 알림 받기

#### 4. 물리적 저장소

많은 레지스트리가 물리적 DNA 샘플을 유지합니다:

**샘플 관리**:
- -80°C 냉동고의 냉동 글리세롤 스톡
- 플라스미드 DNA 제제
- 필터 페이퍼에 건조된 DNA
- 주문형 유전자 합성

**배포 시스템**:
- 샘플 요청 처리
- 배송 전 품질 관리
- 물질 이전 계약
- 배송 물류

**예: Addgene의 시스템**:
- 과학자들이 완전한 문서와 함께 플라스미드 기탁
- Addgene이 서열 및 품질 확인
- 전 세계 요청 과학자에게 배포
- 마스터 스톡 및 백업 복사본 유지

#### 5. 계산 도구

레지스트리 데이터 작업을 위한 통합 도구:

**서열 분석**:
- 유사성 검색을 위한 BLAST
- 다중 서열 정렬
- 특징 예측 (프로모터, RBS 등)
- 제한 효소 매핑

**설계 도구**:
- 유전 회로 설계
- 부품 조립 계획
- 최적화 (코돈 사용, 2차 구조)
- 시뮬레이션 및 모델링

**시각화**:
- 플라스미드 맵
- 유전 회로 다이어그램 (SBOL Visual)
- 서열 정렬
- 발현 데이터 플롯

## 1.4 레지스트리 시스템 유형

생물학적 레지스트리는 각각 다른 요구를 충족하는 다양한 형태로 제공됩니다:

### 공개 커뮤니티 레지스트리

**iGEM Registry of Standard Biological Parts**:
- 모든 연구자에게 개방
- 주로 학생 기여
- 20,000개 이상 부품
- 다양한 품질 수준
- 강력한 교육적 초점

**Addgene**:
- 연구자 간 공유
- 더 높은 품질 임계값
- 동료 심사된 플라스미드
- 물리적 배포 강조
- 무료 및 상업 모델 모두

**강점**:
- 광범위한 참여
- 다양한 부품 컬렉션
- 낮은 진입 장벽
- 커뮤니티 주도 혁신

**과제**:
- 가변적인 품질
- 불완전한 문서
- 부품이 유지되지 않을 수 있음
- 제한된 지원

### 기관 레지스트리

많은 조직이 내부 레지스트리를 유지합니다:

**학술 실험실**:
- 실험실 특정 부품의 비공개 컬렉션
- 종종 Benchling 또는 유사한 플랫폼에 구축
- Electronic Lab Notebooks (ELN)와 통합
- 발표 후 공개 공유 가능

**연구 기관**:
- JBEI (Joint BioEnergy Institute)
- Broad Institute
- Sanger Institute
- 종종 특정 연구 분야에 중점

**강점**:
- 높은 품질 표준
- 잘 특성화된 부품
- 제작자의 직접 지원
- 특정 응용 프로그램에 특화

**과제**:
- 제한된 공개 접근
- 더 작은 부품 컬렉션
- 공개 노력을 중복할 수 있음

### 상업 레지스트리

독점 레지스트리를 유지하는 회사:

**Ginkgo Bioworks**:
- 대규모 내부 레지스트리 (100,000개 이상 부품)
- 머신러닝 기반 특성화
- 자동화된 설계-구축-테스트 플랫폼
- 선택적으로 부품 공개 공유

**Twist Bioscience**:
- 유전자 합성 회사
- 최적화된 서열 레지스트리
- 합성 파이프라인과 통합
- 고객별 컬렉션

**Zymergen** (역사적):
- 균주 최적화를 위한 머신러닝
- 광범위한 게놈 변이 라이브러리
- 독점 특성화 데이터

**Benchling**:
- 레지스트리 기능이 있는 클라우드 플랫폼
- 200,000명 이상의 과학자가 사용
- 공개 및 비공개 레지스트리 옵션
- 전체 R&D 워크플로우와 통합

**강점**:
- 고처리량 특성화
- 고급 자동화
- 전문적인 유지 관리
- 품질 보증

**과제**:
- 주로 독점
- 접근에 종종 지불 필요
- 커뮤니티 입력 감소

### 특수 레지스트리

특정 도메인에 중점:

**CRISPR 라이브러리**:
- Addgene의 CRISPR 컬렉션
- Broad Institute GPP (Genetic Perturbation Platform)
- 게놈 전체 녹아웃/활성화 라이브러리

**바이러스 레지스트리**:
- ATCC 바이러스 컬렉션
- BEI Resources (생물 방어)
- 바이러스 벡터 저장소

**단백질 레지스트리**:
- UniProt (포괄적인 단백질 서열)
- PDB (Protein Data Bank 구조)
- 단백질 발현 벡터 컬렉션

**경로 레지스트리**:
- KEGG (대사 경로)
- BioCyc (경로/게놈 데이터베이스)
- Reactome (생물학적 경로)

### 분산 레지스트리 네트워크

연합 레지스트리를 위한 새로운 모델:

**SynBioHub 네트워크**:
- 분산 레지스트리 인스턴스
- 공통 API 및 데이터 형식 (SBOL)
- 각 기관이 자체 인스턴스 호스팅
- 레지스트리 간 검색 및 발견

**장점**:
- 기관이 통제 유지
- 데이터 주권
- 상호운용성
- 확장성

**기술적 구현**:
- 쉬운 배포를 위한 Docker 컨테이너
- 통합을 위한 RESTful API
- 의미론적 쿼리를 위한 SPARQL
- 자동 복제 및 백업

## 1.5 합성생물학 생태계에서의 역할

### 모듈식 설계 가능

레지스트리는 생물학적 엔지니어링을 진정으로 모듈식으로 만듭니다:

**레지스트리 이전**:
- 각 실험실이 처음부터 부품 개발
- 다른 출처의 부품 결합 어려움
- 표준화 또는 상호운용성 없음
- 높은 노력 중복

**레지스트리와 함께**:
- 기존의 특성화된 부품 재사용
- 글로벌 컬렉션에서 혼합 및 매칭
- 커뮤니티 지식을 기반으로 구축
- 새로운 기여에 집중

**예제 워크플로우**:
1. 원하는 강도의 프로모터를 레지스트리에서 검색
2. 적절한 번역 속도를 가진 RBS (ribosome binding site) 찾기
3. 서열 데이터베이스에서 관심 유전자 추가
4. 발현 카세트를 완성하기 위해 종결자 포함
5. 표준 벡터에 조립
6. 테스트 및 특성화
7. 개선된 설계를 레지스트리에 다시 제출

### 연구 가속화

레지스트리는 개념에서 프로토타입까지의 시간을 극적으로 줄입니다:

**정량적 영향**:
- 부품 선택: 며칠 → 몇 시간
- 부품 획득: 몇 주 → 며칠 (또는 합성 부품의 경우 즉시)
- 조립 계획: 며칠 → 몇 시간 (계산 도구 사용)
- 문서화: 몇 주 → 며칠 (표준화된 형식)

**사례 연구: 2019 iGEM 팀**:
- 프로젝트: 중금속 오염에 대한 바이오센서
- iGEM Registry에서 15개 부품 사용
- 2주 만에 맞춤형 회로 설계
- 8주 만에 기능 프로토타입
- 총 프로젝트: 12주 (레지스트리 없이 6-12개월 대 12주)

### 교육 촉진

레지스트리는 합성생물학 교육의 장벽을 낮춥니다:

**교육적 이점**:
- 학생들이 실제 연구 품질 부품에 접근
- 표준화된 시스템으로 실습 경험
- 전문적인 관행을 조기에 학습
- 글로벌 지식 기반에 기여
- 커뮤니티 지식 기반에 기여

**교육 플랫폼으로서의 iGEM**:
- 50,000명 이상의 학생 참여 (2004-2025)
- 실험실 기술 개발
- 문서화 및 커뮤니케이션 가르침
- IP 및 윤리적 고려사항 소개
- 합성생물학 인력 파이프라인

### 산업 지원

상업 합성생물학은 레지스트리에 크게 의존합니다:

**산업 응용**:
- 바이오 연료 생산 (DuPont, Genomatica)
- 제약 제조 (Amyris, Zymergen)
- 화학 합성 (Ginkgo, Genomatica)
- 식품 생산 (Impossible Foods, Perfect Day)
- 재료 (Bolt Threads, Modern Meadow)

**회사 사용 사례**:

**Ginkgo Bioworks**:
- 100,000개 이상 유전자 부품의 내부 레지스트리
- 자동화된 특성화 플랫폼
- 산업 전반의 고객을 위한 생물체 설계
- 레지스트리로 빠른 프로토타이핑 및 반복 가능

**Zymergen** (전환 이전):
- 머신러닝 기반 균주 최적화
- 수백만 개의 유전적 변이 테스트
- 조건에 걸친 성능을 추적하는 레지스트리
- 데이터 기반 설계 가능

### 재현성 가능

레지스트리는 합성생물학의 재현성 문제를 다룹니다:

**레지스트리 없는 문제**:
- 발표된 작업의 서열 오류
- 모호한 설명
- 플라스미드 및 균주 손실
- 정확한 자료를 얻을 수 없음

**레지스트리 솔루션**:
- 확정적인 서열 기록
- 고유 식별자 (BBa_K000001)
- 물리적 샘플 가용성
- 변경에 대한 버전 관리

**재현성을 위한 표준**:
- 완전한 서열 기탁
- 상세한 특성화 프로토콜
- 원시 데이터 가용성
- 계산 코드 공유 (GitHub)

### 협력 촉진

레지스트리는 전 세계 연구자를 연결합니다:

**협력 유형**:
- 대륙 간 부품 공유
- 다중 실험실 검증 연구
- 분산 특성화 노력
- 공동 표준 개발

**예: COVID-19 대응**:
- SARS-CoV-2 서열의 신속한 공유
- Addgene을 통해 배포된 진단 분석 부품
- 백신 구성 요소 레지스트리
- 전례 없는 규모의 글로벌 협력
