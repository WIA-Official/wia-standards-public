# Chapter 4: DNA Sequence Databases and Part Characterization

## Overview

This chapter explores the major international sequence databases (NCBI GenBank, ENA, DDBJ), their structure and submission protocols, data formats, characterization standards, and how they complement synthetic biology registries like iGEM and Addgene.

## 4.1 The International Nucleotide Sequence Database Collaboration

### INSDC Overview

The **International Nucleotide Sequence Database Collaboration (INSDC)** is a partnership among three organizations:
- **NCBI** (National Center for Biotechnology Information) - USA
- **ENA** (European Nucleotide Archive) - Europe
- **DDBJ** (DNA Data Bank of Japan) - Japan

**Mission**: Comprehensive collection of all publicly available nucleotide sequences

**Key Principles**:
- **Synchronization**: Daily exchange ensures all three contain identical data
- **Permanence**: Sequence records never deleted, only updated
- **Openness**: Free, unrestricted access to all data
- **Standardization**: Common data formats and standards
- **Attribution**: Proper credit to sequence submitters

**Scale** (2025):
- Over 250 billion nucleotide bases
- 500+ million sequence records
- Doubling approximately every 18 months
- Petabytes of associated data

### Historical Development

**Pre-Database Era** (before 1980s):
- Sequences published in journal articles
- Manual transcription from papers
- Limited access and searchability
- High error rates

**Early Databases** (1980s):
- **1982**: EMBL Data Library founded (now ENA)
- **1988**: GenBank established at NCBI
- **1986**: DDBJ founded in Japan
- Thousands of sequences

**Sanger Era** (1990s-2000s):
- Human Genome Project
- Model organism sequencing
- Millions of sequences
- Gigabases of data

**Next-Gen Era** (2005-2015):
- Illumina, 454, SOLiD sequencing
- Massive throughput increase
- Metagenomics, transcriptomics
- Billions of short reads

**Third-Gen Era** (2015-present):
- PacBio and Oxford Nanopore
- Long reads (10kb-100kb+)
- Real-time sequencing
- Portable devices (MinION)

**Future** (2025+):
- Single-cell genomics
- Spatial transcriptomics
- Real-time clinical sequencing
- Planetary-scale biodiversity catalogs

# Set email (required by NCBI)

### SBOL (Synthetic Biology Open Language)

**Purpose**: Specifically designed for synthetic biology

**Version 3.0 Structure** (RDF/XML):
```xml
<?xml version="1.0" encoding="UTF-8"?>
<rdf:RDF xmlns:rdf="http://www.w3.org/1999/02/22-rdf-syntax-ns#"
         xmlns:sbol="http://sbols.org/v3#">
    <sbol:Component rdf:about="https://synbiohub.org/BBa_E0040">
        <sbol:displayId>BBa_E0040</sbol:displayId>
        <sbol:name>GFP</sbol:name>
        <sbol:description>Green Fluorescent Protein</sbol:description>
        <sbol:type rdf:resource="http://www.biopax.org/release/biopax-level3.owl#Dna"/>
        <sbol:sequence rdf:resource="https://synbiohub.org/BBa_E0040_sequence"/>
        <sbol:role rdf:resource="http://identifiers.org/so/SO:0000316"/>
    </sbol:Component>

    <sbol:Sequence rdf:about="https://synbiohub.org/BBa_E0040_sequence">
        <sbol:elements>ATGCGTAAAGGAGAAGAACTTTTCACTGGAGTT...</sbol:elements>
        <sbol:encoding rdf:resource="http://www.chem.qmul.ac.uk/iupac/misc/naseq.html"/>
    </sbol:Sequence>
</rdf:RDF>
```

**Advantages**:
- Design intent captured
- Machine-readable
- Tool interoperability
- Hierarchical designs
- Semantic web compatible

**SBOL Visual**:
Graphical notation standard:
- Arrow: CDS (coding sequence)
- Right angle: Promoter
- T-shape: Terminator
- Circle: Origin of replication
- Hexagon: Operator
- Diamond: Restriction site

## 4.6 Sequence Submission Protocols

### Why Submit Sequences?

**Scientific Requirements**:
- Journal publication requirements
- Grant agency mandates
- Data sharing principle
- Reproducibility

**Community Benefit**:
- Others can use your sequences
- Accelerate research
- Enable meta-analyses
- Build on your work

**Attribution**:
- Permanent record
- Citable accession numbers
- Recognition for contribution

### NCBI Submission Process

**Preparation**:
1. Sequence files (FASTA)
2. Annotations (feature tables)
3. Metadata (organism, source, methods)
4. Publication info (if available)

**BankIt** (web-based, simple):
- Upload sequences
- Fill web forms
- Automated checking
- Suitable for 1-100 sequences

**Sequin** (desktop, detailed):
- Downloadable software
- Complex annotations
- Multiple sequences
- More control

**tbl2asn** (command-line, programmatic):
- Batch submissions
- Automated pipelines
- Large-scale projects
- Requires feature tables

**Example Submission**:
1. Create FASTA file with sequences
2. Create feature table (.tbl):
```
>Feature seq1
1   753   gene
        gene    gfp
1   753   CDS
        product Green Fluorescent Protein
        protein_id  gnl|YourLab|GFP1
```
3. Run tbl2asn:
```bash
tbl2asn -t template.sbt -i sequences.fsa -f features.tbl -o submission.sqn
```
4. Upload .sqn file to NCBI
5. Receive accession numbers

**Processing Time**:
- Simple submissions: 1-3 days
- Complex submissions: 1-2 weeks
- Genome submissions: weeks to months

### ENA Submission Process

**Webin** (web interface):
- Create study
- Add samples
- Upload sequences
- Submit

**Webin-CLI** (command-line):
```bash
# Download Webin-CLI
wget https://github.com/enasequence/webin-cli/releases/download/latest/webin-cli.jar

# Prepare manifest file
cat > manifest.txt <<EOF
STUDY   PRJEB12345
SAMPLE  ERS123456
NAME    My_Synthetic_Construct
ASSEMBLY_TYPE   isolate
COVERAGE    100

PLATFORM    Illumina
FASTA   sequence.fasta
EOF

# Submit
java -jar webin-cli.jar -context=genome -manifest=manifest.txt -submit
```

**REST API** (programmatic):
- POST metadata in XML
- Upload data files
- Automate large submissions

### DDBJ Submission Process

**NSSS** (for sequences):

## 4.7 Part Characterization Standards

### Why Characterization Matters

**Problems with Poor Characterization**:
- Parts don't work as expected
- Cannot compare between labs
- Wasted effort and resources
- Low reproducibility
- Limits reuse

**Benefits of Good Characterization**:
- Predictable behavior
- Rational design possible
- Easy to compare alternatives
- Builds trust in registry
- Accelerates research

### Types of Characterization

**Basic Characterization**:
- Sequence verification
- Expression confirmation
- Qualitative function

**Quantitative Characterization**:
- Promoter strength (expression level)
- RBS translation rate
- Protein activity (enzyme kinetics)
- Response curves (inducers, repressors)

**Contextual Characterization**:
- Different host strains
- Various growth conditions
- Temperature dependence
- Media effects
- Copy number variations

**Dynamic Characterization**:
- Time-course behavior
- Response kinetics
- Adaptation effects
- Long-term stability

### Measurement Standards

**iGEM Measurement Committee**:
Established standards for part characterization

**Interlab Study** (annual):
- Multiple labs measure same parts
- Standardized protocols
- Calibration to absolute units
- Assess reproducibility

**Key Advances**:

**Fluorescence Calibration**:
- Fluorescein standard curves
- Silica bead microsphere standards
- Convert arbitrary units to molecules

**Cell Counting**:
- Microsphere standards
- Flow cytometry counting
- OD600 to cell number conversion

**Protocols**:
```
1. Calibration:
   - Measure fluorescein standard curve
   - Measure microsphere count vs. OD600

2. Transform strains:
   - Positive control
   - Negative control
   - Test devices

3. Measure:
   - 0, 2, 4, 6 hour timepoints
   - Plate reader: fluorescence + OD600
   - Flow cytometry (if available)

4. Calculate:
   - Fluorescence / cell
   - Convert to absolute units
   - Statistical analysis
```

**Results**:
- 2018 Interlab: >250 teams, successful calibration
- Improved reproducibility across labs
- Foundation for engineering standards

### Characterization Metadata

**Minimum Information**:
- Host organism (strain)
- Growth medium
- Temperature
- Measurement method
- Instrument details
- Data processing

**Comprehensive Information**:
- Complete protocol
- Raw data files

## 4.9 Integration with Synthetic Biology Registries

### Cross-References

**Best Practice**: Link between databases

**Example - BBa_E0040 (GFP)**:
- **iGEM Registry**: BBa_E0040
- **GenBank**: U17997 (original GFP from jellyfish)
- **UniProt**: P42212 (GFP protein)
- **PDB**: 1GFL (GFP structure)
- **InterPro**: IPR011584 (GFP family)

**Implementation**:
```xml
<part id="BBa_E0040">
  <name>GFP</name>
  <cross_references>
    <xref db="GenBank" id="U17997"/>
    <xref db="UniProt" id="P42212"/>
    <xref db="PDB" id="1GFL"/>
    <xref db="NCBI_Taxonomy" id="6100"/>
  </cross_references>
</part>
```

### Data Flow

**Typical Workflow**:
1. Design part based on sequence from GenBank/UniProt
2. Synthesize or clone
3. Characterize in lab
4. Submit to iGEM Registry (BioBrick)
5. Submit sequence to GenBank (accession)
6. Publish paper (PubMed)
7. Links maintained between all resources

### SBOL as Universal Format

**SBOL Enables**:
- Import from GenBank
- Export to registries
- Tool interoperability
- Design exchange

**Example Conversion**:
```python
from sbol import *

# Import from GenBank
doc = Document()
doc.read('genbank_file.gb')

# Add synthetic biology annotations
component = doc.components[0]
component.roles = SO_PROMOTER

# Export as SBOL
doc.write('part.xml')

# Upload to SynBioHub
# (API calls to registry)
```
