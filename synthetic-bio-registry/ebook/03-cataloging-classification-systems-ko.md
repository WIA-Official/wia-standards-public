# 제3장: 카탈로그 및 분류 시스템

## WIA-SYNTHETIC-BIO-REGISTRY 표준
**버전:** 1.0  
**상태:** 공식 WIA 표준  
**철학:** 弘益人間 (홍익인간)

---

## 3.1 체계적 분류의 중요성

생물학적 부품 레지스트리가 수백 개에서 수만 개의 항목으로 성장함에 따라 효과적인 카탈로그 및 분류 시스템이 필수적이 되었습니다. 체계적인 조직이 없으면 사용자는 관련 부품을 효율적으로 발견할 수 없으며 레지스트리의 가치는 크기 증가에도 불구하고 감소합니다.

생물학적 부품은 제조 부품과 달리 고유한 분류 과제를 제시합니다. 단일 DNA 서열이 맥락에 따라 여러 기능을 수행할 수 있으며 부품은 개별 모티프에서 완전한 경로까지 여러 수준의 조직에 존재하며 중첩 분류 체계가 필요합니다.

## 3.2 계층적 분류 프레임워크

대부분의 생물학적 부품 레지스트리는 여러 수준의 세분성을 가진 계층적 분류 시스템을 사용합니다.

### 주요 기능 카테고리

최상위 수준은 광범위한 기능 클래스를 구분합니다:

#### 조절 부품
유전자 발현 타이밍, 수준 및 조건 제어
**예시:** 프로모터, 오퍼레이터, 인핸서, 절연체, 터미네이터

#### 코딩 부품  
기능 제품 인코딩(단백질 또는 RNA 분자)
**예시:** 유전자, 오픈 리딩 프레임, RNA 코딩 서열

#### 인식 부위
결합 또는 인식 특이성 제공
**예시:** 제한 부위, 재조합 효소 부위, 프라이머 결합 부위, 오퍼레이터 서열

#### 조립 구성 요소
구성 및 조작 촉진
**예시:** 복제 기점, 선택 마커, 다중 클로닝 부위

#### 구조 부품
물리적 조직 또는 스캐폴딩 제공
**예시:** 스페이서, 절연체, 링커 서열, 스캐폴드 부착 영역

### 2차 하위 유형 분류

각 주요 카테고리는 특정 하위 유형으로 세분화됩니다. 프로모터의 경우 구성적, 유도성, 억제성, 조절 논리 프로모터로 나뉩니다. 코딩 부품은 리포터 단백질, 조절 단백질, 대사 효소, 구조 단백질로 분류됩니다.

## 3.3 온톨로지 기반 분류

현대 레지스트리는 점점 더 공식 온톨로지를 채택합니다. 온톨로지는 용어 간에 정의된 관계를 가진 구조화된 어휘를 제공합니다.

### 서열 온톨로지(SO)

서열 온톨로지는 게놈 특징에 대한 표준화된 어휘를 제공합니다:

**핵심 용어:**
```
SO:0000167 - promoter (프로모터)
├── SO:0001203 - constitutive promoter (구성적 프로모터)
├── SO:0001204 - inducible promoter (유도성 프로모터)
└── SO:0001205 - repressible promoter (억제성 프로모터)

SO:0000316 - CDS (코딩 서열)
├── SO:0000101 - enzymatic activity (효소 활성)
├── SO:0000357 - transcription factor (전사 인자)
└── SO:0000103 - reporter gene (리포터 유전자)
```

**장점:**
- 데이터베이스 전반에 걸친 표준화된 용어
- 기계 판독 가능 관계
- 계산 추론 기능
- 게놈 데이터베이스와의 통합

### 시스템 생물학 온톨로지(SBO)

SBO는 정량적 매개변수 및 수학적 표현을 설명합니다. 부품 특성화에 사용되는 용어에는 프로모터 강도(SBO:0000064), 최대 전사 속도(SBO:0000186), 동역학 상수(SBO:0000009) 등이 포함됩니다.

### 유전자 온톨로지(GO)

생물학적 기능, 세포 구성 요소 및 프로세스를 설명합니다. 분자 기능, 생물학적 프로세스, 세포 구성 요소의 세 가지 주요 영역으로 나뉩니다.

## 3.4 메타데이터 표준

포괄적인 메타데이터는 부품 유용성 및 발견 가능성을 향상시킵니다.

### 최소 정보 표준

WIA-SYNTHETIC-BIO-REGISTRY는 최소 메타데이터 요구 사항을 정의합니다:

#### 필수 메타데이터(필수)
```json
{
  "part_id": "BBa_J23100",
  "name": "구성적 프로모터 (Anderson)",
  "type": "regulatory",
  "subtype": "promoter",
  "sequence": "ttgacggctagctcagtcctaggtacagtgctagc",
  "length": 35,
  "author": "John Anderson",
  "creation_date": "2006-07-15",
  "license": "Open MTA",
  "safety_level": 1
}
```

#### 권장 메타데이터
유기체, 테스트된 샤시, 조립 표준, 응용 프로그램, 설명, 설계 노트 및 참고 문헌을 포함합니다.

#### 확장 메타데이터
서열 특징, 특성화 요약, 품질 플래그 및 상세한 실험 데이터를 포함합니다.

### 서열 주석

표준 형식을 사용한 상세한 서열 특징. GenBank 형식은 보편적으로 인식되며 모든 서열 뷰어에서 지원됩니다. SBOL(합성생물학 개방 언어)은 합성 생물학 설계를 표현하기 위한 주요 표준을 제공합니다.

## 3.5 태그 및 패싯 분류

계층적 시스템 외에도 태그 및 패싯은 유연한 다차원 분류를 가능하게 합니다.

### 사용자 생성 태그

커뮤니티 구성원은 설명 태그를 추가합니다:

**기능 태그:** #biosensor #reporter #inducible #constitutive #strong #weak #tunable

**응용 프로그램 태그:** #metabolic-engineering #therapeutics #biofuels #biosensing #optogenetics

**품질 태그:** #well-characterized #works-reliably #needs-optimization #context-dependent

**유기체 태그:** #ecoli #yeast #mammalian #plant #bacillus

**프로젝트 태그:** #iGEM2024 #Anderson-collection #community-favorite

### 패싯 검색

여러 독립적인 속성이 정밀한 필터링을 가능하게 합니다. 사용자는 유형, 강도, 조절, 유기체, 품질 계층, 조립 및 연도별로 동시에 필터링할 수 있습니다.

## 3.6 교차 레지스트리 분류 표준

여러 레지스트리가 등장함에 따라 상호 운용성에는 공유 분류 체계가 필요합니다.

### 범용 부품 식별자(UPI)

전 세계적으로 고유한 부품 식별자에 대한 제안된 표준:

**형식:** `[레지스트리]:[컬렉션]:[부품]:[버전]`

**예시:**
- `iGEM:Anderson:J23100:1` - iGEM 레지스트리, Anderson 컬렉션, 부품 J23100, 버전 1
- `AddGene:CRISPR:12345:2` - AddGene, CRISPR 컬렉션, 플라스미드 12345, 버전 2
- `WIA:Standard:PROM001:1` - WIA 레지스트리, 표준 컬렉션, 프로모터 001, 버전 1

### SBOL Visual 표준

생물학적 부품에 대한 그래픽 기호는 보편적인 시각적 분류를 가능하게 합니다:

**표준 기호:**
```
→   프로모터 (오른쪽을 가리키는 화살표)
⭘   RBS (원)
▭   CDS (직사각형 상자)
⊤   터미네이터 (T자형)
◇   오퍼레이터 (다이아몬드)
⬟   복제 기점 (+ 표시가 있는 원)
◀   프라이머 결합 부위 (왼쪽 화살표)
```

**색상 규칙:**
- 녹색: 기능적이고 특성화됨
- 노란색: 부분적으로 특성화됨
- 빨간색: 문제가 있거나 사용 중단됨
- 회색: 이론적이거나 특성화되지 않음

### SBOL 데이터 모델

합성생물학 개방 언어는 설계 교환을 위한 데이터 모델을 제공합니다. 핵심 클래스에는 ComponentDefinition(추상 부품 설명), Component(설계의 특정 인스턴스), Module(구성 요소의 기능적 그룹화) 및 Interaction(구성 요소 간 관계)이 포함됩니다.

## 3.6 부품 유형별 상세 분류

### 프로모터 부품 분류 (총 3,200개+)

**구성적 프로모터 (Constitutive Promoters):**
- Anderson 프로모터 패밀리: 18개의 특성화된 프로모터, 강도 0.01-1.0 상대 단위
- T7 프로모터: T7 RNA 폴리머라제의 강력한 인식
- SP6 프로모터: 박테리오파지 SP6 RNA 폴리머라제용
- T3 프로모터: 박테리오파지 T3 RNA 폴리머라제용
- 강도 범위: 약한 프로모터(< 0.1), 중간 프로모터(0.1-0.5), 강한 프로모터(> 0.5)

**유도성 프로모터 (Inducible Promoters):**
- araBAD 프로모터: L-아라비노스에 의해 유도, 최대 1,000배 활성화
- Tet 프로모터: 테트라사이클린 계열 항생제에 반응
- Lac 프로모터: IPTG에 의해 유도
- Trp 프로모터: 트립토판 기아에 반응
- rhaPBAD 프로모터: L-라므노스에 의해 유도

**억제성 프로모터 (Repressible Promoters):**
- Trp 오페론: 트립토판 존재 시 억제
- His 오페론: 히스티딘 존재 시 억제
- Phe 오페론: 페닐알라닌 존재 시 억제

**조절 프로모터 (Regulated Promoters):**
- 합성 로직 게이트: AND, OR, NOT 게이트 기능
- 탠덤 프로모터: 여러 제어 요소를 포함
- 상황 응답형: 온도, pH, 산화 스트레스에 반응

### 리보솜 결합 부위 분류 (총 1,800개+)

**천연 RBS (Native RBS):**
- E. coli 샤시에서 자연 발생하는 리보솜 결합 서열
- Shine-Dalgarno 시퀀스: AGGAGG 또는 유사 모티프
- 번역 효율성: 0.1-1.0 상대 단위

**합성 RBS (Synthetic RBS):**
- BCD RBS 계산기: 계산적으로 설계된 강도 조절 가능
- Stanford RBS 라이브러리: 다양한 강도 선택 가능
- Uppsala RBS 라이브러리: 스웨덴의 iGEM 팀에서 특성화
- 설계 범위: 매우 약함(< 1%), 약함(1-10%), 중간(10-50%), 강함(50-90%), 매우 강함(> 90%)

**조절 RBS:**
- 리보스위치와 연결: 특정 신호에 반응하여 번역 개시
- 온도 민감: 온도에 따라 2차 구조 변경
- 동적 조절: 조절 단백질에 의해 번역 조절

### 코딩 서열 분류 (총 7,500개+)

**리포터 단백질 (Reporter Proteins):**
- GFP (Green Fluorescent Protein) 및 변이체: sfGFP, mGFP, 약 800개 변이
- RFP (Red Fluorescent Protein) 계열: mCherry, tdTomato, 약 300개 변이
- YFP, CFP: 스펙트럼 다양화를 위해
- 효소 리포터: β-갈락토시다제, 루시페라제, β-글루쿠로니다제
- 발광 단백질: VVD, miniSOG, 광활성 단백질

**조절 단백질 (Regulatory Proteins):**
- 전사 인자: LacI, TetR, AraC, 수십 개의 다른 조절자
- 억제자 단백질: CI (λ 파지 억제자)
- 활성화제: CAP (캐탈라이트 작용 단백질)
- 항체 조각: scFv, Fab 단편

**효소 (Enzymes):**
- 대사 효소: 글루코오스 산화효소, 알코올 산화효소
- 생합성 효소: 카로테노이드 생합성 효소, 리코펜 합성효소
- 분해 효소: 셀룰라제, 단백질분해효소
- DNA 수정 효소: 메틸라제, 제한효소

**구조 단백질 (Structural Proteins):**
- 스캐폴드 단백질: 효소 복합체 구성
- 세포벽 단백질: 세포 표면 디스플레이
- 막 단백질: 채널, 운반체

## 3.7 기계 학습 향상 분류

현대 레지스트리는 기계 학습을 사용하여 분류 및 발견을 개선합니다.

### 자동화된 특징 추출

ML 모델은 서열 특징 및 기능 모티프를 식별합니다. 프로모터 예측, RBS 강도 예측, 터미네이터 효율성 평가 등에 사용됩니다.

### 기능 주석 예측

특성화된 부품에 대해 훈련된 ML 모델은 새로운 서열의 기능을 예측합니다. 입력 특징에는 DNA 서열(k-mer 표현), 서열 모티프, 2차 구조, 코돈 사용, GC 함량 등이 포함됩니다.

### 유사성 기반 클러스터링

비지도 학습은 관련 부품을 식별합니다. 클러스터링 방법에는 서열 임베딩에 대한 K-평균 클러스터링, 기능별 계층적 클러스터링, 이상치 감지를 위한 DBSCAN 등이 포함됩니다.

### 추천 시스템

협업 필터링 및 콘텐츠 기반 추천:
- 사용자 기반: "부품 A를 사용한 사용자는 부품 B, C, D도 사용했습니다"
- 콘텐츠 기반: "A와 유사한 부품에는 서열 및 기능을 기반으로 B, C, D가 포함됩니다"
- 하이브리드: 사용자 행동 및 부품 특성 결합

## 3.8 BioBricks RFC 표준과 분류

BioBricks 표준 제안 (RFC)은 부품 분류의 근간을 이루고 있습니다.

### RFC 표준 계층 구조

**RFC 10: BioBricks 조립 표준**
- 부품 경계: EcoRI(외측)/XbaI(내측) - SpeI(내측)/PstI(외측)
- 방향성 조립 가능
- 모든 부품이 호환 가능
- 1,000개 이상의 부품이 이 표준을 따름

**RFC 21: BioBricks 부품 구성**
- 순서: 프로모터-RBS-CDS-터미네이터
- 조절 요소는 부품과 분리 가능
- 각 계층에서 예측 가능성 유지
- 2,000개 이상의 합성 부품

**RFC 23: BioBricks 명명과 버전 관리**
- BBa_[X][#####][버전] 형식
- 같은 서열의 다양한 버전 추적
- 여러 부품의 진화 과정 기록
- 버전별 성능 비교 가능

**RFC 25: 표준화된 프로모터 강도 단위**
- 상대적 강도 단위 (Relative Strength Units, RSU)
- Anderson 기준 프로모터 = 1.0 RSU
- 0.01-1.0 범위의 다양한 강도 제공
- 미터법 같은 보편적 단위 시스템

## 3.8 품질 기반 분류

부품 품질 자체는 중요한 분류 차원입니다.

### 품질 차원

**서열 무결성:**
- 시퀀싱으로 확인된 DNA 서열
- 예상치 못한 돌연변이 없음
- 올바른 길이
- 오염 서열 없음

**기능 검증:**
- 실험적으로 확인된 행동
- 제공된 정량적 측정
- 여러 조건 테스트
- 재현성 입증

**문서 완전성:**
- 상세한 설명
- 설계 근거
- 구성 방법
- 제공된 프로토콜
- 인용된 참고 문헌

**커뮤니티 검증:**
- 여러 사용자가 기능 확인
- 독립적인 실험실 확인
- 보고된 일관된 결과
- 문서화된 문제 해결

**특성화 깊이:**
- 기본 기능 확인(계층 1)
- 정량적 데이터(계층 2)
- 다중 조건 테스트(계층 3)
- 최적화 연구(계층 4)

### 품질 메트릭

각 차원에 대한 정량적 점수:

**서열 품질 점수(0-100):**
```
SQS = (확인_상태 * 40) +
      (특징_주석 * 30) +
      (조립_호환성 * 30)

여기서 각 구성 요소 ∈ [0, 1]
```

**기능 품질 점수(0-100):**
```
FQS = (실험_검증 * 50) +
      (정량적_데이터 * 30) +
      (재현성 * 20)
```

**문서 품질 점수(0-100):**
```
DQS = (설명_완전성 * 30) +
      (프로토콜_상세 * 25) +
      (설계_근거 * 25) +
      (참고문헌 * 20)
```

**전체 품질 점수(0-100):**
```
OQS = (SQS * 0.3) + (FQS * 0.4) + (DQS * 0.3)
```

### 품질 시각화

레지스트리 인터페이스는 품질 정보를 표시합니다:

**배지 시스템:**
```
🏅 인증됨 (OQS ≥ 90)
⭐ 고품질 (OQS ≥ 75)
✓  검증됨 (OQS ≥ 60)
📝 문서화됨 (OQS ≥ 40)
❓ 예비 (OQS < 40)
```

**품질 대시보드:**
```
부품: BBa_J23100
━━━━━━━━━━━━━━━━━━━━━━━━━
서열 확인됨        ████████████ 100%
기능적으로 유효함   ██████████░░  85%
잘 문서화됨        ███████████░  92%
커뮤니티 지원      █████████░░░  75%
━━━━━━━━━━━━━━━━━━━━━━━━━
전체 품질          ███████████░  88%

```

## 3.9 동적 분류

이해가 진화함에 따라 분류 시스템은 적응해야 합니다.

### 버전 관리 및 업데이트

부품 및 분류는 시간이 지남에 따라 변경될 수 있습니다. 버전 제어를 통해 변경 사항을 추적하고 초기 제출, 서열 재확인, 추가 특성화, 새로운 실험 데이터 등의 기록을 유지합니다.

### 사용 중단 및 은퇴

부품이 쓸모없게 될 수 있습니다. 사용 중단 이유에는 더 나은 대안 사용 가능, 서열 오류 발견, 보고된 일관성 없는 행동, 식별된 안전 문제, 구식 조립 표준 등이 포함됩니다.

### 재분류

새로운 정보는 재분류를 요구할 수 있습니다. 트리거에는 새로운 실험 데이터, 기능에 대한 이해 변경, 추가 역할 발견, 업데이트된 유기체 호환성, 개정된 안전 분류 등이 포함됩니다.
