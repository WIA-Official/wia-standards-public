Chapter 3: Blockchain for Traceability

Why Blockchain for Food Safety?

Traditional food traceability systems rely on centralized databases and paper records. When contamination occurs, tracing the source can take 7-14 days—during which contaminated products spread across the country.

Blockchain Solution: Distributed ledger technology provides immutable, transparent records of every transaction in the food supply chain. Traceability time: < 2.2 seconds.

How Blockchain Works

A blockchain is a distributed ledger of transactions, stored across multiple computers (nodes). Each "block" contains:

  • Data: Product information, batch ID, temperature logs, CCP records
  • Hash: Unique cryptographic fingerprint of the block (SHA-256)
  • Previous Hash: Links this block to the previous block, forming a chain
  • Timestamp: When the block was created

Key Properties:

  • Immutable: Once written, data cannot be altered without breaking the chain
  • Transparent: All participants can view the ledger (with appropriate permissions)
  • Decentralized: No single point of failure or control
  • Consensus: Changes require agreement from network participants

Real-World Case Study: Walmart + IBM Food Trust

In 2018, Walmart partnered with IBM to pilot blockchain traceability for mangoes:

  • Traditional System: 6 days, 18 hours to trace mangoes from farm to store
  • Blockchain System: 2.2 seconds to trace complete supply chain

Impact: Walmart now requires all leafy green suppliers to use blockchain traceability. The system tracks 25+ products from 100+ suppliers.

Smart Contracts for Food Safety

Smart contracts are self-executing programs on the blockchain that automatically enforce rules. Example use cases:

Example: Temperature Violation Smart Contract

contract TemperatureMonitoring {
    function logTemperature(
        bytes32 batchId,
        int16 temp,
        uint256 timestamp
    ) external {
        // Critical limit: 4°C
        if (temp > 40) {  // 40 = 4.0°C (stored as int * 10)
            // Automatic quarantine
            products[batchId].status = Status.QUARANTINED;
            emit TemperatureViolation(batchId, temp, timestamp);

            // Trigger alert to supervisor
            alertSupervisor(batchId);
        }

        // Store temperature log on blockchain
        tempLogs[batchId].push(TemperatureLog({
            temp: temp,
            timestamp: timestamp
        }));
    }
}

One-Up/One-Back Traceability

FDA FSMA requires food companies to maintain "one-up, one-back" traceability:

  • One-Up: Where did this batch come from? (Supplier, batch ID, date received)
  • One-Back: Where did this batch go? (Customer, shipment ID, date sent)
WIA-FOOD_SAFETY Enhancement: Provides "one-step" traceability—full visibility from farm to consumer in a single query, not just one step up/down the chain.

QR Code Integration

Every food product registered in WIA-FOOD_SAFETY receives a unique QR code that links to its blockchain record. Consumers can scan the QR code to see:

  • Farm origin and GPS coordinates
  • Harvest date and expiration date
  • Organic/GAP certifications
  • Temperature history (cold chain compliance)
  • Lab test results (pathogen screening)
  • Safety status (PASS / RECALLED)
