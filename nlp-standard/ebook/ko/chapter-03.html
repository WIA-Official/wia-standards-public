단어 임베딩과 표현

3.1 단어에서 벡터로

머신러닝 모델은 텍스트를 직접 처리할 수 없습니다—수치적 표현이 필요합니다. 단어 임베딩은 이산적인 단어를 기하학적 속성을 통해 의미론적 관계가 보존되는 연속적인 벡터 공간으로 변환합니다. 이 변환은 언어에 대한 수학적 연산을 가능하게 하고 현대 NLP 시스템을 강력하게 합니다.

단어 임베딩의 기본 통찰은 분포 가설입니다: 유사한 맥락에 나타나는 단어는 유사한 의미를 가지는 경향이 있습니다. 대규모 텍스트 말뭉치에서 단어 동시 출현 패턴을 분석함으로써, 임베딩 알고리즘은 의미론적 및 구문론적 관계를 포착하는 밀집 벡터 표현을 학습합니다.

단어 임베딩이 중요한 이유

  • 의미적 유사성: 유사한 단어는 유사한 벡터를 가짐
  • 차원성 축소: 희소한 원-핫 인코딩 대비 밀집 벡터
  • 전이 학습: 사전 학습된 임베딩이 언어학적 지식을 인코딩
  • 산술 연산: 벡터 수학이 의미론적 관계를 포착
  • 일반화: 모델이 학습 데이터를 넘어 일반화되는 패턴 학습

3.2 전통적 접근법: 원-핫 및 TF-IDF

단어 임베딩이 표준이 되기 전에, NLP는 제한된 의미론적 정보로 단어를 고차원 벡터로 인코딩하는 희소 표현에 의존했습니다.

원-핫 인코딩

원-핫 인코딩은 각 단어를 어휘 크기와 같은 길이의 벡터로 표현하며, 단일 1과 나머지 모든 값이 0입니다. 이 간단한 접근법은 심각한 제한이 있습니다: 대규모 차원성, 의미론적 관계 없음, 비효율적인 계산. 

import numpy as np

# 어휘
vocab = ['자연', '언어', '처리', '기계', '학습']

# 원-핫 인코딩 함수
def one_hot_encode(word, vocab):
    vector = np.zeros(len(vocab))
    if word in vocab:
        vector[vocab.index(word)] = 1
    return vector

# 예제
word = '언어'
one_hot = one_hot_encode(word, vocab)
print(f"{word}: {one_hot}")
# 출력: [0. 1. 0. 0. 0.]

# 문제: 의미론적 관계 없음
vec_language = one_hot_encode('언어', vocab)
vec_processing = one_hot_encode('처리', vocab)
similarity = np.dot(vec_language, vec_processing)
print(f"유사도: {similarity}")  # 항상 0!

TF-IDF (용어 빈도-역문서 빈도)

TF-IDF는 중요도에 따라 단어에 가중치를 부여합니다: 문서에서 자주 나타나지만 말뭉치 전체에서는 희귀한 단어가 더 높은 가중치를 받습니다. 원-핫보다 정보가 풍부하지만, TF-IDF는 여전히 의미론적 관계를 포착하지 않는 희소 벡터를 생성합니다. 

from sklearn.feature_extraction.text import TfidfVectorizer

documents = [
    "자연어 처리는 기본적입니다",
    "기계 학습이 NLP 시스템을 가능하게 합니다",
    "딥러닝이 자연어 이해를 혁신했습니다"
]

vectorizer = TfidfVectorizer()
tfidf_matrix = vectorizer.fit_transform(documents)

print("어휘:", vectorizer.get_feature_names_out())
print("TF-IDF 행렬 형태:", tfidf_matrix.shape)
print("\n문서 0 벡터:\n", tfidf_matrix[0].toarray())

3.3 Word2Vec: 맥락으로부터 학습

2013년 Mikolov 등이 도입한 Word2Vec은 대규모 텍스트 말뭉치로부터 밀집 단어 임베딩을 학습하여 NLP에 혁명을 일으켰습니다. Word2Vec은 단어 맥락 예측 작업에 대해 학습된 얕은 신경망을 사용합니다.

Word2Vec 아키텍처

CBOW (연속 단어 가방)

CBOW는 주변 맥락 단어로부터 대상 단어를 예측합니다. 창 내의 맥락 단어가 주어지면, 모델은 중심 단어를 예측하도록 학습합니다. CBOW는 더 빠르며 빈번한 단어에 잘 작동합니다.

Skip-gram

Skip-gram은 반대로 작동합니다: 대상 단어가 주어지면, 주변 맥락 단어를 예측합니다. Skip-gram은 희귀 단어와 더 작은 데이터셋에서 더 잘 작동하며, 드물게 나타나는 용어에 대해 더 높은 품질의 임베딩을 생성합니다. 

from gensim.models import Word2Vec
import gensim.downloader as api

# 사전 학습된 모델 로드 (또는 자체 학습)
# 데모를 위해 작은 말뭉치 사용
sentences = [
    ['자연', '언어', '처리'],
    ['기계', '학습', '알고리즘'],
    ['딥', '학습', '신경', '네트워크'],
    ['자연', '언어', '이해'],
    ['텍스트', '처리', '기법']
]

# Word2Vec 모델 학습
model = Word2Vec(sentences,
                 vector_size=100,  # 임베딩 차원
                 window=5,         # 맥락 창
                 min_count=1,      # 최소 단어 빈도
                 workers=4,        # 병렬 스레드
                 sg=1)             # Skip-gram (0은 CBOW)

# 단어 벡터 가져오기
vector = model.wv['언어']
print(f"벡터 형태: {vector.shape}")
print(f"벡터 (처음 10개 차원): {vector[:10]}")

# 유사 단어 찾기
similar = model.wv.most_similar('언어', topn=3)
print(f"\n'언어'와 유사한 단어: {similar}")

# 벡터 산술
result = model.wv.most_similar(positive=['처리', '기계'],
                               negative=['자연'],
                               topn=1)
print(f"\n처리 + 기계 - 자연 = {result}")

3.4 GloVe: 전역 벡터

GloVe(Global Vectors for Word Representation)는 행렬 분해 방법의 이점과 지역 맥락 창 접근법을 결합합니다. Word2Vec이 지역 맥락을 사용하는 것과 달리, GloVe는 전체 말뭉치로부터 전역 단어 동시 출현 통계를 활용합니다.

GloVe 작동 방식

  1. 말뭉치로부터 단어 동시 출현 행렬 구성
  2. 단어 벡터를 학습하기 위해 행렬 분해
  3. 동시 출현 확률을 보존하도록 최적화
# 사전 학습된 GloVe 임베딩 사용
import numpy as np

def load_glove_embeddings(glove_file):
    """파일에서 GloVe 임베딩 로드"""
    embeddings = {}
    with open(glove_file, 'r', encoding='utf-8') as f:
        for line in f:
            values = line.split()
            word = values[0]
            vector = np.asarray(values[1:], dtype='float32')
            embeddings[word] = vector
    return embeddings

# 사용 예 (glove.6B.100d.txt가 사용 가능하다고 가정)
# embeddings = load_glove_embeddings('glove.6B.100d.txt')

# 유사도 계산
def cosine_similarity(vec1, vec2):
    return np.dot(vec1, vec2) / (np.linalg.norm(vec1) * np.linalg.norm(vec2))

# 가상 벡터를 사용한 예
vec_king = np.random.randn(100)
vec_queen = np.random.randn(100)
similarity = cosine_similarity(vec_king, vec_queen)
print(f"유사도: {similarity}")

3.5 FastText: 서브워드 임베딩

FastText는 단어를 문자 n-그램의 가방으로 표현함으로써 Word2Vec을 확장합니다. 이 접근법은 어휘 외 단어를 처리하고, 형태론적 정보를 포착하며, 형태론적으로 풍부한 언어에 잘 작동합니다. 

from gensim.models import FastText

# FastText 학습
sentences = [
    ['자연', '언어', '처리'],
    ['자연화', '자연적으로', '자연'],
    ['처리', '처리중', '처리됨', '처리들']
]

model = FastText(sentences,
                 vector_size=100,
                 window=5,
                 min_count=1,
                 min_n=2,      # 최소 n-그램 길이
                 max_n=5,      # 최대 n-그램 길이
                 workers=4)

# OOV 단어 처리
# '자연화하는'이 학습에 없었더라도 FastText는 처리 가능
vector = model.wv['자연화하는']
print(f"OOV 단어 벡터 형태: {vector.shape}")

# 서브워드 정보
print(f"\n'처리'에 대한 서브워드:")
print(model.wv.key_to_index.keys())

3.6 문맥적 임베딩: ELMo 그리고 그 이후

전통적인 임베딩은 맥락과 관계없이 각 단어에 단일 벡터를 할당합니다. "은행"은 "강 은행"과 "저축 은행"에서 동일한 표현을 갖습니다. 문맥적 임베딩은 주변 단어를 기반으로 다른 표현을 생성함으로써 이를 해결합니다.

ELMo (언어 모델로부터의 임베딩)

ELMo는 양방향 LSTM 언어 모델을 사용하여 맥락 의존적 표현을 생성합니다. 각 단어는 문장에서의 위치와 맥락에 따라 고유한 벡터를 받습니다.

모델 유형 문맥적 연도
Word2Vec 예측 기반 아니오 2013
GloVe 카운트 기반 아니오 2014
FastText 서브워드 기반 아니오 2016
ELMo LSTM 기반 예 2018
BERT 트랜스포머 기반 예 2018

3.7 임베딩 평가

임베딩 품질 평가는 본질적 및 외부적 방법을 모두 포함합니다. 본질적 평가는 임베딩을 직접 테스트하고, 외부적 평가는 후속 작업에서의 성능을 측정합니다.

본질적 평가 방법

  • 단어 유사도: 인간 유사도 판단과의 상관관계
  • 단어 유추: 유추 작업 해결 (왕 - 남자 + 여자 = 여왕)
  • 클러스터링: 의미론적으로 유사한 단어 그룹화
from scipy.stats import spearmanr

def evaluate_word_similarity(embeddings, word_pairs, human_scores):
    """
    단어 유사도 데이터셋을 사용하여 임베딩 평가
    word_pairs: (word1, word2) 튜플 리스트
    human_scores: 인간 유사도 등급 리스트
    """
    predicted_scores = []

    for word1, word2 in word_pairs:
        if word1 in embeddings and word2 in embeddings:
            vec1 = embeddings[word1]
            vec2 = embeddings[word2]
            similarity = cosine_similarity(vec1, vec2)
            predicted_scores.append(similarity)
        else:
            predicted_scores.append(0.0)

    # 스피어먼 상관관계 계산
    correlation, p_value = spearmanr(human_scores, predicted_scores)
    return correlation, p_value

# 사용 예
word_pairs = [('고양이', '개'), ('자동차', '자동차'), ('컴퓨터', '키보드')]
human_scores = [0.8, 0.95, 0.7]

3.9 실용적 고려사항

프로덕션 시스템에서 단어 임베딩을 작업할 때, 여러 실용적 고려사항이 성능, 효율성 및 신뢰성에 영향을 미칩니다.

임베딩 차원 선택

  • 50-100 차원: 작은 데이터셋, 제한된 컴퓨팅
  • 200-300 차원: 범용, 좋은 균형
  • 500-1000 차원: 대규모 데이터셋, 고성능 요구

어휘 외 단어 처리

  • 서브워드 정보를 위해 FastText 사용
  • 어휘 내 유사 단어의 임베딩 평균
  • 특수 [UNK] 토큰 사용
  • 도메인별 데이터에 대한 임베딩 미세 조정

3.10 고급 주제

현대 임베딩 연구는 제한사항을 해결하고 기능을 확장하면서 계속 발전하고 있습니다.

도메인별 임베딩

범용 임베딩은 도메인별 용어와 관계를 포착하지 못할 수 있습니다. 의료, 법률 또는 과학 말뭉치에 대한 임베딩 학습은 전문 애플리케이션의 성능을 향상시킵니다.

다국어 임베딩

다국어 임베딩은 다른 언어의 단어를 공유 벡터 공간으로 매핑하여 교차 언어 전이 및 기계 번역을 가능하게 합니다. 

# 예: 다국어 임베딩 사용
from transformers import AutoTokenizer, AutoModel

# 다국어 모델 로드
model_name = "bert-base-multilingual-cased"
tokenizer = AutoTokenizer.from_pretrained(model_name)
model = AutoModel.from_pretrained(model_name)

# 다양한 언어로 텍스트 처리
texts = {
    'en': 'Natural language processing',
    'ko': '자연어 처리',
    'es': 'Procesamiento del lenguaje natural'
}

for lang, text in texts.items():
    inputs = tokenizer(text, return_tensors='pt')
    outputs = model(**inputs)
    print(f"{lang}: {outputs.last_hidden_state.shape}")

요약

핵심 내용:

  • 단어 임베딩은 이산적인 단어를 의미론적 관계를 보존하는 연속 벡터로 변환
  • 전통적인 희소 표현(원-핫, TF-IDF)은 의미론적 정보 부족
  • Word2Vec은 맥락 예측을 사용하여 밀집 임베딩의 효율적인 학습 도입
  • GloVe는 임베딩 생성을 위해 전역 동시 출현 통계 활용
  • FastText는 서브워드 표현을 통해 어휘 외 단어 처리
  • 문맥적 임베딩(ELMo, BERT)은 맥락에 따라 다른 벡터 생성
  • 임베딩 평가는 본질적(유사도, 유추) 및 외부적(작업 성능) 방법 사용
  • WIA-AI-023 표준은 시스템 간 임베딩 상호운용성 보장
