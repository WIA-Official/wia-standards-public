텍스트 전처리와 토큰화

2.1 텍스트 전처리의 중요성

텍스트 전처리는 모든 NLP 파이프라인에서 중요한 첫 단계입니다. 인터넷, 문서 또는 사용자 입력에서 가져온 원시 텍스트에는 분석 전에 정제하고 표준화해야 하는 노이즈, 불일치 및 형식이 포함되어 있습니다. 효과적인 전처리는 데이터 복잡성을 줄이고 의미있는 패턴에 집중함으로써 후속 NLP 작업의 성능을 크게 향상시킬 수 있습니다.

전처리의 품질은 모델 성능에 직접적인 영향을 미칩니다. 잘못 전처리된 데이터는 노이즈를 도입하고 정확도를 떨어뜨리며 NLP 시스템에 편향을 생성할 수 있습니다. 반대로 잘 설계된 전처리 파이프라인은 노이즈에서 신호를 추출하고 변동을 정규화하며 효율적인 처리를 위해 텍스트를 준비합니다.

텍스트 전처리의 목표

  • 관련없는 정보와 노이즈 제거
  • 텍스트 형식 및 인코딩 표준화
  • 의미를 보존하면서 어휘 크기 축소
  • 단어 형태의 변동 정규화
  • 특수 문자, 구두점 및 공백 처리
  • 효율적인 토큰화를 위한 텍스트 준비

2.2 텍스트 정제 및 정규화

텍스트 정제는 원시 텍스트에서 원하지 않는 요소를 제거하는 것을 포함합니다. 여기에는 HTML 태그, URL, 이메일 주소, 특수 문자, 과도한 공백 및 의미론적 의미에 기여하지 않는 기타 아티팩트가 포함됩니다.

일반적인 정제 작업

import re
import unicodedata

def clean_text(text):
    """
    포괄적인 텍스트 정제 함수
    """
    # 소문자로 변환
    text = text.lower()

    # URL 제거
    text = re.sub(r'http\S+|www.\S+', '', text)

    # 이메일 주소 제거
    text = re.sub(r'\S+@\S+', '', text)

    # HTML 태그 제거
    text = re.sub(r'<.*?>', '', text)

    # 특수 문자 제거하되 구두점은 유지
    text = re.sub(r'[^a-zA-Z0-9\s.,!?;:\'-]', '', text)

    # 유니코드 문자 정규화
    text = unicodedata.normalize('NFKD', text)

    # 추가 공백 제거
    text = ' '.join(text.split())

    return text

# 사용 예
raw_text = "https://example.com을 확인하세요! 이메일: user@domain.com 
HTML 콘텐츠
"
cleaned = clean_text(raw_text)
print(cleaned)
# 출력: "을 확인하세요 이메일"

2.3 토큰화 기초

토큰화는 텍스트를 토큰이라고 하는 더 작은 단위로 분해하는 과정입니다. 이러한 토큰은 토큰화 전략에 따라 단어, 서브워드, 문자 또는 문장이 될 수 있습니다. 토큰화는 연속적인 텍스트를 머신러닝 모델이 처리할 수 있는 개별 요소로 변환하기 때문에 기본적입니다.

토큰화 유형

유형 설명 사용 사례
단어 토큰화 텍스트를 단어로 분할 전통적인 NLP 작업, 간단한 분석
서브워드 토큰화 의미있는 하위 단위로 분할 현대 트랜스포머, 희귀 단어 처리
문자 토큰화 개별 문자를 토큰으로 문자 수준 모델, 다국어
문장 토큰화 텍스트를 문장으로 분할 문서 분석, 요약

2.4 단어 토큰화 기법

단어 토큰화는 간단해 보입니다—공백으로 분할—하지만 언어는 축약형, 하이픈으로 연결된 단어, 구두점 첨부 및 단어 경계가 명확하지 않은 형태론적으로 풍부한 언어를 포함하여 수많은 과제를 제시합니다. 

import nltk
from nltk.tokenize import word_tokenize, WhitespaceTokenizer
from nltk.tokenize import RegexpTokenizer

# 필요한 리소스 다운로드
nltk.download('punkt')

text = "WIA-AI-023은 단순한 표준이 아닙니다—혁신적입니다!"

# NLTK 단어 토크나이저
tokens_nltk = word_tokenize(text)
print("NLTK:", tokens_nltk)
# 출력: ['WIA-AI-023은', '단순한', '표준이', '아닙니다', '—', '혁신적입니다', '!']

# 공백 토크나이저
ws_tokenizer = WhitespaceTokenizer()
tokens_ws = ws_tokenizer.tokenize(text)
print("공백:", tokens_ws)
# 출력: ['WIA-AI-023은', '단순한', '표준이', '아닙니다—혁신적입니다!']

# 정규식 토크나이저 (단어만)
regexp_tokenizer = RegexpTokenizer(r'\w+')
tokens_regexp = regexp_tokenizer.tokenize(text)
print("정규식:", tokens_regexp)
# 출력: ['WIA', 'AI', '023은', '단순한', '표준이', '아닙니다', '혁신적입니다']

2.5 서브워드 토큰화

서브워드 토큰화는 현대 NLP 모델의 표준이 되었습니다. 희귀하거나 알려지지 않은 단어를 의미있는 하위 단위로 분해함으로써 어휘 크기와 의미론적 풍부함 사이의 균형을 맞춥니다. 이 접근법은 어휘 외 단어를 우아하게 처리하고 여러 언어에서 잘 작동합니다.

인기있는 서브워드 알고리즘

바이트 페어 인코딩(BPE)

BPE는 가장 빈번한 문자 또는 문자 시퀀스 쌍을 반복적으로 병합합니다. 개별 문자로 시작하여 훈련 데이터 빈도를 기반으로 서브워드 단위의 어휘를 구축합니다. BPE는 GPT 모델 및 기타 많은 트랜스포머에서 사용됩니다.

워드피스(WordPiece)

BPE와 유사하지만 병합 결정에 우도 최대화를 사용합니다. 워드피스는 BERT 및 많은 Google 모델에서 사용됩니다. 서브워드 연속에 "##" 접두사를 추가하여 토큰화를 투명하게 만듭니다.

센텐스피스(SentencePiece)

텍스트를 유니코드 문자 시퀀스로 처리하는 언어 독립적 서브워드 토큰화입니다. 사전 토큰화가 필요하지 않으며 원시 텍스트에서 직접 작동하므로 다국어 모델에 이상적입니다. 

from transformers import BertTokenizer, GPT2Tokenizer

# BERT 토크나이저 (워드피스)
bert_tokenizer = BertTokenizer.from_pretrained('bert-base-uncased')
text = "자연어 처리 표준화"
tokens_bert = bert_tokenizer.tokenize(text)
print("BERT:", tokens_bert)
# 출력: ['자연어', '처리', '표준', '##화']

# GPT-2 토크나이저 (BPE)
gpt2_tokenizer = GPT2Tokenizer.from_pretrained('gpt2')
tokens_gpt2 = gpt2_tokenizer.tokenize(text)
print("GPT-2:", tokens_gpt2)
# 출력: ['자연어', 'Ġ처리', 'Ġ표준', '화']

# 토큰을 ID로 변환
ids_bert = bert_tokenizer.convert_tokens_to_ids(tokens_bert)
print("토큰 ID:", ids_bert)

2.6 텍스트 정규화 기법

정규화는 텍스트 변형을 표준 형식으로 축소하여 어휘 크기를 줄이고 모델 효율성을 향상시킵니다. 일반적인 정규화 기법에는 대소문자 변환, 어간 추출, 표제어 추출 및 맞춤법 교정이 포함됩니다.

대소문자 정규화

모든 텍스트를 소문자로 변환하는 것이 가장 간단한 정규화입니다. 그러나 대소문자 정보는 의미가 있을 수 있으므로 (예: 회사 "Apple" 대 과일 "apple") 대소문자 보존이 때때로 필요합니다.

어간 추출

어간 추출은 접미사를 제거하여 단어를 어근 형태로 축소합니다. 빠르지만 비단어를 생성할 수 있으며 과도하게 또는 불충분하게 어간을 추출할 수 있습니다. 

from nltk.stem import PorterStemmer, SnowballStemmer

porter = PorterStemmer()
snowball = SnowballStemmer('english')

words = ['running', 'runs', 'ran', 'easily', 'fairly']

print("Porter 어간 추출기:")
for word in words:
    print(f"{word} -> {porter.stem(word)}")
# running -> run, runs -> run, ran -> ran, easily -> easili, fairly -> fairli

print("\nSnowball 어간 추출기:")
for word in words:
    print(f"{word} -> {snowball.stem(word)}")
# 약간의 차이가 있는 유사한 결과

표제어 추출

표제어 추출은 언어학적 지식을 사용하여 단어를 사전 형태(표제어)로 변환합니다. 어간 추출보다 정확하지만 계산 비용이 많이 들고 품사 정보가 필요합니다. 

from nltk.stem import WordNetLemmatizer
from nltk.corpus import wordnet
import nltk

nltk.download('wordnet')
nltk.download('averaged_perceptron_tagger')

lemmatizer = WordNetLemmatizer()

def get_wordnet_pos(word):
    """POS 태그를 워드넷 POS 태그로 매핑"""
    tag = nltk.pos_tag([word])[0][1][0].upper()
    tag_dict = {"J": wordnet.ADJ,
                "N": wordnet.NOUN,
                "V": wordnet.VERB,
                "R": wordnet.ADV}
    return tag_dict.get(tag, wordnet.NOUN)

words = ['running', 'runs', 'ran', 'better', 'geese']

for word in words:
    pos = get_wordnet_pos(word)
    lemma = lemmatizer.lemmatize(word, pos)
    print(f"{word} ({pos}) -> {lemma}")
# running (VERB) -> run
# runs (VERB) -> run
# ran (VERB) -> run
# better (ADJ) -> good
# geese (NOUN) -> goose

2.7 불용어 제거

불용어는 "the", "is", "at"과 같이 자주 나타나지만 의미론적 의미가 거의 없는 일반적인 단어입니다. 이들을 제거하면 차원성을 줄이고 효율성을 향상시킬 수 있지만, 현대 문맥화 모델은 종종 이들을 유지하는 것으로부터 이익을 얻습니다. 

from nltk.corpus import stopwords
import nltk

nltk.download('stopwords')

stop_words = set(stopwords.words('english'))

text = "WIA-AI-023 표준은 자연어 처리를 변화시키고 있습니다"
tokens = word_tokenize(text.lower())

filtered_tokens = [w for w in tokens if w not in stop_words]

print("원본:", tokens)
print("필터링:", filtered_tokens)
# 원본: ['wia-ai-023', '표준은', '자연어', '처리를', '변화시키고', '있습니다']
# 필터링: ['wia-ai-023', '표준은', '자연어', '처리를', '변화시키고', '있습니다']

2.8 특수 케이스 처리

실제 텍스트에는 신중한 처리가 필요한 수많은 특수 케이스가 포함되어 있습니다: 숫자, 날짜, URL, 이모지, 해시태그, 멘션 및 도메인별 용어.

숫자 및 수치 개체

숫자는 정규화될 수 있습니다 (예: "100", "hundred", "one hundred" → "100"), 철자로 표기되거나 특수 토큰으로 대체될 수 있습니다. 선택은 수치 값이 의미론적 중요성을 가지는지에 달려 있습니다.

이모지 및 이모티콘

이모지는 현대 텍스트에서 감정과 의미를 전달합니다. 이들은 텍스트 설명으로 변환되거나 제거되거나 감정 분석을 위한 특수 토큰으로 처리될 수 있습니다. 

import emoji

text = "저는 NLP를 사랑합니다! 😍 정말 놀랍습니다! 🚀"

# 이모지를 텍스트로 변환
text_with_descriptions = emoji.demojize(text)
print(text_with_descriptions)
# 출력: "저는 NLP를 사랑합니다! :smiling_face_with_heart-eyes: 정말 놀랍습니다! :rocket:"

# 이모지 제거
text_no_emoji = emoji.replace_emoji(text, replace='')
print(text_no_emoji)
# 출력: "저는 NLP를 사랑합니다!  정말 놀랍습니다! "

2.10 전처리 파이프라인 구축

잘 설계된 전처리 파이프라인은 여러 작업을 올바른 순서로 연결하고, 오류를 우아하게 처리하며, 다양한 사용 사례에 대한 유연성을 제공합니다. 

class TextPreprocessor:
    def __init__(self,
                 lowercase=True,
                 remove_urls=True,
                 remove_html=True,
                 remove_emoji=False,
                 tokenizer='word',
                 remove_stopwords=False,
                 lemmatize=False):
        self.lowercase = lowercase
        self.remove_urls = remove_urls
        self.remove_html = remove_html
        self.remove_emoji = remove_emoji
        self.tokenizer = tokenizer
        self.remove_stopwords = remove_stopwords
        self.lemmatize = lemmatize

        if self.remove_stopwords:
            self.stop_words = set(stopwords.words('english'))
        if self.lemmatize:
            self.lemmatizer = WordNetLemmatizer()

    def preprocess(self, text):
        # 정제 작업 적용
        if self.lowercase:
            text = text.lower()
        if self.remove_urls:
            text = re.sub(r'http\S+|www.\S+', '', text)
        if self.remove_html:
            text = re.sub(r'<.*?>', '', text)
        if self.remove_emoji:
            text = emoji.replace_emoji(text, replace='')

        # 토큰화
        tokens = word_tokenize(text)

        # 불용어 제거
        if self.remove_stopwords:
            tokens = [t for t in tokens if t not in self.stop_words]

        # 표제어 추출
        if self.lemmatize:
            tokens = [self.lemmatizer.lemmatize(t) for t in tokens]

        return tokens

# 사용
preprocessor = TextPreprocessor(
    lowercase=True,
    remove_urls=True,
    remove_stopwords=True,
    lemmatize=True
)

text = "https://wiastandards.com을 확인하세요! NLP 전처리는 필수적입니다."
result = preprocessor.preprocess(text)
print(result)
# 출력: ['확인', '##하', '##세요', 'nlp', '전처리', '##는', '필수적', '##입니다', '.']

요약

핵심 내용:

  • 텍스트 전처리는 NLP 분석 전에 원시 텍스트를 정제하고 표준화하는 데 필수적
  • 토큰화는 텍스트를 처리 가능한 단위로 분해: 단어, 서브워드, 문자 또는 문장
  • 서브워드 토큰화(BPE, 워드피스, 센텐스피스)는 현대 트랜스포머의 표준
  • 어간 추출 및 표제어 추출을 포함한 정규화 기법은 어휘 크기를 축소
  • 불용어 제거는 효율성을 향상시킬 수 있지만 문맥화 모델에는 이익이 되지 않을 수 있음
  • 이모지, URL 및 숫자와 같은 특수 케이스는 신중한 처리가 필요
  • WIA-AI-023 표준은 상호운용성을 위한 구조화된 전처리 가이드라인 제공
  • 잘 설계된 전처리 파이프라인은 올바른 순서로 작업을 연결

  2. 어간 추출과 표제어 추출의 차이를 설명하시오. 각각 언제 사용하는가?
  3. 단어 토큰화와 서브워드 토큰화를 비교하시오. 서브워드 토큰화가 제공하는 장점은?
  4. 워드피스 알고리즘이 바이트 페어 인코딩(BPE)과 어떻게 다른가?
  5. NLP 전처리에서 불용어를 항상 제거해야 하는가? 이유를 설명하시오.
  6. 해시태그, 멘션 및 이모지를 포함하는 소셜 미디어 텍스트를 위한 전처리 파이프라인을 설계하시오.

다음 장 안내

3장에서는 전처리된 토큰을 머신러닝 모델이 이해할 수 있는 수치 형식으로 변환하는 단어 임베딩과 벡터 표현을 탐구합니다. Word2Vec, GloVe, FastText 및 문맥적 임베딩에 대해 배우게 됩니다.
