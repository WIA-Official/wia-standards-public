트랜스포머와 어텐션 메커니즘

4.1 트랜스포머 혁명

"Attention is All You Need" (Vaswani et al., 2017)에서 소개된 트랜스포머 아키텍처는 NLP를 근본적으로 변화시켰습니다. 순환 및 합성곱 계층을 어텐션 메커니즘으로 대체함으로써, 트랜스포머는 병렬 처리를 가능하게 하고, 장거리 의존성을 더 효과적으로 포착하며, 사실상 모든 NLP 작업에서 최첨단 결과를 달성했습니다.

트랜스포머 이전에는 RNN과 LSTM이 시퀀스 처리를 지배했습니다. 그러나 이들은 순차적 계산 병목, 기울기 소실/폭발 문제, 장거리 의존성 포착의 어려움을 겪었습니다. 트랜스포머는 셀프 어텐션과 위치 인코딩을 통해 이러한 문제를 해결했습니다.

트랜스포머의 핵심 혁신

  • 셀프 어텐션: 관련성에 기반하여 입력 요소에 가중치 부여
  • 병렬 처리: RNN과 같은 순차적 의존성 없음
  • 위치 인코딩: 시퀀스 순서 정보 통합
  • 멀티헤드 어텐션: 여러 표현 부분공간 학습
  • 레이어 정규화: 깊은 네트워크의 학습 안정화

4.2 어텐션 메커니즘 기초

어텐션 메커니즘은 모델이 출력을 생성할 때 입력의 관련 부분에 집중할 수 있게 합니다. 전체 시퀀스를 고정 크기 벡터로 인코딩하는 대신, 어텐션은 모든 입력 요소의 가중 조합을 동적으로 계산합니다.

어텐션 공식

핵심 어텐션 메커니즘은 쿼리(Q), 키(K), 값(V) 사이의 점수를 계산합니다: 

Attention(Q, K, V) = softmax(QK^T / sqrt(d_k)) * V

여기서:
- Q (쿼리): 찾고 있는 것
- K (키): 비교 대상
- V (값): 검색할 실제 콘텐츠
- d_k: 키 벡터의 차원 (스케일링용)

스케일드 닷-프로덕트 어텐션 구현

import torch
import torch.nn as nn
import math

def scaled_dot_product_attention(query, key, value, mask=None):
    """
    스케일드 닷-프로덕트 어텐션 계산

    Args:
        query: (batch, heads, seq_len, d_k)
        key: (batch, heads, seq_len, d_k)
        value: (batch, heads, seq_len, d_v)
        mask: 패딩/미래 토큰에 대한 선택적 마스크
    """
    d_k = query.size(-1)

    # 어텐션 점수 계산
    scores = torch.matmul(query, key.transpose(-2, -1)) / math.sqrt(d_k)

    # 마스크 적용 (제공된 경우)
    if mask is not None:
        scores = scores.masked_fill(mask == 0, -1e9)

    # 소프트맥스 적용
    attention_weights = torch.softmax(scores, dim=-1)

    # 값에 어텐션 적용
    output = torch.matmul(attention_weights, value)

    return output, attention_weights

4.3 멀티헤드 어텐션

멀티헤드 어텐션은 여러 어텐션 메커니즘을 병렬로 실행하며, 각각 단어 간 관계의 다른 측면을 학습합니다. 이를 통해 모델이 다양한 표현 부분공간의 정보에 동시에 주의를 기울일 수 있습니다. 

class MultiHeadAttention(nn.Module):
    def __init__(self, d_model, num_heads):
        super().__init__()
        assert d_model % num_heads == 0

        self.d_model = d_model
        self.num_heads = num_heads
        self.d_k = d_model // num_heads

        # Q, K, V에 대한 선형 계층
        self.W_q = nn.Linear(d_model, d_model)
        self.W_k = nn.Linear(d_model, d_model)
        self.W_v = nn.Linear(d_model, d_model)
        self.W_o = nn.Linear(d_model, d_model)

    def forward(self, query, key, value, mask=None):
        batch_size = query.size(0)

        # 선형 투영 및 헤드로 분할
        Q = self.W_q(query).view(batch_size, -1, self.num_heads, self.d_k).transpose(1, 2)
        K = self.W_k(key).view(batch_size, -1, self.num_heads, self.d_k).transpose(1, 2)
        V = self.W_v(value).view(batch_size, -1, self.num_heads, self.d_k).transpose(1, 2)

        # 어텐션 적용
        x, attention = scaled_dot_product_attention(Q, K, V, mask)

        # 헤드 연결
        x = x.transpose(1, 2).contiguous().view(batch_size, -1, self.d_model)

        # 최종 선형 투영
        return self.W_o(x)

4.4 위치 인코딩

트랜스포머는 모든 위치를 병렬로 처리하므로 명시적인 위치 정보가 필요합니다. 위치 인코딩은 입력 임베딩에 위치 의존적 패턴을 추가하여 모델이 시퀀스 순서를 활용할 수 있게 합니다. 

class PositionalEncoding(nn.Module):
    def __init__(self, d_model, max_len=5000):
        super().__init__()

        # 위치 인코딩 행렬 생성
        pe = torch.zeros(max_len, d_model)
        position = torch.arange(0, max_len).unsqueeze(1).float()
        div_term = torch.exp(torch.arange(0, d_model, 2).float() *
                           -(math.log(10000.0) / d_model))

        # 짝수 인덱스에 사인 적용
        pe[:, 0::2] = torch.sin(position * div_term)
        # 홀수 인덱스에 코사인 적용
        pe[:, 1::2] = torch.cos(position * div_term)

        pe = pe.unsqueeze(0)
        self.register_buffer('pe', pe)

    def forward(self, x):
        # 입력에 위치 인코딩 추가
        return x + self.pe[:, :x.size(1)]

4.5 트랜스포머 인코더 아키텍처

트랜스포머 인코더는 스택된 계층으로 구성되며, 각 계층은 멀티헤드 어텐션과 위치별 피드포워드 네트워크를 포함하고, 잔차 연결 및 레이어 정규화가 있습니다.

구성요소 목적 주요 파라미터
입력 임베딩 토큰을 벡터로 변환 vocab_size, d_model
위치 인코딩 위치 정보 추가 max_len, d_model
멀티헤드 어텐션 모든 위치에 주의 num_heads, d_k, d_v
피드포워드 네트워크 위치별 변환 d_ff (일반적으로 4*d_model)
레이어 정규화 학습 안정화 epsilon

4.6 트랜스포머 디코더 아키텍처

디코더는 미래 토큰에 주의하는 것을 방지하기 위한 추가 마스킹된 멀티헤드 어텐션 계층과 입력 시퀀스의 관련 부분에 집중하기 위한 인코더-디코더 어텐션을 포함합니다.

마스킹된 셀프 어텐션

학습 중에 디코더는 미래 토큰을 볼 수 없어야 합니다. 마스킹된 어텐션은 위치 i가 위치 ≤ i에만 주의할 수 있도록 삼각 마스크를 적용합니다. 

def create_causal_mask(seq_len):
    """인과적(자기회귀) 어텐션을 위한 마스크 생성"""
    mask = torch.triu(torch.ones(seq_len, seq_len), diagonal=1).bool()
    return ~mask  # 반전: True는 주의, False는 마스크

# 사용 예
seq_len = 5
mask = create_causal_mask(seq_len)
print(mask)
# 출력:
# tensor([[ True, False, False, False, False],
#         [ True,  True, False, False, False],
#         [ True,  True,  True, False, False],
#         [ True,  True,  True,  True, False],
#         [ True,  True,  True,  True,  True]])

4.7 BERT: 양방향 인코더 표현

BERT는 양방향 맥락으로 트랜스포머 인코더만 사용합니다. 마스킹된 언어 모델링과 다음 문장 예측에 대해 사전 학습된 BERT는 후속 작업에 잘 전이되는 깊은 양방향 표현을 학습합니다.

BERT 사전 학습 작업

  • 마스킹된 언어 모델링(MLM): 무작위로 마스킹된 토큰 예측
  • 다음 문장 예측(NSP): 두 문장이 연속적인지 판단
from transformers import BertTokenizer, BertModel

# 사전 학습된 BERT 로드
tokenizer = BertTokenizer.from_pretrained('bert-base-uncased')
model = BertModel.from_pretrained('bert-base-uncased')

# 텍스트 인코딩
text = "트랜스포머를 사용한 자연어 처리"
inputs = tokenizer(text, return_tensors='pt', padding=True)

# 임베딩 가져오기
outputs = model(**inputs)
last_hidden_states = outputs.last_hidden_state
pooled_output = outputs.pooler_output

print(f"마지막 은닉 상태 형태: {last_hidden_states.shape}")
print(f"풀링된 출력 형태: {pooled_output.shape}")

4.8 GPT: 생성적 사전 학습 트랜스포머

GPT는 자기회귀 언어 모델링을 위해 트랜스포머 디코더를 사용합니다. 이전 토큰이 주어졌을 때 다음 토큰을 예측하도록 학습된 GPT는 텍스트 생성과 퓨샷 학습에서 뛰어납니다.

GPT 대 BERT

측면 BERT GPT
아키텍처 인코더만 디코더만
어텐션 양방향 단방향(인과적)
주요 용도 이해 작업 생성 작업
사전 학습 MLM + NSP 다음 토큰 예측

4.9 트랜스포머 변형

원래 트랜스포머 이후, 효율성, 성능을 개선하거나 특정 작업에 특화하기 위해 수많은 변형이 개발되었습니다.

주목할 만한 트랜스포머 변형

  • RoBERTa: 향상된 학습으로 강력하게 최적화된 BERT
  • ALBERT: 파라미터 효율적인 BERT 변형
  • T5: 통합된 작업 처리를 위한 텍스트-투-텍스트 트랜스포머
  • XLNet: 순열 기반 언어 모델링
  • Longformer: 긴 문서를 위한 효율적인 어텐션
  • ELECTRA: MLM 대신 교체된 토큰 탐지

요약

핵심 내용:

  • 트랜스포머는 셀프 어텐션과 병렬 처리를 통해 NLP를 혁신
  • 스케일드 닷-프로덕트 어텐션은 관련성에 기반하여 입력 요소에 가중치 부여
  • 멀티헤드 어텐션은 여러 표현 부분공간을 동시에 학습
  • 위치 인코딩은 트랜스포머에 시퀀스 순서 정보 제공
  • 인코더 아키텍처(BERT)는 양방향 맥락으로 이해 작업에서 뛰어남
  • 디코더 아키텍처(GPT)는 인과적 어텐션으로 생성에서 뛰어남
  • 수많은 트랜스포머 변형이 효율성, 성능 또는 특정 사용 사례에 최적화
