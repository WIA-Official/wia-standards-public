CHAPTER 4

Transformers and Attention Mechanisms

4.1 The Transformer Revolution

The transformer architecture, introduced in "Attention is All You Need" (Vaswani et al., 2017), fundamentally changed NLP. By replacing recurrent and convolutional layers with attention mechanisms, transformers enabled parallel processing, captured long-range dependencies more effectively, and achieved state-of-the-art results across virtually all NLP tasks.

Before transformers, RNNs and LSTMs dominated sequence processing. However, they suffered from sequential computation bottlenecks, gradient vanishing/exploding problems, and difficulty capturing long-range dependencies. Transformers solved these issues through self-attention and positional encoding.

Key Innovations in Transformers

  • Self-Attention: Weighting input elements based on relevance
  • Parallel Processing: No sequential dependencies like RNNs
  • Positional Encoding: Incorporating sequence order information
  • Multi-Head Attention: Learning multiple representation subspaces
  • Layer Normalization: Stabilizing training of deep networks

4.2 Attention Mechanism Basics

Attention mechanisms allow models to focus on relevant parts of the input when producing outputs. Instead of encoding entire sequences into fixed-size vectors, attention computes weighted combinations of all input elements dynamically.

Attention Formula

The core attention mechanism computes scores between queries (Q), keys (K), and values (V): 

Attention(Q, K, V) = softmax(QK^T / sqrt(d_k)) * V

Where:
- Q (Query): What we're looking for
- K (Key): What we're comparing against
- V (Value): The actual content to retrieve
- d_k: Dimension of key vectors (for scaling)

Scaled Dot-Product Attention Implementation

import torch
import torch.nn as nn
import math

def scaled_dot_product_attention(query, key, value, mask=None):
    """
    Compute scaled dot-product attention

    Args:
        query: (batch, heads, seq_len, d_k)
        key: (batch, heads, seq_len, d_k)
        value: (batch, heads, seq_len, d_v)
        mask: Optional mask for padding/future tokens
    """
    d_k = query.size(-1)

    # Compute attention scores
    scores = torch.matmul(query, key.transpose(-2, -1)) / math.sqrt(d_k)

    # Apply mask if provided
    if mask is not None:
        scores = scores.masked_fill(mask == 0, -1e9)

    # Apply softmax
    attention_weights = torch.softmax(scores, dim=-1)

    # Apply attention to values
    output = torch.matmul(attention_weights, value)

    return output, attention_weights

4.3 Multi-Head Attention

Multi-head attention runs multiple attention mechanisms in parallel, each learning different aspects of relationships between words. This allows the model to attend to information from different representation subspaces simultaneously. 

class MultiHeadAttention(nn.Module):
    def __init__(self, d_model, num_heads):
        super().__init__()
        assert d_model % num_heads == 0

        self.d_model = d_model
        self.num_heads = num_heads
        self.d_k = d_model // num_heads

        # Linear layers for Q, K, V
        self.W_q = nn.Linear(d_model, d_model)
        self.W_k = nn.Linear(d_model, d_model)
        self.W_v = nn.Linear(d_model, d_model)
        self.W_o = nn.Linear(d_model, d_model)

    def forward(self, query, key, value, mask=None):
        batch_size = query.size(0)

        # Linear projections and split into heads
        Q = self.W_q(query).view(batch_size, -1, self.num_heads, self.d_k).transpose(1, 2)
        K = self.W_k(key).view(batch_size, -1, self.num_heads, self.d_k).transpose(1, 2)
        V = self.W_v(value).view(batch_size, -1, self.num_heads, self.d_k).transpose(1, 2)

        # Apply attention
        x, attention = scaled_dot_product_attention(Q, K, V, mask)

        # Concatenate heads
        x = x.transpose(1, 2).contiguous().view(batch_size, -1, self.d_model)

        # Final linear projection
        return self.W_o(x)

4.4 Positional Encoding

Since transformers process all positions in parallel, they need explicit positional information. Positional encoding adds position-dependent patterns to input embeddings, allowing the model to utilize sequence order. 

class PositionalEncoding(nn.Module):
    def __init__(self, d_model, max_len=5000):
        super().__init__()

        # Create positional encoding matrix
        pe = torch.zeros(max_len, d_model)
        position = torch.arange(0, max_len).unsqueeze(1).float()
        div_term = torch.exp(torch.arange(0, d_model, 2).float() *
                           -(math.log(10000.0) / d_model))

        # Apply sine to even indices
        pe[:, 0::2] = torch.sin(position * div_term)
        # Apply cosine to odd indices
        pe[:, 1::2] = torch.cos(position * div_term)

        pe = pe.unsqueeze(0)
        self.register_buffer('pe', pe)

    def forward(self, x):
        # Add positional encoding to input
        return x + self.pe[:, :x.size(1)]

4.5 Transformer Encoder Architecture

The transformer encoder consists of stacked layers, each containing multi-head attention followed by a position-wise feedforward network, with residual connections and layer normalization.

Component Purpose Key Parameters
Input Embedding Convert tokens to vectors vocab_size, d_model
Positional Encoding Add position information max_len, d_model
Multi-Head Attention Attend to all positions num_heads, d_k, d_v
Feedforward Network Position-wise transformation d_ff (typically 4*d_model)
Layer Normalization Stabilize training epsilon

4.6 Transformer Decoder Architecture

The decoder includes an additional masked multi-head attention layer to prevent attending to future tokens, plus encoder-decoder attention to focus on relevant parts of the input sequence.

Masked Self-Attention

During training, the decoder must not see future tokens. Masked attention applies a triangular mask ensuring position i can only attend to positions ≤ i. 

def create_causal_mask(seq_len):
    """Create mask for causal (autoregressive) attention"""
    mask = torch.triu(torch.ones(seq_len, seq_len), diagonal=1).bool()
    return ~mask  # Invert: True means attend, False means mask

# Example usage
seq_len = 5
mask = create_causal_mask(seq_len)
print(mask)
# Output:
# tensor([[ True, False, False, False, False],

#         [ True,  True,  True, False, False],
#         [ True,  True,  True,  True, False],
#         [ True,  True,  True,  True,  True]])

4.7 BERT: Bidirectional Encoder Representations

BERT uses only the transformer encoder with bidirectional context. Pre-trained on masked language modeling and next sentence prediction, BERT learns deep bidirectional representations that transfer well to downstream tasks.

BERT Pre-training Tasks

  • Masked Language Modeling (MLM): Predict randomly masked tokens
  • Next Sentence Prediction (NSP): Determine if two sentences are consecutive
from transformers import BertTokenizer, BertModel

# Load pre-trained BERT
tokenizer = BertTokenizer.from_pretrained('bert-base-uncased')
model = BertModel.from_pretrained('bert-base-uncased')

# Encode text
text = "Natural language processing with transformers"
inputs = tokenizer(text, return_tensors='pt', padding=True)

# Get embeddings
outputs = model(**inputs)
last_hidden_states = outputs.last_hidden_state
pooled_output = outputs.pooler_output

print(f"Last hidden state shape: {last_hidden_states.shape}")
print(f"Pooled output shape: {pooled_output.shape}")

4.8 GPT: Generative Pre-trained Transformer

GPT uses the transformer decoder for autoregressive language modeling. Trained to predict the next token given previous tokens, GPT excels at text generation and few-shot learning.

GPT vs BERT

Aspect BERT GPT
Architecture Encoder only Decoder only
Attention Bidirectional Unidirectional (causal)
Primary Use Understanding tasks Generation tasks
Pre-training MLM + NSP Next token prediction

4.9 Transformer Variants

Since the original transformer, numerous variants have been developed to improve efficiency, performance, or specialize for specific tasks.

Notable Transformer Variants

  • RoBERTa: Robustly optimized BERT with improved training
  • ALBERT: Parameter-efficient BERT variant
  • T5: Text-to-text transformer for unified task handling
  • XLNet: Permutation-based language modeling
  • Longformer: Efficient attention for long documents
  • ELECTRA: Replaced token detection instead of MLM
