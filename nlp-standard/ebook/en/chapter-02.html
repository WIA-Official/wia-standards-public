CHAPTER 2

Text Preprocessing and Tokenization

2.1 The Importance of Text Preprocessing

Text preprocessing is the crucial first step in any NLP pipeline. Raw text from the internet, documents, or user inputs contains noise, inconsistencies, and formatting that must be cleaned and standardized before analysis. Effective preprocessing can significantly improve the performance of downstream NLP tasks by reducing data complexity and focusing on meaningful patterns.

The quality of preprocessing directly impacts model performance. Poorly preprocessed data can introduce noise, reduce accuracy, and create biases in NLP systems. Conversely, well-designed preprocessing pipelines extract signal from noise, normalize variations, and prepare text for efficient processing.

Goals of Text Preprocessing

  • Remove irrelevant information and noise
  • Standardize text format and encoding
  • Reduce vocabulary size while preserving meaning
  • Normalize variations in word forms
  • Handle special characters, punctuation, and whitespace
  • Prepare text for efficient tokenization

2.2 Text Cleaning and Normalization

Text cleaning involves removing unwanted elements from raw text. This includes HTML tags, URLs, email addresses, special characters, excessive whitespace, and other artifacts that don't contribute to semantic meaning.

Common Cleaning Operations

import re
import unicodedata

def clean_text(text):
    """
    Comprehensive text cleaning function
    """
    # Convert to lowercase
    text = text.lower()

    # Remove URLs
    text = re.sub(r'http\S+|www.\S+', '', text)

    # Remove email addresses
    text = re.sub(r'\S+@\S+', '', text)

    # Remove HTML tags
    text = re.sub(r'<.*?>', '', text)

    # Remove special characters but keep punctuation
    text = re.sub(r'[^a-zA-Z0-9\s.,!?;:\'-]', '', text)

    # Normalize unicode characters
    text = unicodedata.normalize('NFKD', text)

    # Remove extra whitespace
    text = ' '.join(text.split())

    return text

# Example usage
raw_text = "Check out https://example.com! Email: user@domain.com 
HTML content
"
cleaned = clean_text(raw_text)
print(cleaned)
# Output: "check out email"

2.3 Tokenization Fundamentals

Tokenization is the process of breaking text into smaller units called tokens. These tokens can be words, subwords, characters, or sentences depending on the tokenization strategy. Tokenization is fundamental because it transforms continuous text into discrete elements that can be processed by machine learning models.

Types of Tokenization

Type Description Use Case
Word Tokenization Split text into words Traditional NLP tasks, simple analysis
Subword Tokenization Split into meaningful subunits Modern transformers, handling rare words
Character Tokenization Individual characters as tokens Character-level models, multilingual
Sentence Tokenization Split text into sentences Document analysis, summarization

2.4 Word Tokenization Techniques

Word tokenization appears simple—split on whitespace—but languages present numerous challenges including contractions, hyphenated words, punctuation attachment, and morphologically rich languages where word boundaries aren't clear. 

import nltk
from nltk.tokenize import word_tokenize, WhitespaceTokenizer
from nltk.tokenize import RegexpTokenizer

# Download required resources
nltk.download('punkt')

text = "Don't forget: WIA-AI-023 isn't just a standard—it's transformative!"

# NLTK word tokenizer
tokens_nltk = word_tokenize(text)
print("NLTK:", tokens_nltk)
# Output: ['Do', "n't", 'forget', ':', 'WIA-AI-023', 'is', "n't", 'just', ...]

# Whitespace tokenizer
ws_tokenizer = WhitespaceTokenizer()
tokens_ws = ws_tokenizer.tokenize(text)
print("Whitespace:", tokens_ws)
# Output: ["Don't", 'forget:', 'WIA-AI-023', "isn't", ...]

# Regular expression tokenizer (words only)
regexp_tokenizer = RegexpTokenizer(r'\w+')
tokens_regexp = regexp_tokenizer.tokenize(text)
print("Regexp:", tokens_regexp)
# Output: ['Don', 't', 'forget', 'WIA', 'AI', '023', ...]

2.5 Subword Tokenization

Subword tokenization has become the standard for modern NLP models. It balances vocabulary size with semantic richness by breaking rare or unknown words into meaningful subunits. This approach handles out-of-vocabulary words gracefully and works well across languages.

Popular Subword Algorithms

Byte Pair Encoding (BPE)

BPE iteratively merges the most frequent pairs of characters or character sequences. Starting with individual characters, it builds a vocabulary of subword units based on training data frequency. BPE is used in GPT models and many other transformers.

WordPiece

Similar to BPE but uses likelihood maximization for merging decisions. WordPiece is used in BERT and many Google models. It adds "##" prefix to subword continuations, making tokenization transparent.

SentencePiece

Language-independent subword tokenization that treats text as a sequence of Unicode characters. It doesn't require pre-tokenization and works directly on raw text, making it ideal for multilingual models. 

from transformers import BertTokenizer, GPT2Tokenizer

# BERT tokenizer (WordPiece)
bert_tokenizer = BertTokenizer.from_pretrained('bert-base-uncased')
text = "Natural language processing standardization"
tokens_bert = bert_tokenizer.tokenize(text)
print("BERT:", tokens_bert)
# Output: ['natural', 'language', 'processing', 'standard', '##ization']

# GPT-2 tokenizer (BPE)
gpt2_tokenizer = GPT2Tokenizer.from_pretrained('gpt2')
tokens_gpt2 = gpt2_tokenizer.tokenize(text)
print("GPT-2:", tokens_gpt2)
# Output: ['Natural', 'Ġlanguage', 'Ġprocessing', 'Ġstandard', 'ization']

# Convert tokens to IDs
ids_bert = bert_tokenizer.convert_tokens_to_ids(tokens_bert)
print("Token IDs:", ids_bert)

2.6 Text Normalization Techniques

Normalization reduces text variations to standard forms, decreasing vocabulary size and improving model efficiency. Common normalization techniques include case folding, stemming, lemmatization, and spell correction.

Case Normalization

Converting all text to lowercase is the simplest normalization. However, case information can be meaningful (e.g., "Apple" the company vs. "apple" the fruit), so case preservation is sometimes necessary.

Stemming

Stemming reduces words to their root form by removing suffixes. It's fast but can produce non-words and may over-stem or under-stem. 

from nltk.stem import PorterStemmer, SnowballStemmer

porter = PorterStemmer()
snowball = SnowballStemmer('english')

words = ['running', 'runs', 'ran', 'easily', 'fairly']

print("Porter Stemmer:")
for word in words:
    print(f"{word} -> {porter.stem(word)}")
# running -> run, runs -> run, ran -> ran, easily -> easili, fairly -> fairli

print("\nSnowball Stemmer:")
for word in words:
    print(f"{word} -> {snowball.stem(word)}")
# Similar results with slight differences

Lemmatization

Lemmatization uses linguistic knowledge to convert words to their dictionary form (lemma). It's more accurate than stemming but computationally expensive and requires part-of-speech information. 

from nltk.stem import WordNetLemmatizer
from nltk.corpus import wordnet
import nltk

nltk.download('wordnet')
nltk.download('averaged_perceptron_tagger')

lemmatizer = WordNetLemmatizer()

def get_wordnet_pos(word):
    """Map POS tag to WordNet POS tag"""
    tag = nltk.pos_tag([word])[0][1][0].upper()
    tag_dict = {"J": wordnet.ADJ,
                "N": wordnet.NOUN,
                "V": wordnet.VERB,
                "R": wordnet.ADV}
    return tag_dict.get(tag, wordnet.NOUN)

words = ['running', 'runs', 'ran', 'better', 'geese']

for word in words:
    pos = get_wordnet_pos(word)
    lemma = lemmatizer.lemmatize(word, pos)
    print(f"{word} ({pos}) -> {lemma}")
# running (VERB) -> run
# runs (VERB) -> run
# ran (VERB) -> run
# better (ADJ) -> good
# geese (NOUN) -> goose

2.7 Stop Word Removal

Stop words are common words like "the", "is", "at" that appear frequently but carry little semantic meaning. Removing them can reduce dimensionality and improve efficiency, though modern contextualized models often benefit from keeping them. 

from nltk.corpus import stopwords
import nltk

nltk.download('stopwords')

stop_words = set(stopwords.words('english'))

text = "The WIA-AI-023 standard is transforming natural language processing"
tokens = word_tokenize(text.lower())

filtered_tokens = [w for w in tokens if w not in stop_words]

print("Original:", tokens)
print("Filtered:", filtered_tokens)
# Original: ['the', 'wia-ai-023', 'standard', 'is', 'transforming', ...]
# Filtered: ['wia-ai-023', 'standard', 'transforming', ...]

2.8 Handling Special Cases

Real-world text contains numerous special cases that require careful handling: numbers, dates, URLs, emojis, hashtags, mentions, and domain-specific terminology.

Numbers and Numerical Entities

Numbers can be normalized (e.g., "100", "hundred", "one hundred" → "100"), spelled out, or replaced with special tokens. The choice depends on whether numerical values carry semantic importance.

Emojis and Emoticons

Emojis convey emotion and meaning in modern text. They can be converted to textual descriptions, removed, or treated as special tokens for sentiment analysis. 

import emoji

text = "I love NLP! 😍 It's amazing! 🚀"

# Convert emoji to text
text_with_descriptions = emoji.demojize(text)
print(text_with_descriptions)
# Output: "I love NLP! :smiling_face_with_heart-eyes: It's amazing! :rocket:"

# Remove emojis
text_no_emoji = emoji.replace_emoji(text, replace='')
print(text_no_emoji)
# Output: "I love NLP!  It's amazing! "

2.10 Building a Preprocessing Pipeline

A well-designed preprocessing pipeline chains multiple operations in the correct order, handles errors gracefully, and provides flexibility for different use cases. 

class TextPreprocessor:
    def __init__(self,
                 lowercase=True,
                 remove_urls=True,
                 remove_html=True,
                 remove_emoji=False,
                 tokenizer='word',
                 remove_stopwords=False,
                 lemmatize=False):
        self.lowercase = lowercase
        self.remove_urls = remove_urls
        self.remove_html = remove_html
        self.remove_emoji = remove_emoji
        self.tokenizer = tokenizer
        self.remove_stopwords = remove_stopwords
        self.lemmatize = lemmatize

        if self.remove_stopwords:
            self.stop_words = set(stopwords.words('english'))
        if self.lemmatize:
            self.lemmatizer = WordNetLemmatizer()

    def preprocess(self, text):
        # Apply cleaning operations
        if self.lowercase:
            text = text.lower()
        if self.remove_urls:
            text = re.sub(r'http\S+|www.\S+', '', text)
        if self.remove_html:
            text = re.sub(r'<.*?>', '', text)
        if self.remove_emoji:
            text = emoji.replace_emoji(text, replace='')

        # Tokenize
        tokens = word_tokenize(text)

        # Remove stopwords
        if self.remove_stopwords:
            tokens = [t for t in tokens if t not in self.stop_words]

        # Lemmatize
        if self.lemmatize:
            tokens = [self.lemmatizer.lemmatize(t) for t in tokens]

        return tokens

# Usage
preprocessor = TextPreprocessor(
    lowercase=True,
    remove_urls=True,
    remove_stopwords=True,
    lemmatize=True
)

text = "Check out https://wiastandards.com! NLP preprocessing is essential."
result = preprocessor.preprocess(text)
print(result)
# Output: ['check', 'nlp', 'preprocessing', 'essential', '.']

Chapter Summary

Key Takeaways:

  • Text preprocessing is essential for cleaning and standardizing raw text before NLP analysis
  • Tokenization breaks text into processable units: words, subwords, characters, or sentences
