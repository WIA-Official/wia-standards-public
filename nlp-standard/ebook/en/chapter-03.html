CHAPTER 3

Word Embeddings and Representations

3.1 From Words to Vectors

Machine learning models cannot process text directly—they require numerical representations. Word embeddings transform discrete words into continuous vector spaces where semantic relationships are preserved through geometric properties. This transformation enables mathematical operations on language and powers modern NLP systems.

The fundamental insight behind word embeddings is the distributional hypothesis: words that appear in similar contexts tend to have similar meanings. By analyzing word co-occurrence patterns in large text corpora, embedding algorithms learn dense vector representations that capture semantic and syntactic relationships.

Why Word Embeddings Matter

  • Semantic similarity: Similar words have similar vectors
  • Dimensionality reduction: Dense vectors vs. sparse one-hot encoding
  • Transfer learning: Pre-trained embeddings encode linguistic knowledge
  • Arithmetic operations: Vector math captures semantic relationships
  • Generalization: Models learn patterns that generalize beyond training data

3.2 Traditional Approaches: One-Hot and TF-IDF

Before word embeddings became standard, NLP relied on sparse representations that encoded words as high-dimensional vectors with limited semantic information.

One-Hot Encoding

One-hot encoding represents each word as a vector with length equal to vocabulary size, with a single 1 and all other values 0. This simple approach has severe limitations: massive dimensionality, no semantic relationships, and inefficient computation. 

import numpy as np

# Vocabulary
vocab = ['natural', 'language', 'processing', 'machine', 'learning']

# One-hot encoding function
def one_hot_encode(word, vocab):
    vector = np.zeros(len(vocab))
    if word in vocab:
        vector[vocab.index(word)] = 1
    return vector

# Example
word = 'language'
one_hot = one_hot_encode(word, vocab)
print(f"{word}: {one_hot}")
# Output: [0. 1. 0. 0. 0.]

# Problem: No semantic relationship
vec_language = one_hot_encode('language', vocab)
vec_processing = one_hot_encode('processing', vocab)
similarity = np.dot(vec_language, vec_processing)
print(f"Similarity: {similarity}")  # Always 0!

TF-IDF (Term Frequency-Inverse Document Frequency)

TF-IDF weights words by their importance: frequently appearing in a document but rare across the corpus receive higher weights. While more informative than one-hot, TF-IDF still produces sparse vectors without capturing semantic relationships. 

from sklearn.feature_extraction.text import TfidfVectorizer

documents = [
    "Natural language processing is fundamental",
    "Machine learning enables NLP systems",
    "Deep learning revolutionized natural language understanding"
]

vectorizer = TfidfVectorizer()
tfidf_matrix = vectorizer.fit_transform(documents)

print("Vocabulary:", vectorizer.get_feature_names_out())
print("TF-IDF matrix shape:", tfidf_matrix.shape)
print("\nDocument 0 vector:\n", tfidf_matrix[0].toarray())

3.3 Word2Vec: Learning from Context

Word2Vec, introduced by Mikolov et al. in 2013, revolutionized NLP by learning dense word embeddings from large text corpora. Word2Vec uses shallow neural networks trained on word context prediction tasks.

Word2Vec Architectures

CBOW (Continuous Bag of Words)

CBOW predicts a target word from surrounding context words. Given context words within a window, the model learns to predict the center word. CBOW is faster and works well with frequent words.

Skip-gram

Skip-gram does the opposite: given a target word, it predicts surrounding context words. Skip-gram works better with rare words and smaller datasets, producing higher quality embeddings for infrequent terms. 

from gensim.models import Word2Vec
import gensim.downloader as api

# Load pre-trained model (or train your own)
# For demo, using a small corpus
sentences = [
    ['natural', 'language', 'processing'],
    ['machine', 'learning', 'algorithms'],
    ['deep', 'learning', 'neural', 'networks'],
    ['natural', 'language', 'understanding'],
    ['text', 'processing', 'techniques']
]

# Train Word2Vec model
model = Word2Vec(sentences,
                 vector_size=100,  # Embedding dimension
                 window=5,         # Context window
                 min_count=1,      # Minimum word frequency
                 workers=4,        # Parallel threads
                 sg=1)             # Skip-gram (0 for CBOW)

# Get word vector
vector = model.wv['language']
print(f"Vector shape: {vector.shape}")
print(f"Vector (first 10 dims): {vector[:10]}")

# Find similar words
similar = model.wv.most_similar('language', topn=3)
print(f"\nWords similar to 'language': {similar}")

# Vector arithmetic
result = model.wv.most_similar(positive=['processing', 'machine'],
                               negative=['natural'],
                               topn=1)
print(f"\nprocessing + machine - natural = {result}")

3.4 GloVe: Global Vectors

GloVe (Global Vectors for Word Representation) combines the benefits of matrix factorization methods with local context window approaches. Unlike Word2Vec which uses local context, GloVe leverages global word co-occurrence statistics from the entire corpus.

How GloVe Works

  1. Construct a word co-occurrence matrix from the corpus
  2. Factorize the matrix to learn word vectors
  3. Optimize to preserve co-occurrence probabilities
# Using pre-trained GloVe embeddings
import numpy as np

def load_glove_embeddings(glove_file):
    """Load GloVe embeddings from file"""
    embeddings = {}
    with open(glove_file, 'r', encoding='utf-8') as f:
        for line in f:
            values = line.split()
            word = values[0]
            vector = np.asarray(values[1:], dtype='float32')
            embeddings[word] = vector
    return embeddings

# Example usage (assuming glove.6B.100d.txt is available)
# embeddings = load_glove_embeddings('glove.6B.100d.txt')

# Compute similarity
def cosine_similarity(vec1, vec2):
    return np.dot(vec1, vec2) / (np.linalg.norm(vec1) * np.linalg.norm(vec2))

# Example with hypothetical vectors
vec_king = np.random.randn(100)
vec_queen = np.random.randn(100)
similarity = cosine_similarity(vec_king, vec_queen)
print(f"Similarity: {similarity}")

3.5 FastText: Subword Embeddings

FastText extends Word2Vec by representing words as bags of character n-grams. This approach handles out-of-vocabulary words, captures morphological information, and works well for morphologically rich languages. 

from gensim.models import FastText

# Training FastText
sentences = [
    ['natural', 'language', 'processing'],
    ['naturalization', 'naturally', 'nature'],
    ['process', 'processing', 'processed', 'processes']
]

model = FastText(sentences,
                 vector_size=100,
                 window=5,
                 min_count=1,
                 min_n=2,      # Minimum n-gram length
                 max_n=5,      # Maximum n-gram length
                 workers=4)

# Handle OOV words
# Even if 'naturalizing' wasn't in training, FastText can handle it
vector = model.wv['naturalizing']
print(f"OOV word vector shape: {vector.shape}")

# Subword information
print(f"\nSubwords for 'processing':")
print(model.wv.key_to_index.keys())

3.6 Contextual Embeddings: ELMo and Beyond

Traditional embeddings assign a single vector to each word regardless of context. "Bank" has the same representation in "river bank" and "savings bank." Contextual embeddings solve this by generating different representations based on surrounding words.

ELMo (Embeddings from Language Models)

ELMo uses bidirectional LSTM language models to generate context-dependent representations. Each word gets a unique vector based on its position and context in the sentence.

3.7 Embedding Evaluation

Evaluating embedding quality involves both intrinsic and extrinsic methods. Intrinsic evaluation tests embeddings directly, while extrinsic evaluation measures performance on downstream tasks.

Intrinsic Evaluation Methods

  • Word similarity: Correlation with human similarity judgments
  • Word analogy: Solving analogy tasks (king - man + woman = queen)
  • Clustering: Grouping semantically similar words
from scipy.stats import spearmanr

def evaluate_word_similarity(embeddings, word_pairs, human_scores):
    """
    Evaluate embeddings using word similarity dataset
    word_pairs: list of (word1, word2) tuples
    human_scores: list of human similarity ratings
    """
    predicted_scores = []

    for word1, word2 in word_pairs:
        if word1 in embeddings and word2 in embeddings:
            vec1 = embeddings[word1]
            vec2 = embeddings[word2]
            similarity = cosine_similarity(vec1, vec2)
            predicted_scores.append(similarity)
        else:
            predicted_scores.append(0.0)

    # Compute Spearman correlation
    correlation, p_value = spearmanr(human_scores, predicted_scores)
    return correlation, p_value

# Example usage
word_pairs = [('cat', 'dog'), ('car', 'automobile'), ('computer', 'keyboard')]
human_scores = [0.8, 0.95, 0.7]
# correlation, p_value = evaluate_word_similarity(embeddings, word_pairs, human_scores)

3.9 Practical Considerations

When working with word embeddings in production systems, several practical considerations affect performance, efficiency, and reliability.

Choosing Embedding Dimensions

  • 50-100 dimensions: Small datasets, limited compute
  • 200-300 dimensions: General purpose, good balance
  • 500-1000 dimensions: Large datasets, high performance needs

Handling Out-of-Vocabulary Words

  • Use FastText for subword information
  • Average embeddings of similar in-vocabulary words
  • Use special [UNK] token
  • Fine-tune embeddings on domain-specific data

3.10 Advanced Topics

Modern embedding research continues to advance, addressing limitations and expanding capabilities.

Domain-Specific Embeddings

General purpose embeddings may not capture domain-specific terminology and relationships. Training embeddings on medical, legal, or scientific corpora improves performance for specialized applications.

Multilingual Embeddings

Multilingual embeddings map words from different languages into a shared vector space, enabling cross-lingual transfer and machine translation. 

# Example: Using multilingual embeddings
from transformers import AutoTokenizer, AutoModel

# Load multilingual model
model_name = "bert-base-multilingual-cased"
tokenizer = AutoTokenizer.from_pretrained(model_name)
model = AutoModel.from_pretrained(model_name)

# Process text in different languages
texts = {
    'en': 'Natural language processing',
    'ko': '자연어 처리',
    'es': 'Procesamiento del lenguaje natural'
}

for lang, text in texts.items():
    inputs = tokenizer(text, return_tensors='pt')
    outputs = model(**inputs)
    print(f"{lang}: {outputs.last_hidden_state.shape}")
