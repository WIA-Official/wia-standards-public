Named Entity Recognition and POS Tagging

5.1 Introduction to Sequence Labeling

Sequence labeling assigns labels to each element in a sequence. Named Entity Recognition (NER) and Part-of-Speech (POS) tagging are fundamental sequence labeling tasks that extract structured information from unstructured text. These tasks serve as building blocks for many downstream NLP applications including information extraction, question answering, and knowledge graph construction.

Unlike classification tasks that assign a single label to entire texts, sequence labeling requires understanding context and relationships between neighboring words. Modern approaches use contextual embeddings and transformer models to achieve state-of-the-art performance.

5.2 Part-of-Speech Tagging

POS tagging assigns grammatical categories (noun, verb, adjective, etc.) to each word. Understanding parts of speech helps with syntactic parsing, named entity recognition, and semantic analysis.

Common POS Tags

VBDVerb, past tenseran, ate, processed

5.3 Named Entity Recognition Fundamentals

NER identifies and classifies named entities (people, organizations, locations, dates, etc.) in text. Accurate NER is crucial for information extraction, question answering, and knowledge base construction.

Common Entity Types

  • PERSON: People, characters (e.g., "Albert Einstein", "Tim Cook")
  • ORGANIZATION: Companies, agencies (e.g., "Apple Inc.", "United Nations")
  • LOCATION: Countries, cities, landmarks (e.g., "Paris", "Pacific Ocean")
  • DATE: Dates and times (e.g., "January 2025", "Monday")
  • MONEY: Monetary values (e.g., "$100", "€50")
  • PRODUCT: Products and services (e.g., "iPhone", "Windows 11")
  • EVENT: Named events (e.g., "Olympics", "World War II")

5.4 BIO Tagging Scheme

The BIO (Begin, Inside, Outside) tagging scheme represents entity boundaries in sequence labeling. This scheme handles multi-word entities and prevents ambiguity in entity spans. 

# BIO Tagging Example
sentence = "Apple CEO Tim Cook visited New York"

# Word       Tag
# Apple      B-ORG
# CEO        O
# Tim        B-PER
# Cook       I-PER
# visited    O
# New        B-LOC
# York       I-LOC

# B- : Beginning of entity
# I- : Inside entity (continuation)
# O  : Outside any entity

Alternative Tagging Schemes

  • IOB2: Variant of BIO with stricter rules
  • BILOU: Begin, Inside, Last, Outside, Unit (single-word entities)
  • IOBES: Inside, Outside, Begin, End, Single

5.5 Traditional NER Approaches

Before deep learning, NER relied on rule-based systems, gazetteers, and statistical models like Hidden Markov Models (HMM) and Conditional Random Fields (CRF).

Conditional Random Fields (CRF)

CRFs model sequential dependencies between labels, making them effective for structured prediction tasks. They consider features like word shape, capitalization, prefixes, suffixes, and POS tags. 

from sklearn_crfsuite import CRF
from sklearn_crfsuite.metrics import flat_classification_report

def word_features(sentence, i):
    """Extract features for word at position i"""
    word = sentence[i]

    features = {
        'word.lower': word.lower(),
        'word.isupper': word.isupper(),
        'word.istitle': word.istitle(),
        'word.isdigit': word.isdigit(),
        'word.prefix-2': word[:2],
        'word.prefix-3': word[:3],
        'word.suffix-2': word[-2:],
        'word.suffix-3': word[-3:],
    }

    if i > 0:
        prev_word = sentence[i-1]
        features.update({
            'prev_word.lower': prev_word.lower(),
            'prev_word.istitle': prev_word.istitle(),
        })

    if i < len(sentence) - 1:
        next_word = sentence[i+1]
        features.update({
            'next_word.lower': next_word.lower(),
            'next_word.istitle': next_word.istitle(),
        })

    return features

# Train CRF model

5.6 Deep Learning for NER

Deep learning transformed NER through bidirectional LSTMs with CRF layers (BiLSTM-CRF) and more recently, transformer-based models that achieve state-of-the-art results.

BiLSTM-CRF Architecture

import torch
import torch.nn as nn

class BiLSTM_CRF(nn.Module):
    def __init__(self, vocab_size, tag_size, embedding_dim, hidden_dim):
        super().__init__()
        self.embedding = nn.Embedding(vocab_size, embedding_dim)
        self.lstm = nn.LSTM(embedding_dim, hidden_dim // 2,
                           num_layers=1, bidirectional=True, batch_first=True)
        self.hidden2tag = nn.Linear(hidden_dim, tag_size)
        self.crf = CRF(tag_size, batch_first=True)

    def forward(self, sentence, tags=None, mask=None):
        # Embedding
        embeds = self.embedding(sentence)

        # BiLSTM
        lstm_out, _ = self.lstm(embeds)

        # Projection to tag space
        emissions = self.hidden2tag(lstm_out)

        if tags is not None:
            # Training: compute loss
            loss = -self.crf(emissions, tags, mask=mask)
            return loss
        else:
            # Inference: decode
            return self.crf.decode(emissions, mask=mask)

5.7 Transformer-Based NER

BERT and other transformer models revolutionized NER by providing rich contextual representations. Fine-tuning pre-trained transformers on NER datasets achieves excellent performance with relatively small labeled datasets. 

from transformers import BertForTokenClassification, BertTokenizer
import torch

# Load pre-trained BERT for token classification
model = BertForTokenClassification.from_pretrained('bert-base-cased', num_labels=9)
tokenizer = BertTokenizer.from_pretrained('bert-base-cased')

# Example sentence
text = "Apple CEO Tim Cook announced new products in Cupertino"
inputs = tokenizer(text, return_tensors="pt", padding=True, truncation=True,
                  return_offsets_mapping=True)

# Get predictions
with torch.no_grad():
    outputs = model(**{k: v for k, v in inputs.items() if k != 'offset_mapping'})
    predictions = torch.argmax(outputs.logits, dim=2)

# Map predictions to labels
label_list = ['O', 'B-PER', 'I-PER', 'B-ORG', 'I-ORG', 'B-LOC', 'I-LOC', 'B-MISC', 'I-MISC']
tokens = tokenizer.convert_ids_to_tokens(inputs['input_ids'][0])

print("NER Results:")
for token, pred in zip(tokens, predictions[0]):
    if token not in ['[CLS]', '[SEP]', '[PAD]']:
        print(f"{token:15} -> {label_list[pred]}")

5.8 Evaluation Metrics for NER

Evaluating NER systems requires metrics that consider exact entity boundary matching, not just individual token classification accuracy.

Key Metrics

Metric Description Formula
Precision Correct predictions / Total predictions TP / (TP + FP)
Recall Correct predictions / Total ground truth TP / (TP + FN)
F1 Score Harmonic mean of precision and recall 2 * (P * R) / (P + R) 
from seqeval.metrics import classification_report, f1_score

y_true = [['O', 'O', 'B-PER', 'I-PER', 'O', 'B-LOC']]
y_pred = [['O', 'O', 'B-PER', 'I-PER', 'O', 'O']]

# Compute metrics (exact entity match)
print(classification_report(y_true, y_pred))

f1 = f1_score(y_true, y_pred)
print(f"F1 Score: {f1:.4f}")

5.9 Domain-Specific NER

General NER models work well for common entities, but specialized domains (medical, legal, scientific) require domain-specific models trained on annotated data from those domains.

Biomedical NER Example

Medical texts contain specialized entities like diseases, drugs, proteins, and genes. Models like BioBERT, trained on PubMed abstracts, excel at biomedical NER.

  • DISEASE: "diabetes", "hypertension", "COVID-19"
  • DRUG: "aspirin", "metformin", "ibuprofen"
  • GENE: "BRCA1", "TP53", "EGFR"
  • PROTEIN: "hemoglobin", "insulin", "antibody"
