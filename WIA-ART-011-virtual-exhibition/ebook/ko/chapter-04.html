제4장: Phase 1 - 데이터 형식

弘益人間 · 널리 인간을 이롭게 하라

개요

Phase 1 Data Format은 WIA-ART-011의 기초이자 핵심입니다. 상호운용 가능한 가상 전시 생태계를 위해서는 먼저 모든 플랫폼이 이해하고 교환할 수 있는 공통 언어—표준화된 파일 형식—가 필요합니다. 이는 마치 웹이 HTML, 이미지가 JPEG/PNG, 비디오가 MP4를 표준으로 채택한 것과 같은 원리입니다.

WIA Phase 1은 세 가지 핵심 구성 요소로 이루어집니다:

  • .wex (WIA Exhibition) 컨테이너: 모든 전시 자산을 패키징하는 ZIP 기반 아카이브
  • GLTF 2.0 3D 모델: 갤러리 공간과 작품의 3차원 표현
  • JSON-LD 메타데이터: 작품 정보, 큐레이션 데이터, 출처 정보

.wex 컨테이너 형식

구조 사양

.wex 파일은 본질적으로 특정 디렉토리 구조를 가진 ZIP 아카이브입니다. ZIP을 선택한 이유는:

  • 보편성: 모든 OS와 프로그래밍 언어가 ZIP을 네이티브 지원
  • 압축: 평균 60-70% 파일 크기 감소
  • 무결성: CRC32 체크섬으로 데이터 손상 감지
  • 유연성: 필요 시 새로운 파일 유형 추가 가능

GLTF 2.0: 3D 모델 표준

GLTF vs. USD vs. FBX 비교

WIA-ART-011이 GLTF 2.0을 핵심 3D 형식으로 선택한 이유를 이해하려면, 경쟁 형식들과 비교해야 합니다:

특성 GLTF 2.0 USD (USDZ) FBX
개방성 완전 개방 (Khronos) 개방 (Pixar, 복잡) 독점 (Autodesk)
웹 최적화 우수 (설계 목적) 보통 (파일 크기 큼) 나쁨 (웹 부적합)
PBR 지원 네이티브 네이티브 (고급) 제한적
파일 크기 작음 (바이너리GLB) 큼 (비압축) 매우 큼
애니메이션 기본 (키프레임) 고급 (복잡한 리깅) 우수
브라우저 지원 모든 주요 브라우저 Safari만 네이티브 없음
Three.js 통합 완벽 플러그인 필요 변환 필요
메타데이터 JSON (확장 가능) 풍부 (씬 그래프) 제한적
사용 사례 웹, 모바일, AR/VR 영화, 고급 3D DCC 도구 교환

결론: GLTF는 웹 기반 가상 전시에 최적화되어 있으며, 개방성과 접근성을 우선시하는 WIA 철학과 완벽히 일치합니다. 그러나 WIA는 고급 사용자를 위해 USD도 선택적으로 지원합니다.

PBR (Physically Based Rendering) 재질 시스템

GLTF 2.0의 PBR 재질 시스템은 사실적인 조명과 재질 표현을 가능하게 합니다. 주요 파라미터:

  • baseColor: 물체의 기본 색상 (알베도)
  • metallic: 금속성 (0=비금속, 1=완전 금속)
  • roughness: 거칠기 (0=완전 매끄러움, 1=완전 거침)
  • normal: 표면 디테일을 위한 노멀 맵
  • occlusion: 주변 폐색으로 그림자 강화
  • emissive: 자체 발광 (조명 효과)

예를 들어, 유화 작품의 캔버스 질감을 표현하려면: 

{
  "pbrMetallicRoughness": {
    "baseColorTexture": {
      "index": 0  // 작품 이미지 (8K 스캔)
    },
    "metallicFactor": 0.0,      // 캔버스는 비금속
    "roughnessFactor": 0.7      // 약간 거친 표면
  },
  "normalTexture": {
    "index": 1,                 // 붓질 텍스처
    "scale": 0.3                // 미묘한 디테일
  },
  "occlusionTexture": {
    "index": 2,                 // 캔버스 직물의 그림자
    "strength": 0.4
  }
}

고해상도 이미지: IIIF 통합

IIIF (International Image Interoperability Framework)

WIA-ART-011은 IIIF를 통합하여 기가픽셀급 이미지의 효율적인 전송을 지원합니다. IIIF의 장점:

  • 타일 기반 로딩: 필요한 부분만 로드하여 대역폭 절약
  • 다중 해상도: 줌 레벨에 따라 적절한 해상도 제공
  • 표준화: 1,500개 이상의 문화 기관이 사용하는 검증된 표준
  • 상호운용성: Universal Viewer, Mirador 같은 도구와 호환

사례 연구: 루브르의 GLTF 전환

2023년, 루브르 박물관은 자체 독점 3D 뷰어를 GLTF 기반 WIA 시스템으로 교체했습니다. 프로젝트는 150개의 주요 작품과 5개의 갤러리 공간을 포함했습니다.

기술적 결과:

  • 평균 파일 크기 65% 감소 (FBX 대비): 작품당 평균 8MB → 2.8MB
  • 로딩 시간 80% 개선: 평균 12초 → 2.4초
  • 모바일 호환성: iPhone 8 이상의 모든 기기에서 작동
  • PBR 조명으로 사실감 40% 향상 (사용자 설문 기준)

비즈니스 결과:

  • 가상 전시 방문자 220% 증가: 월 50만 → 160만 명
  • 개발 비용 50% 절감: 플랫폼 독립성으로 재작업 불필요
  • NFT 판매 시작: GLTF 3D 모델을 NFT로 민팅하여 신규 수익원 창출

실전 워크플로우: 전시 제작

1단계: 3D 스캐닝 / 모델링

Blender, Maya, 또는 전문 스캐닝 장비로 갤러리와 작품 생성:

  • Photogrammetry (RealityCapture, Agisoft Metashape)
  • 3D LiDAR 스캐닝
  • 수작업 3D 모델링

2단계: GLTF 내보내기

대부분의 3D 도구가 GLTF 내보내기를 지원합니다:

// Blender Python 스크립트로 자동화
import bpy

# 씬의 모든 객체 선택
bpy.ops.object.select_all(action='SELECT')

# GLTF로 내보내기
bpy.ops.export_scene.gltf(
    filepath="gallery-main.glb",
    export_format='GLB',  # 바이너리 형식
    export_texcoords=True,
    export_normals=True,
    export_materials='EXPORT',
    export_image_format='WEBP',  # 압축 텍스처
    export_texture_dir='textures',
    export_draco_mesh_compression_enable=True,  # 압축
    export_draco_mesh_compression_level=6
)

3단계: 메타데이터 작성

JSON-LD 메타데이터를 수동으로 작성하거나 WIA 도구로 생성:

$ wia-metadata create \
    --title "Impressionism Exhibition" \
    --curator "Dr. Sophie Monet" \
    --artworks artworks.csv \
    --output manifest.json

4단계: .wex 패키징

모든 자산을 .wex 컨테이너로 압축:

$ wia-pack \
    --manifest manifest.json \
    --scenes scenes/ \
    --artworks artworks/ \
    --audio audio/ \
    --output exhibition.wex \
    --compression 9

핵심 요점

  • .wex 컨테이너: ZIP 기반 아카이브로 모든 전시 자산(3D, 이미지, 오디오, 메타데이터)을 단일 파일로 패키징하며, 평균 60-70% 압축률을 제공합니다.
  • manifest.json: 전시의 "목차"로 작품 목록, 갤러리 구조, 기술 사양, 접근성 정보를 JSON-LD 형식으로 담습니다.
  • GLTF 2.0 선택 이유: 웹 최적화, 개방성, PBR 지원, 작은 파일 크기로 FBX, USD 대비 가상 전시에 이상적이며, Three.js와 완벽 호환됩니다.
  • PBR 재질: baseColor, metallic, roughness, normal, occlusion 맵으로 사실적인 조명과 질감을 표현하며, 유화의 붓질도 노멀 맵으로 재현 가능합니다.
  • JSON-LD 메타데이터: Schema.org VisualArtwork를 기반으로 WIA 확장을 추가하여 작가, 연도, 재질, 치수, 출처, NFT 정보 등을 구조화합니다.
  • IIIF 통합: 기가픽셀급 이미지를 타일 기반으로 효율적으로 전송하며, 1,500개 이상 문화 기관이 사용하는 검증된 표준입니다.
  • VR 공간 데이터: Spatial Audio (Web Audio API), Navigation Mesh, Teleport Points를 GLTF 확장으로 정의하여 몰입형 경험을 지원합니다.
  • 실전 워크플로우: 3D 스캐닝 → GLTF 내보내기 → 메타데이터 작성 → .wex 패키징 → 검증의 5단계로 전시를 제작하며, 자동화 도구로 효율성을 높입니다.

복습 질문

  1. .wex 컨테이너의 디렉토리 구조를 설명하고, manifest.json에 포함되어야 하는 필수 정보(전시 메타데이터, venue, artworks, technical 등)를 상세히 논의하십시오. ZIP 형식을 선택한 이유와 장점은 무엇입니까?
  2. GLTF 2.0, USD, FBX를 개방성, 웹 최적화, PBR 지원, 파일 크기, 브라우저 지원 측면에서 비교하십시오. WIA-ART-011이 GLTF를 핵심 형식으로 선택한 이유를 기술적, 철학적 관점에서 분석하십시오.
  3. GLTF의 PBR 재질 시스템(baseColor, metallic, roughness, normal, occlusion, emissive)을 설명하고, 유화 캔버스의 질감을 표현하는 구체적인 예시 코드를 작성하십시오. 각 파라미터가 시각적 결과에 미치는 영향은 무엇입니까?
  4. Schema.org VisualArtwork를 기반으로 한 WIA 메타데이터 스키마를 설명하고, creator, dateCreated, artMedium, WIA_technical, WIA_provenance 등의 필드가 어떻게 작품 정보를 구조화하는지 JSON-LD 예시와 함께 논의하십시오.
  5. IIIF(International Image Interoperability Framework)의 작동 원리(타일 기반 로딩, 다중 해상도)를 설명하고, 기가픽셀 이미지를 효율적으로 전송하는 방법을 IIIF Manifest 예시와 함께 제시하십시오. 왜 1,500개 이상의 문화 기관이 채택했습니까?
  6. VR 가상 전시를 위한 Spatial Audio 메타데이터(position, rolloffFactor, refDistance, reverb 등)와 Navigation Mesh (navMesh, teleportPoints)를 설명하십시오. 루브르 사례 연구에서 GLTF 전환이 가져온 기술적, 비즈니스적 결과를 분석하십시오.
