제6장: Phase 3 - 프로토콜

弘益人間 · 널리 인간을 이롭게 하라

개요

Phase 3 Protocol은 WIA-ART-011의 "사회적 차원"입니다. 데이터 형식(Phase 1)과 API(Phase 2)가 단일 사용자 경험을 다룬다면, Protocol은 다중 사용자 실시간 협업을 가능하게 합니다. 메타버스 가상 전시의 핵심은 여러 관람객이 동시에 같은 공간에서 만나고, 대화하며, 예술을 함께 감상하는 것입니다.

WIA Phase 3는 네 가지 핵심 프로토콜을 정의합니다:

  • WebSocket 실시간 동기화: 아바타 위치, 상태, 채팅 메시지
  • WebRTC P2P: 음성/비디오 채팅, 저지연 데이터 전송
  • Adaptive Streaming: 네트워크 상태 기반 품질 조정
  • Spatial Audio: 3D 위치 기반 오디오

WebRTC P2P 통신

음성 채팅 연결

저지연 음성 통신을 위해 WIA는 WebRTC를 사용합니다: 

// Signaling (WebSocket을 통해)
// 1. Offer 생성
const pc = new RTCPeerConnection(configuration);
const offer = await pc.createOffer({
  offerToReceiveAudio: true,
  offerToReceiveVideo: false
});

await pc.setLocalDescription(offer);

// WebSocket으로 전송
ws.send({
  type: "webrtc.offer",
  from: "user-123",
  to: "user-456",
  data: {
    sdp: offer.sdp,
    type: offer.type
  }
});

// 2. Answer 수신
ws.on('message', async (msg) => {
  if (msg.type === 'webrtc.answer') {
    await pc.setRemoteDescription(new RTCSessionDescription(msg.data));
  }
  if (msg.type === 'webrtc.ice-candidate') {
    await pc.addIceCandidate(new RTCIceCandidate(msg.data));
  }
});

// 3. 오디오 스트림 추가
const stream = await navigator.mediaDevices.getUserMedia({
  audio: {
    echoCancellation: true,
    noiseSuppression: true,
    autoGainControl: true
  }
});

stream.getTracks().forEach(track => pc.addTrack(track, stream));

// 4. 원격 스트림 수신
pc.ontrack = (event) => {
  const remoteAudio = new Audio();
  remoteAudio.srcObject = event.streams[0];
  remoteAudio.play();
  
  // Spatial Audio 적용 (Web Audio API)
  applySpatialAudio(remoteAudio, user456Position);
};

Mesh vs. SFU 토폴로지

특성 Mesh (P2P) SFU (Server)
최대 참가자 ~10명 수백 명
지연시간 매우 낮음 (50-100ms) 낮음 (100-200ms)
대역폭 사용 N*(N-1) 연결 (높음) N 연결 (낮음)
서버 비용 낮음 높음
사용 사례 소규모 그룹 투어 대형 이벤트, 강연

WIA는 참가자 수에 따라 자동으로 토폴로지를 전환합니다:

  • ≤ 8명: Full Mesh (모두가 모두와 직접 연결)
  • 9-30명: Partial Mesh (근처 사용자만 P2P, 나머지는 서버)
  • > 30명: 완전 SFU 모드

Adaptive Streaming

네트워크 감지 및 품질 조정

// Network Information API 활용
const connection = navigator.connection || navigator.mozConnection || navigator.webkitConnection;

function getQualityLevel() {
  const bandwidth = connection.downlink;  // Mbps
  const rtt = connection.rtt;            // ms
  const effectiveType = connection.effectiveType;  // '4g', '3g', '2g', 'slow-2g'
  
  if (bandwidth > 10 && rtt < 100) {
    return 'ultra';  // 8K 텍스처, 높은 폴리곤
  } else if (bandwidth > 5 && rtt < 200) {
    return 'high';   // 4K 텍스처, 중간 폴리곤
  } else if (bandwidth > 2) {
    return 'medium'; // 2K 텍스처, 낮은 폴리곤
  } else {
    return 'low';    // 1K 텍스처, 최소 폴리곤
  }
}

// 품질 변경 시 점진적 전환
gallery.on('quality.change', (from, to) => {
  console.log(`Quality: ${from} → ${to}`);
  // 부드러운 LOD 전환 (crossfade)
});

// 사용자 수동 오버라이드 허용
gallery.setQuality('high', { userPreference: true });

Bandwidth Estimation

// WebRTC Stats를 활용한 실시간 대역폭 측정
async function estimateBandwidth(peerConnection) {
  const stats = await peerConnection.getStats();
  let bytesReceived = 0;
  let timestamp = 0;
  
  stats.forEach(report => {
    if (report.type === 'inbound-rtp' && report.mediaType === 'video') {
      bytesReceived = report.bytesReceived;
      timestamp = report.timestamp;
    }
  });
  
  // 이전 측정과 비교하여 Mbps 계산
  const bandwidth = calculateBandwidth(bytesReceived, timestamp);
  
  if (bandwidth < 2) {
    gallery.setQuality('low');
  } else if (bandwidth < 5) {
    gallery.setQuality('medium');
  }
}

// 5초마다 측정

audioContext.listener.positionY.value = myPosition.y;

// 음원 위치 (다른 사용자 또는 작품 오디오 가이드)

panner.positionZ.value = sourcePosition.z;

panner.coneOuterGain = 0.3;   // 외부 음량

// 오디오 연결

사례 연구: MoMA의 500명 동시 접속 VR 강연

2023년 5월, 뉴욕 현대미술관(MoMA)는 WIA Phase 3 프로토콜을 사용하여 "현대 미술의 미래" VR 강연을 개최했습니다. 전 세계에서 500명이 동시 접속하여 큐레이터의 실시간 가이드를 들었습니다.

기술 아키텍처:

  • WebSocket 서버: AWS ELB 뒤의 10개 인스턴스, Redis Pub/Sub로 메시지 라우팅
  • WebRTC SFU: Mediasoup 기반, 3개 리전(미국, 유럽, 아시아)에 배포
  • CDN: Cloudflare for 3D 자산, 엣지 캐싱으로 로딩 시간 80% 감소
  • Adaptive Streaming: 5단계 품질(ultra/high/medium/low/minimum)

성능 결과:

  • 평균 WebSocket 지연시간: 45ms (서버 간 Redis 통신 포함)
  • WebRTC 음성 지연: 120ms (글로벌 평균)
  • 대역폭 사용: 사용자당 평균 2.5Mbps (adaptive로 1-8Mbps 범위)
  • 동시 접속 500명, 피크 메시지 처리율 15,000 msg/sec
  • 99%ile 응답 시간: 200ms 미만

사용자 경험:

  • 참가자 만족도: 4.7/5.0
  • 음성 품질 평가: 4.5/5.0 (약간의 에코 보고)
  • 88%가 "물리적 강연만큼 몰입적"이라고 응답
  • 평균 체류 시간: 52분 (계획된 45분 강연 초과)

교훈:

  • 지역별 SFU 배치가 글로벌 지연시간 감소에 결정적
  • Adaptive quality가 모바일 사용자(35%) 참여 가능하게 함
  • Spatial audio가 혼란 감소 및 몰입감 증가
  • 텍스트 채팅(15,000 메시지)이 음성 보완 및 접근성 향상

Rate Limiting & Anti-Spam

// 메시지 타입별 rate limit
const limits = {
  'position.update': 20,   // 20 msg/sec
  'chat.message': 1,       // 1 msg/sec
  'action.emote': 0.2      // 1 msg/5sec
};

// 서버 측 토큰 버킷 알고리즘
class RateLimiter {
  constructor(rate, capacity) {
    this.tokens = capacity;
    this.rate = rate;
    this.lastRefill = Date.now();
  }
  
  tryConsume() {
    this.refill();
    if (this.tokens >= 1) {
      this.tokens -= 1;
      return true;
    }
    return false;
  }
  
  refill() {
    const now = Date.now();
    const elapsed = (now - this.lastRefill) / 1000;
    this.tokens = Math.min(
      this.capacity,
      this.tokens + elapsed * this.rate
    );
    this.lastRefill = now;
  }
}

핵심 요점

  • WebSocket 프로토콜: Presence, Position, Chat, Actions 메시지 타입으로 실시간 동기화하며, JSON 기반 표준 형식을 사용합니다.
  • WebRTC P2P: 저지연 음성 채팅을 위해 Mesh (≤8명)와 SFU (>30명) 토폴로지를 참가자 수에 따라 자동 전환합니다.
  • Adaptive Streaming: Network Information API로 대역폭을 측정하고 ultra/high/medium/low 품질로 자동 조정하여 모든 네트워크 환경을 지원합니다.
  • Spatial Audio: Web Audio API Panner Node로 3D 위치 기반 오디오를 구현하며, Convolution Reverb로 갤러리 음향을 시뮬레이션합니다.
  • 확장성: MoMA 사례에서 500명 동시 접속, 15,000 msg/sec 처리, 평균 45ms 지연시간을 달성했습니다.
  • 보안: JWT 인증, 권한 기반 접근 제어, 메시지 타입별 rate limiting, 스팸 감지로 안전한 환경을 보장합니다.
  • 사용자 경험: Spatial audio와 adaptive quality로 88%가 물리적 강연만큼 몰입적이라고 평가했으며, 모바일 사용자 35%도 참여 가능했습니다.
  • 인프라: 지역별 SFU 배치, Redis Pub/Sub, CDN 엣지 캐싱으로 글로벌 성능을 최적화하고 대역폭 사용을 최소화합니다.
