제5장: Phase 2 - API 인터페이스

弘益人間 · 널리 인간을 이롭게 하라

개요

Phase 1이 데이터의 "언어"를 정의했다면, Phase 2 API Interface는 그 언어로 "대화"하는 방법을 정의합니다. 파일 교환만으로는 진정한 상호운용성을 달성할 수 없습니다—개발자가 가상 전시를 프로그래밍 방식으로 생성, 수정, 쿼리, 렌더링할 수 있어야 합니다.

WIA Phase 2는 세 가지 주요 구성 요소를 제공합니다:

  • RESTful API: HTTP 기반의 표준 엔드포인트
  • SDK (Software Development Kits): JavaScript, Python, C# 라이브러리
  • WebXR API 통합: 브라우저 기반 VR/AR 경험

REST API 아키텍처

핵심 엔드포인트

WIA-ART-011 REST API는 리소스 중심 설계를 따르며, 표준 HTTP 메서드를 사용합니다: 

Base URL: https://api.wia.org/art-011/v1

# 전시 관리
GET    /exhibitions                    # 전시 목록
GET    /exhibitions/{id}               # 전시 상세
POST   /exhibitions                    # 전시 생성
PUT    /exhibitions/{id}               # 전시 업데이트
DELETE /exhibitions/{id}               # 전시 삭제

# 작품 관리
GET    /exhibitions/{id}/artworks      # 전시 내 작품 목록
GET    /artworks/{id}                  # 작품 상세
POST   /artworks                       # 작품 추가
PUT    /artworks/{id}                  # 작품 수정
DELETE /artworks/{id}                  # 작품 삭제

# 3D 씬 관리
GET    /exhibitions/{id}/scenes        # 갤러리 씬 목록
POST   /scenes                         # 씬 업로드
GET    /scenes/{id}/gltf               # GLTF 다운로드

# 메타데이터 쿼리
GET    /artworks/search?q={query}      # 작품 검색
GET    /artists/{id}                   # 아티스트 정보
GET    /artworks/{id}/provenance       # 출처 정보

# 렌더링 및 미리보기
GET    /exhibitions/{id}/preview       # 미리보기 이미지
GET    /artworks/{id}/thumbnail        # 썸네일
POST   /exhibitions/{id}/render        # 렌더링 요청

인증: OAuth 2.0

API 접근은 OAuth 2.0 프로토콜로 보호됩니다. 두 가지 플로우를 지원합니다: 

# 1. Authorization Code Flow (웹 앱)
# 사용자를 인증 페이지로 리다이렉트
https://auth.wia.org/oauth/authorize?
    response_type=code&
    client_id=YOUR_CLIENT_ID&
    redirect_uri=https://yourapp.com/callback&
    scope=exhibitions:read exhibitions:write

# 콜백에서 인증 코드 받기
# 코드를 액세스 토큰으로 교환
POST https://auth.wia.org/oauth/token
Content-Type: application/json

{
  "grant_type": "authorization_code",
  "code": "AUTHORIZATION_CODE",
  "client_id": "YOUR_CLIENT_ID",
  "client_secret": "YOUR_CLIENT_SECRET",
  "redirect_uri": "https://yourapp.com/callback"
}

# 응답
{
  "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "token_type": "Bearer",
  "expires_in": 3600,
  "refresh_token": "tGzv3JOkF0XG5Qx2TlKWIA",
  "scope": "exhibitions:read exhibitions:write"
}

# 2. Client Credentials Flow (서버 간 통신)
POST https://auth.wia.org/oauth/token
Content-Type: application/json

{
  "grant_type": "client_credentials",
  "client_id": "YOUR_CLIENT_ID",
  "client_secret": "YOUR_CLIENT_SECRET",
  "scope": "exhibitions:read"
}

JavaScript SDK

설치 및 초기화

# npm으로 설치
npm install @wia/art-011

# 또는 yarn
yarn add @wia/art-011

# 또는 CDN (브라우저)
<script src="https://cdn.wia.org/sdk/art-011/v1/wia-art-011.min.js"></script>

// ES6 모듈
import { WIAClient, WIAGallery } from '@wia/art-011';

// 클라이언트 초기화
const client = new WIAClient({
  apiKey: 'YOUR_API_KEY',
  // 또는 OAuth 토큰
  accessToken: 'YOUR_ACCESS_TOKEN'
});

// 이벤트 리스너

WebXR API 통합

브라우저 기반 VR

WIA SDK는 W3C WebXR Device API와 완벽히 통합되어 VR 헤드셋 없이도 브라우저에서 VR 전시를 경험할 수 있습니다: 

// WebXR 지원 확인
if (navigator.xr) {
  const vr supported = await navigator.xr.isSessionSupported('immersive-vr');
  const arSupported = await navigator.xr.isSessionSupported('immersive-ar');
  
  console.log(`VR: ${vrSupported}, AR: ${arSupported}`);
}

// VR 세션 시작
const gallery = new WIAGallery({
  container: '#viewer',
  xr: {
    mode: 'immersive-vr',
    requiredFeatures: ['local-floor'],
    optionalFeatures: ['hand-tracking', 'eye-tracking']
  }
});

await gallery.present(exhibition);

// VR 모드 진입
document.getElementById('enter-vr').addEventListener('click', async () => {
  await gallery.enterVR();
});

// AR 모드 (모바일)
document.getElementById('enter-ar').addEventListener('click', async () => {
  await gallery.enterAR({
    hitTest: true,  // 바닥 감지
    anchors: true   // 공간 앵커
  });
});

사례 연구: 테이트 모던의 SDK 통합

2024년, 테이트 모던은 WIA JavaScript SDK를 자체 웹사이트에 통합하여 100개 이상의 현대 미술 작품을 가상 전시로 제공했습니다. 프로젝트는 두 명의 프론트엔드 개발자가 3주 만에 완료했습니다.

기술 스택:

  • React 18 + TypeScript
  • @wia/art-011 SDK
  • Three.js (SDK 내부 사용)
  • WebXR Device API

구현 결과:

  • 개발 시간 80% 단축: 예상 12주 → 3주 (SDK 덕분)
  • 코드 라인 90% 감소: 순수 Three.js 대비 10,000줄 → 1,000줄
  • VR 지원 추가에 단 2일 소요: WebXR 통합이 SDK에 내장
  • 모바일 성능 우수: iPhone 12에서 60fps 유지

사용자 반응:

  • 가상 전시 방문자 월 8,000명 → 120,000명 (1400% 증가)
  • 평균 체류 시간 12분 (물리적 전시는 8분)
  • 모바일 사용자 65% (SDK의 반응형 디자인)

// SDK 자동 캐싱

Rate Limiting

WIA API는 공정한 사용을 보장하기 위해 rate limiting을 적용합니다:

  • 무료 티어: 1,000 요청/시간, 10,000 요청/일
  • Bronze: 5,000 요청/시간, 50,000 요청/일
  • Silver: 20,000 요청/시간, 200,000 요청/일
  • Gold: 무제한 (공정 사용 정책 적용)
// Rate limit 정보는 응답 헤더에 포함
X-RateLimit-Limit: 5000
X-RateLimit-Remaining: 4823
X-RateLimit-Reset: 1640995200

// SDK는 자동으로 rate limit을 처리
const client = new WIAClient({
  apiKey: 'YOUR_API_KEY',
  rateLimit: {
    autoRetry: true,  // 429 응답 시 자동 재시도
    maxRetries: 3,
    backoff: 'exponential'  // 지수 백오프
  }
});

핵심 요점

  • RESTful API: 리소스 중심 설계로 exhibitions, artworks, scenes 등을 HTTP 메서드로 조작하며, OAuth 2.0으로 보안을 보장합니다.
  • JavaScript SDK: @wia/art-011 패키지로 전시 로드, 렌더링, 실시간 편집을 간단한 API로 제공하며, 테이트 모던은 3주 만에 통합했습니다.
  • Python SDK: 데이터 분석과 배치 작업에 최적화되어 pandas 통합, 대량 업로드, 통계 분석을 지원합니다.
  • GraphQL API: 복잡한 쿼리와 효율적인 데이터 페칭을 위해 선택적으로 제공하며, 중첩된 리소스를 한 번의 요청으로 가져올 수 있습니다.
  • WebXR 통합: W3C WebXR Device API와 완벽 호환되어 브라우저에서 VR/AR 경험을 제공하며, 핸드 트래킹과 시선 추적도 지원합니다.
  • 성능 최적화: 자동 캐싱, Progressive Loading, Rate Limiting으로 효율성과 공정성을 보장하며, 모바일에서도 60fps를 유지합니다.
  • 개발자 경험: TypeScript 타입 정의, 상세한 에러 메시지, 풍부한 이벤트 시스템으로 개발 시간을 80% 단축하고 코드 라인을 90% 감소시킵니다.
  • 다중 언어 지원: JavaScript, Python, C# SDK를 제공하며, REST API는 모든 언어에서 사용 가능하여 생태계 확장을 촉진합니다.
