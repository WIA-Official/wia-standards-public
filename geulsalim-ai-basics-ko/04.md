## 이론

### 생태계를 지도로 이해하라

처음 AI 개발을 시작하면 라이브러리 이름들이 뒤엉켜 혼란스럽다. `torch`, `transformers`, `diffusers`, `accelerate`, `peft`... 이것들이 서로 어떻게 다른지, 왜 다 설치해야 하는지 모른 채로 코드를 복붙하다 보면 오류가 나도 어디를 봐야 할지 모른다.

지도가 필요하다. 전체 구조를 먼저 보면 개별 코드가 자연스럽게 이해된다.

### 계층 구조

```
┌─────────────────────────────────────────┐
│         내 학습 코드 (train.py)          │  ← 내가 작성
├─────────────────────────────────────────┤
│  peft     │  accelerate  │  wandb       │  ← 파인튜닝 도우미
├───────────┴──────────────┴──────────────┤
│  transformers  │  diffusers  │ datasets │  ← Hugging Face 핵심
├─────────────────────────────────────────┤
│              PyTorch (torch)             │  ← 기반 엔진
├─────────────────────────────────────────┤
│         CUDA (GPU 드라이버)              │  ← 하드웨어
└─────────────────────────────────────────┘
```

**PyTorch (`torch`)**: GPU 연산의 기반. 텐서 계산, 자동 미분, 신경망 레이어가 모두 여기서 나온다. "엔진"이라고 생각하면 된다.

**Transformers (`transformers`)**: BERT, GPT, LLaMA 같은 언어 모델을 불러오고 학습시키는 라이브러리. `AutoModelForCausalLM.from_pretrained("meta-llama/Llama-3.1-8B")`처럼 모델명만 써도 다운로드된다.

**Diffusers (`diffusers`)**: Stable Diffusion, SDXL, Flux 같은 이미지 생성 모델 전용. 파이프라인 개념으로 동작한다. `StableDiffusionXLPipeline.from_pretrained(...)`.

**Datasets (`datasets`)**: Hugging Face Hub의 수천 개 데이터셋을 한 줄로 불러온다. 캐시, 스트리밍, 전처리까지 통합 처리.

**PEFT (`peft`)**: "Parameter-Efficient Fine-Tuning." LoRA, QLoRA 같은 파인튜닝 기법의 구현체. 이 라이브러리 없이는 VRAM 24GB 안에서 파인튜닝이 불가능하다.

**Accelerate (`accelerate`)**: 단일 GPU → 멀티 GPU → TPU 전환을 코드 수정 없이 처리. `accelerate launch train.py`만으로 확장된다.

### Hugging Face Hub

`huggingface.co`는 AI 모델의 GitHub이다. 2024년 기준 70만 개 이상의 모델이 올라와 있다. 검색하고, 다운로드하고, 내 모델을 업로드할 수 있다.

모델 카드(Model Card)에는 반드시 확인해야 할 세 가지가 있다.
1. **라이선스**: Apache 2.0은 상업적 사용 가능. CC BY-NC는 비상업 전용.
2. **기반 모델**: 어떤 모델에서 파인튜닝됐는지.
3. **학습 데이터**: 어떤 데이터로 학습됐는지. 저작권 이슈 확인.

## 실전 코드

```python
# Hugging Face 생태계 핵심 사용법 모음
# 이 파일 하나로 기본 패턴 전부 익힌다

from transformers import AutoTokenizer, AutoModelForCausalLM
from diffusers import StableDiffusionXLPipeline
from datasets import load_dataset
from peft import LoraConfig, get_peft_model
import torch

# ── 1. 언어 모델 불러오기 ──────────────────────────────
def load_language_model(model_name: str = "meta-llama/Llama-3.1-8B"):
    """
    Hugging Face Hub에서 언어 모델을 불러온다.
    처음 실행 시 자동으로 다운로드 (캐시 위치: ~/.cache/huggingface/)
    """
    print(f"모델 로딩: {model_name}")
    
    # 토크나이저: 텍스트 → 숫자(토큰) 변환기
    tokenizer = AutoTokenizer.from_pretrained(model_name)
    
    # 모델: FP16으로 로드해 VRAM 절약
    model = AutoModelForCausalLM.from_pretrained(
        model_name,
        torch_dtype=torch.float16,  # FP32 대비 VRAM 절반
        device_map="auto"           # GPU 자동 배치
    )
    
    print(f"파라미터 수: {model.num_parameters() / 1e9:.1f}B")
    return tokenizer, model


# ── 2. 이미지 생성 파이프라인 ─────────────────────────
def load_sdxl():
    """SDXL 파이프라인 로드. 처음 실행 시 약 7GB 다운로드."""
    pipe = StableDiffusionXLPipeline.from_pretrained(
        "stabilityai/stable-diffusion-xl-base-1.0",
        torch_dtype=torch.float16,
        variant="fp16",
        use_safetensors=True
    ).to("cuda")
    
    # xformers로 메모리 최적화 (설치된 경우)
    try:
        pipe.enable_xformers_memory_efficient_attention()
        print("xformers 메모리 최적화 활성화")
    except Exception:
        pass
    
    return pipe


# ── 3. 데이터셋 불러오기 ──────────────────────────────
def explore_dataset(dataset_name: str = "laion/laion2B-en"):
    """
    Hugging Face Hub 데이터셋 탐색.
    스트리밍 모드: 전체를 다운로드하지 않고 필요한 만큼만 읽는다.
    """
    # 스트리밍으로 처음 5개만 확인
    dataset = load_dataset(dataset_name, split="train", streaming=True)
    
    print(f"데이터셋: {dataset_name}")
    for i, sample in enumerate(dataset):
        print(f"\n샘플 {i+1}:")
        for key, value in sample.items():
            v_str = str(value)[:100] + "..." if len(str(value)) > 100 else str(value)
            print(f"  {key}: {v_str}")
        if i >= 4:
            break


# ── 4. LoRA 설정 (파인튜닝 준비) ─────────────────────
def apply_lora(model, rank: int = 4):
    """
    모델에 LoRA 어댑터를 추가한다.
    rank가 작을수록 VRAM 적게 사용, 학습 속도 빠름 (품질 약간 낮음)
    """
    config = LoraConfig(
        r=rank,                      # LoRA 랭크 (4, 8, 16, 32, 64)
        lora_alpha=rank * 2,         # 스케일링 계수 (보통 rank의 2배)
        target_modules=["q_proj", "v_proj"],  # 어텐션 레이어에만 적용
        lora_dropout=0.05,           # 과적합 방지
        bias="none",
        task_type="CAUSAL_LM"
    )
    
    model = get_peft_model(model, config)
    
    # 학습 가능한 파라미터 수 출력
    trainable = sum(p.numel() for p in model.parameters() if p.requires_grad)
    total = sum(p.numel() for p in model.parameters())
    print(f"학습 파라미터: {trainable:,} / {total:,} ({trainable/total*100:.2f}%)")
    
    return model


# ── 5. Hub에서 모델 검색 ──────────────────────────────
def search_hub():
    """Hugging Face Hub API로 모델 검색"""
    from huggingface_hub import HfApi
    
    api = HfApi()
    
    # SDXL 파인튜닝된 모델 검색
    models = api.list_models(
        filter="stable-diffusion-xl",
        sort="downloads",
        direction=-1,  # 내림차순
        limit=5
    )
    
    print("인기 SDXL 모델 TOP 5:")
    for model in models:
        print(f"  {model.id} — {model.downloads:,} 다운로드")


if __name__ == "__main__":
    # 생태계 탐색 (다운로드 없이)
    search_hub()
    
    # 데이터셋 미리보기 (스트리밍)
    explore_dataset("laion/laion-art")
```

## 함정 카드 — 자주 실수하는 5가지

1. **"캐시 경로를 모른다"** → 다운로드한 모델은 `~/.cache/huggingface/hub/`에 쌓인다. 클라우드 인스턴스에서는 홈 디렉토리 용량이 제한적이다. `HF_HOME=/data/hf_cache` 환경변수로 경로를 데이터 디스크로 변경해야 한다.

2. **"`device_map='auto'`를 안 쓴다"** → 수동으로 `.to('cuda')`를 하면 모델 전체가 GPU 1장에 올라가려고 해서 OOM이 난다. `device_map='auto'`는 여러 GPU, GPU+CPU를 자동으로 분산한다.

3. **"라이선스를 확인 안 한다"** → `meta-llama/Llama-3.1-8B`는 Meta 라이선스 동의가 필요하다. `gated` 표시가 있는 모델은 HF 계정으로 로그인 후 접근 요청을 해야 다운로드된다.

4. **"`streaming=True` 없이 대용량 데이터셋을 로드한다"** → `load_dataset("laion/laion2B-en")`을 스트리밍 없이 실행하면 수백 GB가 다운로드된다. 탐색 목적이라면 반드시 `streaming=True`를 붙인다.

5. **"LoRA `target_modules`를 잘못 지정한다"** → 모델마다 레이어 이름이 다르다. LLaMA는 `q_proj`, GPT-2는 `c_attn`. 모델 구조를 `print(model)`로 확인하고 어텐션 레이어 이름을 직접 찾아야 한다.

## 연습문제

**문제 1.** `transformers`와 `diffusers`의 차이를 한 문장으로 설명하라. 어떤 상황에서 각각을 사용하는가?

> **해설:** `transformers`는 텍스트 처리 모델(LLM, BERT 등) 전용이고, `diffusers`는 이미지 생성/편집 모델(Stable Diffusion, Flux 등) 전용이다. 텍스트 생성이면 `transformers`, 이미지 생성이면 `diffusers`.

**문제 2.** LoRA rank를 4에서 64로 높이면 어떤 변화가 생기는가? 트레이드오프 3가지를 설명하라.

> **해설:** ① VRAM 사용량 증가 (rank 비례) ② 학습 속도 감소 ③ 표현력 증가 (더 복잡한 패턴 학습 가능). 일반적으로 rank 4~16이 가성비 좋다. rank 64는 Full fine-tune에 가까운 품질이지만 VRAM이 많이 필요하다.

**문제 3.** `HF_HOME` 환경변수를 `/data/hf_cache`로 설정하고 `load_sdxl()` 함수를 실행하는 bash 명령을 작성하라.

> **해설:** `HF_HOME=/data/hf_cache python -c "from ch04 import load_sdxl; load_sdxl()"` 또는 `export HF_HOME=/data/hf_cache`를 `.bashrc`에 추가 후 `source ~/.bashrc`.
