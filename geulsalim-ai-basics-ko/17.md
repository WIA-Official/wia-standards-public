# 챕터 17 · FastAPI로 모델 서빙 — 프로덕션 수준 API 구축

## 학습 목표
- FastAPI로 SDXL 추론 API 엔드포인트를 구현하고 비동기 처리를 적용할 수 있다
- 요청 큐 · 타임아웃 · 에러 핸들링을 포함한 프로덕션 수준 서빙 아키텍처를 설계할 수 있다
- uvicorn + gunicorn 조합으로 멀티 워커 API 서버를 배포할 수 있다

---

## 이론

### 왜 FastAPI인가

모델을 학습했다고 서비스가 되는 게 아니다. 고문서 복원 API는 다음을 처리해야 한다:
- 여러 사용자가 동시에 요청
- 이미지 업로드 (~수 MB)
- 처리 시간 ~8초 (긴 I/O)
- 실패 시 적절한 에러 응답

FastAPI가 최적인 이유:
- **비동기 처리**: `async/await`로 I/O 대기 중 다른 요청 처리
- **자동 문서화**: `/docs` 엔드포인트로 Swagger UI 자동 생성
- **타입 검증**: Pydantic 모델로 입력 자동 검증
- **성능**: Starlette 기반으로 Node.js 급 처리량
- **ML 통합**: PyTorch/diffusers와 자연스러운 통합

2025년 기준 ML 서빙 프레임워크 중 **FastAPI + uvicorn**이 가장 광범위하게 사용된다.

### GPU 서빙의 핵심 문제: 동시성

GPU 모델은 **한 번에 하나**의 추론 요청만 처리할 수 있다 (기본 설정).

```
사용자 A 요청 (8초)
사용자 B 요청 → 8초 대기
사용자 C 요청 → 16초 대기
```

해결책:
1. **배치 처리**: 여러 요청을 모아 한 번에 처리 (latency vs throughput 트레이드오프)
2. **큐 시스템**: 요청을 큐에 넣고 순서대로 처리
3. **멀티 GPU**: GPU당 하나의 서버 프로세스

### API 설계 패턴

**동기 방식 (소규모)**:
```
POST /restore → 이미지 업로드 → 처리 → 결과 반환 (8~30초 blocking)
```

**비동기 방식 (프로덕션)**:
```
POST /restore/submit → job_id 즉시 반환
GET  /restore/status/{job_id} → 처리 상태 확인
GET  /restore/result/{job_id} → 완료 시 이미지 반환
```

중간심사 데모 수준에서는 동기 방식으로 시작하고, 사용자가 늘면 비동기로 전환.

## 실전 코드

```python
# 고문서 복원 FastAPI 서버
# pip install fastapi uvicorn[standard] python-multipart Pillow torch

import os
import io
import uuid
import asyncio
import logging
from pathlib import Path
from typing import Optional
from contextlib import asynccontextmanager

import torch
import numpy as np
from PIL import Image
from fastapi import FastAPI, File, UploadFile, HTTPException, BackgroundTasks
from fastapi.responses import Response, JSONResponse
from pydantic import BaseModel, Field

logging.basicConfig(level=logging.INFO)
logger = logging.getLogger(__name__)


# ──────────────────────────────────────────────
# 1. 전역 상태 (모델은 앱 시작 시 한 번만 로드)
# ──────────────────────────────────────────────
class ModelRegistry:
    """GPU 모델 싱글톤 — 매 요청마다 로드하면 메모리 낭비"""
    pipe = None
    device = "cuda" if torch.cuda.is_available() else "cpu"
    job_results: dict = {}   # {job_id: {"status": ..., "result": bytes}}


registry = ModelRegistry()


def load_model():
    """앱 시작 시 1회 모델 로드"""
    try:
        from diffusers import StableDiffusionXLInpaintPipeline
        
        model_id = os.environ.get(
            "MODEL_ID",
            "diffusers/stable-diffusion-xl-1.0-inpainting-0.1"
        )
        dtype = torch.float16 if registry.device == "cuda" else torch.float32
        
        logger.info(f"모델 로드 시작: {model_id}")
        registry.pipe = StableDiffusionXLInpaintPipeline.from_pretrained(
            model_id,
            torch_dtype=dtype,
        ).to(registry.device)
        
        if registry.device == "cuda":
            registry.pipe.enable_attention_slicing()
            registry.pipe.enable_vae_slicing()
        
        registry.pipe.set_progress_bar_config(disable=True)
        logger.info("모델 로드 완료")
        
    except Exception as e:
        logger.warning(f"모델 로드 실패 ({e}). 데모 모드로 실행.")
        registry.pipe = None


# ──────────────────────────────────────────────
# 2. Lifespan — 앱 시작/종료 이벤트
# ──────────────────────────────────────────────
@asynccontextmanager
async def lifespan(app: FastAPI):
    # 시작 시
    logger.info("서버 시작 중...")
    await asyncio.get_event_loop().run_in_executor(None, load_model)
    logger.info("서버 준비 완료")
    
    yield  # 앱 실행 중
    
    # 종료 시
    logger.info("서버 종료 중...")
    if registry.pipe:
        del registry.pipe
        torch.cuda.empty_cache()


# ──────────────────────────────────────────────
# 3. FastAPI 앱 생성
# ──────────────────────────────────────────────
app = FastAPI(
    title="글살림 AI — 고문서 복원 API",
    description="SDXL Inpainting 기반 역사 문서 복원 서비스",
    version="1.0.0",
    lifespan=lifespan,
)


# ──────────────────────────────────────────────
# 4. 요청/응답 스키마
# ──────────────────────────────────────────────
class RestoreRequest(BaseModel):
    prompt:          str   = "historical Korean document, clear ink brushstroke, aged paper"
    negative_prompt: str   = "blurry, modern, digital artifact, watermark"
    num_steps:       int   = Field(default=20, ge=4, le=100)
    strength:        float = Field(default=0.80, ge=0.1, le=1.0)
    guidance_scale:  float = Field(default=7.5, ge=1.0, le=20.0)
    seed:            int   = Field(default=42)


class JobStatus(BaseModel):
    job_id:  str
    status:  str   # "queued" | "processing" | "done" | "failed"
    message: Optional[str] = None


# ──────────────────────────────────────────────
# 5. 헬스 체크
# ──────────────────────────────────────────────
@app.get("/health")
async def health():
    return {
        "status": "ok",
        "model_loaded": registry.pipe is not None,
        "device": registry.device,
        "cuda_available": torch.cuda.is_available(),
        "gpu_memory_gb": (
            round(torch.cuda.memory_allocated() / 1e9, 2)
            if torch.cuda.is_available() else None
        ),
    }


# ──────────────────────────────────────────────
# 6. 동기 복원 엔드포인트 (데모용)
# ──────────────────────────────────────────────
# GPU Lock: 동시 요청 방지
_gpu_lock = asyncio.Lock()


@app.post(
    "/restore",
    response_class=Response,
    responses={200: {"content": {"image/png": {}}}},
    summary="고문서 이미지 복원 (동기)"
)
async def restore_document(
    image: UploadFile = File(..., description="복원할 고문서 이미지 (PNG/JPG)"),
    mask:  Optional[UploadFile] = File(None, description="손상 마스크 (없으면 자동 생성)"),
    num_steps:       int   = 20,
    strength:        float = 0.80,
    guidance_scale:  float = 7.5,
    seed:            int   = 42,
):
    """
    고문서 이미지를 업로드하면 AI가 복원하여 PNG로 반환합니다.
    처리 시간: 5~30초 (GPU 모델 및 이미지 크기에 따라)
    """
    if registry.pipe is None:
        raise HTTPException(503, "모델 로드 중. 잠시 후 재시도하세요.")
    
    # 입력 검증
    if image.content_type not in ["image/png", "image/jpeg"]:
        raise HTTPException(400, "PNG 또는 JPEG 이미지만 지원합니다.")
    
    if image.size and image.size > 20 * 1024 * 1024:  # 20MB 제한
        raise HTTPException(413, "이미지 크기는 20MB 이하여야 합니다.")
    
    try:
        # 이미지 로드
        img_bytes = await image.read()
        input_img = Image.open(io.BytesIO(img_bytes)).convert("RGB")
        
        # 크기 정규화 (SDXL: 1024×1024)
        w, h = input_img.size
        target = 1024
        input_img = input_img.resize((target, target), Image.LANCZOS)
        
        # 마스크 처리
        if mask:
            mask_bytes = await mask.read()
            mask_img = Image.open(io.BytesIO(mask_bytes)).convert("L")
            mask_img = mask_img.resize((target, target), Image.LANCZOS)
        else:
            # 자동 마스크: 밝기 임계값 기반
            gray = np.array(input_img.convert("L"))
            mask_arr = (gray > 230).astype(np.uint8) * 255
            mask_img = Image.fromarray(mask_arr)
        
        # GPU 처리 (Lock으로 직렬화)
        async with _gpu_lock:
            generator = torch.Generator(registry.device).manual_seed(seed)
            
            loop = asyncio.get_event_loop()
            result = await loop.run_in_executor(
                None,
                lambda: registry.pipe(
                    prompt="historical Korean document, clear traditional ink brushstroke, aged paper texture",
                    negative_prompt="blurry, modern font, digital artifact, watermark",
                    image=input_img,
                    mask_image=mask_img,
                    num_inference_steps=num_steps,
                    strength=strength,
                    guidance_scale=guidance_scale,
                    generator=generator,
                ).images[0]
            )
        
        # PNG 직렬화
        buf = io.BytesIO()
        result.save(buf, format="PNG", optimize=True)
        buf.seek(0)
        
        logger.info(f"복원 완료: {w}×{h} → {target}×{target}")
        return Response(content=buf.read(), media_type="image/png")
    
    except Exception as e:
        logger.error(f"복원 실패: {e}")
        raise HTTPException(500, f"처리 중 오류 발생: {str(e)}")


# ──────────────────────────────────────────────
# 7. 비동기 큐 엔드포인트 (프로덕션용)
# ──────────────────────────────────────────────
@app.post("/restore/submit", response_model=JobStatus)
async def submit_restore_job(
    background_tasks: BackgroundTasks,
    image: UploadFile = File(...),
    num_steps: int = 20,
):
    """
    복원 작업을 큐에 등록하고 job_id를 즉시 반환.
    /restore/status/{job_id}로 상태 확인.
    """
    job_id = str(uuid.uuid4())[:8]
    registry.job_results[job_id] = {"status": "queued"}
    
    img_bytes = await image.read()
    
    async def process_job():
        registry.job_results[job_id]["status"] = "processing"
        try:
            input_img = Image.open(io.BytesIO(img_bytes)).convert("RGB")
            input_img = input_img.resize((1024, 1024), Image.LANCZOS)
            
            gray = np.array(input_img.convert("L"))
            mask_arr = (gray > 230).astype(np.uint8) * 255
            mask_img = Image.fromarray(mask_arr)
            
            if registry.pipe:
                async with _gpu_lock:
                    loop = asyncio.get_event_loop()
                    result = await loop.run_in_executor(
                        None,
                        lambda: registry.pipe(
                            prompt="historical Korean document, clear traditional ink",
                            image=input_img,
                            mask_image=mask_img,
                            num_inference_steps=num_steps,
                        ).images[0]
                    )
                buf = io.BytesIO()
                result.save(buf, format="PNG")
                registry.job_results[job_id]["result"] = buf.getvalue()
            
            registry.job_results[job_id]["status"] = "done"
        except Exception as e:
            registry.job_results[job_id] = {"status": "failed", "message": str(e)}
    
    background_tasks.add_task(process_job)
    return JobStatus(job_id=job_id, status="queued")


@app.get("/restore/status/{job_id}", response_model=JobStatus)
async def get_job_status(job_id: str):
    if job_id not in registry.job_results:
        raise HTTPException(404, "Job not found")
    job = registry.job_results[job_id]
    return JobStatus(job_id=job_id, status=job["status"])


@app.get("/restore/result/{job_id}", response_class=Response)
async def get_job_result(job_id: str):
    if job_id not in registry.job_results:
        raise HTTPException(404, "Job not found")
    job = registry.job_results[job_id]
    if job["status"] != "done":
        raise HTTPException(202, f"처리 중: {job['status']}")
    return Response(content=job["result"], media_type="image/png")


# ──────────────────────────────────────────────
# 8. 서버 실행
# ──────────────────────────────────────────────
if __name__ == "__main__":
    import uvicorn
    
    uvicorn.run(
        "main:app",
        host="0.0.0.0",
        port=8000,
        reload=False,           # 프로덕션에서는 False
        workers=1,              # GPU 서버는 worker 1개 (모델 공유 불가)
        log_level="info",
        access_log=True,
    )
    
    # 프로덕션 실행 명령:
    # gunicorn main:app -w 1 -k uvicorn.workers.UvicornWorker --bind 0.0.0.0:8000
    #
    # API 테스트:
    # curl -X POST http://localhost:8000/restore \
    #      -F "image=@sample.png" \
    #      --output restored.png
```

## 함정 카드 — 자주 실수하는 5가지

1. **"매 요청마다 모델을 로드한다"** → 내가 처음 이 실수를 했다. 요청 하나에 40초, 다음 요청에도 40초. SDXL 로딩이 20초, 실제 추론 8초였다. 모델은 **앱 시작 시 한 번만** 로드하고 `registry`에 전역 보관.

2. **"async def에서 동기 GPU 코드를 직접 호출한다"** → `async def`에서 `pipe(...)`를 직접 호출하면 이벤트 루프가 블록된다. 반드시 `await loop.run_in_executor(None, lambda: pipe(...))`로 스레드풀에서 실행. GPU 작업은 CPU 비동기 컨텍스트에서 실행 불가.

3. **"GPU Lock 없이 동시 요청을 처리한다"** → GPU는 병렬 추론이 가능하지만 단일 파이프라인 인스턴스는 thread-safe하지 않다. `asyncio.Lock()`으로 한 번에 하나의 추론만 실행. 여러 GPU가 있다면 파이프라인 풀을 구성.

4. **"에러를 모두 500으로 반환한다"** → 이미지 크기 초과는 413, 형식 오류는 400, 모델 미로드는 503. 클라이언트가 에러 원인을 파악할 수 없으면 디버깅이 불가능. FastAPI의 HTTPException 상태 코드를 적절히 구분.

5. **"API 문서를 따로 작성한다"** → FastAPI는 Pydantic 모델과 docstring에서 Swagger UI(`/docs`)를 자동 생성한다. 별도 문서 작성 시간을 낭비하지 말고, 코드의 타입 힌트와 `summary=`, `description=` 파라미터로 문서를 대체.

## 연습문제

**문제 1.** 현재 `/restore` 엔드포인트는 요청당 최대 1개의 이미지를 처리한다. 여러 이미지를 한 번에 배치로 업로드하여 처리하는 `/restore/batch` 엔드포인트를 설계하라. 입력과 출력 형식을 정의하라.

> **해설:** 입력: `images: List[UploadFile]`, 크기 제한: 최대 10개. 출력: `{"results": [{"filename": "xxx.png", "data": "<base64>", "success": bool}]}`. 구현: 이미지 리스트 수집 후 `pipe(prompt=[p]*n, image=[...], mask_image=[...])`로 배치 추론. 결과를 base64 인코딩하여 JSON 반환. 배치가 GPU VRAM을 초과하면 `torch.cuda.OutOfMemoryError`를 잡아 자동으로 개별 처리 폴백.

**문제 2.** `/restore/submit` 비동기 엔드포인트에서 `registry.job_results`를 딕셔너리로 사용했다. 이 방식의 문제점과 프로덕션 환경에서의 대안을 제시하라.

> **해설:** 문제점: ① 메모리 누수 — 완료된 작업이 계속 쌓임 ② 서버 재시작 시 모든 작업 기록 소멸 ③ 멀티 프로세스 환경에서 프로세스 간 공유 불가. 대안: **Redis** — `redis.setex(job_id, 3600, json.dumps(status))`로 TTL 1시간 설정. 또는 **SQLite** (소규모) — 완료 후 30분 이상 된 레코드 주기적 삭제. 프로덕션: Celery + Redis 큐.

**문제 3.** `/health` 엔드포인트를 확장하여 Kubernetes liveness probe와 readiness probe에 각각 사용할 수 있도록 설계하라. 두 probe의 차이는 무엇인가?

> **해설:** **liveness probe** (`/health/live`): "컨테이너가 살아있는가" — 앱이 응답하면 200. 죽으면 컨테이너 재시작. 간단: `return {"alive": True}`. **readiness probe** (`/health/ready`): "요청을 받을 준비가 됐는가" — 모델 로드 완료 여부 포함. `{"ready": registry.pipe is not None, "model": ...}`. 모델 로드 중이면 503 반환 → Kubernetes가 트래픽 라우팅 차단. 두 probe를 분리하면 모델 로딩 중에도 컨테이너가 재시작되지 않음.
