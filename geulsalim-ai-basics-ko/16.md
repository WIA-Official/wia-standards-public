# 챕터 16 · 추론 최적화 — FP16 · INT8 · torch.compile · TensorRT

## 학습 목표
- 훈련된 모델의 추론 속도를 최적화하는 4가지 전략(FP16, INT8, torch.compile, TensorRT)을 적용할 수 있다
- 고문서 복원 서비스에서 이미지 한 장 처리 시간을 10초 이하로 줄이는 파이프라인을 구성할 수 있다
- 품질 손실 없이 속도를 높이는 최적화 우선순위를 결정할 수 있다

---

## 이론

### 추론 최적화가 왜 필요한가

A100 GPU에서 SDXL Inpainting 1장 처리:
- 최적화 없음: ~30초
- FP16: ~15초
- torch.compile + FP16: ~8초
- TensorRT: ~5초

**비용 차이**: 1,000장 처리 시 30,000초 vs 5,000초 = **A100 6.9시간 vs 1.4시간** = 약 $10 vs $1.7.

### FP16 추론 — 가장 간단한 최적화

학습에서 이미 다뤘지만 추론에서도 적용 가능.

```python
with torch.autocast("cuda", dtype=torch.float16):
    output = model(input)
```

**장점**: 코드 한 줄, VRAM 절반, 속도 1.5~2배
**품질 영향**: 거의 없음 (복원 이미지 SSIM 차이 < 0.01)

### torch.compile — PyTorch 2.0의 혁신

2022년 PyTorch 2.0에서 도입. Python 인터프리터 오버헤드 제거 + 연산 융합.

```python
import torch

model = torch.compile(
    model,
    mode="reduce-overhead",  # 또는 "max-autotune" (더 느린 컴파일, 더 빠른 추론)
    fullgraph=True,
)
```

**성능**: 기본 PyTorch 대비 1.8~10배 향상 (모델 아키텍처에 따라)
**주의**: 첫 실행 시 컴파일 시간 1~5분 소요. 캐싱되면 이후 실행부터 빠름.

최신 동향 (2026): NVIDIA AITune이 `torch.compile`과 TensorRT를 자동으로 선택해주는 도구로 출시 (Apache 2.0 라이선스).

### INT8 양자화 — 극한 최적화

FP32(32비트) → INT8(8비트)로 가중치 양자화.

- VRAM 4배 절약
- 속도 2~4배 향상 (Integer 연산이 Float보다 빠름)
- 품질 손실: 약간 있음 (SSIM -0.02~-0.05 수준)

**2가지 INT8 방식**:
- **PTQ (Post-Training Quantization)**: 학습 완료 후 양자화. 빠르지만 품질 손실 있음
- **QAT (Quantization-Aware Training)**: 학습 중 INT8 시뮬레이션. 품질 손실 최소화

### TensorRT — NVIDIA 최고 성능

NVIDIA GPU에 특화된 추론 최적화 엔진.

```
PyTorch 모델 → ONNX 변환 → TensorRT 최적화 엔진 → 2~3배 추가 속도
```

**torch.compile vs TensorRT**:
- 비교 실험(Llama-7b, Mistral): 두 방법 성능 거의 동일
- torch.compile: 통합 쉬움, 코드 변경 최소
- TensorRT: 배포 환경에서 더 일관된 성능

**Torch-TensorRT** (2025): PyTorch API 유지하면서 TensorRT 성능 활용.

### 추론 파이프라인 최적화 체크리스트

```
[즉시 적용]
✅ torch.float16 (or bfloat16) 추론
✅ model.eval() 호출 (dropout, BN 비활성화)
✅ torch.no_grad() 컨텍스트
✅ enable_attention_slicing() (대형 이미지)

[중급 최적화]
✅ torch.compile(mode="reduce-overhead")
✅ DDIM 샘플러 20~30 스텝 (vs 기본 50)
✅ LCM (Latent Consistency Model) 4~8 스텝

[고급 최적화]
✅ TensorRT 또는 Torch-TensorRT
✅ INT8 PTQ (품질 허용 시)
✅ Flash Attention 2 (xformers 또는 내장)
```

## 실전 코드

```python
# 추론 최적화 파이프라인
# pip install torch diffusers xformers

import time
import torch
from diffusers import StableDiffusionXLInpaintPipeline
from contextlib import contextmanager


# ──────────────────────────────────────────────
# 1. 기본 최적화 파이프라인 빌드
# ──────────────────────────────────────────────
def build_optimized_pipeline(
    model_id: str = "diffusers/stable-diffusion-xl-1.0-inpainting-0.1",
    device: str = "cuda",
    dtype: torch.dtype = torch.float16,
    use_xformers: bool = True,
    use_compile: bool = False,  # 첫 실행 시 컴파일 대기 필요
) -> StableDiffusionXLInpaintPipeline:
    """
    SDXL Inpainting 추론 최적화 파이프라인
    순서: 로드 → FP16 → xformers → 슬라이싱 → (선택) compile
    """
    print("파이프라인 로드 중...")
    pipe = StableDiffusionXLInpaintPipeline.from_pretrained(
        model_id,
        torch_dtype=dtype,
        variant="fp16",
        use_safetensors=True,
    ).to(device)
    
    pipe.set_progress_bar_config(disable=True)  # 배치 처리 시 진행바 끄기
    
    # Attention 최적화
    if use_xformers:
        try:
            pipe.enable_xformers_memory_efficient_attention()
            print("xformers 메모리 효율적 어텐션: 활성화")
        except Exception:
            pipe.enable_attention_slicing()
            print("xformers 없음 → attention slicing 사용")
    
    # VAE 최적화
    pipe.enable_vae_slicing()    # 대형 이미지 VAE 처리
    pipe.enable_vae_tiling()     # 매우 큰 이미지 (2048px+)
    
    # torch.compile (선택적)
    if use_compile and torch.__version__ >= "2.0":
        print("UNet torch.compile 시작 (첫 추론에 1~5분 소요)...")
        pipe.unet = torch.compile(
            pipe.unet,
            mode="reduce-overhead",
            fullgraph=True,
        )
        print("torch.compile 완료")
    
    return pipe


# ──────────────────────────────────────────────
# 2. 추론 속도 벤치마크
# ──────────────────────────────────────────────
@contextmanager
def timer(label: str):
    start = time.perf_counter()
    yield
    elapsed = time.perf_counter() - start
    print(f"[{label}] {elapsed:.2f}초")


def benchmark_inference(
    pipe,
    n_steps_list: list = [10, 20, 30, 50],
    image_size: int = 1024,
    n_warmup: int = 1,
):
    """
    스텝 수별 추론 시간 벤치마크
    LCM scheduler: 4~8 스텝으로 30 스텝급 품질
    DDIM: 20~30 스텝으로 고품질
    """
    from PIL import Image
    import numpy as np
    
    # 더미 입력
    dummy_image = Image.fromarray(
        np.random.randint(0, 255, (image_size, image_size, 3), dtype=np.uint8)
    )
    dummy_mask = Image.fromarray(
        np.random.randint(0, 255, (image_size, image_size), dtype=np.uint8)
    )
    prompt = "historical Korean document, clear ink text, aged paper"
    
    results = {}
    
    for n_steps in n_steps_list:
        # 워밍업
        for _ in range(n_warmup):
            with torch.no_grad():
                pipe(
                    prompt=prompt,
                    image=dummy_image,
                    mask_image=dummy_mask,
                    num_inference_steps=n_steps,
                    height=image_size,
                    width=image_size,
                )
        
        # 실제 측정 (3회 평균)
        times = []
        for _ in range(3):
            start = time.perf_counter()
            with torch.no_grad():
                pipe(
                    prompt=prompt,
                    image=dummy_image,
                    mask_image=dummy_mask,
                    num_inference_steps=n_steps,
                    height=image_size,
                    width=image_size,
                )
            times.append(time.perf_counter() - start)
        
        avg_time = sum(times) / len(times)
        results[n_steps] = avg_time
        print(f"  {n_steps:3d} steps: {avg_time:.1f}초")
    
    return results


# ──────────────────────────────────────────────
# 3. INT8 PTQ 양자화 (torch.ao 사용)
# ──────────────────────────────────────────────
def apply_int8_quantization(model, calibration_data=None):
    """
    Post-Training Quantization (INT8)
    torch 2.0+ 기준
    """
    try:
        from torch.ao.quantization import quantize_dynamic
        
        # Dynamic Quantization: Linear 레이어를 INT8로
        quantized_model = quantize_dynamic(
            model,
            {torch.nn.Linear},
            dtype=torch.qint8,
        )
        
        print("INT8 Dynamic Quantization 적용")
        
        # 파라미터 수 비교
        orig_size = sum(p.numel() * p.element_size() for p in model.parameters())
        # 양자화 후 크기 추정 (Linear 레이어의 50% 감소 가정)
        print(f"원본 크기: {orig_size / 1024 / 1024:.0f} MB")
        print("INT8 후 예상: 약 50~75% 감소")
        
        return quantized_model
    except Exception as e:
        print(f"INT8 양자화 실패: {e}")
        return model


# ──────────────────────────────────────────────
# 4. 처리량 최대화 — 배치 처리
# ──────────────────────────────────────────────
def batch_restore_documents(
    pipe,
    image_paths: list,
    output_dir: str = "./restored",
    batch_size: int = 4,
    n_steps: int = 20,
):
    """
    대량 고문서 이미지 배치 복원
    메모리 효율적 배치 처리
    """
    import os
    from PIL import Image
    import numpy as np
    
    os.makedirs(output_dir, exist_ok=True)
    
    prompt = "historical Korean document, clear traditional ink brushstroke, aged paper texture, restored, high detail"
    negative = "blurry, modern, digital artifact, watermark, noise"
    
    total = len(image_paths)
    
    for batch_start in range(0, total, batch_size):
        batch_paths = image_paths[batch_start:batch_start + batch_size]
        
        images = []
        masks  = []
        
        for path in batch_paths:
            img = Image.open(path).convert("RGB").resize((1024, 1024))
            # 간단 자동 마스크: 매우 밝은 영역
            gray = np.array(img.convert("L"))
            mask_arr = (gray > 230).astype(np.uint8) * 255
            mask = Image.fromarray(mask_arr)
            images.append(img)
            masks.append(mask)
        
        with torch.no_grad(), torch.autocast("cuda", dtype=torch.float16):
            results = pipe(
                prompt=[prompt] * len(images),
                negative_prompt=[negative] * len(images),
                image=images,
                mask_image=masks,
                num_inference_steps=n_steps,
                guidance_scale=7.5,
                strength=0.8,
            ).images
        
        for path, result in zip(batch_paths, results):
            out_name = os.path.basename(path)
            result.save(os.path.join(output_dir, out_name))
        
        print(f"처리: {batch_start + len(batch_paths)}/{total}")
    
    print(f"배치 복원 완료: {output_dir}")


if __name__ == "__main__":
    print("추론 최적화 설정 가이드")
    print("=" * 50)
    
    settings = [
        ("기본 (최적화 없음)",  "30초/장", "필요 없음"),
        ("FP16",               "15초/장", "코드 1줄"),
        ("FP16 + xformers",    "10초/장", "pip install xformers"),
        ("FP16 + torch.compile", "6초/장", "첫 실행 3분 컴파일"),
        ("FP16 + DDIM 20스텝", "6초/장",  "num_steps 줄이기"),
        ("FP16 + LCM 8스텝",   "2.5초/장","스케줄러 교체"),
    ]
    
    print(f"{'방법':<25} {'속도':>10} {'준비':>20}")
    print("-" * 60)
    for method, speed, prep in settings:
        print(f"{method:<25} {speed:>10} {prep:>20}")
```

## 함정 카드 — 자주 실수하는 5가지

1. **"model.eval()을 빠뜨린다"** → `model.eval()`이 없으면 Dropout과 BatchNorm이 학습 모드로 동작. 추론 결과가 매 실행마다 달라지며 성능도 저하. 항상 `with torch.no_grad(): model.eval()` 조합.

2. **"torch.compile을 모든 모델에 적용한다"** → torch.compile은 정적 그래프가 필요. 동적 shape(이미지 크기가 매번 달라지는 경우) 사용 시 매 실행마다 재컴파일. 해결: 입력을 고정 크기로 패딩하거나 `torch.compile(dynamic=True)`.

3. **"xformers 없이 큰 이미지를 처리한다"** → 1024px 이상 SDXL을 xformers 없이 실행하면 Attention의 O(n²) 메모리가 폭증. `enable_xformers_memory_efficient_attention()` 또는 `enable_attention_slicing()` 필수.

4. **"INT8 양자화 후 정확도 검증을 안 한다"** → INT8은 품질 손실이 있다. 고문서 복원처럼 세밀한 문자 복원이 필요한 작업에서 INT8 후 SSIM/LPIPS 반드시 재측정. 품질 하락이 허용 기준 초과 시 INT8 적용 재고.

5. **"LCM 4스텝이 항상 빠르고 좋다고 생각한다"** → LCM(Latent Consistency Model)은 4~8스텝에서 빠른 생성을 지원하지만, 고문서 복원처럼 원본 이미지 구조 유지가 중요한 작업에서는 `strength` 조정이 까다롭다. DDIM 20~30스텝이 복원 품질 면에서 더 안정적인 경우 많음.

## 연습문제

**문제 1.** torch.compile을 적용했더니 첫 추론에 5분이 걸렸다. 다음 추론부터는 6초가 됐다. 이 컴파일 결과를 캐싱하여 서버 재시작 후에도 재사용하는 방법은?

> **해설:** `torch._dynamo.config.cache_size_limit = 64` 설정 + `torch.export`로 컴파일된 모델 저장. 또는 TorchScript `torch.jit.script()` + `torch.jit.save()`로 재사용. 현실적으로는 서버 시작 시 1회 컴파일 워밍업 루틴을 포함하거나, TensorRT로 .engine 파일을 저장하는 방식이 더 안정적.

**문제 2.** DDIM 20 스텝과 LCM 4 스텝으로 고문서 복원 결과를 비교한다. 어떤 지표로 두 방법을 평가하겠는가? SSIM이 낮더라도 LCM을 선택할 수 있는 기준은 무엇인가?

> **해설:** 평가 지표: SSIM, LPIPS, 처리 시간, 처리 비용(GPU $/장). LCM 선택 기준: ① SSIM 차이가 0.05 이내이고 ② 1차 목적이 대량 처리(수천 장/일)이며 ③ 전문가 육안 평가에서 두 방법의 차이가 구별 불가할 때. 단, 중간심사 시연용 샘플 복원에는 DDIM으로 최고 품질 확보 권장.

**문제 3.** A100 GPU에서 SDXL Inpainting 1장 처리에 8초, 비용이 $1.29/hr이다. 하루 10,000장을 처리할 때의 GPU 비용을 계산하고, torch.compile로 6초로 줄이면 얼마나 절약되는가?

> **해설:** 8초 × 10,000장 = 80,000초 = 22.2시간. 비용: 22.2 × $1.29 = **$28.6/일**. torch.compile 6초: 60,000초 = 16.7시간. 비용: 16.7 × $1.29 = $21.5/일. **절약: $7.1/일**. 연간 $2,592 절약. 최적화의 누적 효과는 무시할 수 없다.
