## 이론

### 모래사장 비유

깨끗한 모래사장이 있다고 하자. 여기에 돌을 던지면 파문이 생기고, 바람이 불면 모래가 흩어진다. 시간이 지날수록 원래 형태는 사라지고 **무작위 노이즈** 상태가 된다.

이제 역방향을 생각해보자. 완전히 흩어진 모래사장을 보고 "원래 어떤 형태였는지" 복원할 수 있는가? 불가능해 보인다.

하지만 만약 "흩어지는 과정의 매 순간"을 전부 기록해두었다면? 그 역방향 과정을 학습한 AI가 있다면?

이것이 확산 모델의 아이디어다.

### Forward Process — 노이즈 추가

원본 이미지 x₀에 매 단계(timestep)마다 약간의 가우시안 노이즈를 추가한다. T번 반복하면 순수한 노이즈 xT가 된다.

```
x₀ (원본) → x₁ (약간 흐림) → x₂ (더 흐림) → ... → xT (순수 노이즈)
```

각 단계의 노이즈 크기는 **β 스케줄**로 조절한다. 처음에는 조금씩, 나중에는 많이 추가하는 cosine schedule이 현재 표준이다.

이 과정은 **수학적으로 닫힌 형태**가 있다. 즉, x₅₀₀의 상태를 계산하려면 x₁부터 x₄₉₉까지 순서대로 계산할 필요 없이, x₀와 타임스텝 500만으로 직접 계산할 수 있다.

### Reverse Process — 노이즈 제거 학습

모델이 학습하는 것은 딱 한 가지다.

> "이 noisy한 이미지 xt에서, 어떤 노이즈 ε가 추가된 건지 예측하라."

즉 모델은 **노이즈를 예측**하는 함수 εθ(xt, t)를 학습한다. 예측된 노이즈를 빼면 약간 더 깨끗한 이미지가 나오고, 이를 T번 반복하면 원본이 복원된다.

### DDPM vs DDIM

**DDPM** (Denoising Diffusion Probabilistic Models, 2020): 원조. T=1000 스텝. 이미지 1장 생성에 분 단위 소요.

**DDIM** (Denoising Diffusion Implicit Models, 2021): 스텝 수를 20~50으로 줄임. 속도 10~50배 향상. 현재 대부분의 제품에서 사용.

**DPM-Solver++**: 수치 ODE solver 활용. 5~10 스텝으로 고품질 생성. SDXL의 기본 샘플러.

### 고문서 복원에 적합한 이유

고문서 이미지는 두 가지 문제를 가진다.

1. **노이즈 문제**: 세월이 지나면서 얼룩, 변색, 번짐이 생긴다.
2. **결손 문제**: 일부 영역이 완전히 사라졌다 (Inpainting 필요).

확산 모델은 두 가지 모두를 처리할 수 있다.

- **Image-to-Image**: 노이즈 있는 원본을 입력하면 복원된 이미지 출력
- **Inpainting**: 마스킹된 영역을 주변 맥락으로 채움

## 실전 코드

```python
# 확산 모델 핵심 연산 직접 구현
# forward process와 역방향 추론의 원리를 이해한다

import torch
import numpy as np
import matplotlib
matplotlib.use('Agg')  # 서버 환경 (GUI 없음)
import matplotlib.pyplot as plt
from PIL import Image

# ── 1. 노이즈 스케줄 ─────────────────────────────────
def cosine_beta_schedule(timesteps: int, s: float = 0.008):
    """
    Cosine noise schedule (Improved DDPM, 2021)
    처음에는 천천히, 나중에는 빠르게 노이즈 추가
    """
    steps = timesteps + 1
    x = torch.linspace(0, timesteps, steps)
    alphas_cumprod = torch.cos(((x / timesteps) + s) / (1 + s) * np.pi * 0.5) ** 2
    alphas_cumprod = alphas_cumprod / alphas_cumprod[0]
    betas = 1 - (alphas_cumprod[1:] / alphas_cumprod[:-1])
    return torch.clamp(betas, 0.0001, 0.9999)


class DiffusionProcess:
    """확산 프로세스 — forward와 reverse 핵심 연산"""
    
    def __init__(self, timesteps: int = 1000):
        self.T = timesteps
        
        # 노이즈 스케줄 계산
        self.betas = cosine_beta_schedule(timesteps)
        self.alphas = 1.0 - self.betas
        self.alphas_cumprod = torch.cumprod(self.alphas, dim=0)
        self.sqrt_alphas_cumprod = torch.sqrt(self.alphas_cumprod)
        self.sqrt_one_minus_alphas_cumprod = torch.sqrt(1.0 - self.alphas_cumprod)
    
    def q_sample(self, x0: torch.Tensor, t: torch.Tensor) -> tuple:
        """
        Forward process: 원본 이미지에 t 단계 노이즈 추가
        닫힌 형태(closed form) — x0에서 xt를 직접 계산
        
        xt = √ᾱt · x0 + √(1-ᾱt) · ε
        """
        noise = torch.randn_like(x0)  # 표준 가우시안 노이즈
        
        # t 단계의 alpha 값 추출
        sqrt_alpha = self.sqrt_alphas_cumprod[t].view(-1, 1, 1, 1)
        sqrt_one_minus = self.sqrt_one_minus_alphas_cumprod[t].view(-1, 1, 1, 1)
        
        # 노이즈 추가
        xt = sqrt_alpha * x0 + sqrt_one_minus * noise
        
        return xt, noise  # noisy 이미지와 추가된 노이즈 반환
    
    def visualize_forward(self, image_path: str = None):
        """Forward process 시각화 — 노이즈가 쌓이는 과정"""
        
        if image_path:
            img = Image.open(image_path).resize((256, 256))
            x0 = torch.tensor(np.array(img)).float() / 127.5 - 1.0  # [-1, 1] 정규화
            x0 = x0.permute(2, 0, 1).unsqueeze(0)  # [1, C, H, W]
        else:
            # 데모용 랜덤 이미지
            x0 = torch.randn(1, 3, 256, 256)
        
        timesteps_to_show = [0, 100, 250, 500, 750, 999]
        
        fig, axes = plt.subplots(1, len(timesteps_to_show), figsize=(15, 3))
        
        for ax, t in zip(axes, timesteps_to_show):
            if t == 0:
                xt = x0
            else:
                t_tensor = torch.tensor([t])
                xt, _ = self.q_sample(x0, t_tensor)
            
            # 시각화용 변환
            img_show = xt.squeeze().permute(1, 2, 0).clamp(-1, 1)
            img_show = ((img_show + 1) / 2).numpy()
            
            ax.imshow(img_show)
            ax.set_title(f"t={t}")
            ax.axis('off')
        
        plt.suptitle("Forward Process: 노이즈가 쌓이는 과정")
        plt.tight_layout()
        plt.savefig("forward_process.png", dpi=150, bbox_inches='tight')
        print("forward_process.png 저장됨")


# ── 2. 간단한 학습 루프 (의사 코드) ─────────────────
def training_step(model, x0: torch.Tensor, diffusion: DiffusionProcess) -> torch.Tensor:
    """
    확산 모델 학습의 한 스텝
    모델이 학습하는 것: "이 noisy 이미지에서 어떤 노이즈가 추가됐는가"
    """
    batch_size = x0.shape[0]
    
    # 1. 랜덤 타임스텝 샘플링
    t = torch.randint(0, diffusion.T, (batch_size,))
    
    # 2. Forward: 노이즈 추가
    xt, noise_added = diffusion.q_sample(x0, t)
    
    # 3. 모델 예측: 추가된 노이즈 추정
    noise_predicted = model(xt, t)  # U-Net 등
    
    # 4. 손실 계산: 예측 노이즈 vs 실제 추가된 노이즈
    loss = torch.nn.functional.mse_loss(noise_predicted, noise_added)
    
    return loss

    # mask = Image.open("damage_mask.png")  # 흰색=복원 영역, 검정=보존 영역

    #     image=image,

if __name__ == "__main__":
    diffusion = DiffusionProcess(timesteps=1000)
```

## 함정 카드 — 자주 실수하는 5가지

1. **"timestep이 많을수록 품질이 좋다"** → DDIM으로 20~50 스텝도 충분한 품질이 나온다. 1000 스텝은 DDPM 원본 방식이며 현재는 거의 사용하지 않는다. 스텝 수 대비 품질 곡선을 확인하고 최소 스텝을 찾는 것이 실용적이다.

2. **"strength=1.0으로 Inpainting한다"** → `strength=1.0`은 원본 이미지를 완전히 무시하고 처음부터 생성한다. 고문서 복원 시에는 0.7~0.9 범위가 원본 맥락을 유지하면서 결손 부분을 채운다.

3. **"마스크를 반전해서 쓴다"** → SDXL Inpainting에서 마스크의 **흰색 영역이 복원 대상**이다. 직관과 반대로 느껴질 수 있다. 검정=유지, 흰색=복원.

4. **"guidance_scale을 너무 높게 설정한다"** → `guidance_scale=15` 이상은 artifact(기형적 결과물)가 생길 수 있다. 고문서 복원에는 7~9 범위가 적당하다.

5. **"forward process를 직접 학습하려 한다"** → forward process는 수식으로 정의되어 **학습 대상이 아니다**. 오직 reverse process (노이즈 제거 모델)만 학습한다. 이 구분을 혼동하면 학습 코드 구조가 완전히 잘못된다.

## 연습문제

**문제 1.** 고문서 이미지 복원 작업에서 Inpainting 마스크를 자동으로 생성하려면 어떤 방법을 사용할 수 있는가? (최소 2가지 방법 제시)

> **해설:** ① 이진 임계값(Otsu binarization): 밝기 기준으로 손상 영역 자동 분리. ② 색차이 기반: 원본 색상 범위를 정의하고 벗어난 픽셀을 마스킹. ③ 딥러닝 세그멘테이션: Segment Anything Model (SAM)으로 손상 영역 자동 감지.

**문제 2.** `q_sample()` 함수에서 `t=0`과 `t=999`를 입력했을 때 각각 어떤 결과가 나오는가? `sqrt_alphas_cumprod[0]`과 `sqrt_alphas_cumprod[999]`의 값 범위를 추론하라.

> **해설:** `t=0`: `sqrt_alphas_cumprod[0]`≈1.0, `sqrt_one_minus_alphas_cumprod[0]`≈0.0 → 거의 원본 이미지. `t=999`: `sqrt_alphas_cumprod[999]`≈0.0, `sqrt_one_minus_alphas_cumprod[999]`≈1.0 → 거의 순수 노이즈.

**문제 3.** DDIM과 DDPM의 스텝 수 차이가 왜 품질에 영향을 주지 않는가? DDIM의 핵심 아이디어를 한 문단으로 설명하라.

> **해설:** DDPM은 확률적(stochastic) 과정이라 각 스텝에서 노이즈를 샘플링한다. DDIM은 결정론적(deterministic) ODE로 변환하여 큰 스텝으로 건너뛸 수 있다. 수학적으로 동일한 분포를 따르지만 경로를 효율화한 것이므로 품질 차이가 최소화된다.
