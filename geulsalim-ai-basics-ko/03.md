## 이론

### 왜 클라우드 GPU인가

내가 이 프로젝트를 시작할 때 가장 먼저 막혔던 자리가 이 질문이었다. "GPU를 사야 할까, 빌려야 할까." 당시 A100 80GB 중고 한 장이 1,500만 원을 달라고 했고, 제대로 돌리려면 전기·냉각·공간까지 따로 두어야 했다. 연간 유지비가 구매가의 20~30%를 따라붙었다. 1인 개발자한테 이 숫자는 현실이 아니었다.

그래서 클라우드로 방향을 잡았다. 시간 단위로 빌리고, 실험 끝나면 끄고, 다시 필요할 때 켠다. 이 책의 실습 전체를 클라우드 GPU로 끝내는 데 내가 실제로 쓴 돈은 **대략 30만~80만 원 사이**였다. 학습 한 사이클 실패할 때마다 2~5만 원이 사라져서, 처음 한 달은 영수증을 들여다보며 생각이 많아졌다.

### 4대 서비스 비교

| 서비스 | A100 80GB 시간당 | 특징 | 추천 용도 |
|---|---|---|---|
| **Lambda Labs** | $1.99/hr | 안정적, 재고 부족 빈번 | 중요한 장기 학습 |
| **AWS g5.2xlarge** | $1.21/hr | A10G 24GB, 안정성 최고 | 실험 및 개발 |
| **Paperspace** | $2.30/hr | GUI 노트북, 쉬운 설정 | 초보자, 시각화 |
| **Vast.ai** | $0.4~0.8/hr | 가장 저렴, 신뢰성 낮음 | 단기 실험 |

**권장 전략:**
- 처음 환경 테스트 → Paperspace (GUI로 쉽게)
- 본격 학습 → Lambda Labs (안정성)
- 비용 절감이 중요한 실험 → Vast.ai
- AWS 크레딧 있으면 → g5 시리즈

나는 네 군데를 전부 써봤다. 이 표는 그 시간의 요약이다. 첫 몇 주는 Paperspace에서 GUI로 익히며 기본기를 쌓았고, 본격 학습은 Lambda Labs로 옮겼다가, 하이퍼파라미터 스윕 같은 병렬 실험은 Vast.ai로 돌려 예산을 지켰다. 어느 하나가 정답이 아니라 **학습 단계마다 맞는 도구가 달랐다**. 처음부터 Lambda만 쓰겠다고 결심했다가 재고 부족으로 3일을 버린 경험이 있다. 멀티 프로바이더를 알고 있으면 그런 막힘에서 자유로워진다.

### GPU 스펙 이해

파인튜닝에서 VRAM 크기가 거의 모든 것을 결정한다.

| VRAM | 가능한 작업 |
|---|---|
| 8GB (RTX 3070) | LoRA rank 4, batch 1, 512px |
| 16GB (T4, V100) | LoRA rank 16, batch 2, 768px |
| 24GB (A10G, RTX 3090) | QLoRA Full, batch 4, 1024px |
| 40GB (A100 40GB) | Full fine-tune 소형 모델 |
| 80GB (A100 80GB) | Full fine-tune 대형 모델 |

이 책의 실습은 **24GB (A10G)** 기준으로 설계되었다. AWS g5.2xlarge는 A10G 24GB를 탑재하고 있어 최적의 가성비를 제공한다.

나는 초반에 "A100 80GB를 써야 제대로 배운다"고 믿고 고비용 인스턴스를 계속 잡아뒀다가, 한 달 영수증을 보고 정신이 들었다. 실제로 이 책 60장 대부분의 실습은 **A10G 24GB 한 장이면 충분**하다. 파인튜닝은 "비싼 GPU를 잘 쓰는 기술"이기도 하지만, 그 전에 **"주어진 GPU에서 낭비하지 않는 기술"** 이다. 24GB에서 돌아가게 설계한 파이프라인은 80GB로 옮기는 것도 쉽다. 반대는 어렵다.

```python
# GPU 사용량 실시간 모니터링 스크립트
# 학습 중 별도 터미널에서 실행

import time
import subprocess

def get_gpu_stats():
    """nvidia-smi로 GPU 상태 조회"""
    result = subprocess.run(
        ['nvidia-smi', '--query-gpu=utilization.gpu,memory.used,memory.total,temperature.gpu',
         '--format=csv,noheader,nounits'],
        capture_output=True, text=True
    )
    if result.returncode != 0:
        return None
    
    values = result.stdout.strip().split(', ')
    return {
        'gpu_util': int(values[0]),
        'mem_used': int(values[1]),
        'mem_total': int(values[2]),
        'temp': int(values[3])
    }

def monitor(interval: int = 5):
    """GPU 상태를 interval 초마다 출력"""
    print("GPU 모니터링 시작 (Ctrl+C로 종료)")
    print(f"{'시각':12} {'GPU%':6} {'VRAM':14} {'온도':6}")
    print("-" * 45)
    
    while True:
        stats = get_gpu_stats()
        if stats:
            vram_bar = "█" * (stats['mem_used'] // (stats['mem_total'] // 20))
            print(
                f"{time.strftime('%H:%M:%S'):12} "
                f"{stats['gpu_util']:5}% "
                f"{stats['mem_used']:5}/{stats['mem_total']}MB "
                f"{stats['temp']:4}°C"
            )
        time.sleep(interval)

if __name__ == "__main__":
    monitor()
```

## 함정 카드 — 내가 돈으로 배운 5가지

1. **"인스턴스를 종료하지 않고 잊는다"** → 가장 비싼 실수. 나는 어느 금요일 저녁 학습을 돌려두고 주말 동안 잊어버려서 월요일에 30만 원이 녹아있는 걸 봤다. 이후로 Lambda Labs는 학습 스크립트 마지막 줄에 `auto_shutdown()` 함수를 심어두고, Vast.ai는 인스턴스 생성 시 **max_runtime** 을 반드시 설정한다.

2. **"SSH 키를 잃어버린다"** → Lambda Labs, AWS는 SSH 키를 처음 1번만 다운로드할 수 있다. 나는 첫 주에 이걸 몰라서 새 인스턴스 만들 때마다 키를 새로 발급받았다. `~/.ssh/` 폴더에 안전하게 백업하고, 1Password 같은 도구에 한 번 더 복사해 두는 게 편하다.

3. **"CUDA 버전과 PyTorch 버전이 맞지 않는다"** → 내가 처음 반나절을 날린 지점. `pip install torch` 만 쳤다가 CUDA 11.8에 PyTorch CPU 빌드가 깔린 적이 있다. `nvcc --version` 으로 CUDA를 먼저 확인하고, 해당 버전에 맞는 PyTorch 휠을 명시적으로 설치해야 한다. 위 스크립트는 CUDA 12.1 기준이다.

4. **"tmux 없이 학습을 돌린다"** → 공용 Wi-Fi에서 SSH를 유지한 채 학습 돌리다가, 연결이 한 번 끊기는 순간 3시간치 학습이 사라졌다. 그 날 이후로 첫 명령은 무조건 `tmux new -s 이름` 이다. `screen` 도 같은 역할.

5. **"모델 파일을 로컬에 백업하지 않는다"** → Vast.ai 인스턴스가 종료되면서 체크포인트가 같이 사라진 적이 있다. 이후로는 학습 루프 안에 **10분마다 S3 또는 Hugging Face Hub에 주기 업로드** 하는 코드를 기본으로 넣는다. GPU가 비싸다고 생각하지만, **잃어버린 체크포인트를 다시 만드는 시간이 더 비싸다**.

## 연습문제

**문제 1.** 다음 조건에서 최적의 GPU 클라우드를 선택하고 이유를 설명하라: "예산 20만 원, 24시간 이내 완료 필요, SDXL LoRA 파인튜닝."

> **해설:** Vast.ai (A100 기준 $0.8/hr × 24hr ≈ $19 ≈ 2.7만원) 또는 Lambda Labs ($1.99/hr × 24hr ≈ $48 ≈ 7만원). 예산을 최대한 아끼려면 Vast.ai, 안정성이 중요하면 Lambda Labs. 24시간이면 tmux 필수.

**문제 2.** `get_gpu_stats()` 함수를 수정하여 GPU가 여러 장인 경우(multi-GPU) 모든 GPU의 통계를 리스트로 반환하도록 개선하라.

> **해설:** `nvidia-smi`에 `--id=0,1,2`를 제거하거나, 출력 결과를 `splitlines()`로 분할하여 각 줄을 파싱한다. 반환값을 `List[dict]`로 변경한다.

**문제 3.** 위 세팅 스크립트에서 `pip install` 명령 이후 실제로 설치된 패키지 버전을 확인하는 명령을 작성하라. 예상치 못한 버전이 설치됐을 때 어떻게 대응해야 하는가?

> **해설:** `pip show torch transformers diffusers | grep Version`으로 확인한다. 버전이 다르면 `pip install "torch==2.1.0" --force-reinstall`로 강제 재설치한다. 의존성 충돌 시 `pip install 패키지 --no-deps`로 의존성 없이 설치 후 수동 확인한다.
