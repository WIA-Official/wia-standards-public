## 이론

### 모니터링 없는 학습은 눈 감고 운전하는 것

GPU 학습은 시간당 수천 원이 소모된다. 실시간 모니터링 없이는:
- 손실이 발산했는데 4시간 후에야 알게 된다
- 과적합이 1,000 스텝에서 이미 시작됐지만 5,000 스텝까지 학습한다
- 어느 실험이 가장 좋은지 비교 불가능

**모니터링 핵심 지표**:
| 지표 | 의미 | 정상 패턴 | 이상 패턴 |
|---|---|---|---|
| train_loss | 학습 손실 | 꾸준히 감소 | 발산(NaN/급등) |
| val_loss | 검증 손실 | train 따라 감소 | train 감소, val 증가 = 과적합 |
| lr | 현재 학습률 | 스케줄 따라 변화 | 갑작스런 0 = 스케줄러 버그 |
| grad_norm | Gradient 크기 | 0.1~2.0 사이 안정 | 15+ 반복 = 발산 위험 |
| gpu_util | GPU 사용률 | 70~95% | 30% 이하 = 데이터 병목 |
| gpu_memory | VRAM 사용량 | 80~90% | 95%+ = OOM 위험 |

### wandb — 클라우드 실험 추적

2019년 창립된 Weights & Biases (wandb). 현재 ML 실험 추적의 사실상 표준.

**핵심 기능**:
- **실시간 대시보드**: 학습 중 손실·지표 시각화
- **실험 비교**: 여러 run을 한 화면에서 비교
- **아티팩트 관리**: 모델 체크포인트, 데이터셋 버전 관리
- **Hyperparameter Sweep**: 자동 하이퍼파라미터 탐색

**무료 플랜**: 개인 100GB, 팀 공유 OK

**통합 3줄**:
```python
import wandb
wandb.init(project="geulsalim-ai", name="sdxl-lora-rank16")
wandb.log({"loss": loss, "lr": lr})
```

### TensorBoard — 로컬 모니터링

Google에서 개발. 서버에 데이터를 보내지 않는 로컬 모니터링.

**적합 상황**: 인터넷 연결 불안정, 데이터 보안 요구, 오프라인 환경

```bash
# 설치
pip install tensorboard

# 실행 (port 6006)
tensorboard --logdir ./runs --port 6006

# 브라우저: http://localhost:6006
```

### Early Stopping — 손실을 멈추는 기준

검증 손실이 n 에폭 동안 개선되지 않으면 학습 중단.

```
patience = 5   # 5 에폭 개선 없으면 중단
```

**고문서 복원 기준**:
- val_loss가 50 스텝 연속 개선 없으면 lr을 10배 낮추거나 중단
- SSIM이 0.85 이상 도달하면 추가 학습은 수확 체감 구간

## 실전 코드

```python
# wandb + TensorBoard 통합 학습 모니터링
# pip install wandb torch tensorboard

import os
import math
import time
import torch
import wandb
from torch.utils.tensorboard import SummaryWriter
from dataclasses import dataclass, asdict
from typing import Optional


# ──────────────────────────────────────────────
# 1. 학습 설정 (Config) 클래스
# ──────────────────────────────────────────────
@dataclass
class TrainConfig:
    """학습 설정 — wandb에 자동 기록"""
    project:         str   = "geulsalim-ai-restoration"
    run_name:        str   = "sdxl-lora-rank16-v1"
    
    # 모델
    base_model:      str   = "diffusers/stable-diffusion-xl-1.0-inpainting-0.1"
    lora_rank:       int   = 16
    lora_alpha:      int   = 32
    
    # 학습
    lr:              float = 1e-4
    batch_size:      int   = 2
    grad_accum:      int   = 2
    total_steps:     int   = 2000
    warmup_steps:    int   = 100
    weight_decay:    float = 1e-4
    
    # 데이터
    train_data_dir:  str   = "./data/train"
    val_data_dir:    str   = "./data/val"
    image_size:      int   = 1024
    
    # 체크포인트
    save_every:      int   = 200
    output_dir:      str   = "./checkpoints"
    
    @property
    def effective_batch(self):
        return self.batch_size * self.grad_accum


# ──────────────────────────────────────────────
# 2. 통합 Logger 클래스
# ──────────────────────────────────────────────
class TrainLogger:
    """wandb + TensorBoard 동시 로깅"""
    
    def __init__(self, config: TrainConfig, use_wandb: bool = True, use_tb: bool = True):
        self.config = config
        self.use_wandb = use_wandb
        self.use_tb = use_tb
        self.step = 0
        self.start_time = time.time()
        
        # wandb 초기화
        if use_wandb:
            wandb.init(
                project=config.project,
                name=config.run_name,
                config=asdict(config),
                tags=["sdxl", "lora", "inpainting", "historical-document"],
            )
            print(f"wandb 초기화: {wandb.run.url}")
        
        # TensorBoard 초기화
        if use_tb:
            tb_dir = os.path.join("runs", config.run_name)
            self.writer = SummaryWriter(log_dir=tb_dir)
            print(f"TensorBoard: tensorboard --logdir {tb_dir}")
        
        os.makedirs(config.output_dir, exist_ok=True)
    
    def log(self, metrics: dict, step: Optional[int] = None):
        """지표 동시 기록"""
        s = step or self.step
        
        # 경과 시간 추가
        elapsed = time.time() - self.start_time
        metrics["elapsed_min"] = elapsed / 60
        
        if self.use_wandb:
            wandb.log(metrics, step=s)
        
        if self.use_tb:
            for k, v in metrics.items():
                if isinstance(v, (int, float)):
                    self.writer.add_scalar(k, v, global_step=s)
        
        self.step = s + 1
    
    def log_image(self, tag: str, images: torch.Tensor, step: int):
        """이미지 비교 로깅 (입력 vs 복원 결과)"""
        # [-1,1] → [0,1]
        imgs = (images.clamp(-1, 1) + 1) / 2
        
        if self.use_wandb:
            wandb_imgs = [wandb.Image(img.permute(1,2,0).cpu().numpy())
                          for img in imgs[:4]]
            wandb.log({tag: wandb_imgs}, step=step)
        
        if self.use_tb:
            from torchvision.utils import make_grid
            grid = make_grid(imgs[:4], nrow=2, normalize=False)
            self.writer.add_image(tag, grid, global_step=step)
    
    def log_model_gradient_stats(self, model, step: int):
        """Gradient norm 분포 기록"""
        total_norm = 0.0
        layer_norms = {}
        
        for name, param in model.named_parameters():
            if param.grad is not None:
                norm = param.grad.data.norm(2).item()
                total_norm += norm ** 2
                if "lora" in name.lower():
                    layer_norms[f"grad/{name[:50]}"] = norm
        
        total_norm = math.sqrt(total_norm)
        
        metrics = {"grad_norm/total": total_norm, **layer_norms}
        self.log(metrics, step=step)
        
        # 경고
        if total_norm > 5.0:
            print(f"⚠️  Step {step}: Gradient norm 높음 ({total_norm:.2f}). 발산 주의!")
    
    def finish(self):
        if self.use_wandb:
            wandb.finish()
        if self.use_tb:
            self.writer.close()


# ──────────────────────────────────────────────
# 3. Early Stopping
# ──────────────────────────────────────────────
class EarlyStopping:
    """val_loss 기반 Early Stopping"""
    
    def __init__(self, patience: int = 5, min_delta: float = 1e-4):
        self.patience = patience
        self.min_delta = min_delta
        self.counter = 0
        self.best_loss = float("inf")
        self.should_stop = False
    
    def __call__(self, val_loss: float) -> bool:
        if val_loss < self.best_loss - self.min_delta:
            self.best_loss = val_loss
            self.counter = 0
        else:
            self.counter += 1
            print(f"[EarlyStopping] 개선 없음 {self.counter}/{self.patience}")
            if self.counter >= self.patience:
                self.should_stop = True
                print("조기 종료 트리거됨")
        return self.should_stop


# ──────────────────────────────────────────────
# 4. 학습 루프 스켈레톤
# ──────────────────────────────────────────────
def training_loop_skeleton(config: TrainConfig):
    """
    실제 학습 루프의 구조만 표시 (실제 모델 없이 실행 가능)
    wandb/TensorBoard 통합 패턴을 이해하기 위한 예시
    """
    logger = TrainLogger(config, use_wandb=False, use_tb=False)  # 예시: 둘 다 끔
    early_stop = EarlyStopping(patience=5)
    
    # 가상 학습 루프
    for step in range(min(config.total_steps, 20)):  # 예시: 20 스텝만
        # ── 시뮬레이션 지표 ──────────────────
        import random
        train_loss = max(0.01, 1.0 * math.exp(-step * 0.1) + random.gauss(0, 0.02))
        val_loss   = max(0.01, 1.1 * math.exp(-step * 0.09) + random.gauss(0, 0.03))
        lr         = 1e-4 * (0.5 + 0.5 * math.cos(math.pi * step / config.total_steps))
        grad_norm  = 0.5 + random.gauss(0, 0.1)
        
        # ── 로깅 ─────────────────────────────
        logger.log({
            "train/loss":    train_loss,
            "val/loss":      val_loss,
            "train/lr":      lr,
            "train/grad_norm": abs(grad_norm),
        }, step=step)
        
        if step % 5 == 0:
            print(f"Step {step:4d} | train_loss={train_loss:.4f} | "
                  f"val_loss={val_loss:.4f} | lr={lr:.2e}")
        
        # ── 체크포인트 저장 ───────────────────
        if step % config.save_every == 0 and step > 0:
            ckpt_path = os.path.join(config.output_dir, f"step_{step:05d}")
            print(f"체크포인트 저장: {ckpt_path}")
        
        # ── Early Stopping ────────────────────
        if step % 5 == 0:
            if early_stop(val_loss):
                print(f"Step {step}에서 조기 종료")
                break
    
    logger.finish()
    print("\n학습 루프 완료")


if __name__ == "__main__":
    config = TrainConfig(
        run_name="test-run-01",
        total_steps=100,
        save_every=50,
    )
    training_loop_skeleton(config)
    
    print("\n실제 wandb 사용 시:")
    print("  1. pip install wandb")
    print("  2. wandb login  (브라우저에서 API 키 복사)")
    print("  3. TrainLogger(config, use_wandb=True) 로 변경")
    print("  4. https://wandb.ai 에서 실시간 대시보드 확인")
```

## 함정 카드 — 자주 실수하는 5가지

1. **"wandb.log() 매 스텝마다 호출한다"** → 고빈도 로깅은 I/O 오버헤드로 학습 속도를 느리게 한다. 매 10~50 스텝마다 로깅이 적당. 이미지 로깅은 더 드물게 (매 100~500 스텝).

2. **"val_loss를 train set으로 계산한다"** → 반드시 학습에 사용하지 않은 **검증 세트**로 val_loss 계산. train_loss만 보면 과적합 감지 불가. 데이터셋의 10~20%를 검증 세트로 분리.

3. **"체크포인트를 너무 드물게 저장한다"** → GPU 인스턴스는 예고 없이 종료될 수 있다 (Spot Instance). 최소 500 스텝마다 저장. wandb Artifacts로 클라우드 백업도 병행.

4. **"GPU 사용률을 확인하지 않는다"** → `nvidia-smi`로 GPU 사용률이 70% 미만이면 데이터 파이프라인 개선 필요. wandb의 System Metrics(자동 기록)에서 GPU 사용률 히스토그램으로 확인 가능.

5. **"학습 곡선의 노이즈를 이상으로 판단한다"** → 소배치 학습에서 train_loss의 스텝별 노이즈는 정상. **이동 평균** (smoothing factor 0.6~0.9)으로 트렌드를 봐야 한다. wandb 대시보드의 Smoothing 슬라이더 활용.

## 연습문제

**문제 1.** wandb에서 두 실험 run을 비교할 때, val_loss 이외에 어떤 지표를 추가로 기록하면 고문서 복원 실험 비교에 도움이 되는가? 3가지 제시하라.

> **해설:** ① `val/ssim` — 구조적 유사도로 복원 품질 직접 측정 ② `val/lpips` — 지각적 유사도 (낮을수록 좋음) ③ `examples/restored_vs_target` — 실제 복원 이미지 vs 정답 이미지 나란히 기록. 숫자만으로는 실제 복원 결과가 인간에게 얼마나 자연스러운지 알 수 없으므로 시각적 비교가 필수.

**문제 2.** Early Stopping patience=5로 설정했지만, 실제 val_loss가 plateau(정체) 후 다시 개선되는 경우가 있다. 이를 처리하기 위해 EarlyStopping 클래스에 어떤 기능을 추가하겠는가?

> **해설:** ① **ReduceLROnPlateau 통합**: patience 절반 도달 시 lr을 10배 줄이고 counter 리셋. ② **최소 학습 스텝 보장**: `min_steps` 파라미터를 추가하여 초반 plateau를 무시. ③ **Delta 기반 판단 완화**: 개선 기준 `min_delta`를 처음에는 작게 설정하고 단계적으로 줄이는 adaptive 버전.

**문제 3.** TensorBoard에서 `add_image()` 호출 시 텐서 형태가 잘못되어 에러가 발생했다. `torch.Size([4, 3, 1024, 1024])`를 `add_image()`에 올바르게 전달하는 코드를 작성하라.

> **해설:** `add_image()`는 단일 이미지 `(C, H, W)` 또는 그리드를 기대. 배치를 그리드로 변환: `from torchvision.utils import make_grid; grid = make_grid(tensor[:4], nrow=2, normalize=True, value_range=(-1,1)); writer.add_image("tag", grid, step)`. `add_images()`(복수형)을 사용하면 배치 전체 `(N, C, H, W)`를 직접 전달 가능.
