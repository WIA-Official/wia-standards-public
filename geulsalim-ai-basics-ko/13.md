# 챕터 13 · 하이퍼파라미터 기초 — lr · batch · scheduler

## 학습 목표
- 학습률(lr)이 모델 수렴에 미치는 영향을 직관적으로 이해한다
- 배치 크기와 학습률의 관계(linear scaling rule)를 적용할 수 있다
- 고문서 복원 SDXL 파인튜닝에 최적화된 하이퍼파라미터 레시피를 적용한다

---

## 이론

### 하이퍼파라미터란

모델이 학습 과정에서 스스로 조정하는 **파라미터(가중치, 편향)**와 달리, 학습 시작 전 사람이 설정해야 하는 값을 **하이퍼파라미터**라 한다.

"연구자는 모델에게 올바른 환경을 제공할 책임이 있다." — 잘못된 학습률 하나가 수십만 원의 GPU 비용을 낭비시킨다.

### 학습률 (Learning Rate, lr)

가중치를 얼마나 크게 갱신할지 결정하는 스칼라.

```
w_new = w_old - lr × gradient
```

**lr이 너무 크면**: 손실이 발산(NaN) 또는 진동
**lr이 너무 작으면**: 수렴이 지나치게 느림 (학습 비용 증가)
**최적 lr**: 손실이 꾸준히 감소, 최솟값 근방에서 안정

**SDXL LoRA 파인튜닝 권장 lr**:
- UNet: `1e-4` ~ `5e-5` (낮은 lr로 사전 학습 파괴 방지)
- Text Encoder: `1e-5` ~ `5e-6` (더 민감하므로 더 낮게)
- LoRA rank 높을수록 lr 낮춰야 안정적

### Warmup — 학습 초반 안정화

학습 첫 단계에서 lr을 0에서 목표 lr까지 서서히 올린다.

**왜 필요한가**: 초기 무작위 LoRA 가중치가 큰 gradient를 만들어 기반 모델을 손상시킬 수 있다. Warmup이 초기 충격 완화.

권장 warmup 스텝: 전체 학습 스텝의 5~10%
```
steps=1000이면 warmup_steps=50~100
```

### 배치 크기 (Batch Size)

한 번의 gradient 업데이트에 사용하는 이미지 수.

**큰 배치**: 안정적 gradient 추정, 더 빠른 수렴 (에폭 당), VRAM 많이 필요
**작은 배치**: VRAM 절약, 더 시끄러운 gradient (과적합 방지 효과 있음)

**Linear Scaling Rule** (2017, Facebook):
> 배치 크기를 k배 키우면 학습률도 k배 올려야 동일한 수렴

예: 배치 4 + lr 1e-4 → 배치 8 + lr 2e-4

**Gradient Accumulation**: VRAM 부족 시 배치를 나눠 gradient를 누적한 후 한 번에 업데이트
```python
# 유효 배치 크기 = 배치 크기 × gradient_accumulation_steps
batch_size=2, gradient_accumulation_steps=4 → 유효 배치=8
```

### Learning Rate Scheduler

학습 진행에 따라 lr을 자동 조정.

**Cosine Annealing** (현재 최대 권장):
```
lr_t = lr_min + (lr_max - lr_min) × (1 + cos(π × t/T)) / 2
```
- t: 현재 스텝, T: 전체 스텝
- 처음에는 높고, 끝으로 갈수록 낮아짐
- 마지막에 다시 올라가는 Cosine Restart 변형도 존재

**Linear**: 선형 감소. 단순하지만 효과적
**Constant**: lr 고정. Warmup 이후 일정하게 유지

**SDXL LoRA 권장**: `cosine_with_restarts` 또는 `linear`

### Weight Decay — 과적합 방지

```
w_new = w_old × (1 - weight_decay) - lr × gradient
```
- 가중치 크기를 지속적으로 줄여 과적합 방지
- 권장: `1e-4` ~ `1e-2`
- AdamW optimizer가 weight decay를 올바르게 구현 (Adam은 잘못 구현됨)

## 실전 코드

```python
# 하이퍼파라미터 설정 및 스케줄러 구현
# pip install torch diffusers transformers

import torch
from torch.optim import AdamW
from torch.optim.lr_scheduler import (
    CosineAnnealingLR,
    CosineAnnealingWarmRestarts,
    LinearLR,
    SequentialLR,
)
import math


# ──────────────────────────────────────────────
# 1. SDXL LoRA 파인튜닝 기준 하이퍼파라미터 레시피
# ──────────────────────────────────────────────
SDXL_LORA_RECIPES = {
    "minimal_8gb": {
        "lr":                       5e-5,
        "batch_size":               1,
        "gradient_accumulation":    4,   # 유효 배치 = 4
        "effective_batch":          4,
        "warmup_steps":             50,
        "total_steps":              1000,
        "weight_decay":             1e-4,
        "scheduler":                "cosine",
        "lora_rank":                4,
        "vram":                     "8GB",
    },
    "standard_12gb": {
        "lr":                       1e-4,
        "batch_size":               2,
        "gradient_accumulation":    2,   # 유효 배치 = 4
        "effective_batch":          4,
        "warmup_steps":             100,
        "total_steps":              2000,
        "weight_decay":             1e-4,
        "scheduler":                "cosine",
        "lora_rank":                16,
        "vram":                     "12GB",
    },
    "quality_24gb": {
        "lr":                       1e-4,
        "batch_size":               4,
        "gradient_accumulation":    2,   # 유효 배치 = 8
        "effective_batch":          8,
        "warmup_steps":             200,
        "total_steps":              5000,
        "weight_decay":             1e-4,
        "scheduler":                "cosine_with_restarts",
        "lora_rank":                32,
        "vram":                     "24GB",
    },
}

def print_recipe(name: str):
    r = SDXL_LORA_RECIPES[name]
    print(f"\n[{name}] 하이퍼파라미터 레시피")
    print("-" * 40)
    for k, v in r.items():
        print(f"  {k:<28} {v}")


# ──────────────────────────────────────────────
# 2. Warmup + Cosine 스케줄러 조합
# ──────────────────────────────────────────────
def build_scheduler(
    optimizer: AdamW,
    warmup_steps: int,
    total_steps: int,
    scheduler_type: str = "cosine",
    n_restarts: int = 3,
):
    """
    Warmup → Main Scheduler 조합
    1. LinearLR: 0 → lr (warmup_steps 동안)
    2. CosineAnnealingLR 또는 CosineAnnealingWarmRestarts
    """
    warmup_scheduler = LinearLR(
        optimizer,
        start_factor=1e-7,    # 매우 작은 값에서 시작
        end_factor=1.0,
        total_iters=warmup_steps,
    )
    
    main_steps = total_steps - warmup_steps
    
    if scheduler_type == "cosine":
        main_scheduler = CosineAnnealingLR(
            optimizer,
            T_max=main_steps,
            eta_min=1e-7,
        )
    elif scheduler_type == "cosine_with_restarts":
        main_scheduler = CosineAnnealingWarmRestarts(
            optimizer,
            T_0=main_steps // n_restarts,
            T_mult=1,
            eta_min=1e-7,
        )
    else:  # linear
        main_scheduler = LinearLR(
            optimizer,
            start_factor=1.0,
            end_factor=0.0,
            total_iters=main_steps,
        )
    
    combined = SequentialLR(
        optimizer,
        schedulers=[warmup_scheduler, main_scheduler],
        milestones=[warmup_steps],
    )
    
    return combined


# ──────────────────────────────────────────────
# 3. 학습률 시각화
# ──────────────────────────────────────────────
def visualize_lr_schedule(
    lr: float = 1e-4,
    warmup_steps: int = 100,
    total_steps: int = 2000,
    scheduler_type: str = "cosine",
):
    """학습률 스케줄 시뮬레이션 (GPU 불필요)"""
    # 더미 파라미터
    dummy_param = torch.nn.Parameter(torch.zeros(1))
    optimizer = AdamW([dummy_param], lr=lr, weight_decay=1e-4)
    scheduler = build_scheduler(optimizer, warmup_steps, total_steps, scheduler_type)
    
    lrs = []
    for step in range(total_steps):
        lrs.append(optimizer.param_groups[0]["lr"])
        optimizer.zero_grad()
        dummy_param.grad = torch.ones(1)
        optimizer.step()
        scheduler.step()
    
    # 텍스트 기반 시각화
    key_steps = [0, warmup_steps//2, warmup_steps,
                 total_steps//4, total_steps//2, total_steps-1]
    print(f"\n학습률 스케줄 ({scheduler_type})")
    print("-" * 40)
    for s in key_steps:
        bar = "█" * int(lrs[s] / lr * 20)
        print(f"Step {s:5d}: {lrs[s]:.2e} {bar}")
    
    return lrs


# ──────────────────────────────────────────────
# 4. AdamW 옵티마이저 + Gradient Clipping
# ──────────────────────────────────────────────
def build_optimizer_with_clipping(model, recipe_name: str = "standard_12gb"):
    """
    AdamW + Gradient Clipping 설정
    Gradient Clipping: gradient norm이 max_norm을 초과하면 스케일링
    → 초기 학습에서 폭발적 gradient 방지
    """
    recipe = SDXL_LORA_RECIPES[recipe_name]
    
    optimizer = AdamW(
        [p for p in model.parameters() if p.requires_grad],
        lr=recipe["lr"],
        betas=(0.9, 0.999),    # Adam 모멘텀 파라미터 (기본값 권장)
        eps=1e-8,
        weight_decay=recipe["weight_decay"],
    )
    
    def training_step_with_clip(loss, max_norm: float = 1.0):
        """학습 스텝 + gradient clipping"""
        optimizer.zero_grad()
        loss.backward()
        
        # Gradient norm 계산
        total_norm = 0.0
        for p in model.parameters():
            if p.grad is not None:
                total_norm += p.grad.data.norm(2).item() ** 2
        total_norm = math.sqrt(total_norm)
        
        # Clipping
        torch.nn.utils.clip_grad_norm_(
            [p for p in model.parameters() if p.requires_grad],
            max_norm=max_norm
        )
        
        optimizer.step()
        return total_norm
    
    return optimizer, training_step_with_clip


if __name__ == "__main__":
    for recipe_name in SDXL_LORA_RECIPES:
        print_recipe(recipe_name)
    
    print("\n학습률 스케줄 시뮬레이션")
    lrs = visualize_lr_schedule(
        lr=1e-4,
        warmup_steps=100,
        total_steps=2000,
        scheduler_type="cosine"
    )
    print(f"\n최고 lr: {max(lrs):.2e} (step {lrs.index(max(lrs))})")
    print(f"최저 lr: {min(lrs):.2e} (step {lrs.index(min(lrs))})")
```

## 함정 카드 — 자주 실수하는 5가지

1. **"lr=1e-3으로 LoRA를 학습한다"** → LoRA는 기반 모델 위에 올라타는 구조라 높은 lr은 기반 모델을 파괴한다. `1e-4`를 상한으로 보수적으로 시작. 손실이 발산하면 10배 낮춰라.

2. **"Adam을 쓰고 weight_decay를 설정한다"** → `torch.optim.Adam`은 weight decay를 L2 정규화로 잘못 구현한다. 반드시 `torch.optim.AdamW`를 사용해야 correct weight decay가 적용된다. 이 차이는 장기 학습에서 성능 차이를 유발한다.

3. **"배치 크기 2개로 학습한다"** → 배치 2는 BN 통계가 불안정하다. SDXL은 BN 없이 GroupNorm을 쓰지만, 배치 크기가 작으면 gradient 분산이 커진다. `gradient_accumulation_steps`로 유효 배치를 4 이상 확보 권장.

4. **"스케줄러 step()을 optimizer step() 전에 호출한다"** → PyTorch에서는 `optimizer.step()` 이후에 `scheduler.step()`을 호출해야 한다. 순서 잘못되면 첫 스텝 lr이 이미 업데이트된 상태로 시작.

5. **"warmup 없이 1e-4로 학습 시작한다"** → 초기 LoRA 가중치가 모두 0 근처이므로 첫 few 스텝에서 gradient가 불규칙적으로 크다. Warmup 없으면 기반 모델의 중요한 가중치가 초반에 손상될 수 있다.

## 연습문제

**문제 1.** "유효 배치 크기 8, lr=2e-4"로 실험하고 싶지만 VRAM이 4GB뿐이다. 배치 크기 1로 동등한 학습 설정을 만들려면 어떻게 해야 하는가?

> **해설:** `batch_size=1, gradient_accumulation_steps=8`로 설정. 유효 배치 = 1 × 8 = 8. 단, Linear Scaling Rule에 따라 lr도 조정 가능: 유효 배치 8 기준으로 `lr = 2e-4`는 유지. 실제 각 mini-batch step에서 `loss / gradient_accumulation_steps`로 나누어 평균 gradient를 유지해야 함. 8 스텝마다 한 번 `optimizer.step()`과 `scheduler.step()` 호출.

**문제 2.** Cosine Annealing 스케줄러에서 `T_max=1000`으로 설정했는데 실제 학습이 1500 스텝 필요하다면 어떤 일이 생기는가?

> **해설:** Step 1000 이후 lr이 `eta_min`(거의 0)에 도달한 후 더 이상 감소하지 않고 `eta_min`에 머문다. 사실상 1000~1500 스텝은 거의 학습이 되지 않는 셈. 해결: ① `T_max=1500`으로 재설정 ② `CosineAnnealingWarmRestarts`로 주기적 재시작 ③ 남은 500 스텝은 `eta_min`을 약간 높게(1e-6) 설정.

**문제 3.** Gradient norm clipping을 `max_norm=1.0`으로 설정했다. 학습 중 gradient norm이 주로 0.1 ~ 0.3인데, 특정 배치에서 갑자기 15.0까지 올라갔다. 이 현상의 원인과 대응 방법을 설명하라.

> **해설:** Gradient spike 원인: ① 이상치 이미지(outlier) — 손상이 극심한 이미지가 loss를 폭증 ② 마스크 비율이 매우 높은 샘플 — 복원 면적이 커서 loss × area 증가. 대응: ① 데이터 전처리에서 극단적 샘플 제거 (마스크 비율 5~40% 범위만 허용) ② `max_norm=1.0` 클리핑이 정확히 이 케이스를 처리함 (15.0 → 1.0으로 스케일링) ③ 학습 로그에서 norm > 3.0 이상 빈번하면 lr을 낮추거나 데이터 전처리 개선.
