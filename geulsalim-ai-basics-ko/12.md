## 학습 목표

- 고문서 복원 특화 평가 지표(SSIM, LPIPS)의 필요성과 사용법을 이해한다

## 이론

### 왜 "눈으로 보기"만으로는 부족한가

파인튜닝 실험을 할 때마다 "이번이 더 좋은 것 같다"는 주관적 판단은 재현 불가능하고 비교 불가능하다. 특히:
- 100개 실험을 비교할 때 육안 평가는 한계
- 팀원 간 품질 기준 통일 불가능
- 모델 버전 관리 시 객관적 기준 필요

따라서 **정량적 지표**가 반드시 필요하다.

### FID — Fréchet Inception Distance

2017년 제안. 현재 이미지 생성 품질 평가의 **업계 표준**.

**원리**:
1. 실제 이미지와 생성 이미지를 각각 Inception-v3 모델에 통과
2. 마지막 레이어(2048차원 피처) 추출
3. 두 피처 분포를 가우시안으로 가정하고 평균(μ)과 공분산(Σ) 계산
4. 두 가우시안 사이의 Fréchet distance 계산:

```
FID = ||μ_real - μ_fake||² + Tr(Σ_real + Σ_fake - 2√(Σ_real × Σ_fake))
```

**해석**:
- **낮을수록 좋음**: 0에 가까울수록 생성 이미지가 실제와 유사
- SDXL 기반: 일반 이미지에서 FID ~5~15 수준이 우수
- 고문서 복원: 도메인 특화이므로 **절대값보다 상대적 개선도**가 중요

**주의사항**:
- 최소 **2,048개** 이미지가 있어야 통계적으로 신뢰 가능
- 이미지 전처리(리사이즈 방법, 정규화)에 따라 FID가 달라짐
- 동일 전처리 파이프라인으로 비교해야 의미 있음

### CLIP Score — 텍스트-이미지 정렬 측정

2021년 OpenAI CLIP 모델 기반.

**원리**:
- 생성된 이미지를 CLIP 비전 인코더로 임베딩
- 프롬프트 텍스트를 CLIP 텍스트 인코더로 임베딩
- 두 임베딩의 코사인 유사도 × 100

**값 범위**: 0~100 (높을수록 텍스트-이미지 정렬 우수)
- 35 이상: 양호
- 25 이하: 프롬프트를 잘 따르지 못함

**고문서 복원에서의 활용**:
- 프롬프트: `"historical Korean document, clear ink text, aged paper"`
- 복원 전 vs 후 CLIP Score 비교
- 복원이 프롬프트 방향으로 개선됐는지 확인

### SSIM — 구조적 유사도 (고문서 복원 특화)

FID는 생성 품질을 보지만, **복원 작업**은 "원본과 얼마나 가까운가"가 핵심이다.

**SSIM (Structural Similarity Index)**:
- 밝기(luminance) · 대비(contrast) · 구조(structure) 세 요소 측정
- 값 범위: -1 ~ 1 (1 = 완벽히 동일)
- 인간 시각 시스템과 상관관계가 높음

### LPIPS — 딥러닝 기반 지각 유사도

**LPIPS (Learned Perceptual Image Patch Similarity)**:
- AlexNet/VGG의 중간 레이어 피처로 유사도 계산
- 픽셀 단위 MSE보다 인간 판단과 훨씬 높은 상관관계
- 값 범위: 0 ~ 1 (0 = 완벽히 동일)
- 고문서 복원에서 SSIM보다 신뢰도 높음

### 고문서 복원 평가 대시보드

| 지표 | 목적 | 좋은 값 | 학습 중 사용 |
|---|---|---|---|
| Loss (MSE) | 픽셀 수준 복원 | 최소화 | ✅ 매 배치 |
| SSIM | 구조 보존 | 최대화 (→1) | ✅ 매 에폭 |
| LPIPS | 지각 품질 | 최소화 (→0) | ✅ 매 에폭 |
| FID | 분포 품질 | 최소화 | ⚠️ 학습 후 |
| CLIP Score | 프롬프트 정렬 | 최대화 (→100) | ⚠️ 학습 후 |
| 전문가 점수 | 역사 정확성 | 5점 만점 | 최종 검수 |

## 실전 코드

```python
# 이미지 생성/복원 품질 평가 종합
# pip install torchmetrics[image] lpips Pillow torch

import torch
import numpy as np
from PIL import Image
from pathlib import Path
from torchmetrics.image.fid import FrechetInceptionDistance
from torchmetrics.image import StructuralSimilarityIndexMeasure
from torchmetrics.functional.multimodal import clip_score
import torchvision.transforms as T


# ──────────────────────────────────────────────
# 1. 이미지 로드 헬퍼
# ──────────────────────────────────────────────
def load_image_tensor(
    path: str,
    size: int = 299,  # Inception-v3 입력 크기
    normalize: bool = True,
) -> torch.Tensor:
    """
    이미지 → [0,255] uint8 텐서 (FID용) 또는 [-1,1] float 텐서 (CLIP/SSIM용)
    """
    img = Image.open(path).convert("RGB").resize((size, size), Image.LANCZOS)
    arr = np.array(img)
    tensor = torch.from_numpy(arr).permute(2, 0, 1)  # (3, H, W)
    
    if normalize:
        tensor = tensor.float() / 255.0  # [0, 1]
    
    return tensor.unsqueeze(0)  # (1, 3, H, W)


def load_image_batch(
    directory: str,
    max_images: int = 2048,
    size: int = 299,
) -> torch.Tensor:
    """디렉토리의 이미지들을 배치로 로드"""
    paths = list(Path(directory).glob("*.png"))[:max_images]
    tensors = []
    for p in paths:
        try:
            t = load_image_tensor(str(p), size=size, normalize=False)
            tensors.append(t)
        except Exception:
            continue
    return torch.cat(tensors, dim=0)  # (N, 3, H, W)


# ──────────────────────────────────────────────
# 2. FID 계산
# ──────────────────────────────────────────────
def compute_fid(
    real_dir: str,
    fake_dir: str,
    device: str = "cpu",
    n_images: int = 2048,
) -> float:
    """
    FID 계산 — 최소 2048개 이미지 권장
    real_dir: 실제(ground truth) 이미지 폴더
    fake_dir: 생성/복원된 이미지 폴더
    """
    fid = FrechetInceptionDistance(feature=2048, normalize=True).to(device)
    
    print(f"실제 이미지 로드 중: {real_dir}")
    real_imgs = load_image_batch(real_dir, max_images=n_images).to(device)
    fid.update(real_imgs, real=True)
    
    print(f"생성 이미지 로드 중: {fake_dir}")
    fake_imgs = load_image_batch(fake_dir, max_images=n_images).to(device)
    fid.update(fake_imgs, real=False)
    
    score = float(fid.compute())
    print(f"FID: {score:.2f} (낮을수록 좋음)")
    return score


# ──────────────────────────────────────────────
# 3. CLIP Score 계산
# ──────────────────────────────────────────────
def compute_clip_score_batch(
    image_dir: str,
    prompt: str = "historical Korean document, clear ink brushstroke, aged paper texture",
    device: str = "cpu",
    n_images: int = 100,
) -> float:
    """
    CLIP Score: 이미지-텍스트 정렬 측정
    clip_score()는 torchmetrics 0.11+ 필요
    """
    paths = list(Path(image_dir).glob("*.png"))[:n_images]
    
    transform = T.Compose([
        T.Resize((224, 224)),
        T.ToTensor(),
    ])
    
    images = []
    for p in paths:
        try:
            img = Image.open(p).convert("RGB")
            t = transform(img)
            images.append(t)
        except Exception:
            continue
    
    if not images:
        return 0.0
    
    batch = torch.stack(images).to(device)  # (N, 3, 224, 224)
    prompts = [prompt] * len(images)
    
    score = clip_score(batch, prompts, model_name_or_path="openai/clip-vit-base-patch32")
    result = float(score)
    
    print(f"CLIP Score: {result:.2f} (높을수록 좋음, 기준: 35+)")
    return result


# ──────────────────────────────────────────────
# 4. SSIM + LPIPS 계산 (복원 품질)
# ──────────────────────────────────────────────
def compute_restoration_metrics(
    restored_dir: str,
    target_dir: str,
    device: str = "cpu",
) -> dict:
    """
    복원 이미지 vs 정답 이미지 비교
    SSIM: 구조적 유사도
    LPIPS: 지각 유사도
    PSNR: 픽셀 단위 신호 대 잡음비
    """
    try:
        import lpips
        lpips_fn = lpips.LPIPS(net="vgg").to(device)
    except ImportError:
        print("pip install lpips 필요")
        lpips_fn = None
    
    ssim_fn = StructuralSimilarityIndexMeasure(data_range=1.0).to(device)
    
    restored_paths = sorted(Path(restored_dir).glob("*.png"))
    target_paths   = sorted(Path(target_dir).glob("*.png"))
    
    ssim_scores, lpips_scores, psnr_scores = [], [], []
    
    transform = T.Compose([T.Resize((512, 512)), T.ToTensor()])
    
    for r_path, t_path in zip(restored_paths[:100], target_paths[:100]):
        try:
            r = transform(Image.open(r_path).convert("RGB")).unsqueeze(0).to(device)
            t = transform(Image.open(t_path).convert("RGB")).unsqueeze(0).to(device)
            
            # SSIM
            ssim_val = float(ssim_fn(r, t))
            ssim_scores.append(ssim_val)
            
            # PSNR (MSE 기반)
            mse = float(torch.mean((r - t) ** 2))
            psnr = 10 * np.log10(1.0 / mse) if mse > 0 else 100.0
            psnr_scores.append(psnr)
            
            # LPIPS
            if lpips_fn:
                r_lpips = r * 2 - 1  # [0,1] → [-1,1]
                t_lpips = t * 2 - 1
                lpips_val = float(lpips_fn(r_lpips, t_lpips))
                lpips_scores.append(lpips_val)
        except Exception:
            continue
    
    results = {
        "ssim":  np.mean(ssim_scores)  if ssim_scores  else 0.0,
        "psnr":  np.mean(psnr_scores)  if psnr_scores  else 0.0,
        "lpips": np.mean(lpips_scores) if lpips_scores else 0.0,
    }
    
    print("복원 품질 지표:")
    print(f"  SSIM:  {results['ssim']:.4f} (기준: 0.8+)")
    print(f"  PSNR:  {results['psnr']:.1f} dB (기준: 28+)")
    print(f"  LPIPS: {results['lpips']:.4f} (기준: 0.1 이하)")
    
    return results


# ──────────────────────────────────────────────
# 5. 학습 중 빠른 SSIM 평가 (매 에폭)
# ──────────────────────────────────────────────
class EpochMetrics:
    """학습 루프 내 지표 누적 및 출력"""
    
    def __init__(self, device="cuda"):
        self.ssim = StructuralSimilarityIndexMeasure(data_range=1.0).to(device)
        self.reset()
    
    def reset(self):
        self.losses = []
        self.ssim.reset()
    
    def update(self, pred: torch.Tensor, target: torch.Tensor, loss: float):
        self.losses.append(loss)
        # [0,1]로 정규화 후 SSIM
        pred_norm   = (pred.clamp(-1, 1) + 1) / 2
        target_norm = (target.clamp(-1, 1) + 1) / 2
        self.ssim.update(pred_norm, target_norm)
    
    def compute(self) -> dict:
        return {
            "loss": np.mean(self.losses),
            "ssim": float(self.ssim.compute()),
        }
    
    def log(self, epoch: int):
        metrics = self.compute()
        print(f"[Epoch {epoch:03d}] Loss: {metrics['loss']:.4f} | SSIM: {metrics['ssim']:.4f}")
        return metrics


if __name__ == "__main__":
    print("평가 지표 요약")
    print("=" * 50)
    
    guide = [
        ("FID",         "낮을수록 좋음", "< 15: 우수, < 30: 양호"),
        ("CLIP Score",  "높을수록 좋음", "> 35: 우수, > 25: 양호"),
        ("SSIM",        "높을수록 좋음", "> 0.90: 우수, > 0.80: 양호"),
        ("PSNR",        "높을수록 좋음", "> 32dB: 우수, > 28dB: 양호"),
        ("LPIPS",       "낮을수록 좋음", "< 0.05: 우수, < 0.10: 양호"),
    ]
    for name, direction, standard in guide:
        print(f"{name:<12} {direction:<15} {standard}")
```

## 함정 카드 — 자주 실수하는 5가지

1. **"FID를 샘플 100개로 계산한다"** → FID는 **최소 2,048개, 권장 10,000개** 이상의 이미지가 필요하다. 샘플이 적으면 분산이 커서 의미 없는 숫자가 나온다. `torchmetrics FrechetInceptionDistance`가 경고를 출력하니 무시하지 말 것.

2. **"서로 다른 전처리로 FID를 비교한다"** → 리사이즈 방법(LANCZOS vs BILINEAR), 정규화 범위([0,255] vs [0,1])가 다르면 FID가 크게 달라진다. 모든 실험에서 **동일한 전처리 파이프라인** 사용 필수.

3. **"CLIP Score로 복원 품질을 판단한다"** → CLIP Score는 텍스트-이미지 정렬을 보는 것이지, 복원 정확도를 보는 게 아니다. 고문서 복원 정확도는 SSIM과 LPIPS로 판단. 용도 혼동 주의.

4. **"SSIM=1이면 완벽하다고 생각한다"** → SSIM은 밝기와 구조를 보지만, 고문서의 경우 원본 텍스트의 의미적 정확성(문자 판독 가능 여부)은 별도의 OCR 정확도 지표로 측정해야 한다.

5. **"지표 하나로만 판단한다"** → FID가 좋지만 LPIPS가 나쁜 경우가 자주 있다. 복원 품질은 **SSIM + LPIPS + 전문가 시각 평가** 세 가지를 복합 사용해야 신뢰할 수 있다.

## 연습문제

**문제 1.** 파인튜닝 실험 A (FID=12, CLIP=38, SSIM=0.82)와 실험 B (FID=18, CLIP=42, SSIM=0.88) 중 고문서 복원 목적에 어느 실험이 더 적합한가? 이유를 설명하라.

> **해설:** **실험 B 권장**. 고문서 복원의 핵심 지표는 SSIM(원본 구조 보존)과 CLIP Score(복원 방향). SSIM 0.88이 0.82보다 원본 문서 구조를 더 잘 보존. CLIP Score 42가 더 높아 "역사 문서" 프롬프트 방향으로 복원이 더 정확. FID가 낮은 A가 전반적 이미지 품질은 좋지만, 복원 작업에서는 원본 충실도가 더 중요.

**문제 2.** FID 계산 시 `normalize=True` 파라미터의 역할을 설명하라. `normalize=False`로 설정하면 어떤 에러가 발생할 수 있는가?

> **해설:** `normalize=True`는 입력 이미지가 [0,1] float 범위임을 명시. 내부적으로 Inception-v3 입력을 위해 적절히 변환. `normalize=False`는 [0,255] uint8 기대. 이를 혼동하면 Inception 피처가 완전히 다른 분포를 갖게 되어 FID가 의미 없는 값(수백~수천)이 나온다. 에러 메시지 없이 잘못된 값이 출력되므로 주의.

**문제 3.** LPIPS가 SSIM보다 고문서 복원 평가에 더 적합한 이유를 설명하라. 어떤 경우에 두 지표가 반대 방향을 가리킬 수 있는가?

> **해설:** SSIM은 픽셀 블록의 밝기·대비·구조를 보지만, 고문서의 붓글씨처럼 얇고 복잡한 선은 SSIM이 놓칠 수 있다. LPIPS는 VGG/AlexNet이 학습한 인간 시각의 perceptual feature를 사용하여 섬세한 질감 차이를 더 잘 잡는다. 반대 방향 예: 원본 문서의 전체 밝기가 조금 달라진 경우 — SSIM은 크게 하락하지만(밝기 민감), LPIPS는 텍스처 구조가 유지되면 낮게 유지.
