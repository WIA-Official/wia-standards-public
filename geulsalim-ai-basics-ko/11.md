# 챕터 11 · WebDataset · Parquet · HDF5 — 대용량 데이터 실전

## 학습 목표
- WebDataset · Parquet · HDF5 세 포맷의 특성과 적합한 사용 시나리오를 설명할 수 있다
- GPU 학습 중 데이터 로딩 병목(bottleneck)을 진단하고 해결할 수 있다
- 고문서 복원 데이터셋을 WebDataset tar 포맷으로 변환하고 학습에 연결할 수 있다

## 이론

### 데이터 로딩 병목 문제

A100 GPU는 초당 312 TFLOPS를 처리한다. 하지만 학습 중 GPU 사용률이 30~50%에 머문다면? 대부분 **데이터 로딩 병목** 때문이다.

GPU가 배치를 처리하는 동안 CPU가 다음 배치를 준비해야 한다. 이 준비가 늦으면 GPU가 멈추고 대기한다. 낭비되는 컴퓨팅 = 낭비되는 돈.

**병목 진단**:
```bash
# 학습 중 GPU 사용률 모니터링
watch -n 1 nvidia-smi

# 70% 이상 유지가 이상적
# 30% 이하면 데이터 파이프라인 개선 필요
```

### 포맷 비교: 언제 무엇을 쓰는가

| 포맷 | 구조 | 최적 시나리오 | HDFS 지원 | 스트리밍 |
|---|---|---|---|---|
| PNG/JPEG 파일들 | 개별 파일 | 로컬 소규모 | ❌ | ❌ |
| HDF5 | 단일 바이너리 파일 | 소~중규모, 연구 | ❌ | 부분 |
| Parquet | 컬럼형 파일 | 메타데이터 + 이미지 | ✅ | ✅ |
| WebDataset (tar) | tar 아카이브 | **대규모 분산 학습** | ✅ | ✅ |

### HDF5 — 연구자 표준

- `.h5` 파일 하나에 수천~수만 이미지 저장
- 계층적 구조: `/train/images`, `/train/masks`, `/val/images`
- NumPy 배열 직접 접근: 인덱스 기반 랜덤 액세스 O(1)
- 단점: 병렬 쓰기 불가 (단일 파일 락), 클라우드 스트리밍 비효율

### Parquet — 메타데이터와 이미지 통합

- 아파치 Arrow 기반 컬럼형 포맷
- Hugging Face 데이터셋 기본 포맷 (`datasets` 라이브러리)
- 이미지를 바이트 버퍼로 저장: `{"image": bytes, "mask": bytes, "caption": str, "license": str}`
- 장점: 메타데이터 쿼리 빠름, Spark/Dask와 통합 용이
- 단점: 개별 이미지 랜덤 액세스 상대적으로 느림

### WebDataset — 대규모 학습의 표준

Google Brain, Stability AI 등이 채택한 방식. `tar` 아카이브를 스트리밍 방식으로 읽는다.

```
shard-000.tar
  000001.png    ← 손상 이미지
  000001.mask.png
  000001.target.png
  000001.json   ← 메타데이터 {"source": "nl.go.kr", "year": 1890}
shard-001.tar
  ...
```

**장점**:
- S3/GCS에서 직접 스트리밍: 디스크 공간 절약
- 멀티-워커 병렬 로딩: CPU 코어 수만큼 확장
- Shuffling: 샤드 단위 셔플 + 내부 버퍼 셔플

**권장 샤드 크기**: 200~500MB per shard (네트워크 전송 효율 최적)

## 실전 코드

```python
# 데이터 포맷 변환 및 DataLoader 설정
# pip install webdataset h5py pyarrow pandas torch torchvision

import os
import io
import json
import tarfile
from pathlib import Path
import numpy as np
import h5py
import webdataset as wds
import torch
from torch.utils.data import DataLoader
from torchvision import transforms
from PIL import Image
from tqdm import tqdm


# ──────────────────────────────────────────────
# 1. 표준 파일 폴더 → HDF5 변환
# ──────────────────────────────────────────────
def images_to_hdf5(
    data_dir: str,
    output_path: str = "dataset.h5",
    image_size: int = 1024,
    split: float = 0.9,
):
    """
    {data_dir}/images/, masks/, targets/ → dataset.h5
    """
    images_dir = Path(data_dir) / "images"
    masks_dir  = Path(data_dir) / "masks"
    targets_dir = Path(data_dir) / "targets"
    
    files = sorted([p.stem for p in images_dir.glob("*.png")])
    n_train = int(len(files) * split)
    
    with h5py.File(output_path, "w") as hf:
        for split_name, split_files in [
            ("train", files[:n_train]),
            ("val", files[n_train:])
        ]:
            n = len(split_files)
            grp = hf.create_group(split_name)
            
            # 데이터셋 미리 할당 (효율적 쓰기)
            grp.create_dataset("images",  shape=(n, 3, image_size, image_size),
                               dtype=np.uint8, compression="lzf")
            grp.create_dataset("masks",   shape=(n, 1, image_size, image_size),
                               dtype=np.uint8, compression="lzf")
            grp.create_dataset("targets", shape=(n, 3, image_size, image_size),
                               dtype=np.uint8, compression="lzf")
            
            for i, stem in enumerate(tqdm(split_files, desc=f"HDF5 {split_name}")):
                def load_img(path, grayscale=False):
                    img = Image.open(path).convert("L" if grayscale else "RGB")
                    img = img.resize((image_size, image_size), Image.LANCZOS)
                    arr = np.array(img)
                    if grayscale:
                        arr = arr[np.newaxis, :, :]  # (1, H, W)
                    else:
                        arr = arr.transpose(2, 0, 1)  # (3, H, W)
                    return arr
                
                grp["images"][i]  = load_img(images_dir / f"{stem}.png")
                grp["masks"][i]   = load_img(masks_dir / f"{stem}.png", grayscale=True)
                grp["targets"][i] = load_img(targets_dir / f"{stem}.png")
    
    size_mb = os.path.getsize(output_path) / 1024 / 1024
    print(f"HDF5 생성 완료: {output_path} ({size_mb:.0f} MB)")
    print(f"  train: {n_train}개, val: {len(files)-n_train}개")


# ──────────────────────────────────────────────
# 2. 표준 파일 폴더 → WebDataset tar 샤드 변환
# ──────────────────────────────────────────────
def images_to_webdataset(
    data_dir: str,
    output_dir: str = "./shards",
    shard_size: int = 500,  # 샤드당 이미지 수
):
    """
    images/ masks/ targets/ → shard-000000.tar shard-000001.tar ...
    """
    os.makedirs(output_dir, exist_ok=True)
    images_dir = Path(data_dir) / "images"
    files = sorted([p.stem for p in images_dir.glob("*.png")])
    
    shard_idx = 0
    current_shard_files = []
    
    for i, stem in enumerate(tqdm(files, desc="WebDataset 변환")):
        current_shard_files.append(stem)
        
        # 샤드 크기 도달 또는 마지막 파일
        if len(current_shard_files) >= shard_size or i == len(files) - 1:
            shard_name = f"shard-{shard_idx:06d}.tar"
            shard_path = os.path.join(output_dir, shard_name)
            
            with tarfile.open(shard_path, "w") as tf:
                for s in current_shard_files:
                    for suffix, sub_dir in [
                        (".png", "images"),
                        (".mask.png", "masks"),
                        (".target.png", "targets"),
                    ]:
                        src = Path(data_dir) / sub_dir
                        # 파일 이름 통일: {s}.png / {s}.mask.png / {s}.target.png
                        actual = src / f"{s}.png"
                        if actual.exists():
                            tf.add(str(actual), arcname=f"{s}{suffix}")
                    
                    # 메타데이터 JSON
                    meta = {"key": s, "source": "geulsalim-ai-dataset"}
                    meta_bytes = json.dumps(meta).encode()
                    info = tarfile.TarInfo(name=f"{s}.json")
                    info.size = len(meta_bytes)
                    tf.addfile(info, io.BytesIO(meta_bytes))
            
            size_mb = os.path.getsize(shard_path) / 1024 / 1024
            print(f"  {shard_name}: {len(current_shard_files)}개 ({size_mb:.0f}MB)")
            shard_idx += 1
            current_shard_files = []
    
    print(f"WebDataset 변환 완료: {shard_idx}개 샤드")


# ──────────────────────────────────────────────
# 3. WebDataset DataLoader 구성
# ──────────────────────────────────────────────
def build_webdataset_loader(
    shard_pattern: str = "./shards/shard-{000000..000009}.tar",
    batch_size: int = 4,
    image_size: int = 1024,
    num_workers: int = 4,
    shuffle_buffer: int = 200,
) -> DataLoader:
    """
    WebDataset 기반 학습 DataLoader
    shard_pattern 예: "s3://my-bucket/shards/shard-{000000..000999}.tar"
    """
    transform = transforms.Compose([
        transforms.Resize((image_size, image_size)),
        transforms.ToTensor(),
        transforms.Normalize([0.5, 0.5, 0.5], [0.5, 0.5, 0.5]),  # [-1, 1]
    ])
    mask_transform = transforms.Compose([
        transforms.Resize((image_size, image_size)),
        transforms.ToTensor(),  # [0, 1]
    ])
    
    def decode_sample(sample):
        """tar에서 꺼낸 바이트 → 텐서"""
        image  = Image.open(io.BytesIO(sample[".png"])).convert("RGB")
        mask   = Image.open(io.BytesIO(sample[".mask.png"])).convert("L")
        target = Image.open(io.BytesIO(sample[".target.png"])).convert("RGB")
        return {
            "image":  transform(image),
            "mask":   mask_transform(mask),
            "target": transform(target),
        }
    
    dataset = (
        wds.WebDataset(shard_pattern, resampled=True)   # resampled=True: 무한 반복
        .shuffle(shuffle_buffer)
        .map(decode_sample)
        .batched(batch_size, partial=False)
    )
    
    loader = DataLoader(
        dataset,
        batch_size=None,       # WebDataset이 배치 처리
        num_workers=num_workers,
        pin_memory=True,       # GPU 전송 속도 향상
        prefetch_factor=2,     # 워커당 미리 읽어두는 배치 수
    )
    
    return loader


# ──────────────────────────────────────────────
# 4. DataLoader 성능 벤치마크
# ──────────────────────────────────────────────
def benchmark_dataloader(loader, n_batches: int = 20):
    """DataLoader 처리 속도 측정 (images/sec)"""
    import time
    
    start = time.time()
    total_images = 0
    
    for i, batch in enumerate(loader):
        if i >= n_batches:
            break
        total_images += batch["image"].shape[0]
    
    elapsed = time.time() - start
    throughput = total_images / elapsed
    
    print(f"처리 속도: {throughput:.0f} images/sec")
    print(f"총 {total_images}개 이미지, {elapsed:.1f}초")
    
    # GPU 학습 기준: 배치 크기 4 × 초당 처리 배치 수
    # A100 기준 목표: 10+ batches/sec (40+ images/sec at 1024px)
    return throughput


if __name__ == "__main__":
    # 가상 데이터로 WebDataset 변환 테스트
    print("WebDataset 샤드 변환 테스트")
    print("실제 데이터 디렉토리를 data_dir에 전달하세요")
    
    # 성능 벤치마크 예시
    # loader = build_webdataset_loader("./shards/shard-{000000..000009}.tar")
    # benchmark_dataloader(loader)
```

## 함정 카드 — 자주 실수하는 5가지

1. **"DataLoader num_workers=0으로 둔다"** → 기본값 0은 단일 프로세스 로딩. 멀티코어 CPU에서는 `num_workers = min(os.cpu_count(), 8)`로 설정. Windows에서는 `if __name__ == '__main__':` 가드 필수 (spawn 방식).

2. **"WebDataset에서 셔플을 제대로 설정하지 않는다"** → `shuffle_buffer` 크기가 너무 작으면 모델이 샤드 내 순서를 학습한다. 최소 배치 크기의 50배 이상 권장. `resampled=True`는 무한 반복 스트리밍 시 필수.

3. **"HDF5를 다중 프로세스에서 동시에 쓴다"** → HDF5는 기본적으로 단일 쓰기 허용. 병렬 쓰기가 필요하면 `h5py.File(path, "w", libver="latest", swmr=True)` SWMR 모드 사용. 읽기는 다중 프로세스 OK.

4. **"pin_memory=True를 항상 쓴다"** → `pin_memory=True`는 GPU 전송 속도를 높이지만 CPU 메모리 사용량이 증가한다. CPU RAM이 부족한 환경에서는 오히려 OOM 유발. 시스템 RAM > 32GB일 때만 권장.

5. **"전처리를 DataLoader 내에서 한다"** → 이미지 리사이즈, 정규화 같은 고비용 전처리는 **오프라인에서 미리 처리**하고 저장하는 게 효율적. 온라인 증강(플립, 회전)만 DataLoader에서 수행.

## 연습문제

**문제 1.** WebDataset의 샤드당 최적 크기(파일 수 or 바이트 수)가 왜 중요한가? 너무 작거나 너무 크면 어떤 문제가 생기는가?

> **해설:** 너무 작은 샤드(< 50MB): 파일 오픈/닫기 오버헤드가 전체 처리 시간의 큰 비율 차지. S3에서 읽을 때 latency 비용 상승. 너무 큰 샤드(> 2GB): 셔플링 세분도가 떨어짐 (샤드 단위 셔플이므로). 장애 시 재전송 비용 큼. 권장: 100~500MB, 200~1000개 샘플/샤드. Stability AI 공개 학습 코드 기준 500MB가 표준.

**문제 2.** HDF5와 WebDataset 중 어느 포맷이 다음 시나리오에 적합한가? (a) 로컬 SSD 서버, 데이터 50,000장, 빠른 랜덤 액세스 필요 (b) AWS S3 저장, 데이터 5,000,000장, 분산 학습

> **해설:** (a) **HDF5** — 로컬 NVMe SSD에서 HDF5 인덱스 접근은 O(1). 50,000장은 단일 .h5 파일로 관리 가능. (b) **WebDataset** — S3 직접 스트리밍 지원. 수백만 장을 샤드로 분산. 다중 노드 학습에서 각 워커가 다른 샤드를 읽어 자동 병렬화.

**문제 3.** DataLoader `num_workers=4`일 때 메모리 사용량을 계산하라. 배치 크기 4, 이미지 크기 1024×1024 RGB, `prefetch_factor=2`.

> **해설:** 이미지 텐서 하나: 1024 × 1024 × 3 × 4바이트(FP32) = 12MB. 배치 4개: 48MB. 마스크 + 타깃 포함하면 48 × 3 = 144MB/배치. prefetch_factor=2이므로 워커당 2배치 버퍼. 4워커 × 2배치 × 144MB = **1,152MB ≈ 1.1GB** 추가 RAM 사용. `pin_memory=True` 시 pinned 메모리로 추가 할당.
