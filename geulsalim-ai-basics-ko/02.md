# 챕터 02 · Claude Code와 함께하는 ML 워크플로우

## 학습 목표
- Claude Code를 ML 개발 파이프라인의 각 단계에서 활용하는 방법을 익힌다
- `-p` 플래그를 이용한 비대화형 자동화 패턴을 이해한다
- 코드 생성 → 검증 → 저장의 반복 사이클을 구축할 수 있다

---

## 이론

### ML 개발의 병목은 어디인가

전통적인 ML 개발 사이클은 이렇게 생겼다.

```
아이디어 → 코드 작성(3일) → 버그 수정(2일) → 실험(1일) → 결과 분석(1일) → 반복
```

코드 작성과 버그 수정에 전체 시간의 70% 이상이 소모된다. 모델 아키텍처를 고민하고 데이터를 분석하는 **진짜 ML 작업**은 30%에 불과하다.

Claude Code는 이 병목을 제거한다. 코드 작성과 디버깅을 대화로 처리하면, 1인 개발자가 집중할 수 있는 영역이 늘어난다.

### Claude Code의 세 가지 활용 모드

**모드 1: 대화형 (Interactive)**
터미널에서 `claude`를 실행하면 대화 모드가 시작된다. 자연어로 요청하면 코드를 작성하고 즉시 실행까지 할 수 있다.

```bash
claude
> PyTorch로 이미지 분류 모델 만들어줘. ResNet18 기반으로.
```

**모드 2: 비대화형 (`-p` 플래그)**
스크립트나 크론탭에서 Claude Code를 자동으로 호출한다. ML 파이프라인 자동화의 핵심이다.

```bash
claude -p "훈련 손실이 0.001 이하로 떨어지면 체크포인트를 저장하는 콜백 클래스 작성해줘"
```

**모드 3: 파일 기반 (`--input`/`--output`)**
프롬프트 파일을 읽어서 결과를 파일로 저장한다. 대량 코드 생성에 사용한다.

```bash
claude -p "$(cat prompt.md)" > generated_code.py
```

### ML 워크플로우 5단계별 활용법

**1단계: 데이터 준비**
데이터 전처리 스크립트, 라이선스 검증, 파일 형식 변환 코드를 Claude Code에 맡긴다.

```bash
claude -p "AI-Hub에서 내려받은 이미지 데이터셋을 WebDataset 형식으로 변환하는 스크립트 작성해줘. 이미지 크기 512×512 리사이즈, PNG → JPEG 변환 포함."
```

**2단계: 모델 선택 및 설정**
Hugging Face Hub에서 모델을 검색하고 파인튜닝 설정을 구성한다.

**3단계: 학습 실행**
학습 스크립트 작성, 하이퍼파라미터 실험 계획, wandb 연동.

**4단계: 평가**
FID, CLIP Score 계산 스크립트, 시각화 코드.

**5단계: 배포**
FastAPI 서버 코드, Docker 이미지 빌드, 모델 최적화.

### 프롬프트 엔지니어링 — ML 코드 요청 패턴

Claude Code에서 좋은 ML 코드를 얻으려면 프롬프트에 세 가지를 명시해야 한다.

1. **라이브러리 버전**: `torch 2.1, diffusers 0.25, transformers 4.37`
2. **하드웨어 제약**: `VRAM 24GB, A100 1장`
3. **출력 형식**: `클래스 기반, 한국어 주석, type hint 포함`

## 실전 코드

```python
# Claude Code CLI를 Python에서 직접 호출하는 유틸리티
# ML 파이프라인 자동화에 활용

import subprocess
import json
import tempfile
import os
from pathlib import Path

def ask_claude(prompt: str, output_file: str = None) -> str:
    """
    Claude Code CLI를 호출하여 응답을 받는 함수
    
    Args:
        prompt: Claude에게 보낼 프롬프트
        output_file: 응답을 저장할 파일 경로 (선택)
    
    Returns:
        Claude의 응답 텍스트
    """
    # 프롬프트를 임시 파일에 저장 (긴 프롬프트 처리용)
    with tempfile.NamedTemporaryFile(mode='w', suffix='.txt', 
                                     delete=False, encoding='utf-8') as f:
        f.write(prompt)
        prompt_file = f.name
    
    try:
        result = subprocess.run(
            ['claude', '-p', prompt],
            capture_output=True,
            text=True,
            timeout=120  # 2분 제한
        )
        response = result.stdout.strip()
        
        if output_file:
            Path(output_file).write_text(response, encoding='utf-8')
            print(f"저장됨: {output_file}")
        
        return response
    
    except subprocess.TimeoutExpired:
        return "오류: Claude 응답 시간 초과 (120초)"
    except FileNotFoundError:
        return "오류: claude CLI가 설치되지 않았거나 PATH에 없음"
    finally:
        os.unlink(prompt_file)


def generate_training_script(
    model_name: str,
    dataset_path: str,
    output_dir: str,
    epochs: int = 10,
    batch_size: int = 4
) -> str:
    """ML 학습 스크립트를 Claude Code로 자동 생성"""
    
    prompt = f"""
    다음 사양으로 PyTorch 학습 스크립트를 작성해줘:
    
    - 기반 모델: {model_name}
    - 데이터셋 경로: {dataset_path}
    - 출력 디렉토리: {output_dir}
    - 에포크: {epochs}
    - 배치 크기: {batch_size}
    
    요구사항:
    - Hugging Face Trainer 사용
    - wandb 학습 로깅
    - 매 에포크 체크포인트 저장
    - 한국어 주석
    - type hint 포함
    """
    
    output_path = f"{output_dir}/train.py"
    os.makedirs(output_dir, exist_ok=True)
    
    code = ask_claude(prompt, output_path)
    return code


# 사용 예시
if __name__ == "__main__":
    # 학습 스크립트 자동 생성
    script = generate_training_script(
        model_name="stabilityai/stable-diffusion-xl-base-1.0",
        dataset_path="/data/korean-historical-images",
        output_dir="./output",
        epochs=20,
        batch_size=2
    )
    print("학습 스크립트 생성 완료")
    print(f"미리보기:\n{script[:500]}...")
```

**실행:**
```bash
pip install anthropic  # claude CLI 설치 포함
python claude_ml_utils.py
```

## 함정 카드 — 자주 실수하는 5가지

1. **"Claude Code가 생성한 코드는 무조건 맞다"** → 거짓. 항상 실행해서 확인해야 한다. 특히 라이브러리 버전에 따라 API가 다를 수 있다. 오류 메시지를 Claude에게 붙여넣으면 즉시 수정해 준다.

2. **"`-p` 플래그 없이 자동화하려 한다"** → `claude`만 실행하면 대화형 모드로 들어가 자동화가 안 된다. 스크립트에서는 반드시 `-p "프롬프트"` 형식을 써야 한다.

3. **"프롬프트에 하드웨어 정보를 안 준다"** → VRAM 8GB에서 돌아가는 코드와 24GB에서 돌아가는 코드는 다르다. 하드웨어 제약을 명시하지 않으면 실행 불가능한 코드가 나온다.

4. **"한 프롬프트에 모든 것을 요청한다"** → "데이터 전처리 + 학습 + 배포 코드 전부 써줘"는 품질이 낮다. 단계를 나눠서 각각 요청하면 훨씬 정확한 코드가 나온다.

5. **"응답을 복붙 후 수정 없이 쓴다"** → Claude Code 응답은 출발점이다. 경로, 파일명, 하이퍼파라미터는 실제 환경에 맞게 반드시 수정해야 한다.

## 연습문제

**문제 1.** Claude Code의 `-p` 플래그가 ML 자동화에 중요한 이유를 설명하라. 대화형 모드와의 차이점을 포함할 것.

> **해설:** `-p` 플래그는 단일 명령으로 응답을 받고 종료한다. 크론탭, 쉘 스크립트, Python subprocess에서 호출 가능하다. 대화형 모드는 사람이 개입해야 하므로 자동화 파이프라인에 사용할 수 없다. ML에서 반복적인 하이퍼파라미터 실험, 배치 코드 생성 등에 필수다.

**문제 2.** 다음 프롬프트의 문제점을 찾고 개선하라: `"이미지 분류 모델 만들어줘"`

> **해설:** ① 기반 모델 미지정 ② 데이터셋 형식 미지정 ③ 하드웨어 제약 미지정 ④ 출력 형식 미지정. 개선: `"ResNet18 기반 이미지 분류 모델 만들어줘. 데이터는 ImageFolder 형식, VRAM 8GB 환경, 클래스 기반, 한국어 주석, type hint 포함."`

**문제 3.** `generate_training_script()` 함수를 확장하여 VRAM 크기에 따라 배치 사이즈를 자동으로 결정하는 로직을 추가하라. (8GB → batch 1, 16GB → batch 2, 24GB+ → batch 4)

> **해설:** `import torch; vram = torch.cuda.get_device_properties(0).total_memory / 1e9`로 VRAM을 확인한 후, 조건문으로 배치 사이즈를 결정한다. 이 로직을 `generate_training_script()` 호출 전에 실행하면 된다.
