# 챕터 09 · LoRA · QLoRA · Full Fine-tune — 언제 무엇을 쓰는가

## 학습 목표
- LoRA · QLoRA · Full Fine-tune 세 전략의 수학적 원리 차이를 설명할 수 있다
- GPU VRAM과 품질 요구사항을 기준으로 파인튜닝 전략을 결정할 수 있다
- SDXL Inpainting에 LoRA rank와 learning rate를 설정하는 실전 감각을 갖는다

---

## 이론

### 파인튜닝이란 무엇인가

파운데이션 모델(SDXL, Flux, Llama 등)은 인터넷 규모의 데이터로 학습됐다. 하지만 고문서 복원처럼 특정 도메인에 특화하려면 추가 학습이 필요하다. 이 추가 학습을 **파인튜닝(Fine-tuning)**이라 한다.

문제는 SDXL 같은 모델의 파라미터 수가 **약 35억 개**라는 것이다. 이를 모두 다시 학습하면:
- H100 GPU 필요 (메모리 80GB)
- 수백만 원의 컴퓨팅 비용
- 수 주~수 달의 학습 시간

1인 개발자에게 현실적이지 않다. 그래서 등장한 것이 **PEFT (Parameter-Efficient Fine-Tuning)** 계열 방법들이다.

### Full Fine-tuning

모든 파라미터를 갱신한다. 최고 품질이지만:
- 35억 파라미터 × 4바이트(FP32) = 약 14GB (파라미터 저장만)
- Gradient + Optimizer state 포함하면 80~160GB VRAM 필요
- **1인 개발자에게 비현실적**. 스킵.

### LoRA — Low-Rank Adaptation

2021년 Microsoft에서 제안. 핵심 아이디어:

> 모델의 가중치 변화(ΔW)는 낮은 랭크(low rank)로 근사할 수 있다.

수학적으로:
```
ΔW = A × B
where A: (d × r), B: (r × k), r << min(d, k)
```

d=1024, k=1024, r=4 라면:
- 원래 ΔW 크기: 1024 × 1024 = **1,048,576 파라미터**
- LoRA ΔW 크기: (1024×4) + (4×1024) = **8,192 파라미터** (128배 절약)

**핵심 파라미터**:
- `rank (r)`: 분해 행렬의 랭크. 값이 클수록 표현력 ↑, VRAM ↑
  - 실험/스타일: rank 4~8
  - 콘텐츠 학습: rank 16~32
  - 복잡 도메인: rank 64~128
- `lora_alpha`: LoRA 스케일링 계수. 일반적으로 `alpha = 2 × rank`
- `target_modules`: 어떤 레이어에 LoRA를 붙이냐. 보통 `["q_proj", "v_proj"]` 또는 U-Net attention blocks

**SDXL에서 LoRA**:
- 품질: Full Fine-tune 대비 **90~95% 수준**
- VRAM: 8~12GB (rank 16, 1024px 기준)
- 파일 크기: 원본 모델 6.7GB → LoRA 파일 50~200MB

### QLoRA — Quantized LoRA

2023년 Tim Dettmers 팀 제안. LoRA를 한 단계 더 발전:

1. 기반 모델을 **4-bit NF4 양자화**로 메모리 압축
2. 양자화된 기반 모델 위에 FP16 LoRA 어댑터 훈련
3. Double Quantization으로 추가 압축

결과:
- VRAM: LoRA 대비 40~60% 추가 절약
- 품질: Full Fine-tune 대비 **80~90%** (LoRA 대비 약간 낮음)
- SDXL 기준: 6~8GB VRAM에서 가능

**언제 QLoRA를 선택하는가**: VRAM 8GB GPU + SDXL처럼 큰 모델 조합. 단, 학습 속도는 LoRA보다 느리다.

### 고문서 복원 최적 전략 결정 트리

```
GPU VRAM이 얼마인가?
├── 8GB 미만 → SD 1.5 LoRA (rank 4)
├── 8~12GB → SDXL QLoRA (rank 16) 또는 SDXL LoRA (rank 4)
├── 12~24GB → SDXL LoRA (rank 16~32)  ← 권장
└── 24GB 이상 → SDXL LoRA (rank 64) 또는 Full Fine-tune 부분적

품질이 가장 중요한가?
├── Yes → A100 80GB 렌탈 ($0.78~$1.29/hr) + Full Fine-tune
└── No → LoRA rank 16~32로 충분
```

### 최신 동향 (2025~2026)

**DoRA (Weight-Decomposed Low-Rank Adaptation, 2024)**: LoRA를 방향(direction)과 크기(magnitude)로 분해. 품질 향상.

**LoRA+**: 학습률을 A, B 행렬에 다르게 적용. A는 낮게, B는 높게. 수렴 속도 개선.

**Rank-Stabilized LoRA (rsLoRA)**: `alpha = alpha/sqrt(r)` 설정으로 랭크에 무관한 안정적 학습. Hugging Face PEFT의 `use_rslora=True` 옵션.

## 실전 코드

```python
# SDXL Inpainting LoRA 파인튜닝 설정
# 실행 환경: CUDA GPU 12GB+, peft>=0.10.0, diffusers>=0.28.0

import torch
from peft import LoraConfig, get_peft_model
from diffusers import UNet2DConditionModel
from peft import PeftModel


# ──────────────────────────────────────────────
# 1. LoRA 설정 비교 (rank별 VRAM/품질 트레이드오프)
# ──────────────────────────────────────────────
def get_lora_config(preset: str = "standard") -> LoraConfig:
    """
    preset:
      "minimal"  — 8GB GPU, 빠른 실험
      "standard" — 12GB GPU, 고문서 복원 기본
      "quality"  — 24GB GPU, 최고 품질
    """
    presets = {
        "minimal": dict(r=4,  lora_alpha=8,  target_modules=["to_q","to_v"]),
        "standard": dict(r=16, lora_alpha=32, target_modules=["to_q","to_k","to_v","to_out.0"]),
        "quality":  dict(r=64, lora_alpha=64, target_modules=["to_q","to_k","to_v","to_out.0","ff.net.0.proj","ff.net.2"]),
    }
    p = presets[preset]
    return LoraConfig(
        r=p["r"],
        lora_alpha=p["lora_alpha"],
        target_modules=p["target_modules"],
        lora_dropout=0.1,
        bias="none",
        use_rslora=True,        # Rank-Stabilized LoRA (2024 권장)
    )


# ──────────────────────────────────────────────
# 2. SDXL UNet에 LoRA 적용
# ──────────────────────────────────────────────
def apply_lora_to_unet(unet: UNet2DConditionModel, preset: str = "standard"):
    """UNet에 LoRA 레이어 삽입 후 학습 가능 파라미터 수 출력"""
    config = get_lora_config(preset)
    unet = get_peft_model(unet, config)
    
    trainable = sum(p.numel() for p in unet.parameters() if p.requires_grad)
    total     = sum(p.numel() for p in unet.parameters())
    print(f"[{preset}] 학습 파라미터: {trainable:,} / {total:,} ({trainable/total*100:.2f}%)")
    return unet


# ──────────────────────────────────────────────
# 3. QLoRA: 4-bit 양자화 + LoRA
# ──────────────────────────────────────────────
def apply_qlora_to_unet(model_id: str = "stabilityai/stable-diffusion-xl-base-1.0"):
    """
    QLoRA 설정 — bitsandbytes 필요
    pip install bitsandbytes>=0.41.0
    """
    try:
        from transformers import BitsAndBytesConfig
        
        bnb_config = BitsAndBytesConfig(
            load_in_4bit=True,
            bnb_4bit_use_double_quant=True,    # Double Quantization (추가 메모리 절약)
            bnb_4bit_quant_type="nf4",          # NF4: 정규분포 파라미터에 최적화
            bnb_4bit_compute_dtype=torch.bfloat16,
        )
        print("QLoRA BitsAndBytes 설정 완료")
        print("  - 4-bit NF4 양자화")
        print("  - Double Quantization: VRAM 추가 ~20% 절약")
        return bnb_config
    except ImportError:
        print("bitsandbytes 미설치: pip install bitsandbytes")
        return None


# ──────────────────────────────────────────────
# 4. LoRA 저장 및 불러오기
# ──────────────────────────────────────────────
def save_lora_weights(model, output_dir: str = "./lora_output"):
    """LoRA 가중치만 저장 — 수십 MB"""
    model.save_pretrained(output_dir)
    print(f"LoRA 저장 완료: {output_dir}")
    
    import os
    for f in os.listdir(output_dir):
        size_mb = os.path.getsize(os.path.join(output_dir, f)) / 1024 / 1024
        print(f"  {f}: {size_mb:.1f} MB")


def load_lora_weights(base_model, lora_dir: str = "./lora_output"):
    """저장된 LoRA를 기반 모델에 로드"""
    model = PeftModel.from_pretrained(base_model, lora_dir)
    model = model.merge_and_unload()  # LoRA를 기반 모델에 병합 (추론 속도 최적화)
    return model


# ──────────────────────────────────────────────
# 5. 파인튜닝 전략 비교표 출력
# ──────────────────────────────────────────────
def print_strategy_comparison():
    strategies = [
        ("Full Fine-tune",     "35B", "160GB+", "H100×2+", "$200+/실험", "100%", "매우 어려움"),
        ("LoRA rank 64",        "170M", "24GB",  "A100",    "$5~10/실험",  "95%",  "중간"),
        ("LoRA rank 16",        "43M",  "12GB",  "RTX 4090","$2~5/실험",   "93%",  "쉬움"),
        ("LoRA rank 4",         "11M",  "8GB",   "RTX 3080","$1~2/실험",   "90%",  "쉬움"),
        ("QLoRA rank 16",       "43M",  "6GB",   "RTX 3060","$1/실험",     "87%",  "쉬움"),
    ]
    header = f"{'전략':<20} {'학습파라미터':>12} {'VRAM':>8} {'GPU':>12} {'비용':>12} {'품질':>6} {'난이도':<10}"
    print(header)
    print("-" * 85)
    for row in strategies:
        print(f"{row[0]:<20} {row[1]:>12} {row[2]:>8} {row[3]:>12} {row[4]:>12} {row[5]:>6} {row[6]:<10}")


if __name__ == "__main__":
    print("=" * 85)
    print("파인튜닝 전략 비교 (SDXL 기준)")
    print("=" * 85)
    print_strategy_comparison()
    print()
    
    # LoRA 설정 확인 (GPU 없어도 실행 가능)
    for preset in ["minimal", "standard", "quality"]:
        cfg = get_lora_config(preset)
        print(f"[{preset}] rank={cfg.r}, alpha={cfg.lora_alpha}, modules={cfg.target_modules}")
```

## 함정 카드 — 자주 실수하는 5가지

1. **"rank를 높이면 무조건 품질이 좋아진다"** → rank 과다는 오히려 과적합(overfitting)을 유발한다. 데이터셋 크기 대비 rank를 결정해야 한다. 이미지 100장 이하 → rank 4~8, 1,000장 이상 → rank 16~32, 10,000장 이상 → rank 64 고려.

2. **"alpha = rank로 항상 설정한다"** → `use_rslora=True`를 쓰면 alpha/sqrt(rank) 스케일링이 자동 적용된다. rsLoRA 미사용 시에는 `alpha = 2 × rank`가 경험 법칙. alpha가 너무 크면 학습이 불안정해진다.

3. **"QLoRA는 모든 모델에 적용 가능하다"** → Diffusion 모델의 U-Net에 QLoRA를 적용하는 것은 LLM보다 복잡하다. `bitsandbytes`가 공식 지원하는 레이어 타입이 제한적. 2025년 기준 SDXL U-Net QLoRA는 커뮤니티 패치가 필요한 경우 있음.

4. **"LoRA 학습 후 추론이 느려진다"** → `merge_and_unload()`로 LoRA를 기반 모델에 병합하면 추론 속도가 원본과 동일해진다. 단, 병합 후에는 LoRA를 다시 분리할 수 없으니 원본 가중치를 백업해둘 것.

5. **"target_modules를 잘못 설정한다"** → SDXL U-Net의 경우 `to_q`, `to_k`, `to_v` (Attention Q/K/V)만 지정하면 Efficient LoRA, `ff.net` (Feed-Forward)까지 포함하면 Rich LoRA. 레이어 이름은 `model.named_modules()` 또는 `model.print_trainable_parameters()`로 확인.

## 연습문제

**문제 1.** 데이터셋 크기 500장, GPU VRAM 12GB 환경에서 SDXL을 고문서 복원에 파인튜닝한다. LoRA rank를 4, 16, 64 중 선택하고 이유를 설명하라.

> **해설:** **rank 16 권장**. 데이터 500장은 rank 4(과소학습 위험)와 rank 64(과적합 위험) 중간. 12GB VRAM에서 rank 16은 여유 있게 학습 가능. `lora_alpha=32`, `use_rslora=True`, `target_modules=["to_q","to_k","to_v","to_out.0"]` 기본 설정으로 시작.

**문제 2.** LoRA rank 16의 총 추가 파라미터 수를 계산하라. SDXL U-Net의 Attention 레이어가 각 Cross-Attention block당 `to_q`, `to_k`, `to_v`, `to_out` 4개이고, 각 입출력 차원이 1024이며, 총 32개 블록이 있다고 가정.

> **해설:** 레이어 하나의 LoRA 파라미터 = A(1024×16) + B(16×1024) = 16,384 + 16,384 = 32,768. 총 4레이어 × 32블록 = 128레이어 × 32,768 = **4,194,304 파라미터 (약 420만)**.  SDXL 전체 35억 파라미터 대비 0.12%만 학습.

**문제 3.** rsLoRA(Rank-Stabilized LoRA)가 기존 LoRA보다 안정적인 이유를 설명하라. `lora_alpha/r`과 `lora_alpha/sqrt(r)` 스케일링의 차이는 무엇인가?

> **해설:** 기존 LoRA는 `alpha/r` 스케일. rank가 커질수록 스케일이 감소하여 고랭크 LoRA에서 학습률 재조정이 필요. rsLoRA는 `alpha/sqrt(r)` 스케일. rank 증가에 따른 스케일 감소 속도가 더 완만하여, 같은 alpha 설정으로 다양한 rank에서 일관된 학습 다이나믹스를 유지. 특히 rank 64 이상의 고랭크 LoRA에서 rsLoRA 효과가 두드러짐.
