# 챕터 18 · Docker · Kubernetes — GPU 모델 배포

## 학습 목표
- NVIDIA Container Toolkit을 포함한 GPU 지원 Docker 이미지를 빌드할 수 있다
- docker-compose로 FastAPI + GPU 서비스를 로컬에서 실행할 수 있다
- Kubernetes GPU 노드에 Deployment와 Service를 배포하는 기본 패턴을 적용할 수 있다

---

## 이론

### 왜 Docker인가 — "내 컴퓨터에서는 됐는데"

"내 서버에서는 완벽히 동작합니다." — 그런데 고객 서버에서는 안 된다. Python 버전, CUDA 버전, diffusers 버전이 다를 때 발생하는 문제다.

Docker는 **환경 전체를 이미지로 포장**한다. 한 번 만든 이미지는 어디서든 동일하게 동작한다.

나는 Lambda Labs A100 서버에서 완벽히 동작하던 코드가 RunPod H100에서 CUDA 버전 불일치로 실패하는 경험을 했다. 그 이후 Docker 없이는 ML 서비스를 배포하지 않는다.

### GPU Docker 핵심 구성

일반 Docker와 GPU Docker의 차이:
```
일반: CPU + RAM
GPU Docker: CPU + RAM + NVIDIA GPU (CUDA 드라이버 공유)
```

필요한 것:
1. **NVIDIA GPU 드라이버** (호스트에 설치)
2. **NVIDIA Container Toolkit** (`nvidia-docker2`)
3. **CUDA 베이스 이미지** (`nvidia/cuda:12.1.0-cudnn8-devel-ubuntu22.04`)

### Dockerfile 핵심 최적화

**레이어 캐싱**:
```dockerfile
# 자주 바뀌는 것은 나중에 → 캐시 효율
COPY requirements.txt .    # 의존성 먼저
RUN pip install ...        # 캐시됨
COPY . .                   # 코드 나중에 (자주 바뀜)
```

**이미지 크기 최소화**:
- Multi-stage build: 빌드 환경과 실행 환경 분리
- `.dockerignore`로 불필요한 파일 제외
- PyTorch CPU only 버전으로 개발, GPU 버전은 프로덕션만

**모델 캐시**:
- Hugging Face 모델은 `/root/.cache/huggingface/`에 저장됨
- Docker 볼륨으로 마운트하면 재다운로드 방지
- 또는 이미지 빌드 시 미리 다운로드 (이미지 크기 증가)

### Kubernetes GPU 배포

K8s에서 GPU를 사용하려면:
1. GPU 노드에 NVIDIA device plugin 설치
2. Pod spec에 `resources.limits: nvidia.com/gpu: 1` 명시
3. NVIDIA Container Runtime이 자동으로 GPU 연결

## 실전 코드

```dockerfile
# Dockerfile — GPU 지원 SDXL FastAPI 서버
# 베이스: CUDA 12.1 + PyTorch 2.x 호환

FROM nvidia/cuda:12.1.0-cudnn8-runtime-ubuntu22.04

# 시스템 패키지
RUN apt-get update && apt-get install -y \
    python3.11 python3-pip git curl \
    libgl1-mesa-glx libglib2.0-0 \
    && rm -rf /var/lib/apt/lists/*

# Python 심볼릭 링크
RUN ln -sf /usr/bin/python3.11 /usr/bin/python3 && \
    ln -sf /usr/bin/python3.11 /usr/bin/python

WORKDIR /app

# 의존성 먼저 설치 (캐시 최대화)
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

# 모델 사전 다운로드 (선택: 이미지 크기 ~15GB 증가)
# ARG DOWNLOAD_MODEL=false
# RUN if [ "$DOWNLOAD_MODEL" = "true" ]; then \
#     python -c "from diffusers import StableDiffusionXLInpaintPipeline; \
#     StableDiffusionXLInpaintPipeline.from_pretrained('diffusers/stable-diffusion-xl-1.0-inpainting-0.1')"; \
#     fi

# 앱 코드 복사
COPY . .

EXPOSE 8000

# 헬스 체크
HEALTHCHECK --interval=30s --timeout=10s --start-period=120s --retries=3 \
    CMD curl -f http://localhost:8000/health || exit 1

# 프로덕션 실행
CMD ["python", "-m", "uvicorn", "main:app", \
     "--host", "0.0.0.0", "--port", "8000", \
     "--workers", "1", "--log-level", "info"]
```

```yaml
# docker-compose.yml — 로컬 GPU 서버 실행
version: "3.8"

services:
  geulsalim-restore-api:
    build:
      context: .
      dockerfile: Dockerfile
    image: geulsalim-ai:v1.0.0
    
    # GPU 연결 (NVIDIA Container Toolkit 필요)
    deploy:
      resources:
        reservations:
          devices:
            - driver: nvidia
              count: 1
              capabilities: [gpu]
    
    ports:
      - "8000:8000"
    
    environment:
      - NVIDIA_VISIBLE_DEVICES=all
      - CUDA_VISIBLE_DEVICES=0
      - MODEL_ID=diffusers/stable-diffusion-xl-1.0-inpainting-0.1
      - HF_HOME=/models/.cache      # Hugging Face 캐시 경로
    
    volumes:
      # 모델 캐시를 호스트에 저장 (재다운로드 방지)
      - /data/hf-cache:/models/.cache
      # 복원 결과 저장
      - /data/restored:/app/restored
    
    restart: unless-stopped
    
    # 컨테이너 리소스 제한
    mem_limit: 32g
    shm_size: 2g          # /dev/shm: PyTorch DataLoader 공유 메모리

  # Nginx 리버스 프록시 (HTTPS, Rate Limiting)
  nginx:
    image: nginx:alpine
    ports:
      - "80:80"
      - "443:443"
    volumes:
      - ./nginx.conf:/etc/nginx/nginx.conf:ro
      - /etc/letsencrypt:/etc/letsencrypt:ro
    depends_on:
      - geulsalim-restore-api
    restart: unless-stopped
```

```yaml
# kubernetes/deployment.yaml — K8s GPU 배포
apiVersion: apps/v1
kind: Deployment
metadata:
  name: geulsalim-restore-api
  labels:
    app: geulsalim-ai
    version: "1.0.0"
spec:
  replicas: 1          # GPU 서비스는 replica 수 = GPU 수
  selector:
    matchLabels:
      app: geulsalim-ai
  template:
    metadata:
      labels:
        app: geulsalim-ai
    spec:
      # GPU 노드에만 스케줄링
      nodeSelector:
        nvidia.com/gpu.present: "true"
      
      # 모델 초기화 대기 시간
      terminationGracePeriodSeconds: 60
      
      containers:
        - name: restore-api
          image: your-registry/geulsalim-ai:v1.0.0
          
          ports:
            - containerPort: 8000
          
          # GPU 리소스 요청
          resources:
            requests:
              memory: "8Gi"
              cpu: "4"
              nvidia.com/gpu: "1"
            limits:
              memory: "24Gi"
              cpu: "8"
              nvidia.com/gpu: "1"
          
          env:
            - name: CUDA_VISIBLE_DEVICES
              value: "0"
            - name: HF_HOME
              value: /models/.cache
          
          volumeMounts:
            - name: model-cache
              mountPath: /models/.cache
          
          # 헬스 체크
          livenessProbe:
            httpGet:
              path: /health/live
              port: 8000
            initialDelaySeconds: 30
            periodSeconds: 30
          
          readinessProbe:
            httpGet:
              path: /health/ready
              port: 8000
            initialDelaySeconds: 120    # 모델 로드 대기
            periodSeconds: 10
      
      volumes:
        - name: model-cache
          persistentVolumeClaim:
            claimName: model-cache-pvc

---
apiVersion: v1
kind: Service
metadata:
  name: geulsalim-restore-service
spec:
  selector:
    app: geulsalim-ai
  ports:
    - protocol: TCP
      port: 80
      targetPort: 8000
  type: LoadBalancer
```

```bash
#!/bin/bash
# deploy.sh — 빌드 및 배포 자동화

set -e

VERSION=$(git describe --tags --always)
IMAGE="your-registry/geulsalim-ai:${VERSION}"

echo "1. Docker 이미지 빌드: ${IMAGE}"
docker build -t "${IMAGE}" .

echo "2. 이미지 테스트"
docker run --rm --gpus all "${IMAGE}" python -c "
import torch
print(f'CUDA: {torch.cuda.is_available()}')
print(f'GPU: {torch.cuda.get_device_name(0) if torch.cuda.is_available() else \"없음\"}')
"

echo "3. 레지스트리 푸시"
docker push "${IMAGE}"

echo "4. K8s 배포 업데이트"
kubectl set image deployment/geulsalim-restore-api \
    restore-api="${IMAGE}" \
    --record

echo "5. 롤아웃 대기"
kubectl rollout status deployment/geulsalim-restore-api

echo "배포 완료: ${IMAGE}"
```

## 함정 카드 — 자주 실수하는 5가지

1. **"CUDA 버전과 PyTorch 버전을 무시한다"** → PyTorch 2.3은 CUDA 11.8 또는 12.1을 요구한다. 이를 무시하고 최신 CUDA 12.4에 설치하면 런타임에 segment fault. 반드시 `nvidia-smi`로 드라이버 버전 확인 후 호환 PyTorch 선택. [pytorch.org/get-started/locally](https://pytorch.org) 에서 매트릭스 확인.

2. **"shm_size를 너무 작게 설정한다"** → PyTorch DataLoader의 `num_workers > 0`은 `/dev/shm` (공유 메모리)를 사용한다. Docker의 기본 shm_size는 64MB로 DataLoader 사용 시 "Bus error" 발생. `shm_size: 2g` 이상 설정.

3. **"모델을 이미지에 번들링한다"** → SDXL 모델 크기 ~7GB. 이미지에 포함하면 docker pull에 수 GB 트래픽. 대신 볼륨 마운트로 호스트에 캐시. 이미지는 코드만, 데이터는 볼륨이 원칙.

4. **"K8s replica를 GPU 수보다 많이 설정한다"** → GPU 노드 1개에 1 GPU, replica 3으로 설정하면 2개의 Pod가 Pending 상태로 영원히 스케줄링 대기. GPU 서비스는 `replica = GPU 노드 수 × GPU/노드`.

5. **"컨테이너 종료 시그널을 처리하지 않는다"** → K8s는 Pod 종료 시 SIGTERM을 보낸다. FastAPI + uvicorn은 기본으로 처리하지만, 학습 중인 모델 서버라면 체크포인트 저장 후 종료하는 `signal.signal(signal.SIGTERM, handler)` 추가 필요.

## 연습문제

**문제 1.** GPU가 없는 개발 환경에서 GPU Docker 이미지를 테스트하려면 어떻게 해야 하는가? `CUDA_VISIBLE_DEVICES`와 CPU fallback을 활용하는 방법을 설명하라.

> **해설:** ① `CUDA_VISIBLE_DEVICES=""` 환경변수로 GPU 감추기. ② `device = "cuda" if torch.cuda.is_available() else "cpu"` 조건 분기. ③ 모델 로드 시 `torch_dtype=torch.float32` (CPU는 float16 지원 안 함). ④ 소형 SD 1.5 모델로 테스트 후 프로덕션에서 SDXL 전환. CI/CD 파이프라인에서 GPU 없이 코드 검증 후 GPU 인스턴스에 배포하는 흐름.

**문제 2.** 다음 docker-compose 설정에서 문제점을 찾아라: `shm_size: 64m, volumes: ["/tmp/models:/models"]`

> **해설:** ① `shm_size: 64m` → DataLoader num_workers > 0 사용 시 Bus error. 최소 `2g` 필요. ② `/tmp/models` → 시스템 재시작 또는 tmpfs 정리 시 모델 캐시 소멸. `/data/models` 또는 `/opt/models` 같은 영구 경로 사용. 추가: 볼륨 마운트에서 `:ro` (read-only) 플래그를 추가하면 컨테이너가 실수로 모델을 덮어쓰는 것을 방지.

**문제 3.** K8s에서 GPU Pod가 `CrashLoopBackOff` 상태다. 디버깅 순서를 설명하라.

> **해설:** ① `kubectl logs <pod-name> --previous` — 이전 컨테이너 로그 확인 ② `kubectl describe pod <pod-name>` — 이벤트 섹션에서 OOM 또는 CUDA 오류 확인 ③ `kubectl exec -it <pod-name> -- nvidia-smi` — GPU 인식 여부 ④ `kubectl get events --sort-by='.lastTimestamp'` — 네임스페이스 이벤트 ⑤ 간단한 테스트 Pod 배포: `kubectl run test --image=nvidia/cuda:12.1.0-base-ubuntu22.04 --restart=Never -- nvidia-smi`.
