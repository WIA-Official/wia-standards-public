## 이론

### 도서관 사서의 비유

트랜스포머를 이해하는 가장 좋은 비유는 도서관 사서다.

당신이 사서에게 "1950년대 한국 전쟁 관련 문학 작품"을 요청한다고 하자. 사서는 도서관에 있는 수천 권의 책 중에서 **관련성이 높은 것들에 집중**해 답을 찾는다. "1950년대", "한국 전쟁", "문학"이라는 키워드에 각각 다른 **가중치**를 부여하면서.

이것이 어텐션(Attention)이다. 입력 문장의 각 단어가 다른 단어들과 얼마나 관련 있는지를 계산하고, 관련성 높은 부분에 집중한다.

### Q, K, V — 세 가지 역할

어텐션은 세 가지 벡터로 동작한다.

**Query (Q)**: 내가 찾고 싶은 것. "전쟁 관련 문학을 찾고 있어."
**Key (K)**: 책들의 색인 카드. "이 책은 전쟁, 1950년대, 시 관련이야."
**Value (V)**: 책의 실제 내용. Q와 K의 유사도에 따라 V를 가중합산한다.

수식으로 쓰면:
```
Attention(Q, K, V) = softmax(Q × K^T / √d_k) × V
```

`√d_k`로 나누는 이유: 차원이 커질수록 내적값이 커져서 softmax가 극단적으로 작동하는 것을 방지한다.

### 세 가지 트랜스포머 구조

**Encoder 전용 (BERT 계열)**
- 입력 문장 전체를 양방향으로 이해
- 텍스트 분류, 감정 분석, 문서 임베딩에 사용
- 텍스트를 생성하지 않고 "이해"만 함

**Decoder 전용 (GPT, LLaMA 계열)**
- 앞의 토큰들만 보면서 다음 토큰을 예측 (인과적 어텐션)
- 텍스트 생성, 코드 생성, 대화에 사용
- 우리가 쓰는 Claude, ChatGPT의 기반

**Encoder-Decoder (T5, BART 계열)**
- 인코더가 입력을 이해하고, 디코더가 출력을 생성
- 번역, 요약, 질의응답에 사용

파인튜닝 목적에 따라 선택이 달라진다. 이미지 복원 설명문 생성 → Decoder. 고문서 텍스트 이해 → Encoder. 한문→한글 번역 → Encoder-Decoder.

### 트랜스포머가 CNN, RNN을 이긴 이유

**RNN의 문제**: 문장을 순서대로 처리해서 느리다. 긴 문장의 앞부분 정보가 뒤로 갈수록 희석된다.

**CNN의 문제**: 지역적(local) 패턴만 본다. 문장 전체의 맥락을 파악하기 어렵다.

**트랜스포머의 강점**: 모든 위치의 토큰이 동시에 서로를 참조한다. 병렬 처리 가능. 긴 거리 의존성 포착.

## 실전 코드

```python
# 어텐션 메커니즘 직접 구현 — 수식을 코드로 이해하기
import numpy as np

def softmax(x: np.ndarray) -> np.ndarray:
    """수치 안정적인 softmax"""
    e_x = np.exp(x - np.max(x, axis=-1, keepdims=True))
    return e_x / e_x.sum(axis=-1, keepdims=True)

def scaled_dot_product_attention(
    Q: np.ndarray,
    K: np.ndarray,
    V: np.ndarray,
    mask: np.ndarray = None
) -> tuple:
    """
    스케일드 닷프로덕트 어텐션 구현
    
    Args:
        Q: Query  행렬 [seq_len, d_k]
        K: Key    행렬 [seq_len, d_k]
        V: Value  행렬 [seq_len, d_v]
        mask: 디코더용 인과적 마스크 (선택)
    
    Returns:
        (출력, 어텐션 가중치)
    """
    d_k = Q.shape[-1]
    
    # 1. Q와 K의 내적: 유사도 계산
    scores = Q @ K.T / np.sqrt(d_k)  # [seq_len, seq_len]
    
    # 2. 마스킹 (디코더: 미래 토큰 참조 금지)
    if mask is not None:
        scores = np.where(mask == 0, -1e9, scores)
    
    # 3. Softmax: 유사도를 확률로 변환
    weights = softmax(scores)  # 각 행의 합 = 1
    
    # 4. Value 가중합산
    output = weights @ V
    
    return output, weights


def demonstrate_attention():
    """어텐션이 무엇을 하는지 시각적으로 확인"""
    np.random.seed(42)
    
    # 간단한 예시: 4개 토큰, 차원 8
    seq_len, d_k = 4, 8
    tokens = ["나는", "한국", "역사를", "좋아한다"]
    
    # 임의 Q, K, V (실제로는 학습된 가중치로 생성)
    Q = np.random.randn(seq_len, d_k)
    K = np.random.randn(seq_len, d_k)
    V = np.random.randn(seq_len, d_k)
    
    output, weights = scaled_dot_product_attention(Q, K, V)
    
    print("=== 어텐션 가중치 행렬 ===")
    print("(행: Query 토큰, 열: Key 토큰에 얼마나 집중하는가)")
    print(f"\n{'':10}", end="")
    for t in tokens:
        print(f"{t:8}", end="")
    print()
    
    for i, token in enumerate(tokens):
        print(f"{token:10}", end="")
        for w in weights[i]:
            print(f"{w:.4f}  ", end="")
        print()
    
    print(f"\n출력 형태: {output.shape}")
    print("각 토큰은 모든 토큰의 Value를 가중합산한 결과를 가진다.")


def causal_mask(seq_len: int) -> np.ndarray:
    """
    디코더용 인과적 마스크 생성
    미래 위치는 0, 과거/현재는 1
    """
    mask = np.tril(np.ones((seq_len, seq_len)))
    return mask


def demonstrate_causal_attention():
    """GPT 스타일 디코더 어텐션"""
    seq_len, d_k = 4, 8
    tokens = ["나는", "역사를", "좋아", "한다"]
    
    Q = np.random.randn(seq_len, d_k)
    K = np.random.randn(seq_len, d_k)
    V = np.random.randn(seq_len, d_k)
    
    mask = causal_mask(seq_len)
    output, weights = scaled_dot_product_attention(Q, K, V, mask)
    
    print("\n=== 인과적 어텐션 가중치 (GPT 스타일) ===")
    print("(미래 토큰은 참조 불가 — 0으로 마스킹)")
    print(f"\n{'':8}", end="")
    for t in tokens:
        print(f"{t:8}", end="")
    print()
    
    for i, token in enumerate(tokens):
        print(f"{token:8}", end="")
        for j, w in enumerate(weights[i]):
            if j > i:
                print(f"{'---':8}", end="")  # 마스킹된 위치
            else:
                print(f"{w:.4f}  ", end="")
        print()


if __name__ == "__main__":
    demonstrate_attention()
    demonstrate_causal_attention()
```

## 함정 카드 — 자주 실수하는 5가지

1. **"트랜스포머 = GPT"** → GPT는 트랜스포머의 한 구현체다. BERT, T5, ViT(이미지), Whisper(음성)도 모두 트랜스포머 기반이다. 아키텍처(설계도)와 구현체(제품)를 구분해야 한다.

2. **"어텐션이 클수록 무조건 좋다"** → 어텐션 헤드 수, 레이어 수가 많을수록 표현력은 올라가지만 VRAM과 연산량이 제곱으로 증가한다. 파인튜닝 목적에는 과도한 크기가 오히려 불리하다.

3. **"`d_k`로 나누는 이유를 모른다"** → 차원이 커지면 내적값의 분산이 커진다. 분산이 크면 softmax가 거의 0/1로 극단화되어 그래디언트 소실이 발생한다. `√d_k`로 나눠 분산을 정규화한다.

4. **"인코더와 디코더 어텐션의 차이를 모른다"** → 인코더는 양방향(모든 토큰 참조), 디코더는 단방향(과거만 참조). 마스크 유무가 핵심이다.

5. **"위치 인코딩을 빠뜨린다"** → 트랜스포머는 순서 정보가 없다. "나는 너를 좋아해"와 "너를 나는 좋아해"를 구분하려면 위치 정보를 별도로 추가해야 한다. Positional Encoding이 그 역할이다.

## 연습문제

**문제 1.** 인코더 전용, 디코더 전용, 인코더-디코더 구조 중 "고문서 이미지에서 텍스트를 추출해 현대 한국어로 번역"하는 작업에 가장 적합한 것은? 이유를 설명하라.

> **해설:** 인코더-디코더. 인코더가 고문서 이미지/텍스트 입력을 이해하고, 디코더가 현대 한국어를 생성한다. 입력과 출력이 다른 형태일 때 Encoder-Decoder가 최적이다.

**문제 2.** `scaled_dot_product_attention()` 함수에서 `mask` 파라미터를 제거하고 인코더 어텐션으로 사용하면 어떤 변화가 생기는가?

> **해설:** 모든 토큰이 모든 위치를 참조할 수 있게 된다. 가중치 행렬이 상삼각 부분도 채워진다. 이것이 양방향(bidirectional) 어텐션이며 BERT 스타일이다.

**문제 3.** 시퀀스 길이가 512인 경우 어텐션 가중치 행렬의 크기는? 이것이 왜 "긴 문서 처리"의 병목이 되는지 설명하라.

> **해설:** 512 × 512 = 262,144개 원소. 멀티헤드 어텐션에서는 헤드 수 × 배치 크기만큼 복사된다. 시퀀스 길이가 n이면 메모리는 O(n²)로 증가한다. 4096 토큰이면 16M, 32K 토큰이면 1B — 이것이 "긴 컨텍스트 창"이 어려운 이유다.
