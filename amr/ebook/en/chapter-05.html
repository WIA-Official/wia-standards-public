🔌 Chapter 5: Phase 2 - API Interface

Phase 2 of the WIA-AMR standard defines the application programming interfaces that enable software systems to interact with AMRs and fleet management platforms. This chapter covers the complete REST API specification, authentication mechanisms, and practical implementation patterns using the official WIA-AMR SDK.

5.1 API Design Philosophy

The WIA-AMR API follows RESTful principles with additional conventions that ensure consistency and ease of use across implementations:

Core Design Principles

  • Resource-Oriented: APIs are organized around resources (robots, tasks, maps) with standard CRUD operations
  • Consistent Naming: Plural nouns for collections, lowercase with hyphens for multi-word paths
  • Versioned Endpoints: All APIs include version number in path (/api/v1/)
  • JSON by Default: Request and response bodies use JSON format
  • Pagination: Collection endpoints support cursor-based pagination
  • Filtering: Consistent query parameter patterns for filtering results

Base URL Structure

https://{host}/api/v1/{resource}/{id?}/{sub-resource?}

Examples:
GET  /api/v1/robots                  # List all robots
GET  /api/v1/robots/AMR-001          # Get specific robot
GET  /api/v1/robots/AMR-001/tasks    # Get robot's tasks
POST /api/v1/tasks                   # Create new task
PUT  /api/v1/tasks/TASK-001/cancel   # Cancel task

Task Status Transitions

Status Description Valid Transitions
pending Task created, awaiting assignment assigned, cancelled
assigned Robot assigned in_progress, cancelled
in_progress Robot executing task completed, failed, cancelled
completed Task successfully finished (terminal)
failed Task could not complete pending (retry)
cancelled Task cancelled (terminal)

5.5 Authentication and Authorization

OAuth 2.0 Client Credentials Flow

// Step 1: Request access token
POST /oauth/token
Content-Type: application/x-www-form-urlencoded

grant_type=client_credentials
&client_id=your_client_id
&client_secret=your_client_secret
&scope=robots:read robots:write tasks:read tasks:write

Response: 200 OK
{
  "access_token": "eyJhbGciOiJSUzI1NiIsInR5cCI6IkpXVCJ9...",
  "token_type": "Bearer",
  "expires_in": 3600,
  "scope": "robots:read robots:write tasks:read tasks:write"
}

// Step 2: Use token in API requests
GET /api/v1/robots
Authorization: Bearer eyJhbGciOiJSUzI1NiIsInR5cCI6IkpXVCJ9...

API Key Authentication (Alternative)

// Using API Key header
GET /api/v1/robots
X-API-Key: wia_live_abc123xyz456

// Or as query parameter
GET /api/v1/robots?api_key=wia_live_abc123xyz456

Permission Scopes

Scope Description
robots:read Read robot status and telemetry
robots:write Control robots (pause, resume, navigate)
tasks:read Read task information
tasks:write Create, update, cancel tasks
fleet:admin Fleet management operations
maps:read Read map data
maps:write Update maps and zones

Standard Error Codes

HTTP Status Code Description
400 INVALID_REQUEST Malformed request body or parameters

403 FORBIDDEN Insufficient permissions

409 CONFLICT Resource state conflict

5.7 Rate Limiting

API rate limits are communicated via response headers:

HTTP/1.1 200 OK
X-RateLimit-Limit: 1000
X-RateLimit-Remaining: 985
X-RateLimit-Reset: 1705330800

// When rate limited
HTTP/1.1 429 Too Many Requests
Retry-After: 60
{
  "error": {
    "code": "RATE_LIMITED",
    "message": "Rate limit exceeded. Please retry after 60 seconds."
  }
}

5.8 SDK Implementation

// TypeScript SDK usage example
import { WIAAMRClient } from '@wia/amr-sdk';

// Initialize client
const client = new WIAAMRClient({
  endpoint: 'https://fleet.example.com/api/v1',
  auth: {
    type: 'oauth2',
    clientId: process.env.WIA_CLIENT_ID,
    clientSecret: process.env.WIA_CLIENT_SECRET
  }
});

// List robots with filtering
const robots = await client.robots.list({
  status: 'idle',
  limit: 10
});

// Get specific robot
const robot = await client.robots.get('AMR-001');
console.log(`Battery: ${robot.battery.stateOfCharge}%`);

// Create transport task
const task = await client.tasks.create({
  taskType: 'transport',
  priority: 5,
  source: { nodeId: 'PICK-15' },
  destination: { nodeId: 'PACK-03' }
});

// Monitor task status
client.tasks.onStatusChange(task.taskId, (status) => {
  console.log(`Task ${task.taskId}: ${status}`);
});

// Navigate robot to position
await client.robots.goto('AMR-001', {
  destination: { position: { x: 10, y: 15 } }
});

// Get fleet status
const fleetStatus = await client.fleet.getStatus();
console.log(`Active robots: ${fleetStatus.summary.byStatus.navigating}`);

5.9 Webhooks and Events

The WIA-AMR API supports webhooks for real-time event notifications:

// Register webhook
POST /api/v1/webhooks
{
  "url": "https://your-server.com/wia-events",
  "events": ["task.completed", "robot.error", "robot.low_battery"],
  "secret": "your_webhook_secret"
}

// Webhook payload example
POST https://your-server.com/wia-events
X-WIA-Signature: sha256=abc123...
{
  "eventId": "evt_12345",
  "eventType": "task.completed",
  "timestamp": "2025-01-15T14:45:00Z",
  "data": {
    "taskId": "TASK-2025-00001",
    "robotId": "AMR-001",

    "result": "success"
  }
}
