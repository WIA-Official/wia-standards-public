📡 Chapter 6: Phase 3 - Protocol

Phase 3 of the WIA-AMR standard defines the real-time communication protocols that enable AMRs to exchange messages with fleet management systems and coordinate with other robots. Built on VDA 5050 compatibility with significant extensions, this chapter covers the complete protocol specification for mission-critical AMR communication.

6.1 Protocol Architecture

The WIA-AMR protocol layer operates between the API layer (Phase 2) and the physical robot systems, handling real-time message exchange that requires lower latency and higher reliability than REST APIs can provide:

Communication Patterns

Pattern Use Case Transport
Request-Response Commands requiring acknowledgment MQTT QoS 1/2
Publish-Subscribe State broadcasting, telemetry MQTT QoS 0/1
Streaming Continuous sensor data WebSocket
Event-Driven Safety alerts, status changes MQTT QoS 2

6.2 VDA 5050 Compatibility

VDA 5050 is the industry-standard interface developed by the German Mechanical Engineering Association (VDMA) for communication between fleet management systems and autonomous vehicles. WIA-AMR Phase 3 is fully compatible with VDA 5050 version 2.0:

VDA 5050 Message Types

Message Direction Purpose
Order FMS → Robot Navigation and task commands
State Robot → FMS Robot status reporting
Instant Action FMS → Robot Immediate commands
Visualization Robot → FMS Position for display
Connection Bidirectional Connectivity status

MQTT Topic Structure (VDA 5050)

// VDA 5050 topic pattern
{interface}/{version}/{manufacturer}/{serialNumber}/{topic}

// Example topics
wia/v2/mir/AMR-001/order        // Orders from FMS to robot
wia/v2/mir/AMR-001/state        // State from robot to FMS
wia/v2/mir/AMR-001/instantAction // Immediate commands
wia/v2/mir/AMR-001/visualization // Position updates
wia/v2/mir/AMR-001/connection    // Connection status

6.7 Connection Management

Connection State Machine

State Description Actions
DISCONNECTED No connection Attempt connection
CONNECTING Connection in progress Wait for handshake
ONLINE Connected, operational Normal operation
OFFLINE Lost connection Reconnect, safe mode

Heartbeat and Keep-Alive

6.8 Quality of Service

Message Type QoS Level Rationale
Safety alerts QoS 2 Exactly once delivery critical
Orders QoS 1 At least once, robot confirms
State QoS 1 Reliable status updates
Visualization QoS 0 High frequency, loss acceptable

Emergency stop <50ms On event

📌 Chapter Summary: Key Takeaways

  1. VDA 5050 Compatible: WIA-AMR Phase 3 is fully compatible with VDA 5050 v2.0
