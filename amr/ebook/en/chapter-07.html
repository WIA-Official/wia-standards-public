🔗 Chapter 7: Phase 4 - Integration

Phase 4 represents the culmination of WIA-AMR standardization, enabling seamless integration between AMR systems and enterprise software platforms. This chapter covers integration patterns, standard adapters for major WMS/ERP systems, analytics pipelines, and monitoring dashboards that transform AMR deployments into fully connected automation ecosystems.

7.1 Integration Architecture

The WIA-AMR integration architecture follows a layered approach that decouples AMR operations from enterprise systems while ensuring reliable data flow:

Integration Layers

Layer Components Function
Enterprise WMS, ERP, MES Business logic, order management
Integration Adapters, message broker Protocol translation, routing
Fleet Fleet management system Robot coordination, optimization
Robot AMR systems Physical execution

7.2 WMS Integration Patterns

Order-Driven Integration

The most common integration pattern where warehouse orders drive AMR task creation:

// WMS order received
{
  "orderId": "WMS-ORD-2025-00001",
  "orderType": "pick",
  "priority": "high",
  "items": [
    { "sku": "SKU-12345", "quantity": 2, "location": "A1-015" },
    { "sku": "SKU-67890", "quantity": 1, "location": "B3-042" }
  ],
  "destination": "PACK-STATION-03",
  "deadline": "2025-01-15T15:30:00Z"
}

// Transformed to WIA-AMR tasks
[
  {
    "taskId": "TASK-2025-00001",
    "taskType": "transport",
    "source": { "nodeId": "A1-015" },
    "destination": { "nodeId": "PACK-STATION-03" },
    "priority": 8,
    "metadata": { "wmsOrderId": "WMS-ORD-2025-00001" }
  },
  {
    "taskId": "TASK-2025-00002",
    "taskType": "transport",
    "source": { "nodeId": "B3-042" },
    "destination": { "nodeId": "PACK-STATION-03" },
    "priority": 8,
    "metadata": { "wmsOrderId": "WMS-ORD-2025-00001" }
  }
]

Status Synchronization

// AMR task completion → WMS update
{
  "eventType": "task.completed",
  "taskId": "TASK-2025-00001",
  "wmsOrderId": "WMS-ORD-2025-00001",
  "completedAt": "2025-01-15T15:15:30Z",
  "robotId": "AMR-001",
  "metrics": {
    "duration": 245,
    "distance": 85.2,
    "energyUsed": 2.1
  }
}

7.3 Standard WMS Adapters

WIA-AMR provides pre-built adapters for major WMS platforms:

Platform Adapter Integration Method
SAP EWM wia-adapter-sap RFC/BAPI, OData
Manhattan Associates wia-adapter-manhattan REST API, MQ
Blue Yonder wia-adapter-blueyonder REST API
Oracle WMS wia-adapter-oracle REST, JMS
Infor WMS wia-adapter-infor REST API

Adapter Configuration Example

// SAP EWM adapter configuration
{
  "adapter": "wia-adapter-sap",
  "version": "1.2.0",
  "connection": {
    "host": "sap-ewm.example.com",
    "client": "100",
    "language": "EN",
    "auth": {
      "type": "oauth2",
      "tokenUrl": "https://sap-ewm.example.com/oauth/token"
    }
  },
  "mapping": {
    "orderType": {
      "PICK": "transport",
      "PUTAWAY": "transport",
      "REPLENISHMENT": "transport"
    },
    "locations": {
      "format": "SAP_STORAGE_BIN",
      "prefix": "WH01-"
    }
  },
  "sync": {
    "orderPollInterval": 5000,
    "statusUpdateMode": "realtime"
  }
}

7.5 Analytics and Reporting

Key Performance Indicators

KPI Description Target
Throughput Tasks completed per hour ≥50/hour/robot
Utilization Active time percentage ≥75%
Task Success Rate Completed vs. failed tasks ≥99%
Average Task Duration Time from assignment to completion ≤10 min
Fleet Uptime Operational availability ≥98%

Analytics Pipeline

// Telemetry data flow

    ↓

    ↓

    ↓

    ↓
Dashboards & Reports

7.6 Monitoring Dashboard

The WIA-AMR standard includes specifications for monitoring dashboards that provide real-time visibility:

Dashboard Components

  • Fleet Map: Real-time robot positions on facility map
  • Status Overview: Robot counts by status (idle, navigating, charging)
  • Task Queue: Pending and in-progress tasks
  • Performance Metrics: KPIs with trend charts
  • Alert Panel: Active alerts and warnings
  • Battery Status: Fleet battery levels and charging schedule

弘益人間 (Hongik Ingan)

"Benefit All Humanity"

The WIA-AMR integration layer embodies this philosophy by democratizing enterprise connectivity—making advanced WMS integration accessible to organizations of all sizes through standardized adapters rather than expensive custom development.
