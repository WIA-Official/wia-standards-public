6.1 실시간 통신의 필요성

앞서 살펴본 REST API는 요청-응답(Request-Response) 패턴에 적합합니다. 클라이언트가 요청을 보내면 서버가 응답하는 단방향 통신입니다. 그러나 AMR 시스템에서는 이것만으로는 부족합니다.

실시간 양방향 통신이 필요한 시나리오:

로봇 상태 스트리밍: 로봇의 위치, 속도, 배터리 상태가 초당 여러 번 변경됩니다. 폴링(polling) 방식으로는 지연이 발생하고 서버 부하가 증가합니다.

즉각적인 명령 전달: 비상 정지, 경로 변경 등 긴급 명령은 지연 없이 전달되어야 합니다. HTTP의 연결 오버헤드가 문제가 됩니다.

다중 로봇 조율: 여러 로봇이 동일한 영역에서 작업할 때, 실시간으로 위치와 경로 정보를 공유해야 충돌을 방지할 수 있습니다.

이벤트 알림: 작업 완료, 오류 발생, 배터리 부족 등의 이벤트를 즉시 관제 시스템에 알려야 합니다.

통신 패턴 적합한 프로토콜 예시
요청-응답 REST API (HTTP) 로봇 정보 조회, 작업 생성
실시간 스트리밍 MQTT, WebSocket 위치 업데이트, 센서 데이터
발행-구독 MQTT 이벤트 알림, 상태 변경
양방향 메시징 WebSocket 대화형 제어, 대시보드

6.2 MQTT 프로토콜

MQTT(Message Queuing Telemetry Transport)는 경량 발행-구독(Publish-Subscribe) 메시징 프로토콜입니다. IoT와 M2M(Machine-to-Machine) 통신에 널리 사용되며, WIA-AMR의 주요 실시간 통신 프로토콜입니다.

6.2.1 MQTT 기본 개념

브로커(Broker): 메시지를 중계하는 중앙 서버입니다. 발행자와 구독자가 직접 연결되지 않고 브로커를 통해 통신합니다.

토픽(Topic): 메시지가 전달되는 채널입니다. 계층적 구조로 구성됩니다 (예: /wia/amr/robots/amr-001/state).

발행(Publish): 특정 토픽에 메시지를 보냅니다.

구독(Subscribe): 특정 토픽의 메시지를 수신하도록 등록합니다. 와일드카드(+, #) 지원.

QoS(Quality of Service): 메시지 전달 보장 수준을 정의합니다.

QoS 레벨 설명 사용 시나리오
QoS 0 최대 1회 전달 (fire and forget) 센서 데이터 (손실 허용)
QoS 1 최소 1회 전달 (중복 가능) 상태 업데이트
QoS 2 정확히 1회 전달 중요 명령, 트랜잭션

6.2.2 WIA-AMR 토픽 구조

WIA-AMR은 표준화된 토픽 구조를 정의합니다:

# 토픽 구조: /wia/amr/{version}/{category}/{identifier}/{type} # 로봇 상태 /wia/amr/v1/robots/{robotId}/state # 로봇 상태 발행 /wia/amr/v1/robots/{robotId}/telemetry # 텔레메트리 데이터 /wia/amr/v1/robots/{robotId}/events # 이벤트 알림 # 명령 /wia/amr/v1/robots/{robotId}/commands # 명령 수신 /wia/amr/v1/robots/{robotId}/commands/ack # 명령 확인 응답 # 플릿 /wia/amr/v1/fleet/status # 전체 플릿 상태 /wia/amr/v1/fleet/alerts # 플릿 경고 # 시스템 /wia/amr/v1/system/heartbeat # 시스템 하트비트 /wia/amr/v1/system/config # 설정 변경

💡 토픽 와일드카드

• +: 단일 레벨 와일드카드

예: /wia/amr/v1/robots/+/state - 모든 로봇의 상태

• #: 다중 레벨 와일드카드

예: /wia/amr/v1/robots/amr-001/# - amr-001의 모든 메시지

6.3 VDA 5050 호환성

VDA 5050은 독일 자동차산업협회(VDA)가 개발한 AGV/AMR 통신 표준입니다. 유럽 자동차 산업에서 광범위하게 채택되었으며, WIA-AMR은 VDA 5050과 완전히 호환됩니다.

6.3.1 VDA 5050 핵심 개념

마스터 컨트롤: 중앙 관제 시스템으로, 작업 할당, 경로 계획, 트래픽 관리를 담당합니다.

Order: 마스터 컨트롤이 AGV에 전송하는 이동 명령입니다. 노드와 엣지로 구성된 경로를 포함합니다.

State: AGV가 마스터 컨트롤에 보고하는 현재 상태입니다. 위치, 속도, 배터리, 에러 등을 포함합니다.

Visualization: 시각화를 위한 고빈도 위치 데이터입니다.

6.4 WebSocket 통신

WebSocket은 브라우저 기반 클라이언트와의 실시간 양방향 통신에 사용됩니다. 대시보드, 모니터링 UI, 원격 제어 인터페이스에 적합합니다.

6.4.1 WebSocket vs MQTT

특성 MQTT WebSocket
사용 환경 IoT, 백엔드 브라우저, 프론트엔드
메시지 패턴 발행-구독 양방향 스트림
QoS 0, 1, 2 지원 없음 (TCP 신뢰)
오버헤드 매우 낮음 낮음
브라우저 지원 MQTT over WS 필요 네이티브

6.4.2 WebSocket 메시지 형식

// 연결 설정 const ws = new WebSocket('wss://api.example.com/ws/v1'); // 인증 ws.onopen = () => { ws.send(JSON.stringify({ type: 'AUTH', token: 'Bearer eyJhbGciOiJSUzI1NiIs...' })); }; // 구독 요청 ws.send(JSON.stringify({ type: 'SUBSCRIBE', channels: [ 'robots.*.state', 'fleet.alerts' ] })); // 메시지 수신 ws.onmessage = (event) => { const message = JSON.parse(event.data); switch (message.type) { case 'ROBOT_STATE': updateRobotPosition(message.data); break; case 'ALERT': showNotification(message.data); break; } }; // 명령 전송 ws.send(JSON.stringify({ type: 'COMMAND', robotId: 'amr-001', command: { commandType: 'PAUSE', reason: '수동 일시정지' } }));

6.5 다중 로봇 조율

여러 로봇이 동일한 환경에서 작업할 때, 효율적인 조율이 필수입니다. WIA-AMR은 다중 로봇 조율을 위한 프로토콜 패턴을 정의합니다.

6.5.1 트래픽 관리

로봇 간 충돌을 방지하고 효율적인 경로를 보장하기 위한 중앙 집중식 트래픽 관리:

영역 예약 시스템: 로봇이 특정 영역을 통과하기 전에 예약을 요청합니다. 마스터 컨트롤이 충돌 검사 후 승인/거부합니다.

우선순위 기반 양보: 우선순위가 높은 작업을 수행하는 로봇이 우선권을 가집니다. 낮은 우선순위 로봇은 대기하거나 우회합니다.

교착 상태 감지 및 해결: 순환 대기 상황을 감지하고 자동으로 해결합니다.

⚠️ 분산 vs 중앙 집중 조율

WIA-AMR은 기본적으로 중앙 집중식 트래픽 관리를 권장합니다. 분산 방식은 확장성이 좋지만 복잡한 충돌 상황에서 비효율적일 수 있습니다. 대규모 플릿(100대 이상)에서는 계층적 하이브리드 접근을 고려하세요.

6.5.2 조율 프로토콜 예제

// 영역 예약 요청 (로봇 → 마스터 컨트롤) // 토픽: /wia/amr/v1/traffic/reservation/request { "requestId": "res-001", "robotId": "amr-001", "timestamp": "2025-01-15T10:30:00Z", "zones": ["zone-A1", "zone-A2"], "estimatedDuration": 30, "priority": 80, "currentPosition": { "x": 10.0, "y": 20.0 } } // 영역 예약 응답 (마스터 컨트롤 → 로봇) // 토픽: /wia/amr/v1/robots/amr-001/traffic/reservation/response { "requestId": "res-001", "status": "GRANTED", "timestamp": "2025-01-15T10:30:00.050Z", "validUntil": "2025-01-15T10:30:30Z", "zones": ["zone-A1", "zone-A2"] } // 영역 해제 (로봇 → 마스터 컨트롤) // 토픽: /wia/amr/v1/traffic/reservation/release { "robotId": "amr-001", "zones": ["zone-A1"], "timestamp": "2025-01-15T10:30:15Z" }
