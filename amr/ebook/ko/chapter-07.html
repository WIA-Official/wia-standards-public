🔗 4단계: 통합

WMS, ERP, MES와의 연동 패턴, 이벤트 기반 아키텍처, 플릿 모니터링 대시보드 구축을 다룹니다.

7.1 시스템 통합 개요

AMR 시스템은 단독으로 작동하지 않습니다. 창고관리시스템(WMS), 전사적자원관리(ERP), 제조실행시스템(MES) 등 기존 기업 시스템과 유기적으로 연동되어야 진정한 가치를 발휘합니다. WIA-AMR 표준의 4단계는 이러한 통합을 위한 패턴과 모범 사례를 정의합니다.

弘益人間
"널리 인간을 이롭게 하라" - 기술은 인간의 업무를 보조하고 효율성을 높여야 합니다

7.1.1 통합의 목표

원활한 데이터 흐름: 시스템 간 정보가 지연 없이 정확하게 전달되어야 합니다. 재고 이동, 작업 상태, 위치 정보가 실시간으로 동기화됩니다.

자동화된 워크플로우: 수동 개입 없이 작업이 자동으로 할당되고 실행되어야 합니다. WMS에서 생성된 피킹 작업이 자동으로 AMR에 전달됩니다.

통합 모니터링: 단일 대시보드에서 전체 운영 현황을 파악할 수 있어야 합니다. AMR 상태와 WMS 재고가 한눈에 보입니다.

확장 가능한 아키텍처: 새로운 시스템이나 로봇을 추가할 때 기존 통합에 영향을 주지 않아야 합니다.

7.1.2 통합 아키텍처 패턴

WIA-AMR은 세 가지 통합 아키텍처 패턴을 지원합니다:

패턴 설명 적합한 환경
점대점(Point-to-Point) 시스템 간 직접 연결 소규모, 시스템 수 적음
허브앤스포크(Hub-and-Spoke) 중앙 통합 허브 경유 중규모, 중앙 관리 필요
이벤트 기반(Event-Driven) 메시지 브로커 통한 비동기 대규모, 실시간, 확장성

7.2 WMS 통합

창고관리시스템(Warehouse Management System)은 AMR 통합의 가장 일반적인 대상입니다. 재고 위치, 주문 처리, 피킹 작업 등을 관리합니다.

7.2.1 작업 할당 흐름

WMS에서 생성된 창고 작업이 AMR에 할당되는 흐름을 정의합니다:

  1. 주문 접수: ERP/OMS에서 주문이 WMS로 전달됩니다.
  2. 피킹 작업 생성: WMS가 재고 위치를 확인하고 피킹 작업을 생성합니다.
  3. 작업 전송: WMS가 WIA-AMR API를 통해 작업을 FMS에 전송합니다.
  4. 로봇 할당: FMS가 적합한 로봇을 선택하고 작업을 할당합니다.
  5. 작업 수행: 로봇이 지정된 위치로 이동하여 작업을 수행합니다.
  6. 완료 보고: 로봇이 작업 완료를 FMS에 보고합니다.
  7. WMS 업데이트: FMS가 WMS에 작업 완료를 알립니다.
  8. 재고 갱신: WMS가 재고 위치와 수량을 업데이트합니다.

7.2.3 위치 매핑

WMS와 AMR 시스템은 서로 다른 위치 표현 방식을 사용합니다. WMS는 논리적 위치(랙/열/단)를 사용하고, AMR은 물리적 좌표(x, y)를 사용합니다.

WMS 위치 AMR 좌표 설명
RACK-A01-02-03 x: 45.5, y: 28.3 A구역 1열 2단 3번
PACK-STATION-01 x: 100.0, y: 50.0 포장 스테이션 1
CHARGE-01 x: 5.0, y: 5.0 충전 스테이션 1

💡 위치 매핑 모범 사례

• 중앙 위치 마스터 데이터베이스를 유지하세요

• 위치 변경 시 양 시스템에 자동 동기화되어야 합니다

• 새 위치 추가 시 좌표 캘리브레이션을 수행하세요

• 위치 ID는 고유하고 의미 있는 명명 규칙을 따르세요

7.3 ERP 통합

전사적자원관리(Enterprise Resource Planning) 시스템과의 통합은 비즈니스 프로세스 전체에 AMR 데이터를 연결합니다.

7.3.1 통합 시나리오

재고 트랜잭션: AMR이 물리적 이동을 완료하면 ERP에 재고 트랜잭션이 생성됩니다.

작업 지시: ERP의 생산 계획이나 출하 지시가 AMR 작업으로 변환됩니다.

비용 추적: AMR 운영 비용(에너지, 유지보수)이 ERP 원가 센터에 반영됩니다.

KPI 리포팅: AMR 성능 지표가 ERP 분석 모듈에 통합됩니다.

7.3.2 SAP 통합 예제

// SAP RFC 호출을 통한 재고 이동 기록 // WIA-AMR FMS → SAP Middleware → SAP ERP // 미들웨어에서 SAP RFC 호출 const sapClient = new SAPClient(config); await sapClient.call('BAPI_GOODSMVT_CREATE', { GOODSMVT_HEADER: { PSTNG_DATE: '2025-01-15', DOC_DATE: '2025-01-15', PR_UNAME: 'AMR_SYSTEM' }, GOODSMVT_CODE: { GM_CODE: '04' // 창고 간 이동 }, GOODSMVT_ITEM: [{ MATERIAL: 'MAT-12345', PLANT: '1000', STGE_LOC: 'WH01', MOVE_TYPE: '311', // 창고 내 이동 ENTRY_QNT: 3, ENTRY_UOM: 'EA', COSTCENTER: 'CC-AMR-01' }] }); // WIA-AMR 이벤트로부터 트리거 eventBus.subscribe('task.completed', async (event) => { if (event.taskType === 'PICK_AND_TRANSPORT') { await recordSAPMovement(event); } });

7.4 이벤트 기반 아키텍처

대규모 시스템 통합에는 이벤트 기반 아키텍처가 적합합니다. 시스템 간 느슨한 결합을 유지하면서 실시간 데이터 흐름을 지원합니다.

7.4.1 이벤트 카탈로그

WIA-AMR은 표준 이벤트 카탈로그를 정의합니다:

이벤트 유형 발행자 구독자 설명
task.created WMS/ERP FMS 새 작업 생성됨
task.assigned FMS WMS, 대시보드 로봇에 작업 할당됨
task.completed FMS WMS, ERP 작업 완료됨
robot.state.changed 로봇 FMS, 대시보드 로봇 상태 변경
robot.error 로봇/FMS 대시보드, 알림 오류 발생
battery.low 로봇 FMS 배터리 부족
inventory.moved FMS WMS, ERP 재고 이동 완료

7.6 분석 및 리포팅

장기적인 운영 최적화를 위해 데이터 분석과 리포팅 시스템이 필요합니다.

7.6.1 데이터 웨어하우스 통합

AMR 운영 데이터를 데이터 웨어하우스에 적재하여 심층 분석을 수행합니다:

히스토리 데이터: 로봇 이동 경로, 작업 이력, 에너지 소비 패턴

예측 분석: 유지보수 필요 시점 예측, 수요 예측

최적화 인사이트: 병목 구간 식별, 경로 최적화 기회

7.6.2 표준 리포트

리포트 주기 주요 내용
일일 운영 요약 일간 처리량, 가동률, 오류 건수
주간 성능 분석 주간 트렌드, 피크 시간대, 효율성
월간 비용 분석 월간 에너지 비용, 유지보수 비용, ROI
유지보수 예측 주간 예상 부품 교체, 서비스 일정

✅ 핵심 요약

• 이벤트 기반 아키텍처로 시스템 간 느슨한 결합 유지

• WMS 통합: 작업 할당, 위치 매핑, 재고 동기화

• ERP 통합: 재고 트랜잭션, 비용 추적, KPI 리포팅

• 실시간 대시보드로 플릿 상태 모니터링

• 데이터 웨어하우스 통합으로 장기 분석 지원
