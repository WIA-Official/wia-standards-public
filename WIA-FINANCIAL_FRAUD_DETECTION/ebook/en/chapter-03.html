3.2 Feature Store Architecture

Technology: Feast (Feature Store)

  • Offline Store: Historical features for model training (Parquet on S3)
  • Online Store: Low-latency features for real-time inference (Redis)
  • Feature Registry: Metadata about features (YAML definitions)

3.2.1 Example Feature Definition

feature_view:
  name: user_transaction_statistics_30d
  entities:
    - customer
  ttl: 86400s  # 24 hours
  features:
    - name: user_avg_transaction_amount_30d
      dtype: FLOAT
    - name: user_total_transactions_30d
      dtype: INT64
  batch_source:
    type: BigQuerySource
    query: |
      SELECT
        customer_id,
        timestamp,
        AVG(amount) as user_avg_transaction_amount_30d,
        COUNT(*) as user_total_transactions_30d
      FROM transactions
      WHERE timestamp >= TIMESTAMP_SUB(CURRENT_TIMESTAMP(), INTERVAL 30 DAY)
      GROUP BY customer_id, DATE(timestamp)

3.4 Feature Engineering Best Practices

3.4.1 Handle Missing Values

  • Numerical features: Median imputation or -999 placeholder
  • Categorical features: "UNKNOWN" category
  • Boolean features: Create "is_missing" indicator

3.4.2 Feature Scaling

# StandardScaler (mean=0, std=1)
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X_train)

# MinMaxScaler (0-1 range)
scaler = MinMaxScaler()
X_scaled = scaler.fit_transform(X_train)

3.4.3 Categorical Encoding

  • One-Hot Encoding: For low-cardinality features (< 10 unique values)
  • Target Encoding: Replace categories with mean fraud rate
  • Frequency Encoding: Replace with count/frequency

    np.triu(np.ones(correlation_matrix.shape), k=1).astype(bool)
)

           if any(upper_triangle[column] > 0.95)]
