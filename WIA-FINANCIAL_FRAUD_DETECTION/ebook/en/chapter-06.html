Chapter 6

Security & Compliance

6.1 Overview

Implement PCI DSS v4.0.1, ISO 27001, and GDPR compliant fraud detection systems. This chapter covers encryption, tokenization, secure data handling, audit logging, and regulatory reporting.

6.2 Key Topics

  • PCI DSS v4.0.1: Payment card data security requirements
  • Encryption: AES-256 at rest, TLS 1.3 in transit
  • Tokenization: Replace PAN with tokens (never store full card numbers)
  • Key Management: HashiCorp Vault, AWS KMS for secrets
  • GDPR Compliance: Right to access, erasure, data portability
  • Audit Logging: ELK Stack for centralized log management
  • Network Security: VPC, security groups, WAF, DDoS protection
  • Container Security: Image scanning, pod security policies

6.3 Implementation Example

# Encrypt sensitive data with AES-256
from cryptography.fernet import Fernet
import base64
import hashlib

def generate_key(password: str) -> bytes:
    """Generate encryption key from password"""
    return base64.urlsafe_b64encode(
        hashlib.sha256(password.encode()).digest()
    )

def encrypt_data(data: str, key: bytes) -> str:
    """Encrypt data with Fernet (AES-256)"""
    f = Fernet(key)
    encrypted = f.encrypt(data.encode())
    return encrypted.decode()

def decrypt_data(encrypted: str, key: bytes) -> str:
    """Decrypt data"""
    f = Fernet(key)
    decrypted = f.decrypt(encrypted.encode())
    return decrypted.decode()

# Example: Encrypt card PAN
key = generate_key("my_secret_password")
pan = "4242424242424242"
encrypted_pan = encrypt_data(pan, key)
print(f"Encrypted: {encrypted_pan}")

6.4 Best Practices

  • Never store full PAN (Primary Account Number) - use tokenization
  • Never store CVV/CVC (prohibited by PCI DSS)
  • Encrypt data at rest (AES-256) and in transit (TLS 1.3)
  • Rotate encryption keys every 90 days
  • Implement principle of least privilege for access control
  • Log all access to sensitive data for audit trails
