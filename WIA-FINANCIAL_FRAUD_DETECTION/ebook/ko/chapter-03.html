3.1 80+ 특성 프레임워크

특성 엔지니어링은 사기 탐지에서 가장 중요한 구성 요소입니다. 적절한 특성은 70% 탐지율과 95% 이상 탐지율의 차이를 만들 수 있습니다. 이 장에서는 80개 이상의 사기 탐지 특성을 생성하는 체계적인 접근 방식을 다룹니다.

3.2 특성 저장소 아키텍처

기술: Feast (Feature Store)

  • 오프라인 저장소: 모델 학습을 위한 과거 특성 (S3의 Parquet)
  • 온라인 저장소: 실시간 추론을 위한 저지연 특성 (Redis)
  • 특성 레지스트리: 특성에 대한 메타데이터 (YAML 정의)

3.2.1 특성 정의 예시

feature_view:
  name: user_transaction_statistics_30d
  entities:
    - customer
  ttl: 86400s  # 24시간
  features:
    - name: user_avg_transaction_amount_30d
      dtype: FLOAT
    - name: user_total_transactions_30d
      dtype: INT64
  batch_source:
    type: BigQuerySource
    query: |
      SELECT
        customer_id,
        timestamp,
        AVG(amount) as user_avg_transaction_amount_30d,
        COUNT(*) as user_total_transactions_30d
      FROM transactions
      WHERE timestamp >= TIMESTAMP_SUB(CURRENT_TIMESTAMP(), INTERVAL 30 DAY)
      GROUP BY customer_id, DATE(timestamp)

3.4 특성 엔지니어링 모범 사례

3.4.1 결측값 처리

  • 숫자형 특성: 중앙값 대체 또는 -999 플레이스홀더
  • 범주형 특성: "UNKNOWN" 범주
  • 부울형 특성: "is_missing" 표시자 생성

3.4.2 특성 스케일링

# StandardScaler (평균=0, 표준편차=1)
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X_train)

# MinMaxScaler (0-1 범위)
scaler = MinMaxScaler()
X_scaled = scaler.fit_transform(X_train)

3.4.3 범주형 인코딩

  • 원-핫 인코딩: 낮은 카디널리티 특성용 (< 10개 고유값)
  • 타겟 인코딩: 범주를 평균 사기율로 대체
  • 빈도 인코딩: 개수/빈도로 대체

3.5.2 상관관계 분석

# 높은 상관관계를 가진 특성 제거 (> 0.95)
correlation_matrix = X_train.corr().abs()
upper_triangle = correlation_matrix.where(
    np.triu(np.ones(correlation_matrix.shape), k=1).astype(bool)
)
to_drop = [column for column in upper_triangle.columns
           if any(upper_triangle[column] > 0.95)]
X_train_reduced = X_train.drop(columns=to_drop)
