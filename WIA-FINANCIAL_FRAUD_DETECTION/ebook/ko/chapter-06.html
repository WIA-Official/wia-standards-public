Chapter 6

보안 및 규정 준수

6.1 개요

PCI DSS v4.0.1, ISO 27001, GDPR을 준수하는 사기 탐지 시스템을 구현합니다. 이 장에서는 암호화, 토큰화, 안전한 데이터 처리, 감사 로깅, 규제 보고를 다룹니다.

6.2 주요 내용

  • PCI DSS v4.0.1: 결제 카드 데이터 보안 요구사항
  • 암호화: 저장 시 AES-256, 전송 시 TLS 1.3
  • 토큰화: PAN을 토큰으로 교체 (전체 카드 번호 저장 금지)
  • 키 관리: 비밀을 위한 HashiCorp Vault, AWS KMS
  • GDPR 준수: 액세스, 삭제, 데이터 이동성에 대한 권리
  • 감사 로깅: 중앙 집중식 로그 관리를 위한 ELK Stack
  • 네트워크 보안: VPC, 보안 그룹, WAF, DDoS 보호

6.3 핵심 개념

보안은 깊이 있는 방어가 필요합니다: (1) 애플리케이션 계층 - 입력 검증, 출력 인코딩, CSRF 보호. (2) API 계층 - OAuth 2.0, JWT, API 키 순환, 속도 제한. (3) 데이터 계층 - 저장 시 암호화(AES-256), 전송 시 암호화(TLS 1.3), PAN 토큰화. (4) 네트워크 계층 - VPC 격리, 보안 그룹, WAF, DDoS 보호. (5) 컨테이너 계층 - 이미지 스캔, 런타임 보호, Pod 보안 정책. (6) 인프라 계층 - 데이터 센터 보안(클라우드 제공업체 책임).

핵심 요약

PCI DSS v4.0.1은 결제 카드 데이터 보안을 요구합니다. AES-256(저장 시) 및 TLS 1.3(전송 시)으로 암호화가 필수입니다. PAN 토큰화는 전체 카드 번호 저장을 방지합니다(CVV 저장 금지). 비밀을 위해 HashiCorp Vault 또는 AWS KMS를 사용합니다. GDPR은 액세스, 삭제, 데이터 이동성에 대한 권리를 부여합니다. 감사 로깅(ELK Stack)은 중앙 집중식 로그 관리를 제공합니다. 네트워크 보안에는 VPC, 보안 그룹, WAF, DDoS 보호가 포함됩니다. 컨테이너 보안에는 이미지 스캔 및 Pod 보안 정책이 포함됩니다.
