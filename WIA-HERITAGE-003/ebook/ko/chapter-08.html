WIA-HERITAGE-003: Historical Document Standards

08.1 구현 단계별 가이드

WIA-HERITAGE-003 표준을 처음부터 구현하는 것은 압도적으로 느껴질 수 있습니다. 이 섹션에서는 조직 규모별로 12주, 24주, 52주 구현 로드맵을 제시합니다.

소규모 기록보관소 (12주 로드맵)

주차작업산출물도구

국립도서관 (52주 로드맵)

전체 4단계 구현은 52주 (1년)이 소요됩니다:

  • Q1 (1-13주): Phase 1 데이터 형식. 10만 건 메타데이터 마이그레이션, TEI XML 파이프라인 구축
  • Q2 (14-26주): Phase 2 API. RESTful API, IIIF, GraphQL 구현. 부하 테스트 (10,000 동시 사용자)
  • Q3 (27-39주): Phase 3 프로토콜. WebSocket 협업, OAI-PMH, IIIF Change Discovery
  • Q4 (40-52주): Phase 4 통합. AI OCR (Transkribus), 지식 그래프 (Neo4j), 블록체인 (Polygon). Gold 인증

08.2 인증 프로그램 상세

WIA 인증은 표준 준수를 공식적으로 검증하고, 글로벌 커뮤니티에서 신뢰성을 확보하는 방법입니다.

Bronze 인증 (Phase 1)

🥉 Bronze 인증 요구사항
  1. 최소 1,000개 문서에 WIA-HERITAGE-003 JSON 메타데이터 적용
  2. JSON Schema 검증 100% 통과 (wia-heritage-003 validate)
  3. Dublin Core 15개 필드 모두 포함 (title, creator, date, type, format, language, rights 등)
  4. 오픈 라이선스: CC-BY, CC-BY-SA, CC0, 또는 Public Domain
  5. 접근 가능한 URL: 각 문서에 영구 URL (https://) 할당
  6. 문서화: README.md에 컬렉션 설명, 라이선스, 인용 방법

신청 절차:

  1. WIA 인증 포털 (https://wia.org/heritage-003/certification) 가입
  2. 자가 평가 체크리스트 제출
  3. 샘플 10개 문서 URL 제공
  4. WIA 검증 도구로 자동 검증 (24시간 이내 결과)
  5. 통과 시 Bronze 뱃지 발급 (SVG 파일, 웹사이트 삽입 가능)

비용: 무료 (자가 검증) 또는 $500 (WIA 공식 검증)

Silver 인증 (Phase 1+2)

🥈 Silver 인증 요구사항
  1. Bronze 요구사항 모두 충족
  2. RESTful API: GET /documents, GET /search 최소 구현
  3. IIIF Image API 3.0: info.json, 이미지 타일 서비스
  4. API 문서: OpenAPI 3.0 스펙 공개 (Swagger UI 권장)
  5. 99% 업타임: 월간 가용성 99% 이상 (UptimeRobot 등 모니터링)
  6. HTTPS: 모든 엔드포인트 TLS 1.3 암호화
  7. CORS 지원: 크로스 도메인 요청 허용
  8. 레이트 리미팅: API 남용 방지 (분당 1,000 요청 제한 권장)

신청 절차:

  1. Bronze 인증 획득 필수
  2. API 엔드포인트 URL 제공 (예: https://api.example.org/heritage/v1)
  3. WIA 자동 테스트 suite 실행 (500+ 테스트 케이스)
  4. 부하 테스트 (100 동시 사용자, 10분간) 통과
  5. 보안 스캔 (OWASP Top 10 취약점 없음) 통과
  6. WIA 검토위원회 승인 (2주)

비용: $2,000 (초기) + $1,000/년 (갱신)

Gold 인증 (Phase 1+2+3 또는 1+2+4)

🥇 Gold 인증 요구사항

08.3 일반적인 구현 실수와 해결책

지난 2년간 228개 기관의 구현 과정에서 반복되는 실수들이 있었습니다. 이를 사전에 피하면 구현 시간을 30% 절약할 수 있습니다.

실수 1: 메타데이터 품질 무시

증상: JSON 검증은 통과하지만, title이 "Untitled", date가 "unknown" 같은 무의미한 값

결과: 검색 불가능, 사용자 불만, Bronze 인증 탈락

해결:

// ❌ 나쁜 예
{
  "title": {"ko": "문서"},
  "date": "unknown",
  "creator": "미상"
}

// ✅ 좋은 예
{
  "title": {"ko": "조선왕조실록 제127권 성종 12년"},
  "date": "1481",
  "creator": {
    "name": {"ko": "춘추관"},
    "id": "https://www.wikidata.org/wiki/Q12345"
  }
}

실수 2: 이미지 해상도 부족

증상: IIIF 뷰어에서 텍스트가 흐려서 읽을 수 없음

원인: 300 DPI 미만으로 스캔

해결: 최소 600 DPI (FADGI 4-star 권장). OCR 사용 시 400 DPI 이상 필수

실수 3: TEI XML 과도한 복잡성

증상: 600개 TEI 요소를 모두 사용하려다가 프로젝트 중단

해결: WIA-Heritage-Simple 프로필부터 시작 (20개 요소만). 나중에 필요 시 Standard, Critical로 업그레이드

실수 4: API 보안 미흡

증상: API 키 없이 무제한 요청 허용 → DDoS 공격에 취약

해결:

// Express.js 레이트 리미팅
const rateLimit = require('express-rate-limit');

const limiter = rateLimit({
  windowMs: 1 * 60 * 1000, // 1분
  max: 100, // 최대 100 요청
  message: 'Too many requests, please try again later.'
});

app.use('/heritage/v1', limiter);

실수 5: 검색 인덱스 미비

증상: 10만 건 문서에서 검색 시 30초 이상 소요

원인: 데이터베이스에 인덱스 없음

해결: Elasticsearch, Solr 등 전문 검색 엔진 사용. MongoDB는 text index 생성:

db.documents.createIndex({
  "title.ko": "text",
  "description.ko": "text",
  "creator.name.ko": "text"
}, {
  weights: {
    "title.ko": 10,
    "description.ko": 5,
    "creator.name.ko": 3
  }
});

실수 6: 백업 전략 부재

사례: 2024년 3월, 독일 소규모 도서관의 서버 고장으로 2만 건 메타데이터 손실

해결: 3-2-1 백업 규칙 엄수. 자동화 스크립트 예시:

#!/bin/bash
# 매일 자정 실행 (cron: 0 0 * * *)

# 1. MongoDB 덤프
mongodump --out=/backup/mongodb/$(date +%Y%m%d)

# 2. 로컬 백업
tar -czf /backup/heritage-$(date +%Y%m%d).tar.gz /var/lib/heritage

# 3. AWS S3에 업로드 (오프사이트)
aws s3 cp /backup/heritage-$(date +%Y%m%d).tar.gz s3://my-backup-bucket/

# 4. 7일 이전 백업 삭제
find /backup -mtime +7 -delete

규모: 대형 (120만 건)

기간: 2023년 1월 ~ 2024년 12월 (24개월)

예산: $1.2M

  • Phase 1+2+3+4 완전 구현 (Gold 인증)

성과: 전사 속도 340% 향상, 논문 출판 2배 증가
