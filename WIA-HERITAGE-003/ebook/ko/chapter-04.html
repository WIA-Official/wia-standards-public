Phase 1: 데이터 형식

WIA-HERITAGE-003: Historical Document Standards

04.1 JSON 메타데이터 스키마

WIA-HERITAGE-003의 핵심은 JSON 기반 메타데이터 스키마입니다. 모든 역사 문서는 JSON 파일로 기술되며, 이는 사람이 읽을 수 있으면서도 기계가 처리하기 쉽습니다.

필수 필드 (Required Fields)

모든 문서는 다음 필드를 반드시 포함해야 합니다:

{
  "@context": "https://wia.org/heritage-003/v1/context.json",
  "@type": "HistoricalDocument",
  "id": "https://example.org/documents/joseon-annals-vol127",
  "title": {
    "ko": "조선왕조실록 제127권 성종 12년",
    "en": "Annals of the Joseon Dynasty, Vol. 127, King Seongjong Year 12"
  },
  "date": "1481",
  "language": ["hanja", "ko"],
  "type": "manuscript"
}
필드타입설명예시
@contextstringJSON-LD 컨텍스트 URLhttps://wia.org/heritage-003/v1/context.json
@typestring리소스 타입HistoricalDocument, Manuscript, Letter
idURI고유 식별자 (URI)https://example.org/doc/123
titleobject다국어 제목{"ko": "...", "en": "..."}
datestring생성/작성 날짜 (ISO 8601)"1481", "1796-04-03"
languagearray문서 언어 (ISO 639-3)["ko", "hanja", "en"]
typestring문서 유형manuscript, letter, official, diary

04.4 TEI XML 텍스트 인코딩

전사(transcription)된 텍스트는 TEI P5 XML로 저장합니다. WIA-HERITAGE-003은 세 가지 TEI 프로필을 정의합니다.

WIA-Heritage-Simple 프로필

가장 기본적인 전사. 레이아웃과 구조만 표현:

<TEI xmlns="http://www.tei-c.org/ns/1.0">
  <teiHeader>
    <fileDesc>
      <titleStmt>
        <title>조선왕조실록 성종 12년 1월</title>
      </titleStmt>
      <publicationStmt>
        <publisher>국립중앙도서관</publisher>
        <date>2023</date>
      </publicationStmt>
      <sourceDesc>
        <p>조선왕조실록 원본 필사본</p>
      </sourceDesc>
    </fileDesc>
  </teiHeader>
  <text>
    <body>
      <div type="entry" n="1">
        <p>
          <lb/>辛丑正月初一日癸卯
          <lb/>上行正至禮于昭格殿
          <lb/>遣承旨宣醞于宗廟
        </p>
      </div>
    </body>
  </text>
</TEI>

WIA-Heritage-Standard 프로필

개체명 인식 추가. 인물, 장소, 날짜를 태깅:

<div type="entry" n="1">
  <p>
    <lb/><date when="1481-01-01">辛丑正月初一日癸卯</date>
    <lb/><persName ref="#seongjong">上</persName>行正至禮于
    <placeName ref="#jongmyo">昭格殿</placeName>
    <lb/>遣<persName role="seungji">承旨</persName>宣醞于
    <placeName ref="#shrine">宗廟</placeName>
  </p>
</div>

<!-- 개체 목록 -->
<listPerson>
  <person xml:id="seongjong">
    <persName xml:lang="ko">성종</persName>
    <persName xml:lang="hanja">成宗</persName>
    <birth when="1457">1457년</birth>
    <death when="1494">1494년</death>
    <occupation>조선 9대 왕</occupation>
  </person>
</listPerson>

<listPlace>
  <place xml:id="jongmyo">
    <placeName xml:lang="ko">종묘</placeName>
    <placeName xml:lang="en">Jongmyo Shrine</placeName>
    <location>
      <geo>37.5743 126.9940</geo>
    </location>
  </place>
</listPlace>

WIA-Heritage-Critical 프로필

비평적 편집. 필사자의 삭제, 추가, 수정 기록:

<p>
  <lb/>上曰臣下之
  <choice>
    <sic>罪</sic>
    <corr>非</corr>
  </choice>
  <lb/>可
  <del rend="strikethrough">容恕</del>
  <add place="above">赦免</add>
  也
</p>

이는 "上曰臣下之非可赦免也"로 읽히며, 원본에는 "罪"가 "非"로 교정되고 "容恕"가 삭제된 뒤 위에 "赦免"이 추가된 것을 보여줍니다.

IIIF의 장점

  • 표준 뷰어: Mirador, Universal Viewer, Diva.js 등 20+ 뷰어가 모두 작동
  • 줌/팬: Deep Zoom 기술로 100MP 이미지도 브라우저에서 부드럽게 탐색
  • 비교 보기: 여러 기관의 문서를 나란히 비교
  • 주석: W3C Web Annotation 표준으로 이미지에 주석 추가
  • API 액세스: 이미지 타일을 프로그래밍 방식으로 요청

04.8 장 요약

✅ 핵심 내용
  1. JSON 메타데이터는 7개 필수 필드(@context, @type, id, title, date, language, type)와 Dublin Core 15개 요소를 지원합니다.
  2. 물리적 메타데이터는 재질, 크기, 상태, 인장, 보존 이력을 포함하며, 5단계 상태 평가 기준(excellent~critical) 사용합니다.
  3. 디지털화 메타데이터는 스캐너, 해상도, 색상 관리, FADGI 품질 레벨(1~4-star)을 기록하여 재현성 보장합니다.
  4. TEI XML은 세 가지 프로필 제공: Simple(구조만), Standard(개체명 인식), Critical(비평적 편집).
  5. IIIF 통합으로 Presentation API 3.0 매니페스트 생성. 표준 뷰어(Mirador, UV)로 고해상도 이미지 제공.
  6. 검증 도구: JSON Schema 검증, TEI 프로필 검증, Dublin Core/MARC21 변환 지원. npm으로 설치 가능.

04.9 복습 문제

  1. WIA-HERITAGE-003 JSON 메타데이터의 7개 필수 필드를 나열하고, 각각의 역할을 설명하세요.
  2. FADGI 4-star 품질 레벨의 최소 해상도는 무엇이며, 이러한 고품질 디지털화가 필요한 이유는 무엇인가요?
  3. TEI의 세 가지 프로필(Simple, Standard, Critical)을 비교하고, 각각 어떤 종류의 프로젝트에 적합한지 설명하세요.
  4. IIIF Presentation API 3.0 매니페스트의 기본 구조를 설명하고, Canvas, AnnotationPage, Annotation의 관계를 기술하세요.
  5. 물리적 메타데이터에서 "seals" 필드가 중요한 이유는 무엇인가요? 한국 역사 문서에서 인장의 역할을 논하세요.
  6. MARC21 260 $c 필드(publication date)를 WIA-HERITAGE-003 JSON으로 변환할 때 주의해야 할 점은 무엇인가요?
