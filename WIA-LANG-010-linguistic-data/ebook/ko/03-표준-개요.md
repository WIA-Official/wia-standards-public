# 제3장: WIA-LANG-010 표준 개요

## 개요

> **弘益人間 (홍익인간)** - 널리 인간을 이롭게 하라

WIA-LANG-010 언어 데이터 표준은 언어 자원의 생성, 주석, 저장, 교환을 위한 포괄적인 프레임워크입니다. 이 장에서는 표준의 구조, 설계 원칙, 핵심 구성 요소를 상세히 설명합니다.

## 표준의 계층 구조

WIA-LANG-010은 다음과 같은 계층적 구조로 설계되었습니다:

```
WIA-LANG-010 표준 아키텍처
│
├─ Layer 1: 데이터 모델 계층 (Data Model Layer)
│  ├─ 추상 데이터 모델 (Abstract Data Model)
│  ├─ 개념 스키마 (Conceptual Schema)
│  └─ 관계 정의 (Relationship Definitions)
│
├─ Layer 2: 표현 계층 (Representation Layer)
│  ├─ XML 직렬화 (XML Serialization)
│  ├─ JSON 직렬화 (JSON Serialization)
│  ├─ RDF 직렬화 (RDF Serialization)
│  └─ 이진 형식 (Binary Format)
│
├─ Layer 3: 의미 계층 (Semantic Layer)
│  ├─ 주석 체계 (Annotation Schemes)
│  ├─ 태그셋 정의 (Tagset Definitions)
│  ├─ 관계 유형 (Relation Types)
│  └─ 온톨로지 매핑 (Ontology Mappings)
│
├─ Layer 4: 인터페이스 계층 (Interface Layer)
│  ├─ API 명세 (API Specification)
│  ├─ 쿼리 언어 (Query Language)
│  ├─ 검증 규칙 (Validation Rules)
│  └─ 변환 도구 (Conversion Tools)
│
└─ Layer 5: 응용 계층 (Application Layer)
   ├─ 참조 구현 (Reference Implementations)
   ├─ 도구 및 라이브러리 (Tools and Libraries)
   ├─ 샘플 데이터 (Sample Data)
   └─ 베스트 프랙티스 (Best Practices)
```

## 설계 원칙

### 1. 모듈성 (Modularity)

표준은 독립적으로 사용 가능한 모듈로 구성됩니다:

```yaml
modules:
  core:
    name: "WIA-LANG-010-CORE"
    description: "핵심 데이터 모델 및 메타데이터"
    required: true
    dependencies: []

  morphology:
    name: "WIA-LANG-010-MORPH"
    description: "형태론적 주석"
    required: false
    dependencies: ["core"]
    includes:
      - tokenization
      - morpheme_segmentation
      - pos_tagging
      - lemmatization

  syntax:
    name: "WIA-LANG-010-SYNTAX"
    description: "통사론적 주석"
    required: false
    dependencies: ["core", "morphology"]
    includes:
      - constituency_parsing
      - dependency_parsing
      - grammatical_functions

  semantics:
    name: "WIA-LANG-010-SEM"
    description: "의미론적 주석"
    required: false
    dependencies: ["core", "morphology"]
    includes:
      - named_entities
      - word_senses
      - semantic_roles
      - event_structures

  discourse:
    name: "WIA-LANG-010-DISC"
    description: "담화 수준 주석"
    required: false
    dependencies: ["core", "morphology", "syntax"]
    includes:
      - coreference
      - discourse_relations
      - information_structure

  lexicon:
    name: "WIA-LANG-010-LEX"
    description: "어휘 데이터베이스"
    required: false
    dependencies: ["core"]
    includes:
      - wordnet
      - framenet
      - verbnet
```

### 2. 확장성 (Extensibility)

새로운 주석 유형과 언어 현상을 쉽게 추가할 수 있습니다:

```xml
<!-- 기본 구조 -->
<linguisticData xmlns="http://wia.org/lang-010/v1.0"
                xmlns:custom="http://myorg.org/custom-annotation">

  <!-- 표준 주석 -->
  <annotation type="pos">
    <token id="t1" pos="NOUN"/>
  </annotation>

  <!-- 사용자 정의 확장 -->
  <custom:annotation type="sentiment">
    <token id="t1" sentiment="positive" intensity="0.8"/>
  </custom:annotation>

  <!-- 새로운 언어 현상 주석 -->
  <custom:annotation type="honorifics">
    <token id="t1" honorific-level="formal" subject-honorific="true"/>
  </custom:annotation>

</linguisticData>
```

### 3. 상호운용성 (Interoperability)

다른 표준 및 형식과의 호환성:

| 외부 표준 | 통합 방식 | 용도 |
|---------|---------|-----|
| Unicode | 완전 지원 | 문자 인코딩 |
| ISO 639-3 | 언어 코드 | 언어 식별 |

### TypeScript 인터페이스 정의

```typescript
// 핵심 인터페이스
interface LinguisticResource {
  id: string;
  metadata: Metadata;
  content: Content;
}

interface Metadata {
  // 필수 메타데이터
  language: Language;
  creator: Creator;
  created: Date;
  license: License;
  version: string;

  // 권장 메타데이터
  description?: string;
  domain?: string[];
  genre?: string[];
  size?: Size;

  // 선택 메타데이터
  funding?: string;
  citation?: string;
  doi?: string;
}

interface Language {
  iso639_1?: string;  // "ko"
  iso639_3: string;   // "kor" (필수)
  name: string;       // "Korean"
  script?: string;    // "Hangul"
  family?: string;    // "Koreanic"
}

interface Token {
  id: string;
  form: string;
  start: number;
  end: number;

  // 형태론적 정보 (선택)
  lemma?: string;
  pos?: string;
  morphemes?: Morpheme[];

  // 통사론적 정보 (선택)
  head?: string;  // 지배소 토큰 ID
  deprel?: string;  // 의존 관계

  // 의미론적 정보 (선택)
  entity?: EntityAnnotation;
  sense?: SenseAnnotation;
  role?: RoleAnnotation;
}

interface Morpheme {
  form: string;
  lemma: string;
  tag: string;  // 형태소 태그
  gloss?: string;  // 의미 설명
}

interface EntityAnnotation {
  type: string;  // PER, ORG, LOC, etc.
  confidence?: number;
  uri?: string;  // 지식베이스 URI
}

interface Annotation {
  type: string;
  layer: AnnotationLayer;
  features: Record<string, any>;
  confidence?: number;
  annotator?: string;
  timestamp?: Date;
}

enum AnnotationLayer {
  MORPHOLOGY = "morphology",
  SYNTAX = "syntax",
  SEMANTICS = "semantics",
  DISCOURSE = "discourse",
  PHONOLOGY = "phonology"
}
```
