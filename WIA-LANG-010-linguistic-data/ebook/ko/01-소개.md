# 제1장: 언어 데이터 표준 소개

## 개요

> **弘益人間 (홍익인간)** - 널리 인간을 이롭게 하라

언어는 인류의 가장 근본적인 소통 수단이며, 문화와 지식을 전달하는 핵심 매체입니다. WIA-LANG-010 언어 데이터 표준은 다양한 언어 자원을 체계적으로 구조화하고, 언어학적 데이터를 효율적으로 교환하며, 언어 기술의 발전을 촉진하기 위해 설계되었습니다.

이 표준은 홍익인간의 정신에 따라 모든 언어와 문화가 디지털 시대에 동등하게 표현되고 보존될 수 있도록 돕습니다.

## 언어 데이터의 중요성

### 1. 언어 다양성 보존

전 세계에는 약 7,000개 이상의 언어가 존재하지만, 그 중 많은 언어가 소멸 위기에 처해 있습니다. 언어 데이터 표준화는:

- **언어 문서화**: 소멸 위기 언어의 체계적 기록
- **문화 유산 보존**: 언어를 통한 전통 지식과 문화 보존
- **세대 간 전승**: 디지털 형태로 다음 세대에 언어 전달
- **학술 연구 지원**: 언어학자들의 연구 자료 제공

### 2. 언어 기술 발전

현대 언어 기술은 방대한 언어 데이터에 의존합니다:

| 기술 분야 | 필요 데이터 | 표준화 효과 |
|---------|----------|-----------|
| 기계 번역 | 병렬 코퍼스 | 번역 품질 향상 |
| 음성 인식 | 음성 주석 코퍼스 | 인식 정확도 개선 |
| 자연어 이해 | 의미 주석 데이터 | 문맥 이해 강화 |
| 질의응답 시스템 | 지식 그래프, 온톨로지 | 정확한 답변 생성 |
| 텍스트 생성 | 대규모 텍스트 코퍼스 | 자연스러운 문장 생성 |
| 감성 분석 | 감성 레이블 데이터 | 감정 분류 정확도 향상 |

### 3. 언어학 연구

언어학자들은 체계적인 언어 데이터를 통해:

- **이론 검증**: 언어 이론의 경험적 검증
- **유형론 연구**: 언어 간 비교 연구
- **통시적 연구**: 언어 변화 추적
- **사회언어학**: 언어 변이 연구

## WIA-LANG-010 표준의 목표

### 1. 상호운용성 (Interoperability)

다양한 언어 자원과 도구가 원활하게 소통할 수 있도록:

```xml
<!-- 표준화된 언어 데이터 교환 형식 예시 -->
<linguisticData xmlns="http://wia.org/lang-010/v1.0">
  <metadata>
    <language code="ko" name="Korean" family="Koreanic"/>
    <dataType>syntactic-treebank</dataType>
    <encoding>UTF-8</encoding>
    <version>1.0.0</version>
  </metadata>
  <corpus>
    <sentence id="s001">
      <text>언어 데이터 표준화는 중요합니다.</text>
      <annotation type="dependency">
        <token id="t1" form="언어" lemma="언어" pos="NNG" head="2" deprel="nmod"/>
        <token id="t2" form="데이터" lemma="데이터" pos="NNG" head="3" deprel="nmod"/>
        <token id="t3" form="표준화는" lemma="표준화" pos="NNG" head="4" deprel="nsubj"/>
        <token id="t4" form="중요합니다" lemma="중요하다" pos="VV" head="0" deprel="root"/>
      </annotation>
    </sentence>
  </corpus>
</linguisticData>
```

### 2. 재사용성 (Reusability)

한 번 구축된 언어 자원이 다양한 용도로 재사용:

- **다목적 활용**: 단일 코퍼스를 여러 연구/응용에 활용
- **증분적 확장**: 기존 자원에 새로운 주석 계층 추가
- **교차 언어 적용**: 동일한 주석 체계를 여러 언어에 적용

### 3. 확장성 (Scalability)

소규모부터 대규모까지 다양한 규모의 언어 데이터 지원:

| 규모 | 데이터 크기 | 활용 사례 |
|-----|----------|---------|
| 소규모 | 1만~10만 토큰 | 언어 문서화, 파일럿 연구 |
| 중규모 | 10만~1천만 토큰 | 언어학 연구, 도구 개발 |
| 대규모 | 1천만~10억 토큰 | 기계 학습, 언어 모델 |
| 초대규모 | 10억 토큰 이상 | 대규모 언어 모델 (LLM) |

### 4. 품질 보증 (Quality Assurance)

일관되고 신뢰할 수 있는 언어 데이터:

```python
# 언어 데이터 품질 검증 예시
from wia_lang_010 import DataValidator, QualityMetrics

validator = DataValidator(standard="WIA-LANG-010")
corpus = load_corpus("korean_treebank.xml")

# 품질 검증
results = validator.validate(corpus)
print(f"구조 검증: {results.structural_validity}")
print(f"주석 일관성: {results.annotation_consistency}")
print(f"메타데이터 완전성: {results.metadata_completeness}")

# 품질 메트릭스
metrics = QualityMetrics(corpus)
print(f"Inter-annotator Agreement: {metrics.iaa_score}")
print(f"Coverage: {metrics.lexical_coverage}%")
print(f"Annotation Density: {metrics.annotation_density}")
```

## 언어 데이터의 주요 유형

### 1. 코퍼스 (Corpora)

실제 언어 사용의 대규모 수집:

**종류별 분류:**
- **원시 코퍼스 (Raw Corpus)**: 주석 없는 순수 텍스트
- **주석 코퍼스 (Annotated Corpus)**: 언어학적 정보가 추가된 텍스트
- **병렬 코퍼스 (Parallel Corpus)**: 두 개 이상 언어의 대응 텍스트
- **비교 코퍼스 (Comparable Corpus)**: 유사한 주제/장르의 다언어 텍스트

**주석 계층별 분류:**

```
코퍼스 주석 계층
│
├─ 형태론 계층
│  ├─ 토큰화 (Tokenization)
│  ├─ 형태소 분석 (Morphological Analysis)
│  └─ 품사 태깅 (POS Tagging)
│
├─ 통사론 계층
│  ├─ 구문 구조 (Phrase Structure)
│  ├─ 의존 구조 (Dependency Structure)
│  └─ 기능 표지 (Functional Labels)
│
├─ 의미론 계층
│  ├─ 개체명 인식 (Named Entity Recognition)
│  ├─ 의미역 (Semantic Role)
│  └─ 어휘 의미 (Word Sense)
│
└─ 화용론 계층
   ├─ 담화 관계 (Discourse Relations)
   ├─ 공지시 (Coreference)
   └─ 화행 (Speech Acts)
```

### 2. 어휘 데이터베이스 (Lexical Databases)

체계적으로 구조화된 어휘 정보:

**WordNet 형식:**


### 3. 문법 자원 (Grammar Resources)

언어의 문법 규칙을 형식화:

| 문법 유형 | 형식 | 활용 |
|---------|------|-----|
| 문맥 자유 문법 (CFG) | 생성 규칙 | 구문 분석 |
| 의존 문법 | 의존 관계 | 통사 분석 |
| 범주 문법 | 범주 조합 | 의미 구성 |
| HPSG | 자질 구조 | 정밀 문법 |

### 4. 온톨로지 (Ontologies)

언어 개념의 체계적 분류:

```turtle
gold:CommonNoun rdfs:subClassOf gold:Noun ;
    rdfs:label "보통명사"@ko .

    rdfs:label "고유명사"@ko .
```
