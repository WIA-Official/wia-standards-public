Chapter 6: Phase 4 — Integration

Phase 4 specifies how WIA Smart Breeding composes with existing genomic toolchains and with neighbouring WIA standards.

6.1 VCF Integration

VCF (Variant Call Format) v4.3 is the de-facto wire format for genotype data. WIA Smart Breeding integrates with VCF via two paths:

  • Inline embed: small SNP panels (under 100 MiB encoded) ride inside the genotype envelope as a base64url-encoded VCF blob.
  • External reference: larger panels and whole-genome sequencing data live as VCF files at a stable URL, with the genotype envelope carrying the URL plus a SHA-256 integrity hash.

WIA hosts MAY publish a VCF-to-WIA bridge endpoint that accepts a vendor's existing VCF and emits a WIA genotype envelope, so labs can on-board without changing their pipeline.

6.2 PLINK Integration

PLINK BED/BIM/FAM bundles are the standard format for population-genetics tools (GCTA, GEMMA, ssGBLUP implementations). The WIA host MAY publish a PLINK-export endpoint that materialises a population's federated genotype + pedigree as a PLINK bundle for downstream tooling. The export MUST honour the audience controls of Phase 3 §5.5 — for example, only the evaluation audience can request a full PLINK bundle.

6.3 HapMap and GFF3 Integration

HapMap reference panels are referenced by URL plus integrity hash. GFF3 annotation files (gene model coordinates) are likewise referenced rather than embedded; the WIA host MAY cache them but MUST NOT re-publish them as canonical WIA records.

6.4 WIA Family Integration

  • WIA-OMNI-API — breeder identity and credentials (cooperative membership, certified-organic status, AI/ET licence) live in WIA-OMNI-API.
  • WIA-AIR-SHIELD — transport hardening for cross-organisation data exchange. Hosts MAY refuse handshakes from peers below a configured AIR-SHIELD score.
  • WIA-INTENT — a breeder's "find me top 10 sires for milk yield in my breed and price band" intent lowers to mating-plan endpoint queries scoped by the breeder's owner consent.
  • WIA-ACCESSIBILITY — breeder-facing UI surfaces accommodate accessibility profiles (large-text mode for ageing breeders, voice-input for hands-free pen-side data entry).

6.5 Worked Example — Joint Genetic Evaluation

Three breeding cooperatives (α in Korea, β in Denmark, γ in Australia) want a joint Holstein milk-yield evaluation without merging their member databases. The flow:

  1. Each cooperative publishes a discovery document and an owner-consent envelope (signed by each member breeder) authorising the cooperative-aggregation scope.
  2. An evaluation centre (did:wia:evaluation:interbull-mirror) requests federated handshakes with α, β, γ.
  3. α, β, γ each issue receipts and serve their members' genotype + phenotype under audience evaluation.
  4. The evaluation centre runs ssGBLUP on the federated dataset and emits per-individual GEBV envelopes back to each cooperative.
  5. Each cooperative writes the GEBV envelopes to its own host; member breeders read their own animals' GEBV with audience commercial.

No raw genotypes ever leave the federation; only signed envelopes pass between hosts, and every read is audit-logged by the source.
