Chapter 4: Phase 2 — API Interface

Phase 2 specifies the HTTP surface that a host (typically a breeding cooperative or a national herd-book) exposes for the data families defined in Phase 1.

4.1 Transport

  • HTTPS only, TLS 1.3 (IETF RFC 8446) or later.
  • HTTP/2 RECOMMENDED, HTTP/1.1 SHALL be supported as fallback.
  • Strict-Transport-Security per IETF RFC 6797.
  • Compression: gzip, br.

4.2 Discovery

GET https://<host>/.well-known/wia-smart-breeding

{
  "wia_smart_breeding_version": "1.0.0",
  "operator_id": "did:wia:operator:herdbook-A",
  "endpoints": {
    "individual": "https://breed.example/sb/individual",
    "genotype":   "https://breed.example/sb/genotype",
    "phenotype":  "https://breed.example/sb/phenotype",
    "gebv":       "https://breed.example/sb/gebv",
    "pedigree":   "https://breed.example/sb/pedigree",
    "population": "https://breed.example/sb/population",
    "mating":     "https://breed.example/sb/mating"
  },
  "supported_signatures": ["Ed25519"],
  "supported_species": ["Bos taurus", "Sus scrofa", "Gallus gallus", "Triticum aestivum"],
  "rate_limits": {
    "anonymous":     { "rps": 1,  "burst": 5 },
    "authenticated": { "rps": 30, "burst": 100 }
  }
}

4.3 Individual Endpoints

GET    /sb/individual/{individual_id}
PUT    /sb/individual/{individual_id}    (signed)
DELETE /sb/individual/{individual_id}    (signed; tombstone, audit retention)
GET    /sb/individual?species=Bos+taurus&breed=Holstein&since={cursor}

4.4 Genotype Endpoints

POST /sb/genotype                   (submit; large payloads MAY use chunked transfer)
GET  /sb/genotype/{genotype_id}
GET  /sb/genotype?individual_id={id}

Genotype payloads larger than 100 MiB MUST use the external-reference form per Phase 1 §3.3.

4.5 Phenotype, GEBV, Pedigree, Population, Mating

The remaining endpoint families follow the same shape: POST/GET/PATCH on /sb/{family}/{id}. PATCH on a phenotype is forbidden (corrections append a new envelope referencing the original). PATCH on a GEBV is permitted only when the method changes (e.g. annual reference population refresh).

4.6 Authentication

Writes require HTTP Message Signatures per IETF RFC 9421 covering method, target URI, Date, Content-Digest, and Content-Length. Reads MAY be anonymous subject to the discovery rate limit; PII-bearing reads (owner identifiers) MUST be authenticated.

4.7 Errors

Errors follow IETF RFC 9457 problem details. Reserved error types live under https://wiastandards.com/smart-breeding/errors/.

4.8 Conformance

A Phase 2 conformant host MUST publish discovery, implement individual / genotype / phenotype / GEBV / pedigree routes, honour Idempotency-Key on all writes, and emit problem-detail JSON for 4xx/5xx responses.
