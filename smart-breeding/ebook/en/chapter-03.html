Chapter 3: Phase 1 — Data Format

Phase 1 fixes the JSON shapes that every WIA Smart Breeding implementation must read and emit. The format is engineered to interoperate with the existing genomic toolchain (VCF, PLINK, HapMap, GFF3) while adding the breeding-specific structures that vendor platforms keep in private databases today.

3.1 Object Families

FamilyPurpose
individualOne animal or plant with identifying metadata, pedigree pointers, and trait history
genotypeSNP array, sequencing readout, or imputed genotype payload
phenotypeMeasured trait observation tied to an individual and a recording event
gebvGenomic Estimated Breeding Value for one trait, with reliability and method
pedigreeMulti-generation ancestor graph + computed inbreeding coefficient
populationGroup of individuals with diversity metrics
mating_planRecommended pairings with predicted offspring values

3.3 Genotype Record

Genotype payloads ride either inline (small SNP panels) or by reference to a VCF / PLINK / HapMap object hosted out-of-band. Inline form:

{
  "wia_smart_breeding_version": "1.0.0",
  "type": "genotype",
  "genotype_id": "gt_BULL-2025-001-snp-50k",
  "individual_id": "BULL-2025-001",
  "platform": "Illumina BovineSNP50 v3",
  "marker_count": 53218,
  "format": "vcf-inline",
  "vcf_header_url": "https://refs.example/bovine-snp50-v3.vcf.header",
  "calls": "data:application/x-vcf;base64,##fileformat=VCFv4.3...",
  "captured_at": "2025-04-02T00:00:00Z",
  "signature": { "alg": "Ed25519", "value": "..." }
}

External-reference form replaces calls with vcf_url and an integrity hash; receivers fetch and verify before use.

3.4 Phenotype Record

{
  "wia_smart_breeding_version": "1.0.0",
  "type": "phenotype",
  "phenotype_id": "ph_BULL-2025-001-2025-q4",
  "individual_id": "BULL-2025-001",
  "trait": "milk_yield_305d",
  "value": 12450,
  "unit": "kg",
  "recorded_at": "2025-12-31",
  "recording_protocol": "ICAR-2024",
  "lactation_number": 1,
  "contemporary_group_id": "cg_2024_holstein_q4",
  "signature": { "alg": "Ed25519", "value": "..." }
}

The contemporary_group_id field anchors the observation to a comparable cohort so downstream BLUP / GEBV computation is not contaminated by herd-level confounders.

  "signature": { "alg": "Ed25519", "value": "..." }
}

3.6 Encoding Rules

  • UTF-8 JSON per IETF RFC 8259, snake_case keys.
  • Timestamps RFC 3339 in UTC, Z suffix.
  • Identifiers URI-shaped or ULID where applicable.
  • Numerical fields in IEEE 754 double precision.
  • Genotype payloads MAY use base64url; integrity hash MUST accompany external references.

3.7 Conformance

A Phase 1 conformant implementation MUST round-trip every object family byte-identically through encode/decode, reject objects missing required fields per JSON Schema, treat unknown optional fields as non-fatal, and validate genotype integrity hashes when the payload rides as an external reference.
