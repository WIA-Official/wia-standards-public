CHAPTER 6

🔀 다중 접속 기술

6.1 다중 접속 개요

위성 통신에서는 수십~수천 명의 사용자가 하나의 위성을 공유해야 합니다. 제한된 주파수 대역과 트랜스폰더 용량을 여러 사용자가 효율적으로 나눠 쓰는 방법을 다중 접속(Multiple Access) 기술이라고 합니다.

다중 접속 기술은 크게 세 가지 차원으로 자원을 분할합니다: 주파수(Frequency), 시간(Time), 코드(Code). 각 방식은 고유한 장단점을 가지며, 응용 분야에 따라 선택됩니다.

6.2 FDMA (Frequency Division Multiple Access)

6.2.1 FDMA 원리

FDMA는 주파수 대역을 여러 개의 좁은 채널로 분할하고, 각 사용자에게 고정된 주파수 채널을 할당하는 방식입니다. 아날로그 TV 방송처럼 각 채널이 고유한 주파수를 독점합니다.

주파수
  ↑
  │ ┌───┐ ┌───┐ ┌───┐ ┌───┐
  │ │ 4 │ │ 3 │ │ 2 │ │ 1 │
  │ └───┘ └───┘ └───┘ └───┘
  └────────────────────────→ 시간
     각 사용자가 고정 주파수 점유

6.2.2 FDMA의 장점

  • 단순성: 구현이 간단하고 저렴. 아날로그 시스템에도 적용 가능
  • 연속 접속: 실시간 음성 통화에 적합. 지연 없음
  • 독립성: 각 채널이 독립적. 다른 사용자 영향 없음
  • 간섭 제한: 보호 대역(Guard Band)으로 인접 채널 간섭 방지

6.2.3 FDMA의 단점

  • 주파수 낭비: 사용자가 전송하지 않아도 채널 점유. 효율 낮음
  • 보호 대역: 채널 간 간격으로 10~20% 스펙트럼 손실
  • 상호변조: 위성 증폭기 비선형성으로 채널 간 간섭 발생
  • 유연성 부족: 채널 재할당이 어려움

6.2.4 FDMA 응용

  • 아날로그 위성 전화: Inmarsat 초기 시스템
  • TV 중계: 방송국 간 비디오 피드
  • SCPC (Single Channel Per Carrier): 전용선, 음성 회선

6.3 TDMA (Time Division Multiple Access)

6.3.1 TDMA 원리

TDMA는 전체 주파수 대역을 모든 사용자가 공유하되, 각 사용자는 할당된 시간 슬롯에만 전송합니다. 마치 회의에서 순서대로 발언하는 것처럼, 시간을 나눠 씁니다.

주파수
  ↑
  │ ┌───────────────────┐
  │ │ 1│ 2│ 3│ 4│ 1│ 2│
  │ └───────────────────┘
  └────────────────────→ 시간
     모든 사용자가 전체 대역 공유
     각자 할당된 시간에만 전송

6.3.2 TDMA 프레임 구조

TDMA는 시간을 프레임으로 나누고, 각 프레임을 다시 타임 슬롯으로 분할합니다:

  • 프레임 길이: 보통 2~20 ms
  • 타임 슬롯: 프레임을 수십~수백 개로 분할
  • 프리앰블: 각 슬롯 시작에 동기·주소 정보
  • 페이로드: 실제 데이터
  • 가드 타임: 슬롯 간 충돌 방지 여유 시간

6.3.3 TDMA의 장점

  • 높은 효율: 보호 대역 불필요. 스펙트럼 활용률 90% 이상
  • 유연성: 트래픽에 따라 슬롯 동적 재할당 가능
  • 디지털 친화적: 패킷 데이터에 적합
  • 단일 반송파: 위성 증폭기를 선형 영역에서 운용. 상호변조 감소

6.3.4 TDMA의 단점

  • 복잡한 동기: 모든 단말이 정밀한 시간 동기 필요
  • 버스트 전송: 순간적으로 높은 전력 필요. 단말 증폭기 부담
  • 지연 민감: 프레임 주기로 인한 추가 지연 발생
  • 거리 보정: GEO 위성의 경우 240ms 전파 지연 보정 복잡

6.3.5 TDMA 응용

  • VSAT 네트워크: iDirect, Hughes 등 대부분의 상용 VSAT
  • 이동통신: GSM, Iridium 위성 전화
  • DVB-RCS2: 양방향 위성 인터넷 리턴 채널

6.4 CDMA (Code Division Multiple Access)

6.4.1 CDMA 원리

CDMA는 모든 사용자가 동일한 주파수와 시간을 동시에 사용하되, 각자 고유한 확산 코드로 신호를 구별합니다. 마치 여러 사람이 동시에 다른 언어로 대화하는 것처럼, 코드로 분리합니다.

주파수
  ↑
  │ ┌───────────────────┐
  │ │ 모든 사용자 동시 │
  │ │ 코드로 구별      │
  │ └───────────────────┘
  └────────────────────→ 시간
     직교 또는 의사잡음 코드 사용

6.4.2 확산 스펙트럼 (Spread Spectrum)

CDMA는 원래 신호를 훨씬 넓은 대역폭으로 확산시킵니다. 이를 통해:

  • 간섭 내성: 협대역 간섭이 들어와도 확산으로 희석
  • 비밀성: 코드를 모르면 신호 감지 어려움 (군사 응용)
  • 다중 경로 극복: 반사파를 구별하여 결합 가능

6.4.3 CDMA의 장점

  • 간섭 강인성: 재밍, 협대역 간섭에 강함
  • 보안: 코드 없이는 해독 불가능
  • 소프트 용량: 사용자 수가 증가해도 급격한 성능 저하 없이 점진적 품질 하락
  • 주파수 재사용: 인접 셀에서 같은 주파수 사용 가능

6.4.4 CDMA의 단점

  • Near-Far 문제: 가까운 사용자 신호가 먼 사용자 신호를 압도. 정밀한 전력 제어 필수
  • 복잡한 수신기: 상관 검출, 코드 동기 등 복잡한 신호처리
  • 넓은 대역폭: 확산으로 인해 FDMA 대비 수십 배 대역 사용
  • 전력 소비: 복잡한 연산으로 단말 배터리 소모 증가

6.4.5 CDMA 응용

  • GPS: 모든 위성이 동일 주파수 1.575 GHz 사용, 코드로 구별
  • Globalstar: LEO 위성 전화 시스템
  • IS-95, CDMA2000: 3G 이동통신 (현재는 LTE/5G로 대체)
  • 군사 위성 통신: 보안과 간섭 내성 중요

6.5 하이브리드 및 고급 기술

6.5.1 FDMA/TDMA 결합

주파수를 여러 채널로 나누고(FDMA), 각 채널 내에서 시간 슬롯으로 다시 분할(TDMA)하는 방식. 대부분의 상용 위성 시스템이 채택하고 있습니다.

  • DVB-S2: 다운링크는 TDM, 업링크는 MF-TDMA
  • Intelsat: 트랜스폰더별로 FDMA, 각 반송파 내에서 TDMA

6.5.2 MF-TDMA (Multi-Frequency TDMA)

여러 주파수 반송파를 동시에 사용하고, 각 반송파에서 TDMA를 적용합니다. 트래픽 변화에 따라 반송파 수를 동적으로 조정할 수 있어 유연합니다.

6.5.3 OFDMA (Orthogonal Frequency Division Multiple Access)

LTE/5G에서 사용하는 기술로, 주파수를 수백~수천 개의 직교 부반송파로 분할하고, 각 사용자에게 필요한 만큼의 부반송파를 할당합니다. 위성 통신에도 도입이 시작되었습니다.

  • 장점: 주파수 효율 극대화, 다중 경로 페이딩 강인성, 유연한 대역폭 할당
  • 단점: 높은 PAPR(Peak-to-Average Power Ratio)로 전력 증폭기 효율 저하, 도플러에 민감

6.5.4 NOMA (Non-Orthogonal Multiple Access)

5G/6G에서 연구 중인 차세대 기술로, 동일 자원을 여러 사용자가 비직교적으로 공유하고, 수신측에서 연속 간섭 제거(SIC)로 신호를 분리합니다. 위성 통신에도 적용 연구 중입니다.

6.6 랜덤 액세스 프로토콜

다중 접속 기술 외에도, 사용자가 필요할 때만 전송하는 랜덤 액세스 방식이 있습니다.

6.6.1 ALOHA

사용자가 데이터가 있을 때 즉시 전송. 충돌 발생 시 랜덤 시간 후 재전송. 단순하지만 효율이 낮습니다(최대 18%).

6.6.2 Slotted ALOHA

시간을 슬롯으로 나누고, 슬롯 시작에만 전송 허용. 충돌 확률이 절반으로 감소하여 효율 36%까지 향상.

6.6.3 CRDSA (Contention Resolution Diversity Slotted ALOHA)

동일 패킷을 여러 슬롯에 중복 전송하여, 수신측에서 간섭 제거 기법으로 충돌 해결. 효율 60~80%까지 달성. DVB-RCS2에서 채택.

6.6.4 응용: IoT 위성 통신

수백만 개의 IoT 센서가 간헐적으로 소량 데이터를 전송하는 경우, 랜덤 액세스가 효율적입니다. Swarm, Kinéis 등 IoT 전문 위성이 ALOHA 변형 기법을 사용합니다.

6.7 주파수 재사용 (Frequency Reuse)

동일한 주파수를 여러 지역에서 동시에 사용하여 전체 시스템 용량을 배가시키는 기술입니다.

6.7.1 스팟 빔 (Spot Beam)

위성 안테나가 지구 표면의 좁은 영역만을 조명하는 빔입니다. 인접하지 않은 빔끼리는 같은 주파수를 재사용할 수 있습니다.

  • 빔 직경: 100~500 km
  • 재사용 인수: 4~7 (주파수 효율 4~7배 향상)
  • 응용: HTS는 수백 개의 스팟 빔 운용

6.7.2 편파 재사용 (Polarization Reuse)

동일 주파수를 수평 편파와 수직 편파(또는 좌우 원편파)로 동시 전송. 주파수 효율 2배 향상.

  • 교차 편파 간섭(XPI): 25~30 dB 격리 필요
  • 비 감쇠 시 편파 회전: 폭우 시 XPI 저하로 간섭 증가

6.7.3 4색 재사용 패턴

지상 셀룰러처럼 위성 스팟 빔도 주파수를 4~7개 그룹으로 나누고, 인접 빔끼리 다른 주파수를 사용합니다. Viasat-3는 수백 개 빔으로 주파수를 수십 번 재사용하여 Tbps급 용량을 달성합니다.

6.9 수요 할당 (Demand Assignment)

고정 할당 대신, 트래픽 수요에 따라 동적으로 자원을 할당하는 방식입니다.

6.9.1 DAMA (Demand Assigned Multiple Access)

  • 사용자가 데이터 전송을 원할 때 중앙 제어국에 요청
  • 제어국이 빈 채널/슬롯을 할당
  • 전송 완료 후 자원 반환
  • 장점: 효율 극대화 (80~95%)
  • 단점: 요청-할당 지연, 중앙 제어 필요

6.9.2 고정 vs. 수요 할당

구분고정 할당수요 할당
효율낮음 (30~50%)높음 (80~95%)
지연없음요청-할당 시간
복잡도낮음높음
응용실시간 음성버스티 데이터

6.10 마치며

다중 접속 기술은 제한된 위성 자원을 여러 사용자가 효율적으로 공유하는 핵심 기술입니다. FDMA는 단순하지만 비효율적이고, TDMA는 디지털 시대의 표준으로 높은 효율과 유연성을 제공하며, CDMA는 간섭 내성과 보안이 뛰어나지만 복잡합니다.

현대 위성 시스템은 FDMA/TDMA 하이브리드를 주로 사용하며, 스팟 빔과 편파 재사용으로 주파수 효율을 극대화합니다. 차세대 HTS는 OFDMA와 NOMA 등 최신 기술을 도입하여 Tbps급 용량을 실현하고 있습니다.
