2단계: API 인터페이스 설계

API 아키텍처 개요

WIA-LEG-006의 2단계는 상속 계획의 프로그래밍 방식 관리를 위한 포괄적인 API 인터페이스를 제공합니다. API는 REST와 GraphQL 엔드포인트를 모두 지원하여 개발자에게 유연성을 제공합니다.

API 설계 원칙

  • RESTful 규칙: HTTP 메서드 및 상태 코드의 표준 사용
  • 버전 관리: URL 경로를 통한 명시적 API 버전 관리
  • 보안 우선: 모든 엔드포인트에 대한 HTTPS 및 인증 필수
  • 속도 제한: 남용 방지 및 공정한 사용 보장
  • 페이지네이션: 대규모 데이터 세트의 효율적인 처리
  • 오류 처리: 명확하고 일관된 오류 응답

{

  "assetDetails": {

    "ticker": "ETH",

      "amount": 10,
      "unit": "ETH"

GET/api/v1/plans/{planId}/assets

계획의 모든 자산 목록 조회

쿼리 매개변수:

  • type - 자산 유형별 필터링 (cryptocurrency, nft, defi, etc.)
  • blockchain - 블록체인별 필터링
  • minValue - 최소 가치별 필터링
  • page - 페이지 번호 (기본값: 1)
  • limit - 페이지당 항목 수 (기본값: 20)

GraphQL 인터페이스

스키마 정의

type InheritancePlan {
  id: ID!
  version: String!
  status: PlanStatus!
  owner: Owner!
  assets: [Asset!]!
  beneficiaries: [Beneficiary!]!
  distribution: Distribution!
  triggers: [Trigger!]!
  createdAt: DateTime!
  updatedAt: DateTime!
}

type Asset {
  id: ID!
  type: AssetType!
  blockchain: String
  estimatedValue: Value!
  metadata: AssetMetadata!
}

type Beneficiary {
  id: ID!
  personalInfo: PersonalInfo!
  allocation: Allocation!
  conditions: [Condition!]
  priority: Priority!
}

enum PlanStatus {
  DRAFT
  ACTIVE
  TRIGGERED
  EXECUTED
  ARCHIVED
}

enum AssetType {
  CRYPTOCURRENCY
  NFT
  DEFI_POSITION
  DOMAIN
  GAMING_ASSET
}

  X-API-Key: wia_live_abc123xyz

속도 제한

  • 인증된 요청: 시간당 1000 요청
  • 미인증 요청: 시간당 100 요청
  • 민감한 작업: 분당 10 요청
Response Headers:
X-RateLimit-Limit: 1000
X-RateLimit-Remaining: 987
X-RateLimit-Reset: 1705319400

HTTP 상태 코드

  • 200 OK - 성공적인 요청
  • 201 Created - 리소스 생성 성공
  • 400 Bad Request - 잘못된 요청
  • 401 Unauthorized - 인증 실패
  • 403 Forbidden - 권한 부족
  • 404 Not Found - 리소스 없음
  • 429 Too Many Requests - 속도 제한 초과
  • 500 Internal Server Error - 서버 오류

핵심 요약

  1. 이중 프로토콜: REST와 GraphQL을 모두 지원하여 개발자에게 유연성 제공
  2. 포괄적인 CRUD: 상속 계획, 자산, 수혜자에 대한 전체 생성, 읽기, 업데이트, 삭제 작업
  3. 실시간 업데이트: 웹훅과 GraphQL 구독을 통한 즉각적인 알림
  4. 엔터프라이즈 보안: JWT 인증, API 키, 속도 제한 및 감사 로그
  5. 개발자 친화적: 명확한 문서, 일관된 오류 처리, 직관적인 엔드포인트
