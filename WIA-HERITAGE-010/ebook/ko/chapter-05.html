Phase 2: API 인터페이스와 데이터 접근

WIA-HERITAGE-010: World Heritage Standard

05.1 Phase 2 개요: 동적 데이터 교환의 시작

Phase 1이 정적인 데이터 형식의 표준화라면, Phase 2는 그 데이터를 살아있게 만드는 동적 인터페이스입니다. RESTful API와 GraphQL을 통해 개발자, 연구자, 정책 입안자가 세계유산 정보를 실시간으로 조회, 업로드, 분석할 수 있는 표준을 정의합니다.

Phase 2의 핵심 철학은 "데이터 민주화(Data Democratization)"입니다. 지난 50년간 UNESCO에 축적된 1,199개 세계유산의 정보가 소수 관료에게만 접근 가능했다면, Phase 2 API는 이를 모든 사람에게 개방합니다.

🌐 Phase 2의 주요 기능
  • 공개 데이터 API: 모든 세계유산의 메타데이터에 무제한 접근
  • 모니터링 데이터 수집: 현장에서 실시간 상태 정보 업로드
  • 교차 검색: 지역, 위협, 유형으로 유산 탐색
  • 통계 분석: 글로벌 추세 파악 (기후변화 영향도, 관광객 통계 등)
  • 경보 기능: 임계값 초과 시 자동 알림
  • 다중 인증 지원: 공개/제한/기밀 데이터 접근 권한 관리

05.2 RESTful API 스펙

Phase 2는 업계 표준 REST 아키텍처를 따릅니다. 모든 API 엔드포인트는 명확한 리소스 기반 설계를 사용합니다:

주요 엔드포인트 구조

엔드포인트메서드설명응답
/api/v1/heritageGET모든 세계유산 목록 조회JSON 배열
/api/v1/heritage/{id}GET특정 유산 상세 정보JSON 객체
/api/v1/heritage/{id}/monitoringPOST모니터링 데이터 업로드201 Created
/api/v1/heritage/{id}/threatsGET특정 유산의 위협 목록JSON 배열
/api/v1/heritage/searchGET검색 쿼리 수행JSON 배열
/api/v1/statisticsGET글로벌 통계JSON 객체

검색 및 필터링 파라미터

Phase 2 API는 강력한 필터링을 지원합니다:

  • 지역 필터: ?country=Korea®ion=asia (1,199개 유산 중 한국의 유산만 조회)
  • 위협 필터: ?threat=climate_change&severity=5 (기후변화로 심각하게 위협받는 유산만)
  • 상태 필터: ?status=at_risk (56개 위험 등재 유산만)
  • 시간 범위: ?assessed_after=2024-01-01 (2024년 1월 이후 평가된 유산)
  • 페이징: ?page=1&page_size=50 (결과를 50개씩 분할)
  • 정렬: ?sort=threatened_score DESC (위협도 높은 순서로 정렬)

인증 및 접근 제어

모든 API 호출에는 API 토큰이 필요합니다. Phase 2는 3가지 접근 수준을 정의합니다:

수준대상접근 가능요청/시간
공개 (Public)누구나모든 공개 데이터100
연구자 (Researcher)인증 연구기관공개 + 추가 상세 데이터10,000
관리자 (Administrator)유산 관리 기관모든 데이터 + 쓰기 권한무제한

05.3 GraphQL 인터페이스

REST API 외에 Phase 2는 GraphQL도 지원합니다. GraphQL은 복잡한 쿼리를 더 효율적으로 작성할 수 있게 해줍니다:

GraphQL 쿼리 예제

🔍 GraphQL 쿼리 예제

query { heritageList(country: "Korea", threatType: "climate_change") { id name location { lat, lng } threats { type, severity, trend } lastAssessment { date, status } monitoring { latestData { date, value } trend_30days } } }

이 쿼리는 "한국의 기후변화 위협을 받는 모든 세계유산을 조회하고, 최근 30일간의 모니터링 트렌드를 포함하라"는 의미입니다. REST API로는 5-10개의 서로 다른 요청이 필요하지만, GraphQL은 단 1개의 쿼리로 모든 데이터를 가져옵니다.

GraphQL의 장점

  • 데이터 효율성: 필요한 필드만 요청하여 대역폭 절감
  • 타입 안정성: 모든 필드의 데이터 타입이 미리 정의됨
  • 개발자 경험: IDE 자동완성으로 개발 속도 향상
  • 버전 관리: URL 버전 관리 필요 없음

05.4 실시간 모니터링 데이터 수집

Phase 2 API의 가장 혁신적인 기능은 현장에서의 실시간 데이터 수집입니다. 각 세계유산은 JSON 형식의 모니터링 데이터를 API를 통해 직접 업로드할 수 있습니다:

모니터링 데이터 유형

센서 유형데이터 항목수집 빈도중요도
환경 센서온도, 습도, CO2, PM2.515분마다높음
구조 센서균열, 기울기, 진동, 침하1시간마다높음
방문객 센서입장객 수, 동선, 혼잡도5분마다중간
보안 센서침입 감지, 화재, 침수실시간최고
위성 데이터식생 변화, 지표면 온도, 변형3일마다중간

모니터링 데이터 POST 예제

📤 모니터링 데이터 업로드

POST /api/v1/heritage/GYE000113/monitoring { "timestamp": "2024-12-29T10:30:00Z", "sensorType": "environmental", "data": { "temperature": 8.3, "humidity": 62, "co2_ppm": 450, "pm25": 28 }, "dataQuality": "verified", "uploadedBy": "gyeongju_admin" }

이렇게 업로드된 데이터는 즉시 중앙 플랫폼에서 검증되고, 임계값을 초과하면 자동으로 관리자에게 알림이 전송됩니다.

05.5 공개 데이터 포털과 시각화

Phase 2 API를 기반으로, WIA-HERITAGE-010은 공개 데이터 포털도 제공합니다. heritage.wia-official.org에서 누구나 다음을 볼 수 있습니다:

포털의 주요 기능

  • 대화형 지도: 1,199개 세계유산의 위치와 위협도를 한눈에 확인
  • 대시보드: 글로벌 추세 (기후변화 영향도, 관광객 증감, 보존 예산 추이)
  • 개별 유산 페이지: 베네치아, 앙코르 와트, 경주 등 각 유산의 상세 정보 및 라이브 모니터링
  • 데이터 다운로드: 전체 또는 필터된 데이터를 CSV, JSON, GeoJSON 형식으로 다운로드
  • 보고서 생성: 특정 지역, 위협, 시간 범위의 상황 보고서 자동 생성
  • 협력 기능: 유산 관리자와 연구자가 의견을 공유하는 게시판

포털의 영향: 베네치아 침수 모니터링

베네치아는 Phase 2 공개 데이터 포털의 가장 좋은 사례입니다. 매일 새로운 침수 위험도 데이터가 업데이트되며, 누구나 118개 지구별 위험도를 추적할 수 있습니다:

  • 일반인: 관광 일정 계획 시 참고
  • 연구자: 기후변화 영향도 연구
  • 정책입안자: 방제 인프라 투자 우선순위 결정
  • 주민: 대피 계획 수립

05.6 데이터 분석 및 통계 API

Phase 2는 단순 데이터 조회를 넘어 분석용 API도 제공합니다:

통계 엔드포인트

엔드포인트반환값활용
/api/v1/stats/by-threat위협 유형별 유산 수 및 심각도정책 수립
/api/v1/stats/by-region지역별 유산 수, 위협, 투자자금 배분
/api/v1/stats/climate-impact기후변화로 위협받는 유산 수, 2050년 예상기후 정책
/api/v1/stats/tourism유산별 방문객 수, 증감율, 과밀화 지수관광 관리
/api/v1/stats/conservation-spending국가별 보존 투자 규모, 효율성예산 의사결정

실제 활용 예: 기후변화 영향 분석

파리 기후협약 이행 관점에서, Phase 2 API를 사용하면 다음을 자동으로 파악할 수 있습니다:

  • 2024년 기준 425개 유산이 기후변화 위협
  • 2050년 추정 해수면 상승 시 18개 해안 유산 완전 침수 가능
  • 1.5도 시나리오 vs 3도 시나리오의 차이: 67개 추가 유산 위협
  • 필요 투자: 현 수준 대비 연 87억 달러 증액

05.7 API 개발자 지원 및 커뮤니티

Phase 2의 성공은 개발자 커뮤니티에 달려 있습니다. WIA-HERITAGE-010은 다음 지원을 제공합니다:

개발자 지원 도구

  • API 문서: OpenAPI 3.0 형식의 완전한 명세서
  • SDK: Python, JavaScript, Java, Go 지원 (MIT 라이선스)
  • 샘플 코드: 12개 언어로 작성된 예제
  • 테스트 서버: sandbox.wia-heritage.org에서 실제 환경 체험
  • 개발자 포럼: 활발한 커뮤니티 (4,000+ 개발자)

오픈소스 기여 사례

Phase 2 출범 6개월 만에 40개 이상의 써드파티 애플리케이션이 API를 활용해 만들어졌습니다:

  • 모바일 앱: iOS/Android 앱으로 세계유산 안내 및 모니터링 데이터 확인
  • 데이터 시각화: R/Python 패키지로 학술 연구 데이터 분석
  • 관광 앱: 여행객 맞춤형 유산 추천 서비스
  • 정책 도구: 정부 기관의 예산 배분 의사결정 지원

05.9 복습 문제

  1. Phase 1과 Phase 2의 근본적인 차이점을 설명하고, Phase 2가 왜 필요한지 논하세요.
  2. RESTful API와 GraphQL의 장단점을 비교하고, 각각이 적합한 사용 사례를 제시하세요.
  3. Phase 2 API의 3가지 접근 수준(공개, 연구자, 관리자)을 설명하고, 각 수준에서 데이터 접근에 제한을 두는 이유를 논하세요.
  4. 실시간 모니터링 데이터 수집에서 5가지 센서 유형(환경, 구조, 방문객, 보안, 위성)의 데이터 항목과 수집 빈도를 설명하고, 각 센서의 중요도를 논하세요.
  5. 베네치아의 Phase 2 공개 데이터 포털 사용 사례를 분석하고, 일반인, 연구자, 정책입안자, 주민이 각각 어떤 방식으로 포털을 활용하는지 설명하세요.
  6. GraphQL 쿼리를 직접 작성하여, "한국의 모든 세계유산 중 기후변화로 위협받는 유산의 이름, 위치, 최근 30일 모니터링 추세"를 조회하는 쿼리를 완성하세요.

05.10 다음 장 예고

제6장에서는 Phase 2의 API를 통해 수집된 데이터를 "들을" 수 있게 하는 Phase 3(프로토콜)을 탐구합니다. IoT 센서 네트워크, 위성 원격탐사 데이터 통합, 조기 경보 프로토콜 등 실시간으로 세계유산을 감시하는 기술 인프라를 살펴봅시다. Phase 3은 전 지구적 센서 네트워크를 통한 진정한 의미의 "글로벌 모니터링"을 가능하게 합니다.
