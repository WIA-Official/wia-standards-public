1장: 센서 융합 기초

1.1 센서 융합 소개

센서 융합은 서로 다른 출처에서 얻은 감각 데이터를 결합하여 개별 소스를 사용할 때보다 불확실성이 적은 정보를 만들어내는 과정입니다. 이 기본 개념은 물리적 세계에 대한 신뢰할 수 있는 인식이 필요한 현대 자율 시스템, 로봇공학 및 인공지능 응용 프로그램의 핵심입니다.

자연에서 생물학적 시스템은 정교한 다중 감각 통합 기능을 진화시켜 왔습니다. 인간은 시각, 청각, 촉각 및 전정 정보를 원활하게 결합하여 환경을 탐색하고 상호작용합니다. 센서 융합 분야는 이러한 생물학적 기능을 인공 시스템에서 복제하고 심지어 능가하는 것을 목표로 합니다.

센싱 기술의 확산과 함께 센서 융합의 중요성은 기하급수적으로 증가했습니다. 현대 자율 주행차는 카메라, LiDAR, 레이더, 초음파 센서, GPS 및 관성 측정 장치(IMU)를 포함한 수십 개의 센서를 사용할 수 있습니다. 각 센서 모달리티는 고유한 강점과 약점을 가지고 있으며, 지능적인 융합을 통해서만 안전하고 신뢰할 수 있는 작동에 필요한 강력한 인식을 달성할 수 있습니다.

1.2 센서 융합이 중요한 이유

단일 센서 기술은 환경에 대한 완벽한 정보를 제공하지 않습니다. 각 감지 모달리티에는 다중 센서 통합을 통해서만 극복할 수 있는 고유한 한계가 있습니다:

센서 유형 강점 한계 융합 이점
카메라 풍부한 텍스처, 색상, 높은 해상도 저조도에서 성능 저하, 직접적인 깊이 없음 LiDAR와 결합하여 3D 이해
LiDAR 정확한 3D 기하학, 거리 비용이 높음, 희소 데이터, 날씨 민감 카메라가 의미론적 정보 추가
레이더 모든 날씨에서 작동, 속도 데이터 낮은 해상도, 제한된 객체 분류 비전이 객체 식별 제공
IMU 높은 속도, 자체 포함 시간이 지남에 따라 드리프트, 절대 참조 없음 GPS가 절대 위치 제공
GPS 전역 절대 위치 낮은 속도, 다중 경로 오류, 실내 불가 IMU가 GPS 중단 연결
초음파 간단함, 저비용, 근접성에 적합 짧은 범위, 온도 민감 카메라 사각 지대 보완

1.3 융합 아키텍처 개요

센서 융합 시스템은 데이터 통합이 발생하는 수준에 따라 분류할 수 있습니다. 이러한 아키텍처를 이해하는 것은 효과적인 다중 센서 시스템을 설계하는 데 필수적입니다:

저수준 (데이터 수준) 융합

원시 센서 데이터가 중요한 처리 전에 직접 결합됩니다. 이 접근 방식은 최대 정보를 보존하지만 호환 가능한 데이터 표현과 정밀한 시공간 정렬을 가진 센서가 필요합니다.

# 저수준 융합 예제: 포인트 클라우드 병합
class LowLevelFusion:
    def fuse_point_clouds(self, lidar_points, stereo_points):
        """
        LiDAR와 스테레오 카메라의 포인트 클라우드를
        원시 데이터 수준에서 병합
        """
        # 공통 좌표 프레임으로 변환
        lidar_transformed = self.transform_to_world(lidar_points, self.lidar_pose)
        stereo_transformed = self.transform_to_world(stereo_points, self.camera_pose)

        # 복제본 제거를 위한 복셀 그리드 필터링으로 병합
        merged = np.vstack([lidar_transformed, stereo_transformed])

        # 복셀 그리드 다운샘플링 적용
        voxel_size = 0.05  # 5cm 해상도
        fused_cloud = self.voxel_filter(merged, voxel_size)

        return fused_cloud

중간 수준 (특징 수준) 융합

각 센서에서 특징이 독립적으로 추출된 후 결합됩니다. 이 접근 방식은 정보 보존과 계산 효율성 사이의 균형을 제공합니다.

# 중간 수준 융합 예제: 특징 연관
class FeatureLevelFusion:
    def fuse_detections(self, camera_detections, radar_tracks):
        """
        헝가리안 알고리즘을 사용하여 카메라 객체 감지와
        레이더 트랙 연관
        """
        # 특징 추출
        cam_features = self.extract_camera_features(camera_detections)
        radar_features = self.extract_radar_features(radar_tracks)

        # 공간 및 외관 유사성을 기반으로 비용 행렬 구축
        cost_matrix = np.zeros((len(cam_features), len(radar_features)))
        for i, cf in enumerate(cam_features):
            for j, rf in enumerate(radar_features):
                spatial_cost = self.spatial_distance(cf.position, rf.position)
                velocity_cost = self.velocity_consistency(cf, rf)
                cost_matrix[i, j] = spatial_cost + 0.5 * velocity_cost

        # 할당 문제 해결
        row_ind, col_ind = linear_sum_assignment(cost_matrix)

        # 융합된 객체 생성
        fused_objects = []
        for i, j in zip(row_ind, col_ind):
            if cost_matrix[i, j] < self.association_threshold:
                fused = self.merge_features(cam_features[i], radar_features[j])
                fused_objects.append(fused)

        return fused_objects

고수준 (결정 수준) 융합

각 센서가 독립적인 결정 또는 분류를 생성한 후 투표, 베이지안 방법 또는 학습된 융합 규칙을 사용하여 결합됩니다.

# 고수준 융합 예제: 다중 센서 분류
class DecisionLevelFusion:
    def fuse_classifications(self, sensor_decisions, sensor_confidences):
        """
        과거 신뢰성을 기반으로 한 가중 투표를 사용하여
        여러 센서의 분류 결정 결합
        """
        class_votes = defaultdict(float)

        for sensor_id, decision in sensor_decisions.items():
            # 센서 신뢰도 및 과거 신뢰성으로 가중치 부여
            reliability = self.sensor_reliability[sensor_id]
            confidence = sensor_confidences[sensor_id]
            weight = reliability * confidence

            class_votes[decision] += weight

        # 사후 확률을 얻기 위해 정규화
        total = sum(class_votes.values())
        posteriors = {k: v/total for k, v in class_votes.items()}

        # 가장 높은 사후 확률을 가진 클래스 반환
        final_class = max(posteriors, key=posteriors.get)
        final_confidence = posteriors[final_class]

        return final_class, final_confidence

1.4 수학적 기초

센서 융합은 확률 이론, 추정 이론 및 선형 대수학의 기본 개념에 의존합니다. 이러한 기초를 이해하는 것은 효과적인 융합 알고리즘을 설계하고 구현하는 데 필수적입니다.

센서 데이터의 확률적 표현

센서 측정은 본질적으로 불확실합니다. 우리는 이 불확실성을 확률 분포를 사용하여 모델링합니다:

z = h(x) + v, 여기서 v ~ N(0, R)

여기서 z는 측정값, x는 실제 상태, h()는 측정 함수, v는 공분산 R을 가진 측정 노이즈입니다. 이 확률적 프레임워크는 여러 불확실한 측정의 원칙적인 융합을 가능하게 합니다.

센서 융합을 위한 베이즈 정리

베이지안 추론은 최적의 센서 융합을 위한 이론적 기초를 제공합니다:

P(x|z1, z2) = P(z1, z2|x) * P(x) / P(z1, z2)

상태가 주어졌을 때 센서 측정이 조건부 독립인 경우:

P(x|z1, z2) ∝ P(z1|x) * P(z2|x) * P(x) 
# 베이지안 센서 융합 구현
class BayesianFusion:
    def fuse_measurements(self, prior_mean, prior_cov, measurements, noise_covs):
        """
        여러 가우시안 측정의 최적 베이지안 융합

        매개변수:
        - prior_mean: 사전 상태 추정 (n,)
        - prior_cov: 사전 공분산 (n, n)
        - measurements: 측정 벡터 리스트
        - noise_covs: 측정 노이즈 공분산 리스트

        반환:
        - posterior_mean: 융합된 상태 추정
        - posterior_cov: 융합된 공분산
        """
        # 사전 분포로 시작
        info_matrix = np.linalg.inv(prior_cov)  # 정보 행렬
        info_vector = info_matrix @ prior_mean   # 정보 벡터

        # 각 측정에서 정보 누적
        for z, R in zip(measurements, noise_covs):
            H = np.eye(len(z))  # 직접 측정 가정
            R_inv = np.linalg.inv(R)

            info_matrix += H.T @ R_inv @ H
            info_vector += H.T @ R_inv @ z

        # 평균/공분산 형태로 변환
        posterior_cov = np.linalg.inv(info_matrix)
        posterior_mean = posterior_cov @ info_vector

        return posterior_mean, posterior_cov

1.7 캘리브레이션 기초

정확한 센서 융합은 센서 간의 공간적 및 시간적 관계에 대한 정밀한 지식이 필요합니다. 캘리브레이션은 내부 매개변수(내부 센서 특성)와 외부 매개변수(공통 참조 프레임에 대한 센서 자세)를 모두 결정합니다.

내부 캘리브레이션

내부 센서 매개변수를 결정합니다. 카메라의 경우 초점 거리, 주점 및 렌즈 왜곡이 포함됩니다:

# 카메라 내부 캘리브레이션
class CameraCalibration:
    def calibrate_intrinsics(self, checkerboard_images, pattern_size, square_size):
        """
        체커보드 패턴을 사용하여 카메라 내부 매개변수 캘리브레이션
        """
        # 객체 점 준비 (체커보드 프레임의 3D 점)
        objp = np.zeros((pattern_size[0] * pattern_size[1], 3), np.float32)
        objp[:, :2] = np.mgrid[0:pattern_size[0], 0:pattern_size[1]].T.reshape(-1, 2)
        objp *= square_size  # 실제 크기로 스케일

        obj_points = []  # 월드 좌표의 3D 점
        img_points = []  # 이미지 평면의 2D 점

        for img in checkerboard_images:
            gray = cv2.cvtColor(img, cv2.COLOR_BGR2GRAY)
            ret, corners = cv2.findChessboardCorners(gray, pattern_size, None)

            if ret:
                # 코너 위치 정제
                corners_refined = cv2.cornerSubPix(
                    gray, corners, (11, 11), (-1, -1),
                    (cv2.TERM_CRITERIA_EPS + cv2.TERM_CRITERIA_MAX_ITER, 30, 0.001)
                )
                obj_points.append(objp)
                img_points.append(corners_refined)

        # 캘리브레이션
        ret, K, dist, rvecs, tvecs = cv2.calibrateCamera(
            obj_points, img_points, gray.shape[::-1], None, None
        )

        return {
            'camera_matrix': K,           # 3x3 내부 행렬
            'dist_coeffs': dist,          # 왜곡 계수
            'reprojection_error': ret     # RMS 재투영 오차
        }

핵심 내용

  • 센서 융합은 여러 소스의 데이터를 결합하여 불확실성을 줄이고 인식 신뢰성을 향상시킵니다
  • 단일 센서는 완전한 정보를 제공하지 않으며, 융합은 다양한 모달리티의 상호 보완적 강점을 활용합니다
  • 세 가지 주요 융합 아키텍처: 저수준(데이터), 중간 수준(특징), 고수준(결정) 융합이 있습니다

  • 시간 정렬은 서로 다른 센서의 데이터가 동일한 시점에 해당하도록 보장합니다
  • WIA-AI-SENSOR-FUSION 표준은 모달리티 불가지론적이고 불확실성을 인식하는 프레임워크를 제공합니다
