Chapter 07: Phase 4 — Integration with Farm Management Systems

7.1 Phase 4 in one paragraph

Phase 4 specifies how WIA Smart Farm integrates with existing precision agriculture standards: ISO 19156 (Observations & Measurements), AgGateway ADAPT, ISOBUS (ISO 11783) tractor electronics, FAO AgroVoc crop vocabulary, USDA Soil Taxonomy, and the WIA family.

7.2 ISO 19156 O&M alignment

ISO 19156 (Observations and Measurements) provides the conceptual foundation for sensor reading shapes. The standard\x27s sensor_reading envelope is structurally compatible with O&M; downstream OGC-based GIS systems can ingest WIA sensor data via the bridge.

7.3 AgGateway ADAPT bridge

AgGateway ADAPT is the precision-agriculture data exchange specification used widely by farm management software. The bridge maps WIA envelopes to ADAPT objects so existing FMS platforms can ingest WIA data.

7.4 ISOBUS (ISO 11783) tractor bridge

ISOBUS is the tractor-implement electronics standard. The bridge translates ISOBUS Task Controller messages to WIA fertilisation_event and harvest_record envelopes. Variable-rate prescription maps emitted as ISOBUS Task XML can be derived from WIA field + sensor data.

7.5 FAO AgroVoc crop concept vocabulary

FAO AgroVoc is the multilingual concept vocabulary for agriculture. The standard\x27s current_crop and harvest_record.crop fields use AgroVoc concept URIs so cross-language consumers can resolve crop names without ambiguity.

7.6 USDA Soil Taxonomy

USDA Soil Taxonomy provides the soil classification vocabulary. The standard\x27s soil_class field uses USDA tokens (Mollisol, Alfisol, Inceptisol, etc.); jurisdictions outside the US can use WRB (World Reference Base) tokens with a wrb_class field.

7.7 WIA family integration

WIA-OMNI-API holds farmer credentials. WIA Water Scarcity Response composes — irrigation events flow as supply augmentation data into basin-level water scarcity tracking. WIA Smart Breeding composes — variety performance from harvest records feeds into breeding program evaluation. WIA Time-Series Data composes — high-cadence sensor streams ride as time-series envelopes when bulk historical analysis is needed.

Farmer-First Design

Every choice in this standard is filtered through: does this make farm data more useful to the farmer who collected it, or just more accessible to the vendor who sold the equipment? When the two conflict, farmer sovereignty wins. That is why the standard mandates explicit farmer consent for cross-vendor data sharing, refuses to allow vendor terms-of-service to bypass the consent envelope, and requires implementations to surface vendor data flows so farmers can verify what their data is doing.

Implementer Notes

Three operational concerns recur in smart-farm infrastructure and are baked into the standard. First, sensor data volume during peak season: a 100-hectare farm with 20 soil sensors per hectare emitting at 5-minute cadence produces ~500 000 sensor readings per day. Hosts MUST provision for this peak load and SHOULD downsample to hourly aggregates after 30 days. Second, farmer consent revocation: when a farmer revokes consent, the data must stop flowing within minutes; the consent envelope MUST carry an explicit revocation timestamp and downstream consumers MUST honour it. Third, ISOBUS Task XML round-trip: variable-rate prescription maps emitted as ISOBUS Task XML must round-trip through WIA envelopes without losing the per-cell prescription rates; the harvest_record envelope embeds GeoJSON yield maps that are convertible back to ISOBUS Task XML for the next planting cycle.
