WIA 스마트 팜이 다루는 핵심

WIA 스마트 팜은 정밀 농업 데이터 교환을 위한 개방형 표준입니다. 4-페이즈 아키텍처: Phase 1(데이터 형식)은 필드/구획·센서 판독·관개 사건·시비 사건·수확 기록·작물 정찰 관측 봉투를 정의합니다. Phase 2(API 인터페이스)는 발행·조회·스트림·페더레이션을 위한 HTTP 표면. Phase 3(페더레이션 프로토콜)는 농민 통제 청중 통제와 함께 벤더 간 데이터 흐름 처리. Phase 4(통합)는 ISO 19156(O&M), AgGateway ADAPT, ISOBUS(ISO 11783), FAO AgroVoc, USDA Soil Taxonomy, WIA 패밀리와 연결.

한국 환경에서의 적용 노트

한국 농촌진흥청과 농림축산식품부는 ICT 융합 스마트팜 사업을 추진하고 있으며 다수의 비닐하우스·노지 농장에 IoT 센서가 설치되어 있습니다. 한국 농가는 NH농협, 신경식가축병원, 농약사·종자사 등 다양한 외부 행위자와 데이터를 교환하지만 농가 데이터 주권에 대한 표준화된 동의 메커니즘이 없습니다. WIA 스마트 팜의 farmer_consent 봉투(Phase 3 §3)는 한국 농가가 외부 데이터 공유에 대한 명시적 통제권을 가질 수 있도록 표준화된 형식을 제공합니다.

핵심 객체 가족

  • field — 형상·면적·현재 작물·토양 분류
  • sensor_reading — 토양/기상/작물 센서 측정과 교정 메타데이터
  • irrigation_event — 부피·방법·출처·기간
  • fertilisation_event — N/P/K 시비량·적용 방법·장비 ID
  • harvest_record — 수확량·수분·품질 지표·장비 ID
  • crop_scouting_observation — 위치 정보가 있는 해충/질병/잡초 존재
  • farmer_consent — 벤더 간 데이터 공유에 대한 서명된 권한 부여

농민 주권의 의미

표준은 농가 데이터에 대한 농민 소유권을 Phase 3 핵심 원리로 다룹니다. 실용적 결과: 장비 벤더는 농민의 명시적 동의 봉투 서명 없이 종자 회사·금융 기관·보험 제공자와 농가 데이터를 공유할 수 없습니다. 기본값은 동의 없음입니다. 이는 벤더 약관이 기본적으로 광범위한 데이터 사용 권한을 부여하는 지배적 산업 패턴과 대조됩니다.

WIA 패밀리 합성

WIA-OMNI-API(농민 자격증), WIA Water Scarcity Response(관개 사건 → 유역 단위 공급 증대 데이터), WIA Smart Breeding(수확 기록 → 품종 평가), WIA Time-Series Data(고주기 센서 스트림)와 자연스럽게 합성됩니다.

구현자 노트

스마트 팜 인프라에서 반복되는 세 가지 운영 관심사가 표준에 박혀 있습니다. 첫째, 성수기 센서 데이터 볼륨: 100 헥타르 농장에 헥타르당 20개 토양 센서가 5분 주기로 발행하면 일일 ~50만 센서 판독을 생산합니다. 호스트는 이 피크 부하를 위해 프로비저닝해야 하며 30일 후 시간별 집계로 다운샘플링해야 합니다. 둘째, 농민 동의 취소: 농민이 동의를 취소할 때 데이터는 분 단위로 흐름이 멈춰야 합니다. 동의 봉투는 명시적 취소 타임스탬프를 포함해야 하며 다운스트림 컨슈머는 이를 존중해야 합니다. 셋째, ISOBUS Task XML 왕복: ISOBUS Task XML로 발행된 가변 시비 처방 맵은 셀별 처방률을 잃지 않고 WIA 봉투를 통해 왕복해야 합니다. harvest_record 봉투는 다음 파종 주기를 위해 ISOBUS Task XML로 변환 가능한 GeoJSON 수확 맵을 포함합니다.
