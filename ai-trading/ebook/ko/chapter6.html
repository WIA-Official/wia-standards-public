실행 및 인프라

거래 실행이 중요합니다

거래를 효율적으로 실행할 수 없다면 최고의 예측도 가치가 없습니다. 부실한 실행은 슬리피지, 지연, 시장 영향을 통해 모든 이익을 잠식할 수 있습니다.

실행 알고리즘

VWAP (거래량 가중 평균 가격)

  • 대규모 주문을 작은 청크로 분할
  • 역사적 일중 거래량 패턴과 일치
  • 시장 영향 최소화
  • 벤치마크: VWAP를 이겼는가?

TWAP (시간 가중 평균 가격)

  • VWAP보다 단순, 동일한 슬라이스
  • 거래량 패턴을 모를 때 사용
  • 단일 대규모 주문보다 낮은 시장 영향

Implementation Shortfall

  • 결정 가격과 실행 가격의 차이 최소화
  • 알파를 포착하기 위해 초기에 공격적으로 거래
  • 시장 영향 대 기회 비용 균형

POV (거래량 비율)

  • 예: 모든 시장 거래량의 10% 거래
  • 시장 유동성에 적응
  • 완료 시간은 시장 활동에 따라 다름

스마트 주문 라우팅 (SOR)

최적의 실행 거래소로 주문을 자동으로 라우팅

실행 거래소

  • Lit 거래소: NYSE, NASDAQ 등 (가시적 주문)
  • 다크 풀: 숨겨진 유동성, 시장 영향 없음
  • ECNs: 전자 통신 네트워크
  • 대체 거래 시스템(ATS)

SOR 논리

  • 모든 거래소에서 가격 비교
  • 수수료 및 리베이트 고려
  • 체결 확률과 속도 고려
  • 최상의 순가격으로 라우팅

저지연 인프라

지연 원인

  • 네트워크: 물리적 거리, 라우팅 홉
  • 처리: 전략 계산, 주문 생성
  • I/O: 디스크 액세스, 데이터베이스 쿼리
  • 직렬화: 데이터 형식 변환

최적화 기법

  • 코로케이션: 거래소와 같은 데이터센터에 서버 배치
  • 직접 시장 접근(DMA): 브로커 라우팅 건너뛰기
  • FIX 프로토콜: 업계 표준 메시징
  • FPGA 가속: 하드웨어 레벨 처리
  • 커널 우회: OS 네트워크 스택 오버헤드 회피

HFT를 위한 언어 선택

  • C++: 최대 성능, 완전한 제어
  • Rust: C++ 성능과 메모리 안전성
  • Java: 더 간단한 개발로 좋은 성능
  • Python: 연구 및 백테스팅만(HFT에는 너무 느림)

프로덕션 배포

배포 체크리스트

  • ✓ 아웃-오브-샘플 데이터에서 전략 검증
  • ✓ 리스크 한도 구성 및 테스트
  • ✓ 페이퍼 트레이딩 결과 수용 가능
  • ✓ 실행 인프라 테스트
  • ✓ 모니터링 및 경고 구성
  • ✓ 재해 복구 계획 수립

모니터링

  • 실시간 P&L 추적
  • 포지션 모니터링
  • 주문 실행 품질(TCA)
  • 시스템 상태(지연, 오류, 가동 시간)
  • 모델 성능 저하 감지
