# 제2장: 다중빔 소나 시스템

## 2.1 다중빔 반향 측심의 원리

다중빔 반향 측심기(MBES)는 현대 해저 매핑의 주력 기술을 대표합니다. 이러한 정교한 음향 시스템은 선박의 이동 방향에 수직인 부채꼴 패턴으로 여러 음파 빔을 방출하여 넓은 해저 스와스에 걸쳐 동시에 깊이 측정을 가능하게 합니다. 선박 바로 아래만 측정하는 단일 빔 반향 측심기와 달리 다중빔 시스템은 수심의 3배에서 7배에 이르는 스와스를 매핑할 수 있어 조사 효율성을 극적으로 향상시킵니다.

다중빔 소나의 기본 원리는 간단합니다. 음향 펄스가 소나 변환기에서 해저까지 왕복하는 데 필요한 시간을 측정한 다음 해수에서 알려진 음속을 사용하여 거리를 계산합니다. 그러나 다중빔 시스템에서 이 원리를 구현하려면 현대 응용 프로그램에 필요한 정확도와 해상도를 달성하기 위해 정교한 엔지니어링과 신호 처리가 필요합니다.

일반적인 다중빔 시스템은 송신과 수신을 위한 두 개의 수직 변환기 배열로 구성됩니다. 송신 배열은 선박 트랙을 따라 좁은 빔과 트랙에 수직인 넓은 빔을 생성합니다. 수신 배열은 트랙에 수직인 좁은 빔을 생성하여 각 반환 신호에 대한 정밀한 각도 결정을 가능하게 합니다. 정밀한 이동 시간 측정과 정확한 각도 정보를 결합함으로써 시스템은 스와스 내의 각 해저 지점의 3차원 위치를 계산합니다.

현대 MBES 시스템은 핑당 수백 개의 개별 깊이 측정을 생성할 수 있으며, 얕은 물에서 핑 속도가 50Hz를 초과합니다. 이 높은 데이터 속도는 느린 조사 속도에서도 밀집된 해저 범위를 가능하게 합니다. 현대 다중빔 시스템으로 하루 조사하면 수백만 개의 개별 깊이 측정을 생성하여 해저 지형의 상세한 그림을 만들 수 있습니다.

## 2.2 시스템 아키텍처 및 구성 요소

완전한 다중빔 소나 시스템은 각각 데이터 수집에서 중요한 역할을 하는 여러 통합 구성 요소로 구성됩니다:

**소나 변환기**: 변환기 배열은 전기 신호를 송신용 음파로 변환하고 수신된 음파를 다시 전기 신호로 변환합니다. 변환기는 일반적으로 선체 장착 돔 또는 수축 가능한 킬에 장착되어 유동 소음을 최소화하고 물과의 최적 음향 결합을 보장합니다. 현대 변환기는 시스템의 작동 주파수 범위에서 에너지를 효율적으로 변환할 수 있는 압전 세라믹 또는 복합 재료를 사용합니다.

**신호 처리 장치**: 소나 프로세서는 송신 타이밍을 제어하고, 빔형성 알고리즘을 적용하여 개별 빔을 분리하고, 바닥 에코를 감지하고, 예비 깊이 추정을 계산합니다. 현대 프로세서는 FPGA(Field-Programmable Gate Array)와 DSP(Digital Signal Processor)를 사용하여 높은 핑 속도에서 수백 개의 빔에 대한 실시간 빔형성 및 바닥 감지의 엄청난 계산 요구를 처리합니다.

**운동 참조 장치(MRU)**: 조사 선박이 파도에 반응하여 지속적으로 움직이기 때문에 선박 운동의 정확한 측정은 깊이 측정을 수정하는 데 필수적입니다. MRU는 관성 센서와 GPS/GNSS 수신기를 사용하여 롤, 피치, 히브 및 헤딩을 높은 정밀도로 측정합니다. 현대 MRU는 센티미터 수준의 정확도로 히브를 측정할 수 있으며, 이는 거친 바다에서 수심 정확도를 유지하는 데 중요합니다.

**음속 프로파일러(SVP)**: 해수에서의 음속은 온도, 염분 및 압력에 따라 변합니다. 수주 전체의 음속 프로파일에 대한 정확한 지식은 올바른 깊이 계산과 빔 스티어링에 중요합니다. SVP는 일반적으로 선박에서 내려진 프로브 또는 AUV에 부착된 프로브를 사용하여 다양한 깊이에서 음속을 측정합니다.

**위치 시스템**: 수심 데이터를 지리 참조하려면 선박 위치에 대한 정밀한 지식이 필요합니다. 현대 조사는 RTK(Real-Time Kinematic) 또는 PPP(Precise Point Positioning) 기술과 함께 GNSS(Global Navigation Satellite Systems)를 사용하여 데시미터 수준 이상의 위치 정확도를 달성합니다.

**수집 소프트웨어**: 수집 시스템은 모든 센서의 데이터를 통합하고, 실시간 보정을 적용하고, 운영자 검토를 위해 데이터를 표시하고, 후처리를 위해 데이터를 기록합니다. 현대 수집 시스템은 운영자가 조사 중 데이터 품질을 평가하고 필요에 따라 조정할 수 있도록 하는 정교한 시각화 도구를 제공합니다.

## 2.3 주파수 선택 및 성능 트레이드오프

다중빔 시스템은 일반적으로 12kHz에서 400kHz 이상까지 광범위한 음향 주파수에서 작동합니다. 주파수 선택에는 범위, 해상도 및 빔 폭 간의 기본적인 트레이드오프가 포함됩니다:

**저주파 시스템(12-50kHz)**: 낮은 주파수는 감쇠가 적어 물을 통해 더 멀리 전파되므로 심해 매핑에 적합합니다. 12kHz 시스템은 6,000미터를 초과하는 수심에서 효과적으로 작동할 수 있습니다. 그러나 낮은 주파수는 파장이 더 크므로 더 큰 음향 발자국과 낮은 해상도를 초래합니다. 심해 시스템은 일반적으로 수 미터에서 수십 미터의 수평 해상도를 달성합니다.

**중주파 시스템(70-200kHz)**: 이 시스템은 범위와 해상도 사이의 절충안을 나타내며 대륙붕과 경사 매핑에 적합합니다. 3,000-4,000미터 깊이까지 효과적으로 작동할 수 있으며 수심과 시스템 구성에 따라 1-5미터의 수평 해상도를 제공합니다.

**고주파 시스템(300-400kHz 이상)**: 고주파 시스템은 우수한 해상도를 제공하지만 음향 감쇠 증가로 인해 범위가 제한됩니다. 이 시스템은 서브미터 해상도가 필요한 얕은 물 응용 분야, 연안 조사 및 상세 검사에서 탁월합니다. 일부 특수 시스템은 매우 얕은 물에서 매우 높은 해상도 이미징을 위해 훨씬 더 높은 주파수(1MHz 이상)에서 작동합니다.

주파수와 해상도 사이의 관계는 기본 음향 원리에서 따릅니다. 해저의 음향 발자국 크기는 빔 폭과 범위에 의해 결정됩니다. 빔 폭은 음향 파장과 변환기 배열의 크기에 따라 달라집니다. 더 높은 주파수(더 짧은 파장)는 더 좁은 빔과 더 작은 발자국을 가능하게 하여 더 높은 해상도를 생성합니다.

## 2.4 스와스 범위 및 조사 계획

다중빔 시스템의 주요 장점 중 하나는 단일 통과로 넓은 해저 스와스를 매핑할 수 있다는 것입니다. 스와스 폭은 여러 요인에 따라 달라집니다:

**최대 사용 가능 스와스**: 다중빔 시스템이 이론적으로 매우 넓은 스와스를 달성할 수 있지만 실용적 한계가 사용 가능한 스와스 폭을 제한합니다. 외부 빔은 경사 각도로 해저에 도달하여 더 큰 발자국과 낮은 해상도를 초래합니다. 신호 대 잡음비는 경로 길이 및 빔 확산 증가로 인해 외부 빔에 대해 감소합니다. 대부분의 조사는 전체 폭에 걸쳐 일관된 데이터 품질을 유지하기 위해 사용 가능한 스와스를 제한합니다.

**깊이 의존성**: 스와스 폭은 수심에 따라 조정됩니다. 100미터 물에서 140도 스와스 각도를 가진 시스템은 약 350미터의 해저 폭을 매핑할 수 있습니다. 4,000미터 물에서 동일한 시스템은 14킬로미터를 덮습니다. 이 스케일링은 조사 효율성이 실제로 수심 증가와 함께 향상된다는 것을 의미합니다(단위 시간당 덮인 면적 측면에서), 비록 이것은 더 깊은 물에서 종종 필요한 느린 선박 속도에 의해 부분적으로 상쇄됩니다.

**라인 간격 및 중첩**: 조사 계획자는 격차 없이 완전한 범위를 보장하고 과도한 중첩을 피하기 위해 적절한 라인 간격을 결정해야 합니다. 일반적으로 인접한 조사 라인은 격차가 없고 품질 평가를 위한 중복 데이터를 제공하기 위해 10-20% 중첩으로 계획됩니다. 중첩 백분율은 복잡한 지형 지역이나 매우 높은 데이터 밀도가 필요한 경우 증가할 수 있습니다.

**조사 속도**: 선박 속도는 트랙을 따라 데이터 밀도에 영향을 미칩니다. 더 빠른 속도는 조사 효율성을 높이지만 트랙을 따라 데이터 밀도를 줄입니다. 최적 속도는 시스템 핑 속도, 원하는 데이터 밀도, 해상 조건 및 선박 기능에 따라 달라집니다. 일반적인 조사 속도는 이러한 요인에 따라 4-12노트 범위입니다.

## 2.5 데이터 품질 및 불확실성

불확실성을 이해하고 정량화하는 것은 수심 데이터의 적절한 사용에 중요합니다. 여러 요인이 전체 불확실성 예산에 기여합니다:

**측정 불확실성**: 이동 시간 측정의 기본 정밀도는 이론적 수직 해상도를 결정합니다. 현대 시스템은 나노초 정밀도로 이동 시간을 측정하여 서브센티미터 범위 해상도를 가능하게 합니다. 그러나 다른 요인이 일반적으로 전체 불확실성 예산을 지배합니다.

**음속 불확실성**: 가정된 음속 프로파일의 오류는 깊이 오류로 직접 변환됩니다. 음속의 1% 오류는 계산된 깊이의 1% 오류를 생성합니다. 4,000미터 물에서 이것은 40미터의 수직 오류를 나타낼 수 있습니다. 정기적인 음속 프로파일은 이 오류 소스를 최소화하는 데 필수적입니다.

**운동 보상**: 선박 운동의 불완전한 측정 또는 보상은 수직 및 수평 오류를 발생시킵니다. 현대 고성능 MRU는 이 오류 소스를 최소화하지만 매우 거친 바다에서는 특히 외부 빔에 대해 운동 유발 오류가 여전히 중요할 수 있습니다.

**위치 오류**: 수평 위치 오류는 깊이 측정의 지리적 위치에 영향을 미칩니다. 현대 GNSS 위치 지정을 사용하면 수평 오류는 일반적으로 1미터 미만이며, 대부분의 응용 프로그램에 대한 다른 오류 소스에 비해 무시할 수 있습니다.

**해저 특성**: 해저의 특성은 측정 품질에 영향을 미칩니다. 딱딱하고 매끄러운 표면은 강하고 명확한 반환을 생성합니다. 부드러운 퇴적물은 음향 에너지를 흡수하여 반환을 약화시킵니다. 복잡한 미세 지형은 혼란스러운 반환 신호를 생성할 수 있습니다. 이러한 요인은 바닥 감지 신뢰성과 정확성에 영향을 미칩니다.

## 2.6 고급 기능

현대 다중빔 시스템은 단순한 깊이 측정을 넘어서는 기능을 제공합니다:

**후방 산란 이미징**: 깊이 측정 외에도 다중빔 시스템은 반환된 신호의 강도를 기록합니다. 후방 산란 강도는 해저 구성, 거칠기 및 입사각에 따라 달라집니다. 후방 산란 데이터를 처리하면 해저 질감과 구성을 나타내는 음향 이미지가 생성되어 수심 데이터를 보완합니다.

**수주 이미징**: 일부 다중빔 시스템은 해저 에코뿐만 아니라 전체 수주 반환을 기록할 수 있습니다. 수주 데이터는 어군, 가스 누출, 열수 플룸 및 기타 중수 특징을 나타냅니다. 이 기능은 다중빔 시스템을 해저 매핑 도구에서 포괄적인 해양 관측 기기로 변환합니다.

**스니펫 데이터**: 스니펫 데이터는 각 빔의 바닥 감지 주변의 시계열 샘플로 구성됩니다. 이 추가 정보는 후처리에서 더 정교한 바닥 감지 알고리즘을 가능하게 하고 해저 산란 특성에 대한 통찰력을 제공합니다.

**다중 주파수**: 일부 고급 시스템은 동시에 또는 순차적으로 여러 주파수로 송신하여 다양한 범위-해상도 트레이드오프를 가진 보완 데이터를 제공할 수 있습니다. 이 기능은 복잡한 해저 환경을 매핑하는 데 특히 유용합니다.

## 2.7 TypeScript 구현 예제

다음은 주요 다중빔 데이터 처리 개념을 보여주는 TypeScript 구현입니다:

```typescript
/**
 * WIA-OCEAN-005: 다중빔 소나 데이터 처리
 * Ocean Blue Theme: #0EA5E9
 */

interface MultibeamPing {
  timestamp: Date;
  vesselPosition: Position;
  vesselAttitude: Attitude;
  beams: BeamData[];
  soundVelocityProfile: SoundVelocity[];
}

interface Position {
  latitude: number;
  longitude: number;
  altitude: number;
}

interface Attitude {
  roll: number;      // 도
  pitch: number;     // 도
  heading: number;   // 진북 기준 도
  heave: number;     // 미터
}

interface BeamData {
  beamNumber: number;
  angle: number;           // 천저에서 도
  travelTime: number;      // 초
  backscatter: number;     // dB
  quality: number;         // 0-100
}

interface SoundVelocity {
  depth: number;           // 미터
  velocity: number;        // 미터/초
}

interface SeafloorPoint {
  position: Position;
  depth: number;
  uncertainty: number;
  backscatter: number;
}

/**
 * 다중빔 핑을 처리하여 해저 위치 계산
 */
function processPing(ping: MultibeamPing): SeafloorPoint[] {
  const points: SeafloorPoint[] = [];

  for (const beam of ping.beams) {
    // 저품질 빔 건너뛰기
    if (beam.quality < 50) continue;

    // 음속 프로파일을 통한 광선 추적을 사용하여 경사 범위 계산
    const slantRange = calculateRange(
      beam.travelTime,
      ping.soundVelocityProfile
    );

    // 운동 보상 적용
    const correctedAngle = applyMotionCorrection(
      beam.angle,
      ping.vesselAttitude
    );

    // 선박 참조 프레임에서 해저 지점 계산
    const acrossTrack = slantRange * Math.sin(correctedAngle * Math.PI / 180);
    const depth = slantRange * Math.cos(correctedAngle * Math.PI / 180);

    // 지리적 좌표로 변환
    const position = transformToGeographic(
      ping.vesselPosition,
      ping.vesselAttitude.heading,
      acrossTrack,
      0  // 트랙 따라 오프셋
    );

    // 불확실성 추정
    const uncertainty = estimateUncertainty(
      depth,
      beam.angle,
      ping.soundVelocityProfile
    );

    points.push({
      position: position,
      depth: ping.vesselPosition.altitude - depth,
      uncertainty: uncertainty,
      backscatter: beam.backscatter
    });
  }

  return points;
}

// 예제 사용
const examplePing: MultibeamPing = {
  timestamp: new Date(),
  vesselPosition: {
    latitude: 35.5,
    longitude: 139.7,
    altitude: 0
  },
  vesselAttitude: {
    roll: 2.5,
    pitch: -1.2,
    heading: 045,
    heave: 0.8
  },
  beams: Array.from({ length: 256 }, (_, i) => ({
    beamNumber: i,
    angle: (i - 128) * 0.5,
    travelTime: 0.5 + Math.random() * 0.1,
    backscatter: -20 + Math.random() * 10,
    quality: 80 + Math.random() * 20
  })),
  soundVelocityProfile: [
    { depth: 0, velocity: 1500 },
    { depth: 100, velocity: 1490 },
    { depth: 500, velocity: 1485 }
  ]
};

const seafloorPoints = processPing(examplePing);
console.log(`${seafloorPoints.length}개의 해저 지점 처리 완료`);
```
