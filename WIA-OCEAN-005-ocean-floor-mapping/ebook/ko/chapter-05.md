# 제5장: 3D 모델링 및 시각화

## 5.1 시각적 표현의 힘

인간의 뇌는 시각 정보 처리에 탁월합니다. 잘 만들어진 3D 시각화는 숫자 테이블이나 2D 지도에서는 숨겨져 있을 수 있는 패턴, 관계 및 특징을 나타낼 수 있습니다. 해저 매핑의 경우 3D 시각화는 수심 데이터를 추상적인 깊이 값에서 수중 경관의 직관적인 표현으로 변환하여 연구자, 항해사 및 대중이 해저의 복잡성과 아름다움을 이해할 수 있게 합니다.

현대 시각화 기술은 강력한 그래픽 하드웨어와 정교한 소프트웨어를 활용하여 수백만 개의 데이터 포인트를 실시간으로 렌더링합니다. 사용자는 가상 수중 경관을 날아다니고, 모든 각도에서 특징을 검사하고, 다양한 색상 구성표를 적용하여 지형의 다양한 측면을 강조할 수 있습니다. 이러한 기능은 해저를 탐험, 분석 및 정보 전달하는 방식을 혁신했습니다.

WIA-OCEAN-005 표준은 수심 데이터가 다양한 소프트웨어 플랫폼에서 효과적으로 렌더링될 수 있도록 시각화 요구 사항을 다룹니다. 표준화된 형식, 좌표 시스템 및 메타데이터는 시각화 기술 및 사용자 인터페이스의 혁신을 허용하면서 상호 운용성을 가능하게 합니다.

## 5.2 디지털 고도 모델

3D 해저 시각화의 기초는 수심 격자 또는 디지털 지형 모델(DTM)이라고도 하는 디지털 고도 모델(DEM)입니다. DEM은 해저를 정규 고도 값 격자로 나타내어 이산 측정에서 연속 표면 모델을 제공합니다.

**격자 구조**: 대부분의 DEM은 지리적 또는 투영된 좌표와 정렬된 정규 직사각형 격자를 사용합니다. 각 셀에는 해당 위치의 해저 고도를 나타내는 고도 값(또는 수심의 깊이 값)이 포함됩니다. 격자 간격은 해상도와 파일 크기를 결정합니다. 더 미세한 격자는 더 많은 세부 사항을 보존하지만 더 많은 저장 공간과 처리 능력이 필요합니다.

**데이터 형식**: 수많은 파일 형식이 격자화된 수심 데이터를 저장합니다:
- **GeoTIFF**: 내장된 지리 참조가 있는 널리 지원되는 래스터 형식
- **NetCDF**: 해양학 응용 프로그램에서 널리 사용되는 자체 설명 형식
- **BAG(Bathymetric Attributed Grid)**: 수심 데이터용으로 특별히 설계되었으며 불확실성 레이어 포함
- **ASCII Grid**: 간단한 텍스트 형식, 사람이 읽을 수 있지만 대규모 데이터 세트에는 비효율적
- **Cloud-Optimized GeoTIFF(COG)**: 네트워크를 통해 대규모 데이터 세트의 일부에 효율적으로 액세스 가능

**다중 해상도 표현**: 대면적 DEM은 종종 여러 해상도를 저장하는 계층적 구조를 사용합니다. 이를 통해 다양한 확대/축소 수준에서 효율적인 시각화가 가능합니다. 지역 보기에는 거친 해상도, 세부 영역으로 확대하면 미세 해상도가 사용됩니다. 쿼드트리 및 타일 피라미드와 같은 기술은 다중 해상도 데이터를 효율적으로 구성합니다.

## 5.3 표면 렌더링 기술

고도 값 격자를 매력적인 3D 시각화로 변환하려면 정교한 렌더링 기술이 필요합니다:

**삼각 불규칙 네트워크(TIN)**: TIN은 표면을 겹치지 않는 삼각형 네트워크로 나타냅니다. 각 삼각형의 꼭지점에는 알려진 위치와 고도가 있습니다. TIN은 가변 세부 사항으로 표면을 효율적으로 나타냅니다. 복잡한 지역에는 작은 삼각형, 평평한 지역에는 큰 삼각형입니다. 그러나 격자화된 데이터에서 최적의 TIN을 만들려면 정교한 알고리즘이 필요합니다.

**음영 기복**: 인공 조명은 2D 디스플레이에서도 3D 지형의 모양을 만듭니다. 렌더링 알고리즘은 가상 광원에 대한 방향에 따라 각 표면 요소가 받는 빛의 양을 계산합니다. 빛을 향한 표면은 밝게 나타나고 멀어진 표면은 어둡게 나타납니다. 결과는 자연 세계에서 모양을 인식하는 데 도움이 되는 그림자와 하이라이트를 모방합니다.

**다중 방향 음영**: 단일 소스 조명은 그림자의 특징을 가릴 수 있습니다. 다중 방향 음영은 여러 각도의 조명을 결합하여 모든 특징이 최소한 부분적으로 조명되도록 합니다. 이 기술은 3D 모양을 유지하면서 복잡한 지형의 세부 사항을 나타냅니다.

**텍스처 매핑**: 추가 정보를 텍스처로 오버레이하면 시각화가 향상됩니다. 소나 시스템의 후방 산란 강도는 수심 표면 위에 드레이프될 수 있어 모양과 음향 특성을 모두 보여줍니다. 분류된 서식지 지도, 지질학적 해석 또는 기타 파생 제품은 텍스처로 표시될 수 있습니다.

**수직 과장**: 해저 경사는 종종 미묘합니다. 대면적은 불과 몇 퍼센트의 기울기를 가질 수 있습니다. 수직 과장은 수평에 비해 수직 스케일을 증폭하여 미묘한 특징을 더 명확하게 만듭니다. 그러나 과도한 과장은 오해의 소지가 있는 인상을 만들 수 있습니다. WIA-OCEAN-005 표준은 수직 과장 계수를 명확하게 표시할 것을 권장합니다.

## 5.4 색상 매핑 및 기호화

효과적인 색상 구성표는 이해를 향상시키면서 오해를 방지합니다:

**순차 색상 램프**: 이 구성표는 점진적인 색상 전환을 사용하여 연속적인 깊이 변화를 나타냅니다. 전통적인 파란색에서 흰색 구성표는 더 깊은 물을 더 어두운 파란색으로, 얕아지면 더 밝은 파란색으로, 마지막으로 흰색으로 나타냅니다. 다른 구성표는 특정 응용 프로그램에 최적화된 파란색-녹색-노란색-빨간색 또는 사용자 정의 범위를 사용합니다.

**지각적으로 균일한 색상 스케일**: 전통적인 무지개 색상 맵은 인식된 색상 차이가 데이터 차이와 균일하게 일치하지 않기 때문에 오해의 소지가 있을 수 있습니다. Viridis와 같은 현대 지각적으로 균일한 구성표는 동일한 데이터 단계가 동일한 인식된 색상 단계를 생성하도록 합니다.

**수심 색상 팔레트**: WIA-OCEAN-005에 지정된 Ocean Blue 테마(#0EA5E9)는 시작점을 제공하지만 효과적인 시각화에는 신중하게 설계된 색상 진행이 필요합니다:

```typescript
/**
 * WIA-OCEAN-005: 수심 색상 매핑
 * Ocean Blue Theme: #0EA5E9
 */

function createOceanPalette(): BathymetricColorScheme {
  return {
    name: 'WIA Ocean Blue',
    colors: [
      { r: 14, g: 165, b: 233 },   // 얕은: #0EA5E9
      { r: 6, g: 182, b: 212 },    // 중간-얕은: #06B6D4
      { r: 8, g: 145, b: 178 },    // 중간: #0891B2
      { r: 14, g: 116, b: 144 },   // 중간-깊은: #0E7490
      { r: 12, g: 74, b: 110 },    // 깊은: #0C4A6E
      { r: 3, g: 24, b: 60 }       // 매우 깊은: #03183C
    ],
    depths: [0, -50, -200, -1000, -4000, -6000]
  };
}
```

## 5.5 대화형 3D 시각화

현대 웹 기술은 웹 브라우저에서 직접 정교한 3D 시각화를 가능하게 합니다:

**WebGL 렌더링**: WebGL API는 웹 브라우저에서 GPU 가속 3D 그래픽에 대한 액세스를 제공합니다. Three.js, Babylon.js 및 Cesium과 같은 라이브러리는 WebGL 개발을 단순화하여 저수준 그래픽 프로그래밍 없이 대화형 3D 뷰어를 만들 수 있습니다.

**세부 수준 관리**: 대규모 수심 데이터 세트를 효율적으로 렌더링하려면 세부 수준(LOD) 기술이 필요합니다. 시각화 시스템은 현재 보기에 필요한 데이터만 로드하고 표시하며, 사용자가 확대/축소함에 따라 해상도 수준 간에 자동으로 전환합니다.

**사용자 상호 작용**: 효과적인 시각화는 직관적인 상호 작용을 지원합니다:
- 팬: 수평으로 시점 이동
- 확대/축소: 보기 거리 조정
- 회전: 보기 각도 변경
- 측정: 거리, 면적 및 고도 차이 계산
- 쿼리: 특정 위치에 대한 정보 표시
- 애니메이션: 비행 경로 및 시계열 시각화 생성
