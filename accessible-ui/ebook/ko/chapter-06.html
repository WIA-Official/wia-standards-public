제6장: 보조기술 통합

보조기술(Assistive Technology, AT)은 장애를 가진 사용자가 디지털 콘텐츠에 접근할 수 있게 해주는 하드웨어와 소프트웨어입니다. 이 장에서는 다양한 보조기술의 작동 원리, WAI-ARIA의 상세 사용법, 화면 판독기 호환성, 그리고 접근성 API에 대해 다룹니다.

보조기술 유형

화면 판독기 (Screen Reader)

화면의 텍스트와 UI 요소를 음성으로 읽어주는 소프트웨어입니다. 시각 장애인이 주로 사용합니다.

  • JAWS: Windows용, 가장 널리 사용되는 상용 화면 판독기
  • NVDA: Windows용 무료 오픈소스 화면 판독기
  • VoiceOver: Apple 기기 내장 (macOS, iOS)
  • TalkBack: Android 내장 화면 판독기
  • Narrator: Windows 내장 화면 판독기

화면 확대기 (Screen Magnifier)

저시력 사용자를 위해 화면의 일부를 확대하여 표시합니다. 색상 반전, 커서 강조 등의 기능도 제공합니다.

  • ZoomText: Windows용 상용 확대 소프트웨어
  • Windows 확대경: Windows 내장 기능
  • macOS 확대/축소: macOS 내장 기능

대체 입력 장치

표준 키보드나 마우스를 사용할 수 없는 사용자를 위한 장치입니다.

  • 스위치: 단일 또는 소수의 버튼으로 입력
  • 시선 추적: 눈의 움직임으로 포인터 제어
  • 음성 인식: Dragon NaturallySpeaking, Voice Control
  • 헤드 포인터: 머리 움직임으로 제어

접근성 API와 접근성 트리

운영체제는 접근성 API를 통해 보조기술에 UI 정보를 제공합니다. 브라우저는 DOM을 기반으로 접근성 트리(Accessibility Tree)를 구성하여 이 API에 정보를 전달합니다.

접근성 트리 구성 요소:
  • 이름 (Name): 요소를 식별하는 텍스트 (레이블)
  • 역할 (Role): 요소의 유형 (버튼, 링크, 체크박스 등)
  • 상태 (State): 현재 상태 (선택됨, 확장됨, 비활성화 등)
  • 값 (Value): 입력 필드의 값, 진행률 등
// 접근성 트리 정보 확인 (개발자 도구)
// Chrome DevTools > Elements > Accessibility 탭

// HTML 요소의 접근성 정보 예시
/*
버튼 요소:
Name: "제출"
Role: button
State: focusable, enabled

체크박스 요소:
Name: "이용약관 동의"
Role: checkbox
State: focusable, checked
*/

// JavaScript로 접근성 정보 확인
const button = document.querySelector('button');
console.log('역할:', button.getAttribute('role') || button.tagName);
console.log('이름:', button.textContent || button.getAttribute('aria-label'));
console.log('상태:', button.disabled ? '비활성화' : '활성화');

WAI-ARIA 상세 가이드

WAI-ARIA(Accessible Rich Internet Applications)는 HTML의 접근성을 보완하는 속성 집합입니다. 동적 콘텐츠와 사용자 정의 위젯의 접근성을 향상시킵니다.

ARIA 역할 (Roles)

랜드마크 역할

페이지의 주요 영역을 정의하여 화면 판독기 사용자가 빠르게 탐색할 수 있게 합니다.

<!-- 랜드마크 역할 -->
<header role="banner">
    <!-- 사이트 헤더, 로고, 전역 탐색 -->
</header>

<nav role="navigation" aria-label="주요 탐색">
    <!-- 주요 탐색 메뉴 -->
</nav>

<main role="main">
    <!-- 페이지의 주요 콘텐츠 -->

    <article role="article">
        <!-- 독립적인 콘텐츠 블록 -->
    </article>

    <aside role="complementary">
        <!-- 보조 콘텐츠 -->
    </aside>

    <form role="search" aria-label="사이트 검색">
        <!-- 검색 양식 -->
    </form>

    <section role="region" aria-labelledby="section-heading">
        <h2 id="section-heading">섹션 제목</h2>
        <!-- 이름이 있는 영역 -->
    </section>
</main>

<footer role="contentinfo">
    <!-- 페이지 바닥글, 저작권, 연락처 -->
</footer>

위젯 역할

ARIA 상태와 속성

속성 용도 예시
aria-expanded 확장/축소 상태 아코디언, 드롭다운
aria-selected 선택 상태 탭, 리스트 아이템
aria-checked 체크 상태 체크박스, 토글
aria-pressed 눌림 상태 토글 버튼
aria-disabled 비활성화 상태 비활성화된 컨트롤
aria-hidden 접근성 트리에서 숨김 장식용 요소
aria-live 동적 콘텐츠 알림 상태 메시지, 오류
aria-describedby 설명 연결 힌트, 오류 메시지

라이브 리전 (Live Regions)

라이브 리전은 동적으로 변경되는 콘텐츠를 화면 판독기에 자동으로 알립니다:

<!-- 라이브 리전 예시 -->

<!-- 상태 메시지 (예의적 알림) -->
<div role="status" aria-live="polite" aria-atomic="true">
    장바구니에 상품이 추가되었습니다.
</div>

<!-- 긴급 알림 (즉시 알림) -->
<div role="alert" aria-live="assertive">
    세션이 곧 만료됩니다. 계속하시겠습니까?
</div>

<!-- 로그/채팅 (새 항목만 알림) -->
<div role="log" aria-live="polite" aria-atomic="false">
    <p>사용자A: 안녕하세요</p>
    <p>사용자B: 반갑습니다</p>
    <!-- 새 메시지가 추가되면 자동 알림 -->
</div>

<!-- 진행 상태 -->
<div role="progressbar"
     aria-valuenow="70"
     aria-valuemin="0"
     aria-valuemax="100"
     aria-label="파일 업로드 진행률">
    70%
</div>

<!-- JavaScript로 라이브 리전 업데이트 -->
<script>
function showNotification(message, type = 'polite') {
    const container = document.getElementById('notification-container');
    container.setAttribute('aria-live', type);
    container.textContent = message;

    // 잠시 후 내용 비우기 (다음 알림 준비)
    setTimeout(() => {
        container.textContent = '';
    }, 5000);
}

// 사용 예
showNotification('저장되었습니다.');
showNotification('오류가 발생했습니다!', 'assertive');
</script>

화면 판독기 테스트

실제 화면 판독기로 테스트하는 것이 중요합니다. 다음은 주요 테스트 포인트입니다:

화면 판독기 테스트 체크리스트:
  1. 모든 이미지에 적절한 대체 텍스트가 있는가?
  2. 폼 필드에 레이블이 올바르게 연결되어 있는가?
  3. 페이지 제목이 페이지 내용을 반영하는가?
  4. 제목 구조가 논리적인가?
  5. 링크 텍스트만으로 목적지를 알 수 있는가?
  6. 동적 콘텐츠 변경이 알려지는가?
  7. 모달이 열릴 때 포커스가 이동하는가?
  8. 오류 메시지가 즉시 알려지는가?

화면 판독기 단축키

기능 NVDA VoiceOver (macOS)
다음 제목 H VO + Command + H
다음 링크 K VO + Command + L
다음 폼 필드 F VO + Command + J
랜드마크 이동 D VO + U (로터)
요소 목록 NVDA + F7 VO + U

ARIA 사용 규칙

ARIA 사용 5가지 규칙:
  1. 네이티브 HTML 요소가 있으면 ARIA 대신 그것을 사용하세요.
  2. 네이티브 의미론을 변경하지 마세요.
  3. 모든 대화형 ARIA 컨트롤은 키보드로 사용 가능해야 합니다.
  4. 포커스 가능한 요소에 role="presentation"이나 aria-hidden="true"를 사용하지 마세요.
  5. 모든 대화형 요소에는 접근 가능한 이름이 있어야 합니다.
