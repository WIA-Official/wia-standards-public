제5장: 보안과 인증 접근성

보안은 모든 디지털 서비스의 핵심이지만, 보안 메커니즘이 접근성 장벽이 되어서는 안 됩니다. 이 장에서는 접근 가능하면서도 안전한 인증 방식, CAPTCHA 대안, 그리고 개인정보 보호와 접근성의 균형에 대해 다룹니다. WCAG 2.2에서 새롭게 추가된 접근 가능한 인증 기준도 포함합니다.

접근 가능한 인증의 원칙

핵심 원칙

인증 과정은 사용자의 인지 능력에 과도한 부담을 주지 않아야 합니다. 기억력, 문제 해결 능력, 시각적 인식에 의존하는 인증 방식은 다양한 장애를 가진 사용자에게 장벽이 될 수 있습니다.

WCAG 2.2 접근 가능한 인증 기준

WCAG 2.2는 인지 기능 테스트에 의존하지 않는 인증 방법을 요구합니다:

  • 3.3.8 접근 가능한 인증 (최소) - AA: 인지 기능 테스트(퍼즐, 기억, 전사 등)가 필요한 경우 대안을 제공해야 합니다.
  • 3.3.9 접근 가능한 인증 (강화) - AAA: 인지 기능 테스트 없이 인증할 수 있어야 합니다.

인증 방식별 접근성

비밀번호 인증

전통적인 비밀번호 인증의 접근성을 높이는 방법:

  • 비밀번호 표시/숨기기 토글 버튼 제공
  • 복사-붙여넣기 허용 (비밀번호 관리자 사용 가능)
  • 명확한 비밀번호 요구사항 안내
  • 실시간 유효성 검사 피드백
<!-- 접근 가능한 비밀번호 입력 -->
<div class="password-field">
    <label for="password">비밀번호</label>

    <div class="input-wrapper">
        <input type="password"
               id="password"
               name="password"
               autocomplete="current-password"
               aria-describedby="password-requirements password-strength">

        <button type="button"
                class="toggle-password"
                aria-pressed="false"
                aria-label="비밀번호 표시">
            <span aria-hidden="true">👁</span>
        </button>
    </div>

    <div id="password-requirements" class="hint">
        8자 이상, 대문자, 소문자, 숫자 포함
    </div>

    <div id="password-strength"
         role="status"
         aria-live="polite"
         aria-atomic="true">
        <!-- 비밀번호 강도 표시 -->
    </div>
</div>

<script>
// 비밀번호 표시 토글
const toggleBtn = document.querySelector('.toggle-password');
const passwordInput = document.getElementById('password');

toggleBtn.addEventListener('click', () => {
    const isVisible = passwordInput.type === 'text';
    passwordInput.type = isVisible ? 'password' : 'text';
    toggleBtn.setAttribute('aria-pressed', !isVisible);
    toggleBtn.setAttribute('aria-label',
        isVisible ? '비밀번호 표시' : '비밀번호 숨기기');
});
</script>

2단계 인증 (2FA)

2단계 인증의 접근성 고려사항:

  • SMS, 이메일, 인증 앱 등 다양한 옵션 제공
  • 인증 코드 입력에 충분한 시간 허용
  • 인증 코드 복사-붙여넣기 지원
  • 음성 전화 옵션 제공 (청각 장애인용 문자 대안 포함)
<!-- 접근 가능한 OTP 입력 -->
<fieldset>
    <legend>인증 코드 입력</legend>

    <p id="otp-instructions">
        휴대폰으로 전송된 6자리 코드를 입력하세요.
        코드는 5분간 유효합니다.
    </p>

    <!-- 단일 입력 필드 방식 (권장) -->
    <label for="otp-input" class="visually-hidden">
        인증 코드 6자리
    </label>
    <input type="text"
           id="otp-input"
           name="otp"
           inputmode="numeric"
           pattern="[0-9]{6}"
           maxlength="6"
           autocomplete="one-time-code"
           aria-describedby="otp-instructions otp-timer">

    <div id="otp-timer"
         role="timer"
         aria-live="polite"
         aria-atomic="true">
        남은 시간: <span>4:59</span>
    </div>

    <button type="button" class="resend-code">
        코드 재전송
    </button>
</fieldset>

생체 인증

지문, 얼굴 인식 등 생체 인증의 접근성:

  • 항상 대체 인증 방법 제공 (생체 정보 사용 불가한 경우)
  • 명확한 지시와 피드백 제공
  • 개인정보 보호 설명 제공

패스키 (Passkeys)

FIDO2/WebAuthn 기반 패스키는 접근성 측면에서 우수합니다:

  • 기억해야 할 비밀번호 없음
  • 피싱에 안전
  • 다양한 인증 방식 지원 (생체인식, PIN, 보안키)
// WebAuthn 패스키 인증 핵심 코드
async function authenticateWithPasskey() {
    const status = document.getElementById('auth-status');
    status.textContent = '장치에서 인증을 완료해주세요.';
    status.setAttribute('aria-busy', 'true');

    try {
        const credential = await navigator.credentials.get({
            publicKey: { challenge, timeout: 60000 }
        });
        status.textContent = '인증 완료';
        status.setAttribute('aria-busy', 'false');
    } catch (error) {
        status.textContent = '인증 실패. 다른 방법을 시도하세요.';
    }
}

CAPTCHA 접근성

전통적인 CAPTCHA는 심각한 접근성 문제가 있습니다. 시각 장애인은 이미지 기반 CAPTCHA를, 청각 장애인은 오디오 CAPTCHA를 사용할 수 없습니다.

CAPTCHA의 문제점:
  • 이미지 CAPTCHA: 시각 장애인, 인지 장애인에게 불가능
  • 오디오 CAPTCHA: 청각 장애인, 비원어민에게 어려움
  • 퍼즐 CAPTCHA: 운동 장애인, 인지 장애인에게 장벽
  • 시간 제한: 처리 속도가 느린 사용자에게 불리

접근 가능한 CAPTCHA 대안

대안 장점 고려사항
reCAPTCHA v3 사용자 상호작용 불필요, 백그라운드 분석 점수 임계값 조정 필요
hCaptcha 접근성 쿠키 접근성 통과 옵션 제공 사전 등록 필요
허니팟 필드 완전히 투명, 사용자 부담 없음 단독으로는 보안 약함
시간 기반 분석 사용자 부담 없음 추가 보안 조치 필요 
<!-- 허니팟 필드 구현 -->
<form action="/submit" method="POST">
    <!-- 실제 필드 -->
    <label for="real-email">이메일</label>
    <input type="email" id="real-email" name="email" required>

    <!-- 허니팟 필드 (봇만 채움) -->
    <div class="visually-hidden" aria-hidden="true">
        <label for="website">웹사이트를 입력하지 마세요</label>
        <input type="text"
               id="website"
               name="website"
               tabindex="-1"
               autocomplete="off">
    </div>

    <button type="submit">제출</button>
</form>

세션 관리와 타임아웃

세션 타임아웃은 보안상 필요하지만, 일부 사용자는 작업 완료에 더 많은 시간이 필요합니다:

// 접근 가능한 세션 경고 대화상자
function showTimeoutWarning() {
    const dialog = document.createElement('div');
    dialog.setAttribute('role', 'alertdialog');
    dialog.setAttribute('aria-modal', 'true');
    dialog.setAttribute('aria-labelledby', 'timeout-title');
    dialog.innerHTML = `
        <h2 id="timeout-title">세션 만료 경고</h2>
        <p>5분 후 로그아웃됩니다.</p>
        <button id="extend-session">세션 연장</button>
    `;
    document.body.appendChild(dialog);
    document.getElementById('extend-session').focus();
}

데이터 보호와 프라이버시

프라이버시 접근성: 개인정보 보호 설정, 동의 양식, 데이터 관리 인터페이스도 접근 가능해야 합니다. 복잡한 법적 문서는 읽기 쉬운 요약과 함께 제공하세요.
  • 개인정보 처리방침을 명확하고 이해하기 쉽게 작성
  • 동의 체크박스에 명확한 레이블 제공
  • 데이터 삭제 요청 등의 기능을 접근 가능하게 구현
  • 민감한 정보 표시 시 화면 판독기 고려 (예: aria-hidden으로 마스킹)
