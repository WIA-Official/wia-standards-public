제4장: 상호작용 패턴

접근 가능한 사용자 인터페이스는 다양한 입력 방식을 지원해야 합니다. 이 장에서는 키보드 탐색, 터치 인터페이스, 음성 제어, 그리고 포인터 장치에 대한 접근성 패턴을 다룹니다. 모든 사용자가 자신에게 편한 방식으로 인터페이스와 상호작용할 수 있도록 설계하는 것이 목표입니다.

키보드 접근성

키보드 접근성은 웹 접근성의 핵심입니다. 마우스를 사용할 수 없는 사용자(시각 장애인, 운동 장애인 등)는 키보드나 키보드를 에뮬레이션하는 보조기술에 의존합니다.

기본 키보드 탐색

키 기능
Tab 다음 포커스 가능한 요소로 이동
Shift + Tab 이전 포커스 가능한 요소로 이동
Enter 링크 활성화, 버튼 클릭
Space 버튼 클릭, 체크박스 토글
Esc 모달 닫기, 드롭다운 닫기
방향키 메뉴/탭/라디오 버튼 내 이동

포커스 관리

포커스 관리는 사용자가 현재 위치를 파악하고 다음 행동을 결정하는 데 필수적입니다:

/* 포커스 표시 스타일 */
:focus {
    outline: 2px solid #0066cc;
    outline-offset: 2px;
}

/* 마우스 클릭 시에만 아웃라인 숨기기 (키보드 사용자에게는 표시) */
:focus:not(:focus-visible) {
    outline: none;
}

:focus-visible {
    outline: 3px solid #0066cc;
    outline-offset: 2px;
}

/* 고대비 모드 지원 */
@media (forced-colors: active) {
    :focus {
        outline: 3px solid CanvasText;
    }
}

탭 순서 관리

// 포커스 관리 핵심 함수
function saveFocus() {
    window.lastFocused = document.activeElement;
}

function restoreFocus() {
    if (window.lastFocused) window.lastFocused.focus();
}

function getFocusableElements(container) {
    return container.querySelectorAll(
        'a[href], button:not([disabled]), input:not([disabled]), [tabindex]:not([tabindex="-1"])'
    );
}
tabindex 사용 가이드:
  • tabindex="0": 자연스러운 탭 순서에 포함
  • tabindex="-1": 프로그래밍적으로만 포커스 가능 (탭 순서에서 제외)
  • tabindex="1+": 사용 지양 - DOM 순서와 탭 순서 불일치 유발

포커스 트랩

모달 대화상자와 같은 UI에서는 포커스를 해당 영역 내에 가두어야 합니다:

// 포커스 트랩 구현
class FocusTrap {
    constructor(container) {
        this.container = container;
        this.focusableElements = this.getFocusableElements();
        this.firstElement = this.focusableElements[0];
        this.lastElement = this.focusableElements[this.focusableElements.length - 1];

        this.handleKeyDown = this.handleKeyDown.bind(this);
    }

    getFocusableElements() {
        return this.container.querySelectorAll(
            'button, [href], input, select, textarea, [tabindex]:not([tabindex="-1"])'
        );
    }

    activate() {
        this.container.addEventListener('keydown', this.handleKeyDown);
        // 첫 번째 요소에 포커스
        this.firstElement?.focus();
    }

    deactivate() {
        this.container.removeEventListener('keydown', this.handleKeyDown);
    }

    handleKeyDown(e) {
        if (e.key !== 'Tab') return;

        // Shift + Tab: 첫 번째 요소에서 마지막으로 이동
        if (e.shiftKey && document.activeElement === this.firstElement) {
            e.preventDefault();
            this.lastElement.focus();
        }
        // Tab: 마지막 요소에서 첫 번째로 이동
        else if (!e.shiftKey && document.activeElement === this.lastElement) {
            e.preventDefault();
            this.firstElement.focus();
        }
    }
}

// 모달에 적용
const modal = document.querySelector('[role="dialog"]');
const trap = new FocusTrap(modal);

function openModal() {
    modal.hidden = false;
    modal.setAttribute('aria-hidden', 'false');
    trap.activate();
}

function closeModal() {
    trap.deactivate();
    modal.hidden = true;
    modal.setAttribute('aria-hidden', 'true');
    // 원래 포커스로 복귀
}

터치 인터페이스

터치 인터페이스는 모바일 기기 사용자와 터치스크린 장치 사용자를 위한 접근성을 고려해야 합니다.

터치 대상 크기

WCAG 2.2 새 기준 (2.5.8 Target Size): 터치 대상은 최소 24x24 CSS 픽셀이어야 하며, AAA 수준에서는 44x44 CSS 픽셀을 권장합니다. 인접한 대상 사이에는 충분한 간격이 필요합니다. 
/* 터치 대상 크기 */
.touch-target {
    /* 최소 크기 보장 */
    min-width: 44px;
    min-height: 44px;

    /* 패딩으로 터치 영역 확대 */
    padding: 12px;
}

/* 아이콘 버튼의 경우 */
.icon-button {
    width: 44px;
    height: 44px;
    display: flex;
    align-items: center;
    justify-content: center;
}

.icon-button svg {
    width: 24px;
    height: 24px;
}

/* 인접 버튼 간격 */
.button-group {
    display: flex;
    gap: 8px;
}

/* 링크 목록 */
.link-list li {
    padding: 12px 0;
}

/* 폼 컨트롤 */
input[type="checkbox"],
input[type="radio"] {
    width: 24px;
    height: 24px;
}

/* 사용자 정의 체크박스 */
.custom-checkbox {
    position: relative;
    padding-left: 44px;
    min-height: 44px;
    display: flex;
    align-items: center;
}

.custom-checkbox input {
    position: absolute;
    opacity: 0;
    width: 44px;
    height: 44px;
    left: 0;
    top: 0;
    cursor: pointer;
}

.custom-checkbox .checkmark {
    position: absolute;
    left: 10px;
    width: 24px;
    height: 24px;
    border: 2px solid #333;
    border-radius: 4px;
}

제스처 대안

복잡한 제스처(핀치, 스와이프 등)에는 단순한 대안을 제공해야 합니다:

제스처와 대안 매핑

  • 핀치 줌: + / - 버튼 제공
  • 스와이프 삭제: 삭제 버튼 제공
  • 드래그 앤 드롭: 이동 버튼 또는 컨텍스트 메뉴
  • 더블 탭: 단일 탭으로 동일 기능 제공

음성 제어

음성 제어는 운동 장애가 있는 사용자에게 필수적인 입력 방식입니다. Dragon NaturallySpeaking, Voice Control(macOS/iOS) 등의 음성 인식 소프트웨어와 호환되어야 합니다.

<!-- 음성 제어 친화적 마크업 -->

<!-- 1. 보이는 레이블과 접근 가능한 이름 일치 -->
<button>제출</button>  <!-- "제출" 클릭이라고 말하면 동작 -->

<!-- 잘못된 예 -->
<button aria-label="양식 제출하기">제출</button>
<!-- 사용자가 "제출"이라고 말해도 인식 안 될 수 있음 -->

<!-- 2. 폼 레이블 명확하게 -->
<label for="search-input">검색어</label>
<input type="text" id="search-input" name="search">
<!-- "검색어"라고 말하면 해당 필드에 포커스 -->

<!-- 3. 링크 텍스트 고유하게 -->
<a href="/products">제품 보기</a>
<a href="/services">서비스 보기</a>
<!-- "제품 보기" 또는 "서비스 보기"로 구분 가능 -->

<!-- 잘못된 예: 동일한 링크 텍스트 -->
<a href="/products">더 보기</a>
<a href="/services">더 보기</a>
<!-- "더 보기"라고 말하면 어느 링크인지 불명확 -->
WCAG 2.5.3 Label in Name: 사용자 인터페이스 구성요소에 텍스트나 이미지가 포함된 레이블이 있는 경우, 접근 가능한 이름에 시각적으로 표시되는 텍스트가 포함되어야 합니다.

포인터 입력

마우스, 트랙패드, 스타일러스 등 다양한 포인터 장치를 지원해야 합니다:

/* 포인터 유형에 따른 스타일 */

/* 정밀한 포인터 (마우스, 스타일러스) */
@media (pointer: fine) {
    .button {
        padding: 8px 16px;
    }
}

/* 거친 포인터 (터치) */
@media (pointer: coarse) {
    .button {
        padding: 12px 24px;
        min-height: 44px;
    }
}

/* 호버 가능한 장치 */
@media (hover: hover) {
    .button:hover {
        background-color: #0056b3;
    }

    .tooltip-trigger:hover + .tooltip {
        display: block;
    }
}

/* 호버 불가능한 장치 */
@media (hover: none) {
    /* 호버 대신 터치/클릭으로 활성화 */
    .tooltip {
        /* 항상 표시하거나 클릭으로 토글 */
    }
}

모션과 애니메이션

애니메이션은 일부 사용자에게 어지러움, 메스꺼움을 유발할 수 있으므로 제어 옵션을 제공해야 합니다:

/* 사용자의 모션 감소 설정 존중 */
@media (prefers-reduced-motion: reduce) {
    *,
    *::before,
    *::after {
        animation-duration: 0.01ms !important;
        animation-iteration-count: 1 !important;
        transition-duration: 0.01ms !important;
        scroll-behavior: auto !important;
    }
}

/* 기본 애니메이션 */
.animated-element {
    transition: transform 0.3s ease, opacity 0.3s ease;
}

/* 안전한 애니메이션만 사용 */
.safe-animation {
    /* 작은 움직임, 페이드 효과는 일반적으로 안전 */
    transition: opacity 0.2s ease;
}

/* 위험한 애니메이션 피하기 */
.avoid-this {
    /* 큰 움직임, 회전, 확대/축소는 주의 */
    /* animation: spin 1s infinite; -- 피해야 함 */
}
발작 위험: 초당 3회 이상 깜빡이는 콘텐츠는 광과민성 발작을 유발할 수 있습니다. 절대 사용하지 마세요.
