제3장: 시각적 접근성

시각적 접근성은 저시력 사용자, 색맹 사용자, 그리고 다양한 시각적 환경에서 콘텐츠를 사용하는 모든 사용자를 위한 설계 원칙입니다. 이 장에서는 색상 대비, 색상에만 의존하지 않는 정보 전달, 타이포그래피, 그리고 레이아웃의 접근성을 다룹니다.

색상 대비 요구사항

WCAG는 텍스트와 배경 간의 최소 대비 비율을 규정합니다. 대비 비율은 1:1(동일 색상)부터 21:1(검정과 흰색)까지의 범위를 가집니다.

WCAG 대비 요구사항

텍스트 유형 AA 수준 AAA 수준
일반 텍스트 (18pt 미만) 4.5:1 7:1
큰 텍스트 (18pt 이상 또는 14pt 굵게) 3:1 4.5:1
UI 컴포넌트 및 그래픽 객체 3:1 해당 없음
대비 계산: 대비 비율은 두 색상의 상대 휘도를 기반으로 계산됩니다. 온라인 대비 검사 도구나 브라우저 개발자 도구를 사용하여 검사할 수 있습니다.

대비 검사 및 수정

// JavaScript로 대비 비율 계산하기
function getLuminance(r, g, b) {
    const [rs, gs, bs] = [r, g, b].map(c => {
        c = c / 255;
        return c <= 0.03928
            ? c / 12.92
            : Math.pow((c + 0.055) / 1.055, 2.4);
    });
    return 0.2126 * rs + 0.7152 * gs + 0.0722 * bs;
}

function getContrastRatio(color1, color2) {
    const l1 = getLuminance(...color1);
    const l2 = getLuminance(...color2);
    const lighter = Math.max(l1, l2);
    const darker = Math.min(l1, l2);
    return (lighter + 0.05) / (darker + 0.05);
}

// 사용 예시
const textColor = [33, 37, 41];     // #212529
const bgColor = [255, 255, 255];    // #ffffff
const ratio = getContrastRatio(textColor, bgColor);
console.log(`대비 비율: ${ratio.toFixed(2)}:1`);  // 약 16:1

// WCAG 준수 여부 확인
function meetsWCAG(ratio, level = 'AA', isLargeText = false) {
    const requirements = {
        'AA': { normal: 4.5, large: 3 },
        'AAA': { normal: 7, large: 4.5 }
    };
    const required = isLargeText
        ? requirements[level].large
        : requirements[level].normal;
    return ratio >= required;
}

조합 1: 배경 #ffffff, 텍스트 #1a5f7a - 대비 비율 7.2:1 (AAA 통과)
조합 2: 배경 #1a5f7a, 텍스트 #ffffff - 대비 비율 7.2:1 (AAA 통과)

색상에만 의존하지 않기

색상은 정보를 전달하는 유일한 수단이 되어서는 안 됩니다. 색맹 사용자(남성의 약 8%, 여성의 약 0.5%)를 위해 색상 외의 시각적 단서를 추가로 제공해야 합니다.

잘못된 예: "빨간색 항목은 오류, 녹색 항목은 성공을 나타냅니다."
올바른 예: "오류 항목에는 X 아이콘과 빨간색, 성공 항목에는 체크 아이콘과 녹색이 표시됩니다."

색상 외 정보 전달 방법

<!-- 잘못된 예: 색상만으로 상태 표시 -->
<span style="color: red">필수 항목</span>

<!-- 올바른 예: 색상 + 텍스트 + 아이콘 -->
<span class="required-field">
    <span class="icon" aria-hidden="true">*</span>
    필수 항목
</span>

<!-- 차트에서의 구분 -->

타이포그래피 접근성

읽기 쉬운 타이포그래피는 모든 사용자의 콘텐츠 이해를 돕습니다. 특히 난독증이나 시각 장애가 있는 사용자에게 중요합니다.

폰트 선택 가이드라인

  • 산세리프 폰트 권장: 화면에서 읽기 쉬운 Malgun Gothic, Noto Sans KR 등
  • 코드용 고정폭 폰트: D2Coding, Consolas 등
  • 장식용 폰트 제한: 본문 텍스트에는 장식 폰트 사용 자제
  • 문자 구분: l(소문자 L), 1(숫자 1), I(대문자 i)가 명확히 구분되는 폰트

텍스트 크기와 줄 간격

/* 접근 가능한 타이포그래피 CSS */
:root {
    /* 기본 폰트 크기 - 브라우저 설정 존중 */
    font-size: 100%; /* 16px가 기본값 */
}

body {
    /* 상대 단위 사용 */
    font-size: 1rem;

    /* 충분한 줄 간격 (1.5 이상 권장) */
    line-height: 1.6;

    /* 적절한 문단 간격 */
    letter-spacing: 0.01em;
}

/* 단락 간격 */
p {
    margin-bottom: 1.5em;
}

/* 제목 계층 */
h1 {
    font-size: 2.5rem;
    line-height: 1.2;
    margin-bottom: 1rem;
}

h2 {
    font-size: 2rem;
    line-height: 1.3;
    margin-top: 2rem;
    margin-bottom: 0.75rem;
}

h3 {
    font-size: 1.5rem;
    line-height: 1.4;
    margin-top: 1.5rem;
    margin-bottom: 0.5rem;
}

/* 최대 줄 길이 - 가독성을 위해 45-75자 권장 */
.content {
    max-width: 70ch;
}

/* 텍스트 정렬 - 양쪽 정렬은 난독증 사용자에게 어려울 수 있음 */
.readable-text {
    text-align: left;
}

/* 사용자 설정 존중 */
@media (prefers-reduced-motion: reduce) {
    * {
        scroll-behavior: auto !important;
    }
}

/* 고대비 모드 지원 */
@media (prefers-contrast: high) {
    body {
        background-color: #000;
        color: #fff;
    }

    a {
        color: #ffff00;
    }
}
WCAG 2.1 새 기준: 텍스트 간격을 사용자가 조정해도 콘텐츠나 기능 손실 없이 사용할 수 있어야 합니다 (줄 높이 1.5배, 단락 간격 2배, 글자 간격 0.12배, 단어 간격 0.16배까지).

레이아웃 접근성

레이아웃은 콘텐츠의 논리적 흐름을 지원하고, 다양한 화면 크기와 확대 수준에서 사용 가능해야 합니다.

반응형 설계

/* 모바일 우선 반응형 설계 */
.container {
    width: 100%;
    padding: 1rem;
}

/* 콘텐츠 영역 */
.content {
    max-width: 900px;
    margin: 0 auto;
}

/* 400% 확대 지원 - 320px 뷰포트에서 사용 가능해야 함 */
@media (max-width: 320px) {
    .container {
        padding: 0.5rem;
    }

    h1 {
        font-size: 1.5rem;
    }

    /* 수평 스크롤 방지 */
    img, video, iframe {
        max-width: 100%;
        height: auto;
    }
}

/* 그리드 레이아웃 - 콘텐츠 순서 유지 */
.grid-layout {
    display: grid;
    grid-template-columns: repeat(auto-fit, minmax(280px, 1fr));
    gap: 1.5rem;
}

/* Flexbox - 순서 변경 주의 */
.flex-container {
    display: flex;
    flex-wrap: wrap;
    gap: 1rem;
}

/* 시각적 순서와 DOM 순서 일치 */
/* order 속성 사용 시 주의 - 스크린 리더는 DOM 순서를 따름 */
.sidebar {
    /* order: -1; -- 접근성 문제 발생 가능 */
}

확대/축소 지원

WCAG 2.1은 200% 확대까지 콘텐츠 손실 없이 사용할 수 있어야 하며, WCAG 2.2는 텍스트만 400%까지 확대하는 경우도 고려합니다.

검증 방법:
  1. 브라우저 확대/축소를 200%로 설정
  2. 모든 콘텐츠가 표시되는지 확인
  3. 수평 스크롤 없이 읽을 수 있는지 확인
  4. 모든 기능이 작동하는지 확인

이미지 접근성

이미지는 적절한 대체 텍스트와 함께 제공되어야 하며, 이미지 유형에 따라 대체 텍스트 전략이 달라집니다:

이미지 유형 대체 텍스트 전략 예시
정보 전달 이미지가 전달하는 정보 설명 alt="2024년 매출 그래프: 전년 대비 15% 증가"
기능적 버튼/링크의 기능 설명 alt="검색" 또는 aria-label="검색"
장식용 빈 alt 속성 alt="" role="presentation"
복잡한 이미지 간단한 alt + 상세 설명 alt="조직도" aria-describedby="org-desc"

요약

시각적 접근성은 색상 대비, 색상 독립적 정보 전달, 읽기 쉬운 타이포그래피, 유연한 레이아웃을 포함합니다. 이러한 원칙을 적용하면 시각 장애가 있는 사용자뿐 아니라 다양한 환경에서 콘텐츠를 사용하는 모든 사용자의 경험이 향상됩니다. 다음 장에서는 상호작용 패턴과 키보드 접근성을 다룹니다.
