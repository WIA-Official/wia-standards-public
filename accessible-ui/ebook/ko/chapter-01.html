제1장: 접근성 UI 기초

디지털 접근성은 장애를 가진 사람들이 웹사이트, 도구, 기술을 사용할 수 있도록 보장하는 것을 의미합니다. 이 장에서는 접근성의 기본 개념, 핵심 용어, 그리고 접근성 표준의 역사적 발전 과정을 살펴봅니다. 접근성 UI 설계의 토대가 되는 지식을 제공하여 이후 장에서 다루는 기술적 내용을 이해하는 데 도움을 줍니다.

접근성의 정의와 중요성

웹 접근성(Web Accessibility)은 모든 사람이 웹을 인식하고, 이해하고, 탐색하고, 상호작용할 수 있으며, 웹에 기여할 수 있음을 의미합니다. 접근성은 다양한 장애를 가진 사람들에게 혜택을 주며, 여기에는 청각, 인지, 신경, 신체, 언어, 시각 장애가 포함됩니다.

통계: 세계 인구의 약 15%인 10억 명 이상이 어떤 형태로든 장애를 갖고 있습니다. 디지털 접근성을 무시하는 것은 상당한 규모의 사용자층을 배제하는 것과 같습니다.

접근성의 중요성은 다음과 같은 측면에서 강조됩니다:

  • 인권: 디지털 접근은 현대 사회에서 기본권으로 인식되고 있습니다.
  • 법적 의무: 많은 국가에서 디지털 접근성을 법으로 의무화하고 있습니다.
  • 비즈니스 가치: 더 넓은 사용자층에 도달하고, 브랜드 평판을 높이며, 검색엔진 최적화에 기여합니다.
  • 기술적 우수성: 접근 가능한 코드는 일반적으로 더 구조적이고 유지보수가 용이합니다.

핵심 용어 정의

WCAG (Web Content Accessibility Guidelines)

W3C의 웹 접근성 이니셔티브(WAI)가 개발한 국제 표준 지침입니다. 웹 콘텐츠를 더 접근 가능하게 만들기 위한 권고사항을 제공합니다. 현재 최신 버전은 WCAG 2.2입니다.

WAI-ARIA (Accessible Rich Internet Applications)

동적 콘텐츠와 고급 사용자 인터페이스 컨트롤의 접근성을 향상시키기 위한 기술 사양입니다. HTML 속성을 추가하여 보조기술에 추가 정보를 제공합니다.

보조기술 (Assistive Technology)

장애인이 디지털 콘텐츠에 접근하는 데 도움을 주는 하드웨어나 소프트웨어입니다. 화면 판독기, 화면 확대기, 대체 키보드, 스위치 장치 등이 포함됩니다.

화면 판독기 (Screen Reader)

시각 장애인을 위해 화면의 텍스트와 UI 요소를 음성으로 읽어주는 소프트웨어입니다. JAWS, NVDA, VoiceOver, TalkBack 등이 대표적입니다.

장애 유형과 접근성 요구사항

효과적인 접근성 설계를 위해서는 다양한 장애 유형과 각각의 요구사항을 이해해야 합니다:

장애 유형 주요 특성 접근성 요구사항
시각 장애 전맹, 저시력, 색맹 대체 텍스트, 색상 대비, 화면 판독기 호환성
청각 장애 농아, 난청 자막, 수화 비디오, 시각적 알림
운동 장애 미세 운동 조절 어려움 키보드 접근성, 큰 클릭 영역, 충분한 시간
인지 장애 학습 장애, 주의력 장애 단순한 언어, 일관된 탐색, 명확한 지시
발작 장애 광과민성 발작 깜빡임 제한, 애니메이션 제어

접근성 표준의 역사

웹 접근성 표준은 인터넷의 발전과 함께 진화해 왔습니다:

1997년

W3C가 웹 접근성 이니셔티브(WAI)를 설립하여 웹 접근성 향상을 위한 활동을 시작했습니다.

1999년

WCAG 1.0이 발표되어 최초의 국제 웹 접근성 표준이 되었습니다.

2008년

WCAG 2.0이 발표되어 기술 중립적인 접근 방식을 도입했습니다.

2014년

WAI-ARIA 1.0이 권고안으로 발표되어 동적 웹 콘텐츠의 접근성을 향상시켰습니다.

2018년

WCAG 2.1이 발표되어 모바일 접근성, 저시력 사용자, 인지 장애인을 위한 기준을 추가했습니다.

2023년

WCAG 2.2가 발표되어 인지 장애, 이동 장애, 모바일 사용자를 위한 새로운 성공 기준을 추가했습니다.

WCAG 적합성 수준

WCAG는 세 가지 적합성 수준을 정의합니다:

  • Level A: 최소 접근성 수준으로, 반드시 충족해야 하는 기본 요구사항입니다.
  • Level AA: 권장되는 접근성 수준으로, 대부분의 법규에서 요구하는 수준입니다.
  • Level AAA: 최고 수준의 접근성으로, 모든 콘텐츠에 적용하기 어려울 수 있습니다.
권장 수준: 일반적으로 WCAG 2.1 Level AA 준수를 목표로 합니다. 이는 법적 요구사항을 충족하면서도 실질적으로 달성 가능한 수준입니다.

접근성 설계의 기본 원칙

다음은 접근 가능한 인터페이스를 설계할 때 따라야 할 기본 원칙입니다:

/* 접근성 기본 CSS 설정 */

/* 1. 기본 폰트 크기 설정 - 사용자 설정 존중 */
html {
    font-size: 100%; /* 브라우저 기본값 16px */
}

/* 2. 상대 단위 사용 */
body {
    font-size: 1rem;
    line-height: 1.5;
}

h1 { font-size: 2rem; }
h2 { font-size: 1.5rem; }

/* 3. 포커스 표시 유지 */
:focus {
    outline: 2px solid #0066cc;
    outline-offset: 2px;
}

/* 4. 포커스 표시 제거 금지 - 잘못된 예 */
/* :focus { outline: none; } - 절대 사용하지 마세요! */

/* 5. 색상 대비 확보 */
.text-content {
    color: #333333; /* 배경 #ffffff 대비 12.6:1 */
    background-color: #ffffff;
}

/* 6. 움직임 감소 설정 존중 */
@media (prefers-reduced-motion: reduce) {
    * {
        animation: none !important;
        transition: none !important;
    }
}

/* 7. 고대비 모드 지원 */
@media (prefers-contrast: high) {
    .button {
        border: 2px solid currentColor;
    }
}

국내 접근성 법규

한국에서는 다음과 같은 법규와 표준이 웹 접근성을 규정합니다:

  • 장애인차별금지 및 권리구제 등에 관한 법률: 2008년 시행, 웹 접근성 의무화의 법적 근거
  • 국가정보화 기본법: 공공기관의 웹 접근성 준수 의무 명시
  • 한국형 웹 콘텐츠 접근성 지침 2.2: WCAG를 기반으로 한 국내 표준
  • 웹 접근성 품질마크: 접근성 준수 사이트에 부여되는 인증 마크
주의: 접근성 의무 미준수 시 장애인차별금지법에 따라 손해배상 청구 및 시정명령의 대상이 될 수 있습니다. 공공기관은 물론 민간 기업도 해당됩니다.

접근성 설계의 이점

접근성 원칙을 적용하면 다음과 같은 추가적인 이점이 있습니다:

  • SEO 향상: 시맨틱 마크업과 대체 텍스트는 검색엔진 최적화에 기여합니다.
  • 모바일 호환성: 터치 대상 크기, 반응형 설계는 모바일 사용성을 높입니다.
  • 노년층 지원: 고령 사용자의 시력, 청력, 운동 능력 저하를 보완합니다.
  • 상황적 장애 대응: 밝은 햇빛, 시끄러운 환경 등 일시적 제약 상황에도 대응합니다.
  • 기술 부채 감소: 처음부터 접근성을 고려하면 나중에 수정하는 비용을 절감합니다.

요약

이 장에서는 접근성 UI의 기초 개념과 용어, 역사적 발전 과정을 살펴보았습니다. 접근성은 단순히 장애인을 위한 것이 아니라 모든 사용자의 경험을 향상시키는 보편적 설계 원칙입니다. 다음 장에서는 이러한 기초 위에 구체적인 아키텍처 설계와 WCAG 원칙의 실제 적용 방법을 알아봅니다.
