Chapter 07

Testing and Monitoring

7.1 The Testing Pyramid

Automated testing catches 20-30% of issues. Manual testing catches issues requiring human judgment. User testing with people with disabilities reveals real-world usability issues.

Testing Layers
  • Base - Automated: Fast, frequent, run on every commit
  • Middle - Manual: Keyboard and screen reader testing per release
  • Top - User Testing: With people with disabilities for major features

7.2 Automated Testing Tools

axe-core

Industry-standard testing engine. Zero false positives by design.

Lighthouse

Built into Chrome DevTools. Part of broader performance analysis.

Pa11y

Command-line testing. Excellent for CI pipelines.

WAVE

Visual feedback showing issues directly on page.

// Jest + Puppeteer + axe-core
const { AxePuppeteer } = require('@axe-core/puppeteer');

test('Page has no accessibility violations', async () => {
  await page.goto('http://localhost:3000');

  const results = await new AxePuppeteer(page)
    .withTags(['wcag2a', 'wcag2aa'])
    .analyze();

  expect(results.violations).toHaveLength(0);
});

// Cypress + cypress-axe
it('Has no detectable violations', () => {
  cy.visit('/');
  cy.injectAxe();
  cy.checkA11y();
});

// Keyboard Testing Protocol
const keyboardTests = [
  'Tab through all interactive elements',
  'Verify focus order matches visual order',

  'Verify skip links work',
  'Confirm focus indicator always visible'

  'Check dynamic content updates'

7.6 User Testing

User Testing Best Practices
  • Recruit participants who regularly use assistive technology
  • Compensate participants fairly for their expertise
  • Allow participants to use their own devices and AT
  • Focus on task completion, not just issue finding
  • Ask participants to think aloud about their experience

7.7 Continuous Monitoring

Production monitoring catches issues that slip through testing and tracks accessibility metrics over time. Set up automated scanning on a regular schedule, monitor for regressions after deployments, and establish alerting thresholds for accessibility score drops.

7.8 Summary

Comprehensive accessibility testing combines automated scanning, manual expert review, and user testing with people with disabilities. By integrating these approaches into your development workflow and continuously monitoring production, you maintain high accessibility standards and catch issues before they impact users.
