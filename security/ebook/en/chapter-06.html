Chapter 06

Security Operations Center

6.1 Introduction to Security Operations

A Security Operations Center (SOC) serves as the central hub for an organization's security monitoring, detection, and response capabilities. The SOC combines people, processes, and technology to continuously monitor and improve the organization's security posture while preventing, detecting, analyzing, and responding to cybersecurity incidents.

Modern SOCs have evolved from simple monitoring rooms to sophisticated operations centers leveraging artificial intelligence, automation, and advanced analytics. This chapter explores how to build, operate, and optimize an effective SOC aligned with WIA-SEC-001 standards.

6.1.1 SOC Functions

Core Functions

  • 24/7 security monitoring
  • Alert triage and investigation
  • Incident detection and response
  • Threat intelligence integration
  • Vulnerability management

Advanced Functions

  • Threat hunting
  • Forensic investigation
  • Security automation
  • Metrics and reporting
  • Continuous improvement

6.1.2 SOC Models

Model Description Advantages Considerations
In-House SOC Fully staffed and operated internally Full control, institutional knowledge High cost, staffing challenges
Managed SOC (MSSP) Outsourced to security provider Cost-effective, 24/7 coverage Less control, integration challenges
Hybrid SOC Combination of internal and managed Flexibility, optimized costs Coordination complexity
Virtual SOC Distributed team, no physical center Reduced overhead, flexibility Communication challenges

6.2.2 Log Sources

Effective SIEM requires comprehensive log collection from across the enterprise:

Essential Log Sources for SIEM: ================================ Network Security: • Firewalls (allow/deny, connection logs) • IDS/IPS (alerts, detected threats) • VPN (connection logs, authentication) • DNS (queries, responses) • Proxy/Web Gateway (URL access, blocks) Identity & Access: • Active Directory (authentication, changes) • Identity Providers (SSO, MFA events) • Privileged Access Management (sessions) • Badge/Physical Access (entry/exit) Endpoint: • EDR/Antivirus (detections, actions) • Operating System (security events) • Application logs Cloud: • Cloud provider logs (CloudTrail, Activity Logs) • SaaS application logs • Container/Kubernetes logs Applications: • Web servers • Database audit logs • Business applications

6.2.3 Correlation Rules

SIEM correlation rules identify security events by analyzing relationships between log entries:

Rule Types

  • Single-event rules (pattern match)
  • Threshold rules (count over time)
  • Sequence rules (ordered events)
  • Statistical rules (deviation from baseline)
  • Lookup rules (against watchlists)

Common Detection Scenarios

  • Brute force attacks
  • Privilege escalation
  • Lateral movement
  • Data exfiltration
  • Malware execution

6.3 SOAR Implementation

Security Orchestration, Automation, and Response (SOAR) platforms extend SIEM capabilities by automating incident response workflows, orchestrating actions across security tools, and managing the incident lifecycle.

6.3.1 SOAR Components

The Three Pillars of SOAR

Orchestration: Connecting disparate security tools to work together through APIs and integrations

Automation: Executing repetitive tasks without human intervention using playbooks and workflows

Response: Managing the incident lifecycle from detection through containment and remediation

6.3.2 Automation Use Cases

Use Case Manual Time Automated Time Actions Automated
Phishing Triage 30 minutes 2 minutes Extract IOCs, check reputation, block sender
Malware Alert 45 minutes 5 minutes Isolate host, collect artifacts, create ticket
User Offboarding 2 hours 10 minutes Disable accounts, revoke access, audit logs
Vulnerability Scan 4 hours 30 minutes Run scan, enrich findings, create tickets

6.4 Incident Response

Incident response is the organized approach to addressing and managing security incidents. Effective incident response minimizes damage, reduces recovery time and costs, and captures lessons for continuous improvement.

6.4.1 Incident Response Phases

  1. Preparation: Develop response plans, train teams, and implement detection capabilities
  2. Identification: Detect and determine if an event is a security incident
  3. Containment: Limit the scope and impact of the incident
  4. Eradication: Remove the threat from the environment
  5. Recovery: Restore systems to normal operation
  6. Lessons Learned: Document findings and improve processes

6.4.2 Incident Classification

Severity Definition Response Time Escalation
Critical (P1) Active breach, widespread impact Immediate CISO, Executive Team
High (P2) Confirmed threat, limited impact Within 1 hour Security Manager
Medium (P3) Suspicious activity, needs investigation Within 4 hours SOC Lead
Low (P4) Policy violation, minor security event Within 24 hours SOC Analyst

6.4.3 Containment Strategies

Short-Term Containment

  • Network isolation
  • Account disabling
  • Firewall blocks
  • Traffic rerouting

Long-Term Containment

  • System patching
  • Temporary systems
  • Enhanced monitoring
  • Access restrictions

Evidence Preservation

During containment and eradication, preserve evidence for forensic analysis and potential legal proceedings. Document all actions, capture system states, and maintain chain of custody for all collected evidence.

6.5 SOC Metrics and KPIs

Measuring SOC performance is essential for demonstrating value, identifying improvement opportunities, and meeting WIA-SEC-001 requirements.

6.5.1 Key Performance Indicators

< 30 min

< 5%
False Positive Rate

6.5.2 Operational Metrics

SOC Operational Metrics: ========================= Detection Metrics: • Events per Second (EPS) processed • Alerts generated per day • Coverage (% of assets monitored) • Detection rate by attack type Response Metrics: • Mean Time to Acknowledge (MTTA) • Mean Time to Contain (MTTC) • Mean Time to Remediate (MTTR) • Incidents per analyst Quality Metrics: • False positive rate • Escalation rate • Reopened incidents • SLA compliance Team Metrics: • Analyst utilization • Training hours completed • Certifications obtained • Employee satisfaction

6.5.3 WIA-SEC-001 SOC Requirements

Certification Level SOC Requirements
Bronze Basic SIEM, business hours monitoring, documented IR plan
Silver 24/7 monitoring (internal or MSSP), SIEM with correlation, formal IR procedures
Gold Advanced SIEM/SOAR, threat hunting program, MTTD < 30 min, regular exercises
Platinum AI-enhanced detection, automated response, MTTD < 15 min, continuous improvement

6.6.2 Skills and Training

SOC analysts require a combination of technical skills and soft skills:

Technical Skills

  • SIEM operation and query languages
  • Network protocols and analysis
  • Operating system internals
  • Malware analysis basics
  • Scripting (Python, PowerShell)

Soft Skills

  • Analytical thinking
  • Communication
  • Attention to detail
  • Stress management
  • Continuous learning

Chapter Summary

A well-functioning SOC is essential for detecting and responding to security threats. Key takeaways:

  • SOCs combine people, processes, and technology for comprehensive security operations
  • SIEM provides the foundation for log collection, correlation, and alerting
  • SOAR enables automation and orchestration to improve efficiency
  • Incident response follows defined phases from preparation through lessons learned
  • Metrics and KPIs measure SOC effectiveness and drive improvement
  • Skilled personnel organized in tiered structures are essential for SOC success
