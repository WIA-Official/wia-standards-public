Chapter 2: Threat Modeling

2.1 Introduction to Threat Modeling

Threat modeling is a structured approach to identifying, quantifying, and addressing security threats throughout the software development lifecycle. It enables organizations to proactively discover vulnerabilities, assess risks, and implement appropriate countermeasures before systems are deployed to production.

Core Philosophy: The goal of threat modeling is not to eliminate all risks, but to understand them well enough to make informed decisions about which risks to accept, mitigate, transfer, or avoid.

Threat Modeling Methodologies

Methodology Focus Best For Complexity
STRIDE Threat categorization Software applications, System design Medium
PASTA Risk-centric approach Enterprise applications, Business alignment High
VAST Visual, Agile, Simple DevOps, Continuous integration Low-Medium
LINDDUN Privacy threats GDPR compliance, Privacy-sensitive apps Medium
Attack Trees Attack path analysis Specific security controls, Red teaming Medium

2.2 STRIDE Threat Model

STRIDE is a threat modeling framework developed by Microsoft that categorizes threats into six main categories. Each letter in STRIDE represents a different type of security threat that can compromise system security.

STRIDE Categories

Category Threat Type Security Property Example Attack Mitigation
Spoofing Identity Authentication Session hijacking, Credential theft MFA, Strong authentication
Tampering Data Integrity Integrity SQL injection, Data modification Digital signatures, Hashing
Repudiation Non-repudiation Non-repudiation Denying actions, Log tampering Audit logs, Digital signatures
Information Disclosure Confidentiality Confidentiality Data breach, Eavesdropping Encryption, Access controls
Denial of Service Availability Availability DDoS, Resource exhaustion Rate limiting, Load balancing
Elevation of Privilege Authorization Authorization Privilege escalation, Admin bypass Least privilege, RBAC

2.4 Threat Modeling API Endpoints

The WIA Security Standard provides comprehensive APIs for managing threat models throughout their lifecycle.

POST /api/v1/threat-models

Create a new threat model for an application or system

GET /api/v1/threat-models/{modelId}

Retrieve a specific threat model by ID

PUT /api/v1/threat-models/{modelId}

Update an existing threat model

DELETE /api/v1/threat-models/{modelId}

Archive a threat model

POST /api/v1/threat-models/{modelId}/threats

Add a new threat to an existing threat model

PUT /api/v1/threat-models/{modelId}/threats/{threatId}

Update threat details or risk rating

POST /api/v1/threat-models/{modelId}/threats/{threatId}/mitigations

Add mitigation strategy to a specific threat

GET /api/v1/threat-models/{modelId}/analysis

Get threat analysis and risk summary

POST /api/v1/threat-models/{modelId}/validate

Validate threat model completeness and quality

GET /api/v1/threat-models/search

Search threat models by application, owner, or risk level

POST /api/v1/threat-models/{modelId}/export

Export threat model in various formats (PDF, JSON, Excel)

GET /api/v1/threat-intelligence/feed

Get latest threat intelligence and emerging threats

2.5 Implementation Examples

Example 1: STRIDE Threat Analysis (Python)

Python - STRIDE Threat Analyzer
from typing import List, Dict
from enum import Enum
import uuid
from datetime import datetime

class ThreatCategory(Enum):
    SPOOFING = "Spoofing"
    TAMPERING = "Tampering"
    REPUDIATION = "Repudiation"
    INFORMATION_DISCLOSURE = "InformationDisclosure"
    DENIAL_OF_SERVICE = "DenialOfService"
    ELEVATION_OF_PRIVILEGE = "ElevationOfPrivilege"

class RiskLevel(Enum):
    LOW = "Low"
    MEDIUM = "Medium"
    HIGH = "High"
    CRITICAL = "Critical"

class ThreatAnalyzer:
    """
    STRIDE-based threat analysis engine for WIA Security Standard
    """

    def __init__(self):
        self.threat_patterns = self._load_threat_patterns()

    def create_threat_model(self, application_name: str, version: str) -> Dict:
        """Create a new threat model"""
        threat_model = {
            "threatModelId": str(uuid.uuid4()),
            "metadata": {
                "applicationName": application_name,
                "version": version,
                "createdDate": datetime.utcnow().isoformat(),
                "lastUpdated": datetime.utcnow().isoformat(),
                "methodology": "STRIDE"
            },
            "systemDescription": {
                "overview": "",
                "architecture": "",
                "dataFlow": [],
                "trustBoundaries": []
            },
            "threats": [],
            "assumptions": [],
            "dependencies": []
        }
        return threat_model

    def add_threat(self, threat_model: Dict, threat_data: Dict) -> Dict:
        """Add a threat to the threat model"""
        threat = {
            "threatId": str(uuid.uuid4()),
            "title": threat_data["title"],
            "category": threat_data["category"],
            "description": threat_data["description"],
            "affectedComponents": threat_data.get("affectedComponents", []),
            "attackVector": threat_data.get("attackVector", ""),
            "riskRating": self.calculate_risk(
                threat_data.get("likelihood", "Medium"),
                threat_data.get("impact", "Medium")
            ),
            "mitigations": [],
            "status": "Open"
        }

        threat_model["threats"].append(threat)
        threat_model["metadata"]["lastUpdated"] = datetime.utcnow().isoformat()
        return threat

    def calculate_risk(self, likelihood: str, impact: str) -> Dict:
        """Calculate overall risk based on likelihood and impact"""
        risk_matrix = {
            ("Low", "Low"): "Low",
            ("Low", "Medium"): "Low",
            ("Low", "High"): "Medium",
            ("Low", "Critical"): "High",
            ("Medium", "Low"): "Low",
            ("Medium", "Medium"): "Medium",
            ("Medium", "High"): "High",
            ("Medium", "Critical"): "Critical",
            ("High", "Low"): "Medium",
            ("High", "Medium"): "High",
            ("High", "High"): "Critical",
            ("High", "Critical"): "Critical",
            ("Critical", "Low"): "High",
            ("Critical", "Medium"): "Critical",
            ("Critical", "High"): "Critical",
            ("Critical", "Critical"): "Critical"
        }

        overall_risk = risk_matrix.get((likelihood, impact), "Medium")

        return {
            "likelihood": likelihood,
            "impact": impact,
            "overallRisk": overall_risk
        }

    def add_mitigation(self, threat: Dict, mitigation_data: Dict) -> Dict:
        """Add mitigation strategy to a threat"""
        mitigation = {
            "mitigationId": str(uuid.uuid4()),
            "strategy": mitigation_data["strategy"],
            "implementation": mitigation_data["implementation"],
            "status": mitigation_data.get("status", "Planned"),
            "effectiveness": mitigation_data.get("effectiveness", "Medium")
        }

        threat["mitigations"].append(mitigation)
        return mitigation

    def analyze_stride_coverage(self, threat_model: Dict) -> Dict:
        """Analyze STRIDE coverage of the threat model"""
        coverage = {category.value: 0 for category in ThreatCategory}

        for threat in threat_model["threats"]:
            category = threat["category"]
            coverage[category] = coverage.get(category, 0) + 1

        total_threats = len(threat_model["threats"])
        gaps = [cat for cat, count in coverage.items() if count == 0]

        return {
            "totalThreats": total_threats,
            "categoryDistribution": coverage,
            "gaps": gaps,
            "coverageScore": (6 - len(gaps)) / 6 * 100
        }

    def get_high_risk_threats(self, threat_model: Dict) -> List[Dict]:
        """Get all high and critical risk threats"""
        high_risk = []
        for threat in threat_model["threats"]:
            risk = threat["riskRating"]["overallRisk"]
            if risk in ["High", "Critical"]:
                high_risk.append(threat)
        return high_risk

    def _load_threat_patterns(self) -> Dict:
        """Load common threat patterns for each STRIDE category"""
        return {
            "Spoofing": [
                "Session hijacking",
                "Credential theft",
                "IP spoofing",
                "Cookie manipulation"
            ],
            "Tampering": [
                "SQL injection",
                "XSS attacks",
                "Data manipulation",
                "Configuration tampering"
            ],
            "Repudiation": [
                "Log deletion",
                "Action denial",
                "Timestamp manipulation"
            ],
            "InformationDisclosure": [
                "Data leakage",
                "Eavesdropping",
                "Memory dumps",
                "Error message disclosure"
            ],
            "DenialOfService": [
                "DDoS attacks",
                "Resource exhaustion",
                "Algorithmic complexity attacks"
            ],
            "ElevationOfPrivilege": [
                "Privilege escalation",
                "Authorization bypass",
                "Token manipulation"
            ]
        }

Example 2: Threat Model API Client (TypeScript)

TypeScript - Threat Model API Client
import axios, { AxiosInstance } from 'axios';

interface ThreatModel {
    threatModelId: string;
    metadata: {
        applicationName: string;
        version: string;
        createdDate: string;
        lastUpdated: string;
        methodology: string;
    };
    threats: Threat[];
}

interface Threat {
    threatId: string;
    title: string;
    category: string;
    description: string;
    riskRating: RiskRating;
    mitigations: Mitigation[];
    status: string;
}

interface RiskRating {
    likelihood: 'Low' | 'Medium' | 'High' | 'Critical';
    impact: 'Low' | 'Medium' | 'High' | 'Critical';
    overallRisk: 'Low' | 'Medium' | 'High' | 'Critical';
}

interface Mitigation {
    mitigationId: string;
    strategy: string;
    implementation: string;
    status: 'Planned' | 'InProgress' | 'Implemented' | 'Validated';
    effectiveness: 'Low' | 'Medium' | 'High';
}

class ThreatModelClient {
    private client: AxiosInstance;

    constructor(baseURL: string, apiKey: string) {
        this.client = axios.create({
            baseURL,
            headers: {
                'Authorization': `Bearer ${apiKey}`,
                'Content-Type': 'application/json'
            }
        });
    }

    async createThreatModel(data: Partial): Promise {
        const response = await this.client.post('/api/v1/threat-models', data);
        return response.data;
    }

    async getThreatModel(modelId: string): Promise {
        const response = await this.client.get(`/api/v1/threat-models/${modelId}`);
        return response.data;
    }

    async updateThreatModel(modelId: string, data: Partial): Promise {
        const response = await this.client.put(`/api/v1/threat-models/${modelId}`, data);
        return response.data;
    }

    async addThreat(modelId: string, threat: Partial): Promise {
        const response = await this.client.post(
            `/api/v1/threat-models/${modelId}/threats`,
            threat
        );
        return response.data;
    }

    async addMitigation(
        modelId: string,
        threatId: string,
        mitigation: Partial
    ): Promise {
        const response = await this.client.post(
            `/api/v1/threat-models/${modelId}/threats/${threatId}/mitigations`,
            mitigation
        );
        return response.data;
    }

    async getAnalysis(modelId: string): Promise {
        const response = await this.client.get(`/api/v1/threat-models/${modelId}/analysis`);
        return response.data;
    }

    async validateModel(modelId: string): Promise<{ isValid: boolean; issues: string[] }> {
        const response = await this.client.post(`/api/v1/threat-models/${modelId}/validate`);
        return response.data;
    }

    async searchThreatModels(criteria: {
        applicationName?: string;
        riskLevel?: string;
        owner?: string;
    }): Promise {
        const response = await this.client.get('/api/v1/threat-models/search', {
            params: criteria
        });
        return response.data;
    }
}

export default ThreatModelClient;

Example 3: Attack Surface Analysis (Java)

Java - Attack Surface Analyzer
package com.wia.security.threatmodeling;

import java.util.*;
import java.util.stream.Collectors;

public class AttackSurfaceAnalyzer {

    private Map> attackSurface;

    public AttackSurfaceAnalyzer() {
        this.attackSurface = new HashMap<>();
    }

    public void analyzeComponent(String componentName, List exposedInterfaces) {
        List vectors = new ArrayList<>();

        for (String interfaceType : exposedInterfaces) {
            vectors.addAll(identifyAttackVectors(interfaceType));
        }

        attackSurface.put(componentName, vectors);
    }

    private List identifyAttackVectors(String interfaceType) {
        List vectors = new ArrayList<>();

        switch (interfaceType.toLowerCase()) {
            case "http":
            case "https":
                vectors.add(new AttackVector("HTTP", "SQL Injection", "High"));
                vectors.add(new AttackVector("HTTP", "XSS", "High"));
                vectors.add(new AttackVector("HTTP", "CSRF", "Medium"));
                vectors.add(new AttackVector("HTTP", "Authentication Bypass", "Critical"));
                break;

            case "api":
                vectors.add(new AttackVector("API", "Broken Authentication", "Critical"));
                vectors.add(new AttackVector("API", "Excessive Data Exposure", "High"));
                vectors.add(new AttackVector("API", "Rate Limiting Bypass", "Medium"));
                vectors.add(new AttackVector("API", "Mass Assignment", "High"));
                break;

            case "database":
                vectors.add(new AttackVector("Database", "SQL Injection", "Critical"));
                vectors.add(new AttackVector("Database", "Privilege Escalation", "High"));
                vectors.add(new AttackVector("Database", "Data Leakage", "High"));
                break;

            case "file_upload":
                vectors.add(new AttackVector("Upload", "Malware Upload", "Critical"));
                vectors.add(new AttackVector("Upload", "Path Traversal", "High"));
                vectors.add(new AttackVector("Upload", "File Type Bypass", "High"));
                break;
        }

        return vectors;
    }

    public Map getRiskSummary() {
        Map summary = new HashMap<>();
        summary.put("Critical", 0);
        summary.put("High", 0);
        summary.put("Medium", 0);
        summary.put("Low", 0);

        for (List vectors : attackSurface.values()) {
            for (AttackVector vector : vectors) {
                summary.put(vector.getSeverity(),
                    summary.get(vector.getSeverity()) + 1);
            }
        }

        return summary;
    }

    public List getCriticalVectors() {
        return attackSurface.values().stream()
            .flatMap(List::stream)
            .filter(v -> "Critical".equals(v.getSeverity()))
            .collect(Collectors.toList());
    }

    public static class AttackVector {
        private String surface;
        private String threatType;
        private String severity;

        public AttackVector(String surface, String threatType, String severity) {
            this.surface = surface;
            this.threatType = threatType;
            this.severity = severity;
        }

        public String getSurface() { return surface; }
        public String getThreatType() { return threatType; }
        public String getSeverity() { return severity; }

        @Override
        public String toString() {
            return String.format("[%s] %s on %s", severity, threatType, surface);
        }
    }
}

2.6 WIA Threat Modeling Certification

Organizations must demonstrate comprehensive threat modeling capabilities to achieve WIA Security Standard certification:

Certification Requirements

  • Threat modeling process documented and integrated into SDLC
  • Threat modeling methodology selected (STRIDE, PASTA, or equivalent)
  • Threat models created for all critical and high-risk applications
  • Data flow diagrams (DFD) documented for all applications
  • Trust boundaries identified and documented in threat models
  • All STRIDE categories analyzed for each application
  • Risk ratings assigned using standardized likelihood/impact matrix
  • Minimum 10 threats identified per critical application
  • Mitigation strategies documented for all high and critical threats
  • Threat models reviewed and approved by security team
  • Threat models updated quarterly or after significant changes
  • Attack surface analysis performed for all external-facing systems
  • Third-party dependencies assessed for security risks
  • Threat intelligence integrated into threat modeling process
  • Automated threat detection tools integrated where applicable
  • Threat model validation performed before production deployment
  • Security requirements derived from threat model findings
  • Penetration testing scoped based on threat model results
  • Residual risks documented and accepted by business stakeholders
  • Threat modeling training provided to development teams
  • Threat model repository maintained with version control
  • Metrics tracked for threat model coverage and effectiveness
  • Annual threat modeling process review and improvement
  • Threat models integrated with incident response planning
Best Practice: Conduct threat modeling sessions early in the design phase, not just before deployment. Early identification of threats reduces remediation costs by up to 90%.
