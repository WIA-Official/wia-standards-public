Chapter 6: Incident Response

6.1 Introduction to Incident Response

Incident response is the systematic approach to preparing for, detecting, containing, and recovering from security incidents. The WIA Security Standard mandates a comprehensive incident response capability that minimizes damage, reduces recovery time, and prevents future incidents through lessons learned.

Core Principle: It's not a question of if a security incident will occur, but when. Organizations must be prepared to respond quickly and effectively to minimize impact and restore normal operations.

NIST Incident Response Lifecycle

Phase Objectives Key Activities Deliverables
1. Preparation Establish IR capability Build IR team, Tools, Playbooks, Training IR plan, Team roster, Tooling
2. Detection & Analysis Identify security incidents Monitoring, Alert triage, Investigation Incident classification, Scope
3. Containment Limit incident spread Isolate systems, Preserve evidence Containment strategy
4. Eradication Remove threat from environment Remove malware, Close vulnerabilities Clean systems
5. Recovery Restore normal operations System restoration, Validation, Monitoring Restored services
6. Post-Incident Learn and improve Lessons learned, Report, Process improvement Incident report, Recommendations

6.2 Incident Classification and Severity

Proper incident classification enables appropriate response and resource allocation based on severity and impact.

Incident Severity Levels

Severity Impact Response Time Examples
Critical (P1) Severe business disruption, Data breach Immediate (15 min) Ransomware, Major data breach, Complete outage
High (P2) Significant impact, Limited data exposure 1 hour Malware outbreak, Targeted attack, Service degradation
Medium (P3) Moderate impact, Potential risk 4 hours Suspicious activity, Policy violation, Minor compromise
Low (P4) Minimal impact, Informational 24 hours Scan attempts, False positives, Reconnaissance

Example 1: Incident Response Orchestration (Python)

Python - Incident Response Orchestrator
from datetime import datetime
from enum import Enum
from typing import List, Dict, Optional
import uuid

class IncidentSeverity(Enum):
    CRITICAL = "critical"
    HIGH = "high"
    MEDIUM = "medium"
    LOW = "low"

class IncidentStatus(Enum):
    NEW = "new"
    INVESTIGATING = "investigating"
    CONTAINED = "contained"
    ERADICATED = "eradicated"
    RECOVERED = "recovered"
    CLOSED = "closed"

class IncidentResponseOrchestrator:
    """
    WIA Security Standard Incident Response Orchestration
    Coordinates incident response activities and automation
    """

    def __init__(self):
        self.incidents = {}
        self.playbooks = self._load_playbooks()
        self.notification_service = NotificationService()
        self.siem = SIEMIntegration()

    def create_incident(self, incident_data: Dict) -> str:
        """Create and initialize new security incident"""
        incident_id = str(uuid.uuid4())

        incident = {
            "incidentId": incident_id,
            "metadata": {
                "title": incident_data["title"],
                "description": incident_data["description"],
                "detectedAt": datetime.utcnow().isoformat(),
                "reportedBy": incident_data["reportedBy"],
                "status": IncidentStatus.NEW.value,
                "severity": incident_data["severity"],
                "category": incident_data["category"]
            },
            "affectedSystems": incident_data.get("affectedSystems", []),
            "indicators": incident_data.get("indicators", {}),
            "timeline": [{
                "timestamp": datetime.utcnow().isoformat(),
                "event": "Incident created",
                "phase": "detection",
                "actor": incident_data["reportedBy"],
                "details": "Initial incident report"
            }],
            "response": {
                "team": [],
                "containmentActions": [],
                "eradicationActions": [],
                "recoveryActions": []
            }
        }

        self.incidents[incident_id] = incident

        # Trigger automated response based on severity
        self._initiate_response(incident)

        return incident_id

    def _initiate_response(self, incident: Dict):
        """Initiate automated incident response"""
        severity = incident["metadata"]["severity"]
        category = incident["metadata"]["category"]

        # Assemble response team
        team = self._assemble_team(severity)
        incident["response"]["team"] = team

        # Notify team members
        self.notification_service.notify_team(
            incident["incidentId"],
            team,
            severity
        )

        # Execute playbook if available
        playbook = self.playbooks.get(category)
        if playbook:
            self._execute_playbook(incident, playbook)

        # Update SIEM
        self.siem.create_case(incident)

    def _assemble_team(self, severity: str) -> List[Dict]:
        """Assemble incident response team based on severity"""
        team = [
            {"name": "SOC Analyst", "role": "lead", "contactInfo": "soc@company.com"}
        ]

        if severity in ["critical", "high"]:
            team.extend([
                {"name": "Security Manager", "role": "manager", "contactInfo": "secmgr@company.com"},
                {"name": "Legal Team", "role": "legal", "contactInfo": "legal@company.com"}
            ])

        if severity == "critical":
            team.extend([
                {"name": "CISO", "role": "executive", "contactInfo": "ciso@company.com"},
                {"name": "PR Team", "role": "communications", "contactInfo": "pr@company.com"}
            ])

        return team

    def _execute_playbook(self, incident: Dict, playbook: Dict):
        """Execute incident response playbook"""
        for step in playbook["steps"]:
            if step["automated"]:
                self._execute_automated_action(incident, step)
            else:
                self._create_manual_task(incident, step)

    def contain_incident(self, incident_id: str, actions: List[Dict]) -> bool:
        """Execute containment actions"""
        incident = self.incidents.get(incident_id)
        if not incident:
            return False

        for action in actions:
            # Execute containment action
            result = self._execute_containment_action(action)

            # Record action in incident
            incident["response"]["containmentActions"].append({
                "action": action["description"],
                "timestamp": datetime.utcnow().isoformat(),
                "performer": action["performer"],
                "result": result
            })

            # Update timeline
            self._add_timeline_event(
                incident,
                f"Containment: {action['description']}",
                "containment",
                action["performer"]
            )

        # Update status
        incident["metadata"]["status"] = IncidentStatus.CONTAINED.value

        return True

    def _execute_containment_action(self, action: Dict) -> str:
        """Execute specific containment action"""
        action_type = action["type"]

        if action_type == "isolate_host":
            return self._isolate_host(action["target"])
        elif action_type == "block_ip":
            return self._block_ip(action["target"])
        elif action_type == "disable_account":
            return self._disable_account(action["target"])
        elif action_type == "quarantine_file":
            return self._quarantine_file(action["target"])

        return "unknown_action"

    def _isolate_host(self, host_id: str) -> str:
        """Isolate compromised host from network"""
        # Integration with EDR/network controls
        print(f"Isolating host: {host_id}")
        return "success"

    def _block_ip(self, ip_address: str) -> str:
        """Block malicious IP at firewall"""
        print(f"Blocking IP: {ip_address}")
        return "success"

    def _disable_account(self, user_id: str) -> str:
        """Disable compromised user account"""
        print(f"Disabling account: {user_id}")
        return "success"

    def _quarantine_file(self, file_hash: str) -> str:
        """Quarantine malicious file"""
        print(f"Quarantining file: {file_hash}")
        return "success"

    def add_evidence(self, incident_id: str, evidence: Dict) -> bool:
        """Add evidence to incident"""
        incident = self.incidents.get(incident_id)
        if not incident:
            return False

        if "evidence" not in incident:
            incident["evidence"] = []

        evidence_entry = {
            "evidenceId": str(uuid.uuid4()),
            "type": evidence["type"],
            "location": evidence["location"],
            "collectedBy": evidence["collectedBy"],
            "collectedAt": datetime.utcnow().isoformat(),
            "chainOfCustody": [{
                "custodian": evidence["collectedBy"],
                "timestamp": datetime.utcnow().isoformat(),
                "action": "collected"
            }]
        }

        incident["evidence"].append(evidence_entry)
        return True

    def close_incident(self, incident_id: str, final_report: Dict) -> bool:
        """Close incident with final report"""
        incident = self.incidents.get(incident_id)
        if not incident:
            return False

        # Add root cause analysis
        incident["rootCause"] = final_report.get("rootCause", {})

        # Add impact assessment
        incident["impact"] = final_report.get("impact", {})

        # Update status
        incident["metadata"]["status"] = IncidentStatus.CLOSED.value

        # Add final timeline event
        self._add_timeline_event(
            incident,
            "Incident closed",
            "post-incident",
            final_report.get("closedBy", "system")
        )

        # Generate lessons learned
        self._generate_lessons_learned(incident)

        return True

    def _add_timeline_event(self, incident: Dict, event: str,
                           phase: str, actor: str):
        """Add event to incident timeline"""
        incident["timeline"].append({
            "timestamp": datetime.utcnow().isoformat(),
            "event": event,
            "phase": phase,
            "actor": actor,
            "details": ""
        })

    def _generate_lessons_learned(self, incident: Dict):
        """Generate lessons learned report"""
        print(f"Generating lessons learned for incident {incident['incidentId']}")
        # Analysis and recommendations for improvement

    def _load_playbooks(self) -> Dict:
        """Load incident response playbooks"""
        return {
            "malware": {
                "steps": [
                    {"action": "Isolate affected systems", "automated": True},
                    {"action": "Collect malware sample", "automated": False},

Example 2: Automated Containment (Bash Script)

Bash - Automated Host Isolation Script
#!/bin/bash
# WIA Security Standard - Automated Host Isolation
# This script isolates a compromised host from the network

set -euo pipefail

INCIDENT_ID="$1"
HOST_IP="$2"
LOG_FILE="/var/log/incident-response/containment.log"

log() {
    echo "[$(date +'%Y-%m-%d %H:%M:%S')] $*" | tee -a "$LOG_FILE"
}

isolate_host() {
    local host_ip="$1"

    log "Starting host isolation for $host_ip (Incident: $INCIDENT_ID)"

    # Block host at firewall
    log "Blocking host at firewall..."
    sudo iptables -I INPUT -s "$host_ip" -j DROP
    sudo iptables -I OUTPUT -d "$host_ip" -j DROP

    # Disable network interface on compromised host (if accessible)
    log "Attempting to disable network interface on target host..."
    ssh -o StrictHostKeyChecking=no "admin@$host_ip" \
        "sudo ip link set eth0 down" 2>/dev/null || \
        log "Could not SSH to target host - host may already be isolated"

    # Update network ACLs
    log "Updating network ACLs..."
    # Integration with network management system
    curl -X POST "https://network-api/acl/block" \
        -H "Authorization: Bearer ${API_TOKEN}" \
        -d "{\"ip\": \"$host_ip\", \"incident\": \"$INCIDENT_ID\"}"

    # Document containment action
    log "Documenting containment action..."
    curl -X POST "https://ir-api/v1/incidents/$INCIDENT_ID/containment" \
        -H "Authorization: Bearer ${API_TOKEN}" \
        -H "Content-Type: application/json" \
        -d "{
            \"action\": \"Host isolation\",
            \"target\": \"$host_ip\",
            \"timestamp\": \"$(date -u +%Y-%m-%dT%H:%M:%SZ)\",
            \"performer\": \"automated-system\",
            \"result\": \"success\"
        }"

    log "Host isolation completed for $host_ip"
}

# Main execution
isolate_host "$HOST_IP"

Example 3: Incident Metrics Dashboard (JavaScript)

JavaScript - Incident Response Metrics
class IncidentMetrics {
    constructor(apiClient) {
        this.api = apiClient;
    }

    async calculateMTTD(incidentId) {
        // Mean Time To Detect
        const incident = await this.api.getIncident(incidentId);
        const detectedAt = new Date(incident.metadata.detectedAt);
        const occurredAt = new Date(incident.timeline[0].timestamp);

        return (detectedAt - occurredAt) / 1000 / 60; // minutes
    }

    async calculateMTTR(incidentId) {
        // Mean Time To Respond
        const incident = await this.api.getIncident(incidentId);
        const detectedAt = new Date(incident.metadata.detectedAt);

        const firstResponse = incident.timeline.find(
            e => e.phase === 'containment'
        );

        if (!firstResponse) return null;

        const respondedAt = new Date(firstResponse.timestamp);
        return (respondedAt - detectedAt) / 1000 / 60; // minutes
    }

    async calculateMTTC(incidentId) {
        // Mean Time To Contain
        const incident = await this.api.getIncident(incidentId);
        const detectedAt = new Date(incident.metadata.detectedAt);

        const containedEvent = incident.timeline.find(
            e => e.event.includes('Containment')
        );

        if (!containedEvent) return null;

        const containedAt = new Date(containedEvent.timestamp);
        return (containedAt - detectedAt) / 1000 / 60; // minutes
    }

    async getIncidentStatistics(dateRange) {
        const incidents = await this.api.getIncidents(dateRange);

        return {
            total: incidents.length,
            bySeverity: this.groupBy(incidents, 'metadata.severity'),
            byCategory: this.groupBy(incidents, 'metadata.category'),
            byStatus: this.groupBy(incidents, 'metadata.status'),
            avgMTTD: await this.calculateAvgMTTD(incidents),
            avgMTTR: await this.calculateAvgMTTR(incidents),
            avgMTTC: await this.calculateAvgMTTC(incidents)
        };
    }

    groupBy(array, path) {
        return array.reduce((acc, obj) => {
            const key = path.split('.').reduce((o, i) => o[i], obj);
            acc[key] = (acc[key] || 0) + 1;
            return acc;
        }, {});
    }
}

export default IncidentMetrics;

6.6 WIA Incident Response Certification

Organizations must demonstrate comprehensive incident response capabilities to achieve WIA certification:

Certification Requirements

  • Incident response plan documented and approved by executive leadership
  • Incident response team established with defined roles and responsibilities
  • 24/7 security operations center (SOC) or equivalent monitoring capability
  • Incident classification and severity system implemented
  • Response time SLAs defined for each severity level
  • Incident response playbooks created for common attack scenarios
  • SIEM or log management system deployed for detection
  • Automated alerting configured for security events
  • Incident tracking system implemented for case management
  • Forensic tools and capabilities available for investigation
  • Evidence collection and chain of custody procedures documented
  • Containment procedures for different incident types defined
  • Communication templates for incident notifications prepared
  • Legal and regulatory notification procedures established
  • Business continuity and disaster recovery plans integrated with IR
  • Incident response drills and tabletop exercises conducted quarterly
  • Post-incident review process mandatory for all incidents
  • Lessons learned database maintained and reviewed
  • Threat intelligence integration for IOC enrichment
  • Incident metrics tracked (MTTD, MTTR, MTTC)
  • External IR retainer or partnership established
  • Data breach response plan compliant with regulations (GDPR, etc.)
  • Annual IR plan review and update process
  • IR team training completed at least semi-annually
Critical Success Factor: The effectiveness of incident response is directly proportional to preparation. Regular testing and continuous improvement are essential.
