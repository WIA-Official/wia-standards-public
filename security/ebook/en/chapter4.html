Chapter 4: Authentication & Authorization

4.1 Introduction to Identity and Access Management

Authentication and authorization form the foundation of secure access control in modern systems. Authentication verifies the identity of users and systems, while authorization determines what authenticated entities are permitted to do. The WIA Security Standard mandates robust identity management practices that balance security with usability.

Core Principle: Authentication answers "Who are you?" while Authorization answers "What can you do?" Both must work together to enforce the principle of least privilege and protect system resources.

Authentication Methods Comparison

Method Security Level User Experience Implementation Complexity Use Cases
Password Only Low Good Low Not recommended
MFA (TOTP) High Good Medium Standard user authentication
Biometric High Excellent High Mobile apps, Physical access
Hardware Token Very High Medium Medium Privileged access, Finance
Certificate-based Very High Good (after setup) High Service accounts, API access
SSO/SAML High Excellent High Enterprise applications

4.2 Authorization Models

The WIA Security Standard supports multiple authorization models to accommodate different organizational and application requirements.

Authorization Model Comparison

Model Description Complexity Best For Limitations

ABAC Attribute-Based Access Control High Complex, dynamic policies Performance overhead
ReBAC Relationship-Based Access Control Medium Social platforms, Collaboration Complex relationship management
PBAC Policy-Based Access Control High Enterprise-wide governance Requires policy engine

Example 1: OAuth 2.0 Authorization Server (Python/Flask)

Python - OAuth 2.0 Authorization Server
from flask import Flask, request, jsonify, redirect
from authlib.integrations.flask_oauth2 import AuthorizationServer
from authlib.oauth2.rfc6749 import grants
from authlib.jose import jwt
import secrets
from datetime import datetime, timedelta
from werkzeug.security import gen_salt

app = Flask(__name__)
app.config['SECRET_KEY'] = secrets.token_hex(32)

class OAuth2AuthorizationServer:
    """
    WIA Security Standard compliant OAuth 2.0 Authorization Server
    """

    def __init__(self, app):
        self.app = app
        self.server = AuthorizationServer(app)
        self.setup_grants()

    def setup_grants(self):
        """Configure supported OAuth 2.0 grants"""
        # Authorization Code Grant
        self.server.register_grant(AuthorizationCodeGrant)

        # Client Credentials Grant
        self.server.register_grant(ClientCredentialsGrant)

        # Refresh Token Grant
        self.server.register_grant(RefreshTokenGrant)

    def generate_access_token(self, client, user, scope):
        """Generate JWT access token"""
        now = datetime.utcnow()
        payload = {
            'iss': 'https://auth.wia-security.org',
            'sub': user.id,
            'aud': client.client_id,
            'scope': scope,
            'iat': now,
            'exp': now + timedelta(hours=1),
            'client_id': client.client_id
        }

        # Sign token with RS256
        token = jwt.encode(
            header={'alg': 'RS256'},
            payload=payload,
            key=self.load_private_key()
        )

        return token.decode('utf-8')

    def generate_refresh_token(self):
        """Generate secure refresh token"""
        return secrets.token_urlsafe(32)

    def validate_pkce(self, code_challenge, code_verifier, method='S256'):
        """Validate PKCE for authorization code flow"""
        if method == 'S256':
            import hashlib
            import base64

            # Compute SHA256 hash of verifier
            h = hashlib.sha256(code_verifier.encode('utf-8')).digest()
            computed_challenge = base64.urlsafe_b64encode(h).decode('utf-8').rstrip('=')

            return computed_challenge == code_challenge

        return False

    """OAuth 2.0 token endpoint"""
    grant_type = request.form.get('grant_type')

    if grant_type == 'authorization_code':
        code = request.form.get('code')
        client_id = request.form.get('client_id')
        client_secret = request.form.get('client_secret')
        redirect_uri = request.form.get('redirect_uri')
        code_verifier = request.form.get('code_verifier')

        # Validate client credentials
        if not validate_client(client_id, client_secret):
            return jsonify({'error': 'invalid_client'}), 401

        # Retrieve authorization code
        auth_code_data = get_authorization_code(code)

        if not auth_code_data or auth_code_data['expires_at'] < datetime.utcnow():
            return jsonify({'error': 'invalid_grant'}), 400

        # Validate PKCE if present
        if auth_code_data.get('code_challenge'):
            if not validate_pkce(
                auth_code_data['code_challenge'],
                code_verifier,
                'S256'
            ):
                return jsonify({'error': 'invalid_grant'}), 400

        # Generate tokens
        access_token = generate_access_token(
            client_id=client_id,
            user_id=auth_code_data['user_id'],
            scope=auth_code_data['scope']
        )

        refresh_token = generate_refresh_token()

        # Invalidate authorization code (one-time use)
        delete_authorization_code(code)

        return jsonify({
            'access_token': access_token,
            'token_type': 'Bearer',

            'refresh_token': refresh_token,
            'scope': auth_code_data['scope']
        })

@app.route('/oauth/userinfo', methods=['GET'])
def userinfo():
    """OIDC UserInfo endpoint"""
    # Extract and validate access token
    auth_header = request.headers.get('Authorization')

    if not auth_header or not auth_header.startswith('Bearer '):
        return jsonify({'error': 'invalid_token'}), 401

    access_token = auth_header.split(' ')[1]

    # Decode and validate JWT
    try:
        payload = jwt.decode(access_token, load_public_key())
        user_id = payload['sub']

        # Retrieve user information
        user = get_user_by_id(user_id)

        return jsonify({
            'sub': user.id,
            'email': user.email,
            'email_verified': user.email_verified,
            'name': user.name,
            'given_name': user.given_name,
            'family_name': user.family_name,
            'picture': user.picture_url
        })

    except Exception as e:
        return jsonify({'error': 'invalid_token'}), 401

Example 2: RBAC Implementation (Java/Spring)

Java - Role-Based Access Control
package com.wia.security.authz;

import org.springframework.security.access.prepost.PreAuthorize;
import org.springframework.stereotype.Service;
import java.util.*;

@Service
public class RBACAuthorizationService {

    private final RoleRepository roleRepository;
    private final UserRepository userRepository;
    private final PermissionRepository permissionRepository;

    public RBACAuthorizationService(
        RoleRepository roleRepository,
        UserRepository userRepository,
        PermissionRepository permissionRepository
    ) {
        this.roleRepository = roleRepository;
        this.userRepository = userRepository;
        this.permissionRepository = permissionRepository;
    }

    /**
     * Evaluate if user has required permission
     */
    public boolean hasPermission(String userId, String resource, String action) {
        User user = userRepository.findById(userId)
            .orElseThrow(() -> new UserNotFoundException(userId));

        Set userPermissions = getUserPermissions(user);

        return userPermissions.stream()
            .anyMatch(p -> p.getResource().equals(resource) &&
                          p.getAction().equals(action));
    }

    /**
     * Get all permissions for a user (including inherited from roles)
     */
    public Set getUserPermissions(User user) {
        Set permissions = new HashSet<>();

        // Direct permissions
        permissions.addAll(user.getPermissions());

        // Role-based permissions
        for (Role role : user.getRoles()) {
            permissions.addAll(getRolePermissions(role));
        }

        return permissions;
    }

    /**
     * Get all permissions for a role (including inherited roles)
     */
    private Set getRolePermissions(Role role) {
        Set permissions = new HashSet<>();
        permissions.addAll(role.getPermissions());

        // Handle role hierarchy
        for (Role parentRole : role.getParentRoles()) {
            permissions.addAll(getRolePermissions(parentRole));
        }

        return permissions;
    }

    /**
     * Assign role to user
     */
    @PreAuthorize("hasAuthority('MANAGE_USERS')")
    public void assignRole(String userId, String roleId) {
        User user = userRepository.findById(userId)
            .orElseThrow(() -> new UserNotFoundException(userId));

        Role role = roleRepository.findById(roleId)
            .orElseThrow(() -> new RoleNotFoundException(roleId));

        user.getRoles().add(role);
        userRepository.save(user);

        auditLog("ROLE_ASSIGNED", userId, roleId);
    }

    /**
     * Create new role with permissions
     */
    @PreAuthorize("hasAuthority('MANAGE_ROLES')")
    public Role createRole(String roleName, Set permissionIds) {
        Role role = new Role();
        role.setName(roleName);
        role.setCreatedAt(new Date());

        Set permissions = new HashSet<>();
        for (String permId : permissionIds) {
            Permission perm = permissionRepository.findById(permId)
                .orElseThrow(() -> new PermissionNotFoundException(permId));
            permissions.add(perm);
        }

        role.setPermissions(permissions);
        return roleRepository.save(role);
    }

    /**
     * Evaluate complex authorization policy
     */
    public AuthorizationDecision evaluate(AuthorizationRequest request) {
        // Check if user exists and is active
        User user = userRepository.findById(request.getUserId())
            .orElseThrow(() -> new UserNotFoundException(request.getUserId()));

        if (!user.isActive()) {
            return AuthorizationDecision.deny("User account is inactive");
        }

        // Check permissions
        boolean hasPermission = hasPermission(
            request.getUserId(),
            request.getResource(),
            request.getAction()
        );

        if (!hasPermission) {
            return AuthorizationDecision.deny("Insufficient permissions");
        }

        // Additional context-based checks
        if (request.requiresMFA() && !user.isMFAEnabled()) {
            return AuthorizationDecision.deny("MFA required but not enabled");
        }

        return AuthorizationDecision.allow();
    }

    private void auditLog(String action, String userId, String roleId) {
        // Log to audit trail
        System.out.println(
            String.format("[AUDIT] %s - User: %s, Role: %s", action, userId, roleId)
        );
    }
}

Example 3: Session Management (Node.js)

JavaScript/Node.js - Secure Session Management
const session = require('express-session');
const RedisStore = require('connect-redis')(session);
const redis = require('redis');
const crypto = require('crypto');

class SessionManager {
    constructor(config) {
        this.redisClient = redis.createClient(config.redis);
        this.sessionConfig = this.buildSessionConfig(config);
    }

    buildSessionConfig(config) {
        return {
            store: new RedisStore({ client: this.redisClient }),

            // Session ID generation
            genid: (req) => {
                return crypto.randomBytes(32).toString('hex');
            },

            // Session name (don't use default 'connect.sid')
            name: 'wia.sid',

            // Secret for signing session ID cookie
            secret: config.sessionSecret,

            // Resave and saveUninitialized
            resave: false,
            saveUninitialized: false,

            // Cookie configuration
            cookie: {
                secure: true,        // HTTPS only
                httpOnly: true,      // Not accessible via JavaScript
                sameSite: 'strict',  // CSRF protection
                maxAge: 3600000,     // 1 hour
                domain: config.cookieDomain
            },

            // Rolling session
            rolling: true
        };
    }

    getMiddleware() {
        return session(this.sessionConfig);
    }

    /**
     * Track concurrent sessions per user
     */
    async trackSession(userId, sessionId) {
        const key = `user_sessions:${userId}`;

        // Add session to user's active sessions
        await this.redisClient.sadd(key, sessionId);

        // Set expiration
        await this.redisClient.expire(key, 86400); // 24 hours
    }

    /**
     * Terminate all sessions for a user
     */
    async terminateAllUserSessions(userId) {
        const key = `user_sessions:${userId}`;
        const sessions = await this.redisClient.smembers(key);

        for (const sessionId of sessions) {
            await this.redisClient.del(`sess:${sessionId}`);
        }

        await this.redisClient.del(key);
    }

    /**
     * Session validation middleware
     */
    validateSession = async (req, res, next) => {
        if (!req.session || !req.session.userId) {
            return res.status(401).json({
                error: 'Session invalid or expired'
            });
        }

        // Check if session exists in Redis
        const sessionExists = await this.redisClient.exists(
            `sess:${req.sessionID}`
        );

        if (!sessionExists) {
            return res.status(401).json({
                error: 'Session not found'
            });
        }

        // Verify session hasn't been compromised
        if (req.session.ipAddress !== req.ip) {
            // IP address changed - potential session hijacking
            req.session.destroy();
            return res.status(401).json({
                error: 'Session security violation'
            });
        }

        next();
    };
}

module.exports = SessionManager;

4.6 WIA Authentication & Authorization Certification

Organizations must implement comprehensive identity and access management to achieve WIA certification:

Certification Requirements

  • Multi-factor authentication (MFA) enforced for all user accounts
  • OAuth 2.0 with PKCE implemented for web and mobile applications
  • OpenID Connect (OIDC) used for user authentication
  • Single Sign-On (SSO) implemented for enterprise applications
  • Role-Based Access Control (RBAC) or ABAC implemented organization-wide
  • Principle of least privilege enforced in permission assignments
  • Password policy requiring minimum 12 characters with complexity
  • Password hashing using Argon2id, bcrypt, or PBKDF2
  • Account lockout after 5 failed authentication attempts
  • Session timeout configured for maximum 60 minutes inactivity
  • Secure session management with HttpOnly and Secure flags
  • API authentication using JWT or OAuth 2.0 client credentials
  • Service-to-service authentication using mutual TLS or certificates
  • Just-in-Time (JIT) access provisioning for privileged operations
  • Privileged access management (PAM) for administrative accounts
  • Access reviews performed quarterly for all user permissions
  • Automated deprovisioning when users leave organization
  • Identity lifecycle management integrated with HR systems
  • Federated identity support for external partners (SAML/OIDC)
  • Biometric authentication option available for high-security scenarios
  • Hardware token support for privileged users
  • Authorization audit logging for all access decisions
  • Anomaly detection for unusual authentication patterns
  • Regular penetration testing of authentication mechanisms
Security Best Practice: Never trust, always verify. Implement continuous authentication and authorization validation, not just at initial login.
