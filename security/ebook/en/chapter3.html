Chapter 3: Encryption Standards

3.1 Introduction to Cryptography

Encryption is the cornerstone of modern information security, providing confidentiality, integrity, and authenticity for data at rest and in transit. The WIA Security Standard mandates strong encryption algorithms and proper key management practices to protect sensitive information from unauthorized access.

Core Principle: Encryption should be the default state for all sensitive data. Assume that perimeter defenses will be breached and ensure that encrypted data remains protected even when accessed by unauthorized parties.

Encryption Types and Use Cases

Type Algorithm Examples Use Cases Key Size Performance
Symmetric AES, ChaCha20 Data at rest, Bulk encryption 128, 192, 256 bits Fast
Asymmetric RSA, ECC, Ed25519 Key exchange, Digital signatures 2048-4096 bits (RSA) Slower
Hash Functions SHA-256, SHA-3, BLAKE2 Data integrity, Password storage 256-512 bits output Fast
Key Derivation PBKDF2, Argon2, scrypt Password hashing, Key generation Variable Intentionally slow
Authenticated AES-GCM, ChaCha20-Poly1305 Encryption + Authentication 256 bits + tag Fast

The WIA Security Standard specifies minimum encryption requirements for different data classifications and transmission scenarios.

Encryption Standards by Data Classification

3.4 Encryption API Endpoints

The WIA Security Standard provides comprehensive APIs for encryption operations and key management.

POST /api/v1/encryption/encrypt

Encrypt data using specified algorithm and key

POST /api/v1/encryption/decrypt

Decrypt encrypted data with proper authorization

POST /api/v1/encryption/keys/generate

Generate new encryption key with specified parameters

GET /api/v1/encryption/keys/{keyId}

Retrieve key metadata (not the key material)

POST /api/v1/encryption/keys/{keyId}/rotate

Rotate encryption key and re-encrypt affected data

DELETE /api/v1/encryption/keys/{keyId}

Schedule key for deletion (with grace period)

POST /api/v1/encryption/hash

Generate cryptographic hash of data

POST /api/v1/encryption/sign

Create digital signature for data integrity

POST /api/v1/encryption/verify

Verify digital signature authenticity

GET /api/v1/encryption/compliance/validate

Validate encryption configuration against compliance requirements

GET /api/v1/encryption/audit-trail

Retrieve encryption operation audit logs

POST /api/v1/encryption/tls/certificate/renew

Renew TLS certificate before expiration

3.5 Implementation Examples

Example 1: AES-256-GCM Encryption (Python)

Python - AES-GCM Encryption Service
from cryptography.hazmat.primitives.ciphers.aead import AESGCM
from cryptography.hazmat.primitives import hashes
from cryptography.hazmat.primitives.kdf.pbkdf2 import PBKDF2HMAC
import os
import base64
import json
from typing import Tuple, Dict

class EncryptionService:
    """
    WIA Security Standard compliant encryption service using AES-256-GCM
    """

    def __init__(self, kms_client=None):
        self.kms_client = kms_client
        self.algorithm = "AES-256-GCM"

    def encrypt_data(self, plaintext: bytes, key_id: str,
                     associated_data: bytes = None) -> Dict[str, str]:
        """
        Encrypt data using AES-256-GCM with authenticated encryption

        Args:
            plaintext: Data to encrypt
            key_id: KMS key identifier
            associated_data: Additional authenticated data (AAD)

        Returns:
            Dictionary containing ciphertext, nonce, tag, and metadata
        """
        # Get encryption key from KMS
        key = self._get_key_from_kms(key_id)

        # Generate random nonce (96 bits for GCM)
        nonce = os.urandom(12)

        # Initialize AES-GCM cipher
        aesgcm = AESGCM(key)

        # Encrypt and authenticate
        ciphertext = aesgcm.encrypt(
            nonce=nonce,
            data=plaintext,
            associated_data=associated_data
        )

        # Return encrypted data with metadata
        return {
            "algorithm": self.algorithm,
            "keyId": key_id,
            "ciphertext": base64.b64encode(ciphertext).decode('utf-8'),
            "nonce": base64.b64encode(nonce).decode('utf-8'),
            "version": "1.0"
        }

    def decrypt_data(self, encrypted_data: Dict[str, str],
                     associated_data: bytes = None) -> bytes:
        """
        Decrypt AES-256-GCM encrypted data

        Args:
            encrypted_data: Dictionary containing encrypted data and metadata
            associated_data: Additional authenticated data (must match encryption)

        Returns:
            Decrypted plaintext
        """
        # Get decryption key from KMS
        key = self._get_key_from_kms(encrypted_data["keyId"])

        # Decode ciphertext and nonce
        ciphertext = base64.b64decode(encrypted_data["ciphertext"])
        nonce = base64.b64decode(encrypted_data["nonce"])

        # Initialize AES-GCM cipher
        aesgcm = AESGCM(key)

        # Decrypt and verify authentication
        try:
            plaintext = aesgcm.decrypt(
                nonce=nonce,
                data=ciphertext,
                associated_data=associated_data
            )
            return plaintext
        except Exception as e:
            raise ValueError(f"Decryption failed: Authentication verification failed")

    def encrypt_field(self, data: Dict, field_name: str, key_id: str) -> Dict:
        """
        Encrypt a specific field in a dictionary (for database encryption)
        """
        if field_name not in data:
            raise ValueError(f"Field {field_name} not found in data")

        # Convert field value to bytes
        field_value = str(data[field_name]).encode('utf-8')

        # Encrypt the field
        encrypted = self.encrypt_data(
            plaintext=field_value,
            key_id=key_id,
            associated_data=field_name.encode('utf-8')
        )

        # Replace field with encrypted version
        data[field_name] = encrypted
        data[f"{field_name}_encrypted"] = True

        return data

    def rotate_encryption_key(self, old_key_id: str, new_key_id: str,
                              encrypted_data: Dict[str, str]) -> Dict[str, str]:
        """
        Rotate encryption key by decrypting with old key and re-encrypting with new key
        """
        # Decrypt with old key
        plaintext = self.decrypt_data(encrypted_data)

        # Re-encrypt with new key
        re_encrypted = self.encrypt_data(plaintext, new_key_id)

        return re_encrypted

    def _get_key_from_kms(self, key_id: str) -> bytes:
        """
        Retrieve encryption key from KMS
        In production, this would call actual KMS service
        """
        if self.kms_client:
            return self.kms_client.get_key(key_id)
        else:
            # For demo purposes - in production, NEVER hardcode keys
            # This would retrieve from AWS KMS, Azure Key Vault, etc.
            return os.urandom(32)  # 256-bit key

    def derive_key_from_password(self, password: str, salt: bytes = None) -> Tuple[bytes, bytes]:
        """
        Derive encryption key from password using PBKDF2
        """
        if salt is None:
            salt = os.urandom(16)

        kdf = PBKDF2HMAC(
            algorithm=hashes.SHA256(),
            length=32,  # 256 bits
            salt=salt,
            iterations=480000,  # OWASP recommended minimum
        )

        key = kdf.derive(password.encode('utf-8'))
        return key, salt

Example 2: TLS Configuration (Node.js)

JavaScript/Node.js - TLS 1.3 Server Configuration
const https = require('https');
const fs = require('fs');
const crypto = require('crypto');

class SecureTLSServer {
    constructor(config) {
        this.config = config;
        this.tlsOptions = this.buildTLSOptions();
    }

    buildTLSOptions() {
        // WIA Security Standard compliant TLS configuration
        const options = {
            // TLS version enforcement
            minVersion: 'TLSv1.3',
            maxVersion: 'TLSv1.3',

            // Certificate configuration
            key: fs.readFileSync(this.config.keyPath),
            cert: fs.readFileSync(this.config.certPath),
            ca: this.config.caPath ? fs.readFileSync(this.config.caPath) : undefined,

            // Cipher suite configuration (TLS 1.3)
            ciphers: [
                'TLS_AES_256_GCM_SHA384',
                'TLS_CHACHA20_POLY1305_SHA256',
                'TLS_AES_128_GCM_SHA256'
            ].join(':'),

            // Security options
            honorCipherOrder: true,
            requestCert: this.config.mutualTLS || false,
            rejectUnauthorized: true,

            // Disable insecure features
            secureOptions:
                crypto.constants.SSL_OP_NO_SSLv2 |
                crypto.constants.SSL_OP_NO_SSLv3 |
                crypto.constants.SSL_OP_NO_TLSv1 |
                crypto.constants.SSL_OP_NO_TLSv1_1,

            // ALPN protocols
            ALPNProtocols: ['h2', 'http/1.1'],

            // Session management
            sessionTimeout: 300, // 5 minutes
            sessionIdContext: 'wia-security-standard'
        };

        return options;
    }

    createServer(app) {
        const server = https.createServer(this.tlsOptions, app);

        // TLS connection event logging
        server.on('secureConnection', (tlsSocket) => {
            this.logTLSConnection(tlsSocket);
        });

        // Certificate validation
        server.on('tlsClientError', (error, tlsSocket) => {
            console.error('TLS Client Error:', error.message);
            this.logSecurityEvent('TLS_ERROR', {
                error: error.message,
                remoteAddress: tlsSocket.remoteAddress
            });
        });

        return server;
    }

    logTLSConnection(tlsSocket) {
        const connectionInfo = {
            protocol: tlsSocket.getProtocol(),
            cipher: tlsSocket.getCipher(),
            peerCertificate: tlsSocket.getPeerCertificate(),
            authorized: tlsSocket.authorized,
            remoteAddress: tlsSocket.remoteAddress,
            timestamp: new Date().toISOString()
        };

        // Verify TLS 1.3 is being used
        if (connectionInfo.protocol !== 'TLSv1.3') {
            console.warn('Non-TLS 1.3 connection attempted:', connectionInfo);
            this.logSecurityEvent('WEAK_TLS', connectionInfo);
        }

        console.log('Secure TLS connection established:', connectionInfo);
    }

    validateCertificate(cert) {
        // Check certificate expiration
        const now = new Date();
        const validTo = new Date(cert.valid_to);
        const daysUntilExpiration = (validTo - now) / (1000 * 60 * 60 * 24);

        if (daysUntilExpiration < 30) {
            console.warn(`Certificate expires in ${daysUntilExpiration} days`);
            this.logSecurityEvent('CERT_EXPIRING', {
                subject: cert.subject,
                daysUntilExpiration
            });
        }

        // Verify certificate chain
        if (!cert.issuerCertificate) {
            throw new Error('Certificate chain validation failed');
        }

        return true;
    }

    logSecurityEvent(eventType, details) {
        // Log to SIEM or security monitoring system
        const event = {
            eventType,
            timestamp: new Date().toISOString(),
            details,
            severity: this.getSeverity(eventType)
        };

        console.log('[SECURITY EVENT]', JSON.stringify(event));
    }

    getSeverity(eventType) {
        const severityMap = {
            'TLS_ERROR': 'HIGH',
            'WEAK_TLS': 'CRITICAL',
            'CERT_EXPIRING': 'WARNING',
            'CERT_EXPIRED': 'CRITICAL'
        };

        return severityMap[eventType] || 'INFO';
    }
}

// Usage example
const server = new SecureTLSServer({
    keyPath: '/path/to/private-key.pem',
    certPath: '/path/to/certificate.pem',
    caPath: '/path/to/ca-bundle.pem',
    mutualTLS: true
});

module.exports = SecureTLSServer;

Example 3: Password Hashing with Argon2 (Go)

Go - Argon2id Password Hashing
package encryption

import (
    "crypto/rand"
    "crypto/subtle"
    "encoding/base64"
    "errors"
    "fmt"
    "strings"

    "golang.org/x/crypto/argon2"
)

// PasswordHasher implements WIA Security Standard password hashing
type PasswordHasher struct {
    memory      uint32
    iterations  uint32
    parallelism uint8
    saltLength  uint32
    keyLength   uint32
}

// NewPasswordHasher creates a hasher with WIA recommended parameters
func NewPasswordHasher() *PasswordHasher {
    return &PasswordHasher{
        memory:      64 * 1024, // 64 MB
        iterations:  3,
        parallelism: 2,
        saltLength:  16,
        keyLength:   32, // 256 bits
    }
}

// HashPassword creates an Argon2id hash of the password
func (ph *PasswordHasher) HashPassword(password string) (string, error) {
    // Generate random salt
    salt := make([]byte, ph.saltLength)
    if _, err := rand.Read(salt); err != nil {
        return "", err
    }

    // Generate hash
    hash := argon2.IDKey(
        []byte(password),
        salt,
        ph.iterations,
        ph.memory,
        ph.parallelism,
        ph.keyLength,
    )

    // Encode hash with parameters for verification
    encodedHash := ph.encodeHash(hash, salt)
    return encodedHash, nil
}

// VerifyPassword checks if password matches the hash
func (ph *PasswordHasher) VerifyPassword(password, encodedHash string) (bool, error) {
    // Decode hash and extract parameters
    hash, salt, err := ph.decodeHash(encodedHash)
    if err != nil {
        return false, err
    }

    // Generate hash from provided password
    computedHash := argon2.IDKey(
        []byte(password),
        salt,
        ph.iterations,
        ph.memory,
        ph.parallelism,
        ph.keyLength,
    )

    // Constant-time comparison to prevent timing attacks
    if subtle.ConstantTimeCompare(hash, computedHash) == 1 {
        return true, nil
    }

    return false, nil
}

// encodeHash creates encoded string with format:
// $argon2id$v=19$m=65536,t=3,p=2$salt$hash
func (ph *PasswordHasher) encodeHash(hash, salt []byte) string {
    b64Salt := base64.RawStdEncoding.EncodeToString(salt)
    b64Hash := base64.RawStdEncoding.EncodeToString(hash)

    return fmt.Sprintf(
        "$argon2id$v=%d$m=%d,t=%d,p=%d$%s$%s",
        argon2.Version,
        ph.memory,
        ph.iterations,
        ph.parallelism,
        b64Salt,
        b64Hash,
    )
}

// decodeHash extracts hash and salt from encoded string
func (ph *PasswordHasher) decodeHash(encodedHash string) (hash, salt []byte, err error) {
    parts := strings.Split(encodedHash, "$")
    if len(parts) != 6 {
        return nil, nil, errors.New("invalid hash format")
    }

    if parts[1] != "argon2id" {
        return nil, nil, errors.New("incompatible algorithm")
    }

    salt, err = base64.RawStdEncoding.DecodeString(parts[4])
    if err != nil {
        return nil, nil, err
    }

    hash, err = base64.RawStdEncoding.DecodeString(parts[5])
    if err != nil {
        return nil, nil, err
    }

    return hash, salt, nil
}

3.6 WIA Encryption Standards Certification

Organizations must implement and maintain robust encryption practices across all systems to achieve certification:

Certification Requirements

  • AES-256 encryption implemented for all sensitive data at rest
  • TLS 1.3 enforced for all data in transit (TLS 1.2 minimum acceptable)
  • Key management system (KMS) deployed for centralized key management
  • Hardware Security Module (HSM) used for critical key storage
  • Encryption key rotation automated and performed quarterly minimum
  • Weak cipher suites disabled (RC4, DES, 3DES, MD5)
  • Perfect Forward Secrecy (PFS) enabled for all TLS connections
  • Certificate management automated with renewal 30 days before expiration
  • Mutual TLS (mTLS) implemented for service-to-service communication
  • Password hashing using Argon2id, bcrypt, or PBKDF2 with 100k+ iterations
  • Salt generation using cryptographically secure random number generator
  • Encryption keys never stored in source code or configuration files
  • Key backup and recovery procedures documented and tested
  • Encryption audit logging enabled for all key operations
  • Field-level encryption implemented for PII and sensitive data
  • Database encryption at rest enabled (TDE or equivalent)
  • Encryption performance impact monitored and optimized
  • FIPS 140-2 or 140-3 validated cryptographic modules used
  • Digital signature verification implemented for critical operations
  • Secure key destruction procedures when keys are rotated or retired
  • Encryption compliance validated against regulatory requirements
  • Annual cryptographic algorithm review and update process
  • Zero-knowledge encryption for highly sensitive user data
  • Post-quantum cryptography readiness assessment completed
Critical Warning: Never implement custom cryptography. Always use well-tested, industry-standard cryptographic libraries and follow established best practices.
