Chapter 1: Security Fundamentals

1.1 Introduction to Information Security

Information security is the practice of protecting information by mitigating information risks. It is part of information risk management and involves preventing or reducing the probability of unauthorized access, use, disclosure, disruption, deletion, corruption, modification, inspection, recording, or devaluation of information.

Key Principle: Security is not a product, but a process. It requires continuous monitoring, assessment, and improvement to adapt to evolving threats and technologies.

The CIA Triad

The foundation of information security rests on three core principles, commonly known as the CIA Triad:

Principle Definition Implementation Examples Threat Examples
Confidentiality Ensuring that information is accessible only to authorized individuals Encryption, Access Controls, Authentication Data breaches, Eavesdropping, Social engineering
Integrity Maintaining accuracy and completeness of data Hashing, Digital signatures, Version control Malware, Unauthorized modifications, Data corruption
Availability Ensuring authorized users have reliable access to information Redundancy, Backups, DDoS protection DoS attacks, Hardware failures, Natural disasters

1.2 Defense in Depth

Defense in depth is a cybersecurity strategy that uses multiple layers of security controls throughout an IT system. This approach ensures that if one defensive layer fails, additional layers continue to provide protection.

Security Layers

Layer Controls Technologies Responsibility
Physical Access control, Surveillance Badges, Biometrics, CCTV Facilities Team
Network Firewalls, Segmentation NGFW, IDS/IPS, VPN Network Security
Endpoint Antivirus, EDR CrowdStrike, SentinelOne, Defender Security Operations
Application WAF, Input validation ModSecurity, Cloudflare, OWASP AppSec Team
Data Encryption, DLP AES-256, TLS, Tokenization Data Security
Identity MFA, SSO, IAM Okta, Auth0, Azure AD Identity Management

1.4 Security API Endpoints

The WIA Security Standard defines RESTful APIs for managing security configurations, policies, and monitoring. Below are the core security management endpoints:

Security Configuration APIs

GET /api/v1/security/config

Retrieve current security configuration for the organization

POST /api/v1/security/config

Create or update security configuration

GET /api/v1/security/policies

List all security policies

POST /api/v1/security/policies

Create a new security policy

PUT /api/v1/security/policies/{policyId}

Update an existing security policy

DELETE /api/v1/security/policies/{policyId}

Delete a security policy

GET /api/v1/security/audit-logs

Retrieve security audit logs with filtering options

POST /api/v1/security/scan

Initiate a security vulnerability scan

GET /api/v1/security/scan/{scanId}/results

Retrieve vulnerability scan results

POST /api/v1/security/compliance/validate

Validate compliance against specified framework

GET /api/v1/security/risk-assessment

Get current risk assessment and security posture

POST /api/v1/security/alerts

Create a security alert or notification

Example 1: Security Configuration Implementation (Python)

Python - Security Configuration Manager
import json
import requests
from typing import Dict, List, Optional
from datetime import datetime
import logging

class SecurityConfigManager:
    """
    WIA Security Configuration Manager
    Manages security configurations and policy enforcement
    """

    def __init__(self, api_base_url: str, api_key: str):
        self.api_base_url = api_base_url
        self.headers = {
            'Authorization': f'Bearer {api_key}',
            'Content-Type': 'application/json'
        }
        self.logger = logging.getLogger(__name__)

    def get_security_config(self, org_id: str) -> Dict:
        """Retrieve current security configuration"""
        try:
            response = requests.get(
                f'{self.api_base_url}/api/v1/security/config',
                headers=self.headers,
                params={'organizationId': org_id}
            )
            response.raise_for_status()
            return response.json()
        except requests.exceptions.RequestException as e:
            self.logger.error(f"Failed to retrieve config: {e}")
            raise

    def update_security_config(self, config: Dict) -> bool:
        """Update security configuration"""
        try:
            # Validate configuration against schema
            self._validate_config(config)

            response = requests.post(
                f'{self.api_base_url}/api/v1/security/config',
                headers=self.headers,
                json=config
            )
            response.raise_for_status()
            self.logger.info(f"Security config updated for org {config['organizationId']}")
            return True
        except Exception as e:
            self.logger.error(f"Failed to update config: {e}")
            return False

    def _validate_config(self, config: Dict) -> bool:
        """Validate security configuration"""
        required_fields = ['organizationId', 'securityProfile', 'networkSecurity']
        for field in required_fields:
            if field not in config:
                raise ValueError(f"Missing required field: {field}")

        # Validate MFA requirement
        if config.get('accessControl', {}).get('mfaRequired') is False:
            self.logger.warning("MFA is not required - security risk!")

        # Validate encryption standards
        encryption = config.get('encryption', {})
        if encryption.get('atRest', {}).get('algorithm') != 'AES-256':
            raise ValueError("AES-256 required for data at rest")

        return True

    def apply_baseline_config(self, org_id: str, level: str = 'enhanced') -> Dict:
        """Apply baseline security configuration"""
        baseline_config = {
            "organizationId": org_id,
            "securityProfile": {
                "level": level,
                "complianceFrameworks": ["SOC2", "ISO27001"],
                "riskTolerance": "low"
            },
            "networkSecurity": {
                "firewallEnabled": True,
                "idsEnabled": True,
                "ipsEnabled": True,
                "segmentation": [
                    {"name": "dmz", "cidr": "10.0.1.0/24", "securityZone": "public"},
                    {"name": "internal", "cidr": "10.0.2.0/24", "securityZone": "private"}
                ]
            },
            "accessControl": {
                "mfaRequired": True,
                "passwordPolicy": {
                    "minLength": 16,
                    "requireSpecialChar": True,
                    "requireNumbers": True,
                    "expirationDays": 90
                },
                "sessionTimeout": 3600
            },
            "encryption": {
                "atRest": {"algorithm": "AES-256", "keyManagement": "AWS-KMS"},
                "inTransit": {"tlsVersion": "1.3", "cipherSuites": ["TLS_AES_256_GCM_SHA384"]}
            },
            "monitoring": {
                "loggingEnabled": True,
                "siemIntegration": True,
                "alertThresholds": {"failedLogins": 5, "dataExfiltration": "1MB"}
            }
        }

        self.update_security_config(baseline_config)
        return baseline_config

Example 2: Access Control Implementation (Node.js)

JavaScript/Node.js - Access Control Middleware
const jwt = require('jsonwebtoken');
const bcrypt = require('bcrypt');
const speakeasy = require('speakeasy');

class AccessControlManager {
    constructor(config) {
        this.jwtSecret = config.jwtSecret;
        this.mfaRequired = config.mfaRequired || true;
        this.sessionTimeout = config.sessionTimeout || 3600;
    }

    /**
     * Authentication middleware
     */
    authenticate = async (req, res, next) => {
        try {
            const token = req.headers.authorization?.split(' ')[1];

            if (!token) {
                return res.status(401).json({
                    error: 'Authentication required',
                    code: 'AUTH_TOKEN_MISSING'
                });
            }

            const decoded = jwt.verify(token, this.jwtSecret);

            // Check session expiration
            if (Date.now() >= decoded.exp * 1000) {
                return res.status(401).json({
                    error: 'Session expired',
                    code: 'SESSION_EXPIRED'
                });
            }

            // Check MFA validation
            if (this.mfaRequired && !decoded.mfaValidated) {
                return res.status(401).json({
                    error: 'MFA validation required',
                    code: 'MFA_REQUIRED'
                });
            }

            req.user = decoded;
            next();
        } catch (error) {
            return res.status(401).json({
                error: 'Invalid authentication token',
                code: 'AUTH_INVALID'
            });
        }
    };

    /**
     * Authorization middleware - Role-based access control
     */
    authorize = (requiredRoles) => {
        return (req, res, next) => {
            if (!req.user) {
                return res.status(401).json({
                    error: 'Authentication required',
                    code: 'AUTH_REQUIRED'
                });
            }

            const userRoles = req.user.roles || [];
            const hasPermission = requiredRoles.some(role => userRoles.includes(role));

            if (!hasPermission) {
                return res.status(403).json({
                    error: 'Insufficient permissions',
                    code: 'FORBIDDEN',
                    required: requiredRoles,
                    current: userRoles
                });
            }

            next();
        };
    };

    /**
     * Validate MFA token
     */
    validateMFA = (secret, token) => {
        return speakeasy.totp.verify({
            secret: secret,
            encoding: 'base32',
            token: token,
            window: 2
        });
    };

    /**
     * Generate session token with security claims
     */
    generateToken = (user, mfaValidated = false) => {
        const payload = {
            userId: user.id,
            email: user.email,
            roles: user.roles,
            mfaValidated: mfaValidated,
            iat: Math.floor(Date.now() / 1000),
            exp: Math.floor(Date.now() / 1000) + this.sessionTimeout
        };

        return jwt.sign(payload, this.jwtSecret, { algorithm: 'HS256' });
    };
}

module.exports = AccessControlManager;

Example 3: Security Logging Implementation (Go)

Go - Security Event Logger
package security

import (
    "encoding/json"
    "fmt"
    "log"
    "time"
)

type SecurityEvent struct {
    EventID     string                 `json:"eventId"`
    Timestamp   time.Time             `json:"timestamp"`
    EventType   string                `json:"eventType"`
    Severity    string                `json:"severity"`
    UserID      string                `json:"userId,omitempty"`
    IPAddress   string                `json:"ipAddress"`
    Resource    string                `json:"resource"`
    Action      string                `json:"action"`
    Result      string                `json:"result"`
    Details     map[string]interface{} `json:"details,omitempty"`
}

type SecurityLogger struct {
    orgID      string
    siemEnabled bool
}

func NewSecurityLogger(orgID string, siemEnabled bool) *SecurityLogger {
    return &SecurityLogger{
        orgID:      orgID,
        siemEnabled: siemEnabled,
    }
}

// LogAuthenticationAttempt logs authentication events
func (sl *SecurityLogger) LogAuthenticationAttempt(userID, ipAddress, result string) {
    event := SecurityEvent{
        EventID:   generateEventID(),
        Timestamp: time.Now().UTC(),
        EventType: "AUTHENTICATION",
        Severity:  sl.determineSeverity(result),
        UserID:    userID,
        IPAddress: ipAddress,
        Resource:  "auth-service",
        Action:    "login",
        Result:    result,
    }

    sl.writeLog(event)
}

// LogAccessControlViolation logs authorization failures
func (sl *SecurityLogger) LogAccessControlViolation(userID, resource, action, ipAddress string) {
    event := SecurityEvent{
        EventID:   generateEventID(),
        Timestamp: time.Now().UTC(),
        EventType: "ACCESS_VIOLATION",
        Severity:  "HIGH",
        UserID:    userID,
        IPAddress: ipAddress,
        Resource:  resource,
        Action:    action,
        Result:    "DENIED",
        Details: map[string]interface{}{
            "organizationId": sl.orgID,
            "violation_type": "insufficient_permissions",
        },
    }

    sl.writeLog(event)
    sl.triggerAlert(event)
}

// LogDataAccess logs data access events for audit trail
func (sl *SecurityLogger) LogDataAccess(userID, resource, action, ipAddress string, recordCount int) {
    event := SecurityEvent{
        EventID:   generateEventID(),
        Timestamp: time.Now().UTC(),
        EventType: "DATA_ACCESS",
        Severity:  "INFO",
        UserID:    userID,
        IPAddress: ipAddress,
        Resource:  resource,
        Action:    action,
        Result:    "SUCCESS",
        Details: map[string]interface{}{
            "record_count": recordCount,
        },
    }

    sl.writeLog(event)
}

func (sl *SecurityLogger) writeLog(event SecurityEvent) {
    jsonData, err := json.Marshal(event)
    if err != nil {
        log.Printf("Error marshaling security event: %v", err)
        return
    }

    // Write to local logs
    log.Printf("[SECURITY] %s", string(jsonData))

    // Send to SIEM if enabled
    if sl.siemEnabled {
        sl.sendToSIEM(event)
    }
}

func (sl *SecurityLogger) determineSeverity(result string) string {
    if result == "FAILED" {
        return "WARNING"
    }
    return "INFO"
}

func (sl *SecurityLogger) triggerAlert(event SecurityEvent) {
    // Trigger real-time security alert
    fmt.Printf("[ALERT] Security violation detected: %+v\n", event)
}

    return fmt.Sprintf("SEC-%d", time.Now().UnixNano())

1.6 WIA Security Fundamentals Certification

Organizations implementing the WIA Security Standard must validate compliance with the following security fundamentals requirements:

Certification Requirements

  • Documented information security policy approved by executive leadership
  • CIA triad principles implemented across all systems and data classifications
  • Defense in depth strategy with minimum 4 security layers deployed
  • Network segmentation implemented with DMZ and internal zones
  • Firewall configured and actively monitoring all network traffic
  • Intrusion Detection System (IDS) deployed and monitored 24/7
  • Intrusion Prevention System (IPS) actively blocking malicious traffic
  • Endpoint protection (EDR) deployed on all workstations and servers
  • Multi-factor authentication (MFA) enforced for all user accounts
  • Password policy requiring minimum 12 characters with complexity
  • Session timeout configured not exceeding 60 minutes of inactivity
  • Encryption at rest using AES-256 for all sensitive data
  • Encryption in transit using TLS 1.2 or higher for all communications
  • Security logging enabled and centralized for all critical systems
  • SIEM integration for real-time security event correlation
  • Security audit logs retained for minimum 12 months
  • Vulnerability scanning performed at least monthly
  • Penetration testing conducted at least annually
  • Security awareness training completed by all employees annually
  • Incident response plan documented and tested quarterly
  • Data classification scheme implemented and enforced
  • Access control policies based on principle of least privilege
  • Regular security configuration reviews performed quarterly
  • Security metrics and KPIs tracked and reported to leadership
Certification Process: Complete all 24 requirements above and submit evidence to WIA Security Standards Board for review and certification approval.
