챕터 06

데이터 보호 및 개인정보

6.1 데이터 보호 개요

데이터 보호는 데이터의 전체 생명주기에 걸쳐 무단 접근, 손상 및 도난으로부터 데이터를 보호하는 프로세스입니다. 데이터 침해의 비용이 증가하고 규제가 강화됨에 따라 데이터 보호는 모든 조직의 최우선 과제가 되었습니다.

데이터 보호의 핵심 원칙

기밀성: 권한 있는 사용자만 데이터에 접근

무결성: 데이터가 정확하고 변경되지 않음

가용성: 필요할 때 데이터에 접근 가능

개인정보 보호: 개인 데이터의 적절한 처리

6.1.1 데이터 분류

효과적인 데이터 보호는 데이터 분류에서 시작합니다. 모든 데이터에 동일한 수준의 보호를 적용하는 것은 비효율적이며, 민감도에 따라 적절한 통제를 적용해야 합니다.

분류 설명 예시 보호 수준
공개 일반 공개 가능 마케팅 자료, 공개 보고서 기본
내부 직원용 내부 정보 내부 통신, 절차 표준
기밀 제한된 접근 필요 재무 데이터, 고객 정보 강화
극비 엄격히 제한 영업 비밀, 개인 건강 정보 최고

6.2 암호화

암호화는 데이터 보호의 기본 기술입니다. 데이터를 읽을 수 없는 형식으로 변환하여 무단 접근 시에도 데이터를 보호합니다.

6.2.1 암호화 유형

저장 데이터 암호화

  • 전체 디스크 암호화 (FDE)
  • 파일/폴더 수준 암호화
  • 데이터베이스 암호화 (TDE)
  • 클라우드 스토리지 암호화

전송 중 데이터 암호화

  • TLS/SSL 프로토콜
  • VPN 터널
  • 이메일 암호화 (S/MIME, PGP)
  • API 암호화

6.2.2 암호화 표준

WIA-SEC-001 암호화 요구사항: ============================ 대칭 암호화: • AES-256-GCM (권장) • AES-256-CBC (허용) • 3DES는 2025년까지 단계적 폐지 비대칭 암호화: • RSA 2048비트 이상 • ECDSA P-256 이상 • Ed25519 (권장) 해시 함수: • SHA-256 이상 • SHA-3 (권장) • MD5, SHA-1 사용 금지 TLS 요구사항: • TLS 1.2 이상 (브론즈/실버) • TLS 1.3 권장 (골드) • TLS 1.3 필수 (플래티넘)

6.3 데이터 손실 방지 (DLP)

DLP는 민감한 데이터가 조직 외부로 유출되는 것을 방지하는 기술과 프로세스입니다. 의도적인 데이터 탈취뿐만 아니라 우발적인 데이터 유출도 방지합니다.

6.3.1 DLP 유형

네트워크 DLP

네트워크 트래픽 모니터링, 이메일/웹 게이트웨이 통합, 외부 전송 차단

엔드포인트 DLP

워크스테이션/노트북 모니터링, USB/프린터 제어, 클립보드 제어

클라우드 DLP

클라우드 앱 모니터링, SaaS 데이터 보호, API 기반 검사

6.3.2 DLP 정책

일반적인 DLP 정책

  • 신용카드 번호: 16자리 숫자 패턴, Luhn 알고리즘 검증
  • 주민등록번호: 형식 매칭, 컨텍스트 분석
  • 건강 정보: 의료 용어 및 코드 탐지
  • 소스 코드: 파일 유형 및 패턴 매칭
  • 재무 데이터: 키워드 및 패턴 조합

6.4 개인정보 보호

개인정보 보호는 개인 데이터의 수집, 사용, 저장 및 공유를 적절하게 관리하는 것입니다. GDPR, 개인정보보호법 등 규제 준수가 필수적입니다.

6.4.1 주요 개인정보 보호 원칙

데이터 주체 권리

  • 접근권 - 자신의 데이터 열람
  • 정정권 - 부정확한 데이터 수정
  • 삭제권 - 데이터 삭제 요청
  • 이동권 - 데이터 이전 요청
  • 거부권 - 처리 거부

조직 의무

  • 적법한 처리 근거 확보
  • 목적 제한 준수
  • 데이터 최소화
  • 정확성 유지
  • 보관 기간 제한
  • 보안 조치 구현

6.4.2 개인정보 영향 평가 (PIA)

PIA 필수 상황

다음 경우 개인정보 영향 평가를 수행해야 합니다:

  • 대규모 개인정보 처리
  • 민감정보 처리
  • 자동화된 의사결정
  • 체계적인 모니터링
  • 새로운 기술 도입

6.5 데이터 백업 및 복구

백업은 데이터 손실, 손상 또는 랜섬웨어 공격으로부터 복구할 수 있는 마지막 방어선입니다.

6.5.1 백업 전략

3-2-1 백업 규칙: ================ • 3개의 데이터 복사본 유지 • 2개의 다른 저장 매체에 저장 • 1개는 오프사이트에 보관 추가 고려사항: • 정기적인 백업 테스트 • 암호화된 백업 • 에어갭 백업 (랜섬웨어 방어) • 복구 시간 목표 (RTO) 정의 • 복구 지점 목표 (RPO) 정의

6.5.2 백업 유형

유형 설명 장점 단점
전체 백업 모든 데이터 백업 간단한 복구 시간/저장공간 소요
증분 백업 마지막 백업 이후 변경분 빠른 백업 복구 시 모든 증분 필요
차등 백업 전체 백업 이후 변경분 중간 복구 시간 증분보다 큰 백업

챕터 요약

이 챕터에서는 데이터 보호 및 개인정보의 핵심 요소를 다루었습니다:

  • 데이터 분류: 민감도에 따른 데이터 분류 및 보호
  • 암호화: 저장 및 전송 데이터 암호화 표준
  • DLP: 데이터 손실 방지 기술 및 정책
  • 개인정보 보호: 규제 준수 및 데이터 주체 권리
  • 백업: 데이터 복구 전략
