챕터 05

위협 탐지 및 대응

5.1 위협 탐지 개요

위협 탐지는 조직의 시스템, 네트워크 및 데이터에 대한 잠재적인 보안 위협을 식별하는 프로세스입니다. 효과적인 위협 탐지는 공격의 조기 발견과 신속한 대응을 가능하게 합니다.

현대 위협 환경에서 전통적인 시그니처 기반 탐지만으로는 충분하지 않습니다. 조직은 AI/ML 기반 이상 탐지, 행동 분석, 위협 인텔리전스를 결합한 다층 접근 방식이 필요합니다.

위협 탐지의 진화

1세대: 시그니처 기반 - 알려진 악성코드 패턴 매칭

2세대: 휴리스틱 - 의심스러운 행동 패턴 탐지

3세대: 행동 분석 - 정상 행동에서의 편차 식별

4세대: AI/ML - 머신러닝을 통한 고급 위협 탐지

5.1.1 탐지 방법론

알려진 위협 탐지

  • 시그니처 매칭
  • IOC (침해 지표) 기반
  • 위협 인텔리전스 피드
  • 블랙리스트/화이트리스트

알려지지 않은 위협 탐지

  • 이상 탐지
  • 행동 분석 (UEBA)
  • ML 기반 분류
  • 샌드박스 분석

5.2 AI 기반 위협 탐지

인공지능과 머신러닝은 위협 탐지 능력을 혁명적으로 향상시켰습니다. AI는 대량의 데이터를 실시간으로 분석하여 인간 분석가가 놓칠 수 있는 패턴과 이상을 식별합니다.

5.2.1 ML 탐지 모델

기술 애플리케이션 장점 제한사항
지도 학습 악성코드 분류, 스팸 탐지 알려진 위협에 높은 정확도 레이블된 학습 데이터 필요
비지도 학습 이상 탐지, 클러스터링 알려지지 않은 위협 탐지 높은 오탐률
딥러닝 네트워크 트래픽 분석 복잡한 패턴 인식 높은 컴퓨팅 요구
강화 학습 적응형 대응, 게임 이론 방어 시간에 따른 자가 개선 신중한 보상 설계 필요

5.3 위협 인텔리전스

위협 인텔리전스는 현재 및 잠재적 공격에 대한 증거 기반 지식으로, 위협에 대한 정보에 입각한 결정을 내리는 데 사용됩니다.

5.3.1 위협 인텔리전스 유형

전략적 인텔리전스

경영진 대상, 고수준 위협 동향, 위험 평가에 사용

전술적 인텔리전스

보안 관리자 대상, 공격자 TTP, 방어 전략 수립

운영적 인텔리전스

SOC 분석가 대상, IOC, 실시간 탐지에 사용

5.3.2 위협 인텔리전스 활용

위협 인텔리전스 통합

  • SIEM 통합: IOC 자동 상관관계 분석
  • 방화벽: 악성 IP/도메인 자동 차단
  • EDR: 악성 해시 탐지
  • 이메일 보안: 피싱 캠페인 차단
  • 취약점 관리: 악용 위협 우선순위

5.4 사고 대응

사고 대응은 보안 사고를 처리하고 관리하는 조직적인 접근 방식입니다. 효과적인 사고 대응은 피해를 최소화하고 복구 시간과 비용을 줄입니다.

5.4.2 사고 분류

심각도 정의 대응 시간 에스컬레이션
심각 (P1) 활성 침해, 광범위한 영향 즉시 CISO, 경영진
높음 (P2) 확인된 위협, 제한된 영향 1시간 이내 보안 관리자
중간 (P3) 의심스러운 활동, 조사 필요 4시간 이내 SOC 리드
낮음 (P4) 정책 위반, 경미한 보안 이벤트 24시간 이내 SOC 분석가

5.5 위협 헌팅

위협 헌팅은 기존 보안 통제를 회피한 위협을 사전에 찾는 활동입니다. 반응적인 알림과 달리, 위협 헌팅은 적대자가 이미 환경에 존재할 수 있다고 가정하고 그들의 활동 증거를 적극적으로 찾습니다.

5.5.1 위협 헌팅 방법론

가설 기반 헌팅

위협 인텔리전스, 최근 사고 또는 산업 동향을 기반으로 잠재적 위협에 대한 이론으로 시작합니다. 가설을 증명하거나 반증하기 위한 쿼리와 분석을 개발합니다.

베이스라인 헌팅

정상 행동 베이스라인을 수립하고 편차를 조사합니다. "정상"이 무엇인지 이해하여 위협을 나타낼 수 있는 이상을 식별하는 데 중점을 둡니다.

5.5.2 MITRE ATT&CK 기반 헌팅

ATT&CK 기반 헌팅 예시: ===================== 초기 접근 (TA0001): 헌트: 이메일 보안을 우회한 피싱 시도 식별 쿼리: 외부 발신자의 매크로 활성화 문서 첨부파일 지표: 비정상 발신자 도메인, 내장 URL, 비밀번호 보호 파일 실행 (TA0002): 헌트: 스크립트 기반 실행 시도 탐지 쿼리: 인코딩된 명령 또는 다운로드 크래들이 있는 PowerShell/WMI 이벤트 지표: -enc 플래그, IEX(New-Object Net.WebClient), 의심스러운 부모 프로세스 지속성 (TA0003): 헌트: 무단 지속성 메커니즘 찾기 쿼리: 레지스트리 실행 키, 예약된 작업, 시작 폴더 수정 지표: 최근 생성된 항목, 서명되지 않은 실행 파일, 숨겨진 파일 측면 이동 (TA0008): 헌트: 내부 확산 탐지 쿼리: SMB 세션, RDP 연결, WMI 원격 실행 지표: 비정상 서비스 계정, 업무 외 시간 활동, 새로운 연결
