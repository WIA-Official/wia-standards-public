제1장: 보안 기초

1.1 정보 보안 소개

정보 보안은 정보 위험을 완화하여 정보를 보호하는 실무입니다. 정보 위험 관리의 일부이며 무단 액세스, 사용, 공개, 중단, 삭제, 손상, 수정, 검사, 기록 또는 평가 절하의 확률을 방지하거나 줄이는 것을 포함합니다.

핵심 원칙: 보안은 제품이 아니라 프로세스입니다. 진화하는 위협과 기술에 적응하기 위해 지속적인 모니터링, 평가 및 개선이 필요합니다.

CIA 트라이어드

정보 보안의 기초는 CIA 트라이어드로 일반적으로 알려진 세 가지 핵심 원칙에 기반합니다:

원칙 정의 구현 예제 위협 예제
기밀성 (Confidentiality) 권한이 있는 개인만 정보에 액세스할 수 있도록 보장 암호화, 액세스 제어, 인증 데이터 침해, 도청, 사회 공학
무결성 (Integrity) 데이터의 정확성과 완전성 유지 해싱, 디지털 서명, 버전 제어 맬웨어, 무단 수정, 데이터 손상
가용성 (Availability) 권한이 있는 사용자가 정보에 안정적으로 액세스할 수 있도록 보장 이중화, 백업, DDoS 보호 DoS 공격, 하드웨어 장애, 자연 재해

1.2 심층 방어

심층 방어는 IT 시스템 전체에 걸쳐 여러 계층의 보안 제어를 사용하는 사이버보안 전략입니다. 이 접근 방식은 하나의 방어 계층이 실패하더라도 추가 계층이 계속 보호를 제공하도록 합니다.

1.3 보안 아키텍처 프레임워크

강력한 보안 아키텍처는 조직 전체에 걸쳐 보안 제어를 구현하고 유지 관리하기 위한 청사진을 제공합니다.

JSON 스키마 - 보안 구성 모델
{
  "$schema": "http://json-schema.org/draft-07/schema#",
  "title": "WIA Security Configuration",
  "type": "object",
  "properties": {
    "organizationId": {
      "type": "string",
      "description": "Unique organization identifier"
    },
    "securityProfile": {
      "type": "object",
      "properties": {
        "level": {"type": "string", "enum": ["basic", "enhanced", "maximum"]},
        "complianceFrameworks": {
          "type": "array",
          "items": {"type": "string", "enum": ["SOC2", "ISO27001", "NIST", "PCI-DSS", "HIPAA"]}
        }
      }
    }
  }
}

1.4 보안 API 엔드포인트

GET /api/v1/security/config

조직의 현재 보안 구성 검색

POST /api/v1/security/config

보안 구성 생성 또는 업데이트

GET /api/v1/security/policies

모든 보안 정책 나열

POST /api/v1/security/policies

새 보안 정책 생성

PUT /api/v1/security/policies/{policyId}

기존 보안 정책 업데이트

DELETE /api/v1/security/policies/{policyId}

보안 정책 삭제

GET /api/v1/security/audit-logs

필터링 옵션을 사용하여 보안 감사 로그 검색

POST /api/v1/security/scan

보안 취약점 스캔 시작

GET /api/v1/security/scan/{scanId}/results

취약점 스캔 결과 검색

POST /api/v1/security/compliance/validate

지정된 프레임워크에 대한 규정 준수 검증

GET /api/v1/security/risk-assessment

현재 위험 평가 및 보안 태세 가져오기

POST /api/v1/security/alerts

보안 경고 또는 알림 생성

1.5 WIA 보안 기초 인증

인증 요구 사항

  • 경영진이 승인한 문서화된 정보 보안 정책
  • 모든 시스템 및 데이터 분류에 걸쳐 구현된 CIA 트라이어드 원칙
  • 최소 4개의 보안 계층이 배포된 심층 방어 전략
  • DMZ 및 내부 영역으로 구현된 네트워크 세그먼테이션
  • 구성되고 모든 네트워크 트래픽을 적극적으로 모니터링하는 방화벽
  • 24/7 모니터링되는 침입 탐지 시스템(IDS) 배포
  • 악성 트래픽을 적극적으로 차단하는 침입 방지 시스템(IPS)
  • 모든 워크스테이션 및 서버에 배포된 엔드포인트 보호(EDR)
  • 모든 사용자 계정에 적용되는 다단계 인증(MFA)
  • 복잡성을 포함한 최소 12자 이상의 비밀번호 정책
  • 비활성 60분을 초과하지 않도록 구성된 세션 시간 초과
  • 모든 민감한 데이터에 대해 AES-256을 사용한 저장 시 암호화
  • 모든 통신에 대해 TLS 1.2 이상을 사용한 전송 중 암호화
  • 모든 중요 시스템에 대해 활성화되고 중앙 집중화된 보안 로깅
  • 실시간 보안 이벤트 상관 관계를 위한 SIEM 통합
  • 최소 12개월 동안 보관된 보안 감사 로그
  • 최소 월별로 수행되는 취약점 스캔
  • 최소 연간 수행되는 침투 테스트
  • 모든 직원이 연간 완료하는 보안 인식 교육
  • 분기별로 문서화되고 테스트되는 사고 대응 계획
  • 구현되고 시행되는 데이터 분류 체계
  • 최소 권한 원칙을 기반으로 한 액세스 제어 정책
  • 분기별로 수행되는 정기적인 보안 구성 검토
  • 리더십에 추적 및 보고되는 보안 메트릭 및 KPI
