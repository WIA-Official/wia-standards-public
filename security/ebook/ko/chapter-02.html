챕터 02

사이버보안 기초

2.1 사이버보안 개요

사이버보안은 컴퓨터 시스템, 네트워크, 프로그램 및 데이터를 디지털 공격으로부터 보호하는 실천입니다. 이러한 공격은 일반적으로 민감한 정보에 접근, 변경 또는 파괴하거나, 사용자로부터 금전을 갈취하거나, 정상적인 비즈니스 프로세스를 방해하는 것을 목표로 합니다.

효과적인 사이버보안 접근 방식은 컴퓨터, 네트워크, 프로그램 또는 사용자가 공격 대상으로 의도하는 데이터 전반에 걸쳐 여러 계층의 보호를 갖추고 있습니다. 조직에서 사람, 프로세스 및 기술은 효과적인 사이버 위협 방어를 만들기 위해 모두 상호 보완해야 합니다.

2.1.1 CIA 삼원칙

정보보안의 핵심 원칙인 CIA 삼원칙은 모든 보안 프로그램의 기초입니다:

기밀성 (Confidentiality)

권한 있는 사용자만 민감한 정보에 접근할 수 있도록 보장합니다. 암호화, 접근 제어 및 데이터 분류를 통해 구현됩니다.

무결성 (Integrity)

데이터가 무단 변경 없이 정확하고 완전하게 유지되도록 보장합니다. 해시, 디지털 서명 및 변경 관리를 통해 보호됩니다.

가용성 (Availability)

권한 있는 사용자가 필요할 때 시스템과 데이터에 접근할 수 있도록 보장합니다. 중복성, 백업 및 재해 복구를 통해 달성됩니다.

2.1.2 추가 보안 원칙

확장된 보안 원칙

  • 인증 (Authentication): 사용자나 시스템의 신원을 확인
  • 권한 부여 (Authorization): 인증된 엔터티의 접근 권한 결정
  • 부인 방지 (Non-repudiation): 행위자가 자신의 행동을 부인할 수 없도록 보장
  • 책임 추적성 (Accountability): 행동을 특정 엔터티에 추적

2.2 위협 환경

현대 사이버 위협 환경은 복잡하고 끊임없이 진화합니다. 조직은 다양한 위협 행위자와 공격 벡터에 대비해야 합니다.

2.2.1 위협 행위자 유형

행위자 유형 동기 능력 수준 일반적인 공격
국가 지원 행위자 스파이, 파괴, 영향력 매우 높음 APT, 제로데이, 공급망 공격
조직 범죄 그룹 금전적 이득 높음 랜섬웨어, BEC, 카드 사기
핵티비스트 이념적, 정치적 중간 DDoS, 웹사이트 훼손, 데이터 유출
내부자 불만, 금전, 실수 다양함 데이터 탈취, 사보타지, 과실
스크립트 키디 명성, 재미 낮음 알려진 취약점 악용, 피싱

2.2.3 MITRE ATT&CK 프레임워크

MITRE ATT&CK은 실제 관찰을 기반으로 한 적대적 전술, 기술 및 절차(TTP)의 지식 기반입니다. 이 프레임워크는 위협 탐지 및 대응 전략을 개발하는 데 필수적입니다.

MITRE ATT&CK 전술 개요: ======================= 1. 정찰 (Reconnaissance) - 목표 정보 수집 - 소셜 미디어 조사 - 기술 스캐닝 2. 자원 개발 (Resource Development) - 인프라 구매/임대 - 악성코드 개발 - 계정 생성 3. 초기 접근 (Initial Access) - 피싱 - 취약한 서비스 악용 - 공급망 침해 4. 실행 (Execution) - 명령줄 인터페이스 - 스크립팅 - 사용자 실행 5. 지속성 (Persistence) - 레지스트리 수정 - 예약된 작업 - 유효한 계정 6. 권한 상승 (Privilege Escalation) - 취약점 악용 - 접근 토큰 조작 - 바이패스 UAC 7. 방어 회피 (Defense Evasion) - 난독화 - 도구 삭제 - 로깅 비활성화 8. 자격 증명 접근 (Credential Access) - 무차별 대입 - 자격 증명 덤핑 - 키로깅 9. 탐색 (Discovery) - 네트워크 스캐닝 - 시스템 정보 수집 - 계정 열거 10. 측면 이동 (Lateral Movement) - 원격 서비스 - 내부 스피어피싱 - SMB/Windows Admin Shares 11. 수집 (Collection) - 데이터 스테이징 - 스크린 캡처 - 키보드 입력 캡처 12. 명령 및 제어 (Command and Control) - 암호화된 채널 - 프록시 - 웹 서비스 13. 유출 (Exfiltration) - C2 채널 통한 유출 - 대체 프로토콜 - 물리적 매체 14. 영향 (Impact) - 데이터 암호화 - 데이터 파괴 - 서비스 거부

2.3 심층 방어 전략

2.3.2 각 계층별 보안 통제

데이터 계층

  • 저장 데이터 암호화 (AES-256)
  • 전송 중 데이터 암호화 (TLS 1.3)
  • 데이터 손실 방지 (DLP)
  • 데이터 분류 및 라벨링
  • 데이터 마스킹 및 토큰화

애플리케이션 계층

  • 보안 소프트웨어 개발 생명주기 (SSDLC)
  • 웹 애플리케이션 방화벽 (WAF)
  • 코드 검토 및 정적 분석
  • 동적 애플리케이션 보안 테스트
  • API 보안

호스트 계층

  • 엔드포인트 탐지 및 대응 (EDR)
  • 안티바이러스/안티맬웨어
  • 호스트 기반 방화벽
  • 시스템 하드닝
  • 패치 관리

네트워크 계층

  • 차세대 방화벽 (NGFW)
  • 침입 탐지/방지 시스템
  • 네트워크 세분화
  • DNS 보안
  • 네트워크 접근 제어 (NAC)

2.4 위험 관리

위험 관리는 보안 프로그램의 핵심입니다. 조직은 위험을 식별, 평가 및 처리하여 수용 가능한 수준으로 유지해야 합니다.

2.4.1 위험 평가 프로세스

  1. 자산 식별: 보호해야 할 자산 목록 작성 (데이터, 시스템, 인력)
  2. 위협 식별: 각 자산에 대한 잠재적 위협 파악
  3. 취약점 식별: 악용될 수 있는 약점 발견
  4. 위험 분석: 가능성 × 영향으로 위험 수준 계산
  5. 위험 평가: 위험 수용 기준과 비교하여 우선순위 결정
  6. 위험 처리: 완화, 전가, 수용 또는 회피 결정

2.4.2 위험 처리 옵션

옵션 설명 예시
위험 완화 통제를 구현하여 위험 감소 방화벽 배치, 암호화 적용
위험 전가 위험을 제3자에게 이전 사이버 보험, 아웃소싱
위험 수용 위험을 인지하고 수용 저위험 시스템의 알려진 취약점
위험 회피 위험을 발생시키는 활동 중단 위험한 서비스 폐기

위험 관리 주의사항

위험을 완전히 제거하는 것은 불가능합니다. 효과적인 위험 관리의 목표는 위험을 조직의 위험 수용 범위 내로 줄이는 것입니다. 모든 위험 결정은 비즈니스 맥락에서 이루어져야 하며, 보안 팀과 비즈니스 이해관계자 간의 협력이 필수적입니다.

2.5 보안 프레임워크

보안 프레임워크는 조직이 보안 프로그램을 구조화하고 관리하는 데 도움이 되는 지침과 모범 사례를 제공합니다.

2.5.1 주요 보안 프레임워크

NIST 사이버보안 프레임워크

5개 핵심 기능으로 구성:

  • 식별 (Identify)
  • 보호 (Protect)
  • 탐지 (Detect)
  • 대응 (Respond)
  • 복구 (Recover)

ISO/IEC 27001

정보보안 관리 시스템(ISMS) 표준:

  • 정책 및 조직
  • 인적 보안
  • 자산 관리
  • 접근 제어
  • 암호화

CIS Controls

우선순위가 지정된 방어 조치:

  • 자산 인벤토리 및 제어
  • 소프트웨어 자산 인벤토리
  • 데이터 보호
  • 보안 구성
  • 계정 관리

COBIT

IT 거버넌스 프레임워크:

  • 평가, 지시, 모니터링
  • 계획 및 조직
  • 구축 및 구현
  • 제공 및 지원
  • 모니터링 및 평가

2.5.2 WIA-SEC-001과 타 프레임워크 매핑

프레임워크 정렬 수준 커버리지
ISO 27001 완전 정렬 95%+
NIST CSF 5개 기능 모두 매핑 100%
CIS Controls 모든 통제 포함 100%
PCI DSS 모든 요구사항 대응 85%+

2.6 보안 인식 및 교육

사람은 보안의 가장 약한 고리이자 가장 강력한 방어선이 될 수 있습니다. 효과적인 보안 인식 프로그램은 직원들이 위협을 인식하고 적절하게 대응하도록 교육합니다.

2.6.1 보안 인식 프로그램 구성요소

WIA-SEC-001 보안 교육 요구사항

  • 신입 직원 보안 오리엔테이션
  • 연간 보안 인식 교육 (모든 직원)
  • 역할 기반 보안 교육 (개발자, 관리자 등)
  • 정기적인 피싱 시뮬레이션
  • 보안 정책 검토 및 서명
  • 사고 보고 절차 교육

2.6.2 교육 주제

기본 주제

  • 피싱 및 사회 공학 인식
  • 비밀번호 보안
  • 물리적 보안
  • 데이터 분류 및 처리
  • 사고 보고

고급 주제

  • 보안 코딩 관행
  • 클라우드 보안
  • 모바일 장치 보안
  • 규제 준수
  • 사고 대응
