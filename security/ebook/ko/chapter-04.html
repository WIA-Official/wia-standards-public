챕터 04

신원 및 접근 관리

4.1 IAM 개요

신원 및 접근 관리(IAM)는 디지털 신원을 관리하고 사용자가 적절한 리소스에 적절한 시간에 적절한 이유로 접근할 수 있도록 보장하는 프레임워크입니다. IAM은 현대 기업 보안의 핵심 구성 요소입니다.

제로 트러스트 환경에서 IAM은 더욱 중요해집니다. "절대 신뢰하지 말고, 항상 검증하라"는 원칙은 강력한 신원 확인과 지속적인 접근 검증을 요구합니다.

IAM의 핵심 구성요소

신원 관리: 사용자 계정의 생명주기 관리 (생성, 수정, 삭제)

인증: 사용자가 주장하는 신원이 맞는지 확인 (로그인)

권한 부여: 인증된 사용자가 무엇을 할 수 있는지 결정

감사: 접근 활동을 기록하고 분석

4.1.1 IAM 성숙도 모델

레벨 특성 기능
레벨 1: 기초 분산된 신원 관리 기본 비밀번호 정책, 수동 프로비저닝
레벨 2: 통합 중앙화된 디렉토리 SSO, 기본 MFA, 셀프서비스
레벨 3: 자동화 자동화된 프로비저닝 역할 기반 접근, PAM, 접근 검토
레벨 4: 적응형 위험 기반 접근 적응형 인증, AI 분석, 제로 트러스트

4.2 인증 메커니즘

인증은 사용자의 신원을 확인하는 프로세스입니다. 현대 인증 시스템은 여러 요소를 결합하여 보안을 강화합니다.

4.2.1 다중 요소 인증 (MFA)

지식 요소

사용자가 알고 있는 것

  • 비밀번호
  • PIN
  • 보안 질문

소유 요소

사용자가 가지고 있는 것

  • 스마트폰 (TOTP)
  • 하드웨어 토큰
  • 스마트 카드

고유 요소

사용자 자체의 특성

  • 지문
  • 안면 인식
  • 음성

4.2.2 생체 인증

생체 인증은 고급 WIA-SEC-001 인증의 핵심 요구사항입니다. 다양한 생체 인식 방식이 지원됩니다.

4.2.3 패스워드리스 인증

FIDO2/WebAuthn 표준

패스워드리스 인증의 핵심 기술:

  • FIDO2: 피싱 방지, 공개키 기반 인증
  • WebAuthn: 웹 브라우저 표준 API
  • CTAP: 인증 장치 통신 프로토콜
  • 지원 장치: 보안 키, 플랫폼 인증자, 스마트폰

4.3 권한 부여 및 접근 제어

권한 부여는 인증된 사용자가 어떤 리소스에 접근하고 어떤 작업을 수행할 수 있는지 결정합니다.

4.3.1 접근 제어 모델

모델 설명 사용 사례
RBAC 역할 기반 - 역할에 권한 할당 대부분의 기업 환경
ABAC 속성 기반 - 사용자/리소스 속성으로 결정 복잡한 정책, 클라우드
PBAC 정책 기반 - 명시적 정책 규칙 규제 준수 환경
ReBAC 관계 기반 - 엔터티 간 관계로 결정 소셜, 협업 애플리케이션

4.3.2 최소 권한 원칙

최소 권한 구현 전략: ==================== 1. 기본 거부 (Default Deny) - 모든 접근을 기본적으로 거부 - 명시적으로 필요한 권한만 부여 2. 역할 세분화 - 광범위한 역할 대신 세분화된 역할 - 직무 분리 원칙 적용 3. 시간 제한 접근 - 영구 권한 대신 시간 제한 권한 - Just-In-Time (JIT) 접근 4. 정기적 검토 - 분기별 접근 권한 검토 - 미사용 권한 자동 제거 5. 승인 워크플로우 - 민감 리소스 접근에 승인 필요 - 다중 승인자 요구 (고위험)

4.4 특권 접근 관리 (PAM)

특권 접근 관리는 관리자 계정 및 높은 권한을 가진 계정을 보호하고 관리하는 보안 솔루션입니다. 이러한 계정은 공격자의 주요 표적입니다.

4.4.1 PAM 핵심 기능

비밀번호 볼트

  • 특권 계정 비밀번호 중앙 저장
  • 자동 비밀번호 교체
  • 체크아웃/체크인 워크플로우
  • 비밀번호 복잡성 적용

세션 관리

  • 특권 세션 녹화
  • 실시간 세션 모니터링
  • 명령 필터링
  • 세션 종료 기능

4.4.2 Just-In-Time 특권 접근

4.4.3 서비스 계정 관리

서비스 계정 보안 위험

서비스 계정은 종종 간과되지만 높은 권한을 가진 경우가 많습니다:

  • 정적 비밀번호로 인한 유출 위험
  • 과도한 권한 부여
  • 비활성 계정 방치
  • 소유자 불명확

대책: 서비스 계정 인벤토리, 정기 비밀번호 교체, 최소 권한, 소유자 지정

4.5 싱글 사인온 (SSO)

SSO는 사용자가 한 번의 인증으로 여러 애플리케이션에 접근할 수 있게 합니다. 이는 사용자 경험을 개선하고 비밀번호 피로를 줄여 보안을 향상시킵니다.

4.5.1 SSO 프로토콜

프로토콜 용도 특징
SAML 2.0 엔터프라이즈 웹 SSO XML 기반, 성숙한 표준
OAuth 2.0 API 권한 부여 토큰 기반, 위임된 접근
OIDC 현대적 인증 OAuth 2.0 + 신원 레이어
Kerberos 온프레미스 환경 티켓 기반, Windows 통합

4.5.2 SSO 아키텍처

SAML SSO 인증 흐름: =================== 1. 사용자 → 서비스 제공자(SP) 접근 시도 2. SP → 사용자를 신원 제공자(IdP)로 리디렉션 3. IdP → 사용자 인증 (비밀번호/MFA) 4. IdP → SAML 어설션 생성 및 서명 5. 사용자 → SAML 어설션을 SP에 제출 6. SP → 어설션 검증 후 접근 허용 보안 고려사항: - SAML 어설션 서명 및 암호화 - 어설션 유효 기간 제한 - 세션 타임아웃 설정 - 로그아웃 전파 (SLO)

4.6 신원 거버넌스

신원 거버넌스는 조직 전체의 신원 및 접근 권한을 관리하고 규제 준수를 보장하는 프레임워크입니다.

4.6.1 접근 검토

WIA-SEC-001 접근 검토 요구사항

  • 분기별: 모든 사용자 접근 권한 검토
  • 월별: 특권 계정 검토
  • 즉시: 역할 변경 시 접근 권한 조정
  • 연간: 역할 및 권한 모델 전체 검토

4.6.2 직무 분리 (SoD)

직무 분리는 한 사람이 중요 프로세스의 모든 단계를 수행할 수 없도록 하여 사기 및 오류를 방지합니다.

SoD 충돌 예시

  • 공급업체 생성 + 지불 승인
  • 계정 생성 + 권한 부여
  • 개발 + 프로덕션 배포
  • 주문 생성 + 주문 승인

SoD 구현

  • 역할 간 충돌 매트릭스 정의
  • 자동화된 충돌 탐지
  • 예외 승인 워크플로우
  • 보상 통제 구현

챕터 요약

이 챕터에서는 신원 및 접근 관리의 핵심 요소를 다루었습니다:

  • IAM 기초: 신원 관리, 인증, 권한 부여, 감사의 핵심 구성요소
  • 인증 메커니즘: MFA, 생체 인증, 패스워드리스 인증
  • 접근 제어: RBAC, ABAC 및 최소 권한 원칙
  • PAM: 특권 계정 보호 및 Just-In-Time 접근
  • SSO: 통합 인증을 통한 사용자 경험 및 보안 향상
  • 거버넌스: 접근 검토 및 직무 분리
