06장: Phase 3 — 페더레이션 프로토콜

6.1 한 단락으로 보는 Phase 3

Phase 3은 독립 관할권이 데이터베이스를 병합하지 않고 물 부족 데이터를 페더레이트하고, 리플레이를 방어하고, 청중 통제 경계를 강제하는 방법을 명세합니다.

6.2 역할

관할권 기관, 유역 조정자, 공급자, 규제 기관, 연구 컨소시엄, 공공. 신뢰는 페더레이션 핸드셰이크로 수립되고 서명된 영수증에 기록됨.

6.3 페더레이션 핸드셰이크

WIA-SOCIAL Phase 3 §5 영수증 형태를 재사용. 상태 머신: IDLE → PENDING(원본이 서명 검증) → ACCEPTED(원본이 영수증 발행) → 선택적 REVOKED. 영수증은 일반적인 국제 수자원 데이터 보존 규범에 따라 최소 25년 지속.

6.4 리플레이 방어

각 서명된 봉투는 96비트 nonce + RFC 3339 타임스탬프 보유. 수신자는 ±300초 이상 편차 거부, 지난 600초 동안 본 (signer, nonce) 튜플 거부, 본 nonce 캐시를 최소 600초 유지해야 함.

6.5 청중 통제

청중가시성
public유역 집계 통계, 가뭄 지수 관측, 제한 선언
jurisdiction_partner할당별 세부(개별 권리 보유자), 비상 대응 세부
basin_coordinator전체 유역 할당 + 제한 + 비상 데이터
regulator규제 관할 내 전체 데이터
research지역 개인정보 규칙에 따른 비식별 데이터

6.6 공공 협의 봉투

주요 할당 변경에 대한 공공 협의를 의무화하는 관할권을 위해 표준은 의견 수렴 기간과 처리를 기록하는 public_consultation 봉투(Phase 1 §4.7)를 제공합니다. 페더레이션 피어는 결과적 할당 변경을 존중하기 전에 협의가 수행되었는지 검증할 수 있습니다.

구현자 노트

물 부족 인프라에서 반복되는 운영 관심사 세 가지가 표준에 박혀 있습니다. 첫째, 관할 권한 검증: 할당/제한/비상 엔드포인트 쓰기는 작성자가 관할권 권한을 입증해야 함(WIA-OMNI-API jurisdictional_authority 클레임으로 검증). 둘째, 비상 엔드포인트 리플레이 방어: 중복 비상 선언은 중복 대응 활성화를 트리거하므로 본 nonce 캐시는 호스트 재시작에 걸쳐 영구적이어야 함. 셋째, 공공 대시보드 속도 제한: 가뭄 지수 스트림은 건기에 대시보드 트래픽을 끌어모으므로 호스트는 인제스트 경로 응답성을 유지하기 위해 읽기 복제본을 프로비저닝해야 함.

유역 우선 설계

이 표준의 모든 결정은 다음 질문을 거칩니다: 이것이 유역 인구가 자기 유역의 물 부족 상황을 더 잘 볼 수 있게 하는가, 아니면 단지 관료 중앙에 더 접근 가능하게 만드는가? 두 가지가 충돌하면 유역 인구 가시성이 이깁니다. 그래서 표준은 집계 유역 데이터에 대한 공공 읽기를 의무화하고, 데이터를 단일 조정자로 중앙집중화하기를 거부하며, 모든 관할권 간 읽기를 감사 로그로 남겨 유역 인구가 자기 물 데이터가 무엇을 하는지 검증할 수 있게 합니다.
