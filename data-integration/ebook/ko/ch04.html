04장: API 기반 데이터 통합

홍익인간 (弘益人間)

"널리 인간을 이롭게 하라"

현대적인 API 기술을 활용하여 시스템 간 데이터를 효율적으로 통합하고, 실시간 동기화를 통해 모든 이해관계자에게 최신 데이터를 제공합니다.

개요

API 기반 데이터 통합은 현대 애플리케이션에서 가장 보편적인 통합 방식입니다. REST, GraphQL, gRPC, 웹훅 등 다양한 API 기술을 이해하고 적절히 활용하는 것이 성공적인 데이터 통합의 핵심입니다.

이 장에서는 각 API 유형의 특징, 장단점, 사용 사례, 그리고 실제 구현 방법을 상세히 다룹니다.

REST API 통합

REST (REpresentational State Transfer)는 HTTP 프로토콜을 기반으로 하는 가장 널리 사용되는 API 아키텍처 스타일입니다.

REST API 핵심 원칙

원칙 설명 예시
무상태성
(Stateless)		 각 요청은 독립적이며 서버는 클라이언트 상태 저장 안함 인증 토큰을 매 요청마다 전송
캐시 가능
(Cacheable)		 응답은 캐시 가능 여부 명시 Cache-Control, ETag 헤더 사용
균일한 인터페이스
(Uniform Interface)		 일관된 URI와 HTTP 메서드 사용 GET /users, POST /users, PUT /users/:id
계층화 시스템
(Layered System)		 클라이언트는 중간 계층 존재 알 필요 없음 로드 밸런서, 프록시, 게이트웨이
리소스 기반
(Resource-Based)		 URI는 리소스를 나타냄 /customers/123/orders/456

REST API 구현 예시

# Python: REST API 데이터 통합
import requests
from typing import Dict, List
import time

class RESTAPIIntegration:
    """
    REST API를 통한 데이터 통합
    """
    def __init__(self, base_url: str, api_key: str):
        self.base_url = base_url
        self.headers = {
            "Authorization": f"Bearer {api_key}",
            "Content-Type": "application/json"
        }
        self.session = requests.Session()
        self.session.headers.update(self.headers)

    def get_customers(self, page: int = 1, limit: int = 100) -> List[Dict]:
        """
        고객 데이터 조회 (페이지네이션)
        """
        url = f"{self.base_url}/customers"
        params = {"page": page, "limit": limit}

        response = self.session.get(url, params=params)
        response.raise_for_status()

        return response.json()["data"]

    def get_all_customers(self) -> List[Dict]:
        """
        모든 고객 데이터 조회 (전체 페이지네이션)
        """
        all_customers = []
        page = 1

        while True:
            customers = self.get_customers(page=page)
            if not customers:
                break

            all_customers.extend(customers)
            page += 1

            # Rate limiting 준수
            time.sleep(0.1)

        return all_customers

    def create_customer(self, customer_data: Dict) -> Dict:
        """
        고객 생성
        """
        url = f"{self.base_url}/customers"
        response = self.session.post(url, json=customer_data)
        response.raise_for_status()

        return response.json()

    def update_customer(self, customer_id: str, updates: Dict) -> Dict:
        """
        고객 정보 업데이트
        """
        url = f"{self.base_url}/customers/{customer_id}"
        response = self.session.put(url, json=updates)
        response.raise_for_status()

        return response.json()

    def batch_sync(self, source_data: List[Dict], target_key: str = "id"):
        """
        배치 동기화: 소스 데이터와 REST API 동기화
        """
        existing_customers = {c[target_key]: c for c in self.get_all_customers()}

        for record in source_data:
            customer_id = record[target_key]

            if customer_id in existing_customers:
                # 업데이트
                self.update_customer(customer_id, record)
            else:
                # 생성
                self.create_customer(record)

# 사용 예시
api = RESTAPIIntegration(
    base_url="https://api.example.com/v1",
    api_key="your-api-key"
)

# 전체 고객 데이터 가져오기
customers = api.get_all_customers()
print(f"Total customers: {len(customers)}")

REST API 모범 사례

  • 버전 관리: URL에 API 버전 포함 (/v1/, /v2/)
  • HTTP 메서드 올바른 사용: GET(조회), POST(생성), PUT(전체 업데이트), PATCH(부분 업데이트), DELETE(삭제)
  • 상태 코드 활용: 200(성공), 201(생성), 400(잘못된 요청), 401(인증 실패), 404(없음), 500(서버 오류)
  • 페이지네이션: 대량 데이터 조회 시 페이징 구현
  • Rate Limiting: API 호출 제한 준수
  • 에러 처리: 재시도 로직, 지수 백오프 구현
  • 캐싱: ETag, Last-Modified 헤더 활용

GraphQL API 통합

GraphQL은 Facebook이 개발한 쿼리 언어로, 클라이언트가 필요한 데이터만 정확히 요청할 수 있습니다.

GraphQL vs REST 비교

특징 REST GraphQL
엔드포인트 여러 엔드포인트 (각 리소스별) 단일 엔드포인트 (/graphql)
데이터 가져오기 Over-fetching 또는 Under-fetching 가능 필요한 필드만 정확히 요청
버전 관리 URL 버전 (/v1/, /v2/) 스키마 진화, 버전 불필요
중첩 데이터 여러 요청 필요 단일 쿼리로 중첩 데이터 조회
캐싱 HTTP 캐싱 자동 지원 클라이언트 측 캐싱 필요
학습 곡선 낮음 (HTTP 기본) 중간 (GraphQL 문법 학습 필요)
최적 사용 표준 CRUD, 간단한 API 복잡한 쿼리, 모바일 앱

GraphQL 구현 예시

# Python: GraphQL API 데이터 통합
from gql import gql, Client
from gql.transport.requests import RequestsHTTPTransport

class GraphQLIntegration:
    """
    GraphQL API를 통한 데이터 통합
    """
    def __init__(self, endpoint: str, api_key: str):
        transport = RequestsHTTPTransport(
            url=endpoint,
            headers={"Authorization": f"Bearer {api_key}"},
            use_json=True,
        )

        self.client = Client(
            transport=transport,
            fetch_schema_from_transport=True,
        )

    def get_customer_with_orders(self, customer_id: str):
        """
        고객과 주문 정보를 단일 쿼리로 조회
        """
        query = gql("""
            query GetCustomerWithOrders($customerId: ID!) {
                customer(id: $customerId) {
                    id
                    name
                    email
                    createdAt
                    orders {
                        id
                        totalAmount
                        status
                        items {
                            productId
                            productName
                            quantity
                            price
                        }
                    }
                }
            }
        """)

        variables = {"customerId": customer_id}
        result = self.client.execute(query, variable_values=variables)

        return result["customer"]

    def search_products(self, keyword: str, min_price: float, max_price: float):
        """
        제품 검색 (필요한 필드만 요청)
        """
        query = gql("""
            query SearchProducts($keyword: String!, $minPrice: Float!, $maxPrice: Float!) {
                products(
                    filter: {
                        search: $keyword,
                        priceRange: { min: $minPrice, max: $maxPrice }
                    }
                ) {
                    id
                    name
                    price
                    inStock
                    category {
                        name
                    }
                }
            }
        """)

        variables = {
            "keyword": keyword,
            "minPrice": min_price,
            "maxPrice": max_price
        }

        result = self.client.execute(query, variable_values=variables)
        return result["products"]

    def bulk_create_orders(self, orders: List[Dict]):
        """
        대량 주문 생성 (Mutation)
        """
        mutation = gql("""
            mutation CreateOrders($orders: [OrderInput!]!) {
                createOrders(input: $orders) {
                    id
                    status
                    totalAmount
                }
            }
        """)

        variables = {"orders": orders}
        result = self.client.execute(mutation, variable_values=variables)

        return result["createOrders"]

# 사용 예시
graphql = GraphQLIntegration(
    endpoint="https://api.example.com/graphql",
    api_key="your-api-key"
)

# 단일 쿼리로 고객과 주문 정보 조회
customer = graphql.get_customer_with_orders("customer-123")
print(f"Customer: {customer['name']}")
print(f"Orders: {len(customer['orders'])}")

gRPC 통합

gRPC는 Google이 개발한 고성능 RPC 프레임워크로, HTTP/2와 Protocol Buffers를 사용합니다.

gRPC 특징

특징 설명 장점
Protocol Buffers 바이너리 직렬화 형식 JSON보다 작고 빠름 (5-10배)
HTTP/2 멀티플렉싱, 헤더 압축, 양방향 스트리밍 단일 연결로 여러 요청 동시 처리
강타입 .proto 파일로 스키마 정의 컴파일 시 타입 검증, 자동 코드 생성
스트리밍 Unary, Server Streaming, Client Streaming, Bidirectional 실시간 데이터 전송, 효율적인 대량 전송
성능 바이너리 프로토콜, HTTP/2 REST보다 7-10배 빠른 처리

gRPC 구현 예시

# Proto 파일 정의
// customer.proto
syntax = "proto3";

package customer;

service CustomerService {
    // Unary RPC: 단일 요청/응답
    rpc GetCustomer(GetCustomerRequest) returns (Customer);

    // Server Streaming: 서버가 여러 응답 스트리밍
    rpc ListCustomers(ListCustomersRequest) returns (stream Customer);

    // Client Streaming: 클라이언트가 여러 요청 스트리밍
    rpc BatchCreateCustomers(stream Customer) returns (BatchCreateResponse);

    // Bidirectional Streaming: 양방향 스트리밍
    rpc SyncCustomers(stream Customer) returns (stream SyncResponse);
}

message Customer {
    string id = 1;
    string name = 2;
    string email = 3;
    int64 created_at = 4;
}

message GetCustomerRequest {
    string id = 1;
}

message ListCustomersRequest {
    int32 page_size = 1;
    string page_token = 2;
}

message BatchCreateResponse {
    int32 created_count = 1;
    repeated string customer_ids = 2;
}

message SyncResponse {
    string customer_id = 1;
    string status = 2;
}

# Python: gRPC 클라이언트
import grpc
import customer_pb2
import customer_pb2_grpc

class GRPCCustomerClient:
    """
    gRPC를 통한 고객 데이터 통합
    """
    def __init__(self, server_address: str):
        self.channel = grpc.insecure_channel(server_address)
        self.stub = customer_pb2_grpc.CustomerServiceStub(self.channel)

    def get_customer(self, customer_id: str):
        """Unary RPC: 단일 고객 조회"""
        request = customer_pb2.GetCustomerRequest(id=customer_id)
        response = self.stub.GetCustomer(request)
        return response

    def list_customers(self, page_size: int = 100):
        """Server Streaming: 고객 목록 스트리밍"""
        request = customer_pb2.ListCustomersRequest(page_size=page_size)

        # 서버가 고객 데이터를 스트리밍
        for customer in self.stub.ListCustomers(request):
            yield customer

    def batch_create_customers(self, customers):
        """Client Streaming: 대량 고객 생성"""
        def customer_generator():
            for customer_data in customers:
                yield customer_pb2.Customer(**customer_data)

        response = self.stub.BatchCreateCustomers(customer_generator())
        return response

    def sync_customers(self, local_customers):
        """Bidirectional Streaming: 양방향 동기화"""
        def request_generator():
            for customer in local_customers:
                yield customer_pb2.Customer(**customer)

        # 양방향 스트리밍
        responses = self.stub.SyncCustomers(request_generator())

        for response in responses:
            print(f"Synced {response.customer_id}: {response.status}")

# 사용 예시
client = GRPCCustomerClient('localhost:50051')

# Server Streaming: 전체 고객 목록 조회
for customer in client.list_customers(page_size=1000):
    print(f"Customer: {customer.name}")

웹훅 (Webhook) 통합

웹훅은 이벤트가 발생할 때 서버가 미리 등록된 URL로 HTTP 콜백을 보내는 방식입니다.

웹훅 vs 폴링 비교

측면 폴링 (Polling) 웹훅 (Webhook)
데이터 흐름 클라이언트가 주기적으로 서버에 요청 서버가 이벤트 발생 시 클라이언트에 푸시
실시간성 폴링 간격만큼 지연 즉시 알림
리소스 사용 높음 (빈번한 요청) 낮음 (이벤트 발생 시만)
구현 복잡도 낮음 중간 (웹훅 엔드포인트 구현 필요)
적합한 경우 간단한 시스템, 낮은 빈도 실시간 알림, 이벤트 기반 처리

# Python: 웹훅 수신 서버 (Flask)
from flask import Flask, request, jsonify
import hmac
import hashlib

app = Flask(__name__)
WEBHOOK_SECRET = "your-webhook-secret"

def verify_webhook_signature(payload, signature):
    """
    웹훅 서명 검증 (보안)
    """
    expected_signature = hmac.new(
        WEBHOOK_SECRET.encode(),
        payload,
        hashlib.sha256
    ).hexdigest()

    return hmac.compare_digest(signature, expected_signature)

@app.route('/webhooks/orders', methods=['POST'])
def handle_order_webhook():
    """
    주문 이벤트 웹훅 처리
    """
    # 서명 검증
    signature = request.headers.get('X-Webhook-Signature')
    if not verify_webhook_signature(request.data, signature):
        return jsonify({"error": "Invalid signature"}), 401

    # 웹훅 페이로드 처리
    event = request.json

    event_type = event.get('type')
    order_data = event.get('data')

API 인증 및 보안

API 통합에서 보안은 필수적입니다. 다양한 인증 방식을 이해하고 적절히 사용해야 합니다.

인증 방식 설명 장점 단점
API Key 간단한 문자열 키 구현 간단, 빠른 검증 노출 위험, 권한 세분화 어려움
OAuth 2.0 액세스 토큰 기반 인증 표준, 권한 위임, 토큰 만료 구현 복잡도 높음
JWT JSON Web Token 무상태, 확장 가능, 클레임 포함 토큰 크기 큼, 취소 어려움
mTLS 상호 TLS 인증 강력한 보안, 양방향 인증 인증서 관리 복잡

OAuth 2.0 구현 예시

# Python: OAuth 2.0 클라이언트
from requests_oauthlib import OAuth2Session

class OAuth2APIClient:
    """
    OAuth 2.0을 사용한 API 클라이언트
    """
    def __init__(self, client_id, client_secret, authorization_base_url, token_url):
        self.client_id = client_id
        self.client_secret = client_secret
        self.authorization_base_url = authorization_base_url
        self.token_url = token_url
        self.token = None

    def authorize(self, redirect_uri):
        """
        OAuth 2.0 인증 흐름
        """
        oauth = OAuth2Session(self.client_id, redirect_uri=redirect_uri)

        # 1. 인증 URL 생성
        authorization_url, state = oauth.authorization_url(
            self.authorization_base_url
        )

        print(f'Please go to {authorization_url} and authorize access.')
        authorization_response = input('Enter the full callback URL: ')

        # 2. 액세스 토큰 획득
        self.token = oauth.fetch_token(
            self.token_url,
            authorization_response=authorization_response,
            client_secret=self.client_secret
        )

        return self.token

    def refresh_token(self):
        """
        토큰 갱신
        """
        oauth = OAuth2Session(self.client_id, token=self.token)

        self.token = oauth.refresh_token(
            self.token_url,
            client_id=self.client_id,
            client_secret=self.client_secret
        )

        return self.token

    def make_request(self, url):
        """
        인증된 요청 수행
        """
        oauth = OAuth2Session(self.client_id, token=self.token)
        response = oauth.get(url)

        return response.json()

API Rate Limiting 처리

대부분의 API는 요청 제한(Rate Limiting)을 두고 있습니다. 이를 올바르게 처리하는 것이 중요합니다.

# Python: Rate Limiting 처리
import time
from functools import wraps

class RateLimiter:
    """
    API Rate Limiting 핸들러
    """
    def __init__(self, max_calls, time_window):
        self.max_calls = max_calls
        self.time_window = time_window
        self.calls = []

    def __call__(self, func):
        @wraps(func)
        def wrapper(*args, **kwargs):
            now = time.time()

            # 시간 윈도우 밖의 호출 제거
            self.calls = [call for call in self.calls if now - call < self.time_window]

            if len(self.calls) >= self.max_calls:
                # Rate limit 초과: 대기
                sleep_time = self.time_window - (now - self.calls[0])
                print(f"Rate limit reached. Sleeping for {sleep_time:.2f} seconds...")
                time.sleep(sleep_time)
                self.calls = []

            self.calls.append(now)
            return func(*args, **kwargs)

        return wrapper

# 사용 예시
@RateLimiter(max_calls=100, time_window=60)  # 분당 100회
def api_call():
    # API 호출
    pass

API 에러 처리 및 재시도

안정적인 API 통합을 위해서는 적절한 에러 처리와 재시도 로직이 필수입니다.

# Python: 지수 백오프를 사용한 재시도
import requests
from requests.adapters import HTTPAdapter
from urllib3.util.retry import Retry

def create_retry_session(
    retries=3,
    backoff_factor=0.3,
    status_forcelist=(500, 502, 504)
):
    """
    자동 재시도 기능이 있는 Session 생성
    """
    session = requests.Session()

    retry = Retry(
        total=retries,
        read=retries,
        connect=retries,
        backoff_factor=backoff_factor,
        status_forcelist=status_forcelist,
        allowed_methods=["GET", "POST", "PUT", "DELETE"]
    )

    adapter = HTTPAdapter(max_retries=retry)
    session.mount('http://', adapter)
    session.mount('https://', adapter)

    return session

# 사용 예시
session = create_retry_session()
response = session.get('https://api.example.com/data')

모니터링 및 관찰성

API 통합의 건강성을 모니터링하는 핵심 메트릭:

  • 응답 시간: P50, P95, P99 레이턴시
  • 성공률: 2xx 응답 비율
  • 에러율: 4xx, 5xx 에러 비율
  • Rate Limit 상태: 남은 요청 수, 리셋 시간
  • 처리량: 초당 요청 수 (RPS)
  • 재시도 횟수: 재시도가 필요한 요청 비율

요약

이 장에서 다룬 주요 내용:

  • REST API: 가장 보편적, HTTP 기반, 리소스 중심
  • GraphQL: 유연한 쿼리, 필요한 데이터만 요청
  • gRPC: 고성능, HTTP/2, Protocol Buffers, 스트리밍
  • 웹훅: 이벤트 기반, 푸시 방식, 실시간 알림
  • 인증: API Key, OAuth 2.0, JWT, mTLS
  • 에러 처리: 재시도, 지수 백오프, Rate Limiting

각 API 유형은 고유한 장단점과 적합한 사용 사례가 있습니다. 프로젝트 요구사항에 따라 적절한 API 기술을 선택하고, 보안과 안정성을 고려한 구현이 중요합니다.
