Chapter 7: Data Integration Tools and Platforms

Hongik Ingan (弘益人間)

"Benefit All Humanity"

The right tools empower teams to build robust data pipelines efficiently. Understanding the landscape of integration tools helps organizations choose solutions that best serve their needs and capabilities.

7.1 Tool Categories

Data integration tools span multiple categories, each serving specific needs:

Category Purpose Examples
Ingestion (ELT) Extract and load data from sources to warehouses Fivetran, Airbyte, Stitch, Meltano
Transformation Transform data in the warehouse dbt, Dataform, Matillion
Orchestration Schedule and monitor data workflows Airflow, Prefect, Dagster
Streaming Real-time event streaming Kafka, Kinesis, Pub/Sub
CDC Capture database changes Debezium, Maxwell, Oracle GoldenGate
Reverse ETL Push warehouse data to operational tools Hightouch, Census, Polytomic
Data Quality Monitor and improve data quality Great Expectations, Soda, Monte Carlo

7.2 Ingestion Tools (ELT)

7.2.1 Fivetran

Fivetran is the leading automated ELT platform with 500+ pre-built connectors.

Key Features:

  • Fully automated: No code, no maintenance
  • 500+ connectors for SaaS apps, databases, files, events
  • Automatic schema drift detection and handling
  • Incremental updates with CDC where available
  • Column-level lineage and impact analysis
  • Transformation with dbt integration

Pricing: Monthly active rows (MAR) model

7.2.2 Airbyte

Airbyte is the open-source alternative to Fivetran with 300+ connectors.

Advantages:

  • Open source with self-hosted option
  • 300+ connectors with active community
  • Custom connector development kit (CDK)
  • Supports ELT to data warehouses, lakes, databases
  • Cloud and self-hosted deployments
  • Free for self-hosted, usage-based pricing for cloud

7.2.3 Comparison: Fivetran vs Airbyte

Aspect Fivetran Airbyte
Deployment Cloud only (SaaS) Cloud or self-hosted
Connectors 500+ pre-built, highly reliable 300+ connectors, growing fast
Pricing Higher (enterprise SaaS) Lower (open source or cloud)
Customization Limited to available connectors Build custom connectors easily
Maturity Highly mature, battle-tested Rapidly maturing
Best For Enterprises wanting zero-maintenance Cost-sensitive or custom connector needs

7.3 Transformation Tools

7.3.1 dbt (data build tool)

dbt has become the standard for SQL-based transformations in modern data stacks.

Core Capabilities:

  • SQL Transformations: Write SELECT statements, dbt handles DDL
  • Modularity: Reference models with {{ ref() }}, build DAGs automatically
  • Testing: Built-in tests (not null, unique, accepted values, relationships)
  • Documentation: Auto-generated docs with lineage graphs
  • Macros: Reusable Jinja SQL templates
  • Packages: Install community packages (dbt_utils, audit_helper)
  • Incremental Models: Process only new/changed data
  • Snapshots: Type 2 slowly changing dimensions automatically

Example dbt project structure:

my_dbt_project/
├── models/
│   ├── staging/
│   │   ├── stg_customers.sql
│   │   └── stg_orders.sql
│   └── marts/
│       └── customers/
│           └── customer_lifetime_value.sql
├── tests/
│   └── assert_positive_ltv.sql
├── macros/
│   └── custom_macros.sql
└── dbt_project.yml

7.4 Orchestration Tools

7.4.1 Apache Airflow

Apache Airflow is the most popular open-source workflow orchestration platform.

Key Features:

  • Python-based DAG (Directed Acyclic Graph) definitions
  • Rich web UI for monitoring and troubleshooting
  • 100+ operators (Bash, Python, SQL, Spark, Kubernetes)
  • Dynamic pipeline generation with Python
  • Backfilling: re-run historical dates
  • Extensive plugin ecosystem
  • Managed options: Astronomer, AWS MWAA, Google Cloud Composer

7.4.2 Prefect

Prefect is a modern workflow orchestration platform with focus on developer experience.

Advantages over Airflow:

  • Native Python (no DAG bag loading issues)
  • Dynamic workflows (no complex templating)
  • Hybrid execution model (cloud UI, local/cloud execution)
  • Automatic retries and caching
  • Modern UI with better observability
  • Cloud offering with free tier

7.4.3 Dagster

Dagster focuses on data-aware orchestration with software-defined assets.

Unique Features:

  • Software-Defined Assets: Declare assets (tables, files) not just tasks
  • Asset Lineage: Automatic lineage tracking across assets
  • Type System: Type-checked data flows
  • Testing: First-class support for testing pipelines
  • dbt Integration: Treat dbt models as Dagster assets
  • Dagster Cloud: Managed orchestration platform

7.4.4 Orchestrator Comparison

Tool Best For Learning Curve Maturity
Airflow Complex workflows, large teams Steep Very mature
Prefect Modern Python workflows Low Mature
Dagster Data assets, software engineering practices Medium Growing
dbt Cloud Simple dbt scheduling only Very low Mature for dbt

7.5 Streaming and CDC Tools

7.5.1 Apache Kafka Ecosystem

Kafka provides a complete ecosystem for event streaming:

  • Kafka Core: Distributed log for events
  • Kafka Connect: 100+ source/sink connectors
  • Kafka Streams: Stream processing library
  • ksqlDB: SQL interface for stream processing
  • Schema Registry: Schema versioning and compatibility

7.5.2 Debezium for CDC

Debezium supports CDC for:

  • MySQL and MariaDB
  • PostgreSQL
  • MongoDB
  • SQL Server
  • Oracle
  • Cassandra

Debezium publishes database changes to Kafka topics in near real-time.

7.6 Reverse ETL Tools

Reverse ETL syncs data from warehouses back to operational tools (CRM, marketing, support).

7.6.1 Popular Reverse ETL Tools

Tool Strengths Pricing
Hightouch 200+ destinations, visual mapper, audience sync Usage-based
Census Strong governance, field-level permissions, orchestration Usage-based

Use Cases:

  • Sync customer segments from warehouse to marketing tools (Braze, Iterable)

7.7 Data Quality Tools

7.7.1 Great Expectations

Great Expectations is an open-source data quality framework.

Features:

  • Expectation-based testing (expect_column_values_to_not_be_null)
  • Data docs: auto-generated documentation
  • Integration with Airflow, dbt, notebooks
  • Profiling: auto-generate expectations from data
  • Data validation checkpoints

7.7.2 Data Observability Platforms

7.8 Choosing the Right Tools

Decision framework for tool selection:

7.8.1 For Small Teams (< 10 people)

Recommended stack:

  • Ingestion: Fivetran or Airbyte (managed)
  • Warehouse: Snowflake, BigQuery, or Redshift
  • Transformation: dbt Cloud
  • Orchestration: dbt Cloud scheduler (start) or Prefect Cloud
  • BI: Looker, Tableau, or Metabase

Rationale: Minimize operational overhead with managed services, focus on business logic.

7.8.2 For Mid-Size Teams (10-50 people)

Recommended stack:

  • Ingestion: Fivetran + Airbyte (mix of managed + custom)
  • Warehouse: Snowflake or Databricks
  • Transformation: dbt Core with Airflow or dbt Cloud
  • Orchestration: Airflow (Astronomer or MWAA)
  • Data Quality: Great Expectations or Soda
  • Reverse ETL: Hightouch or Census
  • BI: Looker or Tableau

7.8.3 For Large Enterprises (50+ people)

Recommended stack:

  • Ingestion: Fivetran, Airbyte, custom Spark jobs
  • Warehouse: Snowflake, Databricks, or multi-warehouse
  • Transformation: dbt + Spark for complex logic
  • Orchestration: Airflow or Dagster at scale
  • Streaming: Kafka + Flink for real-time
  • Data Quality: Monte Carlo + Great Expectations
  • Governance: Alation, Collibra for data catalogs
  • Observability: Datadog, custom Grafana dashboards

Summary

The modern data integration landscape offers specialized tools for each stage:

  • Ingestion: Fivetran (ease) vs Airbyte (flexibility)
  • Transformation: dbt is the clear standard
  • Orchestration: Airflow (mature) vs Prefect (modern) vs Dagster (asset-oriented)
  • Streaming: Kafka ecosystem dominates
  • Reverse ETL: Hightouch, Census for warehouse-to-SaaS sync
  • Data Quality: Great Expectations, Monte Carlo, Soda

Choose tools based on team size, budget, technical maturity, and operational capabilities. Start simple with managed services, add complexity as needs grow.
