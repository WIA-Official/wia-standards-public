7.1 데이터 품질 차원

데이터 품질은 6가지 핵심 차원으로 측정됩니다. 각 차원은 독립적으로 평가되며, 전체 품질 점수는 가중 평균으로 계산됩니다.

// 품질 점수 계산 function calculateQualityScore(metrics: QualityMetrics): number { const weights = { accuracy: 0.25, completeness: 0.20, consistency: 0.20, timeliness: 0.15, validity: 0.10, uniqueness: 0.10 }; return Object.entries(metrics).reduce((score, [key, value]) => { return score + (value * weights[key]); }, 0); } // 사용 예시 const metrics = { accuracy: 0.99, completeness: 0.96, consistency: 0.97, timeliness: 0.98, validity: 0.99, uniqueness: 0.99 }; const score = calculateQualityScore(metrics); console.log(`품질 점수: ${(score * 100).toFixed(2)}%`); // 출력: 품질 점수: 97.85%

class BusinessRuleEngine { private rules: Rule[] = []; addRule(rule: Rule) { this.rules.push(rule); } async validate(data: any): Promise<RuleValidationResult> { const violations = []; for (const rule of this.rules) { const result = await rule.check(data); if (!result.passed) { violations.push({ ruleId: rule.id, message: result.message, severity: rule.severity }); } } return { passed: violations.length === 0, violations }; } } // 규칙 예시 const ageRule: Rule = { id: 'AGE_EMPLOYMENT_CHECK', description: '고용 가능 나이 확인', severity: 'error', check: async (data) => { if (data.employed && data.age < 16) { return { passed: false, message: '16세 미만은 고용될 수 없습니다' }; } return { passed: true }; } };

class DataProfiler { async profileColumn( dataset: any[], columnName: string ): Promise<ColumnProfile> { const values = dataset.map(row => row[columnName]); const nonNullValues = values.filter(v => v != null); return { totalCount: values.length, nullCount: values.length - nonNullValues.length, nullPercentage: ((values.length - nonNullValues.length) / values.length) * 100, uniqueCount: new Set(nonNullValues).size, dataTypes: this.detectDataTypes(nonNullValues), patterns: this.detectPatterns(nonNullValues), statistics: this.calculateStatistics(nonNullValues) }; } private calculateStatistics(values: any[]) { const numbers = values.filter(v => typeof v === 'number'); if (numbers.length === 0) return null; return { min: Math.min(...numbers), max: Math.max(...numbers), mean: numbers.reduce((a, b) => a + b, 0) / numbers.length, median: this.calculateMedian(numbers), stdDev: this.calculateStdDev(numbers) }; } }

7.4 이상 탐지

머신러닝 기반 이상 탐지로 데이터 품질 문제를 자동으로 식별합니다.

7.4.1 통계 기반 이상 탐지

class AnomalyDetector { /** * Z-Score 기반 이상치 탐지 * |z| > 3이면 이상치로 판단 */ detectOutliers(values: number[], threshold: number = 3): number[] { const mean = values.reduce((a, b) => a + b) / values.length; const stdDev = Math.sqrt( values.reduce((sum, v) => sum + Math.pow(v - mean, 2), 0) / values.length ); return values.filter(v => { const zScore = Math.abs((v - mean) / stdDev); return zScore > threshold; }); } /** * IQR (Interquartile Range) 기반 */ detectOutliersIQR(values: number[]): {value: number, index: number}[] { const sorted = [...values].sort((a, b) => a - b); const q1 = sorted[Math.floor(sorted.length * 0.25)]; const q3 = sorted[Math.floor(sorted.length * 0.75)]; const iqr = q3 - q1; const lowerBound = q1 - 1.5 * iqr; const upperBound = q3 + 1.5 * iqr; return values .map((value, index) => ({ value, index })) .filter(({ value }) => value < lowerBound || value > upperBound); } }

7.4.2 머신러닝 기반 탐지

// Isolation Forest 알고리즘 class IsolationForest { private trees: IsolationTree[] = []; private numTrees: number = 100; fit(data: number[][]) { for (let i = 0; i < this.numTrees; i++) { const sample = this.randomSample(data); const tree = this.buildTree(sample); this.trees.push(tree); } } predict(point: number[]): number { const avgPathLength = this.trees.reduce((sum, tree) => { return sum + tree.pathLength(point); }, 0) / this.numTrees; // 정규화된 이상 점수 (0~1, 1에 가까울수록 이상) return Math.pow(2, -avgPathLength / this.averagePathLength); } }

7.5 데이터 정제 및 변환

발견된 품질 문제를 자동으로 수정하는 정제 파이프라인을 구성합니다.

정제 작업방법적용 시점

class DataCleaner { /** * 중복 레코드 탐지 및 제거 */ async deduplicateRecords( records: any[], keyFields: string[] ): Promise<DeduplicationResult> { const seen = new Map(); const duplicates = []; const unique = []; for (const record of records) { const key = keyFields.map(f => record[f]).join('|'); if (seen.has(key)) { duplicates.push({ record, duplicateOf: seen.get(key) }); } else { seen.set(key, record); unique.push(record); } } return { unique, duplicates }; } /** * Fuzzy 중복 탐지 */ async fuzzyDeduplicate( records: any[], field: string, threshold: number = 0.8 ): Promise<FuzzyMatch[]> { const matches = []; for (let i = 0; i < records.length; i++) { for (let j = i + 1; j < records.length; j++) { const similarity = this.levenshteinSimilarity( records[i][field], records[j][field] ); if (similarity >= threshold) { matches.push({ record1: records[i], record2: records[j], similarity }); } } } return matches; } private levenshteinSimilarity(str1: string, str2: string): number { const distance = this.levenshteinDistance(str1, str2); const maxLength = Math.max(str1.length, str2.length); return 1 - (distance / maxLength); } }

7.6 품질 모니터링 대시보드

실시간으로 데이터 품질을 모니터링하고 시각화합니다.

대시보드 핵심 메트릭

  • 실시간 품질 점수: 현재 데이터셋의 전체 품질 점수 (0-100)
  • 차원별 점수: 6가지 품질 차원별 세부 점수
  • 추세 분석: 시간에 따른 품질 변화 그래프
  • 문제 알림: 품질 기준 미달 시 즉시 알림
  • 수정 제안: AI 기반 자동 수정 방안 제시
// 실시간 품질 모니터링 class QualityMonitor { private metrics: QualityMetricsCollector; private alertManager: AlertManager; async monitor(interval: number = 60000) { setInterval(async () => { const currentMetrics = await this.metrics.collect(); const qualityScore = calculateQualityScore(currentMetrics); // 품질 점수가 90% 미만이면 경고 if (qualityScore < 0.90) { await this.alertManager.sendAlert({ level: 'warning', message: `데이터 품질 저하: ${(qualityScore * 100).toFixed(2)}%`, metrics: currentMetrics }); } // 품질 점수가 80% 미만이면 긴급 if (qualityScore < 0.80) { await this.alertManager.sendAlert({ level: 'critical', message: `데이터 품질 심각: ${(qualityScore * 100).toFixed(2)}%`, metrics: currentMetrics, actions: ['즉시 조치 필요', '데이터 정제 실행'] }); } // 메트릭 저장 await this.saveMetrics(currentMetrics, qualityScore); }, interval); } }

핵심 요점

  • 데이터 품질은 정확성(25%), 완전성(20%), 일관성(20%), 적시성(15%), 유효성(10%), 유일성(10%)의 6가지 차원으로 측정됩니다.
  • 검증은 JSON Schema를 통한 스키마 검증과 비즈니스 규칙 엔진을 통한 규칙 검증의 두 단계로 수행됩니다.
  • 데이터 프로파일링은 열 프로파일링, 패턴 분석, 통계 분석, 관계 분석의 4가지 유형으로 구성됩니다.
  • 이상 탐지는 Z-Score, IQR 등 통계적 방법과 Isolation Forest 등 머신러닝 방법을 결합합니다.
  • 데이터 정제는 중복 제거, NULL 처리, 표준화, 정규화 작업을 포함하며, Fuzzy matching으로 유사 중복을 탐지합니다.
  • 실시간 품질 모니터링으로 품질 점수가 90% 미만이면 경고, 80% 미만이면 긴급 알림을 발송합니다.
