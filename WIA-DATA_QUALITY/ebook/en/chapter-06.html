✅ Chapter 6: Phase 3 - Protocol

6.1 Overview of Phase 3

Phase 3 of the WIA-DATA_QUALITY Standard introduces real-time protocols for continuous data quality monitoring. While Phases 1 and 2 focus on data formats and request/response APIs, Phase 3 enables streaming quality metrics, live dashboards, and proactive alerting. This phase is essential for organizations that need to detect and respond to quality issues as they occur rather than discovering them in periodic batch assessments.

The protocols defined in Phase 3 support multiple use cases: streaming quality validation for real-time data pipelines, live quality dashboards that update without page refresh, proactive alerts that notify stakeholders of quality degradation, and event-driven quality workflows that automate remediation. These capabilities transform data quality from a reactive discipline to a proactive one.

Phase 3 builds on established streaming technologies including WebSocket for bidirectional communication, Server-Sent Events (SSE) for unidirectional streaming, and Apache Kafka for enterprise-scale event streaming. Organizations can choose the technology that best fits their infrastructure while maintaining interoperability through standard message formats.

The protocols are designed for both producer and consumer roles. Quality services can publish quality events (acting as producers), and applications can subscribe to quality streams (acting as consumers). This publish-subscribe model enables loose coupling between quality services and the applications that depend on quality information.

6.4 WebSocket Protocol

WebSocket provides bidirectional, real-time communication for interactive quality applications. The Phase 3 WebSocket protocol defines connection establishment, subscription management, and message formats.

Connection and Authentication

// Establish WebSocket connection
const ws = new WebSocket('wss://quality.example.com/v1/stream');

// Authenticate after connection
ws.onopen = () => {
  ws.send(JSON.stringify({
    type: 'authenticate',
    token: 'Bearer eyJhbGciOiJIUzI1NiIs...'
  }));
};

// Handle authentication response
ws.onmessage = (event) => {
  const message = JSON.parse(event.data);
  if (message.type === 'authenticated') {
    console.log('Connected successfully');
    subscribeToTopics();
  }
};

Subscription Management

// Subscribe to quality events
function subscribeToTopics() {
  ws.send(JSON.stringify({
    type: 'subscribe',
    topics: [
      'quality.threshold.breached/*',
      'quality.metric.updated/customer_master',
      'quality.anomaly.detected/*'
    ],
    options: {
      fromOffset: 'latest',
      includeHistory: 300  // last 5 minutes
    }
  }));
}

// Handle incoming events
ws.onmessage = (event) => {
  const message = JSON.parse(event.data);
  switch (message.type) {
    case 'quality.threshold.breached':
      handleThresholdBreach(message.data);
      break;
    case 'quality.metric.updated':
      updateDashboard(message.data);
      break;
    case 'quality.anomaly.detected':
      investigateAnomaly(message.data);
      break;
  }
};

6.5 Server-Sent Events (SSE)

For applications that only need to receive quality events (not send commands), Server-Sent Events provide a simpler alternative to WebSocket. SSE is particularly useful for dashboard applications and monitoring displays.

const eventSource = new EventSource(
  'https://quality.example.com/v1/events?topics=quality.metric.updated/*',
  { headers: { 'Authorization': 'Bearer token...' } }
);

eventSource.addEventListener('quality.metric.updated', (event) => {
  const data = JSON.parse(event.data);
  updateMetricDisplay(data.dimension, data.metric.current);
});

eventSource.addEventListener('quality.threshold.breached', (event) => {
  const data = JSON.parse(event.data);
  showAlert(data.dimension, data.metric);
});

eventSource.onerror = (error) => {
  console.error('SSE connection error:', error);
  // Automatic reconnection is handled by the browser
};

6.8 Real-Time Dashboard Integration

Phase 3 provides patterns for building real-time quality dashboards. These patterns ensure efficient updates, handle connection failures gracefully, and maintain consistent state across dashboard instances.

// Dashboard state management with real-time updates
class QualityDashboard {
  constructor() {
    this.metrics = new Map();
    this.eventSource = null;
  }

  connect() {
    this.eventSource = new EventSource('/v1/events?topics=quality.*');

    this.eventSource.addEventListener('quality.metric.updated', (e) => {
      const data = JSON.parse(e.data);
      this.metrics.set(data.dimension, data.metric.current);
      this.render();
    });

    // Periodic full refresh for data consistency
    setInterval(() => this.fullRefresh(), 60000);
  }

  async fullRefresh() {
    const response = await fetch('/v1/assess/current');
    const data = await response.json();
    for (const [dim, score] of Object.entries(data.dimensions)) {
      this.metrics.set(dim, score.score);
    }
    this.render();
  }

  render() {
    // Update dashboard UI with current metrics
    for (const [dimension, score] of this.metrics) {
      document.getElementById(`metric-${dimension}`)
        .textContent = `${score.toFixed(1)}%`;
    }
  }
}
