✅ Chapter 5: Phase 2 - API Interface

5.1 Overview of Phase 2

Phase 2 of the WIA-DATA_QUALITY Standard defines RESTful APIs for programmatic access to data quality operations. While Phase 1 establishes the data formats, Phase 2 specifies how to interact with quality services—profiling data, validating records, calculating scores, and managing quality rules. This phase enables automation of quality processes that would otherwise require manual intervention.

The APIs defined in Phase 2 follow REST architectural principles and OpenAPI 3.0 specification for documentation. This ensures wide compatibility with existing development tools, testing frameworks, and API management platforms. Organizations can implement the APIs on any technology stack that supports HTTP/REST communication.

Phase 2 also includes SDK specifications for popular programming languages including TypeScript/JavaScript, Python, Java, and C#. These SDKs provide strongly-typed interfaces that simplify integration and reduce the likelihood of errors. SDK implementations handle authentication, error handling, and retry logic, allowing developers to focus on business logic.

The API design emphasizes both batch and real-time operations. Large-scale data quality assessments can be submitted as asynchronous jobs, while individual record validation supports synchronous sub-second response times. This flexibility ensures the APIs can support diverse use cases from periodic batch quality checks to real-time data pipeline validation.

5.2 API Architecture

The Phase 2 API architecture organizes endpoints into logical resource groups, each handling a specific aspect of data quality management. This organization makes the API intuitive to navigate and enables fine-grained access control.

Resource Group Base Path Purpose
Profile /api/v1/profile Data profiling operations
Validate /api/v1/validate Record validation
Assess /api/v1/assess Quality assessment and scoring
Rules /api/v1/rules Quality rule management
Reports /api/v1/reports Quality reports and history
Jobs /api/v1/jobs Async job management

5.3 Profiling API

The Profiling API enables discovery of data characteristics and quality issues without predefined rules. Profiling analyzes data to determine statistics, patterns, and anomalies that inform quality management decisions.

Create Profile Job

POST /api/v1/profile/jobs
Content-Type: application/json
Authorization: Bearer {token}

{
  "source": {
    "type": "database",
    "connection": "postgresql://host/db",
    "table": "customers"
  },
  "options": {
    "sampleSize": 100000,
    "analyzePatterns": true,
    "detectOutliers": true,
    "computeDistributions": true
  },
  "fields": ["*"]  // or specific field list
}

Response:
{
  "jobId": "prof-12345-67890",
  "status": "queued",
  "estimatedDuration": "PT5M",
  "createdAt": "2025-01-08T10:30:00Z"
}

5.4 Validation API

The Validation API applies quality rules to data records, returning detailed results about rule violations and suggested corrections. It supports both single-record synchronous validation and batch asynchronous validation.

Single Record Validation

POST /api/v1/validate/record
Content-Type: application/json

{
  "schema": "customer_v1",
  "record": {
    "customer_id": "CUST-001",
    "email": "invalid-email",
    "phone": "555-1234",
    "created_date": "2025-01-08"
  }
}

Response:
{
  "valid": false,
  "score": 75.0,
  "violations": [
    {
      "field": "email",
      "rule": "email-format",
      "severity": "high",
      "message": "Invalid email format",
      "value": "invalid-email",
      "suggestion": null
    }
  ],
  "dimensions": {
    "completeness": 100,
    "validity": 75,
    "accuracy": null
  }
}

Batch Validation

5.5 Assessment API

The Assessment API calculates comprehensive quality scores for datasets, combining multiple dimension scores into overall quality metrics. It supports configurable scoring profiles and threshold-based pass/fail determinations.

POST /api/v1/assess
Content-Type: application/json

{
  "target": {
    "type": "dataset",
    "identifier": "customer_master"
  },
  "profile": "standard",  // or "strict", "lenient", custom
  "dimensions": ["completeness", "accuracy", "consistency", "validity", "uniqueness"],
  "options": {
    "includeDetails": true,
    "compareToBaseline": "2025-01-01",
    "generateRecommendations": true
  }
}

Response:
{

  "timestamp": "2025-01-08T10:45:00Z",

5.6 Rules API

The Rules API manages quality rules including creation, modification, testing, and deployment. It supports both standard rules from the WIA library and custom organization-specific rules.

Endpoint Method Purpose
/rules GET List all rules
/rules POST Create new rule
/rules/{id} GET Get rule details
/rules/{id} PUT Update rule
/rules/{id}/test POST Test rule against sample data
/rules/{id}/deploy POST Deploy rule to production 
POST /api/v1/rules
Content-Type: application/json

{
  "name": "valid-customer-status",
  "description": "Customer status must be one of valid values",
  "type": "lookup",
  "field": "status",
  "parameters": {
    "values": ["active", "inactive", "suspended", "pending"]
  },
  "severity": "high",
  "schemas": ["customer_v1", "customer_v2"]
}

5.7 Authentication and Authorization

Phase 2 specifies OAuth 2.0 for authentication and a role-based access control (RBAC) model for authorization. This ensures secure access to quality operations while supporting integration with enterprise identity providers.

Role Permissions
quality-viewer Read profiles, assessments, reports
quality-analyst All viewer + create profiles, run validations
quality-steward All analyst + manage rules, remediate issues
quality-admin All permissions + manage users, configure system

5.8 SDK Overview

The WIA-DATA_QUALITY SDKs provide idiomatic interfaces for popular programming languages. Here's an example using the TypeScript SDK:

import { WIADataQuality } from '@wia/data-quality';

const dq = new WIADataQuality({
  baseUrl: 'https://api.example.com',
  apiKey: process.env.WIA_API_KEY
});

// Profile a dataset
const profile = await dq.profile.create({
  source: { type: 'database', connection: connString, table: 'customers' },
  options: { analyzePatterns: true }
});

// Validate a record
const result = await dq.validate.record({
  schema: 'customer_v1',
  record: customerData
});

if (!result.valid) {
  console.log('Violations:', result.violations);
}

// Run assessment
const assessment = await dq.assess.dataset({
  identifier: 'customer_master',
  profile: 'standard'
});

console.log(`Quality Score: ${assessment.overallScore}%`);
