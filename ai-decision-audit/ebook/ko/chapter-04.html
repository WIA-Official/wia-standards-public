Chapter 4: 감사 추적 관리

불변적이고 검증 가능한 감사 기록 유지

감사 추적의 중요성

감사 추적(Audit Trail)은 AI 의사결정의 완전하고 불변적인 역사적 기록입니다. 효과적인 감사 추적 관리는 규제 준수, 사고 조사, 시스템 개선 및 책임성 확보에 필수적입니다.

강력한 감사 추적은 다음을 보장합니다:

  • 불변성: 기록이 변조되지 않았음을 보장
  • 완전성: 모든 중요한 결정이 캡처됨
  • 검증성: 기록의 진위를 확인 가능
  • 추적성: 결정의 전체 체인을 재구성 가능
  • 내구성: 장기간 안정적으로 보존

불변 스토리지 아키텍처

감사 추적의 불변성을 보장하는 스토리지 아키텍처:

추가 전용 로그 구조

기록은 추가만 가능하며 수정이나 삭제가 불가능합니다:

// 추가 전용 감사 스토어
class AppendOnlyAuditStore {
  private logs: AuditRecord[] = [];
  private sealed: boolean = false;

  // 새 감사 기록 추가
  append(record: AuditRecord): string {
    if (this.sealed) {
      throw new Error('스토어가 봉인되었습니다');
    }

    // 이전 기록의 해시를 포함하여 체인 생성
    const previousHash = this.getLastHash();
    const enriched = {
      ...record,
      sequence: this.logs.length,
      previousHash: previousHash,
      timestamp: new Date().toISOString()
    };

    // 현재 기록의 해시 계산
    enriched.hash = this.calculateHash(enriched);

    this.logs.push(enriched);
    return enriched.hash;
  }

  // 마지막 해시 가져오기
  private getLastHash(): string {
    if (this.logs.length === 0) {
      return '0'.repeat(64); // 제네시스 해시
    }
    return this.logs[this.logs.length - 1].hash;
  }

  // 해시 계산
  private calculateHash(record: any): string {
    const crypto = require('crypto');
    const content = JSON.stringify(record);
    return crypto.createHash('sha256').update(content).digest('hex');
  }

  // 체인 무결성 검증
  verify(): VerificationResult {
    for (let i = 1; i < this.logs.length; i++) {
      const current = this.logs[i];
      const previous = this.logs[i - 1];

      // 이전 해시 검증
      if (current.previousHash !== previous.hash) {
        return {
          valid: false,
          error: `기록 ${i}의 체인이 끊어졌습니다`,
          position: i
        };
      }

      // 해시 재계산 및 검증
      const recalculated = this.calculateHash({
        ...current,
        hash: undefined
      });

      if (recalculated !== current.hash) {
        return {
          valid: false,
          error: `기록 ${i}이 변조되었습니다`,
          position: i
        };
      }
    }

    return { valid: true };
  }
}

머클 트리 기반 검증

효율적인 무결성 검증을 위한 머클 트리 구조:

// 머클 트리 구현
class MerkleAuditTree {
  private leaves: string[] = [];
  private tree: string[][] = [];

  // 감사 기록 추가
  addRecord(record: AuditRecord): string {
    const hash = this.hashRecord(record);
    this.leaves.push(hash);
    this.rebuildTree();
    return hash;
  }

  // 트리 재구성
  private rebuildTree(): void {
    this.tree = [this.leaves];

    while (this.tree[this.tree.length - 1].length > 1) {
      const currentLevel = this.tree[this.tree.length - 1];
      const nextLevel: string[] = [];

      for (let i = 0; i < currentLevel.length; i += 2) {
        const left = currentLevel[i];
        const right = i + 1 < currentLevel.length
          ? currentLevel[i + 1]
          : left;

        nextLevel.push(this.hashPair(left, right));
      }

      this.tree.push(nextLevel);
    }
  }

  // 루트 해시 가져오기
  getRootHash(): string {
    if (this.tree.length === 0) return '';
    return this.tree[this.tree.length - 1][0];
  }

  // 머클 증명 생성
  getProof(index: number): MerkleProof {
    const proof: MerkleProof = {
      leaf: this.leaves[index],
      path: []
    };

    let currentIndex = index;

    for (let level = 0; level < this.tree.length - 1; level++) {
      const isRightNode = currentIndex % 2 === 1;
      const siblingIndex = isRightNode
        ? currentIndex - 1
        : currentIndex + 1;

      if (siblingIndex < this.tree[level].length) {
        proof.path.push({
          hash: this.tree[level][siblingIndex],
          position: isRightNode ? 'left' : 'right'
        });
      }

      currentIndex = Math.floor(currentIndex / 2);
    }

    return proof;
  }

  // 증명 검증
  verifyProof(proof: MerkleProof, rootHash: string): boolean {
    let currentHash = proof.leaf;

    for (const step of proof.path) {
      if (step.position === 'left') {
        currentHash = this.hashPair(step.hash, currentHash);
      } else {
        currentHash = this.hashPair(currentHash, step.hash);
      }
    }

    return currentHash === rootHash;
  }

  private hashRecord(record: AuditRecord): string {
    const crypto = require('crypto');
    return crypto.createHash('sha256')
      .update(JSON.stringify(record))
      .digest('hex');
  }

  private hashPair(left: string, right: string): string {
    const crypto = require('crypto');
    return crypto.createHash('sha256')
      .update(left + right)
      .digest('hex');
  }
}

블록체인 기반 감사 추적

분산 원장 기술을 활용한 감사 추적:

// 블록체인 기반 감사 시스템
class BlockchainAuditTrail {
  private chain: Block[] = [];
  private difficulty: number = 4;

  constructor() {
    // 제네시스 블록 생성
    this.chain.push(this.createGenesisBlock());
  }

  private createGenesisBlock(): Block {
    return {
      index: 0,
      timestamp: '2024-01-01T00:00:00Z',
      records: [],
      previousHash: '0',
      hash: this.calculateBlockHash({
        index: 0,
        timestamp: '2024-01-01T00:00:00Z',
        records: [],
        previousHash: '0',
        nonce: 0
      }),
      nonce: 0
    };
  }

  // 새 블록 추가
  addBlock(records: AuditRecord[]): Block {
    const previousBlock = this.chain[this.chain.length - 1];

    const newBlock: Block = {
      index: previousBlock.index + 1,
      timestamp: new Date().toISOString(),
      records: records,
      previousHash: previousBlock.hash,
      hash: '',
      nonce: 0
    };

    // 작업 증명 (Proof of Work)
    newBlock.hash = this.mineBlock(newBlock);

    this.chain.push(newBlock);
    return newBlock;
  }

  // 블록 채굴 (작업 증명)
  private mineBlock(block: Block): string {
    const target = '0'.repeat(this.difficulty);

    while (true) {
      const hash = this.calculateBlockHash(block);

      if (hash.startsWith(target)) {
        return hash;
      }

      block.nonce++;
    }
  }

  // 블록 해시 계산
  private calculateBlockHash(block: Partial): string {
    const crypto = require('crypto');
    const content = JSON.stringify({
      index: block.index,
      timestamp: block.timestamp,
      records: block.records,
      previousHash: block.previousHash,
      nonce: block.nonce
    });

    return crypto.createHash('sha256').update(content).digest('hex');
  }

  // 체인 검증
  validateChain(): boolean {
    for (let i = 1; i < this.chain.length; i++) {
      const current = this.chain[i];
      const previous = this.chain[i - 1];

      // 현재 블록 해시 검증
      const recalculated = this.calculateBlockHash(current);
      if (current.hash !== recalculated) {
        return false;
      }

      // 이전 블록 링크 검증
      if (current.previousHash !== previous.hash) {
        return false;
      }

      // 작업 증명 검증
      const target = '0'.repeat(this.difficulty);
      if (!current.hash.startsWith(target)) {
        return false;
      }
    }

    return true;
  }

  // 감사 기록 조회
  findRecord(recordId: string): AuditRecord | null {
    for (const block of this.chain) {
      const record = block.records.find(r => r.id === recordId);
      if (record) return record;
    }
    return null;
  }
}

스토리지 전략

효율적인 감사 추적 스토리지를 위한 다층 접근 방식:

계층 스토리지 유형 보존 기간 액세스 패턴
핫 스토리지 SSD / 메모리 30일 빈번한 쿼리, 실시간 분석
웜 스토리지 HDD / S3 Standard 1년 주기적 조회, 규정 준수 검사
콜드 스토리지 S3 Glacier 7년 드문 액세스, 장기 보존
아카이브 S3 Deep Archive 영구 법적 요구, 역사적 기록

계층형 스토리지 구현

// 계층형 감사 스토리지
class TieredAuditStorage {
  private hotStorage: HotStorage;
  private warmStorage: WarmStorage;
  private coldStorage: ColdStorage;

  // 감사 기록 저장
  async store(record: AuditRecord): Promise {
    // 핫 스토리지에 저장
    await this.hotStorage.store(record);

    // 메타데이터 인덱싱
    await this.indexRecord(record);
  }

  // 감사 기록 조회
  async retrieve(recordId: string): Promise {
    // 핫 스토리지 먼저 확인
    let record = await this.hotStorage.get(recordId);
    if (record) return record;

    // 웜 스토리지 확인
    record = await this.warmStorage.get(recordId);
    if (record) {
      // 핫 스토리지로 승격 (자주 액세스되는 경우)
      await this.promoteToHot(record);
      return record;
    }

    // 콜드 스토리지 확인
    record = await this.coldStorage.get(recordId);
    if (record) return record;

    throw new Error(`기록을 찾을 수 없습니다: ${recordId}`);
  }

  // 자동 계층 전환
  async archiveOldRecords(): Promise {
    const now = Date.now();
    const thirtyDaysAgo = now - 30 * 24 * 60 * 60 * 1000;
    const oneYearAgo = now - 365 * 24 * 60 * 60 * 1000;

    // 핫 → 웜
    const toWarm = await this.hotStorage.getOlderThan(thirtyDaysAgo);
    for (const record of toWarm) {
      await this.warmStorage.store(record);
      await this.hotStorage.delete(record.id);
    }

    // 웜 → 콜드
    const toCold = await this.warmStorage.getOlderThan(oneYearAgo);
    for (const record of toCold) {
      await this.coldStorage.store(record);
      await this.warmStorage.delete(record.id);
    }
  }

  private async promoteToHot(record: AuditRecord): Promise {
    await this.hotStorage.store(record);

    // Elasticsearch 또는 다른 검색 엔진 사용
  }

암호화 및 서명

감사 기록의 기밀성과 진위성을 보장:

디지털 서명

// 디지털 서명을 통한 감사 기록 검증
class SignedAuditRecord {
  private privateKey: string;
  private publicKey: string;

  constructor(keyPair: KeyPair) {
    this.privateKey = keyPair.private;
    this.publicKey = keyPair.public;
  }

  // 감사 기록 서명
  sign(record: AuditRecord): SignedRecord {
    const crypto = require('crypto');

    // 기록 직렬화
    const content = JSON.stringify(record);

    // 서명 생성
    const sign = crypto.createSign('SHA256');
    sign.update(content);
    sign.end();

    const signature = sign.sign(this.privateKey, 'hex');

    return {
      record: record,
      signature: signature,
      publicKey: this.publicKey,
      timestamp: new Date().toISOString()
    };
  }

  // 서명 검증
  verify(signedRecord: SignedRecord): boolean {
    const crypto = require('crypto');

    const content = JSON.stringify(signedRecord.record);

    const verify = crypto.createVerify('SHA256');
    verify.update(content);
    verify.end();

    return verify.verify(
      signedRecord.publicKey,
      signedRecord.signature,
      'hex'
    );
  }
}

종단간 암호화

// 종단간 암호화된 감사 스토리지
class EncryptedAuditStorage {
  private encryptionKey: Buffer;

  // 감사 기록 암호화 및 저장
  async storeEncrypted(record: AuditRecord): Promise {
    const crypto = require('crypto');

    // IV (Initialization Vector) 생성
    const iv = crypto.randomBytes(16);

    // 암호화
    const cipher = crypto.createCipheriv(
      'aes-256-gcm',
      this.encryptionKey,
      iv
    );

    const plaintext = JSON.stringify(record);
    const encrypted = Buffer.concat([
      cipher.update(plaintext, 'utf8'),
      cipher.final()
    ]);

    // 인증 태그
    const authTag = cipher.getAuthTag();

    // 암호화된 데이터 저장
    const encryptedRecord = {
      id: record.id,
      iv: iv.toString('hex'),
      authTag: authTag.toString('hex'),
      encrypted: encrypted.toString('hex')
    };

    await this.storage.store(encryptedRecord);
    return record.id;
  }

  // 감사 기록 복호화 및 조회
  async retrieveDecrypted(recordId: string): Promise {
    const crypto = require('crypto');

    // 암호화된 기록 조회
    const encrypted = await this.storage.retrieve(recordId);

    // 복호화
    const decipher = crypto.createDecipheriv(
      'aes-256-gcm',
      this.encryptionKey,
      Buffer.from(encrypted.iv, 'hex')
    );

    decipher.setAuthTag(Buffer.from(encrypted.authTag, 'hex'));

    const decrypted = Buffer.concat([
      decipher.update(Buffer.from(encrypted.encrypted, 'hex')),
      decipher.final()
    ]);

    return JSON.parse(decrypted.toString('utf8'));
  }
}

백업 및 복구

감사 추적의 내구성과 가용성을 보장하는 전략:

3-2-1 백업 규칙
  • 3개의 복사본: 원본 + 2개의 백업
  • 2개의 다른 매체: SSD + 클라우드 스토리지
  • 1개의 오프사이트: 다른 지리적 위치
// 백업 및 복구 시스템
class AuditBackupSystem {
  // 증분 백업
  async incrementalBackup(): Promise {
    const lastBackup = await this.getLastBackupTime();
    const newRecords = await this.getRecordsSince(lastBackup);

    // 백업 생성
    const backup = {
      timestamp: new Date().toISOString(),
      type: 'incremental',
      recordCount: newRecords.length,
      records: newRecords
    };

    // 압축
    const compressed = await this.compress(backup);

    // 여러 위치에 저장
    await Promise.all([
      this.storePrimary(compressed),
      this.storeSecondary(compressed),
      this.storeOffsite(compressed)
    ]);

    return {
      success: true,
      backupId: backup.timestamp,
      size: compressed.length
    };
  }

  // 전체 백업
  async fullBackup(): Promise {
    const allRecords = await this.getAllRecords();

    const backup = {
      timestamp: new Date().toISOString(),
      type: 'full',
      recordCount: allRecords.length,
      records: allRecords,
      merkleRoot: this.calculateMerkleRoot(allRecords)
    };

    const compressed = await this.compress(backup);

    await Promise.all([
      this.storePrimary(compressed),
      this.storeSecondary(compressed),
      this.storeOffsite(compressed)
    ]);

    return {
      success: true,
      backupId: backup.timestamp,
      size: compressed.length
    };
  }

  // 복구
  async restore(backupId: string): Promise {
    // 백업 조회 (우선순위: primary → secondary → offsite)
    let backup = await this.retrievePrimary(backupId);

    if (!backup) {
      backup = await this.retrieveSecondary(backupId);
    }

    if (!backup) {
      backup = await this.retrieveOffsite(backupId);
    }

    if (!backup) {
      throw new Error('백업을 찾을 수 없습니다');
    }

    // 압축 해제
    const decompressed = await this.decompress(backup);

    // 무결성 검증
    const valid = await this.verifyBackup(decompressed);
    if (!valid) {
      throw new Error('백업이 손상되었습니다');
    }

    // 복원
    await this.restoreRecords(decompressed.records);
  }
}

감사 추적 쿼리

효율적인 감사 기록 조회를 위한 쿼리 인터페이스:

// 감사 추적 쿼리 엔진
class AuditQueryEngine {
  // 시간 범위 쿼리
  async queryByTimeRange(
    start: Date,
    end: Date
  ): Promise {
    return await this.storage.query({
      timestamp: {
        $gte: start.toISOString(),
        $lte: end.toISOString()
      }
    });
  }

  // 모델별 쿼리
  async queryByModel(modelId: string): Promise {
    return await this.storage.query({
      'model.id': modelId
    });
  }

  // 복잡한 쿼리
  async advancedQuery(criteria: QueryCriteria): Promise {
    const query: any = {};

    // 시간 필터
    if (criteria.timeRange) {
      query.timestamp = {
        $gte: criteria.timeRange.start,
        $lte: criteria.timeRange.end
      };
    }

    // 모델 필터
    if (criteria.modelId) {
      query['model.id'] = criteria.modelId;
    }

    // 결정 유형 필터
    if (criteria.decisionType) {
      query['output.decision'] = criteria.decisionType;
    }

    // 신뢰도 필터
    if (criteria.minConfidence) {
      query['output.confidence'] = { $gte: criteria.minConfidence };
    }

    // 사용자 필터
    if (criteria.userId) {
      query['context.userId'] = criteria.userId;
    }

    return await this.storage.query(query);
  }

  // 전문 검색 (Full-text search)
  async search(searchTerm: string): Promise {
    // Elasticsearch 또는 유사한 검색 엔진 사용
    return await this.searchEngine.search({
      query: {
        multi_match: {
          query: searchTerm,
          fields: [
            'reasoning.explanationText',
            'input.*',
            'output.*'
          ]
        }
      }
    });
  }
}

성능 최적화

  • 인덱싱: timestamp, modelId, userId 등 자주 쿼리되는 필드에 인덱스 생성

  • 압축: 오래된 기록을 압축하여 스토리지 절약
  • 비동기 처리: 백그라운드에서 무결성 검증 및 아카이빙 수행
