Chapter 7: Reporting and Visualization

Communicating Audit Insights to Stakeholders

The Power of Effective Communication

Audit data without effective communication is merely stored bits—valuable insights remain locked away. Reporting transforms raw audit trails into actionable intelligence, enabling stakeholders to understand AI behavior, identify issues, and make informed decisions.

The principle of 弘익人間 reminds us that transparency serves humanity. Effective audit reporting democratizes AI understanding, empowering diverse stakeholders—from executives to affected individuals—to engage meaningfully with algorithmic decision-making.

Individual Decision Explanation

Clear, accessible explanations for affected individuals:

class IndividualExplanationGenerator {
  async generateExplanation(
    decisionId: string,
    userId: string
  ): Promise {
    const log = await this.loadDecision(decisionId);

    // Verify authorization
    if (log.context.user_id !== userId) {
      throw new UnauthorizedError();
    }

    return {
      decision_summary: {
        decision: this.humanReadableDecision(log.output.decision),
        date: new Date(log.timestamp).toLocaleDateString(),
        confidence: `${Math.round(log.output.confidence * 100)}%`
      },

      why_this_decision: this.generateNaturalLanguageExplanation(log),

      key_factors: log.reasoning.key_features.map(f => ({
        factor: this.humanReadableFactor(f.name),
        importance: this.importanceDescription(f.importance),
        your_value: this.formatValue(f.value),
        impact: f.importance > 0.2 ? "Strong influence" :
                f.importance > 0.1 ? "Moderate influence" :
                "Minor influence"
      })),

      what_if_scenarios: log.reasoning.counterfactuals?.map(cf =>
        this.formatCounterfactual(cf)
      ) || [],

      your_rights: {
        explanation: "You have received this explanation as required by GDPR Article 22",
        challenge: "You have the right to challenge this decision",
        human_review: "You can request human review",
        contact: "privacy@example.com"
      },

      next_steps: this.generateNextSteps(log)
    };
  }

  private generateNaturalLanguageExplanation(log: AuditLog): string {
    // Convert technical explanation to plain language
    const template = this.getExplanationTemplate(log.context.decision_type);

    return template
      .replace("{decision}", log.output.decision)
      .replace("{primary_factor}", log.reasoning.key_features[0].name)
      .replace("{confidence}", `${Math.round(log.output.confidence * 100)}%`);
  }
}

Visualization Best Practices

Time Series Visualization

Track metrics over time to identify trends:

class TimeSeriesVisualizer {
  generateComplianceTimeSeries(logs: AuditLog[]): ChartData {
    const dailyCompliance = this.groupByDay(logs).map(day => ({
      date: day.date,
      compliance_rate: this.calculateComplianceRate(day.logs),
      violation_count: this.countViolations(day.logs),
      total_decisions: day.logs.length
    }));

    return {
      type: "line",
      title: "Compliance Rate Over Time",
      data: dailyCompliance,
      x_axis: "date",
      y_axis: "compliance_rate",
      target_line: 0.95  // 95% target
    };
  }

  generateRiskTimeSeries(logs: AuditLog[]): ChartData {
    const dailyRisk = this.groupByDay(logs).map(day => ({
      date: day.date,
      average_risk: this.calculateAverageRisk(day.logs),
      high_risk_count: this.countHighRisk(day.logs),
      critical_risk_count: this.countCriticalRisk(day.logs)
    }));

    return {
      type: "line",
      title: "Risk Trend Analysis",
      data: dailyRisk,
      series: ["average_risk", "high_risk_count", "critical_risk_count"]
    };
  }
}

Distribution Visualization

Understand the distribution of key metrics:

class DistributionVisualizer {
  generateConfidenceDistribution(logs: AuditLog[]): ChartData {
    const bins = [0, 0.5, 0.6, 0.7, 0.8, 0.9, 0.95, 1.0];
    const distribution = this.binByConfidence(logs, bins);

    return {
      type: "histogram",
      title: "Confidence Score Distribution",
      data: distribution,
      x_axis: "confidence_bin",
      y_axis: "count",
      color_scale: (bin) => bin < 0.7 ? "red" :
                             bin < 0.85 ? "yellow" :
                             "green"
    };
  }

  generateRiskHeatmap(logs: AuditLog[]): ChartData {
    const heatmap = this.createRiskHeatmap(logs);

    return {
      type: "heatmap",
      title: "Risk Distribution by Decision Type and Time",
      data: heatmap,
      x_axis: "hour_of_day",
      y_axis: "decision_type",
      color_scale: "red_yellow_green",
      cell_value: "average_risk_score"
    };
  }
}

Comparative Analysis

Compare performance across models, time periods, or segments:

class ComparativeAnalyzer {
  compareModels(models: string[], timeRange: DateRange): ChartData {
    const comparison = models.map(async model => {
      const logs = await this.loadLogsForModel(model, timeRange);
      return {
        model,
        accuracy: this.calculateAccuracy(logs),
        compliance_rate: this.calculateComplianceRate(logs),
        average_risk: this.calculateAverageRisk(logs),
        prediction_count: logs.length
      };
    });

    return {
      type: "bar",
      title: "Model Performance Comparison",
      data: await Promise.all(comparison),
      x_axis: "model",
      y_axes: ["accuracy", "compliance_rate", "average_risk"]
    };
  }

  generateFairnessComparison(logs: AuditLog[]): ChartData {
    const groups = this.inferDemographicGroups(logs);
    const approvalRates = groups.map(group => ({
      group: group.name,
      approval_rate: this.calculateApprovalRate(group.logs),
      sample_size: group.logs.length
    }));

    return {
      type: "bar",
      title: "Approval Rates by Demographic Group",
      data: approvalRates,
      x_axis: "group",
      y_axis: "approval_rate",
      reference_line: 0.80  // 80% rule threshold
    };
  }
}

Automated Report Generation and Distribution

Schedule regular reports for stakeholders:

class AutomatedReportingService {
  scheduleReports(): void {
    // Daily executive summary
    this.scheduler.daily("07:00", async () => {
      const report = await this.executiveSummary.generate("yesterday");
      await this.email.send({
        to: "executives@company.com",
        subject: "Daily AI Decision Summary",
        body: this.formatEmailReport(report),
        attachments: [await this.exportToPDF(report)]
      });
    });

    // Weekly compliance report
    this.scheduler.weekly("Monday 09:00", async () => {
      const report = await this.complianceReport.generate("last_week");
      await this.email.send({
        to: "compliance@company.com",
        subject: "Weekly Compliance Report",
        body: this.formatComplianceEmail(report),
        attachments: [
          await this.exportToPDF(report),
          await this.exportToCSV(report.violations)
        ]
      });
    });

    // Monthly model performance report
    this.scheduler.monthly("1st 10:00", async () => {
      const models = await this.getActiveModels();

      for (const model of models) {
        const report = await this.modelReport.generate(model, "last_month");
        await this.email.send({
          to: model.owner_email,
          subject: `Monthly Performance: ${model.name}`,
          body: this.formatModelReport(report),
          attachments: [await this.exportToPDF(report)]
        });
      }
    });
  }
}

Export Formats

Support multiple export formats for different use cases:

class ReportExporter {
  async exportToPDF(report: Report): Promise {
    const pdf = new PDFDocument();

    // Add header
    pdf.fontSize(24).text(report.title);
    pdf.fontSize(12).text(`Generated: ${new Date().toLocaleString()}`);

    // Add sections
    for (const section of report.sections) {
      pdf.addPage();
      pdf.fontSize(18).text(section.title);
      pdf.fontSize(10).text(section.content);

      // Add charts as images
      if (section.chart) {
        const chartImage = await this.renderChartToImage(section.chart);
        pdf.image(chartImage);
      }
    }

    return pdf.outputSync();
  }

  async exportToCSV(data: any[]): Promise {
    const headers = Object.keys(data[0]);
    const rows = data.map(row =>
      headers.map(h => JSON.stringify(row[h])).join(',')
    );

    return [headers.join(','), ...rows].join('\n');
  }

  async exportToExcel(report: Report): Promise {
    const workbook = new ExcelWorkbook();

    for (const table of report.tables) {
      const worksheet = workbook.addWorksheet(table.name);
      worksheet.addTable({
        name: table.name,
        ref: 'A1',
        headers: table.headers,
        rows: table.rows
      });
    }

    return workbook.outputAsync();
  }

  async exportToJSON(report: Report): Promise {
    return JSON.stringify(report, null, 2);
  }
}

Chapter Summary

  • Effective reporting transforms audit data into actionable intelligence for diverse stakeholders
  • Executive dashboards provide high-level metrics, trends, and strategic insights for C-level decision-making
  • Compliance officer reports detail violations, framework-specific analysis, and regulatory filing preparation
  • Data scientist reports provide technical performance metrics, drift analysis, fairness metrics, and error analysis
  • Individual explanations offer clear, accessible decision reasoning to affected persons
  • Time series visualization tracks compliance, risk, and performance trends over time
  • Distribution visualization reveals confidence distributions, risk heatmaps, and comparative analysis
  • Interactive dashboards enable dynamic data exploration with filters, drill-down, and export capabilities
  • Automated reporting schedules regular distribution of daily, weekly, and monthly reports to stakeholders
  • Multiple export formats (PDF, CSV, Excel, JSON) support different consumption and analysis needs
