Chapter 4: Phase 1 - Data Format

In this chapter: JSON schemas for cultural events, participant profiles, heritage metadata, media assets, and validation rules. Foundation layer enabling data interoperability.

4.1 Cultural Event Schema

The Cultural Event is the central entity in WIA-UNI-011. Every exchange activity - whether arts exhibition, sports competition, music performance, or heritage project - is represented as a standardized event record.

Core Event Fields

FieldTypeRequiredDescription
eventIdstringYesUnique identifier (UNI-011-{TYPE}-{TIMESTAMP})
eventTypeenumYesarts, sports, music, festival, heritage, education
titlei18nYesMultilingual event name (ko, en, etc.)
descriptioni18nYesDetailed multilingual description
startDateISO 8601YesEvent start date and time with timezone
endDateISO 8601YesEvent end date and time with timezone
locationLocationYesStructured location with coordinates
organizersOrganization[]YesOrganizing institutions
participantsParticipant[]NoEvent participants with roles
mediaMediaAsset[]NoPhotos, videos, documents
tagsstring[]NoClassification and discovery tags

Example: Music Festival Event

{
  "standard": "WIA-UNI-011",
  "version": "2.0",
  "eventId": "UNI-011-MUSIC-2025-001",
  "eventType": "music",
  "title": {
    "ko": "남북 통일 음악 축제",
    "en": "Inter-Korean Unity Music Festival",
    "ko-north": "북남 통일 음악 축전"
  },
  "description": {
    "ko": "남북 음악가들이 함께하는 평화 음악 축제",
    "en": "Peace music festival featuring artists from both Koreas"
  },
  "startDate": "2025-06-15T18:00:00+09:00",
  "endDate": "2025-06-17T22:00:00+09:00",
  "location": {
    "name": { "ko": "DMZ 평화공원", "en": "DMZ Peace Park" },
    "address": "Paju, Gyeonggi Province",
    "coordinates": { "lat": 37.8813, "lng": 126.7569 }
  },
  "organizers": [
    { "id": "ORG-SK-CULTURE-001", "name": "Ministry of Culture (ROK)" },
    { "id": "ORG-NK-CULTURE-001", "name": "Ministry of Culture (DPRK)" }
  ],
  "unescoCategory": "intangible",
  "expectedParticipants": 500,
  "tags": ["music", "peace", "unity", "traditional", "modern"]
}

4.2 Participant Schema

Participant records capture information about individuals involved in cultural exchange while respecting privacy and security requirements.

Participant Fields

FieldTypePrivacy LevelDescription
participantIdstringPublicUnique identifier
namei18nControlledFull name in multiple scripts
roleenumPublicartist, athlete, organizer, audience, etc.
organizationOrganizationPublicAffiliated institution
regionenumPublicsouth, north, international
specialtystring[]PublicAreas of expertise/performance
biographyi18nPublicProfessional biography
consentLevelenumPrivateData sharing consent level

Privacy-Preserving Design

Participant data uses three-tier access control:

  • Public: Role, specialty, organization - visible to all
  • Controlled: Name, biography - visible to authorized parties
  • Private: Contact info, nationality - strictly restricted

4.3 Heritage Item Schema

Cultural heritage items receive rich metadata following UNESCO standards while documenting regional variations across divided Korea.

Heritage Fields

FieldTypeRequiredDescription
heritageIdstringYesUnique identifier
namei18nYesHeritage item name with variants
categoryenumYesmusic, dance, craft, food, ritual, architecture
unescoStatusenumNoregistered, nominated, candidate, local
descriptioni18nYesDetailed description
historicalContexti18nYesHistorical background
regionalVariationsVariation[]NoNorth/South differences
preservationStatusenumYesexcellent, good, fair, endangered, critical
mediaMediaAsset[]NoDocumentation media

Regional Variation Documentation

Heritage items often have different names, practices, or preservation states in North and South Korea. WIA-UNI-011 explicitly documents these variations:

{
  "heritageId": "HER-KR-MUSIC-001",
  "name": {
    "ko": "아리랑",
    "en": "Arirang",
    "ko-north": "아리랑",
    "hanja": "阿里郞"
  },
  "category": "music",
  "unescoStatus": "registered",
  "regionalVariations": [
    {
      "region": "south",
      "localName": "정선아리랑",
      "characteristics": "Slower tempo, unique melodic structure",
      "preservationStatus": "excellent"
    },
    {
      "region": "north",
      "localName": "평양아리랑",
      "characteristics": "Faster tempo, orchestral arrangements",
      "preservationStatus": "excellent"
    }
  ]
}

4.4 Media Asset Schema

Cultural exchange generates massive amounts of media. Proper metadata ensures these assets remain discoverable, contextual, and preserved long-term.

Media Asset Fields

FieldTypeRequiredDescription
assetIdstringYesUnique identifier
typeenumYesphoto, video, audio, document, 3d
titlei18nYesAsset title
descriptioni18nNoDetailed description
creatorstringYesCreator/photographer/videographer
createdISO 8601YesCreation date and time
licenseLicenseYesUsage rights and restrictions
fileFileYesFile details (URL, format, size, checksum)
relatedEventstringNoEvent ID if applicable
tagsstring[]NoDiscovery tags

Rights Management

Media assets include detailed licensing information enabling proper usage tracking and rights management:

  • Creative Commons: CC0, CC-BY, CC-BY-SA, CC-BY-NC, CC-BY-ND
  • Public Domain: Public domain dedication for historical materials
  • Rights Reserved: All rights reserved with permission required
  • Government Work: Government-created content with special status

4.5 Multilingual Data Structures

WIA-UNI-011 uses the i18n (internationalization) type for all user-facing text. This structure supports multiple languages and scripts:

{
  "title": {
    "ko": "한글 제목",
    "ko-north": "조선말 제목",
    "en": "English Title",
    "zh": "中文标题",
    "ja": "日本語タイトル"
  }
}

Language Codes

  • ko - Korean (South Korean standard)
  • ko-north - Korean (North Korean variation)
  • en - English
  • hanja - Chinese characters (Hanja)
  • zh, ja, ru - Other languages as needed

4.6 Validation Rules

All WIA-UNI-011 data must pass validation before being accepted into systems. Validation ensures data quality, consistency, and interoperability.

Standard Validation Rules

  • All required fields must be present
  • Dates must be valid ISO 8601 format with timezone
  • Coordinates must be within valid latitude/longitude ranges
  • Enum values must match allowed options
  • At least one language (ko or en) required for i18n fields
  • References (eventId, organizationId) must resolve to valid entities
  • File checksums must match actual file content

4.7 Data Migration Strategy

Organizations with existing cultural databases can migrate to WIA-UNI-011 format using provided conversion tools:

  1. Export existing data in CSV, JSON, or Excel format
  2. Map fields from legacy schema to WIA-UNI-011 schema
  3. Enrich data with missing required fields
  4. Validate using WIA-UNI-011 validation tools
  5. Import into WIA-UNI-011 compliant systems
