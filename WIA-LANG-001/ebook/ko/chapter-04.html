WIA-LANG-001

04.1 데이터 형식의 중요성

언어 자원의 데이터 형식은 WIA-LANG-001 표준의 가장 기본적인 계층입니다. 표준화된 데이터 형식 없이는 시스템 간 상호운용성, 데이터 재사용성, 장기 보존이 불가능합니다.

멸종 위기 언어의 경우, 수집된 데이터가 유일한 기록일 수 있습니다. 따라서 데이터 형식 선택은 단순한 기술적 결정이 아니라 문화유산 보존과 직결된 중대한 문제입니다.

🎯 데이터 형식 선택 기준
  • 개방성: 독점 형식 대신 개방형 표준 사용
  • 지속가능성: 50년 후에도 읽을 수 있는 형식
  • 품질: 무손실 또는 최소 손실 압축
  • 호환성: 기존 도구 및 라이브러리 지원
  • 확장성: 메타데이터 및 주석 추가 가능

데이터 유형별 분류

WIA-LANG-001은 다음 주요 데이터 유형을 정의합니다:

데이터 유형주요 형식용도우선순위
오디오 WAV, FLAC, MP3 음성 녹음, ASR 훈련 최고
텍스트 UTF-8, XML, JSON 전사, 코퍼스, 메타데이터 최고
비디오 MP4 (H.264), WebM 수화, 문화 기록 높음
이미지 PNG, TIFF, JPEG 문서 스캔, 문자 체계 중간
주석 ELAN (.eaf), Praat (.TextGrid) 언어학적 분석 높음

04.2 오디오 데이터 형식

오디오는 멸종 위기 언어 보존의 핵심 자료입니다. 많은 언어가 문자 체계를 갖추지 못했거나, 구술 전통이 강하기 때문입니다.

WAV (Waveform Audio File Format)

WAV는 보관용 원본 데이터의 표준 형식입니다:

📊 WAV 권장 스펙
  • 샘플링 레이트: 16kHz (ASR용), 44.1kHz (음악/고품질), 48kHz (방송 표준)
  • 비트 깊이: 16-bit (최소), 24-bit (권장), 32-bit float (전문가용)
  • 채널: Mono (단일 화자), Stereo (스테레오 녹음)
  • 인코딩: PCM (Pulse Code Modulation) - 무압축

WAV 파일 헤더 구조:

RIFF Header (12 bytes):
  "RIFF" (4 bytes)          // 파일 형식 식별자
  File Size - 8 (4 bytes)   // 전체 파일 크기
  "WAVE" (4 bytes)          // 파일 타입

Format Chunk (24 bytes):
  "fmt " (4 bytes)          // 포맷 청크 식별자
  16 (4 bytes)              // 청크 크기
  1 (2 bytes)               // 오디오 형식 (1 = PCM)
  1 or 2 (2 bytes)          // 채널 수
  16000/44100/48000 (4 b)   // 샘플링 레이트
  Byte Rate (4 bytes)       // = SampleRate * NumChannels * BitsPerSample/8
  Block Align (2 bytes)     // = NumChannels * BitsPerSample/8
  16/24/32 (2 bytes)        // 비트 깊이

Data Chunk:
  "data" (4 bytes)          // 데이터 청크 식별자
  Data Size (4 bytes)       // 오디오 데이터 크기
  [Audio Samples]           // 실제 오디오 데이터

ASR을 위한 오디오 전처리

원본 WAV 파일을 ASR 훈련에 사용하기 전 다음 전처리를 수행합니다:

  1. 리샘플링: 16kHz로 다운샘플링 (대부분 ASR 모델의 표준)
  2. 정규화: 음량을 -3dB 또는 -1dB peak로 정규화
  3. 잡음 제거: Spectral subtraction 또는 Wiener filtering
  4. 침묵 구간 제거: VAD (Voice Activity Detection) 적용
# Python 예제: FFmpeg로 오디오 전처리

import subprocess

def preprocess_audio(input_path, output_path):
    """WAV 파일을 ASR 표준 형식으로 변환"""
    cmd = [
        'ffmpeg', '-i', input_path,
        '-ar', '16000',           # 16kHz 리샘플링
        '-ac', '1',               # 모노로 변환
        '-sample_fmt', 's16',     # 16-bit PCM
        '-af', 'loudnorm=I=-16:TP=-1.5:LRA=11',  # 음량 정규화
        output_path
    ]
    subprocess.run(cmd, check=True)

# 사용 예
preprocess_audio('original_recording.wav', 'processed_16k.wav')

FLAC (Free Lossless Audio Codec)

FLAC는 무손실 압축 형식으로, WAV 대비 40-60% 용량 절감이 가능합니다:

  • 장점: 무손실, 메타데이터 지원, 오픈소스, WAV보다 작은 크기
  • 단점: CPU 사용량 증가 (압축/해제 시)
  • 사용 시나리오: 장기 보관, 클라우드 백업, 배포용
# FLAC 압축 예제

# WAV를 FLAC로 변환 (최대 압축)
flac --best original.wav -o compressed.flac

# FLAC를 WAV로 복원 (완벽한 원본 재현)
flac -d compressed.flac -o restored.wav

# MD5 체크섬 확인
md5sum original.wav restored.wav
# 두 파일의 MD5 해시가 동일함을 확인

MP3 (MPEG-1 Audio Layer 3)

MP3는 손실 압축 형식이지만, 일반 배포 및 스트리밍에 유용합니다:

비트레이트품질용도파일 크기 (1분당)
320 kbps 최고 (near-CD) 음악, 고품질 배포 ~2.4 MB
192 kbps 높음 일반 배포, 언어 학습 ~1.4 MB
128 kbps 중간 스트리밍, 저대역폭 ~960 KB
64 kbps 낮음 음성 전용 (비추천) ~480 KB
⚠️ 중요: 보관용은 절대 MP3 사용 금지

MP3는 손실 압축이므로, 원본 오디오를 완벽하게 재현할 수 없습니다. 보관용 원본은 반드시 WAV 또는 FLAC를 사용하세요. MP3는 오직 배포 및 접근성 향상 목적으로만 사용합니다.

04.3 텍스트 데이터 형식

텍스트 데이터는 언어 코퍼스, 전사, 번역, 사전 등 다양한 형태로 존재합니다.

UTF-8 인코딩 (필수)

모든 텍스트 파일은 UTF-8 인코딩을 사용해야 합니다. 이는 세계 모든 문자 체계를 지원하는 유일한 실용적 표준입니다.

🔤 UTF-8의 장점
  • 보편성: 유니코드 14.0 기준 144,697개 문자 지원
  • 역호환성: ASCII와 100% 호환
  • 효율성: 가변 길이 인코딩 (1-4 바이트)
  • 오류 감지: 잘못된 시퀀스 감지 가능
  • 표준화: RFC 3629, ISO/IEC 10646 표준

문자 정규화 (Normalization)

유니코드는 동일한 문자를 여러 방식으로 표현할 수 있습니다. 예: "é" = U+00E9 (단일 문자) 또는 U+0065 U+0301 (e + 결합 accent)

정규화 형식설명사용 시나리오
NFC Canonical Composition (합성) 일반 텍스트 저장 (권장)
NFD Canonical Decomposition (분해) 언어학적 분석, 검색
NFKC Compatibility Composition 호환성이 필요한 경우
NFKD Compatibility Decomposition 최대 분해가 필요한 경우 
# Python 유니코드 정규화 예제

import unicodedata

text = "Māori"  # ā는 U+0101 (단일 문자)

# NFC 정규화 (권장)
nfc = unicodedata.normalize('NFC', text)
print(f"NFC: {nfc} ({len(nfc)} chars)")  # "Māori" (5 chars)

# NFD 정규화
nfd = unicodedata.normalize('NFD', text)
print(f"NFD: {nfd} ({len(nfd)} chars)")  # "Māori" (6 chars: M + a + ̄ + o + r + i)

# 파일 저장 전 항상 NFC 정규화
with open('text.txt', 'w', encoding='utf-8') as f:
    f.write(unicodedata.normalize('NFC', text))

CoNLL-U 형식 (Universal Dependencies)

CoNLL-U는 형태통사론적 주석을 위한 표준 형식입니다:

# sent_id = mri_001
# text = Kei te haere au ki te kura.
# translation[en] = I am going to school.
# translation[ko] = 나는 학교에 가고 있습니다.
1	Kei	kei	ADP	_	_	3	aux	_	Gloss=PROG
2	te	te	DET	_	_	3	det	_	Gloss=the
3	haere	haere	VERB	_	_	0	root	_	Gloss=go
4	au	au	PRON	_	Number=Sing|Person=1	3	nsubj	_	Gloss=I
5	ki	ki	ADP	_	_	7	case	_	Gloss=to
6	te	te	DET	_	_	7	det	_	Gloss=the
7	kura	kura	NOUN	_	_	3	obl	_	Gloss=school|SpaceAfter=No
8	.	.	PUNCT	_	_	3	punct	_	_

CoNLL-U 형식의 10개 필드:

  1. ID: 단어 인덱스
  2. FORM: 단어 표층형
  3. LEMMA: 기본형
  4. UPOS: Universal POS 태그
  5. XPOS: 언어 특화 POS 태그
  6. FEATS: 형태론적 자질 (Number, Tense 등)
  7. HEAD: 지배소 ID
  8. DEPREL: 의존 관계 (nsubj, obj 등)
  9. DEPS: 향상된 의존 관계
  10. MISC: 기타 정보

04.4 비디오 데이터 형식

비디오는 수화 언어, 제스처, 문화적 맥락 기록에 필수적입니다.

MP4 (MPEG-4 Part 14)

MP4는 가장 널리 지원되는 비디오 컨테이너 형식입니다:

🎬 MP4 권장 설정
  • 비디오 코덱: H.264 (AVC) - 보편적 지원
  • 오디오 코덱: AAC LC - 고품질, 낮은 비트레이트
  • 해상도: 1920x1080 (Full HD) 최소, 3840x2160 (4K) 권장
  • 프레임레이트: 30 fps (표준), 60 fps (고품질)
  • 비트레이트: 5-10 Mbps (1080p), 25-50 Mbps (4K)

수화 언어를 위한 특별 고려사항

수화는 손의 미세한 움직임, 표정, 신체 언어를 포착해야 합니다:

요소요구사항이유
해상도 최소 1080p, 4K 권장 손가락 철자, 표정 상세 캡처
프레임레이트 60 fps 권장 빠른 손 움직임 캡처
조명 균일한 전면 조명 손 모양 명확성
배경 단색 (파란색/녹색) 서명자와 배경 구분
프레이밍 머리에서 허리까지 전신 수화 포착

WebM 형식

WebM은 오픈소스 비디오 형식으로, 웹 스트리밍에 최적화되어 있습니다:

  • 비디오 코덱: VP9 또는 AV1 (최신)
  • 오디오 코덱: Opus
  • 장점: 로열티 프리, 효율적 압축, 브라우저 지원
  • 사용 시나리오: 웹 배포, 온라인 학습 플랫폼
# FFmpeg로 수화 비디오 최적화

ffmpeg -i sign_language_original.mov \
  -c:v libx264 \              # H.264 코덱
  -preset slow \              # 최고 압축 효율
  -crf 18 \                   # 거의 무손실 품질
  -vf "scale=1920:1080" \     # 1080p로 스케일
  -r 60 \                     # 60fps
  -c:a aac \                  # AAC 오디오
  -b:a 192k \                 # 192kbps 오디오
  -movflags +faststart \      # 웹 스트리밍 최적화
  sign_language_optimized.mp4

# 4K 버전 (보관용)
ffmpeg -i sign_language_original.mov \
  -c:v libx265 \              # H.265 (더 높은 압축률)
  -crf 18 \
  -vf "scale=3840:2160" \     # 4K
  -r 60 \
  -c:a aac -b:a 256k \
  -tag:v hvc1 \               # Apple 호환성
  sign_language_4k_archive.mp4

04.5 주석 데이터 형식

언어학적 주석은 연구 및 AI 모델 훈련에 필수적입니다.

ELAN (.eaf) 형식

ELAN (EUDICO Linguistic Annotator)는 다중 계층 주석을 위한 표준 도구입니다:

📝 ELAN 주요 기능
  • 시간 정렬: 오디오/비디오와 텍스트 동기화
  • 계층 구조: 화자, 전사, 번역, 형태소 분석 등 다중 계층
  • 통제 어휘: 일관성 있는 주석을 위한 사전 정의 값
  • 검색 기능: 복잡한 쿼리 지원
  • 상호운용성: Praat, Toolbox 등과 연동

04.7 데이터 품질 검증

데이터 품질은 AI 모델 성능에 직접적인 영향을 미칩니다.

자동 품질 검사

# Python 데이터 품질 검증 스크립트

import os
import json
import hashlib
import librosa
import soundfile as sf
from typing import Dict, List

class AudioQualityChecker:
    """오디오 파일 품질 검증"""

    def __init__(self, target_sr=16000, target_channels=1):
        self.target_sr = target_sr
        self.target_channels = target_channels

    def check_file(self, audio_path: str) -> Dict:
        """단일 오디오 파일 검증"""
        results = {
            "file": audio_path,
            "passed": True,
            "errors": [],
            "warnings": []
        }

        # 파일 존재 확인
        if not os.path.exists(audio_path):
            results["errors"].append("File not found")
            results["passed"] = False
            return results

        try:
            # 오디오 로드
            y, sr = librosa.load(audio_path, sr=None, mono=False)

            # 샘플링 레이트 확인
            if sr != self.target_sr:
                results["warnings"].append(
                    f"Sample rate {sr} != target {self.target_sr}"
                )

            # 채널 확인
            channels = 1 if y.ndim == 1 else y.shape[0]
            if channels != self.target_channels:
                results["warnings"].append(
                    f"Channels {channels} != target {self.target_channels}"
                )

            # 길이 확인 (최소 1초)
            duration = len(y) / sr if y.ndim == 1 else len(y[0]) / sr
            if duration < 1.0:
                results["errors"].append(
                    f"Duration {duration:.2f}s too short"
                )
                results["passed"] = False

            # 클리핑 확인
            if y.ndim == 1:
                clipping_pct = (np.abs(y) >= 0.99).sum() / len(y) * 100
            else:
                clipping_pct = (np.abs(y) >= 0.99).sum() / y.size * 100

            if clipping_pct > 1.0:
                results["errors"].append(
                    f"Clipping detected: {clipping_pct:.2f}%"
                )
                results["passed"] = False
            elif clipping_pct > 0.1:
                results["warnings"].append(
                    f"Minor clipping: {clipping_pct:.2f}%"
                )

            # 침묵 확인 (RMS)
            if y.ndim == 1:
                rms = np.sqrt(np.mean(y**2))
            else:
                rms = np.sqrt(np.mean(y**2))

            if rms < 0.001:
                results["errors"].append("Audio appears to be silent")
                results["passed"] = False

            # SNR 추정 (간단한 방법)
            # 상위 10% RMS vs 하위 10% RMS
            if y.ndim == 1:
                sorted_rms = np.sort(librosa.feature.rms(y=y)[0])
            else:
                sorted_rms = np.sort(librosa.feature.rms(y=y[0])[0])

            signal = np.mean(sorted_rms[-len(sorted_rms)//10:])
            noise = np.mean(sorted_rms[:len(sorted_rms)//10])
            snr = 20 * np.log10(signal / (noise + 1e-10))

            if snr < 10:
                results["warnings"].append(
                    f"Low SNR: {snr:.1f} dB"
                )

            results["metrics"] = {
                "duration": duration,
                "sample_rate": sr,
                "channels": channels,
                "rms": float(rms),
                "snr_db": float(snr),
                "clipping_pct": float(clipping_pct)
            }

        except Exception as e:
            results["errors"].append(f"Error processing file: {str(e)}")
            results["passed"] = False

        return results

    def check_directory(self, directory: str) -> List[Dict]:
        """디렉토리 내 모든 오디오 파일 검증"""
        results = []
        for root, dirs, files in os.walk(directory):
            for file in files:
                if file.endswith(('.wav', '.flac', '.mp3')):
                    path = os.path.join(root, file)
                    results.append(self.check_file(path))
        return results

# 사용 예
checker = AudioQualityChecker(target_sr=16000, target_channels=1)
results = checker.check_directory('/path/to/audio/corpus')

# 결과 요약
passed = sum(1 for r in results if r["passed"])
failed = len(results) - passed
print(f"Checked {len(results)} files: {passed} passed, {failed} failed")

# 실패한 파일 상세 출력
for r in results:
    if not r["passed"]:
        print(f"\nFAILED: {r['file']}")
        for error in r["errors"]:
            print(f"  ERROR: {error}")
