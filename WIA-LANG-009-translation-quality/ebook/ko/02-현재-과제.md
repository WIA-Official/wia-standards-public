# 제2장: 번역 품질의 현재 과제

## 번역 품질 평가의 근본적 어려움

번역 품질 평가는 언어학, 문화학, 컴퓨터 과학이 교차하는 복잡한 영역입니다. 기술이 발전했음에도 불구하고, 번역 품질을 정확하고 일관되게 평가하는 것은 여전히 큰 도전 과제입니다.

### 주관성의 문제

번역 품질은 본질적으로 주관적입니다. 같은 번역에 대해서도 평가자의 배경, 언어 능력, 문화적 맥락에 따라 다른 평가가 나올 수 있습니다.

**평가자 간 불일치 사례**:

| 번역 예시 | 평가자 A | 평가자 B | 불일치 이유 |
|----------|----------|----------|------------|
| "break a leg" → "다리를 부러뜨려" | 1/5 (직역) | 3/5 (의미 전달) | 관용구 처리 |
| "CEO" → "최고경영자" | 4/5 (형식적) | 2/5 (불필요) | 독자층 인식 차이 |
| "you" → "너/당신/귀하" | 3/5 | 5/5 | 격식 수준 판단 |

**주관성 감소 전략**:
- 명확한 평가 가이드라인 제공
- 다수 평가자의 평가 집계
- 도메인별 구체적 기준 설정
- 평가자 교육 및 보정

### 맥락 의존성

번역의 품질은 사용 목적과 독자에 따라 달라집니다.

```json
{
  "source": "The product is cool",
  "contexts": [
    {
      "domain": "technical",
      "target": "제품의 온도가 낮습니다",
      "quality": "high_in_context"
    },
    {
      "domain": "marketing",
      "target": "이 제품 정말 멋져요",
      "quality": "high_in_context"
    },
    {
      "domain": "scientific",
      "target": "제품이 시원합니다",
      "quality": "low_precision"
    }
  ]
}
```

## 자동 평가 메트릭의 한계

### BLEU의 근본적 문제

BLEU는 가장 널리 사용되는 자동 평가 메트릭이지만, 여러 한계가 있습니다.

**BLEU가 놓치는 것들**:

1. **의미론적 유사성**
```
참조: "The cat sat on the mat"
번역A: "A feline was positioned on the rug"  → BLEU: 0.00
번역B: "The mat cat the on sat"              → BLEU: 1.00

실제 품질: 번역A > 번역B
BLEU 점수: 번역A < 번역B
```

2. **어순의 차이**
```
영어: "I love you"
한국어 참조: "나는 너를 사랑해"
번역: "너를 나는 사랑해"

의미: 완전히 동일
BLEU: 낮은 점수 (어순 불일치)
```

3. **동의어 인식 실패**
```
참조: "big house"
번역A: "large house"    → BLEU: 0.5 (big 불일치)
번역B: "big building"   → BLEU: 0.5 (house 불일치)

실제: 번역A가 더 정확
```

### 메트릭별 한계 비교

| 메트릭 | 주요 한계 | 영향 받는 언어 유형 |
|--------|----------|-------------------|
| **BLEU** | 동의어 미인식, 어순 의존 | 한국어, 일본어, 터키어 |
| **METEOR** | 언어별 리소스 필요 | 저자원 언어 |
| **TER** | 편집 거리만 고려 | 형태론이 복잡한 언어 |
| **chrF** | 의미론적 오류 탐지 약함 | 모든 언어 |
| **BERTScore** | 계산 비용 높음, 언어별 모델 필요 | 저자원 언어 |

### 학습형 메트릭의 데이터 편향

COMET, BLEURT 같은 신경망 기반 메트릭은 학습 데이터에 의존합니다.

**데이터 편향 문제**:
```json
{
  "training_data_distribution": {
    "european_languages": "78%",
    "asian_languages": "15%",
    "african_languages": "4%",
    "indigenous_languages": "3%"
  },
  "performance_correlation": {
    "european_languages": 0.85,
    "asian_languages": 0.72,
    "african_languages": 0.58,
    "indigenous_languages": 0.41
  }
}
```

## 인간 평가의 과제

### 비용과 확장성

인간 평가는 정확하지만 비용이 많이 들고 느립니다.

**인간 평가 비용 분석** (2024년 기준):

| 평가 유형 | 시간 (단어당) | 비용 (단어당) | 1만 단어 평가 시간 |
|----------|-------------|-------------|------------------|
| 간단 평가 (1-5점) | 3초 | $0.05 | 8.3시간 |
| MQM 상세 평가 | 15초 | $0.25 | 41.7시간 |
| 후편집 기반 | 45초 | $0.75 | 125시간 |

**확장성 문제**:
- 대규모 콘텐츠 실시간 평가 불가능
- 일관성 유지 어려움
- 전문 분야 평가자 부족

### 평가자 간 일치도

전문 평가자들 사이에서도 일치도가 낮은 경우가 많습니다.

**Cohen's Kappa 분석**:
```
κ < 0.20: 거의 일치 없음
κ = 0.21-0.40: 낮은 일치도
κ = 0.41-0.60: 보통 일치도
κ = 0.61-0.80: 상당한 일치도
κ = 0.81-1.00: 거의 완벽한 일치도

실제 번역 품질 평가:
- 일반 텍스트: κ = 0.45-0.65
- 문학 텍스트: κ = 0.25-0.45
- 기술 문서: κ = 0.65-0.85
```

### 평가 피로도

장시간 평가는 품질 저하를 초래합니다.

```json
{
  "evaluation_quality_over_time": {
    "first_hour": {
      "accuracy": 0.92,
      "consistency": 0.88
    },
    "second_hour": {
      "accuracy": 0.87,
      "consistency": 0.81
    },
    "third_hour": {
      "accuracy": 0.79,
      "consistency": 0.72
    },
    "recommendation": "최대 2시간, 휴식 필수"
  }
}
```

## 언어별 특수 과제

### 형태론이 복잡한 언어

한국어, 터키어, 핀란드어 등은 형태론적 복잡성으로 인해 평가가 어렵습니다.

**한국어 번역 평가의 특수성**:

1. **존댓말 체계**
```json
{
  "source_en": "What's your name?",
  "korean_translations": [
    {
      "text": "이름이 뭐야?",
      "formality": "informal",
      "appropriate_for": ["friends", "children"]
    },
    {
      "text": "이름이 어떻게 되세요?",
      "formality": "formal",
      "appropriate_for": ["adults", "strangers", "business"]
    },
    {
      "text": "성함이 어떻게 되십니까?",
      "formality": "highly_formal",
      "appropriate_for": ["elderly", "formal_documents"]
    }
  ],
  "challenge": "맥락 없이 품질 판단 불가능"
}
```

2. **조사 선택**
```
"책을" vs "책이" vs "책은" vs "책만" vs "책도"
각각 다른 의미와 뉘앙스
자동 평가 메트릭으로 구분 불가능
```

### 저자원 언어의 평가 난제

저자원 언어는 평가 자체가 어렵습니다.

**저자원 언어 평가 장벽**:

| 과제 | 영향 | 예시 언어 |
|------|------|----------|
| 참조 번역 부족 | BLEU 등 사용 불가 | 크메르어, 암하라어 |
| 평가자 부족 | 인간 평가 어려움 | 티그리냐어, 조지아어 |
| 언어 도구 부재 | 전처리 불가능 | 버마어, 라오어 |
| 표준 정서법 없음 | 일관성 평가 불가 | 일부 아프리카 언어 |

**케이스 스터디: 요루바어 번역 평가**

```json
{
  "language": "Yoruba",
  "speakers": "40M",
  "challenges": {
    "tone_marks": "3가지 성조, 표기 불일치",
    "orthography": "표준 정서법 미정착",
    "resources": {
      "parallel_corpus": "100K 문장 (vs 영독 100M)",
      "monolingual": "500K 문장",
      "evaluators": "전문가 < 10명"
    },
    "mt_quality": "BLEU 15.3 (vs 영독 42.3)"
  }
}
```

### 문화적 뉘앙스

문화 특정적 표현은 번역과 평가 모두 어렵습니다.

**문화 불일치 사례**:

1. **색상 은유**
```
영어: "green with envy" (시기심)
한국어: "배가 아프다" (시기심)
직역: 의미 손실
의역: 문화 적응

평가 기준: 문화적 동등성? 직접성?
```

2. **경어 체계**
```
일본어: 존경어, 겸양어, 정중어
영어: Mr./Ms./Dr. 수준의 구분

번역 방향:
영→일: 맥락 추론 필요
일→영: 정보 손실 불가피
```

## 도메인 특화 과제

### 전문 용어 일관성

기술, 의료, 법률 번역은 용어 일관성이 중요합니다.

**용어 불일치 문제**:
```json
{
  "document": "Software Manual - 150 pages",
  "term_inconsistencies": [
    {
      "source": "user interface",
      "translations": [
        "사용자 인터페이스",
        "사용자 화면",
        "UI",
        "유저 인터페이스"
      ],
      "occurrences": [45, 23, 18, 9],
      "quality_impact": "심각한 혼란"
    },
    {
      "source": "database",
      "translations": ["데이터베이스", "DB", "자료기지"],
      "occurrences": [67, 34, 2],
      "quality_impact": "중간 수준 혼란"
    }
  ]
}
```

### 법률 번역의 특수성

법률 번역은 법적 효력을 가지므로 오류가 치명적입니다.

**법률 번역 오류 사례**:

| 원문 | 잘못된 번역 | 법적 영향 |
|------|------------|----------|
| "shall" (의무) | "할 수 있다" (선택) | 계약 무효화 가능 |
| "and/or" | "및/또는" (모호) | 해석 분쟁 |
| "force majeure" | "불가항력" vs "천재지변" | 적용 범위 차이 |

### 의료 번역의 생명 안전성

의료 번역 오류는 생명에 직접적 위험을 초래합니다.

**치명적 오류 사례** (실제 사건):

```
원문 (영어): "Take on an empty stomach"
오역 (스페인어): "Tome un estómago vacío"
의도: "공복에 복용하세요"
오역 해석: "빈 위장을 드세요"

결과: 환자가 약 복용법을 이해하지 못함
```

## 기계 번역 고유의 과제

### 환각 (Hallucination)

신경망 기계 번역은 때때로 원문에 없는 내용을 생성합니다.

**환각 유형**:

1. **내용 추가**
```json
{
  "source": "The meeting is at 3pm",
  "reference": "회의는 오후 3시입니다",
  "hallucinated": "중요한 회의는 오후 3시에 회의실 A에서 있습니다",
  "added": ["중요한", "회의실 A에서"],
  "severity": "medium"
}
```

2. **내용 변경**
```json
{
  "source": "Quarterly revenue decreased by 5%",
  "reference": "분기 수익이 5% 감소했습니다",
  "hallucinated": "분기 수익이 5% 증가했습니다",
  "changed": "감소 → 증가",
  "severity": "critical"
}
```

### 편향 (Bias)

기계 번역 시스템은 학습 데이터의 편향을 반영합니다.

**성별 편향 사례**:

```json
{
  "source_turkish": "O bir doktor",
  "literal": "That person is a doctor (성별 중립)",
  "biased_translations": {
    "english_male": "He is a doctor",
    "english_female": "She is a doctor",
    "frequency": {
      "male": 0.87,
      "female": 0.13
    }
  },
  "bias_source": "학습 데이터의 성별 분포"
}
```

**직업별 성별 편향**:

| 직업 | 남성 대명사 비율 | 여성 대명사 비율 |
|------|----------------|----------------|
| Doctor | 87% | 13% |
| Engineer | 92% | 8% |
| Nurse | 8% | 92% |
| Teacher | 35% | 65% |

### 저품질 학습 데이터

웹 크롤링 데이터는 품질이 일정하지 않습니다.

**학습 데이터 품질 문제**:
```json
{
  "data_source": "Web crawled parallel corpus",
  "quality_issues": {
    "misalignment": "15%",
    "machine_translated": "23%",
    "low_quality_human": "31%",
    "good_quality": "31%"
  },
  "impact": "모델 품질 상한선 제약"
}
```

## 후편집 (Post-Editing) 과제

### 후편집 노력 예측

기계 번역의 후편집이 필요한 정도를 예측하기 어렵습니다.

**후편집 노력 수준**:

| 수준 | 설명 | 시간 비율 |
|------|------|----------|
| 경미 | 문장 부호, 띄어쓰기 | 10-20% |
| 중간 | 어순 조정, 용어 변경 | 40-60% |
| 심각 | 문장 재구성 | 80-100% |
| 재번역 | 처음부터 다시 번역 | 150%+ |

### 후편집자 피로도

단조로운 후편집 작업은 집중력 저하를 초래합니다.

```json
{
  "post_editing_fatigue": {
    "minor_errors_only": {
      "description": "단순 오류만 반복 수정",
      "fatigue_rate": "높음",
      "error_detection_drop": "25% after 2 hours"
    },
    "mixed_difficulty": {
      "description": "다양한 수준의 오류",
      "fatigue_rate": "중간",
      "error_detection_drop": "15% after 2 hours"
    }
  }
}
```

## 품질 추정 (Quality Estimation) 과제

### 참조 번역 없이 평가하기

QE는 참조 번역 없이 번역 품질을 예측해야 합니다.

**QE의 어려움**:
- 원문과 번역문만으로 품질 판단
- 언어 쌍별로 다른 모델 필요
- 저자원 언어에서 정확도 낮음

**QE 성능 비교**:
```json
{
  "language_pairs": {
    "en_de": {
      "pearson_correlation": 0.82,
      "data_availability": "high"
    },
    "en_zh": {
      "pearson_correlation": 0.76,
      "data_availability": "high"
    },
    "en_sw": {
      "pearson_correlation": 0.54,
      "data_availability": "low"
    }
  }
}
```

## 번역 메모리 시스템의 한계

### 부분 일치의 품질 불확실성

번역 메모리는 유사한 문장을 재사용하지만, 부분 일치는 위험합니다.

**부분 일치 문제**:
```json
{
  "tm_entry": {
    "source": "Click the Save button",
    "target": "저장 버튼을 클릭하세요",
    "match": "100%"
  },
  "new_segment": "Click the Delete button",
  "tm_suggestion": {
    "text": "삭제 버튼을 클릭하세요",
    "match": "85%",
    "changed": ["Save → Delete", "저장 → 삭제"]
  },
  "risk": "자동 적용 시 오류 가능성"
}
```

### 맥락 무시

TM은 문장 단위로 작동하여 맥락을 무시합니다.

```
문서 A: "The application is running"
TM: "애플리케이션이 실행 중입니다"

문서 B (채용 공고): "Review the application"
TM 제안: "애플리케이션을 검토하세요" (잘못됨)
올바른 번역: "지원서를 검토하세요"
```

## 표준화 부재의 문제

### 평가 기준의 파편화

산업 전반에 통일된 품질 기준이 없습니다.

**현재 사용 중인 다양한 표준**:

| 표준/프레임워크 | 개발자 | 채택 영역 | 호환성 |
|---------------|--------|----------|--------|
| MQM | QT21/TAUS | 학계, 일부 기업 | 낮음 |
| DQF | TAUS | 번역 산업 | 중간 |
| SAE J2450 | SAE | 자동차 산업 | 없음 |
| ISO 17100 | ISO | 번역 서비스 | 중간 |
| LISA QA | LISA (defunct) | 레거시 시스템 | 낮음 |

### 데이터 형식 불일치

품질 평가 데이터가 표준화되지 않았습니다.

```json
{
  "format_diversity": [
    "XML (XLIFF)",
    "JSON (custom schemas)",
    "CSV (various columns)",
    "Excel (company-specific)",
    "Proprietary formats"
  ],
  "interoperability": "매우 낮음"
}
```

## WIA-LANG-009 표준이 해결할 과제

### 통합 평가 프레임워크

다양한 평가 메트릭을 통합하는 표준 제공:
- 자동 메트릭 (BLEU, COMET 등)
- 인간 평가 (MQM, DQF 등)
- 품질 추정 (QE)

### 저자원 언어 지원

모든 언어에 적용 가능한 유연한 프레임워크:
- 최소 리소스 요구사항
- 언어 독립적 평가 방법
- 전이 학습 지원

### 상호운용 가능한 데이터 형식

표준화된 품질 데이터 교환:
- JSON 기반 명세
- 기존 시스템과의 호환성
- 확장 가능한 스키마

## 핵심 요약

**주요 과제:**

1. **평가의 주관성**: 평가자 간 불일치, 맥락 의존성, 문화적 차이

2. **자동 메트릭 한계**: BLEU의 의미론적 한계, 언어별 편향, 계산 비용

3. **인간 평가 문제**: 높은 비용, 확장성 부족, 평가자 피로도

4. **언어별 특수성**: 형태론 복잡성, 저자원 언어, 문화적 뉘앙스

5. **기계 번역 문제**: 환각, 편향, 저품질 학습 데이터

6. **표준화 부재**: 파편화된 평가 기준, 데이터 형식 불일치

## 복습 문제

1. 번역 품질 평가에서 주관성이 문제가 되는 이유와 이를 감소시키는 방법을 설명하시오.

2. BLEU 메트릭의 구체적인 한계 3가지를 예시와 함께 서술하시오.

3. 저자원 언어 번역 평가에서 직면하는 주요 장벽을 나열하고 해결 방안을 제시하시오.

4. 기계 번역의 환각 현상이 무엇이며, 왜 발생하는지 설명하시오.

5. 번역 메모리 시스템의 부분 일치가 품질 문제를 일으킬 수 있는 상황을 예시를 들어 설명하시오.

6. 현재 번역 품질 평가 표준이 파편화된 상황과 이것이 산업에 미치는 영향을 논하시오.
