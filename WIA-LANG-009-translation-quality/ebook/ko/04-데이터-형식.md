# 제4장: Phase 1 - 데이터 형식 사양

## 표준화된 데이터 구조의 중요성

번역 품질 데이터의 표준화는 시스템 간 상호운용성, 재현 가능한 평가, 그리고 장기적 데이터 축적을 가능하게 합니다. Phase 1은 WIA-LANG-009의 기초를 형성하는 데이터 형식 명세를 정의합니다.

## 핵심 데이터 스키마

### 문서 메타데이터

모든 번역 품질 평가의 최상위 구조입니다.


### 필수 및 선택 필드

| 필드 경로 | 필수 | 타입 | 설명 |
|----------|------|------|------|
| `wia_lang_009.version` | ✓ | string | 표준 버전 |
| `document.id` | ✓ | string | 문서 고유 식별자 |
| `metadata.source_language` | ✓ | string | BCP 47 언어 코드 |
| `metadata.target_language` | ✓ | string | BCP 47 언어 코드 |
| `metadata.domain` | ✓ | enum | 도메인 분류 |
| `metadata.created` | ✓ | datetime | ISO 8601 형식 |
| `quality_summary.overall_score` | ✓ | number | 0-100 범위 |
| `metadata.subdomain` | | string | 세부 도메인 |
| `statistics` | | object | 통계 정보 |

## 데이터 검증

### 스키마 검증 예시

```python
import jsonschema
import json

# 스키마 로드
with open('quality-assessment.schema.json') as f:
    schema = json.load(f)

# 데이터 로드
with open('en-ko_technical_20250101_quality.json') as f:
    data = json.load(f)

# 검증
try:
    jsonschema.validate(instance=data, schema=schema)
    print("✓ 데이터가 WIA-LANG-009 스키마를 준수합니다")
except jsonschema.exceptions.ValidationError as e:
    print(f"✗ 검증 오류: {e.message}")
```

### CLI 도구

```bash
# 단일 파일 검증
wia-lang-009 validate --schema quality-assessment \
  --file en-ko_technical_20250101_quality.json

# 디렉토리 전체 검증
wia-lang-009 validate --schema quality-assessment \
  --dir ./quality-data/assessments/

# 형식 변환
wia-lang-009 convert --from json --to csv \
  --input quality.json --output quality.csv

# 통계 생성
wia-lang-009 stats --input quality.json --output report.html
```

## 핵심 요약

**데이터 형식:**

1. **계층적 구조**: 문서 → 세그먼트 → 품질 → 메트릭/오류

2. **표준 메트릭**: BLEU, COMET, chrF, TER, BERTScore 등 일관된 형식

3. **오류 주석**: MQM 기반 체계적 오류 분류 및 심각도

4. **용어 관리**: 용어집 및 일관성 검증 데이터

5. **다중 형식**: JSON (기본), CSV (분석), XML (XLIFF 호환)

6. **검증 체계**: JSON Schema 기반 엄격한 데이터 검증
