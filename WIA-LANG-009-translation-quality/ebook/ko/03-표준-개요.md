# 제3장: WIA-LANG-009 표준 개요

## 표준의 비전과 미션

### 비전

**"모든 언어, 모든 사람을 위한 고품질 번역"**

WIA-LANG-009 표준은 언어의 다양성을 존중하면서도 일관되고 측정 가능한 번역 품질을 보장하는 통합 프레임워크를 제공합니다.

### 미션

1. **보편적 접근**: 모든 언어 쌍에 적용 가능한 품질 평가 체계
2. **객관적 측정**: 재현 가능하고 투명한 품질 메트릭
3. **실용적 통합**: 기존 번역 워크플로우와의 원활한 통합
4. **지속적 발전**: 기술 발전과 함께 진화하는 표준

## 弘益人間 철학의 구현

> **弘益人間 (홍익인간) - "널리 인간을 이롭게 하라"**

### 철학의 실천

WIA-LANG-009 표준은 弘益人間 철학을 다음과 같이 구현합니다:

**1. 언어 평등 (Language Equality)**
```json
{
  "principle": "언어 평등",
  "implementation": {
    "high_resource": "영어, 중국어 등 - 고급 메트릭 지원",
    "mid_resource": "한국어, 아랍어 등 - 적응형 평가",
    "low_resource": "요루바어, 크메르어 등 - 최소 리소스 평가",
    "guarantee": "모든 언어가 동등한 품질 표준 적용"
  }
}
```

**2. 접근성 (Accessibility)**
- 오픈소스 도구 제공
- 명확한 문서화
- 다국어 가이드라인
- 무료 기본 구현

**3. 투명성 (Transparency)**
- 모든 메트릭 공식 공개
- 평가 기준 명시
- 데이터 형식 표준화
- 인증 절차 투명화

**4. 협력 (Collaboration)**
- 산업, 학계, 커뮤니티 협력
- 오픈 표준 개발
- 피드백 기반 개선
- 글로벌 기여 환영

### Phase 1: 데이터 형식 (Data Format)

**목적**: 번역 품질 데이터의 표준화된 표현

**핵심 구성 요소**:

| 구성 요소 | 설명 | 형식 |
|----------|------|------|
| Quality Metrics Schema | 품질 메트릭 정의 및 값 | JSON Schema |
| Evaluation Data | 평가 결과 데이터 | JSON |
| Error Annotation | 오류 분류 및 주석 | JSON-LD |
| Segment Metadata | 문장 단위 메타데이터 | JSON |

**데이터 계층 구조**:
```json
{
  "wia_lang_009": {
    "version": "1.0",
    "document": {
      "id": "doc-001",
      "source_language": "en",
      "target_language": "ko",
      "domain": "technical",
      "segments": [
        {
          "id": "seg-001",
          "source": "...",
          "target": "...",
          "quality": {
            "overall_score": 0.85,
            "metrics": {...},
            "errors": [...]
          }
        }
      ]
    }
  }
}
```

### Phase 2: API 인터페이스

**목적**: 프로그래매틱한 품질 평가 접근

**API 카테고리**:

1. **Assessment API**
   - 자동 품질 평가
   - 메트릭 계산
   - 실시간 품질 예측

2. **Annotation API**
   - 오류 주석 생성
   - 평가자 인터페이스
   - 품질 피드백 수집

3. **Reporting API**
   - 품질 리포트 생성
   - 통계 분석
   - 대시보드 데이터

**RESTful 엔드포인트 예시**:
```
POST   /api/v1/assess              # 품질 평가 수행
GET    /api/v1/metrics/{id}        # 메트릭 조회
POST   /api/v1/annotate            # 오류 주석
GET    /api/v1/report/{doc_id}     # 품질 리포트
```

### Phase 3: 프로토콜

**목적**: 표준화된 품질 보증 프로세스

**프로토콜 유형**:

| 프로토콜 | 적용 단계 | 목적 |
|---------|----------|------|
| Pre-translation QA | 번역 전 | 원문 품질 확인 |
| In-translation QA | 번역 중 | 실시간 품질 모니터링 |
| Post-translation QA | 번역 후 | 최종 품질 검증 |
| Continuous QA | 지속적 | 피드백 루프 |

**워크플로우 예시**:
```
원문 입력
   ↓
[Phase 3.1: 원문 품질 검증]
   ↓
기계 번역 / 인간 번역
   ↓
[Phase 3.2: 자동 품질 평가]
   ↓
품질 점수 < 임계값?
   ↓ Yes
[Phase 3.3: 인간 후편집]
   ↓
[Phase 3.4: 최종 검증]
   ↓
번역 완료
```

### Phase 4: 시스템 통합

**목적**: 기존 번역 생태계와의 통합

**통합 대상**:

1. **번역 관리 시스템 (TMS)**
   - Phrase, Smartling, XTM
   - 품질 메트릭 임베딩
   - 워크플로우 자동화

2. **CAT 도구**
   - SDL Trados, memoQ, Memsource
   - 실시간 품질 피드백
   - 번역 메모리 품질 평가

3. **MT 엔진**
   - Google Translate, DeepL, NLLB
   - 품질 추정 (QE)
   - 자동 후편집 결정

4. **기업 시스템**
   - CMS, DAM, PIM 통합
   - CI/CD 파이프라인
   - 품질 대시보드

## 품질 차원 정의

WIA-LANG-009는 5가지 핵심 품질 차원을 정의합니다.

### 1. 정확성 (Accuracy)

**정의**: 원문의 의미를 정확하게 전달하는 정도

**측정 요소**:
```json
{
  "accuracy": {
    "dimensions": [
      {
        "name": "semantic_fidelity",
        "description": "의미적 충실도",
        "weight": 0.40
      },
      {
        "name": "terminology_correctness",
        "description": "용어 정확성",
        "weight": 0.30
      },
      {
        "name": "factual_accuracy",
        "description": "사실적 정확성",
        "weight": 0.30
      }
    ]
  }
}
```

**오류 유형**:
- 오역 (Mistranslation)
- 누락 (Omission)
- 추가 (Addition)
- 의미 왜곡 (Meaning Distortion)

### 2. 유창성 (Fluency)

**정의**: 목표 언어로서의 자연스러움

**측정 요소**:
```json
{
  "fluency": {
    "dimensions": [
      {
        "name": "grammaticality",
        "description": "문법적 정확성",
        "weight": 0.35
      },
      {
        "name": "naturalness",
        "description": "자연스러움",
        "weight": 0.35
      },
      {
        "name": "readability",
        "description": "가독성",
        "weight": 0.30
      }
    ]
  }
}
```

**오류 유형**:
- 문법 오류 (Grammar Error)
- 어색한 표현 (Unnatural Expression)
- 어휘 선택 부적절 (Poor Word Choice)
- 문체 불일치 (Style Inconsistency)

### 3. 일관성 (Consistency)

**정의**: 문서 내 및 문서 간 용어와 스타일의 통일성

**측정 범위**:

| 일관성 유형 | 범위 | 예시 |
|-----------|------|------|
| 용어 일관성 | 문서 전체 | "user interface" 항상 동일하게 |
| 스타일 일관성 | 문서 전체 | 존댓말/반말 통일 |
| 형식 일관성 | 문서 전체 | 날짜 형식, 숫자 형식 |
| 브랜드 일관성 | 프로젝트 전체 | 브랜드명, 제품명 |

### 4. 적절성 (Adequacy)

**정의**: 목적, 독자, 도메인에 맞는 번역

**평가 기준**:
```json
{
  "adequacy": {
    "factors": [
      {
        "name": "audience_appropriateness",
        "description": "독자층 적합성",
        "levels": ["expert", "professional", "general", "children"]
      },
      {
        "name": "purpose_fit",
        "description": "목적 부합성",
        "types": ["informative", "persuasive", "instructional", "legal"]
      },
      {
        "name": "cultural_adaptation",
        "description": "문화적 적응",
        "aspects": ["idioms", "metaphors", "cultural_references"]
      }
    ]
  }
}
```

### 5. 형식 (Format)

**정의**: 레이아웃, 서식, 특수 요소의 정확한 유지

**검증 항목**:
- HTML/XML 태그 일치
- 플레이스홀더 보존
- 숫자, 날짜, 시간 형식
- 특수 문자 및 기호
- 레이아웃 구조

## 오류 분류 체계

### MQM 기반 오류 분류

WIA-LANG-009는 MQM (Multidimensional Quality Metrics)을 기반으로 확장된 오류 분류를 사용합니다.

```
오류 분류
├── Accuracy (정확성)
│   ├── Mistranslation (오역)
│   ├── Omission (누락)
│   ├── Addition (추가)
│   ├── Untranslated (미번역)
│   └── Improper exact TM match (부적절한 TM 사용)
│
├── Fluency (유창성)
│   ├── Grammar (문법)
│   │   ├── Word order (어순)
│   │   ├── Tense/mood/aspect (시제/서법/상)
│   │   └── Agreement (일치)
│   ├── Spelling (철자)
│   ├── Typography (조판)
│   └── Unintelligible (이해 불가)
│
├── Terminology (용어)
│   ├── Inconsistent with termbase (용어집 불일치)
│   ├── Inconsistent use of terminology (용어 사용 불일치)
│   └── Wrong term (잘못된 용어)
│
├── Style (스타일)
│   ├── Inappropriate register (부적절한 격식)
│   ├── Inconsistent style (스타일 불일치)
│   └── Unidiomatic (관용적이지 않음)
│
└── Locale convention (지역 관습)
    ├── Currency format (통화 형식)
    ├── Date/time format (날짜/시간 형식)
    ├── Number format (숫자 형식)
    └── Address format (주소 형식)
```

### 오류 심각도

**4단계 심각도 분류**:

| 수준 | 설명 | 영향 | 점수 영향 |
|------|------|------|----------|
| **Critical** | 의미 완전 왜곡, 안전 위험 | 사용 불가 | -10점 |
| **Major** | 의미 부분 왜곡, 이해 곤란 | 수정 필수 | -5점 |
| **Minor** | 자연스럽지 않음, 경미한 오류 | 수정 권장 | -1점 |
| **Neutral** | 선호도 차이 | 수용 가능 | -0.1점 |

**심각도 판단 예시**:
```json
{
  "examples": [
    {
      "source": "Do not use near fire",
      "translation": "불 근처에서 사용하세요",
      "error": "의미 반대",
      "severity": "critical",
      "reason": "안전 지침 반대로 번역"
    },
    {
      "source": "Save the file",
      "translation": "파일을 저장하십니다",
      "error": "문법 (시제 불일치)",
      "severity": "minor",
      "reason": "의미는 전달되나 어색함"
    }
  ]
}
```

## 품질 점수 체계

### 종합 품질 점수 (Overall Quality Score)

**계산 공식**:
```
OQS = (w_acc × Accuracy + w_flu × Fluency + w_con × Consistency +
       w_ade × Adequacy + w_fmt × Format) × 100

기본 가중치:
w_acc = 0.35  (정확성)
w_flu = 0.25  (유창성)
w_con = 0.20  (일관성)
w_ade = 0.15  (적절성)
w_fmt = 0.05  (형식)
```

### 품질 등급

**5단계 품질 등급**:

| 등급 | 점수 범위 | 설명 | 권장 조치 |
|------|----------|------|----------|
| Excellent | 90-100 | 출판 가능 품질 | 최소 검토 |
| Good | 75-89 | 경미한 후편집 | 간단한 수정 |
| Acceptable | 60-74 | 중간 수준 후편집 | 체계적 수정 |
| Poor | 40-59 | 대폭 후편집 | 전면 수정 |
| Unacceptable | 0-39 | 재번역 필요 | 재번역 |

## 표준 준수 수준

### 3단계 준수 레벨

**Level 1: Basic Compliance (기본 준수)**
- Phase 1 데이터 형식 사용
- 최소 3개 자동 메트릭 지원
- 오류 분류 체계 적용

**Level 2: Advanced Compliance (고급 준수)**
- Phase 1 + Phase 2 구현
- 5개 이상 메트릭 지원
- API 통합
- 인간 평가 프레임워크

**Level 3: Full Compliance (완전 준수)**
- Phase 1-4 전체 구현
- 하이브리드 평가 체계
- 시스템 통합
- WIA 인증 획득

## 핵심 요약

**표준 구조:**

1. **4-Phase 아키텍처**: 데이터 형식 → API → 프로토콜 → 시스템 통합

2. **5개 품질 차원**: 정확성, 유창성, 일관성, 적절성, 형식
