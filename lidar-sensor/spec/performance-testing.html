WIA-SEMI-014: LiDAR Performance Testing Methodology

Test Standard Version: 1.0
Applicable to: WIA-SEMI-014 Specification v1.0+

1. Range Performance Testing

1.1 Static Target Range Test

Objective: Verify maximum detection range and range accuracy across reflectivity range.

Equipment Required:

  • Calibrated flat targets: 10%, 20%, 50%, 80%, 90% reflectivity @ sensor wavelength
  • Optical rail or linear stage (300m+ length for long-range sensors)
  • Laser rangefinder reference (±1mm accuracy)
  • Data acquisition system

Procedure:

  1. Mount LiDAR sensor on stable platform with optical axis parallel to rail
  2. Position 10% reflectivity target at 10m distance, verify detection
  3. Record 100 range measurements, calculate mean and standard deviation
  4. Increment target distance in 10m steps until sensor no longer detects target
  5. Record maximum detection distance
  6. Repeat for each reflectivity level

Pass Criteria:

  • Maximum range @ 10% reflectivity ≥ 200m (or manufacturer specification)
  • Range accuracy (1σ) ≤ ±2cm across full range
  • Range precision (1σ) ≤ ±1cm

2. Angular Resolution and FOV Testing

2.1 Field of View Verification

Procedure:

  1. Position LiDAR sensor at known reference point
  2. Place targets at ±45°, ±60°, ±90° horizontal angles (for 180° HFOV sensor)
  3. Verify detection at specified angles
  4. Measure actual FOV boundaries where detection drops to 50%
  5. Repeat for vertical FOV

2.2 Angular Resolution Test

Equipment: Two small targets (10cm × 10cm) on precision angular mount

  1. Position targets at 50m distance, separated by specified angular resolution (e.g., 0.1°)
  2. Verify LiDAR resolves targets as separate objects in point cloud
  3. Decrease angular separation until targets merge in point cloud
  4. Record minimum resolvable angle

3. Environmental Testing

3.1 Temperature Cycling

Phase Temperature Duration Ramp Rate
Cold Soak -40°C 4 hours ≤5°C/min
Hot Soak +85°C 4 hours ≤5°C/min
Cycles -40°C to +85°C 1000 cycles -

Functional Test: Perform range accuracy test at -40°C, +25°C (room temp), and +85°C. Verify performance degradation within acceptable limits (±10% range, ±5cm accuracy).

3.2 Rain Testing (per ISO 22628)

Setup: Rainfall simulator with controlled precipitation rate

Rain Intensity Test Distance Minimum Detection Rate
Light (10mm/h) 200m @ 10% reflectivity ≥95% of clear weather
Heavy (50mm/h) 100m @ 10% reflectivity ≥90% detection rate

4. Point Cloud Quality Assessment

4.1 Point Density Test

Test Configuration:
- Target: White wall @ 80% reflectivity
- Distance: 10m
- Duration: 10 seconds @ 10 Hz frame rate

Calculation:
Points per frame = Total points collected / Number of frames
Points per square meter = Points per frame / (Target area in m²)

Minimum Requirement: ≥1000 points/m² @ 10m distance

4.2 Intensity Calibration Verification

  1. Position calibrated reflectivity targets (10%, 30%, 50%, 80%) at same distance
  2. Collect point cloud data for 100 frames
  3. Calculate mean intensity for each target
  4. Verify monotonic relationship: I(80%) > I(50%) > I(30%) > I(10%)
  5. Check intensity uniformity across FOV (coefficient of variation <10%)

5. Safety Testing

5.1 Laser Safety Measurement (IEC 60825-1)

Equipment: Calibrated laser power meter, integrating sphere, collimating optics

Procedure:

  1. Measure laser output at sensor aperture using calibrated detector
  2. Record peak power, pulse duration, PRF (pulse repetition frequency)
  3. Calculate AEL (Accessible Emission Limit) per IEC 60825-1: 
    Class 1: AEL = 0.39 mW @ 905nm (for viewing through optics)
Class 1: AEL = 10 mW @ 1550nm (eye-safe IR)
  4. Verify sensor output ≤ AEL for specified class
  5. Test worst-case scenarios: maximum power mode, single point stare

Pass Criteria: Measured output ≤ 100% of AEL for claimed safety class

6. Interference and Multi-Sensor Testing

6.1 LiDAR-to-LiDAR Interference

Setup: Two or more identical LiDAR sensors operating simultaneously

  1. Operate single LiDAR, record baseline point cloud quality metrics
  2. Activate second LiDAR sensor aimed at overlapping target area
  3. Measure degradation in range accuracy, false positive rate, detection rate
  4. Repeat with 3, 5, 10 sensors (simulating dense traffic scenario)

Pass Criteria (ToF sensors): <10% degradation with 3 sensors, <20% with 10 sensors

Pass Criteria (FMCW sensors): No degradation (interference immune by design)

7. Data Quality Metrics

Metric Calculation Target Value
Detection Rate (Detected Objects / Ground Truth Objects) × 100% ≥99% for safety-critical objects
False Positive Rate Spurious detections per kilometer driven <1 per 1000 km
Localization Error RMS error in object position (X, Y, Z) ≤10cm @ 100m distance
Latency Time from photon detection to data output <100ms (perception), <50ms (safety)

8. Test Report Format

All test reports SHALL include:

  1. Test Identification: Date, test engineer, DUT (Device Under Test) serial number
  2. Test Conditions: Temperature, humidity, equipment used, calibration dates
  3. Results Summary: Table of pass/fail for each test
  4. Detailed Data: Raw measurements, plots, statistical analysis
  5. Non-Conformances: Any failures with root cause analysis
  6. Certification: Test engineer signature, approval authority
