Security & Cryptography

Cryptographic Foundations

Digital wallet security relies on robust cryptographic principles. Understanding these foundations is essential for implementing secure wallet systems.

Public Key Cryptography

The backbone of digital wallets, public key cryptography uses asymmetric key pairs:

  • Private Key: Secret key that must never be shared. Used to sign transactions and prove ownership.
  • Public Key: Derived from private key, can be shared freely. Used to verify signatures and receive payments.
  • Address: Hash of public key, serves as the wallet identifier.

Encryption Algorithms

  • AES-256: Symmetric encryption for data at rest
  • RSA-2048+: Asymmetric encryption for key exchange
  • ECDSA: Elliptic Curve Digital Signature Algorithm for Bitcoin/Ethereum
  • EdDSA: Edwards-curve Digital Signature Algorithm (Curve25519)

Hashing Functions

  • SHA-256: Secure Hash Algorithm for Bitcoin addresses
  • Keccak-256: Used in Ethereum for address generation
  • RIPEMD-160: Combined with SHA-256 for Bitcoin addresses
  • Argon2: Memory-hard function for password hashing

Key Management

Proper key management is critical to wallet security. Loss of keys means loss of funds.

Key Generation

  1. Entropy Collection: Gather high-quality random data from secure sources
  2. Seed Generation: Create mnemonic seed phrase (12-24 words, BIP39)
  3. Master Key Derivation: Use PBKDF2 or similar to derive master key
  4. Child Key Derivation: Generate hierarchical keys (BIP32/BIP44)

Key Storage Options

  • Software Storage: Encrypted on device (mobile/desktop)
  • Hardware Security Modules (HSM): Dedicated cryptographic processors
  • Hardware Wallets: Offline devices like Ledger, Trezor
  • Secure Enclaves: iOS Secure Enclave, Android StrongBox
  • Multi-Party Computation (MPC): Distributed key shares

Backup and Recovery

  • Mnemonic Phrases: 12-24 word backup for HD wallets
  • Encrypted Backups: Cloud storage with strong encryption
  • Shamir's Secret Sharing: Split recovery into multiple shares
  • Social Recovery: Guardians help recover wallet

Authentication Mechanisms

Biometric Authentication

Modern devices offer hardware-backed biometric security:

  • Fingerprint: Touch ID, in-display sensors
  • Facial Recognition: Face ID, 3D depth sensing
  • Iris Scanning: High-security applications
  • Voice Recognition: Emerging authentication method

Multi-Factor Authentication (MFA)

  • Something You Know: Password, PIN, security questions
  • Something You Have: Phone, hardware token, smart card
  • Something You Are: Biometrics (fingerprint, face, iris)
  • Somewhere You Are: Geolocation, IP address verification

Time-Based One-Time Passwords (TOTP)

Implementing TOTP adds an additional security layer:

  • Synchronized time-based codes that expire every 30 seconds
  • Compatible with Google Authenticator, Authy, 1Password
  • Offline generation doesn't require network access

Transaction Security

Digital Signatures

Every transaction must be cryptographically signed to prove authorization:

  1. Create transaction data structure
  2. Hash the transaction (SHA-256 or similar)
  3. Sign hash with private key (ECDSA/EdDSA)
  4. Broadcast transaction with signature
  5. Network verifies signature with public key

Transaction Limits

  • Daily Limits: Maximum spend per 24-hour period
  • Per-Transaction Limits: Maximum single transaction amount
  • Velocity Checks: Rate limiting on transaction frequency
  • Geographic Restrictions: Block transactions from high-risk regions

Confirmation Requirements

  • Low Value: Biometric only (under $50)
  • Medium Value: Biometric + PIN ($50-$500)
  • High Value: Biometric + PIN + 2FA (over $500)
  • Very High Value: Additional human approval (over $10,000)

Fraud Prevention

AI-Powered Fraud Detection

Machine learning models analyze patterns to identify suspicious activity:

  • Transaction amount anomalies
  • Unusual merchant categories
  • Geographic impossibilities (rapid location changes)
  • Device fingerprint changes
  • Behavioral biometrics (typing patterns, swipe gestures)

Risk Scoring

Assign risk scores to transactions based on multiple factors:

  • Low Risk (0-30): Proceed automatically
  • Medium Risk (31-60): Additional authentication required
  • High Risk (61-85): Manual review or block
  • Critical Risk (86-100): Automatic block, user notification

Common Attack Vectors

  1. Phishing: Fake websites/apps to steal credentials
  2. Malware: Keyloggers, screen recorders, clipboard hijackers
  3. SIM Swapping: Hijack phone number for SMS 2FA
  4. Man-in-the-Middle: Intercept communication between user and service
  5. Social Engineering: Manipulate users into revealing secrets

Privacy-Preserving Technologies

Zero-Knowledge Proofs

Prove knowledge without revealing the information itself. Applications include:

  • Anonymous credentials (prove age without revealing birthdate)
  • Private transactions (hide amounts and participants)
  • Selective disclosure (share only necessary information)

Confidential Transactions

Hide transaction amounts while maintaining verifiability using cryptographic commitments.

Mixing Services

Enhance privacy by breaking the link between sender and receiver addresses (use with caution for regulatory compliance).

Hardware Wallet Integration

Benefits

  • Private keys never leave the device
  • Immune to malware on host computer
  • Physical confirmation for transactions
  • Secure element chip protection

Popular Hardware Wallets

  • Ledger: Nano S, Nano X with Bluetooth
  • Trezor: Model One, Model T with touchscreen
  • KeepKey: Large display for verification
  • BitBox: Minimalist Swiss design

Integration Protocol

  1. Connect hardware wallet via USB or Bluetooth
  2. Wallet app requests transaction signing
  3. User verifies transaction on device screen
  4. Physical button press confirms transaction
  5. Device signs transaction and returns signature
  6. App broadcasts signed transaction to network

Quantum-Resistant Cryptography

Preparing for the quantum computing era:

  • Lattice-Based Cryptography: NTRU, Kyber
  • Hash-Based Signatures: SPHINCS+, XMSS
  • Code-Based Cryptography: McEliece, BIKE
  • Multivariate Cryptography: Rainbow
Future-Proofing: While quantum computers capable of breaking current cryptography don't exist yet, implementing quantum-resistant algorithms now ensures long-term security as the technology evolves.

Security Best Practices

  1. Defense in Depth: Multiple layers of security
  2. Principle of Least Privilege: Minimal permissions required
  3. Regular Security Audits: Third-party penetration testing
  4. Bug Bounty Programs: Incentivize responsible disclosure
  5. Incident Response Plan: Prepare for security breaches
  6. User Education: Teach users about security risks
  7. Keep Software Updated: Patch vulnerabilities promptly
  8. Monitor Threat Intelligence: Stay informed about new attacks
