Chapter 7: Calibration and Personalization

弘益人間 (Hongik Ingan)

"Benefit All Humanity"

Personalized calibration and continuous adaptation optimize prosthetic control for each individual's unique physiology, preferences, and daily activities, ensuring reliable performance for all users.

7.1 The Need for Personalization

No two prosthetic users are identical. Residual limb anatomy, muscle strength, EMG signal characteristics, motor control strategies, and functional needs vary dramatically across individuals. Generic "one-size-fits-all" prosthetic control systems deliver suboptimal performance for everyone. Proper calibration and personalization adapt systems to individual characteristics, dramatically improving functionality and user satisfaction.

7.1.1 Sources of Inter-Individual Variability

Anatomical Differences: Residual limb length, muscle volume, skin condition, subcutaneous fat thickness, and nerve distribution vary widely. These affect EMG signal amplitude (±300%), electrode positioning options, and available control sites.

Physiological Variability: Muscle fiber composition (slow vs. fast-twitch ratios), neuromuscular efficiency, and motor unit firing patterns differ. Some users generate strong, clear EMG signals with minimal effort; others produce weak or noisy signals requiring maximum contraction.

Amputation-Specific Factors: Cause of amputation (trauma vs. disease), time since amputation (acute vs. chronic), presence of TMR/RPNI, and surgical technique all influence available musculature and signal quality.

Cognitive and Motor Learning: Prior experience with prosthetics, motor learning ability, cognitive capacity, and age affect how quickly users learn new control strategies and their ultimate performance ceiling.

Lifestyle and Goals: Occupational requirements (manual labor vs. desk work), hobbies, living situation (independent vs. assisted), and personal preferences shape which prosthetic functions are most valuable.

7.2 Initial Calibration Procedures

Proper initial setup establishes baseline performance and trains the system to the user's specific signals.

7.2.1 Signal Acquisition Calibration

Electrode Placement Optimization:

  1. Anatomical Survey: Prosthetist palpates residual limb to identify muscle locations, assesses skin quality and scarring.
  2. Signal Screening: Place electrodes at candidate sites; user performs standard contractions; EMG amplitudes and signal-to-noise ratios are measured.
  3. Cross-Talk Assessment: User activates each muscle independently; EMG from other sites is monitored to ensure good selectivity (low cross-talk).
  4. Final Placement: Based on signal quality and selectivity, optimal electrode locations are marked for socket integration.

Typical duration: 30-60 minutes for transradial user with 6-8 electrode sites.

Gain Adjustment: Amplifier gains are set to utilize full ADC range without clipping. User performs maximum voluntary contractions (MVC); gains adjusted so MVCs approach but don't exceed ADC maximum (e.g., 95% of range).

Baseline Noise Measurement: With limb completely relaxed, baseline noise is recorded for each channel. This establishes minimum signal thresholds for control algorithms.

7.2.2 Control Threshold Calibration

For Direct/Proportional Control:

Rest Threshold: Set just above baseline noise level. Signals below this threshold are considered "off" (muscle relaxed). Typical values: 5-15% of MVC, adjusted per channel.

Activation Threshold: Minimum signal level to trigger prosthetic action. Set above rest threshold to prevent accidental activation. Typical: 15-25% of MVC.

Maximum Level: Signal level corresponding to maximum prosthetic speed/force. Usually set to 80-90% of MVC (not 100%) to prevent fatigue-induced loss of control.

Mapping Function: Relationship between signal amplitude and prosthetic response. Linear mapping is simplest (proportional); some users prefer non-linear (e.g., exponential for finer low-force control, or deadband-linear-saturation curves).

7.2.3 Pattern Recognition Training

Gesture Library Selection: Prosthetist and user collaborate to choose desired gestures based on functional goals. Common starting set (8-10 gestures):

  • Hand open
  • Hand close / Power grasp
  • Pinch / Precision grip
  • Wrist extension
  • Wrist flexion
  • Wrist pronation (palm down)
  • Wrist supination (palm up)
  • Rest / No movement

Data Collection Protocol:

  1. Demonstration: Prosthetist demonstrates each gesture; user practices to ensure they understand the intended movement.
  2. Recording: User performs each gesture 4-5 times, holding each for 3-5 seconds. System records EMG from all channels.
  3. Rest Periods: 5-10 second breaks between gestures prevent fatigue.
  4. Quality Check: Real-time signal quality monitoring flags inconsistent or weak recordings for immediate re-collection.

Total training data collection: 5-8 minutes for 8 gestures.

Classifier Training: System extracts features, trains classifier (typically takes 5-30 seconds depending on algorithm complexity), and validates on held-out test data.

Performance Verification: User tests the trained classifier with real-time feedback. Target: ≥90% classification accuracy. If performance is inadequate, additional training data is collected or gesture set adjusted.

7.3 User-Specific Customization

Beyond initial calibration, systems can be customized to individual preferences and use patterns.

7.3.1 Control Parameter Tuning

Response Speed: Maximum prosthetic movement velocity. Some users prefer fast, responsive control for efficiency; others prefer slower, more controlled movements for precision. WIA standard recommends user-adjustable speed: 25%-200% of nominal.

Grip Force Limits: Maximum allowed grip force. Protects objects and prevents uncomfortable proprioception. Typical range: 20-140 N, user-adjustable.

Latency/Smoothing Trade-Off: More signal smoothing reduces noise/misclassifications but increases latency. Users can adjust this balance based on their priorities (reliability vs. responsiveness).

Dead Zone Sizes: Width of "no control" zones between gestures in feature space. Larger dead zones reduce misclassifications but make boundary gestures harder to activate. Adjustable per user comfort.

7.3.2 Functional Customization

Custom Gesture Mappings: Map detected patterns to desired prosthetic actions. User might prefer different grasp patterns than defaults. Example: map "wrist flexion" pattern to "trigger grip" instead of actual wrist flexion if wrist function less important than grip variety.

Multi-Grip Configurations: For prosthetic hands with multiple grips, user can customize which grips are available and how to select them. Some may prefer just 3-4 frequently-used grips for simplicity; others want access to all 10+ options.

Mode Switching Preferences: Customize how control modes are switched (co-contraction, hold, external button, app-based). Different strategies suit different users and tasks.

7.3.3 Feedback Customization

Vibrotactile Preferences: Users can adjust vibration intensity, frequency ranges, and mapping functions. Some prefer strong, obvious feedback; others want subtle cues. Frequency preferences vary with individual sensitivity.

Sensory Priority Selection: Users choose which sensory information is most important (grip force, position, slip warning, etc.), and system allocates feedback channels accordingly.

Adaptive Intensity: Feedback intensity can auto-adjust based on ambient conditions (stronger in noisy environments, weaker in quiet settings) or time of day (reduced at night).

7.4 Adaptive and Online Learning

Static calibration degrades over time due to electrode shift, muscle fatigue, learning, and physiological changes. Adaptive systems continuously update to maintain performance.

7.4.1 Unsupervised Adaptation

Drift Tracking: System monitors feature values over time, detecting slow drift in signal characteristics. Classifier decision boundaries are adjusted to follow the drift, maintaining separation between gesture classes.

Implementation: Exponentially weighted moving average of feature means and covariances; classifier parameters updated based on new statistics. Update rate: typically very slow (hours to days) to avoid chasing noise.

Results: Studies show unsupervised adaptation reduces accuracy degradation from ~15% to ~5% over a week without retraining.

7.4.2 Supervised Online Learning

Active Retraining: System detects when confidence is consistently low or user performance degrades, prompting for short retraining sessions (1-2 minutes, few gesture repetitions).

Incremental Learning: New training data is incorporated into existing classifier without full retraining. Algorithms like online SVM or incremental neural networks enable this. Preserves old knowledge while adapting to new conditions.

Selective Updates: Only update classifier regions showing degradation. If "hand close" accuracy remains high but "wrist rotate" degrades, retrain only wrist-related patterns.

7.4.3 Transfer Learning Across Sessions

Challenge: Daily electrode placement variations cause feature distribution shifts, degrading inter-session performance (accuracy drops from 95% within-session to 75-85% across sessions).

Domain Adaptation Solutions:

  • Procrustes Analysis: Geometrically aligns new session feature space to previous sessions via rotation/scaling/translation transforms. Reduces retraining needs.
  • Subspace Alignment: Projects features into subspace robust to domain shifts. Maintains 85-90% accuracy across sessions without retraining.
  • Quick Recalibration: Collect just 1-2 gesture repetitions at session start; use to adapt previous model. Requires <30 seconds vs. 5+ minutes for full training.

7.5 Long-Term Adaptation and Learning

Over weeks to months, users' motor control, residual limb condition, and functional needs evolve. Systems should support this long-term trajectory.

7.5.1 Motor Learning Support

Progressive Difficulty: Initially, use lenient classification thresholds and larger tolerance zones, making control easy. As user skill improves, gradually tighten thresholds, requiring more precise muscle activation. This scaffolded learning optimizes the learning curve.

Performance Metrics and Feedback: Track success rates, reaction times, and classification confidence over sessions. Visualize progress to motivate users and inform therapists about training effectiveness.

Automated Difficulty Adjustment: System monitors performance; when consistently high (e.g., 95%+ success for multiple sessions), automatically increases challenge level. Prevents plateauing on easy settings.

7.5.2 Anticipatory Adaptation

Context Recognition: System learns user's activity patterns (morning routine, work tasks, evening activities) and predicts likely upcoming gestures, pre-adjusting parameters for expected tasks.

Usage Patterns: Tracks which gestures/grips are most frequently used, which are never used. Can suggest removing unused options to simplify control, or retraining underperforming frequently-used gestures.

Fatigue Prediction: Learn user's fatigue patterns (time-of-day effects, activity-related fatigue). Proactively adjust control parameters before performance degrades.

7.6 Multi-User and Clinical Workflows

Clinical settings serve many users. Efficient workflows and data management are essential.

7.6.1 User Profiles and Cloud Storage

Profile Management: Each user has a stored profile containing calibration parameters, trained classifiers, customizations, and usage history. Enables quick setup when switching between users (family members sharing device, rental/trial scenarios, research studies).

Cloud Backup: Profiles backed up to HIPAA-compliant cloud storage. Protects against device failure, enables access from any compatible system, supports remote troubleshooting by clinicians.

Version Control: Maintains history of profile changes. If new calibration performs poorly, can roll back to previous working configuration.

7.6.2 Clinician Interfaces

Calibration Wizards: Step-by-step guided procedures for signal acquisition, electrode placement verification, and classifier training. Reduces required expertise, ensures consistency across clinics.

Remote Monitoring: Clinicians can view user performance metrics remotely, identifying issues before in-person visits. Enables proactive intervention when degradation detected.

Troubleshooting Tools: Diagnostic interfaces showing real-time signals, classifier confidence, error rates. Helps clinicians identify problems (electrode issues, poor training, user technique).

7.6.3 Standardized Assessment Protocols

The WIA standard recommends regular assessment to track outcomes and guide optimization:

Initial Assessment (fitting/training):

  • Signal quality metrics for all channels
  • Classifier accuracy (≥90% target)
  • Basic functional test (Box and Blocks, Clothespin Relocation)
  • User satisfaction questionnaire

3-Month Assessment:

  • Classifier accuracy revalidation
  • Comprehensive functional test (SHAP)
  • User feedback on control quality, comfort, functionality
  • Recalibration if needed

Annual Assessment:

  • Full signal quality reassessment
  • Complete recalibration
  • Functional testing
  • Psychosocial outcomes (TAPES questionnaire)
  • Discussion of evolving needs/goals

7.7.2 Minimum Calibration Requirements

WIA-compliant systems must support:

  • Initial Calibration: Guided procedure completing in ≤20 minutes
  • Gain Adjustment: Automatic or semi-automatic per-channel gain setting
  • Threshold Calibration: User-adjustable thresholds with recommended defaults
  • Classifier Training: ≤10 minutes training time, achieving ≥80% accuracy
  • Profile Storage: Save/load calibration profiles
  • Performance Validation: Automatic accuracy testing with feedback
