Chapter 6: 사이버 범죄 국제 협약

사이버 공간의 무국경적 특성으로 인해 사이버 범죄는 국제적 협력 없이는 효과적으로 대응하기 어렵습니다. 부다페스트 협약(Budapest Convention)을 중심으로 한 국제 사이버 범죄 협약 체계는 각국의 형사법을 조화시키고, 수사 권한을 현대화하며, 국제 협력을 강화하는 포괄적인 법적 프레임워크를 제공합니다. 본 장에서는 사이버 범죄의 국제적 규제 체계, 협약의 주요 내용, 국제 공조 메커니즘, 그리고 한국의 대응 현황을 상세히 분석합니다.

6.1 부다페스트 협약의 개요 및 구조

2001년 유럽평의회(Council of Europe)에서 채택된 사이버범죄협약, 통칭 부다페스트 협약은 사이버 범죄에 관한 최초의 구속력 있는 국제 조약입니다. 이 협약은 유럽 국가뿐만 아니라 미국, 일본, 캐나다, 호주 등 비유럽 국가도 가입하여 글로벌 표준으로 자리잡았습니다. 협약은 크게 실체법적 규정, 절차법적 규정, 국제 협력 규정의 세 부분으로 구성됩니다.

실체법 부분에서는 컴퓨터 시스템에 대한 불법 접근, 불법 감청, 데이터 및 시스템 방해, 컴퓨터 관련 위조 및 사기 등의 범죄를 정의합니다. 절차법 부분에서는 저장된 컴퓨터 데이터의 신속한 보존, 제공 명령, 수색 및 압수, 실시간 데이터 수집 등에 관한 권한을 규정합니다. 국제 협력 부분에서는 범죄인 인도, 상호 법률 지원, 자발적 정보 제공 등을 다룹니다.

협약의 제정 배경과 목적

1990년대 후반 인터넷의 급속한 확산과 함께 사이버 범죄가 급증하면서, 기존의 국내법만으로는 효과적인 대응이 불가능하다는 인식이 확산되었습니다. 특히 사이버 범죄의 국경 초월적 특성, 디지털 증거의 휘발성, 관할권 충돌 문제 등이 주요 과제로 대두되었습니다. 부다페스트 협약은 이러한 문제에 대응하기 위해 각국 형사법의 조화(harmonization), 수사 권한의 현대화, 신속하고 효과적인 국제 협력 체계 구축을 목표로 제정되었습니다.

협약 구성 주요 조항 규율 내용 특징
제1장 실체법 제2-13조 사이버 범죄 유형 정의, 범죄화 의무 4개 범주 9개 범죄 유형
제2장 절차법 제14-22조 수사권한, 데이터 보존, 압수수색 전자증거 수집 절차 표준화
제3장 국제협력 제23-35조 상호법률지원, 범죄인 인도 24/7 네트워크 운영
추가의정서 2003년 채택 인종차별 콘텐츠 규제 표현의 자유와의 긴장
제2추가의정서 2021년 채택 강화된 협력, 직접 요청 클라우드 증거 수집 개선

6.2 협약상 사이버 범죄 유형

부다페스트 협약은 사이버 범죄를 네 가지 범주로 분류합니다. 첫째, 컴퓨터 데이터 및 시스템의 기밀성, 무결성, 가용성에 대한 범죄로서 불법 접근(제2조), 불법 감청(제3조), 데이터 방해(제4조), 시스템 방해(제5조), 장치 남용(제6조)이 포함됩니다. 둘째, 컴퓨터 관련 범죄로서 컴퓨터 관련 위조(제7조)와 사기(제8조)가 있습니다.

셋째, 콘텐츠 관련 범죄로서 아동 포르노그래피(제9조)를 규정합니다. 넷째, 저작권 및 관련 권리 침해(제10조)를 다룹니다. 추가의정서에서는 컴퓨터 시스템을 통한 인종차별적, 외국인 혐오적 자료의 유포를 범죄화합니다. 각 범죄에 대해 협약은 각국에 국내법상 범죄화를 의무지우며, 미수범과 공범에 대한 처벌, 법인의 형사책임도 규정합니다.

기밀성·무결성·가용성 침해 범죄

불법 접근(Illegal Access)은 컴퓨터 시스템의 전부 또는 일부에 권한 없이 고의로 접근하는 행위입니다. 협약은 각국이 보안 조치를 우회한 경우나 부정한 의도가 있는 경우로 범위를 제한할 수 있도록 허용합니다. 불법 감청(Illegal Interception)은 기술적 수단을 이용하여 컴퓨터 시스템으로 또는 컴퓨터 시스템 내에서 이루어지는 비공개 데이터 전송을 권한 없이 감청하는 행위입니다.

데이터 방해(Data Interference)는 컴퓨터 데이터를 권한 없이 손상, 삭제, 훼손, 변경, 억제하는 행위를 의미합니다. 시스템 방해(System Interference)는 컴퓨터 데이터의 입력, 전송, 손상, 삭제, 훼손, 변경, 억제를 통해 컴퓨터 시스템의 기능을 심각하게 방해하는 행위입니다. DDoS 공격이 대표적인 예입니다. 장치 남용(Misuse of Devices)은 위 범죄를 저지르기 위한 도구의 생산, 판매, 취득, 수입, 배포 등을 처벌합니다.

범죄 유형 협약 조항 행위 요건 적용 사례
불법 접근 제2조 권한 없는 시스템 접근 해킹, 무단 로그인
불법 감청 제3조 데이터 전송의 무단 감청 패킷 스니핑, 도청
데이터 방해 제4조 데이터의 손상, 삭제, 변경 랜섬웨어, 데이터 파괴
시스템 방해 제5조 시스템 기능의 심각한 방해 DDoS 공격
장치 남용 제6조 범죄 도구의 생산, 배포 해킹 툴 판매, 익스플로잇 거래

6.3 절차법적 규정과 전자증거 수집

부다페스트 협약의 절차법 규정은 사이버 범죄 수사에 필요한 권한과 절차를 표준화합니다. 저장된 컴퓨터 데이터의 신속한 보존(제16조)은 데이터가 삭제되기 전에 보존 명령을 발할 수 있는 권한을 규정합니다. 이는 디지털 증거의 휘발성에 대응하기 위한 핵심적인 조치입니다. 트래픽 데이터의 신속한 보존 및 부분 공개(제17조)는 통신 경로 추적에 필요한 정보를 확보할 수 있게 합니다.

제공 명령(제18조)은 특정인이나 서비스 제공자에게 보유한 컴퓨터 데이터나 가입자 정보의 제출을 명할 수 있는 권한을 부여합니다. 수색 및 압수(제19조)는 컴퓨터 시스템, 저장매체, 컴퓨터 데이터에 대한 수색 및 압수 권한을 규정합니다. 실시간 트래픽 데이터 수집(제20조)과 콘텐츠 데이터 감청(제21조)은 통신감청에 해당하는 권한으로, 중대한 범죄 수사에 한해 적용됩니다.

// 사이버 범죄 국제 협력 요청 처리 시스템
interface MutualLegalAssistanceRequest {
  requestId: string;
  requestingState: string;
  requestedState: string;
  requestType: 'preservation' | 'disclosure' | 'search' | 'realtime';
  urgency: 'routine' | 'urgent' | 'emergency';

  // 범죄 정보
  offenseDescription: string;
  offenseArticles: BudapestConventionArticle[];
  dualCriminality: boolean;

  // 대상 데이터 정보
  targetData: {
    type: 'subscriber' | 'traffic' | 'content';
    serviceProvider: string;
    dataIdentifiers: string[];
    timePeriod: { start: Date; end: Date };
  };

  // 법적 근거
  legalBasis: {
    domesticLaw: string;
    treatyBasis: string;
    judicialAuthorization: JudicialOrder;
  };
}

class CyberCrimeCooperation {
  private twentyFourSevenNetwork: Map;

  async submitPreservationRequest(
    targetState: string,
    dataSpecification: DataSpecification
  ): Promise {
    // 24/7 네트워크를 통한 긴급 보존 요청
    const contactPoint = this.twentyFourSevenNetwork.get(targetState);

    if (!contactPoint) {
      throw new Error(`No 24/7 contact for state: ${targetState}`);
    }

    const request: PreservationRequest = {
      type: 'expedited_preservation',
      article: 'Article 29', // Budapest Convention
      data: dataSpecification,
      preservationPeriod: 60, // days, extendable to 90
      timestamp: new Date()
    };

    // 긴급 채널을 통한 요청 전송
    return await contactPoint.sendUrgentRequest(request);
  }

  async processIncomingMLA(
    request: MutualLegalAssistanceRequest
  ): Promise {
    // 1. 형식적 요건 검토
    const formalReview = await this.reviewFormalRequirements(request);
    if (!formalReview.satisfied) {
      return { status: 'rejected', reason: formalReview.deficiencies };
    }

    // 2. 쌍방가벌성 검토 (필요시)
    if (this.requiresDualCriminality(request.requestType)) {
      const dualCrimCheck = await this.checkDualCriminality(request);
      if (!dualCrimCheck.satisfied) {
        return { status: 'rejected', reason: 'dual_criminality_not_met' };
      }
    }

    // 3. 국내법상 집행
    const domesticOrder = await this.obtainDomesticAuthorization(request);
    const collectedData = await this.executeDataCollection(
      request.targetData,
      domesticOrder
    );

    // 4. 응답 전송
    return {
      status: 'fulfilled',
      data: collectedData,
      restrictions: this.determineUseRestrictions(request)
    };
  }
}

// 증거 체인 관리
class EvidenceChainManager {
  async preserveDigitalEvidence(
    source: DataSource,
    preservationOrder: PreservationOrder
  ): Promise {
    // 증거 무결성 보장을 위한 해시 계산
    const evidenceHash = await this.computeForensicHash(source);

    const preserved: PreservedEvidence = {
      id: generateEvidenceId(),
      source: source,
      hash: {
        algorithm: 'SHA-256',
        value: evidenceHash,
        timestamp: new Date()
      },
      chainOfCustody: [{
        action: 'initial_preservation',
        actor: this.getCurrentAuthority(),
        timestamp: new Date(),
        location: this.getSecureStorageLocation()
      }],
      legalBasis: preservationOrder
    };

    await this.secureStorage.store(preserved);
    return preserved;
  }
}

6.4 국제 협력 메커니즘

부다페스트 협약의 가장 혁신적인 부분은 국제 협력에 관한 제3장입니다. 협약은 범죄인 인도(제24조)와 상호 법률 지원의 일반 원칙(제25-28조)을 규정하며, 특히 사이버 범죄 수사를 위한 특별한 협력 조항을 두고 있습니다. 저장된 데이터의 보존에 관한 상호 지원(제29-30조), 저장된 데이터에 대한 접근에 관한 상호 지원(제31조), 실시간 데이터 수집에 관한 상호 지원(제33-34조) 등이 포함됩니다.

24/7 네트워크(제35조)는 협약의 핵심적인 협력 메커니즘입니다. 각 체약국은 사이버 범죄 수사를 위한 즉각적인 지원을 제공할 수 있는 24시간 연락 창구를 지정해야 합니다. 이 네트워크는 긴급한 보존 요청, 기술적 조언, 범죄 관련 정보의 신속한 교환을 가능하게 합니다. 2021년 채택된 제2추가의정서는 이러한 협력 체계를 더욱 강화하여, 서비스 제공자에 대한 직접적인 협력 요청, 긴급 상황에서의 신속한 절차, 합동 수사팀 구성 등을 규정합니다.

한국의 사이버 범죄 국제 협력 체계

한국은 2015년에 부다페스트 협약에 정식으로 가입하여 아시아 최초의 비유럽 가입국이 되었습니다. 한국은 협약 가입 이전부터 국내법을 협약 기준에 맞추어 정비해 왔으며, 정보통신망 이용촉진 및 정보보호 등에 관한 법률, 형법 전자기록 관련 조항, 통신비밀보호법 등이 협약의 실체법 및 절차법 요건을 충족합니다.

대검찰청 사이버수사과가 24/7 네트워크의 연락 창구 역할을 수행하며, 경찰청 사이버수사국과 긴밀히 협력합니다. 한국은 미국, 일본, EU 회원국 등과 양자간 상호법률지원조약(MLAT)을 체결하여 협력 채널을 다변화하고 있습니다. 최근에는 클라우드 증거 수집을 위한 제2추가의정서 가입도 검토 중이며, 실시간 국제 공조 역량 강화에 주력하고 있습니다.

협력 유형 근거 조항 처리 시한 담당 기관
긴급 보존 요청 제29조 24시간 내 응답 24/7 연락창구
가입자 정보 요청 제31조 10일 이내 법무부 국제협력과
트래픽 데이터 요청 제31조 30일 이내 법무부, 검찰
콘텐츠 데이터 요청 제31조 60일 이내 법무부, 법원
범죄인 인도 제24조 조약별 상이 법무부, 외교부

6.5 신종 사이버 범죄와 협약의 발전

2001년 협약 채택 이후 사이버 범죄의 양상은 크게 변화했습니다. 랜섬웨어 공격, 암호화폐 관련 범죄, 딥페이크를 이용한 범죄, IoT 기기 해킹, AI 기반 사이버 공격 등 새로운 유형의 범죄가 등장했습니다. 부다페스트 협약의 기본 조항들은 기술 중립적으로 설계되어 상당 부분 새로운 범죄에도 적용 가능하지만, 일부 영역에서는 해석론적 한계가 있습니다.

유럽평의회는 협약의 현대화를 위해 지속적인 노력을 기울이고 있습니다. 2021년 제2추가의정서는 클라우드 컴퓨팅 환경에서의 증거 수집, 국경간 직접 협력, 합동 수사, 긴급 상호 지원 절차 등을 규정합니다. 또한 사이버범죄협약위원회(T-CY)는 클라우드 증거, 관할권, 테러리스트의 인터넷 이용 등에 관한 가이던스 노트를 채택하여 협약의 현대적 해석을 지원합니다.

class CooperationMetrics {
  async analyzeResponseTimes(
    requestType: MLARequestType,
    period: DateRange
  ): Promise {
    const requests = await this.getMLARequests(requestType, period);

    return {
      averageResponseTime: this.calculateAverage(requests),
      medianResponseTime: this.calculateMedian(requests),
      complianceRate: this.calculateComplianceRate(requests),
      byRequestedState: this.groupByState(requests),
      bottlenecks: this.identifyBottlenecks(requests),
      recommendations: this.generateRecommendations(requests)
    };
  }

  async benchmarkAgainstConventionStandards(
    stateId: string
  ): Promise {
    return {
      article29Compliance: await this.checkPreservationCapability(stateId),
      article35Compliance: await this.check24x7NetworkStatus(stateId),
      legislativeAlignment: await this.assessLegislativeHarmonization(stateId),
      practicalEffectiveness: await this.measurePracticalCooperation(stateId)
    };
  }
}

6.6 협약의 한계와 대안적 접근

부다페스트 협약은 사이버 범죄 국제 협력의 선구적 역할을 했지만, 여러 한계도 지적됩니다. 첫째, 가입국의 지리적 편중 문제입니다. 러시아, 중국, 인도 등 주요 국가들이 협약에 가입하지 않아 글로벌 커버리지에 한계가 있습니다. 둘째, 절차의 지연 문제입니다. 상호법률지원 절차가 수개월에서 수년이 소요되어 디지털 증거의 신속한 확보에 어려움이 있습니다.

이러한 한계를 극복하기 위해 다양한 대안이 모색되고 있습니다. UN 차원에서는 러시아의 주도로 새로운 사이버범죄협약 협상이 진행 중입니다. 미국의 CLOUD Act와 EU의 e-Evidence 규칙은 서비스 제공자에 대한 직접 접근 방식을 채택합니다. 양자간 또는 다자간 데이터 접근 협정(Data Access Agreements)도 확산되고 있습니다. 민간 부문에서는 테크 기업들의 자발적 협력 프로그램(Trusted Partner Programs)이 운영됩니다.

⚖️ 협약의 미래: 사이버 범죄의 진화와 디지털 증거 수집의 기술적 발전에 발맞추어, 부다페스트 협약도 지속적으로 발전하고 있습니다. 제2추가의정서의 채택, UN 차원의 새로운 협약 논의, 클라우드 시대에 맞는 증거법 현대화 등 다양한 노력이 진행 중입니다. 한국은 협약의 적극적인 참여국으로서 국제 사이버 보안 거버넌스에서 중요한 역할을 담당하고 있습니다.

📚 핵심 요약

  1. 부다페스트 협약: 2001년 채택된 최초의 구속력 있는 사이버범죄 국제 조약으로, 실체법, 절차법, 국제협력의 세 축으로 구성됩니다.
  2. 사이버 범죄 유형: 협약은 CIA 침해 범죄, 컴퓨터 관련 범죄, 콘텐츠 범죄, 저작권 침해의 네 범주로 사이버 범죄를 분류합니다.
  3. 절차법 규정: 데이터 보존, 제공 명령, 수색 및 압수, 실시간 수집 등 전자증거 수집을 위한 표준화된 절차를 제공합니다.
  4. 24/7 네트워크: 각국의 상시 연락 창구를 통해 긴급한 국제 공조가 가능하며, 신속한 데이터 보존 요청을 처리합니다.
  5. 한국의 참여: 2015년 아시아 최초로 협약에 가입하여 대검찰청이 24/7 연락창구 역할을 수행하고 있습니다.
  6. 신종 범죄 대응: 랜섬웨어, 암호화폐 범죄, 딥페이크 등 새로운 위협에 대응하기 위해 협약 해석과 추가의정서가 발전하고 있습니다.
  7. 향후 과제: 비가입국 확대, 절차 지연 해소, 클라우드 증거 수집 개선 등이 주요 과제이며, 제2추가의정서가 이를 보완합니다.

  2. 협약 제2조부터 제6조에 규정된 CIA(기밀성, 무결성, 가용성) 침해 범죄 5가지를 열거하고 각각의 행위 요건을 설명하시오.

  4. 24/7 네트워크(제35조)의 운영 방식과 긴급 보존 요청 처리 절차를 설명하시오.

  6. 랜섬웨어 공격에 대해 부다페스트 협약의 어떤 조항들이 적용될 수 있는지 분석하고, 국제 공조의 어려움을 논하시오.

  8. 2021년 제2추가의정서의 주요 내용을 설명하고, 클라우드 컴퓨팅 환경에서의 증거 수집에 어떤 개선을 가져오는지 분석하시오.
