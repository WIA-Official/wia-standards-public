API Specifications

RESTful APIs for Booking, Tracking, and Integration

Introduction to Cargo APIs

Modern air cargo operations require robust APIs to enable seamless integration between airlines, freight forwarders, ground handlers, customs authorities, and technology platforms. RESTful APIs have become the industry standard, providing standardized interfaces for booking, tracking, documentation, and operational workflows.

This chapter provides comprehensive API specifications following industry best practices, including authentication mechanisms, data models, error handling, rate limiting, and versioning strategies. These specifications align with IATA ONE Record and Cargo-XML standards while providing practical implementation guidance.

Authentication and Authorization

// OAuth 2.0 Authentication Implementation
interface AuthToken {
  access_token: string;
  token_type: "Bearer";
  expires_in: number;                   // Seconds until expiration
  refresh_token?: string;
  scope: string[];                      // Granted permissions
}

interface AuthRequest {
  grant_type: "client_credentials" | "authorization_code" | "refresh_token";
  client_id: string;
  client_secret: string;
  scope?: string;
  refresh_token?: string;
}

// API Client with OAuth 2.0
class CargoAPIClient {
  private baseURL: string;
  private clientId: string;
  private clientSecret: string;
  private accessToken?: string;
  private tokenExpiry?: Date;

  constructor(baseURL: string, clientId: string, clientSecret: string) {
    this.baseURL = baseURL;
    this.clientId = clientId;
    this.clientSecret = clientSecret;
  }

  // Authenticate and get access token
  async authenticate(scope: string[] = ["cargo.read", "cargo.write"]): Promise<void> {
    const authRequest: AuthRequest = {
      grant_type: "client_credentials",
      client_id: this.clientId,
      client_secret: this.clientSecret,
      scope: scope.join(" ")
    };

    const response = await fetch(`${this.baseURL}/oauth/token`, {
      method: "POST",
      headers: {
        "Content-Type": "application/json"
      },
      body: JSON.stringify(authRequest)
    });

    if (!response.ok) {
      throw new Error(`Authentication failed: ${response.status}`);
    }

    const token: AuthToken = await response.json();
    this.accessToken = token.access_token;
    this.tokenExpiry = new Date(Date.now() + token.expires_in * 1000);

    console.log("Authentication successful, token expires:", this.tokenExpiry);
  }

  // Make authenticated API request
  async request<T>(
    method: string,
    endpoint: string,
    body?: any
  ): Promise<T> {
    // Check token expiry and refresh if needed
    if (!this.accessToken || (this.tokenExpiry && new Date() >= this.tokenExpiry)) {
      await this.authenticate();
    }

    const headers: Record<string, string> = {
      "Authorization": `Bearer ${this.accessToken}`,
      "Content-Type": "application/json",
      "Accept": "application/json"
    };

    const options: RequestInit = {
      method,
      headers
    };

    if (body) {
      options.body = JSON.stringify(body);
    }

    const response = await fetch(`${this.baseURL}${endpoint}`, options);

    if (!response.ok) {
      const error = await response.json();
      throw new Error(`API Error: ${error.message}`);
    }

    return await response.json();
  }

  // Convenience methods
  async get<T>(endpoint: string): Promise<T> {
    return this.request<T>("GET", endpoint);
  }

  async post<T>(endpoint: string, body: any): Promise<T> {
    return this.request<T>("POST", endpoint, body);
  }

  async put<T>(endpoint: string, body: any): Promise<T> {
    return this.request<T>("PUT", endpoint, body);
  }

  async delete<T>(endpoint: string): Promise<T> {
    return this.request<T>("DELETE", endpoint);
  }
}

interface PartyDetails {
  name: string;
  address: {
    street: string;
    city: string;

    country: string;
  };

    name: string;
    phone: string;
    email: string;

interface BookingResponse {
  bookingId: string;
  bookingReference: string;

  awbNumber?: string;

  createdAt: string;

interface FlightAllocation {
  flightNumber: string;
  flightDate: string;
  departureTime: string;
  arrivalTime: string;
  aircraftType: string;
  allocatedPieces: number;
  allocatedWeight: Weight;
}

// Booking API Service
class BookingAPI {
  constructor(private client: CargoAPIClient) {}

  // Create booking
  async createBooking(request: BookingRequest): Promise<BookingResponse> {
    const response = await this.client.post<BookingResponse>(
      "/api/v1/bookings",
      request
    );

    console.log("Booking created:", response.bookingReference);
    return response;
  }

  // Get booking details
  async getBooking(bookingId: string): Promise<BookingResponse> {
    return this.client.get<BookingResponse>(`/api/v1/bookings/${bookingId}`);
  }

  // Update booking
  async updateBooking(
    bookingId: string,
    updates: Partial<BookingRequest>
  ): Promise<BookingResponse> {
    return this.client.put<BookingResponse>(
      `/api/v1/bookings/${bookingId}`,
      updates
    );
  }

  // Cancel booking
  async cancelBooking(bookingId: string): Promise<{ success: boolean }> {
    return this.client.delete(`/api/v1/bookings/${bookingId}`);
  }

  // Get available flights
  async getAvailableFlights(
    origin: string,
    destination: string,
    date: string,
    weight: number
  ): Promise<FlightAllocation[]> {
    const params = new URLSearchParams({
      origin,
      destination,
      date,
      weight: weight.toString()
    });

    return this.client.get<FlightAllocation[]>(
      `/api/v1/flights/available?${params}`
    );
  }

  // Calculate rate quote
  async getRateQuote(request: {
    origin: string;
    destination: string;
    weight: Weight;
    volume: Volume;
    commodityCode: string;
  }): Promise<{
    totalCharge: Money;
    breakdown: ChargeBreakdown[];
    validUntil: string;
  }> {
    return this.client.post("/api/v1/rates/quote", request);
  }
}

interface ChargeBreakdown {
  chargeType: string;
  description: string;
  amount: Money;
}

// Tracking API Implementation
interface TrackingResponse {
  awbNumber: string;
  origin: string;
  destination: string;
  currentStatus: string;
  currentLocation: string;
  pieces: number;
  weight: Weight;
  estimatedDelivery: string;
  milestones: TrackingMilestone[];
  _links: {
    self: { href: string };
    booking: { href: string };
    pod: { href: string };              // Proof of delivery
  };
}

interface TrackingMilestone {
  timestamp: string;
  statusCode: string;
  statusDescription: string;
  location: string;
  pieces?: number;
  weight?: Weight;
  remarks?: string;
  flightNumber?: string;
}

interface MultiTrackingRequest {
  awbNumbers: string[];
}

interface MultiTrackingResponse {
  shipments: TrackingResponse[];
  notFound: string[];
}

// Tracking API Service
class TrackingAPI {
  constructor(private client: CargoAPIClient) {}

  // Track single shipment
  async trackShipment(awbNumber: string): Promise<TrackingResponse> {
    const formatted = awbNumber.replace("-", "");
    return this.client.get<TrackingResponse>(`/api/v1/tracking/${formatted}`);
  }

  // Track multiple shipments
  async trackMultipleShipments(awbNumbers: string[]): Promise<MultiTrackingResponse> {
    return this.client.post<MultiTrackingResponse>(
      "/api/v1/tracking/bulk",
      { awbNumbers }
    );
  }

  // Get live flight status
  async getFlightStatus(flightNumber: string, flightDate: string): Promise<{
    flightNumber: string;
    flightDate: string;
    status: "SCHEDULED" | "DEPARTED" | "AIRBORNE" | "ARRIVED" | "CANCELLED";
    actualDeparture?: string;
    actualArrival?: string;
    cargo: {
      totalPieces: number;
      totalWeight: Weight;
      awbCount: number;
    };
  }> {
    const params = new URLSearchParams({ date: flightDate });
    return this.client.get(`/api/v1/flights/${flightNumber}?${params}`);
  }

  // Subscribe to tracking updates
  async subscribeToUpdates(
    awbNumber: string,
    webhookUrl: string
  ): Promise<{
    subscriptionId: string;
    awbNumber: string;
    webhookUrl: string;
    active: boolean;
  }> {
    return this.client.post("/api/v1/tracking/subscriptions", {
      awbNumber,
      webhookUrl
    });
  }

  // Get proof of delivery
  async getProofOfDelivery(awbNumber: string): Promise<{
    awbNumber: string;
    deliveryDate: string;
    deliveryTime: string;
    recipientName: string;
    recipientSignature?: string;        // Base64 encoded image
    deliveryLocation: string;
    pieces: number;
    remarks?: string;
  }> {
    return this.client.get(`/api/v1/tracking/${awbNumber}/pod`);
  }
}

// Example usage
const trackingAPI = new TrackingAPI(apiClient);

// Track shipment
const tracking = await trackingAPI.trackShipment("020-12345678");

console.log("=== Shipment Tracking ===");

console.log("Estimated Delivery:", tracking.estimatedDelivery);

console.log("\nMilestone History:");
tracking.milestones.forEach(milestone => {
  console.log(`  ${milestone.timestamp} - ${milestone.statusDescription} at ${milestone.location}`);
});

// Track multiple shipments
const multiTracking = await trackingAPI.trackMultipleShipments([
  "020-12345678",
  "020-12345679",
  "020-99999999"  // Does not exist
]);

console.log("\n=== Multi-Tracking Results ===");
console.log("Found:", multiTracking.shipments.length);
console.log("Not found:", multiTracking.notFound);

  "020-12345678",
  "https://myapp.example.com/webhooks/cargo-updates"

// Customs API Service
class CustomsAPI {
  constructor(private client: CargoAPIClient) {}

  // Submit customs declaration
  async submitDeclaration(
    request: CustomsDeclarationRequest
  ): Promise<CustomsDeclarationResponse> {
    return this.client.post("/api/v1/customs/declarations", request);
  }

  // Get declaration status
  async getDeclarationStatus(declarationId: string): Promise<CustomsDeclarationResponse> {
    return this.client.get(`/api/v1/customs/declarations/${declarationId}`);
  }

  // Get HS code details
  async getHSCodeInfo(hsCode: string, country: string): Promise<{
    hsCode: string;
    description: string;
    dutyRate: number;
    additionalDuties: string[];
    restrictions: string[];
  }> {
    const params = new URLSearchParams({ country });
    return this.client.get(`/api/v1/customs/hscodes/${hsCode}?${params}`);

  async calculateDuties(
    country: string,
    commodities: CommodityDeclaration[]
  ): Promise<{
    totalDuties: Money;
    totalTaxes: Money;
    breakdown: {
      hsCode: string;
      duty: Money;
      tax: Money;
    }[];
  }> {
    return this.client.post("/api/v1/customs/calculate-duties", {
      country,
      commodities
    });
  }
}

// Error handling
interface APIError {

const ERROR_CODES = {

// Rate limiting headers
interface RateLimitHeaders {
  "X-RateLimit-Limit": string;          // Max requests per window
  "X-RateLimit-Remaining": string;      // Remaining requests
  "X-RateLimit-Reset": string;          // Unix timestamp when limit resets
}

// Example error handling
try {
  const tracking = await trackingAPI.trackShipment("INVALID-AWB");
} catch (error: any) {
  if (error.response) {
    const apiError: APIError = await error.response.json();
    console.error("API Error:", apiError.error.code);
    console.error("Message:", apiError.error.message);
    console.error("Request ID:", apiError.error.requestId);
  }
}

Key Takeaways

  • RESTful API design with OAuth 2.0 authentication provides secure, scalable access to cargo systems
  • Booking API enables automated space reservation, flight allocation, and rate quote retrieval
  • Tracking API provides real-time shipment visibility with milestone history and webhook subscriptions
  • Customs API streamlines declaration submission, duty calculation, and HS code validation
  • Standard error response formats and HTTP status codes enable consistent error handling
  • Rate limiting protects API infrastructure while ensuring fair usage across all clients
  • HATEOAS links enable API discoverability and reduce coupling between client and server
  • TypeScript interfaces provide type safety and self-documenting code for API integration

Reference Standards and Implementation Notes

This standard is designed to interoperate with established international frameworks. Implementations SHOULD treat the following as the single source of truth and use this PHASE document as the interoperability layer that maps existing data into a machine-readable, signed, and auditable shape.

  • OpenAPI Specification 3.1 — REST API definition
  • JSON Schema 2020-12 — schema validation
  • ISO/IEC 27001:2022 — information security management
  • ISO/IEC 27018:2019 — PII protection in public clouds
  • ISO/IEC 17065:2012 — product certification bodies
  • CycloneDX 1.5 / SPDX 2.3 — software bill of materials
  • Sigstore (DSSE envelope, Rekor transparency log) — signature anchoring
  • in-toto Attestation Framework 1.0 — build provenance
  • IETF RFC 9700 — OAuth 2.1
  • IETF RFC 9457 — Problem Details for HTTP APIs
  • IETF RFC 8615 — well-known URIs

Practical Implementation Checklist

  • Manifest integrity — every published manifest is signed via Sigstore DSSE. Tampering is detectable without trusting an intermediary. Updates are issued as new manifests; the prior manifest is preserved with a supersede marker.
  • Retention — keep signed evidence for at least seven years to satisfy ISO/IEC 17065:2012 audit expectations. Some regulators require longer retention; in that case the deployment policy MUST extend the window rather than relying on the defaults.
  • Versioning — follow Semantic Versioning 2.0.0. Major version bumps require at least a 90-day overlap with the prior major version on every WIA-published reference implementation.
  • Negative-result reporting — a skipped indicator without recorded justification is treated as a failed indicator at audit time.
  • Key rotation — manifest signing keys are rotated per the deployment policy. The previous key is preserved in a published key history so that auditors can verify historical manifests offline.

Reference Topologies

Implementations declare their operational topology in the manifest: single-tenant edge (one runtime per organization), multi-tenant gateway (shared runtime with header-based isolation), federated mesh (multiple runtimes peer to one another and publish manifests to a directory service), or air-gapped batch (signed evidence package transported out-of-band). Each topology ships with reference test vectors so that implementations can validate their deployment matches the canonical pattern.

Open Governance

Issues, errata, and proposals are tracked at github.com/WIA-Official/wia-standards/issues with the appropriate label. The WIA Standards working group reviews open issues at the start of every minor release cycle and publishes the resulting decision log alongside the release notes. Errata are issued as patch releases; new normative requirements trigger minor bumps; backwards-incompatible changes trigger major bumps.
