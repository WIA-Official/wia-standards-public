Special Cargo Operations

Perishables, Pharmaceuticals, Live Animals, and Specialized Handling

Introduction to Special Cargo

Special cargo encompasses shipments requiring non-standard handling procedures, specialized equipment, or stringent environmental controls. These commodities represent high-value, time-sensitive segments of air cargo with specific regulatory requirements and handling protocols.

Special cargo categories include temperature-sensitive perishables, pharmaceutical products requiring GDP compliance, live animals with welfare requirements, human remains, valuable cargo, and oversized/heavy shipments. Each category demands specialized knowledge, equipment, and operational procedures to ensure product integrity and regulatory compliance.

Categories and Requirements

Perishable cargo includes food products, flowers, and other commodities with limited shelf life requiring rapid transport and temperature control:

Cool Chain Management

Maintaining temperature integrity from origin to destination requires specialized equipment and procedures:

  • Cool Rooms: Temperature-controlled storage at airports (2-8°C, 15-25°C zones)
  • Refrigerated ULDs: Active or passive cooling containers (RKN, RKE, RAP types)
  • Gel Packs/Dry Ice: Passive cooling for short transits (note: dry ice is Class 9 DG)
  • Thermal Blankets: Insulated covers for temperature protection
  • Temperature Monitoring: Data loggers recording temperature throughout journey
  • Priority Handling: Expedited processing to minimize dwell time

Pharmaceutical logistics must comply with GDP guidelines ensuring product quality and integrity throughout the supply chain. This includes stringent temperature control, qualified personnel, validated processes, and comprehensive documentation.

interface PharmaProduct {
  productName: string;
  activeIngredient: string;
  dosageForm: string;
  manufacturer: string;
  batchNumber: string;
  expiryDate: Date;
  quantity: number;
  unitOfMeasure: string;
  regulatoryStatus: string;           // FDA approved, EMA authorized, etc.
}

interface TemperatureRequirement {
  minTemperature: number;
  maxTemperature: number;
  unit: "C" | "F";
  criticalDuration?: number;          // Maximum minutes outside range
  storageConditions: string;          // "2-8°C", "15-25°C", "Frozen -20°C"
}

interface GDPCompliance {
  gdpCertified: boolean;
  certifyingBody: string;
  certificateNumber: string;
  validUntil: Date;
  qualifiedLanes: string[];           // Validated shipping routes
  lastAuditDate: Date;
  auditResult: "PASS" | "CONDITIONAL" | "FAIL";
}

interface PackagingInfo {
  packagingType: "ACTIVE" | "PASSIVE" | "HYBRID";
  packagingQualification: string;     // Validation study reference
  coolantType?: "GEL_PACKS" | "DRY_ICE" | "LIQUID_NITROGEN" | "PCM";
  coolantQuantity?: number;
  insulationMaterial: string;
  dataLoggers: DataLogger[];
}

interface DataLogger {
  loggerId: string;
  serialNumber: string;
  calibrationDate: Date;
  calibrationDue: Date;
  sampleInterval: number;             // Minutes
  alarmSettings: {
    lowAlarm: number;
    highAlarm: number;
  };
}

interface TemperatureDataPoint {
  timestamp: Date;
  temperature: number;
  unit: "C" | "F";
  location: string;
  loggerId: string;
  inRange: boolean;
}

interface TemperatureExcursion {
  excursionId: string;
  startTime: Date;
  endTime: Date;
  duration: number;                   // Minutes
  minTemperature: number;
  maxTemperature: number;
  meanTemperature: number;
  severity: "MINOR" | "MODERATE" | "MAJOR";
  impactAssessment?: string;
  resolution: "ACCEPTED" | "REJECTED" | "PENDING_REVIEW";
}

interface Certification {
  certificateType: "ORIGIN" | "HEALTH" | "QUALITY" | "COLD_CHAIN" | "GDP";
  certificateNumber: string;
  issuingAuthority: string;
  issueDate: Date;
  expiryDate?: Date;
}

// Pharmaceutical Cargo Manager
class PharmaCargoManager {
  private shipments: Map<string, PharmaShipment>;

  constructor() {
    this.shipments = new Map();
  }

  // Create pharma shipment
  createPharmaShipment(shipmentData: PharmaShipment): void {
    this.shipments.set(shipmentData.awbNumber, shipmentData);
    console.log(`Pharma shipment ${shipmentData.awbNumber} created`);
    console.log(`Temperature requirement: ${shipmentData.temperatureRequirements.storageConditions}`);
  }

  // Record temperature reading
  recordTemperature(
    awbNumber: string,
    dataPoint: TemperatureDataPoint
  ): void {
    const shipment = this.shipments.get(awbNumber);

    if (!shipment) {
      throw new Error(`Shipment ${awbNumber} not found`);
    }

    shipment.temperatureData.push(dataPoint);

    // Check if temperature is out of range
    if (!dataPoint.inRange) {
      this.checkForExcursion(awbNumber);
    }
  }

  // Detect and record temperature excursions
  private checkForExcursion(awbNumber: string): void {
    const shipment = this.shipments.get(awbNumber);

    if (!shipment) return;

    const tempReq = shipment.temperatureRequirements;
    const readings = shipment.temperatureData;

    // Find consecutive out-of-range readings
    let excursionStart: Date | null = null;
    let excursionTemps: number[] = [];

    for (let i = 0; i < readings.length; i++) {
      const reading = readings[i];

      if (reading.temperature < tempReq.minTemperature ||
          reading.temperature > tempReq.maxTemperature) {

        if (!excursionStart) {
          excursionStart = reading.timestamp;
        }
        excursionTemps.push(reading.temperature);
      } else if (excursionStart) {
        // Excursion ended
        this.recordExcursion(awbNumber, excursionStart, readings[i - 1].timestamp, excursionTemps);
        excursionStart = null;
        excursionTemps = [];
      }
    }
  }

  // Record temperature excursion
  private recordExcursion(
    awbNumber: string,
    startTime: Date,
    endTime: Date,
    temperatures: number[]
  ): void {
    const shipment = this.shipments.get(awbNumber);

    if (!shipment) return;

    const duration = (endTime.getTime() - startTime.getTime()) / (1000 * 60);
    const minTemp = Math.min(...temperatures);
    const maxTemp = Math.max(...temperatures);
    const meanTemp = temperatures.reduce((a, b) => a + b, 0) / temperatures.length;

    // Determine severity
    let severity: TemperatureExcursion["severity"];
    const tempReq = shipment.temperatureRequirements;
    const deviation = Math.max(
      Math.abs(minTemp - tempReq.minTemperature),
      Math.abs(maxTemp - tempReq.maxTemperature)
    );

    if (deviation < 2 && duration < 30) {
      severity = "MINOR";
    } else if (deviation < 5 && duration < 120) {
      severity = "MODERATE";
    } else {
      severity = "MAJOR";
    }

    const excursion: TemperatureExcursion = {
      excursionId: `EXC-${Date.now()}`,
      startTime,
      endTime,
      duration,
      minTemperature: minTemp,
      maxTemperature: maxTemp,
      meanTemperature: meanTemp,
      severity,
      resolution: "PENDING_REVIEW"
    };

    shipment.excursions.push(excursion);

    console.log(`⚠️  Temperature Excursion Detected!`);
    console.log(`   AWB: ${awbNumber}`);
    console.log(`   Duration: ${duration.toFixed(0)} minutes`);
    console.log(`   Range: ${minTemp.toFixed(1)}°C to ${maxTemp.toFixed(1)}°C`);
    console.log(`   Severity: ${severity}`);
  }

  // Validate GDP compliance
  validateGDPCompliance(awbNumber: string): {
    compliant: boolean;
    issues: string[];
  } {
    const shipment = this.shipments.get(awbNumber);

    if (!shipment) {
      return {
        compliant: false,
        issues: ["Shipment not found"]
      };
    }

    const issues: string[] = [];

    // Check GDP certification
    if (!shipment.gdpCompliance.gdpCertified) {
      issues.push("Handler is not GDP certified");
    }

    // Check certificate validity
    if (shipment.gdpCompliance.validUntil < new Date()) {
      issues.push("GDP certificate has expired");
    }

    // Check data logger calibration
    for (const logger of shipment.packagingDetails.dataLoggers) {
      if (logger.calibrationDue < new Date()) {
        issues.push(`Data logger ${logger.loggerId} calibration expired`);
      }
    }

    // Check for major excursions
    const majorExcursions = shipment.excursions.filter(e => e.severity === "MAJOR");
    if (majorExcursions.length > 0) {
      issues.push(`${majorExcursions.length} major temperature excursion(s) detected`);
    }

    // Check required certifications
    const requiredCerts = ["ORIGIN", "HEALTH", "GDP"];
    const providedCerts = shipment.certifications.map(c => c.certificateType);

    for (const required of requiredCerts) {
      if (!providedCerts.includes(required as any)) {
        issues.push(`Missing ${required} certificate`);
      }
    }

    return {
      compliant: issues.length === 0,
      issues
    };
  }

  // Generate temperature report
  generateTemperatureReport(awbNumber: string): {
    awbNumber: string;
    totalReadings: number;
    inRangePercentage: number;
    meanTemperature: number;
    minTemperature: number;
    maxTemperature: number;
    excursionsCount: number;
    complianceStatus: "COMPLIANT" | "NON_COMPLIANT";
  } {
    const shipment = this.shipments.get(awbNumber);

    if (!shipment) {
      throw new Error(`Shipment ${awbNumber} not found`);
    }

    const readings = shipment.temperatureData;
    const inRangeCount = readings.filter(r => r.inRange).length;
    const temps = readings.map(r => r.temperature);

    return {
      awbNumber,
      totalReadings: readings.length,
      inRangePercentage: (inRangeCount / readings.length) * 100,
      meanTemperature: temps.reduce((a, b) => a + b, 0) / temps.length,
      minTemperature: Math.min(...temps),
      maxTemperature: Math.max(...temps),
      excursionsCount: shipment.excursions.length,
      complianceStatus: shipment.excursions.filter(e => e.severity === "MAJOR").length === 0
        ? "COMPLIANT"
        : "NON_COMPLIANT"
    };
  }
}

Live Animals

IATA Live Animals Regulations (LAR)

Transport of live animals requires strict compliance with IATA LAR, ensuring animal welfare, handler safety, and regulatory compliance. Requirements vary by species, with specialized containers and procedures for each animal type.

Animal Category Common Examples Container Requirements Special Considerations
Pets Dogs, cats, rabbits IATA CR82 compliant kennels Health certificate, vaccination records
Birds Parrots, finches, poultry Ventilated boxes with perches Avian influenza clearance, CITES permits
Aquatic Animals Fish, shrimp, seahorses Sealed plastic bags with oxygen Water quality, temperature control
Reptiles Snakes, lizards, turtles Escape-proof containers Temperature control, CITES compliance
Laboratory Animals Mice, rats, primates Specialized LAR-compliant containers Research permits, ethical approvals
Livestock Horses, cattle, sheep Aircraft stalls, specialized pallets Groom/handler travel, veterinary cert

⚠️ Critical Live Animal Requirements:

  • Health certificate issued by licensed veterinarian within 10 days of travel
  • CITES permits for endangered species required
  • Containers must meet IATA LAR specifications for species
  • No sedation of animals unless prescribed by veterinarian
  • Ambient temperature limits: usually 10-27°C
  • Maximum transit time varies by species (typically 24-48 hours)
  • Priority loading/unloading to minimize stress
  • 24-hour advance notification to airline required

Other Special Cargo Categories

Human Remains

Transport of human remains requires sensitivity, regulatory compliance, and specialized handling. Two main forms: cremated remains (ashes) or intact remains (in caskets).

  • Documentation: Death certificate, embalming certificate, consular documentation
  • Packaging: Hermetically sealed casket or urn in protective outer container
  • Handling: Respectful, discreet handling; priority loading; special handling code HUM
  • Regulations: Vary by country; some require health authority permits

Valuables and High-Value Cargo

Valuable cargo (diamonds, jewelry, currency, artwork, electronics) requires enhanced security measures:

  • Security: Locked containers, dedicated secure storage areas, armed escorts
  • Insurance: Declared value required, special cargo insurance
  • Tracking: Real-time GPS tracking, continuous monitoring
  • Chain of Custody: Documented handoffs with signatures at every transfer
  • Special Handling Code: VAL (valuable cargo)

Oversized and Heavy Cargo

Cargo exceeding standard dimensions or weight limits requires specialized handling:

Category Threshold Aircraft Type Handling Requirements
Heavy Cargo (HEA) >150 kg per piece Freighters, some widebody Forklifts, cranes, reinforced pallets
Oversized (BIG) >300 cm any dimension B747F, An-124, specialized Main deck loading, special clearance
Aircraft Parts (AOG) Varies Any available aircraft Priority handling, expedited customs
Machinery >1000 kg Freighters only Engineering review, load planning

Key Takeaways

  • Special cargo represents 20% of volume but often 50%+ premium pricing due to specialized handling requirements
  • Perishable cargo requires temperature-controlled cool chain with validated processes and rapid transit
  • Pharmaceutical logistics must comply with GDP guidelines including qualified personnel, validated equipment, and comprehensive documentation
  • Temperature excursions must be detected, documented, and assessed for product impact with severity classification
  • Live animals require IATA LAR-compliant containers, health certificates, and species-specific handling procedures
  • Human remains transport demands respectful handling with death certificates and embalming/cremation documentation
  • Valuable cargo needs enhanced security including locked containers, armed escorts, and real-time GPS tracking
  • TypeScript implementations enable automated temperature monitoring, excursion detection, and GDP compliance validation

Reference Standards and Implementation Notes

This standard is designed to interoperate with established international frameworks. Implementations SHOULD treat the following as the single source of truth and use this PHASE document as the interoperability layer that maps existing data into a machine-readable, signed, and auditable shape.

  • OpenAPI Specification 3.1 — REST API definition
  • JSON Schema 2020-12 — schema validation
  • ISO/IEC 27001:2022 — information security management
  • ISO/IEC 27018:2019 — PII protection in public clouds
  • ISO/IEC 17065:2012 — product certification bodies
  • CycloneDX 1.5 / SPDX 2.3 — software bill of materials
  • Sigstore (DSSE envelope, Rekor transparency log) — signature anchoring
  • in-toto Attestation Framework 1.0 — build provenance
  • IETF RFC 9700 — OAuth 2.1
  • IETF RFC 9457 — Problem Details for HTTP APIs
  • IETF RFC 8615 — well-known URIs

Practical Implementation Checklist

  • Manifest integrity — every published manifest is signed via Sigstore DSSE. Tampering is detectable without trusting an intermediary. Updates are issued as new manifests; the prior manifest is preserved with a supersede marker.
  • Retention — keep signed evidence for at least seven years to satisfy ISO/IEC 17065:2012 audit expectations. Some regulators require longer retention; in that case the deployment policy MUST extend the window rather than relying on the defaults.
  • Versioning — follow Semantic Versioning 2.0.0. Major version bumps require at least a 90-day overlap with the prior major version on every WIA-published reference implementation.
  • Negative-result reporting — a skipped indicator without recorded justification is treated as a failed indicator at audit time.
  • Key rotation — manifest signing keys are rotated per the deployment policy. The previous key is preserved in a published key history so that auditors can verify historical manifests offline.

Reference Topologies

Implementations declare their operational topology in the manifest: single-tenant edge (one runtime per organization), multi-tenant gateway (shared runtime with header-based isolation), federated mesh (multiple runtimes peer to one another and publish manifests to a directory service), or air-gapped batch (signed evidence package transported out-of-band). Each topology ships with reference test vectors so that implementations can validate their deployment matches the canonical pattern.

Open Governance

Issues, errata, and proposals are tracked at github.com/WIA-Official/wia-standards/issues with the appropriate label. The WIA Standards working group reviews open issues at the start of every minor release cycle and publishes the resulting decision log alongside the release notes. Errata are issued as patch releases; new normative requirements trigger minor bumps; backwards-incompatible changes trigger major bumps.
