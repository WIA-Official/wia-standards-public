Understanding the Global Air Cargo Market and Supply Chain

Introduction to Air Cargo

Air cargo represents one of the most critical components of global trade, moving high-value, time-sensitive goods across continents within hours. Despite accounting for only 1% of global freight volume, air cargo handles approximately 35% of global trade by value, exceeding $6 trillion annually.

The air cargo industry encompasses a complex ecosystem of airlines, freight forwarders, ground handlers, customs brokers, and technology providers working together to ensure efficient movement of goods. Understanding this ecosystem is essential for anyone involved in international logistics, e-commerce, or supply chain management.

Supply Chain Dynamics

Air Cargo Supply Chain Flow

The air cargo supply chain involves multiple touchpoints from shipper to consignee. Understanding this flow is critical for optimizing operations and implementing digital solutions.

Typical Supply Chain Journey:

  1. Booking: Shipper or freight forwarder books space with airline
  2. Pick-up: Cargo collected from shipper's location
  3. Origin Terminal: Cargo received, documented, screened, and stored
  4. Build-up: Cargo loaded into ULDs or loose-loaded onto aircraft
  5. Flight: Air transport to destination airport
  6. Break-down: Cargo unloaded from aircraft and ULDs
  7. Destination Terminal: Customs clearance and storage
  8. Delivery: Last-mile delivery to consignee

TypeScript Implementation: Cargo Shipment Tracking

Here's a comprehensive TypeScript implementation for modeling air cargo shipments and tracking their journey through the supply chain:

// Core types for air cargo shipment tracking
interface CargoShipment {
  awbNumber: string;                    // Air Waybill Number (e.g., "020-12345678")
  masterAWB?: string;                   // Master AWB for consolidated shipments
  houseAWB?: string;                    // House AWB for freight forwarder shipments
  shipper: Party;
  consignee: Party;
  origin: AirportCode;
  destination: AirportCode;
  pieces: number;
  weight: Weight;
  volume: Volume;
  cargoDescription: string;
  specialHandlingCodes: string[];       // SPX, EAW, PER, etc.
  declaredValue: Money;
  incoterms: string;                    // DDP, DDU, EXW, etc.
  currentStatus: ShipmentStatus;
  milestones: Milestone[];
  flightDetails: FlightSegment[];
}

interface Party {
  name: string;
  address: Address;
  contactPerson: string;
  phone: string;
  email: string;
  accountNumber?: string;
}

interface Address {
  street: string;
  city: string;
  state: string;
  postalCode: string;
  country: string;
}

type AirportCode = string;              // IATA 3-letter code (e.g., "JFK", "HKG")

interface Weight {
  value: number;
  unit: "KG" | "LBS";
}

interface Volume {
  value: number;
  unit: "CBM" | "CFT";
}

interface Money {
  amount: number;
  currency: string;                     // ISO 4217 code
}

enum ShipmentStatus {
  BOOKED = "BOOKED",
  RECEIVED = "RECEIVED",
  IN_TRANSIT = "IN_TRANSIT",
  ARRIVED = "ARRIVED",
  CUSTOMS_CLEARANCE = "CUSTOMS_CLEARANCE",
  OUT_FOR_DELIVERY = "OUT_FOR_DELIVERY",
  DELIVERED = "DELIVERED",
  EXCEPTION = "EXCEPTION"
}

interface Milestone {
  timestamp: Date;
  status: ShipmentStatus;
  location: AirportCode | string;
  description: string;
  updatedBy: string;
}

interface FlightSegment {
  flightNumber: string;
  departureAirport: AirportCode;
  arrivalAirport: AirportCode;
  scheduledDeparture: Date;
  scheduledArrival: Date;
  actualDeparture?: Date;
  actualArrival?: Date;
  aircraftType: string;
  pieces: number;
  weight: Weight;
}

// Air Cargo Management System
class AirCargoManager {
  private shipments: Map<string, CargoShipment>;

  constructor() {
    this.shipments = new Map();
  }

  // Create a new shipment booking
  createShipment(shipmentData: Omit<CargoShipment, "currentStatus" | "milestones">): CargoShipment {
    const shipment: CargoShipment = {
      ...shipmentData,
      currentStatus: ShipmentStatus.BOOKED,
      milestones: [{
        timestamp: new Date(),
        status: ShipmentStatus.BOOKED,
        location: shipmentData.origin,
        description: "Shipment booked",
        updatedBy: "SYSTEM"
      }]
    };

    this.shipments.set(shipment.awbNumber, shipment);
    console.log(`Shipment ${shipment.awbNumber} created successfully`);

    return shipment;
  }

  // Update shipment status
  updateStatus(
    awbNumber: string,
    newStatus: ShipmentStatus,
    location: string,
    description: string
  ): void {
    const shipment = this.shipments.get(awbNumber);

    if (!shipment) {
      throw new Error(`Shipment ${awbNumber} not found`);
    }

    shipment.currentStatus = newStatus;
    shipment.milestones.push({
      timestamp: new Date(),
      status: newStatus,
      location,
      description,
      updatedBy: "SYSTEM"
    });

    console.log(`Shipment ${awbNumber} updated to ${newStatus}`);
  }

  // Track shipment with full milestone history
  trackShipment(awbNumber: string): CargoShipment | undefined {
    return this.shipments.get(awbNumber);
  }

  // Calculate transit time
  calculateTransitTime(awbNumber: string): number | null {
    const shipment = this.shipments.get(awbNumber);

    if (!shipment || shipment.milestones.length < 2) {
      return null;
    }

    const firstMilestone = shipment.milestones[0];
    const lastMilestone = shipment.milestones[shipment.milestones.length - 1];

    const transitTime = lastMilestone.timestamp.getTime() - firstMilestone.timestamp.getTime();
    return Math.floor(transitTime / (1000 * 60 * 60)); // Convert to hours
  }

  // Get shipments by status
  getShipmentsByStatus(status: ShipmentStatus): CargoShipment[] {
    return Array.from(this.shipments.values())
      .filter(shipment => shipment.currentStatus === status);
  }

  // Calculate dimensional weight (for pricing)
  calculateDimensionalWeight(
    length: number,
    width: number,
    height: number,
    unit: "CM" | "IN" = "CM"
  ): Weight {
    const divisor = unit === "CM" ? 6000 : 366; // IATA standard
    const dimWeight = (length * width * height) / divisor;

    return {
      value: dimWeight,
      unit: "KG"
    };
  }

  // Validate AWB number format
  static validateAWBNumber(awb: string): boolean {
    // Format: XXX-XXXXXXXX (carrier prefix + 8-digit serial)
    const awbPattern = /^\d{3}-\d{8}$/;
    return awbPattern.test(awb);
  }
}

Industry Challenges and Opportunities

Current Challenges

The air cargo industry faces several structural challenges that require innovative solutions:

  • Capacity Constraints: Limited belly capacity due to reduced passenger flights during disruptions
  • Infrastructure Bottlenecks: Congestion at major cargo hubs affecting handling times
  • Digitalization Gap: Paper-based processes still dominate many operations
  • Rate Volatility: Significant price fluctuations based on supply-demand imbalances
  • Sustainability Pressure: Growing environmental regulations and carbon reduction targets
  • Security Compliance: Increasing regulatory requirements for cargo screening
  • Labor Shortages: Difficulty attracting and retaining skilled workforce

Emerging Opportunities

Despite challenges, the industry presents significant growth opportunities:

  • E-commerce Boom: Cross-border online shopping driving express cargo growth
  • Digital Transformation: E-AWB, blockchain, and IoT improving efficiency
  • Pharmaceutical Logistics: Growing demand for temperature-controlled shipping
  • Dedicated Freighter Demand: Investment in new freighter aircraft
  • Emerging Markets: Growing trade volumes in Asia, Africa, and Latin America
  • Sustainable Aviation Fuels: Transition to lower-carbon operations

Practical Implementation Checklist

  • Manifest integrity — every published manifest is signed via Sigstore DSSE. Tampering is detectable without trusting an intermediary. Updates are issued as new manifests; the prior manifest is preserved with a supersede marker.
  • Retention — keep signed evidence for at least seven years to satisfy ISO/IEC 17065:2012 audit expectations. Some regulators require longer retention; in that case the deployment policy MUST extend the window rather than relying on the defaults.
  • Versioning — follow Semantic Versioning 2.0.0. Major version bumps require at least a 90-day overlap with the prior major version on every WIA-published reference implementation.
  • Negative-result reporting — a skipped indicator without recorded justification is treated as a failed indicator at audit time.
  • Key rotation — manifest signing keys are rotated per the deployment policy. The previous key is preserved in a published key history so that auditors can verify historical manifests offline.

Reference Topologies

Implementations declare their operational topology in the manifest: single-tenant edge (one runtime per organization), multi-tenant gateway (shared runtime with header-based isolation), federated mesh (multiple runtimes peer to one another and publish manifests to a directory service), or air-gapped batch (signed evidence package transported out-of-band). Each topology ships with reference test vectors so that implementations can validate their deployment matches the canonical pattern.

Open Governance
