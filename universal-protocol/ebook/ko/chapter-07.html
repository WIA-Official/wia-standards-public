장 요약

핵심 요점:

  1. OAuth 2.0은 권한 부여 코드, 클라이언트 자격 증명, 장치 코드, 갱신 토큰과 같은 권한 부여 유형을 통해 위임된 권한 부여를 제공하여 암호 공유 없이 안전한 제3자 액세스를 가능하게 하며 공개 클라이언트 및 모바일 앱을 위한 PKCE(Proof Key for Code Exchange)를 지원합니다.
  2. OpenID Connect(OIDC)는 ID 토큰(JWT 형식), UserInfo 엔드포인트 및 디스커버리 프로토콜을 사용하여 인증 계층으로 OAuth 2.0을 확장하여 여러 애플리케이션 전반에 걸친 단일 사인온(SSO), ID 페더레이션 및 클레임 기반 ID 관리를 가능하게 합니다.
  3. JWT(JSON Web Tokens)는 암호화 서명(HMAC-SHA256, RS256, ES256)을 통해 상태 비저장 인증을 제공하며 클레임 기반 권한 부여, 토큰 만료 및 갱신 토큰 회전을 지원하면서 세션 저장소 요구 사항 없이 수평 확장을 가능하게 합니다.
  4. mTLS(Mutual TLS)는 서비스 간 통신을 위한 인증서 기반 인증을 제공하며 클라이언트와 서버 모두 양방향 검증을 위해 X.509 인증서를 제시하여 제로 트러스트 아키텍처에서 암호화된 채널을 보장하고 중간자 공격을 방지합니다.
  5. RBAC(Role-Based Access Control)는 역할 계층(관리자, 편집자, 뷰어)을 통해 권한을 할당하는 반면 ABAC(Attribute-Based Access Control)는 세분화되고 동적인 권한 부여를 위해 사용자 속성, 리소스 속성 및 환경 컨텍스트를 결합하는 정책 기반 결정을 사용합니다.
  6. 토큰 버킷, 리키 버킷 또는 슬라이딩 윈도우 알고리즘을 사용한 API 속도 제한은 남용으로부터 보호하며 Redis 또는 인메모리 저장소를 통해 시행되는 사용자별, IP별 및 API 키별 제한과 함께 Retry-After 헤더와 함께 429 Too Many Requests를 반환합니다.
  7. 보안 헤더(HSTS, CSP, X-Frame-Options, X-Content-Type-Options) 및 CORS 정책은 일반적인 공격(XSS, 클릭재킹, MIME 스니핑)으로부터 보호하는 반면 감사 로깅은 규정 준수 및 포렌식 분석을 위해 인증 이벤트, 권한 부여 결정 및 민감한 작업을 캡처합니다.

복습 질문

  1. OAuth 2.0 권한 부여 유형(권한 부여 코드, 클라이언트 자격 증명, 장치 코드, 암시적, 암호)을 비교하십시오. 웹 애플리케이션, 모바일 앱, IoT 장치 및 서버 간 통신을 지원하는 인증 시스템을 설계하십시오. PKCE, 상태 매개변수 및 갱신 토큰 회전을 언제 사용하는지 설명하십시오. 권한 부여 코드 가로채기 및 토큰 도난을 어떻게 방지합니까?
  2. OAuth 2.0과 OpenID Connect 간의 관계를 설명하십시오. ID 토큰, UserInfo 엔드포인트 및 디스커버리 프로토콜을 사용하여 OIDC를 사용하는 단일 사인온(SSO) 시스템을 설계하십시오. 세션 관리, 여러 앱 전반에 걸친 로그아웃 전파 및 ID 클레임 검증을 어떻게 처리합니까? ID 토큰과 액세스 토큰을 비교하십시오.
  3. 액세스 토큰 및 갱신 토큰을 사용하여 JWT 기반 인증 시스템을 설계하십시오. RS256 서명을 사용한 토큰 생성, 클레임 검증, 만료 처리 및 갱신 토큰 회전을 구현하십시오. 상태 비저장 시스템에서 토큰 해지를 어떻게 처리합니까? JWT 저장소 옵션(localStorage, sessionStorage, httpOnly 쿠키)과 보안 의미를 비교하십시오.
  4. 마이크로서비스 아키텍처에서 서비스 간 인증을 위한 mTLS를 구현하십시오. cert-manager 또는 Vault를 사용하여 인증서 프로비저닝, 회전 및 해지 워크플로를 설계하십시오. mTLS가 서비스 메시(Istio, Linkerd)와 어떻게 통합됩니까? 서비스 인증을 위한 mTLS와 JWT 토큰을 비교하십시오. 각 접근 방식을 언제 사용하시겠습니까?
  5. RBAC와 ABAC를 모두 지원하는 권한 부여 시스템을 설계하십시오. OPA(Open Policy Agent) 또는 Cedar를 사용하여 역할 계층, 권한 상속 및 정책 기반 액세스 제어를 구현하십시오. 사용자 역할, 리소스 속성, 시간 기반 제한 및 지리적 제약을 결합하는 복잡한 정책을 어떻게 평가합니까? 정책 복잡성에 따라 권한 부여가 어떻게 확장됩니까?
  6. 토큰 버킷 및 슬라이딩 윈도우 알고리즘을 사용하여 Redis를 사용하는 분산 속도 제한을 구현하십시오. 무료 계층과 유료 계층에 대한 다양한 할당량으로 사용자별, IP별 및 API 키별 제한을 설계하십시오. 여러 API 게이트웨이 인스턴스 전반에 걸쳐 속도 제한 동기화를 어떻게 처리합니까? 분산 속도 제한에 대한 정확성과 성능 간의 트레이드오프를 설명하십시오.
  7. 보안 헤더(HSTS, CSP, X-Frame-Options), CORS 정책, 입력 검증, 출력 인코딩 및 감사 로깅을 포함한 포괄적인 API 보안을 설계하십시오. OWASP Top 10 위협(인젝션, 손상된 인증, XSS, CSRF)에 대한 보호를 구현하십시오. GDPR 및 SOC 2 규정 준수를 위해 민감한 작업을 어떻게 감사합니까? 어떤 이벤트가 보안 알림을 트리거해야 합니까?
