장 요약

핵심 요점:

  1. Protocol Buffers(protobuf)는 JSON/XML에 비해 3-10배 작은 메시지 크기와 20-100배 빠른 파싱을 제공하는 효율적인 바이너리 직렬화를 제공하여 고처리량 마이크로서비스 통신 및 대역폭 제약이 있는 모바일 애플리케이션에 이상적입니다.
  2. 의미 체계 버전 관리(semver)를 Protocol Buffers 필드 번호 지정과 결합한 메시지 형식 버전 관리는 하위 및 상위 호환성을 가능하게 하여 기존 클라이언트를 손상시키거나 조정된 릴리스를 요구하지 않고 독립적인 서비스 배포를 허용합니다.
  3. HTTP Accept/Content-Type 헤더를 통한 콘텐츠 협상은 동일한 API 내에서 다양한 메시지 형식(JSON, XML, protobuf, MessagePack)을 가능하게 하여 형식 어댑터를 통한 통합 백엔드 구현을 유지하면서 다양한 클라이언트 요구 사항을 지원합니다.
  4. 스키마 레지스트리(Confluent Schema Registry, AWS Glue Schema Registry)는 Kafka, MQTT 및 이벤트 기반 아키텍처를 위한 중앙 집중식 스키마 관리를 제공하여 수집 시 스키마 검증을 시행하고 스키마 진화 정책(하위, 상위, 전체 호환성)을 가능하게 합니다.
  5. GraphQL 스키마 정의 언어(SDL) 및 인트로스펙션은 클라이언트 기반 필드 선택을 통해 강력한 타입의 메시지 계약을 가능하게 하여 과다 가져오기 및 과소 가져오기 문제를 줄이면서 자동 API 문서 및 타입 안전 코드 생성을 제공합니다.
  6. gRPC 메시지 스트리밍 패턴(단일, 서버 스트리밍, 클라이언트 스트리밍, 양방향 스트리밍)은 HTTP/2 멀티플렉싱 및 흐름 제어를 활용하여 효율적인 실시간 통신을 지원하며 파일 업로드에서 라이브 비디오 스트리밍까지 사용 사례를 지원합니다.
  7. gzip, Brotli 또는 LZ4를 사용한 메시지 압축은 최소한의 CPU 오버헤드를 추가하면서 텍스트 기반 형식(JSON, XML)에 대해 네트워크 대역폭을 60-90% 줄이며 HTTP Accept-Encoding 헤더를 통한 자동 압축 협상을 제공합니다.

복습 질문

  1. 유니버설 프로토콜 구현에서 메시지 직렬화를 위한 Protocol Buffers, JSON, XML 및 MessagePack을 비교하십시오. 와이어 크기, 파싱 성능, 인간 가독성, 스키마 진화 및 생태계 도구 지원 측면에서 트레이드오프를 분석하십시오. 각 형식을 언제 선택하시겠습니까?
  2. 하위 및 상위 호환성을 지원하는 Protocol Buffers 스키마에 대한 버전 관리 전략을 설계하십시오. 필드 번호 지정 규칙, 더 이상 사용되지 않는 필드 처리, 예약된 필드 번호 및 안전한 스키마 진화 패턴을 설명하십시오. 중단 변경 사항을 어떻게 처리합니까?
  3. 여러 메시지 형식(JSON, XML, protobuf)을 지원하는 REST API에서 콘텐츠 협상을 구현하십시오. HTTP 헤더 협상 논리, 형식 어댑터, 지원되지 않는 형식에 대한 오류 처리 및 형식별 응답에 대한 캐싱 전략을 설계하십시오.
  4. 이벤트 기반 아키텍처에서 스키마 레지스트리(Confluent, AWS Glue)가 스키마 검증을 시행하는 방법을 설명하십시오. 스키마 진화, 호환성 확인 및 스키마 버전 마이그레이션 전략을 처리하는 스키마 레지스트리 통합을 통해 Kafka 프로듀서/컨슈머 구현을 설계하십시오.
  5. 다양한 사용 사례에 대해 JSON을 사용한 REST, Protocol Buffers를 사용한 gRPC 및 GraphQL을 비교하십시오. 메시지 형식 선택이 API 설계, 클라이언트 복잡성, 성능 특성 및 모니터링, 디버깅 및 속도 제한과 같은 운영 고려 사항에 어떻게 영향을 미치는지 분석하십시오.
  6. 대역폭 감소와 CPU 오버헤드의 균형을 맞추는 유니버설 프로토콜에 대한 메시지 압축 전략을 설계하십시오. 다양한 페이로드 크기 및 유형에 대해 gzip, Brotli 및 LZ4를 비교하십시오. 자동 압축 협상을 구현하고 압축 오류를 처리하려면 어떻게 해야 합니까?
  7. gRPC 스트리밍 패턴(단일, 서버 스트리밍, 클라이언트 스트리밍, 양방향 스트리밍) 및 메시지 형식 의미를 설명하십시오. 양방향 스트리밍을 사용하여 실시간 채팅 애플리케이션을 설계하고 메시지 순서 지정, 흐름 제어, 배압 및 오류 복구를 다루십시오.
