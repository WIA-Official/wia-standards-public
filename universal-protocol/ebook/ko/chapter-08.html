장 요약

핵심 요점:

  1. OpenTelemetry는 언어 전반에 걸친 표준화된 API 및 SDK를 사용하여 분산 추적, 메트릭 수집 및 로그 상관관계를 통해 통합 관찰 가능성을 제공하며 HTTP, gRPC 및 데이터베이스 클라이언트에 대한 자동 계측을 통해 마이크로서비스 전반에 걸친 엔드투엔드 가시성을 가능하게 합니다.
  2. W3C Trace Context 전파를 사용한 분산 추적은 추적 ID, 스팬 ID 및 부모-자식 관계를 통해 서비스 경계를 넘어 요청을 추적하여 복잡한 분산 시스템에서 지연 시간 분석, 종속성 매핑 및 병목 현상 식별을 가능하게 합니다.
  3. 서킷 브레이커 패턴(Hystrix, Resilience4j)은 오류 임계값을 감지하고 회로를 열어 요청을 빠르게 실패하도록 하여 연쇄 장애를 방지하며 자동 복구를 위한 구성 가능한 반개방 상태, 타임아웃 처리 및 다운스트림 서비스를 보호하는 폴백 전략을 제공합니다.
  4. Redis, Memcached 또는 CDN 엣지 캐싱을 사용한 캐싱 전략은 캐시 어사이드, 라이트 스루 및 라이트 비하인드 패턴을 통해 지연 시간과 백엔드 로드를 줄이며 TTL 기반 만료, 캐시 무효화 및 캐시 워밍 전략을 통해 신선도와 성능의 균형을 맞춥니다.
  5. GraphQL 페더레이션은 여러 서비스 전반에 걸친 분산 스키마 구성을 가능하게 하여 팀이 독립적으로 서브그래프를 개발할 수 있도록 하면서 서비스 전반에 걸쳐 엔티티를 해석하는 통합 API 게이트웨이를 유지하며 중단 변경 없이 스키마 진화를 지원합니다.
  6. Protocol buffer 확장 및 사용자 지정 옵션은 검증 규칙, 권한 부여 정책 및 코드 생성 힌트를 위한 .proto 파일의 도메인별 주석을 가능하게 하여 protobuf 호환성 및 스키마 진화를 유지하면서 프레임워크별 동작을 지원합니다.
  7. 서비스 메시 데이터 플레인(Envoy, Linkerd-proxy)은 카오스 엔지니어링을 위한 트래픽 분할, 재시도 예산, 타임아웃 전파 및 장애 주입을 포함한 고급 기능을 제공하여 애플리케이션 코드 변경 없이 점진적 롤아웃, A/B 테스트 및 복원력 테스트를 가능하게 합니다.

복습 질문

  1. HTTP/REST 및 gRPC를 사용하는 마이크로서비스 아키텍처를 위한 OpenTelemetry 계측 전략을 설계하십시오. 추적 컨텍스트 전파(W3C Trace Context), 스팬 생성, 메트릭 수집(요청 기간, 오류율) 및 로그 상관관계를 구현하십시오. Jaeger 또는 Prometheus로 텔레메트리 데이터를 내보내려면 어떻게 합니까? 어떤 샘플링 전략이 관찰 가능성과 오버헤드의 균형을 맞춥니까?
  2. 추적 ID, 스팬 ID 및 부모-자식 관계를 사용한 분산 추적 분석을 설명하십시오. HTTP → gRPC → 데이터베이스 → 캐시 스팬을 보여주는 추적이 주어지면 성능 병목 현상을 식별하고 임계 경로 지연 시간을 분석하며 서비스 종속성을 감지하십시오. SLO 위반에 대한 추적 기반 알림을 어떻게 구현합니까?
  3. gRPC 및 REST 클라이언트를 위해 Resilience4j를 사용하여 서킷 브레이커를 구현하십시오. 장애 감지(오류 임계값, 타임아웃), 회로 상태(닫힘, 열림, 반개방), 자동 복구 및 폴백 전략을 설계하십시오. 서킷 브레이커가 재시도 정책과 어떻게 상호 작용합니까? 장애 도메인을 격리하기 위해 벌크헤드를 언제 사용해야 합니까?
  4. CDN 엣지 캐시, Redis 애플리케이션 캐시 및 데이터베이스 쿼리 캐시를 사용하는 다층 캐싱 전략을 설계하십시오. TTL 만료, 업데이트 시 캐시 무효화 및 고트래픽 키에 대한 캐시 워밍을 사용하여 캐시 어사이드 패턴을 구현하십시오. 다양한 사용 사례에 대해 Redis와 Memcached를 비교하십시오. 캐시 만료 중 캐시 스탬피드를 어떻게 처리합니까?
  5. Apollo Federation 또는 Schema Stitching을 사용하여 GraphQL 페더레이션을 구현하십시오. 다양한 팀이 소유한 서브그래프, 서비스 전반에 걸친 엔티티 해석 및 게이트웨이 구성을 설계하십시오. 스키마 진화, 중단 변경 및 하위 호환성을 어떻게 처리합니까? 페더레이션을 BFF(Backend for Frontend) 패턴과 비교하십시오.
  6. 검증, 권한 부여 및 코드 생성을 위한 사용자 지정 옵션을 사용하여 Protocol Buffer 스키마 확장을 설계하십시오. 필드 수준 검증 규칙(필수, 최소/최대, 정규식), 서비스 수준 권한 부여 정책을 구현하고 내장 검증을 통해 클라이언트 SDK를 생성하십시오. 사용자 지정 옵션이 스키마 진화 및 하위 호환성에 어떻게 영향을 미칩니까?
  7. 서비스 메시(Istio, Linkerd)를 사용하여 고급 트래픽 관리를 구현하십시오. 카나리 배포를 위한 트래픽 분할(95% 안정, 5% 카나리), 재시도 폭풍을 방지하기 위한 재시도 예산, 서비스 전반에 걸친 타임아웃 전파 및 카오스 엔지니어링을 위한 장애 주입을 설계하십시오. 카나리 상태를 모니터링하고 오류 시 자동 롤백을 구현하려면 어떻게 합니까?
