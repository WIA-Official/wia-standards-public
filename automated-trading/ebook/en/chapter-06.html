Chapter 6: Backtesting and Simulation

Chapter Overview: Backtesting validates trading strategies using historical data before risking real capital. This chapter covers backtesting methodology, performance metrics, common pitfalls, walk-forward analysis, and Monte Carlo simulation. Proper backtesting is essential for understanding strategy behavior and avoiding costly mistakes.

1. Backtesting Fundamentals

Backtesting simulates strategy performance using historical market data. The goal is to estimate how a strategy would have performed in the past, providing insights into expected returns, risk characteristics, and potential failure modes. However, backtesting has significant limitations that must be understood and mitigated.

Backtesting Pitfall Description Mitigation Strategy
Survivorship Bias Using only currently active symbols Include delisted/bankrupt companies in data
Look-Ahead Bias Using future information in signals Careful timestamp validation, point-in-time data
Overfitting Strategy too optimized to historical data Out-of-sample testing, walk-forward analysis
Transaction Cost Ignorance Not accounting for commissions and slippage Model realistic costs, use bid-ask spread
Market Impact Assuming unlimited liquidity Volume constraints, progressive slippage model
Data Mining Testing too many strategies on same data Bonferroni correction, hold-out validation set

2. Backtesting Engine Implementation

TypeScript: Professional Backtesting Framework
interface BacktestConfig {
  startDate: Date;
  endDate: Date;
  initialCapital: number;
  commission: number;          // Per trade commission
  slippageBps: number;        // Slippage in basis points
  maxPositionSize?: number;
  enableMarginTrading?: boolean;
}

interface HistoricalBar {
  timestamp: number;
  open: number;
  high: number;
  low: number;
  close: number;
  volume: number;
}

interface BacktestTrade {
  entryTime: number;
  exitTime: number;
  entryPrice: number;
  exitPrice: number;
  quantity: number;
  side: 'LONG' | 'SHORT';
  pnl: number;
  pnlPercent: number;
  commission: number;
  slippage: number;
}

interface BacktestMetrics {
  totalReturn: number;
  annualizedReturn: number;
  sharpeRatio: number;
  sortinoRatio: number;
  maxDrawdown: number;
  maxDrawdownDuration: number;
  winRate: number;
  profitFactor: number;
  averageWin: number;
  averageLoss: number;
  totalTrades: number;
  totalCommission: number;
  totalSlippage: number;
  calmarRatio: number;
}

class BacktestEngine {
  private config: BacktestConfig;
  private trades: BacktestTrade[] = [];
  private equity: number[];
  private dailyReturns: number[];
  private currentPosition: number = 0;
  private cash: number;
  private entryPrice: number = 0;
  private entryTime: number = 0;

  constructor(config: BacktestConfig) {
    this.config = config;
    this.cash = config.initialCapital;
    this.equity = [config.initialCapital];
    this.dailyReturns = [];
  }

  async run(
    data: HistoricalBar[],
    strategy: TradingStrategy
  ): Promise {
    // Filter data to backtest period
    const filteredData = data.filter(bar =>
      bar.timestamp >= this.config.startDate.getTime() &&
      bar.timestamp <= this.config.endDate.getTime()
    );

    // Iterate through historical data
    for (let i = 0; i < filteredData.length; i++) {
      const bar = filteredData[i];
      const historicalData = filteredData.slice(0, i + 1);

      // Generate signal from strategy
      const signal = strategy.generateSignal(historicalData);

      // Execute trading logic
      if (signal.action === 'BUY' && this.currentPosition === 0) {
        this.enterLong(bar, signal.quantity || this.calculatePositionSize(bar));
      } else if (signal.action === 'SELL' && this.currentPosition > 0) {
        this.closeLong(bar);
      } else if (signal.action === 'SHORT' && this.currentPosition === 0) {
        this.enterShort(bar, signal.quantity || this.calculatePositionSize(bar));
      } else if (signal.action === 'COVER' && this.currentPosition < 0) {
        this.closeShort(bar);
      }

      // Update equity curve
      const portfolioValue = this.calculatePortfolioValue(bar.close);
      this.equity.push(portfolioValue);

      // Calculate daily return
      if (this.equity.length > 1) {
        const dailyReturn = (portfolioValue - this.equity[this.equity.length - 2]) /
                           this.equity[this.equity.length - 2];
        this.dailyReturns.push(dailyReturn);
      }
    }

    // Close any open position at end of backtest
    const lastBar = filteredData[filteredData.length - 1];
    if (this.currentPosition !== 0) {
      this.currentPosition > 0
        ? this.closeLong(lastBar)
        : this.closeShort(lastBar);
    }

    // Calculate metrics
    const metrics = this.calculateMetrics();

    return {
      metrics,
      trades: this.trades,
      equity: this.equity,
      dailyReturns: this.dailyReturns
    };
  }

  private enterLong(bar: HistoricalBar, quantity: number): void {
    const slippage = bar.close * (this.config.slippageBps / 10000);
    const entryPrice = bar.close + slippage;
    const commission = this.config.commission;
    const cost = (quantity * entryPrice) + commission;

    if (cost > this.cash) {
      return; // Insufficient capital
    }

    this.currentPosition = quantity;
    this.entryPrice = entryPrice;
    this.entryTime = bar.timestamp;
    this.cash -= cost;
  }

  private closeLong(bar: HistoricalBar): void {
    const slippage = bar.close * (this.config.slippageBps / 10000);
    const exitPrice = bar.close - slippage;
    const commission = this.config.commission;
    const proceeds = (this.currentPosition * exitPrice) - commission;

    const pnl = proceeds - (this.currentPosition * this.entryPrice);
    const pnlPercent = pnl / (this.currentPosition * this.entryPrice);

    this.trades.push({
      entryTime: this.entryTime,
      exitTime: bar.timestamp,
      entryPrice: this.entryPrice,
      exitPrice,
      quantity: this.currentPosition,
      side: 'LONG',
      pnl,
      pnlPercent,
      commission: this.config.commission * 2,
      slippage: slippage * this.currentPosition * 2
    });

    this.cash += proceeds;
    this.currentPosition = 0;
  }

  private enterShort(bar: HistoricalBar, quantity: number): void {
    const slippage = bar.close * (this.config.slippageBps / 10000);
    const entryPrice = bar.close - slippage;
    const commission = this.config.commission;
    const proceeds = (quantity * entryPrice) - commission;

    this.currentPosition = -quantity;
    this.entryPrice = entryPrice;
    this.entryTime = bar.timestamp;
    this.cash += proceeds;
  }

  private closeShort(bar: HistoricalBar): void {
    const slippage = bar.close * (this.config.slippageBps / 10000);
    const exitPrice = bar.close + slippage;
    const commission = this.config.commission;
    const cost = (Math.abs(this.currentPosition) * exitPrice) + commission;

    const pnl = (Math.abs(this.currentPosition) * this.entryPrice) - cost;
    const pnlPercent = pnl / (Math.abs(this.currentPosition) * this.entryPrice);

    this.trades.push({
      entryTime: this.entryTime,
      exitTime: bar.timestamp,
      entryPrice: this.entryPrice,
      exitPrice,
      quantity: Math.abs(this.currentPosition),
      side: 'SHORT',
      pnl,
      pnlPercent,
      commission: this.config.commission * 2,
      slippage: slippage * Math.abs(this.currentPosition) * 2
    });

    this.cash -= cost;
    this.currentPosition = 0;
  }

  private calculatePositionSize(bar: HistoricalBar): number {
    const maxPositionValue = this.config.maxPositionSize ||
                            (this.cash * 0.95);
    return Math.floor(maxPositionValue / bar.close);
  }

  private calculatePortfolioValue(currentPrice: number): number {
    const positionValue = this.currentPosition * currentPrice;
    return this.cash + positionValue;
  }

  private calculateMetrics(): BacktestMetrics {
    const finalValue = this.equity[this.equity.length - 1];
    const totalReturn = (finalValue - this.config.initialCapital) /
                       this.config.initialCapital;

    // Calculate Sharpe Ratio
    const avgReturn = this.dailyReturns.reduce((a, b) => a + b, 0) /
                     this.dailyReturns.length;
    const stdDev = Math.sqrt(
      this.dailyReturns.reduce(
        (sum, r) => sum + Math.pow(r - avgReturn, 2),
        0
      ) / this.dailyReturns.length
    );
    const sharpeRatio = (avgReturn / stdDev) * Math.sqrt(252);

    // Calculate Sortino Ratio (downside deviation only)
    const downsideReturns = this.dailyReturns.filter(r => r < 0);
    const downsideDeviation = downsideReturns.length > 0
      ? Math.sqrt(
          downsideReturns.reduce((sum, r) => sum + Math.pow(r, 2), 0) /
          downsideReturns.length
        )
      : 0;
    const sortinoRatio = downsideDeviation > 0
      ? (avgReturn / downsideDeviation) * Math.sqrt(252)
      : 0;

    // Calculate Maximum Drawdown
    let maxDrawdown = 0;
    let peak = this.equity[0];
    let maxDrawdownDuration = 0;
    let currentDrawdownDuration = 0;

    for (const value of this.equity) {
      if (value > peak) {
        peak = value;
        currentDrawdownDuration = 0;
      } else {
        const drawdown = (peak - value) / peak;
        maxDrawdown = Math.max(maxDrawdown, drawdown);
        currentDrawdownDuration++;
        maxDrawdownDuration = Math.max(maxDrawdownDuration, currentDrawdownDuration);
      }
    }

    // Win rate and profit factor
    const winningTrades = this.trades.filter(t => t.pnl > 0);
    const losingTrades = this.trades.filter(t => t.pnl < 0);
    const winRate = this.trades.length > 0
      ? winningTrades.length / this.trades.length
      : 0;

    const totalWin = winningTrades.reduce((sum, t) => sum + t.pnl, 0);
    const totalLoss = Math.abs(losingTrades.reduce((sum, t) => sum + t.pnl, 0));
    const profitFactor = totalLoss > 0 ? totalWin / totalLoss : 0;

    const averageWin = winningTrades.length > 0
      ? totalWin / winningTrades.length
      : 0;
    const averageLoss = losingTrades.length > 0
      ? totalLoss / losingTrades.length
      : 0;

    // Calculate annualized return
    const years = (this.config.endDate.getTime() - this.config.startDate.getTime()) /
                 (365.25 * 24 * 60 * 60 * 1000);
    const annualizedReturn = Math.pow(1 + totalReturn, 1 / years) - 1;

    // Calmar Ratio (annualized return / max drawdown)
    const calmarRatio = maxDrawdown > 0 ? annualizedReturn / maxDrawdown : 0;

    // Total costs
    const totalCommission = this.trades.reduce((sum, t) => sum + t.commission, 0);
    const totalSlippage = this.trades.reduce((sum, t) => sum + t.slippage, 0);

    return {
      totalReturn,
      annualizedReturn,
      sharpeRatio,
      sortinoRatio,
      maxDrawdown,
      maxDrawdownDuration,
      winRate,
      profitFactor,
      averageWin,
      averageLoss,
      totalTrades: this.trades.length,
      totalCommission,
      totalSlippage,
      calmarRatio
    };
  }
}

interface TradingStrategy {
  generateSignal(data: HistoricalBar[]): {
    action: 'BUY' | 'SELL' | 'SHORT' | 'COVER' | 'HOLD';
    quantity?: number;
  };
}

interface BacktestResult {
  metrics: BacktestMetrics;
  trades: BacktestTrade[];
  equity: number[];
  dailyReturns: number[];
}

3. Performance Metrics

Comprehensive performance analysis requires multiple metrics that capture different aspects of strategy behavior. No single metric tells the complete story.

Metric Formula/Description Good Value Interpretation
Sharpe Ratio Excess return / Standard deviation > 1.5 Risk-adjusted return; higher is better
Sortino Ratio Excess return / Downside deviation > 2.0 Like Sharpe but only penalizes downside
Maximum Drawdown Peak-to-trough decline < 20% Worst historical loss period
Calmar Ratio Annual return / Max drawdown > 1.0 Return per unit of downside risk
Win Rate Winning trades / Total trades 40-60% Percentage of profitable trades
Profit Factor Gross profit / Gross loss > 1.5 Total winnings vs total losses

4. Walk-Forward Analysis

Walk-forward analysis is a more robust validation technique than simple train-test splits. The strategy is optimized on a training window, tested on an out-of-sample period, then the window rolls forward. This simulates continuous re-optimization as would occur in live trading.

Walk-Forward Process

  1. Define Windows: Training period (e.g., 12 months) and test period (e.g., 3 months)
  2. Optimize: Find best parameters on training data
  3. Test: Evaluate performance on out-of-sample test period
  4. Roll Forward: Move windows forward by test period duration
  5. Repeat: Continue until all data is processed
  6. Aggregate: Combine all out-of-sample periods for final metrics

5. Monte Carlo Simulation

Monte Carlo simulation generates thousands of possible future scenarios by randomly resampling historical returns. This provides a distribution of possible outcomes rather than a single point estimate, helping quantify uncertainty and tail risks.

6. Common Pitfalls and Solutions

Avoiding Backtest Pitfalls

  • Use Realistic Costs: Model commissions, slippage, and market impact accurately
  • Point-in-Time Data: Ensure no look-ahead bias; use data as it was known historically
  • Out-of-Sample Testing: Always reserve data that was never used in optimization
  • Multiple Regimes: Test across bull markets, bear markets, and high volatility periods
  • Diversify Strategies: Don't rely on a single strategy; combine multiple uncorrelated approaches
  • Track Degradation: Monitor live performance vs backtest; investigate significant deviations

Summary

Key Takeaways

  • Backtest Carefully: Avoid look-ahead bias, survivorship bias, and overfitting through rigorous methodology
  • Multiple Metrics: Evaluate strategies using diverse metrics; no single number tells the full story
  • Walk-Forward Validation: More realistic than static train-test splits; simulates ongoing optimization
  • Model Costs: Transaction costs dramatically impact high-frequency strategies; always include realistic estimates
  • Expect Degradation: Live performance typically worse than backtest; plan for 20-30% degradation

Practical Implementation Checklist

  • Manifest integrity — every published manifest is signed via Sigstore DSSE. Tampering is detectable without trusting an intermediary. Updates are issued as new manifests; the prior manifest is preserved with a supersede marker.
  • Retention — keep signed evidence for at least seven years to satisfy ISO/IEC 17065:2012 audit expectations. Some regulators require longer retention; in that case the deployment policy MUST extend the window rather than relying on the defaults.
  • Versioning — follow Semantic Versioning 2.0.0. Major version bumps require at least a 90-day overlap with the prior major version on every WIA-published reference implementation.
  • Negative-result reporting — a skipped indicator without recorded justification is treated as a failed indicator at audit time.
  • Key rotation — manifest signing keys are rotated per the deployment policy. The previous key is preserved in a published key history so that auditors can verify historical manifests offline.

Reference Topologies

Implementations declare their operational topology in the manifest: single-tenant edge (one runtime per organization), multi-tenant gateway (shared runtime with header-based isolation), federated mesh (multiple runtimes peer to one another and publish manifests to a directory service), or air-gapped batch (signed evidence package transported out-of-band). Each topology ships with reference test vectors so that implementations can validate their deployment matches the canonical pattern.
