Chapter 3: Technical Architecture

Chapter Overview: This chapter explores the technical infrastructure required for automated trading systems, from order management and execution to low-latency networking and co-location strategies. We examine architectural patterns, system components, and performance optimization techniques for building robust trading platforms.

1. System Architecture Overview

Modern automated trading systems are complex distributed systems that must handle high-frequency data streams, execute trades with microsecond latency, maintain risk controls, and ensure regulatory compliance. The architecture typically consists of multiple layers working in concert.

Component Function Latency Target Critical Features
Market Data Gateway Real-time market data ingestion < 1 microsecond Multicast, normalization, failover
Strategy Engine Signal generation and decision making < 10 microseconds Event-driven, parallel processing
Risk Manager Pre-trade and post-trade risk checks < 5 microseconds Real-time limits, circuit breakers
Order Management System Order lifecycle management < 20 microseconds State machine, audit trail
Execution Gateway Exchange connectivity and order routing < 50 microseconds FIX protocol, smart routing
Position Manager Real-time position tracking and P&L < 100 microseconds Multi-asset, multi-currency

2. Order Management System (OMS)

The Order Management System is the central hub for managing the complete lifecycle of trading orders, from creation through execution to settlement. A robust OMS must handle thousands of orders per second while maintaining perfect accuracy and auditability.

class OrderManagementSystem {
  private orders: Map = new Map();
  private orderIdCounter: number = 1;
  private eventListeners: OrderEventListener[] = [];

  async submitOrder(request: OrderRequest): Promise {
    // Generate unique order ID
    const orderId = this.generateOrderId();

    // Create order object
    const order: Order = {
      orderId,
      clientOrderId: request.clientOrderId,
      symbol: request.symbol,
      side: request.side,
      type: request.type,
      quantity: request.quantity,
      price: request.price,
      stopPrice: request.stopPrice,
      timeInForce: request.timeInForce || 'GTC',
      status: OrderStatus.PENDING,
      filledQuantity: 0,
      averagePrice: 0,
      timestamp: Date.now(),
      lastUpdateTime: Date.now(),
      strategyId: request.strategyId,
      accountId: request.accountId
    };

    // Pre-trade risk checks
    const riskCheckResult = await this.performRiskChecks(order);
    if (!riskCheckResult.passed) {
      order.status = OrderStatus.REJECTED;
      this.emitEvent('ORDER_REJECTED', order, riskCheckResult.reason);
      throw new Error(`Order rejected: ${riskCheckResult.reason}`);
    }

    // Store order
    this.orders.set(orderId, order);

    // Submit to execution gateway
    try {
      await this.sendToExecutionGateway(order);
      order.status = OrderStatus.SUBMITTED;
      order.lastUpdateTime = Date.now();
      this.emitEvent('ORDER_SUBMITTED', order);
    } catch (error) {
      order.status = OrderStatus.REJECTED;
      this.emitEvent('ORDER_REJECTED', order, error.message);
      throw error;
    }

    return order;
  }

  async cancelOrder(orderId: string): Promise {
    const order = this.orders.get(orderId);
    if (!order) {
      throw new Error(`Order not found: ${orderId}`);
    }

    if (order.status === OrderStatus.FILLED ||
        order.status === OrderStatus.CANCELLED) {
      throw new Error(`Cannot cancel order in status ${order.status}`);
    }

    // Send cancel request to exchange
    await this.sendCancelRequest(orderId);

    order.status = OrderStatus.CANCELLED;
    order.lastUpdateTime = Date.now();
    this.emitEvent('ORDER_CANCELLED', order);
  }

  async modifyOrder(
    orderId: string,
    modifications: Partial
  ): Promise {
    const order = this.orders.get(orderId);
    if (!order) {
      throw new Error(`Order not found: ${orderId}`);
    }

    // Create new order with modifications
    const modifiedOrder = {
      ...order,
      ...modifications,
      lastUpdateTime: Date.now()
    };

    // Risk checks on modified order
    const riskCheckResult = await this.performRiskChecks(modifiedOrder);
    if (!riskCheckResult.passed) {
      throw new Error(`Modification rejected: ${riskCheckResult.reason}`);
    }

    // Send modification to exchange
    await this.sendModificationRequest(orderId, modifications);

    // Update stored order
    Object.assign(order, modifications);
    order.lastUpdateTime = Date.now();
    this.emitEvent('ORDER_MODIFIED', order);

    return order;
  }

  onExecutionReport(report: ExecutionReport): void {
    const order = this.orders.get(report.orderId);
    if (!order) {
      console.error(`Unknown order ID in execution report: ${report.orderId}`);
      return;
    }

    // Update order based on execution report
    switch (report.executionType) {
      case 'FILL':
      case 'PARTIAL_FILL':
        const fillQuantity = report.lastQuantity;
        const fillPrice = report.lastPrice;

        order.filledQuantity += fillQuantity;
        order.averagePrice = (
          (order.averagePrice * (order.filledQuantity - fillQuantity)) +
          (fillPrice * fillQuantity)
        ) / order.filledQuantity;

        order.status = order.filledQuantity >= order.quantity
          ? OrderStatus.FILLED
          : OrderStatus.PARTIALLY_FILLED;

        this.emitEvent('ORDER_FILLED', order, report);
        break;

      case 'CANCELLED':
        order.status = OrderStatus.CANCELLED;
        this.emitEvent('ORDER_CANCELLED', order);
        break;

      case 'REJECTED':
        order.status = OrderStatus.REJECTED;
        this.emitEvent('ORDER_REJECTED', order, report.rejectReason);
        break;

      case 'EXPIRED':
        order.status = OrderStatus.EXPIRED;
        this.emitEvent('ORDER_EXPIRED', order);
        break;
    }

    order.lastUpdateTime = Date.now();
  }

  getOrder(orderId: string): Order | undefined {
    return this.orders.get(orderId);
  }

  getOrdersByStrategy(strategyId: string): Order[] {
    return Array.from(this.orders.values())
      .filter(order => order.strategyId === strategyId);
  }

  getActiveOrders(): Order[] {
    return Array.from(this.orders.values())
      .filter(order =>
        order.status === OrderStatus.SUBMITTED ||
        order.status === OrderStatus.PARTIALLY_FILLED
      );
  }

  private generateOrderId(): string {
    const timestamp = Date.now();
    const counter = this.orderIdCounter++;
    return `ORD-${timestamp}-${counter}`;
  }

  private async performRiskChecks(order: Order): Promise {

  private emitEvent(
    eventType: string,
    order: Order,
    additionalData?: any
  ): void {

      listener.onOrderEvent({
        type: eventType,

3. Execution Management System (EMS)

The Execution Management System optimizes order execution by intelligently routing orders to venues and employing sophisticated algorithms to minimize market impact and transaction costs. EMS strategies include TWAP (Time-Weighted Average Price), VWAP (Volume-Weighted Average Price), and implementation shortfall algorithms.

VWAP Algorithm Implementation

TypeScript: VWAP Execution Algorithm
interface VWAPConfig {
  symbol: string;
  totalQuantity: number;
  side: 'BUY' | 'SELL';
  startTime: Date;
  endTime: Date;
  participationRate: number;  // 0.0 to 1.0
  maxSliceSize: number;
}

class VWAPExecutionAlgorithm {
  private config: VWAPConfig;
  private executedQuantity: number = 0;
  private volumeProfile: number[] = [];
  private scheduledSlices: OrderSlice[] = [];

  constructor(config: VWAPConfig) {
    this.config = config;
    this.calculateVolumeProfile();
    this.scheduleOrderSlices();
  }

  private async calculateVolumeProfile(): Promise {
    // Fetch historical intraday volume data
    const historicalVolume = await this.fetchHistoricalVolume(
      this.config.symbol,
      this.config.startTime,
      this.config.endTime
    );

    // Calculate typical intraday volume distribution
    const totalVolume = historicalVolume.reduce((a, b) => a + b, 0);
    this.volumeProfile = historicalVolume.map(v => v / totalVolume);
  }

  private scheduleOrderSlices(): void {
    const duration = this.config.endTime.getTime() -
                    this.config.startTime.getTime();
    const intervals = this.volumeProfile.length;
    const intervalDuration = duration / intervals;

    let remainingQuantity = this.config.totalQuantity;

    for (let i = 0; i < intervals; i++) {
      const targetQuantity = Math.round(
        this.config.totalQuantity * this.volumeProfile[i]
      );

      const sliceQuantity = Math.min(
        targetQuantity,
        this.config.maxSliceSize,
        remainingQuantity
      );

      const executeTime = new Date(
        this.config.startTime.getTime() + (i * intervalDuration)
      );

      this.scheduledSlices.push({
        quantity: sliceQuantity,
        executeTime,
        executed: false
      });

      remainingQuantity -= sliceQuantity;
    }
  }

  async execute(
    oms: OrderManagementSystem,
    marketData: MarketDataService
  ): Promise {
    const fills: Fill[] = [];

    for (const slice of this.scheduledSlices) {
      // Wait until scheduled time
      await this.waitUntil(slice.executeTime);

      // Get current market conditions
      const quote = await marketData.getQuote(this.config.symbol);

      // Adjust quantity based on current volume
      const currentVolume = await marketData.getCurrentVolume(
        this.config.symbol
      );
      const adjustedQuantity = Math.min(
        slice.quantity,
        currentVolume * this.config.participationRate
      );

      if (adjustedQuantity > 0) {
        try {
          // Submit order slice
          const order = await oms.submitOrder({
            clientOrderId: `VWAP-${Date.now()}`,
            symbol: this.config.symbol,
            side: this.config.side,
            type: OrderType.LIMIT,
            quantity: adjustedQuantity,
            price: this.config.side === 'BUY' ? quote.ask : quote.bid,
            timeInForce: 'IOC',  // Immediate or Cancel
            accountId: 'ALGO_TRADING'
          });

          // Wait for fill
          const fill = await this.waitForFill(oms, order.orderId, 5000);

          if (fill) {
            fills.push(fill);
            this.executedQuantity += fill.quantity;
            slice.executed = true;
          }
        } catch (error) {
          console.error(`VWAP slice execution error:`, error);
        }
      }

      // Check if target quantity reached
      if (this.executedQuantity >= this.config.totalQuantity) {
        break;
      }
    }

    return this.calculateExecutionSummary(fills);
  }

  private async waitUntil(targetTime: Date): Promise {
    const delay = targetTime.getTime() - Date.now();
    if (delay > 0) {
      await new Promise(resolve => setTimeout(resolve, delay));
    }
  }

  private async waitForFill(
    oms: OrderManagementSystem,
    orderId: string,
    timeoutMs: number
  ): Promise {
    const startTime = Date.now();

    while (Date.now() - startTime < timeoutMs) {
      const order = oms.getOrder(orderId);

      if (order && (order.status === OrderStatus.FILLED ||
                    order.status === OrderStatus.PARTIALLY_FILLED)) {
        return {
          quantity: order.filledQuantity,
          price: order.averagePrice,
          timestamp: order.lastUpdateTime
        };
      }

      await new Promise(resolve => setTimeout(resolve, 100));
    }

    return null;
  }

  private async fetchHistoricalVolume(
    symbol: string,
    startTime: Date,
    endTime: Date
  ): Promise {
    // Implementation would fetch from market data service
    // Return typical intraday volume distribution
    return [
      0.05, 0.08, 0.10, 0.12, 0.15, 0.18, 0.20, 0.12
    ];  // Example: 8 intervals
  }

  private calculateExecutionSummary(fills: Fill[]): ExecutionSummary {
    const totalQuantity = fills.reduce((sum, f) => sum + f.quantity, 0);
    const averagePrice = fills.reduce(
      (sum, f) => sum + (f.price * f.quantity),
      0
    ) / totalQuantity;

    return {
      executedQuantity: totalQuantity,
      targetQuantity: this.config.totalQuantity,
      averagePrice,
      fills,
      fillRate: totalQuantity / this.config.totalQuantity,
      startTime: this.config.startTime,
      endTime: new Date()
    };
  }
}

4. Low-Latency Optimization

For high-frequency trading strategies, every microsecond matters. Latency optimization involves multiple layers from hardware selection to network topology to software design patterns.

Optimization Area Technique Latency Improvement Implementation Cost
Hardware FPGA-based trading systems 90-99% reduction Very High
Network Kernel bypass (DPDK, Solarflare) 50-80% reduction High
Co-location Exchange proximity hosting 95% reduction Medium-High
Software Lock-free data structures 30-50% reduction Medium
Software Memory-mapped I/O 20-40% reduction Low-Medium
Protocol Binary protocols vs FIX 40-60% reduction Medium

5. Infrastructure Design Patterns

High-Performance Architecture Principles

  • Event-Driven Design: Asynchronous processing with message queues for maximum throughput
  • Memory Locality: Keep hot data in CPU cache, minimize pointer chasing
  • Lock-Free Algorithms: Use atomic operations and compare-and-swap instead of locks
  • Zero-Copy Techniques: Avoid unnecessary data copying between network and application buffers
  • Mechanical Sympathy: Understand hardware behavior (cache lines, branch prediction, etc.)
  • Batching: Process multiple events together when possible to amortize overhead
  • Hot/Cold Path Separation: Optimize critical path, defer non-critical operations

6. Monitoring and Observability

Production trading systems require comprehensive monitoring to detect issues before they cause financial losses. Key metrics include order-to-execution latency, fill rates, slippage, rejection rates, and system resource utilization.

Summary

Key Takeaways

  • Layered Architecture: Separate concerns into distinct components (data, strategy, risk, execution) for maintainability and scalability
  • Latency Hierarchy: Different strategies require different latency budgets; design architecture accordingly
  • Fault Tolerance: Trading systems must handle failures gracefully with automatic failover and recovery
  • Audit Trail: Maintain complete records of all orders and executions for regulatory compliance
  • Testing Environment: Robust testing infrastructure including simulators and replay capabilities is essential

Reference Standards and Implementation Notes

This standard is designed to interoperate with established international frameworks. Implementations SHOULD treat the following as the single source of truth and use this PHASE document as the interoperability layer that maps existing data into a machine-readable, signed, and auditable shape.

  • OpenAPI Specification 3.1 — REST API definition
  • JSON Schema 2020-12 — schema validation
  • ISO/IEC 27001:2022 — information security management
  • ISO/IEC 27018:2019 — PII protection in public clouds
  • ISO/IEC 17065:2012 — product certification bodies
  • CycloneDX 1.5 / SPDX 2.3 — software bill of materials
  • Sigstore (DSSE envelope, Rekor transparency log) — signature anchoring
  • in-toto Attestation Framework 1.0 — build provenance
  • IETF RFC 9700 — OAuth 2.1
  • IETF RFC 9457 — Problem Details for HTTP APIs
  • IETF RFC 8615 — well-known URIs

Practical Implementation Checklist

  • Manifest integrity — every published manifest is signed via Sigstore DSSE. Tampering is detectable without trusting an intermediary. Updates are issued as new manifests; the prior manifest is preserved with a supersede marker.
  • Retention — keep signed evidence for at least seven years to satisfy ISO/IEC 17065:2012 audit expectations. Some regulators require longer retention; in that case the deployment policy MUST extend the window rather than relying on the defaults.
  • Versioning — follow Semantic Versioning 2.0.0. Major version bumps require at least a 90-day overlap with the prior major version on every WIA-published reference implementation.
  • Negative-result reporting — a skipped indicator without recorded justification is treated as a failed indicator at audit time.
  • Key rotation — manifest signing keys are rotated per the deployment policy. The previous key is preserved in a published key history so that auditors can verify historical manifests offline.

Reference Topologies

Implementations declare their operational topology in the manifest: single-tenant edge (one runtime per organization), multi-tenant gateway (shared runtime with header-based isolation), federated mesh (multiple runtimes peer to one another and publish manifests to a directory service), or air-gapped batch (signed evidence package transported out-of-band). Each topology ships with reference test vectors so that implementations can validate their deployment matches the canonical pattern.

Open Governance
