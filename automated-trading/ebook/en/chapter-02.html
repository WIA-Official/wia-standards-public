2. Trend Following Strategies

Trend following strategies attempt to identify and profit from sustained directional movements in asset prices. These strategies are based on the observation that markets often exhibit momentum, where rising prices tend to continue rising and falling prices tend to continue falling.

Moving Average Crossover

One of the simplest and most widely used trend following approaches involves two moving averages of different periods. When the shorter-period average crosses above the longer-period average, a buy signal is generated. When it crosses below, a sell signal is generated.

TypeScript: Moving Average Crossover Strategy
interface TrendFollowingStrategy {
  shortPeriod: number;
  longPeriod: number;
  calculateSignal(prices: number[]): 'BUY' | 'SELL' | 'HOLD';
}

class MovingAverageCrossover implements TrendFollowingStrategy {
  constructor(
    public shortPeriod: number = 50,
    public longPeriod: number = 200
  ) {}

  private calculateSMA(prices: number[], period: number): number {
    const slice = prices.slice(-period);
    return slice.reduce((a, b) => a + b, 0) / period;
  }

  calculateSignal(prices: number[]): 'BUY' | 'SELL' | 'HOLD' {
    if (prices.length < this.longPeriod) {
      return 'HOLD';
    }

    const shortMA = this.calculateSMA(prices, this.shortPeriod);
    const longMA = this.calculateSMA(prices, this.longPeriod);
    const prevShortMA = this.calculateSMA(
      prices.slice(0, -1),
      this.shortPeriod
    );
    const prevLongMA = this.calculateSMA(
      prices.slice(0, -1),
      this.longPeriod
    );

    // Golden cross: short MA crosses above long MA
    if (prevShortMA <= prevLongMA && shortMA > longMA) {
      return 'BUY';
    }

    // Death cross: short MA crosses below long MA
    if (prevShortMA >= prevLongMA && shortMA < longMA) {
      return 'SELL';
    }

    return 'HOLD';
  }

  backtest(
    prices: number[],
    initialCapital: number = 10000
  ): BacktestResult {
    let capital = initialCapital;
    let position = 0;
    let trades: Trade[] = [];

    for (let i = this.longPeriod; i < prices.length; i++) {
      const signal = this.calculateSignal(prices.slice(0, i + 1));
      const price = prices[i];

      if (signal === 'BUY' && position === 0) {
        position = capital / price;
        capital = 0;
        trades.push({ type: 'BUY', price, quantity: position });
      } else if (signal === 'SELL' && position > 0) {
        capital = position * price;
        trades.push({ type: 'SELL', price, quantity: position });
        position = 0;
      }
    }

    const finalValue = position > 0
      ? position * prices[prices.length - 1]
      : capital;
    const totalReturn = (finalValue - initialCapital) / initialCapital;

    return { finalValue, totalReturn, trades };
  }
}

interface Trade {
  type: 'BUY' | 'SELL';
  price: number;
  quantity: number;
}

interface BacktestResult {
  finalValue: number;
  totalReturn: number;
  trades: Trade[];
}

Breakout Strategies

Breakout strategies identify when price moves beyond established support or resistance levels, interpreting this as the beginning of a new trend. Donchian channels, which track the highest high and lowest low over a specified period, provide a systematic framework for breakout trading.

3. Mean Reversion Strategies

Mean reversion strategies are based on the principle that asset prices tend to revert to their historical average over time. When prices deviate significantly from the mean, these strategies take positions expecting a return to equilibrium.

TypeScript: Bollinger Bands Mean Reversion
interface BollingerBandsConfig {
  period: number;
  standardDeviations: number;
}

class BollingerBandsMeanReversion {
  constructor(
    private config: BollingerBandsConfig = {
      period: 20,
      standardDeviations: 2
    }
  ) {}

  private calculateStandardDeviation(
    prices: number[],
    mean: number
  ): number {
    const squaredDiffs = prices.map(p => Math.pow(p - mean, 2));
    const variance = squaredDiffs.reduce((a, b) => a + b) / prices.length;
    return Math.sqrt(variance);
  }

  private getBands(prices: number[]): {
    upper: number;
    middle: number;
    lower: number;
  } {
    const slice = prices.slice(-this.config.period);
    const middle = slice.reduce((a, b) => a + b) / slice.length;
    const stdDev = this.calculateStandardDeviation(slice, middle);

    return {
      upper: middle + (stdDev * this.config.standardDeviations),
      middle,
      lower: middle - (stdDev * this.config.standardDeviations)
    };
  }

  generateSignal(prices: number[]): TradingSignal {
    if (prices.length < this.config.period) {
      return { action: 'HOLD', confidence: 0 };
    }

    const currentPrice = prices[prices.length - 1];
    const bands = this.getBands(prices);

    // Price below lower band: oversold, buy signal
    if (currentPrice < bands.lower) {
      const deviation = (bands.lower - currentPrice) / bands.middle;
      return {
        action: 'BUY',
        confidence: Math.min(deviation * 10, 1),
        entry: currentPrice,
        target: bands.middle,
        stopLoss: currentPrice * 0.98
      };
    }

    // Price above upper band: overbought, sell signal
    if (currentPrice > bands.upper) {
      const deviation = (currentPrice - bands.upper) / bands.middle;
      return {
        action: 'SELL',
        confidence: Math.min(deviation * 10, 1),
        entry: currentPrice,
        target: bands.middle,
        stopLoss: currentPrice * 1.02
      };
    }

    return { action: 'HOLD', confidence: 0 };
  }
}

interface TradingSignal {
  action: 'BUY' | 'SELL' | 'HOLD';
  confidence: number;
  entry?: number;
  target?: number;
  stopLoss?: number;
}

4. Arbitrage Strategies

Arbitrage strategies exploit price discrepancies between related instruments or markets. These strategies aim to generate risk-free profits by simultaneously buying and selling related securities.

Arbitrage Type Description Typical Duration Risk Level
Triangular Arbitrage Exploits currency exchange rate discrepancies Milliseconds Very Low
Statistical Arbitrage Pairs trading based on statistical relationships Hours to Days Low to Medium
Index Arbitrage Exploits mispricing between index and components Minutes Low
Merger Arbitrage Trades on M&A deal spreads Weeks to Months Medium
Convertible Arbitrage Long convertible bonds, short underlying stock Months Medium
TypeScript: Pairs Trading Strategy
interface PairsTradingConfig {
  lookbackPeriod: number;
  entryThreshold: number;  // Z-score threshold to enter
  exitThreshold: number;   // Z-score threshold to exit
  positionSize: number;
}

class PairsTradingStrategy {
  constructor(
    private config: PairsTradingConfig = {
      lookbackPeriod: 60,
      entryThreshold: 2.0,
      exitThreshold: 0.5,
      positionSize: 10000
    }
  ) {}

  private calculateSpread(
    pricesA: number[],
    pricesB: number[]
  ): number[] {
    const hedge = this.calculateHedgeRatio(pricesA, pricesB);
    return pricesA.map((pA, i) => pA - hedge * pricesB[i]);
  }

  private calculateHedgeRatio(
    pricesA: number[],
    pricesB: number[]
  ): number {
    // Simple linear regression to find hedge ratio
    const n = pricesA.length;
    const meanA = pricesA.reduce((a, b) => a + b) / n;
    const meanB = pricesB.reduce((a, b) => a + b) / n;

    let numerator = 0;
    let denominator = 0;

    for (let i = 0; i < n; i++) {
      numerator += (pricesA[i] - meanA) * (pricesB[i] - meanB);
      denominator += Math.pow(pricesB[i] - meanB, 2);
    }

    return numerator / denominator;
  }

  private calculateZScore(spread: number[]): number {
    const mean = spread.reduce((a, b) => a + b) / spread.length;
    const stdDev = Math.sqrt(
      spread.reduce((sum, val) => sum + Math.pow(val - mean, 2), 0) /
      spread.length
    );
    const currentSpread = spread[spread.length - 1];
    return (currentSpread - mean) / stdDev;
  }

  generateSignal(
    pricesA: number[],
    pricesB: number[]
  ): PairsSignal {
    if (pricesA.length < this.config.lookbackPeriod) {
      return { action: 'HOLD', zScore: 0 };
    }

    const recentA = pricesA.slice(-this.config.lookbackPeriod);
    const recentB = pricesB.slice(-this.config.lookbackPeriod);
    const spread = this.calculateSpread(recentA, recentB);
    const zScore = this.calculateZScore(spread);
    const hedgeRatio = this.calculateHedgeRatio(recentA, recentB);

    // Spread is too high: short A, long B
    if (zScore > this.config.entryThreshold) {
      return {
        action: 'SHORT_A_LONG_B',
        zScore,
        hedgeRatio,
        quantityA: this.config.positionSize / pricesA[pricesA.length - 1],
        quantityB: this.config.positionSize * hedgeRatio /
                   pricesB[pricesB.length - 1]
      };
    }

    // Spread is too low: long A, short B
    if (zScore < -this.config.entryThreshold) {
      return {
        action: 'LONG_A_SHORT_B',
        zScore,
        hedgeRatio,
        quantityA: this.config.positionSize / pricesA[pricesA.length - 1],
        quantityB: this.config.positionSize * hedgeRatio /
                   pricesB[pricesB.length - 1]
      };
    }

    // Spread returned to normal: exit position
    if (Math.abs(zScore) < this.config.exitThreshold) {
      return { action: 'CLOSE_POSITION', zScore };
    }

    return { action: 'HOLD', zScore };
  }
}

interface PairsSignal {
  action: 'LONG_A_SHORT_B' | 'SHORT_A_LONG_B' | 'CLOSE_POSITION' | 'HOLD';
  zScore: number;
  hedgeRatio?: number;
  quantityA?: number;
  quantityB?: number;
}

5. Market Making Strategies

Market makers provide liquidity by simultaneously posting buy and sell orders, profiting from the bid-ask spread. These strategies require sophisticated inventory management and real-time risk controls.

Market Making Challenges

  • Adverse Selection: Risk of trading with informed counterparties
  • Inventory Risk: Managing position accumulation and directional exposure
  • Competition: Multiple market makers competing for the same spreads
  • Latency: Need for ultra-low latency to adjust quotes quickly
  • Regulatory Requirements: Obligations to maintain quotes and provide liquidity
TypeScript: Basic Market Making Strategy
interface MarketMakingConfig {
  spreadBps: number;           // Spread in basis points
  maxInventory: number;        // Maximum position size
  inventorySkew: number;       // Adjustment factor for inventory
  minOrderSize: number;
  quoteTTL: number;           // Quote time-to-live in seconds
}

class MarketMakingStrategy {
  private inventory: number = 0;

  constructor(
    private config: MarketMakingConfig = {
      spreadBps: 10,
      maxInventory: 1000,
      inventorySkew: 0.0002,
      minOrderSize: 10,
      quoteTTL: 1
    }
  ) {}

  generateQuotes(
    midPrice: number,
    volatility: number,
    orderBook: OrderBook
  ): { bid: Quote; ask: Quote } {
    const baseSpread = (this.config.spreadBps / 10000) * midPrice;

    // Adjust spread based on volatility
    const volAdjustment = volatility * midPrice * 2;
    const adjustedSpread = baseSpread + volAdjustment;

    // Skew quotes based on inventory
    const inventoryRatio = this.inventory / this.config.maxInventory;
    const skew = inventoryRatio * this.config.inventorySkew * midPrice;

    // Calculate bid and ask prices
    let bidPrice = midPrice - adjustedSpread / 2 + skew;
    let askPrice = midPrice + adjustedSpread / 2 + skew;

    // Adjust for order book imbalance
    const imbalance = this.calculateOrderBookImbalance(orderBook);
    const imbalanceAdjustment = imbalance * midPrice * 0.0001;

    bidPrice -= imbalanceAdjustment;
    askPrice -= imbalanceAdjustment;

    // Calculate quantities
    const bidQuantity = Math.max(
      this.config.minOrderSize,
      this.config.maxInventory - this.inventory
    );
    const askQuantity = Math.max(
      this.config.minOrderSize,
      this.config.maxInventory + this.inventory
    );

    return {
      bid: {
        price: this.roundPrice(bidPrice),
        quantity: bidQuantity,
        side: 'BUY',
        ttl: this.config.quoteTTL
      },
      ask: {
        price: this.roundPrice(askPrice),
        quantity: askQuantity,
        side: 'SELL',
        ttl: this.config.quoteTTL
      }
    };
  }

  private calculateOrderBookImbalance(orderBook: OrderBook): number {
    const bidVolume = orderBook.bids
      .slice(0, 5)
      .reduce((sum, level) => sum + level.quantity, 0);
    const askVolume = orderBook.asks
      .slice(0, 5)
      .reduce((sum, level) => sum + level.quantity, 0);

    return (bidVolume - askVolume) / (bidVolume + askVolume);
  }

  onFill(side: 'BUY' | 'SELL', quantity: number): void {
    if (side === 'BUY') {
      this.inventory += quantity;
    } else {
      this.inventory -= quantity;
    }
  }

  private roundPrice(price: number): number {
    return Math.round(price * 100) / 100;
  }

  getInventoryStatus(): InventoryStatus {
    const utilizationPct = Math.abs(this.inventory) /
                          this.config.maxInventory * 100;

    return {
      currentInventory: this.inventory,
      maxInventory: this.config.maxInventory,
      utilizationPct,
      riskLevel: utilizationPct > 80 ? 'HIGH' :
                utilizationPct > 50 ? 'MEDIUM' : 'LOW'
    };
  }
}

interface Quote {
  price: number;
  quantity: number;
  side: 'BUY' | 'SELL';
  ttl: number;
}

interface OrderBook {
  bids: Array<{ price: number; quantity: number }>;
  asks: Array<{ price: number; quantity: number }>;
}

interface InventoryStatus {
  currentInventory: number;
  maxInventory: number;
  utilizationPct: number;
  riskLevel: 'LOW' | 'MEDIUM' | 'HIGH';
}

6. Momentum Strategies

Momentum strategies exploit the tendency of assets that have performed well (or poorly) in the recent past to continue performing well (or poorly) in the near future. These strategies can be applied across timeframes from intraday to multi-month periods.

Strategy Selection Framework

Selection Criteria Trend Following Mean Reversion Arbitrage Market Making
Market Conditions Strong trends, momentum Range-bound, cyclical Inefficient pricing Liquid, tight spreads
Capital Requirements $50K - $500K $50K - $500K $1M+ $1M+
Infrastructure Basic Moderate Advanced Ultra-low latency
Win Rate 30-40% 50-60% 70-90% 55-65%
Risk/Reward Asymmetric (large wins) Symmetric Low risk/low reward Consistent small gains

Summary

Key Takeaways

  • Strategy Diversity: Different strategies perform better in different market conditions; diversification across strategies reduces overall risk
  • Implementation Matters: The same strategy concept can have vastly different results based on implementation details and parameter choices
  • Risk-Adjusted Returns: Focus on Sharpe ratios and risk-adjusted metrics rather than absolute returns
  • Market Evolution: Strategies that worked historically may become crowded or obsolete; continuous research and adaptation is essential
  • Transaction Costs: High-frequency strategies are particularly sensitive to trading costs; always account for slippage, commissions, and market impact

Practical Implementation Checklist

  • Manifest integrity — every published manifest is signed via Sigstore DSSE. Tampering is detectable without trusting an intermediary. Updates are issued as new manifests; the prior manifest is preserved with a supersede marker.
  • Retention — keep signed evidence for at least seven years to satisfy ISO/IEC 17065:2012 audit expectations. Some regulators require longer retention; in that case the deployment policy MUST extend the window rather than relying on the defaults.
  • Versioning — follow Semantic Versioning 2.0.0. Major version bumps require at least a 90-day overlap with the prior major version on every WIA-published reference implementation.
  • Negative-result reporting — a skipped indicator without recorded justification is treated as a failed indicator at audit time.
  • Key rotation — manifest signing keys are rotated per the deployment policy. The previous key is preserved in a published key history so that auditors can verify historical manifests offline.

Reference Topologies

Implementations declare their operational topology in the manifest: single-tenant edge (one runtime per organization), multi-tenant gateway (shared runtime with header-based isolation), federated mesh (multiple runtimes peer to one another and publish manifests to a directory service), or air-gapped batch (signed evidence package transported out-of-band). Each topology ships with reference test vectors so that implementations can validate their deployment matches the canonical pattern.
