5.7 Endpoint catalogue

Phase 2 fixes endpoints in three groups. Device endpoints cover commissioning, listing, control, and decommissioning. Group and scenario endpoints cover bulk control and saved scenarios. Subscription endpoints cover the normal event WebSocket and the dedicated emergency-channel WebSocket. Every mutating call accepts an Idempotency-Key header for safe retry.

5.8 The emergency channel contract

The emergency channel exists because in real households the normal event stream is too noisy. A typical instrumented home produces tens of events per second under busy conditions; a fall alert sandwiched between thirty thermostat updates is a fall alert that gets missed. Phase 2 specifies that emergency events are emitted on a separate WebSocket connection, are duplicated on the normal stream for audit completeness, and require an explicit ack from at least one subscribed client within thirty seconds. Failure to ack within the window triggers automatic escalation to the resident's external contacts (SMS, voice call) and to emergency services if the household has opted in.

5.9 Worked control sequence — switch-input scanning

A user with single-switch input scans through devices in their living room. The mobile app issues a sequence of preview commands as the scanner advances and a final commit when the user activates the switch. The sequence: 

POST /devices/living_room_light/preview {"action":"highlight"}
POST /devices/living_room_speaker/preview {"action":"highlight"}
POST /devices/living_room_blinds/preview {"action":"highlight"}
POST /devices/living_room_blinds/control {"action":"open","percent":75}

The hub treats preview as a non-state-changing visual / haptic cue (a brief LED flash, a 100 ms vibration on a paired wearable) so the user can confirm the scan position before committing.

5.10 Operational telemetry expectations

A conformant hub exposes Prometheus-style metrics on a separate port (default 9091, distinct from the public service port). Metrics include device-online counts, normal-event throughput, emergency events emitted by category, alert-acknowledgement latency histograms, idempotency-key cache hit rates, and downgrade events emitted by the WIA-AIR-SHIELD scoring layer. Telemetry must not carry resident identifiers or device-serial values as label values.
