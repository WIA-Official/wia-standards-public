4.7 다섯 envelope 가족, 상세히

Phase 1 은 다섯 envelope 가족을 정의합니다. Device envelope 는 commissioning 된 디바이스를 기술합니다. 벤더, 모델, 펌웨어, 지원 cluster, 접근성 확장 capability, PASE commissioning 시 바인딩된 서명 키. Command envelope 는 한 디바이스 또는 한 그룹을 대상으로 하는 단일 지시를 운반합니다. 발급 principal(거주자, 돌봄인, 시나리오 엔진)이 서명하고 idempotency key 를 운반합니다. Event envelope 는 허브 또는 디바이스가 관측한 상태 변화를 기술합니다. 디바이스 서명을 운반하므로 감사자는 "디바이스가 상태 X 를 보고했다"와 "허브가 디바이스가 상태 X 를 보고했다고 주장했다"를 구별할 수 있습니다. Emergency envelope 는 비상 채널로 플래그된 이벤트로, 추가 severity 필드와 escalation 체인을 운반합니다. Profile envelope 는 거주자의 접근성 프로필(선호 입력 모달리티, 알림 선호도, 스캐닝 타이밍)을 보유하며, 거주자를 위해 디바이스 동작을 사용자화하는 어댑터의 게이팅 레코드입니다.

4.9 Round-trip 정규화

서명된 envelope 는 conformant 중간자를 통한 encode/decode 를 살아남아야 합니다. Phase 1 은 RFC 8785 JSON Canonicalisation Scheme 을 정규형으로 명시합니다. 키는 코드 포인트로 정렬, 무의미한 공백 없음, 숫자는 최단 round-trip 형식. 수신자는 서명 검증 전에 재정규화합니다. 편차는 깨진 중간자를 의미하므로 수신자는 추측이 아닌 envelope 거부를 선택해야 합니다.

4.10 envelope 의 버전 관리

각 envelope 는 wia_smarthome_version을 운반합니다. 1.x 라인 내 에서 추가 필드는 허용되며 consumer 는 알 수 없는 필드를 무시해야 합니다. 제거된 필드는 2.x 메이저 bump 와 문서화된 deprecation 윈도우를 요구합니다. 1.x 허브에 commissioning 된 디바이스는 문서화된 호환성 envelope 변환기를 통해 2.x 허브 하에서 계속 작동합니다. 변환기 자체는 서명되며 WIA 표준 저장소에 의해 게시됩니다.
