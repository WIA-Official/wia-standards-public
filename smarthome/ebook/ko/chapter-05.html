5.7 엔드포인트 카탈로그

Phase 2 는 엔드포인트를 세 그룹으로 고정합니다. 디바이스 엔드포인트 는 commissioning, 목록, 제어, decommissioning 을 다룹니다. 그룹 및 시나리오 엔드포인트는 일괄 제어와 저장된 시나리오를 다룹니다. 구독 엔드포인트는 일반 이벤트 WebSocket 과 전용 비상 채널 WebSocket 을 다룹니다. 모든 mutating 호출은 안전한 재시도를 위해 Idempotency-Key 헤더를 받아들입니다.

5.8 비상 채널 계약

실제 가구에서 일반 이벤트 스트림이 너무 시끄러워서 비상 채널이 존재합니다. 전형적인 계기화된 가구는 바쁜 조건에서 초당 수십 개의 이벤트를 생성합니다. 30 개의 온도 조절기 업데이트 사이에 끼인 낙상 알림은 놓치는 낙상 알림입니다. Phase 2 는 비상 이벤트가 별도 WebSocket 연결로 송출되고, 감사 완전성을 위해 일반 스트림에서 중복되며, 30 초 이내 적어도 하나의 구독 클라이언트로부터 명시적 ack 를 요구한다고 명시합니다. 윈도우 내 ack 실패는 거주자의 외부 연락처(SMS, 음성 통화)와 — 가구가 옵트인한 경우 — 응급 서비스로의 자동 escalation 을 트리거합니다.

5.9 워크된 제어 시퀀스 — 스위치 입력 스캐닝

단일 스위치 입력 사용자가 거실 디바이스를 스캔합니다. 모바일 앱은 스캐너가 진행되면서 미리보기 명령 시퀀스를 발행하고, 사용자가 스위치를 활성화하면 최종 commit 을 발행합니다. 

POST /devices/living_room_light/preview {"action":"highlight"}
POST /devices/living_room_speaker/preview {"action":"highlight"}
POST /devices/living_room_blinds/preview {"action":"highlight"}
POST /devices/living_room_blinds/control {"action":"open","percent":75}

허브는 preview를 비상태 변경의 시각/햅틱 단서(짧은 LED 깜빡임, 페어링된 wearable 의 100 ms 진동)로 처리하므로 사용자는 commit 전에 스캔 위치를 확인할 수 있습니다.

5.10 운영 텔레메트리 기대치

conformant 허브는 별도 포트(기본 9091, 공개 서비스 포트와 구별)에 Prometheus 스타일 메트릭을 노출합니다. 메트릭은 디바이스 온라인 카운트, 일반 이벤트 처리량, 카테고리별 송출된 비상 이벤트, 알림 ack 지연 히스토그램, idempotency key 캐시 히트 비율, 그리고 WIA-AIR-SHIELD 점수화 레이어가 송출한 강등 이벤트 를 포함합니다. 텔레메트리는 거주자 식별자나 디바이스 시리얼 값을 라벨 값으로 운반하지 않아야 합니다.
