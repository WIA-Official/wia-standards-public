6.7 허브 관점의 프로토콜 스택

허브는 여러 남행(southbound) 프로토콜(Wi-Fi 위 Matter, Thread 위 Matter, Z-Wave 800, Zigbee 3.0, wearable 을 위한 Bluetooth LE)과 단일 북행(northbound) WIA 프로토콜 표면을 말합니다. 변환 레이어가 Phase 3 의 핵심입니다. 각 남행 디바이스의 native cluster 어휘를 표준 WIA envelope shape 으로 변환하고, 들어오는 WIA 제어 envelope 를 디바이스가 실제로 말하는 남행 프로토콜로 변환합니다.

6.8 다중 fabric 협업, 상세히

가구는 흔히 적어도 하나의 벤더 fabric(Apple Home, Google Home, SmartThings)과 WIA fabric 을 가집니다. Phase 3 은 WIA fabric 이 경쟁이 아닌 공존하는 방식을 명시합니다. 두 fabric 에 commissioning 된 디바이스는 양쪽에 이벤트를 송출합니다. 양쪽으로부터의 제어 명령은 디바이스에서 50 ms 충돌 윈도우로 큐잉됩니다. 윈도우 내에서 가장 최근 commissioning 된 fabric 의 명령이 우선합니다. 윈도우 밖에서는 각 fabric 의 명령이 FIFO 입니다. 이는 같은 초에 두 음성 어시스턴트가 모순되는 지시를 내리는 실패 모드를 회피합니다.

6.9 오프라인 작동

Phase 3 은 인터넷 없이도 가구의 핵심 기능이 작동할 것을 요구합니다. 로컬 LAN 디바이스 목록 및 제어, 그룹 명령, 허브에 모델이 설치된 경우 로컬 NLU 음성 명령, LAN 연결된 모바일 클라이언트로의 비상 알림. 클라우드 의존 기능(원격 접근, 로컬 NLU 없는 음성 어시스턴트, 제3자 시나리오 공유)은 우아하게 강등되고 재연결 시 재개됩니다. conformant 허브는 거주자의 접근성 프로필을 로컬 캐시해 클라우드 장애가 거주자의 스위치 입력 또는 알림 선호도가 잘못 적용되는 상태로 거주자를 고립시키지 않도록 해야 합니다.

6.10 재생 방어

모든 서명된 envelope 는 96 비트 nonce 와 RFC 3339 timestamp 를 운반합니다. 수신자는 ±300 초 초과의 skew 를 가진 envelope 를 거부하고, 지난 600 초 내에 보인 (signer, nonce) tuple 의 envelope 를 거부하며, seen-nonce 캐시를 허브 재시작 사이에 영속시켜 전원 사이클이 이전에 차단된 재생을 위한 윈도우를 다시 열지 않도록 합니다.

6.11 신뢰 목록 유지 관리

각 허브는 연합 peer (벤더 클라우드, 가구 간 게스트 허브, 돌봄 에이전트)의 서명된 신뢰 목록을 유지합니다. 신뢰 목록은 적어도 매월 재게시됩니다. peer 는 60 일보다 오래된 stale 목록을 거부합니다. peer 는 목록 새로고침 윈도우 사이에 즉시 신뢰를 끊기 위해 revocation envelope 를 자체 게시할 수 있습니다. 신뢰 목록 자체는 envelope 이며 Phase 1 정규화 규칙을 따릅니다.
