📚 제2장: 교육 접근성의 현재 과제

"모든 사람을 위해 교육을 접근 가능하게 만드는 기술은 존재합니다. 문제는 어떻게 함께 협력할 것인지에 대해 합의하지 않았다는 것입니다."

2.1 파편화된 환경

교육 기술 산업은 지난 20년간 빠르게 성장했으며, 수천 개의 학습 관리 시스템(LMS), 콘텐츠 플랫폼 및 평가 도구가 시장에 진입했습니다. 이러한 성장이 혁신을 가져왔지만, 동시에 깊이 파편화된 접근성 환경을 만들었습니다.

2.1.1 LMS 시장 파편화

플랫폼 시장 점유율 접근성 접근 방식 프로필 이식성
Canvas 약 35% WCAG 2.1 AA 준수 독점적
Moodle 약 22% WCAG 2.1 AA 목표 독점 플러그인
Blackboard 약 18% WCAG 2.1 AA 준수 독점적
D2L Brightspace 약 10% WCAG 2.1 AA 준수 독점적

핵심 문제: 각 플랫폼은 접근성을 다르게 구현합니다. Canvas에서 설정한 학습자의 선호도는 Moodle로 전송되지 않습니다. Blackboard에 문서화된 편의제공은 D2L에서 재구성해야 합니다.

2.2 기술적 장벽

2.2.1 일관되지 않은 API 구현

// 현재 상태: 각 플랫폼이 접근성을 다르게 정의

// 결과: 플랫폼 간 상호 운용성 없음

2.3 행정적 부담

2.3.1 편의제공 요청 프로세스

일반적인 편의제공 일정:

1주차: 학생이 문서 제출
  └── 의료 문서
  └── 이전 IEP/504 계획
  └── 자가 식별 양식

2-3주차: 검토 및 확인
  └── 장애 서비스에서 문서 검토
  └── 추가 정보 요청 가능
  └── 위원회가 편의제공 결정

4주차: 강사에게 통보
  └── 각 강사에게 개별 서신 발송
  └── 학생이 각 강사가 서신을 받았는지 확인

매 학기: 이 과정의 상당 부분 반복

위기 지점: 많은 장애 서비스 사무실이 인력 부족과 업무 과중 상태입니다. 일반적인 사무실은 3-5명의 직원으로 1,000명 이상의 학생에게 서비스를 제공하며, 학기당 수백 건의 편의제공 요청을 처리합니다.

2.6 WIA EDU의 필요성

이 장에서 설명한 과제들은 교육 접근성에 대한 통합 접근 방식의 시급한 필요성을 보여줍니다. WIA EDU 표준은 다음을 통해 이러한 문제를 해결합니다:

2.6.1 기술 솔루션

  • 통합 스키마: 학습자 프로필을 위한 하나의 표준 형식
  • API 표준화: 플랫폼 간 일관된 엔드포인트
  • AT 통합: 보조 기술을 위한 표준 인터페이스

2.6.2 행정 솔루션

  • 휴대용 프로필: 기관 간 편의제공 전송
  • 자동 매칭: 학습자 요구를 과정 기능에 매칭
  • 간소화된 워크플로우: 수동 프로세스 감소

2.7 장 요약

핵심 내용:

  1. 파편화: 독점적 접근성 구현을 가진 여러 LMS 플랫폼
  2. 기술 장벽: 일관되지 않은 API, 불량한 AT 통합
  3. 행정 부담: 수동 프로세스가 장애 서비스를 압도
  4. 경제적 영향: 수십억 달러의 비용, 증가하는 법적 책임
  5. 인간적 영향: 낮은 유지율, 정신 건강 영향, 옹호 피로
