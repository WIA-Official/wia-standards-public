📚 제3장: WIA EDU 표준 개요

홍익인간 (弘益人間)

WIA EDU 표준은 플랫폼이나 장애 유형에 관계없이 원활한 학습 경험을 가능하게 하는 교육 접근성을 위한 보편적 프레임워크를 만듭니다.

3.1 미션과 목표

3.1.1 미션 선언문

WIA EDU 표준의 미션은 장애를 가진 학습자가 동료들과 동일한 용이성과 효과로 교육 콘텐츠와 플랫폼에 접근할 수 있도록 하는 개방적이고 상호 운용 가능한 교육 접근성 프레임워크를 구축하는 것입니다.

3.1.2 주요 목표

  1. 이식성: 기관과 플랫폼 간 이동하는 학습자 접근성 프로필
  2. 상호운용성: LMS 플랫폼과 보조 기술을 위한 표준화된 인터페이스
  3. 자동화: 지능형 매칭을 통한 수동 편의제공 프로세스 감소
  4. 개인정보 보호: 세분화된 공개 옵션을 갖춘 학습자 제어 데이터
  5. 규정 준수: WCAG, Section 508, AccessForAll 표준에 대한 내장 지원

3.2.1 스키마 관계

스키마 목적 주요 관계
학습자 프로필 학습자 접근성 선호도 및 요구 저장 WIA AAC, BCI, Eye Gaze 프로필에 연결

평가 평가 편의제공 정의 QTI 접근성 패턴, 시간 규칙

3.3 지원 장애 유형

WIA EDU 표준은 모든 주요 장애 범주에 대한 포괄적인 지원을 제공합니다.

3.3.1 시각 장애

상태 프로필 설정 자동 적응
실명 screen_reader: true, vision: "none" 대체 텍스트 필수, 화면 해설, 구조적 탐색
저시력 magnification: 2.0+, high_contrast: true 확장 가능 콘텐츠, 리플로우 지원, 대비 모드
색맹 colorblind_type: "deuteranopia" 색상 독립적 정보, 대체 색상 체계

3.3.2 청각 장애

상태 프로필 설정 자동 적응
청각 상실 hearing: "none", sign_language: "KSL" 자막 필수, 수어 영상, 시각적 알림
난청 hearing: "partial", captions: true 자막, 대본, 볼륨 컨트롤

3.4 설계 원칙

3.4.1 AccessForAll 정렬

WIA EDU 표준은 ISO/IEC 24751 AccessForAll 프레임워크를 구현합니다:

  • 개인 요구 및 선호도 (PNP): 학습자 프로필 스키마에 매핑
  • 디지털 리소스 설명 (DRD): 콘텐츠 메타데이터 스키마에 매핑
  • 접근성 매칭: API 계층에서 자동화

3.4.2 보편적 학습 설계 (UDL)

표준은 모든 세 가지 UDL 원칙을 지원합니다:

{
  "udl_options": {
    "engagement": {
      "interest_triggers": ["choice", "relevance", "authenticity"],
      "collaboration_preference": "small_groups"
    },
    "representation": {
      "preferred_modalities": ["visual", "auditory"],
      "background_knowledge_support": true
    },
    "action_expression": {
      "expression_preferences": ["multimedia", "verbal"],
      "planning_support": true
    }
  }
}

3.5 개발 단계

Phase 1: 데이터 형식 표준

학습자 프로필, 과정, 콘텐츠 및 평가를 위한 JSON 스키마. 데이터 기반 구축.

Phase 2: API 인터페이스 표준

CRUD 작업, 프로필 매칭 및 LTI 통합을 위한 RESTful API 설계.

Phase 3: 통신 프로토콜

LTI 1.3 확장, OAuth 2.0 흐름 및 실시간 WebSocket 업데이트.

Phase 4: 생태계 통합

WIA AAC, BCI, Eye Gaze 및 기타 보조 기술 표준과의 통합.

3.6 장 요약

핵심 내용:

  1. 미션: 개방적이고 상호 운용 가능한 교육 접근성 프레임워크
  2. 아키텍처: 4-스키마 설계 (학습자, 과정, 콘텐츠, 평가)
  3. 범위: 포괄적인 편의제공을 갖춘 모든 장애 유형
  4. 원칙: AccessForAll, UDL 및 WCAG 정렬
  5. 단계: 데이터 형식 → API → 프로토콜 → 통합
