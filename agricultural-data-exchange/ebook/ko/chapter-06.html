6장: IoT 및 센서 네트워크

농업 사물인터넷(IoT) 시스템은 정밀 농업과 실시간 의사결정 지원을 가능하게 하는 환경, 장비, 작물 데이터의 연속 스트림을 생성합니다. 이 장에서는 센서 유형, 연결 프로토콜, 엣지 컴퓨팅 아키텍처, 다양한 농업 환경에서 상호운용 가능한 농업 IoT 생태계를 가능하게 하는 데이터 표준을 다룹니다.

6.1 농업용 센서 유형

범주 센서 유형 매개변수 배치
토양 정전용량, TDR, 저항성 수분, 온도, EC, pH 지중 프로브
기상 온도계, 습도계, 풍속계 온도, 습도, 바람, 강수, 일사 기상관측소
작물 NDVI, 엽록소, 군락 온도 건강, 스트레스, 생육 단계 근접, 항공
수분 유량계, 수위 센서 유량, 압력, 수질 관개 시스템
장비 GPS, CAN 버스, 텔레매틱스 위치, 속도, 연료, 진단 기계 장착
가축 RFID, 가속도계, GPS ID, 활동, 위치, 건강 태그, 목걸이, 볼루스

6.2 센서 데이터 모델

// OGC SensorThings API 데이터 모델
interface SensorThingsDataModel {
  // Thing - 물리적 장치 또는 객체
  Thing: {
    id: string;
    name: string;
    description: string;
    properties: Record<string, any>;
    Locations: Location[];
    Datastreams: Datastream[];
  };

  // Location - 지리공간 위치
  Location: {
    id: string;
    name: string;
    description: string;
    encodingType: 'application/geo+json';
    location: GeoJSON.Point | GeoJSON.Polygon;
  };

  // Sensor - 측정 절차
  Sensor: {
    id: string;
    name: string;
    description: string;
    encodingType: 'application/pdf' | 'http://www.opengis.net/doc/IS/SensorML/2.0';
    metadata: string;            // 센서 설명 링크
  };

  // ObservedProperty - 측정 대상
  ObservedProperty: {
    id: string;
    name: string;
    definition: string;          // 어휘에 대한 URI
    description: string;
  };

  // Datastream - 관측 시리즈
  Datastream: {
    id: string;
    name: string;
    description: string;
    observationType: string;
    unitOfMeasurement: {
      name: string;
      symbol: string;
      definition: string;
    };
    observedArea?: GeoJSON.Polygon;
    phenomenonTime?: string;     // 시간 범위
    resultTime?: string;
    Thing: Thing;
    Sensor: Sensor;
    ObservedProperty: ObservedProperty;
  };

  // Observation - 개별 측정
  Observation: {
    id: string;
    phenomenonTime: ISO8601DateTime;
    resultTime: ISO8601DateTime;
    result: number | object;
    resultQuality?: QualityElement[];
    validTime?: string;
    parameters?: Record<string, any>;
    Datastream: Datastream;
    FeatureOfInterest: FeatureOfInterest;
  };
}

6.4 농업 IoT용 MQTT

MQTT 프로토콜 특성

  • 경량: 제한된 장치를 위한 최소 오버헤드
  • 발행/구독: 생산자와 소비자의 분리
  • QoS 레벨: 전달 보장 (0, 1, 2)
  • 보존 메시지: 최신 값 항상 사용 가능
  • 유언: 예기치 않은 연결 끊김 시 알림
// 농업용 MQTT 토픽 구조
const mqttTopicStructure = {
  // 계층적 토픽 설계
  pattern: 'wia/{farmId}/{fieldId}/{deviceType}/{deviceId}/{measurement}',

  examples: [
    'wia/farm123/field456/soil_sensor/ss001/moisture',
    'wia/farm123/field456/weather_station/ws001/temperature',
    'wia/farm123/+/irrigation/+/flow_rate',  // 와일드카드
    'wia/farm123/#'  // 모든 농장 데이터
  ],

  // 메시지 페이로드 (JSON)
  payload: {
    timestamp: '2025-06-15T10:30:00Z',
    value: 0.32,
    unit: 'm3/m3',
    quality: 'good',
    location: { lat: 37.123, lng: 127.234 }
  },

  // 산업 IoT용 Sparkplug B 사양
  sparkplugB: {
    namespace: 'spBv1.0',
    groupId: 'farm123',
    edgeNodeId: 'gateway001',
    deviceId: 'soil_sensor_001',
    messageTypes: ['NBIRTH', 'NDATA', 'NDEATH', 'DBIRTH', 'DDATA', 'DDEATH']
  }
};

6.7 시계열 데이터 관리

시계열 데이터베이스 요구사항

농업 IoT는 높은 쓰기 처리량, 효율적인 압축, 빠른 시간 범위 쿼리에 최적화된 특수 저장 솔루션이 필요한 대량의 타임스탬프 데이터를 생성합니다.

// 시계열 데이터 아키텍처
