Chapter 3

API Design and Implementation

This chapter explores the RESTful API architecture of WIA-AGRI-020, covering authentication, endpoints, request/response patterns, error handling, and best practices for building robust agricultural data APIs.

RESTful API Architecture

The WIA-AGRI-020 API follows RESTful principles, making it intuitive for developers familiar with modern web services. The API is organized around resources (farms, sensors, data) and uses standard HTTP methods (GET, POST, PUT, DELETE) for operations.

Base URL Structure

All API requests are made to a base URL with versioning in the path:

Production: https://api.wia-agri.org/v1
Testing:    https://api-test.wia-agri.org/v1
Sandbox:    https://sandbox.wia-agri.org/v1

The version number (v1) in the URL ensures backward compatibility. When breaking changes are introduced, a new version (v2) is released while maintaining support for previous versions.

Authentication

API Key Authentication

The simplest authentication method uses API keys passed in request headers:

GET /api/v1/data/query HTTP/1.1
Host: api.wia-agri.org
X-API-Key: wia_live_abc123def456ghi789
Content-Type: application/json

API keys come in two types:

  • Test Keys: Prefix wia_test_ - Limited to sandbox environments, rate-limited, and work with sample data only.
  • Live Keys: Prefix wia_live_ - Full production access with standard rate limits based on your subscription tier.

OAuth 2.0 Flow

For third-party integrations and applications accessing data on behalf of farmers, OAuth 2.0 provides secure delegated access:

POST /oauth/token HTTP/1.1
Host: auth.wia-agri.org
Content-Type: application/x-www-form-urlencoded

grant_type=client_credentials&
client_id=YOUR_CLIENT_ID&
client_secret=YOUR_CLIENT_SECRET&
scope=data:read data:write

Response:

{
  "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "token_type": "Bearer",
  "expires_in": 3600,
  "scope": "data:read data:write"
}

Scopes and Permissions

Scope Description Typical Use
data:read Read sensor data and queries Analytics dashboards
data:write Submit new data IoT sensors, equipment
farm:manage Manage farm profiles and settings Farm management software
device:register Register and manage IoT devices Sensor installation apps
analytics:access Access advanced analytics AI/ML platforms

Core API Endpoints

Data Submission

Submit a single data record:

POST /api/v1/data/submit
Content-Type: application/json
X-API-Key: wia_live_abc123def456ghi789

{
  "standardVersion": "WIA-AGRI-020-v1.0",
  "farmId": "FARM-2025-001",
  "dataType": "SOIL_SENSOR",
  "timestamp": "2025-12-26T10:30:00.000Z",
  "readings": {
    "soilMoisture": {"value": 45.5},
    "soilPH": {"value": 6.8}
  }
}

Success Response (201 Created):

{
  "status": "success",
  "dataId": "DATA-20251226-001",
  "timestamp": "2025-12-26T10:30:00.000Z",
  "validationResult": {
    "isValid": true,
    "warnings": [],
    "errors": []
  },
  "receivedAt": "2025-12-26T10:30:01.234Z"
}

Batch Submission

Submit multiple records in one request for efficiency:

POST /api/v1/data/batch
Content-Type: application/json

{
  "batchId": "BATCH-20251226-001",
  "farmId": "FARM-2025-001",
  "data": [
    {"dataId": "SOIL-001", "dataType": "SOIL_SENSOR", "readings": {...}},
    {"dataId": "WEATHER-001", "dataType": "WEATHER", "current": {...}},
    {"dataId": "CROP-001", "dataType": "CROP_HEALTH", "healthMetrics": {...}}
  ]
}

The API processes all records atomically - either all succeed or all fail together, preventing partial data corruption.

Data Retrieval

Query historical data with flexible filtering:

GET /api/v1/data/query?farmId=FARM-2025-001&dataType=SOIL_SENSOR&from=2025-12-20&to=2025-12-26&limit=100

Response includes pagination for large datasets:

{

    "count": 45,

    "page": 1,

Get Latest Reading

Retrieve the most recent data point for quick status checks:

GET /api/v1/data/latest?farmId=FARM-2025-001&dataType=SOIL_SENSOR&fieldId=FIELD-NORTH-01

Rate Limiting

To ensure fair usage and system stability, the API implements rate limiting. Each response includes headers showing your current status:

HTTP/1.1 200 OK
X-RateLimit-Limit: 1000
X-RateLimit-Remaining: 999
X-RateLimit-Reset: 1703592000

Rate Limits by Tier

Tier Requests/Hour Burst Limit Concurrent Connections
Free 1,000 100 2
Basic 10,000 500 10
Pro 100,000 2,000 50
Enterprise Unlimited Unlimited Unlimited

When rate limits are exceeded, the API returns a 429 status code with retry information:

HTTP/1.1 429 Too Many Requests
Retry-After: 3600

{
  "status": "error",
  "code": "RATE_LIMIT_EXCEEDED",
  "message": "Rate limit exceeded. Try again in 3600 seconds.",
  "limit": 1000,
  "remaining": 0,
  "resetAt": "2025-12-26T11:00:00.000Z"
}

Error Handling

The API uses standard HTTP status codes and provides detailed error information in responses:

Error Response Format

{
  "status": "error",
  "code": "VALIDATION_ERROR",
  "message": "Data validation failed",
  "details": [
    {
      "field": "readings.soilMoisture.value",
      "error": "Value 150 exceeds maximum allowed (100)",
      "constraint": "range"
    }
  ],
  "requestId": "req_abc123",
  "timestamp": "2025-12-26T10:30:00.000Z"
}

Common Error Codes

Code HTTP Status Description Solution
VALIDATION_ERROR 400 Invalid data format Check field types and ranges
AUTHENTICATION_FAILED 401 Invalid API key Verify your API key is correct
AUTHORIZATION_FAILED 403 Insufficient permissions Request appropriate scopes
NOT_FOUND 404 Resource not found Check farmId, dataId exists
RATE_LIMIT_EXCEEDED 429 Too many requests Wait and retry, upgrade tier
INTERNAL_ERROR 500 Server error Contact support with requestId

Real-Time Data Streaming

WebSocket Connections

For real-time data, establish a WebSocket connection:

const ws = new WebSocket('wss://stream.wia-agri.org/v1/stream');

ws.onopen = function() {
  // Authenticate
  ws.send(JSON.stringify({
    action: 'authenticate',
    token: 'wia_live_abc123def456ghi789'
  }));

  // Subscribe to data streams
  ws.send(JSON.stringify({
    action: 'subscribe',
    farmId: 'FARM-2025-001',
    dataTypes: ['SOIL_SENSOR', 'WEATHER']
  }));
};

ws.onmessage = function(event) {
  const data = JSON.parse(event.data);
  console.log('Real-time data:', data);
};

Webhooks

Configure webhooks to receive notifications when events occur:

POST /api/v1/webhooks
Content-Type: application/json

{
  "url": "https://your-server.com/webhook",
  "events": ["data.submitted", "device.offline"],
  "farmId": "FARM-2025-001",
  "secret": "whsec_abc123xyz789"
}

When events occur, the API sends POST requests to your webhook URL with the event payload.

SDK Examples

JavaScript/Node.js

const WIAAgri = require('wia-agri-sdk');

const client = new WIAAgri({
  apiKey: 'wia_live_abc123def456ghi789'
});

// Submit data
await client.data.submit({
  farmId: 'FARM-2025-001',
  dataType: 'SOIL_SENSOR',
  readings: {
    soilMoisture: {value: 45.5}
  }
});

// Query data
const results = await client.data.query({
  farmId: 'FARM-2025-001',
  dataType: 'SOIL_SENSOR',
  from: '2025-12-20',
  to: '2025-12-26'
});

Python

from wia_agri import WIAAgriClient

client = WIAAgriClient(api_key='wia_live_abc123def456ghi789')

# Submit data
client.data.submit(
    farm_id='FARM-2025-001',
    data_type='SOIL_SENSOR',
    readings={'soilMoisture': {'value': 45.5}}
)

# Query data
results = client.data.query(
    farm_id='FARM-2025-001',
    data_type='SOIL_SENSOR',
    from_date='2025-12-20',
    to_date='2025-12-26'
)

🎯 Best Practices

  • Use batch endpoints for bulk operations to reduce API calls
  • Implement exponential backoff for retries after rate limit errors
  • Cache authentication tokens and reuse them until expiration
  • Always validate data client-side before API submission
  • Monitor rate limit headers to avoid hitting limits
  • Use webhooks instead of polling for event-driven updates
  • Include requestId in support tickets for faster debugging

Chapter Summary

The WIA-AGRI-020 API provides a robust, RESTful interface for agricultural data exchange. It supports multiple authentication methods, comprehensive error handling, rate limiting, real-time streaming via WebSocket, and webhook notifications. Official SDKs for major programming languages simplify integration and follow API best practices.
