Chapter 6: IoT and Sensor Networks

Agricultural Internet of Things (IoT) systems generate continuous streams of environmental, equipment, and crop data that enable precision agriculture and real-time decision support. This chapter covers sensor types, connectivity protocols, edge computing architectures, and the data standards that enable interoperable agricultural IoT ecosystems across diverse farming environments.

6.1 Agricultural Sensor Types

Category Sensor Types Parameters Deployment
Soil Capacitance, TDR, resistive Moisture, temperature, EC, pH In-ground probes
Weather Thermometer, hygrometer, anemometer Temp, humidity, wind, rain, solar Weather stations
Crop NDVI, chlorophyll, canopy temp Health, stress, growth stage Proximal, aerial
Water Flow meters, level sensors Flow rate, pressure, quality Irrigation systems
Equipment GPS, CAN bus, telematics Position, speed, fuel, diagnostics Mounted on machinery
Livestock RFID, accelerometer, GPS ID, activity, location, health Tags, collars, boluses

6.2 Sensor Data Model

// OGC SensorThings API Data Model
interface SensorThingsDataModel {
  // Thing - Physical device or object
  Thing: {
    id: string;
    name: string;
    description: string;
    properties: Record<string, any>;
    Locations: Location[];
    Datastreams: Datastream[];
  };

  // Location - Geospatial position
  Location: {
    id: string;
    name: string;
    description: string;
    encodingType: 'application/geo+json';
    location: GeoJSON.Point | GeoJSON.Polygon;
  };

  // Sensor - Measurement procedure
  Sensor: {
    id: string;
    name: string;
    description: string;
    encodingType: 'application/pdf' | 'http://www.opengis.net/doc/IS/SensorML/2.0';
    metadata: string;            // Link to sensor description
  };

  // ObservedProperty - What is being measured
  ObservedProperty: {
    id: string;
    name: string;
    definition: string;          // URI to vocabulary
    description: string;
  };

  // Datastream - Series of observations
  Datastream: {
    id: string;
    name: string;
    description: string;
    observationType: string;
    unitOfMeasurement: {
      name: string;
      symbol: string;
      definition: string;
    };
    observedArea?: GeoJSON.Polygon;
    phenomenonTime?: string;     // Time extent
    resultTime?: string;
    Thing: Thing;
    Sensor: Sensor;
    ObservedProperty: ObservedProperty;
  };

  // Observation - Individual measurement
  Observation: {
    id: string;
    phenomenonTime: ISO8601DateTime;
    resultTime: ISO8601DateTime;
    result: number | object;
    resultQuality?: QualityElement[];
    validTime?: string;
    parameters?: Record<string, any>;
    Datastream: Datastream;
    FeatureOfInterest: FeatureOfInterest;
  };
}

6.4 MQTT for Agricultural IoT

MQTT Protocol Characteristics

  • Lightweight: Minimal overhead for constrained devices
  • Publish/Subscribe: Decoupled producers and consumers
  • QoS Levels: Delivery guarantees (0, 1, 2)
  • Retained Messages: Latest value always available
  • Last Will: Notification on unexpected disconnect
// MQTT Topic Structure for Agriculture
const mqttTopicStructure = {
  // Hierarchical topic design
  pattern: 'wia/{farmId}/{fieldId}/{deviceType}/{deviceId}/{measurement}',

  examples: [
    'wia/farm123/field456/soil_sensor/ss001/moisture',
    'wia/farm123/field456/weather_station/ws001/temperature',
    'wia/farm123/+/irrigation/+/flow_rate',  // Wildcards
    'wia/farm123/#'  // All farm data
  ],

  // Message payload (JSON)
  payload: {
    timestamp: '2025-06-15T10:30:00Z',
    value: 0.32,
    unit: 'm3/m3',
    quality: 'good',
    location: { lat: 42.123, lng: -93.234 }
  },

  // Sparkplug B specification for industrial IoT
  sparkplugB: {
    namespace: 'spBv1.0',
    groupId: 'farm123',
    edgeNodeId: 'gateway001',
    deviceId: 'soil_sensor_001',
    messageTypes: ['NBIRTH', 'NDATA', 'NDEATH', 'DBIRTH', 'DDATA', 'DDEATH']

};

6.7 Time Series Data Management

Time Series Database Requirements

Agricultural IoT generates massive volumes of time-stamped data requiring specialized storage solutions optimized for high write throughput, efficient compression, and fast time-range queries.

// Time Series Data Architecture
interface TimeSeriesArchitecture {
  databases: {
    influxDB: {
      model: 'tags + fields + timestamp';
      query: 'InfluxQL / Flux';
      retention: 'configurable policies';
    };
    timescaleDB: {
      model: 'PostgreSQL extension';
      query: 'SQL';
      features: 'hypertables, continuous aggregates';
    };
    questDB: {
      model: 'column-oriented';
      query: 'SQL';
      performance: 'high ingestion rate';
    };
  };

  // Data Retention Policies
  retention: {
    raw: { duration: '7d', resolution: 'original' },
    hourly: { duration: '90d', aggregation: 'avg, min, max' },
    daily: { duration: '2y', aggregation: 'avg, min, max' },
    monthly: { duration: 'forever', aggregation: 'avg, min, max' }
  };

  // Downsampling Functions
  aggregations: [
    'mean', 'min', 'max', 'sum', 'count',
    'first', 'last', 'median', 'percentile',
    'spread', 'stddev'
  ];

  // Query Patterns

    latest: 'SELECT last(*) FROM soil_moisture WHERE device=$1',
