Chapter 2

Data Types and Formats

This chapter provides an in-depth exploration of the data types supported by WIA-AGRI-020, including detailed field specifications, validation rules, and practical examples for each category of agricultural data.

Understanding Agricultural Data Structure

Every piece of agricultural data in the WIA-AGRI-020 standard follows a consistent structure that includes metadata, measurements, and quality indicators. This uniformity makes it possible for any system to understand and process data regardless of its source.

The Universal Data Container

All data submissions share a common outer structure:

{
  "standardVersion": "WIA-AGRI-020-v1.0",
  "dataId": "unique-identifier",
  "timestamp": "ISO-8601-timestamp",
  "farmId": "farm-identifier",
  "location": { /* location data */ },
  "dataType": "SOIL_SENSOR|WEATHER|CROP_HEALTH|EQUIPMENT|YIELD",
  "source": { /* data source info */ },
  "readings": { /* actual measurements */ },
  "metadata": { /* quality and context */ }
}

Soil Data Fields Reference

Field Type Range Description
soilMoisture Number 0-100% Volumetric water content
soilTemperature Number -30 to 60°C Temperature at specified depth
soilPH Number 0-14 Acidity/alkalinity measurement
soilEC Number 0-20 dS/m Electrical conductivity (salinity)
nitrogen (N) Number 0-200 ppm Available nitrogen content
phosphorus (P) Number 0-100 ppm Available phosphorus content
potassium (K) Number 0-500 ppm Available potassium content

Data Validation Rules

The standard includes comprehensive validation rules to ensure data quality:

  • Required Fields: standardVersion, dataId, timestamp, farmId, dataType must always be present
  • Timestamp Format: ISO 8601 UTC format (e.g., "2025-12-26T10:30:00.000Z")
  • Numeric Ranges: All measurements must fall within physically possible ranges
  • Unit Consistency: Units must be explicitly specified and match the measurement type
  • Location Accuracy: GPS coordinates must be valid and within farm boundaries when known

🎯 Best Practices

  • Always include source information for traceability
  • Use consistent timezone (UTC recommended)
  • Include calibration dates for sensor data
  • Specify measurement depth for soil sensors
  • Add quality indicators to enable data filtering

Chapter Summary

WIA-AGRI-020 defines comprehensive data structures for soil sensors, weather data, crop health metrics, equipment telemetry, and yield information. Each data type has specific fields, validation rules, and quality indicators that ensure consistency and reliability across all agricultural data exchange.
