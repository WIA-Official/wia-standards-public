챕터 1: 작물 모니터링 소개

1.1 농업의 진화

농업은 세 가지 주요 혁명을 거쳤습니다. 제1차 농업혁명(기원전 10,000년)은 수렵채집에서 농경으로의 전환을 의미합니다. 녹색혁명(1960~1980년대)은 고수확 품종, 비료, 기계화를 도입하여 수십억 명을 먹여 살렸습니다. 이제 우리는 데이터, AI, IoT가 농업을 정밀한 최적화 과학으로 변화시키는 디지털 농업혁명에 진입하고 있습니다.

전통 농업은 경험과 관찰에 의존합니다. 농민들은 자신의 밭을 걸으며, 작물을 육안으로 검사하고, 직관에 기반하여 결정을 내립니다. 이 접근법은 효과적이지만:

  • 노동 집약적: 지속적인 수작업 모니터링 필요
  • 반응적: 증상이 나타난 후에야 문제 감지
  • 부정확: 작물 건강을 객관적으로 정량화하기 어려움
  • 규모 제한: 한 농민이 모니터링할 수 있는 토지 한계

🌍 글로벌 식량 과제

2050년까지 세계 인구는 97억 명에 도달하여 70% 증가한 식량 생산이 필요합니다. 그러나 기후 변화, 물 부족, 토양 황폐화가 수확량을 위협합니다. 작물 모니터링을 통한 정밀농업은 선택이 아니라 식량 안보를 위한 필수입니다.

1.2 작물 모니터링이란?

작물 모니터링은 센서, 카메라, 드론, AI를 사용하여 작물의 생육, 건강, 환경 조건을 지속적으로 관찰하고 측정하는 것입니다. 농업을 반응적에서 사전 예방적으로, 직관에서 데이터 기반 의사결정으로 변화시킵니다.

핵심 구성 요소:

  • 생육 추적: 초장, 엽면적, 생체량, 생육단계
  • 건강 평가: 병해 감지, 해충 식별, 영양 결핍
  • 환경 모니터링: 토양 수분, 온도, 습도, 광
  • 수확량 예측: 수확 계획을 위한 예측 모델

1.3 왜 작물 모니터링이 중요한가

1.3.1 조기 병해 감지

식물 병해는 전체 수확물을 황폐화시킬 수 있습니다. 1840년대 감자역병은 아일랜드 감자 작물을 파괴하여 대기근을 일으켰습니다. 오늘날에도 병해는 농민들에게 연간 수십억 달러의 손실을 초래합니다. AI 기반 작물 모니터링으로 육안 검사보다 수일 또는 수주 앞서 병해를 감지하여 피해가 확산되기 전에 예방 처치가 가능합니다.

📊 병해 감지 통계

  • 수작업 검사: 30-40% 감염 시 병해 감지
  • AI 기반 카메라: 5-10% 감염 시 감지
  • 조기 감지로 수확량의 70-90% 절약 가능

1.3.2 자원 사용 최적화

물, 비료, 농약은 과다 사용 시 비용이 많이 들고 환경에 해롭습니다. 작물 모니터링은 정밀 적용을 가능하게 합니다—필요한 것을, 필요한 곳에, 필요한 때에만 정확히 적용합니다.

  • 물 절약: 토양 수분 모니터링으로 20-40% 감소
  • 비료 효율성: 영양소 감지로 15-30% 감소
  • 농약 감소: 표적 병해 감지로 50-70% 감소

1.4 생육단계(Phenology) 이해

생육단계학은 식물 생애주기 단계를 연구하는 학문입니다. BBCH 스케일(독일 연방생물학연구소, 연방품종사무소, 화학산업)은 작물 생육단계를 식별하기 위한 십진법 코드 시스템으로, 종간 표준화되어 있습니다.

BBCH 스케일 개요:

BBCH 코드 | 단계        | 예시 (벼)
---------------------------------------
BBCH-00   | 건조 종자   | 파종 전
BBCH-10   | 묘목기      | 첫 잎 출현
BBCH-30   | 신장기      | 영양생장
BBCH-50   | 개화기      | 이삭 출현
BBCH-70   | 결실기      | 낟알 충진
BBCH-90   | 성숙기      | 수확 준비

🌾 생육단계가 중요한 이유

다른 생육단계는 다른 요구사항을 가집니다. BBCH-30(영양생장기)의 벼는 잎 성장을 위해 많은 질소가 필요합니다. BBCH-70(낟알 충진)에서는 인과 칼륨이 필요합니다. 작물 모니터링은 생육단계를 자동으로 추적하여 각 단계에서 올바른 투입이 이루어지도록 합니다.

1.5 작물 모니터링의 주요 지표

1.5.1 생육 지표

  • 초장: cm 단위로 측정, 생장률 표시
  • 엽면적지수(LAI): 지상 면적당 총 잎 표면적 (0-10)
  • 생체량: 총 식물 질량, 수확량 예측 인자 (kg/ha)

1.5.2 건강 지표

  • 엽록소 함량(SPAD): 광합성 능력 (0-60 척도)
  • NDVI(정규식생지수): 원격감지 건강 지표 (-1~1)
  • 병해 심각도: 피해 면적 백분율

1.5.3 환경 지표

  • 토양 수분: 수분 함량 (0-100%)
  • 온도: 대기 및 토양 온도 (°C)
  • 습도: 상대습도, 병해 위험 영향 (%)

1.6 WIA 작물 모니터링 표준

WIA-AGRI-006 표준은 작물 모니터링을 위한 통합 프레임워크를 제공하여 장치, 플랫폼, 국가 간 상호운용성을 보장합니다. 다음을 정의합니다:

  • 데이터 형식: 작물 데이터를 위한 표준화된 JSON 스키마
  • API 인터페이스: RESTful 엔드포인트 및 SDK
  • 통신 프로토콜: IoT 장치용 MQTT, WebSocket, CoAP
  • 통합 표준: 기상, AI, 시장, 보험 API

1.7 실제 영향

사례 연구: 한국 벼농사

1계절 후 결과:

참조 표준 및 적용 지침

본 표준은 다음 ALLOW 출처를 단일 진실원으로 삼아 데이터 교환·서명· 감사 증적 인터페이스를 제공합니다. 구현체는 이들 원전을 대체하지 않고, 데이터 교환·감사 증적·자동 검증을 위한 상호운용 계층을 정의합니다.

  • OpenAPI Specification 3.1 — REST API 정의
  • JSON Schema 2020-12 — 스키마 검증
  • ISO/IEC 27001:2022 — 정보보안 경영시스템
  • ISO/IEC 27018:2019 — 공용 클라우드의 PII 보호
  • ISO/IEC 17065:2012 — 제품 인증 기관
  • CycloneDX 1.5 / SPDX 2.3 — 소프트웨어 BOM
  • Sigstore (DSSE 봉투, Rekor 투명성 로그) — 서명 앵커링
  • in-toto Attestation Framework 1.0 — 빌드 출처 검증
  • IETF RFC 9700 — OAuth 2.1
  • IETF RFC 9457 — Problem Details for HTTP APIs
  • IETF RFC 8615 — well-known URIs

실무 적용 체크리스트

  • 매니페스트 무결성: 모든 발급은 SHA-256 또는 Sigstore 서명을 부착해 변조를 차단합니다. 발급 후 변경은 새 매니페스트로 처리 되며 이전 매니페스트는 supersede 마킹과 함께 보존됩니다.
  • 보관 기간: ISO/IEC 17065:2012 감사 기대치를 충족하기 위해 7년 이상의 보관 정책을 명시합니다. 관할 규정이 더 길면 그것을 따릅니다.
  • 버전 관리: 정정·확장은 시맨틱 버저닝(SemVer 2.0.0)을 따릅니다. 필수 요건의 추가는 마이너, 호환 단절은 메이저 버전 갱신을 트리거합니다. 메이저 변경은 최소 90일 병행 호환 기간을 거치며 그 기간 동안 신·구 두 버전 모두 WIA 등록부에 Stable로 표기 됩니다.
  • 부정결과 보고: 누락된 지표는 사유와 함께 매니페스트에 기록 되어야 합니다. 사유가 없는 누락은 감사 시 실패로 간주됩니다.
  • 키 회전: 매니페스트 서명 키는 배포 정책에 따라 회전됩니다. 이전 키는 공개된 키 이력에 보존되어 과거 매니페스트의 사후 검증을 보장합니다.

참조 토폴로지

구현체의 운용 토폴로지는 다음 4가지 중 하나로 매니페스트에 선언됩니다: 단일 테넌트 엣지(조직당 1개 런타임), 멀티 테넌트 게이트웨이(헤더 기반 격리로 다중 테넌트 서비스), 연합 메시(다수 런타임이 디렉토리 서비스 통해 상호 인식), 에어갭 배치(라이브 연결 없이 서명된 패키지를 out-of-band 채널로 전송). 각 토폴로지는 참조 테스트 벡터를 포함하며 구현체가 자신의 배포가 표준 패턴과 일치하는지 검증할 수 있도록 합니다.
