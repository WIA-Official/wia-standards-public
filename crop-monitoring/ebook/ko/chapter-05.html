챕터 5: 수확량 예측 및 예보

5.1 왜 수확량 예측이 중요한가

정확한 수확량 예측은 다음을 가능하게 합니다:

  • 시장 계획: 최적 가격에 작물 선판매
  • 물류: 수확 인력 및 장비 준비
  • 보험: 보장 범위 확인 및 청구 제출
  • 재무 계획: 수익 예측, 현금 흐름 관리

5.2 예측 입력

생육 지표

수확량 모델 입력:
- 시간 경과에 따른 초장
- 엽면적지수(LAI)
- 위성/드론 이미지의 NDVI
- 생육단계 진행 (BBCH 코드)
- 생체량 축적

환경 데이터

  • 온도 (생장 적산온도)
  • 강수량 (누적, 시기)
  • 일사량
  • 토양 수분

5.3 예측 모델

머신러닝 접근법

import numpy as np
from sklearn.ensemble import RandomForestRegressor

# 특성: 초장, LAI, 엽록소, 온도, 강수량
X_train = [[45, 3.8, 42, 25, 500], ...]
y_train = [5500, ...]  # 실제 수확량

model = RandomForestRegressor(n_estimators=100)
model.fit(X_train, y_train)

# 현재 작물 수확량 예측
current_data = [[50, 4.2, 45, 26, 520]]
predicted_yield = model.predict(current_data)
print(f"예상 수확량: {predicted_yield[0]} kg/ha")

5.5 시나리오 분석

관리 결정을 최적화하기 위해 "만약" 시나리오를 테스트합니다.

POST /forecast/scenario
{
  "cropId": "CROP-2025-001",
  "scenario": {
    "weatherOverride": {"rainfall": "+20%", "temperature": "+2°C"},
    "interventions": ["extra_fertilizer", "pest_control"]
  }
}

응답:
{
  "baselineYield": 5500,
  "scenarioYield": 6200,
  "difference": "+12.7%",
  "recommendation": "투자 정당화됨"
}

📊 예측 정확도

초기 계절 (파종 후 30일): ±25% 정확도
중기 계절 (60일): ±15% 정확도
말기 계절 (수확 2주 전): ±5% 정확도

5.6 실제 적용

사례 연구: 브라질 옥수수 농민들은 수확량 예측을 사용하여 선도 계약을 협상했습니다. 정확한 예측(실제 대비 8% 이내)으로 3개월 전 가격을 고정하여 현물 시장 판매 대비 헥타르당 15만원 증가.

실무 적용 체크리스트

  • 매니페스트 무결성: 모든 발급은 SHA-256 또는 Sigstore 서명을 부착해 변조를 차단합니다. 발급 후 변경은 새 매니페스트로 처리 되며 이전 매니페스트는 supersede 마킹과 함께 보존됩니다.
  • 보관 기간: ISO/IEC 17065:2012 감사 기대치를 충족하기 위해 7년 이상의 보관 정책을 명시합니다. 관할 규정이 더 길면 그것을 따릅니다.
  • 버전 관리: 정정·확장은 시맨틱 버저닝(SemVer 2.0.0)을 따릅니다. 필수 요건의 추가는 마이너, 호환 단절은 메이저 버전 갱신을 트리거합니다. 메이저 변경은 최소 90일 병행 호환 기간을 거치며 그 기간 동안 신·구 두 버전 모두 WIA 등록부에 Stable로 표기 됩니다.
  • 부정결과 보고: 누락된 지표는 사유와 함께 매니페스트에 기록 되어야 합니다. 사유가 없는 누락은 감사 시 실패로 간주됩니다.
  • 키 회전: 매니페스트 서명 키는 배포 정책에 따라 회전됩니다. 이전 키는 공개된 키 이력에 보존되어 과거 매니페스트의 사후 검증을 보장합니다.

참조 토폴로지

구현체의 운용 토폴로지는 다음 4가지 중 하나로 매니페스트에 선언됩니다: 단일 테넌트 엣지(조직당 1개 런타임), 멀티 테넌트 게이트웨이(헤더 기반 격리로 다중 테넌트 서비스), 연합 메시(다수 런타임이 디렉토리 서비스 통해 상호 인식), 에어갭 배치(라이브 연결 없이 서명된 패키지를 out-of-band 채널로 전송). 각 토폴로지는 참조 테스트 벡터를 포함하며 구현체가 자신의 배포가 표준 패턴과 일치하는지 검증할 수 있도록 합니다.
