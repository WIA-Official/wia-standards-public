## 📋 Overview

The WIA Crop Monitoring Standard (WIA-AGRI-006) provides a comprehensive framework for real-time crop health intelligence, enabling standardized plant health tracking, disease detection, growth stage monitoring, and AI-powered crop analytics.

**English Title:** Crop Monitoring
**Korean Title:** 작물 모니터링 (Jakmul Moniteo-ring)

This standard unifies crop monitoring systems across different platforms, enabling farmers to track crop health from germination to harvest with precision and early disease intervention.

---

## 🎯 Key Features

- **Plant Health Tracking:** Real-time NDVI and health indices
- **Disease Detection:** AI-powered early disease identification
- **Growth Stage Monitoring:** Automated phenological tracking
- **Pest Detection:** Computer vision-based pest identification
- **Stress Analysis:** Drought, heat, and nutrient stress detection
- **Yield Forecasting:** Growth-based yield prediction
- **Multi-Crop Support:** Rice, wheat, corn, vegetables, fruits

## 🎮 Quick Start

1. **View Landing Page:** Open `index.html` in your browser
2. **Try Simulator:** Navigate to `simulator/index.html` for crop analysis demo
3. **Read Ebook:** Access comprehensive guides in `ebook/en/` or `ebook/ko/`

## 📐 Specification Phases

### Phase 1: Data Format (15 KB)
- Crop health data structures
- NDVI and vegetation indices
- Disease classification schemas
- Growth stage definitions
- Image metadata formats

### Phase 2: API Interface (14 KB)
- REST API for crop data management
- Real-time health monitoring endpoints
- Disease detection APIs
- Historical trend analysis
- Alert and notification services

### Phase 3: Protocol (13 KB)
- MQTT for real-time health updates
- WebSocket for live image streaming
- HTTP/REST for batch processing
- Image upload protocols
- Mobile app synchronization

### Phase 4: Integration (16 KB)
- AI/ML platforms (TensorFlow, PyTorch)
- Drone and satellite imagery integration
- Weather API correlation
- Farm management systems
- Agricultural advisory platforms
