Chapter 2: Data Schema

"The structure of data determines the quality of insights. A well-designed schema is the foundation of climate action at scale."

2.1 Base Message Structure

All WIA Climate messages share a common base structure, regardless of the specific climate domain. This ensures consistency and enables unified processing pipelines.

2.1.1 Universal Message Format

{
  "$schema": "https://wia.live/climate/data/v1/schema.json",
  "version": "1.0.0",
  "type": "<data_type>",
  "timestamp": {
    "unix_ms": <milliseconds>,
    "iso8601": "<ISO 8601 string>"
  },
  "location": {
    "latitude": <degrees>,
    "longitude": <degrees>,
    "altitude_m": <meters>,
    "crs": "EPSG:4326"
  },
  "device": {
    "manufacturer": "<string>",
    "model": "<string>",
    "serial": "<string>"
  },
  "data": {
    // Domain-specific data
  },
  "meta": {
    "quality_score": <0.0-1.0>,
    "uncertainty": <number>,
    "source": "<string>"
  }
}

2.1.2 Required vs Optional Fields

Field Type Required Description
version string ✅ Yes Semantic version (e.g., "1.0.0")
type string ✅ Yes Data type identifier
timestamp object ✅ Yes Measurement time in UTC
location object ✅ Yes Geographic coordinates (WGS84)
device object ✅ Yes Data source information
data object ✅ Yes Domain-specific measurements
$schema string ❌ Optional JSON Schema URL for validation
meta object ❌ Optional Quality and metadata

2.2 Carbon Capture Schema

Carbon capture technologies remove CO2 from the atmosphere or from industrial emissions. This schema tracks capture rates, energy efficiency, and storage methods.

2.2.1 Data Type

"type": "carbon_capture"

2.2.2 Carbon Capture Fields

Field Type Unit Description
technology string - Capture technology type: "dac", "post_combustion", "pre_combustion", "oxy_fuel", "bioenergy_ccs"
capture_rate_kg_per_hour number kg/h CO2 capture rate
co2_concentration_ppm number ppm Inlet CO2 concentration
co2_purity_percentage number % Captured CO2 purity (0-100)
energy_consumption_kwh number kWh Energy used for capture
sorbent_status object - Sorbent material information
storage object - CO2 storage details

2.2.3 Complete Example

{
  "version": "1.0.0",
  "type": "carbon_capture",
  "timestamp": {
    "unix_ms": 1702468800000,
    "iso8601": "2024-12-14T12:00:00.000Z"
  },
  "location": {
    "latitude": 64.0,
    "longitude": -21.0,
    "altitude_m": 100
  },
  "device": {
    "manufacturer": "Climeworks",
    "model": "Orca DAC",
    "serial": "ORCA-2024-001"
  },
  "data": {
    "technology": "dac",
    "capture_rate_kg_per_hour": 125.5,
    "co2_concentration_ppm": 415,
    "co2_purity_percentage": 99.2,
    "energy_consumption_kwh": 2500,
    "sorbent_status": {
      "type": "solid_amine",
      "efficiency_percentage": 85.5,
      "cycles_completed": 1250
    },
    "storage": {
      "method": "geological",
      "pressure_mpa": 10.5,
      "depth_m": 2000,
      "formation_type": "basalt"
    }
  },
  "meta": {
    "quality_score": 0.98,
    "source": "sensor"
  }
}

2.3 Weather Control Schema

Weather control operations include cloud seeding, fog dispersal, and precipitation enhancement. This schema captures intervention parameters and atmospheric results.

2.3.1 Data Type

"type": "weather_control"

2.3.2 Weather Control Fields

Field Type Description
operation_type string "cloud_seeding", "fog_dispersal", "hail_suppression", "rain_enhancement"
seeding_agent object Type, mass, and concentration of seeding material
delivery_method string How agent was deployed: "aircraft", "ground_generator", "rocket"
target_cloud object Cloud characteristics (type, altitude, coverage)
atmospheric_conditions object Temperature, humidity, wind at operation time
result object Measured outcomes (precipitation, duration, effectiveness)

2.3.3 Seeding Agent Types

  • silver_iodide: Most common, effective below -5°C
  • potassium_iodide: Alternative to silver iodide
  • dry_ice: For rapid cooling of supercooled clouds
  • liquid_propane: Expands rapidly on release
  • salt: For warm cloud seeding over oceans

2.4 Geoengineering Schema

Geoengineering interventions aim to counteract global warming through solar radiation management or carbon dioxide removal. This schema tracks intervention parameters and monitored effects.

2.4.1 Data Type

"type": "geoengineering"

2.4.2 Geoengineering Fields

Field Description Examples
intervention_type Specific geoengineering technique "stratospheric_aerosol_injection", "marine_cloud_brightening", "ocean_fertilization"
category High-level classification "solar_radiation_management" (SRM) or "carbon_dioxide_removal" (CDR)
deployment How intervention was deployed Aerosol type, injection altitude, particle size, injection rate
coverage Geographic extent Area in km², latitude/longitude ranges
effects Measured environmental impacts Radiative forcing (W/m²), temperature change, precipitation change
monitoring Safety and side effect tracking Ozone impact, acid deposition, ecosystem effects

Important: Geoengineering is highly experimental and controversial. Data transparency is critical for informed public discourse and governance. All geoengineering data should include comprehensive monitoring of potential negative effects.

2.5 Vertical Farming Schema

Vertical farms use controlled environment agriculture to grow crops indoors with minimal land and water. This schema tracks environmental controls, nutrient delivery, and crop performance.

2.5.1 Data Type

"type": "vertical_farming"

2.5.2 Vertical Farming Fields

Category Key Fields Typical Values
System Type hydroponics, aeroponics, aquaponics, substrate -
Environment temperature_celsius, humidity_percentage, co2_ppm, vpd_kpa 22°C, 65%, 800ppm, 0.85kPa
Lighting ppfd_umol_per_m2_s, photoperiod_hours, spectrum 450 µmol/m²/s, 16h, full_spectrum
Nutrient Solution ph, ec_ms_per_cm, temperature, dissolved_oxygen 6.0, 1.8 mS/cm, 20°C, 8.0 mg/L
Crop Info species, variety, growth_stage, days_after_planting lactuca_sativa, butterhead, vegetative, 21
Yield fresh_weight_kg, kg_per_m2 42.5 kg, 8.5 kg/m²

2.5.3 Key Metrics Explained

  • PPFD (Photosynthetic Photon Flux Density): Measures usable light for photosynthesis in µmol/m²/s
  • VPD (Vapor Pressure Deficit): Difference between air moisture content and saturation, drives transpiration
  • EC (Electrical Conductivity): Measures nutrient concentration in solution (mS/cm)
  • DLI (Daily Light Integral): Total photosynthetic light received per day (mol/m²/day)

2.6 Ocean Cleanup Schema

Ocean cleanup operations remove plastic and other debris from marine environments. This schema tracks collection efficiency, debris composition, and water quality.

2.6.1 Data Type

"type": "ocean_cleanup"

2.6.2 Ocean Cleanup Fields

Field Unit Description
operation_type - "floating_barrier", "river_interceptor", "beach_cleanup", "autonomous_vessel", "drone_collection"
collection.total_mass_kg kg Total debris collected
collection.plastic_mass_kg kg Plastic component of collection
collection.microplastic_count - Number of microplastic particles (0.1-5mm)
area.swept_km2 km² Ocean area covered by operation
area.efficiency_kg_per_km2 kg/km² Collection efficiency metric
water_conditions various Temperature, salinity, pH, current speed, wave height

2.7 Climate Model Schema

Climate models generate projections of future climate conditions under various scenarios. This schema follows CMIP6 conventions for maximum compatibility with research infrastructure.

2.7.1 Data Type

"type": "climate_model"

2.7.2 Climate Model Fields

Field Description
model.source_id CMIP6 model identifier (e.g., "CESM2", "UKESM1")
model.experiment_id Experiment/scenario (e.g., "ssp245", "ssp585", "historical")
variable.name Standard variable name (e.g., "tas", "pr", "psl")
variable.units Physical units (e.g., "K", "kg m-2 s-1", "Pa")
grid.resolution_deg Spatial resolution in degrees
time.frequency "yr", "mon", "day", "6hr", "3hr", "1hr"
scenario.ssp Shared Socioeconomic Pathway (e.g., "SSP2-4.5")

2.7.3 Common CMIP6 Variables

Variable Name Unit
tas Near-surface air temperature K
pr Precipitation kg m-2 s-1
psl Sea level pressure Pa
uas, vas Eastward/northward wind m s-1
hurs Relative humidity %
rsds Surface downwelling shortwave radiation W m-2
