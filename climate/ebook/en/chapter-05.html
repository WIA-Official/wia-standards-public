Chapter 5: Phase 2 — API Interface

Overview: Phase 2 binds Phase-1 records to four equally normative HTTP façades — OGC API — Features / Coverages / EDR / Records, plus STAC 1.0 — and connects authority systems (UNFCCC ETF, IFRS S2 filers, registries) without re-keying any data.

5.1 Four API façades, one truth

FaçadePathUse case
OGC API — Features/v1/collections/{id}/itemsVector geometry: facility footprints, project boundaries, treaty-defined polygons
OGC API — Coverages/v1/collections/{id}/coverage?bbox=…&subset=time(…)Subset of CF NetCDF / Zarr by bbox, time, level
OGC API — EDR/v1/collections/{id}/position?coords=…Time-series for a point or trajectory (drone, ship, balloon)
OGC API — Records (+ STAC 1.0)/v1/collectionsCatalogue, search, broker subscription

An implementation MUST surface the same state through all four. The conformance suite (https://conformance.wia-climate.org) executes equivalent queries on each and compares result hashes.

5.3 OGC API — Coverages (the workhorse)

GET /v1/collections/cmip6.scenariomip.kma-k-ace.ssp245.tas/coverage
   ?bbox=124,33,131,39
   &subset=time("2050-01-01"/"2099-12-31")
   &f=netcdf
Authorization: Bearer $TOKEN

→ HTTP/1.1 200 OK
   Content-Type: application/x-netcdf;version=4
   Content-Disposition: attachment; filename="tas_…20500101-20991231.nc"
   ETag: "W/\"sha256-3a2f…\""

Output formats are negotiated through the f query parameter or Accept header: netcdf, zarr, covjson (CoverageJSON, CF-aligned), geotiff.

5.4 OGC API — EDR for point time-series

GET /v1/collections/era5.t2m/position
   ?coords=POINT(126.98 37.57)
   &datetime=2025-01-01T00:00:00Z/2025-12-31T23:00:00Z
   &f=covjson

Returns a CoverageJSON document the analyst can plot directly. The same path supports cube, area, trajectory, and corridor queries (e.g., a ship's track, an airline route).

5.5 STAC catalogue and search

POST /v1/search
{
  "collections": ["cmip6.scenariomip.*.ssp245.tas"],
  "bbox": [124, 33, 131, 39],
  "datetime": "2050-01-01/2099-12-31",
  "query": {
    "wia:scenario": {"eq": "ssp245"},
    "wia:variant_label": {"in": ["r1i1p1f1","r2i1p1f1"]}
  },
  "limit": 50
}

STAC 1.0 items embed asset HREFs that point to NetCDF, Zarr, or a presigned object-storage URL. The self link is durable; ETag-based caching cuts repeated downloads to a 304.

Chapter Summary

Key Takeaways

  1. Four OGC API façades (Features, Coverages, EDR, Records) plus STAC must surface identical state.
  2. Idempotency-Key + ETag/If-Match handle network blips and concurrent updates safely.
  3. RFC 9457 problem+json gives clients stable failure types — including CF-validation faults.
  4. Disclosures are filed once and served as both JSON-LD and iXBRL.
  5. UNFCCC ETF/CRT 2.0 BTR submission is a first-class endpoint, signed with the NDA's key.
