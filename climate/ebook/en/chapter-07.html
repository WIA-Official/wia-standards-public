Chapter 7: Phase 4 — Integration

Overview: Phase 4 is what turns the standard into outcomes — UNFCCC Enhanced Transparency Framework treaty submissions, IFRS S2 + ESRS E1 corporate disclosures, voluntary carbon-market registry bridges, ESG dashboards, and climate-justice tags that prevent under-counting of historical responsibility.

7.1 UNFCCC ETF / CRT 2.0 Biennial Transparency Report

Decision 18/CMA.1 (Katowice 2018) established the Enhanced Transparency Framework. From 2024 onward each Party submits a Biennial Transparency Report (BTR) including a national greenhouse-gas inventory in the Common Reporting Tables 2.0 schema, mitigation progress, projections, financial flows, and (for least-developed countries) capacity-building needs.

The full BTR envelope is signed by the National Designated Authority's key and submitted via POST /etf/v1/btr (Chapter 5 §5.9).

7.2 IFRS S2 corporate disclosure twin

IFRS S2 ("Climate-related Disclosures") is effective for annual periods beginning 1 January 2024 in early-adopter jurisdictions and subsequently across the IFRS network. The standard requires Scope 1, 2, 3 totals; transition risks; physical risks; targets; and governance. WIA-CLIMATE-022 generates the disclosure once as JSON-LD and emits an iXBRL twin via the IFRS Sustainability Disclosure Taxonomy (2024).

// Core disclosure path
GET /v1/disclosures/ifrs-s2/2026/213800ABCDEFGHIJKL00
Accept: application/ld+json   →  JSON-LD (machine consumption)
Accept: application/inline-xbrl+xml  →  iXBRL (regulator filing)
Accept: text/html             →  Human-readable disclosure narrative

7.3 ESRS E1 + EU CSRD digital tagging

EU large undertakings file ESRS E1 disclosures via the European Single Electronic Format (ESEF) using the EFRAG XBRL taxonomy. Mapping: WIA-CLIMATE-022 inventory categories → EFRAG datapoints. The mapping table is normative and lives at https://schemas.wia-climate.org/mappings/efrag-esrs-e1-2024.json.

7.5 ESG dashboard integration

CDP, MSCI, S&P Trucost, ISS-ESG and Bloomberg ESG consume the same JSON-LD disclosure. The standard publishes a schema:Observation JSON-LD endpoint that ESG vendors hit instead of asking the company to fill yet another questionnaire.

7.6 Climate-justice tags (mandatory)

  • Historical responsibility — every disclosure includes the cumulative emissions for the entity's home jurisdiction since 1850, sourced from AR6 WG-III Chapter 2 Table 2.5 + UNEP Emissions Gap Report 2024.
  • Just-transition flow — finance flows tagged with destination geography (LDC/SIDS/non-LDC), mapped to OECD CRS purpose code 41081 (climate change adaptation) or 41082 (mitigation).
  • Loss-and-damage contributions — separate line item, not netted into mitigation finance.
  • Indigenous-territory overlap — for land-based projects, polygon overlap with the LandMark Global Platform indigenous-lands dataset is reported with an aware/consented/objected status.

7.7 WIA ecosystem integration

  • WIA-VC — issues verifier reports, baseline studies, third-party assurance certificates.
  • WIA-IDENTITY — DIDs for verifiers, NDAs, project developers.
  • WIA-IOT — onboards CEMS, smart meters, fuel-flow scales.
  • WIA-AGRI-014 — agricultural Scope 3 cold-chain ingest into corporate inventories.
  • WIA-CARBON — ISO 14083 transport calculation drop-in.
  • WIA-BLOCKCHAIN — Merkle-root anchoring for tamper evidence.

Chapter Summary

Key Takeaways

  1. UNFCCC ETF, IFRS S2, and ESRS E1 are first-class endpoints, not adapters bolted onto an internal system.
  2. Voluntary carbon-market registries federate through W3C Verifiable Credentials, with corresponding-adjustment status mandatory.
  3. Climate-justice tags (historical responsibility, just-transition, loss-and-damage, indigenous overlap) are non-optional.
  4. WIA ecosystem standards (VC, IOT, AGRI-014, CARBON, BLOCKCHAIN, IDENTITY) plug in cleanly and reduce duplicate plumbing.
  5. Meter-to-disclosure-to-treaty becomes one lifecycle, not seven systems.
