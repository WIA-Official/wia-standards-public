Chapter 4: Phase 1 — Data Format

Overview: Phase 1 binds the wire bytes — CF-1.10 NetCDF for gridded fields, Zarr v3 for cloud-native cubes, JSON-LD for GHG inventories and disclosures, GeoJSON / OGC CRS84 for vectors, and UCUM for every unit on the bus.

4.1 CF-1.10 NetCDF for gridded climate data

Every gridded field — observation, reanalysis, model output — uses NetCDF-4 with the CF Conventions 1.10 (released 2024). The Conventions standardise variable names, coordinate axes, units, and metadata so a generic reader can plot a temperature field without a vendor manual.

// ncdump -h tas_day_KMA-K-ACE_ssp245_r1i1p1f1_gn_20260101-20261231.nc
netcdf tas_day_KMA-K-ACE_ssp245_r1i1p1f1_gn_20260101-20261231 {
dimensions:
    time = UNLIMITED ;
    lat  = 192 ;
    lon  = 288 ;
variables:
    float tas(time, lat, lon) ;
        tas:standard_name = "air_temperature" ;
        tas:long_name     = "Near-Surface Air Temperature" ;
        tas:units         = "K" ;
        tas:cell_methods  = "time: mean" ;
    double time(time) ;
        time:standard_name = "time" ;
        time:units         = "days since 1850-01-01 00:00:00" ;
        time:calendar      = "proleptic_gregorian" ;
    double lat(lat), lon(lon) ;

// global attributes:
    :Conventions          = "CF-1.10" ;
    :title                = "Near-Surface Air Temperature, KMA K-ACE SSP2-4.5" ;
    :institution          = "Korea Meteorological Administration / NIMS" ;
    :source               = "K-ACE 1.0" ;
    :experiment_id        = "ssp245" ;
    :variant_label        = "r1i1p1f1" ;
    :tracking_id          = "hdl:21.14100/abcdef-1234" ;
    :wia_compliance       = "WIA-CLIMATE-022:2026" ;
    :wia_signature        = "ed25519:wia:climate:kma-k-ace-2026#k1" ;
}

The standard_name values come from the CF Standard Name Table (currently version 84, June 2024). Custom names are forbidden; if you need a name that is not in the table, the path is to propose it to the CF community.

4.2 DRS filename convention (CMIP6-aligned)

<variable>_<table>_<source_id>_<experiment_id>_<variant_label>_<grid_label>_<date_range>.nc

tas_day_KMA-K-ACE_ssp245_r1i1p1f1_gn_20260101-20261231.nc
│   │   │         │      │        │  │
│   │   │         │      │        │  └─ ISO 8601 range (YYYYMMDD-YYYYMMDD)
│   │   │         │      │        └──── grid label (gn = native, gr = regridded)
│   │   │         │      └───────────── variant (realisation/initialisation/physics/forcing)
│   │   │         └──────────────────── ScenarioMIP experiment id
│   │   └────────────────────────────── source identifier (institution + model)
│   └────────────────────────────────── CMIP6 table id (day, mon, Amon, etc.)
└────────────────────────────────────── CF standard name short form

4.3 Zarr v3 for cloud-native subsets

NetCDF-4 is fast on local disks but expensive over object storage. WIA-CLIMATE-022 mirrors every NetCDF granule into Zarr v3 with consistent chunking (typical: {time: 30, lat: 192, lon: 288}) so analysts can xarray.open_zarr a 30 TB store and read a single bounding box in milliseconds.

import xarray as xr
ds = xr.open_zarr(
    "s3://wia-climate/cmip6/ScenarioMIP/KMA/K-ACE/ssp245/r1i1p1f1/Amon/tas/gn/v20260315",
    consolidated=True,
)
seoul = ds.tas.sel(lat=37.57, lon=126.98, method="nearest").sel(time=slice("2050","2099"))

4.5 Units: UCUM as the single source

The Unified Code for Units of Measure (UCUM, ISO 80000-1 aligned) is the only unit vocabulary on the wire. Examples:

TemperatureKKelvin (CF requires SI)

Emission factorkg/GJKilogram per gigajoule
Concentration10*-6 (mol/mol)parts per million by volume

4.6 Spatial reference and geometry

Every coordinate on the wire is OGC CRS84 (WGS 84 longitude/latitude, decimal degrees). Polygons use GeoJSON Feature (RFC 7946); rasters declare their CRS in the CF grid_mapping attribute. Projected CRS strings are EPSG codes (e.g., EPSG:5179 for Korea 2000 / Unified TM).

4.7 Validation

  • NetCDF files MUST pass cf-checker with no warnings against CF 1.10.
  • JSON-LD documents MUST round-trip through canonicalisation (RFC 8785) and signature verification.
  • UCUM unit strings MUST parse with libucum; symbol-only forms are forbidden in machine-readable fields.
  • Coordinate values MUST round-trip through CRS84; precision is bounded at 7 decimal degrees.
  • Time values MUST use ISO 8601-1 with explicit calendar attribute (proleptic_gregorian default for climate data).

Chapter Summary

Key Takeaways

  1. CF-1.10 NetCDF (and its Zarr v3 mirror) is the only on-the-wire form for gridded climate fields.
  2. DRS filenames carry full provenance — variable, model, scenario, ensemble member, grid, time range.
  3. JSON-LD with a pinned @context is the only on-the-wire form for inventories and disclosures.
  4. UCUM is the unit system; symbol-only "tCO₂e" is rejected.
  5. OGC CRS84 + GeoJSON + EPSG-coded projections cover all spatial geometry.
