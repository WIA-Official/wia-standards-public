Chapter 6: Phase 3 — Protocol

Overview: Phase 3 is the data plane: WMO WIS 2.0 MQTT 5 brokers for atmospheric observations, Kafka 3 with exactly-once semantics for inventory event streams, NetCDF-over-HTTPS with chunked-transfer for bulk, OGC SensorThings 1.1 for ground stations, and TLS 1.3 / OAuth 2.1 + DPoP / Ed25519 throughout.

6.1 WMO WIS 2.0 MQTT broker network

WIS 2.0 became the production WMO Information System in 2023, replacing the legacy GTS message-switched network. It is built on MQTT 5 brokers run by Global Information System Centres (e.g., DWD, NCAR, KMA) and Data Collection or Production Centres. Subscribers receive WIS2 Notification Messages (WNM) — small JSON payloads that point to the actual data in object storage.

// Subscribe to KMA's surface synop notifications
mosquitto_sub -h gisc.kma.go.kr -p 8883 \
  --cafile wmo-ca.pem --cert client.crt --key client.key \
  -t "origin/a/wis2/+/data/core/weather/surface-based-observations/synop/#"

// Notification payload (WNM)
{
  "id":"urn:wmo:md:kma:weather-obs:2026-04-25T12:00:00Z",
  "conformsTo":["https://wis.wmo.int/spec/wnm/1/conf/core"],
  "type":"Feature",
  "geometry":{"type":"Point","coordinates":[126.98,37.57]},
  "properties":{
    "data_id":"weather/surface-based-observations/synop/2026-04-25T12:00:00Z/47108.bufr",
    "datetime":"2026-04-25T12:00:00Z",
    "pubtime":"2026-04-25T12:01:18Z",
    "wmo:dataPolicy":"core",
    "integrity":{"method":"sha512","value":"…"}
  },
  "links":[
    {"rel":"canonical","href":"https://gisc.kma.go.kr/data/…/47108.bufr","type":"application/x-bufr"}
  ]
}

WIA-CLIMATE-022 ingests these notifications, downloads referenced data, transcodes BUFR/GRIB2 into CF-1.10 NetCDF, and re-publishes through the OGC API endpoints from Chapter 5.

6.2 MQTT 5 features the standard relies on

  • Session Expiry Interval — long-lived subscribers (regulator dashboards) survive transient drops.
  • Payload Format Indicator — distinguishes JSON WNM from binary BUFR fast-paths.
  • Subscription Identifier — multiplexes hundreds of topic filters per client.
  • Shared Subscriptions — load-balancing across consumer pods ($share/group/topic).
  • AUTH packet + Reason Codes — surfaces enhanced auth failures.

6.4 Kafka 3 for inventory streaming with exactly-once

GHG inventory events (meter readings, fuel deliveries, process measurements) are streamed through Kafka 3 with the three-rule recipe for exactly-once semantics:

  1. enable.idempotence=true on producers (since Kafka 3.0 default).
  2. isolation.level=read_committed on consumers.
  3. Transactional producer with a stable transactional.id per ingest worker.
// Inventory ingest event
{
  "schemaId": "wia.climate.ingest.v1",
  "facilityLei": "213800ABCDEFGHIJKL00",
  "meterId": "POH-CEMS-7",
  "ts": "2026-04-25T12:00:00+09:00",
  "channel": "co2_concentration_dry_corrected",
  "value": 12.4,
  "unit": "%v/v",
  "quality": {"validation":"ok","calibrationDue":"2026-09-12"},
  "lineage": {"sourceFile":"s3://kr-cems/poh-7/2026-04-25.csv","row":34812}
}

6.5 NetCDF-over-HTTPS for bulk

For bulk historical reanalyses (ERA5, JRA-3Q), the standard supports HTTP/2 chunked transfer with byte-range fetches. The NetCDF library (netCDF-c 4.9 with the byterange driver) opens the remote file as if it were local; only the requested chunks travel.

import netCDF4 as nc
url = "https://archive.wia-climate.org/era5/single-level/t2m/2025/era5_t2m_2025.nc"
ds = nc.Dataset(url + "#mode=bytes")
seoul = ds.variables["t2m"][:, 280, 530]   # one grid cell, 8760 hours, <10 MB downloaded

6.6 OGC SensorThings 1.1 for ground stations

Air-quality stations, pyranometers, eddy-covariance towers and CEMS smokestack monitors register through OGC SensorThings — a lightweight REST/MQTT vocabulary aligned with ISO 19156 Observations & Measurements.

6.7 Tamper-evident anchoring

Every Kafka ingest window (default 30 seconds) and every OGC API write produces a Merkle root that is anchored on WIA Chain PoA and onward to BSC mainnet. External auditors verify integrity without seeing the underlying data.

async function anchorIngestWindow(rootHex, windowStartIso) {
  const tx = await wiaChain.submit({
    network:"WIA-CLIMATE",
    payload:{ type:"INGEST_WINDOW", root: rootHex, windowStart: windowStartIso }
  });
  return tx.transactionId;
}

6.8 Security baseline

LayerRequired
TransportTLS 1.3 with TLS_AES_256_GCM_SHA384 or TLS_CHACHA20_POLY1305_SHA256
Token formatJWT EdDSA, audience-restricted, 30 min lifetime, DPoP-bound (RFC 9449)
Application signingEd25519 (RFC 8032) over RFC 8785 canonicalised JSON
Key managementHSM-backed, RFC 7517 JWKS rotation, 90-day max
BUFR/GRIB integritySHA-512 of payload published in WNM integrity field

6.9 Failure modes the protocol layer must handle

  • Broker outage — gateways MUST buffer up to 24 h locally and replay with original timestamps.
  • Late observations — re-publish under cache/... topic prefix, not origin/....
  • Clock drift — NTP every 6 h; ingest rejects readings with |now − ts| > 600 s (climate windows are slower than logistics).
  • Replay attacks — RFC 9421 nonces + JWT JTI cache.
  • Schema evolution — Kafka subject schema registry with backwards-compatibility enforced.

Chapter Summary

Key Takeaways

  1. WIS 2.0 MQTT is the inbound observation pipeline; the standard republishes through OGC APIs.
  2. Kafka 3 with idempotent + transactional producers gives exactly-once for inventory events.
  3. NetCDF-over-HTTPS byte-range reads let analysts open multi-TB archives without downloading them.
  4. OGC SensorThings 1.1 standardises ground-station ingest, including stack-emissions monitors.
  5. Merkle-root anchoring on WIA Chain PoA + BSC gives external auditability without data exposure.
