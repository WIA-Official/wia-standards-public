3.2 Three core principles

3.2.1 Interoperability first

Every identifier, every unit, every coordinate reference, every code list MUST come from an external authority — IPCC AR6, UNFCCC, WMO, OGC, ISO, IFRS, GHG Protocol, UCUM. WIA-CLIMATE-022 does not invent code lists; it composes existing ones with explicit version pins. A facility identifier is an LEI (ISO 17442). A pollutant code is a CAS Registry Number. A unit is a UCUM symbol. A coordinate reference is OGC CRS84.

3.2.2 Verifiability by default

Every primary measurement record has a hash; every aggregated total carries the Merkle root of its inputs; every disclosure is signed with EdDSA-JCS-2022 (W3C Data Integrity Proofs 1.0). An auditor's job becomes verifying signatures and walking links, not reconciling spreadsheets.

3.2.3 Climate justice as design constraint

Methodologies that systematically under-count least-developed-country emissions or over-count their forest baselines are violations of the standard. The standard requires explicit reporting of biome-level baselines (FAO Global Forest Resources Assessment 2025), historical responsibility (cumulative emissions since 1850 per AR6 WG-III Chapter 2), and just-transition tags on transition-finance flows.

3.4 Key components in one paragraph each

Atmospheric observation pipeline

WIS 2.0 broker subscriptions deliver WMO-grade observations as CF-1.10 NetCDF granules. WIA-CLIMATE-022 wraps each granule in a STAC 1.0 item, exposes it through an OGC API — Records endpoint, and notifies subscribers through a CloudEvents 1.0 webhook signed with HTTP Message Signatures (RFC 9421).

Climate-model output service

CMIP6 / ScenarioMIP / DCPP / CORDEX outputs are mirrored into Zarr v3 stores with consistent DRS naming. An OGC API — Coverages endpoint serves arbitrary subset requests by bounding box, time, and SSP scenario.

Corporate GHG inventory

Facility-level meter data, fuel deliveries, and process emissions are captured as JSON-LD records with GHG Protocol scope tags, ISO 14064-1 categorisation, and UCUM-coded units. Aggregations roll up to the legal entity (LEI) and corporate group.

Disclosure twin

A single source-of-truth inventory generates both an iXBRL filing (IFRS S2 / ESRS E1) and a JSON-LD twin used by the regulator's machine-readable consumption pipeline.

Carbon registry interface

Verra VCS, Gold Standard 4PR, ACR, and CAR all expose credit issuance/retirement events as W3C VCs. WIA-CLIMATE-022 federates these into a unified credit ledger view with corresponding-adjustment status.

Treaty-reporting bridge

The standard maps a national inventory expressed as JSON-LD into the UNFCCC CRT 2.0 schema, generates a Biennial Transparency Report (BTR) JSON envelope, and signs it with the National Designated Authority's key.

3.6 Integration with the WIA ecosystem

  • WIA-VC — issues verifiable certifications (verifier reports, baseline studies).
  • WIA-BLOCKCHAIN — anchors Merkle roots of inventory aggregations onto WIA Chain PoA + BSC.
  • WIA-IDENTITY — DIDs for verifiers, project developers, registry operators.
  • WIA-IOT — facility-level sensor onboarding (CEMS, smart meters, fuel scales).
  • WIA-CARBON — Scope-3 transport calculation per ISO 14083:2023.
  • WIA-AGRI-014 — agricultural Scope 3 cold-chain handoff.

3.7 The "why standard" test

A system implementing WIA-CLIMATE-022 produces output that another implementer's verifier can read, walk, and re-derive without prior coordination — only by reading this book and consulting the cited authorities. That property — read by a stranger, derive the same answer — is the standard's only ultimate test.

Chapter Summary

Key Takeaways

  1. Five layers (data → protocol → API → application → disclosure) cleanly separate concerns; phase n+1 builds only on phase n.
  2. Interoperability, verifiability, and climate justice are non-negotiable design constraints.
  3. Six components (observations, models, inventory, disclosure, registries, treaty) share one identifier and unit system.
  4. Reference technology is open-source; alternatives are first-class but must pass conformance.
  5. The standard is interoperable with the broader WIA ecosystem and explicitly with WIA-AGRI-014, WIA-CARBON, WIA-VC, WIA-IOT, WIA-IDENTITY, WIA-BLOCKCHAIN.
