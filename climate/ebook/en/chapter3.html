Chapter 3: API Endpoints

"APIs are the arteries of the digital ecosystem. Well-designed endpoints enable climate data to flow where it's needed most."

3.1 API Overview

The WIA Climate API provides RESTful HTTP endpoints for climate data ingestion, querying, and management. The API follows OpenAPI 3.0 specifications and supports both synchronous and asynchronous operations.

3.1.1 Base URL

https://api.wia.live/climate/v1

3.1.2 Authentication

All API requests require authentication using API keys or JWT tokens:

Authorization: Bearer <your_api_key>

3.1.3 Standard Response Format

{
  "success": true,
  "data": { ... },
  "meta": {
    "timestamp": "2024-12-14T12:00:00Z",
    "request_id": "req_abc123"
  }
}

3.2 Data Ingestion Endpoints

3.2.1 Submit Single Climate Message

POST /data

Description: Submit a single climate data message.

Request Body:

{
  "version": "1.0.0",
  "type": "carbon_capture",
  "timestamp": { ... },
  "location": { ... },
  "device": { ... },
  "data": { ... }
}

Response (201 Created):

{
  "success": true,
  "data": {
    "message_id": "msg_abc123",
    "status": "accepted",
    "validation": {
      "passed": true,
      "quality_score": 0.98
    }
  }
}

3.2.2 Submit Batch Climate Messages

POST /data/batch

Description: Submit multiple climate messages in a single request (max 1000 messages).

Request Body:

{
  "messages": [
    { /* message 1 */ },
    { /* message 2 */ },
    ...
  ]
}

Response (202 Accepted):

{
  "success": true,
  "data": {
    "batch_id": "batch_xyz789",
    "accepted": 995,
    "rejected": 5,
    "errors": [
      {
        "index": 42,
        "reason": "Invalid latitude value"
      }
    ]
  }
}

3.3 Data Query Endpoints

3.3.1 Query Climate Data

GET /data

Query Parameters:

Parameter Type Description Example
type string Filter by data type carbon_capture
start_time ISO8601 Start of time range 2024-12-01T00:00:00Z
end_time ISO8601 End of time range 2024-12-31T23:59:59Z
bbox string Bounding box (min_lon,min_lat,max_lon,max_lat) -22,63,-20,65
device_id string Filter by device ORCA-001
limit integer Max results (1-1000) 100
offset integer Pagination offset 0

Example Request:

GET /data?type=carbon_capture&start_time=2024-12-01T00:00:00Z&limit=50

Response:

{
  "success": true,
  "data": {
    "messages": [ ... ],
    "pagination": {
      "total": 5420,
      "limit": 50,
      "offset": 0,
      "has_more": true
    }
  }
}

3.3.2 Get Data by ID

GET /data/{message_id}

Description: Retrieve a specific climate message by its ID.

3.4 Device Management Endpoints

3.4.1 Register Device

POST /devices

Request Body:

{
  "device_id": "ORCA-001",
  "manufacturer": "Climeworks",
  "model": "Orca DAC",

    "latitude": 64.0,

3.4.2 List Devices

GET /devices

Query Parameters: type, manufacturer, active_only

3.4.3 Update Device

PUT /devices/{device_id}

3.4.4 Get Device Telemetry

GET /devices/{device_id}/telemetry

Description: Retrieve recent data from a specific device.

3.5 Real-time Streaming Endpoints

3.5.1 WebSocket Connection

WS /stream

URL: wss://api.wia.live/climate/v1/stream

Connection Example:

const ws = new WebSocket('wss://api.wia.live/climate/v1/stream?auth=token');

ws.onmessage = (event) => {
  const message = JSON.parse(event.data);
  console.log('Climate data received:', message);
};

// Subscribe to specific data types
ws.send(JSON.stringify({
  action: 'subscribe',
  filters: {
    type: 'carbon_capture',
    bbox: '-22,63,-20,65'
  }
}));

3.5.2 Server-Sent Events (SSE)

GET /stream/sse

Description: Subscribe to climate data updates via Server-Sent Events.

GET /stream/sse?type=carbon_capture

event: data
data: {"type":"carbon_capture","data":{...}}

event: data
data: {"type":"carbon_capture","data":{...}}

3.6 Analytics & Aggregation Endpoints

3.6.1 Time-Series Aggregation

GET /analytics/timeseries

Query Parameters:

Parameter Description Values
type Data type carbon_capture, vertical_farming, etc.
metric Metric to aggregate capture_rate_kg_per_hour, temperature_celsius
aggregation Aggregation function avg, sum, min, max, count
interval Time bucket size 1h, 1d, 1w, 1M
start_time Start time ISO8601
end_time End time ISO8601

Example:

GET /analytics/timeseries?type=carbon_capture&metric=capture_rate_kg_per_hour&aggregation=avg&interval=1d&start_time=2024-12-01T00:00:00Z&end_time=2024-12-31T23:59:59Z

Response:

{
  "success": true,
  "data": {
    "metric": "capture_rate_kg_per_hour",
    "aggregation": "avg",
    "interval": "1d",
    "series": [
      {
        "timestamp": "2024-12-01T00:00:00Z",
        "value": 127.3
      },
      {
        "timestamp": "2024-12-02T00:00:00Z",
        "value": 129.8
      }
    ]
  }
}

3.6.2 Geospatial Aggregation

GET /analytics/geospatial

Description: Aggregate data by geographic regions (grid cells, countries, custom polygons).

3.6.3 Device Performance

GET /analytics/devices/{device_id}/performance

Description: Get performance statistics for a specific device over time.

3.7 Command & Control Endpoints

3.7.1 Send Device Command

POST /devices/{device_id}/commands

Request Body:

{
  "action": "set_capture_rate",
  "parameters": {
    "target_rate_kg_per_hour": 150.0
  },
  "timeout_seconds": 30
}

Response:

{
  "success": true,
  "data": {
    "command_id": "cmd_123",
    "status": "pending",
    "submitted_at": "2024-12-14T12:00:00Z"
  }
}

3.7.2 Get Command Status

GET /commands/{command_id}

Response:

{
  "success": true,
  "data": {
    "command_id": "cmd_123",
    "status": "completed",
    "result": {
      "new_capture_rate": 150.0,
      "confirmation": "Rate adjusted successfully"
    },
    "completed_at": "2024-12-14T12:00:15Z"
  }
}

3.8 Error Responses

3.8.1 Standard Error Format

{
  "success": false,
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "Invalid latitude value",
    "details": {
      "field": "location.latitude",
      "value": 100.0,
      "constraint": "must be between -90 and 90"
    },
    "request_id": "req_abc123"
  }
}

3.8.2 HTTP Status Codes

Code Meaning When Used
200 OK Successful GET request
201 Created Successful POST (resource created)
202 Accepted Request accepted for async processing
400 Bad Request Invalid request format or parameters
401 Unauthorized Missing or invalid authentication
403 Forbidden Authenticated but not authorized
404 Not Found Resource doesn't exist
422 Unprocessable Entity Valid format but semantic errors
429 Too Many Requests Rate limit exceeded
500 Internal Server Error Server-side error
503 Service Unavailable Temporary maintenance or overload
