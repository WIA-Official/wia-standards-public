기본 메시지 구조

이 챕터는 WIA 기후 데이터 형식 표준의 핵심 개념을 다룹니다.

2.1 범용 메시지 형식

{
  "version": "1.0.0",
  "type": "<data_type>",
  "timestamp": {
    "unix_ms": <milliseconds>,
    "iso8601": "<ISO 8601 string>"
  },
  "location": {
    "latitude": <degrees>,
    "longitude": <degrees>,
    "altitude_m": <meters>
  },
  "device": {
    "manufacturer": "<string>",
    "model": "<string>",
    "serial": "<string>"
  },
  "data": {
    // 영역별 고유 데이터
  },
  "meta": {
    "quality_score": <0.0-1.0>
  }
}

2.2 탄소 포집 스키마

필드타입단위설명
technologystring-포집 기술: "dac", "post_combustion" 등
capture_rate_kg_per_hournumberkg/hCO2 포집 속도
co2_purity_percentagenumber%포집된 CO2 순도
energy_consumption_kwhnumberkWh에너지 소비량

2.3 기상 제어 스키마

기상 제어 작업에는 구름 씨뿌리기, 안개 제거 및 강수 증가가 포함됩니다.

2.4 지구공학 스키마

지구공학 개입은 태양복사 관리 또는 이산화탄소 제거를 통해 지구 온난화에 대응합니다.

2.5 수직 농장 스키마

수직 농장은 제어 환경 농업을 사용하여 최소한의 토지와 물로 실내에서 작물을 재배합니다.

2.6 해양 정화 스키마

해양 정화 작업은 해양 환경에서 플라스틱 및 기타 쓰레기를 제거합니다.

2.7 기후 모델 스키마

기후 모델은 다양한 시나리오에서 미래 기후 조건의 예측을 생성합니다.
