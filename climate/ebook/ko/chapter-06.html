6장: Phase 3 — 프로토콜

장 개요: 데이터 평면 — WMO WIS 2.0 MQTT 5 브로커, Kafka 3 exactly‑once, NetCDF over HTTPS 바이트 범위, OGC SensorThings 1.1, TLS 1.3 + DPoP + Ed25519 — 를 한자리에 정렬한다.

6.1 WMO WIS 2.0 MQTT 브로커

WIS 2.0은 2023년부터 GTS를 대체한 운영 시스템이다. 글로벌 정보 시스템 센터(DWD·NCAR·KMA 등)가 MQTT 5 브로커를 운영하며, 구독자는 WNM(WIS2 Notification Message) JSON을 받고 그 안의 링크로 실제 데이터를 가져온다.

// KMA의 지상 SYNOP 통보 구독
mosquitto_sub -h gisc.kma.go.kr -p 8883 \
  --cafile wmo-ca.pem --cert client.crt --key client.key \
  -t "origin/a/wis2/+/data/core/weather/surface-based-observations/synop/#"

// WNM 페이로드
{
  "id":"urn:wmo:md:kma:weather-obs:2026-04-25T12:00:00Z",
  "conformsTo":["https://wis.wmo.int/spec/wnm/1/conf/core"],
  "type":"Feature",
  "geometry":{"type":"Point","coordinates":[126.98,37.57]},
  "properties":{
    "data_id":"weather/surface-based-observations/synop/2026-04-25T12:00:00Z/47108.bufr",
    "datetime":"2026-04-25T12:00:00Z",
    "pubtime":"2026-04-25T12:01:18Z",
    "wmo:dataPolicy":"core",
    "integrity":{"method":"sha512","value":"…"}
  },
  "links":[
    {"rel":"canonical","href":"https://gisc.kma.go.kr/data/…/47108.bufr","type":"application/x-bufr"}
  ]
}

6.2 표준이 의존하는 MQTT 5 기능

  • Session Expiry Interval — 규제기관 대시보드 같은 장기 구독자가 일시 단절을 견딘다.
  • Payload Format Indicator — JSON WNM과 바이너리 BUFR 빠른 경로를 구분.
  • Subscription Identifier — 한 클라이언트가 수백 개 토픽 필터를 다중화.
  • Shared Subscription — $share/group/topic으로 컨슈머 파드 부하 분산.
  • AUTH 패킷 + Reason Codes — 강화 인증 실패 사유가 게이트웨이 로그에 그대로.

6.4 Kafka 3 exactly‑once 인벤토리 스트리밍

사업장 미터·연료 인수·공정 측정은 Kafka 3 exactly‑once 3법칙으로 흐른다.

  1. 프로듀서 enable.idempotence=true (3.0 이후 기본값).
  2. 컨슈머 isolation.level=read_committed.
  3. 안정적인 transactional.id를 가진 트랜잭션 프로듀서.
// 인벤토리 ingest 이벤트
{
  "schemaId": "wia.climate.ingest.v1",
  "facilityLei": "213800ABCDEFGHIJKL00",
  "meterId": "POH-CEMS-7",
  "ts": "2026-04-25T12:00:00+09:00",
  "channel": "co2_concentration_dry_corrected",
  "value": 12.4,
  "unit": "%v/v",
  "quality": {"validation":"ok","calibrationDue":"2026-09-12"},
  "lineage": {"sourceFile":"s3://kr-cems/poh-7/2026-04-25.csv","row":34812}
}

6.5 NetCDF over HTTPS 바이트 범위

대용량 재분석(ERA5, JRA‑3Q)을 위해 HTTP/2 chunked transfer + 바이트 범위 fetch 사용. netCDF‑c 4.9의 byterange 드라이버로 원격 파일을 로컬처럼 연다.

import netCDF4 as nc
url = "https://archive.wia-climate.org/era5/single-level/t2m/2025/era5_t2m_2025.nc"
ds = nc.Dataset(url + "#mode=bytes")
seoul = ds.variables["t2m"][:, 280, 530]   # 한 격자, 8760시간, <10MB 다운로드

6.7 변조 불가능 앵커링

30초 단위 ingest 윈도와 OGC API 쓰기마다 머클 루트가 WIA Chain PoA → BSC 메인넷에 앵커링된다. 외부 감사인은 데이터 본문을 보지 않고도 시점 무결성을 검증한다.

6.8 보안 베이스라인

6.9 프로토콜 계층 실패 모드

  • 브로커 다운 — 24시간 로컬 버퍼링 + 원본 ts 리플레이.
  • 지연 관측 — cache/... 토픽 접두사로 재발행.
  • 시계 드리프트 — 6시간 NTP, |now − ts| > 600 s 거절(기후 윈도는 물류보다 느리다).
  • 리플레이 공격 — RFC 9421 nonce + JWT JTI 캐시.
  • 스키마 진화 — Kafka subject schema registry, 후방 호환 강제.

장 요약

핵심 요점

  1. WIS 2.0 MQTT는 인바운드 관측 파이프라인이며 표준은 OGC API로 재발행한다.
  2. Kafka 3 idempotent + transactional이 인벤토리 exactly‑once를 만든다.
  3. NetCDF/HTTPS 바이트 범위가 다국 TB 아카이브를 다운로드 없이 연다.
  4. OGC SensorThings 1.1이 CEMS 등 사업장 센서 인입을 표준화한다.
  5. WIA Chain PoA + BSC 머클 루트 앵커링이 외부 감사 가능성을 만든다.
