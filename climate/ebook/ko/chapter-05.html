5장: Phase 2 — API 인터페이스

장 개요: 4개의 동등 normative façade(OGC API Features/Coverages/EDR/Records + STAC 1.0)를 묶고, IFRS S2 트윈, UNFCCC ETF 제출, CloudEvents 1.0 웹훅까지 호출 단위로 펼친다.

5.1 4 façade, 같은 진실

Façade경로활용
OGC API Features/v1/collections/{id}/items벡터 — 사업장 폴리곤, 프로젝트 경계, 협약 영역
OGC API Coverages/v1/collections/{id}/coverage?bbox=…&subset=time(…)CF NetCDF/Zarr 부분집합
OGC API EDR/v1/collections/{id}/position?coords=…점·궤적 시계열
OGC API Records (+ STAC 1.0)/v1/collections카탈로그·검색·구독

conformance suite는 동등 시나리오를 4 façade로 동시 실행해 결과 해시를 비교한다.

5.3 OGC API Coverages — 주력

GET /v1/collections/cmip6.scenariomip.kma-k-ace.ssp245.tas/coverage
   ?bbox=124,33,131,39
   &subset=time("2050-01-01"/"2099-12-31")
   &f=netcdf
Authorization: Bearer $TOKEN

→ HTTP/1.1 200 OK
   Content-Type: application/x-netcdf;version=4
   Content-Disposition: attachment; filename="tas_…20500101-20991231.nc"
   ETag: "W/\"sha256-3a2f…\""

출력 형식은 f 쿼리 또는 Accept로 협상: netcdf, zarr, covjson, geotiff.

장 요약

핵심 요점

  1. 4 OGC API façade + STAC가 동치성을 가져야 conformance를 통과한다.
  2. Idempotency‑Key + ETag/If‑Match가 네트워크 단절·동시 갱신을 안전하게 만든다.
  3. RFC 9457 problem+json은 클라이언트 처리의 안정 식별 키다.
  4. 공시는 한 번 제출 → JSON‑LD와 iXBRL 두 형식으로 동시 노출.
  5. UNFCCC ETF/CRT 2.0 BTR 제출은 1급 엔드포인트로, NDA 키로 서명한다.
