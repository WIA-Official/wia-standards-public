3.2 3대 핵심 원칙

3.2.1 상호운용을 우선으로

식별자·단위·좌표·코드는 외부 권위에서 비롯된다 — IPCC AR6, UNFCCC, WMO, OGC, ISO, IFRS, GHG Protocol, UCUM. 표준은 코드 리스트를 만들지 않는다, 기존 권위를 버전 핀과 함께 조합한다. 사업장은 LEI(ISO 17442), 화학종은 CAS, 단위는 UCUM, 좌표는 OGC CRS84.

3.2.2 검증가능성을 기본값으로

모든 1차 측정에는 해시, 모든 집계에는 입력 머클 루트, 모든 공시에는 EdDSA‑JCS‑2022 서명(W3C Data Integrity Proofs 1.0). 검증인은 서명 검증과 링크 추적이 본업이 된다 — 스프레드시트 정합 맞추기가 아니다.

3.2.3 기후 정의(climate justice)를 설계 제약으로

최저개발국 배출량을 체계적으로 과소산정하거나 산림 기준선을 과다산정하는 방법론은 표준 위반이다. 표준은 1850년 이래 누적 배출량(AR6 WG‑III 2.5절), OECD CRS 41081/41082 적응·완화 자금 흐름, 토지 프로젝트의 원주민 영토 중첩(LandMark Global Platform 기반)을 명시 보고하도록 요구한다.

3.4 핵심 컴포넌트

기상 관측 파이프라인

WIS 2.0 브로커 구독으로 CF‑1.10 NetCDF 그래뉼이 도착. STAC 1.0 아이템으로 감싸 OGC API Records로 노출, CloudEvents 1.0 웹훅(RFC 9421 서명)으로 통보.

기후 모델 출력 서비스

CMIP6/ScenarioMIP/DCPP/CORDEX‑EAS를 Zarr v3 스토어에 거울 복제, DRS 명명 일관성 유지. OGC API Coverages가 임의 부분집합(bbox·time·SSP)을 서비스.

기업 GHG 인벤토리

사업장 미터·연료 인수·공정 배출이 JSON‑LD 레코드로 기록. GHG Protocol scope 태그·ISO 14064‑1 분류·UCUM 단위. 법인(LEI)·기업집단으로 롤업.

공시 트윈

단일 진실원에서 iXBRL(IFRS S2/ESRS E1)과 JSON‑LD 트윈을 생성한다 — 규제기관 기계 소비와 감사인 도구가 같은 출처를 본다.

탄소 등록부 인터페이스

Verra·Gold Standard·ACR·CAR 발행/회수 이벤트가 W3C VC로 통일. corresponding adjustment 상태가 1급 시민이다.

국가 보고 브리지

JSON‑LD 국가 인벤토리를 UNFCCC CRT 2.0 스키마로 매핑, BTR JSON 봉투를 NDA 키로 서명해 제출.

3.6 WIA 생태계와의 결합

  • WIA‑VC — 검증 보고서·기준선 연구·3자 보증서 발급.
  • WIA‑BLOCKCHAIN — 인벤토리 집계 머클 루트를 WIA Chain PoA + BSC에 앵커링.
  • WIA‑IDENTITY — 검증인·NDA·프로젝트 개발자 DID.
  • WIA‑IOT — CEMS·스마트미터·연료 유량 등 사업장 센서 온보딩.
  • WIA‑CARBON — Scope 3 운송 ISO 14083:2023 자동.
  • WIA‑AGRI‑014 — 농산물 콜드체인 Scope 3 핸드오프.

3.7 “표준”의 단 하나의 시험

본 표준을 구현한 시스템의 출력은 다른 구현자의 검증인이 사전 조율 없이도, 본 책과 인용 권위만 보고 같은 결론에 닿을 수 있어야 한다. 모르는 사람이 같은 답을 도출할 수 있는가 — 이 한 가지가 표준의 궁극 시험이다.

장 요약

핵심 요점

  1. 5계층(데이터→프로토콜→API→응용→공시)이 단일 책임으로 분리되며 phase n+1은 phase n 위에서만 동작한다.
  2. 상호운용·검증가능·기후정의는 협상 불가능한 설계 제약이다.
  3. 여섯 컴포넌트(관측·모델·인벤토리·공시·등록부·국가보고)가 단일 식별자/단위 시스템을 공유한다.
  4. 레퍼런스 기술은 오픈소스이며, 대안은 동등한 conformance 통과를 요구한다.
  5. WIA‑VC, WIA‑IOT, WIA‑AGRI‑014, WIA‑CARBON, WIA‑BLOCKCHAIN, WIA‑IDENTITY와 매끄럽게 결합한다.
