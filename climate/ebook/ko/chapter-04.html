4장: Phase 1 — 데이터 형식

장 개요: CF‑1.10 NetCDF로 격자 데이터, Zarr v3로 클라우드 큐브, JSON‑LD로 GHG 인벤토리·공시, GeoJSON/CRS84로 벡터, UCUM으로 단위 — 와이어 바이트의 기본을 깐다.

4.1 CF‑1.10 NetCDF

모든 격자 필드는 NetCDF‑4 + CF Conventions 1.10(2024년 발표)을 사용한다. 변수명·좌표축·단위·메타데이터가 표준화돼 있어 일반 리더가 벤더 매뉴얼 없이 온도장을 그릴 수 있다.

// ncdump -h tas_day_KMA-K-ACE_ssp245_r1i1p1f1_gn_20260101-20261231.nc
netcdf tas_day_KMA-K-ACE_ssp245_r1i1p1f1_gn_20260101-20261231 {
dimensions:
    time = UNLIMITED ; lat = 192 ; lon = 288 ;
variables:
    float tas(time, lat, lon) ;
        tas:standard_name = "air_temperature" ;
        tas:units         = "K" ;
        tas:cell_methods  = "time: mean" ;
    double time(time) ;
        time:standard_name = "time" ;
        time:units         = "days since 1850-01-01 00:00:00" ;
        time:calendar      = "proleptic_gregorian" ;

// global attributes:
    :Conventions     = "CF-1.10" ;
    :institution     = "기상청 / 국립기상과학원" ;
    :source          = "K-ACE 1.0" ;
    :experiment_id   = "ssp245" ;
    :variant_label   = "r1i1p1f1" ;
    :tracking_id     = "hdl:21.14100/abcdef-1234" ;
    :wia_compliance  = "WIA-CLIMATE-022:2026" ;
    :wia_signature   = "ed25519:wia:climate:kma-k-ace-2026#k1" ;
}

standard_name 값은 CF Standard Name Table v84(2024.06) 기준. 표에 없는 이름은 CF 커뮤니티 제안을 거쳐야 한다.

4.3 Zarr v3 클라우드 큐브

NetCDF‑4는 로컬 디스크에서 빠르지만 객체 저장소에서는 비싸다. WIA‑CLIMATE‑022는 모든 NetCDF 그래뉼을 Zarr v3로 거울 복제(전형 청크 {time:30, lat:192, lon:288}). 30TB 스토어에서 바운딩 박스 한 개를 밀리초에 읽는다.

import xarray as xr
ds = xr.open_zarr(
    "s3://wia-climate/cmip6/ScenarioMIP/KMA/K-ACE/ssp245/r1i1p1f1/Amon/tas/gn/v20260315",
    consolidated=True,
)
seoul = ds.tas.sel(lat=37.57, lon=126.98, method="nearest").sel(time=slice("2050","2099"))

4.6 좌표 참조

모든 좌표는 OGC CRS84(WGS 84 경위도, 십진도). 한국 통합 TM 좌표는 EPSG:5179. 폴리곤은 GeoJSON Feature(RFC 7946), 격자는 CF grid_mapping으로 CRS 선언.

4.7 검증 규칙

  • NetCDF는 cf-checker를 CF 1.10에 대해 무경고로 통과해야 한다.
  • JSON‑LD는 RFC 8785(JCS) 정규화 + 서명 라운드트립이 가능해야 한다.
  • UCUM 단위 문자열은 libucum으로 파싱돼야 한다.
  • 좌표는 CRS84 라운드트립, 정밀도 7자리 이하.
  • 시간은 ISO 8601‑1, 명시 calendar 속성(proleptic_gregorian 기본).

장 요약

핵심 요점

  1. CF‑1.10 NetCDF(+Zarr v3 거울)는 격자 데이터의 유일한 와이어 형식이다.
  2. DRS 파일명이 변수·모델·시나리오·앙상블·격자·기간의 출처를 담는다.
  3. JSON‑LD + 핀된 @context가 인벤토리·공시의 유일한 와이어 형식이다.
  4. UCUM이 단위 시스템이다 — 기호만 있는 “tCO₂e”는 거절된다.
  5. OGC CRS84 + GeoJSON + EPSG 코드(5179 등)가 모든 공간 기하를 다룬다.
