04장: 프로토콜 사양

"이 챕터는 영문 버전에서 번역되었습니다. 전체 내용은 영문 버전을 참조하십시오."

프로토콜 아키텍처

이 챕터는 WIA 기후 데이터 형식 표준의 핵심 개념을 다룹니다.

4.1 프로토콜 아키텍처

WIA 기후 프로토콜은 전송 계층(WebSocket, MQTT, HTTP) 위에 있는 애플리케이션 수준 프로토콜입니다.

4.2 프로토콜 메시지 형식

{
  "protocol": "wia-climate",
  "version": "1.0.0",
  "messageId": "uuid",
  "timestamp": 1702483200000,
  "type": "data",
  "payload": { ... }
}

4.3 연결 수명 주기

연결 시퀀스: connect → connect_ack → 데이터 교환 → disconnect

4.4 전송 계층 지원

  • WebSocket: wss://api.wia.live/climate/v1/ws
  • MQTT: mqtts://mqtt.wia.live:8883
  • HTTP: https://api.wia.live/climate/v1
  • Serial: 로컬 IoT 장치용

4.5 서비스 품질 (QoS)

QoS 레벨보장사용 사례
0최대 한 번고빈도 센서 스트림
1최소 한 번중요 이벤트, 명령
2정확히 한 번중요 제어 명령
