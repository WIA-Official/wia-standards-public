Phase 1: 데이터 형식과 이야기 온톨로지

WIA-HERITAGE-008: Folklore Archive Standard

04.1 JSON 스키마 설계 원칙

Phase 1은 WIA-HERITAGE-008의 기반으로서 구전 전통을 디지털 형식으로 인코딩하는 방법을 정의합니다. JSON(JavaScript Object Notation)을 주요 형식으로 선택한 이유는 인간이 읽을 수 있고, 모든 프로그래밍 언어가 지원하며, 웹 API와 완벽하게 호환되기 때문입니다.

설계 철학

  • 단순성: 비기술자도 텍스트 에디터로 편집 가능
  • 확장성: 새 필드 추가 시 기존 데이터 호환
  • 검증 가능: JSON Schema로 자동 유효성 검사
  • 다국어 지원: UTF-8로 7,000개 언어 모두 표현
  • 타입 안전성: 명시적 데이터 타입으로 오류 방지
📝 최소 유효 Story 객체 예시
{
  "@context": "https://wia.org/heritage-008/v1",
  "@type": "FolkStory",
  "id": "story:kr:jeju:001",
  "title": "설문대할망 천지개벽",
  "language": "jje",
  "dateRecorded": "2024-03-15",
  "narrator": {
    "name": "김영희",
    "birthYear": 1942,
    "community": "제주시 조천읍"
  },
  "transcription": "엣날엣날 설문대할망이..."
}

이 최소 객체만으로도 유효한 WIA-HERITAGE-008 데이터이며, 필요에 따라 수십 개의 선택적 필드를 추가할 수 있습니다.

04.2 Aarne-Thompson-Uther 모티프 색인

ATU(Aarne-Thompson-Uther) 색인은 민담을 분류하는 국제 표준입니다. 핀란드 민속학자 안티 아르네가 1910년 개발했고, 스티스 톰프슨이 1928년 확장했으며, 한스-외르크 우터가 2004년 개정했습니다. 현재 2,500개 이상의 이야기 유형을 분류합니다.

ATU 코드유형설명한국 예시
ATU 300-749마법 이야기초자연적 요소가 있는 이야기선녀와 나무꾼 (ATU 400)
ATU 750-849종교 이야기신, 성자, 악마 이야기석가모니 전생담 (ATU 759)
ATU 850-999신부 이야기영리한 신부가 문제 해결혹부리 영감 (ATU 503)
ATU 1000-1199바보 이야기어리석은 사람들의 행동바보 온달 (ATU 1540)
ATU 1200-1999우화와 농담짧고 재치있는 이야기토끼와 거북이 (ATU 275A)
ATU 2000-2399연쇄담반복 구조의 이야기콩쥐 팥쥐 (ATU 510A)

JSON 스키마에서 ATU 표현

{
  "motifs": [
    {
      "code": "ATU 400",
      "title": "The Man on a Quest for His Lost Wife",
      "confidence": 0.95,
      "notes": "한국 버전은 날개옷 모티프가 핵심"
    },
    {
      "code": "D361.1",
      "title": "Swan maiden",
      "type": "Thompson Motif",
      "confidence": 0.87
    }
  ]
}

04.3 다국어 텍스트 인코딩

구전 전통의 가장 큰 도전은 언어의 다양성입니다. 같은 이야기라도 원어, 방언, 번역, 주석이 모두 필요합니다. WIA-HERITAGE-008은 이를 위한 다층 텍스트 모델을 제공합니다.

언어 코드 체계

ISO 639-3 표준을 사용하여 7,900개 언어를 3글자 코드로 표현합니다. 이는 ISO 639-1(2글자, 184개 언어만 지원)보다 훨씬 포괄적입니다.

언어ISO 639-3ISO 639-1비고
한국어korko표준어
제주어jje없음별도 언어로 분류
아이누어ain없음소멸 위기
나바호어navnv북미 원주민
스와힐리어swasw동아프리카

다층 텍스트 구조

{
  "text": {
    "original": {
      "language": "jje",
      "content": "엣날엣날 설문대할망이 천지를 창조햇주",
      "ipa": "ennal ennal sʌlmundɛ halmɑŋi...",
      "notes": "제주 북부 방언"
    },
    "standardized": {
      "language": "kor",
      "content": "옛날 옛날 설문대할망이 천지를 창조했습니다"
    },
    "translations": [
      {
        "language": "eng",
        "content": "Long long ago, Grandmother Seolmundae created heaven and earth",
        "translator": "Park Minho",
        "date": "2024-04-10"
      },
      {
        "language": "jpn",
        "content": "むかしむかし、ソルムンデおばあさんが天地を創造しました"
      }
    ],
    "glosses": [
      {
        "original": "엣날",
        "literal": "old-day",
        "meaning": "long ago",
        "grammatical": "adverb"
      }
    ]
  }
}

04.4 오디오 메타데이터와 타임스탬프

오디오는 구전 전통의 핵심입니다. 목소리의 억양, 리듬, 감정은 텍스트로 담을 수 없습니다. WIA-HERITAGE-008은 오디오 파일에 대한 풍부한 메타데이터를 정의합니다.

오디오 기술 사양

항목아카이브 버전스트리밍 버전
포맷FLAC, WAVOpus, AAC
샘플레이트48kHz48kHz
비트뎁스24-bit16-bit
비트레이트무손실64kbps (음성 최적화)
채널Stereo/MonoMono
파일크기 (1시간)~600MB~28MB

타임스탬프 주석

이야기의 특정 순간에 주석을 달 수 있습니다. 이는 팟캐스트의 챕터 마커와 유사하지만 훨씬 세밀합니다.

{
  "audio": {
    "files": [
      {
        "type": "archive",
        "format": "flac",
        "url": "https://archive.wia.org/story-kr-jeju-001.flac",
        "duration": 1847,
        "checksum": "sha256:a3f2..."
      }
    ],
    "annotations": [
      {
        "timestamp": 42.5,
        "type": "narrative_structure",
        "label": "서사 시작 (Complication)",
        "note": "할망이 문제에 직면"
      },
      {
        "timestamp": 156.2,
        "type": "cultural_reference",
        "label": "제주 방언 '영'",
        "explanation": "표준어 '아주'의 뜻"
      },
      {
        "timestamp": 423.8,
        "type": "audience_reaction",
        "label": "청중 웃음",
        "duration": 3.2
      }
    ]
  }
}

04.6 지리 정보와 분포 매핑

민담은 특정 지역에서 발생하고 전파됩니다. 지리 정보를 기록하면 이야기의 이동 경로를 시각화할 수 있습니다.

지리 유형설명예시
발생지이야기가 실제 일어난 곳한라산 백록담
구연지이번 구연이 녹음된 곳조천읍 마을회관
전승 범위이야기가 알려진 지역제주도 전역
변형 지역비슷한 이야기가 있는 곳울릉도, 독도

GeoJSON 형식으로 지도에 바로 표시할 수 있습니다.

04.7 문화적 맥락과 접근 제어

일부 이야기는 문화적으로 민감하여 제한된 접근만 허용해야 합니다. Phase 1은 이를 위한 메타데이터를 정의합니다 (실제 접근 제어는 Phase 2 API에서 구현).

04.8 장 요약

✅ 핵심 내용
  1. JSON 스키마: 단순하면서도 확장 가능한 데이터 형식, UTF-8로 7,000개 언어 지원
  2. ATU 색인: 2,500개 이야기 유형 분류, Thompson Motif Index 통합
  3. 다국어 텍스트: 원어, 표준화, 번역, 주석, IPA 발음 기호를 다층 구조로 표현
  4. 오디오 메타데이터: FLAC 아카이브 + Opus 스트리밍, 타임스탬프 주석으로 세밀한 분석
  5. 구연자 프로필: 배경, 레퍼토리, 전승 이력 상세 기록
  6. 접근 제어: 6단계 민감도 레벨로 문화적 보호와 학술 개방 균형
弘益人間

데이터 형식은 기술이지만, 그 안에 담긴 것은 인류의 영혼입니다

04.9 복습 문제

  1. JSON을 WIA-HERITAGE-008의 주요 형식으로 선택한 5가지 이유를 설명하세요.
  2. ATU 색인의 6가지 주요 카테고리를 나열하고, 각각의 한국 민담 예시를 제시하세요.
  3. 다층 텍스트 모델에서 original, standardized, translations, glosses의 역할과 차이점을 설명하세요.
  4. 오디오 아카이브 버전과 스트리밍 버전의 기술 사양 차이와 그 이유를 설명하세요.
  5. 구연자 프로필에 repertoire를 기록하는 것이 왜 중요한지 논하세요.
  6. 문화적 접근 제어 6단계 레벨을 설명하고, 각 레벨의 실제 사용 사례를 제시하세요.
