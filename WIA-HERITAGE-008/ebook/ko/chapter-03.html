표준 개요 및 전체 아키텍처

WIA-HERITAGE-008: Folklore Archive Standard

03.1 WIA-HERITAGE-008 아키텍처 개요

WIA-HERITAGE-008은 모듈형 레이어드 아키텍처를 채택하여 각 구성요소가 독립적으로 진화하면서도 전체적으로 조화롭게 작동하도록 설계되었습니다. 이는 소프트웨어 공학의 "관심사의 분리(Separation of Concerns)" 원칙에 기반합니다.

표준의 전체 구조는 OSI 7계층 모델처럼 4개 레이어로 나뉘며, 각 레이어는 명확한 책임과 인터페이스를 가집니다. 하위 레이어는 상위 레이어의 기반을 제공하고, 상위 레이어는 하위 레이어의 복잡성을 추상화합니다.

레이어Phase핵심 기능의존성
통합 계층Phase 4AI, VR, 인터랙티브 체험Phase 1~3 모두
프로토콜 계층Phase 3네트워크 통신, 스트리밍Phase 1~2

데이터 계층Phase 1형식, 스키마, 메타데이터없음 (기반)

설계 철학

아키텍처 설계 시 다음 원칙을 따랐습니다:

  • 점진적 채택: Phase 1만 구현해도 즉시 가치 제공
  • 하위 호환성: 상위 버전이 하위 버전 데이터를 항상 읽을 수 있음
  • 확장성: 새로운 기능 추가 시 기존 시스템에 영향 최소화
  • 상호운용성: 다른 표준(Dublin Core, TEI, IIIF)과 완전 호환
  • 개방성: 모든 사양은 MIT 라이선스로 공개

03.2 Phase 1: 데이터 형식 계층

Phase 1은 표준의 기초로서 구전 전통을 디지털로 표현하는 데이터 모델을 정의합니다. 이는 "민속학의 DNA"라 할 수 있는 핵심 개념들을 컴퓨터가 이해할 수 있는 형식으로 인코딩합니다.

핵심 데이터 엔티티

엔티티설명필수 속성
Story이야기 본체title, language, text/audio, date_recorded
Narrator구연자 정보name, birth_year, community, languages
Performance구연 이벤트narrator, story, location, audience, duration
Motif서사 모티프ATU_code, description, variants
Community문화 공동체name, location, language, traditions
Collection이야기 모음name, curator, license, stories[]

메타데이터 스키마

WIA-HERITAGE-008은 Dublin Core를 기본으로 하고 Folklore 전용 확장을 추가합니다. JSON-LD 형식으로 시맨틱 웹과 호환되며, Schema.org의 CreativeWork 타입을 확장합니다.

📋 표준 메타데이터 필드 (32개)
  • Dublin Core 15개: title, creator, subject, description, publisher, contributor, date, type, format, identifier, source, language, relation, coverage, rights
  • Folklore 확장 17개: narrator, performance_date, recording_quality, transcription_method, ATU_index, cultural_context, sacred_level, access_restrictions, original_language, dialect, performance_duration, audience_type, accompanying_gestures, musical_elements, cultural_keywords, geolocation, community_permissions

파일 형식 표준

오디오와 비디오는 개방형 무손실 포맷을 우선하되, 웹 호환성을 위한 압축 버전도 제공합니다.

  • 오디오 아카이브: FLAC (24-bit/48kHz) 또는 WAV
  • 오디오 스트리밍: Opus (64kbps 음성 최적화) 또는 AAC
  • 비디오 아카이브: FFV1 (무손실) 또는 ProRes
  • 비디오 스트리밍: AV1 (4K) 또는 H.265/HEVC
  • 텍스트: TEI XML (학술), Markdown (일반), JSON (데이터 교환)
  • 이미지: TIFF (아카이브), WebP (웹)

03.3 Phase 2: API 서비스 계층

Phase 2는 Phase 1의 데이터에 접근하고 조작하기 위한 프로그래밍 인터페이스를 제공합니다. RESTful API와 GraphQL 두 가지 스타일을 모두 지원하여 개발자가 선호하는 방식을 선택할 수 있습니다.

검색 기능

민속 자료 검색은 일반 문서 검색과 다른 특수성이 있습니다. 같은 이야기의 변형을 찾거나, 유사한 모티프를 가진 다른 전통의 이야기를 발견하는 것이 중요합니다.

  • 전문 검색: Elasticsearch로 68개 언어 형태소 분석
  • 퍼지 검색: 철자 오류, 방언 변형 허용 (Levenshtein distance)
  • 시맨틱 검색: GPT 임베딩으로 의미적 유사도 검색
  • 모티프 검색: ATU 색인으로 서사 구조 패턴 매칭
  • 지리 검색: 특정 지역 반경 내 이야기 검색
  • 오디오 검색: Shazam 스타일 음향 지문으로 멜로디 검색

인증 및 권한 관리

문화적으로 민감한 자료를 보호하기 위해 세분화된 권한 시스템을 구현합니다. OAuth 2.0과 JWT 토큰 기반이며, 커뮤니티가 자신들의 이야기에 대한 접근을 완전히 통제할 수 있습니다.

  • 공개(Public): 누구나 접근 가능
  • 등록(Registered): 계정 생성 후 접근
  • 연구자(Researcher): 학술 기관 인증 필요
  • 커뮤니티(Community): 원주민/공동체 구성원만
  • 장로(Elders): 공동체 지정 장로만
  • 비공개(Private): 특정 개인만

03.4 Phase 3: 프로토콜 계층

Phase 3은 분산된 아카이브 간 데이터 동기화, 실시간 협업, 대용량 미디어 스트리밍을 위한 네트워크 프로토콜을 정의합니다.

핵심 프로토콜

프로토콜용도기술
FolkloreSync아카이브 간 동기화Dat Protocol / IPFS
AudioStream오디오 스트리밍WebRTC + Opus
CollabEdit협업 전사/주석Operational Transform
P2PShare피어 투 피어 배포BitTorrent + WebTorrent
LivePerform실시간 구연 방송WebRTC + SRT

오프라인 동기화

인터넷이 불안정한 지역을 위해 "Offline-First" 프로토콜을 설계했습니다. 모바일 앱에서 로컬에 데이터를 저장하고, 인터넷 접속 시 자동으로 중앙 서버와 동기화합니다.

충돌 해결은 CRDTs(Conflict-free Replicated Data Types)를 사용하여, 여러 사람이 동시에 오프라인으로 편집해도 나중에 자동 병합됩니다.

03.5 Phase 4: 통합 계층

Phase 4는 AI, VR, AR, 게임 등 첨단 기술과 통합하여 구전 전통을 새로운 방식으로 경험하게 합니다.

AI 통합

  • 자동 전사: Whisper AI로 100개 언어 음성→텍스트 변환
  • 자동 번역: M2M100으로 200개 언어 쌍 번역
  • 모티프 분류: GPT-4로 ATU 색인 자동 태깅
  • 이야기 복원: 파손된 녹음의 누락 부분을 AI가 추정
  • 인터랙티브 스토리텔링: 사용자가 챗봇 구연자와 대화
  • 패턴 발견: 수천 개 이야기에서 문화 전파 경로 추적

VR/AR 체험

VR 헤드셋으로 과거로 시간 여행하여 전통 구연 현장에 참석하는 것처럼 체험할 수 있습니다. 1920년대 아프리카 마을 모닥불 앞에서 그리오의 이야기를 듣거나, 중세 유럽 성에서 음유시인의 서사시를 감상할 수 있습니다.

03.6 기존 표준과의 호환성

WIA-HERITAGE-008은 바퀴를 재발명하지 않습니다. 검증된 기존 표준들을 최대한 활용하고 민속학 특화 부분만 추가합니다.

기존 표준용도통합 방식
Dublin Core기본 메타데이터15개 코어 필드 그대로 사용
TEI (Text Encoding Initiative)텍스트 전사TEI P5 스키마 + 민속 모듈
IIIF (International Image Interoperability Framework)이미지/비디오IIIF Presentation API 3.0
JSON-LD링크드 데이터Schema.org 확장
BagIt장기 보존아카이브 패키징
PREMIS보존 메타데이터디지털 보존 이력

상호운용성 시나리오

한국 국립민속박물관의 WIA-HERITAGE-008 아카이브에서 판소리 데이터를 IIIF 형식으로 내보내면, 미국 스미소니언 박물관의 IIIF 뷰어에서 바로 재생할 수 있습니다. 메타데이터는 Dublin Core이므로 일반 도서관 시스템에서도 검색 가능합니다.

03.7 확장성과 미래 대비

기술은 빠르게 변합니다. 2024년의 최신 기술이 2034년에는 구식이 될 수 있습니다. WIA-HERITAGE-008은 이를 대비하여 확장 메커니즘을 내장했습니다.

확장 포인트

  • 커스텀 메타데이터: "x-" 접두사로 기관별 필드 추가 가능
  • 플러그인 시스템: 새로운 미디어 포맷, AI 모델을 플러그인으로 추가
  • 버전 관리: 시맨틱 버저닝으로 하위 호환성 보장
  • 실험 기능: "experimental" 플래그로 표준화 전 기능 테스트

03.9 복습 문제

  1. WIA-HERITAGE-008의 4계층 아키텍처를 설명하고, 각 계층의 의존성 관계를 그려보세요.
