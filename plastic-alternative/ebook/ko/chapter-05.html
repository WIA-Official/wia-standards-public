제5장

API 레퍼런스 및 통합

WIA-PLASTIC_ALTERNATIVE 시스템과 통합하기 위한 완전한 API 문서

API 개요

WIA-PLASTIC_ALTERNATIVE API는 모든 표준 기능에 대한 프로그래밍 방식 접근을 제공하여 엔터프라이즈 시스템, 파트너 플랫폼 및 제3자 애플리케이션과의 원활한 통합을 가능하게 합니다. JSON 페이로드를 사용하는 RESTful 원칙을 기반으로 구축된 API는 통합 복잡성을 최소화하고 구현자의 가치 실현 시간을 단축하기 위해 보안, 버전 관리 및 문서화에 대한 업계 모범 사례를 따릅니다.

API는 플라스틱 대체재 생태계의 핵심 엔터티를 나타내는 핵심 리소스를 중심으로 구성됩니다: 재료, 인증서, 조직, 테스트, 공급업체. 각 리소스는 인증 워크플로우 및 추적성 쿼리와 같은 도메인별 기능을 위한 특수 엔드포인트와 함께 표준 CRUD 작업을 지원합니다.

API 기본 정보

기본 URLhttps://api.wia-plastic.org/v1
프로토콜HTTPS (TLS 1.2+)
형식JSON (application/json)
인증OAuth 2.0 Bearer Token
속도 제한1000 요청/시간 (표준 티어)
페이지네이션커서 기반 (limit/after 매개변수)

인증

API는 서버 간 통합을 위한 클라이언트 자격 증명 흐름과 사용자 위임 액세스를 위한 권한 부여 코드 흐름을 모두 지원하는 OAuth 2.0을 인증에 사용합니다. 모든 요청은 Authorization 헤더에 유효한 Bearer 토큰을 포함해야 합니다.

액세스 토큰 획득

WIA 개발자 포털에서 애플리케이션을 등록하여 클라이언트 자격 증명을 얻습니다. 이러한 자격 증명을 사용하여 권한 부여 서버에서 액세스 토큰을 요청합니다. 

// 토큰 요청 (클라이언트 자격 증명 흐름)
POST https://auth.wia-plastic.org/oauth/token
Content-Type: application/x-www-form-urlencoded

grant_type=client_credentials
&client_id=YOUR_CLIENT_ID
&client_secret=YOUR_CLIENT_SECRET
&scope=materials:read certificates:read

// 응답
{
  "access_token": "eyJhbGciOiJSUzI1NiIsInR5cCI6IkpXVCJ9...",
  "token_type": "Bearer",
  "expires_in": 3600,
  "scope": "materials:read certificates:read"
}

재료 API

재료 API는 WIA-PLASTIC_ALTERNATIVE 시스템에 등록된 재료 정의에 대한 액세스를 제공합니다. 작업에는 재료 정보 검색, 재료 레지스트리 검색, 권한 있는 사용자의 경우 재료 레코드 생성 및 업데이트가 포함됩니다.

재료 목록

GET /materials

지정된 기준과 일치하는 페이지네이션된 재료 목록을 검색합니다.

쿼리 매개변수

매개변수유형설명
categorystring재료 범주별 필터 (biodegradable, bio-based, compostable)
certification_levelstring인증 수준별 필터 (bronze, silver, gold)
searchstring재료 이름 및 설명에 대한 전체 텍스트 검색
limitinteger반환할 최대 결과 수 (기본값: 20, 최대: 100)
afterstring페이지네이션용 커서 (이전 응답에서)

추적성 API

추적성 API는 재료 및 제품의 순추적 및 역추적을 통해 공급망 가시성을 가능하게 합니다. 이러한 엔드포인트는 규정 준수 검증, 리콜 관리 및 지속가능성 보고를 지원합니다.

순추적

GET /traceability/forward/{batch_id}

공급망을 통해 재료 배치를 순추적합니다.

웹훅

API는 통합과 관련된 이벤트의 실시간 알림을 위해 웹훅을 지원합니다. 지정된 이벤트가 발생할 때 HTTP POST 요청을 수신하도록 웹훅 엔드포인트를 구성합니다.

지원되는 이벤트

이벤트 유형설명
certificate.issued새 인증서 발급됨
certificate.expiring인증서 90일 내 만료
certificate.suspended인증서 정지됨
material.updated재료 레코드 업데이트됨
application.status_changed인증 신청 상태 변경
compliance.alert규정 준수 문제 감지됨

SDK 라이브러리

WIA는 API 통합을 간소화하기 위해 인기 있는 프로그래밍 언어에 대한 공식 SDK 라이브러리를 제공합니다. SDK는 인증, 요청 형식화, 오류 처리 및 응답 파싱을 처리하여 개발자가 API 메커닉이 아닌 비즈니스 로직에 집중할 수 있게 합니다.

사용 가능한 SDK

  • TypeScript/JavaScript: npm install @wia/plastic-alternative
  • Python: pip install wia-plastic-alternative
  • Go: go get github.com/wia-official/plastic-alternative-go
  • Java: Maven/Gradle 의존성 사용 가능
  • C#/.NET: NuGet 패키지 사용 가능

TypeScript SDK 예시

import { WIAClient } from '@wia/plastic-alternative';

const client = new WIAClient({
  clientId: process.env.WIA_CLIENT_ID,
  clientSecret: process.env.WIA_CLIENT_SECRET
});

// 골드 인증 생분해성 재료 목록
const materials = await client.materials.list({
  category: 'biodegradable',
  certificationLevel: 'gold',
  limit: 10
});

// 인증서 검증
const verification = await client.certificates.verify('cert_8nM4kL2pQw');
console.log(`인증서 유효: ${verification.valid}`);

오류 처리

API는 표준 HTTP 상태 코드를 사용하여 요청 결과를 나타냅니다. 오류 응답에는 디버깅 및 사용자 피드백을 용이하게 하는 구조화된 오류 정보가 포함됩니다.

상태 코드의미일반적인 원인
400잘못된 요청잘못된 매개변수, 잘못된 형식의 JSON
401인증되지 않음누락되거나 잘못된 인증
403금지됨작업에 대한 권한 부족
404찾을 수 없음리소스가 존재하지 않음
429너무 많은 요청속도 제한 초과
500내부 오류서버 측 문제 (백오프로 재시도)

장 요약

이 장에서는 WIA-PLASTIC_ALTERNATIVE 시스템과 통합하기 위한 포괄적인 API 문서를 제공했습니다. RESTful API는 보안되고 잘 문서화된 엔드포인트를 통해 재료 레지스트리, 인증 관리 및 추적성 기능에 대한 전체 액세스를 제공합니다.

핵심 요점

  • RESTful API는 모든 표준 기능에 대한 프로그래밍 방식 액세스를 제공합니다
  • OAuth 2.0 인증은 범위 지정된 권한으로 API 액세스를 보호합니다
  • 재료, 인증서 및 추적성 API가 핵심 사용 사례를 다룹니다
  • 웹훅은 관련 이벤트에 대한 실시간 알림을 가능하게 합니다
  • 공식 SDK는 인기 있는 언어에서 통합을 간소화합니다
