챕터 4: 1단계: 데이터 형식 사양

"弘益人間 (홍익인간) - 널리 인간을 이롭게 하라"

1단계는 모든 식량 안보 시스템 간의 상호 운용성을 가능하게 하는 표준화된 데이터 형식을 설정합니다.

주요 내용

  • 설계 원칙: 포괄적 범위, JSON 우선 접근
  • 핵심 데이터 구조: 지역, 비축, 공급망, 기후, 분배, 비상
  • 데이터 검증 규칙 및 메타데이터 표준
  • 구현 가이드 및 스키마 채택

핵심 요점

표준화된 데이터 형식은 식량 안보 상호 운용성의 기초입니다. 모든 엔터티는 UUID, 타임스탬프, 지역 식별자를 포함해야 합니다.

상세 설명

데이터 구조는 식량 안보의 모든 차원(가용성, 접근성, 활용, 안정성)을 다루며 JSON 형식을 사용하여 보편적 호환성을 보장합니다.

실용적 응용

WIA 식량 안보 표준은 다음을 가능하게 합니다:

  • 실시간 식량 안보 모니터링
  • 예측 분석을 통한 조기 경고
  • 긴급 상황 시 신속한 대응
  • 블록체인 기반 투명성
  • WIA 생태계와의 통합

기술 사양

완전한 기술 사양은 공식 WIA 식량 안보 표준 문서를 참조하십시오: wiastandards.com

구현 가이드

  1. 현재 시스템 평가
  2. WIA 데이터 형식 채택
  3. API 통합 구현
  4. 프로토콜 및 통신 설정
  5. WIA 생태계와 연결
  6. 인증 신청

참조 표준 및 적용 지침

이 장에서 다룬 절차는 국제 식량안보·추적성 프레임워크와 상호운용되도록 설계되었습니다. 구현체는 다음 ALLOW 출처를 단일 진실원으로 삼아야 하며, 본 표준은 이들 원전을 대체하지 않고 데이터 교환·감사 증적·자동 검증을 위한 상호운용 계층을 제공합니다.

  • FAO 식량불안 경험 척도(FIES) 보고 프로토콜 — 가구 단위 지표의 기준 및 라쉬 모형 기반 비교가능성 산출의 출처.
  • ISO 22005:2007 — 사료 및 식품 체인 추적성 표준; PHASE-1의 로트·관리연쇄 객체의 근거.
  • 국제식품규격위원회(Codex Alimentarius) CAC/GL 60-2006 — 일반 추적성 원칙; 로트 식별 규칙의 근거.
  • UN/CEFACT Buy-Ship-Pay 참조 데이터 모델 — PHASE-2의 API에서 사용한 원산지·운송정보 용어 체계.
  • OpenAPI 3.1, JSON Schema 2020-12, Sigstore (DSSE + Rekor) — 본 문서가 재사용한 상호운용 기본 요소.

실무 적용 체크리스트

  • 기록 무결성: 모든 보고서는 서명된 매니페스트 해시를 부착합니다. 중개자에 대한 신뢰 없이도 변조를 감지할 수 있어야 합니다.
  • 보관 기간: ISO/IEC 17065:2012 감사 기대치를 충족하기 위해 7년 이상의 보관 정책을 명시합니다. 더 긴 기간을 요구하는 규정이 있으면 그 규정이 우선합니다.
  • 버전 관리: 정정·확장은 시맨틱 버저닝을 따릅니다. 필수 요건의 추가는 마이너, 호환 단절은 메이저 버전 갱신을 트리거합니다.
  • 부정결과 보고: 누락된 지표는 사유와 함께 매니페스트에 기록되어야 합니다. 사유가 없는 누락은 감사 시 실패로 간주됩니다.

이 권고는 기존 현장 운영과 본 표준 사이의 가교를 제공하기 위한 것이며 정규(normative) 절(節)에 명시된 의무를 넘어선 새로운 의무를 부과하지 않습니다.

거버넌스 및 변경 관리

본 표준의 변경 제안은 GitHub 이슈 트래커의 food-security 레이블로 접수되며 WIA 표준 워킹그룹이 매 마이너 릴리스 사이클 시작 시점에 검토합니다. 결정 로그는 릴리스 노트와 함께 공개됩니다. 정정(에라타)은 패치 버전으로 발급되며, 새로운 의무 요건의 추가는 마이너, 호환 단절은 메이저 버전 갱신을 트리거합니다.
