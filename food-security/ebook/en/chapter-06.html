Reference Standards and Implementation Notes

The procedures in this chapter are designed to interoperate with established international food-security and traceability frameworks. Implementations SHOULD treat those frameworks as the single source of truth and use this PHASE document as the interoperability layer that maps existing data into a machine-readable, signed, and auditable shape.

  • FAO Food Insecurity Experience Scale (FIES) reporting protocol — reference for household-level indicators and Rasch-modelled comparability.
  • ISO 22005:2007 — traceability in the feed and food chain; the basis for the lot/custody-chain object in the supply-chain manifest.
  • Codex Alimentarius CAC/GL 60-2006 — general principles of traceability that informed the lot identification rules in PHASE-1.
  • UN/CEFACT Buy-Ship-Pay reference data model — origin and consignment vocabulary used in the API in PHASE-2.
  • OpenAPI 3.1, JSON Schema 2020-12, Sigstore (DSSE + Rekor) — the interoperability primitives this PHASE document re-uses.

Practical Adoption Checklist

  • Record integrity — every report has a signed manifest hash. Tampering is detectable without trusting an intermediary.
  • Retention — keep signed evidence for at least seven years to satisfy ISO/IEC 17065:2012 audit expectations; extend as required by the regulator with jurisdiction.
  • Versioning — follow Semantic Versioning so consumers can plan migration windows when normative requirements change.
  • Negative-result reporting — a skipped indicator without justification is treated by auditors as a failed indicator. Always record the reason for omission in the manifest.

These notes are intended as a bridge between existing field operations and this standard. They impose no additional obligations beyond those already declared in the normative sections of the corresponding PHASE documents.
