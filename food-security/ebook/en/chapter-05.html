Chapter 5: Phase 2 - API Interface Specification

Phase 2 builds on the data formats from Phase 1 by providing standardized APIs (Application Programming Interfaces) that enable systems to interact, exchange data, and coordinate actions. This chapter details the RESTful API architecture that powers food security interoperability.

5.1 API Architecture

RESTful Design

The WIA Food Security API follows REST (Representational State Transfer) principles:

  • Resource-Based: URLs represent resources (reserves, regions, emergencies)
  • HTTP Methods: Standard verbs (GET, POST, PUT, DELETE) for operations
  • Stateless: Each request contains all necessary information
  • JSON Responses: Data returned in standardized JSON format

Base URL Structure

https://api.food-security.wia/{version}/{service}/{resource}

Examples:

GET https://api.food-security.wia/v1/security/status?regionId=ASIA-KR-SEOUL
POST https://api.food-security.wia/v1/reserves/monitor
GET https://api.food-security.wia/v1/emergency/emerg-123/status

5.2 Authentication and Authorization

OAuth 2.0

Authorization: Bearer {access_token}

API Key

X-API-Key: {api_key}
X-API-Secret: {api_secret}

Authorization Scopes

  • food-security:read - Read food security data
  • reserves:manage - Manage strategic reserves
  • emergency:respond - Emergency response operations
  • admin:full - Full administrative access

5.3 Core API Endpoints

5.3.1 Food Security Status

GET /api/v1/security/status

Get overall food security status for a region.

Request:

GET /api/v1/security/status?regionId=ASIA-KR-SEOUL
Authorization: Bearer {token}

Response:

{
  "status": "success",
  "data": {
    "regionId": "ASIA-KR-SEOUL",
    "foodSecurityIndex": {
      "overall": 82.5,
      "availability": 85.0,
      "access": 78.0,
      "utilization": 84.0,
      "stability": 83.0
    },
    "status": "adequate",
    "alerts": []
  }
}

5.3.2 Reserve Management

POST /api/v1/reserves/monitor

Record reserve monitoring data.

Request:

POST /api/v1/reserves/monitor
Content-Type: application/json

{
  "reserveId": "a1b2c3d4-5678-90ab-cdef-1234567890ab",
  "quantity": 50000,
  "quality": {
    "grade": "A",
    "contaminationStatus": "clean"
  }
}

Response:

{
  "status": "success",
  "message": "Reserve monitoring data recorded",
  "monitoringId": "m1n2o3p4-qrst-uvwx-yz12-345678901234"
}

5.3.3 Emergency Declaration

POST /api/v1/emergency/declare

Declare a food security emergency.

Request:

POST /api/v1/emergency/declare
Authorization: Bearer {token}

{
  "regionId": "ASIA-KR-AFFECTED",
  "emergencyType": "drought",
  "severity": "level_2",
  "affectedPopulation": 500000,
  "requestedSupport": {
    "food": 5000,
    "cash": 5000000
  }
}

Response:

{
  "status": "success",
  "emergencyId": "emerg-234567890",
  "message": "Emergency declared",
  "responseTeamNotified": true
}

5.4 Error Handling

Standard error response format:

{
  "status": "error",
  "error": {
    "code": "INSUFFICIENT_PERMISSIONS",
    "message": "You do not have permission to access this resource",
    "details": {
      "requiredScope": "reserves:manage",
      "yourScope": "food-security:read"
    }
  }
}

Common Error Codes

  • INVALID_REQUEST - Malformed request
  • AUTHENTICATION_REQUIRED - Missing/invalid authentication
  • INSUFFICIENT_PERMISSIONS - Lack of authorization
  • RESOURCE_NOT_FOUND - Resource does not exist
  • RATE_LIMIT_EXCEEDED - Too many requests

5.5 Rate Limiting

Rate limits protect API infrastructure:

  • Free Tier: 100 requests/hour
  • Basic Tier: 1,000 requests/hour
  • Premium Tier: 10,000 requests/hour
  • Enterprise: Custom limits

Rate limit headers:

X-RateLimit-Limit: 1000
X-RateLimit-Remaining: 750
X-RateLimit-Reset: 1642262400

5.7 SDK and Client Libraries

Official SDKs simplify integration:

JavaScript:

import { FoodSecurityAPI } from '@wia/food-security-sdk';

const client = new FoodSecurityAPI({
  apiKey: 'your_api_key'
});

const status = await client.security.getStatus({
  regionId: 'ASIA-KR-SEOUL'
});

console.log(status.foodSecurityIndex.overall); // 82.5

Python:

from wia_food_security import FoodSecurityAPI

client = FoodSecurityAPI(api_key='your_api_key')

status = client.security.get_status(region_id='ASIA-KR-SEOUL')
print(status['foodSecurityIndex']['overall']) # 82.5
