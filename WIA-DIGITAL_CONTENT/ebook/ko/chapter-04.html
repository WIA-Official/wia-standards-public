📖 Chapter 04

WIA-DIGITAL_CONTENT: 시스템 통합

1. 시스템 통합 개요

WIA-DIGITAL_CONTENT를 기존 시스템과 통합하는 것은 실무에서 가장 중요한 과제입니다. 네이버 웹툰, 카카오페이지, 넷플릭스 한국처럼 이미 운영 중인 플랫폼에 표준을 적용하려면 레거시 시스템과의 호환성을 유지하면서 점진적으로 마이그레이션해야 합니다.

📌 학습 목표

이 장을 마치면 다음 내용을 구현할 수 있습니다:

  • 레거시 시스템과의 단계적 통합 전략
  • 네이버/카카오 플랫폼 API 연동
  • 결제 시스템 (PG사) 통합
  • AWS S3/CloudFront CDN 연동
  • Elasticsearch 검색 엔진 통합
  • Grafana/Prometheus 모니터링 구축

2. 레거시 시스템 통합 전략

대부분의 디지털 콘텐츠 플랫폼은 이미 자체 시스템을 운영 중입니다. WIA-DIGITAL_CONTENT 도입 시 기존 서비스를 중단하지 않고 통합하는 전략이 필요합니다.

2.1 Strangler Fig Pattern

마틴 파울러가 제안한 Strangler Fig 패턴으로 레거시 시스템을 점진적으로 대체합니다.

Phase 1: 신규 기능부터 WIA 표준 적용

새로 출시하는 웹툰/웹소설부터 WIA-DIGITAL_CONTENT 메타데이터와 DRM을 사용하고, 기존 콘텐츠는 그대로 유지합니다.

Phase 2: Adapter Layer 구축

레거시 데이터를 WIA 형식으로 변환하는 어댑터를 개발하여, 기존 콘텐츠도 표준 API로 접근 가능하게 만듭니다.

Phase 3: 배치 마이그레이션

사용자 영향이 적은 시간대에 레거시 데이터를 WIA 형식으로 전환합니다. 한 번에 전체가 아닌, 카테고리별로 점진적으로 수행합니다.

Phase 4: 레거시 시스템 폐기

모든 데이터와 기능이 WIA 표준으로 전환되면, 레거시 시스템을 단계적으로 폐기합니다.

2.2 데이터 마이그레이션 예시

// 레거시 → WIA 메타데이터 변환
async function migrateLegacyContent(legacyContent) {
  const wiaMetadata = {
    '@context': 'https://wia.org/standards/digital-content/v1.0',
    '@type': 'Webtoon',
    contentId: `migrated:${legacyContent.id}`,
    title: {
      ko: legacyContent.title,
      en: legacyContent.title_en || null
    },
    creator: {
      '@type': 'Person',
      name: legacyContent.author_name,
      identifier: `legacy:${legacyContent.author_id}`
    },
    genre: parseGenre(legacyContent.category),
    publishedDate: legacyContent.created_at,
    rights: {
      copyrightHolder: legacyContent.copyright || legacyContent.author_name,
      territory: ['KR'], // 레거시는 한국만 지원했다고 가정
      validFrom: legacyContent.created_at,
      validUntil: new Date('2099-12-31')
    }
  };

  // WIA 시스템에 등록
  await wiaDigitalContent.createContent(wiaMetadata);

  // 레거시 DB에 마이그레이션 완료 표시
  await db.query('UPDATE contents SET wia_migrated = true WHERE id = ?', [legacyContent.id]);
}

// 배치 마이그레이션 (10,000건씩)
async function batchMigration() {
  let offset = 0;
  const batchSize = 10000;

  while (true) {
    const legacyContents = await db.query(
      'SELECT * FROM contents WHERE wia_migrated = false LIMIT ? OFFSET ?',
      [batchSize, offset]
    );

    if (legacyContents.length === 0) break;

    for (const content of legacyContents) {
      try {
        await migrateLegacyContent(content);
        console.log(`Migrated: ${content.id}`);
      } catch (error) {
        console.error(`Failed to migrate ${content.id}:`, error);
      }
    }

    offset += batchSize;
  }
}

3. 플랫폼 연동

네이버, 카카오 등 주요 플랫폼의 API와 WIA-DIGITAL_CONTENT를 연동합니다.

3.1 네이버 웹툰 연동

기능 네이버 API WIA API 연동 방식
인증 Naver OAuth 2.0 WIA JWT OAuth 토큰으로 WIA JWT 발급
콘텐츠 등록 POST /webtoon/upload POST /api/v1/contents Webhook으로 자동 동기화
구매 Naver Pay WIA Settlement 구매 완료 시 정산 API 호출
DRM 자체 DRM WIA DRM 하이브리드 (양쪽 모두 적용)

3.2 카카오페이지 연동

// 카카오페이지 Webhook 핸들러
app.post('/webhooks/kakao/purchase', async (req, res) => {
  const { userId, episodeId, price, currency, timestamp } = req.body;

  // 1. 카카오 서명 검증
  const isValid = verifyKakaoSignature(req.headers['x-kakao-signature'], req.body);
  if (!isValid) {
    return res.status(401).json({ error: 'Invalid signature' });
  }

  // 2. WIA Settlement API 호출
  await wiaDigitalContent.recordPurchase({
    platform: 'kakao',
    contentId: episodeId,
    userId: userId,
    amount: price,
    currency: currency,
    timestamp: timestamp
  });

  // 3. 창작자 수익 자동 정산 (70:30 비율)
  await wiaDigitalContent.distributRevenue({
    episodeId,
    platformShare: 0.30,
    creatorShare: 0.70
  });

  res.json({ success: true });
});

4. 결제 시스템 통합

토스페이먼츠, 나이스페이먼츠 등 한국 PG사와 연동하여 결제 및 정산을 자동화합니다.

4.1 토스페이먼츠 연동

결제 흐름:

  • 1. 사용자가 웹툰 에피소드 구매 버튼 클릭
  • 2. 프론트엔드에서 토스페이먼츠 SDK 호출
  • 3. 결제 성공 시 토스 서버에서 Webhook 전송
  • 4. WIA API가 Webhook을 받아 라이선스 발급
  • 5. 사용자는 DRM 해제 키를 받아 콘텐츠 열람

4.2 정산 자동화

수익원 창작자 플랫폼 PG 수수료
유료 구매 70% 30% 3.3% (플랫폼 부담)
광고 수익 50% 50% 없음
IP 라이선스 80% 20% 없음

5. CDN 및 스토리지 통합

AWS S3에 콘텐츠를 저장하고, CloudFront CDN으로 글로벌 사용자에게 빠르게 전송합니다.

5.1 S3 업로드

import AWS from 'aws-sdk';
import { WIADRM } from '@wia/digital-content-sdk';

const s3 = new AWS.S3({
  accessKeyId: process.env.AWS_ACCESS_KEY,
  secretAccessKey: process.env.AWS_SECRET_KEY,
  region: 'ap-northeast-2' // 서울 리전
});

async function uploadEncryptedContent(file, contentId) {
  // 1. DRM 암호화
  const drm = new WIADRM();
  const encrypted = await drm.encrypt(file.buffer, contentId);

  // 2. S3 업로드
  const params = {
    Bucket: 'naver-webtoon-content',
    Key: `encrypted/${contentId}/${file.originalname}.enc`,
    Body: encrypted.data,
    ContentType: 'application/octet-stream',
    ServerSideEncryption: 'AES256',
    Metadata: {
      'x-wia-content-id': contentId,
      'x-wia-drm-version': '1.0',
      'x-wia-encryption-algo': 'AES-256-GCM'
    }
  };

  const result = await s3.upload(params).promise();

  // 3. CloudFront CDN URL 생성
  const cdnUrl = `https://d123abc.cloudfront.net/${params.Key}`;

  return {
    s3Url: result.Location,
    cdnUrl: cdnUrl,
    encryptionKey: encrypted.key,
    iv: encrypted.iv
  };
}

5.2 CloudFront Signed URL

구매한 사용자만 CDN에서 콘텐츠를 다운로드할 수 있도록 서명된 URL을 생성합니다.

보안 흐름:

  • 사용자가 콘텐츠 요청
  • 서버가 구매 여부 확인
  • CloudFront Signed URL 생성 (1시간 유효)
  • 사용자는 Signed URL로 CDN에서 다운로드
  • 클라이언트에서 WIA DRM 키로 복호화

Elasticsearch를 사용하여 웹툰/웹소설 검색 기능을 구현합니다.

6.1 메타데이터 인덱싱

// Elasticsearch 인덱스 생성
PUT /wia-contents
{
  "mappings": {
    "properties": {
      "contentId": { "type": "keyword" },
      "title": {
        "type": "text",
        "fields": {
          "ko": { "type": "text", "analyzer": "korean" },
          "en": { "type": "text", "analyzer": "english" }

      },
      "creator": { "type": "text" },
      "genre": { "type": "keyword" },
      "publishedDate": { "type": "date" },
      "rating": { "type": "float" },
      "viewCount": { "type": "long" }
    }
  }
}

8. 弘益人間: 생태계 전체를 이롭게

시스템 통합의 궁극적 목표는 창작자, 플랫폼, 소비자 모두가 이익을 얻는 것입니다.

창작자: 수익 증가

멀티 플랫폼 배포로 독자층 확대, 투명한 정산으로 신뢰 확보

플랫폼: 운영 효율화

표준화로 개발 비용 절감, 콘텐츠 확보 경쟁력 강화

소비자: 편의성 향상

다양한 콘텐츠 접근, 공정한 가격, 안정적인 서비스
