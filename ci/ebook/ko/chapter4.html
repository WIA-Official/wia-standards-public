4.1 통신 아키텍처

WIA-CI는 RF(프로세서 ↔ 임플란트)와 Bluetooth LE(프로세서 ↔ 모바일 앱) 프로토콜을 모두 지원하는 다층 통신 아키텍처를 정의합니다.

4.2 RF 통신 프로토콜

RF 링크는 외부 사운드 프로세서를 피부를 통과하여 이식된 장치에 연결합니다.

RF 링크 사양

파라미터 값 비고
캐리어 주파수 49 MHz ISM 대역
대역폭 2 MHz
변조 ASK / FSK 진폭 또는 주파수 편이 변조
데이터 전송률 1 Mbps
전력 전송 최대 40 mW 유도 결합

4.3 Bluetooth LE 프로토콜

BLE는 CI 프로세서와 모바일 장치 간의 무선 통신을 가능하게 합니다.

GATT 서비스 정의

WIA-CI GATT 서비스
UUID: 0000FE01-0000-1000-8000-00805F9B34FB

특성:
├─ STATUS (FE02)
│  속성: Read, Notify
│  설명: CI 프로세서 상태
│
├─ AUDIO_STREAM (FE03)
│  속성: Notify
│  설명: 실시간 오디오 데이터
│
├─ CONFIG_WRITE (FE04)
│  속성: Write
│  설명: 설정 업데이트
│
└─ WIA_EXTENDED (FE10)
   속성: Read, Write, Notify
   설명: WIA 옥타브 향상

4.4 실시간 오디오 처리

WIA-CI는 엄격한 지연 요구사항으로 실시간으로 오디오를 처리합니다.

처리 파이프라인

오디오 입력 (16ms 프레임)
    ↓
[1] 전처리 (1ms)
    - AGC (자동 게인 제어)
    - 프리엠퍼시스
    ↓
[2] 주파수 분석 (5ms)
    - FFT
    - Gammatone 필터뱅크
    ↓
[3] F0 및 옥타브 감지 (10ms)
    - YIN 알고리즘
    - 하모닉 분석
    - 옥타브 분류
    ↓
[4] 엔벨로프 추출 (2ms)
    - Hilbert 변환
    - 평활화
    - 옥타브 변조
    ↓
[5] 채널 선택 (1ms)
    - ACE (n-of-m)
    - 하모닉 우선순위
    ↓
[6] 자극 생성 (1ms)
    - 진폭 매핑
    - 인터리빙
    ↓
총 지연: ~20ms

4.5 신호 인코딩

옥타브 감지 알고리즘

function detectOctave(audioFrame: Float32Array) {
  // 1. F0 추정 (YIN 알고리즘)
  const f0Result = yinEstimateF0(audioFrame, sampleRate);

  // 2. 옥타브 분류
  const octaveInfo = classifyOctave(f0Result.f0);

  // 3. 하모닉 분석
  const spectrum = fft(audioFrame);
  const harmonics = analyzeHarmonics(spectrum, f0Result.f0);

  // 4. 신뢰도 계산
  const confidence = calculateConfidence(f0Result, harmonics);

  return {
    f0: f0Result.f0,
    octave: octaveInfo,
    harmonics: harmonics,
    confidence: confidence
  };
}

4.6 성능 지표

지표 요구사항 목표 측정
종단간 지연 < 50 ms < 20 ms 오디오 입력 → 자극 출력
F0 추정 < 20 ms < 10 ms 프레임당
패킷 손실 < 1% < 0.1% 1시간 이상
처리량 > 100 fps > 200 fps 초당 프레임 수

안전 제한

  • 최대 전류: 모든 전극에서 총 5 mA
  • 전극당 전류: 최대 2 mA
  • 펄스 레이트: 최대 32,000 펄스/초 (모든 채널 합계)
  • 온도: 임플란트는 41°C 이하 유지
