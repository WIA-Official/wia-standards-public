6장: 보안 및 프라이버시

6.1 보안 개요

WIA-CI는 환자 데이터를 보호하고, 장치 무결성을 보장하며, 무단 액세스를 방지하기 위한 포괄적인 보안 조치를 구현합니다.

보안 원칙

  • 심층 방어: 여러 보안 계층
  • 최소 권한: 필요한 최소 액세스
  • 제로 트러스트: 항상 확인, 절대 신뢰하지 않음
  • 설계에 의한 프라이버시: 처음부터 내장

6.2 인증 및 권한 부여

장치 페어링

interface DevicePairing {

// 페어링 프로세스
async function pairDevice(deviceId: string) {
  const request = generatePairingRequest(deviceId);
  const code = displayPairingCode();  // e.g., "123456"
  const confirmed = await waitForUserConfirmation();

  if (confirmed) {
    const bond = await completePairing(deviceId);
    storeBond(deviceId, bond);
  }
}

6.3 암호화

저장된 데이터 암호화

interface EncryptionAtRest {
  // 설정 데이터
  config: {
    algorithm: 'AES-256-GCM';
    keyDerivation: 'PBKDF2';
    iterations: 100000;
  };

  // 환자 데이터 (HIPAA 준수)
  patientData: {
    algorithm: 'AES-256-GCM';
    keyRotation: true;
    rotationPeriod: 2592000;  // 30일
  };
}

// 민감한 데이터 암호화
async function encryptData(data: Uint8Array, key: CryptoKey) {
  const iv = crypto.getRandomValues(new Uint8Array(12));

  const encrypted = await crypto.subtle.encrypt(
    { name: 'AES-GCM', iv },
    key,
    data
  );

  return {
    ciphertext: new Uint8Array(encrypted),
    iv: iv,
    algorithm: 'AES-256-GCM'
  };
}

전송 중 데이터 암호화

interface EncryptionInTransit {
  // BLE 암호화
  ble: {
    mode: 'AES-128-CCM';
    keyExchange: 'ECDH';
    perfectForwardSecrecy: true;
  };

  // HTTPS/TLS
  https: {
    version: 'TLS-1.3';
    cipherSuites: [
      'TLS_AES_256_GCM_SHA384',
      'TLS_CHACHA20_POLY1305_SHA256'
    ];
    certificatePinning: true;
  };
}

6.4 오디오 데이터 보호

민감한 정보를 포함할 수 있는 오디오 데이터 보호를 위한 특별 고려사항.

오디오 데이터 처리

interface AudioDataProtection {
  // 저장 정책
  storage: {
    neverStorePermanent: true;
    bufferOnly: true;
    maxBufferTime: 5000;      // ms
    clearOnDisconnect: true;
  };

  // 전송
  transmission: {
    encrypted: true;
    compression: 'opus';
    streamOnly: true;         // 녹음 안함
  };

  // 처리
  processing: {
    onDeviceOnly: boolean;
    cloudProcessing: 'opt-in';
    anonymizeBeforeCloud: true;
  };
}
