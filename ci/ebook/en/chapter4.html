4.2 RF Communication Protocol

The RF link connects the external sound processor to the implanted device through the skin.

RF Link Specifications

Parameter Value Notes
Carrier Frequency 49 MHz ISM band
Bandwidth 2 MHz
Modulation ASK / FSK Amplitude or Frequency Shift Keying
Data Rate 1 Mbps
Power Transfer 40 mW max Inductive coupling
Range 10 mm max Through skin

RF Packet Structure

interface RFPacket {
  header: {
    sync: 0xAA55;               // Sync word
    type: RFPacketType;
    length: number;             // Payload bytes
    sequence: number;           // 0-255
  };

  payload: StimulationPayload | TelemetryPayload | CommandPayload;

  footer: {
    crc16: number;              // CRC-16-CCITT
  };
}

type RFPacketType =
  | 0x01   // Stimulation Data
  | 0x02   // Telemetry Request
  | 0x03   // Telemetry Response
  | 0x04   // Configuration
  | 0x05   // Status
  | 0x10   // Extended (WIA Octave);

Compressed Stimulation Format

To minimize RF bandwidth, stimulation commands are compressed into 4 bytes:

// 32-bit packed stimulation command
// [electrode:5][amplitude:11][pulseWidth:6][flags:10]

function packStimulation(stim) {
  return (
    ((stim.electrode & 0x1F) << 27) |
    ((stim.amplitude & 0x7FF) << 16) |
    ((stim.pulseWidth & 0x3F) << 10) |
    (stim.flags & 0x3FF)
  );
}

4.4 Real-time Audio Processing

WIA-CI processes audio in real-time with strict latency requirements.

Processing Pipeline

Audio Input (16ms frame)
    ↓
[1] Pre-processing (1ms)
    - AGC (Automatic Gain Control)
    - Pre-emphasis
    ↓
[2] Frequency Analysis (5ms)
    - FFT
    - Gammatone filterbank
    ↓
[3] F0 & Octave Detection (10ms)
    - YIN algorithm
    - Harmonic analysis
    - Octave classification
    ↓
[4] Envelope Extraction (2ms)
    - Hilbert transform
    - Smoothing
    - Octave modulation
    ↓
[5] Channel Selection (1ms)
    - ACE (n-of-m)
    - Harmonic priority
    ↓
[6] Stimulation Generation (1ms)
    - Amplitude mapping
    - Interleaving
    ↓
Total Latency: ~20ms

Octave Detection Algorithm

function detectOctave(audioFrame: Float32Array) {
  // 1. F0 Estimation (YIN algorithm)
  const f0Result = yinEstimateF0(audioFrame, sampleRate);

  // 2. Octave Classification
  const octaveInfo = classifyOctave(f0Result.f0);
  // octaveInfo = { octaveNumber: 4, noteName: 'C', cents: -5 }

  // 3. Harmonic Analysis
  const spectrum = fft(audioFrame);
  const harmonics = analyzeHarmonics(spectrum, f0Result.f0);

  // 4. Confidence Calculation
  const confidence = calculateConfidence(f0Result, harmonics);

  return {
    f0: f0Result.f0,
    octave: octaveInfo,
    harmonics: harmonics,
    confidence: confidence
  };
}

4.5 Signal Encoding

Envelope Encoding

// Enhanced envelope with octave information
function extractEnhancedEnvelope(signal, f0, octaveInfo) {
  // 1. Base envelope extraction
  const baseEnvelope = hilbertEnvelope(signal);

  // 2. Generate octave-specific modulation
  const octaveModulation = generateOctaveModulation(
    signal.length,
    octaveInfo.octaveNumber,
    modulationRate: 8,  // Hz
    depth: 0.3          // 30% modulation
  );

  // 3. Combine
  const enhancedEnvelope = baseEnvelope.map((val, i) =>
    val * (1 + octaveModulation[i])
  );

  return {
    base: baseEnvelope,
    enhanced: enhancedEnvelope,
    octaveInfo: octaveInfo
  };
}

Temporal Fine Structure (TFS) Encoding

// Phase-locking for low frequencies
function encodeTFS(signal, electrode) {
  // Apply gammatone filter
  const filtered = applyGammatone(signal, electrode);

  // Hilbert transform for analytic signal
  const analytic = hilbertTransform(filtered);

  // Extract instantaneous phase
  const phase = computePhase(analytic);

  // Zero-crossing detection for pulse timing
  const pulses = detectZeroCrossings(phase);

  return {
    pulses: pulses,
    electrode: electrode,
    method: 'phase_locking'
  };
}

4.7 Quality of Service (QoS)

Packet Loss Handling

function handlePacketLoss(lastSeq, currentSeq) {
  const lostCount = currentSeq - lastSeq - 1;

  if (lostCount > MAX_CONSECUTIVE_LOSS) {
    initiateReconnect();
    return;
  }

  // Packet loss concealment
  switch (concealmentMethod) {
    case 'repeat':
      repeatLastFrame(lostCount);
      break;
    case 'interpolate':
      interpolateFrames(lastFrame, currentFrame, lostCount);
      break;
    case 'mute':
      insertSilence(lostCount);
      break;
  }
}

4.8 Error Handling

Error Codes

Code Category Description Recovery
0x0101 Connection Connection timeout Auto-retry
0x0103 Connection RF link failure Fallback to BLE
0x0202 Protocol CRC error Request retransmit
0x0404 Hardware Compliance limit Reduce current
0x1002 WIA Octave detection failed Fallback to standard

Recovery Strategy

async function handleError(error) {
  const severity = classifyError(error.code);

  switch (severity) {
    case 'critical':
      await emergencyStop();
      notifyUser(error, 'critical');
      break;

    case 'recoverable':
      for (let i = 0; i < maxRetries; i++) {
        await delay(backoffTime * Math.pow(2, i));
        const result = await retryOperation();
        if (result.success) return result;
      }
      applyFallback(error);
      break;

    case 'warning':
      logWarning(error);
      continue;
  }
}

Safety Limits

  • Maximum Current: 5 mA total across all electrodes
  • Per-Electrode Current: 2 mA maximum
  • Pulse Rate: 32,000 pulses/sec maximum (all channels combined)
  • Temperature: Implant must stay below 41°C
