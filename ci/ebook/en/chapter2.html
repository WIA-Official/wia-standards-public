2.1 Audio Input Format

WIA-CI defines a flexible audio input format that balances quality with real-time processing requirements.

Basic Audio Specification

interface CIAudioInput {
  sampleRate: 16000 | 22050 | 44100 | 48000;  // Hz
  bitDepth: 16 | 24 | 32;
  channels: 1;  // Mono (CI is unilateral)

  frameSize: number;     // Samples (recommended: 256 @ 16kHz)
  hopSize: number;       // Frame advance (recommended: 128)

  encoding: 'pcm_s16le' | 'pcm_f32le' | 'pcm_s24le';
}

Recommended Configurations

Use Case Sample Rate Frame Size Hop Size Latency
Real-time Speech 16 kHz 256 128 16 ms
Music 44.1 kHz 1024 512 23 ms
Low Latency 16 kHz 128 64 8 ms
High Quality 48 kHz 2048 1024 43 ms

2.2 Electrode Mapping

The 22-channel frequency allocation table (FAT) is the cornerstone of WIA-CI electrode mapping.

Frequency Allocation Table Structure

interface FrequencyAllocationTable {
  electrodeCount: 22;

  // Center frequencies (Hz) - Base to Apex
  centerFrequencies: [
    7438, 6500, 5688, 5000, 4375, 3813, 3313, 2875,
    2500, 2188, 1938, 1688, 1438, 1250, 1125, 1000,
    875, 750, 625, 500, 375, 250
  ];

  // Bandwidth for each electrode
  bandwidths: number[];  // Calculated from frequency ranges

  // Logarithmic distribution parameters
  logarithmic: {
    minFreq: 250;   // Hz
    maxFreq: 8000;  // Hz
    octaves: 5;     // Total octave range
  };
}

Electrode Configuration

interface ElectrodeMap {
  electrode: number;        // 1-22
  centerFreq: number;       // Hz
  lowCutoff: number;        // Hz (band edge)
  highCutoff: number;       // Hz (band edge)
  insertionDepth: number;   // mm (cochlear position)
  tonotopicAngle: number;   // degrees (Greenwood function)
}

Greenwood Function

WIA-CI uses the Greenwood function to map frequency to cochlear position: f = 165.4 × (10^(0.06x) - 1) where x is distance from apex in mm. This ensures physiologically accurate frequency-to-place mapping.

2.3 Personalized Mapping

Individual patients may require customized electrode mappings based on their specific cochlear anatomy and neural response patterns.

interface PersonalizedMap {
  patientId: string;

  // Audiometric thresholds (T-levels)
  audiometricThresholds: number[];  // μA per electrode

  // Comfort levels (C-levels)
  comfortLevels: number[];          // μA per electrode

  // Tonotopic mismatch correction
  frequencyShift: number[];         // Hz adjustment per electrode

  // Active electrodes
  activeElectrodes: number[];       // Enabled electrode numbers
  disabledElectrodes: number[];     // Disabled (e.g., facial nerve)
}

2.4 Envelope Data Format

The envelope represents the amplitude modulation of each frequency channel over time.

Basic Envelope Structure

interface EnvelopeFrame {
  timestamp: number;        // ms (frame start time)
  frameIndex: number;       // Sequential frame number

  // 22-channel envelope values
  envelopes: Float32Array;  // length: 22, range: 0.0-1.0

  // Signal statistics
  rms: number;              // Overall RMS level
  peak: number;             // Peak level

  // Selected channels (n-of-m strategy)
  selectedChannels: number[]; // ACE: top 8-12 channels
}

Extended Envelope with Octave Information

interface ExtendedEnvelopeFrame extends EnvelopeFrame {
  // WIA-CI Enhancement: Octave information
  octaveInfo: {
    fundamentalFreq: number;     // Estimated F0 (Hz)
    octaveNumber: number;        // 0-8
    octaveConfidence: number;    // 0.0-1.0

    // Harmonic analysis
    harmonics: HarmonicInfo[];
  };

  // Temporal modulation parameters
  temporalModulation: {
    pattern: 'AM' | 'FM' | 'mixed';
    frequency: number;           // Modulation rate (Hz)
    depth: number;               // 0.0-1.0
  };
}

interface HarmonicInfo {
  harmonic: number;      // 1=fundamental, 2=2nd harmonic...
  frequency: number;     // Hz
  amplitude: number;     // Relative amplitude
  present: boolean;      // Detected yes/no
}

2.5 Stimulation Data Format

Stimulation commands specify the electrical pulses delivered to each electrode.

Stimulation Command

interface StimulationCommand {
  timestamp: number;          // μs (microsecond precision)
  electrode: number;          // 1-22

  // Stimulation parameters
  amplitude: number;          // μA (0-2000)
  pulseWidth: number;         // μs (typically 25-50)
  interphaseGap: number;      // μs (typically 8)

  // Polarity
  polarity: 'cathodic_first' | 'anodic_first';

  // WIA-CI Extension: Octave modulation
  octaveModulation?: {
    enabled: boolean;
    pattern: number[];        // Modulation pattern
    rate: number;             // Hz
  };
}

Stimulation Frame

interface StimulationFrame {
  frameTimestamp: number;     // ms
  stimulations: StimulationCommand[];

  // CIS interleaving (no simultaneous stimulation)
  interleaved: boolean;

  // Safety limits
  totalCurrent: number;       // μA (total across all electrodes)
  compliance: boolean;        // Voltage compliance met
}

2.6 Stream Format

Real-time streaming uses a packetized format for efficient transmission.

Stream Packet Structure

interface CIStreamPacket {
  header: {
    magic: 0x57494143;        // 'WIAC' (WIA Cochlear)
    version: number;          // Protocol version
    packetType: PacketType;
    sequenceNumber: number;
    timestamp: number;        // μs
    payloadLength: number;
  };

  payload: AudioPayload | EnvelopePayload | StimulationPayload;

  checksum: number;           // CRC32
}

type PacketType =
  | 0x01  // Audio Input
  | 0x02  // Envelope Data
  | 0x03  // Stimulation Commands
  | 0x04  // Status/Telemetry
  | 0x05  // Configuration
  | 0x10  // Extended (with Octave);

Binary Layout

┌────────────────────────────────────────────┐
│ Header (16 bytes)                          │
├────────────────────────────────────────────┤
│ Magic    │ Ver  │ Type │ Seq  │ Timestamp │
│ 4 bytes  │ 2 b  │ 1 b  │ 2 b  │ 4 bytes   │
├────────────────────────────────────────────┤
│ PayloadLen │ Reserved                      │
│ 2 bytes    │ 1 byte                        │
├────────────────────────────────────────────┤
│ Payload (variable length)                  │
│ ...                                        │
├────────────────────────────────────────────┤
│ Checksum (4 bytes)                         │
└────────────────────────────────────────────┘

2.7 File Format (.wiac)

WIA-CI defines a standardized file format for storing and exchanging CI data.

interface WIACFile {
  header: {
    magic: 'WIAC';
    version: [1, 0, 0];

    // Metadata
    createdAt: string;  // ISO8601
    duration: number;   // ms

    // Device information
    device: {
      manufacturer: string;   // 'Cochlear', 'MED-EL', 'AB'
      model: string;
      serialNumber?: string;
    };

    // Patient information (anonymized)
    patient?: {
      id: string;             // Anonymous ID
      implantSide: 'left' | 'right';
      implantDate?: string;   // ISO8601
    };
  };

  // Configuration
  configuration: {
    audioInput: CIAudioInput;
    frequencyTable: FrequencyAllocationTable;
    processingStrategy: 'CIS' | 'ACE' | 'FSP' | 'WIA-OCTAVE';
    personalizedMap?: PersonalizedMap;
  };

  // Data blocks
  blocks: DataBlock[];
}

2.8 JSON Schema

WIA-CI provides JSON schemas for validation and interoperability.

{
  "$schema": "http://json-schema.org/draft-07/schema#",
  "$id": "https://wiastandards.com/schemas/ci/config.json",
  "title": "WIA-CI Configuration",
  "type": "object",
  "required": ["version", "audioInput", "electrodeMap"],
  "properties": {
    "version": {
      "type": "string",
      "pattern": "^\\d+\\.\\d+\\.\\d+$"
    },
    "audioInput": {
      "type": "object",
      "properties": {
        "sampleRate": { "enum": [16000, 22050, 44100, 48000] },
        "bitDepth": { "enum": [16, 24, 32] },
        "frameSize": { "type": "integer", "minimum": 64 },
        "hopSize": { "type": "integer", "minimum": 32 }
      }
    },
    "electrodeMap": {
      "type": "object",
      "properties": {
        "electrodeCount": { "type": "integer", "default": 22 },
        "centerFrequencies": {
          "type": "array",
          "items": { "type": "number" }
        }
      }
    }
  }
}
