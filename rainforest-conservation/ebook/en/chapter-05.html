Chapter 05: Phase 2 — API Interface

5.1 Phase 2 in one paragraph

Phase 2 specifies the HTTPS surface for forest-condition data: observation publish, query, stream, federation handshake. Transport is TLS 1.3 (IETF RFC 8446) only; HTTP/2 RECOMMENDED. The surface follows REST conventions with cursor-based pagination and Server-Sent Events for live deforestation alerts.

5.2 Discovery

Every host publishes a discovery document at /.well-known/wia-rainforest-conservation listing endpoints, supported signatures, supported FAO FRA vocabulary version, and rate limits. Discovery responses are cacheable for 300 seconds.

5.3 Observation endpoints

POST /rfc/observation                    (signed)
GET  /rfc/observation/{observation_id}
GET  /rfc/observation?bbox={minlon,minlat,maxlon,maxlat}&from={ts}&to={ts}

Spatial bounding-box queries are the typical pattern for clients building map-based dashboards. Hosts MAY decline very large bbox queries with 413 Payload Too Large and a problem document directing the client to chunk the query.

5.4 Deforestation event stream

GET /rfc/deforestation/stream?bbox=...
Accept: text/event-stream

Live deforestation alerts via Server-Sent Events. The stream is the primary mechanism for early-warning systems; reconnecting clients pass Last-Event-Id per WHATWG HTML §9.2.

5.5 Biodiversity, indigenous land claim, carbon stock, restoration plan endpoints

Each family follows the same shape: POST/GET/PATCH on /rfc/{family}/{id}. Indigenous land claim endpoints have additional authorization: only the community custodian can publish or amend a claim, verified via WIA-OMNI-API guardian-link claims.

5.6 Authentication

Writes require HTTP Message Signatures per IETF RFC 9421. PHI-equivalent reads (observations within an indigenous-claimed territory) MUST be authenticated and MUST present a valid community consent envelope per Phase 3 §4. Public reads of aggregate forest statistics MAY be anonymous.

Cross-Chapter Anchors

The reference container wia/rainforest-conservation-host:1.0.0 implements every Phase 2 endpoint with mock data. The companion CLI at cli/rainforest-conservation.sh ships sample envelope generators.

Indigenous-Sovereignty First Design

Every choice in this standard is filtered through the question: does this make community-claimed territory data more useful while honouring indigenous sovereignty, or just more accessible to outside actors? When the two conflict, sovereignty wins. That is why the standard mandates explicit community consent before cross-host data flow within community territory, refuses to allow national agencies to bypass community custodians, and requires implementations to surface custodian provenance so downstream consumers can verify whose data they are working with.
