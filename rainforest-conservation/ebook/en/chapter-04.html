Chapter 04: Phase 1 — Data Format

4.1 Phase 1 in one paragraph

Phase 1 specifies the JSON envelopes for forest-condition data. Every envelope is signed with Ed25519 (IETF RFC 8032) and round-trips byte-identically through encode/decode. Coordinate references use WGS84 by default; alternative CRS values follow EPSG codes registered at https://epsg.io/.

4.2 Forest condition observation

The base envelope. Required fields: observation_id, captured_at, location (point or polygon), forest_class (FAO FRA vocabulary), source_method, signature.

{
  "wia_rainforest_conservation_version": "1.0.0",
  "type": "forest_condition_observation",
  "observation_id": "obs_01HXY",
  "captured_at": "2026-04-27T10:00:00Z",
  "location": { "type": "Point", "coordinates": [-62.5, -3.2], "crs": "EPSG:4326" },
  "forest_class": "primary_humid_tropical",
  "canopy_cover_pct": 92,
  "source_method": "sentinel-2_l2a",
  "scene_id": "S2A_MSIL2A_20260427T142751",
  "signature": { "alg": "Ed25519", "value": "..." }
}

4.3 Deforestation event

A localised forest-loss event. The polygon geometry plus area_hectares plus detection method are the required fields; the standard does not pick a single detection threshold (vendor algorithms vary), so the algorithm identifier is preserved for auditability.

4.4 Biodiversity survey

Species-occurrence records. The standard is Darwin Core compatible: a Phase 4 bridge can transform WIA biodiversity_survey envelopes to Darwin Core Archive format for direct submission to GBIF.

4.5 Indigenous land claim

Community-claimed territory boundary with custodian attribution. The custodian DID is the legal authority that signs subsequent consent envelopes governing data flow within the territory.

4.6 Carbon stock estimate

Above-ground biomass, below-ground biomass, dead wood, litter, soil organic carbon. Each component carries its own uncertainty estimate per IPCC 2019 Refinement guidance; downstream REDD+ MRV bridges read these envelopes directly.

4.7 Restoration plan

Planned reforestation/restoration intervention with milestones. The standard tracks plans through `proposed`, `funded`, `in_progress`, `monitoring`, `complete`, `failed` lifecycle states; downstream consumers can filter to actively-funded plans for impact reporting.

Indigenous-Sovereignty First Design

Every choice in this standard is filtered through the question: does this make community-claimed territory data more useful while honouring indigenous sovereignty, or just more accessible to outside actors? When the two conflict, sovereignty wins. That is why the standard mandates explicit community consent before cross-host data flow within community territory, refuses to allow national agencies to bypass community custodians, and requires implementations to surface custodian provenance so downstream consumers can verify whose data they are working with.
