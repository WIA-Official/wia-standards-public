🌳 [rainforest-conservation] 표준 전자책 | 08장 (총 8장)

08장: 적합성·원주민 주권·로드맵

8.1 적합성 테스트 스위트

블랙박스 적합성 테스트 스위트가 https://github.com/WIA-Official/wia-rainforest-conservation-conformance에 발행되어 있으며, 디스커버리 문서 왕복, 관측 발행/읽기/철회, bbox 조회, 벌채 사건 스트림 구독/재연결, 생물다양성 조사 Darwin Core 왕복, 후견인 검증을 통한 원주민 토지 주장 발행, 공동체 동의 봉투 시행, 페더레이션 핸드셰이크, 속도 제한 헤더 및 소진 동작을 점검합니다. bridge_profile=Full을 발행하는 호스트는 추가로 위성 영상 소스 1개와 GBIF Darwin Core 브리지에 대한 확장 테스트를 통과해야 합니다.

8.2 참조 컨테이너

wia/rainforest-conservation-host:1.0.0 컨테이너 이미지는 모든 Phase 2 엔드포인트를 모의 데이터로 구현합니다. 컨테이너는 공동체-조직 규모 배포(주장당 최대 100,000 관측)에 적합한 SQLite 백업 저장소와 내장 FAO FRA 2020 어휘 변환기를 제공합니다.

8.3 원주민 주권 고려

표준은 원주민 공동체 동의를 부수적이 아니라 Phase 3 핵심 원리로 다룹니다. 실용적 결과: 공동체 주장 영토 내 벌채 경보를 발행하는 위성 벤더는 국가 기관이나 REDD+ 검증 기관 청중에 경보를 제공하기 전에 동의 봉투를 검증해야 합니다. 기본값은 동의 없음입니다.

UNDRIP Article 18(원주민의 의사결정 참여권)와 Article 31(문화유산 통제 유지권)이 표준 설계를 안내합니다. 공동체 후견인 DID는 동의의 진실 원천이며, 표준은 국가 기관이 후견인을 우회하는 것을 허용하지 않습니다.

8.5 호환성 약속

1.x 라인 내에서 모든 Phase 2 엔드포인트는 도달 가능하게 유지되며 문서화된 상태 코드와 콘텐츠 형태를 계속 따라야 합니다. 호스트는 선택적 쿼리 파라미터·응답 필드·새 엔드포인트·미디어 타입을 추가할 수 있습니다. 호스트는 기존 것을 제거하거나 용도 변경할 수 없습니다. 호환성 깨짐은 메이저 버전 변경에 동반되며 IETF RFC 8594와 RFC 9745에 따른 12개월 사용 중단 창이 선행되어야 합니다.

8.6 마무리 노트

열대우림 보전은 깊이 지역적 이해관계자가 있는 글로벌 공동선 문제입니다. 오늘날의 지배적 패턴은 통합된 그림을 막고 자주 데이터셋이 다루는 영토의 원주민 공동체를 우회하는 벤더별 NGO별 사일로입니다. WIA 열대우림 보전은 두 격차를 모두 해소하기 위해 존재합니다 — 데이터를 위한 개방형 와이어 형식, 사회적 계약을 위한 핵심 원리로서의 원주민 공동체 동의.

장 간 연결

이 장은 WIA 열대우림 보전 8장 시리즈에 속합니다. 적합성 테스트 스위트(https://github.com/WIA-Official/wia-rainforest-conservation-conformance)가 구현이 통과해야 하는 단일 진실 원천입니다. 참조 컨테이너 wia/rainforest-conservation-host:1.0.0이 모든 Phase 2 엔드포인트를 모의 데이터로 구현. 동반 CLI cli/rainforest-conservation.sh가 표본 봉투 생성기를 제공.

원주민 주권 우선 설계

이 표준의 모든 결정은 다음 질문을 거칩니다: 이것이 공동체 주장 영토 데이터를 더 유용하게 만들면서 원주민 주권을 존중하는가, 아니면 단지 외부 행위자에게 더 접근 가능하게 만드는가? 두 가지가 충돌하면 주권이 이깁니다.

다음에 볼 곳

  • Phase 1 사양 spec/PHASE-1-DATA-FORMAT.md — 산림 상태·벌채 사건·생물다양성 조사·원주민 토지 주장·탄소 저장량·복원 계획에 대한 JSON 봉투 형태.
  • Phase 2 사양 spec/PHASE-2-API.md — 발행·조회·스트림·페더레이션 핸드셰이크를 위한 HTTP 표면.
  • Phase 3 사양 spec/PHASE-3-PROTOCOL.md — 페더레이션 핸드셰이크·리플레이 방어·원주민 공동체 동의·청중 통제.
  • Phase 4 사양 spec/PHASE-4-INTEGRATION.md — REDD+ MRV·FAO FRA·GBIF Darwin Core·Sentinel-2/Landsat 위성 인제스트·WIA 패밀리 합성.
  • UNFCCC REDD+ 웹 플랫폼 — 국가에서 기후 협약으로 산림 상태 데이터가 흐르는 방식의 국제 정책 프레임워크.
  • FAO 글로벌 산림자원평가 — Phase 1 어휘가 정렬되는 국가 수준 산림 분류체계.
  • UNDRIP(원주민 권리에 관한 UN 선언) — Phase 3 공동체 동의가 인코딩하는 국제 인권 프레임워크.

구현자 노트

산림 데이터 인프라에는 통합 시 종종 과소평가되는 운영 고려사항 세 가지가 표준에 박혀 있습니다. 첫째, 공간 인덱스: 좌표 기반 조회는 PostGIS 같은 공간 데이터베이스를 강력히 요구합니다. 둘째, 위성 영상 메타데이터 보존: scene_id 필드는 Sentinel-2/Landsat 장면 ID를 보존하여 다운스트림 컨슈머가 원본 영상을 재페치할 수 있게 합니다. 셋째, 공동체 후견인 신원 확인: WIA-OMNI-API의 guardian-link 클레임이 후견인 정당성의 진실 원천이며, 호스트는 stale guardian-link로 서명된 동의 봉투를 거부해야 합니다.
