🌳 [rainforest-conservation] 표준 전자책 | 01장 (총 8장)

01장: 개요 — 왜 열대우림에 개방형 표준이 필요한가

1.1 이 책이 다루는 내용

WIA 열대우림 보전 표준은 산림 상태 관측치(벌채 사건, 황폐화 단계, 생물다양성 조사, 원주민 영토 주장, 탄소 저장량 추정치)에 대한 개방형 와이어 형식과 페더레이션 프로토콜을 제공합니다. 이 책은 실무자용 입문서로서 4개 페이즈 아키텍처(데이터 형식·API 인터페이스·페더레이션 프로토콜·통합)를 구체적인 봉투 예제와 함께 설명합니다.

1.2 왜 열대우림 데이터에 개방형 표준이 필요한가

현재 산림 상태 데이터는 분리된 행위자들에 의해 생산됩니다. 국가 산림청, 위성 모니터링 벤더(Planet Labs, Maxar, Sentinel Hub), 학술 원격탐사 연구실, NGO 지상 조사팀, 원주민 공동체 토지 매핑 프로젝트가 모두 자기 형식으로 데이터를 발행합니다. 정책 결정자나 감사관은 이를 직접 조정해야 하며, 원주민 공동체는 기술 결정 테이블에 거의 참여하지 못합니다.

1.3 WIA 열대우림 보전이 바꾸는 것

이 표준은 벤더 분단을 하나의 와이어 형식으로 통합하면서, Phase 3에 원주민 공동체 동의 게이트를 명시적으로 박았습니다. 위성 벤더의 벌채 경보와 공동체 지상 조사가 동일한 와이어를 통해 흐를 수 있지만, 수신자는 동의 범위에 따라 필터링하고 적절한 증거에 기반해 결정할 수 있습니다.

1.4 4-Phase 아키텍처

Phase 1(데이터 형식)은 산림 상태 관측·벌채 사건·생물다양성 조사·원주민 토지 주장·탄소 저장량 추정 봉투를 정의합니다. Phase 2(API 인터페이스)는 발행·조회·스트림·페더레이션을 위한 HTTP 표면을 명세합니다. Phase 3(페더레이션 프로토콜)는 공동체 동의 봉투를 게이트로 사용하여 호스트 간 데이터 흐름을 처리합니다. Phase 4(통합)는 FAO 글로벌 산림자원평가, REDD+ MRV, GBIF 생물다양성 데이터, Sentinel-2/Landsat 위성 영상, WIA 패밀리 표준과 연결됩니다.

1.5 원주민 주권을 핵심 원리로

이전 산림 데이터 표준들은 원주민 공동체 동의를 부수적으로 다뤘습니다. WIA 열대우림 보전 표준은 이를 Phase 3 핵심 원리로 다룹니다 — 공동체가 주장하는 영토와 관련된 관측을 수반하는 모든 호스트 간 데이터 흐름은 공동체 후견인이 서명한 새로운 동의 봉투로 게이트되어야 합니다. 기본값은 동의 없음입니다.

1.6 이 책의 구성

2-3장은 환경 분석과 표준 개요. 4-7장은 4개 페이즈를 구체 예제와 함께 다룸. 8장은 적합성·원주민 주권 고려·로드맵을 마무리.

장 간 연결

이 장은 WIA 열대우림 보전 8장 시리즈에 속합니다. 적합성 테스트 스위트(https://github.com/WIA-Official/wia-rainforest-conservation-conformance)가 구현이 통과해야 하는 단일 진실 원천입니다. 참조 컨테이너 wia/rainforest-conservation-host:1.0.0이 모든 Phase 2 엔드포인트를 모의 데이터로 구현. 동반 CLI cli/rainforest-conservation.sh가 표본 봉투 생성기를 제공.

원주민 주권 우선 설계

이 표준의 모든 결정은 다음 질문을 거칩니다: 이것이 공동체 주장 영토 데이터를 더 유용하게 만들면서 원주민 주권을 존중하는가, 아니면 단지 외부 행위자에게 더 접근 가능하게 만드는가? 두 가지가 충돌하면 주권이 이깁니다.

다음에 볼 곳

  • Phase 1 사양 spec/PHASE-1-DATA-FORMAT.md — 산림 상태·벌채 사건·생물다양성 조사·원주민 토지 주장·탄소 저장량·복원 계획에 대한 JSON 봉투 형태.
  • Phase 2 사양 spec/PHASE-2-API.md — 발행·조회·스트림·페더레이션 핸드셰이크를 위한 HTTP 표면.
  • Phase 3 사양 spec/PHASE-3-PROTOCOL.md — 페더레이션 핸드셰이크·리플레이 방어·원주민 공동체 동의·청중 통제.
  • Phase 4 사양 spec/PHASE-4-INTEGRATION.md — REDD+ MRV·FAO FRA·GBIF Darwin Core·Sentinel-2/Landsat 위성 인제스트·WIA 패밀리 합성.
  • UNFCCC REDD+ 웹 플랫폼 — 국가에서 기후 협약으로 산림 상태 데이터가 흐르는 방식의 국제 정책 프레임워크.
  • FAO 글로벌 산림자원평가 — Phase 1 어휘가 정렬되는 국가 수준 산림 분류체계.
  • UNDRIP(원주민 권리에 관한 UN 선언) — Phase 3 공동체 동의가 인코딩하는 국제 인권 프레임워크.

구현자 노트

산림 데이터 인프라에는 통합 시 종종 과소평가되는 운영 고려사항 세 가지가 표준에 박혀 있습니다. 첫째, 공간 인덱스: 좌표 기반 조회는 PostGIS 같은 공간 데이터베이스를 강력히 요구합니다. 둘째, 위성 영상 메타데이터 보존: scene_id 필드는 Sentinel-2/Landsat 장면 ID를 보존하여 다운스트림 컨슈머가 원본 영상을 재페치할 수 있게 합니다. 셋째, 공동체 후견인 신원 확인: WIA-OMNI-API의 guardian-link 클레임이 후견인 정당성의 진실 원천이며, 호스트는 stale guardian-link로 서명된 동의 봉투를 거부해야 합니다.
