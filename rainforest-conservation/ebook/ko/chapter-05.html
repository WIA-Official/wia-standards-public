05장: Phase 2 — API 인터페이스

5.1 한 단락으로 보는 Phase 2

Phase 2는 산림 데이터 HTTPS 표면을 명세합니다 — 관측 발행·조회·스트림·페더레이션 핸드셰이크. 전송은 TLS 1.3(IETF RFC 8446) 전용. HTTP/2 권장. 표면은 REST 관행을 따르며 커서 기반 페이지네이션과 라이브 벌채 경보를 위한 SSE를 사용합니다.

5.2 디스커버리

모든 호스트는 /.well-known/wia-rainforest-conservation에 디스커버리 문서를 발행합니다. 엔드포인트, 지원 서명 방식, 지원 FAO FRA 어휘 버전, 속도 제한이 나열됩니다. 디스커버리 응답은 300초 캐시 가능.

5.3 관측 엔드포인트

POST /rfc/observation                    (서명)
GET  /rfc/observation/{observation_id}
GET  /rfc/observation?bbox={minlon,minlat,maxlon,maxlat}&from={ts}&to={ts}

공간 경계 상자 조회는 지도 기반 대시보드 클라이언트의 일반적 패턴. 호스트는 너무 큰 bbox 조회를 413 Payload Too Large로 거부할 수 있습니다.

5.4 벌채 사건 스트림

GET /rfc/deforestation/stream?bbox=...
Accept: text/event-stream

SSE를 통한 라이브 벌채 경보. 조기 경보 시스템의 주요 메커니즘. 재연결 클라이언트는 WHATWG HTML §9.2에 따라 Last-Event-Id를 전달.

5.5 생물다양성·원주민 토지 주장·탄소 저장량·복원 계획 엔드포인트

각 가족은 동일한 형태를 따름: /rfc/{family}/{id}에 POST/GET/PATCH. 원주민 토지 주장 엔드포인트는 추가 권한 부여를 가짐 — 공동체 후견인만 주장을 발행하거나 수정할 수 있으며, WIA-OMNI-API guardian-link 클레임으로 검증.

5.6 인증

쓰기는 IETF RFC 9421에 따른 HTTP 메시지 서명 필요. 원주민 주장 영토 내 관측에 대한 PHI 등가 읽기는 인증되어야 하며 Phase 3 §4에 따른 유효한 공동체 동의 봉투를 제시해야 합니다.

장 간 연결

이 장은 WIA 열대우림 보전 8장 시리즈에 속합니다. 적합성 테스트 스위트(https://github.com/WIA-Official/wia-rainforest-conservation-conformance)가 구현이 통과해야 하는 단일 진실 원천입니다. 참조 컨테이너 wia/rainforest-conservation-host:1.0.0이 모든 Phase 2 엔드포인트를 모의 데이터로 구현. 동반 CLI cli/rainforest-conservation.sh가 표본 봉투 생성기를 제공.

원주민 주권 우선 설계

이 표준의 모든 결정은 다음 질문을 거칩니다: 이것이 공동체 주장 영토 데이터를 더 유용하게 만들면서 원주민 주권을 존중하는가, 아니면 단지 외부 행위자에게 더 접근 가능하게 만드는가? 두 가지가 충돌하면 주권이 이깁니다.

다음에 볼 곳

  • Phase 1 사양 spec/PHASE-1-DATA-FORMAT.md — 산림 상태·벌채 사건·생물다양성 조사·원주민 토지 주장·탄소 저장량·복원 계획에 대한 JSON 봉투 형태.
  • Phase 2 사양 spec/PHASE-2-API.md — 발행·조회·스트림·페더레이션 핸드셰이크를 위한 HTTP 표면.
  • Phase 3 사양 spec/PHASE-3-PROTOCOL.md — 페더레이션 핸드셰이크·리플레이 방어·원주민 공동체 동의·청중 통제.
  • Phase 4 사양 spec/PHASE-4-INTEGRATION.md — REDD+ MRV·FAO FRA·GBIF Darwin Core·Sentinel-2/Landsat 위성 인제스트·WIA 패밀리 합성.
  • UNFCCC REDD+ 웹 플랫폼 — 국가에서 기후 협약으로 산림 상태 데이터가 흐르는 방식의 국제 정책 프레임워크.
  • FAO 글로벌 산림자원평가 — Phase 1 어휘가 정렬되는 국가 수준 산림 분류체계.
  • UNDRIP(원주민 권리에 관한 UN 선언) — Phase 3 공동체 동의가 인코딩하는 국제 인권 프레임워크.

구현자 노트

산림 데이터 인프라에는 통합 시 종종 과소평가되는 운영 고려사항 세 가지가 표준에 박혀 있습니다. 첫째, 공간 인덱스: 좌표 기반 조회는 PostGIS 같은 공간 데이터베이스를 강력히 요구합니다. 둘째, 위성 영상 메타데이터 보존: scene_id 필드는 Sentinel-2/Landsat 장면 ID를 보존하여 다운스트림 컨슈머가 원본 영상을 재페치할 수 있게 합니다. 셋째, 공동체 후견인 신원 확인: WIA-OMNI-API의 guardian-link 클레임이 후견인 정당성의 진실 원천이며, 호스트는 stale guardian-link로 서명된 동의 봉투를 거부해야 합니다.
