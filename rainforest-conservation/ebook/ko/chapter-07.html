07장: Phase 4 — 지구 관측 파이프라인 통합

7.1 한 단락으로 보는 Phase 4

Phase 4는 WIA 열대우림 보전이 기존 산림 데이터 시스템과 통합되는 방식을 명세합니다 — REDD+ MRV 보고 템플릿, FAO 글로벌 산림자원평가 어휘, GBIF Darwin Core 생물다양성 데이터, Sentinel-2/Landsat 위성 영상 파이프라인, OpenStreetMap landcover 레이어, WIA 패밀리 표준.

7.2 REDD+ MRV 브리지

브리지는 WIA forest_condition_observation + deforestation_event + carbon_stock_estimate 봉투를 REDD+ MRV 보고 템플릿에 매핑. 브리지를 구독하는 국가 산림청은 공동체와 벤더 데이터를 하나의 파이프로 인제스트하고 UNFCCC에 국가 수준 MRV 보고서 제출 가능.

7.3 FAO 글로벌 산림자원평가 어휘

FAO FRA는 5년마다 국가 수준 산림 분류체계를 발행. WIA Phase 1 어휘가 FAO FRA 항목과 정렬됨. 정렬 표는 spec/profiles/fao-fra.md. FAO가 새 FRA 판을 발행하면 정렬 표가 업데이트되며, 표준은 추가형 마이너 버전 규칙을 통해 신·구 어휘 토큰을 모두 지원.

7.4 GBIF Darwin Core 브리지

Global Biodiversity Information Facility(GBIF)는 종 출현 데이터를 Darwin Core Archive 형식으로 받음. 브리지는 WIA biodiversity_survey 봉투를 Darwin Core로 변환하여 GBIF에 직접 제출. 원주민 공동체 동의가 Darwin Core 데이터셋 메타데이터로 흘러들어 GBIF 소비자가 출처를 검증 가능.

7.5 위성 영상 파이프라인

Sentinel-2(Copernicus/ESA 경유)와 Landsat(USGS 경유)는 벌채 탐지에 가장 일반적으로 사용되는 공공 도메인 위성 영상 소스. 브리지가 장면 메타데이터를 인제스트하고 scene_id 필드를 통해 WIA forest_condition_observation 봉투와 연결. 벤더 위성 영상(Planet Labs, Maxar)은 벤더별 장면 ID 형식으로 유사한 패턴을 따름.

7.6 WIA 패밀리 통합

WIA-OMNI-API는 공동체 후견인 신원, 정부 기관 자격증, 위성 벤더 계정 보관. WIA-AIR-SHIELD는 조직 간 페더레이션을 위한 전송 강화 제공. WIA-ACCESSIBILITY는 원격 공동체 접근을 위한 저대역폭 모드와 원주민 언어 다국어 라벨을 포함한 공동체 대면 UI 편의 사항을 추진.

7.7 컴플라이언스 매핑

WIA 의료 표준의 환자 동의 패턴을 공동체 동의로 재사용. UNDRIP(원주민 권리에 관한 UN 선언) Article 18 의사결정 참여권은 원주민 공동체 동의 봉투에 인코딩됨. UNFCCC REDD+ MRV 보고 요구사항은 Phase 4 REDD+ 브리지를 통해 충족.

장 간 연결

이 장은 WIA 열대우림 보전 8장 시리즈에 속합니다. 적합성 테스트 스위트(https://github.com/WIA-Official/wia-rainforest-conservation-conformance)가 구현이 통과해야 하는 단일 진실 원천입니다. 참조 컨테이너 wia/rainforest-conservation-host:1.0.0이 모든 Phase 2 엔드포인트를 모의 데이터로 구현. 동반 CLI cli/rainforest-conservation.sh가 표본 봉투 생성기를 제공.

원주민 주권 우선 설계

이 표준의 모든 결정은 다음 질문을 거칩니다: 이것이 공동체 주장 영토 데이터를 더 유용하게 만들면서 원주민 주권을 존중하는가, 아니면 단지 외부 행위자에게 더 접근 가능하게 만드는가? 두 가지가 충돌하면 주권이 이깁니다.

다음에 볼 곳

  • Phase 1 사양 spec/PHASE-1-DATA-FORMAT.md — 산림 상태·벌채 사건·생물다양성 조사·원주민 토지 주장·탄소 저장량·복원 계획에 대한 JSON 봉투 형태.
  • Phase 2 사양 spec/PHASE-2-API.md — 발행·조회·스트림·페더레이션 핸드셰이크를 위한 HTTP 표면.
  • Phase 3 사양 spec/PHASE-3-PROTOCOL.md — 페더레이션 핸드셰이크·리플레이 방어·원주민 공동체 동의·청중 통제.
  • Phase 4 사양 spec/PHASE-4-INTEGRATION.md — REDD+ MRV·FAO FRA·GBIF Darwin Core·Sentinel-2/Landsat 위성 인제스트·WIA 패밀리 합성.
  • UNFCCC REDD+ 웹 플랫폼 — 국가에서 기후 협약으로 산림 상태 데이터가 흐르는 방식의 국제 정책 프레임워크.
  • FAO 글로벌 산림자원평가 — Phase 1 어휘가 정렬되는 국가 수준 산림 분류체계.
  • UNDRIP(원주민 권리에 관한 UN 선언) — Phase 3 공동체 동의가 인코딩하는 국제 인권 프레임워크.

구현자 노트

산림 데이터 인프라에는 통합 시 종종 과소평가되는 운영 고려사항 세 가지가 표준에 박혀 있습니다. 첫째, 공간 인덱스: 좌표 기반 조회는 PostGIS 같은 공간 데이터베이스를 강력히 요구합니다. 둘째, 위성 영상 메타데이터 보존: scene_id 필드는 Sentinel-2/Landsat 장면 ID를 보존하여 다운스트림 컨슈머가 원본 영상을 재페치할 수 있게 합니다. 셋째, 공동체 후견인 신원 확인: WIA-OMNI-API의 guardian-link 클레임이 후견인 정당성의 진실 원천이며, 호스트는 stale guardian-link로 서명된 동의 봉투를 거부해야 합니다.
