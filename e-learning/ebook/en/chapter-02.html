Chapter 2

Content Format & SCORM Compliance

This chapter covers the standardized formats for educational content, ensuring compatibility across different learning management systems.

SCORM (Sharable Content Object Reference Model)

SCORM is the most widely adopted e-learning standard. WIA-EDU-002 supports SCORM 2004 4th Edition:

  • Run-Time Environment: Communication between content and LMS
  • Content Packaging: How courses are structured and delivered
  • Sequencing & Navigation: Learning path control
  • Metadata: Course descriptions and discoverability

xAPI (Experience API / Tin Can)

xAPI is the next-generation learning standard that tracks learner experiences beyond traditional LMS:

  • Captures learning from mobile apps, simulations, and games
  • Works offline with data synchronization
  • Stores data in a Learning Record Store (LRS)
  • Uses simple JSON statements: "Actor-Verb-Object"

Content Types Supported

1. Video Content

  • HTML5 video (MP4, WebM, OGG)
  • Adaptive bitrate streaming (HLS, DASH)
  • Interactive video with embedded quizzes
  • Closed captions and transcripts (WebVTT)

2. Interactive Content (H5P)

  • Interactive videos
  • Drag-and-drop exercises
  • Fill-in-the-blanks
  • Timeline presentations
  • Virtual tours (360° images)

3. Documents

  • PDF viewers with annotation support
  • Markdown-based lessons
  • Microsoft Office files (via Office Online)
  • EPUB for e-books

Example: SCORM Manifest

<?xml version="1.0"?>
<manifest identifier="com.wia.course.001"
          xmlns="http://www.imsglobal.org/xsd/imscp_v1p1"
          xmlns:adlcp="http://www.adlnet.org/xsd/adlcp_v1p3">
  <metadata>
    <schema>ADL SCORM</schema>
    <schemaversion>2004 4th Edition</schemaversion>
  </metadata>
  <organizations default="org_001">
    <organization identifier="org_001">
      <title>Introduction to Web Development</title>
      <item identifier="item_001" identifierref="resource_001">
        <title>Lesson 1: HTML Basics</title>
      </item>
    </organization>
  </organizations>
</manifest>

Example: xAPI Statement

{
  "actor": {
    "name": "John Doe",
    "mbox": "mailto:john@example.com"
  },
  "verb": {
    "id": "http://adlnet.gov/expapi/verbs/completed",
    "display": { "en-US": "completed" }
  },
  "object": {
    "id": "http://example.com/course/web-dev/lesson-1",
    "definition": {
      "name": { "en-US": "HTML Basics" },
      "description": { "en-US": "Introduction to HTML tags" }
    }
  },
  "result": {
    "score": { "scaled": 0.95 },
    "completion": true,
    "duration": "PT30M"
  }
}

Best Practices

  • Always include metadata for better discoverability
  • Provide multiple formats (video + text) for accessibility
  • Keep SCORM packages under 100MB for faster loading
  • Use xAPI for tracking real-world skill application
  • Include transcripts and captions for all video content
