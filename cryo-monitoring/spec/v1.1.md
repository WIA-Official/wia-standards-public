# WIA CRYO-008 Cryo Monitoring Standard v1.1 - API Interface

**Version:** 1.1.0  
**Status:** Final  
**Published:** December 2025

## Overview

This specification defines the REST API interface for cryo monitoring systems, building upon the data format specification in v1.0.

## 1. API Endpoints

### 1.1 Monitoring Data

**POST /api/v1/monitoring**
- Submit new monitoring record
- Request Body: MonitoringRecord (see v1.0)
- Response: 201 Created with monitoringId

**GET /api/v1/monitoring/{monitoringId}**
- Retrieve specific monitoring record
- Response: MonitoringRecord object

**GET /api/v1/monitoring/subject/{subjectId}**
- Query monitoring history for subject
- Query Params: `start`, `end`, `limit`, `offset`
- Response: Array of MonitoringRecord

### 1.2 Real-time Streaming

**WebSocket ws://host/api/v1/stream**
- Subscribe to real-time monitoring updates
- Protocol: JSON messages
- Authentication: JWT token

**MQTT Topic: monitoring/{facilityId}/{subjectId}**
- Publish/Subscribe to sensor readings
- QoS: 2 (exactly once)
- Retain: false

## 2. Authentication

All API requests MUST include:
- Authorization header: `Bearer {JWT_TOKEN}`
- Token expiry: 1 hour (configurable)
- Refresh token: 30 days

## 3. Error Codes

| Code | Status | Description |
|------|--------|-------------|
| 400 | Bad Request | Invalid data format |
| 401 | Unauthorized | Missing or invalid token |
| 403 | Forbidden | Insufficient permissions |
| 404 | Not Found | Resource not found |
| 429 | Too Many Requests | Rate limit exceeded |
| 500 | Internal Server Error | Server-side error |

## 4. Rate Limiting

- Standard: 1000 requests/hour
- Burst: 100 requests/minute
- Streaming: 10 concurrent connections per facility
