# 챕터 08 · 학습 루프 구현 — Forward/Backward 한 사이클

## 학습 목표
- SDXL Inpainting LoRA 학습 루프 한 스텝을 VAE 인코딩 → 노이즈 추가 → UNet Forward → Loss → Backward 단계로 구현할 수 있다
- `DDPMScheduler`로 랜덤 타임스텝을 샘플링하고 `add_noise`로 학습 타깃을 만들 수 있다
- AdamW 옵티마이저와 코사인 스케줄러를 연결해 안정적인 학습 커브를 만들 수 있다

---

## 이론

### 한 스텝의 수학

SDXL Inpainting 학습은 표준 DDPM 손실이다:

```
1. 원본 이미지 x₀ → VAE 인코딩 → latent z₀ (128×128×4)
2. 타임스텝 t ∈ [0, 999] 랜덤 샘플
3. 노이즈 ε ~ N(0, I) 샘플
4. 노이지 latent z_t = α_t · z₀ + σ_t · ε
5. 조건 입력 = [z_t, z₀ × (1-m), m]  (9채널)
6. UNet 예측 ε̂ = UNet(조건, t, text)
7. loss = ||ε - ε̂||²
```

모델은 "주어진 노이즈 레벨 t에서 원본 노이즈가 무엇이었는지" 맞히도록 학습된다. 추론 시에는 이 예측을 스케줄러로 풀어 이미지를 생성한다.

### 왜 마스크된 latent도 함께 입력하는가

일반 text→image UNet은 `z_t`만 받는다. Inpainting UNet은 추가로 **`z₀ × (1-m)` (마스크 밖 원본)** 과 **`m` (마스크)** 을 받는다. 학습 시 이 9채널 구조로 forward해야 사전학습된 Inpainting 가중치가 올바르게 작동한다.

나는 처음에 마스크를 conv_in 이전에 latent와 concat하는 걸 깜빡하고 4채널로 forward했다가 "이상하게 loss가 0.5에서 더 안 떨어진다"며 이틀을 날렸다. **`torch.cat([noisy_latent, masked_latent, mask_down], dim=1)`** 이 Inpainting UNet 입력의 정석.

### 타임스텝 샘플링 편향

DDPM 논문은 t를 `[0, T-1]` 균등 샘플링한다. 하지만 SDXL 학습에서는 **중간 타임스텝(200~800)이 학습 신호가 가장 강하다**. 극단적 t (0 근처 거의 원본 / 1000 근처 거의 노이즈)는 학습 기여도가 낮다.

권장: 기본은 uniform 유지, 과적합 시 `logit_normal` 샘플링(Stable Diffusion 3 논문 방식)으로 전환. 초보는 uniform으로 시작.

### AdamW + Cosine Warmup

LoRA 학습의 표준 조합:
- **옵티마이저**: AdamW, LR 1e-4, β=(0.9, 0.999), weight_decay=1e-2
- **스케줄러**: Cosine Annealing with Warmup, warmup 500 steps
- **gradient clipping**: max_norm=1.0 (NaN 방지)

나는 LR 5e-4로 "빨리 수렴시키자"는 유혹에 넘어갔다가 10 스텝 안에 발산했다. **LoRA는 1e-4가 안전하고 2e-4 이상은 NaN 도박**이다.

## 실전 코드

```python
# SDXL Inpainting LoRA 학습 루프 한 스텝 (최소 작동 코드)
# pip install diffusers transformers accelerate peft torch

import torch
import torch.nn.functional as F
from diffusers import StableDiffusionXLInpaintPipeline, DDPMScheduler
from diffusers.optimization import get_cosine_schedule_with_warmup
from peft import LoraConfig
from torch.optim import AdamW


MODEL_ID  = "diffusers/stable-diffusion-xl-1.0-inpainting-0.1"
DEVICE    = "cuda"
WEIGHT_DTYPE = torch.bfloat16  # 학습은 bf16 mixed precision


# ──────────────────────────────────────────────
# 1. 파이프라인·스케줄러 준비
# ──────────────────────────────────────────────
def setup_training():
    pipe = StableDiffusionXLInpaintPipeline.from_pretrained(
        MODEL_ID, torch_dtype=torch.float32,
    )

    # VAE, Text Encoder는 고정 + fp16 (메모리 절약)
    pipe.vae.to(DEVICE, dtype=WEIGHT_DTYPE)
    pipe.vae.requires_grad_(False)
    pipe.text_encoder.to(DEVICE, dtype=WEIGHT_DTYPE)
    pipe.text_encoder.requires_grad_(False)
    pipe.text_encoder_2.to(DEVICE, dtype=WEIGHT_DTYPE)
    pipe.text_encoder_2.requires_grad_(False)

    # UNet은 fp32 유지 + LoRA 주입
    pipe.unet.to(DEVICE, dtype=torch.float32)
    lora_cfg = LoraConfig(
        r=16, lora_alpha=16,
        target_modules=["to_q", "to_k", "to_v", "to_out.0"],
        lora_dropout=0.0, bias="none",
    )
    pipe.unet.add_adapter(lora_cfg)
    for n, p in pipe.unet.named_parameters():
        p.requires_grad = "lora" in n.lower()

    # 학습용 DDPM 스케줄러 — 반드시 DDPMScheduler
    # num_train_timesteps=1000 고정, 1000개 노이즈 레벨 균등 샘플링
    # 추론 스케줄러(DPMSolver++, Euler a, LCM 등)와 혼동 금지 — 학습은 무조건 DDPM
    noise_scheduler = DDPMScheduler.from_pretrained(MODEL_ID, subfolder="scheduler")

    return pipe, noise_scheduler


# ──────────────────────────────────────────────
# 2. 텍스트 임베딩 사전 계산
# ──────────────────────────────────────────────
@torch.no_grad()
def encode_prompt(pipe, prompt: str, batch_size: int):
    """
    SDXL은 두 텍스트 인코더 출력을 concat한다.
    학습 시 프롬프트가 고정이면 사전 계산으로 2배 빠르게.
    """
    tokenizers    = [pipe.tokenizer,      pipe.tokenizer_2]
    text_encoders = [pipe.text_encoder,   pipe.text_encoder_2]

    prompt_embeds_list = []
    pooled_embed = None

    for tok, enc in zip(tokenizers, text_encoders):
        text_inputs = tok(
            [prompt] * batch_size,
            padding="max_length", max_length=tok.model_max_length,
            truncation=True, return_tensors="pt",
        )
        input_ids = text_inputs.input_ids.to(DEVICE)
        outputs = enc(input_ids, output_hidden_states=True)
        # 두 번째 인코더의 pooled 사용
        pooled_embed = outputs[0]
        # 마지막 히든 스테이트 (마지막-1 계층)
        prompt_embeds_list.append(outputs.hidden_states[-2])

    # [B, 77, 768+1280] = [B, 77, 2048]
    prompt_embeds = torch.cat(prompt_embeds_list, dim=-1)
    return prompt_embeds, pooled_embed


# ──────────────────────────────────────────────
# 3. 학습 한 스텝 구현
# ──────────────────────────────────────────────
def training_step(
    batch:           dict,
    pipe,
    noise_scheduler,
    prompt_embeds:   torch.Tensor,
    pooled_embed:    torch.Tensor,
) -> torch.Tensor:
    """
    배치 한 개로 loss 계산.

    batch:
        pixel_values:        (B, 3, 1024, 1024) [-1, 1]
        masked_pixel_values: (B, 3, 1024, 1024)
        mask:                (B, 1, 1024, 1024) [0, 1]
    """
    vae  = pipe.vae
    unet = pipe.unet

    pixels      = batch["pixel_values"].to(DEVICE, dtype=WEIGHT_DTYPE)
    masked      = batch["masked_pixel_values"].to(DEVICE, dtype=WEIGHT_DTYPE)
    mask        = batch["mask"].to(DEVICE, dtype=WEIGHT_DTYPE)
    B = pixels.shape[0]

    # (1) VAE 인코딩 (grad 없음, no_grad로 메모리 절약)
    with torch.no_grad():
        latents        = vae.encode(pixels).latent_dist.sample() * vae.config.scaling_factor
        masked_latents = vae.encode(masked).latent_dist.sample() * vae.config.scaling_factor
    latents        = latents.to(dtype=torch.float32)
    masked_latents = masked_latents.to(dtype=torch.float32)

    # (2) 마스크 다운샘플 (1024 → 128)
    mask_small = F.interpolate(mask, size=(latents.shape[-2], latents.shape[-1]), mode="nearest")
    mask_small = mask_small.to(dtype=torch.float32)

    # (3) 타임스텝 + 노이즈 샘플
    noise = torch.randn_like(latents)
    timesteps = torch.randint(
        0, noise_scheduler.config.num_train_timesteps, (B,),
        device=DEVICE, dtype=torch.long,
    )
    noisy_latents = noise_scheduler.add_noise(latents, noise, timesteps)

    # (4) 9채널 입력 구성
    unet_input = torch.cat([noisy_latents, masked_latents, mask_small], dim=1)

    # (5) SDXL added_cond_kwargs
    add_time_ids = torch.tensor(
        [[1024, 1024, 0, 0, 1024, 1024]] * B,
        device=DEVICE, dtype=prompt_embeds.dtype,
    )
    added_cond = {"text_embeds": pooled_embed, "time_ids": add_time_ids}

    # (6) UNet Forward
    with torch.autocast(device_type="cuda", dtype=WEIGHT_DTYPE):
        noise_pred = unet(
            unet_input, timesteps,
            encoder_hidden_states=prompt_embeds,
            added_cond_kwargs=added_cond,
        ).sample

    # (7) Loss: 전체 영역 MSE (9장에서 마스크 가중 loss로 개선)
    loss = F.mse_loss(noise_pred.float(), noise.float(), reduction="mean")
    return loss


# ──────────────────────────────────────────────
# 4. 옵티마이저 + 스케줄러
# ──────────────────────────────────────────────
def build_optimizer(unet, lr: float = 1e-4, weight_decay: float = 1e-2):
    trainable = [p for p in unet.parameters() if p.requires_grad]
    optimizer = AdamW(trainable, lr=lr, betas=(0.9, 0.999),
                      weight_decay=weight_decay, eps=1e-8)
    return optimizer


def build_scheduler(optimizer, total_steps: int, warmup_steps: int = 500):
    return get_cosine_schedule_with_warmup(
        optimizer, num_warmup_steps=warmup_steps,
        num_training_steps=total_steps,
    )


# ──────────────────────────────────────────────
# 5. 학습 루프 (간단)
# ──────────────────────────────────────────────
def train(loader, pipe, noise_scheduler, total_steps: int = 30000):
    prompt = "Korean traditional document restoration"
    prompt_embeds, pooled_embed = encode_prompt(
        pipe, prompt, batch_size=loader.batch_size or 4
    )

    optimizer = build_optimizer(pipe.unet, lr=1e-4)
    scheduler = build_scheduler(optimizer, total_steps=total_steps, warmup_steps=500)

    pipe.unet.train()
    step = 0
    loss_window = []

    for batch in loader:
        optimizer.zero_grad(set_to_none=True)

        loss = training_step(batch, pipe, noise_scheduler, prompt_embeds, pooled_embed)
        loss.backward()

        # Gradient clipping
        torch.nn.utils.clip_grad_norm_(
            [p for p in pipe.unet.parameters() if p.requires_grad],
            max_norm=1.0,
        )

        optimizer.step()
        scheduler.step()

        loss_window.append(loss.item())
        if step % 50 == 0:
            avg = sum(loss_window[-50:]) / min(len(loss_window), 50)
            lr_now = scheduler.get_last_lr()[0]
            print(f"step {step:6d} | loss {avg:.4f} | lr {lr_now:.2e}")

        step += 1
        if step >= total_steps:
            break
```

**정상 학습의 로스 커브 (내 실측):**
```
step      0 | loss 0.9823 | lr 2.00e-07
step    500 | loss 0.1842 | lr 1.00e-04   ← warmup 끝
step   2000 | loss 0.0921 | lr 9.87e-05
step  10000 | loss 0.0543 | lr 6.54e-05
step  20000 | loss 0.0412 | lr 2.34e-05
step  30000 | loss 0.0389 | lr 1.00e-07
```

첫 500 스텝은 loss가 출렁댄다 (warmup 중 LR이 올라가면서). 500 스텝 이후 안정 하강하면 정상.

## 함정 카드

1. **"VAE 출력에 `scaling_factor`를 곱하지 않는다"** → SDXL VAE는 `config.scaling_factor = 0.13025`. latent에 이 값을 곱하지 않으면 UNet이 본 분포와 달라져 loss가 내려가지 않는다. 디코딩 시 반대로 나눠야 함. `vae.encode(...).latent_dist.sample() * vae.config.scaling_factor` 외워라.

2. **"학습 스케줄러로 DDPMScheduler 대신 추론 스케줄러(DPMSolver 등)를 쓴다"** → 학습은 반드시 **DDPMScheduler** (num_train_timesteps=1000). 추론 스케줄러는 step 수가 적고 샘플링 알고리즘이 다르다. 학습용 `noise_scheduler`를 파이프라인 스케줄러와 구분해서 별도 로드.

3. **"텍스트 임베딩을 매 스텝 계산한다"** → 학습 중 프롬프트가 고정이면(복원 태스크는 보통 그렇다) 사전 계산 후 재사용. 매 스텝 Text Encoder 2개를 돌리면 VRAM 8GB 낭비 + 시간 2배. `@torch.no_grad()`로 한 번만 계산.

4. **"gradient clipping을 빼먹는다"** → SDXL LoRA는 초반 warmup에서 그래디언트가 튀기 쉽다. `clip_grad_norm_(max_norm=1.0)` 없이는 100 스텝 안에 NaN이 떠서 재시작이 흔하다. 무조건 포함.

5. **"loss가 0.1 밑으로 안 내려간다고 LR을 올린다"** → 노이즈 예측 loss는 **0.03~0.10이 정상**이다. 0에 수렴하는 게 아니다. LR을 올리면 발산. 수렴 판단은 **validation SSIM/PSNR**로 하지 training loss로 하지 말 것 (챕터 12 참조).

## 연습문제

**문제 1.** `prompt_embeds.shape == (B, 77, 2048)`, `pooled_embed.shape == (B, 1280)`이 나와야 한다. 만약 `pooled_embed.shape == (B, 768)`이면 원인은?

> **해설:** 두 번째 텍스트 인코더(`text_encoder_2`, OpenCLIP ViT-G, 1280차원)의 pooled를 써야 하는데 첫 번째(`text_encoder`, CLIP ViT-L, 768차원)의 pooled를 쓴 것. 코드에서 `pooled_embed = outputs[0]`이 두 인코더 루프 안에서 매번 덮어쓰이므로 **마지막 반복(text_encoder_2)의 pooled가 남아야 맞음**. 만약 768이 나왔다면 루프 순서가 뒤집혔거나 `outputs[0]` 인덱스가 다른 버전에선 다른 의미일 수 있음. `outputs.text_embeds` 또는 `outputs[0]`을 SDXL 문서와 대조해 확인.

**문제 2.** 마스크가 전부 0(손상 없음)인 샘플이 배치에 섞여 있다. 학습에 해로운가?

> **해설:** **해롭지 않다, 오히려 유익할 수 있다**. 마스크=0이면 모델은 "이미지 그대로 복원"을 학습 — identity mapping을 강화한다. 단, 비율이 10% 이하로 유지되어야 한다. 너무 많으면 모델이 "복원 신호가 약한" 편향을 가진다. 챕터 04 마스크 생성기에서 `mask_ratio_min=0.05`로 하한을 둔 이유가 이것.

**문제 3.** 학습 중 `loss.backward()`에서 `out of memory`가 난다. rank, batch_size, resolution, 다른 것을 먼저 줄여야 하는가?

> **해설:** **우선순위: batch_size → resolution → rank**. ① batch_size는 메모리·수렴속도에만 영향, 품질 영향 적음. gradient_accumulation_steps로 유효 배치 보존 가능. ② resolution 1024 → 768은 메모리 1/1.78로 줄지만 SDXL 네이티브와 어긋나 품질 저하. ③ rank 16 → 8은 최후의 수단. 파라미터 수가 줄어도 activation 메모리는 거의 안 줄어 효과 미미. 실용 답: **batch_size=1, gradient_accumulation=8, gradient_checkpointing 활성화** (챕터 10에서 상세).
