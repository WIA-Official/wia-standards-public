# 챕터 18 · A/B 평가 + 인간 평가자 루프 + Elo 순위

## 학습 목표
- 메트릭 자동 평가가 잡지 못하는 "사람 눈" 품질을 인간 평가자(Human-in-the-loop)로 검증하는 프로토콜을 설계할 수 있다
- 체크포인트 N개를 Elo 레이팅으로 서열화해 "사실상 최고 모델"을 정하는 파이프라인을 구현할 수 있다
- 평가자 간 합의(Cohen's Kappa) 수준을 측정하고 낮을 경우 재훈련/기준 명확화를 결정할 수 있다

---

## 이론

### 왜 인간 평가가 필요한가

챕터 12의 자동 메트릭(SSIM·PSNR·LPIPS·CLIP)은 **80%는 맞지만 20%가 틀린다**. 전형적 실패:

- SSIM 0.92 & LPIPS 0.12 — "메트릭은 최고"지만 사람이 보면 **마스크 경계가 디지털 아트처럼 어색**.
- SSIM 0.78 & LPIPS 0.22 — "중간 수준 숫자"지만 사람은 "오히려 자연스럽다"고 판단.

나는 자동 메트릭으로만 뽑은 체크포인트를 클라이언트에게 데모했다가 "이게 복원이에요? 조작 같아요"라는 말을 들었다. 그 날 이후로 **배포 전 최소 50 이미지 사람 눈 리뷰**를 거친다.

### 평가 유형 3가지

| 유형 | 프로토콜 | 장점 | 단점 |
|---|---|---|---|
| **절대 점수** | 평가자가 1~5점 부여 | 빠름 | 평가자 편향 |
| **A/B 비교** | 두 결과 중 선택 | 명확, 합의 높음 | 랭킹만 가능 |
| **체인 랭킹** | N개 순서 매기기 | 정밀 | 시간 오래 |

실무 권장: **A/B 비교 + Elo 집계**. 상대 평가라 "2점과 3점 차이가 뭐야" 같은 혼란 없음.

### Elo 레이팅 — 체스에서 가져온 방법

초기 레이팅 1500에서 시작, A/B 매칭 결과에 따라 업데이트:
```
Expected_A = 1 / (1 + 10^((R_B - R_A) / 400))
R_A_new = R_A + K × (score_A - Expected_A)
K = 32 (초반) → 16 (안정화 후)
```

50~100 매치면 10개 모델의 순위가 안정화된다. **챗봇 Arena(LMSYS)** 가 이 방식.

### 평가자 간 합의 측정 — Cohen's Kappa

두 평가자가 서로 다른 판단을 하면 결과 신뢰도가 낮다. Kappa 해석:
- κ > 0.75: 높은 합의
- 0.60 ~ 0.75: 양호
- 0.40 ~ 0.60: 중간 — **기준 명확화 필요**
- < 0.40: 낮음 — **프로토콜 재설계**

### 평가자 수와 표본 크기

- **모델 A vs B 비교**: 최소 30 이미지 × 2 평가자
- **4 모델 Elo**: 최소 4×3/2 = 6 조합 × 30 이미지 = 180 매치
- **10 모델 Elo**: 45 조합 × 20 = 900 매치 — 크라우드 평가 또는 자동 대리 평가 필요

크라우드(MTurk·Prolific) 시간당 $8~15, 한 매치 15초라면 **100 매치 $3~5**.

```python
# A/B 평가 + Elo 레이팅 + 평가자 합의 측정
# pip install streamlit pandas scipy matplotlib

import json
import random
import hashlib
from pathlib import Path
from dataclasses import dataclass, asdict
from typing import List, Dict
import pandas as pd
from scipy.stats import kendalltau


# ──────────────────────────────────────────────
# 1. 평가 매치 생성기
# ──────────────────────────────────────────────
@dataclass
class MatchRecord:
    match_id:      str
    source_image:  str
    model_A:       str
    model_B:       str
    image_A_path:  str
    image_B_path:  str
    # 평가자 입력
    winner:        str = ""        # "A" | "B" | "tie"
    rater_id:      str = ""
    confidence:    int = 0         # 1~5
    notes:         str = ""


class MatchGenerator:
    """
    모델 체크포인트 N개에서 N×(N-1)/2 조합 × 샘플 M개 매치 생성.
    A/B 순서는 랜덤.
    """
    def __init__(
        self,
        prediction_dir: str,            # prediction_dir/{model_tag}/{image_name}.jpg
        source_images:  List[str],       # 정답 이미지 목록
        num_samples:    int = 30,
    ):
        self.prediction_dir = Path(prediction_dir)
        self.source_images  = source_images
        self.num_samples    = num_samples

    def generate(self, model_tags: List[str]) -> List[MatchRecord]:
        """
        각 매치 = (source, model_A_pred, model_B_pred)
        model pair 순서 랜덤, rater가 편향되지 않도록.
        """
        matches = []
        sample_imgs = random.sample(self.source_images, self.num_samples)

        for i in range(len(model_tags)):
            for j in range(i + 1, len(model_tags)):
                for img_name in sample_imgs:
                    a, b = model_tags[i], model_tags[j]
                    if random.random() < 0.5:
                        a, b = b, a     # 순서 랜덤

                    match_id = hashlib.md5(
                        f"{a}-{b}-{img_name}".encode()
                    ).hexdigest()[:12]

                    matches.append(MatchRecord(
                        match_id     = match_id,
                        source_image = img_name,
                        model_A      = a,
                        model_B      = b,
                        image_A_path = str(self.prediction_dir / a / img_name),
                        image_B_path = str(self.prediction_dir / b / img_name),
                    ))
        random.shuffle(matches)
        return matches


# ──────────────────────────────────────────────
# 2. Streamlit 평가 UI (간소화)
# ──────────────────────────────────────────────
STREAMLIT_APP = '''

    st.stop()

st.sidebar.write(f"진행: {len(completed)} / {len(matches)}")

    st.success("모든 매치 평가 완료 🎉")
    st.stop()

                  horizontal=True, key=f"ch_{m['match_id']}")

        **m, "winner": winner, "rater_id": rater_id,

    }

# ──────────────────────────────────────────────
# 3. Elo 레이팅 집계
# ──────────────────────────────────────────────
class EloRating:
    """
    체스식 Elo 구현.
    K_FACTOR: 매치 당 레이팅 변화 최대값
    """
    def __init__(self, initial: int = 1500, k_factor: int = 32):
        self.ratings: Dict[str, float] = {}
        self.initial = initial
        self.k = k_factor

    def get(self, player: str) -> float:
        return self.ratings.setdefault(player, self.initial)

    def _expected(self, r_a: float, r_b: float) -> float:
        return 1.0 / (1.0 + 10 ** ((r_b - r_a) / 400.0))

    def update(self, winner: str, loser: str, is_tie: bool = False) -> None:
        r_a, r_b = self.get(winner), self.get(loser)
        e_a = self._expected(r_a, r_b)
        e_b = 1 - e_a

        if is_tie:
            s_a = s_b = 0.5
        else:
            s_a, s_b = 1.0, 0.0

        self.ratings[winner] = r_a + self.k * (s_a - e_a)
        self.ratings[loser]  = r_b + self.k * (s_b - e_b)

    def from_responses(self, responses: List[Dict]) -> None:
        """평가 응답 리스트에서 Elo 갱신"""
        # Tie는 양방향 업데이트
        for r in responses:
            if r["winner"] == "tie":
                self.update(r["model_A"], r["model_B"], is_tie=True)
            elif r["winner"] == "A":
                self.update(r["model_A"], r["model_B"])
            elif r["winner"] == "B":
                self.update(r["model_B"], r["model_A"])

    def leaderboard(self) -> pd.DataFrame:
        df = pd.DataFrame([
            {"model": k, "elo": round(v, 1)}
            for k, v in self.ratings.items()
        ])
        return df.sort_values("elo", ascending=False).reset_index(drop=True)


# ──────────────────────────────────────────────
# 4. 평가자 간 합의 (Cohen's Kappa)
# ──────────────────────────────────────────────
def compute_agreement(
    rater_responses: Dict[str, List[Dict]],     # {rater_id: [response, ...]}
    rater_a: str,
    rater_b: str,
) -> dict:
    """
    같은 match_id에 대해 두 평가자 판단 비교.
    """
    from sklearn.metrics import cohen_kappa_score

    # match_id → winner
    a_map = {r["match_id"]: r["winner"] for r in rater_responses[rater_a]}
    b_map = {r["match_id"]: r["winner"] for r in rater_responses[rater_b]}

    common = set(a_map.keys()) & set(b_map.keys())
    if not common:
        return {"error": "공통 평가 매치 없음"}

    labels_a = [a_map[m] for m in common]
    labels_b = [b_map[m] for m in common]

    kappa = cohen_kappa_score(labels_a, labels_b)
    agreement = sum(1 for x, y in zip(labels_a, labels_b) if x == y) / len(common)

    return {
        "n_common":    len(common),
        "agreement":   round(agreement, 3),
        "cohen_kappa": round(kappa, 3),
        "interpretation":
            "높음" if kappa > 0.75 else
            "양호" if kappa > 0.60 else
            "중간" if kappa > 0.40 else
            "낮음 - 프로토콜 재설계",
    }


# ──────────────────────────────────────────────
# 5. 메인 파이프라인
# ──────────────────────────────────────────────
def run_evaluation():
    # (1) 매치 생성
    generator = MatchGenerator(
        prediction_dir="./predictions",
        source_images=[f for f in Path("./val/images").iterdir()],
        num_samples=30,
    )
    matches = generator.generate(
        model_tags=["baseline", "step_10k", "step_20k", "step_30k", "lcm"]
    )
    with open("matches.json", "w") as f:
        json.dump([asdict(m) for m in matches], f, ensure_ascii=False, indent=2)
    print(f">> 매치 {len(matches)}개 생성 → matches.json")

    # (2) Streamlit app.py 쓰기
    Path("app.py").write_text(STREAMLIT_APP)
    print(">> UI 파일: streamlit run app.py")
    print(">> 평가자들이 평가 완료하면 responses/{rater_id}.jsonl 생성됨")

    # (3) 응답 수집 후 분석 (예시)
    rater_responses = {}
    for f in Path("responses").glob("*.jsonl"):
        rater_id = f.stem
        with open(f) as fh:
            rater_responses[rater_id] = [json.loads(line) for line in fh]

    # (4) Elo 집계
    all_responses = [r for rs in rater_responses.values() for r in rs]
    elo = EloRating()
    elo.from_responses(all_responses)
    board = elo.leaderboard()
    print("\n=== Elo 리더보드 ===")
    print(board)

    # (5) 평가자 간 합의
    raters = list(rater_responses.keys())
    if len(raters) >= 2:
        agreement = compute_agreement(rater_responses, raters[0], raters[1])
        print(f"\n=== {raters[0]} vs {raters[1]} 합의 ===")
        print(json.dumps(agreement, ensure_ascii=False, indent=2))

    # (6) 자동 메트릭 vs Elo 상관성
    try:
        auto_scores = pd.read_csv("./eval_output/comparison.csv")
        auto_scores["model"] = auto_scores.index
        merged = board.merge(auto_scores, on="model", how="inner")
        tau, _ = kendalltau(merged["elo"], merged["ssim_inside"])
        print(f"\n자동 SSIM_inside vs 인간 Elo 순위 상관 (Kendall τ): {tau:.3f}")
        # τ > 0.7면 자동 메트릭이 믿을 만, <0.4면 인간 평가 의존도 높음
    except FileNotFoundError:
        pass


if __name__ == "__main__":
    run_evaluation()
```

**운영 흐름:**
```
(1) 4개 체크포인트에 대해 같은 30장 검증 이미지로 예측 생성 (predictions/{tag}/)
(2) MatchGenerator.generate → matches.json (180 매치)
(3) 평가자 2~3명이 streamlit으로 180 매치씩 평가 (약 40분)
(4) EloRating.leaderboard → 최종 순위
(5) compute_agreement → κ > 0.6 확인 (낮으면 재평가)
(6) 자동 메트릭과 Kendall τ 상관 확인 (높으면 자동 메트릭 신뢰)
```

**내 실측 예시:**
```
Elo 리더보드:
  step_30k  1547.3
  step_20k  1523.5
  lcm       1501.2
  step_10k  1484.7
  baseline  1443.3

평가자 A vs B 합의: 83.2% (Cohen's κ = 0.72, 양호)
자동 SSIM_inside vs Elo 순위 Kendall τ = 0.82 (강 상관)
→ 이후 평가는 자동 메트릭으로 충분, 분기별 1회만 인간 평가
```

## 함정 카드

1. **"A/B 순서를 고정한다"** → "왼쪽이 더 좋아 보인다"는 좌우 편향(position bias)가 심하다. 매치마다 A/B 배치를 **랜덤 셔플**해 평가자가 위치가 아닌 내용으로 판단하게 강제.

2. **"평가자 1명에게 전부 맡긴다"** → 개인 편향(세밀한 디테일 vs 전체 톤)이 결과를 지배. **최소 2명, 이상적으로 3명** + Kappa 측정. 3명 의견이 2:1로 갈리면 majority voting 적용.

3. **"평가 매치 수를 너무 적게"** → 10 매치로 Elo를 돌리면 레이팅이 ±100 요동쳐 순위 뒤집힘. **최소 모델 페어당 20 샘플**, 이상적 30+. 10 모델 비교는 자동화된 Prolific/MTurk 필수.

4. **"인간 평가 기준을 명세 없이 시작"** → 평가자마다 "자연스러움"을 다르게 해석. 사전에 **평가 가이드라인 문서 + 3 예시(좋음/중간/나쁨)** 를 공유. Kappa 0.4 미만은 거의 가이드라인 문제.

5. **"Elo를 한 번 계산 후 유지 안 함"** → 평가가 더 쌓이면 Elo도 갱신되어야 한다. 모델을 지속 추가·평가 중이면 **전체 응답에서 매번 Elo 재계산**. 초기 시드 중요(셔플 순서에 따라 미세 차이).

## 연습문제

**문제 1.** 자동 메트릭 기준 모델 순위와 Elo 순위가 완전히 일치한다. 인간 평가를 계속 할 가치가 있는가?

> **해설:** **분기 1회만 유지**. Kendall τ > 0.85라면 자동 메트릭이 충분히 대리(proxy) 역할. 하지만 데이터 분포나 모델 아키텍처가 바뀌면 상관성이 깨질 수 있어 **정기 검증**은 필요. 또한 자동 메트릭이 **새로운 실패 모드(예: 신규 아티팩트)** 를 놓치는 경우가 많아, 소규모(10 샘플) 수시 육안 검수는 계속 권장. Elo 풀 세트는 분기별 or 주요 릴리스 전.

**문제 2.** 평가자 A는 step_30k 우위, B는 step_20k 우위로 정반대 판단. Kappa = 0.28 (낮음). 행동 계획은?

> **해설:** ① **가이드라인 재작성**: "마스크 경계 자연스러움이 최우선"과 같이 구체화 ② **캘리브레이션 세션**: A, B, 관리자 1명이 같이 앉아 5~10 샘플 공동 평가 후 의견 조율 ③ **3번째 평가자 C 추가**: 2:1 majority로 결정. κ < 0.3은 **데이터 문제보다 프로토콜 문제**일 가능성 높음. 재측정 전에 무리한 분석은 의미 없음.

**문제 3.** Elo 1547 vs 1443 차이는 실제로 얼마나 큰 차이인가?

> **해설:** Elo 1547의 1443 대비 기대 승률 = `1 / (1 + 10^((1443-1547)/400))` = 약 **64.4%**. 일반 대중에게 "100명이 보면 64명이 1547 모델을 선호한다"로 해석. 챗봇 Arena 기준으로 **100점 차이는 유의미한 품질 차이**, 50점 이하는 경계 내 혼재. step_30k vs step_20k만 104점 차이로 통계적 유의. 하지만 "100점 안정 분리"에는 최소 매치 300+ 필요.
