# 챕터 13 · 하이퍼파라미터 스윕 — 베이지안 탐색으로 최적 찾기

## 학습 목표
- Grid·Random·Bayesian(TPE) 탐색의 비용·효율 차이를 설명하고 LoRA 튜닝에 적합한 방법을 고를 수 있다
- wandb Sweep으로 rank·alpha·LR·inside_weight를 자동 탐색하는 설정 YAML을 작성할 수 있다
- 제한된 예산($100, GPU 20시간)에서 최적 근방을 찾는 실무적 탐색 전략을 세울 수 있다

---

## 이론

### 3가지 탐색 전략

| 전략 | 탐색 방식 | 비용 (N 시도) | 최적 근접도 |
|---|---|---|---|
| **Grid Search** | 파라미터 격자 전수 | 파라미터 개수 ^ 분할 수 | 격자 해상도에 제한 |
| **Random Search** | 균등/log 균등 샘플링 | N (자유 설정) | 중간 |
| **Bayesian (TPE)** | 이전 결과로 후보 예측 | N (적은 편) | 가장 높음 ⭐ |

하이퍼파라미터 4개 × 각 5단계 Grid면 **625 시도**. 한 시도에 6시간·$8이면 $5,000. 실용적이지 않다.

**Bayesian TPE (Tree-structured Parzen Estimator)** 는 이전 결과를 보고 "다음에 어떤 파라미터 조합이 좋을지" 확률 모델로 예측한다. Optuna, wandb Sweep 둘 다 지원. **15~25 시도로 TOP 3 조합**을 찾을 수 있다.

나는 처음 rank·alpha·LR 3개를 Grid로 돌리다 학습 예산을 40% 날리고 포기했다. wandb Sweep + Bayesian으로 전환하니 **16 시도로 최적 근접 조합**을 찾아냈고 비용이 $120로 줄었다.

### 탐색할 하이퍼파라미터 우선순위

| 순위 | 파라미터 | 범위 | 영향도 |
|---|---|---|---|
| 1 | Learning Rate | 5e-5 ~ 3e-4 (log) | 매우 큼 |
| 2 | LoRA rank | 8, 16, 32 | 큼 |
| 3 | inside_weight | 0.5 ~ 0.9 | 중간 |
| 4 | LoRA alpha | rank, 2×rank | 중간 |
| 5 | Batch size | 2, 4, 8 (유효) | 작음 (수렴 속도만) |
| 6 | Warmup steps | 200, 500, 1000 | 작음 |

**LR을 먼저 고정**하고 나머지를 탐색하는 게 실용적. LR은 정답 범위가 좁고(1e-4 근방), 다른 파라미터 영향을 지배하므로 LR부터 확정.

### 조기 종료 (Early Termination)

"이 조합은 loss가 5,000 스텝까지도 0.2 이하로 안 내려감 → 최적 후보 아님"을 빨리 식별해 돈을 아낀다.

- **wandb Sweep Hyperband**: 각 시도를 짧게 돌리고 성능 하위를 자동 Kill
- **Median Stopping Rule**: 같은 스텝의 이전 시도 중앙값보다 나쁘면 중단

### 탐색 예산 설계 (6시간 × $8 = $48/시도 기준)

| 예산 | 전략 | 탐색 |
|---|---|---|
| $100 | 짧은 돌림 2시간 × 6회 | LR만 탐색 |
| $400 | 짧은 돌림 × 15회 | LR + rank |
| $1,200 | Bayesian Sweep × 20회 | 4 파라미터 |
| $3,000 | 20회 + TOP 3 풀 학습 | 최종 선택 |

초심자 권장: **$100~400 범위에서 LR·rank만 탐색**. 나머지는 권장값 고정.

## 실전 코드

```python
# wandb Sweep + Bayesian TPE 하이퍼파라미터 탐색
# pip install wandb optuna

import wandb
import os
import yaml
from pathlib import Path


# ──────────────────────────────────────────────
# 1. Sweep 설정 (wandb)
# ──────────────────────────────────────────────
SWEEP_CONFIG = {
    "method": "bayes",                  # bayes | grid | random
    "metric": {
        "name": "val/ssim_inside",       # 마스크 안쪽 SSIM 최대화
        "goal": "maximize",
    },
    "early_terminate": {
        "type": "hyperband",
        "min_iter": 3,                   # 최소 3,000 스텝은 돌림
    },
    "parameters": {
        "lr": {
            "distribution": "log_uniform_values",
            "min": 5e-5,
            "max": 3e-4,
        },
        "rank": {
            "values": [8, 16, 32],
        },
        "alpha_multiplier": {            # alpha = rank × multiplier
            "values": [1, 2],
        },
        "inside_weight": {
            "distribution": "uniform",
            "min": 0.5,
            "max": 0.9,
        },
        "warmup_steps": {
            "values": [200, 500, 1000],
        },
        # 고정 파라미터 (탐색 안 함)
        "batch_size":         {"value": 2},
        "accumulation_steps": {"value": 4},
        "total_steps":        {"value": 5000},    # 탐색은 짧게
        "checkpoint_every":   {"value": 2500},
        "validation_every":   {"value": 500},
    },
}


# ──────────────────────────────────────────────
# 2. Sweep 실행 Agent
# ──────────────────────────────────────────────
def sweep_train_agent():
    """
    wandb.agent가 호출하는 학습 함수.
    wandb.config에서 파라미터를 읽어 학습 후 메트릭 로그.
    """
    wandb.init()
    cfg = dict(wandb.config)

    # alpha 파생
    cfg["alpha"] = cfg["rank"] * cfg["alpha_multiplier"]

    print(f">> Sweep 시도: rank={cfg['rank']}, alpha={cfg['alpha']}, "
          f"lr={cfg['lr']:.2e}, inside_w={cfg['inside_weight']:.2f}")

    # ─── 학습 모듈 호출 (챕터 11의 train 함수 재사용) ───
    from train import (
        build_accelerator, setup_memory_optimized_training,
        build_dataloader, encode_prompt, train_with_checkpointing,
    )
    from diffusers import DDPMScheduler
    import torch

    accelerator = build_accelerator(
        output_dir=f"./sweep/{wandb.run.id}",
        accumulation_steps=cfg["accumulation_steps"],
        mixed_precision="bf16",
    )
    pipe, unet, noise_scheduler = setup_memory_optimized_training(
        accelerator, rank=cfg["rank"], alpha=cfg["alpha"],
    )
    loader = build_dataloader(
        shard_pattern="./shards/shard-{000000..000009}.tar",
        batch_size=cfg["batch_size"], num_workers=8,
    )
    prompt_embeds, pooled_embed = encode_prompt(
        pipe, "Korean traditional document restoration", cfg["batch_size"],
    )

    # ─── 학습 실행 (metrics 자동 로그) ───
    train_with_checkpointing(
        accelerator, unet, pipe.vae, noise_scheduler,
        loader, pipe, prompt_embeds, pooled_embed,
        val_image=load_val_image(), val_mask=load_val_mask(),
        val_prompt="Korean traditional document",
        config={
            **cfg,
            "output_dir": f"./sweep/{wandb.run.id}",
            "resume_from": None,
            "log_every": 50,
        },
    )

    # ─── 최종 검증 메트릭 계산 (챕터 12 평가기) ───
    from evaluate import CheckpointEvaluator, EvalConfig
    evaluator = CheckpointEvaluator(
        base_model_id="diffusers/stable-diffusion-xl-1.0-inpainting-0.1",
        cfg=EvalConfig(
            val_image_dir="./val/images",
            val_mask_dir ="./val/masks",
            output_dir   =f"./sweep/{wandb.run.id}/eval",
            num_samples=30,    # Sweep 중엔 30장만 (시간 절약)
        ),
    )
    lora_path = f"./sweep/{wandb.run.id}/checkpoints/lora_final.safetensors"
    df = evaluator.evaluate_one_checkpoint(lora_path, tag="final")

    # wandb에 최종 메트릭 보고 (이 값을 Sweep이 최적화)
    wandb.log({
        "val/psnr_full":   df["psnr_full"].mean(),
        "val/ssim_full":   df["ssim_full"].mean(),
        "val/lpips":       df["lpips"].mean(),
        "val/clip_score":  df["clip_score"].mean(),
        "val/psnr_inside": df["psnr_inside"].mean(),
        "val/ssim_inside": df["ssim_inside"].mean(),
    })
    wandb.finish()


def load_val_image():
    from PIL import Image
    return Image.open("./val/sample_01.jpg").convert("RGB").resize((1024, 1024))


def load_val_mask():
    from PIL import Image
    return Image.open("./val/sample_01_mask.png").convert("L").resize((1024, 1024))


# ──────────────────────────────────────────────
# 3. Sweep 생성 + 실행 (CLI)
# ──────────────────────────────────────────────
def create_sweep() -> str:
    sweep_id = wandb.sweep(SWEEP_CONFIG, project="sdxl-inpaint-restoration")
    print(f"Sweep ID: {sweep_id}")
    print(f"Agent 실행: wandb agent <entity>/sdxl-inpaint-restoration/{sweep_id}")
    return sweep_id


def run_agent(sweep_id: str, count: int = 20):
    wandb.agent(sweep_id, function=sweep_train_agent, count=count)


if __name__ == "__main__":
    import argparse
    p = argparse.ArgumentParser()
    p.add_argument("--mode", choices=["create", "agent"], required=True)
    p.add_argument("--sweep_id", help="agent 모드일 때 필요")
    p.add_argument("--count", type=int, default=20)
    args = p.parse_args()

    if args.mode == "create":
        create_sweep()
    else:
        run_agent(args.sweep_id, args.count)
```

## 함정 카드

1. **"탐색 범위를 너무 넓게 잡는다"** → LR을 1e-6 ~ 1e-2로 열어두면 90%는 발산/비수렴. **사전 지식으로 합리적 범위**(5e-5~3e-4)부터. 경계를 벗어나면 다음 스윕에서 확장.

2. **"탐색 중 체크포인트를 지우지 않는다"** → 20 시도 × 3 체크포인트 × 60MB = 3.6GB. 괜찮아 보이지만 여러 스윕 누적되면 디스크 폭발. 스윕 종료 후 **TOP 3만 남기고 삭제**하는 자동 정리 스크립트 필수.

3. **"매 시도마다 검증 세트 100장 전부 평가"** → 평가만 30분. 스윕 중엔 30~50장으로 충분 (노이즈는 크지만 경향 파악 가능). 최종 확정 전에만 100~500장 전수 평가.

4. **"Bayesian에서 초반 10 시도가 랜덤이라고 실망"** → TPE는 **초반 10~15 시도는 거의 랜덤** (사전 분포 부족). 그 이후부터 집중 탐색이 시작된다. 너무 빨리 포기하지 말 것. 최소 15 시도는 돌려야 Bayesian 효과.

5. **"탐색 메트릭으로 training loss를 쓴다"** → training loss는 과적합 감지 불가. **반드시 validation 메트릭**(ssim_inside 또는 lpips)을 최적화 대상으로. LPIPS면 `goal: minimize`로 변경.

## 연습문제

**문제 1.** Sweep 16번째 시도에서 wandb 대시보드를 보니 TOP 3 모두 `rank=16, alpha=16` 조합이다. 다음 행동 2가지 선택지의 장단점은?

**(A)** 스윕을 종료하고 rank=16·alpha=16으로 풀 학습 실행  
**(B)** rank·alpha를 고정한 채 LR·inside_weight만 정밀 탐색하는 2차 스윕

> **해설:** **(B)가 더 안전**. rank·alpha가 수렴했어도 LR·inside_weight는 아직 최적이 아닐 가능성. 2차 스윕(10 시도, $80)으로 LR·inside_weight를 좁은 범위(0.8~1.5e-4, 0.75~0.90)로 정밀 탐색 후 최종 확정이 엔지니어링 관점에서 바람직. **(A)** 장점: 즉시 진행 가능, 비용 절약. 단점: "LR 1.2e-4 vs 1.5e-4" 같은 미세 조정 기회 놓침.

**문제 2.** Hyperband Early Termination으로 스윕 중 5번째 시도가 `min_iter=3` 단계에서 Kill됐다. 남은 체크포인트·로그를 어떻게 할 것인가?

> **해설:** ① **로그 보존** — 왜 나빴는지(LR이 너무 높아 발산? inside_weight가 극단?) 분석해 다음 스윕 범위 개선에 반영 ② **체크포인트 삭제** — 절반에 끊긴 LoRA는 가치 없음 ③ **이 시도의 하이퍼파라미터 조합을 향후 스윕의 탐색 금지 영역으로 문서화** (`excluded_regions.yaml` 같은 식). 실패 정보도 자산이다.

**문제 3.** TOP 조합으로 5,000 스텝 탐색했을 때 ssim_inside 0.81, 30,000 스텝 풀 학습에서는 0.83에서 평평해진다. 4만 스텝으로 연장할 가치가 있는가?

> **해설:** **보통 없음**. 증가 곡선이 30,000에서 평평이면 추가 1만 스텝에서 ±0.01 수준. 비용 $3 = 유의미한 품질 향상 아님. 대신 투자할 곳: ① **데이터 품질/양 증가** (라이선스 검증 후 +5,000장) ② **rank 32로 재학습** — capacity 이슈면 효과 큼 ③ **λ_lpips 0.3으로 올려 재학습** (최종 1만 스텝만). 학습 시간 연장은 평형 상태에선 ROI 낮다.
