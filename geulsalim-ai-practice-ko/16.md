# 챕터 16 · FastAPI 서빙 — GPU 큐 + 비동기 처리

## 학습 목표
- FastAPI + asyncio 기반의 Inpainting 추론 서버를 구축하고 동시 요청을 안전하게 처리할 수 있다
- GPU 큐(worker pool)로 단일 GPU를 여러 요청이 공유할 때 처리량과 응답 지연을 균형 있게 관리할 수 있다
- 모니터링(처리량·지연·실패율)과 rate limit을 추가해 실제 프로덕션 API를 운영할 수 있다

---

## 이론

### 요구사항 정리

- 사용자가 이미지를 업로드 → 2~10초 안에 복원 결과 반환
- 동시 10명 접속 시에도 대기 시간 예측 가능
- GPU 1장으로 서비스 시작, 필요 시 수평 확장
- 실패 시 명확한 에러 메시지 + 자동 복구

### 단일 프로세스 vs GPU 큐

**단일 프로세스 직접 호출**: 요청이 GPU를 블로킹. 동시 2 요청이면 한쪽은 즉시 대기. 스레드 안전 문제.

**GPU 큐 모델**:
```
Client 요청 → FastAPI → asyncio Queue → GPU Worker(1~N) → 결과 반환
```

워커 1개만 GPU를 소유하고 요청을 순차(또는 배치) 처리. Queue에 대기 중인 요청 수로 **예상 지연 시간** 계산 가능.

### 배치 vs 순차

워커가 요청 1개씩 처리하면 GPU 사용률이 낮다. **동적 배치**: "Queue에 2~4 요청 쌓이면 한 번에 처리" 전략.

트레이드오프:
- 배치 대기 시간: 최대 500ms
- 처리량: 3배 이상
- 단일 요청 지연: +500ms (대기)

**낮은 트래픽엔 순차, 피크엔 배치**가 이상적 — 초기에는 **순차로 시작**하고 트래픽 증가 후 배치 최적화.

### 비동기 I/O의 경계

FastAPI의 `async def` 엔드포인트는 I/O 작업(파일 업로드, 네트워크)에 유리. 하지만 **GPU 호출은 CPU bound(Python GIL)**. `asyncio.to_thread` 또는 전용 worker 프로세스로 GPU 호출을 분리해야 메인 이벤트 루프가 멈추지 않는다.

## 실전 코드

```python
# FastAPI Inpainting 서버 — GPU 큐 + 동기화 + 모니터링
# pip install fastapi uvicorn python-multipart aiofiles prometheus-client

import asyncio
import io
import time
import uuid
from contextlib import asynccontextmanager
from pathlib import Path
from typing import Optional

import torch
from PIL import Image
from fastapi import FastAPI, UploadFile, File, Form, HTTPException
from fastapi.responses import StreamingResponse, JSONResponse
from fastapi.middleware.cors import CORSMiddleware
from pydantic import BaseModel

# 챕터 15의 OptimizedInpaintPipeline 재사용
from optimized_pipeline import OptimizedInpaintPipeline
from otsu_sam import OtsuMaskDetector    # 챕터 14


# ──────────────────────────────────────────────
# 1. GPU Worker — 큐에서 Job을 꺼내 순차 처리
# ──────────────────────────────────────────────
class InferenceJob(BaseModel):
    job_id:    str
    prompt:    str
    mode:      str = "otsu"        # "otsu" | "provided"
    cfg:       float = 7.5
    steps:     int = 25
    seed:      int = 42


class GPUWorker:
    """
    단일 GPU 워커. asyncio.Queue에서 Job을 꺼내 순차 처리.
    결과는 asyncio.Future로 비동기 반환.
    """
    def __init__(self, pipeline: OptimizedInpaintPipeline):
        self.pipeline    = pipeline
        self.otsu        = OtsuMaskDetector()
        self.queue:      asyncio.Queue = asyncio.Queue()
        self.stop_event: asyncio.Event = asyncio.Event()
        self.metrics = {
            "processed": 0,
            "failed":    0,
            "queue_size_max": 0,
            "latency_sum":    0.0,
        }

    async def start(self):
        """워커 루프 시작 (백그라운드 태스크)"""
        print(">> GPU Worker 시작")
        while not self.stop_event.is_set():
            try:
                job, image_bytes, mask_bytes, future = await asyncio.wait_for(
                    self.queue.get(), timeout=1.0
                )
            except asyncio.TimeoutError:
                continue

            start = time.perf_counter()
            try:
                result = await asyncio.to_thread(
                    self._process_sync, job, image_bytes, mask_bytes
                )
                future.set_result(result)
                self.metrics["processed"] += 1
            except Exception as e:
                future.set_exception(e)
                self.metrics["failed"] += 1
                print(f"[ERROR] job {job.job_id}: {e}")

            latency = time.perf_counter() - start
            self.metrics["latency_sum"] += latency

    def _process_sync(self, job: InferenceJob,
                      image_bytes: bytes, mask_bytes: Optional[bytes]) -> bytes:
        """실제 GPU 호출 (블로킹)"""
        img = Image.open(io.BytesIO(image_bytes)).convert("RGB").resize((1024, 1024))

        if job.mode == "otsu":
            mask_np = self.otsu.detect(img)
            mask = Image.fromarray(mask_np).convert("L")
        else:
            mask = Image.open(io.BytesIO(mask_bytes)).convert("L").resize((1024, 1024))

        mask_ratio = (torch.tensor(list(mask.getdata())) > 127).float().mean().item()
        if mask_ratio < 0.005:
            raise ValueError(f"mask too small ({mask_ratio:.4f})")
        if mask_ratio > 0.5:
            raise ValueError(f"mask too large ({mask_ratio:.4f})")

        result = self.pipeline.infer(
            img, mask,
            prompt=job.prompt,
            steps=job.steps, cfg=job.cfg, seed=job.seed,
        )

        buf = io.BytesIO()
        result.save(buf, format="JPEG", quality=90)
        return buf.getvalue()

    async def submit(self, job: InferenceJob,
                     image_bytes: bytes,
                     mask_bytes: Optional[bytes] = None) -> bytes:
        """Job 제출 후 결과 await"""
        future = asyncio.get_event_loop().create_future()
        await self.queue.put((job, image_bytes, mask_bytes, future))
        self.metrics["queue_size_max"] = max(
            self.metrics["queue_size_max"], self.queue.qsize()
        )
        return await future

    async def stop(self):
        self.stop_event.set()


# ──────────────────────────────────────────────
# 2. FastAPI 앱 — 라이프스팬 관리
# ──────────────────────────────────────────────
worker: Optional[GPUWorker] = None
worker_task = None


@asynccontextmanager
async def lifespan(app: FastAPI):
    """시작 시 모델 로드·워커 시작. 종료 시 정리."""
    global worker, worker_task

    print(">> 파이프라인 로드 중 (모델 + torch.compile 워밍업)...")
    pipeline = OptimizedInpaintPipeline(
        lora_path="./output/lora_final.safetensors",
        mode="dpm",
        compile_unet=True,
    )

    # 워밍업 (torch.compile 컴파일 완료)
    dummy = Image.new("RGB", (1024, 1024), color=(200, 200, 200))
    dummy_mask = Image.new("L", (1024, 1024), color=255).point(
        lambda _: 255 if _ else 0
    )
    for _ in range(2):
        _ = pipeline.infer(dummy, dummy_mask, prompt="warmup", steps=6, cfg=2.0)
    print(">> 워밍업 완료")

    worker = GPUWorker(pipeline)
    worker_task = asyncio.create_task(worker.start())

    yield

    print(">> 서버 종료 중...")
    await worker.stop()
    await worker_task


app = FastAPI(title="SDXL Inpainting API", version="1.0", lifespan=lifespan)
app.add_middleware(
    CORSMiddleware, allow_origins=["*"],
    allow_credentials=True, allow_methods=["*"], allow_headers=["*"],
)


# ──────────────────────────────────────────────
# 3. 엔드포인트
# ──────────────────────────────────────────────
@app.post("/restore")
async def restore(
    image: UploadFile = File(..., description="JPEG/PNG 이미지"),
    mask:  Optional[UploadFile] = File(None, description="선택, 없으면 Otsu 자동"),
    prompt: str = Form("Korean traditional document restoration"),
    cfg:    float = Form(7.5),
    steps:  int   = Form(25),
    seed:   int   = Form(42),
):
    """
    이미지 업로드 → 마스크 감지 → Inpainting → JPEG 반환.
    """
    if image.content_type not in ("image/jpeg", "image/png"):
        raise HTTPException(400, "이미지 포맷은 JPEG 또는 PNG만 지원")

    image_bytes = await image.read()
    if len(image_bytes) > 10 * 1024 * 1024:
        raise HTTPException(413, "이미지는 10MB 이하")

    mask_bytes = await mask.read() if mask else None

    job = InferenceJob(
        job_id=str(uuid.uuid4()),
        prompt=prompt, cfg=cfg, steps=steps, seed=seed,
        mode="provided" if mask else "otsu",
    )

    try:
        result_bytes = await asyncio.wait_for(
            worker.submit(job, image_bytes, mask_bytes),
            timeout=60.0,   # 60초 타임아웃
        )
    except asyncio.TimeoutError:
        raise HTTPException(504, "추론 타임아웃 — 서버 혼잡")
    except ValueError as e:
        raise HTTPException(400, str(e))
    except Exception as e:
        raise HTTPException(500, f"추론 오류: {e}")

    return StreamingResponse(
        io.BytesIO(result_bytes),
        media_type="image/jpeg",
        headers={"X-Job-ID": job.job_id},
    )


@app.get("/health")
async def health():
    return {"status": "ok", "worker_ready": worker is not None}


@app.get("/metrics")
async def metrics():
    """Prometheus-친화적 메트릭"""
    if worker is None:
        return {"error": "worker not ready"}

    m = worker.metrics
    total = m["processed"] + m["failed"]
    avg_latency = m["latency_sum"] / m["processed"] if m["processed"] else 0

    return {
        "total_requests":   total,
        "success":          m["processed"],
        "failed":           m["failed"],
        "success_rate":     m["processed"] / max(total, 1),
        "avg_latency_sec":  round(avg_latency, 3),
        "queue_size_current": worker.queue.qsize(),
        "queue_size_max":   m["queue_size_max"],
        "gpu_memory_gb":    torch.cuda.memory_allocated() / 1024**3,
    }


# ──────────────────────────────────────────────
# 4. Rate Limit (IP 기반 간단 구현)
# ──────────────────────────────────────────────
from collections import defaultdict
from fastapi import Request

_request_times: dict[str, list] = defaultdict(list)
_RATE_LIMIT_PER_MIN = 10


@app.middleware("http")
async def rate_limit(request: Request, call_next):
    if request.url.path not in ("/restore",):
        return await call_next(request)

    client_ip = request.client.host if request.client else "unknown"
    now = time.time()

    times = _request_times[client_ip]
    times[:] = [t for t in times if now - t < 60]   # 최근 1분
    if len(times) >= _RATE_LIMIT_PER_MIN:
        return JSONResponse(
            {"error": f"Rate limit: {_RATE_LIMIT_PER_MIN}/min"},
            status_code=429,
        )
    times.append(now)
    return await call_next(request)


if __name__ == "__main__":
    import uvicorn
    uvicorn.run(app, host="0.0.0.0", port=8000, log_level="info")
```

## 함정 카드

1. **"모델 로드를 엔드포인트 안에서 한다"** → 첫 요청이 5분 걸린다(모델 로드 + compile). 반드시 **lifespan의 startup 단계**에서 로드 + 워밍업 완료 후 트래픽 수신.

2. **"GPU 호출을 `async def` 안에서 직접 호출"** → Python GIL + GPU 블로킹으로 이벤트 루프 전체 멈춤. 다른 요청 타임아웃 연쇄. 반드시 `asyncio.to_thread`(또는 별도 프로세스) 경유.

3. **"UploadFile의 read() 를 여러 번 부른다"** → `UploadFile.read()`는 1회만 가능. 두 번 부르면 빈 바이트. 재사용 필요하면 한 번 읽어 변수에 보관.

4. **"Worker 예외를 조용히 삼킨다"** → `future.set_exception(e)` 없이 워커만 종료되면 클라이언트는 영원히 대기. 모든 Worker 경로에 try/except + future로 에러 전파.

5. **"torch.compile 컴파일 캐시 경로를 안 설정"** → 기본은 `/tmp`이라 컨테이너 재시작 시 날아감. `TORCHINDUCTOR_CACHE_DIR=/app/cache`로 persistent volume에 저장하면 재시작 워밍업 시간 90% 단축.

## 연습문제

**문제 1.** 피크 시간(오후 2시) Queue 크기가 30 이상으로 쌓인다. 추가 GPU 구매 외 해결책 2가지는?

> **해설:** ① **동적 배치** — Queue에 2~4 요청 쌓이면 `pipe(image=[imgs], mask_image=[masks])`로 배치 추론. 처리량 3배 증가. 구현: worker 루프에서 `queue.get` 한 번 후 50ms 동안 추가 요청 흡수하여 배치 구성. ② **LCM 모드 자동 전환** — Queue > 10이면 LCM(6-step)으로 폴백. 품질은 약간 낮지만 응답 시간 유지. ③ **Accept-level 제한** — Queue 50 초과 시 429(Busy) 즉시 반환, 사용자에게 "30초 후 재시도" 안내.

**문제 2.** 동일 이미지를 10명이 동시에 보냈다. 중복 추론을 피하는 방법은?

> **해설:** **이미지 해시 기반 결과 캐시**. ① 이미지 SHA-256 계산 → 메모리/Redis 캐시 확인 ② 캐시 히트면 즉시 반환 ③ 미스면 추론 진행 + 완료 후 캐시 저장 (TTL 1시간). 10명 같은 이미지면 첫 1명만 추론, 9명은 즉시 반환. 주의: prompt/cfg/seed가 같아야 캐시 공유. 키는 `sha256(image) + prompt + cfg + steps + seed`.

**문제 3.** 사용자가 "복원이 이상하다"고 불만이다. 서버에서 어떤 로그를 남겨야 이슈 재현이 가능한가?

> **해설:** ① **요청 아카이브**: job_id 별로 `input_image.jpg`, `mask.png` (Otsu 결과), `output.jpg`, `request.json` (prompt, cfg, steps, seed, client_ip) 30일 보관 ② **메트릭**: mask_ratio, inference_time, gpu_mem_peak ③ **모델 버전**: lora_hash (safetensors SHA), base_model_id 기록 (학습 업데이트 추적). 불만 제보 시 job_id만 있으면 **정확 재현**이 가능해야 한다. 이 정보 없이는 디버그 불가.
