# Korean Historical Document Restorer (LoRA)

LoRA adapter trained on Korean historical documents for inpainting-based restoration.

## Model Details

- **Developed by**: <your name>
- **Language(s)**: Korean (조선시대 한글·한자)
- **License**: CreativeML Open RAIL-M
- **Finetuned from model**: `diffusers/stable-diffusion-xl-1.0-inpainting-0.1`
- **Training compute**: 1× NVIDIA A100 40GB, 6 hours
- **Dataset**: 10,112 public-domain images (공공누리 1유형)

## Uses

### Direct Use
Restoring damaged areas of Korean historical document images.

### Out-of-Scope Use
- Modern document restoration (not trained on this domain)
- Signature forgery, document tampering (ethically prohibited)
- Deepfake or identity manipulation

## Bias & Limitations
- Performance drops for non-Joseon dynasty material
- Biased toward the style of training sources (museum scans, Wikimedia photographs)
- May generate plausible but historically inaccurate content in large masks (>40%)

## How to Use

\`\`\`python
import torch
from diffusers import StableDiffusionXLInpaintPipeline, LCMScheduler
from peft import LoraConfig
from safetensors.torch import load_file

pipe = StableDiffusionXLInpaintPipeline.from_pretrained(
    "diffusers/stable-diffusion-xl-1.0-inpainting-0.1",
    torch_dtype=torch.float16,
).to("cuda")

lora_cfg = LoraConfig(r=16, lora_alpha=16,
    target_modules=["to_q", "to_k", "to_v", "to_out.0"])
pipe.unet.add_adapter(lora_cfg)
state = load_file("lora_final.safetensors")
pipe.unet.load_state_dict(state, strict=False)

# LCM mode for fast inference
pipe.load_lora_weights("latent-consistency/lcm-lora-sdxl", adapter_name="lcm")
pipe.set_adapters(["default", "lcm"], adapter_weights=[1.0, 1.0])
pipe.scheduler = LCMScheduler.from_config(pipe.scheduler.config)

result = pipe(
    prompt="Korean traditional document restoration",
    image=damaged_image, mask_image=mask,
    num_inference_steps=6, guidance_scale=2.0,
).images[0]

# demo/app.py — Streamlit 데모
import streamlit as st
import torch, io
from PIL import Image

st.set_page_config(page_title="고문서 복원", layout="wide")
st.title("🏛️ 조선 고문서 자동 복원")
st.caption("손상된 이미지를 업로드하면 AI가 자동으로 복원합니다.")


@st.cache_resource
def load_pipe():
    from diffusers import StableDiffusionXLInpaintPipeline, LCMScheduler
    from peft import LoraConfig
    from huggingface_hub import hf_hub_download
    from safetensors.torch import load_file

    has_gpu = torch.cuda.is_available()
    dtype = torch.float16 if has_gpu else torch.float32
    device = "cuda" if has_gpu else "cpu"

    pipe = StableDiffusionXLInpaintPipeline.from_pretrained(
        "diffusers/stable-diffusion-xl-1.0-inpainting-0.1",
        torch_dtype=dtype,
    ).to(device)

    lora_cfg = LoraConfig(r=16, lora_alpha=16,
        target_modules=["to_q", "to_k", "to_v", "to_out.0"])
    pipe.unet.add_adapter(lora_cfg, adapter_name="style")
    lora_file = hf_hub_download(
        "<user>/korean-document-restorer",
        filename="lora_final.safetensors",
    )
    state = load_file(lora_file, device="cpu")
    pipe.unet.load_state_dict(state, strict=False)

    if has_gpu:
        pipe.load_lora_weights(
            "latent-consistency/lcm-lora-sdxl", adapter_name="lcm",
        )
        pipe.set_adapters(["style", "lcm"], adapter_weights=[1.0, 1.0])
        pipe.scheduler = LCMScheduler.from_config(pipe.scheduler.config)

    pipe.set_progress_bar_config(disable=True)
    return pipe, device


pipe, device = load_pipe()

uploaded = st.file_uploader("손상된 이미지 업로드", type=["jpg", "png"])

col1, col2 = st.columns(2)

if uploaded:
    img = Image.open(uploaded).convert("RGB")
    img_1024 = img.resize((1024, 1024))

    with col1:
        st.image(img, caption="원본", use_container_width=True)

    with st.sidebar:
        steps = st.slider("추론 스텝", 4, 40, 6 if device == "cuda" else 20)
        cfg   = st.slider("CFG", 1.0, 10.0, 2.0 if device == "cuda" else 7.5)
        seed  = st.number_input("시드", value=42)

    if st.button("🚀 복원 실행"):
        # Otsu 마스크 자동 감지 (간소화)
        import cv2, numpy as np
        gray = np.array(img_1024.convert("L"))
        _, mask = cv2.threshold(gray, 0, 255,
            cv2.THRESH_BINARY_INV + cv2.THRESH_OTSU)
        kernel = np.ones((9, 9), np.uint8)
        mask = cv2.morphologyEx(mask, cv2.MORPH_CLOSE, kernel, iterations=2)
        mask_pil = Image.fromarray(mask).convert("L")

        with st.spinner(f"추론 중 ({device})..."):
            generator = torch.Generator(device).manual_seed(seed)
            with torch.autocast(device, dtype=torch.float16) if device == "cuda" else contextlib.nullcontext():
                result = pipe(
                    prompt="Korean traditional document restoration",
                    image=img_1024, mask_image=mask_pil,
                    num_inference_steps=steps,
                    guidance_scale=cfg,
                    generator=generator,
                ).images[0]

        with col2:
            st.image(result, caption="복원 결과", use_container_width=True)

        buf = io.BytesIO()
        result.save(buf, format="JPEG")
        st.download_button("다운로드", data=buf.getvalue(),
                          file_name="restored.jpg", mime="image/jpeg")

st.divider()
st.caption("Built with SDXL Inpainting + LoRA · [GitHub](https://github.com/<user>/korean-document-restorer)")

## 함정 카드

1. **"가중치를 GitHub에 commit"** → 60MB는 감당되지만 여러 버전을 올리다 보면 레포 용량이 GB 단위로. `.gitignore`에 `*.safetensors` 추가 + **HF Hub에만 가중치 올림**. GitHub는 코드 전용.

2. **"Model Card의 한계·윤리 섹션을 비워둠"** → HF 심사가 경고 달리거나 배포 거부. 반드시 "학습 데이터 한계", "악용 우려", "검수 권장 시나리오" 명시. 특히 복원/생성 모델은 필수.

3. **"데모 앱에서 사용자 이미지를 서버에 저장"** → 개인정보 이슈. Streamlit 데모는 메모리에서만 처리, 디스크 쓰기 없음. 로그를 남겨야 한다면 **이미지 자체는 저장 안 하고 메타데이터(해상도, 처리 시간)만** 기록.

4. **"HF Spaces 무료 tier에서 SDXL이 돌 거라고 기대"** → 무료 tier는 CPU + 16GB RAM. SDXL은 VRAM 14GB 필요. CPU 모드에서는 1 이미지 5분 소요. **Pro tier A10G**($9/월)으로 실질 데모 가능. 또는 CPU fallback + "추론 시간 오래 걸림" 공지.

5. **"README에 demo.gif를 빠뜨린다"** → 포트폴리오 평가에서 **gif가 있고 없음의 심리적 차이가 크다**. 5초 gif 하나가 "이 프로젝트는 실제로 작동한다"를 증명. QuickTime 녹화 + `gifski`로 최적화한 5초 GIF (500KB 이하) 필수.

**문제 1.** GitHub repo를 공개한 뒤 1주일간 Star 0개, Issue 0개다. 프로젝트 발견성을 올리는 3가지 실천을 제시하라.

> **해설:** ① **HackerNews·Reddit(/r/MachineLearning) 포스팅** — "조선 고문서 복원 LoRA 공개: SDXL + LCM-LoRA 6-step, 6시간 학습" 같은 기술적 훅 제목. ② **HF Spaces 데모 링크를 README 최상단**에 — 사람들은 코드 전에 "작동하는지"를 보고 싶어 함. ③ **블로그 포스트** — Medium/개인 블로그에 "학습 3번 실패 후 배운 것" 같은 구체 경험담. 추가 전략: 같은 도메인 프로젝트(한국어 NLP·컴퓨터비전) repo에 relevant Issue로 조심스럽게 언급, 컨퍼런스(한국소프트웨어공학회·NAVER DEVIEW) 포스터 세션 지원.

**문제 2.** 한 사용자가 "모델이 내가 올린 조선시대 지도를 엉뚱한 그림으로 바꿨다"고 Issue를 달았다. 대응 시나리오는?

> **해설:** ① **재현 요청** — "이미지를 익명으로 공유 가능하면 mail 주소" 또는 유사 공개 이미지 링크. job_id가 있으면 서버 로그 재현 가능. ② **원인 진단** — ③-1 지도는 학습 데이터에 부족한 도메인 → 한계 인정 ③-2 Otsu 마스크가 지도 범례·라벨 전체를 잡아 50%+ → 마스크 과다 ③ **개선 약속** — 다음 학습 데이터에 지도류 추가 or "지도 복원용 v2" 로드맵 제시. ④ **Model Card 업데이트** — "조선시대 지도는 제한적 성능" 한계 명시. Issue 응답은 방어적이지 않고 **기술적 솔직함 + 개선 약속**.

**문제 3.** 이 책에서 완성한 프로젝트를 취업/대학원/창업 포트폴리오로 활용할 때, 어떤 스토리라인이 가장 효과적인가?

> **해설:** **"실패 → 학습 → 성공 → 수치" 4단 스토리**. ① **실패**: "초기 100장 데이터로 돌려보니 모델이 같은 패턴만 반복, 2,000 스텝에서 포기" ② **학습**: "라이선스 검증·마스크 다양성·Bayesian 스윕 등 19개 챕터를 직접 구현" ③ **성공**: "최종 Elo 1547로 4개 체크포인트 1위, 인간 평가자 합의 Kappa 0.72" ④ **수치**: "$8 학습비·6시간·10,112 라이선스 검증 이미지". 3~5분 발표 기준. 취업은 **재현 가능한 코드** + 기술적 통찰 강조, 대학원은 **실패 교훈과 향후 연구 방향**, 창업은 **시장 타당성(문화재청·박물관·고문서 소장가 수요)** 을 추가.

## 다음 단계 — 실전 책이 끝났다

20개 챕터를 완주했다면 이제 당신은:

- SDXL Inpainting LoRA 학습·평가·배포 전 과정을 독립적으로 수행할 수 있다
- 학습 데이터 1만 장 이상을 라이선스 검증·전처리·WebDataset 포맷까지 다룰 수 있다
- A100 1장으로 6시간·$8에 배포 가능한 모델을 학습시킬 수 있다
- 메트릭·인간 평가·드리프트 감지까지 운영 레이어를 설계할 수 있다
- GitHub·Hugging Face·Streamlit으로 포트폴리오를 완성할 수 있다

### 다음으로 향할 곳

이 책은 **Inpainting** 태스크에 집중했다. 다음 도전:

- **ControlNet 파인튜닝**: 마스크 대신 스케치/엣지/깊이 조건
- **DreamBooth**: 소량 데이터(5~20장)로 특정 스타일·인물 학습
- **Video Inpainting**: 시간축을 추가한 영상 복원 (AnimateDiff·CogVideoX)
- **ControlNet + IPAdapter**: 레퍼런스 이미지 기반 제어
- **Self-host LLM + 이미지 모델**: Llama 4 + Flux 같은 하이브리드 파이프라인

### 마지막 말

기술 스택은 매 6개월마다 변한다. 하지만 이 책에서 체득한 **"데이터 → 학습 → 평가 → 배포 → 관측"** 의 루프는 어떤 신규 모델에도 적용된다. SDXL이 Flux로 바뀌어도, LoRA가 DoRA로 진화해도, **라이선스 검증·마스크 다양성·체크포인트 재개·인간 평가·드리프트 감지**의 원칙은 변하지 않는다.

20개 챕터를 완주했다면 당신은 이미 "읽기만 하는" 단계를 벗어났다. 다음 단계는 **당신이 발견한 실패·교훈을 다시 기록**하는 것이다 — 블로그 포스트, Issue 기여, 후속 모델 공개로.

**행운을 빈다.**
