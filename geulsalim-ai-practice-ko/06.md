# 챕터 06 · SDXL Inpainting 베이스모델 해부

## 학습 목표
- SDXL Inpainting 파이프라인의 UNet·VAE·Text Encoder 세 컴포넌트 역할을 각각 설명할 수 있다
- 텍스트→이미지 UNet(in_channels=4)과 Inpainting UNet(in_channels=9)의 입력 채널 차이를 이해한다
- LoRA를 주입할 어텐션 레이어 위치를 UNet 모듈 트리에서 식별할 수 있다

---

## 이론

### SDXL Inpainting 파이프라인 전체 구조

```
입력: (원본 이미지, 마스크, 텍스트 프롬프트)

  ┌─────────── VAE Encoder ───────────┐
  원본 이미지 (1024×1024×3) → latent (128×128×4)

  ┌─────────── 마스크 처리 ───────────┐
  마스크 (1024×1024×1) → 다운샘플 (128×128×1)
  마스크된 latent = latent × (1 - mask_down)

  ┌─────────── Text Encoder ──────────┐
  프롬프트 → CLIP ViT-L + OpenCLIP ViT-G → text_embed (77×2048)

  ┌─────────── UNet (9 채널 입력) ─────┐
  입력: [노이즈(4) + 마스크된 latent(4) + 마스크(1)] = 9 채널
  조건: text_embed + time_step + pooled_embed
  출력: 예측된 노이즈 ε

  ┌─────────── Sampler (DDIM/DPM++) ──┐
  50 step 반복 샘플링

  ┌─────────── VAE Decoder ───────────┐
  예측 latent (128×128×4) → 복원 이미지 (1024×1024×3)
```

### 텍스트→이미지 UNet vs Inpainting UNet

| | SDXL text-to-image | SDXL Inpainting |
|---|---|---|
| `in_channels` | 4 (latent만) | **9** (latent 4 + masked_latent 4 + mask 1) |
| 베이스모델 | `stabilityai/stable-diffusion-xl-base-1.0` | `stabilityai/stable-diffusion-xl-base-1.0` + SDv2-Inpainting 가중치 이식 또는 `diffusers/stable-diffusion-xl-1.0-inpainting-0.1` |
| 학습 목적 | 프롬프트만으로 생성 | 마스크 영역만 생성, 나머지 보존 |

**핵심 차이**: Inpainting UNet은 `conv_in` 레이어가 9채널 입력을 받도록 확장됐다. 이 레이어는 Stability AI가 text-to-image 가중치 위에 **추가 학습**한 결과라, 저차(rank 16) LoRA로 수정하기엔 표현력이 부족하다. **LoRA 학습 시 `conv_in`에 어댑터를 붙이면 절대 안 된다** — 사전학습 9채널 매핑을 파괴하면 처음부터 재학습해야 하고, 재학습 비용은 A100 기준 $2,000 이상이다. `conv_in`·`conv_out`은 건드리지 말고 어텐션 프로젝션(to_q/k/v/out)만 타겟팅한다.

### UNet 구조 트리 (주요 경로)

```
UNet2DConditionModel
├─ conv_in                    ← 9ch 입력 (LoRA 금지)
├─ time_embedding
├─ add_embedding              ← pooled text + size
├─ down_blocks (3개)
│  ├─ DownBlock2D             ← ResNet만 (어텐션 없음)
│  ├─ CrossAttnDownBlock2D    ← ResNet + Transformer ⭐
│  └─ CrossAttnDownBlock2D    ← ResNet + Transformer ⭐
├─ mid_block                  ← CrossAttn ⭐
├─ up_blocks (3개)
│  ├─ CrossAttnUpBlock2D      ← Transformer ⭐
│  ├─ CrossAttnUpBlock2D      ← Transformer ⭐
│  └─ UpBlock2D               ← ResNet만
└─ conv_out                   ← 4ch 출력 (LoRA 금지)
```

Transformer 블록 내부:
```
Transformer2DModel
└─ transformer_blocks
   └─ BasicTransformerBlock
      ├─ attn1 (self-attention)      ← to_q, to_k, to_v, to_out.0 ⭐
      ├─ attn2 (cross-attention)     ← to_q, to_k, to_v, to_out.0 ⭐
      └─ ff (feed-forward)            ← proj (선택)
```

⭐ 표시 = LoRA 주입 후보. 2026년 표준 구성: `["to_q", "to_k", "to_v", "to_out.0"]` 4개 모듈.

### 왜 어텐션만 건드리는가

UNet 총 파라미터 ≈ 2.6B. 어텐션 프로젝션 (to_q/k/v/out) 합계 ≈ 400M. **전체 학습**은 A100 80GB 4장이 필요하고 3일 걸리지만, **LoRA rank 16**은 학습 파라미터 15M (전체의 0.6%)로 A100 1장·6시간에 끝난다.

어텐션은 "무엇을 어디에 넣을지" 결정하는 층이다. 스타일·컨셉 학습은 여기서 일어난다. 픽셀-레벨 디테일은 ResNet에 묶여 있으므로 건드리지 않아도 된다.

## 실전 코드

```python
# SDXL Inpainting 모델 로드 + 구조 분석 + LoRA 타깃 식별
# pip install diffusers transformers accelerate torch

import torch
from diffusers import StableDiffusionXLInpaintPipeline, AutoencoderKL, UNet2DConditionModel
from collections import defaultdict


MODEL_ID = "diffusers/stable-diffusion-xl-1.0-inpainting-0.1"
DEVICE   = "cuda" if torch.cuda.is_available() else "cpu"
DTYPE    = torch.float16


# ──────────────────────────────────────────────
# 1. 파이프라인 로드 + 컴포넌트 분리
# ──────────────────────────────────────────────
def load_inpaint_pipeline():
    """SDXL Inpainting 파이프라인 로드"""
    pipe = StableDiffusionXLInpaintPipeline.from_pretrained(
        MODEL_ID,
        torch_dtype = DTYPE,
        variant     = "fp16",
        use_safetensors = True,
    )
    pipe.to(DEVICE)
    pipe.enable_vae_slicing()          # 메모리 절약
    pipe.enable_xformers_memory_efficient_attention()

    return {
        "pipe":         pipe,
        "unet":         pipe.unet,
        "vae":          pipe.vae,
        "text_encoder": pipe.text_encoder,
        "text_encoder_2": pipe.text_encoder_2,
        "scheduler":    pipe.scheduler,
    }


# ──────────────────────────────────────────────
# 2. UNet 구조 진단
# ──────────────────────────────────────────────
def diagnose_unet(unet: UNet2DConditionModel) -> dict:
    """UNet 구조 요약 + LoRA 타깃 모듈 통계"""
    conv_in_channels  = unet.conv_in.in_channels
    conv_out_channels = unet.conv_out.out_channels

    # 타깃 모듈 패턴 수집
    target_patterns = ["to_q", "to_k", "to_v", "to_out.0"]
    module_count = defaultdict(int)
    module_params = defaultdict(int)

    for name, module in unet.named_modules():
        for pat in target_patterns:
            if name.endswith(pat):
                module_count[pat]  += 1
                module_params[pat] += sum(p.numel() for p in module.parameters())

    # 어텐션 블록 위치별 개수
    block_stats = defaultdict(int)
    for name, _ in unet.named_modules():
        if ".attn1." in name and name.endswith("to_q"):
            if   "down_blocks" in name: block_stats["down_self_attn"] += 1
            elif "mid_block"   in name: block_stats["mid_self_attn"]  += 1
            elif "up_blocks"   in name: block_stats["up_self_attn"]   += 1
        if ".attn2." in name and name.endswith("to_q"):
            if   "down_blocks" in name: block_stats["down_cross_attn"] += 1
            elif "mid_block"   in name: block_stats["mid_cross_attn"]  += 1
            elif "up_blocks"   in name: block_stats["up_cross_attn"]   += 1

    total_params   = sum(p.numel() for p in unet.parameters())
    trainable_candidate = sum(module_params.values())

    return {
        "total_params":        total_params,
        "conv_in_channels":    conv_in_channels,
        "conv_out_channels":   conv_out_channels,
        "is_inpainting":       conv_in_channels == 9,
        "module_count":        dict(module_count),
        "module_params":       dict(module_params),
        "block_stats":         dict(block_stats),
        "lora_candidate_params": trainable_candidate,
        "lora_candidate_ratio": trainable_candidate / total_params,
    }


# ──────────────────────────────────────────────
# 3. LoRA 타깃 모듈 이름 수집
# ──────────────────────────────────────────────
def collect_lora_targets(
    unet: UNet2DConditionModel,
    target_suffixes: list = None,
) -> list:
    """LoRA 삽입 대상 모듈 전체 경로 반환"""
    if target_suffixes is None:
        target_suffixes = ["to_q", "to_k", "to_v", "to_out.0"]

    targets = []
    for name, module in unet.named_modules():
        for suf in target_suffixes:
            if name.endswith(suf) and isinstance(module, torch.nn.Linear):
                targets.append(name)
                break
    return targets


# ──────────────────────────────────────────────
# 4. VAE + Text Encoder 간단 요약
# ──────────────────────────────────────────────
def summarize_components(components: dict) -> None:
    """각 컴포넌트 파라미터 수 요약"""
    def count_params(m): return sum(p.numel() for p in m.parameters())

    print("=" * 60)
    print("SDXL Inpainting 파이프라인 컴포넌트")
    print("=" * 60)
    print(f"UNet:             {count_params(components['unet'])         / 1e9:.2f}B")
    print(f"VAE:              {count_params(components['vae'])          / 1e6:.1f}M")
    print(f"Text Encoder 1:   {count_params(components['text_encoder']) / 1e6:.1f}M")
    print(f"Text Encoder 2:   {count_params(components['text_encoder_2'])/ 1e6:.1f}M")
    print(f"Scheduler:        {type(components['scheduler']).__name__}")


# ──────────────────────────────────────────────
# 5. 메인
# ──────────────────────────────────────────────
if __name__ == "__main__":
    comps = load_inpaint_pipeline()
    summarize_components(comps)

    diag = diagnose_unet(comps["unet"])
    print("\n" + "=" * 60)
    print("UNet 진단")
    print("=" * 60)
    print(f"전체 파라미터:        {diag['total_params'] / 1e9:.2f}B")
    print(f"conv_in 채널:         {diag['conv_in_channels']} (Inpainting={diag['is_inpainting']})")
    print(f"conv_out 채널:        {diag['conv_out_channels']}")
    print(f"LoRA 후보 모듈:       {diag['module_count']}")
    print(f"LoRA 후보 파라미터:   {diag['lora_candidate_params'] / 1e6:.1f}M "
          f"({diag['lora_candidate_ratio']*100:.2f}%)")
    print(f"어텐션 블록 분포:     {diag['block_stats']}")

    targets = collect_lora_targets(comps["unet"])
    print(f"\nLoRA 타깃 모듈 총 {len(targets)}개 (처음 5개):")
    for t in targets[:5]:
        print(f"  {t}")
```

## 함정 카드

1. **"text-to-image SDXL 베이스모델을 Inpainting LoRA로 학습한다"** → `conv_in`이 4채널이라 학습은 되지만 추론 시 9채널 입력을 받지 못한다. 반드시 Inpainting 전용 체크포인트(`diffusers/stable-diffusion-xl-1.0-inpainting-0.1`)에서 시작.

2. **"`conv_in`·`conv_out`에 LoRA를 붙인다"** → 여기는 입출력 채널 변환 레이어로, LoRA 랭크 저차 근사로는 표현력이 부족해 수렴하지 않는다. 어텐션 프로젝션만 타겟팅.

3. **"Text Encoder도 LoRA로 학습한다"** → 가능하지만 스타일 학습에는 불필요. Text Encoder LoRA는 새로운 토큰(프롬프트 단어 의미)을 추가할 때만 쓴다. 복원 태스크는 시각 품질이 중요하므로 UNet만 학습.

4. **"fp16으로 바로 학습한다"** → SDXL은 fp16에서 NaN이 자주 난다. 반드시 **mixed precision (bf16)** 사용, 또는 weights는 fp32 유지하고 forward만 autocast. 이 대응은 챕터 10에서 다룸.

5. **"`enable_xformers_memory_efficient_attention`을 학습 시 그대로 쓴다"** → Xformers는 추론 가속용이다. 학습 시엔 PyTorch 2.0+ `torch.nn.functional.scaled_dot_product_attention`(SDPA)이 더 안정적이고 그래디언트 흐름도 정확하다.

## 연습문제

**문제 1.** UNet 9채널 입력의 9 = 4 + 4 + 1 구성 의미를 설명하라.

> **해설:** ① **첫 4채널**: 현재 스텝 노이즈 latent (64×64×4, VAE 인코딩 후 해상도) — 샘플링 중 매 스텝 갱신. ② **다음 4채널**: 원본 이미지 latent × (1 - 마스크). 모델이 "보존해야 할 영역"을 알려주는 조건. ③ **마지막 1채널**: 다운샘플된 마스크 (64×64×1, 0=보존 / 1=복원). 모델이 "어디를 생성해야 하는지" 아는 조건. 이 구조 덕에 모델은 마스크 영역만 채우고 나머지는 원본 latent를 그대로 복사해 복원한다.

**문제 2.** LoRA 타깃에 `to_out.0` 대신 `to_out`을 넣으면 왜 실패하는가?

> **해설:** `to_out`은 `nn.ModuleList`로 `[Linear, Dropout]` 두 요소를 가진다. LoRA는 Linear 레이어에만 삽입 가능하므로 ModuleList 자체에 붙으면 타입 오류. 첫 번째 요소 Linear을 가리키는 `to_out.0`이 올바른 경로. 일부 PEFT 버전은 자동 탐지하지만, 명시적으로 `.0`을 쓰는 게 안전.

**문제 3.** `enable_vae_slicing`·`enable_vae_tiling`을 학습 시 쓰면 안 되는 이유는?

> **해설:** `vae_slicing`은 배치를 한 샘플씩 쪼개 메모리 절약하는 추론 최적화 기능이다. 학습 시 전체 배치를 한 번에 역전파해야 그래디언트가 올바르게 누적되므로 slicing을 쓰면 그래디언트가 분리된 서브그래프에서만 흘러 학습 신호가 왜곡된다. `vae_tiling`도 마찬가지로 타일 경계에 아티팩트가 생기고 역전파 경로가 복잡해진다. 메모리 절약은 **gradient checkpointing + mixed precision**으로 해결 (챕터 10).
