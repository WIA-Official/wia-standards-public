# 챕터 07 · LoRA 어댑터 설계 — rank·alpha·타깃 결정

## 학습 목표
- PEFT 라이브러리로 SDXL UNet에 LoRA 어댑터를 삽입하고 학습 가능 파라미터를 검증할 수 있다
- rank·alpha·dropout 세 하이퍼파라미터의 상호작용과 권장값 범위를 근거와 함께 설명할 수 있다
- 학습된 LoRA 가중치를 safetensors로 저장·로드하는 최소 코드를 작성할 수 있다

---

## 이론

### LoRA 수학 — 왜 저차 행렬 두 개인가

UNet 어텐션의 `to_q` 같은 Linear 레이어는 `W ∈ ℝ^{d×d}` 행렬이다 (SDXL 기준 d=1280 또는 2048). 전체 학습은 이 W 전체를 업데이트한다.

LoRA는 `W` 자체를 고정하고 **증분만 저차 분해**로 표현한다:
```
W_effective = W + ΔW
ΔW = B · A     (B ∈ ℝ^{d×r},  A ∈ ℝ^{r×d},  r ≪ d)
```

rank r=16이면 업데이트 파라미터는 `d×r + r×d = 2dr`. d=2048·r=16이면 65,536개 — 원래 `d²=4,194,304`의 1.6%.

### alpha = rank × 2 관행

LoRA 논문은 `α/r`을 스케일 팩터로 쓴다:
```
output = W·x + (α/r) · B · A · x
```

`α`와 `r`이 같으면 스케일 = 1. 실무에서는 **α = r** (스케일 1) 또는 **α = 2r** (스케일 2) 둘 중 하나를 쓴다.
- `α = r`: 안정적, 학습 신호 약함, LR을 2e-4 이상으로 올려야 함
- `α = 2r`: 공격적, 빠른 수렴, 과적합 위험, LR 1e-4 권장

나는 rank 16·alpha 32로 시작했다가 1,000 스텝에서 이미 과적합 조짐(validation SSIM이 떨어지기 시작)을 봤다. **rank 16·alpha 16이 복원 태스크 기본값**으로 안전하다.

### rank 선택 — 태스크 복잡도에 맞춰라

| rank | 용도 | 학습 시간 | 성공률 |
|---|---|---|---|
| 4  | 단일 개념·작은 스타일 | 2시간 | 낮음 (디테일 부족) |
| 8  | 단순 스타일 (컬러·필터) | 3시간 | 중간 |
| 16 | 복잡 스타일·도메인 전이 ⭐ | 6시간 | 높음 |
| 32 | 멀티 스타일·구조 변경 | 10시간 | 매우 높음 (과적합 위험) |
| 64 | 전면 재학습 수준 | 20시간+ | LoRA 의미 약함 |

고문서 Inpainting은 "실제 균열 패턴·손상 질감"을 학습해야 하므로 단순 필터가 아니다. **rank 16이 기본**, 1차 실험 후 품질 부족이면 rank 32로 올린다.

### Dropout 효과 미미함

LoRA Dropout은 B 행렬 출력에 드롭아웃을 적용한다. 이론상 과적합 방지이지만, rank가 이미 작아 표현력 제한이 있어 추가 드롭아웃은 큰 효과 없다. **0.0 또는 0.05** 권장. 그 이상은 학습 신호를 낮추기만 한다.

## 실전 코드

```python
# SDXL UNet LoRA 어댑터 주입 + 검증 + 저장/로드
# pip install peft==0.11+ diffusers transformers safetensors torch

import torch
from pathlib import Path
from diffusers import StableDiffusionXLInpaintPipeline, UNet2DConditionModel
from peft import LoraConfig, get_peft_model, PeftModel
from safetensors.torch import save_file, load_file


MODEL_ID = "diffusers/stable-diffusion-xl-1.0-inpainting-0.1"
DEVICE   = "cuda" if torch.cuda.is_available() else "cpu"


# ──────────────────────────────────────────────
# 1. LoRA 설정
# ──────────────────────────────────────────────
def build_lora_config(
    rank:       int   = 16,
    alpha:      int   = 16,
    dropout:    float = 0.0,
    extended:   bool  = False,
) -> LoraConfig:
    """
    SDXL UNet용 LoRA 설정.

    Args:
        rank:     저차 분해 차원 (권장 16)
        alpha:    스케일 팩터 (rank와 같게 권장)
        dropout:  LoRA 어댑터 드롭아웃 (0.0 기본)
        extended: True면 proj_in/out/conv도 타겟 (VRAM +4GB)
    """
    target_modules = ["to_q", "to_k", "to_v", "to_out.0"]
    if extended:
        target_modules += ["proj_in", "proj_out", "conv1", "conv2"]

    return LoraConfig(
        r              = rank,
        lora_alpha     = alpha,
        target_modules = target_modules,
        lora_dropout   = dropout,
        bias           = "none",
        init_lora_weights = "gaussian",
    )


# ──────────────────────────────────────────────
# 2. UNet에 어댑터 주입
# ──────────────────────────────────────────────
def inject_lora(unet: UNet2DConditionModel, config: LoraConfig):
    """
    UNet에 LoRA 어댑터 주입.
    diffusers는 PEFT backend를 지원하므로 add_adapter를 쓴다.
    (2026-04 기준 diffusers 0.29+ 권장)
    """
    # diffusers native API
    unet.add_adapter(config)

    # 학습 대상만 requires_grad=True
    for name, param in unet.named_parameters():
        if "lora" in name.lower():
            param.requires_grad = True
        else:
            param.requires_grad = False

    return unet


def print_trainable_parameters(model) -> None:
    """학습 가능 / 전체 파라미터 출력"""
    trainable = sum(p.numel() for p in model.parameters() if p.requires_grad)
    total     = sum(p.numel() for p in model.parameters())
    print(f"학습 가능: {trainable:,} ({trainable/total*100:.3f}%)")
    print(f"전체:      {total:,}")
    print(f"고정:      {total - trainable:,}")


# ──────────────────────────────────────────────
# 3. LoRA 가중치 저장/로드
# ──────────────────────────────────────────────
def save_lora_weights(unet: UNet2DConditionModel, output_path: str) -> None:
    """LoRA 가중치만 safetensors로 저장"""
    lora_state = {
        k: v.cpu() for k, v in unet.state_dict().items()
        if "lora" in k.lower()
    }
    Path(output_path).parent.mkdir(parents=True, exist_ok=True)
    save_file(lora_state, output_path)
    size_mb = Path(output_path).stat().st_size / 1024 / 1024
    print(f"LoRA 저장: {output_path} ({size_mb:.1f} MB, {len(lora_state)} 텐서)")


def load_lora_weights(unet: UNet2DConditionModel, weight_path: str) -> None:
    """safetensors에서 LoRA 가중치 복원"""
    state = load_file(weight_path, device="cpu")
    missing, unexpected = unet.load_state_dict(state, strict=False)

    lora_missing = [k for k in missing if "lora" in k.lower()]
    if lora_missing:
        raise RuntimeError(f"LoRA 키 누락: {lora_missing[:3]}...")
    if unexpected:
        print(f"⚠️  예상 외 키: {len(unexpected)}개 (무시)")

    print(f"LoRA 로드: {len(state)} 텐서 복원")


# ──────────────────────────────────────────────
# 4. Sanity 체크 — 순전파·역전파 작동 확인
# ──────────────────────────────────────────────
def sanity_check(unet: UNet2DConditionModel) -> None:
    """주입 직후 순전파/역전파 검증"""
    unet.to(DEVICE).train()

    B, C, H, W = 1, 9, 128, 128
    noise    = torch.randn(B, C, H, W, device=DEVICE, dtype=unet.dtype)
    timestep = torch.tensor([500], device=DEVICE)
    # SDXL은 added_cond_kwargs 필수
    encoder_hidden_states = torch.randn(B, 77, 2048, device=DEVICE, dtype=unet.dtype)
    time_ids  = torch.tensor([[1024, 1024, 0, 0, 1024, 1024]],
                             device=DEVICE, dtype=unet.dtype)
    text_embeds = torch.randn(B, 1280, device=DEVICE, dtype=unet.dtype)

    out = unet(
        noise,
        timestep,
        encoder_hidden_states=encoder_hidden_states,
        added_cond_kwargs={"time_ids": time_ids, "text_embeds": text_embeds},
    ).sample

    assert out.shape == (B, 4, H, W), f"출력 shape 오류: {out.shape}"
    loss = out.mean()
    loss.backward()

    # LoRA 파라미터에 gradient가 실제로 흘렀는지
    lora_with_grad = sum(
        1 for n, p in unet.named_parameters()
        if "lora" in n.lower() and p.grad is not None
    )
    print(f"LoRA gradient 흐름 확인: {lora_with_grad}개 파라미터에 grad")


# ──────────────────────────────────────────────
# 5. 메인
# ──────────────────────────────────────────────
if __name__ == "__main__":
    print(">> SDXL Inpainting UNet 로드")
    pipe = StableDiffusionXLInpaintPipeline.from_pretrained(
        MODEL_ID, torch_dtype=torch.float32,  # 학습은 fp32 로드 후 mixed precision
    )
    unet = pipe.unet

    print("\n>> LoRA 설정 (rank=16, alpha=16)")
    cfg = build_lora_config(rank=16, alpha=16, dropout=0.0)
    print(f"타깃 모듈: {cfg.target_modules}")

    print("\n>> 어댑터 주입")
    unet = inject_lora(unet, cfg)
    print_trainable_parameters(unet)

    print("\n>> Sanity 체크")
    sanity_check(unet)

    print("\n>> LoRA 저장")
    save_lora_weights(unet, "./output/lora_init.safetensors")

    print("\n>> 재로드 검증")
    pipe2 = StableDiffusionXLInpaintPipeline.from_pretrained(
        MODEL_ID, torch_dtype=torch.float32,
    )
    pipe2.unet.add_adapter(cfg)
    load_lora_weights(pipe2.unet, "./output/lora_init.safetensors")
```

**예상 출력:**
```
>> LoRA 설정 (rank=16, alpha=16)
타깃 모듈: ['to_q', 'to_k', 'to_v', 'to_out.0']

>> 어댑터 주입
학습 가능: 15,252,480 (0.593%)
전체:      2,573,126,784
고정:      2,557,874,304

>> Sanity 체크
LoRA gradient 흐름 확인: 1120개 파라미터에 grad

>> LoRA 저장
LoRA 저장: ./output/lora_init.safetensors (61.0 MB, 1120 텐서)
```

## 함정 카드

1. **"`add_adapter` 후 `requires_grad`를 설정 안 한다"** → `add_adapter`는 LoRA만 학습하도록 자동 설정하지만, 일부 diffusers 버전은 기존 파라미터도 grad를 남겨둔다. 명시적으로 `for n, p: p.requires_grad = "lora" in n.lower()`를 해서 이중 안전장치.

2. **"LoRA 주입 후 `unet.train()`을 깜빡한다"** → `eval()` 모드에서는 Dropout이 작동 안 하고 BatchNorm 통계가 고정된다. SDXL은 GroupNorm만 써서 덜 치명적이지만, 어쨌든 학습 루프 진입 전 반드시 `unet.train()` 호출.

3. **"rank를 너무 크게 잡는다"** → rank 64 이상에서는 "LoRA의 장점(파라미터 효율성)"이 사라지면서 과적합만 심해진다. 데이터셋 1만 장 미만이면 rank 16으로 충분. 많은 스타일 학습이면 rank 32.

4. **"LoRA를 UNet에만 주입하고 Text Encoder를 건드린다"** → Text Encoder LoRA는 복원 태스크에서 불필요하며, 두 개를 동시에 학습시키면 학습 속도가 2배 느려지고 수렴 난이도도 올라간다. 스타일 LoRA만 만들 땐 UNet만.

5. **"safetensors 대신 `torch.save`로 pt 파일을 쓴다"** → .pt는 pickle 기반으로 악성 코드 로드 가능. Hugging Face Hub 업로드 시 거부되거나 경고가 뜬다. 항상 `safetensors`로. 크기도 10~20% 작다.

## 연습문제

**문제 1.** rank 16·alpha 32·LR 1e-4로 학습 중 1,500 스텝에서 loss가 발산(NaN)했다. rank·alpha·LR 셋 중 먼저 조정할 것과 이유를 서술하라.

> **해설:** **LR을 먼저 낮춘다** (1e-4 → 5e-5). 이유: α=2r이면 효과적 학습률이 α/r=2배로 증폭되므로 LR 1e-4는 실질 2e-4가 된다. SDXL mixed precision은 2e-4에서 NaN 흔함. rank·alpha를 조정하면 저장된 가중치 구조가 바뀌어 재현성이 흔들리지만, LR 조정은 하이퍼파라미터만 바꾸므로 먼저 시도. 그래도 NaN이면 alpha를 16으로 낮춰 스케일 1로.

**문제 2.** 학습 후 저장한 LoRA safetensors 파일이 61MB가 아니라 10GB로 나왔다. 원인은?

> **해설:** `save_lora_weights`에서 `"lora"` 필터가 작동 안 해 UNet 전체 state_dict를 저장한 경우. 원인 후보: ① PEFT 버전이 다른 키 이름 규칙 사용 (예: `base_layer.lora_A` vs `lora_A.weight`). ② 소문자 비교 실패로 `LoRA`가 매치 안 됨 → `k.lower()` 확인. ③ `add_adapter` 대신 `get_peft_model`을 썼다면 키 접두사가 `base_model.model.`로 다름. 수정: `any(tag in k.lower() for tag in ["lora_a", "lora_b"])`로 필터 더 엄격히.

**문제 3.** Extended 모드(`proj_in/out/conv1/conv2` 추가)가 기본보다 성능이 나쁠 수 있는 경우 2가지를 예시하라.

> **해설:** ① **데이터셋 1만 장 미만**: 추가 파라미터가 늘면 과적합. conv LoRA는 공간 정보에 직접 개입하므로 데이터 부족 시 텍스처 과적합. ② **스타일 전이가 아닌 복원**: 고문서 복원은 원본 내용을 건드리면 안 된다. conv1/conv2 LoRA는 ResNet 피처를 변경해 "복원 시 원본 구조까지 바뀌는" 부작용. 어텐션만 수정하는 기본 설정이 "어떤 콘텐츠가 그 자리에 있어야 하는가" 결정에 집중되어 복원 태스크에 적합.
