# 챕터 09 · 손실 함수 설계 — Masked MSE + Perceptual

## 학습 목표
- 전체 영역 MSE가 Inpainting 태스크에 왜 불충분한지 설명하고, 마스크 가중 MSE를 구현할 수 있다
- LPIPS 기반 perceptual loss를 latent 공간이 아닌 RGB 공간에서 계산하는 코드를 작성할 수 있다
- 세 손실(MSE, masked MSE, LPIPS)의 가중합 하이퍼파라미터를 실험적으로 튜닝하는 절차를 기술할 수 있다

---

## 이론

### 전체 MSE의 결함

챕터 08에서 `F.mse_loss(noise_pred, noise)`를 썼다. 이 loss는 **128×128×4 latent 전체에 균등**하게 적용된다. 마스크 영역이 전체의 20%라면, 80%는 "거의 손대지 말 것"을 학습하고 20%만 "새로 생성"을 학습한다.

결과: **모델이 "안전하게 복사"에 과최적화**되고, 마스크 영역 생성 품질은 정체된다. 실제로 내 첫 학습에서 전체 MSE로만 돌렸을 때, 이미지 전체 SSIM은 0.89까지 올랐지만 마스크 영역만 잘라서 측정하면 SSIM이 0.52밖에 안 됐다. **마스크 안쪽만 보면 복원 품질이 형편없다는 뜻.**

### Masked MSE — 마스크 영역 가중치

```
loss = mse_loss(ε̂_outside, ε_outside) × (1 - w) 
     + mse_loss(ε̂_inside,  ε_inside)  × w
```

보통 `w = 0.7~0.9`. 마스크 영역에 70~90% 가중치를 줘서 모델이 "복원에 집중"하도록 강제한다.

단순 구현:
```python
mse_per_pixel = (noise_pred - noise).pow(2)
loss_outside = (mse_per_pixel * (1 - mask_small)).sum() / (1 - mask_small).sum()
loss_inside  = (mse_per_pixel *      mask_small ).sum() /      mask_small .sum()
loss = 0.2 * loss_outside + 0.8 * loss_inside
```

두 영역을 각각 평균내고 가중 합한다. `.sum() / mask.sum()`이 핵심 — 마스크 면적으로 정규화하지 않으면 큰 마스크 샘플만 loss가 커진다.

### Perceptual Loss — LPIPS

MSE는 "픽셀 값 차이"만 본다. 하지만 사람 눈엔 비슷해 보여도 MSE가 큰 경우가 많다(예: 1픽셀 시프트). **LPIPS(Learned Perceptual Image Patch Similarity)** 는 VGG 피처 공간에서 거리를 재, 지각적 유사성에 맞춘다.

LPIPS는 **RGB 이미지**에 대해 작동한다. Latent 공간에서는 의미가 없다. 따라서:
1. UNet 예측 `ε̂` → scheduler로 `ẑ₀` 추정 → VAE 디코딩 → 예측 RGB
2. 원본 latent `z₀` → VAE 디코딩 → 타깃 RGB
3. `LPIPS(예측 RGB, 타깃 RGB)` 계산

**비용**: VAE 디코딩이 매 스텝 추가되면 학습 속도가 30% 느려진다. 그래서 **10 스텝마다 한 번**만 LPIPS를 계산하는 전략이 현실적이다.

### 3 손실 가중합

```
loss_total = λ_mse · L_mse + λ_masked · L_masked + λ_lpips · L_lpips

권장 초기값: λ_mse = 1.0, λ_masked = 1.5, λ_lpips = 0.1
```

LPIPS 절대값이 MSE보다 10배 크기 때문에 λ_lpips를 작게 잡는다. 학습 로그에서 세 loss를 분리 기록하고 **각 loss의 크기가 비슷한 스케일**이 되도록 weight를 조정한다.

## 실전 코드

```python
# Masked MSE + LPIPS Perceptual Loss 구현
# pip install lpips torch torchvision

import torch
import torch.nn.functional as F
from typing import Optional


# ──────────────────────────────────────────────
# 1. Masked MSE
# ──────────────────────────────────────────────
def masked_mse_loss(
    pred:        torch.Tensor,       # (B, 4, H, W) 예측 노이즈
    target:      torch.Tensor,       # (B, 4, H, W) 정답 노이즈
    mask_small:  torch.Tensor,       # (B, 1, H, W) [0, 1]
    inside_weight: float = 0.8,
) -> dict:
    """
    마스크 안팎 분리 MSE. 각 영역 면적 정규화 후 가중합.

    Returns:
        {"loss": 총합, "mse_inside": 안쪽, "mse_outside": 바깥}
    """
    # (B, 4, H, W) 픽셀별 제곱오차
    se = (pred.float() - target.float()).pow(2)  # mean over channels 생략 — channel별 독립

    # 4채널 latent 동일 마스크 적용
    mask_b = mask_small.expand_as(se)            # (B, 4, H, W)

    inside_count  = mask_b.sum().clamp(min=1.0)
    outside_count = (1 - mask_b).sum().clamp(min=1.0)

    mse_in  = (se * mask_b).sum()       / inside_count
    mse_out = (se * (1 - mask_b)).sum() / outside_count

    loss = inside_weight * mse_in + (1 - inside_weight) * mse_out
    return {
        "loss":        loss,
        "mse_inside":  mse_in.detach(),
        "mse_outside": mse_out.detach(),
    }


# ──────────────────────────────────────────────
# 2. LPIPS Perceptual Loss (RGB 공간)
# ──────────────────────────────────────────────
class LPIPSLoss:
    """
    LPIPS는 [-1, 1] RGB 입력을 기대한다.
    VGG 기반이라 VRAM ~500MB 추가 소모.
    """
    def __init__(self, device: str = "cuda"):
        import lpips
        self.model = lpips.LPIPS(net="vgg").to(device).eval()
        for p in self.model.parameters():
            p.requires_grad = False

    @torch.no_grad()
    def decode_latent_to_rgb(self, latent: torch.Tensor, vae) -> torch.Tensor:
        """latent → RGB [-1, 1]"""
        # SDXL VAE scaling_factor = 0.13025
        # encode 시 × factor, decode 시 ÷ factor (챕터 08 참조)
        latent = latent / vae.config.scaling_factor
        return vae.decode(latent.to(vae.dtype)).sample.clamp(-1, 1).float()

    def __call__(
        self,
        pred_latent:   torch.Tensor,
        target_latent: torch.Tensor,
        vae,
    ) -> torch.Tensor:
        pred_rgb   = self.decode_latent_to_rgb(pred_latent,   vae)
        target_rgb = self.decode_latent_to_rgb(target_latent, vae)
        # LPIPS는 (B, 3, H, W) 입력
        return self.model(pred_rgb, target_rgb).mean()


# ──────────────────────────────────────────────
# 3. Scheduler로 ε̂ → ẑ₀ 추정
# ──────────────────────────────────────────────
def predict_z0_from_noise(
    noise_pred: torch.Tensor,
    noisy_latent: torch.Tensor,
    timesteps: torch.Tensor,
    noise_scheduler,
) -> torch.Tensor:
    """
    DDPM 역방향: noisy_latent와 예측 ε로부터 원본 latent 추정.
    ẑ₀ = (z_t - σ_t · ε̂) / α_t
    """
    # 누적 α_t
    alphas_cumprod = noise_scheduler.alphas_cumprod.to(noise_pred.device)
    alpha_t = alphas_cumprod[timesteps].view(-1, 1, 1, 1)
    sigma_t = (1 - alpha_t).sqrt()
    alpha_sqrt = alpha_t.sqrt()

    z0_pred = (noisy_latent - sigma_t * noise_pred) / alpha_sqrt.clamp(min=1e-6)
    return z0_pred


# ──────────────────────────────────────────────
# 4. 통합 Loss
# ──────────────────────────────────────────────
class InpaintingLoss:
    """
    3 손실 가중합.
    LPIPS는 10 스텝에 한 번만 계산(비용 절약).
    """
    def __init__(
        self,
        lambda_mse:     float = 1.0,
        lambda_masked:  float = 1.5,
        lambda_lpips:   float = 0.1,
        lpips_interval: int   = 10,
        device:         str   = "cuda",
    ):
        self.l_mse    = lambda_mse
        self.l_masked = lambda_masked
        self.l_lpips  = lambda_lpips
        self.lpips_interval = lpips_interval
        self.lpips = LPIPSLoss(device) if lambda_lpips > 0 else None
        self._step = 0

    def __call__(
        self,
        noise_pred:    torch.Tensor,
        noise:         torch.Tensor,
        noisy_latents: torch.Tensor,
        latents:       torch.Tensor,
        timesteps:     torch.Tensor,
        mask_small:    torch.Tensor,
        noise_scheduler,
        vae,
    ) -> dict:
        self._step += 1
        logs = {}

        # (1) 전체 MSE
        loss_mse = F.mse_loss(noise_pred.float(), noise.float())
        logs["loss_mse"] = loss_mse.detach()

        # (2) Masked MSE
        md = masked_mse_loss(noise_pred, noise, mask_small, inside_weight=0.8)
        loss_masked = md["loss"]
        logs["loss_masked"]   = loss_masked.detach()
        logs["mse_inside"]    = md["mse_inside"]
        logs["mse_outside"]   = md["mse_outside"]

        total = self.l_mse * loss_mse + self.l_masked * loss_masked

        # (3) LPIPS (주기적만)
        if self.lpips is not None and self._step % self.lpips_interval == 0:
            z0_pred = predict_z0_from_noise(noise_pred, noisy_latents, timesteps, noise_scheduler)
            loss_lpips = self.lpips(z0_pred, latents, vae)
            logs["loss_lpips"] = loss_lpips.detach()
            total = total + self.l_lpips * loss_lpips

        logs["loss_total"] = total.detach()
        return {"loss": total, "logs": logs}


# ──────────────────────────────────────────────
# 5. 사용 예시
# ──────────────────────────────────────────────
if __name__ == "__main__":
    # Mock tensors
    B, C, H, W = 2, 4, 128, 128
    noise_pred    = torch.randn(B, C, H, W, device="cuda", requires_grad=True)
    noise         = torch.randn(B, C, H, W, device="cuda")
    noisy_latents = torch.randn(B, C, H, W, device="cuda")
    latents       = torch.randn(B, C, H, W, device="cuda")
    timesteps     = torch.randint(0, 1000, (B,), device="cuda")
    mask_small    = (torch.rand(B, 1, H, W, device="cuda") > 0.7).float()

    from diffusers import DDPMScheduler, AutoencoderKL
    scheduler = DDPMScheduler.from_pretrained(
        "diffusers/stable-diffusion-xl-1.0-inpainting-0.1", subfolder="scheduler",
    )
    vae = AutoencoderKL.from_pretrained(
        "diffusers/stable-diffusion-xl-1.0-inpainting-0.1", subfolder="vae",
    ).to("cuda", dtype=torch.float16)

    loss_fn = InpaintingLoss(lambda_mse=1.0, lambda_masked=1.5, lambda_lpips=0.1)
    out = loss_fn(noise_pred, noise, noisy_latents, latents, timesteps,
                  mask_small, scheduler, vae)
    print(f"총 loss: {out['loss'].item():.4f}")
    for k, v in out["logs"].items():
        print(f"  {k}: {v.item():.4f}")

    out["loss"].backward()
    print(f"\nnoise_pred.grad 존재: {noise_pred.grad is not None}")
```

**학습 초반 로그 예시 (내 실측):**
```
step    0 | loss_total 1.24 | mse 0.98 | masked 0.89 | mse_in 0.94 | mse_out 0.85
step  500 | loss_total 0.31 | mse 0.18 | masked 0.21 | mse_in 0.24 | mse_out 0.13
step 5000 | loss_total 0.14 | mse 0.09 | masked 0.11 | mse_in 0.13 | mse_out 0.06
               ↑ mse_in은 mse_out보다 항상 2배 정도 큼 (예상)
```

`mse_inside > mse_outside`가 정상이다. 마스크 영역이 더 어려우니까. 만약 비슷하거나 반대면 `inside_weight`를 올려라.

## 함정 카드

1. **"마스크 가중치를 0.5로 둔다"** → 0.5는 기본 MSE와 같다. 마스크 영역에 실제로 더 집중시키려면 **0.7 이상**이 필요. 0.9까지 올려도 되지만 그 이상은 outside 복사 품질이 떨어진다. 0.8이 sweet spot.

2. **"LPIPS를 매 스텝 계산한다"** → VAE 디코딩은 1024 해상도에서 배치당 1~2초 추가. 30,000 스텝 × 2초 = 추가 17시간. 10 스텝 간격으로도 학습 신호는 충분하다. 정말 필요하면 `lpips_interval=5`까지.

3. **"LPIPS 모델의 `requires_grad`를 해제 안 한다"** → 기본 PyTorch 모듈은 모든 파라미터가 `requires_grad=True`다. VGG는 학습 대상이 아니므로 반드시 `.eval()` + `requires_grad = False`. 안 그러면 LPIPS 파라미터에도 gradient가 누적되어 VRAM 낭비·학습 혼란.

4. **"3 loss를 모두 동일 스케일로 가중한다"** → MSE는 보통 0.05~0.20, LPIPS는 0.5~2.0. 가중치 1:1:1로 합하면 LPIPS만 loss를 지배한다. 반드시 스케일 맞춘 뒤 가중.

5. **"마스크 영역 면적이 0인 샘플에서 `mse_inside` 계산이 NaN"** → `mask.sum() = 0`이면 `0/0 = NaN`. `.clamp(min=1.0)` 대신 `.clamp(min=1e-6)`은 위험(너무 작으면 수치 불안정). `1.0`이 안전.

## 연습문제

**문제 1.** 학습 중 `mse_inside`가 계속 0.3에서 정체되고 `mse_outside`는 0.02로 잘 떨어진다. 진단과 해결책은?

> **해설:** 모델이 "마스크 바깥 복사"만 쉽게 학습하고 **마스크 안 복원은 전혀 학습 못 하는** 전형적 증상. 원인 3가지: ① `inside_weight`가 너무 낮음 (0.5 → 0.85로) ② 마스크 영역이 과도하게 크거나 복잡해 모델 capacity 부족 (rank 16 → 32) ③ 데이터셋에 "복원 어려운" 극단 마스크만 있음. 해결 순서: inside_weight 상향 → LPIPS 가중치 0.1 → 0.3 → rank 증가 → 데이터 마스크 분포 재조정.

**문제 2.** LPIPS loss 값이 매 스텝 거의 같은 값(0.85±0.02)만 찍힌다. 무엇이 잘못됐는가?

> **해설:** VAE 디코딩에서 `scaling_factor`로 나누는 걸 빠뜨렸거나, `z0_pred`가 극단값이라 디코딩 후 RGB가 전부 같은 색(포화)이 된 경우. 디버그: ① `vae.decode` 결과를 실제 이미지로 저장해 육안 확인 ② `z0_pred.abs().max()`가 100 이상이면 scheduler 추정식 오류 (`alpha_sqrt.clamp(min=1e-6)`이 너무 커 분모 발산) ③ LPIPS 입력이 [-1, 1] 범위 벗어난 경우 — `.clamp(-1, 1)` 명시.

**문제 3.** 어떤 경우 λ_lpips = 0으로 두는 게 나은가? 3가지를 제시하라.

> **해설:** ① **VRAM 부족**: LPIPS는 VGG +500MB, 메모리 한계 시 첫 희생 대상 ② **학습 초기 검증**: 파이프라인 작동 확인용 smoke test에서는 손실 최소화 ③ **Pure latent 학습 전략**: latent 공간에서만 loss를 계산해 학습 속도 최대화 후, 마지막 1,000 스텝만 LPIPS를 켜는 staged training. 반대로 **꼭 켜야 할 때**는 ① 마스크 영역 텍스처 품질 우선 ② SSIM은 좋은데 사람 눈엔 흐린 결과 ③ 생성 모델 대회 심사.
