## 학습 목표
- 2026년 현재 주요 GPU 클라우드(Lambda Labs·RunPod·Vast.ai·AWS Spot)의 가격·가용성을 비교할 수 있다
- On-demand와 Spot/Interruptible 인스턴스의 트레이드오프를 이해하고 체크포인트 전략으로 Spot을 안전하게 활용할 수 있다

### On-demand vs Spot/Interruptible

| 유형 | 장점 | 단점 |
|---|---|---|
| **On-demand** | 중단 없음, 예측 가능 | 비쌈, 부하 여유 시간 낭비 |
| **Spot (AWS/GCP)** | 60~80% 할인 | 2분 전 알림 후 회수 |
| **RunPod Community** | 60~80% 할인 | 가끔 종료, 보안 제한 |
| **Vast.ai** | 최저가, 경매 | 신뢰도 다양, 호스트에 따라 품질 차 |

### 학습 vs 추론 전략 분리

**학습(30k step = 6시간)**: 중단되어도 체크포인트 재개 가능 (챕터 11). **Spot/Community에 최적**. 내 경우 Lambda on-demand $8에서 Vast.ai $2.5로 학습 비용 69% 절감.

**추론 서비스 (24/7)**: 중단 허용 X. **On-demand 또는 Serverless**. 낮은 트래픽엔 **Serverless GPU (RunPod Serverless, Replicate, Modal)** 이 유리 — 요청당 과금.

| 트래픽 | 전략 | 월 비용 (A100 기준) |
|---|---|---|
| 0~100 req/day | Serverless | ~$30 |
| 100~1000 req/day | Serverless or 작은 on-demand | $50~$200 |
| 1000~10000 req/day | on-demand GPU 1대 | $900~$1,500 |
| 10000+ req/day | auto-scale 여러 대 | $2,000+ |

### Serverless GPU 비교 (요청당 과금)

| 서비스 | A100 초당 비용 | 콜드 스타트 | 메모 |
|---|---|---|---|
| **Replicate** | ~$0.0015/s | 15~60s | 모델 공개 시 인기 서비스, 직접 배포 |
| **Modal** | ~$0.0011/s | 5~15s | Python 친화적, 스냅샷 지원 |
| **RunPod Serverless** | ~$0.0010/s | 10~30s | 커스텀 Docker, 가장 유연 |
| **Hugging Face Inference Endpoints** | ~$0.0018/s | 30~90s | HF Hub 연동, 간편하나 비쌈 |

**Replicate** 선택 기준: 복원 2.5초 × $0.0015 = $0.004/요청. 하루 1,000 요청 = $4 = 월 $120.

### 추론 최적화로 Serverless 비용 추가 절감

- LCM-LoRA 6-step → 2.5초 대신 0.5초 → 비용 5배 절감
- 콜드 스타트 최소화: **model pre-load** + **컨테이너 warm pool** (Modal 기본 제공)
- 배치 모드: N 이미지를 1 요청에 → 인당 추론 시간 60% 감소

### 1. Spot 학습을 위한 자동 재개 스크립트

```bash
#!/usr/bin/env bash
# spot-train.sh — Spot 인스턴스에서 학습 자동 재개
# crontab: @reboot /root/spot-train.sh &>> /root/train.log

set -e

OUTPUT_DIR="/workspace/training_output"
S3_BUCKET="s3://my-restoration-checkpoints"

# (1) 이전 체크포인트 S3에서 동기화
aws s3 sync $S3_BUCKET/$OUTPUT_DIR $OUTPUT_DIR --quiet

# (2) 학습 실행 (재개 모드)
accelerate launch \
    --num_processes=1 --mixed_precision=bf16 \
    train.py --resume_from=latest \
             --total_steps=30000 \
             --output_dir=$OUTPUT_DIR

# (3) 학습 완료 시 S3 백업 (cron으로 10분마다 실행해도 됨)
aws s3 sync $OUTPUT_DIR $S3_BUCKET/$OUTPUT_DIR --quiet

echo "학습 완료: $(date)"
```

**주기적 체크포인트 S3 업로드** (학습 중):

```python
    ])  # 비동기
```

### 2. Modal로 Serverless 배포

```python
# modal_deploy.py — Modal 기반 Serverless 추론
# pip install modal

import modal

app = modal.App("inpaint-service")

image = (
    modal.Image.debian_slim(python_version="3.11")
    .apt_install("ffmpeg", "libsm6", "libxext6")
    .pip_install(
        "torch==2.4", "diffusers==0.29", "transformers==4.43",
        "peft==0.11", "safetensors", "accelerate", "Pillow",
    )
)

# LoRA 가중치를 Modal Volume에 저장
volume = modal.Volume.from_name("inpaint-models", create_if_missing=True)


@app.cls(
    gpu="A10G",              # $0.60/h, LCM 모드면 충분
    image=image,
    volumes={"/models": volume},
    container_idle_timeout=300,    # 5분 놀면 종료
    concurrency_limit=10,          # 동시 10 컨테이너까지
    timeout=120,
)
class InpaintService:
    @modal.enter()
    def load_model(self):
        """컨테이너 시작 시 1회 모델 로드"""
        from diffusers import StableDiffusionXLInpaintPipeline, LCMScheduler
        from peft import LoraConfig
        from safetensors.torch import load_file
        import torch

        self.pipe = StableDiffusionXLInpaintPipeline.from_pretrained(
            "diffusers/stable-diffusion-xl-1.0-inpainting-0.1",
            torch_dtype=torch.float16, variant="fp16",
        ).to("cuda")

        lora_cfg = LoraConfig(r=16, lora_alpha=16,
            target_modules=["to_q", "to_k", "to_v", "to_out.0"])
        self.pipe.unet.add_adapter(lora_cfg, adapter_name="style")

        state = load_file("/models/lora_final.safetensors", device="cpu")
        self.pipe.unet.load_state_dict(state, strict=False)

        self.pipe.load_lora_weights(
            "latent-consistency/lcm-lora-sdxl", adapter_name="lcm",
        )
        self.pipe.set_adapters(["style", "lcm"], adapter_weights=[1.0, 1.0])
        self.pipe.scheduler = LCMScheduler.from_config(self.pipe.scheduler.config)
        self.pipe.set_progress_bar_config(disable=True)

    @modal.method()
    def restore(self, image_bytes: bytes, mask_bytes: bytes,
                prompt: str = "Korean document restoration") -> bytes:
        import io, torch
        from PIL import Image

        img  = Image.open(io.BytesIO(image_bytes)).convert("RGB").resize((1024, 1024))
        mask = Image.open(io.BytesIO(mask_bytes)).convert("L").resize((1024, 1024))

        with torch.autocast("cuda", dtype=torch.float16):
            result = self.pipe(
                prompt=prompt,
                image=img, mask_image=mask,
                num_inference_steps=6, guidance_scale=2.0,
                generator=torch.Generator("cuda").manual_seed(42),
            ).images[0]

        buf = io.BytesIO()
        result.save(buf, format="JPEG", quality=90)
        return buf.getvalue()


@app.function(image=image)
@modal.web_endpoint(method="POST")
def restore_endpoint(image_bytes: bytes, mask_bytes: bytes):
    service = InpaintService()
    return service.restore.remote(image_bytes, mask_bytes)
```

```python
# cost_simulator.py — 월 비용 시뮬레이션

def estimate_monthly_cost(
    daily_requests:   int,
    avg_seconds:      float,        # 추론 시간
    platform:         str = "modal_a10g",
):
    """다양한 플랫폼 월 비용 추산"""
    rates = {
        "lambda_a100_40":    (1.29, "on-demand"),
        "runpod_community":  (0.59, "community"),
        "modal_a10g":        (0.0006, "serverless_per_sec"),  # $2.16/h
        "replicate_a100":    (0.0015, "serverless_per_sec"),
        "runpod_serverless": (0.0010, "serverless_per_sec"),
    }
    rate, billing = rates[platform]

    if billing == "serverless_per_sec":
        # 추론 시간만 과금
        monthly = daily_requests * 30 * avg_seconds * rate
    else:
        # 시간당 고정 (24/7 가정)
        monthly = rate * 24 * 30

    return {
        "platform":       platform,
        "monthly_usd":    round(monthly, 2),
        "billing_model":  billing,
    }


if __name__ == "__main__":
    for daily in [100, 1000, 10000]:
        print(f"\n=== 하루 {daily} 요청 (평균 2.5초 추론) ===")
        for p in ["modal_a10g", "replicate_a100", "runpod_serverless",
                  "lambda_a100_40", "runpod_community"]:
            r = estimate_monthly_cost(daily, 2.5, p)
            print(f"  {r['platform']:22s}: ${r['monthly_usd']:>8.2f}/월  ({r['billing_model']})")
```

## 함정 카드

1. **"Vast.ai 최저가 호스트를 아무거나 고른다"** → 신뢰도 점수(`reliability`) 95% 미만 호스트는 갑자기 끊기거나 네트워크 느림. **최소 98% + GPU 완전 사용 가능(dph_per_gpu_idle=0)** 필터 후 가격 순 정렬. 가격 차 20%는 안정성으로 만회.

2. **"Spot 인스턴스 학습에서 S3 동기화를 잊는다"** → Spot 회수 시 로컬 디스크도 날아감. **10분마다 S3 sync** 또는 persistent volume(EBS gp3) 필수. 내 경우 S3 sync 없이 돌리다 4시간 학습이 순식간에 사라짐 경험 2회.

3. **"Modal/Replicate 콜드 스타트를 로드 테스트에 반영 안 한다"** → 첫 요청 20초, 두 번째 2초로 **사용자 경험 불균등**. 해결: `container_idle_timeout`을 트래픽 패턴 맞춰 조정 (300~600초), 또는 주기적 "keep-alive" 요청으로 warm 상태 유지.

4. **"RunPod Community GPU에 민감한 고객 데이터"** → Community는 개인 호스트 물리 머신 공유. **고객 이미지가 해당 호스트에 남을 가능성** 있음. NDA/법적 요구가 있는 복원 작업은 Secure Cloud 또는 개인 소유 GPU로.

5. **"월 예산 알림 없이 돌린다"** → 실수로 A100 4대 병렬 학습 걸어두고 휴가 다녀오면 $3,000 고지서. AWS/GCP는 예산 알림 필수, Lambda/RunPod도 대시보드 확인 습관. 스팟은 **최대 기간(`--max_runtime=24h`) 명시**.

## 연습문제

**문제 1.** 학습 1회 = 30,000 step × A100 40GB × 6시간. Lambda on-demand vs Vast.ai 경매 vs RunPod Community 각 플랫폼의 총 비용은? (평균가 기준)

> **해설:** ① **Lambda on-demand**: $1.29/h × 6h = **$7.74** (중단 없음) ② **Vast.ai 경매**: 평균 $0.60/h × 6h = **$3.60** (중단 가능성, reliability 98%+면 중단 드묾) ③ **RunPod Community**: $0.59/h × 6h = **$3.54** (가끔 종료 알림). Spot 학습 시 S3 sync 오버헤드 ~5% 감안해 **$3.8~$4.0**이 현실적. 60% 절감.

**문제 2.** 하루 평균 200 요청, 피크 시간 500 요청/h이면 Serverless vs on-demand 중 어느 쪽을 쓸 것인가?

> **해설:** 하루 200 × 2.5초 = 500초/일 실제 GPU 사용. On-demand 24시간 × 30일 = 2,592,000초. 사용률 500/86,400 ≈ **0.6%**. **Serverless 압도적 경제적** (Modal A10G ≈ $9/월 vs on-demand $900/월). 피크 500/h는 초당 0.14 = 실시간 GPU 1개로 충분하지만 Serverless라도 concurrency_limit 5~10으로 병렬 처리 가능.

**문제 3.** 학습 6시간 중 4시간 30분 경과 시점에 Spot 회수 알림이 왔다. 2분 안에 무엇을 해야 하는가?

> **해설:** ① **긴급 체크포인트 저장** — `accelerator.save_state` 즉시 호출 ② **S3 최종 sync** — 백그라운드 sync가 한 번 더 실행되도록 `aws s3 sync ... --only-show-errors` 포그라운드 실행 ③ **W&B로 현재 state 보고** — 재개 정보 전송. 대부분 Spot 회수 알림은 **2분 전**이므로 빠른 저장 가능. 재개는 다른 Spot 인스턴스에서 `--resume_from=latest`로 즉시 이어감. 챕터 11의 CheckpointManager가 이 시나리오 핵심 대응.
