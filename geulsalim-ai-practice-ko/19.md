# 챕터 19 · 프로덕션 배포 — HF Endpoints · Docker · 모니터링 · 드리프트

## 학습 목표
- Hugging Face Inference Endpoints에 LoRA 어댑터 모델을 배포하고 사용자 정의 handler를 작성할 수 있다
- Docker + Kubernetes(HPA)로 자체 인프라에 auto-scale 배포를 구축할 수 있다
- Sentry/Grafana/Prometheus로 에러·지연·실패율을 실시간 감시하고 데이터 드리프트를 자동 감지할 수 있다

## 이론

### 배포 옵션 비교 (2026-04 기준)

| 옵션 | 운영 부담 | 비용 모델 | 확장성 | 적합 상황 |
|---|---|---|---|---|
| **HF Inference Endpoints** | 낮음 | 시간당 고정 | 수동 | 빠른 출시, HF Hub 이미 사용 |
| **Modal/Replicate Serverless** | 매우 낮음 | 요청당 | 자동 | 트래픽 불규칙 |
| **Docker + RunPod/Lambda** | 중간 | 시간당 고정 | 수동 | 비용 최적화, 커스텀 |
| **Kubernetes(k8s) + HPA** | 높음 | 노드 비용 | 자동 | 대규모, 다중 모델 |
| **직접 베어메탈 + Nginx** | 매우 높음 | 하드웨어 고정 | 수동 | 극한 비용 절감 |

### 빠른 출시 경로 (우선순위)

1. **HF Inference Endpoints**로 MVP 배포 (30분)
2. 트래픽 검증되면 **Modal 또는 Docker**로 비용 절감
3. 월 $2,000+ 트래픽이면 **k8s**로 자체 인프라

### 모니터링 계층

**Layer 1 — 인프라**: CPU/GPU 사용률, 메모리, 디스크 I/O (Grafana + Prometheus)  
**Layer 2 — 애플리케이션**: 요청 지연, 에러율, 큐 깊이 (Prometheus + FastAPI `/metrics`)  
**Layer 3 — 에러 추적**: 스택 트레이스, 재현 컨텍스트 (Sentry)  
**Layer 4 — 비즈니스**: 복원 성공률, 마스크 비율 분포, 사용자 재요청률 (자체 DB + Metabase)

### 데이터 드리프트 감지

모델이 "점점 나빠지는" 방식은 가중치가 아닌 **입력 분포 변화**. 학습 당시 고문서는 스캔 해상도 300dpi였는데 실제 서비스 입력이 4K 고해상도로 들어오면 모델이 본 적 없는 분포 → 품질 저하.

감지 방법:
- **입력 이미지 히스토그램**: 주기적으로 평균·표준편차 측정, 학습 분포와 비교
- **마스크 비율 분포**: 학습 시 5~35% 범위, 실제 서비스에서 50%+가 늘면 경고
- **LPIPS 자동 평가**: 랜덤 샘플 100건을 가상 타깃(입력 × 0.9 + 평균 × 0.1) 대비 정기 측정

## 실전 코드

### 1. Hugging Face Inference Endpoints — 커스텀 Handler

```python
# handler.py — HF Inference Endpoint 진입점
# 업로드 구조: handler.py + requirements.txt + lora_final.safetensors

from typing import Dict, Any
import torch, io, base64
from PIL import Image
from diffusers import StableDiffusionXLInpaintPipeline, LCMScheduler
from peft import LoraConfig
from safetensors.torch import load_file


class EndpointHandler:
    def __init__(self, path: str = ""):
        """
        모델 로드 — path는 repo 루트.
        lora_final.safetensors가 여기에 있어야 함.
        """
        self.pipe = StableDiffusionXLInpaintPipeline.from_pretrained(
            "diffusers/stable-diffusion-xl-1.0-inpainting-0.1",
            torch_dtype=torch.float16, variant="fp16",
        ).to("cuda")

        lora_cfg = LoraConfig(
            r=16, lora_alpha=16,
            target_modules=["to_q", "to_k", "to_v", "to_out.0"],
        )
        self.pipe.unet.add_adapter(lora_cfg, adapter_name="style")
        state = load_file(f"{path}/lora_final.safetensors", device="cpu")
        self.pipe.unet.load_state_dict(state, strict=False)

        self.pipe.load_lora_weights(
            "latent-consistency/lcm-lora-sdxl", adapter_name="lcm"
        )
        self.pipe.set_adapters(["style", "lcm"], adapter_weights=[1.0, 1.0])
        self.pipe.scheduler = LCMScheduler.from_config(self.pipe.scheduler.config)
        self.pipe.set_progress_bar_config(disable=True)

    def __call__(self, data: Dict[str, Any]) -> Dict[str, Any]:
        """
        Request JSON 형식:
        {
            "inputs": {
                "image":  base64_encoded_jpg,
                "mask":   base64_encoded_png,
                "prompt": "..."
            },
            "parameters": {
                "cfg": 2.0, "steps": 6, "seed": 42
            }
        }
        """
        inputs     = data.get("inputs", {})
        parameters = data.get("parameters", {})

        img  = Image.open(io.BytesIO(base64.b64decode(inputs["image"]))).convert("RGB")
        mask = Image.open(io.BytesIO(base64.b64decode(inputs["mask"]))).convert("L")
        img, mask = img.resize((1024, 1024)), mask.resize((1024, 1024))
        prompt = inputs.get("prompt", "Korean traditional document restoration")

        generator = torch.Generator("cuda").manual_seed(parameters.get("seed", 42))
        with torch.autocast("cuda", dtype=torch.float16):
            result = self.pipe(
                prompt=prompt, image=img, mask_image=mask,
                num_inference_steps=parameters.get("steps", 6),
                guidance_scale=parameters.get("cfg", 2.0),
                generator=generator,
            ).images[0]

        buf = io.BytesIO()
        result.save(buf, format="JPEG", quality=90)
        return {"image": base64.b64encode(buf.getvalue()).decode()}
```

### 3. Prometheus 메트릭 + Grafana

```python
# server.py — Prometheus exporter 통합
from prometheus_client import Counter, Histogram, Gauge, generate_latest
from fastapi import Response

_req_counter   = Counter("inpaint_requests_total", "Total requests", ["status"])
_latency_hist  = Histogram("inpaint_latency_seconds", "Request latency",
                           buckets=[0.5, 1, 2, 5, 10, 20, 60])
_queue_gauge   = Gauge("inpaint_queue_size", "Current queue size")
_gpu_mem_gauge = Gauge("inpaint_gpu_memory_gb", "GPU memory in GB")


@app.get("/metrics")
async def metrics_endpoint():
    if worker:
        _queue_gauge.set(worker.queue.qsize())
    _gpu_mem_gauge.set(torch.cuda.memory_allocated() / 1024**3)
    return Response(content=generate_latest(), media_type="text/plain")


@app.middleware("http")
async def metrics_middleware(request, call_next):
    if request.url.path == "/restore":
        import time
        start = time.perf_counter()
        try:
            response = await call_next(request)
            _req_counter.labels(status="success").inc()
        except Exception:
            _req_counter.labels(status="error").inc()
            raise
        finally:
            _latency_hist.observe(time.perf_counter() - start)
        return response
    return await call_next(request)
```

### 5. 드리프트 감지 데몬

```python
# drift_monitor.py — 주기 실행 cron 또는 systemd
import torch, numpy as np
from pathlib import Path
from PIL import Image
from datetime import datetime


class DriftMonitor:
    """
    최근 24시간 서비스 입력의 통계를 계산하고 학습 분포와 비교.
    Slack/이메일 알림.
    """
    def __init__(
        self,
        baseline_stats:    dict,      # 학습 분포 통계 (사전 계산)
        sample_dir:        str,
        alert_threshold:   float = 2.0,    # z-score 임계
    ):
        self.baseline = baseline_stats
        self.sample_dir = Path(sample_dir)
        self.threshold  = alert_threshold

    def compute_stats(self) -> dict:
        files = list(self.sample_dir.glob("*.jpg"))[:500]
        mask_ratios, brightness, resolution = [], [], []

        for f in files:
            try:
                img = Image.open(f).convert("RGB")
                arr = np.array(img)
                brightness.append(arr.mean() / 255)
                resolution.append(max(img.size))
                # mask_ratio는 별도 로그에서 가져옴
            except Exception:
                continue

        return {
            "brightness_mean":  float(np.mean(brightness)),
            "brightness_std":   float(np.std(brightness)),
            "resolution_mean":  float(np.mean(resolution)),
            "sample_count":     len(files),
        }

    def check_drift(self) -> list:
        current = self.compute_stats()
        alerts = []

        for key, base in self.baseline.items():
            if key.endswith("_mean"):
                std_key = key.replace("_mean", "_std")
                baseline_std = self.baseline.get(std_key, 1.0) or 1.0
                z = abs(current[key] - base) / baseline_std
                if z > self.threshold:
                    alerts.append({
                        "metric":  key,
                        "baseline": base,
                        "current": current[key],
                        "z_score": round(z, 2),
                        "severity": "high" if z > 3 else "medium",
                    })
        return alerts

    def run_and_alert(self) -> None:
        alerts = self.check_drift()
        if not alerts:
            print(f"[{datetime.now()}] 드리프트 없음")
            return

        message = "\n".join([
            f"⚠️ {a['severity'].upper()}: {a['metric']} 변화 (z={a['z_score']})"
            f"\n  baseline: {a['baseline']:.3f} → current: {a['current']:.3f}"
            for a in alerts
        ])
        self._send_slack(message)

    def _send_slack(self, text: str) -> None:
        import requests, os
        url = os.environ.get("SLACK_WEBHOOK_URL")
        if url:
            requests.post(url, json={"text": f"🔥 Drift Alert\n{text}"})
        print(text)


if __name__ == "__main__":
    # 학습 분포 통계 (사전 계산 결과)
    baseline = {
        "brightness_mean":  0.48,
        "brightness_std":   0.11,
        "resolution_mean":  1550.0,
        "resolution_std":   320.0,
    }
    monitor = DriftMonitor(
        baseline_stats=baseline,
        sample_dir="/var/log/inpaint/uploads",
    )
    monitor.run_and_alert()
```

## 함정 카드

1. **"readinessProbe initialDelay를 너무 짧게"** → 모델 로드 + torch.compile에 90~150초 걸리는데 initialDelay=30이면 Kubernetes가 "죽은 pod"로 판단해 재시작 루프. 반드시 **120~180초** + `startupProbe` 별도 설정으로 초기 지연 허용.

2. **"HF Inference Endpoint에 가중치를 직접 업로드 안 한다"** → `from_pretrained("hf-repo")` 만 쓰면 공개 모델 다운로드 지연 + 버전 관리 문제. LoRA 가중치는 **같은 repo에 포함**하거나 별도 `datasets` repo에 두고 handler에서 명시적 다운로드.

3. **"메트릭 noise를 전부 드리프트로 판단"** → 주말/공휴일엔 트래픽 패턴이 다름. `z_score > 2`만 알림으로 노이즈 많음 → **3시간 연속 + 동일 방향**일 때만 알림 강화. 또는 `prophet` 같은 시계열 모델로 정상 범위 학습.

4. **"Sentry에 민감 데이터(이미지·경로) 전송"** → 고객 이미지가 Sentry에 스택 트레이스로 유출. `before_send` 훅에서 `request.data`·`image` 필드 필터링. GDPR 관련 문제 가능.

5. **"배포 후 기준 성능을 저장 안 한다"** → 배포 v1.0.0 시점의 SSIM·지연·에러율을 **Golden Metric**으로 기록. v1.1.0 배포 시 **5% 이상 악화**하면 자동 롤백. 이 기준선 없이는 "나빠지고 있는지" 판단 불가.

## 연습문제

**문제 1.** 월 트래픽 3,000 req/day, 평균 2.5초 추론. HF Endpoint vs Modal Serverless vs 자체 k8s 중 무엇을 쓸 것인가?

> **해설:** 월 총 추론 시간 ≈ 3,000 × 30 × 2.5 = **225,000초 ≈ 62시간**. 일 24시간 사용률 8%. ① HF A10G: $0.60/h × 720 = $432 (사용률 낮아 과잉) ② Modal: 225,000 × $0.0006 = **$135** (최적) ③ 자체 k8s A10G: 노드 $450 + 운영 공수. **Modal 추천**. 트래픽 5배 증가 시 k8s 재검토.

**문제 2.** 드리프트 감지기가 `resolution_mean`이 1550 → 3200으로 급증했다고 알림. 즉각 조치할 일 3가지는?

> **해설:** ① **확인**: 실제 업로드 샘플 10장을 열어 고해상도 스캔이 늘었는지 확인 ② **자동 리사이즈 점검**: `img.resize((1024, 1024))` 로직이 작동하는지 확인 — 리사이즈가 제대로 되면 품질 영향 작음 ③ **데이터 수집 반영**: 새 분포의 이미지 1,000장을 수집해 다음 학습에 추가. 추가로 **일시적 대응**: 입력 분포 다양성 감시 강화, 고해상도 입력에 대한 별도 A/B 메트릭 측정 기준 추가.

**문제 3.** Kubernetes HPA가 큐 5 기준으로 스케일 업하는데 새 pod가 뜨기까지 2분 + 컨테이너 준비 2분 = 4분. 이 4분 동안 사용자 경험은? 해결책은?

> **해설:** **스케일 업 지연 중 큐가 계속 쌓임** → 사용자는 10~30초 타임아웃 경험. 해결: ① **Overprovisioning** — 평소 replicas=2로 유지, 대기 pod 보유 (비용 +60% 하지만 UX 보장) ② **Proactive Scaling** — 큐 5가 아니라 큐 3에서 선제 스케일 업 (4분 미래 대비) ③ **KEDA** — 트래픽 예측 기반 자동 스케일러 도입 ④ **Serverless로 이관** — Modal 같은 플랫폼은 콜드 스타트 10~30초로 빠름. 일반적으로 **Proactive + 평시 replicas=2** 조합이 가장 실무적.
