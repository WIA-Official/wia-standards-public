Game Metadata Schema

Phase 1 defines the foundational data structures for educational games. The Game Metadata schema captures essential information about the game itself, including educational context, technical requirements, and accessibility features.

Core Game Metadata Structure

{
  "id": "game_abc123",
  "standard": "WIA-EDU-014",
  "version": "1.0",
  "title": "Math Quest Adventures",
  "type": "educational-game",
  "category": "puzzle",
  "subject": "mathematics",
  "topics": ["algebra", "geometry", "problem-solving"],
  "gradeLevel": {"min": 6, "max": 8},
  "difficulty": "intermediate",
  "targetAge": {"min": 11, "max": 14},
  "language": ["en", "es", "fr", "ko"],
  "platform": ["web", "ios", "android"],
  "accessibility": {
    "wcagLevel": "AA",
    "features": ["screen-reader", "high-contrast", "subtitles"]
  },
  "learningObjectives": [
    {"id": "obj1", "description": "Solve linear equations"},
    {"id": "obj2", "description": "Apply geometric reasoning"}
  ]
}

Progress data captures the learner's journey through the game, including levels completed, skills mastered, time invested, and areas requiring additional support. This data drives adaptive learning and teacher insights.

FieldTypePurpose
currentLevelintegerPlayer's current progression point
totalPlayTimedurationCumulative time in game
lastPlayeddatetimeMost recent session timestamp
completionPercentagefloatOverall game completion (0-100)
skillsMasteredarrayList of demonstrated competencies

Achievement and Badge System

Achievements recognize player accomplishments, milestone completions, and demonstrated competencies. The standardized achievement schema enables portable credentials that follow learners across platforms.

Achievement Types

  • Completion Badges: Awarded for finishing levels, modules, or entire games
  • Mastery Badges: Recognize demonstrated competency in specific skills
  • Performance Badges: Honor exceptional performance (perfect scores, speed records)
  • Persistence Badges: Reward dedication and consistent engagement
  • Collaboration Badges: Acknowledge teamwork and peer support
  • Innovation Badges: Celebrate creative problem-solving approaches

Achievement Schema Example

{
  "achievementId": "badge_algebra_master",
  "type": "mastery",
  "name": "Algebra Master",
  "description": "Demonstrated mastery of algebraic concepts",
  "criteria": {
    "skillsMastered": ["linear-equations", "quadratic-equations"],
    "minAccuracy": 85,
    "minProblems": 100
  },
  "iconUrl": "https://cdn.wia.com/badges/algebra-master.png",
  "earnedBy": "player_xyz789",
  "earnedDate": "2025-12-25T14:30:00Z",
  "verifiable": true,
  "credentialUrl": "https://verify.wia.com/vc/abc123"
}

Assessment Data Structure

Assessment data connects in-game performance to learning standards and curriculum objectives. This enables educators to understand what students are learning through gameplay and identify intervention opportunities.

Assessment TypePurposeFrequency
FormativeTrack ongoing progress and adjust difficultyContinuous
SummativeEvaluate overall learning at module completionEnd of unit
DiagnosticIdentify knowledge gaps and misconceptionsTriggered by performance
AdaptiveDetermine optimal next challenge levelAfter each challenge

Data Validation and Quality

All data structures include JSON Schema validation to ensure data integrity, consistency, and interoperability. Validation catches errors early and prevents incompatible data from entering the ecosystem.

Validation Requirements

• All required fields must be present
• Data types must match schema definitions
• Enumerated values must be from approved lists
• Dates must use ISO 8601 format
• Durations must use ISO 8601 duration format
• URLs must be valid and accessible
• IDs must follow WIA naming conventions

Privacy and Data Protection

Phase 1 specifications include privacy-by-design principles. Personal data is minimized, sensitive information is encrypted, and consent requirements are built into the data models.

  • Personally identifiable information (PII) is separated from learning data
  • Data retention policies are specified in metadata
  • Consent records are linked to player profiles
  • Encryption requirements are defined for sensitive fields
  • Data portability is enabled through standardized export formats
