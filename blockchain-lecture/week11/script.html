대체가능(Fungible) vs 대체불가능(Non-Fungible)

핵심 개념: 대체불가능성이란?

1만원짜리 지폐는 다른 1만원짜리와 교환해도 가치가 같습니다(대체가능). 하지만 여러분의 졸업증서는 다른 사람의 졸업증서와 교환할 수 없습니다(대체불가능). NFT는 이 대체불가능성을 블록체인으로 증명합니다.

  • Fungible Token (FT): ERC-20 - 모든 토큰이 동일한 가치
  • Non-Fungible Token (NFT): ERC-721 - 각 토큰이 고유
  • Semi-Fungible Token (SFT): ERC-1155 - 둘 다 지원
  • NFT = 디지털 자산의 소유권 증명서
15:00 - 28:00

ERC-721 표준 분석

ERC-721은 NFT의 기반이 되는 기술 표준입니다. 이 표준이 정의하는 함수들을 살펴보면 NFT의 작동 원리를 이해할 수 있습니다.
// ERC-721 핵심 인터페이스 interface IERC721 { function balanceOf(address owner) → uint256; function ownerOf(uint256 tokenId) → address; function transferFrom(address from, address to, uint256 tokenId); function approve(address to, uint256 tokenId); function safeTransferFrom(address from, address to, uint256 tokenId); }
  • ownerOf: 특정 토큰의 소유자 조회 - 소유권의 핵심
  • transferFrom: 소유권 이전 - 거래의 기반
  • approve: 대리 전송 권한 부여 - 마켓플레이스 연동
  • 각 tokenId는 전 세계에서 유일 → 희소성 보장

NFT 메타데이터 구조

NFT = 온체인 토큰 + 오프체인 메타데이터

NFT 자체는 블록체인에 저장되지만, 이미지나 영상 등의 실제 콘텐츠는 대부분 오프체인에 저장됩니다. 메타데이터가 콘텐츠와 토큰을 연결하는 다리 역할을 합니다.

{ "name": "대전대 블록체인 수료증 #001", "description": "대전대학교 대학원 블록체인 과정 수료를 증명합니다", "image": "ipfs://QmX...abc/certificate.png", "attributes": [ { "trait_type": "학기", "value": "2026년 1학기" }, { "trait_type": "과목", "value": "블록체인 금융" }, { "trait_type": "성적", "value": "A+" } ] }
  • 저장소 옵션: IPFS, Arweave, 중앙 서버
  • IPFS: 콘텐츠 주소 지정(CID) - 내용이 바뀌면 주소도 바뀜
  • Arweave: 영구 저장 (permaweb)
  • 중앙 서버: 편리하지만 SPOF(단일장애점) 위험
NFT는 이미지 자체가 아닙니다. 블록체인에는 메타데이터의 URI만 저장되어 있습니다. 만약 IPFS 노드가 모두 사라지면? 이 문제가 NFT의 근본적 한계 중 하나입니다.
