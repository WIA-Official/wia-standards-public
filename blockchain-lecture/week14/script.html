0:05-0:15 의료 데이터의 현재 문제점

  • 환자 주권 부재: 환자가 자신의 데이터를 통제할 수 없음
  • 연구 데이터 무결성: 임상시험 데이터 조작 가능성
교수 멘트

0:28-0:38 동의 관리와 데이터 공유
// 간소화된 환자 동의 스마트 계약 contract PatientConsent { struct Consent { address patient; address provider; // 의료기관 string dataType; // "진료기록", "영상", "처방" 등 uint256 expiry; // 동의 만료 시점 bool isActive; } mapping(bytes32 => Consent) public consents; function grantAccess( address _provider, string memory _dataType, uint256 _duration ) public { bytes32 id = keccak256(abi.encodePacked(msg.sender, _provider, _dataType)); consents[id] = Consent(msg.sender, _provider, _dataType, block.timestamp + _duration, true); } function revokeAccess(address _provider, string memory _dataType) public { bytes32 id = keccak256(abi.encodePacked(msg.sender, _provider, _dataType)); require(consents[id].patient == msg.sender, "Not authorized"); consents[id].isActive = false; } }
교수 멘트
"이 모델의 핵심은 환자가 자신의 데이터에 대한 접근 권한을 직접 부여하고 철회할 수 있다는 점입니다. 기존에는 병원이 데이터의 관리 주체였다면, 블록체인 모델에서는 환자가 주체가 됩니다."
