0:05-0:18 Reentrancy (재진입) 공격
  • Reentrancy 공격의 원리: 외부 호출 중 상태 변경 전에 재진입
  • The DAO 사건에서의 실제 공격 메커니즘
  • 취약한 코드 vs 안전한 코드 비교
// 취약한 코드 예시 function withdraw(uint amount) public { require(balances[msg.sender] >= amount); // 위험: 상태 변경 전에 외부 호출 (bool success, ) = msg.sender.call{value: amount}(""); require(success); balances[msg.sender] -= amount; // 이 줄이 재진입 시 실행되지 않음 } // 안전한 코드 (Checks-Effects-Interactions 패턴) function withdrawSafe(uint amount) public { require(balances[msg.sender] >= amount); balances[msg.sender] -= amount; // 먼저 상태 변경 (bool success, ) = msg.sender.call{value: amount}(""); require(success); }
교수 멘트
"핵심은 Checks-Effects-Interactions 패턴입니다. 먼저 조건을 확인하고(Checks), 상태를 변경한 후(Effects), 마지막으로 외부 호출(Interactions)을 합니다. 이 순서를 지키는 것만으로 재진입 공격의 대부분을 방어할 수 있습니다."
0:18-0:30 Integer Overflow / Underflow
  • 정수 오버플로우/언더플로우의 개념
  • Solidity 0.8.0 이전의 취약점
  • SafeMath 라이브러리와 Solidity 0.8+ 기본 보호
  • 실제 공격 사례: BEC 토큰 사건 (2018)
// Solidity 0.7.x - 오버플로우 취약 uint8 balance = 255; balance += 1; // 결과: 0 (오버플로우 발생!) // Solidity 0.8.x - 자동 보호 uint8 balance = 255; balance += 1; // revert 발생 (안전) // unchecked 블록 사용 시 주의 unchecked { uint8 balance = 255; balance += 1; // 다시 0이 됨 - 의도적인 경우에만 사용 }
학생들에게 Solidity 버전 차이에 따른 보안 영향을 강조하세요. 레거시 코드를 다룰 때 특히 중요합니다.
0:30-0:43 Front-running 공격
  • MEV (Miner/Maximal Extractable Value) 개념
  • 트랜잭션 순서 조작의 원리
  • 샌드위치 공격 (DEX에서의 Front-running)
  • 방어 기법: Commit-Reveal 스킴, Flashbots
교수 멘트
"Front-running은 전통 금융에서는 불법이지만, 블록체인에서는 구조적으로 발생할 수 있는 문제입니다. 멤풀(mempool)에 있는 미확인 트랜잭션을 보고, 높은 가스비를 지불하여 먼저 처리되게 하는 것이죠. DEX에서 대량 구매 주문을 미리 감지하고 앞서서 매수한 뒤, 가격 상승 후 매도하는 샌드위치 공격이 대표적입니다."

2:10-2:15 챌린지 1: 재진입 공격 (초급)
// Challenge 1: EtherVault contract EtherVault { mapping(address => uint) public balances; function deposit() public payable { balances[msg.sender] += msg.value; } function withdraw() public { uint bal = balances[msg.sender]; require(bal > 0); (bool sent, ) = msg.sender.call{value: bal}(""); require(sent, "Failed to send Ether"); balances[msg.sender] = 0; } } // 힌트: withdraw 함수의 상태 변경 순서를 살펴보세요.
학생들이 막힐 경우, Checks-Effects-Interactions 패턴을 다시 상기시켜 주세요.
