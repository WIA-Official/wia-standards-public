1 해시값 만들기

SHA-256 해시 함수의 기본 동작을 체험합니다.

  • 시뮬레이터에서 해시 함수 탭을 선택합니다
  • 입력란에 자신의 이름을 입력하고 SHA-256 해시값을 생성합니다 결과가 64자 16진수인지 확인하세요
  • 같은 입력을 다시 넣어 동일한 해시가 나오는지 확인합니다 (결정성)
  • 팀원의 이름으로도 해시를 생성하여 비교합니다
  • 결과를 아래 표에 기록합니다
입력SHA-256 해시값 (앞 16자)
(나의 이름)________________
(팀원 이름)________________
"대전대학교"________________

2 눈사태 효과 관찰

입력을 미세하게 변경하면 해시값이 얼마나 크게 달라지는지 확인합니다.

  • "blockchain"을 입력하여 해시값을 기록합니다
  • "Blockchain" (B를 대문자로)을 입력하여 해시값을 비교합니다 대소문자 하나 차이로 해시가 완전히 달라지는 것을 확인
  • "blockchain1"과 "blockchain2"의 해시값을 비교합니다
  • 변경된 비트 수를 세어 변화율(약 50%)을 확인합니다
예시: "blockchain" → a1b2c3d4e5f6... "Blockchain" → 9f8e7d6c5b4a... → 완전히 다른 값! (이것이 눈사태 효과)

3 키 쌍 생성

공개키와 개인키 쌍을 생성하고 비대칭 암호화의 원리를 체험합니다.

  • 시뮬레이터에서 "키 생성" 탭으로 이동합니다
  • ECC (secp256k1) 알고리즘으로 키 쌍을 생성합니다 개인키(32바이트)와 공개키(65바이트)의 크기 차이를 확인
  • 생성된 공개키에서 블록체인 주소가 도출되는 과정을 관찰합니다
  • 팀원과 공개키를 교환합니다 (개인키는 절대 공유하지 않습니다!)

4 서명 및 검증

디지털 서명의 생성과 검증 과정을 실습합니다.

  • "나는 [이름]이고, 이 메시지를 승인합니다"라는 메시지를 작성합니다
  • 자신의 개인키로 메시지에 서명합니다
  • 서명값과 공개키를 팀원에게 전달합니다
  • 팀원의 공개키로 팀원의 서명을 검증합니다 "유효(Valid)" 결과가 나와야 합니다
  • 메시지를 1글자 변경한 후 검증하여 "무효(Invalid)"가 나오는 것을 확인합니다
