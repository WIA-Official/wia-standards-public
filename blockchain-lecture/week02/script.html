Week 02 — 암호학 기초 (WIA-SEC-002)

대전대학교 대학원 IT 소프트웨어공학 · 교수: 연삼흠 · 총 90분 (전반 40분 + 휴식 15분 + 후반 35분)

세션 1 해시 함수, SHA-256, 눈사태 효과 45분

00:00해시 함수의 개념

지난 주에 블록체인의 구조를 배웠는데, 블록 간의 연결을 가능하게 하는 핵심 기술이 바로 해시 함수입니다. 오늘은 이 해시 함수를 깊이 있게 살펴보겠습니다.
  • 해시 함수란: 임의의 길이의 입력을 고정된 길이의 출력으로 변환하는 수학적 함수
  • 5가지 핵심 속성:
속성설명의미
결정성같은 입력 → 항상 같은 출력검증 가능성
단방향성출력에서 입력 역추적 불가데이터 보호
충돌 저항성같은 출력을 가진 두 입력을 찾기 어려움무결성 보장
눈사태 효과입력의 미세한 변화 → 출력의 대폭 변화위변조 감지
빠른 계산해시값 계산이 효율적실용성

00:10SHA-256 상세

비트코인이 사용하는 해시 함수가 SHA-256입니다. SHA는 Secure Hash Algorithm의 약자이고, 256은 출력 비트 수입니다.
  • NSA가 설계, NIST가 2001년 표준으로 발표
  • 출력: 256비트 = 32바이트 = 64자 16진수 문자열
  • 가능한 출력 수: 2^256 (우주의 원자 수보다 많음)
입력: "대전대학교" SHA-256: 7a8b3f2e1d... (64자) 입력: "대전대학교!" (느낌표 하나 추가) SHA-256: c4e9f1a8b2... (완전히 다른 값)
학생들에게 직접 온라인 SHA-256 계산기를 사용하게 하면서 비교해 보도록 유도하세요.
SHA 패밀리 비교
알고리즘출력 크기상태용도
MD5128비트취약 (사용 금지)레거시 시스템
SHA-1160비트취약Git (전환 중)
SHA-256256비트안전비트코인, SSL

00:25눈사태 효과 (Avalanche Effect)

해시 함수에서 가장 재미있는 특성 중 하나가 눈사태 효과입니다. 입력을 딱 1비트만 바꿔도 출력이 완전히 달라집니다.
입력 A: "Block1" → 해시: 3a7f2b... (64자) 입력 B: "Block2" → 해시: e1c4d9... (64자) 입력 차이: 단 1글자 출력 차이: 약 50%의 비트가 변경됨 이것이 바로 "눈사태 효과" 산 위의 작은 눈 조각 → 거대한 눈사태
  • 이상적인 해시 함수는 1비트 변경 시 출력 비트의 약 50%가 변경
  • 이 특성 덕분에 블록 데이터 위변조를 즉시 감지 가능
  • 패턴 분석을 통한 역추적도 방지
이제 왜 지난 주 실습에서 블록 데이터를 조금만 바꿔도 해시가 완전히 달라졌는지 이해가 되시죠?

세션 2 공개키 암호화, 디지털 서명, 양자내성 45분

01:00공개키/개인키 암호화

해시 함수가 "자물쇠"라면, 공개키 암호화는 "열쇠 시스템"입니다. 여러분은 두 개의 열쇠를 가집니다 — 하나는 공개하고, 하나는 비밀로 합니다.
┌─────────────────────────────────────────────┐ │ 키 쌍 (Key Pair) │ │ │ │ 🔓 공개키 (Public Key) │ │ ├─ 누구에게나 공개 │ │ ├─ 암호화에 사용 │ │ └─ 주소 생성에 사용 │ │ │ │ 🔐 개인키 (Private Key) │ │ ├─ 반드시 비밀로 보관 │ │ ├─ 복호화에 사용 │ │ └─ 서명 생성에 사용 │ │ │ │ 개인키 → 공개키 (가능) │ │ 공개키 → 개인키 (불가능!) │ └─────────────────────────────────────────────┘
  • 비대칭 암호화: 암호화 키 ≠ 복호화 키
  • 타원곡선 암호(ECC): 비트코인과 이더리움이 사용하는 secp256k1 곡선
  • 지갑 주소: 공개키 → SHA-256 → RIPEMD-160 → Base58Check = 비트코인 주소
학생들에게 "개인키를 잃어버리면 코인도 영원히 잃어버린다"는 점을 강조하세요. 실제 사례(하드디스크를 버린 사람)를 소개하면 효과적입니다.

01:20디지털 서명

디지털 서명은 "이 트랜잭션을 내가 승인했다"는 것을 증명하는 방법입니다. 오프라인의 서명(싸인)과 같은 역할이지만, 위조가 수학적으로 불가능합니다.
서명 생성 (송신자): 메시지 → Hash → 해시값 + 개인키 → 서명 서명 검증 (수신자): 메시지 → Hash → 해시값 ↘ 서명 + 공개키 → 복원된 해시값 → 비교 → 일치하면 유효!
  1. 송신자가 트랜잭션 데이터를 해시
  2. 해시값을 개인키로 서명 (ECDSA 알고리즘)
  3. 서명과 트랜잭션을 네트워크에 전파
  4. 수신자/검증자가 공개키로 서명을 검증

01:35양자내성 암호 (Post-Quantum Cryptography)

양자 컴퓨터가 발전하면 현재 사용하는 RSA나 ECC 같은 암호 알고리즘이 깨질 수 있습니다. 이에 대비한 것이 양자내성 암호입니다. WIA-SEC-002에서 이 부분을 중요하게 다루고 있습니다.
  • 양자 위협: Shor 알고리즘으로 RSA, ECC 해독 가능 (이론적으로)
  • 해시 함수는 안전: Grover 알고리즘으로 해시 보안 강도가 절반으로 감소하지만, SHA-256은 여전히 128비트 보안 유지
  • NIST PQC 표준화: CRYSTALS-Kyber(키 교환), CRYSTALS-Dilithium(서명) 등이 선정
  • WIA-SEC-002의 권고: 하이브리드 방식(기존 + PQC)으로 점진적 전환
양자 컴퓨터가 "내일 당장" 위협이 되는 것은 아니지만, 암호화된 데이터가 수십 년간 보호되어야 하는 경우(정부, 금융) "지금부터" 대비해야 한다는 점을 강조하세요.

02:00WIA PQ-Crypto 시뮬레이터 실습

이론을 충분히 배웠으니 이제 직접 체험해 봅시다. WIA에서 제공하는 PQ-Crypto 시뮬레이터를 사용하겠습니다. 실습 가이드를 화면에 띄워 드리겠습니다.
  • 시뮬레이터 URL: https://wiastandards.com/pq-crypto/simulator
  • 학생들에게 lab-guide.html을 프로젝터에 표시
  • 실습 흐름:
  1. 해시값 만들기 (10분): 다양한 문자열의 SHA-256 해시 생성 및 비교
  2. 눈사태 효과 관찰 (5분): 1글자 변경 후 해시 변화 관찰
  3. 키 쌍 생성 (10분): ECC 키 쌍 생성 및 주소 도출 과정
  4. 서명/검증 (10분): 메시지에 서명하고 상대방이 검증
  5. PQC 비교 (10분): 기존 ECC vs 양자내성 알고리즘 키 크기/속도 비교

02:35마무리 및 다음 주 예고

오늘 배운 암호학은 블록체인의 보안 기반입니다. 해시 함수로 데이터 무결성을, 공개키 암호로 신원 확인과 트랜잭션 승인을 보장합니다. 양자 시대에 대비한 PQC도 기억해 주세요. 다음 주에는 합의 알고리즘 — 여러 노드가 어떻게 하나의 진실에 동의하는지를 배우겠습니다.
  • 핵심 정리: 해시(무결성) + 공개키(신원) + 서명(승인) = 블록체인 보안의 3요소
  • 다음 주: 합의 알고리즘 (PoW, PoS, PBFT 등)
  • 과제: 실습 가이드 마지막 페이지 참조
