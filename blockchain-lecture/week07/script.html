1.1 디지털 지갑이란? (10분)

▼
지난 두 주간 스마트 계약을 배웠습니다. 오늘은 블록체인과 사용자를 연결하는 핵심 인터페이스, 디지털 지갑에 대해 알아보겠습니다.

디지털 지갑은 실제로 암호화폐를 "보관"하지 않습니다. 블록체인에 기록된 자산에 접근하기 위한 개인키(Private Key)를 관리하는 도구입니다.

  • 개인키(Private Key): 256비트 난수, 자산에 대한 서명 권한
  • 공개키(Public Key): 개인키에서 파생, 타원곡선 암호화(ECDSA)
  • 주소(Address): 공개키의 해시, 수신 주소로 사용
지갑은 "열쇠 보관함"이라고 생각하면 됩니다. 돈이 들어있는 것이 아니라, 돈에 접근할 수 있는 열쇠를 보관하는 것입니다. 열쇠를 잃어버리면 돈에 영영 접근할 수 없습니다.

1.2 지갑 종류: 핫 월렛 vs 콜드 월렛 (15분)

▼
지갑은 인터넷 연결 여부에 따라 크게 두 가지로 나뉩니다.
구분핫 월렛 (Hot Wallet)콜드 월렛 (Cold Wallet)
인터넷항상 연결오프라인
편의성높음 (즉시 사용)낮음 (연결 필요)
보안상대적 낮음매우 높음
예시MetaMask, Trust WalletLedger, Trezor, 페이퍼 월렛
용도일상 거래, DApp 상호작용장기 보관, 대규모 자산

추가 분류:

  • 소프트웨어 지갑: 모바일 앱, 데스크톱, 브라우저 확장 (MetaMask)
  • 하드웨어 지갑: 전용 기기 (Ledger Nano, Trezor)
  • 페이퍼 월렛: 개인키를 종이에 인쇄
  • 커스터디 지갑: 거래소가 키를 관리 (Binance, Coinbase)
  • 비커스터디 지갑: 사용자가 키를 직접 관리 (MetaMask)
"Not your keys, not your coins"라는 유명한 격언이 있습니다. 거래소에 맡기면 편하지만, 거래소가 해킹되거나 파산하면 자산을 잃을 수 있습니다. 2022년 FTX 파산 사건을 기억하세요.

1.3 HD 지갑과 BIP-39 니모닉 (20분)

▼
HD 지갑은 현대 암호화폐 지갑의 핵심 기술입니다. 하나의 시드(seed)에서 무한한 키 쌍을 파생합니다.

HD(Hierarchical Deterministic) 지갑 - BIP-32:

  • 하나의 마스터 시드에서 트리 구조로 키를 파생
  • 각 계정, 체인, 주소가 계층적으로 관리됨
  • 마스터 시드만 백업하면 모든 키를 복원 가능

BIP-39 니모닉 (Mnemonic):

  • 256비트 엔트로피를 12~24개의 영어 단어로 변환
  • 2048개 단어 목록에서 선택 (BIP-39 wordlist)
  • 체크섬 포함으로 오타 검증 가능
  • 사람이 읽고 기록할 수 있는 형태
니모닉 생성 과정: 1. 128~256비트 엔트로피(난수) 생성 2. SHA-256 해시의 앞부분을 체크섬으로 추가 3. 11비트씩 분할 (2^11 = 2048) 4. 각 11비트 값을 단어 목록의 인덱스로 변환 예시 (12단어): "abandon ability able about above absent absorb abstract absurd abuse access accident" 니모닉 -> PBKDF2(2048회) -> 512비트 시드 시드 -> HMAC-SHA512 -> 마스터 키

BIP-44 파생 경로:

m / purpose' / coin_type' / account' / change / address_index 예: m/44'/60'/0'/0/0 (이더리움 첫 번째 주소) m/44'/60'/0'/0/1 (이더리움 두 번째 주소) m/44'/0'/0'/0/0 (비트코인 첫 번째 주소)
12개 단어만 안전하게 보관하면 모든 계정과 자산을 복원할 수 있습니다. 반대로 말하면, 이 12개 단어를 아는 사람은 여러분의 모든 자산에 접근할 수 있습니다. 절대 디지털 형태로 저장하지 마세요. 종이에 적어 안전한 곳에 보관하세요.

보안 경고

니모닉을 스크린샷, 클라우드, 이메일, 메신저로 저장하면 안 됩니다. 물리적 매체(종이, 금속판)에 기록하고 안전한 장소에 보관하세요.

2.1 키 관리 모범 사례 (15분)

▼
개인키 관리는 블록체인 보안의 핵심입니다. 키를 분실하면 자산을 영영 잃고, 키가 유출되면 자산을 탈취당합니다.

개인키 보관 방법 비교:

방법보안편의성복구 가능
니모닉 종이 보관높음낮음분실 시 불가
하드웨어 지갑매우 높음보통니모닉으로 복구
암호화된 키스토어보통높음비밀번호 필요
거래소 보관낮음매우 높음거래소 의존
MPC(다자간 연산)높음높음임계값에 따름

키 관리 모범 사례:

  1. 니모닉을 2부 이상 만들어 서로 다른 장소에 보관
  2. 큰 금액은 하드웨어 지갑 사용
  3. 일상 거래용과 보관용 지갑을 분리
  4. 정기적으로 백업 상태 확인
  5. 피싱 사이트 주의 (URL 항상 확인)

2.2 지갑 복구 메커니즘 (15분)

▼
기존 금융에서는 비밀번호를 분실하면 은행에 연락하여 재설정할 수 있습니다. 하지만 블록체인에는 "고객센터"가 없습니다.

복구 시나리오:

  • 니모닉 보유: 어떤 지갑 앱에서든 "지갑 복구"로 복원 가능
  • 니모닉 분실, 지갑 앱 접근 가능: 즉시 새 지갑 생성 후 자산 이전
  • 니모닉 분실 + 지갑 접근 불가: 자산 영구 손실

소셜 리커버리 (Social Recovery):

  • Vitalik Buterin이 제안한 차세대 복구 방식
  • 신뢰할 수 있는 "가디언" 그룹 지정
  • 과반수 가디언의 승인으로 키 교체 가능
  • Account Abstraction (ERC-4337)과 결합
소셜 리커버리는 "블록체인의 사용성 문제"를 해결하려는 시도입니다. 일반 사용자가 12개 단어를 안전하게 보관하기란 쉽지 않으니까요. 이 분야가 현재 활발히 연구 중입니다.

2.3 WIA-FIN-015 디지털 지갑 표준 (15분)

▼
WIA-FIN-015는 디지털 지갑의 설계, 보안, 사용자 경험에 대한 국제 가이드라인입니다.

WIA-FIN-015의 주요 원칙:

  • 키 관리 보안: 개인키 암호화 저장, 안전한 엔트로피 소스 사용
  • 사용자 인증: 생체인증, PIN, 2FA 등 다중 인증
  • 트랜잭션 확인: 사용자에게 명확한 트랜잭션 정보 표시
  • 백업 및 복구: 안전한 백업 절차, 복구 메커니즘 제공
  • 네트워크 보안: 안전한 통신 채널, 피싱 방지
  • 규제 준수: KYC/AML 연동 가능한 구조

WIA-FIN-015 vs WIA-FIN-007

WIA-FIN-007은 스마트 계약(서버 측)의 표준이고, WIA-FIN-015는 디지털 지갑(클라이언트 측)의 표준입니다. 둘은 상호 보완적으로, 안전한 블록체인 금융 서비스를 위해 함께 적용됩니다.

3.1 MetaMask 설치 및 지갑 생성 (15분)

▼
이제 직접 MetaMask를 설치하고 지갑을 만들어 봅시다. MetaMask는 가장 널리 사용되는 이더리움 지갑입니다.
  1. Chrome 웹 스토어에서 MetaMask 검색 및 설치
  2. "시작하기" 클릭 후 "지갑 생성" 선택
  3. 비밀번호 설정 (8자 이상, 특수문자 포함 권장)
  4. 비밀 복구 구문(Secret Recovery Phrase) 12단어 확인
  5. 종이에 12단어 기록 (화면 캡처 금지!)
  6. 확인 테스트 통과
학생들에게 반드시 종이에 적게 하세요. 이 12단어가 BIP-39 니모닉입니다. 실습용이라도 보안 습관을 들이는 것이 중요합니다.

학생 안내

이 지갑에 실제 자산을 넣지 마세요. 실습용으로만 사용합니다. 테스트넷 이더만 사용합니다.

3.2 테스트넷 연결 (15분)

▼
메인넷에서는 실제 이더가 필요하므로, 무료 테스트넷을 사용합니다.
  1. MetaMask 설정 -> 고급 -> "테스트 네트워크 표시" 활성화
  2. 네트워크 드롭다운에서 "Sepolia 테스트 네트워크" 선택
  3. Sepolia Faucet에서 테스트 이더 받기
    • sepoliafaucet.com 접속
    • MetaMask 주소 복사하여 입력
    • "Send Me ETH" 클릭
  4. MetaMask에서 잔액 확인 (몇 분 소요될 수 있음)
테스트넷은 메인넷과 동일한 기능을 제공하지만, 이더가 실제 가치가 없습니다. 개발과 학습에 안전하게 사용할 수 있습니다. Sepolia는 현재 이더리움의 공식 테스트넷입니다.

3.3 지갑 기능 탐색 (15분)

▼
MetaMask의 주요 기능을 살펴봅시다.
  1. 주소 확인: 상단의 계정 이름 클릭 -> 주소 복사
  2. 계정 추가: "계정 만들기"로 새 계정 생성 (동일 니모닉에서 파생)
  3. 토큰 추가: "토큰 가져오기"에서 토큰 계약 주소 입력
  4. 전송 테스트: 두 번째 계정으로 소량의 테스트 ETH 전송
  5. 트랜잭션 확인: 가스비 설정, 트랜잭션 상세 정보 확인
  6. 활동 기록: "활동" 탭에서 트랜잭션 이력 확인
  7. 네트워크 추가: 커스텀 네트워크 추가 방법 (RPC URL, Chain ID)
계정을 여러 개 만들어 보세요. 모두 같은 니모닉에서 파생된 주소입니다. BIP-44 경로의 address_index가 0, 1, 2로 증가합니다.
