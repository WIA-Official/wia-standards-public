## Part 1: 이론 (30분)

### 1-1. 도입 및 지난 주 복습 (5분)

**교수 멘트:**

> 안녕하세요, 여러분. 연삼흠입니다. 지난 2주 동안 스마트 계약을 집중적으로 다뤘죠? HelloWorld 계약도 만들고, 투표 DApp도 만들어 봤습니다. 오늘은 주제가 바뀝니다. **디지털 지갑**입니다.
>
> 여러분, 스마트 계약을 배포하고 함수를 호출할 때 Remix에서 계정을 선택했잖아요. 그 "계정"이 바로 지갑입니다. 지갑이 없으면 블록체인과 상호작용할 수 없어요. 트랜잭션을 보낼 수도, 스마트 계약을 호출할 수도 없죠.
>

### 1-2. 디지털 지갑이란? (8분)

**교수 멘트:**

> 흔히 "지갑"이라고 하면 돈을 보관하는 물건을 떠올리죠? 그런데 블록체인 디지털 지갑은 조금 다릅니다. 실제로 암호화폐를 "저장"하는 게 아니에요. 지갑은 **개인키**를 관리하는 소프트웨어입니다. 암호화폐 자체는 블록체인에 기록되어 있고, 지갑은 그 자산에 접근할 수 있는 "열쇠"를 보관하는 거예요.

#### 핵심 개념

| 개념 | 설명 |
|------|------|
| **개인키 (Private Key)** | 256비트 난수. 절대 남에게 공개하면 안 됨 |
| **공개키 (Public Key)** | 개인키에서 타원곡선 암호로 생성. 공개 가능 |
| **주소 (Address)** | 공개키를 해시하여 생성. 20바이트(40자리 16진수) |
| **서명 (Signature)** | 개인키로 트랜잭션에 서명. 소유권 증명 |

**교수 멘트:**

> 이 관계를 이해하는 게 중요합니다. 개인키 → 공개키 → 주소, 이 방향으로만 계산이 가능합니다. 역방향은 불가능해요. 주소에서 공개키를 역추적할 수 없고, 공개키에서 개인키를 알아낼 수 없습니다. 이게 블록체인 보안의 수학적 기반입니다.
>
> 비유하자면, 주소는 "계좌번호"이고, 개인키는 "비밀번호+공인인증서+도장"을 합쳐놓은 겁니다. 개인키가 유출되면 모든 자산을 잃을 수 있어요. 은행은 비밀번호를 바꿀 수 있지만, 블록체인에서는 개인키를 변경할 수 없습니다.

### 1-3. 지갑의 종류 (7분)

**교수 멘트:**

> 디지털 지갑은 크게 두 가지로 나뉩니다. 핫월렛과 콜드월렛이에요.

#### 핫월렛 (Hot Wallet) - 인터넷 연결

| 유형 | 예시 | 장점 | 단점 |
|------|------|------|------|
| 웹 지갑 | MetaMask (브라우저 확장) | 편리, DApp 연동 | 피싱 위험 |
| 모바일 지갑 | Trust Wallet, Klip | 이동성 | 분실/해킹 위험 |
| 데스크톱 지갑 | Electrum | 기능 풍부 | 악성코드 위험 |
| 거래소 지갑 | 업비트, 빗썸 | 초보자 친화 | 자산 통제권 없음 |

#### 콜드월렛 (Cold Wallet) - 인터넷 비연결

| 유형 | 예시 | 장점 | 단점 |
|------|------|------|------|
| 하드웨어 지갑 | Ledger, Trezor | 최고 보안 | 비용, 불편 |
| 종이 지갑 | QR코드 인쇄 | 해킹 불가 | 분실/훼손 위험 |
| 에어갭 지갑 | 오프라인 PC | 높은 보안 | 전문 지식 필요 |

**교수 멘트:**

> "Not your keys, not your coins"라는 유명한 말이 있습니다. 거래소 지갑은 여러분이 개인키를 가지고 있는 게 아니라, 거래소가 관리하는 거예요. 거래소가 해킹되거나 파산하면? 여러분의 자산도 위험해집니다. 실제로 2022년에 FTX 거래소가 파산하면서 수많은 사용자의 자금이 동결되었죠.
>
> 반면에 MetaMask 같은 지갑은 개인키를 여러분의 브라우저에 저장합니다. 여러분이 직접 관리하는 거죠. 자유가 크지만 책임도 큽니다.

### 1-4. HD 지갑과 니모닉 (5분)

**교수 멘트:**

> 자, 여기서 중요한 개념 하나 더. HD 지갑입니다. HD는 "Hierarchical Deterministic"의 약자예요. 계층적 결정론적 지갑. 이름이 어렵죠? 쉽게 말하면, **하나의 시드(씨앗)에서 수많은 주소를 만들어낼 수 있는 지갑**입니다.

#### HD 지갑 구조

```
시드(Seed)
  └─ 마스터 키
       ├─ 계정 1
       │    ├─ 주소 1-1
       │    ├─ 주소 1-2
       │    └─ 주소 1-3
       ├─ 계정 2
       │    ├─ 주소 2-1
       │    └─ 주소 2-2
       └─ 계정 3
            └─ ...
```

#### 니모닉 (Mnemonic Phrase)

**교수 멘트:**

> 시드를 256비트 숫자 그대로 기억하는 건 불가능하죠. 그래서 사람이 읽을 수 있는 단어 12개(또는 24개)로 변환한 것이 **니모닉**입니다.

```
예시: apple banana cherry dog elephant fish garden hotel island jungle kitchen lemon
```

> 이 12개 단어만 있으면 모든 개인키와 주소를 복원할 수 있습니다. 그래서 니모닉은 개인키보다도 중요해요. 절대 디지털로 저장하지 마세요. 스크린샷, 메모장, 클라우드 절대 안 됩니다. 종이에 적어서 안전한 곳에 보관하는 것이 최선입니다.
>
> BIP-39 표준에서 정의한 2048개의 영어 단어 목록에서 선택되며, 마지막 단어에는 체크섬이 포함되어 있어서 단어를 잘못 적으면 감지할 수 있습니다.

### 1-5. WIA-FIN-015 표준 (5분)

**교수 멘트:**

> WIA-FIN-015는 디지털 지갑에 대한 국제 표준입니다. 지갑 개발과 사용에 관한 보안 가이드라인을 제시합니다.

#### WIA-FIN-015 주요 항목

1. **키 생성 및 관리**: 안전한 난수 생성기(CSPRNG) 사용, 키 암호화 저장
2. **니모닉 관리**: BIP-39 준수, 사용자에게 안전한 보관 안내 필수
3. **트랜잭션 서명**: 서명 전 사용자 확인 절차 필수
4. **네트워크 보안**: HTTPS 통신, 피싱 방지, 도메인 검증
5. **백업 및 복구**: 니모닉 기반 복구 절차, 다중 백업 권장
6. **사용자 인터페이스**: 주소 표시 시 앞뒤 일부 문자 강조, 금액 확인 절차

**교수 멘트:**

> 이 표준에서 특히 강조하는 게 "사용자에게 니모닉의 중요성을 반드시 안내해야 한다"는 점입니다. 많은 지갑 앱이 처음 설정할 때 "이 단어들을 안전하게 보관하세요"라고 경고하죠? 그게 WIA-FIN-015를 따르고 있는 겁니다.
>
> 시뮬레이터에서 이 표준의 각 항목을 실제로 확인해 볼 수 있으니까, 실습 시간에 같이 해봅시다.

## Part 2: 실습 (60분)

### 2-1. MetaMask 설치 및 설정 (20분)

**교수 멘트:**

> 자, 오늘 실습의 핵심은 MetaMask입니다. 이더리움 생태계에서 가장 많이 사용되는 브라우저 확장 지갑이에요. 실습용으로 사용하기 딱 좋습니다.

#### Step 1: MetaMask 설치

1. Chrome 웹 스토어에서 "MetaMask" 검색
2. **반드시 공식 확장 프로그램인지 확인** (제작자: metamask.io)
3. "Chrome에 추가" 클릭
4. 브라우저 우측 상단에 여우 아이콘 확인

**교수 멘트:**

> 주의! 가짜 MetaMask 확장 프로그램이 있을 수 있습니다. 반드시 공식 사이트(metamask.io)에서 설치하거나, Chrome 웹 스토어에서 제작자를 확인하세요. 이것도 WIA-FIN-015에서 말하는 피싱 방지의 일환입니다.

#### Step 2: 지갑 생성

1. MetaMask 아이콘 클릭 → "시작하기" 버튼 클릭
2. **"새 지갑 만들기"** 선택
3. 비밀번호 설정 (8자 이상, 영문+숫자+특수문자 조합 권장)
4. **비밀 복구 구문(니모닉) 확인**
   - 12개 단어가 표시됩니다
   - **반드시 종이에 적으세요** (실습용이므로 메모장도 허용)
   - 순서가 중요합니다!
5. 니모닉 확인 단계: 단어를 올바른 순서로 클릭

**교수 멘트:**

> 방금 보신 12개 단어가 니모닉입니다. 이론에서 배웠죠? 이 단어들만 있으면 다른 컴퓨터에서도, 다른 지갑 앱에서도 여러분의 계정을 복원할 수 있습니다.
>
> 오늘은 실습용이니까 메모장에 적어도 되지만, 실제로 본인 자산을 관리하는 지갑의 니모닉은 절대 디지털로 저장하면 안 됩니다. 종이에 적어서 금고에 넣어두세요. 진짜로요.

#### Step 3: 테스트넷 설정

**교수 멘트:**

> 메인넷에서 실습하면 진짜 돈이 듭니다. 우리는 테스트넷을 사용할 겁니다. 테스트넷은 메인넷과 동일하게 동작하지만, 사용하는 ETH가 무료이고 실제 가치가 없어요.

1. MetaMask 좌측 상단 네트워크 선택 드롭다운 클릭
2. "네트워크 표시/숨기기" 또는 "테스트 네트워크 표시" 옵션 활성화
3. **"Sepolia 테스트 네트워크"** 선택
4. 상단에 "Sepolia" 표시 확인

#### Step 4: 테스트 ETH 받기 (Faucet)

1. Google에서 "Sepolia faucet" 검색
2. 추천 Faucet 사이트:
   - https://sepoliafaucet.com
   - https://faucets.chain.link/sepolia
3. MetaMask에서 주소 복사 (계정 이름 아래 `0x...` 클릭)
4. Faucet 사이트에 주소 붙여넣기
5. "Send me ETH" 또는 유사 버튼 클릭
6. 1~2분 후 MetaMask에서 잔액 확인 (보통 0.1~0.5 SepoliaETH)

**교수 멘트:**

> Faucet은 테스트용 ETH를 무료로 나눠주는 사이트입니다. "수도꼭지"라는 뜻인데, 조금씩 뚝뚝 떨어지니까 그렇게 부릅니다. 사이트에 따라 하루 1회 제한이 있을 수 있어요.
>
> ETH가 바로 안 들어올 수도 있습니다. 테스트넷도 블록이 생성되어야 하니까 1~2분 정도 기다려 주세요.

### 2-2. 트랜잭션 실습 (15분)

**교수 멘트:**

> 자, 이제 ETH가 들어왔으면 트랜잭션을 보내봅시다. 옆자리 친구와 주소를 교환해서 서로 ETH를 보내볼 거예요.

#### Step 5: ETH 전송

1. MetaMask에서 "보내기" 버튼 클릭
2. 받는 주소 입력 (옆자리 동료의 주소 또는 자신의 다른 계정)
3. 금액 입력: `0.01` ETH
4. 가스비 확인 (예상 가스비가 표시됨)
5. "확인" 클릭
6. 트랜잭션 해시 확인

#### Step 6: 트랜잭션 확인

1. 트랜잭션을 클릭하여 상세 정보 확인
2. "블록 탐색기에서 보기" 클릭 (Etherscan Sepolia)
3. Etherscan에서 확인할 내용:
   - Transaction Hash (고유 식별자)
   - Status (Success/Pending)
   - From / To 주소
   - Value (전송량)
   - Gas Used / Gas Price

**교수 멘트:**

> Etherscan에서 여러분의 트랜잭션이 보이시죠? 이게 블록체인의 투명성입니다. 누구나 모든 트랜잭션을 확인할 수 있어요. 보낸 사람, 받는 사람, 금액, 가스비까지 전부 공개됩니다.
>
> "From" 주소가 여러분의 MetaMask 주소와 일치하는지 확인하세요. "Gas Used"를 보면 단순 ETH 전송에는 21,000 가스가 소모됩니다. 이건 이더리움에서 정해진 기본 전송 가스량이에요.

### 2-3. 계정 관리 실습 (10분)

#### Step 7: 새 계정 생성 (HD 지갑 체험)

1. MetaMask에서 계정 아이콘 클릭
2. "계정 추가" 또는 "새 계정 만들기" 클릭
3. 계정 이름 설정 (예: "테스트 계정 2")
4. 새 주소가 생성된 것을 확인

**교수 멘트:**

> 새 계정을 만들었는데, 니모닉을 다시 입력하지 않았죠? 이게 HD 지갑의 힘입니다. 하나의 시드(니모닉)에서 무한히 많은 주소를 파생할 수 있어요. Account 1은 파생 경로 m/44'/60'/0'/0/0, Account 2는 m/44'/60'/0'/0/1, 이런 식으로 순서대로 생성됩니다.

#### Step 8: 개인키 내보내기 (주의해서 확인만)

1. 계정 메뉴 → "계정 세부 정보" 클릭
2. "개인키 내보내기" 클릭
3. MetaMask 비밀번호 입력
4. 64자리 16진수 개인키가 표시됨
5. **확인만 하고 절대 공유하지 마세요**

**교수 멘트:**

> 이게 여러분의 개인키입니다. 64자리 16진수, 즉 256비트 숫자예요. 이걸 아는 사람은 여러분의 모든 자산을 마음대로 가져갈 수 있습니다. 비밀번호 없이도, MetaMask 없이도요.
>
> 실습이 끝나면 이 화면을 바로 닫으세요. 혹시 캡처한 분은 즉시 삭제하세요. 실습용 계정이라 자산이 없지만, 이런 습관이 나중에 실제 자산을 보호합니다.

### 2-4. WIA 시뮬레이터 실습 (15분)

**교수 멘트:**

> 마지막으로 WIA 시뮬레이터에서 디지털 지갑 관련 실습을 해봅시다.

## Part 3: 정리 (30분)

### 3-1. 핵심 내용 정리 (10분)

**교수 멘트:**

> 자, 오늘 배운 내용을 정리합시다.

#### 오늘 배운 핵심 내용

1. **디지털 지갑** = 개인키를 관리하는 소프트웨어 (자산 자체를 저장하지 않음)
2. **키 구조**: 개인키 → 공개키 → 주소 (단방향만 가능)
3. **핫월렛**: 인터넷 연결, 편리, 보안 취약 (MetaMask, Trust Wallet)
4. **콜드월렛**: 인터넷 비연결, 불편, 보안 강력 (Ledger, Trezor)
5. **HD 지갑**: 하나의 시드에서 무한한 주소 파생 (BIP-32/44)
6. **니모닉**: 12/24개 단어로 시드를 인간이 읽을 수 있는 형태로 변환 (BIP-39)
7. **WIA-FIN-015**: 디지털 지갑 보안 표준 (키 관리, 니모닉, 트랜잭션 서명)

**교수 멘트:**

> 오늘 여러분이 MetaMask를 설치하고, 테스트넷에서 실제로 ETH를 주고받아 봤습니다. "블록체인 사용자"로서의 첫 경험이에요. 이제 지갑이 뭔지, 개인키가 왜 중요한지 몸으로 느꼈을 겁니다.
>
> 한 가지만 기억하세요. **니모닉은 절대 디지털로 저장하지 마세요.** 이것만 지키면 대부분의 자산 도난을 방지할 수 있습니다.

### 3-2. Q&A 및 토론 (10분)

**교수 멘트:**

> 질문 시간입니다. 오늘 실습하면서 궁금했던 점이 있으면 말씀하세요.

**예상 질문과 답변:**

**Q1: MetaMask 비밀번호를 잊어버리면 어떻게 하나요?**

> 비밀번호는 로컬 장치에서만 사용하는 잠금 해제용입니다. 비밀번호를 잊으면 MetaMask를 재설치하고, 니모닉으로 복구하면 됩니다. 니모닉만 있으면 언제든 복구가 가능해요. 그래서 니모닉이 최종 백업 수단인 겁니다.

**Q2: 하드웨어 지갑은 어떻게 동작하나요?**

> 하드웨어 지갑은 개인키를 USB 장치 내부의 보안 칩에 저장합니다. 트랜잭션에 서명할 때, 개인키가 장치 밖으로 나오지 않고 장치 내부에서 서명을 수행해요. 그래서 컴퓨터에 악성코드가 있어도 개인키를 탈취할 수 없습니다. 장기 보유 자산은 하드웨어 지갑에 보관하는 것을 권장합니다.

**Q3: 테스트넷 ETH를 메인넷에서 사용할 수 있나요?**

> 절대 안 됩니다. 테스트넷과 메인넷은 완전히 별개의 네트워크예요. 테스트넷 ETH는 실제 가치가 없습니다. 반대로 메인넷 ETH를 테스트넷에 보낼 수도 없어요. 네트워크를 잘못 선택하지 않도록 항상 MetaMask 상단에 어떤 네트워크가 선택되어 있는지 확인하세요.

**Q4: 같은 니모닉을 MetaMask와 다른 지갑 앱에서 동시에 사용할 수 있나요?**

> 네, 가능합니다. BIP-39/44 표준을 따르는 지갑이라면 같은 니모닉으로 동일한 주소를 복원할 수 있어요. 이것이 표준의 힘입니다. 특정 회사나 앱에 종속되지 않죠.

### 3-3. 다음 주 예고 및 과제 (10분)

**교수 멘트:**

> 다음 주에는 **토큰 경제학**으로 넘어갑니다. ERC-20 토큰이 뭔지, 어떻게 만드는지, 그리고 NFT(ERC-721)까지 다룰 예정입니다. 오늘 배운 지갑 지식이 토큰을 이해하는 데 기반이 되니까 잘 복습해 두세요.

#### 과제

1. **MetaMask 활용 실습**
   - 계정 3개 이상 생성하고, 각 계정 간에 테스트 ETH 전송
   - 각 트랜잭션을 Etherscan에서 확인하고 스크린샷 캡처
   - 가스비(Gas Fee) 구성 요소 분석: Base Fee, Priority Fee, Gas Limit
2. **보안 실습**
   - MetaMask의 보안 설정 옵션 탐색 (자동 잠금 시간, 피싱 감지 등)
   - WIA-FIN-015 체크리스트 기준으로 자신의 보안 수준 자체 평가
3. **WIA 시뮬레이터**
   - 시뮬레이터에서 지갑 보안 점검 완료
   - 결과 캡처 및 개선 사항 정리

**교수 멘트:**

> 과제의 핵심은 "직접 해보기"입니다. 계정 간 ETH 전송을 여러 번 해보면서 트랜잭션의 흐름을 체감하세요. Etherscan에서 트랜잭션 세부 정보를 읽는 연습도 중요합니다. 블록체인 개발자라면 블록 탐색기를 자유자재로 활용할 수 있어야 해요.
>
> 그리고 보안 실습! MetaMask의 설정 메뉴를 꼼꼼히 둘러보세요. 자동 잠금 시간을 짧게 설정하고, 피싱 감지 기능이 켜져 있는지 확인하세요. 이런 사소한 설정이 보안의 첫 걸음입니다. 그럼 다음 주에 만나요!
