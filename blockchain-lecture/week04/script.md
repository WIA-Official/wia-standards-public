### 1-1. 암호화폐란? 디지털 자산의 개념 (3분)

> **교수 멘트:**
> "암호화폐를 정의해봅시다. 암호화폐는 **암호학적 기법을 사용하여 보안을 확보하는 디지털 자산**입니다."

**암호화폐의 특징:**
- **탈중앙화:** 중앙 발행 기관이 없음 (중앙은행 X)
- **암호학 기반:** 공개키/개인키로 소유권 증명
- **블록체인 기록:** 모든 거래가 블록체인에 투명하게 기록
- **프로그래밍 가능:** 스마트 컨트랙트를 통한 자동화 가능 (이더리움)
- **전 세계 전송:** 국경 없이 24시간 전송 가능

**디지털 자산 vs 디지털 화폐 vs 암호화폐:**

| 구분 | 예시 | 특징 |
|------|------|------|
| **디지털 자산** | 게임 아이템, NFT | 디지털 형태의 가치 있는 자산 |
| **디지털 화폐** | CBDC (디지털 원화) | 중앙기관이 발행하는 디지털 돈 |
| **암호화폐** | BTC, ETH | 탈중앙화 + 암호학 기반 |

> **교수 멘트:**
> "중요한 구분입니다. 최근 한국은행에서 추진하는 CBDC(디지털 원화)는 '디지털 화폐'이지 '암호화폐'는 아닙니다. 중앙은행이 발행하니까요. 비트코인과 이더리움은 누구도 통제하지 않는 진정한 암호화폐입니다."

### 1-2. 비트코인 구조: UTXO 모델 (7분)

> **교수 멘트:**
> "비트코인의 가장 핵심적인 개념 중 하나가 **UTXO(Unspent Transaction Output, 미사용 트랜잭션 출력)** 모델입니다. 처음에는 좀 어려울 수 있는데, 현금에 비유하면 쉽습니다."

**UTXO 모델 설명:**
- **UTXO = 아직 사용하지 않은 거래 출력값**
- 비유: 지갑에 있는 지폐들
  - 5만 원짜리 1장 + 1만 원짜리 3장 = 잔액 8만 원
  - 각 지폐가 하나의 UTXO

**UTXO 거래 예시:**
```
앨리스가 밥에게 3 BTC를 보내고 싶다.
앨리스의 UTXO: [5 BTC]

트랜잭션:
  입력(Input): 5 BTC (UTXO 소비)
  출력(Output) 1: 3 BTC → 밥 (전송)
  출력(Output) 2: 1.9999 BTC → 앨리스 (거스름돈)
  수수료: 0.0001 BTC (채굴자에게)
```

> **교수 멘트:**
> "5만 원짜리 지폐로 3만 원짜리 물건을 사면 2만 원을 거스름돈으로 받죠? UTXO도 마찬가지입니다. 5 BTC UTXO를 써서 3 BTC를 보내면, 나머지는 거스름돈으로 자기 자신에게 돌아옵니다. 거스름돈과 원래 금액의 차이가 수수료가 됩니다."

**비트코인 블록 보상과 반감기:**
- **블록 보상:** 새 블록을 생성한 채굴자에게 주어지는 비트코인
- **초기 보상:** 50 BTC (2009년)
- **반감기(Halving):** 약 4년(21만 블록)마다 보상이 절반으로 감소
  - 2009년: 50 BTC
  - 2012년: 25 BTC
  - 2016년: 12.5 BTC
  - 2020년: 6.25 BTC
  - 2024년: 3.125 BTC
- **총 발행량:** 2,100만 BTC (2140년경 모든 발행 완료 예정)

> **교수 멘트:**
> "반감기 때마다 비트코인 가격이 크게 변동하는 경향이 있습니다. 공급이 줄어드니까요. 2024년에 가장 최근 반감기가 있었죠. 이런 설계가 비트코인을 '디지털 금'이라 부르는 이유 중 하나입니다."

### 1-3. 이더리움 구조: 어카운트 모델 (7분)

> **교수 멘트:**
> "이더리움은 비트코인과 다르게 **어카운트(Account) 모델**을 사용합니다. 이건 우리가 익숙한 은행 계좌 방식과 유사합니다."

**어카운트 모델:**
- 각 계정(주소)에 잔액이 직접 기록됨
- 은행 계좌처럼 잔액이 증감하는 방식
- 두 가지 계정 유형:
  - **EOA (Externally Owned Account):** 일반 사용자 계정 (개인키로 제어)
  - **CA (Contract Account):** 스마트 컨트랙트 계정 (코드로 제어)

**UTXO vs 어카운트 비교:**

| 구분 | UTXO (비트코인) | 어카운트 (이더리움) |
|------|----------------|-------------------|
| **비유** | 현금 지갑 | 은행 계좌 |
| **잔액 표현** | UTXO 합계 | 계정 잔액 |
| **프라이버시** | 높음 (주소 변경) | 낮음 (고정 주소) |
| **스마트 컨트랙트** | 제한적 | 강력한 지원 |
| **병렬 처리** | 용이 | 어려움 |

**가스비(Gas Fee):**
- 이더리움에서 트랜잭션을 실행하려면 **가스비**를 지불
- 가스 = 연산 비용의 단위
- 가스비 = 가스 사용량 x 가스 가격 (Gwei 단위)
- 복잡한 스마트 컨트랙트 실행 → 더 많은 가스 소비

> **교수 멘트:**
> "가스비를 택시 요금에 비유하면 이해하기 쉽습니다. 가까운 곳은 요금이 적고, 먼 곳은 비싸죠? 단순 송금은 가스가 적게 들고, 복잡한 스마트 컨트랙트 실행은 가스가 많이 듭니다."

**EVM (Ethereum Virtual Machine):**
- 이더리움의 스마트 컨트랙트를 실행하는 가상 머신
- 전 세계 모든 이더리움 노드에서 동일하게 실행
- "세계 컴퓨터(World Computer)"라는 별명

### 1-4. 트랜잭션 구조 (5분)

> **교수 멘트:**
> "이제 트랜잭션의 구조를 자세히 살펴봅시다. 트랜잭션은 블록체인에 기록되는 하나의 거래 기록입니다."

**트랜잭션의 주요 필드:**

```
{
  "sender":    "0xAbCd...1234",     // 보내는 사람 주소
  "receiver":  "0xEfGh...5678",     // 받는 사람 주소
  "amount":    "1.5 ETH",           // 전송 금액
  "gas":       "21000",             // 가스 한도 (Gas Limit)
  "gasPrice":  "20 Gwei",           // 가스 가격
  "nonce":     "42",                // 발신자의 트랜잭션 순번
  "signature": "0x3a7f...b2c1",     // 디지털 서명
  "data":      "0x..."              // 추가 데이터 (스마트 컨트랙트 호출 시)
}
```

**각 필드 설명:**
- **sender:** 보내는 사람의 지갑 주소 (공개키에서 파생)
- **receiver:** 받는 사람의 지갑 주소
- **amount:** 전송할 금액
- **gas:** 이 트랜잭션에 사용할 최대 가스량
- **gasPrice:** 가스 1단위당 지불할 가격
- **nonce:** 해당 계정에서 발생한 트랜잭션의 순번 (순서 보장, 재전송 방지)
- **signature:** 발신자의 개인키로 생성한 디지털 서명 (소유권 증명)

> **교수 멘트:**
> "nonce가 왜 중요한지 생각해보세요. 만약 nonce가 없다면, 누군가가 이전 트랜잭션을 복사해서 다시 보낼 수 있습니다. nonce가 있으면 같은 nonce의 트랜잭션은 한 번만 처리됩니다. 이것도 이중지불 방지의 한 방법이에요."

### 실습 2: 테스트넷 지갑 주소 생성 (10분)

> **교수 멘트:**
> "이제 여러분만의 지갑 주소를 만들어봅시다. 지갑 주소가 어떻게 생성되는지, 공개키와 개인키의 관계를 직접 확인합니다."

**진행 순서:**
1. 시뮬레이터에서 "새 지갑 생성" 클릭
2. 생성 과정 관찰:
   - 개인키(Private Key) 생성 → 랜덤 256비트 숫자
   - 공개키(Public Key) 생성 → 개인키에서 파생 (타원곡선 암호)
   - 지갑 주소 생성 → 공개키를 해시하여 생성
3. 생성된 지갑 정보 기록:
   - 내 지갑 주소: ____________________
   - 공개키: ____________________
   - 개인키: ____________________ (절대 타인에게 공유하지 않기!)

> **교수 멘트:**
> "개인키를 절대로 다른 사람에게 알려주면 안 됩니다. 실제 블록체인에서 개인키가 유출되면 그 지갑의 모든 자산을 잃게 됩니다. 시뮬레이터에서도 이 습관을 들이는 것이 중요해요."

**개인키 → 공개키 → 주소의 관계:**
```
개인키 (256비트 랜덤)
    ↓ 타원곡선 곱셈 (ECDSA)
공개키 (512비트)
    ↓ Keccak-256 해시 → 마지막 20바이트
지갑 주소 (0x로 시작하는 40자리 16진수)
```

> **교수 멘트:**
> "개인키에서 공개키는 쉽게 만들 수 있지만, 공개키에서 개인키를 역추적하는 것은 수학적으로 불가능합니다. 이것이 **비대칭 암호학**의 핵심이죠. 2주차에서 배운 내용 기억나시죠?"

### 실습 3: 가상 코인 전송 체험 (15분)

> **교수 멘트:**
> "지갑이 만들어졌으니, 이제 코인을 주고받아 봅시다. 시뮬레이터에서 제공하는 테스트 코인을 사용합니다."

**진행 순서:**
1. 시뮬레이터에서 테스트 코인 받기 (Faucet 기능)
   - "테스트 코인 받기" 버튼 클릭
   - 10 테스트 코인 수령 확인
2. 옆 자리 동료의 지갑 주소 확인
3. 트랜잭션 생성:
   - 받는 사람 주소 입력
   - 전송 금액 입력 (예: 2.5 코인)
   - 가스비 설정
   - 서명 및 전송
4. 전송 결과 확인:
   - 내 잔액 변화 확인
   - 상대방 잔액 변화 확인
   - 트랜잭션 상태(Pending → Confirmed) 관찰

> **교수 멘트:**
> "전송 버튼을 누르면 바로 완료되는 것이 아니라, '대기 중(Pending)' 상태를 거쳐 '확인됨(Confirmed)'으로 바뀌는 것을 관찰하세요. 실제 비트코인은 이 과정이 약 10분, 이더리움은 약 12초 걸립니다."

**트랜잭션 생성 시 확인할 사항:**
- 가스비를 너무 낮게 설정하면? → 트랜잭션이 오래 걸리거나 실패
- 잔액보다 많은 금액을 보내려고 하면? → 트랜잭션 거부
- nonce가 자동으로 증가하는지 확인

### 실습 4: 트랜잭션 해시 확인 및 분석 (15분)

> **교수 멘트:**
> "여러분이 방금 보낸 트랜잭션의 해시를 확인해봅시다. 트랜잭션 해시는 그 거래의 **고유 ID**와 같습니다."

**진행 순서:**
1. 전송 완료된 트랜잭션의 해시(TxHash) 복사
2. 시뮬레이터의 트랜잭션 조회 기능에서 해시 입력
3. 트랜잭션 상세 정보 분석:

| 필드 | 내 트랜잭션 값 | 설명 |
|------|--------------|------|
| TxHash | | 트랜잭션 고유 식별자 |
| Status | | 성공/실패/대기 |
| Block | | 포함된 블록 번호 |
| From | | 발신자 주소 |
| To | | 수신자 주소 |
| Value | | 전송 금액 |
| Gas Used | | 실제 사용된 가스 |
| Gas Price | | 가스 단가 |
| Nonce | | 발신자 트랜잭션 순번 |

> **교수 멘트:**
> "트랜잭션 해시만 있으면 전 세계 누구나 이 거래 내용을 확인할 수 있습니다. 이것이 블록체인의 **투명성**입니다. 모든 거래가 공개되어 있어서 감사(audit)가 가능하죠."

4. 여러 트랜잭션의 가스비를 비교
5. nonce 값의 변화 패턴 관찰

### 실습 5: 블록 탐색기로 실제 트랜잭션 조회 (10분)

> **교수 멘트:**
> "마지막으로, 실제 비트코인과 이더리움 블록체인에서 트랜잭션을 직접 조회해봅시다. 실제 메인넷의 거래 데이터를 볼 수 있습니다."

**블록 탐색기 사이트:**
- **비트코인:** https://blockchain.info (또는 https://blockchair.com)
- **이더리움:** https://etherscan.io

**진행 순서:**
1. Etherscan 접속 → 최근 블록 확인
2. 아무 블록 클릭 → 해당 블록에 포함된 트랜잭션 목록 확인
3. 아무 트랜잭션 클릭 → 상세 정보 분석
4. 유명한 지갑 주소 조회해보기:
   - 비트코인 제네시스 블록 주소 (사토시 나카모토)
   - 이더리움 대형 스마트 컨트랙트 주소

> **교수 멘트:**
> "Etherscan에서 보이는 모든 데이터가 실제 이더리움 메인넷의 데이터입니다. 여러분의 트랜잭션도 이더리움을 사용하면 이렇게 전 세계에 공개됩니다. 한번 기록되면 삭제하거나 수정할 수 없죠. 이것이 블록체인의 **불변성(Immutability)**입니다."

**관찰 과제:**
- 실제 이더리움 트랜잭션의 가스비는 얼마인지 확인
- 블록 하나에 몇 개의 트랜잭션이 들어있는지 확인
- 시뮬레이터와 실제 블록 탐색기의 트랜잭션 구조 비교

## Part 3: 정리 (30분)

### 핵심 내용 정리 (10분)

> **교수 멘트:**
> "오늘 암호화폐의 핵심을 정리하겠습니다."

**오늘의 핵심:**

1. **암호화폐:** 암호학 기반의 탈중앙화 디지털 자산
2. **비트코인 UTXO 모델:** 현금처럼 미사용 출력값을 소비하는 방식, 거스름돈 개념
3. **반감기:** 약 4년마다 블록 보상 절반 감소, 총 2,100만 BTC
4. **이더리움 어카운트 모델:** 은행 계좌처럼 잔액을 직접 관리하는 방식
5. **가스비:** 이더리움 트랜잭션 실행 비용, 복잡할수록 비쌈
6. **트랜잭션 구조:** sender, receiver, amount, gas, nonce, signature
7. **WIA-FIN-003:** 암호화폐 관련 국제 표준

### 비트코인 vs 이더리움 최종 비교 (5분)

| 구분 | 비트코인 | 이더리움 |
|------|---------|---------|
| **탄생** | 2009년 | 2015년 |
| **창시자** | 사토시 나카모토 | 비탈릭 부테린 |
| **합의 방식** | PoW | PoS (The Merge 이후) |
| **계정 모델** | UTXO | 어카운트 |
| **블록 시간** | ~10분 | ~12초 |
| **주요 용도** | 가치 저장 ("디지털 금") | 스마트 컨트랙트 플랫폼 |
| **총 발행량** | 2,100만 BTC | 제한 없음 (EIP-1559로 소각) |
| **프로그래밍** | 제한적 (Script) | 강력 (Solidity, EVM) |

> **교수 멘트:**
> "비트코인과 이더리움은 목적 자체가 다릅니다. 비트코인은 '디지털 금'으로 가치 저장에 초점, 이더리움은 '세계 컴퓨터'로 프로그래밍 가능한 블록체인 플랫폼에 초점을 맞추고 있습니다."

### 실무 연결 (5분)

> **교수 멘트:**
> "실무에서 암호화폐 관련 프로젝트를 진행할 때는 WIA-FIN-003 같은 표준을 반드시 참고해야 합니다. 특히 거래소를 운영하거나 결제 서비스를 개발할 때 트랜잭션 검증 절차, 지갑 보안 가이드라인을 준수하는 것이 중요합니다."

**WIA-FIN-003 표준의 실무 적용:**
- 암호화폐 거래소: 트랜잭션 모니터링, AML/KYC 절차
- 지갑 서비스: 개인키 관리, 다중 서명 지갑
- DeFi 서비스: 스마트 컨트랙트 감사, 가스비 최적화
- wiastandards.com에서 최신 표준 문서 확인 가능
