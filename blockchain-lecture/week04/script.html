Week 04 — 암호화폐 (WIA-FIN-003)

대전대학교 대학원 IT 소프트웨어공학 · 교수: 연삼흠 · 총 90분 (전반 40분 + 휴식 15분 + 후반 35분)

세션 1 비트코인 UTXO, 이더리움 Account 45분

00:00비트코인의 UTXO 모델

지난 3주간 블록체인의 기반 기술을 배웠습니다. 이제 그 기술 위에서 작동하는 암호화폐의 구체적인 구조를 살펴보겠습니다. 먼저 비트코인의 UTXO 모델입니다.
  • UTXO (Unspent Transaction Output): 사용되지 않은 트랜잭션 출력
  • 비트코인에는 "잔고"라는 개념이 없음 — 대신 UTXO의 합계가 잔고
  • 마치 지갑 안의 지폐들과 같음 (5만원권 2장 + 1만원권 3장 = 13만원)
UTXO 모델 예시: Alice의 UTXO: [UTXO #1: 0.5 BTC] + [UTXO #2: 0.3 BTC] = 잔고 0.8 BTC Alice → Bob에게 0.6 BTC 전송: 입력 (Input): UTXO #1 (0.5 BTC) + UTXO #2 (0.3 BTC) = 0.8 BTC 출력 (Output): Bob에게 0.6 BTC (새 UTXO) Alice에게 0.19 BTC (거스름돈, 새 UTXO) 수수료: 0.01 BTC (채굴자에게) UTXO #1, #2는 소멸 → 새 UTXO 2개 생성
은행처럼 잔고를 저장하는 것이 아니라, 사용되지 않은 "동전들"을 관리하는 것입니다. 10만원짜리 물건을 살 때 5만원권 2장을 내고 거스름돈을 받는 것과 같습니다.
"비트코인에는 잔고가 없다"는 말이 학생들에게 혼란을 줄 수 있습니다. 지갑 앱에서 보이는 잔고는 UTXO의 합계를 표시한 것일 뿐이라는 점을 명확히 설명하세요.

00:25이더리움의 Account 모델

이더리움은 비트코인과 다른 접근법을 택했습니다. 우리가 일상적으로 사용하는 은행 계좌와 비슷한 모델입니다.
  • Account 모델: 각 주소가 상태(잔고, 논스, 코드 등)를 직접 저장
  • 두 가지 계정 유형:
    • EOA (Externally Owned Account): 개인키로 제어, 일반 사용자 계정
    • CA (Contract Account): 스마트 컨트랙트 코드로 제어
비교 항목비트코인 (UTXO)이더리움 (Account)
잔고 관리UTXO 합계 계산계정에 직접 저장
프라이버시높음 (주소 재사용 방지)낮음 (같은 주소 반복 사용)
병렬 처리용이 (독립적 UTXO)어려움 (계정 상태 충돌)
스마트 컨트랙트제한적 (Script)튜링 완전 (Solidity)
복잡도트랜잭션은 단순, 검증은 복잡트랜잭션은 유연, 상태 관리 복잡
이더리움 Account 모델: ┌─────────────────────────┐ │ EOA (사용자 계정) │ │ 주소: 0xAbc... │ │ 잔고: 2.5 ETH │ │ 논스: 7 │ │ (코드 없음) │ └─────────────────────────┘ ┌─────────────────────────┐ │ CA (컨트랙트 계정) │ │ 주소: 0xDef... │ │ 잔고: 100 ETH │ │ 논스: 0 │ │ 코드: 스마트 컨트랙트 │ │ 스토리지: 상태 변수 │ └─────────────────────────┘
☕ 15분 휴식 (세션 1 종료)

세션 2 트랜잭션 구조, 수수료, WIA-FIN-003 45분

01:00트랜잭션 구조

트랜잭션은 블록체인의 기본 단위입니다. "A에서 B로 코인을 보낸다"는 행위가 어떤 구조로 기록되는지 살펴봅시다.

비트코인 트랜잭션:

{ "version": 2, "inputs": [ { "prev_tx": "abc123...", "output_index": 0, "scriptSig": "3045022100...", // 서명 "sequence": 0xffffffff } ], "outputs": [ { "value": 60000000, // 0.6 BTC (satoshi) "scriptPubKey": "OP_DUP OP_HASH160..." // 수신자 조건 }, { "value": 19000000, // 0.19 BTC (거스름돈) "scriptPubKey": "OP_DUP OP_HASH160..." // 자신에게 반환 } ], "locktime": 0 }

이더리움 트랜잭션:

{ "from": "0xAbc...", "to": "0xDef...", "value": "1000000000000000000", // 1 ETH (wei) "gas": 21000, "gasPrice": "20000000000", // 20 Gwei "nonce": 7, "data": "0x", // 단순 전송 시 빈 값 "signature": { v, r, s } }

01:20수수료 (가스)

블록체인에서 트랜잭션을 처리하려면 수수료를 지불해야 합니다. 이더리움에서는 이를 "가스(Gas)"라고 부릅니다.
  • 비트코인 수수료: 트랜잭션 크기(바이트)에 비례, 입력 UTXO가 많을수록 비쌈
  • 이더리움 가스: 연산 복잡도에 비례
    • Gas Limit: 최대 사용 가능 가스량
    • Gas Price: 가스 단위당 가격 (Gwei)
    • 수수료 = Gas Used x Gas Price
    • EIP-1559 이후: Base Fee(소각) + Priority Fee(검증자에게)
이더리움 연산가스 비용
단순 ETH 전송21,000
ERC-20 토큰 전송~65,000
스마트 컨트랙트 배포수백만
Uniswap 스왑~150,000
가스 시스템은 "무한 루프" 같은 악의적 코드가 네트워크를 마비시키는 것을 방지합니다. 연산에 비용을 부과함으로써 자원 남용을 경제적으로 억제하는 것이죠.

02:00테스트넷 실습

오늘의 실습은 실제 테스트넷에서 코인을 주고받아 봅니다. 실습 가이드를 화면에 띄워 드리겠습니다.
  • WIA 시뮬레이터: https://wiastandards.com/cryptocurrency/simulator
  • 학생들에게 lab-guide.html을 프로젝터에 표시
  • 실습 흐름:
  1. 시뮬레이터 접속 및 지갑 생성 (5분)
  2. 테스트넷 코인(Faucet) 수령 (5분)
  3. 팀원에게 코인 전송 (10분): 트랜잭션 생성 → 서명 → 전파 → 확인
  4. 트랜잭션 분석 (10분): 전송된 트랜잭션의 구조를 분석
  5. 수수료 비교 (10분): 다양한 가스 가격으로 전송하여 처리 속도 비교
테스트넷 Faucet에서 코인을 받는 데 시간이 걸릴 수 있으므로, 수업 전에 미리 Faucet URL을 공유하고 받아두도록 안내하는 것이 좋습니다.
