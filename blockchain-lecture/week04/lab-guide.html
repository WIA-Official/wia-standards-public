1 시뮬레이터 지갑 생성

WIA 시뮬레이터에서 테스트용 지갑을 생성하고 구조를 파악합니다.

  • WIA 시뮬레이터에 접속합니다
  • "지갑 생성" 버튼을 클릭하여 새 지갑을 만듭니다 개인키, 공개키, 주소가 생성되는 것을 확인하세요
  • 생성된 주소 형식을 확인합니다 (0x로 시작하는 42자)
  • 개인키를 안전한 곳에 기록합니다 실제 환경에서는 절대 디지털로 저장하지 않습니다
지갑 정보 기록: 주소: 0x________________________ 공개키: 04________________________ 개인키: (비밀 보관) 잔고: 0 TEST (초기 상태)

2 테스트넷 코인 수령 (Faucet)

테스트넷 Faucet에서 무료 테스트 코인을 수령합니다.

  • 시뮬레이터의 Faucet 기능을 찾습니다
  • 자신의 지갑 주소를 입력하여 테스트 코인을 요청합니다
  • 잔고에 코인이 반영되는 것을 확인합니다 반영까지 몇 초 ~ 몇 분이 걸릴 수 있습니다
  • 수령한 트랜잭션의 해시를 기록합니다

3 코인 전송 실습

팀원에게 테스트 코인을 전송하고 트랜잭션 과정을 이해합니다.

  • 팀원의 지갑 주소를 확인합니다
  • 팀원에게 1 TEST 코인을 전송합니다 수신 주소, 금액, 가스비를 설정하세요
  • 트랜잭션이 서명되고 전파되는 과정을 관찰합니다
  • 트랜잭션 해시를 기록하고 상태(Pending → Confirmed)를 확인합니다
  • 팀원으로부터 1 TEST 코인을 수신하여 잔고 변화를 확인합니다
전송 정보기록
수신 주소0x______________
전송 금액1 TEST
가스비_____ Gwei
트랜잭션 해시0x______________
확인 소요 시간_____ 초

4 트랜잭션 분석

전송한 트랜잭션의 상세 구조를 분석합니다.

  • 트랜잭션 해시를 이용하여 상세 정보를 조회합니다
  • 다음 항목들을 식별합니다: from, to, value, gas, nonce 각 필드가 무엇을 의미하는지 수업 내용과 대조하세요
  • 블록 번호와 블록 내 트랜잭션 위치를 확인합니다
  • 서명값(v, r, s)을 확인하고 2주차 디지털 서명과 연결합니다
  • 가스 사용량(Gas Used)과 실제 수수료를 계산합니다 수수료 = Gas Used x Gas Price

  2. 트랜잭션 분석 보고서 (A4 1페이지): 오늘 실습에서 전송한 트랜잭션의 구조를 스크린샷과 함께 분석하고, 각 필드의 의미를 설명하세요.
