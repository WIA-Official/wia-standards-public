Week 03 — 합의 알고리즘

대전대학교 대학원 IT 소프트웨어공학 · 교수: 연삼흠 · 총 90분 (토의 중심)

세션 1 이중지불 문제, 비잔틴 장군 문제 45분

00:00이중지불 문제 (Double Spending Problem)

디지털 세계에서 돈을 복사하면 어떻게 될까요? 종이 화폐는 물리적이라 복사가 어렵지만, 디지털 데이터는 복사가 쉽습니다. 이것이 이중지불 문제이고, 블록체인이 해결하려는 핵심 과제입니다.
  • 문제 정의: 같은 디지털 화폐를 두 번 이상 사용하는 것
  • 기존 해결책: 은행 같은 중앙 기관이 잔고를 관리 (중앙화)
  • 블록체인 해결책: 모든 참여자가 거래 내역을 공유하고 합의 (탈중앙화)
이중지불 시나리오: Alice 잔고: 10 BTC 거래 1: Alice → Bob (10 BTC) ← 정상 거래 2: Alice → Charlie (10 BTC) ← 이중지불! 해결: 네트워크가 거래 1을 먼저 확인하면 거래 2는 잔고 부족으로 거부됨 → 이를 위해 "합의 알고리즘"이 필요!
학생들에게 "만약 카카오페이 잔고를 복사할 수 있다면?"이라는 질문으로 시작하면 직관적으로 이해합니다.

00:20비잔틴 장군 문제 (Byzantine Generals Problem)

1982년 Lamport, Shostak, Pease가 제시한 고전적인 분산 시스템 문제입니다. 비잔틴 제국의 장군들이 성을 공격할지 퇴각할지 합의해야 하는데, 일부 장군이 배신자라면 어떻게 할까요?
┌─────────────────────────────────────────┐ │ 비잔틴 장군 문제 │ │ │ │ 장군A(충성) ──"공격"──→ 장군B(충성) │ │ │ │ │ │ │ ┌──────────┐ │ │ │ └──→│ 장군C │←───┘ │ │ │ (배신자!) │ │ │ └──────────┘ │ │ │ │ │ "퇴각" (거짓 정보 전달) │ │ │ │ 충성스러운 장군들이 어떻게 합의할 것인가? │ └─────────────────────────────────────────┘
  • 핵심: 신뢰할 수 없는 참여자가 있는 환경에서 어떻게 합의에 도달할 것인가?
  • BFT 조건: 배신자가 전체의 1/3 미만이면 합의 가능
  • 블록체인에서의 적용: 악의적 노드가 있어도 정직한 다수가 합의를 이룸
블록체인의 합의 알고리즘은 바로 이 비잔틴 장군 문제에 대한 실용적 해결책입니다. 각 알고리즘이 이 문제를 어떻게 풀어내는지 세션 2에서 비교해 보겠습니다.
☕ 15분 휴식 (세션 1 종료)

세션 2 PoW, PoS, DPoS, PBFT 비교 45분

01:00작업 증명 (Proof of Work)

가장 먼저 등장한 합의 알고리즘이 PoW입니다. "일을 많이 한 사람이 블록을 만든다"는 단순한 원칙입니다.
  • 원리: 난이도 조건을 만족하는 해시값(논스)을 먼저 찾는 채굴자가 블록 생성권 획득
  • 보안: 공격 비용 = 전체 해시파워의 51% 이상 (경제적으로 비합리적)
  • 단점: 막대한 에너지 소비, 느린 처리 속도, 채굴 풀의 중앙화 경향
  • 대표 체인: 비트코인, 라이트코인
비트코인 네트워크의 연간 전력 소비가 일부 국가의 전력 소비와 맞먹는다는 통계를 보여주면 학생들이 PoS 전환의 필요성을 실감합니다.

01:15지분 증명 (Proof of Stake)

PoW의 에너지 낭비 문제를 해결하기 위해 등장한 것이 PoS입니다. "많이 가진 사람이 검증한다"는 원칙이죠. 이더리움이 2022년 PoW에서 PoS로 전환한 'The Merge'가 대표적 사례입니다.
  • 원리: 코인을 예치(스테이킹)한 검증자 중에서 블록 생성자를 선정
  • 보안: 부정행위 시 스테이킹한 자산을 슬래싱(Slashing)으로 몰수
  • 장점: 에너지 효율적, 참여 장벽 낮음
  • 단점: 부자가 더 부자가 되는 구조(Nothing at Stake 문제)
  • 대표 체인: 이더리움 2.0, 카르다노, 솔라나

01:30DPoS와 PBFT

PoW와 PoS 외에도 다양한 합의 알고리즘이 있습니다. 각각의 장단점을 비교해 봅시다.

DPoS (Delegated Proof of Stake):

  • 토큰 보유자가 대표자(Delegate/Witness)를 투표로 선출
  • 소수의 대표자가 블록을 생성하므로 속도가 빠름
  • 대표 체인: EOS, TRON

PBFT (Practical Byzantine Fault Tolerance):

  • 참여자가 3단계 투표(Pre-prepare, Prepare, Commit)를 통해 합의
  • 허가형(Private) 블록체인에 적합
  • 대표 체인: Hyperledger Fabric
알고리즘속도에너지탈중앙화보안적합 환경
PoW느림매우 높음높음매우 높음퍼블릭
PoS빠름낮음중간높음퍼블릭
DPoS매우 빠름낮음낮음중간퍼블릭
PBFT빠름낮음낮음높음프라이빗
여기서 중요한 것은 "최고의 합의 알고리즘"은 없다는 것입니다. 각각 트레이드오프가 있고, 용도에 따라 적합한 알고리즘이 다릅니다. 이것이 블록체인 트릴레마(보안-탈중앙화-확장성)의 핵심입니다.
📋 수업 정리

세션 3 역할극 실습 — PoS 검증자 역할극, 합의 토론 45분

02:00역할극: 비잔틴 장군 & PoS 검증자

이론만으로는 합의 알고리즘의 미묘한 차이를 체감하기 어렵습니다. 직접 역할을 맡아서 체험해 봅시다. 실습 가이드를 화면에 띄워 드리겠습니다.

실습 1: 비잔틴 장군 역할극 (10분)

  1. 5~6명을 한 조로 구성 (장군 역할)
  2. 1명을 비밀리에 "배신자"로 지정
  3. 각 장군은 쪽지를 통해 "공격" 또는 "퇴각" 메시지 전달
  4. 배신자는 서로 다른 메시지를 보냄
  5. 충성스러운 장군들이 올바른 합의에 도달할 수 있는지 관찰

실습 2: PoS 검증자 역할극 (15분)

  1. 각 학생에게 가상 코인(사탕, 종이 등)을 차등 배분
  2. 스테이킹 금액에 비례하여 검증자 선정 확률 결정
  3. 주사위를 굴려 검증자 선정
  4. 검증자가 트랜잭션을 검증하고 보상 수령
  5. 부정행위 시 스테이킹 몰수(슬래싱) 시뮬레이션
사탕이나 초콜릿을 코인 대용으로 사용하면 학생들의 참여도가 높아집니다. 슬래싱 시 실제로 사탕을 회수하면 더 효과적입니다!

02:25합의 토론: "최적의 합의 알고리즘은?"

이제 조별로 토론을 해 봅시다. 각 조는 하나의 합의 알고리즘을 맡아 그 알고리즘이 가장 우수하다는 논리를 준비하세요.
  • 조별로 PoW, PoS, DPoS, PBFT 중 하나를 배정
  • 5분간 논리 준비 → 조당 2분 발표 → 자유 토론 5분
  • 토론 주제: "2030년에 가장 널리 쓰일 합의 알고리즘은?"

02:35마무리

오늘 수업에서 가장 중요한 것은 "완벽한 합의 알고리즘은 없다"는 것입니다. 보안, 탈중앙화, 확장성 — 이 세 가지를 동시에 만족시키기 어렵고, 이것이 블록체인 트릴레마입니다. 다음 주에는 암호화폐의 구체적 구조를 배우겠습니다.
  • 핵심 정리: 이중지불 → 비잔틴 장군 → 합의 알고리즘(PoW/PoS/DPoS/PBFT)
  • 다음 주: 암호화폐 — 비트코인 UTXO, 이더리움 Account 모델
  • 과제: 실습 가이드 마지막 페이지 참조
