## Part 1: 이론 (30분)

### 도입 (5분)

> **교수 멘트:**
> "여러분, 지난 시간에 블록체인의 구조와 해시에 대해 배웠죠? 오늘은 블록체인의 **핵심 중의 핵심**인 합의 알고리즘을 다뤄보겠습니다. 질문 하나 드릴게요. 서로 모르는 사람들끼리, 중앙 관리자 없이, 어떻게 '이 거래가 맞다'고 동의할 수 있을까요? 은행 없이 돈을 보내려면 누가 그걸 확인해줄까요?"

> **교수 멘트:**
> "이 문제를 해결하는 것이 바로 **합의 알고리즘**입니다. 합의 알고리즘은 블록체인 네트워크의 참여자들이 동일한 데이터 상태에 합의하는 방법을 말합니다."

### 1-1. 왜 합의가 필요한가? - 이중지불 문제 (5분)

> **교수 멘트:**
> "합의가 왜 필요한지, 이중지불(Double Spending) 문제로 설명해볼게요. 여러분이 디지털 파일 하나를 가지고 있다고 생각해보세요. 사진 파일은 복사-붙여넣기가 되죠? 만약 디지털 화폐도 복사가 된다면 어떻게 될까요?"

**이중지불 문제란?**
- 동일한 디지털 화폐를 두 번 이상 사용하는 것
- 디지털 데이터는 복제가 쉬움 → 돈도 복제 가능?
- 기존 금융: 은행이라는 **중앙기관**이 잔액을 관리하여 해결
- 블록체인: 중앙기관 없이 **네트워크 참여자들의 합의**로 해결

> **교수 멘트:**
> "앨리스가 1 BTC를 가지고 있는데, 밥에게도 1 BTC를 보내고 찰리에게도 1 BTC를 보내면요? 둘 중 하나는 가짜 거래가 됩니다. 은행이 없는 환경에서 네트워크가 '밥에게 보낸 게 진짜다'라고 합의해야 하는 거죠. 이걸 해결하는 메커니즘이 합의 알고리즘입니다."

### 1-2. 비잔틴 장군 문제 (5분)

> **교수 멘트:**
> "합의 문제를 이해하기 위해 유명한 사고실험을 하나 소개합니다. 1982년에 레슬리 램포트 등이 발표한 **비잔틴 장군 문제(Byzantine Generals Problem)**입니다."

**비잔틴 장군 문제 설명:**
- 비잔틴 제국의 여러 장군이 적의 성을 포위
- 공격 또는 후퇴를 **동시에** 결정해야 함 (일부만 공격하면 패배)
- 장군들은 전령(메시지)으로만 소통
- **문제:** 장군 중에 **배신자**가 있을 수 있음
  - 배신자는 서로 다른 장군에게 다른 메시지를 보냄
  - "나는 공격한다" vs "나는 후퇴한다" 동시에 전달

> **교수 멘트:**
> "이 문제를 블록체인에 대입해보면, 장군은 노드(컴퓨터), 전령은 네트워크 메시지, 배신자는 악의적인 노드입니다. 배신자가 있어도 나머지 정직한 노드들이 올바른 합의에 도달할 수 있어야 합니다. 이것이 **비잔틴 장애 허용(BFT)**의 핵심이에요."

**핵심 조건:**
- 전체 노드 중 배신자가 1/3 미만이면 합의 가능
- 배신자가 1/3 이상이면 합의 불가능 (수학적으로 증명됨)

### 1-3. PoW - 작업증명 (5분)

> **교수 멘트:**
> "자, 이제 실제 블록체인에서 사용하는 합의 알고리즘을 하나씩 살펴봅시다. 첫 번째는 **PoW(Proof of Work, 작업증명)**입니다. 비트코인이 사용하는 방식이죠."

**PoW (Proof of Work, 작업증명):**
- **핵심 원리:** 어려운 수학 문제를 가장 먼저 푸는 노드가 블록 생성 권한을 얻음
- 실제로는 해시값에서 앞에 0이 특정 개수 이상 나오는 nonce를 찾는 것
- **채굴(Mining):** 이 과정을 채굴이라 부르고, 채굴에 성공한 노드에게 보상(블록 보상 + 수수료)

**장점:**
- 검증된 보안성 (비트코인 15년 이상 운영)
- 단순하고 직관적인 메커니즘
- 공격 비용이 매우 높음 (51% 공격 비용 막대)

**단점:**
- 막대한 전력 소모 (비트코인 네트워크 = 중소국가 전력 소비)
- 느린 처리 속도 (비트코인: ~7 TPS)
- 채굴 집중화 문제 (대형 채굴 풀)

> **교수 멘트:**
> "비트코인 채굴에 들어가는 전력이 어마어마합니다. 아르헨티나 한 나라 수준이라는 보고도 있어요. 환경 문제로 많은 비판을 받고 있고, 이 때문에 다른 합의 알고리즘이 등장했습니다."

### 1-4. PoS - 지분증명 (3분)

> **교수 멘트:**
> "PoW의 에너지 문제를 해결하기 위해 나온 것이 **PoS(Proof of Stake, 지분증명)**입니다. 이더리움이 2022년 '더 머지(The Merge)'를 통해 PoW에서 PoS로 전환했죠."

**PoS (Proof of Stake, 지분증명):**
- **핵심 원리:** 코인을 많이 예치(스테이킹)한 노드가 블록 생성 권한을 확률적으로 얻음
- 수학 문제를 풀 필요 없음 → 에너지 절약
- **스테이킹(Staking):** 자신의 코인을 담보로 맡기는 것
  - 이더리움: 32 ETH 스테이킹 → 검증자(Validator) 참여 가능
  - 부정행위 시 스테이킹한 코인 **슬래싱(삭감)**

**장점:**
- 에너지 효율적 (PoW 대비 99.95% 에너지 절감)
- 더 빠른 트랜잭션 처리

**단점:**
- 부익부 빈익부: 코인 부자가 더 많은 보상 획득
- Nothing at Stake 문제
- 초기 분배 공정성 문제

### 1-5. DPoS - 위임지분증명 (3분)

**DPoS (Delegated Proof of Stake, 위임지분증명):**
- **대표 프로젝트:** EOS
- **핵심 원리:** 코인 보유자가 **대표자(위임자)**를 투표로 선출, 대표자가 블록 생성
- EOS의 경우 21명의 블록 프로듀서(BP)를 선출

> **교수 멘트:**
> "DPoS는 대의 민주주의와 비슷합니다. 국민이 직접 법을 만들지 않고 국회의원을 뽑아서 위임하듯이, 코인 보유자가 블록 생성자를 뽑는 거죠. 장점은 빠르고 효율적이지만, 소수에게 권력이 집중될 수 있다는 단점이 있어요."

**장점:** 빠른 처리 속도, 확장성 우수
**단점:** 중앙화 우려, 담합 가능성

### 1-6. PBFT - 실용적 비잔틴 장애 허용 (2분)

**PBFT (Practical Byzantine Fault Tolerance):**
- **대표 프로젝트:** Hyperledger Fabric
- **핵심 원리:** 모든 노드가 서로 통신하여 2/3 이상 합의 시 블록 확정
- 노드 수가 적은 허가형(프라이빗) 블록체인에 적합
- Pre-prepare → Prepare → Commit 3단계 프로토콜

> **교수 멘트:**
> "PBFT는 참여 노드가 정해져 있는 프라이빗 블록체인에서 주로 사용됩니다. 노드 수가 많아지면 통신량이 기하급수적으로 늘어나는 한계가 있지만, 금융기관 같은 곳에서는 아주 유용해요."

### 1-7. 합의 알고리즘 비교표 (2분)

| 구분 | PoW | PoS | DPoS | PBFT |
|------|-----|-----|------|------|
| **대표 프로젝트** | 비트코인 | 이더리움 2.0 | EOS | Hyperledger |
| **블록 생성 기준** | 연산력 | 지분(코인) | 투표된 대표자 | 전체 노드 합의 |
| **에너지 효율** | 매우 낮음 | 높음 | 높음 | 높음 |
| **처리 속도 (TPS)** | ~7 | ~30 | ~4,000 | ~3,000 |
| **탈중앙화 정도** | 높음 | 중간 | 낮음 | 낮음 |
| **보안성** | 매우 높음 | 높음 | 중간 | 높음 (노드 제한) |
| **블록체인 유형** | 퍼블릭 | 퍼블릭 | 퍼블릭 | 프라이빗/컨소시엄 |
| **최종성(Finality)** | 확률적 | 확률적 | 빠른 확정 | 즉시 확정 |

> **교수 멘트:**
> "이 비교표를 잘 기억해두세요. 합의 알고리즘은 블록체인의 성격을 결정짓는 핵심 요소입니다. 어떤 알고리즘이 '최고'인 것은 아니고, 블록체인의 목적에 따라 적합한 것을 선택하는 것이 중요합니다. 자, 이제 실습으로 넘어가겠습니다!"

## Part 2: 실습 (60분)

> **교수 멘트:**
> "이번 주 실습은 시뮬레이터가 아닌 **역할극과 토의** 중심으로 진행합니다. 직접 몸으로 체험해야 합의 알고리즘의 원리가 확실히 이해됩니다. 5명씩 조를 편성해주세요!"

### 실습 1: 비잔틴 장군 문제 역할극 (15분)

> **교수 멘트:**
> "첫 번째 실습입니다. 5명이 한 조가 되어 비잔틴 장군 문제를 직접 체험해봅니다."

**진행 방법:**
1. 5명 중 역할 배정 (교수가 비밀리에 배정)
   - **정직한 장군 4명:** 진실만 말함
   - **배신자 1명:** 거짓 메시지를 보낼 수 있음
2. 목표: **공격** 또는 **후퇴** 중 하나로 합의
3. 각 장군은 종이에 자신의 의견을 적어 다른 장군에게 전달
4. 배신자는 서로 다른 장군에게 다른 내용을 전달 가능
5. 3라운드 진행 후 최종 합의 시도

**관찰 포인트:**
- 배신자가 1명일 때: 합의가 가능한가?
- 배신자를 찾아낼 수 있는가?
- 어떤 전략이 효과적인가?

> **교수 멘트:**
> "자, 배신자가 누군지 밝히지 마시고요. 3라운드가 끝난 후에 여러분이 누가 배신자인지 추측해보세요. 정직한 장군들이 올바르게 합의에 도달했는지 확인해봅시다."

**라운드 종료 후 토의:**
- 정직한 장군들이 합의에 성공했는가?
- 배신자를 올바르게 식별했는가?
- 배신자가 2명이면 어떻게 될까? (5명 중 2명이면 2/5 > 1/3 → 합의 불가능 시연)

### 실습 2: PoW 시뮬레이션 - 주사위 채굴 (15분)

> **교수 멘트:**
> "이번에는 PoW를 주사위로 체험해봅시다. 채굴이 얼마나 힘들고 운에 좌우되는지 직접 느껴보세요."

**준비물:** 각 조별 주사위 2개

**진행 방법:**
1. **목표:** 주사위 2개를 굴려서 합이 **특정 조건**을 만족하는 것
   - 난이도 1: 합이 7 이상 (쉬움)
   - 난이도 2: 합이 11 이상 (어려움)
   - 난이도 3: 합이 정확히 12 (매우 어려움)
2. 가장 먼저 조건을 만족한 학생이 **채굴 성공** → 보상(가상 포인트) 획득
3. 5라운드 진행, 난이도를 점점 올림

**기록표:**

| 라운드 | 난이도 | 채굴 성공자 | 시도 횟수 |
|--------|--------|------------|----------|
| 1 | 합 ≥ 7 | | |
| 2 | 합 ≥ 9 | | |
| 3 | 합 ≥ 10 | | |
| 4 | 합 ≥ 11 | | |
| 5 | 합 = 12 | | |

> **교수 멘트:**
> "난이도가 올라갈수록 시간이 오래 걸리죠? 이것이 바로 비트코인의 **난이도 조절**과 같은 원리입니다. 비트코인은 약 10분에 한 블록이 생성되도록 난이도를 자동 조절합니다. 채굴 장비가 좋아지면 난이도가 올라가고, 빠져나가면 난이도가 내려가죠."

**토의 포인트:**
- 주사위를 더 많이 가진 사람(= 채굴 장비가 좋은 사람)이 유리하지 않은가?
- 이것이 채굴 집중화 문제와 어떤 관련이 있는가?

### 실습 3: PoS 역할극 - 스테이킹과 검증 (10분)

> **교수 멘트:**
> "이번에는 PoS를 체험합니다. 각자 가상 코인을 가지고 시작합니다."

**진행 방법:**
1. 각 조원에게 가상 코인 배분 (불균등하게)
   - 학생 A: 50 코인
   - 학생 B: 30 코인
   - 학생 C: 10 코인
   - 학생 D: 7 코인
   - 학생 E: 3 코인
2. 스테이킹 양에 비례하여 검증자 선택 확률 결정
3. 주사위 또는 제비뽑기로 검증자 선정 (확률 반영)
4. 검증자가 블록 생성, 보상 획득
5. 5라운드 반복

> **교수 멘트:**
> "코인을 많이 가진 학생이 검증자로 선택될 확률이 높죠? 그리고 보상을 받으면 코인이 더 많아지니까 다음에도 더 유리해집니다. 이것이 PoS의 **부익부 빈익부** 문제입니다. 여러분, 이 시스템이 공정하다고 생각하시나요?"

**토의 포인트:**
- PoW와 PoS 중 어느 것이 더 공정한가?
- 부익부 빈익부 문제의 해결 방법은?
- 스테이킹에 참여하지 못하는 소액 보유자의 입장은?

### 실습 4: 합의 알고리즘 비교 토론 (15분)

> **교수 멘트:**
> "자, 이제 각 합의 알고리즘을 직접 체험해봤으니 본격적으로 토론해봅시다. 각 조는 하나의 합의 알고리즘을 맡아서 **장점을 옹호**하는 입장에서 발표합니다."

**토론 주제:** "가장 이상적인 합의 알고리즘은 무엇인가?"

**조별 배정:**
- 1조: PoW 옹호팀 - "보안성이 가장 검증된 알고리즘이다"

- 4조: PBFT 옹호팀 - "기업 환경에 가장 적합하다"

**토론 진행:**
1. 각 조 5분간 준비

**오늘의 핵심:**

1. **합의가 필요한 이유:** 중앙기관 없이 이중지불 문제를 해결하기 위해
2. **비잔틴 장군 문제:** 배신자가 있는 환경에서 합의를 달성하는 문제
3. **PoW:** 연산력으로 경쟁, 보안성 높지만 에너지 낭비
4. **PoS:** 지분으로 검증, 에너지 효율적이지만 부익부 문제
5. **DPoS:** 대표자 선출, 빠르지만 중앙화 우려
6. **PBFT:** 전체 노드 합의, 프라이빗에 적합하지만 확장성 한계
