1 WIA 표준 탐색

WIA(Web & Internet Architecture) 표준 사이트를 탐색하며 블록체인 표준화 현황을 파악합니다.

참고 사이트: https://wiastandards.com
  • wiastandards.com에 접속하여 메인 페이지를 살펴봅니다 사이트 구조와 주요 표준 영역을 파악하세요
  • WIA-SEC (보안) 섹션을 찾아 주요 항목을 메모합니다 특히 암호학 관련 항목에 주목하세요
  • WIA-FIN (금융) 섹션을 찾아 암호화폐 관련 표준을 확인합니다
  • 발견한 내용을 팀원과 공유하고 토론합니다 (3분)

2 블록 생성 실습

블록의 구성 요소를 직접 입력하고 해시 생성 과정을 체험합니다.

  • 블록 시뮬레이터를 열고 Block #1을 생성합니다 데이터 필드에 "대전대학교 블록체인 첫 번째 블록"을 입력하세요
  • 생성된 해시값을 기록합니다 해시값이 몇 자리인지, 어떤 문자로 구성되어 있는지 관찰하세요
  • Block #2를 생성하여 이전 블록 해시와 연결을 확인합니다
  • Block #3까지 생성하여 체인 형태를 완성합니다
예시 해시값: Block #1: 0000a3f2... (SHA-256, 64자 16진수) Block #2: 0000b7c1... (Previous Hash = Block #1의 해시) Block #3: 00004e8d... (Previous Hash = Block #2의 해시)

3 위변조 실험

블록 데이터를 변경하면 해시 무결성이 어떻게 깨지는지 확인합니다.

  • 3개 블록 체인이 완성된 상태에서 Block #2의 데이터를 수정합니다 "대전대학교"를 "서울대학교"로 변경해 보세요
  • Block #2의 해시값이 변경되는 것을 확인합니다
  • Block #3의 "이전 블록 해시"와 불일치가 발생하는 것을 관찰합니다 시뮬레이터에서 빨간색으로 표시되는 부분을 확인하세요
  • 이 결과가 블록체인 보안에 어떤 의미를 가지는지 팀원과 토론합니다 (5분)

4 2인 1조 네트워크 실습

팀원과 함께 분산 네트워크의 개념을 시뮬레이션합니다.

  • 팀원 A: 동일한 데이터로 블록 3개를 생성합니다
  • 팀원 B: 동일한 데이터로 블록 3개를 생성합니다
  • 두 체인의 해시값이 동일한지 비교합니다 동일한 입력 → 동일한 출력 (결정적 함수의 특성)
  • 팀원 A만 Block #2를 수정한 후, 두 체인을 다시 비교합니다 정직한 다수가 원본을 유지하면 위변조를 감지할 수 있음을 확인

확인 퀴즈

Q1. 블록의 헤더에 포함되지 않는 것은?

Q2. 분산원장의 가장 큰 장점은?

과제 (Homework)

  2. 블록체인 활용 사례 조사 (A4 1페이지): 블록체인이 금융 외 분야(의료, 물류, 투표 등)에서 활용되는 실제 사례 하나를 조사하여 간단히 보고서를 작성하세요.
