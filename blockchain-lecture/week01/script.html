Week 01 — 블록체인 개론

대전대학교 대학원 IT 소프트웨어공학 · 교수: 연삼흠 · 총 90분 (전반 40분 + 휴식 15분 + 후반 35분)

세션 1 블록체인 정의, 역사, 분산원장 45분

00:00블록체인이란 무엇인가?

여러분, 오늘부터 블록체인 과목을 시작하겠습니다. 먼저 간단히 질문을 드리겠습니다 — "블록체인" 하면 가장 먼저 떠오르는 단어가 무엇인가요? 비트코인? 암호화폐? 네, 그것도 맞지만, 블록체인은 그보다 훨씬 넓은 개념입니다.
  • 핵심 정의: 블록체인은 분산 환경에서 데이터 무결성을 보장하는 연결 리스트(linked list) 기반의 데이터 구조입니다.
  • 각 블록(Block)은 이전 블록의 해시값을 포함하여 체인(Chain)을 형성
  • 중앙 관리자가 없어도 참여자들의 합의를 통해 데이터 신뢰성 확보
  • 3가지 핵심 속성: 탈중앙화(Decentralization), 투명성(Transparency), 불변성(Immutability)
학생들에게 "은행 없이 송금이 가능할까?"라는 질문으로 호기심을 유도하면 효과적입니다.

00:15블록체인의 역사

블록체인의 역사는 2008년 사토시 나카모토의 백서에서 시작된다고 알려져 있지만, 사실 그 이전에 많은 기반 기술들이 있었습니다. 하나씩 살펴보겠습니다.
연도사건의의
1991Haber & Stornetta — 타임스탬프 체인최초의 블록 연결 개념
2004Hal Finney — Reusable PoW작업 증명 재사용 개념
2008사토시 나카모토 백서P2P 전자화폐 시스템 제안
2009비트코인 제네시스 블록최초의 블록체인 가동
2015이더리움 출시스마트 컨트랙트 플랫폼
2020+DeFi, NFT, CBDC블록체인 대중화 시대
사토시 나카모토가 누구인지 학생들에게 추측하게 하면 흥미를 유발할 수 있습니다. 아직까지 정체가 밝혀지지 않았다는 점을 강조하세요.

00:30분산원장(DLT) 개념

기존 시스템은 은행 같은 중앙기관이 장부를 관리합니다. 그런데 만약 참여자 모두가 동일한 장부를 가지고 있다면 어떨까요? 이것이 분산원장의 핵심 아이디어입니다.
┌──────────────────────────┐ ┌──────────────────────────┐ │ 중앙화 시스템 │ │ 분산원장 시스템 │ │ │ │ │ │ ┌──────┐ │ │ ┌──┐ ┌──┐ ┌──┐ │ │ │ 서버 │ │ │ │A │──│B │──│C │ │ │ └──┬───┘ │ │ └┬─┘ └┬─┘ └┬─┘ │ │ ┌────┼────┐ │ │ │ │ │ │ │ A B C │ │ ┌┴─┐ ┌┴─┐ ┌┴─┐ │ │ (클라이언트) │ │ │D │──│E │──│F │ │ │ │ │ └──┘ └──┘ └──┘ │ │ 단일 장애점 존재 │ │ 모든 노드가 장부 보유 │ └──────────────────────────┘ └──────────────────────────┘
  • 중앙화 vs 분산화: 단일 장애점(Single Point of Failure) 제거
  • DLT의 유형: 퍼블릭, 프라이빗, 컨소시엄
  • 블록체인은 DLT의 한 구현 방식 (모든 DLT가 블록체인은 아님)

세션 2 WIA 표준 소개, 블록 구조 45분

01:00WIA 표준 소개

블록체인 기술이 발전하면서 표준화의 필요성이 대두되었습니다. WIA(Web & Internet Architecture) 표준은 블록체인 기술의 상호운용성과 보안성을 위한 국제 프레임워크입니다.
  • WIA 표준이란: 블록체인 및 분산 기술의 국제 표준화 프레임워크
  • 주요 영역: 보안(SEC), 금융(FIN), 신원(ID), 데이터(DAT)
  • 공식 사이트: wiastandards.com
  • 이 수업에서 다룰 WIA 표준: WIA-SEC-002(암호학), WIA-FIN-003(암호화폐)
학생들에게 wiastandards.com을 미리 즐겨찾기하도록 안내하세요. 이후 실습에서 자주 참조합니다.

01:20블록의 구조

이제 블록 하나를 해부해 보겠습니다. 블록은 크게 헤더(Header)와 바디(Body)로 나뉩니다.
┌─────────────────────────────────────┐ │ Block #N │ ├─────────────────────────────────────┤ │ Block Header │ │ ┌─────────────────────────────┐ │ │ │ Previous Block Hash │ │ │ │ Timestamp │ │ │ │ Nonce │ │ │ │ Merkle Root │ │ │ │ Difficulty Target │ │ │ └─────────────────────────────┘ │ ├─────────────────────────────────────┤ │ Block Body │ │ ┌────┐ ┌────┐ ┌────┐ ┌────┐ │ │ │Tx 1│ │Tx 2│ │Tx 3│ │Tx 4│ │ │ └────┘ └────┘ └────┘ └────┘ │ └─────────────────────────────────────┘
  • 이전 블록 해시: 체인 연결의 핵심, 이전 블록 헤더의 SHA-256 해시
  • 타임스탬프: 블록 생성 시각
  • 논스(Nonce): 채굴 과정에서 조정하는 값
  • 머클 루트: 모든 트랜잭션의 요약 해시 (다음 주 암호학에서 상세 학습)
  • 난이도 목표: 채굴 난이도 조절 파라미터
머클 트리(Merkle Tree) 미리보기
Merkle Root / \ H(AB) H(CD) / \ / \ H(A) H(B) H(C) H(D) | | | | Tx A Tx B Tx C Tx D

머클 트리는 트랜잭션들을 쌍으로 묶어 해시하고, 이를 반복하여 최종 루트 해시를 만드는 구조입니다. 2주차에서 자세히 다룹니다.

블록 하나가 변조되면 해시값이 변하고, 그 뒤의 모든 블록도 연쇄적으로 변해야 합니다. 이것이 블록체인의 불변성을 보장하는 핵심 원리입니다.

세션 3 실습 — 블록 생성 시뮬레이터 45분

02:00블록 생성 시뮬레이터 체험

이론을 충분히 다뤘으니, 이제 직접 체험해 봅시다. 실습 가이드를 화면에 띄워 드리겠습니다. 2인 1조로 진행합니다.
  • 학생들에게 lab-guide.html을 프로젝터에 표시
  • 2인 1조 팀을 구성 (총 인원에 따라 조정)
  • 블록 생성 시뮬레이터에서 데이터 입력 → 해시 생성 과정 관찰
  • 각 팀이 서로 다른 데이터를 입력하여 해시값 비교
학생들이 시뮬레이터에 접속하는 데 시간이 걸릴 수 있으므로, 미리 URL을 채팅이나 LMS에 공유해 두세요.

02:20위변조 실험

이번에는 블록 하나의 데이터를 의도적으로 변경해 봅시다. 어떤 일이 벌어지는지 직접 확인하세요.
  1. 3개의 블록이 연결된 체인을 생성
  2. 중간 블록(Block #2)의 데이터를 수정
  3. 해시 불일치가 연쇄적으로 발생하는 과정을 관찰
  4. 각 팀이 결과를 발표 (2분씩)
보셨죠? 블록 하나만 바꿔도 그 뒤의 모든 블록이 "깨집니다". 51% 이상의 노드를 동시에 바꾸지 않는 한, 위변조는 사실상 불가능합니다.
