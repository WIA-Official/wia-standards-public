Chapter 2

Data Structures for Genome Editing

2.1 Introduction to Genomic Data Formats

Effective genome editing requires standardized data structures to represent sequences, edits, validation results, and clinical outcomes. This chapter explores the essential data formats used throughout the genome editing pipeline, from initial sequence upload to final therapeutic validation.

Modern genome editing workflows generate massive amounts of data: sequencing reads, variant calls, editing efficiency metrics, off-target analysis, and clinical outcomes. Standardized data formats ensure interoperability between different tools, enable reproducible research, and facilitate regulatory compliance for clinical applications.

2.2 Sequence Data Formats

FASTA Format

FASTA is the simplest and most widely used format for representing nucleotide or protein sequences. Each sequence begins with a header line starting with '>' followed by the sequence identifier and optional description, followed by the sequence data on subsequent lines.

>HBB_human_chr11:5227002-5229395 Hemoglobin subunit beta ATGGTGCATCTGACTCCTGAGGAGAAGTCTGCCGTTACTGCCCTGTGGGGCAAGGTGAAC GTGGATGAAGTTGGTGGTGAGGCCCTGGGCAGGTTGGTATCAAGGTTACAAGACAGGTTT AAGGAGACCAATAGAAACTGGGCATGTGGAGACAGAGAAGACTCTTGGGTTTCTGATAGG

FASTQ Format

FASTQ extends FASTA by including base quality scores, essential for Next-Generation Sequencing (NGS) data. Each sequence record consists of four lines: header, sequence, separator (+), and quality scores encoded as ASCII characters.

@SEQ_ID_001 GTGCACCTGACTCCTGAGGAGAAGTCTGCCGTTACTGCCCTGTGGGGCAAG + IIIIIIIHHHGGGFFFEEEDDCCCBBBAAAZZZYYYXXXWWWVVVUUUTTT

BAM/SAM Format

Sequence Alignment/Map (SAM) format stores aligned sequencing reads against a reference genome. BAM is the compressed binary version of SAM, dramatically reducing file sizes. These formats are critical for variant calling and editing validation.

Format Type Use Case Compression Typical Size
FASTA Text Reference sequences None (or .gz) 3 GB (human genome)
FASTQ Text Raw sequencing reads .gz typical 50-200 GB per run
SAM Text Aligned reads (human-readable) None 100-300 GB
BAM Binary Aligned reads (compressed) BGZF 20-50 GB
VCF Text Variant calls .gz typical 1-10 GB

2.3 Guide RNA Data Structures

Guide RNAs are central to CRISPR-based genome editing. Standardized data structures for gRNAs must capture the sequence, target location, predicted efficiency, and off-target potential.

Core gRNA Fields

{ "guide_rna": { "id": "gRNA-HBB-001", "sequence": "GTGCACCTGACTCCTGAGGAGAA", "length": 23, "pam_sequence": "AGG", "pam_position": "3prime", "target_gene": "HBB", "chromosome": "11", "genomic_position": 5227020, "strand": "+", "design_score": 95.2, "on_target_score": 97.3, "specificity_score": 98.5, "predicted_efficiency": 89.5, "gc_content": 56.5, "off_target_count": 3 } }

Off-Target Prediction Data

Off-target sites must be systematically cataloged with their genomic locations, mismatch patterns, and predicted cleavage scores. This information is critical for safety assessment.

{ "off_target_sites": [ { "rank": 1, "chromosome": "15", "position": 8234501, "sequence": "GTGCACCTGACTGCTGAGGAGAA", "mismatches": 1, "mismatch_positions": [12], "score": 12.4, "in_gene": false, "risk_level": "low" } ] }

2.4 Edit Outcome Data Structures

CRISPR-Cas9 Editing Outcomes

CRISPR-Cas9 creates double-strand breaks repaired by either Non-Homologous End Joining (NHEJ) or Homology-Directed Repair (HDR). Data structures must capture the distribution of repair outcomes.

{ "crispr_outcome": { "edit_id": "CRISPR-001", "target": "HBB_exon1", "total_alleles": 100000, "edited_alleles": 85300, "editing_efficiency": 85.3, "repair_pathway_distribution": { "nhej": 78.2, "hdr": 12.5, "unedited": 9.3 }, "indel_spectrum": { "1bp_deletion": 28.4, "2bp_deletion": 15.6, "1bp_insertion": 12.3, "larger_deletions": 21.9 } } }

Base Editing Outcomes

Base editors produce cleaner outcomes than CRISPR-Cas9, primarily generating the desired base conversion with some bystander editing and minimal indels.

{ "base_edit_outcome": { "edit_id": "BE-HBB-001", "editor": "ABE8.20-m", "target_base": "A", "target_position": 5227021, "edited_base": "G", "on_target_efficiency": 68.5, "product_purity": 92.3, "allele_distribution": { "wildtype": 24.8, "desired_edit": 68.5, "indels": 2.1, "bystander_edits": 4.6 }, "bystander_editing": { "position_5227018": {"change": "A>G", "frequency": 5.2} } } }

2.5 Validation and Quality Metrics

Rigorous quality control is essential for therapeutic genome editing. Validation data structures must capture sequencing quality, editing outcomes, and safety assessments.

Sequencing Quality Metrics

  • Phred Quality Score (Q): Q30 means 99.9% base call accuracy (1 in 1000 error)
  • Coverage Depth: Number of reads covering each position (50,000x for clinical validation)
  • Mapping Quality: Confidence that reads are correctly aligned (≥60 for high confidence)
  • Strand Bias: Balanced representation from forward and reverse strands (0.3-0.7 optimal)
Metric Research Grade Clinical Grade Measurement Method
Base Quality Q20 (99%) Q30 (99.9%) Phred score from sequencer
Coverage Depth 1,000-10,000x 50,000x+ Read count per position
Mapping Quality ≥30 ≥60 BWA/Bowtie alignment score
On-Target Efficiency ≥30% ≥50% CRISPResso2 analysis
Off-Target Rate ≤5% of on-target ≤1% of on-target GUIDE-seq or targeted NGS

2.6 Clinical Data Standards

HL7 FHIR for Genome Editing

Fast Healthcare Interoperability Resources (FHIR) is becoming the standard for representing healthcare data. Genome editing workflows integrate with Electronic Health Records (EHRs) through FHIR resources.

FHIR Patient Resource with Genetic Data: FHIR enables linking patient demographics, diagnoses, treatments, and genome editing outcomes in a standardized, interoperable format that can be shared across healthcare systems.

Gene Therapy Manufacturing Data

For FDA-approved therapies like Casgevy, comprehensive manufacturing data must be captured: cell counts, editing efficiency, sterility testing, potency assays, and release criteria.

{ "manufacturing_lot": { "lot_number": "CASGEVY-2026-001", "patient_id": "PT-SCD-001", "apheresis_date": "2026-01-05", "cd34_input": 12.5e6, "editing_efficiency": 78.5, "final_cell_dose": 8.5e6, "viability": 97.2, "sterility": "pass", "mycoplasma": "negative", "endotoxin_level": 2.3, "release_status": "approved" } }

2.7 Data Integration and Interoperability

Modern genome editing workflows require integration of data from multiple sources: sequencing platforms, bioinformatics tools, LIMS systems, and EHRs. Standardized APIs and data formats enable this integration.

REST API JSON Responses

Web APIs use JSON (JavaScript Object Notation) for data exchange due to its human-readable format and universal language support.

2.8 Best Practices for Data Management

  • Use Standard Formats: FASTA, FASTQ, BAM, VCF for compatibility
  • Version Everything: Track versions of reference genomes, analysis tools, and data schemas
  • Validate Data Quality: Implement quality checks at each pipeline stage
  • Ensure Traceability: Maintain complete audit trails from sample to result
  • Implement Encryption: Protect patient data with AES-256 encryption
  • Use Checksums: SHA-256 hashes to verify data integrity
  • Backup Redundantly: Multiple geographic locations for critical data
  • Document Metadata: Who, what, when, where, why, and how for every dataset

Key Takeaways

  • Standardized data formats (FASTA, FASTQ, BAM, VCF) enable interoperability between different genome editing tools and platforms.
  • Guide RNA data structures must capture sequence, target location, design scores, and comprehensive off-target predictions for safety assessment.
  • Edit outcome data structures differ between technologies: CRISPR-Cas9 produces indel spectra, while base editors generate primarily desired base conversions with bystander editing.
  • Clinical-grade validation requires Q30 base quality (99.9% accuracy), 50,000x coverage depth, and off-target rates below 1% of on-target editing.
  • HL7 FHIR integration enables genome editing data to flow seamlessly into Electronic Health Records for clinical applications.
  • Manufacturing data for FDA-approved therapies must capture cell counts, editing efficiency, sterility, mycoplasma testing, and comprehensive quality control metrics.
  • Data management best practices include using standard formats, maintaining version control, ensuring traceability, implementing encryption, and comprehensive metadata documentation.
