제2장: 데이터 스키마

핵심 데이터 타입 및 구조

데이터 스키마

2.1 개요

음성-수화 변환 시스템은 각 단계가 다음 단계의 입력으로 사용되는 특정 데이터 타입을 생성하는 구조화된 데이터 파이프라인을 사용합니다. 이 장에서는 AudioInput, TranscriptionResult, SignGloss 및 SignPose의 핵심 데이터 스키마를 정의합니다.

2.2 AudioInput (오디오 입력)

AudioInput 스키마는 음성 인식 처리를 위한 오디오 데이터 및 관련 메타데이터를 캡처합니다.

interface AudioInput {
  audio_id: string;        // 고유 식별자
  timestamp: string;       // ISO 8601
  format: AudioFormat;     // 오디오 형식
  sample_rate: number;     // 샘플레이트 (Hz)
  channels: number;        // 채널 수
  duration_ms: number;     // 지속 시간 (밀리초)
  language_hint?: string;  // 언어 힌트
}

2.3 TranscriptionResult (전사 결과)

TranscriptionResult는 자동 음성 인식(ASR) 엔진의 출력을 포함하며, 단어 수준 타이밍 및 신뢰도 점수를 포함합니다.

interface TranscriptionResult {
  transcription_id: string;
  audio_id: string;
  language: string;         // 감지된 언어
  confidence: number;       // 신뢰도 (0.0-1.0)
  text: string;            // 전체 텍스트
  words: TranscribedWord[];
}

2.4 SignGloss (수화 글로스)

SignGloss는 텍스트와 물리적 포즈 사이의 중간 수화 표기법을 나타냅니다. 수화의 의미론적 및 구문적 구조를 캡처합니다.

interface SignGlossSequence {
  gloss_id: string;
  transcription_id: string;
  sign_language: SignLanguageCode;  // ASL, BSL, KSL 등
  glosses: SignGloss[];
  sentence_type: SentenceType;
  confidence: number;
}

2.5 SignPose (수화 포즈)

SignPose는 수화의 물리적 실행을 나타내는 3D 골격 애니메이션 데이터를 포함합니다. 이 데이터는 아바타 시스템에서 렌더링할 수 있습니다.

interface SignPoseSequence {
  pose_sequence_id: string;
  gloss_id: string;
  frame_rate: number;      // FPS (30, 60)
  total_frames: number;
  skeleton: SkeletonDefinition;
  frames: PoseFrame[];
}
