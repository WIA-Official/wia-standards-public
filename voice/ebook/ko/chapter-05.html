05.8 음성 데이터 보호 원칙

음성 데이터는 본질적으로 생체 식별 정보 (biometric identifier) 입니다. 표준은 다음 세 가지 보호 원칙을 모든 페이즈에 일관되게 박아 두었습니다.

  • 명시적 동의: 모든 음성 봉투는 사용자의 서명된 동의 봉투 ID (consent_envelope_id) 를 가리켜야 하며, 동의 범위 (전사, 화자 인식, TTS 클로닝 등) 가 일치하지 않으면 호스트는 처리를 거부합니다.
  • 모델 식별: 모든 봉투는 사용된 모델 식별자 (ASR 모델, TTS 음성, 화자 임베더) 를 포함해야 합니다. 모델이 갱신될 때 이전 결과를 재현할 수 있도록 보장합니다.
  • 감사 로그: 모든 외부 읽기 (다른 조직이 사용자 음성 데이터를 가져갈 때) 는 추가 전용 감사 로그에 기록되며, 사용자는 자신의 음성 데이터에 대한 모든 접근 이력을 조회할 수 있습니다.

05.9 한국 환경에서의 적용 노트

한국에서 WIA Voice 표준을 적용할 때 고려해야 할 두 가지 추가 요소가 있습니다. 첫째, 개인정보보호법상 생체 정보는 별도의 동의 절차가 필요하므로 동의 봉투의 한국어 표기를 명확히 해야 합니다. 둘째, 한국어 ASR / TTS 모델은 영어 모델 대비 학습 데이터가 적어 화자별 적응 (speaker adaptation) 의 효과가 더 큽니다. 표준은 화자별 적응 모델을 별도 객체로 정의하지 않지만, voice.voice_id 필드를 통해 화자별 음성 모델을 지정할 수 있습니다.

05.11 참고 표준

  • IETF RFC 8032 — Ed25519 (전자 서명)
  • IETF RFC 8446 — TLS 1.3 (전송 보안)

  • IETF RFC 9457 — Problem Details for HTTP APIs
  • ITU-T G.711 / G.722 — 음성 코덱 (전화품)
  • IETF RFC 6716 — Opus 코덱 (광대역)
  • W3C Web Speech API — 브라우저 음성 인터페이스
  • WIA-OMNI-API — 사용자 자격증 보관
  • WIA-ACCESSIBILITY — 음성 UI 접근성 프로파일
