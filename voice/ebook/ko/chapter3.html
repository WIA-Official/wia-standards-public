제3장: API 엔드포인트

오디오 입력, 전사, 수화 생성 및 아바타 제어

API 엔드포인트

3.1 개요

Voice-Sign API는 동기식 요청을 위한 RESTful 엔드포인트와 실시간 스트리밍을 위한 WebSocket 연결을 제공합니다. 모든 엔드포인트는 API 키 또는 OAuth 2.0 토큰을 통한 인증이 필요합니다.

기본 URL

https://api.wia.org/voice-sign/v1

3.2 번역 엔드포인트

오디오를 수화 표현으로 변환하는 주요 엔드포인트입니다.

POST /translate

Request:
{
  "audio": "<오디오 파일 또는 base64>",
  "target_language": "ASL",
  "output": {
    "include_transcript": true,
    "include_gloss": true,
    "include_pose": true
  }
}

Response:
{
  "transcript": { "text": "Hello, how are you?" },
  "gloss": { "glosses": [...] },
  "pose": { "frames": [...] }
}

3.3 오디오 처리 엔드포인트

오디오 업로드

POST /audio/upload

Response:
{
  "audio_id": "aud_789",
  "duration_ms": 2000,
  "status": "ready"
}

3.4 전사 엔드포인트

음성 인식

POST /transcribe

Request:
{
  "audio_id": "aud_789",
  "language": "ko",
  "word_timestamps": true
}

Response:
{
  "transcription_id": "tr_012",
  "text": "안녕하세요, 어떻게 지내세요?",
  "language": "ko",
  "confidence": 0.95,
  "words": [...]
}

3.5 수화 번역 엔드포인트

텍스트-수화 글로스

POST /sign/translate

Request:
{
  "text": "안녕하세요",
  "source_language": "ko",
  "target_language": "KSL"
}

Response:
{
  "sign_language": "KSL",
  "glosses": [
    {"gloss": "안녕", "sign_type": "lexical"}
  ]
}

3.6 스트리밍 API (WebSocket)

WebSocket: wss://api.wia.org/voice-sign/v1/stream

// 연결 헤더
Authorization: Bearer {token}

// 스트림 시작
{
  "type": "start",
  "config": {
    "target_language": "KSL",
    "audio_format": "pcm"
  }
}

// 오디오 청크 전송
{
  "type": "audio_chunk",
  "audio_data": "base64_encoded_audio"
}
