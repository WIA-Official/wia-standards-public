Chapter 3: API Endpoints

Audio Input, Transcription, Sign Generation, and Avatar Control

API Endpoints

3.1 Overview

The Voice-Sign API provides RESTful endpoints for synchronous requests and WebSocket connections for real-time streaming. All endpoints require authentication via API keys or OAuth 2.0 tokens.

Base URL

https://api.wia.org/voice-sign/v1

3.2 Translation Endpoint

The primary endpoint for converting audio to sign language representation.

Full Translation

POST /translate

Content-Type: multipart/form-data
Authorization: Bearer {token}

Request:
{
  "audio": <audio file or base64>,
  "target_language": "ASL",
  "output": {
    "include_transcript": true,
    "include_gloss": true,
    "include_pose": true,
    "include_render": false
  },
  "options": {
    "language_detection": true,
    "streaming": false
  }
}

Response (200 OK):
{
  "request_id": "req_123",
  "response_id": "res_456",
  "status": "success",
  "transcript": {
    "text": "Hello, how are you?",
    "language": "en",
    "confidence": 0.95,
    "words": [...]
  },
  "gloss": {
    "sign_language": "ASL",
    "glosses": [
      {"gloss": "HELLO", "start_ms": 0, "end_ms": 600},
      {"gloss": "HOW", "start_ms": 650, "end_ms": 1000},
      {"gloss": "YOU", "start_ms": 1050, "end_ms": 1400}
    ],
    "confidence": 0.91
  },
  "pose": {
    "frame_rate": 30,
    "total_frames": 60,
    "frames": [...]
  },
  "processing": {
    "total_time_ms": 450,
    "asr_time_ms": 180,
    "translation_time_ms": 150,
    "pose_generation_time_ms": 120
  }
}

3.3 Audio Processing Endpoints

Audio Upload

POST /audio/upload

Content-Type: multipart/form-data

Request:
- audio: (binary file)
- format: "wav"
- sample_rate: 16000
- language_hint: "en"

Response (201 Created):
{
  "audio_id": "aud_789",
  "duration_ms": 2000,
  "format": "wav",
  "sample_rate": 16000,
  "status": "ready"
}

Audio Preprocessing

POST /audio/{audio_id}/preprocess

Request:
{
  "noise_reduction": true,
  "voice_activity_detection": true,
  "normalization": true
}

Response (200 OK):
{
  "audio_id": "aud_789_processed",
  "processing_applied": ["noise_reduction", "vad", "normalization"],
  "speech_ratio": 0.75,
  "quality_score": 0.88
}

  "avatar_id": "avatar_default_asl",
  "resolution": {"width": 1920, "height": 1080},

  "format": "mp4",

  "render_id": "rnd_901",
  "status": "processing",
  "estimated_time_ms": 5000,
  "status_url": "/avatar/render/rnd_901/status"

Request:

3.7 Streaming API (WebSocket)

Connection

WebSocket: wss://api.wia.org/voice-sign/v1/stream

Connection Headers:
Authorization: Bearer {token}
Sec-WebSocket-Protocol: voice-sign-v1

Client Messages

// Start streaming
{
  "type": "start",
  "config": {
    "target_language": "ASL",
    "audio_format": "pcm",
    "sample_rate": 16000,
    "output_preferences": {
      "include_transcript": true,
      "include_gloss": true,
      "include_pose": true
    }
  }
}

// Audio chunk
{
  "type": "audio_chunk",
  "audio_data": "base64_encoded_audio",
  "sequence_number": 1
}

// Stop streaming
{
  "type": "stop"
}

Server Messages

// Transcript delta
{
  "type": "transcript",
  "transcript_delta": {
    "text": "Hello",
    "is_final": false,
    "confidence": 0.85
  },
  "latency_ms": 120
}

// Gloss delta
{
  "type": "gloss",
  "gloss_delta": {
    "gloss": {
      "gloss": "HELLO",
      "sign_type": "lexical",
      "confidence": 0.92
    },
    "is_final": true
  },
  "latency_ms": 150
}

// Pose frame
{
  "type": "pose",
  "pose_frame": {
    "frame_index": 0,
    "timestamp_ms": 0,
    "joints": [...]
  },
  "latency_ms": 180
}

// Error
{
  "type": "error",
  "error": {
    "code": "AUDIO_QUALITY_LOW",
    "message": "Audio quality is too low for accurate transcription"
  }
}

3.9 Rate Limiting

API endpoints are rate-limited to ensure fair usage and system stability.

Tier Requests/Minute Concurrent Streams Audio Minutes/Day
Free 10 1 60
Basic 60 3 500
Pro 300 10 5000
Enterprise Custom Custom Unlimited

Rate Limit Headers

X-RateLimit-Limit: 60
X-RateLimit-Remaining: 45
X-RateLimit-Reset: 1642262400

Response (429 Too Many Requests):
{
  "error": "rate_limit_exceeded",
  "message": "Rate limit exceeded. Please retry after 60 seconds.",
  "retry_after": 60
}

3.10 Error Responses

Status Code Error Description
400 Bad Request Invalid request format or parameters
401 Unauthorized Missing or invalid authentication
403 Forbidden Insufficient permissions
404 Not Found Resource not found
429 Too Many Requests Rate limit exceeded
500 Internal Server Error Server-side processing error
503 Service Unavailable Service temporarily unavailable
Best Practices
  • Implement exponential backoff for retry logic
  • Monitor rate limit headers to avoid throttling
  • Use streaming endpoints for real-time applications
  • Cache frequently requested translations
  • Compress audio before upload to reduce bandwidth
