1. WAF (Web Application Firewall)

WAF는 HTTP/HTTPS 트래픽을 검사하여 악의적인 요청을 차단합니다. DDoS뿐 아니라 SQL Injection, XSS 등도 방어합니다.

# ModSecurity (오픈소스 WAF) 설정 # Apache/Nginx에 통합 가능 # 1. 설치 (Ubuntu) apt-get install libapache2-mod-security2 # 2. 설정 파일 cp /etc/modsecurity/modsecurity.conf-recommended /etc/modsecurity/modsecurity.conf # 3. 규칙 활성화 SecRuleEngine On # 4. DDoS 방어 규칙 # HTTP Flood 방어 SecRule REMOTE_ADDR "@unconditionalMatch" \ "id:1001,phase:1,pass,nolog,setvar:ip.requests_per_minute=+1,expirevar:ip.requests_per_minute=60" SecRule IP:REQUESTS_PER_MINUTE "@gt 100" \ "id:1002,phase:1,deny,status:429,msg:'Rate limit exceeded'" # Slowloris 방어 SecRequestBodyNoFilesLimit 131072 # 128KB 제한 SecRequestBodyLimit 10485760 # 10MB 제한 SecRequestBodyLimitAction Reject # User-Agent 검증 (봇 차단) SecRule REQUEST_HEADERS:User-Agent "^$" \ "id:1003,phase:1,deny,status:403,msg:'Missing User-Agent'"

2. Rate Limiting Algorithms

2.1 Token Bucket

가장 널리 사용되는 Rate Limiting 알고리즘으로, 버스트 트래픽을 허용하면서도 장기 평균을 제한합니다.

# Token Bucket 원리 # 버킷에 토큰이 일정 속도로 추가됨 # 요청 시 토큰 1개 소비 # 토큰 없으면 요청 거부 import time class TokenBucket: def __init__(self, capacity, refill_rate): self.capacity = capacity # 최대 토큰 수 self.tokens = capacity # 현재 토큰 self.refill_rate = refill_rate # 초당 토큰 추가 수 self.last_refill = time.time() def refill(self): now = time.time() elapsed = now - self.last_refill tokens_to_add = elapsed * self.refill_rate self.tokens = min(self.capacity, self.tokens + tokens_to_add) self.last_refill = now def consume(self, tokens=1): self.refill() if self.tokens >= tokens: self.tokens -= tokens return True # 요청 허용 return False # 요청 거부 # 사용 예시 bucket = TokenBucket(capacity=100, refill_rate=10) # 최대 100개, 초당 10개 추가 for i in range(150): if bucket.consume(): print(f"Request {i}: Allowed") else: print(f"Request {i}: Rate limited") time.sleep(0.05) # 초당 20 요청

2.2 Leaky Bucket

# Leaky Bucket 원리 # 요청을 큐에 추가 # 일정 속도로 처리 (물이 새듯) from collections import deque import time class LeakyBucket: def __init__(self, capacity, leak_rate): self.capacity = capacity # 큐 최대 크기 self.leak_rate = leak_rate # 초당 처리 수 self.queue = deque() self.last_leak = time.time() def leak(self): now = time.time() elapsed = now - self.last_leak leaks = int(elapsed * self.leak_rate) for _ in range(min(leaks, len(self.queue))): self.queue.popleft() self.last_leak = now def add_request(self): self.leak() if len(self.queue) < self.capacity: self.queue.append(time.time()) return True return False # Nginx rate limiting (Leaky Bucket 기반) # nginx.conf http { limit_req_zone $binary_remote_addr zone=mylimit:10m rate=10r/s; server { location /api/ { limit_req zone=mylimit burst=20; # rate=10r/s: 초당 10 요청 # burst=20: 버스트 20개 허용 } } }

2.3 Sliding Window

# Sliding Window Rate Limiting (Redis) # 정확한 Rate Limiting, 시간 경계 문제 해결 import redis import time r = redis.Redis(host='localhost', port=6379) def rate_limit(user_id, limit=100, window=60): """ user_id: 사용자 식별자 limit: 윈도우당 최대 요청 수 window: 윈도우 크기 (초) """ key = f"rate_limit:{user_id}" now = time.time() # 1. 오래된 요청 제거 (윈도우 밖) r.zremrangebyscore(key, 0, now - window) # 2. 현재 요청 수 확인 current_requests = r.zcard(key) if current_requests < limit: # 3. 요청 추가 r.zadd(key, {now: now}) r.expire(key, window) return True # 허용 return False # 거부 # 사용 예시 for i in range(150): if rate_limit("user123", limit=100, window=60): print(f"Request {i}: Allowed") else: print(f"Request {i}: Rate limited") time.sleep(0.3) # 초당 3.3 요청

3. CAPTCHA & Challenge

3.1 Google reCAPTCHA v3

# 서버 측 검증 (Python) import requests def verify_recaptcha(token): secret_key = "YOUR_SECRET_KEY" response = requests.post('https://www.google.com/recaptcha/api/siteverify', data={ 'secret': secret_key, 'response': token }) result = response.json() # score: 0.0 (봇) ~ 1.0 (사람) if result['success'] and result['score'] >= 0.5: return True # 사람으로 판단 return False # 봇으로 판단

5. Connection Pooling & Queue
