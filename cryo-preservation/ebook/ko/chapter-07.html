7.1 경보 시스템 아키텍처
--------------------------------------------------------------------------------

견고한 경보 시스템은 보존된 대상자를 위협할 수 있는 비상 상황에 대한
첫 번째 방어선입니다.

[CRYO] 경보 레벨:

    +------------------------------------------------------------------------+
    | 레벨      | 트리거                       | 응답 시간  | 에스컬레이션 |
    +------------------------------------------------------------------------+
    | INFO      | 일상적 알림                  | 해당없음   | 없음         |
    | WARNING   | 파라미터가 한계에 접근       | 30분       | 당직자       |
    | CRITICAL  | 파라미터가 한계 초과         | 5분        | 관리자       |
    | EMERGENCY | 대상자에 대한 즉각적 위협    | 즉시       | 전 직원      |
    +------------------------------------------------------------------------+

[CRYO] 경보 트리거 조건:

    WARNING 레벨:
    +------------------------------------------------------------------+
    | - LN2 레벨 < 60%                                                 |
    | - 기준선에서 온도 편차 > 2 C                                     |
    | - 단일 센서 고장                                                 |
    | - 예정된 유지보수 지연                                           |
    | - 전원 변동 감지                                                 |
    +------------------------------------------------------------------+

    CRITICAL 레벨:
    +------------------------------------------------------------------+
    | - LN2 레벨 < 40%                                                 |
    | - 기준선에서 온도 편차 > 5 C                                     |
    | - 다중 센서 고장                                                 |
    | - 전원 손실 (UPS 활성)                                           |
    | - 무단 접근 시도                                                 |
    | - 통신 시스템 고장                                               |
    +------------------------------------------------------------------+

    EMERGENCY 레벨:
    +------------------------------------------------------------------+
    | - LN2 레벨 < 20%                                                 |
    | - 온도 > -180 C (가온 감지)                                      |
    | - 용기 파손 감지                                                 |
    | - 시설 내 화재 또는 침수                                         |
    | - 완전 정전                                                      |
    | - 시설 구조적 손상                                               |
    +------------------------------------------------------------------+

7.2 비상 대응 프로토콜
--------------------------------------------------------------------------------

사전 정의된 대응 프로토콜은 비상 시 일관되고 효과적인 조치를
보장합니다.

[CRYO] 프로토콜: LN2 고갈 비상

    트리거: LN2 레벨 < 20% 또는 고갈률이 < 4시간을 표시

    즉각 조치 (0-5분):
    +------------------------------------------------------------------+
    | 1. 경보가 오경보가 아닌지 확인 (중복 센서 확인)                  |
    | 2. 현장 비상 대응자 파견                                         |
    | 3. 비상 LN2 배송 활성화                                          |
    | 4. 시설 관리자 통보                                              |
    +------------------------------------------------------------------+

    단기 조치 (5-30분):
    +------------------------------------------------------------------+
    | 1. 백업 저장소에서 비상 LN2 이송 시작                            |
    | 2. 비상 충전을 위한 휴대용 듀어 준비                             |
    | 3. 영향받은 용기 평가 및 우선순위 지정                           |
    | 4. 배송 지연 시 상호 지원 시설 연락                              |
    +------------------------------------------------------------------+

    비상 대책 (LN2 불가 시):
    +------------------------------------------------------------------+
    | 1. 대체 용기로 대상자 이송 시작                                  |
    | 2. 드라이아이스를 임시 냉각으로 사용 (권장하지 않음)             |
    | 3. 지역 비상 네트워크 활성화                                     |
    | 4. 사고 보고서를 위한 모든 조치 문서화                           |
    +------------------------------------------------------------------+

[CRYO] 프로토콜: 온도 이탈

    트리거: 온도 > -180 C (안전 임계값 이상 가온)

    평가 (0-2분):
    +------------------------------------------------------------------+
    | 1. 영향받은 용기 식별                                            |
    | 2. LN2 레벨 확인 - 충전 문제인가?                                |
    | 3. 진공 게이지 확인 - 단열 손상인가?                             |
    | 4. 물리적 손상 또는 파손 확인                                    |
    +------------------------------------------------------------------+

    원인별 대응:

    LN2 고갈인 경우:
      -> LN2 고갈 비상 프로토콜 실행

    진공 손실인 경우:
    +------------------------------------------------------------------+
    | 1. 이것은 용기 고장 - 빠른 수리 불가                             |
    | 2. 즉시 대체 용기 준비                                           |
    | 3. 비상 대상자 이송 계획                                         |
    | 4. 이송 준비 중 최대 LN2 충전 유지                               |
    +------------------------------------------------------------------+

    물리적 파손인 경우:
    +------------------------------------------------------------------+
    | 1. 위험 존재 시 접근 금지                                        |
    | 2. 구역 확보 및 안전 평가                                        |
    | 3. 안전한 경우 대상자 비상 이송                                  |
    | 4. 구조적 손상 시 응급 서비스 연락                               |
    +------------------------------------------------------------------+

[CRYO] 프로토콜: 시설 대피

    트리거: 화재, 침수, 구조적 손상, 또는 외부 위협

    1단계: 인원 안전 (즉시)
    +------------------------------------------------------------------+
    | 1. 대피 경보 발령                                                |
    | 2. 모든 인원 지정 경로로 대피                                    |
    | 3. 집합 장소에서 모든 직원 확인                                  |
    | 4. 응급 서비스 연락                                              |
    +------------------------------------------------------------------+

    2단계: 대상자 보호 (안전한 경우에만)
    +------------------------------------------------------------------+
    | 1. 가능하면 원격으로 용기 상태 확인                              |
    | 2. 응급 서비스 허가 없이 시설 재진입 금지                        |
    | 3. 잠재적 이송을 위한 상호 지원 협정 활성화                      |
    | 4. 안전한 위치에서 용기 텔레메트리 모니터링                      |
    +------------------------------------------------------------------+

    3단계: 복구
    +------------------------------------------------------------------+
    | 1. 허가 시 시설 손상 평가                                        |
    | 2. 모든 용기 무결성 확인                                         |
    | 3. 시설 손상 시 이송 실행                                        |
    | 4. 사고 문서화 완료                                              |
    +------------------------------------------------------------------+

7.3 시설간 이송 프로토콜
--------------------------------------------------------------------------------

보존된 대상자를 시설 간에 이송하는 것은 고위험이며 신중한 계획과
실행이 필요합니다.

[CRYO] 이송 결정 기준:

    이송해야 할 경우:
    +------------------------------------------------------------------+
    | - 시설 폐쇄 또는 이전                                            |
    | - 비상 대피 요구                                                 |
    | - 대상자 요청 (법적으로 해당되는 경우)                           |
    | - 더 나은 시설로 품질 업그레이드                                 |
    | - 연구 프로토콜 요구                                             |
    +------------------------------------------------------------------+

    이송하지 말아야 할 경우:
    +------------------------------------------------------------------+
    | - 편의 (위험이 이점 초과)                                        |
    | - 극한 기상 조건 중                                              |
    | - 수신 시설 준비 확인 없이                                       |
    | - 적절한 운송 장비 없이                                          |
    | - 완전한 문서화 체인 없이                                        |
    +------------------------------------------------------------------+

7.4 운송 장비 사양
--------------------------------------------------------------------------------

보존된 대상자의 안전한 운송에는 특수 장비가 필요합니다.

[CRYO] 운송 용기 요구사항:

    +------------------------------------------------------------------------+
    | 사양                 | 요구사항                                        |
    +------------------------------------------------------------------------+
    | 정적 유지 시간       | 최소 72시간 (120+ 권장)                         |
    | LN2 용량             | 유지 시간 + 50% 여유분 충분                     |
    | 온도 모니터링        | 셀룰러/위성 업링크로 실시간                     |
    | 충격 보호            | 차량 운송 등급                                  |
    | 유출 억제            | 전체 LN2 용량 억제                              |
    | 접근 보안            | 변조 방지 봉인                                  |
    +------------------------------------------------------------------------+

[CRYO] 차량 요구사항:

    육상 운송:
    +------------------------------------------------------------------+
    | - 기후 제어 화물 공간                                            |
    | - LN2 가스 배출을 위한 환기                                      |
    | - 화물 공간 산소 모니터링                                        |
    | - 지오펜싱 경보가 있는 GPS 추적                                  |
    | - 시설과 직접 통신                                               |
    | - 모니터링 장비용 백업 발전기                                    |
    +------------------------------------------------------------------+

    항공 운송:
    +------------------------------------------------------------------+
    | - 가압 화물칸 (LN2 호환)                                         |
    | - LN2에 대한 항공사 위험물 인증                                  |
    | - 온도 제어 또는 단열 화물칸                                     |
    | - 우선 처리 준비                                                 |
    | - 국제 운송의 경우 세관 사전 통관                                |
    +------------------------------------------------------------------+

[i] 운송 모니터링:

    class TransportMonitor:
        def __init__(self, transfer_id, subject_id):
            self.transfer_id = transfer_id
            self.subject_id = subject_id
            self.readings = []
            self.alerts = []

        def record_reading(self, timestamp, data):
            reading = {
                'timestamp': timestamp,
                'temperature': data['temperature'],
                'ln2_level': data['ln2_level'],
                'location': data['gps_coords'],
                'shock_events': data['accelerometer_events']
            }
            self.readings.append(reading)

            # 경보 조건 확인
            self.check_alerts(reading)

            # 모니터링 시스템으로 전송
            self.transmit_reading(reading)

        def check_alerts(self, reading):
            if reading['temperature'] > -180:
                self.raise_alert('TEMPERATURE_HIGH', reading)

            if reading['ln2_level'] < 40:
                self.raise_alert('LN2_LOW', reading)

            if reading['shock_events'] > 0:
                self.raise_alert('SHOCK_DETECTED', reading)

7.6 상호 지원 협정
--------------------------------------------------------------------------------

시설은 비상 지원을 위해 다른 냉동보존 시설과 상호 지원 협정을
체결해야 합니다.

[CRYO] 상호 지원 프레임워크:

    협정 구성요소:
    +------------------------------------------------------------------+
    | 1. 비상 LN2 공급                                                 |
    |    - 4시간 내 LN2 제공 약속                                      |
    |    - 최소 수량 보장                                              |
    |    - 비용 분담 준비                                              |

    | 2. 비상 저장                                                     |
    |    - 비상 이송을 위한 예약 슬롯                                  |

    |    - 임시 vs 영구 저장 조건                                      |
    +------------------------------------------------------------------+
    | 3. 인력 지원                                                     |
    |    - 비상 대응에 가용한 기술 직원                                |
    |    - 운송 팀 지원                                                |
    |    - 장비 대여                                                   |
    +------------------------------------------------------------------+
    | 4. 정보 공유                                                     |
    |    - 사고 알림                                                   |
    |    - 모범 사례 공유                                              |
    |    - 합동 훈련 연습                                              |
    +------------------------------------------------------------------+
