5.2 액체 질소 관리
--------------------------------------------------------------------------------

LN2 레벨 관리는 장기 저장의 가장 중요한 측면입니다.
증발은 지속적이며, 모니터링은 연속적이어야 합니다.

[CRYO] LN2 레벨 구간:

    100% |============| 최대 용량
         |            |
     80% |============| 최적 - 정상 운영 범위
         |            |
     60% |============| 주의 - 충전 예약
         |            |
     40% |============| 경고 - 긴급 충전 필요
         |            |
     20% |============| 위험 - 비상 프로토콜
         |            |
      0% |____________| 재앙 - 대상자 위험

[!] 경보 임계값:

    +------------------------------------------------------------------------+
    | 레벨      | 상태         | 필요 조치                                   |
    +------------------------------------------------------------------------+
    | > 80%     | 최적         | 정상 모니터링                               |
    | 60-80%    | 주의         | 48시간 내 충전 예약                         |
    | 40-60%    | 경고         | 즉시 충전 예약                              |
    | 20-40%    | 위험         | 비상 충전, 백업 활성화                      |
    | < 20%     | 비상         | 대상자 대피, 전체 비상 프로토콜             |
    +------------------------------------------------------------------------+

[CRYO] LN2 소비 예측:

    class LN2Predictor:
        def __init__(self, container):
            self.container = container
            self.base_rate = container.specs.static_evaporation_rate
            self.history = []

        def predict_depletion(self, current_level_pct):
            """위험 레벨(20%)까지 시간 예측"""
            current_volume = self.container.capacity * (current_level_pct / 100)
            critical_volume = self.container.capacity * 0.20

            available = current_volume - critical_volume

            # 환경 요인 조정
            adjusted_rate = self.base_rate * self.get_rate_multiplier()

            days_until_critical = available / adjusted_rate

            return {
                'current_level_pct': current_level_pct,
                'current_volume_L': current_volume,
                'daily_consumption_L': adjusted_rate,
                'days_until_critical': days_until_critical,
                'recommended_refill_date': self.calculate_refill_date(
                    days_until_critical
                )
            }

        def get_rate_multiplier(self):
            """환경 조정 요인"""
            multiplier = 1.0

            # 주변 온도 조정
            ambient_temp = self.get_ambient_temp()
            if ambient_temp > 25:
                multiplier += 0.1 * (ambient_temp - 25) / 5

            # 접근 빈도 조정
            daily_accesses = self.get_access_frequency()
            multiplier += 0.05 * daily_accesses

            # 듀어 노후화 조정
            years_old = self.container.age_years
            if years_old > 10:
                multiplier += 0.02 * (years_old - 10)

            return multiplier

여러 중복 레이어로 24/7 작동해야 합니다.

5.4 슬롯 할당 및 추적
--------------------------------------------------------------------------------

각 대상자는 용기 내 특정 슬롯에 할당되어야 하며, 저장 기간 동안
위치의 정밀한 추적이 필요합니다.

[CRYO] 슬롯 주소 스키마:

    형식: [DEWAR_ID]-[레벨]-[위치]

    예: DEWAR-A-001-L3-P05
      |-- DEWAR-A-001: 용기 식별자
      |-- L3: 레벨 3 (수직 위치)
      +-- P05: 위치 5 (방사형 위치)

    용기 슬롯의 시각적 표현:

    평면도 (레벨당):

              P01
           P08    P02
        P07    *    P03
           P06    P04
              P05

    * = 중앙 코어 (해당하는 경우)

    측면도 (레벨):

    +------ 뚜껑 ------+
    |   L1: 상단       |
    |   L2             |
    |   L3             |
    |   L4             |
    |   L5: 하단       |
    +-----------------+

[!] 슬롯 관리 규칙:

    1. 이중 할당 없음 - 슬롯당 하나의 대상자
    2. 모든 슬롯 변경을 전체 감사 추적과 함께 기록
    3. 대상자 유형과 슬롯 호환성 확인
    4. 슬롯 선택 시 온도 구배 고려
    5. 안정성을 위한 용기 로딩 균형
