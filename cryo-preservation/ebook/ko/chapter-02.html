2.1 대상자 식별 스키마
--------------------------------------------------------------------------------

모든 냉동보존 대상자는 보존 기간 동안 지속되는 고유하고 전역적으로
검증 가능한 식별자가 필요합니다.

[CRYO] 대상자 문서 스키마:

    {
      "@context": [
        "https://www.w3.org/2018/credentials/v1",
        "https://wia.org/cryo-preservation/v1"
      ],
      "type": ["VerifiableCredential", "CryoSubjectCredential"],
      "id": "urn:uuid:cryo:subject:a1b2c3d4-e5f6-7890-abcd-ef1234567890",

      "credentialSubject": {
        "subjectId": "CRYO-2025-001-WB-001",
        "subjectType": "WHOLE_BODY",
        "species": "HOMO_SAPIENS",

        "identifier": {
          "type": "FACILITY_ASSIGNED",
          "value": "ACME-CRYO-2025-00001",
          "issuer": "urn:facility:acme-cryonics"
        },

        "demographics": {
          "birthYear": 1960,
          "biologicalSex": "MALE",
          "bloodType": "A_POSITIVE"
        },

        "biometrics": {
          "heightCm": 175,
          "weightKg": 70,
          "bodyMassIndex": 22.9
        }
      },

      "preservationStatus": {
        "currentState": "VITRIFIED",
        "containerAssignment": "DEWAR-A-001-SLOT-05",
        "temperatureCelsius": -196,
        "preservationDate": "2025-03-15T14:30:00Z"
      }
    }

[i] 대상자 ID 형식:
    CRYO-[연도]-[시설]-[유형]-[순번]

    예시:
    - CRYO-2025-001-WB-001  (전신, 시설 001)
    - CRYO-2025-042-NR-015  (신경, 시설 042)
    - CRYO-2025-007-EM-203  (배아, 시설 007)


--------------------------------------------------------------------------------
2.2 동의 및 법적 문서화
--------------------------------------------------------------------------------

냉동보존은 강력한 동의 문서화가 필요하며, 특히 여러 세대에 걸쳐
저장될 수 있는 인간 대상자의 경우 더욱 그렇습니다.

[CRYO] 동의 자격증명 구조:

    ConsentDocument
    +------------------------------------------------------------------+
    | consentId         | 고유 동의 문서 식별자                        |
    | consentType       | PATIENT | NEXT_OF_KIN | LEGAL_GUARDIAN       |
    | consentDate       | ISO 8601 타임스탬프                          |
    | consentScope      | 승인된 절차 배열                             |
    +------------------------------------------------------------------+
    | signatories       | SignatoryRecord 배열                         |
    |   - name          | 법적 성명                                    |
    |   - relationship  | SELF | SPOUSE | CHILD | LEGAL_REP            |
    |   - signature     | 암호화 서명                                  |
    |   - witnessed     | 증인 증명                                    |
    +------------------------------------------------------------------+
    | revivalConsent    | RevivalConsentRecord                         |
    |   - authorized    | 불리언                                       |
    |   - conditions    | 소생 조건 배열                               |
    |   - decisionMaker | 소생 결정권자                                |
    +------------------------------------------------------------------+

[!] 법적 요구사항:
    - 동의는 보존 시작 전에 획득해야 함
    - 인간 대상자의 경우 복수 증인 필요
    - 소생 결정을 위한 법적 대리인 지정
    - 장기 저장의 경우 정기적 동의 갱신 (대상자 가능시)

2.3 병력 기록
--------------------------------------------------------------------------------

완전한 의료 문서화는 최적의 보존 프로토콜을 보장하고 잠재적 소생
절차를 준비합니다.

[CRYO] 병력 기록 스키마:

    MedicalHistory

    |     |-- icd10Code: "I21.9"
    |     |-- description: "급성 심근경색"
    |     |-- timeOfLegalDeath: "2025-03-15T14:00:00Z"
    |     +-- pronouncingPhysician: "김의사, MD"
    |
    +-- preExistingConditions[]
    |     |-- condition: "제2형 당뇨병"
    |     |-- icd10Code: "E11"
    |     |-- diagnosisDate: "2010-05-20"
    |     +-- managementNotes: "메트포르민으로 조절"
    |
    +-- medications[]
    |     |-- name: "메트포르민"
    |     |-- dosage: "500mg 1일 2회"
    |     +-- lastAdministered: "2025-03-15T08:00:00Z"
    |
    +-- allergies[]
    |     |-- allergen: "페니실린"
    |     |-- severity: "SEVERE"
    |     +-- reaction: "아나필락시스"
    |
    +-- surgicalHistory[]
    |     |-- procedure: "충수절제술"
    |     |-- date: "1995-08-12"
    |     +-- notes: "합병증 없음"
    |
    +-- geneticMarkers
          |-- bloodType: "A_POSITIVE"
          |-- hlaType: "A*02:01, B*07:02, DRB1*15:01"

[i] 데이터 개인정보 보호:
    병력 기록은 대상자별 키로 암호화됩니다.
    접근에는 명시적 권한 자격증명이 필요합니다.
    선택적 공개로 특정 필드만 공유할 수 있습니다.

2.4 보존 상태 기록
--------------------------------------------------------------------------------

상태 기록은 보존된 대상자의 현재 상태와 전체 이력을 추적합니다.

[CRYO] 상태 열거:

    +------------------------------------------------------------------------+
    | 상태              | 설명                                               |
    +------------------------------------------------------------------------+
    | PENDING           | 보존 절차 대기 중                                  |
    | STANDBY           | 활성 대기, 보존 준비 완료                          |
    | COOLING           | 온도 감소 진행 중                                  |
    | VITRIFYING        | 유리화 과정 진행 중                                |
    | VITRIFIED         | 유리화 성공, 안정적 저장 중                        |
    | TRANSFERRING      | 용기/시설 간 이동 중                               |
    | COMPROMISED       | 품질 저하 감지                                     |
    | WARMING           | 소생 가온 과정 시작                                |
    | REVIVED           | 성공적 소생                                        |
    | TERMINATED        | 보존 종료 (법적/동의)                              |
    +------------------------------------------------------------------------+

[CRYO] 상태 이력 기록:

    StatusTransition

      "parameters": {
        "temperatureAtTransition": -40,
        "coolingRateAchieved": -1.2,
        "cpaConcentration": 7.2
      },
      "verificationSignature": "0x7f8e9a..."
    }

2.5 보관연쇄 문서화
--------------------------------------------------------------------------------

보존된 대상자와의 모든 상호작용은 불변의 보관연쇄 기록에
문서화되어야 합니다.

[CRYO] 보관 이벤트 유형:

    INITIAL_INTAKE
      |-- 시설에서 대상자 접수
      |-- 초기 문서화 확인
      +-- 보존 프로토콜 할당

    PROCEDURE_START
      |-- 특정 절차 시작
      |-- 운영자 자격증명 확인
      +-- 장비 교정 확인

    CONTAINER_ASSIGNMENT
      |-- 대상자를 저장 용기에 배치
      |-- 위치 좌표 기록
      +-- 모니터링 센서 활성화

    INSPECTION
      |-- 정기 또는 비상 검사
      |-- 시각/기기 평가
      +-- 결과 문서화

    TRANSFER_OUT
      |-- 대상자가 현재 보관을 떠남
      |-- 목적지 시설 확인
      +-- 운송 조건 지정

    TRANSFER_IN
      |-- 이송에서 대상자 도착
      |-- 상태 평가
      +-- 새 용기 할당

    REVIVAL_INITIATION
      |-- 소생 과정 승인
      |-- 의료팀 구성
      +-- 프로토콜 선택

[CRYO] 보관연쇄 예시:

    대상자: CRYO-2025-001-WB-001

    이벤트 1: INITIAL_INTAKE
    +------------------------------------------------------------------+
    | 타임스탬프: 2025-03-15T14:30:00Z                                 |
    | 시설: ACME 크라이오닉스, 서울                                    |
    | 운영자: 김박사 (ID: OP-2025-0042)                                |
    | 증인: 이간호사 (ID: OP-2025-0089)                                |
    | 비고: 대기팀 통해 대상자 도착, 양호한 상태                       |
    | 서명: 0x8a9b7c6d...                                              |
    +------------------------------------------------------------------+

    이벤트 2: PROCEDURE_START
    +------------------------------------------------------------------+
    | 타임스탬프: 2025-03-15T14:45:00Z                                 |
    | 절차: 전신 유리화 프로토콜 WB-V3.2                               |
    | 운영자: 김박사 (ID: OP-2025-0042)                                |
    | 장비: 관류 시스템 PS-100 (교정: 2025-03-01)                      |
    | CPA 용액: M22 유리화 용액, 배치 VIT-2025-0156                    |
    | 서명: 0x3e4f5a6b...                                              |

2.6 자격증명 검증
--------------------------------------------------------------------------------

모든 자격증명은 표준 DID(탈중앙화 식별자) 해석 및 암호화 서명
검증을 통해 검증 가능해야 합니다.

[i] 검증 흐름:

    +----------------+     +------------------+     +----------------+
    |  자격증명      | --> |  DID 해석기      | --> |  공개키        |
    |  (서명 포함)   |     |  did:wia:cryo:.. |     |  조회          |
    +----------------+     +------------------+     +----------------+
                                                            |
                                                            v
    +----------------+     +------------------+     +----------------+
    |  검증됨        | <-- |  서명            | <-- |  서명 검증     |
    |  자격증명      |     |  유효/무효       |     |  키 대비       |
    +----------------+     +------------------+     +----------------+

[CRYO] 검증 코드 예시:

    async function verifySubjectCredential(credential) {
      // 1. 발급자 DID 추출
      const issuerDid = credential.issuer;

      // 2. DID 문서 해석
      const didDocument = await didResolver.resolve(issuerDid);

      // 3. 검증 방법 획득
      const verificationMethod = didDocument.verificationMethod[0];

      // 4. 서명 검증
      const isValid = await verifySignature(
        credential,
        credential.proof.jws,
        verificationMethod.publicKeyJwk
      );

      // 5. 폐지 상태 확인
      const isRevoked = await checkRevocationRegistry(credential.id);

      return {
        signatureValid: isValid,
        notRevoked: !isRevoked,
        issuerTrusted: await checkTrustRegistry(issuerDid)
      };
    }
