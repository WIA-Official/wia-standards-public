5.2 LIQUID NITROGEN MANAGEMENT
--------------------------------------------------------------------------------

LN2 level management is the most critical aspect of long-term storage.
Evaporation is constant; monitoring must be continuous.

[CRYO] LN2 LEVEL ZONES:

    100% |============| Full capacity
         |            |
     80% |============| OPTIMAL - Normal operating range
         |            |
     60% |============| ATTENTION - Schedule refill
         |            |
     40% |============| WARNING - Urgent refill needed
         |            |
     20% |============| CRITICAL - Emergency protocol
         |            |
      0% |____________| CATASTROPHIC - Subjects at risk

[!] ALERT THRESHOLDS:

    +------------------------------------------------------------------------+
    | Level     | Status       | Action Required                            |
    +------------------------------------------------------------------------+
    | > 80%     | OPTIMAL      | Normal monitoring                          |
    | 60-80%    | ATTENTION    | Schedule refill within 48 hours            |
    | 40-60%    | WARNING      | Immediate refill scheduling                |
    | 20-40%    | CRITICAL     | Emergency refill, activate backup          |
    | < 20%     | EMERGENCY    | Evacuate subjects, full emergency protocol |
    +------------------------------------------------------------------------+

[CRYO] LN2 CONSUMPTION PREDICTION:

    class LN2Predictor:
        def __init__(self, container):
            self.container = container
            self.base_rate = container.specs.static_evaporation_rate
            self.history = []

        def predict_depletion(self, current_level_pct):
            """Predict time until critical level (20%)"""
            current_volume = self.container.capacity * (current_level_pct / 100)
            critical_volume = self.container.capacity * 0.20

            available = current_volume - critical_volume

            # Adjust for environmental factors
            adjusted_rate = self.base_rate * self.get_rate_multiplier()

            days_until_critical = available / adjusted_rate

            return {
                'current_level_pct': current_level_pct,
                'current_volume_L': current_volume,
                'daily_consumption_L': adjusted_rate,
                'days_until_critical': days_until_critical,
                'recommended_refill_date': self.calculate_refill_date(
                    days_until_critical
                )
            }

        def get_rate_multiplier(self):
            """Environmental adjustment factors"""
            multiplier = 1.0

            # Ambient temperature adjustment
            ambient_temp = self.get_ambient_temp()
            if ambient_temp > 25:
                multiplier += 0.1 * (ambient_temp - 25) / 5

            # Access frequency adjustment
            daily_accesses = self.get_access_frequency()
            multiplier += 0.05 * daily_accesses

            # Age of dewar adjustment
            years_old = self.container.age_years
            if years_old > 10:
                multiplier += 0.02 * (years_old - 10)

            return multiplier

5.3 CONTINUOUS MONITORING REQUIREMENTS
--------------------------------------------------------------------------------

Automated monitoring systems must operate 24/7 with multiple redundancy
layers to ensure no failure mode can go undetected.

[CRYO] MONITORING PARAMETERS:

    +------------------------------------------------------------------------+
    | Parameter        | Frequency      | Method               | Redundancy |
    +------------------------------------------------------------------------+
    | LN2 Level        | 15 minutes     | Capacitance probe    | 2 sensors  |
    | Temperature      | 1 minute       | PT100 RTD            | 3 sensors  |
    | Vacuum pressure  | 1 hour         | Ion gauge            | 1 sensor   |
    | Ambient temp     | 5 minutes      | Thermocouple         | 2 sensors  |
    | Power status     | Continuous     | UPS monitoring       | Automatic  |
    | Access events    | On occurrence  | Door/lid sensors     | Logged     |
    +------------------------------------------------------------------------+

[CRYO] MONITORING SYSTEM ARCHITECTURE:

    +------------------+     +------------------+     +------------------+
    |  SENSOR LAYER    |     |  AGGREGATION     |     |  ANALYSIS        |
    +------------------+     +------------------+     +------------------+
    | PT100 Temp (x3)  | --> | Local Controller | --> | Trend Analysis   |
    | LN2 Level (x2)   | --> | Data Logging     | --> | Anomaly Detection|
    | Vacuum Gauge     | --> | Alert Generation | --> | Prediction       |
    | Door Sensors     | --> | Communication    | --> | Reporting        |
    +------------------+     +------------------+     +------------------+
            |                        |                        |
            v                        v                        v
    +------------------+     +------------------+     +------------------+
    |  POWER BACKUP    |     |  COMMUNICATION   |     |  ALERTING        |
    +------------------+     +------------------+     +------------------+
    | UPS (4 hours)    |     | Primary: Ethernet|     | SMS/Phone        |
    | Generator backup |     | Backup: Cellular |     | Email            |
    | Solar (optional) |     | Failover: Radio  |     | On-site alarms   |
    +------------------+     +------------------+     +------------------+

[i] DATA RETENTION REQUIREMENTS:

    Real-time data (1-minute resolution):    30 days
    Hourly aggregated data:                  1 year
    Daily summary data:                      10 years
    Event/alert logs:                        Indefinitely
    Access logs:                             Indefinitely

5.4 SLOT ASSIGNMENT AND TRACKING
--------------------------------------------------------------------------------

Each subject must be assigned to a specific slot within a container,
with precise tracking of position throughout storage.

[CRYO] SLOT ADDRESSING SCHEMA:

    Format: [DEWAR_ID]-[LEVEL]-[POSITION]

    Example: DEWAR-A-001-L3-P05
      |-- DEWAR-A-001: Container identifier
      |-- L3: Level 3 (vertical position)
      +-- P05: Position 5 (radial position)

    Visual representation of container slots:

    Top View (per level):

              P01
           P08    P02
        P07    *    P03
           P06    P04
              P05

    * = Central core (if applicable)

    Side View (levels):

    +------ Lid ------+
    |   L1: Top       |
    |   L2            |
    |   L3            |
    |   L4            |
    |   L5: Bottom    |
    +-----------------+

5.6 SECURITY AND ACCESS CONTROL
--------------------------------------------------------------------------------

Physical and digital security protects subjects from unauthorized access,
tampering, and environmental threats.

[CRYO] ACCESS CONTROL LAYERS:

    LAYER 1: Facility Perimeter
    +------------------------------------------------------------------+
    | - Fenced compound with controlled entry points                   |
    | - 24/7 security monitoring                                       |
    | - Vehicle barriers at entry                                      |
    +------------------------------------------------------------------+

    LAYER 2: Building Access
    +------------------------------------------------------------------+
    | - Biometric + badge authentication                               |
    | - Mantrap entry (dual door system)                              |
    | - Visitor logging and escort requirements                        |
    +------------------------------------------------------------------+

    LAYER 3: Storage Vault Access
    +------------------------------------------------------------------+
    | - Two-person rule for vault entry                               |
    | - Additional biometric verification                              |
    | - Real-time video monitoring                                     |
    | - All access logged with credential verification                 |
    +------------------------------------------------------------------+

    LAYER 4: Container Access
    +------------------------------------------------------------------+
    | - Individual container alarms                                    |
    | - Lid opening triggers immediate alert                          |
    | - Access requires work order authorization                       |
    | - Photo/video documentation of all access                        |
    +------------------------------------------------------------------+

[i] DIGITAL SECURITY:

    - Encrypted data transmission (TLS 1.3 minimum)
    - Air-gapped backup systems
    - Regular security audits
    - Intrusion detection systems
    - Blockchain-anchored audit logs
