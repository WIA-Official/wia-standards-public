# 제6장: 프로토콜 사양

## 학습 목표

이 장을 마친 후 다음을 할 수 있게 됩니다:

1. 품질 보증 및 품질 관리(QA/QC) 절차 구현
2. 환경 센서에 대한 보정 프로토콜 따르기
3. 표준화된 현장 샘플링 방법 적용
4. 데이터 관리 및 검증 워크플로우 문서화
5. 실험실 및 관찰자에 대한 숙련도 테스트 수행

## 6.1 품질 보증 개요

### 6.1.1 QA vs. QC

```

┌─────────────────────────────────────────────────────────────────────┐
│                                                                     │
│  품질 보증(QA) - 계획된 체계적 활동                                    │
│  ├─ 표준 운영 절차(SOP)                                              │
│  ├─ 교육 및 인증 프로그램                                            │
│  ├─ 장비 보정 일정                                                   │
│  ├─ 데이터 관리 계획                                                 │
│  └─ 감사 및 검토 프로세스                                            │
│                                                                     │
│  품질 관리(QC) - 문제를 감지하는 특정 기술                             │
│  ├─ 현장 블랭크 및 복제본                                            │
│  ├─ 실험실 중복 및 스파이크                                          │
│  ├─ 자동화된 데이터 검증                                             │
│  ├─ 이상치 감지                                                     │
│  └─ 교차 매개변수 일관성 검사                                        │
│                                                                     │
│  목표: 방어 가능한 출판 품질 데이터 생성                               │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 6.1.2 데이터 유형별 QA/QC 요구사항

| 데이터 유형 | 현장 QC | 실험실 QC | 자동 검증 |
|-----------|---------|---------|---------|
| 종 관찰 | 10% 현장 복제본 | 전문가 검증 | 분류학적, 공간적, 시간적 |
| 수질 | 5% 현장 블랭크, 10% 복제본 | 중복, 스파이크, 블랭크 | 범위, 변화율 |
| 대기질 | 공동 배치 테스트 | 보정 검증 | 플랫라인, 스파이크 감지 |
| 센서 데이터 | 월별 보정 | 연간 공장 보정 | 다중 매개변수 일관성 |

## 6.2 현장 품질 관리

### 6.2.1 현장 QC 요구사항

**현장 블랭크 (샘플의 5%):**
- 목적: 수집, 운송 또는 처리 중 오염 감지
- 절차: 현장 조건에 노출된 깨끗한 물/용액
- 수용: 분석물 농도 < 검출 한계

**장비 블랭크:**
- 목적: 세척 절차 검증
- 절차: 샘플링 장비를 통해 깨끗한 물 실행
- 빈도: 각 샘플링 이벤트 전후
- 수용: < 검출 한계

**현장 복제본 (샘플의 10%):**
- 목적: 샘플링 정밀도 및 자연적 변동성 평가
- 절차: 같은 위치/시간에서 2-3개의 샘플 수집
- 수용: 상대 백분율 차이(RPD) < 20%

```
RPD = |샘플1 - 샘플2| / ((샘플1 + 샘플2) / 2) × 100%
```

**양성 대조군:**
- 목적: 탐지 방법이 작동하는지 확인
- 절차: 대상 유기체/분석물이 있는 알려진 샘플
- 예: 종 ID용 참조 표본, 화학용 표준 용액
- 수용: 정확한 식별 또는 회수율 80-120%

## 6.3 실험실 품질 관리

### 6.3.1 실험실 QC 요구사항

**방법 블랭크 (각 분석 배치):**
- 목적: 실험실 오염 감지
- 절차: 전체 분석 프로세스를 통한 깨끗한 매트릭스
- 수용: < 검출 한계

**보정 검증 (샘플 10개마다):**
- 목적: 보정 곡선이 유효한지 확인
- 절차: 중간점 보정 표준 분석
- 수용: 회수율 90-110%

**스파이크 회수 (샘플의 10%):**
- 목적: 매트릭스 간섭 및 회수 효율 평가
- 절차: 분석 전 샘플에 알려진 양의 분석물 추가
- 계산: `회수율 = (스파이크된 샘플 - 스파이크되지 않은 샘플) / 스파이크 양 × 100%`
- 수용: 75-125%

**중복 분석 (샘플의 10%):**
- 목적: 분석 정밀도 평가
- 절차: 동일한 샘플을 두 번 분석
- 수용: RPD < 20%

**인증 표준 물질 (각 배치):**
- 목적: 방법 정확도 검증
- 절차: 알려진 농도의 인증 표준 분석
- 수용: 인증 값의 ±2 표준편차 이내

**블라인드 QC 샘플 (샘플의 5%):**
- 목적: 실험실 성능의 편향되지 않은 평가
- 절차: 알려진 샘플을 미지의 것처럼 제출
- 수용: 수용 기준 이내

**변화율 감지:**
```typescript
function detectRateOfChange(readings) {
  for (let i = 1; i < readings.length; i++) {
    const dt = readings[i].timestamp - readings[i-1].timestamp; // 초
    const dv = readings[i].value - readings[i-1].value;
    const rate = Math.abs(dv / (dt / 3600)); // 시간당 변화

    if (rate > MAX_CHANGE_RATE) {
      readings[i].qc_flag = 'questionable';
      readings[i].qc_notes = `변화율 ${rate.toFixed(2)}/hr이 최대 ${MAX_CHANGE_RATE}/hr을 초과`;
    }
  }
}

// 예제 임계값
const MAX_CHANGE_RATES = {
  air_temperature: 10, // °C/hr
  water_temperature: 5, // °C/hr
  water_level: 50, // cm/hr (홍수 제외)
};
```

**플랫라인 감지:**
```typescript
function detectFlatline(readings, minVariance = 0.001) {
  const window = 24; // 24개의 연속 판독값 확인
  for (let i = 0; i < readings.length - window; i++) {
    const segment = readings.slice(i, i + window);
    const values = segment.map(r => r.value);
    const variance = calculateVariance(values);

    if (variance < minVariance) {
      segment.forEach(r => {
        r.qc_flag = 'questionable';
        r.qc_notes = '플랫라인 감지 - 센서가 정지했을 수 있음';
      });
    }
  }
}
```

**스파이크 감지:**
```typescript
function detectSpikes(readings, threshold = 4.0) {
  // 견고성을 위해 중앙값 절대 편차(MAD) 사용
  const values = readings.map(r => r.value);
  const median = calculateMedian(values);
  const mad = calculateMAD(values, median);

  readings.forEach(r => {
    const deviation = Math.abs(r.value - median) / mad;
    if (deviation > threshold) {
      r.qc_flag = 'questionable';
      r.qc_notes = `값이 중앙값에서 ${deviation.toFixed(1)} MAD 벗어남`;
    }
  });
}
```

**교차 매개변수 일관성:**
```typescript
function checkWaterOxygenTemperature(sample) {
  // 용존 산소는 온도가 증가함에 따라 감소해야 함
  const temp = sample.temperature_c;
  const do_mgl = sample.dissolved_oxygen_mgl;

  // 이론적 포화도 계산 (간소화된 Benson & Krause)
  const do_sat = 14.652 - 0.41022 * temp + 0.007991 * temp**2 - 0.000077774 * temp**3;

  // 측정된 DO > 110% 포화인 경우 플래그 지정 (과포화 소스 없이는 가능성 낮음)
  if (do_mgl > 1.1 * do_sat) {
    sample.quality_flags.push('DO가 온도에 대한 예상 포화도를 초과');
  }

  // 온도가 적당할 때 측정된 DO < 50% 포화인 경우 플래그 지정 (센서 문제 가능)
  if (temp < 25 && do_mgl < 0.5 * do_sat) {
    sample.quality_flags.push('DO가 온도에 비해 비정상적으로 낮음');
  }
}
```

## 6.6 현장 샘플링 프로토콜

### 6.6.1 지점 계수 프로토콜 (조류)

**표준 절차:**

1. **사이트 선택**
   - 계층화된 무작위 샘플링을 통해 선택된 고정 지점
   - 지점 사이 최소 250m (이중 계수 방지)
   - 연구 지역의 서식지 유형 대표

2. **타이밍**
   - 번식기 (위도 및 종에 따라 다름)
   - 해돋이부터 해돋이 후 4시간까지 (최고 활동)
   - 비, 강풍(>20 km/h) 피함

3. **절차**
   - 관찰자가 조용히 접근, 조류를 안정시키기 위해 1분 대기
   - 5-10분 동안 감지된 모든 조류 기록
   - 거리 밴드 기록: 0-50m, 50-100m, >100m
   - 탐지 방법 기록: 노래, 울음, 시각적

4. **노력 문서화**
   - 시작 시간, 종료 시간, 지속 시간
   - 날씨 조건 (구름 덮임, 바람, 온도)
   - 관찰자 ID
   - 배경 소음 수준

5. **반복 방문**
   - 시즌당 최소 3회 방문
   - 7-14일 간격
   - 탐지 확률 추정 가능

### 6.6.2 수질 샘플링 프로토콜

**표준 절차:**

1. **준비**
   - 깨끗한 니트릴 장갑 착용
   - 사전 세척된 병 사용 (금속용 산 세척, 유기물용 고압증기멸균)
   - 현장에 가기 전에 방수 마커로 병에 라벨 부착

2. **접근**
   - 샘플링 위치에서 하류에서 접근
   - 샘플 위치 상류의 퇴적물 교란 피함

3. **현장 매개변수**
   - 온도, pH, DO, 전도도를 현장에서 측정
   - 하루 시작 시 측정기 보정
   - 측정 전 프로브 3회 헹굼
   - 판독값이 안정화된 후 기록 (±2%)

4. **샘플 수집**
   - 주변 물로 병 3회 헹굼
   - 중류, 중간 깊이에서 수집
   - 표면(기름막) 및 바닥(퇴적물) 피함
   - DO의 경우: 완전히 채움, 헤드스페이스 없음
   - 금속의 경우: 즉시 필터링 (0.45 μm), 산성화 (HNO₃를 pH <2로)
   - 영양소의 경우: 필터링 및 냉동

5. **문서화**
   - 사이트 ID, GPS 좌표
   - 날짜 및 시간
   - 수집 깊이
   - 시각적 관찰 (색상, 냄새, 투명도)
   - 관리 연속성 양식

6. **운송 및 보관**
   - 즉시 얼음 위에 놓기
   - 6시간 이내에 실험실로 운송 (또는 매개변수 보관 시간당)
   - 운송 중 4°C 유지

### 6.6.3 카메라 트랩 배치

**표준 절차:**

1. **사이트 선택**
   - 게임 트레일, 수원 또는 체계적 그리드
   - 과열 및 눈부심 방지를 위해 직사광선 피함

2. **카메라 설정**
   - 지상 30-50cm 높이에 장착 (지상 서식 포유류)
   - 트레일과 평행하게 조준 (수직이 아님)
   - 앞 3m의 식물 제거 (잘못된 트리거 방지)
   - 떠나기 전에 트리거 테스트

3. **구성**
   - 사진 모드: 3장 연속 촬영
   - 지연: 1-5분 (탐지 vs. 저장 균형)
   - 감도: 중간-높음 (사이트에 맞게 조정)
   - 날짜/시간 스탬프: 켜짐
   - 위치 스탬프: 카메라가 지원하는 경우 켜짐

4. **메타데이터 문서화**
   ```json

     "camera_id": "RECONYX-PC900-001",
   ```

5. **유지보수**
   - 2-4주마다 배터리 및 메모리 확인
   - 렌즈 및 센서 청소
   - 탐지 영역으로 자라는 식물 제거
   - 이미지 다운로드, 필요시 카드 지우기

## 6.7 데이터 관리 프로토콜

### 6.7.1 데이터 입력

**모범 사례:**

- **이중 데이터 입력** (중요 데이터셋용)
  - 두 사람이 독립적으로 동일한 데이터 입력
  - 항목 비교 및 차이 조정
  - 데이터 입력 오류율을 ~1-3%에서 <0.1%로 줄임

- **실시간 검증**
  - 데이터 입력 중 검증 규칙 적용
  - 데이터 입력자에게 즉각적인 피드백
  - 오류 전파 방지

- **디지털 데이터 캡처**
  - 오프라인 기능이 있는 모바일 앱
  - 사전 채워진 목록 (종, 사이트)로 오타 방지
  - 자동 좌표를 위한 GPS 통합
  - 관찰과 연결된 사진

- **버전 관리**
  - 타임스탬프 및 사용자 ID로 모든 편집 기록
  - 이전 버전으로 되돌릴 수 있는 기능
  - 원시 데이터(절대 편집되지 않음)와 처리된 데이터의 분리

## 6.8 복습 질문

### 질문 1
수질 샘플의 스파이크 회수율이 65%입니다(허용 범위: 75-125%). 가능한 원인과 취할 수 있는 시정 조치는 무엇인가?

### 질문 2
온도, 습도 및 강수량 센서가 있는 원격 기상 관측소에 대한 보정 일정을 설계하시오. 어떤 빈도와 방법을 사용하겠는가?

### 질문 3
두 관찰자가 같은 위치에서 독립적으로 조류 조사를 수행합니다. 관찰자 A는 15종을 감지하고, 관찰자 B는 12종을 감지하며, 10종이 공통입니다. 관찰자 간 일치도를 계산하고 결과를 해석하시오.

### 질문 4
용존 산소 센서가 온도가 변함에도 불구하고 48시간 동안 플랫라인 동작(일정한 8.5 mg/L)을 보입니다. 가능성 있는 문제를 진단하고 해결책을 제안하시오.

### 질문 5
100명의 자원봉사자가 있는 시민 과학 나비 모니터링 프로그램에 대한 QA/QC를 설계하고 있습니다. 관찰자 전문성의 변동성을 고려할 때 가장 중요한 품질 관리 조치는 무엇인가?

## 6.9 주요 요점

| 프로토콜 영역 | 주요 요구사항 |
|-------------|-------------|
| **현장 QC** | 블랭크 5%, 복제본 10%, 관찰자 보정 |
| **실험실 QC** | 중복, 스파이크, 블랭크, 인증 참조 |
| **보정** | 센서별 간격, 문서화된 절차 |
| **검증** | 자동화된 범위, 속도, 플랫라인, 스파이크 감지 |
| **샘플링** | 각 데이터 유형에 대한 표준화된 프로토콜 |
| **문서화** | 포괄적인 메타데이터, 버전 관리 |

### 중요한 통계
- **10%**: 최소 현장 복제본
- **75-125%**: 허용 가능한 스파이크 회수 범위
- **< 20%**: 중복에 대한 최대 허용 RPD
- **6개월**: 일반적인 온도 센서 보정 간격
- **3-5회 방문**: 탐지 확률 추정을 위한 최소
