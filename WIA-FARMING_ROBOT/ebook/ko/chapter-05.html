제5장

API 인터페이스 및 통신 프로토콜

장 개요

이 장에서는 농업 로봇 제어 및 모니터링을 위한 RESTful API와 실시간 통신 프로토콜을 탐구합니다. WIA-FARMING_ROBOT 표준의 API 엔드포인트, 인증, MQTT 메시징, WebSocket 연결 및 데이터 보안을 다룹니다.

5.1 REST API 아키텍처

WIA-FARMING_ROBOT 표준은 로봇 관리, 작업 제어, 센서 데이터 수집 및 분석을 위한 포괄적인 RESTful API를 정의합니다.

5.1.1 기본 URL 및 버전 관리

기본 URL: https://api.farming-robot.wia.org/v1

버전 관리 전략:
- URL 경로 버전: /v1/, /v2/
- 헤더 버전: Accept-Version: 1.0
- 이전 버전과의 호환성: 2개 주요 버전
- 폐기 통지: 최소 6개월

5.4 MQTT 통신 프로토콜

MQTT는 원격 측정, 상태 업데이트 및 센서 데이터에 사용되는 경량 게시/구독 프로토콜입니다.

5.4.1 MQTT 토픽 구조

주제 구조:
wia/farming/{farmId}/{robotId}/{messageType}

예제:
wia/farming/farm-001/robot-550e/telemetry
wia/farming/farm-001/robot-550e/status
wia/farming/farm-001/robot-550e/alerts
wia/farming/farm-001/robot-550e/sensors/lidar
wia/farming/farm-001/robot-550e/sensors/camera
wia/farming/farm-001/robot-550e/commands

QoS 수준:
- QoS 0: 원격 측정 데이터 (고주파, 손실 허용)
- QoS 1: 상태 업데이트 (최소 한 번 전달)
- QoS 2: 제어 명령 (정확히 한 번 전달)

// JavaScript WebSocket 클라이언트
const ws = new WebSocket('wss://api.farming-robot.wia.org/v1/realtime');

ws.onopen = () => {
  // 인증
  ws.send(JSON.stringify({

ws.onmessage = (event) => {
  const message = JSON.parse(event.data);

      break;
      
    case 'TELEMETRY':

      break;

      updateTaskProgress(message.data);
      break;
      
    case 'SENSOR_DATA':
      processSensorData(message.data);
      break;

5.6.2 메시지 서명 및 검증

// 메시지 서명 생성
function signMessage(payload, secretKey) {
  const timestamp = Date.now();
  const nonce = generateNonce();
  const message = JSON.stringify(payload) + timestamp + nonce;
  
  const signature = crypto
    .createHmac('sha256', secretKey)
    .update(message)
    .digest('hex');
  
  return {
    payload: payload,
    timestamp: timestamp,
    nonce: nonce,
    signature: signature
  };
}

// 메시지 검증
function verifyMessage(signedMessage, secretKey) {
  const { payload, timestamp, nonce, signature } = signedMessage;
  
  // 타임스탬프 확인 (최대 5초 오래됨)
  if (Date.now() - timestamp > 5000) {
    throw new Error('Message expired');
  }
  
  // 서명 재계산 및 확인
  const message = JSON.stringify(payload) + timestamp + nonce;
  const expectedSignature = crypto
    .createHmac('sha256', secretKey)
    .update(message)
    .digest('hex');
  
  if (signature !== expectedSignature) {
    throw new Error('Invalid signature');
  }
  
  return payload;
}

5.7.2 재시도 전략

class APIClient {
  async requestWithRetry(endpoint, options, maxRetries = 3) {
    let lastError;
    
    for (let attempt = 0; attempt < maxRetries; attempt++) {
      try {
        const response = await fetch(endpoint, options);
        
        // 성공
        if (response.ok) {
          return await response.json();
        }
        
        // 재시도 불가능한 오류 (4xx 클라이언트 오류)
        if (response.status >= 400 && response.status < 500) {
          throw new Error(`Client error: ${response.status}`);
        }
        
        // 재시도 가능한 오류 (5xx 서버 오류)
        lastError = new Error(`Server error: ${response.status}`);
        
      } catch (error) {
        lastError = error;
      }
      
      // 지수 백오프 (exponential backoff)
      const delay = Math.min(1000 * Math.pow(2, attempt), 10000);
      await new Promise(resolve => setTimeout(resolve, delay));
    }
    
    throw lastError;
  }
}

핵심 요약

  • WIA REST API는 로봇 관리, 작업 제어 및 데이터 수집을 위한 포괄적인 엔드포인트를 제공합니다
  • JWT 기반 인증은 API 키 및 로봇 ID와 함께 안전한 액세스를 보장합니다
  • MQTT는 다양한 QoS 수준으로 경량 게시/구독 메시징을 제공합니다: QoS 0(원격 측정), QoS 1(상태), QoS 2(명령)
  • WebSocket은 양방향 실시간 제어 및 모니터링을 위한 지속적인 연결을 제공합니다
  • TLS 1.3 암호화는 전송 중인 모든 데이터를 보호하며, HMAC-SHA256 서명은 메시지 무결성을 보장합니다
  • 오류 처리는 표준 오류 코드 및 지수 백오프를 사용한 재시도 로직을 포함합니다
  • 속도 제한은 시스템 안정성을 보장하며, 엔드포인트 유형에 따라 다른 제한이 적용됩니다
