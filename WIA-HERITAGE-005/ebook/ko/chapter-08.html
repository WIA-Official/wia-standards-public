08.4 인증 프로세스

WIA 인증은 독립적인 3자 감사 기관이 수행하며, 기술 준수와 윤리 준수를 모두 평가합니다.

인증 단계

단계기간활동비용
1. 사전 평가2주자가 진단 도구로 준비도 확인무료
2. 신청1주신청서 제출, 증빙 자료 업로드$500
3. 문서 검토2주감사 기관이 문서 심사$2K-5K
4. 기술 감사1주API 테스트, 코드 리뷰, 침투 테스트$5K-15K
5. 공동체 검증2주공동체 대표 인터뷰, 만족도 조사$3K-8K
6. 최종 심사1주WIA 인증위원회 승인 결정포함
7. 인증서 발급1주디지털 인증서 + NFT 배지포함

공동체 검증 - 가장 중요한 단계

기술 준수는 필요조건이지만, 공동체 검증은 충분조건입니다. 인증 과정의 30%는 공동체가 직접 평가합니다.

🗣️ 공동체 검증 인터뷰 질문 (실제 사례)

감사 기관은 최소 3개 공동체의 대표자와 인터뷰합니다:

  1. 이 조직과 협력하기 전에 충분한 정보를 받았습니까? (FPIC)
  2. 협력 조건을 이해하고 자유롭게 결정할 수 있었습니까?
  3. 약속된 금전적/비금전적 혜택을 받았습니까?
  4. 신성한 지식이 보호되었습니까?
  5. 연구 진행 상황을 정기적으로 통보받았습니까?
  6. 문제 제기 시 신속히 해결되었습니까?
  7. 다른 공동체에 이 조직을 추천하시겠습니까? (1-10점)

인증 기준: 7개 질문 모두 "예", 추천 점수 평균 8점 이상

실패 원인 분석

TCM University는 기술 요구사항은 충족했으나, 공동체 검증에서 탈락했습니다.

  • 톱다운 접근: 대학이 일방적으로 데이터 수집, 공동체 참여 최소화
  • FPIC 부재: 고문헌 기반이라 FPIC 불필요하다고 착각
  • 수익 독점: 로열티를 대학이 100% 보유, 공동체 배분 0%

교훈: WIA-HERITAGE-005는 단순히 "데이터를 RDF로 변환"하는 것이 아니라, "공동체와 함께" 하는 것입니다. 2024년 재신청 시 소수민족 마을 5곳과 파트너십 체결 후 진행 중.

08.6 흔한 구현 실수 Top 10

⚠️ 실수 1: "기술만 하면 된다"

증상: API는 완벽한데, 공동체가 사용하지 않거나 불만

원인: 공동체를 "데이터 소스"로만 보고, 파트너로 존중하지 않음

해결: 초기부터 공동체를 거버넌스 위원회에 포함, 수익 공유 명시

⚠️ 실수 2: "우리는 특별하니 표준을 커스터마이즈"

증상: 표준을 과도하게 변형, 다른 시스템과 호환 불가

원인: "우리 조직은 다르다" 신드롬

해결: 표준은 80%를 커버, 20%는 확장 필드로 해결

⚠️ 실수 3: "보안은 나중에"

증상: 출시 직전 침투 테스트에서 치명적 취약점 발견, 몇 달 지연

원인: 보안을 사후 점검으로 생각

해결: 설계 단계부터 보안 전문가 참여, "Security by Design"

⚠️ 실수 4: "블록체인은 buzzword라서 넣었다"

증상: 블록체인 비용만 높고, 실제 투명성 개선 없음

원인: 블록체인을 목적이 아닌 도구로 이해 못함

해결: 변조 방지가 정말 필요한 곳에만 사용 (로열티, 동의 기록)

⚠️ 실수 5: "다국어는 나중에"

증상: 영어로만 구축 후, 원주민 언어 추가 시 전체 재설계

원인: i18n을 사후 기능으로 간주

해결: 초기부터 UTF-8, 원주민 언어를 1급 시민으로 설계
