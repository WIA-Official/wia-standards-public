Phase 1: 데이터 형식 - 지식 그래프 설계

WIA-HERITAGE-005: Traditional Knowledge Standard

04.1 전통 지식의 구조화 과제

전통 지식을 디지털 형식으로 표현하는 것은 단순한 데이터베이스 설계를 넘어서는 복잡한 도전입니다. 서구 과학 지식과 달리, 전통 지식은 본질적으로 맥락 의존적이고, 다층적 관계로 엮여 있으며, 문화적 규범과 불가분의 관계에 있습니다.

기존 데이터 형식의 한계

전통 지식을 저장하기 위한 기존 시도들은 대부분 관계형 데이터베이스나 단순 XML 스키마에 의존했습니다. 이러한 접근법의 문제점:

  • 관계의 단순화: 약용 식물과 질병 사이의 "치료" 관계를 단순한 1:1 또는 1:N 관계로 표현하면, 계절, 환자 상태, 다른 식물과의 조합, 의례적 맥락 등의 복잡한 조건들을 잃게 됩니다.
  • 다국어 지원 부족: 원주민 언어의 복잡한 의미론적 뉘앙스를 단순한 번역으로 표현할 수 없습니다. 예를 들어, 케추아어의 "sumaq kawsay"(좋은 삶)는 서구 개념의 "웰빙"과는 근본적으로 다른 세계관을 내포합니다.
  • 접근 제어 미흡: 지식 항목 수준의 단순한 권한 설정으로는 "여성만 알 수 있는", "성인식 후에만 전승되는", "건기에만 논의 가능한" 등의 문화적 규범을 표현할 수 없습니다.
  • 출처 추적 불가: 누가 언제 어떤 지식을 기여했는지, 그 지식이 어떻게 변형되고 확산되었는지 추적할 메커니즘이 없습니다.
📊 케이스 스터디: 아마존 약용 식물 데이터베이스의 실패

2008년, 브라질 정부는 아마존 원주민의 약용 식물 지식을 MySQL 데이터베이스에 기록하는 프로젝트를 시작했습니다. 3년 후 프로젝트는 다음의 이유로 중단되었습니다:

  • 1,200종의 식물 중 겨우 37종만 "올바르게" 기록됨
  • 샤먼들이 데이터베이스에 기록된 정보가 "죽은 지식"이라고 거부
  • 의례적 맥락이 제거된 약용 정보가 실제로는 효과가 없거나 위험함
  • 공동체 간 지식 공유 프로토콜을 무시하여 문화적 갈등 발생

이 실패는 전통 지식이 단순한 "팩트"의 집합이 아니라, 문화적, 영적, 생태학적 맥락 속에서만 의미를 가지는 살아있는 체계임을 보여줍니다.

04.2 지식 그래프: 전통 지식을 위한 이상적 모델

지식 그래프(Knowledge Graph)는 전통 지식의 복잡성을 표현하기에 가장 적합한 데이터 모델입니다. 관계형 데이터베이스의 고정된 테이블 대신, 지식 그래프는 노드(개체)와 엣지(관계)의 유연한 네트워크로 지식을 표현합니다.

지식 그래프의 핵심 구성 요소

구성 요소설명전통 지식 사례
노드 (Node)개별 지식 항목 또는 개체식물, 질병, 의례, 계절, 장로, 공동체
엣지 (Edge)노드 간의 관계치료, 사용, 전승, 금기, 공존, 대체
속성 (Property)노드나 엣지의 특성학명, 현지명, 채집 시기, 복용량, 금기 대상
온톨로지 (Ontology)노드와 관계의 유형 정의식물 분류법, 관계 유형 체계, 접근 등급

RDF 기반 표현

WIA-HERITAGE-005는 W3C의 RDF (Resource Description Framework) 표준을 확장하여 전통 지식을 표현합니다. RDF는 주어(Subject) - 술어(Predicate) - 목적어(Object)의 삼중구조(Triple)로 모든 지식을 표현합니다.

간단한 예시: "Ayahuasca는 우울증을 치료한다"

@prefix tk: <http://wia.org/heritage/vocab#> .
@prefix rdfs: <http://www.w3.org/2000/01/rdf-schema#> .

tk:Ayahuasca a tk:MedicinalPlant ;
    rdfs:label "Ayahuasca"@quy ;
    rdfs:label "아야와스카"@ko ;
    tk:scientificName "Banisteriopsis caapi" ;
    tk:treats tk:Depression ;
    tk:accessLevel tk:CommunityOnly .

tk:Depression a tk:MentalCondition ;
    rdfs:label "Llaki"@quy ;
    rdfs:label "우울증"@ko .

하지만 실제 전통 지식은 이보다 훨씬 복잡합니다. 아야와스카의 사용은 샤먼의 감독, 특정 의례, 식이 제한, 다른 식물과의 조합 등 수많은 조건에 의존합니다.

04.3 WIA-TK 온톨로지: 전통 지식 표현 체계

WIA-HERITAGE-005는 전통 지식을 위한 포괄적 온톨로지인 WIA-TK (WIA Traditional Knowledge Ontology)를 정의합니다. 이는 기존의 Dublin Core, FOAF, SKOS 등의 표준 온톨로지를 확장합니다.

핵심 클래스 계층

tk:TraditionalKnowledge (최상위 클래스)
├── tk:EcologicalKnowledge (생태학적 지식)
│   ├── tk:MedicinalPlant (약용 식물)
│   ├── tk:EdiblePlant (식용 식물)
│   ├── tk:SacredSite (신성한 장소)
│   └── tk:AnimalBehavior (동물 행동)
├── tk:TechnicalKnowledge (기술 지식)
│   ├── tk:CraftTechnique (공예 기술)
│   ├── tk:ConstructionMethod (건축 기법)
│   └── tk:FoodPreservation (식품 보존)
├── tk:CulturalPractice (문화적 관행)
│   ├── tk:Ritual (의례)
│   ├── tk:OralTradition (구전 전통)
│   └── tk:SocialNorm (사회 규범)
└── tk:AgriculturalKnowledge (농업 지식)
    ├── tk:CropVariety (작물 품종)
    ├── tk:SoilManagement (토양 관리)
    └── tk:ClimateForecasting (기후 예측)

관계 유형 (Property)

전통 지식의 복잡한 관계를 표현하기 위해 60개 이상의 관계 유형을 정의합니다:

관계 카테고리관계 예시의미
치료 관계tk:treats, tk:prevents, tk:diagnoses건강 관련 효과
생태 관계tk:growsWith, tk:indicatorOf, tk:competesFor생태계 내 상호작용
문화 관계tk:sacredTo, tk:tabooFor, tk:restrictedBy문화적 규범과 금기
시간 관계tk:harvestDuring, tk:usedInSeason, tk:celebratedOn계절성과 타이밍
전승 관계tk:taughtBy, tk:inherits, tk:modifiedFrom지식의 계승과 변화
공동체 관계tk:ownedBy, tk:sharedWith, tk:tradedTo지식의 소유와 공유

복잡한 예시: 안데스 퀴노아 재배 지식

@prefix tk: <http://wia.org/heritage/vocab#> .
@prefix geo: <http://www.w3.org/2003/01/geo/wgs84_pos#> .
@prefix time: <http://www.w3.org/2006/time#> .

# 퀴노아 품종
tk:QuinoaRealBlanca a tk:CropVariety ;
    rdfs:label "Quinoa Real Blanca"@es ;
    rdfs:label "유라크 퀴노아"@ay ;  # 아이마라어
    tk:scientificName "Chenopodium quinoa var. real" ;
    tk:ownedBy tk:AymaraCommunity_Uyuni ;
    tk:cultivatedAt tk:UyuniAltiplano ;
    tk:altitude "3,600-4,200m" ;
    tk:accessLevel tk:PublicWithAttribution .

# 재배 시기 (남반구 계절)
tk:QuinoaPlantingSeason a time:Interval ;
    time:hasBeginning tk:October_SouthernSpring ;
    time:hasEnd tk:November_SouthernSpring ;
    tk:determinedBy tk:RainySeasonOnset ;
    tk:indicatorSpecies tk:Sikuya_Frog .  # 개구리 울음으로 판단

# 토양 준비 기법
tk:QuinoaSoilPreparation a tk:SoilManagement ;
    rdfs:label "Mink'a Chakra Wakichay"@quy ;  # 케추아어
    tk:involves tk:LlamaMannureApplication ;
    tk:rotationWith tk:PotatoCrop ;
    tk:rotationCycle "3-5 years" ;
    tk:taughtBy tk:Elder_MariaSisaY ;
    tk:recordedDate "2023-08-15"^^xsd:date .

# 수확 의례
tk:QuinoaHarvestRitual a tk:Ritual ;
    rdfs:label "Jamp'atu"@ay ;
    tk:performedBy tk:CommunityElders ;
    tk:invokesSpiritOf tk:Pachamama ;  # 대지의 여신
    tk:accessLevel tk:CommunityOnly ;
    tk:genderRestriction "women-elders-only" .

# 관계 정의
tk:QuinoaRealBlanca tk:requiresRitual tk:QuinoaHarvestRitual ;
    tk:plantedDuring tk:QuinoaPlantingSeason ;
    tk:requiresSoilPrep tk:QuinoaSoilPreparation ;
    tk:resistantTo tk:FrostDown-8C ;
    tk:sharedWith tk:PeruvianQuechuaCommunities .

04.4 다국어 지원과 언어학적 도전

전통 지식의 가장 큰 과제 중 하나는 언어의 다양성입니다. 전 세계 7,000개 이상의 언어 중 많은 수가 문자 체계가 없거나, 디지털 폰트가 없거나, 기계 번역이 불가능합니다.

다국어 레이블 시스템

WIA-TK 온톨로지는 각 개체에 대해 무제한의 언어 레이블을 허용하며, 각 레이블은 다음의 메타데이터를 포함합니다:

  • 언어 코드: ISO 639-3 3글자 코드 (예: quy=케추아어, ay=아이마라어)
  • 방언 변이: 같은 언어 내 지역별 차이 (예: ay-north, ay-south)
  • 의미론적 범위: 번역이 원어의 의미를 완전히 포착하는지 부분적인지 표시
  • 문화적 맥락: 특정 문화권에서만 의미가 있는 개념인지 표시
🗣️ 언어적 상대성: 번역 불가능한 개념들

일부 전통 지식 개념은 다른 언어로 번역이 불가능하거나, 번역하면 의미가 왜곡됩니다:

  • Sumaq Kawsay (케추아어): "좋은 삶"으로 번역되지만, 실제로는 자연, 공동체, 영성의 조화로운 균형을 의미하는 세계관 전체
  • Ubuntu (줄루어): "나는 우리가 있기에 존재한다" - 개인주의 문화권에서는 개념화하기 어려운 집단적 존재론
  • Dadirri (호주 원주민 언어): 깊은 경청과 자연과의 조용한 교감 - "명상"이나 "관찰"로는 의미 손실

WIA-HERITAGE-005는 이러한 개념을 번역하지 않고 원어로 보존하면서, 상세한 문화적 설명을 별도 속성으로 추가할 것을 권장합니다.

음성 및 시각 데이터 통합

많은 전통 지식은 문자보다 구전, 실연, 의례를 통해 전승됩니다. WIA-TK는 멀티미디어 데이터를 지식 그래프에 통합합니다:

tk:AyahuascaPreparation a tk:RitualPractice ;
    rdfs:label "Ayahuasca Ikaro"@quy ;
    tk:description "샤먼이 아야와스카 조제 시 부르는 치유의 노래"@ko ;
    tk:audioRecording <https://heritage.wia.org/audio/ikaro_001.ogg> ;
    tk:recordedBy tk:Shaman_DonJuanTananta ;
    tk:recordingDate "2023-06-12"^^xsd:date ;
    tk:consent tk:ConsentDoc_2023_FPIC_042 ;
    tk:accessLevel tk:RestrictedResearch ;
    tk:culturalSensitivity "sacred-music" .

04.5 접근 제어 및 문화적 민감성 등급

전통 지식의 가장 중요한 측면 중 하나는 모든 지식이 공개되어서는 안 된다는 것입니다. 많은 원주민 문화에서 특정 지식은 성별, 연령, 이니시에이션 단계, 사회적 역할에 따라 제한됩니다.

4단계 접근 등급 체계

등급명칭접근 조건사례
Level 1공개 (Public)누구나 접근 가능, 출처 표시 필수일반적인 약용 식물 용도, 전통 요리법, 공예 기법
Level 2제한적 공개 (Restricted)연구자/학생, 공동체 승인 필요특정 의례의 일반적 설명, 멸종위기 식물 위치
Level 3공동체 전용 (Community Only)해당 공동체 구성원만상세한 의료 처방, 신성한 장소의 정확한 위치
Level 4신성/비밀 (Sacred/Secret)특정 자격 보유자만 (장로, 샤먼 등)이니시에이션 의례, 샤먼 훈련 지식, 성별 제한 지식

동적 접근 제어

접근 등급은 고정된 것이 아니라, 시간, 계절, 개인의 상태에 따라 변할 수 있습니다:

tk:SacredPlantKnowledge a tk:MedicinalKnowledge ;
    tk:baseAccessLevel tk:CommunityOnly ;
    tk:accessRule [
        a tk:ConditionalAccess ;
        tk:condition "requester.age >= 18" ;
        tk:condition "requester.initiationComplete = true" ;
        tk:condition "currentSeason != 'rainy-season'" ;
        tk:condition "requester.gender = 'male'" ;  # 특정 지식은 성별 제한
        tk:grantsAccessTo tk:SacredPlantFullKnowledge
    ] .

문화적 민감성 태그

접근 등급 외에도, 지식의 문화적 맥락을 표시하는 태그를 추가합니다:

  • sacred: 영적/종교적으로 신성한 지식
  • gender-restricted: 특정 성별만 접근 가능
  • age-restricted: 특정 연령 이상만 접근 가능
  • seasonal: 특정 계절에만 논의 가능
  • endangered: 멸종 위기 종과 관련되어 위치 정보 숨김 필요
  • commercially-sensitive: 상업적 착취 위험이 높음

04.6 출처 추적 및 버전 관리

전통 지식은 정적이지 않습니다. 세대를 거치며 진화하고, 새로운 발견이 추가되며, 환경 변화에 적응합니다. 이러한 변화를 추적하는 것은 지식의 진정성과 공동체의 권리를 보호하는 데 필수적입니다.

지식 출처 메타데이터 (Provenance)

각 지식 항목은 다음의 출처 정보를 포함해야 합니다:

tk:QuinoaFrostResistance a tk:AgriculturalKnowledge ;
    rdfs:label "퀴노아의 서리 저항성 지식"@ko ;

    # 원 기여자
    prov:wasAttributedTo tk:Elder_PedroMamani ;
    prov:generatedAtTime "2022-11-03T14:30:00Z"^^xsd:dateTime ;
    prov:atLocation tk:Village_Salinas_de_Garci_Mendoza ;

    # 기록자
    prov:wasRecordedBy tk:Researcher_MariaJohnson ;
    prov:withConsent tk:FPIC_Consent_2022_089 ;

    # 검증자
    tk:verifiedBy tk:Elder_RosaCondori ;
    tk:verifiedBy tk:Elder_JuanQuispe ;
    tk:verificationDate "2022-11-15"^^xsd:date ;

    # 역사적 계승
    prov:wasDerivedFrom tk:AncestralKnowledge_Quinoa ;
    tk:transmittedThrough "7-generations"^^xsd:string ;
    tk:firstRecordedBy "Spanish chronicler Pedro de Cieza, 1553" .

지식 진화 추적

전통 지식이 시간에 따라 어떻게 변화했는지 추적합니다:

# 버전 1: 전통적 방법 (2020년 기록)
tk:QuinoaStorageMethod_v1 a tk:FoodPreservation ;
    rdfs:label "전통 퀴노아 저장법"@ko ;
    tk:method "흙 저장고, 점토 용기" ;
    prov:generatedAtTime "2020-03-10"^^xsd:date .

# 버전 2: 기후 변화 적응 (2023년 기록)
tk:QuinoaStorageMethod_v2 a tk:FoodPreservation ;
    rdfs:label "개량된 퀴노아 저장법"@ko ;
    tk:method "흙 저장고 + 플라스틱 라이너 (습도 증가 대응)" ;
    prov:wasRevisionOf tk:QuinoaStorageMethod_v1 ;
    prov:generatedAtTime "2023-07-22"^^xsd:date ;
    tk:modificationReason "기후 변화로 인한 우기 강수량 40% 증가" ;
    tk:adoptionRate "78% of community members" .

04.7 상호운용성 및 기존 표준과의 연계

04.8 장 요약

✅ 핵심 내용
  1. 전통 지식의 복잡성: 단순한 데이터베이스로는 맥락 의존적, 다층적, 문화적으로 민감한 전통 지식의 본질을 포착할 수 없습니다. 과거의 실패 사례들은 문화적 맥락을 무시한 대가를 보여줍니다.
  2. 지식 그래프 모델: RDF 기반 지식 그래프는 노드와 엣지의 유연한 네트워크로 복잡한 관계와 맥락을 자연스럽게 표현합니다. 주어-술어-목적어 삼중구조로 모든 지식을 표현합니다.
  3. WIA-TK 온톨로지: 60개 이상의 관계 유형과 4개 주요 클래스(생태, 기술, 문화, 농업)로 구성된 포괄적 온톨로지를 정의합니다. Dublin Core, FOAF, Darwin Core 등 기존 표준을 확장합니다.
  4. 다국어 지원: 7,000개 언어를 위한 무제한 다국어 레이블 시스템을 제공하며, 번역 불가능한 개념은 원어로 보존하고 상세한 문화적 설명을 추가합니다. 음성 및 시각 데이터도 통합합니다.
  5. 4단계 접근 제어: 공개, 제한적 공개, 공동체 전용, 신성/비밀의 4단계 등급으로 문화적 규범을 존중합니다. 시간, 계절, 성별, 연령에 따른 동적 접근 제어를 지원합니다.
  6. 출처 및 진화 추적: PROV-O 표준으로 지식의 기여자, 기록자, 검증자, 역사적 계승을 추적하며, 버전 관리로 지식의 시간적 변화와 기후 변화 적응을 기록합니다.
弘益人間

전통 지식의 디지털화는 보존이 아니라 존중입니다. 지식 그래프는 단순히 데이터를 저장하는 것이 아니라, 지식이 태어난 문화적 생태계를 디지털 공간에 재현합니다.

04.9 복습 문제

  1. 2008년 아마존 약용 식물 데이터베이스 프로젝트가 실패한 주요 원인 4가지를 분석하고, WIA-TK 온톨로지가 이러한 문제를 어떻게 해결하는지 설명하세요.
  2. 지식 그래프가 관계형 데이터베이스보다 전통 지식 표현에 적합한 이유를 노드-엣지 구조의 특성과 함께 설명하고, 퀴노아 재배 지식의 RDF 예시에서 최소 5개의 삼중구조를 식별하세요.
  3. WIA-TK 온톨로지의 4개 주요 클래스 계층과 각각의 하위 클래스를 나열하고, 각 클래스에 적용 가능한 관계 유형(property)를 3개씩 제시하세요.
  4. "Sumaq Kawsay"와 같이 번역 불가능한 전통 지식 개념을 WIA-HERITAGE-005에서 어떻게 표현하는지 설명하고, 이것이 단순 번역보다 나은 이유를 문화적 맥락 보존 관점에서 논의하세요.
  5. 4단계 접근 제어 체계(공개, 제한적 공개, 공동체 전용, 신성/비밀)의 각 등급에 해당하는 전통 지식 사례를 제시하고, 동적 접근 제어가 필요한 상황을 2가지 이상 설명하세요.
  6. PROV-O 기반 출처 추적이 생물해적행위 방지에 어떻게 기여하는지 설명하고, 지식의 버전 관리가 기후 변화에 따른 전통 지식의 적응을 어떻게 문서화하는지 퀴노아 저장법 예시를 통해 분석하세요.

04.10 다음 장 예고

제5장에서는 정의된 데이터 형식을 실제로 활용하기 위한 API 인터페이스를 다룹니다. 전통 지식 데이터베이스에 접근하기 위한 RESTful API와 GraphQL 엔드포인트, 공동체 중심의 OAuth 인증 시스템, FPIC(사전 정보 동의) 자동화 워크플로우, 그리고 모든 접근을 블록체인에 기록하는 투명성 메커니즘을 상세히 살펴봅니다. 또한 아마존 지역의 실제 API 구현 사례와 제약 회사의 윤리적 연구 접근 프로토콜을 분석합니다.
