Chapter 4: Phase 1: Data Format Specification

4.1 Data Schema Overview

Phase 1 of WIA-SOC-011 establishes comprehensive data schemas representing all elements of gas supply infrastructure and operations. These schemas use JSON Schema format, providing human-readable specifications that are also machine-processable for automated validation. The data model follows object-oriented principles with inheritance, composition, and polymorphism enabling flexible yet consistent representations.

4.2 Pipeline Data Structures

Pipeline objects represent physical gas transmission and distribution infrastructure. Core attributes include unique identifiers (UUID format), geometric representations (GeoJSON for spatial data), material specifications (steel grade, coating type, wall thickness), pressure ratings (maximum operating pressure, design pressure), installation metadata (construction date, manufacturer, installer), and operational status (active, inactive, abandoned, planned).

4.3 Operational Data Formats

Operational data captures real-time and historical measurements from gas systems. Measurement objects include timestamps (ISO 8601 format with timezone), values with units (SI units by default, conversions defined for common alternatives), quality indicators (good, uncertain, bad), and data source identification (sensor ID, measurement method). Time series data uses efficient encoding for high-frequency measurements while maintaining compatibility with standard JSON.

4.4 Equipment Specifications

Equipment data structures cover all mechanical and electronic components. Compressor objects detail capacity curves, efficiency characteristics, maintenance schedules, and performance monitoring. Valve specifications include type (gate, ball, butterfly), actuation method (manual, motorized, pneumatic), position feedback, and safety interlocks. Meter definitions encompass measurement technology (turbine, ultrasonic, Coriolis), accuracy class, calibration data, and communication interfaces.

4.5 Gas Quality Data

Gas composition and quality parameters require detailed representation for custody transfer, regulatory compliance, and operational optimization. Composition data lists molecular components with mole fractions, calculated properties include heating value, Wobbe index, density, and compressibility. Quality attributes track moisture content, hydrogen sulfide levels, and other contaminants against specification limits.

4.6 Event and Alarm Data

Events and alarms receive structured representation enabling consistent handling across systems. Event objects include severity levels (informational, warning, alarm, emergency), event classification (operational, maintenance, safety, security), affected assets (pipeline sections, equipment, facilities), and response requirements (acknowledgment needed, action required, escalation procedures). State machines define alarm progression through detected, acknowledged, and cleared states.

4.7 Spatial Data Integration

Gas infrastructure inherently has spatial characteristics requiring geographic data integration. WIA-SOC-011 adopts GeoJSON for all spatial representations, enabling direct use in mapping applications and GIS systems. Coordinate reference systems follow EPSG standards with WGS84 (EPSG:4326) as default. Linear referencing systems accommodate pipeline-centric location methods (e.g., "kilometer 45.3 on pipeline PL-001").

4.8 Data Validation and Quality

Comprehensive validation rules ensure data quality and consistency. Schema validation confirms all required fields exist with correct types. Range validation checks values against physical limits (e.g., pressure cannot exceed pipe rating). Relationship validation ensures referential integrity (e.g., meter references valid pipeline). Temporal validation detects time sequence errors and future timestamps. Custom validators can add domain-specific business rules.

📌 Key Takeaways

  • JSON Schema provides human-readable, machine-processable data format specifications
  • Comprehensive schemas cover pipelines, operational data, equipment, gas quality, and events
  • GeoJSON integration enables seamless spatial data handling in mapping applications
  • Multi-layer validation ensures data quality, consistency, and referential integrity
  • Flexible schema design accommodates diverse infrastructure types and operating requirements
