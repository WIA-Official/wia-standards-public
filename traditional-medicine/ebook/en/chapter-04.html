Chapter 4: Phase 1 - Data Format

Standardized JSON Schemas for Integrative Medicine

4.1 Overview of Data Standards

The WIA-TRADITIONAL-MEDICINE data format establishes a common language for representing traditional medicine concepts in modern healthcare systems. The format is designed to be:

  • Comprehensive: Captures the full richness of traditional diagnostic information
  • Interoperable: Compatible with HL7 FHIR and other healthcare standards
  • Multilingual: Preserves original terminology alongside translations
  • Extensible: Allows for addition of new traditional medicine systems
  • Validated: Includes schema validation for data quality

4.5 Data Validation

The WIA-TRADITIONAL-MEDICINE standard includes JSON Schema definitions for validation:

  • Required fields for each data type
  • Enumerated values for standardized terminology
  • Range constraints for numerical values
  • Reference validation for related entities
Implementation Note: All schemas are published at https://wia.live/schemas/traditional-medicine/ and can be used with standard JSON Schema validators. SDKs are available for TypeScript, Python, and Java.
