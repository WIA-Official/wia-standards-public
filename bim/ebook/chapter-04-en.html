Chapter 4: Clash Detection and Coordination

Automated Conflict Identification and Multi-Discipline Coordination

4.1 Introduction to Clash Detection

Clash detection is one of the most valuable capabilities of BIM, enabling project teams to identify and resolve conflicts between building systems before construction begins. Traditional 2D workflows often resulted in clashes being discovered during construction, leading to costly rework, schedule delays, and coordination failures. BIM-based clash detection transforms this reactive approach into a proactive one.

A clash occurs when two or more building elements occupy the same physical space or when required clearances are violated. These conflicts can range from obvious hard clashes (physical intersections) to more subtle soft clashes (clearance violations) and workflow clashes (sequencing conflicts). Early detection and resolution of these issues significantly reduces construction costs and improves project outcomes.

Cost Impact of Clashes

Industry studies indicate that undetected clashes can cost 10-30 times more to resolve during construction than during the design phase. A single major clash discovered in the field can result in weeks of delay and hundreds of thousands of dollars in additional costs.

4.2 Types of Clashes

Clash Type Description Examples Detection Method
Hard Clash Physical intersection of two elements Duct through beam, pipe through wall Geometry intersection
Soft Clash (Clearance) Violation of required clearance zones Insufficient maintenance access, code clearances Buffer zone analysis
4D Clash (Time) Construction sequence conflicts Equipment installed before access path Schedule-linked analysis
Workflow Clash Design intent or standards violation Accessibility issues, code non-compliance Rule-based checking
Duplicate Clash Same element modeled by multiple parties Double-counted walls, overlapping grids Identity matching

4.4 Clash Test Configuration

// WIA-BIM Clash Test Configuration Schema
interface ClashTestConfig {
    testId: string;
    name: string;
    description?: string;
    selectionA: ElementSelection;      // First selection set
    selectionB: ElementSelection;      // Second selection set
    testType: ClashTestType;
    tolerance: ToleranceSettings;
    exclusions: ExclusionRule[];
    priority: ClashPriority;
}

interface ElementSelection {
    source: 'MODEL' | 'SAVED_SELECTION' | 'SEARCH';
    modelIds?: string[];
    ifcTypes?: string[];               // e.g., ['IfcDuctSegment', 'IfcPipeSegment']
    propertyFilters?: PropertyFilter[];
    spatialFilters?: SpatialFilter[];
}

interface ToleranceSettings {
    clashType: 'HARD' | 'SOFT' | 'CLEARANCE';
    hardTolerance: number;             // mm - ignore clashes smaller than this
    clearanceDistance?: number;        // mm - required clearance for soft clash
    touchTolerance: number;            // mm - elements within this are "touching"
}

interface ExclusionRule {
    type: 'SAME_ELEMENT' | 'SAME_GROUP' | 'CONNECTED' | 'CONTAINER' | 'CUSTOM';
    description: string;
    condition?: string;                // Custom condition expression
}

type ClashTestType = 'INTERSECTION' | 'CLEARANCE' | 'DUPLICATE' | 'HARD_SOFT';
type ClashPriority = 'CRITICAL' | 'HIGH' | 'MEDIUM' | 'LOW';

// Example: MEP vs Structure Clash Test
const mepStructureTest: ClashTestConfig = {
    testId: 'CT-001',
    name: 'MEP vs Structure Hard Clash',
    description: 'Detect intersections between MEP systems and structural elements',
    selectionA: {
        source: 'MODEL',
        ifcTypes: ['IfcDuctSegment', 'IfcPipeSegment', 'IfcCableCarrierSegment'],
        propertyFilters: [{
            propertySet: 'Pset_*Common',
            property: 'Status',
            operator: 'NOT_EQUALS',
            value: 'DEMOLISHED'
        }]
    },
    selectionB: {
        source: 'MODEL',
        ifcTypes: ['IfcBeam', 'IfcColumn', 'IfcSlab', 'IfcWall'],
        propertyFilters: [{
            propertySet: 'Pset_*Common',
            property: 'LoadBearing',
            operator: 'EQUALS',
            value: true
        }]
    },
    testType: 'HARD_SOFT',
    tolerance: {
        clashType: 'HARD',
        hardTolerance: 10,           // Ignore clashes < 10mm
        clearanceDistance: 50,       // 50mm clearance required
        touchTolerance: 1            // 1mm touch tolerance
    },
    exclusions: [
        { type: 'SAME_ELEMENT', description: 'Ignore self-clashes' },
        { type: 'CONNECTED', description: 'Ignore designed connections' }
    ],
    priority: 'CRITICAL'
};

4.5 BCF (BIM Collaboration Format)

BCF (BIM Collaboration Format) is the open standard for communicating BIM issues, including clash detection results. BCF enables clash reports to be exchanged between different software platforms, ensuring that issues identified in one tool can be reviewed and resolved in another.

4.6 Clash Detection API

// WIA-BIM SDK: Clash Detection Service
import { WIABimClient, ClashEngine, BCFExporter } from '@wia/bim-sdk';

class ClashDetectionService {
    private client: WIABimClient;
    private engine: ClashEngine;

    constructor(config: ServiceConfig) {
        this.client = new WIABimClient(config);
        this.engine = new ClashEngine();
    }

    async runClashTests(
        federatedModelId: string,
        tests: ClashTestConfig[]
    ): Promise<ClashTestResults> {
        // Load federated model
        const model = await this.client.loadFederatedModel(federatedModelId);

        const results: ClashTestResult[] = [];

        for (const test of tests) {
            // Get element selections
            const selectionA = await this.resolveSelection(model, test.selectionA);
            const selectionB = await this.resolveSelection(model, test.selectionB);

            // Run clash detection
            const clashes = await this.engine.detectClashes({
                elementsA: selectionA,
                elementsB: selectionB,
                testType: test.testType,
                tolerance: test.tolerance,
                exclusions: test.exclusions
            });

            results.push({
                testId: test.testId,
                testName: test.name,
                priority: test.priority,
                clashCount: clashes.length,
                clashes: clashes.map(c => this.formatClash(c, test))
            });
        }

        return {
            modelId: federatedModelId,
            timestamp: new Date().toISOString(),
            testResults: results,
            summary: this.generateSummary(results)
        };
    }

    async exportToBCF(
        results: ClashTestResults,
        options: BCFExportOptions
    ): Promise<Buffer> {
        const exporter = new BCFExporter(options);

        for (const testResult of results.testResults) {
            for (const clash of testResult.clashes) {
                await exporter.addTopic({
                    title: `${clash.elementA.name} vs ${clash.elementB.name}`,
                    description: clash.description,
                    priority: this.mapPriority(testResult.priority),
                    topicType: 'Clash',
                    labels: ['Coordination', clash.disciplineA, clash.disciplineB],
                    viewpoint: await this.generateViewpoint(clash),
                    relatedElements: [clash.elementA.globalId, clash.elementB.globalId]
                });
            }
        }

        return await exporter.export();
    }

    private formatClash(clash: RawClash, test: ClashTestConfig): FormattedClash {
        return {
            id: clash.id,
            elementA: {
                globalId: clash.elementA.globalId,
                name: clash.elementA.name,
                type: clash.elementA.ifcType,
                model: clash.elementA.sourceModel
            },
            elementB: {
                globalId: clash.elementB.globalId,
                name: clash.elementB.name,
                type: clash.elementB.ifcType,
                model: clash.elementB.sourceModel
            },
            clashType: clash.type,
            distance: clash.distance,
            location: clash.intersectionPoint,
            volume: clash.intersectionVolume,
            description: this.generateDescription(clash),
            disciplineA: this.getDiscipline(clash.elementA),
            disciplineB: this.getDiscipline(clash.elementB)
        };
    }
}

// Usage example
const service = new ClashDetectionService({ projectId: 'PRJ001' });
const results = await service.runClashTests(federatedModelId, [mepStructureTest]);
const bcfFile = await service.exportToBCF(results, { version: '3.0' });

Clash Grouping Best Practice

A single design issue can generate hundreds of individual clashes. For example, a duct running through a beam may create clashes with every segment. Effective clash management requires intelligent grouping to prevent overwhelming coordinators with duplicate issues. WIA-BIM recommends grouping clashes by spatial proximity, related elements, and common resolution approach.

4.7 Coordination Meeting Workflow

  • Pre-meeting preparation: Run all clash tests and filter results by priority
  • Issue assignment: Assign each clash group to responsible discipline
  • Resolution tracking: Document proposed solutions and deadlines
  • Verification: Re-run tests after model updates to confirm resolution
  • Documentation: Maintain audit trail of all clash resolution decisions
