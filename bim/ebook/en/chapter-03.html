Chapter 03

Data Exchange Formats (IFC)

Chapter Overview

Effective BIM requires seamless data exchange between different software platforms and project stakeholders. This chapter covers the Industry Foundation Classes (IFC) standard and other key data exchange formats that enable interoperability in the AEC industry.

Introduction to Open BIM

Open BIM is a universal approach to the collaborative design, realization, and operation of buildings based on open standards and workflows. The foundation of Open BIM is the ability to exchange building information using vendor-neutral formats that preserve data integrity and meaning.

While proprietary BIM software excels at authoring and manipulation, the AEC industry requires open standards to enable collaboration across the diverse software ecosystem. buildingSMART International leads the development of these open standards, with IFC serving as the cornerstone of interoperable BIM.

Data Exchange Formats Overview

IFC
Industry Foundation Classes
.ifc, .ifcxml, .ifczip

The primary open standard for BIM data exchange. Contains complete building geometry and properties in a vendor-neutral format.

BCF
BIM Collaboration Format
.bcf, .bcfzip

Standard for communication of coordination issues, clash reports, and model comments without transferring full model data.

COBie
Construction Operations Building
.xlsx, .xml

Spreadsheet-based format for facility management handover, capturing asset information for building operations.

gbXML
Green Building XML
.xml

Schema for transferring building properties needed for energy analysis and sustainability calculations.

Understanding IFC

Industry Foundation Classes (IFC) is an open, international standard (ISO 16739) for Building Information Model data exchange. IFC defines a data schema that represents building and construction industry data, enabling software applications to exchange and share building models regardless of which application was used to create them.

IFC Schema Architecture

The IFC schema is organized as a layered hierarchy, with abstract concepts at the top and concrete elements at the bottom. Understanding this structure is essential for effective IFC implementation.

IfcRoot - Base class for all IFC entities with identity
IfcObjectDefinition - Abstract generalization of objects
IfcObject - Independent objects with occurrence
IfcProduct - Physical or spatial elements
IfcElement - Building elements
IfcBuildingElement - Walls, Slabs, Doors, etc.

IFC Versions

Version Release Status Key Features
IFC2x3 2006 Legacy Widely implemented, stable baseline
IFC4 2013 Current ISO 16739:2013, improved geometry, 4D/5D support
IFC4.3 2022 Current Infrastructure extensions (rail, road, bridge)
IFC5 Planned Future Modular architecture, API-first design
WIA-BIM IFC Requirements

WIA-BIM mandates IFC4 as the minimum version for project deliverables. IFC2x3 is accepted only for legacy system integration with documented justification. All new projects should target IFC4 or IFC4.3 for infrastructure projects.

IFC Data Structure

IFC files contain structured data representing every aspect of a building model. Understanding this structure enables better control over export quality and troubleshooting of exchange issues.

/* Sample IFC File Structure (IFC4) */

ISO-10303-21;
HEADER;
FILE_DESCRIPTION(('ViewDefinition [CoordinationView]'),'2;1');
FILE_NAME('SampleBuilding.ifc','2025-01-15T10:30:00',('Architect'),
    ('WIA-BIM Certified Org'),'IFC4','Authoring Software','');
FILE_SCHEMA(('IFC4'));
ENDSEC;

DATA;
/* Project structure */
#1=IFCPROJECT('2nDvK9Vu95A8nZFqP6LKrz',#2,'Sample Project',$,$,$,$,(#20),#7);
#2=IFCOWNERHISTORY(#3,#6,$,.ADDED.,$,$,$,1705312200);
#3=IFCPERSONANDORGANIZATION(#4,#5,$);
#4=IFCPERSON($,'Designer','John',$,$,$,$,$);
#5=IFCORGANIZATION($,'WIA-BIM Org',$,$,$);
#6=IFCAPPLICATION(#5,'1.0','BIM Application','WIA-App');

/* Geometric context */
#7=IFCUNITASSIGNMENT((#8,#9,#10));
#8=IFCSIUNIT(*,.LENGTHUNIT.,.MILLI.,.METRE.);
#9=IFCSIUNIT(*,.AREAUNIT.,$,.SQUARE_METRE.);
#10=IFCSIUNIT(*,.VOLUMEUNIT.,$,.CUBIC_METRE.);

/* Building hierarchy */
#20=IFCGEOMETRICREPRESENTATIONCONTEXT($,'Model',3,1.E-05,#21,$);
#21=IFCAXIS2PLACEMENT3D(#22,$,$);
#22=IFCCARTESIANPOINT((0.,0.,0.));

/* Site */
#100=IFCSITE('3eDJk8Ll55x9A2xPqR7Mcd',#2,'Building Site',$,$,#101,$,$,.ELEMENT.,$,$,$,$,$);
#101=IFCLOCALPLACEMENT($,#102);
#102=IFCAXIS2PLACEMENT3D(#22,$,$);

/* Building */
#200=IFCBUILDING('1aFGh7Jk92C4bNoPqS5Tuv',#2,'Main Building',$,$,#201,$,$,.ELEMENT.,$,$,$);

/* Storey */
#300=IFCBUILDINGSTOREY('4rSTu2Vw36D8eXyZaB9Cde',#2,'Ground Floor',$,$,#301,$,$,.ELEMENT.,0.);

/* Wall example */
#400=IFCWALLSTANDARDCASE('5vWXy3Za47E9fGhIjK0Lmn',#2,'Exterior Wall',$,$,#401,#410,$);
#401=IFCLOCALPLACEMENT(#301,#402);

ENDSEC;
END-ISO-10303-21;

Model View Definitions (MVD)

Model View Definitions specify which parts of the IFC schema are required for particular use cases. MVDs ensure that IFC exports contain the necessary information for their intended purpose.

Reference View (RV)

Optimized for viewing, coordination, and reference purposes. Geometry is simplified for smaller file sizes and faster loading.

  • Use CaseModel viewing, coordination
  • GeometryTessellated (triangulated)
  • File SizeSmaller
Design Transfer View (DTV)

Preserves parametric geometry for editing in receiving applications. Ideal for design collaboration where models need modification.

  • Use CaseDesign collaboration, editing
  • GeometryParametric (B-rep, CSG)
  • File SizeLarger

IFC Validation

Validation ensures IFC files meet schema requirements and contain the expected information. WIA-BIM requires validation before file distribution.

📋
Syntax Validation

Verifies the IFC file conforms to the STEP physical file format specification and IFC schema syntax rules.

🏗️
Schema Validation

Checks that all entity instances comply with the IFC schema, including correct attribute types and cardinality.

📐
MVD Validation

Confirms the file meets requirements of the specified Model View Definition, including mandatory entities and properties.

✅
Business Rule Validation

Applies project-specific rules to verify naming conventions, required parameters, and organizational standards.

BCF for Issue Communication

BIM Collaboration Format (BCF) provides a lightweight method for communicating model issues without exchanging entire models. BCF files contain viewpoint information, comments, and issue tracking data.
