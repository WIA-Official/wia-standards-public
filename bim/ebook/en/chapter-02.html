Chapter 02

BIM Modeling Standards

Chapter Overview

This chapter establishes the fundamental modeling standards for WIA-BIM compliant projects. These standards ensure consistency, quality, and interoperability across all BIM deliverables, regardless of the software platform used.

Naming Conventions

Consistent naming conventions are essential for organizing and managing BIM data. WIA-BIM adopts a structured naming system that provides clarity and enables automated processing of model elements.

File Naming Standard

All BIM files must follow the WIA-BIM file naming convention to ensure proper identification and version control.

WIA-BIM File Naming Structure:
[Project]-[Discipline]-[Zone]-[Level]-[Type]-[Revision].[ext]

Components:
├── Project:     3-6 character project identifier
├── Discipline:  2-3 character discipline code
├── Zone:        Optional zone/building identifier
├── Level:       Level or floor identifier
├── Type:        Document/model type code
├── Revision:    Version identifier
└── Extension:   File format extension

Examples:
PROJ01-ARC-A-L02-MOD-R03.rvt    (Architectural Model, Building A, Level 2)
PROJ01-STR-ALL-MOD-R05.ifc      (Structural Model, All Levels, IFC Export)
PROJ01-MEP-B-L01-MOD-R02.rvt    (MEP Model, Building B, Level 1)

Discipline Codes

Code Discipline Description
ARC Architectural Building architecture, interior design
STR Structural Structural engineering elements
MEP Mechanical, Electrical, Plumbing Building services and systems
CIV Civil Site work, earthwork, utilities
LAN Landscape Landscape architecture
FED Federated Combined coordination model

Element Naming

Individual model elements must be named using a consistent pattern that identifies their category, type, and unique characteristics.

Element Naming Pattern:
[Category]_[Type]_[Material]_[Size/Specification]

Examples:
├── Wall_Exterior_Concrete_300mm
├── Door_Single_Wood_900x2100
├── Column_Circular_Steel_D400
├── Duct_Supply_Galvanized_400x300
├── Pipe_ColdWater_Copper_DN50
└── Window_Fixed_Aluminum_1200x1500

Coordinate System Standards

Proper coordinate system management is critical for model coordination and clash detection. All WIA-BIM projects must establish and maintain consistent coordinate systems.

Project Base Point (PBP)

    N (Y+)
      ↑
      │
      │    Project Base Point
      │    ┌───┐
      │    │ × │ ← (0,0,0) Project Origin
      │    └───┘
      │
 ─────┼──────────────→ E (X+)
      │
      │
    Elevation Reference: ±0.00m

The Project Base Point defines the local coordinate system for the project. All model elements are positioned relative to this point. The PBP typically aligns with a significant project feature such as the main building corner or site entrance.

Survey Point Requirements

The Survey Point establishes the relationship between the project coordinate system and real-world geographic coordinates. This connection is essential for:

  • Site coordination with civil engineering models
  • GIS integration and mapping applications
  • Construction layout and surveying
  • Regulatory compliance and permit submissions

Coordinate System Checklist

✓ Project Base Point established at logical project origin
✓ Survey Point linked to real-world coordinates (geodetic datum specified)
✓ True North rotation defined and documented
✓ All discipline models share identical coordinate systems
✓ Shared coordinates file distributed to all team members
✓ Level elevations reference consistent datum

Units and Measurement

WIA-BIM mandates the use of the International System of Units (SI) as the primary measurement system. Projects in regions using imperial units must maintain metric equivalents for interoperability.

Measurement Type Primary Unit Precision
Length (large) Meters (m) 3 decimal places
Length (detail) Millimeters (mm) 0 decimal places
Area Square meters (m²) 2 decimal places
Volume Cubic meters (m³) 3 decimal places
Angle Decimal degrees (°) 4 decimal places

Quality Assurance Standards

Model quality directly impacts the value derived from BIM throughout the project lifecycle. WIA-BIM establishes quality criteria that all models must meet.

Geometric Accuracy

All model elements must be placed with precision appropriate to their LOD. Elements must not float, overlap unintentionally, or extend beyond logical boundaries.

Data Completeness

Required parameters must be populated for all elements. Empty or placeholder values are not acceptable for mandated properties.

Naming Compliance

All elements, views, sheets, and families must follow the established naming conventions without deviation.

Model Performance

Models must maintain acceptable performance. File sizes and element counts must be monitored and optimized as needed.

// Model Quality Validation Script (Pseudocode)
function validateModel(model) {
    let issues = [];
    
    // Check naming conventions
    for (element of model.elements) {
        if (!element.name.matches(NAMING_PATTERN)) {
            issues.push({
                type: 'NAMING',
                element: element.id,
                message: 'Non-compliant element name'
            });
        }
    }
    
    // Check required parameters
    for (element of model.elements) {
        for (param of REQUIRED_PARAMS[element.category]) {
            if (element.getParameter(param) === null) {
                issues.push({
                    type: 'DATA',
                    element: element.id,
                    message: `Missing required parameter: ${param}`
                });
            }
        }
    }
    
    // Check geometric validity
    for (element of model.elements) {
        if (element.hasOverlap() || element.isFloating()) {
            issues.push({
                type: 'GEOMETRY',
                element: element.id,
                message: 'Geometric issue detected'
            });
        }
    }
    
    return {
        passed: issues.length === 0,
        issues: issues
    };
}
Common Modeling Errors to Avoid

1. Duplicated Elements: Elements copied in place, resulting in overlapping geometry and inflated quantities.

2. Incorrect Hosting: Elements hosted to wrong references, causing failures when references change.

3. Missing Constraints: Elements not properly constrained to levels or grids, leading to coordination issues.

4. Inconsistent Worksets: Elements placed in incorrect worksets, affecting visibility and performance.

View and Sheet Standards

Views and sheets extracted from BIM models must meet documentation standards for construction and regulatory submissions.

View Naming Convention:
[Level/Section]-[Discipline]-[View Type]-[Description]

Examples:
├── L01-ARC-FP-Overall           (Level 1 Architectural Floor Plan)
├── L02-STR-FP-Framing           (Level 2 Structural Framing Plan)
├── SEC-A-ARC-Building Section    (Architectural Building Section A)
├── DET-01-STR-Connection Detail  (Structural Detail 01)
└── 3D-FED-Coordination View      (Federated 3D Coordination)

Sheet Naming Convention:
[Discipline]-[Sheet Type][Number]-[Description]

Examples:
├── A-101-Ground Floor Plan
├── S-201-Foundation Plan
├── M-301-HVAC Floor Plan Level 1
└── E-401-Lighting Layout

By adhering to these modeling standards, WIA-BIM projects achieve the consistency and quality necessary for effective collaboration, automated analysis, and lifecycle value creation.
