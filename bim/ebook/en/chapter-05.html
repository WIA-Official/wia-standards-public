5.1 API Architecture Overview

The WIA-BIM API layer provides standardized programmatic interfaces for interacting with BIM data. This layer abstracts the underlying data format, enabling applications to work with building information without needing to understand the full schema complexity.

🔌 API Design Principles

  • RESTful Foundation: Resource-oriented architecture with standard HTTP methods
  • GraphQL Support: Flexible queries for complex data retrieval
  • Real-time Streams: WebSocket connections for live collaboration
  • Consistent Responses: Standardized response formats and error codes

5.1.1 API Layers

Layer Purpose Protocol Use Case
Query API Read operations REST GET / GraphQL Retrieve objects, search, filter
Mutation API Write operations REST POST/PUT/DELETE Create, update, delete objects
Stream API Real-time updates WebSocket Live collaboration, notifications
Validation API Data verification REST POST Schema validation, rule checking
Batch API Bulk operations REST POST Mass updates, imports, exports

Authorization: Bearer {token}

// Response

  "data": {

        "properties": {...}
      },
      {
        "guid": "wall-002-guid",
        "type": "Wall",
        "name": "Interior Partition P1",
        "properties": {...}
      }
    ],
    "pagination": {
      "total": 156,
      "page": 1,
      "pageSize": 50,
      "hasMore": true
    }
  }
}

// Get single object by GUID
GET /api/v1/projects/{projectId}/objects/{guid}

// Get object with related elements
GET /api/v1/projects/{projectId}/objects/{guid}?include=relationships,materials,geometry

// Spatial query - objects within bounding box
GET /api/v1/projects/{projectId}/objects?spatial=bbox&minX=0&minY=0&maxX=10000&maxY=8000&z=0

5.2.2 GraphQL Query Interface

// GraphQL Schema Definition
type Query {
  project(id: ID!): Project
  object(guid: ID!): BIMObject
  objects(filter: ObjectFilter, pagination: Pagination): ObjectConnection
  spaces(buildingId: ID!): [Space]
  searchObjects(query: String!, types: [ObjectType]): [BIMObject]
}

type BIMObject {
  guid: ID!
  type: ObjectType!
  name: String!
  description: String
  geometry: Geometry
  properties: PropertySet
  relationships: [Relationship]
  containedIn: SpatialElement
  materials: [MaterialLayer]
  metadata: Metadata
}

// Example GraphQL Query
query GetWallsWithDoors($projectId: ID!, $level: String) {
  objects(
    filter: {
      projectId: $projectId,
      type: WALL,
      level: $level
    }
  ) {
    edges {
      node {
        guid
        name
        properties {
          isExternal
          fireRating
        }
        relationships(type: VOIDS) {
          relatedObjects {
            guid
            type
            name
          }
        }
        geometry {
          boundingBox {
            min { x y z }
            max { x y z }
          }
        }
      }
    }
    pageInfo {
      hasNextPage
      endCursor
    }
  }
}

5.2.3 Query Filters and Operators

Operator Description Example
eq Equals type eq 'Door'
ne Not equals status ne 'Demolished'
gt, gte Greater than (or equal) fireRating gte 60
lt, lte Less than (or equal) height lte 2400
contains String contains name contains 'External'
in Value in list type in ['Door', 'Window']
within Spatial within geometry within $boundingBox
intersects Spatial intersection geometry intersects $polygon

5.3.2 Update Operations

// Update object properties
PATCH /api/v1/projects/{projectId}/objects/{guid}
Content-Type: application/json

{
  "properties": {
    "fireRating": 90,
    "acousticRating": "STC-45"
  }
}

// Move object to new location
PATCH /api/v1/projects/{projectId}/objects/{guid}/geometry
Content-Type: application/json

{
  "transform": {
    "translation": {"x": 500, "y": 0, "z": 0}
  }
}

// Batch update multiple objects
POST /api/v1/projects/{projectId}/objects/batch-update
Content-Type: application/json

{
  "updates": [
    {
      "guid": "wall-001",
      "properties": {"status": "ForConstruction"}
    },
    {
      "guid": "wall-002",
      "properties": {"status": "ForConstruction"}
    }
  ],
  "options": {
    "validateAll": true,
    "atomicTransaction": true
  }
}

5.4 Streaming API

5.4.1 WebSocket Connection

// Establish WebSocket connection
const ws = new WebSocket('wss://api.wia-bim.org/v1/stream');

ws.onopen = () => {
  // Authenticate
  ws.send(JSON.stringify({
    type: 'auth',
    token: 'Bearer your-jwt-token'
  }));

  // Subscribe to project changes
  ws.send(JSON.stringify({
    type: 'subscribe',
    channel: 'project:prj-001',
    events: ['object.created', 'object.updated', 'object.deleted']
  }));
};

ws.onmessage = (event) => {
  const message = JSON.parse(event.data);

  switch (message.type) {
    case 'object.created':
      handleObjectCreated(message.data);
      break;
    case 'object.updated':
      handleObjectUpdated(message.data);
      break;
    case 'object.deleted':
      handleObjectDeleted(message.data);
      break;
  }
};

// Incoming message format
{
  "type": "object.updated",
  "timestamp": "2025-01-20T14:30:00.123Z",
  "project": "prj-001",
  "data": {
    "guid": "wall-001",
    "changes": {
      "properties.fireRating": {
        "old": 60,
        "new": 90
      }
    },
    "author": "user@company.com",
    "version": 15
  }
}

5.4.2 Event Types

Event Trigger Payload
object.created New object added Full object data
object.updated Object modified Changed fields only
object.deleted Object removed GUID and metadata
relationship.created New relationship Relationship data
conflict.detected Edit conflict Conflicting versions
validation.failed Rule violation Validation errors

5.5 Validation API

5.5.1 Schema Validation

// Validate object before creation
POST /api/v1/validate/object
Content-Type: application/json

{
  "type": "Door",
  "name": "Office Door",
  "properties": {
    "width": 900,
    "height": 2100,
    "fireRating": 60
  }
}

// Validation response
{
  "valid": true,
  "warnings": [
    {
      "path": "properties.acousticRating",
      "message": "Acoustic rating recommended for office doors",
      "severity": "warning"
    }
  ]
}

// Invalid object response
{
  "valid": false,
  "errors": [
    {
      "path": "properties.width",
      "message": "Width must be between 600 and 1200 for standard doors",
      "code": "RANGE_ERROR"
    }
  ]
}

5.5.2 Business Rule Validation

// Validate against project rules
POST /api/v1/projects/{projectId}/validate/rules
Content-Type: application/json

{
  "objects": ["wall-001", "door-001", "door-002"],
  "rulesets": ["fire-safety", "accessibility", "energy-efficiency"]
}

// Response
{
  "valid": false,
  "results": [
    {
      "ruleset": "fire-safety",
      "passed": false,
      "violations": [
        {
          "rule": "FS-101",
          "message": "Exit door requires 90-minute fire rating",
          "object": "door-001",
          "severity": "error"
        }
      ]
    },
    {
      "ruleset": "accessibility",
      "passed": true,
      "violations": []
    }
  ]
}

5.6 Error Handling

5.6.1 Standard Error Response

{
  "success": false,
  "error": {
    "code": "OBJECT_NOT_FOUND",
    "message": "Object with GUID 'xyz-123' not found in project",
    "details": {
      "guid": "xyz-123",
      "projectId": "prj-001"
    },
    "timestamp": "2025-01-20T14:30:00Z",
    "requestId": "req-abc-123"
  }
}

5.6.2 Error Codes

Code HTTP Status Description
UNAUTHORIZED 401 Invalid or missing authentication
FORBIDDEN 403 Insufficient permissions
OBJECT_NOT_FOUND 404 Object does not exist
VALIDATION_ERROR 422 Invalid data format
CONFLICT 409 Version conflict
RATE_LIMITED 429 Too many requests
