Chapter 1: IFC Standards and Data Schemas

Industry Foundation Classes for Open BIM Data Exchange

1.1 Introduction to IFC

The Industry Foundation Classes (IFC) represent the cornerstone of open BIM data exchange. Developed and maintained by buildingSMART International, IFC provides a vendor-neutral, standardized data schema that enables seamless interoperability between different BIM software applications. IFC is registered as ISO 16739, establishing it as a global standard for building and construction data.

IFC defines not just geometry but a rich semantic model encompassing building elements, relationships, properties, processes, and actors involved in the building lifecycle. This comprehensive approach enables true data exchange rather than mere geometric transfer, preserving the intelligence embedded in BIM models across different platforms and phases of a project.

Why IFC Matters

Without IFC, each BIM software would exist as an isolated silo, requiring manual re-entry of data when moving between platforms. IFC breaks down these barriers, enabling architects, engineers, and contractors to collaborate using their preferred tools while maintaining data integrity throughout the project lifecycle.

1.2 IFC Schema Architecture

The IFC schema is organized into four conceptual layers, each building upon the one below it. This layered architecture provides flexibility while maintaining consistency across different building domains and project types. 

IFC Schema Layer Architecture
=============================

                      |
+--------------------------------------------------+

1.4 EXPRESS Schema Language

IFC is formally defined using the EXPRESS data modeling language (ISO 10303-11). EXPRESS provides a rigorous, unambiguous definition of data structures, relationships, and constraints that ensure consistent implementation across different software platforms. 

-- EXPRESS Schema Example: IfcWall Definition
-- ISO 10303-11 EXPRESS Language

ENTITY IfcWall
  SUPERTYPE OF (ONEOF
    (IfcWallStandardCase, IfcWallElementedCase))
  SUBTYPE OF (IfcBuildingElement);
    PredefinedType : OPTIONAL IfcWallTypeEnum;
  WHERE
    CorrectPredefinedType : NOT(EXISTS(PredefinedType)) OR
      (PredefinedType <> IfcWallTypeEnum.USERDEFINED) OR
      ((PredefinedType = IfcWallTypeEnum.USERDEFINED) AND
       EXISTS(SELF\IfcObject.ObjectType));
    CorrectTypeAssigned : (SIZEOF(IsTypedBy) = 0) OR
      ('IFC4X3.IFCWALLTYPE' IN TYPEOF(SELF\IfcObject.IsTypedBy[1].RelatingType));
END_ENTITY;

ENTITY IfcWallType
  SUBTYPE OF (IfcBuildingElementType);
    PredefinedType : IfcWallTypeEnum;
  WHERE
    CorrectPredefinedType : (PredefinedType <> IfcWallTypeEnum.USERDEFINED) OR
      ((PredefinedType = IfcWallTypeEnum.USERDEFINED) AND
       EXISTS(SELF\IfcElementType.ElementType));
END_ENTITY;

TYPE IfcWallTypeEnum = ENUMERATION OF
    (MOVABLE,
     PARAPET,
     PARTITIONING,
     PLUMBINGWALL,
     SHEAR,
     SOLIDWALL,
     STANDARD,
     POLYGONAL,
     ELEMENTEDWALL,
     RETAININGWALL,
     WAVEWALL,
     USERDEFINED,
     NOTDEFINED);
END_TYPE;

1.5 IFC File Formats

IFC data can be serialized in multiple file formats, each suited to different use cases and performance requirements.

1.5.1 STEP Physical File (SPF)

The traditional IFC format uses STEP Part 21 encoding (.ifc extension). This human-readable text format is widely supported and excellent for debugging and validation, though file sizes can be large for complex models. 

/* IFC-SPF Example: Simple Wall Instance */
ISO-10303-21;
HEADER;
FILE_DESCRIPTION(('WIA-BIM Example'),'2;1');
FILE_NAME('example.ifc','2025-01-15T10:30:00',('Architect'),('WIA'),'WIA-BIM SDK','IFC4X3_ADD2','');
FILE_SCHEMA(('IFC4X3_ADD2'));
ENDSEC;

DATA;
#1=IFCPROJECT('2m6OwYT5z9tQJy',#2,'Sample Project',$,$,$,$,(#10),#5);
#2=IFCOWNERHISTORY(#3,#4,$,.NOCHANGE.,$,$,$,1705312200);
#3=IFCPERSONANDORGANIZATION(#6,#7,$);
#4=IFCAPPLICATION(#7,'1.0','WIA-BIM SDK','WIA-BIM');
#5=IFCUNITASSIGNMENT((#8,#9));
#6=IFCPERSON($,'Smith','John',$,$,$,$,$);
#7=IFCORGANIZATION($,'WIA','World Industry Association',$,$);
#8=IFCSIUNIT(*,.LENGTHUNIT.,$,.METRE.);
#9=IFCSIUNIT(*,.AREAUNIT.,$,.SQUARE_METRE.);
#10=IFCGEOMETRICREPRESENTATIONCONTEXT($,'Model',3,1.0E-05,#11,$);
#11=IFCAXIS2PLACEMENT3D(#12,$,$);
#12=IFCCARTESIANPOINT((0.,0.,0.));

/* Wall Instance */
#100=IFCWALL('3vB2YO$5X4xOQ8',#2,'Wall-001','Exterior Wall',$,#101,#110,$,.STANDARD.);
#101=IFCLOCALPLACEMENT($,#102);
#102=IFCAXIS2PLACEMENT3D(#103,#104,#105);
#103=IFCCARTESIANPOINT((0.,0.,0.));
#104=IFCDIRECTION((0.,0.,1.));
#105=IFCDIRECTION((1.,0.,0.));
#110=IFCPRODUCTDEFINITIONSHAPE($,$,(#111));
#111=IFCSHAPEREPRESENTATION(#10,'Body','SweptSolid',(#112));
#112=IFCEXTRUDEDAREASOLID(#113,#11,#104,3000.);
#113=IFCRECTANGLEPROFILEDEF(.AREA.,$,$,200.,6000.);

ENDSEC;
END-ISO-10303-21;

1.6 IFC Version History

The IFC schema has evolved significantly since its initial release, with each version expanding capabilities and addressing industry needs.

Version Year Key Features ISO Status
IFC 1.0 1997 Initial release, basic building elements -
IFC 2x3 2006 Most widely deployed, stable schema ISO 16739:2013
IFC 4 2013 Enhanced geometry, parametric elements ISO 16739-1:2018
IFC 4.3 2024 Infrastructure, rail, roads, bridges ISO 16739-1:2024

WIA-BIM Compliance

WIA-BIM mandates support for IFC 4.3 as the baseline requirement, with backward compatibility for IFC 2x3 models. All WIA-certified BIM software must pass the buildingSMART IFC certification program to ensure reliable data exchange.

1.7 API Integration Example

// WIA-BIM SDK: Complete IFC Model Operations
import { WIABimClient, IfcModel, IfcWall } from '@wia/bim-sdk';

class IFCModelManager {
    private client: WIABimClient;
    private model: IfcModel;

    constructor(config: { projectId: string; endpoint: string }) {
        this.client = new WIABimClient(config);
    }

    async loadModel(filePath: string): Promise<void> {
        this.model = await this.client.loadModel(filePath);
        console.log(`Loaded model: ${this.model.getProject().name}`);
        console.log(`Schema version: ${this.model.schemaVersion}`);
    }

    async analyzeWalls(): Promise<WallAnalysis[]> {
        const walls = await this.model.getElementsByType('IfcWall');
        const analysis: WallAnalysis[] = [];

        for (const wall of walls) {
            const geometry = await wall.getGeometry();
            const properties = await wall.getPropertySets();
            const materials = await wall.getMaterials();

            analysis.push({
                globalId: wall.globalId,
                name: wall.name,
                length: geometry.boundingBox.length,
                height: geometry.boundingBox.height,
                thickness: geometry.boundingBox.width,
                area: geometry.surfaceArea,
                volume: geometry.volume,
                materials: materials.map(m => m.name),
                thermalTransmittance: properties['Pset_WallCommon']?.ThermalTransmittance
            });
        }

        return analysis;
    }

    async exportToJSON(outputPath: string): Promise<void> {
        const jsonData = await this.model.toJSON();
        await this.client.writeFile(outputPath, JSON.stringify(jsonData, null, 2));
    }
}

// Usage
const manager = new IFCModelManager({
    projectId: 'building-001',
    endpoint: 'https://api.wia-bim.org'
});

await manager.loadModel('architecture.ifc');
const wallAnalysis = await manager.analyzeWalls();
console.log(`Analyzed ${wallAnalysis.length} walls`);
