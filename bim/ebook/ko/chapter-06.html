6.1 프로토콜 레이어 개요

프로토콜 레이어는 WIA-BIM 데이터가 분산 시스템 간에 교환, 동기화, 관리되는 방식을 정의합니다. 이 레이어는 BIM 프로젝트의 모든 참여자 간에 신뢰할 수 있고, 안전하며, 일관된 통신을 보장합니다.

📡 프로토콜 설계 목표

  • 신뢰성: 확인 응답을 통한 보장된 메시지 전달
  • 일관성: 모든 참여자가 동일한 데이터 상태 확인
  • 보안: 종단간 암호화 및 인증
  • 효율성: 델타 전송으로 최소 대역폭
  • 확장성: 수천 명의 동시 사용자 지원

6.3 동기화 프로토콜

6.3.1 동기화 모드

모드 설명 용도 지연
실시간 즉시 전파 공동 저작 세션 < 100ms
준실시간 주기적 동기화 팀 협업 1-5분
배치 예약된 동기화 조직 간 교환 시간/일
온디맨드 수동 트리거 마일스톤 전달 트리거 시

6.5 충돌 해결

6.5.1 충돌 유형

유형 설명 해결 전략
속성 충돌 동일 속성, 다른 값 자동 병합 또는 수동
형상 충돌 겹치는 수정 시각화 + 수동
관계 충돌 호환되지 않는 관계 규칙 기반 해결
스키마 충돌 다른 스키마 버전 마이그레이션 + 검증
의미론적 충돌 논리적 불일치 전문가 검토

6.6.2 데이터 암호화

레이어 암호화 키 관리
전송 TLS 1.3 인증 기관
메시지 AES-256-GCM 세션 키
저장 AES-256 고객 관리 키
패키지 RSA-4096 + AES 수신자 공개 키

6.7 장 요약

핵심 내용

  • 교환 프로토콜은 당사자 간 신뢰할 수 있고 검증된 데이터 전송 보장
  • 동기화는 다양한 워크플로우를 위해 실시간부터 배치 모드까지 지원
  • 델타 동기화는 변경 사항만 전송하여 대역폭 최소화
  • 브랜칭을 통한 버전 제어는 병렬 개발 트랙 가능
  • 충돌 해결은 자동화된 규칙과 인간 감독 결합
  • 보안 프로토콜은 인증, 권한 부여, 암호화 제공
