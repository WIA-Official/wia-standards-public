Chapter 02: 모델링 표준

BIM 모델의 품질과 일관성을 보장하기 위한 핵심 표준인 LOD(Level of Development), IFC(Industry Foundation Classes), COBie(Construction Operations Building Information Exchange)에 대해 상세히 알아봅니다. 이 표준들은 BIM 프로젝트의 성공적 수행을 위한 기반입니다.

2.1 LOD (Level of Development)

LOD(Level of Development)는 BIM 요소의 개발 수준을 정의하는 프레임워크입니다. 미국 건축사협회(AIA)에서 정의한 이 개념은 모델 요소가 얼마나 상세하게 개발되었는지, 그리고 해당 정보가 얼마나 신뢰할 수 있는지를 나타냅니다.

LOD의 두 가지 측면

LOG (Level of Geometry): 형상의 상세도 - 모델 요소가 얼마나 정밀하게 형상화되었는가

LOI (Level of Information): 정보의 상세도 - 모델 요소에 얼마나 많은 속성 정보가 포함되었는가

LOD = LOG + LOI, 즉 형상과 정보가 함께 개발 수준을 결정합니다.

LOD 단계별 정의

LOD 100 - 개념 설계

요소의 존재 여부를 나타내는 수준. 매스 모델, 기호, 또는 일반적 표현. 면적, 높이, 체적, 위치, 방향 등 개략적 정보만 포함.

LOD 200 - 기본 설계

일반화된 형상, 대략적 수량, 크기, 형태, 위치, 방향 정보 포함. 그래픽 요소가 공간 내에서 대략적 위치와 형상으로 표현됨.

LOD 300 - 실시 설계

정확한 수량, 크기, 형태, 위치, 방향이 포함된 특정 시스템, 객체, 조립체로 표현. 비형상 정보도 요소에 연결 가능.

LOD 350 - 시공 문서

LOD 300 + 다른 건물 시스템과의 인터페이스 포함. 지지부, 연결부 등이 모델링되어 분야 간 조율에 사용 가능.

LOD 400 - 제작/조립

제작, 조립, 설치에 필요한 상세 정보 포함. 제작 도면 수준의 상세도와 정확한 크기, 형태, 위치, 수량, 방향 정보.

LOD 500 - 준공 (As-Built)

실제 시공된 상태를 반영한 현장 검증된 모델. 시설 관리(FM)에 필요한 모든 정보가 포함된 최종 상태.

2.2 IFC (Industry Foundation Classes)

IFC는 buildingSMART에서 개발한 개방형 국제 표준 데이터 형식입니다. 서로 다른 BIM 소프트웨어 간의 정보 교환을 가능하게 하며, ISO 16739로 국제 표준화되어 있습니다.

IFC의 목적

건설 및 시설 관리 산업에서 다양한 소프트웨어 애플리케이션 간의 상호운용성을 제공하는 것입니다. 특정 벤더에 종속되지 않는 중립적인 데이터 형식으로, openBIM의 핵심 기술입니다.

IFC 데이터 구조

IFC는 계층적 객체 지향 데이터 모델을 사용합니다. 최상위에 IfcRoot가 있고, 그 아래 다양한 엔티티 클래스가 정의되어 있습니다.

버전 발표 연도 주요 특징

2.3 COBie (Construction Operations Building Information Exchange)

COBie는 시설 관리에 필요한 정보를 교환하기 위한 표준 형식입니다. BIM 모델에서 FM(Facility Management)에 필요한 데이터를 추출하여 스프레드시트 형태로 제공합니다.

COBie의 필요성

BIM 모델에는 방대한 정보가 포함되어 있지만, 시설 관리자에게는 불필요한 정보도 많습니다. COBie는 FM에 필수적인 정보만 추출하여 간편하게 전달하는 역할을 합니다. 특히 BIM 소프트웨어가 없는 환경에서도 Excel로 정보 활용이 가능합니다.

2.4 분류 체계

BIM에서 분류 체계는 모델 요소를 체계적으로 구분하고 관리하기 위한 기준입니다. 국제적으로 OmniClass, UniFormat, MasterFormat 등이 사용되며, 국내에서는 건설정보분류체계가 활용됩니다.

분류 체계 특징 주요 용도
OmniClass 15개 테이블, 포괄적 분류 BIM 요소 분류, 정보 관리
UniFormat 기능별 분류 비용 산정, 성능 분석
MasterFormat 시공 분야별 분류 시방서, 입찰, 계약

2.5 모델 품질 검증

BIM 모델의 품질을 보장하기 위해서는 체계적인 검증이 필요합니다. 모델 품질 검증은 기하학적 정확성, 정보 완전성, 표준 준수 여부 등 다양한 측면을 확인합니다.

BIM 모델 품질 검증 항목
  • 형상 검증: 충돌, 간섭, 중복 요소, 열린 형상
  • 정보 검증: 필수 속성, 분류 코드, 명명 규칙
  • 표준 검증: LOD, IFC 스키마, 프로젝트 표준
  • 좌표 검증: 원점, 좌표계, 레벨 정확성

2.6 명명 규칙

일관된 명명 규칙은 BIM 프로젝트의 정보 관리에 필수적입니다. 파일, 모델 요소, 뷰, 시트 등 모든 항목에 대한 체계적인 명명 규칙이 필요합니다.

BIM 파일 명명 규칙 예시
=====================================

[프로젝트]-[건물]-[분야]-[단계]-[일련번호]

예시:
PRJ001-BLD01-ARC-DD-001.rvt
PRJ001-BLD01-STR-DD-001.rvt
PRJ001-BLD01-MEP-DD-001.rvt

구성 요소:
• PRJ001  : 프로젝트 코드
• BLD01   : 건물 식별자
• ARC/STR/MEP : 분야 코드
• DD      : 단계 (SD/DD/CD/CA)
• 001     : 일련번호

분야 코드:
• ARC : 건축 (Architecture)
• STR : 구조 (Structure)
• MEP : 기계/전기/배관 (Mechanical/Electrical/Plumbing)
• CIV : 토목 (Civil)
• LND : 조경 (Landscape)

모델 요소 명명 규칙

모델 요소의 명명은 타입명과 인스턴스명을 구분하여 관리합니다. 타입명은 제품 유형의 특성을, 인스턴스명은 개별 요소의 위치나 용도를 반영합니다.

2.8 정리

BIM 모델링 표준은 프로젝트 참여자 간의 원활한 협업과 정보 교환을 위한 기반입니다. LOD, IFC, COBie를 이해하고 올바르게 적용하는 것이 성공적인 BIM 프로젝트의 첫걸음입니다.

핵심 요약
  • LOD는 모델 요소의 개발 수준을 정의 (100~500)
  • IFC는 개방형 데이터 교환 표준 (ISO 16739)
  • COBie는 시설 관리용 정보 교환 형식
  • 분류 체계는 요소의 체계적 관리 기준
  • 품질 검증으로 모델 신뢰성 확보
  • 명명 규칙과 좌표계 설정은 협업의 기반
