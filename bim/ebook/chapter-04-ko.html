4장: 간섭 검토 및 조정

자동 충돌 식별 및 다분야 조정

4.1 간섭 검토 소개

간섭 검토(Clash Detection)는 프로젝트 팀이 시공 전에 건물 시스템 간의 충돌을 식별하고 해결할 수 있게 하는 BIM의 가장 가치 있는 기능 중 하나입니다. 전통적인 2D 워크플로우는 종종 시공 중에 간섭이 발견되어 비용이 많이 드는 재작업, 일정 지연 및 조정 실패로 이어졌습니다. BIM 기반 간섭 검토는 이러한 사후 대응 방식을 사전 예방 방식으로 변환합니다.

간섭은 둘 이상의 건물 요소가 동일한 물리적 공간을 점유하거나 필요한 여유 공간이 위반될 때 발생합니다. 이러한 충돌은 명백한 하드 클래시(물리적 교차)에서 더 미묘한 소프트 클래시(여유 공간 위반) 및 워크플로우 클래시(순서 충돌)에 이르기까지 다양합니다. 이러한 문제의 조기 발견과 해결은 시공 비용을 크게 줄이고 프로젝트 결과를 개선합니다.

간섭의 비용 영향

업계 연구에 따르면 감지되지 않은 간섭은 설계 단계에서 해결하는 것보다 시공 중에 해결하는 데 10-30배 더 많은 비용이 들 수 있습니다. 현장에서 발견된 단일 주요 간섭은 몇 주의 지연과 수십만 달러의 추가 비용을 초래할 수 있습니다.

4.2 간섭 유형

간섭 유형 설명 예시 검출 방법
하드 클래시 두 요소의 물리적 교차 보를 통과하는 덕트, 벽을 통과하는 배관 기하학 교차
소프트 클래시 (여유) 필요한 여유 공간 위반 불충분한 유지보수 접근, 법규 여유 버퍼 영역 분석
4D 클래시 (시간) 시공 순서 충돌 접근 경로 전에 설치된 장비 일정 연계 분석
워크플로우 클래시 설계 의도 또는 표준 위반 접근성 문제, 법규 비준수 규칙 기반 검사
중복 클래시 여러 당사자가 동일한 요소 모델링 이중 계산된 벽, 겹치는 그리드 ID 매칭

4.4 간섭 테스트 설정

// WIA-BIM 간섭 테스트 설정 스키마
interface ClashTestConfig {
    testId: string;
    name: string;
    description?: string;
    selectionA: ElementSelection;      // 첫 번째 선택 세트
    selectionB: ElementSelection;      // 두 번째 선택 세트
    testType: ClashTestType;
    tolerance: ToleranceSettings;
    exclusions: ExclusionRule[];
    priority: ClashPriority;
}

interface ElementSelection {
    source: 'MODEL' | 'SAVED_SELECTION' | 'SEARCH';
    modelIds?: string[];
    ifcTypes?: string[];               // 예: ['IfcDuctSegment', 'IfcPipeSegment']
    propertyFilters?: PropertyFilter[];
    spatialFilters?: SpatialFilter[];
}

interface ToleranceSettings {
    clashType: 'HARD' | 'SOFT' | 'CLEARANCE';
    hardTolerance: number;             // mm - 이보다 작은 간섭 무시
    clearanceDistance?: number;        // mm - 소프트 클래시에 필요한 여유
    touchTolerance: number;            // mm - 이 범위 내의 요소는 "접촉"
}

interface ExclusionRule {
    type: 'SAME_ELEMENT' | 'SAME_GROUP' | 'CONNECTED' | 'CONTAINER' | 'CUSTOM';
    description: string;
    condition?: string;                // 사용자 정의 조건 표현식
}

type ClashTestType = 'INTERSECTION' | 'CLEARANCE' | 'DUPLICATE' | 'HARD_SOFT';
type ClashPriority = 'CRITICAL' | 'HIGH' | 'MEDIUM' | 'LOW';

// 예시: MEP vs 구조 간섭 테스트
const mepStructureTest: ClashTestConfig = {
    testId: 'CT-001',
    name: 'MEP vs 구조 하드 클래시',
    description: 'MEP 시스템과 구조 요소 간의 교차 검출',
    selectionA: {
        source: 'MODEL',
        ifcTypes: ['IfcDuctSegment', 'IfcPipeSegment', 'IfcCableCarrierSegment'],
        propertyFilters: [{
            propertySet: 'Pset_*Common',
            property: 'Status',
            operator: 'NOT_EQUALS',
            value: 'DEMOLISHED'
        }]
    },
    selectionB: {
        source: 'MODEL',
        ifcTypes: ['IfcBeam', 'IfcColumn', 'IfcSlab', 'IfcWall'],
        propertyFilters: [{
            propertySet: 'Pset_*Common',
            property: 'LoadBearing',
            operator: 'EQUALS',
            value: true
        }]
    },
    testType: 'HARD_SOFT',
    tolerance: {
        clashType: 'HARD',
        hardTolerance: 10,           // 10mm 미만 간섭 무시
        clearanceDistance: 50,       // 50mm 여유 필요
        touchTolerance: 1            // 1mm 접촉 공차
    },
    exclusions: [
        { type: 'SAME_ELEMENT', description: '자기 간섭 무시' },
        { type: 'CONNECTED', description: '설계된 연결 무시' }
    ],
    priority: 'CRITICAL'
};

4.5 BCF (BIM 협업 형식)

BCF(BIM Collaboration Format)는 간섭 검토 결과를 포함한 BIM 이슈를 전달하기 위한 개방형 표준입니다. BCF는 간섭 보고서가 서로 다른 소프트웨어 플랫폼 간에 교환될 수 있게 하여, 한 도구에서 식별된 이슈가 다른 도구에서 검토되고 해결될 수 있도록 합니다.

4.6 간섭 검토 API

// WIA-BIM SDK: 간섭 검토 서비스
import { WIABimClient, ClashEngine, BCFExporter } from '@wia/bim-sdk';

class ClashDetectionService {
    private client: WIABimClient;
    private engine: ClashEngine;

    constructor(config: ServiceConfig) {
        this.client = new WIABimClient(config);
        this.engine = new ClashEngine();
    }

    async runClashTests(
        federatedModelId: string,
        tests: ClashTestConfig[]
    ): Promise<ClashTestResults> {
        // 통합 모델 로드
        const model = await this.client.loadFederatedModel(federatedModelId);

        const results: ClashTestResult[] = [];

        for (const test of tests) {
            // 요소 선택 조회
            const selectionA = await this.resolveSelection(model, test.selectionA);
            const selectionB = await this.resolveSelection(model, test.selectionB);

            // 간섭 검출 실행
            const clashes = await this.engine.detectClashes({
                elementsA: selectionA,
                elementsB: selectionB,
                testType: test.testType,
                tolerance: test.tolerance,
                exclusions: test.exclusions
            });

            results.push({
                testId: test.testId,
                testName: test.name,
                priority: test.priority,
                clashCount: clashes.length,
                clashes: clashes.map(c => this.formatClash(c, test))
            });
        }

        return {
            modelId: federatedModelId,
            timestamp: new Date().toISOString(),
            testResults: results,
            summary: this.generateSummary(results)
        };
    }

    async exportToBCF(
        results: ClashTestResults,
        options: BCFExportOptions
    ): Promise<Buffer> {
        const exporter = new BCFExporter(options);

        for (const testResult of results.testResults) {
            for (const clash of testResult.clashes) {
                await exporter.addTopic({
                    title: `${clash.elementA.name} vs ${clash.elementB.name}`,
                    description: clash.description,
                    priority: this.mapPriority(testResult.priority),
                    topicType: 'Clash',
                    labels: ['Coordination', clash.disciplineA, clash.disciplineB],
                    viewpoint: await this.generateViewpoint(clash),
                    relatedElements: [clash.elementA.globalId, clash.elementB.globalId]
                });
            }
        }

        return await exporter.export();
    }
}

// 사용 예시
const service = new ClashDetectionService({ projectId: 'PRJ001' });
const results = await service.runClashTests(federatedModelId, [mepStructureTest]);
const bcfFile = await service.exportToBCF(results, { version: '3.0' });
console.log(`${results.summary.totalClashes}개 간섭 발견`);

간섭 그룹화 모범 사례

단일 설계 문제가 수백 개의 개별 간섭을 생성할 수 있습니다. 예를 들어, 보를 통과하는 덕트는 모든 세그먼트와 간섭을 생성할 수 있습니다. 효과적인 간섭 관리는 코디네이터가 중복 이슈로 압도당하는 것을 방지하기 위해 지능적인 그룹화가 필요합니다. WIA-BIM은 공간적 근접성, 관련 요소 및 공통 해결 방법에 따라 간섭을 그룹화할 것을 권장합니다.

4.7 조정 회의 워크플로우

  • 회의 전 준비: 모든 간섭 테스트를 실행하고 우선순위별로 결과 필터링
  • 이슈 할당: 각 간섭 그룹을 책임 분야에 할당
  • 해결 추적: 제안된 솔루션과 마감일 문서화
  • 검증: 모델 업데이트 후 해결 확인을 위해 테스트 재실행
  • 문서화: 모든 간섭 해결 결정의 감사 추적 유지
