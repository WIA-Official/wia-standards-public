Python Example
from wia_standard import Client, ValidationError

# Initialize client with configuration
client = Client(
    api_key="your-api-key",
    environment="production",
    timeout=30
)

try:
    # Create resource with validation
    resource = client.create_resource({
        "name": "Example Resource",
        "type": "standard-compliant",
        "metadata": {
            "version": "1.0",
            "created_by": "user@example.com"
        }
    })

    # Automatic compliance validation
    validation_result = client.validate(resource)

    if validation_result.is_valid:
        print(f"Resource created successfully: {resource.id}")
    else:
        print(f"Validation warnings: {validation_result.warnings}")

except ValidationError as e:
    print(f"Validation failed: {e.message}")
    print(f"Details: {e.details}")

Java Example
import com.wia.standard.*;

public class WIAExample {
    public static void main(String[] args) {
        // Configuration using builder pattern
        WIAClient client = WIAClient.builder()
            .apiKey(System.getenv("WIA_API_KEY"))
            .environment(Environment.PRODUCTION)
            .connectionTimeout(Duration.ofSeconds(30))
            .build();

        try {
            // Create resource with validation
            Resource resource = Resource.builder()
                .name("Example Resource")
                .type("standard-compliant")
                .metadata(Map.of(
                    "version", "1.0",
                    "createdBy", "user@example.com"
                ))
                .build();

            Resource created = client.createResource(resource);
            System.out.println("Created: " + created.getId());

        } catch (ValidationException e) {
            System.err.println("Validation failed: " + e.getMessage());
        } finally {
            client.close();
        }
    }
}

Phase 2: Metadata Schema

WIA-HERITAGE-001

5.1 Importance of Metadata

Metadata is the essential element that carries the context and meaning of digital cultural heritage. High-quality images or 3D models without proper metadata are merely collections of bits. The WIA-HERITAGE metadata schema integrates technical information and cultural context to ensure long-term understanding and utilization.

Effective metadata improves discoverability, proves provenance, manages access rights, and tracks preservation history. This allows digital objects to retain their value and meaning even when separated from the original.

📋 Core Metadata Functions
  • Discovery: Supports search and browsing
  • Provenance: Origin and ownership history
  • Context: Cultural and historical background
  • Rights: Copyright, access restrictions
  • Preservation: Technical characteristics, transformation history

5.2 WIA-HERITAGE Schema Structure

The WIA-HERITAGE metadata schema is based on Dublin Core and integrates CIDOC-CRM's event-centric model with cultural heritage-specific extensions.

All text fields support multiple languages. The original language is prioritized, with English translation and romanization provided together. This ensures both international accessibility and cultural accuracy.
