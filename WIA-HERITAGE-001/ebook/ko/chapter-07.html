API 사양

코드 예제 및 패턴

인기 있는 언어의 일반적인 구현 패턴 및 코드 샘플:

Python 예제
from wia_standard import Client, ValidationError

# 구성으로 클라이언트 초기화
client = Client(
    api_key="your-api-key",
    environment="production",
    timeout=30
)

try:
    # 검증과 함께 리소스 생성
    resource = client.create_resource({
        "name": "예제 리소스",
        "type": "standard-compliant",
        "metadata": {
            "version": "1.0",
            "created_by": "user@example.com"
        }
    })

    # 자동 준수 검증
    validation_result = client.validate(resource)

    if validation_result.is_valid:
        print(f"리소스가 성공적으로 생성되었습니다: {resource.id}")
    else:
        print(f"검증 경고: {validation_result.warnings}")

except ValidationError as e:
    print(f"검증 실패: {e.message}")
    print(f"세부사항: {e.details}")

Java 예제
import com.wia.standard.*;

public class WIAExample {
    public static void main(String[] args) {
        // 빌더 패턴을 사용한 구성
        WIAClient client = WIAClient.builder()
            .apiKey(System.getenv("WIA_API_KEY"))
            .environment(Environment.PRODUCTION)
            .connectionTimeout(Duration.ofSeconds(30))
            .build();

        try {
            // 검증과 함께 리소스 생성
            Resource resource = Resource.builder()
                .name("예제 리소스")
                .type("standard-compliant")
                .metadata(Map.of(
                    "version", "1.0",
                    "createdBy", "user@example.com"
                ))
                .build();

            Resource created = client.createResource(resource);
            System.out.println("생성됨: " + created.getId());

        } catch (ValidationException e) {
            System.err.println("검증 실패: " + e.getMessage());
        } finally {
            client.close();
        }
    }
}

7.1 커뮤니티 중심 보존

문화유산은 그것을 창조하고 전승해온 커뮤니티에 속합니다. WIA-HERITAGE-001은 기술적 우수성만큼이나 커뮤니티 참여를 중요하게 여깁니다. 디지털화 프로젝트는 외부 전문가의 일방적 작업이 아니라, 커뮤니티와의 진정한 협력이어야 합니다.

커뮤니티 구성원은 자신의 유산에 대한 가장 깊은 지식을 보유하고 있습니다. 그들의 참여는 기술적 문서화를 넘어 문화적 의미와 맥락을 풍부하게 하고, 프로젝트의 정당성과 지속가능성을 확보합니다.

  • 호혜: 상호 이익이 되는 관계 구축
  • 투명성: 프로세스와 결과의 공개
  • 역량 강화: 커뮤니티 주도권 지원
  • 지속성: 일회성이 아닌 장기적 파트너십

7.2 FPIC 프로세스

자유롭고, 사전적이며, 정보에 입각한 동의(Free, Prior, and Informed Consent)는 원주민 및 지역 커뮤니티와의 모든 협력의 기초입니다. FPIC는 단순한 허가가 아니라, 지속적인 대화와 협의의 과정입니다.

FPIC 단계

단계활동결과물
1. 접촉적절한 대표자 확인 및 연락초기 관계 구축
2. 정보 제공프로젝트 목적, 방법, 영향 설명정보 자료 (현지어)
3. 협의충분한 시간을 두고 내부 논의커뮤니티 피드백
4. 동의/거부자율적 의사결정서면 합의 또는 거부
5. 지속 협의프로젝트 전 과정에서 소통수정 합의, 모니터링
⚠️ FPIC 핵심 요소
  • 자유로운(Free): 강압, 협박, 조작 없이
  • 사전적(Prior): 활동 시작 전에 충분한 시간
  • 정보에 입각한(Informed): 이해 가능한 언어와 형식
  • 동의(Consent): 거부할 권리 포함

7.3 접근 권한 관리

모든 디지털 문화유산이 모든 사람에게 공개되어야 하는 것은 아닙니다. 일부 유산은 특정 성별, 연령, 지위에게만 접근이 허용되는 문화적 프로토콜을 가지고 있습니다. WIA-HERITAGE-001은 다층적 접근 권한 관리를 지원합니다.

접근 수준

수준설명예시
공개제한 없는 접근일반 역사 자료
등록신원 확인된 사용자연구 목적 자료
커뮤니티출처 커뮤니티 구성원내부 전통 지식
의례특정 자격자만성별/연령 제한 의례
비공개지정된 관리자만신성한 비밀 지식

7.4 이익 공유

디지털 문화유산의 상업적 활용에서 발생하는 이익은 출처 커뮤니티와 공유되어야 합니다. 이는 금전적 보상뿐 아니라 역량 강화, 기술 이전, 고용 기회 등을 포함합니다.

이익 공유 형태

  • 금전적: 로열티, 라이선싱 수익 분배
  • 역량 강화: 교육, 훈련 프로그램
  • 기술 이전: 장비, 소프트웨어, 노하우
  • 접근 보장: 디지털화된 자료의 무료 접근
  • 공동 저작권: 지적재산권 공유

7.5 역량 강화

지속가능한 보존은 커뮤니티 스스로가 자신의 유산을 기록하고 관리할 역량을 갖출 때 가능합니다. WIA-HERITAGE-001은 커뮤니티 역량 강화 프로그램을 권장합니다.

📚 역량 강화 영역
  • 디지털화 기술 (촬영, 스캐닝, 녹음)
  • 메타데이터 작성 및 관리
  • 디지털 아카이브 운영
  • 저작권 및 접근 권한 관리
  • 보조금 신청 및 프로젝트 관리

  • 다층적 접근 권한으로 문화적 프로토콜 존중
