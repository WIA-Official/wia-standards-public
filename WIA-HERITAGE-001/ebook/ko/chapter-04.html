WIA-HERITAGE-001: 문화유산 디지털화 표준

4.1 3D 스캐닝 표준

3차원 문화유산 캡처는 구조광 스캐닝(Structured Light), 포토그래메트리(Photogrammetry), LiDAR의 세 가지 주요 기술을 활용합니다. 각 기술은 특정 유형의 유산에 최적화되어 있으며, WIA-HERITAGE-001은 각 기술의 최소 품질 요구사항을 명시합니다.

3D 디지털화의 목적은 물리적 객체의 기하학적 형태, 표면 텍스처, 색상 정보를 정확하게 기록하여 가상 환경에서 재현하고, 측정 및 분석을 가능하게 하며, 복제와 복원의 기반을 마련하는 것입니다. 이를 위해 마이크로미터(μm) 단위의 정밀도부터 센티미터(cm) 단위의 효율성까지 다양한 스케일의 캡처 기술이 필요합니다.

모든 3D 캡처는 다음 형식으로 보관되어야 합니다:

  • 메쉬(Mesh): OBJ glTF 2.0 PLY (오픈 형식 필수, 독점 형식 보조 허용)
  • 포인트 클라우드(Point Cloud): E57 LAS/LAZ 1.4 PTS
  • 텍스처(Texture): PNG (무손실 압축, 최소 4K 해상도) TIFF (마스터용)
  • 메타데이터: JSON 형식으로 캡처 장비, 날짜, 환경 조건, 좌표계, 스케일 정보 포함

3D 캡처 워크플로우

  1. 사전 준비: 객체 상태 점검, 조명 설정, 색상 차트 배치, 스케일 바 설치
  2. 데이터 획득: 다각도 스캔 (최소 360도 커버리지, 중복률 30% 이상)
  3. 데이터 처리: 정렬(alignment), 병합(merging), 노이즈 제거, 홀 메우기
  4. 텍스처 매핑: 색상 보정, UV 언래핑, 텍스처 베이킹
  5. 최적화: 폴리곤 감소 (여러 LOD 버전 생성), 텍스처 압축
  6. 품질 검증: 기하학적 정확도 측정, 색상 재현도 확인, 완전성 평가
  7. 메타데이터 생성: 캡처 프로세스 문서화, 파일 정보 기록

4.2 문서 이미징

역사적 문서, 필사본, 지도의 디지털화는 특수한 고려가 필요합니다. 표면 상태, 잉크 성분, 워터마크, 손상 부위 등의 정보를 보존하기 위해 다중 스펙트럼 이미징(Multispectral Imaging)이 권장됩니다. 문서 디지털화의 목표는 시각적 재현뿐 아니라 OCR(광학 문자 인식), 보존 상태 평가, 원본 재료 분석을 지원하는 것입니다.

필사본/희귀본 600 PPI 48-bit RGB TIFF 6.0 무압축 색 차트, 스케일 바, 메타데이터 임베딩
대형 지도 400 PPI 48-bit RGB JPEG2000 무손실 타일 스캔, 자동 스티칭, GeoTIFF 메타데이터
사진 아카이브 4000 PPI (필름) 48-bit RGB + IR DNG (RAW) + TIFF 먼지 제거, ICE 기술, 밀도 범위 4.0 이상
신문/인쇄물 300 PPI 24-bit RGB 또는 8-bit 흑백 PDF/A-2u OCR 레이어, 구조적 메타데이터, ALTO XML

멀티스펙트럼 이미징 프로토콜

멀티스펙트럼 이미징은 가시광선 외에 자외선(UV), 적외선(IR), 형광 등 여러 파장 대역에서 문서를 촬영하여 육안으로 보이지 않는 정보를 기록합니다. 이는 팔림프세스트(덮어쓴 문서), 손상된 텍스트, 위조 감별, 잉크 성분 분석에 필수적입니다.

스펙트럼 대역 파장 범위 주요 용도 장비 요구사항
UV (자외선) 365nm 잉크 구별, 수정 감지, 형광 분석 UV 조명, UV-pass 필터
가시광선 400-700nm 표준 이미징, 색상 기록 D50/D65 조명, 색 관리
IR (근적외선) 700-1000nm 덮어쓴 텍스트, 깊이 침투 IR-pass 필터, IR 민감 센서
RTI (Reflectance Transformation Imaging) 가시광 표면 텍스처, 새겨진 문자 돔 또는 멀티 라이트 시스템
📷 이미징 환경 표준
  • 조명: 색온도 5000K (D50) 또는 6500K (D65), CRI ≥ 95, 균일도 ±5%
  • 카메라: 풀프레임 센서 (최소 50MP), 14-bit RAW 지원, 색 공간 Adobe RGB 또는 ProPhoto RGB
  • 렌즈: 왜곡 < 0.5%, 색수차 보정, f/8-f/11 권장 조리개
  • 색 관리: X-Rite ColorChecker Classic 또는 Golden Thread 사용, ICC 프로파일 생성
  • 환경: 온도 20±2°C, 습도 40-50%, 먼지 제어

4.3 오디오-비주얼 기록

무형문화유산, 공연 예술, 구전 전통의 기록은 고품질 오디오-비주얼 캡처를 요구합니다. 모션 캡처 기술은 춤, 의례적 동작, 전통 공예 기술의 정밀한 기록을 가능하게 합니다. AV 기록의 목표는 공연의 시각적·청각적 경험을 재현하고, 동작과 소리의 뉘앙스를 보존하며, 학술 연구와 교육에 활용할 수 있는 데이터를 생성하는 것입니다.

AV 캡처 표준

  • 비디오:
    • 해상도: 최소 4K UHD (3840×2160), 권장 6K 또는 8K
    • 프레임 레이트: 최소 60fps (고속 동작은 120fps 이상)
    • 색 샘플링: 4:2:2 이상, 권장 4:4:4
    • 색 깊이: 10-bit 이상, HDR 지원시 12-bit
    • 코덱: ProRes 422 HQ, ProRes 4444, DNxHR, FFV1 (무손실)
    • 색 공간: Rec. 709 (SDR) 또는 Rec. 2020 (HDR)
  • 오디오:
    • 샘플 레이트: 최소 96kHz (음악 녹음), 48kHz (대화 중심)
    • 비트 깊이: 24-bit
    • 채널: 최소 스테레오, 권장 다채널 (5.1, Ambisonic B-format)
    • 형식: WAV (BWF), FLAC, 공간 오디오는 ADM BWF
    • 마이크: 콘덴서 마이크, 주파수 응답 20Hz-20kHz ±2dB
  • 모션 캡처:
    • 프레임 레이트: 최소 120fps, 고속 동작 240fps
    • 마커 수: 전신 최소 41개 마커 (Vicon Plug-in-Gait 마커셋)
    • 정확도: <1mm @ 5m 거리
    • 출력 형식: C3D, BVH, FBX
    • 마커리스 옵션: OpenPose, MediaPipe (보조 데이터)
  • 360° 비디오:
    • 해상도: 최소 8K (7680×3840) 모노스코픽, 6K 스테레오스코픽
    • 스티칭: 자동 또는 수동, 시차 오류 <2 픽셀
    • 공간 오디오: Ambisonic 1st order 이상, 권장 3rd order
    • 형식: Equirectangular JPEG/PNG 시퀀스 또는 H.265 비디오
🎬 다중 카메라 세팅

복잡한 공연이나 의례의 경우, 최소 3개 이상의 동기화된 카메라를 사용하여 다양한 각도에서 기록합니다. 마스터 타임코드(SMPTE LTC 또는 PTP IEEE 1588)와 클래퍼 싱크를 통해 후반 작업에서의 정확한 동기화를 보장합니다.

권장 카메라 배치:

  • Master shot: 전체 무대/공간을 넓게 담는 고정 카메라 (4K 60fps)
  • Detail shots: 연기자/수행자의 표정, 손동작 등을 담는 클로즈업 카메라 (2대 이상)
  • Contextual shot: 관객 반응, 환경 등을 담는 보조 카메라
  • Overhead/specialty: 특수 각도 또는 360° 카메라 (옵션)

4.4 품질 검증

모든 디지털 캡처는 엄격한 품질 검증 프로세스를 거쳐야 합니다. WIA-HERITAGE-001은 자동화된 품질 검사 도구와 수동 검토 체크리스트를 제공합니다. 품질 검증의 목표는 캡처된 데이터가 명시된 기술 사양을 충족하고, 원본을 충실하게 재현하며, 장기 보존과 재사용에 적합한지 확인하는 것입니다.

품질 검증 체크리스트

  1. 해상도 및 정확도 측정값 검증
    • 3D: 스케일 바를 사용한 치수 정확도 확인 (허용 오차 ±0.5%)
    • 이미징: ISO 16067-1/2 테스트 타겟으로 해상도 검증
    • AV: 테스트 패턴(SMPTE 컬러 바, 1kHz 톤)으로 신호 품질 확인
  2. 컬러 차트를 통한 색 재현성 확인
    • X-Rite ColorChecker 또는 Golden Thread 포함된 캡처 사용
    • Delta E (ΔE) 값 측정: ΔE < 3 (양호), ΔE < 1 (우수)
    • 화이트 밸런스, 색온도, 색 공간 일관성 확인
  3. 기하학적 왜곡 분석
    • 렌즈 왜곡: 격자 패턴 왜곡 측정, 보정 적용 여부 확인
    • 3D 정렬: ICP (Iterative Closest Point) 오차 측정
    • 스케일: 알려진 치수 객체와 비교, 일관성 확인
  4. 메타데이터 완전성 검토
    • 필수 필드: 식별자, 날짜, 장비, 파라미터, 파일 정보 모두 기록
    • 임베딩: EXIF, XMP, IPTC (이미지), Matroska 메타데이터 (비디오)
    • 외부 파일: JSON 또는 XML 사이드카 파일 생성 및 검증
  5. 파일 무결성 체크섬 생성
    • 알고리즘: SHA-256 (최소), SHA-512 (권장)
    • 체크섬 파일: MD5SUM, SHA256SUM 형식 또는 BagIt manifest
    • 검증: 수집 직후 및 저장 후 즉시 체크섬 확인
  6. 완전성 및 커버리지 평가
    • 3D: 전체 표면 커버리지, 홀(hole) 크기 < 5mm²
    • 문서: 모든 페이지 캡처, 순서 확인, 누락 검사
    • AV: 시작부터 끝까지 연속 녹화, 프레임 드롭 검사
🔍 자동화 품질 검사 도구

WIA는 다음 오픈소스 및 상용 도구를 권장합니다:

  • 이미지 QC: Adobe Lightroom, Capture One, ExifTool (메타데이터), ImageMagick (자동화)
  • 3D QC: CloudCompare (정렬 오차, 노이즈), MeshLab (메쉬 품질), Geomagic Control (정확도)
  • 비디오 QC: FFmpeg (프레임 분석), MediaInfo (메타데이터), QCTools (신호 분석)
  • 오디오 QC: iZotope RX (스펙트럼 분석), Adobe Audition (파형 검사)
  • 체크섬: md5sum, sha256sum, BagIt (파일 집합 무결성)

4.5 특수 재료 및 기술

반사성/투명 재질

금속, 유리, 보석 등 반사성 또는 투명한 재질은 표준 캡처 기술로는 충분히 기록되지 않습니다. 교차 편광(cross-polarization), HDR 촬영, 다중 조명 RTI 등의 특수 기법이 필요합니다.

대형 객체 및 건축물

건물, 대형 조각, 유적지 등은 크기와 접근성의 제약으로 특수한 캡처 전략이 필요합니다. 지상 및 항공 LiDAR, 드론 포토그래메트리, 파노라마 이미징을 결합하여 완전한 문서화를 달성합니다.

4.6 장 요약

✅ 핵심 내용
  • 3D 캡처: 구조광, 포토그래메트리, LiDAR 기술별 표준 사양 및 적용 대상
  • 문서: 멀티스펙트럼 이미징 (UV, 가시광, IR), 고해상도 TIFF/JPEG2000
  • AV: 최소 4K 60fps, 10-bit 색심도, 96kHz/24-bit 오디오, 다채널 및 공간 오디오
  • 모션 캡처: 120fps 이상, 전신 마커 시스템 또는 마커리스
  • 품질 검증: 해상도, 색 정확도, 기하학적 정밀도, 메타데이터 완전성, 체크섬
  • 특수 재료: 반사성/투명 재질을 위한 교차 편광, HDR, RTI 기법
  • 오픈 형식 우선: OBJ, glTF, E57, TIFF, WAV 등 독점 형식 회피

복습 질문

  1. 구조광 스캐닝, 포토그래메트리, 지상 LiDAR의 최소 해상도와 정확도를 비교하고, 각 기술이 최적화된 문화유산 유형을 설명하시오.
  2. 멀티스펙트럼 이미징에서 UV, 가시광선, IR 각 대역이 문서 분석에 어떻게 활용되는지 예를 들어 설명하시오.
  3. 무형문화유산 공연 기록을 위한 다중 카메라 세팅의 권장 구성(Master/Detail/Contextual shot)을 설명하고, 동기화 방법을 논하시오.
  4. 3D 메쉬 출력 형식으로 OBJ, glTF, PLY가 권장되는 이유는 무엇이며, 각 형식의 장단점을 비교하시오.
  5. 색 재현성 검증에 사용되는 Delta E (ΔE) 값의 의미를 설명하고, 양호/우수 기준을 제시하시오.
  6. 금속 유물과 같은 반사성 재질을 디지털화할 때 발생하는 문제와 교차 편광 기법의 원리를 설명하시오.
  7. AV 기록에서 최소 96kHz 샘플 레이트와 24-bit 깊이가 권장되는 이유를 음향학적 관점에서 논하시오.
  8. BagIt과 같은 파일 집합 무결성 도구가 단순 SHA-256 체크섬보다 유리한 점을 설명하시오.
