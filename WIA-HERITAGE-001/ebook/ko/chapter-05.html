WIA-HERITAGE-001: 문화유산 디지털화 표준

5.1 메타데이터의 중요성

메타데이터는 디지털 문화유산의 맥락과 의미를 담는 핵심 요소입니다. 고품질 이미지나 3D 모델도 적절한 메타데이터 없이는 단순한 비트의 집합에 불과합니다. WIA-HERITAGE 메타데이터 스키마는 기술적 정보와 문화적 맥락을 통합하여 장기적인 이해와 활용을 보장합니다.

효과적인 메타데이터는 검색 가능성을 높이고, 출처를 증명하며, 접근 권한을 관리하고, 보존 이력을 추적합니다. 이는 디지털 객체가 원본과 분리되었을 때도 그 가치와 의미를 유지하게 합니다. "메타데이터 없이는 데이터가 아니라 노이즈일 뿐"이라는 도서관 정보학의 격언이 문화유산 영역에서도 그대로 적용됩니다.

디지털 문화유산 메타데이터는 단순히 파일명, 크기, 생성일을 기록하는 것을 넘어, 객체의 역사적 맥락, 문화적 의미, 제작 기법, 재료 구성, 보존 상태, 이전 소유자, 전시 이력, 학술적 해석 등을 포괄합니다. 이러한 풍부한 맥락 정보는 디지털 객체를 살아있는 지식 자원으로 만듭니다.

📋 메타데이터 핵심 기능
  • 발견(Discovery): 검색 및 탐색 지원, 필터링, 패싯 브라우징
  • 출처(Provenance): 기원, 소유 이력, 수집 경위, 거래 기록
  • 맥락(Context): 문화적, 역사적, 사회적 배경, 관련 사건
  • 권리(Rights): 저작권, 접근 제한, 사용 조건, 라이선스
  • 보존(Preservation): 기술적 특성, 변환 이력, 무결성 검증
  • 관계(Relationships): 다른 객체, 컬렉션, 인물, 사건과의 연결
  • 진본성(Authenticity): 원본 증명, 디지털 서명, 타임스탬프

메타데이터 품질 속성

고품질 메타데이터는 다음 속성을 만족해야 합니다:

  • 정확성(Accuracy): 사실에 기반하고 검증된 정보
  • 완전성(Completeness): 필수 필드 모두 기록, 권장 필드 최대한 포함
  • 일관성(Consistency): 용어, 형식, 구조의 통일성
  • 명확성(Clarity): 모호하지 않고 이해하기 쉬운 표현
  • 최신성(Currency): 변경사항 반영, 정기적 갱신
  • 접근성(Accessibility): 기계 가독성, 다국어 지원

5.2 WIA-HERITAGE 스키마 구조

WIA-HERITAGE 메타데이터 스키마는 Dublin Core를 기반으로 하며, CIDOC-CRM의 이벤트 중심 모델과 문화유산 특화 확장을 통합합니다. 이는 기존 표준과의 호환성을 유지하면서도 문화유산의 고유한 요구사항을 충족하는 균형잡힌 접근입니다.

핵심 메타데이터 필드

카테고리 필드 의무성 설명 형식
식별 wia:identifier 필수 고유 식별자 UUID v4 또는 ARK
식별 wia:title 필수 공식 명칭 다국어 문자열
식별 wia:alternateTitle 권장 별칭, 통칭 다국어 문자열 배열
출처 wia:creator 필수 제작자/창작자 개인 또는 조직 (VIAF/ISNI 링크)
출처 wia:date 필수 제작 시기 EDTF (Extended Date/Time Format)
출처 wia:provenance 권장 소유/수집 이력 시간순 이벤트 배열
맥락 wia:culture 필수 문화권/문명 Getty AAT 또는 자유 텍스트
맥락 wia:place 권장 원산지/발견지 TGN 또는 GeoNames + 좌표
맥락 wia:subject 권장 주제/키워드 통제 어휘 (LCSH, AAT)
설명 wia:description 필수 상세 설명 다국어 장문 텍스트
설명 wia:material 권장 재료/매체 AAT 재료 어휘
설명 wia:technique 권장 제작 기법 AAT 기법 어휘
설명 wia:dimensions 권장 크기/치수 구조화된 측정값 (단위 포함)
기술 wia:format 필수 파일 형식 MIME type + 확장자
기술 wia:extent 필수 파일 크기 바이트 + 인간 가독 표기
권리 wia:rights 필수 권리 설명 권리 선언문 텍스트
권리 wia:license 권장 라이선스 SPDX 라이선스 식별자 또는 URL
권리 wia:accessRights 필수 접근 수준 열거형 (public, registered, community, restricted)

5.3 문화적 맥락 확장

문화유산은 기술적 메타데이터만으로는 충분히 설명되지 않습니다. WIA-HERITAGE 스키마는 문화적 맥락을 기록하기 위한 확장 필드를 제공합니다. 이는 객체가 단순한 물건이 아니라 문화적 의미와 정체성의 담지자임을 인정하는 것입니다.

🌏 다국어 지원

모든 텍스트 필드는 다국어를 지원합니다. 원어 표기를 우선하고, 영어 번역과 로마자 표기를 함께 제공합니다. 이는 국제적 접근성과 문화적 정확성을 동시에 보장합니다.

언어 태그는 IETF BCP 47 표준을 따릅니다: ko(한국어), ko-Hani(한자), ko-Latn(로마자 표기), en(영어), ja(일본어), zh-Hans(중국어 간체) 등.

5.4 전통 지식 라벨 (Traditional Knowledge Labels)

Local Contexts 이니셔티브의 전통 지식(TK) 라벨과 생물문화(BC) 라벨을 통합 지원합니다. 이 라벨들은 원주민 커뮤니티의 문화적 프로토콜을 디지털 환경에서 표현하며, 서구 중심의 저작권 체계가 보호하지 못하는 전통 지식의 권리를 옹호합니다.

라벨 유형 URI 의미 사용 예
TK Attribution (TK A) localcontexts.org/tk/a/1.0 커뮤니티 귀속 명시 출처 커뮤니티 이름, 위치, 연락처 명시
TK Family (TK F) localcontexts.org/tk/f/1.0 가족/씨족 소유 특정 가계에 속하는 지식, 노래, 디자인
TK Non-Commercial (TK NC) localcontexts.org/tk/nc/1.0 비상업적 사용 상업적 이용 금지, 연구/교육 목적만 허용
TK Seasonal (TK S) localcontexts.org/tk/s/1.0 계절적 제한 특정 시기에만 공개 가능 (의례 기간 등)
TK Secret/Sacred (TK SS) localcontexts.org/tk/ss/1.0 비밀/신성한 지식 제한된 접근, 썸네일/프리뷰 불가
TK Men Restricted (TK MR) localcontexts.org/tk/mr/1.0 남성 전용 여성에게 공개 불가
TK Women General (TK WG) localcontexts.org/tk/wg/1.0 여성 일반 여성 커뮤니티 구성원 일반 접근 가능
TK Community Use Only (TK CO) localcontexts.org/tk/co/1.0 커뮤니티 전용 외부인 접근 완전 불가
TK Verified (TK V) localcontexts.org/tk/v/1.0 커뮤니티 검증 커뮤니티가 내용 정확성 확인

5.5 상호운용성 (Interoperability)

WIA-HERITAGE 스키마는 주요 국제 표준과의 매핑을 제공하여 데이터 교환과 통합 검색을 지원합니다. 이는 고립된 데이터 사일로를 방지하고 글로벌 문화유산 지식 그래프에 기여합니다.

🔗 링크드 데이터 (Linked Data)

WIA-HERITAGE 메타데이터는 JSON-LD 형식으로 제공되어 시맨틱 웹과의 통합을 지원합니다. 주요 온톨로지 및 어휘와의 링크:

  • Dublin Core Terms: http://purl.org/dc/terms/
  • CIDOC-CRM: http://www.cidoc-crm.org/cidoc-crm/
  • Getty AAT: http://vocab.getty.edu/aat/
  • GeoNames: http://www.geonames.org/
  • VIAF: http://viaf.org/viaf/
  • WIA Namespace: https://wia.org/ns/heritage#

5.6 메타데이터 품질 관리

메타데이터 품질은 디지털 문화유산의 유용성을 결정합니다. WIA-HERITAGE는 메타데이터 작성, 검증, 개선을 위한 종합적인 품질 관리 프레임워크를 제공합니다.

품질 검증 체크리스트

  1. 완전성 검사: 필수 필드 모두 입력, 권장 필드 80% 이상
  2. 정확성 검증: 날짜 형식, 식별자 유효성, 외부 참조 링크 확인
  3. 일관성 확인: 통제 어휘 준수, 명명 규칙 일치, 단위 통일
  4. 중복 제거: 동일 객체의 중복 레코드 병합
  5. 관계 검증: 관련 객체 간 양방향 링크 일치성
  6. 권리 명확화: 모호한 권리 선언문 구체화
🛠️ 메타데이터 도구
  • 작성: CollectiveAccess, Omeka S, ResourceSpace (웹 기반 폼)
  • 검증: OpenRefine (데이터 클리닝), JSON Schema Validator
  • 보강: Getty Vocabularies API, VIAF Autosuggest, GeoNames Search
  • 변환: XSLT (XML), jq (JSON), Python pandas (대용량)
  • 분석: Tableau, Power BI (메타데이터 품질 대시보드)

5.7 장 요약

✅ 핵심 내용
  • 메타데이터는 디지털 유산의 맥락과 의미를 담는 핵심 - "메타데이터 없이는 노이즈일 뿐"
  • Dublin Core + CIDOC-CRM 기반 WIA-HERITAGE 스키마, 18개 필수/권장 필드
  • 문화적 맥락 확장: 의미, 전통, 상징, 구전 역사, 커뮤니티 연결
  • 전통 지식 라벨 (TK Labels) 통합 지원 - 원주민 권리와 문화적 프로토콜 존중
  • 주요 국제 표준과의 상호운용성: DC, CIDOC-CRM, EDM, LIDO 크로스워크
  • JSON-LD 링크드 데이터 지원으로 시맨틱 웹 통합
  • 메타데이터 품질 관리: 완전성, 정확성, 일관성, 명확성, 최신성, 접근성
