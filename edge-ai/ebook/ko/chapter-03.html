모델 최적화

최적화의 필요성

최첨단 신경망 모델은 종종 수억 또는 수십억 개의 파라미터를 포함하며 기가바이트의 메모리와 막대한 컴퓨팅 리소스를 필요로 합니다. 반면 엣지 디바이스는 엄격한 제약을 가지고 있습니다: 제한된 RAM(1-8GB), 적당한 컴퓨팅 성능(10-100 TOPS), 그리고 타이트한 전력 예산(0.1-5W). 클라우드 규모의 모델을 엣지 디바이스에 직접 배포하는 것은 불가능합니다.

모델 최적화는 이 격차를 메우며, 크고 정확한 모델을 가능한 한 많은 정확도를 보존하면서 엣지 배포에 적합한 컴팩트하고 효율적인 버전으로 변환합니다. 목표: 최소한의 정확도 손실(<1-3%)로 10-100배 크기 감소와 3-10배 속도 향상을 달성하는 것입니다.

양자화 (Quantization)

수치 정밀도 이해

신경망은 전통적으로 가중치와 활성화에 32비트 부동소수점(FP32) 숫자를 사용합니다. 각 파라미터는 4바이트의 저장 공간을 필요로 합니다. 양자화는 수치 정밀도를 더 작은 데이터 타입으로 줄입니다:

데이터 타입 비트 범위 크기 감소
FP32 32 ±3.4×10³⁸ 기준 (1x)
FP16 16 ±6.5×10⁴ 2배 작음
INT8 8 -128 ~ 127 4배 작음
INT4 4 -8 ~ 7 8배 작음

훈련 후 양자화 (PTQ)

PTQ는 재훈련 없이 훈련된 FP32 모델을 낮은 정밀도로 변환합니다. 프로세스:

  1. 보정: 대표 데이터를 모델에 통과시켜 활성화 통계 수집
  2. 범위 결정: 각 레이어의 최소/최대 값 계산
  3. 매핑: FP32 값을 INT8 범위로 변환
  4. 검증: 정확도 저하 측정
// TensorFlow Lite 훈련 후 INT8 양자화
import tensorflow as tf

converter = tf.lite.TFLiteConverter.from_saved_model('model/')
converter.optimizations = [tf.lite.Optimize.DEFAULT]

def representative_dataset():
    for data in calibration_samples:
        yield [data.astype(np.float32)]

converter.representative_dataset = representative_dataset
converter.target_spec.supported_ops = [tf.lite.OpsSet.TFLITE_BUILTINS_INT8]
converter.inference_input_type = tf.int8
converter.inference_output_type = tf.int8

quantized_model = converter.convert()
with open('model_int8.tflite', 'wb') as f:
    f.write(quantized_model)

결과: PTQ는 일반적으로 1-2% 정확도 손실로 4배 크기 감소(FP32 → INT8)를 달성합니다. 재훈련이 필요 없고 빠른 변환(몇 분)이 가능합니다.

양자화 인식 훈련 (QAT)

QAT는 훈련 중 양자화를 시뮬레이션하여 모델이 감소된 정밀도에 적응할 수 있게 합니다. 이는 PTQ에 비해 정확도 손실을 최소화합니다.

// PyTorch 양자화 인식 훈련
import torch.quantization as quant

model = YourModel()
model.qconfig = quant.get_default_qat_qconfig('fbgemm')
model_prepared = quant.prepare_qat(model)

# 시뮬레이션된 양자화로 정상적으로 훈련
for epoch in range(num_epochs):
    train(model_prepared, train_loader)

# 양자화된 모델로 변환
model_quantized = quant.convert(model_prepared)

결과: QAT는 0.5% 미만의 정확도 손실로 4배 크기 감소를 달성합니다. 재훈련이 필요하지만(몇 시간에서 며칠) PTQ보다 더 나은 정확도 보존이 가능합니다.

혼합 정밀도 양자화

모든 레이어가 양자화를 동일하게 허용하는 것은 아닙니다. 민감한 레이어(첫 번째, 마지막, 어텐션)는 FP16을 사용하고 중간 레이어는 INT8을 사용할 수 있습니다. 이는 크기 감소와 정확도 사이의 균형을 맞춥니다.

프루닝 (Pruning)

비구조적 프루닝

작은 크기의 개별 가중치를 제거합니다. 0에 가까운 가중치는 모델 출력에 최소한으로 기여하며 정확히 0으로 설정할 수 있습니다.

// 간단한 크기 기반 프루닝
import torch.nn.utils.prune as prune

# Conv 레이어에서 50%의 가중치 프루닝
prune.l1_unstructured(model.conv1, name='weight', amount=0.5)

# 프루닝을 영구적으로 만들기
prune.remove(model.conv1, 'weight')

대형 네트워크의 경우 50-90%의 가중치를 프루닝하는 것이 일반적입니다. 프루닝된 모델은 실제로 크기를 줄이기 위해 희소 행렬 형식(CSR, CSC)이 필요합니다.

과제: 희소 행렬은 많은 엣지 가속기에서 잘 지원되지 않습니다. 비구조적 프루닝은 전문 하드웨어 없이는 추론 속도를 향상시키지 못할 수 있습니다.

구조적 프루닝

개별 가중치가 아닌 전체 채널, 필터 또는 뉴런을 제거합니다. 이는 표준 하드웨어에서 효율적으로 실행되는 모델을 만듭니다.

// 채널 프루닝 예제
# L1 노름을 기반으로 30%의 채널 제거
for layer in model.conv_layers:
    channel_importance = compute_channel_norms(layer)
    keep_channels = select_top_k(channel_importance, k=0.7)
    layer.weight = layer.weight[keep_channels, :, :, :]
    layer.bias = layer.bias[keep_channels]

결과: 2-5% 정확도 손실로 30-50% 파라미터 감소. 모델 구조가 밀집 상태로 유지되므로 엣지 하드웨어에서 실제 속도 향상이 있습니다.

반복적 프루닝 및 미세 조정

최선의 방법: 반복 사이에 미세 조정을 통해 점진적으로 프루닝합니다.

  1. 10-20%의 파라미터 프루닝
  2. 5-10 에폭 동안 미세 조정하여 정확도 회복
  3. 목표 압축에 도달할 때까지 반복

지식 증류 (Knowledge Distillation)

교사-학생 프레임워크

작은 "학생" 모델을 큰 "교사" 모델을 모방하도록 훈련합니다. 학생은 실제 레이블과 교사의 소프트 예측 모두에서 학습합니다.

// 지식 증류 손실
def distillation_loss(student_logits, teacher_logits, labels, temp=3.0, alpha=0.7):
    # 교사의 소프트 타겟 (평활화된 확률)
    soft_targets = F.softmax(teacher_logits / temp, dim=1)
    soft_student = F.log_softmax(student_logits / temp, dim=1)
    distillation = F.kl_div(soft_student, soft_targets, reduction='batchmean')

    # 실제 레이블의 하드 타겟
    hard_loss = F.cross_entropy(student_logits, labels)

    # 결합 손실
    return alpha * (temp ** 2) * distillation + (1 - alpha) * hard_loss

# 훈련 루프
for images, labels in dataloader:
    with torch.no_grad():
        teacher_logits = teacher_model(images)

    student_logits = student_model(images)
    loss = distillation_loss(student_logits, teacher_logits, labels)
    loss.backward()
    optimizer.step()

교사의 소프트 예측은 하드 레이블보다 풍부한 정보를 제공합니다. "개"로 95% 확신으로 분류된 이미지의 경우, 교사는 3% "늑대", 1% "고양이"라는 것도 보여줍니다—학생이 배울 수 있는 구조적 지식입니다.

학생 아키텍처 선택

일반적인 접근 방식:

  • 축소된 버전: 교사와 동일한 아키텍처이지만 더 적은 레이어/채널 (예: ResNet-152 → ResNet-18)
  • 효율적인 아키텍처: 엣지에 최적화된 다른 아키텍처 (예: ResNet-50 → MobileNetV3)
  • 작업별: 대상 하드웨어용 전문 아키텍처

결과: 교사 정확도의 95-99%를 유지하면서 5-20배 모델 크기 감소. 학생은 종종 작은 아키텍처를 직접 훈련하는 것보다 더 나은 성능을 보입니다.

신경 아키텍처 검색 (NAS)

하드웨어 인식 NAS

대상 엣지 하드웨어에 최적화된 신경망 아키텍처를 자동으로 발견합니다. 검색 목표는 다음 사항의 균형을 맞춥니다:

  • 검증 세트의 정확도
  • 대상 디바이스의 지연시간
  • 모델 크기 (파라미터, 저장소)
  • 에너지 소비

인기 있는 NAS 방법:

  • EfficientNet: 복합 스케일링을 사용하여 깊이, 너비, 해상도 균형
  • MobileNetV3: 모바일 디바이스용 플랫폼 인식 NAS를 통해 발견
  • FBNet: Facebook의 모바일 배포용 NAS
  • Once-for-All (OFA): 한 번 훈련하고 여러 하드웨어 대상에 배포

엣지를 위한 AutoML

TensorFlow Model Optimization Toolkit 및 Neural Network Intelligence (NNI)와 같은 도구는 양자화, 프루닝, NAS를 체계적으로 적용하여 최적화 프로세스를 자동화합니다.

연산자 융합 및 그래프 최적화

레이어 융합

순차적 연산을 단일 커널로 결합하여 메모리 트래픽과 오버헤드를 줄입니다.

일반적인 융합:

  • Conv + BatchNorm + ReLU: 3개 연산 → 1개 융합 커널
  • MatMul + Bias + Activation: 활성화를 가진 Dense 레이어
  • Add + ReLU: 활성화를 가진 Residual 연결
// 융합 전 (3개 연산, 3번 메모리 통과)
x = conv2d(input, weights)
x = batch_norm(x, bn_params)
output = relu(x)

// 융합 후 (1개 연산, 1번 메모리 통과)
output = fused_conv_bn_relu(input, weights, bn_params)

융합은 메모리 읽기/쓰기를 최소화하여 지연시간을 20-40% 줄입니다.

상수 폴딩

컴파일 시간에 상수 입력을 가진 연산을 미리 계산합니다.

// 전: 런타임에 계산
scale = 0.5
output = input * scale * 2.0

// 상수 폴딩 후: 0.5 * 2.0 = 1.0 사전 계산
output = input  // 곱셈이 완전히 제거됨!

데드 코드 제거

사용되지 않는 출력, 도달할 수 없는 연산, 중복 계산을 그래프에서 제거합니다.

저랭크 인수분해

텐서 분해

저랭크 분해를 사용하여 가중치 행렬을 근사화합니다. 가중치 행렬 W ∈ R^(m×n)에 대해, W ≈ U × V로 분해합니다. 여기서 U ∈ R^(m×k), V ∈ R^(k×n)이고 k ≪ min(m,n)입니다.

// 압축을 위한 특이값 분해 (SVD)
U, S, V = torch.svd(weight_matrix)

# 상위 k개의 특이값만 유지
k = 64
U_reduced = U[:, :k]
S_reduced = S[:k]
V_reduced = V[:, :k]

# 원래 가중치 근사
weight_approx = U_reduced @ torch.diag(S_reduced) @ V_reduced.t()

# 하나의 큰 레이어를 두 개의 작은 레이어로 교체
layer1.weight = U_reduced @ torch.diag(torch.sqrt(S_reduced))
layer2.weight = torch.diag(torch.sqrt(S_reduced)) @ V_reduced.t()

결과: 최소한의 정확도 손실로 완전 연결 레이어에 대해 2-5배 압축.

압축 파이프라인

기술 결합

최대 압축은 여러 기술을 결합하여 얻습니다:

  1. 프루닝: 50%의 파라미터 제거 (2배 감소)
  2. 양자화: FP32 → INT8 (4배 감소)
  3. 지식 증류: 효율적인 학생 훈련 (3배 감소)
  4. 총계: 2 × 4 × 3 = 24배 전체 압축
// 완전한 최적화 파이프라인
// 1. 사전 훈련된 모델로 시작
model = load_pretrained_model()

// 2. 60%의 가중치 프루닝
model = iterative_prune(model, target_sparsity=0.6, epochs=10)

// 3. 효율적인 아키텍처로 지식 증류
student = MobileNetV3()
student = distill(teacher=model, student=student, epochs=50)

// 4. 양자화 인식 훈련
student = quantization_aware_train(student, epochs=20)

// 5. INT8로 변환
model_int8 = convert_to_int8(student)

// 6. 엣지 배포를 위해 내보내기
export_to_tflite(model_int8, 'optimized_model.tflite')

정확도-효율성 트레이드오프

파레토 프론티어

정확도 대 지연시간/크기에 모델 변형을 플로팅합니다. 파레토 프론티어는 지배되지 않는 모델을 보여줍니다—한 지표를 다른 지표를 해치지 않고 개선할 수 없는 모델입니다.

모델 변형 정확도 지연시간 크기
원본 FP32 92.5% 120ms 98 MB
+ 프루닝 50% 91.8% 100ms 50 MB
+ INT8 양자화 91.2% 35ms 12 MB
+ 증류 89.5% 18ms 4 MB

작업별 요구 사항

애플리케이션 요구 사항에 따라 최적화 수준을 선택합니다:

  • 의료 진단: 정확도 우선 (91%+), 더 높은 지연시간/크기 허용
  • 실시간 AR: 지연시간 우선 (<16ms for 60fps), 일부 정확도 손실 허용
  • IoT 센서: 크기/전력 우선 (100KB 미만 모델), 상당한 정확도 감소 허용

弘益人間 최적화 원칙:

가장 광범위한 디바이스에서 실행되도록 모델을 최적화하여 하드웨어에 관계없이 모든 사람이 AI에 접근할 수 있도록 합니다. 10배 크기 감소는 전 세계적으로 10배 더 많은 디바이스에 AI를 제공합니다.

요약

모델 최적화는 엣지 AI 배포에 필수적입니다. 주요 기술:

  • 양자화: 수치 정밀도 감소 (FP32 → INT8)로 최소한의 정확도 손실로 4배 크기 감소
  • 프루닝: 불필요한 가중치/채널 제거로 2-10배 파라미터 감소
  • 지식 증류: 작은 모델을 큰 모델을 모방하도록 훈련하여 5-20배 압축 달성
  • NAS: 하드웨어 최적화 아키텍처 자동 발견
  • 그래프 최적화: 연산 융합, 데드 코드 제거, 상수 폴딩

기술을 결합하면 10-100배 총 압축을 얻습니다. 최적화 파이프라인: 프루닝 → 증류 → 양자화 → 그래프 최적화. 정확도, 지연시간, 크기, 전력 간의 트레이드오프는 애플리케이션별로 다릅니다.

최신 프레임워크(TensorFlow Lite, PyTorch, ONNX Runtime)는 자동화된 최적화를 위한 강력한 도구를 제공합니다. 실제 성능 향상을 검증하기 위해 대상 하드웨어에서 결과를 측정하십시오.
