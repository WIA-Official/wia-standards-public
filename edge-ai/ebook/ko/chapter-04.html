TinyML과 임베디드 AI

TinyML이란 무엇인가?

TinyML(Tiny Machine Learning)은 마이크로컨트롤러와 같은 극도로 리소스가 제한된 디바이스에서 머신러닝 모델을 실행하는 기술입니다. 이러한 디바이스는 일반적으로 다음과 같은 특성을 가집니다:

  • 메모리: 1KB-1MB RAM, 수십 KB-수 MB 플래시
  • 처리 능력: 1-100 MHz CPU, 전용 AI 가속기 없음
  • 전력: 1mW-100mW (배터리 수명: 수개월-수년)
  • 비용: 디바이스당 $0.50-$5

TinyML은 센서 노드, 웨어러블, 스마트 홈 디바이스, 산업 IoT 센서 등 수십억 개의 초저전력 디바이스에 AI를 제공합니다.

TinyML의 주요 특징

초저전력 작동

TinyML 디바이스는 배터리나 에너지 하베스팅(태양광, 진동, 열)으로 작동합니다. 추론당 에너지 소비는 마이크로와트 범위여야 합니다:

  • 항상 켜짐 감지: 10-100 μW (배터리 수명: 1-5년)
  • 주기적 추론: 100-1000 μW 평균 (배터리 수명: 몇 개월)
  • 이벤트 기반: <10 μW 대기, 트리거 시 1-10 mW

극단적 모델 압축

TinyML 모델은 극도로 작아야 합니다. 일반적인 크기:

  • 초소형: 1-10 KB (간단한 분류)
  • 소형: 10-100 KB (키워드 감지, 간단한 이미지)
  • 중형: 100KB-1MB (복잡한 이미지/오디오)

실시간 추론

마이크로컨트롤러는 추론 시간이 예측 가능하고 결정론적입니다. OS 오버헤드가 없어 1-100ms의 일관된 지연시간을 보장합니다.

TinyML 하드웨어 플랫폼

Cortex-M4 80-200 MHz 64-256 KB 오디오/신호 처리
Cortex-M7 400-600 MHz 512KB-1MB 이미지 처리

전용 TinyML 칩

  • Syntiant NDP120: 항상 켜짐 오디오 ML (140 μW)
  • Eta Compute TENSAI: 마이크로와트급 AI 가속
  • GreenWaves GAP9: 이미지/오디오용 초저전력 프로세서

TensorFlow Lite Micro

개요

TensorFlow Lite Micro는 마이크로컨트롤러용으로 설계된 TensorFlow Lite의 경량 버전입니다. 주요 특징:

  • C++ 런타임 (동적 메모리 할당 없음)
  • 16 KB 코드 풋프린트
  • FP32, INT8, INT16 양자화 지원
  • ARM, ESP32, Arduino, RISC-V 지원

TFLite Micro로 모델 배포

// 1. Python에서 TFLite 모델 변환
import tensorflow as tf

converter = tf.lite.TFLiteConverter.from_keras_model(model)
converter.optimizations = [tf.lite.Optimize.DEFAULT]
converter.target_spec.supported_ops = [tf.lite.OpsSet.TFLITE_BUILTINS_INT8]
tflite_model = converter.convert()

# C 배열로 변환
with open('model.cc', 'w') as f:
    f.write('alignas(8) const unsigned char model_data[] = {')
    f.write(','.join([str(b) for b in tflite_model]))
    f.write('};')

// 2. 마이크로컨트롤러에서 모델 로드
#include "tensorflow/lite/micro/micro_interpreter.h"
#include "tensorflow/lite/micro/micro_mutable_op_resolver.h"
#include "model.h"

// 오퍼레이터 등록
static tflite::MicroMutableOpResolver<3> resolver;
resolver.AddFullyConnected();
resolver.AddRelu();
resolver.AddSoftmax();

// 인터프리터 생성 (정적 메모리 버퍼 사용)
constexpr int kTensorArenaSize = 8 * 1024;
uint8_t tensor_arena[kTensorArenaSize];

static tflite::MicroInterpreter interpreter(
    model, resolver, tensor_arena, kTensorArenaSize);
interpreter.AllocateTensors();

// 3. 추론 실행
TfLiteTensor* input = interpreter.input(0);
for (int i = 0; i < input->bytes; i++) {
    input->data.uint8[i] = sensor_data[i];
}

interpreter.Invoke();

TfLiteTensor* output = interpreter.output(0);
int8_t max_score = output->data.int8[0];
int max_index = 0;
for (int i = 1; i < output->dims->data[0]; i++) {
    if (output->data.int8[i] > max_score) {
        max_score = output->data.int8[i];
        max_index = i;
    }
}

Serial.print("예측: "); Serial.println(max_index);

TinyML 사용 사례

키워드 감지 (Wake Word Detection)

"Hey Google", "Alexa"와 같은 웨이크 워드를 항상 켜짐 모드로 감지합니다. 요구사항:

  • 모델 크기: 10-50 KB
  • 추론 시간: 20-50 ms (오디오 프레임당)
  • 전력: 100-500 μW
  • 정확도: >95% (낮은 False Positive Rate)
// 키워드 감지 예제
// 오디오 특징 추출 (MFCC)
const int kFeatureSliceSize = 40;
const int kFeatureSliceCount = 49;

void extract_features(int16_t* audio_buffer, int8_t* features) {
    // MFCC (Mel-frequency Cepstral Coefficients) 추출
    for (int slice = 0; slice < kFeatureSliceCount; slice++) {
        int16_t* audio_slice = &audio_buffer[slice * 160];
        float mfcc[kFeatureSliceSize];

        // FFT, Mel 필터뱅크, DCT 적용
        compute_mfcc(audio_slice, mfcc);

        // 양자화하여 저장
        for (int i = 0; i < kFeatureSliceSize; i++) {
            features[slice * kFeatureSliceSize + i] =
                quantize(mfcc[i]);
        }
    }
}

// 추론
extract_features(audio_buffer, input_features);
memcpy(input->data.int8, input_features, input->bytes);
interpreter.Invoke();

// 결과: 0 = 배경, 1 = "hey google"
int prediction = output->data.int8[0] < output->data.int8[1] ? 1 : 0;
if (prediction == 1) {
    wake_main_processor();
}

예측 유지보수

진동 센서를 사용하여 기계 고장을 예측합니다:

  • 진동 패턴 분석
  • 정상/이상 분류
  • 고장 전 경고 제공
  • 연결 없이 로컬에서 작동
// 진동 이상 감지
const int kWindowSize = 128;
const int kNumFeatures = 16;

void extract_vibration_features(float* accel_data,
                                float* features) {
    // 시간 영역 특징
    features[0] = mean(accel_data, kWindowSize);
    features[1] = std_dev(accel_data, kWindowSize);
    features[2] = rms(accel_data, kWindowSize);
    features[3] = peak_to_peak(accel_data, kWindowSize);

    // 주파수 영역 특징
    float fft_output[kWindowSize/2];
    compute_fft(accel_data, fft_output);
    features[4] = dominant_frequency(fft_output);
    features[5] = spectral_centroid(fft_output);
    // ... 추가 특징
}

// 100Hz 샘플링, 1.28초 윈도우
float accel_buffer[kWindowSize];
collect_samples(accel_buffer, kWindowSize);

float features[kNumFeatures];
extract_vibration_features(accel_buffer, features);

// INT8 양자화 및 추론
quantize_and_infer(features, kNumFeatures);

// 0 = 정상, 1 = 이상
if (output->data.int8[0] < output->data.int8[1]) {
    send_alert("기계 이상 감지!");
}

TinyML 모델 설계 원칙

아키텍처 선택

TinyML에 적합한 모델 아키텍처:

  • Fully Connected (FC): 10-100 KB, 간단한 분류
  • 1D CNN: 오디오/센서 시계열 데이터
  • Depthwise Separable Conv: 효율적인 2D CNN
  • MobileNetV2/V3: 이미지 (100KB-1MB)
  • Simple RNN/LSTM: 시퀀스 모델 (주의: 메모리 집약적)

양자화 전략

TinyML은 공격적인 양자화가 필요합니다:

  • INT8: 표준, 4배 압축
  • INT16: DSP가 있는 MCU용
  • Binary/Ternary: {-1, 0, +1} 가중치, 32배 압축
// INT8 양자화 예제
# 훈련 후 양자화
converter = tf.lite.TFLiteConverter.from_keras_model(model)
converter.optimizations = [tf.lite.Optimize.DEFAULT]

def representative_dataset():
    for _ in range(100):
        data = np.random.rand(1, 49, 40, 1).astype(np.float32)
        yield [data]

converter.representative_dataset = representative_dataset
converter.target_spec.supported_ops = [
    tf.lite.OpsSet.TFLITE_BUILTINS_INT8
]
converter.inference_input_type = tf.int8
converter.inference_output_type = tf.int8

tflite_quant_model = converter.convert()

메모리 최적화

메모리는 TinyML의 가장 큰 제약입니다:

  • In-place 연산: 입력 버퍼 재사용
  • 버퍼 재사용: 레이어 간 메모리 공유
  • 가중치 압축: 플래시에 압축, 런타임에 압축 해제
  • 스트리밍 추론: 데이터를 청크로 처리

TinyML 개발 워크플로우

1. 데이터 수집

실제 타겟 하드웨어에서 데이터를 수집합니다:

  • 동일한 센서 사용
  • 동일한 샘플링 레이트
  • 실제 환경 조건
  • 다양한 사용자/조건

2. 모델 훈련

클라우드/PC에서 FP32로 훈련합니다:

# Keras로 모델 정의
model = tf.keras.Sequential([
    tf.keras.layers.Conv1D(8, 3, activation='relu',
                           input_shape=(49, 40, 1)),
    tf.keras.layers.MaxPooling1D(2),
    tf.keras.layers.Conv1D(16, 3, activation='relu'),
    tf.keras.layers.Flatten(),
    tf.keras.layers.Dense(16, activation='relu'),
    tf.keras.layers.Dense(2, activation='softmax')
])

model.compile(optimizer='adam',
              loss='categorical_crossentropy',
              metrics=['accuracy'])

model.fit(train_data, train_labels, epochs=50)

3. 양자화 및 변환

TFLite로 변환하고 INT8로 양자화합니다.

4. 벤치마킹

타겟 하드웨어에서 측정합니다:

  • 추론 시간
  • 메모리 사용량 (RAM, 플래시)
  • 에너지 소비
  • 정확도 (양자화 후)

5. 최적화 반복

필요시 모델 아키텍처를 조정합니다:

  • 레이어 수 줄이기
  • 필터/뉴런 수 감소
  • 더 공격적인 양자화
  • 프루닝 적용

TinyML의 과제와 해결책

과제 1: 극한 리소스 제약

해결책:

  • 모델 아키텍처 단순화
  • 공격적인 양자화 (INT8, INT4, 바이너리)
  • 지식 증류로 작은 모델 훈련
  • NAS로 효율적인 아키텍처 찾기

과제 2: 실제 디바이스 디버깅

해결책:

  • UART/Serial 로깅
  • JTAG 디버거 사용
  • PC에서 먼저 검증
  • 단위 테스트

과제 3: 전력 소비

해결책:

  • 듀티 사이클링 (주기적 슬립)
  • 이벤트 기반 웨이크업
  • 계층적 처리 (간단한 모델 먼저)
  • 저전력 센서 선택

弘益人間 TinyML 원칙:

TinyML은 인터넷 연결이나 클라우드 인프라가 없는 세계 어디에서나 AI를 제공합니다. $1 마이크로컨트롤러에 AI를 탑재하면 수십억 개의 디바이스에 지능을 제공하여 농업, 의료, 환경 모니터링 등 모든 곳에서 인류에게 이익을 줍니다.

요약

TinyML은 극도로 리소스가 제한된 마이크로컨트롤러에서 머신러닝을 실행합니다:

  • 하드웨어: 1KB-1MB RAM, 1-100 MHz, 1-100 mW
  • 모델: 1-100 KB, INT8/INT16 양자화, 단순 아키텍처
  • 용도: 키워드 감지, 예측 유지보수, 제스처 인식, 환경 모니터링
  • 프레임워크: TensorFlow Lite Micro, Edge Impulse, STM32Cube.AI
  • 장점: 초저전력, 오프라인 작동, 저비용, 프라이버시

TinyML은 수십억 개의 IoT 디바이스에 AI를 제공하여 이전에는 불가능했던 애플리케이션을 가능하게 합니다.
