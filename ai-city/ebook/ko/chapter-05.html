5.1 REST API 설계 원칙

WIA AI-City API는 RESTful 규칙을 따르며, 모든 준수 구현에서 일관성을 보장하기 위한 추가 제약 조건을 가지고 있습니다.

기본 URL 구조

https://api.{도시-도메인}/wia/v1/{리소스}

예시:

  • https://api.seoul.ai-city.kr/wia/v1/traffic
  • https://api.barcelona.smart-city.es/wia/v1/environment
  • https://api.singapore.gov.sg/wia/v1/citizens

표준 응답 형식

{
  "@context": "https://schema.wia.org/ai-city/v1",
  "success": true,
  "data": { ... },
  "meta": {
    "requestId": "REQ-2024-ABC123",
    "timestamp": "2024-12-24T10:30:00Z",
    "pagination": {
      "page": 1,
      "pageSize": 20,
      "totalItems": 156,
      "totalPages": 8
    }
  }
}

오류 응답 형식

{
  "@context": "https://schema.wia.org/ai-city/v1",
  "success": false,
  "error": {
    "code": "CONSENT_REQUIRED",
    "message": "위치 데이터 접근에는 시민 동의가 필요합니다",
    "details": {
      "dataType": "L2",
      "requiredScope": "location:read"
    },
    "helpUrl": "https://docs.wia.org/errors/CONSENT_REQUIRED"
  },
  "meta": {
    "requestId": "REQ-2024-XYZ789",
    "timestamp": "2024-12-24T10:30:00Z"
  }
}

5.2 핵심 API 엔드포인트

교통 도메인

GET/traffic/sensors

필터링 옵션과 함께 모든 교통 센서 목록 조회

쿼리 매개변수:

  • zone - 구역 ID로 필터링
  • status - 상태로 필터링 (active, maintenance, offline)
  • type - 센서 유형으로 필터링

GET/traffic/sensors/{sensorId}/readings

특정 센서의 과거 측정값 조회

GET/traffic/congestion

실시간 도시 전체 혼잡도 상태

환경 도메인

GET/environment/air-quality

구역별 현재 대기질 지수

GET/environment/weather

현재 기상 조건 및 예보

나눔 엔진 도메인

GET/sharing/stats

집계된 기여도 통계

GET/sharing/allocations

카테고리별 펀드 배분 세부사항

5.3 시민 포털 API

시민 포털 API는 데이터 주권 권리를 위한 기술적 기반을 제공합니다. 이러한 엔드포인트를 통해 시민은 자신의 개인 데이터를 조회하고 관리하며 통제할 수 있습니다.

동의 관리

GET/citizens/me/consents

모든 활성 동의 승인 목록 조회

POST/citizens/me/consents

데이터 사용에 대한 새로운 동의 부여

{
  "dataType": "L2",
  "purpose": "traffic_optimization",
  "scope": ["location", "travel_patterns"],
  "expiresAt": "2025-12-24T00:00:00Z"
}

DELETE/citizens/me/consents/{consentId}

기존 동의 철회

데이터 접근

GET/citizens/me/data

도시가 보유한 모든 내 데이터 조회

GET/citizens/me/access-log

누가 언제 내 데이터에 접근했는지 확인

POST/citizens/me/data/export

데이터 내보내기 요청 (이동권)

DELETE/citizens/me/data

완전한 데이터 삭제 요청 (삭제권)

5.4 GraphQL 인터페이스

복잡한 쿼리와 실시간 구독을 위해 WIA AI-City는 REST와 함께 GraphQL 인터페이스를 제공합니다.

스키마 개요

type Query {
  traffic(zone: ID, limit: Int): TrafficData
  environment(type: EnvironmentType): EnvironmentData
  sharing: SharingStats
  citizen: CitizenProfile
}

type Mutation {
  grantConsent(input: ConsentInput!): Consent
  revokeConsent(consentId: ID!): Boolean
  requestDataExport: ExportRequest
  requestDataDeletion: DeletionRequest
}

type Subscription {
  trafficUpdates(zone: ID): TrafficReading
  alertsForZone(zone: ID!): Alert
  congestionChanges: CongestionEvent
}

쿼리 예시

# 중첩된 센서 정보와 함께 교통 데이터 가져오기
query {
  traffic(zone: "gangnam") {
    sensors {
      id
      status
      lastReading {
        vehicleCount
        averageSpeed
        timestamp
      }
    }
    congestionLevel
  }
}
# 실시간 교통 업데이트 구독
subscription {
  trafficUpdates(zone: "gangnam") {
    sensorId
    vehicleCount
    averageSpeed
    congestionLevel
    timestamp
  }
}

5.5 인증 및 권한 부여

WIA AI-City는 다양한 사용 사례를 위한 여러 승인 유형을 지원하는 OpenID Connect와 함께 OAuth 2.0을 사용합니다.

지원되는 승인 유형

승인 유형 사용 사례 토큰 수명
Authorization Code + PKCE 시민 대면 애플리케이션 1시간 (갱신: 30일)
Client Credentials 기계 간 통신 (도시 시스템) 1시간
Device Authorization IoT 장치, 키오스크 15분 (갱신: 90일)

범위 계층 구조

# 데이터 유형 범위 (분류 수준)
L1:*         # 모든 L1 데이터 (외부에 절대 부여되지 않음)
L2:read      # 위치 데이터 읽기
L2:write     # 위치 기본 설정 업데이트
L3:read      # 행동 데이터 읽기
L4:read      # 공개 데이터 읽기 (기본값)

# 도메인 범위
traffic:read
environment:read
sharing:read
citizen:profile:read
citizen:consents:manage
citizen:data:export
citizen:data:delete

토큰 요청 예시

POST /oauth/token
Content-Type: application/x-www-form-urlencoded

grant_type=authorization_code
&code=AUTH_CODE_HERE
&redirect_uri=https://myapp.example.com/callback
&client_id=APP_CLIENT_ID
&code_verifier=PKCE_VERIFIER

// 클라이언트 초기화
const client = new AICityClient({
  baseUrl: 'https://api.seoul.ai-city.kr/wia/v1',

// 교통 데이터
const traffic = await client.traffic.getCongestion({ zone: 'gangnam' });

// 시민 동의 관리
const consents = await client.citizen.listConsents();
await client.citizen.revokeConsent('CST-ABC123');

Rust SDK

// Cargo.toml
[dependencies]
wia-ai-city = "1.0"
use wia_ai_city::{Client, Result};

#[tokio::main]
async fn main() -> Result<()> {
    let client = Client::from_env().await?;

    // 교통 센서 조회
    let sensors = client.traffic().sensors()
        .zone("gangnam")
        .status("active")
        .fetch().await?;

    for sensor in sensors {
        println!("{}: {} 대", sensor.id, sensor.reading.vehicle_count);
    }

    Ok(())
}

5.8 웹훅 통합

이벤트 기반 아키텍처를 위해 WIA AI-City는 실시간 알림을 위한 웹훅을 지원합니다.

웹훅 등록

POST /webhooks
{
  "url": "https://myapp.example.com/wia-webhook",
  "events": [
    "traffic.congestion.high",
    "environment.alert.air_quality",
    "sharing.contribution.milestone"
  ],
  "secret": "webhook_signing_secret"
}

웹훅 페이로드

{
  "@context": "https://schema.wia.org/ai-city/v1",
  "@type": "WebhookEvent",
  "event": "traffic.congestion.high",
  "timestamp": "2024-12-24T10:30:00Z",
  "data": {
    "zone": "gangnam",
    "congestionLevel": 0.85,
    "affectedRoutes": ["route-a", "route-b"]
  },
  "signature": "sha256=abc123..."
}

웹훅 보안

서명 검증: 모든 웹훅 페이로드는 HMAC-SHA256으로 서명됩니다. 수신자는 등록 시 제공한 비밀 키를 사용하여 서명을 검증해야 합니다.

const crypto = require('crypto');

function verifyWebhookSignature(payload, signature, secret) {
  const hmac = crypto.createHmac('sha256', secret);
  const digest = 'sha256=' + hmac.update(payload).digest('hex');
  return crypto.timingSafeEqual(
    Buffer.from(signature),
    Buffer.from(digest)
  );
}

5.9 API 버전 관리

WIA AI-City API는 의미론적 버전 관리를 사용하며, 주요 버전은 URL 경로에 포함됩니다.

변경 유형 버전 영향 하위 호환성
새 엔드포인트 추가 부 버전 증가 유지됨
새 선택적 필드 추가 부 버전 증가 유지됨
필수 필드 제거 주 버전 증가 손상됨
응답 형식 변경 주 버전 증가 손상됨

버전 지원 정책

중요: 각 주요 API 버전은 최소 24개월 동안 지원됩니다. 버전 지원 중단은 최소 12개월 전에 공지됩니다.

5.10 요약

핵심 요점

  • REST 설계: API는 JSON-LD 응답, 표준 오류 형식, 일관된 페이지네이션을 갖춘 RESTful 규칙을 따릅니다.
  • 핵심 엔드포인트: 교통, 환경, 나눔 엔진, 시민 도메인에 걸쳐 85개 이상의 표준 엔드포인트가 제공됩니다.
  • 시민 포털: 동의 관리, 데이터 접근, 내보내기, 삭제를 위한 API가 데이터 주권 원칙을 기술적으로 구현합니다.
  • GraphQL: 복잡한 사용 사례를 위해 유연한 쿼리와 실시간 구독이 REST를 보완합니다.
  • OAuth 2.0: 인증은 다양한 클라이언트를 위한 여러 승인 유형을 지원하는 OpenID Connect와 함께 OAuth 2.0을 사용합니다.
  • 범위: 세분화된 범위가 데이터 분류 수준 및 도메인 권한에 매핑됩니다.
  • 속도 제한: 계층별 속도 제한이 정당한 대용량 접근을 가능하게 하면서 시스템을 보호합니다.
  • SDK: TypeScript, Python, Rust용 공식 SDK가 통합 마찰을 줄입니다.
  • 웹훅: 이벤트 기반 통합이 중요한 도시 이벤트에 대한 실시간 알림을 가능하게 합니다.

5.12 다음 단계

다음: 6장 - 3단계: AIDC-Link 프로토콜

API가 애플리케이션 수준 접근을 제공하면, 3단계는 완전한 AIDC-Link 프로토콜 스택을 구현합니다:

  • 물리 계층: 광섬유, 5G/6G, 양자 준비 링크를 위한 네트워크 인프라 사양.
  • 메시지 프로토콜: 효율적인 노드 간 통신을 위한 바이너리 및 텍스트 메시지 형식.
  • 엣지 AI 배포: 엣지 컴퓨팅 노드 및 로컬 AI 처리 사양.
  • 보안 강화: 종단 간 암호화, 인증서 관리, 침입 탐지.
  • 연합: 도시 간 조정을 위한 도시 간 통신 프로토콜.

API는 애플리케이션에 말을 합니다. 이제 도시가 스스로에게 말하게 만드는 프로토콜을 구축해봅시다.
