[MED-003] WIA 생체신호 스트리밍 표준 전자책 | 제4장 (총 8장)

[MED-003] 제4장: Phase 1 - 신호 형식 표준

"데이터의 언어를 통일하면 모든 기기가 대화할 수 있다"

Phase 1은 WIA-MED-003의 기초로, 모든 생체신호 데이터의 표준 형식을 정의합니다.

4.1 기본 신호 구조

4.1.1 공통 메타데이터

모든 WIA-MED-003 신호는 다음 공통 필드를 포함해야 합니다:

{
  "standard": "WIA-MED-003",
  "version": "1.0.0",
  "signalType": "...",          // ECG, SPO2, BP, HRV 등
  "deviceId": "...",             // 기기 고유 식별자
  "deviceManufacturer": "...",   // 제조사 (선택)
  "deviceModel": "...",          // 모델명 (선택)
  "timestamp": "...",            // ISO 8601 형식
  "patientId": "...",            // 환자 ID (선택, 익명화)
  "sessionId": "...",            // 세션 ID (선택)
  "data": { ... }                // 신호별 데이터
}

4.1.2 타임스탬프 형식

모든 시간 정보는 ISO 8601 형식을 사용합니다:

형식: YYYY-MM-DDTHH:mm:ss.sssZ

예시:
- UTC 시간: "2025-01-15T10:30:00.000Z"
- 밀리초 정밀도: "2025-01-15T10:30:00.123Z"
- 타임존 포함: "2025-01-15T10:30:00.000+09:00" (KST)

권장사항:
✅ 항상 UTC 사용
✅ 밀리초 정밀도 유지
✅ 'Z' 접미사로 UTC 명시

4.2.2 ECG 리드 시스템

리드 시스템 리드 수 리드 이름 용도
단일 리드 1 Lead I 또는 유사 웨어러블, 리듬 모니터링
3-리드 3 I, II, III 기본 모니터링
5-리드 5 I, II, III, aVR, aVL 중환자실
12-리드 12 I, II, III, aVR, aVL, aVF, V1-V6 진단용, 병원

4.2.3 ECG 이상 분류

표준 이상 코드:

{
  "abnormalities": [
    {
      "type": "arrhythmia",
      "subtype": "atrial_fibrillation",
      "severity": "moderate",
      "confidence": 0.95,
      "timestamp": "2025-01-15T10:30:05.230Z"
    },
    {
      "type": "morphology",
      "subtype": "st_elevation",
      "severity": "high",
      "confidence": 0.87,
      "leads": ["V1", "V2", "V3"]
    }
  ]
}

이상 유형:
- arrhythmia: 부정맥 (AF, VT, VF, PVC 등)
- morphology: 파형 이상 (ST 상승/하강, Q파 등)
- conduction: 전도 장애 (AV block, BBB 등)
- rate: 심박수 이상 (빈맥, 서맥)

4.3.2 SpO2 경고 임계값

{
  "standard": "WIA-MED-003",
  "version": "1.0.0",
  "signalType": "HRV",
  "deviceId": "HRV-DEVICE-22222",
  "timestamp": "2025-01-15T10:30:00.000Z",
  "measurement": {
    "duration": 300,             // 초 (5분 측정 권장)
    "rrIntervals": [800, 820, 810, 825, ...],  // ms
    "totalBeats": 360
  },
  "timeDomain": {
    "meanRR": 815,               // ms
    "sdnn": 50,                  // ms
    "rmssd": 42,                 // ms
    "pnn50": 15.2,               // %
    "sdsd": 35                   // ms
  },
  "frequencyDomain": {
    "vlf": 200,                  // ms² (Very Low Frequency: 0-0.04 Hz)
    "lf": 450,                   // ms² (Low Frequency: 0.04-0.15 Hz)
    "hf": 350,                   // ms² (High Frequency: 0.15-0.4 Hz)
    "lfHfRatio": 1.29,
    "totalPower": 1000           // ms²
  },
  "nonLinear": {
    "sd1": 30,                   // Poincaré plot
    "sd2": 70,
    "sampEn": 1.2                // Sample Entropy
  },
  "interpretation": {
    "stressLevel": "low",        // low, moderate, high
    "autonomicBalance": "balanced",
    "recovery": "good"
  }
}

4.5.2 HRV 지표 해석

지표 의미 정상 범위 해석
SDNN 전체 변동성 50-100 ms 높을수록 좋은 건강
RMSSD 부교감신경 활동 20-50 ms 회복 능력 반영
LF/HF 비율 자율신경 균형 1.5-2.0 높으면 스트레스
pNN50 심박 변동 정도 5-20% 건강한 변동성

4.9 데이터 압축 및 최적화

4.9.1 압축 방법

대용량 생체신호 데이터의 효율적 전송:

1. 델타 인코딩:
   원본: [100, 102, 104, 103, 105, 107]
   델타: [100, +2, +2, -1, +2, +2]
   압축률: ~40-60%

2. Run-Length Encoding:
   연속된 같은 값 압축
   [95, 95, 95, 95, 96] → [95×4, 96×1]

3. Gzip 압축:
   JSON 전체를 gzip으로 압축
   압축률: ~70-80%

4. 다운샘플링 (품질 유지):
   1000Hz → 500Hz (항-앨리어싱 필터 적용)
   데이터 크기 50% 감소

4.9.2 배치 전송

실시간성과 효율성 균형:

{
  "batch": {
    "batchId": "batch-123",
    "startTime": "2025-01-15T10:00:00.000Z",
    "endTime": "2025-01-15T10:05:00.000Z",
    "signals": [
      { /* 신호 1 */ },
      { /* 신호 2 */ },
      // ... 최대 100개 신호
    ],
    "compression": "gzip",
    "totalSize": 245678,         // bytes (압축 전)
    "compressedSize": 49135      // bytes (압축 후)
  }
}

4.10 장 요약

[OK] 핵심 내용:

  1. 공통 구조: 모든 신호는 표준 메타데이터 포함
  2. ECG: 다중 리드 지원, 파형 분석, 이상 감지
  3. SpO2: 산소포화도, PPG 파형, 품질 지표
  4. 혈압: 수축기/이완기, 연속 모니터링 지원
  5. HRV: 시간/주파수 영역 분석, 스트레스 평가
  6. 체온: 다양한 측정 위치, 연속 추적
  7. 호흡수: 호흡 패턴, 파형 분석
  8. JSON Schema: 엄격한 데이터 검증
  9. 최적화: 압축 및 배치 전송

  2. ECG 신호의 리드 시스템 차이점을 설명하세요 (1-리드 vs 12-리드).
