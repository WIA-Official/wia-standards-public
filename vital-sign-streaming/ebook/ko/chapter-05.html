"좋은 API는 복잡함을 숨기고 단순함을 드러낸다"

Phase 2는 개발자가 생체신호 기기와 쉽게 통신할 수 있도록 표준 API를 제공합니다.

5.1 코어 API 설계

5.1.1 VitalSignStream 클래스

모든 WIA-MED-003 구현의 중심이 되는 클래스입니다:

class VitalSignStream {
  // 생성자
  constructor(config: StreamConfig)

  // 장치 관리
  async discoverDevices(filter?: DeviceFilter): Promise<Device[]>
  async connect(deviceId: string): Promise<Connection>
  async disconnect(deviceId: string): Promise<void>
  getDeviceStatus(deviceId: string): DeviceStatus

  // 데이터 스트리밍
  async startStream(deviceId: string, config?: StreamOptions): Promise<Stream>
  async stopStream(streamId: string): Promise<void>
  onData(callback: (data: VitalSignData) => void): void
  onError(callback: (error: Error) => void): void

  // 데이터 저장 및 검색
  async saveData(data: VitalSignData, storage?: StorageConfig): Promise<string>
  async loadData(query: DataQuery): Promise<VitalSignData[]>
  async exportData(format: ExportFormat, query: DataQuery): Promise<Blob>

  // 알림 시스템
  setAlert(config: AlertConfig): string
  removeAlert(alertId: string): void
  onAlert(callback: (alert: Alert) => void): void
}

5.3 실시간 데이터 스트리밍

5.3.1 스트림 구성

interface StreamOptions {
  signalType: SignalType;
  samplingRate?: number;       // Hz (기본값은 신호 유형별)
  duration?: number;           // 초 (0 = 무한)
  compression?: 'none' | 'gzip' | 'delta';
  bufferSize?: number;         // 샘플 수
  realtime?: boolean;          // true = 즉시 전송, false = 배치
  filters?: {
    lowpass?: number;          // Hz
    highpass?: number;         // Hz
    notch?: number;            // Hz (50/60Hz 제거)
  };
}

// 예제: 고품질 ECG 스트리밍
const ecgStream = await stream.startStream(deviceId, {
  signalType: 'ECG',
  samplingRate: 1000,          // 1000Hz
  duration: 0,                 // 무한 스트리밍
  compression: 'delta',
  bufferSize: 100,
  realtime: true,
  filters: {
    lowpass: 100,              // 100Hz 저역통과
    highpass: 0.5,             // 0.5Hz 고역통과
    notch: 60                  // 60Hz 노치 필터 (전원 노이즈 제거)
  }
});

5.3.2 백프레셔 및 흐름 제어

네트워크 혼잡을 방지하기 위한 흐름 제어:

// 백프레셔 메커니즘
class StreamController {
  private buffer: VitalSignData[] = [];
  private maxBufferSize = 1000;
  private pauseThreshold = 800;
  private resumeThreshold = 200;

  onData(data: VitalSignData) {
    this.buffer.push(data);

    // 버퍼가 가득 차면 스트림 일시정지
    if (this.buffer.length > this.pauseThreshold) {
      this.pauseStream();
    }

    // 버퍼 처리
    this.processBuffer();

    // 버퍼가 비면 스트림 재개
    if (this.buffer.length < this.resumeThreshold) {
      this.resumeStream();
    }
  }

  private pauseStream() {
    // 장치에 PAUSE 신호 전송
    this.sendControl({ command: 'PAUSE' });
  }

  private resumeStream() {
    // 장치에 RESUME 신호 전송
    this.sendControl({ command: 'RESUME' });
  }
}

5.6 오류 처리

5.6.1 오류 유형

enum ErrorCode {
  // 연결 오류
  DEVICE_NOT_FOUND = 1001,
  CONNECTION_FAILED = 1002,
  CONNECTION_LOST = 1003,
  AUTHENTICATION_FAILED = 1004,

  // 스트리밍 오류
  STREAM_START_FAILED = 2001,
  STREAM_INTERRUPTED = 2002,
  BUFFER_OVERFLOW = 2003,
  DATA_CORRUPTED = 2004,

  // 데이터 오류
  INVALID_DATA_FORMAT = 3001,
  SCHEMA_VALIDATION_FAILED = 3002,
  STORAGE_FAILED = 3003,
  QUERY_FAILED = 3004,

  // 시스템 오류
  LOW_BATTERY = 4001,
  SENSOR_FAILURE = 4002,
  FIRMWARE_ERROR = 4003,
  NETWORK_ERROR = 4004
}

interface WIAError extends Error {
  code: ErrorCode;
  message: string;
  details?: any;
  timestamp: string;
  recoverable: boolean;
}

5.6.2 오류 처리 패턴

// 자동 재연결
stream.onError(async (error) => {
  if (error.code === ErrorCode.CONNECTION_LOST && error.recoverable) {
    console.log('연결 끊김. 재연결 시도 중...');

    for (let attempt = 1; attempt <= 5; attempt++) {
      try {
        await stream.connect(deviceId);
        console.log('재연결 성공!');
        return;
      } catch (e) {
        console.log(`재연결 시도 ${attempt}/5 실패`);
        await sleep(attempt * 1000);  // 지수 백오프
      }
    }

    console.error('재연결 실패. 사용자 개입 필요.');
    notifyUser('기기 연결이 끊겼습니다. 다시 연결해주세요.');
  }
});

// 데이터 손실 방지
stream.onError(async (error) => {
  if (error.code === ErrorCode.BUFFER_OVERFLOW) {
    // 버퍼 오버플로우 시 로컬 저장
    console.warn('버퍼 오버플로우. 로컬 저장으로 전환.');
    await stream.changeStorage({ type: 'local' });
  }
});

// 센서 오류 감지
stream.onError((error) => {
  if (error.code === ErrorCode.SENSOR_FAILURE) {
    console.error('센서 오류:', error.details);

    // 사용자에게 안내
    showError({
      title: '센서 오류',
      message: '센서를 다시 부착하거나 기기를 재시작해주세요.',
      actions: [
        { label: '재시작', onClick: () => stream.restart(deviceId) },
        { label: '지원팀 연락', onClick: () => contactSupport() }
      ]
    });
  }
});

  4. 데이터 저장: 로컬/클라우드, 쿼리, 내보내기
  5. 알림 시스템: 규칙 기반, 복합 조건, 다중 채널
  6. 오류 처리: 자동 복구, 사용자 알림
