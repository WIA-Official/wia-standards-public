4.6 Frame Envelope (canonical)

The Phase 1 specification fixes a single canonical frame envelope that wraps any vital signal type. Each frame is the unit of streaming, signing, and replay defence. Required fields: wia_vital_sign_streaming_version, type, stream_id, patient_id, channel, sample_rate_hz, captured_at, samples, encoding, signature. Optional fields include calibration, lead_id, frame_seq, qos, and compression.

{
  "wia_vital_sign_streaming_version": "1.0.0",
  "type": "frame",
  "stream_id": "stream-icu-A-bed-7-ecg",
  "patient_id": "did:wia:patient:01HXY",
  "channel": "ecg",
  "lead_id": "II",
  "sample_rate_hz": 500,
  "captured_at": "2026-04-27T10:00:00.000Z",
  "encoding": "int16",
  "calibration": { "lsb": 0.000244, "offset": 0, "unit": "mV" },
  "frame_seq": 12483017,
  "qos": "assured",
  "samples": [3, 4, 5, 4, 3, 2, 1, 0, -1, -2, -1, 0, 1, 2, 3, 4],
  "signature": { "alg": "Ed25519", "value": "AY3o..." }
}

4.7 Channel Descriptor

For high-rate streams the producer MAY publish a one-shot channel descriptor that the receiver caches; subsequent frames omit the descriptor's fields and inherit them. This optimisation matters for ECG and EEG streams where per-frame metadata overhead would otherwise dominate the wire bandwidth.

{
  "wia_vital_sign_streaming_version": "1.0.0",
  "type": "channel_descriptor",
  "stream_id": "stream-icu-A-bed-7-ecg",
  "channel": "ecg",
  "default_calibration": { "lsb": 0.000244, "offset": 0, "unit": "mV" },
  "default_encoding": "int16",
  "default_lead_id": "II",
  "issued_at": "2026-04-27T08:00:00Z",
  "valid_until": "2026-04-27T20:00:00Z",
  "signature": { "alg": "Ed25519", "value": "Bz9A..." }
}

4.8 Reserved Channels

The canonical channel registry includes ecg, ppg, eeg, emg, spo2, hr, sbp, dbp, map, temp, rr, and etco2. Implementations MUST treat unknown channels as unrecognised rather than fail the frame. Future minor versions add channels but never remove them.

4.9 Encoding Rules

  • UTF-8 JSON per IETF RFC 8259, snake_case keys.
  • Timestamps RFC 3339 in UTC with Z suffix; ECG and other ≥ 100 Hz channels MUST encode millisecond precision.
  • Numbers fit in IEEE 754 double precision; integer fields fit in signed 64-bit.
  • Binary fields base64url without padding per IETF RFC 4648 §5.
  • The frame MUST be self-describing: a receiver presented with one envelope MUST be able to reconstruct calibrated values without out-of-band metadata.

4.10 Conformance

A Phase 1 conformant implementation MUST emit and accept the frame envelope above with all required fields, honour the channel descriptor inheritance rules, round-trip every frame byte-identically through encode/decode when no normalisation is requested, reject frames that violate the JSON Schema, and treat unknown optional fields as non-fatal extensions.
