4.1 Protocol Embracer (프로토콜 수용자)

The Protocol Embracer is the heart of WIA-OMNI-API's "mother" philosophy. It accepts and understands all API protocols, current and future, adapting to each client's preferred communication style.

Supported Protocols

4.2 Intent Interpreter (의도 해석기)

The Intent Interpreter understands what clients want to achieve, not just what they literally request. It supports multiple levels of intent expression.

Intent Levels

Level 1: Protocol-Specific (Traditional)

// REST
GET /api/users/123

// GraphQL
query { user(id: 123) { name email } }

// gRPC
UserService.GetUser({ user_id: 123 })

Level 2: Natural Language

{
  "intent": "Get user 123's profile information"
}

// Mother interprets and executes optimal query

Level 3: WIA-INTENT (Structured Intent)

{
  "intent": {
    "type": "wia-intent",
    "code": `
      intent GetUserProfile {
        given: user_id = 123
        want: user_profile
        constraints {
          include: [name, email, avatar]
          exclude: [password, internal_id]
          freshness: <= 5.minutes
        }
        certainty: >= 0.95
      }
    `
  }
}

Level 4: Contextual Intent

{
  "intent": "show me his orders too",
  "context": {
    "session_id": "abc123",
    "previous_requests": [
      "get user 123"
    ]
  }
}

// Mother understands "his" refers to user 123 from context

4.3 Protocol Auto-Selection

When clients don't specify a protocol preference, Mother selects the optimal one based on multiple factors.

Selection Criteria

Scenario Optimal Protocol Reason
Browser, simple query REST Wide support, cacheable
Complex nested data GraphQL Precise field selection
High-performance service gRPC Binary encoding, type-safe
Real-time updates WebSocket Bidirectional, low latency
IoT device, limited bandwidth MQTT Lightweight, QoS support
Unknown/AI client Intent-based Highest abstraction

Auto-Selection Example

// Client request (no protocol specified)
{
  "intent": "subscribe to user 123's activity updates"
}

// Mother's decision process:
{
  "analysis": {
    "intent_type": "subscription",
    "data_pattern": "stream",
    "client_type": "browser",
    "network_quality": "good"
  },
  "protocol_scores": {
    "websocket": 0.95,  // ✓ Real-time, browser-friendly
    "graphql_subscription": 0.85,  // Good, but heavier
    "mqtt": 0.40,  // Not typical for browser
    "rest_polling": 0.20  // Inefficient for real-time
  },
  "selected": "websocket",
  "reason": "Real-time subscription from browser"
}

4.4 Protocol Interoperability

Different protocols can interoperate seamlessly through Mother's translation layer.

Cross-Protocol Communication

// REST client subscribes to GraphQL subscription
// Client sends (REST polling):
GET /api/subscribe/user/123/activity
Connection: keep-alive

// Mother translates to GraphQL subscription:
subscription {
  userActivity(userId: 123) {
    type
    timestamp
    details
  }
}

// Mother streams updates back via Server-Sent Events (SSE):
data: {"type": "login", "timestamp": "2024-01-01T10:00:00Z"}
data: {"type": "order", "timestamp": "2024-01-01T10:05:00Z"}

4.5 Error Handling Across Protocols

Mother normalizes error responses across all protocols, providing compassionate, actionable error messages.

Unified Error Format

{
  "success": false,
  "error": {
    "code": "USER_NOT_FOUND",
    "message": "User with ID 999 does not exist",
    "comfort": "Don't worry, this happens often.",
    "cause": "The requested user ID is not in the system",
    "solution": "Please verify the user ID or search by name",
    "http_status": 404,
    "grpc_status": "NOT_FOUND",
    "graphql_extensions": {
      "code": "USER_NOT_FOUND"
    },
    "recovery_options": [
      {
        "action": "search_similar",
        "description": "Search for users with similar names",
        "endpoint": "/users/search",
        "auto_applicable": false,
        "confidence": 0.7
      },
      {
        "action": "list_recent",
        "description": "Show recently active users",
        "endpoint": "/users/recent",
        "auto_applicable": true,
        "confidence": 0.5
      }
    ]
  }
}
