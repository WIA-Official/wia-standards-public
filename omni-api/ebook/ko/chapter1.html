제1장: 소개 및 개요

메타 API 개념 & 모든 프로토콜 수용

1.1 현재 API의 문제점

현대 소프트웨어 개발은 단편화된 API 환경에 직면해 있으며, 서로 다른 프로토콜이 서로 다른 목적을 수행하고, 각각 고유한 한계를 가지고 있습니다:

REST API → 제한된 동사, 오버페칭, 언더페칭 GraphQL → 복잡한 쿼리, 캐싱 어려움 gRPC → 브라우저 지원 제한, 가파른 학습 곡선 WebSocket → 복잡한 상태 관리, 리소스 집약적 MQTT → IoT 특화, 범용성 부족 결과: - 각각 따로 배워야 함 - 프로토콜 간 상호 운용성 없음 - 버전 변경 시 클라이언트 손상 - 클라이언트가 복잡성 부담
실제 비용: 개발자들은 실제 기능을 구축하는 것보다 API 호환성 관리, 버전 마이그레이션, 프로토콜 변환에 더 많은 시간을 소비합니다. 이러한 복잡성은 전통적으로 클라이언트에 전가되어 통합을 어렵게 만듭니다.

1.2 어머니의 철학

어머니는 자식이 무엇을 원하는지 들으면,
나머지는 모두 자동으로 처리해준다.

어머니는 자식에게 복잡함을 전가하지 않는다.
어머니는 자식이 변하라고 강요하지 않는다.
어머니는 모든 자식을 품는다.

A mother listens to what her child wants,
then handles everything else automatically.

한국 문화의 모성애 개념에 뿌리를 둔 이 철학은 WIA-OMNI-API의 기초를 형성합니다. 어머니가 엄격한 규칙을 강요하는 대신 각 자식의 고유한 요구에 적응하듯이, 이 API 표준은 각 클라이언트의 능력과 선호도에 맞춰집니다.

1.3 WIA-OMNI-API 원칙

1. 모두 품기 (Embrace All)

차별 없이 모든 프로토콜, 포맷, 버전을 지원합니다. 클라이언트가 REST, GraphQL, gRPC, WebSocket, MQTT, 또는 아직 발명되지 않은 프로토콜로 통신하든, 게이트웨이는 이해하고 적절히 응답합니다.

2. 자식에게 맞추기 (Adapt to Child)

서버가 클라이언트에 맞추며, 그 반대는 없습니다. 레거시 클라이언트가 이전 버전이나 포맷을 사용하면 서버가 자동으로 변환합니다. 호환성의 부담은 항상 서버(어머니)에게 있으며, 클라이언트(자식)에게 없습니다.

3. 자기 진화 (Self-Evolving)

사용 패턴으로부터 학습하고 자동으로 진화합니다. 시스템은 클라이언트가 API를 어떻게 사용하는지 관찰하고 그에 따라 최적화합니다 - 자주 요청되는 데이터 캐싱, 일반적으로 함께 요청되는 리소스 번들링, 개선 사항 제안.

4. 절대 깨지 않기 (Never Break)

절대적인 하위 호환성을 유지합니다. 일단 API 계약이 수립되면 영원히 지원됩니다. 클라이언트는 서버 변경으로 인해 업데이트할 필요가 없습니다. 이것이 WIA-OMNI-API의 신성한 약속입니다.

5. 의도 기반 (Intent-Based)

문자 그대로의 요청이 아니라 클라이언트의 의도를 이해합니다. WIA-INTENT와의 통합을 통해 클라이언트가 달성하고자 하는 것을 표현할 수 있으며, 게이트웨이가 그 의도를 충족하는 최적의 방법을 결정합니다.

1.4 메타 API 개념

WIA-OMNI-API는 단순히 또 다른 API 프로토콜이 아닙니다 - 모든 기존 프로토콜 위에 위치하는 메타 프로토콜로, 각 기본 기술의 강점을 보존하면서 통합된 인터페이스를 제공합니다.

1.5 다양한 이해관계자를 위한 혜택

프론트엔드 개발자

  • 자연어나 간단한 의도 문장으로 요청 작성
  • 여러 API 프로토콜을 배울 필요 없음
  • 자동 에러 복구 및 재시도 로직
  • API 버전 업데이트 걱정 불필요

백엔드 개발자

  • 한 번 구축하면 모든 프로토콜 자동 지원
  • 중단 없이 점진적 진화
  • 사용 패턴 기반 자동 최적화
  • 유지보수 부담 감소

DevOps 팀

  • 모니터링하고 보안할 단일 게이트웨이
  • 모든 API 진화의 포괄적인 감사 추적
  • 학습된 패턴 기반 자동 스케일링
  • 서비스 메시의 복잡성 감소

비즈니스 이해관계자

  • 더 빠른 기능 개발
  • 낮은 통합 비용
  • 더 나은 고객 경험
  • 미래 보장 기술 투자

1.6 홍익인간 (弘益人間) - 창립 정신

홍익인간 - "모든 인류에게 이익을" - 의 원칙은 수천 년 동안 한국 문명을 이끌어왔습니다. WIA-OMNI-API는 이 시대를 초월한 지혜를 현대 API 설계에 적용합니다:

  • 보편적 이익: 기술이나 버전에 관계없이 모든 클라이언트가 혜택
  • 포용적 설계: 클라이언트를 뒤에 남겨두지 않음; 모두 동등하게 포용
  • 적응적 돌봄: 어머니처럼 시스템이 개별 요구에 맞춤
  • 지속 가능한 성장: 파괴가 아닌 강화하는 진화
