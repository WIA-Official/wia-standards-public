Security & Privacy

Military-grade encryption and privacy-preserving technologies protecting sensitive communications

🔐 Security First, Privacy Always

Inter-Korean communication operates in one of the most sensitive geopolitical contexts in the world. Both governments have legitimate security concerns. Families have legitimate privacy needs. International observers need verification capabilities. WIA-UNI-001 balances all these requirements through advanced cryptographic techniques.

Core Principle: Security and privacy are not negotiable. Every message must be:
  • Encrypted end-to-end so only sender and recipient can read it
  • Authenticated so everyone knows it's genuine
  • Verifiable so observers can confirm proper handling
  • Private so no third party can access content

Military-Grade Encryption

AES-256 with Perfect Forward Secrecy

Every message transmitted through WIA-UNI-001 is encrypted using AES-256, the same encryption standard used by governments worldwide to protect top-secret information. But we go further: we implement Perfect Forward Secrecy.

Perfect Forward Secrecy means that even if an encryption key is somehow compromised in the future, it cannot be used to decrypt past messages. Each message uses a unique, ephemeral key that is destroyed immediately after use. 

// Message encryption with perfect forward secrecy
async function encryptMessage(message: string, recipientPublicKey: string) {
  // Generate ephemeral key pair for this message only
  const ephemeralKeys = await generateKeyPair();

  // Derive shared secret using ECDH
  const sharedSecret = await ecdh(
    ephemeralKeys.privateKey,
    recipientPublicKey
  );

  // Encrypt message with AES-256-GCM
  const encrypted = await aes256gcm.encrypt(message, sharedSecret);

  // Destroy ephemeral private key immediately
  await securelyDelete(ephemeralKeys.privateKey);

  return {
    ciphertext: encrypted,
    ephemeralPublicKey: ephemeralKeys.publicKey
  };
}

Zero-Knowledge Proofs

One of the most innovative aspects of WIA-UNI-001 is the use of zero-knowledge proofs. This cryptographic technique allows the system to verify that messages are legitimate without actually seeing their content.

For example, a zero-knowledge proof can verify:

  • "This message was sent by an authorized user" - without revealing who
  • "This message contains no prohibited content" - without reading it
  • "This message was delivered successfully" - without knowing what it said

This allows international observers to verify system integrity while protecting family privacy.

Multi-Layer Privacy Protection

Layer 1: Content Privacy

Message content is never stored unencrypted. It's never logged. It's never accessible to system administrators. Only the sender and recipient hold the keys.

Layer 2: Metadata Protection

Even metadata (who is talking to whom, when, how often) can be sensitive. WIA-UNI-001 minimizes metadata collection and uses differential privacy techniques to add statistical noise to aggregate data, making it impossible to identify individual communications.

Layer 3: Network Privacy

Messages are routed through multiple independent channels, making it difficult to trace the path of any specific message. This protects against network surveillance and traffic analysis.

Layer 4: Storage Privacy

Messages are stored encrypted. Encryption keys are split using Shamir's Secret Sharing across multiple trust anchors. No single party can decrypt stored messages. 

// Split encryption key across 4 trust anchors
// Requires any 3 to reconstruct
const keyShares = shamirSplit(encryptionKey, {
  shares: 4,
  threshold: 3
});

await distributeShares({
  rok: keyShares[0],
  dprk: keyShares[1],
  un: keyShares[2],
  icrc: keyShares[3]
});

// Key is destroyed after splitting
securelyDelete(encryptionKey);

Authentication & Non-Repudiation

Security isn't just about keeping secrets - it's also about knowing who sent what. Every message includes cryptographic signatures that prove:

  • Authentication: The message really came from the claimed sender
  • Integrity: The message wasn't modified in transit
  • Non-repudiation: The sender can't deny sending it later

Threat Model & Protections

Threat: Man-in-the-Middle Attacks

Protection: Certificate pinning, multi-path verification, trust anchor consensus

Threat: Network Surveillance

Protection: End-to-end encryption, multi-channel routing, traffic obfuscation

Threat: Server Compromise

Protection: Zero-knowledge architecture, no plaintext storage, key splitting

Threat: Coercion/Forced Disclosure

Protection: Plausible deniability modes, duress codes, distributed key management

Threat: Traffic Analysis

Protection: Constant-rate traffic padding, decoy messages, timing obfuscation

Privacy Rights & Guarantees

Privacy Guarantees for Families:
  • Your messages can ONLY be read by you and your family member
  • No government, including ROK or DPRK, can decrypt your messages
  • International observers can verify system integrity but cannot read content
  • Message content is NEVER stored permanently in readable form
  • You can request deletion of your messages at any time
  • You have full control over your data and communication
