📁 제3장: 디지털 슬라이드 형식 및 표준

파일 형식의 중요성

디지털 병리학의 성공은 표준화된 파일 형식에 달려 있습니다. 상호운용 가능한 형식은 시스템 간 데이터 교환을 가능하게 하고, 장기 보존을 보장하며, AI 알고리즘의 광범위한 적용을 지원합니다.

3.1 주요 파일 형식

3.1.1 DICOM (Digital Imaging and Communications in Medicine)

DICOM Supplement 145 - Whole Slide Microscopic Image IOD

역사:
  - 2010년 승인
  - 병리학 이미지를 DICOM 표준에 통합
  - PACS 시스템과의 호환성

구조:
  DICOM 파일
    ├─> 헤더 (Header)
    │   ├─> 환자 정보 (Patient Module)
    │   ├─> 연구 정보 (Study Module)
    │   ├─> 시리즈 정보 (Series Module)
    │   └─> 장비 정보 (Equipment Module)
    │
    └─> 이미지 데이터 (Pixel Data)
        ├─> 프레임 1 (타일 1)
        ├─> 프레임 2 (타일 2)
        └─> ... (모든 타일)

주요 태그:
  (0008,0060) Modality = "SM" (Slide Microscopy)
  (0028,0010) Rows = 타일 높이
  (0028,0011) Columns = 타일 너비
  (0048,0001) Imaged Volume Width
  (0048,0002) Imaged Volume Height
  (0048,0003) Total Pixel Matrix Rows
  (0048,0006) Total Pixel Matrix Columns

장점:
  ✅ 의료 표준 준수
  ✅ PACS 통합 용이
  ✅ 방사선 이미지와 통합 관리
  ✅ HL7 연동
  ✅ 보안 및 암호화 지원

단점:
  ❌ 복잡한 구조
  ❌ 큰 파일 크기
  ❌ 제한된 소프트웨어 지원

3.1.2 TIFF (Tagged Image File Format)

BigTIFF (TIFF 6.0 확장):
  - 4GB 이상 파일 지원
  - 64비트 오프셋
  - WSI에 적합

PyramidTIFF:
  - 다중 해상도 레벨
  - IFD (Image File Directory) 체인
  - 타일 기반 구조

구조:
  TIFF 파일
    ├─> IFD 0 (최고 해상도 - 40배율)
    │   ├─> 태그 (메타데이터)
    │   └─> 타일 오프셋 포인터
    │
    ├─> IFD 1 (20배율)
    ├─> IFD 2 (10배율)
    ├─> IFD 3 (5배율)
    ├─> IFD 4 (2.5배율)
    └─> IFD 5 (1.25배율)

주요 태그:
  256: ImageWidth
  257: ImageLength
  258: BitsPerSample (8, 8, 8 for RGB)
  259: Compression (1=None, 7=JPEG, 33003=AdobeDeflate)
  262: PhotometricInterpretation (2=RGB)
  322: TileWidth (256)
  323: TileLength (256)
  339: SampleFormat (1=unsigned integer)

장점:
  ✅ 간단한 구조
  ✅ 광범위한 소프트웨어 지원
  ✅ 유연한 압축 옵션
  ✅ 오픈 표준

단점:
  ❌ 표준화된 메타데이터 부족
  ❌ 의료 정보 통합 한계

특징:
  - TIFF + XML 메타데이터
  - 연구용 표준
  - 생물 의학 이미징 커뮤니티

장점:
  ✅ 풍부한 메타데이터
  ✅ 확장 가능
  ✅ 다차원 이미징 지원
  ✅ 오픈소스 도구 지원

단점:
  ❌ 복잡한 구조
  ❌ 임상 사용 제한적

3.1.4 독점 형식

형식 제조사 특징 기반 형식
SVS Aperio (Leica) 가장 널리 사용, TIFF 기반 TIFF
NDPI Hamamatsu 일본 시장 주도, TIFF 유사 TIFF
MRXS 3DHistech 멀티파일 구조, 유럽 시장 독점
SCN Leica (구형) 레거시 형식 독점
BIF Ventana (Roche) 압축 최적화 독점
VSI Olympus 멀티채널 지원 독점

독점 형식의 문제점:

❌ 상호운용성 부족
   - 각 형식마다 전용 리더 필요
   - 소프트웨어 개발 부담

❌ 벤더 종속
   - 제조사 변경 시 마이그레이션 어려움
   - 장기 보존 위험

❌ 제한된 도구
   - 오픈소스 도구 지원 부족
   - AI 개발 장애

❌ 표준화 저해
   - 시장 파편화
   - 혁신 억제

3.2 WIA-DPS (WIA Digital Pathology Standard) 형식

3.2.1 설계 원칙

1. 상호운용성 (Interoperability)
   → DICOM, TIFF 호환
   → 기존 도구와 연동

2. 확장성 (Extensibility)
   → 미래 기술 수용
   → 플러그인 아키텍처

3. 효율성 (Efficiency)
   → 최적화된 압축
   → 빠른 접근 속도

4. 단순성 (Simplicity)
   → 구현 용이
   → 명확한 사양

5. 개방성 (Openness)
   → 오픈 표준
   → 무료 라이선스

3.2.2 파일 구조

WIA-DPS v1.0 구조:

파일명: specimen_id.wdps (또는 .tif)

[헤더 섹션]
  - 매직 넘버: "WDPS" (4 bytes)
  - 버전: 1.0 (2 bytes)
  - 플래그: 압축, 암호화 등 (2 bytes)
  - 오프셋: 메타데이터, 이미지 데이터 (8 bytes each)

[메타데이터 섹션]
  - 형식: JSON
  - 압축: gzip
  - 구조:
    {
      "version": "1.0.0",
      "specimen": {...},
      "scan": {...},
      "image": {...},
      "clinical": {...},
      "extensions": {...}
    }

[이미지 피라미드]
  Level 0 (40배율, 0.25μm/픽셀)
    └─> 타일 0, 1, 2, ... (256×256 픽셀)

  Level 1 (20배율, 0.5μm/픽셀)
    └─> 타일 0, 1, 2, ...

  Level 2 (10배율, 1.0μm/픽셀)
    └─> 타일 0, 1, 2, ...

  ... (추가 레벨)

[주석 섹션] (선택)
  - 형식: GeoJSON
  - 마킹, 측정, 분류

[AI 결과 섹션] (선택)
  - 모델 출력
  - 확률 맵
  - 세그멘테이션

3.3 압축 및 저장 기술

3.3.1 압축 알고리즘 비교

알고리즘 유형 압축률 품질 속도 사용 사례
무압축 None 1:1 완벽 가장 빠름 품질 검증용
LZW 무손실 1.5:1~3:1 완벽 빠름 형광 이미지
Deflate 무손실 2:1~4:1 완벽 보통 IHC 이미지
JPEG 손실 10:1~20:1 우수 매우 빠름 H&E (표준)
JPEG2000 손실/무손실 15:1~30:1 뛰어남 느림 고품질 보관
WebP 손실/무손실 12:1~25:1 우수 빠름 웹 뷰어
HEIF 손실/무손실 20:1~40:1 뛰어남 보통 차세대 형식

3.3.2 WIA-MED-008 압축 권장사항

염색 유형별 권장 압축:

1. H&E (Hematoxylin & Eosin)
   권장: JPEG (품질 90)
   이유:
     - 높은 압축률 (15:1)
     - 육안으로 구별 불가능한 품질
     - 빠른 인코딩/디코딩
     - 광범위한 지원

   예시:
     원본: 5GB
     압축: 330MB (93% 감소)
     품질: 진단에 영향 없음

2. IHC (Immunohistochemistry)
   권장: JPEG2000 (무손실) 또는 Deflate
   이유:
     - 색상 정확도 중요
     - 정량적 분석 필요
     - 바이오마커 강도 측정

   예시:
     원본: 5GB
     압축: 1.7GB (66% 감소)
     품질: 완벽 보존

3. 형광 (Fluorescence)
   권장: 무손실 압축 (LZW, Deflate)
   이유:
     - 신호 강도 정량화
     - 공초점 현미경 데이터
     - 3D 재구성

   예시:
     원본: 8GB (4채널)
     압축: 3GB (63% 감소)
     품질: 완벽 보존

4. 멀티스펙트럴 (Multispectral)
   권장: JPEG2000 (무손실)
   이유:
     - 다중 채널 (6-10개)
     - 스펙트럼 분해 필요
     - 고급 분석

   예시:
     원본: 20GB (8채널)
     압축: 6GB (70% 감소)
     품질: 완벽 보존

3.3.3 적응형 압축 전략

영역별 차등 압축:

1. 관심 영역 (ROI)
   - 압축: 무손실 또는 고품질 JPEG (95)
   - 이유: 진단 정확도 최우선

2. 조직 배경
   - 압축: 중간 품질 JPEG (85)
   - 이유: 문맥 정보 제공, 덜 중요

3. 빈 영역 (배경 유리)
   - 압축: 저품질 JPEG (70) 또는 제외
   - 이유: 진단 가치 없음

예시:
  슬라이드 영역 분석:
    - ROI (종양): 10% → 무손실 (1GB)
    - 조직 배경: 60% → JPEG 85 (500MB)
    - 빈 영역: 30% → JPEG 70 (50MB)

  전체: 1.55GB (원본 5GB 대비 69% 감소)
  vs 균일 압축: 330MB (품질 손실)
       또는 1.7GB (과도한 크기)

  장점:
    ✅ 진단 품질 유지
    ✅ 파일 크기 최적화
    ✅ 스토리지 비용 절감

3.4 상호운용성 및 변환

3.4.1 형식 변환 도구

WIA-DPS Converter:

기능:
  1. 다중 형식 읽기
     - SVS, NDPI, MRXS, SCN, BIF
     - DICOM, OME-TIFF
     - 일반 TIFF

  2. WIA-DPS로 변환
     - 메타데이터 추출
     - 이미지 피라미드 재구성
     - 품질 검증

  3. 역변환
     - WIA-DPS → DICOM
     - WIA-DPS → OME-TIFF
     - 호환성 유지

사용 예시:
  # SVS를 WIA-DPS로 변환
  $ wia-dps-convert input.svs output.wdps \
      --compression jpeg \
      --quality 90 \
      --validate

  # 배치 변환
  $ wia-dps-convert *.svs \
      --output-dir ./converted \
      --format wdps \
      --parallel 4

  # DICOM으로 변환
  $ wia-dps-convert input.wdps output.dcm \
      --format dicom \
      --patient-id P123456 \
      --study-uid 1.2.3.4.5

3.5 장기 보존 전략

  ❌ 형식 노후화 (Format Obsolescence)

  ❌ 소프트웨어 호환성
  ❌ 하드웨어 노후화

해결 방안:
  ✅ 개방형 표준 사용 (WIA-DPS, DICOM)
  ✅ 정기적 마이그레이션
  ✅ 다중 복사본
  ✅ 클라우드 백업

3.5.2 보존 아키텍처

3-2-1 백업 규칙:

  3개의 복사본:
    1. 운영 스토리지 (온프레미스 SSD)
    2. 로컬 백업 (NAS, 테이프)
    3. 원격 백업 (클라우드)

  2개의 다른 미디어:
    - 디스크 (SSD/HDD)
    - 테이프 또는 클라우드

  1개의 오프사이트:
    - 지리적으로 분리된 위치
    - 재해 복구

스토리지 계층:

  Tier 1 (Hot): 최근 30일
    - SSD 스토리지
    - 즉시 접근
    - 높은 비용

  Tier 2 (Warm): 1개월~1년
    - HDD 스토리지
    - 빠른 접근 (초 단위)
    - 중간 비용

  Tier 3 (Cold): 1년~10년
    - 테이프 또는 클라우드 아카이브
    - 접근 시간: 시간 단위
    - 낮은 비용

  Tier 4 (Frozen): 10년 이상
    - 클라우드 glacier
    - 접근 시간: 일 단위
    - 매우 낮은 비용

비용 예시 (10,000 슬라이드, 3GB 평균):
  총 데이터: 30TB

  Tier 1 (10%): 3TB × $0.20/GB/월 = $600/월
  Tier 2 (20%): 6TB × $0.05/GB/월 = $300/월
  Tier 3 (50%): 15TB × $0.01/GB/월 = $150/월
  Tier 4 (20%): 6TB × $0.002/GB/월 = $12/월

  총: $1,062/월 ($12,744/년)

3.5.3 무결성 검증

정기 검증 스케줄:

일일:
  - 새로 추가된 파일 체크섬 검증
  - 백업 작업 성공 확인

월간:
  - Tier 1 전체 체크섬 검증
  - 손상된 파일 복구

분기:
  - Tier 2 샘플링 검증 (10%)
  - 메타데이터 일관성 확인

연간:
  - 전체 아카이브 검증
  - 마이그레이션 평가
  - 재해 복구 훈련

체크섬 알고리즘:
  - SHA-256 (권장)
  - MD5 (레거시 지원)
  - CRC32 (빠른 검증)

  $ wia-dps-verify --file image.wdps
