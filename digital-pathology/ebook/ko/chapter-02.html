[MED-008] WIA 디지털 병리학 표준 전자책 | 제2장 (총 8장)

🔬 제2장: 전체 슬라이드 이미징 (WSI)

Whole Slide Imaging

WSI는 디지털 병리학의 핵심 기술입니다. 현미경 슬라이드 전체를 고해상도 디지털 이미지로 변환하여, 병리학자가 컴퓨터 화면에서 마치 현미경을 보듯 진단할 수 있게 합니다.

2.1 WSI 기술 개요

2.1.1 WSI란?

전체 슬라이드 이미징 (Whole Slide Imaging, WSI)은 조직 병리학 슬라이드 전체를 고해상도로 디지털화하는 기술입니다.

기본 개념:

  물리적 슬라이드:
    - 크기: 75mm × 25mm (표준 슬라이드)
    - 조직 영역: 약 15mm × 15mm
    - 두께: 3-5μm

  디지털 이미지:
    - 해상도: 40배율 (0.25μm/픽셀)
    - 파일 크기: 1-5GB
    - 픽셀 수: 약 100,000 × 100,000 (10 기가픽셀)
    - 타일 구조: 256×256 픽셀 타일로 분할

2.1.2 WSI의 작동 원리

스캐닝 프로세스:

1. 슬라이드 로딩
   └─> 자동 또는 수동으로 슬라이드 트레이에 배치
   └─> 바코드 스캔으로 식별
   └─> 최대 400개 슬라이드 자동 처리 (일부 모델)

2. 조직 감지 (Tissue Detection)
   └─> 저배율 프리스캔 (예: 2.5배율)
   └─> 이미지 분석으로 조직 영역 식별
   └─> 스캔할 영역 결정 (ROI - Region of Interest)
   └─> 배경 제외로 파일 크기 최적화

3. 자동 초점 (Autofocus)
   └─> 다중 z-레벨에서 샘플 이미지 캡처
   └─> 초점 점수 계산 (대비 기반)
   └─> 최적 초점 평면 결정
   └─> 각 타일마다 초점 조정 (고급 모델)

4. 이미지 획득
   └─> 조직 영역을 작은 타일로 분할
   └─> 각 타일을 고해상도로 촬영
   └─> Line scanning 또는 TDI (Time Delay Integration) 사용
   └─> 색상 보정 적용

5. 이미지 조합 (Stitching)
   └─> 개별 타일을 하나의 이미지로 결합
   └─> 타일 간 경계 조정 (blending)
   └─> 왜곡 보정
   └─> 색상 일관성 유지

6. 압축 및 저장
   └─> 다중 해상도 피라미드 생성
   └─> JPEG, JPEG2000 등으로 압축
   └─> 메타데이터 첨부
   └─> 표준 형식으로 저장

2.2 스캐너 유형

2.2.1 스캐닝 방법에 따른 분류

1. 타일 스캐닝 (Tile-Based Scanning)

작동 방식:
  1. 슬라이드를 격자 형태로 분할
  2. 각 타일을 순차적으로 촬영
  3. 스테이지 이동 → 촬영 → 이동 → 촬영 반복
  4. 모든 타일 조합하여 전체 이미지 생성

장점:
  ✅ 높은 해상도
  ✅ 우수한 이미지 품질
  ✅ 유연한 스캔 영역 선택

단점:
  ❌ 상대적으로 느린 속도
  ❌ 스티칭 아티팩트 가능성
  ❌ 기계적 마모

대표 제품:
  - Aperio AT2 (Leica)
  - NanoZoomer (Hamamatsu)
  - Pannoramic (3DHistech)

속도: 슬라이드당 60-120초 (40배율)

2. 라인 스캐닝 (Line Scanning)

작동 방식:
  1. 선형 카메라 사용 (1차원 센서)
  2. 슬라이드를 연속적으로 이동
  3. 이동하면서 라인 단위로 촬영
  4. 라인들을 조합하여 이미지 생성

장점:
  ✅ 빠른 스캔 속도
  ✅ 스티칭 아티팩트 없음
  ✅ 연속적인 이미지

단점:
  ❌ 복잡한 광학 시스템
  ❌ 높은 비용
  ❌ 진동에 민감

대표 제품:
  - Aperio GT450 (Leica)
  - IntelliSite Pathology (Philips)

속도: 슬라이드당 30-60초 (40배율)

3. TDI (Time Delay Integration) 스캐닝

작동 방식:
  1. 특수 TDI 카메라 사용
  2. 센서와 슬라이드 동기화 이동
  3. 여러 센서 라인의 신호 누적
  4. 높은 감도와 속도 달성

장점:
  ✅ 매우 빠른 속도
  ✅ 높은 감도
  ✅ 낮은 노이즈

단점:
  ❌ 매우 높은 비용
  ❌ 복잡한 캘리브레이션

대표 제품:
  - Aperio ScanScope (일부 모델)

속도: 슬라이드당 30초 이하 (40배율)

2.3 이미지 해상도 및 품질 기준

2.3.1 해상도 표준

배율 μm/픽셀 용도 파일 크기 (15×15mm)

2.5× 4.0 조직 감지, 프리스캔 ~100MB
5× 2.0 저배율 검토 ~300MB

WIA-MED-008 권장 사항:

임상 진단:
  - 주 스캔: 40× (0.25μm/픽셀)
  - 피라미드 레벨: 1.25×, 2.5×, 5×, 10×, 20×, 40×
  - 최소 요구: 20× (0.5μm/픽셀)

연구용:

  - 필요시 100× 지원

2.3.2 품질 지표

1. 초점 품질 (Focus Quality)

측정 방법:
  - Brenner 그래디언트
  - Tenengrad
  - 라플라시안 분산

기준:
  ✅ 우수: FQ > 90%
  ✅ 양호: FQ 80-90%
  ⚠️  경고: FQ 70-80%
  ❌ 불량: FQ < 70%

불량 원인:
  - 슬라이드 두께 불균일
  - 조직 주름
  - 기포
  - 스캐너 캘리브레이션 문제

해결책:
  1. 다중 z-레벨 스캔 (z-stack)
  2. 적응형 초점 조정
  3. 재스캔

2. 색상 정확도 (Color Accuracy)

표준 색상 타겟:
  - Macbeth ColorChecker
  - IT8 차트
  - 표준 H&E 슬라이드

측정 지표:
  - ΔE (Delta E): 색상 차이
    • ΔE < 2: 거의 인지 불가
    • ΔE 2-10: 인지 가능
    • ΔE > 10: 명확히 다름

  WIA-MED-008 기준:
    ✅ ΔE < 5 (H&E 염색)
    ✅ ΔE < 3 (IHC 염색)

캘리브레이션:
  - 주간: 표준 슬라이드 스캔
  - 월간: 전체 시스템 캘리브레이션
  - 연간: 전문가 서비스

3. 해상도 및 선명도

해상도 테스트:
  - USAF 1951 해상도 차트
  - ISO 21550 슬라이드

측정:
  - MTF (Modulation Transfer Function)
  - 공간 주파수 응답

기준:
  ✅ MTF50 > 200 lp/mm (40배율)
  ✅ 대비비 > 1000:1

4. 신호 대 잡음비 (SNR)

계산:
  SNR = 평균 신호 / 표준편차(노이즈)

기준:
  ✅ SNR > 40dB (우수)
  ✅ SNR 30-40dB (양호)
  ⚠️  SNR 20-30dB (수용 가능)
  ❌ SNR < 20dB (불량)

개선 방법:
  - 조명 세기 최적화
  - 카메라 게인 조정
  - TDI 스캐닝 사용
  - 노이즈 감소 알고리즘

2.4 스캐닝 워크플로우

2.4.1 표준 워크플로우

단계별 프로세스:

준비 단계 (10분):
  1. 스캐너 예열 (5분)
     └─> 광원 안정화
     └─> 온도 평형

  2. 일일 품질 관리 (5분)
     └─> 표준 슬라이드 스캔
     └─> 품질 지표 확인
     └─> 이상 시 조치

슬라이드 준비 (5분/배치):
  3. 슬라이드 점검
     └─> 라벨 확인
     └─> 바코드 부착
     └─> 청결 상태 확인

  4. 트레이 로딩
     └─> 순서대로 배치
     └─> 방향 정렬
     └─> 최대 용량 확인

스캐닝 (60-120초/슬라이드):
  5. 자동 스캔 시작
     └─> 바코드 인식
     └─> 조직 감지
     └─> 초점 조정
     └─> 이미지 캡처
     └─> 품질 검사

  6. 실시간 모니터링
     └─> 진행 상황 확인
     └─> 에러 처리
     └─> 재스캔 큐 관리

후처리 (30초/슬라이드):
  7. 이미지 처리
     └─> 스티칭
     └─> 압축
     └─> 피라미드 생성
     └─> 썸네일 생성

  8. 품질 검증
     └─> 자동 QC
     └─> 기준 통과 확인
     └─> 불량 플래그

저장 및 배포 (10초/슬라이드):
  9. 저장
     └─> 로컬 저장
     └─> PACS 전송
     └─> 백업

  10. 알림
      └─> LIS 업데이트
      └─> 병리학자 알림
      └─> 워크리스트 갱신

전체 시간 (100 슬라이드 기준):
  - 준비: 15분
  - 스캐닝: 100-200분 (1.7-3.3시간)
  - 후처리: 50분
  - 저장: 17분
  총: 약 3-5시간 (자동 처리)

2.4.2 긴급 케이스 처리

빠른 경로 (Fast-Track):

  1. 우선순위 큐
     └─> 긴급 슬라이드 즉시 처리
     └─> 일반 스캔 일시 중단 가능

  2. 신속 스캔 모드
     └─> 20배율 사용 (40배율 대신)
     └─> 스캔 시간: 30초
     └─> 진단 가능한 품질 유지

  3. 즉시 알림
     └─> 스캔 완료 즉시 통보
     └─> 모바일 푸시 알림
     └─> 긴급 플래그 표시

  목표 시간:
    슬라이드 도착 → 디지털 이미지 가용
    < 5분 (긴급 케이스)

2.4.3 배치 처리 최적화

효율성 팁:

  1. 슬라이드 그룹화
     └─> 유사 조직 타입
     └─> 동일 염색
     └─> 최적화된 스캔 설정

  2. 야간 배치
     └─> 대량 슬라이드 야간 처리
     └─> 무인 작동
     └─> 아침에 결과 준비

  3. 부하 분산
     └─> 여러 스캐너 활용
     └─> 우선순위 기반 할당
     └─> 스캐너 상태 모니터링

  4. 예측 스케줄링
     └─> 과거 데이터 분석
     └─> 피크 시간 예측
     └─> 리소스 사전 할당

  실제 사례:
    대형 병원 (일일 500 슬라이드)
    - 스캐너 3대 운영
    - 야간 배치: 300 슬라이드
    - 일간 긴급: 200 슬라이드
    - 처리율: 99.5%

2.5 WSI 품질 관리

2.5.1 일일 QC 프로토콜

매일 아침 루틴:

1. 표준 슬라이드 스캔 (5분)
   └─> H&E 참조 슬라이드
   └─> IHC 참조 슬라이드 (필요시)

2. 품질 지표 확인
   초점 품질:
     [OK] FQ = 94% (기준: >90%)
   색상 정확도:
     [OK] ΔE = 3.2 (기준: <5)
   해상도:
     [OK] MTF50 = 215 lp/mm (기준: >200)
   SNR:
     [OK] 42dB (기준: >40dB)

3. 기준 미달 시 조치
   경고 (Warning):
     → 재스캔 후 재확인
     → 문제 지속 시 캘리브레이션

   불량 (Failure):
     → 즉시 스캐닝 중단
     → 기술 지원 요청
     → 대체 스캐너 사용

4. 기록 유지
   └─> QC 결과 데이터베이스 저장
   └─> 추세 분석
   └─> 월간 보고서

2.6 WIA-MED-008 WSI 사양

2.6.1 표준 이미지 형식

필수 지원 형식:
  1. WIA-DPS (Digital Pathology Standard)
     - 기본 컨테이너: TIFF
     - 압축: JPEG, JPEG2000, WebP
     - 메타데이터: JSON embedded
     - 피라미드: 다중 레벨

  2. DICOM
     - 표준: DICOM Supplement 145
     - 전송: DICOM DIMSE
     - 저장: PACS 호환

권장 지원:
  3. OME-TIFF
     - 오픈 마이크로스코피 환경
     - XML 메타데이터
     - 연구용 표준

레거시 지원:
  4. SVS, NDPI, MRXS 등
     - 기존 형식 읽기
     - WIA-DPS로 변환

2.7 장 요약

주요 학습 내용:

  1. WSI 원리: 조직 감지 → 초점 조정 → 타일 캡처 → 조합 → 압축
  2. 스캐너 유형: 타일, 라인, TDI 스캐닝 방법
  3. 품질 기준: 초점, 색상, 해상도, SNR 지표
  4. 워크플로우: 준비 → 스캔 → 후처리 → 저장
  5. QC 프로토콜: 일일, 주간, 월간 점검
  6. WIA 표준: 형식, 메타데이터, 품질 요구사항
