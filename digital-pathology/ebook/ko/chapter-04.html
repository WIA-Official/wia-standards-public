🤖 제4장: AI 기반 병리학 분석

AI의 잠재력

인공지능은 병리학 진단의 정확성, 효율성, 재현성을 혁명적으로 향상시킵니다. 병리학자를 대체하는 것이 아니라, 더 나은 환자 치료를 위한 강력한 도구를 제공합니다.

4.1 딥러닝 기초

4.1.1 신경망 아키텍처

컨볼루션 신경망 (Convolutional Neural Networks, CNN)

기본 구조:

입력 이미지 (256×256×3)
    ↓
[Convolutional Layer 1]
  - 필터: 32개, 크기: 3×3
  - 활성화: ReLU
  - 출력: 256×256×32
    ↓
[Pooling Layer 1]
  - Max Pooling: 2×2
  - 출력: 128×128×32
    ↓
[Convolutional Layer 2]
  - 필터: 64개, 크기: 3×3
  - 활성화: ReLU
  - 출력: 128×128×64
    ↓
[Pooling Layer 2]
  - Max Pooling: 2×2
  - 출력: 64×64×64
    ↓
[Convolutional Layer 3]
  - 필터: 128개, 크기: 3×3
  - 활성화: ReLU
  - 출력: 64×64×128
    ↓
[Global Average Pooling]
  - 출력: 128
    ↓
[Fully Connected Layer]
  - 뉴런: 256
  - 활성화: ReLU
    ↓
[Output Layer]
  - 뉴런: 2 (양성/음성)
  - 활성화: Softmax
  - 출력: 확률 [0.85, 0.15]

결과: 양성 (85% 확신도)

4.1.2 주요 모델 아키텍처

모델 연도 특징 병리학 응용 성능
ResNet 2015 잔차 연결, 깊은 네트워크 조직 분류, 전이 학습 기반 매우 우수
Inception 2014 다중 스케일 특징 다양한 세포 크기 감지 우수
DenseNet 2017 밀집 연결, 파라미터 효율 제한된 데이터셋 우수
U-Net 2015 인코더-디코더, 픽셀 단위 조직 세그멘테이션 뛰어남
Vision Transformer 2020 Self-attention, 글로벌 문맥 전체 슬라이드 분석 매우 뛰어남

4.1.3 전이 학습 (Transfer Learning)

전이 학습 워크플로우:

1. 사전 훈련 모델 선택
   └─> ImageNet 사전 훈련 ResNet50
   └─> 1.4M 자연 이미지로 학습됨

2. 특징 추출기로 사용
   └─> 마지막 레이어 제거
   └─> 저레벨 특징 재사용 (엣지, 텍스처)

3. 병리학 데이터로 미세 조정
   └─> 새로운 분류 레이어 추가
   └─> 소규모 데이터셋으로 학습 (1,000개 이미지)
   └─> 학습 시간: 2-4시간 (vs 수일)

4. 성능 향상
   └─> 정확도: 92% (전이 학습)
   └─> vs 75% (처음부터 학습)

장점:
  ✅ 적은 데이터로 학습 가능
  ✅ 빠른 수렴
  ✅ 더 나은 일반화
  ✅ 계산 비용 절감

사용 사례:
  - 희귀 암 유형
  - 제한된 라벨 데이터
  - 빠른 프로토타이핑

4.2 이미지 분할 및 분류

4.2.1 시맨틱 세그멘테이션

픽셀 단위 분류

U-Net 아키텍처:

입력: 조직 이미지 (512×512×3)

[Contracting Path - 인코더]
  Conv 3×3, ReLU (512×512×64)
  Conv 3×3, ReLU (512×512×64)
  Max Pool 2×2 (256×256×64)
    ↓
  Conv 3×3, ReLU (256×256×128)
  Conv 3×3, ReLU (256×256×128)
  Max Pool 2×2 (128×128×128)
    ↓
  [더 깊은 레이어...]
    ↓
[Bottleneck]
  Conv 3×3, ReLU (32×32×512)

[Expanding Path - 디코더]
  Up-conv 2×2 (64×64×256)
  Concatenate with encoder (skip connection)
  Conv 3×3, ReLU (64×64×256)
    ↓
  Up-conv 2×2 (128×128×128)
  Concatenate with encoder
  Conv 3×3, ReLU (128×128×128)
    ↓
  [더 얕은 레이어...]
    ↓
출력: 세그멘테이션 맵 (512×512×N_classes)

클래스:
  0: 배경
  1: 정상 조직
  2: 종양
  3: 괴사
  4: 염증
  5: 간질

결과:
  각 픽셀에 클래스 레이블 할당

4.2.2 인스턴스 세그멘테이션

Mask R-CNN:

목표: 개별 세포/핵 감지 및 분할

1. 영역 제안 (Region Proposal Network)
   └─> 객체가 있을 가능성 영역 제안

2. ROI Align
   └─> 각 영역에서 특징 추출

3. 분류 + 바운딩 박스
   └─> 클래스 예측 (세포 유형)
   └─> 위치 정제

4. 마스크 예측
   └─> 각 인스턴스의 픽셀 레벨 마스크

출력 예시:
  감지된 객체: 1,253개 핵

  핵 #1:
    - 클래스: 종양 세포
    - 신뢰도: 0.95
    - 바운딩 박스: (x:120, y:340, w:45, h:48)
    - 마스크: [픽셀별 마스크]

  핵 #2:
    - 클래스: 림프구
    - 신뢰도: 0.88
    - 바운딩 박스: (x:200, y:380, w:28, h:30)
    - 마스크: [픽셀별 마스크]

  ...

응용:
  - 세포 계수
  - 형태학적 분석 (크기, 모양)
  - 공간적 분포 분석

4.2.3 패치 기반 분류

전체 슬라이드 이미지 분류:

문제: WSI 크기 (100,000×100,000 픽셀)는 GPU 메모리 초과

해결책: 패치 기반 접근

1. 슬라이드 패치 분할
   └─> 패치 크기: 256×256 픽셀
   └─> 겹침: 50% (128 픽셀)
   └─> 총 패치: ~40,000개

2. 각 패치 분류
   └─> CNN 모델로 추론
   └─> 출력: 종양 확률

   패치 #1: 확률 = 0.02 (정상)
   패치 #2: 확률 = 0.15 (정상)
   패치 #3: 확률 = 0.89 (종양)
   패치 #4: 확률 = 0.92 (종양)
   ...

3. 히트맵 생성
   └─> 각 패치의 확률을 슬라이드에 매핑
   └─> 보간법으로 부드러운 맵 생성

4. 슬라이드 레벨 예측
   └─> 집계 방법:
       a) 최대 확률
       b) 상위 10% 평균
       c) 가중 평균
       d) Attention 기반

5. 최종 진단
   └─> 종양 존재: 예 (확률 95%)
   └─> 종양 면적: 12% (슬라이드 대비)
   └─> 고위험 영역: 3개

최적화:
  - GPU 배치 처리 (배치 크기: 64)
  - 멀티프로세싱
  - 조직 영역만 처리 (배경 제외)
  - 처리 시간: 슬라이드당 2-5분

4.3 병변 검출 알고리즘

4.3.1 종양 검출

다단계 검출 파이프라인:

1단계: 조직 검출 및 세그멘테이션
  입력: 저배율 이미지 (2.5배율)
  모델: U-Net
  출력: 조직 마스크
  목적: 분석 영역 제한

2단계: 종양 영역 검출
  입력: 고배율 패치 (20배율 또는 40배율)
  모델: ResNet50 분류기
  출력: 종양 확률 맵
  임계값: 0.5 (양성 판정)

3단계: 종양 경계 정제
  입력: 종양 후보 영역
  모델: U-Net 세그멘테이션
  출력: 정확한 종양 윤곽선

4단계: 거짓 양성 제거
  입력: 검출된 종양 영역
  모델: DenseNet 분류기
  특징: 형태학적 특징
  출력: 최종 종양 영역

5단계: 후처리
  - 작은 노이즈 제거 (면적 < 0.1mm²)
  - 형태학적 연산 (opening, closing)
  - 영역 병합

결과 예시:
  총 검출: 15개 영역
  거짓 양성 제거 후: 3개 영역

  종양 영역 #1:
    - 위치: (x:5200, y:8100)
    - 크기: 2.3 mm²
    - 신뢰도: 0.95
    - 유형: 침윤성 관암종

  종양 영역 #2:
    - 위치: (x:12000, y:3500)
    - 크기: 1.8 mm²
    - 신뢰도: 0.88
    - 유형: 침윤성 관암종

  종양 영역 #3:
    - 위치: (x:8500, y:15200)
    - 크기: 0.9 mm²
    - 신뢰도: 0.82
    - 유형: 관상피내암 (DCIS)

성능:
  - 감도 (Sensitivity): 94%
  - 특이도 (Specificity): 89%
  - F1 점수: 0.91

4.3.2 핵 검출 및 분할

핵 검출의 중요성:
  - 세포 계수 (증식 지수)
  - 핵 형태 분석 (이형성)
  - 유사분열 지수
  - 공간적 배열 패턴

접근 방법:

방법 1: 전통적 이미지 처리
  1. 색상 분리 (Hematoxylin 채널)
  2. 이진화 (Otsu 임계값)
  3. Watershed 세그멘테이션
  4. 형태학적 필터링

  장점: 빠름, 해석 가능
  단점: 정확도 한계 (~80%)

방법 2: 딥러닝
  모델: Mask R-CNN 또는 StarDist

  StarDist 예시:
    입력: 조직 패치 (512×512)
    출력: 핵 중심점 + 방사형 거리

    각 핵:
      중심: (x, y)
      거리: [r₁, r₂, ..., r₃₂] (32방향)

    장점: 겹친 핵 분리 가능
    정확도: ~92%

방법 3: 앙상블
  - 여러 모델 결합
  - 투표 또는 평균
  - 정확도: ~95%

핵 특징 추출:
  형태학적:
    - 면적, 둘레
    - 원형도 (4π×면적/둘레²)
    - 편심률
    - 장축/단축 비율

  텍스처:
    - Haralick 특징
    - LBP (Local Binary Patterns)

  색상:
    - 평균 강도
    - 표준편차
    - 색조, 채도

응용:
  Ki-67 지수 계산:
    양성 핵 / 전체 핵 × 100%
    예: 200/1000 × 100% = 20% (증식 활발)

  유사분열 계수:
    10 HPF (High Power Field)당 유사분열 핵 수
    예: 15개/10HPF → Grade II

4.3.3 림프절 전이 검출

림프절 슬라이드 분석:

도전과제:
  - 매우 작은 미세전이 (0.2-2mm)
  - 정상 림프구와 유사한 외형
  - 슬라이드당 많은 림프절 (5-20개)

CAMELYON 챌린지 접근법:

1. 다중 스케일 분석
   저배율 (2.5배율):
     └─> 림프절 영역 검출
   중배율 (10배율):
     └─> 의심 영역 찾기
   고배율 (40배율):
     └─> 미세전이 확인

2. Attention 기반 MIL (Multiple Instance Learning)
   가정: 슬라이드에 하나라도 양성 패치 존재 → 양성

   Attention 메커니즘:
     각 패치에 가중치 할당
     의심 영역에 더 높은 가중치

   최종 예측:
     가중 평균으로 슬라이드 레벨 분류

3. 결과
   성능 (CAMELYON16 데이터셋):
     - AUC: 0.994
     - 감도: 92.5% (병리학자 수준)
     - 거짓 양성: 8개/400 슬라이드

   임상 영향:
     - 미세전이 놓치는 비율 감소
     - 병리학자 워크로드 50% 감소
     - 진단 시간 단축

4.4 모델 학습 및 검증

4.4.1 데이터셋 구축

데이터 수집:

요구사항:
  - 다양성: 여러 기관, 스캐너, 염색
  - 규모: 최소 1,000-10,000 슬라이드
  - 라벨 품질: 전문가 검증

예시: 유방암 분류 데이터셋

  클래스별 분포:
    정상: 3,000 슬라이드
    양성 종양: 1,500 슬라이드
    악성 종양:
      - 침윤성 관암종: 4,000 슬라이드
      - 침윤성 소엽암종: 1,200 슬라이드
      - 관상피내암: 800 슬라이드
      - 기타: 500 슬라이드

  총: 11,000 슬라이드

주석 프로세스:
  1. 초기 주석 (병리학자 1명)
  2. 검토 (병리학자 2명)
  3. 불일치 해결 (3명 합의)
  4. 품질 검증 (무작위 10% 재검토)

데이터 분할:
  훈련: 70% (7,700 슬라이드)
  검증: 15% (1,650 슬라이드)
  테스트: 15% (1,650 슬라이드)

중요: 환자 레벨 분할
  └─> 같은 환자의 슬라이드가 여러 세트에 들어가지 않도록

4.4.2 데이터 증강

증강 기법:

1. 기하학적 변환
   - 회전: -180° ~ +180°
   - 반전: 수평, 수직
   - 확대/축소: 0.8x ~ 1.2x
   - 전단 변형: ±10°

2. 색상 변환
   - H&E 색상 정규화
   - 밝기 조정: ±20%
   - 대비 조정: ±20%
   - 색조 이동: ±10°

3. 노이즈 추가
   - 가우시안 노이즈: σ = 5
   - Salt-and-pepper: 밀도 = 0.01

4. 블러링
   - 가우시안 블러: σ = 0.5-1.0
   - 모션 블러

5. 엘라스틱 변형
   - 조직 변형 시뮬레이션
   - α = 50, σ = 5

적용 예시:
  원본 패치 1개
    ↓
  증강 후 8개
    1. 원본
    2. 90° 회전
    3. 180° 회전
    4. 수평 반전
    5. 색상 정규화
    6. 밝기 증가
    7. 가우시안 노이즈
    8. 엘라스틱 변형

효과:
  - 데이터셋 크기: 8배 증가
  - 과적합 방지
  - 일반화 성능 향상: 85% → 92%

4.4.3 학습 전략

하이퍼파라미터:

모델: ResNet50
입력 크기: 256×256×3
배치 크기: 32
에포크: 100

옵티마이저: Adam
  - 학습률: 0.001 → 0.0001 (코사인 감쇠)
  - β₁: 0.9
  - β₂: 0.999

손실 함수:
  - 이진 분류: Binary Cross-Entropy
  - 다중 클래스: Categorical Cross-Entropy
  - 세그멘테이션: Dice Loss + Cross-Entropy

정규화:
  - L2 정규화: λ = 0.0001
  - Dropout: 0.5 (FC layer)
  - Batch Normalization

학습 스케줄:

에포크 1-20: 워밍업
  - 학습률: 0 → 0.001 (선형 증가)
  - 목적: 안정적 시작

에포크 21-80: 주 학습
  - 학습률: 0.001 (고정)
  - 검증 손실 모니터링
  - 조기 종료: 10 에포크 개선 없으면 중지

에포크 81-100: 미세 조정
  - 학습률: 0.0001
  - 모든 레이어 학습

4.4.4 성능 평가

평가 지표:

1. 분류 메트릭
   혼동 행렬:
                예측
              양성  음성
     실제 양성 TP   FN
          음성 FP   TN

   정확도 (Accuracy):
     (TP + TN) / (TP + TN + FP + FN)
     = (850 + 740) / (850 + 740 + 40 + 20)
     = 96.4%

   감도 (Sensitivity/Recall):
     TP / (TP + FN)
     = 850 / (850 + 20)
     = 97.7%

   특이도 (Specificity):
     TN / (TN + FP)
     = 740 / (740 + 40)
     = 94.9%

   정밀도 (Precision):
     TP / (TP + FP)
     = 850 / (850 + 40)
     = 95.5%

   F1 점수:
     2 × (Precision × Recall) / (Precision + Recall)
     = 2 × (0.955 × 0.977) / (0.955 + 0.977)
     = 0.966

2. ROC 곡선 및 AUC
   ROC (Receiver Operating Characteristic):
     - x축: 거짓 양성률 (1 - 특이도)
     - y축: 참 양성률 (감도)
     - 다양한 임계값에서 계산

   AUC (Area Under Curve):
     = 0.987 (완벽: 1.0)

   해석:
     98.7% 확률로 무작위 양성 샘플이
     무작위 음성 샘플보다 높은 점수

3. 세그멘테이션 메트릭
   Dice 계수:
     2 × |예측 ∩ 실제| / (|예측| + |실제|)
     = 2 × 8500 / (9000 + 9200)
     = 0.934

   IoU (Intersection over Union):
     |예측 ∩ 실제| / |예측 ∪ 실제|
     = 8500 / 9700
     = 0.876

4. 임상 관련 메트릭
   종양 검출률:
     감지된 종양 / 실제 종양
     = 945 / 970
     = 97.4%

   거짓 양성률:
     거짓 양성 / 슬라이드
     = 45 / 1650
     = 2.7%
