# 제3장: WIA 전자폐기물 관리 표준 개요

## 학습 목표

이 장을 완료하면 다음을 수행할 수 있습니다:

1. 전자폐기물 관리를 위한 WIA 4단계 아키텍처 설명
2. 제품 생애주기 추적 프레임워크 이해
3. 규정 준수 수준 및 인증 요구사항 식별
4. 이해관계자 역할과 책임 설명
5. WIA 표준을 기존 규제 프레임워크에 매핑

## 3.1 표준 아키텍처

### 3.1.1 WIA 4단계 프레임워크

WIA 전자폐기물 관리 표준은 모든 WIA 표준에 적용되는 검증된 4단계 아키텍처를 따릅니다:

```
WIA 전자폐기물 관리 표준 아키텍처:
┌─────────────────────────────────────────────────────────────────────┐
│                                                                     │
│  1단계: 데이터 형식                                                 │
│  ├─ 기기 식별 스키마                                               │
│  ├─ 재료 구성 기록                                                 │
│  ├─ 관리 연속성 데이터 모델                                        │
│  └─ 규정 준수 문서 형식                                            │
│                                                                     │
│  2단계: API 인터페이스                                              │
│  ├─ 기기 등록 API                                                  │
│  ├─ 수거 이벤트 API                                                │
│  ├─ 처리 시설 API                                                  │
│  ├─ 재료 회수 보고 API                                             │
│  └─ 규제 준수 API                                                  │
│                                                                     │
│  3단계: 프로토콜                                                    │
│  ├─ 관리 연속성 프로토콜                                           │
│  ├─ 재료 취급 사양                                                 │
│  ├─ 유해물질 프로토콜                                              │
│  ├─ 품질 인증 프로토콜                                             │
│  └─ 감사 및 검증 프로토콜                                          │
│                                                                     │
│  4단계: 시스템 통합                                                 │
│  ├─ 생산자 책임 시스템                                             │
│  ├─ 수거 네트워크 통합                                             │
│  ├─ 재활용 시설 시스템                                             │
│  ├─ 규제 보고 시스템                                               │
│  └─ 순환경제 플랫폼                                                │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 3.2.2 생애주기 이벤트 모델

```typescript
// 핵심 생애주기 이벤트 구조
interface LifecycleEvent {
  // 이벤트 식별
  eventId: string;              // UUID
  eventType: EventType;
  timestamp: string;            // ISO 8601
  timezone: string;

  // 기기 참조
  deviceId: string;             // WIA 기기 ID
  deviceCategory: DeviceCategory;

  // 행위자 정보
  actor: {
    actorId: string;            // WIA 등록 엔티티
    actorType: "producer" | "retailer" | "collector" | "processor" | "consumer";
    facilityId?: string;        // 시설 기반 운영의 경우
    location: GeoLocation;
  };

  // 이벤트별 데이터
  eventData: EventSpecificData;

  // 관리 연속성
  previousEvent?: string;       // 이전 이벤트 링크
  custodyTransfer?: {
    fromActor: string;
    toActor: string;
    transferDocument?: string;
  };

  // 검증
  verification: {
    method: "self_declared" | "witnessed" | "automated" | "audited";
    verifier?: string;
    evidence?: string[];        // 사진, 문서, 센서 데이터
  };

  // 디지털 서명
  signature: {
    algorithm: "ES256" | "RS256";
    value: string;
    publicKey: string;
  };
}

// 이벤트 유형 열거
type EventType =
  | "production"           // 생산
  | "distribution"         // 유통
  | "sale"                 // 판매
  | "ownership_transfer"   // 소유권 이전
  | "repair"               // 수리
  | "collection"           // 수거
  | "assessment"           // 평가
  | "refurbishment"        // 리퍼비시
  | "facility_transfer"    // 시설 이전
  | "disassembly"          // 분해
  | "material_recovery"    // 재료 회수
  | "hazardous_treatment"  // 유해물질 처리
  | "final_disposal"       // 최종 처분
  | "lifecycle_closure";   // 생애주기 종결
```

## 3.6 핵심 요약

| 구성요소 | 목적 | 핵심 기능 |
|---------|-----|---------|
| 4단계 아키텍처 | 포괄적 커버리지 | 데이터, API, 프로토콜, 통합 |
| 생애주기 프레임워크 | 종단간 추적 | 6단계, 이벤트 기반 모델 |

- **계층적 준수**: 모든 규모의 운영자에게 접근 가능
- **API 우선 설계**: 기본적으로 상호운용 가능
- **감사 추적**: 완전한 관리 연속성
