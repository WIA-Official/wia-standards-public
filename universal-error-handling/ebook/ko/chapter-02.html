핵심 요점

  • JSON 오류 응답 형식은 기계가 파싱 가능하며 동시에 사람이 읽기 쉽다.
  • 타임스탬프, 요청 ID, 사용자 ID 등의 메타데이터는 오류 분석을 풍부하게 한다.
  • 재시도 가능 오류에 대한 명확한 표시는 클라이언트의 복구 전략을 최적화한다.
  • 오류 문서 링크 제공은 개발자가 신속하게 해결 방법을 찾도록 돕는다.
  • 스택 트레이스 필터링은 민감한 정보 노출을 방지하면서 디버깅 정보를 제공한다.
