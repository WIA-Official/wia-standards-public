챕터 7: 거버넌스와 변경 관리

이 장의 목적

이 장은 WIA-LEG-008 데이터 이동권 표준의 챕터 7: 거버넌스와 변경 관리 영역을 다룹니다. 표준은 GDPR Article 20 (Right to Data Portability), CCPA §1798.100, LGPD Article 18, K-PIPA Article 35조의2의 데이터 이동권 의무를 상호운용 가능한 데이터 교환 인터페이스로 구현합니다.

참조 표준

  • GDPR (EU 2016/679) Article 20 — Right to data portability
  • CCPA (California Consumer Privacy Act) §1798.100
  • LGPD (Brazil Lei Geral de Proteção de Dados) Article 18
  • K-PIPA (Personal Information Protection Act) Article 35조의2
  • OpenAPI Specification 3.1 — REST API 정의
  • JSON Schema 2020-12 — 데이터 형식 검증
  • IETF RFC 9457 — Problem Details for HTTP APIs
  • IETF RFC 9700 — OAuth 2.1
  • ISO/IEC 27001:2022 — 정보보안 경영시스템
  • ISO/IEC 27018:2019 — 공용 클라우드의 PII 보호
  • ISO/IEC 27701:2019 — 프라이버시 정보 관리
  • Sigstore (DSSE 봉투, Rekor 투명성 로그) — 서명 앵커링

실무 절차

데이터 주체는 자신의 데이터를 한 컨트롤러에서 다른 컨트롤러로 이전 요청할 수 있습니다. 이 표준은 그 요청과 응답의 데이터 형식·인증 절차·서명 방식을 정의합니다.

요청 절차

  1. 주체 인증 — OAuth 2.1 (RFC 9700) PKCE 흐름으로 주체의 신원 확인. mTLS 바인딩 권장.
  2. 요청 매니페스트 발급 — 이전 범위(scope), 대상 컨트롤러, 포맷(JSON/XML/CSV), 발급 시각이 매니페스트에 명시되며 Sigstore DSSE 봉투로 서명됩니다.
  3. 원천 컨트롤러 검증 — 매니페스트 서명·범위·만료 시각 검증. 부적합 시 RFC 9457 Problem Details로 거부.
  4. 데이터 패키지 발행 — 표준 데이터 형식으로 패키지 생성. SHA-256 해시·Sigstore 서명·검증 가능한 SBOM 첨부.
  5. 대상 컨트롤러 수령 — 서명 검증 후 수령. 수령 후에는 별도 매니페스트로 수령 사실을 anchoring.

이행 토폴로지

구현체는 다음 4가지 토폴로지 중 하나를 매니페스트에 선언합니다.

  • 단일 테넌트 엣지: 컨트롤러 자체 운영. 직접 서명·발행. 초기 도입에 적합.
  • 멀티 테넌트 게이트웨이: 1개 인프라가 다중 컨트롤러 서비스. 헤더 기반 격리·OAuth 2.1 신원 공급자.
  • 연합 메시: 다수 컨트롤러가 디렉토리 서비스 통해 상호 인식. 디렉토리는 집계 인덱스를 별도 키로 서명.
  • 에어갭 배치: 라이브 연결 없음. 서명된 패키지 out-of-band 전송. 규제 환경 배포에 사용.

거버넌스

정정·확장은 시맨틱 버저닝(SemVer 2.0.0)을 따릅니다. 패치는 편집상 정정만, 마이너는 새 의무 요건의 추가, 메이저는 호환 단절을 의미합니다. 메이저 변경은 최소 90일 병행 호환 기간을 거치며 그 기간 동안 신·구 두 버전 모두 WIA 등록부에 Stable로 표기됩니다.

모든 변경 제안과 정정 요청은 GitHub Issues 트래커에 data-portability 레이블로 접수됩니다. WIA 표준 워킹그룹은 매 마이너 릴리스 사이클의 시작 시점에 열린 이슈를 검토하고, 결정 로그를 릴리스 노트와 함께 공개합니다.

마이그레이션 노트

이전 메이저 버전에서 새 메이저 버전으로의 마이그레이션은 최소 90일 병행 호환 기간을 거칩니다. 이전 매니페스트 키는 공개된 키 이력에 보존되어 사후 감사 가능성을 유지합니다. 이전 매니페스트는 supersede 마킹과 함께 영구 보존됩니다 — 삭제하지 않습니다 (감사 요구). 마이그레이션 노트는 GitHub Releases와 WIA 등록부 양쪽에 동시 공시되며, 결정 로그는 워킹그룹의 검토 결과를 함께 담습니다. 데이터 주체의 이전 권리는 마이그레이션 기간에도 끊김 없이 보장되며, 신·구 버전 모두 동일한 데이터 패키지 포맷을 발급할 수 있어야 합니다.
