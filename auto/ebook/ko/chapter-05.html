제5장: 안전과 검증

안전은 자율주행차 배포의 가장 중요한 관심사입니다. 이 장에서는 기능 안전, 검증 방법론, 자율주행차 안전을 입증하는 데 필요한 포괄적인 테스트에 대한 WIA-AUTO 요구사항을 자세히 설명합니다.

5.1 기능 안전 프레임워크

WIA-AUTO는 도로 차량의 기능 안전을 위한 ISO 26262를 기반으로 하며, 자율주행 시스템에 대한 개념을 확장합니다.

모션 제어ASIL D차량 움직임 직접 제어

5.2 의도된 기능의 안전(SOTIF)

SOTIF(ISO 21448)는 시스템이 설계대로 작동하더라도 기능적 불충분으로 인해 발생하는 안전 위험을 다룹니다.

SOTIF 범위

  • 센서 한계 (범위, 해상도, 환경 조건)
  • 알고리즘 한계 (엣지 케이스, 분포 외 입력)
  • 설계 중 예상하지 못한 트리거 조건
  • 예측 가능한 범위 내 사용자 오용
  • 예측 불가능한 도로 사용자 행동과의 상호작용

5.3 검증 방법론

자율주행차 안전을 입증하려면 시뮬레이션, 트랙 테스트 및 실제 주행을 포함한 여러 방법론에 걸친 광범위한 검증이 필요합니다.

시뮬레이션 요구사항
===================
환경 충실도:
- 물리학: 실제 세계의 5% 이내 차량 동역학
- 센서: 현실적인 노이즈, 아티팩트, 실패 모드
- 렌더링: 카메라 테스트를 위한 포토리얼리스틱

시나리오 커버리지:
- 기능 시나리오: 1,000+ 시나리오 카테고리
- 엣지 케이스: 높은 결과를 가진 희귀 이벤트
- 적대적 시나리오: 한계를 테스트하도록 설계
- 몬테카를로: 파라미터의 통계적 변동

검증 메트릭:
- 10,000+ 시뮬레이션 시간 동안 충돌 없음
- 엣지 케이스 라이브러리의 95% 안전 처리
- ODD 외 시나리오에서 우아한 저하

5.5 시나리오 기반 테스트

시나리오 계층
==============
기능 시나리오
│   (추상 설명)
│   예: "차량이 더 느린 교통에 접근"
│
├── 논리 시나리오
│   │   (파라미터 범위)
│   │   예: "자차 속도 50-80 km/h, 선행 차량 30-50 km/h"
│   │
│   └── 구체 시나리오
│           (특정 파라미터)
│           예: "자차 72 km/h, 선행 45 km/h, 간격 50m"
│
└── 테스트 케이스
        (시나리오 + 합격/불합격 기준)
        예: "충돌 불가피 시 AEB 활성화"

5.6 런타임 안전 모니터링

안전 모니터 기능

  • 인지 타당성: 일관성을 위해 센서 데이터 교차 확인
  • 계획 검증: 궤적이 안전 제약을 준수하는지 확인
  • ODD 모니터링: 조건이 운영 영역을 초과하는지 감지
  • 시스템 상태: 계산 리소스, 센서 상태 모니터링
  • 행동 모니터링: 비정상적인 시스템 행동 감지

5.7 지속적 개선

  • 현장 데이터 수집: 운영 데이터의 체계적 수집
  • 사고 조사: 안전 이벤트에 대한 근본 원인 분석
  • OTA 업데이트 프로세스: 검증 요구사항이 있는 보안 업데이트
  • 보고 요구사항: 당국에 대한 의무 보고
