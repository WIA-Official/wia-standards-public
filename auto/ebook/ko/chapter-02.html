제2장: 센서 시스템과 인지

인지 시스템은 자율주행차의 눈과 귀 역할을 하며, 원시 센서 데이터를 실행 가능한 환경 이해로 변환합니다. 이 장에서는 센서 선택, 캘리브레이션, 융합 및 인지 알고리즘 성능에 대한 WIA-AUTO 요구사항을 정의합니다.

2.1 인지 시스템 아키텍처

견고한 인지 시스템은 모든 운영 조건에서 신뢰할 수 있는 환경 이해를 달성하기 위해 조화롭게 작동하는 여러 보완적인 센서가 필요합니다. WIA-AUTO 표준은 안전 등급 자율주행 시스템에 중요한 이중화 및 내결함성을 제공하는 다중 모달 센서 접근 방식을 의무화합니다.

2.2 LiDAR 시스템

LiDAR(Light Detection and Ranging) 센서는 레이저 펄스의 비행 시간 측정을 통해 정밀한 3D 환경 매핑을 제공하며 대부분의 자율주행차 인지 시스템의 중추를 형성합니다.

LiDAR 기술 유형

  • 기계식 회전 LiDAR: 360도 커버리지를 제공하는 회전 레이저 어셈블리가 있는 전통적인 설계
  • 솔리드 스테이트 LiDAR: 움직이는 부품이 없으며 MEMS 미러 또는 광학 위상 배열 활용
  • 플래시 LiDAR: 전체 장면을 동시에 조명하여 단일 펄스로 3D 이미지 캡처
  • FMCW LiDAR: 동시 범위 및 속도 제공하는 주파수 변조 연속파 기술

2.3 레이더 시스템

레이더 센서는 모든 기상 조건에서 신뢰할 수 있는 객체 탐지 및 속도 측정을 제공하여 LiDAR와 카메라를 보완합니다. LiDAR보다 낮은 각도 해상도를 제공하지만, 레이더는 이동 객체 탐지와 악천후 투과에서 탁월합니다.

2.4 카메라 시스템

카메라는 교통 표지판 인식, 신호등 탐지, 차선 표시 식별 및 객체 분류를 포함한 장면 이해에 필수적인 풍부한 시각 정보를 제공합니다.

카메라 유형 및 구성

  • 단안 카메라: 분류 및 탐지를 위한 2D 이미지를 제공하는 단일 카메라
  • 스테레오 카메라: 시차 기반 깊이 추정을 가능하게 하는 쌍 카메라
  • 서라운드 뷰: 360도 시각적 커버리지를 제공하는 다중 광각 카메라
  • 열화상/IR 카메라: 저조도 조건에서 보행자 탐지를 위한 적외선 카메라

2.5 센서 융합 접근법

센서 융합은 개별 센서 한계를 극복하고 견고한 인지를 달성하기 위해 여러 센서의 데이터를 결합합니다. WIA-AUTO는 여러 융합 접근법을 인정합니다:

// 다중 모달 센서 융합 예시
class SensorFusion {
    WorldModel fuse(LiDARData lidar, RadarData radar, CameraData cameras) {
        // 각 모달리티에서 탐지 가져오기
        lidar_objects = lidarDetector.detect(lidar);
        radar_objects = radarDetector.detect(radar);
        camera_objects = cameraDetector.detect(cameras);

        // 모달리티 간 탐지 연관
        associations = hungarianMatching(
            lidar_objects, radar_objects, camera_objects
        );

        // 칼만 필터를 사용하여 연관된 객체 융합
        fused_objects = [];
        for (assoc in associations) {
            fused = kalmanFuse(assoc.lidar, assoc.radar, assoc.camera);
            fused_objects.append(fused);
        }
        return WorldModel(fused_objects);
    }
}

2.7 센서 캘리브레이션 요구사항

정확한 센서 캘리브레이션은 신뢰할 수 있는 인지에 필수적입니다. WIA-AUTO는 내부 캘리브레이션(내부 센서 파라미터)과 외부 캘리브레이션(센서 간 상대적 자세) 모두에 대해 지정된 정확도 요구사항을 의무화합니다.

캘리브레이션 유지관리

자율주행차는 캘리브레이션 품질을 지속적으로 모니터링하고 저하를 감지해야 합니다. 자연 특징을 사용한 온라인 캘리브레이션 검증은 캘리브레이션 드리프트가 허용 가능한 임계값을 초과할 때 경고를 트리거해야 합니다.
