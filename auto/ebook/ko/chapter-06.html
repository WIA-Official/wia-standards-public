제6장: 사이버보안 프레임워크

자율주행차가 점점 더 연결되고 소프트웨어 정의가 됨에 따라, 사이버보안은 차량 안전에 필수적이 됩니다. 이 장에서는 전체 수명주기에 걸쳐 자율주행차를 사이버 위협으로부터 보호하기 위한 WIA-AUTO 요구사항을 수립합니다.

6.1 사이버보안 현황

자율주행차 공격 표면
=====================
외부 인터페이스:
├── V2X 통신 (DSRC/C-V2X)
├── 셀룰러 연결 (4G/5G)
├── Wi-Fi (업데이트, 진단용)
├── 블루투스 (폰 연결)
├── GPS/GNSS 신호
├── 물리적 포트 (OBD-II, USB, 디버그)
└── 센서 입력 (카메라, LiDAR 스푸핑)

내부 시스템:
├── 인포테인먼트 시스템
├── 텔레매틱스 제어 유닛
├── 바디 제어 모듈
├── 자율주행 컴퓨터
├── V2X 온보드 유닛
├── 게이트웨이/방화벽
└── 안전 중요 ECU

6.2 위협 모델링

위협 카테고리공격 예시잠재적 영향
스푸핑GPS 스푸핑, V2X 메시지 위조잘못된 방향, 거짓 장애물
변조펌웨어 수정, CAN 주입시스템 오작동
부인로그 조작, 이벤트 삭제조사 방해
정보 유출위치 추적, 데이터 도난프라이버시 침해
서비스 거부네트워크 플러딩, 리소스 고갈시스템 불가용
권한 상승취약점 악용전체 시스템 침해

6.4 보안 부팅 및 런타임 무결성

보안 부팅 체인
===============
단계 1: 하드웨어 신뢰 루트
   └── 불변 부팅 ROM
   └── 하드코딩된 루트 공개키 해시

단계 2: 1단계 부트로더
   └── 2단계 서명 검증
   └── HSM을 사용한 암호화 검증

단계 3: 2단계 부트로더
   └── 커널/OS 서명 검증
   └── 검증된 운영 체제 로드

단계 4: 운영 체제
   └── 애플리케이션 서명 검증
   └── 검증된 애플리케이션 시작

단계 5: 애플리케이션 런타임
   └── 지속적인 무결성 모니터링
   └── 코드 서명 검증

모든 단계 실패 → 안전 복구 모드로 부팅

6.5 네트워크 보안

CAN 버스 보안 조치

  • 메시지 인증 코드: 중요 CAN 메시지에 MAC 추가
  • 게이트웨이 필터링: 경계에서 승인되지 않은 메시지 ID 차단
  • 침입 탐지: 비정상적인 메시지 패턴 모니터링
  • 암호화: 대역폭이 허용하는 경우 민감한 데이터 암호화

6.6 OTA 업데이트 보안

안전한 OTA 업데이트 프로세스
============================
1. 업데이트 패키지 생성
   └── 제조업체 개인키로 서명
   └── 버전 메타데이터 포함
   └── 민감한 콘텐츠 암호화

2. 안전한 배포
   └── 인증된 HTTPS 서버
   └── 인증서 피닝
   └── 무결성 검증

3. 차량 측 검증
   └── 서명 검증
   └── 버전 확인 (롤백 방지)
   └── 호환성 검증

4. 안전한 설치
   └── A/B 파티션 체계
   └── 설치 후 검증
   └── 실패 시 자동 롤백

단계활동요구사항

교훈분석, 방어 개선사고 후 보고서

6.8 프라이버시 보호

  • 데이터 최소화: 필요한 데이터만 수집
  • 목적 제한: 명시된 목적으로만 데이터 사용
  • 익명화: 식별 정보 제거
  • 사용자 제어: 투명성과 제어 제공
  • 안전한 저장: 저장 및 전송 중 개인 데이터 암호화
