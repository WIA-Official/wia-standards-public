Chapter 6: Cybersecurity Framework

As autonomous vehicles become increasingly connected and software-defined, cybersecurity becomes essential to vehicle safety. This chapter establishes WIA-AUTO requirements for protecting autonomous vehicles against cyber threats throughout their lifecycle.

6.1 Cybersecurity Landscape

Autonomous Vehicle Attack Surface
=================================
External Interfaces:
├── V2X Communication (DSRC/C-V2X)
├── Cellular Connectivity (4G/5G)
├── Wi-Fi (for updates, diagnostics)
├── Bluetooth (phone connectivity)
├── GPS/GNSS Signals
├── Physical Ports (OBD-II, USB, debug)
└── Sensor Inputs (camera, LiDAR spoofing)

Internal Systems:
├── Infotainment System
├── Telematics Control Unit
├── Body Control Module
├── Autonomous Driving Computer
├── V2X On-Board Unit
├── Gateway/Firewall
└── Safety-Critical ECUs

6.4 Secure Boot and Runtime Integrity

Secure Boot Chain
=================
Step 1: Hardware Root of Trust
   └── Immutable boot ROM
   └── Hardcoded root public key hash

Step 2: First Stage Bootloader
   └── Verify signature of second stage
   └── Cryptographic verification using HSM

Step 3: Second Stage Bootloader
   └── Verify kernel/OS signature
   └── Load verified operating system

Step 4: Operating System
   └── Verify application signatures
   └── Launch verified applications

Step 5: Application Runtime
   └── Continuous integrity monitoring
   └── Code signing verification

Failure at any step → Boot into safe recovery mode

6.5 Network Security

CAN Bus Security Measures

  • Message Authentication Codes: Append MACs to critical CAN messages
  • Gateway Filtering: Block unauthorized message IDs at boundaries
  • Intrusion Detection: Monitor for anomalous message patterns
  • Encryption: Encrypt sensitive data where bandwidth permits

6.6 Over-the-Air Update Security

Secure OTA Update Process
=========================
1. Update Package Creation
   └── Sign with manufacturer private key
   └── Include version metadata
   └── Encrypt sensitive content

2. Secure Distribution
   └── Authenticated HTTPS servers
   └── Certificate pinning
   └── Integrity verification

3. Vehicle-Side Verification
   └── Signature verification
   └── Version checking (prevent rollback)
   └── Compatibility verification

4. Safe Installation
   └── A/B partition scheme
   └── Post-installation verification
   └── Automatic rollback on failure

6.8 Privacy Protection

  • Data Minimization: Collect only necessary data
  • Purpose Limitation: Use data only for specified purposes
  • Anonymization: Remove identifying information
  • User Control: Provide transparency and control
  • Secure Storage: Encrypt personal data at rest and in transit
