Chapter 2: Sensor Systems and Perception

The perception system serves as the eyes and ears of an autonomous vehicle, transforming raw sensor data into actionable environmental understanding. This chapter defines WIA-AUTO requirements for sensor selection, calibration, fusion, and perception algorithm performance.

2.1 Perception System Architecture

A robust perception system requires multiple complementary sensors working in concert to achieve reliable environmental understanding under all operational conditions. The WIA-AUTO standard mandates a multi-modal sensor approach that provides redundancy and fault tolerance critical for safety-rated autonomous driving systems.

2.2 LiDAR Systems

Light Detection and Ranging (LiDAR) sensors form the backbone of most autonomous vehicle perception systems, providing precise 3D environmental mapping through the measurement of laser pulse time-of-flight.

LiDAR Technology Types

  • Mechanical Spinning LiDAR: Traditional design with rotating laser assembly providing 360-degree coverage
  • Solid-State LiDAR: No moving parts, utilizing MEMS mirrors or optical phased arrays
  • Flash LiDAR: Illuminates entire scene simultaneously, capturing 3D images in single pulse
  • FMCW LiDAR: Frequency-modulated continuous wave technology providing simultaneous range and velocity

2.3 Radar Systems

Radar sensors complement LiDAR and cameras by providing reliable object detection and velocity measurement in all weather conditions. While offering lower angular resolution than LiDAR, radar excels in detecting moving objects and penetrating adverse weather.

2.4 Camera Systems

Cameras provide rich visual information essential for scene understanding, including traffic sign recognition, traffic light detection, lane marking identification, and object classification.

Camera Types and Configurations

  • Monocular Cameras: Single camera providing 2D images for classification and detection
  • Stereo Cameras: Paired cameras enabling disparity-based depth estimation
  • Surround View: Multiple wide-angle cameras providing 360-degree visual coverage
  • Thermal/IR Cameras: Infrared cameras for pedestrian detection in low-light conditions

2.5 Sensor Fusion Approaches

Sensor fusion combines data from multiple sensors to overcome individual sensor limitations and achieve robust perception. WIA-AUTO recognizes multiple fusion approaches:

// Multi-Modal Sensor Fusion Example
class SensorFusion {
    WorldModel fuse(LiDARData lidar, RadarData radar, CameraData cameras) {
        // Get detections from each modality
        lidar_objects = lidarDetector.detect(lidar);
        radar_objects = radarDetector.detect(radar);
        camera_objects = cameraDetector.detect(cameras);

        // Associate detections across modalities
        associations = hungarianMatching(
            lidar_objects, radar_objects, camera_objects
        );

        // Fuse associated objects using Kalman filter
        fused_objects = [];
        for (assoc in associations) {
            fused = kalmanFuse(assoc.lidar, assoc.radar, assoc.camera);
            fused_objects.append(fused);
        }
        return WorldModel(fused_objects);
    }
}

2.7 Sensor Calibration Requirements

Accurate sensor calibration is essential for reliable perception. WIA-AUTO mandates both intrinsic calibration (internal sensor parameters) and extrinsic calibration (relative pose between sensors) with specified accuracy requirements.

Calibration Maintenance

Autonomous vehicles must continuously monitor calibration quality and detect degradation. Online calibration verification using natural features should trigger alerts when calibration drift exceeds acceptable thresholds.
