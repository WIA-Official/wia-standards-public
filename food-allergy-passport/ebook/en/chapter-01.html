Chapter 1: Digital Allergy Profiles

Comprehensive allergy profile management and medical verification

1.1 Introduction to Digital Allergy Profiles

Digital allergy profiles represent the cornerstone of the WIA Food Allergy Passport Standard. These comprehensive digital records contain detailed information about an individual's food allergies, intolerances, and dietary restrictions, providing a standardized format for communication between allergic individuals, food service providers, healthcare professionals, and emergency responders.

A digital allergy profile goes beyond simple lists of allergens. It includes severity classifications, cross-reactivity information, emergency contact details, medication protocols, medical verification credentials, and historical reaction data. This holistic approach ensures that all stakeholders have access to complete and accurate information necessary for maintaining food safety.

Core Components of Allergy Profiles

Each digital allergy profile must contain the following standardized components:

  • Primary Allergens: IgE-mediated food allergies with severity classifications
  • Cross-Reactivity Data: Related foods that may trigger reactions due to protein similarities
  • Dietary Restrictions: Additional restrictions beyond allergies (religious, ethical, health-related)
  • Medical Verification: Healthcare provider attestations and diagnostic test results
  • Emergency Protocols: Response procedures, medications, and emergency contacts
  • Reaction History: Documented past allergic reactions with severity and treatments
  • Safe Foods Database: Pre-approved foods and restaurant dishes
  • Preference Settings: Risk tolerance levels and notification preferences

Allergy Severity Classification

Severity Level Code Description Response Required
Life-Threatening SEVERITY_1 History of anaphylaxis, requires immediate EpiPen Complete allergen elimination, emergency preparation
Severe SEVERITY_2 Systemic reactions, may progress to anaphylaxis Strict avoidance, antihistamines available
Moderate SEVERITY_3 Localized reactions, uncomfortable but not life-threatening Avoidance recommended, medication as needed
Mild SEVERITY_4 Minor symptoms, oral allergy syndrome Caution advised, patient may tolerate cooked forms
Intolerance SEVERITY_5 Non-IgE mediated, digestive issues Dietary management, no emergency risk

Medical Verification Requirements

Verification Type Required For Validity Period Documentation
Allergist Confirmation All SEVERITY_1 and SEVERITY_2 allergies 12 months Signed medical letter with license number
Skin Prick Test Primary allergen diagnosis 24 months Lab results with wheal size measurements
Blood IgE Test Confirmation testing 24 months Specific IgE levels (kU/L)

1.3 API Endpoints for Profile Management

The WIA Food Allergy Passport Standard defines comprehensive RESTful API endpoints for creating, reading, updating, and managing digital allergy profiles.

POST /api/v1/profiles

Create a new allergy profile. Requires user authentication and initial medical verification.

GET /api/v1/profiles/{profileId}

Retrieve complete allergy profile by ID. Returns full profile data including all allergies and medical history.

PUT /api/v1/profiles/{profileId}

Update existing allergy profile. Version control enforced to prevent conflicting updates.

DELETE /api/v1/profiles/{profileId}

Soft delete allergy profile. Data retained for 90 days for recovery, then permanently deleted.

POST /api/v1/profiles/{profileId}/allergies

Add new allergen to existing profile. Requires medical verification for SEVERITY_1 and SEVERITY_2.

PUT /api/v1/profiles/{profileId}/allergies/{allergenId}

Update allergen details including severity, test results, or reaction history.

DELETE /api/v1/profiles/{profileId}/allergies/{allergenId}

Remove allergen from profile. Requires allergist confirmation via oral food challenge.

POST /api/v1/profiles/{profileId}/verify

Submit medical verification documents. Accepts physician letters, test results, and prescription images.

GET /api/v1/profiles/{profileId}/verification-status

Check verification status for all allergies. Returns expiration dates and renewal requirements.

POST /api/v1/profiles/{profileId}/reactions

Log new allergic reaction incident. Captures symptoms, severity, treatment, and location data.

GET /api/v1/profiles/{profileId}/cross-reactivities

Get comprehensive cross-reactivity analysis for all allergies in profile.

POST /api/v1/profiles/{profileId}/emergency-access

Generate time-limited emergency access token for first responders.

Code Example: Creating an Allergy Profile

// JavaScript/Node.js Example
const axios = require('axios');

async function createAllergyProfile(userData) {
  try {
    const response = await axios.post('https://api.wia-foodallergy.org/api/v1/profiles', {
      personalInfo: {
        firstName: userData.firstName,
        lastName: userData.lastName,
        dateOfBirth: userData.dateOfBirth,
        emergencyContact: {
          name: userData.emergencyContactName,
          phone: userData.emergencyContactPhone,
          relationship: userData.emergencyContactRelationship
        }
      },
      allergies: userData.allergies.map(allergy => ({
        allergenId: allergy.allergenId,
        severityLevel: allergy.severityLevel,
        diagnosisDate: allergy.diagnosisDate,
        verifiedBy: {
          physicianName: allergy.physicianName,
          licenseNumber: allergy.licenseNumber,
          specialty: 'Allergist/Immunologist'
        }
      })),
      safetyPreferences: {
        crossContaminationTolerance: 'ZERO_TOLERANCE',
        sharedEquipmentAcceptable: false,
        requiresDedicatedPreparationArea: true
      },
      emergencyAccessEnabled: true
    }, {
      headers: {
        'Authorization': `Bearer ${userData.authToken}`,
        'Content-Type': 'application/json',
        'X-API-Version': '1.0.0'
      }
    });

    console.log('Profile created successfully:', response.data.profileId);
    return response.data;
  } catch (error) {
    console.error('Error creating profile:', error.response.data);
    throw error;
  }
}

Code Example: Retrieving Profile Data

// Python Example
import requests
from datetime import datetime

class AllergyProfileClient:
    def __init__(self, api_base_url, auth_token):
        self.base_url = api_base_url
        self.headers = {
            'Authorization': f'Bearer {auth_token}',
            'Content-Type': 'application/json',
            'X-API-Version': '1.0.0'
        }

    def get_profile(self, profile_id):
        """Retrieve complete allergy profile"""
        try:
            response = requests.get(
                f'{self.base_url}/api/v1/profiles/{profile_id}',
                headers=self.headers
            )
            response.raise_for_status()
            return response.json()
        except requests.exceptions.HTTPError as e:
            print(f'Error retrieving profile: {e}')
            raise

    def check_allergen_safety(self, profile_id, ingredient_list):
        """Check if ingredients are safe for profile"""
        profile = self.get_profile(profile_id)
        unsafe_ingredients = []

        for allergy in profile['allergies']:
            allergen_name = allergy['allergenName'].lower()

            # Check direct matches
            for ingredient in ingredient_list:
                if allergen_name in ingredient.lower():
                    unsafe_ingredients.append({
                        'ingredient': ingredient,
                        'allergen': allergy['allergenName'],
                        'severity': allergy['severityLevel']
                    })

            # Check cross-reactivities
            for cross_reactive in allergy.get('crossReactivities', []):
                for ingredient in ingredient_list:
                    if cross_reactive.lower() in ingredient.lower():
                        unsafe_ingredients.append({
                            'ingredient': ingredient,
                            'allergen': f'{allergy["allergenName"]} (cross-reactive)',
                            'severity': allergy['severityLevel']
                        })

        return {
            'safe': len(unsafe_ingredients) == 0,
            'unsafe_ingredients': unsafe_ingredients,
            'checked_at': datetime.utcnow().isoformat()
        }

# Usage
client = AllergyProfileClient(
    'https://api.wia-foodallergy.org',
    'user-jwt-token-here'
)

profile = client.get_profile('uuid-profile-id')
safety_check = client.check_allergen_safety(
    'uuid-profile-id',
    ['peanut oil', 'wheat flour', 'soy lecithin', 'milk']
)

print(f"Meal is safe: {safety_check['safe']}")
if not safety_check['safe']:
    print("Unsafe ingredients detected:")
    for item in safety_check['unsafe_ingredients']:
        print(f"  - {item['ingredient']}: {item['allergen']} ({item['severity']})")

1.4 Certification Requirements for Profile Management

To achieve WIA Food Allergy Passport certification, implementations must meet the following requirements for digital allergy profile management:

CERT-PROFILE-001: System must support creation of comprehensive allergy profiles with minimum fields: user identification, allergen list, severity levels, emergency contacts, and medical verification.
CERT-PROFILE-002: All SEVERITY_1 (life-threatening) allergies must require medical verification from licensed allergist or immunologist.
CERT-PROFILE-003: Profile data must be encrypted at rest using AES-256 encryption and in transit using TLS 1.3 or higher.
CERT-PROFILE-004: System must implement version control for all profile updates with complete audit trail.
CERT-PROFILE-005: Emergency access mode must allow first responders to view critical allergy information without authentication.
CERT-PROFILE-006: Cross-reactivity database must include minimum 500 documented allergen relationships.
CERT-PROFILE-007: Profile photos must be compressed and optimized to ensure QR code generation under 3KB.
CERT-PROFILE-008: Medical verification documents must be stored securely with HIPAA-compliant access controls.
CERT-PROFILE-009: System must support multi-language profile display for minimum 12 languages.
CERT-PROFILE-010: Allergy severity levels must follow standardized SEVERITY_1 through SEVERITY_5 classification.
CERT-PROFILE-011: Profile must include expiration tracking for all time-sensitive medical verifications.
CERT-PROFILE-012: System must send automated reminders 30 days before medical verification expiration.
CERT-PROFILE-013: Reaction history logging must capture minimum data: date, symptoms, treatment, location, severity.
CERT-PROFILE-014: EpiPen and emergency medication tracking must include expiration alerts.
CERT-PROFILE-015: Profile backup and export functionality must be available in standardized JSON format.
CERT-PROFILE-016: System must support household profiles linking multiple family member profiles.
CERT-PROFILE-017: Parental controls must be implemented for minor children's profiles with guardian approval workflows.
CERT-PROFILE-018: Profile sharing permissions must allow granular control (full access, emergency only, allergen list only).
CERT-PROFILE-019: API rate limiting must prevent abuse while allowing legitimate high-frequency access patterns.
CERT-PROFILE-020: Profile deletion must implement 90-day soft delete with recovery option before permanent deletion.
CERT-PROFILE-021: System must log all profile access attempts for security auditing purposes.
CERT-PROFILE-022: Offline profile access must be supported through encrypted local storage on mobile devices.
CERT-PROFILE-023: Profile synchronization across multiple devices must resolve conflicts using last-write-wins with user notification.
CERT-PROFILE-024: Medication dosage calculations must account for patient weight, age, and severity level.
CERT-PROFILE-025: System must integrate with major EHR systems via FHIR standard for medical record import.
