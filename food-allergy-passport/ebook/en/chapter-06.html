Chapter 6: Data Standards & Security

The WIA Food Allergy Passport Standard implements comprehensive data security and privacy protections to safeguard sensitive medical information while enabling life-saving access during emergencies. This chapter defines encryption standards, HIPAA compliance requirements, GDPR alignment, blockchain verification, and complete audit logging protocols.

6.3 Security API Endpoints

POST/api/v1/security/authenticate

Authenticate user with multi-factor authentication support.

POST/api/v1/security/refresh-token

Refresh expired JWT token without re-authentication.

GET/api/v1/security/audit-log/{profileId}

Retrieve complete audit log for profile access and modifications.

POST/api/v1/security/data-export

Export all personal data (GDPR Article 20 - Data Portability).

POST/api/v1/security/data-deletion

Request permanent data deletion (GDPR Article 17 - Right to Erasure).

GET/api/v1/security/access-history

View all access attempts and successful logins.

POST/api/v1/security/revoke-access

Revoke third-party access permissions immediately.

POST/api/v1/security/encryption/rotate-keys

Manually trigger encryption key rotation.

GET/api/v1/security/compliance-report

Generate compliance report for HIPAA/GDPR auditing.

POST/api/v1/security/breach-notification

Trigger data breach notification workflow.

Code Example: Secure Data Access

// Node.js - Secure Profile Access with Encryption
const crypto = require('crypto');
const axios = require('axios');

class SecureProfileAccess {
  constructor(apiKey, encryptionKey) {
    this.apiKey = apiKey;
    this.encryptionKey = Buffer.from(encryptionKey, 'hex');
  }

  async authenticateWithMFA(username, password, mfaCode) {
    const response = await axios.post(
      'https://api.wia-foodallergy.org/api/v1/security/authenticate',
      {
        username: username,
        password: await this.hashPassword(password),
        mfaCode: mfaCode,
        deviceId: this.getDeviceId()
      }
    );
    
    this.accessToken = response.data.accessToken;
    this.refreshToken = response.data.refreshToken;
    return response.data;
  }

  async getEncryptedProfile(profileId) {
    const response = await axios.get(
      `https://api.wia-foodallergy.org/api/v1/profiles/${profileId}`,
      {headers: {'Authorization': `Bearer ${this.accessToken}`}}
    );

    // Decrypt sensitive fields
    const profile = response.data;
    profile.allergies = await this.decryptFieldArray(profile.allergies);
    profile.emergencyContacts = await this.decryptField(profile.emergencyContacts);
    
    return profile;
  }

  async decryptField(encryptedData) {
    const decipher = crypto.createDecipheriv(
      'aes-256-gcm',
      this.encryptionKey,
      Buffer.from(encryptedData.iv, 'hex')
    );
    decipher.setAuthTag(Buffer.from(encryptedData.authTag, 'hex'));
    
    let decrypted = decipher.update(encryptedData.data, 'hex', 'utf8');
    decrypted += decipher.final('utf8');
    return JSON.parse(decrypted);
  }

  async hashPassword(password) {
    return crypto.createHash('sha256').update(password).digest('hex');
  }

  getDeviceId() {
    return crypto.createHash('sha256').update(navigator.userAgent).digest('hex');
  }
}

// Usage
const secureAccess = new SecureProfileAccess(apiKey, encryptionKey);
await secureAccess.authenticateWithMFA('user@email.com', 'password', '123456');
const profile = await secureAccess.getEncryptedProfile('profile-uuid');

6.4 Security Certification Requirements

CERT-SECURITY-001: All PHI must be encrypted at rest using AES-256-GCM or stronger.
CERT-SECURITY-002: TLS 1.3 required for all data transmission, TLS 1.2 minimum for legacy support.
CERT-SECURITY-003: Multi-factor authentication required for all profile modifications.
CERT-SECURITY-004: Passwords must meet NIST 800-63-3 standards (12+ characters, complexity requirements).
CERT-SECURITY-005: Encryption keys must rotate every 90 days with automated key management.
CERT-SECURITY-006: All access attempts must be logged with timestamp, IP address, and outcome.
CERT-SECURITY-007: Audit logs must be immutable and retained for minimum 7 years (HIPAA).
CERT-SECURITY-008: Data breach notification within 72 hours of discovery (GDPR Article 33).
CERT-SECURITY-009: Right to data portability via standardized JSON export (GDPR Article 20).
CERT-SECURITY-010: Right to erasure implementation with 30-day hard delete (GDPR Article 17).
CERT-SECURITY-011: Business Associate Agreement required for all third-party processors (HIPAA).
CERT-SECURITY-012: Annual penetration testing by certified security firm.
CERT-SECURITY-013: Vulnerability scanning and patching within 30 days of critical CVE publication.
CERT-SECURITY-014: Database backups encrypted and tested monthly for recovery.
CERT-SECURITY-015: Role-based access control with least privilege principle.
CERT-SECURITY-016: Session tokens expire after 1 hour of inactivity.
CERT-SECURITY-017: Failed login lockout after 5 attempts with exponential backoff.
CERT-SECURITY-018: API rate limiting to prevent abuse (100 requests/minute per user).
CERT-SECURITY-019: SQL injection prevention via parameterized queries and ORM.
CERT-SECURITY-020: XSS protection via content security policy and input sanitization.
CERT-SECURITY-021: CSRF protection via synchronizer tokens on all state-changing operations.
CERT-SECURITY-022: Blockchain or cryptographic signatures for medical verification documents.
CERT-SECURITY-023: Secure credential storage using industry-standard vaults (HashiCorp Vault, AWS Secrets Manager).
CERT-SECURITY-024: Privacy impact assessment documented and reviewed annually.
CERT-SECURITY-025: SOC 2 Type II certification or equivalent security audit.
