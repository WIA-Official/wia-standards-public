Chapter 2: Restaurant Integration

Real-time menu analysis and kitchen communication protocols

2.1 Restaurant Integration Overview

Restaurant integration represents one of the most critical components of the WIA Food Allergy Passport Standard. This chapter defines comprehensive protocols for connecting digital allergy profiles with restaurant menu systems, enabling real-time ingredient analysis, allergen warnings, and safe meal recommendations.

The restaurant integration system creates a seamless bridge between allergic individuals and food service providers. When a customer scans their allergy passport QR code at a restaurant, the system instantly analyzes the entire menu against their allergy profile, identifying safe dishes, potentially dangerous items, and providing customization recommendations to make dishes allergen-free.

Integration Architecture Components

  • Menu Database Standardization: Structured ingredient data with allergen tagging
  • Real-Time Analysis Engine: Instant menu scanning against allergy profiles
  • Kitchen Communication System: Direct alerts to preparation staff
  • Cross-Contamination Tracking: Shared equipment and facility warnings
  • Staff Training Verification: Certification tracking for allergen-aware personnel
  • Modification Suggestions: Automated recipe adjustments for allergen removal
  • Confidence Scoring: Risk assessment for each menu item
  • Audit Trail: Complete logging of all allergen-related communications

Restaurant Certification Levels

Level Name Requirements Capabilities
Level 3 Allergy Excellence Full WIA integration, dedicated allergen-free kitchen zone, certified staff Can safely serve all SEVERITY_1 customers, custom meal preparation
Level 2 Allergy Aware Complete menu database, allergen training, separate prep areas Can accommodate most allergies with proper precautions
Level 1 Basic Compliance Allergen labeling, basic staff training, standard protocols Can identify allergens in menu items, limited customization
Level 0 Not Certified No formal allergen program Customer assumes all risk, no system integration

Menu Item Safety Classification

Safety Status Code Criteria Customer Guidance
Safe SAFE_VERIFIED No allergens, no cross-contamination risk, verified by chef Recommended for ordering
Safe with Modification SAFE_MODIFIED Contains allergen but can be removed/substituted Order with specified modifications
Caution CAUTION_CROSS_CONTACT No direct allergen but shares equipment/fryer Discuss with staff, risk assessment required
Unsafe UNSAFE_CONTAINS Contains declared allergen as primary ingredient Do not order
Unknown UNKNOWN_INCOMPLETE Incomplete ingredient data or complex preparation Requires detailed discussion with chef

2.3 Restaurant Integration API Endpoints

The WIA standard defines comprehensive APIs for restaurant integration, enabling real-time menu analysis and kitchen communication.

POST /api/v1/restaurants/register

Register new restaurant with system. Requires business verification and initial menu upload.

GET /api/v1/restaurants/{restaurantId}

Retrieve restaurant details including certification level and allergen protocols.

POST /api/v1/restaurants/{restaurantId}/menu

Upload or update complete restaurant menu with ingredient data.

GET /api/v1/restaurants/{restaurantId}/menu

Retrieve complete menu with all ingredient and allergen information.

POST /api/v1/restaurants/{restaurantId}/analyze

Analyze restaurant menu against customer allergy profile. Returns safe/unsafe items with confidence scores.

POST /api/v1/restaurants/{restaurantId}/orders

Place allergen-aware order with kitchen alerts and special preparation instructions.

GET /api/v1/restaurants/{restaurantId}/certification

Get restaurant certification status, level, and expiration date.

POST /api/v1/restaurants/{restaurantId}/staff/verify

Verify staff allergen training certification status.

GET /api/v1/restaurants/search

Search for allergen-aware restaurants by location, certification level, and cuisine type.

POST /api/v1/restaurants/{restaurantId}/incidents/report

Report allergen-related incident or cross-contamination event.

GET /api/v1/restaurants/{restaurantId}/safety-score

Get restaurant safety score based on incident history and compliance audits.

PUT /api/v1/restaurants/{restaurantId}/menu/items/{itemId}

Update specific menu item with new ingredients or allergen information.

Code Example: Menu Analysis

// JavaScript/Node.js Example - Restaurant Menu Analysis
const axios = require('axios');

async function analyzeMenuForCustomer(restaurantId, profileId, authToken) {
  try {
    const response = await axios.post(
      `https://api.wia-foodallergy.org/api/v1/restaurants/${restaurantId}/analyze`,
      {
        profileId: profileId,
        includeModifications: true,
        minimumConfidenceScore: 0.90,
        excludeCautionItems: false
      },
      {
        headers: {
          'Authorization': `Bearer ${authToken}`,
          'Content-Type': 'application/json',
          'X-API-Version': '1.0.0'
        }
      }
    );

    const analysis = response.data;

    console.log(`\n=== Menu Analysis for ${analysis.restaurantName} ===`);
    console.log(`Restaurant Certification: ${analysis.restaurantCertificationLevel}`);
    console.log(`\nSafe Dishes (${analysis.menuAnalysis.safeItems}):`);

    analysis.recommendedDishes
      .filter(item => item.safetyStatus === 'SAFE_VERIFIED')
      .forEach(dish => {
        console.log(`  ✓ ${dish.itemName} (Confidence: ${dish.confidenceScore * 100}%)`);
      });

    console.log(`\nSafe with Modifications (${analysis.menuAnalysis.safeWithModification}):`);
    analysis.recommendedDishes
      .filter(item => item.safetyStatus === 'SAFE_MODIFIED')
      .forEach(dish => {
        console.log(`  ⚠ ${dish.itemName}`);
        dish.requiredModifications.forEach(mod => {
          console.log(`     - ${mod}`);
        });
      });

    console.log(`\nUnsafe Dishes (${analysis.menuAnalysis.unsafeItems}):`);
    analysis.unsafeDishes.forEach(dish => {
        console.log(`  ✗ ${dish.itemName}: ${dish.reason}`);
    });

    return analysis;
  } catch (error) {
    console.error('Menu analysis failed:', error.response?.data || error.message);
    throw error;
  }
}

// Usage
const menuAnalysis = await analyzeMenuForCustomer(
  'restaurant-uuid-here',
  'customer-profile-uuid-here',
  'user-auth-token-here'
);

// Python Example - Restaurant Registration
import requests
from typing import List, Dict

class RestaurantIntegration:
    def __init__(self, api_base_url: str, api_key: str):
        self.base_url = api_base_url
        self.headers = {
            'Authorization': f'Bearer {api_key}',
            'Content-Type': 'application/json',
            'X-API-Version': '1.0.0'
        }

    def register_restaurant(self, restaurant_data: Dict) -> Dict:
        """Register new restaurant with WIA Food Allergy system"""
        endpoint = f'{self.base_url}/api/v1/restaurants/register'

        payload = {
            'restaurantName': restaurant_data['name'],
            'location': {
                'address': restaurant_data['address'],
                'city': restaurant_data['city'],
                'state': restaurant_data['state'],
                'zipCode': restaurant_data['zip'],
                'country': restaurant_data['country'],
                'gps': restaurant_data['coordinates']
            },
            'contactInfo': {
                'phone': restaurant_data['phone'],
                'email': restaurant_data['email'],
                'website': restaurant_data.get('website')
            },
            'facilityInfo': {
                'hasDedicatedAllergenFreeZone': restaurant_data['dedicated_zone'],
                'separateFryers': restaurant_data['separate_fryers'],
                'allergenTrainedStaff': restaurant_data['trained_staff_count'],
                'certifiedAllergenChef': restaurant_data['certified_chef'],
                'facilityAllergens': restaurant_data['facility_allergens']
            },
            'cuisineTypes': restaurant_data['cuisine_types'],
            'requestedCertificationLevel': restaurant_data['target_certification']
        }

        try:
            response = requests.post(endpoint, json=payload, headers=self.headers)
            response.raise_for_status()
            return response.json()
        except requests.exceptions.HTTPError as e:
            print(f'Registration failed: {e}')
            raise

    def upload_menu(self, restaurant_id: str, menu_data: Dict) -> Dict:
        """Upload complete menu with ingredient data"""
        endpoint = f'{self.base_url}/api/v1/restaurants/{restaurant_id}/menu'

        try:
            response = requests.post(endpoint, json=menu_data, headers=self.headers)
            response.raise_for_status()
            return response.json()
        except requests.exceptions.HTTPError as e:
            print(f'Menu upload failed: {e}')
            raise

    allergen: string;
    severity: string;

  items: Array<{

    specialInstructions: string;
  }>;
  alertLevel: 'CRITICAL' | 'HIGH' | 'MEDIUM';
}

      auth: {
        token: localStorage.getItem('kitchenToken'),
        restaurantId: localStorage.getItem('restaurantId')

2.4 Certification Requirements for Restaurant Integration

CERT-RESTAURANT-001: Restaurant must maintain complete ingredient database for all menu items with allergen tagging.
CERT-RESTAURANT-002: All menu data must be updated within 24 hours of any ingredient changes.
CERT-RESTAURANT-003: Level 2 and Level 3 restaurants must have minimum 80% of staff allergen-trained and certified.
CERT-RESTAURANT-004: Kitchen must implement visual alert system (red ticket) for allergen orders.
CERT-RESTAURANT-005: Cross-contamination protocols must be documented and verifiable through audit.
CERT-RESTAURANT-006: Level 3 certification requires dedicated allergen-free preparation zone.
CERT-RESTAURANT-007: Separate fryers and cooking equipment required for Level 2 and Level 3 certification.
CERT-RESTAURANT-008: Restaurant must designate certified allergen manager on duty during all operating hours.
CERT-RESTAURANT-009: Menu analysis API response time must be under 2 seconds for standard menu sizes.
CERT-RESTAURANT-010: Kitchen alert system must provide real-time notifications to preparation staff.
CERT-RESTAURANT-011: All allergen-related incidents must be logged and reported within 2 hours.
CERT-RESTAURANT-012: Restaurant must maintain EpiPen supply and trained staff for emergency response.
CERT-RESTAURANT-013: Ingredient supplier documentation must be retained for minimum 2 years.
CERT-RESTAURANT-014: Menu modifications must be confirmable by kitchen staff before order preparation.
CERT-RESTAURANT-015: Restaurant must pass quarterly allergen compliance audits to maintain certification.
CERT-RESTAURANT-016: Confidence scoring algorithm must achieve minimum 95% accuracy on test datasets.
CERT-RESTAURANT-017: Double-check procedures required for all SEVERITY_1 allergen orders.
CERT-RESTAURANT-018: Manager approval required before serving customers with multiple SEVERITY_1 allergies.
CERT-RESTAURANT-019: System must maintain complete audit trail of all allergen-related communications.
CERT-RESTAURANT-020: Restaurant must provide allergen awareness training to new staff within 7 days of hiring.
CERT-RESTAURANT-021: Emergency contact information for local hospitals must be readily accessible to all staff.
CERT-RESTAURANT-022: Cleaning and sanitization protocols must prevent allergen cross-contact on surfaces and equipment.
CERT-RESTAURANT-023: Recipe standardization required to ensure consistent allergen content across locations.
CERT-RESTAURANT-024: System must flag when shared equipment is used for allergen-containing and allergen-free items.
CERT-RESTAURANT-025: Customer feedback system must capture allergen-related experiences and safety concerns.
