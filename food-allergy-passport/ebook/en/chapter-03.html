Chapter 3: QR Code Systems

Secure QR code generation and instant profile sharing

3.1 QR Code System Architecture

The WIA Food Allergy Passport QR code system enables instant, secure sharing of allergy profiles with restaurants, healthcare providers, and emergency responders. The QR code serves as a universal access point to critical allergy information while maintaining privacy controls and offline functionality.

Unlike traditional QR codes that simply encode URLs, the WIA standard implements a sophisticated hybrid system that embeds critical emergency data directly in the QR code while referencing extended profile data through secure API endpoints. This ensures that life-saving information remains accessible even without internet connectivity, while detailed medical history and preferences remain protected behind authentication layers.

QR Code Data Layers

Layer Data Type Availability Maximum Size
Emergency Layer Life-threatening allergies, EpiPen location, emergency contacts Embedded in QR (offline) 2KB
Profile Reference Profile UUID, API endpoint, authentication token Embedded in QR 512 bytes
Extended Profile Complete allergy history, test results, dietary preferences API fetch (online) Unlimited
Medical Records Physician notes, lab results, prescriptions API fetch (authenticated) Unlimited

QR Code Security Levels

Security Level Access Method Data Available Use Case
Emergency Access No authentication required SEVERITY_1 allergies, emergency contacts, current medications First responders, paramedics
Restaurant Access One-time session token All allergies, dietary restrictions, safe food preferences Restaurant staff during dining
Healthcare Access Medical professional authentication Complete medical history, test results, treatment plans Doctors, allergists, nurses
Full Access User authentication required All data including personal settings and preferences Profile owner only

3.3 QR Code API Endpoints

POST /api/v1/qr/generate

Generate new QR code for allergy profile with specified security settings and expiration.

GET /api/v1/qr/{qrCodeId}

Scan QR code and retrieve emergency data plus session token for extended access.

POST /api/v1/qr/{qrCodeId}/refresh

Refresh expired session token to maintain access during active dining session.

GET /api/v1/qr/{qrCodeId}/profile

Retrieve full allergy profile using valid session token from QR code scan.

POST /api/v1/qr/{qrCodeId}/scan-log

Log QR code scan event with scanner information and location data.

GET /api/v1/qr/{qrCodeId}/scan-history

Retrieve scan history for QR code including dates, locations, and scanner types.

PUT /api/v1/qr/{qrCodeId}/settings

Update QR code settings including security level, expiration, and notification preferences.

POST /api/v1/qr/{qrCodeId}/revoke

Immediately revoke QR code access and invalidate all active session tokens.

GET /api/v1/qr/{qrCodeId}/verify

Verify QR code authenticity and check for tampering or expiration.

POST /api/v1/qr/{qrCodeId}/emergency-access

Request emergency access override for first responders with verification.

GET /api/v1/qr/batch-generate

Generate multiple QR codes for family members or different access levels.

POST /api/v1/qr/{qrCodeId}/notify-owner

Send notification to profile owner about QR code scan or access attempt.

Code Example: QR Code Generation

// JavaScript/Node.js - QR Code Generation
const QRCode = require('qrcode');
const axios = require('axios');

async function generateAllergyPassportQR(profileId, settings = {}) {
  try {
    // Request QR code generation from API
    const response = await axios.post(
      'https://api.wia-foodallergy.org/api/v1/qr/generate',
      {
        profileId: profileId,
        securityLevel: settings.securityLevel || 'MULTI_TIER',
        expirationDays: settings.expirationDays || 365,
        includeOfflineData: settings.includeOfflineData !== false,
        emergencyAccessEnabled: settings.emergencyAccessEnabled !== false,
        notifyOnScan: settings.notifyOnScan !== false,
        allowedScannerTypes: settings.allowedScannerTypes || [
          'RESTAURANT', 'HEALTHCARE', 'EMERGENCY'
        ]
      },
      {
        headers: {
          'Authorization': `Bearer ${settings.authToken}`,
          'Content-Type': 'application/json',
          'X-API-Version': '1.0.0'
        }
      }
    );

    const qrData = response.data;
    console.log('QR Code generated:', qrData.qrCodeId);

    // Generate actual QR code image
    const qrCodeDataUrl = await QRCode.toDataURL(
      JSON.stringify(qrData.qrCodePayload),
      {
        errorCorrectionLevel: 'H',
        type: 'image/png',
        quality: 1.0,
        margin: 2,
        width: 512,
        color: {
          dark: '#000000',
          light: '#FFFFFF'
        }
      }
    );

    return {
      qrCodeId: qrData.qrCodeId,
      qrCodeImage: qrCodeDataUrl,
      expiresAt: qrData.expiresAt,
      emergencyUrl: qrData.emergencyUrl,
      printableVersion: qrData.printableUrl
    };

  } catch (error) {
    console.error('QR generation failed:', error.response?.data || error.message);
    throw error;
  }
}

// Usage
const qrCode = await generateAllergyPassportQR('customer-profile-uuid', {
  authToken: 'user-jwt-token',
  expirationDays: 365,
  notifyOnScan: true,
  securityLevel: 'MULTI_TIER'
});

console.log('QR Code ready for download:', qrCode.qrCodeImage);
console.log('Emergency access URL:', qrCode.emergencyUrl);

Code Example: QR Code Scanning

// Python - QR Code Scanner Implementation
import requests
import json
from datetime import datetime, timedelta

class QRCodeScanner:
    def __init__(self, scanner_id, scanner_type, api_key):
        self.scanner_id = scanner_id
        self.scanner_type = scanner_type  # RESTAURANT, HEALTHCARE, EMERGENCY
        self.api_key = api_key
        self.base_url = 'https://api.wia-foodallergy.org'
        self.active_sessions = {}

    def scan_qr_code(self, qr_code_data, location=None):
        """Scan QR code and retrieve allergy profile"""
        try:
            # Parse QR code data
            qr_payload = json.loads(qr_code_data)
            qr_code_id = qr_payload.get('qrCodeId')

            # Call scan endpoint
            response = requests.get(
                f'{self.base_url}/api/v1/qr/{qr_code_id}',
                headers={
                    'Authorization': f'Bearer {self.api_key}',
                    'X-Scanner-ID': self.scanner_id,
                    'X-Scanner-Type': self.scanner_type,
                    'Content-Type': 'application/json'
                },
                json={
                    'location': location,
                    'scanTimestamp': datetime.utcnow().isoformat()
                }
            )
            response.raise_for_status()

            scan_result = response.json()

            # Store session token for continued access
            self.active_sessions[qr_code_id] = {
                'sessionToken': scan_result['accessGrant']['sessionToken'],
                'expiresAt': scan_result['accessGrant']['sessionExpiry'],
                'profileId': scan_result['profileId']
            }

            return scan_result

        except Exception as e:
            print(f'QR scan failed: {e}')
            raise

    def get_full_profile(self, qr_code_id):
        """Retrieve full allergy profile using active session"""
        if qr_code_id not in self.active_sessions:
            raise ValueError('No active session for this QR code')

        session = self.active_sessions[qr_code_id]

        # Check if session expired
        if datetime.fromisoformat(session['expiresAt'].replace('Z', '+00:00')) < datetime.utcnow():
            # Attempt to refresh session
            self.refresh_session(qr_code_id)

        response = requests.get(
            f'{self.base_url}/api/v1/qr/{qr_code_id}/profile',
            headers={
                'Authorization': f'Bearer {session["sessionToken"]}',
                'X-Scanner-ID': self.scanner_id
            }
        )
        response.raise_for_status()
        return response.json()

    def refresh_session(self, qr_code_id):
        """Refresh expired session token"""
        session = self.active_sessions.get(qr_code_id)
        if not session:
            raise ValueError('No session to refresh')

        response = requests.post(
            f'{self.base_url}/api/v1/qr/{qr_code_id}/refresh',
            headers={
                'Authorization': f'Bearer {session["sessionToken"]}',
                'X-Scanner-ID': self.scanner_id
            }
        )
        response.raise_for_status()

        new_token_data = response.json()
        self.active_sessions[qr_code_id].update({
            'sessionToken': new_token_data['sessionToken'],
            'expiresAt': new_token_data['expiresAt']
        })

    def display_emergency_info(self, scan_result):
        """Display emergency information for first responders"""
        emergency = scan_result['emergencyData']

        print("\n" + "="*60)
        print("FOOD ALLERGY EMERGENCY INFORMATION")
        print("="*60)
        print(f"Patient: {emergency['patientName']}")
        print(f"Age: {emergency['age']}  |  Blood Type: {emergency['bloodType']}")
        print(f"\nCRITICAL ALLERGIES:")
        for allergy in emergency['criticalAllergies']:
            print(f"  ⚠️  {allergy}")
        print(f"\nEPIPEN: {emergency['epiPenLocation']}")
        print(f"Expires: {emergency['epiPenExpiration']}")
        print(f"\n⚡ EMERGENCY CONTACT: {emergency['emergencyContact']}")
        print(f"\n📋 INSTRUCTIONS:")
        print(f"   {emergency['instructions']}")
        print("="*60 + "\n")

# Usage
scanner = QRCodeScanner(
    scanner_id='restaurant-uuid-here',
    scanner_type='RESTAURANT',
    api_key='restaurant-api-key'
)

# Scan QR code
qr_data = '{"qrCodeId":"uuid-here","version":"1.0.0",...}'
result = scanner.scan_qr_code(qr_data, location={
    'latitude': 47.6062,
    'longitude': -122.3321
})

# Display emergency info
scanner.display_emergency_info(result)

# Get full profile for menu analysis
full_profile = scanner.get_full_profile(result['qrCodeId'])
print(f"Total allergies: {full_profile['totalAllergies']}")

3.4 Certification Requirements for QR Code Systems

CERT-QR-001: QR codes must use error correction level H (30% redundancy) to ensure scannability even if damaged.
CERT-QR-002: Emergency data must be embedded directly in QR code for offline access without internet connectivity.
CERT-QR-003: QR code data must be encrypted using AES-256-GCM with authenticated encryption.
CERT-QR-004: Session tokens must expire after maximum 6 hours or when dining session ends.
CERT-QR-005: QR code must include tamper detection mechanism to prevent data modification.
CERT-QR-006: System must log all QR code scans with timestamp, location, and scanner identification.
CERT-QR-007: Profile owner must receive notification when QR code is scanned (configurable).
CERT-QR-008: QR code must be regenerable without changing underlying profile data.
CERT-QR-009: Emergency access layer must be accessible without authentication for first responders.
CERT-QR-010: QR code image must be minimum 512x512 pixels for reliable scanning.
CERT-QR-011: System must support multiple active QR codes per profile for different contexts (wallet card, phone, wristband).
CERT-QR-012: QR code must include version number for backward compatibility with future schema changes.
CERT-QR-013: Revoked QR codes must return clear error message when scanned.
CERT-QR-014: QR code generation must complete within 3 seconds for optimal user experience.
CERT-QR-015: System must support batch QR code generation for families (minimum 10 profiles).
CERT-QR-016: Printable QR code must include human-readable emergency information alongside code.
CERT-QR-017: QR code must work on both iOS and Android native camera apps without additional software.
CERT-QR-018: Session refresh must be automatic and transparent during active dining sessions.
CERT-QR-019: QR code must support geofencing to restrict scans to authorized locations (optional feature).
CERT-QR-020: System must maintain QR code scan history for minimum 90 days for security auditing.
CERT-QR-021: Emergency access must display photo verification to confirm patient identity.
CERT-QR-022: QR code must degrade gracefully if API is unavailable, showing cached emergency data.
CERT-QR-023: Multi-language support required for emergency data display (minimum 12 languages).
CERT-QR-024: QR code must include checksum validation to detect scanning errors or corruption.
CERT-QR-025: System must support custom QR code branding while maintaining scanning compatibility.
