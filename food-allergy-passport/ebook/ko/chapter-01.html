1.1 디지털 알레르기 프로필 소개

디지털 알레르기 프로필은 WIA 식품 알레르기 여권 표준의 초석을 나타냅니다. 이러한 포괄적인 디지털 기록에는 개인의 식품 알레르기, 불내성 및 식이 제한에 대한 자세한 정보가 포함되어 있으며 알레르기가 있는 개인, 식품 서비스 제공업체, 의료 전문가 및 응급 구조대 간의 의사소통을 위한 표준화된 형식을 제공합니다.

알레르기 심각도 분류

심각도 수준 코드 설명 필요한 대응
생명을 위협하는 SEVERITY_1 아나필락시스 병력, 즉시 에피펜 필요 완전한 알레르겐 제거, 비상 준비
심각 SEVERITY_2 전신 반응, 아나필락시스로 진행될 수 있음 엄격한 회피, 항히스타민제 사용 가능
중간 SEVERITY_3 국소 반응, 불편하지만 생명을 위협하지 않음 회피 권장, 필요에 따라 약물
경미 SEVERITY_4 경미한 증상, 구강 알레르기 증후군 주의 권고, 환자가 조리된 형태는 견딜 수 있음
불내성 SEVERITY_5 비-IgE 매개, 소화 문제 식이 관리, 응급 위험 없음

의료 검증 요구 사항

검증 유형 필요 대상 유효 기간 문서
알레르기 전문의 확인 모든 SEVERITY_1 및 SEVERITY_2 알레르기 12개월 면허 번호가 있는 서명된 의료 서신
피부 단자 검사 일차 알레르겐 진단 24개월 팽진 크기 측정이 있는 실험실 결과
혈액 IgE 검사 확인 검사 24개월 특정 IgE 수준 (kU/L)
경구 식품 챌린지 내성 검증, 알레르기 해소 12개월 감독된 챌린지 프로토콜 결과
성분 검사 교차 반응성 평가 24개월 분자 알레르겐 성분 결과
에피펜 처방 모든 아나필락시스 위험 환자 12개월 복용량 지침이 있는 활성 처방전

1.3 프로필 관리를 위한 API 엔드포인트

POST /api/v1/profiles

새로운 알레르기 프로필을 생성합니다. 사용자 인증 및 초기 의료 검증이 필요합니다.

GET /api/v1/profiles/{profileId}

ID로 완전한 알레르기 프로필을 검색합니다. 모든 알레르기 및 의료 기록을 포함한 전체 프로필 데이터를 반환합니다.

PUT /api/v1/profiles/{profileId}

기존 알레르기 프로필을 업데이트합니다. 충돌하는 업데이트를 방지하기 위해 버전 제어가 적용됩니다.

DELETE /api/v1/profiles/{profileId}

알레르기 프로필을 소프트 삭제합니다. 복구를 위해 90일 동안 데이터가 보관된 후 영구적으로 삭제됩니다.

POST /api/v1/profiles/{profileId}/allergies

기존 프로필에 새 알레르겐을 추가합니다. SEVERITY_1 및 SEVERITY_2에 대한 의료 검증이 필요합니다.

PUT /api/v1/profiles/{profileId}/allergies/{allergenId}

심각도, 검사 결과 또는 반응 이력을 포함한 알레르겐 세부 정보를 업데이트합니다.

DELETE /api/v1/profiles/{profileId}/allergies/{allergenId}

프로필에서 알레르겐을 제거합니다. 경구 식품 챌린지를 통한 알레르기 전문의 확인이 필요합니다.

POST /api/v1/profiles/{profileId}/verify

의료 검증 문서를 제출합니다. 의사 서신, 검사 결과 및 처방전 이미지를 허용합니다.

GET /api/v1/profiles/{profileId}/verification-status

모든 알레르기에 대한 검증 상태를 확인합니다. 만료 날짜 및 갱신 요구 사항을 반환합니다.

POST /api/v1/profiles/{profileId}/reactions

새로운 알레르기 반응 사고를 기록합니다. 증상, 심각도, 치료 및 위치 데이터를 캡처합니다.

GET /api/v1/profiles/{profileId}/cross-reactivities

프로필의 모든 알레르기에 대한 포괄적인 교차 반응성 분석을 가져옵니다.

POST /api/v1/profiles/{profileId}/emergency-access

응급 구조대를 위한 시간 제한 비상 액세스 토큰을 생성합니다.

코드 예제: 알레르기 프로필 생성

// JavaScript/Node.js 예제
const axios = require('axios');

async function createAllergyProfile(userData) {
  try {
    const response = await axios.post('https://api.wia-foodallergy.org/api/v1/profiles', {
      personalInfo: {
        firstName: userData.firstName,
        lastName: userData.lastName,
        dateOfBirth: userData.dateOfBirth,
        emergencyContact: {
          name: userData.emergencyContactName,
          phone: userData.emergencyContactPhone,
          relationship: userData.emergencyContactRelationship
        }
      },
      allergies: userData.allergies.map(allergy => ({
        allergenId: allergy.allergenId,
        severityLevel: allergy.severityLevel,
        diagnosisDate: allergy.diagnosisDate,
        verifiedBy: {
          physicianName: allergy.physicianName,
          licenseNumber: allergy.licenseNumber,
          specialty: '알레르기/면역학'
        }
      })),
      safetyPreferences: {
        crossContaminationTolerance: 'ZERO_TOLERANCE',
        sharedEquipmentAcceptable: false,
        requiresDedicatedPreparationArea: true
      },
      emergencyAccessEnabled: true
    }, {
      headers: {
        'Authorization': `Bearer ${userData.authToken}`,
        'Content-Type': 'application/json',
        'X-API-Version': '1.0.0'
      }
    });

    console.log('프로필이 성공적으로 생성되었습니다:', response.data.profileId);
    return response.data;
  } catch (error) {
    console.error('프로필 생성 오류:', error.response.data);
    throw error;
  }
}

// 사용법

1.4 프로필 관리를 위한 인증 요구 사항

CERT-PROFILE-001: 시스템은 최소 필드로 포괄적인 알레르기 프로필 생성을 지원해야 합니다: 사용자 식별, 알레르겐 목록, 심각도 수준, 비상 연락처 및 의료 검증.
CERT-PROFILE-002: 모든 SEVERITY_1(생명을 위협하는) 알레르기는 면허가 있는 알레르기 전문의 또는 면역학자의 의료 검증이 필요합니다.
CERT-PROFILE-003: 프로필 데이터는 AES-256 암호화를 사용하여 저장 시 암호화되고 TLS 1.3 이상을 사용하여 전송 중 암호화되어야 합니다.
CERT-PROFILE-004: 시스템은 완전한 감사 추적과 함께 모든 프로필 업데이트에 대한 버전 제어를 구현해야 합니다.
CERT-PROFILE-005: 비상 액세스 모드는 응급 구조대가 인증 없이 중요한 알레르기 정보를 볼 수 있도록 해야 합니다.
CERT-PROFILE-006: 교차 반응성 데이터베이스에는 최소 500개의 문서화된 알레르겐 관계가 포함되어야 합니다.
CERT-PROFILE-007: 프로필 사진은 3KB 미만의 QR 코드 생성을 보장하기 위해 압축 및 최적화되어야 합니다.
CERT-PROFILE-008: 의료 검증 문서는 HIPAA 규정 준수 액세스 제어와 함께 안전하게 저장되어야 합니다.
CERT-PROFILE-009: 시스템은 최소 12개 언어에 대한 다국어 프로필 표시를 지원해야 합니다.
CERT-PROFILE-010: 알레르기 심각도 수준은 표준화된 SEVERITY_1부터 SEVERITY_5 분류를 따라야 합니다.
CERT-PROFILE-011: 프로필에는 모든 시간에 민감한 의료 검증에 대한 만료 추적이 포함되어야 합니다.
CERT-PROFILE-012: 시스템은 의료 검증 만료 30일 전에 자동 알림을 보내야 합니다.
CERT-PROFILE-013: 반응 이력 로깅은 최소 데이터를 캡처해야 합니다: 날짜, 증상, 치료, 위치, 심각도.
CERT-PROFILE-014: 에피펜 및 비상 약물 추적에는 만료 알림이 포함되어야 합니다.
CERT-PROFILE-015: 프로필 백업 및 내보내기 기능은 표준화된 JSON 형식으로 사용할 수 있어야 합니다.
CERT-PROFILE-016: 시스템은 여러 가족 구성원 프로필을 연결하는 가구 프로필을 지원해야 합니다.
CERT-PROFILE-017: 미성년 자녀의 프로필에 대해 보호자 승인 워크플로와 함께 부모 제어를 구현해야 합니다.
CERT-PROFILE-018: 프로필 공유 권한은 세분화된 제어를 허용해야 합니다(전체 액세스, 비상만, 알레르겐 목록만).
CERT-PROFILE-019: API 속도 제한은 남용을 방지하면서 합법적인 고빈도 액세스 패턴을 허용해야 합니다.
CERT-PROFILE-020: 프로필 삭제는 영구 삭제 전에 복구 옵션과 함께 90일 소프트 삭제를 구현해야 합니다.
CERT-PROFILE-021: 시스템은 보안 감사 목적으로 모든 프로필 액세스 시도를 기록해야 합니다.
CERT-PROFILE-022: 모바일 장치의 암호화된 로컬 저장소를 통해 오프라인 프로필 액세스를 지원해야 합니다.
CERT-PROFILE-023: 여러 장치 간의 프로필 동기화는 사용자 알림과 함께 최근 기록 우선을 사용하여 충돌을 해결해야 합니다.
CERT-PROFILE-024: 약물 복용량 계산은 환자의 체중, 나이 및 심각도 수준을 고려해야 합니다.
CERT-PROFILE-025: 시스템은 의료 기록 가져오기를 위해 FHIR 표준을 통해 주요 EHR 시스템과 통합되어야 합니다.
