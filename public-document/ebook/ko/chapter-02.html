2.3 메타데이터 표준

메타데이터는 문서의 속성과 맥락을 설명하는 데이터입니다. WIA-SOCIAL은 Dublin Core, ISO 15836, PREMIS 등 국제 메타데이터 표준을 준수하며, 공공문서 특화 메타데이터를 추가합니다.

2.4 생체 인식 데이터 인코딩

현대 신원 문서는 인증을 위해 생체 인식 데이터를 점점 더 많이 통합하고 있습니다. 표준화된 생체 인식 인코딩은 캡처 장치, 저장 시스템 및 검증 시스템 간의 상호 운용성을 보장합니다.

ISO/IEC 19794 생체 인식 데이터 형식

ISO/IEC 19794는 다양한 생체 인식 양식에 대한 표준 형식을 정의합니다. Part 2는 지문 특이점 데이터(지문의 고유 지점)를 다룹니다. Part 4는 지문 이미지 데이터를 지정합니다. Part 5는 자세, 조명 및 해상도 요구 사항을 포함하여 얼굴 이미지 데이터를 정의합니다. Part 6은 홍채 이미지 데이터를 다룹니다. 이러한 표준은 생체 인식 데이터가 다른 공급업체의 시스템 간에 공유될 수 있도록 보장합니다.

생체 인식 템플릿 보호

원시 생체 인식 데이터는 개인정보 보호 위험을 초래합니다. 지문과 홍채 스캔은 손상되면 변경할 수 없습니다. 템플릿 보호 기술은 생체 인식 데이터의 단방향 변환을 생성하여 매칭에는 적합하지만 재구성은 불가능하게 만듭니다. Fuzzy Extractor는 생체 인식 데이터에서 암호화 키를 생성하여 생체 인식 템플릿을 저장하지 않고도 인증을 가능하게 합니다. 이러한 기술은 개인정보 보호 규정에서 점점 더 요구되고 있습니다.

생체 인식 유형 표준 형식 일반적 크기 매칭 정확도
얼굴 사진 ISO 19794-5, JPEG 2000 15-45 KB FAR 0.1%, FRR 1-3%
지문 특이점 ISO 19794-2, ANSI 378 200-600 bytes FAR 0.01%, FRR 0.5-2%
지문 이미지 ISO 19794-4, WSQ 10-30 KB FAR 0.001%, FRR 0.5%
홍채 이미지 ISO 19794-6 2-10 KB FAR 0.0001%, FRR 0.1%
음성 템플릿 ISO 19794-13 1-5 KB FAR 2%, FRR 5-10%

2.5 기계 판독 영역 및 시각적 인코딩

많은 공공문서는 디지털 정보를 물리적 형태로 인코딩하여 검증 장치가 빠르게 읽을 수 있도록 합니다. 이러한 기계 판독 영역은 디지털 시스템과 물리적 문서 간의 연결을 제공합니다.

QR 코드 및 2D 바코드

QR 코드는 최대 4,296자의 영숫자 데이터를 저장할 수 있어 문서 ID, 검증 URL 또는 암호화 해시를 인코딩하는 데 적합합니다. 오류 수정 기능은 부분적으로 손상된 코드도 읽을 수 있게 합니다. Data Matrix 코드는 작은 공간에 더 적합합니다. PDF417은 더 많은 데이터를 인코딩할 수 있지만 더 큰 공간이 필요합니다.

NFC/RFID 칩

전자 여권과 신원 카드에는 생체 인식 데이터와 문서 정보를 저장하는 NFC 칩이 포함됩니다. ICAO 9303 표준은 e-Passport의 논리적 데이터 구조를 정의합니다. 칩은 무선 독취기와 통신하여 국경 통제 시 빠른 검증을 가능하게 합니다. Basic Access Control (BAC) 및 Extended Access Control (EAC)은 무단 읽기를 방지합니다.

기계 판독 영역 (MRZ)

여권과 비자의 하단에 있는 MRZ는 OCR 기계가 읽을 수 있는 표준화된 형식으로 핵심 정보를 인코딩합니다. TD1(신용카드 크기), TD2, TD3(여권 크기) 형식은 이름, 국적, 문서 번호, 생년월일 및 체크 디지트를 포함합니다. 이러한 형식은 수십 년 동안 사용되어 왔으며 전 세계적으로 표준화되어 있습니다.

2.6 PDF/A 및 장기 보관 형식

공공문서는 수십 년, 때로는 영구적으로 보관되어야 합니다. PDF/A (ISO 19005) 표준은 장기 보관을 위한 전자 문서 형식으로, 문서의 시각적 외관이 시간이 지나도 동일하게 유지되도록 보장합니다.

📄 PDF/A 규격 비교

  • PDF/A-1: 기본 수준, 외부 참조 금지, 투명도 미지원 (출생증명서, 단순 문서)
  • PDF/A-2: JPEG 2000 지원, 디지털 서명 강화, 투명도 지원 (여권, 사진 포함 문서)
  • PDF/A-3: 임의 파일 형식 첨부 가능, XML 메타데이터 (복합 문서, 증거 자료 첨부)
  • PDF/A-4: PDF 2.0 기반, 개선된 보안 및 접근성 (최신 표준, 2020년 발표)

PDF/A 생성 코드 예제 (Python)

from reportlab.pdfgen import canvas
from reportlab.lib.pagesizes import A4
from pypdf import PdfWriter, PdfReader
import datetime

def create_pdfa_birth_certificate(data, output_path):
    """출생증명서를 PDF/A-2b 형식으로 생성"""

    # 1. PDF 생성
    c = canvas.Canvas(output_path, pagesize=A4)
    width, height = A4

    # 2. 메타데이터 설정
    c.setTitle(f"출생증명서 - {data['fullName']}")
    c.setAuthor("Seoul City Government")
    c.setSubject("Official Birth Certificate")
    c.setCreator("WIA-SOCIAL Document System")
    c.setProducer("WIA PDF/A Generator v1.0")

    # 3. 내용 작성
    c.setFont("Helvetica-Bold", 24)
    c.drawCentredString(width/2, height-100, "출생증명서")
    c.drawCentredString(width/2, height-130, "Birth Certificate")

    c.setFont("Helvetica", 12)
    y = height - 200

    fields = [
        ("성명 / Name:", data['fullName']),
        ("생년월일 / Date of Birth:", data['dateOfBirth']),
        ("출생지 / Place of Birth:", data['placeOfBirth']),
        ("성별 / Gender:", data['gender']),
        ("부 / Father:", data['father']),
        ("모 / Mother:", data['mother']),
        ("등록번호 / Registration No:", data['registrationNumber']),
        ("발급일 / Issue Date:", data['issueDate'])
    ]

    for label, value in fields:
        c.drawString(100, y, f"{label} {value}")
        y -= 30

    # 4. QR 코드 추가 (검증용)
    qr_data = f"WIA:{data['documentId']}"
    # QR 코드 생성 로직...

    # 5. 디지털 서명 영역
    c.drawString(100, 100, "디지털 서명: [Digitally Signed]")
    c.drawString(100, 80, f"발급 기관: {data['issuingAuthority']}")

    c.save()

    # 6. PDF/A-2b로 변환
    convert_to_pdfa(output_path)

    return output_path

def convert_to_pdfa(pdf_path):
    """PDF를 PDF/A-2b로 변환"""
    reader = PdfReader(pdf_path)
    writer = PdfWriter()

    # 모든 페이지 복사
    for page in reader.pages:
        writer.add_page(page)

    # XMP 메타데이터 추가 (PDF/A 준수)
    xmp_metadata = f"""

  
    
      2
      B
    
  

"""

    writer.add_metadata({
        "/Type": "/Metadata",
        "/Subtype": "/XML",
        "/Length": len(xmp_metadata)
    })

    # 저장
    with open(pdf_path, "wb") as output_file:
        writer.write(output_file)

2.8 데이터 검증 및 무결성

문서 데이터의 정확성과 무결성을 보장하기 위해 다층 검증 시스템이 필요합니다. WIA-SOCIAL은 스키마 검증, 비즈니스 규칙 검증, 암호학적 검증을 결합합니다.

검증 단계 검증 내용 사용 기술
1. 구문 검증 JSON/XML 구문의 유효성 JSON Schema, XSD
2. 스키마 검증 필수 필드, 데이터 타입, 형식 Ajv, Joi, Yup
3. 비즈니스 규칙 논리적 일관성, 도메인 규칙 Custom Validators
4. 참조 무결성 외래 키, 관계형 제약 Database Constraints
5. 암호학적 검증 디지털 서명, 체크섬 RSA, ECDSA, SHA-256

요약

공공문서의 데이터 형식과 표준은 디지털 문서 시스템의 핵심 기반입니다. JSON, XML, PDF/A 등 다양한 형식을 지원하고, 국제 표준을 준수하며, 엄격한 검증 체계를 적용함으로써 문서의 신뢰성과 상호운용성을 보장할 수 있습니다. 다음 장에서는 이러한 데이터 형식을 다루는 API 인터페이스를 살펴보겠습니다.
