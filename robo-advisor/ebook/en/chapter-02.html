Chapter 2: Investment Algorithms and Portfolio Theory

2.1 Foundations of Modern Portfolio Theory

Modern Portfolio Theory (MPT), developed by Harry Markowitz in 1952, forms the mathematical foundation of robo-advisor algorithms. MPT introduced the revolutionary concept that risk and return should be evaluated for the entire portfolio rather than individual securities.

Core Principles of MPT

  • Diversification: Combining assets with low correlation reduces overall portfolio risk
  • Risk-Return Tradeoff: Higher expected returns require accepting higher risk
  • Efficient Frontier: The set of optimal portfolios offering maximum return for a given risk level
  • Market Efficiency: Prices reflect all available information, making active management difficult

2.2 Mathematical Foundations

2.2.1 Expected Return

The expected return of a portfolio is the weighted average of individual asset returns:

E(Rp) = Σ wi × E(Ri)

Where wi is the weight of asset i and E(Ri) is its expected return.

2.2.2 Portfolio Variance

Portfolio risk (variance) accounts for both individual asset volatility and correlations:

σ²p = Σ Σ wi wj Cov(Ri, Rj)

2.2.3 Sharpe Ratio

The Sharpe ratio measures risk-adjusted returns:

Sharpe = (Rp - Rf) / σp

Where Rf is the risk-free rate and σp is portfolio standard deviation.

2.3 Asset Allocation Algorithms

2.3.1 Strategic Asset Allocation

Strategic allocation establishes long-term target weights based on risk tolerance, time horizon, and goals. Common approaches include:

  • Age-Based Rules: (120 - age)% in stocks, remainder in bonds
  • Goal-Based Allocation: Customized portfolios aligned with specific financial objectives
  • Risk Parity: Equalizing risk contribution across asset classes
  • Black-Litterman Model: Combining market equilibrium with investor views

2.3.2 Tactical Asset Allocation

Some robo-advisors employ tactical adjustments based on market conditions, economic indicators, or valuation metrics. However, pure index-based robo-advisors typically avoid market timing.

2.4 Optimization Techniques

2.4.1 Mean-Variance Optimization

The classic Markowitz optimization seeks to maximize return for a given risk level or minimize risk for a target return: 

minimize: w' Σ w
subject to:
    - w' μ ≥ r_target
    - w' 1 = 1
    - w ≥ 0 (no short selling)

2.4.2 Black-Litterman Optimization

Addresses estimation error in expected returns by starting with market equilibrium and incorporating investor views with confidence levels.

2.4.3 Robust Optimization

Accounts for parameter uncertainty by optimizing worst-case scenarios within confidence intervals, producing more stable portfolios.

2.5 Rebalancing Algorithms

2.5.1 Threshold-Based Rebalancing

Rebalance when any asset class drifts beyond ±X% from target allocation: 

if abs(current_weight - target_weight) > threshold:
    rebalance_to_target()

2.5.2 Calendar Rebalancing

Fixed schedule rebalancing (monthly, quarterly, annually) regardless of drift:

  • Pros: Predictable, simple to implement
  • Cons: May rebalance too frequently or not frequently enough

2.5.3 Hybrid Approaches

Combine threshold and calendar: check allocations quarterly but only rebalance if drift exceeds threshold.

2.6 Tax-Loss Harvesting Algorithms

Tax-loss harvesting (TLH) systematically sells securities at a loss to offset capital gains:

TLH Implementation Logic

for each security in portfolio:
    if current_price < cost_basis:
        loss = cost_basis - current_price
        if loss > min_threshold:
            # Sell losing position
            sell_security(security, quantity)

            # Buy similar security (avoid wash sale)
            replacement = find_correlated_security(security)
            buy_security(replacement, quantity)

            # Track harvested loss for tax reporting
            record_loss(loss, tax_year)

Wash Sale Rules

IRS wash sale rule prohibits claiming losses if substantially identical securities are purchased within 30 days before or after the sale. Robo-advisors use ETF pairs to maintain exposure while avoiding wash sales:

  • VTI (Total Market) ↔ SCHB (Total Market alternative)
  • VEA (Developed Markets) ↔ IEFA (Developed Markets alternative)
  • AGG (Aggregate Bonds) ↔ BND (Total Bond alternative)

2.7 Risk Parity Strategies

Risk parity allocates capital such that each asset class contributes equally to portfolio risk:

Risk Contributioni = wi × (∂σp / ∂wi)

All Weather Portfolio

Ray Dalio's All Weather portfolio uses risk parity principles:

  • 30% Stocks (equities)
  • 40% Long-term bonds
  • 15% Intermediate bonds
  • 7.5% Gold
  • 7.5% Commodities

2.8 Factor-Based Investing

Advanced robo-advisors incorporate factor tilts:

  • Value: Stocks with low price-to-book ratios
  • Size: Small-cap premium
  • Momentum: Recent outperformers
  • Quality: High profitability, low debt
  • Low Volatility: Defensive stocks with lower beta

2.9 Algorithm Performance Metrics

Key Performance Indicators

  • Sharpe Ratio: Risk-adjusted returns
  • Sortino Ratio: Downside deviation focus
  • Maximum Drawdown: Peak-to-trough decline
  • Alpha: Excess return vs. benchmark
  • Beta: Systematic risk exposure
  • Information Ratio: Active return per unit of tracking error

2.10 Practical Implementation Example

class RoboAdvisorAlgorithm:
    def __init__(self, risk_score):
        self.risk_score = risk_score  # 0-100
        self.stock_allocation = self.calculate_stock_allocation()
        self.bond_allocation = 100 - self.stock_allocation

    def calculate_stock_allocation(self):
        """Risk score determines equity allocation"""
        return min(90, max(10, self.risk_score))

    def build_portfolio(self, amount):
        """Construct portfolio based on allocation"""
        portfolio = {
            'stocks': {
                'VTI': amount * self.stock_allocation * 0.4 / 100,  # 40% US
                'VEA': amount * self.stock_allocation * 0.3 / 100,  # 30% Intl
                'VWO': amount * self.stock_allocation * 0.3 / 100   # 30% EM
            },
            'bonds': {
                'BND': amount * self.bond_allocation * 0.7 / 100,   # 70% Total
                'BNDX': amount * self.bond_allocation * 0.3 / 100   # 30% Intl
            }
        }
        return portfolio

    def check_rebalance(self, current_portfolio, threshold=5):
        """Determine if rebalancing is needed"""
        current_stock_pct = self.calculate_current_stock_pct(current_portfolio)
        drift = abs(current_stock_pct - self.stock_allocation)
        return drift > threshold
