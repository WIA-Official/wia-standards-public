제3장 · WIA Robo-Advisor v1.0.0

API 표면

2단계 API 표면은 발행·조회·스트리밍·연합 핸드셰이크의 4가지 기본 동작을 노출합니다. 모든 요청은 RFC 9421 HTTP 메시지 서명을 운반하며, 오류 응답은 RFC 9457 Problem Details를 사용합니다.

3.1 엔드포인트 표면

POST /ra/v1/investors                    투자자 프로파일 생성
GET  /ra/v1/investors/{id}               최신 서명 프로파일
POST /ra/v1/portfolios                   새 포트폴리오 바인딩
GET  /ra/v1/portfolios/{id}              최신 포트폴리오 스냅샷
POST /ra/v1/rebalance                    재배분 이벤트 발행
POST /ra/v1/tlh                          세금 손실 수확 이벤트
GET  /ra/v1/stream/{investor_id}         SSE: 라이브 봉투 스트림

3.2 인증과 서명

모든 요청은 (request-target)·@authority·content-digest·@created·@expires 필수 커버 컴포넌트 위에 HTTP 메시지 서명을 운반해야 합니다. 베어러 토큰만 사용한 인증은 거부됩니다.

3.3 멱등성

POST 엔드포인트는 Idempotency-Key 헤더(UUIDv7 권장)를 수용합니다. 24시간 내 동일 키 재요청은 자원 재생성 없이 원본 응답을 반환합니다.

3.4 스트리밍 의미론

/stream/{investor_id} 엔드포인트는 Server-Sent Events를 발행합니다. 각 이벤트는 완전한 서명 봉투(부분 프레임 없음)입니다. 서버는 연결 시 마지막 60초 이벤트를 재생해 클라이언트 재연결을 누락 없이 흡수하며, 클라이언트는 봉투 식별자로 중복 제거를 책임집니다.

3.5 능력 발견

GET /.well-known/wia-robo-advisor는 서명된 능력 광고 봉투를 반환합니다 — 엔드포인트 표면·지원 봉투 유형·세금 처리 지원 관할·활성 알고리즘 증명. 사전 통합 없이도 소비자가 한 문서를 읽고 시작할 수 있게 만드는 핵심 메커니즘입니다.

3.6 한국 적용 노트

마이데이터 사업자 응답 SLA(요청 후 60초 내 응답)는 본 표준의 폴-기반 조회와 정확히 일치합니다. 추가로 SSE 스트림으로 푸시 알림을 보내면 마이데이터 신선도 SLA를 자동 만족합니다 — 별도 신선도 보장 인프라 불필요.

3.7 속도 제한

기본 제한은 어드바이저 클라이언트 신원당 분당 600 요청, 초당 100 버스트입니다. 한도 초과 응답은 HTTP 429와 Retry-After 초입니다. 어그리게이터의 적응형 백오프가 이 헤더를 존중해야 합니다.

워크 예제

투자자 대시보드 클라이언트가 SSE 스트림을 구독해 실시간 포트폴리오 갱신을 받습니다. 분기 보고서 생성 시 다른 클라이언트가 같은 봉투 스트림을 NDJSON 벌크 내보내기로 받아 PDF로 렌더링합니다. 두 클라이언트 모두 같은 서명을 검증하므로 동일 진실을 봅니다 — 한 쪽이 다른 쪽보다 신선하지도 정확하지도 않습니다.

편집자 노트

본 한국어 챕터는 영문판의 직역이 아니라 한국 자본시장 환경(자본시장법·금융소비자보호법·마이데이터·KR-GIPS)을 염두에 둔 응용 노트와 함께 재작성됐습니다. 직역이 필요한 경우 영문판을 우선 인용해 주십시오. 봉투 서명·검증 코드 예제와 한국 마이데이터 사업자 연결 예제는 별도 부속서에 수록될 예정입니다.

다음 단계

다음 장에서 다음 봉투 클래스 또는 운영 측면을 다룹니다. 8개 장 전체는 약 2시간이면 읽을 수 있도록 의도됐습니다 — 일하는 어드바이저 운영자가 분기 보고 사이에 책을 끝낼 수 있을 만큼 짧습니다. 각 장의 워크 예제는 실제 한국 중규모 로보어드바이저(운용자산 1조원 미만, 투자자 5만명 이하)를 가정합니다.
