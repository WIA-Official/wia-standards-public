제7장 · WIA Robo-Advisor v1.0.0

알고리즘 증명과 GIPS

로보어드바이저의 가장 큰 약속은 알고리즘이 사람보다 더 결정적·투명하게 작동한다는 것입니다. 약속이 검증 가능하려면 알고리즘 자체가 봉투에 서명되어 사후 검증이 가능해야 합니다.

7.1 알고리즘 증명 봉투

{
  "wia_robo_advisor_version": "1.0.0",
  "type": "algorithm_attestation",
  "algorithm_id": "ra-glide-2026q1-v3",
  "algorithm_kind": "glide-path",
  "rebalance_policy": "drift-band",
  "drift_threshold_pct": 5,
  "version": "3.1.2",
  "training_data_window": {
    "from": "1990-01-01T00:00:00Z",
    "to":   "2025-12-31T23:59:59Z"
  },
  "validation_metrics": {
    "out_of_sample_sharpe_estimate": 0.62,
    "max_drawdown_in_sample_pct": 28
  },
  "signature": { "alg": "Ed25519", "value": "..." }
}

7.2 알고리즘 종류

표준은 6가지 알고리즘 종류를 인정합니다: mean-variance(Markowitz), risk-parity(Bridgewater 스타일), black-litterman(베이지안 결합), factor-tilt(Fama-French 5요인 등), glide-path(나이/지평선 기반), ml-derived(ML 학습). 종류는 봉투에 명시되어 후속 성과 비교 시 동일 종류 동일 비교됩니다.

7.3 라이브 알고리즘 변경 추적

알고리즘이 변경될 때마다 — 버전·종류·학습 윈도우·정책 매개변수 — 새 algorithm_attestation이 발행됩니다. 투자자의 포트폴리오 이벤트는 활성 시점의 algorithm_id를 참조합니다. 알고리즘 버전 간 결과 조정이 어드바이저 측 성과 귀속의 핵심입니다 — "이 분기 부진이 모델 변경 탓인가, 시장 탓인가"에 답할 수 있게 됩니다.

7.4 GIPS 정렬

표준의 성과 보고는 GIPS 2020 규약과 정렬됩니다. 시간가중수익률(TWR), 모듬 정의, 수수료 차감 전·후 보고, 벤치마크 비교 — 모두 봉투의 결정적 필드로 인코딩됩니다. GIPS 적합성 자체는 외부 검증자가 인증하지만, 본 표준의 봉투는 검증자 작업의 입력이 됩니다.

7.5 한국 적용 노트

국내 KR-GIPS 한국기관투자자포럼 가이드는 GIPS 2020을 거의 그대로 준수합니다. 본 표준의 봉투는 KR-GIPS 보고에도 직접 사용 가능합니다 — 별도 변환 불필요. 금감원 영업보고서의 운용 성과 항목 역시 같은 봉투 스트림에서 자동 산출됩니다.

7.6 ML-파생 알고리즘의 특수 의무

ML-파생 알고리즘은 추가 의무를 집니다 — 학습 윈도우 명시·아웃-오브-샘플 샤프 추정·인-샘플 최대 손실폭. 단지 "신경망"이라는 라벨로는 부족합니다. 사후 분쟁 시 모델의 학습 분포가 운영 분포와 부합했는가가 첫 질문이며, training_data_window가 그 질문의 출발점입니다.

워크 예제

2026-Q2, 어드바이저가 글라이드 패스 v3.1.2를 v3.2.0으로 업그레이드 — 50대 앵커를 65/35에서 60/40으로 보수화. 새 algorithm_attestation 발행. 영향 받는 모든 투자자 포트폴리오에 사전 알림 전송. 다음 분기 성과 귀속에서 v3.1.2 시기와 v3.2.0 시기를 분리 보고 — 모델 변경 효과가 시장 효과와 분리되어 정직하게 측정됩니다.

편집자 노트

본 한국어 챕터는 영문판의 직역이 아니라 한국 자본시장 환경(자본시장법·금융소비자보호법·마이데이터·KR-GIPS)을 염두에 둔 응용 노트와 함께 재작성됐습니다. 직역이 필요한 경우 영문판을 우선 인용해 주십시오. 봉투 서명·검증 코드 예제와 한국 마이데이터 사업자 연결 예제는 별도 부속서에 수록될 예정입니다.
