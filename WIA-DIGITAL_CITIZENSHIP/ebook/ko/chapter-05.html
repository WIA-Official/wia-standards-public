WIA-DIGITAL_CITIZENSHIP

디지털 시민권 관리를 위한 REST API

5.1 개요

이 장에서는 WIA-DIGITAL_CITIZENSHIP 표준의 REST API 인터페이스를 다룹니다. 디지털 시민권 발급, 검증, 관리를 위한 API 엔드포인트와 한국 정부 시스템과의 연동 방법을 설명합니다.

📌 학습 목표

  • REST API 엔드포인트 설계 이해
  • 디지털 시민권 발급/검증 API 구현
  • 한국 정부 시스템 연동 방법
  • 인증 및 보안 구현
  • SDK 사용법 습득

5.2 API 기본 구성

5.2.1 Base URL 및 인증

Base URL: https://api.wia.live/citizenship/v1

인증 방법:
- Bearer Token (JWT): 일반 애플리케이션
- API Key: 서버 간 통신
- OAuth 2.0: 제3자 앱
- PASS 인증: 한국 사용자 본인 인증

5.2.2 주요 API 엔드포인트

엔드포인트 Method 설명 인증
/citizenship/apply POST 디지털 시민권 신청 Required
/citizenship/{id} GET 시민권 정보 조회 Required
/citizenship/verify POST 시민권 검증 Optional
/citizenship/update PUT 시민권 정보 갱신 Required
/citizenship/revoke DELETE 시민권 취소 Required (Admin)

5.4 한국 정부 시스템 연동

디지털 시민권 발급을 위해 한국 정부의 다양한 시스템과 API 연동이 필요합니다.

5.4.1 정부 시스템 연동 현황

정부 기관 시스템 API 용도 연동 상태 일일 호출량
행정안전부 주민등록 시스템 신원 확인, 주소 검증 완전 통합 ~45,000회
법무부 출입국관리 시스템 국적 확인, 체류 자격 완전 통합 ~18,000회
경찰청 범죄경력조회 시스템 신원 조회 제한적 연동 ~12,000회
과학기술정보통신부 PASS 인증 본인 인증 완전 통합 ~85,000회
금융위원회 KYC/AML 시스템 금융 거래 검증 파일럿 ~8,000회

5.4.2 행정안전부 주민등록 연동

// 행정안전부 API 연동 예시
import { MoisAPI } from '@kr-gov/mois-api';

class CitizenshipVerification {
  private moisClient: MoisAPI;

  async verifyIdentity(applicant: Applicant): Promise {
    // 1. 주민등록번호 실명 확인
    const nameVerification = await this.moisClient.verifyRealName({
      name: applicant.full_name,
      resident_number: applicant.national_id,
      auth_key: process.env.MOIS_API_KEY
    });

    if (!nameVerification.verified) {
      throw new Error('실명 인증 실패');
    }

    // 2. 주소 검증
    const addressVerification = await this.moisClient.verifyAddress({
      resident_number: applicant.national_id,
      address: applicant.contact.address
    });

    // 3. 가족관계 확인 (선택적)
    const familyRelation = await this.moisClient.getFamilyRelation({
      resident_number: applicant.national_id
    });

    return {
      verified: true,
      verification_level: 'high',
      identity_confirmed: nameVerification.verified,
      address_confirmed: addressVerification.verified,
      government_issued_id: true,
      verification_timestamp: new Date().toISOString(),
      verifier: '행정안전부 주민등록시스템'
    };
  }
}

5.6 인증 및 보안

5.6.1 JWT 기반 인증

// JWT 토큰 발급 예시
import jwt from 'jsonwebtoken';

interface CitizenshipToken {
  sub: string; // citizenship_id
  name: string;
  nationality: string;
  rights: string[];
  iat: number;
  exp: number;
}

function generateCitizenshipToken(citizenship: Citizenship): string {
  const payload: CitizenshipToken = {
    sub: citizenship.id,
    name: citizenship.holder.name,
    nationality: citizenship.holder.nationality,
    rights: citizenship.rights,
    iat: Math.floor(Date.now() / 1000),
    exp: Math.floor(Date.now() / 1000) + (60 * 60) // 1시간
  };

  return jwt.sign(payload, process.env.JWT_SECRET, {
    algorithm: 'RS256',
    issuer: 'WIA-CITIZENSHIP',
    audience: 'https://api.wia.live'
  });
}

// JWT 검증
function verifyCitizenshipToken(token: string): CitizenshipToken {
  try {
    return jwt.verify(token, process.env.JWT_PUBLIC_KEY, {
      algorithms: ['RS256'],
      issuer: 'WIA-CITIZENSHIP'
    }) as CitizenshipToken;
  } catch (error) {
    throw new Error('유효하지 않은 시민권 토큰');
  }
}

5.7 SDK 사용법

5.7.1 TypeScript SDK

npm install @wia/citizenship-sdk

// 사용 예시
import { CitizenshipClient } from '@wia/citizenship-sdk';

const client = new CitizenshipClient({
  apiKey: 'your-api-key',
  region: 'kr-seoul',
  environment: 'production'
});

// 시민권 신청
async function applyCitizenship() {
  const application = await client.citizenship.apply({
    applicant: {
      full_name: '김철수',
      date_of_birth: '1990-05-15',
      nationality: 'KR',
      national_id: '900515-1234567'
    },
    contact: {
      email: 'chulsoo@example.com',
      phone: '+82-10-1234-5678'
    },
    digital_identity: {
      wallet_address: '0x742d35Cc...',
      verification_method: 'ethereum_signature'
    }
  });

  console.log(`신청 ID: ${application.citizenship_id}`);
  console.log(`상태: ${application.status}`);
  console.log(`예상 처리 시간: ${application.estimated_processing_time}`);
}

// 시민권 검증
async function verifyCitizenship(citizenshipId: string) {
  const verification = await client.citizenship.verify({
    citizenship_id: citizenshipId
  });

  if (verification.valid) {
    console.log('✓ 유효한 시민권');
    console.log(`보유 권리: ${verification.citizenship.rights.join(', ')}`);
  } else {
    console.log('✗ 유효하지 않음');
  }
}

    print(f"보유자: {citizenship.holder.name}")

    print(f"권리: {', '.join(citizenship.rights)}")

# 실시간 모니터링
async def monitor_applications():
    async with client.monitor_applications() as stream:
        async for event in stream:
            if event.type == 'application_submitted':
                print(f"새 신청: {event.application_id}")
            elif event.type == 'verification_completed':
                print(f"검증 완료: {event.citizenship_id}")

5.8 오류 처리

5.8.1 표준 오류 코드

오류 코드 HTTP 상태 설명 해결 방법
INVALID_NATIONAL_ID 400 주민등록번호 형식 오류 유효한 주민번호 입력

UNAUTHORIZED 401 인증 토큰 없음/만료 토큰 갱신
RATE_LIMIT_EXCEEDED 429 요청 한도 초과 플랜 업그레이드

5.10 Chapter Summary

핵심 요점

  1. REST API 설계: WIA-DIGITAL_CITIZENSHIP는 RESTful 원칙을 따르는 직관적인 API를 제공하며, 시민권 신청/검증/관리 전체 라이프사이클을 지원합니다.
  2. 정부 시스템 통합: 행정안전부, 법무부, 경찰청, 과기정통부 등 5개 정부 기관과 API 연동하여 일일 168,000건의 신원 확인을 처리합니다.

  4. SDK 지원: TypeScript, Python SDK를 제공하여 개발자 친화적이며, 5분 내 통합 가능한 간편한 인터페이스를 제공합니다.
