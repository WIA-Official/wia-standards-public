Chapter 5: Text and Unstructured Data Analytics

5.1 Introduction to Unstructured Data

Unstructured data lacks a predefined data model or organization. Unlike structured data (databases, spreadsheets), unstructured data includes text documents, emails, social media posts, images, videos, and audio. It's estimated that 80-90% of all data generated today is unstructured, making it a critical area for analytics.

5.2 Natural Language Processing (NLP) Fundamentals

NLP enables computers to understand, interpret, and generate human language.

5.2.1 Text Preprocessing Pipeline

Stemming/LemmatizationReduce words to root formPorter Stemmer, WordNet Lemmatizer

# NLP Preprocessing Example
import nltk
from nltk.tokenize import word_tokenize
from nltk.corpus import stopwords
from nltk.stem import WordNetLemmatizer

nltk.download('punkt')
nltk.download('stopwords')
nltk.download('wordnet')

def preprocess_text(text):
    # Lowercase
    text = text.lower()
    
    # Tokenize
    tokens = word_tokenize(text)
    
    # Remove stopwords and punctuation
    stop_words = set(stopwords.words('english'))
    tokens = [t for t in tokens if t.isalnum() and t not in stop_words]
    
    # Lemmatize
    lemmatizer = WordNetLemmatizer()
    tokens = [lemmatizer.lemmatize(t) for t in tokens]
    
    return tokens

# Example
text = "The quick brown foxes are jumping over the lazy dogs!"
processed = preprocess_text(text)
print(processed)  # ['quick', 'brown', 'fox', 'jumping', 'lazy', 'dog']

5.4 Sentiment Analysis

Sentiment analysis determines the emotional tone of text: positive, negative, or neutral.

5.4.1 Approaches to Sentiment Analysis

  • Lexicon-Based: Uses pre-built dictionaries of positive/negative words (VADER, TextBlob)
  • Machine Learning: Train classifiers on labeled data (Naive Bayes, SVM, Random Forest)
  • Deep Learning: Neural networks (LSTM, BERT) for complex sentiment
# Sentiment Analysis with VADER
from vaderSentiment.vaderSentiment import SentimentIntensityAnalyzer

analyzer = SentimentIntensityAnalyzer()

texts = [
    "This product is amazing! I love it!",
    "Terrible experience. Would not recommend.",
    "It's okay, nothing special."
]

for text in texts:
    scores = analyzer.polarity_scores(text)
    print(f"Text: {text}")
    print(f"Scores: {scores}")
    print(f"Sentiment: {'Positive' if scores['compound'] > 0.05 else 'Negative' if scores['compound'] < -0.05 else 'Neutral'}\n")

5.6 Named Entity Recognition (NER)

Identify and classify named entities (people, organizations, locations, dates) in text.

# NER with spaCy
import spacy

nlp = spacy.load("en_core_web_sm")
text = "Apple Inc. was founded by Steve Jobs in Cupertino, California in 1976."

doc = nlp(text)
for ent in doc.ents:
    print(f"{ent.text}: {ent.label_}")

# Output:
# Apple Inc.: ORG
# Steve Jobs: PERSON
# Cupertino: GPE
# California: GPE
# 1976: DATE

5.11 Practical NLP Pipeline with WIA-DATA-012

# Complete NLP Pipeline
from wia_analytics import TextAnalytics

# Initialize
text_analytics = TextAnalytics(api_key="your_api_key")

# Sample data
reviews = [
    "Great product! Exceeded my expectations.",
    "Poor quality. Broke after one week.",
    "Average. Nothing special but works fine."
]

# Analyze sentiment
sentiments = text_analytics.analyze_sentiment(reviews)
for review, sentiment in zip(reviews, sentiments):
    print(f"Review: {review}")
    print(f"Sentiment: {sentiment['label']} (score: {sentiment['score']:.2f})\n")

# Extract key phrases
key_phrases = text_analytics.extract_key_phrases(reviews)

# Named entity recognition
entities = text_analytics.extract_entities(reviews)

# Topic modeling
topics = text_analytics.discover_topics(reviews, num_topics=3)

# Text classification
categories = text_analytics.classify(
    texts=reviews,
    categories=["positive", "negative", "neutral"]
)

5.12 Challenges in Unstructured Data Analytics

Common Challenges:
  • Data Quality: Noise, typos, slang, abbreviations
  • Context Understanding: Sarcasm, idioms, cultural references
  • Language Variations: Dialects, multilingual content
  • Computational Resources: Processing large volumes of text/images
  • Privacy Concerns: Handling sensitive textual data
  • Domain Specificity: Medical, legal, technical jargon

5.13 Best Practices

  • Start with clean, representative data
  • Choose appropriate preprocessing for your use case
  • Consider domain-specific lexicons and models
  • Validate results with domain experts
  • Monitor model performance over time
  • Handle multilingual content appropriately
  • Respect privacy and ethical considerations
