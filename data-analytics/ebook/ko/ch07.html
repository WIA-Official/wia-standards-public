7.1 분석 생태계 개요

분석 생태계는 데이터 수집, 저장, 처리, 시각화 및 배포 등 다양한 목적을 제공하는 다양한 도구 및 플랫폼으로 구성됩니다. 현대 데이터 스택은 상호운용 가능한 도구들이 원활하게 통합되어 엔드투엔드 데이터 파이프라인을 구축합니다.

올바른 도구 선택은 조직의 성공에 결정적입니다. 잘못된 도구는 개발 속도를 늦추고, 비용을 증가시키며, 확장성 문제를 야기할 수 있습니다. 반면, 적절한 도구 조합은 팀의 생산성을 극대화하고 인사이트까지의 시간을 단축시킵니다.

7.1.1 분석 도구 카테고리

데이터 수집소스에서 데이터 추출Fivetran, Airbyte, Kafka

7.2 비즈니스 인텔리전스 (BI) 도구

BI 도구는 데이터를 시각화하고 비즈니스 사용자가 인사이트를 발견할 수 있게 합니다. 각 도구는 고유한 강점과 사용 사례가 있습니다.

7.2.1 Tableau

개요

Tableau는 강력한 시각화 기능과 직관적인 드래그 앤 드롭 인터페이스로 유명한 업계 선도적인 BI 플랫폼입니다.

주요 기능:

  • 풍부한 시각화 라이브러리 (100+ 차트 유형)
  • 실시간 데이터 연결 및 빠른 쿼리
  • 데이터 블렌딩 - 여러 소스 결합
  • Tableau Public으로 공개 대시보드 공유
  • 자연어 쿼리 (Ask Data)
  • AI 기반 인사이트 (Explain Data)

가격: Creator $70/월, Explorer $42/월, Viewer $15/월

최적 용도: 복잡한 시각화, 대시보드 스토리텔링, 대기업

7.2.2 Power BI

개요

Microsoft Power BI는 강력한 Microsoft 생태계 통합과 경쟁력 있는 가격으로 빠르게 성장하는 BI 도구입니다.

7.5 SQL 기반 분석

SQL은 데이터베이스 쿼리의 표준 언어로, 모든 데이터 분석가가 마스터해야 할 필수 기술입니다.

7.5.1 고급 SQL 쿼리

-- 윈도우 함수를 사용한 고급 분석

-- 1. 순위 및 순서
SELECT
    product_id,
    product_name,
    category,
    sales_amount,
    -- 전체 순위
    ROW_NUMBER() OVER (ORDER BY sales_amount DESC) as overall_rank,
    -- 카테고리별 순위
    RANK() OVER (PARTITION BY category ORDER BY sales_amount DESC) as category_rank,
    -- 백분위수
    PERCENT_RANK() OVER (ORDER BY sales_amount) as percentile
FROM product_sales
WHERE date >= '2024-01-01';

-- 2. 이동 평균 및 누적 합계
SELECT
    date,
    product_id,
    daily_sales,
    -- 7일 이동 평균
    AVG(daily_sales) OVER (
        PARTITION BY product_id
        ORDER BY date
        ROWS BETWEEN 6 PRECEDING AND CURRENT ROW
    ) as moving_avg_7d,
    -- 누적 합계
    SUM(daily_sales) OVER (
        PARTITION BY product_id
        ORDER BY date
        ROWS UNBOUNDED PRECEDING
    ) as cumulative_sales,
    -- 전년 동기 비교
    LAG(daily_sales, 365) OVER (
        PARTITION BY product_id
        ORDER BY date
    ) as sales_last_year
FROM daily_product_sales;

-- 3. 코호트 분석
WITH first_purchase AS (
    SELECT
        customer_id,
        DATE_TRUNC('month', MIN(purchase_date)) as cohort_month
    FROM orders
    GROUP BY customer_id
),
cohort_data AS (
    SELECT
        fp.cohort_month,
        DATE_TRUNC('month', o.purchase_date) as purchase_month,
        COUNT(DISTINCT o.customer_id) as customers,
        SUM(o.amount) as revenue
    FROM orders o
    JOIN first_purchase fp ON o.customer_id = fp.customer_id
    GROUP BY fp.cohort_month, DATE_TRUNC('month', o.purchase_date)
)
SELECT
    cohort_month,
    purchase_month,
    customers,
    revenue,
    -- 코호트 크기 대비 비율
    customers * 100.0 / FIRST_VALUE(customers) OVER (
        PARTITION BY cohort_month
        ORDER BY purchase_month
    ) as retention_rate
FROM cohort_data
ORDER BY cohort_month, purchase_month;

-- 4. RFM 분석 (Recency, Frequency, Monetary)
WITH customer_rfm AS (
    SELECT
        customer_id,
        MAX(purchase_date) as last_purchase,
        COUNT(*) as purchase_frequency,
        SUM(amount) as total_spent,
        -- Recency (최근성) - 마지막 구매로부터 경과일
        CURRENT_DATE - MAX(purchase_date) as days_since_last_purchase
    FROM orders
    WHERE purchase_date >= CURRENT_DATE - INTERVAL '1 year'
    GROUP BY customer_id
)
SELECT
    customer_id,
    days_since_last_purchase,
    purchase_frequency,
    total_spent,
    -- RFM 점수 (1-5)
    NTILE(5) OVER (ORDER BY days_since_last_purchase DESC) as recency_score,
    NTILE(5) OVER (ORDER BY purchase_frequency) as frequency_score,
    NTILE(5) OVER (ORDER BY total_spent) as monetary_score
FROM customer_rfm;

-- 1. 인덱스 활용

  AND region = 'US';

-- 3. CTE vs. 서브쿼리 (가독성과 성능)
-- CTE 사용 (권장)
WITH monthly_sales AS (
    SELECT
        DATE_TRUNC('month', date) as month,
        SUM(amount) as total
    FROM orders
    GROUP BY DATE_TRUNC('month', date)
),
averages AS (
    SELECT
        AVG(total) as avg_monthly_sales
    FROM monthly_sales
)
SELECT *
FROM monthly_sales
CROSS JOIN averages
WHERE total > avg_monthly_sales;

-- 4. 집계 최적화
-- 나쁜 예: 여러 번 스캔
SELECT
    (SELECT COUNT(*) FROM orders WHERE status = 'completed') as completed,
    (SELECT COUNT(*) FROM orders WHERE status = 'pending') as pending,
    (SELECT COUNT(*) FROM orders WHERE status = 'cancelled') as cancelled;

-- 좋은 예: 한 번 스캔
SELECT
    COUNT(*) FILTER (WHERE status = 'completed') as completed,
    COUNT(*) FILTER (WHERE status = 'pending') as pending,
    COUNT(*) FILTER (WHERE status = 'cancelled') as cancelled
FROM orders;

-- 5. EXPLAIN ANALYZE로 쿼리 계획 분석
EXPLAIN ANALYZE
SELECT
    c.customer_name,
    COUNT(*) as order_count,
    SUM(o.amount) as total_spent
FROM customers c
JOIN orders o ON c.customer_id = o.customer_id
WHERE o.date >= '2024-01-01'
GROUP BY c.customer_name
HAVING SUM(o.amount) > 1000;

    order_date,
    status,
    -- 데이터 정제
    CAST(amount AS DECIMAL(10,2)) as amount,
    -- 타임스탬프 표준화
    TIMESTAMP(order_date) as order_timestamp,
    -- NULL 처리
    COALESCE(discount, 0) as discount
FROM {{ source('raw', 'orders') }}
WHERE order_date IS NOT NULL
  AND amount > 0

-- 팩트 테이블 - 비즈니스 로직 적용

    materialized='incremental',
    unique_key='order_id'

SELECT
    o.order_id,
    o.customer_id,
    o.order_date,
    o.amount,
    o.discount,
    -- 계산 필드
    o.amount - o.discount as net_amount,
    -- 차원 테이블 조인
    c.customer_segment,
    p.product_category,
    -- 윈도우 함수
    ROW_NUMBER() OVER (
        PARTITION BY o.customer_id
        ORDER BY o.order_date
    ) as customer_order_number
FROM {{ ref('stg_orders') }} o
LEFT JOIN {{ ref('dim_customers') }} c ON o.customer_id = c.customer_id
LEFT JOIN {{ ref('dim_products') }} p ON o.product_id = p.product_id

7.8 워크플로우 오케스트레이션

데이터 파이프라인은 복잡한 종속성과 스케줄링이 필요합니다. 오케스트레이션 도구는 이를 자동화합니다.

7.8.1 Apache Airflow DAG

# airflow_pipeline.py - ETL 파이프라인 정의

from airflow import DAG
from airflow.operators.python import PythonOperator
from airflow.providers.postgres.operators.postgres import PostgresOperator
from airflow.providers.amazon.aws.transfers.s3_to_redshift import S3ToRedshiftOperator
from datetime import datetime, timedelta

default_args = {
    'owner': 'data-team',
    'depends_on_past': False,
    'start_date': datetime(2024, 1, 1),
    'email': ['alerts@company.com'],
    'email_on_failure': True,
    'email_on_retry': False,
    'retries': 3,
    'retry_delay': timedelta(minutes=5),
}

dag = DAG(
    'daily_sales_pipeline',
    default_args=default_args,
    description='일일 판매 데이터 ETL',
    schedule_interval='0 2 * * *',  # 매일 오전 2시
    catchup=False,
    tags=['sales', 'etl'],
)

# Task 1: S3에서 원시 데이터 추출
extract_task = S3ToRedshiftOperator(
    task_id='extract_from_s3',
    schema='raw',
    table='sales_raw',
    s3_bucket='company-data',
    s3_key='sales/{{ ds }}/sales.csv',
    copy_options=['CSV', 'IGNOREHEADER 1'],
    dag=dag,
)

# Task 2: 데이터 변환
transform_task = PostgresOperator(
    task_id='transform_data',
    postgres_conn_id='redshift',
    sql='''
        INSERT INTO analytics.fct_sales
        SELECT
            order_id,
            customer_id,
            order_date,
            amount,
            product_id
        FROM raw.sales_raw
        WHERE order_date = '{{ ds }}'
          AND amount > 0;
    ''',
    dag=dag,
)

# Task 3: 집계 계산
aggregate_task = PostgresOperator(
    task_id='calculate_aggregates',
    postgres_conn_id='redshift',
    sql='sql/daily_aggregates.sql',
    dag=dag,
)

# Task 4: 데이터 품질 검사
def check_data_quality(**context):
    from airflow.providers.postgres.hooks.postgres import PostgresHook

    hook = PostgresHook(postgres_conn_id='redshift')
    records = hook.get_records('''
        SELECT COUNT(*) FROM analytics.fct_sales
        WHERE order_date = '{{ ds }}'
    ''')

    if records[0][0] < 100:
        raise ValueError(f"데이터 품질 검사 실패: {records[0][0]} 레코드만 발견됨")

quality_check = PythonOperator(
    task_id='quality_check',
    python_callable=check_data_quality,
    dag=dag,
)

# 종속성 정의
extract_task >> transform_task >> aggregate_task >> quality_check

7.9 도구 선택 가이드

의사결정 프레임워크

1. 팀 규모 및 기술

  • 소규모 팀 (< 5명): Power BI, Metabase, Google Sheets
  • 중규모 팀 (5-20명): Tableau, Looker, Snowflake
  • 대규모 팀 (20명+): 엔터프라이즈 솔루션, 맞춤형 스택

2. 데이터 볼륨

  • 소량 (< 100GB): PostgreSQL, MySQL, Power BI
  • 중량 (100GB-10TB): Snowflake, BigQuery, Redshift
  • 대량 (> 10TB): BigQuery, Databricks, Spark

3. 예산

  • 제한적 (< $1K/월): 오픈소스 도구 (Metabase, Superset, PostgreSQL)
  • 중간 ($1K-$10K/월): Power BI, Snowflake 소규모, dbt Cloud
  • 높음 (> $10K/월): Tableau, Looker, 엔터프라이즈 Snowflake

4. 사용 사례

  • 실시간 분석: Kafka, Flink, Kinesis
  • 비즈니스 리포팅: Tableau, Power BI, Looker
  • 머신러닝: Databricks, SageMaker, Vertex AI
  • 데이터 거버넌스: Collibra, Alation, Looker

요약

핵심 포인트:

  • 도구 선택은 팀 기술, 데이터 규모, 예산, 사용 사례에 따라 결정됩니다
  • BI 도구 (Tableau, Power BI, Looker)는 시각화와 셀프서비스 분석을 제공합니다
  • Python/Pandas와 R은 프로그래밍 방식 분석과 고급 통계에 강력합니다
  • SQL은 모든 분석가의 필수 기술이며, 클라우드 웨어하우스에서 대규모로 실행됩니다
  • dbt는 데이터 변환을 코드화하고, Airflow는 파이프라인을 오케스트레이션합니다
  • 현대적인 데이터 스택은 상호운용 가능한 도구들을 조합하여 구축됩니다

마지막 장에서는 분석의 미래 동향과 증강 및 자동화된 분석으로의 진화를 살펴보겠습니다.
