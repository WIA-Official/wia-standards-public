4.1 처방적 분석 소개

처방적 분석은 무엇이 일어날 것인지 예측하는 것을 넘어 무엇을 해야 하는지를 권장합니다. 예측 모델을 비즈니스 규칙 및 최적화 알고리즘과 결합하여 최선의 행동 방침을 제안합니다. 기술적 분석이 무엇이 일어났는지 알려주고 예측 분석이 무엇이 일어날 것인지 예측하는 반면, 처방적 분석은 원하는 결과를 달성하기 위해 취해야 할 조치에 대해 조언합니다.

처방적 분석의 핵심 가치:
  • 의사결정 자동화: 반복적인 의사결정을 시스템에서 자동으로 처리
  • 최적 솔루션 발견: 수천 개의 가능한 옵션 중에서 최선의 선택
  • 리스크 관리: 불확실성을 고려한 강건한 의사결정
  • ROI 극대화: 제한된 자원으로 최대 가치 창출
  • 시나리오 분석: 다양한 "만약" 시나리오 평가

4.1.1 분석 성숙도 모델

단계유형질문기법가치
1기술적무엇이 일어났는가?보고, 대시보드, SQL기본
2진단적왜 일어났는가?드릴다운, 상관관계 분석중간
3예측적무엇이 일어날 것인가?회귀, 시계열, ML높음
4처방적무엇을 해야 하는가?최적화, 시뮬레이션, RL매우 높음

4.2 최적화 기초

최적화는 모든 가능한 솔루션 중에서 최선의 솔루션을 찾는 프로세스입니다. 다음이 포함됩니다:

  • 목적 함수: 최대화 또는 최소화할 목표 (예: 이익, 비용, 시간)
  • 결정 변수: 제어할 수 있는 변수
  • 제약 조건: 만족되어야 하는 제한 또는 요구사항
  • 가능 영역: 제약 조건을 만족하는 모든 솔루션의 집합

4.2.1 최적화 문제 유형

유형설명예시복잡도
선형 프로그래밍선형 목적 함수 및 제약 조건자원 할당, 생산 계획P (다항시간)
정수 프로그래밍일부/모든 변수가 정수여야 함시설 위치, 일정 계획NP-hard
비선형 프로그래밍비선형 목적 함수 또는 제약 조건포트폴리오 최적화, 매개변수 추정변동적
동적 프로그래밍다단계 의사결정 문제재고 관리, 최단 경로Pseudo-polynomial
확률적 최적화매개변수의 불확실성위험 관리, 강건한 최적화높음
다목적 최적화여러 상충하는 목표트레이드오프 분석, 파레토 최적화매우 높음

4.2.2 최적화 접근법 비교

접근법방법론장점단점
정확한 방법단체법, 분기한정법최적해 보장대규모 문제에 느림
휴리스틱탐욕 알고리즘, 규칙 기반빠름, 간단함최적성 보장 없음
메타휴리스틱유전 알고리즘, 시뮬레이티드 어닐링복잡한 문제에 적합파라미터 튜닝 필요
근사 알고리즘성능 보장 있는 근사품질 보장, 효율적문제 특화적

4.3 선형 프로그래밍

선형 프로그래밍(LP)은 가장 널리 사용되는 최적화 기법 중 하나입니다. 목적 함수와 제약 조건이 모두 선형일 때 최적의 솔루션을 찾습니다.

표준 형식:
최대화: c₁x₁ + c₂x₂ + ... + cₙxₙ
제약 조건:
a₁₁x₁ + a₁₂x₂ + ... + a₁ₙxₙ ≤ b₁
a₂₁x₁ + a₂₂x₂ + ... + a₂ₙxₙ ≤ b₂
...
x₁, x₂, ..., xₙ ≥ 0 
# Python 선형 프로그래밍 예제
from scipy.optimize import linprog
import numpy as np

# 문제: 제품 A와 B의 최적 생산량 결정
# 최대화: 3x + 4y (이익)
# 제약 조건:
#   x + 2y ≤ 14 (노동 시간)
#   3x + y ≤ 18 (원자재)
#   x, y ≥ 0

# linprog는 최소화만 지원하므로 부호 반전
c = [-3, -4]  # 목적 함수 계수

# 부등식 제약 (Ax ≤ b)
A_ub = [[1, 2], [3, 1]]
b_ub = [14, 18]

# 경계 (x, y ≥ 0)
x_bounds = (0, None)
y_bounds = (0, None)
bounds = [x_bounds, y_bounds]

# 최적화 실행
result = linprog(c, A_ub=A_ub, b_ub=b_ub, bounds=bounds, method='highs')

if result.success:
    print(f"최적 솔루션:")
    print(f"  제품 A: {result.x[0]:.2f}개")
    print(f"  제품 B: {result.x[1]:.2f}개")
    print(f"  최대 이익: ${-result.fun:.2f}")
    print(f"  여유 자원: {b_ub - A_ub @ result.x}")
else:
    print("최적화 실패:", result.message)

4.4 정수 프로그래밍

많은 실제 문제에서 결정 변수는 정수여야 합니다 (예: 직원 수, 트럭 대수, 프로젝트 선택).

# Python PuLP를 사용한 정수 프로그래밍
from pulp import *

# 문제 정의: 배낭 문제 (Knapsack Problem)
# 최대 무게 50kg으로 가치를 최대화

# 아이템 데이터
items = ['아이템1', '아이템2', '아이템3', '아이템4', '아이템5']
values = {'아이템1': 10, '아이템2': 40, '아이템3': 30,
          '아이템4': 50, '아이템5': 35}
weights = {'아이템1': 5, '아이템2': 20, '아이템3': 10,
           '아이템4': 25, '아이템5': 15}
max_weight = 50

# 문제 생성
prob = LpProblem("배낭문제", LpMaximize)

# 결정 변수 (0 또는 1)
item_vars = LpVariable.dicts("아이템", items, cat='Binary')

# 목적 함수: 총 가치 최대화
prob += lpSum([values[i] * item_vars[i] for i in items])

# 제약 조건: 총 무게 제한
prob += lpSum([weights[i] * item_vars[i] for i in items]) <= max_weight

# 해결
prob.solve()

# 결과 출력
print(f"상태: {LpStatus[prob.status]}")
print(f"최대 가치: {value(prob.objective)}")
print("\n선택된 아이템:")
for i in items:
    if item_vars[i].varValue == 1:
        print(f"  {i}: 가치={values[i]}, 무게={weights[i]}")

total_weight = sum([weights[i] * item_vars[i].varValue for i in items])
print(f"\n총 무게: {total_weight}kg / {max_weight}kg")

4.5 처방적 분석을 위한 의사결정 트리

의사결정 트리는 다양한 시나리오와 예상 결과를 기반으로 조치를 권장하는 데 처방적으로 사용될 수 있습니다.

구성 요소목적예시계산
결정 노드선택이 이루어지는 지점신제품 출시? (예/아니요)최대 EV 선택
기회 노드불확실한 결과가 있는 지점시장 반응 (높음/중간/낮음)EV = Σ(p × 값)
종료 노드최종 결과 및 보상이익: $500K순 현재 가치
기대값확률 가중 평균EV = Σ(확률 × 값)역방향 계산
# Python으로 의사결정 트리 분석
import numpy as np

class DecisionNode:
    def __init__(self, name, options):
        self.name = name
        self.options = options  # {option_name: expected_value}

    def best_decision(self):
        best = max(self.options.items(), key=lambda x: x[1])
        return best[0], best[1]

class ChanceNode:
    def __init__(self, name, outcomes):
        self.name = name
        self.outcomes = outcomes  # {outcome: (probability, value)}

    def expected_value(self):
        ev = sum(prob * val for prob, val in self.outcomes.values())
        return ev

# 예: 신제품 출시 의사결정
# 옵션 1: 출시 ($200K 투자)
market_high = 0.3 * 1000000  # 30% 확률, $1M 수익
market_medium = 0.5 * 400000  # 50% 확률, $400K 수익
market_low = 0.2 * 100000    # 20% 확률, $100K 수익

launch_ev = market_high + market_medium + market_low - 200000

# 옵션 2: 출시 안 함 ($0 투자, $0 수익)
no_launch_ev = 0

# 의사결정
decision = DecisionNode("신제품출시", {
    "출시": launch_ev,
    "출시안함": no_launch_ev
})

best_choice, best_value = decision.best_decision()
print(f"최선의 결정: {best_choice}")
print(f"기대값: ${best_value:,.0f}")

# 민감도 분석
print("\n민감도 분석 (높은 시장 반응 확률 변화):")
for p_high in np.arange(0.1, 0.6, 0.1):
    p_medium = 0.5
    p_low = 1 - p_high - p_medium
    ev = (p_high * 1000000 + p_medium * 400000 + p_low * 100000) - 200000
    print(f"  P(높음)={p_high:.1f}: EV=${ev:,.0f}")

4.6 시뮬레이션 및 시나리오 분석

시뮬레이션은 여러 시나리오를 실행하여 가능한 결과를 이해함으로써 복잡한 시스템의 동작을 모델링합니다.

4.6.1 몬테카를로 시뮬레이션

무작위 샘플링을 사용하여 시스템의 불확실성과 변동성을 모델링합니다.

# 몬테카를로 시뮬레이션 - 프로젝트 리스크 분석
import numpy as np
import matplotlib.pyplot as plt

# 프로젝트 비용 및 수익 불확실성 시뮬레이션
np.random.seed(42)
n_simulations = 10000

# 비용 요소 (정규분포)
labor_cost = np.random.normal(500000, 50000, n_simulations)
material_cost = np.random.normal(300000, 40000, n_simulations)
overhead = np.random.normal(100000, 20000, n_simulations)

# 수익 (삼각분포 - 최소, 최빈, 최대)
revenue = np.random.triangular(800000, 1200000, 1500000, n_simulations)

# 순이익 계산
total_cost = labor_cost + material_cost + overhead
net_profit = revenue - total_cost

# 결과 분석
mean_profit = np.mean(net_profit)
median_profit = np.median(net_profit)
std_profit = np.std(net_profit)
percentile_5 = np.percentile(net_profit, 5)
percentile_95 = np.percentile(net_profit, 95)

# 리스크 메트릭
prob_loss = np.mean(net_profit < 0) * 100
prob_target = np.mean(net_profit > 200000) * 100

print("프로젝트 리스크 분석 결과:")
print(f"  평균 순이익: ${mean_profit:,.0f}")
print(f"  중간값: ${median_profit:,.0f}")
print(f"  표준편차: ${std_profit:,.0f}")
print(f"  90% 신뢰구간: [${percentile_5:,.0f}, ${percentile_95:,.0f}]")
print(f"\n리스크 메트릭:")
print(f"  손실 확률: {prob_loss:.1f}%")
print(f"  목표($200K) 달성 확률: {prob_target:.1f}%")

# VaR (Value at Risk) 계산
var_95 = percentile_5
print(f"  95% VaR: ${-var_95:,.0f} (5% 확률로 이보다 큰 손실)")

4.6.2 이산 사건 시뮬레이션

# SimPy를 사용한 이산 사건 시뮬레이션 - 콜센터
import simpy
import random
import numpy as np

class CallCenter:
    def __init__(self, env, num_agents):
        self.env = env
        self.agents = simpy.Resource(env, num_agents)
        self.wait_times = []
        self.service_times = []

    def handle_call(self, call_duration):
        """통화 처리 시뮬레이션"""
        yield self.env.timeout(call_duration)

    def customer(self, name):
        """고객 프로세스"""
        arrival_time = self.env.now

        with self.agents.request() as request:
            yield request
            wait_time = self.env.now - arrival_time
            self.wait_times.append(wait_time)

            service_time = random.expovariate(1.0/5.0)  # 평균 5분
            yield self.env.process(self.handle_call(service_time))
            self.service_times.append(service_time)

def generate_customers(env, call_center, arrival_rate):
    """고객 도착 생성"""
    customer_count = 0
    while True:
        yield env.timeout(random.expovariate(arrival_rate))
        customer_count += 1
        env.process(call_center.customer(f'고객{customer_count}'))

# 시나리오 비교: 상담원 수 최적화
print("콜센터 시뮬레이션 - 상담원 수 최적화\n")

for num_agents in [3, 4, 5, 6]:
    env = simpy.Environment()
    call_center = CallCenter(env, num_agents)
    env.process(generate_customers(env, call_center, 0.8))  # 시간당 48통화

    env.run(until=480)  # 8시간 시뮬레이션

    avg_wait = np.mean(call_center.wait_times)
    max_wait = np.max(call_center.wait_times)
    service_level = np.mean(np.array(call_center.wait_times) < 2) * 100

    print(f"상담원 {num_agents}명:")
    print(f"  평균 대기시간: {avg_wait:.2f}분")
    print(f"  최대 대기시간: {max_wait:.2f}분")
    print(f"  서비스 수준 (2분 내): {service_level:.1f}%")
    print(f"  처리된 통화: {len(call_center.wait_times)}건\n")

4.7 추천 시스템

추천 시스템은 사용자 행동 및 선호도를 기반으로 개인화된 행동 (예: 구매할 제품, 소비할 콘텐츠)을 처방합니다.

접근 방식작동 방식사용 사례장단점
협업 필터링유사한 사용자의 행동 사용Netflix, Amazon 제품 추천강력하지만 콜드 스타트 문제
콘텐츠 기반유사한 항목 추천음악 추천, 기사 제안과적합 위험
하이브리드여러 접근 방식 결합YouTube, Spotify최상의 성능, 복잡함
지식 기반도메인 지식 및 규칙 사용금융 자문, 의료 추천설명 가능, 확장성 제한
# Python 협업 필터링 추천 시스템
import numpy as np
from sklearn.metrics.pairwise import cosine_similarity

# 사용자-아이템 평점 행렬 (5점 만점)
ratings = np.array([
    [5, 3, 0, 1, 4],  # 사용자1
    [4, 0, 0, 1, 2],  # 사용자2
    [1, 1, 0, 5, 4],  # 사용자3
    [0, 0, 5, 4, 0],  # 사용자4
    [2, 1, 3, 0, 0],  # 사용자5
])

# 아이템 이름
items = ['제품A', '제품B', '제품C', '제품D', '제품E']

def collaborative_filtering(ratings, user_id, k=2):
    """
    협업 필터링 추천
    user_id: 추천 대상 사용자 (0-based index)
    k: 고려할 유사 사용자 수
    """
    # 사용자 간 코사인 유사도 계산
    # 0으로 나눗셈 방지를 위해 작은 값 추가
    similarity = cosine_similarity(ratings + 1e-9)

    # 자기 자신 제외
    similarity[user_id, user_id] = 0

    # 가장 유사한 k명의 사용자 찾기
    similar_users = np.argsort(similarity[user_id])[::-1][:k]

    # 타겟 사용자가 평점을 주지 않은 아이템 찾기
    user_ratings = ratings[user_id]
    unrated_items = np.where(user_ratings == 0)[0]

    # 예측 평점 계산
    predictions = []
    for item in unrated_items:
        # 유사 사용자의 가중 평균 평점
        similar_ratings = ratings[similar_users, item]
        similar_weights = similarity[user_id, similar_users]

        # 평점을 준 사용자만 고려
        rated_mask = similar_ratings > 0
        if rated_mask.sum() > 0:
            weighted_sum = (similar_ratings[rated_mask] *
                          similar_weights[rated_mask]).sum()
            weight_sum = similar_weights[rated_mask].sum()
            pred_rating = weighted_sum / weight_sum if weight_sum > 0 else 0
        else:
            pred_rating = 0

        predictions.append((item, pred_rating))

    # 예측 평점 순으로 정렬
    predictions.sort(key=lambda x: x[1], reverse=True)

    return predictions

# 사용자 0에 대한 추천
user_id = 0
recommendations = collaborative_filtering(ratings, user_id, k=2)

print(f"사용자 {user_id+1}의 현재 평점:")
for i, rating in enumerate(ratings[user_id]):
    if rating > 0:
        print(f"  {items[i]}: {rating}점")

print(f"\n추천 아이템:")
for item_idx, pred_rating in recommendations[:3]:
    print(f"  {items[item_idx]}: 예상 평점 {pred_rating:.2f}점")

4.8.1 Q-Learning 예제

# Q-Learning - 간단한 재고 관리 문제
import numpy as np
import random

class InventoryEnvironment:
    def __init__(self, max_inventory=20, holding_cost=1,
                 stockout_cost=10, order_cost=5):
        self.max_inventory = max_inventory
        self.holding_cost = holding_cost
        self.stockout_cost = stockout_cost
        self.order_cost = order_cost
        self.reset()

    def reset(self):
        self.inventory = 10
        return self.inventory

    def step(self, order_qty):
        """
        행동 실행: order_qty만큼 주문
        반환: (다음 상태, 보상, 종료여부)
        """
        # 수요는 포아송 분포 (평균 5)
        demand = np.random.poisson(5)

        # 비용 계산
        order_cost_total = self.order_cost * (order_qty > 0)

        # 재고 업데이트
        self.inventory = min(self.inventory + order_qty, self.max_inventory)

        # 판매 및 재고 부족 처리
        if self.inventory >= demand:
            sales = demand
            stockout = 0
        else:
            sales = self.inventory
            stockout = demand - self.inventory

        self.inventory -= sales

        # 보상 계산 (비용 최소화이므로 음수)
        holding_cost_total = self.holding_cost * self.inventory
        stockout_cost_total = self.stockout_cost * stockout
        reward = -(order_cost_total + holding_cost_total + stockout_cost_total)

        return self.inventory, reward, False

# Q-Learning 알고리즘
def q_learning(env, episodes=1000, alpha=0.1, gamma=0.95, epsilon=0.1):
    """
    Q-Learning 훈련
    alpha: 학습률
    gamma: 할인율
    epsilon: 탐험 비율
    """
    # Q-테이블 초기화 (상태 x 행동)
    Q = np.zeros((env.max_inventory + 1, env.max_inventory + 1))

    for episode in range(episodes):
        state = env.reset()
        total_reward = 0

        for _ in range(30):  # 30일 시뮬레이션
            # Epsilon-greedy 정책
            if random.random() < epsilon:
                action = random.randint(0, env.max_inventory - state)
            else:
                action = np.argmax(Q[state, :env.max_inventory - state + 1])

            # 행동 실행
            next_state, reward, done = env.step(action)
            total_reward += reward

            # Q-값 업데이트
            best_next_action = np.argmax(Q[next_state])
            Q[state, action] += alpha * (
                reward + gamma * Q[next_state, best_next_action] - Q[state, action]
            )

            state = next_state

        if (episode + 1) % 100 == 0:
            print(f"에피소드 {episode+1}: 평균 보상 = {total_reward/30:.2f}")

    return Q

# 훈련
env = InventoryEnvironment()
Q_table = q_learning(env, episodes=1000)

# 학습된 정책 추출
print("\n학습된 재주문 정책:")
print("현재재고 -> 최적주문량")
for state in range(0, 21, 5):
    best_action = np.argmax(Q_table[state])
    print(f"  {state}개 -> {best_action}개 주문")

4.9 다기준 의사결정 분석 (MCDA)

MCDA는 여러 상충하는 기준을 고려해야 할 때 의사결정을 돕습니다.

방법설명최적 용도복잡도
가중 합 모델기준에 가중치를 곱하고 합산간단한 결정, 비교 가능한 기준낮음

# TOPSIS 구현 예제
import numpy as np

def topsis(decision_matrix, weights, is_benefit):
    """
    TOPSIS (Technique for Order Preference by Similarity to Ideal Solution)

    decision_matrix: 대안 x 기준 행렬
    weights: 기준별 가중치
    is_benefit: 기준이 이익 기준인지 (True) 비용 기준인지 (False)
    """
    # 1. 정규화
    normalized = decision_matrix / np.sqrt((decision_matrix ** 2).sum(axis=0))

    # 2. 가중 정규화 행렬
    weighted_normalized = normalized * weights

    # 3. 이상적 및 부정적 이상적 솔루션
    ideal_best = np.zeros(len(weights))
    ideal_worst = np.zeros(len(weights))

    for i in range(len(weights)):
        if is_benefit[i]:
            ideal_best[i] = weighted_normalized[:, i].max()
            ideal_worst[i] = weighted_normalized[:, i].min()
        else:
            ideal_best[i] = weighted_normalized[:, i].min()
            ideal_worst[i] = weighted_normalized[:, i].max()

    # 4. 유클리드 거리 계산
    distance_to_best = np.sqrt(((weighted_normalized - ideal_best) ** 2).sum(axis=1))
    distance_to_worst = np.sqrt(((weighted_normalized - ideal_worst) ** 2).sum(axis=1))

    # 5. 상대적 근접도
    closeness = distance_to_worst / (distance_to_best + distance_to_worst)

    return closeness

# 예: 공급업체 선택
# 기준: 가격(낮을수록 좋음), 품질(높을수록 좋음), 배송시간(낮을수록 좋음)
suppliers = ['공급업체A', '공급업체B', '공급업체C', '공급업체D']

decision_matrix = np.array([
    [250, 8, 3],   # 공급업체A: 가격=$250K, 품질=8, 배송=3일
    [200, 6, 5],   # 공급업체B
    [300, 9, 2],   # 공급업체C
    [280, 7, 4],   # 공급업체D
])

weights = np.array([0.4, 0.35, 0.25])  # 가격, 품질, 배송시간 가중치
is_benefit = [False, True, False]       # 가격/배송은 낮을수록, 품질은 높을수록 좋음

# TOPSIS 실행

# 결과 출력
print("공급업체 선택 분석 (TOPSIS):\n")

for rank, (supplier, score) in enumerate(ranked, 1):
    print(f"{rank}. {supplier}: 점수 {score:.4f}")
