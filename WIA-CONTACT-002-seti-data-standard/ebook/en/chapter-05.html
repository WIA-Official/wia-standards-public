Chapter 5: Multi-site Coordination

The power of SETI lies not in any single telescope but in the collective capability of a global network. When a potential signal is detected at one facility, rapid verification by independent observatories is essential to distinguish genuine extraterrestrial transmissions from local interference. This chapter defines the protocols and infrastructure that enable seamless coordination across dozens of facilities spanning every continent and time zone.

5.3 Response Coordination

Facility Response Protocol

When a verification request arrives, facilities must respond with their availability and capability:

{
  "message_type": "VERIFICATION_RESPONSE",
  "request_id": "VR-2024-GBT-00042",
  "responding_facility": "ATA",
  "response_timestamp": "2024-06-15T03:47:23Z",

  "availability": {
    "status": "AVAILABLE",
    "earliest_observation_start": "2024-06-15T04:00:00Z",
    "maximum_duration_available_s": 1800
  },

  "capability_match": {
    "frequency_coverage": true,
    "sensitivity_adequate": true,
    "sky_position_observable": true,
    "estimated_snr": 8.2
  },

  "constraints": {
    "scheduled_maintenance": null,
    "weather_conditions": "clear",
    "rfi_environment": "moderate"
  }
}

Observation Scheduling

The regional hub coordinates responses and schedules observations to maximize verification confidence:

  • Geographic diversity: Prioritize facilities on different continents to rule out terrestrial interference
  • Time distribution: Stagger observations to catch intermittent signals
  • Equipment diversity: Include different antenna types and receiver systems
  • Sensitivity matching: Ensure responding facilities can detect the signal if present

5.5 Consensus Protocols

Verification Voting

After observations complete, facilities submit their verification votes:

{
  "message_type": "VERIFICATION_VOTE",
  "request_id": "VR-2024-GBT-00042",
  "voting_facility": "ATA",
  "observation_id": "OBS-2024-ATA-12345",

  "vote": "CONFIRMED",
  "confidence": 0.92,

  "observation_summary": {
    "signal_detected": true,
    "measured_frequency_hz": 1420405751.803,
    "measured_snr": 9.4,
    "measured_drift_rate": 0.41,
    "duration_observed_s": 450
  },

  "notes": "Clear detection, parameters consistent with original report"
}

Vote Categories

Vote Meaning Weight Modifier
CONFIRMED Signal detected, parameters match +1.0
PARTIAL Detection but some parameters differ +0.5
NOT_DETECTED No signal found at expected parameters -0.5
INCONCLUSIVE Unable to determine (RFI, weather, etc.) 0
REJECTED Identified as RFI or artifact -1.0

Consensus Threshold

A signal is considered VERIFIED when the weighted consensus score exceeds +3.0 with at least 3 independent confirmations from facilities on at least 2 continents. A score below -2.0 with at least 3 non-detections results in REJECTED status.

5.6 Time Synchronization

Accurate time synchronization across facilities is critical for signal correlation and Doppler analysis.

Timing Requirements

Requirement Specification Purpose
Absolute accuracy < 1 microsecond Signal arrival time correlation
Relative stability < 100 nanoseconds Phase coherence for VLBI
Time system UTC via GPS Universal reference

5.7 Communication Redundancy

Network reliability is essential—verification failures due to communication outages are unacceptable.

Redundancy Layers

  • Primary: Dedicated fiber connections between major facilities
  • Secondary: Commercial internet with VPN encryption
  • Tertiary: Satellite backup for remote locations
  • Emergency: HF radio network for catastrophic internet failure

5.8 Coordination Exercises

Regular exercises maintain network readiness and identify issues before real events.

Exercise Types

Exercise Frequency Scope

Stress Test Annually Multiple simultaneous events, failure injection

Chapter 5 Summary

Multi-site coordination under WIA-CONTACT-002 provides:

  1. Global network architecture with regional hubs for low-latency coordination
  2. Verification request protocol enabling rapid response to candidate signals
  3. Real-time data sharing for collaborative analysis during observations
  4. Consensus protocols providing objective signal confirmation or rejection
  5. Time synchronization ensuring cross-facility data correlation
  6. Communication redundancy maintaining network availability
