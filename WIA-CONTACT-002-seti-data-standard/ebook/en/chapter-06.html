Raw telescope data must be transformed into actionable intelligence through sophisticated analysis pipelines. From spectral analysis to machine learning classification, these pipelines form the analytical backbone of the global SETI network.

6.2 Radio Frequency Interference (RFI) Mitigation

RFI is the primary enemy of SETI detection. Terrestrial signals vastly outnumber any potential extraterrestrial transmissions, and sophisticated mitigation is essential.

RFI Detection Methods

  • Spectral Kurtosis: Identifies non-Gaussian signals indicating RFI
  • Median Absolute Deviation (MAD): Robust outlier detection
  • On-Off Comparison: Compares target pointings with off-source observations
  • Multi-Beam Anti-Coincidence: RFI appears in all beams simultaneously
  • Known RFI Database: Matches against catalog of known terrestrial sources

RFI Excision Pipeline

// RFI excision implementation
class RFIExcisionPipeline {

  excise(data: TimeSeries): TimeSeries {
    // Stage 1: Statistical detection
    const kurtosis = this.computeSpectralKurtosis(data);
    const rfiMask = kurtosis.map(k => k > this.kurtosisThreshold);

    // Stage 2: Database matching
    const knownRFI = this.rfiDatabase.match(data.frequencies);
    rfiMask.or(knownRFI);

    // Stage 3: Persistent narrowband detection
    const persistent = this.detectPersistentNarrowband(data);
    rfiMask.or(persistent);

    // Stage 4: Apply mask with interpolation
    return this.applyMaskWithInterpolation(data, rfiMask);
  }
}

6.3 Spectral Analysis

FFT Processing

The Fast Fourier Transform converts time-domain voltage data into frequency-domain power spectra. Standard parameters ensure cross-facility consistency:

Parameter Standard Value Rationale
FFT Length 2^20 (1M points) ~1 Hz resolution at typical sample rates
Window Function Hanning Good frequency resolution, moderate leakage
Overlap 50% Optimal time resolution
Averaging Welch method Reduces variance

Doppler Drift Detection

Extraterrestrial signals will exhibit Doppler drift due to relative motion between transmitter and receiver. The de-Doppler search is a key SETI technique:

// Doppler drift search algorithm
function searchDopplerSpace(waterfall: DynamicSpectrum): Candidate[] {
  const candidates = [];

  // Search drift rates from -10 to +10 Hz/s
  for (let driftRate = -10; driftRate <= 10; driftRate += 0.01) {

    // Apply de-drifting correction
    const dedrifted = applyDriftCorrection(waterfall, driftRate);

    // Integrate along time axis
    const integrated = dedrifted.sumAlongTime();

    // Find peaks above threshold
    const peaks = findPeaks(integrated, SNR_THRESHOLD);

    for (const peak of peaks) {
      candidates.push({
        frequency: peak.frequency,
        driftRate: driftRate,
        snr: peak.snr,
        significance: calculateSignificance(peak)
      });
    }
  }

  return candidates;
}

6.4 Signal Detection Algorithms

Narrowband Search

The primary SETI search targets narrowband signals—spectral features with bandwidth less than a few Hz, far narrower than any known natural phenomenon.

  • turboSETI: GPU-accelerated Doppler drift search
  • seticore: CPU-optimized with advanced RFI rejection
  • hyperseti: Hierarchical search for computational efficiency

Training Data

Models are trained on curated datasets:

  • RFI samples: Millions of confirmed terrestrial interference examples
  • Natural sources: Pulsars, masers, FRBs, and other cosmic phenomena
  • Synthetic ETI: Injected artificial signals with known parameters
  • Negative samples: Pure noise with no signals

Model Validation

All ML models must achieve >99% recall on synthetic ETI signals and >99.9% precision in RFI rejection before deployment. Models are retrained quarterly with new data and evaluated against held-out test sets from each participating facility.

6.7 Reproducibility

Scientific credibility requires reproducible analysis. WIA-CONTACT-002 mandates:

  • Version tracking: All pipeline software versions recorded in metadata
  • Parameter logging: Complete configuration captured with each run
  • Containerization: Reference implementations in Docker containers
  • Regression testing: Standard test datasets with known outputs
