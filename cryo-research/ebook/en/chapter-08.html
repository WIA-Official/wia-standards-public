8.1 DATA RELEASE PREPARATION
--------------------------------------------------------------------------------

Publishing research data alongside manuscripts maximizes scientific impact
and enables reproducibility.

[CRYO] DATA RELEASE CHECKLIST:

    BEFORE SUBMISSION
    ☑ Complete all experiments and analyses
    ☑ Perform quality assurance on all data files
    ☑ De-identify any human subject data (HIPAA compliant)
    ☑ Organize files in logical directory structure
    ☑ Create comprehensive README file
    ☑ Write data dictionary defining all variables
    ☑ Document analysis code with comments
    ☑ Test that code runs on clean environment
    ☑ Choose appropriate repository (Zenodo, Dryad, etc.)
    ☑ Select open license (CC-BY-4.0 recommended)
    ☑ Obtain co-author approval for release

    FILE ORGANIZATION
    ┌──────────────────────────────────────────────────────────────────┐
    │ research-data/                                                   │
    │ ├── README.md                   # Overview and guide             │

8.2 SUPPLEMENTARY MATERIAL FORMATS
--------------------------------------------------------------------------------

Journals require specific formats for supplementary materials to ensure
accessibility.

[!] SUPPLEMENTARY MATERIAL GUIDELINES:

    ACCEPTABLE FORMATS (Most Journals)
    ┌──────────────────────────────────────────────────────────────────┐
    │ Type               | Format        | Max Size  | Notes            │
    ├────────────────────┼───────────────┼───────────┼─────────────────┤
    │ Text/Methods       | PDF, DOCX     | 10 MB     | Searchable PDF   │
    │ Tables             | Excel, CSV    | 5 MB      | One sheet/file   │
    │ Figures            | PDF, TIFF,PNG | 50 MB     | High resolution  │
    │ Videos             | MP4, AVI      | 500 MB    | Compressed       │
    │ 3D structures      | PDB, CIF      | 10 MB     | Standard format  │
    │ Code               | ZIP, TAR.GZ   | 50 MB     | Include README   │
    │ Large datasets     | Link to repo  | N/A       | Zenodo, Dryad    │
    └──────────────────────────────────────────────────────────────────┘

    SUPPLEMENTARY FILE NAMING
    ┌──────────────────────────────────────────────────────────────────┐
    │ Convention: [Paper]_[Type][Number].[ext]                         │
    │                                                                  │
    │ Examples:                                                        │
    │ - Smith2025_TableS1.xlsx                                         │
    │ - Smith2025_FigureS2.pdf                                         │
    │ - Smith2025_MethodsS1.pdf                                        │
    │ - Smith2025_CodeS1.zip                                           │
    │                                                                  │
    │ Avoid spaces, special characters; use underscores                │
    └──────────────────────────────────────────────────────────────────┘

[CRYO] SUPPLEMENTARY CONTENT CHECKLIST:

    ESSENTIAL SUPPLEMENTARY MATERIALS
    ☑ Detailed protocols (beyond main text Methods)
    ☑ Complete statistical analyses
    ☑ Full data tables (main text shows summaries)
    ☑ Additional figures supporting conclusions
    ☑ Equipment specifications and calibration data
    ☑ CPA formulation details (complete compositions)
    ☑ Raw data or link to repository
    ☑ Analysis code (R, Python, etc.)
    ☑ Negative results (failed experiments)

8.3 DIGITAL OBJECT IDENTIFIERS (DOIs)
--------------------------------------------------------------------------------

DOIs provide persistent identifiers for publications and datasets.

[OK] DOI SYSTEM OVERVIEW:

    DOI STRUCTURE
    ┌──────────────────────────────────────────────────────────────────┐
    │ Format: https://doi.org/[prefix]/[suffix]                        │
    │                                                                  │
    │ Example: https://doi.org/10.1016/j.cryobiol.2025.03.001          │
    │          ├─────────────┘ └────┘ └──────────────────────┘         │
    │          URL prefix     Publisher  Publisher-specific ID         │
    │                                                                  │
    │ Dataset DOI: https://doi.org/10.5281/zenodo.123456               │
    │              └─────────────┘ └─────┘ └──────┘                    │
    │              URL prefix      Zenodo  Record ID                   │
    └──────────────────────────────────────────────────────────────────┘

    DOI REGISTRATION SERVICES
    ┌──────────────────────────────────────────────────────────────────┐
    │ Service       | Type           | Cost      | Best For             │
    ├───────────────┼────────────────┼───────────┼─────────────────────┤
    │ Zenodo        | Datasets       | Free      | General research data│
    │ Dryad         | Datasets       | $120/sub  | Large datasets       │
    │ figshare      | Datasets/Figs  | Free      | Figures, small data  │
    │ OSF           | Projects       | Free      | Preprints, protocols │
    │ Crossref      | Publications   | Via pub   | Journal articles     │
    └──────────────────────────────────────────────────────────────────┘

    DOI METADATA REQUIREMENTS
    ☑ Creator(s) with ORCID
    ☑ Title
    ☑ Publication year
    ☑ Publisher/Repository
    ☑ Resource type (Dataset, Software, etc.)
    ☑ Description/Abstract
    ☑ Subject keywords
    ☑ License
    ☑ Related identifiers (e.g., paper DOI)

8.4 REPRODUCIBILITY CHECKLISTS
--------------------------------------------------------------------------------

Reproducibility checklists ensure sufficient detail for others to replicate
your work.

[!] MINIMUM INFORMATION FOR PUBLICATION:

    EXPERIMENTAL DESIGN
    ☑ Subject/sample type and source
    ☑ Sample size and power analysis
    ☑ Randomization method (if applicable)
    ☑ Blinding (who was blinded, to what)
    ☑ Inclusion/exclusion criteria
    ☑ Experimental vs control groups

    MATERIALS
    ☑ CPA components with supplier, catalog #, lot #
    ☑ Cell culture media and supplements
    ☑ Equipment manufacturer and model
    ☑ Software name and version
    ☑ Antibodies/assay kits with lot numbers

    PROCEDURES
    ☑ Complete CPA preparation protocol
    ☑ Cooling protocol with rates and endpoints
    ☑ Storage conditions (temperature, duration)
    ☑ Warming/revival protocol
    ☑ CPA removal steps
    ☑ Viability assay protocol with timing
    ☑ Functional assay protocols

    DATA ANALYSIS
    ☑ Statistical tests used (with justification)
    ☑ Software and packages/libraries with versions
    ☑ Significance threshold (α level)
    ☑ Multiple comparison corrections (if applicable)
    ☑ Handling of outliers
    ☑ Missing data treatment

[CRYO] REPRODUCIBILITY STATEMENT TEMPLATE:

    "All experiments were performed in biological triplicate with
    technical replicates as stated. Sample size was determined by
    power analysis (α=0.05, β=0.20, effect size=0.5, calculated n=24
    per group). Assessors were blinded to experimental conditions
    during viability counting. Primary human hepatocytes were procured
    from [source] under IRB protocol [number]. Cryoprotectant components
    were from Sigma-Aldrich: DMSO (D2650, lot SLCD1234), trehalose
    (T9531, lot SLBW5678). Controlled-rate freezing used CryoMed
    CRF-2000 (S/N 12345). Viability was assessed by Trypan Blue
    exclusion within 2 hours post-thaw. Statistical analysis used
    GraphPad Prism v9.5.1 with two-tailed t-tests and p<0.05
    considered significant. All data and analysis code available at
    DOI:10.5281/zenodo.123456."

8.5 RAW DATA ARCHIVAL
--------------------------------------------------------------------------------

Raw data archival preserves the primary evidence supporting published findings.

[OK] ARCHIVAL BEST PRACTICES:

    WHAT TO ARCHIVE
    ┌──────────────────────────────────────────────────────────────────┐
    │ Priority 1 (Essential):                                          │
    │ - Raw viability counts (not just percentages)                    │
    │ - Temperature time-series (complete logs)                        │
    │ - Microscopy images (original, unprocessed)                      │
    │ - Spectrophotometry readings                                     │
    │ - Flow cytometry FCS files                                       │
    │                                                                  │
    │ Priority 2 (Important):                                          │
    │ - Intermediate data processing files                             │
    │ - Calibration records                                            │
    │ - Experimental notes/lab notebooks                               │
    │                                                                  │
    │ Priority 3 (Nice to have):                                       │
    │ - Failed experiments (negative results)                          │
    │ - Pilot studies                                                  │
    └──────────────────────────────────────────────────────────────────┘

    FILE FORMAT LONGEVITY
    ┌──────────────────────────────────────────────────────────────────┐
    │ Preferred Formats (Open, Stable):                                │
    │ - Text: TXT, CSV, JSON, XML                                      │
    │ - Images: TIFF (uncompressed), PNG                               │
    │ - Documents: PDF/A (archival PDF)                                │
    │ - Data: HDF5, NetCDF                                             │
    │                                                                  │
    │ Acceptable (Convert if Possible):                                │
    │ - Excel: Convert to CSV for long-term                            │
    │ - Word: Convert to PDF/A                                         │
    │ - Proprietary: Include reader software or convert                │
    └──────────────────────────────────────────────────────────────────┘

    METADATA FOR ARCHIVAL
    ☑ File creation date
    ☑ Creator/operator
    ☑ Instrument/software used
    ☑ Experimental context (which experiment/sample)
    ☑ Processing history (what was done to raw data)
    ☑ Variable names and units
    ☑ Quality flags or notes

8.6 CODE AND ANALYSIS SHARING
--------------------------------------------------------------------------------

Sharing analysis code is essential for computational reproducibility.

[!] CODE SHARING CHECKLIST:

    CODE PREPARATION
    ☑ Remove hard-coded file paths (use relative paths)
    ☑ Add comments explaining each analysis step
    ☑ Include session info (R) or environment (Python)
    ☑ Test code runs on clean environment
    ☑ Remove debugging code and commented-out sections
    ☑ Organize into logical sections/functions
    ☑ Include example data if possible

    DOCUMENTATION REQUIREMENTS
    ☑ README with installation instructions
    ☑ Dependencies list (requirements.txt, environment.yml)
    ☑ Step-by-step execution guide
    ☑ Expected output description
    ☑ Troubleshooting tips
    ☑ License (MIT or GPL recommended)
    ☑ Contact information for questions

[CRYO] CODE REPOSITORY EXAMPLE:

    GitHub Repository: username/hepatocyte-cryo-analysis
    ├── README.md                    # Overview and instructions
    ├── LICENSE                       # MIT License
    ├── requirements.txt             # Python dependencies
    ├── environment.yml              # Conda environment
    ├── data/
    │   ├── example_data.csv         # Small test dataset
    │   └── README.md                 # Data source information
    ├── scripts/
    │   ├── 01_data_cleaning.R       # Data preprocessing
    │   ├── 02_viability_analysis.R  # Primary analysis
    │   ├── 03_figures.R             # Generate publication figures
    │   └── 04_statistics.R          # Statistical tests
    ├── notebooks/
    │   └── exploratory_analysis.ipynb # Jupyter notebook
    ├── results/
    │   ├── figures/                 # Generated plots
    │   └── tables/                  # Generated tables
    └── docs/
        └── analysis_plan.md         # Detailed methodology

8.7 PEER REVIEW GUIDELINES
--------------------------------------------------------------------------------

As reviewer or author, understanding peer review standards ensures quality.

[OK] REVIEWER RESPONSIBILITIES:

    SCIENTIFIC RIGOR
    ☑ Are methods sufficiently detailed to reproduce?
    ☑ Is sample size justified (power analysis)?
    ☑ Are statistical methods appropriate?
    ☑ Are conclusions supported by data?
    ☑ Are limitations acknowledged?
    ☑ Is prior work properly cited?

    DATA QUALITY
    ☑ Are controls appropriate?
    ☑ Are error bars clearly defined (SD, SEM, CI)?
    ☑ Are individual data points shown or available?
    ☑ Is data analysis transparent?
    ☑ Are datasets deposited in repository?

    CRYOPRESERVATION-SPECIFIC
    ☑ Is CPA formulation completely specified?
    ☑ Are cooling/warming rates documented?
    ☑ Is viability assessment timing stated?
    ☑ Are functional assays appropriate for cell type?
    ☑ Is storage duration and temperature recorded?

[CRYO] REVIEWER CHECKLIST (WIA-CRYO-010 COMPLIANCE):

    EXPERIMENTAL DESIGN
    [ ] Subject classification uses standard taxonomy (Section 2.3)
    [ ] Hypothesis pre-registered or clearly stated (Section 2.5)
    [ ] Sample size justified with power analysis (Section 6.6)

    MATERIALS AND METHODS
    [ ] CPA formulation documented per Section 3.2
        - All components with CAS#, supplier, lot#
        - Concentrations with units
        - Preparation protocol
    [ ] Cooling protocol per Section 4.2
        - Method, rate, equipment
        - Start/end temperatures
    [ ] Revival protocol per Section 4.2-4.4
        - Warming method and rate
        - CPA removal steps
    [ ] Viability assessment per Section 5.1-5.2
        - Method, timing, calculation

    DATA PRESENTATION
    [ ] Viability reported with confidence intervals (Section 5.6)
    [ ] Statistical tests appropriate and justified (Section 5.6)
    [ ] Individual data points shown or available
    [ ] Negative results (if any) reported

    DATA AVAILABILITY
    [ ] Raw data deposited with DOI (Section 8.3)
    [ ] Analysis code shared (Section 8.6)
    [ ] Protocols detailed in supplementary materials (Section 8.2)

8.8 POST-PUBLICATION UPDATES
--------------------------------------------------------------------------------

Science progresses; corrections and updates maintain research integrity.

[!] TYPES OF POST-PUBLICATION UPDATES:

    CORRECTION
    ┌──────────────────────────────────────────────────────────────────┐
    │ For: Minor errors not affecting conclusions                      │
    │ Examples:                                                        │
    │ - Typo in table                                                  │
    │ - Incorrect author affiliation                                   │
    │ - Missing acknowledgment                                         │
    │ Process: Submit correction to journal                            │
    │ Timeline: 1-2 months                                             │
    └──────────────────────────────────────────────────────────────────┘

    ERRATUM
    ┌──────────────────────────────────────────────────────────────────┐
    │ For: Errors affecting results but not conclusions                │
    │ Examples:                                                        │
    │ - Calculation error in one analysis                              │
    │ - Mislabeled figure panel                                        │
    │ - Incorrect statistics (but same conclusion)                     │
    │ Process: Formal erratum published                                │
    │ Timeline: 2-3 months                                             │
    └──────────────────────────────────────────────────────────────────┘

    RETRACTION
    ┌──────────────────────────────────────────────────────────────────┐
    │ For: Major errors invalidating conclusions                       │
    │ Examples:                                                        │
    │ - Fundamental methodological flaw                                │
    │ - Contaminated samples                                           │
    │ - Fabricated data                                                │
    │ Process: Retraction notice with explanation                      │
    │ Impact: Serious consequences                                     │
    └──────────────────────────────────────────────────────────────────┘

    DATASET VERSION UPDATE
    ┌──────────────────────────────────────────────────────────────────┐
    │ For: Adding new data or correcting dataset errors                │
    │ Process:                                                         │
    │ 1. Upload new version to repository                              │
    │ 2. Increment version number (v1.0 → v1.1)                        │
    │ 3. Document changes in version notes                             │
    │ 4. Previous versions remain accessible                           │
    │ Example DOI: 10.5281/zenodo.123456 (all versions)               │
    │              10.5281/zenodo.123457 (version 1.1)                 │
    └──────────────────────────────────────────────────────────────────┘

[CRYO] POST-PUBLICATION PROTOCOL:

    IF ERROR DISCOVERED:
    1. Assess severity immediately
    2. Consult co-authors
    3. Notify journal editor if publication affected
    4. Update dataset in repository if data error
    5. Post correction on preprint server if applicable
    6. Update institutional repository
    7. Notify anyone who requested data
    8. Document in research records

    TRANSPARENCY PRINCIPLE:
    "Better to publish a correction than hide an error. Scientific
    integrity requires acknowledging and fixing mistakes promptly."

CHAPTER SUMMARY
--------------------------------------------------------------------------------

This final chapter covered the complete publication lifecycle from data
preparation through post-publication updates.

Key Takeaways:

  [OK] 1. DATA RELEASE
       Organized file structure with README, data dictionary, raw/processed
       data, analysis code, and documentation enables reproducibility.

  [OK] 2. SUPPLEMENTARY MATERIALS
       Journal-compliant formats (PDF, CSV, etc.) with clear naming and
       comprehensive content (protocols, data, code, negative results).

  [OK] 3. DOI SYSTEM
       Persistent identifiers from Zenodo, Dryad, or other repositories
       enable citation, discovery, and impact tracking.

  [OK] 4. REPRODUCIBILITY
       Checklists ensure sufficient methodological detail (materials with
       lot numbers, complete protocols, statistical methods, code).

  [OK] 5. RAW DATA ARCHIVAL
       Long-term preservation in open formats (CSV, TIFF, PDF/A) with
       multiple backups and format migration plans.

  [OK] 6. CODE SHARING
       GitHub/GitLab repositories with documentation, dependencies,
       examples, and licenses enable computational reproducibility.

  [OK] 7. PEER REVIEW
       Standardized checklists for WIA-CRYO-010 compliance ensure quality
       and completeness of cryopreservation research publications.

  [OK] 8. POST-PUBLICATION UPDATES
       Corrections, errata, retractions, and dataset version updates
       maintain scientific integrity when errors are discovered.
