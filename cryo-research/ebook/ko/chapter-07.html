================================================================================
제07장: WIA-CRYO-010 냉동보존 연구 데이터 표준
================================================================================본 장은 영문 Chapter 7의 한국어 번역본입니다. 
WIA-CRYO-010 표준의 일부로 냉동보존 연구 데이터 관리에 대한 포괄적인 지침을 제공합니다.

弘益人間 (홍익인간) - 널리 인간을 이롭게 하라

이 철학은 모든 WIA 표준의 핵심입니다. 냉동보존 연구 데이터의 표준화를 통해
우리는 전 세계 연구자들이 협력하고, 재현 가능한 과학을 수행하며, 인류에게
이익이 되는 발견을 가속화할 수 있도록 합니다.

--------------------------------------------------------------------------------
주요 내용 (이 장의 핵심 개념)
--------------------------------------------------------------------------------

이 장에서는 냉동보존 연구의 다음 측면을 다룹니다:

[체크] 표준화된 데이터 형식과 스키마
[체크] 메타데이터 요구 사항 및 구조
[체크] 재현 가능한 연구를 위한 문서화 프로토콜
[체크] 품질 관리 및 검증 절차
[체크] 기관 간 데이터 공유 방법
[체크] 통계 분석 및 보고 표준
[체크] 윤리적 고려 사항 및 개인정보 보호
[체크] 장기 데이터 보존 및 접근성

--------------------------------------------------------------------------------

| 분산된 데이터         | 통찰력 상실              | 통합 데이터베이스       |
| 다양한 측정 기준      | 비교 불가능한 연구        | 공통 측정               |
| 사일로화된 연구       | 중복 노력                | 데이터 공유             |
| 출판 편향            | 숨겨진 실패              | 포괄적 보고             |
+--------------------------------------------------------------------------+

데이터 품질 보증
--------------------------------------------------------------------------------

품질 관리는 데이터 무결성을 보장합니다:

실험 전 점검:
☑ 장비 교정 확인
☑ 시약 품질 테스트 (삼투압, pH)
☑ 대조군 샘플 준비
☑ 데이터 수집 시스템 테스트
☑ 프로토콜 검토 및 승인

실험 중 점검:
☑ 실시간 온도 모니터링
☑ 자동 범위 검사 (사양 외 값에 대한 경고)
☑ 중간 생존력 검사
☑ 대조군 추적
☑ 데이터 백업 확인

실험 후 점검:
☑ 데이터 완전성 검사
☑ 통계적 이상값 탐지
☑ 대조군 검증
☑ 측정 일관성 검토
☑ 메타데이터 완전성 확인

--------------------------------------------------------------------------------
통계 분석 표준
--------------------------------------------------------------------------------

엄격한 통계 분석은 유효한 결론을 보장합니다:

표본 크기 요구 사항:
+--------------------------------------------------------------------------+
| 연구 유형          | 최소 n  | 권장    | 근거                          |
+--------------------------------------------------------------------------+
| 파일럿/타당성       | 3       | 6       | 주요 문제 탐지                 |
| 최적화             | 6       | 10-12   | 조건 비교                     |
| 검증               | 20      | 30+     | 통계적 검정력                  |
| 임상               | 50      | 100+    | 규제 요구 사항                 |
+--------------------------------------------------------------------------+

통계 검정:
+--------------------------------------------------------------------------+
| 비교               | 검정              | 요구 사항                      |
+--------------------------------------------------------------------------+
| 두 그룹            | t-검정            | 정규 분포                      |
| 다중 그룹          | ANOVA + 사후 검정  | 정규, 등분산                   |
| 비모수적           | Mann-Whitney      | 모든 분포                      |
| 쌍 데이터          | 쌍 t-검정         | 이전/이후                      |
| 비율               | 카이제곱          | 범주형 데이터                  |
+--------------------------------------------------------------------------+

보고 요구 사항:
☑ 평균 ± 표준 편차 (또는 정당화가 있는 SEM)
☑ 각 그룹의 표본 크기 (n)
☑ 사용된 통계 검정
☑ P-값 (p<0.05가 아닌 정확한 값 보고)
☑ 신뢰 구간 (95% CI 권장)
☑ 효과 크기 (Cohen's d 또는 유사)

--------------------------------------------------------------------------------
데이터 공유 및 협력
--------------------------------------------------------------------------------

기관 간 협력은 상호 운용성이 필요합니다:

상호 운용성 계층:

계층 1: 데이터 형식
   - 표준: WIA-CRYO-010 컨텍스트가 있는 JSON-LD
   - 인코딩: UTF-8
   - 스키마: 검증을 위한 JSON 스키마
   - 대체 형식: CSV, XML (레거시 시스템용)

계층 2: 의미론적 상호 운용성
   - 제어 어휘:
     • 종: NCBI 분류학
     • 해부학: Uberon 온톨로지
     • 세포 유형: 세포 온톨로지
     • 분석: 생의학 조사를 위한 온톨로지 (OBI)
     • 단위: QUDT

계층 3: API 인터페이스
   - 프로토콜: RESTful HTTP/HTTPS
   - 인증: OAuth 2.0
   - 쿼리 언어: GraphQL 또는 OData
   - 페이지 매김: 큰 데이터셋용 커서 기반
   - 속도 제한: 남용 방지

계층 4: 조직적
   - 데이터 공유 계약
   - 윤리/IRB 승인
   - 데이터 거버넌스 정책
   - 접근 요청 절차

장 요약
--------------------------------------------------------------------------------

이 장에서 다룬 핵심 개념:

[확인] 1. 데이터 표준화의 중요성
       일관된 문서화는 재현성을 가능하게 하고 과학적 발전을 가속화합니다.

[확인] 2. 구조화된 데이터 형식
       JSON-LD 및 표준 스키마는 기계 판독 가능성과 상호 운용성을 제공합니다.

[확인] 3. 품질 보증
       다단계 QC (실험 전, 실험 중, 실험 후)는 데이터 무결성을 보장합니다.

[확인] 4. 통계적 엄격성
       적절한 표본 크기, 검정 및 보고는 유효한 결론을 보장합니다.

[확인] 5. 개인정보 보호
       비식별화 및 윤리적 관행은 인간 대상을 보호합니다.

[확인] 6. 데이터 공유
       표준화된 형식과 API는 협력을 가능하게 합니다.

[확인] 7. 장기 보존
       여러 위치와 형식 마이그레이션은 영구 접근을 보장합니다.
