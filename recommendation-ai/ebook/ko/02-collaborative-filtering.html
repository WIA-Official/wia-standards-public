🤝 2장: 협업 필터링

🎯 집단 지성의 힘

협업 필터링은 간단하지만 강력한 아이디어에 기반합니다: 과거에 동의한 사람들은 미래에도 동의하는 경향이 있습니다. 사용자 A와 B가 모두 영화 X, Y, Z를 좋아했고, 사용자 A가 영화 W도 좋아했다면, 사용자 B도 영화 W를 좋아할 것입니다.

핵심 통찰: 협업 필터링은 집단 지성을 활용합니다. 아이템이 무엇인지가 아니라 사람들이 어떻게 상호작용하는지에 관한 것입니다.

📊 사용자 기반 협업 필터링

사용자 기반 CF는 대상 사용자와 유사한 사용자를 찾고 그 유사한 사용자가 좋아한 아이템을 추천합니다.

3단계 프로세스:

  • 1단계: 유사한 사용자 찾기 (코사인 유사도, 피어슨 상관계수)
  • 2단계: 유사도로 가중치를 둔 평점 집계
  • 3단계: 예측 평점이 높은 아이템 추천

🎬 아이템 기반 협업 필터링

사용자가 아이템 A를 좋아했다면, 사용자 평점 패턴을 기반으로 A와 유사한 아이템을 추천합니다.

아이템 기반이 사용자 기반보다 나은 이유:

  • 안정성: 아이템 유사도는 사용자 선호도보다 덜 자주 변경됩니다
  • 확장성: 일반적으로 사용자보다 아이템이 적어 사전 계산이 가능합니다
  • 설명 가능성: "X를 좋아한 사용자가 Y도 좋아했습니다"를 설명하기 쉽습니다
  • 희소성: 희소한 데이터 행렬을 더 잘 처리합니다
// 아이템 기반 협업 필터링 class ItemBasedCF { private itemSimilarities: Map>; // 아이템-아이템 유사도 사전 계산 precomputeSimilarities(ratings: Map>): void { // 모든 아이템 쌍에 대해 유사도 계산 // 캐싱하여 빠른 추천 생성 } // 사용자를 위한 추천 생성 recommend(userId: string, k: number = 10): Array<{item: string, score: number}> { // 사용자가 평가한 각 아이템에 대해 // 유사한 아이템을 찾고 점수 집계 // 상위 k개 반환 } }

⚖️ 유사도 메트릭

  • 코사인 유사도: 벡터 각도 측정, 희소 데이터에 적합
  • 피어슨 상관계수: 선형 관계, 평점 스케일 처리
  • 자카드 유사도: 집합 교집합, 이진 데이터용
  • 유클리드 거리: 기하학적 거리, 밀집 데이터용

🎯 희소 데이터 처리

실제 평점 행렬은 일반적으로 99% 이상 희소합니다. 대부분의 사용자는 매우 적은 수의 아이템만 평가합니다.

과제:

  • 제한된 중복: 두 사용자가 유사하지만 공동 평가 아이템이 거의 없음
  • 콜드 스타트: 신규 사용자나 아이템은 평점이 없어 추천이 불가능
  • 계산 비용: 희소 행렬은 특수 데이터 구조와 알고리즘이 필요

🚀 최적화 기법

  • 차원 축소: SVD를 사용하여 평점 행렬을 저차원 공간으로 축소
  • 근사 최근접 이웃: 빠른 근사 결과를 위한 ANN 알고리즘
  • 증분 업데이트: 처음부터 다시 계산하지 않고 새 평점에 따라 모델 업데이트
  • 캐싱: 유사도 계산 및 인기 아이템/사용자에 대한 추천 캐시

⚠️ 한계 및 과제

  • 인기 편향: 인기 있는 아이템이 더 많이 추천되어 다양성 감소
  • 필터 버블: 사용자가 이미 좋아한 것과 유사한 아이템만 보게 됨
  • 시링 공격: 악의적인 사용자가 가짜 프로필로 추천 조작
  • 회색 양 문제: 고유한 취향을 가진 사용자는 좋은 추천을 받지 못함

📝 요약

  • 협업 필터링은 집단 사용자 행동을 활용하여 추천합니다
  • 사용자 기반 CF는 유사한 사용자를, 아이템 기반 CF는 유사한 아이템을 찾습니다
  • 아이템 기반은 더 나은 안정성과 확장성으로 사용자 기반보다 우수합니다
  • 희소성은 특수 데이터 구조가 필요한 근본적인 과제입니다
  • 최적화에는 캐싱, 차원 축소, 근사 이웃이 포함됩니다
  • 한계에는 인기 편향, 필터 버블, 콜드 스타트 문제가 포함됩니다
