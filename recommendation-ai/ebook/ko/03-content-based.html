📊 3장: 콘텐츠 기반 필터링

🎯 아이템 특징 이해

콘텐츠 기반 필터링은 아이템의 고유 특징을 분석하고 사용자 선호도와 매칭하여 추천합니다. 군중의 지혜에 의존하는 협업 필터링과 달리, "이 아이템의 어떤 속성이 이 사용자에게 매력적인가?"를 묻습니다.

🏗️ 아이템 프로필 구축

구조화된 특징

  • 영화: 장르, 감독, 배우, 연도
  • 제품: 카테고리, 브랜드, 가격대, 사양
  • 음악: 장르, 아티스트, 템포, 키

텍스트 특징 (TF-IDF)

설명이나 리뷰와 같은 텍스트 콘텐츠의 경우 TF-IDF(Term Frequency-Inverse Document Frequency)를 사용하여 의미 있는 특징을 추출합니다.

👤 사용자 프로필 구축

아이템이 특징 벡터로 표현되면, 상호작용한 아이템의 특징을 집계하여 사용자 프로필을 구축합니다.

📏 유사도 계산

  • 코사인 유사도: 희소, 고차원 벡터용
  • 유클리드 거리: 밀집, 저차원 데이터용
  • 피어슨 상관: 평점 예측용
  • 자카드: 집합 기반 특징용

🔍 고급 특징 엔지니어링

  • 단어 임베딩: TF-IDF 대신 Word2Vec, BERT 사용
  • 이미지 특징: 시각적 아이템을 위한 CNN
  • 오디오 특징: 음악을 위한 MFCC
  • 지식 그래프: 관계와 엔티티로 아이템 표현 강화

⚖️ 과전문화 문제 해결

콘텐츠 기반 필터링의 주요 한계는 과전문화입니다: 사용자가 이미 본 것과 유사한 추천만 받아 필터 버블을 만듭니다.

완화 전략:

  • 우연성 주입: 가끔 다양한 아이템 추천
  • 시간적 감쇠: 오래된 선호도에 덜 가중치 부여
  • 부정적 피드백: 사용자가 싫어하는 것 학습
  • 탐색-활용: 익숙한 것과 새로운 것 균형

🎯 하이브리드 콘텐츠 접근법

여러 콘텐츠 기반 기법을 결합하면 더 나은 결과를 얻는 경우가 많습니다.

📝 요약

  • 콘텐츠 기반 필터링은 아이템 특징을 사용자 선호도와 매칭
  • 특징 추출은 아이템을 벡터 표현으로 변환
  • 사용자 프로필은 좋아한 아이템의 특징을 집계
  • 주요 한계는 필터 버블로 이어지는 과전문화
  • 콘텐츠 기반은 풍부한 메타데이터가 있고 콜드 스타트가 문제일 때 우수
