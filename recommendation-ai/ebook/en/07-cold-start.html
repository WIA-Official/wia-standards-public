🆕 Chapter 7: Cold Start Problem

WIA-AI-024 Recommendation AI Standard

🎯 The Bootstrap Challenge

The cold start problem is one of the most persistent challenges in recommendation systems: how do you recommend items to new users with no interaction history? How do you recommend new items that no one has rated yet? This chicken-and-egg problem affects every recommendation system.

There are three types of cold start scenarios: new users, new items, and completely new systems. Each requires different strategies, and production systems typically employ multiple approaches simultaneously.

Key Challenge: Collaborative filtering fails completely for cold start scenarios because it depends on interaction history. Solutions must leverage alternative information sources.

👤 New User Cold Start

When a user first joins your platform, you have no data about their preferences. How do you make that crucial first impression?

Strategy 1: Popularity-Based Recommendations

Start with what works for most people—popular items, trending content, or bestsellers:

// Popularity-based cold start recommender class PopularityRecommender { private popularityScores: Map; constructor() { this.popularityScores = new Map(); } // Calculate popularity from interactions calculatePopularity( interactions: Array<{userId: string, itemId: string, weight?: number}> ): void { const itemCounts = new Map(); for (const interaction of interactions) { const weight = interaction.weight || 1; const current = itemCounts.get(interaction.itemId) || 0; itemCounts.set(interaction.itemId, current + weight); } // Normalize scores const maxCount = Math.max(...itemCounts.values()); for (const [itemId, count] of itemCounts) { this.popularityScores.set(itemId, count / maxCount); } } // Get trending items (time-decayed popularity) getTrending( interactions: Array<{itemId: string, timestamp: Date}>, decayDays: number = 7, k: number = 10 ): string[] { const now = Date.now(); const decayMs = decayDays * 24 * 60 * 60 * 1000; const scores = new Map(); for (const {itemId, timestamp} of interactions) { const age = now - timestamp.getTime(); const decayFactor = Math.exp(-age / decayMs); const current = scores.get(itemId) || 0; scores.set(itemId, current + decayFactor); } return Array.from(scores.entries()) .sort(([, a], [, b]) => b - a) .slice(0, k) .map(([itemId]) => itemId); } recommend(k: number = 10): string[] { return Array.from(this.popularityScores.entries()) .sort(([, a], [, b]) => b - a) .slice(0, k) .map(([itemId]) => itemId); } }

Strategy 2: Onboarding Questionnaire

Ask users about their preferences explicitly during signup:

// Questionnaire-based profile builder interface OnboardingResponse { categories: string[]; // Preferred categories favoriteItems?: string[]; // Known items they like dislikes?: string[]; // Items/categories to avoid demographics?: { age?: number; location?: string; gender?: string; }; } class OnboardingRecommender { buildInitialProfile( response: OnboardingResponse, categoryItemMap: Map, itemFeatures: Map ): { profileVector: number[], blacklist: Set } { const profileVector: number[] = []; const blacklist = new Set(response.dislikes || []); // Build profile from liked items if (response.favoriteItems && response.favoriteItems.length > 0) { const likedVectors = response.favoriteItems .map(id => itemFeatures.get(id)) .filter(v => v !== undefined) as number[][]; if (likedVectors.length > 0) { // Average feature vectors for (let i = 0; i < likedVectors[0].length; i++) { const avg = likedVectors.reduce((sum, v) => sum + v[i], 0) / likedVectors.length; profileVector.push(avg); } } } return { profileVector, blacklist }; } recommendFromOnboarding( response: OnboardingResponse, allItems: string[], categoryMap: Map, k: number = 10 ): string[] { // Gather candidate items from preferred categories const candidates = new Set(); for (const category of response.categories) { const categoryItems = categoryMap.get(category) || []; categoryItems.forEach(item => candidates.add(item)); } // Remove dislikes const dislikes = new Set(response.dislikes || []); const filtered = Array.from(candidates).filter(item => !dislikes.has(item)); // Return top k (could add additional ranking logic) return filtered.slice(0, k); } }

Strategy 3: Demographic-Based Recommendations

Use demographic information (age, location, gender) to find similar users:

// Demographic-based recommender interface UserDemographics { age: number; location: string; gender: string; } class DemographicRecommender { findSimilarUsers( newUser: UserDemographics, existingUsers: Map, k: number = 50 ): string[] { const similarities: Array<{userId: string, score: number}> = []; for (const [userId, demographics] of existingUsers) { const score = this.calculateDemographicSimilarity(newUser, demographics); similarities.push({ userId, score }); } return similarities .sort((a, b) => b.score - a.score) .slice(0, k) .map(s => s.userId); } private calculateDemographicSimilarity( user1: UserDemographics, user2: UserDemographics ): number { let score = 0; // Age similarity (closer ages = higher similarity) const ageDiff = Math.abs(user1.age - user2.age); const ageScore = Math.exp(-ageDiff / 10); // Decay function score += ageScore * 0.3; // Location match if (user1.location === user2.location) { score += 0.4; } // Gender match if (user1.gender === user2.gender) { score += 0.3; } return score; } recommendFromDemographics( newUser: UserDemographics, existingUsers: Map, userPreferences: Map, k: number = 10 ): string[] { // Find similar users const similarUsers = this.findSimilarUsers(newUser, existingUsers, 50); // Aggregate their preferences const itemScores = new Map(); for (const userId of similarUsers) { const preferences = userPreferences.get(userId) || []; for (const itemId of preferences) { const current = itemScores.get(itemId) || 0; itemScores.set(itemId, current + 1); } } return Array.from(itemScores.entries()) .sort(([, a], [, b]) => b - a) .slice(0, k) .map(([itemId]) => itemId); } }

📦 New Item Cold Start

How do you recommend a brand new movie, product, or article that no one has interacted with yet?

Strategy 1: Content-Based Features

Use item metadata to find similar items:

Movies Genre, director, actors, plot TF-IDF, embeddings
Products Category, brand, specs, images CNN features, metadata
News Topics, entities, sentiment NLP, knowledge graphs

Strategy 2: Active Learning

Strategically show new items to diverse user groups to quickly gather feedback:

// Active learning for new items class ActiveLearningColdStart { // Select users to show new item for maximum information gain selectUsersForNewItem( newItemId: string, newItemFeatures: number[], users: Map, // User preference vectors alreadyShown: Set, k: number = 100 ): string[] { const candidates: Array<{userId: string, score: number}> = []; for (const [userId, userVector] of users) { if (alreadyShown.has(userId)) continue; // Score based on expected information gain // Higher score for users whose preferences match item features const similarity = this.cosineSimilarity(userVector, newItemFeatures); // Also consider user diversity (not all similar users) const diversityBonus = this.calculateDiversityScore(userId, Array.from(alreadyShown), users); const score = similarity * 0.7 + diversityBonus * 0.3; candidates.push({ userId, score }); } return candidates .sort((a, b) => b.score - a.score) .slice(0, k) .map(c => c.userId); } private cosineSimilarity(a: number[], b: number[]): number { let dot = 0, magA = 0, magB = 0; for (let i = 0; i < a.length; i++) { dot += a[i] * b[i]; magA += a[i] * a[i]; magB += b[i] * b[i]; } return dot / (Math.sqrt(magA) * Math.sqrt(magB) + 1e-8); } private calculateDiversityScore( userId: string, shownUsers: string[], allUsers: Map ): number { if (shownUsers.length === 0) return 1; const userVec = allUsers.get(userId); if (!userVec) return 0; // Calculate average similarity to already shown users let avgSimilarity = 0; for (const shownUserId of shownUsers) { const shownVec = allUsers.get(shownUserId); if (shownVec) { avgSimilarity += this.cosineSimilarity(userVec, shownVec); } } avgSimilarity /= shownUsers.length; // Return diversity (inverse of similarity) return 1 - avgSimilarity; } }

Strategy 3: Expert Curation

🌟 Hybrid Cold Start Strategies

The most effective approach combines multiple strategies:

🎵 Spotify's New User Experience
  1. Onboarding: Ask for favorite artists and genres
  2. Popularity: Show trending playlists in selected genres
  3. Content-based: Recommend similar artists to favorites
  4. Active learning: Ask for thumbs up/down on initial recommendations
  5. Quick adaptation: Update recommendations after every interaction

🔄 From Cold to Warm: Transition Strategies

As users accumulate interactions, smoothly transition from cold start methods to collaborative filtering:

// Hybrid recommender that transitions from cold to warm class TransitionRecommender { recommendHybrid( userId: string, interactionCount: number, demographics: UserDemographics, recentInteractions: string[], k: number = 10 ): string[] { // Weight different strategies based on interaction count const coldWeight = Math.exp(-interactionCount / 10); // Decay as user interacts const warmWeight = 1 - coldWeight; let recommendations: string[] = []; if (coldWeight > 0.5) { // Still mostly cold - use demographic + popularity const demographic = this.getDemographicRecs(demographics, k); const popular = this.getPopularRecs(k); recommendations = this.merge(demographic, popular, 0.6, 0.4, k); } else if (warmWeight > 0.5) { // Warming up - use content-based + early collaborative const contentBased = this.getContentBasedRecs(recentInteractions, k); const collaborative = this.getCollaborativeRecs(userId, k); recommendations = this.merge(contentBased, collaborative, coldWeight, warmWeight, k); } else { // Fully warm - primarily collaborative with content-based fallback const collaborative = this.getCollaborativeRecs(userId, k); const contentBased = this.getContentBasedRecs(recentInteractions, Math.floor(k * 0.2)); recommendations = [...collaborative.slice(0, Math.floor(k * 0.8)), ...contentBased]; } return recommendations.slice(0, k); } private merge( list1: string[], list2: string[], weight1: number, weight2: number, k: number ): string[] { const count1 = Math.floor(k * weight1); const count2 = k - count1; const merged = [...list1.slice(0, count1), ...list2.slice(0, count2)]; return merged; } // Placeholder methods (implement with actual logic) private getDemographicRecs(demographics: UserDemographics, k: number): string[] { return []; } private getPopularRecs(k: number): string[] { return []; } private getContentBasedRecs(interactions: string[], k: number): string[] { return []; } private getCollaborativeRecs(userId: string, k: number): string[] { return []; } }

🎯 Multi-Armed Bandit Approach

Treat cold start as an exploration-exploitation problem using bandit algorithms:

  • Epsilon-greedy: With probability ε, explore random items; otherwise exploit best known
  • Upper Confidence Bound (UCB): Balance exploration of uncertain items with exploitation
  • Thompson Sampling: Bayesian approach sampling from posterior distributions

🌐 Cross-Domain Transfer Learning

Leverage data from related domains or platforms:

  • Social media connections predict content preferences
  • Movie preferences correlate with book tastes
  • Music listening patterns relate to podcast interests

⚠️ Best Practices

Do's

  • Make the first experience count—bad initial recommendations hurt retention
  • Ask for minimal information—long onboarding reduces conversion
  • Learn quickly from early interactions
  • Provide explanations—users are more forgiving when they understand
  • Offer control—let users customize or skip recommendations

Don'ts

  • Don't show only popular items—lacks personalization
  • Don't ignore implicit signals—views, time spent, skips all provide data
  • Don't treat all new users the same—demographics and context matter
  • Don't wait too long to personalize—adapt after first few interactions

📝 Summary

  • Cold start affects new users, new items, and new systems
  • Popularity-based recommendations provide safe defaults for new users
  • Onboarding questionnaires collect explicit preferences efficiently
  • Demographic information enables user similarity without interaction history
  • Content-based features allow new item recommendations immediately
  • Active learning strategically gathers feedback on new items
  • Hybrid approaches combine multiple strategies for robustness
  • Smooth transitions from cold to warm state improve user experience
