📊 Chapter 6: Evaluation Metrics

WIA-AI-024 Recommendation AI Standard

🎯 Measuring Success

"What gets measured gets improved." Evaluation metrics are critical for assessing recommendation quality, comparing algorithms, and optimizing for business objectives. However, no single metric captures all aspects of a good recommendation system.

This chapter explores accuracy metrics, ranking metrics, diversity measures, business metrics, and online evaluation methods. Understanding when to use each metric—and their limitations—is essential for building effective systems.

Key Principle: Optimize for what matters to users and business, not just what's easy to measure. High accuracy doesn't always mean better user experience.

📐 Accuracy Metrics

These metrics measure how well predicted ratings match actual user preferences.

Root Mean Square Error (RMSE)

Measures average prediction error for rating predictions:

RMSE = √(Σ(predicted - actual)² / n)
// RMSE calculation function calculateRMSE( predictions: Array<{predicted: number, actual: number}> ): number { const squaredErrors = predictions.map(p => Math.pow(p.predicted - p.actual, 2) ); const mse = squaredErrors.reduce((a, b) => a + b, 0) / predictions.length; return Math.sqrt(mse); } // Example usage const predictions = [ { predicted: 4.2, actual: 4.0 }, { predicted: 3.8, actual: 4.5 }, { predicted: 4.9, actual: 5.0 }, { predicted: 2.1, actual: 2.0 } ]; console.log(`RMSE: ${calculateRMSE(predictions).toFixed(3)}`); // Output: RMSE: 0.346

Mean Absolute Error (MAE)

Less sensitive to outliers than RMSE:

MAE = Σ|predicted - actual| / n

🎯 Classification Metrics

For binary recommendations (relevant vs. not relevant):

Precision

Of items recommended, what fraction are relevant?

Precision = True Positives / (True Positives + False Positives)

Recall

Of all relevant items, what fraction did we recommend?

Recall = True Positives / (True Positives + False Negatives)

F1 Score

Harmonic mean of precision and recall:

F1 = 2 × (Precision × Recall) / (Precision + Recall)
// Classification metrics calculator class ClassificationMetrics { static precision(recommended: Set, relevant: Set): number { const truePositives = this.intersection(recommended, relevant).size; return recommended.size > 0 ? truePositives / recommended.size : 0; } static recall(recommended: Set, relevant: Set): number { const truePositives = this.intersection(recommended, relevant).size; return relevant.size > 0 ? truePositives / relevant.size : 0; } static f1Score(precision: number, recall: number): number { if (precision + recall === 0) return 0; return 2 * (precision * recall) / (precision + recall); } static precisionAtK(recommended: string[], relevant: Set, k: number): number { const topK = new Set(recommended.slice(0, k)); return this.precision(topK, relevant); } static recallAtK(recommended: string[], relevant: Set, k: number): number { const topK = new Set(recommended.slice(0, k)); return this.recall(topK, relevant); } private static intersection(a: Set, b: Set): Set { return new Set([...a].filter(x => b.has(x))); } } // Example const recommended = ['item1', 'item2', 'item3', 'item4', 'item5']; const relevant = new Set(['item2', 'item5', 'item7', 'item9']); const p = ClassificationMetrics.precisionAtK(recommended, relevant, 5); const r = ClassificationMetrics.recallAtK(recommended, relevant, 5); const f1 = ClassificationMetrics.f1Score(p, r); console.log(`Precision@5: ${p.toFixed(3)}`); // 0.400 (2/5) console.log(`Recall@5: ${r.toFixed(3)}`); // 0.500 (2/4) console.log(`F1@5: ${f1.toFixed(3)}`); // 0.444

📊 Ranking Metrics

These metrics consider the order of recommendations, not just which items are included.

Normalized Discounted Cumulative Gain (NDCG)

The gold standard for ranking evaluation. It considers both relevance and position, with higher-ranked items contributing more to the score.

// NDCG implementation class RankingMetrics { static ndcgAtK( recommended: string[], relevanceScores: Map, k: number ): number { const dcg = this.dcgAtK(recommended, relevanceScores, k); const idcg = this.idealDCG(relevanceScores, k); return idcg > 0 ? dcg / idcg : 0; } private static dcgAtK( recommended: string[], relevanceScores: Map, k: number ): number { let dcg = 0; const items = recommended.slice(0, k); for (let i = 0; i < items.length; i++) { const relevance = relevanceScores.get(items[i]) || 0; const position = i + 1; dcg += relevance / Math.log2(position + 1); } return dcg; } private static idealDCG(relevanceScores: Map, k: number): number { const sortedRelevance = Array.from(relevanceScores.values()) .sort((a, b) => b - a) .slice(0, k); let idcg = 0; for (let i = 0; i < sortedRelevance.length; i++) { idcg += sortedRelevance[i] / Math.log2(i + 2); } return idcg; } // Mean Average Precision static mapAtK( recommended: string[], relevant: Set, k: number ): number { let sum = 0; let hits = 0; const items = recommended.slice(0, k); for (let i = 0; i < items.length; i++) { if (relevant.has(items[i])) { hits++; const precision = hits / (i + 1); sum += precision; } } return relevant.size > 0 ? sum / relevant.size : 0; } // Mean Reciprocal Rank static mrr(recommended: string[], relevant: Set): number { for (let i = 0; i < recommended.length; i++) { if (relevant.has(recommended[i])) { return 1 / (i + 1); } } return 0; } } // Example const rec = ['item1', 'item2', 'item3', 'item4']; const scores = new Map([ ['item1', 0], ['item2', 3], // Highly relevant ['item3', 1], // Somewhat relevant ['item4', 0] ]); const ndcg = RankingMetrics.ndcgAtK(rec, scores, 4); console.log(`NDCG@4: ${ndcg.toFixed(3)}`);

🌈 Beyond Accuracy: Diversity & Coverage

Accuracy alone can lead to filter bubbles and poor user experience. Diversity and coverage metrics ensure varied recommendations.

Intra-List Diversity

How different are items within a single recommendation list?

// Diversity metrics class DiversityMetrics { static intraListDiversity( items: string[], similarityFunc: (a: string, b: string) => number ): number { if (items.length < 2) return 0; let totalDissimilarity = 0; let comparisons = 0; for (let i = 0; i < items.length; i++) { for (let j = i + 1; j < items.length; j++) { const similarity = similarityFunc(items[i], items[j]); totalDissimilarity += (1 - similarity); comparisons++; } } return totalDissimilarity / comparisons; } // Catalog coverage: percentage of catalog recommended to at least one user static catalogCoverage( allRecommendations: string[][], catalogSize: number ): number { const recommendedItems = new Set(); for (const userRecs of allRecommendations) { userRecs.forEach(item => recommendedItems.add(item)); } return recommendedItems.size / catalogSize; } // Gini coefficient: measure of recommendation concentration static giniCoefficient(itemCounts: number[]): number { const sorted = itemCounts.slice().sort((a, b) => a - b); const n = sorted.length; let sum1 = 0, sum2 = 0; for (let i = 0; i < n; i++) { sum1 += (i + 1) * sorted[i]; sum2 += sorted[i]; } return (2 * sum1) / (n * sum2) - (n + 1) / n; } // Novelty: average popularity of recommended items (lower = more novel) static novelty( recommended: string[], popularityScores: Map ): number { if (recommended.length === 0) return 0; const totalPopularity = recommended.reduce((sum, item) => { return sum + (popularityScores.get(item) || 0); }, 0); return totalPopularity / recommended.length; } }

🧪 Offline vs Online Evaluation

Offline Evaluation

Uses historical data to evaluate models before deployment:

  • Pros: Fast, cheap, reproducible
  • Cons: May not reflect real user behavior, limited by data quality

Online Evaluation (A/B Testing)

Tests with real users in production:

  • Pros: Measures actual impact on user behavior
  • Cons: Slow, expensive, requires significant traffic
// A/B test analysis class ABTestAnalyzer { static calculateSignificance( controlMetrics: {conversions: number, impressions: number}, treatmentMetrics: {conversions: number, impressions: number} ): { controlRate: number, treatmentRate: number, lift: number, pValue: number, significant: boolean } { const controlRate = controlMetrics.conversions / controlMetrics.impressions; const treatmentRate = treatmentMetrics.conversions / treatmentMetrics.impressions; const lift = (treatmentRate - controlRate) / controlRate; // Z-test for proportion difference const p1 = controlRate; const p2 = treatmentRate; const n1 = controlMetrics.impressions; const n2 = treatmentMetrics.impressions; const pooledP = (controlMetrics.conversions + treatmentMetrics.conversions) / (n1 + n2); const se = Math.sqrt(pooledP * (1 - pooledP) * (1/n1 + 1/n2)); const zScore = (p2 - p1) / se; // Approximate p-value (two-tailed) const pValue = 2 * (1 - this.normalCDF(Math.abs(zScore))); return { controlRate, treatmentRate, lift, pValue, significant: pValue < 0.05 }; } private static normalCDF(z: number): number { // Approximation of standard normal CDF const t = 1 / (1 + 0.2316419 * Math.abs(z)); const d = 0.3989423 * Math.exp(-z * z / 2); const probability = d * t * (0.3193815 + t * (-0.3565638 + t * (1.781478 + t * (-1.821256 + t * 1.330274)))); return z > 0 ? 1 - probability : probability; } } // Example const result = ABTestAnalyzer.calculateSignificance( { conversions: 1000, impressions: 10000 }, // Control: 10% conversion { conversions: 1150, impressions: 10000 } // Treatment: 11.5% conversion ); console.log(`Control Rate: ${(result.controlRate * 100).toFixed(2)}%`); console.log(`Treatment Rate: ${(result.treatmentRate * 100).toFixed(2)}%`); console.log(`Lift: ${(result.lift * 100).toFixed(2)}%`); console.log(`P-Value: ${result.pValue.toFixed(4)}`); console.log(`Significant: ${result.significant}`);

⚖️ Choosing the Right Metrics

Select metrics based on your goals and constraints:

Goal Primary Metrics Secondary Metrics
Accuracy RMSE, NDCG, Precision@K F1, MAP
Discovery Novelty, Serendipity Coverage, Diversity
Engagement CTR, Watch Time Session Length, Return Rate
Revenue Conversion Rate, RPR AOV, LTV
⚠️ Common Pitfalls
  • Optimizing for the wrong metric: High RMSE doesn't guarantee user satisfaction
  • Ignoring diversity: Accurate but repetitive recommendations harm discovery
  • Short-term focus: Maximizing immediate clicks may hurt long-term engagement
  • Statistical significance: Small sample sizes can lead to false conclusions

🔄 Multi-Objective Optimization

Real systems must balance multiple objectives:

  • Accuracy vs. Diversity
  • Short-term engagement vs. Long-term retention
  • User satisfaction vs. Business revenue

Approaches include Pareto optimization, weighted combinations, and constraint-based methods.

📝 Summary

  • No single metric captures all aspects of recommendation quality
  • Accuracy metrics (RMSE, Precision, Recall) measure prediction correctness
  • Ranking metrics (NDCG, MAP, MRR) consider item order importance
  • Diversity and coverage metrics prevent filter bubbles
  • Business metrics (CTR, conversion, revenue) measure real-world impact
  • Offline evaluation is fast but limited; online A/B testing measures true impact
  • Choose metrics aligned with business goals and user needs
  • Multi-objective optimization balances competing priorities
