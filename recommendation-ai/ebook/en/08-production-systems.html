🚀 Chapter 8: Production Recommendation Systems

WIA-AI-024 Recommendation AI Standard

🎯 From Research to Reality

Building recommendation systems in production is fundamentally different from research prototypes. You must handle millions of users, serve recommendations in milliseconds, adapt to changing preferences in real-time, monitor for failures, and ensure fairness and privacy—all while maintaining 99.9% uptime.

This chapter covers the architecture, infrastructure, deployment strategies, monitoring, and operational considerations for production recommendation systems at scale.

Core Principle: Production systems prioritize reliability, latency, and maintainability over model complexity. A simple model that serves consistently beats a sophisticated model that fails.

🏗️ System Architecture

Modern recommendation systems follow a multi-stage architecture:

Stage 1: Candidate Generation

Quickly retrieve hundreds to thousands of candidates from millions of items. Uses fast retrieval methods like nearest neighbor search, collaborative filtering, or simple rules.

Stage 2: Ranking

Apply more sophisticated models to score and rank candidates. Can use deep learning, ensemble methods, or complex features.

Stage 3: Filtering & Business Logic

Apply business rules, diversity constraints, legal requirements, and user preferences.

Stage 4: Serving

Return formatted results with explanations, metadata, and tracking information.

// Production recommendation pipeline interface RecommendationRequest { userId: string; context: { device: string; location?: string; timeOfDay: string; sessionId: string; }; count: number; filters?: { categories?: string[]; priceRange?: [number, number]; excludeItems?: string[]; }; } interface RecommendationResponse { items: Array<{ itemId: string; score: number; reason: string; metadata: any; }>; requestId: string; latencyMs: number; } class ProductionRecommendationPipeline { private candidateGenerator: CandidateGenerator; private ranker: Ranker; private filterEngine: FilterEngine; private logger: Logger; private cache: Cache; async recommend(request: RecommendationRequest): Promise { const startTime = Date.now(); const requestId = this.generateRequestId(); try { // Try cache first const cached = await this.cache.get(this.getCacheKey(request)); if (cached) { this.logger.info('Cache hit', { requestId, userId: request.userId }); return cached; } // Stage 1: Candidate Generation (100-1000 items) const candidates = await this.candidateGenerator.generate( request.userId, request.context, 1000 ); // Stage 2: Ranking (score all candidates) const ranked = await this.ranker.rank( request.userId, candidates, request.context ); // Stage 3: Filtering (apply business rules) const filtered = this.filterEngine.apply(ranked, request.filters); // Stage 4: Prepare response const topItems = filtered.slice(0, request.count); const response: RecommendationResponse = { items: topItems.map(item => ({ itemId: item.id, score: item.score, reason: item.explanation, metadata: item.metadata })), requestId, latencyMs: Date.now() - startTime }; // Cache result await this.cache.set(this.getCacheKey(request), response, 300); // 5 min TTL // Log metrics this.logger.info('Recommendation served', { requestId, userId: request.userId, latencyMs: response.latencyMs, itemCount: response.items.length }); return response; } catch (error) { this.logger.error('Recommendation failed', { requestId, userId: request.userId, error }); // Fallback to popular items return this.getFallbackRecommendations(request, requestId); } } private generateRequestId(): string { return `req_${Date.now()}_${Math.random().toString(36).substr(2, 9)}`; } private getCacheKey(request: RecommendationRequest): string { return `rec:${request.userId}:${request.count}:${JSON.stringify(request.filters || {})}`; } private getFallbackRecommendations( request: RecommendationRequest, requestId: string ): RecommendationResponse { // Return popular items as fallback return { items: [], requestId, latencyMs: 0 }; } } // Placeholder classes class CandidateGenerator { async generate(userId: string, context: any, count: number): Promise { return []; ⚡ Latency Optimization

Users expect recommendations in 50-200ms. Achieving this requires multiple strategies:

1. Caching Strategies

  • User-level cache: Cache recommendations per user (short TTL)
  • Item-level cache: Cache item features and similarities
  • Model cache: Cache model predictions for common inputs
  • Partial computation: Precompute item embeddings, similarities

2. Approximate Methods

  • ANN (Approximate Nearest Neighbors): FAISS, Annoy, HNSW
  • Quantization: Reduce precision for faster computation
  • Model compression: Distillation, pruning, quantization

3. Asynchronous Processing

  • Pre-computation: Generate recommendations offline for known users
  • Background updates: Update models without blocking requests
  • Event-driven: Trigger recomputation on user actions
// Fast nearest neighbor search using approximation class FastSimilaritySearch { private index: Map; private buckets: Map>; private numBuckets: number = 100; constructor() { this.index = new Map(); this.buckets = new Map(); } // Locality-Sensitive Hashing for fast approximate search addItem(itemId: string, vector: number[]): void { this.index.set(itemId, vector); // Hash to buckets const hash = this.lshHash(vector); if (!this.buckets.has(hash)) { this.buckets.set(hash, new Set()); } this.buckets.get(hash)!.add(itemId); } // Fast approximate k-NN search findSimilar(queryVector: number[], k: number = 10): Array<{itemId: string, similarity: number}> { const hash = this.lshHash(queryVector); const candidates = this.buckets.get(hash) || new Set(); // If bucket is empty or too small, check neighboring buckets if (candidates.size < k * 2) { // Expand search to all items (fallback) candidates.clear(); for (const itemId of this.index.keys()) { candidates.add(itemId); } } // Compute exact similarities for candidates const similarities: Array<{itemId: string, similarity: number}> = []; for (const itemId of candidates) { const itemVector = this.index.get(itemId)!; const sim = this.cosineSimilarity(queryVector, itemVector); similarities.push({ itemId, similarity: sim }); } return similarities .sort((a, b) => b.similarity - a.similarity) .slice(0, k); } private lshHash(vector: number[]): string { // Simple LSH: project onto random hyperplanes let hash = ''; for (let i = 0; i < 10; i++) { const projection = vector.reduce((sum, val, idx) => { // Use deterministic pseudo-random projection const weight = Math.sin(idx * i); return sum + val * weight; }, 0); hash += projection > 0 ? '1' : '0'; } return hash; } private cosineSimilarity(a: number[], b: number[]): number { let dot = 0, magA = 0, magB = 0; for (let i = 0; i < a.length; i++) { dot += a[i] * b[i]; magA += a[i] * a[i]; magB += b[i] * b[i]; } return dot / (Math.sqrt(magA) * Math.sqrt(magB) + 1e-8); } }

🔄 Real-Time Updates

User preferences change constantly. Production systems must adapt:

Online Learning

Update models incrementally as new data arrives without full retraining:

// Online learning for recommendations class OnlineRecommender { private userFactors: Map; private itemFactors: Map; private learningRate: number = 0.01; // Update model with new interaction updateOnInteraction( userId: string, itemId: string, feedback: number // 1 for positive, 0 for negative ): void { let userVec = this.userFactors.get(userId); let itemVec = this.itemFactors.get(itemId); // Initialize if new if (!userVec) { userVec = this.randomVector(50); this.userFactors.set(userId, userVec); } if (!itemVec) { itemVec = this.randomVector(50); this.itemFactors.set(itemId, itemVec); } // Predict and compute error const prediction = this.dotProduct(userVec, itemVec); const error = feedback - this.sigmoid(prediction); // Gradient update for (let i = 0; i < userVec.length; i++) { const userGrad = error * itemVec[i]; const itemGrad = error * userVec[i]; userVec[i] += this.learningRate * userGrad; itemVec[i] += this.learningRate * itemGrad; } } private randomVector(dim: number): number[] { return Array.from({ length: dim }, () => (Math.random() - 0.5) * 0.1); } private dotProduct(a: number[], b: number[]): number { return a.reduce((sum, val, i) => sum + val * b[i], 0); } private sigmoid(x: number): number { return 1 / (1 + Math.exp(-x)); } }

🛡️ Reliability & Fault Tolerance

Fallback Strategies

  • Popular items: When personalization fails
  • Cached recommendations: Serve stale data if fresh fails
  • Rule-based: Simple rules as last resort
  • Multi-model: Switch to backup model on failure

Circuit Breakers

Prevent cascading failures by breaking calls to failing services:

// Circuit breaker pattern for recommendations class CircuitBreaker { private failures: number = 0; private lastFailure: number = 0; private state: 'closed' | 'open' | 'half-open' = 'closed'; private threshold: number = 5; private timeout: number = 60000; // 1 minute async call(fn: () => Promise, fallback: () => T): Promise { if (this.state === 'open') { // Check if timeout has passed if (Date.now() - this.lastFailure > this.timeout) { this.state = 'half-open'; } else { console.log('Circuit breaker open, using fallback'); return fallback(); } } try { const result = await fn(); // Success - reset if half-open if (this.state === 'half-open') { this.state = 'closed'; this.failures = 0; } return result; } catch (error) { this.failures++; this.lastFailure = Date.now(); if (this.failures >= this.threshold) { this.state = 'open'; console.log('Circuit breaker opened due to failures'); } return fallback(); } } }

📈 A/B Testing Infrastructure

Continuous experimentation is essential for improvement:

// A/B testing framework class ABTestingFramework { private experiments: Map = new Map(); registerExperiment(experiment: Experiment): void { this.experiments.set(experiment.id, experiment); } // Assign user to variant getVariant(experimentId: string, userId: string): string { const experiment = this.experiments.get(experimentId); if (!experiment || !experiment.active) { return 'control'; } // Consistent hashing for stable assignment const hash = this.hashCode(userId + experimentId); const bucket = hash % 100; let cumulative = 0; for (const [variant, percentage] of Object.entries(experiment.variants)) { cumulative += percentage; if (bucket < cumulative) { return variant; } } return 'control'; } // Log experiment exposure logExposure(experimentId: string, userId: string, variant: string): void { // Send to analytics console.log('Experiment exposure:', { experimentId, userId, variant }); } // Log experiment outcome logOutcome(experimentId: string, userId: string, metric: string, value: number): void { // Send to analytics console.log('Experiment outcome:', { experimentId, userId, metric, value }); } private hashCode(str: string): number { let hash = 0; for (let i = 0; i < str.length; i++) { const char = str.charCodeAt(i); hash = ((hash << 5) - hash) + char; hash |= 0; // Convert to 32-bit integer } return Math.abs(hash); } } interface Experiment { id: string; name: string; active: boolean; variants: { control: number; // percentage (0-100) treatment: number; // percentage (0-100) }; }

🔒 Privacy & Ethics

Production systems must respect user privacy and avoid harmful biases:

Privacy Considerations

  • Data minimization: Collect only necessary data
  • Anonymization: Remove or hash PII where possible
  • User control: Allow opt-out and data deletion
  • Encryption: Encrypt data in transit and at rest
  • Federated learning: Train without centralizing data

Fairness & Bias

  • Diverse training data: Ensure representation
  • Bias detection: Monitor for discriminatory patterns
  • Fair ranking: Apply fairness constraints
  • Transparency: Explain recommendations
