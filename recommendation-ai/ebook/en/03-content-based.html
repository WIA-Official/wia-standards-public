📊 Chapter 3: Content-Based Filtering

WIA-AI-024 Recommendation AI Standard

🎯 Understanding Item Features

Content-based filtering recommends items by analyzing their intrinsic features and matching them to user preferences. Unlike collaborative filtering, which relies on crowd wisdom, content-based systems ask: "What properties make this item appealing to this user?"

This approach builds a user profile based on features of items they've liked, then recommends new items with similar features. For movies, features might include genre, director, actors, year, and plot keywords. For products, they might include category, brand, price range, and specifications.

Key Advantage: Content-based systems don't suffer from the cold start problem for new items (as long as we have their features) and can provide explanations like "Recommended because you liked similar sci-fi movies."

🏗️ Building Item Profiles

The first step is representing items as feature vectors. This process, called feature extraction, transforms items into mathematical representations.

Structured Features

For items with clear attributes, create structured feature vectors:

// Item profile with structured features interface MovieProfile { id: string; title: string; features: { genres: string[]; // ['Sci-Fi', 'Action', 'Thriller'] year: number; // 2010 director: string; // 'Christopher Nolan' actors: string[]; // ['Leonardo DiCaprio', ...] rating: number; // 8.8 duration: number; // 148 minutes language: string; // 'English' }; } // Convert to feature vector class FeatureExtractor { private genreIndex: Map = new Map(); private directorIndex: Map = new Map(); buildFeatureVector(movie: MovieProfile): number[] { const vector: number[] = []; // One-hot encode genres (assuming 20 genres) const genreVector = new Array(20).fill(0); for (const genre of movie.features.genres) { const idx = this.genreIndex.get(genre); if (idx !== undefined) genreVector[idx] = 1; } vector.push(...genreVector); // Normalize year to [0, 1] const normalizedYear = (movie.features.year - 1900) / 124; vector.push(normalizedYear); // Rating already in [0, 10] vector.push(movie.features.rating / 10); // Normalize duration vector.push(movie.features.duration / 300); return vector; } }

Text Features (TF-IDF)

For textual content like descriptions or reviews, use TF-IDF (Term Frequency-Inverse Document Frequency) to extract meaningful features:

// TF-IDF implementation for text features class TFIDFVectorizer { private vocabulary: Map = new Map(); private idf: Map = new Map(); private documentCount: number = 0; // Build vocabulary and calculate IDF fit(documents: string[]): void { this.documentCount = documents.length; const docFrequency = new Map(); // Count document frequencies for (const doc of documents) { const terms = new Set(this.tokenize(doc)); for (const term of terms) { docFrequency.set(term, (docFrequency.get(term) || 0) + 1); } } // Build vocabulary and calculate IDF let idx = 0; for (const [term, df] of docFrequency) { this.vocabulary.set(term, idx++); this.idf.set(term, Math.log(this.documentCount / df)); } } // Transform document to TF-IDF vector transform(document: string): number[] { const terms = this.tokenize(document); const termFreq = new Map(); // Calculate term frequencies for (const term of terms) { termFreq.set(term, (termFreq.get(term) || 0) + 1); } // Normalize by document length const docLength = terms.length; for (const [term, freq] of termFreq) { termFreq.set(term, freq / docLength); } // Build TF-IDF vector const vector = new Array(this.vocabulary.size).fill(0); for (const [term, tf] of termFreq) { const idx = this.vocabulary.get(term); const idf = this.idf.get(term); if (idx !== undefined && idf !== undefined) { vector[idx] = tf * idf; } } return vector; } private tokenize(text: string): string[] { return text.toLowerCase() .replace(/[^\w\s]/g, '') .split(/\s+/) .filter(word => word.length > 2); // Remove short words } }

👤 Building User Profiles

Once items are represented as feature vectors, build user profiles by aggregating features of items they've interacted with:

// User profile construction class UserProfileBuilder { buildProfile( userInteractions: Array<{itemId: string, rating: number}>, itemProfiles: Map ): number[] { if (userInteractions.length === 0) return []; const featureLength = itemProfiles.values().next().value.length; const profile = new Array(featureLength).fill(0); let totalWeight = 0; // Weighted average of item features for (const {itemId, rating} of userInteractions) { const itemVector = itemProfiles.get(itemId); if (!itemVector) continue; const weight = rating; // Use rating as weight for (let i = 0; i < featureLength; i++) { profile[i] += itemVector[i] * weight; } totalWeight += weight; } // Normalize if (totalWeight > 0) { for (let i = 0; i < featureLength; i++) { profile[i] /= totalWeight; } } return profile; } // Alternative: Boolean profile (liked/disliked) buildBooleanProfile( likedItems: string[], itemProfiles: Map ): number[] { if (likedItems.length === 0) return []; const featureLength = itemProfiles.values().next().value.length; const profile = new Array(featureLength).fill(0); for (const itemId of likedItems) { const itemVector = itemProfiles.get(itemId); if (!itemVector) continue; for (let i = 0; i < featureLength; i++) { profile[i] += itemVector[i]; } } // Average for (let i = 0; i < featureLength; i++) { profile[i] /= likedItems.length; } return profile; } }

📏 Similarity Calculation

With user and item profiles as vectors, calculate similarity to generate recommendations:

Method Use Case Pros Cons
Cosine Similarity Sparse, high-dimensional vectors Scale invariant, fast Ignores magnitude
Euclidean Distance Dense, low-dimensional data Intuitive, considers magnitude Sensitive to scale
Pearson Correlation Rating predictions Handles different scales More computationally expensive
Jaccard Set-based features Simple, works for binary Loses information
// Content-based recommender class ContentBasedRecommender { private itemProfiles: Map; private userProfiles: Map; constructor() { this.itemProfiles = new Map(); this.userProfiles = new Map(); } addItemProfile(itemId: string, features: number[]): void { this.itemProfiles.set(itemId, features); } updateUserProfile(userId: string, profile: number[]): void { this.userProfiles.set(userId, profile); } // Generate recommendations recommend( userId: string, k: number = 10, excludeItems: Set = new Set() ): Array<{itemId: string, score: number}> { const userProfile = this.userProfiles.get(userId); if (!userProfile) return []; const scores: Array<{itemId: string, score: number}> = []; for (const [itemId, itemProfile] of this.itemProfiles) { if (excludeItems.has(itemId)) continue; const similarity = this.cosineSimilarity(userProfile, itemProfile); scores.push({ itemId, score: similarity }); } return scores .sort((a, b) => b.score - a.score) .slice(0, k); } private cosineSimilarity(vecA: number[], vecB: number[]): number { let dotProduct = 0; let magA = 0; let magB = 0; for (let i = 0; i < vecA.length; i++) { dotProduct += vecA[i] * vecB[i]; magA += vecA[i] * vecA[i]; magB += vecB[i] * vecB[i]; } if (magA === 0 || magB === 0) return 0; return dotProduct / (Math.sqrt(magA) * Math.sqrt(magB)); } // Explain recommendation explainRecommendation( userId: string, itemId: string, featureNames: string[] ): Array<{feature: string, contribution: number}> { const userProfile = this.userProfiles.get(userId); const itemProfile = this.itemProfiles.get(itemId); if (!userProfile || !itemProfile) return []; const explanations: Array<{feature: string, contribution: number}> = []; for (let i = 0; i < featureNames.length; i++) { const contribution = userProfile[i] * itemProfile[i]; if (contribution > 0.01) { // Threshold for relevance explanations.push({ feature: featureNames[i], contribution }); } } return explanations.sort((a, b) => b.contribution - a.contribution); } }

🔍 Advanced Feature Engineering

Sophisticated content-based systems employ advanced feature extraction techniques:

1. Word Embeddings

Instead of TF-IDF, use pre-trained embeddings (Word2Vec, GloVe, BERT) to capture semantic meaning.

2. Image Features

For visual items, extract features using CNNs (Convolutional Neural Networks) pre-trained on ImageNet.

3. Audio Features

For music, extract mel-frequency cepstral coefficients (MFCCs), tempo, key, and other acoustic features.

4. Knowledge Graphs

Leverage structured knowledge (DBpedia, Wikidata) to enrich item representations with relationships and entities.

🎵 Example: Music Recommendation

A music recommender might combine:

  • Audio features: Tempo (120 BPM), key (C major), energy (0.8)
  • Metadata: Genre (Electronic), artist, year
  • Lyrics: TF-IDF or BERT embeddings of lyrics
  • Social: Playlist co-occurrence patterns

⚖️ Addressing Overspecialization

A key limitation of content-based filtering is overspecialization: users only get recommendations similar to what they've already seen, creating filter bubbles.

Mitigation Strategies

  • Serendipity injection: Occasionally recommend diverse items
  • Temporal decay: Give less weight to older preferences
  • Negative feedback: Learn what users dislike
  • Exploration-exploitation: Balance familiar and novel recommendations
// Adding diversity to recommendations class DiverseContentRecommender extends ContentBasedRecommender { recommendWithDiversity( userId: string, k: number = 10, diversityWeight: number = 0.3 ): Array<{itemId: string, score: number}> { // Get initial recommendations const candidates = this.recommend(userId, k * 3); // Get 3x candidates const selected: Array<{itemId: string, score: number}> = []; const selectedProfiles: number[][] = []; while (selected.length < k && candidates.length > 0) { let bestIdx = 0; let bestScore = -Infinity; for (let i = 0; i < candidates.length; i++) { const candidate = candidates[i]; const itemProfile = this.itemProfiles.get(candidate.itemId); if (!itemProfile) continue; // Calculate diversity from already selected items let minDiversity = Infinity; for (const selectedProfile of selectedProfiles) { const similarity = this.cosineSimilarity(itemProfile, selectedProfile); const diversity = 1 - similarity; minDiversity = Math.min(minDiversity, diversity); } // Combined score: relevance + diversity const relevanceScore = candidate.score; const diversityScore = selectedProfiles.length === 0 ? 1 : minDiversity; const combinedScore = (1 - diversityWeight) * relevanceScore + diversityWeight * diversityScore; if (combinedScore > bestScore) { bestScore = combinedScore; bestIdx = i; } } // Add best candidate const best = candidates.splice(bestIdx, 1)[0]; selected.push(best); selectedProfiles.push(this.itemProfiles.get(best.itemId)!); } return selected; } }

🎯 Hybrid Content Approaches

Combining multiple content-based techniques often yields better results:

Multi-Modal Learning

Combine different types of features (text, images, metadata) into a unified representation.

Ensemble Methods

Train multiple models on different feature sets and aggregate their predictions.

Deep Learning Integration

Use neural networks to automatically learn relevant features from raw data.

📊 Evaluation Metrics

Content-based systems should be evaluated on multiple dimensions:

  • Accuracy: Precision, recall, F1-score
  • Diversity: Intra-list similarity, coverage
  • Novelty: Percentage of unfamiliar items
  • Serendipity: Unexpected but relevant recommendations
  • Explainability: Can we explain why items were recommended?

🚀 Real-World Applications

Content-based filtering excels in specific domains:

News Recommendation

Recommend articles based on topic, entities, writing style, and user reading history.

Job Matching

Match candidates to jobs based on skills, experience, education, and location.

Academic Paper Recommendation

Suggest papers based on citations, topics, authors, and abstract similarity.

Fashion Recommendation

Recommend clothing based on visual features, style, brand, and user preferences.

📝 Summary

  • Content-based filtering recommends items by matching their features to user preferences
  • Feature extraction transforms items into vector representations
  • User profiles aggregate features of items users have liked
  • Similarity metrics (cosine, Euclidean) compare user and item profiles
  • Advanced techniques include embeddings, CNNs for images, and knowledge graphs
  • Main limitation is overspecialization leading to filter bubbles
  • Diversity can be promoted through serendipity injection and exploration strategies
  • Content-based excels when items have rich metadata and cold start is a concern
