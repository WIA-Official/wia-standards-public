🤝 Chapter 2: Collaborative Filtering

WIA-AI-024 Recommendation AI Standard

🎯 The Wisdom of the Crowd

Collaborative filtering (CF) is based on a simple but powerful idea: people who agreed in the past tend to agree in the future. If User A and User B both loved movies X, Y, and Z, and User A also loved movie W, then User B will probably like movie W too.

This approach doesn't require understanding what a movie is about, who directed it, or what genre it belongs to. It discovers patterns purely from user behavior. This makes collaborative filtering domain-agnostic and capable of finding surprising connections that content analysis might miss.

Key Insight: Collaborative filtering harnesses collective intelligence. It's not about what items are, but about how people interact with them.

📊 User-Based Collaborative Filtering

User-based CF finds users similar to the target user and recommends items those similar users liked. The process involves three steps:

Step 1: Find Similar Users

Calculate similarity between the target user and all other users based on their rating patterns. Common similarity metrics include:

Cosine Similarity = cos(θ) = (A · B) / (||A|| × ||B||)
// User-based collaborative filtering implementation class UserBasedCF { private ratings: Map>; constructor(ratings: Map>) { this.ratings = ratings; } // Calculate cosine similarity between two users cosineSimilarity(userA: string, userB: string): number { const ratingsA = this.ratings.get(userA); const ratingsB = this.ratings.get(userB); if (!ratingsA || !ratingsB) return 0; // Find common items const commonItems = new Set( [...ratingsA.keys()].filter(item => ratingsB.has(item)) ); if (commonItems.size === 0) return 0; // Calculate dot product and magnitudes let dotProduct = 0; let magA = 0; let magB = 0; for (const item of commonItems) { const rA = ratingsA.get(item)!; const rB = ratingsB.get(item)!; dotProduct += rA * rB; magA += rA * rA; magB += rB * rB; } if (magA === 0 || magB === 0) return 0; return dotProduct / (Math.sqrt(magA) * Math.sqrt(magB)); } // Find k most similar users findSimilarUsers(targetUser: string, k: number = 10): Array<{user: string, similarity: number}> { const similarities: Array<{user: string, similarity: number}> = []; for (const user of this.ratings.keys()) { if (user !== targetUser) { const sim = this.cosineSimilarity(targetUser, user); if (sim > 0) { similarities.push({ user, similarity: sim }); } } } return similarities .sort((a, b) => b.similarity - a.similarity) .slice(0, k); } // Generate recommendations recommend(targetUser: string, k: number = 10): Array<{item: string, score: number}> { const similarUsers = this.findSimilarUsers(targetUser, k); const targetRatings = this.ratings.get(targetUser); if (!targetRatings) return []; // Aggregate scores from similar users const scores = new Map(); const weights = new Map(); for (const {user, similarity} of similarUsers) { const userRatings = this.ratings.get(user)!; for (const [item, rating] of userRatings) { // Skip items the target user has already rated if (targetRatings.has(item)) continue; const currentScore = scores.get(item) || 0; const currentWeight = weights.get(item) || 0; scores.set(item, currentScore + rating * similarity); weights.set(item, currentWeight + similarity); } } // Normalize scores const recommendations: Array<{item: string, score: number}> = []; for (const [item, score] of scores) { const weight = weights.get(item)!; recommendations.push({ item, score: weight > 0 ? score / weight : 0 }); } return recommendations .sort((a, b) => b.score - a.score) .slice(0, k); } }

Step 2: Aggregate Ratings

Once similar users are identified, aggregate their ratings weighted by similarity. Users more similar to the target have more influence on the recommendation.

Step 3: Generate Predictions

Predict ratings for items the target user hasn't interacted with and recommend the highest-scoring items.

📝 Example: User-Based CF in Action

User Alice has rated: Inception (5★), Matrix (4★), Interstellar (5★)
User Bob has rated: Inception (5★), Matrix (4★), Interstellar (4★), Avatar (5★)
User Charlie has rated: Inception (4★), Matrix (5★), Avatar (4★)

Alice and Bob are highly similar (both love sci-fi). Since Bob loved Avatar and Alice hasn't seen it, the system recommends Avatar to Alice with a predicted rating of ~4.5★.

🎬 Item-Based Collaborative Filtering

Item-based CF flips the approach: instead of finding similar users, find similar items. If a user liked item A, recommend items similar to A based on user rating patterns.

Why Item-Based Often Outperforms User-Based

  • Stability: Item similarities change less frequently than user preferences
  • Scalability: Usually fewer items than users, making precomputation feasible
  • Explainability: Easier to explain "users who liked X also liked Y"
  • Sparsity: Better handles sparse data matrices
// Item-based collaborative filtering class ItemBasedCF { private ratings: Map>; private itemSimilarities: Map>; constructor(ratings: Map>) { this.ratings = ratings; this.itemSimilarities = new Map(); this.precomputeItemSimilarities(); } // Precompute item-item similarities (can be cached) private precomputeItemSimilarities(): void { const items = this.getAllItems(); for (let i = 0; i < items.length; i++) { const itemA = items[i]; const simMap = new Map(); for (let j = i + 1; j < items.length; j++) { const itemB = items[j]; const sim = this.calculateItemSimilarity(itemA, itemB); if (sim > 0) { simMap.set(itemB, sim); // Store symmetric similarity if (!this.itemSimilarities.has(itemB)) { this.itemSimilarities.set(itemB, new Map()); } this.itemSimilarities.get(itemB)!.set(itemA, sim); } } this.itemSimilarities.set(itemA, simMap); } } private getAllItems(): string[] { const items = new Set(); for (const userRatings of this.ratings.values()) { for (const item of userRatings.keys()) { items.add(item); } } return Array.from(items); } private calculateItemSimilarity(itemA: string, itemB: string): number { // Find users who rated both items const ratingsA: number[] = []; const ratingsB: number[] = []; for (const userRatings of this.ratings.values()) { if (userRatings.has(itemA) && userRatings.has(itemB)) { ratingsA.push(userRatings.get(itemA)!); ratingsB.push(userRatings.get(itemB)!); } } if (ratingsA.length < 2) return 0; // Need minimum co-ratings // Pearson correlation return this.pearsonCorrelation(ratingsA, ratingsB); } private pearsonCorrelation(x: number[], y: number[]): number { const n = x.length; const sumX = x.reduce((a, b) => a + b, 0); const sumY = y.reduce((a, b) => a + b, 0); const sumXY = x.reduce((sum, xi, i) => sum + xi * y[i], 0); const sumX2 = x.reduce((sum, xi) => sum + xi * xi, 0); const sumY2 = y.reduce((sum, yi) => sum + yi * yi, 0); const numerator = n * sumXY - sumX * sumY; const denominator = Math.sqrt( (n * sumX2 - sumX * sumX) * (n * sumY2 - sumY * sumY) ); return denominator === 0 ? 0 : numerator / denominator; } // Generate recommendations for a user recommend(targetUser: string, k: number = 10): Array<{item: string, score: number}> { const userRatings = this.ratings.get(targetUser); if (!userRatings) return []; const scores = new Map(); const weights = new Map(); // For each item the user has rated for (const [ratedItem, rating] of userRatings) { const similarities = this.itemSimilarities.get(ratedItem); if (!similarities) continue; // Look at similar items for (const [similarItem, similarity] of similarities) { // Skip if user already rated this item if (userRatings.has(similarItem)) continue; const currentScore = scores.get(similarItem) || 0; const currentWeight = weights.get(similarItem) || 0; scores.set(similarItem, currentScore + rating * similarity); weights.set(similarItem, currentWeight + Math.abs(similarity)); } } // Normalize and sort const recommendations: Array<{item: string, score: number}> = []; for (const [item, score] of scores) { const weight = weights.get(item)!; recommendations.push({ item, score: weight > 0 ? score / weight : 0 }); } return recommendations .sort((a, b) => b.score - a.score) .slice(0, k); } }

Metric Formula Type Range Best For
Cosine Similarity Vector angle -1 to 1 Sparse data, rating patterns
Pearson Correlation Linear relationship -1 to 1 Handling rating scales, user bias
Jaccard Similarity Set intersection 0 to 1 Binary data (liked/not liked)
Euclidean Distance Geometric distance 0 to ∞ Dense data, continuous ratings

🎯 Handling Sparse Data

Real-world rating matrices are typically 99%+ sparse. Most users rate very few items. This sparsity creates several challenges:

Challenge 1: Limited Overlap

Two users might be similar but have few co-rated items, making similarity calculations unreliable.

Challenge 2: Cold Start

New users or items have no ratings, making them impossible to recommend or be recommended.

Challenge 3: Computational Cost

Sparse matrices require special data structures and algorithms to avoid wasting computation on zero values.

// Efficient sparse matrix representation class SparseMatrix { private data: Map>; private userIndex: Map; private itemIndex: Map; constructor() { this.data = new Map(); this.userIndex = new Map(); this.itemIndex = new Map(); } set(user: string, item: string, rating: number): void { if (!this.data.has(user)) { this.data.set(user, new Map()); this.userIndex.set(user, this.userIndex.size); } if (!this.itemIndex.has(item)) { this.itemIndex.set(item, this.itemIndex.size); } this.data.get(user)!.set(item, rating); } get(user: string, item: string): number | undefined { return this.data.get(user)?.get(item); } // Get only non-zero entries for a user getUserRatings(user: string): Map { return this.data.get(user) || new Map(); } // Calculate sparsity getSparsity(): number { const totalCells = this.userIndex.size * this.itemIndex.size; let nonZeroCells = 0; for (const userRatings of this.data.values()) { nonZeroCells += userRatings.size; } return 1 - (nonZeroCells / totalCells); } }

🚀 Optimization Techniques

Production collaborative filtering systems employ various optimizations:

1. Dimensionality Reduction

Reduce the rating matrix to a lower-dimensional space using techniques like SVD (Singular Value Decomposition).

2. Approximate Nearest Neighbors

Instead of computing exact similarities with all users/items, use ANN algorithms (LSH, HNSW) for faster approximate results.

3. Incremental Updates

Update models incrementally as new ratings arrive rather than recomputing from scratch.

4. Caching

Cache similarity computations and recommendations for popular items or users.

// Caching layer for collaborative filtering class CachedCF { private cf: ItemBasedCF; private cache: Map>; private cacheExpiry: Map; private ttl: number = 3600000; // 1 hour constructor(cf: ItemBasedCF) { this.cf = cf; this.cache = new Map(); this.cacheExpiry = new Map(); } recommend(user: string, k: number = 10): Array<{item: string, score: number}> { const cacheKey = `${user}_${k}`; const now = Date.now(); // Check cache if (this.cache.has(cacheKey)) { const expiry = this.cacheExpiry.get(cacheKey)!; if (now < expiry) { return this.cache.get(cacheKey)!; } } // Generate fresh recommendations const recommendations = this.cf.recommend(user, k); // Store in cache this.cache.set(cacheKey, recommendations); this.cacheExpiry.set(cacheKey, now + this.ttl); return recommendations; } invalidateUser(user: string): void { // Remove all cached entries for a user for (const key of this.cache.keys()) { if (key.startsWith(user + '_')) { this.cache.delete(key); this.cacheExpiry.delete(key); } } } }

⚠️ Limitations and Challenges

While powerful, collaborative filtering has inherent limitations:

Popularity Bias

Popular items get recommended more, creating a rich-get-richer effect that reduces diversity and hurts niche items.

Filter Bubble

Users only see items similar to what they've already liked, limiting discovery and serendipity.

Shilling Attacks

Malicious users can manipulate recommendations by creating fake profiles with strategic ratings.

Gray Sheep Problem

Users with unique tastes don't fit any cluster, receiving poor recommendations.

🛡️ Mitigation Strategies
  • Diversity promotion: Explicitly boost diverse items in rankings
  • Exploration: Mix in random or trending items
  • Anomaly detection: Identify and filter suspicious rating patterns
  • Hybrid approaches: Combine with content-based methods

📈 Performance Considerations

Scaling collaborative filtering to millions of users and items requires careful engineering:

Memory Management

Storing all user-item pairs is prohibitive. Use sparse representations and consider approximate methods.

Computational Complexity

  • User-based: O(|U|²) to compute all user similarities
  • Item-based: O(|I|²) to compute all item similarities
  • Recommendation generation: O(|U| × |I|) in worst case

Real-Time Requirements

Precompute what you can (item similarities), cache aggressively, and use approximate methods for interactive latencies.

📝 Summary

  • Collaborative filtering leverages collective user behavior to make recommendations
  • User-based CF finds similar users; item-based CF finds similar items
  • Item-based often outperforms user-based due to better stability and scalability
  • Common similarity metrics include cosine, Pearson, Jaccard, and Euclidean
  • Sparsity is a fundamental challenge requiring specialized data structures
  • Optimizations include caching, dimensionality reduction, and approximate neighbors
  • Limitations include popularity bias, filter bubbles, and cold start problems
  • Production systems require careful engineering for scale and performance
