# 챕터 17 · End-to-End 오케스트레이션 — 4 Phase 자동 통과

## 학습 목표
- 손상 고문서 이미지 한 장이 4 Phase(복원·구조화·변환·전달)를 자동 통과하는 오케스트레이션 DAG를 설계할 수 있다
- 각 Phase의 산출물이 다음 Phase의 입력이 되는 데이터 흐름을 Prefect 또는 Airflow로 구축할 수 있다
- 파이프라인 실패 지점을 식별하고 부분 재실행(retry from failed step) 전략을 적용할 수 있다

---

## 이론

### 오케스트레이션의 필요

챕터 1~16의 모든 모듈은 독립적으로 작동한다. 하지만 사용자 입장에서는 "이미지 한 장 올렸더니 웹페이지 + EPUB + 점자 + 음성 + 수어가 나왔다"가 이상적. **16개 단계를 한 번에 실행**하고, 각 단계 실패에 견고해야 한다.

### 파이프라인 도구 비교 (2026)

| 도구 | 장점 | 단점 | 적합 |
|---|---|---|---|
| **Prefect 3** | Python 네이티브, 시각 UI, retry 쉬움 | 학습 곡선 낮음 | 권장 |
| **Airflow** | 대규모·다양한 백엔드 | 복잡, SQLAlchemy 의존 | 엔터프라이즈 |
| **Dagster** | 데이터 asset 중심, 타입 체크 | ML 친화, 초기 개념 많음 | 데이터 플랫폼 |
| **Argo Workflows** | Kubernetes 네이티브 | 운영 부담 | k8s 인프라 있을 때 |
| **직접 구현** | 간단 | retry·모니터링 직접 작성 | 프로토타입 |

WIA 글살림 권장: **Prefect 3**. Python으로 바로 쓰고, Prefect Cloud 무료 tier 있음, 시각 UI 있어 비기술자도 상태 파악.

### 4 Phase DAG 설계

```
  ┌─ Phase 1 복원 ─┐
  │               │
  ▼               ▼
이미지 복원    텍스트 복원
(SDXL Inp)    (LLM 누락 예측)
  │               │
  └──┬────────────┘
     ▼
  CrossCheck 통합 결과
     │
     ▼
  Phase 1 증명서 발행 (Merkle + WIA Chain)
     │
     ├──┬──┬──┬──┐
     ▼  ▼  ▼  ▼  ▼
  Phase 2   Phase 3   Phase 4
  (HTML)    (점자)    (수어)
  (EPUB)    (음성)    (VR)
  (ONIX)
     │
     └──┬──┴──┘
        ▼
     Phase 4 최종 번들 (zip)
     + 접근성 감사 리포트
     + 법규 준수 리포트
     + End-to-End 증명서 갱신
```

Phase 2~4는 **병렬** 실행 가능 (서로 의존성 없음). Prefect는 자동 병렬화.

### 실패 대응 전략

**failure vs degradation**:
- **Failure**: 전체 중단 (예: SDXL 모델 로드 실패)
- **Degradation**: 일부 누락된 채 진행 (예: 수어 인식 실패 → 수어 없이 다른 모달리티만)

사용자 입장에서 "부분 실패 + 누락 명시"가 "전체 실패"보다 낫다. Prefect의 `allow_failure` 태스크로 구현.

### 비용 관리

한 건 처리 비용 추정:
- Phase 1: $0.20 (SDXL + LLM GPU 시간)
- Phase 2: $0.01 (CPU 템플릿)
- Phase 3: $0.15 (TTS GPU + 점자 CPU)
- Phase 4: $0.10 (수어 클립 조합 CPU + VR 씬 생성)
- **합: $0.46 per document**

1만 건 처리 = $4,600. Serverless 사용으로 절반 가능.

```python
# 4 Phase End-to-End 오케스트레이션 (Prefect 3)
# pip install prefect==3.0 pydantic

from prefect import flow, task
from prefect.task_runners import ConcurrentTaskRunner
from prefect.artifacts import create_markdown_artifact
from pathlib import Path
from dataclasses import asdict
import json


# ──────────────────────────────────────────────
# Phase 1 — 복원
# ──────────────────────────────────────────────
@task(retries=2, retry_delay_seconds=10, log_prints=True)
def task_image_inpaint(image_path: str, output_dir: str) -> dict:
    """SDXL Inpainting (실전편 챕터 14)"""
    from inference import DocumentRestorer
    restorer = DocumentRestorer(
        base_model_id="diffusers/stable-diffusion-xl-1.0-inpainting-0.1",
        lora_path="./models/sdxl_inpaint_korean.safetensors",
        mask_mode="otsu",
    )
    return restorer.restore(image_path, f"{output_dir}/restored_image.jpg")


@task(retries=2)
def task_text_predict(image_path: str, output_dir: str) -> dict:
    """LLM 누락 글자 예측 (챕터 01)"""
    from advanced_ch01 import HybridRestorer
    restorer = HybridRestorer(
        inpaint_lora_path="./models/sdxl_inpaint_korean.safetensors",
        llm_lora_path="./models/solar_hanmun_lora",
    )
    return restorer.restore(image_path, output_dir)


@task(retries=1)
def task_multimodal_crosscheck(image_path: str, restoration: dict) -> dict:
    """멀티모달 CrossCheck (챕터 02)"""
    from advanced_ch02 import AdvancedHybridRestorer
    advanced = AdvancedHybridRestorer(
        inpaint_lora_path="./models/sdxl_inpaint_korean.safetensors",
        text_lora_path="./models/solar_hanmun_lora",
    )
    return advanced.restore(image_path, restoration.get("output_dir"))


@task(retries=3, retry_delay_seconds=5)
def task_issue_certificate(
    original_image: str, restoration: dict, output_dir: str,
) -> dict:
    """진위 증명서 발행 (챕터 03)"""
    from advanced_ch03 import CertificateBuilder, save_certificate
    builder = CertificateBuilder()
    cert = builder.build(
        original_image_path=original_image,
        restored_image_path=restoration["image_path"],
        mask_image_path=restoration["image_path"].replace(".jpg", ".mask.png"),
        restoration_json=restoration,
        lora_weights_path="./models/sdxl_inpaint_korean.safetensors",
        document_title=restoration.get("document_title", "Unknown"),
    )
    cert = builder.anchor_to_wia_chain(cert)
    cert_path = f"{output_dir}/certificate.json"
    save_certificate(cert, cert_path)
    return {"path": cert_path, "cert": asdict(cert)}


# ──────────────────────────────────────────────
# Phase 2 — 구조화 (HTML + EPUB + ONIX)
# ──────────────────────────────────────────────
@task(retries=2)
def task_generate_alt_text(restoration: dict, output_dir: str) -> dict:
    """AI alt-text 생성 (챕터 06)"""
    from advanced_ch06 import AltTextGenerator, AltQualityEvaluator
    gen = AltTextGenerator()
    alts = gen.generate_both(restoration["image_path"])
    evaluator = AltQualityEvaluator()
    quality = evaluator.evaluate_all(
        image_path=restoration["image_path"],
        predicted_alt=alts["alt"],
    )
    alts["quality"] = quality
    with open(f"{output_dir}/alt_text.json", "w", encoding="utf-8") as f:
        json.dump(alts, f, ensure_ascii=False, indent=2)
    return alts


@task(retries=2)
def task_generate_html(
    restoration: dict, cert: dict, alt: dict, output_dir: str,
) -> str:
    """WCAG 2.2 AA HTML (챕터 04)"""
    from advanced_ch04 import generate_and_audit
    result = generate_and_audit(
        restoration_json_path=f"{output_dir}/restoration.json",
        certificate_json_path=cert["path"],
        ai_alt_text=alt["alt"],
        output_html_path=f"{output_dir}/page.html",
    )
    return f"{output_dir}/page.html"


@task(retries=2)
def task_generate_epub(
    restoration: dict, cert: dict, output_dir: str,
    tts_audio: str = None,
) -> dict:
    """접근성 EPUB + ONIX (챕터 05)"""
    from advanced_ch05 import build_restoration_epub
    return build_restoration_epub(
        restoration_json_path=f"{output_dir}/restoration.json",
        certificate_json_path=cert["path"],
        restored_image_path=restoration["image_path"],
        tts_audio_path=tts_audio,
        output_epub_path=f"{output_dir}/document.epub",
    )


# ──────────────────────────────────────────────
# Phase 3 — 변환 (점자 + 음성)
# ──────────────────────────────────────────────
@task(retries=1)
def task_braille(restoration: dict, output_dir: str) -> dict:
    """점자 변환 (챕터 09/10)"""
    from wia_braille import convert_document
    return convert_document(
        text_ko=restoration.get("modern_translation", ""),
        text_original=restoration.get("masked_text", ""),
        output_path=f"{output_dir}/braille.json",
    )


@task(retries=2, retry_delay_seconds=15)
def task_tts(restoration: dict, output_dir: str) -> dict:
    """TTS 생성 + 품질 검증 (챕터 11/12)"""
    from tts_engine import WIAVoiceAPI
    from advanced_ch12 import TTSQualityValidator
    api = WIAVoiceAPI()
    validator = TTSQualityValidator(api)
    text = restoration.get("modern_translation", "")
    if not text:
        return {"skipped": True, "reason": "번역 없음"}
    result = validator.validate(text, lang="ko")
    # 결과 오디오를 output_dir로 복사
    import shutil
    src = result["final"]["audio_path"]
    dst = f"{output_dir}/narration_ko.wav"
    shutil.copy(src, dst)
    return {"path": dst, "cer": result["final"]["cer"]}


# ──────────────────────────────────────────────
# Phase 4 — 전달 (수어 + VR 메타)
# ──────────────────────────────────────────────
@task(retries=1)
def task_sign_video(restoration: dict, output_dir: str) -> dict:
    """3D 수어 변환 (챕터 13/14)"""
    from advanced_ch14 import translate_sentence
    text = restoration.get("modern_translation", "")
    return translate_sentence(
        sentence=text, src_lang="ko", target_sign="KSL",
        registry_path="./data/concepts.json",
    )


@task(retries=1)
def task_vr_scene(restoration: dict, output_dir: str) -> str:
    """VR 씬 메타 생성 (챕터 15)"""
    from advanced_ch15 import build_vr_scene_meta
    build_vr_scene_meta(
        restoration_json_path=f"{output_dir}/restoration.json",
        output_path=f"{output_dir}/vr_scene.json",
    )
    return f"{output_dir}/vr_scene.json"


# ──────────────────────────────────────────────
# 집계 + 최종 번들
# ──────────────────────────────────────────────
@task
def task_bundle(output_dir: str, manifests: dict) -> str:
    """최종 ZIP 번들"""
    import zipfile
    bundle_path = f"{output_dir}/final_bundle.zip"
    with zipfile.ZipFile(bundle_path, "w", zipfile.ZIP_DEFLATED) as zf:
        for name, path in manifests.items():
            if isinstance(path, str) and Path(path).exists():
                zf.write(path, arcname=Path(path).name)

    # 최종 manifest.json
    manifest = {
        "generated_at": datetime.now(timezone.utc).isoformat(),
        "artifacts":    manifests,
    }
    with open(f"{output_dir}/manifest.json", "w", encoding="utf-8") as f:
        json.dump(manifest, f, ensure_ascii=False, indent=2)
    return bundle_path


@task
def task_accessibility_audit(html_path: str) -> dict:
    """WCAG 감사 (챕터 07)"""
    import subprocess
    result = subprocess.run(
        ["python", "scripts/run_audit.py",
         "--url", f"file://{Path(html_path).resolve()}",
         "--output-json", html_path.replace(".html", ".audit.json"),
         "--min-score", "90", "--max-errors", "3"],
        capture_output=True, text=True,
    )
    import json
    with open(html_path.replace(".html", ".audit.json"), encoding="utf-8") as f:
        return json.load(f)


@task
def task_legal_compliance(audit: dict, output_dir: str) -> dict:
    """글로벌 법규 준수 리포트 (챕터 08)"""
    from advanced_ch08 import generate_global_report
    manual = {
        "accessibility_statement_published": True,
        "user_feedback_mechanism_exists":    True,
        "user_testing_with_disabled_participants": False,
        # ...
    }
    return generate_global_report(
        audit_report=audit,
        manual_verifications=manual,
        standards=["EU_EAA", "USA_ADA", "KR_KS_X_3253"],
        output_path=f"{output_dir}/legal.html",
    )


# ──────────────────────────────────────────────
# 메인 Flow
# ──────────────────────────────────────────────
@flow(
    name="WIA Geulsalim 4-Phase Pipeline",
    task_runner=ConcurrentTaskRunner(),
    log_prints=True,
    timeout_seconds=1800,
)
def process_document(
    image_path: str,
    output_dir: str,
    document_title: str = "Untitled Document",
) -> dict:
    """
    손상 고문서 이미지 → 4 Phase 산출물 자동 생성.
    """
    Path(output_dir).mkdir(parents=True, exist_ok=True)
    print(f">> Processing: {image_path}")

    # ==== Phase 1: 복원 ====
    # Image + Text 병렬
    inpaint_result = task_image_inpaint.submit(image_path, output_dir)
    text_result    = task_text_predict.submit(image_path, output_dir)
    restoration = task_multimodal_crosscheck(image_path, text_result.result())

    # 증명서
    cert = task_issue_certificate(image_path, restoration, output_dir)

    # ==== Phase 2 + 3 + 4: 병렬 실행 ====
    alt       = task_generate_alt_text.submit(restoration, output_dir)
    braille   = task_braille.submit(restoration, output_dir)
    tts       = task_tts.submit(restoration, output_dir)
    sign      = task_sign_video.submit(restoration, output_dir)
    vr_scene  = task_vr_scene.submit(restoration, output_dir)

    # HTML은 alt-text 결과 필요
    html_path = task_generate_html(restoration, cert, alt.result(), output_dir)

    # EPUB은 TTS 오디오 기다림 (선택)
    tts_result = tts.result()
    epub_result = task_generate_epub(
        restoration, cert, output_dir,
        tts_audio=tts_result.get("path") if not tts_result.get("skipped") else None,
    )

    # 감사 + 법규
    audit = task_accessibility_audit(html_path)
    legal = task_legal_compliance(audit, output_dir)

    # 최종 번들
    manifests = {
        "image":        restoration["image_path"],
        "certificate":  cert["path"],
        "html":         html_path,
        "epub":         epub_result["epub"],
        "onix":         epub_result["onix"],
        "braille":      f"{output_dir}/braille.json",
        "tts":          tts_result.get("path"),
        "sign":         f"{output_dir}/sign_clips.json",
        "vr_scene":     f"{output_dir}/vr_scene.json",
        "audit":        f"{output_dir}/page.audit.json",
        "legal":        f"{output_dir}/legal.html",
    }
    bundle = task_bundle(output_dir, manifests)

    # 리포트 생성
    create_markdown_artifact(
        key=f"report-{Path(output_dir).name}",
        markdown=f"""# 처리 완료: {document_title}

- 복원 이미지: {restoration['image_path']}
- 증명서:     [{cert['path']}]({cert['path']})
- HTML:       [{html_path}]({html_path})
- EPUB:       [{epub_result['epub']}]({epub_result['epub']})
- 접근성 점수: {audit.get('score', 'N/A')}/100
- 법규 준수:   {legal}
- 번들:       [{bundle}]({bundle})
""",
    )
    return {"bundle": bundle, "manifests": manifests}


# ──────────────────────────────────────────────
# 실행
# ──────────────────────────────────────────────
if __name__ == "__main__":
    from datetime import datetime, timezone

    result = process_document(
        image_path="./samples/sangso_damaged_01.jpg",
        output_dir=f"./output/run_{datetime.now(timezone.utc).strftime('%Y%m%d_%H%M%S')}",
        document_title="조선시대 상소문 #01",
    )
    print(f"\n>> 완료: {result['bundle']}")
```

# 폴더 내 모든 이미지 병렬 처리
@flow(name="Batch Processing", task_runner=ConcurrentTaskRunner())
def process_batch(input_dir: str, output_root: str, max_concurrent: int = 4):
    from pathlib import Path
    images = sorted(Path(input_dir).glob("*.jpg"))
    print(f">> Batch: {len(images)} images")

    results = []
    # 동시 실행 제한 (GPU 메모리 고려)
    for i in range(0, len(images), max_concurrent):
        batch = images[i:i + max_concurrent]
        futures = [
            process_document.submit(
                str(img),
                f"{output_root}/{img.stem}",
                img.stem,
            ) for img in batch
        ]
        for f in futures:
            results.append(f.result())
    return results


if __name__ == "__main__":
    process_batch(
        input_dir="./input_documents",
        output_root="./batch_output",
        max_concurrent=4,
    )

### 사용자 관점의 오케스트레이션

이 파이프라인을 처음 다섯 명에게 시연했을 때, 그들은 DAG 다이어그램에 관심이 없었다. 한 분은 복원된 문서 파일과 음성 파일을 받아 집에 가져가면서 이렇게 물었다.

> "이걸 어머니께 읽어드리려고요. 어머니는 한문을 모르세요. 음성 파일에 현대어 번역이 들어 있나요?"

그 순간 나는 우리가 만든 Prefect flow 16개 task의 실제 용도를 알게 됐다. 오케스트레이션은 "DAG를 돌리는 기술"이 아니라 **한 장의 훼손된 종이가, 한 사람의 하루 속 어떤 자리에 가닿을 때까지 길을 잃지 않게 하는 일**이다. 실패 복구·병렬·배치는 모두 그 도달을 위해 있다.

그래서 이 장의 모든 코드를 다시 볼 때, `task_tts`의 언어 기본값이 한국어 현대어 번역인지, `task_bundle` 이 어머니가 열 수 있는 파일 형식인지 — 그런 질문부터 확인하자. 기술은 그 다음 자리다.

## 함정 카드

1. **"Phase 1 실패 시 Phase 2~4도 실행하려고 시도"** → Phase 1 없이는 후속 단계 데이터 없음. Prefect의 **`wait_for`** 명시적 의존성 선언. 또는 Phase 1 실패 시 전체 flow 중단.

2. **"모든 task를 동시 실행"** → GPU 메모리 고갈. SDXL + LLM + Whisper 동시 로드 = 50GB+. `max_concurrent=1`로 GPU-heavy 태스크는 순차, CPU 태스크만 병렬.

3. **"실패 시 처음부터 재시작"** → Phase 3의 TTS만 실패해도 Phase 1 SDXL부터 재실행 = 5분 낭비. Prefect의 **부분 재시작** (`from_failed`) 활용: 이미 완성된 태스크는 cache 사용.

4. **"로그가 단일 파일에 축적"** → 배치 1,000개면 10GB 로그. Prefect는 기본 구조화 로깅 + Prefect Cloud UI. 개별 run ID로 필터링. 별도 JSON 로그는 S3로 주기 저장.

5. **"Prefect Cloud 사용 요금 모르고 과대 트래픽"** → 무료 tier는 월 2만 task run. 이미지 하나가 16 task × 1만 건 = 월 16만. 초과 요금 발생. **자체 호스팅 Prefect Server** (무료) + self-hosted Postgres가 안전.

## 연습문제

**문제 1.** 한 단계가 매번 실패해 pipeline 전체가 멈춘다. 이 단계를 "선택적"으로 만들어 다른 단계는 계속 진행하려면?

> **해설:** Prefect `allow_failure=True` 플래그 사용. 예: `task_vr_scene` 실패해도 `process_document` flow 자체는 진행 (부분 산출물만 생성). 후속 태스크에서 `vr_scene.result(raise_on_failure=False)`로 결과 체크 후 None 처리. 단, 주요 Phase(복원)는 실패하면 중단이 합리적 — 접근성 산출물은 degradation 허용, 원본 복원은 무결성.

**문제 2.** 배치 1,000건 처리 중 503번째에서 SDXL 모델 OOM. 처음 502개는 보존하고 503부터 재개하는 방법은?

> **해설:** Prefect의 **idempotent run**. 각 document의 `run_id = hash(image_path + config)`를 flow_run의 `name`으로 지정. 실패 후 재시작 시 이미 성공한 `run_id`는 skip. 또는 체크포인트 파일 — `{output_root}/.completed.txt`에 완료된 이미지 파일명 append, flow 시작 시 이 파일 참조해 skip. OOM 원인은 별도 해결: `max_concurrent` 축소, GPU 메모리 주기 cleanup(`torch.cuda.empty_cache`), 모델 unload 후 다음 배치.

**문제 3.** Prefect Cloud 대신 자체 호스팅을 선택했을 때 어떤 인프라가 필요한가?

> **해설:** ① **Prefect Server**: Python 기반, Docker 또는 systemd (1 vCPU, 2GB RAM 충분) ② **PostgreSQL**: 메타데이터 저장 (15GB 디스크, 백업 주기 1일) ③ **Worker pool**: 실제 task 실행, GPU 유무에 따라 분리 (`gpu-worker-pool`, `cpu-worker-pool`) ④ **Redis** (선택): 분산 캐시·큐 ⑤ **Grafana + Prometheus**: 메트릭 모니터링. 비용: AWS t3.medium + RDS db.t3.small ≈ $60/월 + GPU 인스턴스 별도. Prefect Cloud 대비 운영 공수 증가하지만 task run 무제한.
