# 챕터 16 · WIA Live — 농인·시각장애인·일반인 3자 실시간 통신

## 학습 목표
- 3자 실시간 통신에서 각 모달리티(수어·음성·텍스트·점자)의 지연 예산을 배분하고 사용자 경험 임계치(P99 1.5초)를 설계할 수 있다
- WebRTC + Socket.IO + 실시간 STT/TTS/수어 변환을 결합한 플랫폼 아키텍처를 구축할 수 있다
- 음성·텍스트·수어 간 양방향 변환 지연을 최소화하는 스트리밍 전략을 적용할 수 있다

---

## 이론

### 왜 3자 통신인가

일반 비디오 통화는 "같은 언어, 같은 감각" 전제. 접근성 도구들은 1:1 변환(수어→텍스트, 음성→텍스트 등)에 집중. **3자 동시 소통**은 거의 없다.

예시 시나리오:
```
A (농인, 한국 수어로 말함)
B (시각장애인, 한국어 음성으로 말함)
C (외국인, 영어 텍스트로 말함)
```

전통 방식: 통역사 2명 이상 + 각자 다른 화면. WIA Live는 **AI 기반 실시간 3자 변환**으로 통역사 없이 소통.

### 지연 예산

사람이 "실시간"으로 느끼는 임계:
- 대화 자연성: **300ms 이하**
- 허용 지연: **1,000ms 이하**
- 불편: 1,500ms+
- 끊김: 3,000ms+

각 변환의 현재 기술 한계 (2026):
| 작업 | 평균 지연 |
|---|---|
| STT (Whisper streaming) | 200~400ms |
| TTS (Piper) | 300~500ms |
| Text → 수어 (concept lookup + keyframe) | 100~300ms |
| 수어 → Text (2D 포즈 추정 → LSTM 분류) | 500~800ms |
| WebRTC 네트워크 | 50~150ms |

**음성 → 수어** = STT(300) + 번역(200) + 수어 렌더(150) = **650ms** (허용 내)  
**수어 → 음성** = 수어 인식(700) + TTS(400) = **1,100ms** (경계)

### 스트리밍 필수

전체 문장 완성 후 처리하면 지연 축적. 각 단계 **부분 결과(partial)** 를 다음 단계로 즉시 전달:

```
음성 흐름:      "안녕하세요 저는"  (아직 계속)
STT 부분 결과:  "안녕하세요"        ← 이 시점 이미 다음 단계로
  ↓
번역:           "Hello"             ← 스트리밍 번역
  ↓
TTS:            부분 합성 시작
```

Whisper의 streaming mode (`faster-whisper` + VAD), OpenAI Realtime API, Claude Realtime (2026 중 출시 예정) 활용.

### 신뢰 등급

실시간은 오류 있기 마련. 사용자에게 **신뢰도 가시화** 필수:
- 확정(confirmed): 검은색 텍스트
- 중간 결과(interim): 회색 이탤릭
- 수정됨(corrected): 취소선 + 새 텍스트

### 아키텍처

```
┌─ Client A (농인)  ─┬─ Signaling Server ─┬─ Client B (시각장애인)
│  웹캠 → 수어 인식   │  (WebSocket)       │  마이크 → Whisper
│                   │                    │  스피커 ← TTS
└───────────────────┤                    ├───────────────────
                    │                    │
                    ▼                    ▼
                 AI Worker Pool (GPU)
                 ├ Whisper STT (A100)
                 ├ Claude Translate
                 ├ Piper TTS
                 ├ Sign Language MT
                 └ Pose Classifier
```

각 워커는 독립 컨테이너. Kafka 또는 Redis Streams로 이벤트 버스.

```python
# WIA Live 서버 아키텍처 (FastAPI + WebSocket + Redis Streams)
# pip install fastapi==0.110 websockets redis==5.0 aiortc==1.7

import asyncio
import json
import uuid
from fastapi import FastAPI, WebSocket, WebSocketDisconnect
from fastapi.middleware.cors import CORSMiddleware
import redis.asyncio as redis
from typing import Dict, Literal
from dataclasses import dataclass, asdict


app = FastAPI(title="WIA Live")
app.add_middleware(
    CORSMiddleware, allow_origins=["*"],
    allow_credentials=True, allow_methods=["*"], allow_headers=["*"],
)


# ──────────────────────────────────────────────
# 1. 참가자·세션 관리
# ──────────────────────────────────────────────
ParticipantMode = Literal["sign", "voice", "text", "text_braille"]


@dataclass
class Participant:
    user_id:     str
    mode:        ParticipantMode     # 이 사용자의 입력/출력 방식
    language:    str = "ko"          # 자연어
    sign_lang:   str = "KSL"         # 수어 언어 (sign 모드일 때)
    ws:          WebSocket = None


class LiveSession:
    def __init__(self, session_id: str):
        self.session_id = session_id
        self.participants: Dict[str, Participant] = {}
        self.transcript = []          # 세션 전체 기록

    def add(self, p: Participant):
        self.participants[p.user_id] = p

    def remove(self, user_id: str):
        self.participants.pop(user_id, None)

    async def broadcast(self, exclude_user: str, payload: dict):
        """나를 제외한 모두에게 전송 (모달리티별 변환 후)"""
        for uid, p in self.participants.items():
            if uid == exclude_user:
                continue
            # 수신자 모달리티에 맞춰 변환
            adapted = await self._adapt_payload(payload, p)
            try:
                await p.ws.send_json(adapted)
            except Exception:
                pass

    async def _adapt_payload(self, payload: dict, receiver: Participant) -> dict:
        """
        송신자 모달리티 → 수신자 모달리티 변환.
        """
        content_type = payload["type"]
        content = payload["content"]
        sender_lang = payload.get("language", "ko")

        # 1) 텍스트 확보 (중간 표현)
        if content_type == "voice_audio":
            text = await stt_stream(content, lang=sender_lang)
        elif content_type == "sign_video":
            text = await sign_recognize(content, sign_lang=payload.get("sign_lang", "KSL"))
        else:
            text = content

        # 2) 수신자 언어로 번역 (필요시)
        if sender_lang != receiver.language:
            text = await translate(text, sender_lang, receiver.language)

        # 3) 수신자 모달리티로 변환
        if receiver.mode == "voice":
            audio_url = await tts_stream(text, lang=receiver.language)
            return {
                "type": "audio", "url": audio_url,
                "text_caption": text, "sender": payload.get("sender"),
            }
        elif receiver.mode == "sign":
            sign_clips = await text_to_sign(text, receiver.sign_lang)
            return {
                "type": "sign_clips", "clips": sign_clips,
                "text_caption": text, "sender": payload.get("sender"),
            }
        elif receiver.mode == "text_braille":
            braille = await text_to_braille(text, lang=receiver.language)
            return {
                "type": "braille", "braille": braille,
                "text": text, "sender": payload.get("sender"),
            }
        else:   # text
            return {
                "type": "text", "text": text,
                "sender": payload.get("sender"),
            }


# ──────────────────────────────────────────────
# 2. 전역 세션 저장소
# ──────────────────────────────────────────────
sessions: Dict[str, LiveSession] = {}


def get_or_create_session(session_id: str) -> LiveSession:
    if session_id not in sessions:
        sessions[session_id] = LiveSession(session_id)
    return sessions[session_id]


# ──────────────────────────────────────────────
# 3. AI 서비스 stub (실제는 별도 워커)
# ──────────────────────────────────────────────
async def stt_stream(audio_bytes: bytes, lang: str = "ko") -> str:
    """Whisper streaming STT — 여기선 간단 호출"""
    # 실제: faster-whisper 서버에 gRPC / HTTP 요청
    await asyncio.sleep(0.3)    # 시뮬레이션
    return "[안녕하세요]"


async def tts_stream(text: str, lang: str = "ko") -> str:
    """Piper TTS — URL 반환"""
    # 실제: 챕터 11 WIAVoiceAPI 호출
    await asyncio.sleep(0.4)
    return f"/cache/audio_{hash(text)}.mp3"


async def translate(text: str, src_lang: str, dst_lang: str) -> str:
    """Claude Realtime 또는 Google Translate API"""
    if src_lang == dst_lang:
        return text
    await asyncio.sleep(0.2)
    return f"[translated {src_lang}→{dst_lang}] {text}"


async def text_to_sign(text: str, sign_lang: str) -> list:
    """챕터 14 수어 변환기 호출"""
    await asyncio.sleep(0.15)
    return [{"word": w, "clip_url": f"/clips/{sign_lang}/{w}.json"}
            for w in text.split()]


async def sign_recognize(video_bytes: bytes, sign_lang: str) -> str:
    """2D 포즈 추정 → 수어 분류 → 텍스트"""
    # MediaPipe Hands + Pose → LSTM 분류
    await asyncio.sleep(0.7)
    return "[수어 인식 결과]"


async def text_to_braille(text: str, lang: str = "ko") -> str:
    """챕터 09/10 점자 변환기"""
    await asyncio.sleep(0.05)
    return "⠛⠣⠒ ⠉⠱⠶ ⠚⠣ ⠠⠝ ⠛⠬"     # 샘플


# ──────────────────────────────────────────────
# 4. WebSocket 엔드포인트
# ──────────────────────────────────────────────
@app.websocket("/ws/{session_id}/{user_id}")
async def live_ws(websocket: WebSocket, session_id: str, user_id: str):
    await websocket.accept()

    # 첫 메시지: 참가자 설정
    init = await websocket.receive_json()
    participant = Participant(
        user_id=user_id,
        mode=init["mode"],
        language=init.get("language", "ko"),
        sign_lang=init.get("sign_lang", "KSL"),
        ws=websocket,
    )

    session = get_or_create_session(session_id)
    session.add(participant)
    await websocket.send_json({
        "type": "session_joined",
        "session_id": session_id,
        "participants": [p.user_id for p in session.participants.values()],
    })

    try:
        while True:
            msg = await websocket.receive_json()
            # 송신: "나"의 발언을 모두에게 브로드캐스트
            payload = {
                "type":      msg["type"],    # "voice_audio" | "sign_video" | "text"
                "content":   msg["content"],
                "sender":    user_id,
                "language":  participant.language,
                "sign_lang": participant.sign_lang,
            }
            await session.broadcast(exclude_user=user_id, payload=payload)
    except WebSocketDisconnect:
        session.remove(user_id)
        for p in session.participants.values():
            await p.ws.send_json({
                "type": "participant_left", "user_id": user_id,
            })


# ──────────────────────────────────────────────
# 5. Health 엔드포인트
# ──────────────────────────────────────────────
@app.get("/")
async def root():
    return {"service": "WIA Live", "sessions": len(sessions)}


@app.get("/sessions/{session_id}")
async def session_info(session_id: str):
    s = sessions.get(session_id)
    if not s:
        return {"error": "not found"}
    return {
        "session_id": session_id,
        "participants": [
            {"user_id": p.user_id, "mode": p.mode, "lang": p.language}
            for p in s.participants.values()
        ],
    }


if __name__ == "__main__":
    import uvicorn
    uvicorn.run(app, host="0.0.0.0", port=8001)
```

## 함정 카드

1. **"모든 변환을 동기 처리"** → STT + 번역 + TTS 순차면 지연 1.5초+. **파이프라인 병렬화**: STT 부분 결과 → 번역 → TTS를 동시 진행 (스트리밍). 완전 병렬은 아니지만 대기 시간 60% 감소.

2. **"수어 인식을 1초에 한 프레임"** → 수어는 초당 3~4 동작. 1fps면 정보 손실. 최소 **10fps 포즈 추정**. MediaPipe는 30fps 가능, 네트워크 대역폭이 병목 — **클라이언트에서 포즈 landmark만 전송** (이미지 압축 불필요, 132 float/frame).

3. **"번역 모델을 매번 cold start"** → Claude/OpenAI API 호출 cold start 500ms+. **세션 시작 시 warm pool 생성**, keep-alive 연결. 또는 로컬 소형 번역 모델(NLLB-200 distilled) 서버 유지.

4. **"참가자 3명+ 확장에서 N×N 변환"** → 3명이면 6개 방향 (각자→상대 2명), 5명이면 20개. **허브 패턴**으로 모두 공통 중간 표현(예: 영어 텍스트)으로 모은 뒤 각자 번역. 변환 수 N×2로 감소.

5. **"세션 녹화·녹음을 미리 고지 안 함"** → GDPR·한국 개인정보보호법 위반. 세션 시작 시 명확한 동의 UI, 녹화 중 표시(빨간 점), **수어 사용자에게는 수어 동의** 애니메이션. 접근성 자체가 법적 리스크 관리.

## 연습문제

**문제 1.** 세션 중 네트워크가 불안정해 1초 끊기면 수어 인식이 실패한다. 어떻게 복원력을 높이는가?

> **해설:** ① **클라이언트 버퍼링**: 끊김 감지 시 최근 2초 프레임을 로컬 저장, 복구 시 일괄 전송 ② **서버 리시프트**: 마지막 완성 문장 이후 처리. 미완성 구간은 재시작 ③ **UX 알림**: 수어 사용자에겐 "연결 복구 중" 시각 표시 (점멸 아이콘), 상대에게는 "상대 연결 불안정" 텍스트 ④ **WebRTC DataChannel** 재시도 로직 — `reliable: true` 대신 부분 순서 허용. 실제로는 완벽 복원 불가 — 끊김이 발생했음을 **명시**해서 오해 방지가 더 중요.

**문제 2.** 3자 통신에서 지연 예산 각 단계를 수치로 설계하라.

> **해설:** **목표: 1,000ms 이하 (대화 자연성은 포기하고 허용 임계)**. 내 분배:
> - WebRTC 입력 → 서버: 100ms
> - STT 스트리밍 (부분 결과): 200ms
> - 번역 (Claude Realtime): 150ms
> - 수신자 모달리티 변환 (TTS 또는 수어 렌더): 300ms
> - 서버 → WebRTC 출력: 100ms
> - **합: 850ms**
> 여유 150ms는 네트워크 지터 버퍼. 초과 시 "부분 결과 전송" 활성화로 사용자 체감 지연 단축. 수어 → 음성은 수어 인식(700ms)이 크니 목표 1,500ms로 상향.

**문제 3.** 프로덕션에서 AI 워커 cost가 월 $5,000이다. 어떻게 절감하는가?

> **해설:** ① **캐시 재사용**: "안녕하세요", "감사합니다" 같은 고빈도 문장의 TTS·수어 클립 Redis 캐시 — 50% 히트율이면 TTS 비용 반감 ② **모델 경량화**: Whisper large → medium (품질 5% 감소, 비용 70% 절감) ③ **배치 처리**: 유휴 워커를 동기식 요청이 아닌 micro-batch(10~50ms 단위)로 묶어 GPU 활용률 향상 ④ **엣지 추론**: 간단 번역(한→영)은 브라우저 WebLLM(Llama 3 1B) — 서버 비용 0 ⑤ **Serverless GPU**: Modal · Replicate 사용량 기반 과금. 피크 외에 대기 비용 없음. 조합 적용으로 $5,000 → $1,800 경험 (64% 절감).
