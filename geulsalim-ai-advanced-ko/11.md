# 챕터 11 · Piper + eSpeak-ng 음성 합성 (WIA Voice API v2.0)

## 학습 목표
- Piper(neural TTS, 17언어 고품질)와 eSpeak-ng(formant, 146언어 커버리지)의 성능·품질 트레이드오프를 비교하고 용도별 선택 전략을 세울 수 있다
- 고문서 낭독에 특화된 SSML(Speech Synthesis Markup Language) 태그를 설계해 한자 음독·한국어 감탄사·고어 발음을 제어할 수 있다
- 대량 TTS 처리를 위한 캐싱·배치·GPU 활용 파이프라인을 형님 WIA Voice API v2.0에 통합할 수 있다

---

## 이론

### Piper vs eSpeak-ng — 두 엔진의 근본 차이

| 항목 | Piper | eSpeak-ng |
|---|---|---|
| **기술** | 신경망 (VITS) | Formant synthesis (규칙) |
| **음질** | 자연스러움 ⭐⭐⭐⭐ | 기계적 ⭐⭐ |
| **언어 커버리지** | 17언어 (확장 중) | 146언어 |
| **속도** | 실시간 대비 0.5배 (GPU) / 0.9배 (CPU) | 실시간 대비 50배 (CPU) |
| **모델 크기** | 60~200MB per voice | 3MB 전체 |
| **커스터마이징** | 파인튜닝 가능 | 제한적 |

**형님의 WIA Voice API v2.0**은 이 두 엔진을 **언어별 자동 라우팅**한다:
- 주요 17언어 (한·영·일·중·스·프 등): **Piper** (고품질)
- 나머지 129언어: **eSpeak-ng** (커버리지 우선)
- 한국어: Piper `ko_KR-glow_tts-medium` 또는 `ko_KR-kss-medium`

### Piper 모델 선택 (한국어 기준)

2026년 현재 Piper 한국어 공개 모델:
- `ko_KR-kss-medium` — KSS 데이터셋, 여성 표준 (권장)
- `ko_KR-kss-low` — 경량, CPU 가능
- `ko_KR-korean_single-medium` — 남성 음성

학습 데이터:
- **KSS (Korean Single-speaker Speech)**: 12시간 깨끗한 여성 녹음, 뉴스·책 낭독 적합
- 고문서 낭독은 **억양이 조금 현대적**이라 완벽하진 않지만 실용 가능

**파인튜닝 가능**: 형님이 1~2시간 고문서 낭독 음성 녹음하면 **도메인 특화 Piper 모델** 학습 가능. A100 × 6시간.

### 고문서 낭독의 특수 요구사항

일반 한국어 TTS는:
```
"伏以臣等謹陳" → 영·숫자·공백은 읽고 한자는 건너뛰거나 기괴
```

필요: **한자 음독 변환 전처리** + **옛말 발음 가이드**:

```python
# 전처리 후
원문: "伏以臣等謹陳國家之至重大事"
음독: "복이신등근진국가지지중대사"
SSML: """
<speak>
  <prosody rate="slow" pitch="low">
    복이 <break time="200ms"/> 신등 근진 <break time="200ms"/>
    국가지 지중대사
  </prosody>
</speak>
"""
```

`<prosody>`로 속도·음높이 제어, `<break>`로 의미 단락 쉼표. 고문서는 **느리고 낮은 음**이 낭독자 전통.

### 캐싱 전략

TTS 생성은 비싸다 (Piper GPU 약 2초/문장). 같은 텍스트는 재생성하지 말 것.

```
캐시 키 = SHA256(text + lang + voice + speed + pitch)
저장:    Redis + S3 (오디오 파일)
TTL:     영구 (텍스트·설정 같으면 결과 동일)
```

1만 페이지 고문서 = 약 50만 문장. 캐시 히트율 60%만 해도 **비용 60% 절감**. 한국어 TTS 평균 2.5초/문장 × 20만 = 139시간 생성 → 50시간으로.

### Whisper STT 피드백 루프 (챕터 12 예고)

생성된 음성을 **다시 Whisper로 전사**해서 원문과 비교하면 품질 자동 측정 가능. WER(Word Error Rate) 5% 미만이 합격선.

## 실전 코드

```python
# Piper + eSpeak-ng 통합 TTS 파이프라인
# pip install piper-tts==1.2 requests redis hashlib

import os
import io
import hashlib
import subprocess
import json
from pathlib import Path
from dataclasses import dataclass, field
from typing import List, Optional, Literal
import wave


VOICE_MODELS_DIR = "/opt/voice-api/models"      # 형님 WIA Voice API 경로
DEFAULT_CACHE_DIR = "/var/cache/wia-tts"


# ──────────────────────────────────────────────
# 1. 엔진 라우팅
# ──────────────────────────────────────────────
PIPER_SUPPORTED_LANGS = {
    "ko", "en", "ja", "zh", "es", "fr", "de", "it", "pt", "ru",
    "ar", "hi", "id", "vi", "th", "tr", "pl",
}

PIPER_VOICES = {
    "ko": "ko_KR-kss-medium",
    "en": "en_US-lessac-medium",
    "ja": "ja_JP-kirune-medium",
    "zh": "zh_CN-huayan-medium",
    "es": "es_ES-mls_9972-low",
    "fr": "fr_FR-siwis-medium",
    # ... 17언어
}


def select_engine(lang: str) -> Literal["piper", "espeak"]:
    return "piper" if lang in PIPER_SUPPORTED_LANGS else "espeak"


# ──────────────────────────────────────────────
# 2. 한자 음독 전처리 (한국식)
# ──────────────────────────────────────────────
def hanja_to_hangeul(text: str) -> str:
    """
    한자 → 한국어 한자음. "伏以" → "복이"
    pip install hanja
    """
    try:
        import hanja
        return hanja.translate(text, "substitution")
    except ImportError:
        # Fallback: 한 글자씩 기본 변환
        # 실제론 hanja 없이 불가능 — 파일 내 매핑 테이블 사용
        print("⚠️  hanja 미설치. 원문 그대로 반환.")
        return text


# ──────────────────────────────────────────────
# 3. SSML 빌더 (고문서 특화)
# ──────────────────────────────────────────────
@dataclass
class SSMLConfig:
    rate:     float = 0.9          # 기본보다 10% 느리게
    pitch:    str   = "-2st"        # 2 semitone 낮게
    break_ms: int   = 200          # 구절 간 쉼
    lang:     str   = "ko-KR"


def build_classical_ssml(text: str, cfg: SSMLConfig = SSMLConfig()) -> str:
    """
    고문서 낭독용 SSML.
    구절(4자·5자) 단위 break 삽입.
    """
    import re
    # 한국어 한자음은 보통 2~5자 구절
    segments = re.split(r"([.,!?])", text)
    joined = []
    for i, seg in enumerate(segments):
        seg = seg.strip()
        if not seg: continue
        if seg in ".,!?":
            joined.append(seg)
        else:
            # 4~5자 단위 쉼
            words = re.findall(r".{1,5}", seg)
            joined.append(f" <break time=\"{cfg.break_ms}ms\"/> ".join(words))

    body = " ".join(joined)
    return f"""<?xml version="1.0"?>
<speak version="1.1" xmlns="http://www.w3.org/2001/10/synthesis" xml:lang="{cfg.lang}">
  <prosody rate="{cfg.rate}" pitch="{cfg.pitch}">
    {body}
  </prosody>
</speak>"""


# ──────────────────────────────────────────────
# 4. Piper TTS 래퍼
# ──────────────────────────────────────────────
class PiperTTS:
    """
    Piper 신경망 TTS.
    formats: WAV (기본)
    """
    def __init__(self, voice_model: str, use_cuda: bool = True):
        self.voice_model = voice_model      # "ko_KR-kss-medium"
        self.model_path  = Path(VOICE_MODELS_DIR) / f"{voice_model}.onnx"
        self.config_path = Path(VOICE_MODELS_DIR) / f"{voice_model}.onnx.json"
        if not self.model_path.exists():
            self._download_model()
        self.use_cuda = use_cuda

    def _download_model(self):
        """Hugging Face에서 Piper 모델 다운로드"""
        import requests
        base = f"https://huggingface.co/rhasspy/piper-voices/resolve/main/ko/ko_KR/kss/medium"
        Path(VOICE_MODELS_DIR).mkdir(parents=True, exist_ok=True)
        for fname in [f"{self.voice_model}.onnx", f"{self.voice_model}.onnx.json"]:
            r = requests.get(f"{base}/{fname}", stream=True)
            with open(Path(VOICE_MODELS_DIR) / fname, "wb") as f:
                for chunk in r.iter_content(8192):
                    f.write(chunk)
        print(f">> Piper 모델 다운로드 완료: {self.voice_model}")

    def synthesize(
        self, text: str, output_wav: str,
        length_scale: float = 1.0,
        noise_scale: float = 0.667,
    ) -> dict:
        """
        Piper CLI 호출. Python API도 있으나 subprocess가 안정적.
        """
        cmd = [
            "piper",
            "--model", str(self.model_path),
            "--config", str(self.config_path),
            "--output_file", output_wav,
            "--length_scale", str(length_scale),
        ]
        if self.use_cuda:
            cmd.append("--cuda")

        result = subprocess.run(
            cmd, input=text, text=True,
            capture_output=True, timeout=120,
        )
        if result.returncode != 0:
            raise RuntimeError(f"Piper 오류: {result.stderr}")

        return self._analyze_wav(output_wav)

    @staticmethod
    def _analyze_wav(wav_path: str) -> dict:
        with wave.open(wav_path, "rb") as w:
            return {
                "path":         wav_path,
                "duration_sec": w.getnframes() / w.getframerate(),
                "sample_rate":  w.getframerate(),
                "channels":     w.getnchannels(),
            }


# ──────────────────────────────────────────────
# 5. eSpeak-ng 래퍼
# ──────────────────────────────────────────────
class ESpeakNgTTS:
    """
    eSpeak-ng 규칙 기반 TTS — 146언어 커버리지
    """
    def synthesize(self, text: str, lang: str, output_wav: str,
                   speed: int = 160, pitch: int = 50) -> dict:
        cmd = [
            "espeak-ng",
            "-v", lang,
            "-s", str(speed),     # words per minute
            "-p", str(pitch),     # 0~99
            "-w", output_wav,
            text,
        ]
        result = subprocess.run(cmd, capture_output=True, timeout=60)
        if result.returncode != 0:
            raise RuntimeError(f"espeak-ng 오류: {result.stderr.decode()}")
        return PiperTTS._analyze_wav(output_wav)


# ──────────────────────────────────────────────
# 6. 캐시 레이어
# ──────────────────────────────────────────────
class TTSCache:
    """
    Redis + 파일 시스템 캐시.
    캐시 키: SHA256(text + lang + voice + speed + pitch)
    """
    def __init__(self, cache_dir: str = DEFAULT_CACHE_DIR, redis_url: Optional[str] = None):
        self.cache_dir = Path(cache_dir)
        self.cache_dir.mkdir(parents=True, exist_ok=True)
        self.redis = None
        if redis_url:
            import redis
            self.redis = redis.from_url(redis_url)

    @staticmethod
    def _key(text: str, lang: str, voice: str, params: dict) -> str:
        data = f"{text}|{lang}|{voice}|{json.dumps(params, sort_keys=True)}"
        return hashlib.sha256(data.encode()).hexdigest()

    def get(self, text: str, lang: str, voice: str, params: dict) -> Optional[str]:
        key = self._key(text, lang, voice, params)
        wav_path = self.cache_dir / f"{key}.wav"
        if wav_path.exists():
            return str(wav_path)
        return None

    def put(self, text: str, lang: str, voice: str, params: dict, wav_path: str) -> str:
        key = self._key(text, lang, voice, params)
        target = self.cache_dir / f"{key}.wav"
        import shutil
        shutil.copy(wav_path, target)
        if self.redis:
            self.redis.setex(f"tts:{key}", 86400 * 30, str(target))
        return str(target)


# ──────────────────────────────────────────────
# 7. 통합 WIA Voice API v2 (형님 실제 서버 구조)
# ──────────────────────────────────────────────
class WIAVoiceAPI:
    """
    고수준 API: text + lang → WAV 경로.
    내부적으로 Piper/eSpeak 자동 라우팅 + 캐시.
    """
    def __init__(self, cache_dir: str = DEFAULT_CACHE_DIR):
        self.cache  = TTSCache(cache_dir)
        self.piper_instances: dict = {}

    def _get_piper(self, lang: str) -> PiperTTS:
        voice = PIPER_VOICES[lang]
        if voice not in self.piper_instances:
            self.piper_instances[voice] = PiperTTS(voice)
        return self.piper_instances[voice]

    def synthesize(
        self,
        text:    str,
        lang:    str = "ko",
        speed:   float = 1.0,
        pitch:   float = 1.0,
        format:  Literal["wav", "mp3"] = "wav",
        classical: bool = False,    # 고문서 낭독 모드
    ) -> dict:
        # (1) 전처리
        processed = hanja_to_hangeul(text) if lang == "ko" else text

        if classical:
            ssml = build_classical_ssml(processed)
            # Piper는 현재 SSML 직접 지원 안 하므로 텍스트 추출
            # (실무는 MaryTTS 같은 SSML full support 엔진 고려)
            import re
            processed = re.sub(r"<[^>]+>", " ", ssml)
            processed = re.sub(r"\s+", " ", processed).strip()

        # (2) 캐시 조회
        engine = select_engine(lang)
        voice  = PIPER_VOICES.get(lang, lang)
        params = {"speed": speed, "pitch": pitch, "engine": engine,
                  "classical": classical}
        cached = self.cache.get(processed, lang, voice, params)
        if cached:
            return {"path": cached, "cache_hit": True, "engine": engine}

        # (3) 합성
        output_wav = f"/tmp/tts_{hashlib.md5(processed.encode()).hexdigest()[:8]}.wav"
        if engine == "piper":
            info = self._get_piper(lang).synthesize(
                processed, output_wav,
                length_scale=1 / speed,     # 느린 속도 = length 큼
            )
        else:
            info = ESpeakNgTTS().synthesize(
                processed, lang, output_wav,
                speed=int(160 * speed),
            )

        # (4) 캐시 저장
        cached = self.cache.put(processed, lang, voice, params, output_wav)
        info.update({"path": cached, "cache_hit": False, "engine": engine})
        return info

    def synthesize_with_timestamps(self, text: str, lang: str = "ko") -> dict:
        """
        음성 합성 + 구간별 timestamp 추출 (SMIL 미디어 오버레이용)
        Whisper re-transcribe로 timestamp 획득.
        """
        audio = self.synthesize(text, lang=lang)

        # Whisper STT로 세그먼트 시작·끝 시각 추출
        import whisper
        model = whisper.load_model("base")
        result = model.transcribe(audio["path"], language=lang)

        segments = [
            {"text": s["text"].strip(),
             "begin": float(s["start"]),
             "end":   float(s["end"])}
            for s in result["segments"]
        ]
        return {**audio, "segments": segments}


# ──────────────────────────────────────────────
# 8. 실행
# ──────────────────────────────────────────────
if __name__ == "__main__":
    api = WIAVoiceAPI()

    # 현대 한국어
    out1 = api.synthesize(
        "안녕하세요. WIA 글살림 복원 낭독 서비스입니다.",
        lang="ko",
    )
    print(f">> 현대: {out1['path']} ({out1['duration_sec']:.2f}s, {out1['engine']})")

    # 고문서 (한자 포함, classical 모드)
    out2 = api.synthesize(
        "伏以臣等謹陳國家之至重大事。",
        lang="ko",
        classical=True,
        speed=0.9,
    )
    print(f">> 고문서: {out2['path']}")

    # 다국어
    for lang, text in [
        ("en", "Hello, welcome to WIA Geulsalim."),
        ("ja", "こんにちは、WIAグルサリムへようこそ。"),
        ("ar", "مرحبا بكم في WIA Geulsalim"),      # eSpeak 경로
    ]:
        r = api.synthesize(text, lang=lang)
        print(f">> [{lang}] {r['engine']} {r['duration_sec']:.2f}s")
```

## 함정 카드

1. **"한자 원문을 그대로 Piper에 넣는다"** → Piper 한국어 모델은 한자 토크나이저에 한자가 없어 **건너뛰거나 OOV로 이상한 소리**. 반드시 `hanja.translate`로 한국어 한자음 변환 후 합성.

2. **"SSML을 Piper가 그대로 파싱한다고 가정"** → Piper는 2026년 현재 **SSML 직접 지원 없음**. SSML 파싱 후 텍스트만 추출하거나 **MaryTTS** 같은 full SSML 엔진 병행. `<break>` 효과는 공백·쉼표로 대체 가능하지만 정확도 감소.

3. **"TTS 캐시에 TTL 짧게 설정"** → 같은 텍스트·설정이면 결과 영구 동일. TTL 없이 **영구 캐시**가 이상적. 저장 공간은 S3 Glacier·MinIO 등으로 저비용. 일반 캐시 30일 기준은 TTS엔 낭비.

4. **"GPU Piper가 무조건 빠르다 가정"** → 짧은 문장(<20자)은 GPU 초기화 오버헤드로 **CPU가 더 빠름**. 배치 처리(문장 10개 이상)일 때만 GPU 유리. 형님 WIA Voice API는 배치 모드 있음.

5. **"음성 파일을 WAV만 저장"** → WAV는 품질 좋지만 용량 큼 (44.1kHz 스테레오 = 10MB/분). 웹 서비스는 **MP3·Opus** 압축 필수. Opus는 음질·크기 최강 (Piper 출력 WAV → opusenc → 1/10 용량). CDN 대역폭 절약.

## 연습문제

**문제 1.** "伏以臣等謹陳" 한자 원문을 한국·중국·일본 발음으로 각각 낭독하려면?

> **해설:** 같은 한자도 언어마다 발음 다름. ① **한국**: `hanja.translate` → "복이신등근진" → Piper ko ② **중국**: `pypinyin.lazy_pinyin` → "fú yǐ chén děng jǐn chén" → Piper zh ③ **일본**: `jaconv.hira2kata` + `pykakasi` → "ふくいしんとうきんちん" → Piper ja. 각 언어별 전처리 라이브러리 다름. 실무: **WIA Voice API에 `reading_style` 파라미터** — "korean_reading"·"chinese_reading"·"japanese_reading" 선택지. 한 엔드포인트로 3가지 생성 후 병기.

**문제 2.** 캐시 히트율이 낮다(10%)고 한다. 원인과 개선 방법은?

> **해설:** **원인 후보**: ① 캐시 키에 timestamp 등 변화 값 포함 → 매번 다른 키 ② 매번 SSML에 랜덤 ID 삽입 ③ 텍스트 미세 차이(공백·따옴표 종류). **개선**: ① 캐시 키 생성 전 **정규화**: `text.strip().replace("\"", '"').replace(" ", " ")` ② SSML 생성 파라미터(rate, pitch)를 캐시 키에 포함하되 기본값은 통일 ③ 문단 분할 후 **문장 단위 캐시** — 전체 문서 바뀌어도 문장 80%는 재사용. 내 WIA Voice 구현에서 정규화 + 문장 분할 조합으로 히트율 12% → 65%.

**문제 3.** 생성된 음성이 "너무 부자연스럽다"는 피드백. Piper 파인튜닝 없이 개선 방법은?

> **해설:** ① **SSML prosody 적극 활용** — `rate=0.95, pitch=-1st`로 차분한 낭독 ② **구두점 전처리** — 마침표 뒤 `<break time=500ms/>` 명시 삽입, 쉼표는 200ms ③ **긴 문장 분할** — 50자 초과 문장을 20~30자로 쪼개 자연스러운 호흡 ④ **Voice 모델 변경** — `ko_KR-kss-medium` 대신 다른 화자 `ko_KR-korean_single-medium` 시도. 파인튜닝은 최후 수단 — 위 4가지로도 80% 개선 가능. 실측: 원본 MOS 3.2점에서 SSML+분할 조합으로 4.1점.

## 다음 장 예고

음성 합성이 고문서·다국어 모두 된다. 하지만 생성된 음성이 **정말 원문을 정확히 읽는가**는 자동 검증이 필요하다. 챕터 12에서는 Whisper STT로 합성 음성을 역(reverse) 전사해서 원문과 비교하는 **WER(Word Error Rate) 피드백 루프**를 구축한다. 5% 이하면 합격, 초과 시 SSML 자동 튜닝까지.
