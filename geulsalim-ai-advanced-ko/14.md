# 챕터 14 · 한국 수어(KSL) vs ASL · 다국어 수어 스위칭

## 학습 목표
- 한국 수어(KSL)·미국 수어(ASL)·일본 수어(JSL)·국제 수어(IS)의 언어 간 구조 차이를 설명할 수 있다
- 수어 사전 데이터 포맷 SignWriting과 HamNoSys를 비교하고 WIA Talk 내부 표현을 선택할 수 있다
- 복원된 고문서의 한국어 문장을 여러 국가 수어로 동시 변환하는 라우팅 시스템을 구축할 수 있다

---

## 이론

### 수어는 하나의 언어가 아니다

오해: "수어는 세계 공통"  
진실: **각 나라마다 독립된 언어**다. 한국인 농인과 미국 농인은 수어로 대화 불가.

주요 수어:
| 언어 | 화자 수 (추정) | 문법 구조 | 특징 |
|---|---|---|---|
| **ASL** (미국) | 50만 | SVO → Topic-Comment | Classifier 체계 발달 |
| **KSL** (한국) | 40만 | SOV 기반 | 한국어 어순 유사, 지화(지문자) 한글 |
| **JSL** (일본) | 32만 | SOV | 口型(입모양) 중요 |
| **BSL** (영국) | 15만 | OSV | ASL과 전혀 다름 |
| **IS** (국제 수어) | — | Pidgin | 국제 회의용, 문법 최소 |

국제 수어(IS)는 **완전한 언어 아니다** — 각국 수어의 공통 어휘 기반 축약. 회의·올림픽용.

### 단어·문법·공간 사용의 차이

**같은 "고맙다" 수어:**
- **KSL**: 손바닥을 턱 아래에서 앞으로 뻗음
- **ASL**: 손끝을 입술에 대었다가 앞으로 내밀고 손바닥 위로 펼침
- **JSL**: 양손을 가슴 앞에서 살짝 앞으로 숙임

**공간 사용:**
- ASL: 3인칭 위치 지시 정교함 (왼쪽 공간 = 그 사람 A, 오른쪽 = B)
- KSL: 3인칭 공간 활용 있지만 ASL보다 덜 정교
- JSL: 공간 시간축 (왼쪽 = 과거, 오른쪽 = 미래) 관습

### 수어 데이터 포맷

**1. SignWriting (수어쓰기)**
- 수어를 **시각 기호**로 표기하는 문자 체계
- ISWA 2010 기호 세트
- 유니코드 U+1D800~U+1DAAF
- 장점: 사람이 읽고 쓸 수 있음
- 단점: 애니메이션용 데이터로 변환 필요

**2. HamNoSys (Hamburg Notation System)**
- 언어학 연구용 표기
- 손 모양·위치·방향·움직임 세밀 기술
- ASCII/유니코드 기호 조합
- 장점: 3D 애니메이션으로 직접 변환 가능
- 단점: 전문가만 읽음

**3. SiGML (Signing Gesture Markup Language)**
- HamNoSys 기반 XML
- JASigning·eSIGN 시스템이 애니메이션화
- 장점: 표준화된 XML

**WIA Talk 내부 표현**: SiGML 기반 + 형님의 JSON 확장 (Quaternion keyframes). HamNoSys 읽기/쓰기는 변환 레이어.

### 언어 간 Cross-language Sign Transfer

같은 "물"(water) 개념을 KSL·ASL·JSL에서 다르게 수어한다. 자동 번역 전략:

**방법 1 — Direct Lookup** (1:1 사전)
```
water (EN) → KSL: 물수어.json
water (EN) → ASL: water_asl.json
water (EN) → JSL: water_jsl.json
```
단점: 10만 단어 × 언어 수 = 관리 부담

**방법 2 — Semantic Concept ID**
```
Concept: C_WATER
├ KSL: {sign_clip_korean}
├ ASL: {sign_clip_american}
├ JSL: {sign_clip_japanese}
└ IS:  {sign_clip_international}
```
단어가 아닌 **개념(ontology)** 에 기반. ConceptNet·WordNet 활용.
장점: 단어 동의어 통합, 언어 추가 용이
단점: 개념 정의 자체가 어렵고 문화별 다름

**방법 3 — GPT/Claude 번역 + 수어 검색**
```
원문 (KR) → LLM → 수어 대상 언어 자연어 (US) → ASL 사전 검색
```
실용적이지만 이중 번역 오류 누적.

**WIA Talk 권장**: 방법 2 (Concept ID) + 방법 3 fallback. 주요 5,000 개념은 수동 매핑, 나머지는 LLM.

```python
# 다국어 수어 라우팅 + Concept ID 시스템
# pip install transformers torch

import json
import re
from pathlib import Path
from dataclasses import dataclass, field
from typing import List, Dict, Optional, Literal
from enum import Enum


SignLanguage = Literal["KSL", "ASL", "JSL", "BSL", "IS"]


# ──────────────────────────────────────────────
# 1. Concept ID 시스템
# ──────────────────────────────────────────────
@dataclass
class SignConcept:
    """
    언어 독립 개념. 각 수어 언어별 클립을 하위로 둠.
    """
    concept_id:    str                  # "C_WATER", "C_THANKS" 등
    canonical_en:  str                  # "water"
    aliases:       Dict[str, List[str]] = field(default_factory=dict)   # {"ko": ["물"], "ja": ["水"]}
    clips:         Dict[SignLanguage, str] = field(default_factory=dict) # {"KSL": "path/to/water_ksl.json"}
    category:      str = "general"      # noun, verb, adj, classifier 등


class ConceptRegistry:
    """
    Concept → 각국 수어 클립 파일 경로 매핑.
    """
    def __init__(self, registry_json: str):
        with open(registry_json, encoding="utf-8") as f:
            raw = json.load(f)
        self.concepts: Dict[str, SignConcept] = {
            c["concept_id"]: SignConcept(**c) for c in raw
        }
        self._build_text_index()

    def _build_text_index(self):
        """각 언어 텍스트 → concept_id 역색인"""
        self.text_index: Dict[str, Dict[str, str]] = {
            "ko": {}, "en": {}, "ja": {}, "zh": {},
        }
        for cid, c in self.concepts.items():
            self.text_index["en"][c.canonical_en.lower()] = cid
            for lang, words in c.aliases.items():
                for w in words:
                    self.text_index.setdefault(lang, {})[w] = cid

    def lookup_by_text(self, text: str, lang: str) -> Optional[SignConcept]:
        cid = self.text_index.get(lang, {}).get(text.strip().lower())
        return self.concepts.get(cid) if cid else None

    def get_clip_path(self, concept_id: str, sign_lang: SignLanguage) -> Optional[str]:
        c = self.concepts.get(concept_id)
        if c is None:
            return None
        return c.clips.get(sign_lang)


# ──────────────────────────────────────────────
# 2. 자연어 → 수어 시퀀스 변환
# ──────────────────────────────────────────────
class SignLanguageTranslator:
    """
    한국어/영어 문장 → 수어 클립 시퀀스 (목표 수어 언어로)
    """
    def __init__(
        self,
        registry: ConceptRegistry,
        llm_fallback: Optional[callable] = None,
    ):
        self.reg = registry
        self.llm = llm_fallback    # Claude/GPT: (text, src_lang, dst_lang) → text

    def translate(
        self,
        text:       str,
        src_lang:   str = "ko",
        target_sign: SignLanguage = "KSL",
    ) -> List[dict]:
        """
        결과: [{"word": "물", "concept": "C_WATER", "clip": "path", "mode": "lookup"}, ...]
        """
        results = []

        # 어휘 토큰화 (공백·구두점 기준)
        tokens = re.findall(r"\w+", text)

        for token in tokens:
            # 1) 직접 사전 검색
            concept = self.reg.lookup_by_text(token, src_lang)
            if concept:
                clip = self.reg.get_clip_path(concept.concept_id, target_sign)
                if clip:
                    results.append({
                        "word":    token,
                        "concept": concept.concept_id,
                        "clip":    clip,
                        "mode":    "lookup",
                    })
                    continue

            # 2) LLM fallback — 영어로 변환 후 개념 검색
            if self.llm:
                en_word = self.llm(token, src_lang, "en")
                concept = self.reg.lookup_by_text(en_word, "en")
                if concept:
                    clip = self.reg.get_clip_path(concept.concept_id, target_sign)
                    if clip:
                        results.append({
                            "word":       token,
                            "via_en":     en_word,
                            "concept":    concept.concept_id,
                            "clip":       clip,
                            "mode":       "llm_bridge",
                        })
                        continue

            # 3) 지문자 (finger-spelling) fallback
            results.append({
                "word":   token,
                "mode":   "finger_spelling",
                "hangul": [c for c in token if 0xAC00 <= ord(c) <= 0xD7A3],
            })

        return results


# ──────────────────────────────────────────────
# 3. 한국어 지문자(지화) 생성
# ──────────────────────────────────────────────
# 한글 자모 지문자 매핑 (clip 파일명)
HANGUL_FINGERSPELLING = {
    "ㄱ": "hangul_giyeok.json", "ㄴ": "hangul_nieun.json",
    "ㄷ": "hangul_digeut.json", "ㄹ": "hangul_rieul.json",
    "ㅁ": "hangul_mieum.json",  "ㅂ": "hangul_bieup.json",
    "ㅅ": "hangul_siot.json",   "ㅇ": "hangul_ieung.json",
    "ㅈ": "hangul_jieut.json",  "ㅊ": "hangul_chieut.json",
    "ㅋ": "hangul_kieuk.json",  "ㅌ": "hangul_tieut.json",
    "ㅍ": "hangul_pieup.json",  "ㅎ": "hangul_hieut.json",
    "ㅏ": "hangul_a.json",      "ㅑ": "hangul_ya.json",
    "ㅓ": "hangul_eo.json",     "ㅕ": "hangul_yeo.json",
    "ㅗ": "hangul_o.json",      "ㅛ": "hangul_yo.json",
    "ㅜ": "hangul_u.json",      "ㅠ": "hangul_yu.json",
    "ㅡ": "hangul_eu.json",     "ㅣ": "hangul_i.json",
}


def word_to_finger_spelling(word: str) -> List[str]:
    """
    한글 단어 → 지문자 클립 리스트.
    자모 분해 후 각각 지문자로.
    """
    from advanced_ch10 import decompose_hangul
    clips = []
    for ch in word:
        cho, jung, jong = decompose_hangul(ch)
        if jung:    # 한글 음절
            if cho: clips.append(HANGUL_FINGERSPELLING.get(cho))
            if jung: clips.append(HANGUL_FINGERSPELLING.get(jung))
            if jong: clips.append(HANGUL_FINGERSPELLING.get(jong))
    return [c for c in clips if c]


# ──────────────────────────────────────────────
# 4. 문장 전체 → 수어 시퀀스
# ──────────────────────────────────────────────
def translate_sentence(
    sentence: str,
    src_lang: str = "ko",
    target_sign: SignLanguage = "KSL",
    registry_path: str = "./data/concepts.json",
) -> dict:
    """
    문장 하나를 수어 클립 시퀀스로 변환.
    지문자 fallback 포함.
    """
    reg = ConceptRegistry(registry_path)
    translator = SignLanguageTranslator(reg)

    tokens = translator.translate(sentence, src_lang=src_lang, target_sign=target_sign)

    # 지문자 해석
    for t in tokens:
        if t["mode"] == "finger_spelling":
            t["finger_clips"] = word_to_finger_spelling(t["word"])

    # 순서 조정: KSL은 SOV 어순 준수 필요 (원문 어순이 한국어면 대부분 OK)
    # ASL 목표면 S→Topic→Comment 로 재배열 필요 (LLM 또는 규칙)
    if target_sign == "ASL":
        tokens = _reorder_to_asl(tokens)

    return {
        "source":      sentence,
        "src_lang":    src_lang,
        "target_sign": target_sign,
        "tokens":      tokens,
        "total_clips": sum(
            len(t.get("finger_clips", [])) if t["mode"] == "finger_spelling" else 1
            for t in tokens
        ),
    }


def _reorder_to_asl(tokens: List[dict]) -> List[dict]:
    """
    간이 ASL 어순 변환.
    "나는 사과를 먹는다" (SOV) → "나 사과 먹다" (SVO 또는 Topic-Comment).
    실무: 규칙 + LLM 조합.
    """
    # Placeholder — 실제 구현은 POS 태깅 + 규칙 기반
    return tokens


# ──────────────────────────────────────────────
# 5. Concept Registry 샘플 데이터
# ──────────────────────────────────────────────
SAMPLE_REGISTRY = [

        "canonical_en": "water",
        "aliases": {
            "ko": ["물", "워터"],
            "ja": ["水", "みず"],
            "zh": ["水", "shuǐ"],
        },
        "clips": {
            "KSL": "./clips/ksl/water.json",
            "ASL": "./clips/asl/water.json",
            "JSL": "./clips/jsl/water.json",
            "IS":  "./clips/is/water.json",
        },
        "category": "noun",
    },
    {
        "concept_id":   "C_THANKS",
        "canonical_en": "thank_you",
        "aliases": {
            "ko": ["감사", "고맙다", "고마워"],
            "ja": ["ありがとう", "感謝"],
        },
        "clips": {
            "KSL": "./clips/ksl/thanks.json",
            "ASL": "./clips/asl/thanks.json",
        },
        "category": "expression",
    },
    # ... 5,000 concepts
]


if __name__ == "__main__":
    # 샘플 registry 생성
    Path("./data").mkdir(exist_ok=True)
    with open("./data/concepts.json", "w", encoding="utf-8") as f:
        json.dump(SAMPLE_REGISTRY, f, ensure_ascii=False, indent=2)

    # 한국어 문장 → KSL
    result_ksl = translate_sentence(
        "물 감사합니다", src_lang="ko", target_sign="KSL",
    )
    print("=== KSL ===")
    print(json.dumps(result_ksl, ensure_ascii=False, indent=2))

    # 같은 문장 → ASL (LLM bridge)
    result_asl = translate_sentence(
        "물 감사합니다", src_lang="ko", target_sign="ASL",

    print("\n=== ASL ===")
```

## 함정 카드

1. **"1:1 단어 번역으로 수어 완성"** → 수어는 문법·어순이 다르다. "나는 밥을 먹었다"를 ASL로 단어 직역하면 어색. **Topic-Comment 재배열**("밥 나 먹다 완료") 필수. 규칙 기반 + LLM 검토 조합.

2. **"지문자(지화)를 과도하게 사용"** → 농인은 지문자로 전체 문장 읽으면 피곤해한다. **고유명사·신조어만 지문자**, 일반 단어는 반드시 수어 사전. 미등록 단어 많으면 사전 확장 우선.

3. **"ASL classifier를 단순 명사 취급"** → ASL의 classifier (예: CL:3 = 차량)는 형태소지 단어가 아님. 단어 사전 매핑으로만 처리하면 **classifier가 포함된 문장이 깨짐**. 언어학자 자문 필수.

4. **"표정(non-manual markers) 무시"** → 수어는 표정도 문법이다. ASL은 의문문에서 눈썹 올림. KSL은 부정문에서 고개 돌림. 3D 아바타의 표정 블렌드셰이프가 없으면 **문장 의미 손실**.

5. **"Concept 매핑 없는 단어를 지문자 fallback"** → 대량 지문자는 UX 나쁨. **의역/우회**: "친환경" 개념 매핑 없으면 "자연 + 좋다"로 합성. 수어 번역 품질의 핵심은 **의미 전달이지 단어 일치가 아님**.

## 연습문제

**문제 1.** KSL로 수어 비디오 제작 후, 같은 아바타·동작으로 ASL 사용자에게 "비슷하게" 이해시킬 수 있는가?

> **해설:** **단어는 거의 이해 못하지만, 국제 수어(IS) 어휘로 선별된 기본 단어 50~100개는 가능**. 올림픽·UN에서 쓰이는 IS 공통 어휘(안녕·감사·도와주세요 등). 이 범위만 공통 녹화하고 나머지는 언어별 분리. 또는 **bilingual subtitle** — 화면 하단에 해당 수어 언어 텍스트 캡션. 완전 이해는 불가능하나 초보 소통 가능. 실무: 글로벌 데모는 IS로 녹화, 국가별 완전판은 해당 국가 수어.

**문제 2.** Concept Registry가 커질수록 검색 O(N)이다. 10만 개념 규모에서 어떻게 최적화?

> **해설:** ① **메모리 해시맵**: 이미 구현 (text_index dict) — O(1) 검색 ② **Trie**: 부분 매칭이 필요하면 prefix tree ③ **임베딩 기반 유사 검색**: 사용자가 "죽음"을 찾는데 사전엔 "사망"만 있을 때, SentenceTransformer 임베딩 유사도 top-3 반환 ④ **DB 인덱싱**: PostgreSQL + pg_trgm으로 fuzzy search + 메모리 캐시. 10만 개념은 메모리 수 MB라 RAM 캐시가 최적. 검색 레이어 다단계: exact → fuzzy → semantic.

**문제 3.** "과학적 용어(mRNA, CRISPR 등)"가 사전에 없다. 수어 번역은?

> **해설:** ① **지문자 (finger-spelling)**: 알파벳·한글 자모로 써줌. 단점: 길다. ② **의역 + 설명**: "MRNA → 메신저 리보핵산 (유전 정보 전달자)" 같이 풀어 수어. ③ **등록 요청**: 한국농아인협회 등에 새 수어 표준 건의, 커뮤니티 승인 후 사전 등록. ④ **컨텍스트 반복**: 첫 언급 시 지문자 + 설명, 이후는 고유 손짓 (lexicalized) — 수어 커뮤니티 자연스러운 어휘 확장 방식. **자동 해결 X, 수어 언어학자·커뮤니티 협력 필수**.

## 다음 장 예고

3D 수어가 다국어 지원된다. 하지만 평면 2D 비디오로 보는 것과 **3D 공간에서 아바타와 함께 고문서 관람**하는 경험은 다르다. 챕터 15에서는 WebXR로 VR 공간에서 고문서를 펼쳐놓고 3D 수어 아바타가 옆에서 설명하는 몰입 경험을 구축한다. 촉각 피드백(햅틱)과 공간 오디오까지.
