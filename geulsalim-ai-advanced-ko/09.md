# 챕터 09 · WIA Braille — IPA 기반 254언어 점자 변환

## 학습 목표
- 점자의 물리적 구조(6점·8점)와 표기 체계(Grade 1·Grade 2)의 차이를 설명할 수 있다
- IPA(국제 음성 기호) 기반 공통 점자 매핑으로 254 언어를 단일 엔진으로 변환하는 원리를 이해한다
- 한국어·영어·아랍어·태국어 등 다양한 문자 체계의 점자 변환 코드를 작성할 수 있다

---

## 이론

### 점자 기본 — 6점 vs 8점

점자는 **6점 셀(2×3)** 이 국제 표준(Louis Braille 1824). **8점 셀(2×4)** 은 컴퓨터용 (유니코드 ⠿⣿).

```
6점          8점
⠿  ⠄         ⣿  ⠄
⠿  ⠂         ⠿  ⠂
⠿  ⠁         ⠿  ⠁
            ⣿  ⠈

bit 배치: 
  1 4
  2 5
  3 6
(7 8 — 8점에만)
```

유니코드 점자는 U+2800 ~ U+28FF 범위. `⠁`(U+2801, 1점)부터 `⣿`(U+28FF, 전체) 까지 256자.

### Grade 1 vs Grade 2

- **Grade 1**: 1대1 문자 대응 (abcdef → ⠁⠃⠉⠙⠑⠋). 초보·이름·기술 용어.
- **Grade 2 (Contracted)**: 축약형. "the"→⠮, "and"→⠯. 영어 점자 95% 실사용.

한국어 점자:
- **한글 점자 Grade 1**: 초성·중성·종성 단위 (KS X 6201)
- **Grade 2 축약**: 자주 쓰는 단어(것·수·이·그 등)를 짧게

### IPA 기반 통일 변환의 아이디어

254개 언어 × 각기 다른 점자 규칙을 개별 매핑하면 유지보수 지옥. **대안**: 모든 언어를 IPA 음성 기호로 먼저 변환 → IPA → 점자 단일 규칙.

```
한국어  "안녕" → IPA [an njʌŋ] → 점자 ⠁⠉⠊⠒⠁⠒
영어    "hello" → IPA [həˈloʊ] → 점자 (유사)
중국어  "你好"  → IPA [ni˨˩hɑʊ˨˩] → 점자
```

장점:
- 공통 엔진, 언어별 로케일만 추가
- IPA→점자 규칙이 언어 독립적

한계:
- 성조·강세 표기는 언어별 특수 기호 필요
- 완전 1:1 매핑 불가능한 언어 있음 (중국어 汉字 → 병음 변환 후 IPA)

**실무 전략**: IPA를 기본 경로로, **중요 언어는 언어별 최적화 모듈**도 갖추는 이중 구조. 한국어·영어·일본어는 최적화 모듈, 나머지 200+ 언어는 IPA 경로.

### 형님의 WIA Braille 시스템

- **254언어** IPA 기반 변환 지원
- **Piper + eSpeak-ng** 음성 합성 엔진이 이미 IPA 레벨로 작동 → 점자도 같은 경로 공유
- WIA Standards 765개 공개 표준 중 10+ 개가 점자 관련

```python
# IPA 기반 점자 변환 엔진
# pip install phonemizer==3.2 epitran==1.25 unicodedata

import unicodedata
import re
from typing import List, Dict, Optional, Literal
from dataclasses import dataclass


# ──────────────────────────────────────────────
# 1. IPA → 점자 매핑 (공통 규칙)
# ──────────────────────────────────────────────
# IPA 기호의 점자 대응 (Unified English Braille 확장 기반)
IPA_TO_BRAILLE: Dict[str, str] = {
    # 모음
    "a": "⠁", "ɑ": "⠁", "ə": "⠢", "e": "⠑", "ɛ": "⠢",
    "i": "⠊", "ɪ": "⠊", "o": "⠕", "ɔ": "⠕", "u": "⠥", "ʊ": "⠥",
    "y": "⠽", "ø": "⠪", "œ": "⠪", "ɯ": "⠯",

    # 자음 (영어 기준)
    "p": "⠏", "b": "⠃", "t": "⠞", "d": "⠙",
    "k": "⠅", "g": "⠛", "f": "⠋", "v": "⠧",
    "s": "⠎", "z": "⠵", "ʃ": "⠩", "ʒ": "⠮",
    "m": "⠍", "n": "⠝", "ŋ": "⠬", "l": "⠇",
    "ɹ": "⠗", "r": "⠗", "h": "⠓",
    "j": "⠚", "w": "⠺",
    "θ": "⠹", "ð": "⠺",      # th sounds
    "tʃ": "⠡", "dʒ": "⠚",    # ch, j
    "ɸ": "⠋", "β": "⠧",      # 스페인어 등

    # 성조 마크 (중국어·태국어·베트남어)
    "˥": "⠜", "˦": "⠜⠁", "˧": "⠜⠃", "˨": "⠜⠉", "˩": "⠜⠙",
    "́": "⠄", "̀": "⠄⠁", "̂": "⠄⠃",    # 강세 마크

    # 특수
    " ": " ",
    ".": "⠲", ",": "⠂", "?": "⠦", "!": "⠖",
    "-": "⠤", "'": "⠄",
}

# 언어별 IPA 변환기 (phonemizer backend)
LANG_TO_PHONEMIZER = {
    "ko":  "espeak",       # eSpeak-ng 한국어
    "en":  "espeak",
    "ja":  "espeak",
    "zh":  "espeak",
    "ar":  "espeak",
    "th":  "espeak",
    "vi":  "espeak",
    # 특수 필요 언어는 Epitran
}


# ──────────────────────────────────────────────
# 2. 통합 점자 변환기
# ──────────────────────────────────────────────
@dataclass
class BrailleResult:
    source_text:     str
    source_lang:     str
    ipa:             str
    braille:         str
    grade:           Literal["grade1", "grade2"]
    cell_count:      int


class WIABrailleConverter:
    """
    254언어 범용 점자 변환기.
    주 경로: text → IPA → 점자
    특수: 한국어·영어는 최적화 모듈 (로컬 규칙 우선)
    """
    UNKNOWN = "⠿"         # 매핑 실패 시 (6점 모두 채움)

    def __init__(self):
        try:
            from phonemizer import phonemize as _phon
            self._phon = _phon
        except ImportError:
            self._phon = None
            print("⚠️  phonemizer 미설치 — IPA 경로 비활성. 한국어만 작동.")

    # ────────── 경로 1: 텍스트 → IPA ──────────
    def text_to_ipa(self, text: str, lang: str = "en") -> str:
        if self._phon is None:
            return text  # fallback: 원문 그대로
        backend = LANG_TO_PHONEMIZER.get(lang, "espeak")
        try:
            ipa = self._phon(
                text, language=lang, backend=backend,
                with_stress=True, preserve_punctuation=True,
            )
            return ipa.strip()
        except Exception as e:
            print(f"⚠️  phonemize 실패 ({lang}): {e}")
            return text

    # ────────── 경로 2: IPA → 점자 ──────────
    def ipa_to_braille(self, ipa: str) -> str:
        out = []
        i = 0
        # 2글자 우선 매칭 (tʃ, dʒ 등)
        while i < len(ipa):
            ch = ipa[i]
            next_ch = ipa[i + 1] if i + 1 < len(ipa) else ""
            bigram = ch + next_ch
            if bigram in IPA_TO_BRAILLE:
                out.append(IPA_TO_BRAILLE[bigram])
                i += 2
                continue
            # 유니코드 정규화 (NFD로 분리된 악센트)
            normalized = unicodedata.normalize("NFD", ch)
            if normalized != ch and normalized[0] in IPA_TO_BRAILLE:
                out.append(IPA_TO_BRAILLE[normalized[0]])
                i += 1
                continue
            # 단일 매핑
            if ch in IPA_TO_BRAILLE:
                out.append(IPA_TO_BRAILLE[ch])
            elif ch.isspace():
                out.append(" ")
            else:
                out.append(self.UNKNOWN)
            i += 1
        return "".join(out)

    # ────────── 통합 ──────────
    def convert(
        self,
        text:  str,
        lang:  str = "en",
        grade: Literal["grade1", "grade2"] = "grade1",
        use_language_module: bool = True,
    ) -> BrailleResult:
        # 언어별 최적화 모듈 우선
        if use_language_module and lang == "ko":
            from ks_x_6201 import KoreanBrailleConverter  # 챕터 10
            conv = KoreanBrailleConverter()
            braille = conv.convert(text, grade=grade)
            ipa = self.text_to_ipa(text, lang="ko")
            return BrailleResult(text, lang, ipa, braille, grade,
                                  cell_count=len(braille.replace(" ", "")))

        if use_language_module and lang == "en":
            # UEB (Unified English Braille) 간이 Grade 2
            braille = self._ueb_grade2(text) if grade == "grade2" else self._ueb_grade1(text)
            ipa = self.text_to_ipa(text, lang="en")
            return BrailleResult(text, lang, ipa, braille, grade,
                                  cell_count=len(braille.replace(" ", "")))

        # 일반 경로: IPA 기반
        ipa = self.text_to_ipa(text, lang=lang)
        braille = self.ipa_to_braille(ipa)
        return BrailleResult(text, lang, ipa, braille, grade,
                              cell_count=len(braille.replace(" ", "")))

    # ────────── UEB 간이 ──────────
    _UEB_G1 = {
        "a": "⠁", "b": "⠃", "c": "⠉", "d": "⠙", "e": "⠑",
        "f": "⠋", "g": "⠛", "h": "⠓", "i": "⠊", "j": "⠚",
        "k": "⠅", "l": "⠇", "m": "⠍", "n": "⠝", "o": "⠕",
        "p": "⠏", "q": "⠟", "r": "⠗", "s": "⠎", "t": "⠞",
        "u": "⠥", "v": "⠧", "w": "⠺", "x": "⠭", "y": "⠽", "z": "⠵",
        " ": " ", ".": "⠲", ",": "⠂", "?": "⠦", "!": "⠖",
    }
    _UEB_G2 = {
        "the": "⠮", "and": "⠯", "of": "⠷", "for": "⠿", "with": "⠾",
        "ch": "⠡", "sh": "⠩", "th": "⠹", "st": "⠌", "er": "⠻",
    }

    def _ueb_grade1(self, text: str) -> str:
        out = []
        for ch in text.lower():
            out.append(self._UEB_G1.get(ch, self.UNKNOWN))
        return "".join(out)

    def _ueb_grade2(self, text: str) -> str:
        """간이 Grade 2 — 축약만 일부 (실무는 liblouis)"""
        lower = text.lower()
        # 단어 축약 먼저
        for word, b in sorted(self._UEB_G2.items(), key=lambda x: -len(x[0])):
            if len(word) > 1 and word in lower:
                lower = re.sub(rf"\b{word}\b", b, lower)
        # 남은 글자를 Grade 1로
        return self._ueb_grade1(lower)


# ──────────────────────────────────────────────
# 3. liblouis 통합 (프로덕션 권장)
# ──────────────────────────────────────────────
class LibLouisConverter:
    """
    liblouis 공식 점자 엔진 래퍼 (프로덕션 환경).
    254언어 전부 공식 테이블 제공.
    pip install louis + liblouis 시스템 설치.
    """
    LANG_TABLE = {
        "ko":  "ko-g2.ctb",         # 한국어 Grade 2
        "en":  "en-ueb-g2.ctb",     # UEB Grade 2
        "ja":  "ja.ctb",
        "zh":  "zh-tw.ctb",
        "ar":  "ar.ctb",
        "es":  "es.ctb",
        "fr":  "fr-bfu-g2.ctb",
        "de":  "de-g2.ctb",
        # ... 254개
    }

    def __init__(self):
        try:
            import louis
            self.louis = louis
        except ImportError:
            self.louis = None
            print("⚠️  louis(liblouis) 미설치 — 간이 엔진 사용")

    def convert(self, text: str, lang: str = "en") -> str:
        if self.louis is None:
            return WIABrailleConverter().convert(text, lang=lang).braille

        table = self.LANG_TABLE.get(lang, "en-ueb-g2.ctb")
        return self.louis.translateString([table], text)


# ──────────────────────────────────────────────
# 4. 배치 변환 (복원 고문서용)
# ──────────────────────────────────────────────
def convert_document(
    text_ko:         str,        # 현대어 번역
    text_original:   str,        # 한자 원문 (세로쓰기)
    output_path:     str,
) -> dict:
    """복원 고문서의 한국어 번역 + 한자 원문 점자 변환"""
    converter = WIABrailleConverter()

    # 현대어 (KS X 6201)
    result_ko = converter.convert(text_ko, lang="ko", grade="grade2")

    # 한문 (중국어 번체 테이블)
    # 한자는 직접 점자 매핑 어려워 병음→IPA→점자 경로
    try:
        from pypinyin import lazy_pinyin
        pinyin = " ".join(lazy_pinyin(text_original))
        result_han = converter.convert(pinyin, lang="zh", grade="grade1")
    except ImportError:
        result_han = None

    output = {
        "modern_korean": {
            "source":  text_ko,
            "ipa":     result_ko.ipa,
            "braille": result_ko.braille,
            "cells":   result_ko.cell_count,
        },
        "classical_chinese": {
            "source":  text_original,
            "braille": result_han.braille if result_han else "(Pypinyin 필요)",
        } if result_han else None,
    }

    import json
    with open(output_path, "w", encoding="utf-8") as f:
        json.dump(output, f, ensure_ascii=False, indent=2)

    print(f">> 점자 변환: {output_path}")
    print(f"   한국어: {result_ko.cell_count} 셀")
    return output


if __name__ == "__main__":
    converter = WIABrailleConverter()

    # 기본 테스트
    tests = [
        ("안녕하세요",       "ko"),
        ("Hello world",      "en"),
        ("こんにちは",       "ja"),
        ("مرحبا",           "ar"),
        ("สวัสดี",           "th"),
    ]
    for text, lang in tests:
        r = converter.convert(text, lang=lang, grade="grade1")
        print(f"[{lang}] {text}")
        print(f"  IPA:     {r.ipa}")
        print(f"  Braille: {r.braille}")
        print(f"  Cells:   {r.cell_count}\n")

    # 고문서 변환
    convert_document(
        text_ko="엎드려 아뢰옵건대, 신들이 삼가 국가의 지극히 중대한 일을 진술하고자 합니다.",
        text_original="伏以臣等謹陳國家之至重大事",
        output_path="./output/sangso_01/braille.json",
    )
```

## 함정 카드

1. **"간이 IPA 매핑으로 모든 언어 커버 주장"** → `IPA_TO_BRAILLE` 테이블은 학습·프로토타입용. 실제 서비스는 **liblouis** (공식 254 테이블) 사용 필수. 간이 엔진은 아랍어 RTL·태국어 성조·중국어 성조 완전 커버 불가.

2. **"Grade 2 축약을 직접 구현"** → UEB Grade 2만 해도 규칙 180+. 한국어 Grade 2도 수십. 직접 구현은 버그 온상. **liblouis 테이블**이 정답. 유지보수 비용 0.

3. **"점자 출력 후 길이 검증 안 함"** → 점자 디스플레이(Refreshable Braille)는 **20·40·80 셀** 제약. 한 줄 초과 시 자동 줄바꿈 로직 필요. 실용: `<br class="braille-break">` 마크업 + CSS로 점자 단말기용 스타일.

4. **"한자를 병음 거치지 않고 직접 점자"** → 한자 자체는 표음이 아니라 점자 1:1 매핑 없음. 반드시 **병음(pypinyin) → IPA → 점자** 경로. 또는 한문 번역문을 점자로 (고문서 사용자 편의).

5. **"eSpeak-ng 언어 테이블 미설치 시 조용히 원문 반환"** → 시스템에 eSpeak-ng 없으면 phonemize 실패 → fallback으로 원문 그대로 점자 변환 → **완전 틀린 출력**. 반드시 예외 로그 + CI에서 `espeak-ng --voices` 검증 후 배포.

## 연습문제

**문제 1.** "café"(프랑스어)를 점자 변환한다. UEB Grade 1로 갈 때 `é`를 어떻게 처리하는가?

> **해설:** ① UEB는 `é`를 **억양 기호 (⠈) + e (⠑)** 로 표기 = `c-a-f-⠈⠑`. ② 또는 프랑스어 점자 테이블(fr-bfu-g1)로 변환 시 직접 매핑. ③ IPA 경로는 `café` → `[kafe]` → 점자. 프랑스어 단어는 프랑스어 점자 테이블 쓰는 게 정확. UEB는 영어 본문 중 외국어 단어에 쓰는 **통일 표기** 용도. 구분: "이 책이 주로 어떤 언어인가"로 테이블 선택.

**문제 2.** 고문서 판독문에 한자 80% + 한글 20%가 섞여 있다. 점자 변환 시 권장 전략은?

> **해설:** ① **섹션 분리**: 원문 한자 섹션 → 병음 변환 후 중국어 점자, 한글 섹션 → 한국어 점자 ② **동시 제공**: 한자 원문은 참고용으로만 병음 점자 포함, **주 콘텐츠는 한국어 번역**의 점자. 실제 시각장애인 사용자는 **한문 병음 점자 읽기 어려워함** — 현대어 번역 점자가 우선. ③ EPUB에서 `<section lang="ko-Hant">`, `<section lang="ko">` 마크업으로 점자 단말기가 언어 인식 가능하게 함.

**문제 3.** 점자 변환 결과를 음성 합성(TTS)과 **동시 제공**하는 이유는?

> **해설:** 시각장애인 중 **점자 활용자 10% 미만** (한국). 대부분 음성만 사용. 하지만 ① 학습 중인 학생 ② 소음 환경 ③ 중복 장애(청각+시각)에겐 점자가 생명선. **점자+음성 동시 제공**이 접근성 최대화. 구현: SMIL로 오디오 싱크 (챕터 05) + EPUB `schema:accessibilityFeature: synchronizedAudioText` 선언. 사용자는 자기 환경 맞춰 선택. 이것이 "정당한 편의 제공"의 핵심.
