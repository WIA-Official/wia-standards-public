# 챕터 07 · axe-core + Pa11y CI 자동 감사 · 100점 게이트

## 학습 목표
- GitHub Actions 또는 GitLab CI에 axe-core·Pa11y 이중 엔진 감사를 통합해 PR마다 자동 검증할 수 있다
- 감사 점수가 임계(95점) 미만이면 머지 차단하는 Status Check를 구축할 수 있다
- 감사 결과를 Grafana·Datadog 대시보드로 시각화해 접근성 점수 추세를 추적할 수 있다

---

## 이론

### 접근성은 "배포 후 검증"이 아니다

수동 감사는 배포 후 문제 발견 → 사용자가 불편 → 리포트 → 수정 → 재배포 사이클. **접근성 회귀**가 배포마다 발생. 한 PR이 CSS 규칙 하나 바꿔 focus-visible 깨지는 경우 흔하다.

해결책: **감사를 CI 파이프라인에 넣고, 임계 미달이면 머지 자체를 차단**. 형님의 WIA Book 프로젝트가 `a11y.wiabook.com`에 이미 이 구조를 갖추고 있다 ([WIA Book 핸드오프 문서](https://github.com/WIA-Official/wia-book) 참고).

### 감사 실행 위치 3가지

| 위치 | 속도 | 실제성 | 용도 |
|---|---|---|---|
| **로컬 pre-commit** | 즉시 | 정적 HTML | 개발자 조기 피드백 |
| **PR CI (ephemeral preview)** | 3~5분 | 실제 빌드 | 머지 차단 게이트 |
| **프로덕션 주기 감사** | 10분 | 실 URL | 회귀 감지 |

셋 다 갖추는 게 이상적. 최소 2번째(PR CI)는 필수.

### 100점의 현실

"WCAG 2.2 AA 100/100"은 **완전 자동 검증 기준**이다. 자동 감사는 전체 WCAG 기준의 약 30~40%만 잡는다 (나머지는 문맥·의미 해석 필요). 즉 100점 = "자동 잡히는 건 다 잡았다"이지 "완벽 접근성"은 아니다.

실무 임계:
- **개발 중**: 90점 + 경고 허용
- **PR 머지**: 95점 + 에러 0
- **프로덕션**: 100점 (자동 기준) + 월 1회 수동 감사 (SR·휠체어 시뮬레이션)

### 캐시 전략

매 PR마다 Playwright + axe-core + Pa11y + Puppeteer 설치하면 5분 → 30초로 단축 가능한 포인트:
- GitHub Actions `actions/cache`로 `node_modules`, `~/.cache/puppeteer`, Playwright 브라우저 캐시
- Docker 이미지로 프리빌드 (accessibility-auditor 이미지 분리)
- axe-core 룰셋을 shard 하여 병렬 실행

## 실전 코드

### 1. GitHub Actions 워크플로

```yaml
# .github/workflows/accessibility-audit.yml
name: Accessibility Audit

on:
  pull_request:
    branches: [main]
  push:
    branches: [main]
  schedule:
    - cron: "0 2 * * *"   # 매일 02:00 UTC 프로덕션 재감사

env:
  THRESHOLD_SCORE: 95
  THRESHOLD_ERRORS: 0

jobs:
  audit:
    runs-on: ubuntu-latest
    timeout-minutes: 15

    strategy:
      fail-fast: false
      matrix:
        page:
          - "index.html"
          - "docs/sangso_01.html"
          - "docs/jijuk_02.html"

    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Setup Node
        uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: "npm"

      - name: Cache Playwright
        uses: actions/cache@v4
        with:
          path: ~/.cache/ms-playwright
          key: ${{ runner.os }}-playwright-${{ hashFiles('**/package-lock.json') }}

      - name: Install dependencies
        run: |
          npm ci
          npx playwright install chromium --with-deps
          npm install -g @axe-core/cli pa11y @daisy/ace

      - name: Build site
        run: npm run build

      - name: Start preview server
        run: |
          npx serve -s dist -p 3000 &
          sleep 3

      - name: Run audits — ${{ matrix.page }}
        id: audit
        run: |
          python scripts/run_audit.py \
            --url "http://localhost:3000/${{ matrix.page }}" \
            --output-json "./audit-reports/${{ matrix.page }}.json" \
            --min-score $THRESHOLD_SCORE \
            --max-errors $THRESHOLD_ERRORS

      - name: Upload report
        uses: actions/upload-artifact@v4
        if: always()
        with:
          name: audit-${{ matrix.page }}
          path: ./audit-reports/

      - name: Comment PR
        if: github.event_name == 'pull_request'
        uses: actions/github-script@v7
        with:
          script: |
            const fs = require('fs');
            const report = JSON.parse(
              fs.readFileSync('./audit-reports/${{ matrix.page }}.json')
            );
            const grade = report.score >= 95 ? '✅' : '❌';
            const body = `## ${grade} Accessibility Audit — \`${{ matrix.page }}\`
            
            | Metric | Value |
            |---|---|
            | **Score** | ${report.score} / 100 |
            | **Grade** | ${report.grade} |
            | axe-core violations | ${report.axe.violations} |
            | Pa11y errors | ${report.pa11y.errors} |
            | Pa11y warnings | ${report.pa11y.warnings} |
            
            ${report.failures ? '### 🔥 Failures\\n' + report.failures.map(f => `- ${f}`).join('\\n') : ''}
            `;
            github.rest.issues.createComment({
              issue_number: context.issue.number,
              owner: context.repo.owner, repo: context.repo.repo,
              body,
            });
```

### 2. 감사 실행 스크립트

```python
# scripts/run_audit.py — CI 진입점
import argparse
import json
import sys
from pathlib import Path
import subprocess


def run_pa11y(url: str, standard: str = "WCAG2AA") -> dict:
    result = subprocess.run(
        ["pa11y", "--standard", standard, "--reporter", "json",
         "--timeout", "30000", url],
        capture_output=True, text=True, timeout=60,
    )
    try:
        issues = json.loads(result.stdout)
        errors   = [i for i in issues if i["type"] == "error"]
        warnings = [i for i in issues if i["type"] == "warning"]
        return {
            "tool":    "pa11y",
            "errors":  len(errors),
            "warnings": len(warnings),
            "issues":  issues,
        }
    except json.JSONDecodeError:
        return {"tool": "pa11y", "errors": -1, "fatal": result.stderr}


def run_axe(url: str) -> dict:
    """axe-core via Playwright"""
    from playwright.sync_api import sync_playwright
    with sync_playwright() as p:
        browser = p.chromium.launch(headless=True)
        page = browser.new_page()
        page.goto(url, wait_until="networkidle")
        page.add_script_tag(
            url="https://cdnjs.cloudflare.com/ajax/libs/axe-core/4.10.0/axe.min.js",
        )
        result = page.evaluate("async () => await axe.run()")
        browser.close()

    return {
        "tool":       "axe-core",
        "violations": len(result["violations"]),
        "passes":     len(result["passes"]),
        "incomplete": len(result["incomplete"]),
        "violation_details": [
            {
                "id":        v["id"],
                "impact":    v["impact"],
                "help":      v["help"],
                "nodes":     len(v["nodes"]),
            }
            for v in result["violations"]
        ],
    }


def compute_score(axe: dict, pa11y: dict) -> dict:
    """통합 점수"""
    total_issues = axe.get("violations", 0) + pa11y.get("errors", 0)
    total_passes = axe.get("passes", 0)
    denom = total_passes + total_issues
    score = int(100 * total_passes / denom) if denom else 0

    failures = []
    if pa11y.get("errors", 0) > 0:
        failures.append(f"Pa11y errors: {pa11y['errors']}")
    for v in axe.get("violation_details", []):
        if v["impact"] in ("critical", "serious"):
            failures.append(f"axe {v['id']} ({v['impact']}): {v['help']}")

    return {
        "score":    score,
        "grade":    "AA" if score >= 95 else "A" if score >= 85 else "FAIL",
        "axe":      axe,
        "pa11y":    pa11y,
        "failures": failures,
    }


def main():
    parser = argparse.ArgumentParser()
    parser.add_argument("--url", required=True)
    parser.add_argument("--output-json", required=True)
    parser.add_argument("--min-score",  type=int, default=95)
    parser.add_argument("--max-errors", type=int, default=0)
    args = parser.parse_args()

    print(f">> 감사 시작: {args.url}")
    pa11y = run_pa11y(args.url)
    axe   = run_axe(args.url)
    report = compute_score(axe, pa11y)
    report["url"] = args.url

    Path(args.output_json).parent.mkdir(parents=True, exist_ok=True)
    with open(args.output_json, "w", encoding="utf-8") as f:
        json.dump(report, f, ensure_ascii=False, indent=2)

    print(f"점수: {report['score']}/100 ({report['grade']})")
    if report["failures"]:
        print("실패:")
        for f in report["failures"]:
            print(f"  - {f}")

    # 게이트 검사
    if report["score"] < args.min_score:
        print(f"❌ 점수 {report['score']} < 임계 {args.min_score}")
        sys.exit(1)
    if pa11y.get("errors", 0) > args.max_errors:
        print(f"❌ Pa11y 에러 {pa11y['errors']} > 허용 {args.max_errors}")
        sys.exit(1)
    print("✅ 통과")
    sys.exit(0)


if __name__ == "__main__":
    main()
```

### 3. Grafana 대시보드 연동

```python
# scripts/report_to_prometheus.py — 감사 결과를 Pushgateway로
import requests
import json
from pathlib import Path


def push_metrics(audit_json_path: str, pushgateway_url: str, job: str = "a11y_audit"):
    with open(audit_json_path, encoding="utf-8") as f:
        report = json.load(f)

    # Prometheus text format
    metrics = f"""
# HELP a11y_score WCAG 2.2 compliance score (0-100)
# TYPE a11y_score gauge
a11y_score{{url="{report['url']}"}} {report['score']}

# HELP a11y_axe_violations axe-core 위반 개수
# TYPE a11y_axe_violations counter
a11y_axe_violations{{url="{report['url']}"}} {report['axe']['violations']}

# HELP a11y_pa11y_errors Pa11y 에러 개수
# TYPE a11y_pa11y_errors counter
a11y_pa11y_errors{{url="{report['url']}"}} {report['pa11y']['errors']}
"""

    response = requests.post(
        f"{pushgateway_url}/metrics/job/{job}",
        data=metrics.encode("utf-8"),
        headers={"Content-Type": "text/plain"},
        timeout=30,
    )
    response.raise_for_status()
    print(f">> Pushgateway 전송 완료: {response.status_code}")


if __name__ == "__main__":
    import sys
    push_metrics(
        audit_json_path=sys.argv[1],
        pushgateway_url="http://prometheus-pushgateway:9091",
    )
```

## 함정 카드

1. **"CI에서 `sleep`으로 dev 서버 기동 대기"** → 고정 sleep 3초는 느린 머신에서 실패, 빠른 머신에선 낭비. `wait-on http://localhost:3000` 또는 health check endpoint 도입 권장.

2. **"프리뷰 환경의 URL을 CI에서 못 잡음"** → Netlify·Vercel 프리뷰 배포 URL은 PR마다 랜덤. GitHub Action의 `needs.preview.outputs.url`로 체이닝하거나 `wait-for-preview-url` 액션 사용.

3. **"audit-reports 아티팩트를 지우지 않음"** → 매 PR마다 저장해 GB 누적. `actions/upload-artifact`의 `retention-days: 7`로 자동 삭제. 대시보드 메트릭은 별도 DB.

4. **"axe-core 룰 전체를 매번 실행"** → ~100 룰. 대규모 페이지면 30초+. 변경 없는 룰은 캐시 스킵하는 `axe-core-reporter-incremental` 플러그인 고려. 또는 중요 룰만 (`--rules wcag2a,wcag2aa,best-practice`) 제한.

5. **"감사 결과 코멘트가 PR에 누적됨"** → 같은 PR에 푸시할 때마다 댓글이 쌓여 가독성 망함. `github-script`에 **기존 댓글 탐색 후 업데이트** 로직 추가 (`octokit.issues.updateComment`).

## 연습문제

**문제 1.** PR CI에서 axe-core는 0 violations인데 Pa11y는 3 errors. 어느 쪽을 신뢰해야 하나?

> **해설:** **둘 다 신뢰**. 서로 다른 규칙 세트를 본다. Pa11y의 3 errors가 axe-core 룰셋에 없는 경우 (예: HTML validator 수준 이슈, 특정 ARIA attribute 조합). 해결: ① Pa11y 에러 3건을 상세 확인 — 실제 WCAG 위반인지, false positive인지 ② WCAG 번호로 해결 가능성 판단 ③ axe-core에서 같은 이슈를 잡는 룰 활성화 여부 체크. **결정**: Pa11y errors 0으로 수정 후 다시 돌려 axe도 0 확인. 이중 엔진의 목적이 각자 놓치는 것을 잡는 것이므로 합집합 통과.

**문제 2.** 감사 점수가 95점인 PR을 머지했는데 프로덕션에서 100점이 나온다. 왜?

> **해설:** ① **CDN 캐시** — CI는 빌드 직후, 프로덕션은 CDN 경유로 일부 외부 자원(폰트·아이콘) 변환 → 감사 룰 영향 ② **환경 변수 차이** — 개발 빌드는 sourcemap 포함, prod은 제거, 감사 룰이 개발 전용 디버그 표시 잡아냄 ③ **렌더링 방식 차이** — CSR에서 감사와 SSR 후 감사 결과 다름 (초기 HTML과 JS 후 차이) ④ **타임아웃** — Pa11y 30초 타임아웃으로 일부 리소스 로드 전 스크립트 실행. 해결: **CI 환경을 프로덕션과 최대한 동일화**(Docker 이미지 공유) + 프로덕션 감사도 정기 실행.

**문제 3.** 3개 페이지 중 1개만 실패하면 PR 전체 차단되는데, 때로는 "다른 팀 담당 페이지라 모른다"는 경우가 있다. 어떻게 세분화하는가?

> **해설:** ① **CODEOWNERS 활용** — `.github/CODEOWNERS`에 페이지별 담당 팀 명시. 감사 실패 시 해당 담당에 자동 알림. ② **필터 기반 머지 차단** — Branch protection에 `required_status_checks`를 PR 변경 파일과 페이지 경로를 매핑해 **변경된 페이지만 블록**. `path-filter` GitHub Action로 구현 가능. ③ **경고 vs 차단 분리** — "내 PR이 직접 건드린 페이지"는 차단, 기존 실패한 페이지는 **경고 댓글만** 남기고 통과 (`allow-failing-existing: true` 옵션 패턴). 이 방식은 전체 접근성 하락 없이 작업 플로우 유지.
