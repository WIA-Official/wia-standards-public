# 챕터 12 · Whisper STT 피드백 루프 — 합성→재인식 품질 검증

## 학습 목표
- 합성된 TTS 음성을 Whisper로 역전사(reverse transcription)해 원문과 비교함으로써 TTS 품질을 정량화할 수 있다
- WER(Word Error Rate)·CER(Character Error Rate) 지표를 계산하고 한국어·다국어 품질 임계값을 설정할 수 있다
- WER 초과 시 SSML 파라미터를 자동 튜닝하는 피드백 루프를 구축할 수 있다

---

## 이론

### 역전사 검증의 필요성

TTS가 "伏以臣等"을 "복이씬덩"으로 이상하게 발음해도 오디오 파일만 보면 알 수 없다. 청취자가 실제 들어야 품질 판단 가능. 이걸 자동화하는 방법:

```
원문 텍스트 ──→ TTS ──→ 음성 파일 ──→ STT(Whisper) ──→ 재전사 텍스트
                                                            │
                                                            ▼
                                                     원문과 비교
                                                    (WER/CER 계산)
```

WER 0%는 "완벽 발음", 5% 미만은 "실사용 OK", 10% 초과는 "다시 해야 함".

### WER vs CER

**WER (Word Error Rate)**: 단어 단위 오류율
```
WER = (삽입 + 삭제 + 치환) / 전체 단어 수
```

**CER (Character Error Rate)**: 글자 단위 오류율. 한국어처럼 **단어 경계 애매한 언어**에 적합.

```
한국어 원문: "안녕하세요 반갑습니다"
STT 결과:   "안녕 하세요 반갑습니다"
WER: 2/2 = 100% (단어 분리 다름 → 치환)
CER: 0/16 = 0%  (글자는 동일)
```

**권장**: 한국어·일본어·중국어는 **CER 우선**, 영어·유럽어는 WER.

### Whisper 모델 선택

| 모델 | 크기 | VRAM | 한국어 CER (추정 범위) |
|---|---|---|---|
| `tiny` | 39M | 1GB | 12~18% |
| `base` | 74M | 1GB | 8~12% |
| `small` | 244M | 2GB | 5~8% |
| `medium` | 769M | 5GB | 3~6% |
| `large-v3` | 1550M | 10GB | 2~5% |

위 수치는 일반 한국어 기준이고, **고문서 도메인(희귀 한자음·옛말·고유명사)** 에서는 각 모델 CER이 1.5~2배 높게 나오는 걸 직접 겪었다. 공식 벤치마크 숫자만 믿고 SLO를 설정하면 배포 후 이탈이 커진다. **자체 검증 세트 100~500문장으로 실측 후 임계 설정**이 필수.

**역전사 검증 용도**는 TTS 품질 평가이므로 **STT 자체도 고품질** 필요. `large-v3` 권장. 다만 10GB VRAM 요구.

**절충**: 1차 `small`로 거르고, WER 5% 초과 건만 `large-v3`로 재확인. 비용 절반.

### WER 임계값 (도메인별)

| 도메인 | CER 합격 | WER 합격 |
|---|---|---|
| 일반 대화 | < 5% | < 7% |
| 뉴스 낭독 | < 3% | < 5% |
| 고문서 낭독 | < 8% (희귀 한자음) | < 10% |
| 도서 낭독 | < 4% | < 6% |

고문서는 희귀 한자음 때문에 Whisper가 "못 맞히는" 경우 있어 **허용 기준 완화**.

### SSML 자동 튜닝

WER 초과 시 무작정 재생성하지 않고 체계적으로 튜닝:

1. **속도 감소** (rate 0.9 → 0.8) → 자음·모음 분리 명확
2. **피치 조정** (pitch 0 → -2st) → Whisper가 잘 듣는 주파수 영역
3. **break 삽입** — 긴 문장을 쪼개 Whisper 세그먼트 인식 돕기
4. **Piper voice 변경** — 특정 화자로 Whisper가 더 정확하면 그 voice 사용

순차 적용하며 WER 목표 달성.

```python
# TTS → STT 역전사 검증 + SSML 자동 튜닝
# pip install openai-whisper==20240930 jiwer==3.0 librosa

import re
import json
import unicodedata
from pathlib import Path
from dataclasses import dataclass, field
from typing import List, Optional, Dict
import whisper
import jiwer


DEVICE = "cuda"


# ──────────────────────────────────────────────
# 1. 텍스트 정규화 (공정 비교)
# ──────────────────────────────────────────────
def normalize_text(text: str, lang: str = "ko") -> str:
    """
    WER/CER 계산 전 텍스트 정규화.
    언어별 불필요한 차이(구두점·공백·유니코드 이형) 제거.
    """
    # 유니코드 NFC 정규화
    text = unicodedata.normalize("NFC", text)

    # 구두점 제거
    text = re.sub(r"[.,!?;:·\"'()【】『』「」『』“”‘’]", " ", text)

    # 연속 공백 하나로
    text = re.sub(r"\s+", " ", text).strip()

    # 한국어는 띄어쓰기 무시 (CER만 계산)
    if lang in ("ko", "ja", "zh"):
        text = text.replace(" ", "")

    return text.lower() if text.isascii() else text


# ──────────────────────────────────────────────
# 2. Whisper STT 래퍼
# ──────────────────────────────────────────────
class WhisperSTT:
    def __init__(self, model_size: str = "medium", device: str = DEVICE):
        self.model = whisper.load_model(model_size, device=device)

    def transcribe(
        self,
        audio_path: str,
        lang: str = "ko",
        temperature: float = 0.0,
    ) -> dict:
        """
        transcribe 옵션:
        - temperature=0: 결정론적 (동일 입력엔 동일 출력)
        - task="transcribe": 원어 그대로
        """
        result = self.model.transcribe(
            audio_path,
            language=lang,
            temperature=temperature,
            task="transcribe",
            condition_on_previous_text=False,    # 독립 세그먼트
            initial_prompt=None,
        )
        return {
            "text":     result["text"].strip(),
            "segments": [
                {"text": s["text"].strip(),
                 "start": float(s["start"]),
                 "end":   float(s["end"]),
                 "avg_logprob": float(s.get("avg_logprob", 0))}
                for s in result["segments"]
            ],
            "language": result.get("language"),
        }


# ──────────────────────────────────────────────
# 3. WER/CER 계산
# ──────────────────────────────────────────────
@dataclass
class QualityMetrics:
    wer:          float
    cer:          float
    reference:    str
    hypothesis:   str
    substitutions: int
    insertions:    int
    deletions:     int


def compute_metrics(
    reference: str,
    hypothesis: str,
    lang: str = "ko",
) -> QualityMetrics:
    """
    jiwer로 WER/CER 계산.
    한국어는 CER 우선.
    """
    ref_n = normalize_text(reference, lang)
    hyp_n = normalize_text(hypothesis, lang)

    # WER (단어 기반)
    if lang in ("ko", "ja", "zh"):
        # 한국어는 글자 단위 WER = CER
        word_ref = list(ref_n)
        word_hyp = list(hyp_n)
    else:
        word_ref = ref_n.split()
        word_hyp = hyp_n.split()

    wer_result = jiwer.compute_measures(
        " ".join(word_ref), " ".join(word_hyp),
    )
    # CER — 항상 글자 단위
    cer = jiwer.cer(ref_n, hyp_n)

    return QualityMetrics(
        wer=wer_result["wer"],
        cer=cer,
        reference=reference,
        hypothesis=hypothesis,
        substitutions=wer_result["substitutions"],
        insertions=wer_result["insertions"],
        deletions=wer_result["deletions"],
    )


# ──────────────────────────────────────────────
# 4. TTS 품질 검증기 (루프 통합)
# ──────────────────────────────────────────────
@dataclass
class TTSValidationConfig:
    wer_threshold:   float = 0.10     # 10%
    cer_threshold:   float = 0.05     # 5% (한국어·일본어·중국어)
    max_retries:     int = 3
    stt_model_size:  str = "medium"
    fallback_model:  str = "large-v3"


class TTSQualityValidator:
    """
    TTS 생성 → Whisper STT → WER/CER → 임계 미달 시 SSML 튜닝 재생성
    """
    def __init__(self, tts_api, config: TTSValidationConfig = TTSValidationConfig()):
        self.tts = tts_api
        self.cfg = config
        self.stt = WhisperSTT(model_size=config.stt_model_size)
        self.fallback_stt = None

    def validate(
        self,
        text: str,
        lang: str = "ko",
        initial_params: Optional[Dict] = None,
    ) -> dict:
        """
        반복 튜닝 루프.
        """
        params = initial_params or {"speed": 1.0, "pitch": 0, "break_ms": 200}
        attempts = []

        for attempt in range(self.cfg.max_retries):
            # (1) TTS 생성
            audio = self.tts.synthesize(text, lang=lang, **params)

            # (2) STT 역전사
            stt = self.stt.transcribe(audio["path"], lang=lang)

            # (3) WER/CER
            metrics = compute_metrics(text, stt["text"], lang=lang)

            result = {
                "attempt":       attempt + 1,
                "params":        params.copy(),
                "audio_path":    audio["path"],
                "hypothesis":    stt["text"],
                "wer":           metrics.wer,
                "cer":           metrics.cer,
                "passed":        self._judge_pass(metrics, lang),
            }
            attempts.append(result)

            if result["passed"]:
                print(f">> 시도 {attempt+1}: WER={metrics.wer:.3f} CER={metrics.cer:.3f} ✅")
                return {"final": result, "history": attempts}

            print(f">> 시도 {attempt+1}: WER={metrics.wer:.3f} CER={metrics.cer:.3f} — 튜닝 필요")

            # (4) 튜닝
            params = self._tune(params, metrics, attempt)

        # 최대 시도 초과 시 Whisper large로 fallback 재검증
        final = attempts[-1]
        if self.fallback_stt is None:
            self.fallback_stt = WhisperSTT(model_size=self.cfg.fallback_model)
        stt2 = self.fallback_stt.transcribe(final["audio_path"], lang=lang)
        metrics2 = compute_metrics(text, stt2["text"], lang=lang)
        if self._judge_pass(metrics2, lang):
            print(f">> large-v3 재검증으로 통과: CER={metrics2.cer:.3f}")
            final["passed"] = True
            final["fallback_cer"] = metrics2.cer
            final["fallback_hypothesis"] = stt2["text"]

        return {"final": final, "history": attempts, "expert_review_required": not final["passed"]}

    def _judge_pass(self, m: QualityMetrics, lang: str) -> bool:
        if lang in ("ko", "ja", "zh"):
            return m.cer < self.cfg.cer_threshold
        return m.wer < self.cfg.wer_threshold

    def _tune(self, params: dict, metrics: QualityMetrics, attempt: int) -> dict:
        """
        WER/CER 기반 SSML 파라미터 튜닝.
        """
        new_params = params.copy()
        # Round 1: 속도 감소
        if attempt == 0:
            new_params["speed"] = max(0.7, params.get("speed", 1.0) - 0.1)
        # Round 2: 피치 조정 + break 증가
        elif attempt == 1:
            new_params["pitch"] = -2
            new_params["break_ms"] = 300
        # Round 3: 다른 voice 시도 (경량)
        elif attempt == 2:
            new_params["voice_override"] = "ko_KR-korean_single-medium"

        return new_params


# ──────────────────────────────────────────────
# 5. 배치 검증 (고문서 전체)
# ──────────────────────────────────────────────
def validate_document_tts(
    document_text:  str,
    output_json:    str,
    lang:           str = "ko",
) -> dict:
    """
    문서 전체를 문장 단위로 분할 후 각각 검증.
    """
    from tts_engine import WIAVoiceAPI   # 챕터 11
    tts = WIAVoiceAPI()
    validator = TTSQualityValidator(tts)

    sentences = re.split(r"(?<=[.!?])\s+", document_text.strip())
    results = []

    for i, sent in enumerate(sentences):
        if not sent.strip():
            continue
        print(f"\n[{i+1}/{len(sentences)}] {sent[:40]}...")
        result = validator.validate(sent, lang=lang)
        results.append({
            "index":         i,
            "text":          sent,
            "passed":        result["final"]["passed"],
            "final_cer":     result["final"]["cer"],
            "final_wer":     result["final"]["wer"],
            "attempts":      len(result["history"]),
            "audio_path":    result["final"]["audio_path"],
            "expert_review": result.get("expert_review_required", False),
        })

    # 집계
    total = len(results)
    passed = sum(1 for r in results if r["passed"])
    avg_cer = sum(r["final_cer"] for r in results) / total

    summary = {
        "total_sentences": total,
        "passed":          passed,
        "pass_rate":       passed / total,
        "avg_cer":         avg_cer,
        "expert_review":   sum(1 for r in results if r["expert_review"]),
        "sentences":       results,
    }

    with open(output_json, "w", encoding="utf-8") as f:
        json.dump(summary, f, ensure_ascii=False, indent=2)

    print(f"\n=== 전체 요약 ===")
    print(f"통과: {passed}/{total} ({passed/total*100:.1f}%)")
    print(f"평균 CER: {avg_cer:.3f}")
    print(f"전문가 검토 필요: {summary['expert_review']}건")
    return summary


if __name__ == "__main__":
    # 단일 문장 검증
    from tts_engine import WIAVoiceAPI
    tts = WIAVoiceAPI()
    validator = TTSQualityValidator(tts)

    text = "안녕하세요. WIA 글살림입니다."
    result = validator.validate(text, lang="ko")
    print(json.dumps(result, ensure_ascii=False, indent=2))

    # 문서 전체
    validate_document_tts(
        document_text="""엎드려 아뢰옵건대, 신들이 삼가 국가의 지극히 중대한 일을 진술하고자 합니다.
        경들의 뜻을 받들어 깊이 검토하겠노라.
        왕께서 친히 붓을 들어 비답하셨다.""",
        output_json="./output/tts_quality_report.json",
        lang="ko",
    )
```

## 함정 카드

1. **"Whisper가 인식 못 하면 TTS 탓"** → Whisper 자체도 오류 5~10%. 특히 희귀 한자음, 고어, 인명·지명에 약함. **Whisper WER = TTS 오류 + Whisper 오류** 합산. 고문서는 Whisper만으로 평가 불가 — `large-v3` + **인간 검수 10%** 샘플링 필수.

2. **"WER 계산 전 텍스트 정규화 생략"** → 원문 "안녕, 반가워"와 STT "안녕 반가워"는 쉼표 하나 차이로 WER 100%. 반드시 `normalize_text`로 구두점·유니코드 이형 제거 후 비교.

3. **"한국어에 WER만 계산"** → 한국어는 띄어쓰기가 불일치 흔함. WER은 단어 기반이라 띄어쓰기 차이만으로 40%+ 오류율. **CER**이 실제 품질 반영. WER은 참고용.

4. **"Whisper 온도를 0.7로 랜덤 샘플링"** → 같은 음성을 여러 번 돌리면 다른 결과. 재현성 없음. 반드시 `temperature=0`으로 결정론적 출력. WER 측정 가능.

5. **"튜닝 루프에서 화자 교체 너무 빨리"** → 화자 변경 시 음색이 바뀌어 **청자 경험 불일치**. 한 문서 내에서 화자는 일관 유지가 원칙. SSML 파라미터(속도·피치·break) 조정을 먼저, 화자 교체는 최후의 수단.

## 연습문제

**문제 1.** TTS → STT 역전사 결과 CER이 0%인데 실제 사람이 들으면 부자연스럽다. 왜?

> **해설:** **CER은 "글자 정확성"만 보고, 음운·운율은 안 본다.** Whisper가 "어떤 글자를 발음했는지"만 판단. 로봇 같은 기계적 발음이어도 글자만 맞으면 CER 0%. 해결: ① **MOS(Mean Opinion Score) 인간 평가** 추가 — 5점 척도 (1=매우 부자연, 5=사람 같음) ② **prosody metric** — F0 표준편차(음높이 변동), 말하기 속도 분포. Piper 출력이 단조로우면 F0 std 10Hz 미만, 자연스러운 낭독은 30Hz+. 자동 측정은 `librosa.pyin` 활용. CER만으로는 "정확하지만 부자연스러운" 문제 못 잡음.

**문제 2.** "伏以" 한 단어의 CER이 50%다. Whisper가 "복이"를 "봉이"로 인식했다. TTS 튜닝으로 해결되는가?

> **해설:** **TTS는 정상, Whisper 한계**. Whisper `base`·`medium`은 한국어 희귀 한자음 약함. 해결: ① **initial_prompt 활용** — `transcribe(audio, initial_prompt="이 음성은 조선시대 고문서 낭독이며 伏以, 臣等 같은 한자음이 나옵니다.")` → Whisper가 도메인 인식 ② **large-v3 모델** 사용 ③ **custom vocabulary** — 고문서 희귀 한자음 100~500개를 Whisper의 `vocabulary_hints`로 (API 옵션에 따라). TTS 튜닝은 **무의미** — 원인이 STT니까.

**문제 3.** 1000 문장 검증 중 50개가 실패해 fallback까지 가도 통과 못 했다. 운영상 어떻게 처리해야 하나?

> **해설:** ① **수동 검토 대기열**로 이관 — 50 문장을 한국어 장애인용 도서 낭독 전문가에게 발송. 비용 ₩3,000/문장 × 50 = ₩150,000 ② **패턴 분석** — 실패 문장의 공통점(고유명사 많음? 수식 표현? 외래어?)을 추출해 **SSML 사전 규칙 추가**. 예: 특정 한자 단어는 항상 괄호 안에 한글 음독 넣기 ③ **점진적 학습** — 실패 문장 + 전문가 수정본을 **Piper 파인튜닝 데이터**로 누적. 월 50 문장 × 12개월 = 600 문장의 전문가 검증 코퍼스가 쌓이면 Piper 고문서 특화 모델 학습 가능. **운영은 자동 100%를 목표하지 말고 "5% 수동 검수" 전제로 프로세스 설계**.
