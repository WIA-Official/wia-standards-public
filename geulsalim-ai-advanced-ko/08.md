# 챕터 08 · 글로벌 접근성 법·표준 체크리스트 자동화

## 학습 목표
- EU EAA(유럽 접근성법)·미국 ADA·한국 KS X 3253·일본 JIS X 8341·캐나다 AODA 등 글로벌 접근성 법·표준의 공통점과 차이점을 설명할 수 있다
- 법·표준별 요구사항을 WCAG 기준 번호로 매핑하는 JSON DB를 구축하고, 자동 감사 결과와 법규 준수 리포트를 연결할 수 있다
- 법규 위반 리스크를 식별하고 배포 전 법적 검토 필요 여부를 자동 판정할 수 있다

---

## 이론

### 접근성 법은 나라마다 다르지만 뿌리는 같다

| 국가 | 법·표준 | 기반 | 발효 | 과태료 |
|---|---|---|---|---|
| EU | EAA (Directive 2019/882) | WCAG 2.1 AA → 2.2 전환 중 | 2025-06-28 | 최대 €50,000 (국가마다 다름) |
| 미국 | ADA Title III / Section 508 | WCAG 2.1 AA | 오래됨 / 최근 판례 강화 | 소송 합의 $10,000~$500,000 |
| 한국 | KS X 3253 / 장애인차별금지법 | WCAG 2.1 (2.2 준비 중) | 2008 장차법 · KS 개정 중 | 3천만 원 이하 과태료 |
| 일본 | JIS X 8341-3:2016 | WCAG 2.0 AA | 2016 (2.2 갱신 예정) | 소송 리스크 |
| 캐나다 | AODA | WCAG 2.0 AA | 2021 (대기업 · 2025 전체) | CA$100,000/일 |
| 호주 | DDA + Web Accessibility National Transition Strategy | WCAG 2.0 AA | 1992 (강화 중) | 소송 |
| 영국 | Equality Act 2010 + PSBAR 2018 | WCAG 2.1 AA | 2018 (공공부문) | 소송 |

**공통 기반**: WCAG. 차이: 버전(2.0 vs 2.1 vs 2.2), 적용 범위(공공만 vs 민간 포함), 시행 시기.

**형님 글살림 프로젝트**처럼 글로벌 공개하는 서비스는 **가장 엄격한 기준(EU EAA + KS X 3253 + AODA)**을 상위 교집합으로 삼아야 안전.

### 법규 준수 vs 자동 감사의 간극

axe-core/Pa11y는 **기술 기준(WCAG SC 번호)** 만 검사한다. 하지만 법은 더 복잡:

- EU EAA: 기술 기준 + **사용자 테스트 의무** + **접근성 진술서 게시** 요구
- 한국 장차법: "정당한 편의 제공 거부" 판단 — 법원이 사례별 판단
- ADA: 판례법이라 추상적

즉 자동 감사 100점 ≠ 법규 100% 준수. 법규 리포트는 기술 점수 + **수동 확인 항목** + **증거 문서** 통합.

### 자동화 전략

1. **법규 → WCAG 매핑 DB** 구축 (JSON)
2. 자동 감사 결과 + 매핑으로 **법규별 준수표** 생성
3. 수동 확인 항목은 **사람이 체크할 양식** 생성
4. 최종 리포트는 기술 + 수동 + 증거 조합

# 2. 감사 결과 → 법규 준수 매핑
# ──────────────────────────────────────────────
@dataclass
class LegalComplianceReport:
    standard_id:            str
    standard_name:          str
    technical_score:        float       # 0~100 (자동 감사)
    technical_failed_criteria: List[str]
    manual_items_total:     int
    manual_items_verified:  int
    overall_compliance:     str         # "compliant" | "partial" | "non-compliant"
    risk_level:             str         # "low" | "medium" | "high"
    recommendations:        List[str]
    evidence_urls:          List[str]


class LegalComplianceChecker:
    """
    자동 감사 결과 + 수동 체크리스트 → 법규 준수 리포트
    """

    def __init__(self, standards_db: dict = None):
        self.db = standards_db or LEGAL_STANDARDS_DB

    def check(
        self,
        audit_report: dict,                   # 챕터 07의 감사 결과 JSON
        manual_verifications: Dict[str, bool],   # {"accessibility_statement_published": True, ...}
        standard_id: str,
    ) -> LegalComplianceReport:
        standard = self.db[standard_id]

        # (1) 기술 기준 매칭
        required_criteria = set(standard["technical_criteria"])
        failed_criteria = self._extract_failed_criteria(audit_report, required_criteria)
        tech_score = 100 * (1 - len(failed_criteria) / len(required_criteria))

        # (2) 수동 기준 확인
        manual_items = standard.get("manual_requirements", [])
        verified = sum(1 for item in manual_items if manual_verifications.get(item, False))

        # (3) 종합 판정
        overall, risk, recs = self._judge(
            tech_score, failed_criteria, manual_items, verified,
        )

        return LegalComplianceReport(
            standard_id=standard_id,
            standard_name=standard["name"],
            technical_score=round(tech_score, 1),
            technical_failed_criteria=sorted(failed_criteria),
            manual_items_total=len(manual_items),
            manual_items_verified=verified,
            overall_compliance=overall,
            risk_level=risk,
            recommendations=recs,
            evidence_urls=[],
        )

    def _extract_failed_criteria(
        self,
        audit_report: dict,
        required_criteria: set,
    ) -> List[str]:
        """axe-core/Pa11y 결과에서 WCAG 번호 추출 후 필수와 교집합"""
        failed = set()

        # axe-core violations
        for v in audit_report.get("axe", {}).get("violation_details", []):
            tags = v.get("tags", [])
            for tag in tags:
                # tag 예: "wcag2aa", "wcag143" → "1.4.3"
                if tag.startswith("wcag") and len(tag) >= 6:
                    num = tag[4:]
                    formatted = f"{num[0]}.{num[1]}.{num[2:]}"
                    if formatted in required_criteria:
                        failed.add(formatted)

        # Pa11y는 WCAG 번호를 code에 포함
        for issue in audit_report.get("pa11y", {}).get("issues", []):
            if issue.get("type") != "error":
                continue
            code = issue.get("code", "")
            # 예: "WCAG2AA.Principle1.Guideline1_4.1_4_3.G18.Fail"
            match = self._parse_pa11y_code(code)
            if match and match in required_criteria:
                failed.add(match)

        return list(failed)

    @staticmethod
    def _parse_pa11y_code(code: str) -> Optional[str]:
        import re
        match = re.search(r"(\d)_(\d)_(\d+)", code)
        if match:
            return f"{match.group(1)}.{match.group(2)}.{match.group(3)}"
        return None

    def _judge(
        self, tech_score: float, failed_criteria: List[str],
        manual_items: List[str], verified: int,
    ) -> tuple:
        manual_rate = verified / max(len(manual_items), 1)

        if tech_score >= 95 and manual_rate == 1.0:
            return ("compliant", "low", [
                "현 수준 유지",
                "분기별 재감사 권장",
            ])
        if tech_score >= 85 and manual_rate >= 0.7:
            return ("partial", "medium", [
                f"기술 실패 {len(failed_criteria)}건 해결: {', '.join(failed_criteria[:5])}",
                f"수동 항목 {len(manual_items) - verified}건 완료 필요",
                "법무 검토 권장",
            ])
        return ("non-compliant", "high", [
            "긴급 조치 필요",
            f"기술 점수 {tech_score:.1f}점 (임계 95 미달)",
            f"수동 확인 {verified}/{len(manual_items)} 미흡",
            "배포 전 법무 필수 검토 · 영향 받는 사용자 공지",
        ])


# ──────────────────────────────────────────────
# 3. 다국가 통합 리포트
# ──────────────────────────────────────────────
def generate_global_report(
    audit_report: dict,
    manual_verifications: Dict[str, bool],
    standards: List[str] = None,
    output_path: str = "./output/legal_compliance.html",
) -> Dict[str, LegalComplianceReport]:
    checker = LegalComplianceChecker()
    standards = standards or list(LEGAL_STANDARDS_DB.keys())

    reports = {}
    for sid in standards:
        reports[sid] = checker.check(audit_report, manual_verifications, sid)

    # HTML 리포트
    html = _render_global_report_html(reports, audit_report)
    Path(output_path).parent.mkdir(parents=True, exist_ok=True)
    with open(output_path, "w", encoding="utf-8") as f:
        f.write(html)

    # JSON 요약
    json_path = output_path.replace(".html", ".json")
    with open(json_path, "w", encoding="utf-8") as f:
        json.dump(
            {sid: {
                "compliance": r.overall_compliance,
                "risk":       r.risk_level,
                "tech_score": r.technical_score,
                "failures":   r.technical_failed_criteria,
                "manual":     f"{r.manual_items_verified}/{r.manual_items_total}",
            } for sid, r in reports.items()},
            f, ensure_ascii=False, indent=2,
        )

    print(f">> 통합 리포트: {output_path}")
    print(f">> JSON 요약:   {json_path}")

    # 콘솔 요약
    for sid, r in reports.items():
        emoji = {"compliant": "✅", "partial": "⚠️ ", "non-compliant": "❌"}[r.overall_compliance]
        print(f"{emoji} {sid}: {r.technical_score:.0f}점 / {r.overall_compliance} ({r.risk_level})")

    return reports


def _render_global_report_html(reports: dict, audit: dict) -> str:
    rows = []
    for sid, r in reports.items():
        color = {
            "compliant": "#10b981", "partial": "#f59e0b", "non-compliant": "#ef4444"

        rows.append(f"""
        <tr>
          <td>{r.standard_name}</td>
          <td>{r.technical_score:.1f}</td>
          <td style="color:{color}; font-weight:bold">
            {r.overall_compliance.upper()}
          </td>
          <td>{r.risk_level}</td>
          <td>{r.manual_items_verified}/{r.manual_items_total}</td>
          <td>{len(r.technical_failed_criteria)}</td>
        </tr>""")

    return f"""<!DOCTYPE html>

<title>글로벌 접근성 법규 준수 리포트 — WIA 글살림</title>

  body {{ font-family: sans-serif; max-width: 1200px; margin: 2em auto; padding: 0 1em; }}
  table {{ border-collapse: collapse; width: 100%; }}
  th, td {{ border: 1px solid #ddd; padding: 12px; text-align: left; }}
  th {{ background: #6c3ce1; color: white; }}
</style>
</head><body>
<h1>글로벌 접근성 법규 준수 리포트</h1>
<p>생성: {datetime.now(timezone.utc).isoformat()}</p>
<p>전체 감사 점수: <strong>{audit.get('score', 0)} / 100</strong></p>

<table>
<thead><tr>
  <th>표준</th><th>기술 점수</th><th>준수</th><th>리스크</th>

{"".join(f"<li><strong>{sid}</strong>: {', '.join(r.recommendations)}</li>"
         for sid, r in reports.items())}

## 함정 카드

1. **"법규 준수 = 소송 면제라고 오해"** → EAA·ADA·장차법 모두 "합리적 편의 제공" 기준이라 **감사 100점이라도 특정 사용자가 소송 가능**. 예방: ① 접근성 불만 접수 채널 명시 ② 사용자 피드백 반영 프로세스 ③ 주기적 장애인 사용자 테스트. 기술 점수만으론 불충분.

2. **"EU EAA를 WCAG 2.1로만 맞춤"** → 2025-06 발효 시점엔 2.1 기반이었으나 **2026년 개정 작업 중, 2.2 전환 확실시**. 지금부터 2.2 AA로 구현해 개정 대응. 2.1 맞추는 건 2년 내 재작업 필수.

3. **"한국 KS X 3253 = 형식 요건만"이라 오해** → 장애인차별금지법은 실제 소송 가능. 2024년 한국 대형 쇼핑몰이 시각장애인 원고에 패소 (정당한 편의 제공 거부). **KWCAG + 실제 사용성 개선**이 필수.

4. **"법무 검토 없이 접근성 진술서 자동 생성"** → 진술서에 **"100% 준수"** 등 과장 표현은 소송 근거가 됨. 실제 수준에 맞춰 "부분 준수, 특정 영역 개선 중"이라 **정직하게 기재**. 법무 템플릿 활용.

5. **"다국가 지원이라면서 특정 국가만 감사"** → 글로벌 공개 시 모든 국가 법규 해당. EU 사용자가 한국에서 호스팅된 사이트 접근 시 **EU EAA 적용**. 서버 위치가 아닌 **사용자 위치** 기준. 반드시 다국가 감사.

## 연습문제

**문제 1.** 감사 점수 97점인데 EU EAA 준수 "partial"로 나왔다. 원인은?

> **해설:** **수동 요건 미충족**. EAA는 기술 기준 외 3가지 수동 요구: ① 접근성 진술서 게시 ② 사용자 피드백 메커니즘 ③ 장애인 참여 사용자 테스트. 97점은 기술 ✓지만 수동 1~2건 누락이면 judge에서 `manual_rate < 1.0`으로 partial. 해결: 누락 항목 순차 이행 — 진술서·피드백은 1주, 사용자 테스트는 3~6개월 프로젝트. 단계적 compliant 진입.

**문제 2.** 한 사이트를 EU·미국·한국·일본 동시에 배포한다. 가장 엄격한 기준만 따르면 되는가?

> **해설:** **기본은 맞지만 국가별 고유 요건 있음**. 공통 기반 = WCAG 2.2 AA가 상위 교집합. 하지만 ① 한국: 웹접근성 품질마크·한국어 접근성 진술서 ② EU: EU 언어별 진술서 (최소 자국어 + 영어) ③ 일본: JIS 고유 서식 (정부 입찰 필요시) ④ 미국: VPAT 2508문서 (정부 조달 시). 기술은 하나로, **진술서·인증·서식은 국가별 별도 발급** 필요. 법무와 협의.

**문제 3.** 매달 감사 점수 변동이 95~99 사이 오간다. 어떤 자동화를 추가하면 법규 리스크 관리에 도움되는가?

> **해설:** ① **스파이크 알림**: 95 미만 24시간 지속 시 Slack + 법무팀 이메일. 월간 트렌드(prophet 예측)로 사전 경고. ② **페이지 단위 드릴다운**: 전체 97점이어도 특정 페이지 80점이면 그 페이지만 부분 비공개 처리. ③ **롤백 자동화**: 배포 후 5분 내 점수 급락 → 자동 이전 버전 롤백 + incident 생성. ④ **주기 법무 리포트**: 분기 1회 `generate_global_report` 실행 + PDF 생성 + 법무팀 승인 로그. 점수 숫자만이 아니라 **시간·페이지·역사** 축으로 관리해야 법규 리스크 정량화 가능.
