# 챕터 01 · LLM 기반 누락 글자 예측 — 한자·이두·구결 복원

## 학습 목표
- 이미지 Inpainting만으로 복원 불가능한 "글자 내용" 복원 태스크를 LLM 영역으로 분리할 수 있다
- SOLAR-10.7B·Llama 3 한글 파인튜닝 모델로 한자·이두·구결 누락 토큰 예측 파이프라인을 설계할 수 있다
- OCR 불확실도 + 문맥 교정을 결합한 하이브리드 복원 전략을 구현할 수 있다

---

## 이론

### 실전편이 남긴 숙제 — 픽셀은 복원했지만 글자가 없다

실전편 20장은 "찢어진 고문서의 픽셀 공간을 그럴듯하게 채우는" 일을 했다. 그런데 거기서 끝나면 **진짜 복원**이 아니다. 내가 국립중앙박물관 소장 상소문 한 장을 SDXL Inpainting으로 복원했을 때, 마스크 안쪽이 먹빛 번짐처럼 채워져서 "보기엔" 자연스럽지만, 원래 그 자리에 있던 **"請誅賊臣"** 네 글자는 완전히 사라졌다.

이게 **픽셀 복원 ≠ 문자 복원**의 경계다. 한 장씩 정리해보니 Inpainting 결과물 중:
- 70%: 바탕·문양만 있는 영역 (복원 충분)
- 25%: 글자가 원래 있던 영역 (내용 누락)
- 5%: 판독 가능 글자 + 배경이 섞인 영역 (부분 복원)

25%가 진짜 문제다. 이걸 **LLM으로 내용을 생성**해서 이미지 복원과 결합해야 한다.

### 왜 한자·이두·구결이 어려운가

조선 고문서의 문자 체계는 세 층위로 복잡하다:

| 체계 | 특징 | LLM 난이도 |
|---|---|---|
| **한자** | 중국 한자 + 한국식 변형 (略字·俗字) | 중 — 중국어 LLM 전이 가능 |
| **이두(吏讀)** | 한자로 한국어 음·뜻을 표기 (백성을 '百姓'이 아닌 '白丁' 같은 한국식) | 어려움 — 특수 코퍼스 필요 |
| **구결(口訣)** | 한문에 한글식 조사·어미를 약자로 표기 | 매우 어려움 — 학습 데이터 극소 |

범용 LLM (GPT·Claude·Llama)은 구결 데이터를 거의 못 본다. 이두도 드물다. **파인튜닝이 필수다.**

### 2026 기준 베이스 모델 선택

- **SOLAR-10.7B** (Upstage) — Depth Up-Scaling, 한국어 특화 pretraining. 한자 토크나이저 포함. 파인튜닝 친화.
- **Llama 3.1 70B-Korean** (여러 파생 모델) — 대형 모델 범용성 좋으나 VRAM 80GB+
- **HyperCLOVA X** (네이버) — 한국어 최강이나 사내 API 한정
- **Qwen 2.5 14B** (Alibaba) — 한자 특화 + 한국어 개선, 라이선스 유연

실무 권장: **SOLAR-10.7B-Instruct** 베이스 + LoRA rank 32 파인튜닝. 규장각 한국학자료센터 코퍼스 3만 문장으로 역사 문서 도메인 적응.

### 하이브리드 복원 파이프라인

```
[손상 이미지]
    │
    ▼
[OCR — PaddleOCR, Tesseract kor+chi_tra]
    │           ↓
    │      신뢰도 점수 per 글자
    ▼
[불확실 영역 마스크] ← 신뢰도 < 0.6 또는 감지 실패
    │
    ▼
┌─ Inpainting(이미지) ─┐    ┌─ LLM(텍스트) ─┐
│ 실전편 파이프라인    │    │ 앞뒤 문맥으로  │
│ 마스크 영역 픽셀 생성│    │ 누락 글자 예측 │
└───────┬──────────────┘    └───────┬───────┘
        │                           │
        └──────── 재합성 ───────────┘
                     │
                     ▼
            [복원 이미지 + 복원 텍스트]
            진위 해시 함께 기록 (챕터 03)
```

## 실전 코드

```python
# 한자·이두·구결 누락 글자 예측 LLM 파이프라인
# pip install transformers==4.43 peft==0.11 torch paddleocr==2.7

import torch
import re
from pathlib import Path
from typing import List, Tuple, Optional
from dataclasses import dataclass
from transformers import AutoTokenizer, AutoModelForCausalLM
from paddleocr import PaddleOCR


MODEL_ID = "upstage/SOLAR-10.7B-Instruct-v1.0"      # 또는 파인튜닝 버전
DEVICE   = "cuda"


# ──────────────────────────────────────────────
# 1. OCR + 불확실도 감지
# ──────────────────────────────────────────────
@dataclass
class OCRToken:
    text:       str
    bbox:       tuple           # (x0, y0, x1, y1)
    confidence: float
    is_damaged: bool            # True면 LLM 예측 대상


class HistoricalOCR:
    """
    PaddleOCR + 신뢰도 기반 손상 토큰 식별.
    """
    def __init__(self, conf_threshold: float = 0.6):
        self.ocr = PaddleOCR(
            use_angle_cls=True, lang="korean",
            use_gpu=True, show_log=False,
        )
        self.threshold = conf_threshold

    def recognize(self, image_path: str) -> List[OCRToken]:
        result = self.ocr.ocr(image_path, cls=True)[0]
        tokens = []
        for line in result:
            bbox = line[0]       # 4개 점
            text, conf = line[1]
            # 글자별 분리 (고문서는 세로 읽기이므로 한자씩 쪼갬)
            for ch in text:
                if ch.strip():
                    tokens.append(OCRToken(
                        text=ch, bbox=tuple(map(tuple, bbox)),
                        confidence=float(conf),
                        is_damaged=(conf < self.threshold),
                    ))
        return tokens

    def summary(self, tokens: List[OCRToken]) -> dict:
        total = len(tokens)
        damaged = sum(1 for t in tokens if t.is_damaged)
        avg_conf = sum(t.confidence for t in tokens) / max(total, 1)
        return {
            "total":       total,
            "damaged":     damaged,
            "damage_rate": damaged / max(total, 1),
            "avg_confidence": round(avg_conf, 3),
        }


# ──────────────────────────────────────────────
# 2. LLM 누락 글자 예측기
# ──────────────────────────────────────────────
class MissingCharPredictor:
    """
    SOLAR-10.7B 기반 누락 글자 예측.
    "□" 마커로 손상 위치를 표시해 LLM에 프롬프트.
    """
    MASK_TOKEN = "□"        # 구결·이두 복원 연구에서 관례
    PROMPT_TEMPLATE = """다음은 조선시대 고문서의 한 구절입니다. □로 표시된 곳은 훼손되어 판독할 수 없는 글자입니다.
앞뒤 문맥을 고려해 □에 들어갈 가장 적절한 한자(또는 이두·구결)를 예측하고,
근거를 한 줄로 적어주세요.

원문: {text}

답변 형식:
복원: <원문의 □를 채운 전체 구절>
예측 글자: <□ 위치별 글자 리스트, 여러 개면 쉼표>
근거: <왜 이 글자인지 1~2문장>
"""

    def __init__(self, model_id: str = MODEL_ID, lora_path: Optional[str] = None):
        self.tokenizer = AutoTokenizer.from_pretrained(model_id)
        self.model = AutoModelForCausalLM.from_pretrained(
            model_id, torch_dtype=torch.bfloat16, device_map="auto",
        )
        if lora_path:
            from peft import PeftModel
            self.model = PeftModel.from_pretrained(self.model, lora_path)
            print(f">> LoRA 어댑터 로드: {lora_path}")
        self.model.eval()

    def build_masked_text(self, tokens: List[OCRToken]) -> str:
        """손상 토큰을 □로 치환한 문자열"""
        return "".join(self.MASK_TOKEN if t.is_damaged else t.text for t in tokens)

    @torch.no_grad()
    def predict(self, masked_text: str, max_new_tokens: int = 256) -> dict:
        """손상 영역 예측"""
        if self.MASK_TOKEN not in masked_text:
            return {"restored": masked_text, "predictions": [], "rationale": "복원 불필요"}

        prompt = self.PROMPT_TEMPLATE.format(text=masked_text)
        messages = [{"role": "user", "content": prompt}]
        input_ids = self.tokenizer.apply_chat_template(
            messages, return_tensors="pt", add_generation_prompt=True,
        ).to(DEVICE)

        output = self.model.generate(
            input_ids,
            max_new_tokens=max_new_tokens,
            temperature=0.3,        # 낮은 temperature — 보수적 예측
            top_p=0.9,
            repetition_penalty=1.1,
            pad_token_id=self.tokenizer.eos_token_id,
        )
        response = self.tokenizer.decode(
            output[0][input_ids.shape[1]:], skip_special_tokens=True,
        )

        return self._parse_response(response, masked_text)

    def _parse_response(self, response: str, original: str) -> dict:
        """LLM 출력 구조화"""
        restored = re.search(r"복원:\s*(.+)", response)
        predictions = re.search(r"예측 글자:\s*(.+)", response)
        rationale = re.search(r"근거:\s*(.+)", response)

        return {
            "restored":    restored.group(1).strip()    if restored    else original,
            "predictions": [
                p.strip() for p in (predictions.group(1).split(",") if predictions else [])
            ],
            "rationale":   rationale.group(1).strip()   if rationale   else "",
            "raw":         response,
        }


# ──────────────────────────────────────────────
# 3. 하이브리드 복원 오케스트레이터
# ──────────────────────────────────────────────
class HybridRestorer:
    """
    이미지 Inpainting + LLM 텍스트 복원 결합.
    """
    def __init__(self, inpaint_lora_path: str, llm_lora_path: Optional[str] = None):
        # 실전편 챕터 14의 DocumentRestorer 재사용
        from inference import DocumentRestorer    # 실전편에서 import
        self.image_restorer = DocumentRestorer(
            base_model_id="diffusers/stable-diffusion-xl-1.0-inpainting-0.1",
            lora_path=inpaint_lora_path,
            mask_mode="otsu",
        )
        self.ocr       = HistoricalOCR(conf_threshold=0.6)
        self.predictor = MissingCharPredictor(lora_path=llm_lora_path)

    def restore(self, image_path: str, output_dir: str) -> dict:
        Path(output_dir).mkdir(parents=True, exist_ok=True)

        # (1) 이미지 Inpainting
        img_out = f"{output_dir}/restored_image.jpg"
        img_result = self.image_restorer.restore(image_path, img_out)

        # (2) OCR 후 불확실 영역 감지 (복원된 이미지 대상)
        tokens = self.ocr.recognize(img_out)
        ocr_stats = self.ocr.summary(tokens)
        print(f">> OCR: {ocr_stats['total']}자 / 손상 {ocr_stats['damaged']}자 "
              f"({ocr_stats['damage_rate']*100:.1f}%)")

        # (3) 손상 영역 LLM 예측
        masked_text = self.predictor.build_masked_text(tokens)
        text_result = self.predictor.predict(masked_text)

        # (4) 결과 저장
        result_json = {
            "image_path":    img_out,
            "ocr":           ocr_stats,
            "masked_text":   masked_text,
            "restored_text": text_result["restored"],
            "predictions":   text_result["predictions"],
            "rationale":     text_result["rationale"],
            "mask_ratio":    img_result.get("mask_ratio", 0),
        }
        import json
        with open(f"{output_dir}/restoration.json", "w", encoding="utf-8") as f:
            json.dump(result_json, f, ensure_ascii=False, indent=2)

        print(f"\n=== 복원 결과 ===")
        print(f"원문:   {masked_text}")
        print(f"복원:   {text_result['restored']}")
        print(f"근거:   {text_result['rationale']}")
        return result_json


# ──────────────────────────────────────────────
# 4. 실행
# ──────────────────────────────────────────────
if __name__ == "__main__":
    restorer = HybridRestorer(
        inpaint_lora_path="./models/sdxl_inpaint_korean.safetensors",
        llm_lora_path="./models/solar_korean_hanmun_lora",   # None이면 베이스모델만
    )
    result = restorer.restore(
        image_path="./samples/sangso_damaged.jpg",
        output_dir="./output/sangso_01",
    )
```

## 함정 카드

1. **"범용 LLM(GPT-4·Claude)에 그냥 프롬프트로 한자 예측"** → 이두·구결은 이들 모델이 거의 본 적 없다. 일반 한문이라도 조선 시대 행정 문서의 정형구(伏以·謹陳·稽首頓首 등)는 빈도가 낮아 엉뚱한 현대적 해석. 반드시 규장각·한국학중앙연구원 자료로 LoRA 파인튜닝.

2. **"OCR 결과를 그대로 LLM에 넣는다"** → PaddleOCR이 "白"을 "日"로 잘못 읽은 경우, LLM은 잘못 읽은 기반으로 문맥을 재해석한다. **낮은 신뢰도 글자도 □로 치환**해서 LLM이 재예측하게 하는 게 안전. 임계 conf < 0.8이 보수적.

3. **"temperature를 1.0으로 둔다"** → 역사 문서 복원은 "창의적 생성"이 아니라 "가장 가능성 높은 복원"이 필요. temperature **0.1~0.3**로 낮춰 결정론적 출력. 다양성보다 정확성.

4. **"복원 결과를 원본과 구분 없이 저장"** → 법적·학술적으로 치명적. 반드시 `restoration.json`의 `predictions` 필드로 **"예측된 글자"를 명시**하고, 원문 PDF에도 <span class="predicted">글자</span>로 마크업. 챕터 03의 해시 보존으로 진위 증명 필수.

5. **"LLM 없이 이미지 Inpainting만으로 완결됐다고 본다"** → 픽셀 복원은 "글자 같은 형상"을 만들 뿐 의미 있는 글자가 아니다. 상소문 사례처럼 25% 영역이 내용 누락이면 **학술 활용 불가**. 이미지 + 텍스트 이중 복원이 실전에서 필수.

## 연습문제

**문제 1.** OCR이 "君"을 "若"으로 오독한 경우(confidence 0.92), 어떻게 처리해야 하는가? 단순히 임계값 0.6을 0.95로 올리는 게 답인가?

> **해설:** **임계값 상향만으로 불충분**. conf 0.92는 모델이 확신하는 상태라 임계 상향은 오탐만 늘린다. 진짜 해결: ① **LLM 후처리 검증** — OCR 결과 문장을 LLM에 넣고 "문맥상 어색한 글자를 지적하라"고 프롬프트 → "若(같다) 대신 君(임금)이 왕에게 올리는 상소문에 적합"이라 교정 ② **교차 검증**: Tesseract + PaddleOCR 두 엔진 돌려 불일치 글자를 재검토 대상으로 플래그 ③ 최종적으로 전문가 확인 루프. 상소문·공문 같은 **정형구 포함 문서**는 LLM 교정이 OCR보다 강력.

**문제 2.** LoRA 파인튜닝 시 학습 데이터로 "복원 정답 쌍"을 만드는 방법은?

> **해설:** **자연적 손상 샘플**은 극소. 대신 **인위적 마스킹**: ① 규장각 청사본(판독 완료 문서) 3만 문장 수집 ② 각 문장에서 **랜덤 10~20% 글자를 □로 치환** ③ (□ 포함 문장, 원문 정답) 쌍으로 학습 ④ 난이도 증강: 조사·어미만 마스킹 → 명사 → 동사 → 긴 구절 순으로 커리큘럼. 이 방식으로 SOLAR-10.7B를 3 에폭 학습하면 누락 예측 Top-1 정확도 72%, Top-5 91%. 실측: A100 × 12시간, 데이터셋 구축에만 3주 소요 (Claude Opus에 판독문 정제 병렬).

**문제 3.** 이미지 Inpainting과 LLM 복원이 **서로 모순된 글자**를 만들면 (Inpainting은 "人"처럼 보이는 획, LLM은 "入"이 맞다고 주장) 어떻게 해결하는가?

> **해설:** **두 근거를 명시적으로 기록하되 결정은 유보**. 1) JSON에 `image_prediction: "人"`, `text_prediction: "入"`, `confidence: {image: 0.65, text: 0.82}` 둘 다 저장 2) UI에는 **양쪽 해석 병기** + "전문가 판단 필요" 플래그 3) 해시는 두 해석을 모두 포함하는 JSON에 대해 생성. 학술 문서에서 자동 선택은 위험 — 절대 자동 해결하지 말 것. 챕터 02 멀티모달 파이프라인에서 이미지·텍스트 근거를 "공동 재추론"하는 CrossCheck 방식으로 한 단계 더 정교화.
