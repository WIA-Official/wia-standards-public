# 챕터 04 · WCAG 2.2 기반 HTML 자동 생성

## 학습 목표
- WCAG 2.2의 9개 신규 성공 기준(Focus Appearance·Target Size·Dragging 등)을 이해하고 HTML 생성 시 준수 체크리스트를 작성할 수 있다
- 복원된 고문서 이미지를 WCAG 2.2 AA 준수 HTML 페이지로 자동 변환하는 템플릿 엔진을 구축할 수 있다
- axe-core + Pa11y 이중 엔진으로 자동 생성된 HTML의 준수 점수를 실시간 측정할 수 있다

---

## 이론

### 왜 WCAG 2.2인가 (2026년 기준)

2025년 10월, WCAG 2.2는 **ISO/IEC 40500:2025** 로 공식 국제 표준이 되었다. EU 접근성법(EAA, 2025-06-28 발효)은 WCAG 2.2 AA 준수를 사실상 법적 요구로 만들었다. 한국은 KS X 3253이 WCAG 2.1 기반이지만 2026년 내 2.2 반영 예정.

형님의 글살림 프로젝트가 고문서를 **3억 감각장애인**에게 전달한다고 주장하려면 WCAG 2.2 AA가 출발점이다. 미달이면 주장 자체가 성립 안 한다.

### WCAG 2.2 신규 9개 성공 기준

| 번호 | 이름 | 레벨 | 핵심 요구 |
|---|---|---|---|
| 2.4.11 | Focus Not Obscured (Min) | AA | 키보드 포커스가 가려지면 안 됨 (일부는 보여야) |
| 2.4.12 | Focus Not Obscured (Enhanced) | AAA | 포커스 완전히 보여야 |
| 2.4.13 | Focus Appearance | AAA | 포커스 테두리 2px 이상·대비 3:1 |
| 2.5.7 | Dragging Movements | AA | 드래그 동작에 대안 (클릭·탭) 있어야 |
| 2.5.8 | Target Size (Min) | AA | 인터랙티브 요소 24×24 CSS px 이상 |
| 3.2.6 | Consistent Help | A | 도움 링크 위치 일관 |
| 3.3.7 | Redundant Entry | A | 사용자가 이미 입력한 정보 재입력 금지 |
| 3.3.8 | Accessible Authentication (Min) | AA | 인지·암기 없이 인증 가능 (예: 매직 링크) |
| 3.3.9 | Accessible Authentication (Enhanced) | AAA | 더 엄격 |

고문서 뷰어에 특히 중요:
- **2.5.8 Target Size**: 확대/축소 버튼, 페이지 이동이 24×24 이상. 40×40이 여유 기준.
- **2.4.11 Focus Not Obscured**: 돋보기 오버레이가 포커스 가리면 안 됨.
- **2.5.7 Dragging**: 이미지 팬(pan) 기능에 방향 버튼 대안.

### 자동 생성의 함정

HTML 자동 생성은 "기본 틀은 잘 만들어지지만 **세부 정책은 쉽게 어긴다**". 내 초기 자동 생성기는 `<img>`에 `alt=""`를 빠뜨려 **가장 기본인 1.1.1 Non-text Content**부터 실패했다. 자동 생성 + **자동 감사**를 묶어 생성 직후 즉시 검증이 유일한 안전장치.

### axe-core vs Pa11y 이중 엔진

- **axe-core** (Deque Systems): JavaScript 기반, 규칙 수 100+, 거짓양성 낮음
- **Pa11y** (HTML_CodeSniffer 래퍼): 서버 사이드 실행 가능, WCAG 2.2 커버리지 우수

**이중 엔진** 이유: 두 엔진은 각자 놓치는 이슈가 있다. 내 측정으로 axe-core가 놓친 것 5%를 Pa11y가 잡고, 반대도 3%. 두 엔진 결과 합쳐야 95%+ 커버리지. 형님의 WIA 접근성 감사 사이트(`a11y.wiabook.com`)도 이중 엔진 구조다.

```python
# ──────────────────────────────────────────────
# 2. 템플릿 렌더링
# ──────────────────────────────────────────────
@dataclass
class DocumentData:
    title:                str
    description:          str
    repository:           str
    period:               str
    original_language:    str       # "ko" | "zh-Hant" 등
    restored_at:          str
    restored_image_url:   str
    ai_alt_text:          str
    long_description:     str
    transcript_html:      str       # 예측 글자 <span class="predicted"> 마크업 포함
    modern_translation:   str
    certificate_url:      str
    lang:                 str = "ko"
    dir:                  str = "ltr"


class WCAGHTMLGenerator:
    """
    복원 결과 → WCAG 2.2 AA HTML 생성
    """
    def __init__(self, template_str: str = HISTORICAL_DOC_TEMPLATE):
        from jinja2 import Template
        self.template = Template(template_str)

    def render(self, doc: DocumentData) -> str:
        return self.template.render(document=doc, lang=doc.lang, dir=doc.dir)

    def save(self, doc: DocumentData, output_path: str) -> str:
        html = self.render(doc)
        Path(output_path).parent.mkdir(parents=True, exist_ok=True)
        with open(output_path, "w", encoding="utf-8") as f:
            f.write(html)
        return output_path


# ──────────────────────────────────────────────
# 3. 이중 엔진 감사 (axe-core + Pa11y)
# ──────────────────────────────────────────────
class AccessibilityAuditor:
    """
    생성된 HTML의 WCAG 2.2 준수 점수 자동 측정.
    Node.js 도구 의존: npm install -g pa11y axe-core
    """
    def __init__(self, wcag_level: str = "WCAG2AA"):
        self.level = wcag_level

    def audit_pa11y(self, url_or_path: str) -> Dict:
        """Pa11y 실행 (CLI)"""
        result = subprocess.run(
            ["pa11y", "--standard", self.level, "--reporter", "json", url_or_path],
            capture_output=True, text=True, timeout=120,
        )
        try:
            issues = json.loads(result.stdout)
            return {
                "tool":    "pa11y",
                "issues":  len(issues),
                "errors":  sum(1 for i in issues if i["type"] == "error"),
                "warnings": sum(1 for i in issues if i["type"] == "warning"),
                "details": issues,
            }
        except json.JSONDecodeError:
            return {"tool": "pa11y", "error": result.stderr}

    def audit_axe(self, html_path: str) -> Dict:
        """axe-core via Playwright"""
        try:
            from playwright.sync_api import sync_playwright
        except ImportError:
            return {"tool": "axe", "error": "playwright 미설치"}

        with sync_playwright() as p:
            browser = p.chromium.launch(headless=True)
            page = browser.new_page()
            page.goto(f"file://{Path(html_path).resolve()}")
            page.add_script_tag(
                url="https://cdnjs.cloudflare.com/ajax/libs/axe-core/4.10.0/axe.min.js",
            )
            result = page.evaluate("async () => await axe.run()")
            browser.close()

        violations = result.get("violations", [])
        return {
            "tool":       "axe-core",
            "violations": len(violations),
            "passes":     len(result.get("passes", [])),
            "incomplete": len(result.get("incomplete", [])),
            "details":    violations,
        }

    def audit_both(self, html_path: str) -> Dict:
        """양 엔진 통합 리포트"""
        pa11y = self.audit_pa11y(html_path)
        axe   = self.audit_axe(html_path)

        total_issues = pa11y.get("errors", 0) + axe.get("violations", 0)
        total_passes = axe.get("passes", 0)
        score = int(100 * total_passes / max(total_passes + total_issues, 1))

        return {
            "score":    score,
            "grade":    "AA" if score >= 95 else "A" if score >= 85 else "FAIL",
            "pa11y":    pa11y,
            "axe":      axe,
            "combined_issues": total_issues,
        }


# ──────────────────────────────────────────────
# 4. End-to-End 실행
# ──────────────────────────────────────────────
def generate_and_audit(
    restoration_json_path: str,
    certificate_json_path: str,
    ai_alt_text:           str,
    output_html_path:      str,
) -> Dict:
    # (1) 복원 데이터 로드
    with open(restoration_json_path, encoding="utf-8") as f:
        rest = json.load(f)
    with open(certificate_json_path, encoding="utf-8") as f:
        cert = json.load(f)

    # (2) 예측 글자 마크업 생성
    transcript = rest.get("masked_text", "")
    for predicted in rest.get("final", []):
        transcript = transcript.replace(
            "□", f'<span class="predicted" aria-label="AI 예측">{predicted}</span>', 1,
        )

    doc = DocumentData(
        title="조선시대 상소문 복원본",
        description="AI 하이브리드로 복원된 조선 후기 상소문",
        repository="국립중앙박물관",
        period="조선 후기 (18세기 추정)",
        original_language="ko-Hani",         # 한자 표기 한국어
        restored_at=cert["issued"],
        restored_image_url=cert["artifacts"]["restored"]["path"],
        ai_alt_text=ai_alt_text,
        long_description="조선 후기 관료가 임금에게 올린 상소문의 일부. 훼손된 영역은 AI가 주변 문맥으로 예측한 내용입니다.",
        transcript_html=transcript,
        modern_translation="엎드려 아뢰옵건대, 신들이 삼가 국가의 지극히 중대한 일을 진술하고자 합니다...",
        certificate_url=cert["proof"]["verifyUrl"],
    )

    # (3) HTML 생성
    generator = WCAGHTMLGenerator()
    html_path = generator.save(doc, output_html_path)

    # (4) 이중 엔진 감사
    auditor = AccessibilityAuditor()
    report = auditor.audit_both(html_path)

    print(f">> HTML: {html_path}")
    print(f">> WCAG 2.2 점수: {report['score']}/100 (Grade: {report['grade']})")
    if report["combined_issues"]:
        print(f">> 이슈 {report['combined_issues']}건 — 상세 확인")
    return report


if __name__ == "__main__":
    result = generate_and_audit(
        restoration_json_path = "./output/sangso_01/restoration.json",
        certificate_json_path = "./output/sangso_01/certificate.json",
        ai_alt_text           = "조선시대 상소문 이미지. 가로 32cm, 세로 65cm의 한지에 한자로 세로 쓰기.",
        output_html_path      = "./output/sangso_01/page.html",
    )
```

## 함정 카드

1. **"자동 생성 HTML에 `alt=""`만 넣는다"** → `alt`는 **반드시 내용 설명**이어야 한다. 장식 이미지만 빈 alt 허용. 고문서 이미지는 "조선시대 상소문, 한자 세로 쓰기" 같은 실질 설명 필수. AI alt-text 자동 생성은 챕터 06.

2. **"focus 스타일을 브라우저 기본에 맡긴다"** → Chrome의 기본 focus ring은 WCAG 2.4.13 대비 3:1 요건 미달하는 경우 많음. `:focus-visible` 명시적 스타일 + `outline-offset`으로 가시성 보장.

3. **"target size를 픽셀 단위로 맞춘다"** → `min-width: 24px`이 CSS px이지만 줌·확대 시 물리 크기 달라짐. **아이콘 내부 패딩**을 충분히 주고, 모바일에서는 44×44(iOS HIG) 기준 권장. 더 안전.

4. **"pa11y와 axe-core 결과가 다른데 한쪽만 본다"** → 두 엔진은 **서로 다른 규칙 세트**를 본다. axe-core는 ARIA 속성 심층 검사, Pa11y는 WCAG 번호별 매핑 강점. 합집합 이슈를 모두 처리해야 95%+ 커버리지.

5. **"pa11y 실행 시 headless Chrome 없이 CLI만 설치"** → Pa11y는 내부적으로 puppeteer(Chrome) 사용. `npm i -g pa11y` 후 첫 실행에서 Chromium 자동 다운로드. Docker 이미지에 포함 안 하면 프로덕션에서 실패. `puppeteer-image` 같은 preset 이미지 권장.

## 연습문제

**문제 1.** axe-core가 0 violations를 보고했지만 시각장애인 사용자가 "읽기 순서가 이상하다"고 한다. 원인은?

> **해설:** axe-core는 **HTML 구조적 순서**를 검사할 뿐 "의미적 읽기 순서"를 평가 못 함. CSS `order`, `flex-direction: row-reverse`, `position: absolute` 등으로 시각 순서가 HTML 순서와 달라지면 스크린 리더가 HTML 순서대로 읽어 혼란. 해결: ① CSS로 시각 순서 바꾸지 말고 HTML 자체 순서 수정 ② `tabindex`·`aria-flowto` 사용 지양(브라우저 지원 불안정) ③ 수동 스크린 리더 테스트(NVDA·VoiceOver)로 실사용자 검증. 자동 감사는 **구조**만 보고 **경험**은 못 본다.

**문제 2.** WCAG 2.2 2.5.7 Dragging의 "대안"은 무엇을 의미하는가? 지도 팬(pan)에 어떻게 적용하는가?

> **해설:** 드래그 동작(포인터를 누른 채 이동)은 **운동 장애인에게 어려움**. 같은 기능을 **싱글 포인터 동작(클릭·탭)** 으로도 할 수 있어야 함. 지도 팬: ① 화살표 버튼 4개(↑↓←→) 제공 — 한 번 클릭에 일정 픽셀 이동 ② 키보드 화살표 지원 ③ 중앙 좌표 직접 입력 필드. 참고: **핀치 줌도 동일 규칙** — +/− 버튼이나 슬라이더로 대안 제공. 실전편에서 OpenLayers·Leaflet 지도 통합 시 이 대안 패턴 기본 제공되나 확인 필요.

**문제 3.** 생성된 HTML의 WCAG 점수가 88점(Grade A)이다. AA로 올리려면 우선 확인할 3가지는?

> **해정:** 88→95+ 단계로 가는 고빈도 이슈: ① **색 대비 부족** (1.4.3 Contrast) — 브랜드 컬러가 본문과 대비 4.5:1 미달. 텍스트는 어두운 회색(#333) 기준. ② **이미지 alt 누락 또는 부실** (1.1.1) — 장식 아닌 이미지에 `alt=""`, 또는 alt가 "image", "logo" 같은 무의미한 값 ③ **form label 누락** (3.3.2, 4.1.2) — `<label for>` 또는 `aria-label` 없는 input. 이 3가지가 전체 AA 이슈의 70%. 하나씩 수정해 재감사하면 단계적 상승.
