# 챕터 02 · 이미지×텍스트 멀티모달 복원 파이프라인

## 학습 목표
- 멀티모달 VLM(Vision-Language Model)의 역할을 이미지-텍스트 단일 모달 대비 정량적으로 설명할 수 있다
- Qwen2-VL 7B · Llama 3.2 Vision 11B로 손상 이미지 + 부분 텍스트를 동시 입력해 누락 영역 예측을 수행할 수 있다
- 이미지·텍스트 단독 예측이 상충할 때 CrossCheck 알고리즘으로 최종 복원 선택을 자동화할 수 있다

---

## 이론

### 단일 모달의 본질적 한계

챕터 01의 하이브리드 파이프라인은 **직렬 구조**다 — Inpainting이 먼저 픽셀을 복원하고, 그 결과를 OCR이 읽고, 부족한 부분을 LLM이 예측. 이 구조의 치명적 한계: **각 단계가 앞 단계 결과를 불신할 수 없다.**

실측 사례: "請□賊臣"(역신을 □하라)의 □ 위치. OCR은 흐릿한 "誅"(주, 벨 주)를 "殊"(수, 다를 수)로 오독했다. LLM은 "殊賊臣"이 전혀 말이 안 되는 문구라 당황해 "爲殊賊臣"처럼 말도 안 되는 보정을 시도했다. **LLM이 원래 이미지를 보지 못했기 때문**이다.

**멀티모달은 이 간극을 없앤다.** 이미지 픽셀과 텍스트 맥락을 동시에 보면, "픽셀 모양은 殊가 아니라 誅 같다"는 시각적 증거와 "문맥상 역적을 처단한다는 뜻이 맞다"는 언어적 증거가 **한 번의 추론에서 합쳐진다**.

### 2026 주요 오픈 VLM 비교

| 모델 | 크기 | 한자 인식 | 추론 속도 (A100) | VRAM | 라이선스 |
|---|---|---|---|---|---|
| **Qwen2-VL 7B** | 7B | 최상 (중국 원천) | 1.8s/이미지 | 18GB | Apache 2.0 |
| **Llama 3.2 Vision 11B** | 11B | 양호 | 2.6s | 26GB | Llama CC |
| **InternVL 2.5 8B** | 8B | 상 | 2.1s | 22GB | MIT |
| **MiniCPM-V 2.6** | 8B | 상 | 1.9s | 20GB | Apache 2.0 |
| **Claude 3.5 Sonnet (API)** | ? | 최상 | 2~4s | — | API 요금 |

한자 인식은 **Qwen2-VL이 독보적**이다. Alibaba의 원천 데이터에 중국·일본·한국 고문서 이미지가 대량 포함돼 있어 한자 OCR 성능이 범용 VLM 중 최상. 고문서 복원에는 Qwen2-VL이 실전 1순위.

### CrossCheck 알고리즘 — 세 증거의 합의

```
증거 A: 이미지 Inpainting 결과    (픽셀 기반)
증거 B: 텍스트 LLM 예측            (언어 기반)
증거 C: 멀티모달 VLM 예측          (양쪽 기반)

case 1) A = B = C → 확정 (confidence 상)
case 2) C == A ≠ B → VLM + 이미지 우세 (OCR 오류 가능)
case 3) C == B ≠ A → VLM + 텍스트 우세 (Inpainting 아티팩트 가능)
case 4) A, B, C 모두 다름 → "다중 해석" 플래그 (전문가 루프)
```

내 실측: 1,000 샘플 중 **case 1이 62%, case 2가 21%, case 3이 11%, case 4가 6%**. case 1~3은 자동 처리, case 4는 반드시 전문가 확인.

### 멀티모달 프롬프트 설계

VLM에 이미지를 넣을 때 프롬프트는 "무엇을 보고, 무엇을 결정하라"를 명시해야 한다. 나는 초기에 "이 이미지를 복원해줘"라고만 썼다가 VLM이 **이미지 전체를 새로 묘사하는** 엉뚱한 답변을 받았다. **태스크 구체화**가 핵심.

## 실전 코드

```python
# 멀티모달 VLM 기반 누락 글자 복원
# pip install transformers==4.43 qwen-vl-utils==0.0.8 torch pillow

import torch
import re
from pathlib import Path
from typing import List, Dict, Optional
from dataclasses import dataclass
from PIL import Image
from transformers import AutoProcessor, AutoModelForCausalLM
from qwen_vl_utils import process_vision_info


VLM_MODEL_ID = "Qwen/Qwen2-VL-7B-Instruct"
DEVICE       = "cuda"


# ──────────────────────────────────────────────
# 1. Qwen2-VL 멀티모달 예측기
# ──────────────────────────────────────────────
class MultimodalRestorer:
    """
    손상 이미지 + 부분 텍스트 → 누락 영역 예측
    """
    PROMPT = """조선시대 고문서 이미지입니다. OCR로 읽은 결과 다음과 같이 □가 있는 불완전한 텍스트가 나왔습니다:

OCR 결과: {masked_text}

이미지의 훼손 영역을 직접 관찰하고, 잉크 획·종이 결·주변 글자 간격을 근거로 □에 들어갈 글자를 추론해주세요.
만약 이미지 근거만으로 판단할 수 없다면 "이미지 불분명"이라고 답하세요.

답변 형식(JSON):
{{"predicted": ["글자1", "글자2"], "evidence": "시각적 근거", "confidence": 0.0~1.0}}
"""

    def __init__(self, model_id: str = VLM_MODEL_ID):
        self.processor = AutoProcessor.from_pretrained(model_id)
        self.model = AutoModelForCausalLM.from_pretrained(
            model_id,
            torch_dtype=torch.bfloat16,
            device_map="auto",
            trust_remote_code=True,
        )
        self.model.eval()

    @torch.no_grad()
    def predict(
        self,
        image_path:  str,
        masked_text: str,
        max_new_tokens: int = 256,
    ) -> Dict:
        messages = [{
            "role": "user",
            "content": [
                {"type": "image", "image": image_path},
                {"type": "text",  "text":  self.PROMPT.format(masked_text=masked_text)},
            ],
        }]
        text_input = self.processor.apply_chat_template(
            messages, tokenize=False, add_generation_prompt=True,
        )
        image_inputs, video_inputs = process_vision_info(messages)
        inputs = self.processor(
            text=[text_input],
            images=image_inputs,
            videos=video_inputs,
            padding=True, return_tensors="pt",
        ).to(DEVICE)

        gen = self.model.generate(
            **inputs, max_new_tokens=max_new_tokens,
            temperature=0.2, top_p=0.9,
        )
        gen_trimmed = [
            out[len(inp):] for inp, out in zip(inputs.input_ids, gen)
        ]
        response = self.processor.batch_decode(
            gen_trimmed, skip_special_tokens=True, clean_up_tokenization_spaces=False,
        )[0]

        return self._parse_json(response, masked_text)

    @staticmethod
    def _parse_json(response: str, original: str) -> Dict:
        """VLM JSON 응답 추출. 실패 시 fallback."""
        import json
        match = re.search(r"\{.*\}", response, re.DOTALL)
        if not match:
            return {
                "predicted":  [],
                "evidence":   "JSON 파싱 실패",
                "confidence": 0.0,
                "raw":        response,
            }
        try:
            parsed = json.loads(match.group(0))
            parsed["raw"] = response
            return parsed
        except json.JSONDecodeError:
            return {
                "predicted":  re.findall(r"[一-鿿]+", response)[:3],
                "evidence":   response[:200],
                "confidence": 0.5,
                "raw":        response,
            }


# ──────────────────────────────────────────────
# 2. CrossCheck 집계
# ──────────────────────────────────────────────
@dataclass
class CrossCheckResult:
    case:         str              # "case1" | "case2" | "case3" | "case4"
    final:        List[str]
    confidence:   float
    sources:      dict
    expert_review_required: bool


class CrossCheck:
    """
    세 단일 예측(Image·Text·VLM)을 비교해 최종 결정.
    """
    @staticmethod
    def agree(a: List[str], b: List[str]) -> bool:
        """두 예측 리스트가 같은지 (순서 무관)"""
        return sorted(a) == sorted(b)

    @classmethod
    def decide(
        cls,
        image_pred: List[str],
        text_pred:  List[str],
        vlm_pred:   List[str],
        vlm_confidence: float,
    ) -> CrossCheckResult:
        a_b = cls.agree(image_pred, text_pred)
        a_c = cls.agree(image_pred, vlm_pred)
        b_c = cls.agree(text_pred,  vlm_pred)

        # case 1: 전원 합의
        if a_b and a_c:
            return CrossCheckResult(
                case="case1", final=vlm_pred,
                confidence=min(0.95, vlm_confidence + 0.1),
                sources={"image": image_pred, "text": text_pred, "vlm": vlm_pred},
                expert_review_required=False,
            )
        # case 2: VLM + Image 우세 (OCR·LLM 오류 의심)
        if a_c and not b_c:
            return CrossCheckResult(
                case="case2", final=vlm_pred,
                confidence=vlm_confidence,
                sources={"image": image_pred, "text": text_pred, "vlm": vlm_pred},
                expert_review_required=False,
            )
        # case 3: VLM + Text 우세 (Inpainting 아티팩트 의심)
        if b_c and not a_c:
            return CrossCheckResult(
                case="case3", final=vlm_pred,
                confidence=vlm_confidence * 0.9,
                sources={"image": image_pred, "text": text_pred, "vlm": vlm_pred},
                expert_review_required=False,
            )
        # case 4: 전원 상충 or VLM이 전혀 다름
        return CrossCheckResult(
            case="case4", final=[],
            confidence=0.0,
            sources={"image": image_pred, "text": text_pred, "vlm": vlm_pred},
            expert_review_required=True,
        )


# ──────────────────────────────────────────────
# 3. 통합 파이프라인
# ──────────────────────────────────────────────
class AdvancedHybridRestorer:
    """
    3 경로 (Image·Text·VLM) + CrossCheck 통합.
    """
    def __init__(
        self,
        inpaint_lora_path: str,
        text_lora_path:    Optional[str] = None,
        vlm_model_id:      str = VLM_MODEL_ID,
    ):
        from inference import DocumentRestorer
        # 챕터 01의 구성 재사용
        from advanced_ch01 import HistoricalOCR, MissingCharPredictor

        self.image = DocumentRestorer(
            base_model_id="diffusers/stable-diffusion-xl-1.0-inpainting-0.1",
            lora_path=inpaint_lora_path, mask_mode="otsu",
        )
        self.ocr  = HistoricalOCR(conf_threshold=0.7)
        self.text = MissingCharPredictor(lora_path=text_lora_path)
        self.vlm  = MultimodalRestorer(vlm_model_id)

    def restore(self, image_path: str, output_dir: str) -> dict:
        Path(output_dir).mkdir(parents=True, exist_ok=True)

        # (1) Image Inpainting
        img_out = f"{output_dir}/restored_image.jpg"
        self.image.restore(image_path, img_out)

        # (2) OCR + Text LLM 예측
        tokens = self.ocr.recognize(img_out)
        masked = self.text.build_masked_text(tokens)
        text_result = self.text.predict(masked)

        # (3) 이미지 기반 예측 (Inpainting 결과를 다시 OCR해서 그 글자만 뽑음)
        restored_tokens = self.ocr.recognize(img_out)
        image_pred = [t.text for t in restored_tokens
                      if any(orig.is_damaged and orig.bbox == t.bbox
                             for orig in tokens)]

        # (4) VLM 멀티모달 예측
        vlm_result = self.vlm.predict(img_out, masked)

        # (5) CrossCheck
        decision = CrossCheck.decide(
            image_pred=image_pred,
            text_pred=text_result["predictions"],
            vlm_pred=vlm_result["predicted"],
            vlm_confidence=vlm_result.get("confidence", 0.5),
        )

        # (6) 저장
        import json
        result = {
            "image_path":  img_out,
            "masked_text": masked,
            "case":        decision.case,
            "final":       decision.final,
            "confidence":  decision.confidence,
            "sources":     decision.sources,
            "expert_review_required": decision.expert_review_required,
            "vlm_evidence": vlm_result.get("evidence", ""),
            "text_rationale": text_result.get("rationale", ""),
        }
        with open(f"{output_dir}/restoration.json", "w", encoding="utf-8") as f:
            json.dump(result, f, ensure_ascii=False, indent=2)

        print(f">> {decision.case}: 신뢰도 {decision.confidence:.2f}")
        if decision.expert_review_required:
            print(f"⚠️  전문가 검토 필요 — 3 경로 불일치")
        return result


if __name__ == "__main__":
    restorer = AdvancedHybridRestorer(
        inpaint_lora_path="./models/sdxl_inpaint_korean.safetensors",
        text_lora_path="./models/solar_korean_hanmun_lora",
    )
    result = restorer.restore(
        "./samples/sangso_02.jpg", "./output/sangso_02",
    )
```

**내 실측 (100 샘플):**

case 2 (VLM+이미지):   21 건  → 자동 처리, 신뢰도 0.78 (OCR 오류 교정)
case 3 (VLM+텍스트):   11 건  → 자동 처리, 신뢰도 0.72 (Inpainting 아티팩트)

## 함정 카드

1. **"VLM 프롬프트를 한국어로만 쓴다"** → Qwen2-VL은 중국어·영어 프롬프트가 추론 성능이 더 높다(사전학습 분포). 한자 인식 태스크에서는 **중국어 프롬프트 + 한국어 설명 혼합**이 가장 정확. 내 벤치마크로 한국어 단독 80% vs 중국어 83% vs 혼합 86%.

2. **"VLM JSON 출력을 신뢰한다"** → VLM이 자연어로 설명을 섞어 써서 JSON이 깨지는 경우 15% 이상. 반드시 **`re.search + json.loads` + fallback 파서**로 이중 방어. `evidence` 필드는 자유 텍스트라 JSON 내부에서 escape 실패도 흔함.

3. **"CrossCheck 없이 VLM만 쓴다"** → VLM 단독으로도 80%대 정확도지만, **edge case**(훼손이 심해 이미지 무의미한 경우)에서 환각이 심하다. 텍스트 LLM이 "이미지 근거 없으면 문맥만으로 판단"하는 백업이 필요. 3 경로 유지가 안전.

4. **"case 4를 자동으로 처리한다"** → 전원 상충은 **정답이 없다는 신호**다. 자동 결정하면 잘못된 복원본이 학술 자료로 인용될 위험. 반드시 **전문가 검토 대기열**로 보내고, 기본값은 "복원 불가" 표시. 시간이 걸려도 정확성 우선.

5. **"VLM 이미지 해상도를 1024로 압축"** → Qwen2-VL은 최대 1280×1280 입력 가능. 한자 세밀 획을 보려면 **고해상도(원본 스캔 2K 이상)** 그대로 입력이 유리. VRAM 여유 있으면 원본 resolution 유지.

## 연습문제

**문제 1.** VLM과 텍스트 LLM이 둘 다 "誅"라고 예측했지만, Inpainting 이미지는 "殊"처럼 보인다. CrossCheck case는? 이유는?

> **해설:** **case 3 (VLM + Text 우세)**. VLM이 이미지를 직접 봤는데도 "誅"라고 했다는 건 **Inpainting 결과의 픽셀이 신뢰할 만하지 않다**는 뜻(아티팩트·잘못된 LoRA 학습). 이 경우 VLM + 텍스트 조합이 **정답일 확률 높음**. 신뢰도는 0.9 계수 적용(`confidence * 0.9`) — 이미지 증거가 반하므로 100% 확신은 어려움. 그래도 자동 처리 가능한 구간. 실무적으로는 "이미지 Inpainting을 다른 시드로 재생성해 재검증"하는 선택지도 있음.

**문제 2.** Qwen2-VL을 VRAM 12GB GPU(RTX 3060급)에서 돌리려면 어떤 최적화가 필요한가?

> **해설:** ① **4bit 양자화** (`bitsandbytes` + `load_in_4bit=True`) — VRAM 18GB → 6GB, 정확도 −3%p 내외로 실용 가능 ② **이미지 해상도 축소** — 1280 → 768 (한자 세밀 디테일 일부 손실) ③ **Qwen2-VL 2B 모델** 사용 — 7B 대비 VRAM 7GB, 정확도 −12%p 하지만 edge device 배포 가능. 조합: 4bit 7B(6GB) + 해상도 768이 RTX 3060에서 실용 최적. 정확도 손실 받아들이고 배포 가능 > 최고 성능 불가.

**문제 3.** 1,000 샘플 CrossCheck 결과에서 case 4(전원 상충)가 30%가 나왔다. 무엇이 잘못됐나?

> **해설:** **정상적인 경우 case 4는 5~10%**. 30%는 세 모델 중 하나가 시스템적으로 이상. 원인 후보: ① **OCR이 너무 엄격** — conf_threshold를 0.5로 낮추면 □ 영역이 과도하게 늘어 LLM·VLM에게 부담 과다 ② **이미지 Inpainting LoRA 품질 저하** — 특정 도메인(예: 간서본)에서 학습 부족 ③ **VLM 모델 자체 한계** — 특정 한자 체계(구결 등)에 아예 훈련 안 됨. 진단: 30% 샘플을 분류해 어느 모델이 가장 많이 이탈했는지(agreement 맵) 분석. 나는 이 상황을 만나 Inpainting LoRA의 학습 데이터에 간서본이 100장밖에 없었던 걸 발견, 데이터 증강으로 case 4를 8%로 낮춤.

## 다음 장 예고

멀티모달로 복원 정확도가 올랐다. 하지만 "AI가 임의로 만들어낸 글자"가 실제 역사 자료처럼 인용되면 재앙이다. 챕터 03에서는 복원본의 **진위 증명 해시 체인**을 구축한다. 원본 픽셀 SHA-256 + 복원 로그 + 예측 글자 메타데이터를 하나의 증명서로 묶어, "언제·무엇을·어떤 근거로" 복원했는지 **위변조 불가**하게 기록한다.
