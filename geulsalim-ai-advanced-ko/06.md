# 챕터 06 · AI alt-text 자동 생성 · 품질 평가

## 학습 목표
- VLM 기반 alt-text 자동 생성의 짧은 alt(≤125자)와 긴 설명(longdesc, 상세)을 분리 설계할 수 있다
- BLEU·BLEURT·CLIP 유사도·인간 평가의 4가지 지표로 alt-text 품질을 정량 측정할 수 있다
- 고문서 도메인 특화 프롬프트 템플릿과 체크리스트로 alt-text의 "사실성"을 담보할 수 있다

---

## 이론

### 왜 alt-text 자동화가 필요한가

복원된 고문서 1만 장을 전부 수동으로 alt-text 작성하면 **장당 5분 × 10,000 = 833시간 = 4개월**. 비현실적이다. 자동 생성이 필요하지만, 자동 생성 alt는 **"사실 오류"** 위험이 크다. 나는 초기 VLM 출력을 쓰다가 alt가 "조선 시대 여성이 한복을 입고 웃는 모습"인데 **실제 이미지는 상소문**인 경우를 봤다. VLM이 제목 한자를 사람 얼굴로 오인한 환각.

### alt-text의 2 층위

| 유형 | 위치 | 길이 | 용도 |
|---|---|---|---|
| **짧은 alt** | `<img alt="...">` | 125자 이내 | 스크린 리더 1차 읽기 |
| **긴 설명** | `longdesc`, `<figcaption>`, `<details>` | 무제한 | 시각장애인이 요청 시 |

짧은 alt는 "핵심 파악", 긴 설명은 "맥락·배경·구성"을 담는다. 내 경험상 **짧은 alt 1줄 + 긴 설명 3~5문장** 조합이 최적.

### 고문서 도메인 특화 프롬프트

일반 "describe this image"는 현대 사진에 맞춰 설계된 VLM 기본. 고문서에는 다음 4축을 강제:

1. **물리 속성**: 재질(한지·비단), 크기, 서체(해서·행서·초서)
2. **내용 유형**: 상소문·지도·호적·회화·족보 등 문서 유형
3. **훼손 상태**: 균열·얼룩·결손 위치와 정도
4. **복원 마크**: "AI 복원 영역 포함"임을 명시

**중요 원칙**: alt-text에 **"추정이 포함된 문장"은 표시**. 예: "18세기 상소문으로 추정" vs "18세기 상소문". 확실한 것과 추정을 구분하지 않으면 alt가 거짓 정보의 출처가 된다.

### 평가 지표 4종

| 지표 | 무엇을 보는가 | 장단점 |
|---|---|---|
| **BLEU-4** | 정답 alt과 n-gram 일치 | 빠름·객관 / 의역 감지 못함 |
| **BLEURT** | 문장 의미 유사도 (BERT 기반) | 의미 포착 / GPU 필요 |
| **CLIP 유사도** | 이미지-텍스트 임베딩 코사인 | 이미지↔alt 일관성 / 세밀 사실 못 봄 |
| **인간 평가** | 사실성·유용성 5점 척도 | 정답 / 비용 ↑ |

실무: 자동 3지표로 필터링 → 하위 20%만 인간 평가. 비용 1/5.

## 실전 코드

```python
# AI alt-text 자동 생성 + 4지표 품질 평가
# pip install transformers torch pillow evaluate bleurt-pytorch open-clip-torch

import torch
import re
import json
import numpy as np
from pathlib import Path
from typing import List, Dict, Optional
from dataclasses import dataclass, asdict
from PIL import Image
from transformers import AutoProcessor, AutoModelForCausalLM
from qwen_vl_utils import process_vision_info


VLM_MODEL_ID = "Qwen/Qwen2-VL-7B-Instruct"
DEVICE       = "cuda"


# ──────────────────────────────────────────────
# 1. 도메인 특화 프롬프트
# ──────────────────────────────────────────────
ALT_PROMPT_SHORT = """이 이미지는 조선시대 고문서입니다. 스크린 리더용 짧은 대체 텍스트를 작성하세요.

요구사항:
- 125자 이내 한국어 1문장
- 문서 유형 명시 (상소문·지도·호적·회화·족보 등)
- 서체 또는 재질 간단 언급
- 추측하는 정보엔 "추정" 붙이기
- AI 복원본임 표시 포함

답변은 alt 텍스트 한 줄만. 따옴표·설명 불필요."""

ALT_PROMPT_LONG = """이 이미지는 조선시대 고문서이며 AI로 복원된 것입니다. 시각장애인이 상세 설명을 요청할 때 읽어줄 긴 설명을 작성하세요.

4가지 축으로 3~5문장:
1. 물리 속성 (재질, 크기 추정, 서체)
2. 내용 유형 (문서의 용도·기능)
3. 훼손/복원 상태 (어디가 AI 복원 영역)
4. 문맥 (시대 배경, 관련 인물·사건 추정)

원칙:
- 확실한 사실과 추정을 구분 ("~으로 보인다", "~로 추정" 사용)
- 사실 확인 불가한 것은 서술 생략 (환각 금지)
- 한국어 자연스러운 설명체"""


# ──────────────────────────────────────────────
# 2. VLM alt 생성기
# ──────────────────────────────────────────────
class AltTextGenerator:
    def __init__(self, model_id: str = VLM_MODEL_ID):
        self.processor = AutoProcessor.from_pretrained(model_id)
        self.model = AutoModelForCausalLM.from_pretrained(
            model_id, torch_dtype=torch.bfloat16,
            device_map="auto", trust_remote_code=True,
        )
        self.model.eval()

    @torch.no_grad()
    def _generate(self, image_path: str, prompt: str, max_new: int = 256) -> str:
        messages = [{
            "role": "user",
            "content": [
                {"type": "image", "image": image_path},
                {"type": "text",  "text": prompt},
            ],
        }]
        text_input = self.processor.apply_chat_template(
            messages, tokenize=False, add_generation_prompt=True,
        )
        image_inputs, _ = process_vision_info(messages)
        inputs = self.processor(
            text=[text_input], images=image_inputs,
            padding=True, return_tensors="pt",
        ).to(DEVICE)
        out = self.model.generate(
            **inputs, max_new_tokens=max_new,
            temperature=0.2, top_p=0.9,
            repetition_penalty=1.1,
        )
        trimmed = [o[len(i):] for i, o in zip(inputs.input_ids, out)]
        response = self.processor.batch_decode(
            trimmed, skip_special_tokens=True,
        )[0].strip()
        return response

    def short_alt(self, image_path: str) -> str:
        alt = self._generate(image_path, ALT_PROMPT_SHORT, max_new=150)
        return self._clip_length(alt, max_chars=125)

    def long_description(self, image_path: str) -> str:
        return self._generate(image_path, ALT_PROMPT_LONG, max_new=400)

    @staticmethod
    def _clip_length(text: str, max_chars: int = 125) -> str:
        """문장 단위 자르기"""
        if len(text) <= max_chars:
            return text
        # 마지막 종결 부호 찾아 자름
        cut = text[:max_chars]
        last_period = max(cut.rfind("."), cut.rfind("다"), cut.rfind("요"))
        return cut[: last_period + 1] if last_period > 50 else cut + "..."

    def generate_both(self, image_path: str) -> Dict[str, str]:
        return {
            "alt":              self.short_alt(image_path),
            "long_description": self.long_description(image_path),
        }


# ──────────────────────────────────────────────
# 3. 품질 평가 4지표
# ──────────────────────────────────────────────
class AltQualityEvaluator:
    """
    BLEU-4, BLEURT, CLIP 유사도, 사실성 체크 통합.
    """

    def __init__(self):
        # BLEU
        try:
            import evaluate
            self.bleu = evaluate.load("bleu")
        except Exception:
            self.bleu = None

        # BLEURT
        try:
            from bleurt_pytorch import BleurtForSequenceClassification, BleurtTokenizer
            self.bleurt_tokenizer = BleurtTokenizer.from_pretrained("lucadiliello/BLEURT-20")
            self.bleurt_model = BleurtForSequenceClassification.from_pretrained(
                "lucadiliello/BLEURT-20",
            ).to(DEVICE).eval()
        except ImportError:
            self.bleurt_model = None

        # CLIP
        try:
            import open_clip
            self.clip_model, _, self.clip_preprocess = open_clip.create_model_and_transforms(
                "ViT-L-14", pretrained="openai", device=DEVICE,
            )
            self.clip_tokenizer = open_clip.get_tokenizer("ViT-L-14")
        except Exception:
            self.clip_model = None

    # ──────── BLEU ────────
    def bleu_score(self, predictions: List[str], references: List[str]) -> float:
        if self.bleu is None:
            return 0.0
        # 한국어는 문자 단위 토큰화가 더 정확
        result = self.bleu.compute(
            predictions=[list(p) for p in predictions],
            references=[[list(r)] for r in references],
            max_order=4,
        )
        return result.get("bleu", 0.0)

    # ──────── BLEURT ────────
    @torch.no_grad()
    def bleurt_score(self, predictions: List[str], references: List[str]) -> List[float]:
        if self.bleurt_model is None:
            return [0.0] * len(predictions)
        inputs = self.bleurt_tokenizer(
            references, predictions,
            padding=True, truncation=True, return_tensors="pt",
        ).to(DEVICE)
        scores = self.bleurt_model(**inputs).logits.flatten().cpu().tolist()
        return scores

    # ──────── CLIP 이미지-텍스트 유사도 ────────
    @torch.no_grad()
    def clip_similarity(self, image_path: str, alt_text: str) -> float:
        if self.clip_model is None:
            return 0.0
        img = self.clip_preprocess(Image.open(image_path).convert("RGB")).unsqueeze(0).to(DEVICE)
        txt = self.clip_tokenizer([alt_text]).to(DEVICE)
        img_f = self.clip_model.encode_image(img)
        txt_f = self.clip_model.encode_text(txt)
        img_f /= img_f.norm(dim=-1, keepdim=True)
        txt_f /= txt_f.norm(dim=-1, keepdim=True)
        return float((img_f @ txt_f.T).item())

    # ──────── 사실성 자가 체크 ────────
    def factuality_check(self, alt: str, restoration_json: Optional[dict] = None) -> Dict:
        """
        alt-text가 지켜야 할 규칙 자동 체크.
        """
        issues = []

        # 1. 길이
        if len(alt) > 125:
            issues.append(f"길이 초과: {len(alt)}자 (125자 제한)")

        # 2. 문서 유형 키워드
        types = ["상소문", "지도", "호적", "족보", "회화", "문집", "교지", "간찰", "판본"]
        if not any(t in alt for t in types):
            issues.append("문서 유형 미명시")

        # 3. 환각 패턴
        hallucination_patterns = [
            r"한복",         # 복원 이미지에 인물 없는데 한복 언급
            r"웃[고는]",     # 표정 묘사
            r"[0-9]{4}년 ",  # 확정적 연도 (18세기 "추정" 없이)
        ]
        for pat in hallucination_patterns:
            if re.search(pat, alt):
                issues.append(f"환각 의심 패턴: {pat}")

        # 4. AI 복원본 표시
        if "복원" not in alt and "AI" not in alt:
            issues.append("AI 복원본 표시 누락")

        # 5. 추정 표현 (확실 정보와 구분)
        if "추정" not in alt and "~로 보인다" not in alt:
            # 짧은 alt라면 OK (핵심만), 긴 설명이면 필수
            pass

        return {
            "passed":         len(issues) == 0,
            "issue_count":    len(issues),
            "issues":         issues,
        }

    # ──────── 통합 ────────
    def evaluate_all(
        self,
        image_path:  str,
        predicted_alt: str,
        reference_alt: Optional[str] = None,
        restoration_json: Optional[dict] = None,
    ) -> Dict:
        report = {
            "alt":        predicted_alt,
            "length":     len(predicted_alt),
            "clip_sim":   self.clip_similarity(image_path, predicted_alt),
            "factuality": self.factuality_check(predicted_alt, restoration_json),
        }
        if reference_alt:
            report["bleu"]   = self.bleu_score([predicted_alt], [reference_alt])
            report["bleurt"] = self.bleurt_score([predicted_alt], [reference_alt])[0]

        # 종합 점수 (0~100)
        score = 0
        score += 30 * min(report["clip_sim"] / 0.30, 1.0)   # CLIP 0.30 이상이면 만점
        score += 30 * (1.0 if report["factuality"]["passed"] else 0.3)
        if reference_alt:
            score += 20 * min(report["bleu"] / 0.20, 1.0)
            score += 20 * min((report["bleurt"] + 1) / 2, 1.0)
        else:
            score += 40   # 참조 없으면 factuality 비중 높임
        report["total_score"] = round(score, 1)
        return report


# ──────────────────────────────────────────────
# 4. 배치 생성 + 저장
# ──────────────────────────────────────────────
def batch_generate(
    image_dir:   str,
    output_json: str,
    reference_csv: Optional[str] = None,
) -> dict:
    generator = AltTextGenerator()
    evaluator = AltQualityEvaluator()

    images = sorted(Path(image_dir).glob("*.jpg"))
    results = []

    refs = {}
    if reference_csv and Path(reference_csv).exists():
        import csv
        with open(reference_csv, encoding="utf-8") as f:
            for row in csv.DictReader(f):
                refs[row["filename"]] = row["alt"]

    from tqdm import tqdm
    for img_path in tqdm(images, desc="alt 생성 · 평가"):
        try:
            alts = generator.generate_both(str(img_path))
            report = evaluator.evaluate_all(
                image_path=str(img_path),
                predicted_alt=alts["alt"],
                reference_alt=refs.get(img_path.name),
            )
            results.append({
                "filename":         img_path.name,
                "alt":              alts["alt"],
                "long_description": alts["long_description"],
                "quality":          report,
            })
        except Exception as e:
            print(f"오류 {img_path.name}: {e}")

    # 저장
    with open(output_json, "w", encoding="utf-8") as f:
        json.dump(results, f, ensure_ascii=False, indent=2)

    # 요약
    scores = [r["quality"]["total_score"] for r in results]
    failed = [r for r in results if not r["quality"]["factuality"]["passed"]]
    print(f"\n=== 요약 ===")
    print(f"평균 점수: {np.mean(scores):.1f}")
    print(f"중앙값:    {np.median(scores):.1f}")
    print(f"사실성 실패: {len(failed)} / {len(results)}")
    print(f"  (이들은 인간 평가 큐로 이관 권장)")
    return {"count": len(results), "avg_score": float(np.mean(scores)),
            "failures": len(failed)}


if __name__ == "__main__":
    result = batch_generate(
        image_dir    = "./output/restored_images",
        output_json  = "./output/alt_texts.json",
        reference_csv= "./reference/alt_ground_truth.csv",    # 선택
    )
```

## 함정 카드

1. **"alt와 longdesc 내용이 같다"** → alt는 핵심 1줄, longdesc는 맥락 포함 상세 설명이어야 한다. 내용이 같으면 longdesc는 무의미. **서로 겹치지 않게 역할 분담** — alt "무엇인가", longdesc "왜·어떻게·어디서".

2. **"VLM이 생성한 alt를 그대로 저장"** → 도메인 프롬프트로도 환각 10~20% 발생. 반드시 `factuality_check`로 규칙 검증 + CLIP 유사도 < 0.20인 샘플은 **인간 검수 큐로**. 자동만 믿으면 환각이 웹에 공개됨.

3. **"평가 지표 1개만 쓴다"** → BLEU만 보면 "단어 맞췄지만 의미는 완전 틀림" 놓침. CLIP만 보면 "이미지와 일반적 유사"는 있어도 세부 사실 놓침. **4지표 앙상블**이 안전.

4. **"참조 alt가 없으면 평가 불가라고 건너뜀"** → 참조 없이도 CLIP 유사도 + factuality check로 **자동 스크리닝** 가능. 참조는 상위 그레이드(gold) 샘플 50~100장만 있어도 임계값 교정에 충분.

5. **"alt-text를 영어로 생성"** → 한국어 사용자 스크린 리더는 **언어 감지 + 한국어 엔진**을 쓴다. 영어 alt가 섞이면 Pitch·속도가 튀어 멀미. 반드시 콘텐츠 언어에 맞춘 alt. 다국어 사이트면 언어별 alt (`alt-ko`, `alt-en` 따로 저장 후 렌더링 시 선택).

## 연습문제

**문제 1.** 생성된 alt가 "조선시대 여성이 한복을 입고 웃는 초상화"인데 실제는 상소문(글자만)이다. 감지 단계별로 무엇이 먼저 잡아야 하는가?

> **해설:** ① **factuality_check**가 "한복", "웃[고는]" 환각 패턴으로 1차 차단 (가장 빠름, 0ms) ② **CLIP similarity**로 이미지-텍스트 유사도 측정 시 0.15 미만 나올 것 (상소문 이미지와 "한복 여성 초상화" 텍스트 미스매치) ③ **BLEU/BLEURT**는 참조가 "상소문"인데 "한복 초상화"면 단어 일치 없어 스코어 극저. 가장 빨리 막는 건 1번 규칙 기반. 그 다음 CLIP. 이 순서로 비용 최소화.

**문제 2.** 같은 이미지에 다른 시점(6개월 후) VLM 재실행하면 alt가 달라진다. 이 변동성을 어떻게 관리하는가?

> **해설:** ① **alt 생성을 한 번만 하고 고정**: 첫 생성 시점 결과를 DB에 저장, 이미지 해시와 쌍으로 보관. 이미지가 바뀌지 않는 한 alt도 바뀌지 않음. ② **모델 버전 추적**: alt 레코드에 `vlm_model: "Qwen2-VL-7B-Instruct@v2.0"`, `generated_at: "2026-04-24"` 메타 저장. 업그레이드 시 **재생성 가치 평가 후 선택적 재생성**. 자동 재생성은 접근성 일관성 깨짐. ③ **사용자 피드백 채널**: "이 alt가 부정확하다" 리포트 받아 수동 교정 후 `alt_manual_override` 필드로 영구 고정. 자동 생성은 초기값, 수정은 기록.

**문제 3.** `long_description` 품질 평가에 BLEU보다 어떤 지표가 더 적합한가?

> **해설:** **BLEURT** 또는 **BERTScore** (문장 의미 기반). 긴 설명은 의역·재배열이 자연스러운데 BLEU는 n-gram 일치만 봐서 의역을 penalize. BLEURT는 BERT 기반 의미 거리라 "단어는 다르지만 같은 뜻"을 인식. 또 다른 선택지: **factuality-focused 평가** — 긴 설명은 여러 문장이니 **각 문장을 claim으로 분해 후 이미지 근거 확인** (NLI 기반 FACTScore). 이는 사실성 강조가 필수인 고문서 도메인에 적합. BLEU는 짧은 alt에만, 긴 설명엔 의미적·사실성 평가 조합.
