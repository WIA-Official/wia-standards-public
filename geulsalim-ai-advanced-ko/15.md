# 챕터 15 · WebXR VR 고문서 관람 · 촉각 피드백

## 학습 목표
- WebXR 표준으로 브라우저에서 VR/AR 몰입 경험을 구축하는 핵심 API를 이해할 수 있다
- Three.js 기반 3D 공간에서 고문서 이미지·수어 아바타·공간 오디오를 배치하는 장면 그래프를 작성할 수 있다
- WebXR Haptic Actuators API로 점자 컨트롤러 촉각 피드백을 전달하고, 고문서 요철을 재현할 수 있다

---

## 이론

### 왜 VR인가 — 단순 웹페이지로는 부족한 것

2D 웹에서 고문서를 본다 = 박물관 유리 너머로 보는 경험. VR에서 **공간감 있게 펼쳐놓고, 수어 아바타와 함께** 보는 것 = 박물관 큐레이터 동반 관람.

특히 접근성 관점:
- **시각장애인**: 3D 공간 오디오로 "왼쪽에 상소문, 오른쪽에 해설자"
- **청각장애인**: 수어 아바타가 바로 옆에 서서 설명, 몰입감 ↑
- **일반 사용자**: 고문서를 360도 돌려보며 뒷면(판권지) 확인

### WebXR 표준

- **WebXR Device API**: 브라우저에서 VR/AR 세션 제어 (`navigator.xr`)
- **Immersive Web Emulator**: 개발 중 VR 디바이스 없이 테스트
- **완전 지원 (2026)**: Chrome, Edge, Meta Quest Browser
- **부분 지원**: Vision Pro Safari — 기본 `immersive-vr` 세션은 가능하나 `hand-tracking` 같은 고급 API는 미지원 또는 실험적 단계
- **미지원**: 일반 Safari (iPhone), Firefox (부분적)

Vision Pro Safari는 2026년 기준 WebXR 커버리지가 완전하지 않다. 특히 `hand-tracking`·`hit-test`를 요구하는 장면은 iPhone/iPad 사용자에게 도달하지 않는다는 점을 설계 초기에 인지해야 한다. 우리 프로젝트는 **핸드 트래킹 필수 기능을 네이티브 Meta Quest Browser 경로에만** 배치하고, Vision Pro는 방문 관람형 시나리오로 제한했다.

```js
// 세션 시작
navigator.xr.requestSession('immersive-vr', {
  requiredFeatures: ['local-floor', 'hand-tracking'],
  optionalFeatures: ['hit-test', 'layers'],
}).then(session => {
  // ...
});
```

### Three.js VR 확장

Three.js는 WebXR를 1줄로 활성화:
```js
renderer.xr.enabled = true;
document.body.appendChild(VRButton.createButton(renderer));
```

이후 렌더 루프는 `renderer.setAnimationLoop` 사용 (기존 requestAnimationFrame 대신). VR 모드에서 양안 뷰 자동 렌더.

### Haptic Actuators API — 촉각 피드백

2026년 Chrome에서 `XRGamepad.hapticActuators` 지원. 강도(0~1), 지속 시간(ms)으로 진동 제어.

```js
gamepad.hapticActuators[0].pulse(0.8, 200);  // 80% 강도, 200ms
```

응용:
- **점자 시뮬레이션**: 6점 패턴을 6번의 short pulse로 (점자 컨트롤러 있으면 실 점자)
- **고문서 종이 요철**: 손이 문서 가장자리 통과 시 살짝 진동
- **수어 강조**: 중요 단어 수어 시점에 미세 피드백

### 공간 오디오 (Spatial Audio)

WebAudio API + HRTF(Head-Related Transfer Function) 필터:
```js
const panner = audioCtx.createPanner();
panner.panningModel = 'HRTF';
panner.positionX.value = -2;  // 왼쪽 2m
panner.positionY.value = 1.6; // 머리 높이
panner.positionZ.value = -1;  // 앞 1m
```

시각장애인 사용자 VR 경험에 필수. 소리로 공간 구조 파악.

# WebXR 경험의 접근성 지원을 위한 서버 측 메타데이터 생성
# pip install pydantic

from dataclasses import dataclass
from typing import List, Dict


@dataclass
class VRSceneMeta:
    """
    VR 씬의 접근성 메타.
    스크린 리더나 AT 기기가 공간 구조를 이해할 수 있도록.
    """
    objects: List[dict]       # [{"name": "document", "type": "artifact", "position": [x,y,z]}]
    narration_tracks: List[dict]    # [{"lang": "ko", "url": "...", "duration": 120}]
    sign_language_tracks: Dict[str, str]    # {"KSL": "...", "ASL": "..."}
    haptic_patterns: List[dict]     # 점자 변환 결과 등

    def to_json(self) -> str:
        import json
        return json.dumps({
            "scene": self.objects,
            "audio": self.narration_tracks,
            "sign":  self.sign_language_tracks,
            "haptic": self.haptic_patterns,
        }, ensure_ascii=False, indent=2)


def build_vr_scene_meta(restoration_json_path: str, output_path: str) -> VRSceneMeta:
    import json
    with open(restoration_json_path, encoding="utf-8") as f:
        rest = json.load(f)

    meta = VRSceneMeta(
        objects=[
            {"name": "document_main",
             "type": "artifact",
             "position": [0, 1.6, -1.5],
             "size": [1.5, 2.2],
             "title": "조선시대 상소문",
             "image_url": rest["image_path"],
             "alt_text": "AI로 복원된 조선 후기 상소문"},
            {"name": "narrator_source",
             "type": "audio_source",
             "position": [-1.5, 1.6, -1.2],
             "label": "내레이션 시작점"},
            {"name": "sign_avatar",
             "type": "sign_language_avatar",
             "position": [1.5, 0, -1.2],
             "language": "KSL",
             "label": "3D 수어 해설자"},
        ],
        narration_tracks=[
            {"lang": "ko",
             "url": "/assets/narration_ko.mp3",
             "duration_sec": 120,
             "transcript_url": "/assets/narration_ko.vtt"},
            {"lang": "en",
             "url": "/assets/narration_en.mp3",
             "duration_sec": 115,
             "transcript_url": "/assets/narration_en.vtt"},
        ],
        sign_language_tracks={
            "KSL": "/assets/sign_ksl_clip.json",
            "ASL": "/assets/sign_asl_clip.json",
        },
        haptic_patterns=[
            {"trigger": "document_focus",

            {"trigger": "braille_cell_0xAC00",    # "가"
             "pattern": [
                {"strength": 0.8, "duration_ms": 80},

                {"strength": 0.8, "duration_ms": 80},
             ]},
        ],
    )

    with open(output_path, "w", encoding="utf-8") as f:
        f.write(meta.to_json())
    return meta


if __name__ == "__main__":

## 함정 카드

1. **"VR 세션에서 UI를 2D HTML로"** → VR 모드 진입 시 HTML overlay는 안 보인다. 중요 정보는 **3D 공간 내 텍스트 메시**(TextGeometry 또는 HTML rendering to texture)로. `WebXR DOM Overlay`는 일부 브라우저만 지원.

2. **"Haptic feedback을 연속 발생"** → 사용자 손이 계속 진동하면 피로·불쾌. 중요 순간만 100~200ms 펄스. 연속 패턴은 점자 시뮬레이션 같은 명시적 용도만.

3. **"공간 오디오 설정을 HRTF 없이"** → 기본 `panningModel: 'equalpower'`는 좌우만. **`'HRTF'`** 로 설정해야 위·아래·앞·뒤도 식별 가능. 시각장애인 공간 인식 필수.

4. **"수어 아바타를 너무 멀리 배치"** → 2m 이상 멀면 손·손가락 디테일 안 보임. **1~1.5m 거리**가 수어 가독성 최적. 대신 문서는 더 멀리 배치해 "관람 거리" 확보.

5. **"VR을 비지원 브라우저에서 조용히 실패"** → iPhone Safari 등 WebXR 미지원 기기는 VRButton 자체 비활성. 반드시 **fallback UI** — 2D 모드에서도 같은 콘텐츠 접근 가능해야 함. `renderer.xr.isPresenting` false일 때의 조작법 별도 구현.

## 연습문제

**문제 1.** Meta Quest 3 사용자가 VR 모드 진입 후 10분 만에 "멀미"라 호소한다. 원인 후보와 해결책은?

> **해설:** **원인**: ① **낮은 프레임 레이트** — Quest 3는 90Hz 타깃. 45 이하 떨어지면 멀미. ② **카메라 강제 이동** — 유저 의사 없이 scene을 움직임. 수어 아바타가 카메라 주변 이동이면 멀미 ③ **렌더 지연** — 머리 움직임 후 화면 갱신 지연. ④ **긴 세션**. **해결**: 1) `renderer.setPixelRatio(1.0)` (초고해상도 지양) 2) 오브젝트 수 < 500, 메시 폴리곤 < 50만 3) 움직임은 항상 유저 주도 (teleport는 fade in/out 0.2초) 4) 5분마다 "휴식 알림" 표시. 접근성 설정에 "vignette during movement" 옵션 추가.

**문제 2.** 시각장애인 사용자가 VR을 "볼" 수 없다. 그래도 VR 경험이 가치 있는가?

> **해설:** **있다**. VR의 본질은 시각이 아니라 **공간감**이다. 시각장애인 사용자에겐 ① **공간 오디오**로 "왼쪽 2m에 문서, 오른쪽 1m에 해설자" 파악 ② **햅틱**으로 "이 방향에 벽, 이 방향에 오브젝트" ③ **수어 아바타**는 안 보여도 내레이션은 공간감 있게 들림. 실제로 American Foundation for the Blind는 **audio-first VR**이 시각장애인 몰입 경험에 탁월하다고 발표. 일반 웹의 평면 오디오 대비 "있는 곳에 있다"는 현실감 차이가 크다.

**문제 3.** WebXR 미지원 기기(iPhone Safari 등)를 위한 fallback 전략은?

> **해설:** ① **2D 탐색 모드**: Three.js 그대로 사용, `OrbitControls`로 마우스/터치 드래그 탐색. 카메라 앵글 사용자 제어 ② **360도 비디오**: WebXR 씬을 사전에 360도 렌더링 후 mp4/webm으로 저장, `<video>` + WebGL texture. `A-Frame` 프레임워크가 쉽게 처리 ③ **사진 슬라이드쇼 + 오디오**: 가장 단순, VR 씬의 주요 시점을 사진으로 캡처 후 내레이션과 동기화. 접근성 관점에서는 **3번이 최고** — 스크린 리더 호환, 저대역 네트워크 OK. VR은 "증강", 2D가 "기본".

## 다음 장 예고
