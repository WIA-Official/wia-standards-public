# 챕터 03 · 복원 진위 해시 체인 — "복원 ≠ 조작" 증명

## 학습 목표
- 복원 AI가 만든 결과물이 "원본에 추가된 예측"인지 "원본의 위조"인지 암호학적으로 구분할 수 있는 증명서를 설계할 수 있다
- SHA-256 해시 체인 + 메타데이터 JSON으로 복원 이력을 위변조 불가하게 기록할 수 있다
- WIA Chain(자체 PoA) 또는 공공 BSC에 해시 앵커링해 제3자 검증 가능한 구조를 구축할 수 있다

---

## 이론

### 왜 해시 증명이 필수인가

2025년 10월, 어느 지방 박물관 웹사이트에서 "AI가 복원한 조선시대 호적부"가 공개됐다. 그런데 얼마 뒤 역사학자 한 분이 복원된 이름 중 하나가 자기 가문 족보에 없는 인물이라고 지적했다. 박물관 측은 "AI가 예측한 부분"이라 답했지만, **이미 여러 논문에 인용된 뒤**였다.

이 사건이 내게 준 교훈: **AI 복원은 "생성"이다. 생성물을 원본과 섞어 공개하는 순간 학술적 오염이 시작된다.** 해결책은 기술이 아니라 **투명성**이다.

### 증명서가 담아야 할 것

```
1. 원본 이미지 SHA-256         ← "처음 받은 건 이거"
2. 복원 이미지 SHA-256         ← "이렇게 복원했어"
3. 마스크 이미지 SHA-256       ← "어디를 AI가 채웠어"
4. 복원 텍스트 JSON            ← "이 글자들은 AI 예측"
5. 모델 버전 + LoRA 해시       ← "어떤 모델이 했어"
6. 타임스탬프 + 서명자         ← "언제 · 누가"
7. Merkle root                 ← "1~6 묶어 한 해시로"
8. 공공 체인 트랜잭션 ID        ← "제3자가 검증 가능"
```

### Merkle Tree 구조

```
                    ROOT (전체 증명 해시)
                   /                    \
              LEFT                       RIGHT
             /    \                    /         \
      H(원본)  H(복원)            H(마스크)  H(메타JSON)
```

개별 파일 중 하나만 바뀌어도 ROOT 해시가 변한다 — **위변조 감지 자동**.

### 공공 앵커링 전략

자체 DB에만 해시를 저장하면 서버 관리자가 "데이터 수정 + 해시도 수정"을 할 수 있다(자가 위조). 반드시 **제3자가 들여다볼 수 있는 공개 장소**에 앵커링.

2026년 실무 선택지:
- **WIA Chain** (형님 자체 PoA 체인): 가스 없음, 완전 통제, 신뢰는 WIA 운영자 한정
- **Binance Smart Chain (BSC)**: 트랜잭션 수수료 ≈ $0.02, 글로벌 공개 검증 가능
- **Polygon**: BSC보다 빠름, 수수료 ≈ $0.001
- **IPFS** (무체인): 블록체인 아니지만 내용 주소 기반 분산 저장, 공공 검증 가능

**권장 전략**: WIA Chain에 원본 앵커링 + 분기별 1회 BSC·Polygon 배치 앵커링. 일상은 WIA Chain, 학술 공개 시점엔 공공 체인.

```python
# 복원 진위 해시 체인 + 증명서 생성
# pip install hashlib pydantic web3==6.17 requests

import hashlib
import json
import uuid
import time
from dataclasses import dataclass, field, asdict
from datetime import datetime, timezone
from pathlib import Path
from typing import List, Dict, Optional


# ──────────────────────────────────────────────
# 1. 해시 유틸
# ──────────────────────────────────────────────
def sha256_file(path: str) -> str:
    """파일 SHA-256 (증명서용)"""
    h = hashlib.sha256()
    with open(path, "rb") as f:
        for chunk in iter(lambda: f.read(8192), b""):
            h.update(chunk)
    return h.hexdigest()


def sha256_json(obj: dict) -> str:
    """JSON 정규화(키 정렬) 후 SHA-256 — 같은 내용이면 항상 같은 해시"""
    canonical = json.dumps(obj, sort_keys=True, ensure_ascii=False, separators=(",", ":"))
    return hashlib.sha256(canonical.encode("utf-8")).hexdigest()


def sha256_concat(*hashes: str) -> str:
    """여러 해시를 붙여 다시 해시 (Merkle node)"""
    concatenated = "".join(hashes)
    return hashlib.sha256(concatenated.encode("utf-8")).hexdigest()


# ──────────────────────────────────────────────
# 2. Merkle Tree 빌더
# ──────────────────────────────────────────────
class MerkleTree:
    """
    간단한 Merkle Tree (leaves 짝수 아니면 마지막 복제).
    """
    def __init__(self, leaves: List[str]):
        if not leaves:
            raise ValueError("leaves 비어있음")
        self.leaves = leaves[:]
        self.levels = self._build(leaves)

    @staticmethod
    def _build(level: List[str]) -> List[List[str]]:
        levels = [level]
        while len(level) > 1:
            if len(level) % 2 == 1:
                level.append(level[-1])      # 마지막 복제
            next_level = [
                sha256_concat(level[i], level[i + 1])
                for i in range(0, len(level), 2)
            ]
            levels.append(next_level)
            level = next_level
        return levels

    @property
    def root(self) -> str:
        return self.levels[-1][0]

    def proof(self, leaf_index: int) -> List[Dict]:
        """특정 leaf의 Merkle proof 경로"""
        proof = []
        idx = leaf_index
        for level in self.levels[:-1]:
            sibling_idx = idx + 1 if idx % 2 == 0 else idx - 1
            if sibling_idx < len(level):
                proof.append({
                    "hash": level[sibling_idx],
                    "side": "right" if idx % 2 == 0 else "left",
                })
            idx //= 2
        return proof

    @staticmethod
    def verify(leaf: str, proof: List[Dict], root: str) -> bool:
        """leaf + proof 경로로 root 재계산"""
        current = leaf
        for step in proof:
            current = (
                sha256_concat(current, step["hash"])
                if step["side"] == "right"
                else sha256_concat(step["hash"], current)
            )
        return current == root


# ──────────────────────────────────────────────
# 3. 복원 증명서 빌더
# ──────────────────────────────────────────────
@dataclass
class RestorationCertificate:
    """W3C Verifiable Credentials 스타일 증명서"""
    context:        str = "https://wiageulsalim.com/schema/restoration/v1"
    id:             str = field(default_factory=lambda: f"urn:wia:restoration:{uuid.uuid4()}")
    type:           str = "AIRestoration"
    issued:         str = field(default_factory=lambda: datetime.now(timezone.utc).isoformat())
    issuer:         str = "did:wia:geulsalim"      # 발행자 DID
    subject:        dict = field(default_factory=dict)
    artifacts:      dict = field(default_factory=dict)
    model_info:     dict = field(default_factory=dict)
    predictions:    dict = field(default_factory=dict)
    proof:          dict = field(default_factory=dict)


class CertificateBuilder:
    """
    복원 결과물에서 증명서 자동 생성.
    """
    def __init__(self, wia_chain_endpoint: Optional[str] = None):
        self.wia_endpoint = wia_chain_endpoint or "https://wia.money/wia-chain/api/v1"

    def build(
        self,
        original_image_path:  str,
        restored_image_path:  str,
        mask_image_path:      str,
        restoration_json:     dict,
        lora_weights_path:    str,
        document_title:       str,
        issuer_did:           str = "did:wia:geulsalim",
    ) -> RestorationCertificate:
        # (1) 각 artifact 해시
        h_original = sha256_file(original_image_path)
        h_restored = sha256_file(restored_image_path)
        h_mask     = sha256_file(mask_image_path)
        h_meta     = sha256_json(restoration_json)
        h_model    = sha256_file(lora_weights_path)

        # (2) Merkle Tree
        leaves = [h_original, h_restored, h_mask, h_meta, h_model]
        tree = MerkleTree(leaves)

        cert = RestorationCertificate(
            issuer=issuer_did,
            subject={
                "originalDocument": document_title,
                "originalHash":     f"sha256:{h_original}",
                "restoredHash":     f"sha256:{h_restored}",
                "maskHash":         f"sha256:{h_mask}",
            },
            artifacts={
                "original":  {"path": original_image_path, "hash": h_original},
                "restored":  {"path": restored_image_path, "hash": h_restored},
                "mask":      {"path": mask_image_path,      "hash": h_mask},
            },
            model_info={
                "base_model":  "diffusers/stable-diffusion-xl-1.0-inpainting-0.1",
                "lora_hash":   f"sha256:{h_model}",
                "lora_path":   lora_weights_path,
                "llm_model":   restoration_json.get("llm_model", "solar-10.7b-lora"),
                "vlm_model":   restoration_json.get("vlm_model", "Qwen/Qwen2-VL-7B-Instruct"),
            },
            predictions={
                "masked_text":   restoration_json.get("masked_text", ""),
                "final_text":    restoration_json.get("final", []),
                "crosscheck_case": restoration_json.get("case", ""),
                "confidence":    restoration_json.get("confidence", 0),
                "expert_review": restoration_json.get("expert_review_required", True),
            },
            proof={
                "type":        "Sha256ChainAnchor2026",
                "merkleRoot":  f"sha256:{tree.root}",
                "leaves":      [f"sha256:{h}" for h in leaves],
                "anchorChain": None,          # 앵커링 후 채움
                "anchorTx":    None,
                "verifyUrl":   None,
            },
        )
        return cert

    def anchor_to_wia_chain(self, cert: RestorationCertificate) -> RestorationCertificate:
        """
        WIA Chain에 Merkle root 트랜잭션 기록.
        실제 BSC/Polygon도 유사한 패턴.
        """
        import requests
        root = cert.proof["merkleRoot"].replace("sha256:", "")
        response = requests.post(
            f"{self.wia_endpoint}/anchor",
            json={
                "type":       "restoration",
                "merkleRoot": root,
                "issuer":     cert.issuer,
                "issuedAt":   cert.issued,
            },
            timeout=30,
        )
        response.raise_for_status()
        data = response.json()

        cert.proof["anchorChain"] = "WIA"
        cert.proof["anchorTx"]    = data["txHash"]
        cert.proof["verifyUrl"]   = f"https://wia.money/wia-chain/tx/{data['txHash']}"
        cert.proof["anchoredAt"]  = data["timestamp"]
        return cert

    def anchor_to_bsc(self, cert: RestorationCertificate, private_key: str) -> RestorationCertificate:
        """
        Binance Smart Chain 공공 앵커링.
        OP_RETURN 유사 — 데이터만 실리고 수신자 0x0.
        """
        from web3 import Web3
        w3 = Web3(Web3.HTTPProvider("https://bsc-dataseed.binance.org/"))
        acct = w3.eth.account.from_key(private_key)

        root_bytes = bytes.fromhex(cert.proof["merkleRoot"].replace("sha256:", ""))
        # 32 bytes data를 tx.data에 실음 (0x0 수신자)
        tx = {
            "from":      acct.address,
            "to":        "0x0000000000000000000000000000000000000000",
            "value":     0,
            "data":      "0x" + root_bytes.hex(),
            "gas":       25000,
            "gasPrice":  w3.eth.gas_price,
            "nonce":     w3.eth.get_transaction_count(acct.address),
            "chainId":   56,
        }
        signed = acct.sign_transaction(tx)
        tx_hash = w3.eth.send_raw_transaction(signed.rawTransaction).hex()

        cert.proof["anchorChainBSC"] = "BSC"
        cert.proof["anchorTxBSC"]    = tx_hash
        cert.proof["verifyUrlBSC"]   = f"https://bscscan.com/tx/{tx_hash}"
        return cert


# ──────────────────────────────────────────────
# 4. 증명서 검증기 (공개)
# ──────────────────────────────────────────────
class CertificateVerifier:
    """
    제3자가 증명서를 검증할 때 쓰는 모듈.
    """
    @staticmethod
    def verify_locally(
        cert: dict,
        artifact_paths: Dict[str, str],    # {"original": "/path", "restored": "/path", ...}
    ) -> dict:
        """로컬에서 파일 해시 재계산 + Merkle root 일치 확인"""
        results = {"status": "ok", "checks": [], "failures": []}

        # (1) 각 artifact 해시 재계산
        for key, path in artifact_paths.items():
            computed = sha256_file(path)
            claimed = cert["artifacts"].get(key, {}).get("hash", "")
            if computed != claimed:
                results["status"] = "FAIL"
                results["failures"].append(f"{key} 해시 불일치: {computed[:16]}... vs {claimed[:16]}...")
            else:
                results["checks"].append(f"{key} 해시 일치")

        # (2) Merkle root 재계산
        leaves = [f"sha256:{sha256_file(p)}" for p in artifact_paths.values()]
        # 필요 시 meta, model 해시도 포함 (cert에서 가져옴)
        # Tree 재구성 후 root 비교
        # (실무: cert.proof.leaves 그대로 쓰고 root만 재계산)
        local_root = MerkleTree(
            [h.replace("sha256:", "") for h in cert["proof"]["leaves"]]
        ).root
        claimed_root = cert["proof"]["merkleRoot"].replace("sha256:", "")
        if local_root != claimed_root:
            results["status"] = "FAIL"
            results["failures"].append(f"Merkle root 불일치: {local_root[:16]}... vs {claimed_root[:16]}...")
        else:
            results["checks"].append("Merkle root 일치")

        return results

    @staticmethod
    def verify_chain_anchor(cert: dict) -> dict:
        """WIA Chain 또는 BSC에서 앵커링 실존 확인"""
        import requests
        tx_url = cert["proof"].get("verifyUrl", "")
        if not tx_url:
            return {"status": "SKIP", "reason": "앵커링 없음"}

        # 간소화: 실제로는 체인 RPC 호출
        try:
            r = requests.get(tx_url + ".json", timeout=15)
            tx_data = r.json()
            anchored_root = tx_data.get("data", "").replace("0x", "")
            expected = cert["proof"]["merkleRoot"].replace("sha256:", "")
            if anchored_root == expected:
                return {"status": "ok", "chain": cert["proof"]["anchorChain"]}
            return {"status": "FAIL", "reason": "앵커된 해시와 증명서 해시 불일치"}
        except Exception as e:
            return {"status": "ERROR", "reason": str(e)}


# ──────────────────────────────────────────────
# 5. 파일 저장 + 서명
# ──────────────────────────────────────────────
def save_certificate(cert: RestorationCertificate, output_path: str) -> None:
    """증명서 JSON 저장 (정규화 형식)"""
    with open(output_path, "w", encoding="utf-8") as f:
        json.dump(asdict(cert), f, ensure_ascii=False, indent=2, sort_keys=True)
    print(f">> 증명서: {output_path}")


if __name__ == "__main__":
    builder = CertificateBuilder()
    cert = builder.build(
        original_image_path="./samples/sangso_damaged.jpg",
        restored_image_path="./output/sangso_01/restored_image.jpg",
        mask_image_path    ="./output/sangso_01/restored_image.mask.png",
        restoration_json   = json.load(open("./output/sangso_01/restoration.json")),
        lora_weights_path  ="./models/sdxl_inpaint_korean.safetensors",
        document_title     ="국립중앙박물관 상소문 샘플 #01",
    )
    # WIA Chain 앵커링
    cert = builder.anchor_to_wia_chain(cert)
    save_certificate(cert, "./output/sangso_01/certificate.json")

    # 제3자 검증 시나리오
    verify_result = CertificateVerifier.verify_locally(
        asdict(cert),
        artifact_paths={
            "original": "./samples/sangso_damaged.jpg",
            "restored": "./output/sangso_01/restored_image.jpg",
            "mask":     "./output/sangso_01/restored_image.mask.png",
        },
    )
    print(f"로컬 검증: {verify_result['status']}")
    print(f"  ✓ {len(verify_result['checks'])} 체크 통과")
    if verify_result["failures"]:
        for f in verify_result["failures"]:
            print(f"  ✗ {f}")
```

## 함정 카드

1. **"해시만 저장하고 원본은 지운다"** → 원본 파일 없이 해시만 있으면 "해시의 원본이 뭐였는지" 증명 불가. **원본·복원·마스크 3종 모두 보관** 필수. IPFS·S3 아카이브 + 여러 복제본.

2. **"JSON 직렬화 방식을 표준화 안 한다"** → `json.dumps` 기본값은 키 순서 무작위 → 같은 내용도 다른 해시. **`sort_keys=True, separators=(",", ":"), ensure_ascii=False`** 고정 규칙. 표준 JCS (RFC 8785) 준수가 이상적.

3. **"BSC 앵커링 수수료가 올라 앵커링 건너뛴다"** → BSC 수수료는 $0.01~0.05. 연 1,000 건이면 $50. 무시할 수 있는 비용. **앵커링 안 하면 자가 위조 방어 불가**. 수수료 변동 있어도 반드시 유지.

4. **"복원 LoRA 가중치 해시를 생략한다"** → "어떤 모델이 어떻게 복원했는지"도 증명 일부. 가중치 해시 없으면 모델을 슬쩍 업데이트해도 증명서 통과. 반드시 `lora_hash` 포함 + 모델 업데이트 시 새 증명서 발행.

5. **"전문가 검토 필요(case 4)인데 증명서 발급"** → 자동 복원 실패를 공식 기록으로 남기면 신뢰 훼손. case 4는 **증명서 발급 전 보류** 상태로 표시하고, 전문가 검증 완료 후 `proof.expertReview: {verifiedBy, date, comments}` 추가 후 발급.

## 연습문제

**문제 1.** 누군가가 증명서의 `restoredHash` 필드만 수정해 다른 복원본을 "같은 증명서"인 것처럼 속이려 한다. 왜 불가능한가?

> **해설:** `restoredHash`를 바꾸면 Merkle Tree의 leaves가 바뀌고 **root가 재계산됐을 때 달라진다**. 공격자는 root도 바꿔야 하지만, root는 이미 WIA Chain/BSC에 **불변으로 기록**돼 있다. 블록체인 기록을 바꾸려면 체인 전체 51% 공격 필요 — BSC는 Binance의 21명 검증자 네트워크로 현실적 불가. 따라서 공격자는 "가짜 증명서 새로 발행" 정도만 가능하나, 그건 타임스탬프·앵커 TX가 다르므로 별개 증명서임.

**문제 2.** WIA Chain과 BSC 둘 다에 앵커링하는 이유는? 하나만 쓰면 안 되는가?

> **해설:** **신뢰의 다층화**. WIA Chain은 자체 운영이라 "WIA가 망하거나 악의적 운영 시" 증명이 사라질 위험. BSC는 글로벌 분산이라 중단·위조 가능성 극소하지만, 모든 건마다 앵커링하면 비용·지연 문제. **실무 조합**: 일상 증명은 WIA Chain (즉시·무료), 분기 1회 또는 주요 공개 시점엔 BSC 배치 앵커링 (글로벌 신뢰). 2중 앵커링으로 운영 리스크 + 비용 균형.

**문제 3.** 증명서를 수년 후(예: 2032년) 검증하려 한다. 그때 어떤 장애가 있을 수 있고 어떻게 대비하는가?

> **해설:** **장애 후보:** ① WIA Chain 운영 중단 → BSC 앵커링이 백업 ② BSC RPC 엔드포인트 URL 변경 → `anchorTx` 해시로 전체 BSC 아카이브(bscscan·Etherscan 같은 explorer)에서 검색 가능 ③ SHA-256이 양자컴퓨터로 깨짐(2030+ 예측) → **알고리즘 마이그레이션**: 2028년 이후부터 SHA-3-256 또는 BLAKE3 병기. 증명서 스키마 `v1 → v2`에서 해시 알고리즘 필드 명시. ④ 원본 파일 유실 → IPFS CID 저장으로 분산 보관. **핵심 원칙**: 증명서 스키마 자체에 `version`과 `algorithm`을 명시해 미래 진화 대비.
