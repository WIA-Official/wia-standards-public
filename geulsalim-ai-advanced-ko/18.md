## 이론

### 정적 DAG의 한계 — 예상 못한 입력

챕터 17의 Prefect DAG는 "정의된 경로"만 실행한다. 하지만 실전에서 겪는 상황:

1. **원문이 산스크리트어** — 한국어 LLM·수어 시스템 기본 경로 부적합
2. **복원 이미지 LPIPS 점수가 경계** — "한 번 더 다른 seed로 재생성하면 좋아질 것 같은데"
3. **OCR 신뢰도가 전체 낮음** — Inpainting 전에 손상 영역을 사람이 검토해야 함
4. **수어 클립 누락 70%** — 이 문서는 텍스트 전달만 시도하는 게 현실적

각 상황마다 분기하는 DAG를 모두 짜두면 DAG 폭발. **Agentic 접근**: LLM이 중간 결과를 보고 **다음 단계를 판단**.

### LangGraph 기본 개념

**Graph**: 노드(Node) = 실행 단위, 엣지(Edge) = 전이 조건. 상태(State)를 공유.

```python
graph = StateGraph(PipelineState)
graph.add_node("inpaint", inpaint_fn)
graph.add_node("evaluate", evaluate_fn)
graph.add_node("retry", retry_fn)
graph.add_edge("inpaint", "evaluate")
graph.add_conditional_edges(
    "evaluate",
    lambda s: "retry" if s["score"] < 0.7 else "done",
)
```

특징:
- **Durable**: 체크포인트로 중단·재개
- **Human-in-the-loop**: 특정 노드에서 사용자 입력 대기
- **Observable**: LangSmith로 모든 전이 로그
- **Model-agnostic**: Claude, GPT, Gemini 교체 가능

### Claude Agent SDK와 결합

2026년 표준 조합: **LangGraph (workflow 뼈대) + Claude Agent SDK (노드 내 heavy lifting)**.

```
LangGraph:    전체 흐름 · 상태 관리 · 조건부 분기 · 재시도
Claude SDK:   개별 노드 안에서 도구 호출(search, OCR, 이미지 분석)
```

### Self-correction 루프

실패 → 원인 → 수정 → 재시도. 단순 "retry 3번"이 아닌 **의미 기반 수정**:

```
Step 1: Inpainting 결과 CER 25% (목표 8% 이하)
Step 2: Agent 판단 — "마스크가 너무 넓음, 40% 영역"
Step 3: 다음 액션 — "마스크 비율 30%로 제한해 재실행"
Step 4: 재실행 → CER 15%
Step 5: Agent — "여전히 임계 초과, 원인: LoRA 학습 부족"
Step 6: 다음 액션 — "대체 LoRA(historical_v2.safetensors) 시도"
Step 7: 재실행 → CER 6%
Step 8: 목표 달성, 종료
```

이 루프는 사람이 설계한 if-else로는 포괄 불가. LLM이 상황을 읽고 **열린 문제 해결**.

## 실전 코드

```python
# LangGraph + Claude 기반 자가 수정 복원 파이프라인
# pip install langgraph==0.2 langchain-anthropic==0.2

from langgraph.graph import StateGraph, END
from langgraph.checkpoint.memory import MemorySaver
from langchain_core.messages import HumanMessage, AIMessage
from langchain_anthropic import ChatAnthropic
from typing import TypedDict, List, Optional, Annotated
from dataclasses import asdict
import json
import operator


# ──────────────────────────────────────────────
# 1. 파이프라인 상태
# ──────────────────────────────────────────────
class PipelineState(TypedDict):
    # 입력
    input_image:         str
    document_title:      str
    output_dir:          str

    # 중간 상태
    restoration_result:  Optional[dict]
    current_attempt:     int
    max_attempts:        int
    quality_scores:      Annotated[List[dict], operator.add]

    # 의사결정
    next_action:         str
    reasoning:           str

    # 최종
    final_artifacts:     Optional[dict]
    success:             bool


# ──────────────────────────────────────────────
# 2. Agent LLM 래퍼
# ──────────────────────────────────────────────
llm = ChatAnthropic(
    model="claude-opus-4-7",     # 2026-04 기준
    max_tokens=2048,
    temperature=0.2,
)


def agent_analyze(state: PipelineState, context: str, options: List[str]) -> dict:
    """
    Agent에 현재 상황 + 선택지 제시, 최선 판단 요청.
    """
    prompt = f"""당신은 AI 고문서 복원 파이프라인의 오케스트레이터입니다.

현재 상황:
{context}

시도 횟수: {state['current_attempt']}/{state['max_attempts']}
이전 품질 기록: {json.dumps(state.get('quality_scores', []), ensure_ascii=False)}

선택지:
{chr(10).join(f'{i+1}. {o}' for i, o in enumerate(options))}

답변 형식 (JSON):
{{"action": "선택한 액션 명", "reasoning": "왜 이것인지 한국어 2문장", "parameters": {{필요한 경우 추가 파라미터}}}}
"""
    response = llm.invoke([HumanMessage(content=prompt)])
    import re
    match = re.search(r"\{.*\}", response.content, re.DOTALL)
    if match:
        try:
            return json.loads(match.group(0))
        except json.JSONDecodeError:
            pass
    return {"action": options[0], "reasoning": "파싱 실패, 첫 선택지 기본값"}


# ──────────────────────────────────────────────
# 3. 노드 구현
# ──────────────────────────────────────────────
def node_initial_analysis(state: PipelineState) -> dict:
    """입력 이미지 특성 분석"""
    # 이미지 해상도, 손상 정도, 언어 추정
    from PIL import Image
    img = Image.open(state["input_image"])
    w, h = img.size

    # (실무: VLM로 문서 유형·언어 판단)
    analysis = {
        "width":  w, "height": h,
        "estimated_lang": "ko-Hani",      # 임시
        "estimated_type": "상소문",
    }

    return {
        "reasoning": f"초기 분석: {w}×{h}, 추정 유형 {analysis['estimated_type']}",
        "next_action": "inpaint",
    }


def node_inpaint(state: PipelineState) -> dict:
    """이미지 복원 + LLM 예측"""
    from advanced_ch02 import AdvancedHybridRestorer
    restorer = AdvancedHybridRestorer(
        inpaint_lora_path="./models/sdxl_inpaint_korean.safetensors",
        text_lora_path="./models/solar_hanmun_lora",
    )

    # 이전 시도에서 힌트 활용
    mask_ratio_limit = 0.35 if state["current_attempt"] == 1 else 0.25
    result = restorer.restore(state["input_image"], state["output_dir"])

    return {
        "restoration_result": result,
        "current_attempt": state["current_attempt"] + 1,
    }


def node_evaluate_quality(state: PipelineState) -> dict:
    """자동 메트릭 + Agent 종합 평가"""
    result = state["restoration_result"]
    confidence = result.get("confidence", 0)
    case = result.get("case", "unknown")

    score_record = {
        "attempt":    state["current_attempt"],
        "confidence": confidence,
        "case":       case,
        "cer":        result.get("cer", None),
    }

    # Agent 판단
    context = f"""복원 결과:
- CrossCheck case: {case}
- 신뢰도: {confidence:.2f}
- 전문가 검토 필요: {result.get('expert_review_required', False)}
- 최종 글자: {result.get('final', [])}"""

    options = [
        "done — 품질 충분, 다음 단계로",
        "retry_different_seed — 같은 파이프라인 다른 seed로 재시도",
        "retry_different_lora — 다른 LoRA 모델 사용",
        "retry_reduce_mask — 마스크 영역 축소",
        "human_review — 전문가 검토 큐로 이관",
        "skip_phase2_sign — Phase 4 수어만 제외하고 진행",
    ]

    decision = agent_analyze(state, context, options)

    return {
        "quality_scores":  [score_record],
        "next_action":     decision["action"].split(" — ")[0],
        "reasoning":       decision["reasoning"],
    }


def node_retry_different_seed(state: PipelineState) -> dict:
    """다른 seed로 재시도"""
    import random
    new_seed = random.randint(0, 99999)
    # 재시도 시 seed 주입은 파이프라인 인자로
    print(f">> 재시도 seed={new_seed}")
    return {"next_action": "inpaint"}


def node_retry_different_lora(state: PipelineState) -> dict:
    """대체 LoRA 시도"""
    print(">> 대체 LoRA 로드: historical_v2")
    # config 업데이트
    return {"next_action": "inpaint"}


def node_human_review(state: PipelineState) -> dict:
    """전문가 검토 대기열"""
    # DB에 작업 등록 + Slack 알림
    review_id = f"review_{state['document_title']}_{state['current_attempt']}"
    print(f">> 검토 대기열: {review_id}")
    return {
        "success": False,
        "next_action": "done",
        "final_artifacts": {"status": "expert_review_pending", "review_id": review_id},
    }


def node_finalize(state: PipelineState) -> dict:
    """최종 4 Phase 실행 (챕터 17)"""
    from ch17_orchestration import process_document
    result = process_document(
        image_path=state["input_image"],
        output_dir=state["output_dir"],
        document_title=state["document_title"],
    )
    return {
        "final_artifacts": result,
        "success": True,
    }


# ──────────────────────────────────────────────
# 4. Graph 구성
# ──────────────────────────────────────────────
def build_graph() -> StateGraph:
    graph = StateGraph(PipelineState)

    graph.add_node("analyze",      node_initial_analysis)
    graph.add_node("inpaint",      node_inpaint)
    graph.add_node("evaluate",     node_evaluate_quality)
    graph.add_node("retry_seed",   node_retry_different_seed)
    graph.add_node("retry_lora",   node_retry_different_lora)
    graph.add_node("human_review", node_human_review)
    graph.add_node("finalize",     node_finalize)

    graph.set_entry_point("analyze")
    graph.add_edge("analyze", "inpaint")
    graph.add_edge("inpaint", "evaluate")
    graph.add_edge("retry_seed", "inpaint")
    graph.add_edge("retry_lora", "inpaint")
    graph.add_edge("finalize",    END)
    graph.add_edge("human_review", END)

    # evaluate 후 조건부 분기
    def route_after_eval(state: PipelineState) -> str:
        action = state.get("next_action", "done")
        if state["current_attempt"] >= state["max_attempts"]:
            return "human_review"
        if action == "done":
            return "finalize"
        if action == "retry_different_seed":
            return "retry_seed"
        if action == "retry_different_lora":
            return "retry_lora"
        if action == "retry_reduce_mask":
            return "retry_seed"     # 파라미터 조정 후 동일 흐름
        if action == "human_review":
            return "human_review"
        if action == "skip_phase2_sign":
            return "finalize"
        return "human_review"

    graph.add_conditional_edges("evaluate", route_after_eval)

    return graph.compile(checkpointer=MemorySaver())


# ──────────────────────────────────────────────
# 5. 실행
# ──────────────────────────────────────────────
def run_agentic_pipeline(image_path: str, output_dir: str, title: str):
    app = build_graph()

    config = {"configurable": {"thread_id": f"session_{title}"}}
    initial_state: PipelineState = {
        "input_image":      image_path,
        "document_title":   title,
        "output_dir":       output_dir,
        "restoration_result": None,
        "current_attempt":  0,
        "max_attempts":     3,
        "quality_scores":   [],
        "next_action":      "",
        "reasoning":        "",
        "final_artifacts":  None,
        "success":          False,
    }

    # invoke 또는 스트리밍
    for event in app.stream(initial_state, config):
        for node, state in event.items():
            print(f"\n>>> Node: {node}")
            if state.get("reasoning"):
                print(f"   판단: {state['reasoning']}")
            if state.get("next_action"):
                print(f"   다음: {state['next_action']}")

    final = app.get_state(config).values
    print(f"\n=== 최종 ===")
    print(f"성공: {final['success']}")
    print(f"시도 횟수: {final['current_attempt']}")
    print(f"품질 기록: {final['quality_scores']}")
    return final


if __name__ == "__main__":
    result = run_agentic_pipeline(
        image_path="./samples/sangso_damaged_01.jpg",
        output_dir="./output/agentic_run_01",
        title="sangso_01",
    )
```

### Agent가 "모른다"를 말할 수 있어야 한다

Agentic 시스템을 오래 돌려 보면서 배운 가장 중요한 한 가지는, **Agent가 "나는 이 결정을 못 내리겠습니다"라고 말할 수 있어야 한다**는 것이다. 우리 CrossCheck case 4 → `human_review` 분기가 바로 그 자리다.

한 번은 Agent가 "expert_review" 를 선택한 복원 건을 한문학 전공 교수님께 보냈다. 며칠 뒤 그분이 보내주신 답장에 이렇게 적혀 있었다.

> "흥미롭네요. 이 AI는 자기가 모른다는 걸 알고 있군요. 그게 제일 어려운 일인데."

모르는 걸 모른다고 말하는 모델을 만드는 것 — 실은 가장 어려운 엔지니어링이다. 환각 대신 침묵을, 자신감 있는 오답 대신 정직한 유보를 택하게 하는 것. Agentic 설계에서 놓치지 말아야 할 원칙.

## 함정 카드

1. **"Agent가 무한 루프"** → `max_attempts` 강제 종료 조건 필수. LangGraph의 `recursion_limit` 설정. Agent 판단이 흔들리면 무한 재시도 가능.

2. **"Agent 호출이 비싸다"** → 매 평가 노드에 Claude Opus 호출 = 건당 $0.01~0.05. 배치 1,000건이면 $50. 비용 관리: ① 간단 규칙으로 거를 수 있는 경우(CER > 0.5 같은 명확 실패)는 규칙 기반 ② Claude Haiku 4.5($0.001/호출)로 1차 판단, 애매할 때만 Opus 호출.

3. **"State에 이미지 binary 포함"** → LangGraph state는 체크포인트마다 직렬화. 10MB 이미지 100번 체크포인트 = 1GB 저장소 낭비. **파일 경로만 저장**, binary는 S3 등 외부.

4. **"Agent가 상상으로 도구 호출"** → Claude가 실제 없는 `.restore_with_magic()` 같은 함수 호출 시도. Tool 정의를 엄격하게, 없는 도구는 `NotImplementedError` 던져 Agent가 현실 인지하게.

5. **"LangSmith 로그에 민감 데이터 유출"** → 사용자 고문서 이미지·판독문이 LangSmith 서버로. GDPR·개인정보 문제. 자체 호스팅 LangGraph + 자체 관측 스택(OpenTelemetry + Jaeger) 권장. LangSmith는 개발만.

## 연습문제

**문제 1.** Agent가 "retry_different_seed" 반복만 선택해 품질 개선 없이 max_attempts 도달. 원인은?

> **해설:** Agent가 **루프 탈출 인센티브 부족**. 이전 시도 결과를 제대로 학습 안 함. 개선: ① **명시적 반례 프롬프트** — "seed 재시도 2회 이미 했음, 품질 변화 없음. 같은 선택 금지" ② **score 추세 분석** — 마지막 2시도 개선률 < 5%면 다른 전략 강제 ③ **선택지 동적 조정** — 재시도 N회 후 "retry_different_seed" 옵션을 선택지에서 제거. Agentic 시스템 고질 문제 — "탐색 vs 활용" 균형.

**문제 2.** LangGraph 체크포인트를 PostgreSQL에 저장하려면?

> **해설:** `from langgraph.checkpoint.postgres import PostgresSaver`로 Memory 대신 Postgres 저장소 사용. 연결 문자열 `postgresql://user:pass@host:5432/langgraph`. 장점: 서버 재시작해도 세션 유지, 멀티 인스턴스에서 공유. 스키마는 LangGraph가 자동 생성. 주의: 체크포인트가 많아지면 DB 용량 폭증 — 주기 정리 또는 TTL(30일). 프로덕션은 MemorySaver 대신 Postgres 또는 Redis.

**문제 3.** Agentic 파이프라인과 정적 DAG를 동시에 유지할 가치가 있는가?

> **해설:** **있다, 용도 분리**. ① **정적 DAG (Prefect)**: 검증된 흐름의 대량 배치 처리. 예측 가능, 비용 낮음 ② **Agentic (LangGraph)**: 미지 도메인·edge case·품질 민감 건. 예측 어렵지만 깊은 대응. 실무 구조: 기본은 DAG, DAG의 실패 케이스만 Agentic으로 재처리. 또는 문서 유형별로 라우팅 — 표준 상소문은 DAG, 외국 문서·특수 자료는 Agentic. 두 시스템 유지 복잡도 있으나 **품질 + 비용 최적** 조합.
