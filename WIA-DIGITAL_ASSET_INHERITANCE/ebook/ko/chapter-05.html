1. Phase 2 개요

Phase 2는 2025년 하반기(Q3-Q4)에 진행되며, 데이터 포맷 위에 실제 플랫폼 간 통신을 가능하게 하는 API 계층을 구축합니다. 이 단계에서는 5개 파일럿 플랫폼에서 20개 주요 플랫폼으로 확대되며, 월 1,000건 이상의 상속 케이스를 처리할 수 있는 확장성을 확보합니다.

🎯 Phase 2 목표

  • WIA API Gateway 구축 및 운영
  • RESTful API + GraphQL 이중 지원
  • Pass/NPKI 통합 인증 게이트웨이 완성

  • 디지털 자산 조회/이전 API 구현

3. RESTful API 엔드포인트 설계

WIA는 직관적이고 RESTful한 API 설계를 따릅니다:

엔드포인트 메서드 설명 인증
/api/v1/inheritance POST 상속 케이스 생성 Pass/NPKI
/api/v1/inheritance/{id} GET 상속 케이스 조회 JWT
/api/v1/assets/discover POST 디지털 자산 검색 OAuth 2.0
/api/v1/assets/transfer POST 자산 이전 실행 Pass + OTP
/api/v1/beneficiary/verify POST 유족 신원 확인 행안부 API
/api/v1/tax/calculate POST 상속세 계산 국세청 API
/api/v1/platforms GET 지원 플랫폼 목록 None
/api/v1/audit/{id} GET 감사 추적 조회 JWT

5. 자산 이전 API 상세

디지털 자산을 실제로 이전하는 API입니다. 보안이 가장 중요합니다:

⚠️ 자산 이전 API 보안 요구사항

  • 다중 인증: Pass + OTP + 생체인식 (3-factor authentication)
  • 승인 프로세스: 복수 상속인 시 전원 동의 필요
  • 금액 제한: 1회 1억원 이상 시 추가 검증
  • 쿨다운 타임: 대량 이전 시 24시간 대기 기간
  • 블록체인 기록: 모든 이전 내역 영구 기록
  • 이의신청 기간: 이전 후 30일간 이의 제기 가능

6. GraphQL 스키마

복잡한 쿼리와 실시간 업데이트를 위한 GraphQL도 지원합니다:

# WIA Digital Inheritance GraphQL Schema

type Query {
  # 상속 케이스 조회
  inheritance(id: ID!): Inheritance
  inheritances(filter: InheritanceFilter): [Inheritance!]!

  # 디지털 자산 조회
  assets(deceasedId: ID!): AssetDiscoveryResult
  asset(id: ID!): DigitalAsset

  # 플랫폼 정보
  platforms: [Platform!]!
  platform(name: String!): Platform

  # 유족 정보
  beneficiaries(inheritanceId: ID!): [Beneficiary!]!
}

type Mutation {
  # 상속 프로세스
  createInheritance(input: CreateInheritanceInput!): Inheritance!
  updateInheritanceStatus(id: ID!, status: InheritanceStatus!): Inheritance!

  # 자산 관리
  discoverAssets(input: AssetDiscoveryInput!): AssetDiscoveryResult!
  transferAssets(input: AssetTransferInput!): AssetTransferResult!

  # 유족 확인
  verifyBeneficiary(input: BeneficiaryVerificationInput!): BeneficiaryVerificationResult!

  # 세무
  calculateTax(inheritanceId: ID!): TaxCalculation!
}

type Subscription {
  # 실시간 상속 프로세스 업데이트
  inheritanceUpdated(id: ID!): Inheritance!

  # 자산 검색 진행 상황
  assetDiscoveryProgress(discoveryId: ID!): DiscoveryProgress!

  # 플랫폼 응답 상태
  platformStatus: PlatformStatusUpdate!
}

type Inheritance {
  id: ID!
  deceased: Person!
  beneficiaries: [Beneficiary!]!
  assets: [DigitalAsset!]!
  status: InheritanceStatus!
  timeline: Timeline!
  taxInformation: TaxInformation
  auditTrail: [AuditEntry!]!
}

type DigitalAsset {
  id: ID!
  platform: String!
  category: AssetCategory!
  assetType: String!
  currentValue: MonetaryAmount!
  metadata: JSON!
}

enum InheritanceStatus {
  INITIATED
  BENEFICIARY_VERIFIED
  ASSETS_DISCOVERED
  TAX_CALCULATED
  TAX_PAID
  TRANSFER_PENDING
  TRANSFER_COMPLETED
  OBJECTION_PERIOD
  COMPLETED
  DISPUTED
}

8. Pass/NPKI 통합 인증 게이트웨이

한국 고유의 인증 체계를 완벽하게 통합한 게이트웨이:

🔐 통합 인증 프로세스

1단계: 사용자 인증 선택

  • Pass 인증서 (모바일 간편 인증) - 권장
  • NPKI 공동인증서 (PC 인증)
  • 생체인증 (지문, 얼굴인식) - Pass 연동
  • 카카오/네이버 간편 로그인 - 초기 접근

2단계: WIA Gateway 검증

  • Pass Alliance API로 실시간 CI(Connecting Information) 검증
  • NPKI 인증서 PKCS#7 서명 검증
  • 행안부 주민등록 시스템으로 실명 확인
  • 검증 결과 JWT 토큰 발급 (1시간 유효)

3단계: 권한 확인

  • 행안부 전자가족관계등록시스템 API로 상속 권한 확인
  • 법원 전산망으로 상속 포기/한정승인 여부 조회
  • 금융위원회 API로 금융사고 정보 확인
  • 모든 검증 통과 시 자산 접근 권한 부여

4단계: 블록체인 기록

  • 인증 성공 시각, 방법, IP 주소 블록체인 기록
  • 이후 모든 API 호출 감사 추적
  • 의심 활동 시 실시간 알림

9. 에러 처리 및 재시도 로직

20개 플랫폼과의 안정적인 통신을 위한 에러 처리 전략:

에러 유형 HTTP 코드 재시도 전략 사용자 조치
인증 실패 401 재시도 안함 재인증 필요
권한 부족 403 재시도 안함 관리자 문의
리소스 없음 404 재시도 안함 요청 정보 확인
요청 제한 초과 429 Exponential Backoff 1분 후 재시도
서버 오류 500 3회 재시도 5초, 15초, 45초 대기
게이트웨이 타임아웃 504 2회 재시도 플랫폼 점검 여부 확인
