1. WIA-DIGITAL_ASSET_INHERITANCE 표준 전체 구조

WIA 표준은 4계층 아키텍처로 설계되어 있으며, 각 계층은 명확한 책임과 인터페이스를 가집니다.

계층 역할 주요 구성 요소 한국 구현
1. 데이터 계층 상속 정보 표준 포맷 JSON-LD, Schema.org 확장 한글 필드, 주민번호 대응
2. API 계층 플랫폼 간 통신 RESTful API, GraphQL Pass/NPKI 통합
3. 프로토콜 계층 상속 프로세스 정의 State Machine, 스마트 계약 법원/국세청 연동
4. 통합 계층 엔드유저 서비스 웹/모바일 앱, CLI 카카오/네이버 SSO

2. 핵심 구성 요소

WIA 표준은 다음 7개 핵심 모듈로 구성됩니다:

🔧 핵심 모듈

  • 인증 모듈 (Authentication Module): Pass, NPKI, 생체인증 통합
  • 자산 검색 모듈 (Asset Discovery): 플랫폼 크롤링 및 자산 탐지
  • 유족 확인 모듈 (Beneficiary Verification): 행안부 API 가족관계 확인
  • 데이터 전송 모듈 (Data Transfer): 플랫폼 간 안전한 자산 이동
  • 세무 연동 모듈 (Tax Integration): 홈택스 자동 신고
  • 감사 추적 모듈 (Audit Trail): 블록체인 기록
  • 프라이버시 보호 모듈 (Privacy Protection): AI 마스킹, 암호화
// WIA 표준 아키텍처 구조
interface WIAArchitecture {
  // 1. 데이터 계층 - 표준 상속 정보 포맷
  dataLayer: {
    format: "JSON-LD",
    schema: {
      "@context": "https://wia.org/contexts/digital-inheritance/v1",
      "@type": "DigitalAssetInheritance",
      deceased: {
        id: "주민번호 해시",
        name: "홍길동",
        deathDate: "2024-01-01",
        digitalAssets: [
          {
            platform: "카카오페이",
            assetType: "금융",
            value: 5_000_000,
            accountId: "encrypted"
          }
        ]
      },
      beneficiaries: [
        {
          relationship: "배우자",
          shareRatio: 0.5,
          verified: true,
          verificationMethod: "Pass인증서"
        }
      ]
    }
  },

  // 2. API 계층 - 플랫폼 연동 인터페이스
  apiLayer: {
    authentication: {
      provider: "WIA Auth Gateway",
      supported: ["Pass", "NPKI", "생체인증", "OAuth2"],
      koreanIntegration: {
        pass: "PASS Alliance 27개 은행",
        npki: "행안부 공동인증서",
        codef: "CODEF 금융 API"
      }
    },

    endpoints: {
      discovery: "POST /api/v1/assets/discover",
      transfer: "POST /api/v1/assets/transfer",
      verify: "POST /api/v1/beneficiary/verify",
      tax: "POST /api/v1/tax/declare"
    }
  },

  // 3. 프로토콜 계층 - 상속 프로세스
  protocolLayer: {
    stateMachine: [
      "INITIATED",      // 상속 시작
      "VERIFIED",       // 유족 확인 완료
      "DISCOVERED",     // 자산 발견 완료
      "TAX_CALCULATED", // 상속세 산정
      "TAX_PAID",       // 납세 완료
      "TRANSFERRED",    // 자산 이전 완료
      "COMPLETED"       // 상속 종료
    ],

    koreanProcess: {
      courtIntegration: "전자가족관계등록시스템",
      taxIntegration: "홈택스 상속세 신고",
      notaryIntegration: "대한법무사협회 전자공증"
    }
  },

  // 4. 통합 계층 - 사용자 인터페이스
  integrationLayer: {
    web: "https://inheritance.wia.org",
    mobile: ["Android", "iOS"],
    cli: "wia-inheritance-cli",

    koreanPartners: {
      kakao: "카카오 싱크 연동",
      naver: "네이버 아이디 로그인",
      toss: "토스 간편 인증"
    }
  }
}

4. 한국형 디지털 상속 아키텍처

한국의 독특한 법적, 기술적 환경을 반영한 특화 설계입니다:

🇰🇷 한국 특화 설계

  • Pass Alliance 통합: KB국민, 신한, 하나 등 27개 은행 Pass 인증서 한 번에 검증
  • 행안부 API 직접 연동: 전자가족관계등록시스템으로 실시간 가족관계 확인 (위조 불가)
  • 국세청 홈택스 연계: 상속세 자동 계산 및 전자신고 (5분 내 완료)
  • 법원 전산망 접속: 상속 포기/한정승인 여부 즉시 조회
  • CODEF 금융 API: 100개 이상 금융사 계좌 일괄 조회
  • 카카오/네이버 SSO: 전 국민 95% 보유 계정으로 간편 로그인

5. 4단계 구현 로드맵 상세

WIA 표준은 점진적 도입을 위해 4개 Phase로 구성됩니다:

Phase 기간 목표 한국 파트너 성공 지표
Phase 1 2025 Q1-Q2 데이터 포맷 정의 카카오, 네이버, 토스, 업비트, 넷마블 5개 플랫폼 참여
Phase 2 2025 Q3-Q4 API 인터페이스 구축 20개 주요 플랫폼 확대 월 1,000건 처리
Phase 3 2026 상반기 프로토콜 완전 구현 행안부, 금융위, 국세청 시장 50% 커버
Phase 4 2026 하반기 국제 표준 연동 Google, Apple, Meta 글로벌 상속 지원

7. 상속 프로세스 플로우

WIA 표준의 전형적인 상속 프로세스는 다음과 같습니다:

📋 표준 상속 프로세스 (7단계)

1단계: 사망 통지 및 초기화 (1일차)

  • 행안부 사망신고 연동으로 자동 감지
  • 등록된 디지털 자산 목록 생성
  • 유족에게 SMS/이메일 자동 발송

2단계: 유족 신원 확인 (2-3일차)

  • Pass 인증서 또는 NPKI로 로그인
  • 행안부 API로 가족관계 실시간 검증
  • 복수 상속인 시 전원 확인 완료 필요

3단계: 디지털 자산 탐색 (3-4일차)

  • CODEF API로 100개 금융사 일괄 조회
  • 카카오/네이버/토스 등 주요 플랫폼 스캔
  • 암호화폐 거래소 5개사 잔액 확인

4단계: 자산 평가 및 상속세 계산 (4-5일차)

  • WIA 표준 평가 기준 적용 (암호화폐는 3개소 평균)
  • 국세청 홈택스 API로 상속세 자동 계산
  • 공제 및 감면 항목 자동 적용

5단계: 상속세 납부 (6일차)

  • 토스/카카오페이 간편 납부
  • 분할 납부 옵션 제공
  • 납부 즉시 블록체인 기록

6단계: 자산 이전 집행 (7일차)

  • WIA API 통해 각 플랫폼에 이전 요청
  • 스마트 계약 자동 실행
  • 지분율에 따라 자동 분배

7단계: 완료 및 감사 (8일차 이후)

  • 전체 프로세스 블록체인 영구 기록
  • 유족에게 완료 증명서 발급
  • 6개월 간 이의신청 기간

9. 표준 준수 인증 프로그램

WIA는 플랫폼 사업자를 위한 3단계 인증 프로그램을 운영합니다:

🏅 WIA 인증 레벨

  • Bronze 인증: 데이터 포맷 준수 (Phase 1 완료)
  • Silver 인증: API 인터페이스 구현 (Phase 2 완료)
  • Gold 인증: 완전한 프로토콜 구현 (Phase 3-4 완료)

한국 인증 현황 (2025.01 기준)

  • Gold: 카카오, 토스 (2개사)
  • Silver: 네이버, 업비트, 빗썸 (3개사)
  • Bronze: 쿠팡, 배민, 당근마켓 등 (15개사)
