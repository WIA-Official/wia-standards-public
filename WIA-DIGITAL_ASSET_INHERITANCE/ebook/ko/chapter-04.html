📖 Chapter 04

Phase 1: 데이터 포맷 정의

JSON-LD 스키마와 한국 특화 구현

1. Phase 1 개요

WIA 표준의 첫 번째 단계는 디지털 상속 정보의 표준 데이터 포맷을 정의하는 것입니다. 2025년 상반기 동안 5개 파일럿 플랫폼(카카오, 네이버, 토스, 업비트, 넷마블)이 이 포맷을 채택하여 상호운용성의 기반을 마련합니다.

🎯 Phase 1 목표

  • JSON-LD 기반 표준 스키마 정의
  • 한국 법률 및 인증 체계 반영 (주민번호, Pass, NPKI)
  • 5개 파일럿 플랫폼 데이터 포맷 통일
  • 암호화 및 프라이버시 보호 규격 수립
  • 정부 시스템 (행안부, 국세청) 연동 포맷 확정

4. 자산 유형별 스키마

WIA는 5가지 주요 자산 유형에 대한 세부 스키마를 정의합니다:

💰 자산 유형별 필수 필드

1. 금융 자산 (FinancialAsset)

  • platform: 카카오페이, 네이버페이, 토스 등
  • accountNumber: 암호화된 계좌번호
  • balance: 잔액 (KRW)
  • lastTransaction: 마지막 거래일시

2. 암호화폐 (CryptoCurrency)

  • cryptocurrency: BTC, ETH, KLAY 등
  • amount: 보유 수량
  • walletAddress: 지갑 주소
  • exchange: 보관 거래소
  • valuationMethod: WIA 3거래소 평균

3. 게임 자산 (GameAsset)

  • gameName: 게임명
  • characterInfo: 캐릭터 레벨, 아이템
  • transferability: 상속 가능 여부 (약관 확인)
  • estimatedValue: 추정 가치 (참고용)

4. SNS/계정 (SocialAccount)

  • platform: 카카오톡, 네이버, 인스타그램
  • accountId: 사용자 ID
  • contentValue: 콘텐츠 가치 (유튜버 등)
  • memorialization: 추모 계정 전환 여부

5. 디지털 콘텐츠 (DigitalContent)

  • contentType: 사진, 동영상, 문서
  • storageLocation: 네이버 클라우드, 구글 드라이브
  • copyright: 저작권 정보
  • commercialValue: 상업적 가치 (유료 콘텐츠)

6. 5개 파일럿 플랫폼 구현 사례

2025년 Q1-Q2 기간 동안 5개 플랫폼이 Phase 1을 구현합니다:

넷마블 게임 자산 스키마 90% 준수 게임 아이템 평가 어려움

8. 데이터 검증 규칙

WIA 스키마 준수를 위한 필수 검증 규칙들:

검증 항목 규칙 오류 처리 한국 특수성
주민번호 형식 13자리 + SHA-256 해시 원본 저장 금지 개인정보보호법 준수
금액 단위 KRW 정수형 소수점 반올림 원화 단위만 허용
날짜 형식 ISO 8601 + KST UTC 변환 후 저장 한국 시간대 +09:00
암호화 필드 AES-256 이상 평문 저장 차단 ARIA/SEED 허용
블록체인 해시 Keccak-256 재계산 후 검증 Klaytn 호환

  • □ 블록체인 감사 추적 기록
  • □ 행안부 API 연동 (가족관계 확인)
  • □ 데이터 검증 로직 구현
  • □ 개인정보보호법 준수 (원본 주민번호 미저장)
  • □ WIA 공식 검증 도구 테스트 통과
  • □ 최소 100건의 테스트 케이스 성공
