1. 암호화폐의 탄생: 비트코인과 블록체인

2008년 금융 위기 직후, 사토시 나카모토(Satoshi Nakamoto)라는 가명의 인물이 "비트코인: 개인 간 전자 화폐 시스템(Bitcoin: A Peer-to-Peer Electronic Cash System)"이라는 백서를 발표하며 암호화폐 시대가 시작되었습니다. 비트코인은 중앙 기관 없이도 신뢰할 수 있는 거래를 가능하게 하는 혁신적인 기술인 블록체인을 세상에 선보였습니다.

📌 학습 목표

이 장에서는 암호화폐와 블록체인 기술의 핵심 개념을 학습하고, 한국의 암호화폐 생태계를 이해합니다.

  • 비트코인과 블록체인의 작동 원리 이해
  • 합의 알고리즘(PoW, PoS 등) 비교
  • 이더리움과 스마트 계약 개념 학습
  • 한국 주요 거래소 (업비트, 빗썸, 코인원) 현황 파악
  • DeFi, NFT 등 새로운 응용 분야 이해

블록체인은 거래 기록을 블록으로 묶고, 이 블록들을 체인으로 연결하여 변경 불가능한 장부를 만드는 기술입니다. 각 블록은 이전 블록의 해시값을 포함하고 있어, 한 블록을 변경하려면 그 이후의 모든 블록을 재계산해야 하므로 사실상 불가능합니다.

2. 블록체인의 핵심 구성 요소

블록체인 기술은 다음과 같은 핵심 요소들로 구성됩니다:

구성 요소 설명 기능
블록 (Block) 거래 데이터를 담는 단위 헤더(메타데이터) + 거래 목록
해시 (Hash) 블록의 고유 지문 데이터 무결성 보장, 체인 연결
노드 (Node) 블록체인을 저장하고 검증하는 컴퓨터 거래 검증, 블록 생성, 네트워크 유지
합의 알고리즘 네트워크 참여자 간 합의 방법 이중 지불 방지, 보안 유지
공개키/개인키 암호화 키 쌍 소유권 증명, 거래 서명
P2P 네트워크 분산 네트워크 구조 중앙 서버 없이 직접 통신

3. 합의 알고리즘: 신뢰의 메커니즘

블록체인의 핵심은 중앙 기관 없이 어떻게 참여자들이 거래의 유효성에 합의하느냐 하는 것입니다. 이를 해결하는 것이 합의 알고리즘입니다.

알고리즘 원리 장점 단점 사용 예
작업증명
(PoW)		 복잡한 수학 문제 해결 높은 보안성, 검증된 방식 막대한 전력 소비, 느린 속도 비트코인
지분증명
(PoS)		 보유 지분에 비례한 검증권 낮은 에너지 소비, 빠른 속도 부자가 더 부유해지는 구조 이더리움 2.0
위임지분증명
(DPoS)		 대표자 선출 후 검증 위임 매우 빠른 속도, 확장성 중앙화 가능성 EOS, 트론
실용적 비잔틴
(PBFT)		 투표를 통한 합의 빠른 최종성 제한적 노드 수 하이퍼레저

⚡ 비트코인의 전력 소비

비트코인 네트워크는 연간 약 150 TWh의 전력을 소비하는 것으로 추정되며, 이는 아르헨티나 전체의 전력 소비량과 비슷합니다. 이러한 환경 문제로 인해 이더리움은 2022년 PoW에서 PoS로 전환(The Merge)하여 에너지 소비를 99.95% 줄였습니다.

4. 블록체인 데이터 구조

WIA-DIGITAL_CURRENCY 표준에 따른 블록체인 데이터 모델입니다:

interface Block {
  // 블록 헤더
  header: {
    version: number;              // 프로토콜 버전
    previousHash: string;         // 이전 블록 해시 (SHA-256)
    merkleRoot: string;           // 머클 루트 (거래들의 해시 트리)
    timestamp: number;            // 블록 생성 시간 (Unix timestamp)
    difficulty: number;           // 난이도 (PoW의 경우)
    nonce: number;                // 논스값 (PoW의 경우)
  };

  // 블록 본문
  transactions: Transaction[];    // 거래 목록

  // 블록 메타데이터
  hash: string;                   // 이 블록의 해시
  height: number;                 // 블록 높이 (제네시스 블록부터의 순서)
  size: number;                   // 블록 크기 (bytes)
  txCount: number;                // 포함된 거래 수

  // 검증 정보
  validator?: {
    address: string;              // 검증자 주소 (PoS의 경우)
    signature: string;            // 검증자 서명
    stake?: number;               // 스테이킹한 코인 양
  };
}

interface Transaction {
  txid: string;                   // 거래 ID
  version: number;                // 거래 버전

  // 입력 (코인 출처)
  inputs: {
    txid: string;                 // 이전 거래 ID
    vout: number;                 // 출력 인덱스
    scriptSig: string;            // 서명 스크립트
    sequence: number;             // 시퀀스 번호
  }[];

  // 출력 (코인 목적지)
  outputs: {
    value: number;                // 코인 양
    scriptPubKey: string;         // 공개키 스크립트 (잠금)
    address: string;              // 수신자 주소
  }[];

  // 추가 정보
  lockTime: number;               // 잠금 시간
  fee: number;                    // 거래 수수료
  size: number;                   // 거래 크기
  confirmations: number;          // 확인 수
}

5. 이더리움과 스마트 계약

비트코인이 디지털 화폐에 집중했다면, 2015년 탄생한 이더리움(Ethereum)은 "세계 컴퓨터"를 지향하며 블록체인을 범용 플랫폼으로 확장했습니다. 이더리움의 핵심은 스마트 계약(Smart Contract)입니다.

스마트 계약은 블록체인에 저장되어 자동으로 실행되는 프로그램입니다. 조건이 충족되면 코드가 자동으로 실행되어 중개자 없이도 계약을 이행할 수 있습니다.

// Solidity 스마트 계약 예제: ERC-20 토큰
pragma solidity ^0.8.0;

contract SimpleToken {
    // 상태 변수
    string public name = "My Token";
    string public symbol = "MTK";
    uint8 public decimals = 18;
    uint256 public totalSupply;

    // 잔액 매핑
    mapping(address => uint256) public balanceOf;
    mapping(address => mapping(address => uint256)) public allowance;

    // 이벤트
    event Transfer(address indexed from, address indexed to, uint256 value);
    event Approval(address indexed owner, address indexed spender, uint256 value);

    // 생성자
    constructor(uint256 initialSupply) {
        totalSupply = initialSupply * 10 ** uint256(decimals);
        balanceOf[msg.sender] = totalSupply;
    }

    // 전송 함수
    function transfer(address to, uint256 value) public returns (bool) {
        require(balanceOf[msg.sender] >= value, "Insufficient balance");
        require(to != address(0), "Invalid address");

        balanceOf[msg.sender] -= value;
        balanceOf[to] += value;

        emit Transfer(msg.sender, to, value);
        return true;
    }

    // 승인 함수
    function approve(address spender, uint256 value) public returns (bool) {
        allowance[msg.sender][spender] = value;
        emit Approval(msg.sender, spender, value);
        return true;
    }

    // 위임 전송 함수
    function transferFrom(
        address from,
        address to,
        uint256 value
    ) public returns (bool) {
        require(balanceOf[from] >= value, "Insufficient balance");
        require(allowance[from][msg.sender] >= value, "Allowance exceeded");
        require(to != address(0), "Invalid address");

        balanceOf[from] -= value;
        balanceOf[to] += value;
        allowance[from][msg.sender] -= value;

        emit Transfer(from, to, value);
        return true;
    }
}

7. DeFi(탈중앙화 금융): 금융의 혁명

DeFi(Decentralized Finance)는 블록체인 기반의 탈중앙화 금융 서비스로, 은행이나 금융 기관 없이도 대출, 예금, 거래, 투자 등을 할 수 있게 합니다.

전통 금융 (CeFi)

  • 중앙화된 기관이 통제
  • 신원 확인 필수 (KYC)
  • 영업 시간 제한
  • 중개 수수료 높음
  • 규제 강함
  • 예금자 보호

탈중앙화 금융 (DeFi)

  • 스마트 계약이 자동 실행
  • 지갑 주소만 있으면 참여
  • 24/7 운영
  • 낮은 수수료
  • 규제 불명확
  • 자기 책임 원칙

주요 DeFi 서비스:

  • 탈중앙화 거래소(DEX): Uniswap, SushiSwap - 중개자 없이 암호화폐를 직접 거래
  • 대출/차입: Aave, Compound - 암호화폐를 담보로 대출하거나 예치하여 이자 수익
  • 스테이블코인: DAI, USDC - 법정화폐에 가치가 고정된 암호화폐
  • 유동성 풀: 사용자가 자금을 제공하고 거래 수수료를 나눠 받는 구조

8. NFT: 디지털 자산의 소유권 증명

NFT(Non-Fungible Token, 대체 불가능 토큰)는 블록체인에 기록되는 고유한 디지털 자산입니다. 각 NFT는 고유한 식별자를 가지며, 디지털 예술품, 음악, 게임 아이템, 부동산 등 다양한 자산의 소유권을 증명할 수 있습니다.

특성 일반 토큰 (FT) NFT
대체 가능성 대체 가능 (1 BTC = 1 BTC) 고유함 (각각 다른 가치)
분할 가능성 분할 가능 (0.001 BTC) 분할 불가능 (통째로만 거래)
표준 ERC-20 (이더리움) ERC-721, ERC-1155
용도 화폐, 거래 수단 소유권 증명, 수집품

9. 암호화폐 지갑 구현

암호화폐를 안전하게 보관하고 거래하기 위한 지갑 구현 예제입니다:

import { ec as EC } from 'elliptic';
import { createHash } from 'crypto';

class CryptoWallet {
  private keyPair: any;
  private ec: any;

  constructor() {
    this.ec = new EC('secp256k1'); // 비트코인/이더리움이 사용하는 타원곡선
  }

  // 새 지갑 생성
  generateWallet(): {
    privateKey: string;
    publicKey: string;
    address: string;
  } {
    this.keyPair = this.ec.genKeyPair();
    const privateKey = this.keyPair.getPrivate('hex');
    const publicKey = this.keyPair.getPublic('hex');
    const address = this.generateAddress(publicKey);

    return { privateKey, publicKey, address };
  }

  // 개인키로부터 지갑 복원
  importWallet(privateKey: string) {
    this.keyPair = this.ec.keyFromPrivate(privateKey, 'hex');
    return {
      privateKey,
      publicKey: this.keyPair.getPublic('hex'),
      address: this.generateAddress(this.keyPair.getPublic('hex'))
    };
  }

  // 주소 생성 (공개키 → 주소)
  private generateAddress(publicKey: string): string {
    // SHA-256 해시
    const sha256Hash = createHash('sha256')
      .update(Buffer.from(publicKey, 'hex'))
      .digest();

    // RIPEMD-160 해시
    const ripemd160Hash = createHash('ripemd160')
      .update(sha256Hash)
      .digest('hex');

    // 간단하게 앞에 '1' 붙이기 (실제로는 Base58Check 인코딩)
    return '1' + ripemd160Hash;
  }

  // 거래 서명
  signTransaction(txData: any): string {
    const txHash = createHash('sha256')
      .update(JSON.stringify(txData))
      .digest();

    const signature = this.keyPair.sign(txHash);
    return signature.toDER('hex');
  }

  // 서명 검증
  static verifySignature(
    txData: any,
    signature: string,
    publicKey: string
  ): boolean {
    const ec = new EC('secp256k1');
    const key = ec.keyFromPublic(publicKey, 'hex');
    const txHash = createHash('sha256')
      .update(JSON.stringify(txData))
      .digest();

    return key.verify(txHash, signature);
  }
}

// 사용 예시
const wallet = new CryptoWallet();
const myWallet = wallet.generateWallet();
console.log('내 지갑:', myWallet);

// 거래 생성 및 서명
const transaction = {
  from: myWallet.address,
  to: '1ReceiverAddressXXXXXXXXXXXXXXX',
  amount: 0.5,
  timestamp: Date.now()
};

const signature = wallet.signTransaction(transaction);
console.log('거래 서명:', signature);

// 서명 검증
const isValid = CryptoWallet.verifySignature(
  transaction,
  signature,
  myWallet.publicKey
);
console.log('서명 유효:', isValid);

10. 弘益人間: 암호화폐로 금융 민주화

암호화폐와 블록체인 기술은 弘益人間의 정신을 실현하는 강력한 도구가 될 수 있습니다:

  • 금융 접근성 확대: 은행이 없는 지역에 사는 25억 명의 사람들도 스마트폰만 있으면 글로벌 금융 시스템에 참여할 수 있습니다.
  • 송금 혁명: 필리핀, 베트남 등에서 일하는 외국인 근로자들이 본국으로 송금할 때 암호화폐를 사용하면 수수료가 7%에서 1% 이하로 줄어듭니다.
  • 투명한 기부: 블록체인에 기록된 기부금은 추적 가능하여, 실제로 필요한 곳에 사용되는지 확인할 수 있습니다.
  • 경제적 주권: 인플레이션이 심한 국가의 국민들이 자산을 보호할 수 있는 수단이 됩니다.
  • 예술가 지원: NFT를 통해 중개자 없이 예술가가 직접 작품을 판매하고, 재판매 시에도 로열티를 받을 수 있습니다.

  • 이더리움과 스마트 계약: 자동 실행되는 계약 코드로 DeFi, NFT 등 다양한 응용이 가능합니다.
  • 한국 3대 거래소: 업비트, 빗썸, 코인원이 일일 수조 원의 거래량을 기록하며, 실명 계좌가 의무화되어 있습니다.
  • DeFi: 은행 없이 대출, 예금, 거래가 가능한 탈중앙화 금융 서비스입니다.
  • NFT: 고유한 디지털 자산의 소유권을 블록체인에 기록하는 기술로, 예술, 게임, 부동산 등에 활용됩니다.
  • 弘益人間 구현: 금융 접근성 확대, 송금 혁명, 투명한 기부 등을 통해 인류를 이롭게 합니다.
