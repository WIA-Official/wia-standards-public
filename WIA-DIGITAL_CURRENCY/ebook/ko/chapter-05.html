1. Phase 2 개요: API 설계의 중요성

WIA-DIGITAL_CURRENCY 표준의 Phase 2는 API 인터페이스 규격을 정의합니다. Phase 1에서 정의한 데이터 포맷을 기반으로, 실제 시스템 간 통신에 사용되는 API를 표준화합니다. 잘 설계된 API는 개발자 경험을 향상시키고, 시스템 통합 비용을 절감합니다.

1.1 API 설계 원칙

원칙 설명 적용
RESTful 자원 중심의 URI 설계 /accounts, /transactions
일관성 모든 엔드포인트에서 동일한 패턴 응답 구조, 에러 형식
버전 관리 URL 기반 버전 명시 /api/v1/, /api/v2/
문서화 OpenAPI 3.0 기반 명세 자동 문서 생성
보안 우선 모든 요청에 인증 필수 OAuth 2.0, JWT

1.2 지원 API 형식

WIA-DIGITAL_CURRENCY는 세 가지 API 형식을 지원합니다:

  • REST API: 기본 CRUD 작업, 동기 요청/응답
  • GraphQL API: 복잡한 쿼리, 유연한 데이터 요청
  • WebSocket API: 실시간 스트리밍, 이벤트 구독

2. RESTful API 엔드포인트

REST API는 WIA-DIGITAL_CURRENCY의 기본 API 형식입니다. 모든 엔드포인트는 JSON 요청/응답을 사용하며, HTTP 표준 메서드를 따릅니다.

3. GraphQL API

GraphQL API는 복잡한 데이터 요청과 관계 탐색에 적합합니다. 클라이언트는 필요한 데이터만 정확하게 요청할 수 있어 네트워크 효율성이 향상됩니다.

3.1 GraphQL 스키마

# 타입 정의
type Account {
  id: ID!
  type: AccountType!
  status: AccountStatus!
  owner: Owner!
  wallets: [Wallet!]!
  limits: Limits!
  transactions(
    first: Int
    after: String
    filter: TransactionFilter
  ): TransactionConnection!
  createdAt: DateTime!
  updatedAt: DateTime!
}

type Wallet {
  id: ID!
  type: WalletType!
  network: String!
  address: Address!
  balances: [Balance!]!
  transactions(first: Int, after: String): TransactionConnection!
}

type Transaction {
  id: ID!
  type: TransactionType!
  status: TransactionStatus!
  sender: Address!
  recipient: Address!
  amount: Amount!
  fee: Amount
  timestamp: DateTime!
  metadata: JSON
  signatures: [Signature!]!
  confirmations: Int
}

type Amount {
  value: String!
  currency: String!
  decimals: Int!
  displayValue: String!
}

# Enum 타입
enum AccountType {
  INDIVIDUAL
  BUSINESS
  INSTITUTIONAL
  CUSTODIAL
}

enum TransactionStatus {
  PENDING
  CONFIRMED
  FAILED
  CANCELLED
  EXPIRED
}

enum WalletType {
  HOT
  COLD
  HARDWARE
  MULTISIG
  SMART_CONTRACT
}

3.2 쿼리 (Query)

# 스키마 정의
type Query {
  # 계정 조회
  account(id: ID!): Account
  accounts(filter: AccountFilter, pagination: Pagination): AccountConnection!

  # 트랜잭션 조회
  transaction(id: ID!): Transaction
  transactions(filter: TransactionFilter, pagination: Pagination): TransactionConnection!

  # 지갑 조회
  wallet(id: ID!): Wallet
  wallets(accountId: ID!, filter: WalletFilter): [Wallet!]!

  # 잔액 조회
  balance(walletId: ID!, currency: String!): Balance

  # 환율 조회
  exchangeRate(from: String!, to: String!): ExchangeRate
}

4. WebSocket API

WebSocket API는 실시간 데이터 스트리밍에 사용됩니다. 트랜잭션 상태 변경, 잔액 업데이트, 시장 데이터 등을 실시간으로 수신할 수 있습니다.

4.1 연결 및 인증

// WebSocket 연결
const ws = new WebSocket('wss://stream.wia-dc.org/v1');

// 연결 시 인증
ws.onopen = () => {
  ws.send(JSON.stringify({
    type: 'authenticate',
    payload: {
      token: 'Bearer {access_token}',
      clientId: 'client-123'
    }
  }));
};

// 인증 응답 처리
ws.onmessage = (event) => {
  const message = JSON.parse(event.data);

  if (message.type === 'authenticated') {
    console.log('연결 성공:', message.payload.sessionId);
    // 채널 구독 시작
    subscribeToChannels();
  }
};

4.2 채널 구독

// 채널 구독 요청
function subscribeToChannels() {
  // 트랜잭션 상태 구독
  ws.send(JSON.stringify({
    type: 'subscribe',
    channel: 'transactions',
    payload: {
      accountId: 'WIA-ACC-KR-0001234567',
      filter: {
        status: ['pending', 'confirmed']
      }
    }
  }));

  // 잔액 변경 구독
  ws.send(JSON.stringify({
    type: 'subscribe',
    channel: 'balances',
    payload: {
      walletIds: ['WIA-WAL-HOT-123', 'WIA-WAL-HOT-456']
    }
  }));

  // 시장 데이터 구독
  ws.send(JSON.stringify({
    type: 'subscribe',
    channel: 'market',
    payload: {
      pairs: ['BTC/KRW', 'ETH/KRW', 'USDC/KRW']
    }
  }));
}

// 메시지 수신 처리
ws.onmessage = (event) => {
  const message = JSON.parse(event.data);

  switch (message.channel) {
    case 'transactions':
      handleTransactionUpdate(message.payload);
      break;
    case 'balances':
      handleBalanceUpdate(message.payload);
      break;
    case 'market':
      handleMarketData(message.payload);
      break;
  }
};

8.2 TypeScript SDK 사용 예시

import { WiaDigitalCurrencyClient } from '@wia-dc/sdk';

// 클라이언트 초기화
const client = new WiaDigitalCurrencyClient({
  apiKey: process.env.WIA_API_KEY,
  environment: 'production',
  timeout: 30000,
  retries: 3
});

// 계정 조회
const account = await client.accounts.get('WIA-ACC-KR-0001234567');
console.log(account.status, account.owner.name);

// 트랜잭션 생성
const transaction = await client.transactions.create({
  type: 'transfer',
  sender: {
    type: 'internal',
    network: 'wia-dc',
    address: 'WIA-WAL-HOT-sender123'
  },
  recipient: {
    type: 'blockchain',
    network: 'ethereum-mainnet',
    address: '0x742d35Cc6634C0532925a3b844Bc454e4438f44e'
  },
  amount: {
    value: '1000.00',
    currency: 'USDC',
    decimals: 6
  }
});

console.log(`트랜잭션 생성됨: ${transaction.id}`);

// 실시간 상태 모니터링
const stream = client.transactions.watch(transaction.id);
stream.on('statusChanged', (status) => {
  console.log(`상태 변경: ${status.previous} -> ${status.current}`);
});
stream.on('confirmed', () => {
  console.log('트랜잭션 확인됨!');
  stream.close();
});

핵심 요약

  • REST API: 기본 CRUD 작업에 사용하며, JSON 요청/응답과 표준 HTTP 메서드를 따릅니다
  • GraphQL API: 복잡한 쿼리와 관계 탐색에 적합하며, 필요한 데이터만 효율적으로 요청합니다
  • WebSocket API: 실시간 이벤트 스트리밍에 사용하며, 트랜잭션 상태와 잔액 변경을 즉시 수신합니다
  • 보안: OAuth 2.0과 JWT 기반 인증을 사용하며, 모든 통신은 TLS 1.3으로 암호화됩니다
  • SDK: TypeScript, Python, Java, Go 공식 SDK를 제공하여 빠른 통합을 지원합니다

복습 문제

  1. REST API와 GraphQL API의 장단점을 비교하고, 각각 어떤 상황에서 사용하는 것이 적합한지 설명하시오.
  2. OAuth 2.0의 Authorization Code 플로우와 Client Credentials 플로우의 차이점을 설명하시오.
  3. Rate Limiting 정책의 목적과 클라이언트에서 429 응답을 받았을 때의 올바른 처리 방법을 설명하시오.
  4. WebSocket을 사용한 실시간 데이터 구독 시스템을 설계하고, 연결 끊김 시 재연결 전략을 제시하시오.
  5. API 버전 관리에서 하위 호환성을 유지하면서 새로운 기능을 추가하는 방법을 설명하시오.
  6. Idempotency Key의 역할과 이를 사용해야 하는 API 엔드포인트를 식별하고 그 이유를 설명하시오.
