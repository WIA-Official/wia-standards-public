1. CBDC란 무엇인가?

CBDC(Central Bank Digital Currency, 중앙은행 디지털 화폐)는 중앙은행이 직접 발행하고 보증하는 디지털 형태의 법정화폐입니다. 물리적인 지폐나 동전과 동일한 법적 지위를 가지며, 중앙은행의 부채로 기록됩니다.

이 장에서는 CBDC의 개념, 유형, 그리고 한국은행의 CBDC 프로젝트를 포함한 세계적인 개발 동향을 학습합니다.

  • CBDC의 정의와 기존 화폐와의 차이점 이해
  • 소매형 CBDC와 도매형 CBDC 구분
  • 한국은행 CBDC 파일럿 프로젝트 현황 파악
  • 주요 국가의 CBDC 개발 동향 학습
  • CBDC 구현을 위한 기술 아키텍처 이해

CBDC는 기존의 중앙은행 화폐 및 상업은행 예금과 다음과 같이 구분됩니다:

특성 현금 (지폐/동전) 상업은행 예금 CBDC
발행 주체 중앙은행 상업은행 중앙은행
형태 물리적 전자적 (장부) 전자적 (디지털)

접근성 누구나 소지 가능 계좌 개설 필요 디지털 지갑으로 접근
이자 지급 없음 가능 정책에 따라 가능
거래 추적 불가능 가능 가능

2. CBDC가 필요한 이유

전 세계 중앙은행들이 CBDC 개발에 나서는 이유는 다양하지만, 주요 동기는 다음과 같습니다:

  • 현금 사용 감소 대응: 디지털 결제 증가로 현금 사용이 급감하면서, 중앙은행은 디지털 시대에도 공공 화폐의 역할을 유지하고자 합니다.
  • 금융 포용성 강화: 은행 계좌 없이도 디지털 화폐에 접근할 수 있게 하여, 금융 서비스에서 소외된 계층을 포용합니다.
  • 결제 시스템 효율성: 24/7 실시간 결제가 가능하고, 국제 송금 비용과 시간을 대폭 줄일 수 있습니다.
  • 통화 주권 보호: 빅테크 기업의 스테이블코인이나 외국 CBDC의 확산에 대응하여 자국 통화 주권을 지킵니다.
  • 금융 정책 효과성: 이자 지급이 가능한 CBDC를 통해 통화 정책 전달 경로를 개선할 수 있습니다.
  • 불법 거래 방지: 거래 추적이 가능하여 자금 세탁, 탈세, 불법 거래를 효과적으로 단속할 수 있습니다.

💡 한국의 현금 사용 현황

한국은행 자료에 따르면, 한국의 현금 결제 비중은 2016년 34.3%에서 2021년 21.3%로 급감했습니다. 특히 코로나19 팬데믹 이후 비대면 결제가 일상화되면서 현금 없는 사회로의 전환이 가속화되고 있습니다. 토스, 카카오페이 등의 간편결제 서비스가 일상 결제의 주요 수단이 되었습니다.

3. 소매형 CBDC vs 도매형 CBDC

CBDC는 사용 대상과 목적에 따라 두 가지 유형으로 분류됩니다:

구분 소매형 CBDC (Retail) 도매형 CBDC (Wholesale)
사용 대상 일반 국민, 기업 금융기관, 중앙은행
주요 용도 일상적인 결제, 송금 대규모 은행 간 결제, 증권 결제
거래 규모 소액 거래 중심 대액 거래 중심
접근 방식 모바일 앱, 카드 등 전용 네트워크
익명성 제한적 익명성 고려 완전 식별
개발 난이도 높음 (복잡한 요구사항) 상대적으로 낮음
금융 시스템 영향 크다 (은행 예금 이동 가능) 제한적

대부분의 국가는 소매형 CBDC에 더 큰 관심을 두고 있으며, 일반 국민의 일상 결제 수단으로 활용하는 것을 목표로 합니다. 한국은행 역시 소매형 CBDC를 중심으로 파일럿 프로젝트를 진행하고 있습니다.

한국은행은 2020년부터 CBDC 연구를 본격화했으며, 2021년부터 2단계에 걸친 파일럿 프로젝트를 진행하고 있습니다.

한국은행은 Ground X(카카오 자회사)를 기술 파트너로 선정하여 프로젝트를 진행했으며, 블록체인 기술을 기반으로 한 CBDC 플랫폼을 구축했습니다. 주요 기술적 성과는 다음과 같습니다:

  • 높은 처리 성능: 초당 2,000건 이상의 거래 처리 (비자카드 수준)
  • 오프라인 결제: 인터넷 연결 없이도 거래 가능한 기술 검증
  • 프라이버시 보호: 영지식 증명(Zero-Knowledge Proof) 등 프라이버시 강화 기술 적용
  • 스마트 계약: 조건부 결제, 자동 이체 등 프로그래밍 가능한 화폐 기능

6. CBDC 아키텍처: 직접형 vs 중개형

CBDC를 구현하는 방식은 크게 두 가지 아키텍처로 나뉩니다:

직접형 (Direct Model)

  • 구조: 중앙은행이 직접 계좌 관리
  • 장점: 중앙은행의 직접 통제, 금융 정책 효과성 극대화
  • 단점: 중앙은행 부담 증가, 민간 은행 역할 축소
  • 사례: 바하마 Sand Dollar

중개형 (Intermediated Model)

  • 구조: 민간 은행이 고객 관리, 중앙은행은 백엔드 운영
  • 장점: 기존 금융 시스템 활용, 민간 혁신 유도
  • 단점: 시스템 복잡도 증가, 은행 간 조율 필요
  • 사례: 중국 e-CNY, 한국은행 파일럿

대부분의 국가는 기존 금융 시스템을 활용하고 민간 은행의 역할을 유지하기 위해 중개형 모델을 선호합니다. 한국은행도 중개형 모델을 채택하여, 시중은행들이 CBDC 지갑 서비스를 제공하고 중앙은행은 핵심 인프라를 운영하는 방식을 검토하고 있습니다.

7. CBDC 데이터 모델

WIA-DIGITAL_CURRENCY 표준에 따른 CBDC 데이터 모델은 다음과 같이 정의됩니다:

interface CBDC extends DigitalCurrency {
  type: "CBDC";

  // 중앙은행 정보
  centralBank: {
    name: string;                  // 예: "Bank of Korea"
    code: string;                  // BIS 코드 (예: "KRW")
    country: string;               // ISO 3166-1 alpha-2 (예: "KR")
    website: string;               // 공식 웹사이트
    contactInfo: {
      email: string;
      phone: string;
      address: string;
    };
  };

  // CBDC 유형
  cbdcType: "RETAIL" | "WHOLESALE" | "HYBRID";

  // 아키텍처 모델
  architecture: {
    model: "DIRECT" | "INTERMEDIATED" | "HYBRID";
    intermediaries?: string[];    // 중개 기관 목록 (중개형인 경우)
    technology: {
      platform: string;            // 예: "Hyperledger Fabric", "Corda"
      consensus: string;           // 합의 알고리즘
      throughput: number;          // 초당 거래 처리량 (TPS)
      latency: number;             // 평균 거래 지연 시간 (ms)
    };
  };

  // 통화 정책 기능
  monetaryPolicy: {
    interestBearing: boolean;      // 이자 지급 여부
    programmable: boolean;         // 프로그래밍 가능 여부
    tieredStructure: boolean;      // 계층별 이자율 차등 적용
    holdingLimit?: number;         // 개인 보유 한도 (있는 경우)
  };

  // 프라이버시 설정
  privacy: {
    level: "FULL" | "PSEUDO_ANONYMOUS" | "IDENTIFIED";
    anonymityThreshold?: number;   // 익명 거래 허용 한도
    kycRequired: boolean;          // KYC 필수 여부
    dataRetentionPeriod: number;   // 데이터 보관 기간 (일)
  };

  // 오프라인 기능
  offline: {
    supported: boolean;            // 오프라인 결제 지원 여부
    technology?: string;           // 예: "NFC", "Bluetooth"
    maxAmount?: number;            // 오프라인 거래 한도
    syncInterval?: number;         // 동기화 주기 (시간)
  };

  // 법적 기반
  legalFramework: {
    legislation: string[];         // 관련 법령
    legalTender: boolean;          // 법정화폐 지위 여부
    enforceableDate: string;       // 시행일 (ISO 8601)
  };
}

8. CBDC 발행 및 유통 프로세스

CBDC의 발행과 유통은 중개형 모델을 기준으로 다음과 같은 프로세스를 따릅니다:

class CBDCIssuanceSystem {
  private centralBankLedger: Ledger;
  private intermediaries: Map<string, Intermediary>;

  constructor() {
    this.centralBankLedger = new Ledger("CENTRAL_BANK");
    this.intermediaries = new Map();
  }

  // 1단계: CBDC 발행 (중앙은행 → 중개기관)
  async issueCBDC(
    intermediaryId: string,
    amount: number,
    collateral: Collateral
  ): Promise<IssuanceRecord> {
    // 담보 검증
    if (!this.validateCollateral(collateral, amount)) {
      throw new Error("Insufficient or invalid collateral");
    }

    // CBDC 발행
    const issuance = {
      id: generateId(),
      timestamp: new Date().toISOString(),
      intermediaryId,
      amount,
      collateral,
      status: "ISSUED"
    };

    // 중앙은행 장부에 기록
    await this.centralBankLedger.credit(intermediaryId, amount);

    // 담보 동결
    await this.lockCollateral(collateral);

    return issuance;
  }

  // 2단계: CBDC 배포 (중개기관 → 일반 사용자)
  async distributeCBDC(
    intermediaryId: string,
    userId: string,
    amount: number
  ): Promise<Distribution> {
    const intermediary = this.intermediaries.get(intermediaryId);

    if (!intermediary) {
      throw new Error("Intermediary not found");
    }

    // 중개기관의 보유액 확인
    const balance = await this.centralBankLedger.getBalance(intermediaryId);
    if (balance < amount) {
      throw new Error("Insufficient CBDC balance");
    }

    // 사용자 계정에 배포
    const distribution = await intermediary.creditUserAccount(userId, amount);

    return distribution;
  }

  // 3단계: CBDC 거래 (사용자 간)
  async transferCBDC(
    fromUserId: string,
    toUserId: string,
    amount: number
  ): Promise<Transaction> {
    // KYC/AML 검증
    await this.verifyCompliance(fromUserId, toUserId, amount);

    // 거래 실행
    const tx = await this.executeTransfer(fromUserId, toUserId, amount);

    // 거래 기록
    await this.centralBankLedger.recordTransaction(tx);

    return tx;
  }

  // 4단계: CBDC 환수 (중개기관 → 중앙은행)
  async redeemCBDC(
    intermediaryId: string,
    amount: number
  ): Promise<Redemption> {
    // 중앙은행 장부에서 차감
    await this.centralBankLedger.debit(intermediaryId, amount);

    // 담보 해제
    const collateral = await this.unlockCollateral(intermediaryId, amount);

    // 환수 기록
    const redemption = {
      id: generateId(),
      timestamp: new Date().toISOString(),
      intermediaryId,
      amount,
      collateral,
      status: "REDEEMED"
    };

    return redemption;
  }

  private validateCollateral(collateral: Collateral, amount: number): boolean {
    // 담보 검증 로직

9. CBDC의 도전 과제

CBDC 도입은 많은 이점을 제공하지만, 동시에 해결해야 할 복잡한 문제들도 안고 있습니다:

  • 은행 예금 이탈 (Bank Disintermediation): CBDC가 매력적이면 은행 예금이 대량 이탈할 수 있어, 은행의 대출 능력이 저하되고 금융 중개 기능이 약화될 수 있습니다. 한국은행은 보유 한도 설정이나 비이자부 운영 등을 고려하고 있습니다.
  • 프라이버시 vs 투명성: 익명성을 높이면 불법 거래에 악용될 수 있고, 추적성을 강화하면 개인 프라이버시가 침해될 수 있습니다. 소액 거래는 익명 허용, 고액 거래는 실명 의무화 등의 절충안이 논의됩니다.
  • 사이버 보안: 국가 전체의 화폐 시스템이 디지털화되면 사이버 공격의 표적이 될 수 있습니다. 분산 시스템, 다층 보안, 실시간 모니터링 등이 필수적입니다.
  • 기술적 복잡성: 수천만 명의 사용자가 동시에 사용하는 시스템을 안정적으로 운영하려면 매우 높은 수준의 기술력이 필요합니다.
  • 국제 조율: 각국의 CBDC가 호환되지 않으면 국제 거래가 오히려 복잡해질 수 있습니다. WIA-DIGITAL_CURRENCY 같은 국제 표준이 중요한 이유입니다.
  • 디지털 격차: 고령층이나 디지털 취약 계층은 CBDC 사용에 어려움을 겪을 수 있어, 현금 병행 사용 기간이 필요합니다.

10. 弘益人間: CBDC로 모두를 이롭게

한국은행의 CBDC 연구는 단순히 기술적 혁신을 넘어, 弘益人間(널리 인간을 이롭게 하라)의 철학을 구현하는 것을 목표로 합니다:

🌟 CBDC의 弘益人間 비전

  • 금융 소외 해소: 은행 계좌 없이도 CBDC 지갑 하나로 안전한 금융 서비스를 이용할 수 있게 합니다. 외국인 노동자, 신용 불량자, 미성년자도 디지털 경제에 참여할 수 있습니다.
  • 결제 비용 제로화: 중앙은행이 직접 운영하므로 민간 결제 서비스의 수수료 부담이 사라집니다. 소상공인의 카드 수수료 부담이 크게 줄어듭니다.
  • 재난 지원 신속화: 코로나19 재난지원금 지급 때처럼, CBDC를 통해 국민에게 즉시 지원금을 전달할 수 있습니다.
  • 국제 송금 혁신: 국내 체류 외국인 근로자가 본국 가족에게 송금할 때, 기존 7-10%의 수수료를 1% 이하로 낮출 수 있습니다.
  • 경제 양극화 완화: 모든 국민이 동등하게 안전한 디지털 화폐를 사용함으로써, 금융 서비스 접근성의 격차를 줄입니다.

📚 핵심 요약

  • CBDC 정의: 중앙은행이 직접 발행하는 법정 디지털 화폐로, 현금과 동일한 법적 지위를 가집니다.
  • 소매형 vs 도매형: 일반인용 소매형 CBDC가 주목받으며, 한국은행도 소매형 중심으로 연구 중입니다.
  • 한국은행 파일럿: 2021-2022년 2단계 파일럿을 성공적으로 완료, 초당 2,000건 처리 성능 검증했습니다.
  • 중국 선도: 중국 e-CNY가 세계에서 가장 앞선 CBDC 프로젝트로, 2억 명 이상이 사용 중입니다.
  • 중개형 모델: 대부분 국가는 민간 은행이 고객 접점을 담당하고 중앙은행은 인프라를 운영하는 중개형 모델을 선호합니다.
  • 주요 과제: 은행 예금 이탈, 프라이버시 균형, 사이버 보안, 디지털 격차 등이 해결 과제입니다.
  • 弘益人間 구현: CBDC는 금융 포용성 확대, 거래 비용 절감, 재난 지원 신속화를 통해 모두를 이롭게 합니다.

🤔 생각해볼 질문

  1. 한국은행이 CBDC를 발행한다면, 여러분은 기존 은행 예금을 CBDC로 전환하시겠습니까? 그 이유는 무엇인가요?
  2. CBDC가 완전히 익명이라면 어떤 문제가 발생할까요? 반대로 모든 거래가 추적된다면 어떤 우려가 있을까요?
  3. 중국의 디지털 위안화가 국제 결제에 널리 사용된다면, 미국 달러의 기축통화 지위에 어떤 영향을 미칠까요?
  4. CBDC가 이자를 지급한다면(혹은 마이너스 이자를 부과한다면), 경제에 어떤 변화가 생길까요?
  5. 은행들이 CBDC 도입으로 예금이 줄어든다면, 어떻게 대출 재원을 마련해야 할까요?
