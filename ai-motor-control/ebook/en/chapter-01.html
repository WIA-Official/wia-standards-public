Chapter 1: Motor Control Fundamentals

1.1 Introduction to Electric Motors

Electric motors are the workhorses of modern automation and robotics. They convert electrical energy into mechanical motion, enabling everything from industrial robots to electric vehicles. Understanding motor fundamentals is essential for implementing effective control systems, whether using classical methods or advanced AI techniques.

The basic principle of all electric motors relies on electromagnetic force: when current flows through a conductor in a magnetic field, a force is generated. This force creates rotational motion (torque) that can be harnessed for useful work. The challenge of motor control lies in precisely regulating this electromagnetic interaction to achieve desired motion characteristics.

In the context of AI motor control, understanding these fundamentals allows us to design intelligent systems that can adapt to varying loads, optimize energy consumption, predict maintenance needs, and achieve motion precision that surpasses traditional control methods.

1.3 Motor Electrical Characteristics

Understanding motor electrical parameters is crucial for control system design. These parameters determine how the motor responds to control inputs and what performance can be achieved:

Key Electrical Parameters

Parameter Symbol Unit Description
Armature Resistance Ra Ohms (Ω) Winding resistance, causes I²R losses
Armature Inductance La Henries (H) Opposes current changes, affects dynamics
Back-EMF Constant Ke V/(rad/s) Voltage generated per unit speed
Torque Constant Kt Nm/A Torque produced per unit current
Rated Voltage Vrated Volts (V) Maximum continuous operating voltage
Rated Current Irated Amperes (A) Maximum continuous operating current

Motor Electrical Model

The DC motor can be modeled electrically as:

V = I × Ra + La × (dI/dt) + Ke × ω

Where V is applied voltage, I is armature current, and ω is angular velocity.

1.4 Motor Mechanical Characteristics

Mechanical parameters define how the motor converts electrical energy to mechanical motion:

Parameter Symbol Unit Description
Rotor Inertia Jm kg·m² Resistance to angular acceleration
Viscous Friction B Nm/(rad/s) Speed-dependent friction torque
Coulomb Friction Tc Nm Static friction torque
Rated Torque Trated Nm Maximum continuous torque
Peak Torque Tpeak Nm Maximum short-duration torque
Rated Speed ωrated rad/s or RPM Maximum continuous speed

Mechanical Dynamics

The mechanical equation of motion:

Tmotor = Jtotal × (dω/dt) + B × ω + Tload

Where Tmotor is motor torque, Jtotal is total inertia (motor + load), and Tload is external load torque.

1.5 Motor Drive Electronics

Motor drives (inverters, amplifiers) convert control signals into power to drive motors. Understanding drive topologies is essential for implementing control:

H-Bridge for DC Motors

     +Vdc
       |
   Q1-----Q3
       |   |
       +---+---- Motor ----+---+
       |   |               |   |
   Q2-----Q4               |   |
       |                   |   |
      GND                 GND

Control Logic:
- Q1 + Q4 ON: Forward rotation
- Q2 + Q3 ON: Reverse rotation
- Q1 + Q3 or Q2 + Q4: Braking
- All OFF: Coast (free-running)

Three-Phase Inverter for BLDC/PMSM

     +Vdc
       |
   Q1--Q3--Q5
    |   |   |
    +---+---+---- Motor (A, B, C phases)
    |   |   |
   Q2--Q4--Q6
       |
      GND

PWM Modulation Techniques:
- Six-step commutation (BLDC)
- Space Vector PWM (SVPWM) for PMSM
- Sinusoidal PWM (SPWM)
- Field-Oriented Control (FOC)

Encoder Signal Processing

// Quadrature Encoder Decoding (4x resolution)
class QuadratureDecoder:
    def __init__(self, ppr):
        self.ppr = ppr  # Pulses per revolution
        self.count = 0
        self.last_a = 0
        self.last_b = 0

    def update(self, a, b):
        # State transition table for 4x decoding
        state = (a << 1) | b
        last_state = (self.last_a << 1) | self.last_b

        # Determine direction and increment
        if (last_state == 0b00 and state == 0b01) or \
           (last_state == 0b01 and state == 0b11) or \
           (last_state == 0b11 and state == 0b10) or \
           (last_state == 0b10 and state == 0b00):
            self.count += 1  # Forward
        elif (last_state == 0b00 and state == 0b10) or \
             (last_state == 0b10 and state == 0b11) or \
             (last_state == 0b11 and state == 0b01) or \
             (last_state == 0b01 and state == 0b00):
            self.count -= 1  # Reverse

        self.last_a = a
        self.last_b = b
        return self.count

    def get_position_rad(self):
        return (self.count / (self.ppr * 4)) * 2 * math.pi

1.7 Control Loop Hierarchy

Modern motor control systems use a hierarchical control structure with nested loops operating at different frequencies:

Control Loop Hierarchy (Cascaded Control):

┌─────────────────────────────────────────────────────────────┐
│  Position Loop (Outer Loop) - 1-10 kHz                      │
│  ┌─────────────────────────────────────────────────────────┐│
│  │  Velocity Loop (Middle Loop) - 1-10 kHz                 ││
│  │  ┌─────────────────────────────────────────────────────┐││
│  │  │  Current/Torque Loop (Inner Loop) - 10-50 kHz       │││
│  │  │                                                      │││
│  │  │  [Current Ref] → [PI] → [PWM] → [Motor] → [Current] │││
│  │  └─────────────────────────────────────────────────────┘││
│  │  [Velocity Ref] → [PI] → [Current Ref]                  ││
│  └─────────────────────────────────────────────────────────┘│
│  [Position Ref] → [P/PID] → [Velocity Ref]                  │
└─────────────────────────────────────────────────────────────┘

Bandwidth Requirements:
- Current loop: Fastest (10-50 kHz) - controls motor physics
- Velocity loop: Medium (1-10 kHz) - smooth speed control
- Position loop: Slowest (100 Hz-1 kHz) - trajectory tracking

1.8 Digital Control Implementation

Modern motor control is implemented digitally using microcontrollers or DSPs. Key considerations include sampling rate, computational latency, and quantization:

// Basic Motor Control Loop Structure (C-like pseudocode)
#define CONTROL_FREQ_HZ  10000  // 10 kHz control loop
#define DT_SEC           (1.0 / CONTROL_FREQ_HZ)

struct MotorController {
    // Feedback
    float position_rad;
    float velocity_rad_s;
    float current_a;

    // References
    float position_ref;
    float velocity_ref;
    float current_ref;

    // PID states
    PIDController position_pid;
    PIDController velocity_pid;
    PIDController current_pid;

    // Limits
    float max_current;
    float max_velocity;
    float max_position;
    float min_position;
};

void control_loop_isr(MotorController* mc) {
    // 1. Read feedback sensors
    mc->position_rad = read_encoder();
    mc->current_a = read_current_sensor();

    // 2. Estimate velocity (from position or direct measurement)
    mc->velocity_rad_s = estimate_velocity(mc->position_rad);

    // 3. Position loop (if in position mode)
    mc->velocity_ref = pid_compute(&mc->position_pid,
                                    mc->position_ref,
                                    mc->position_rad);
    mc->velocity_ref = clamp(mc->velocity_ref, -mc->max_velocity, mc->max_velocity);

    // 4. Velocity loop
    mc->current_ref = pid_compute(&mc->velocity_pid,
                                   mc->velocity_ref,
                                   mc->velocity_rad_s);
    mc->current_ref = clamp(mc->current_ref, -mc->max_current, mc->max_current);

    // 5. Current loop
    float voltage = pid_compute(&mc->current_pid,
                                mc->current_ref,
                                mc->current_a);

    // 6. Apply PWM output
    set_pwm_duty(voltage_to_duty(voltage));
}

1.10 Introduction to AI in Motor Control

Artificial intelligence enhances motor control by enabling adaptive, predictive, and optimized operation beyond what classical control can achieve:

AI Enhancement Opportunities

  • Adaptive Control: Neural networks that learn and compensate for changing motor parameters (temperature, wear)
  • Predictive Maintenance: ML models that detect anomalies and predict failures before they occur
  • Optimal Control: Reinforcement learning for energy-efficient trajectories
  • Disturbance Rejection: AI-based observers for load and friction compensation
  • Self-Tuning: Automatic PID tuning using optimization algorithms
  • Sensor Fusion: Neural networks for improved state estimation from multiple sensors
// AI-Enhanced Motor Control Concept
class AIMotorController:
    def __init__(self):
        self.classical_controller = PIDController()
        self.neural_compensator = NeuralNetwork()
        self.anomaly_detector = AnomalyDetector()

    def control_step(self, reference, feedback):
        # Classical PID output
        pid_output = self.classical_controller.compute(reference, feedback)

        # AI compensation for nonlinearities
        ai_compensation = self.neural_compensator.predict(
            state=[reference, feedback, self.velocity, self.temperature]
        )

        # Combined output
        control_output = pid_output + ai_compensation

        # Anomaly detection for safety
        if self.anomaly_detector.is_anomaly(feedback, control_output):
            self.trigger_safe_mode()

        return control_output

Key Takeaways

  • Electric motors convert electrical energy to mechanical motion through electromagnetic force
  • Major motor types (DC, BLDC, PMSM, stepper, induction) have distinct characteristics suited for different applications
  • Key parameters include resistance, inductance, back-EMF constant, torque constant, and inertia
  • Motor drives (H-bridge, three-phase inverters) convert control signals to motor power
  • Feedback sensors (encoders, resolvers, current sensors) enable closed-loop control
  • Cascaded control loops (position → velocity → current) operate at different frequencies
  • Digital implementation requires attention to sampling rate, latency, and quantization
  • AI enhances motor control through adaptation, prediction, and optimization
