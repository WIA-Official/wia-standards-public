Chapter 2: PID Control & Classical Methods

2.1 Introduction to PID Control

Proportional-Integral-Derivative (PID) control is the most widely used control algorithm in industrial motor control systems. Despite its conceptual simplicity, PID control provides robust performance for a wide range of applications when properly tuned. Over 95% of industrial control loops use PID or its variants.

The PID controller continuously calculates an error value as the difference between a desired setpoint and a measured process variable. It applies a correction based on proportional, integral, and derivative terms, each addressing different aspects of control performance.

u(t) = Kp·e(t) + Ki·∫e(τ)dτ + Kd·(de/dt)

Where u(t) is the control output, e(t) is the error (setpoint - measured value), Kp is proportional gain, Ki is integral gain, and Kd is derivative gain.

2.2 PID Components Explained

Proportional Term (P)

The proportional term produces an output proportional to the current error. It provides the main driving force to reduce the error.

Pout = Kp × e(t)
  • High Kp: Faster response but may cause overshoot and oscillation
  • Low Kp: Slower response, larger steady-state error
  • Pure P control: Always has steady-state error for step inputs (droop)

Integral Term (I)

The integral term accumulates past errors and eliminates steady-state error. It provides the "memory" of the controller.

Iout = Ki × ∫e(τ)dτ
  • High Ki: Fast elimination of steady-state error but may cause overshoot and oscillation
  • Low Ki: Slow elimination of steady-state error
  • Integral windup: When actuator saturates, integral term can grow excessively

Derivative Term (D)

The derivative term predicts future error based on the rate of change. It provides damping and reduces overshoot.

Dout = Kd × (de/dt)
  • High Kd: Strong damping but amplifies noise
  • Low Kd: Reduced damping, more overshoot
  • Noise sensitivity: Often requires filtering or derivative on measurement

Increase Kp Decrease Increase Small change Decrease Degrade
Increase Ki Decrease Increase Increase Eliminate Degrade

2.4 Discrete-Time PID Implementation

In digital control systems, the PID algorithm must be discretized. The most common approach is the backward Euler method:

// Discrete PID Controller Implementation
class PIDController {
private:
    float kp, ki, kd;           // Gains
    float integral;              // Integral accumulator
    float prev_error;            // Previous error for derivative
    float output_min, output_max;// Output limits
    float integral_min, integral_max; // Anti-windup limits
    float dt;                    // Sample time in seconds

public:
    PIDController(float p, float i, float d, float sample_time) {
        kp = p;
        ki = i;
        kd = d;
        dt = sample_time;
        integral = 0.0f;
        prev_error = 0.0f;
        output_min = -1e9f;
        output_max = 1e9f;
        integral_min = -1e9f;
        integral_max = 1e9f;
    }

    void set_limits(float out_min, float out_max) {
        output_min = out_min;
        output_max = out_max;
        // Anti-windup: limit integral to prevent saturation
        integral_max = out_max / ki;
        integral_min = out_min / ki;
    }

    float compute(float setpoint, float measurement) {
        // Calculate error
        float error = setpoint - measurement;

        // Proportional term
        float p_term = kp * error;

        // Integral term with anti-windup
        integral += error * dt;
        integral = clamp(integral, integral_min, integral_max);
        float i_term = ki * integral;

        // Derivative term (on measurement to avoid derivative kick)
        float derivative = (measurement - prev_measurement) / dt;
        float d_term = -kd * derivative;  // Negative because derivative on measurement

        prev_measurement = measurement;

        // Calculate output
        float output = p_term + i_term + d_term;

        // Clamp output
        output = clamp(output, output_min, output_max);

        return output;
    }

    void reset() {
        integral = 0.0f;
        prev_error = 0.0f;
    }

private:
    float prev_measurement = 0.0f;

    float clamp(float value, float min_val, float max_val) {
        if (value < min_val) return min_val;
        if (value > max_val) return max_val;
        return value;
    }
};

2.5 PID Tuning Methods

Proper tuning is critical for PID performance. Several methods exist for determining optimal gains:

Ziegler-Nichols Method (Closed-Loop)

  1. Set Ki = 0 and Kd = 0
  2. Increase Kp until system oscillates with constant amplitude
  3. Record critical gain Ku and oscillation period Tu
  4. Calculate gains using table:
Control Type Kp Ki Kd
P 0.5 × Ku - -
PI 0.45 × Ku 0.54 × Ku / Tu -
PID 0.6 × Ku 1.2 × Ku / Tu 0.075 × Ku × Tu

Cohen-Coon Method (Open-Loop)

Based on step response characterization:

  1. Apply step input to open-loop system
  2. Measure: Process gain K, Dead time L, Time constant T
  3. Calculate gains based on response characteristics
// Cohen-Coon Tuning Calculator
struct ProcessParameters {
    float K;    // Process gain
    float L;    // Dead time (delay)
    float T;    // Time constant
};

struct PIDGains cohen_coon_tuning(ProcessParameters proc, char control_type) {
    PIDGains gains;
    float ratio = L / T;

    switch(control_type) {
        case 'P':
            gains.kp = (1.03 + 0.35 * ratio) * T / (K * L);
            gains.ki = 0;
            gains.kd = 0;
            break;

        case 'I':  // PI
            gains.kp = (0.9 + 0.083 * ratio) * T / (K * L);
            gains.ki = gains.kp / (L * (0.9 + 0.083 * ratio) / (1.27 + 0.6 * ratio));
            gains.kd = 0;
            break;

        case 'D':  // PID
            gains.kp = (1.35 + 0.25 * ratio) * T / (K * L);
            float ti = L * (0.54 * ratio + 0.33) / (1.35 + 0.25 * ratio);
            float td = L * 0.5 / (1.35 + 0.25 * ratio);
            gains.ki = gains.kp / ti;
            gains.kd = gains.kp * td;
            break;
    }

    return gains;
}

Auto-Tuning with Relay Feedback

// Relay Auto-Tuning Algorithm
class RelayAutoTuner {
private:
    float relay_amplitude;
    float hysteresis;
    bool relay_state;
    float peak_value, valley_value;
    float period_start_time;
    int oscillation_count;

public:
    RelayAutoTuner(float amplitude, float hyst) {
        relay_amplitude = amplitude;
        hysteresis = hyst;
        relay_state = true;
        oscillation_count = 0;
    }

    float relay_output(float error, float current_time) {
        // Relay with hysteresis
        if (relay_state && error < -hysteresis) {
            relay_state = false;
            valley_value = error;
        } else if (!relay_state && error > hysteresis) {
            relay_state = true;
            peak_value = error;
            oscillation_count++;

            if (oscillation_count == 1) {
                period_start_time = current_time;
            }
        }

        return relay_state ? relay_amplitude : -relay_amplitude;
    }

    bool is_complete() {
        return oscillation_count >= 4;  // Need enough cycles
    }

    PIDGains calculate_gains(float elapsed_time) {
        // Ultimate gain from describing function analysis
        float oscillation_amplitude = (peak_value - valley_value) / 2.0f;
        float Ku = 4.0f * relay_amplitude / (PI * oscillation_amplitude);

        // Ultimate period
        float Tu = elapsed_time / (oscillation_count - 1);

        // Ziegler-Nichols PID
        PIDGains gains;
        gains.kp = 0.6f * Ku;
        gains.ki = 1.2f * Ku / Tu;
        gains.kd = 0.075f * Ku * Tu;

        return gains;
    }
};

2.6 Anti-Windup Techniques

Integral windup occurs when the actuator saturates but the integral term continues accumulating error. Several techniques prevent this:

Clamping (Conditional Integration)

// Stop integration when output saturates
if (output < output_max && output > output_min) {
    integral += error * dt;
} else {
    // Don't accumulate when saturated
}

Back-Calculation

// Discharge integral based on saturation difference
float output_unsat = p_term + i_term + d_term;
float output_sat = clamp(output_unsat, output_min, output_max);
float saturation_error = output_sat - output_unsat;

// Back-calculate integral correction
integral += (error + saturation_error / kb) * dt;
// kb is back-calculation gain, typically kb = sqrt(ki * kd)

2.7 Derivative Filtering

Pure derivative is sensitive to noise. A first-order filter is commonly applied:

D(s) = Kd × s / (1 + τds)
// Filtered Derivative Implementation
class FilteredDerivative {
private:
    float tau;           // Filter time constant
    float filtered_deriv; // Filtered derivative value
    float prev_measurement;
    float dt;

public:
    FilteredDerivative(float filter_tau, float sample_time) {
        tau = filter_tau;
        dt = sample_time;
        filtered_deriv = 0.0f;
        prev_measurement = 0.0f;
    }

    float compute(float measurement, float kd) {
        // Raw derivative
        float raw_deriv = (measurement - prev_measurement) / dt;
        prev_measurement = measurement;

        // First-order low-pass filter
        float alpha = dt / (tau + dt);
        filtered_deriv = (1.0f - alpha) * filtered_deriv + alpha * raw_deriv;

        return kd * filtered_deriv;
    }
};

2.8 Cascaded PID Control

Motor control typically uses cascaded (nested) loops where the output of one controller becomes the setpoint of another:

// Cascaded Position-Velocity-Current Control
class CascadedMotorController {
private:
    PIDController position_controller;
    PIDController velocity_controller;
    PIDController current_controller;

    float vel_limit;
    float current_limit;

public:
    CascadedMotorController()
        : position_controller(10.0f, 0.0f, 0.5f, 0.001f),   // P, I, D, dt
          velocity_controller(5.0f, 20.0f, 0.0f, 0.001f),
          current_controller(2.0f, 100.0f, 0.0f, 0.0001f)
    {
        vel_limit = 100.0f;      // rad/s
        current_limit = 10.0f;   // Amps
    }

    float control(float pos_ref, float pos_fb, float vel_fb, float curr_fb) {
        // Position loop -> velocity reference
        float vel_ref = position_controller.compute(pos_ref, pos_fb);
        vel_ref = clamp(vel_ref, -vel_limit, vel_limit);

        // Velocity loop -> current reference
        float curr_ref = velocity_controller.compute(vel_ref, vel_fb);
        curr_ref = clamp(curr_ref, -current_limit, current_limit);

        // Current loop -> voltage output
        float voltage = current_controller.compute(curr_ref, curr_fb);

        return voltage;
    }

    // Allow direct velocity mode (bypass position loop)
    float velocity_control(float vel_ref, float vel_fb, float curr_fb) {
        vel_ref = clamp(vel_ref, -vel_limit, vel_limit);
        float curr_ref = velocity_controller.compute(vel_ref, vel_fb);
        curr_ref = clamp(curr_ref, -current_limit, current_limit);
        return current_controller.compute(curr_ref, curr_fb);
    }

    // Allow direct torque/current mode
    float torque_control(float curr_ref, float curr_fb) {
        curr_ref = clamp(curr_ref, -current_limit, current_limit);
        return current_controller.compute(curr_ref, curr_fb);
    }
};

2.9 Feed-Forward Control

Feed-forward compensation improves tracking by adding a model-based term that anticipates the required control effort:

// PID with Feed-Forward
class PIDWithFeedForward {
private:
    PIDController pid;
    float kff_velocity;   // Velocity feed-forward gain
    float kff_accel;      // Acceleration feed-forward gain
    float kt_inv;         // Inverse torque constant (1/Kt)
    float inertia;        // Total inertia

public:
    float compute(float pos_ref, float pos_fb,
                  float vel_ref, float accel_ref) {
        // Feedback term from PID
        float feedback_term = pid.compute(pos_ref, pos_fb);

        // Velocity feed-forward (back-EMF compensation)
        float vel_ff = kff_velocity * vel_ref;

        // Acceleration feed-forward (inertia compensation)
        // Torque = J * acceleration, Current = Torque / Kt
        float accel_ff = kff_accel * inertia * kt_inv * accel_ref;

        // Combined output
        return feedback_term + vel_ff + accel_ff;
    }
};

2.10 Advanced PID Variants

PI-D Controller (Derivative on Measurement)

Avoids derivative kick by computing derivative on measurement instead of error:

u(t) = Kp·e(t) + Ki·∫e(τ)dτ - Kd·(dy/dt)

I-PD Controller (Proportional and Derivative on Measurement)

Further reduces overshoot for setpoint changes:

u(t) = Ki·∫e(τ)dτ - Kp·y - Kd·(dy/dt)

Two-Degrees-of-Freedom PID

Separate tuning for setpoint tracking and disturbance rejection:

// 2-DOF PID Controller
float compute_2dof(float setpoint, float measurement) {
    float error = setpoint - measurement;

    // Setpoint weighting factors (0-1)
    float b = 0.7;  // Proportional setpoint weight
    float c = 0.0;  // Derivative setpoint weight

    // Modified error terms
    float p_error = b * setpoint - measurement;  // Weighted for P
    float i_error = error;                        // Full error for I
    float d_error = c * setpoint - measurement;  // Weighted for D

    float p_term = kp * p_error;
    float i_term = ki * integral;  // integral of i_error
    float d_term = kd * derivative_of(d_error);

    return p_term + i_term + d_term;
}

Key Takeaways

  • PID control combines proportional (current error), integral (accumulated error), and derivative (rate of change) terms
  • Proportional gain determines response speed; integral eliminates steady-state error; derivative provides damping
  • Discrete implementation requires attention to sample time, anti-windup, and derivative filtering
  • Ziegler-Nichols and Cohen-Coon provide systematic tuning methods
  • Anti-windup techniques prevent integral term from growing excessively during saturation
  • Cascaded control (position → velocity → current) is the standard architecture for motor control
  • Feed-forward compensation improves tracking by anticipating required control effort
  • Advanced variants (PI-D, I-PD, 2-DOF) offer better performance for specific applications
