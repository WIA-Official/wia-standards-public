1장: 모터 제어 기초

1.1 전기 모터 개요

전기 모터는 현대 자동화 및 로봇 공학의 핵심 동력원입니다. 전기 에너지를 기계적 운동으로 변환하여 산업용 로봇부터 전기 자동차에 이르기까지 다양한 응용 분야에서 활용됩니다. 모터 기초를 이해하는 것은 고전적 방법이든 고급 AI 기술이든 효과적인 제어 시스템을 구현하는 데 필수적입니다.

모든 전기 모터의 기본 원리는 전자기력에 기반합니다. 자기장 내의 도체에 전류가 흐르면 힘이 발생하고, 이 힘이 회전 운동(토크)을 생성하여 유용한 작업을 수행할 수 있습니다. 모터 제어의 핵심은 원하는 운동 특성을 달성하기 위해 이러한 전자기 상호작용을 정밀하게 조절하는 것입니다.

AI 모터 제어의 맥락에서 이러한 기초를 이해하면 변화하는 부하에 적응하고, 에너지 소비를 최적화하며, 유지보수 필요성을 예측하고, 전통적인 제어 방법을 능가하는 운동 정밀도를 달성할 수 있는 지능형 시스템을 설계할 수 있습니다.

1.2 전기 모터의 종류

다양한 모터 유형은 각각 고유한 특성을 가지며 다양한 응용 분야에 적합합니다. WIA-AI-MOTOR-CONTROL 표준은 모든 주요 모터 유형을 지원합니다:

모터 유형 작동 원리 주요 특성 일반적인 응용
DC 브러시 모터 정류자 기반 전류 전환 단순한 제어, 선형 토크-속도 특성, 브러시 마모 소형 로봇, 장난감, 저비용 시스템
브러시리스 DC (BLDC) 홀 센서를 통한 전자 정류 높은 효율, 긴 수명, 드라이버 필요 드론, 전기차, 산업 자동화
영구자석 동기 모터 (PMSM) 계자 배향을 통한 정현파 AC 구동 최고 효율, 부드러운 토크, 복잡한 제어 정밀 로봇, CNC 기계
스테퍼 모터 펄스당 이산적인 각도 스텝 개루프 위치 결정, 유지 토크, 공진 3D 프린터, CNC, 위치 결정 시스템
AC 유도 모터 회전 자기장이 회전자 전류 유도 견고함, 저비용, 슬립 특성 펌프, 팬, 컨베이어, HVAC
서보 모터 통합 피드백이 있는 BLDC/PMSM 고정밀, 빠른 응답, 폐루프 로봇 공학, 자동화, 모션 제어

1.3 모터 전기적 특성

모터 전기적 파라미터를 이해하는 것은 제어 시스템 설계에 매우 중요합니다. 이러한 파라미터는 모터가 제어 입력에 어떻게 반응하고 어떤 성능을 달성할 수 있는지를 결정합니다:

주요 전기적 파라미터

파라미터 기호 단위 설명
전기자 저항 Ra 옴 (Ω) 권선 저항, I²R 손실 발생
전기자 인덕턴스 La 헨리 (H) 전류 변화에 대한 저항, 동특성에 영향
역기전력 상수 Ke V/(rad/s) 단위 속도당 발생 전압
토크 상수 Kt Nm/A 단위 전류당 발생 토크
정격 전압 Vrated 볼트 (V) 최대 연속 운전 전압
정격 전류 Irated 암페어 (A) 최대 연속 운전 전류

모터 전기적 모델

DC 모터는 전기적으로 다음과 같이 모델링할 수 있습니다:

V = I × Ra + La × (dI/dt) + Ke × ω

여기서 V는 인가 전압, I는 전기자 전류, ω는 각속도입니다.

1.4 모터 기계적 특성

기계적 파라미터는 모터가 전기 에너지를 기계적 운동으로 변환하는 방식을 정의합니다:

파라미터 기호 단위 설명
회전자 관성 Jm kg·m² 각가속도에 대한 저항
점성 마찰 B Nm/(rad/s) 속도에 비례하는 마찰 토크
쿨롱 마찰 Tc Nm 정지 마찰 토크
정격 토크 Trated Nm 최대 연속 토크
최대 토크 Tpeak Nm 최대 순간 토크
정격 속도 ωrated rad/s 또는 RPM 최대 연속 속도

기계적 동역학

운동의 기계적 방정식:

Tmotor = Jtotal × (dω/dt) + B × ω + Tload

여기서 Tmotor는 모터 토크, Jtotal은 총 관성(모터 + 부하), Tload는 외부 부하 토크입니다.

1.5 모터 드라이브 전자장치

모터 드라이브(인버터, 앰프)는 제어 신호를 모터 구동 전력으로 변환합니다. 드라이브 토폴로지를 이해하는 것은 제어 구현에 필수적입니다:

DC 모터용 H-브리지

     +Vdc
       |
   Q1-----Q3
       |   |
       +---+---- 모터 ----+---+
       |   |               |   |
   Q2-----Q4               |   |
       |                   |   |
      GND                 GND

제어 로직:
- Q1 + Q4 ON: 정방향 회전
- Q2 + Q3 ON: 역방향 회전
- Q1 + Q3 또는 Q2 + Q4: 제동
- 모두 OFF: 코스트 (자유 회전)

BLDC/PMSM용 3상 인버터

     +Vdc
       |
   Q1--Q3--Q5
    |   |   |
    +---+---+---- 모터 (A, B, C 상)
    |   |   |
   Q2--Q4--Q6
       |
      GND

PWM 변조 기법:
- 6-스텝 정류 (BLDC)
- 공간 벡터 PWM (SVPWM) - PMSM용
- 정현파 PWM (SPWM)
- 계자 배향 제어 (FOC)

1.6 피드백 센서

정확한 피드백은 폐루프 모터 제어에 필수적입니다. 다양한 센서가 위치, 속도 및 전류 정보를 제공합니다:

센서 유형 측정값 분해능 특성
증분형 엔코더 상대 위치/속도 500-10000+ PPR 원점 복귀 필요, A/B/Z 직교 신호
절대형 엔코더 절대 위치 12-23 비트 원점 복귀 불필요, 높은 비용
레졸버 절대 위치 12-16 비트 유효 견고함, 아날로그, RDC 필요
홀 효과 센서 대략적인 위치 (60°/120°) 회전당 6 상태 BLDC 정류, 저비용
전류 센서 상 전류 12-16 비트 ADC 홀 효과, 션트 저항, CT
타코미터 속도 아날로그 또는 디지털 직접 속도 측정

엔코더 신호 처리

// 직교 엔코더 디코딩 (4배 분해능)
class QuadratureDecoder:
    def __init__(self, ppr):
        self.ppr = ppr  # 회전당 펄스 수
        self.count = 0
        self.last_a = 0
        self.last_b = 0

    def update(self, a, b):
        # 4배 디코딩을 위한 상태 전이 테이블
        state = (a << 1) | b
        last_state = (self.last_a << 1) | self.last_b

        # 방향 결정 및 증가
        if (last_state == 0b00 and state == 0b01) or \
           (last_state == 0b01 and state == 0b11) or \
           (last_state == 0b11 and state == 0b10) or \
           (last_state == 0b10 and state == 0b00):
            self.count += 1  # 정방향
        elif (last_state == 0b00 and state == 0b10) or \
             (last_state == 0b10 and state == 0b11) or \
             (last_state == 0b11 and state == 0b01) or \
             (last_state == 0b01 and state == 0b00):
            self.count -= 1  # 역방향

        self.last_a = a
        self.last_b = b
        return self.count

    def get_position_rad(self):
        return (self.count / (self.ppr * 4)) * 2 * math.pi

1.7 제어 루프 계층 구조

현대 모터 제어 시스템은 서로 다른 주파수로 동작하는 중첩된 루프를 가진 계층적 제어 구조를 사용합니다:

제어 루프 계층 구조 (캐스케이드 제어):

┌─────────────────────────────────────────────────────────────┐
│  위치 루프 (외부 루프) - 1-10 kHz                            │
│  ┌─────────────────────────────────────────────────────────┐│
│  │  속도 루프 (중간 루프) - 1-10 kHz                        ││
│  │  ┌─────────────────────────────────────────────────────┐││
│  │  │  전류/토크 루프 (내부 루프) - 10-50 kHz              │││
│  │  │                                                      │││
│  │  │  [전류 기준] → [PI] → [PWM] → [모터] → [전류]        │││
│  │  └─────────────────────────────────────────────────────┘││
│  │  [속도 기준] → [PI] → [전류 기준]                        ││
│  └─────────────────────────────────────────────────────────┘│
│  [위치 기준] → [P/PID] → [속도 기준]                         │
└─────────────────────────────────────────────────────────────┘

대역폭 요구사항:
- 전류 루프: 가장 빠름 (10-50 kHz) - 모터 물리학 제어
- 속도 루프: 중간 (1-10 kHz) - 부드러운 속도 제어
- 위치 루프: 가장 느림 (100 Hz-1 kHz) - 궤적 추종

1.8 디지털 제어 구현

현대 모터 제어는 마이크로컨트롤러 또는 DSP를 사용하여 디지털로 구현됩니다. 주요 고려사항에는 샘플링 속도, 계산 지연 및 양자화가 포함됩니다:

// 기본 모터 제어 루프 구조 (C 유사 의사 코드)
#define CONTROL_FREQ_HZ  10000  // 10 kHz 제어 루프
#define DT_SEC           (1.0 / CONTROL_FREQ_HZ)

struct MotorController {
    // 피드백
    float position_rad;
    float velocity_rad_s;
    float current_a;

    // 기준값
    float position_ref;
    float velocity_ref;
    float current_ref;

    // PID 상태
    PIDController position_pid;
    PIDController velocity_pid;
    PIDController current_pid;

    // 제한값
    float max_current;
    float max_velocity;
    float max_position;
    float min_position;
};

void control_loop_isr(MotorController* mc) {
    // 1. 피드백 센서 읽기
    mc->position_rad = read_encoder();
    mc->current_a = read_current_sensor();

    // 2. 속도 추정 (위치로부터 또는 직접 측정)
    mc->velocity_rad_s = estimate_velocity(mc->position_rad);

    // 3. 위치 루프 (위치 모드인 경우)
    mc->velocity_ref = pid_compute(&mc->position_pid,
                                    mc->position_ref,
                                    mc->position_rad);
    mc->velocity_ref = clamp(mc->velocity_ref, -mc->max_velocity, mc->max_velocity);

    // 4. 속도 루프
    mc->current_ref = pid_compute(&mc->velocity_pid,
                                   mc->velocity_ref,
                                   mc->velocity_rad_s);
    mc->current_ref = clamp(mc->current_ref, -mc->max_current, mc->max_current);

    // 5. 전류 루프
    float voltage = pid_compute(&mc->current_pid,
                                mc->current_ref,
                                mc->current_a);

    // 6. PWM 출력 적용
    set_pwm_duty(voltage_to_duty(voltage));
}

1.10 모터 제어에서의 AI 소개

인공지능은 고전적 제어가 달성할 수 있는 것 이상으로 적응적, 예측적, 최적화된 운전을 가능하게 하여 모터 제어를 향상시킵니다:

AI 향상 기회

  • 적응 제어: 변화하는 모터 파라미터(온도, 마모)를 학습하고 보상하는 신경망
  • 예측 유지보수: 발생 전에 이상을 감지하고 고장을 예측하는 ML 모델
  • 최적 제어: 에너지 효율적인 궤적을 위한 강화 학습
  • 외란 억제: 부하 및 마찰 보상을 위한 AI 기반 관측기
  • 자동 튜닝: 최적화 알고리즘을 사용한 자동 PID 튜닝
  • 센서 융합: 여러 센서에서 개선된 상태 추정을 위한 신경망
// AI 향상 모터 제어 개념
class AIMotorController:
    def __init__(self):
        self.classical_controller = PIDController()
        self.neural_compensator = NeuralNetwork()
        self.anomaly_detector = AnomalyDetector()

    def control_step(self, reference, feedback):
        # 고전적 PID 출력
        pid_output = self.classical_controller.compute(reference, feedback)

        # 비선형성에 대한 AI 보상
        ai_compensation = self.neural_compensator.predict(
            state=[reference, feedback, self.velocity, self.temperature]
        )

        # 결합된 출력
        control_output = pid_output + ai_compensation

        # 안전을 위한 이상 감지
        if self.anomaly_detector.is_anomaly(feedback, control_output):
            self.trigger_safe_mode()

        return control_output

핵심 내용

  • 전기 모터는 전자기력을 통해 전기 에너지를 기계적 운동으로 변환합니다
  • 주요 모터 유형(DC, BLDC, PMSM, 스테퍼, 유도)은 다양한 응용에 적합한 고유한 특성을 가집니다
  • 주요 파라미터에는 저항, 인덕턴스, 역기전력 상수, 토크 상수 및 관성이 포함됩니다
  • 모터 드라이브(H-브리지, 3상 인버터)는 제어 신호를 모터 전력으로 변환합니다
  • 피드백 센서(엔코더, 레졸버, 전류 센서)는 폐루프 제어를 가능하게 합니다
  • 캐스케이드 제어 루프(위치 → 속도 → 전류)는 서로 다른 주파수로 동작합니다
  • 디지털 구현은 샘플링 속도, 지연 및 양자화에 주의가 필요합니다
  • AI는 적응, 예측 및 최적화를 통해 모터 제어를 향상시킵니다
