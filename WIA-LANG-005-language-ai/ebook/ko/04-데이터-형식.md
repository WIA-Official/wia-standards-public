# 제4장: 데이터 형식 명세

## Phase 1: 표준화된 데이터 구조

언어 AI 시스템의 성능은 데이터의 품질과 구조에 크게 의존합니다. WIA-LANG-005 Phase 1은 훈련 데이터, 모델, 평가 데이터셋을 위한 표준화된 형식을 정의하여 재현성, 공유성, 상호운용성을 보장합니다.

## 메타데이터 스키마

### 품질 보증

**데이터 검증 스크립트**:

```python
class WIALang005Validator:
    """WIA-LANG-005 데이터 형식 검증기"""

    def validate_dataset(self, dataset_path):
        """데이터셋 검증"""
        errors = []

        # 1. 스키마 검증
        if not self.validate_schema(dataset_path):
            errors.append("Schema validation failed")

        # 2. 메타데이터 검증
        metadata = self.load_metadata(dataset_path)
        if not self.validate_metadata(metadata):
            errors.append("Invalid metadata")

        # 3. 언어 코드 검증
        if not self.validate_language_code(metadata['language']['code']):
            errors.append("Invalid ISO 639-3 language code")

        # 4. 데이터 무결성 검증
        if not self.validate_data_integrity(dataset_path):
            errors.append("Data integrity check failed")

        # 5. 파일 체크섬 검증
        if not self.validate_checksums(dataset_path):
            errors.append("Checksum mismatch")

        return len(errors) == 0, errors

    def validate_schema(self, data):
        """JSON 스키마 검증"""
        from jsonschema import validate
        schema = self.load_wia_schema()
        try:
            validate(instance=data, schema=schema)
            return True
        except Exception as e:
            return False
```

## 핵심 요약

**주요 내용:**

1. **메타데이터 표준화**: 언어 정보, 데이터셋 정보, 품질 메트릭을 포함한 포괄적 메타데이터 스키마를 정의합니다.

2. **작업별 형식**: 기계 번역, ASR, TTS, 언어 모델링, NER 등 각 작업에 특화된 데이터 형식을 명시합니다.

3. **모델 형식**: HuggingFace 호환 형식, 어댑터 기반 미세 조정, ONNX 배포 형식을 지원합니다.

4. **품질 보증**: 데이터 검증 도구와 벤치마크 구조로 품질을 보장합니다.

5. **데이터 증강**: 텍스트 및 오디오 증강 기법을 표준화하여 데이터 효율성을 높입니다.

6. **프라이버시 및 문화적 민감성**: 개인정보 보호와 문화적 금기 사항 관리를 내장합니다.

4. 데이터 증강이 왜 저자원 언어 NLP에 특히 중요한지 설명하고, 텍스트와 오디오에 적용 가능한 증강 기법을 각각 3가지 이상 제시하시오.

5. 문화적으로 민감한 언어 데이터(예: 신성한 의례 언어)를 다룰 때 필요한 접근 제어 및 윤리적 고려사항을 설명하시오.
