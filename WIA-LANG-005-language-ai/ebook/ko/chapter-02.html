2.1 데이터 수집의 현실적 어려움

저자원 언어가 직면한 가장 근본적인 문제는 디지털 데이터의 부족입니다. 현대 AI 모델은 방대한 양의 데이터를 필요로 하지만, 대부분의 위기 언어는 이를 충족할 수 없습니다.

2.3 모델링 과제

// 과적합 완화 전략
class LowResourceTrainer {
  applyRegularization(config: TrainingConfig): TrainingConfig {
    return {
      ...config,
      dropout: 0.3,              // 뉴런 비활성화
      l2Penalty: 0.01,           // L2 정규화
      earlyStoppingPatience: 5,  // 조기 종료
      dataAugmentation: true,    // 데이터 증강
      transferLearning: true     // 전이 학습
    };
  }

  // 온도 기반 균형 샘플링
  calculateSamplingProbs(
    datasets: LanguageDataset[],
    temperature: number = 5
  ): Map<string, number> {
    const probs = new Map<string, number>();
    let total = 0;

    for (const ds of datasets) {
      const adjusted = Math.pow(ds.size, 1 / temperature);
      probs.set(ds.language, adjusted);
      total += adjusted;
    }

    // 정규화
    for (const [lang, prob] of probs) {
      probs.set(lang, prob / total);
    }

    return probs;
  }
}

2.4 사회문화적 도전

🚨 기술 식민주의의 패턴:
  1. 외국 연구팀이 방문
  2. 언어 데이터 수집
  3. 본국으로 가져가 연구
  4. 논문 발표, 학위 취득
  5. 공동체는 아무 것도 못 받음
ℹ️ 성공적 협력 원칙: "Nothing About Us Without Us"
  • 공동체가 의사결정 참여
  • 데이터 사용 승인 권한
  • 개발 도구 무상 제공
  • 경제적 이익 공유

📚 핵심 요약

  1. 데이터 희소성: 저자원 언어는 1:1,000 ~ 1:10,000 수준의 데이터 격차가 있습니다.
  2. 기술적 장벽: 형태론적 복잡성, 음운론적 다양성, 문자 체계 미지원이 과제입니다.
  3. 모델링 과제: 과적합, 언어 간 간섭, 제로샷 학습의 한계를 극복해야 합니다.
  4. 인프라 불평등: 컴퓨팅 자원과 전문가 부족이 개발을 제약합니다.
  5. 사회문화적 도전: 언어 공동체와의 신뢰 구축이 기술만큼 중요합니다.
  6. 윤리적 고려: 기술 식민주의 방지, 데이터 주권 보장이 핵심입니다.
  7. 弘益人間: 과제를 인식해야 모든 언어 공동체를 이롭게 할 수 있습니다.
