제6장: 3단계 - 통신 프로토콜

弘益人間 (홍익인간) - 널리 인간을 이롭게 하라

6.1 통신 프로토콜 개요

WIA-ROB-011은 다양한 환경과 요구사항에 맞춰 여러 통신 프로토콜을 지원합니다. 각 프로토콜은 특정 사용 사례에 최적화되어 있으며, 필요에 따라 조합하여 사용할 수 있습니다.

프로토콜 용도 특징 권장 사용
WebSocket 실시간 양방향 통신 저지연, 지속 연결 상태 모니터링, 원격 제어
MQTT IoT 메시징 경량, Pub/Sub 센서 데이터, 이벤트
CoAP 제약 환경 초경량, UDP 저전력 기기
HTTP/2 일반 통신 멀티플렉싱, 압축 API 호출

6.2 WebSocket 프로토콜

WebSocket은 클라이언트와 서버 간 지속적인 양방향 통신 채널을 제공합니다. 실시간 상태 업데이트와 원격 제어에 이상적입니다.

6.2.1 연결 설정

// WebSocket 연결
wss://robot.example.com/ws/v1/robots/CR-2025-001

// 연결 헤더
Upgrade: websocket
Connection: Upgrade
Sec-WebSocket-Key: dGhlIHNhbXBsZSBub25jZQ==
Sec-WebSocket-Version: 13
Authorization: Bearer eyJhbGc...

// 서버 응답
HTTP/1.1 101 Switching Protocols
Upgrade: websocket
Connection: Upgrade
Sec-WebSocket-Accept: s3pPLMBiTxaQ9kYGzzhZRbK+xOo=

6.2.2 메시지 형식

// 클라이언트 → 서버 (명령)
{
  "type": "command",
  "id": "cmd_001",
  "timestamp": "2025-12-29T10:30:00Z",
  "command": "start_cleaning",
  "parameters": {
    "mode": "auto",
    "rooms": ["living_room"]
  }
}

// 서버 → 클라이언트 (응답)
{
  "type": "response",
  "request_id": "cmd_001",
  "timestamp": "2025-12-29T10:30:01Z",
  "status": "success",
  "data": {
    "task_id": "task_001",
    "estimated_duration": 1800
  }
}

// 서버 → 클라이언트 (상태 업데이트)
{
  "type": "status_update",
  "timestamp": "2025-12-29T10:30:15Z",
  "data": {
    "state": "cleaning",
    "battery": 85,
    "location": {"x": 1250, "y": 3400},
    "progress": 12
  }
}

6.2.3 하트비트 (Heartbeat)

연결 상태를 확인하기 위해 주기적으로 핑-퐁 메시지를 교환합니다. 

// 클라이언트 → 서버 (30초마다)
{
  "type": "ping",
  "timestamp": "2025-12-29T10:30:00Z"
}

// 서버 → 클라이언트
{
  "type": "pong",
  "timestamp": "2025-12-29T10:30:00Z"
}

// 응답 없으면 재연결 시도

6.3 MQTT 프로토콜

MQTT는 경량 메시징 프로토콜로, IoT 환경에서 효율적인 Publish/Subscribe 패턴을 제공합니다.

6.3.1 토픽 구조

// 토픽 네이밍 규칙
wia-rob-011/{version}/{robot_id}/{category}/{subcategory}

// 예시
wia-rob-011/v1/CR-2025-001/status/battery
wia-rob-011/v1/CR-2025-001/status/location
wia-rob-011/v1/CR-2025-001/commands/start
wia-rob-011/v1/CR-2025-001/events/cleaning_complete
wia-rob-011/v1/CR-2025-001/sensors/dust_level

6.3.2 주요 토픽

토픽 방향 QoS 설명
status/# Robot → Cloud 0 상태 정보
commands/# Cloud → Robot 1 제어 명령
events/# Robot → Cloud 1 이벤트 알림
sensors/# Robot → Cloud 0 센서 데이터
config/# 양방향 2 설정 변경

6.3.3 QoS 레벨 선택

  • QoS 0 (At most once): 실시간 센서 데이터, 상태 정보 - 지연 최소화
  • QoS 1 (At least once): 명령, 이벤트 - 중요하지만 중복 허용
  • QoS 2 (Exactly once): 설정 변경, 펌웨어 업데이트 - 중복 불가

6.3.4 메시지 예시

// 배터리 상태 게시 (QoS 0)
Topic: wia-rob-011/v1/CR-2025-001/status/battery
Payload:
{
  "timestamp": "2025-12-29T10:30:00Z",
  "level": 85,
  "charging": false,
  "voltage": 14.8,
  "health": 95
}

// 청소 시작 명령 (QoS 1)
Topic: wia-rob-011/v1/CR-2025-001/commands/start
Payload:
{
  "command_id": "cmd_001",
  "timestamp": "2025-12-29T10:30:00Z",
  "mode": "auto",
  "rooms": ["living_room"]
}

// 청소 완료 이벤트 (QoS 1)
Topic: wia-rob-011/v1/CR-2025-001/events/cleaning_complete
Payload:
{
  "event_id": "evt_001",
  "timestamp": "2025-12-29T11:30:00Z",
  "session_id": "clean_001",
  "duration": 3600,
  "area_cleaned": 85.3
}

6.4 CoAP 프로토콜

CoAP(Constrained Application Protocol)은 제약된 환경(저전력, 저대역폭)에서 사용하는 경량 프로토콜입니다.

6.4.1 CoAP 요청

// 상태 조회
coap://robot.local:5683/status

Method: GET
Type: CON (Confirmable)
Token: 0x1234

Response:
{
  "state": "idle",
  "battery": 95
}

// 청소 시작
coap://robot.local:5683/commands/start

Method: POST
Type: CON
Payload:
{
  "mode": "auto"
}

6.4.2 관찰 (Observe) 패턴

// 상태 변화 관찰
GET /status
Observe: 0

// 서버는 상태 변경 시마다 알림
Notification 1:
{
  "state": "cleaning",
  "battery": 90
}

Notification 2:
{
  "state": "cleaning",
  "battery": 85
}

6.5 보안 통신

모든 통신은 암호화되어야 하며, 강력한 인증 메커니즘을 사용해야 합니다.

6.5.1 전송 계층 보안

프로토콜 보안 버전 암호화 최소 버전
WebSocket WSS (WebSocket Secure) TLS 1.3 TLS 1.2
MQTT MQTTS TLS 1.3 TLS 1.2
CoAP CoAPS (DTLS) DTLS 1.3 DTLS 1.2
HTTP HTTPS TLS 1.3 TLS 1.2

6.5.2 인증서 관리

// TLS 설정 예시
{
  "tls": {
    "version": "1.3",
    "cipher_suites": [
      "TLS_AES_256_GCM_SHA384",
      "TLS_CHACHA20_POLY1305_SHA256"
    ],
    "certificates": {
      "robot_cert": "/path/to/robot.crt",
      "robot_key": "/path/to/robot.key",
      "ca_cert": "/path/to/ca.crt"
    },
    "verify_peer": true,
    "verify_hostname": true
  }
}

6.5.3 메시지 수준 보안

전송 계층 보안 외에도 메시지 자체를 암호화하고 서명할 수 있습니다. 

// 메시지 서명
{
  "type": "command",
  "command": "start_cleaning",
  "timestamp": "2025-12-29T10:30:00Z",
  "nonce": "random_nonce_123",
  "signature": "sha256_hmac_signature...",
  "data": {
    "mode": "auto"
  }
}

// 서명 검증
HMAC-SHA256(secret_key, message_content + timestamp + nonce)

6.6 연결 관리

6.6.1 재연결 전략

네트워크 불안정 시 자동 재연결 메커니즘이 필요합니다. 

// 지수 백오프 재연결
class ReconnectionManager {
  constructor() {
    this.baseDelay = 1000;      // 1초
    this.maxDelay = 60000;      // 60초
    this.maxRetries = 10;
    this.retryCount = 0;
  }

  async reconnect() {
    if (this.retryCount >= this.maxRetries) {
      throw new Error('Max retries exceeded');
    }

    const delay = Math.min(
      this.baseDelay * Math.pow(2, this.retryCount),
      this.maxDelay
    );

    await sleep(delay);
    this.retryCount++;

    try {
      await this.connect();
      this.retryCount = 0; // 성공 시 리셋
    } catch (error) {
      await this.reconnect(); // 재귀 재시도
    }
  }
}

6.6.2 연결 상태 관리

상태 설명 다음 행동
DISCONNECTED 연결 안됨 연결 시도
CONNECTING 연결 중 대기
CONNECTED 연결됨 정상 통신
RECONNECTING 재연결 중 백오프 대기
FAILED 연결 실패 오류 보고

6.7.2 배칭 (Batching)

여러 센서 데이터를 묶어서 한 번에 전송합니다. 

// 개별 전송 (비효율)
publish('sensors/dust', {level: 3});
publish('sensors/battery', {level: 85});
publish('sensors/water', {level: 45});

// 배칭 전송 (효율)
publish('sensors/batch', {
  timestamp: "2025-12-29T10:30:00Z",
  readings: [
    {sensor: 'dust', value: 3},
    {sensor: 'battery', value: 85},
    {sensor: 'water', value: 45}
  ]
});

6.7.3 델타 인코딩

변경된 필드만 전송하여 대역폭을 절약합니다. 

// 전체 상태 (1000 bytes)
{
  "state": "cleaning",
  "battery": 85,
  "location": {"x": 1250, "y": 3400},
  "cleaning": {...},
  "sensors": {...}
}

// 델타 업데이트 (50 bytes)
{
  "type": "delta",
  "changes": {
    "battery": 84,
    "location.x": 1260

}

6.8 멀티캐스트 및 브로드캐스트

여러 로봇을 동시에 제어하거나 모니터링할 때 효율적입니다. 

// MQTT 토픽 와일드카드
// 모든 로봇의 배터리 상태 구독

// 특정 건물의 모든 로봇 제어

// 그룹 명령
publish('wia-rob-011/v1/group/floor-1/commands/start', {
  mode: 'auto'
});

6.9 오프라인 처리

네트워크 연결이 끊겨도 로봇은 계속 작동하고, 재연결 시 동기화됩니다. 

// 오프라인 큐
class OfflineQueue {
  constructor() {
    this.queue = [];
    this.maxSize = 1000;
  }

  enqueue(message) {
    if (this.queue.length >= this.maxSize) {
      this.queue.shift(); // 오래된 메시지 제거
    }
    message.queued_at = Date.now();
    this.queue.push(message);
  }

  async flush() {
    while (this.queue.length > 0) {
      const message = this.queue.shift();
      try {
        await send(message);
      } catch (error) {
        this.queue.unshift(message); // 실패 시 재삽입
        break;
      }
    }
  }
}

핵심 내용 요약

  • 4가지 주요 프로토콜: WebSocket, MQTT, CoAP, HTTP/2
  • WebSocket: 실시간 양방향 통신, 하트비트 메커니즘
  • MQTT: 경량 Pub/Sub, 3단계 QoS 레벨
  • CoAP: 제약 환경용, Observe 패턴 지원
  • 필수 보안: TLS 1.2 이상, 메시지 서명
  • 재연결: 지수 백오프 전략
  • 최적화: 압축, 배칭, 델타 인코딩
  • 오프라인 처리: 큐 기반 동기화
