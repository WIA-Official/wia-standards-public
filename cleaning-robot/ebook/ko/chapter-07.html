7.2 음성 비서 통합

자연어 명령을 WIA-ROB-011 API 호출로 변환합니다.

7.2.1 지원 명령 예시

음성 명령 의도 (Intent) API 호출
"청소 시작해줘" StartCleaning POST /commands/start
"거실만 청소해" CleanRoom POST /commands/start {rooms: ["living_room"]}
"배터리 몇 퍼센트야?" QueryBattery GET /status (battery field)
"청소 중지" StopCleaning POST /commands/stop
"충전하러 가" ReturnToDock POST /commands/return

7.2.2 자연어 처리 예시

// 음성 명령 파싱
class VoiceCommandParser {
  parse(utterance) {
    const patterns = {
      start: /청소|시작|작동/,
      stop: /중지|멈춰|정지/,
      pause: /일시정지|잠깐/,
      resume: /재개|다시|계속/,
      return: /복귀|충전|돌아가/,
      room: /거실|침실|주방|화장실/
    };

    let command = {};

    if (patterns.start.test(utterance)) {
      command.action = 'start';
    } else if (patterns.stop.test(utterance)) {
      command.action = 'stop';
    }

    const roomMatch = utterance.match(patterns.room);
    if (roomMatch) {
      command.rooms = this.mapRoomName(roomMatch[0]);
    }

    return command;
  }

  mapRoomName(koreanName) {
    const mapping = {
      '거실': 'living_room',
      '침실': 'bedroom',
      '주방': 'kitchen',
      '화장실': 'bathroom'
    };
    return [mapping[koreanName]];
  }
}

7.3.2 데이터 분석 파이프라인

데이터 흐름:
  1. 로봇 → MQTT → IoT Core
  2. IoT Core → Kinesis Data Streams
  3. Kinesis → Lambda (실시간 처리)
  4. Lambda → S3 (원본 저장)
  5. S3 → Glue (ETL)
  6. Glue → Redshift (분석용 DW)
  7. Redshift → QuickSight (대시보드)

7.3.3 분석 쿼리 예시

-- 일별 청소 통계
SELECT
  DATE(start_time) as date,
  COUNT(*) as cleaning_sessions,
  AVG(duration) as avg_duration,
  SUM(area_cleaned) as total_area,
  AVG(battery_consumed) as avg_battery
FROM cleaning_history
WHERE robot_id = 'CR-2025-001'
  AND start_time >= CURRENT_DATE - INTERVAL '30 days'
GROUP BY DATE(start_time)
ORDER BY date DESC;

-- 방별 청소 빈도
SELECT
  room_name,
  COUNT(*) as times_cleaned,
  AVG(duration) as avg_duration
FROM cleaning_sessions
CROSS JOIN UNNEST(rooms_cleaned) AS t(room_name)
WHERE robot_id = 'CR-2025-001'
GROUP BY room_name
ORDER BY times_cleaned DESC;

7.5 멀티 로봇 조정

여러 대의 청소 로봇을 효율적으로 관리하고 조정합니다.

7.5.1 작업 분배 알고리즘

// 구역 기반 작업 분배
class MultiRobotCoordinator {
  assignTasks(robots, areas) {
    const assignments = [];

    // 1. 로봇 가용성 확인
    const available = robots.filter(r =>
      r.status.state === 'idle' &&
      r.status.battery > 30
    );

    // 2. 배터리 수준으로 정렬
    available.sort((a, b) =>
      b.status.battery - a.status.battery
    );

    // 3. 면적 기준 내림차순 정렬
    areas.sort((a, b) => b.area - a.area);

    // 4. 그리디 알고리즘으로 할당
    for (let i = 0; i < Math.min(available.length, areas.length); i++) {
      assignments.push({
        robot_id: available[i].id,
        area: areas[i],
        priority: i
      });
    }

    return assignments;
  }

  async execute(assignments) {
    const promises = assignments.map(async (task) => {
      return await this.startCleaning(task.robot_id, {
        zones: [task.area],
        priority: task.priority
      });
    });

    return await Promise.all(promises);
  }
}

7.7 펌웨어 업데이트

OTA(Over-The-Air) 업데이트를 통해 원격으로 펌웨어를 갱신합니다.

7.7.1 업데이트 프로세스

안전한 OTA 업데이트 절차:
  1. 사전 검증: 배터리 > 50%, 청소 중이 아님
  2. 다운로드: 펌웨어 파일 다운로드 (체크섬 확인)
  3. 검증: 디지털 서명 확인
  4. 백업: 현재 펌웨어 백업
  5. 설치: 새 펌웨어 플래싱
  6. 재부팅: 시스템 재시작
  7. 검증: 정상 작동 확인
  8. 롤백: 실패 시 이전 버전 복원
