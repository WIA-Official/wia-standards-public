5.1 API 설계 원칙

WIA-ROB-011 API는 RESTful 아키텍처를 기반으로 설계되었으며, 직관적이고 일관된 인터페이스를 제공합니다. 개발자는 최소한의 학습으로 청소 로봇의 모든 기능에 접근할 수 있습니다.

핵심 설계 원칙

  • 일관성: 모든 엔드포인트가 동일한 패턴을 따름
  • 예측 가능성: 직관적인 URL 구조와 HTTP 메서드 사용
  • 버전 관리: API 버전을 URL에 명시하여 하위 호환성 보장
  • 오류 처리: 표준화된 오류 응답 형식
  • 문서화: OpenAPI(Swagger) 사양 준수

5.2 API 기본 구조

5.2.1 Base URL

https://api.example.com/wia-rob-011/v1

5.2.2 인증

모든 API 요청은 OAuth 2.0 또는 JWT 토큰을 사용한 인증이 필요합니다. 

// OAuth 2.0 인증
Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...

// API 키 (개발/테스트 용도)
X-API-Key: wia_rob_011_api_key_example

5.5 오류 처리

API는 표준 HTTP 상태 코드와 상세한 오류 정보를 제공합니다.

상태 코드 의미 사용 사례
200 OK 요청 성공
201 Created 리소스 생성 성공
202 Accepted 명령 접수 (비동기)
400 Bad Request 잘못된 요청 파라미터
401 Unauthorized 인증 실패
403 Forbidden 권한 없음
404 Not Found 리소스 없음
429 Too Many Requests 요청 한도 초과
500 Internal Server Error 서버 오류 
// 오류 응답 형식
Response 400 Bad Request:
{
  "error": {
    "code": "INVALID_PARAMETER",
    "message": "Invalid power_level value",
    "details": {
      "field": "power_level",
      "value": "ultra",
      "allowed": ["quiet", "standard", "high", "max"]
    },
    "timestamp": "2025-12-29T10:30:00Z",
    "request_id": "req_abc123"
  }
}

5.6 속도 제한 (Rate Limiting)

API 남용 방지와 공정한 리소스 사용을 위해 속도 제한을 적용합니다.

엔드포인트 유형 제한 기간
상태 조회 60회 분당
제어 명령 30회 분당
지도 다운로드 10회 시간당
이력 조회 100회 시간당 
// 속도 제한 헤더
HTTP/1.1 200 OK
X-RateLimit-Limit: 60
X-RateLimit-Remaining: 45
X-RateLimit-Reset: 1703848800

// 제한 초과 시
HTTP/1.1 429 Too Many Requests
Retry-After: 30
{
  "error": {
    "code": "RATE_LIMIT_EXCEEDED",
    "message": "Rate limit exceeded. Please retry after 30 seconds."
  }
}

5.8.1 웹훅 등록

    "cleaning.completed",
    "battery.low",
    "error.occurred"

  "secret": "webhook_secret_key"

{
  "webhook_id": "wh_abc123",
  "url": "https://my-app.example.com/webhook",

5.9 SDK 및 클라이언트 라이브러리

주요 프로그래밍 언어를 위한 공식 SDK를 제공합니다. 

// JavaScript/TypeScript
import { CleaningRobotClient } from '@wia/rob-011-sdk';

const client = new CleaningRobotClient({
  apiKey: 'your_api_key',
  baseURL: 'https://api.example.com'
});

const robots = await client.robots.list();
await client.robots.startCleaning('CR-2025-001', {
  mode: 'auto',
  rooms: ['living_room'],
  powerLevel: 'high'
});

// Python
from wia_rob_011 import CleaningRobotClient

client = CleaningRobotClient(api_key='your_api_key')
robots = client.robots.list()
client.robots.start_cleaning('CR-2025-001',
  mode='auto', rooms=['living_room'])

// Go
import "github.com/wia/rob-011-go"

client := rob011.NewClient("your_api_key")
robots, err := client.Robots.List()
err = client.Robots.StartCleaning("CR-2025-001", ...)

핵심 내용 요약

  • RESTful API와 GraphQL 두 가지 인터페이스 제공
  • OAuth 2.0 / JWT 기반 인증 시스템
  • 주요 엔드포인트: 기기 관리, 상태 조회, 제어 명령, 스케줄, 지도, 이력
  • 표준 HTTP 상태 코드와 상세한 오류 메시지
  • 속도 제한: 분당 30-60회, 시간당 10-100회
  • 웹훅을 통한 실시간 이벤트 알림
  • JavaScript, Python, Go 등 주요 언어 SDK 제공
