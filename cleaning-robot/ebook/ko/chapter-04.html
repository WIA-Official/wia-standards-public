제4장: 1단계 - 데이터 형식

弘益人間 (홍익인간) - 널리 인간을 이롭게 하라

4.1 데이터 형식 표준화의 중요성

데이터 형식은 WIA-ROB-011 표준의 기반입니다. 통일된 데이터 구조 없이는 상호운용성과 데이터 이동성을 달성할 수 없습니다. 모든 청소 로봇은 동일한 방식으로 정보를 표현하고 교환해야 합니다.

4.2 기본 데이터 구조

WIA-ROB-011은 JSON(JavaScript Object Notation)을 기본 데이터 형식으로 사용합니다. JSON은 사람이 읽기 쉽고, 거의 모든 프로그래밍 언어에서 지원되며, 확장성이 뛰어납니다.

4.2.1 기본 구조

{
  "standard": "WIA-ROB-011",
  "version": "1.0.0",
  "timestamp": "2025-12-29T10:30:00Z",
  "device": { /* 기기 정보 */ },
  "status": { /* 상태 정보 */ },
  "capabilities": { /* 기능 정보 */ },
  "map": { /* 지도 데이터 */ },
  "history": { /* 청소 이력 */ }
}

4.2.2 필수 필드와 선택 필드

필드명 타입 필수여부 설명
standard string 필수 표준 식별자
version string 필수 표준 버전
timestamp ISO 8601 필수 데이터 생성 시각
device object 필수 기기 식별 정보
status object 필수 현재 상태
capabilities object 선택 지원 기능 목록
map object 선택 공간 지도 데이터
history array 선택 청소 이력

4.3 기기 정보 (Device Information)

device 객체는 청소 로봇을 고유하게 식별하는 정보를 포함합니다. 

{
  "device": {
    "id": "CR-2025-12345",
    "type": "vacuum_cleaner",
    "manufacturer": "Example Corp",
    "model": "CleanBot Pro X",
    "serial_number": "SN-2025-ABCD-1234",
    "firmware_version": "2.5.1",
    "hardware_version": "1.2",
    "manufacture_date": "2025-01-15",
    "certification": {
      "wia_certified": true,
      "level": "Premium",
      "certificate_id": "WIA-ROB-011-2025-001234",
      "expiry_date": "2027-01-15"
    }
  }
}

4.3.1 기기 타입

타입 값 설명 예시
vacuum_cleaner 흡입식 청소기 일반 로봇 청소기
mopping_robot 걸레질 전용 바닥 세척 로봇
hybrid_cleaner 흡입+걸레질 하이브리드 모델
window_cleaner 창문 청소 창문 청소 로봇
commercial_cleaner 상업용 대형 쇼핑몰, 공항용

4.4 상태 정보 (Status Information)

status 객체는 로봇의 실시간 상태를 나타냅니다. 

{
  "status": {
    "state": "cleaning",
    "battery": {
      "level": 85,
      "charging": false,

    },
    "location": {
      "x": 1250,
      "y": 3400,
      "theta": 45,
      "floor": 1,
      "room": "living_room",
      "confidence": 0.98
    },
    "cleaning": {
      "mode": "auto",
      "power_level": "high",
      "area_cleaned": 42.5,
      "time_elapsed": 1820,
      "progress": 65
    },
    "sensors": {
      "dust_level": 3,
      "water_tank": 45,
      "bin_full": false,
      "cliff_detected": false,
      "bumper_triggered": false
    },
    "errors": []

4.4.1 로봇 상태 (State)

상태값 의미 다음 가능 상태
idle 대기 중 cleaning, charging, docked
cleaning 청소 중 paused, returning, idle
paused 일시 정지 cleaning, returning
returning 복귀 중 docked, charging
charging 충전 중 idle, cleaning
docked 도킹 상태 cleaning, idle
error 오류 발생 idle (오류 해결 후)

4.8 오류 및 경고 (Errors and Warnings)

표준화된 오류 코드 시스템을 통해 일관된 오류 처리가 가능합니다. 

{
  "errors": [
    {
      "code": "ERR-001",
      "severity": "warning",
      "category": "maintenance",
      "message": "Dust bin is 90% full",
      "timestamp": "2025-12-29T10:20:00Z",
      "recoverable": true,
      "action_required": "empty_dustbin"
    },
    {
      "code": "ERR-102",
      "severity": "error",
      "category": "hardware",
      "message": "Left wheel motor malfunction",
      "timestamp": "2025-12-29T10:25:00Z",
      "recoverable": false,
      "action_required": "service_required"
    }
  ]
}

4.8.1 오류 코드 체계

코드 범위 카테고리 예시
ERR-001~099 유지보수 먼지통 가득참, 필터 교체
ERR-100~199 하드웨어 모터 고장, 센서 오류
ERR-200~299 소프트웨어 펌웨어 오류, 메모리 부족
ERR-300~399 네트워크 Wi-Fi 연결 끊김, API 오류
ERR-400~499 환경 미끄럼, 장애물 감지

4.9 데이터 검증 (Data Validation)

JSON Schema를 사용하여 데이터의 유효성을 검증합니다. 

{
  "$schema": "http://json-schema.org/draft-07/schema#",
  "title": "WIA-ROB-011 Cleaning Robot Data",
  "type": "object",
  "required": ["standard", "version", "timestamp", "device", "status"],
  "properties": {
    "standard": {
      "type": "string",
      "const": "WIA-ROB-011"
    },
    "version": {
      "type": "string",
      "pattern": "^[0-9]+\\.[0-9]+\\.[0-9]+$"
    },
    "device": {
      "type": "object",
      "required": ["id", "type", "manufacturer", "model"],
      "properties": {
        "battery": {
          "type": "object",
          "properties": {
            "level": {
              "type": "integer",
              "minimum": 0,
              "maximum": 100
            }
          }
        }
      }
    }
  }
}

4.10 데이터 압축 및 최적화

대용량 데이터(지도, 이력)는 효율적인 전송을 위해 압축할 수 있습니다.

데이터 타입 원본 크기 압축 방식 압축 후 크기
지도 데이터 ~500KB gzip ~50KB (90% 감소)
청소 이력 ~100KB gzip ~20KB (80% 감소)
실시간 상태 ~5KB 압축 안함 ~5KB

핵심 내용 요약

  • WIA-ROB-011은 JSON을 표준 데이터 형식으로 사용
  • 기본 구조: device, status, capabilities, map, history
  • 5가지 기기 타입 정의: vacuum_cleaner, mopping_robot 등
  • 7가지 로봇 상태: idle, cleaning, paused, returning 등
  • 표준화된 오류 코드 시스템 (ERR-001~499)
  • JSON Schema로 데이터 유효성 검증
  • 대용량 데이터는 gzip 압축으로 효율성 확보
