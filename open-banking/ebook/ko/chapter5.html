5장: 구현 가이드

WIA-FIN-022 오픈뱅킹 표준

5.1 은행용 (ASPSP 구현)

필수 조건

  • API 기능을 갖춘 핵심 뱅킹 시스템
  • OAuth 2.0 인증 서버
  • 속도 제한 및 모니터링을 위한 API 게이트웨이
  • 고객 인증 시스템 (SCA 준수)
  • 개발자 포털 인프라

구현 로드맵

1단계: 기반 (1-3개월)

  1. API 인프라 및 게이트웨이 설정
  2. OAuth 2.0 인증 서버 구현
  3. 개발 샌드박스 환경 배포
  4. API 문서 및 OpenAPI 사양 작성
  5. eIDAS 인증서 획득 및 구성

2단계: 핵심 API (4-6개월)

  1. 계정 정보 API 구현
  2. 동의 관리 시스템 개발
  3. 고객 동의 대시보드 구축
  4. SCA 흐름 구현
  5. 모니터링 및 로깅 설정

3단계: 결제 API (7-9개월)

  1. 결제 개시 API 구현
  2. 자금 확인 API 추가
  3. 결제 처리 통합 구축
  4. 사기 탐지 규칙 구현
  5. 보안 테스트 수행

4단계: 프로덕션 출시 (10-12개월)

  1. 규제 준수 검토 완료
  2. 침투 테스트 수행
  3. 파일럿 TPP 온보딩
  4. 개발자 포털 출시
  5. 프로덕션 API로 라이브 이동

5.2 TPP용 (제3자 구현)

등록 프로세스

  1. 규제 승인: 국가 관할 당국에 등록 (FCA, BaFin 등)
  2. eIDAS 인증서: 적격 TSP에서 QWAC 및 QSeal 인증서 획득
  3. 개발자 등록: 은행 개발자 포털에 가입
  4. 샌드박스 테스트: 샌드박스 환경에서 통합 테스트
  5. 프로덕션 온보딩: 인증서와 함께 프로덕션 액세스 요청

SDK 통합

WIA Open Banking SDK를 사용하여 빠르게 통합할 수 있습니다.

5.3 테스트 및 검증

샌드박스 테스트

  • 다양한 시나리오로 모든 API 엔드포인트 테스트
  • 엣지 케이스에 대한 오류 처리 확인
  • 동의 흐름 및 토큰 수명 주기 테스트
  • 보안 구현 검증 (서명, 인증서)
  • 부하 테스트 수행

5.4 프로덕션 배포

모니터링 및 관찰 가능성

주요 메트릭 모니터링:

  • 초당 API 요청 수
  • 평균 응답 시간
  • 오류율
  • 지연 시간 (p95, p99)

배포 체크리스트

  • ✓ 모든 보안 인증서 유효하고 구성됨
  • ✓ 속도 제한이 올바르게 구성됨
  • ✓ 모니터링 및 경고 설정됨
  • ✓ 백업 및 재해 복구 테스트됨
  • ✓ 문서 완료 및 게시됨
  • ✓ 지원 프로세스 확립됨
  • ✓ 규정 준수 요구 사항 충족됨
