Chapter 5: Implementation Guide

WIA-FIN-022 Open Banking Standard

5.1 For Banks (ASPSP Implementation)

5.1.1 Prerequisites

  • Core banking system with API capabilities
  • OAuth 2.0 authorization server
  • API gateway for rate limiting and monitoring
  • Customer authentication system (SCA compliant)
  • Developer portal infrastructure

5.1.2 Implementation Roadmap

Phase 1: Foundation (Months 1-3)

  1. Set up API infrastructure and gateway
  2. Implement OAuth 2.0 authorization server
  3. Deploy development sandbox environment
  4. Create API documentation and OpenAPI specs
  5. Obtain and configure eIDAS certificates

Phase 2: Core APIs (Months 4-6)

  1. Implement Account Information APIs
  2. Develop consent management system
  3. Build customer consent dashboard
  4. Implement SCA flows
  5. Set up monitoring and logging

Phase 3: Payment APIs (Months 7-9)

  1. Implement Payment Initiation APIs
  2. Add Confirmation of Funds API
  3. Build payment processing integration
  4. Implement fraud detection rules
  5. Conduct security testing

Phase 4: Production Launch (Months 10-12)

  1. Complete regulatory compliance review
  2. Conduct penetration testing
  3. Onboard pilot TPPs
  4. Launch developer portal
  5. Go live with production APIs

5.2 For TPPs (Third-Party Implementation)

5.2.1 Registration Process

  1. Regulatory Authorization: Register with national competent authority (FCA, BaFin, etc.)
  2. eIDAS Certificates: Obtain QWAC and QSeal certificates from qualified TSP
  3. Developer Registration: Sign up on bank developer portals
  4. Sandbox Testing: Test integration in sandbox environment
  5. Production Onboarding: Request production access with certificates

5.2.2 SDK Integration

5.2.3 Multi-Bank Integration Strategy

Different banks may have variations in API implementations. Strategies for handling this:

  • Adapter Pattern: Create bank-specific adapters with unified interface
  • Configuration-Driven: Use config files for bank-specific parameters
  • Fallback Handling: Graceful degradation when optional features unavailable
  • Version Management: Support multiple API versions per bank

5.3 Testing and Validation

5.3.1 Sandbox Testing

  • Test all API endpoints with various scenarios
  • Verify error handling for edge cases
  • Test consent flows and token lifecycle
  • Validate security implementations (signatures, certificates)
  • Performance testing under load

5.3.2 Conformance Testing

Many markets provide conformance testing suites:

  • UK: Open Banking Conformance Suite
  • EU: Berlin Group NextGenPSD2 tests
  • Validate against OpenAPI specifications
  • Test with official test harnesses
