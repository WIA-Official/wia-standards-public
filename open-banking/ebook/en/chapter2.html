Chapter 2: Open Banking APIs

WIA-FIN-022 Open Banking Standard

2.1 Account Information Services (AIS)

The Account Information API allows authorized third parties to retrieve customer account data, including balances, transactions, and account details. This is the foundation for personal finance management, budgeting apps, and account aggregation services.

2.1.1 Get Accounts

Retrieve list of accounts for which the PSU has given consent.

GET /accounts Authorization: Bearer {access_token} x-fapi-interaction-id: {unique-id} Response 200 OK: { "Data": { "Account": [ { "AccountId": "22289", "Currency": "GBP", "AccountType": "Personal", "AccountSubType": "CurrentAccount", "Nickname": "Main Account" }, { "AccountId": "31820", "Currency": "EUR", "AccountType": "Personal", "AccountSubType": "Savings" } ] } }

2.1.2 Get Account Balances

Retrieve current and available balance for specified account.

GET /accounts/{AccountId}/balances Authorization: Bearer {access_token} Response 200 OK: { "Data": { "Balance": [ { "AccountId": "22289", "Amount": { "Amount": "1230.45", "Currency": "GBP" }, "CreditDebitIndicator": "Credit", "Type": "InterimAvailable", "DateTime": "2025-12-25T14:23:45+00:00" } ] } }

2.1.3 Get Transactions

Retrieve transaction history for an account with optional filtering.

GET /accounts/{AccountId}/transactions?fromBookingDateTime=2025-01-01T00:00:00Z Authorization: Bearer {access_token} Response 200 OK: { "Data": { "Transaction": [ { "TransactionId": "123", "BookingDateTime": "2025-12-25T14:23:45Z", "Amount": { "Amount": "25.50", "Currency": "GBP" }, "CreditDebitIndicator": "Debit", "Status": "Booked", "MerchantDetails": { "MerchantName": "Coffee Shop", "MerchantCategoryCode": "5814" } } ] } }

2.2 Payment Initiation Services (PIS)

The Payment Initiation API enables third parties to initiate payments directly from customer accounts. This supports use cases like e-commerce checkout, bill payments, and peer-to-peer transfers.

2.2.1 Create Payment Consent

Before initiating a payment, create a consent object with payment details.

POST /domestic-payment-consents Authorization: Bearer {client_credentials_token} x-idempotency-key: {unique-key} x-jws-signature: {detached-signature} Request Body: { "Data": { "Initiation": { "InstructionIdentification": "ACME-PAY-123", "EndToEndIdentification": "E2E-PAY-123", "InstructedAmount": { "Amount": "100.00", "Currency": "GBP" }, "CreditorAccount": { "SchemeName": "UK.OBIE.SortCodeAccountNumber", "Identification": "20202052345678", "Name": "Coffee Shop Ltd" }, "RemittanceInformation": { "Unstructured": "Payment for order #12345" } } }, "Risk": {} } Response 201 Created: { "Data": { "ConsentId": "CNS-123", "Status": "AwaitingAuthorisation", "CreationDateTime": "2025-12-25T14:23:45Z" } }

2.2.2 Submit Payment

After user authorization, submit the payment for processing.

POST /domestic-payments Authorization: Bearer {access_token} x-idempotency-key: {unique-key} x-jws-signature: {detached-signature} Request Body: { "Data": { "ConsentId": "CNS-123", "Initiation": { /* Same as consent */ } } } Response 201 Created: { "Data": { "DomesticPaymentId": "PAY-456", "ConsentId": "CNS-123", "Status": "AcceptedSettlementInProcess", "CreationDateTime": "2025-12-25T14:24:30Z" } }

2.3 Confirmation of Funds (CoF)

The Confirmation of Funds API allows card issuers to check if sufficient funds are available before authorizing a transaction, reducing declined payments and overdraft fees.

POST /funds-confirmations Authorization: Bearer {access_token} Request Body: { "Data": { "InstructedAmount": { "Amount": "50.00", "Currency": "GBP" }, "DebtorAccount": { "SchemeName": "UK.OBIE.SortCodeAccountNumber", "Identification": "20202012345678" } } } Response 200 OK: { "Data": { "FundsConfirmationId": "FC-789", "FundsAvailable": true, "InstructedAmount": { "Amount": "50.00", "Currency": "GBP" } } }

2.4 Variable Recurring Payments (VRP)

VRP enables customers to set up flexible recurring payments where the amount and timing can vary within predefined limits, ideal for subscriptions, utility bills, and savings automation.

2.4.1 Create VRP Consent

POST /domestic-vrp-consents Authorization: Bearer {client_credentials_token} Request Body: { "Data": { "ControlParameters": { "ValidFromDateTime": "2025-01-01T00:00:00Z", "ValidToDateTime": "2026-12-31T23:59:59Z", "MaximumIndividualAmount": { "Amount": "500.00", "Currency": "GBP" }, "PeriodicLimits": [{ "PeriodType": "Month", "Amount": { "Amount": "2000.00", "Currency": "GBP" } }] }, "Initiation": { "CreditorAccount": { "Identification": "20202087654321", "Name": "Utility Company" } } } } Response 201 Created: { "Data": { "ConsentId": "VRP-CNS-321", "Status": "AwaitingAuthorisation" } }

2.5 API Standards and Conventions

Common Headers

  • Authorization: Bearer token for authentication
  • x-fapi-interaction-id: Unique ID for tracking request/response
  • x-idempotency-key: Ensures idempotent behavior for mutations
  • x-jws-signature: Detached JWS signature for message integrity
  • Content-Type: application/json; charset=utf-8

Error Handling

Response 400 Bad Request: { "Errors": [ { "ErrorCode": "UK.OBIE.Field.Invalid", "Message": "The field 'Amount' is invalid", "Path": "Data.Initiation.InstructedAmount.Amount" } ] } Response 401 Unauthorized: { "error": "invalid_token", "error_description": "The access token expired" } Response 403 Forbidden: { "Errors": [ { "ErrorCode": "UK.OBIE.Resource.ConsentMismatch", "Message": "Consent does not match payment details" } ] }
Best Practice: Always implement comprehensive error handling, logging request/response pairs with interaction IDs, and provide clear error messages to help developers diagnose issues quickly.

2.6 Rate Limiting and Quotas

Banks implement rate limits to protect their infrastructure. Common limits include:

  • Account Information: 10 requests per second per TPP
  • Payment Initiation: 5 requests per second per TPP
  • Burst allowance: 2x sustained rate for 10 seconds
Response Headers: X-RateLimit-Limit: 10 X-RateLimit-Remaining: 7 X-RateLimit-Reset: 1735135200 Response 429 Too Many Requests: { "error": "rate_limit_exceeded", "retry_after": 5 }
