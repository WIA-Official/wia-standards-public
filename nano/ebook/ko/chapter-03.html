3.2 3대 핵심 원칙

3.2.1 상호운용을 우선으로

식별자·단위·코드는 외부 권위에서 비롯된다. 어휘는 ISO/TS 80004, 명명법은 IUPAC, 단위는 UCUM, 생물학 코드는 SNOMED CT·ChEBI·NCIt, 기기 어휘는 Allotrope ADF. 표준은 코드를 만들지 않고 기존 권위를 버전 핀과 조합한다.

3.2.2 기본값으로서의 FAIR

Findable·Accessible·Interoperable·Reusable(Wilkinson 2016). 모든 1차 레코드는 콘텐츠 주소 식별자를 가지고, STAC + OGC API로 도달 가능하며, 핀된 어휘를 쓰고, 기계 가독 라이선스로 재사용을 허용한다.

3.2.3 Safety‑by‑design을 설계 제약으로

OECD nanoEHS 종점을 체계적으로 누락하거나 불확도·기기 출처를 숨기는 방법론은 표준 위반이다. OECD TG 318 분산 안정성, NIOSH NMAM 7300 작업자 노출, ISO 19007 in‑vitro 독성이 제품 사양과 같은 1급 시민이다.

3.4 핵심 컴포넌트

특성평가 파이프라인

TEM·DLS·BET·SAXS·ICP‑MS·AFM·XPS가 Allotrope ADF로 데이터 캡처. 각 캡처는 STAC 아이템으로 감싸 OGC API Records로 노출, CloudEvents 1.0 웹훅(RFC 9421 서명)으로 통보.

독성 자료

OECD TG 318/412/413과 ISO 19007 in‑vitro 연구가 FHIR R5 Observation으로 기록. 플랫폼이 OECD WPMN 자료 양식으로 자동 집계.

작업자 안전 감시

NIOSH NMAM 7300/0500과 ISO/TR 27628 측정이 시계열 저장소로 적재. 8시간 TWA가 권장노출한도(REL) 초과 시 알람.

자료 트윈

단일 진실원에서 SPL XML(FDA)과 EMA 기계 소비용 JSON‑LD 트윈을 동시 생성.

공급망 추적성

전구체(예: HAuCl₄)→합성→기능화→제형→폐기까지 모든 변환을 EPCIS 2.0 TransformationEvent로 캡처.

공개 검증

제품 QR 스캔 → 검증 페이지가 W3C VC를 해결, 시료 요약·OECD 종점·ISO 측정·기기 캡처를 한 화면에 표시.

3.6 WIA 생태계 결합

  • WIA‑VC — GLP·ISO 17025 보고서·인증서 발급.
  • WIA‑BLOCKCHAIN — Merkle 루트 앵커링.
  • WIA‑IDENTITY — 스폰서·GLP 실험실·규제기관 DID.
  • WIA‑IOT — TEM·AFM·DLS·ICP‑MS·노출 샘플러 온보딩.
  • WIA‑CARBON — 생산 단계 배출.
  • WIA‑CLIMATE‑022 — 기업 IFRS S2 / ESRS E1 공시 결합.

3.7 표준의 단 하나의 시험

구현된 시스템의 출력은 다른 구현자의 규제기관이 사전 조율 없이도 본 책과 인용 권위만 보고 같은 결론에 닿을 수 있어야 한다. 모르는 사람이 같은 답을 도출할 수 있는가 — 이 한 가지가 conformance의 궁극 시험이다.

장 요약

핵심 요점

  1. 5계층(데이터→프로토콜→API→응용→규제)이 단일 책임으로 분리되며 phase n+1은 phase n 위에서만.
  2. 상호운용·FAIR·safety‑by‑design은 협상 불가 설계 제약.
  3. 여섯 컴포넌트(특성·독성·작업자 안전·자료·공급망·검증)가 같은 식별자/단위 시스템을 공유.
  4. 레퍼런스 기술은 오픈소스, 대안은 conformance 동등 통과 시 1급.
  5. WIA‑VC, WIA‑BLOCKCHAIN, WIA‑IDENTITY, WIA‑IOT, WIA‑CARBON, WIA‑CLIMATE‑022와 매끄럽게 결합.
