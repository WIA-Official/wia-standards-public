5장: Phase 2 — API 인터페이스

장 개요: HTTP 와이어 — OpenAPI 3.1 + AsyncAPI 3.0, OAuth 2.1 + DPoP, idempotent CRUD, FAIR 검색, 자료 트윈(FDA SPL / EMA / PMDA / MFDS / REACH), conformance 동치성.

5.1 두 façade, 같은 진실

  1. REST + OpenAPI 3.1 — /substances, /measurements, /dossiers, /exposure 동기 CRUD.
  2. AsyncAPI 3.0 — 기기 캡처·자료 상태·규제 제출 이벤트 스트림.

https://conformance.wia-nano.org가 두 façade를 동시 실행해 결과 해시 비교.

5.3 시료 CRUD + Idempotency‑Key

curl -X POST https://api.wia-nano.org/v1/substances \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/ld+json" \
  -H "Idempotency-Key: register-zno-20nm-pegylated-2026-04-25" \
  -d @substance.jsonld

응답 201 Created + Location + 강한 ETag. 동시성 안전 patch:

PATCH /v1/substances/zno-20nm-pegylated-2026-04-25
Content-Type: application/merge-patch+json
If-Match: "v1:f3a2e1c"

{ "regulatoryFlags":["EU-REACH-NANO","FDA-NTF-CONSIDERED","KR-MFDS-PHARMA","JP-PMDA"] }

5.4 측정 업로드

POST /v1/substances/zno-20nm-pegylated/measurements
Content-Type: multipart/form-data
Idempotency-Key: meas-zno-tem-2026-04-25-batch1

[ JSON-LD 측정 레코드 + Allotrope ADF 바이너리 ]

응답에 시료의 증거 테이블 Iceberg 스냅샷 ID가 동봉.

POST /v1/dossiers
Content-Type: application/json

{
  "substanceUri":"urn:wia:nano:substance:zno-20nm-pegylated-2026-04-25",
  "targets":[
    {"regulator":"FDA","format":"SPL","section":"3.2.S.3.1"},
    {"regulator":"EMA","format":"eCTD","module":"3.2.S.3.1"},

장 요약

핵심 요점

  1. REST + AsyncAPI 두 façade는 동치성을 가져야 conformance 통과.
  2. Idempotency‑Key + ETag/If‑Match로 재시도·동시 갱신 안전.
  3. RFC 9457 problem+json이 검증 실패의 안정 식별 키.
  4. 자료 트윈이 한 레코드를 FDA SPL·EMA·PMDA·MFDS·ECHA REACH‑IUCLID로 렌더.
  5. 작업자 노출과 시료 데이터는 같은 API에 산다 — 별도 배관 금지.
