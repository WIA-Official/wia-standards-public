Chapter 5: Phase 2 — API Interface

Chapter overview: The HTTP wire — OpenAPI 3.1 + AsyncAPI 3.0, OAuth 2.1 + DPoP, idempotent CRUD on substance records, FAIR-aligned discovery, dossier-twin generation for FDA SPL / EMA / PMDA / MFDS / REACH, and the conformance suite that proves equivalence.

5.1 Two façades, one truth

  1. REST + OpenAPI 3.1 — synchronous CRUD on /substances, /measurements, /dossiers, /exposure.
  2. AsyncAPI 3.0 — event stream of instrument captures, dossier state changes, regulator submissions.

The conformance suite at https://conformance.wia-nano.org exercises both façades and compares result hashes.

5.3 Substance CRUD with Idempotency-Key

curl -X POST https://api.wia-nano.org/v1/substances \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/ld+json" \
  -H "Idempotency-Key: register-zno-20nm-pegylated-2026-04-25" \
  -d @substance.jsonld

Response 201 Created includes Location, strong ETag, and the canonical resource. Patch with concurrency:

PATCH /v1/substances/zno-20nm-pegylated-2026-04-25
Content-Type: application/merge-patch+json
If-Match: "v1:f3a2e1c"

{ "regulatoryFlags": ["EU-REACH-NANO","FDA-NTF-CONSIDERED","KR-MFDS-PHARMA","JP-PMDA"] }

5.7 Dossier twin generation

POST /v1/dossiers
Content-Type: application/json

{
  "substanceUri":"urn:wia:nano:substance:zno-20nm-pegylated-2026-04-25",
  "targets":[
    {"regulator":"FDA","format":"SPL","section":"3.2.S.3.1"},
    {"regulator":"EMA","format":"eCTD","module":"3.2.S.3.1"},
    {"regulator":"PMDA","format":"QbD"},
    {"regulator":"MFDS","format":"KFDA-supplementary"},
    {"regulator":"ECHA","format":"REACH-IUCLID-NanoForm"}
  ]
}

Response provides one signed bundle per regulator, ready to upload through the regulator's portal of record (FDA ESG, EMA EUSA, PMDA Gateway, MFDS Online, ECHA REACH-IT).

Chapter Summary

Key Takeaways

  1. REST + AsyncAPI must surface identical state; conformance compares result hashes.
  2. Idempotency-Key + ETag/If-Match make retries and concurrent updates safe.
  3. RFC 9457 problem+json is the contract for validation failures.
  4. Dossier twin endpoints render one record into FDA SPL, EMA, PMDA, MFDS, and ECHA REACH-IUCLID.
  5. Worker-exposure ingest and dossier twin live in the same API — no parallel data plumbing.
