3.2 Three core principles

3.2.1 Interoperability first

Every identifier, every unit, every code list comes from an external authority. Vocabulary from ISO/TS 80004; nomenclature from IUPAC; units from UCUM; biological codes from SNOMED CT, ChEBI, NCIt; instrument vocabulary from Allotrope ADF. WIA-NANO-016 does not invent code lists; it composes existing ones with explicit version pins.

3.2.2 FAIR by default

Findable, Accessible, Interoperable, Reusable (Wilkinson et al., 2016). Every primary record carries a content-addressed identifier; every record is reachable through STAC + OGC API; every record uses pinned vocabularies; every record is licensed for downstream reuse with explicit machine-readable permissions.

3.2.3 Safety-by-design

Methodologies that systematically under-report nanoEHS endpoints, omit uncertainty, or hide instrument-level provenance are violations of the standard. The standard requires explicit reporting of OECD TG 318 dispersion stability, NIOSH NMAM 7300 worker exposure, and ISO 19007 in-vitro toxicity records as first-class siblings to product specifications.

3.4 Key components in one paragraph each

Characterisation pipeline

TEM, DLS, BET, SAXS, ICP-MS, AFM, XPS, Zetasizer instruments capture data into Allotrope ADF; the platform wraps each capture in a STAC item, exposes it through an OGC API — Records endpoint, and notifies subscribers via signed CloudEvents 1.0 webhooks.

Toxicology dossier

OECD TG 318/412/413 plus ISO 19007 in-vitro studies are recorded as FHIR R5 Observation resources with SNOMED CT codes; the platform aggregates per-substance into the OECD WPMN dossier template.

Worker safety surveillance

NIOSH NMAM 7300/0500 + ISO/TR 27628 measurements feed a time-series store partitioned by facility, exposure-zone, and shift; alerts trigger when 8-hour TWA exceeds the recommended exposure limit (REL).

Dossier twin

A single source-of-truth substance record generates both an SPL XML filing (FDA) and a JSON-LD twin used by EMA's machine-readable consumption pipeline.

Supply-chain traceability

From precursor (e.g., HAuCl₄ for gold nanoparticles) through synthesis, functionalisation, formulation, and disposal, every transformation is captured as an EPCIS 2.0 TransformationEvent.

Public verification

Consumers scan a QR on the product, the verifier resolves a W3C Verifiable Credential, and the platform shows the substance summary, OECD TG endpoints completed, ISO measurement records, and the underlying instrument captures.

3.6 WIA ecosystem integration

  • WIA-VC — issues verifier reports, accreditation certificates, OECD GLP statements.
  • WIA-BLOCKCHAIN — anchors Merkle roots of substance records onto WIA Chain PoA + BSC.
  • WIA-IDENTITY — DIDs for sponsors, GLP labs, regulators.
  • WIA-IOT — instrument onboarding (TEM, AFM, DLS, ICP-MS).
  • WIA-CARBON — production-stage emissions for ESG reporting.
  • WIA-CLIMATE-022 — links nano-products to corporate IFRS S2 disclosures.

3.7 The "why standard" test

A system implementing WIA-NANO-016 produces output that another implementer's regulator can read, walk, and re-derive without prior coordination — only by reading this book and consulting the cited authorities. Read by a stranger, derive the same answer — that is the only ultimate test.

Chapter Summary

Key Takeaways

  1. Five layers (data → protocol → API → application → regulatory) cleanly separate concerns; phase n+1 builds only on phase n.
  2. Interoperability, FAIR, and safety-by-design are non-negotiable design constraints.
  3. Six components (characterisation, toxicology, worker safety, dossier, supply chain, public verification) share one identifier and unit system.
  4. Reference technology is open-source; alternatives are first-class but must pass conformance.
  5. The standard is interoperable with WIA-VC, WIA-BLOCKCHAIN, WIA-IDENTITY, WIA-IOT, WIA-CARBON, and WIA-CLIMATE-022.
