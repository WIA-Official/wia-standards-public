Chapter 2: Data Schema

Data as the Foundation of Trust

In carbon markets, trust is everything. The WIA Micro Carbon Credit data schema provides the standardized structure that ensures every credit is verifiable, traceable, and fraud-proof.

2.1.2 Field Definitions

Field Type Required Description
credit_id String ✅ Unique identifier following WIA-MCC-YYYY-XXXXXXXX format
credit_amount.value Decimal ✅ Amount in tCO2e, minimum 0.00001 (10g CO2)
issuance.timestamp ISO8601 ✅ UTC timestamp of credit creation
verification.method Enum ✅ IOT_GPS, SMART_METER, MANUAL, AI_IMAGE, etc.
verification.confidence_score Float ✅ 0.0-1.0, minimum 0.85 for issuance
ownership.wallet_address String ✅ Blockchain wallet address (Ethereum format)
status.state Enum ✅ PENDING, ACTIVE, RETIRED, CANCELLED, EXPIRED

2.6 Data Validation Rules

2.6.1 Credit Amount Validation

Rule Validation Error Code
Minimum credit amount credit_amount ≥ 0.00001 tCO2e ERR_CREDIT_TOO_SMALL
Maximum credit per activity credit_amount ≤ 10.0 tCO2e ERR_CREDIT_TOO_LARGE
Precision Maximum 5 decimal places ERR_INVALID_PRECISION
Non-negative credit_amount > 0 ERR_NEGATIVE_CREDIT

2.6.2 Verification Confidence Thresholds

Verification Method Minimum Confidence Action if Below Threshold
IoT GPS Tracker 0.85 Request manual review
Smart Meter 0.90 Flag for calibration check
AI Image Recognition 0.80 Require additional images
Manual Verification 0.95 Second verifier required

2.7 Chapter Summary

Key Takeaways:

  1. Credit Unit Schema: Standardized structure for micro carbon credits with unique IDs
  2. Activity Records: Detailed schemas for transport, energy, and recycling activities
  3. Verification Data: IoT device and smart meter verification structures
  4. Transactions: Transfer and retirement transaction schemas
  5. Wallet Schema: User carbon credit wallet with balance tracking
  6. Validation Rules: Minimum thresholds and confidence scores for credit issuance
