Chapter 3: API Endpoints

Building the Interface to Carbon Markets

APIs are the bridges that connect individual actions to the carbon marketplace. The WIA Micro Carbon Credit API provides simple, secure, and scalable endpoints for credit issuance, trading, and verification.

3.1 API Overview

3.1.1 Base URL and Authentication

Base URL: https://api.wia.live/v1/carbon-credits

Authentication: OAuth 2.0 Bearer Token
Authorization: Bearer {access_token}

API Key (alternative): X-API-Key: {your_api_key}

3.1.2 API Endpoint Categories

Category Endpoints Description
Credit Issuance 5 endpoints Submit activities, request verification, issue credits
Trading 8 endpoints Buy, sell, transfer credits, view marketplace
Wallet Management 6 endpoints View balance, transaction history, settings
Verification 7 endpoints Submit verification data, check status, certifications
Analytics 4 endpoints Statistics, impact reports, leaderboards

3.2 Credit Issuance Endpoints

3.2.1 Submit Activity Record

POST /activities

Submit a carbon-reducing activity for verification and credit issuance.

POST https://api.wia.live/v1/carbon-credits/activities
Authorization: Bearer {access_token}
Content-Type: application/json

{
    "user_id": "WIA-USER-2024-SARAH",
    "activity_type": "bicycle_commute",
    "timestamp_start": "2024-12-20T08:00:00Z",
    "timestamp_end": "2024-12-20T08:35:00Z",
    "transport_data": {
        "mode": "bicycle",
        "distance_km": 10.2,
        "route": {
            "start_point": {"lat": 37.7749, "lon": -122.4194},
            "end_point": {"lat": 37.7849, "lon": -122.4094}
        }
    },
    "device_id": "WIA-DEVICE-2024-GPS-BIKE-123",

        "gps_samples": 420,

3.7 Rate Limiting and Error Handling

3.7.1 Rate Limits

Tier Requests per Minute Requests per Hour Cost
Free 60 1,000 $0
Basic 300 10,000 $29/month
Pro 1,000 50,000 $99/month
Enterprise Unlimited Unlimited Custom

3.7.2 Error Codes

Code Message Description
400 Bad Request Invalid request parameters
401 Unauthorized Invalid or missing authentication token
403 Forbidden Insufficient permissions
404 Not Found Resource does not exist
429 Too Many Requests Rate limit exceeded
500 Internal Server Error Server-side error

3.8 Chapter Summary

Key Takeaways:

  1. Credit Issuance: Submit activities via API, get real-time verification
  2. Trading: Marketplace, buy/sell orders, P2P transfers
  3. Wallet Management: Balance tracking, transaction history, auto-trading
  4. Verification: IoT device registration, verification data submission
  5. Analytics: User impact stats, market statistics, rankings
  6. Rate Limiting: Tiered pricing from free to enterprise
