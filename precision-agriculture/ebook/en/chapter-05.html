Chapter 5

API Reference & Integration

Complete technical reference for the WIA-PRECISION_AGRICULTURE API, including authentication, endpoints, data formats, and integration patterns.

API Overview

The WIA-PRECISION_AGRICULTURE API provides programmatic access to precision agriculture data and services. This RESTful API enables integration with farm management systems, equipment controllers, third-party analytics platforms, and custom applications. All compliant precision agriculture platforms must implement this API specification to ensure interoperability across the WIA ecosystem.

The API follows REST architectural principles with resource-oriented URLs, standard HTTP methods, and JSON request/response bodies. Pagination, filtering, and field selection enable efficient data retrieval for diverse use cases. Comprehensive error responses facilitate debugging and graceful error handling in client applications.

Base URL

https://api.precision-ag.wia-standard.org/v1

API Version

Current version: v1.2.0 (Released: December 2025)

Authentication

The WIA-PRECISION_AGRICULTURE API uses OAuth 2.0 for authentication and authorization. All API requests require a valid access token obtained through the OAuth 2.0 authorization flow. The API supports both authorization code flow for user-facing applications and client credentials flow for machine-to-machine integrations.

OAuth 2.0 Authorization Code Flow

User-facing applications use the authorization code flow to obtain access tokens on behalf of users. This flow ensures users explicitly authorize access to their precision agriculture data while maintaining security through server-side token exchange. 

# Step 1: Redirect user to authorization endpoint
https://auth.wia-standard.org/oauth/authorize?
  client_id=YOUR_CLIENT_ID&
  redirect_uri=https://yourapp.com/callback&
  response_type=code&
  scope=read:sensors write:prescriptions&
  state=random_state_value

# Step 2: Exchange authorization code for tokens
POST https://auth.wia-standard.org/oauth/token
Content-Type: application/x-www-form-urlencoded

grant_type=authorization_code&
code=AUTHORIZATION_CODE&
redirect_uri=https://yourapp.com/callback&
client_id=YOUR_CLIENT_ID&
client_secret=YOUR_CLIENT_SECRET

# Response
{
  "access_token": "eyJhbGciOiJSUzI1NiIsInR5cCI6IkpXVCJ9...",
  "token_type": "Bearer",
  "expires_in": 3600,
  "refresh_token": "dGhpcyBpcyBhIHJlZnJlc2ggdG9rZW4...",
  "scope": "read:sensors write:prescriptions"
}

Client Credentials Flow

Machine-to-machine integrations including sensor gateways, equipment controllers, and automated processing systems use the client credentials flow. This flow authenticates the application itself rather than acting on behalf of a specific user. 

# Request access token using client credentials
POST https://auth.wia-standard.org/oauth/token
Content-Type: application/x-www-form-urlencoded

grant_type=client_credentials&
client_id=YOUR_CLIENT_ID&
client_secret=YOUR_CLIENT_SECRET&
scope=write:sensor_data read:fields

# Response
{
  "access_token": "eyJhbGciOiJSUzI1NiIsInR5cCI6IkpXVCJ9...",
  "token_type": "Bearer",
  "expires_in": 3600,
  "scope": "write:sensor_data read:fields"
}

API Scopes

Scope Description
read:fields Read field boundaries and management zones
write:fields Create and modify field boundaries
read:sensors Read sensor data and configurations
write:sensor_data Submit sensor readings (for IoT devices)
read:prescriptions Download variable rate prescriptions
write:prescriptions Create and modify prescriptions
read:imagery Access remote sensing imagery and indices
read:analytics Access analytical reports and predictions

Fields API

The Fields API manages field boundaries, management zones, and associated metadata. Fields serve as the primary organizational unit for precision agriculture data, linking sensor deployments, imagery, prescriptions, and yield records to specific geographic areas.

GET/fields

Retrieve a list of all fields accessible to the authenticated user or application.

Query Parameters

limitMaximum number of results (default: 50, max: 200)
offsetNumber of results to skip for pagination
cropFilter by current crop type
farm_idFilter by parent farm identifier
# Example: List all fields
GET /v1/fields?limit=20&crop=corn
Authorization: Bearer {access_token}

# Response
{
  "data": [
    {
      "id": "fld_abc123",
      "name": "North Quarter",
      "farm_id": "frm_xyz789",
      "area_hectares": 64.5,
      "current_crop": "corn",
      "boundary": {
        "type": "Polygon",
        "coordinates": [[[-95.123, 41.456], [-95.120, 41.456], ...]]
      },
      "created_at": "2024-03-15T10:30:00Z",
      "updated_at": "2025-04-20T14:22:00Z"
    }
  ],
  "pagination": {
    "total": 45,
    "limit": 20,
    "offset": 0,
    "has_more": true
  }
}

GET/fields/{field_id}

Retrieve detailed information for a specific field including management zones.

POST/fields

Create a new field with boundary geometry and metadata.

Sensors API

The Sensors API manages sensor device registrations, configurations, and data retrieval. This API supports both reading historical sensor data and submitting new observations from IoT devices.

GET/sensors

List all sensors with optional filtering by field, type, or status.

GET/sensors/{sensor_id}/data

Retrieve time-series data from a specific sensor.

Query Parameters

startStart timestamp (ISO 8601 format)
endEnd timestamp (ISO 8601 format)
resolutionData resolution: raw, hourly, daily (default: raw)
metricsComma-separated list of metrics to include
# Example: Get soil moisture data for past 7 days
GET /v1/sensors/sns_soil42/data?start=2025-12-03T00:00:00Z&end=2025-12-10T00:00:00Z&resolution=hourly&metrics=volumetric_content,temperature
Authorization: Bearer {access_token}

# Response
{
  "sensor_id": "sns_soil42",
  "field_id": "fld_abc123",
  "location": {
    "type": "Point",
    "coordinates": [-95.122, 41.458]
  },
  "depth_cm": 30,
  "data": [
    {
      "timestamp": "2025-12-03T00:00:00Z",
      "volumetric_content": 0.285,
      "temperature": 12.4
    },
    {
      "timestamp": "2025-12-03T01:00:00Z",
      "volumetric_content": 0.283,
      "temperature": 12.2
    }
  ],
  "units": {
    "volumetric_content": "m³/m³",
    "temperature": "°C"
  }
}

POST/sensors/{sensor_id}/data

Submit new sensor readings. Used by IoT gateways and data collection systems.

# Example: Submit sensor readings

Authorization: Bearer {access_token}

  "accepted": 2,

Prescriptions API

The Prescriptions API manages variable rate application prescriptions, providing creation, retrieval, and export capabilities. Prescriptions define spatially-variable input rates for fertilizers, pesticides, seeds, and irrigation across field zones.

GET/prescriptions

List prescriptions with filtering options.

POST/prescriptions

Create a new variable rate prescription.

# Example: Create nitrogen prescription
POST /v1/prescriptions
Authorization: Bearer {access_token}
Content-Type: application/json

{
  "field_id": "fld_abc123",
  "name": "Spring N Application 2026",
  "product": {
    "type": "fertilizer",
    "name": "Urea 46-0-0",
    "nutrient": "nitrogen"
  },
  "application_date": "2026-04-15",
  "zones": [
    {
      "zone_id": "zone_high",
      "rate": 180,
      "rate_unit": "kg/ha"
    },
    {
      "zone_id": "zone_medium",
      "rate": 150,
      "rate_unit": "kg/ha"
    },
    {
      "zone_id": "zone_low",
      "rate": 120,
      "rate_unit": "kg/ha"
    }
  ]
}

# Response
{
  "id": "prx_n2026spring",
  "field_id": "fld_abc123",
  "status": "draft",
  "created_at": "2025-12-10T11:00:00Z"
}

GET/prescriptions/{prescription_id}/export

Export prescription in equipment-compatible format.

Query Parameters

formatExport format: shapefile, isoxml, geojson (default: shapefile)
equipmentTarget equipment brand for format optimization

Webhooks

Webhooks enable real-time notifications when events occur in the precision agriculture platform. Applications register webhook endpoints to receive POST requests when specific events are triggered.

Supported Events

Event Description
sensor.alert Sensor reading exceeded configured threshold
imagery.available New imagery product available for a field
prescription.approved Prescription approved and ready for download
analytics.complete Analytical processing completed 
# Webhook payload example
POST https://yourapp.com/webhooks/precision-ag
Content-Type: application/json
X-WIA-Signature: sha256=abc123...

{
  "event": "sensor.alert",
  "timestamp": "2025-12-10T14:30:00Z",
  "data": {
    "sensor_id": "sns_soil42",
    "field_id": "fld_abc123",
    "alert_type": "low_moisture",
    "current_value": 0.18,
    "threshold": 0.20,
    "message": "Soil moisture below irrigation trigger point"
  }
}

Error Handling

The API uses standard HTTP status codes and returns detailed error responses to facilitate debugging and graceful error handling.

Common Error Codes

400Bad Request - Invalid parameters or malformed request
401Unauthorized - Missing or invalid access token
403Forbidden - Insufficient permissions for requested resource
404Not Found - Requested resource does not exist
429Too Many Requests - Rate limit exceeded
500Internal Server Error - Server-side processing error

Rate Limiting

API requests are subject to rate limiting to ensure fair usage and system stability. Rate limits vary by authentication type and subscription tier.

Tier Requests/Minute Requests/Day

Chapter Summary

This chapter provided comprehensive API documentation for integrating with WIA-PRECISION_AGRICULTURE compliant platforms. The OAuth 2.0 authentication system ensures secure access, while RESTful endpoints enable programmatic management of fields, sensors, prescriptions, imagery, and analytics.

Key Integration Patterns

  • Use authorization code flow for user-facing applications
  • Use client credentials flow for IoT devices and automation
  • Request minimum necessary scopes for each integration
  • Implement webhook handlers for real-time event processing
  • Respect rate limits and implement exponential backoff
  • Cache frequently-accessed data to reduce API calls
