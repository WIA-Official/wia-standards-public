API 레퍼런스 및 통합

인증, 엔드포인트, 데이터 형식, 통합 패턴을 포함한 WIA-PRECISION_AGRICULTURE API 완벽 기술 가이드

API 개요

WIA-PRECISION_AGRICULTURE API는 정밀 농업 데이터와 서비스에 대한 프로그래밍 방식 접근을 제공합니다. 이 RESTful API는 농장 관리 시스템, 장비 컨트롤러, 서드파티 분석 플랫폼, 커스텀 애플리케이션과의 통합을 가능하게 합니다. 모든 호환 정밀 농업 플랫폼은 WIA 생태계 전반의 상호운용성을 보장하기 위해 이 API 사양을 구현해야 합니다.

API는 리소스 지향 URL, 표준 HTTP 메서드, JSON 요청/응답 본문을 사용하는 REST 아키텍처 원칙을 따릅니다. 페이지네이션, 필터링, 필드 선택은 다양한 사용 사례에 대한 효율적인 데이터 검색을 가능하게 합니다.

기본 URL

https://api.precision-ag.wia-standard.org/v1

API 버전

현재 버전: v1.2.0 (출시: 2025년 12월)

인증

WIA-PRECISION_AGRICULTURE API는 인증 및 권한 부여를 위해 OAuth 2.0을 사용합니다. 모든 API 요청은 OAuth 2.0 인증 플로우를 통해 획득한 유효한 액세스 토큰을 필요로 합니다. API는 사용자 대면 애플리케이션을 위한 인증 코드 플로우와 기계 간 통합을 위한 클라이언트 자격 증명 플로우를 모두 지원합니다.

OAuth 2.0 인증 코드 플로우

사용자 대면 애플리케이션은 인증 코드 플로우를 사용하여 사용자를 대신하여 액세스 토큰을 획득합니다. 이 플로우는 사용자가 정밀 농업 데이터에 대한 접근을 명시적으로 승인하도록 보장하면서 서버 측 토큰 교환을 통해 보안을 유지합니다. 

# 1단계: 사용자를 인증 엔드포인트로 리디렉션
https://auth.wia-standard.org/oauth/authorize?
  client_id=YOUR_CLIENT_ID&
  redirect_uri=https://yourapp.com/callback&
  response_type=code&
  scope=read:sensors write:prescriptions&
  state=random_state_value

# 2단계: 인증 코드를 토큰으로 교환
POST https://auth.wia-standard.org/oauth/token
Content-Type: application/x-www-form-urlencoded

grant_type=authorization_code&
code=AUTHORIZATION_CODE&
redirect_uri=https://yourapp.com/callback&
client_id=YOUR_CLIENT_ID&
client_secret=YOUR_CLIENT_SECRET

# 응답
{
  "access_token": "eyJhbGciOiJSUzI1NiIsInR5cCI6IkpXVCJ9...",
  "token_type": "Bearer",
  "expires_in": 3600,
  "refresh_token": "dGhpcyBpcyBhIHJlZnJlc2ggdG9rZW4...",
  "scope": "read:sensors write:prescriptions"
}

API 스코프

스코프 설명
read:fields 포장 경계 및 관리 구역 읽기
write:fields 포장 경계 생성 및 수정
read:sensors 센서 데이터 및 구성 읽기
write:sensor_data 센서 판독값 제출 (IoT 장치용)
read:prescriptions 가변율 처방 다운로드
write:prescriptions 처방 생성 및 수정
read:imagery 원격탐사 영상 및 지수 접근
read:analytics 분석 보고서 및 예측 접근

포장 API

포장 API는 포장 경계, 관리 구역, 관련 메타데이터를 관리합니다. 포장은 정밀 농업 데이터의 주요 조직 단위로, 센서 배치, 영상, 처방, 수확량 기록을 특정 지리적 영역에 연결합니다.

GET/fields

인증된 사용자 또는 애플리케이션이 접근 가능한 모든 포장 목록을 조회합니다.

쿼리 매개변수

limit최대 결과 수 (기본: 50, 최대: 200)
offset페이지네이션을 위해 건너뛸 결과 수
crop현재 작물 유형으로 필터링
farm_id상위 농장 식별자로 필터링
# 예시: 모든 포장 목록 조회
GET /v1/fields?limit=20&crop=corn
Authorization: Bearer {access_token}

센서 API

센서 API는 센서 장치 등록, 구성, 데이터 검색을 관리합니다. 이 API는 과거 센서 데이터 읽기와 IoT 장치에서 새로운 관측값 제출을 모두 지원합니다.

GET/sensors/{sensor_id}/data

특정 센서의 시계열 데이터를 조회합니다.

쿼리 매개변수

start시작 타임스탬프 (ISO 8601 형식)
end종료 타임스탬프 (ISO 8601 형식)
resolution데이터 해상도: raw, hourly, daily (기본: raw)
metrics포함할 메트릭의 쉼표 구분 목록
# 예시: 최근 7일 토양 수분 데이터 조회
GET /v1/sensors/sns_soil42/data?start=2025-12-03T00:00:00Z&end=2025-12-10T00:00:00Z&resolution=hourly
Authorization: Bearer {access_token}

# 응답
{
  "sensor_id": "sns_soil42",
  "field_id": "fld_abc123",
  "location": {
    "type": "Point",
    "coordinates": [127.122, 37.458]
  },
  "depth_cm": 30,
  "data": [
    {
      "timestamp": "2025-12-03T00:00:00Z",
      "volumetric_content": 0.285,
      "temperature": 12.4
    }
  ],
  "units": {
    "volumetric_content": "m³/m³",
    "temperature": "°C"
  }
}

처방 API

처방 API는 가변율 적용 처방을 관리하며 생성, 검색, 내보내기 기능을 제공합니다. 처방은 포장 구역 전체에 걸쳐 비료, 농약, 종자, 관개의 공간 가변 투입율을 정의합니다.

POST/prescriptions

새로운 가변율 처방을 생성합니다.

# 예시: 질소 처방 생성
POST /v1/prescriptions
Authorization: Bearer {access_token}
Content-Type: application/json

{
  "field_id": "fld_abc123",
  "name": "2026 봄 질소 적용",
  "product": {
    "type": "fertilizer",
    "name": "요소 46-0-0",
    "nutrient": "nitrogen"
  },
  "application_date": "2026-04-15",
  "zones": [
    {"zone_id": "zone_high", "rate": 180, "rate_unit": "kg/ha"},
    {"zone_id": "zone_medium", "rate": 150, "rate_unit": "kg/ha"},
    {"zone_id": "zone_low", "rate": 120, "rate_unit": "kg/ha"}
  ]
}

웹훅

웹훅은 정밀 농업 플랫폼에서 이벤트가 발생할 때 실시간 알림을 가능하게 합니다. 애플리케이션은 특정 이벤트가 트리거될 때 POST 요청을 수신할 웹훅 엔드포인트를 등록합니다.

지원 이벤트

이벤트 설명
sensor.alert 센서 판독값이 구성된 임계값 초과
imagery.available 포장에 새로운 영상 제품 사용 가능
prescription.approved 처방 승인 및 다운로드 준비 완료
analytics.complete 분석 처리 완료

오류 처리

API는 표준 HTTP 상태 코드를 사용하고 디버깅과 우아한 오류 처리를 용이하게 하는 상세한 오류 응답을 반환합니다.

일반 오류 코드

400잘못된 요청 - 유효하지 않은 매개변수 또는 형식 오류
401인증되지 않음 - 액세스 토큰 누락 또는 무효
403금지됨 - 요청된 리소스에 대한 권한 부족
404찾을 수 없음 - 요청된 리소스 존재하지 않음
429요청 과다 - 속도 제한 초과
500내부 서버 오류 - 서버 측 처리 오류

속도 제한

API 요청은 공정한 사용과 시스템 안정성을 보장하기 위해 속도 제한이 적용됩니다. 속도 제한은 인증 유형과 구독 계층에 따라 다릅니다.

장 요약

  • 실시간 이벤트 처리를 위한 웹훅 핸들러 구현
