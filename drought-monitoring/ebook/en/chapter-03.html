Chapter 3

WIA Standard Overview

The Philosophy of 弘益人間 (Benefit All Humanity)

The WIA Drought Monitoring Standard is built upon the foundational philosophy of 弘益人間 (Hong-ik In-gan), an ancient Korean concept meaning "broadly benefit all humanity." This philosophy guides every aspect of the standard's design, from technical architecture to implementation approach.

In the context of drought monitoring, 弘益人間 translates to several core principles:

  • Universal Accessibility: Drought information should be freely available to all, from individual smallholder farmers to multinational agricultural corporations
  • Open Standards: All specifications, protocols, and interfaces are published openly without proprietary restrictions
  • Equity-Focused Design: The standard prioritizes the needs of those most vulnerable to drought impacts
  • Global Collaboration: International cooperation is embedded in the standard's governance and evolution
  • Sustainable Implementation: Solutions must be maintainable by communities with limited resources

This philosophy differentiates WIA from proprietary drought monitoring systems that restrict access based on ability to pay or from fragmented approaches that serve only specific regions or user groups.

The Four-Phase Architecture

The WIA Drought Monitoring Standard is organized into four progressive phases, each building upon the previous one to create a comprehensive ecosystem for drought detection, monitoring, and response.

Phase 1: Data Format Standardization

The foundation of interoperability is agreement on how drought information should be structured and transmitted. Phase 1 defines standardized schemas for all major drought indices and monitoring parameters.

Key components include:

  • Palmer Drought Severity Index (PDSI) data schema
  • Standardized Precipitation Index (SPI) format specifications
  • Soil moisture measurement standards
  • NDVI (Normalized Difference Vegetation Index) data structures
  • Evapotranspiration measurement formats
  • Metadata specifications for data provenance and quality
Data Type Standard Format Key Fields Update Frequency
PDSI JSON Schema v1.0 index, classification, parameters Weekly/Monthly
SPI JSON Schema v1.0 value, period, category, confidence Monthly
Soil Moisture JSON Schema v1.0 depth layers, percentages, thresholds Daily/Weekly
NDVI GeoJSON + metadata value, anomaly, satellite source Daily/Weekly

Phase 2: API Interface Specification

Once data formats are standardized, Phase 2 defines how systems should expose drought information through Application Programming Interfaces (APIs). This enables automated integration and real-time data access.

The WIA API specification includes:

  • RESTful endpoint design for drought data queries
  • Authentication and authorization standards
  • Rate limiting and usage quota guidelines
  • Real-time monitoring data access endpoints
  • Historical drought data query interfaces
  • Alert subscription and notification services
  • Forecast data access protocols
  • Batch data export capabilities
Design Principle: The WIA API is designed to be implementable by both sophisticated satellite data providers and simple community weather station networks. A minimal implementation can be deployed with basic web hosting, while advanced implementations can leverage cloud computing for global scale.

Phase 3: Protocol Standards

Phase 3 addresses how raw satellite and sensor data should be processed into standardized drought indices. This ensures consistency in calculation methods and data quality across different implementations.

Protocol specifications cover:

  • Satellite data acquisition and preprocessing procedures
  • MODIS imagery processing for vegetation indices
  • Landsat thermal data processing for surface temperature
  • Evapotranspiration calculation methodologies
  • Water balance modeling approaches
  • Quality control and validation procedures
  • Data fusion techniques for multiple sensors
  • Uncertainty quantification and reporting
Processing Stage Standard Method Quality Requirements Validation Approach
Atmospheric Correction 6S Radiative Transfer RMSE < 5% Ground reflectance targets
Cloud Masking Fmask Algorithm Accuracy > 95% Visual inspection + metrics
NDVI Calculation (NIR - Red)/(NIR + Red) Precision 0.01 Cross-sensor comparison

Phase 4: System Integration Standards

The final phase defines how drought monitoring integrates with agricultural management systems, irrigation controllers, early warning platforms, and decision support tools.

Integration specifications include:

  • Farm management system connection protocols
  • Irrigation controller communication standards
  • Early warning system alert formats
  • Climate adaptation tool interfaces
  • Crop insurance verification procedures
  • Water resource management system integration
  • Mobile application development guidelines
  • Offline data synchronization protocols

Design Principles

Several key principles guide the WIA standard's technical design:

Simplicity First

The standard prioritizes simple, understandable approaches over complex solutions. A basic weather station with limited computing resources should be able to participate in the WIA ecosystem alongside sophisticated satellite processing centers.

Progressive Enhancement

Implementations can start with basic Phase 1 compliance (standard data formats) and progressively add Phase 2 APIs, Phase 3 processing protocols, and Phase 4 integrations as resources allow. Full value is provided at each phase.

Backward Compatibility

As the standard evolves, new versions maintain compatibility with previous versions wherever possible. Systems implementing WIA v1.0 will continue to function when v2.0 is released.

Language and Culture Neutrality

All standard specifications are available in multiple languages, and technical implementations must support internationalization and localization to serve diverse global communities.

Open Source Reference Implementations

Complete, production-ready implementations of the standard are provided as open source software in multiple programming languages, allowing anyone to deploy a compliant system without starting from scratch.

Governance and Evolution

The WIA Drought Monitoring Standard is governed by a multi-stakeholder committee including representatives from:

  • National meteorological agencies
  • Agricultural organizations and farmer cooperatives
  • Satellite data providers and remote sensing researchers
  • Water resource management agencies
  • International development and humanitarian organizations
  • Technology companies and software developers
  • Academic institutions and research centers

This diverse governance ensures the standard serves real-world needs and evolves based on implementation experience and emerging technologies.

Certification and Compliance

Organizations implementing the WIA standard can seek certification to verify compliance and interoperability with other WIA-compliant systems.

Certification Levels

  • Bronze Certification: Phase 1 compliance - implements standard data formats
  • Silver Certification: Phases 1-2 compliance - includes API implementation
  • Gold Certification: Phases 1-3 compliance - full processing protocol adherence
  • Platinum Certification: All phases plus demonstrated integration with 3+ partner systems

Certification is free for non-profit organizations, educational institutions, and governmental agencies. Commercial entities pay a modest certification fee that supports standard maintenance and developing nation implementation assistance.

Chapter Summary

This chapter introduced the WIA Drought Monitoring Standard's philosophical foundation of 弘益人間 (Benefit All Humanity) and its four-phase progressive architecture. We explored how Phase 1 standardizes data formats, Phase 2 defines API interfaces, Phase 3 establishes processing protocols, and Phase 4 enables system integration. The chapter explained key design principles including simplicity, progressive enhancement, and backward compatibility, and outlined the multi-stakeholder governance model ensuring the standard serves diverse global needs.

Key Takeaways

  • Universal Philosophy: The 弘益人間 principle ensures drought information is freely accessible to all, from smallholder farmers to multinational corporations, prioritizing those most vulnerable to drought impacts.
  • Progressive Architecture: Four phases (Data Format, API Interface, Protocol, Integration) allow implementations to start simple and progressively add capabilities while providing value at each stage.
  • Design for All: Simplicity-first approach enables participation from basic weather stations to sophisticated satellite processing centers, with reference implementations in multiple programming languages.
  • Multi-Stakeholder Governance: Representatives from meteorological agencies, agricultural organizations, researchers, and technology companies ensure the standard evolves to meet real-world needs.
  • Significant Impact: Projected annual benefits of $3.3-5.7 billion globally through reduced crop losses, water conservation, research efficiency, and development cost savings.
