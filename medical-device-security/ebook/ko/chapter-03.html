인슐린 펌프 취약점

장 개요: 이 장에서는 인슐린 펌프와 연속 혈당 모니터링(CGM) 시스템의 사이버 보안 취약점을 다룹니다. 당뇨병 환자의 생명을 유지하는 이 장비들은 점점 더 연결되고 자동화되면서 새로운 보안 도전 과제에 직면하고 있습니다. 실제 해킹 사례, 인공췌장 시스템의 보안, 그리고 효과적인 방어 전략을 살펴봅니다.

인슐린 펌프 시스템의 이해

인슐린 펌프는 제1형 및 제2형 당뇨병 환자가 혈당을 조절하기 위해 사용하는 소형 의료기기입니다. 전통적인 인슐린 주사를 대체하여 24시간 지속적으로 인슐린을 주입하며, 식사 시 추가 용량(볼러스)을 투여할 수 있습니다.

현대 인슐린 펌프의 구성 요소

  • 펌프 본체: 인슐린 저장소, 마이크로프로세서, 배터리, 디스플레이
  • 주입 세트: 펌프와 환자를 연결하는 카테터와 캐뉼라
  • CGM 센서: 피하 조직에서 5분마다 혈당 측정
  • 무선 통신: Bluetooth, 무선 주파수로 스마트폰 및 CGM과 연결
  • 제어 앱: 스마트폰에서 펌프 모니터링 및 제어

가장 진보된 형태의 당뇨병 관리 시스템은 인공췌장으로도 알려진 AID 시스템입니다. 이는 CGM, 인슐린 펌프, 제어 알고리즘을 통합하여 자동으로 혈당을 조절합니다:

시스템 유형 자동화 수준 사용자 개입 보안 복잡도

인슐린 펌프 해킹의 역사

2011: Jay Radcliffe의 자가 해킹

보안 연구자이자 제1형 당뇨병 환자인 Jay Radcliffe는 2011년 Black Hat 컨퍼런스에서 자신이 사용하는 인슐린 펌프를 해킹하는 시연을 했습니다. 그는 다음을 발견했습니다:

  • 암호화 부재: 펌프와 원격 제어기 간 통신이 평문으로 전송됨
  • 인증 없음: 누구나 펌프에 명령을 보낼 수 있음
  • 재생 공격 가능: 캡처한 명령을 나중에 재전송 가능
  • 치명적 용량: 원격으로 치명적인 양의 인슐린 주입 가능
실제 위험: 인슐린 과다 투여는 심각한 저혈당(hypoglycemia)을 유발하여 의식 상실, 발작, 혼수상태, 심지어 사망에 이를 수 있습니다. 반대로 인슐린 투여 중단은 고혈당과 당뇨병성 케톤산증(DKA)을 일으킬 수 있으며, 이 역시 치명적입니다.

2016: FDA의 Animas OneTouch Ping 경고

2016년, FDA는 Johnson & Johnson의 Animas OneTouch Ping 인슐린 펌프에 대한 사이버 보안 경고를 발표했습니다. 이는 FDA가 인슐린 펌프에 대해 발표한 최초의 공식 보안 권고였습니다.

발견된 취약점:

  • 원격 제어기와 펌프 간 암호화되지 않은 무선 통신
  • 공격자가 25피트(약 7.6미터) 이내에서 볼러스 명령 전송 가능
  • 환자가 부주의할 경우 승인되지 않은 인슐린 투여 발생 가능

FDA 권고 사항:

  • 펌프의 볼러스 알림을 항상 활성화할 것
  • 예상치 않은 인슐린 투여 알림 시 즉시 확인할 것
  • 의심스러운 활동 발견 시 제조업체 및 의료진에게 즉시 보고

2018: Medtronic 펌프 대규모 리콜

2018년, Medtronic은 다수의 인슐린 펌프 모델에서 사이버 보안 취약점이 발견되어 리콜을 실시했습니다. 이는 인슐린 펌프 역사상 가장 큰 규모의 보안 관련 리콜이었습니다.

영향받은 모델: MiniMed 508, MiniMed Paradigm series (512, 522, 712, 722), MiniMed Paradigm REAL-Time series 등 총 4,000대 이상의 펌프가 영향을 받았습니다.

Medtronic은 해당 모델의 생산을 중단하고, 환자들을 더 안전한 신형 모델로 무료 교체해주었습니다. 신형 모델에는 암호화된 무선 통신, 상호 인증, 재생 공격 방지 기능이 포함되었습니다.

인슐린 펌프 공격 시나리오

1. 원격 인슐린 주입 공격

가장 치명적인 공격으로, 공격자가 환자 모르게 인슐린을 주입하는 시나리오입니다:

공격 시나리오: 원격 볼러스 주입
# 공격자 프로파일
- 위치: 환자로부터 10m 이내 (카페, 사무실, 대중교통)
- 장비: SDR (소프트웨어 무선), 노트북, 안테나
- 지식: 펌프 무선 프로토콜 리버스 엔지니어링

# 공격 단계

1단계: 타겟 스캔
   - 402-405 MHz 주파수 대역 스캔
   - 인슐린 펌프 신호 식별
   - 펌프 시리얼 번호 및 모델 확인

2단계: 트래픽 분석
   - 정상 통신 패킷 캡처
   - 볼러스 명령 구조 분석
   - 프로토콜 리버스 엔지니어링

3단계: 명령 주입
   - 위조된 볼러스 명령 생성
   - 15 유닛 인슐린 주입 명령 전송
   - (일반 식사 용량의 3-5배)

4단계: 효과
   - 15-30분 후: 혈당 급격히 하락
   - 30-60분 후: 저혈당 증상 (떨림, 혼란, 발한)
   - 60-90분 후: 심각한 저혈당 (의식 저하)
   - 응급 처치 없으면 생명 위험

# 완화 조치
- 볼러스 알림 필수 활성화
- 예상치 않은 알림 시 즉시 펌프 확인
- 최대 볼러스 제한 설정
- 암호화된 통신 사용

2. 인슐린 중단 공격

반대로 인슐린 주입을 중단하는 공격도 위험합니다:

  • 기초 주입 중단: 24시간 지속되는 기초 인슐린을 중단
  • 영향: 몇 시간 내에 혈당이 위험 수준으로 상승
  • 결과: 당뇨병성 케톤산증(DKA), 입원 필요
  • 탐지의 어려움: 환자가 즉시 증상을 느끼지 못할 수 있음

3. CGM 데이터 조작

연속 혈당 모니터링 시스템의 데이터를 조작하여 환자나 인공췌장 알고리즘을 속이는 공격:

공격 유형 조작 방법 환자 반응 결과
허위 고혈당 CGM 데이터를 높게 위조 환자가 추가 인슐린 주입 실제로는 저혈당 유발
허위 저혈당 CGM 데이터를 낮게 위조 환자가 탄수화물 섭취 실제로는 고혈당 유발
데이터 지연 오래된 데이터 반복 전송 혈당 변화를 놓침 부적절한 치료 결정
알림 차단 저/고혈당 경고 억제 위험한 혈당 수준 인지 못함 응급 상황 발생

4. 모바일 앱 취약점

현대 인슐린 펌프는 스마트폰 앱과 연동됩니다. 이 앱들도 공격 표면이 될 수 있습니다:

  • 앱 리버스 엔지니어링: 디컴파일하여 통신 프로토콜 추출
  • 중간자 공격 (MITM): 앱과 펌프 간 통신 가로채기
  • 악성 앱 배포: 가짜 당뇨병 관리 앱으로 위장하여 자격 증명 탈취
  • 클라우드 API 공격: 제조업체 클라우드 서버 침입하여 여러 환자 데이터 탈취

인공췌장 시스템의 특수한 보안 도전

자동화된 인슐린 투여 시스템은 편의성을 제공하지만 새로운 보안 위험도 도입합니다:

1. 알고리즘 조작

인공췌장의 제어 알고리즘은 CGM 데이터를 기반으로 인슐린 용량을 계산합니다. 공격자가 이 알고리즘을 조작하면:

  • 과도하게 공격적인 인슐린 투여 (저혈당 유발)
  • 과도하게 보수적인 인슐린 투여 (고혈당 허용)
  • 인슐린 감수성 계수 조작 (용량 계산 왜곡)
  • 목표 혈당 범위 변경 (위험한 수준 허용)

2. DIY 인공췌장의 위험

일부 당뇨병 커뮤니티는 오픈소스 인공췌장 시스템(OpenAPS, Loop, AndroidAPS)을 개발했습니다. 이들은 강력하지만 보안 위험도 있습니다:

DIY 시스템의 보안 고려사항:
  • 규제 미승인: FDA나 CE 인증을 받지 않음
  • 사용자 책임: 모든 보안은 사용자가 직접 관리
  • 코드 검증: 오픈소스 코드의 보안 감사 필요
  • 업데이트 위험: 악의적인 코드가 포함된 업데이트 가능성
  • 지원 부재: 보안 사고 시 공식 지원 없음

인슐린 펌프 보안 방어 전략

1. 강력한 무선 보안

현대 인슐린 펌프는 다음과 같은 무선 보안 메커니즘을 구현해야 합니다:

권장 무선 보안 구성
# Bluetooth Low Energy (BLE) 보안

암호화: AES-128-CCM (Counter with CBC-MAC)
키 교환: LE Secure Connections with ECDH
페어링: Numeric Comparison (6자리 숫자 확인)
본딩: 장기 키 저장 (재페어링 불필요)

# 페어링 프로세스

1. 초기 페어링:
   - 펌프와 스마트폰 모두에 동일한 6자리 숫자 표시
   - 사용자가 두 숫자가 일치하는지 육안 확인
   - 확인 후에만 페어링 완료

2. 인증된 장치만 통신:
   - 페어링된 장치의 MAC 주소 화이트리스트
   - 미인증 장치의 연결 시도 거부
   - 최대 3개 장치로 페어링 제한

3. 재생 공격 방지:
   - 각 명령에 순차 증가하는 카운터 포함
   - 이전에 수신한 카운터 값 거부
   - 타임스탬프 검증 (±5분 허용)

4. 세션 타임아웃:
   - 30분 사용 없으면 자동 연결 해제
   - 재연결 시 재인증 필요

2. 다층 인증 및 승인

중요한 작업에는 여러 단계의 확인이 필요합니다:

작업 인증 수준 사용자 확인 시간 제한
CGM 데이터 읽기 낮음 불필요 없음
기초 주입률 확인 낮음 불필요 없음
소량 볼러스 (<5 유닛) 중간 펌프에서 확인 버튼 60초
대량 볼러스 (≥5 유닛) 높음 펌프 + 앱 양쪽 확인 30초
설정 변경 높음 PIN 코드 + 펌프 확인 120초
펌웨어 업데이트 최고 PIN + 생체인증 300초

3. 이상 탐지 및 안전 제한

펌프에 내장된 안전 메커니즘으로 비정상적인 명령을 차단:

  • 최대 볼러스 제한: 사용자가 설정한 최대값 이상 투여 불가
  • 일일 총 인슐린 제한: 평소 사용량의 150%를 초과하면 경고
  • 연속 볼러스 방지: 2시간 이내 3회 이상 볼러스 시 추가 확인 요구
  • 급격한 설정 변경 탐지: 기초 주입률이 50% 이상 변경 시 경고
  • 인슐린 온보드 (IOB) 추적: 아직 작용 중인 인슐린 양 계산하여 과다 투여 방지

4. 사용자 교육 및 인식

기술적 보안만큼 중요한 것이 사용자의 보안 인식입니다:

환자를 위한 보안 가이드라인:
  • 알림 설정: 모든 볼러스 알림을 활성화하고 무음 모드 사용 금지
  • 즉시 확인: 예상치 않은 인슐린 투여 알림 시 즉시 펌프 확인
  • 페어링 관리: 신뢰할 수 있는 장치만 페어링, 정기적으로 목록 검토
  • 펌웨어 업데이트: 제조업체 공식 채널에서만 업데이트 받기
  • 물리적 보안: 펌프를 항상 몸에 착용, 분실 시 즉시 신고
  • 의심 보고: 이상한 동작 발견 시 의료진 및 제조업체에 보고
  • 백업 계획: 펌프 고장 시를 대비한 인슐린 펜 또는 주사기 준비

규제 및 표준

인슐린 펌프 보안은 다음 규제 및 표준을 따릅니다:

  • FDA Guidance (2023): 의료기기 사이버 보안 시판 전후 관리
  • IEC 60601-1: 의료 전기 기기 안전 일반 요구사항
  • ISO 11608: 주사기, 펜, 인슐린 주입 시스템 요구사항
  • UL 2900-2-1: 의료 및 건강 관리 시스템 사이버 보안
  • AAMI TIR57: 의료기기 보안 원칙 및 실무

미래의 보안 기술

1. 인공지능 기반 이상 탐지

머신러닝 알고리즘이 환자의 정상적인 인슐린 사용 패턴을 학습하고, 이상 행동을 자동으로 탐지:

  • 평소와 다른 시간대의 대량 볼러스 탐지
  • CGM 데이터와 인슐린 투여 간 불일치 발견
  • 비정상적인 무선 통신 패턴 식별

2. 블록체인 기반 투여 기록

모든 인슐린 투여 기록을 변조 불가능한 블록체인에 저장하여 무결성 보장:

  • 누가, 언제, 얼마나 인슐린을 투여했는지 영구 기록
  • 의심스러운 투여 발견 시 감사 추적 가능
  • 의료 과실 소송 시 객관적 증거 제공

3. 생체 인증 통합

지문, 얼굴 인식 등 생체 인증을 펌프 제어에 통합:

  • 스마트폰 생체 인증 후에만 앱에서 펌프 제어 가능
  • 펌프 자체에 지문 센서 통합 (차세대 모델)
  • 환자의 고유한 심전도(ECG) 패턴으로 인증

결론

인슐린 펌프와 CGM 시스템은 당뇨병 환자의 삶의 질을 크게 향상시켰지만, 연결성 증가는 새로운 보안 위험을 초래했습니다. 다행히 제조업체들은 과거의 취약점으로부터 교훈을 얻어 보안을 크게 강화했습니다.

환자들은 최신 보안 기능을 갖춘 펌프를 사용하고, 모든 알림을 활성화하며, 의심스러운 활동을 즉시 보고해야 합니다. 의료진은 환자에게 사이버 보안의 중요성을 교육하고, 정기적으로 기기 설정을 검토해야 합니다.
