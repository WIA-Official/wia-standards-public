제7장

접근 제어 및 메타데이터

더 나은 발견 및 이해를 위한 데이터 접근 권한, 메타데이터 카탈로그 및 데이터 계보 관리.

데이터 접근 제어

접근 제어는 권한이 있는 개인과 시스템만 역할과 책임에 적합한 데이터에 액세스할 수 있도록 보장합니다. 효과적인 접근 제어는 보안과 사용성의 균형을 맞추어 무단 액세스를 방지하면서 합법적인 데이터 사용을 가능하게 합니다.

접근 제어 모델

역할 기반 접근 제어 (RBAC)

조직 내 사용자 역할에 따라 액세스 권한이 부여됩니다.

  • 역할: 직무 기능 정의 (예: 분석가, 관리자, 관리자)
  • 권한: 개인이 아닌 역할에 권한 할당
  • 사용자: 사용자를 적절한 역할에 할당
  • 이점: 관리 단순화, 오류 감소, 더 쉬운 감사
  • 적합한 경우: 명확한 직무 역할을 가진 구조화된 조직

속성 기반 접근 제어 (ABAC)

사용자, 리소스 및 환경의 속성을 기반으로 한 액세스 결정.

  • 사용자 속성: 부서, 보안 수준, 위치
  • 리소스 속성: 분류, 소유자, 민감도
  • 환경 속성: 시간대, IP 주소, 장치 유형
  • 이점: 세밀한 통제, 동적 결정, 유연한 정책
  • 적합한 경우: 컨텍스트 인식 액세스가 필요한 복잡한 환경

접근 제어 모범 사례

최소 권한 원칙

사용자는 직무 기능을 수행하는 데 필요한 최소한의 액세스 권한을 가져야 합니다.

  • 액세스 없음으로 시작하여 필요에 따라 추가
  • 임시 필요에 대한 시간 제한 액세스
  • 관리자 계정과 사용자 계정 분리
  • 정기적인 액세스 검토 및 정리

직무 분리

한 사람이 전체 중요한 프로세스를 통제해서는 안 됩니다.

  • 승인과 실행을 다른 사람이 수행
  • 개발, 테스트, 프로덕션 액세스 분리
  • 여러 역할에 걸쳐 민감한 작업 분할
  • 사기 및 오류 방지

접근 요청 및 승인

요청 워크플로우

  1. 요청 제출: 사용자가 정당화와 함께 액세스 요청 제출
  2. 비즈니스 승인: 데이터 소유자 또는 관리자가 비즈니스 필요성 승인
  3. 보안 검토: 보안 팀이 규정 준수 및 위험 검토
  4. 프로비저닝: IT가 액세스 권한 구현
  5. 알림: 사용자 및 승인자에게 액세스 권한 부여 통지
  6. 문서화: 감사 추적을 위해 요청 기록

접근 검토 및 재인증

정기 접근 검토

누가 어떤 데이터에 액세스할 수 있는지 정기적으로 검토합니다.

  • 빈도: 민감한 데이터는 분기별, 기타는 연간
  • 검토자: 데이터 소유자, 관리자, 보안 팀
  • 프로세스: 액세스 목록 검토, 지속적인 필요 확인, 불필요한 액세스 제거
  • 문서화: 검토 결정 및 조치 기록

접근 제어 메트릭

  • 액세스 요청 시간: 요청을 충족하는 데 걸리는 평균 시간
  • 승인률: 승인된 요청의 비율
  • 검토 완료: 제시간에 완료된 검토의 비율
  • 액세스 제거: 제거된 불필요한 액세스의 양
  • 위반율: 무단 액세스 시도
  • 고아 계정: 활성 소유자가 없는 계정

메타데이터 관리

메타데이터는 "데이터에 관한 데이터"입니다 — 데이터 자산을 설명, 설명, 위치 지정 또는 달리 검색, 사용 또는 관리하기 쉽게 만드는 정보입니다.

메타데이터 유형

비즈니스 메타데이터

비즈니스 사용자에게 의미 있는 정보:

  • 비즈니스 정의 및 설명
  • 비즈니스 규칙 및 로직
  • 데이터 소유권 및 스튜어드십
  • 데이터 품질 점수 및 문제
  • 사용 가이드라인 및 제한 사항

기술 메타데이터

데이터 구조에 대한 기술적 세부 정보:

  • 데이터 유형, 형식 및 제약 조건
  • 데이터베이스 스키마 및 테이블 구조
  • 컬럼 이름 및 관계
  • 인덱스 및 성능 최적화
  • 시스템 및 애플리케이션 세부 정보

운영 메타데이터

데이터 처리 및 운영에 관한 정보:

  • 데이터 계보 및 변환 로직
  • ETL/ELT 작업 정의 및 일정
  • 처리 로그 및 성능 메트릭
  • 데이터 새로 고침 빈도 및 타임스탬프
  • 오류 로그 및 문제 추적

비즈니스 용어집

공통 이해를 보장하는 비즈니스 용어 및 정의의 중앙 집중식 저장소.

용어집 구성 요소

  • 용어 이름: 표준화된 비즈니스 용어
  • 정의: 명확하고 간결한 설명
  • 동의어: 동일한 개념에 대한 대체 이름
  • 관련 용어: 연결되거나 유사한 용어
  • 비즈니스 규칙: 제약 조건 또는 계산
  • 예제: 샘플 데이터 또는 사용 사례
  • 소유자: 용어에 대해 책임지는 사람
  • 상태: 초안, 승인됨, 사용 중단됨

데이터 카탈로그

검색 가능한 메타데이터를 통해 데이터 발견을 가능하게 하는 데이터 자산의 조직화된 인벤토리.

카탈로그 기능

  • 검색 및 발견: 관련 데이터 자산을 빠르게 찾기
  • 자산 세부 정보: 각 자산에 대한 포괄적인 메타데이터
  • 데이터 미리보기: 샘플 데이터 및 통계
  • 계보 시각화: 데이터 흐름 이해
  • 사용 분석: 누가 어떤 데이터를 어떻게 사용하는지
  • 협업: 댓글, 평가, 추천
  • 액세스 요청: 카탈로그에서 직접 액세스 요청

데이터 계보

소스에서 변환을 거쳐 소비까지 데이터 여정의 시각적 표현.

계보 수준

  • 데이터셋 수준: 시스템 간의 높은 수준의 흐름
  • 컬럼 수준: 상세한 필드 간 매핑
  • 변환 수준: 데이터에 적용되는 로직 및 규칙

계보 사용 사례

  • 영향 분석: 변경의 하류 효과 이해
  • 근본 원인 분석: 데이터 품질 문제를 소스로 추적
  • 규정 준수: 규정에 대한 데이터 흐름 문서화
  • 데이터 발견: 데이터의 출처 이해
  • 마이그레이션 계획: 시스템 마이그레이션 및 업그레이드 계획

메타데이터 관리 모범 사례

  • 가능한 경우 메타데이터 수집 자동화
  • 메타데이터 생성을 개발 프로세스의 일부로 만들기
  • 메타데이터를 검색 가능하고 액세스 가능하게 보장
  • 데이터 변경과 메타데이터 동기화 유지
  • 메타데이터 강화에 비즈니스 사용자 참여
  • 메타데이터 품질 측정 및 보고
  • 메타데이터 관리를 데이터 거버넌스와 통합
  • 메타데이터의 중요성 및 사용에 대한 교육 제공

접근 제어 및 메타데이터의 통합

접근 제어와 메타데이터 관리는 안전하고 정보에 입각한 데이터 사용을 가능하게 하기 위해 함께 작동합니다:

  • 발견: 사용자가 카탈로그를 통해 데이터 찾기
  • 이해: 메타데이터가 데이터의 의미와 사용 방법 설명
  • 액세스 요청: 이해를 바탕으로 액세스 요청
  • 승인: 소유자가 메타데이터(분류, 민감도)를 기반으로 승인
  • 사용: 메타데이터가 적절한 데이터 사용 안내
  • 규정 준수: 계보 및 액세스 로그가 규정 준수 입증

마지막 장에서는 조직에서 포괄적인 데이터 거버넌스 프로그램을 구현하기 위한 실용적인 지침으로 모든 것을 종합합니다.
