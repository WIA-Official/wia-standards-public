제5장: 검사실 및 영상 정보 시스템 (LIS/RIS/PACS)

"검사 결과와 의료 영상은 진단의 핵심입니다. 빠르고 정확한 전달이 환자의 생명을 구합니다."

5.1 검사실 정보 시스템 (LIS)

🧪 LIS 주요 기능

  • 검체 관리 — 바코드로 검체 추적
  • 검사 오더 접수 — HIS/EMR에서 자동 전송
  • 분석 장비 연동 — HL7/ASTM으로 자동 분석기 통신
  • 결과 검증 — 정상 범위 확인 및 이상치 경고
  • 결과 보고 — EMR로 자동 전송
  • 품질 관리 — 내부/외부 정도 관리

📋 검사 워크플로우

1. 의사가 EMR에서 검사 오더 (CBC, Liver function)
   ↓
2. LIS가 오더 접수 → 바코드 라벨 출력
   ↓
3. 채혈실에서 검체 채취 → 바코드 부착
   ↓
4. 검사실 도착 → 바코드 스캔으로 자동 분류
   ↓
5. 자동 분석기로 검사 (Sysmex, Roche, etc.)
   ↓
6. 결과 자동 전송 → LIS 검증
   ↓
7. 정상 범위 체크 → 이상치는 임상병리 전문의 검토
   ↓
8. 승인 후 EMR 자동 전송 → 의사 확인

🔬 검사 종류

분야 검사 항목 TAT 목표
일반혈액 CBC, 혈소판 1시간
화학 간기능, 신기능, 전해질 2시간
응급 Troponin, D-dimer 30분
미생물 배양, 항생제 감수성 48-72시간

TAT (Turnaround Time): 검체 접수부터 결과 보고까지의 시간

🤖 자동화 검사실

Total Laboratory Automation (TLA):

  • 자동 검체 분주 — 원심분리 후 자동 분주
  • 자동 이송 시스템 — 컨베이어 벨트로 검사실 내 이동
  • 자동 보관 — 냉장/냉동 보관소로 자동 이동
  • 24/7 운영 — 야간/주말에도 무인 운영 가능

효과: 삼성서울병원 TLA 도입 후 검사 TAT 40% 단축, 검사 정확도 99.5% 달성

5.2 영상 정보 시스템 (RIS)

📷 RIS 기능

방사선과 업무 관리:

  • 검사 스케줄링 — CT, MRI, X-ray 예약
  • 환자 등록 — EMR과 자동 동기화
  • Worklist 관리 — 촬영 대기 목록
  • 판독 워크플로우 — 판독의 배정 및 진행 상황
  • 보고서 작성 — 음성 인식으로 판독 입력
  • 통계 및 분석 — 검사량, TAT, 판독 시간

🎙️ 음성 인식 판독

방사선 전문의의 생산성 향상:

의사: "환자는 63세 남성입니다. 흉부 CT 소견상..."
      ↓
음성 인식 엔진 (Dragon Medical, etc.)
      ↓
실시간 텍스트 변환 → RIS 판독 시스템
      ↓
의사 검토 및 승인 → 최종 보고서
  • 정확도 — 95% 이상 (의료 용어 학습 후)
  • 속도 — 타이핑 대비 3배 빠름
  • 편의성 — Hands-free 판독 가능

5.3 영상 저장 전송 시스템 (PACS)

🖼️ DICOM 표준

Digital Imaging and Communications in Medicine:

  • 표준 이미지 형식 — .dcm 파일
  • 메타데이터 — 환자 정보, 촬영 정보, 장비 정보
  • DICOM 서비스
    • C-STORE: 영상 저장
    • C-FIND: 영상 검색
    • C-MOVE: 영상 전송
    • C-GET: 영상 가져오기

📦 DICOM 파일 구조

DICOM File
├── File Meta Information
│   ├── Transfer Syntax UID
│   └── Media Storage SOP Class UID
└── Data Set
    ├── Patient Module
    │   ├── (0010,0010) Patient Name: 김민지
    │   ├── (0010,0020) Patient ID: P123456
    │   └── (0010,0030) Birth Date: 19850312
    ├── Study Module
    │   ├── (0020,000D) Study Instance UID
    │   └── (0008,0020) Study Date: 20240115
    ├── Series Module
    │   ├── (0020,000E) Series Instance UID
    │   └── (0008,0060) Modality: CT
    └── Image Module
        ├── (0028,0010) Rows: 512
        ├── (0028,0011) Columns: 512
        └── (7FE0,0010) Pixel Data: [...binary...]

💾 PACS 아키텍처

           촬영 장비 (Modality)
         CT / MRI / X-ray / US
                  ↓ DICOM
          ┌───────────────────┐
          │   PACS Server     │
          │  - 영상 저장       │
          │  - 압축/해제       │
          │  - 백업            │
          └───────────────────┘
                  ↓
    ┌─────────────┼─────────────┐
    │             │             │
Hot Storage  Warm Storage  Cold Storage
(SSD)        (HDD)         (Tape/Cloud)
최근 3개월    1년 이내      1년 이상

🖥️ PACS Viewer

영상 판독 및 조회 도구:

  • 멀티 모니터 — 3-5개 고해상도 모니터 (3MP 이상)
  • Window/Level — 밝기/대비 조정
  • 측정 도구 — 거리, 각도, 면적 측정
  • 3D Reconstruction — 3차원 영상 재구성
  • Hanging Protocol — 검사 종류별 자동 레이아웃

5.4 AI 영상 분석

🤖 AI 보조 진단

딥러닝으로 영상 자동 분석:

  • 폐 결절 검출 — CT에서 폐암 의심 결절 표시
  • 뇌출혈 감지 — 응급 CT에서 출혈 자동 경고
  • 골절 검출 — X-ray에서 골절선 표시
  • 유방암 스크리닝 — Mammography AI 판독

효과: AI 보조 진단 도입 후 폐 결절 검출율 15% 증가, 판독 시간 30% 단축

5.5 통합 및 상호 운용성

🔗 HIS-LIS-RIS-PACS 통합

HIS/EMR
   │
   ├─→ LIS (HL7 ORM 메시지: 검사 오더)
   │    ↓
   │   LIS (HL7 ORU 메시지: 검사 결과)
   │    ↓
   │   HIS/EMR
   │
   └─→ RIS (HL7 ORM 메시지: 영상 오더)
        ↓
       RIS (DICOM Modality Worklist)
        ↓
       촬영 장비
        ↓
       PACS (DICOM C-STORE: 영상 저장)
        ↓
       RIS (판독 완료 → HL7 ORU)
        ↓
       HIS/EMR (판독 결과 표시)

참조 표준 및 적용 지침

본 표준은 다음 ALLOW 출처를 단일 진실원으로 삼아 데이터 교환·서명· 감사 증적 인터페이스를 제공합니다. 구현체는 이들 원전을 대체하지 않고, 데이터 교환·감사 증적·자동 검증을 위한 상호운용 계층을 정의합니다.

  • OpenAPI Specification 3.1 — REST API 정의
  • JSON Schema 2020-12 — 스키마 검증
  • ISO/IEC 27001:2022 — 정보보안 경영시스템
  • ISO/IEC 27018:2019 — 공용 클라우드의 PII 보호
  • ISO/IEC 17065:2012 — 제품 인증 기관
  • CycloneDX 1.5 / SPDX 2.3 — 소프트웨어 BOM
  • Sigstore (DSSE 봉투, Rekor 투명성 로그) — 서명 앵커링
  • in-toto Attestation Framework 1.0 — 빌드 출처 검증
  • IETF RFC 9700 — OAuth 2.1
  • IETF RFC 9457 — Problem Details for HTTP APIs
  • IETF RFC 8615 — well-known URIs

실무 적용 체크리스트

  • 매니페스트 무결성: 모든 발급은 SHA-256 또는 Sigstore 서명을 부착해 변조를 차단합니다. 발급 후 변경은 새 매니페스트로 처리 되며 이전 매니페스트는 supersede 마킹과 함께 보존됩니다.
  • 보관 기간: ISO/IEC 17065:2012 감사 기대치를 충족하기 위해 7년 이상의 보관 정책을 명시합니다. 관할 규정이 더 길면 그것을 따릅니다.
  • 버전 관리: 정정·확장은 시맨틱 버저닝(SemVer 2.0.0)을 따릅니다. 필수 요건의 추가는 마이너, 호환 단절은 메이저 버전 갱신을 트리거합니다. 메이저 변경은 최소 90일 병행 호환 기간을 거치며 그 기간 동안 신·구 두 버전 모두 WIA 등록부에 Stable로 표기 됩니다.
  • 부정결과 보고: 누락된 지표는 사유와 함께 매니페스트에 기록 되어야 합니다. 사유가 없는 누락은 감사 시 실패로 간주됩니다.
  • 키 회전: 매니페스트 서명 키는 배포 정책에 따라 회전됩니다. 이전 키는 공개된 키 이력에 보존되어 과거 매니페스트의 사후 검증을 보장합니다.

참조 토폴로지

구현체의 운용 토폴로지는 다음 4가지 중 하나로 매니페스트에 선언됩니다: 단일 테넌트 엣지(조직당 1개 런타임), 멀티 테넌트 게이트웨이(헤더 기반 격리로 다중 테넌트 서비스), 연합 메시(다수 런타임이 디렉토리 서비스 통해 상호 인식), 에어갭 배치(라이브 연결 없이 서명된 패키지를 out-of-band 채널로 전송). 각 토폴로지는 참조 테스트 벡터를 포함하며 구현체가 자신의 배포가 표준 패턴과 일치하는지 검증할 수 있도록 합니다.
