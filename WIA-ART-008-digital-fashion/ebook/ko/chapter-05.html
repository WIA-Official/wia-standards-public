제5장: Phase 2 - API 인터페이스

弘益人間 · 널리 인간을 이롭게 하라

개요

Phase 1이 데이터 형식을 정의했다면, Phase 2는 이 데이터를 프로그래밍 방식으로 생성·조회·수정·삭제(CRUD)할 수 있는 표준화된 인터페이스를 제공합니다. 개발자는 WIA-ART-008 API를 통해 디지털 패션 자산을 웹 애플리케이션, 모바일 앱, 게임 엔진, 메타버스 플랫폼에 통합할 수 있습니다.

이 장에서는 RESTful API 설계, TypeScript/Python SDK 사용법, GraphQL 쿼리, OAuth 2.0 인증, 대용량 파일 업로드 최적화, 에러 처리, 속도 제한 등 실제 프로덕션 환경에 필요한 모든 요소를 다룹니다.

RESTful API 설계

1. 기본 엔드포인트

WIA-ART-008 API는 REST 아키텍처 원칙을 따르며, 다음과 같은 기본 URL 구조를 사용합니다: 

Base URL: https://api.wiastandards.org/art-008/v1

모든 요청은 다음 헤더를 포함해야 합니다:
- Authorization: Bearer {access_token}
- Content-Type: application/json
- Accept: application/json
엔드포인트 메서드 설명 인증 필요
/garments GET 의상 목록 조회 (필터, 페이징 지원) 선택
/garments/{id} GET 특정 의상 상세 정보 조회 선택
/garments POST 새 의상 생성 (메타데이터만) 필수
/garments/{id} PUT 의상 메타데이터 전체 교체 필수
/garments/{id} PATCH 의상 메타데이터 부분 수정 필수
/garments/{id} DELETE 의상 삭제 필수
/garments/{id}/assets POST 3D 모델/텍스처 업로드 필수
/garments/{id}/assets/{type} GET 특정 자산 파일 다운로드 선택
/collections GET/POST 컬렉션 관리 선택/필수
/creators/{wallet} GET 창작자 프로필 및 작품 목록 선택

2. 의상 생성

import * as fs from 'fs';

async function createGarment() {
  // 1. 메타데이터로 의상 생성
  const garment = await client.garments.create({
    name: "Cyberpunk Jacket",
    description: "Neon-lit futuristic jacket with LED strips",
    tags: ["cyberpunk", "jacket", "sci-fi"],
    creator: {
      name: "FutureFashion Studio",
      wallet: "0x742d35Cc...",
      email: "hello@futurefashion.io"
    },
    licensing: {
      license: "CC-BY-NC-4.0",
      commercial_use: false,
      modifications_allowed: true,
      attribution_required: true
    },
    semantic: {
      garment_type: "jacket",
      style: "cyberpunk",
      target_gender: "unisex",
      body_parts: ["torso", "arms"]
    }
  });

  console.log('Garment created:', garment.id);

  // 2. 3D 모델 및 텍스처 업로드
  const modelBuffer = fs.readFileSync('./jacket.glb');
  const textureBuffers = [
    fs.readFileSync('./diffuse.png'),
    fs.readFileSync('./emissive.png')
  ];

  const uploadResult = await client.garments.uploadAssets(garment.id, {
    model: modelBuffer,
    textures: [
      { type: 'diffuse', buffer: textureBuffers[0] },
      { type: 'emissive', buffer: textureBuffers[1] }
    ]
  });

  console.log('Assets uploaded:', uploadResult);

  return garment;
}

createGarment().catch(console.error);

async function downloadGarment(garmentId: string) {
  // 전체 패키지 다운로드 (.wia008 파일)
  const packageBuffer = await client.garments.downloadPackage(garmentId);
  fs.writeFileSync('./downloaded.wia008', packageBuffer);

  // 또는 개별 자산 다운로드
  const metadata = await client.garments.get(garmentId);
  const modelBuffer = await client.garments.downloadAsset(
    garmentId,
    'model'
  );
  fs.writeFileSync('./model.glb', modelBuffer);

  const diffuseTexture = await client.garments.downloadAsset(
    garmentId,
    'texture',
    { type: 'diffuse' }
  );
  fs.writeFileSync('./diffuse.png', diffuseTexture);
}

인증과 권한 관리

1. OAuth 2.0 인증 플로우

WIA-ART-008 API는 OAuth 2.0 프로토콜을 사용하여 사용자 인증 및 권한 부여를 처리합니다.

플로우 타입 용도 권장 상황
Authorization Code 웹 애플리케이션 백엔드 서버가 있는 경우
PKCE 모바일 앱, SPA 클라이언트 시크릿 보호 어려운 경우
API Key 서버-서버 통신 자동화 스크립트, CI/CD
Wallet Signature Web3 dApp 블록체인 지갑 인증

2. Authorization Code 플로우

// 1. 사용자를 인증 페이지로 리다이렉트
const authUrl = `https://auth.wiastandards.org/authorize?` +
  `client_id=${CLIENT_ID}&` +
  `redirect_uri=${REDIRECT_URI}&` +
  `response_type=code&` +
  `scope=garments:read garments:write`;

window.location.href = authUrl;

// 2. 리다이렉트 콜백에서 authorization code 받기
// http://yourapp.com/callback?code=AUTH_CODE

// 3. Authorization code를 access token으로 교환
const response = await fetch('https://auth.wiastandards.org/token', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    grant_type: 'authorization_code',
    code: 'AUTH_CODE',
    client_id: CLIENT_ID,
    client_secret: CLIENT_SECRET,
    redirect_uri: REDIRECT_URI
  })
});

const { access_token, refresh_token, expires_in } = await response.json();

// 4. Access token으로 API 호출
const garments = await fetch('https://api.wiastandards.org/art-008/v1/garments', {
  headers: {
    'Authorization': `Bearer ${access_token}`
  }
});

3. Web3 지갑 서명 인증

import { ethers } from 'ethers';

async function authenticateWithWallet() {
  // 1. MetaMask 연결
  const provider = new ethers.BrowserProvider(window.ethereum);
  const signer = await provider.getSigner();
  const address = await signer.getAddress();

  // 2. 서버에서 nonce 요청
  const nonceResponse = await fetch(
    `https://api.wiastandards.org/art-008/v1/auth/nonce?wallet=${address}`
  );
  const { nonce } = await nonceResponse.json();

  // 3. nonce에 서명
  const message = `WIA-ART-008 Authentication\nNonce: ${nonce}`;
  const signature = await signer.signMessage(message);

  // 4. 서명으로 access token 받기
  const tokenResponse = await fetch(
    'https://api.wiastandards.org/art-008/v1/auth/wallet',
    {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ address, signature, nonce })
    }
  );

  const { access_token } = await tokenResponse.json();

  // 5. Client에 토큰 설정
  client.setAccessToken(access_token);
}

에러 처리

표준 에러 응답

모든 에러는 일관된 JSON 형식으로 반환됩니다: 

{
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "Invalid garment metadata",
    "details": [
      {
        "field": "garment.technical.polycount",
        "issue": "must be a positive integer"
      }
    ],
    "request_id": "req_abc123",
    "timestamp": "2024-03-15T10:30:00Z"
  }
}
HTTP 코드 에러 코드 설명 해결 방법
400 VALIDATION_ERROR 요청 데이터 검증 실패 details 필드 확인 후 수정
401 UNAUTHORIZED 인증 실패 토큰 갱신 또는 재인증
403 FORBIDDEN 권한 부족 리소스 소유자 확인
404 NOT_FOUND 리소스 없음 ID 확인
413 PAYLOAD_TOO_LARGE 파일 크기 초과 압축 또는 LOD 사용
429 RATE_LIMIT_EXCEEDED 속도 제한 초과 Retry-After 헤더 확인
500 INTERNAL_ERROR 서버 오류 재시도 또는 지원팀 문의

SDK 에러 처리

try {
  const garment = await client.garments.create({ ... });
} catch (error) {
  if (error instanceof WIAValidationError) {
    console.error('Validation failed:', error.details);
    error.details.forEach(detail => {
      console.log(`- ${detail.field}: ${detail.issue}`);
    });
  } else if (error instanceof WIARateLimitError) {
    const retryAfter = error.retryAfter; // seconds
    console.log(`Rate limited. Retry after ${retryAfter}s`);
    await sleep(retryAfter * 1000);
    // 재시도 로직
  } else if (error instanceof WIAAuthenticationError) {
    console.error('Authentication failed. Re-authenticating...');
    await client.refreshToken();
  } else {
    console.error('Unknown error:', error);
  }
}

핵심 요점

  • RESTful API: 표준 HTTP 메서드(GET, POST, PUT, PATCH, DELETE)와 명확한 엔드포인트 구조로 의상 CRUD 작업 수행. 필터링, 페이징, 정렬 지원.
  • SDK 지원: TypeScript와 Python 공식 SDK 제공. 복잡한 HTTP 요청을 간단한 메서드 호출로 추상화. 자동 재시도, 에러 처리, 타입 안전성 보장.
  • 대용량 파일 처리: 멀티파트 폼 데이터와 Resumable Upload로 효율적인 3D 모델/텍스처 업로드. 청크 단위 전송으로 네트워크 오류 대응.
  • OAuth 2.0 인증: Authorization Code, PKCE, API Key, Web3 Wallet Signature 등 다양한 인증 방식 지원. Access token 및 refresh token 관리.
  • 표준 에러 처리: 일관된 JSON 에러 응답 형식. 상세한 검증 오류 정보, request_id로 추적 가능. SDK는 에러 타입별 예외 클래스 제공.
  • 속도 제한: 플랜별 요청 쿼터(분당, 일당). 응답 헤더에 남은 한도와 리셋 시간 포함. 초과 시 429 에러와 Retry-After 정보 제공.
