제4장: Phase 1 - 데이터 형식 표준

弘益人間 · 널리 인간을 이롭게 하라

개요

Phase 1은 WIA-ART-008 표준의 기초이자 가장 중요한 단계입니다. 디지털 패션 자산을 어떻게 저장하고, 어떤 메타데이터를 포함하며, 어떻게 다른 시스템으로 전달할 것인가를 정의합니다. 이 장에서는 JSON 기반 메타데이터 스키마, glTF 2.0 3D 모델 형식, 그리고 디지털 패션 고유의 확장(Extensions)을 상세히 다룹니다.

Phase 1의 핵심 원칙은 "한 번 제작하면 어디서나 사용 가능"입니다. CLO3D에서 만든 의상이 Decentraland, Roblox, VRChat 어디서든 동일한 품질로 작동하려면, 모든 필수 정보를 손실 없이 담을 수 있는 표준 형식이 필요합니다. WIA-ART-008 Phase 1은 바로 이 문제를 해결합니다.

WIA-ART-008 파일 구조

WIA-ART-008 디지털 패션 자산은 다음과 같은 파일들로 구성됩니다: 

my-fashion-item/
├── metadata.json          # WIA-ART-008 메타데이터 (필수)
├── model.glb              # glTF 2.0 바이너리 3D 모델 (필수)
├── textures/              # 텍스처 이미지들
│   ├── diffuse.png        # 기본 색상 맵
│   ├── normal.png         # 노멀 맵
│   ├── metallic.png       # 메탈릭 맵
│   └── roughness.png      # 러프니스 맵
├── animations/            # 애니메이션 파일들 (선택)
│   ├── walk.glb
│   └── idle.glb
└── preview.png            # 미리보기 이미지 (권장)

패키징 형식

실제 배포 시에는 이 모든 파일을 하나의 패키지로 묶어야 합니다. WIA-ART-008은 두 가지 패키징 방법을 지원합니다:

형식 확장자 용도 장점
ZIP 아카이브 .wia008 일반 배포, 다운로드 압축률 높음, 기존 도구 사용 가능
IPFS CAR .car 탈중앙화 저장, NFT 영구 보존, 콘텐츠 주소 지정

JSON 메타데이터 스키마

1. 루트 구조

모든 WIA-ART-008 자산은 metadata.json 파일을 루트에 포함해야 하며, 다음과 같은 최상위 필드를 가집니다: 

{
  "$schema": "https://schemas.wiastandards.org/art-008/v1.0/schema.json",
  "standard": "WIA-ART-008",
  "version": "1.0",
  "garment": {
    // 의상 정보
  }
}
필드 타입 필수 설명
$schema string (URI) 필수 JSON Schema 정의 위치
standard string 필수 표준 식별자, 항상 "WIA-ART-008"
version string 필수 표준 버전, 현재 "1.0"
garment object 필수 의상 상세 정보

2. garment 객체 - 식별 정보

"garment": {

필드 타입 필수 설명

name string 필수 의상 이름
description string 권장 의상 상세 설명
tags string[] 선택 검색용 태그 배열
language string (ISO 639-1) 선택 메타데이터 언어 코드
created string (ISO 8601) 필수 생성 일시
modified string (ISO 8601) 선택 마지막 수정 일시

블록체인 기반 출처 증명

wallet 필드는 이더리움 주소를 사용하여 창작자의 정체성을 암호학적으로 증명합니다. 이 주소로 메타데이터에 디지털 서명을 하면, 누구도 위조할 수 없는 출처 증명이 가능합니다. NFT로 발행 시 이 주소가 원작자 주소로 자동 설정됩니다.

glTF 2.0 확장 (Extensions)

glTF 2.0은 표준 3D 정보를 담지만, 디지털 패션 고유의 정보는 Extensions로 추가합니다. WIA-ART-008은 다음 커스텀 확장을 정의합니다:

1. WIA_fabric_physics 확장

원단의 물리적 특성을 정의하여 실시간 시뮬레이션을 가능하게 합니다. 

{
  "extensions": {
    "WIA_fabric_physics": {
      "fabric_type": "virtual-silk",
      "physical_properties": {
        "density": 0.6,              // g/cm³
        "thickness": 0.2,            // mm
        "stretch_stiffness": 1000,   // N/m
        "bend_stiffness": 0.05,      // N⋅m²/rad
        "shear_stiffness": 500,      // N/m
        "damping": 0.1,              // 감쇠 계수
        "friction": 0.3              // 마찰 계수
      },
      "behavior": {
        "drape": "high",             // low | medium | high
        "wrinkle_resistance": "low", // low | medium | high
        "elasticity": "low"          // low | medium | high
      },
      "appearance": {
        "sheen": 0.8,                // 광택 0-1
        "translucency": 0.2          // 반투명도 0-1
      }
    }
  }
}
원단 타입 밀도 (g/cm³) 신축성 드레이프 광택
virtual-silk 0.6 낮음 높음 0.8
virtual-cotton 0.8 낮음 중간 0.2
virtual-denim 1.2 중간 낮음 0.1
virtual-leather 1.0 낮음 낮음 0.6
virtual-spandex 1.1 높음 중간 0.4

2. WIA_layering 확장

의상의 레이어 구조를 정의합니다. 겉감, 안감, 심지, 장식 등이 어떻게 겹쳐지는지 명시합니다.

}

검증과 품질 보증

1. 스키마 검증

모든 WIA-ART-008 메타데이터는 JSON Schema로 자동 검증됩니다: 

// Node.js에서 검증 예시
import Ajv from 'ajv';
import schema from '@wia/art-008-schema';

const ajv = new Ajv();
const validate = ajv.compile(schema);

const metadata = JSON.parse(fs.readFileSync('metadata.json'));
const valid = validate(metadata);

if (!valid) {
  console.error('검증 실패:', validate.errors);
} else {
  console.log('✅ 유효한 WIA-ART-008 메타데이터');
}

2. 품질 레벨 기준

레벨 폴리곤 텍스처 리깅 물리 용도
모바일 < 10K 512-1024 선택 ❌ 스마트폰 메타버스
표준 10-50K 1024-2048 권장 기본 일반 PC, 웹
고품질 50-150K 2048-4096 필수 고급 고사양 PC, VR
아카이브 > 150K 4096+ 완전 실시간 보존, 영화 제작

핵심 요점

  • 파일 구조: metadata.json (메타데이터) + model.glb (3D 모델) + textures/ (텍스처) + preview.png (미리보기)를 .wia008 ZIP 또는 .car IPFS 패키지로 묶음.
  • JSON 스키마: 표준, 버전, 의상 정보, 창작자, 라이선스, 자산, 기술 사양, 의미 정보, 호환성을 포함하는 포괄적 메타데이터 구조.
  • glTF 2.0 기반: 산업 표준 3D 형식 채택으로 광범위한 호환성 확보. 바이너리 .glb 형식으로 압축 효율 극대화.
  • 커스텀 확장: WIA_fabric_physics (원단 물리), WIA_layering (레이어 구조), WIA_fit_metadata (사이즈/핏), WIA_provenance (블록체인 출처) 확장으로 디지털 패션 고유 정보 저장.
  • 블록체인 통합: 창작자 지갑 주소, NFT 정보, 디지털 서명으로 위조 불가능한 출처 증명. IPFS로 영구 저장.
  • 품질 보증: JSON Schema 자동 검증, 4단계 품질 레벨(모바일/표준/고품질/아카이브)로 용도별 최적화.

복습 질문

  1. WIA-ART-008 파일 구조의 필수 파일(metadata.json, model.glb)과 선택 파일(animations/, preview.png)을 나열하고, 두 가지 패키징 형식(.wia008 ZIP, .car IPFS)의 각 장점과 용도를 설명하십시오.
  2. JSON 메타데이터의 garment 객체 내 주요 하위 객체(creator, licensing, assets, technical, semantic, compatibility)를 설명하고, 각각이 어떤 정보를 담고 있는지 예시와 함께 서술하십시오.
  3. Creative Commons 라이선스(CC0, CC-BY, CC-BY-NC, CC-BY-SA)를 비교하고, 각 라이선스의 상업적 사용, 변형 허용, 재배포 조건, 귀속 표시 요구사항을 표로 정리하십시오.
  4. WIA_fabric_physics 확장의 주요 파라미터(density, stretch_stiffness, bend_stiffness, damping, friction)를 설명하고, 다섯 가지 원단 타입(virtual-silk, cotton, denim, leather, spandex)의 물리적 특성을 비교하십시오.
  5. WIA_layering 확장이 의상의 레이어 구조(겉감, 안감, 장식)를 어떻게 표현하는지 설명하고, WIA_fit_metadata 확장이 사이즈와 핏 정보를 어떻게 표준화하여 아바타 자동 맞춤을 가능하게 하는지 서술하십시오.
  6. WIA_provenance 확장의 블록체인 통합 구조를 설명하고, 창작자 지갑 주소, 스마트 컨트랙트, NFT 토큰 ID, IPFS CID, 디지털 서명이 어떻게 위조 불가능한 출처 증명을 제공하는지 분석하십시오. 품질 레벨 4단계(모바일/표준/고품질/아카이브)의 기술 요구사항을 비교하십시오.
