# 3장: WIA-IND-012 표준 개요

## 개요

이 장은 WIA-IND-012 피트니스 트래킹 표준의 포괄적인 아키텍처 개요를 제공하며, 다양한 구성요소가 어떻게 함께 작동하여 통합되고 상호운용 가능한 피트니스 트래킹 생태계를 만드는지 설명합니다.

## 3.2 핵심 구성요소

### 3.2.1 활동 추적기

**책임:**
- 걸음 수 및 보폭 감지
- 거리 측정 (GPS 및 추정)
- 페이스 및 속도 계산
- 고도 추적
- 경로 기록

**주요 기능:**
```typescript
interface ActivityTracker {
  // 실시간 추적
  startTracking(activityType: ActivityType): void;
  pauseTracking(): void;
  stopTracking(): ActivitySummary;

  // 걸음 감지
  getStepCount(): number;
  getStrideLength(): number;

  // 거리 및 위치
  getCurrentDistance(): number;
  getCurrentPosition(): GPSPoint;
}
```

### 3.2.2 심박수 모니터

**책임:**
- 실시간 심박수 측정
- 심박수 구역 계산
- HRV 분석
- VO2 Max 추정

**심박수 구역 시스템:**
```
구역 5 (최대)      ██████ 최대 심박수의 90-100%
구역 4 (역치)      ██████ 최대 심박수의 80-90%
구역 3 (템포)      ██████ 최대 심박수의 70-80%
구역 2 (유산소)    ██████ 최대 심박수의 60-70%
구역 1 (회복)      ██████ 최대 심박수의 50-60%
```

### 3.2.3 칼로리 계산기

**책임:**
- BMR (기초 대사율) 계산
- TDEE (총 일일 에너지 소비) 추정
- 활동 기반 칼로리 계산
- EPOC (애프터번) 추정

**계산 방법:**
```typescript
interface CalorieCalculator {
  // 기초 대사
  calculateBMR(profile: UserProfile): number;
  calculateTDEE(bmr: number, activityLevel: ActivityLevel): number;

  // 활동 칼로리
  calculateActivityCalories(
    activity: Activity,
    duration: number,
    userWeight: number
  ): number;

  // 방법 선택
  useMETMethod(met: number, weight: number, duration: number): number;
  useHeartRateMethod(avgHR: number, profile: UserProfile): number;
}
```

### 3.2.4 운동 로거

**책임:**
- 구조화된 운동 기록
- 인터벌 및 랩 추적
- 근력 트레이닝 세트/반복 로깅
- 성능 분석

**지원되는 운동 유형:**
- 유산소: 달리기, 사이클링, 수영, 로잉
- 근력: 웨이트 트레이닝, 맨몸 운동, 크로스핏
- 스포츠: 농구, 축구, 테니스, 골프
- 마음-몸: 요가, 필라테스, 태극권

### 3.2.5 건강 지표 관리자

**책임:**
- 신체 구성 추적
- 체중 및 BMI 모니터링
- 수면 품질 분석
- 회복 점수 계산

### 3.2.6 목표 관리자

**책임:**
- 목표 생성 및 추적
- 진행 상황 모니터링
- 업적 시스템
- 알림 및 통지

**일반적인 목표 유형:**
- 일일 걸음 목표 (예: 10,000걸음)
- 주간 거리 (예: 25km)
- 월간 운동 (예: 20회)
- 체중 감량/증가 목표

## 3.3 데이터 흐름 아키텍처

### 3.3.1 실시간 데이터 흐름

```
┌──────────────┐
│   센서       │  (GPS, 심박수, 가속도계 등)
└──────┬───────┘
       │
       ↓
┌──────────────┐
│ 데이터 버퍼  │  (원시 센서 판독값)
└──────┬───────┘
       │
       ↓
┌──────────────┐
│   필터       │  (잡음 감소, 스무딩)
└──────┬───────┘
       │
       ↓
┌──────────────┐
│  프로세서    │  (걸음 감지, 심박수 구역 계산)
└──────┬───────┘
       │
       ↓
┌──────────────┐
│   지표       │  (거리, 페이스, 칼로리)

       │
       ↓
┌──────────────┐

       │
       ↓
┌──────────────┐

└──────────────┘
```

```
┌─────────────┐

└─────────────┘                   │          │
                                  │   앱     │
┌─────────────┐                   │          │
```

### 3.5.2 데이터 개인정보 보호

**설계 단계부터 개인정보 보호 원칙:**

1. **데이터 최소화**
   - 필요한 데이터만 수집
   - 보존 기간 후 삭제

2. **사용자 제어**
   - 세분화된 동의 관리
   - 쉬운 데이터 내보내기

3. **투명성**
   - 명확한 개인정보 보호 정책
   - 데이터 사용 통지

## 3.7 인증 수준

### 레벨 1: 기본
**요구사항:**
- ✓ 걸음 수
- ✓ 기본 활동 로깅
- ✓ 수동 칼로리 입력
- ✓ 간단한 목표

### 레벨 2: 표준
**요구사항:**
- ✓ 모든 레벨 1 기능
- ✓ GPS 추적
- ✓ 심박수 모니터링
- ✓ 자동 칼로리 계산

### 레벨 3: 고급
**요구사항:**
- ✓ 모든 레벨 2 기능
- ✓ 다중 스포츠 지원
- ✓ HRV 및 VO2 Max
- ✓ 트레이닝 부하 계산

### 레벨 4: 전문가
**요구사항:**
- ✓ 모든 레벨 3 기능
- ✓ 의료급 정확도
- ✓ 의료 통합(FHIR)
- ✓ 연구급 데이터 내보내기

## 핵심 요점

✓ WIA-IND-012는 유연성을 위한 모듈식 구성요소 아키텍처 사용

✓ 6개 핵심 구성요소가 활동, 심박수, 칼로리, 운동, 건강, 목표 처리

✓ 데이터는 수집 → 처리 → 저장 → 분석 → 표시를 통해 흐름

✓ 기기, 플랫폼, 타사 서비스를 지원하는 여러 통합 지점

✓ 보안 아키텍처는 심층 방어 및 설계 단계부터 개인정보 보호 구현

✓ 4개 인증 수준이 다양한 사용 사례 및 요구사항 수용
