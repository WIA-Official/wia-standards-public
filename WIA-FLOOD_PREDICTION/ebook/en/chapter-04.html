Chapter 4: Machine Learning for Flood Prediction

4.1 Why Machine Learning?

Traditional hydrological models (HEC-RAS, SWAT) require extensive calibration and struggle with complex non-linear relationships. Machine learning excels at pattern recognition in high-dimensional data.

ML Advantage: Learns from 40+ years of satellite imagery, weather data, and observed floods to identify subtle precursors humans might miss.

4.2 LSTM Networks for Time-Series Forecasting

LSTM (Long Short-Term Memory) is a type of recurrent neural network designed to learn from sequential data with long-term dependencies.

Architecture

Input: 30-day time series (8 features per day)
  ↓
LSTM Layer 1 (128 units) → Captures weekly patterns
  ↓
Dropout (0.2) → Prevents overfitting
  ↓
LSTM Layer 2 (64 units) → Refines predictions
  ↓
Dense Layer (32 units, ReLU)
  ↓
Output: 7-day flood probability (Sigmoid)

Input Features

  1. Accumulated precipitation (mm)
  2. Soil moisture (%)
  3. River discharge (m³/s)
  4. Temperature (°C)
  5. Water extent from SAR (%)
  6. NDWI (Sentinel-2)
  7. Elevation (m)
  8. Distance to river (m)

4.3 CNN for Spatial Pattern Recognition

CNN (Convolutional Neural Network) analyzes 2D satellite images to identify spatial flood patterns.

Architecture

Input: 256×256 image (8 channels)
  ↓
Conv2D (32 filters, 3×3) + ReLU → Detects edges
MaxPool2D (2×2) → Reduces dimensionality
  ↓
Conv2D (64 filters, 3×3) + ReLU → Detects textures
MaxPool2D (2×2)
  ↓
Conv2D (128 filters, 3×3) + ReLU → Detects objects
  ↓
Flatten → Dense(128) → Output: Flood probability map

Input Channels

  1. Sentinel-1 VV backscatter
  2. Sentinel-1 VH backscatter
  3. Sentinel-2 NDWI
  4. DEM (elevation)
  5. Slope (degrees)
  6. TWI (Topographic Wetness Index)
  7. Precipitation forecast (mm)
  8. Distance to river (m)

4.4 Ensemble Fusion

Combine LSTM (temporal) + CNN (spatial) + HEC-RAS (physics-based) predictions:

P_flood = 0.4 × P_LSTM + 0.4 × P_CNN + 0.2 × P_HEC-RAS

Result: 87% accuracy (vs. 82% for LSTM alone, 79% for CNN alone)

4.5 Training Data

  • Historical Floods: 2,400 events (2010-2025) from NOAA Storm Events Database
  • Satellite Imagery: 500,000 Sentinel-1/2 scenes
  • Weather Data: 15 years of NOAA GFS archives
  • River Gauges: 9,000 USGS gauges, 15-minute data

Data Augmentation

  • Rotate satellite images (90°, 180°, 270°)
  • Add Gaussian noise to simulate sensor errors
  • Temporal shifting (±3 days) to increase diversity

4.7 Explainability (XAI)

ML models are "black boxes." To build trust, we use:

  • SHAP (Shapley Additive Explanations): Shows which features drive predictions
  • Saliency Maps: Highlights important regions in satellite images
  • Attention Mechanisms: LSTM learns to focus on critical time steps
Example: SHAP analysis reveals that 3-day precipitation accumulation has 3.2x more impact than soil moisture on flash flood predictions.

4.8 Key Takeaways

  • LSTM networks excel at learning temporal patterns from multi-day sequences
  • CNNs extract spatial flood patterns from satellite imagery
  • Ensemble models (LSTM + CNN + physics) outperform individual approaches
  • Training on 15 years of historical data achieves 87% accuracy
  • Explainability tools (SHAP, saliency maps) make predictions interpretable
