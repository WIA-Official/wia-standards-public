Chapter 3: API Endpoints

Device Management, Data Streaming, and Session Control

WIA BCI Standard v1.0.0

3.1 Overview

The WIA BCI API provides a device-agnostic programming interface for brain-computer interface applications. It abstracts hardware differences, allowing developers to write code that works with any BCI device.

Design Goals

  • Device Agnostic: Same API for all BCI devices
  • Event-Driven: Asynchronous event handling
  • Type-Safe: Strong typing in TypeScript
  • Cross-Platform: TypeScript and Python implementations

3.2 Core API: WiaBci Class

The main entry point for all BCI operations:

TypeScript Interface

class WiaBci {
  // Constructor
  constructor(options?: WiaBciOptions);

  // Connection Management
  connect(config: DeviceConfig): Promise<void>;
  disconnect(): Promise<void>;
  isConnected(): boolean;

  // Device Discovery
  listDevices(): Promise<DeviceInfo[]>;
  getDeviceInfo(): DeviceInfo | null;

  // Data Streaming
  startStream(): Promise<void>;
  stopStream(): Promise<void>;
  isStreaming(): boolean;

  // Event Handling
  on<T extends EventType>(event: T, handler: EventHandler<T>): void;
  off<T extends EventType>(event: T, handler: EventHandler<T>): void;
  once<T extends EventType>(event: T, handler: EventHandler<T>): void;
  emit<T extends EventType>(event: T, data: EventData<T>): void;

  // State Management
  getState(): BciState;
  getChannels(): ChannelInfo[];

  // Resource Cleanup
  dispose(): void;
}

3.7 Usage Examples

Basic Connection (TypeScript)

import { WiaBci } from 'wia-bci';

async function main() {
  const bci = new WiaBci({ logLevel: 'info' });

  // List available devices
  const devices = await bci.listDevices();
  console.log('Available devices:', devices);

  // Connect to device
  await bci.connect({
    type: 'eeg_headset',
    device: { manufacturer: 'OpenBCI' },
    acquisition: {
      samplingRate: 250,
      channels: ['Fp1', 'Fp2', 'C3', 'C4', 'O1', 'O2']
    }
  });

  console.log('Connected:', bci.isConnected());

  // Receive data
  bci.on('signal', (event) => {
    console.log(`Sample ${event.sampleIndex}:`, event.data);
  });

  // Start streaming
  await bci.startStream();

  // Stop after 10 seconds
  setTimeout(async () => {
    await bci.stopStream();
    await bci.disconnect();
    bci.dispose();
  }, 10000);
}

main();

Motor Imagery Classification

import { WiaBci, SignalProcessor, FeatureExtractor } from 'wia-bci';

const bci = new WiaBci();
const buffer: Float32Array[] = [];
const EPOCH_SIZE = 250; // 1 second @ 250Hz

await bci.connect({ type: 'eeg_headset' });

bci.on('signal', (event) => {
  buffer.push(event.data);

  if (buffer.length >= EPOCH_SIZE) {
    // Create 1-second epoch
    const epoch = concatenateBuffers(buffer.splice(0, EPOCH_SIZE));

    // Preprocessing
    const filtered = SignalProcessor.bandpass(epoch, 8, 30, 250);

    // Feature extraction
    const features = FeatureExtractor.extractAll(filtered, 250);
    const bandPowers = SignalProcessor.allBandPowers(filtered, 250);

    // Classify (using pre-trained model)
    const prediction = classifier.predict({ ...features, ...bandPowers });

    // Emit classification event
    bci.emit('classification', {
      timestamp: Date.now(),
      classId: prediction.class,
      className: ['rest', 'left', 'right', 'feet'][prediction.class],
      confidence: prediction.confidence
    });
  }
});

await bci.startStream();

Python Example

import asyncio
from wia_bci import WiaBci

async def main():
    bci = WiaBci(log_level='info')

    # List devices
    devices = await bci.list_devices()
    print(f"Available devices: {devices}")

    # Connect
    await bci.connect(
        type='eeg_headset',
        device={'manufacturer': 'OpenBCI'},
        acquisition={
            'sampling_rate': 250,
            'channels': ['Fp1', 'Fp2', 'C3', 'C4', 'O1', 'O2']
        }
    )

    print(f"Connected: {bci.is_connected()}")

    # Data handler (decorator style)
    @bci.on('signal')
    def on_signal(event):
        print(f"Sample {event.sample_index}: {event.data}")

    # Start streaming
    await bci.start_stream()

    # Wait 10 seconds
    await asyncio.sleep(10)

    # Cleanup
    await bci.stop_stream()
    await bci.disconnect()
    bci.dispose()

asyncio.run(main())

3.8 Error Handling

Error Codes

Code Name Description
E001 CONNECTION_FAILED Device connection failed
E002 DEVICE_NOT_FOUND Device not found
E003 PERMISSION_DENIED Access permission denied
E004 STREAM_ERROR Streaming error
E005 INVALID_CONFIG Invalid configuration
E006 ADAPTER_ERROR Adapter error
E007 TIMEOUT Operation timeout
E008 DISCONNECTED Connection lost

Error Handling Pattern

import { WiaBci, BciError } from 'wia-bci';

const bci = new WiaBci();

try {
  await bci.connect({ type: 'eeg_headset' });
} catch (error) {
  if (error instanceof BciError) {
    switch (error.code) {
      case 'E001':
        console.error('Connection failed:', error.message);
        break;
      case 'E002':
        console.error('Device not found');
        break;
      default:
        console.error('BCI Error:', error);
    }
  }
}

// Event-based error handling
bci.on('error', (error) => {
  console.error(`[${error.code}] ${error.message}`);

  if (!error.recoverable) {
    bci.disconnect();
  }
});

3.9 Best Practices

Resource Management

  • Always call dispose() when finished
  • Use try-finally blocks to ensure cleanup
  • Remove event listeners when no longer needed
  • Disconnect devices before page unload

Performance

  • Process signal data in chunks/epochs
  • Use Web Workers for heavy processing
  • Limit console logging in production
  • Cache feature extraction results

Reliability

  • Implement automatic reconnection
  • Validate signal quality continuously
  • Handle network interruptions gracefully
  • Save data periodically to prevent loss
