Chapter 2: Data Schema

EEG Data Formats, Channel Definitions, and Recording Metadata

WIA BCI Standard v1.0.0

2.1 Overview

The WIA BCI Data Format provides a standardized structure for storing, transmitting, and exchanging brain-computer interface data. This schema ensures interoperability across devices, platforms, and research institutions.

Key Design Goals

  • Self-Describing: Metadata embedded with signal data
  • Efficient: Binary formats for large datasets
  • Human-Readable: JSON for configuration and metadata
  • Compatible: Import/export from EDF+, BIDS, XDF

2.2 Directory Structure

A WIA BCI recording consists of multiple files organized in a directory:

wia-bci-recording/
├── recording.json          # Main recording metadata
├── device.json             # Device information
├── subject.json            # Subject/participant info (optional)
├── channels.json           # Channel definitions
├── events.json             # Event markers
├── data/
│   ├── eeg.bin             # Raw EEG data (binary)
│   ├── eeg.json            # EEG metadata
│   ├── emg.bin             # Raw EMG data (optional)
│   ├── emg.json            # EMG metadata (optional)
│   └── auxiliary.bin       # Auxiliary data (optional)
└── derivatives/            # Processed data (optional)
    ├── filtered/
    └── features/

File Naming Convention

{subject}_{session}_{task}_{modality}.{extension}

Examples:
- sub-001_ses-01_task-motor_eeg.bin
- sub-001_ses-01_task-motor_events.json
- sub-002_ses-03_task-p300_eeg.bin

2.7 Binary Data Format

EEG Data (eeg.bin)

Signal data stored as little-endian float32 with interleaved channels:

Layout:

[ch0_s0, ch1_s0, ..., chN_s0, ch0_s1, ch1_s1, ..., chN_s1, ...]

Data Type: float32 (IEEE 754)
Byte Order: Little-endian
Unit: microvolts (µV) after scaling
File Size = num_channels × num_samples × 4 bytes

2.9 Data Validation

JSON Schema Validation

All JSON files must validate against their respective schemas:

// Using a JSON schema validator
import { validate } from 'jsonschema';

const schema = await fetch('https://wia.live/schemas/bci/recording.schema.json');
const recording = JSON.parse(fs.readFileSync('recording.json'));

const result = validate(recording, schema);
if (!result.valid) {
  console.error('Validation errors:', result.errors);
}

Data Integrity Checks

  • File Size: Binary file size must match num_channels × num_samples × 4
  • Sampling Rate: Must match device specifications
  • Channel Count: Must match channels.json
  • Timestamps: Event timestamps must be within recording duration
  • Checksum: Binary data SHA-256 must match metadata

2.10 Compatibility & Conversion

Import from EDF+

import { convertEdfToWia } from 'wia-bci';

// Convert EDF+ file to WIA BCI format
await convertEdfToWia('input.edf', 'output-dir/');

Export to BIDS

import { convertWiaToBids } from 'wia-bci';

// Convert WIA BCI to BIDS structure
await convertWiaToBids('wia-recording/', 'bids-dataset/');

Stream to LSL

import { WiaBci, LslAdapter } from 'wia-bci';

const bci = new WiaBci();
await bci.connect({
  type: 'eeg_headset',
  connection: { protocol: 'lsl' }
});

2.11 Best Practices

File Organization

  • Use descriptive directory names with timestamps
  • Keep raw data separate from derivatives
  • Include README.txt with human-readable summary
  • Version control metadata files (Git-friendly JSON)

Metadata Completeness

  • Always include device information
  • Document all preprocessing steps
  • Record experimental paradigm details
  • Note any anomalies or artifacts

Data Quality

  • Check impedance before recording
  • Mark bad channels in metadata
  • Annotate artifact periods
  • Validate checksums after transfer
