BCI Fundamentals and Neural Data Standards

WIA BCI Standard v1.0.0

1.1 What is Brain-Computer Interface (BCI)?

A Brain-Computer Interface (BCI) is a direct communication pathway between the brain and an external device. BCIs enable individuals to control computers, prosthetics, wheelchairs, and other assistive technologies using only their brain signals, without requiring muscle movement.

Key Principle

BCIs translate neural activity into digital commands, empowering individuals with severe motor disabilities to communicate and interact with their environment.

Types of BCI Systems

Type Invasiveness Signal Quality Applications
Non-invasive EEG External sensors Moderate Consumer devices, research
Invasive Cortical Implanted electrodes High Medical, severe disabilities
Semi-invasive Subdural/epidural High Clinical research

1.2 Purpose of WIA BCI Standard

The WIA BCI Standard addresses critical challenges in brain-computer interface development:

Current Challenges

  • Fragmented Ecosystems - Each BCI manufacturer uses proprietary data formats
  • Limited Interoperability - Devices don't communicate with each other
  • Research Data Silos - Difficult to share and reproduce research
  • Accessibility Barriers - Complex integration with assistive technologies
  • Security Concerns - Lack of standardized privacy protections for neural data

WIA BCI Solutions

  • Unified Data Format - Single standard for all neural signal types
  • Device-Agnostic APIs - Connect any BCI device with the same code
  • Real-time Protocols - Low-latency streaming for responsive applications
  • Accessibility Integration - Native support for TTS, sign language, braille
  • Privacy by Design - Built-in encryption and consent management

1.3 Design Principles

1. Interoperability First

The WIA BCI standard is designed to work with existing standards and formats:

  • EDF+ (European Data Format) - Clinical EEG recordings
  • BIDS (Brain Imaging Data Structure) - Research data organization
  • XDF (Extensible Data Format) - Multi-stream synchronization
  • LSL (Lab Streaming Layer) - Real-time data streaming

2. Self-Describing Data

All WIA BCI data includes comprehensive metadata:

{
  "$schema": "https://wia.live/schemas/bci/recording.schema.json",
  "wia_version": "1.0.0",
  "device": {
    "manufacturer": "OpenBCI",
    "model": "Cyton",
    "channels": 8,
    "sampling_rate_hz": 250
  },
  "session": {
    "paradigm": "motor_imagery",
    "start_time": "2025-01-15T10:30:00.000Z"
  }
}

3. Human-Readable + Machine-Optimized

The standard uses JSON for metadata (human-readable) and binary formats for signal data (machine-optimized):

wia-bci-recording/
├── recording.json      # Human-readable metadata
├── device.json         # Device specifications
├── channels.json       # Channel definitions
├── events.json         # Event markers
└── data/
    ├── eeg.bin         # Binary signal data (efficient)
    └── eeg.json        # Signal metadata

4. Extensibility

New signal types, devices, and features can be added without breaking existing implementations.

5. Accessibility at Core

Native integration with WIA accessibility standards for inclusive applications.

1.4 Signal Types

Electroencephalography (EEG)

EEG (뇌전도) measures electrical activity of the brain through electrodes placed on the scalp. It's the most common non-invasive BCI signal.

  • Frequency Bands: Delta (0.5-4 Hz), Theta (4-8 Hz), Alpha (8-13 Hz), Beta (13-30 Hz), Gamma (30-100 Hz)
  • Applications: Motor imagery, P300 speller, SSVEP, cognitive state monitoring
  • Placement: 10-20 International System

Electromyography (EMG)

EMG (근전도) records electrical activity produced by skeletal muscles. Often used in hybrid BCIs.

  • Applications: Prosthetic control, gesture recognition, muscle fatigue monitoring
  • Placement: Over target muscle groups

Electrooculography (EOG)

EOG (안전도) measures eye movements and blinks through electrical potentials.

  • Applications: Gaze-based interfaces, artifact detection in EEG
  • Placement: Near eyes (horizontal and vertical channels)

1.5 BCI Paradigms

Motor Imagery (MI)

Motor Imagery (운동 상상) - Users imagine moving body parts (left hand, right hand, feet, tongue) without actual movement. Changes in sensorimotor rhythms are detected and classified.

// Event codes for motor imagery
{
  "left_hand": 10,
  "right_hand": 11,
  "feet": 12,
  "tongue": 13,
  "rest": 20
}

P300 Event-Related Potential

The P300 is a positive voltage spike occurring ~300ms after a rare or significant stimulus. Used in speller applications where users focus on target letters.

Steady-State Visually Evoked Potentials (SSVEP)

Visual stimuli flickering at specific frequencies evoke brain responses at the same frequency. Users look at different targets to select commands.

Passive Monitoring

Continuous monitoring of cognitive states (attention, workload, drowsiness) without requiring active control.

1.6 WIA BCI Architecture

Four-Phase Standard

Phase 1: Data Format

Standardized schemas for storing and exchanging neural data (EEG, EMG, EOG). Covers recording metadata, device information, channel definitions, and event markers.

Chapter Coverage: Chapter 2

Phase 2: API Interface

Programming interfaces for connecting to BCI devices, streaming data, and processing signals. Includes device adapters, event systems, and signal processing utilities.

Chapter Coverage: Chapter 3

Phase 3: Communication Protocol

Real-time protocols for low-latency data transmission. Supports WebSocket, TCP, and LSL transport layers with message formats and error recovery.

Phase 4: Ecosystem Integration

Integration with WIA output standards (TTS, sign language, braille, neurofeedback). Enables BCI signals to control diverse accessibility interfaces.

1.7 Use Cases

Medical & Clinical

  • Communication for ALS/Locked-in Patients - Text generation and speech output
  • Stroke Rehabilitation - Motor imagery training for neural plasticity
  • Epilepsy Monitoring - Seizure detection and prediction
  • Sleep Disorder Assessment - Automated sleep stage classification

Accessibility

  • Wheelchair Control - Navigate using motor imagery or P300 commands
  • Computer Access - Type, browse, and control applications
  • Environmental Control - Operate lights, doors, TV, appliances
  • Multimodal Output - Convert thoughts to speech, sign language, or braille

Research

  • Cognitive Neuroscience - Study attention, memory, decision-making
  • BCI Algorithm Development - Test new classification methods
  • Cross-Lab Collaboration - Share standardized datasets
  • Reproducibility - Validate published findings

Consumer & Gaming

  • Neurofeedback Training - Improve focus, relaxation, meditation
  • Gaming Interfaces - Mind-controlled games
  • Wellness Tracking - Monitor stress, sleep quality, cognitive load

1.8 Standards Compliance

Data Standards

  • EDF+ - Import/export European Data Format files
  • BIDS - Compatible with Brain Imaging Data Structure
  • DICOM - Medical imaging metadata alignment
  • HL7 FHIR - Healthcare interoperability (future)

Medical Device Standards

  • IEC 60601 - Medical electrical equipment safety
  • ISO 13485 - Medical device quality management
  • FDA 21 CFR Part 11 - Electronic records (US)
  • MDR/IVDR - Medical device regulation (EU)

Accessibility Standards

  • WCAG 2.1 - Web Content Accessibility Guidelines
  • Section 508 - US accessibility requirements
  • EN 301 549 - EU accessibility standard

1.9 Getting Started

For Developers

  1. Review the Data Schema (Chapter 2) to understand file formats
  2. Install the WIA BCI SDK (TypeScript or Python)
  3. Connect to a simulator or real device using the API (Chapter 3)
  4. Explore implementation examples (Chapter 7)

For Researchers

  1. Convert existing datasets to WIA BCI format
  2. Use standardized event codes for paradigms
  3. Share data with complete metadata
  4. Validate data quality and compliance (Chapter 8)

For Device Manufacturers

  1. Implement WIA BCI data export from your device
  2. Create a device adapter for the API layer
  3. Support real-time protocol (Chapter 4)
  4. Obtain WIA BCI certification (Chapter 8)

Medical Device Notice: If your application involves medical diagnosis or treatment, ensure compliance with applicable medical device regulations (FDA, MDR, etc.). The WIA BCI standard provides technical specifications but does not replace regulatory requirements.
