제1장: 뇌-컴퓨터 인터페이스 개요

뇌-컴퓨터 인터페이스(Brain-Computer Interface, BCI)는 인간의 뇌와 외부 장치 사이에 직접적인 통신 경로를 구축하는 혁신적인 기술입니다. BCI 시스템은 뇌의 전기적 활동을 감지하고 분석하여 사용자의 의도를 해석한 후, 이를 컴퓨터 명령으로 변환합니다. 이 기술은 척수 손상, 근위축성측삭경화증(ALS), 뇌졸중 등으로 인해 신체 기능을 상실한 환자들에게 의사소통과 환경 제어의 새로운 가능성을 제공합니다.

WIA BCI 표준은 다양한 BCI 장치와 플랫폼 간의 상호운용성을 보장하기 위해 데이터 형식, API 인터페이스, 통신 프로토콜, 시스템 통합 방법을 정의합니다. 이 표준은 EEG(뇌전도), EMG(근전도), EOG(안전도) 등 다양한 신호 유형을 지원하며, 기존 표준인 EDF+, BIDS, XDF와의 호환성을 유지합니다. 이를 통해 연구 데이터의 재현성을 높이고, 임상 적용을 가속화하며, 궁극적으로 더 많은 환자들이 BCI 기술의 혜택을 받을 수 있도록 합니다.

본 장에서는 BCI 기술의 기본 원리와 역사적 발전 과정을 살펴보고, WIA BCI 데이터 형식의 핵심 구조인 녹화 메타데이터, 장치 정보, 채널 정의, 이벤트 마커 등을 상세히 설명합니다. 이를 통해 BCI 시스템 개발자와 연구자들이 표준을 효과적으로 적용할 수 있는 기반을 마련합니다.

BCI 기술의 발전 역사

BCI 연구의 역사는 1920년대 독일의 정신과 의사 한스 베르거(Hans Berger)가 인간의 뇌전도(EEG)를 최초로 기록한 것에서 시작됩니다. 그는 두피에 전극을 부착하여 뇌의 전기적 활동을 측정하고, 이를 "알파파"와 "베타파"로 분류했습니다. 이 발견은 비침습적 방법으로 뇌 활동을 관찰할 수 있음을 입증했으며, 이후 모든 BCI 연구의 기초가 되었습니다.

1970년대에는 UCLA의 자크 비달(Jacques Vidal)이 "뇌-컴퓨터 인터페이스"라는 용어를 처음 사용했습니다. 그는 뇌 신호를 사용하여 컴퓨터 커서를 제어하는 가능성을 시연했으며, 이는 BCI 분야의 공식적인 시작점으로 간주됩니다. 이후 수십 년간 신호 처리 기술과 컴퓨팅 파워의 발전에 힘입어 BCI 기술은 꾸준히 진화해 왔습니다.

2000년대에 들어 BCI 기술은 실험실을 벗어나 실제 환자들에게 적용되기 시작했습니다. BrainGate 프로젝트는 피질에 이식된 전극 배열을 통해 사지마비 환자가 로봇 팔을 제어할 수 있음을 입증했습니다. 동시에 Emotiv, NeuroSky 등의 소비자용 EEG 헤드셋이 출시되어 BCI 기술의 대중화를 이끌었습니다. 현재는 Neuralink 등의 기업이 고밀도 침습적 BCI를 개발하고 있으며, 비침습적 기술도 정확도와 편의성 면에서 크게 발전하고 있습니다.

WIA BCI 데이터 형식 개요

WIA BCI 데이터 형식은 BCI 녹화 데이터의 저장, 전송, 교환을 위한 통합 표준입니다. 이 형식은 자기 기술적(self-describing) 특성을 가지며, JSON 기반의 메타데이터와 이진 형식의 신호 데이터를 결합합니다. 이를 통해 인간이 읽을 수 있는 문서화와 기계 처리에 효율적인 데이터 저장을 모두 달성합니다.

디렉토리 구조

WIA BCI 녹화는 표준화된 디렉토리 구조를 따릅니다. 최상위에는 녹화 메타데이터(recording.json), 장치 정보(device.json), 피험자 정보(subject.json), 채널 정의(channels.json), 이벤트 마커(events.json)가 위치합니다. 실제 신호 데이터는 data/ 하위 디렉토리에 저장되며, 전처리된 파생 데이터는 derivatives/ 디렉토리에 보관됩니다. 

wia-bci-recording/
├── recording.json          # 녹화 메타데이터
├── device.json             # 장치 정보
├── subject.json            # 피험자 정보 (선택)
├── channels.json           # 채널 정의
├── events.json             # 이벤트 마커
├── data/
│   ├── eeg.bin             # 원시 EEG 데이터 (이진)
│   ├── eeg.json            # EEG 메타데이터
│   └── auxiliary.bin       # 보조 데이터 (선택)
└── derivatives/            # 처리된 데이터 (선택)
    ├── filtered/
    └── features/

녹화 메타데이터 (recording.json)

recording.json 파일은 BCI 녹화 세션의 전반적인 정보를 담고 있습니다. WIA 버전, 녹화 ID, 시작/종료 시간, 세션 유형, 패러다임 정보 등이 포함됩니다. 또한 다른 메타데이터 파일과 데이터 파일에 대한 참조를 제공하여 전체 녹화 패키지의 진입점 역할을 합니다.

  • wia_version: WIA BCI 표준의 버전 번호입니다. 상호운용성 검사와 향후 마이그레이션에 사용됩니다.
  • recording_id: 고유한 녹화 식별자입니다. UUID 또는 사이트별 명명 규칙을 따릅니다.
  • session: 세션 유형(연구, 임상, 훈련)과 BCI 패러다임(운동 심상, P300, SSVEP 등)을 지정합니다.
  • data_files: 실제 데이터 파일의 경로와 형식 정보를 포함합니다.
호환성 참고: WIA BCI 형식은 EDF+ (European Data Format), BIDS (Brain Imaging Data Structure), XDF (Extensible Data Format)와 양방향 변환을 지원합니다. 기존 데이터를 WIA 형식으로 가져오거나, WIA 데이터를 다른 형식으로 내보낼 수 있는 변환 도구가 제공됩니다.

장치 정보 (device.json)

device.json 파일은 BCI 기기의 상세 사양을 기록합니다. 제조사, 모델, 펌웨어 버전과 함께 샘플링 속도, 해상도, 기준 전극 유형, 통신 프로토콜 등의 기술적 파라미터를 포함합니다. 이 정보는 데이터 해석과 재현성 확보에 필수적입니다.

표 1.2: BCI 장치 유형
유형 코드 설명 침습성
EEG 헤드셋 eeg 비침습적 EEG 녹화 비침습적
EEG 캡 eeg_cap 연구용 EEG 캡 비침습적
피질 임플란트 implant_cortical 피질 내 전극 배열 침습적

채널 정의 (channels.json)

channels.json 파일은 각 기록 채널의 상세 정보를 정의합니다. 채널 레이블, 유형(EEG, EMG, EOG 등), 단위, 전극 위치(10-20 시스템), 임피던스, 필터 설정 등이 포함됩니다. 특히 전극 위치는 3D 좌표로 표현되어 공간적 분석과 소스 로컬라이제이션에 활용됩니다. 

{
  "channel_count": 8,
  "sampling_rate_hz": 250,
  "channels": [
    {
      "index": 0,
      "label": "Fp1",
      "type": "eeg",
      "unit": "uV",
      "position": {
        "system": "10-20",
        "x": -0.95,
        "y": 0.31,
        "z": 0.0
      },
      "reference": "CAR",
      "impedance_kohm": 5.2,
      "status": "good"
    }
  ]
}

이벤트 마커 (events.json)

이벤트 마커는 BCI 실험에서 자극 제시, 사용자 반응, 시스템 이벤트 등을 기록합니다. 각 이벤트는 타임스탬프, 이벤트 코드, 유형, 지속 시간 등의 정보를 포함합니다. WIA 표준은 운동 심상, SSVEP, P300 등 주요 BCI 패러다임에 대한 표준 이벤트 코드를 정의하여 데이터셋 간의 일관성을 보장합니다.

표 1.3: 표준 이벤트 코드
범위 카테고리 설명
1-9 시스템 세션 시작/종료, 자극 온/오프
10-19 운동 심상 왼손, 오른손, 발, 혀
20-29 상태 휴식, 집중, 이완
30-39 SSVEP 주파수별 자극

이진 데이터 형식

실제 EEG 신호 데이터는 효율적인 저장과 빠른 접근을 위해 이진 형식으로 저장됩니다. WIA 표준은 32비트 부동소수점(float32), 리틀 엔디안, 채널 인터리브 방식을 사용합니다. 이 형식은 대부분의 프로그래밍 언어와 분석 도구에서 직접 읽을 수 있으며, 별도의 변환 없이 NumPy, MATLAB 등에서 바로 처리할 수 있습니다.

표준화의 이점

WIA BCI 표준을 채택함으로써 연구자와 개발자는 다양한 이점을 얻을 수 있습니다. 서로 다른 기기에서 수집된 데이터를 통합 분석할 수 있고, 연구 결과의 재현성이 향상됩니다. 또한 표준화된 도구와 라이브러리를 공유할 수 있어 개발 효율성이 높아집니다.

  • 다양한 BCI 기기와 플랫폼 간의 원활한 데이터 교환
  • 기존 표준(EDF+, BIDS, XDF)과의 호환성을 통한 데이터 마이그레이션 용이
  • 자기 기술적 형식으로 데이터 해석의 모호성 제거
  • 연구 데이터의 재현성과 공유 가능성 향상
  • 임상 적용을 위한 규제 승인 과정 간소화
  • 弘益人間 철학에 따른 기술의 보편적 접근성 확보
