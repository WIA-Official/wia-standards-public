Chapter 3: Signal Processing Algorithms

EEG, ECoG, and Neural Signal Processing Techniques

3.1 Signal Processing Pipeline Overview

Signal processing transforms raw neural signals into meaningful features that can be classified to generate BCI commands. The WIA-BCI standard defines a comprehensive signal processing pipeline that ensures consistent, reliable BCI performance across different implementations. Each stage of the pipeline has specific requirements and performance benchmarks that must be met for WIA-BCI certification.

The signal processing pipeline consists of five main stages: preprocessing, artifact removal, feature extraction, feature selection, and classification. Each stage builds upon the previous one, and errors or noise introduced at early stages propagate through the entire system, making quality control at each step critical for overall performance.

3.3 Artifact Removal Algorithms

Artifacts from eye movements, muscle activity, and other sources can overwhelm neural signals. The WIA-BCI standard requires robust artifact detection and removal capabilities to ensure reliable BCI operation in real-world conditions. 

// WIA-BCI Artifact Removal Implementation

class ArtifactRemoval {
    // Independent Component Analysis for artifact removal
    static ICA_ArtifactRemoval(data, numComponents) {
        // 1. Perform ICA decomposition
        ica = fastica(data, numComponents)
        components = ica.components
        mixingMatrix = ica.mixing

        // 2. Identify artifact components
        artifactIndices = []
        for (i = 0; i < numComponents; i++) {
            component = components[i]

            // EOG artifact detection
            if (hasEOGCharacteristics(component)) {
                artifactIndices.push(i)
                continue
            }

            // EMG artifact detection
            if (hasEMGCharacteristics(component)) {
                artifactIndices.push(i)
                continue
            }

            // ECG artifact detection
            if (hasECGCharacteristics(component)) {
                artifactIndices.push(i)
            }
        }

        // 3. Remove artifact components
        cleanComponents = components.copy()
        for (idx in artifactIndices) {
            cleanComponents[idx] = zeros(component.length)
        }

        // 4. Reconstruct clean signal
        cleanData = mixingMatrix.dot(cleanComponents)
        return cleanData
    }

    // Automatic EOG detection criteria
    static hasEOGCharacteristics(component) {
        // High frontal electrode loading
        frontalLoading = abs(component.spatialPattern[Fp1, Fp2]).mean()
        overallLoading = abs(component.spatialPattern).mean()

        // Low frequency dominance
        lowFreqPower = bandpower(component.timeSeries, 0.5, 4)
        totalPower = bandpower(component.timeSeries, 0.5, 40)

        return (frontalLoading / overallLoading > 2.0) &&
               (lowFreqPower / totalPower > 0.6)
    }

    // Automatic EMG detection criteria
    static hasEMGCharacteristics(component) {
        // High frequency content
        highFreqPower = bandpower(component.timeSeries, 20, 100)
        totalPower = bandpower(component.timeSeries, 0.5, 100)

        // Peripheral electrode loading
        peripheralLoading = abs(component.spatialPattern[T7, T8, F7, F8]).mean()

        return (highFreqPower / totalPower > 0.4) &&
               (peripheralLoading > threshold)
    }
}

Artifact Handling Strategies

  • Rejection: Discard contaminated segments entirely (simplest, loses data)
  • Correction: Remove artifact component while preserving neural signal
  • Regression: Subtract estimated artifact using reference channels
  • Adaptive filtering: Real-time artifact tracking and removal
  • Deep learning: Neural network-based artifact detection and removal

3.4 Feature Extraction Methods

Feature extraction transforms preprocessed signals into numerical features that capture discriminative information for classification. The WIA-BCI standard specifies recommended features for each BCI paradigm and defines minimum performance requirements.

3.4.1 Time Domain Features

Time Domain Features (WIA-BCI-F-001):

class TimeDomainFeatures {
    // Basic statistical features
    static extractBasicFeatures(signal) {
        return {
            mean: mean(signal),
            variance: variance(signal),
            standardDeviation: std(signal),
            skewness: skewness(signal),
            kurtosis: kurtosis(signal),
            rms: sqrt(mean(signal.^2)),
            zeroCrossings: countZeroCrossings(signal),
            peakToPeak: max(signal) - min(signal)
        }
    }

    // Hjorth parameters (activity, mobility, complexity)
    static hjorthParameters(signal) {
        activity = variance(signal)
        derivative1 = diff(signal)
        derivative2 = diff(derivative1)

        mobility = sqrt(variance(derivative1) / activity)
        complexity = sqrt(variance(derivative2) / variance(derivative1)) / mobility

        return {activity, mobility, complexity}
    }

    // Autoregressive model coefficients
    static arCoefficients(signal, order) {
        // Yule-Walker or Burg method
        coefficients = arburg(signal, order)
        return coefficients
    }

    // Sample entropy (complexity measure)
    static sampleEntropy(signal, m, r) {
        // m: embedding dimension (typically 2)
        // r: tolerance (typically 0.2 * std)
        N = signal.length
        templates_m = countTemplates(signal, m, r)
        templates_m1 = countTemplates(signal, m+1, r)
        return -log(templates_m1 / templates_m)
    }
}

Frequency domain features capture the oscillatory patterns that are fundamental to many BCI paradigms. The WIA-BCI standard defines standard frequency bands and feature extraction methods for spectral analysis.

class FrequencyDomainFeatures {
    // Power Spectral Density estimation
    static welchPSD(signal, windowSize, overlap, nfft) {
        // Welch method for robust PSD estimation
        windows = segmentWithOverlap(signal, windowSize, overlap)
        psds = []
        for (window in windows) {
            windowed = window * hamming(windowSize)
            fft_result = fft(windowed, nfft)
            psd = abs(fft_result).^2 / windowSize
            psds.push(psd)
        }
        return mean(psds, axis=0)
    }

    // Band power calculation
    static bandPower(psd, freqs, lowFreq, highFreq) {
        indices = where((freqs >= lowFreq) & (freqs <= highFreq))
        return sum(psd[indices]) * (freqs[1] - freqs[0])
    }

    // Standard band power features
    static extractBandPowers(signal, fs) {
        psd, freqs = welchPSD(signal, fs, fs/2, fs*2)
        return {
            delta: bandPower(psd, freqs, 0.5, 4),
            theta: bandPower(psd, freqs, 4, 8),
            alpha: bandPower(psd, freqs, 8, 13),
            beta: bandPower(psd, freqs, 13, 30),
            gamma: bandPower(psd, freqs, 30, 100),
            totalPower: bandPower(psd, freqs, 0.5, 100)
        }
    }

    // Relative band power (normalized)
    static relativeBandPower(bandPowers) {
        total = bandPowers.totalPower
        return {
            relDelta: bandPowers.delta / total,
            relTheta: bandPowers.theta / total,
            relAlpha: bandPowers.alpha / total,
            relBeta: bandPowers.beta / total,
            relGamma: bandPowers.gamma / total
        }
    }

    // Band power ratios (useful for cognitive state)
    static bandPowerRatios(bandPowers) {
        return {
            thetaAlpha: bandPowers.theta / bandPowers.alpha,
            alphaBeta: bandPowers.alpha / bandPowers.beta,
            thetaBeta: bandPowers.theta / bandPowers.beta
        }
    }
}

3.5 Common Spatial Patterns (CSP)

Common Spatial Patterns is a powerful feature extraction method specifically designed for motor imagery BCI. CSP finds spatial filters that maximize the variance difference between two classes, making it highly effective for distinguishing between different motor imagery tasks. 

// Common Spatial Patterns Implementation

class CSP {
    // Training phase: compute spatial filters
    static train(class1Data, class2Data, numFilters) {
        // Compute covariance matrices for each class
        cov1 = computeAverageCovariance(class1Data)
        cov2 = computeAverageCovariance(class2Data)

        // Composite covariance
        covComposite = cov1 + cov2

        // Whitening transformation
        eigenvalues, eigenvectors = eig(covComposite)
        P = eigenvectors * diag(1.0 / sqrt(eigenvalues))

        // Apply whitening to class 1 covariance
        S1 = P.T * cov1 * P

        // Eigendecomposition of whitened covariance
        eigenvalues1, B = eig(S1)

        // Sort by eigenvalue (variance ratio)
        sortedIndices = argsort(eigenvalues1)[::-1]

        // Select top and bottom filters
        topIndices = sortedIndices[:numFilters/2]
        bottomIndices = sortedIndices[-numFilters/2:]
        selectedIndices = concat(topIndices, bottomIndices)

        // Final spatial filter matrix
        W = P * B[:, selectedIndices]

        return W
    }

    // Apply CSP filters to extract features
    static extractFeatures(data, W) {
        // Project data through spatial filters
        projected = W.T * data

        // Compute log-variance features
        features = []
        for (i = 0; i < W.shape[1]; i++) {
            variance = var(projected[i, :])
            features.push(log(variance))
        }

        // Normalize features
        totalVariance = sum(exp(features))
        features = features - log(totalVariance)

        return features
    }

    // Regularized CSP for small sample sizes
    static trainRegularized(class1Data, class2Data, numFilters, shrinkage) {

3.6 Classification Algorithms

The final stage of the signal processing pipeline converts extracted features into BCI commands using machine learning classifiers. The WIA-BCI standard specifies requirements for classifier performance, real-time operation, and adaptive updating.

WIA-BCI Approved Classifiers

  • Linear Discriminant Analysis (LDA): Standard choice for simplicity and robustness
  • Support Vector Machine (SVM): Effective for high-dimensional feature spaces
  • Random Forest: Handles non-linear relationships, resistant to overfitting
  • Neural Networks: Deep learning for complex patterns (requires more data)
  • Riemannian Classifiers: Geometry-aware methods for covariance features
// WIA-BCI Classification Framework

class BCIClassifier {
    // Linear Discriminant Analysis
    static trainLDA(features, labels) {
        // Compute class means and covariance
        classes = unique(labels)
        means = {}
        for (c in classes) {
            means[c] = mean(features[labels == c], axis=0)
        }

        // Pooled within-class covariance
        Sw = zeros(features.shape[1], features.shape[1])
        for (c in classes) {
            classFeatures = features[labels == c]
            centered = classFeatures - means[c]
            Sw += centered.T * centered
        }
        Sw /= features.shape[0]

        // Regularization for numerical stability
        Sw += 0.01 * eye(Sw.shape[0])

        // Compute projection vector (2-class case)
        w = inv(Sw) * (means[0] - means[1])

        return {weights: w, means: means, threshold: 0}
    }

    // Real-time classification
    static classify(features, model) {
        projection = dot(features, model.weights)
        probability = sigmoid(projection - model.threshold)
        prediction = projection > model.threshold ? 1 : 0
        confidence = abs(probability - 0.5) * 2

        return {prediction, probability, confidence}
    }

    // Adaptive classifier updating
    static updateOnline(model, newFeature, trueLabel, learningRate) {
        // Compute prediction error
        prediction = classify(newFeature, model)
        error = trueLabel - prediction.probability

        // Update model parameters
        model.weights += learningRate * error * newFeature
        model.threshold -= learningRate * error

        return model
    }
}
