2.1 대칭 암호화란?

대칭 암호화는 암호화와 복호화에 동일한 키를 사용합니다. 빠르고 효율적이며, 대용량 데이터 암호화에 이상적입니다.

핵심 개념

암호화 키 = 복호화 키. 양쪽 모두 비밀 키를 알아야 합니다. 키 분배가 주요 과제입니다.

2.2 AES (Advanced Encryption Standard)

AES는 2001년 NIST가 선정한 표준 대칭 암호화 알고리즘입니다:

속성 값
블록 크기 128비트 (16바이트)
키 크기 128, 192, 256비트
유형 블록 암호
라운드 10 (128b), 12 (192b), 14 (256b)

2.2.1 WIA 권장 사항

  • AES-256: 모든 민감한 데이터에 사용
  • AES-128: 성능이 중요한 경우 허용
  • AES-192: 거의 사용되지 않음

2.3 운영 모드

모드 인증 추천
GCM ✅ 내장 ✅ 권장
CBC ❌ 별도 필요 ⚠️ 레거시
CTR ❌ 없음 ⚠️ 주의 필요
ECB ❌ 없음 ❌ 사용 금지

⚠️ ECB 모드 사용 금지

ECB (Electronic Codebook) 모드는 동일한 평문 블록을 동일한 암호문으로 암호화합니다. 이로 인해 패턴이 노출되어 치명적인 보안 취약점이 발생합니다.

2.4 AES-256-GCM 구현

// Node.js에서 AES-256-GCM 구현
const crypto = require('crypto');

function encrypt(plaintext, key) {
  // 12바이트 IV 생성 (GCM에 최적)
  const iv = crypto.randomBytes(12);

  // 암호화 객체 생성
  const cipher = crypto.createCipheriv('aes-256-gcm', key, iv);

  // 암호화 수행
  let ciphertext = cipher.update(plaintext, 'utf8', 'base64');
  ciphertext += cipher.final('base64');

  // 인증 태그 획득
  const authTag = cipher.getAuthTag();

  return { ciphertext, iv, authTag };
}

function decrypt(ciphertext, key, iv, authTag) {
  const decipher = crypto.createDecipheriv('aes-256-gcm', key, iv);
  decipher.setAuthTag(authTag);

  let plaintext = decipher.update(ciphertext, 'base64', 'utf8');
  plaintext += decipher.final('utf8');

  return plaintext;
}

2.5 IV/Nonce 요구사항

속성 요구사항 이유
길이 12바이트 (96비트) GCM에 최적화
고유성 절대 재사용 금지 재사용 시 완전 노출
생성 CSRNG 사용 예측 불가능해야 함

2.6 일반적인 실수

  • IV 재사용: GCM에서 치명적, 키 노출 가능
  • 인증 태그 무시: 데이터 변조 공격 허용
  • 약한 키: 암호학적으로 안전한 난수 사용 필수
  • ECB 모드: 패턴 노출로 보안 붕괴

2.7 장 요약

핵심 내용

  • 대칭 암호화는 동일한 키로 암호화/복호화
  • AES-256-GCM이 WIA 권장 표준
  • GCM은 암호화와 인증을 동시에 제공
  • IV는 반드시 고유해야 하며 재사용 금지
  • ECB 모드는 절대 사용하지 말 것
