제1장: 데이터 암호화 소개

弘益人間 (홍익인간)

"널리 인간을 이롭게 하라"

암호화는 인권을 보호하고, 표현의 자유를 가능하게 하며, 전 세계 개인들에게 힘을 부여합니다. 강력한 암호화는 모든 인류에게 이익이 됩니다.

학습 목표

  • WIA-SEC-011 데이터 암호화 표준의 목적 이해
  • 암호화의 기본 개념 학습
  • 암호 시스템의 세 가지 유형 파악
  • WIA 암호화 4단계 아키텍처 탐구
  • 인증 요구사항 검토

1.1 WIA-SEC-011이란?

WIA-SEC-011 데이터 암호화 표준은 이동 중 및 저장 중 데이터에 대한 포괄적인 보안을 제공합니다. 이 표준은 개인 메시지부터 기업 데이터베이스까지 모든 것을 보호합니다.

표준 개요

WIA-SEC-011은 현대 암호화 모범 사례를 위한 포괄적인 프레임워크를 제공합니다. 4단계 아키텍처와 인증 시스템을 통해 조직이 강력한 데이터 보호를 구현할 수 있도록 합니다.

1.2 왜 암호화가 필요한가?

위협 암호화 없이 암호화 적용
데이터 유출 모든 데이터 노출 데이터는 읽을 수 없음
중간자 공격 트래픽 가로채기 가능 암호화된 트래픽은 안전
내부자 위협 접근 = 노출 키가 필요
물리적 도난 디스크 데이터 접근 가능 데이터는 안전하게 유지

1.3 암호화 기본 개념

1.3.1 핵심 용어

  • 평문: 보호가 필요한 원본 데이터
  • 암호문: 암호화된 (스크램블된) 데이터
  • 키: 암호화/복호화에 사용되는 비밀 값
  • 알고리즘: 데이터를 변환하는 수학적 절차

1.3.2 암호화 과정

단계 입력 출력
암호화 평문 + 키 암호문
복호화 암호문 + 키 평문

1.4 암호 시스템의 유형

유형 키 속도 사용 사례
대칭 하나의 공유 키 빠름 대용량 데이터, 디스크 암호화
비대칭 공개/개인 키 쌍 느림 키 교환, 디지털 서명
하이브리드 둘 다 최적 TLS, 이메일 암호화

1.5 WIA 암호화 4단계 아키텍처

단계 계층 보호 대상
1단계 이동 중 데이터 TLS 1.3, 네트워크 트래픽
2단계 저장 중 데이터 AES-256 디스크/DB 암호화
3단계 키 관리 HSM, KMS, 키 순환
4단계 접근 제어 인증, 권한 부여

1.6 인증 등급

등급 요구사항
브론즈 기본 암호화, TLS 1.2+
실버 AES-256, 키 관리, 감사
골드 HSM, 자동화된 순환, 양자 대비

1.7 장 요약

핵심 내용

  • WIA-SEC-011은 데이터 암호화의 포괄적인 프레임워크 제공
  • 암호화는 유출, 가로채기, 도난으로부터 보호
  • 세 가지 유형: 대칭, 비대칭, 하이브리드
  • 4단계 아키텍처: 이동 중, 저장 중, 키, 접근
  • 弘益人間: 암호화는 모든 인류에게 이익
