3.1 비대칭 암호화란?

비대칭 암호화(공개키 암호화)는 수학적으로 연결된 키 쌍을 사용합니다:

키 보관 용도
공개키 누구나 보유 가능 암호화, 서명 검증
개인키 소유자만 비밀 보관 복호화, 서명 생성

핵심 원리

공개키로 암호화된 것은 개인키로만 복호화할 수 있습니다. 개인키로 서명된 것은 공개키로 검증할 수 있습니다.

3.2 RSA 알고리즘

RSA는 큰 소수의 인수분해 어려움에 기반한 암호화 알고리즘입니다:

3.2.1 RSA 키 크기

키 크기 보안 수준 WIA 권장
2048비트 112비트 최소 요구사항
3072비트 128비트 권장
4096비트 140비트+ 장기 보안용

3.2.2 RSA 구현

// Node.js에서 RSA 키 생성
const crypto = require('crypto');

// 4096비트 RSA 키 쌍 생성
const { publicKey, privateKey } = crypto.generateKeyPairSync('rsa', {
  modulusLength: 4096,
  publicKeyEncoding: { type: 'spki', format: 'pem' },
  privateKeyEncoding: { type: 'pkcs8', format: 'pem' }
});

// 공개키로 암호화
function encryptRSA(data, publicKey) {
  return crypto.publicEncrypt(
    { key: publicKey, padding: crypto.constants.RSA_PKCS1_OAEP_PADDING },
    Buffer.from(data)
  );
}

// 개인키로 복호화
function decryptRSA(encrypted, privateKey) {
  return crypto.privateDecrypt(
    { key: privateKey, padding: crypto.constants.RSA_PKCS1_OAEP_PADDING },
    encrypted
  ).toString();
}

3.3 타원 곡선 암호화 (ECC)

ECC는 타원 곡선 이산 로그 문제에 기반하여 더 작은 키로 동등한 보안을 제공합니다:

ECC 키 RSA 동등 보안 수준
256비트 3072비트 128비트
384비트 7680비트 192비트
521비트 15360비트 256비트

3.3.1 권장 곡선

  • P-256 (secp256r1): 일반 용도, 널리 지원
  • P-384 (secp384r1): 높은 보안, WIA 권장
  • P-521: 최고 보안, 성능 영향
  • Curve25519: 현대적, 빠름, 키 교환용

3.4 디지털 서명

디지털 서명은 메시지의 진본성과 무결성을 보장합니다:

// ECDSA 서명 및 검증
const crypto = require('crypto');

// 서명 생성
function signMessage(message, privateKey) {
  const sign = crypto.createSign('SHA384');
  sign.update(message);
  return sign.sign(privateKey, 'base64');
}

// 서명 검증
function verifySignature(message, signature, publicKey) {
  const verify = crypto.createVerify('SHA384');
  verify.update(message);
  return verify.verify(publicKey, signature, 'base64');
}

3.4.1 서명 알고리즘

알고리즘 기반 권장
RSA-PSS RSA ✅ RSA 사용 시
ECDSA ECC ✅ 일반 용도
EdDSA (Ed25519) Curve25519 ✅ 현대적 선택

3.5 키 교환

Diffie-Hellman 키 교환을 통해 안전하지 않은 채널에서도 공유 비밀을 설정할 수 있습니다:

  • ECDH (Elliptic Curve DH): ECC 기반, 효율적
  • X25519: Curve25519 기반, TLS 1.3에서 사용

3.6 RSA vs ECC

측면 RSA ECC
키 크기 큼 (3072+) 작음 (256-521)
성능 느림 빠름
호환성 보편적 현대 시스템
권장 레거시 지원 신규 시스템

3.7 장 요약

핵심 내용

  • 비대칭 암호화는 공개키/개인키 쌍 사용
  • RSA는 소인수분해 어려움에 기반 (3072비트+ 권장)
  • ECC는 더 작은 키로 동등한 보안 제공 (P-384 권장)
  • 디지털 서명으로 진본성과 무결성 보장
  • 신규 시스템에는 ECC 권장
