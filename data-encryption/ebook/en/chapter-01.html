1.1 What is Encryption?

Encryption is the process of converting readable data (plaintext) into an unreadable format (ciphertext) using a mathematical algorithm and a secret key. Only someone with the correct key can decrypt the ciphertext back to plaintext.

Plaintext:  "Hello, World!"
      ↓
   Encrypt (with key)
      ↓
Ciphertext: "8f7a2c9e1b4d..."
      ↓
   Decrypt (with key)
      ↓
Plaintext:  "Hello, World!"

1.2 Why Encryption Matters

Every day, billions of people rely on encryption without knowing it. When you:

  • Send a message to a friend
  • Make an online purchase
  • Access your bank account
  • Store files in the cloud
  • Use a password manager

Encryption is working silently in the background, protecting your privacy and security.

2024 Post-Quantum Quantum-resistant algorithms

1.4 Kerckhoffs's Principle

The Foundation of Modern Cryptography

"A cryptosystem should be secure even if everything about the system, except the key, is public knowledge."

This means:

  • The algorithm can be publicly known
  • Security depends only on the key
  • Open algorithms can be scrutinized by experts
  • "Security through obscurity" is not real security

1.5 Types of Cryptography

Type Key Usage Example
Symmetric Same key for encrypt/decrypt AES-256
Asymmetric Public/private key pair RSA, ECC
Hash Functions One-way, no key SHA-256

1.6 Chapter Summary

Key Takeaways

  • Encryption transforms readable data into protected ciphertext
  • Modern cryptography began with DES and RSA in 1977
  • Kerckhoffs's Principle: security depends on the key, not the algorithm
  • Three main types: symmetric, asymmetric, and hash functions
  • Encryption protects human rights and enables digital freedom
