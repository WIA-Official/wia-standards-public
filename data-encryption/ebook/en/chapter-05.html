5.1 The Weakest Link

Security Principle

The security of your encrypted data is only as strong as the security of your keys. Most encryption breaches result from key management failures, not algorithm weaknesses.

5.2 Key Lifecycle

Phase Activities Best Practices
Generation Create keys Use CSRNG, sufficient entropy
Distribution Share keys securely Asymmetric encryption, Diffie-Hellman
Storage Protect keys at rest HSM, KMS, encrypted key stores
Usage Use keys for crypto Access control, logging
Rotation Replace keys periodically 90 days symmetric, 365 days asymmetric
Destruction Securely delete keys Cryptographic erasure

5.3 Key Derivation Functions

KDFs derive encryption keys from passwords or other sources:

5.3.1 PBKDF2

function deriveKeyFromPassword(password, salt) {
  return crypto.pbkdf2Sync(
    password,
    salt,
    100000,  // iterations
    32,      // key length
    'sha256'
  );
}

5.3.2 Argon2 (Recommended)

Winner of the Password Hashing Competition, Argon2 is more secure:

  • Memory-hard: Resistant to GPU/ASIC attacks
  • Configurable: Time and memory costs adjustable
  • Three variants: Argon2d, Argon2i, Argon2id (recommended)

5.4 Key Storage Solutions

Solution Security Level Use Case
HSM (Hardware Security Module) Highest Financial, government
Cloud KMS High Cloud applications
Secrets Manager Medium-High Application secrets
Encrypted file Medium Development, testing

5.5 Envelope Encryption

Encrypt data with a Data Encryption Key (DEK), then encrypt the DEK with a Key Encryption Key (KEK):

// Envelope Encryption Pattern
{
  "encrypted_data": "AES-encrypted content...",
  "encrypted_dek": "RSA-encrypted DEK...",
  "kek_id": "master-key-v2",
  "algorithm": "AES-256-GCM"
}

Benefits:

  • Enables key rotation without re-encrypting data
  • KEK stored in HSM or KMS
  • DEK stored with encrypted data
  • Used by AWS, Azure, Google Cloud

5.6 Chapter Summary

Key Takeaways

  • Key management is the weakest link in most systems
  • Keys have a lifecycle: generation, distribution, storage, usage, rotation, destruction
  • Use Argon2id for password-based key derivation
  • Store keys in HSM or cloud KMS for production
  • Envelope encryption enables efficient key rotation
