2.1 AES: The Gold Standard

The Advanced Encryption Standard (AES) is the most widely used symmetric encryption algorithm. Selected by NIST in 2001 after a rigorous competition, AES has withstood decades of cryptanalysis.

Specification Value
Block Size 128 bits (16 bytes)
Key Sizes 128, 192, or 256 bits
Rounds (AES-256) 14 rounds
Structure Substitution-Permutation Network

2.2 Modes of Operation

Block ciphers like AES encrypt fixed-size blocks. To encrypt larger data, we use modes of operation:

2.2.1 GCM (Galois/Counter Mode) - Recommended

Why GCM?

  • Provides encryption AND authentication in one operation
  • Parallelizable (fast with hardware acceleration)
  • Prevents tampering with authentication tag
  • Supports Additional Authenticated Data (AAD)

2.2.2 Mode Comparison

Mode Authentication Recommendation
GCM Built-in AEAD ✅ Recommended
CBC Requires separate MAC ⚠️ Use with caution
CTR Requires separate MAC ⚠️ Use with caution
ECB None ❌ Never use

2.3 Implementation Example

// Node.js AES-256-GCM Example
const crypto = require('crypto');

function encryptAES(plaintext, key) {
  // Generate random IV (NEVER reuse!)
  const iv = crypto.randomBytes(16);

  // Create cipher
  const cipher = crypto.createCipheriv('aes-256-gcm', key, iv);

  // Encrypt
  let ciphertext = cipher.update(plaintext, 'utf8', 'hex');
  ciphertext += cipher.final('hex');

  // Get authentication tag
  const authTag = cipher.getAuthTag();

  return { ciphertext, iv, authTag };
}

2.4 ChaCha20-Poly1305

ChaCha20-Poly1305 is a modern alternative to AES-GCM:

  • Fast in software: Doesn't require special hardware
  • Resistant to timing attacks: Constant-time operations
  • Widely adopted: TLS 1.3, WireGuard VPN, Signal Protocol
  • Recommended: When AES hardware acceleration is unavailable

Rule of Thumb

Use AES-256-GCM on systems with AES-NI support (modern Intel/AMD CPUs). Use ChaCha20-Poly1305 on mobile devices and embedded systems.

2.5 Chapter Summary

Key Takeaways

  • AES-256 is the symmetric encryption standard
  • GCM mode provides both encryption and authentication
  • Never reuse IVs with the same key
  • ECB mode is insecure - never use it
  • ChaCha20-Poly1305 is a great alternative on mobile
