4.1 Best of Both Worlds

Hybrid encryption combines the efficiency of symmetric encryption with the convenience of asymmetric encryption:

  1. Generate random symmetric key (AES-256)
  2. Encrypt data with symmetric key (fast)
  3. Encrypt symmetric key with recipient's public key
  4. Send encrypted data + encrypted key

Real-World Example: HTTPS

When you visit an HTTPS website, your browser uses hybrid encryption:

  • RSA/ECC for initial key exchange
  • AES for encrypting actual web traffic
  • This is how TLS (Transport Layer Security) works

4.2 Implementation

function hybridEncrypt(data, recipientPublicKey) {
  // 1. Generate random AES key
  const aesKey = crypto.randomBytes(32);

  // 2. Encrypt data with AES-GCM
  const { ciphertext, iv, authTag } = encryptAES(data, aesKey);

  // 3. Encrypt AES key with RSA-OAEP
  const encryptedKey = crypto.publicEncrypt(
    {
      key: recipientPublicKey,
      padding: crypto.constants.RSA_PKCS1_OAEP_PADDING
    },
    aesKey
  );

  return { ciphertext, encryptedKey, iv, authTag };
}

function hybridDecrypt(encrypted, privateKey) {
  // 1. Decrypt AES key with private key
  const aesKey = crypto.privateDecrypt(
    {
      key: privateKey,
      padding: crypto.constants.RSA_PKCS1_OAEP_PADDING
    },
    encrypted.encryptedKey
  );

  // 2. Decrypt data with AES key
  return decryptAES(encrypted.ciphertext, aesKey,
                    encrypted.iv, encrypted.authTag);
}

4.3 Performance Benefits

Consider encrypting 1 GB of data:

Method Time Practicality
RSA-4096 only ~92 hours ❌ Impractical
AES-256 only 0.3 seconds ⚠️ Key distribution?
Hybrid ~0.3 seconds ✅ Best approach

4.4 TLS Handshake

The TLS handshake is a perfect example of hybrid encryption in action:

// TLS 1.3 Simplified Handshake
1. Client Hello
   - Supported cipher suites
   - Client random
   - Key share (ECDHE)

2. Server Hello
   - Selected cipher suite
   - Server random
   - Key share (ECDHE)
   - Certificate

3. Key Derivation
   - ECDHE shared secret
   - Derive symmetric keys

4. Encrypted Communication
   - AES-256-GCM for all traffic

4.5 Perfect Forward Secrecy

Modern hybrid systems use ephemeral keys to achieve Perfect Forward Secrecy (PFS):

  • Each session uses a new key pair (ECDHE)
  • Even if long-term keys are compromised, past sessions remain secure
  • TLS 1.3 mandates PFS for all connections

4.6 Chapter Summary

Key Takeaways

  • Hybrid encryption combines symmetric speed with asymmetric convenience
  • TLS uses hybrid encryption for secure web traffic
  • Encrypt data with AES, encrypt AES key with RSA/ECC
  • Performance is essentially the same as pure symmetric
  • Perfect Forward Secrecy protects past communications
