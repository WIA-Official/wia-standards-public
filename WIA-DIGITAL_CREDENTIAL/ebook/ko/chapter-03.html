1. WIA-DIGITAL_CREDENTIAL 표준 개요

1.1 표준의 구조

WIA-DIGITAL_CREDENTIAL은 W3C Verifiable Credentials를 기반으로 하며, 한국의 교육 및 자격 시스템에 최적화된 확장을 포함합니다. 이 표준은 교육부, 한국산업인력공단, 주요 대학이 공동으로 검증했습니다.

계층 표준 역할 관리 기관
국제 표준 W3C VC Data Model 기본 데이터 구조 W3C
산업 표준 IMS Global OpenBadges 교육 배지 IMS Global
지역 표준 WIA-DIGITAL_CREDENTIAL 한국형 확장 WIA
국가 표준 KS X ISO/IEC 18013 모바일 ID 한국표준협회

1.2 표준 버전 관리

WIA-DIGITAL_CREDENTIAL은 시맨틱 버전 관리(Semantic Versioning)를 따릅니다. 2025년 현재 v1.2.0이 최신 버전입니다.

  • v1.0.0 (2023.03): 초기 릴리스, 학위 증명서 지원
  • v1.1.0 (2024.01): 국가기술자격 추가, DID 통합
  • v1.2.0 (2024.09): 온라인 교육 수료증, OpenBadges v3 호환
  • v2.0.0 (2025 Q3 예정): 영지식 증명(ZKP) 지원

2. W3C Verifiable Credentials

2.1 데이터 모델

W3C Verifiable Credentials는 검증 가능한 자격증명을 표현하는 국제 표준입니다. JSON-LD 형식을 사용하며, 발급자(Issuer), 보유자(Holder), 검증자(Verifier) 모델을 따릅니다.

{
  "@context": [
    "https://www.w3.org/2018/credentials/v1",
    "https://wia.org/credentials/v1",
    "https://purl.imsglobal.org/spec/ob/v3p0/context.json"
  ],
  "id": "https://snu.ac.kr/credentials/2024/123456",
  "type": ["VerifiableCredential", "UniversityDegreeCredential"],
  "issuer": {
    "id": "did:wia:snu",
    "name": "서울대학교",
    "url": "https://www.snu.ac.kr",
    "image": "https://www.snu.ac.kr/logo.png"
  },
  "issuanceDate": "2024-02-20T00:00:00Z",
  "expirationDate": "2034-02-20T00:00:00Z",
  "credentialSubject": {
    "id": "did:wia:student:987654",
    "name": "홍길동",
    "studentId": "2020-12345",
    "degree": {
      "type": "BachelorDegree",
      "name": "공학사",
      "field": "컴퓨터공학",
      "graduationDate": "2024-02-20",
      "gpa": 3.85,
      "honors": "Cum Laude"
    }
  },
  "proof": {
    "type": "Ed25519Signature2020",
    "created": "2024-02-20T09:30:00Z",
    "verificationMethod": "did:wia:snu#keys-1",
    "proofPurpose": "assertionMethod",
    "proofValue": "z58DAdF...W3UxQ"
  }
}

2.2 JSON-LD 컨텍스트

JSON-LD(Linked Data)는 JSON에 의미론적 정보를 추가합니다. @context는 각 필드의 의미를 정의합니다.

왜 JSON-LD인가?

  • 상호운용성: 서로 다른 시스템이 같은 의미로 해석
  • 확장성: 새로운 필드를 추가해도 하위 호환성 유지
  • 국제 표준: W3C 권고 표준
  • 링크드 데이터: 웹 전체를 데이터베이스처럼 연결

2.3 디지털 서명 알고리즘

WIA-DIGITAL_CREDENTIAL은 다양한 서명 알고리즘을 지원합니다:

알고리즘 키 길이 속도 사용처
Ed25519 256-bit 매우 빠름 교육부 권장
ECDSA (secp256k1) 256-bit 빠름 이더리움 호환
RSA-2048 2048-bit 느림 레거시 시스템
BLS12-381 381-bit 중간 집계 서명

3. OpenBadges 표준

3.1 OpenBadges v3.0

IMS Global의 OpenBadges는 온라인 교육 수료증 표준입니다. Coursera Korea, Udemy Korea, K-MOOC이 이 표준을 따릅니다.

{
  "@context": "https://purl.imsglobal.org/spec/ob/v3p0/context.json",
  "id": "https://coursera.org/badges/deep-learning-specialist-2024-12345",
  "type": ["OpenBadgeCredential"],
  "issuer": {
    "id": "https://coursera.org",
    "type": "Profile",
    "name": "Coursera",
    "url": "https://www.coursera.org"
  },
  "issuanceDate": "2024-05-15T00:00:00Z",
  "name": "Deep Learning Specialist",
  "credentialSubject": {
    "id": "did:wia:learner:abc123",
    "type": "AchievementSubject",
    "achievement": {
      "id": "https://coursera.org/achievements/deep-learning",
      "type": "Achievement",
      "name": "딥러닝 전문가 과정",
      "description": "신경망, CNN, RNN 등 딥러닝 핵심 기술 습득",
      "criteria": {
        "narrative": "5개 과정 완료, 프로젝트 5개 제출, 최종 시험 80점 이상"
      }
    }
  }
}

3.2 WIA-DIGITAL_CREDENTIAL과의 통합

한국 시스템은 OpenBadges를 WIA 표준에 통합하여, 국내외 모두에서 인정받는 수료증을 발급합니다.

K-MOOC + WIA 통합 사례

K-MOOC은 OpenBadges v3.0과 WIA-DIGITAL_CREDENTIAL을 동시에 준수하는 수료증을 발급합니다. 이는 국내에서는 교육부 시스템과 연동되고, 해외에서는 LinkedIn에 자동 등록됩니다.

  • 국내 인정: 교육부 블록체인에 등록
  • 해외 인정: LinkedIn, Mozilla Backpack 연동
  • 블록체인: 클레이튼 + IPFS

4. 한국 정부 표준

4.1 교육부 표준

교육부는 2023년 "대학 학위 디지털 증명서 표준"을 제정했습니다. 이는 WIA-DIGITAL_CREDENTIAL을 기반으로 합니다.

구분 내용 비고
표준명 교육부 고시 제2023-45호 2023.06 시행
적용 대상 전국 4년제 대학 의무 적용
데이터 형식 JSON-LD (WIA v1.1) W3C VC 호환
블록체인 Hyperledger Fabric 교육부 운영
서명 알고리즘 Ed25519 필수

4.2 한국산업인력공단 표준

한국산업인력공단은 국가기술자격증에 대한 자체 표준을 운영하며, WIA-DIGITAL_CREDENTIAL v1.2와 호환됩니다.

  • 표준명: 국가기술자격 디지털 증명서 규격 (HRD-DC-2024)
  • 발급 형식: ERC-721 NFT + VC
  • 메타데이터: 자격 종목, 등급, 합격일, 유효기간
  • 검증 방법: API, QR 코드, 블록체인 직접 조회

4.3 과학기술정보통신부 DID 표준

과학기술정보통신부는 2024년 "분산신원 기술 표준"을 제정했습니다. 이는 WIA-DIGITAL_CREDENTIAL의 DID 구현 기준이 됩니다.

// 한국 표준 DID 문서 (KS X ISO/IEC 19086 준수)
{
  "@context": [
    "https://www.w3.org/ns/did/v1",
    "https://msit.go.kr/did/v1"
  ],
  "id": "did:wia:korea:1234567890abcdef",
  "controller": "did:wia:korea:1234567890abcdef",
  "verificationMethod": [{
    "id": "did:wia:korea:1234567890abcdef#keys-1",
    "type": "Ed25519VerificationKey2020",
    "controller": "did:wia:korea:1234567890abcdef",
    "publicKeyMultibase": "zH3C2AVvLMv6gmMNam3uVAjZpfkcJCwDwnZn6z3wXmqPV"
  }],
  "authentication": ["did:wia:korea:1234567890abcdef#keys-1"],
  "service": [{
    "id": "did:wia:korea:1234567890abcdef#credential-hub",
    "type": "CredentialHub",
    "serviceEndpoint": "https://hub.wia.or.kr/did/1234567890abcdef"
  }]
}

5. 국제 표준 준수

5.1 ISO/IEC 표준

WIA-DIGITAL_CREDENTIAL은 다음 ISO 표준을 준수합니다:

  • ISO/IEC 18013-5: 모바일 운전면허증 (mDL) - DID 기반
  • ISO/IEC 29115: 신원 보증 레벨 - LoA 3-4 지원
  • ISO/IEC 27001: 정보보안 관리 - 인증 획득
  • ISO/IEC 23220: 교육 배지 메타데이터

5.2 ETSI 표준 (유럽)

유럽 표준 기구 ETSI의 eIDAS 2.0과 호환됩니다. 한국 학위를 유럽에서 인정받을 수 있습니다.

유럽 상호인정 협정

2024년 한국 교육부와 EU 집행위원회는 디지털 자격증명 상호인정 협정을 체결했습니다. WIA-DIGITAL_CREDENTIAL은 ETSI TS 119 612 표준과 매핑되어, 한국 대학 학위가 EU 27개국에서 자동 인정됩니다.

6. API 표준

6.1 RESTful API 스펙

WIA-DIGITAL_CREDENTIAL은 표준 REST API를 정의합니다:

엔드포인트 메서드 설명 인증
/api/v1/credentials POST 자격증명 발급 OAuth 2.0
/api/v1/credentials/{id} GET 자격증명 조회 Bearer Token
/api/v1/verify POST 자격증명 검증 API Key
/api/v1/revoke/{id} DELETE 자격증명 취소 OAuth 2.0

6.2 GraphQL 지원

교육부 시스템은 GraphQL도 지원하여, 클라이언트가 필요한 필드만 쿼리할 수 있습니다:

query GetDegree($studentId: ID!) {
  credential(studentId: $studentId) {
    id
    type
    issuer {
      name
      did
    }
    credentialSubject {
      name
      degree {
        type
        field
        graduationDate
      }
    }
    proof {
      type
      proofValue
    }
  }
}

7. 상호운용성 테스트

7.1 국제 테스트베드

WIA는 2024년 6월 "Global Credential Interoperability Test" 에서 12개국 시스템과 상호운용성을 검증받았습니다.

국가 시스템 테스트 항목 결과
미국 MIT Digital Credentials 학위 교차 검증 통과
독일 eduGAIN 신원 연동 통과
일본 MEXT Digital ID API 호환성 통과
싱가포르 SkillsFuture SG 자격증 매핑 통과

7.2 호환성 인증

WIA는 "WIA Certified" 프로그램을 통해 시스템의 표준 준수를 인증합니다. 2025년 기준 한국 내 인증 시스템:

  • 교육부 대학 학위 블록체인 - Level 3 인증
  • 한국산업인력공단 자격증 시스템 - Level 3 인증
  • 카카오 인증서 - Level 2 인증
  • 네이버 자격증 지갑 - Level 2 인증
  • Coursera Korea - Level 2 인증

8. 보안 및 프라이버시 표준

8.1 암호화 표준

모든 자격증명은 전송 중(TLS 1.3) 및 저장 시(AES-256) 암호화됩니다.

보안 요구사항

  • 전송 암호화: TLS 1.3 필수, 1.2 이하 차단
  • 저장 암호화: AES-256-GCM 또는 ChaCha20-Poly1305
  • 키 관리: HSM 또는 KMS 사용 권장
  • 접근 제어: RBAC + ABAC 조합

8.2 개인정보보호법 준수

한국의 개인정보보호법, EU의 GDPR, 미국의 FERPA를 모두 준수하도록 설계되었습니다.

8.3 영지식 증명 (ZKP)

WIA v2.0 (2025 예정)은 영지식 증명을 지원합니다. 예를 들어, "나는 KAIST 졸업생이다"를 증명하면서도 정확한 졸업 연도, 학과, 성적 등은 공개하지 않을 수 있습니다.

9. 확장 및 커스터마이징

9.1 커스텀 스키마

기관은 WIA 표준을 확장하여 자체 필드를 추가할 수 있습니다:

{
  "@context": [
    "https://www.w3.org/2018/credentials/v1",
    "https://wia.org/credentials/v1",
    // 커스텀 컨텍스트 추가
    "https://kaist.ac.kr/credentials/v1"
  ],
  "type": ["VerifiableCredential", "KAISTDegree"],
  "credentialSubject": {
    "degree": {
      "type": "MasterDegree",
      "field": "전기전자공학",
      // KAIST 고유 필드
      "researchLab": "AI & Robotics Lab",
      "thesisTitle": "딥러닝 기반 자율주행 알고리즘",
      "advisor": "홍길동 교수"
    }
  }
}

9.2 플러그인 아키텍처

WIA SDK는 플러그인 시스템을 제공하여, 기관이 자체 검증 로직을 추가할 수 있습니다.

핵심 요약

  • WIA-DIGITAL_CREDENTIAL은 W3C VC를 기반으로 한국 시스템에 최적화
  • JSON-LD 형식으로 상호운용성과 확장성 보장
  • OpenBadges v3.0과 통합하여 온라인 교육 수료증 지원
  • 교육부, 한국산업인력공단 표준을 국가 차원에서 제정
  • ISO, ETSI 등 국제 표준 준수로 글로벌 인정
  • REST/GraphQL API로 쉬운 통합
  • 12개국 상호운용성 테스트 통과
  • 弘益人間 정신으로 글로벌 표준화 주도
