제4장: 1단계 - 데이터 형식 및 센서 통합

WIA 경비 로봇 표준 - ROB-012

4.1 데이터 형식의 중요성

1단계는 WIA ROB-012 표준의 기반을 형성하며 모든 후속 단계가 구축되는 데이터 표현의 공통 언어를 설정합니다. 표준화된 데이터 형식이 없으면 경비 로봇은 서로 이해하지 못하는 다른 언어를 사용하는 시스템으로 남아 있어 통합이 거의 불가능하고 데이터 교환이 비효율적입니다.

데이터 형식 표준화는 로봇 상태, 센서 판독값, 경고, 위치 정보 및 운영 이벤트를 표현하는 방법에 대한 정확한 사양을 제공합니다. 이러한 일관성을 통해 다른 제조업체의 시스템이 데이터를 사용자 정의 변환 계층 없이 직접 소비하고 해석할 수 있습니다.

JSON을 기본 형식으로 선택

ROB-012는 모든 구조화된 데이터 교환의 기본 형식으로 JSON(JavaScript Object Notation)을 채택합니다. 이 결정은 몇 가지 실용적인 고려 사항을 기반으로 합니다:

  • 보편적 지원: JSON은 모든 주요 프로그래밍 언어에서 기본 지원을 받으며 파서와 생성기가 즉시 사용 가능합니다
  • 사람이 읽을 수 있음: JSON의 텍스트 기반 특성은 디버깅, 로깅 및 수동 검사를 단순화합니다
  • 스키마 검증: JSON 스키마 표준은 형식적인 검증 및 문서화를 제공합니다
  • 효율적인 크기: 압축 시 JSON은 바이너리 형식과 경쟁력 있는 크기를 달성합니다
  • 확장성: JSON의 유연한 구조는 이전 버전과의 호환성을 깨지 않고 선택적 필드를 허용합니다

표준은 압축(gzip, brotli)을 의무화하여 네트워크 전송에서 JSON의 잠재적인 버보시티를 완화합니다. 실제 테스트에 따르면 압축된 JSON은 종종 동등한 바이너리 형식에 접근하면서 디버깅 가능성 이점을 유지합니다.

4.2 핵심 데이터 구조

ROB-012는 모든 경비 로봇이 구현해야 하는 포괄적인 JSON 스키마 세트를 정의합니다. 이러한 스키마는 6가지 주요 범주로 구성됩니다:

4.2.1 로봇 상태 데이터

로봇 상태 스키마는 경비 로봇의 현재 조건 및 운영 모드를 캡처합니다:

    "estimated_runtime_minutes": 245,

      "coordinate_system": "WGS84",

      "heading": 142.5,
      "accuracy_meters": 0.8

      "linear_mps": 1.2,

4.3 타임스탬프 및 시간 동기화

정확하고 일관된 타임스탬프는 보안 시스템에 중요합니다. 이벤트는 정확한 순서로 재구성되어야 하며, 다른 소스의 데이터는 시간적으로 상관되어야 하며, 감사 추적은 법적 검토를 견딜 수 있어야 합니다.

ISO 8601 형식 의무화

ROB-012는 모든 타임스탬프에 대해 ISO 8601 형식을 의무화합니다:

// 기본 형식 (마이크로초 정밀도):
"2025-12-27T14:30:45.123456Z"

// 구성 요소:
YYYY-MM-DD  : 날짜 (연-월-일)
T           : 날짜/시간 구분 기호
HH:MM:SS    : 시간 (24시간 형식)
.microseconds : 선택적 마이크로초 정밀도
Z           : UTC 시간대 지정자

// 잘못된 형식 (ROB-012에서 허용되지 않음):
"12/27/2025 2:30 PM"          // 모호한 미국 형식
"2025-12-27 14:30:45"         // T 구분 기호 없음
"2025-12-27T14:30:45+09:00"   // 로컬 시간대 (UTC만 허용)

NTP 시간 동기화 요구 사항

정확한 타임스탬프를 보장하기 위해 모든 ROB-012 호환 로봇은 NTP(네트워크 시간 프로토콜) 시간 동기화를 구현해야 합니다:

  • 로봇은 부팅 시 및 적어도 매시간 NTP 서버와 동기화해야 합니다
  • 시간 오프셋이 500ms를 초과하면 경고를 트리거해야 합니다
  • 연결 끊김 기간 동안 로봇은 실시간 클럭(RTC)을 사용하여 정확도를 유지해야 합니다
  • 모든 타임스탬프에는 신뢰 수준을 나타내는 정확도 추정치가 함께 제공되어야 합니다

4.4 위치 및 좌표 시스템

경비 로봇은 정확한 자체 위치 파악에 의존하여 순찰 경로를 탐색하고, 이벤트 위치를 기록하고, 여러 로봇 간의 적용 범위를 조정합니다. ROB-012는 실외 및 실내 내비게이션 모두에 대한 표준화된 좌표 시스템을 정의합니다.

WGS84 실외 좌표

실외 작업의 경우 표준은 GPS, GLONASS, Galileo 및 BeiDou를 포함한 모든 주요 GNSS 시스템에서 사용하는 World Geodetic System 1984(WGS84)를 의무화합니다:

{
  "location": {
    "coordinate_system": "WGS84",
    "latitude": 37.566535,
    "longitude": 126.977969,
    "altitude": 45.2,
    "heading": 142.5,
    "accuracy_horizontal_m": 0.8,
    "accuracy_vertical_m": 1.2,
    "altitude_reference": "ellipsoid"
  }
}

로컬 좌표 프레임 (실내)

실내 환경에서 로봇은 종종 건물 평면도에 고정된 로컬 좌표 시스템을 사용합니다:

{
  "location": {
    "coordinate_system": "local_cartesian",
    "frame_id": "building_a_floor_3",
    "x_meters": 12.45,
    "y_meters": 8.32,
    "z_meters": 0.15,
    "heading_degrees": 90.0,
    "origin": {
      "wgs84_latitude": 37.566535,
      "wgs84_longitude": 126.977969,
      "description": "Building A southwest corner"
    }
  }
}
좌표 시스템 사용 사례 정확도 목표 업데이트 속도
WGS84 GPS 실외 순찰, 넓은 지역 1-3 미터 1 Hz
WGS84 RTK-GPS 고정밀 실외 0.02-0.1 미터 5-10 Hz
로컬 데카르트 (LIDAR SLAM) 실내 내비게이션 0.05-0.2 미터 10-20 Hz
로컬 데카르트 (UWB) 고정밀 실내 0.1-0.3 미터 10 Hz
혼합 (GPS + SLAM) 실내-실외 전환 0.5-2 미터 10 Hz

핵심 요점

  • 표준화된 데이터 형식은 상호 운용성의 기반이며 ROB-012 표준 아키텍처의 1단계를 형성합니다
  • JSON은 보편적 지원, 사람이 읽을 수 있음, 스키마 검증 기능 및 압축 시 경쟁력 있는 크기로 인해 기본 형식으로 선택되었습니다
  • 포괄적인 JSON 스키마는 로봇 상태, 다양한 센서 유형, 경고, 위치 및 운영 이벤트를 다룹니다
  • ISO 8601 타임스탬프 형식(UTC)과 NTP 시간 동기화는 이벤트의 정확하고 일관된 시간 순서를 보장합니다
  • WGS84(실외)와 로컬 데카르트 프레임(실내)을 모두 지원하는 표준화된 좌표 시스템은 정확한 위치 추적 및 내비게이션을 가능하게 합니다
  • 각 센서 유형에는 메타데이터, 보정 상태, 품질 지표를 포함한 전용 데이터 스키마가 있어 센서 융합 및 분석을 촉진합니다

복습 문제

  1. 표준 데이터 형식으로 JSON이 Protocol Buffers 또는 MessagePack과 같은 바이너리 형식보다 선호되는 이유를 설명하십시오. 절충안은 무엇입니까?
  2. 로봇 상태 스키마의 핵심 구성 요소는 무엇이며, 각 필드가 운영 모니터링 및 관리에 어떻게 기여합니까?
  3. 표준이 다양한 센서 유형(카메라, LIDAR, 환경)에 대해 전용 스키마를 정의하는 이유와 각 센서 스키마의 고유한 요구 사항을 분석하십시오.
  4. ISO 8601 타임스탬프 형식의 중요성과 UTC 시간대가 의무화되는 이유를 논의하십시오. 로컬 시간대를 허용하지 않는 이유는 무엇입니까?
  5. NTP 시간 동기화 요구 사항(500ms 경고 임계값, 시간당 동기화)이 보안 시스템 운영 및 감사 추적 무결성에 어떻게 기여하는지 설명하십시오.
  6. WGS84 및 로컬 데카르트 좌표 시스템을 비교하십시오. 각 시스템은 언제 적절하며, 혼합 실내-실외 배치는 어떻게 처리됩니까?
  7. 다양한 센서 유형(GPS, RTK-GPS, LIDAR SLAM, UWB)의 정확도 목표 및 업데이트 속도를 평가하고 이러한 차이가 다양한 보안 시나리오의 적합성에 어떤 영향을 미치는지 설명하십시오.
