Chapter 2: Image Acquisition and Cameras

2.1 Camera Technologies Overview

The camera is the eyes of a machine vision system, converting optical images into digital data that can be processed by computers. Modern machine vision employs several camera technologies, each optimized for specific applications and requirements. Understanding these technologies is essential for selecting the right camera for your application.

Area scan cameras are the most common type in machine vision, capturing a rectangular array of pixels in a single snapshot. They're ideal for inspecting discrete parts, reading codes, and general-purpose imaging. Area scan cameras range from small VGA sensors (640x480 pixels) to ultra-high-resolution sensors exceeding 150 megapixels. Frame rates vary from 10 fps for high-resolution sensors to over 1000 fps for specialized high-speed cameras.

Line scan cameras capture a single row of pixels continuously as an object moves past the camera (or the camera moves past the object). This makes them ideal for inspecting continuous materials like paper, fabric, metal, or glass. Line scan cameras can achieve unlimited image length and very high resolution—up to 16,384 pixels across the width. Line rates reach 400 kHz for specialized sensors.

CCD vs CMOS sensors represent two fundamental approaches to image sensing. CCD (Charge-Coupled Device) sensors offer excellent image quality with low noise and high dynamic range. However, they consume more power and are more expensive to manufacture. CMOS (Complementary Metal-Oxide-Semiconductor) sensors integrate more functionality on the chip, enable faster readout, consume less power, and cost less. Modern sCMOS (scientific CMOS) sensors combine the best of both worlds, offering CCD-like image quality with CMOS speed and integration.

2.2 GigE Vision Standard

GigE Vision has become the most widely adopted interface standard for industrial machine vision, leveraging the ubiquity of Gigabit Ethernet infrastructure. The standard specifies how cameras communicate over standard Ethernet networks, enabling long cable runs, multiple cameras per network, and easy integration with existing IT infrastructure.

Key advantages of GigE Vision include cable lengths up to 100 meters using standard Cat5e/Cat6 cables, with longer distances possible using fiber optics or network switches. Multiple cameras can share a single network interface, simplifying cabling in multi-camera systems. Power over Ethernet (PoE) enables both data and power delivery over a single cable. The standard is based on widely available, low-cost networking components.

GigE Vision specifies several key protocols: GVCP (GigE Vision Control Protocol) handles camera discovery, configuration, and control. GVSP (GigE Vision Streaming Protocol) transfers image data from camera to host. The standard defines how cameras are discovered on the network, how parameters are read and set, and how images are packetized and transmitted.

2.3 USB3 Vision and Other Interfaces

USB3 Vision provides plug-and-play connectivity leveraging USB 3.0/3.1 technology. With bandwidth up to 5-10 Gbps, USB3 Vision supports high-resolution and high-speed applications. The interface requires no configuration—cameras are automatically discovered when connected. A single cable provides both data and power for bus-powered cameras. However, cable length is limited to 5 meters with standard cables, or 30+ meters with active cables.

USB3 Vision is ideal for machine-based applications where cameras are located close to the host computer. It's particularly popular in laboratory equipment, medical devices, and embedded vision systems. The simplified cabling and plug-and-play operation reduce installation complexity compared to GigE Vision.

CoaXPress (CXP) is the high-performance interface for demanding applications. Using standard coaxial cable, CXP-12 provides 12.5 Gbps per link. Cameras can use 1, 2, or 4 links for bandwidth up to 50 Gbps. Cable length reaches 40 meters with standard coax, or 100+ meters with optical extenders. An uplink from host to camera enables camera control and triggering over the same cable.

CoaXPress applications include high-speed inspection of flat panel displays, semiconductor wafer inspection, high-resolution line scan for web inspection, and medical imaging requiring both high resolution and high frame rates. The deterministic latency and reliable performance make CXP suitable for critical applications.

Camera Link remains relevant for applications requiring proven technology and deterministic performance. Camera Link Full configuration provides up to 680 MB/s bandwidth. Camera Link HS extends this to 2.1 GB/s. The standard is mature and widely supported, though newer installations typically use GigE Vision or USB3 Vision unless specific performance requirements dictate Camera Link.

2.4 GenICam Universal Interface

GenICam (Generic Interface for Cameras) revolutionized machine vision by providing a uniform software interface across different camera types and manufacturers. Before GenICam, application software needed custom code for each camera model. GenICam enables one application to work with any compliant camera, dramatically simplifying system development and maintenance.

GenICam consists of several key modules. GenApi defines how camera features are described using XML. Each camera includes an XML file that describes its capabilities, parameters, and how to access them. Host software loads this XML to discover what the camera can do and how to control it. This means new camera models work with existing software without code changes.

SFNC (Standard Features Naming Convention) ensures consistent naming across vendors. ExposureTime, Gain, Width, Height, TriggerMode—these features have the same names and behavior whether you're using a Basler, FLIR, Teledyne, or any other GenICam-compliant camera. This standardization simplifies code and enables camera substitution without application changes.

GenTL (Generic Transport Layer) abstracts differences between physical interfaces. The same GenICam code works with GigE Vision, USB3 Vision, CoaXPress, or Camera Link cameras. Only the transport layer producer (a small DLL or shared library) changes. This enables developing applications that are independent of the camera interface.

// Example GenICam code (same for any interface)
INodeMap *nodeMap = camera->GetNodeMap();
CIntegerPtr width = nodeMap->GetNode("Width");
CIntegerPtr height = nodeMap->GetNode("Height");
CFloatPtr exposureTime = nodeMap->GetNode("ExposureTime");

width->SetValue(1920);
height->SetValue(1080);
exposureTime->SetValue(5000.0); // microseconds

camera->StartGrabbing();
CGrabResultPtr ptrGrabResult;
camera->RetrieveResult(5000, ptrGrabResult);

2.5 Lighting Systems

Proper lighting is critical for machine vision success—more important than camera selection in many applications. The maxim "garbage in, garbage out" applies: poor lighting creates images that even the best algorithms cannot process successfully. Good lighting creates high-contrast images that simplify subsequent processing and improve reliability.

Front lighting is the most common technique, illuminating the object from the camera's direction. It works well for general inspection and reveals surface features. Ring lights mounted around the lens provide even illumination. Diffuse dome lights eliminate shadows and specular reflections. Front lighting is versatile but can create glare on shiny surfaces.

Back lighting (silhouette imaging) places the light source behind the object, creating a high-contrast silhouette. This technique excels for dimensional measurement, edge detection, and detecting holes or voids. It's insensitive to surface finish and color variations. However, it only reveals the object's outline, not surface features.

Dark field lighting uses low-angle illumination to highlight surface defects, scratches, and edges. Light oriented at a shallow angle reflects away from the camera for smooth surfaces, while defects scatter light toward the camera, appearing bright against a dark background. Dark field is excellent for detecting subtle surface defects that would be invisible with front lighting.

Structured light projects patterns (lines, grids, or coded patterns) onto objects for 3D measurement. The pattern deformation reveals surface topology. Laser line projectors create precise 3D profiles. DLP projectors enable high-resolution 3D scanning. Structured light is fundamental to modern 3D vision systems.

Coaxial lighting (on-axis illumination) uses a beam splitter to direct light along the camera's optical axis. This creates uniform illumination of surfaces perpendicular to the camera, making it ideal for reflective surfaces like semiconductor wafers, polished metal, and glass. Coaxial lighting eliminates shadows and reveals defects through reflectivity differences.

2.6 Lens Selection and Optical Design

The lens determines what the camera sees, controlling field of view, working distance, magnification, and depth of field. Selecting the appropriate lens is crucial for system performance and often requires balancing competing requirements.

Focal length is the primary lens specification, determining the relationship between working distance and field of view. Longer focal lengths provide narrower fields of view and higher magnification at a given distance. Shorter focal lengths provide wider fields of view. The lens equation relates focal length (f), working distance (d), and field of view (FOV): FOV = (sensor_size × d) / f.

Working distance is the distance from lens to object. Applications have different working distance requirements based on mechanical constraints, lighting needs, and clearance requirements. Longer working distances reduce perspective distortion and ease mechanical design but require longer focal lengths to achieve the desired field of view.

Depth of field (DOF) is the distance range where objects remain acceptably sharp. DOF depends on focal length, aperture (f-number), and working distance. Small apertures (large f-numbers like f/16) increase DOF but require more light. Large apertures (small f-numbers like f/2.8) minimize DOF but maximize light collection. Applications inspecting flat objects benefit from large DOF, while applications requiring precise distance measurement may use shallow DOF deliberately.

Resolution and MTF (Modulation Transfer Function) characterize lens performance. The lens must resolve the smallest features of interest with adequate contrast. MTF curves show how contrast varies with spatial frequency, indicating how well the lens performs. High-quality machine vision lenses maintain high MTF across the entire image field, ensuring consistent performance from center to edge.

Distortion causes straight lines to appear curved. Barrel distortion (lines bow outward) occurs with short focal lengths. Pincushion distortion (lines bow inward) occurs with long focal lengths. Distortion affects dimensional measurements and must be corrected through calibration or choice of low-distortion lenses. Telecentric lenses eliminate perspective distortion entirely, maintaining constant magnification across depth.

2.7 Chapter Summary

This chapter covered the image acquisition subsystem—cameras, interfaces, and lighting—which form the foundation of any machine vision system. We explored camera technologies including area scan and line scan, CCD and CMOS sensors, and various resolution and speed options. We examined interface standards including GigE Vision, USB3 Vision, CoaXPress, and Camera Link, understanding their strengths and appropriate applications.

GenICam provides universal camera control, enabling application software to work with any compliant camera regardless of manufacturer or interface. This standardization simplifies development and maintenance while enabling best-of-breed component selection.

Lighting emerged as perhaps the most critical aspect of image acquisition. Proper illumination can make inspection trivial, while poor lighting can make even simple tasks impossible. We reviewed lighting techniques including front, back, dark field, structured, and coaxial illumination, along with various light sources from LED to laser.

Optical design considerations including lens selection, focal length, working distance, depth of field, and distortion round out the image acquisition system. These elements must work together harmoniously to capture images suitable for the application's requirements.
