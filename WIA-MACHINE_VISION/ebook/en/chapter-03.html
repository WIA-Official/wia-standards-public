Chapter 3: Object Detection and Classification

3.1 Traditional Computer Vision Techniques

Before deep learning dominated machine vision, practitioners relied on traditional computer vision algorithms developed over decades of research. These techniques remain relevant today, often complementing deep learning or providing better solutions for specific applications where rules-based approaches are more appropriate or efficient.

Edge detection identifies boundaries between regions of different intensity, fundamental to many vision tasks. The Sobel operator applies directional filters to detect horizontal and vertical edges. The Canny edge detector uses multiple stages—smoothing, gradient calculation, non-maximum suppression, and hysteresis thresholding—to produce clean single-pixel edges. Edge detection enables dimensionalmeasurement, shape analysis, and object localization.

Thresholding and segmentation separate objects from background based on intensity or color. Simple global thresholding works when object and background have distinctly different intensities. Adaptive thresholding handles non-uniform illumination by computing thresholds locally. Otsu's method automatically determines optimal threshold values. Color-based segmentation uses hue, saturation, and value to distinguish objects with different colors.

Blob analysis identifies and measures connected regions (blobs) in binary images. After thresholding, blob analysis computes properties like area, perimeter, centroid, orientation, and bounding box. These measurements enable part counting, size verification, orientation determination, and defect detection. Modern implementations process thousands of blobs per image at video rates.

Pattern matching locates objects by comparing image regions to reference templates. Normalized cross-correlation measures similarity between template and image region, robust to intensity variations. Geometric pattern matching uses edge-based features to find objects invariant to rotation and scale. Pattern matching enables part localization, alignment verification, and code reading when combined with OCR.

Geometric measurement tools compute distances, angles, circles, lines, and other geometric features. Caliper tools measure edge positions with sub-pixel accuracy. Circle/ellipse finders fit geometric primitives to edge points. Point-to-point distance measurement enables dimensional inspection. These tools achieve micron-level accuracy when properly calibrated.

3.2 Deep Learning Fundamentals

Deep learning has revolutionized machine vision by learning features directly from data rather than requiring hand-crafted algorithms. Convolutional neural networks (CNNs) form the foundation of modern deep learning for vision, leveraging the hierarchical structure of images to learn increasingly complex features at each layer.

A CNN consists of several types of layers. Convolutional layers apply learned filters to detect features. Early layers detect simple features like edges and corners. Deeper layers combine these into more complex features like textures and object parts. The deepest layers recognize complete objects. Pooling layers reduce spatial dimensions while preserving important features, providing translation invariance. Fully connected layers at the network's end combine features for final classification or detection.

Training a neural network requires labeled data—images annotated with the information the network should learn. For classification, this means associating images with class labels (e.g., "good" or "defective"). For object detection, annotations specify bounding boxes around objects plus their class labels. For segmentation, annotations label each pixel with its class.

The training process optimizes millions of parameters. The network processes training images, comparing its predictions to ground truth labels. A loss function quantifies prediction errors. Backpropagation computes gradients indicating how to adjust parameters to reduce loss. An optimizer (like Adam or SGD) updates parameters iteratively. After many training epochs, the network learns to generalize from training data to new images.

3.3 YOLO: Real-Time Object Detection

YOLO (You Only Look Once) represents a breakthrough in real-time object detection, processing images in a single pass through the network. Unlike two-stage detectors that first propose regions then classify them, YOLO frames detection as a single regression problem, directly predicting bounding boxes and class probabilities.

The YOLO architecture divides the input image into a grid. Each grid cell predicts multiple bounding boxes along with confidence scores and class probabilities. The network outputs thousands of candidate detections, which are filtered using non-maximum suppression (NMS) to eliminate duplicates. This single-pass approach enables real-time performance—YOLOv8 achieves 60+ FPS on modern GPUs.

Recent YOLO versions incorporate numerous improvements. YOLOv5 introduced anchor-free detection, simplifying training and improving generalization. YOLOv7 added architectural innovations like ELAN (Efficient Layer Aggregation Network) for better feature extraction. YOLOv8 provides multiple model sizes—nano, small, medium, large, and extra-large—enabling optimization for different deployment scenarios from edge devices to high-end servers.

Training YOLO requires properly formatted annotations. The standard YOLO format specifies each object with class ID and normalized bounding box coordinates (center_x, center_y, width, height). Data augmentation during training—random flips, crops, color jittering, and mosaic composition—improves model robustness. Hyperparameter tuning of learning rate, batch size, image size, and augmentation strength optimizes performance for specific applications.

    input_size=640,
    augmentation='medium'

    dataset=dataset,
    epochs=100,
    batch_size=16,
    device='cuda',
    patience=10

# Evaluate on test set

print(f"mAP@0.5: {metrics.map_50:.3f}")
print(f"Precision: {metrics.precision:.3f}")
print(f"Recall: {metrics.recall:.3f}")
print(f"Inference time: {metrics.inference_ms:.1f}ms")

3.4 Training Data and Annotation

Training data quality determines model performance more than architecture choice. "Garbage in, garbage out" applies: models trained on poor data produce poor results. Collecting, annotating, and managing training data requires significant effort but pays dividends in model accuracy and robustness.

Data collection should capture the full range of variation the model will encounter. This includes normal parts and all defect types, different lighting conditions, various part orientations, different production lots, and edge cases. Insufficient variation leads to models that work on training data but fail in production. Aim for at least 100-200 examples per class, more for complex or variable classes.

Annotation quality is critical. Bounding boxes should tightly bound objects without cutting off parts. Class labels must be consistent—ambiguous cases handled the same way throughout. Multiple annotators should review difficult cases. Inter-annotator agreement metrics like Cohen's kappa quantify consistency. Professional annotation services can ensure quality but understanding your application is essential for handling edge cases correctly.

Class balance affects training. If "good" parts outnumber defects 1000:1, the model might learn to classify everything as "good" and achieve 99.9% accuracy while detecting no defects. Techniques to address imbalance include oversampling rare classes, synthetic data generation, and class-weighted loss functions. The goal is ensuring the model learns all classes equally well.

3.5 Model Deployment and Optimization

After training, models must be deployed for inference in production. This requires optimization for target hardware, integration with the vision system, and monitoring of performance. Deployment environments range from edge devices with limited computing power to cloud servers with multiple GPUs.

Model quantization reduces model size and increases inference speed by using lower precision numerics. Converting from 32-bit floating point (FP32) to 16-bit floating point (FP16) typically doubles speed with negligible accuracy loss. 8-bit integer (INT8) quantization can provide 3-4x speedup but requires careful calibration to maintain accuracy. Post-training quantization works on trained models, while quantization-aware training produces better results by learning quantization-robust weights.

Model pruning removes unnecessary connections and filters from neural networks, reducing model size and computational requirements. Structured pruning removes entire filters or layers, making models smaller and faster without specialized hardware. Unstructured pruning removes individual weights, achieving higher compression but requiring sparse computation support. Pruning can reduce model size by 50-90% with 1-2% accuracy loss.

Inference optimization frameworks provide optimized implementations for specific hardware. TensorRT (NVIDIA) optimizes models for NVIDIA GPUs, often achieving 2-5x speedup over standard PyTorch or TensorFlow inference. OpenVINO (Intel) optimizes for Intel CPUs, GPUs, and VPUs. CoreML (Apple) targets Apple silicon. ONNX Runtime provides broad hardware support with good performance.

Edge deployment requires additional considerations. Model size must fit in device memory, often requiring quantization and pruning. Inference latency must meet real-time requirements—typically 10-50ms for inspection applications. Power consumption matters for battery-powered devices. NVIDIA Jetson, Google Coral, Intel Movidius, and similar edge AI accelerators balance performance and efficiency.

3.6 Advanced Techniques

Beyond standard training and deployment, several advanced techniques enhance deep learning performance for machine vision applications.

Transfer learning leverages pre-trained models rather than training from scratch. Models pre-trained on large datasets like ImageNet have learned general visual features (edges, textures, shapes) applicable to many tasks. Fine-tuning these models for specific applications requires far less data and training time than starting from random weights. Transfer learning is particularly valuable when training data is limited.

Data augmentation artificially increases training set size and diversity by transforming images. Standard augmentations include geometric transforms (rotation, scaling, flipping, cropping), color adjustments (brightness, contrast, saturation), and noise injection. Advanced augmentations like CutMix, MixUp, and Mosaic combine multiple images, improving model robustness to partial occlusions and complex scenes.

Active learning strategically selects which images to annotate next, focusing on examples that most improve model performance. The model identifies images it's uncertain about—conflicting predictions, low confidence, or novel features. Human annotators label these informative examples. This iterative process achieves target accuracy with fewer annotated images than random selection.

Few-shot learning enables models to learn new classes from very few examples—sometimes just one or two. Techniques like prototypical networks and siamese networks learn to compare images in a learned feature space. This is valuable for manufacturing where new defect types emerge occasionally but examples are limited.

Anomaly detection identifies defects without specific training examples by learning what "normal" looks like. Autoencoders reconstruct normal images well but fail on anomalies, with reconstruction error indicating defects. One-class SVMs and isolation forests identify statistical outliers. These approaches handle novel defect types not seen during training.
