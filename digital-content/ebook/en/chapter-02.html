Chapter 2: Multimedia Content Formats

2.1 Overview of Multimedia Standards

WIA-EDU-019 defines comprehensive specifications for all major types of multimedia content. The standard prioritizes open formats, broad compatibility, and future-proof encoding techniques.

2.2 Video Content

Supported Video Formats

Container Codecs Use Case Status
MP4 H.264 (AVC), H.265 (HEVC) General purpose, streaming ✅ Required
WebM VP8, VP9, AV1 Web delivery, open source ✅ Required
MOV ProRes, H.264 High-quality production ⚠️ Optional

Video Specifications

Minimum Requirements:
  • Resolution: 1280x720 (720p HD)
  • Frame Rate: 24-30 fps
  • Bitrate: 2.5 Mbps minimum
  • Color Space: BT.709
Recommended Standards:
  • Resolution: 1920x1080 (1080p Full HD)
  • Frame Rate: 30 fps
  • Bitrate: 5-8 Mbps
  • HDR: HDR10 or HLG (for HDR content)

Adaptive Streaming

All video content MUST support at least one adaptive streaming protocol:

  • HLS (HTTP Live Streaming): Apple standard, broad support
  • DASH (MPEG-DASH): ISO standard, codec-agnostic
  • Smooth Streaming: Microsoft standard
# HLS Manifest Example
#EXTM3U
#EXT-X-VERSION:3
#EXT-X-STREAM-INF:BANDWIDTH=2000000,RESOLUTION=1280x720
720p.m3u8
#EXT-X-STREAM-INF:BANDWIDTH=5000000,RESOLUTION=1920x1080
1080p.m3u8
#EXT-X-STREAM-INF:BANDWIDTH=8000000,RESOLUTION=3840x2160
4k.m3u8

2.3 Audio Content

Supported Audio Formats

Format Codec Quality Status
MP3 MPEG-1 Layer 3 128-320 kbps ✅ Required
AAC Advanced Audio Coding 128-256 kbps ✅ Required
OGG Vorbis, Opus 96-256 kbps ✅ Required
FLAC Free Lossless Lossless ⚠️ Optional

Audio Specifications

  • Sample Rate: 44.1 kHz minimum (48 kHz recommended)
  • Bit Depth: 16-bit minimum (24-bit for high-quality)
  • Channels: Stereo minimum (5.1/7.1 surround optional)
  • Bitrate: 128 kbps minimum (256 kbps recommended)

2.4 Image Content

Supported Image Formats

Format Type Use Case Status
JPEG Lossy Photos, general images ✅ Required
PNG Lossless Graphics, transparency ✅ Required
WebP Both Web optimization ✅ Required
AVIF Both Next-gen compression ⚠️ Recommended
SVG Vector Scalable graphics ✅ Required

Image Specifications

  • Color Space: sRGB for web, Adobe RGB for print
  • Resolution: Minimum 72 DPI for web, 300 DPI for print
  • Compression: Quality 85-95 for JPEG
  • Metadata: EXIF, IPTC, XMP embedded

2.5 3D Content

Supported 3D Formats

  • glTF 2.0: Standard for web and mobile 3D (required)
  • USD: Universal Scene Description (recommended)
  • FBX: Autodesk format for interchange (optional)
  • OBJ: Simple mesh format (optional)

3D Specifications

  • Polygon Count: Optimized for target platform
  • Textures: Power-of-2 dimensions, compressed
  • Materials: PBR (Physically Based Rendering)
  • Animations: Skeletal or morph targets

2.6 Document Formats

Supported Document Formats

  • PDF/A: Archival PDF (required for documents)
  • EPUB 3: Reflowable ebooks (required for books)
  • HTML5: Web-based documents (required)
  • Markdown: Lightweight markup (optional)

2.7 Content Validation

All multimedia content MUST pass validation tests:

# Example validation with WIA tools
wia-validate --type video --format mp4 content.mp4
wia-validate --type audio --format aac audio.aac
wia-validate --type image --format webp image.webp

# Batch validation
wia-validate --recursive ./content/

2.8 Best Practices

  1. Multi-format Support: Provide multiple formats for maximum compatibility
  2. Compression: Balance quality and file size
  3. Accessibility: Include alternative text, captions, transcripts
  4. Metadata: Embed comprehensive metadata in all files
  5. Testing: Validate across multiple devices and platforms
