Chapter 4: Content Packaging & Distribution

4.1 WIA Content Package Format

The WIA Content Package (.wcp) is the standard container format for distributing digital content. It ensures compatibility, integrity, and ease of deployment across all platforms.

Package Structure

content-package.wcp (ZIP archive)
├── manifest.json           # Package metadata and structure
├── metadata.json           # Detailed content metadata
├── license.txt             # License information
├── content/                # Main content files
│   ├── index.html
│   ├── video.mp4
│   ├── audio.mp3
│   ├── images/
│   └── scripts/
├── assets/                 # Supporting assets
│   ├── thumbnail.jpg
│   ├── poster.png
│   └── preview.mp4
├── accessibility/          # Accessibility resources
│   ├── captions_en.vtt
│   ├── captions_ko.vtt
│   ├── transcript.txt
│   └── audio_description.mp3
└── signatures/             # Digital signatures
    ├── package.sig
    └── checksums.sha256

4.3 Compression Standards

Supported Formats

  • ZIP: Standard compression (required)
  • TAR.GZ: UNIX-style archive (optional)
  • 7-Zip: High compression (optional)

Compression Guidelines

  • Use ZIP with deflate compression
  • Compression level: 6-9 (balance speed vs. size)
  • Store already-compressed files uncompressed (MP4, JPG, PNG)
  • Maximum package size: 2 GB (split larger packages)

4.4 Content Delivery Networks (CDN)

CDN Requirements

Content SHOULD be distributed via CDN for optimal performance:

  • Global Distribution: Edge servers worldwide
  • HTTPS Only: Secure delivery
  • HTTP/2 or HTTP/3: Modern protocols
  • Adaptive Bitrate: Dynamic quality adjustment
  • Caching: Intelligent cache policies

CDN Configuration Example

# CloudFront CDN configuration
{
  "Origins": [
    {
      "DomainName": "content.wia.org",
      "OriginProtocolPolicy": "https-only"
    }
  ],
  "CacheBehaviors": {
    "PathPattern": "*.mp4",
    "MinTTL": 86400,
    "MaxTTL": 31536000,
    "Compress": true
  },
  "CustomErrorResponses": [
    {
      "ErrorCode": 404,
      "ResponsePagePath": "/error.html"
    }
  ]
}

4.5 Version Control

Semantic Versioning

Content packages MUST follow semantic versioning (SemVer):

Version Format: MAJOR.MINOR.PATCH
  • MAJOR: Incompatible changes (1.x.x → 2.0.0)
  • MINOR: New features, backwards-compatible (1.0.x → 1.1.0)
  • PATCH: Bug fixes, backwards-compatible (1.0.0 → 1.0.1)

Version Metadata

{
  "version": "2.1.3",
  "releaseDate": "2025-01-15",
  "changelog": "Fixed accessibility issues in video player",
  "deprecated": false,
  "deprecationDate": null,
  "supportedUntil": "2027-01-15"
}

4.6 Update Mechanisms

Delta Updates

For large content packages, support delta (incremental) updates:

  • Only download changed files
  • Binary diff for media files
  • Verify integrity with checksums
  • Rollback capability on failure

Update Process

# Check for updates
wia-update check --package content-package.wcp

# Download and apply delta update
wia-update apply --from 2.1.2 --to 2.1.3

# Verify update integrity
wia-update verify --package content-package.wcp

4.7 Digital Signatures

Package Signing

All content packages SHOULD be digitally signed to ensure authenticity and integrity: 

# Generate signature
wia-sign --key private.pem --package content-package.wcp

# Verify signature
wia-verify --key public.pem --package content-package.wcp

Checksum Verification

# checksums.sha256
3f7a8b2c1d4e5f6a7b8c9d0e1f2a3b4c  content/video.mp4
5d6e7f8a9b0c1d2e3f4a5b6c7d8e9f0a  content/audio.mp3
1a2b3c4d5e6f7a8b9c0d1e2f3a4b5c6d  content/index.html

# Verify all files
sha256sum -c checksums.sha256

4.8 Adaptive Packaging

Multi-Quality Bundles

Packages SHOULD include multiple quality levels:

  • Base Package: Essential content only (smallest)
  • Standard Package: 720p-1080p quality
  • Premium Package: 4K, HDR, lossless audio
  • Accessibility Package: All accessibility resources

Modular Packaging

base-package.wcp          # Core content (100 MB)
├── manifest.json
└── content/
    └── index.html

quality-720p.wcp          # HD video (500 MB)
└── content/video-720p.mp4

quality-1080p.wcp         # Full HD (1.2 GB)
└── content/video-1080p.mp4

accessibility.wcp         # A11y resources (50 MB)
└── accessibility/
    ├── captions.vtt
    └── transcript.txt

4.9 Deployment Strategies

Deployment Methods

  1. Direct Download: Single package file
  2. Progressive Download: Stream while downloading
  3. Chunked Delivery: Download in segments
  4. P2P Distribution: Peer-to-peer sharing (WebTorrent)
Best Practice: Combine CDN delivery with client-side caching for optimal performance and offline capability.
