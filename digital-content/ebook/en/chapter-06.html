Chapter 6: Content Licensing & Rights Management

6.1 Understanding Content Licensing

Proper licensing is essential for protecting creators' rights while enabling content sharing and reuse. WIA-EDU-019 supports multiple licensing frameworks to accommodate diverse needs.

6.4 Digital Rights Management (DRM)

DRM Approaches

WIA Position on DRM:

WIA-EDU-019 supports DRM when necessary but encourages open licensing. DRM, if used, MUST NOT prevent:

  • Accessibility features (screen readers, captions)
  • Fair use and educational exceptions
  • Personal backups for legitimate users
  • Cross-platform compatibility

Supported DRM Systems

  • Widevine: Google's DRM (video streaming)
  • FairPlay: Apple's DRM (iOS/macOS)
  • PlayReady: Microsoft's DRM (Windows, Xbox)
  • Adobe Primetime: Cross-platform DRM

6.5 Open Educational Resources (OER)

OER Principles

WIA strongly supports Open Educational Resources:

  • 5R Permissions: Retain, Reuse, Revise, Remix, Redistribute
  • Open License: CC BY or CC BY-SA preferred
  • Open Format: Non-proprietary file formats
  • No Technical Barriers: Freely accessible

6.7 Attribution Requirements

Proper Attribution

TASL Model for Attribution:
  • T - Title of the work
  • A - Author/Creator name
  • S - Source (URL or location)
  • L - License type

Attribution Example

<div class="attribution">
  <p>
    "<a href="https://example.com/content">Digital Content Guide</a>"
    by <a href="https://example.com/author">Jane Smith</a>
    is licensed under
    <a href="https://creativecommons.org/licenses/by/4.0/">CC BY 4.0</a>
  </p>
</div>

6.8 License Verification

Validation Tools

# Verify license validity
wia-license verify content-package.wcp

# Check license compatibility
wia-license check-compatibility --from "CC BY" --to "CC BY-SA"

# Generate license report
wia-license report --package content-package.wcp > license-report.pdf

6.9 Rights Clearance

Third-Party Content

When incorporating third-party content:

  1. Verify license permits intended use
  2. Obtain necessary permissions
  3. Document all licenses in manifest
  4. Provide proper attribution
  5. Track expiration dates

6.10 Best Practices

Licensing Best Practices:
  1. Choose Appropriate License: Match license to your goals
  2. Be Clear and Visible: Display license prominently
  3. Use Machine-Readable Format: Enable automated compliance
  4. Document Everything: Keep records of all permissions
  5. Respect Upstream Licenses: Honor original creators' terms
  6. Provide License File: Include license.txt in package
  7. Update Regularly: Review licenses periodically
