Chapter 5: Metadata & Classification

5.1 Importance of Metadata

Metadata is the "data about data" that makes digital content discoverable, understandable, and manageable. Comprehensive metadata is crucial for search, classification, accessibility, and rights management.

5.5 Content Classification

Educational Taxonomy

  • Bloom's Taxonomy: Cognitive skill levels
  • Revised Bloom's: Creating, Evaluating, Analyzing, Applying, Understanding, Remembering
  • SOLO Taxonomy: Structure of Observed Learning Outcomes
  • Webb's DOK: Depth of Knowledge levels

Content Type Classification

Primary Categories:
  • Video Content (lecture, demonstration, tutorial)
  • Audio Content (podcast, audiobook, music)
  • Interactive Content (simulation, game, quiz)
  • Document Content (article, ebook, manual)
  • Mixed Media (multimedia presentation)

5.6 Tagging and Keywords

Keyword Best Practices

  • Use 5-15 relevant keywords
  • Include synonyms and related terms
  • Support multiple languages
  • Avoid keyword stuffing
  • Use hierarchical tags (parent → child)

Tagging Example

{
  "keywords": [
    "digital content",
    "multimedia standards",
    "educational technology",
    "accessibility",
    "WCAG compliance"
  ],
  "tags": {
    "subject": ["Computer Science", "Education Technology"],
    "level": ["Advanced"],
    "type": ["Tutorial", "Interactive"],
    "language": ["en", "ko"]
  },
  "categories": {
    "primary": "Educational Technology",
    "secondary": ["Standards", "Accessibility", "Multimedia"]
  }
}

5.8 Metadata Management

Metadata Tools

# Extract metadata
wia-metadata extract content.mp4 > metadata.json

# Validate metadata
wia-metadata validate metadata.json

# Embed metadata
wia-metadata embed --input content.mp4 --metadata metadata.json

# Export to different formats
wia-metadata export --format dublin-core metadata.json
wia-metadata export --format schema.org metadata.json

Metadata Validation

Validation Checklist:
  • ✅ All required fields present
  • ✅ Valid format and encoding
  • ✅ Consistent across languages
  • ✅ No conflicts or duplicates
  • ✅ Embedded in content files
  • ✅ Accessible via API
