Chapter 7: Accessibility & Inclusivity

7.1 The Importance of Accessibility

Accessibility is not optional—it's a fundamental requirement of WIA-EDU-019. Every person, regardless of their abilities, deserves equal access to digital content. This aligns with our core philosophy of 弘益人間 (Benefit All Humanity).

Legal Requirements:
  • ADA: Americans with Disabilities Act (US)
  • Section 508: Federal accessibility standards (US)
  • EN 301 549: European accessibility standard
  • AODA: Accessibility for Ontarians with Disabilities Act (Canada)

7.2 WCAG 2.1 Compliance

Compliance Levels

WIA-EDU-019 requires WCAG 2.1 Level AA compliance minimum:

  • Level A: Basic accessibility (minimum)
  • Level AA: Recommended standard (required for WIA)
  • Level AAA: Enhanced accessibility (encouraged)

Four Principles (POUR)

  1. Perceivable: Information must be presentable in ways users can perceive
  2. Operable: User interface must be operable by all users
  3. Understandable: Information and operation must be understandable
  4. Robust: Content must work with current and future technologies

7.3 Video Accessibility

Captions (Closed Captions)

All video content MUST include accurate, synchronized captions in the primary language. Additional language captions SHOULD be provided.

WebVTT Caption Example

WEBVTT

  <track kind="captions" src="captions_es.vtt" srclang="es" label="Español">

Audio Descriptions

For content with important visual information not conveyed through dialogue, audio descriptions MUST be provided. 

<video controls>
  <source src="video.mp4" type="video/mp4">
  <track kind="descriptions" src="audio_desc.vtt" srclang="en" label="Audio Descriptions">
</video>

Transcripts

A complete text transcript SHOULD be provided for all video and audio content.

7.4 Document Accessibility

Semantic HTML

<!-- Use semantic elements -->
<header>
  <h1>Main Title</h1>
  <nav aria-label="Main navigation">...</nav>
</header>

<main>
  <article>
    <h2>Section Title</h2>
    <p>Content...</p>
  </article>
</main>

<aside aria-label="Related content">...</aside>

<footer>...</footer>

ARIA Labels

<!-- Provide accessible names -->
<button aria-label="Close dialog">×</button>

<!-- Describe purpose -->
<div role="search" aria-label="Site search">
  <input type="search" aria-label="Search query">
  <button type="submit">Search</button>
</div>

<!-- Live regions for dynamic content -->
<div aria-live="polite" aria-atomic="true">
  Loading results...
</div>

Keyboard Navigation

Keyboard Requirements:
  • Tab: Navigate forward through interactive elements
  • Shift+Tab: Navigate backward
  • Enter/Space: Activate buttons and links
  • Arrow Keys: Navigate within components (menus, tabs)
  • Esc: Close dialogs and dismiss overlays

7.5 Color and Contrast

Contrast Ratios

  • Normal Text: 4.5:1 minimum (AA), 7:1 recommended (AAA)
  • Large Text: 3:1 minimum (AA), 4.5:1 recommended (AAA)
  • UI Components: 3:1 minimum for interactive elements

Color Independence

Never rely on color alone to convey information. Use additional indicators: 

<!-- Good: Color + icon + text -->
<button class="success">
  <span class="icon">✓</span>
  <span>Success</span>
</button>

<button class="error">
  <span class="icon">✗</span>
  <span>Error</span>
</button>

7.6 Screen Reader Support

Image Alt Text

<!-- Informative images -->
<img src="chart.jpg" alt="Bar chart showing 45% increase in accessibility compliance from 2023 to 2025">

<!-- Decorative images -->
<img src="decoration.jpg" alt="" role="presentation">

<!-- Complex images -->
<figure>
  <img src="diagram.jpg" alt="System architecture diagram" aria-describedby="diagram-desc">
  <figcaption id="diagram-desc">
    Detailed description of the system architecture showing...
  </figcaption>
</figure>

Form Accessibility

<form>
  <div class="form-group">
    <label for="email">Email Address</label>
    <input
      type="email"
      id="email"
      name="email"
      required
      aria-required="true"
      aria-describedby="email-help"
    >
    <small id="email-help">We'll never share your email.</small>
  </div>

  <div class="form-group">
    <fieldset>
      <legend>Notification Preferences</legend>
      <label>
        <input type="checkbox" name="email-notify">
        Email notifications
      </label>
      <label>
        <input type="checkbox" name="sms-notify">
        SMS notifications
      </label>
    </fieldset>
  </div>
</form>

7.7 Cognitive Accessibility

Clear Language

  • Use simple, concise language
  • Avoid jargon and technical terms when possible
  • Provide definitions for complex terms
  • Use bulleted lists and short paragraphs

Consistent Navigation

  • Keep navigation consistent across pages
  • Use clear, descriptive labels
  • Provide breadcrumbs for deep navigation
  • Include a search function

Error Prevention & Recovery

<!-- Clear error messages -->
<div class="error" role="alert">
  <strong>Error:</strong> Email address is required.
  Please enter a valid email address.
</div>

<!-- Confirmation for important actions -->
<button onclick="confirmDelete()">Delete Account</button>

<script>
function confirmDelete() {
  if (confirm('Are you sure you want to delete your account? This action cannot be undone.')) {
    // Proceed with deletion
  }
}
</script>

7.8 Testing and Validation

Automated Testing

# Use automated accessibility testing tools
npm install -g axe-cli

# Test a webpage
axe https://example.com --tags wcag2aa

# Test local file
axe index.html --tags wcag21aa

# Generate report
axe --output report.json index.html

Manual Testing

Testing Checklist:
  1. ✅ Navigate entire site using only keyboard
  2. ✅ Test with screen reader (NVDA, JAWS, VoiceOver)
  3. ✅ Verify color contrast ratios
  4. ✅ Test with browser zoom at 200%
  5. ✅ Verify all images have alt text
  6. ✅ Check all videos have captions
  7. ✅ Test forms for proper labeling
  8. ✅ Verify ARIA labels are appropriate

7.10 Continuous Improvement

Accessibility is an ongoing commitment, not a one-time achievement:

  • Conduct regular accessibility audits
  • Gather feedback from users with disabilities
  • Stay updated with WCAG guidelines
  • Train content creators on accessibility
  • Budget for accessibility improvements
Remember: Accessibility benefits everyone. Captions help in noisy environments, keyboard navigation benefits power users, and clear language helps non-native speakers. By making content accessible, we make it better for all.
