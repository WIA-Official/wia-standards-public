개요

WIA-ART-002 표준의 진정한 가치는 실제 구현에 있습니다. 표준을 정의하는 것만으로는 충분하지 않으며, 개발자들이 쉽게 통합할 수 있는 RESTful API, 직관적인 SDK, 그리고 실시간 통신을 위한 WebSocket 인터페이스가 필요합니다. 이 장에서는 WIA-ART-002의 실무 구현을 상세히 다룹니다.

현대 AI 이미지 생성 서비스는 단순한 동기식 요청-응답을 넘어섭니다. 고품질 이미지 생성은 수십 초에서 수 분이 걸릴 수 있으며, 배치 처리, 큐 관리, 실시간 진행 상황 스트리밍이 필요합니다. 또한 API 속도 제한(Rate Limiting), 사용량 할당(Quota), 비용 추적, 그리고 OAuth 2.0 기반 인증 시스템을 갖춰야 합니다.

RESTful API 설계 원칙

1. API 엔드포인트 구조

WIA-ART-002 API는 REST 아키텍처 원칙을 따르며, 명확한 리소스 기반 URL 구조를 사용합니다:

엔드포인트 메서드 기능 인증 필요
/api/v1/generate POST 새 이미지 생성 요청 필수
/api/v1/jobs/{jobId} GET 생성 작업 상태 조회 필수
/api/v1/jobs/{jobId}/cancel POST 작업 취소 필수
/api/v1/artworks GET 작품 목록 조회 필수
/api/v1/artworks/{id} GET 특정 작품 상세 정보 선택
/api/v1/artworks/{id}/metadata GET WIA-ART-002 메타데이터 선택
/api/v1/artworks/{id}/verify POST 작품 진위 검증 선택
/api/v1/models GET 사용 가능 모델 목록 선택
/api/v1/quota GET 사용량 할당 확인 필수

TypeScript SDK 구현

1. SDK 아키텍처

WIA-ART-002 TypeScript SDK는 Promise 기반의 현대적인 API를 제공하며, 자동 재시도, 에러 처리, 타입 안정성을 갖춥니다: 

// wia-art-002-sdk/src/client.ts
import axios, { AxiosInstance } from 'axios';
import { EventEmitter } from 'events';
import WebSocket from 'ws';

export interface WIAArtConfig {
  apiKey: string;
  baseUrl?: string;
  timeout?: number;
  retries?: number;
}

export interface GenerationRequest {
  prompt: string;
  negativePrompt?: string;
  model?: string;
  parameters?: GenerationParameters;
  metadata?: ArtworkMetadata;
  options?: GenerationOptions;
}

export interface GenerationParameters {
  width?: number;
  height?: number;
  steps?: number;
  cfgScale?: number;
  sampler?: string;
  seed?: number;
  batchSize?: number;
}

export class WIAArtClient extends EventEmitter {
  private axios: AxiosInstance;
  private config: WIAArtConfig;

  constructor(config: WIAArtConfig) {
    super();
    this.config = {
      baseUrl: 'https://api.wia-art.org',
      timeout: 60000,
      retries: 3,
      ...config
    };

    this.axios = axios.create({
      baseURL: this.config.baseUrl,
      timeout: this.config.timeout,
      headers: {
        'Authorization': `Bearer ${this.config.apiKey}`,
        'Content-Type': 'application/json',
        'User-Agent': 'WIA-ART-002-SDK/1.0'
      }
    });

    this.setupInterceptors();
  }

  // 이미지 생성 시작
  async generate(request: GenerationRequest): Promise {
    try {
      const response = await this.axios.post('/api/v1/generate', request);
      return new GenerationJob(this, response.data);
    } catch (error) {
      throw this.handleError(error);
    }
  }

  // 작업 상태 조회
  async getJob(jobId: string): Promise {
    const response = await this.axios.get(`/api/v1/jobs/${jobId}`);
    return response.data;
  }

  // 작품 조회
  async getArtwork(artworkId: string): Promise {
    const response = await this.axios.get(`/api/v1/artworks/${artworkId}`);
    return response.data;
  }

  // 메타데이터 조회
  async getMetadata(artworkId: string): Promise {
    const response = await this.axios.get(
      `/api/v1/artworks/${artworkId}/metadata`
    );
    return response.data;
  }

  // 작품 검증
  async verifyArtwork(artworkId: string): Promise {
    const response = await this.axios.post(
      `/api/v1/artworks/${artworkId}/verify`
    );
    return response.data;
  }

  // 사용량 할당 확인
  async getQuota(): Promise {
    const response = await this.axios.get('/api/v1/quota');
    return response.data;
  }

  // 에러 처리
  private handleError(error: any): Error {
    if (error.response) {
      const { status, data } = error.response;
      switch (status) {
        case 401:
          return new Error('Invalid API key or expired token');
        case 402:
          return new Error('Insufficient credits');
        case 429:
          return new Error(`Rate limit exceeded: ${data.message}`);
        case 500:
          return new Error('Server error, please try again later');
        default:
          return new Error(data.message || 'Unknown error');
      }
    }
    return error;
  }

  // 자동 재시도 설정
  private setupInterceptors() {
    this.axios.interceptors.response.use(
      response => response,
      async error => {
        const config = error.config;
        if (!config || !config.retry) {
          config.retry = 0;
        }

        if (config.retry >= this.config.retries!) {
          return Promise.reject(error);
        }

        config.retry += 1;

        // 지수 백오프
        const delay = Math.pow(2, config.retry) * 1000;
        await new Promise(resolve => setTimeout(resolve, delay));

        return this.axios(config);
      }
    );
  }
}

2. 생성 작업 클래스

GenerationJob 클래스는 진행 상황 추적, WebSocket 스트리밍, 완료 대기 등을 캡슐화합니다: 

// wia-art-002-sdk/src/job.ts
export class GenerationJob extends EventEmitter {
  private client: WIAArtClient;
  private jobData: any;
  private ws?: WebSocket;
  private pollInterval?: NodeJS.Timeout;

  constructor(client: WIAArtClient, jobData: any) {
    super();
    this.client = client;
    this.jobData = jobData;
  }

  get id(): string {
    return this.jobData.jobId;
  }

  get status(): string {
    return this.jobData.status;
  }

  // WebSocket으로 실시간 진행 상황 스트리밍
  stream(): this {
    if (this.ws) return this;

    this.ws = new WebSocket(this.jobData.websocketUrl, {
      headers: { 'Authorization': `Bearer ${this.client.config.apiKey}` }
    });

    this.ws.on('message', (data: string) => {
      const update = JSON.parse(data);
      this.emit('progress', update);

      if (update.status === 'completed') {
        this.emit('completed', update.results);
        this.cleanup();
      } else if (update.status === 'failed') {
        this.emit('failed', new Error(update.error));
        this.cleanup();
      }
    });

    this.ws.on('error', (error) => {
      this.emit('error', error);
      this.cleanup();
    });

    return this;
  }

  // 폴링으로 진행 상황 확인
  poll(intervalMs: number = 2000): this {
    this.pollInterval = setInterval(async () => {
      try {
        const status = await this.client.getJob(this.id);
        this.emit('progress', status);

        if (status.status === 'completed') {
          this.emit('completed', status.results);
          this.cleanup();
        } else if (status.status === 'failed') {
          this.emit('failed', new Error(status.error));
          this.cleanup();
        }
      } catch (error) {
        this.emit('error', error);
      }
    }, intervalMs);

    return this;
  }

  // 완료까지 대기 (Promise)
  async wait(): Promise {
    return new Promise((resolve, reject) => {
      this.stream()
        .on('completed', resolve)
        .on('failed', reject)
        .on('error', reject);
    });
  }

  // 작업 취소
  async cancel(): Promise {
    await this.client.axios.post(`/api/v1/jobs/${this.id}/cancel`);
    this.cleanup();
  }

  private cleanup() {
    if (this.ws) {
      this.ws.close();
      this.ws = undefined;
    }
    if (this.pollInterval) {
      clearInterval(this.pollInterval);
      this.pollInterval = undefined;
    }
  }
}

3. SDK 사용 예시

// 클라이언트 애플리케이션에서 사용
import { WIAArtClient } from 'wia-art-002-sdk';

const client = new WIAArtClient({
  apiKey: process.env.WIA_ART_API_KEY!
});

// 예시 1: Promise 기반 사용 (async/await)
async function generateImage() {
  try {
    const job = await client.generate({
      prompt: 'A majestic dragon flying over mountains at sunset',
      parameters: {
        width: 1024,
        height: 1024,
        steps: 50,
        cfgScale: 7.5
      }
    });

    console.log(`Job created: ${job.id}`);

    // 완료까지 대기
    const results = await job.wait();
    console.log(`Generated ${results.length} images:`);
    results.forEach(result => {
      console.log(`- ${result.imageUrl}`);
    });
  } catch (error) {
    console.error('Generation failed:', error);
  }
}

// 예시 2: 이벤트 기반 사용 (진행 상황 추적)
function generateWithProgress() {
  client.generate({
    prompt: 'Cyberpunk city at night, neon lights, rain',
    parameters: { batchSize: 4 }
  }).then(job => {
    job.stream()
      .on('progress', (status) => {
        console.log(`Progress: ${(status.progress * 100).toFixed(1)}%`);
        console.log(`Message: ${status.message}`);
      })
      .on('completed', (results) => {
        console.log('Generation completed!');
        results.forEach((result, i) => {
          console.log(`Image ${i + 1}: ${result.imageUrl}`);
        });
      })
      .on('failed', (error) => {
        console.error('Generation failed:', error);
      });
  });
}

// 예시 3: 메타데이터 조회 및 검증
async function verifyArtwork(artworkId: string) {
  // 메타데이터 가져오기
  const metadata = await client.getMetadata(artworkId);
  console.log('Artwork metadata:', JSON.stringify(metadata, null, 2));

  // 블록체인 검증
  const verification = await client.verifyArtwork(artworkId);
  if (verification.verified) {
    console.log('✓ Artwork verified on blockchain');
    console.log(`Transaction: ${verification.transactionHash}`);
  } else {
    console.log('✗ Verification failed');
  }
}

// 예시 4: 사용량 확인

  console.log(`Credits remaining: ${quota.remaining}/${quota.total}`);

인증 및 권한 관리

1. OAuth 2.0 흐름

WIA-ART-002는 OAuth 2.0 Authorization Code Flow를 지원하여 제3자 애플리케이션이 사용자를 대신하여 이미지를 생성할 수 있습니다:

단계 엔드포인트 설명
1. 인증 요청 /oauth/authorize 사용자를 로그인 페이지로 리디렉션
2. 콜백 클라이언트 redirect_uri Authorization Code 수신
3. 토큰 교환 /oauth/token Code를 Access Token으로 교환
4. 토큰 갱신 /oauth/token Refresh Token으로 새 토큰 발급

2. Rate Limiting

API는 사용자당 요청 속도 제한을 적용하며, 응답 헤더로 현재 상태를 알려줍니다: 

// 응답 헤더
X-RateLimit-Limit: 100          // 시간당 최대 요청 수
X-RateLimit-Remaining: 87       // 남은 요청 수
X-RateLimit-Reset: 1705320000   // 리셋 시간 (Unix timestamp)

// 제한 초과 시 (429 Too Many Requests)
{
  "error": {
    "code": "RATE_LIMIT_EXCEEDED",
    "message": "Rate limit exceeded. Try again in 342 seconds.",
    "retryAfter": 342
  }
}

에러 처리 및 재시도 로직

1. 에러 코드 체계

HTTP 상태 에러 코드 설명 재시도 가능
400 INVALID_REQUEST 요청 형식 오류 불가
401 UNAUTHORIZED 인증 실패 불가
402 INSUFFICIENT_CREDITS 크레딧 부족 불가
422 NSFW_DETECTED 안전 가이드라인 위반 불가
429 RATE_LIMIT_EXCEEDED 요청 제한 초과 가능
500 INTERNAL_ERROR 서버 내부 오류 가능
503 SERVICE_UNAVAILABLE 서비스 일시 중단 가능

2. 지수 백오프 재시도

// 재시도 로직 구현
async function retryWithExponentialBackoff(
  fn: () => Promise,
  maxRetries: number = 3,
  baseDelay: number = 1000
): Promise {
  let lastError: Error;

  for (let attempt = 0; attempt <= maxRetries; attempt++) {
    try {
      return await fn();
    } catch (error: any) {
      lastError = error;

      // 재시도 불가능한 에러는 즉시 throw
      const nonRetryable = [
        'INVALID_REQUEST',
        'UNAUTHORIZED',
        'INSUFFICIENT_CREDITS',
        'NSFW_DETECTED'
      ];
      if (nonRetryable.includes(error.code)) {
        throw error;
      }

      // 마지막 시도였다면 throw
      if (attempt === maxRetries) {
        throw error;
      }

      // 지수 백오프 대기
      const delay = baseDelay * Math.pow(2, attempt);
      const jitter = Math.random() * 1000;  // 랜덤 지터 추가
      await new Promise(resolve => setTimeout(resolve, delay + jitter));

      console.log(`Retry attempt ${attempt + 1}/${maxRetries} after ${delay}ms`);
    }
  }

  throw lastError!;
}

핵심 요점

  • RESTful API 설계: 명확한 리소스 기반 엔드포인트, 비동기 Job 패턴, 표준 HTTP 메서드와 상태 코드 사용으로 직관적인 API 제공.
  • TypeScript SDK: Promise 기반 현대적 API, 자동 재시도, 타입 안정성, 이벤트 기반 진행 상황 추적으로 개발자 경험 최적화.
  • WebSocket 스트리밍: 실시간 진행 상황 업데이트, 중간 미리보기, 양방향 통신으로 반응형 사용자 경험 구현.
  • 배치 처리: 대량 생성 작업을 효율적으로 관리하는 배치 API와 큐 시스템으로 리소스 최적화.
  • OAuth 2.0 인증: 표준 인증 프로토콜로 제3자 통합 지원, 세밀한 권한 관리, 토큰 갱신 메커니즘.
  • Rate Limiting: 공정한 리소스 분배, 남용 방지, 명확한 제한 정보 제공으로 서비스 안정성 보장.
  • 에러 처리: 체계적인 에러 코드, 지수 백오프 재시도, 재시도 가능 여부 판단으로 견고한 통합 가능.
