제3장: 데이터 형식과 메타데이터

弘益人間 · 널리 인간을 이롭게 하라

개요

AI 생성 예술 작품은 단순한 픽셀 배열 이상입니다. 각 작품 뒤에는 생성 과정, 창작자 정보, 사용된 모델, 프롬프트, 파라미터 등 풍부한 메타데이터가 있습니다. 이러한 정보를 체계적으로 기록하고 전달하는 것은 투명성, 출처 추적, 저작권 관리, 그리고 생태계 전반의 상호운용성을 위해 필수적입니다.

WIA-ART-002 표준은 AI 생성 예술을 위한 포괄적인 데이터 형식과 메타데이터 스키마를 정의합니다. 이 장에서는 이미지 파일 형식, 메타데이터 구조, 블록체인 통합을 위한 출처 기록, 그리고 다양한 플랫폼 간의 데이터 교환 방법을 다룹니다.

이미지 파일 형식

1. 지원되는 파일 형식

WIA-ART-002는 다양한 용도와 요구사항을 충족하기 위해 여러 이미지 파일 형식을 지원합니다:

형식 특징 용도 메타데이터 지원
PNG 무손실, 투명도 지원 최종 결과물, 보관용 EXIF, XMP, tEXt 청크
JPEG/JPG 손실 압축, 작은 파일 크기 웹 공유, 미리보기 EXIF, XMP, IPTC
WebP 효율적 압축, 투명도 웹 최적화 EXIF, XMP
AVIF 차세대 압축, HDR 고품질 웹 배포 EXIF, XMP
TIFF 무손실, 다층 아카이브, 인쇄 EXIF, IPTC, XMP

2. PNG 파일의 메타데이터 임베딩

PNG는 WIA-ART-002에서 권장하는 주요 형식으로, tEXt 청크를 통해 커스텀 메타데이터를 저장할 수 있습니다: 

// TypeScript: PNG 메타데이터 임베딩
import { PNG } from 'pngjs';
import { WIAArtMetadata } from 'wia-art-002';

function embedMetadata(imageBuffer: Buffer, metadata: WIAArtMetadata): Buffer {
  const png = PNG.sync.read(imageBuffer);

  // WIA-ART-002 메타데이터를 tEXt 청크로 저장
  png.text = {
    'wia-art-002:version': '1.0',
    'wia-art-002:metadata': JSON.stringify(metadata),
    'wia-art-002:signature': signMetadata(metadata)
  };

  return PNG.sync.write(png);
}

3. 생성 파라미터 (Generation Parameters)

재현성(Reproducibility)을 위해 모든 생성 파라미터를 상세히 기록합니다:

파라미터 타입 설명 필수
prompt string 정 프롬프트 (최대 10,000자) 필수
negativePrompt string 부정 프롬프트 선택
seed integer 난수 생성 시드 (재현성) 필수
steps integer 샘플링 단계 수 (1-200) 필수
cfgScale float 분류자 없는 가이던스 스케일 (1-30) 필수
sampler string 샘플러 알고리즘명 필수
size object 이미지 크기 {width, height} 필수
controlNet array ControlNet 설정 선택
lora array LoRA 모델 및 가중치 선택
initImage string img2img 초기 이미지 해시 선택
denoisingStrength float img2img 디노이징 강도 (0-1) 선택

메타데이터 검증 및 서명

1. 디지털 서명

메타데이터의 무결성을 보장하기 위해 ECDSA 디지털 서명을 사용합니다: 

// TypeScript: 메타데이터 서명 및 검증
import { ethers } from 'ethers';

class MetadataSignature {
  // 메타데이터 서명
  static async sign(metadata: object, privateKey: string): Promise {
    const wallet = new ethers.Wallet(privateKey);

    // 메타데이터를 정규화된 JSON으로 변환
    const normalized = JSON.stringify(metadata, Object.keys(metadata).sort());

    // SHA-256 해시
    const hash = ethers.utils.keccak256(ethers.utils.toUtf8Bytes(normalized));

    // 서명
    const signature = await wallet.signMessage(ethers.utils.arrayify(hash));

    return signature;
  }

  // 서명 검증
  static verify(metadata: object, signature: string, address: string): boolean {
    // 메타데이터를 정규화된 JSON으로 변환
    const normalized = JSON.stringify(metadata, Object.keys(metadata).sort());

    // SHA-256 해시
    const hash = ethers.utils.keccak256(ethers.utils.toUtf8Bytes(normalized));

    // 서명으로부터 주소 복구
    const recoveredAddress = ethers.utils.verifyMessage(
      ethers.utils.arrayify(hash),
      signature
    );

    // 주소 비교
    return recoveredAddress.toLowerCase() === address.toLowerCase();
  }
}

// 사용 예시
const metadata = { /* WIA-ART-002 메타데이터 */ };
const privateKey = '0x...';  // 창작자의 개인키

const signature = await MetadataSignature.sign(metadata, privateKey);

// 메타데이터에 서명 추가
metadata.signature = {
  value: signature,
  signer: '0x742d35Cc6634C0532925a3b844Bc9e7595f0bEb',
  timestamp: new Date().toISOString()
};

2. 메타데이터 스키마 검증

JSON Schema를 사용하여 메타데이터의 구조적 유효성을 검증합니다: 

// JSON Schema for WIA-ART-002 Metadata
const wiaArt002Schema = {
  "$schema": "http://json-schema.org/draft-07/schema#",
  "title": "WIA-ART-002 Metadata Schema",
  "type": "object",
  "required": ["standard", "version", "artwork", "creator"],
  "properties": {
    "standard": {
      "type": "string",
      "const": "WIA-ART-002"
    },
    "version": {
      "type": "string",
      "pattern": "^\\d+\\.\\d+$"
    },
    "artwork": {
      "type": "object",
      "required": ["id", "title", "created"],
      "properties": {
        "id": { "type": "string" },
        "title": { "type": "string" },
        "created": { "type": "string", "format": "date-time" }
      }
    },
    "creator": {
      "type": "object",
      "required": ["type"],
      "properties": {
        "type": {
          "type": "string",
          "enum": ["human", "ai", "human-ai-collaboration"]
        }
      }
    }
  }
}

// 검증 함수
import Ajv from 'ajv';

function validateMetadata(metadata: any): { valid: boolean; errors?: string[] } {
  const ajv = new Ajv();
  const validate = ajv.compile(wiaArt002Schema);
  const valid = validate(metadata);

  return {
    valid,
    errors: valid ? undefined : validate.errors?.map(e => e.message)
  };
}

핵심 요점

  • 다양한 파일 형식 지원: PNG (권장), JPEG, WebP, AVIF, TIFF 등 용도에 맞는 형식 선택 가능. PNG의 tEXt 청크를 통한 메타데이터 임베딩이 표준.
  • 포괄적 메타데이터 스키마: 작품 정보, 창작자, 생성 파라미터, 출처, 권리 등을 계층적 JSON 구조로 체계화. 재현성과 투명성 보장.
  • 블록체인 출처 추적: Ethereum, Polygon 등 블록체인을 통한 변조 불가능한 소유권 이력 기록. IPFS와 결합하여 분산 저장.
  • 유연한 라이선싱: CC0, CC-BY, Custom 등 다양한 라이선스 지원. 상업적 사용, 수정, 재배포 권리를 명확히 정의.
  • 디지털 서명 검증: ECDSA 서명을 통한 메타데이터 무결성 보장. JSON Schema로 구조적 유효성 검증.
  • 상호운용성: 표준화된 형식으로 다양한 플랫폼 간 원활한 데이터 교환. OpenSea, Rarible 등 NFT 마켓플레이스와 호환.

복습 질문

  1. PNG, JPEG, WebP, AVIF 각 파일 형식의 특징과 적합한 사용 사례를 비교 설명하십시오. WIA-ART-002가 PNG를 권장 형식으로 선택한 이유는 무엇입니까?
  2. WIA-ART-002 메타데이터 스키마의 주요 섹션 5가지를 나열하고 각각의 목적을 설명하십시오. 왜 계층적 JSON 구조가 사용되었습니까?
  3. AI 생성 예술에서 "창작자" 개념이 복잡한 이유를 설명하고, WIA-ART-002가 이를 어떻게 해결하는지 논의하십시오. human-ai-collaboration 타입의 구조를 상세히 기술하십시오.
  4. 생성 파라미터(prompt, seed, steps, cfgScale 등)를 상세히 기록하는 이유는 무엇입니까? 재현성(reproducibility)의 중요성과 한계를 논의하십시오.
