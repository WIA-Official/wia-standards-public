1. Authenticated Encryption (AEAD)

WIA-DEF-017 mandates AEAD ciphers (AES-256-GCM or ChaCha20-Poly1305) for tactical traffic. Plaintext is encrypted with a session key; integrity is bound to associated data such as the message header.

2. Key Derivation (HKDF)

Tactical sessions derive ephemeral keys from a long-term master and a per-mission salt. WIA-DEF-017 requires HKDF-SHA-256 or HKDF-SHA-384 with explicit purpose labels.

— run derivation —

Demonstration only — production deployments MUST use vetted crypto libraries (libsodium, BoringSSL, etc.).

3. Hybrid Post-Quantum Handshake

WIA-DEF-017 specifies a hybrid handshake combining classical ECDH (X25519) with a PQ KEM (ML-KEM-768) so confidentiality survives a future cryptographically relevant quantum computer.

1) initiator -> responder: ephem_x25519_pub, mlkem_pub 2) responder -> initiator: ct_mlkem, ephem_x25519_pub 3) both: ss_x25519 = X25519(my_priv, peer_pub) 4) both: ss_mlkem = ML-KEM-Decap(my_priv, ct_mlkem) 5) both: master = HKDF-Extract(ss_x25519 || ss_mlkem) 6) both: session = HKDF-Expand(master, "wia-def-017/session/v1", 64)

No key material is exchanged in the clear. A future quantum adversary capturing today's traffic still cannot recover the session key without breaking either ECDH or ML-KEM.

4. Tamper-Evident Audit Trail

Every key operation MUST be appended to an audit log whose entries chain via SHA-256. Removing or altering an entry breaks the chain and is detectable on replay.
