2. 코드 품질 및 아키텍처

2.1 SOLID 원칙 적용

객체지향 설계의 5대 원칙인 SOLID를 준수하면 유지보수가 쉽고 확장 가능한 코드를 작성할 수 있습니다:

원칙 설명 적용 예시
S - 단일 책임 클래스는 하나의 책임만 가져야 함 FacilityService는 시설 관리만, EnvironmentService는 환경 모니터링만 담당
O - 개방-폐쇄 확장에는 열려있고 수정에는 닫혀있어야 함 DatabaseAdapter 인터페이스로 다양한 DB 지원
L - 리스코프 치환 하위 타입은 상위 타입을 대체 가능해야 함 모든 DB 어댑터는 DatabaseAdapter를 완전히 구현
I - 인터페이스 분리 클라이언트는 사용하지 않는 인터페이스에 의존하지 않아야 함 ReadOnlyRepository와 WriteRepository 분리
D - 의존성 역전 구체적인 것이 아닌 추상적인 것에 의존해야 함 서비스는 구체 클래스가 아닌 인터페이스에 의존

✅ 좋은 예: 인터페이스 기반 설계

// 추상화된 인터페이스
interface DataRepository<T> {
  findById(id: string): Promise<T | null>;
  findAll(filter?: Filter): Promise<T[]>;
  create(data: Partial<T>): Promise<T>;
  update(id: string, data: Partial<T>): Promise<T>;
  delete(id: string): Promise<void>;
}

// 구체적인 구현
class FacilityRepository implements DataRepository<Facility> {
  constructor(
    private db: DatabaseAdapter,
    private cache: CacheAdapter
  ) {}

  async findById(id: string): Promise<Facility | null> {
    // 캐시 우선 조회
    const cached = await this.cache.get<Facility>(`facility:${id}`);
    if (cached) return cached;

    // DB 조회 및 캐싱
    const result = await this.db.query(
      'SELECT * FROM facilities WHERE id = $1',
      [id]
    );

    if (result.rows.length === 0) return null;

    const facility = this.mapToEntity(result.rows[0]);
    await this.cache.set(`facility:${id}`, facility, 3600);

    return facility;
  }

  // ... 기타 메서드 구현
}

// 서비스는 인터페이스에만 의존
class FacilityService {
  constructor(private repository: DataRepository<Facility>) {}

  async getFacility(id: string): Promise<Facility | null> {
    return this.repository.findById(id);
  }
}

2.2 에러 처리 전략

체계적인 에러 처리는 시스템 안정성의 핵심입니다:

✅ 좋은 예: 계층별 에러 처리

// 커스텀 에러 클래스 정의
export class ApplicationError extends Error {
  constructor(
    message: string,
    public code: string,
    public statusCode: number = 500,
    public details?: any
  ) {
    super(message);
    this.name = this.constructor.name;
    Error.captureStackTrace(this, this.constructor);
  }
}

export class NotFoundError extends ApplicationError {
  constructor(resource: string, id: string) {
    super(
      `${resource} with id ${id} not found`,
      'NOT_FOUND',
      404
    );
  }
}

export class ValidationError extends ApplicationError {
  constructor(message: string, details?: any) {
    super(message, 'VALIDATION_ERROR', 400, details);
  }
}

// 서비스 레벨에서 의미있는 에러 발생
class FacilityService {
  async getFacility(id: string): Promise<Facility> {
    // 입력 검증
    if (!id || !this.isValidUUID(id)) {
      throw new ValidationError('Invalid facility ID format', { id });
    }

    const facility = await this.repository.findById(id);

    if (!facility) {
      throw new NotFoundError('Facility', id);
    }

    return facility;
  }

  async createFacility(data: CreateFacilityDTO): Promise<Facility> {
    // 데이터 검증
    const validationErrors = await this.validateFacilityData(data);
    if (validationErrors.length > 0) {
      throw new ValidationError(
        'Facility validation failed',
        { errors: validationErrors }
      );
    }

    try {
      return await this.repository.create(data);
    } catch (error) {
      // DB 제약조건 위반 등을 의미있는 에러로 변환
      if (error.code === '23505') { // Unique violation
        throw new ValidationError(
          'Facility with this name already exists',
          { field: 'name', value: data.name }
        );
      }
      throw error;
    }
  }
}

// API 레벨에서 에러를 HTTP 응답으로 변환
app.use((error: Error, req: Request, res: Response, next: NextFunction) => {
  logger.error('Request error:', {
    error: error.message,
    stack: error.stack,
    path: req.path,
    method: req.method
  });

  if (error instanceof ApplicationError) {
    return res.status(error.statusCode).json({
      error: {
        code: error.code,
        message: error.message,
        details: error.details
      }
    });
  }

  // 예상하지 못한 에러
  res.status(500).json({
    error: {
      code: 'INTERNAL_ERROR',
      message: 'An unexpected error occurred'
    }
  });
});

3. 데이터베이스 모범 사례

3.1 효율적인 쿼리 작성

항목 모범 사례 안티 패턴
인덱스 활용 WHERE, JOIN, ORDER BY 절에 사용되는 컬럼에 인덱스 생성 인덱스 없이 Full Table Scan
N+1 문제 JOIN이나 배치 조회로 한 번에 가져오기 루프 안에서 개별 쿼리 실행
SELECT 절 필요한 컬럼만 명시적으로 선택 SELECT * 사용
트랜잭션 짧고 명확한 트랜잭션 범위 불필요하게 긴 트랜잭션
페이징 LIMIT/OFFSET 또는 커서 기반 페이징 모든 데이터를 한 번에 조회

✅ 좋은 예: 최적화된 쿼리

// 효율적인 페이징 및 필터링
async function getFacilitiesWithEnvironmentData(
  page: number = 1,
  pageSize: number = 20,
  filters?: FacilityFilter
): Promise<PaginatedResult<FacilityWithEnv>> {
  // 1. 필요한 컬럼만 선택
  // 2. JOIN으로 관련 데이터 한 번에 조회 (N+1 문제 해결)
  // 3. 인덱스 활용을 위한 WHERE 절
  // 4. 페이징 처리
  const offset = (page - 1) * pageSize;

  const query = `
    SELECT
      f.id,
      f.name,
      f.depth,
      f.capacity,
      f.status,
      COUNT(DISTINCT h.id) as harvest_count,
      AVG(e.temperature) as avg_temperature,
      AVG(e.dissolved_oxygen) as avg_oxygen
    FROM facilities f
    LEFT JOIN harvests h ON f.id = h.facility_id
    LEFT JOIN LATERAL (
      SELECT temperature, dissolved_oxygen
      FROM environment_data
      WHERE facility_id = f.id
      ORDER BY timestamp DESC
      LIMIT 100
    ) e ON true
    WHERE f.status = $1
      AND ($2::int IS NULL OR f.depth >= $2)
      AND ($3::int IS NULL OR f.depth <= $3)
    GROUP BY f.id, f.name, f.depth, f.capacity, f.status
    ORDER BY f.created_at DESC
    LIMIT $4 OFFSET $5
  `;

  const [dataResult, countResult] = await Promise.all([
    db.query(query, [
      filters?.status || 'active',
      filters?.minDepth,
      filters?.maxDepth,
      pageSize,
      offset
    ]),
    db.query(
      'SELECT COUNT(*) FROM facilities WHERE status = $1',
      [filters?.status || 'active']
    )
  ]);

  return {
    data: dataResult.rows,
    total: parseInt(countResult.rows[0].count),
    page,
    pageSize,
    totalPages: Math.ceil(parseInt(countResult.rows[0].count) / pageSize)
  };
}

// 인덱스 생성 (마이그레이션 스크립트)
CREATE INDEX idx_facilities_status ON facilities(status);
CREATE INDEX idx_facilities_depth ON facilities(depth);
CREATE INDEX idx_facilities_created_at ON facilities(created_at DESC);
CREATE INDEX idx_environment_facility_timestamp ON environment_data(facility_id, timestamp DESC);
CREATE INDEX idx_harvests_facility ON harvests(facility_id);

async function completeHarvest(
  harvestId: string,
  actualData: HarvestCompletionData
): Promise<Harvest> {
  const client = await db.getClient();

  try {
    await client.query('BEGIN');

    // 1. 수확 계획 업데이트
    const harvestResult = await client.query(`
      UPDATE harvests
      SET status = 'completed',
          actual_weight = $1,
          actual_date = $2,
          quality_grade = $3,
          updated_at = NOW()
      WHERE id = $4 AND status = 'in_progress'
      RETURNING *
    `, [
      actualData.weight,
      actualData.date,
      actualData.quality,
      harvestId
    ]);

    if (harvestResult.rows.length === 0) {
      throw new Error('Harvest not found or already completed');
    }

    const harvest = harvestResult.rows[0];

    // 2. 시설 용량 업데이트
    await client.query(`
      UPDATE facilities
      SET capacity = capacity + $1,
          updated_at = NOW()
      WHERE id = $2
    `, [actualData.weight, harvest.facility_id]);

    // 3. 재고 기록 생성
    await client.query(`
      INSERT INTO inventory (
        facility_id, species, quantity, quality_grade, harvest_date
      ) VALUES ($1, $2, $3, $4, $5)
    `, [
      harvest.facility_id,
      harvest.species,
      actualData.weight,
      actualData.quality,
      actualData.date
    ]);

    // 4. 감사 로그 기록
    await client.query(`
      INSERT INTO audit_logs (
        entity_type, entity_id, action, user_id, details
      ) VALUES ($1, $2, $3, $4, $5)
    `, [
      'harvest',
      harvestId,
      'completed',
      actualData.userId,
      JSON.stringify(actualData)
    ]);

    await client.query('COMMIT');

    logger.info(`Harvest ${harvestId} completed successfully`);
    return harvest;

  } catch (error) {
    await client.query('ROLLBACK');
    logger.error(`Failed to complete harvest ${harvestId}:`, error);
    throw error;
  } finally {
    client.release();
  }
}

// 비밀번호 해싱 (저장 시)
async function hashPassword(password: string): Promise<string> {
  const saltRounds = 12; // 충분한 복잡도
  return bcrypt.hash(password, saltRounds);
}

// 비밀번호 검증 (로그인 시)
async function verifyPassword(
  password: string,
  hashedPassword: string
): Promise<boolean> {
  return bcrypt.compare(password, hashedPassword);
}

// JWT 토큰 생성
function generateToken(user: User): string {
  const payload = {
    userId: user.id,
    email: user.email,
    roles: user.roles
  };

  return jwt.sign(payload, process.env.JWT_SECRET!, {
    expiresIn: '1h',
    issuer: 'wia-aquaculture',
    audience: 'wia-api'
  });
}

// JWT 검증 미들웨어
async function authenticateToken(
  req: Request,
  res: Response,
  next: NextFunction
) {
  const authHeader = req.headers['authorization'];
  const token = authHeader && authHeader.split(' ')[1];

  if (!token) {
    return res.status(401).json({ error: 'Access token required' });
  }

  try {
    const payload = jwt.verify(token, process.env.JWT_SECRET!) as JWTPayload;
    req.user = await getUserById(payload.userId);
    next();
  } catch (error) {
    return res.status(403).json({ error: 'Invalid or expired token' });
  }
}

// 역할 기반 인가 미들웨어
function requireRole(...roles: string[]) {
  return (req: Request, res: Response, next: NextFunction) => {
    if (!req.user) {
      return res.status(401).json({ error: 'Authentication required' });
    }

    const hasRole = roles.some(role => req.user.roles.includes(role));
    if (!hasRole) {
      return res.status(403).json({
        error: 'Insufficient permissions',
        required: roles,
        current: req.user.roles
      });
    }

    next();
  };
}

const CreateFacilitySchema = z.object({

    .regex(/^[a-zA-Z0-9\s-]+$/, 'Name contains invalid characters'),

    longitude: z.number().min(-180).max(180)

    .int('Depth must be an integer')
    .min(10, 'Minimum depth is 10 meters')
    .max(1000, 'Maximum depth is 1000 meters'),

  capacity: z.number()
    .positive('Capacity must be positive')
    .max(100000, 'Capacity exceeds maximum'),

  species: z.array(z.string())
    .min(1, 'At least one species required')
    .max(10, 'Too many species'),

  email: z.string().email('Invalid email format').optional()
});

// 검증 미들웨어
function validateRequest(schema: z.ZodSchema) {
  return (req: Request, res: Response, next: NextFunction) => {
    try {
      req.body = schema.parse(req.body);
      next();
    } catch (error) {
      if (error instanceof z.ZodError) {
        return res.status(400).json({
          error: 'Validation failed',
          details: error.errors.map(e => ({
            field: e.path.join('.'),
            message: e.message
          }))
        });
      }
      next(error);
    }
  };
}

5. 성능 최적화

5.1 캐싱 전략

✅ 좋은 예: 다층 캐싱과 캐시 무효화

class FacilityService {
  private localCache: LRUCache;
  private redisCache: Redis;

  async getFacility(id: string): Promise<Facility> {
    const cacheKey = `facility:${id}`;

    // L1: 로컬 캐시 (가장 빠름)
    let facility = this.localCache.get<Facility>(cacheKey);
    if (facility) {
      return facility;
    }

    // L2: Redis 캐시
    const cached = await this.redisCache.get(cacheKey);
    if (cached) {
      facility = JSON.parse(cached);
      this.localCache.set(cacheKey, facility);
      return facility;
    }

    // L3: 데이터베이스
    facility = await this.repository.findById(id);
    if (!facility) {
      throw new NotFoundError('Facility', id);
    }

    // 캐시에 저장 (비동기, 블로킹 안함)
    this.cacheInBackground(cacheKey, facility);

    return facility;
  }

  async updateFacility(id: string, data: Partial<Facility>): Promise<Facility> {
    const facility = await this.repository.update(id, data);

    // 캐시 무효화 전략
    await this.invalidateCache(id);

    // 관련 캐시도 무효화
    await this.invalidateRelatedCaches(facility);

    return facility;
  }

  private async invalidateCache(id: string): Promise<void> {
    const cacheKey = `facility:${id}`;

    // L1 캐시 무효화
    this.localCache.delete(cacheKey);

    // L2 캐시 무효화
    await this.redisCache.del(cacheKey);
  }

  private async invalidateRelatedCaches(facility: Facility): Promise<void> {
    // 목록 캐시 무효화
    await this.redisCache.del('facilities:list:*');

    // 통계 캐시 무효화
    await this.redisCache.del(`facility:${facility.id}:stats`);

    // 환경 데이터 요약 캐시 무효화
    await this.redisCache.del(`facility:${facility.id}:env:summary`);
  }

  private async cacheInBackground(key: string, data: any): Promise<void> {
    // 블로킹하지 않고 백그라운드에서 캐싱
    setImmediate(async () => {
      try {
        this.localCache.set(key, data);
        await this.redisCache.setex(
          key,
          3600, // 1시간
          JSON.stringify(data)
        );
      } catch (error) {
        logger.warn('Failed to cache data:', error);
      }
    });
  }
}

  }, {
    repeat: {
      cron: '0 * * * *' // 매시간
    }
  });

  logger.info('Scheduled recurring environment analysis');
}

6. 모니터링 및 로깅

6.1 구조화된 로깅

로그 레벨 용도 예시
ERROR 즉시 조치 필요한 오류 DB 연결 실패, API 호출 실패
WARN 잠재적 문제 캐시 미스율 증가, 느린 쿼리
INFO 중요 이벤트 시설 생성, 수확 완료
DEBUG 디버깅 정보 쿼리 실행 시간, 캐시 히트
TRACE 상세한 실행 흐름 함수 진입/종료, 변수 값

✅ 좋은 예: Winston을 활용한 구조화 로깅

import winston from 'winston';

// 로거 설정
const logger = winston.createLogger({
  level: process.env.LOG_LEVEL || 'info',
  format: winston.format.combine(
    winston.format.timestamp(),
    winston.format.errors({ stack: true }),
    winston.format.json()
  ),
  defaultMeta: {
    service: 'wia-aquaculture',
    version: '1.0.0'
  },
  transports: [
    // 콘솔 출력 (개발 환경)
    new winston.transports.Console({
      format: winston.format.combine(
        winston.format.colorize(),
        winston.format.simple()
      )
    }),
    // 파일 출력 (프로덕션)
    new winston.transports.File({
      filename: 'logs/error.log',
      level: 'error',
      maxsize: 10485760, // 10MB
      maxFiles: 5
    }),
    new winston.transports.File({
      filename: 'logs/combined.log',
      maxsize: 10485760,
      maxFiles: 10
    })
  ]
});

// 컨텍스트 있는 로깅
logger.info('Facility created', {
  facilityId: 'facility-123',
  userId: 'user-456',
  metadata: {
    name: 'Pacific Farm',
    depth: 200,
    capacity: 1000
  }
});

// 에러 로깅
try {
  await riskyOperation();
} catch (error) {
  logger.error('Operation failed', {
    error: {
      message: error.message,
      stack: error.stack,
      code: error.code
    },
    context: {
      operation: 'riskyOperation',
      params: { /* ... */ }
    }
  });
}

// 성능 측정
const startTime = Date.now();
await slowOperation();
const duration = Date.now() - startTime;

logger.debug('Operation completed', {
  operation: 'slowOperation',
  duration,
  threshold: 1000,
  slow: duration > 1000
});

🎯 핵심 요점

  • SOLID 원칙: 객체지향 설계 원칙을 따르면 유지보수가 쉽고 확장 가능한 코드를 작성할 수 있습니다.
  • 체계적 에러 처리: 계층별로 의미있는 에러를 정의하고 처리하면 디버깅이 쉽고 사용자 경험이 향상됩니다.
  • 쿼리 최적화: 인덱스 활용, N+1 문제 해결, 페이징 처리로 데이터베이스 성능을 크게 개선할 수 있습니다.
  • 보안 우선: 입력 검증, 인증/인가, 암호화를 통해 시스템 보안을 강화합니다.
  • 다층 캐싱: 로컬 캐시와 Redis를 조합하여 최적의 성능을 달성합니다.
  • 비동기 처리: 무거운 작업은 큐를 통해 백그라운드에서 처리하여 응답 속도를 개선합니다.
  • 구조화된 로깅: JSON 형식의 구조화된 로그로 문제 추적과 분석이 용이합니다.
  • 弘益人間: 모범 사례를 따라 안정적이고 효율적인 시스템을 구축하여 심해 양식 산업 발전에 기여합니다.
