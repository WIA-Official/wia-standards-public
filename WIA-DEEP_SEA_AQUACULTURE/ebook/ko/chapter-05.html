시스템 통합 (Phase 4)

1. 시스템 통합 개요

Phase 4에서는 WIA-DEEP_SEA_AQUACULTURE를 다른 WIA 표준들과 통합하여 완전한 엔터프라이즈 솔루션을 구축합니다. 개별 표준들은 각자의 영역에서 강력하지만, 진정한 가치는 통합을 통해 실현됩니다.

📌 Phase 4의 중요성

시스템 통합은 다음을 가능하게 합니다:

  • 자연어 제어: "제주 양식장 수온 보여줘" (WIA-INTENT)
  • 통합 API: 단일 엔드포인트로 모든 서비스 접근 (WIA-OMNI-API)
  • 중앙 인증: SSO로 모든 시스템 접근 (WIA-AUTH)
  • 감사 추적: 모든 작업 로깅 및 규정 준수 (WIA-AUDIT)

3. WIA-INTENT 통합

3.1 자연어 쿼리

WIA-INTENT를 통해 사용자는 자연어로 심해 양식 데이터를 조회하고 제어할 수 있습니다.

// WIA-INTENT 통합 예제 import { IntentClient } from '@wia/intent-sdk'; import { AquacultureClient } from '@wia/deep-sea-aquaculture-sdk'; const intentClient = new IntentClient({ apiKey: process.env.WIA_INTENT_API_KEY }); const aquacultureClient = new AquacultureClient({ apiKey: process.env.WIA_AQUACULTURE_API_KEY }); // 자연어 쿼리 처리 async function handleNaturalLanguageQuery(userInput: string) { // 1. Intent 분석 const intent = await intentClient.analyze(userInput); console.log('Detected intent:', intent); // { // action: 'query_temperature', // entities: { // facility: 'jeju-farm-001', // timeframe: 'last_7_days' // }, // confidence: 0.95 // } // 2. Intent에 따라 실행 if (intent.action === 'query_temperature') { const data = await aquacultureClient.records.list({ category: 'sensor_data', filters: { facility_id: intent.entities.facility, timestamp_gte: getTimeframeStart(intent.entities.timeframe) }, sort: '-timestamp', limit: 100 }); return formatResponse(data); } } // 사용 예제 const response = await handleNaturalLanguageQuery( "제주 양식장 최근 일주일 수온 데이터 보여줘" ); console.log(response);

5. WIA-AUTH 통합

5.1 OAuth 2.0 / OpenID Connect

WIA-AUTH는 표준 OAuth 2.0 및 OpenID Connect 프로토콜을 사용하여 통합 인증을 제공합니다.

// OAuth 2.0 Authorization Code Flow // 1. 인증 요청 const authUrl = `https://auth.wia.org/oauth/authorize?` + `client_id=${CLIENT_ID}&` + `redirect_uri=${REDIRECT_URI}&` + `response_type=code&` + `scope=aquaculture.read aquaculture.write&` + `state=${STATE}`; window.location.href = authUrl; // 2. 콜백 처리 (리다이렉트 후) const code = new URLSearchParams(window.location.search).get('code'); // 3. Access Token 획득 const response = await fetch('https://auth.wia.org/oauth/token', { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify({ grant_type: 'authorization_code', code: code, client_id: CLIENT_ID, client_secret: CLIENT_SECRET, redirect_uri: REDIRECT_URI }) }); const { access_token, refresh_token, id_token } = await response.json(); // 4. API 호출 시 사용 const data = await fetch('https://api.wia.org/v1/aquaculture/records', { headers: { 'Authorization': `Bearer ${access_token}` } });

5.2 SSO (Single Sign-On)

한 번의 로그인으로 모든 WIA 서비스에 접근할 수 있습니다.

✅ SSO 이점

  • 사용자 편의성: 한 번만 로그인
  • 보안 강화: 중앙 집중식 인증 정책
  • 관리 효율: 사용자 계정 통합 관리
  • 감사 추적: 모든 로그인 이력 중앙 저장

6. WIA-AUDIT 통합

6.1 감사 로깅

모든 중요한 작업은 WIA-AUDIT에 자동으로 기록됩니다.

// 감사 로그 자동 기록 import { AuditClient } from '@wia/audit-sdk'; const audit = new AuditClient({ apiKey: process.env.WIA_AUDIT_API_KEY }); // 데코레이터를 사용한 자동 감사 로깅 function Auditable(action: string) { return function (target: any, propertyKey: string, descriptor: PropertyDescriptor) { const originalMethod = descriptor.value; descriptor.value = async function (...args: any[]) { const startTime = Date.now(); let result, error; try { result = await originalMethod.apply(this, args); return result; } catch (e) { error = e; throw e; } finally { // 감사 로그 기록 await audit.log({ action: action, resource: 'aquaculture_record', user_id: this.userId, timestamp: new Date().toISOString(), duration: Date.now() - startTime, success: !error, details: { args: args, result: result, error: error?.message } }); } }; return descriptor; }; } class AquacultureService { @Auditable('create_record') async createRecord(data: any) { // 레코드 생성 로직 return await this.client.records.create(data); } @Auditable('delete_record') async deleteRecord(id: string) { // 레코드 삭제 로직 return await this.client.records.delete(id); } }

8. 한국 통합 구축 사례

8.1 국립수산과학원 통합 플랫폼

국립수산과학원은 WIA 표준 5개를 통합하여 국가 단위 양식 모니터링 시스템을 구축했습니다.

✅ 국립수산과학원 통합 시스템

  • 규모: 전국 1,200개 양식장 통합 관리
  • 사용자: 양식업자 5,000명, 연구원 300명
  • 통합 표준: DEEP_SEA_AQUACULTURE, INTENT, OMNI-API, AUTH, AUDIT
  • 데이터량: 일 평균 500GB (센서 데이터 200만 건)
  • 가용성: 99.95% 업타임 (연간 4.4시간 다운타임)
  • 비용 절감: 통합 전 대비 운영비 42% 감소

8.2 제주대 해양생명과학과 연구 플랫폼

제주대학교는 WIA 표준을 활용하여 해양 연구 데이터 플랫폼을 구축했습니다. 제주 해역의 20개 연구 거점에서 수집된 데이터를 통합 분석합니다.

컴포넌트 기술 스택 용도

8.3 해양수산부 스마트 양식 클러스터

해양수산부는 WIA 표준 기반 스마트 양식 클러스터를 전국 5개 지역에 구축했습니다. 삼성중공업 해양플랜트가 시스템 통합을 담당했습니다.

10. 요약 (Summary)

Chapter 05에서는 WIA-DEEP_SEA_AQUACULTURE를 다른 WIA 표준들과 통합하는 방법을 학습했습니다. 시스템 통합은 개별 컴포넌트를 하나의 강력한 솔루션으로 만드는 과정입니다.

WIA-INTENT는 자연어 인터페이스를, WIA-OMNI-API는 통합 API 게이트웨이를, WIA-AUTH는 중앙 인증을, WIA-AUDIT는 감사 로깅을 제공합니다. 이들은 마이크로서비스 아키텍처로 구현되어 확장성과 유연성을 보장합니다.

국립수산과학원, 제주대, 해양수산부의 사례는 WIA 표준 통합의 실제 가치를 입증합니다. Kubernetes 기반 배포와 Prometheus/Grafana 모니터링은 안정적인 운영을 보장합니다.
