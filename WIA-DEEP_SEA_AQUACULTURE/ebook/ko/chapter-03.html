API 인터페이스 (Phase 2)

1. API 인터페이스 개요

Phase 2에서는 심해 양식 시스템을 위한 완전한 RESTful API 인터페이스를 구현합니다. 이 API는 Phase 1에서 정의한 데이터 포맷을 기반으로 하며, 다양한 클라이언트 애플리케이션이 심해 양식 데이터를 생성, 조회, 수정, 삭제할 수 있도록 지원합니다.

📌 Phase 2의 중요성

API는 시스템과 외부 세계를 연결하는 창구입니다. 잘 설계된 API는 개발자 경험을 향상시키고, 시스템 통합을 쉽게 만들며, 장기적인 유지보수 비용을 절감합니다.

  • 표준화된 엔드포인트: RESTful 원칙 준수
  • 타입 안전성: TypeScript SDK 제공
  • 보안: JWT 기반 인증 및 권한 관리
  • 확장성: 버전 관리 및 페이지네이션

2. RESTful API 설계 원칙

2.1 리소스 중심 설계

모든 API 엔드포인트는 리소스를 중심으로 설계됩니다. 심해 양식 시스템에서 주요 리소스는 다음과 같습니다:

리소스 경로 설명 주요 작업
Records /api/v1/records 양식 데이터 레코드 CRUD, 검색, 필터링
Sensors /api/v1/sensors 센서 장비 정보 등록, 조회, 상태 모니터링
Facilities /api/v1/facilities 양식 시설 정보 시설 관리, 위치 추적
Species /api/v1/species 양식 어종 정보 어종 등록, 생육 데이터
Analytics /api/v1/analytics 분석 및 통계 집계, 리포트, 예측

2.2 HTTP 메서드 매핑

HTTP 메서드 작업 예시 경로 멱등성
GET 조회 (Read) GET /api/v1/records/{id} Yes
POST 생성 (Create) POST /api/v1/records No
PUT 전체 수정 (Update) PUT /api/v1/records/{id} Yes
PATCH 부분 수정 (Partial Update) PATCH /api/v1/records/{id} No
DELETE 삭제 (Delete) DELETE /api/v1/records/{id} Yes

4. TypeScript SDK

4.1 SDK 설치 및 초기화

# NPM 설치 npm install @wia/deep-sea-aquaculture-sdk # TypeScript 사용 예제 import { createClient, RecordCategory } from '@wia/deep-sea-aquaculture-sdk'; const client = createClient({ apiKey: process.env.WIA_API_KEY, baseUrl: 'https://api.wia.org/v1', timeout: 30000, retries: 3 }); // 레코드 생성 const record = await client.records.create({ category: RecordCategory.SENSOR_DATA, value: { temperature: 4.2, depth: 1200, salinity: 35.5 }, metadata: { sensor_id: 'sensor-node-42', facility_id: 'jeju-farm-001' } }); console.log('Created record:', record.id); // 레코드 조회 const fetched = await client.records.get(record.id); console.log('Fetched record:', fetched); // 레코드 목록 조회 (페이지네이션) const list = await client.records.list({ page: 1, limit: 20, category: RecordCategory.SENSOR_DATA, filters: { facility_id: 'jeju-farm-001', timestamp_gte: '2025-01-01T00:00:00Z' } }); console.log(`Found ${list.pagination.total} records`); // 레코드 업데이트 await client.records.update(record.id, { value: { ...record.value, temperature: 4.5 } }); // 레코드 삭제 await client.records.delete(record.id);

4.2 SDK 타입 정의

// types.ts export enum RecordCategory { SENSOR_DATA = 'sensor_data', FACILITY_INFO = 'facility_info', SPECIES_DATA = 'species_data', MAINTENANCE = 'maintenance', ALERT = 'alert' } export interface SensorValue { temperature?: number; // 섭씨 (°C) depth?: number; // 미터 (m) salinity?: number; // PSU (Practical Salinity Units) oxygen_level?: number; // mg/L ph_level?: number; // pH turbidity?: number; // NTU (Nephelometric Turbidity Units) current_speed?: number; // m/s } export interface RecordMetadata { sensor_id?: string; facility_id?: string; species_id?: string; location?: GeoLocation; quality?: 'raw' | 'verified' | 'processed'; tags?: string[]; } export interface GeoLocation { latitude: number; // -90 ~ 90 longitude: number; // -180 ~ 180 altitude?: number; // 고도 (m), 음수는 해저 } export interface DeepSeaAquacultureRecord { type: 'WIA-DEEP_SEA_AQUACULTURERecord'; version: string; id: string; timestamp: string; category: RecordCategory; value: SensorValue | any; metadata: RecordMetadata; signature?: string; }

5. 인증 및 권한 관리

5.1 JWT 기반 인증

WIA-DEEP_SEA_AQUACULTURE API는 JWT (JSON Web Token) 기반 인증을 사용합니다. 모든 API 요청은 Authorization 헤더에 유효한 JWT 토큰을 포함해야 합니다.

# 1. 로그인하여 JWT 토큰 획득 POST /api/v1/auth/login Content-Type: application/json { "email": "admin@jeju-farm.kr", "password": "secure_password" } # 응답 { "success": true, "data": { "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...", "refresh_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...", "expires_in": 3600, "token_type": "Bearer" } } # 2. API 요청 시 토큰 사용 GET /api/v1/records Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9... # 3. 토큰 갱신 POST /api/v1/auth/refresh Content-Type: application/json { "refresh_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..." }

5.2 권한 레벨

권한 레벨 설명 허용 작업 사용 사례
Admin 시스템 관리자 모든 작업 시스템 설정, 사용자 관리
Operator 운영자 CRUD (자신의 시설) 일상적인 데이터 입력

8. 한국 심해 양식 사례 연구

8.1 국립수산과학원 API 통합

국립수산과학원(NIFS)은 WIA-DEEP_SEA_AQUACULTURE API를 활용하여 전국 해양 관측 데이터를 통합 관리하고 있습니다. 200개 이상의 센서 노드에서 실시간 데이터를 수집하여 양식업자들에게 제공합니다.

✅ 국립수산과학원 구현 사례

  • 센서 네트워크: 동해, 서해, 남해 200개 센서 노드
  • 데이터 수집: 1분 간격 실시간 수온, 염분, pH 측정
  • API 호출량: 일 평균 28만 건 (초당 3.2건)
  • 데이터 제공: 양식업자 1,200명에게 모바일 앱 제공
  • 효과: 폐사율 35% 감소, 생산성 22% 향상

8.2 제주 해양 목장 시스템

제주특별자치도는 제주 해양 목장에서 WIA-DEEP_SEA_AQUACULTURE API를 활용하여 넙치, 전복, 해삼 등의 양식 관리 시스템을 구축했습니다.

// 제주 해양 목장 API 사용 예제 const client = createClient({ apiKey: process.env.JEJU_FARM_API_KEY, baseUrl: 'https://api.jeju-ocean-ranch.kr/v1' }); // 제주 해역 수온 데이터 조회 const temperatureData = await client.records.list({ category: RecordCategory.SENSOR_DATA, filters: { facility_id: 'jeju-farm-001', 'location.latitude_gte': 33.0, 'location.latitude_lte': 34.0, 'location.longitude_gte': 126.0, 'location.longitude_lte': 127.0, timestamp_gte: new Date(Date.now() - 7 * 24 * 60 * 60 * 1000).toISOString() }, sort: '-timestamp', limit: 100 }); // 평균 수온 계산 const avgTemp = temperatureData.data.reduce( (sum, record) => sum + record.value.temperature, 0 ) / temperatureData.data.length; console.log(`제주 해역 평균 수온 (최근 7일): ${avgTemp.toFixed(2)}°C`);

8.3 현대중공업 해양사업 IoT 플랫폼

현대중공업은 자사의 해양 IoT 플랫폼에 WIA-DEEP_SEA_AQUACULTURE API를 통합하여 심해 양식 시설의 구조물 모니터링과 환경 데이터 수집을 동시에 수행합니다.

구분 모니터링 항목 센서 수 측정 주기
구조물 응력, 변형, 진동 48 10분
환경 수온, 염분, pH, 용존산소 32 1분
해류 유속, 방향 12 5분
생물 어체 크기, 밀도 8 1시간

9. API 버전 관리

WIA-DEEP_SEA_AQUACULTURE API는 URL 경로 기반 버전 관리를 사용합니다. 이를 통해 이전 버전과의 호환성을 유지하면서 새로운 기능을 추가할 수 있습니다.

⚠️ 버전 업그레이드 정책

  • v1.x: 현재 안정 버전, 2027년 말까지 지원
  • v2.x: 2026년 Q2 출시 예정, v1과 병행 지원
  • Deprecation: 최소 12개월 전 공지
  • Migration: 자동 마이그레이션 도구 제공

10. 요약 (Summary)

Chapter 03에서는 WIA-DEEP_SEA_AQUACULTURE의 핵심인 API 인터페이스를 다루었습니다. RESTful 설계 원칙에 따라 구현된 API는 심해 양식 데이터의 생성, 조회, 수정, 삭제 작업을 표준화된 방식으로 제공합니다.

TypeScript SDK를 통해 개발자는 타입 안전성을 보장받으며 쉽게 API를 사용할 수 있습니다. JWT 기반 인증, 역할 기반 권한 관리, 페이지네이션, 필터링 등의 기능은 엔터프라이즈급 시스템 구축에 필요한 모든 요소를 제공합니다.

국립수산과학원, 제주 해양 목장, 현대중공업 등 한국의 주요 기관들이 이미 이 API를 활용하여 성공적인 시스템을 구축했습니다. 이러한 사례들은 WIA-DEEP_SEA_AQUACULTURE API의 실용성과 확장성을 입증합니다.
