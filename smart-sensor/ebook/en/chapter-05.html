Chapter 5

Sensor Fusion & Multi-Modal Sensing

Combining accelerometer, gyroscope, magnetometer, environmental sensors; Kalman filtering, complementary filters, and multi-sensor data integration

Overview

Sensor fusion combines data from multiple sensors to achieve superior performance, reliability, and accuracy compared to individual sensors. By intelligently merging complementary information, smart sensor systems can determine 3D orientation, detect complex activities, estimate position, compensate for sensor drift and noise, and provide redundancy for safety-critical applications. The WIA-SEMI-015 standard encourages multi-sensor architectures, particularly for applications requiring high reliability or complex environmental understanding.

This chapter explores sensor fusion algorithms ranging from simple complementary filters to sophisticated Kalman filters, multi-modal sensing strategies combining motion, environmental, and audio sensors, and practical implementation considerations for resource-constrained microcontrollers.

Key Takeaways

  • Sensor fusion improves accuracy, reduces noise, and provides redundancy
  • Complementary filters offer simplicity and low computational cost
  • Kalman filters provide optimal estimation but require careful tuning
  • 9-axis IMUs (accel + gyro + mag) enable robust 3D orientation tracking
  • Multi-modal sensing (motion + environmental + audio) enhances context awareness

Fundamentals of Sensor Fusion

Individual sensors have inherent limitations: accelerometers suffer from vibration noise and cannot measure rotation around gravity vector, gyroscopes accumulate drift over time due to bias instability, magnetometers are affected by local magnetic disturbances (hard iron, soft iron), and barometric pressure sensors exhibit altitude noise and weather-related variations. Sensor fusion addresses these limitations by combining complementary information.

Complementary Sensor Characteristics

Different sensor types excel at different frequency ranges and timescales. Accelerometers provide accurate static orientation (gravity direction) with high-frequency noise, suitable for DC to 10 Hz accurate measurements. Gyroscopes measure angular velocity with low noise at high frequencies but drift at DC, accurate from 0.1 Hz to 100+ Hz. Magnetometers determine absolute heading (yaw) but are slow to respond, useful for DC to 1 Hz. Fusion algorithms leverage these complementary characteristics by using each sensor where it performs best.

6-Axis Sensor Fusion (Accelerometer + Gyroscope)

The most common fusion configuration combines 3-axis accelerometer and 3-axis gyroscope to estimate device orientation (roll, pitch, yaw). This 6-DOF (Degrees of Freedom) IMU provides reliable pitch and roll but cannot determine absolute yaw without magnetometer.

Complementary Filter

The complementary filter represents the simplest and most computationally efficient fusion approach. It applies a high-pass filter to gyroscope-derived orientation (removing drift) and a low-pass filter to accelerometer-derived orientation (removing noise), then combines the results. 

// Complementary filter for pitch/roll estimation
// alpha = filter coefficient (typically 0.98-0.99)
const float alpha = 0.98;
const float dt = 0.01;  // 10ms sample interval

// Previous orientation estimate (pitch, roll in radians)
float pitch = 0, roll = 0;

// Read sensors
float accel_x, accel_y, accel_z;      // m/s²
float gyro_x, gyro_y, gyro_z;         // rad/s
read_accelerometer(&accel_x, &accel_y, &accel_z);
read_gyroscope(&gyro_x, &gyro_y, &gyro_z);

// Integrate gyroscope (high-frequency component)
pitch += gyro_x * dt;
roll  += gyro_y * dt;

// Calculate orientation from accelerometer (low-frequency)
float accel_pitch = atan2(accel_y, sqrt(accel_x*accel_x + accel_z*accel_z));
float accel_roll  = atan2(-accel_x, accel_z);

// Complementary fusion
pitch = alpha * pitch + (1 - alpha) * accel_pitch;
roll  = alpha * roll  + (1 - alpha) * accel_roll;

The complementary filter provides 5-10° accuracy for gentle motions, runs in <1ms on Cortex-M4, requires minimal memory (<100 bytes state), and is simple to implement and tune. However, it degrades during high acceleration (linear motion confuses accelerometer) and cannot estimate yaw without magnetometer.

Madgwick Filter

The Madgwick filter, developed by Sebastian Madgwick, uses gradient descent optimization to find the orientation quaternion that best aligns predicted gravity/magnetic field with measured values. It offers better performance than complementary filters with similar computational cost.

The algorithm represents orientation as quaternion (q0, q1, q2, q3), runs at 100-500 Hz sampling rate, achieves 2-5° accuracy under normal conditions, and executes in 2-5ms on Cortex-M4. Open-source implementations are widely available, and the single tuning parameter (beta, filter gain) simplifies configuration.

9-Axis Sensor Fusion (Accelerometer + Gyroscope + Magnetometer)

Adding a 3-axis magnetometer enables full 3D orientation estimation including absolute yaw (heading). This 9-DOF IMU configuration is essential for navigation, AR/VR, and robotics applications.

Extended Kalman Filter (EKF)

The Kalman filter provides optimal state estimation in the presence of noise, predicting system state based on motion model and correcting predictions using sensor measurements. The Extended Kalman Filter handles the nonlinear orientation dynamics.

EKF Algorithm Overview

The algorithm maintains state vector (orientation quaternion, gyro bias) and covariance matrix (uncertainty estimate). The prediction step integrates gyroscope measurements to predict orientation change and propagates uncertainty. The update step compares predicted gravity/magnetic field with accelerometer/magnetometer measurements, computes Kalman gain (optimal weighting), and updates state estimate and covariance.

EKF provides optimal estimation (minimum variance), handles sensor noise rigorously, estimates gyro bias online, and achieves 1-3° accuracy with proper tuning. However, it requires higher computational cost (10-30ms on Cortex-M4), needs careful tuning of process/measurement noise covariance, and demands 2-4KB memory for matrices.

Magnetometer Calibration

Magnetometers require calibration to compensate for hard iron (constant offset from ferromagnetic materials) and soft iron (distortion from paramagnetic materials) effects. Without calibration, heading errors can exceed 20-40°.

Hard Iron Calibration

Hard iron effects cause constant offset in all three axes. Calibration involves rotating the sensor through all orientations, collecting magnetometer readings forming a sphere offset from origin, calculating sphere center (hard iron offset), and subtracting offset from all future measurements.

Soft Iron Calibration

Soft iron effects distort the magnetic field, stretching the measurement sphere into an ellipsoid. Calibration determines the transformation matrix converting ellipsoid to sphere. This is more complex than hard iron correction and may require factory calibration or sophisticated field procedures.

Activity Recognition Through Multi-Modal Fusion

Combining motion sensors (accelerometer, gyroscope) with environmental sensors (pressure, temperature) and audio enables sophisticated activity recognition and context awareness.

Common Activity Patterns

Walking exhibits periodic accelerometer pattern (1-2 Hz stride frequency), low gyroscope activity (minimal rotation), and gradual pressure changes (floor-to-floor). Running shows higher frequency accelerometer pattern (2-3 Hz), moderate gyroscope activity, and rapid pressure changes. Cycling presents low-frequency pedaling motion (1-1.5 Hz), steady orientation, and variable pressure (terrain). Ascending stairs displays distinct vertical acceleration, periodic gyroscope rotation, and consistent pressure decrease (~12 Pa per meter).

ML-Based Activity Classification

TinyML models trained on multi-modal sensor data achieve 95%+ accuracy for common activities. The feature extraction process includes accelerometer magnitude and spectral features, gyroscope energy in frequency bands, barometer rate of change, and window-based statistics (mean, variance, peak). The classification employs 1D CNN or LSTM networks (10-30KB models) with 5-20ms inference time and 92-98% accuracy on standard datasets.

Environmental Context from Multi-Sensor Data

Combining environmental sensors provides rich context for smart building, agricultural, and industrial applications.

Indoor/Outdoor Detection

Algorithms combine multiple cues: light level (indoor: <500 lux, outdoor: >5000 lux), UV index (indoor: near zero, outdoor: measurable), temperature variation (indoor: stable, outdoor: diurnal cycle), and barometric pressure patterns. Machine learning models achieve 90-95% accuracy using these features.

HVAC Optimization

Smart building sensors measure temperature (multiple zones), humidity (comfort and mold prevention), CO2 (occupancy proxy, air quality), and VOC (volatile organic compounds, indoor air quality). Fusion algorithms correlate occupancy (PIR, CO2) with thermal comfort, predict HVAC load from weather and schedule, and optimize setpoints for energy efficiency while maintaining comfort.

Implementation Considerations

Practical sensor fusion implementation on microcontrollers requires attention to numerical precision, sampling synchronization, and computational efficiency.

Fixed-Point Arithmetic

For Cortex-M4F and higher, floating-point hardware makes FP32 practical. For Cortex-M0/M3 without FPU, fixed-point representations (Q15, Q31) provide 5-10× speedup. Carefully manage scaling and overflow in filter implementations.

Sensor Synchronization

Fusion algorithms assume simultaneous sensor readings. Implement synchronized sampling using hardware timer triggering all ADC/I2C reads, software timestamping to correct for delays, or interpolation when sensors have different rates. Typical requirements demand <10ms synchronization tolerance for inertial fusion.

Calibration Storage

Store calibration parameters (accelerometer offset/scale, magnetometer hard/soft iron, gyroscope bias) in non-volatile memory. Implement field calibration routines, validate calibration quality, and provide user feedback during calibration procedures.

Power Optimization Strategies

Power consumption is often the primary constraint in smart sensor design. This chapter explores comprehensive strategies for minimizing energy usage while maintaining required functionality, from hardware selection to software optimization and energy harvesting techniques.

Power Budget Analysis

Effective power optimization begins with a thorough understanding of where energy is consumed. A detailed power budget analysis identifies the major consumers and guides optimization efforts to areas with the highest impact. Hardware-Level Optimization

Low-Power Component Selection

Selecting components with the lowest power consumption for the required performance is fundamental. Modern ultra-low-power MCUs achieve active currents below 30 µA/MHz and sleep currents under 1 µA with full RAM retention. Similarly, sensors should be chosen based on both performance and power specifications, considering the specific duty cycle requirements of the application.

Component Selection Criteria:
  • MCU: Active current per MHz, sleep modes, wake-up time, integrated peripherals
  • Sensors: Operating current, standby current, measurement time, on-chip processing
  • Radio: TX/RX current, sleep current, wake-up latency, data rate
  • Regulators: Quiescent current, efficiency at light loads, dropout voltage
  • Memory: Retention current, access energy, standby power

Power Domain Management

Dividing the system into independently controllable power domains allows unused sections to be completely powered off. This is more effective than sleep modes for components that are needed only intermittently. Power gating transistors or load switches enable fine-grained control over power delivery.

// Power Domain Control Example typedef enum { DOMAIN_CORE, // MCU core, always on DOMAIN_SENSORS, // Sensor subsystem DOMAIN_RADIO, // Wireless communication DOMAIN_STORAGE, // External flash/EEPROM DOMAIN_DISPLAY // Optional display } PowerDomain; typedef struct { GPIO_TypeDef* enable_port; uint16_t enable_pin; uint32_t startup_delay_us; float quiescent_current_uA; } DomainConfig; void power_domain_enable(PowerDomain domain) { DomainConfig* cfg = &domain_configs[domain]; // Enable power switch HAL_GPIO_WritePin(cfg->enable_port, cfg->enable_pin, GPIO_PIN_SET); // Wait for power rail to stabilize delay_us(cfg->startup_delay_us); // Initialize peripherals in this domain domain_init_peripherals(domain); } void power_domain_disable(PowerDomain domain) { // Deinitialize peripherals first domain_deinit_peripherals(domain); // Disable power switch DomainConfig* cfg = &domain_configs[domain]; HAL_GPIO_WritePin(cfg->enable_port, cfg->enable_pin, GPIO_PIN_RESET); }

Software-Level Optimization

Duty Cycling Strategies

The most effective power reduction technique is minimizing the time spent in high-power states. Duty cycling involves waking the system only when necessary, performing required operations as quickly as possible, and returning to sleep. The key metrics are duty cycle (active time / total time) and the efficiency of wake/sleep transitions.

Periodic Sampling Wake at fixed intervals 90-99% May miss events

Burst Mode High-rate bursts with long sleep 85-98% Latency between bursts

Algorithm Optimization

Reducing the computational complexity of algorithms directly translates to reduced energy consumption. This includes using integer instead of floating-point arithmetic, lookup tables instead of calculations, and early-exit conditions in processing pipelines. For ML inference, model optimization techniques discussed in Chapter 4 are particularly important.

// Energy-Efficient Algorithm Example: Fixed-Point RMS Calculation // Instead of: sqrt(sum(x^2)/n) with floating point uint16_t calculate_rms_fixed(int16_t* samples, uint16_t count) { uint32_t sum_sq = 0; // Accumulate squared values (Q15 format) for (uint16_t i = 0; i < count; i++) { int32_t sq = ((int32_t)samples[i] * samples[i]) >> 15; sum_sq += sq; } // Calculate mean uint32_t mean_sq = sum_sq / count; // Integer square root using binary search uint16_t rms = 0; uint16_t bit = 0x8000; while (bit > 0) { uint16_t test = rms | bit; if ((uint32_t)test * test <= mean_sq) { rms = test; } bit >>= 1; } return rms; }

Communication Optimization

Wireless communication is typically the most power-hungry operation in IoT sensors. Optimization strategies focus on minimizing transmission time, reducing protocol overhead, and leveraging efficient modulation schemes.

Data Compression

Compressing sensor data before transmission reduces the number of bytes sent, directly reducing radio-on time. Simple compression techniques like delta encoding or run-length encoding add minimal processing overhead while achieving significant compression for sensor data with temporal correlation.

Protocol Efficiency

Communication Power Reduction Techniques:
  • Payload Aggregation: Batch multiple readings into single transmissions
  • Advertising Optimization: Reduce BLE advertising interval and channels
  • Connection Parameters: Optimize connection interval and slave latency
  • Coded PHY: Trade data rate for range, reducing retransmissions
  • LPWAN Protocols: Use LoRa, NB-IoT for long-range, low-power scenarios

Energy Harvesting

Energy harvesting enables battery-free or battery-supplemented operation by capturing energy from the environment. The feasibility depends on available ambient energy sources and the sensor's power requirements.

Power Management ICs

Specialized power management ICs (PMICs) integrate multiple functions needed for efficient power management: voltage regulation, battery charging, energy harvesting interfaces, and power sequencing. Selecting the right PMIC can significantly simplify design and improve overall system efficiency.

Key Takeaways

  • Algorithm Efficiency: Optimize computational complexity for energy savings
  • Communication Optimization: Compress data, batch transmissions, tune protocols
  • Energy Harvesting: Consider ambient energy sources for extended operation
