Chapter 2

Hardware Architecture

MCU selection, MEMS sensor technologies, memory architectures, power management, and hardware accelerators for edge AI

Overview

The hardware architecture of a smart sensor system fundamentally determines its capabilities, power consumption, cost, and suitability for specific applications. This chapter explores the critical hardware components required for WIA-SEMI-015 compliance, including microcontroller selection criteria, MEMS sensor technologies, memory hierarchies, power management subsystems, and optional hardware accelerators that enable efficient machine learning inference at the edge.

Successful hardware design requires balancing competing requirements: processing power versus energy consumption, memory capacity versus cost, sensor accuracy versus size, and wireless range versus battery life. The WIA-SEMI-015 standard establishes minimum requirements while providing flexibility for optimization based on application needs.

Key Takeaways

  • ARM Cortex-M4F or RISC-V RV32IMFC represents minimum MCU requirements
  • Minimum 128KB SRAM and 512KB Flash, recommended 256KB SRAM and 1MB Flash
  • MEMS sensors from Bosch, STMicroelectronics, TDK provide industry-leading performance
  • Hardware accelerators (NPU, DSP) can reduce inference latency by 10-50× while cutting power by 5-10×
  • Ultra-low-power modes (<50µA deep sleep) essential for multi-year battery life

Microcontroller Selection

The microcontroller serves as the brain of the smart sensor, orchestrating sensor readings, executing machine learning inference, managing power states, and handling wireless communication. Selection of the appropriate MCU represents one of the most consequential architectural decisions, as it impacts every aspect of system performance, power consumption, and cost.

ARM Cortex-M Series

ARM Cortex-M processors dominate the embedded market with proven reliability, extensive ecosystem support, and power efficiency. The WIA-SEMI-015 standard specifies Cortex-M4F as the minimum requirement, but Cortex-M7 and M33 offer enhanced capabilities for demanding applications.

The Cortex-M4F incorporates a single-precision floating-point unit (FPU) and DSP extensions, making it suitable for sensor fusion algorithms and lightweight ML inference. Running at 80-120 MHz with 128KB SRAM and 512KB Flash, typical implementations consume 40-80 µA/MHz in active mode and under 5 µA in deep sleep. Popular implementations include STM32F4 series (STMicroelectronics), nRF52840 (Nordic Semiconductor with BLE 5.0), and EFR32MG21 (Silicon Labs with multiprotocol wireless).

The Cortex-M7 provides substantially higher performance with up to 2 MB SRAM, 2 MB Flash, speeds reaching 400-600 MHz, dual-issue superscalar pipeline, and enhanced DSP and floating-point capabilities. This class suits complex sensor fusion, real-time signal processing, and larger ML models. Examples include STM32H7 series (STMicroelectronics), i.MX RT1060 (NXP with LCD controller), and ATSAME70 (Microchip with Ethernet).

The Cortex-M33 introduces ARM TrustZone-M for hardware-enforced security isolation, supporting secure and non-secure execution environments. This architecture is ideal for applications requiring certified security like medical devices, payment systems, and industrial control. Implementations include STM32L5 series (STMicroelectronics), LPC55S69 (NXP), and PSoC 64 (Infineon).

RISC-V Alternatives

RISC-V has emerged as a compelling open-source alternative to ARM, offering royalty-free licensing, customizable instruction sets, and strong support for ML acceleration. The WIA-SEMI-015 standard accepts RISC-V RV32IMFC (32-bit, integer, multiply, FPU, compressed instructions) as equivalent to Cortex-M4F.

Notable RISC-V MCUs include GD32VF103 (GigaDevice, 108 MHz, compatible with STM32F103 pinout), FE310-G002 (SiFive, 320 MHz, hardware multiply/divide), and ESP32-C3 (Espressif, 160 MHz with Wi-Fi and BLE 5.0). The open nature of RISC-V enables custom extensions optimized for specific workloads, though ecosystem maturity still lags behind ARM in areas like tooling and libraries.

Memory Architecture

Memory subsystems profoundly impact both capability and power consumption. Smart sensors must balance volatile SRAM for runtime data and model weights, non-volatile Flash for firmware and persistent storage, and optional external memory for larger models or data logging.

SRAM Requirements

SRAM holds activation tensors during neural network inference, sensor data buffers, wireless protocol stacks, and RTOS kernel data structures. The WIA-SEMI-015 minimum of 128KB suffices for simple models (10-20KB), basic BLE stack (30-50KB), sensor buffers (10-20KB), and RTOS overhead (10-20KB). Recommended 256KB enables more complex models (50-100KB), full-featured wireless stacks (60-80KB), larger buffers for burst sampling, and comfort margin for future features.

Memory layout optimization is critical: place frequently accessed data in fast on-chip SRAM, use DMA for sensor data to avoid CPU overhead, implement ring buffers for streaming data, and leverage memory protection units (MPU) for safety-critical applications. Power-conscious designs can put SRAM sections into retention mode during deep sleep, consuming just 5-10 µA while preserving critical state.

Flash Storage

Flash memory stores firmware code, trained ML models (TFLite, ONNX, or compiled), calibration data and configuration, and logs for diagnostics. The 512KB minimum accommodates bootloader and firmware (200-300KB), one ML model (50-150KB), wireless stack code (100-150KB), and limited logging. Recommended 1MB+ allows dual-bank firmware for safe OTA updates, multiple ML models for different modes, comprehensive logging and diagnostics, and future firmware growth.

Flash endurance matters for devices performing frequent OTA updates or data logging. Modern NOR Flash typically provides 10,000-100,000 erase cycles per block. Wear-leveling algorithms distribute writes across blocks to maximize lifespan, and critical data should use redundant storage with error detection/correction codes.

External Memory Options

Applications with large models, extensive data logging, or vision processing may require external memory. Options include SPI/QSPI NOR Flash (8-128 MB, inexpensive, moderate speed), SPI/QSPI PSRAM (4-64 MB, faster random access, higher cost), and eMMC/SD cards (GB range, mass storage, higher power). Access latency and power consumption must be carefully considered, as external memory access can consume 10-50× more energy than on-chip access.

MEMS Sensor Technologies

Micro-Electro-Mechanical Systems (MEMS) fabricate miniature mechanical structures using semiconductor manufacturing processes, enabling mass production of tiny, low-cost, high-performance sensors. The smart sensor market relies heavily on MEMS technology from leading manufacturers.

Inertial Sensors

Accelerometers measure linear acceleration along one or more axes (typically 3-axis), enabling applications like activity recognition (step counting, fall detection), vibration monitoring for predictive maintenance, tilt sensing and orientation detection, and tap/double-tap gesture recognition. Leading devices include Bosch BMA400 (ultra-low power: 3.5 µA at 100 Hz), STMicroelectronics LIS2DW12 (1.1 mm × 1.1 mm × 0.5 mm package), and TDK ICM-42688-P (high performance: ±16g range, 16-bit resolution).

Gyroscopes measure angular velocity around rotation axes, crucial for motion tracking and stabilization, drone and robotics navigation, gesture recognition, and augmented/virtual reality applications. Performance metrics include noise density (mdps/√Hz), zero-rate offset stability (dps), and bandwidth (up to several kHz). Popular choices include Bosch BMG250 (low noise: 0.007 dps/√Hz), STMicroelectronics LSM6DSO (combined 6-axis IMU with ML core), and TDK IAM-20680 (automotive-grade, AEC-Q100).

Magnetometers sense magnetic field strength for compass heading, indoor positioning (magnetic landmarks), metal detection, and current sensing (non-contact). Devices like Bosch BMM350 (low power: 4 µA average), STMicroelectronics LIS2MDL (high resolution: 0.15 mG LSB), and Asahi Kasei AK09918C (wide range: ±4900 µT) provide robust performance.

Environmental Sensors

Temperature sensors using bandgap references or thermistors enable HVAC control and energy optimization, cold chain monitoring, body temperature tracking, and MCU thermal management. Accuracy ranges from ±0.5°C (basic) to ±0.1°C (precision). Humidity sensors based on capacitive or resistive technologies support weather monitoring, HVAC control, agricultural applications, and museum preservation (art, artifacts). Typical accuracy is ±2-3% RH.

Pressure sensors measure barometric pressure for altitude estimation (GPS-free navigation), weather prediction, HVAC optimization, and medical applications (blood pressure). Devices achieve ±0.5 hPa absolute accuracy with resolutions down to 0.01 hPa. Air quality sensors detect VOCs (Volatile Organic Compounds), CO2, particulate matter (PM2.5/PM10), and NO2/O3 for smart building ventilation control, outdoor air quality monitoring, and health/safety compliance. Bosch BME680 integrates temperature, humidity, pressure, and VOC sensing in a single 3mm × 3mm package.

Hardware Accelerators for ML Inference

While capable MCUs can execute ML inference using optimized software libraries, dedicated hardware accelerators dramatically reduce latency and energy consumption. Modern smart sensors increasingly incorporate neural processing units (NPUs), DSP extensions, or specialized AI engines.

ARM Ethos-U NPUs

ARM's Ethos-U family provides microNPUs optimized for Cortex-M systems, offering 32-256 MACs (multiply-accumulate operations) per cycle, INT8 and INT16 quantization support, 64-512KB tightly coupled memory, and 0.5-4 TOPS/W efficiency. The Ethos-U55 delivers up to 256 MACs/cycle with 512KB SRAM, achieving 10-50× inference speedup versus pure Cortex-M execution while reducing energy by 5-10×. Integration in MCUs like STM32N6 (STMicroelectronics) and Corstone-300 (ARM development platform) demonstrates mainstream adoption.

Vendor-Specific Accelerators

Many MCU vendors integrate proprietary ML accelerators. STMicroelectronics embeds ML cores in LSM6DSO IMU and ISM330DHCX sensors, executing decision trees and simple neural networks directly within the sensor without waking the main MCU. NXP provides eIQ inference engines on i.MX RT crossover MCUs with CMSIS-NN acceleration, Glow neural compiler, and TensorFlow Lite Micro support. Nordic Semiconductor acquired Novelda ultra-wideband radar technology, integrating AI acceleration for presence detection and vital signs monitoring in nRF54 series.

DSP Extensions

Most modern MCUs include DSP instructions that accelerate signal processing and neural network operations: SIMD (Single Instruction Multiple Data) for parallel operations on multiple data elements, MAC (Multiply-ACcumulate) optimized for convolution and matrix operations, and saturating arithmetic to prevent overflow. ARM CMSIS-DSP library provides hand-optimized implementations of common functions (FFT, FIR filters, matrix operations) leveraging these extensions, often achieving 2-5× speedup versus generic C code.

Power Management Subsystems

Achieving multi-year battery life requires sophisticated power management, as average power consumption must remain below 100 µA (often 10-50 µA) for coin cell operation. The WIA-SEMI-015 standard mandates deep sleep modes consuming less than 50 µA.

MCU Power Modes

Modern MCUs support multiple power states with varying wake-up latency and consumption. Active mode (10-200 mW) runs CPU, peripherals, and wireless; deep sleep (10-50 µA) retains SRAM, RTC running, fast wake-up (µs-ms); shutdown mode (<5 µA) loses SRAM, requires reset, slow wake-up (ms). Effective duty cycling alternates between these states, spending 99%+ time in sleep for battery-powered applications.

Power Management ICs (PMICs)

Dedicated PMICs optimize power delivery with multiple voltage rails (1.8V digital, 3.3V analog, etc.), dynamic voltage/frequency scaling (DVFS) to match workload, buck/boost converters for battery voltage variations, and low quiescent current (<10 µA). Examples include Texas Instruments TPS62840 (60 nA quiescent, 95% efficiency), Analog Devices ADP5360 (buck/boost with multiple LDOs), and Maxim MAX77650 (single-inductor multiple-output buck-boost).

Energy Harvesting

Advanced implementations incorporate energy harvesting to extend or eliminate battery replacement: solar cells (indoor: 10-100 µW/cm², outdoor: 1-10 mW/cm²), piezoelectric generators (vibration: 10-500 µW), thermoelectric generators (5-10°C delta: 100-1000 µW), and RF energy harvesting (ambient: 1-10 µW, dedicated: 100 µW-1 mW). Storage elements include supercapacitors (fast charge, unlimited cycles) or thin-film batteries (higher density, limited cycles). Texas Instruments BQ25570 and Analog Devices ADP5090 provide harvesting-optimized power management.

Peripheral Interfaces

Smart sensors require various interfaces for sensor communication, debugging, and expansion. I2C (Inter-Integrated Circuit) runs at 100-400 kHz (standard/fast mode) or 1 MHz+ (fast mode plus), supporting multi-device bus with 7-bit addressing, used by most MEMS sensors, and consuming low power (µW). SPI (Serial Peripheral Interface) achieves 1-50 MHz operation with full-duplex communication, separate CS per device, higher throughput than I2C for high-speed sensors and displays. UART provides asynchronous serial (9600-921600 baud), simple two-wire interface, debugging and logging, and GPS module communication.

Summary

This chapter covered the hardware architecture of WIA-SEMI-015 compliant smart sensors, including microcontroller selection (ARM Cortex-M4F/M7/M33 or RISC-V alternatives), memory requirements (minimum 128KB SRAM, 512KB Flash), MEMS sensor technologies from leading manufacturers (Bosch, STMicroelectronics, TDK), hardware accelerators for efficient ML inference, and sophisticated power management enabling multi-year battery operation. MEMS Accelerometers

MEMS accelerometers measure acceleration forces by detecting the displacement of a proof mass. These devices are ubiquitous in modern electronics, enabling applications from smartphone orientation detection to vehicle airbag deployment systems.

Capacitive MEMS Accelerometer Architecture

The most common MEMS accelerometer design uses capacitive sensing with a suspended proof mass between fixed electrodes. When acceleration occurs, the proof mass moves relative to the fixed electrodes, changing the capacitance. This change is detected by the readout circuitry and converted to an acceleration value.

MEMS Gyroscopes

MEMS gyroscopes measure angular velocity using the Coriolis effect. When a vibrating mass rotates, the Coriolis force causes displacement perpendicular to both the vibration and rotation axes. This displacement is detected capacitively and converted to an angular rate measurement.

MEMS Pressure Sensors

MEMS pressure sensors typically employ a thin diaphragm that deflects under applied pressure. The deflection is measured using piezoresistive elements (silicon resistors whose resistance changes with strain) or capacitive techniques. These sensors achieve exceptional miniaturization while maintaining high accuracy.

Pressure Measurement Types

Absolute Pressure: Measured relative to vacuum reference (e.g., barometric pressure)
Gauge Pressure: Measured relative to atmospheric pressure (e.g., tire pressure)
Differential Pressure: Measures difference between two pressure ports (e.g., flow measurement)

LiDAR and Time-of-Flight Sensors

Light Detection and Ranging (LiDAR) and Time-of-Flight (ToF) sensors measure distance by timing the round-trip of light pulses. These technologies enable precise 3D mapping and ranging applications from autonomous vehicles to smartphone depth cameras.

Image Sensors

Modern image sensors convert optical images into electronic signals using arrays of photodetectors. The two dominant technologies are Charge-Coupled Devices (CCD) and Complementary Metal-Oxide-Semiconductor (CMOS) image sensors.

CCD vs CMOS Image Sensors

CCD: Higher image quality, lower noise, uniform pixel response. Used in scientific and medical imaging where image quality is paramount.

CMOS: Lower power, faster readout, on-chip processing, lower cost. Dominant in consumer electronics, smartphones, and automotive applications.

pH and Ion-Selective Sensors

Electrochemical sensors for pH and specific ions are essential in water quality monitoring, medical diagnostics, and industrial process control. Ion-Selective Electrodes (ISEs) use membranes that selectively respond to specific ions.

// Nernst Equation for Ion-Selective Electrodes // E = E0 + (RT/nF) × ln(a_ion) // Where: // E = electrode potential (V) // E0 = standard electrode potential (V) // R = gas constant (8.314 J/(mol·K)) // T = temperature (K) // n = ion charge // F = Faraday constant (96485 C/mol) // a_ion = ion activity const R = 8.314; const F = 96485; function nernstSlope(temperature: number, ionCharge: number): number { // Slope in mV per decade at given temperature return (2.303 * R * temperature) / (ionCharge * F) * 1000; } // At 25°C (298.15 K), for H+ (n=1): // Slope = 59.16 mV/decade (pH unit) function calculatePH(voltage: number, tempK: number, calibrationPH: number, calibrationV: number): number { const slope = nernstSlope(tempK, 1); return calibrationPH - (voltage - calibrationV) / slope; }

Glucose Biosensors

Glucose biosensors represent the most commercially successful biosensor application, enabling millions of diabetic patients to monitor blood glucose levels at home. These devices have evolved through multiple generations of technology.

Generation Mechanism Characteristics Performance
1st Generation Oxygen electrode with GOx Measures O2 consumption Simple, oxygen dependent
2nd Generation Mediator-based electron transfer Artificial electron acceptors O2 independent, interference issues
3rd Generation Direct electron transfer Wired enzyme or nanomaterials High selectivity, complex fabrication
4th Generation Non-enzymatic (nanomaterials) Metal/metal oxide nanostructures High stability, no enzyme degradation

Biosensor Challenges

Stability: Biological recognition elements can denature or degrade over time.
Selectivity: Cross-reactivity with similar molecules must be minimized.
Reproducibility: Biological variability requires careful quality control.
Sterilization: Some biosensors cannot withstand harsh sterilization methods.

Humidity Sensors

Humidity sensors measure the water vapor content in air or gases. The primary technologies include capacitive, resistive, and thermal conductivity methods, each with specific advantages for different applications.

Capacitive Humidity Sensor

Capacitive humidity sensors use a hygroscopic dielectric material between two electrodes. As humidity changes, the dielectric absorbs or releases water molecules, changing its permittivity and thus the capacitance.

Range 0-100% RH
Accuracy ±2-3% RH
Hysteresis ±1% RH
Response Time 5-30 seconds

2.6 Chapter Summary

This chapter has provided a comprehensive overview of the major sensor technologies employed in smart sensor systems. We have explored the principles, fabrication methods, and applications of MEMS sensors, optical sensors, chemical sensors, and biological sensors.

Key takeaways from this chapter include:

  • MEMS technology enables miniaturized sensors for motion, pressure, and environmental measurement through microfabrication techniques
  • Optical sensors span the electromagnetic spectrum and include photodetectors, LiDAR systems, and image sensors
  • Chemical sensors employ various transduction mechanisms including metal oxide, electrochemical, and infrared absorption
  • Biosensors combine biological recognition elements with physical/chemical transducers for specific analyte detection
  • Environmental sensors monitor temperature, humidity, and atmospheric conditions for diverse applications
  • Selection of appropriate sensor technology requires matching sensor capabilities to application requirements
