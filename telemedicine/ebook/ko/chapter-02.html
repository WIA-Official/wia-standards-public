📹 제2장: 화상 상담 프로토콜

WIA-MED-004 | 원격 의료 표준 | 한국어판

장 개요

이 장에서는 원격 의료의 핵심인 화상 상담을 위한 기술적 프로토콜을 다룹니다. WebRTC 기술, 비디오/오디오 품질 요구사항, 네트워크 최적화, 그리고 안정적인 화상 진료를 위한 구현 방법을 학습합니다.

2.1 WebRTC 기술 개요

WebRTC (Web Real-Time Communication)는 웹 브라우저와 모바일 애플리케이션에서 실시간 오디오, 비디오, 데이터 통신을 가능하게 하는 오픈 소스 프로젝트입니다. WIA-MED-004 표준은 WebRTC를 화상 상담의 기본 프로토콜로 채택합니다.

WebRTC의 핵심 장점

  • 플러그인 불필요: 별도 소프트웨어 설치 없이 브라우저에서 바로 작동
  • 낮은 지연시간: P2P 연결로 50-100ms의 초저지연 통신
  • 고품질: HD 비디오 및 고음질 오디오 지원
  • 보안: DTLS와 SRTP를 통한 엔드투엔드 암호화
  • 적응형 대역폭: 네트워크 상황에 따라 자동으로 품질 조정
  • 크로스 플랫폼: Windows, macOS, Linux, Android, iOS 모두 지원

WebRTC 아키텍처

WebRTC는 세 가지 주요 API로 구성됩니다:

API 기능 의료 용도
getUserMedia() 카메라/마이크 접근 환자/의사 비디오 스트림 수집
RTCPeerConnection P2P 오디오/비디오 통신 실시간 화상 진료 연결
RTCDataChannel P2P 데이터 전송 생체 신호, 채팅, 파일 공유

2.2 비디오 품질 요구사항

의료 상담에서 비디오 품질은 정확한 진단에 직접적인 영향을 미칩니다. WIA-MED-004는 진료 유형별로 최소 비디오 품질 기준을 정의합니다.

진료 유형별 비디오 품질 기준

진료 유형 최소 해상도 권장 해상도 프레임레이트 비트레이트
일반 상담 640x480 (VGA) 1280x720 (HD) 15-24 fps 500-1000 kbps
피부과 1280x720 (HD) 1920x1080 (Full HD) 24-30 fps 1500-3000 kbps
안과 1920x1080 (Full HD) 3840x2160 (4K) 30 fps 3000-8000 kbps
정신과 640x480 (VGA) 1280x720 (HD) 15-24 fps 500-1000 kbps
응급 상담 1280x720 (HD) 1920x1080 (Full HD) 24-30 fps 1500-3000 kbps

색 정확도 요구사항

피부 병변, 발진, 황달 등을 정확히 진단하기 위해서는 색상 재현이 중요합니다.

색공간 및 색온도 기준:

  • 색공간: sRGB 또는 Adobe RGB
  • 색온도: 5000K-6500K (주광)
  • 색 정확도: ΔE < 5 (피부과의 경우 ΔE < 3)
  • 화이트 밸런스: 자동 화이트 밸런스(AWB) 필수

2.3 오디오 품질 요구사항

명확한 의사소통은 정확한 병력 청취와 증상 파악에 필수적입니다. 오디오 품질이 낮으면 오진의 위험이 있습니다.

오디오 사양

항목 최소 요구사항 권장 사양
코덱 Opus Opus 48kHz
비트레이트 16 kbps 32-64 kbps
샘플링 레이트 16 kHz 48 kHz
지연시간 < 150 ms < 100 ms
패킷 손실 허용 < 3% < 1%

음향 최적화 기술

필수 오디오 처리:

  • 에코 제거 (AEC): 스피커에서 나오는 소리가 마이크로 다시 들어가는 현상 제거
  • 노이즈 억제 (NS): 배경 소음 필터링 (에어컨, 키보드 소리 등)
  • 자동 이득 제어 (AGC): 음량 자동 조절로 일정한 볼륨 유지
  • 음성 활동 감지 (VAD): 말하는 순간만 전송하여 대역폭 절약
  • 패킷 손실 은닉 (PLC): 패킷 손실 시 음질 저하 최소화

2.4 네트워크 요구사항

안정적인 화상 진료를 위해서는 충분한 대역폭과 낮은 지연시간이 필요합니다.

대역폭 요구사항

화상 품질 업로드 다운로드 총 대역폭
SD (480p) 0.6-1.0 Mbps 0.6-1.0 Mbps 1.2-2.0 Mbps
HD (720p) 1.2-2.0 Mbps 1.2-2.0 Mbps 2.4-4.0 Mbps
Full HD (1080p) 2.5-4.0 Mbps 2.5-4.0 Mbps 5.0-8.0 Mbps
4K (2160p) 8.0-15.0 Mbps 8.0-15.0 Mbps 16.0-30.0 Mbps

지연시간 및 지터

  • 왕복 지연시간 (RTT): < 150ms (권장 < 100ms)
  • 단방향 지연시간: < 75ms
  • 지터: < 30ms
  • 패킷 손실률: < 1% (최대 3%)

2.5 화면 공유 및 주석 기능

의료 영상, 검사 결과, 교육 자료를 환자와 공유하고 설명하는 기능은 원격 진료에서 필수적입니다.

화면 공유 기능

  • 전체 화면 공유: 의사의 EMR 화면 전체 공유
  • 창 선택 공유: 특정 애플리케이션 창만 공유 (개인정보 보호)
  • 탭 공유: 브라우저의 특정 탭만 공유
  • 의료 영상 공유: X-ray, CT, MRI 등 DICOM 이미지

실시간 주석 도구

주석 기능 요구사항:

  • 펜 도구: 자유 곡선 그리기 (병변 표시)
  • 도형 도구: 원, 사각형, 화살표
  • 텍스트 삽입: 주요 부위 라벨링
  • 색상 선택: 다양한 색상으로 구분
  • 지우개: 특정 주석만 제거
  • 전체 지우기: 모든 주석 제거
  • 저장: 주석이 포함된 이미지 저장

2.6 화상 상담 보안

화상 진료 중 전송되는 모든 데이터는 암호화되어야 합니다.

암호화 프로토콜

프로토콜 용도 암호화 수준
DTLS (Datagram TLS) WebRTC 시그널링 TLS 1.2 이상
SRTP (Secure RTP) 오디오/비디오 스트림 AES-128 이상
HTTPS 웹 애플리케이션 TLS 1.3 권장
SCTP over DTLS 데이터 채널 DTLS 1.2 이상

접근 제어

화상 상담 보안 요구사항:

  • 대기실 기능: 의사가 승인할 때까지 환자 대기
  • 비밀번호 보호: 상담방에 비밀번호 설정 가능
  • 참가자 관리: 허가되지 않은 참가자 강제 퇴장
  • 녹화 제어: 녹화 시작/종료 권한 관리
  • 화면 공유 제어: 누가 화면을 공유할 수 있는지 제어

2.7 다자간 화상 상담

협진, 보호자 참여, 의료진 교육 등을 위해 다자간 화상 상담이 필요할 수 있습니다.

다자간 회의 아키텍처

아키텍처 설명 장점 단점
Mesh (P2P) 모든 참가자가 서로 직접 연결 낮은 지연시간, 서버 불필요 3-4명 이상 시 대역폭 과다
SFU (Selective Forwarding Unit) 서버가 스트림을 라우팅 확장성 우수, 10명 이상 가능 서버 인프라 필요
MCU (Multipoint Control Unit) 서버가 스트림을 합성 대역폭 절약 높은 서버 부하, 지연 증가

WIA-MED-004는 10명 이하의 소규모 협진에는 SFU 방식을, 대규모 원격 교육에는 MCU 방식을 권장합니다.

2.8 네트워크 적응형 비디오

불안정한 네트워크 환경에서도 끊김 없는 화상 진료를 제공하기 위해서는 적응형 비트레이트 조정이 필수입니다.

적응형 기술

  • 시뮬캐스트(Simulcast): 동일한 비디오를 여러 해상도/비트레이트로 동시 전송하여, 수신자가 네트워크 상황에 맞게 선택
  • SVC (Scalable Video Coding): 계층적 비디오 인코딩으로 네트워크 상황에 따라 레이어 조정
  • 동적 비트레이트 조정: 실시간 네트워크 통계(RTT, 패킷 손실)를 기반으로 비트레이트 자동 조절
  • 우선순위 기반 전송: I-frame, P-frame 우선순위 구분하여 중요한 프레임 먼저 전송

2.9 진료 기록 및 녹화

화상 상담 내용을 기록하여 향후 참고 자료로 활용하고, 법적 분쟁 시 증거로 사용할 수 있습니다.

녹화 요구사항

항목 요구사항
사전 동의 환자와 의사 모두 녹화에 명시적 동의
녹화 알림 녹화 시작/종료 시 명확한 시각/청각적 알림
저장 형식 MP4 (H.264 비디오, AAC 오디오) 또는 WebM
암호화 저장 AES-256으로 암호화하여 저장
보관 기간 의료법에 따라 최소 10년 (국가별 상이)
접근 제어 담당 의사와 환자만 접근 가능

2.10 구현 예제

다음은 WebRTC를 활용한 간단한 화상 상담 구현 예제입니다.

JavaScript 기본 구조

// 1. 로컬 미디어 스트림 가져오기
const constraints = {
  video: {
    width: { min: 1280, ideal: 1920 },
    height: { min: 720, ideal: 1080 },
    frameRate: { ideal: 30, max: 30 }
  },
  audio: {
    echoCancellation: true,
    noiseSuppression: true,
    autoGainControl: true
  }
};

const localStream = await navigator.mediaDevices.getUserMedia(constraints);
document.getElementById('localVideo').srcObject = localStream;

// 2. RTCPeerConnection 생성
const config = {
  iceServers: [
    { urls: 'stun:stun.l.google.com:19302' },
    {
      urls: 'turn:turn.example.com:3478',
      username: 'user',
      credential: 'pass'
    }
  ]
};

const pc = new RTCPeerConnection(config);

// 3. 로컬 스트림 추가
localStream.getTracks().forEach(track => {
  pc.addTrack(track, localStream);
});

// 4. 원격 스트림 수신
pc.ontrack = (event) => {
  document.getElementById('remoteVideo').srcObject = event.streams[0];
};

// 5. ICE 후보 처리
pc.onicecandidate = (event) => {
  if (event.candidate) {
    // 시그널링 서버로 candidate 전송
    sendToSignalingServer({
      type: 'candidate',
      candidate: event.candidate
    });
  }
};

// 6. Offer 생성 및 전송
const offer = await pc.createOffer();
await pc.setLocalDescription(offer);
sendToSignalingServer({ type: 'offer', sdp: offer });

// 7. Answer 수신 및 처리
async function handleAnswer(answer) {
  await pc.setRemoteDescription(answer);
}

// 8. 연결 품질 모니터링
setInterval(async () => {
  const stats = await pc.getStats();
  stats.forEach(report => {
    if (report.type === 'inbound-rtp' && report.mediaType === 'video') {
      console.log('Packets lost:', report.packetsLost);
      console.log('Jitter:', report.jitter);
      console.log('Bitrate:', report.bytesReceived / report.timestamp);
    }
  });
}, 1000);

2.11 품질 모니터링 및 문제 해결

화상 진료 중 품질 저하가 발생하면 즉시 감지하고 대응해야 합니다.

실시간 품질 지표

  • 비디오 해상도: 현재 전송/수신 해상도
  • 프레임레이트: 실시간 fps
  • 비트레이트: 현재 전송 속도
  • 패킷 손실률: 손실된 패킷 비율
  • 지터: 패킷 도착 시간 변동성
  • RTT: 왕복 지연시간

자동 품질 저하 대응

문제 자동 대응 사용자 알림
패킷 손실 > 3% 비트레이트 20% 감소 "네트워크 불안정"
RTT > 200ms 프레임레이트 감소 "지연 시간 증가"
대역폭 부족 해상도 한 단계 낮춤 "화질 자동 조정"
연결 끊김 자동 재연결 시도 "재연결 중..."
