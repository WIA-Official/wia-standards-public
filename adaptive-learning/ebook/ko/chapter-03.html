3장: 학습자 모델링

3.1 학습자 모델링의 중요성

학습자 모델링은 적응형 학습 시스템의 핵심입니다. 시스템이 학습자에 대해 더 정확하게 파악할수록, 더 적절한 적응을 제공할 수 있습니다. 학습자 모델은 학습자의 지식, 기술, 선호도, 감정 상태 등을 포착하는 컴퓨터 표현입니다.

모델링 원칙: 학습자 모델은 동적이고 지속적으로 업데이트되어야 합니다. 학습자는 계속 변화하며, 모델은 이러한 변화를 실시간으로 반영해야 합니다.

3.2 지식 추적 (Knowledge Tracing)

지식 추적은 학습자의 지식 상태를 시간에 따라 추론하는 기법입니다:

3.2.1 베이지안 지식 추적 (BKT)

// 베이지안 지식 추적 구현
class BayesianKnowledgeTracing {
    // 모델 파라미터
    private pL0: number;   // 초기 지식 확률
    private pT: number;    // 학습 전이 확률
    private pG: number;    // 추측 확률
    private pS: number;    // 실수 확률

    constructor(params: BKTParams) {
        this.pL0 = params.pL0 || 0.3;
        this.pT = params.pT || 0.1;
        this.pG = params.pG || 0.2;
        this.pS = params.pS || 0.1;
    }

    update(pL: number, correct: boolean): number {
        // 응답이 정답일 경우
        const pLGivenCorrect = correct
            ? (pL * (1 - this.pS)) / (pL * (1 - this.pS) + (1 - pL) * this.pG)
            : (pL * this.pS) / (pL * this.pS + (1 - pL) * (1 - this.pG));

        // 학습 전이 적용
        const pLNext = pLGivenCorrect + (1 - pLGivenCorrect) * this.pT;

        return pLNext;
    }

    predictCorrect(pL: number): number {
        return pL * (1 - this.pS) + (1 - pL) * this.pG;
    }
}

3.3 항목 반응 이론 (IRT)

IRT는 학습자의 능력과 문항의 특성을 동시에 모델링합니다:

모델 파라미터 설명
1-PL (Rasch) 난이도 (b) 모든 문항의 변별력 동일
2-PL 난이도, 변별력 (a, b) 문항별 변별력 다름
3-PL 난이도, 변별력, 추측 (a, b, c) 추측 가능성 반영 
// 3-PL IRT 모델
class IRTModel {
    // 정답 확률 계산
    probability(theta: number, item: IRTItem): number {
        const { a, b, c } = item;
        const exponent = -a * (theta - b);
        return c + (1 - c) / (1 + Math.exp(exponent));
    }

    // 능력 추정 (MLE)
    estimateAbility(responses: Response[], items: IRTItem[]): number {
        // Newton-Raphson 방법으로 최대우도 추정
        let theta = 0;  // 초기값
        const maxIter = 100;
        const tolerance = 0.001;

        for (let i = 0; i < maxIter; i++) {
            const { likelihood, derivative } = this.computeLikelihood(theta, responses, items);
            const delta = derivative / likelihood;

            if (Math.abs(delta) < tolerance) break;
            theta += delta;
        }

        return theta;
    }
}

3.4 학습 스타일 모델링

3.4.1 VARK 모델

  • Visual (시각형): 다이어그램, 차트, 비디오 선호
  • Auditory (청각형): 강의, 토론 선호
  • Read/Write (읽기/쓰기형): 텍스트 기반 학습 선호
  • Kinesthetic (운동감각형): 실습, 체험 학습 선호

3.4.2 Felder-Silverman 모델

// 학습 스타일 추론
interface LearningStyleProfile {
    // Felder-Silverman 차원
    processing: 'active' | 'reflective';      // 정보 처리
    perception: 'sensing' | 'intuitive';       // 정보 인식
    input: 'visual' | 'verbal';                // 정보 입력
    understanding: 'sequential' | 'global';    // 이해 방식

    // 각 차원의 강도 (-11 ~ +11)
    scores: {
        processing: number;
        perception: number;
        input: number;
        understanding: number;
    };
}

class LearningStyleDetector {
    // 행동 데이터에서 학습 스타일 추론
    inferStyle(behaviors: LearnerBehavior[]): LearningStyleProfile {
        const indicators = this.extractIndicators(behaviors);

        return {
            processing: indicators.discussionTime > indicators.readingTime
                ? 'active' : 'reflective',
            perception: indicators.exampleRequests > indicators.theoryRequests
                ? 'sensing' : 'intuitive',
            input: indicators.videoViews > indicators.textReads
                ? 'visual' : 'verbal',
            understanding: indicators.linearProgress
                ? 'sequential' : 'global',
            scores: this.calculateScores(indicators)
        };
    }
}

3.5 감정 상태 모델링

학습자의 감정 상태는 학습 효과에 큰 영향을 미칩니다:

// 감정 상태 감지
interface AffectiveState {
    engagement: number;      // 참여도 (0-1)
    frustration: number;     // 좌절감 (0-1)
    boredom: number;         // 지루함 (0-1)
    confusion: number;       // 혼란 (0-1)
    curiosity: number;       // 호기심 (0-1)
    confidence: number;      // 자신감 (0-1)
}

class AffectDetector {
    // 다중 모드 감정 감지
    async detect(data: MultimodalData): Promise<AffectiveState> {
        const signals = await Promise.all([
            this.analyzeInteractionPatterns(data.interactions),
            this.analyzeResponseTimes(data.responseTimes),
            this.analyzeHelpRequests(data.helpRequests),
            data.facialData ? this.analyzeFacialExpressions(data.facialData) : null
        ]);

        return this.fuseSignals(signals);
    }

    // 상호작용 패턴 분석
    private analyzeInteractionPatterns(interactions: Interaction[]): AffectSignal {
        const recentInteractions = interactions.slice(-10);

        const rapidClicks = this.detectRapidClicks(recentInteractions);
        const longPauses = this.detectLongPauses(recentInteractions);
        const helpSeekingPattern = this.detectHelpSeeking(recentInteractions);

        return {
            frustration: rapidClicks ? 0.7 : 0.2,
            confusion: longPauses ? 0.6 : 0.2,
            boredom: !helpSeekingPattern && longPauses ? 0.5 : 0.1
        };
    }
}
감정과 학습: 연구에 따르면 적절한 수준의 도전은 참여도를 높이지만, 과도한 어려움은 좌절감을 유발합니다. 적응형 시스템은 학습자를 "흐름(Flow)" 상태로 유지하는 것을 목표로 합니다.

3.6 오개념 모델링

// 오개념 탐지 및 모델링
interface MisconceptionModel {
    // 알려진 오개념 목록
    knownMisconceptions: Map<ConceptId, Misconception[]>;

    // 학습자별 오개념 추적
    learnerMisconceptions: Map<LearnerId, DetectedMisconception[]>;
}

class MisconceptionDetector {
    detect(responses: Response[], concept: Concept): DetectedMisconception[] {
        const errorPatterns = this.analyzeErrorPatterns(responses);
        const detected: DetectedMisconception[] = [];

        for (const pattern of errorPatterns) {
            const matchingMisconception = this.matchToKnownMisconception(pattern, concept);
            if (matchingMisconception) {
                detected.push({
                    misconception: matchingMisconception,
                    confidence: pattern.confidence,
                    evidence: pattern.responses
                });
            }
        }

        return detected;
    }
}
