Chapter 6: LMS Integration and Interoperability

Adaptive learning systems rarely operate in isolation. They must integrate with existing educational technology infrastructure, particularly Learning Management Systems (LMS), student information systems, and content repositories. This chapter details the interoperability standards and integration patterns specified by WIA-ADAPTIVE_LEARNING.

Interoperability Standards Landscape

The educational technology ecosystem has developed several interoperability standards to enable systems to work together. The WIA-ADAPTIVE_LEARNING standard requires support for key standards to ensure broad compatibility.

Required Interoperability Standards

  • LTI (Learning Tools Interoperability): Tool integration with LMS platforms
  • xAPI (Experience API): Learning activity data exchange
  • QTI (Question and Test Interoperability): Assessment item portability
  • LIS (Learning Information Services): Roster and enrollment data
  • Caliper Analytics: Learning event data model

LTI 1.3 Integration

Learning Tools Interoperability (LTI) 1.3 is the current industry standard for integrating external tools with Learning Management Systems. LTI 1.3 uses OAuth 2.0 and OpenID Connect for secure authentication and authorization.

// LTI 1.3 Tool Provider Implementation
interface LTI13Config {
  // Platform registration
  platformId: string;
  clientId: string;
  deploymentId: string;

  // URLs
  authLoginUrl: string;
  authTokenUrl: string;
  keysetUrl: string;

  // Tool configuration
  toolPublicKeySet: JsonWebKeySet;
  toolPrivateKey: string;
}

class LTI13ToolProvider {
  constructor(private config: LTI13Config) {}

  // Handle OIDC login initiation
  async handleLoginInitiation(
    request: LoginInitiationRequest
  ): Promise<AuthenticationResponse> {
    // Validate request parameters
    this.validateLoginRequest(request);

    // Generate state and nonce
    const state = this.generateSecureRandom();
    const nonce = this.generateSecureRandom();

    // Store state for validation
    await this.stateStore.save(state, {
      nonce,
      targetLinkUri: request.targetLinkUri,
      timestamp: Date.now()
    });

    // Build authentication request
    const authRequest = {
      scope: 'openid',
      response_type: 'id_token',
      client_id: this.config.clientId,
      redirect_uri: this.config.toolRedirectUri,
      login_hint: request.loginHint,
      lti_message_hint: request.ltiMessageHint,
      state,
      nonce,
      prompt: 'none'
    };

    return {
      redirectUrl: this.buildAuthUrl(authRequest)
    };
  }

  // Handle LTI resource link launch
  async handleResourceLinkLaunch(
    idToken: string
  ): Promise<LaunchContext> {
    // Verify and decode JWT
    const claims = await this.verifyIdToken(idToken);

    // Validate LTI claims
    this.validateLtiClaims(claims);

    // Extract launch context
    const context: LaunchContext = {
      user: {
        id: claims.sub,
        name: claims.name,
        email: claims.email,
        roles: claims['https://purl.imsglobal.org/spec/lti/claim/roles']
      },
      context: {
        id: claims['https://purl.imsglobal.org/spec/lti/claim/context'].id,
        title: claims['https://purl.imsglobal.org/spec/lti/claim/context'].title,
        type: claims['https://purl.imsglobal.org/spec/lti/claim/context'].type
      },
      resourceLink: {
        id: claims['https://purl.imsglobal.org/spec/lti/claim/resource_link'].id,
        title: claims['https://purl.imsglobal.org/spec/lti/claim/resource_link'].title
      },
      customParams: claims['https://purl.imsglobal.org/spec/lti/claim/custom']
    };

    return context;
  }

  // Assignment and Grade Services (AGS)
  async submitScore(
    lineItemUrl: string,
    userId: string,
    score: number,
    maxScore: number
  ): Promise<void> {
    const accessToken = await this.getAccessToken([
      'https://purl.imsglobal.org/spec/lti-ags/scope/score'
    ]);

    const scorePayload = {
      userId,
      scoreGiven: score,
      scoreMaximum: maxScore,
      activityProgress: 'Completed',
      gradingProgress: 'FullyGraded',
      timestamp: new Date().toISOString()
    };

    await fetch(`${lineItemUrl}/scores`, {
      method: 'POST',
      headers: {
        'Authorization': `Bearer ${accessToken}`,
        'Content-Type': 'application/vnd.ims.lis.v1.score+json'
      },
      body: JSON.stringify(scorePayload)
    });
  }
}

xAPI Implementation

The Experience API (xAPI) provides a flexible format for recording learning experiences. Unlike older standards like SCORM, xAPI can capture learning activities that occur anywhere—in classrooms, online, mobile, or simulations.

xAPI Statement Structure

xAPI statements follow the actor-verb-object pattern: "Learner did something to something." This simple structure can represent complex learning activities with rich context.

// xAPI Statement Generation for Adaptive Learning
interface XAPIStatement {
  actor: Actor;
  verb: Verb;
  object: ActivityObject;
  result?: Result;
  context?: Context;
  timestamp: string;
}

class XAPIStatementGenerator {
  private endpoint: string;
  private auth: AuthCredentials;

  // Generate statement for content completion
  generateCompletionStatement(
    learner: Learner,
    content: LearningObject,
    performance: PerformanceData
  ): XAPIStatement {
    return {
      actor: {
        objectType: 'Agent',
        name: learner.name,
        mbox: `mailto:${learner.email}`,
        account: {
          homePage: this.platformUrl,
          name: learner.id
        }
      },
      verb: {
        id: 'http://adlnet.gov/expapi/verbs/completed',
        display: { 'en-US': 'completed' }
      },
      object: {
        objectType: 'Activity',
        id: `${this.contentBaseUrl}/${content.id}`,
        definition: {
          name: { 'en-US': content.title },
          description: { 'en-US': content.description },
          type: 'http://adlnet.gov/expapi/activities/module'
        }
      },
      result: {
        completion: true,
        success: performance.passed,
        score: {
          scaled: performance.score / performance.maxScore,
          raw: performance.score,
          max: performance.maxScore,
          min: 0
        },
        duration: this.formatDuration(performance.duration)
      },
      context: {
        contextActivities: {
          parent: [{
            objectType: 'Activity',
            id: `${this.courseBaseUrl}/${content.courseId}`
          }],
          category: [{
            objectType: 'Activity',
            id: 'https://w3id.org/xapi/wia/adaptive-learning'
          }]
        },
        extensions: {
          'https://w3id.org/xapi/wia/extensions/mastery-level': performance.masteryLevel,
          'https://w3id.org/xapi/wia/extensions/adaptation-path': performance.pathId
        }
      },
      timestamp: new Date().toISOString()
    };
  }

  // Generate statement for adaptive path selection
  generatePathSelectionStatement(
    learner: Learner,
    selectedPath: LearningPath,
    reason: string
  ): XAPIStatement {
    return {
      actor: this.buildActor(learner),
      verb: {
        id: 'https://w3id.org/xapi/wia/verbs/path-selected',
        display: { 'en-US': 'was assigned learning path' }
      },
      object: {
        objectType: 'Activity',
        id: `${this.pathBaseUrl}/${selectedPath.id}`,
        definition: {
          name: { 'en-US': selectedPath.name },
          type: 'https://w3id.org/xapi/wia/activity-types/learning-path'
        }
      },
      context: {
        extensions: {
          'https://w3id.org/xapi/wia/extensions/selection-reason': reason,
          'https://w3id.org/xapi/wia/extensions/path-concepts': selectedPath.concepts,
          'https://w3id.org/xapi/wia/extensions/estimated-duration': selectedPath.estimatedDuration
        }
      },
      timestamp: new Date().toISOString()
    };
  }

  // Send statement to LRS
  async sendStatement(statement: XAPIStatement): Promise<string> {
    const response = await fetch(`${this.endpoint}/statements`, {
      method: 'POST',
      headers: {
        'Authorization': `Basic ${this.auth.encoded}`,
        'Content-Type': 'application/json',
        'X-Experience-API-Version': '1.0.3'
      },
      body: JSON.stringify(statement)
    });

    const result = await response.json();
    return result[0]; // Statement ID
  }
}

QTI for Assessment Items

Question and Test Interoperability (QTI) enables portable assessment items across different platforms. The WIA-ADAPTIVE_LEARNING standard requires QTI 3.0 support for assessment content exchange.

// QTI 3.0 Item Parser and Generator
class QTIHandler {
  // Parse QTI XML to internal format
  parseQTIItem(qtiXml: string): AssessmentItem {
    const parser = new DOMParser();
    const doc = parser.parseFromString(qtiXml, 'application/xml');

    const assessmentItem = doc.querySelector('qti-assessment-item');

    return {
      id: assessmentItem.getAttribute('identifier'),
      title: assessmentItem.getAttribute('title'),
      adaptive: assessmentItem.getAttribute('adaptive') === 'true',
      timeDependent: assessmentItem.getAttribute('time-dependent') === 'true',
      responseDeclarations: this.parseResponseDeclarations(doc),
      outcomeDeclarations: this.parseOutcomeDeclarations(doc),
      itemBody: this.parseItemBody(doc),
      responseProcessing: this.parseResponseProcessing(doc)
    };
  }

  // Generate QTI XML from internal format
  generateQTIItem(item: AssessmentItem): string {
    return `


  
    
      ${item.correctResponse}
    
  

  
    
      0
    
  

  
    ${this.generateItemBody(item)}
  

  
`;
  }
}

LMS Integration Patterns

Different integration scenarios require different approaches. The WIA-ADAPTIVE_LEARNING standard defines several integration patterns to address common use cases.

Integration Pattern Selection

  • Embedded Tool: Adaptive content displayed within LMS interface
  • Standalone with SSO: Separate application with single sign-on
  • Data Sync: Bidirectional data synchronization
  • Hybrid: Combination of patterns for different use cases
// LMS Integration Service
class LMSIntegrationService {
  constructor(
    private ltiProvider: LTI13ToolProvider,
    private xapiClient: XAPIClient,
    private rosterService: OneRosterClient
  ) {}

  // Sync roster data from LMS
  async syncRoster(courseId: string): Promise<RosterSyncResult> {
    // Get enrollments via OneRoster
    const enrollments = await this.rosterService.getEnrollmentsForClass(courseId);

    const results = {
      added: 0,
      updated: 0,
      removed: 0,
      errors: []
    };

    for (const enrollment of enrollments) {
      try {
        const learner = await this.findOrCreateLearner(enrollment.user);

        if (enrollment.status === 'active') {
          await this.enrollLearner(learner.id, courseId, enrollment.role);
          results.added++;
        } else if (enrollment.status === 'inactive') {
          await this.unenrollLearner(learner.id, courseId);
          results.removed++;
        }
      } catch (error) {
        results.errors.push({
          userId: enrollment.user.sourcedId,
          error: error.message
        });
      }
    }

    return results;
  }

  // Push grades back to LMS
  async syncGrades(
    courseId: string,
    assignmentId: string
  ): Promise<GradeSyncResult> {
    // Get all completions for assignment
    const completions = await this.getAssignmentCompletions(assignmentId);

    // Get LTI line item
    const lineItem = await this.ltiProvider.getLineItem(courseId, assignmentId);

    const results = {
      synced: 0,
      errors: []
    };

    for (const completion of completions) {
      try {
        await this.ltiProvider.submitScore(
          lineItem.url,
          completion.learnerId,
          completion.score,
          completion.maxScore
        );
        results.synced++;
      } catch (error) {
        results.errors.push({
          learnerId: completion.learnerId,
          error: error.message
        });
      }
    }

    return results;
  }

  // Handle deep linking for content selection
  async handleDeepLinkingRequest(
    launchContext: LaunchContext
  ): Promise<DeepLinkingResponse> {
    // Get available adaptive modules
    const modules = await this.getAvailableModules(launchContext.context.id);

    // Build deep linking response
    const items = modules.map(module => ({
      type: 'ltiResourceLink',
      title: module.title,
      url: `${this.toolUrl}/launch/${module.id}`,
      custom: {
        moduleId: module.id,
        adaptive: 'true'
      }
    }));

    return this.ltiProvider.createDeepLinkingResponse(items);
  }
}

Summary

Successful adaptive learning deployments depend on seamless integration with existing educational technology infrastructure. The WIA-ADAPTIVE_LEARNING standard ensures interoperability through support for industry standards like LTI, xAPI, and QTI, enabling adaptive systems to work within diverse technology ecosystems.

Integration Checklist

  • LTI 1.3 implementation for tool integration
  • xAPI for learning activity tracking
  • QTI 3.0 for assessment portability
  • OneRoster/LIS for roster synchronization
  • Grade passback via LTI AGS
  • Deep linking for content selection
