Chapter 2: Architecture and Learner Models

The architecture of an adaptive learning system determines its capabilities, scalability, and effectiveness. This chapter details the technical architecture specified by the WIA-ADAPTIVE_LEARNING standard, with particular emphasis on learner modeling—the heart of any adaptive system.

System Architecture Overview

A compliant WIA-ADAPTIVE_LEARNING system consists of multiple interconnected components that work together to deliver personalized learning experiences. The architecture follows a modular, microservices-oriented design that enables independent scaling and updating of components.

Core Components

Learner Service

The Learner Service manages all learner-related data and operations. It maintains learner profiles, tracks learning progress, and provides interfaces for querying learner state. This service is responsible for updating learner models based on interaction data and exposing learner characteristics to other components.

Learner Service Responsibilities

  • Profile management (creation, updates, deletion)
  • Knowledge state tracking and updates
  • Learning preference inference
  • Historical data management
  • Privacy-compliant data access

Adaptation Engine

The Adaptation Engine is the intelligence center of the system. It receives learner state and context information, applies adaptation algorithms, and produces personalization decisions. This component implements the core adaptive logic specified by the WIA-ADAPTIVE_LEARNING standard.

Adaptation Engine Capabilities

The engine must support multiple adaptation strategies that can be configured based on pedagogical requirements:

  • Content Selection: Choosing appropriate learning objects
  • Sequence Optimization: Ordering content for optimal learning
  • Difficulty Adjustment: Matching challenge to ability
  • Scaffolding Control: Managing support levels
  • Feedback Personalization: Tailoring response types

Learner Modeling

The learner model is a computational representation of an individual learner that captures relevant characteristics for adaptation decisions. Effective learner models balance comprehensiveness with computational tractability.

Knowledge Modeling Approaches

Knowledge modeling tracks what a learner knows and can do. The WIA-ADAPTIVE_LEARNING standard supports multiple approaches to accommodate different system requirements.

Approach Description Use Case
Overlay Model Learner knowledge as subset of expert knowledge Structured domains
Bayesian Knowledge Tracing Probabilistic model of concept mastery Skill-based learning
Deep Knowledge Tracing RNN-based temporal knowledge modeling Complex sequences
Item Response Theory Statistical model of item difficulty and learner ability Assessment systems 
// Bayesian Knowledge Tracing Implementation
class BayesianKnowledgeTracer {
  private params: BKTParams;

  constructor(params: BKTParams) {
    this.params = {
      pInit: params.pInit || 0.3,    // Initial knowledge probability
      pLearn: params.pLearn || 0.1,  // Learning rate
      pGuess: params.pGuess || 0.2,  // Guess probability
      pSlip: params.pSlip || 0.1     // Slip probability
    };
  }

  // Update knowledge probability based on observation
  update(priorKnowledge: number, isCorrect: boolean): number {
    const { pGuess, pSlip, pLearn } = this.params;

    // Calculate probability of correct response given knowledge states
    const pCorrectIfKnown = 1 - pSlip;
    const pCorrectIfUnknown = pGuess;

    // Apply Bayes' rule
    let posteriorKnown: number;

    if (isCorrect) {
      const pCorrect = priorKnowledge * pCorrectIfKnown +
                       (1 - priorKnowledge) * pCorrectIfUnknown;
      posteriorKnown = (priorKnowledge * pCorrectIfKnown) / pCorrect;
    } else {
      const pIncorrect = priorKnowledge * pSlip +
                         (1 - priorKnowledge) * (1 - pGuess);
      posteriorKnown = (priorKnowledge * pSlip) / pIncorrect;
    }

    // Account for learning
    return posteriorKnown + (1 - posteriorKnown) * pLearn;
  }

  // Predict probability of correct response
  predictCorrect(knowledge: number): number {
    const { pGuess, pSlip } = this.params;
    return knowledge * (1 - pSlip) + (1 - knowledge) * pGuess;
  }
}

Content Structure

The organization of educational content significantly impacts the effectiveness of adaptation. The WIA-ADAPTIVE_LEARNING standard specifies a hierarchical content structure with rich semantic relationships.

Knowledge Graph Structure

Content is organized around a knowledge graph that represents concepts and their relationships. This graph enables intelligent sequencing by respecting prerequisite dependencies and identifying related concepts for remediation or extension.

// Knowledge Graph Implementation
interface Concept {
  id: string;
  name: string;
  description: string;
  domain: string;
  difficulty: number;
  prerequisites: string[];
  relatedConcepts: {
    conceptId: string;
    relationshipType: 'prerequisite' | 'corequisite' | 'extension' | 'application';
    strength: number; // 0.0 to 1.0
  }[];
}

class KnowledgeGraph {
  private concepts: Map<string, Concept> = new Map();
  private adjacencyList: Map<string, Set<string>> = new Map();

  addConcept(concept: Concept): void {
    this.concepts.set(concept.id, concept);
    this.adjacencyList.set(concept.id, new Set(concept.prerequisites));
  }

  // Topological sort for sequencing
  getTopologicalOrder(): string[] {
    const visited = new Set<string>();
    const result: string[] = [];

    const visit = (id: string): void => {
      if (visited.has(id)) return;
      visited.add(id);

      const prerequisites = this.adjacencyList.get(id) || new Set();
      for (const prereq of prerequisites) {
        visit(prereq);
      }
      result.push(id);
    };

    for (const id of this.concepts.keys()) {
      visit(id);
    }

    return result;
  }

  // Find learning path between concepts
  findLearningPath(from: string, to: string): string[] {
    // BFS to find shortest path respecting prerequisites
    const queue: string[][] = [[from]];
    const visited = new Set<string>([from]);

    while (queue.length > 0) {
      const path = queue.shift()!;
      const current = path[path.length - 1];

      if (current === to) return path;

      const concept = this.concepts.get(current);
      if (!concept) continue;

      for (const related of concept.relatedConcepts) {
        if (!visited.has(related.conceptId)) {
          visited.add(related.conceptId);
          queue.push([...path, related.conceptId]);
        }
      }
    }

    return []; // No path found
  }
}

Content Metadata Requirements

All learning objects must include the following metadata for effective adaptation:

  • Learning objectives: Clear, measurable outcomes
  • Difficulty level: Calibrated difficulty rating
  • Prerequisites: Required prior knowledge
  • Estimated duration: Expected completion time
  • Content modality: Presentation format(s)

Summary

The architecture of a WIA-ADAPTIVE_LEARNING compliant system must balance sophisticated personalization capabilities with practical considerations of scalability, maintainability, and interoperability. The modular design allows organizations to adopt components incrementally while ensuring a coherent overall system.

Architecture Principles

  • Modular components with clear interfaces
  • Event-driven data flow for scalability
  • Separation of learner, content, and adaptation logic
  • Rich metadata enabling intelligent adaptation
  • Privacy-by-design in all components
