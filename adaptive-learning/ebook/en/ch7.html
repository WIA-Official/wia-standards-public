7.1 LMS Integration with LTI

Learning Tools Interoperability (LTI) Overview

LTI is the standard protocol for integrating learning tools with Learning Management Systems like Canvas, Moodle, Blackboard, and D2L. WIA-EDU-003 systems should support LTI 1.3 (the latest version) for seamless integration.

LTI Integration Pattern

Scenario: Instructor wants to embed adaptive learning modules within their LMS course

Implementation:

  1. Register adaptive learning tool in LMS as external tool
  2. Configure tool placement (assignment, module, navigation)
  3. Student clicks tool link in LMS
  4. LMS sends signed LTI launch request to adaptive system
  5. Adaptive system validates signature and creates session
  6. Student interacts with adaptive content in iframe or new window
  7. Adaptive system sends grade back to LMS via LTI Advantage services

LTI 1.3 Implementation

// LTI 1.3 Launch Handler (Node.js/Express)

const lti = require('ltijs').Provider;

// Initialize LTI provider
lti.setup(
  process.env.LTI_KEY,
  {
    url: process.env.DATABASE_URL,
    connection: { database: 'lti_database' }
  },
  {
    appRoute: '/',
    loginRoute: '/login',
    keysetRoute: '/keys',
    cookies: {
      secure: true,
      sameSite: 'None'
    }
  }
);

// Handle LTI launch
lti.onConnect(async (token, req, res) => {
  // Extract user and context information
  const userId = token.user;
  const courseId = token.platformContext.context.id;
  const roles = token.userInfo.roles;

  // Create or retrieve learner profile
  const learner = await getLearnerProfile(userId);

  // Get course-specific content and progress
  const courseProgress = await getCourseProgress(userId, courseId);

  // Render adaptive learning interface
  return res.send(renderAdaptiveInterface(learner, courseProgress));
});

// Send grade back to LMS
async function sendGrade(userId, courseId, score) {
  const gradeService = lti.Grade;
  await gradeService.ScorePublish(
    res.locals.token,
    {
      userId: userId,
      scoreGiven: score,
      scoreMaximum: 100,
      comment: 'Adaptive learning progress',
      timestamp: new Date().toISOString()
    }
  );
}

lti.deploy({ port: 3000 });

    res.locals.token,
    [resource],
    { message: 'Module added successfully' }
  );

  return res.send(form);
});

7.2 Student Information System (SIS) Integration

Roster Synchronization

Automate the process of enrolling students and updating course rosters by integrating with the institution's SIS.

SIS Integration Pattern

Common Integration Methods:

  • OneRoster API: Standard REST API for roster and grade exchange
  • CSV Import/Export: Scheduled file-based synchronization
  • SFTP: Secure file transfer for batch updates
  • Direct Database: Read-only connection to SIS database (less common)

OneRoster Implementation

// OneRoster API Client

class OneRosterClient {
  constructor(baseUrl, clientId, clientSecret) {
    this.baseUrl = baseUrl;
    this.auth = Buffer.from(`${clientId}:${clientSecret}`).toString('base64');
  }

  async getOrgs() {
    return this.request('/orgs');
  }

  async getCourses() {
    return this.request('/classes');
  }

  async getEnrollments(courseId) {
    return this.request(`/classes/${courseId}/enrollments`);
  }

  async getUsers() {
    return this.request('/users');
  }

  async request(endpoint) {
    const response = await fetch(`${this.baseUrl}${endpoint}`, {
      headers: {
        'Authorization': `Basic ${this.auth}`,
        'Accept': 'application/json'
      }
    });
    return response.json();
  }
}

// Sync enrollments
async function syncEnrollments() {
  const oneRoster = new OneRosterClient(
    process.env.ONEROSTER_URL,
    process.env.ONEROSTER_CLIENT_ID,
    process.env.ONEROSTER_SECRET
  );

  // Fetch current courses
  const courses = await oneRoster.getCourses();

  for (const course of courses.classes) {
    // Get enrollments for each course
    const enrollments = await oneRoster.getEnrollments(course.sourcedId);

    // Update local database
    await updateCourseEnrollments(course.sourcedId, enrollments.enrollments);
  }

  console.log('Enrollment sync completed');
}

7.3 Content Repository Integration

External Content Sources

Integrate adaptive learning systems with external content repositories to expand available learning materials without duplicating storage.

Common Content Repository Standards

  • OAI-PMH: Open Archives Initiative Protocol for Metadata Harvesting
  • SRU/SRW: Search/Retrieve via URL/Web Service
  • LOR APIs: Learning Object Repository custom APIs
  • YouTube/Vimeo: Video content integration
  • Khan Academy: Educational content API

OAI-PMH Harvester

# Python OAI-PMH Harvester

from sickle import Sickle

class ContentHarvester:
    def __init__(self, repository_url):
        self.sickle = Sickle(repository_url)

    def harvest_content(self, metadata_prefix='oai_dc', from_date=None):
        """Harvest learning objects from OAI-PMH repository"""
        records = self.sickle.ListRecords(
            metadataPrefix=metadata_prefix,
            from_=from_date
        )

        for record in records:
            learning_object = self.parse_record(record)
            self.store_learning_object(learning_object)

    def parse_record(self, record):
        """Extract learning object metadata"""
        metadata = record.metadata

        return {
            'id': record.header.identifier,
            'title': metadata.get('title', [''])[0],
            'description': metadata.get('description', [''])[0],
            'subject': metadata.get('subject', []),
            'type': metadata.get('type', [''])[0],
            'format': metadata.get('format', [''])[0],
            'url': metadata.get('identifier', [''])[0],
            'language': metadata.get('language', ['en'])[0],
            'lastModified': record.header.datestamp
        }

    def store_learning_object(self, obj):
        """Store in local content database"""
        # Tag for adaptive system
        obj['difficulty'] = self.estimate_difficulty(obj)
        obj['prerequisites'] = self.identify_prerequisites(obj)

        # Save to database
        db.learning_objects.insert(obj)

7.4 Authentication and SSO

SAML 2.0 Integration

Enable Single Sign-On (SSO) for institutional deployments using SAML 2.0, allowing users to access the adaptive learning system with their existing institutional credentials. 

// SAML 2.0 Service Provider (passport-saml)

const passport = require('passport');
const SamlStrategy = require('passport-saml').Strategy;

passport.use(new SamlStrategy({
    callbackUrl: process.env.SAML_CALLBACK_URL,
    entryPoint: process.env.IDP_ENTRY_POINT,
    issuer: process.env.SAML_ISSUER,
    cert: process.env.IDP_CERT,
    acceptedClockSkewMs: -1,
    identifierFormat: null
  },
  async (profile, done) => {
    // Extract user information from SAML assertion
    const user = {
      id: profile.nameID,
      email: profile.email,
      firstName: profile.firstName,
      lastName: profile.lastName,
      roles: profile.roles || ['learner']
    };

    // Create or update user in local database
    const localUser = await findOrCreateUser(user);

    return done(null, localUser);
  }
));

// SSO login route
app.get('/login/sso',
  passport.authenticate('saml', { failureRedirect: '/login/fail' })
);

// SSO callback route
app.post('/login/callback',
  passport.authenticate('saml', { failureRedirect: '/login/fail' }),
  (req, res) => {
    res.redirect('/dashboard');
  }
);

OAuth 2.0 for API Access

Implement OAuth 2.0 to allow third-party applications to access learner data with explicit consent. 

// OAuth 2.0 Authorization Server

const oauth2orize = require('oauth2orize');
const server = oauth2orize.createServer();

// Authorization endpoint
server.grant(oauth2orize.grant.code(async (client, redirectUri, user, ares, done) => {
  // Generate authorization code
  const code = await generateAuthCode(client.id, redirectUri, user.id);
  done(null, code);
}));

// Token endpoint
server.exchange(oauth2orize.exchange.code(async (client, code, redirectUri, done) => {
  // Validate authorization code
  const authCode = await validateAuthCode(code, client.id, redirectUri);
  if (!authCode) {
    return done(null, false);
  }

  // Generate access token
  const accessToken = await generateAccessToken(authCode.userId, client.id);
  const refreshToken = await generateRefreshToken(authCode.userId, client.id);

  done(null, accessToken, refreshToken, { expires_in: 3600 });
}));

7.5 Analytics Integration

xAPI Statement Forwarding

Forward learning activity statements to institutional Learning Record Stores for centralized analytics. 

// xAPI Statement Forwarder

class XAPIForwarder {
  constructor(lrsEndpoint, auth) {
    this.endpoint = lrsEndpoint;
    this.auth = auth;
  }

  async sendStatement(statement) {
    // Validate statement format
    if (!this.validateStatement(statement)) {
      throw new Error('Invalid xAPI statement');
    }

    // Add system context
    statement.context = {
      ...statement.context,
      platform: 'WIA-EDU-003 Adaptive Learning',
      extensions: {
        'http://wia.org/extensions/standard': 'WIA-EDU-003',
        'http://wia.org/extensions/version': '1.0.0'
      }
    };

    // Send to LRS
    const response = await fetch(`${this.endpoint}/statements`, {
      method: 'POST',
      headers: {
        'Authorization': this.auth,
        'Content-Type': 'application/json',
        'X-Experience-API-Version': '1.0.3'
      },
      body: JSON.stringify(statement)
    });

    return response.json();
  }

  validateStatement(statement) {
    return (
      statement.actor &&
      statement.verb &&
      statement.object &&
      statement.timestamp
    );
  }
}

// Usage
const forwarder = new XAPIForwarder(
  process.env.LRS_ENDPOINT,
  process.env.LRS_AUTH
);

// Track learning event
await forwarder.sendStatement({
  actor: {
    mbox: 'mailto:learner@example.com',
    name: 'John Doe'
  },
  verb: {
    id: 'http://adlnet.gov/expapi/verbs/completed',
    display: { 'en-US': 'completed' }
  },
  object: {
    id: 'http://example.com/activity/module-123',
    definition: {
      name: { 'en-US': 'Algebra Module 1' },
      type: 'http://adlnet.gov/expapi/activities/module'

Caliper Analytics

Implement IMS Caliper for standardized learning analytics that can be consumed by institutional analytics platforms. 

# Python Caliper Event Sender

from caliper import sensor, events, entities

# Initialize sensor
sensor_config = {
    'id': 'https://example.com/sensors/adaptive-learning',
    'endpoint': 'https://analytics.example.com/caliper'
}

sensor = sensor.Sensor(sensor_config['id'])
sensor.registerClient('default', sensor_config)

# Send navigation event
def track_navigation(user_id, from_url, to_url):
    event = events.NavigationEvent(
        actor=entities.Person(id=f'https://example.com/users/{user_id}'),
        action=events.Action.NAVIGATED_TO,
        object=entities.WebPage(id=to_url),
        target=None,
        generated=None,
        eventTime=datetime.utcnow().isoformat() + 'Z',
        edApp=entities.SoftwareApplication(id='https://example.com/adaptive-learning'),
        referrer=entities.WebPage(id=from_url)
    )

    sensor.send(event)

# Send assessment event
def track_assessment_submission(user_id, assessment_id, score):
    event = events.AssessmentEvent(
        actor=entities.Person(id=f'https://example.com/users/{user_id}'),
        action=events.Action.SUBMITTED,
        object=entities.Assessment(id=f'https://example.com/assessments/{assessment_id}'),
        generated=entities.Score(
            id=f'https://example.com/scores/{uuid.uuid4()}',
            attempt=entities.Attempt(id=f'https://example.com/attempts/{uuid.uuid4()}'),
            scoreGiven=score,
            scoreMaximum=100.0
        ),
        eventTime=datetime.utcnow().isoformat() + 'Z'
    )

    sensor.send(event)

7.6 Microservices Architecture

API Gateway Pattern

Use an API gateway to manage communication between adaptive learning system and external integrations.

API Gateway Benefits

  • Single Entry Point: Simplifies client integration
  • Authentication/Authorization: Centralized security
  • Rate Limiting: Protect backend services
  • Request Routing: Direct requests to appropriate microservices
  • Response Aggregation: Combine multiple service responses
  • Protocol Translation: Support multiple protocols (REST, GraphQL, gRPC)

Event-Driven Integration

Use message queues and event streaming for asynchronous integration that scales well. 

// Event-Driven Integration with Kafka

const { Kafka } = require('kafkajs');

const kafka = new Kafka({
  clientId: 'adaptive-learning',
  brokers: [process.env.KAFKA_BROKER]
});

const producer = kafka.producer();
const consumer = kafka.consumer({ groupId: 'integration-group' });

// Publish events
async function publishLearningEvent(event) {
  await producer.connect();
  await producer.send({
    topic: 'learning-events',
    messages: [{
      key: event.learnerId,
      value: JSON.stringify(event),
      headers: {
        'event-type': event.type,
        'timestamp': Date.now().toString()
      }
    }]
  });
}

// Subscribe to events from integrated systems
async function subscribeToExternalEvents() {
  await consumer.connect();
  await consumer.subscribe({ topic: 'external-events' });

  await consumer.run({
    eachMessage: async ({ topic, partition, message }) => {
      const event = JSON.parse(message.value.toString());

      switch (event.type) {
        case 'enrollment':
          await handleEnrollment(event);
          break;
        case 'grade-update':
          await handleGradeUpdate(event);
          break;
        case 'content-update':
          await handleContentUpdate(event);
          break;
      }
    }
  });
}
