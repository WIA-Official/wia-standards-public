Chapter 5: Data Privacy and Security

Adaptive learning systems collect and process extensive learner data to enable personalization. This data sensitivity demands robust privacy protections and security measures. This chapter details the privacy and security requirements of the WIA-ADAPTIVE_LEARNING standard, ensuring compliant systems protect learner rights while enabling effective adaptation.

Privacy Framework Overview

The WIA-ADAPTIVE_LEARNING standard adopts a privacy-by-design approach, embedding privacy protections into system architecture rather than treating them as afterthoughts. The framework aligns with major international privacy regulations including GDPR, FERPA, COPPA, and emerging global standards.

Critical Privacy Principles

  • Data Minimization: Collect only data necessary for legitimate educational purposes
  • Purpose Limitation: Use data only for stated purposes
  • Transparency: Clear communication about data practices
  • Learner Control: Rights to access, correct, and delete personal data
  • Security: Appropriate technical and organizational measures

// Privacy-Compliant Data Service
class PrivacyAwareDataService {
  constructor(
    private config: PrivacyConfig,
    private encryptionService: EncryptionService,
    private auditLogger: AuditLogger
  ) {}

  async storeLearnerData(
    learnerId: string,
    data: LearnerData,
    purpose: DataPurpose
  ): Promise<void> {
    // Validate purpose
    if (!this.isValidPurpose(purpose)) {
      throw new PrivacyViolationError('Invalid data purpose');
    }

    // Check consent
    const consent = await this.getConsent(learnerId, purpose);
    if (!consent.granted) {
      throw new ConsentRequiredError('Consent not granted for purpose');
    }

    // Minimize data
    const minimizedData = this.minimizeData(data, purpose);

    // Encrypt sensitive fields
    const encryptedData = await this.encryptSensitiveFields(minimizedData);

    // Store with metadata
    await this.dataStore.save({
      learnerId,
      data: encryptedData,
      purpose,
      consentId: consent.id,
      expiresAt: this.calculateExpiration(),
      createdAt: new Date()
    });

    // Audit log
    await this.auditLogger.log({
      action: 'DATA_STORE',
      learnerId,
      purpose,
      dataCategories: Object.keys(minimizedData)
    });
  }
}

Student Data Protection

Students, particularly minors, require enhanced protection. The WIA-ADAPTIVE_LEARNING standard specifies additional safeguards for K-12 educational contexts, ensuring compliance with regulations like COPPA and FERPA.

FERPA Compliance

The Family Educational Rights and Privacy Act (FERPA) protects student education records in the United States. Adaptive learning systems handling data from U.S. educational institutions must comply with FERPA requirements.

FERPA Requirement Implementation
Parent/Student consent for disclosure Consent management system with audit trail
Right to inspect records Data export and viewing functionality
Right to request correction Data rectification workflow
School official exception Role-based access with legitimate interest verification
Directory information policies Configurable data classification system 
// FERPA-Compliant Data Access Control
class FERPAAccessController {
  private rolePermissions: Map<Role, Permission[]>;

  async canAccess(
    requesterId: string,
    studentId: string,
    dataCategory: DataCategory
  ): Promise<AccessDecision> {
    const requesterRole = await this.getRoleForStudent(requesterId, studentId);

    switch (requesterRole) {
      case Role.PARENT:
        // Parents can access their child's records
        return this.checkParentalAccess(requesterId, studentId, dataCategory);

      case Role.STUDENT:
        // Students 18+ have full access; younger students limited
        return this.checkStudentAccess(requesterId, dataCategory);

      case Role.SCHOOL_OFFICIAL:
        // Must have legitimate educational interest
        return this.checkSchoolOfficialAccess(requesterId, studentId, dataCategory);

      case Role.THIRD_PARTY:
        // Requires consent or exception
        return this.checkThirdPartyAccess(requesterId, studentId, dataCategory);

      default:
        return { allowed: false, reason: 'Unknown role' };
    }
  }

  private async checkSchoolOfficialAccess(
    officialId: string,
    studentId: string,
    dataCategory: DataCategory
  ): Promise<AccessDecision> {
    // Verify legitimate educational interest
    const official = await this.getOfficial(officialId);
    const student = await this.getStudent(studentId);

    // Check if official has legitimate relationship
    const hasLegitimateInterest =
      official.schoolId === student.schoolId &&
      this.officialNeedsData(official.role, dataCategory);

    if (!hasLegitimateInterest) {
      return {
        allowed: false,
        reason: 'No legitimate educational interest established'
      };
    }

    // Log access for audit
    await this.logAccess(officialId, studentId, dataCategory);

    return { allowed: true };
  }
}

GDPR Compliance

The General Data Protection Regulation (GDPR) provides comprehensive data protection for EU residents. The WIA-ADAPTIVE_LEARNING standard incorporates GDPR requirements as baseline privacy protections.

GDPR Data Subject Rights

  • Right to Access: Learners can request all data held about them
  • Right to Rectification: Correction of inaccurate data
  • Right to Erasure: Deletion under certain conditions
  • Right to Portability: Export data in machine-readable format
  • Right to Object: Object to certain processing activities
  • Right to Restrict Processing: Limit how data is used
// GDPR Rights Management Service
class GDPRRightsService {
  constructor(
    private dataStore: LearnerDataStore,
    private exportService: DataExportService,
    private notificationService: NotificationService
  ) {}

  // Right to Access (Article 15)
  async handleAccessRequest(
    learnerId: string,
    requestId: string
  ): Promise<AccessResponse> {
    // Verify identity
    await this.verifyIdentity(learnerId);

    // Collect all data
    const personalData = await this.dataStore.getAllDataForLearner(learnerId);
    const processingInfo = await this.getProcessingInformation(learnerId);

    // Generate report
    const report = {
      requestId,
      generatedAt: new Date(),
      personalData,
      processingPurposes: processingInfo.purposes,
      categories: processingInfo.categories,
      recipients: processingInfo.recipients,
      retentionPeriod: processingInfo.retention,
      source: processingInfo.source,
      automatedDecisions: processingInfo.automatedDecisions
    };

    // Must respond within 30 days
    await this.notificationService.scheduleDeadline(requestId, 30);

    return report;
  }

  // Right to Erasure (Article 17)
  async handleErasureRequest(
    learnerId: string,
    requestId: string
  ): Promise<ErasureResponse> {
    // Check if erasure is required or permitted
    const assessment = await this.assessErasureRequest(learnerId);

    if (!assessment.canErase) {
      return {
        requestId,
        status: 'denied',
        reason: assessment.reason,
        legalBasis: assessment.retentionBasis
      };
    }

    // Perform erasure
    const erasedData = await this.dataStore.eraseAllData(learnerId);

    // Notify third parties
    if (assessment.notifyRecipients) {
      await this.notifyDataRecipients(learnerId, 'erasure');
    }

    // Audit log (anonymized)
    await this.logErasure(requestId);

    return {
      requestId,
      status: 'completed',
      erasedCategories: erasedData.categories,
      completedAt: new Date()
    };
  }

  // Right to Data Portability (Article 20)
  async handlePortabilityRequest(
    learnerId: string,
    format: 'json' | 'csv' | 'xml'
  ): Promise<PortabilityResponse> {
    const data = await this.dataStore.getPortableData(learnerId);

    const exportedFile = await this.exportService.export(data, format);

    return {
      format,
      downloadUrl: exportedFile.url,
      expiresAt: exportedFile.expiration,
      checksum: exportedFile.checksum
    };
  }
}

Security Architecture

Security is foundational to privacy protection. The WIA-ADAPTIVE_LEARNING standard specifies comprehensive security requirements covering data encryption, access control, and secure system design.

Encryption Requirements

// Encryption Configuration and Implementation
interface EncryptionConfig {
  // Data at rest
  atRest: {
    algorithm: 'AES-256-GCM' | 'AES-256-CBC';
    keyManagement: 'HSM' | 'KMS' | 'local';
    keyRotationPeriod: number; // days
  };

  // Data in transit
  inTransit: {
    tlsVersion: '1.2' | '1.3';
    cipherSuites: string[];
    certificatePinning: boolean;
  };

  // Field-level encryption
  fieldLevel: {
    enabled: boolean;
    sensitiveFields: string[];
    searchableEncryption: boolean;
  };
}

class EncryptionService {
  private keyManager: KeyManagementService;

  async encryptSensitiveData(
    data: Record<string, any>,
    sensitiveFields: string[]
  ): Promise<EncryptedData> {
    const encryptedData = { ...data };
    const keyId = await this.keyManager.getCurrentKeyId();

    for (const field of sensitiveFields) {
      if (data[field] !== undefined) {
        const encrypted = await this.encrypt(
          JSON.stringify(data[field]),
          keyId
        );
        encryptedData[field] = {
          ciphertext: encrypted.ciphertext,
          iv: encrypted.iv,
          keyId: keyId,
          algorithm: 'AES-256-GCM'
        };
      }
    }

    return {
      data: encryptedData,
      encryptedFields: sensitiveFields.filter(f => data[f] !== undefined),
      keyVersion: keyId
    };
  }

  private async encrypt(
    plaintext: string,
    keyId: string
  ): Promise<{ ciphertext: string; iv: string }> {
    const key = await this.keyManager.getKey(keyId);
    const iv = crypto.randomBytes(12);

    const cipher = crypto.createCipheriv('aes-256-gcm', key, iv);
    let ciphertext = cipher.update(plaintext, 'utf8', 'base64');
    ciphertext += cipher.final('base64');

    const authTag = cipher.getAuthTag();

    return {
      ciphertext: ciphertext + '.' + authTag.toString('base64'),
      iv: iv.toString('base64')
    };
  }
}

Security Best Practices

  • Use strong encryption (AES-256) for all sensitive data
  • Implement proper key management with regular rotation
  • Enforce TLS 1.2+ for all network communications
  • Apply principle of least privilege for access control
  • Conduct regular security audits and penetration testing
  • Maintain comprehensive audit logs

Consent Management

Obtaining and managing consent is critical for lawful data processing. The WIA-ADAPTIVE_LEARNING standard specifies a comprehensive consent management system.

// Consent Management System
interface ConsentRecord {
  id: string;
  learnerId: string;
  purposes: ConsentPurpose[];
  grantedAt: Date;
  expiresAt?: Date;
  method: 'explicit' | 'implicit' | 'parental';
  version: string; // Privacy policy version
  withdrawable: boolean;
}

class ConsentManager {
  async requestConsent(
    learnerId: string,
    purposes: ConsentPurpose[]
  ): Promise<ConsentRequest> {
    // Generate consent form with clear language
    const form = await this.generateConsentForm(purposes);

    return {
      requestId: generateId(),
      learnerId,
      form,
      purposes,
      expiresAt: addDays(new Date(), 7)
    };
  }

  async recordConsent(
    requestId: string,
    decisions: Map<ConsentPurpose, boolean>
  ): Promise<ConsentRecord> {
    const request = await this.getRequest(requestId);

    const grantedPurposes = Array.from(decisions.entries())
      .filter(([_, granted]) => granted)
      .map(([purpose]) => purpose);

    const record: ConsentRecord = {
      id: generateId(),
      learnerId: request.learnerId,
      purposes: grantedPurposes,
      grantedAt: new Date(),
      method: 'explicit',
      version: this.currentPolicyVersion,
      withdrawable: true
    };

    await this.saveConsent(record);

    return record;
  }

  async withdrawConsent(
    learnerId: string,
    purposes: ConsentPurpose[]
  ): Promise<void> {
    // Mark consent as withdrawn
    await this.updateConsentStatus(learnerId, purposes, 'withdrawn');

    // Stop processing for withdrawn purposes
    await this.notifyProcessingServices(learnerId, purposes, 'stop');

    // Schedule data deletion if required
    await this.scheduleDataCleanup(learnerId, purposes);
  }
}

Summary

Privacy and security are non-negotiable aspects of adaptive learning systems. The WIA-ADAPTIVE_LEARNING standard provides a comprehensive framework for protecting learner data while enabling effective personalization. Compliance with this framework ensures systems meet regulatory requirements and earn learner trust.

Privacy and Security Checklist

  • Privacy-by-design embedded in architecture
  • Regulatory compliance (GDPR, FERPA, COPPA)
  • Robust consent management
  • Strong encryption for data at rest and in transit
  • Comprehensive access control
  • Data subject rights implementation
  • Regular security audits
