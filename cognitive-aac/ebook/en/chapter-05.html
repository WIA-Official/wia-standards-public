WIA 인지 AAC 표준은 인지 능력에 관계없이 모든 사람이 동등하게 의사소통할 수 있는 기술 생태계를 구축합니다.

Version: 1.0.0 Date: 2025-01-14 Status: Draft

철학

홍익인간 (弘益人間) - 널리 인간을 이롭게 하라

1. 개요

1.1 목적

본 명세서는 인지장애를 가진 AAC 사용자의 인지 능력을 표준화된 형식으로 표현하기 위한 데이터 구조를 정의합니다.

1.2 범위

  • 핵심 인지 영역 정의
  • 수준 체계 정의
  • AAC 적응 매개변수 정의
  • 데이터 교환 형식 (JSON Schema)

1.3 용어 정의

용어 정의
CognitiveProfile 개인의 인지 능력을 구조화한 데이터 객체
CognitiveLevel 1-5 척도의 인지 능력 수준
AAC Adaptation 인지 프로파일 기반 AAC 인터페이스 조정

2. 인지 영역 모델

2.1 핵심 인지 영역

CognitiveDomains
├── Memory (기억)
├── Attention (주의)
├── Language (언어)
├── ExecutiveFunction (실행 기능)
├── VisualSpatial (시공간)
└── SocialCognition (사회 인지)

2.2 영역별 상세 정의

2.2.1 기억 영역 (Memory Domain)

하위 영역 설명 측정 지표
Immediate Recall 초 단위 즉시 회상 기억 폭 (span)
Working Memory 정보 조작 능력 복잡도 수준
Long-term Episodic 개인 경험 기억 회상 정확도
Long-term Semantic 사실/개념 기억 지식 접근성
Long-term Procedural 기술/절차 기억 자동화 수준 
interface MemoryDomain {
  immediateRecall: {
    level: CognitiveLevel;     // 1-5
    span: number;              // 기억 폭 (항목 수, 일반적 7±2)
    assessmentDate: string;    // ISO 8601
  };

  workingMemory: {
    level: CognitiveLevel;
    complexity: number;        // 처리 가능 복잡도 (1-10)
  };

  longTermMemory: {
    episodic: CognitiveLevel;
    semantic: CognitiveLevel;
    procedural: CognitiveLevel;
  };
}

2.2.2 주의 영역 (Attention Domain)

하위 영역 설명 측정 지표
Sustained 지속 주의 집중 가능 시간 (분)
Selective 선택적 주의 방해 저항도
Divided 분할 주의 동시 과제 수
Shifting 주의 전환 전환 시간 (초) 
interface AttentionDomain {
  sustained: {
    level: CognitiveLevel;
    durationMinutes: number;   // 최대 집중 시간
  };

  selective: {
    level: CognitiveLevel;
    distractibility: number;   // 1-10 (높을수록 방해받기 쉬움)
  };

  divided: {
    level: CognitiveLevel;
    maxTasks: number;          // 동시 처리 가능 과제 수
  };

  shifting: {
    level: CognitiveLevel;
    transitionTimeSeconds: number;  // 전환에 필요한 시간
  };
}

2.2.3 언어 영역 (Language Domain)

하위 영역 설명 측정 지표
Receptive 수용 언어 이해 복잡도
Expressive 표현 언어 표현 복잡도
Naming 이름대기 정확도/속도
Repetition 따라말하기 길이/정확도 
interface LanguageDomain {
  receptive: {
    level: CognitiveLevel;
    complexityLevel: number;   // 이해 가능한 문장 복잡도
  };

  expressive: {
    level: CognitiveLevel;
    complexityLevel: number;   // 표현 가능한 문장 복잡도
  };

  naming: {
    level: CognitiveLevel;
    accuracy: number;          // 0-100%
  };

  repetition: {
    level: CognitiveLevel;
    maxLength: number;         // 따라말할 수 있는 최대 길이
  };
}

2.2.4 실행 기능 영역 (Executive Function Domain)

하위 영역 설명 측정 지표
Planning 계획 수립 단계 수
Inhibition 억제 오류율
Flexibility 인지 유연성 적응 속도
Problem Solving 문제 해결 복잡도 
interface ExecutiveFunctionDomain {
  planning: {
    level: CognitiveLevel;
    maxSteps: number;          // 계획 가능한 최대 단계
  };

  inhibition: {
    level: CognitiveLevel;
    errorRate: number;         // 억제 실패율 (0-100%)
  };

  flexibility: {
    level: CognitiveLevel;
    adaptationSpeed: number;   // 1-10
  };

  problemSolving: {
    level: CognitiveLevel;
    maxComplexity: number;     // 해결 가능한 문제 복잡도
  };
}

2.2.5 시공간 영역 (Visual-Spatial Domain)

interface VisualSpatialDomain {
  visualPerception: {
    level: CognitiveLevel;
    acuity: string;            // 예: "20/20"
  };

  spatialOrientation: {
    level: CognitiveLevel;
  };

  construction: {
    level: CognitiveLevel;     // 구성 능력
  };
}

2.2.6 사회 인지 영역 (Social Cognition Domain)

interface SocialCognitionDomain {
  emotionRecognition: {
    level: CognitiveLevel;
    modalities: ('face' | 'voice' | 'body')[];
  };

  theoryOfMind: {
    level: CognitiveLevel;     // 타인의 마음 이해
  };

  socialJudgment: {
    level: CognitiveLevel;     // 사회적 판단
  };
}

3. 인지 수준 척도

3.1 CognitiveLevel 정의

수준 값 레이블 설명 AAC 복잡도 권장
PROFOUND 1 심각한 손상 기능 거의 불가 최소 (2-4 항목)
SEVERE 2 중증 손상 심각한 제한 매우 낮음 (4-6 항목)
MODERATE 3 중등도 손상 상당한 지원 필요 낮음 (6-9 항목)
MILD 4 경미한 손상 약간의 지원 필요 중간 (9-16 항목)
TYPICAL 5 정상 범위 지원 불필요 표준 (16+ 항목)

3.2 수준 결정 기준

enum CognitiveLevel {
  PROFOUND = 1,    // < 1st percentile
  SEVERE = 2,      // 1st - 5th percentile
  MODERATE = 3,    // 5th - 16th percentile
  MILD = 4,        // 16th - 25th percentile
  TYPICAL = 5      // > 25th percentile
}

4. 통합 인지 프로파일

4.1 CognitiveProfile 구조

interface CognitiveProfile {
  // 메타데이터
  id: string;                      // UUID
  version: string;                 // 스키마 버전
  createdAt: string;               // ISO 8601
  updatedAt: string;               // ISO 8601

  // 대상자 정보
  subject: {
    id: string;                    // 익명화된 ID
    birthYear?: number;            // 출생년도 (선택)
    primaryDiagnosis?: string;     // 주 진단명 (선택)
  };

  // 인지 영역
  domains: {
    memory: MemoryDomain;
    attention: AttentionDomain;
    language: LanguageDomain;
    executive: ExecutiveFunctionDomain;
    visualSpatial: VisualSpatialDomain;
    socialCognition: SocialCognitionDomain;
  };

  // 전체 요약
  summary: {
    overallLevel: CognitiveLevel;  // 종합 수준
    strengths: string[];           // 강점 영역
    challenges: string[];          // 도전 영역
  };

  // AAC 적응 권장사항
  aacRecommendations: AACAdaptation;

  // 평가 정보
  assessment: {
    tool: string;                  // 사용된 평가 도구
    assessor?: string;             // 평가자 (선택)
    date: string;                  // 평가 일자
    notes?: string;                // 추가 메모
  };
}

4.2 AAC 적응 권장사항

interface AACAdaptation {
  // 인터페이스 복잡도
  interface: {
    maxItemsPerScreen: number;     // 화면당 최대 항목 수
    layoutType: 'grid' | 'list' | 'scene';
    iconSize: 'large' | 'medium' | 'small';
    animationsEnabled: boolean;
  };

  // 상징 유형
  symbols: {
    preferredType: 'photos' | 'pictograms' | 'text' | 'mixed';
    abstractionLevel: 1 | 2 | 3;   // 1=구체적, 3=추상적
    labelPosition: 'above' | 'below' | 'none';
  };

  // 입력 방식
  input: {
    methods: ('touch' | 'switch' | 'eyeGaze' | 'voice')[];
    dwellTimeMs?: number;          // 응시 시간 (시선 추적)
    scanSpeed?: number;            // 스캔 속도 (스위치)
  };

  // 피드백
  feedback: {
    auditory: boolean;
    visual: boolean;
    haptic: boolean;
    confirmationRequired: boolean;
  };

  // 진행 속도
  pacing: {
    autoAdvance: boolean;
    delayBetweenItemsMs: number;
    timeoutEnabled: boolean;
    timeoutMs?: number;
  };
}

5. 프로파일 생성 및 업데이트

5.1 초기 프로파일 생성

function createCognitiveProfile(
  subjectId: string,
  assessmentData: AssessmentData
): CognitiveProfile {
  return {
    id: generateUUID(),
    version: "1.0.0",
    createdAt: new Date().toISOString(),
    updatedAt: new Date().toISOString(),
    subject: {
      id: subjectId,
    },
    domains: mapAssessmentToDomains(assessmentData),
    summary: calculateSummary(assessmentData),
    aacRecommendations: generateRecommendations(assessmentData),
    assessment: {
      tool: assessmentData.toolName,
      date: assessmentData.date,
    }
  };
}

5.2 프로파일 업데이트

function updateCognitiveProfile(
  existing: CognitiveProfile,
  newAssessment: AssessmentData
): CognitiveProfile {
  return {
    ...existing,
    updatedAt: new Date().toISOString(),
    domains: mergeDomains(existing.domains, mapAssessmentToDomains(newAssessment)),
    summary: calculateSummary(newAssessment),
    aacRecommendations: generateRecommendations(newAssessment),
    assessment: {
      tool: newAssessment.toolName,
      date: newAssessment.date,
    }
  };
}

6. 프라이버시 및 보안

6.1 데이터 민감성

인지 프로파일은 민감한 건강 정보로 분류되며, 다음 원칙을 따라야 합니다:

원칙 설명
최소화 필요한 정보만 수집
익명화 직접 식별 정보 제거
암호화 전송/저장 시 암호화
동의 명시적 동의 획득
접근 제어 권한 기반 접근

6.2 동의 프로토콜

interface ConsentRecord {
  profileId: string;
  consentGivenBy: 'self' | 'guardian' | 'legal_representative';
  consentType: 'full' | 'limited' | 'research_only';
  consentDate: string;
  expirationDate?: string;
  withdrawalDate?: string;
}

6.3 대리 결정

인지 능력이 심각하게 저하된 경우, 다음 순서로 대리 동의를 구합니다:

  1. 법정 대리인 (Legal Guardian)
  2. 의료 대리인 (Healthcare Proxy)
  3. 가까운 가족 구성원 (Next of Kin)
  4. 윤리 위원회 결정 (Ethics Committee)

8. 확장

8.1 조건 특화 프로파일

  • AutismProfile: 자폐 스펙트럼 특화 (별도 명세서)
  • DementiaProfile: 치매 특화 (별도 명세서)
  • TBIProfile: 외상성 뇌손상 특화 (향후)
  • StrokeProfile: 뇌졸중 특화 (향후)

8.2 확장 예시

interface AutismProfile extends CognitiveProfile {

  };
}

1. 연구 개요

1.1 목적

인지장애(자폐 스펙트럼, 치매, 지적장애)를 가진 사용자를 위한 AAC 시스템의 효과적 적용을 위해 기존 평가 도구와 인지 프로파일 표준을 조사하고 분석합니다.

1.2 조사 범위

  • 자폐 스펙트럼 진단 도구 (ADOS-2)
  • 적응 행동 평가 도구 (Vineland-3)
  • 치매 선별 도구 (MMSE, MoCA)
  • 의사소통 평가 도구 (Communication Matrix)

2. 자폐 스펙트럼 진단: ADOS-2

2.1 개요

ADOS-2 (Autism Diagnostic Observation Schedule, Second Edition)는 자폐 스펙트럼 장애(ASD) 진단을 위한 표준 도구로, “골드 스탠다드”로 인정받고 있습니다.

2.2 구조

모듈 대상 특징
Toddler 12-30개월 영아 전용
Module 1 31개월+ (비언어적) 단어 없음~단어 일부
Module 2 구절 사용자 문장 미사용
Module 3 언어적 아동/청소년 유창한 언어
Module 4 언어적 성인 성인 전용

2.3 평가 영역

ADOS-2 평가 영역:
├── 사회적 상호작용 (Social Interaction)
│   ├── 사회적 미소
│   ├── 눈맞춤
│   ├── 공동 주의
│   └── 사회적 상호성
├── 의사소통 (Communication)
│   ├── 언어적 의사소통
│   ├── 비언어적 의사소통
│   └── 대화 기술
└── 제한적/반복적 행동 (RRBs)
    ├── 반복적 행동
    ├── 감각 관심사
    └── 루틴 고집

2.4 심리측정적 특성

  • 민감도: 83-91%
  • 특이도: 86-94%
  • 평가자 간 신뢰도: 78-98%

2.5 AAC 적용 시사점

  • 비언어적/최소 언어 사용자를 위한 모듈 존재 (Module 1, 2)
  • 의사소통 능력과 사회적 상호작용 동시 평가
  • AAC 기기 사용자에 대한 별도 지침 필요

3. 적응 행동 평가: Vineland-3

3.1 개요

Vineland Adaptive Behavior Scales, Third Edition (Vineland-3)는 일상생활에 필요한 개인적, 사회적 기술을 평가하는 표준화된 도구입니다.

3.2 평가 영역

영역 하위 영역 설명
의사소통 수용 언어 정보 이해 및 반응
표현 언어 언어적 표현
문자 언어 읽기/쓰기
일상생활 개인 자기 관리
가정 가사 활동
지역사회 사회 활동
사회화 대인 관계 타인과의 관계
놀이/여가 여가 활동
대처 기술 적응 기술
운동 기술 대근육 전체적 움직임
(선택) 소근육 정밀한 움직임

3.3 연령 범위

  • 출생~90세 이상
  • 자폐, 지적장애, 발달장애 진단에 활용

3.4 자폐 스펙트럼 표준 점수 (10세 이상, 무발화)

영역 평균 표준점수
의사소통 32.82
일상생활 29.94
사회화 33.57

3.5 AAC 적용 시사점

  • 표현 언어 하위영역은 주로 구어에 초점
  • AAC 사용자의 의사소통 행동 점수화 시 해석 필요
  • 적응 행동 전반 평가로 AAC 개입 목표 설정 가능

4. 치매 인지 선별: MMSE vs MoCA

4.1 MMSE (Mini-Mental State Examination)

개발

  • 1975년 Marshal F. Folstein 개발
  • 역사적으로 가장 널리 사용된 선별 도구

평가 영역

MMSE 평가 영역 (총 30점):
├── 지남력 (10점)
│   ├── 시간 지남력 (5점)
│   └── 장소 지남력 (5점)
├── 기억력 (6점)
│   ├── 즉시 회상 (3점)
│   └── 지연 회상 (3점)
├── 주의력 및 계산 (5점)
├── 언어 (8점)
│   ├── 명명 (2점)
│   ├── 따라말하기 (1점)
│   ├── 3단계 명령 (3점)
│   ├── 읽기 (1점)
│   └── 쓰기 (1점)
└── 시공간 능력 (1점)

절단점

  • 24-30점: 정상
  • 18-23점: 경도 인지장애
  • 0-17점: 중증 인지장애

제한점

  • 경도 인지장애(MCI) 민감도 낮음 (18-25%)
  • 천장 효과 (ceiling effect) 존재
  • 실행 기능 평가 부재

4.2 MoCA (Montreal Cognitive Assessment)

개발

  • 2005년 Ziad Nasreddine 개발
  • MMSE의 한계 극복 목적

평가 영역

MoCA 평가 영역 (총 30점):
├── 시공간/실행 기능 (5점)
│   ├── Trail Making B
│   ├── 시계 그리기
│   └── 도형 복사
├── 명명 (3점)
├── 기억력 (5점 - 지연 회상)
├── 주의력 (6점)
│   ├── 숫자 폭
│   ├── 집중력
│   └── 연속 빼기
├── 언어 (3점)
│   ├── 따라말하기
│   └── 유창성
├── 추상화 (2점)
└── 지남력 (6점)

절단점

  • 26점 이상: 정상
  • 학력 12년 미만 시 +1점 보정

MoCA 우수성

메트릭 MoCA MMSE
MCI 민감도 90-100% 18-25%
경도 알츠하이머 민감도 100% 78%
실행 기능 평가 포함 미포함
천장 효과 없음 있음

4.3 질환별 권장 도구

질환 권장 도구 이유
경도 인지장애 (MCI) MoCA 높은 민감도
전두측두엽 치매 (FTD) MoCA AUC 0.87 vs 0.80
파킨슨병 인지장애 MoCA 천장 효과 없음
중등도-중증 치매 MMSE 충분한 민감도

4.4 AAC 적용 시사점

  • 의사소통 능력에 따라 평가 도구 수정 필요
  • 비언어적 과제 비중 고려
  • 인지 프로파일과 AAC 복잡도 매칭

5. 의사소통 평가: Communication Matrix

5.1 개요

Communication Matrix는 Charity Rowland와 Philip Schweigert가 개발한 무료 평가 도구로, 초기 의사소통 단계에 있는 모든 연령대를 대상으로 합니다.

5.2 7단계 의사소통 발달

레벨 단계 설명
I 선의도적 행동 반사적, 상태 반영
II 의도적 행동 목적 있으나 의사소통 아님
III 비관습적 의사소통 의도적 + 비관습적 행동
IV 관습적 의사소통 관습적 제스처, 발성
V 구체적 상징 그림, 사물 상징
VI 추상적 상징 수어, 단어
VII 언어 2-3 상징 조합

5.3 의사소통 기능

Communication Matrix가 평가하는 4가지 기능:
├── 거부 (Refusing)
│   └── 원하지 않는 것 거부
├── 획득 (Obtaining)
│   └── 원하는 것 요청
├── 사회적 상호작용 (Social Interaction)
│   └── 인사, 관심 끌기
└── 정보 제공 (Providing Information)
    └── 사물/사건 언급, 질문 답변

5.4 지원 양식

  • 구어 (Speech)
  • 제스처/바디랭귀지 (Gestures)
  • 소리 (Sounds)
  • 눈 응시 (Eye Gaze)
  • 터치/조작 (Touch/Manipulation)
  • 3D 상징 (3D Symbols)
  • 2D 상징 (Pictures/Photos)
  • 추상적 상징 (Abstract Symbols)
  • 문자 (Written Words)

5.5 핵심 원리

“Communication Matrix는 의사소통을 측정하지, 인지 능력을 측정하지 않습니다. 의사소통 부재가 지능 부재를 의미하지 않습니다.”

5.6 AAC 적용 시사점

  • AAC 사용자의 모든 의사소통 형태 수용
  • 발달 단계 기반 개입 계획 수립
  • 무료로 접근 가능 (communicationmatrix.org)

9. 결론

본 리서치를 통해 다음을 확인했습니다:

  1. 기존 평가 도구는 인지 영역을 다각적으로 측정하지만, AAC 사용자를 위한 통합 프로파일 표준은 부재합니다.

  2. 자폐와 치매는 서로 다른 인지 특성을 가지며, 각각에 맞는 특화된 AAC 설계 원칙이 필요합니다.

  3. Communication Matrix는 AAC 친화적 평가 도구로, 인지 AAC 표준의 기반으로 활용 가능합니다.

  4. WIA Cognitive AAC 표준은 이러한 기존 도구들의 장점을 통합하면서, 상호운용 가능한 디지털 표준을 제공해야 합니다.
