유전체 연구 데이터

장 개요: 이 장에서는 차세대 염기서열분석(NGS) 데이터, VCF 포맷, 유전자 발현 분석, 변이 주석, 개인정보 보호 및 유전체 데이터 공유를 다룹니다.

유전체학의 혁명

2003년 인간 게놈 프로젝트 완료는 현대 의학의 전환점이 되었습니다. 30억 달러와 13년이 걸린 최초의 전장 유전체 시퀀싱 비용은 2025년 현재 1,000달러 이하로 떨어졌고, 단 몇 시간 만에 완료됩니다. 이러한 기술 발전은 정밀 의료, 맞춤 치료, 희귀 질환 진단을 현실로 만들었습니다.

유전체 연구는 단일 유전자 질환에서 복합 질환, 암 유전체학, 약물유전체학, 마이크로바이옴까지 확장되었습니다. 하지만 이와 함께 방대한 데이터 관리, 분석, 해석, 공유의 과제가 생겼습니다. 한 사람의 전장 유전체 시퀀싱은 약 200GB의 원시 데이터를 생성하며, 수만 명을 대상으로 한 연구는 페타바이트 규모의 데이터를 다룹니다.

차세대 염기서열분석(NGS)

NGS(Next-Generation Sequencing)는 수백만~수십억 개의 DNA 조각을 동시에 시퀀싱하는 고처리량 기술입니다. 주요 NGS 플랫폼은 다음과 같습니다:

플랫폼 기술 읽기 길이 처리량
Illumina NovaSeq 합성에 의한 시퀀싱(SBS) 150-300 bp 최대 6 Tb/run
Oxford Nanopore 나노포어 시퀀싱 >100 kb (long-read) 최대 2 Tb/run
PacBio Sequel II 단일 분자 실시간(SMRT) 10-30 kb 최대 400 Gb/run
MGI DNBSEQ DNA 나노볼 시퀀싱 100-300 bp 최대 6 Tb/run

NGS 워크플로

  1. DNA 추출: 혈액, 조직, 타액에서 고품질 DNA 추출
  2. 라이브러리 제작: DNA를 조각내고 어댑터 서열 부착
  3. 시퀀싱: NGS 플랫폼에서 염기서열 읽기
  4. 베이스 콜링: 형광 신호를 염기서열(ATCG)로 변환
  5. 품질 관리: 읽기 품질 평가, 낮은 품질 제거
  6. 정렬(Alignment): 참조 유전체에 읽기 매핑
  7. 변이 호출(Variant Calling): 참조와 다른 변이 식별
  8. 주석(Annotation): 변이의 기능적 영향 예측

유전체 데이터 포맷

유전체 데이터는 여러 표준 포맷으로 저장됩니다. 각 포맷은 분석 파이프라인의 특정 단계에 사용됩니다.

1. FASTQ - 원시 시퀀싱 데이터

FASTQ는 NGS의 원시 출력 형식으로, 각 읽기의 염기서열과 품질 점수를 저장합니다.

FASTQ 포맷 예시
@SEQ_ID Description
GATTTGGGGTTCAAAGCAGTATCGATCAAATAGTAAATCCATTTGTTCAACTCACAGTTT
+
!''*((((***+))%%%++)(%%%%).1***-+*''))**55CCF>>>>>>CCCCCCC65

각 읽기는 4줄로 구성됩니다: (1) 읽기 ID, (2) 염기서열, (3) 구분자 '+', (4) 품질 점수 (Phred 스케일, ASCII 인코딩).

2. SAM/BAM - 정렬 데이터

SAM(Sequence Alignment/Map)은 참조 유전체에 정렬된 읽기를 저장하는 텍스트 형식입니다. BAM은 SAM의 압축 이진 버전으로, 디스크 공간을 절약하고 처리 속도가 빠릅니다.

필드 설명
QNAME 읽기 이름
FLAG 정렬 정보 비트 플래그
RNAME 참조 염색체 이름
POS 정렬 시작 위치
MAPQ 매핑 품질
CIGAR 정렬 상세 정보 (매치, 삽입, 삭제)
SEQ 읽기 염기서열
QUAL 염기 품질 점수

3. VCF - 변이 데이터

VCF(Variant Call Format)는 유전 변이를 저장하는 표준 포맷입니다. 단일염기변이(SNV), 삽입/삭제(Indel), 구조적 변이를 포함합니다.

VCF 포맷 예시
##fileformat=VCFv4.2
##reference=GRCh38
##INFO=<ID=DP,Number=1,Type=Integer,Description="Total Depth">
##FORMAT=<ID=GT,Number=1,Type=String,Description="Genotype">
#CHROM  POS     ID      REF  ALT   QUAL  FILTER  INFO         FORMAT  SAMPLE1
chr1    12345   rs123   A    G     99.9  PASS    DP=50        GT:DP   0/1:30
chr2    67890   .       CT   C     85.2  PASS    DP=42        GT:DP   1/1:25

VCF의 주요 필드:

  • CHROM: 염색체 번호
  • POS: 변이 위치
  • ID: dbSNP ID (있는 경우)
  • REF: 참조 염기서열
  • ALT: 대립 염기서열 (변이)
  • QUAL: 변이 호출 품질
  • FILTER: 필터 상태 (PASS, FAIL 등)
  • INFO: 변이 메타데이터 (깊이, 빈도 등)
  • FORMAT/SAMPLE: 샘플별 유전자형 정보

변이 주석(Variant Annotation)

변이 호출 후 각 변이의 기능적 영향을 예측하는 주석 과정이 필요합니다.

주석 도구

  • VEP (Variant Effect Predictor): Ensembl의 변이 효과 예측 도구
  • ANNOVAR: 다양한 데이터베이스를 통합한 주석 도구
  • SnpEff: 변이의 유전자/전사체 영향 예측
  • ClinVar: 임상적으로 의미 있는 변이 데이터베이스
  • gnomAD: 인구집단 빈도 데이터베이스

주석 정보

범주 정보
유전자 정보 유전자 이름, 전사체 ID, 엑손 번호
변이 타입 동의(synonymous), 미스센스, 넌센스, 프레임시프트
기능 예측 SIFT, PolyPhen, CADD 점수
인구 빈도 gnomAD, 1000 Genomes, ExAC 빈도
질병 연관성 ClinVar, OMIM, GWAS catalog
보존성 PhyloP, GERP 점수

유전자 발현 분석

RNA 시퀀싱(RNA-seq)은 유전자 발현 수준을 전장 유전체 규모로 측정합니다.

RNA-seq 분석 파이프라인

  1. RNA 추출 및 품질 확인: RIN(RNA Integrity Number) > 7 권장
  2. 라이브러리 제작: mRNA 선택 또는 rRNA 제거
  3. 시퀀싱: 단일-엔드 또는 페어-엔드 시퀀싱
  4. 품질 관리: FastQC로 읽기 품질 평가
  5. 정렬: STAR, HISAT2로 참조 전사체에 정렬
  6. 정량화: HTSeq, featureCounts로 유전자/전사체 수준 카운트
  7. 정규화: TPM, FPKM, DESeq2 정규화
  8. 차등 발현 분석: DESeq2, edgeR로 조건 간 비교

발현 단위

  • Raw Counts: 각 유전자에 매핑된 읽기 수
  • RPKM/FPKM: Reads/Fragments Per Kilobase per Million mapped reads
  • TPM: Transcripts Per Million (샘플 간 비교에 더 적합)
  • DESeq2 정규화: 통계적으로 더 robust한 정규화 방법

유전체 데이터 저장 및 공유

유전체 데이터의 크기와 민감성 때문에 특별한 저장 및 공유 전략이 필요합니다.

주요 유전체 데이터 저장소

저장소 운영 접근 수준
dbGaP 미국 NCBI 통제 접근
EGA 유럽 EBI 통제 접근
SRA 미국 NCBI 공개
ENA 유럽 EBI 공개
KBGD 한국 국립보건연구원 통제 접근

데이터 계층 모델

GA4GH(Global Alliance for Genomics and Health)는 유전체 데이터를 세 계층으로 분류합니다:

  1. Open Access: 익명화된 요약 데이터 (예: 인구 빈도, GWAS 결과)
  2. Registered Access: 연구자 등록 후 집계 데이터 접근
  3. Controlled Access: IRB 승인 및 DAC(Data Access Committee) 승인 후 개인 수준 데이터 접근

유전체 데이터 프라이버시

유전체 데이터는 매우 민감하고 개인 식별 가능성이 높아 특별한 보호가 필요합니다.

재식별 위험

전통적인 익명화 기법(이름, 주소 제거)만으로는 유전체 데이터를 충분히 보호할 수 없습니다:

  • 유전체 지문: 30-80개 SNP만으로도 개인을 고유하게 식별 가능
  • 가족 연결: 가족 구성원의 유전체 데이터를 통해 재식별 가능
  • 계통 데이터베이스: GEDmatch 같은 공개 계통 DB와 교차 참조로 식별 가능

프라이버시 보호 기법

  1. 접근 제어: DAC를 통한 엄격한 접근 승인 프로세스
  2. 안전한 환경: 데이터 다운로드 금지, 클라우드 분석 환경 제공
  3. 암호화: 전송 및 저장 중 데이터 암호화
  4. 감사 추적: 모든 데이터 접근 및 분석 활동 기록
  5. 사용 제한: 승인된 연구 목적에만 사용, MTA/DUA 체결
  6. 차등 프라이버시: 집계 통계에 노이즈 추가하여 개인 보호

약물유전체학(Pharmacogenomics)

약물유전체학은 유전 변이가 약물 반응에 미치는 영향을 연구합니다. 맞춤 약물 치료의 기초가 됩니다.

주요 PGx 유전자

유전자 기능 관련 약물
CYP2D6 약물 대사 효소 코데인, 타목시펜, 항우울제
CYP2C19 약물 대사 효소 클로피도그렐, 프로톤펌프억제제
TPMT 면역억제제 대사 아자티오프린, 6-MP
SLCO1B1 약물 수송체 스타틴
HLA-B*57:01 면역 반응 아바카비르 (과민반응 예측)

FDA는 현재 400개 이상의 약물에 대해 약물유전체 정보를 라벨에 포함하도록 요구하고 있습니다.

미래 전망

유전체 의학은 계속 진화하고 있습니다:

  • 전장 유전체 시퀀싱 보편화: 출생 시 유전체 시퀀싱이 표준이 될 가능성
  • 액체 생검: 혈액에서 순환 종양 DNA(ctDNA) 검출로 암 조기 진단
  • 단일 세포 시퀀싱: 개별 세포 수준의 유전체/전사체 분석
  • 장기 읽기 시퀀싱: 구조적 변이와 반복 서열의 정확한 해독
  • 다중 오믹스 통합: 유전체, 전사체, 프로테옴, 메타볼롬 데이터 통합 분석
핵심 요약:

차세대 염기서열분석(NGS)은 유전체 연구를 혁신했으며, 비용 감소와 처리량 증가로 대규모 코호트 연구가 가능해졌습니다. FASTQ, SAM/BAM, VCF 같은 표준 포맷을 사용하여 데이터를 저장하고 교환합니다. 변이 주석 도구(VEP, ANNOVAR)를 통해 변이의 기능적 영향을 예측하고 임상적 의미를 해석합니다. RNA-seq는 유전자 발현을 전장 유전체 규모로 측정합니다. 유전체 데이터는 재식별 위험이 높아 엄격한 접근 제어, 암호화, 감사 추적이 필요하며, dbGaP, EGA 같은 통제 접근 저장소를 통해 안전하게 공유됩니다. 약물유전체학은 개인의 유전 변이에 기반한 맞춤 약물 치료를 가능하게 합니다.
