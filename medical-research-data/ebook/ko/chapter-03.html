3장: 데이터 수집 및 문서화

연구 데이터 캡처, 주석 및 설명을 위한 표준

3.1 데이터 관리 계획

효과적인 데이터 수집은 단일 데이터 포인트가 캡처되기 전에 시작됩니다. 데이터 관리 계획(DMP)은 연구 수명주기 동안 데이터가 어떻게 수집, 구성, 문서화, 저장 및 공유될 것인지를 문서화합니다. 대부분의 주요 자금 제공자들은 이제 보조금 신청의 일부로 DMP를 요구합니다.

DMP는 연구 프로젝트와 함께 진화하는 살아있는 문서입니다. 프로젝트 마일스톤에서 재검토되고 데이터 관리 요구 사항이 변경되거나 새로운 요구 사항이 나타나면 업데이트되어야 합니다.

DMP 도구

여러 도구가 DMP 작성을 용이하게 합니다: DMPTool(미국)과 DMPonline(영국)은 자금 제공자 요구 사항에 맞춘 템플릿을 제공합니다. DMP Assistant(캐나다)는 이중 언어 지원을 포함합니다. 이러한 도구는 구조화된 DMP를 생성하고, 버전을 추적하며, 자금 제공자가 요구하는 형식으로 내보낼 수 있습니다. 많은 기관이 지역 DMP 템플릿과 지원 서비스를 제공합니다.

3.2 데이터 수집 표준

표준화된 데이터 수집은 일관성, 품질 및 상호운용성을 보장합니다:

도메인별 표준

도메인 표준 적용
임상 시험 CDISC (CDASH, SDTM, ADaM) 증례 보고서 양식, 제출 데이터셋
관찰 OMOP CDM, PCORnet CDM 실제 데이터 조화
유전체학 GA4GH 스키마, VCF, BAM 시퀀스 데이터, 변이 호출
영상 DICOM, BIDS, NIfTI 의료 영상, 신경 영상
바이오뱅킹 MIABIS, SPREC 샘플 수집 및 처리
설문/PRO PROMIS, SF-36, 검증된 도구 환자 보고 결과

CDISC 표준 스위트

CDISC(Clinical Data Interchange Standards Consortium)는 임상 연구 데이터를 위한 가장 포괄적인 표준을 제공합니다. CDASH는 증례 보고서 양식 내용을 표준화합니다. SDTM은 FDA 및 EMA에서 요구하는 제출 데이터셋 구조를 정의합니다. ADaM은 분석 준비 데이터셋을 지정합니다. ODM-XML은 전자 데이터 캡처 교환을 가능하게 합니다. CDISC 표준 채택은 규제 제출에 필요하며 저널 및 저장소에서 점점 더 기대되고 있습니다.

3.3 메타데이터 표준

메타데이터(데이터에 대한 데이터)는 발견, 이해 및 재사용에 필수적입니다:

메타데이터 수준

  • 기술적: 제목, 작성자, 주제, 키워드, 초록
  • 기술: 파일 형식, 크기, 소프트웨어, 생성 날짜
  • 관리: 권리, 라이선스, 접근 조건
  • 출처: 수집 방법, 처리 단계, 변환
  • 구조: 파일 간 관계, 구성 요소

메타데이터 스키마

스키마 범위 사용 사례
Dublin Core 일반 기술 기본 데이터셋 설명
DataCite 연구 데이터 인용 DOI 메타데이터, 저장소 입금
DDI 사회/건강 설문조사 설문조사 데이터 문서화
ISA-Tab 생명과학 실험 연구/분석/샘플 관계
DCAT 데이터 카탈로그 데이터 포털 상호운용성

3.4 문서화 모범 사례

포괄적인 문서화는 미래 사용자가 데이터를 이해하고 재사용할 수 있게 합니다:

필수 문서

  • README 파일: 데이터셋 내용 개요, 파일 구성, 주요 정보
  • 코드북: 변수 정의, 값 레이블, 단위, 누락 값 코드
  • 데이터 사전: 모든 데이터 요소의 자세한 설명
  • 프로토콜 문서: 사용된 방법, 도구, 절차
  • 분석 코드: 분석 단계를 설명하는 주석이 있는 스크립트
  • 변경 로그: 버전 히스토리, 시간이 지남에 따라 이루어진 수정

README 템플릿 구조

# 데이터셋 제목

## 설명
데이터셋에 대한 간략한 설명

## 인용

## 데이터 파일
- filename1.csv: 설명
- filename2.xlsx: 설명

## 변수
주요 변수 및 정의

## 수집 방법
데이터 수집 방법

## 처리
데이터 정제 및 변환 단계

## 접근 조건
라이선스, 제한, 연락처 정보

## 버전 히스토리
데이터셋 버전 간 변경사항

⚠️ 문서화 부채

문서화를 지연하면 "문서화 부채"가 발생합니다. 세부 사항이 기억에서 희미해지면 필요한 작업이 기하급수적으로 증가합니다. 진행하면서 문서화하세요: 프로젝트 시작 시 README 템플릿 생성, 실행 중인 실험 노트북 유지, 작성된 코드에 주석 달기, 각 데이터 버전으로 문서 업데이트. 오늘 명백해 보이는 것이 1년 후에는 미스터리가 될 것입니다.

3.5 파일 구성 및 명명

체계적인 구성은 검색 가능성을 향상시키고 오류를 줄입니다:

파일 명명 규칙

  • 설명적: 프로젝트, 유형, 날짜, 버전 정보 포함
  • 일관성: 모든 파일에 동일한 패턴 적용
  • 기계 판독 가능: 공백, 특수 문자 피하기; 밑줄이나 하이픈 사용
  • 정렬 가능: 시간순 정렬을 위해 YYYYMMDD 날짜 형식 사용
  • 문서화: 프로젝트 문서에 명명 규칙 기록

📌 핵심 요점

  • 데이터 관리 계획(DMP)은 데이터 수집, 저장, 공유 및 보존 계획을 문서화합니다; 대부분의 자금 제공자가 요구
  • 도메인별 표준(CDISC, OMOP, DICOM, GA4GH)은 일관성과 상호운용성을 보장합니다
  • 여러 수준의 메타데이터(기술적, 기술, 관리, 출처)는 발견과 재사용을 가능하게 합니다
  • 필수 문서에는 README 파일, 코드북, 데이터 사전 및 프로토콜 문서가 포함됩니다
  • 체계적인 파일 명명 및 구성은 검색 가능성을 향상시키고 오류를 줄입니다
  • 코드(Git) 및 데이터 추적을 위한 버전 관리는 재현성과 변경 히스토리를 보장합니다
