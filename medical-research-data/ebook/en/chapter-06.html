Chapter 6: Data Sharing and Access

Repositories, Access Controls, and Collaboration Frameworks

6.1 The Case for Data Sharing

Data sharing accelerates scientific discovery, enables validation and reproducibility, and maximizes return on research investment. Shared data enables meta-analyses with greater statistical power, allows replication of findings, facilitates new discoveries from existing datasets, and supports training of future researchers.

Major funders, publishers, and research institutions increasingly mandate or incentivize data sharing. The question is no longer whether to share, but how to share effectively while protecting participant privacy and researcher interests.

6.2 Research Data Repositories

Repositories provide infrastructure for data preservation, discovery, and access:

Repository Types

Type Examples Best For
Domain-Specific dbGaP, EGA, ArrayExpress, PRIDE Data with established community standards
Generalist Zenodo, Figshare, Dryad, OSF Data without domain repository
Institutional University data archives Institutional compliance, local support
Controlled Access dbGaP, EGA, YODA, Vivli Individual-level sensitive data
National UK Data Service, ICPSR Large surveys, government data

Choosing a Repository

Select repositories based on: data type (domain-specific preferred for standard data); access requirements (controlled vs. open); funder/journal requirements; long-term sustainability; curation services; and community uptake in your field. Use re3data.org to search 2,500+ repositories. Many funders and journals specify acceptable repositories.

6.3 Data Deposit Process

Preparing data for repository deposit requires careful attention to documentation and formatting:

Deposit Preparation

  • Data Preparation: Clean, well-organized files in open formats
  • Documentation: README, data dictionary, methodology
  • Metadata: Complete repository metadata forms
  • Access Level: Determine open vs. controlled access
  • License Selection: Choose appropriate data license
  • Linking: Connect to related publications, code, protocols

Data Licensing

License Permissions Conditions
CC0 Maximum reuse, public domain No conditions (waive all rights)
CC-BY Reuse with attribution Credit original creators
CC-BY-NC Non-commercial reuse Attribution, non-commercial only
DUA Custom terms Varies by agreement
Open Data Commons Database-specific PDDL, ODC-By, ODbL

CC0 Recommendation

For maximum reuse, CC0 (public domain dedication) is preferred for research data. Unlike creative works, facts cannot be copyrighted, so restrictive licenses may not be legally enforceable and create confusion. CC0 clearly signals that data is free for any purpose. Attribution norms in science ensure credit through citation regardless of license.

6.4 Clinical Trial Data Sharing

Clinical trial data sharing presents unique considerations:

Trial Data Sharing Platforms

Platform Data Source Access Model
ClinicalStudyDataRequest Industry sponsors Proposal review, controlled
Vivli Industry, academic trials Proposal review, virtual enclave
YODA Project Johnson & Johnson trials Proposal review, data transfer
NHLBI BioLINCC NHLBI-funded trials Request and agreement
IMMPORT Immunology trials Open or controlled

⚠️ ICMJE Requirements

The International Committee of Medical Journal Editors requires data sharing statements for clinical trials submitted to member journals. Authors must state whether individual participant data will be shared, what data will be shared, when data will become available, and how to access data. This policy applies to trials that began enrollment after January 2019.

6.5 Collaborative Data Networks

Federated approaches enable multi-site research without centralizing data:

Network Models

  • OHDSI: Observational health data science using OMOP CDM across 700+ institutions
  • PCORnet: Patient-Centered Outcomes Research Network with common data model
  • TriNetX: Global health research network with federated queries
  • EHDEN: European Health Data Evidence Network using OMOP
  • GA4GH Beacon: Federated discovery of genomic variants

OHDSI Network

The Observational Health Data Sciences and Informatics (OHDSI) collaborative enables research across 700+ databases worldwide without data centralization. Institutions map local data to the OMOP Common Data Model. Researchers develop analyses using common tools, then execute across the network. Results are aggregated while data stays local, enabling studies on billions of patient records while respecting privacy.

6.6 Data Citation

Proper citation gives credit to data creators and enables tracking of data reuse:

Data Citation Principles

  • Importance: Data citations are legitimate and important research contributions
  • Credit: Citation should facilitate attribution and credit
  • Evidence: Citations should support claims made about data
  • Identification: Citations should include persistent identifiers
  • Access: Citations should facilitate access to data and metadata
  • Persistence: Citations should be as durable as cited objects

Citation Format

Creator (Publication Year). Title. Version. Publisher/Repository. Identifier

Example: Smith J, Jones M. (2024). Diabetes Biomarker Study Dataset. V2. Zenodo. https://doi.org/10.5281/zenodo.123456

✓ Data Sharing Best Practices

Select appropriate repository (domain-specific preferred); prepare data with thorough documentation and open formats; choose permissive licenses (CC0 recommended) to maximize reuse; link data to publications and code; use persistent identifiers (DOIs); provide clear data availability statements; cite data as you would publications; and consider federated approaches for sensitive data.

📌 Key Takeaways

  • Data sharing accelerates discovery, enables validation, and generates 25% higher citation rates
  • Repository selection depends on data type, access requirements, and funder/journal mandates
  • Data licensing determines reuse conditions; CC0 (public domain) maximizes reuse potential
  • Clinical trial data sharing is increasingly mandated by journals and facilitated by specialized platforms
  • Federated networks (OHDSI, PCORnet) enable multi-site research without centralizing sensitive data
  • Proper data citation with persistent identifiers gives credit and enables reuse tracking
