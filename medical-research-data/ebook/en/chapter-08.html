Complete Technical Specification with 4-Phase Implementation Architecture

8.1 Standard Overview

The WIA-MEDICAL-RESEARCH-DATA standard provides a comprehensive framework for managing, sharing, and preserving medical research data throughout its lifecycle. Built on FAIR principles and the WIA 4-phase architecture (Data, API, Protocol, Integration), this standard enables interoperable, ethical, and sustainable research data management.

Guided by the principle of 弘益人間 (Hongik Ingan) - "Benefit All Humanity" - this standard aims to maximize the value of research investments by making data findable, accessible, interoperable, and reusable while protecting participant privacy and respecting researcher contributions.

8.2 WIA 4-Phase Architecture

The WIA-MEDICAL-RESEARCH-DATA standard implements the WIA 4-phase architecture pattern:

Architecture Philosophy

The 4-phase architecture separates concerns into distinct layers: Phase 1 (Data) defines dataset and metadata models aligned with DataCite and domain standards; Phase 2 (API) provides RESTful endpoints for data discovery, access, and management; Phase 3 (Protocol) enables communication between repositories, registries, and analysis platforms; Phase 4 (Integration) connects with institutional systems, federated networks, and preservation infrastructure.

Phase Name Purpose Components
Phase 1 Data Layer Information modeling Dataset, Metadata, Provenance models
Phase 2 API Layer Programmatic access Discovery, Access, Management APIs
Phase 3 Protocol Layer System communication OAI-PMH, SWORD, Signposting
Phase 4 Integration Layer External connectivity ORCID, DOI, Federated networks

8.4 Phase 2: API Layer Specification

The API Layer provides programmatic access to research data infrastructure:

API Endpoints Overview

Category Endpoint Method Description
Discovery /api/v1/datasets GET Search datasets
/api/v1/datasets/{id} GET Get dataset metadata
/api/v1/datasets/{id}/files GET List dataset files
Access /api/v1/access/request POST Submit access request
/api/v1/access/{id}/status GET Check request status
/api/v1/datasets/{id}/download GET Download dataset
Deposit /api/v1/deposits POST Create new deposit
/api/v1/deposits/{id}/files POST Upload files
/api/v1/deposits/{id}/publish POST Publish deposit
Metrics /api/v1/datasets/{id}/metrics GET Usage statistics
/api/v1/datasets/{id}/citations GET Citation tracking

Authentication

APIs use OAuth 2.0 with ORCID integration for researcher authentication. Scopes include: deposit:create, deposit:update, access:request, access:download, admin:review. Machine-to-machine access uses API keys with rate limiting.

8.7 FAIR Implementation

The standard ensures FAIR compliance through specific mechanisms:

Principle Implementation
F1-F4 DOIs, DataCite metadata, OAI-PMH harvesting, Google Dataset Search
A1-A2 HTTPS, OAuth, persistent metadata, access statements
I1-I3 Standard schemas, ontologies, qualified references
R1-R1.3 Rich metadata, machine-readable licenses, provenance, community standards

✓ Implementation Best Practices

  • Assign DOIs to all datasets through DataCite-compliant registration
  • Implement comprehensive metadata using DataCite schema with domain extensions
  • Support tiered access (open, registered, controlled) with clear access statements
  • Enable OAI-PMH harvesting for metadata aggregation
  • Integrate ORCID for researcher identification and attribution
  • Deploy Signposting headers for machine navigation

📌 Key Takeaways

  • WIA-MEDICAL-RESEARCH-DATA implements 4-phase architecture: Data Layer (metadata models), API Layer (REST endpoints), Protocol Layer (OAI-PMH, SWORD), Integration Layer (ORCID, DOI)
  • Core models include Dataset (DataCite-aligned), Metadata (multi-level), and Access Request
  • API layer provides discovery, access, deposit, and metrics endpoints with OAuth authentication
  • Protocol layer enables interoperability through OAI-PMH, SWORD, Signposting, and ResourceSync
  • Integration layer connects with ORCID, DataCite, federated networks, and preservation systems
  • FAIR compliance is built into all layers through specific technical mechanisms
