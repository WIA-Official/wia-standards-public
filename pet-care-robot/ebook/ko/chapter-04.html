제4장: 1단계 - 데이터 형식

4.1 데이터 형식의 중요성

통일된 데이터 형식은 WIA-ROB-014의 기반입니다. 모든 디바이스가 같은 언어로 소통할 수 있게 합니다.

측면 표준 데이터 형식의 이점 예시
상호 운용성 모든 디바이스가 동일한 데이터 이해 A사 급식기 → B사 앱에서 표시
데이터 통합 여러 소스의 데이터를 쉽게 결합 급식 + 활동 + 건강 데이터 통합 분석
검증 가능성 스키마로 데이터 유효성 자동 검증 잘못된 데이터 전송 시 즉시 거부
확장성 새 필드 추가 시 하위 호환성 유지 기존 시스템에 영향 없이 기능 확장

4.7 데이터 검증

모든 데이터는 전송 전 스키마 검증을 거쳐야 합니다.

4.7.1 검증 예제 (TypeScript)

import Ajv from 'ajv';
import addFormats from 'ajv-formats';

const ajv = new Ajv({ allErrors: true });
addFormats(ajv);

// 스키마 로드
const petProfileSchema = require('./schemas/pet-profile.json');
const validate = ajv.compile(petProfileSchema);

// 데이터 검증
function validatePetProfile(data: unknown): boolean {
  const valid = validate(data);

  if (!valid) {
    console.error('검증 실패:', validate.errors);
    return false;
  }

  return true;
}

// 사용 예
const petData = {
  petId: "pet-a1b2c3d4e5f67890a1b2c3d4e5f67890",
  name: "맥스",
  species: "dog",
  birthDate: "2022-03-15",
  ownerId: "user-f9e8d7c6b5a4321098765432109876543"
};

if (validatePetProfile(petData)) {
  console.log('✅ 유효한 데이터');
  // 데이터 전송 진행
} else {
  console.log('❌ 유효하지 않은 데이터');
  // 에러 처리
}

4.8 데이터 버전 관리

스키마 업데이트 시 하위 호환성을 유지하는 방법입니다.

변경 타입 허용 여부 버전 변경 예시
새 선택 필드 추가 ✅ 허용 Minor (1.0 → 1.1) "microchipId" 필드 추가
기존 필드 설명 개선 ✅ 허용 Patch (1.0.0 → 1.0.1) 주석 명확화
필수 필드 추가 ❌ 금지 Major (1.0 → 2.0) 기존 데이터 깨짐
필드 이름 변경 ❌ 금지 Major (1.0 → 2.0) "name" → "petName"
enum 값 추가 ✅ 허용 Minor (1.0 → 1.1) species에 "fish" 추가

4.10 데이터 보안 및 프라이버시

민감한 반려동물 데이터의 보호 방안입니다.

데이터 유형 민감도 보호 방법 접근 권한
기본 프로필 낮음 TLS 전송 소유자, 공유자
건강 데이터 높음 TLS + 암호화 소유자, 수의사
위치 정보 매우 높음 TLS + 암호화 + 익명화 소유자만
영상 데이터 매우 높음 E2EE (엔드투엔드 암호화) 소유자만

🌟 弘益人間 (홍익인간)

널리 인간을 이롭게 하라

표준화된 데이터 형식은 모든 이해관계자가
반려동물의 건강과 행복을 증진시킬 수 있게 합니다.

📌 핵심 내용 요약

  • Pet Profile Schema: petId, name, species, birthDate, weight, medicalInfo 등 포함
  • Activity Data: feeding, exercise, sleep, play, bathroom 5가지 활동 타입
  • Health Monitoring: 심박수, 호흡수, 체온, 혈압, 산소포화도 측정
  • Environment Data: 온도, 습도, 공기질, 조도, 소음 레벨 추적
  • Device Metadata: 디바이스 타입, 제조사, 모델, WIA 준수 레벨
  • 데이터 검증: JSON Schema + Ajv로 실시간 검증
  • 버전 관리: SemVer 방식, 선택 필드 추가는 Minor 업데이트
  • 최적화: 배치 전송, gzip 압축, checksum 검증
  • 보안: 민감도에 따른 차등 보호 (TLS, 암호화, E2EE)
