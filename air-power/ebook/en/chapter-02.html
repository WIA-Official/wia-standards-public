Chapter 2: Data Schema

2.1 Introduction to AIR-POWER Data Models

The WIA AIR-POWER standard defines comprehensive JSON schemas for representing power transmitters, receivers, power zones, and charging configurations. These schemas enable interoperability between different manufacturers and provide a consistent API for power management.

Schema Purpose: These data models ensure that any AIR-POWER compliant device can communicate with any compliant transmitter, regardless of manufacturer - enabling a truly universal wireless power ecosystem.

2.2 Transmitter (TX) Schema

The transmitter schema defines all properties and capabilities of a power transmitter, including supported modes, coverage area, power output, and safety parameters.

JSON Schema - Power Transmitter
{
  "$schema": "http://json-schema.org/draft-07/schema#",
  "title": "WIA AIR-POWER Transmitter",
  "type": "object",
  "properties": {
    "txId": {
      "type": "string",
      "description": "Unique transmitter identifier (UUID)",
      "format": "uuid"
    },
    "txType": {
      "type": "string",
      "enum": ["home", "commercial", "public", "vehicle"],
      "description": "Transmitter deployment type"
    },
    "location": {
      "type": "object",
      "properties": {
        "latitude": {"type": "number"},
        "longitude": {"type": "number"},
        "altitude": {"type": "number"},
        "indoor": {"type": "boolean"},
        "venue": {"type": "string"}
      }
    },
    "capabilities": {
      "type": "object",
      "properties": {
        "supportedModes": {
          "type": "array",
          "items": {
            "type": "string",
            "enum": ["rf", "magnetic_resonance", "infrared", "hybrid"]
          }
        },
        "maxPowerOutput": {
          "type": "number",
          "description": "Maximum total power output in watts"
        },
        "maxDevices": {
          "type": "integer",
          "description": "Maximum simultaneous devices supported"
        },
        "beamforming": {"type": "boolean"},
        "deviceTracking": {"type": "boolean"},
        "multiDevice": {"type": "boolean"}
      },
      "required": ["supportedModes", "maxPowerOutput"]
    },
    "coverageZone": {
      "type": "object",
      "properties": {
        "shape": {
          "type": "string",
          "enum": ["sphere", "cylinder", "cone", "polygon"]
        },
        "radius": {
          "type": "number",
          "description": "Coverage radius in meters"
        },
        "coordinates": {
          "type": "array",
          "description": "Polygon coordinates if shape is polygon",
          "items": {
            "type": "object",
            "properties": {
              "x": {"type": "number"},
              "y": {"type": "number"},
              "z": {"type": "number"}
            }
          }
        }
      }
    },
    "security": {
      "type": "object",
      "properties": {
        "accessControl": {
          "type": "string",
          "enum": ["open", "authenticated", "private"]
        },
        "encryption": {"type": "boolean"},
        "authenticationRequired": {"type": "boolean"}
      }
    },
    "safetyConfig": {
      "type": "object",
      "properties": {
        "sarMonitoring": {"type": "boolean"},
        "humanDetection": {"type": "boolean"},
        "autoShutoff": {"type": "boolean"},
        "maxSAR": {
          "type": "number",
          "description": "Maximum SAR in W/kg"
        }
      },
      "required": ["sarMonitoring", "humanDetection", "autoShutoff"]
    },
    "status": {
      "type": "object",
      "properties": {
        "online": {"type": "boolean"},
        "activeDevices": {"type": "integer"},
        "currentPowerOutput": {"type": "number"},
        "availablePower": {"type": "number"},
        "lastHealthCheck": {
          "type": "string",
          "format": "date-time"
        }
      }
    }
  },
  "required": ["txId", "txType", "capabilities", "safetyConfig"]
}

2.3 Receiver (RX) Schema

The receiver schema describes the capabilities and requirements of power-receiving devices, including power needs, battery status, and charging preferences.

{
  "$schema": "http://json-schema.org/draft-07/schema#",

    "deviceInfo": {
      "type": "object",

          "type": "string",

                   "earbud", "iot", "sensor", "appliance"]
        },
        "manufacturer": {"type": "string"},
        "model": {"type": "string"},
        "serialNumber": {"type": "string"}
      },
      "required": ["deviceType"]
    },
    "powerRequirements": {
      "type": "object",
      "properties": {
        "powerClass": {
          "type": "string",
          "enum": ["class_a", "class_b", "class_c", "class_d", "class_e"]
        },
        "minPower": {
          "type": "number",
          "description": "Minimum power in watts for operation"
        },
        "optimalPower": {
          "type": "number",
          "description": "Optimal power in watts for fast charging"
        },
        "maxPower": {
          "type": "number",

      "type": "object",

          "type": "string",

          "type": "number",
          "minimum": 0,
          "maximum": 100

    },
    "status": {
      "type": "object",
      "properties": {
        "isCharging": {"type": "boolean"},
        "connectedTxId": {"type": "string"},
        "currentPowerReceived": {"type": "number"},
        "chargingMode": {"type": "string"},
        "estimatedFullTime": {
          "type": "integer",
          "description": "Minutes until full charge"
        }
      }
    }
  },
  "required": ["rxId", "deviceInfo", "powerRequirements"]
}

        "geometry": {
          "type": "object",
          "description": "GeoJSON geometry defining coverage area"

          "type": "array",
          "items": {"type": "integer"}

2.7 Power Class Definitions

The WIA AIR-POWER standard defines five power classes to categorize devices based on their power requirements.

Class Name Power Range Typical Devices Charging Characteristic
Class A Ambient Power 0 - 100mW RFID tags, sensors, beacons Battery-free operation possible
Class B Wearable Power 100mW - 2W Smartwatches, fitness bands, earbuds Continuous trickle charge, never depletes
Class C Mobile Power 2W - 15W Smartphones, tablets Continuous top-up, always near full
Class D Computing Power 15W - 100W Laptops, monitors, tablets Sufficient for use while charging
Class E Appliance Power 100W - 1kW+ TVs, vacuums, kitchen appliances Future high-power applications
