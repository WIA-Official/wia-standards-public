Chapter 3: API Endpoints

3.1 API Overview

The WIA AIR-POWER standard defines a comprehensive RESTful API for managing wireless power infrastructure, monitoring power zones, registering devices, and tracking charging sessions. All endpoints follow REST principles and return JSON responses.

Base URL: All API endpoints are relative to the base URL: https://api.airpower.wia.org/v1

Authentication

All API requests require authentication using Bearer tokens:

Authorization: Bearer YOUR_API_TOKEN

3.2 Transmitter Management APIs

Endpoints for managing power transmitters, including registration, configuration, and status monitoring.

Transmitter Endpoints

GET /transmitters

List all transmitters in the system with optional filtering by zone, type, or status.

POST /transmitters

Register a new power transmitter in the system.

GET /transmitters/{txId}

Retrieve detailed information about a specific transmitter.

PUT /transmitters/{txId}

Update transmitter configuration or capabilities.

DELETE /transmitters/{txId}

Deregister a transmitter from the system.

GET /transmitters/{txId}/status

Get real-time status including active devices, power output, and health metrics.

POST /transmitters/{txId}/power-on

Activate a transmitter to begin power transmission.

POST /transmitters/{txId}/power-off

Deactivate a transmitter and stop power transmission.

GET /transmitters/{txId}/connected-devices

List all devices currently receiving power from this transmitter.

POST /transmitters/{txId}/health-check

Perform a comprehensive health check and diagnostics.

3.3 Receiver Management APIs

Endpoints for managing power receivers, including device registration, power requests, and battery monitoring.

Receiver Endpoints

POST /receivers/register

Register a new power receiver device in the system.

GET /receivers/{rxId}

Retrieve detailed information about a specific receiver.

PUT /receivers/{rxId}

Update receiver profile or charging preferences.

GET /receivers/{rxId}/battery-status

Get current battery level, health, and charging status.

POST /receivers/{rxId}/request-power

Request power allocation from available transmitters.

POST /receivers/{rxId}/stop-charging

Stop receiving power and release allocation.

GET /receivers/{rxId}/nearby-transmitters

Discover transmitters within range of this receiver.

PUT /receivers/{rxId}/preferences

Update charging preferences including priority and strategy.

3.4 Power Zone APIs

Endpoints for managing power zones, including creation, configuration, and access control.

Zone Endpoints

GET /zones

List all power zones with optional geographic or type filtering.

POST /zones

Create a new power zone.

GET /zones/{zoneId}

Get detailed information about a specific zone.

PUT /zones/{zoneId}

Update zone configuration, coverage, or policies.

DELETE /zones/{zoneId}

Delete a power zone.

GET /zones/{zoneId}/capacity

Get zone capacity metrics including utilization and available power.

GET /zones/{zoneId}/active-sessions

List all active charging sessions in this zone.

POST /zones/{zoneId}/access-control

Configure access policies and authentication requirements.

GET /zones/nearby

Discover zones near specified coordinates (latitude, longitude).

3.5 Charging Session APIs

Endpoints for managing and monitoring individual charging sessions.

Session Endpoints

POST /sessions/start

Initiate a new charging session between receiver and transmitter.

POST /sessions/{sessionId}/stop

End an active charging session.

GET /sessions/{sessionId}

Get detailed information about a specific session.

GET /sessions/{sessionId}/metrics

Retrieve real-time power metrics for an active session.

GET /sessions/history

Get charging session history for a device or zone.

GET /sessions/active

List all currently active charging sessions.

3.6 Analytics & Monitoring APIs

Endpoints for system-wide analytics, usage statistics, and performance monitoring.

Analytics Endpoints

GET /analytics/usage

Get power usage statistics over specified time period.

GET /analytics/efficiency

Analyze power transfer efficiency across zones or transmitters.

GET /analytics/peak-times

Identify peak usage times and capacity requirements.

GET /analytics/device-types

Breakdown of charging sessions by device type.

GET /monitoring/health

Overall system health status and alerts.

GET /monitoring/alerts

List active alerts and warnings across the system.

3.7 Safety & Compliance APIs

Endpoints for safety monitoring, compliance reporting, and emergency controls.

Safety Endpoints

GET /safety/sar-levels

Get current SAR exposure levels for specified zone or transmitter.

POST /safety/emergency-shutdown

Trigger emergency shutdown of specified transmitters or zones.

GET /safety/incidents

List safety incidents and automatic shutdowns.

GET /compliance/certifications

Get certification status and compliance reports.

GET /compliance/audit-log

Retrieve compliance audit logs for regulatory reporting.
