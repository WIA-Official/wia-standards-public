Chapter 1: Chiplet Fundamentals

This chapter establishes the foundational concepts of chiplet architecture, exploring the evolution from monolithic integrated circuits to modular die disaggregation. Understanding these fundamentals is essential for grasping the advanced topics covered in subsequent chapters.

Monolithic vs. Chiplet Design Philosophy

For over five decades, the semiconductor industry pursued a monolithic design approach, integrating all functionality onto a single silicon die. This approach served the industry well during the era of rapid transistor scaling, but fundamental physical and economic limits have prompted a paradigm shift.

The Monolithic Paradox

As monolithic die sizes increase to accommodate more transistors, manufacturing yields decrease exponentially. A single defect anywhere on the die renders the entire chip unusable. This relationship, known as the yield-area curve, becomes economically untenable beyond certain die sizes.

Monolithic Design Characteristics

// Monolithic Design Model
interface MonolithicChip {
    // Single die contains everything
    dieArea: number;           // mm² - typically 100-800
    transistorCount: number;   // billions
    processNode: string;       // e.g., "5nm", "3nm"

    // All components on same process
    components: {
        cpuCores: Component;
        gpuUnits: Component;
        memoryCtrls: Component;
        ioControllers: Component;
        // All manufactured together
    };

    // Yield calculation
    calculateYield(): number {
        // Poisson yield model
        const defectDensity = 0.1;  // defects per cm²
        const area = this.dieArea / 100;  // convert to cm²
        return Math.exp(-defectDensity * area);
    }
}

The monolithic approach has several inherent limitations:

  • Yield Scaling: Doubling die area more than halves yield in typical manufacturing conditions
  • Process Uniformity: All components must use the same process node, even when suboptimal
  • Design Rigidity: Changes to any component require complete redesign and revalidation
  • Development Time: Complex chips can take 3-5 years from concept to production
  • Manufacturing Cost: Leading-edge nodes cost billions in fab development

Chiplet Design Characteristics

// Chiplet-Based Design Model
interface ChipletSystem {
    chiplets: Chiplet[];
    interconnect: ChipletInterconnect;
    package: AdvancedPackage;

    // Calculate system yield
    calculateSystemYield(): number {
        // Product of individual chiplet yields
        return this.chiplets.reduce((yield, chiplet) => {
            return yield * chiplet.calculateYield();
        }, this.package.assemblyYield);
    }

    // Total effective transistors
    getTotalTransistors(): number {
        return this.chiplets.reduce((sum, c) => sum + c.transistorCount, 0);
    }
}

interface Chiplet {
    id: string;
    function: ChipletFunction;
    dieArea: number;      // Typically 50-150 mm²
    processNode: string;  // Can differ per chiplet
    transistorCount: number;

    calculateYield(): number {
        const defectDensity = this.getDefectDensity();
        return Math.exp(-defectDensity * this.dieArea / 100);
    }
}
Yield Mathematics Example

Consider a 400mm² design with 0.1 defects/cm² density. Monolithic yield: e^(-0.1 × 4) = 67%. Split into four 100mm² chiplets: (e^(-0.1 × 1))^4 × 95% assembly = (90.5%)^4 × 95% = 63.7%. While similar, the chiplet approach allows mix-and-match and design reuse, dramatically improving economics.

Die Disaggregation Principles

Die disaggregation is the process of partitioning a monolithic design into discrete chiplets. This partitioning decision is critical and must consider multiple factors including functionality, bandwidth requirements, power delivery, and manufacturing economics.

Disaggregation Boundaries

// Disaggregation Decision Framework
interface DisaggregationAnalysis {
    // Evaluate potential split points
    evaluateSplitPoint(boundary: ComponentBoundary): SplitMetrics {
        return {
            // Bandwidth across boundary
            crossBoundaryBandwidth: this.calculateBandwidth(boundary),

            // Latency impact
            latencyPenalty: this.estimateLatency(boundary),

            // Power overhead
            interconnectPower: this.calculateInterconnectPower(boundary),

            // Economic benefit
            yieldImprovement: this.compareYield(boundary),

            // Recommendation
            recommendation: this.analyzeTradeoffs(boundary)
        };
    }
}

Bandwidth and Latency Considerations

The interface between chiplets must carefully balance bandwidth, latency, power, and cost. Unlike on-die interconnects, chiplet interfaces cross package boundaries and incur additional overhead.

Interconnect Type Bandwidth Latency Power Efficiency
On-Die (NoC) >10 TB/s <1 ns <0.5 pJ/bit
Die-to-Die (UCIe) 1-4 TB/s 2-5 ns 0.5-1 pJ/bit
Package-Level (EMIB) 500 GB/s - 2 TB/s 3-10 ns 0.75-1.5 pJ/bit
Off-Package (PCIe) 64-256 GB/s 100+ ns >5 pJ/bit

Packaging as Enabling Technology

Advanced packaging is not merely about connecting dies; it enables entirely new system architectures. The progression from 150μm bump pitch to sub-10μm hybrid bonding represents a 15× improvement in interconnect density, fundamentally changing what is achievable in multi-die integration.

Economic Analysis Framework

Understanding the economics of chiplet design requires analyzing multiple factors across the product lifecycle. The WIA-CHIPLET standard provides a framework for this analysis.

// Chiplet Economic Model
class ChipletEconomicAnalysis {
    constructor(
        private monolithicBaseline: MonolithicDesign,
        private chipletAlternative: ChipletDesign
    ) {}

    calculateNRE(): ComparisonResult {
        // Non-Recurring Engineering costs
        return {
            monolithic: {
                designCost: this.monolithicBaseline.designComplexity * 50_000_000,
                maskCost: 15_000_000,  // Full chip mask set
                validationCost: 20_000_000
            },
            chiplet: {
                designCost: this.chipletAlternative.chiplets.reduce(
                    (sum, c) => sum + c.designComplexity * 10_000_000, 0
                ),
                maskCost: this.chipletAlternative.chiplets.length * 5_000_000,
                validationCost: 15_000_000  // Incremental validation
            }
        };
    }

    calculateUnitCost(volume: number): ComparisonResult {
        // Per-unit manufacturing cost at scale
        const monoYield = this.monolithicBaseline.calculateYield();
        const chipletYield = this.chipletAlternative.calculateSystemYield();

        return {
            monolithic: this.monolithicBaseline.waferCost / monoYield /
                        this.monolithicBaseline.diesPerWafer,
            chiplet: this.chipletAlternative.calculateTotalCost()
        };
    }

    calculateBreakeven(): number {
        // Volume at which chiplet becomes economical
        const nreDiff = this.calculateNRE();
        const unitDiff = this.calculateUnitCost(1000000);
        return nreDiff.delta / unitDiff.delta;
    }
}

Chiplet Design Trade-offs

Every chiplet architecture decision involves trade-offs. The following framework helps evaluate these decisions systematically.

Decision Factor Fewer, Larger Chiplets More, Smaller Chiplets
Die Yield Lower per-die yield Higher per-die yield
Interconnect Fewer, simpler interfaces More complex interconnect
Power Efficiency Less interface overhead More interface power
Design Reuse Less granular reuse More reuse opportunity
Assembly Complexity Simpler assembly More assembly steps
Chapter Summary

Chiplet architecture fundamentally changes semiconductor design by replacing monolithic dies with interconnected, specialized silicon modules. This approach improves yield economics, enables process mixing, accelerates time-to-market, and provides greater design flexibility. Understanding these fundamentals is essential for the detailed architecture discussions that follow.
