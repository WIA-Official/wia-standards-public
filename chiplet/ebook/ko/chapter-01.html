챕터 1: 칩렛 기초

이 챕터에서는 칩렛 아키텍처의 기초 개념을 확립하고, 모놀리식 집적 회로에서 모듈식 다이 분해로의 진화를 탐구합니다. 이러한 기초를 이해하는 것은 후속 챕터에서 다루는 고급 주제를 파악하는 데 필수적입니다. 칩렛 개념, SoC와 칩렛의 비교, 그리고 인터커넥트 기술에 대해 상세히 알아보겠습니다.

모놀리식 vs 칩렛 설계 철학

50년 이상 반도체 산업은 모든 기능을 단일 실리콘 다이에 통합하는 모놀리식 설계 접근 방식을 추구했습니다. 이 접근 방식은 급속한 트랜지스터 스케일링 시대에 산업에 잘 작동했지만, 근본적인 물리적, 경제적 한계로 인해 패러다임 전환이 촉발되었습니다.

모놀리식 패러독스

모놀리식 다이 크기가 더 많은 트랜지스터를 수용하기 위해 증가함에 따라 제조 수율은 기하급수적으로 감소합니다. 다이 어느 곳에서든 단일 결함이 발생하면 전체 칩이 사용 불가능해집니다. 수율-면적 곡선으로 알려진 이 관계는 특정 다이 크기를 초과하면 경제적으로 지속 불가능해집니다.

모놀리식 설계 특성

// 모놀리식 설계 모델
interface MonolithicChip {
    // 단일 다이에 모든 것 포함
    dieArea: number;           // mm - 일반적으로 100-800
    transistorCount: number;   // 수십억 개
    processNode: string;       // 예: "5nm", "3nm"

    // 모든 구성 요소가 동일한 공정
    components: {
        cpuCores: Component;
        gpuUnits: Component;
        memoryCtrls: Component;
        ioControllers: Component;
        // 모두 함께 제조
    };

    // 수율 계산
    calculateYield(): number {
        // 포아송 수율 모델
        const defectDensity = 0.1;  // 결함/cm 제곱
        const area = this.dieArea / 100;  // cm 제곱으로 변환
        return Math.exp(-defectDensity * area);
    }
}

모놀리식 접근 방식에는 여러 가지 고유한 한계가 있습니다:

  • 수율 스케일링: 다이 면적을 두 배로 늘리면 일반적인 제조 조건에서 수율이 절반 이하로 감소
  • 공정 균일성: 모든 구성 요소가 최적이 아니더라도 동일한 공정 노드를 사용해야 함
  • 설계 경직성: 어떤 구성 요소라도 변경하면 완전한 재설계 및 재검증 필요
  • 개발 시간: 복잡한 칩은 개념에서 생산까지 3-5년 소요
  • 제조 비용: 최첨단 노드는 팹 개발에 수십억 달러 비용

칩렛 설계 특성

// 칩렛 기반 설계 모델
interface ChipletSystem {
    chiplets: Chiplet[];
    interconnect: ChipletInterconnect;
    package: AdvancedPackage;

    // 시스템 수율 계산
    calculateSystemYield(): number {
        // 개별 칩렛 수율의 곱
        return this.chiplets.reduce((yield, chiplet) => {
            return yield * chiplet.calculateYield();
        }, this.package.assemblyYield);
    }

    // 총 유효 트랜지스터
    getTotalTransistors(): number {
        return this.chiplets.reduce((sum, c) => sum + c.transistorCount, 0);
    }
}

interface Chiplet {
    id: string;
    function: ChipletFunction;
    dieArea: number;      // 일반적으로 50-150 mm 제곱
    processNode: string;  // 칩렛마다 다를 수 있음
    transistorCount: number;

    calculateYield(): number {
        const defectDensity = this.getDefectDensity();
        return Math.exp(-defectDensity * this.dieArea / 100);
    }
}
수율 수학 예제

결함 밀도가 0.1 결함/cm 제곱인 400mm 제곱 설계를 고려해 봅시다. 모놀리식 수율: e^(-0.1 x 4) = 67%. 4개의 100mm 제곱 칩렛으로 분할: (e^(-0.1 x 1))^4 x 95% 조립 = (90.5%)^4 x 95% = 63.7%. 비슷하지만 칩렛 접근 방식은 믹스 앤 매치와 설계 재사용을 허용하여 경제성을 극적으로 개선합니다.

다이 분해 원칙

다이 분해는 모놀리식 설계를 개별 칩렛으로 분할하는 프로세스입니다. 이 분할 결정은 기능, 대역폭 요구사항, 전력 공급 및 제조 경제성을 포함한 여러 요소를 고려해야 하는 중요한 사항입니다.

분해 경계

// 분해 결정 프레임워크
interface DisaggregationAnalysis {
    // 잠재적 분할 지점 평가
    evaluateSplitPoint(boundary: ComponentBoundary): SplitMetrics {
        return {
            // 경계를 넘는 대역폭
            crossBoundaryBandwidth: this.calculateBandwidth(boundary),

            // 지연 영향
            latencyPenalty: this.estimateLatency(boundary),

            // 전력 오버헤드
            interconnectPower: this.calculateInterconnectPower(boundary),

            // 경제적 이점
            yieldImprovement: this.compareYield(boundary),

            // 권장 사항
            recommendation: this.analyzeTradeoffs(boundary)
        };
    }
}

대역폭 및 지연 고려사항

칩렛 간 인터페이스는 대역폭, 지연, 전력 및 비용을 신중하게 균형 맞춰야 합니다. 온다이 인터커넥트와 달리 칩렛 인터페이스는 패키지 경계를 넘어 추가 오버헤드가 발생합니다.

인터커넥트 유형 대역폭 지연 전력 효율
온다이 (NoC) >10 TB/s <1 ns <0.5 pJ/bit
다이-투-다이 (UCIe) 1-4 TB/s 2-5 ns 0.5-1 pJ/bit
패키지 레벨 (EMIB) 500 GB/s - 2 TB/s 3-10 ns 0.75-1.5 pJ/bit
오프 패키지 (PCIe) 64-256 GB/s 100+ ns >5 pJ/bit

패키징 진화

반도체 패키징의 진화는 칩렛 아키텍처를 가능하게 했습니다. 단순한 와이어 본딩에서 고급 2.5D 및 3D 통합까지, 패키징 기술은 칩렛 시스템 설계에서 무엇이 가능한지를 결정합니다.

활성화 기술로서의 패키징

고급 패키징은 단순히 다이를 연결하는 것이 아니라 완전히 새로운 시스템 아키텍처를 가능하게 합니다. 150마이크로미터 범프 피치에서 10마이크로미터 미만 하이브리드 본딩으로의 진행은 인터커넥트 밀도에서 15배 향상을 나타내며, 멀티 다이 통합에서 달성 가능한 것을 근본적으로 변화시킵니다.

경제 분석 프레임워크

칩렛 설계의 경제학을 이해하려면 제품 수명 주기 전반에 걸쳐 여러 요소를 분석해야 합니다.

// 칩렛 경제 모델
class ChipletEconomicAnalysis {
    constructor(
        private monolithicBaseline: MonolithicDesign,
        private chipletAlternative: ChipletDesign
    ) {}

    calculateNRE(): ComparisonResult {
        // 비반복 엔지니어링 비용
        return {
            monolithic: {
                designCost: this.monolithicBaseline.designComplexity * 50_000_000,
                maskCost: 15_000_000,  // 전체 칩 마스크 세트
                validationCost: 20_000_000
            },
            chiplet: {
                designCost: this.chipletAlternative.chiplets.reduce(
                    (sum, c) => sum + c.designComplexity * 10_000_000, 0
                ),
                maskCost: this.chipletAlternative.chiplets.length * 5_000_000,
                validationCost: 15_000_000  // 점진적 검증
            }
        };
    }

    calculateBreakeven(): number {
        // 칩렛이 경제적이 되는 볼륨
        const nreDiff = this.calculateNRE();
        const unitDiff = this.calculateUnitCost(1000000);
        return nreDiff.delta / unitDiff.delta;
    }
}

챕터 요약

칩렛 아키텍처는 모놀리식 다이를 상호 연결된 특화된 실리콘 모듈로 대체하여 반도체 설계를 근본적으로 변화시킵니다. 이 접근 방식은 수율 경제성을 개선하고, 공정 혼합을 가능하게 하며, 시장 출시를 가속화하고, 더 큰 설계 유연성을 제공합니다. 이러한 기초를 이해하는 것은 이어지는 상세한 아키텍처 논의에 필수적입니다.
