3.1 Defense in Depth

Effective cyber weapon defense requires multiple layers of security controls, ensuring that failure of any single layer doesn't compromise the entire system. This strategy acknowledges that no single security measure is impenetrable.

3.1.1 The Seven Layers of Defense

Layer Controls Purpose
1. Perimeter Firewalls, IDS/IPS, DDoS mitigation Filter known threats at boundary
2. Network Segmentation, encryption, monitoring Limit lateral movement
3. Endpoint EDR, antivirus, app whitelisting Protect workstations and servers
4. Application Secure coding, WAF, vulnerability mgmt Prevent application exploits
5. Data Encryption, DLP, access controls Protect sensitive information
6. Identity MFA, privilege mgmt, zero trust Verify every access request
7. Operations SIEM, SOC, incident response Detect, respond, recover

3.2 Zero Trust Architecture

Traditional "castle and moat" security assumes trust within the perimeter. Zero trust assumes breach and verifies every access request regardless of source location.

3.5 Security Orchestration (SOAR)

Security Orchestration, Automation, and Response (SOAR) platforms coordinate detection and response across multiple security tools, enabling rapid, consistent response to threats.

3.5.1 SOAR Capabilities

  • Orchestration: Connect disparate security tools through APIs
  • Automation: Execute playbooks without human intervention
  • Response: Contain threats automatically based on policy
  • Case Management: Track incidents through resolution
  • Metrics: Measure MTTD, MTTR, and other KPIs

3.6 Chapter Summary

Key Takeaways

  • Defense in depth provides multiple barriers to attackers
  • Zero trust assumes breach and verifies all access
  • Network segmentation limits lateral movement
  • Multi-layer detection catches threats at multiple points
  • SOAR enables automated, consistent response
