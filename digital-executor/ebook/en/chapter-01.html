Chapter 1: Executor Authorization

Identity Verification and Authorization Protocols

1.1 Introduction to Executor Authorization

The Digital Executor Authorization framework provides a comprehensive system for verifying and authorizing individuals who will manage digital assets on behalf of deceased or incapacitated account holders. This chapter outlines the complete authorization lifecycle, from initial identity verification through ongoing session management and access control.

Authorization is a multi-layered process that combines legal validation, identity verification, biometric authentication, and continuous monitoring to ensure that only properly authorized executors can access sensitive digital assets. The framework supports multiple authorization methods and provides flexibility for different jurisdictional requirements while maintaining strict security standards.

Key Principle: Executor authorization must balance security with accessibility, ensuring that legitimate executors can access assets while preventing unauthorized access. All authorization attempts must be logged and auditable.

1.2 Authorization Levels and Permissions

The WIA Digital Executor Standard defines five authorization levels, each with specific permissions and access rights. These levels ensure appropriate access control based on the executor's role and the sensitivity of the assets being managed.

Level Name Permissions Duration Requirements
1 Provisional View-only access to account metadata and asset inventory 7 days Death certificate, ID verification
2 Limited Read access to digital assets, no modification rights 30 days Court petition filed, legal representation
3 Standard Read/write access, asset transfer with approval 90 days Letters testamentary, probate approval
4 Full Complete asset control, account closure, beneficiary distribution 1 year Final probate decree, tax clearance
5 Administrative System configuration, audit access, delegation rights Unlimited Court supervision, bonding, quarterly reporting
Security Warning: Authorization levels cannot be escalated without proper legal documentation. Any attempt to bypass authorization controls must trigger immediate security alerts and potential account lockdown.

1.3 Identity Verification Protocol

Identity verification is the foundation of executor authorization. The protocol requires multiple forms of verification to establish the executor's identity beyond reasonable doubt. This multi-factor approach combines document verification, biometric authentication, and knowledge-based authentication.

1.5 Multi-Factor Authentication (MFA) Implementation

All executor authorization requires multi-factor authentication combining at least three independent authentication factors. The system supports various MFA methods and can adapt based on risk assessment and authorization level requirements.

Example 2: MFA Configuration and Validation

// MFA Implementation for Executor Authorization
const ExecutorMFA = {
  // Initialize MFA session
  initializeMFA: async (executorId, authRequest) => {
    const mfaSession = {
      sessionId: generateSecureId(),
      executorId: executorId,
      timestamp: new Date().toISOString(),
      requiredFactors: 3,
      completedFactors: [],
      expiresAt: addMinutes(new Date(), 15),
      maxAttempts: 3,
      currentAttempts: 0
    };

    // Store session in secure storage
    await secureMFAStorage.store(mfaSession.sessionId, mfaSession);

    return {
      sessionId: mfaSession.sessionId,
      requiredMethods: ['BIOMETRIC', 'KNOWLEDGE', 'POSSESSION'],
      expiresIn: 900 // seconds
    };
  },

  // Validate biometric factor
  validateBiometric: async (sessionId, biometricData) => {
    const session = await secureMFAStorage.retrieve(sessionId);

    if (!session || new Date() > new Date(session.expiresAt)) {
      throw new Error('MFA session expired');
    }

    // Perform facial recognition
    const faceMatch = await biometricService.compareFaces(
      biometricData.faceImage,
      session.registeredFaceTemplate
    );

    // Perform liveness detection
    const livenessCheck = await biometricService.detectLiveness(
      biometricData.faceVideo
    );

    if (faceMatch.confidence >= 0.95 && livenessCheck.isLive) {
      session.completedFactors.push({
        factor: 'BIOMETRIC',
        method: 'FACIAL_RECOGNITION',
        timestamp: new Date().toISOString(),
        confidence: faceMatch.confidence
      });

      await secureMFAStorage.update(sessionId, session);

      return {
        success: true,
        factorsRemaining: session.requiredFactors - session.completedFactors.length,
        message: 'Biometric authentication successful'
      };
    }

    session.currentAttempts++;
    await secureMFAStorage.update(sessionId, session);

    if (session.currentAttempts >= session.maxAttempts) {
      await lockExecutorAccount(session.executorId, 'MAX_MFA_ATTEMPTS_EXCEEDED');
    }

    return {
      success: false,
      attemptsRemaining: session.maxAttempts - session.currentAttempts,
      message: 'Biometric authentication failed'
    };
  },

  // Validate knowledge factor
  validateKnowledge: async (sessionId, answers) => {
    const session = await secureMFAStorage.retrieve(sessionId);

    const questions = await retrieveSecurityQuestions(session.executorId);
    let correctAnswers = 0;

    for (const [questionId, answer] of Object.entries(answers)) {
      const expectedAnswer = questions[questionId].answer;
      if (hashAnswer(answer) === expectedAnswer) {
        correctAnswers++;
      }
    }

    const threshold = questions.length * 0.8; // 80% correct required

    if (correctAnswers >= threshold) {
      session.completedFactors.push({
        factor: 'KNOWLEDGE',
        method: 'SECURITY_QUESTIONS',
        timestamp: new Date().toISOString(),
        accuracy: correctAnswers / questions.length
      });

      await secureMFAStorage.update(sessionId, session);
      return { success: true, factorsRemaining: session.requiredFactors - session.completedFactors.length };
    }

    return { success: false, message: 'Insufficient correct answers' };
  },

  // Validate possession factor (TOTP)
  validatePossession: async (sessionId, totpCode) => {
    const session = await secureMFAStorage.retrieve(sessionId);
    const executorSecret = await getExecutorTOTPSecret(session.executorId);

    const isValid = authenticator.verify({
      token: totpCode,
      secret: executorSecret
    });

    if (isValid) {
      session.completedFactors.push({
        factor: 'POSSESSION',
        method: 'TOTP',
        timestamp: new Date().toISOString()
      });

      await secureMFAStorage.update(sessionId, session);
      return { success: true, factorsRemaining: session.requiredFactors - session.completedFactors.length };
    }

    return { success: false, message: 'Invalid TOTP code' };
  },

  // Complete MFA process
  completeMFA: async (sessionId) => {
    const session = await secureMFAStorage.retrieve(sessionId);

    if (session.completedFactors.length >= session.requiredFactors) {
      const authToken = await generateExecutorAuthToken(session.executorId, {
        mfaCompleted: true,
        factors: session.completedFactors,
        sessionId: sessionId
      });

      await logAuthorizationEvent({
        executorId: session.executorId,
        event: 'MFA_COMPLETED',
        factors: session.completedFactors.map(f => f.factor),
        timestamp: new Date().toISOString()
      });

      return {
        success: true,
        authToken: authToken,
        expiresIn: 3600,
        tokenType: 'Bearer'
      };
    }

    return {
      success: false,
      message: 'MFA incomplete',
      factorsRemaining: session.requiredFactors - session.completedFactors.length
    };
  }
};

1.7 Token Management and Session Control

Authorization tokens are issued as JWT (JSON Web Tokens) with specific claims and expiration policies. Token management is critical for maintaining security while providing appropriate access duration.

Example 3: JWT Token Structure and Validation

// JWT Token Generation and Validation
const jwt = require('jsonwebtoken');
const crypto = require('crypto');

class ExecutorTokenManager {
  constructor(privateKey, publicKey) {
    this.privateKey = privateKey;
    this.publicKey = publicKey;
    this.algorithm = 'RS256';
  }

  generateToken(executorData, authLevel, sessionData) {
    const now = Math.floor(Date.now() / 1000);
    const tokenId = crypto.randomBytes(16).toString('hex');

    const payload = {
      // Standard JWT claims
      jti: tokenId,                      // JWT ID
      iss: 'wia-digital-executor-auth',  // Issuer
      sub: executorData.executorId,      // Subject (executor ID)
      aud: 'wia-digital-assets',         // Audience
      iat: now,                           // Issued at
      nbf: now,                           // Not before
      exp: now + this.getExpiryDuration(authLevel), // Expiration

      // Custom claims
      executor: {
        id: executorData.executorId,
        name: `${executorData.firstName} ${executorData.lastName}`,
        email: executorData.email,
        authorizationLevel: authLevel,
        verificationMethod: sessionData.verificationMethod
      },

      deceased: {
        id: executorData.deceasedId,
        name: executorData.deceasedName,
        dateOfDeath: executorData.dateOfDeath
      },

      permissions: {
        level: authLevel,
        platforms: this.getPlatformPermissions(authLevel),
        operations: this.getOperationPermissions(authLevel),
        restrictions: {
          ipWhitelist: executorData.ipWhitelist || [],
          geolocation: executorData.allowedCountries || [],
          timeRestrictions: executorData.accessHours || null
        }
      },

      security: {
        mfaCompleted: true,
        mfaFactors: sessionData.completedFactors,
        deviceFingerprint: sessionData.deviceFingerprint,
        sessionId: sessionData.sessionId
      },

      legal: {
        jurisdiction: executorData.jurisdiction,
        caseNumber: executorData.probateCaseNumber,
        courtApproval: executorData.courtApprovalDate || null
      }
    };

    const token = jwt.sign(payload, this.privateKey, {
      algorithm: this.algorithm,
      header: {
        typ: 'JWT',
        alg: this.algorithm,
        kid: this.getKeyId()
      }
    });

    // Store token metadata for revocation checking
    this.storeTokenMetadata(tokenId, {
      executorId: executorData.executorId,
      issuedAt: now,
      expiresAt: payload.exp,
      sessionId: sessionData.sessionId
    });

    return token;
  }

  validateToken(token, requiredLevel = null) {
    try {
      const decoded = jwt.verify(token, this.publicKey, {
        algorithms: [this.algorithm],
        audience: 'wia-digital-assets',
        issuer: 'wia-digital-executor-auth'
      });

      // Check if token has been revoked
      if (this.isTokenRevoked(decoded.jti)) {
        throw new Error('Token has been revoked');
      }

      // Verify authorization level if required
      if (requiredLevel && decoded.permissions.level < requiredLevel) {
        throw new Error('Insufficient authorization level');
      }

      // Validate IP restrictions
      if (decoded.permissions.restrictions.ipWhitelist.length > 0) {
        const clientIp = this.getClientIp();
        if (!decoded.permissions.restrictions.ipWhitelist.includes(clientIp)) {
          throw new Error('IP address not whitelisted');
        }
      }

      // Validate geolocation restrictions
      if (decoded.permissions.restrictions.geolocation.length > 0) {
        const clientCountry = this.getClientCountry();
        if (!decoded.permissions.restrictions.geolocation.includes(clientCountry)) {
          throw new Error('Access not permitted from this location');
        }
      }

      return {
        valid: true,
        decoded: decoded,
        executor: decoded.executor,
        permissions: decoded.permissions
      };

    } catch (error) {
      return {
        valid: false,
        error: error.message
      };
    }
  }

  refreshToken(currentToken) {
    const validation = this.validateToken(currentToken);

    if (!validation.valid) {
      throw new Error('Cannot refresh invalid token');
    }

    const decoded = validation.decoded;
    const now = Math.floor(Date.now() / 1000);

    // Only allow refresh if token is within refresh window (last 25% of lifetime)
    const tokenLifetime = decoded.exp - decoded.iat;
    const refreshWindow = decoded.exp - (tokenLifetime * 0.25);

    if (now < refreshWindow) {
      throw new Error('Token not eligible for refresh yet');
    }

    // Check if maximum refresh count exceeded
    const refreshCount = decoded.security.refreshCount || 0;
    if (refreshCount >= 5) {
      throw new Error('Maximum token refresh count exceeded. Re-authentication required.');
    }

    // Generate new token with same claims but updated timestamps
    const newTokenId = crypto.randomBytes(16).toString('hex');
    const newPayload = {
      ...decoded,
      jti: newTokenId,
      iat: now,
      exp: now + this.getExpiryDuration(decoded.permissions.level),
      security: {
        ...decoded.security,
        refreshCount: refreshCount + 1,
        previousTokenId: decoded.jti,
        refreshedAt: now
      }
    };

    // Revoke old token
    this.revokeToken(decoded.jti);

    return jwt.sign(newPayload, this.privateKey, {
      algorithm: this.algorithm
    });
  }

  revokeToken(tokenId, reason = 'USER_REQUESTED') {
    const metadata = this.getTokenMetadata(tokenId);

    if (!metadata) {
      throw new Error('Token not found');
    }

    this.tokenRevocationList.add(tokenId, {
      revokedAt: Date.now(),
      reason: reason
    });

    this.logTokenRevocation({
      tokenId: tokenId,
      executorId: metadata.executorId,
      reason: reason,
      timestamp: new Date().toISOString()
    });

    return true;
  }

  getExpiryDuration(authLevel) {
    const durations = {
      1: 3600,      // 1 hour for provisional
      2: 7200,      // 2 hours for limited
      3: 14400,     // 4 hours for standard
      4: 28800,     // 8 hours for full
      5: 43200      // 12 hours for administrative
    };

    return durations[authLevel] || 3600;
  }

  getPlatformPermissions(authLevel) {
    const permissions = {
      1: ['metadata'],
      2: ['metadata', 'email', 'cloud_storage'],
      3: ['metadata', 'email', 'cloud_storage', 'social_media'],
      4: ['metadata', 'email', 'cloud_storage', 'social_media', 'financial'],
      5: ['all']
    };

    return permissions[authLevel] || [];
  }

  getOperationPermissions(authLevel) {
    const operations = {
      1: ['read_metadata'],
      2: ['read_metadata', 'read_content'],
      3: ['read_metadata', 'read_content', 'modify_content', 'transfer_request'],
      4: ['read_metadata', 'read_content', 'modify_content', 'transfer_execute', 'account_close'],
      5: ['all']
    };

    return operations[authLevel] || [];
  }
}

1.9 Security Considerations

Authorization security must be treated as the highest priority. Any compromise in the authorization system could result in unauthorized access to sensitive digital assets and violation of the deceased's wishes. The following security measures are mandatory:

  • Zero Trust Architecture: Never assume trust based on network location or prior authentication
  • Defense in Depth: Multiple layers of security controls to prevent single point of failure
  • Least Privilege: Grant minimum necessary permissions for required tasks
  • Continuous Monitoring: Real-time monitoring of authorization attempts and anomaly detection
  • Encryption Everywhere: Encrypt all sensitive data at rest, in transit, and in use
  • Audit Everything: Comprehensive logging of all authorization activities
  • Regular Security Audits: Quarterly internal audits and annual third-party assessments
  • Incident Response: 24/7 security operations center with incident response capability
  • Vulnerability Management: Continuous vulnerability scanning and patch management
  • Access Reviews: Regular reviews of executor access rights and permissions
Critical Security Warning: Authorization credentials must NEVER be transmitted or stored in plain text. All passwords must be hashed using bcrypt or Argon2 with appropriate work factors. Private keys must be stored in hardware security modules (HSM) or secure key management systems.
