8.1 Standard Scope and Purpose

WIA-SEMI-EQP-001 establishes comprehensive specifications for semiconductor manufacturing equipment integration, enabling seamless communication, data exchange, and interoperability across diverse equipment types and vendors. The standard supports the $110 billion global semiconductor equipment market in achieving efficient, high-yield manufacturing at advanced process nodes.

Scope

This standard applies to:

  • Lithography equipment (DUV, EUV, High-NA EUV)
  • Deposition equipment (CVD, ALD, PVD, Epitaxy)
  • Etch equipment (Plasma etch, ALE, HAR etch)
  • Metrology and inspection equipment
  • Ion implantation and thermal processing equipment
  • Clean room and facility equipment
  • Automated material handling systems (AMHS)

8.2 Core Data Model

The WIA-SEMI-EQP-001 data model defines standardized structures for equipment identification, status, process data, and maintenance information.

Equipment Identification

// Core Equipment Identification Schema
interface EquipmentIdentification {
    // Unique identifiers
    equipmentId: string;          // Globally unique ID (UUID v4)
    vendorCode: string;           // SEMI E10 vendor code
    modelNumber: string;          // Vendor model designation
    serialNumber: string;         // Unique serial number
    
    // Classification
    equipmentType: EquipmentType;
    processCapability: ProcessCapability[];
    nodeCompatibility: string[];  // e.g., ['N3', 'N2', 'A16']
    
    // Installation
    fabId: string;
    bayId: string;
    locationX: number;
    locationY: number;
    installDate: Date;
    
    // Software
    firmwareVersion: string;
    interfaceVersion: string;     // WIA-SEMI-EQP version
    securityCertificate: string;
}

enum EquipmentType {
    LITHOGRAPHY = 'LITHO',
    DEPOSITION = 'DEP',
    ETCH = 'ETCH',
    METROLOGY = 'METRO',
    IMPLANT = 'IMPLANT',
    THERMAL = 'THERMAL',
    CLEAN = 'CLEAN',
    AMHS = 'AMHS',
    FACILITY = 'FAC'
}

8.3 Process Data Specifications

Standardized process data formats enable cross-vendor analytics and advanced process control.

Process Parameter Schema

// Process Data Collection
interface ProcessData {
    // Identification
    equipmentId: string;
    recipeId: string;
    lotId: string;
    waferId: string;
    processStep: number;
    
    // Timing
    startTime: Date;
    endTime: Date;
    
    // Parameters (equipment-specific)
    parameters: ProcessParameter[];
    
    // Results
    results: ProcessResult[];
    quality: QualityMetrics;
}

interface ProcessParameter {
    name: string;
    value: number;
    unit: string;
    setpoint: number;
    tolerance: number;
    timestamp: Date;
    samplingRate: number;        // Hz
}

// Example: Lithography Process Data
interface LithographyData extends ProcessData {
    parameters: {
        exposureDose: number;    // mJ/cm2
        focusOffset: number;     // nm
        overlayX: number;        // nm
        overlayY: number;        // nm
        cdMean: number;          // nm
        cdUniformity: number;    // %
    };
}

8.4 Communication Protocols

WIA-SEMI-EQP-001 specifies communication protocols that build upon industry standards while enabling modern capabilities.

Protocol Stack

Application Layer

  • WIA-SEMI API: RESTful/GraphQL interfaces for control and data access
  • SECS/GEM: SEMI E5/E30 for legacy compatibility
  • OPC UA: Industrial automation interoperability

Transport Layer

  • HSMS: SEMI E37 High-Speed SECS Message Services
  • MQTT: Publish/subscribe for real-time data streaming
  • gRPC: High-performance RPC for critical paths

Security Layer

  • TLS 1.3: Encryption for all communications
  • mTLS: Mutual authentication for equipment
  • OAuth 2.0: Authorization for API access

API Specification

8.5 Recipe Management

Standardized recipe formats enable recipe portability, version control, and manufacturing optimization.

Recipe Schema

// WIA-SEMI-EQP-001 Recipe Format
interface Recipe {
    // Identification
    recipeId: string;            // Unique recipe ID
    name: string;                // Human-readable name
    version: string;             // Semantic versioning
    
    // Metadata
    author: string;
    createdDate: Date;
    modifiedDate: Date;
    approvalStatus: ApprovalStatus;
    
    // Target equipment
    equipmentType: EquipmentType;
    compatibleModels: string[];
    nodeTarget: string;
    
    // Recipe content
    steps: RecipeStep[];
    parameters: RecipeParameter[];
    limits: RecipeLimit[];
    
    // Validation
    checksum: string;
    signature: string;
}

interface RecipeStep {
    stepNumber: number;
    name: string;
    duration: number;           // seconds
    parameters: Map<string, number>;
    transitions: StepTransition[];
    endpointCriteria: EndpointCriteria[];
}

interface RecipeLimit {
    parameter: string;
    warningLow: number;
    warningHigh: number;
    faultLow: number;
    faultHigh: number;
    action: LimitAction;
}

8.8 Security Requirements

Comprehensive security protects sensitive manufacturing data and prevents unauthorized equipment access.

Security Architecture

Mandatory Security Controls

  • Authentication: Certificate-based mutual TLS for all equipment
  • Authorization: Role-based access control (RBAC)
  • Encryption: TLS 1.3 minimum, AES-256 for data at rest
  • Audit: Comprehensive logging of all access and changes
  • Network: Segmented networks, firewalls between zones

Access Control Model

Role Read Data Control Equipment Modify Recipe Admin
Operator Yes Limited No No
Process Engineer Yes Yes Yes No
Equipment Engineer Yes Yes No No
Administrator Yes Yes Yes Yes

SDK Example

// TypeScript SDK Usage Example
import { WIAEquipment, ProcessData } from '@wia/semi-eqp-001';

// Connect to equipment
const equipment = await WIAEquipment.connect({
    host: 'litho-01.fab.example.com',
    port: 5000,
    certificate: './certs/client.pem',
    key: './certs/client-key.pem'
});

// Get current status
const status = await equipment.getStatus();
console.log(`Equipment state: ${status.state}`);
console.log(`Health score: ${status.healthScore}`);

// Subscribe to process data
equipment.onProcessData((data: ProcessData) => {
    console.log(`Wafer ${data.waferId} processed`);
    console.log(`Overlay X: ${data.parameters.overlayX} nm`);
});

// Execute recipe
const result = await equipment.executeRecipe('LITHO_N2_M1', {
    lot: 'LOT-2025-001',
    wafers: [1, 2, 3, 4, 5]
});

// Disconnect
await equipment.disconnect();

  • Security First: mTLS, RBAC, and comprehensive audit capabilities
  • Certification Path: Bronze through Platinum levels for progressive adoption
