제5장: Phase 2 - API 인터페이스

弘益人間 · 널리 인간을 이롭게 하라

개요

2023년, 서울 아트센터 나비의 한 작가는 새로운 인터랙티브 설치 작품을 준비하고 있었습니다. 작가는 Processing으로 프로토타입을 만들었지만, 전시장의 기술 팀은 TouchDesigner를 사용하고 있었습니다. 센서 데이터를 읽고 조명을 제어하는 코드를 두 플랫폼 모두에서 다시 작성해야 했고, 각 플랫폼마다 다른 API를 배워야 했습니다. 결국 작품 설치에만 3주가 소요되었고, 버그 수정과 최적화에 추가 2주가 필요했습니다.

WIA-ART-010의 Phase 2는 바로 이러한 플랫폼 파편화 문제를 해결합니다. 표준화된 API와 SDK를 제공하여 어떤 프로그래밍 언어나 크리에이티브 플랫폼에서든 동일한 방식으로 센서를 읽고 출력 장치를 제어할 수 있게 합니다. 이 장에서는 WIA-ART-010 API의 구조, TypeScript/JavaScript SDK, REST API 엔드포인트, 그리고 주요 플랫폼 통합 방법을 실제 코드 예제와 함께 상세히 살펴봅니다.

1. WIA-ART-010 API 아키텍처

1.1 API 설계 원칙

WIA-ART-010 API는 다음의 핵심 원칙을 따릅니다:

  • 일관성(Consistency): 모든 센서와 출력 장치에 동일한 인터페이스 패턴을 적용합니다.
  • 단순성(Simplicity): 복잡한 하드웨어 제어를 간단한 함수 호출로 추상화합니다.
  • 확장성(Extensibility): 플러그인 아키텍처로 새로운 센서와 장치를 쉽게 추가할 수 있습니다.
  • 성능(Performance): 실시간 인터랙션을 위한 저지연 처리를 보장합니다.
  • 타입 안전성(Type Safety): TypeScript 타입 정의로 개발 시 오류를 사전에 방지합니다.

1.2 API 계층 구조

WIA-ART-010 API는 3계층 아키텍처로 구성됩니다:

계층 역할 구현 언어 사용자
High-Level SDK 간편한 작품 개발 API TypeScript, Python 아티스트, 디자이너
Core API 센서/출력 추상화 TypeScript, C++ 기술 개발자
Hardware Drivers 하드웨어 직접 제어 C++, Rust 시스템 엔지니어

2. TypeScript SDK

2.1 설치 및 초기화

WIA-ART-010 TypeScript SDK를 설치하고 초기화하는 방법: 

// npm 설치
npm install @wia/art-010

// 기본 초기화
import { WIAArtwork, DepthCamera, LEDStrip } from '@wia/art-010';

const artwork = new WIAArtwork({
  manifestPath: './artwork.wia-manifest.json',
  autoConnect: true
});

await artwork.init();

2.2 센서 API

모든 센서는 Sensor 기본 인터페이스를 구현하며, 각 센서 유형마다 특화된 메서드를 제공합니다.

깊이 카메라 API 사용 예제

import { DepthCamera, SkeletonData } from '@wia/art-010';

// 센서 생성 및 연결
const camera = new DepthCamera({
  id: 'sensor-001',
  model: 'Intel RealSense D455',
  resolution: { width: 1280, height: 720, fps: 30 }
});

await camera.connect();

// 실시간 골격 추적
camera.on('skeleton', (skeleton: SkeletonData) => {
  const { head, hands } = skeleton.joints;

  console.log(`Head position: x=${head.x}, y=${head.y}, z=${head.z}`);
  console.log(`Hand height: ${hands.left.y}`);

  // 손 높이에 따라 조명 색상 변경
  const hue = mapRange(hands.left.y, 0.5, 2.5, 0, 360);
  ledStrip.setColor({ h: hue, s: 100, v: 100 });
});

// 제스처 인식
camera.on('gesture', (gesture) => {
  if (gesture.type === 'waving' && gesture.confidence > 0.8) {
    console.log('손 흔들기 감지!');
    ledStrip.flash({ duration: 500, color: '#FFD700' });
  }
});

// 군중 분석
camera.on('crowd', (metrics) => {
  console.log(`현재 관람객 수: ${metrics.totalPeople}`);
  console.log(`평균 거리: ${metrics.averageDistance}m`);
  console.log(`움직임 강도: ${metrics.movementIntensity}`);
});

2.3 출력 장치 API

LED, 프로젝터, 스피커 등 모든 출력 장치는 OutputDevice 인터페이스를 구현합니다. 

import { LEDStrip, DMXController } from '@wia/art-010';

// DMX 컨트롤러를 통한 LED 스트립 제어
const dmx = new DMXController({ port: '/dev/ttyUSB0' });
const ledStrip = new LEDStrip({
  id: 'led-strip-001',
  controller: dmx,
  channelStart: 1,
  channelCount: 5
});

// 밝기 설정 (0-255)
ledStrip.setBrightness(200);

// 색상 설정 (HSV)
ledStrip.setColor({ h: 120, s: 100, v: 80 });

// RGB 직접 설정
ledStrip.setRGB(255, 100, 50);

// 부드러운 페이드 효과
await ledStrip.fadeTo({
  color: { h: 240, s: 100, v: 100 },
  duration: 2000  // 2초
});

// 스트로브 효과
ledStrip.strobe({
  frequency: 5,  // 5 Hz
  duration: 3000  // 3초간
});

2.4 인터랙션 매핑

센서 입력을 출력으로 매핑하는 고수준 API를 제공합니다. 

import { InteractionMapper, Curve } from '@wia/art-010';

const mapper = new InteractionMapper();

// 거리 → 밝기 매핑 (역제곱 법칙)
mapper.map({
  sensor: camera,
  inputPath: 'skeleton.torso.z',
  output: ledStrip,
  outputParameter: 'brightness',
  inputRange: [0.5, 5.0],
  outputRange: [0, 255],
  curve: Curve.InverseSquare,
  smoothing: {
    type: 'exponential',
    alpha: 0.3
  }
});

// 손 위치 → 색상 매핑 (선형)
mapper.map({
  sensor: camera,
  inputPath: 'skeleton.hands.left.y',
  output: ledStrip,
  outputParameter: 'hue',
  inputRange: [0.5, 2.5],
  outputRange: [0, 360],
  curve: Curve.Linear
});

// 매핑 활성화
mapper.start();

3. REST API

3.1 API 엔드포인트

WIA-ART-010 작품은 표준 REST API를 제공하여 원격 모니터링과 제어를 지원합니다.

엔드포인트 메서드 설명 응답
/api/v1/artwork GET 작품 메타데이터 조회 JSON manifest
/api/v1/sensors GET 센서 목록 조회 센서 배열
/api/v1/sensors/:id GET 특정 센서 데이터 실시간 센서 값
/api/v1/outputs GET 출력 장치 목록 출력 장치 배열
/api/v1/outputs/:id POST 출력 장치 제어 실행 결과
/api/v1/interactions GET/POST 인터랙션 모드 관리 모드 상태

4. 플랫폼 통합

4.1 TouchDesigner 통합

TouchDesigner는 실시간 비주얼 제작에 널리 사용되는 노드 기반 플랫폼입니다. WIA-ART-010 WebSocket 브리지를 통해 TouchDesigner와 통합할 수 있습니다.

TouchDesigner 연동 예제

# TouchDesigner Python Script OP

import asyncio
import websocket
import json

# WIA-ART-010 WebSocket 연결
ws_url = "ws://localhost:8080/sensor-001"
ws = websocket.create_connection(ws_url)

def onReceive():
    # 센서 데이터 수신
    data = json.loads(ws.recv())

    # 골격 데이터 추출
    if 'detected_objects' in data:
        person = data['detected_objects'][0]
        head_y = person['skeleton']['head']['y']

        # TouchDesigner CHOP으로 전달
        op('constant1').par.value0 = head_y

        # 손 위치로 색상 제어
        hand_y = person['skeleton']['left_hand']['y']
        hue = (hand_y - 0.5) / 2.0 * 360
        op('constant_hue').par.value0 = hue

# 매 프레임 실행
me.time.rate = 30  # 30 FPS

4.2 Processing 통합

Processing은 비주얼 아티스트를 위한 인기 있는 크리에이티브 코딩 플랫폼입니다. 

// Processing Sketch

import wia.art.*;

WIAArtwork artwork;
DepthCamera camera;
LEDController leds;

void setup() {
  size(800, 600);

  // WIA-ART-010 초기화
  artwork = new WIAArtwork("artwork.wia-manifest.json");
  camera = artwork.getDepthCamera("sensor-001");
  leds = artwork.getLEDController("led-strip-001");

  // 센서 이벤트 리스너
  camera.addSkeletonListener(new SkeletonListener() {
    void onSkeleton(SkeletonData skeleton) {
      PVector head = skeleton.getJoint("head");
      PVector hand = skeleton.getJoint("left_hand");

      // 화면에 골격 그리기
      drawSkeleton(skeleton);

      // 손 높이로 LED 색상 제어
      float hue = map(hand.y, 0.5, 2.5, 0, 360);
      leds.setHSV(hue, 100, 100);
    }
  });
}

void draw() {
  background(0);
  // 매 프레임 업데이트
  artwork.update();
}

void drawSkeleton(SkeletonData skeleton) {
  stroke(255, 200, 0);
  strokeWeight(5);
  for (Joint joint : skeleton.getJoints()) {
    PVector pos = joint.getPosition();
    point(pos.x * width, pos.y * height);
  }
}

4.3 openFrameworks 통합

openFrameworks는 C++ 기반의 고성능 크리에이티브 코딩 툴킷입니다. 

// ofApp.h
#include "ofMain.h"
#include "wia/art-010.h"

class ofApp : public ofBaseApp {
public:
    void setup();
    void update();
    void draw();

    wia::Artwork artwork;
    wia::DepthCamera camera;
    wia::LEDStrip ledStrip;
};

// ofApp.cpp
void ofApp::setup() {
    // WIA-ART-010 초기화
    artwork.loadManifest("artwork.wia-manifest.json");
    camera = artwork.getDepthCamera("sensor-001");
    ledStrip = artwork.getLEDStrip("led-strip-001");

    // 센서 콜백 등록
    camera.setSkeletonCallback([this](const wia::SkeletonData& skeleton) {
        auto head = skeleton.joints["head"];
        auto leftHand = skeleton.joints["left_hand"];

        // 손 높이로 색상 매핑
        float hue = ofMap(leftHand.y, 0.5, 2.5, 0, 255);
        ofColor color = ofColor::fromHsb(hue, 255, 255);
        ledStrip.setColor(color);
    });
}

void ofApp::update() {
    artwork.update();
}

void ofApp::draw() {
    ofBackground(0);

    // 센서 데이터 시각화
    auto skeletons = camera.getSkeletons();
    for (auto& skeleton : skeletons) {
        ofSetColor(255, 200, 0);
        for (auto& [name, joint] : skeleton.joints) {
            ofDrawSphere(joint.x * ofGetWidth(),
                        joint.y * ofGetHeight(),
                        joint.z * 10, 10);
        }
    }
}

5. 실전 예제: teamLab 스타일 작품

5.1 "Interactive Flower Garden" 구현

관람객이 움직이면 꽃이 피고 지는 teamLab 스타일의 인터랙티브 작품을 WIA-ART-010 API로 구현합니다. 

import { WIAArtwork, DepthCamera, Projector, ParticleSystem } from '@wia/art-010';

class FlowerGarden {
  private camera: DepthCamera;
  private projector: Projector;
  private flowers: Flower[] = [];

  async setup() {
    const artwork = new WIAArtwork({
      manifestPath: './flower-garden.wia-manifest.json'
    });

    this.camera = artwork.getDepthCamera('sensor-001');
    this.projector = artwork.getProjector('projector-001');

    // 가상 꽃 500송이 생성
    for (let i = 0; i < 500; i++) {
      this.flowers.push(new Flower({
        x: Math.random() * this.projector.width,
        y: Math.random() * this.projector.height
      }));
    }

    // 사람 감지 이벤트
    this.camera.on('person', (person) => {
      const { x, y, z } = person.position;

      // 사람 근처의 꽃 찾기
      this.flowers.forEach(flower => {
        const distance = flower.distanceTo(x, y);

        if (distance < 1.0) {
          // 1m 이내: 꽃 피우기
          flower.bloom();
        } else if (distance < 2.0) {
          // 2m 이내: 꽃잎 흔들림
          flower.sway(distance);
        }
      });
    });

    // 애니메이션 루프
    this.animate();
  }

  animate() {
    requestAnimationFrame(() => this.animate());

    // 모든 꽃 업데이트
    this.flowers.forEach(flower => flower.update());

    // 프로젝터로 렌더링
    const canvas = this.renderFlowers();
    this.projector.display(canvas);
  }

  renderFlowers() {
    const canvas = document.createElement('canvas');
    const ctx = canvas.getContext('2d');

    this.flowers.forEach(flower => {
      flower.draw(ctx);
    });

    return canvas;
  }
}

class Flower {
  private bloomLevel = 0;  // 0: 봉오리, 1: 만개
  private swayAngle = 0;

  bloom() {
    // 부드럽게 꽃 피우기
    this.bloomLevel += 0.05;
    if (this.bloomLevel > 1) this.bloomLevel = 1;
  }

  sway(distance: number) {
    // 거리에 반비례하여 흔들림
    this.swayAngle = Math.sin(Date.now() / 500) * (2.0 - distance) * 10;
  }

  update() {
    // 시간이 지나면 서서히 시들기
    if (this.bloomLevel > 0) {
      this.bloomLevel -= 0.001;
    }
  }

  draw(ctx: CanvasRenderingContext2D) {
    ctx.save();
    ctx.translate(this.x, this.y);
    ctx.rotate(this.swayAngle * Math.PI / 180);

    // 꽃잎 그리기
    const petalCount = 5;
    const petalLength = 20 * this.bloomLevel;

    for (let i = 0; i < petalCount; i++) {
      const angle = (i / petalCount) * Math.PI * 2;
      ctx.beginPath();
      ctx.ellipse(
        Math.cos(angle) * petalLength,
        Math.sin(angle) * petalLength,
        petalLength, petalLength / 2,
        angle, 0, Math.PI * 2
      );
      ctx.fillStyle = `hsl(${30 + this.bloomLevel * 30}, 100%, 70%)`;
      ctx.fill();
    }

    ctx.restore();
  }
}

// 실행
const garden = new FlowerGarden();
garden.setup();

6. 성능 최적화

6.1 센서 데이터 스로틀링

고속 센서는 초당 수백~수천 개의 데이터를 생성할 수 있습니다. 애플리케이션이 모든 데이터를 처리할 필요가 없다면 스로틀링을 적용하여 성능을 향상시킬 수 있습니다. 

import { DepthCamera, Throttle } from '@wia/art-010';

const camera = new DepthCamera({ id: 'sensor-001' });

// 초당 10회로 제한
const throttled = Throttle(camera, { fps: 10 });

throttled.on('skeleton', (skeleton) => {
  // 초당 10번만 실행됨
  updateVisualization(skeleton);
});

6.2 배치 처리

여러 출력 장치를 동시에 업데이트할 때 배치 처리를 사용하면 네트워크 오버헤드를 줄일 수 있습니다. 

import { LEDController, Batch } from '@wia/art-010';

const controller = new LEDController();
const batch = controller.createBatch();

// 여러 LED를 한 번에 업데이트
for (let i = 0; i < 100; i++) {
  batch.setLED(i, { h: i * 3.6, s: 100, v: 100 });
}

// 한 번의 DMX 전송으로 모든 LED 업데이트
await batch.commit();
최적화 기법 적용 대상 성능 향상 트레이드오프
Throttling 고속 센서 데이터 CPU 사용량 -60% 반응 지연 증가
Batching 다수 출력 장치 네트워크 트래픽 -80% 약간의 시간 지연
Caching 센서 캘리브레이션 초기화 시간 -70% 메모리 사용 증가
Worker Threads 복잡한 계산 메인 스레드 부하 -50% 구현 복잡도 증가

핵심 요점

  • 통합 API: WIA-ART-010은 TypeScript/JavaScript SDK와 REST API를 제공하여 모든 플랫폼에서 일관된 방식으로 센서와 출력 장치를 제어할 수 있습니다.
  • 이벤트 기반 아키텍처: 센서 데이터는 이벤트로 전달되어 반응형 프로그래밍 패턴을 쉽게 구현할 수 있으며, skeleton, gesture, crowd 등 다양한 이벤트를 지원합니다.
  • 고수준 추상화: InteractionMapper를 통해 센서 입력을 출력으로 매핑하는 복잡한 로직을 선언적으로 정의할 수 있어 코드 양을 크게 줄입니다.
  • 플랫폼 통합: TouchDesigner, Processing, openFrameworks 등 주요 크리에이티브 코딩 플랫폼과 원활하게 통합되며, WebSocket과 REST API를 통한 원격 제어를 지원합니다.
  • 성능 최적화: Throttling, Batching, Worker Threads 등의 기법을 통해 실시간 인터랙션에 필요한 저지연 고성능 처리를 보장합니다.
  • 타입 안전성: TypeScript 기반 SDK는 개발 시점에 오류를 발견할 수 있어 안정적인 작품 개발을 지원하며, 자동 완성 기능으로 생산성을 향상시킵니다.

복습 질문

  1. WIA-ART-010 API의 3계층 아키텍처(High-Level SDK, Core API, Hardware Drivers)를 설명하고, 각 계층이 담당하는 역할과 대상 사용자를 논의하십시오. 이러한 계층 분리가 API 설계에서 왜 중요한지 소프트웨어 공학 관점에서 분석하십시오.
  2. TypeScript SDK에서 DepthCamera의 이벤트 기반 API(on('skeleton'), on('gesture'), on('crowd'))와 전통적인 폴링(polling) 방식을 비교하십시오. 실시간 인터랙티브 아트에서 이벤트 기반 아키텍처가 더 적합한 이유를 성능, 코드 가독성, 확장성 측면에서 논의하십시오.
  3. InteractionMapper API를 사용한 선언적 매핑과 수동 코딩을 비교하십시오. 제공된 예제 코드에서 거리→밝기 매핑(inverse-square curve)과 손 위치→색상 매핑(linear curve)이 InteractionMapper 없이 구현된다면 코드가 어떻게 달라질지 작성하고, 두 접근 방식의 장단점을 분석하십시오.
  4. REST API 엔드포인트 설계를 평가하십시오. GET /api/v1/sensors/:id와 POST /api/v1/outputs/:id가 RESTful 원칙을 얼마나 잘 따르고 있는지, 그리고 WebSocket 대신 REST API를 사용하는 것이 적합한 상황과 부적합한 상황을 실시간성과 상태 관리 측면에서 논의하십시오.
  5. "Interactive Flower Garden" 예제에서 500개의 꽃 객체를 매 프레임(60 FPS) 업데이트하는 성능을 분석하십시오. bloomLevel과 swayAngle 계산, 거리 측정(distanceTo), 캔버스 렌더링의 계산 복잡도를 평가하고, 1000개 이상의 꽃으로 확장할 때 필요한 최적화 전략(공간 분할, Level of Detail 등)을 제안하십시오.
  6. 센서 데이터 스로틀링(Throttle)과 출력 배치 처리(Batch)의 트레이드오프를 분석하십시오. 표에 제시된 대로 Throttling은 반응 지연을 증가시키고 Batching은 시간 지연을 발생시킵니다. teamLab 스타일의 몰입형 작품에서 이러한 최적화 기법을 적용할 때 관람객 경험을 해치지 않으면서 성능을 향상시키는 최적의 파라미터(fps, batch size)를 결정하는 방법을 논의하십시오.
