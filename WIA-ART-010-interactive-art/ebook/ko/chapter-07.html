개요

2023년, teamLab은 도쿄, 싱가포르, 상하이, 서울에서 동시에 작품을 전시하는 글로벌 프로젝트를 시작했습니다. 각 도시의 작품은 독립적으로 동작하면서도, 관람객 데이터를 클라우드로 집계하여 실시간으로 전 세계 관람 패턴을 분석하고, 특정 시간대에는 네 도시의 작품이 동기화되어 하나의 거대한 인터랙티브 경험을 제공했습니다. 이를 위해 AWS IoT Core를 활용한 분산 아키텍처를 구축했고, 하루 평균 1억 개의 센서 데이터 포인트를 처리했습니다.

WIA-ART-010의 Phase 4는 바로 이러한 클라우드 기반 분산 인터랙티브 아트 시스템을 위한 표준을 제공합니다. AWS IoT, Azure IoT Hub, Google Cloud IoT를 활용한 클라우드 동기화, 실시간 대시보드 구축, 원격 모니터링 및 관리, 데이터 분석 파이프라인을 다룹니다. 이 장에서는 대규모 다중 장소 전시를 위한 클라우드 아키텍처를 실제 구현 예제와 함께 상세히 살펴봅니다.

2. AWS IoT Core 통합

2.1 디바이스 연결 설정

각 인터랙티브 아트 설치는 AWS IoT Core에 "Thing"으로 등록되며, X.509 인증서를 통해 안전하게 연결됩니다. 

// AWS IoT SDK 초기화
import { iot, mqtt } from 'aws-iot-device-sdk-v2';

class WIAIoTClient {
  private connection: mqtt.MqttClientConnection;

  async connect(config: {
    endpoint: string;
    clientId: string;
    certPath: string;
    keyPath: string;
    caPath: string;
  }) {
    const clientBootstrap = new iot.ClientBootstrap();

    const configBuilder = iot.AwsIotMqttConnectionConfigBuilder
      .new_mtls_builder_from_path(config.certPath, config.keyPath)
      .with_certificate_authority_from_path(undefined, config.caPath)
      .with_clean_session(false)
      .with_client_id(config.clientId)
      .with_endpoint(config.endpoint);

    const mqttConfig = configBuilder.build();

    const client = new mqtt.MqttClient(clientBootstrap);
    this.connection = client.new_connection(mqttConfig);

    await this.connection.connect();
    console.log('Connected to AWS IoT Core');
  }

  // 센서 데이터 발행
  async publishSensorData(artworkId: string, sensorId: string, data: any) {
    const topic = `wia/art-010/${artworkId}/sensor/${sensorId}/data`;
    const payload = {
      protocol: 'wia-art-010-iot',
      version: '1.0',
      sensor_id: sensorId,
      timestamp: Date.now() / 1000,
      data: data
    };

    await this.connection.publish(
      topic,
      JSON.stringify(payload),
      mqtt.QoS.AtLeastOnce
    );
  }

  // 출력 명령 구독
  async subscribeOutputCommands(
    artworkId: string,
    callback: (topic: string, payload: Buffer) => void
  ) {
    const topic = `wia/art-010/${artworkId}/output/+/command`;

    await this.connection.subscribe(
      topic,
      mqtt.QoS.AtLeastOnce,
      callback
    );

    console.log(`Subscribed to output commands: ${topic}`);
  }
}

IoT Rules를 사용하여 센서 데이터를 자동으로 데이터베이스, Lambda 함수, SNS 알림 등으로 라우팅합니다.

2.3 AWS Lambda로 실시간 분석

// Lambda 함수: 관람객 수 집계
import { DynamoDBClient, PutItemCommand } from '@aws-sdk/client-dynamodb';

const dynamodb = new DynamoDBClient({ region: 'ap-northeast-2' });

export const handler = async (event) => {
  // IoT Rule에서 전달된 센서 데이터
  const { sensor_id, timestamp, data } = event;

  // 관람객 수 계산
  const crowdCount = data.detected_objects?.length || 0;

  // 시간별 집계 테이블에 저장
  const hour = Math.floor(timestamp / 3600) * 3600;

  await dynamodb.send(new PutItemCommand({
    TableName: 'wia-art-010-crowd-analytics',
    Item: {
      hour: { N: hour.toString() },
      sensor_id: { S: sensor_id },
      crowd_count: { N: crowdCount.toString() },
      timestamp: { N: timestamp.toString() }
    }
  }));

  // 임계값 초과 시 알림
  if (crowdCount > 50) {
    await sns.publish({
      TopicArn: 'arn:aws:sns:ap-northeast-2:123456789:wia-art-crowd-alert',
      Message: `High crowd detected at ${sensor_id}: ${crowdCount} people`,
      Subject: 'WIA-ART-010 Crowd Alert'
    });
  }

  return {
    statusCode: 200,
    body: JSON.stringify({ crowdCount })
  };
};

  url: 'http://localhost:8086',
  token: process.env.INFLUX_TOKEN

const writeApi = client.getWriteApi('wia-org', 'art-010-sensors');

// 센서 데이터 포인트 작성
function writeSensorData(sensorId: string, data: any) {
  const point = new Point('sensor_data')
    .tag('sensor_id', sensorId)
    .tag('artwork_id', 'interactive-2024-001')
    .intField('crowd_count', data.crowd_metrics?.total_people || 0)
    .floatField('movement_intensity', data.crowd_metrics?.movement_intensity || 0)
    .floatField('head_height', data.detected_objects?.[0]?.skeleton.head.y || 0);

  writeApi.writePoint(point);
}

4. 다중 장소 동기화

4.1 teamLab 스타일 글로벌 전시

여러 도시의 작품을 클라우드를 통해 동기화하여 하나의 거대한 인터랙티브 경험을 제공합니다. 

// 글로벌 동기화 매니저
class GlobalSyncManager {
  private iotClient: WIAIoTClient;
  private locations: string[];

  constructor(locations: string[]) {
    this.locations = locations;  // ['seoul', 'tokyo', 'singapore', 'shanghai']
    this.iotClient = new WIAIoTClient();
  }

  async init() {
    await this.iotClient.connect(/* ... */);

    // 모든 도시의 센서 데이터 구독
    for (const location of this.locations) {
      await this.iotClient.subscribe(
        `wia/art-010/global-2024/${location}/sensor/+/data`,
        (data) => this.handleRemoteSensorData(location, data)
      );
    }

    // 로컬 센서 데이터 발행
    this.startLocalSensorPublishing();

    // 정각마다 글로벌 동기화 이벤트
    this.scheduleGlobalSync();
  }

  private handleRemoteSensorData(location: string, data: any) {
    console.log(`Sensor data from ${location}:`, data);

    // 원격 관람객 수를 로컬 비주얼에 반영
    const remoteCrowdCount = data.crowd_metrics.total_people;
    this.visualizeRemoteCrowd(location, remoteCrowdCount);
  }

  private visualizeRemoteCrowd(location: string, count: number) {
    // 예: 다른 도시 관람객 수만큼 파티클 생성
    const particleColor = this.getLocationColor(location);
    for (let i = 0; i < count; i++) {
      particleSystem.emit({
        color: particleColor,
        label: location
      });
    }
  }

  private scheduleGlobalSync() {
    // 매 정각에 모든 도시 동기화
    const msUntilNextHour = (3600000 - (Date.now() % 3600000));

    setTimeout(() => {
      this.triggerGlobalSync();
      setInterval(() => this.triggerGlobalSync(), 3600000);
    }, msUntilNextHour);
  }

  private async triggerGlobalSync() {
    console.log('🌐 Global Sync Event Triggered');

    // 모든 도시에 동기화 명령 전송
    await this.iotClient.publish(
      'wia/art-010/global-2024/sync/command',
      {
        type: 'global_sync',
        timestamp: Date.now() / 1000,
        mode: 'unified_visual',
        duration: 300  // 5분간 동기화
      }
    );

    // 로컬 작품을 동기화 모드로 전환
    this.enterSyncMode();
  }

  private enterSyncMode() {
    // 모든 도시의 비주얼을 통일
    // 예: 모든 LED가 동일한 색상으로 페이드
    ledController.fadeToColor({ h: 200, s: 100, v: 100 }, 5000);

    // 5분 후 정상 모드 복귀
    setTimeout(() => {
      ledController.resumeNormalMode();
    }, 300000);
  }
}

5.2 관람객 행동 분석

// 관람객 체류 시간 및 인기 지점 분석
SELECT
  DATE_TRUNC('hour', timestamp) as hour,
  artwork_zone,
  COUNT(DISTINCT person_id) as unique_visitors,
  AVG(dwell_time_seconds) as avg_dwell_time,
  MAX(crowd_count) as peak_crowd
FROM sensor_analytics
WHERE timestamp > NOW() - INTERVAL '30 days'
GROUP BY hour, artwork_zone
ORDER BY unique_visitors DESC;

// 결과 예시:
// hour               | artwork_zone | unique_visitors | avg_dwell_time | peak_crowd
// 2024-01-20 14:00   | zone-A       | 324            | 185.3         | 52
// 2024-01-20 15:00   | zone-A       | 412            | 203.7         | 68
// 2024-01-20 14:00   | zone-B       | 198            | 142.1         | 34

핵심 요점

  • 클라우드 IoT 통합: AWS IoT Core, Azure IoT Hub를 통해 전 세계 분산 작품을 안전하게 연결하고, X.509 인증서 기반 암호화 통신을 제공합니다.
  • 실시간 데이터 라우팅: IoT Rules로 센서 데이터를 자동으로 DynamoDB, Lambda, SNS로 라우팅하여 저장, 분석, 알림을 즉시 처리합니다.
  • 확장 가능한 대시보드: Grafana + InfluxDB 또는 React 커스텀 대시보드로 실시간 센서 상태, 관람객 수, 작품 성능을 모니터링합니다.
  • 글로벌 동기화: teamLab 스타일의 다중 도시 전시를 위해 클라우드 MQTT 브로커를 통해 전 세계 작품을 동기화하고, 정각마다 통합 비주얼을 제공합니다.
  • 계층적 데이터 보존: 실시간(7일), 집계(1년), 아카이브(영구) 데이터를 InfluxDB, DynamoDB, S3 Glacier에 분산 저장하여 비용과 성능을 최적화합니다.
  • 관람객 분석: SQL 쿼리로 체류 시간, 인기 지점, 피크 시간대를 분석하여 전시 운영을 최적화하고 작가에게 인사이트를 제공합니다.

  4. GlobalSyncManager 클래스의 scheduleGlobalSync() 메서드가 매 정각에 글로벌 동기화를 트리거합니다. 만약 서울은 오후 2시, 도쿄는 3시, 싱가포르는 1시라면, 타임존 차이를 어떻게 처리해야 하는지 논의하십시오. UTC 기준 동기화와 각 도시 현지 시간 기준 동기화의 차이와 관람객 경험에 미치는 영향을 분석하십시오.
  5. 데이터 보존 전략에서 실시간 데이터(7일), 집계 데이터(1년), 아카이브(영구)로 계층화한 이유를 비용, 성능, 규정 준수 관점에서 설명하십시오. InfluxDB에서 S3 Glacier로 데이터를 마이그레이션하는 ETL 파이프라인을 설계하고, 쿼리 성능 저하를 어떻게 최소화할 수 있는지 논의하십시오.
  6. 관람객 행동 분석 SQL 쿼리 결과를 해석하고, zone-A가 zone-B보다 unique_visitors가 164% 높고 avg_dwell_time이 43% 더 긴 이유를 추론하십시오. 이 데이터를 바탕으로 전시 동선 최적화, 작품 배치 조정, 관람객 만족도 향상을 위한 구체적인 개선 방안을 3가지 제안하십시오.
