제4장: Phase 1 - 데이터 형식 표준

弘益人間 · 널리 인간을 이롭게 하라

개요

2022년, 서울 DDP(동대문디자인플라자)에서 국제 인터랙티브 아트 전시가 열렸습니다. 10개국 30명의 작가가 참여하는 대규모 전시였지만, 준비 과정에서 심각한 문제가 발생했습니다. 각 작가가 사용하는 센서 데이터 형식이 모두 달라서 통합 모니터링 시스템을 구축할 수 없었습니다. 일본의 teamLab은 자체 JSON 형식을, 미국의 Rafael Lozano-Hemmer는 OSC 프로토콜을, 한국의 아트센터 나비 작가들은 각각 다른 XML 형식을 사용했습니다. 결국 전시장 관리자는 30개의 서로 다른 모니터링 도구를 동시에 운영해야 했고, 작품 간 연동은 포기할 수밖에 없었습니다.

WIA-ART-010의 Phase 1은 바로 이러한 데이터 형식 파편화 문제를 해결합니다. 센서 데이터, 상호작용 모드, 설치 메타데이터를 위한 통합 형식을 정의하여, 서로 다른 하드웨어와 소프트웨어 플랫폼 간의 원활한 데이터 교환을 가능하게 합니다. 이 장에서는 WIA-ART-010 데이터 형식의 구조, 센서 데이터 표준, 메타데이터 스키마를 실제 코드 예제와 함께 상세히 살펴봅니다.

1. WIA-ART-010 데이터 형식 아키텍처

1.1 핵심 설계 원칙

WIA-ART-010 데이터 형식은 다음의 핵심 원칙을 기반으로 설계되었습니다:

  • 상호운용성(Interoperability): JSON 기반의 개방형 표준으로 모든 플랫폼에서 사용 가능합니다.
  • 확장성(Extensibility): 향후 새로운 센서와 기술을 위한 확장이 용이한 구조입니다.
  • 가독성(Readability): 사람이 읽고 이해하기 쉬운 구조로 디버깅과 수정이 간편합니다.
  • 완전성(Completeness): 작품 재현에 필요한 모든 정보를 포함합니다.
  • 버전 관리(Versioning): 명확한 버전 체계로 하위 호환성을 보장합니다.

1.2 데이터 형식 계층 구조

WIA-ART-010 데이터는 3계층 구조로 조직됩니다:

계층 역할 주요 데이터 파일 확장자
작품 메타데이터 작품 전체 정보 작가, 제목, 설치 요구사항 .wia-manifest.json
센서 설정 센서 구성 정보 센서 유형, 위치, 캘리브레이션 .wia-sensors.json
상호작용 로직 입력-출력 매핑 인터랙션 룰, 파라미터 .wia-interaction.json

3.3 센서 추상화 레이어

WIA-ART-010은 하드웨어 독립적인 센서 추상화 레이어를 정의합니다. 이를 통해 Kinect를 RealSense로 교체하거나 다른 제조사의 센서를 사용할 때도 상위 애플리케이션 코드를 변경할 필요가 없습니다.

추상 센서 유형 지원 하드웨어 표준 출력 업데이트 주기
depth-camera Kinect v2, RealSense D455, Orbbec Astra 깊이 맵 + 골격 데이터 30 FPS
motion-sensor PIR, Ultrasonic, Radar 위치 + 속도 10-100 Hz
touch-sensor Capacitive, Resistive, Force 터치 포인트 + 압력 60 Hz
audio-sensor MEMS Mic, USB Mic Array 오디오 스펙트럼 + 방향 44.1 kHz
biometric-sensor ECG, PPG, GSR 심박수 + 변동성 100-1000 Hz

4.2 매핑 곡선 (Mapping Curves)

센서 입력을 출력으로 변환할 때 사용하는 다양한 곡선 타입을 지원합니다:

곡선 타입 수식 특성 사용 예
linear y = x 일정한 비율 위치 → 색상
exponential y = x² 가속적 증가 속도 → 음량
logarithmic y = log(x) 점진적 감소 거리 → 밝기
inverse-square y = 1/x² 물리적 감쇠 거리 → 밝기 (자연스러움)
sigmoid y = 1/(1+e⁻ˣ) S자 곡선 신호 → 반응 (부드러운 전환)

6. 데이터 검증 및 스키마

6.1 JSON Schema 정의

WIA-ART-010 데이터의 유효성을 검증하기 위해 JSON Schema를 사용합니다: 

{
  "$schema": "https://json-schema.org/draft/2020-12/schema",
  "$id": "https://wia-official.org/schemas/art-010/manifest/v1.0",
  "title": "WIA-ART-010 Artwork Manifest",
  "type": "object",
  "required": ["wia", "artwork", "technical"],
  "properties": {
    "wia": {
      "type": "object",
      "required": ["standard", "version"],
      "properties": {
        "standard": {
          "type": "string",
          "const": "WIA-ART-010"
        },
        "version": {
          "type": "string",
          "pattern": "^\\d+\\.\\d+$"
        }
      }
    },
    "artwork": {
      "type": "object",
      "required": ["id", "title", "artist", "type"],
      "properties": {
        "id": {
          "type": "string",
          "format": "uuid"
        },
        "type": {
          "type": "string",
          "enum": [
            "motion-based",
            "touch-based",
            "sound-based",
            "biometric",
            "environmental",
            "hybrid"
          ]
        }
      }
    }
  }
}

핵심 요점

  • 통합 데이터 형식: WIA-ART-010은 3계층(작품 메타데이터, 센서 설정, 상호작용 로직)의 JSON 기반 표준 형식을 정의하여 플랫폼 간 상호운용성을 보장합니다.
  • 센서 추상화: 하드웨어 독립적인 센서 추상화 레이어를 통해 Kinect, RealSense 등 다양한 센서를 교체 사용할 수 있습니다.
  • 완전한 작품 정보: 작가 정보, 기술 요구사항, 블록체인 출처 증명까지 작품 재현에 필요한 모든 데이터를 포함합니다.
  • 유연한 인터랙션 정의: 센서 입력을 출력으로 변환하는 매핑 규칙을 선언적으로 정의하며, linear, exponential, inverse-square 등 다양한 곡선을 지원합니다.
  • 검증 가능성: JSON Schema를 통한 자동 검증으로 데이터 무결성을 보장하며, 버전 관리 체계로 하위 호환성을 유지합니다.
  • 실전 적용: teamLab, Rafael Lozano-Hemmer 등 실제 작품을 WIA-ART-010 형식으로 표현 가능하며, 작품 이전과 보존이 용이합니다.

복습 질문

  1. WIA-ART-010 데이터 형식의 3계층 구조(.wia-manifest.json, .wia-sensors.json, .wia-interaction.json)를 설명하고, 각 계층이 담당하는 역할과 주요 데이터를 논의하십시오. 왜 하나의 파일이 아닌 3개의 파일로 분리했는지 설계 이유를 분석하십시오.
  2. .wia-manifest.json 파일의 "provenance" 섹션이 블록체인(Ethereum), 컨트랙트 주소, IPFS 해시를 포함하는 이유를 설명하십시오. 이러한 정보가 작품의 진위성 증명과 장기 보존에 어떻게 기여하는지 구체적으로 논의하십시오.
  3. 센서 추상화 레이어의 개념과 필요성을 설명하고, Kinect v2를 Intel RealSense D455로 교체할 때 WIA-ART-010 형식이 어떻게 코드 변경을 최소화하는지 기술적으로 분석하십시오. 표에 제시된 5가지 추상 센서 유형 중 하나를 선택하여 상세히 논의하십시오.
  4. 인터랙션 매핑 곡선(linear, exponential, logarithmic, inverse-square, sigmoid)의 수학적 특성을 비교하고, 각각이 적합한 사용 사례를 제시하십시오. 특히 inverse-square 곡선이 물리적으로 자연스러운 거리-밝기 매핑을 제공하는 이유를 과학적으로 설명하십시오.
  5. teamLab의 "Forest of Resonating Lamps" 작품에서 "propagation" 메커니즘(wave 알고리즘, speed: 2.0, decay: 0.8, max_hops: 5)을 상세히 분석하십시오. 이 파라미터들이 빛의 전파 패턴에 어떤 영향을 미치며, decay 값을 0.5로 변경하면 관람객 경험이 어떻게 달라지는지 논의하십시오.
  6. Rafael Lozano-Hemmer의 "Pulse Room" 구현에서 "persistence" 기능(duration: 300초, queue_size: 100)의 예술적 의도를 분석하십시오. 이전 관람객의 심박 패턴을 저장하고 재생하는 것이 작품의 메시지와 관람객 경험에 어떤 의미를 부여하는지, 그리고 WIA-ART-010 형식이 이러한 복잡한 시간적 인터랙션을 어떻게 표현하는지 논의하십시오.
