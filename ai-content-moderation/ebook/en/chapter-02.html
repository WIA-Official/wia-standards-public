Chapter 2

Text Moderation with NLP

2.1 Natural Language Processing for Moderation

Text moderation is the cornerstone of content safety systems. With billions of text-based messages, comments, posts, and reviews generated daily across digital platforms, automated text analysis using Natural Language Processing (NLP) has become essential. Modern NLP systems can detect hate speech, harassment, spam, profanity, misinformation, and other policy violations with remarkable accuracy, but the complexity of human language presents ongoing challenges.

The evolution of text moderation mirrors the broader development of NLP technology. Early systems relied on simple keyword matching and regular expressions—if a message contained certain banned words, it was flagged or removed. While computationally efficient, these approaches generated massive numbers of false positives (flagging benign content) and false negatives (missing violations that used creative spelling or slang).

Modern text moderation leverages transformer-based models like BERT, RoBERTa, and GPT, which understand context, semantics, and linguistic nuance. These models can distinguish between hate speech and reclaimed language within communities, identify sarcasm, understand multi-turn conversations, and even detect subtle forms of manipulation or coordinated harassment campaigns.

2.2 Hate Speech Detection

Hate speech—content that attacks or dehumanizes individuals or groups based on protected characteristics such as race, religion, ethnicity, gender, sexual orientation, or disability—represents one of the most challenging moderation problems. The difficulty stems from the inherently contextual and culturally-dependent nature of hateful language.

Hate Speech Type Characteristics Detection Challenges NLP Approach
Explicit Slurs Direct use of derogatory terms Reclaimed usage, educational context Keyword + context analysis
Coded Language Dog whistles, euphemisms, symbols Evolving terminology, plausible deniability Semantic similarity, pattern recognition
Dehumanization Comparing groups to animals/objects Metaphorical language, satire Dependency parsing, sentiment analysis
Calls for Violence Incitement to harm specific groups Hypotheticals, historical references Intent classification, threat detection
Stereotyping Negative generalizations about groups Statistical statements, comedy Bias detection, generalization analysis 
// TypeScript: Hate Speech Detection System

interface HateSpeechClassifier {
    text: string;
    language: string;
    context?: ConversationContext;
    userMetadata?: UserMetadata;
}

interface HateSpeechResult {
    isHateSpeech: boolean;
    confidence: number;
    targetedGroups: string[];
    hateSpeechType: HateSpeechType;
    severity: 'LOW' | 'MEDIUM' | 'HIGH' | 'CRITICAL';
    explanation: string;
    requiresReview: boolean;
}

enum HateSpeechType {
    EXPLICIT_SLUR = 'EXPLICIT_SLUR',
    CODED_LANGUAGE = 'CODED_LANGUAGE',
    DEHUMANIZATION = 'DEHUMANIZATION',
    VIOLENCE_INCITEMENT = 'VIOLENCE_INCITEMENT',
    STEREOTYPING = 'STEREOTYPING',
    THREAT = 'THREAT'
}

class NLPHateSpeechDetector {
    private transformerModel: TransformerModel;
    private slurDatabase: SlurDatabase;
    private contextAnalyzer: ContextualAnalyzer;

    /**
     * Analyzes text for hate speech using multi-stage NLP pipeline
     */
    async detectHateSpeech(input: HateSpeechClassifier): Promise {
        // Stage 1: Lexical Analysis - Check for known slurs and hate terms
        const lexicalSignals = await this.lexicalAnalysis(input.text, input.language);

        // Stage 2: Semantic Analysis - Understand meaning and intent
        const semanticSignals = await this.semanticAnalysis(input.text, input.language);

        // Stage 3: Contextual Analysis - Consider conversation history and user info
        const contextualSignals = input.context 
            ? await this.contextAnalyzer.analyze(input.text, input.context)
            : { neutralizingContext: false, reclaimedLanguage: false };

        // Stage 4: Aggregate signals and make decision
        return this.aggregateAndDecide(
            lexicalSignals,
            semanticSignals,
            contextualSignals,
            input.userMetadata
        );
    }

    /**
     * Lexical analysis: keyword and pattern matching
     */
    private async lexicalAnalysis(
        text: string,
        language: string
    ): Promise {
        const normalizedText = this.normalizeText(text);
        const tokens = this.tokenize(normalizedText);

        // Check against slur database
        const matchedSlurs = this.slurDatabase.findMatches(tokens, language);

        // Pattern matching for coded language (e.g., "1488", "(((echoes)))")
        const codedLanguagePatterns = this.detectCodedPatterns(normalizedText);

        return {
            containsSlurs: matchedSlurs.length > 0,
            slurSeverity: this.calculateSlurSeverity(matchedSlurs),
            codedLanguageDetected: codedLanguagePatterns.length > 0,
            patterns: codedLanguagePatterns
        };
    }

    /**
     * Semantic analysis using transformer models
     */
    private async semanticAnalysis(
        text: string,
        language: string
    ): Promise {
        // Use fine-tuned transformer for hate speech classification
        const embedding = await this.transformerModel.encode(text);
        const prediction = await this.transformerModel.classify(embedding);

        // Detect dehumanization through metaphor analysis
        const dehumanizationScore = await this.detectDehumanization(text);

        // Detect violence incitement
        const violenceScore = await this.detectViolenceIncitement(text);

        // Identify targeted groups
        const targetedGroups = await this.identifyTargetedGroups(text);

        return {
            hateSpeechProbability: prediction.probability,
            hateSpeechType: prediction.type,
            dehumanizationScore,
            violenceScore,
            targetedGroups,
            sentiment: prediction.sentiment
        };
    }

    /**
     * Aggregate signals and make final decision
     */
    private aggregateAndDecide(
        lexical: LexicalSignals,
        semantic: SemanticSignals,
        contextual: ContextualSignals,
        userMetadata?: UserMetadata
    ): HateSpeechResult {
        let confidence = 0;
        let severity: 'LOW' | 'MEDIUM' | 'HIGH' | 'CRITICAL' = 'LOW';

        // Explicit slurs with no neutralizing context
        if (lexical.containsSlurs && !contextual.neutralizingContext) {
            confidence += 0.7;
            severity = lexical.slurSeverity;
        }

        // Transformer model prediction
        confidence += semantic.hateSpeechProbability * 0.3;

        // Boost confidence for dehumanization or violence
        if (semantic.dehumanizationScore > 0.7) {
            confidence += 0.15;
            severity = this.escalateSeverity(severity, 'HIGH');
        }

        if (semantic.violenceScore > 0.7) {
            confidence += 0.2;
            severity = this.escalateSeverity(severity, 'CRITICAL');
        }

        // Reduce confidence for reclaimed language within community
        if (contextual.reclaimedLanguage) {
            confidence *= 0.5;
        }

        // Adjust based on user history
        if (userMetadata?.previousViolations && userMetadata.previousViolations > 3) {
            confidence *= 1.1;
        }

        const isHateSpeech = confidence > 0.6;
        const requiresReview = confidence > 0.5 && confidence < 0.9;

        return {
            isHateSpeech,
            confidence: Math.min(confidence, 1.0),
            targetedGroups: semantic.targetedGroups,
            hateSpeechType: semantic.hateSpeechType,
            severity,
            explanation: this.generateExplanation(lexical, semantic, contextual),
            requiresReview
        };
    }

    /**
     * Detect dehumanization through metaphor analysis
     */
    private async detectDehumanization(text: string): Promise {
        // Look for patterns like comparing groups to animals, vermin, diseases
        const dehumanizingMetaphors = [
            /\b(rats|vermin|cockroaches|parasites|plague|disease|infection)\b/i,
            /\b(animals|beasts|savages|barbarians|primitives)\b/i
        ];

        // Also check if text contains group identifiers
        const containsGroupReference = /\b(they|them|those people|these people)\b/i.test(text);

        for (const pattern of dehumanizingMetaphors) {
            if (pattern.test(text) && containsGroupReference) {
                return 0.8;
            }
        }

        return 0.0;
    }

    /**
     * Detect incitement to violence
     */
    private async detectViolenceIncitement(text: string): Promise {
        const violenceKeywords = [
            /\b(kill|murder|execute|exterminate|eliminate|destroy)\b/i,
            /\b(attack|assault|harm|hurt|beat|strike)\b/i,
            /\b(should (die|be killed|be shot))\b/i
        ];

        for (const pattern of violenceKeywords) {
            if (pattern.test(text)) {
                return 0.85;
            }
        }

        return 0.0;
    }

    /**
     * Identify which groups are being targeted
     */
    private async identifyTargetedGroups(text: string): Promise {
        const groups: string[] = [];
        
        // Use NER (Named Entity Recognition) + custom group detection
        const groupPatterns = {
            'racial': /\b(race|racial|ethnic|ethnicity)\b/i,
            'religious': /\b(religion|religious|muslim|jewish|christian|hindu)\b/i,
            'gender': /\b(women|men|female|male|gender)\b/i,
            'lgbtq': /\b(gay|lesbian|trans|queer|lgbtq)\b/i,
            'disability': /\b(disabled|disability|handicapped)\b/i
        };

        for (const [group, pattern] of Object.entries(groupPatterns)) {
            if (pattern.test(text)) {
                groups.push(group);
            }
        }

        return groups;
    }

    private normalizeText(text: string): string {
        // Handle common evasion techniques
        return text
            .toLowerCase()
            .replace(/[^\w\s]/g, ' ')      // Remove special characters
            .replace(/\s+/g, ' ')           // Normalize whitespace
            .replace(/(.)\1{2,}/g, '$1');  // Remove repeated characters
    }

    private tokenize(text: string): string[] {
        return text.split(/\s+/);
    }

    private detectCodedPatterns(text: string): string[] {
        const patterns: string[] = [];
        
        // Common hate group codes
        if (/\b1488\b/.test(text)) patterns.push('1488');
        if (/\(\(\(.+\)\)\)/.test(text)) patterns.push('echoes');
        
        return patterns;
    }

    private calculateSlurSeverity(slurs: string[]): 'LOW' | 'MEDIUM' | 'HIGH' | 'CRITICAL' {
        if (slurs.length === 0) return 'LOW';
        // Logic to determine severity based on specific slurs
        return 'HIGH';
    }

    private escalateSeverity(
        current: 'LOW' | 'MEDIUM' | 'HIGH' | 'CRITICAL',
        proposed: 'LOW' | 'MEDIUM' | 'HIGH' | 'CRITICAL'
    ): 'LOW' | 'MEDIUM' | 'HIGH' | 'CRITICAL' {
        const severityOrder = ['LOW', 'MEDIUM', 'HIGH', 'CRITICAL'];
        const currentIndex = severityOrder.indexOf(current);
        const proposedIndex = severityOrder.indexOf(proposed);
        return severityOrder[Math.max(currentIndex, proposedIndex)] as any;
    }

    private generateExplanation(
        lexical: LexicalSignals,
        semantic: SemanticSignals,
        contextual: ContextualSignals
    ): string {
        const reasons: string[] = [];
        
        if (lexical.containsSlurs) {
            reasons.push('Contains hate speech terminology');
        }
        if (semantic.dehumanizationScore > 0.7) {
            reasons.push('Uses dehumanizing language');
        }
        if (semantic.violenceScore > 0.7) {
            reasons.push('Contains incitement to violence');
        }
        if (contextual.reclaimedLanguage) {
            reasons.push('May be reclaimed language within community');
        }

        return reasons.join('; ');
    }
}

interface LexicalSignals {
    containsSlurs: boolean;
    slurSeverity: 'LOW' | 'MEDIUM' | 'HIGH' | 'CRITICAL';
    codedLanguageDetected: boolean;
    patterns: string[];
}

interface SemanticSignals {
    hateSpeechProbability: number;
    hateSpeechType: HateSpeechType;
    dehumanizationScore: number;
    violenceScore: number;
    targetedGroups: string[];
    sentiment: number;
}

interface ContextualSignals {
    neutralizingContext: boolean;
    reclaimedLanguage: boolean;
}

interface ConversationContext {
    previousMessages: string[];
    participants: string[];
    channelType: string;
}

interface UserMetadata {
    userId: string;
    previousViolations?: number;
    accountAge?: number;
    reputation?: number;
}

2.3 Spam and Manipulation Detection

Spam moderation requires detecting unwanted promotional content, scams, phishing attempts, and coordinated manipulation campaigns. Unlike hate speech, spam detection relies heavily on behavioral patterns and metadata analysis in addition to content analysis.

Spam Type Indicators Detection Method
Commercial Spam URLs, promotional language, repetition Keyword matching, link analysis, posting frequency
Phishing Urgent language, impersonation, suspicious links URL reputation, brand detection, urgency analysis
SEO Spam Keyword stuffing, irrelevant links Keyword density, link diversity analysis
Bot Networks Coordinated posting, similar content Behavioral clustering, content similarity
Engagement Bait "Like if...", "Share to...", vote manipulation Template matching, engagement pattern analysis

2.4 Context Understanding

One of the most significant advances in modern text moderation is the ability to understand context. The same words can have entirely different meanings depending on who says them, where they're said, and the surrounding conversation. Sarcasm, satire, educational content, and news reporting all require sophisticated contextual understanding.

Context Matters: The sentence "That's so gay" could be hate speech or benign slang depending on context. A news article about terrorism needs different treatment than terrorist propaganda. Educational content about historical atrocities shouldn't be flagged like glorification of violence. Modern NLP systems use multi-turn context windows, user relationship graphs, and content category classification to make these nuanced distinctions.

Key Takeaways

  • Transformer Revolution: Modern NLP uses transformer models (BERT, RoBERTa) that understand context and semantics far better than keyword matching
  • Multi-Stage Pipeline: Effective text moderation combines lexical, semantic, and contextual analysis for maximum accuracy
  • Hate Speech Complexity: Detecting hate speech requires understanding slurs, coded language, dehumanization, and targeted groups while avoiding false positives on reclaimed language
  • Behavioral Signals: Spam detection relies on posting patterns, account behavior, and coordination analysis in addition to content features
  • Context is King: The same text can be harmless or harmful depending on who posts it, where, and in what conversational context
  • Continuous Learning: Language evolves rapidly; moderation systems must continuously retrain on new data to catch emerging slang and evasion techniques

Practical Implementation Checklist

  • Manifest integrity — every published manifest is signed via Sigstore DSSE. Tampering is detectable without trusting an intermediary. Updates are issued as new manifests; the prior manifest is preserved with a supersede marker.
  • Retention — keep signed evidence for at least seven years to satisfy ISO/IEC 17065:2012 audit expectations. Some regulators require longer retention; in that case the deployment policy MUST extend the window rather than relying on the defaults.
  • Versioning — follow Semantic Versioning 2.0.0. Major version bumps require at least a 90-day overlap with the prior major version on every WIA-published reference implementation.
  • Negative-result reporting — a skipped indicator without recorded justification is treated as a failed indicator at audit time.
  • Key rotation — manifest signing keys are rotated per the deployment policy. The previous key is preserved in a published key history so that auditors can verify historical manifests offline.

Reference Topologies

Implementations declare their operational topology in the manifest: single-tenant edge (one runtime per organization), multi-tenant gateway (shared runtime with header-based isolation), federated mesh (multiple runtimes peer to one another and publish manifests to a directory service), or air-gapped batch (signed evidence package transported out-of-band). Each topology ships with reference test vectors so that implementations can validate their deployment matches the canonical pattern.

Open Governance

Issues, errata, and proposals are tracked at github.com/WIA-Official/wia-standards/issues with the appropriate label. The WIA Standards working group reviews open issues at the start of every minor release cycle and publishes the resulting decision log alongside the release notes. Errata are issued as patch releases; new normative requirements trigger minor bumps; backwards-incompatible changes trigger major bumps.
