Chapter 3

Image & Video Moderation

3.1 Computer Vision for Content Safety

Visual content moderation presents unique challenges compared to text analysis. Images and videos can convey harmful content through depictions that bypass language entirely—graphic violence, sexual content, disturbing imagery, and hateful symbols all communicate visually. With over 350 million photos uploaded to Facebook daily and 500 hours of video uploaded to YouTube every minute, manual review is impossible. Computer vision systems powered by deep learning have become essential infrastructure for platform safety.

Modern image moderation systems use Convolutional Neural Networks (CNNs) and vision transformers to analyze visual content. These models can detect nudity, violence, weapons, drugs, hate symbols, graphic injury, and other policy-violating imagery with high accuracy. However, visual content moderation faces unique challenges: context matters enormously (artistic nudity vs. pornography, news violence vs. glorification), cultural differences in acceptable imagery, and adversarial techniques like image manipulation to evade detection.

Visual Content Type Detection Approach Accuracy Key Challenges
NSFW (Adult Content) Skin detection, body part recognition, pose estimation 90-95% Art vs pornography, beach photos, medical content
Violence & Gore Blood detection, injury recognition, weapon identification 85-90% News coverage, action movies, fake blood/effects
Hate Symbols Object detection, OCR for text in images 85-92% Context (education vs promotion), varied designs
CSAM Perceptual hashing (PhotoDNA), age estimation 95-99% False positives on family photos, privacy concerns
Dangerous Acts Action recognition, scene understanding 75-85% Context (stunts vs real danger), cultural activities

3.2 NSFW Detection System

Not Safe For Work (NSFW) content detection is one of the most deployed computer vision moderation systems. While nudity detection might seem straightforward, distinguishing between acceptable contexts (art, education, medical, breastfeeding) and policy violations (pornography, sexual exploitation) requires sophisticated understanding of context, intent, and cultural norms. 

// TypeScript: Image & Video Moderation System

interface ImageModerationRequest {
    imageUrl: string;
    imageId: string;
    context?: ImageContext;
    priority: 'REAL_TIME' | 'BATCH';
}

interface ImageContext {
    source: 'USER_UPLOAD' | 'URL_SHARE' | 'PROFILE_PICTURE' | 'ADVERTISEMENT';
    associatedText?: string;
    userAge?: number;
    platform: string;
}

interface ImageModerationResult {
    imageId: string;
    violations: ImageViolation[];
    overallScore: number;
    action: 'APPROVE' | 'BLUR' | 'REMOVE' | 'REVIEW';
    confidence: number;
    requiresHumanReview: boolean;
    processingTimeMs: number;
}

interface ImageViolation {
    type: ViolationType;
    severity: 'LOW' | 'MEDIUM' | 'HIGH' | 'CRITICAL';
    confidence: number;
    boundingBoxes?: BoundingBox[];
    explanation: string;
}

enum ViolationType {
    NSFW_NUDITY = 'NSFW_NUDITY',
    NSFW_SEXUAL = 'NSFW_SEXUAL',
    VIOLENCE = 'VIOLENCE',
    GORE = 'GORE',
    WEAPONS = 'WEAPONS',
    DRUGS = 'DRUGS',
    HATE_SYMBOLS = 'HATE_SYMBOLS',
    CSAM = 'CSAM',
    SELF_HARM = 'SELF_HARM',
    SHOCKING = 'SHOCKING'
}

interface BoundingBox {
    x: number;
    y: number;
    width: number;
    height: number;
    label: string;
    confidence: number;
}

class ImageModerationSystem {
    private nsfwModel: NSFWDetectionModel;
    private violenceModel: ViolenceDetectionModel;
    private objectDetector: ObjectDetectionModel;
    private hashMatcher: PerceptualHashMatcher;
    private sceneClassifier: SceneClassificationModel;

    /**
     * Analyzes image for policy violations using multiple CV models
     */
    async moderateImage(request: ImageModerationRequest): Promise {
        const startTime = Date.now();
        const violations: ImageViolation[] = [];

        // Step 1: Perceptual Hash Matching (fastest, catches known bad images)
        const hashMatch = await this.hashMatcher.checkImage(request.imageUrl);
        if (hashMatch.matched && hashMatch.severity === 'CRITICAL') {
            return {
                imageId: request.imageId,
                violations: [{
                    type: ViolationType.CSAM,
                    severity: 'CRITICAL',
                    confidence: 1.0,
                    explanation: 'Matched known prohibited content'
                }],
                overallScore: 1.0,
                action: 'REMOVE',
                confidence: 1.0,
                requiresHumanReview: false,
                processingTimeMs: Date.now() - startTime
            };
        }

        // Step 2: NSFW Detection
        const nsfwResult = await this.nsfwModel.analyze(request.imageUrl);
        if (nsfwResult.score > 0.5) {
            violations.push({
                type: this.classifyNSFW(nsfwResult),
                severity: this.calculateSeverity(nsfwResult.score),
                confidence: nsfwResult.score,
                boundingBoxes: nsfwResult.regions,
                explanation: `NSFW content detected: ${nsfwResult.category}`
            });
        }

        // Step 3: Violence & Gore Detection
        const violenceResult = await this.violenceModel.analyze(request.imageUrl);
        if (violenceResult.score > 0.6) {
            violations.push({
                type: violenceResult.hasBlood ? ViolationType.GORE : ViolationType.VIOLENCE,
                severity: this.calculateSeverity(violenceResult.score),
                confidence: violenceResult.score,
                explanation: `Violence detected: ${violenceResult.description}`
            });
        }

        // Step 4: Object Detection (weapons, drugs, hate symbols)
        const objects = await this.objectDetector.detect(request.imageUrl);
        const prohibitedObjects = objects.filter(obj => 
            ['weapon', 'drug', 'hate_symbol'].includes(obj.category)
        );

        for (const obj of prohibitedObjects) {
            violations.push({
                type: this.mapObjectToViolation(obj.category),
                severity: 'HIGH',
                confidence: obj.confidence,
                boundingBoxes: [obj.boundingBox],
                explanation: `Detected: ${obj.label}`
            });
        }

        // Step 5: Scene Classification for Context
        const scene = await this.sceneClassifier.classify(request.imageUrl);
        const contextualAdjustment = this.adjustForContext(
            violations,
            scene,
            request.context
        );

        // Aggregate and decide
        return this.makeDecision(
            request.imageId,
            contextualAdjustment,
            request.context,
            Date.now() - startTime
        );
    }

    /**
     * Classifies NSFW content into specific categories
     */
    private classifyNSFW(result: NSFWResult): ViolationType {
        if (result.category === 'pornography' || result.explicitScore > 0.8) {
            return ViolationType.NSFW_SEXUAL;
        }
        return ViolationType.NSFW_NUDITY;
    }

    /**
     * Adjusts violations based on contextual understanding
     */
    private adjustForContext(
        violations: ImageViolation[],
        scene: SceneClassification,
        context?: ImageContext
    ): ImageViolation[] {
        return violations.map(violation => {
            let adjustedConfidence = violation.confidence;

            // Art galleries, museums reduce NSFW severity
            if (scene.isArt && violation.type === ViolationType.NSFW_NUDITY) {
                adjustedConfidence *= 0.6;
                violation.explanation += ' (possible artistic context)';
            }

            // News context affects violence interpretation
            if (scene.isNews && violation.type === ViolationType.VIOLENCE) {
                adjustedConfidence *= 0.7;
                violation.explanation += ' (possible news content)';
            }

            // Medical context
            if (scene.isMedical && 
                [ViolationType.NSFW_NUDITY, ViolationType.GORE].includes(violation.type)) {
                adjustedConfidence *= 0.5;
                violation.explanation += ' (possible medical/educational content)';
            }

            // Profile pictures have stricter standards
            if (context?.source === 'PROFILE_PICTURE') {
                adjustedConfidence *= 1.2;
            }

            return {
                ...violation,
                confidence: Math.min(adjustedConfidence, 1.0)
            };
        });
    }

    /**
     * Makes final moderation decision
     */
    private makeDecision(
        imageId: string,
        violations: ImageViolation[],
        context?: ImageContext,
        processingTimeMs?: number
    ): ImageModerationResult {
        if (violations.length === 0) {
            return {
                imageId,
                violations: [],
                overallScore: 0,
                action: 'APPROVE',
                confidence: 1.0,
                requiresHumanReview: false,
                processingTimeMs: processingTimeMs || 0
            };
        }

        // Calculate aggregate score
        const maxConfidence = Math.max(...violations.map(v => v.confidence));
        const criticalViolations = violations.filter(v => v.severity === 'CRITICAL');

        // Critical violations = immediate removal
        if (criticalViolations.length > 0 && maxConfidence > 0.9) {
            return {
                imageId,
                violations,
                overallScore: maxConfidence,
                action: 'REMOVE',
                confidence: maxConfidence,
                requiresHumanReview: false,
                processingTimeMs: processingTimeMs || 0
            };
        }

        // High confidence violations = blur or remove
        if (maxConfidence > 0.8) {
            return {
                imageId,
                violations,
                overallScore: maxConfidence,
                action: 'BLUR',
                confidence: maxConfidence,
                requiresHumanReview: maxConfidence < 0.95,
                processingTimeMs: processingTimeMs || 0
            };
        }

        // Medium confidence = human review
        if (maxConfidence > 0.6) {
            return {
                imageId,
                violations,
                overallScore: maxConfidence,
                action: 'REVIEW',
                confidence: maxConfidence,
                requiresHumanReview: true,
                processingTimeMs: processingTimeMs || 0
            };
        }

        return {
            imageId,
            violations,
            overallScore: maxConfidence,
            action: 'APPROVE',
            confidence: 1 - maxConfidence,
            requiresHumanReview: false,
            processingTimeMs: processingTimeMs || 0
        };
    }

    private calculateSeverity(score: number): 'LOW' | 'MEDIUM' | 'HIGH' | 'CRITICAL' {
        if (score > 0.9) return 'CRITICAL';
        if (score > 0.75) return 'HIGH';
        if (score > 0.6) return 'MEDIUM';
        return 'LOW';
    }

    private mapObjectToViolation(category: string): ViolationType {
        const mapping: Record = {
            'weapon': ViolationType.WEAPONS,
            'drug': ViolationType.DRUGS,
            'hate_symbol': ViolationType.HATE_SYMBOLS
        };
        return mapping[category] || ViolationType.SHOCKING;
    }
}

interface NSFWResult {
    score: number;
    category: 'nudity' | 'pornography' | 'safe';
    explicitScore: number;
    regions: BoundingBox[];
}

interface ViolenceResult {
    score: number;
    hasBlood: boolean;
    hasWeapons: boolean;
    description: string;
}

interface SceneClassification {
    isArt: boolean;
    isNews: boolean;
    isMedical: boolean;
    isEducational: boolean;
    primaryScene: string;
}

3.3 Video Moderation

Video moderation presents additional complexity beyond static images. Videos combine visual content, audio, text (captions, overlays), and temporal dynamics. A 10-minute video at 30fps contains 18,000 frames—analyzing every frame is computationally prohibitive. Modern video moderation systems use intelligent sampling strategies, analyzing keyframes, scene changes, and audio peaks rather than every frame.

Video moderation must also detect policy violations that emerge over time: a video might start innocuously but transition to violent content, or harmful behavior might only become clear when viewed in sequence. Live streaming presents even greater challenges, requiring real-time analysis with minimal latency to enable immediate intervention.

Video Moderation Technique Approach Use Case
Keyframe Analysis Sample frames at scene changes and intervals Uploaded videos, efficient processing
Temporal Analysis 3D CNNs analyzing frame sequences Action detection, context understanding
Audio Analysis Speech-to-text + acoustic analysis Hate speech, copyright, violence sounds
Live Stream Monitoring Real-time frame sampling + audio Live broadcasts, immediate intervention
Fingerprinting Video hash matching against database Copyright, known violating content

3.4 Deepfake and Manipulation Detection

The rise of deepfake technology—AI-generated synthetic media that convincingly depicts people saying or doing things they never did—presents an emerging challenge for visual content moderation. Deepfakes can be used for non-consensual pornography, fraud, political manipulation, and misinformation. Detecting deepfakes requires analyzing subtle artifacts in facial movements, blinking patterns, lighting inconsistencies, and temporal coherence across frames.

Deepfake Arms Race: Deepfake detection is an adversarial problem. As detection systems improve, generation techniques evolve to evade them. Platforms must continuously update detection models and combine multiple detection approaches (facial analysis, audio-visual synchronization, provenance tracking) to stay ahead of synthetic media threats.

Key Takeaways

  • Computer Vision Scale: Modern CV models can analyze millions of images daily with 90%+ accuracy on clear violations like NSFW content and violence
  • Context Understanding: Effective visual moderation requires understanding context—artistic nudity vs pornography, news violence vs glorification requires scene classification
  • Multi-Model Approach: Robust systems combine NSFW detection, violence detection, object detection, hash matching, and scene classification for comprehensive coverage
  • Video Complexity: Video moderation requires analyzing visual, audio, and temporal dimensions through keyframe sampling and sequential analysis
  • Real-Time Challenges: Live streaming requires ultra-low latency analysis with immediate intervention capabilities for harmful content
  • Deepfake Threat: Synthetic media detection is an ongoing arms race requiring continuous model updates and multi-faceted detection approaches
  • Hash Matching Critical: Perceptual hashing (PhotoDNA) provides near-perfect detection of known prohibited content like CSAM with minimal false positives

Reference Standards and Implementation Notes

This standard is designed to interoperate with established international frameworks. Implementations SHOULD treat the following as the single source of truth and use this PHASE document as the interoperability layer that maps existing data into a machine-readable, signed, and auditable shape.

  • OpenAPI Specification 3.1 — REST API definition
  • JSON Schema 2020-12 — schema validation
  • ISO/IEC 27001:2022 — information security management
  • ISO/IEC 27018:2019 — PII protection in public clouds
  • ISO/IEC 17065:2012 — product certification bodies
  • CycloneDX 1.5 / SPDX 2.3 — software bill of materials
  • Sigstore (DSSE envelope, Rekor transparency log) — signature anchoring
  • in-toto Attestation Framework 1.0 — build provenance
  • IETF RFC 9700 — OAuth 2.1
  • IETF RFC 9457 — Problem Details for HTTP APIs
  • IETF RFC 8615 — well-known URIs

Practical Implementation Checklist

  • Manifest integrity — every published manifest is signed via Sigstore DSSE. Tampering is detectable without trusting an intermediary. Updates are issued as new manifests; the prior manifest is preserved with a supersede marker.
  • Retention — keep signed evidence for at least seven years to satisfy ISO/IEC 17065:2012 audit expectations. Some regulators require longer retention; in that case the deployment policy MUST extend the window rather than relying on the defaults.
  • Versioning — follow Semantic Versioning 2.0.0. Major version bumps require at least a 90-day overlap with the prior major version on every WIA-published reference implementation.
  • Negative-result reporting — a skipped indicator without recorded justification is treated as a failed indicator at audit time.
  • Key rotation — manifest signing keys are rotated per the deployment policy. The previous key is preserved in a published key history so that auditors can verify historical manifests offline.

Reference Topologies

Implementations declare their operational topology in the manifest: single-tenant edge (one runtime per organization), multi-tenant gateway (shared runtime with header-based isolation), federated mesh (multiple runtimes peer to one another and publish manifests to a directory service), or air-gapped batch (signed evidence package transported out-of-band). Each topology ships with reference test vectors so that implementations can validate their deployment matches the canonical pattern.

Open Governance

Issues, errata, and proposals are tracked at github.com/WIA-Official/wia-standards/issues with the appropriate label. The WIA Standards working group reviews open issues at the start of every minor release cycle and publishes the resulting decision log alongside the release notes. Errata are issued as patch releases; new normative requirements trigger minor bumps; backwards-incompatible changes trigger major bumps.
