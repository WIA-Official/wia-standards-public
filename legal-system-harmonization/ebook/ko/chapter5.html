제 5: Phase 2 - API Interface

5.1 Introduction to Legal APIs

Phase 2 of WIA-UNI-014 builds on the data format foundation by providing RESTful APIs that enable programmatic access to legal systems. These APIs allow applications, services, and users to query legal databases, retrieve documents, check legal status, and perform legal transactions programmatically.

The API layer is crucial for modern legal system integration because it enables:

  • Real-time legal status queries
  • Automated compliance checking
  • Integration with business applications
  • Mobile and web application development
  • Cross-system legal transactions

5.2 API Design Principles

WIA-UNI-014 APIs follow modern REST principles and best practices:

RESTful Architecture

  • Resource-Based: URLs represent legal resources (documents, cases, properties)
  • HTTP Methods: GET for retrieval, POST for creation, PUT for updates, DELETE for removal
  • Stateless: Each request contains all necessary information
  • Cacheable: Responses indicate cacheability to improve performance

Security First

  • HTTPS Only: All API communication over TLS 1.3
  • OAuth 2.0: Industry-standard authorization framework
  • API Keys: Client identification and rate limiting
  • Scoped Permissions: Fine-grained access control

Developer Friendly

  • OpenAPI Specification: Machine-readable API documentation
  • Consistent Naming: Predictable URL and parameter names
  • Error Handling: Clear, actionable error messages
  • Versioning: Stable APIs with backward compatibility

5.6 Authentication and Authorization

WIA-UNI-014 APIs use OAuth 2.0 for secure authentication:

OAuth 2.0 Flow

1. Obtain Access Token:
POST /oauth/token
Content-Type: application/x-www-form-urlencoded

grant_type=client_credentials
&client_id={your_client_id}
&client_secret={your_client_secret}
&scope=legal:read legal:search

Response:
{
  "access_token": "eyJhbGciOiJSUzI1NiIs...",
  "token_type": "Bearer",
  "expires_in": 3600,
  "scope": "legal:read legal:search"
}

2. Use Access Token:
GET /api/v1/documents/{id}
Authorization: Bearer eyJhbGciOiJSUzI1NiIs...

Permission Scopes

Scope Permissions Use Cases
legal:read Read public legal documents Document retrieval, legal research
legal:search Search legal databases Legal discovery, compliance checking
property:read Read property records Due diligence, ownership verification
property:write Initiate property transactions Property transfers, registration
contract:create Create and register contracts Contract management systems

5.7 Rate Limiting and Quotas

To ensure fair access and system stability, APIs implement rate limiting:

Response Headers:
X-RateLimit-Limit: 1000
X-RateLimit-Remaining: 987
X-RateLimit-Reset: 1642582800

When limit exceeded:
HTTP/1.1 429 Too Many Requests
Retry-After: 3600
{
  "error": "rate_limit_exceeded",
  "message": "API rate limit exceeded. Retry after 3600 seconds.",
  "limit": 1000,
  "remaining": 0,
  "resetAt": "2025-01-25T15:00:00Z"
}

Rate Limit Tiers

Tier Requests/Hour Concurrent Requests

Basic 1,000 10
Professional 10,000 50
Enterprise 100,000 200

5.10 SDK and Client Libraries

Official SDKs simplify API integration in popular programming languages:

TypeScript/JavaScript Example

import { WIALegalClient } from '@wia/legal-sdk';

const client = new WIALegalClient({
  apiKey: process.env.WIA_API_KEY,
  environment: 'production'
});

// Get legal document
const document = await client.documents.get('WIA-UNI-014-LEGAL-abc123');
console.log(document.metadata.title);

// Search for laws
const results = await client.search({
  query: 'property rights',
  type: 'law',
  jurisdiction: 'unified'
});

// Check property ownership
const ownership = await client.properties.getOwner('WIA-UNI-014-PROP-seoul-123');
console.log(ownership.currentOwner.name);

제 Summary: Key Takeaways

  1. RESTful APIs enable programmatic access: Phase 2 provides comprehensive APIs for querying and interacting with legal systems.
  2. Security is paramount: OAuth 2.0, HTTPS, and fine-grained permissions protect sensitive legal data.
  3. Real-time legal status queries: APIs enable applications to check contract validity, property ownership, and legal capacity in real-time.
  4. Developer-friendly design: OpenAPI specs, SDKs, clear error messages, and comprehensive documentation ease integration.
  5. Rate limiting ensures stability: Quota systems prevent abuse while allowing fair access to all users.
