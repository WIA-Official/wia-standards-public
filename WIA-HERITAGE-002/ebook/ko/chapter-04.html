Phase 1: 데이터 형식 표준

WIA-HERITAGE-002: Archaeological Data Standards

04.1 JSON 기반 데이터 모델

WIA-HERITAGE-002 Phase 1의 핵심은 JSON(JavaScript Object Notation) 형식을 사용한 구조화된 데이터 모델입니다. JSON은 사람이 읽기 쉽고, 기계가 파싱하기 간단하며, 모든 프로그래밍 언어에서 지원되는 개방형 표준입니다. 또한 XML보다 훨씬 간결하여 저장 공간과 네트워크 대역폭을 절약합니다.

표준은 JSON Schema를 사용하여 데이터 구조를 정의합니다. JSON Schema는 데이터의 타입, 필수 필드, 값의 범위, 형식 등을 명확하게 규정하여 데이터 품질을 보장합니다. 모든 WIA-HERITAGE-002 데이터는 공식 스키마에 대해 자동으로 검증될 수 있습니다.

🎯 JSON 선택 이유
  • 보편성: 모든 현대 프로그래밍 언어와 데이터베이스가 네이티브 지원
  • 가독성: 기술자가 아닌 고고학자도 텍스트 에디터에서 직접 편집 가능
  • 경량성: XML 대비 평균 40% 적은 파일 크기
  • 확장성: 새 필드 추가가 기존 파서를 깨뜨리지 않음
  • 웹 친화적: JavaScript와 완벽 통합, REST API의 사실상 표준
  • 도구 생태계: 검증, 변환, 쿼리를 위한 풍부한 오픈소스 도구

최소 유효 문서

가장 단순한 유물 기록은 다음과 같습니다:

{
  "schema": "https://wia.org/heritage-002/artifact/v1.3.7",
  "id": "KOR-2024-GYJ-001",
  "name": "청자 상감운학문 매병",
  "site": "경주 황남동 98-5번지",
  "date": "2024-03-15",
  "images": ["https://cdn.example.org/artifacts/001/main.jpg"]
}

이 5개 필드만으로도 유효한 WIA-HERITAGE-002 문서가 됩니다. 나머지 필드는 모두 선택적이며, 프로젝트의 필요에 따라 추가할 수 있습니다.

04.2 Artifact Schema: 유물 데이터 구조

Artifact Schema는 표준의 가장 중요한 구성 요소로, 발굴된 유물의 모든 측면을 기술합니다. 스키마는 계층적 구조를 가지며, 필수 필드와 선택 필드로 구분됩니다.

필수 필드 (Required Fields)

필드명타입설명예시

분류 체계 (Classification)

유물 분류는 다층적 계층 구조를 사용합니다. 표준은 Getty Art & Architecture Thesaurus (AAT)와 호환되는 어휘를 제공하지만, 프로젝트별 분류 체계도 허용합니다:

📚 표준 분류 카테고리
  • ceramic: pottery, porcelain, celadon, earthenware, stoneware
  • metal: bronze, iron, gold, silver, copper alloy
  • stone: sculpture, tool, building material, inscription
  • organic: bone, wood, textile, leather, shell
  • glass: vessel, bead, ornament, window
  • composite: multiple materials integrated

04.4 3D Model Specification

3D 모델은 WIA-HERITAGE-002의 핵심 혁신 중 하나입니다. 표준은 glTF 2.0(Graphics Language Transmission Format)을 공식 3D 형식으로 채택했습니다.

glTF 2.0 선택 이유

  • 개방형 표준: Khronos Group이 관리하는 무료 개방형 사양
  • 웹 최적화: WebGL로 브라우저에서 직접 렌더링 가능
  • 효율성: 바이너리 압축으로 OBJ 대비 80% 작은 파일 크기
  • 완전성: 메시, 텍스처, 재질, 애니메이션, 스킨을 하나의 파일에
  • 산업 표준: Blender, Unity, Unreal Engine, Three.js 등 주요 도구 지원
  • 확장성: KHR 확장 메커니즘으로 고고학 전용 메타데이터 추가 가능

04.5 Dating Schema: 연대 측정 데이터

연대 측정은 고고학의 핵심이지만, 다양한 방법과 복잡한 통계 처리로 인해 표준화가 어렵습니다. WIA-HERITAGE-002는 모든 주요 측정 방법을 지원하는 통합 스키마를 제공합니다.

지원하는 연대 측정 방법

📅 연대 측정 기법
  • Radiocarbon (C14): 유기물 연대, 50,000년까지
  • Thermoluminescence (TL): 도자기, 석기, 최대 500,000년
  • Optically Stimulated Luminescence (OSL): 퇴적물, 최대 200,000년
  • Dendrochronology: 나무 나이테, 연단위 정확도
  • Potassium-Argon (K-Ar): 화산암, 수백만 년
  • Uranium-Thorium: 탄산염, 500,000년까지
  • Archaeomagnetism: 가마, 화덕 등
  • Obsidian Hydration: 흑요석 도구

04.7 데이터 검증 및 품질 관리

모든 WIA-HERITAGE-002 데이터는 JSON Schema를 사용하여 자동으로 검증될 수 있습니다. 이는 데이터 품질을 보장하고 오류를 조기에 발견하는 데 필수적입니다.

검증 도구

🔍 공식 검증 도구
  • wia-validator: 공식 CLI 도구, Python 기반
  • WIA Schema Validator Web: 브라우저 기반 온라인 검증기
  • wia-check GitHub Action: CI/CD 파이프라인 통합
  • wia-vscode: Visual Studio Code 확장, 실시간 검증
  • wia-js: JavaScript 라이브러리, 클라이언트 사이드 검증

CLI 검증 예시

# wia-validator 설치
$ pip install wia-heritage-validator

# 단일 파일 검증
$ wia-validate artifact.json
✓ artifact.json: VALID (schema v1.3.7)
  - All required fields present
  - 23 optional fields included
  - 0 warnings, 0 errors

# 디렉토리 전체 검증
$ wia-validate ./artifacts/
Processing 142 files...
✓ 138 files valid
⚠ 3 files with warnings
✗ 1 file with errors

# 상세 보고서 생성
$ wia-validate --report=html ./artifacts/

Step 1: 기존 데이터 감사

현재 보유한 데이터를 평가합니다:

  • 어떤 형식으로 저장되어 있는가? (Excel, Access, 파일 시스템 등)
  • 어떤 정보가 기록되어 있는가?
  • 어떤 정보가 누락되어 있는가?
  • 데이터 품질은 어떠한가? (중복, 오류, 불일치)

Step 2: 변환 스크립트 작성

Python을 사용한 간단한 변환 예시:

import pandas as pd
import json
from datetime import datetime

# Excel 데이터 읽기
df = pd.read_excel('artifacts.xlsx')

# WIA-HERITAGE-002 형식으로 변환
artifacts = []
for idx, row in df.iterrows():
    artifact = {
        "schema": "https://wia.org/heritage-002/artifact/v1.3.7",
        "id": row['ID'],
        "name": row['Name'],
        "site": row['Site'],
        "date": row['Date'].strftime('%Y-%m-%d'),
        "images": [row['ImageURL']] if pd.notna(row['ImageURL']) else [],

        # 선택 필드
        "classification": {
            "category": row['Category'],
            "period": row['Period']
        } if pd.notna(row['Category']) else {},

        "physical": {
            "dimensions": {
                "height": row['Height'],
                "unit": "cm"
            }
        } if pd.notna(row['Height']) else {}
    }
    artifacts.append(artifact)

# JSON 파일로 저장
with open('artifacts_wia.json', 'w', encoding='utf-8') as f:
    json.dump(artifacts, f, ensure_ascii=False, indent=2)

print(f"Converted {len(artifacts)} artifacts to WIA-HERITAGE-002 format")

04.10 장 요약

✅ 핵심 내용
  1. JSON 기반 설계: 읽기 쉽고, 확장 가능하며, 모든 플랫폼에서 지원되는 개방형 형식
  2. 최소 5개 필드: schema, id, name, site, date만으로 유효한 문서 작성 가능, 진입 장벽 최소화
  3. 계층적 스키마: Artifact, Site, 3D Model, Dating 등 전문화된 스키마로 모든 고고학 데이터 표현
  4. glTF 2.0 3D: 웹 최적화된 개방형 3D 표준으로 브라우저 직접 렌더링 및 다중 LOD 지원
  5. 다양한 연대 측정: C14, TL, OSL 등 모든 주요 기법 지원, 보정 곡선 버전 명시로 재현성 보장
  6. 다국어 및 Dublin Core: 유니코드 완전 지원, 다국어 메타데이터, Dublin Core 15개 요소 호환
  7. 자동 검증: JSON Schema 기반 검증 도구로 데이터 품질 보장, CLI 및 웹 도구 제공
  8. 확장 메커니즘: extensions 객체로 표준 깨지 않고 프로젝트별 커스텀 필드 추가 가능
