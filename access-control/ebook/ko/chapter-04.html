🔐 제4장: 1단계 - 데이터 형식

4.1 데이터 모델 개요

WIA 접근 제어 데이터 형식은 다양한 시스템에서 신원, 리소스, 정책 및 접근 결정을 표현하는 표준화된 방법을 제공합니다.

4.1.1 핵심 엔티티

WIA-AC 데이터 모델:
┌─────────────────────────────────────────────────────────┐
│                      주체 (SUBJECT)                      │
│  (사용자, 서비스, 장치)                                  │
│  └── 속성, 자격 증명, 역할                               │
├─────────────────────────────────────────────────────────┤
│                      리소스 (RESOURCE)                   │
│  (데이터, 시스템, API, 물리적)                           │
│  └── 유형, 분류, 소유자, 속성                            │
├─────────────────────────────────────────────────────────┤
│                      작업 (ACTION)                       │
│  (읽기, 쓰기, 실행, 관리)                                │
│  └── 작업, 범위, 제약                                    │
├─────────────────────────────────────────────────────────┤
│                      정책 (POLICY)                       │
│  (규칙, 조건, 효과)                                      │
│  └── 대상, 조건, 효과, 의무                              │
├─────────────────────────────────────────────────────────┤
│                      결정 (DECISION)                     │
│  (허용, 거부, 불확정)                                    │
│  └── 결과, 이유, 의무                                    │
└─────────────────────────────────────────────────────────┘

4.2 JSON 스키마 정의

4.2.1 주체 스키마

{
  "$schema": "https://json-schema.org/draft/2020-12/schema",
  "$id": "https://wiastandards.com/ac/subject-v1.0.json",
  "title": "WIA-AC 주체",
  "type": "object",
  "required": ["id", "type", "attributes"],
  "properties": {
    "id": {
      "type": "string",
      "format": "uri",
      "description": "고유 주체 식별자"
    },
    "type": {
      "type": "string",
      "enum": ["user", "service", "device", "group"]
    },
    "attributes": {
      "type": "object",
      "properties": {
        "name": { "type": "string" },
        "email": { "type": "string", "format": "email" },
        "department": { "type": "string" },
        "roles": { "type": "array", "items": { "type": "string" } },
        "clearance": { "type": "string" },
        "location": { "type": "string" }
      }
    }
  }
}

4.3 핵심 데이터 유형

4.3.1 식별자 유형

유형 형식 예시
주체 ID urn:wia:ac:subject:{type}:{uuid} urn:wia:ac:subject:user:550e8400-e29b
리소스 ID urn:wia:ac:resource:{type}:{path} urn:wia:ac:resource:data:/files/report.pdf
정책 ID urn:wia:ac:policy:{namespace}:{name} urn:wia:ac:policy:hr:employee-data-access
역할 ID urn:wia:ac:role:{org}:{name} urn:wia:ac:role:acme:admin

4.3.2 속성 값 유형

유형 JSON 유형 설명
문자열 "string" 텍스트 값
정수 number (정수) 정수
불리언 true/false 이진 플래그
날짜시간 "string" (ISO 8601) 타임스탬프
IP 주소 "string" IPv4 또는 IPv6
배열 [] 값 목록

4.4 정책 정의 언어

4.4.1 정책 구조

{
  "$schema": "https://wiastandards.com/ac/policy-v1.0.json",
  "id": "urn:wia:ac:policy:finance:budget-access",
  "version": "1.0.0",
  "description": "재무팀 예산 접근 정책",
  "target": {
    "subjects": {
      "match": "any",
      "conditions": [
        { "attribute": "department", "operator": "equals", "value": "finance" }
      ]
    },
    "resources": {
      "match": "any",
      "conditions": [
        { "attribute": "type", "operator": "equals", "value": "budget-data" }
      ]
    },
    "actions": ["read", "write"]
  },
  "rules": [
    {
      "id": "rule-1",
      "effect": "permit",
      "condition": {
        "and": [
          { "attribute": "subject.clearance", "operator": "gte", "value": 3 },
          { "attribute": "environment.time", "operator": "between", "value": ["09:00", "18:00"] }
        ]
      }
    }
  ],
  "obligations": [
    { "type": "log", "parameters": { "level": "audit" } }
  ]
}

4.4.2 조건 연산자

연산자 설명 예시
equals 정확한 일치 "role" equals "admin"
notEquals 같지 않음 "status" notEquals "suspended"
contains 배열에 값 포함 "groups" contains "managers"
startsWith 문자열 접두사 "path" startsWith "/api/"
regex 정규 표현식 "email" regex ".*@company.com"
gt, gte, lt, lte 숫자 비교 "level" gte 5
between 범위 검사 "time" between ["09:00", "17:00"]
in 목록에 값 포함 "country" in ["US", "CA", "UK"]

4.7 레거시 형식에서 마이그레이션

4.7.1 LDAP에서 WIA-AC로 매핑

LDAP 속성 WIA-AC 속성
uid subject.id
cn subject.attributes.name
mail subject.attributes.email
memberOf subject.attributes.roles[]
ou subject.attributes.department

4.7.2 마이그레이션 도구

# WIA-AC CLI 마이그레이션 명령

# LDAP에서 내보내기
wia-ac migrate export --source ldap \
  --host ldap.company.com \
  --base "dc=company,dc=com" \
  --output subjects.json

# WIA-AC 형식으로 변환
wia-ac migrate transform \
  --input subjects.json \
  --mapping ldap-to-wia.yaml \
  --output wia-subjects.json

# WIA-AC로 가져오기
wia-ac migrate import \
  --input wia-subjects.json \
  --target https://ac.company.com

[OK] 핵심 내용:

  1. 데이터 모델: 5가지 핵심 엔티티 - 주체, 리소스, 작업, 정책, 결정
  2. JSON 스키마: 모든 데이터 유형에 대해 잘 정의된 스키마

  5. 요청/응답: 접근 결정을 위한 표준화된 형식
  6. 감사 로그: 규정 준수를 위한 포괄적인 이벤트 로깅
  7. 마이그레이션: 레거시 시스템 마이그레이션을 위한 도구 및 매핑
