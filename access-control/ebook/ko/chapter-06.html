🔐 제6장: 3단계 - 프로토콜

6.1 프로토콜 아키텍처

WIA 접근 제어 프로토콜 계층은 세션 관리, 토큰 교환 및 안전한 시스템 간 통신을 위한 실시간 통신 표준을 정의합니다.

6.1.1 프로토콜 스택

WIA-AC 프로토콜 스택:
┌─────────────────────────────────────────────────────────┐
│                 애플리케이션 프로토콜                    │
│  (WIA-AC 메시지, 정책 업데이트, 이벤트)                 │
├─────────────────────────────────────────────────────────┤
│                 세션 프로토콜                            │
│  (세션 관리, 토큰 교환)                                  │
├─────────────────────────────────────────────────────────┤
│                 전송 프로토콜                            │
│  (WebSocket, gRPC, HTTP/2)                              │
├─────────────────────────────────────────────────────────┤
│                 보안 프로토콜                            │
│  (TLS 1.3, mTLS, DTLS)                                  │
└─────────────────────────────────────────────────────────┘

6.1.2 지원되는 전송

전송 사용 사례 포트
HTTPS (REST) 요청-응답, 동기 443
WebSocket 실시간, 양방향 443 (/ws)
gRPC 고성능, 스트리밍 8443
MQTT IoT 장치, 펍/섭 8883

6.2 세션 관리

6.2.1 세션 수명 주기

세션 상태:
┌──────────────┐
│   생성됨     │
└──────┬───────┘
       │ 인증
       ▼
┌──────────────┐
│   활성      │◄────────┐
└──────┬───────┘         │
       │                 │ 갱신
       │ 타임아웃/로그아웃│
       ▼                 │
┌──────────────┐         │
│   만료됨     │─────────┘
└──────┬───────┘
       │ 종료
       ▼
┌──────────────┐
│   종료됨     │
└──────────────┘

6.2.2 세션 토큰 구조

{
  "header": {
    "alg": "RS256",
    "typ": "JWT",
    "kid": "key-2025-01"
  },
  "payload": {
    "iss": "https://ac.company.com",
    "sub": "urn:wia:ac:subject:user:john.doe",
    "aud": ["app-12345", "api-gateway"],
    "exp": 1705312800,
    "iat": 1705309200,
    "jti": "session-550e8400-e29b",
    "sid": "ses-123456789",
    "scope": "read write",
    "wia_roles": ["developer", "team-lead"],
    "wia_groups": ["engineering", "project-x"]
  }
}

6.2.3 세션 작업

작업 설명 프로토콜
생성 새 세션 초기화 POST /sessions
검증 세션 유효성 확인 GET /sessions/{id}
갱신 세션 수명 연장 POST /sessions/{id}/refresh
취소 세션 종료 DELETE /sessions/{id}
목록 사용자의 활성 세션 조회 GET /subjects/{id}/sessions

6.3 토큰 교환 프로토콜

6.3.1 토큰 교환 흐름

토큰 교환 (RFC 8693):

┌──────────┐                        ┌───────────────┐
│  클라이언트│                        │  토큰 서버    │
└────┬─────┘                        └───────┬───────┘
     │                                      │
     │  POST /token                         │
     │  grant_type=token_exchange           │
     │  subject_token={jwt}                 │
     │  subject_token_type=access_token     │
     │  requested_token_type=access_token   │
     │  audience=target-service             │
     │─────────────────────────────────────►│
     │                                      │
     │  {                                   │
     │    access_token: {new_jwt},          │
     │    token_type: "Bearer",             │
     │    expires_in: 3600                  │
     │  }                                   │
     │◄─────────────────────────────────────│

6.3.2 토큰 유형

토큰 유형 URN 용도
접근 토큰 urn:ietf:params:oauth:token-type:access_token API 권한 부여
갱신 토큰 urn:ietf:params:oauth:token-type:refresh_token 토큰 갱신
ID 토큰 urn:ietf:params:oauth:token-type:id_token 신원 클레임
WIA-AC 토큰 urn:wia:ac:token-type:authorization AC 결정

6.4 실시간 이벤트

6.4.1 WebSocket 연결

# 이벤트 스트림에 연결
ws://ac.company.com/wia-ac/v1/events

# 인증
{
  "type": "auth",
  "token": "eyJhbGciOiJSUzI1NiIs..."
}

# 이벤트 구독
{
  "type": "subscribe",
  "channels": [
    "policy-updates",
    "session-events",
    "access-decisions"
  ]
}

6.4.2 이벤트 유형

이벤트 채널 페이로드
policy.created policy-updates 새 정책 세부 정보
policy.updated policy-updates 변경된 정책
policy.deleted policy-updates 정책 ID
session.created session-events 새 세션 정보
session.terminated session-events 세션 ID, 이유
access.denied access-decisions 거부된 요청 세부 정보

6.6 보안 프로토콜

6.6.1 TLS 요구 사항

요구 사항 사양
최소 버전 TLS 1.2 (TLS 1.3 권장)
암호 모음 AEAD 암호 (AES-GCM, ChaCha20)
키 교환 ECDHE (P-256, P-384, X25519)
인증서 RSA 2048+ 또는 ECDSA P-256+
HSTS 필수, max-age 1년

6.6.2 상호 TLS (mTLS)

mTLS 구성:
{
  "mtls": {
    "enabled": true,
    "clientCertRequired": true,
    "trustedCAs": [
      "-----BEGIN CERTIFICATE-----\n...\n-----END CERTIFICATE-----"
    ],
    "subjectMapping": {
      "cn": "subject.id",
      "o": "subject.attributes.organization"
    }
  }
}

6.6.3 토큰 보안

  • 서명: RS256 또는 ES256 필수
  • 암호화: 민감한 토큰에 A256GCM
  • 키 순환: 최소 90일마다
  • 토큰 바인딩: DPoP 지원

6.7 프로토콜 성능

6.7.1 성능 목표

작업 목표 지연 시간 처리량
권한 부여 결정 < 10ms (P99) 100,000/초
토큰 검증 < 5ms (P99) 200,000/초
세션 조회 < 3ms (P99) 500,000/초
정책 평가 < 15ms (P99) 50,000/초

6.8 장 요약

[OK] 핵심 내용:

  1. 프로토콜 스택: 애플리케이션, 세션, 전송, 보안 계층
  2. 전송: HTTPS, WebSocket, gRPC, MQTT 지원
  3. 세션: 전체 수명 주기 관리가 있는 JWT 기반
  4. 토큰 교환: 서비스 간 RFC 8693 준수
  5. 실시간: 정책 및 세션 업데이트를 위한 WebSocket 이벤트
  6. 연합: OIDC 및 SAML2 신원 공급자 통합
  7. 보안: TLS 1.2+, mTLS, 토큰 서명 및 암호화
