🔐 제1장: 접근 제어 소개

홍익인간 (弘益人間)

"널리 인간을 이롭게 하라"

WIA 접근 제어 표준은 안전하고 공평한 접근 관리가 상호 연결된 세계에서 개인과 조직 모두를 보호하는 데 근본적이라는 철학에 기반합니다.

1.1 접근 제어란 무엇인가?

접근 제어(Access Control)는 누가 또는 무엇이 리소스, 데이터, 시스템 또는 물리적 공간에 접근할 수 있는지를 결정하는 기본적인 보안 개념입니다. 이는 진입 및 사용 권한을 규제하는 데 사용되는 정책, 절차 및 기술을 포함합니다.

[i] 정의: 접근 제어는 리소스에 대한 접근을 선택적으로 제한하는 것입니다. 컴퓨팅 환경이나 물리적 위치에서 누가 또는 무엇이 리소스를 보거나 사용할 수 있는지를 규제하는 보안 기술입니다.

1.1.1 핵심 원칙

접근 제어는 AAA 프레임워크로 알려진 세 가지 기본 보안 개념에 기반합니다:

인증 (Authentication) 사용자, 장치 또는 시스템의 신원 확인
권한 부여 (Authorization) 인증된 주체가 수행할 수 있는 작업 결정
책임 추적 (Accountability) 감사 및 규정 준수 목적으로 작업 추적 및 기록

1.1.2 접근 제어가 중요한 이유

오늘날의 디지털 환경에서 접근 제어는 다음을 위해 중요합니다:

  • 무단 접근으로부터 민감한 데이터 보호
  • 규제 준수 보장 (GDPR, HIPAA, SOX)
  • 내부 위협 및 외부 공격 방지
  • 비즈니스 연속성 및 운영 무결성 유지
  • 고객 및 이해관계자와의 신뢰 구축

1.2 시장 규모 및 성장

접근 제어 시장은 디지털 전환 및 보안 우려로 인해 상당한 성장을 경험했습니다:

연도 시장 규모 (USD) 성장률

2021 94억 달러 9.3%
2022 103억 달러 9.6%

1.2.1 시장 세그먼트

접근 제어 시장은 여러 영역에 걸쳐 있습니다:

  • 물리적 접근 제어 (35%) - 건물 보안, 카드 리더, 생체인식
  • 논리적 접근 제어 (30%) - IT 시스템, 애플리케이션, 네트워크
  • 클라우드 접근 보안 (20%) - 클라우드 인프라, SaaS 애플리케이션
  • 모바일 접근 (10%) - 스마트폰 기반 자격 증명, BYOD
  • IoT 접근 (5%) - 연결된 장치, 스마트 시스템

1.3 접근 제어 모델

1.3.1 임의적 접근 제어 (DAC)

DAC는 리소스 소유자가 누가 리소스에 접근할 수 있는지를 결정할 수 있게 합니다. 소유자가 접근 권한에 대한 완전한 제어권을 갖습니다.

측면 설명
제어 리소스 소유자가 권한 결정
유연성 높음 - 소유자가 자유롭게 접근 부여/취소
예시 Unix 파일 권한, Windows NTFS
적합 소규모 조직, 파일 공유
약점 트로이 목마 공격에 취약

1.3.2 강제적 접근 제어 (MAC)

MAC는 보안 레이블 및 인가 수준에 따라 접근을 강제합니다. 사용자가 아닌 시스템이 접근 결정을 내립니다.

측면 설명
제어 중앙 기관 (시스템 정책)
유연성 낮음 - 엄격한 계층 구조
예시 SELinux, 군사 분류
적합 정부, 군사, 고보안 환경
강점 데이터 유출에 대한 강력한 보호

1.3.3 역할 기반 접근 제어 (RBAC)

RBAC는 개인이 아닌 역할에 권한을 할당합니다. 사용자는 직무 기능에 따라 역할이 할당됩니다.

RBAC 구조:
┌─────────────────────────────────────┐
│              사용자                  │
│   (John, Jane, Bob, Alice...)       │
└──────────────┬──────────────────────┘
               │ 할당됨
               ▼
┌─────────────────────────────────────┐
│              역할                    │
│   (관리자, 매니저, 직원...)          │
└──────────────┬──────────────────────┘
               │ 보유
               ▼
┌─────────────────────────────────────┐
│              권한                    │
│   (읽기, 쓰기, 실행, 삭제)           │
└─────────────────────────────────────┘

1.3.4 속성 기반 접근 제어 (ABAC)

ABAC는 사용자, 리소스, 작업 및 환경의 속성을 기반으로 접근 결정을 내립니다.

  • 주체 속성: 사용자 역할, 부서, 인가 수준
  • 리소스 속성: 분류, 소유자, 유형
  • 작업 속성: 읽기, 쓰기, 승인, 삭제
  • 환경 속성: 시간, 위치, 장치

1.4 접근 제어 구성 요소

1.4.1 주체 (Subjects)

주체는 리소스에 대한 접근을 요청하는 엔티티입니다:

  • 사용자: 자격 증명을 가진 인간 개인
  • 프로세스: 사용자를 대신하여 실행되는 애플리케이션 및 서비스
  • 장치: IoT 장치, 기계 및 시스템
  • 서비스: 자동화된 프로세스 및 API

1.4.2 객체 (Objects)

객체는 보호되는 리소스입니다:

  • 데이터: 파일, 데이터베이스, 레코드
  • 시스템: 서버, 애플리케이션, 서비스
  • 물리적: 건물, 방, 장비
  • 네트워크: 세그먼트, 포트, 프로토콜

1.4.3 접근 권한

권한 설명 예시
읽기 정보 보기 또는 검색 파일 내용 보기
쓰기 정보 수정 또는 업데이트 문서 편집
실행 프로그램 또는 스크립트 실행 애플리케이션 시작
삭제 리소스 제거 레코드 삭제
생성 새 리소스 생성 새 사용자 추가
관리 권한 관리 접근 권한 부여

1.5 인증 방법

1.5.1 알고 있는 것 (Something You Know)

  • 비밀번호: 전통적인 텍스트 기반 자격 증명
  • PIN: 접근용 숫자 코드
  • 보안 질문: 지식 기반 검증
  • 암호 문구: 더 길고 기억하기 쉬운 문자열

1.5.2 가지고 있는 것 (Something You Have)

  • 스마트 카드: 칩이 내장된 카드
  • 보안 토큰: OTP 생성기 (RSA SecurID)
  • 모바일 장치: 스마트폰 인증기
  • 키 폽: 근접 장치

1.5.3 나 자신인 것 (Something You Are)

  • 지문: 고유한 손가락 패턴
  • 안면 인식: 얼굴 형상 분석
  • 홍채/망막 스캔: 눈 패턴 식별
  • 음성 인식: 음성 특성

1.5.4 다중 인증 (MFA)

MFA는 향상된 보안을 위해 두 가지 이상의 인증 요소를 결합합니다:

MFA 예시:
┌──────────────────┐
│ 요소 1: 알고 있음 │  비밀번호: ********
└────────┬─────────┘
         │
         ▼
┌──────────────────┐
│ 요소 2: 가지고 있음│  OTP: 123456 (휴대폰에서)
└────────┬─────────┘
         │
         ▼
┌──────────────────┐
│ 요소 3: 나 자신임  │  지문 확인됨 ✓
└────────┬─────────┘
         │
         ▼
    접근 허가됨

1.6 다양한 도메인에서의 접근 제어

1.6.1 물리적 접근 제어

  • 카드 리더: RFID, 마그네틱 스트라이프, 스마트 카드
  • 생체인식 스캐너: 지문, 얼굴, 홍채
  • 턴스타일/게이트: 제어된 진입점
  • 비디오 감시: 모니터링 및 검증

1.6.2 네트워크 접근 제어 (NAC)

  • 802.1X: 포트 기반 네트워크 접근 제어
  • VPN: 안전한 원격 접근
  • 방화벽: 트래픽 필터링 및 제어
  • 네트워크 분할: 민감한 영역 격리

1.6.3 애플리케이션 접근 제어

  • Single Sign-On (SSO): 통합 인증
  • OAuth/OIDC: 위임된 권한 부여
  • SAML: 연합 및 신원 관리
  • API 보안: 토큰 기반 접근

1.6.4 클라우드 접근 제어

  • IAM: 신원 및 접근 관리
  • CASB: 클라우드 접근 보안 브로커
  • 제로 트러스트: 신뢰하지 않고 항상 검증
  • 최소 권한: 최소한의 필요한 접근

1.7 WIA 철학: 홍익인간

弘益人間 (홍익인간)

"널리 인간을 이롭게 하라"

이 고대 한국 철학이 WIA 접근 제어 표준을 안내합니다. 우리는 다음을 믿습니다:

  • 보안은 합법적인 접근에 장벽을 만들지 않고 보호해야 함
  • 표준은 모든 규모의 조직에 개방적이고 접근 가능해야 함
  • 프라이버시와 보안은 인류를 위해 함께 작동해야 함
  • 포용적인 접근 제어는 다양한 요구와 상황을 존중함
