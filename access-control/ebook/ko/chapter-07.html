7.1.2 패턴 비교

패턴 장점 단점 적합
중앙 집중식 PDP 단순, 일관된 정책 단일 장애 지점 소규모 배포

API 게이트웨이 통합 진입점 게이트웨이 의존성 API 우선 아키텍처
사이드카 언어 무관 리소스 오버헤드 Kubernetes/메시

7.2.2 커넥터 인터페이스

// WIA-AC 커넥터 인터페이스 (TypeScript)
interface WiaACConnector {
  // 수명 주기
  initialize(config: ConnectorConfig): Promise<void>;
  healthCheck(): Promise<HealthStatus>;
  shutdown(): Promise<void>;

  // 신원 동기화
  syncSubjects(options: SyncOptions): Promise<SyncResult>;
  syncGroups(options: SyncOptions): Promise<SyncResult>;

  // 인증
  authenticate(credentials: Credentials): Promise<AuthResult>;
  validateToken(token: string): Promise<TokenInfo>;

  // 프로비저닝
  createSubject(subject: Subject): Promise<Subject>;
  updateSubject(id: string, changes: Partial<Subject>): Promise<Subject>;
  deleteSubject(id: string): Promise<void>;

  // 권한 부여
  getPermissions(subjectId: string): Promise<Permission[]>;
  setPermissions(subjectId: string, permissions: Permission[]): Promise<void>;
}

7.4 클라우드 플랫폼 통합

7.4.1 AWS 통합

# WIA-AC용 AWS IAM 정책
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": [
        "sts:AssumeRoleWithWebIdentity"
      ],
      "Principal": {
        "Federated": "arn:aws:iam::123456789:oidc-provider/ac.company.com"
      },
      "Condition": {
        "StringEquals": {
          "ac.company.com:aud": "aws-integration"
        }
      }
    }
  ]
}

# WIA-AC에서 AWS 역할 매핑
wia_role: "engineer"      → AWS Role: arn:aws:iam::123456789:role/Engineer
wia_role: "admin"         → AWS Role: arn:aws:iam::123456789:role/Admin
wia_role: "data-analyst"  → AWS Role: arn:aws:iam::123456789:role/DataAnalyst

7.4.2 Kubernetes 통합

# WIA-AC Kubernetes 어드미션 컨트롤러
apiVersion: admissionregistration.k8s.io/v1
kind: ValidatingWebhookConfiguration
metadata:
  name: wia-ac-admission
webhooks:
  - name: authorize.wia-ac.io
    clientConfig:
      service:
        name: wia-ac-webhook
        namespace: wia-ac
        path: /authorize
    rules:
      - operations: ["CREATE", "UPDATE", "DELETE"]
        apiGroups: ["*"]
        apiVersions: ["*"]
        resources: ["pods", "deployments", "services"]

7.7 장 요약

[OK] 핵심 내용:

  1. 통합 패턴: 중앙 집중식, 분산, 게이트웨이, 사이드카 옵션
