Chapter 3: Authentication Mechanisms

Authentication is the critical first step in the access control process, establishing the identity of subjects before any authorization decisions can be made. This chapter explores the authentication mechanisms supported by the WIA-ACCESS_CONTROL standard, from traditional passwords to cutting-edge biometric and passwordless solutions.

Understanding Authentication

Authentication answers the fundamental security question: "Who are you?" It is the process of verifying that an entity is who or what it claims to be. Strong authentication is essential because authorization decisions are only as trustworthy as the identity verification that precedes them.

Authentication Factors:
  • Something You Know: Passwords, PINs, security questions
  • Something You Have: Smart cards, hardware tokens, mobile devices
  • Something You Are: Fingerprints, facial recognition, voice patterns
  • Somewhere You Are: Geographic location, IP address, network
  • Something You Do: Behavioral patterns, typing dynamics, gait

Password-Based Authentication

Despite their well-known weaknesses, passwords remain the most common authentication mechanism. The WIA-ACCESS_CONTROL standard provides guidelines for implementing password authentication securely.

Secure Password Storage

Passwords must never be stored in plain text. The WIA-ACCESS_CONTROL standard mandates the use of modern password hashing algorithms with appropriate work factors.

import { Argon2id } from '@wia/crypto';

class SecurePasswordStorage {
    private readonly hashConfig = {
        algorithm: 'argon2id',
        memoryCost: 65536,    // 64 MB
        timeCost: 3,          // 3 iterations
        parallelism: 4,       // 4 parallel threads
        hashLength: 32        // 256-bit hash
    };

    async hashPassword(plaintext: string): Promise<string> {
        // Generate cryptographically secure salt
        const salt = crypto.randomBytes(16);

        // Hash with Argon2id
        const hash = await Argon2id.hash(plaintext, {
            salt: salt,
            ...this.hashConfig
        });

        // Return encoded hash string
        return this.encodeHash(hash, salt, this.hashConfig);
    }

    async verifyPassword(plaintext: string, storedHash: string): Promise<boolean> {
        const { hash, salt, config } = this.decodeHash(storedHash);

        const computedHash = await Argon2id.hash(plaintext, {
            salt: salt,
            ...config
        });

        // Constant-time comparison to prevent timing attacks
        return crypto.timingSafeEqual(hash, computedHash);
    }

    needsRehash(storedHash: string): boolean {
        const { config } = this.decodeHash(storedHash);
        return config.memoryCost < this.hashConfig.memoryCost ||
               config.timeCost < this.hashConfig.timeCost;
    }
}

Multi-Factor Authentication (MFA)

Multi-factor authentication requires users to provide two or more verification factors. This dramatically reduces the risk of unauthorized access even if one factor is compromised.

MFA Effectiveness: According to industry research, MFA blocks over 99.9% of automated attacks. The WIA-ACCESS_CONTROL standard strongly recommends MFA for all privileged access and recommends it for all user accounts.

Time-Based One-Time Passwords (TOTP)

import { TOTP } from '@wia/auth';

class TOTPAuthenticator {
    private readonly config = {
        algorithm: 'SHA256',
        digits: 6,
        period: 30,           // 30-second window
        window: 1             // Allow 1 period drift
    };

    generateSecret(): string {
        // Generate 160-bit secret
        const secret = crypto.randomBytes(20);
        return this.base32Encode(secret);
    }

    generateQRCode(secret: string, user: string, issuer: string): string {
        const uri = `otpauth://totp/${encodeURIComponent(issuer)}:${encodeURIComponent(user)}` +
                   `?secret=${secret}&issuer=${encodeURIComponent(issuer)}` +
                   `&algorithm=${this.config.algorithm}&digits=${this.config.digits}` +
                   `&period=${this.config.period}`;
        return uri;
    }

    verifyToken(secret: string, token: string): VerificationResult {
        const currentTime = Math.floor(Date.now() / 1000);

        for (let offset = -this.config.window; offset <= this.config.window; offset++) {
            const timeStep = Math.floor((currentTime + offset * this.config.period) / this.config.period);
            const expectedToken = this.generateTOTP(secret, timeStep);

            if (crypto.timingSafeEqual(
                Buffer.from(token),
                Buffer.from(expectedToken)
            )) {
                return { valid: true, drift: offset };
            }
        }

        return { valid: false, drift: null };
    }

    private generateTOTP(secret: string, timeStep: number): string {
        const hmac = crypto.createHmac(this.config.algorithm, this.base32Decode(secret));
        hmac.update(this.intToBytes(timeStep));
        const hash = hmac.digest();

        const offset = hash[hash.length - 1] & 0x0f;
        const binary = ((hash[offset] & 0x7f) << 24) |
                      ((hash[offset + 1] & 0xff) << 16) |
                      ((hash[offset + 2] & 0xff) << 8) |
                      (hash[offset + 3] & 0xff);

        const otp = binary % Math.pow(10, this.config.digits);
        return otp.toString().padStart(this.config.digits, '0');
    }
}

Hardware Security Keys (FIDO2/WebAuthn)

Hardware security keys provide the strongest form of authentication. They use public-key cryptography and are resistant to phishing attacks because they verify the origin of authentication requests.

import { WebAuthn } from '@wia/auth';

class FIDO2Authenticator {
    private webAuthn: WebAuthn;

    constructor() {
        this.webAuthn = new WebAuthn({
            rpName: 'WIA Enterprise',
            rpId: 'auth.wia.org',
            origin: 'https://auth.wia.org',
            attestation: 'direct',
            authenticatorSelection: {
                authenticatorAttachment: 'cross-platform',
                userVerification: 'required',
                residentKey: 'preferred'
            }
        });
    }

    async startRegistration(user: User): Promise<RegistrationOptions> {
        const challenge = crypto.randomBytes(32);

        const options = await this.webAuthn.generateRegistrationOptions({
            user: {
                id: user.id,
                name: user.email,
                displayName: user.displayName
            },
            challenge: challenge,
            excludeCredentials: await this.getUserCredentials(user.id),
            timeout: 60000
        });

        // Store challenge for verification
        await this.storeChallenge(user.id, challenge, 'registration');

        return options;
    }

    async completeRegistration(
        userId: string,
        response: RegistrationResponse
    ): Promise<Credential> {
        const challenge = await this.getStoredChallenge(userId, 'registration');

        const verification = await this.webAuthn.verifyRegistrationResponse({
            response: response,
            expectedChallenge: challenge,
            expectedOrigin: this.webAuthn.origin,
            expectedRPID: this.webAuthn.rpId
        });

        if (!verification.verified) {
            throw new AuthenticationError('Registration verification failed');
        }

        // Store credential
        const credential = {
            id: verification.credentialId,
            publicKey: verification.credentialPublicKey,
            counter: verification.counter,
            transports: response.transports,
            createdAt: new Date()
        };

        await this.storeCredential(userId, credential);
        return credential;
    }

    async authenticate(userId: string, response: AuthenticationResponse): Promise<boolean> {
        const credential = await this.getCredential(userId, response.credentialId);
        const challenge = await this.getStoredChallenge(userId, 'authentication');

        const verification = await this.webAuthn.verifyAuthenticationResponse({
            response: response,
            expectedChallenge: challenge,
            expectedOrigin: this.webAuthn.origin,
            expectedRPID: this.webAuthn.rpId,
            credential: credential
        });

        if (verification.verified) {
            // Update counter to prevent replay attacks
            await this.updateCredentialCounter(credential.id, verification.newCounter);
            return true;
        }

        return false;
    }
}

Biometric Authentication

Biometric authentication uses unique physical or behavioral characteristics to verify identity. While convenient, biometrics require careful implementation to address privacy concerns and false acceptance/rejection rates.

Biometric Type FAR FRR Use Cases
Fingerprint 0.001% 0.1% Device unlock, physical access
Facial Recognition 0.01% 0.5% Mobile authentication, surveillance
Iris Scan 0.0001% 0.2% High-security facilities
Voice Recognition 0.1% 2% Phone banking, virtual assistants
Behavioral 1% 5% Continuous authentication
Privacy Consideration: Biometric data cannot be changed if compromised. WIA-ACCESS_CONTROL requires that biometric templates be stored securely, preferably on-device, and that raw biometric data never be transmitted or stored on servers.

Token-Based Authentication

JSON Web Tokens (JWT)

JWTs are widely used for stateless authentication in modern applications. They contain encoded claims about the authenticated subject and are cryptographically signed.

import { JWT } from '@wia/auth';

class JWTService {
    private readonly config = {
        algorithm: 'RS256',
        issuer: 'wia-access-control',
        audience: 'wia-services',
        accessTokenTTL: 900,      // 15 minutes
        refreshTokenTTL: 604800   // 7 days
    };

    private privateKey: crypto.KeyObject;
    private publicKey: crypto.KeyObject;

    async generateTokenPair(user: AuthenticatedUser): Promise<TokenPair> {
        const now = Math.floor(Date.now() / 1000);
        const jti = crypto.randomUUID();

        const accessToken = await JWT.sign({
            // Header
            alg: this.config.algorithm,
            typ: 'JWT',
            kid: this.currentKeyId
        }, {
            // Payload
            iss: this.config.issuer,
            sub: user.id,
            aud: this.config.audience,
            iat: now,
            exp: now + this.config.accessTokenTTL,
            jti: jti,
            // Custom claims
            roles: user.roles,
            permissions: user.permissions,
            sessionId: user.sessionId
        }, this.privateKey);

        const refreshToken = await JWT.sign({
            alg: this.config.algorithm,
            typ: 'JWT'
        }, {
            iss: this.config.issuer,
            sub: user.id,
            iat: now,
            exp: now + this.config.refreshTokenTTL,
            jti: crypto.randomUUID(),
            type: 'refresh',
            accessTokenId: jti
        }, this.privateKey);

        // Store refresh token hash for revocation capability
        await this.storeRefreshToken(user.id, refreshToken);

        return { accessToken, refreshToken };
    }

    async verifyAccessToken(token: string): Promise<TokenPayload> {
        const payload = await JWT.verify(token, this.publicKey, {
            algorithms: [this.config.algorithm],
            issuer: this.config.issuer,
            audience: this.config.audience
        });

        // Check if token has been revoked
        if (await this.isTokenRevoked(payload.jti)) {
            throw new AuthenticationError('Token has been revoked');
        }

        return payload;
    }

    async refreshAccessToken(refreshToken: string): Promise<TokenPair> {
        const payload = await JWT.verify(refreshToken, this.publicKey);

        if (payload.type !== 'refresh') {
            throw new AuthenticationError('Invalid token type');
        }

        // Verify refresh token is still valid
        if (!await this.validateRefreshToken(payload.sub, refreshToken)) {
            throw new AuthenticationError('Refresh token invalid or revoked');
        }

        // Revoke old access token
        await this.revokeToken(payload.accessTokenId);

        // Generate new token pair
        const user = await this.getUserById(payload.sub);
        return this.generateTokenPair(user);
    }
}

Passwordless Authentication

Passwordless authentication eliminates passwords entirely, using alternative factors such as magic links, biometrics, or hardware keys. This approach improves both security and user experience.

class PasswordlessAuth {
    async sendMagicLink(email: string): Promise<void> {
        const user = await this.userService.findByEmail(email);
        if (!user) {
            // Return success even if user not found (prevent enumeration)
            return;
        }

        // Generate secure token
        const token = crypto.randomBytes(32).toString('base64url');
        const expiresAt = Date.now() + 15 * 60 * 1000; // 15 minutes

        // Store token hash
        await this.tokenStore.save({
            tokenHash: crypto.createHash('sha256').update(token).digest('hex'),
            userId: user.id,
            expiresAt: expiresAt,
            type: 'magic-link'
        });

        // Send email
        await this.emailService.send({
            to: email,
            subject: 'Sign in to WIA',
            template: 'magic-link',
            data: {
                link: `https://auth.wia.org/verify?token=${token}`,
                expiresIn: '15 minutes'
            }
        });
    }

    async verifyMagicLink(token: string): Promise<AuthResult> {
        const tokenHash = crypto.createHash('sha256').update(token).digest('hex');

        const storedToken = await this.tokenStore.findByHash(tokenHash);
        if (!storedToken || storedToken.expiresAt < Date.now()) {
            throw new AuthenticationError('Invalid or expired token');
        }

        // Delete token (single use)
        await this.tokenStore.delete(storedToken.id);

        // Authenticate user
        const user = await this.userService.findById(storedToken.userId);
        return this.createSession(user);
    }
}

Adaptive Authentication

Adaptive authentication adjusts authentication requirements based on risk assessment. Low-risk scenarios may require minimal authentication, while high-risk scenarios trigger additional verification steps.

Risk Signals: WIA-ACCESS_CONTROL evaluates multiple signals including device trust, network location, user behavior patterns, time of access, and resource sensitivity to calculate risk scores and determine appropriate authentication levels. 
class AdaptiveAuthEngine {
    async calculateRiskScore(context: AuthContext): Promise<RiskAssessment> {
        const signals = await Promise.all([
            this.evaluateDevice(context.device),
            this.evaluateNetwork(context.network),
            this.evaluateGeolocation(context.location),
            this.evaluateBehavior(context.user, context.request),
            this.evaluateTime(context.timestamp),
            this.evaluateResourceSensitivity(context.resource)
        ]);

        const riskScore = this.aggregateSignals(signals);

        return {
            score: riskScore,
            level: this.getRiskLevel(riskScore),
            factors: signals,
            recommendedAuthLevel: this.getRequiredAuthLevel(riskScore),
            additionalFactors: this.getAdditionalFactors(riskScore, context)
        };
    }

    private getRiskLevel(score: number): RiskLevel {
        if (score < 0.3) return 'low';
        if (score < 0.6) return 'medium';
        if (score < 0.8) return 'high';
        return 'critical';
    }

    private getRequiredAuthLevel(score: number): AuthLevel {
        if (score < 0.3) return { factors: 1, methods: ['password', 'biometric'] };
        if (score < 0.6) return { factors: 2, methods: ['password+totp', 'biometric+pin'] };
        if (score < 0.8) return { factors: 2, methods: ['password+fido2', 'biometric+fido2'] };
        return { factors: 3, methods: ['password+fido2+biometric'], stepUp: true };
    }
}

Summary

This chapter has covered the authentication mechanisms supported by the WIA-ACCESS_CONTROL standard. From traditional passwords with modern security controls, through multi-factor authentication and biometrics, to cutting-edge passwordless and adaptive approaches, the standard provides a comprehensive framework for verifying user identities.

Key Takeaways:
  • Strong authentication is the foundation of effective access control
  • Multi-factor authentication dramatically reduces unauthorized access risk
  • Hardware security keys provide the strongest protection against phishing
  • Biometrics require careful implementation to protect privacy
  • Adaptive authentication balances security with user experience
  • Passwordless authentication represents the future of identity verification
