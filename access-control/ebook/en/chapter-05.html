Chapter 5: Security Protocols and Encryption

Security protocols and encryption form the cryptographic foundation of access control systems. They protect credentials, secure communications, and ensure the integrity of authorization decisions. This chapter explores the security mechanisms that underpin the WIA-ACCESS_CONTROL standard, from transport security to comprehensive audit trails.

Transport Layer Security

All access control communications must be protected in transit. The WIA-ACCESS_CONTROL standard mandates the use of TLS 1.3 or higher for all network communications between access control components.

Mutual TLS (mTLS)

For service-to-service communication within the access control infrastructure, mutual TLS provides bidirectional authentication. Both client and server present certificates, ensuring that only authorized services can communicate.

// mTLS Service Configuration
class MTLSService {
    private server: tls.Server;
    private clientCert: tls.Certificate;

    async initialize(config: MTLSConfig): Promise<void> {
        this.server = tls.createServer({
            key: await this.loadPrivateKey(config.keyPath),
            cert: await this.loadCertificate(config.certPath),
            ca: await this.loadCA(config.caPath),

            // Require client certificate
            requestCert: true,
            rejectUnauthorized: true,

            // TLS 1.3 only
            minVersion: 'TLSv1.3',
            maxVersion: 'TLSv1.3',

            // Session configuration
            sessionTimeout: 300,
            sessionIdContext: 'wia-access-control'
        });

        this.server.on('secureConnection', (socket) => {
            this.validateClientCertificate(socket);
        });
    }

    private validateClientCertificate(socket: tls.TLSSocket): void {
        const cert = socket.getPeerCertificate(true);

        // Verify certificate chain
        if (!cert || !cert.subject) {
            socket.destroy(new Error('No client certificate'));
            return;
        }

        // Check certificate against allowlist
        const fingerprint = cert.fingerprint256;
        if (!this.isAuthorizedService(fingerprint)) {
            socket.destroy(new Error('Unauthorized service'));
            return;
        }

        // Log successful connection
        this.auditLog.log({
            event: 'mtls_connection',
            service: cert.subject.CN,
            fingerprint: fingerprint,
            timestamp: new Date()
        });
    }
}

Key Management

Effective key management is essential for maintaining the security of cryptographic operations. The WIA-ACCESS_CONTROL standard defines a comprehensive key management lifecycle.

// Key Management Service
class KeyManagementService {
    private keyStore: SecureKeyStore;
    private hsm: HardwareSecurityModule;

    // Key lifecycle states
    private readonly keyStates = {
        PENDING_ACTIVATION: 'pending_activation',
        ACTIVE: 'active',
        SUSPENDED: 'suspended',
        DEACTIVATED: 'deactivated',
        COMPROMISED: 'compromised',
        DESTROYED: 'destroyed'
    };

    async generateKey(params: KeyGenerationParams): Promise<KeyMetadata> {
        // Generate key in HSM
        const keyId = await this.hsm.generateKey({
            algorithm: params.algorithm,
            keySize: params.keySize,
            extractable: false,
            usages: params.usages
        });

        // Store key metadata
        const metadata: KeyMetadata = {
            id: keyId,
            algorithm: params.algorithm,
            keySize: params.keySize,
            state: this.keyStates.PENDING_ACTIVATION,
            createdAt: new Date(),
            activationDate: params.activationDate || new Date(),
            expirationDate: this.calculateExpiration(params),
            usages: params.usages,
            owner: params.owner
        };

        await this.keyStore.saveMetadata(metadata);
        await this.auditKeyOperation('generate', metadata);

        return metadata;
    }

    async rotateKey(keyId: string): Promise<KeyRotationResult> {
        const currentKey = await this.keyStore.getMetadata(keyId);

        if (currentKey.state !== this.keyStates.ACTIVE) {
            throw new Error('Can only rotate active keys');
        }

        // Generate new key with same parameters
        const newKey = await this.generateKey({
            algorithm: currentKey.algorithm,
            keySize: currentKey.keySize,
            usages: currentKey.usages,
            owner: currentKey.owner
        });

        // Activate new key
        await this.activateKey(newKey.id);

        // Mark old key for deactivation
        await this.scheduleDeactivation(currentKey.id, {
            reason: 'rotation',
            successor: newKey.id,
            gracePreriod: '7d'
        });

        await this.auditKeyOperation('rotate', {
            oldKeyId: currentKey.id,
            newKeyId: newKey.id
        });

        return {
            previousKeyId: currentKey.id,
            newKeyId: newKey.id,
            rotatedAt: new Date()
        };
    }

    async destroyKey(keyId: string, reason: string): Promise<void> {
        const key = await this.keyStore.getMetadata(keyId);

        // Verify key can be destroyed
        if (key.state === this.keyStates.DESTROYED) {
            throw new Error('Key already destroyed');
        }

        // Destroy key material in HSM
        await this.hsm.destroyKey(keyId);

        // Update metadata
        key.state = this.keyStates.DESTROYED;
        key.destroyedAt = new Date();
        key.destroyReason = reason;

        await this.keyStore.saveMetadata(key);
        await this.auditKeyOperation('destroy', key);
    }
}

Token Security

Access tokens and credentials must be protected throughout their lifecycle. The WIA-ACCESS_CONTROL standard specifies security requirements for token generation, storage, and validation.

Secure Token Generation

// Secure Token Generator
class SecureTokenGenerator {
    private readonly entropySource: crypto.RandomGenerator;

    generateAccessToken(claims: TokenClaims): string {
        const header = {
            alg: 'EdDSA',
            typ: 'JWT',
            kid: this.currentKeyId
        };

        const payload = {
            ...claims,
            iat: Math.floor(Date.now() / 1000),
            exp: Math.floor(Date.now() / 1000) + claims.ttl,
            jti: this.generateTokenId(),
            iss: 'wia-access-control',
            aud: claims.audience
        };

        // Sign with Ed25519
        const signature = this.sign(
            this.encodeBase64Url(header) + '.' + this.encodeBase64Url(payload)
        );

        return `${this.encodeBase64Url(header)}.${this.encodeBase64Url(payload)}.${signature}`;
    }

    generateOpaqueToken(length: number = 32): string {
        // Generate cryptographically secure random bytes
        const bytes = crypto.randomBytes(length);

        // Encode as URL-safe base64
        return bytes.toString('base64url');
    }

    private generateTokenId(): string {
        // UUID v7 for time-ordered, unique identifiers
        return this.generateUUIDv7();
    }

    private generateUUIDv7(): string {
        const timestamp = BigInt(Date.now());
        const random = crypto.randomBytes(10);

        // Construct UUID v7
        const bytes = Buffer.alloc(16);

        // Timestamp (48 bits)
        bytes.writeBigUInt64BE(timestamp << 16n, 0);

        // Version (4 bits) = 7
        bytes[6] = (bytes[6] & 0x0f) | 0x70;

        // Random (12 bits)
        bytes[6] = (bytes[6] & 0xf0) | (random[0] & 0x0f);
        bytes[7] = random[1];

        // Variant (2 bits) = 10
        bytes[8] = (random[2] & 0x3f) | 0x80;

        // Random (62 bits)
        random.copy(bytes, 9, 3, 10);

        return this.formatUUID(bytes);
    }
}

Audit Trail Management

Comprehensive audit trails are essential for security monitoring, incident investigation, and compliance. The WIA-ACCESS_CONTROL standard mandates detailed logging of all access control events.

Audit Requirements: All authentication attempts, authorization decisions, policy changes, and administrative actions must be logged with sufficient detail for forensic analysis. Audit logs must be tamper-evident and retained according to compliance requirements.

Tamper-Evident Logging

// Tamper-Evident Audit Log
class TamperEvidentAuditLog {
    private currentHash: string;
    private logStore: AuditLogStore;
    private signingKey: crypto.KeyObject;

    async appendEvent(event: Omit<AuditEvent, 'previousEventHash' | 'eventHash'>): Promise<string> {
        // Get previous hash
        const previousHash = this.currentHash;

        // Create complete event
        const completeEvent: AuditEvent = {
            ...event,
            previousEventHash: previousHash,
            eventHash: '' // Will be calculated
        };

        // Calculate event hash
        const eventData = this.canonicalize(completeEvent);
        const eventHash = crypto
            .createHash('sha256')
            .update(eventData)
            .digest('hex');

        completeEvent.eventHash = eventHash;

        // Sign the event
        const signature = crypto.sign(
            'sha256',
            Buffer.from(eventHash),
            this.signingKey
        );

        // Store event with signature
        await this.logStore.append({
            event: completeEvent,
            signature: signature.toString('base64'),
            timestamp: new Date()
        });

        // Update current hash
        this.currentHash = eventHash;

        return eventHash;
    }

    async verifyIntegrity(startEventId?: string): Promise<IntegrityReport> {
        const events = await this.logStore.getEvents(startEventId);

        let expectedPreviousHash = startEventId
            ? (await this.logStore.getEvent(startEventId)).eventHash
            : '0'.repeat(64); // Genesis hash

        const issues: IntegrityIssue[] = [];

        for (const entry of events) {
            // Verify chain continuity
            if (entry.event.previousEventHash !== expectedPreviousHash) {
                issues.push({
                    eventId: entry.event.eventId,
                    type: 'chain_break',
                    expected: expectedPreviousHash,
                    actual: entry.event.previousEventHash
                });
            }

            // Verify event hash
            const calculatedHash = this.calculateEventHash(entry.event);
            if (calculatedHash !== entry.event.eventHash) {
                issues.push({
                    eventId: entry.event.eventId,
                    type: 'hash_mismatch',
                    expected: entry.event.eventHash,
                    actual: calculatedHash
                });
            }

            // Verify signature
            const signatureValid = crypto.verify(
                'sha256',
                Buffer.from(entry.event.eventHash),
                this.signingKey,
                Buffer.from(entry.signature, 'base64')
            );

            if (!signatureValid) {
                issues.push({
                    eventId: entry.event.eventId,
                    type: 'invalid_signature'
                });
            }

            expectedPreviousHash = entry.event.eventHash;
        }

        return {
            verified: issues.length === 0,
            eventsChecked: events.length,
            issues: issues
        };
    }
}

Secure Communication Patterns

Access control components must communicate securely. The WIA-ACCESS_CONTROL standard defines patterns for secure inter-component communication.

// Secure Message Exchange
class SecureMessageChannel {
    private encryptionKey: crypto.KeyObject;
    private signingKey: crypto.KeyObject;

    async sendMessage(recipient: string, payload: any): Promise<void> {
        // Generate ephemeral key for this message
        const ephemeralKey = crypto.generateKeyPairSync('x25519');

        // Derive shared secret
        const recipientPublicKey = await this.getRecipientPublicKey(recipient);
        const sharedSecret = crypto.diffieHellman({
            privateKey: ephemeralKey.privateKey,
            publicKey: recipientPublicKey
        });

        // Derive encryption key using HKDF
        const encKey = crypto.hkdfSync(
            'sha256',
            sharedSecret,
            Buffer.from('wia-access-control'),
            Buffer.from('encryption'),
            32
        );

        // Encrypt payload
        const iv = crypto.randomBytes(12);
        const cipher = crypto.createCipheriv('aes-256-gcm', encKey, iv);

        const plaintext = JSON.stringify(payload);
        const ciphertext = Buffer.concat([
            cipher.update(plaintext, 'utf8'),
            cipher.final()
        ]);
        const authTag = cipher.getAuthTag();

        // Create message envelope
        const envelope = {
            version: 1,
            sender: this.identity,
            recipient: recipient,
            ephemeralPublicKey: ephemeralKey.publicKey.export({ type: 'spki', format: 'der' }),
            iv: iv.toString('base64'),
            ciphertext: ciphertext.toString('base64'),
            authTag: authTag.toString('base64'),
            timestamp: new Date().toISOString()
        };

        // Sign envelope
        const signature = crypto.sign(
            'sha256',
            Buffer.from(JSON.stringify(envelope)),
            this.signingKey
        );

        envelope.signature = signature.toString('base64');

        await this.transport.send(recipient, envelope);
    }
}

Summary

This chapter has covered the security protocols and encryption mechanisms that protect the WIA-ACCESS_CONTROL infrastructure. From transport layer security with TLS 1.3 to comprehensive key management and tamper-evident audit trails, these mechanisms ensure the confidentiality, integrity, and accountability of access control operations.

Key Takeaways:
  • TLS 1.3 with strong cipher suites protects all communications
  • Mutual TLS provides bidirectional service authentication
  • Modern cryptographic algorithms ensure data protection
  • Comprehensive key management covers the entire key lifecycle
  • Tamper-evident audit trails enable forensic analysis
  • Secure communication patterns protect inter-component messaging
