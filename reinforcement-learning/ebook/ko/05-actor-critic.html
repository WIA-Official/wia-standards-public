5.1 Actor-Critic 소개

Actor-Critic 방법은 정책 경사법과 가치 기반 접근법의 장점을 결합합니다. "Actor"(행위자)는 정책을 업데이트하고 "Critic"(비평가)은 가치 함수를 추정하여 행동을 평가합니다.

핵심 개념:

Actor: 정책 π_θ(a|s) - 어떤 행동을 취할지 결정

Critic: 가치 함수 V_w(s) 또는 Q_w(s,a) - 행동이 얼마나 좋은지 평가

Actor-Critic의 장점

  • 낮은 분산: Critic이 Monte Carlo보다 낮은 분산 추정치 제공
  • 온라인 학습: 불완전한 에피소드에서도 학습 가능
  • 연속 행동: 연속 행동 공간에서 작동
  • 샘플 효율성: 부트스트래핑으로 학습 가속화

5.2 기본 Actor-Critic 알고리즘

알고리즘: 1단계 Actor-Critic

actor π_θ와 critic V_w 초기화

각 에피소드에 대해:
    s = env.reset()
    
    while not done:
        a ~ π_θ(·|s)  // Actor가 행동 선택
        s', r, done = env.step(a)
        
        δ = r + γ V_w(s') - V_w(s)  // TD 오차 (Critic)
        
        w ← w + α_w δ ∇_w V_w(s)       // Critic 업데이트
        θ ← θ + α_θ δ ∇_θ log π_θ(a|s) // Actor 업데이트
        
        s ← s'

구현

import torch
import torch.nn as nn

class Actor(nn.Module):
    def __init__(self, state_dim, action_dim, hidden_dim=128):
        super(Actor, self).__init__()
        self.network = nn.Sequential(
            nn.Linear(state_dim, hidden_dim),
            nn.ReLU(),
            nn.Linear(hidden_dim, action_dim),
            nn.Softmax(dim=-1)
        )
    
    def forward(self, state):
        return self.network(state)

class Critic(nn.Module):
    def __init__(self, state_dim, hidden_dim=128):
        super(Critic, self).__init__()
        self.network = nn.Sequential(
            nn.Linear(state_dim, hidden_dim),
            nn.ReLU(),
            nn.Linear(hidden_dim, 1)
        )
    
    def forward(self, state):
        return self.network(state)

5.3 Advantage Actor-Critic (A2C)

A2C는 Q값 대신 어드밴티지 함수를 사용합니다:

A(s,a) = Q(s,a) - V(s)
       = r + γV(s') - V(s)  // TD(0) 사용

어드밴티지는 상태 s에서 평균 행동과 비교하여 
행동 a가 얼마나 더 좋은지를 알려줍니다.

어드밴티지를 사용하는 이유

  • 기준선 V(s)를 빼서 분산 감소
  • 값을 0 주위로 중심화 (양수 = 좋음, 음수 = 나쁨)
  • 정책 업데이트에 더 안정적인 그래디언트
  • Q(s,a)가 아닌 V(s)만 학습하면 됨

5.4 A3C (Asynchronous Advantage Actor-Critic)

A3C는 여러 CPU 스레드에서 학습을 병렬화합니다. 각 스레드가 자체 환경을 실행합니다:

핵심 혁신:

각 워커가 독립적으로 공유 글로벌 네트워크를 비동기적으로 업데이트합니다. 이는 경험 재생 없이도 다양한 경험을 제공하고 학습을 안정화합니다.

5.5 Soft Actor-Critic (SAC)

SAC는 보상과 엔트로피를 모두 최대화하는 off-policy actor-critic 알고리즘입니다:

J(π) = E[Σ γ^t (r_t + α H(π(·|s_t)))]

여기서:
- H(π) = -E[log π(a|s)]는 엔트로피
- α는 온도 파라미터
- 높은 엔트로피 → 더 많은 탐색

SAC 주요 특징

  • 최대 엔트로피: 보상을 최대화하면서 탐색 장려
  • Off-Policy: 리플레이 버퍼로 샘플 효율적
  • 확률적 정책: 자연스러운 탐색
  • 안정적: 많은 튜닝 없이 다양한 태스크에서 잘 작동

5.6 TD3 (Twin Delayed DDPG)

TD3는 세 가지 주요 수정으로 DDPG를 개선합니다:

  1. 트윈 Critic: 두 개의 Q 네트워크, 최솟값 사용
  2. 지연 정책 업데이트: Critic보다 덜 자주 정책 업데이트
  3. 타겟 정책 스무딩: 타겟 행동에 노이즈 추가

5.7 Actor-Critic 방법 비교

알고리즘 On/Off-Policy 행동 공간 핵심 특징
A2C On-policy 이산/연속 동기식 병렬 워커
A3C On-policy 이산/연속 비동기식 워커
DDPG Off-policy 연속 결정적 정책 + 리플레이
TD3 Off-policy 연속 트윈 critic + 지연 업데이트
SAC Off-policy 연속 최대 엔트로피 + 자동 튜닝

5.8 실용적 팁

알고리즘 선택

  • 이산 행동: A2C, PPO
  • 연속 행동: SAC, TD3, DDPG
  • 샘플 효율성: SAC, TD3 (리플레이 있는 off-policy)
  • 안정성: SAC, PPO
  • 단순성: A2C, 기본 Actor-Critic

요약

  • Actor-Critic은 정책 경사(actor)와 가치 함수(critic)를 결합
  • 어드밴티지는 순수 정책 경사에 비해 분산 감소
  • A2C는 동기식 병렬 워커를 사용한 데이터 수집
  • A3C는 경험 재생 없이 비동기 업데이트
  • SAC는 탐색을 위한 엔트로피 최대화, 높은 샘플 효율성
  • TD3는 트윈 critic과 지연 업데이트로 DDPG 개선
  • Off-policy 방법(SAC, TD3)이 더 샘플 효율적
  • 선택은 행동 공간, 샘플 효율성, 안정성 요구에 따라 결정
