2.1 MDP 소개

마르코프 결정 과정(Markov Decision Process, MDP)은 강화학습의 수학적 기초를 제공합니다. MDP는 결과가 부분적으로는 무작위이고 부분적으로는 의사결정자의 통제 하에 있는 확률적 환경에서 순차적 의사결정을 형식화합니다.

정의:

MDP는 다음의 튜플 (S, A, P, R, γ)로 정의됩니다:

  • S: 상태 집합
  • A: 행동 집합
  • P: 전이 확률 함수 P(s'|s,a)
  • R: 보상 함수 R(s,a,s')
  • γ: 할인 계수 [0,1]

마르코프 속성

MDP의 핵심 가정은 마르코프 속성입니다: 현재가 주어졌을 때 미래는 과거와 독립적입니다.

P(sₜ₊₁ | sₜ, aₜ, sₜ₋₁, aₜ₋₁, ..., s₀, a₀) = P(sₜ₊₁ | sₜ, aₜ)

"미래는 현재에만 의존하고, 과거에는 의존하지 않는다"

이것은 현재 상태가 과거의 모든 관련 정보를 포함한다는 것을 의미합니다. 에이전트는 과거 상태와 행동의 전체 시퀀스를 기억할 필요가 없습니다.

2.2 상태 가치 함수

상태 가치 함수 V^π(s)는 상태 s에서 시작하여 이후 정책 π를 따를 때의 기대 수익입니다:

V^π(s) = E_π[Gₜ | Sₜ = s]
       = E_π[Rₜ₊₁ + γRₜ₊₂ + γ²Rₜ₊₃ + ... | Sₜ = s]
       = E_π[Σ γᵏRₜ₊ᵏ₊₁ | Sₜ = s]

V^π에 대한 벨만 방정식

가치 함수는 재귀적 관계를 만족합니다:

V^π(s) = Σ_a π(a|s) Σ_s' P(s'|s,a) [R(s,a,s') + γV^π(s')]

설명:
V(s) = 즉각적 기대 보상 + 다음 상태의 할인된 가치
핵심 통찰:

벨만 방정식은 상태의 가치와 그 후속 상태들의 가치 사이의 관계를 표현합니다. 이 재귀적 구조는 MDP를 해결하는 데 기본이 됩니다.

2.3 행동 가치 함수

행동 가치 함수(Q-함수)는 상태 s에서 행동 a를 취하고 이후 정책 π를 따를 때의 기대 수익을 제공합니다:

Q^π(s,a) = E_π[Gₜ | Sₜ = s, Aₜ = a]
         = E_π[Rₜ₊₁ + γV^π(Sₜ₊₁) | Sₜ = s, Aₜ = a]

V와 Q의 관계

// V에서 Q로
Q^π(s,a) = Σ_s' P(s'|s,a) [R(s,a,s') + γV^π(s')]

// Q에서 V로
V^π(s) = Σ_a π(a|s) Q^π(s,a)

// 결정적 정책의 경우
V^π(s) = Q^π(s, π(s))

2.4 최적 가치 함수

최적 상태 가치 함수 V*(s)는 상태 s에서 달성 가능한 최대 가치입니다:

V*(s) = max_π V^π(s)

Q*(s,a) = max_π Q^π(s,a)

벨만 최적 방정식

V*(s) = max_a Σ_s' P(s'|s,a) [R(s,a,s') + γV*(s')]

Q*(s,a) = Σ_s' P(s'|s,a) [R(s,a,s') + γ max_a' Q*(s',a')]
중요:

벨만 최적 방정식은 max 연산자 때문에 비선형입니다. 이것은 고정된 정책에 대해 V^π를 평가하는 것보다 V*를 찾는 것을 더 어렵게 만듭니다.

2.5 최적 정책

최적 정책 π*는 모든 상태에서 최대 기대 수익을 달성합니다:

π*(s) = argmax_a Q*(s,a)
      = argmax_a Σ_s' P(s'|s,a) [R(s,a,s') + γV*(s')]

최적 정책의 속성

  • 항상 적어도 하나의 최적 정책이 존재합니다 (고유하지 않을 수 있음)
  • 모든 최적 정책은 동일한 최적 가치 함수 V*를 공유합니다
  • 최적 정책은 결정적입니다 (확률적 정책도 존재할 수 있지만)
  • Q*를 알면 최적 정책을 추출하는 것은 간단합니다

2.6 정책 평가

정책 평가는 주어진 정책 π에 대해 V^π를 계산합니다. 벨만 방정식을 반복적으로 풀 수 있습니다:

def policy_evaluation(policy, env, gamma=0.99, theta=1e-6):
    """
    MDP에서 정책을 평가합니다.

    Args:
        policy: π(a|s) - 평가할 정책
        env: MDP 환경
        gamma: 할인 계수
        theta: 수렴 임계값
    """
    V = {s: 0 for s in env.states}

    while True:
        delta = 0
        for s in env.states:
            v = V[s]

            # 벨만 업데이트
            V[s] = sum(
                policy[s][a] * sum(
                    env.P[s][a][s_next] * (
                        env.R[s][a][s_next] + gamma * V[s_next]
                    )
                    for s_next in env.states
                )
                for a in env.actions
            )

            delta = max(delta, abs(v - V[s]))

        if delta < theta:
            break

    return V

행렬 형태

결정적 정책의 경우, 벨만 방정식을 행렬 형태로 작성할 수 있습니다:

V^π = R^π + γP^π V^π

직접 풀기:
V^π = (I - γP^π)^(-1) R^π

여기서:
- V^π는 모든 상태에 대한 가치 벡터
- R^π는 즉각적 기대 보상 벡터
- P^π는 정책 π 하의 전이 행렬

2.7 정책 개선

V^π가 주어지면, 탐욕적으로 행동하여 더 나은 정책을 구성할 수 있습니다:

def policy_improvement(V, env, gamma=0.99):
    """
    가치 함수를 기반으로 정책을 개선합니다.
    """
    policy = {}

    for s in env.states:
        # 모든 행동에 대한 Q-값 계산
        q_values = {}
        for a in env.actions:
            q_values[a] = sum(
                env.P[s][a][s_next] * (
                    env.R[s][a][s_next] + gamma * V[s_next]
                )
                for s_next in env.states
            )

        # 탐욕적 행동 선택
        best_action = max(q_values, key=q_values.get)
        policy[s] = best_action

    return policy
정책 개선 정리:

π와 π'가 모든 s에 대해 Q^π(s, π'(s)) ≥ V^π(s)를 만족하는 임의의 결정적 정책 쌍이라면, π'는 π만큼 좋거나 더 좋아야 합니다: 모든 s에 대해 V^π'(s) ≥ V^π(s).

2.8 정책 반복

정책 반복은 수렴할 때까지 정책 평가와 정책 개선을 번갈아 수행합니다:

def policy_iteration(env, gamma=0.99):
    """
    정책 반복을 사용하여 최적 정책을 찾습니다.

    1. 정책을 임의로 초기화
    2. 수렴할 때까지 반복:
        - 정책 평가: V^π 계산
        - 정책 개선: π' ← greedy(V^π)
    """
    # 무작위 정책 초기화
    policy = {s: random.choice(env.actions) for s in env.states}

    while True:
        # 정책 평가
        V = policy_evaluation(policy, env, gamma)

        # 정책 개선
        new_policy = policy_improvement(V, env, gamma)

        # 수렴 확인
        if new_policy == policy:
            break

        policy = new_policy

    return policy, V

수렴 보장

정책 반복은 유한한 반복 횟수(|A|^|S|로 제한됨)에서 최적 정책으로 수렴하는 것이 보장됩니다.

2.9 가치 반복

가치 반복은 정책 평가와 개선을 단일 업데이트로 결합합니다:

def value_iteration(env, gamma=0.99, theta=1e-6):
    """
    가치 반복을 사용하여 최적 가치 함수를 찾습니다.

    V(s) ← max_a Σ_s' P(s'|s,a)[R + γV(s')]
    """
    V = {s: 0 for s in env.states}

    while True:
        delta = 0
        for s in env.states:
            v = V[s]

            # 벨만 최적 업데이트
            V[s] = max(
                sum(
                    env.P[s][a][s_next] * (
                        env.R[s][a][s_next] + gamma * V[s_next]
                    )
                    for s_next in env.states
                )
                for a in env.actions
            )

            delta = max(delta, abs(v - V[s]))

        if delta < theta:
            break

    # 최적 정책 추출
    policy = policy_improvement(V, env, gamma)

    return policy, V

정책 반복 vs 가치 반복

측면 정책 반복 가치 반복
업데이트 규칙 두 개의 별도 단계 결합된 업데이트
반복 횟수 더 적은 외부 반복 더 많은 반복
반복당 비용 더 높음 (전체 평가) 더 낮음 (단일 스윕)
수렴 유한 반복 점근적

2.10 실전 예제: 그리드 월드

class GridWorld:
    def __init__(self, size=5):
        self.size = size
        self.states = [(i,j) for i in range(size) for j in range(size)]
        self.actions = ['up', 'down', 'left', 'right']
        self.goal = (size-1, size-1)

    def transition(self, state, action):
        """현재 상태와 행동이 주어졌을 때 다음 상태를 반환합니다."""
        i, j = state

        if action == 'up':
            next_state = (max(0, i-1), j)
        elif action == 'down':
            next_state = (min(self.size-1, i+1), j)
        elif action == 'left':
            next_state = (i, max(0, j-1))
        elif action == 'right':
            next_state = (i, min(self.size-1, j+1))

        return next_state

    def reward(self, state, action, next_state):
        """전이에 대한 보상을 반환합니다."""
        if next_state == self.goal:
            return 100  # 목표 보상
        else:
            return -1   # 단계 페널티

# GridWorld 풀기
env = GridWorld(size=5)
optimal_policy, V_optimal = value_iteration(env, gamma=0.9)

print("최적 가치 함수:")
for i in range(env.size):
    for j in range(env.size):
        print(f"{V_optimal[(i,j)]:6.2f}", end=" ")
    print()

print("\n최적 정책:")
for i in range(env.size):
    for j in range(env.size):
        action = optimal_policy[(i,j)]
        symbol = {'up':'↑', 'down':'↓', 'left':'←', 'right':'→'}[action]
        print(symbol, end=" ")
    print()

요약

  • MDP는 순차적 의사결정을 위한 수학적 프레임워크를 제공합니다
  • 마르코프 속성은 미래가 현재에만 의존한다고 명시합니다
  • 가치 함수 V(s)와 Q(s,a)는 상태와 상태-행동 쌍의 품질을 측정합니다
  • 벨만 방정식은 상태의 가치를 후속 상태의 가치와 연결합니다
  • 최적 가치 함수 V*와 Q*는 벨만 최적 방정식을 만족합니다
  • 정책 평가는 주어진 정책에 대해 V^π를 계산합니다
  • 정책 개선은 가치 함수로부터 더 나은 정책을 구성합니다
  • 정책 반복은 수렴할 때까지 평가와 개선을 번갈아 수행합니다
  • 가치 반복은 효율적인 계산을 위해 이러한 단계를 결합합니다
  • 두 알고리즘 모두 유한 MDP에 대한 최적 정책을 찾는 것이 보장됩니다

복습 문제

1. 마르코프 속성이란 무엇이며 왜 중요한가요?

답변: 마르코프 속성은 현재 상태가 주어졌을 때 미래가 과거와 독립적이라고 명시합니다. 이것은 전체 과거 이력 없이 현재 상태만을 기반으로 의사결정을 내릴 수 있게 해주기 때문에 중요합니다.

2. V(s)와 Q(s,a)의 차이는 무엇인가요?

답변: V(s)는 상태 s에 있는 것의 가치인 반면, Q(s,a)는 상태 s에서 행동 a를 취하는 것의 가치입니다. Q-값은 전이 모델 없이도 행동 선택을 가능하게 합니다.

3. 벨만 최적 방정식이 비선형인 이유는 무엇인가요?

답변: 비선형인 max 연산자를 포함하기 때문입니다. 이것은 최적 가치를 찾는 것을 정책 평가보다 더 어렵게 만듭니다.

4. 정책 반복과 가치 반복의 주요 차이점은 무엇인가요?

답변: 정책 반복은 별도의 평가와 개선 단계를 가지며 더 적은 반복으로 수렴하지만 반복당 비용이 더 높습니다. 가치 반복은 이러한 단계를 결합하여 더 많은 반복이 필요하지만 각 반복이 더 저렴합니다.
