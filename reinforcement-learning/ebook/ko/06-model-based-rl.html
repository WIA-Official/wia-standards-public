6.1 모델 기반 RL 소개

모델 기반 RL은 환경 역학의 모델을 학습하고 이를 계획에 사용합니다. 이는 명시적 모델 없이 경험에서 직접 가치나 정책을 학습하는 모델 프리 방법과 대조됩니다.

정의:

모델은 현재 상태와 행동이 주어졌을 때 환경이 다음에 무엇을 할지 예측합니다:

  • 전이 모델: P(s'|s,a) - 다음 상태 예측
  • 보상 모델: R(s,a) - 보상 예측

모델을 사용하는 이유

  • 샘플 효율성: 시뮬레이션 경험에서 학습 (실제 환경 상호작용 불필요)
  • 계획: 시행착오 없이 미래 결과에 대해 추론 가능
  • 전이: 모델이 새로운 태스크로 일반화 가능
  • 안전성: 배포 전 시뮬레이션에서 정책 테스트
  • 데이터 생성: 합성 훈련 데이터 생성

6.2 Dyna 아키텍처

Dyna는 계획, 행동, 모델 학습을 통합하여 모델 프리와 모델 기반 학습을 결합합니다:

알고리즘: Dyna-Q

Q(s,a)와 Model(s,a) 초기화

반복:
    // (a) 행동: 환경과 상호작용
    s = 현재 상태
    a = ε-greedy(Q, s)
    a 실행, r, s' 관찰

    // (b) 모델 프리 학습: 직접 RL
    Q(s,a) ← Q(s,a) + α[r + γ max Q(s',a') - Q(s,a)]

    // (c) 모델 학습: 전이/보상 모델 업데이트
    Model(s,a) ← (s', r)

    // (d) 계획: 시뮬레이션 경험에서 학습
    n번 반복:
        s_sim = 이전에 방문한 임의 상태
        a_sim = 이전에 취한 임의 행동
        s'_sim, r_sim = Model(s_sim, a_sim)
        Q(s_sim, a_sim) ← Q + α[r_sim + γ max Q(s'_sim, ·) - Q]

Dyna의 이점

  • 실제 경험 수집과 동시에 모델에서 계획
  • n개의 시뮬레이션 업데이트가 학습 가속화
  • 실제 상호작용과 시뮬레이션 사이의 균형 조정 가능

6.3 학습된 모델

6.3.1 결정론적 모델

import torch.nn as nn

class DeterministicModel(nn.Module):
    def __init__(self, state_dim, action_dim, hidden_dim=256):
        super().__init__()
        self.network = nn.Sequential(
            nn.Linear(state_dim + action_dim, hidden_dim),
            nn.ReLU(),
            nn.Linear(hidden_dim, hidden_dim),
            nn.ReLU(),
            nn.Linear(hidden_dim, state_dim + 1)  # 다음 상태 + 보상
        )
    
    def forward(self, state, action):
        x = torch.cat([state, action], dim=-1)
        output = self.network(x)
        next_state = output[..., :-1]
        reward = output[..., -1]
        return next_state, reward

6.3.2 확률론적 모델

확률론적 모델은 불확실성을 캡처하기 위해 분포를 출력합니다:

class ProbabilisticModel(nn.Module):
    def __init__(self, state_dim, action_dim, hidden_dim=256):
        super().__init__()
        self.shared = nn.Sequential(
            nn.Linear(state_dim + action_dim, hidden_dim),
            nn.ReLU(),
            nn.Linear(hidden_dim, hidden_dim),
            nn.ReLU()
        )
        self.mean_head = nn.Linear(hidden_dim, state_dim)
        self.logvar_head = nn.Linear(hidden_dim, state_dim)
    
    def forward(self, state, action):
        x = torch.cat([state, action], dim=-1)
        h = self.shared(x)
        mean = self.mean_head(h)
        logvar = self.logvar_head(h)
        std = torch.exp(0.5 * logvar)
        return mean, std

6.4 Model Predictive Control (MPC)

MPC는 각 시간 단계에서 학습된 모델로 최적화하여 행동을 선택합니다:

class MPC:
    def __init__(self, model, horizon=10, num_samples=1000):
        self.model = model
        self.horizon = horizon
        self.num_samples = num_samples
    
    def select_action(self, state):
        best_action = None
        best_return = -float('inf')
        
        for _ in range(self.num_samples):
            # 임의 행동 시퀀스 샘플
            actions = sample_random_actions(self.horizon)
            
            # 모델에서 롤아웃
            total_reward = 0
            s = state.clone()
            
            for a in actions:
                s, r = self.model(s, a)
                total_reward += r
            
            if total_reward > best_return:
                best_return = total_reward
                best_action = actions[0]
        
        return best_action

6.5 MBPO (Model-Based Policy Optimization)

MBPO는 실제 데이터로 모델을 훈련하고, 모델 생성 데이터로 정책을 훈련합니다:

  1. 실제 환경에서 데이터 수집
  2. 데이터로 앙상블 모델 훈련
  3. 모델을 사용하여 단기 롤아웃 생성
  4. SAC를 사용하여 모델 생성 데이터로 정책 훈련
  5. 반복

핵심 통찰

  • 단기 롤아웃: 모델 오류가 누적되기 전 짧은 롤아웃 사용
  • 앙상블: 여러 모델로 불확실성 추정
  • 실제 데이터 믹싱: 모델 편향 방지를 위해 실제 데이터도 사용

6.6 World Models

World Models는 생성 모델을 사용하여 환경의 컴팩트한 표현을 학습합니다:

World Models 구성요소:
1. Vision (V): 관찰을 잠재 코드로 압축 (VAE)
2. Memory (M): 잠재 역학 예측 (RNN/MDN)
3. Controller (C): 잠재 공간에서 행동 (작은 네트워크)

학습 프로세스:
1. 임의 정책으로 경험 수집
2. V (VAE) 훈련: 관찰 재구성
3. M (RNN) 훈련: 잠재 전이 예측
4. C 훈련: V와 M이 생성한 "꿈" 안에서 진화

6.7 모델 기반 vs 모델 프리 비교

모델 기반: 샘플 효율적, 계획 가능, 전이 가능. 하지만 모델 오류에 취약.

모델 프리: 더 많은 데이터 필요, 직접 최적화. 복잡한 환경에서도 잘 작동.

측면 모델 기반 모델 프리
샘플 효율성 높음 (10-100배) 낮음
계산량 높음 (모델 + 계획) 낮음
점근 성능 모델 오류에 제한 최적에 수렴 가능

요약

  • 모델 기반 RL은 환경 역학 모델을 학습하고 계획에 사용
  • Dyna는 실제 경험과 모델 시뮬레이션을 결합
  • MPC는 학습된 모델로 온라인 최적화 수행
  • MBPO는 단기 모델 롤아웃으로 정책 학습
  • World Models는 잠재 공간에서 계획
  • 모델 기반 방법은 샘플 효율적이지만 모델 오류에 주의 필요
  • 하이브리드 접근법이 두 패러다임의 장점을 결합
