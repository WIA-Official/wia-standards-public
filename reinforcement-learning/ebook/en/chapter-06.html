Chapter 06: Phase 3 — Federation Protocol

6.1 Phase 3 in one paragraph

Phase 3 specifies how independent RL labs and operators federate without forcing checkpoint upload to a single central registry.

6.2 Roles

Researcher, training operator, evaluation aggregator, deployment operator, safety auditor, research consortium. Trust established by federation handshake recorded in signed receipts.

6.3 Federation handshake

Reuses WIA-SOCIAL Phase 3 §5 receipt shape. Receipts persist for at least 7 years per typical research recordkeeping retention.

6.4 Replay defence

Each signed envelope carries 96-bit nonce + RFC 3339 timestamp. Receivers MUST reject envelopes with skew greater than ±300 seconds, reject envelopes whose (signer, nonce) tuple has been seen within the last 600 seconds, and maintain seen-nonce cache for at least 600 seconds.

6.5 Audience controls

AudienceVisibility
publicAggregate evaluation statistics, environment descriptors (no checkpoints)
researcherOwn training runs + checkpoints + evaluations
collaboratorPer-scope authorisation (typically per-environment-family)
safety_auditorFull safety_metrics across all evaluation runs in audit scope
deployment_operatorFrozen agent_policy + monitoring stream for active deployments
researchDe-identified runs for cross-lab benchmarking

6.6 Reproducibility envelope

For published research, a reproducibility envelope binds an environment_descriptor + checkpoint chain + evaluation_run + framework version into a verifiable bundle. Downstream researchers can re-run the bundle and verify the published numbers match within a documented epsilon.

Safety-First Design

Every choice in this standard is filtered through: does this surface enough evidence for downstream consumers to detect reward hacking and unsafe deployment, or just optimize for headline benchmark numbers? When the two conflict, safety surfacing wins. That is why the standard mandates safety_metrics on every evaluation_run, refuses to allow frozen agent_policy modification post-evaluation, and requires implementations to surface checkpoint provenance so downstream consumers can verify what model is actually deployed.

Implementer Notes

Three operational concerns recur in RL infrastructure and are baked into the standard. First, environment determinism is property of the environment + the framework + the GPU driver stack — the standard's environment_descriptor envelope captures all three so reproducibility studies can pin the exact stack. Second, checkpoint storage scales aggressively with experiment frequency; a typical research lab generates 100 GB of checkpoints per week from active hyperparameter sweeps. The standard's checkpoint envelope references model_url + sha256 rather than embedding bytes, so storage backend choices are deferred. Third, replay buffer size for offline RL — the replay_buffer_url field in the checkpoint envelope is optional but strongly recommended for offline RL workflows where buffer size can dominate disk usage.
