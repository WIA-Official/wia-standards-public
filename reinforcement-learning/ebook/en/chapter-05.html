Chapter 05: Phase 2 — API Interface

5.1 Phase 2 in one paragraph

Phase 2 specifies the HTTPS surface for RL data exchange. Transport is TLS 1.3 (IETF RFC 8446) only; HTTP/2 RECOMMENDED.

5.2 Discovery

Every host publishes a discovery document at /.well-known/wia-reinforcement-learning listing endpoints, supported frameworks, supported environment registries, and rate limits.

5.3 Environment endpoints

POST /rl/environment             (signed)
GET  /rl/environment/{environment_id}
GET  /rl/environment?framework={name}

5.4 Training run endpoints

POST /rl/training-run            (signed; researcher authority)
GET  /rl/training-run/{run_id}
GET  /rl/training-run?env={id}&algorithm={name}&since={cursor}
GET  /rl/training-run/stream?run_id={id}    (Server-Sent Events for live training metrics)

5.5 Evaluation run endpoints

POST /rl/evaluation-run          (signed)
GET  /rl/evaluation-run/{run_id}
GET  /rl/evaluation-run?env={id}&policy={id}

5.6 Checkpoint endpoints

POST /rl/checkpoint              (signed; large model_url payload by reference, not inline)
GET  /rl/checkpoint/{checkpoint_id}
GET  /rl/checkpoint?run_id={id}

5.7 Agent policy + deployment endpoints

Standard POST/GET/PATCH on /rl/agent-policy/{id} and /rl/deployment/{id}. Deployment endpoints additionally support stream subscription for post-deployment monitoring.

5.8 Authentication

Writes require HTTP Message Signatures per IETF RFC 9421. Researcher authority verified via WIA-OMNI-API researcher_credential claims; deployment operator authority verified via deployment_authorization claims under explicit organisation consent.

Safety-First Design

Every choice in this standard is filtered through: does this surface enough evidence for downstream consumers to detect reward hacking and unsafe deployment, or just optimize for headline benchmark numbers? When the two conflict, safety surfacing wins. That is why the standard mandates safety_metrics on every evaluation_run, refuses to allow frozen agent_policy modification post-evaluation, and requires implementations to surface checkpoint provenance so downstream consumers can verify what model is actually deployed.

Implementer Notes

Three operational concerns recur in RL infrastructure and are baked into the standard. First, environment determinism is property of the environment + the framework + the GPU driver stack — the standard's environment_descriptor envelope captures all three so reproducibility studies can pin the exact stack. Second, checkpoint storage scales aggressively with experiment frequency; a typical research lab generates 100 GB of checkpoints per week from active hyperparameter sweeps. The standard's checkpoint envelope references model_url + sha256 rather than embedding bytes, so storage backend choices are deferred. Third, replay buffer size for offline RL — the replay_buffer_url field in the checkpoint envelope is optional but strongly recommended for offline RL workflows where buffer size can dominate disk usage.
