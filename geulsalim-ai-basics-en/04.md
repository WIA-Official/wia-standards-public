## Theory

### Understand the Ecosystem Through a Map

When you start AI development for the first time, library names become tangled and confusing. `torch`, `transformers`, `diffusers`, `accelerate`, `peft`... Without knowing how these differ from each other or why you need to install all of them, you copy-paste code and when errors occur, you don't know where to look.

You need a map. Once you see the overall structure first, individual code naturally makes sense.

### Hierarchical Structure

```
┌─────────────────────────────────────────┐
│         My training code (train.py)      │  ← You write this
├─────────────────────────────────────────┤
│  peft     │  accelerate  │  wandb       │  ← Fine-tuning helpers
├───────────┴──────────────┴──────────────┤
│  transformers  │  diffusers  │ datasets │  ← Hugging Face core
├─────────────────────────────────────────┤
│              PyTorch (torch)             │  ← Base engine
├─────────────────────────────────────────┤
│         CUDA (GPU driver)                │  ← Hardware
└─────────────────────────────────────────┘
```

**PyTorch (`torch`)**: The foundation of GPU computation. Tensor operations, automatic differentiation, and neural network layers all come from here. Think of it as the "engine."

**Transformers (`transformers`)**: A library for loading and training language models like BERT, GPT, and LLaMA. Just write the model name and it downloads automatically, like `AutoModelForCausalLM.from_pretrained("meta-llama/Llama-3.1-8B")`.

**Diffusers (`diffusers`)**: Exclusively for image generation models like Stable Diffusion, SDXL, and Flux. Works with a pipeline concept like `StableDiffusionXLPipeline.from_pretrained(...)`.

**Datasets (`datasets`)**: Load thousands of datasets from the Hugging Face Hub with one line. Handles caching, streaming, and preprocessing all in one place.

**PEFT (`peft`)**: "Parameter-Efficient Fine-Tuning." An implementation of fine-tuning techniques like LoRA and QLoRA. Without this library, fine-tuning within 24GB of VRAM is impossible.

**Accelerate (`accelerate`)**: Handles conversion from single GPU → multi-GPU → TPU without code modifications. Just run `accelerate launch train.py` and it scales.

### Hugging Face Hub

`huggingface.co` is the GitHub of AI models. As of 2024, over 700,000 models are uploaded. You can search, download, and upload your own models.

There are three things you must check in the Model Card:
1. **License**: Apache 2.0 allows commercial use. CC BY-NC is non-commercial only.
2. **Base Model**: What model was this fine-tuned from?
3. **Training Data**: What data was it trained on? Check for copyright issues.

## Practical Code

```python
# Core usage patterns from the Hugging Face ecosystem
# Master all the basics with this single file

from transformers import AutoTokenizer, AutoModelForCausalLM
from diffusers import StableDiffusionXLPipeline
from datasets import load_dataset
from peft import LoraConfig, get_peft_model
import torch

# ── 1. Loading a language model ────────────────────────
def load_language_model(model_name: str = "meta-llama/Llama-3.1-8B"):
    """
    Load a language model from Hugging Face Hub.
    On first run, it automatically downloads (cache location: ~/.cache/huggingface/)
    """
    print(f"Loading model: {model_name}")
    
    # Tokenizer: converts text → numbers (tokens)
    tokenizer = AutoTokenizer.from_pretrained(model_name)
    
    # Model: load in FP16 to save VRAM
    model = AutoModelForCausalLM.from_pretrained(
        model_name,
        torch_dtype=torch.float16,  # Half VRAM compared to FP32
        device_map="auto"           # Automatically distribute across GPUs
    )
    
    print(f"Number of parameters: {model.num_parameters() / 1e9:.1f}B")
    return tokenizer, model


# ── 2. Image generation pipeline ───────────────────────
def load_sdxl():
    """Load SDXL pipeline. First run downloads ~7GB."""
    pipe = StableDiffusionXLPipeline.from_pretrained(
        "stabilityai/stable-diffusion-xl-base-1.0",
        torch_dtype=torch.float16,
        variant="fp16",
        use_safetensors=True
    ).to("cuda")
    
    # Memory optimization with xformers (if installed)
    try:
        pipe.enable_xformers_memory_efficient_attention()
        print("xformers memory optimization enabled")
    except Exception:
        pass
    
    return pipe


# ── 3. Load datasets ───────────────────────────────────
def explore_dataset(dataset_name: str = "laion/laion2B-en"):
    """
    Explore Hugging Face Hub datasets.
    Streaming mode: read only what you need without downloading everything.
    """
    # Stream and view only first 5 samples
    dataset = load_dataset(dataset_name, split="train", streaming=True)
    
    print(f"Dataset: {dataset_name}")
    for i, sample in enumerate(dataset):
        print(f"\nSample {i+1}:")
        for key, value in sample.items():
            v_str = str(value)[:100] + "..." if len(str(value)) > 100 else str(value)
            print(f"  {key}: {v_str}")
        if i >= 4:
            break


# ── 4. LoRA configuration (preparing fine-tuning) ──────
def apply_lora(model, rank: int = 4):
    """
    Add LoRA adapter to the model.
    Smaller rank = less VRAM, faster training (slightly lower quality)
    """
    config = LoraConfig(
        r=rank,                      # LoRA rank (4, 8, 16, 32, 64)
        lora_alpha=rank * 2,         # Scaling coefficient (usually 2x rank)
        target_modules=["q_proj", "v_proj"],  # Apply only to attention layers
        lora_dropout=0.05,           # Prevent overfitting
        bias="none",
        task_type="CAUSAL_LM"
    )
    
    model = get_peft_model(model, config)
    
    # Print number of trainable parameters
    trainable = sum(p.numel() for p in model.parameters() if p.requires_grad)
    total = sum(p.numel() for p in model.parameters())
    print(f"Trainable parameters: {trainable:,} / {total:,} ({trainable/total*100:.2f}%)")
    
    return model


# ── 5. Search models on Hub ────────────────────────────
def search_hub():
    """Search models on Hugging Face Hub using the API"""
    from huggingface_hub import HfApi
    
    api = HfApi()
    
    # Search for fine-tuned SDXL models
    models = api.list_models(
        filter="stable-diffusion-xl",
        sort="downloads",
        direction=-1,  # Descending order
        limit=5
    )
    
    print("Top 5 popular SDXL models:")
    for model in models:
        print(f"  {model.id} — {model.downloads:,} downloads")


if __name__ == "__main__":
    # Explore ecosystem (without downloading)
    search_hub()
    
    # Dataset preview (streaming)
    explore_dataset("laion/laion-art")
```

## Pitfall Cards — 5 Common Mistakes

1. **"I don't know the cache path"** → Downloaded models accumulate at `~/.cache/huggingface/hub/`. On cloud instances, home directory space is limited. Change the path to your data disk using the `HF_HOME=/data/hf_cache` environment variable.

2. **"I don't use `device_map='auto'`"** → If you manually do `.to('cuda')`, the entire model tries to fit on one GPU and causes OOM. `device_map='auto'` automatically distributes across multiple GPUs and CPU memory.

3. **"I don't check the license"** → `meta-llama/Llama-3.1-8B` requires agreeing to Meta's license. Models marked as `gated` require you to log in with your HF account and request access before downloading.

4. **"I load large datasets without `streaming=True`"** → Running `load_dataset("laion/laion2B-en")` without streaming downloads hundreds of GB. Always add `streaming=True` when exploring.

5. **"I specify LoRA `target_modules` incorrectly"** → Different models have different layer names. LLaMA uses `q_proj`, GPT-2 uses `c_attn`. Check the model structure with `print(model)` and find the attention layer names yourself.

## Practice Problems

**Problem 1.** Explain the difference between `transformers` and `diffusers` in one sentence. In what situations would you use each?

> **Answer:** `transformers` is exclusively for text processing models (LLMs, BERT, etc.), while `diffusers` is exclusively for image generation/editing models (Stable Diffusion, Flux, etc.). Use `transformers` for text generation, `diffusers` for image generation.

**Problem 2.** If you increase LoRA rank from 4 to 64, what changes? Explain 3 tradeoffs.

> **Answer:** ① VRAM usage increases (proportional to rank) ② Training speed decreases ③ Representation capacity increases (can learn more complex patterns). Generally rank 4~16 offers good value. Rank 64 approaches full fine-tune quality but requires much more VRAM.

**Problem 3.** Write a bash command to set the `HF_HOME` environment variable to `/data/hf_cache` and run the `load_sdxl()` function.

> **Answer:** `HF_HOME=/data/hf_cache python -c "from ch04 import load_sdxl; load_sdxl()"` or add `export HF_HOME=/data/hf_cache` to `.bashrc` then run `source ~/.bashrc`.
