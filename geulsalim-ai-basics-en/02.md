## Theory

### Where are the bottlenecks in ML development?

Traditional ML development cycles look like this:

```
Idea → Code Writing (3 days) → Bug Fixing (2 days) → Experimentation (1 day) → Result Analysis (1 day) → Repeat
```

More than 70% of total time is consumed by code writing and bug fixing. The **real ML work** of thinking about model architecture and analyzing data accounts for only 30%.

Claude Code removes this bottleneck. By handling code writing and debugging through conversation, solo developers have more time to focus on high-level tasks.

### Three usage modes for Claude Code

**Mode 1: Interactive**
Running `claude` in your terminal starts interactive mode. You can request code in natural language and execute it immediately.

```bash
claude
> Make an image classification model for me. Based on ResNet18.
```

**Mode 2: Non-interactive (`-p` flag)**
Call Claude Code automatically from scripts or cron jobs. This is the core of ML pipeline automation.

```bash
claude -p "Write a callback class that saves a checkpoint when training loss drops below 0.001"
```

**Mode 3: File-based (`--input`/`--output`)**
Read prompts from files and save results to files. Used for bulk code generation.

```bash
claude -p "$(cat prompt.md)" > generated_code.py
```

### Five stages of ML workflows and how to apply Claude Code

**Stage 1: Data Preparation**
Delegate data preprocessing scripts, license validation, and file format conversion code to Claude Code.

```bash
claude -p "Convert image datasets downloaded from AI-Hub to WebDataset format. Include resizing to 512×512 and PNG to JPEG conversion."
```

**Stage 2: Model Selection and Configuration**
Search for models on Hugging Face Hub and configure fine-tuning settings.

**Stage 3: Training Execution**
Training script writing, hyperparameter experiment planning, and wandb integration.

**Stage 4: Evaluation**
FID and CLIP Score calculation scripts, visualization code.

**Stage 5: Deployment**
FastAPI server code, Docker image building, model optimization.

### Prompt Engineering — ML Code Request Patterns

To get good ML code from Claude Code, you need to specify three things in your prompt:

1. **Library versions**: `torch 2.1, diffusers 0.25, transformers 4.37`
2. **Hardware constraints**: `VRAM 24GB, single A100`
3. **Output format**: `class-based, Korean comments, type hints included`

## Practical Code

```python
# Utility for calling Claude Code CLI directly from Python
# Used for automating ML pipelines

import subprocess
import json
import tempfile
import os
from pathlib import Path

def ask_claude(prompt: str, output_file: str = None) -> str:
    """
    Call Claude Code CLI and receive response
    
    Args:
        prompt: Prompt to send to Claude
        output_file: File path to save response (optional)
    
    Returns:
        Claude's response text
    """
    # Save prompt to temporary file (for handling long prompts)
    with tempfile.NamedTemporaryFile(mode='w', suffix='.txt', 
                                     delete=False, encoding='utf-8') as f:
        f.write(prompt)
        prompt_file = f.name
    
    try:
        result = subprocess.run(
            ['claude', '-p', prompt],
            capture_output=True,
            text=True,
            timeout=120  # 2 minute limit
        )
        response = result.stdout.strip()
        
        if output_file:
            Path(output_file).write_text(response, encoding='utf-8')
            print(f"Saved to: {output_file}")
        
        return response
    
    except subprocess.TimeoutExpired:
        return "Error: Claude response timeout (120 seconds)"
    except FileNotFoundError:
        return "Error: claude CLI not installed or not in PATH"
    finally:
        os.unlink(prompt_file)


def generate_training_script(
    model_name: str,
    dataset_path: str,
    output_dir: str,
    epochs: int = 10,
    batch_size: int = 4
) -> str:
    """Auto-generate ML training script with Claude Code"""
    
    prompt = f"""
    Write a PyTorch training script with the following specifications:
    
    - Base model: {model_name}
    - Dataset path: {dataset_path}
    - Output directory: {output_dir}
    - Epochs: {epochs}
    - Batch size: {batch_size}
    
    Requirements:
    - Use Hugging Face Trainer
    - Log training with wandb
    - Save checkpoint every epoch
    - Include Korean comments
    - Include type hints
    """
    
    output_path = f"{output_dir}/train.py"
    os.makedirs(output_dir, exist_ok=True)
    
    code = ask_claude(prompt, output_path)
    return code


# Usage example
if __name__ == "__main__":
    # Auto-generate training script
    script = generate_training_script(
        model_name="stabilityai/stable-diffusion-xl-base-1.0",
        dataset_path="/data/korean-historical-images",
        output_dir="./output",
        epochs=20,
        batch_size=2
    )
    print("Training script generation complete")
    print(f"Preview:\n{script[:500]}...")
```

## Pitfall Cards — Five common mistakes

1. **"Code generated by Claude Code is always correct"** → False. Always execute and verify. APIs can differ by library version. Paste error messages back to Claude and it fixes them immediately.

2. **"Automating without the `-p` flag"** → Running just `claude` enters interactive mode and breaks automation. Scripts must use the `-p "prompt"` format.

3. **"Not providing hardware info in the prompt"** → Code that runs on 8GB VRAM differs from code for 24GB. Omitting hardware constraints produces unusable code.

4. **"Requesting everything in one prompt"** → "Write all the preprocessing + training + deployment code" produces lower quality. Breaking it into stages and requesting separately yields much more accurate code.

5. **"Using responses as-is without modifications"** → Claude Code responses are a starting point. Always adjust paths, filenames, and hyperparameters to match your actual environment.

## Practice Problems

**Problem 1.** Explain why the `-p` flag is important for ML automation in Claude Code. Include differences from interactive mode.

> **Answer:** The `-p` flag receives a response and exits in a single command. It's callable from crontabs, shell scripts, and Python subprocesses. Interactive mode requires human intervention, so it can't be used in automation pipelines. It's essential for ML's repetitive hyperparameter experiments and batch code generation.

**Problem 2.** Find the problems in this prompt and improve it: `"Make an image classification model for me"`

> **Answer:** ① Missing base model ② Unspecified dataset format ③ No hardware constraints ④ No output format specified. Improved: `"Make an image classification model based on ResNet18. Data is ImageFolder format, 8GB VRAM environment, class-based, Korean comments, include type hints."`

**Problem 3.** Extend the `generate_training_script()` function to automatically determine batch size based on VRAM size. (8GB → batch 1, 16GB → batch 2, 24GB+ → batch 4)

> **Answer:** Use `import torch; vram = torch.cuda.get_device_properties(0).total_memory / 1e9` to check VRAM, then use conditional statements to set batch size. Execute this logic before calling `generate_training_script()`.
