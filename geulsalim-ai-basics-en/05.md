## Theory

### The Librarian Analogy

The best way to understand a transformer is through the analogy of a librarian.

Imagine you ask a librarian for "literary works related to the Korean War in the 1950s." The librarian searches through thousands of books in the library by **focusing on highly relevant ones**. They assign different **weights** to keywords like "1950s," "Korean War," and "literature."

This is Attention. It calculates how much each word in an input sentence relates to other words and focuses on highly relevant parts.

### Q, K, V — Three Roles

Attention operates with three vectors.

**Query (Q)**: What I'm looking for. "I'm searching for war-related literature."
**Key (K)**: Index cards of books. "This book is about war, the 1950s, poetry."
**Value (V)**: The actual content of the books. Values are weighted-summed according to the similarity between Q and K.

Mathematically:
```
Attention(Q, K, V) = softmax(Q × K^T / √d_k) × V
```

Why divide by `√d_k`: As dimensions increase, dot products grow larger, causing softmax to operate too extremely. This prevents that problem.

### Three Transformer Architectures

**Encoder-only (BERT family)**
- Understands the entire input sentence bidirectionally
- Used for text classification, sentiment analysis, document embedding
- Only "understands" text without generating it

**Decoder-only (GPT, LLaMA family)**
- Predicts the next token while seeing only previous tokens (causal attention)
- Used for text generation, code generation, dialogue
- The foundation of Claude and ChatGPT that we use

**Encoder-Decoder (T5, BART family)**
- Encoder understands input, decoder generates output
- Used for translation, summarization, question answering

Architecture choice depends on your fine-tuning goal. Image restoration caption generation → Decoder. Understanding ancient texts → Encoder. Classical Chinese → Modern Korean translation → Encoder-Decoder.

### Why Transformers Beat CNNs and RNNs

**RNN's problem**: Processes sentences sequentially, which is slow. Information from earlier parts of long sentences gets diluted.

**CNN's problem**: Only sees local patterns. Difficult to capture the full context of a sentence.

**Transformer's strength**: All tokens simultaneously reference each other at all positions. Enables parallel processing. Captures long-distance dependencies.

## Practical Code

```python
# Implementing attention mechanism directly — understanding the formula through code
import numpy as np

def softmax(x: np.ndarray) -> np.ndarray:
    """Numerically stable softmax"""
    e_x = np.exp(x - np.max(x, axis=-1, keepdims=True))
    return e_x / e_x.sum(axis=-1, keepdims=True)

def scaled_dot_product_attention(
    Q: np.ndarray,
    K: np.ndarray,
    V: np.ndarray,
    mask: np.ndarray = None
) -> tuple:
    """
    Scaled dot-product attention implementation
    
    Args:
        Q: Query matrix [seq_len, d_k]
        K: Key matrix [seq_len, d_k]
        V: Value matrix [seq_len, d_v]
        mask: Causal mask for decoder (optional)
    
    Returns:
        (output, attention_weights)
    """
    d_k = Q.shape[-1]
    
    # 1. Dot product of Q and K: compute similarity
    scores = Q @ K.T / np.sqrt(d_k)  # [seq_len, seq_len]
    
    # 2. Masking (decoder: prevent looking at future tokens)
    if mask is not None:
        scores = np.where(mask == 0, -1e9, scores)
    
    # 3. Softmax: convert similarity to probability
    weights = softmax(scores)  # sum of each row = 1
    
    # 4. Weighted sum of values
    output = weights @ V
    
    return output, weights


def demonstrate_attention():
    """Visually confirm what attention does"""
    np.random.seed(42)
    
    # Simple example: 4 tokens, dimension 8
    seq_len, d_k = 4, 8
    tokens = ["I", "like", "Korean", "history"]
    
    # Random Q, K, V (in practice, generated with learned weights)
    Q = np.random.randn(seq_len, d_k)
    K = np.random.randn(seq_len, d_k)
    V = np.random.randn(seq_len, d_k)
    
    output, weights = scaled_dot_product_attention(Q, K, V)
    
    print("=== Attention Weight Matrix ===")
    print("(rows: Query tokens, columns: how much to focus on Key tokens)")
    print(f"\n{'':10}", end="")
    for t in tokens:
        print(f"{t:8}", end="")
    print()
    
    for i, token in enumerate(tokens):
        print(f"{token:10}", end="")
        for w in weights[i]:
            print(f"{w:.4f}  ", end="")
        print()
    
    print(f"\nOutput shape: {output.shape}")
    print("Each token has the result of weighted sum of all tokens' Values.")


def causal_mask(seq_len: int) -> np.ndarray:
    """
    Generate causal mask for decoder
    Future positions are 0, past/present are 1
    """
    mask = np.tril(np.ones((seq_len, seq_len)))
    return mask


def demonstrate_causal_attention():
    """GPT-style decoder attention"""
    seq_len, d_k = 4, 8
    tokens = ["I", "like", "Korean", "history"]
    
    Q = np.random.randn(seq_len, d_k)
    K = np.random.randn(seq_len, d_k)
    V = np.random.randn(seq_len, d_k)
    
    mask = causal_mask(seq_len)
    output, weights = scaled_dot_product_attention(Q, K, V, mask)
    
    print("\n=== Causal Attention Weights (GPT Style) ===")
    print("(Future tokens cannot be referenced — masked to 0)")
    print(f"\n{'':8}", end="")
    for t in tokens:
        print(f"{t:8}", end="")
    print()
    
    for i, token in enumerate(tokens):
        print(f"{token:8}", end="")
        for j, w in enumerate(weights[i]):
            if j > i:
                print(f"{'---':8}", end="")  # masked position
            else:
                print(f"{w:.4f}  ", end="")
        print()


if __name__ == "__main__":
    demonstrate_attention()
    demonstrate_causal_attention()
```

## Pitfall Cards — 5 Common Mistakes

1. **"Transformer = GPT"** → GPT is one implementation of a transformer. BERT, T5, ViT (image), Whisper (speech) are all transformer-based. You must distinguish between architecture (blueprint) and implementation (product).

2. **"More attention is always better"** → More attention heads and layers increase expressiveness, but VRAM and computation grow quadratically. For fine-tuning purposes, excessive size is often disadvantageous.

3. **"Don't understand why we divide by `d_k`"** → As dimensions increase, the variance of dot products grows. High variance causes softmax to become extremely 0/1, leading to vanishing gradients. Dividing by `√d_k` normalizes the variance.

4. **"Don't understand the difference between encoder and decoder attention"** → Encoder is bidirectional (references all tokens), decoder is unidirectional (references only past). The presence or absence of masking is key.

5. **"Forget positional encoding"** → Transformers have no order information built in. To distinguish "I like you" from "You I like," position information must be explicitly added. Positional Encoding serves this role.

## Practice Problems

**Problem 1.** Among encoder-only, decoder-only, and encoder-decoder architectures, which is most suitable for "extracting text from ancient document images and translating it to modern Korean"? Explain your reasoning.

> **Answer:** Encoder-Decoder. The encoder understands the input (ancient document images/text), and the decoder generates modern Korean output. When input and output are in different forms, Encoder-Decoder is optimal.

**Problem 2.** What happens if you remove the `mask` parameter from the `scaled_dot_product_attention()` function and use it as encoder attention?

> **Answer:** All tokens can reference all positions. The weight matrix fills in the upper triangular part. This becomes bidirectional attention, BERT-style.

**Problem 3.** For a sequence length of 512, what is the size of the attention weight matrix? Why is this a bottleneck for "long document processing"?

> **Answer:** 512 × 512 = 262,144 elements. In multi-head attention, this is replicated by the number of heads × batch size. If sequence length is n, memory is O(n²). For 4,096 tokens it's 16M, for 32K tokens it's 1B — this is why "long context windows" are difficult.
