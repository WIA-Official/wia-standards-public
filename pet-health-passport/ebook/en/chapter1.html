Chapter 1: Pet Health Records Management

WIA Pet Health Passport Standard - Digital Health Records Architecture

1.1 Introduction to Digital Pet Health Records

Digital pet health records represent a paradigm shift in veterinary care management, enabling seamless data exchange, improved care coordination, and enhanced pet owner engagement. The WIA Pet Health Passport Standard defines a comprehensive framework for creating, maintaining, and exchanging pet health information across diverse platforms, jurisdictions, and care providers.

Traditional paper-based veterinary records suffer from numerous limitations including poor portability, difficult duplication, susceptibility to damage or loss, and inability to support real-time data sharing. Digital health records eliminate these challenges while introducing new capabilities such as automated reminders, trend analysis, predictive health monitoring, and integration with diagnostic equipment.

Core Components of Pet Health Records

Every WIA-compliant pet health record must include the following standardized components:

  • Pet Identification: Unique identifiers including microchip number, registration ID, and biometric markers
  • Owner Information: Contact details, emergency contacts, and authorization credentials
  • Medical History: Comprehensive chronological record of all health events, treatments, and observations
  • Vaccination Records: Complete immunization history with dates, products, batch numbers, and next due dates
  • Diagnostic Results: Laboratory tests, imaging studies, pathology reports, and specialist consultations
  • Treatment Plans: Active and historical treatment protocols, medication schedules, and care instructions
  • Behavioral Records: Temperament assessments, training history, and behavioral health documentation
  • Dietary Information: Nutritional requirements, food allergies, and feeding schedules

1.3 API Endpoints for Health Record Management

The WIA Pet Health Passport Standard defines the following RESTful API endpoints for health record operations. All endpoints must implement proper authentication, authorization, and rate limiting.

GET /api/v1/pets/{petId}/health-record

Retrieve complete health record for a specific pet. Requires owner or veterinarian authorization.

POST /api/v1/pets/health-record

Create a new pet health record. Requires veterinarian credentials and owner consent.

PUT /api/v1/pets/{petId}/health-record

Update existing health record. Supports partial updates via JSON Patch (RFC 6902).

GET /api/v1/pets/{petId}/health-record/summary

Retrieve condensed health record summary suitable for quick reference or emergency access.

GET /api/v1/pets/{petId}/health-record/history

Retrieve complete change history with audit trail showing all modifications.

POST /api/v1/pets/{petId}/health-record/share

Generate temporary sharing token for authorized third-party access (e.g., boarding facilities).

GET /api/v1/owners/{ownerId}/pets

List all pets associated with a specific owner account.

GET /api/v1/pets/microchip/{microchipNumber}

Lookup pet health record by microchip number for emergency or found pet scenarios.

POST /api/v1/pets/{petId}/health-record/export

Export health record in multiple formats (PDF, HL7 FHIR, WIA-JSON) for portability.

DELETE /api/v1/pets/{petId}/health-record

Archive health record (soft delete). Maintains data for legal compliance while marking as inactive.

POST /api/v1/pets/{petId}/health-record/validate

Validate health record against WIA schema and certification requirements.

GET /api/v1/pets/{petId}/health-record/attachments

Retrieve list of attached documents (X-rays, lab reports, images) with metadata.

Example 1: Creating a New Pet Health Record (JavaScript)

// Create new pet health record using WIA Pet Health Passport API
const createPetHealthRecord = async (petData) => {
  const endpoint = 'https://api.wia-pet-health.org/v1/pets/health-record';

  const healthRecord = {
    petId: crypto.randomUUID(),
    microchipNumber: '985112001234567',
    species: 'canine',
    breed: 'Golden Retriever',
    name: 'Max',
    dateOfBirth: '2020-05-15',
    sex: 'MN',
    weight: {
      value: 32.5,
      unit: 'kg',
      measuredAt: new Date().toISOString()
    },
    colorMarkings: 'Golden coat, white chest marking',
    ownerId: petData.ownerId,
    primaryVetId: petData.vetId,
    allergies: [
      {
        allergen: 'Chicken',
        severity: 'moderate',
        reaction: 'Gastrointestinal upset',
        diagnosedDate: '2021-03-10'
      }
    ],
    chronicConditions: [],
    createdAt: new Date().toISOString(),
    version: '1.0.0'
  };

  try {
    const response = await fetch(endpoint, {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
        'Authorization': `Bearer ${petData.authToken}`,
        'X-WIA-Version': '1.0'
      },
      body: JSON.stringify(healthRecord)
    });

    if (!response.ok) {
      throw new Error(`API error: ${response.status}`);
    }

    const result = await response.json();
    console.log('Health record created:', result.petId);
    return result;

  } catch (error) {
    console.error('Failed to create health record:', error);
    throw error;
  }
};

// Usage
const newPet = await createPetHealthRecord({

Example 2: Retrieving and Updating Health Record (Python)

import requests
import json
from datetime import datetime

class PetHealthRecordClient:
    def __init__(self, api_key, base_url='https://api.wia-pet-health.org/v1'):
        self.api_key = api_key
        self.base_url = base_url
        self.headers = {
            'Authorization': f'Bearer {api_key}',
            'Content-Type': 'application/json',
            'X-WIA-Version': '1.0'
        }

    def get_health_record(self, pet_id):
        """Retrieve complete health record for a pet"""
        url = f'{self.base_url}/pets/{pet_id}/health-record'
        response = requests.get(url, headers=self.headers)
        response.raise_for_status()
        return response.json()

    def update_weight(self, pet_id, weight_kg):
        """Update pet's weight measurement"""
        url = f'{self.base_url}/pets/{pet_id}/health-record'

        update_data = {
            'weight': {
                'value': weight_kg,
                'unit': 'kg',
                'measuredAt': datetime.utcnow().isoformat() + 'Z'
            },
            'updatedAt': datetime.utcnow().isoformat() + 'Z'
        }

        response = requests.put(url, headers=self.headers, json=update_data)
        response.raise_for_status()
        return response.json()

    def add_allergy(self, pet_id, allergen, severity, reaction):
        """Add new allergy to pet's health record"""
        # First, get current record
        record = self.get_health_record(pet_id)

        # Add new allergy
        new_allergy = {
            'allergen': allergen,
            'severity': severity,
            'reaction': reaction,
            'diagnosedDate': datetime.utcnow().date().isoformat()
        }

        if 'allergies' not in record:
            record['allergies'] = []
        record['allergies'].append(new_allergy)
        record['updatedAt'] = datetime.utcnow().isoformat() + 'Z'

        # Update record
        url = f'{self.base_url}/pets/{pet_id}/health-record'
        response = requests.put(url, headers=self.headers, json=record)
        response.raise_for_status()
        return response.json()

# Example usage
client = PetHealthRecordClient('your-api-key-here')

# Retrieve health record
pet_record = client.get_health_record('pet-uuid-123')
print(f"Pet: {pet_record['name']}, Species: {pet_record['species']}")

# Update weight
updated = client.update_weight('pet-uuid-123', 33.2)
print(f"Weight updated to {updated['weight']['value']} kg")

# Add allergy
client.add_allergy('pet-uuid-123', 'Pollen', 'mild', 'Sneezing and itching')

Example 3: Microchip Lookup and Emergency Access (Java)

import java.net.http.*;
import java.net.URI;
import com.google.gson.*;
import java.time.Instant;

public class PetHealthEmergencyAccess {
    private static final String BASE_URL = "https://api.wia-pet-health.org/v1";
    private final HttpClient httpClient;
    private final String apiKey;
    private final Gson gson;

    public PetHealthEmergencyAccess(String apiKey) {
        this.apiKey = apiKey;
        this.httpClient = HttpClient.newHttpClient();
        this.gson = new Gson();
    }

    /**
     * Emergency lookup of pet health record by microchip number
     * Used when found pet needs immediate medical attention
     */
    public PetHealthRecord lookupByMicrochip(String microchipNumber) throws Exception {
        String endpoint = BASE_URL + "/pets/microchip/" + microchipNumber;

        HttpRequest request = HttpRequest.newBuilder()
            .uri(URI.create(endpoint))
            .header("Authorization", "Bearer " + apiKey)
            .header("X-WIA-Version", "1.0")
            .header("X-Emergency-Access", "true")
            .GET()
            .build();

        HttpResponse response = httpClient.send(request,
            HttpResponse.BodyHandlers.ofString());

        if (response.statusCode() != 200) {
            throw new Exception("Failed to retrieve record: " + response.statusCode());
        }

        return gson.fromJson(response.body(), PetHealthRecord.class);
    }

    /**
     * Retrieve emergency summary with critical information only
     */
    public EmergencySummary getEmergencySummary(String petId) throws Exception {
        String endpoint = BASE_URL + "/pets/" + petId + "/health-record/summary";

        HttpRequest request = HttpRequest.newBuilder()
            .uri(URI.create(endpoint))
            .header("Authorization", "Bearer " + apiKey)
            .header("X-WIA-Version", "1.0")
            .GET()
            .build();

        HttpResponse response = httpClient.send(request,
            HttpResponse.BodyHandlers.ofString());

        return gson.fromJson(response.body(), EmergencySummary.class);
    }

    /**
     * Log emergency access for audit trail
     */
    public void logEmergencyAccess(String petId, String reason, String facilityId)
            throws Exception {
        String endpoint = BASE_URL + "/pets/" + petId + "/health-record/audit";

        JsonObject auditLog = new JsonObject();
        auditLog.addProperty("accessType", "emergency");
        auditLog.addProperty("reason", reason);
        auditLog.addProperty("facilityId", facilityId);
        auditLog.addProperty("timestamp", Instant.now().toString());

        HttpRequest request = HttpRequest.newBuilder()
            .uri(URI.create(endpoint))
            .header("Authorization", "Bearer " + apiKey)
            .header("Content-Type", "application/json")
            .header("X-WIA-Version", "1.0")
            .POST(HttpRequest.BodyPublishers.ofString(gson.toJson(auditLog)))
            .build();

        httpClient.send(request, HttpResponse.BodyHandlers.ofString());
    }

    public static void main(String[] args) {
        try {
            PetHealthEmergencyAccess emergency =
                new PetHealthEmergencyAccess("emergency-vet-api-key");

            // Found pet scenario
            String foundMicrochip = "985112001234567";
            PetHealthRecord record = emergency.lookupByMicrochip(foundMicrochip);

            System.out.println("Found Pet: " + record.getName());
            System.out.println("Owner Contact: " + record.getOwnerContact());
            System.out.println("Allergies: " + record.getAllergies());
            System.out.println("Chronic Conditions: " + record.getChronicConditions());

            // Log the emergency access
            emergency.logEmergencyAccess(
                record.getPetId(),
                "Found injured pet requiring immediate care",
                "emergency-clinic-12345"
            );

        } catch (Exception e) {
            System.err.println("Emergency access failed: " + e.getMessage());
        }
    }
}

1.6 WIA Certification Requirements for Health Records

To achieve WIA Pet Health Passport certification for health records management, implementations must satisfy the following requirements:

CR-1.1: Support complete pet identification with all required fields (petId, microchipNumber, species, breed, dateOfBirth, sex, ownerId)
CR-1.2: Validate all data against WIA JSON schema with proper error reporting for non-compliant submissions
CR-1.3: Implement all 12 required API endpoints with proper RESTful semantics and HTTP status codes
CR-1.4: Maintain complete audit trail of all health record modifications with user identification and timestamps
CR-1.5: Support microchip-based emergency lookup with response time under 500ms
CR-1.6: Implement TLS 1.3 encryption for all API communications with valid certificates
CR-1.7: Provide AES-256 encryption for all stored health records and personally identifiable information
CR-1.8: Support OAuth 2.0 authentication with JWT tokens and proper token expiration
CR-1.9: Implement role-based access control with minimum three roles (owner, veterinarian, administrator)
CR-1.10: Support health record export in PDF, JSON, and HL7 FHIR formats
CR-1.11: Maintain 99.9% uptime for health record retrieval services
CR-1.12: Implement automated daily backups with tested restoration procedures
CR-1.13: Support temporary sharing tokens with configurable expiration (1 hour to 30 days)
CR-1.14: Provide API versioning with backward compatibility for minimum two major versions
CR-1.15: Implement rate limiting to prevent API abuse (minimum 1000 requests/hour per authenticated user)
CR-1.16: Support GDPR-compliant data deletion with 30-day grace period
CR-1.17: Provide owner access portal for viewing and downloading their pet's complete health record
CR-1.18: Support multi-language interfaces with minimum English and one additional language
CR-1.19: Implement automatic weight tracking with trend analysis and alerts for significant changes
CR-1.20: Support attachment of medical images and documents (X-rays, lab reports) with OCR capabilities
CR-1.21: Provide API documentation conforming to OpenAPI 3.0 specification
CR-1.22: Support bulk import of legacy records with validation and error reporting
CR-1.23: Implement search functionality supporting queries by pet name, microchip, owner, and date ranges
CR-1.24: Provide mobile-responsive web interface or native mobile applications for iOS and Android
CR-1.25: Support offline access to emergency health information with sync upon reconnection
