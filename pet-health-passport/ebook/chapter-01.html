1.1 The WIA Philosophy

The World Interoperability Alliance (WIA) was founded on a simple yet profound principle: technology should serve humanity without boundaries. Our standards are designed to break down barriers between systems, nations, and organizations.

  • Open Standards - Free to implement, no licensing fees

  • Verifiable Credentials - Cryptographic proof of authenticity

1.2 Why Pet Health Passports?

Every year, millions of pets travel internationally with their owners. The current system is fragmented, paper-based, and prone to fraud. The WIA Pet Health Passport standard solves these problems:

Problem WIA Solution
Paper documents can be forged Digital signatures with DID verification
Different formats per country Universal JSON-LD schema
Manual verification delays Instant QR code scanning
Lost vaccination records Immutable blockchain anchoring
No real-time health alerts WebSocket event notifications

1.3 The 4-Phase Standard Structure

All WIA standards follow a consistent 4-phase structure, ensuring comprehensive coverage from data formats to real-world integration:

Phase 1: Data Format

Defines the structure of all data entities using JSON-LD schemas with W3C Verifiable Credentials compatibility.

Phase 2: Algorithms

Specifies verification algorithms, validation rules, and computational procedures for processing passport data.

Phase 3: Protocol

Defines REST API endpoints, WebSocket events, and communication protocols for system-to-system interaction.

Phase 4: Integration

Provides integration guides for connecting with existing systems like veterinary databases, government registries, and border control systems.
