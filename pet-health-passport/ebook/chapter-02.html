Chapter 2: Data Format Specification

JSON-LD Schemas, DID Identifiers, and Data Validation

2.1 Overview

Phase 1 defines the complete data format specification for the WIA Pet Health Passport. All data structures use JSON-LD format with W3C Verifiable Credentials compatibility, ensuring global interoperability and cryptographic verifiability.

[KEY] Key Concepts

  • JSON-LD - Linked Data format for semantic web compatibility
  • DID - Decentralized Identifiers for unique entity identification
  • Verifiable Credentials - W3C standard for tamper-proof credentials

2.2 Pet Passport Schema

The core PetPassport credential contains all essential information about a pet and its health records:

pet-passport.json
{
  "@context": [
    "https://www.w3.org/2018/credentials/v1",
    "https://wia.org/contexts/pet-health-passport/v1"
  ],
  "type": ["VerifiableCredential", "PetHealthPassport"],
  "id": "urn:uuid:3a4b5c6d-7e8f-9a0b-1c2d-3e4f5a6b7c8d",
  "issuer": {
    "id": "did:wia:vet:kr:seoul:gangnam:12345",
    "name": "Seoul Pet Hospital",
    "licenseNumber": "VET-KR-2024-12345"
  },
  "issuanceDate": "2024-01-15T09:00:00Z",
  "expirationDate": "2029-01-15T09:00:00Z",
  "credentialSubject": {
    "id": "did:wia:pet:900123456789012",
    "type": "Pet",
    "microchipId": "900123456789012",
    "microchipType": "ISO11784",
    "species": "canine",
    "breed": "Golden Retriever",
    "name": "Max",
    "dateOfBirth": "2020-03-15",
    "sex": "male",
    "neutered": true,
    "color": "Golden",
    "distinguishingMarks": "White patch on chest",
    "photo": "ipfs://QmX4k5v6w7x8y9z...",
    "owner": {
      "id": "did:wia:person:kr:kim-001",

        "country": "KR",

Field Definitions

Field Type Required Description
microchipId string Yes 15-digit ISO microchip number
species string Yes canine, feline, avian, etc.
breed string Yes Official breed name
name string Yes Pet's registered name
dateOfBirth date Yes ISO 8601 date format
sex string Yes male, female
neutered boolean Yes Neutering/spaying status
color string No Primary coat color
photo string No IPFS or URL to photo

2.4 DID Format

The WIA standard uses Decentralized Identifiers (DIDs) for all entities. The DID format follows this structure: 

did:wia:[entity-type]:[country]:[region]:[identifier]

Examples:
- Pet:      did:wia:pet:900123456789012
- Vet:      did:wia:vet:kr:seoul:gangnam:12345
- Person:   did:wia:person:kr:kim-001
- Hospital: did:wia:hospital:kr:seoul:12345

[INFO] DID Resolution

DIDs can be resolved through the WIA DID resolver service or any compatible resolver that supports the did:wia method. The resolver returns a DID Document containing public keys and service endpoints.

2.5 Rust Type Definitions

Here are the Rust type definitions for implementing the data structures:

types.rs
use serde::{Deserialize, Serialize};
use chrono::{DateTime, NaiveDate, Utc};

#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct PetPassport {
    #[serde(rename = "@context")]
    pub context: Vec<String>,
    #[serde(rename = "type")]
    pub credential_type: Vec<String>,
    pub id: String,
    pub issuer: Issuer,
    #[serde(rename = "issuanceDate")]
    pub issuance_date: DateTime<Utc>,
    #[serde(rename = "expirationDate")]
    pub expiration_date: DateTime<Utc>,
    #[serde(rename = "credentialSubject")]
    pub credential_subject: PetSubject,
    pub proof: Option<Proof>,
}

#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct PetSubject {
    pub id: String,  // DID
    #[serde(rename = "type")]
    pub subject_type: String,
    #[serde(rename = "microchipId")]
    pub microchip_id: String,
    pub species: Species,
    pub breed: String,
    pub name: String,
    #[serde(rename = "dateOfBirth")]
    pub date_of_birth: NaiveDate,
    pub sex: Sex,
    pub neutered: bool,
    pub color: Option<String>,
    pub photo: Option<String>,
    pub owner: Owner,
}

#[derive(Debug, Clone, Serialize, Deserialize)]
#[serde(rename_all = "lowercase")]
pub enum Species {
    Canine,
    Feline,
    Avian,
    Rabbit,
    Ferret,
    Other(String),
}

#[derive(Debug, Clone, Serialize, Deserialize)]
#[serde(rename_all = "lowercase")]
pub enum Sex {
    Male,
    Female,
}
