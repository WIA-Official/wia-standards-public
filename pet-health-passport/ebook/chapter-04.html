PHASE 3

Chapter 4: API Protocol Design

REST Endpoints, WebSocket Events, and Authentication

4.1 Overview

Phase 3 defines the communication protocols for the WIA Pet Health Passport system. The API follows RESTful principles with optional WebSocket support for real-time updates.

[API] Base URL

https://api.wia.org/v1/pet-passport

4.2 Passport Endpoints

POST /passports

Create a new pet health passport. Only authorized veterinary authorities can issue passports.

Request Body

{
  "pet": {
    "microchipId": "900123456789012",
    "species": "canine",
    "breed": "Golden Retriever",
    "name": "Max",
    "dateOfBirth": "2020-03-15",
    "sex": "male",
    "neutered": true
  },
  "owner": {
    "name": "Kim Minjun",
    "country": "KR",
    "contact": "+82-10-1234-5678"
  }
}

Response (201 Created)

{
  "id": "did:wia:pet:900123456789012",
  "passportNumber": "WIA-KR-2024-001234",
  "issuedAt": "2024-01-15T09:00:00Z",
  "expiresAt": "2029-01-15T09:00:00Z",
  "qrCode": "data:image/png;base64,..."
}
GET /passports/{id}

Retrieve a pet passport by DID or microchip ID.

Parameter Type Description
id string DID (did:wia:pet:...) or microchip ID
GET /passports/{id}/verify

Verify passport authenticity and check travel eligibility for a destination.

Query Param Type Description
destination string ISO country code (e.g., US, JP, EU)
travelDate date Planned travel date (YYYY-MM-DD)

Response

{
  "verified": true,
  "signatureValid": true,
  "issuerTrusted": true,
  "travelEligibility": {
    "eligible": true,
    "status": "ready",
    "requirementsMet": ["valid_passport", "rabies_vaccination"],
    "quarantine": {
      "required": false,
      "durationDays": 0
    }
  }
}

4.3 Vaccination Endpoints

POST /passports/{id}/vaccinations

Add a new vaccination record to a passport. Requires veterinary authority credentials. 

{
  "type": "rabies",
  "vaccineProduct": {
    "name": "Nobivac Rabies",
    "manufacturer": "MSD Animal Health",
    "batchNumber": "BATCH-2024-001"
  },
  "dateAdministered": "2024-01-15",
  "validUntil": "2027-01-15",
  "veterinarian": {
    "did": "did:wia:vet:person:dr-park-001",
    "licenseNumber": "VET-KR-12345"
  }
}
GET /passports/{id}/vaccinations

Get all vaccination records for a passport.

4.4 WebSocket Events

Connect to the WebSocket endpoint for real-time updates: 

wss://api.wia.org/v1/pet-passport/ws
WebSocket Events
// Subscribe to passport updates
{
  "action": "subscribe",
  "passportId": "did:wia:pet:900123456789012"
}

// Vaccination added event
{
  "event": "vaccination.added",
  "passportId": "did:wia:pet:900123456789012",
  "data": {
    "type": "rabies",
    "dateAdministered": "2024-01-15",
    "validUntil": "2027-01-15"
  },
  "timestamp": "2024-01-15T10:30:00Z"
}

// Travel eligibility changed
{
  "event": "eligibility.changed",
  "passportId": "did:wia:pet:900123456789012",
  "data": {
    "destination": "JP",
    "previousStatus": "pending_vaccination",
    "newStatus": "ready"
  },
  "timestamp": "2024-01-15T10:30:00Z"
}

4.5 Authentication

The API uses JWT tokens with DID-based authentication:

Authentication Header
Authorization: Bearer eyJhbGciOiJFZDI1NTE5IiwidHlwIjoiSldUIn0...

// JWT Payload
{
  "iss": "did:wia:vet:kr:seoul:gangnam:12345",
  "sub": "did:wia:vet:person:dr-park-001",
  "aud": "https://api.wia.org",
  "exp": 1705401600,
  "iat": 1705315200,
  "scope": ["passport:write", "vaccination:write"]
}

[KEY] Permission Scopes

  • passport:read - Read passport data
  • passport:write - Create/update passports (vet only)
  • vaccination:write - Add vaccination records (vet only)
  • verify:read - Verify passports (border control)

4.6 Error Responses

{
  "error": {
    "code": "PASSPORT_NOT_FOUND",
    "message": "No passport found for the given ID",
    "details": {
      "searchedId": "did:wia:pet:000000000000000"
    }
  },
  "timestamp": "2024-01-15T10:30:00Z",
  "traceId": "abc123"
}
Error Code HTTP Status Description
PASSPORT_NOT_FOUND 404 Passport does not exist
INVALID_SIGNATURE 401 JWT signature verification failed
UNAUTHORIZED_ISSUER 403 Issuer not authorized to perform action
VALIDATION_ERROR 400 Request body validation failed
