5.2 Veterinary System Integration

Veterinary clinics and hospitals can integrate directly with the WIA system using our SDK or REST API:

Rust SDK Example - Veterinary Integration
use wia_pet_passport::{Client, VetCredentials, PassportBuilder};

#[tokio::main]
async fn main() -> Result<(), Box> {
    // Initialize client with veterinary credentials
    let credentials = VetCredentials::load_from_env()?;
    let client = Client::new(credentials).await?;

    // Create a new passport for a pet
    let passport = PassportBuilder::new()
        .microchip_id("900123456789012")
        .species(Species::Canine)
        .breed("Golden Retriever")
        .name("Max")
        .date_of_birth(NaiveDate::from_ymd(2020, 3, 15))
        .sex(Sex::Male)
        .neutered(true)
        .owner(Owner {
            name: "Kim Minjun".into(),
            country: "KR".into(),
            contact: "+82-10-1234-5678".into(),
        })
        .build()?;

    let result = client.create_passport(passport).await?;
    println!("Created passport: {}", result.did);

    // Add vaccination record
    let vaccination = VaccinationRecord {
        vaccine_type: VaccineType::Rabies,
        product_name: "Nobivac Rabies".into(),
        manufacturer: "MSD Animal Health".into(),
        batch_number: "BATCH-2024-001".into(),
        date_administered: Utc::now(),
        valid_until: Utc::now() + Duration::days(365 * 3),
    };

    client.add_vaccination(&result.did, vaccination).await?;

    Ok(())
}

5.3 Government Database Integration

European Union

[EU] TRACES NT Integration

The WIA system integrates with TRACES NT (Trade Control and Expert System) for EU pet travel compliance.

  • Automatic health certificate generation
  • Real-time status synchronization
  • EU Pet Passport number linking
United States

[US] USDA APHIS Integration

Integration with USDA Animal and Plant Health Inspection Service for US import/export compliance.

  • USDA health certificate validation
  • CDC import permit integration
  • State-specific requirement checks
South Korea

[KR] MAFRA Integration

Integration with Ministry of Agriculture, Food and Rural Affairs animal registration system.

  • National animal registration sync
  • Quarantine requirement verification
  • Domestic travel compliance
Japan

[JP] MAFF NACCS Integration

Integration with Japan's Animal Quarantine Service through NACCS system.

  • 180-day advance notification
  • Rabies antibody titer verification
  • Quarantine facility booking

5.4 Airline Integration

Airlines can verify pet travel eligibility during booking and check-in:

Airline Integration Example
// Verify pet eligibility for flight
POST /v1/pet-passport/verify/flight
Content-Type: application/json

{
  "passportId": "did:wia:pet:900123456789012",
  "flight": {
    "airline": "KE",
    "flightNumber": "KE001",
    "origin": "ICN",
    "destination": "LAX",
    "departureDate": "2024-06-15",
    "cabin": "cargo"  // or "cabin" for in-cabin travel
  }
}

// Response
{
  "eligible": true,
  "requirements": {
    "healthCertificate": {
      "required": true,
      "status": "valid",
      "expiresAt": "2024-06-25T00:00:00Z"
    },
    "vaccinations": {
      "rabies": "valid",
      "dhpp": "valid"
    },
    "crate": {
      "required": true,
      "dimensions": "IATA compliant required"
    }
  },
  "warnings": []
}

5.5 QR Code Integration

Each passport includes a QR code for instant verification at border control:

[QR] QR Code Contents

The QR code contains a signed JWT with minimal verification data:

  • Passport DID
  • Pet microchip ID
  • Expiration date
  • Verification URL
  • Cryptographic signature
// QR Code JWT payload
{
  "iss": "did:wia:vet:kr:seoul:gangnam:12345",
  "sub": "did:wia:pet:900123456789012",
  "mic": "900123456789012",
  "exp": 1893456000,
  "url": "https://verify.wia.org/p/abc123"
}

// Scan result shows:
// [OK] Passport Valid
// [PET] Max (Golden Retriever)
// [CHIP] 900123456789012
// [EXPIRES] 2029-01-15
