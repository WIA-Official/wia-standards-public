Chapter 2: Data Ingestion

Extracting data without dropping any

Chapter overview: Ingestion patterns (full / incremental / CDC), per-source recipes (database, API, file, stream), validation, error handling, performance tactics, an end-to-end worked example (REST API → S3 → Iceberg), and the metrics every ingestion job MUST emit.

2.1 What is ingestion?

Ingestion is the boundary where data leaves the system that produced it and enters yours. Everything downstream — quality, lineage, freshness — is bounded by what ingestion captured. The cardinal rule: capture loud, drop nothing silently. Anything you can not capture, log explicitly with reason and identifier so a later run can pick it up.

2.2 Three ingestion patterns

Full load

Read the entire source on every run. Simple, correct, expensive. Use it for slowly-changing reference tables (countries, currencies) or in development. Avoid for transactional tables > 10 GB.

Incremental load (watermark)

Read only rows where updated_at > last_watermark. Save the new high-water mark in a metadata table. Caveat: requires a monotonic timestamp on the source; deletes are invisible.

SELECT * FROM orders
WHERE updated_at > :last_watermark
  AND updated_at <= :now
ORDER BY updated_at;

-- Persist the new watermark only after sink commit succeeds.

Change Data Capture (CDC)

Tail the database transaction log. Captures inserts, updates, AND deletes. Debezium 3 + Kafka Connect is the open-source default; AWS DMS, Oracle GoldenGate, and Fivetran HVR cover commercial use cases. Pair with the **Outbox pattern** to avoid dual-write inconsistency.

REST API extractor (with rate-limit etiquette)

import time, requests
from tenacity import retry, wait_exponential_jitter, stop_after_attempt

@retry(wait=wait_exponential_jitter(initial=2, max=60), stop=stop_after_attempt(8))
def fetch_page(url, params, headers):
    r = requests.get(url, params=params, headers=headers, timeout=30)
    if r.status_code == 429:
        retry_after = int(r.headers.get("Retry-After","2"))
        time.sleep(retry_after)
        raise RuntimeError("rate-limited")
    r.raise_for_status()
    return r.json()

cursor, page = None, 0
while True:
    data = fetch_page("https://api.partner.com/v1/orders",
                      params={"cursor":cursor,"limit":500},
                      headers={"Authorization":f"Bearer {TOKEN}"})
    yield from data["items"]
    if not data.get("next_cursor"): break
    cursor = data["next_cursor"]; page += 1

2.4 Validation at the boundary

Validate as soon as data enters. Reject loudly, never silently coerce.

# Pydantic v2 model — runs on every record at ingest
from pydantic import BaseModel, Field, conint, condecimal
from datetime import datetime

class OrderEvent(BaseModel):
    order_id:    conint(ge=1)
    user_id:     conint(ge=1)
    total_cents: conint(ge=0, le=10**11)
    currency:    str = Field(min_length=3, max_length=3)
    created_at:  datetime
    model_config = {"extra":"forbid"}   # reject unknown fields loudly

def validate(record: dict) -> OrderEvent:
    return OrderEvent.model_validate(record)

2.5 Error handling

Every ingestion run produces three buckets:

  1. Accepted — written to bronze.
  2. Quarantined — written to s3://lake/quarantine/orders/dt=…/ with original row plus error reason.
  3. Lost — should be empty. If non-empty, the run fails the quality gate.

Quarantine is reviewable; backfill scripts replay quarantined rows after producers fix the schema.

2.6 Performance tactics

  • Parallel partitions — split the source by primary key range or time bucket; one worker per range.
  • Column pruning — only fetch columns the destination actually needs (saves egress and decode).
  • Compression on the wire — gzip/zstd HTTP responses, lz4 over Kafka.
  • Connection pooling — pgbouncer for Postgres, redis-pool for cache.
  • Micro-batching — flush every 64 MB or 30 s, whichever comes first.

2.7 Worked example: REST API → Iceberg

from datetime import datetime
import pyarrow as pa, pyarrow.parquet as pq, pyiceberg.catalog as ic

catalog = ic.load_catalog("polaris-prod", **{
    "uri":"https://api.wia-data-pipeline.org/iceberg/v1",
    "credential":f"{CLIENT_ID}:{CLIENT_SECRET}"
})

records = list(extract_orders())                  # §2.3 generator
records = [validate(r).model_dump() for r in records]
table = pa.Table.from_pylist(records, schema=ARROW_SCHEMA)

# Land bronze parquet for replay safety
pq.write_table(table, f"s3://bronze/orders/dt={today}/part-0001.parquet",
               compression="zstd", write_page_index=True)

# Append to Iceberg (transactional, snapshot-isolated)
iceberg = catalog.load_table("lake.orders")
iceberg.append(table)

# Emit OpenLineage events: START, COMPLETE
emit_lineage("COMPLETE", run_id=RUN_ID, inputs=["api://partner/orders"],
             outputs=["iceberg://polaris-prod/lake.orders"])
