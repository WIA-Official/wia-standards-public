8.1 Real-time pipeline — Kafka + Spark Structured Streaming

// Bronze ingest — Kafka → Iceberg
val events = spark.readStream
  .format("kafka")
  .option("kafka.bootstrap.servers","kafka:9093")
  .option("subscribe","events.orders")
  .option("startingOffsets","earliest")
  .option("kafka.isolation.level","read_committed")
  .load()

val parsed = events.select(
  from_avro($"value", schemaRegistry, "orders-value").as("o"))
  .select($"o.*", current_timestamp().as("ingest_ts"))

parsed.writeStream
  .format("iceberg")
  .option("path","polaris-prod.bronze.orders")
  .option("checkpointLocation","s3://checkpoints/orders/")
  .trigger(Trigger.ProcessingTime("30 seconds"))
  .outputMode("append")
  .start()

8.2 CDC pipeline — Debezium → Kafka → Iceberg MERGE

-- streaming MERGE — Flink Iceberg sink, exactly-once
INSERT INTO polaris-prod.lake.orders
SELECT * FROM TABLE(
  TUMBLE(TABLE prod-db.public.orders, DESCRIPTOR(updated_at), INTERVAL '1' MINUTES))
GROUP BY order_id, user_id, total_cents, currency,
         created_at, updated_at, window_start, window_end
ON DUPLICATE order_id UPDATE WHEN s.updated_at > t.updated_at;

Debezium's schema.history.internal topic preserves DDL events for replay; new consumers can rebuild state by replaying from the earliest offset.

Lambda still earns its keep when the batch path computes something the streaming path can not (heavy joins on multi-day windows). When that compromise is no longer required, prefer Kappa.

8.4 Data Mesh in practice

  • Domain ownership — finance owns finance datasets, growth owns growth, etc.
  • Data products — every dataset has an owner, an SLA, a contract, an SLO dashboard.
  • Self-service platform — central platform provides Iceberg + Polaris + Airflow + DataHub.
  • Federated governance — global rules (PII tagging, retention) enforced via Open Policy Agent.

      - run: dbt parse --profiles-dir ci-profiles
  test:
    runs-on: ubuntu-22.04
    steps:
      - uses: actions/setup-python@v5
      - run: pip install -r requirements-dev.txt
      - run: pytest tests/dags                  # DAG load tests
      - run: dbt build --target ci --select state:modified+
      - run: soda scan -d ci -c configuration.yml contracts/
  preview:
    needs: [lint, test]
    runs-on: ubuntu-22.04
    steps:
      - run: ./scripts/spin-up-staging.sh
      - run: dbt run --target staging
      - run: soda scan -d staging -c configuration.yml contracts/

8.6 Cost optimisation

  • Right-size files. 128 MB Parquet, ZSTD-3 default, ZSTD-9 for cold archives.
  • Compaction job. Iceberg RewriteFiles weekly removes small-file overhead.
  • Snapshot expiration. Keep 30 days of history; older snapshots become storage-only.
  • Workload separation. Heavy ELT on dedicated warehouse; BI on a separate one.
  • Auto-suspend. Snowflake / BigQuery reservations idle out aggressively.
  • Cold partitioning. Tag old partitions STORAGE_CLASS=GLACIER; reload only on query.

8.8 Pre-flight checklist

  1. Every dataset has a published Bitol data contract.
  2. Every pipeline has an OpenLineage event stream.
  3. Every mutating call carries an Idempotency-Key.
  4. Quality gate runs as the last task and blocks publish on failure.
  5. Snapshot retention <= 30 days; compaction weekly.
  6. Cost tags applied to every warehouse session.
  7. Backfill recipe documented and tested.
  8. Runbook published; on-call rotation in place.
  9. PII columns tokenised at ingest, not at egress.
  10. Schema Registry compatibility level set to BACKWARD_TRANSITIVE.

8.9 Best-practice summary

  • Idempotent everything. If you can not replay safely, you can not recover safely.
  • Contracts first. The contract is the agreement; the code is implementation.
  • Quality at boundaries. Validate at ingest, validate at publish.
  • Lineage by default. Every job emits OpenLineage events.
  • Open formats. Parquet / Avro / Iceberg keep your data portable.
  • Observability is non-optional. Metrics, logs, traces, lineage — all four pillars.
  • Cost discipline. Tag, monitor, alert — the bill controls itself only if you watch.
