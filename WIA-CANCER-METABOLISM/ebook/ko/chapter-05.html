핵심 개념: 잘 설계된 API는 암 대사 데이터를 안전하고 효율적으로 교환할 수 있게 합니다. RESTful 원칙, 명확한 인증 메커니즘, 포괄적인 SDK를 통해 개발자는 WIA-CANCER-METABOLISM 표준을 쉽게 통합할 수 있습니다.

5.1 RESTful API 설계 원칙

WIA-CANCER-METABOLISM API는 REST(Representational State Transfer) 아키텍처 스타일을 따릅니다. RESTful API는 다음과 같은 핵심 원칙을 준수합니다:

  • 클라이언트-서버 분리: 클라이언트와 서버는 독립적으로 발전할 수 있습니다
  • 무상태성(Stateless): 각 요청은 독립적이며 모든 필요한 정보를 포함합니다
  • 캐시 가능: 응답은 캐시 가능 여부를 명시해야 합니다
  • 계층화된 시스템: 클라이언트는 최종 서버와 직접 연결되었는지 알 수 없습니다
  • 통일된 인터페이스: 일관된 URI 구조와 HTTP 메서드 사용

5.1.1 리소스 중심 설계

API는 데이터를 리소스로 모델링하며, 각 리소스는 고유한 URI를 가집니다:

리소스 URI 패턴 설명
대사체 프로파일 /api/v1/profiles 환자의 대사체 프로파일 컬렉션
특정 프로파일 /api/v1/profiles/{profileId} 개별 대사체 프로파일
대사체 /api/v1/metabolites 대사체 참조 데이터
바이오마커 /api/v1/biomarkers 바이오마커 정의 및 참조 범위
대사 경로 /api/v1/pathways 대사 경로 정보

5.1.2 HTTP 메서드 활용

각 HTTP 메서드는 명확한 의미를 가지며 CRUD(Create, Read, Update, Delete) 작업에 대응됩니다:

메서드 작업 예제 멱등성
GET 조회 (Read) GET /api/v1/profiles/{id} ✓
POST 생성 (Create) POST /api/v1/profiles ✗
PUT 전체 수정 (Update) PUT /api/v1/profiles/{id} ✓
PATCH 부분 수정 (Partial Update) PATCH /api/v1/profiles/{id} ✗
DELETE 삭제 (Delete) DELETE /api/v1/profiles/{id} ✓

5.3 인증 및 권한 부여

WIA-CANCER-METABOLISM API는 OAuth 2.0과 JWT(JSON Web Token)를 사용한 강력한 인증 메커니즘을 제공합니다. 의료 데이터의 민감성을 고려하여 다층 보안이 적용됩니다.

5.3.1 OAuth 2.0 인증 흐름

API는 다음과 같은 OAuth 2.0 흐름을 지원합니다:

흐름 유형 사용 사례 보안 수준
Authorization Code 웹 애플리케이션 높음
Client Credentials 서버 간 통신 높음
PKCE 모바일/SPA 앱 매우 높음 
# 1. 인증 코드 요청
GET /oauth/authorize?
  response_type=code&
  client_id=your_client_id&
  redirect_uri=https://yourapp.com/callback&
  scope=read:profiles write:profiles&
  state=random_state_string

# 2. 액세스 토큰 교환
POST /oauth/token
Content-Type: application/x-www-form-urlencoded

grant_type=authorization_code&
code=received_auth_code&
client_id=your_client_id&
client_secret=your_client_secret&
redirect_uri=https://yourapp.com/callback

Response: 200 OK
{
  "access_token": "eyJhbGciOiJSUzI1NiIsInR5cCI6IkpXVCJ9...",
  "token_type": "Bearer",
  "expires_in": 3600,
  "refresh_token": "def502003ca7...",
  "scope": "read:profiles write:profiles"
}

# 3. API 호출 시 액세스 토큰 사용

Authorization: Bearer eyJhbGciOiJSUzI1NiIsInR5cCI6IkpXVCJ9...

5.4.1 TypeScript/JavaScript SDK

import { WIACancerMetabolismClient } from '@wia/cancer-metabolism-sdk';

// 클라이언트 초기화
const client = new WIACancerMetabolismClient({
  apiUrl: 'https://api.wiastandards.com',
  clientId: 'your_client_id',
  clientSecret: 'your_client_secret',
  scope: ['read:profiles', 'write:profiles']
});

// 인증
await client.authenticate();

// 프로파일 생성
const profile = await client.profiles.create({
  patientId: 'PT-2025-0001-ANON',
  sampleDate: new Date('2025-01-15T09:30:00Z'),
  sampleType: 'serum',
  cancerType: 'C34.1',
  metabolites: [
    {
      metaboliteId: 'HMDB0000122',
      name: 'D-Glucose',
      concentration: 5.8,
      unit: 'mM'
    }
  ]
});

console.log(`프로파일 생성됨: ${profile.profileId}`);

// 프로파일 조회
const retrieved = await client.profiles.get('WCM-20250115-A7F3D92E');

// 프로파일 검색 (페이지네이션)
const results = await client.profiles.search({
  cancerType: 'C34.1',
  page: 1,
  limit: 20
});

results.data.forEach(p => {
  console.log(`${p.profileId}: ${p.metabolites.length} metabolites`);
});

// 바이오마커 평가
const evaluation = await client.biomarkers.evaluate({
  profileId: 'WCM-20250115-A7F3D92E'
});

console.log(`전체 위험 점수: ${evaluation.overallRiskScore}`);

5.5.3 SDK 에러 처리 예제

// TypeScript
try {
  const profile = await client.profiles.create(data);
} catch (error) {
  if (error instanceof WIAValidationError) {
    console.error('검증 오류:', error.details);
    error.details.forEach(detail => {
      console.error(`${detail.field}: ${detail.message}`);
    });
  } else if (error instanceof WIAAuthenticationError) {
    console.error('인증 오류:', error.message);
    // 재인증 시도
    await client.authenticate();
  } else if (error instanceof WIARateLimitError) {
    console.error('속도 제한 초과, 대기 중...');
    await sleep(error.retryAfter * 1000);
    // 재시도
  } else {
    console.error('알 수 없는 오류:', error);
  }
}

// Python
try:
    profile = client.profiles.create(data)
except WIAValidationError as e:
    print(f"검증 오류: {e.message}")
    for detail in e.details:
        print(f"{detail['field']}: {detail['message']}")
except WIAAuthenticationError as e:
    print(f"인증 오류: {e.message}")
    client.authenticate()
except WIARateLimitError as e:
    print(f"속도 제한 초과, {e.retry_after}초 후 재시도")
    time.sleep(e.retry_after)
except WIAAPIError as e:
    print(f"API 오류: {e.message}")
