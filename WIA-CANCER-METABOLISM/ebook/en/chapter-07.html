By integrating cancer metabolism research platforms with clinical systems, laboratory workflows, and biobank repositories, we create pathways for discoveries to flow seamlessly from bench to bedside, benefiting all humanity.

7.1 Integration Patterns

Integration architecture fundamentally shapes how cancer metabolism research platforms communicate with external systems. The choice of integration pattern impacts scalability, fault tolerance, data consistency, and operational complexity.

Hub-and-Spoke Architecture

The hub-and-spoke pattern centralizes integration logic in a central hub that mediates all communication between systems. The cancer metabolism platform acts as the hub, with EHR, LIMS, biobank, and omics systems as spokes.

Advantages:

  • Centralized Control: All integration logic, transformation rules, and routing decisions reside in one location
  • Simplified Monitoring: Single point for logging, auditing, and performance tracking
  • Easier Governance: Access control, data quality validation, and compliance checks in central hub
  • Reduced Spoke Complexity: Individual systems require minimal integration logic

Challenges:

  • Single Point of Failure: Hub downtime affects all integrations
  • Scalability Bottleneck: Hub can become performance bottleneck with high message volumes
  • Tight Coupling: Changes to hub impact all connected systems

Peer-to-Peer Integration

Peer-to-peer patterns enable direct communication between systems without central mediation. Each system implements integration logic to communicate with relevant partners.

Advantages:

  • High Performance: Direct communication eliminates intermediary hops
  • Resilience: Failure in one connection doesn't affect others
  • Loose Coupling: Systems evolve independently

Challenges:

  • Complex Mesh: N systems require up to N(N-1)/2 connections
  • Distributed Logic: Integration rules scattered across systems
  • Difficult Governance: No central point for policy enforcement

Federated Architecture

Federated patterns combine benefits of both approaches. Systems maintain autonomy but collaborate through standardized protocols and shared governance frameworks.

WIA-CANCER-METABOLISM Recommendation: For cancer metabolism research platforms, we recommend a hybrid federated approach with a lightweight integration hub that provides message routing and protocol translation, while allowing peer-to-peer communication for high-volume data transfers. This balances governance, performance, and resilience.

7.2 EHR System Integration

Electronic Health Record (EHR) systems contain patient demographics, medical history, medications, laboratory results, and clinical notes essential for contextualizing cancer metabolism research data. Integration with EHR systems enables researchers to correlate metabolic profiles with clinical outcomes, treatment responses, and patient characteristics.

Epic Integration

Epic Systems dominates the U.S. healthcare market with approximately 31% market share. Epic provides multiple integration mechanisms:

FHIR API Integration

Epic's FHIR (Fast Healthcare Interoperability Resources) API enables standardized access to patient data. For cancer metabolism research platforms:

import axios from 'axios';
import { FHIRPatient, FHIRObservation } from '@wia/cancer-metabolism';

interface EpicFHIRConfig {
  baseUrl: string;
  clientId: string;
  clientSecret: string;
  tenantId: string;
}

class EpicFHIRConnector {
  private config: EpicFHIRConfig;
  private accessToken: string | null = null;
  private tokenExpiry: Date | null = null;

  constructor(config: EpicFHIRConfig) {
    this.config = config;
  }

  // OAuth 2.0 authentication
  private async authenticate(): Promise {
    const authUrl = `${this.config.baseUrl}/oauth2/token`;

    const response = await axios.post(authUrl, {
      grant_type: 'client_credentials',
      client_id: this.config.clientId,
      client_secret: this.config.clientSecret
    });

    this.accessToken = response.data.access_token;
    this.tokenExpiry = new Date(Date.now() + response.data.expires_in * 1000);
  }

  private async ensureAuthenticated(): Promise {
    if (!this.accessToken || !this.tokenExpiry || new Date() >= this.tokenExpiry) {
      await this.authenticate();
    }
  }

  // Fetch patient demographics
  async getPatient(patientId: string): Promise {
    await this.ensureAuthenticated();

    const response = await axios.get(
      `${this.config.baseUrl}/api/FHIR/R4/Patient/${patientId}`,
      {
        headers: {
          'Authorization': `Bearer ${this.accessToken}`,
          'Accept': 'application/fhir+json'
        }
      }
    );

    return this.transformPatient(response.data);
  }

  // Fetch laboratory observations
  async getLabResults(patientId: string, loincCodes: string[]): Promise {
    await this.ensureAuthenticated();

    const codeQuery = loincCodes.map(c => `code=${c}`).join('&');
    const url = `${this.config.baseUrl}/api/FHIR/R4/Observation?patient=${patientId}&${codeQuery}`;

    const response = await axios.get(url, {
      headers: {
        'Authorization': `Bearer ${this.accessToken}`,
        'Accept': 'application/fhir+json'
      }
    });

    return response.data.entry?.map((e: any) => this.transformObservation(e.resource)) || [];
  }

  private transformPatient(fhirPatient: any): FHIRPatient {
    return {
      id: fhirPatient.id,
      identifier: fhirPatient.identifier?.[0]?.value,
      name: {
        family: fhirPatient.name?.[0]?.family,
        given: fhirPatient.name?.[0]?.given?.[0]
      },
      gender: fhirPatient.gender,
      birthDate: fhirPatient.birthDate,
      address: fhirPatient.address?.[0]
    };
  }

  private transformObservation(fhirObs: any): FHIRObservation {
    return {
      id: fhirObs.id,
      code: fhirObs.code?.coding?.[0]?.code,
      display: fhirObs.code?.coding?.[0]?.display,
      value: fhirObs.valueQuantity?.value,
      unit: fhirObs.valueQuantity?.unit,
      effectiveDateTime: fhirObs.effectiveDateTime,
      status: fhirObs.status
    };
  }
}

HL7 v2 Integration

For legacy systems or real-time laboratory result feeds, HL7 v2 messaging remains common:

// HL7 v2 message parser for laboratory results
class HL7v2Parser {
  parseORU_R01(message: string): LabResult[] {
    const segments = message.split('\r');
    const results: LabResult[] = [];

    let currentOrder: any = null;

    for (const segment of segments) {
      const fields = segment.split('|');
      const segmentType = fields[0];

      if (segmentType === 'OBR') {
        // Observation Request - new test order
        currentOrder = {
          orderId: fields[2],
          testCode: fields[4]?.split('^')[0],
          testName: fields[4]?.split('^')[1],
          specimenCollectionTime: fields[7],
          observations: []
        };
      } else if (segmentType === 'OBX' && currentOrder) {
        // Observation Result
        currentOrder.observations.push({
          observationId: fields[3]?.split('^')[0],
          observationName: fields[3]?.split('^')[1],
          value: fields[5],
          units: fields[6],
          referenceRange: fields[7],
          abnormalFlag: fields[8],
          resultStatus: fields[11],
          observationDateTime: fields[14]
        });
      }
    }

    return results;
  }
}

class CernerFHIRConnector {
  // Cerner-specific token endpoint
  private async authenticate(): Promise {
    const authUrl = `${this.config.baseUrl}/tenants/${this.config.tenantId}/protocols/oauth2/profiles/smart-v1/token`;

      new URLSearchParams({
        grant_type: 'client_credentials',

        client_secret: this.config.clientSecret,
        scope: 'system/Patient.read system/Observation.read'
      }),

      }
    );

    this.accessToken = response.data.access_token;
  }

  // Cerner uses different FHIR endpoint structure
  async getPatient(patientId: string): Promise {
    await this.ensureAuthenticated();

    const response = await axios.get(
      `${this.config.baseUrl}/tenants/${this.config.tenantId}/instances/${this.config.instanceId}/Patient/${patientId}`,
      {
        headers: {
          'Authorization': `Bearer ${this.accessToken}`,
          'Accept': 'application/fhir+json'
        }
      }
    );

    return this.transformPatient(response.data);
  }
}

7.3 LIMS Integration

Laboratory Information Management Systems (LIMS) manage sample tracking, assay workflows, quality control, and result reporting. Integration with LIMS enables automated data flow from laboratory instruments to cancer metabolism research platforms.

Common LIMS Platforms

Cancer metabolism research typically involves these LIMS platforms:

  • LabWare LIMS: Enterprise-scale LIMS with robust workflow automation
  • Thermo Fisher SampleManager: Industry-standard for pharmaceutical and clinical laboratories
  • LabVantage: Cloud-native LIMS with extensive integration capabilities
  • STARLIMS: Highly configurable for complex research workflows
  • Custom Laboratory Systems: Institution-specific platforms

LIMS Integration Architecture

import { EventEmitter } from 'events';
import { SampleMetadata, AssayResult } from '@wia/cancer-metabolism';

interface LIMSConfig {
  apiUrl: string;
  apiKey: string;
  labId: string;
  pollingInterval: number; // milliseconds
}

class LIMSConnector extends EventEmitter {
  private config: LIMSConfig;
  private pollingTimer: NodeJS.Timer | null = null;
  private lastSyncTimestamp: Date | null = null;

  constructor(config: LIMSConfig) {
    super();
    this.config = config;
  }

  // Start continuous polling for new results
  startPolling(): void {
    if (this.pollingTimer) {
      return; // Already polling
    }

    this.pollingTimer = setInterval(
      async () => await this.pollForNewResults(),
      this.config.pollingInterval
    );

    // Immediate poll
    this.pollForNewResults();
  }

  stopPolling(): void {
    if (this.pollingTimer) {
      clearInterval(this.pollingTimer);
      this.pollingTimer = null;
    }
  }

  private async pollForNewResults(): Promise {
    try {
      const since = this.lastSyncTimestamp || new Date(Date.now() - 24 * 60 * 60 * 1000);
      const results = await this.fetchResultsSince(since);

      for (const result of results) {
        this.emit('assayResult', result);
      }

      this.lastSyncTimestamp = new Date();
      this.emit('syncComplete', { resultCount: results.length });
    } catch (error) {
      this.emit('error', error);
    }
  }

  private async fetchResultsSince(since: Date): Promise {
    const response = await fetch(
      `${this.config.apiUrl}/results?labId=${this.config.labId}&since=${since.toISOString()}`,
      {
        headers: {
          'X-API-Key': this.config.apiKey,
          'Accept': 'application/json'
        }
      }
    );

    if (!response.ok) {
      throw new Error(`LIMS API error: ${response.status}`);
    }

    const data = await response.json();
    return data.results.map((r: any) => this.transformResult(r));
  }

  // Submit sample metadata to LIMS
  async registerSample(sample: SampleMetadata): Promise {
    const response = await fetch(
      `${this.config.apiUrl}/samples`,
      {
        method: 'POST',
        headers: {
          'X-API-Key': this.config.apiKey,
          'Content-Type': 'application/json'
        },
        body: JSON.stringify({
          sampleId: sample.sampleId,
          patientId: sample.patientId,
          sampleType: sample.sampleType,
          collectionDate: sample.collectionDate,
          storageLocation: sample.storageLocation,
          requestedAssays: sample.requestedAssays
        })
      }
    );

    if (!response.ok) {
      throw new Error(`Failed to register sample: ${response.status}`);
    }

    const data = await response.json();
    return data.limsAccessionNumber;
  }

  // Query sample status
  async getSampleStatus(accessionNumber: string): Promise {
    const response = await fetch(
      `${this.config.apiUrl}/samples/${accessionNumber}/status`,
      {
        headers: {
          'X-API-Key': this.config.apiKey,
          'Accept': 'application/json'
        }
      }
    );

    if (!response.ok) {
      throw new Error(`Failed to get sample status: ${response.status}`);
    }

    return await response.json();
  }

  private transformResult(limsResult: any): AssayResult {
    return {
      accessionNumber: limsResult.accessionNumber,
      sampleId: limsResult.sampleId,
      assayType: limsResult.testCode,
      assayName: limsResult.testName,
      resultDate: new Date(limsResult.resultDateTime),
      analytes: limsResult.analytes.map((a: any) => ({
        name: a.analyteName,
        value: a.value,
        unit: a.unit,
        referenceRange: a.referenceRange,
        flag: a.abnormalFlag
      })),
      qcStatus: limsResult.qcStatus,
      instrument: limsResult.instrumentId,
      technician: limsResult.technicianId
    };
  }
}

// Usage
const limsConnector = new LIMSConnector({
  apiUrl: 'https://lims.hospital.org/api/v1',
  apiKey: 'your-api-key',
  labId: 'LAB-001',
  pollingInterval: 300000 // 5 minutes
});

limsConnector.on('assayResult', (result) => {
  console.log('New assay result:', result);
  // Process and store in cancer metabolism platform
});

limsConnector.on('error', (error) => {
  console.error('LIMS integration error:', error);
});

limsConnector.startPolling();

Bidirectional LIMS Integration

Effective integration requires bidirectional communication:

  • To LIMS: Sample registration, test orders, quality control notifications
  • From LIMS: Assay results, sample status updates, instrument data, QC metrics
Data Quality Consideration: LIMS data often contains incomplete records, data entry errors, and inconsistent nomenclature. Implement robust validation and normalization logic. For metabolomics data, standardize to HMDB identifiers; for proteomics, use UniProt accessions.

7.4 Biobank Registry Connection

Biobank registries track biological sample storage, chain of custody, freeze-thaw cycles, and availability for research. Integration with biobank systems enables researchers to link metabolic profiles with physical samples, track sample quality over time, and coordinate sample retrieval for validation studies.

Biobank Integration Connector

class BiobankConnector {
  private apiUrl: string;
  private apiKey: string;

  constructor(apiUrl: string, apiKey: string) {
    this.apiUrl = apiUrl;
    this.apiKey = apiKey;
  }

  // Query available samples matching criteria
  async searchSamples(criteria: BiobankSearchCriteria): Promise {
    const response = await fetch(
      `${this.apiUrl}/samples/search`,
      {
        method: 'POST',
        headers: {
          'Authorization': `Bearer ${this.apiKey}`,
          'Content-Type': 'application/json'
        },
        body: JSON.stringify(criteria)
      }
    );

    if (!response.ok) {
      throw new Error(`Biobank search failed: ${response.status}`);
    }

    return await response.json();
  }

  // Reserve sample for analysis
  async reserveSample(sampleId: string, studyId: string, volumeNeeded: number): Promise {
    const response = await fetch(
      `${this.apiUrl}/samples/${sampleId}/reserve`,
      {
        method: 'POST',
        headers: {
          'Authorization': `Bearer ${this.apiKey}`,
          'Content-Type': 'application/json'
        },
        body: JSON.stringify({
          studyId,
          volumeNeeded,
          expirationDate: new Date(Date.now() + 14 * 24 * 60 * 60 * 1000) // 14 days
        })
      }
    );

    if (!response.ok) {
      throw new Error(`Sample reservation failed: ${response.status}`);
    }

    return await response.json();
  }

  // Record sample usage
  async recordUsage(sampleId: string, usage: SampleUsage): Promise {
    const response = await fetch(
      `${this.apiUrl}/samples/${sampleId}/usage`,
      {
        method: 'POST',
        headers: {
          'Authorization': `Bearer ${this.apiKey}`,
          'Content-Type': 'application/json'
        },
        body: JSON.stringify(usage)
      }
    );

    if (!response.ok) {
      throw new Error(`Failed to record usage: ${response.status}`);
    }
  }
}

Multi-omics Data Integration

class MultiOmicsIntegrator {
  private genomicsConnector: GenomicsPlatformConnector;
  private proteomicsConnector: ProteomicsPlatformConnector;
  private metabolomicsConnector: MetabolomicsPlatformConnector;

  async integratePatientData(patientId: string): Promise {
    // Fetch all omics layers in parallel
    const [variants, proteins, metabolites] = await Promise.all([
      this.genomicsConnector.getVariants(patientId),
      this.proteomicsConnector.getProteinExpression(patientId),
      this.metabolomicsConnector.getMetaboliteProfile(patientId)
    ]);

    // Map genomic variants to affected metabolic pathways
    const variantPathwayMap = this.mapVariantsToPathways(variants);

    // Correlate protein expression with metabolite levels
    const proteinMetaboliteCorrelations = this.correlateProteinMetabolite(
      proteins,
      metabolites
    );

    // Identify pathway dysregulation
    const dysregulatedPathways = this.identifyDysregulatedPathways(
      variants,
      proteins,
      metabolites
    );

    return {
      patientId,
      genomics: { variants, pathwayImpact: variantPathwayMap },
      proteomics: { proteins, enzymeActivity: this.estimateEnzymeActivity(proteins) },
      metabolomics: { metabolites, fluxEstimates: this.estimateFlux(metabolites) },
      correlations: proteinMetaboliteCorrelations,
      pathwayAnalysis: dysregulatedPathways,
      generatedAt: new Date()
    };
  }

  private mapVariantsToPathways(variants: GenomicVariant[]): Map {
    const pathwayMap = new Map();

    for (const variant of variants) {
      const pathways = this.geneToPathwayMap.get(variant.gene) || [];
      pathwayMap.set(variant.gene, pathways);
    }

    return pathwayMap;
  }

  private correlateProteinMetabolite(
    proteins: ProteinExpression[],
    metabolites: MetaboliteConcentration[]
  ): Correlation[] {
    const correlations: Correlation[] = [];

    // For each enzyme, correlate with its substrate/product metabolites
    for (const protein of proteins) {
      const relatedMetabolites = this.getEnzymeSubstratesProducts(protein.geneName);

      for (const metabolite of metabolites) {
        if (relatedMetabolites.includes(metabolite.hmdbId)) {
          const correlation = this.calculateCorrelation(
            protein.abundance,
            metabolite.concentration
          );

          correlations.push({
            protein: protein.geneName,
            metabolite: metabolite.hmdbId,
            correlation,
            biologicalRelationship: 'enzyme-substrate'
          });
        }
      }
    }

    return correlations;
  }
}

// Middleware to track request duration
app.use((req, res, next) => {
  const start = Date.now();

  res.on('finish', () => {
    const duration = (Date.now() - start) / 1000;
    httpRequestDuration.observe(
      {
        method: req.method,
        route: req.route?.path || req.path,
        status_code: res.statusCode
      },
      duration
    );
  });

  next();
});

// Track sample processing
async function processSample(sample: Sample): Promise {
  const startTime = Date.now();

  try {
    await performProcessing(sample);

    const duration = (Date.now() - startTime) / 1000;
    sampleProcessingDuration.observe(
      {
        sample_type: sample.type,
        assay_type: sample.assayType
      },
      duration
    );
  } catch (error) {
    dataProcessingErrors.inc({
      pipeline: 'sample_processing',
      error_type: error.constructor.name
    });
    throw error;
  }
}

Data Quality Monitoring

class DataQualityMonitor {
  async assessDataQuality(dataset: Dataset): Promise {
    const metrics = {
      completeness: await this.calculateCompleteness(dataset),
      accuracy: await this.calculateAccuracy(dataset),
      consistency: await this.calculateConsistency(dataset),
      timeliness: await this.calculateTimeliness(dataset),
      validity: await this.calculateValidity(dataset)
    };

    const overallScore = this.calculateOverallScore(metrics);

    return {
      datasetId: dataset.id,
      assessmentDate: new Date(),
      metrics,
      overallScore,
      issues: await this.identifyIssues(metrics),
      recommendations: await this.generateRecommendations(metrics)
    };
  }

  private async calculateCompleteness(dataset: Dataset): Promise {
    const totalFields = dataset.schema.requiredFields.length;
    const completeRecords = await this.countCompleteRecords(dataset);
    const totalRecords = dataset.recordCount;

    return (completeRecords / totalRecords) * 100;
  }

  private async calculateAccuracy(dataset: Dataset): Promise {
    // Compare against reference standards, range checks
    const validatedFields = await this.validateAgainstRules(dataset);
    const totalFields = dataset.recordCount * dataset.schema.fields.length;

    return (validatedFields / totalFields) * 100;
  }

  private async identifyIssues(metrics: QualityMetrics): Promise {
    const issues: QualityIssue[] = [];

    if (metrics.completeness < 95) {
      issues.push({
        severity: 'high',
        type: 'completeness',
        description: 'Data completeness below threshold',
        affectedRecords: await this.getIncompleteRecords()
      });
    }

    if (metrics.accuracy < 90) {
      issues.push({
        severity: 'critical',
        type: 'accuracy',
        description: 'Data accuracy below acceptable levels',
        affectedRecords: await this.getInaccurateRecords()
      });
    }

    return issues;
  }
}

7.8 Chapter Summary

Key Takeaways

  1. Integration Architecture Matters: The choice between hub-and-spoke, peer-to-peer, and federated patterns significantly impacts scalability, governance, and operational complexity. A hybrid approach often provides the best balance for cancer metabolism research platforms.
  2. EHR Integration Enables Clinical Context: Connecting with Epic, Cerner, and other EHR systems through FHIR APIs and HL7 messaging brings essential clinical context to metabolic research data, enabling correlation of metabolic profiles with patient outcomes and treatment responses.
  3. LIMS Integration Automates Data Flow: Bidirectional integration with Laboratory Information Management Systems eliminates manual data entry, reduces errors, and accelerates the path from sample collection to analyzed results while maintaining complete audit trails.
  4. Multi-omics Integration Reveals Biological Mechanisms: Integrating metabolomics with genomics, transcriptomics, and proteomics platforms provides comprehensive molecular portraits that reveal mechanistic relationships between genetic variations, enzyme expression, and metabolic phenotypes.
  5. Cloud Deployment Enables Scale and Collaboration: Modern cloud platforms (AWS, Azure, GCP) provide HIPAA-compliant infrastructure, specialized healthcare APIs, and scalable computing resources that enable multi-site collaboration and handle growing data volumes while maintaining security and regulatory compliance.
