2장: 시스템 아키텍처

2.1 아키텍처 개요

WIA-AAC 표준은 관심사를 분리하고 상호운용성을 유지하면서 유연성을 가능하게 하는 계층화된 아키텍처를 정의합니다. 이 아키텍처는 모든 부분이 원활하게 함께 작동하도록 보장하면서 다른 구성요소를 독립적으로 개발, 업데이트 및 교체할 수 있게 합니다.

아키텍처는 각 계층이 명확하게 정의된 책임과 인터페이스를 갖는 모듈식 설계 철학을 따릅니다. 이 접근 방식은 제조업체가 더 넓은 AAC 생태계와의 호환성을 보장하면서 특정 구성요소에 전문화할 수 있게 합니다.

설계 원칙: WIA-AAC 아키텍처는 사용자 연속성을 우선시합니다. 사용자는 어휘, 사용자화 또는 의사소통 기록을 잃지 않고 기기, 앱 또는 서비스를 전환할 수 있어야 합니다.

2.2 5계층 모델

WIA-AAC 시스템은 5개의 개별 계층으로 구성됩니다:

계층 이름 책임
5 애플리케이션 계층 사용자 인터페이스, 의사소통 디스플레이, 설정
4 의사소통 엔진 계층 메시지 구성, 예측, 언어 처리
3 콘텐츠 계층 상징, 어휘, 음성 데이터, 사용자 프로필
2 접근 계층 입력 방법, 선택 기술, 보정
1 플랫폼 계층 하드웨어, OS, 기기 기능 
// WIA-AAC 계층 아키텍처 구현
interface WIAAACSystem {
    // 계층 1: 플랫폼
    platform: {
        deviceType: 'dedicated' | 'tablet' | 'phone' | 'computer';
        operatingSystem: string;
        screenSize: { width: number; height: number };
        capabilities: DeviceCapabilities;
    };

    // 계층 2: 접근
    accessManager: {
        availableMethods: AccessMethod[];
        activeMethod: AccessMethod;
        calibrationData: CalibrationProfile;
    };

    // 계층 3: 콘텐츠
    contentRepository: {
        symbolSets: SymbolSet[];
        vocabularies: VocabularySet[];
        userProfiles: UserProfile[];
        voiceData: VoiceBank[];
    };

    // 계층 4: 의사소통 엔진
    communicationEngine: {
        messageComposer: MessageComposer;
        predictor: PredictionEngine;
        languageProcessor: LanguageProcessor;
    };

    // 계층 5: 애플리케이션
    application: {
        displays: CommunicationDisplay[];
        settings: SystemSettings;
        uiTheme: UITheme;
    };
}

2.3 계층 1: 플랫폼 계층

플랫폼 계층은 AAC 시스템이 실행되는 물리적 하드웨어와 운영 체제를 나타냅니다. WIA-AAC는 플랫폼에 구애받지 않도록 설계되어 다음을 지원합니다:

2.3.1 전용 AAC 기기

AAC 사용에 최적화된 목적 제작 하드웨어로, 다음을 특징으로 합니다:

  • 내구성을 위한 견고한 구조
  • 음성 출력을 위한 고품질 스피커
  • 하루 종일 사용할 수 있는 긴 배터리 수명
  • 통합 장착 옵션
  • 환경 제어 통합

2.3.2 소비자 기기

AAC 애플리케이션을 실행하는 주류 태블릿, 전화 및 컴퓨터:

  • iOS 기기 (iPad, iPhone)
  • Android 태블릿 및 전화
  • Windows 컴퓨터
  • macOS 기기
  • Chrome OS 기기

2.3.3 플랫폼 추상화

// 플랫폼 기능 감지
class PlatformAbstraction {
    static detectCapabilities(): DeviceCapabilities {
        return {
            touchScreen: this.hasTouchScreen(),
            eyeTracker: this.hasEyeTracker(),
            switchPorts: this.detectSwitchPorts(),
            speakers: this.detectAudioOutput(),
            microphone: this.hasMicrophone(),
            camera: this.hasCamera(),
            bluetooth: this.hasBluetooth(),
            wifi: this.hasWifi(),
            cellularData: this.hasCellular(),
            gps: this.hasGPS(),
            accelerometer: this.hasAccelerometer()
        };
    }

    static getOptimalConfiguration(): AccessRecommendation {
        const caps = this.detectCapabilities();
        // 사용 가능한 하드웨어를 기반으로 접근 방법 추천
        return this.analyzeCapabilities(caps);
    }
}

2.4 계층 2: 접근 계층

접근 계층은 사용자가 AAC 시스템과 물리적으로 상호작용하는 방법을 관리합니다. 다양한 입력 방법을 상위 계층이 특정 입력 메커니즘을 알 필요 없이 사용할 수 있는 통합 인터페이스로 추상화합니다.

2.4.1 직접 선택 방법

  • 터치: 터치스크린과 손가락 또는 스타일러스 접촉
  • 시선 추적: 시선 추적 카메라로 감지된 시각적 고정
  • 머리 추적: 커서 위치를 제어하는 머리 움직임
  • 마우스/조이스틱: 전통적인 포인팅 장치

2.4.2 스캐닝 방법

  • 자동 스캐닝: 항목이 일정 간격으로 순차적으로 강조 표시됨
  • 단계 스캐닝: 사용자가 항목 간 진행을 제어
  • 그룹-항목 스캐닝: 계층적 선택으로 필요한 선택 수 감소
  • 역 스캐닝: 스위치를 누르는 동안 항목이 순환됨; 놓아서 선택

2.4.3 접근 방법 인터페이스

// 통합 접근 방법 인터페이스
interface AccessMethod {
    readonly type: 'direct' | 'scanning' | 'encoding';
    readonly name: string;

    initialize(): Promise<void>;
    calibrate(): Promise<CalibrationResult>;

    // 통합 선택 이벤트
    onSelection(callback: (target: SelectionTarget) => void): void;

    // 직접 선택 방법용
    onPointerMove?(callback: (position: Point) => void): void;

    // 스캐닝 방법용
    onScanProgress?(callback: (current: ScanPosition) => void): void;

    configure(settings: AccessSettings): void;
    getStatus(): AccessStatus;
    dispose(): void;
}

// 예: 시선 추적 구현
class EyeGazeAccess implements AccessMethod {
    readonly type = 'direct';
    readonly name = '시선 추적';

    private tracker: EyeTracker;
    private dwellTime: number = 800; // ms

    async initialize() {
        this.tracker = await EyeTracker.connect();
        await this.tracker.startTracking();
    }

    async calibrate() {
        return await this.tracker.runCalibration({
            points: 9,
            animation: 'shrinking-circle'
        });
    }

    configure(settings: AccessSettings) {
        this.dwellTime = settings.dwellTime ?? 800;
        this.tracker.setSmoothing(settings.smoothing ?? 'medium');
    }
}
접근성 요구사항: 모든 WIA-AAC 시스템은 사용자가 주요 방법을 사용할 수 없게 될 경우 대안을 보장하기 위해 최소 두 가지 별개의 접근 방법을 지원해야 합니다.

2.5 계층 3: 콘텐츠 계층

콘텐츠 계층은 AAC 시스템에서 사용하는 모든 언어적 및 개인화 리소스를 관리합니다:

2.5.1 상징 관리

상징은 다음을 포함하는 표준화된 패키지에 저장됩니다:

  • 여러 해상도의 벡터 및 래스터 이미지
  • 상징을 개념에 연결하는 의미적 메타데이터
  • 다국어 레이블 및 설명
  • 사용 통계 및 빈도 데이터
  • 라이선스 및 저작자 정보

사용자 프로필에는 개인화된 설정과 데이터가 포함됩니다:

2.6 계층 4: 의사소통 엔진 계층

의사소통 엔진은 사용자 선택을 의미 있는 의사소통으로 변환하는 지능적인 핵심입니다:

2.6.1 메시지 구성

메시지 작성기는 발화 구성을 관리합니다:

  • 상징을 구문으로 연결
  • 메시지 버퍼 및 기록 관리
  • 문법 및 형태론 처리
  • 여러 구성 전략 지원

2.6.2 예측 엔진

// 예측 엔진 인터페이스
interface PredictionEngine {

    // 사용자 패턴에서 학습

2.7 계층 5: 애플리케이션 계층

애플리케이션 계층은 사용자 대면 인터페이스를 제공합니다:

2.7.1 의사소통 디스플레이

  • 하이브리드 디스플레이: 여러 디스플레이 유형의 조합

2.8 계층 간 통신

계층은 잘 정의된 인터페이스와 이벤트를 통해 통신합니다:

2.8.1 이벤트 흐름

일반적인 선택은 다음과 같이 시스템을 통해 흐릅니다:

  1. 플랫폼 계층이 원시 입력 감지 (터치, 스위치 누름, 시선 고정)
  2. 접근 계층이 입력을 해석하고 선택된 대상 식별
  3. 애플리케이션 계층이 시각적 피드백 업데이트
  4. 의사소통 엔진이 선택 처리
  5. 콘텐츠 계층이 상징 및 어휘 데이터 제공
  6. 의사소통 엔진이 메시지 및 예측 업데이트
  7. 애플리케이션 계층이 업데이트된 디스플레이 렌더링 및 구성된 경우 발화

2.8.2 이벤트 버스 아키텍처

// 계층 간 통신을 위한 중앙 이벤트 버스
class AACEventBus {
    private listeners: Map<string, Set<EventHandler>> = new Map();

    emit(event: AACEvent): void {
        const handlers = this.listeners.get(event.type);
        if (handlers) {
            handlers.forEach(handler => handler(event));
        }
    }

    on(eventType: string, handler: EventHandler): Unsubscribe {
        if (!this.listeners.has(eventType)) {
            this.listeners.set(eventType, new Set());
        }
        this.listeners.get(eventType)!.add(handler);
        return () => this.listeners.get(eventType)?.delete(handler);
    }
}

// 이벤트 유형
type AACEvent =
    | { type: 'selection'; target: SelectionTarget }
    | { type: 'message-update'; message: Message }
    | { type: 'speak'; text: string }
    | { type: 'prediction-update'; predictions: Prediction[] }
    | { type: 'page-change'; page: VocabularyPage }
    | { type: 'access-change'; method: AccessMethod };

2.9 확장성 포인트

WIA-AAC 아키텍처는 여러 확장성 포인트를 제공합니다:

  • 사용자 정의 접근 방법: 새로운 입력 기술이 AccessMethod 인터페이스를 구현할 수 있음
  • 상징 세트 플러그인: 콘텐츠 계층을 통해 추가 상징 시스템을 추가할 수 있음
  • 예측 엔진: 대체 예측 알고리즘을 교체할 수 있음
  • 음성 엔진: 다른 TTS 엔진을 통합할 수 있음
  • 디스플레이 템플릿: 사용자 정의 UI 레이아웃을 만들 수 있음

이 계층화되고 확장 가능한 아키텍처는 WIA-AAC 시스템이 호환성과 사용자 연속성을 유지하면서 발전하는 기술과 함께 진화할 수 있도록 보장합니다.
