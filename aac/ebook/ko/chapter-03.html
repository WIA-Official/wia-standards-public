3장: 상징 시스템과 어휘

3.1 AAC 상징 소개

상징은 AAC 시스템에서 시각적 의사소통의 기본 구성 요소입니다. 상징은 사고와 표현 사이의 다리 역할을 하여 사용자가 개념의 표현을 선택하여 메시지를 구성할 수 있게 합니다. WIA-AAC 표준은 상징 표현, 조직 및 상호운용성에 대한 포괄적인 사양을 수립합니다.

효과적인 상징 시스템은 여러 경쟁 요구사항 간의 균형을 유지해야 합니다: 새 사용자가 배울 수 있어야 하고, 숙련된 사용자에게 효율적이어야 하며, 어떤 아이디어도 표현할 수 있을 만큼 포괄적이어야 하고, 개인의 요구와 문화적 맥락에 적응할 수 있을 만큼 유연해야 합니다.

지침 원칙: 상징은 인간 표현을 위한 도구이지 제약이 아닙니다. AAC 시스템은 완전한 표현의 자유를 위해 사용자가 항상 글자별 철자에 접근할 수 있도록 보장하면서 견고한 상징 지원을 제공해야 합니다.

3.2 상징 시스템의 유형

WIA-AAC 표준은 여러 범주의 상징을 지원합니다:

3.2.1 그림 문자 상징

그림 문자 상징은 시각적으로 그것이 나타내는 것과 유사합니다. 새 사용자와 구체적인 개념에 가장 직관적입니다:

  • PCS (그림 의사소통 상징): 일관된 스타일로 널리 사용되는 상용 상징 세트
  • SymbolStix: 다양한 표현을 가진 현대적인 막대 인물 기반 상징
  • ARASAAC: 여러 언어로 제공되는 오픈소스 상징 세트
  • 멀베리 상징: 성인에게 적합한 이미지를 가진 무료 상징 세트

3.2.2 표의 상징

표의 상징은 추상적인 시각적 관계를 통해 아이디어나 개념을 나타냅니다:

  • 블리스상징: 기본 요소에서 의미가 구성되는 의미론적 쓰기 시스템
  • Minspeak/Unity: 문맥이 특정 의미를 결정하는 다중 의미 아이콘

3.2.3 철자 상징

문자 언어를 사용하는 텍스트 기반 상징:

  • 문자 단어와 구문
  • 약어와 두문자어
  • 철자를 위한 알파벳 글자

3.2.4 사진 상징

상징으로 사용되는 실제 사진, 특히 다음에 유용:

  • 개인 어휘 (가족 구성원, 반려동물, 개인 물품)
  • 위치 특정 어휘 (지역 장소, 학교, 직장)
  • 추상적 표현에 어려움이 있는 사용자

3.3 상징 데이터 형식

WIA-AAC 표준은 포괄적인 상징 데이터 형식을 정의합니다:

// WIA-AAC 상징 정의 스키마
interface Symbol {
    // 고유 식별자
    id: string;
    symbolSet: string;

    // 시각적 표현
    images: {
        svg?: string;         // 벡터 형식 선호
        png?: {
            size: number;     // 픽셀 단위 너비/높이
            data: string;     // Base64 인코딩
        }[];
        highContrast?: string;  // 시각 장애용 대안
    };

    // 의미적 정보
    semantics: {
        conceptId: string;    // 개념 온톨로지 참조
        partOfSpeech: PartOfSpeech;
        semanticCategory: string[];
        relatedConcepts: string[];
    };

    // 언어적 레이블
    labels: {
        language: string;
        primary: string;      // 주요 표시 레이블
        spoken: string;       // TTS용 텍스트 (표시와 다를 수 있음)
        alternates: string[]; // 동의어 및 변형
        description: string;  // 화면 판독기 및 툴팁용
    }[];

    // 사용 메타데이터
    metadata: {
        frequency: number;    // 코퍼스 빈도 순위
        ageAppropriate: AgeRange;
        culturalNotes?: string[];
        license: LicenseInfo;
    };
}

3.5 어휘 조직 원칙

상징이 조직되는 방식은 의사소통 효율성에 크게 영향을 미칩니다. WIA-AAC 표준은 여러 조직 전략을 지원합니다:

3.5.1 의미-구문 그리드 조직

상징은 의미(의미론)와 문법적 기능(구문)에 따라 배열됩니다:

  • 단어 범주(동사, 명사, 형용사)에 대한 일관된 위치
  • 품사별 색상 코딩 (수정된 Fitzgerald 키)
  • 핵심 어휘의 두드러진 배치
  • 주제별로 조직된 주변 어휘

3.5.2 PODD 조직

Pragmatic Organization Dynamic Display는 의사소통 기능에 따라 어휘를 배열합니다:

  • 고빈도 사회적 구문에 대한 빠른 접근
  • 상세 주제로의 분기 탐색
  • 항상 사용 가능한 탐색 및 핵심 단어
  • 효율성을 위한 원히트 구문

3.5.3 시각적 장면 디스플레이

맥락적 사진이나 일러스트레이션에 포함된 어휘:

  • 장면 내 의미 있는 요소에 대한 핫스팟
  • 자연스러운 맥락이 이해를 지원
  • 초기 의사소통자에게 특히 효과적
  • 그리드 기반 페이지와 결합 가능

3.6 어휘 데이터 구조

// WIA-AAC 어휘 구조
interface VocabularySet {
    id: string;
    name: string;
    version: string;
    description: string;

    organization: VocabularyOrganization;
    symbolSet: string;

    // 페이지 계층

    settings: {
        gridSize: GridSize;
        colorCoding: ColorScheme;
        navigationStyle: NavigationStyle;
    };

    // 사용자화 추적
    customizations: {
        addedSymbols: Symbol[];
        modifiedCells: CellModification[];
        addedPages: VocabularyPage[];
        hiddenItems: string[];
    };
}

interface VocabularyPage {
    id: string;
    name: string;
    type: PageType;

    // 그리드 구성
    rows: number;
    columns: number;
    cells: Cell[][];

    // 시각적 장면 구성 (해당하는 경우)
    backgroundImage?: string;
    hotspots?: Hotspot[];

    // 탐색
    parentPageId?: string;
    linkedPages: string[];

    behavior: CellBehavior;

    | { type: 'symbol'; symbolId: string; label?: string }

3.7 핵심 어휘 구현

핵심 어휘는 효과적인 AAC에 필수적입니다. WIA-AAC 표준은 핵심 단어 지원에 대한 요구사항을 정의합니다:

3.7.1 필수 핵심 단어

모든 WIA-AAC 호환 시스템은 다음 기본 단어에 대한 상징을 포함해야 합니다:

범주 예시 개수
대명사 나, 너, 그, 그녀, 그것, 우리, 그들, 내, 나의 ~15
동사 원하다, 가다, 얻다, 만들다, 돕다, 좋아하다, 필요하다, 보다, 갖다 ~50
수식어 더, 모든, 다른, 같은, 큰, 작은, 좋은, 나쁜 ~40
전치사 안에, 위에, ~에게, ~을 위해, ~와 함께, ~에 대해, 위로, 아래로 ~20
질문 무엇, 어디, 누구, 언제, 왜, 어떻게 ~10
사회적 예, 아니오, 제발, 감사합니다, 미안해요, 안녕하세요, 안녕히 가세요 ~15

3.7.2 핵심 단어 접근성

// 핵심 어휘는 지정된 선택 수 내에서 접근 가능해야 함
const coreAccessibilityRequirements = {
    // 1-2회 선택으로 접근 가능한 고빈도 단어
    tier1: {
        maxSelections: 2,
        words: ['나', '원해', '더', '도와줘', '멈춰', '가자', '응', '아니']
    },

    // 2-3회 선택으로 접근 가능한 중빈도 핵심
    tier2: {
        maxSelections: 3,
        words: ['너', '그', '그녀', '그것', '좋아', '필요해', '있어', '만들어',
                '얻어', '봐', '좋은', '나쁜', '큰', '작은', '뭐', '어디']
    },

    // 최대 4회 선택으로 접근 가능한 전체 핵심 어휘
    tier3: {
        maxSelections: 4,
        coverage: 'all-core-words'
    }
};

// 검증 함수
function validateCoreAccess(vocabulary: VocabularySet): ValidationResult {
    const results: ValidationResult = { passed: true, issues: [] };

    for (const [tier, req] of Object.entries(coreAccessibilityRequirements)) {
        for (const word of req.words || []) {
            const selections = countSelectionsToReach(vocabulary, word);
            if (selections > req.maxSelections) {
                results.passed = false;
                results.issues.push({
                    severity: 'error',
                    message: `핵심 단어 "${word}"는 ${selections}회 선택이 필요하며, 최대는 ${req.maxSelections}입니다`
                });
            }
        }
    }

    return results;
}
의사소통 효율성: 연구에 따르면 핵심 어휘가 일상적인 의사소통의 80%를 차지합니다. WIA-AAC 시스템은 특정 주제와 개인적 표현을 위한 포괄적인 주변 어휘를 제공하면서도 이러한 단어에 대한 빠른 접근을 우선시합니다.

3.8 주변 어휘 관리

주변 어휘는 개인적 표현에 필수적인 개인화를 제공합니다:

3.8.1 카테고리 조직

주변 어휘는 의미적 카테고리로 조직됩니다:

  • 사람 (가족, 친구, 전문가)
  • 장소 (집, 학교, 지역사회)
  • 활동 (취미, 스포츠, 일과)
  • 음식과 음료
  • 동물
  • 의복과 개인 물품
  • 감정과 느낌
  • 시간과 달력
  • 건강과 신체
  • 기술

3.8.2 개인 어휘 API

// 개인 주변 어휘 관리를 위한 API
class PersonalVocabulary {
    private userId: string;
    private customSymbols: Map<string, Symbol>;

    // 개인 상징 추가 (예: 가족 구성원 사진)
    async addPersonalSymbol(config: {
        name: string;
        image: File | string;
        category: string;
        spokenText?: string;
    }): Promise<Symbol> {
        const symbol = await this.createSymbol(config);
        this.customSymbols.set(symbol.id, symbol);
        await this.syncToCloud();
        return symbol;
    }

    // 다른 기기에서 상징 가져오기
    async importSymbols(source: SymbolExport): Promise<ImportResult> {
        const imported: Symbol[] = [];
        const conflicts: ConflictInfo[] = [];

        for (const symbol of source.symbols) {
            if (this.customSymbols.has(symbol.id)) {
                conflicts.push({ existing: this.customSymbols.get(symbol.id)!, incoming: symbol });
            } else {
                this.customSymbols.set(symbol.id, symbol);
                imported.push(symbol);
            }
        }

        return { imported, conflicts };
    }

    // 백업 또는 전송을 위한 내보내기
    async exportSymbols(): Promise<SymbolExport> {
        return {
            userId: this.userId,
            exportDate: new Date(),
            symbols: Array.from(this.customSymbols.values()),
            format: 'wia-aac-v1'
        };
    }
}

3.9 다국어 지원

WIA-AAC 표준은 포괄적인 다국어 지원을 요구합니다:

3.9.1 상징 현지화

  • 여러 언어로 제공되는 레이블
  • 적절한 경우 문화적 적응
  • 오른쪽에서 왼쪽 언어 지원
  • 모든 스크립트에 대한 문자 세트 지원

3.10 상징 사용자화

사용자와 지원 팀은 시스템 무결성을 유지하면서 상징을 사용자화할 수 있습니다:

  • 레이블 수정: 상징 이미지를 변경하지 않고 표시된 텍스트 변경
  • 이미지 교체: 개인 사진이나 사용자 정의 그림으로 대체
  • 색상 조정: 가시성이나 선호도를 위해 상징 색상 수정
  • 크기 조정: 그리드 내에서 개별 상징 크기 조정
  • 숨겨진 상징: 연령에 부적합하거나 관련 없는 상징 숨기기

모든 사용자화는 사용자 프로필에 저장되고 내보낼 수 있어, 기기와 애플리케이션 간에 개인화가 유지됩니다.
