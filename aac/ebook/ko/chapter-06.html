6장: 통합과 상호운용성

6.1 연결된 AAC 생태계

현대 AAC 시스템은 독립적으로 작동하지 않습니다. 스마트 기기, 의사소통 플랫폼, 생산성 도구 및 환경 제어 시스템을 포함한 더 넓은 기술 생태계와 원활하게 통합되어야 합니다. WIA-AAC 표준은 보안과 사용자 제어를 유지하면서 이러한 통합을 가능하게 하는 프로토콜과 API를 정의합니다.

통합은 여러 목적을 수행합니다: AAC 사용자의 의사소통 범위를 확장하고, 디지털 생활에 참여할 수 있게 하며, 환경 제어를 통해 독립성을 제공하고, 사용자의 요구가 변화함에 따라 AAC 데이터가 시스템 간에 자유롭게 이동할 수 있도록 보장합니다.

상호운용성 의무: WIA-AAC 호환 시스템은 표준화된 데이터 교환 형식과 API를 지원해야 합니다. 사용자는 단일 벤더의 생태계에 종속되어서는 안 됩니다. 데이터 이동성은 기본권입니다.

6.2 사용자 프로필 이동성

사용자 프로필에는 상당한 시간 투자를 나타내는 개인화된 설정이 포함됩니다. 이는 이동 가능해야 합니다:

6.2.1 프로필 내보내기 형식

// WIA-AAC 사용자 프로필 교환 형식
interface UserProfileExport {
    format: 'wia-aac-profile';
    version: '1.0';
    exportDate: string;
    exportedBy: string;  // 애플리케이션 식별자

    // 핵심 사용자 데이터
    user: {
        id: string;
        displayName: string;
        languages: string[];
    };

    // 접근 구성
    access: {
        methods: AccessMethodConfig[];
        activeMethod: string;
        calibrationData: Map<string, CalibrationData>;
    };

    // 음성 선호도
    voice: {
        selectedVoice: string;
        settings: VoiceSettings;
        pronunciationDictionary: PronunciationEntry[];
        bankedVoice?: VoiceBankReference;
    };

    // 어휘 사용자화
    vocabulary: {
        baseVocabulary: string;
        customPages: VocabularyPage[];
        customSymbols: Symbol[];
        hiddenItems: string[];
        buttonCustomizations: ButtonCustomization[];
    };

    // 사용 데이터 (선택사항, 개인정보 제어)
    usage?: {
        frequencyData: WordFrequency[];
        predictionModel?: PredictionModelData;
        communicationStats?: UsageStatistics;
    };

    // 무결성을 위한 디지털 서명
    signature?: string;
}

6.2.2 프로필 가져오기/내보내기 API

// 프로필 이동성 작업
interface ProfilePortability {
    // 사용자 프로필 내보내기
    exportProfile(userId: string, options: ExportOptions): Promise<ProfileExport>;

    // 시스템에 프로필 가져오기
    importProfile(profile: ProfileExport, options: ImportOptions): Promise<ImportResult>;

    // 프로필 병합 (예: 여러 기기에서 결합)
    mergeProfiles(primary: ProfileExport, secondary: ProfileExport): Promise<MergeResult>;

    // 가져오기 전 프로필 유효성 검사
    validateProfile(profile: ProfileExport): ValidationResult;
}

interface ExportOptions {
    includeUsageData: boolean;
    includeVoiceBank: boolean;
    includeCalibration: boolean;
    encryptionKey?: string;  // 민감한 데이터 보호용
}

interface ImportOptions {
    conflictResolution: 'keep-existing' | 'use-imported' | 'merge' | 'ask';
    recalibrationRequired: boolean;
    verifySignature: boolean;
}

6.3 의사소통 플랫폼 통합

AAC 시스템은 현대 의사소통 플랫폼과 통합되어야 합니다:

6.3.1 메시징 통합

플랫폼 통합 유형 기능
SMS/MMS 시스템 통합 AAC 인터페이스에서 텍스트, 사진 송수신
이메일 IMAP/SMTP 또는 API 이메일 작성 및 읽기
화상 통화 WebRTC 또는 앱 통합 통화 참여, 수어를 위한 카메라 지원
소셜 미디어 플랫폼 API 업데이트 게시, 메시지 보내기
채팅 앱 딥 링킹 또는 API WhatsApp, Messenger 등

6.3.2 통합 메시징 API

// AAC용 통합 메시징 인터페이스
interface AACMessaging {
    // 연결된 모든 플랫폼을 통해 메시지 보내기
    sendMessage(message: OutgoingMessage): Promise<SendResult>;

    // 플랫폼 간 메시지 수신
    onMessage(callback: (message: IncomingMessage) => void): void;

    // 플랫폼 관리
    getConnectedPlatforms(): Platform[];
    connectPlatform(platform: Platform, credentials: AuthCredentials): Promise<void>;
    disconnectPlatform(platformId: string): void;
}

interface OutgoingMessage {
    platform: string;
    recipient: Contact;
    content: {
        text: string;
        attachments?: Attachment[];
    };
    speakOnSend: boolean;  // 메시지를 소리내어 말하기도
}

// 예: 메시지 보내기
const messaging = aacSystem.getMessaging();
await messaging.sendMessage({
    platform: 'sms',
    recipient: { name: '엄마', address: '+821012345678' },
    content: { text: '5시에 집에 도착할게요' },
    speakOnSend: true
});
의사소통 형평성: AAC 사용자는 다른 모든 사람과 동일하게 디지털 의사소통에 접근할 수 있어야 합니다. 주류 플랫폼과의 통합은 현대 의사소통의 전체 범위에 참여를 보장합니다.

6.4 환경 제어 통합

AAC 기기는 종종 사용자 환경을 위한 제어 센터 역할을 합니다:

6.4.1 스마트 홈 프로토콜

  • Matter: 새롭게 부상하는 범용 스마트 홈 표준
  • HomeKit: Apple의 홈 자동화 프레임워크
  • Google Home: Google의 스마트 홈 생태계
  • Alexa Smart Home: Amazon의 음성 우선 플랫폼
  • Z-Wave/Zigbee: 저전력 메시 네트워크
  • IR 제어: 레거시 적외선 기기 제어

6.4.2 환경 제어 API

// 환경 제어 통합
interface EnvironmentControl {
    // 기기 검색
    discoverDevices(): Promise<SmartDevice[]>;

    // 기기 제어
    controlDevice(deviceId: string, command: DeviceCommand): Promise<void>;

    // 방/장면 관리
    activateScene(sceneId: string): Promise<void>;

    // 상태 모니터링
    getDeviceStatus(deviceId: string): Promise<DeviceStatus>;
    onStatusChange(callback: (device: SmartDevice, status: DeviceStatus) => void): void;
}

interface SmartDevice {
    id: string;
    name: string;
    room: string;
    type: DeviceType;
    capabilities: DeviceCapability[];
    currentStatus: DeviceStatus;
}

type DeviceType = 'light' | 'thermostat' | 'door-lock' | 'tv' | 'blinds' |
                   'fan' | 'outlet' | 'speaker' | 'camera' | 'bed' | 'phone';

// AAC 어휘 통합
interface ECUVocabularyPage {
    generateControlButtons(devices: SmartDevice[]): VocabularyPage {
        const page: VocabularyPage = {
            id: 'ecu-controls',
            name: '집 제어',
            type: 'ecu',
            rows: 4,
            columns: 6,
            cells: []
        };

        for (const device of devices) {
            page.cells.push({
                symbol: this.getDeviceSymbol(device.type),
                label: device.name,
                action: {
                    type: 'ecu-control',
                    deviceId: device.id,
                    showSubmenu: device.capabilities.length > 2
                }
            });
        }

        return page;
    }
}

6.5.2 오프라인 우선 아키텍처

AAC 시스템은 오프라인에서 완전히 작동해야 하며 연결이 복원되면 동기화해야 합니다:

  • 오프라인에서 모든 핵심 기능 사용 가능
  • 오프라인 기간 동안 변경사항 대기열
  • 연결 복원 시 자동 동기화
  • 동시 편집을 위한 충돌 해결
  • 대역폭 효율적인 델타 동기화
신뢰성 요구사항: 의사소통은 인터넷 연결을 기다릴 수 없습니다. WIA-AAC 시스템은 오프라인에서 전체 기능을 유지해야 하며, 클라우드 기능은 핵심 의사소통 기능을 활성화하는 것이 아니라 향상시켜야 합니다.

6.6 API 표준

WIA-AAC는 타사 통합을 위한 표준 API를 정의합니다:

6.6.1 REST API 사양

// WIA-AAC REST API 엔드포인트
// 기본 URL: https://api.aac-device.local/v1

// 메시지 작업
POST   /messages/speak           // 메시지 말하기
POST   /messages/queue           // 말하기 위한 메시지 대기열
DELETE /messages/queue           // 메시지 대기열 지우기

// 어휘 작업
GET    /vocabulary/pages         // 어휘 페이지 목록
GET    /vocabulary/pages/{id}    // 특정 페이지 가져오기
POST   /vocabulary/pages         // 사용자 정의 페이지 생성
PUT    /vocabulary/pages/{id}    // 페이지 업데이트
GET    /vocabulary/symbols       // 상징 검색

// 접근 제어
GET    /access/methods           // 사용 가능한 방법 목록
PUT    /access/active            // 접근 방법 전환
GET    /access/calibration       // 보정 상태 가져오기

// 프로필 관리
GET    /profile                  // 사용자 프로필 가져오기
PUT    /profile                  // 프로필 업데이트
POST   /profile/export           // 프로필 내보내기
POST   /profile/import           // 프로필 가져오기

// 환경 제어
GET    /ecu/devices              // 연결된 기기 목록
POST   /ecu/devices/{id}/command // 기기 명령 보내기

6.6.2 WebSocket 이벤트

// 통합을 위한 실시간 이벤트 스트림
// 연결: wss://api.aac-device.local/v1/events

interface AACEvent {
    type: EventType;
    timestamp: string;
    data: EventData;
}

type EventType =
    | 'message.spoken'        // 메시지가 말해짐
    | 'message.queued'        // 대기열에 메시지 추가됨
    | 'selection.made'        // 사용자가 선택함
    | 'page.changed'          // 어휘 페이지 변경됨
    | 'access.changed'        // 접근 방법 변경됨
    | 'profile.updated'       // 프로필 수정됨
    | 'ecu.device.status';    // 스마트 기기 상태 변경됨

// 이벤트 구독
const ws = new WebSocket('wss://api.aac-device.local/v1/events');
ws.onmessage = (event) => {
    const aacEvent = JSON.parse(event.data);
    if (aacEvent.type === 'message.spoken') {
        console.log('사용자가 말함:', aacEvent.data.text);
    }
};

6.7 데이터 교환 형식

  • WIA-AAC는 추가 메타데이터로 OBZ를 확장

6.8 보조기술 호환성

AAC 시스템은 다른 보조기술과 함께 작동해야 합니다:

기술 통합 접근법
화면 판독기 ARIA 레이블, 시맨틱 HTML, 포커스 관리
스위치 인터페이스 HID 호환성, 구성 가능한 매핑
전동 휠체어 장착 통합, 공유 제어

보청기 Bluetooth LE Audio, 히어링 루프
범용 설계: AAC 시스템은 더 큰 보조기술 생태계의 일부입니다. 다른 AT 기기와의 원활한 통합은 독립성을 극대화하고 사용자의 기술 부담을 최소화합니다.
