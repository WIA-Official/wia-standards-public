5장: 음성 출력과 음성 뱅킹

5.1 목소리의 중요성

목소리는 개인 정체성의 기본적인 측면입니다. AAC 사용자에게 기기가 생성하는 음성은 세상에서 그들의 목소리가 됩니다. WIA-AAC 표준은 음성 품질, 개인화 및 음성 뱅킹 기술을 통해 자신의 목소리를 보존할 수 있는 능력의 깊은 중요성을 인식합니다.

음성 출력은 시각적 선택을 청각적 의사소통으로 변환하여 AAC 사용자가 대화에 참여하고, 주의를 끌고, 시각 전용 의사소통이 할 수 없는 방식으로 자신을 표현할 수 있게 합니다. 고품질의 자연스러운 음성 출력은 사용자의 경험과 의사소통 파트너가 AAC 매개 의사소통을 인식하고 반응하는 방식 모두에 상당한 영향을 미칩니다.

정체성 원칙: AAC 사용자의 합성 음성은 말하는 사람의 자연스러운 목소리만큼 개인적입니다. WIA-AAC 시스템은 가능한 경우 개인화된 음성 옵션과 함께 연령, 성별 및 문화적으로 적합한 음성을 포함하여 사용자 정체성을 존중하는 음성 옵션을 제공해야 합니다.

5.2 텍스트 음성 변환 기술

현대 TTS 시스템은 텍스트를 자연스러운 음성으로 변환하기 위해 정교한 알고리즘을 사용합니다:

5.2.1 TTS 아키텍처 세대

세대 기술 특성
1세대 포먼트 합성 규칙 기반, 로봇 같은 소리, 작은 풋프린트
2세대 연결 합성 미리 녹음된 단위가 연결됨, 더 자연스러움
3세대 통계적 파라메트릭 HMM 기반, 유연하지만 때때로 먹먹함
4세대 신경망 TTS 딥러닝, 매우 자연스러움, 계산 집약적
5세대 신경망 + 음성 복제 제한된 샘플에서 개인 음성 재생성

5.3 음성 선택 및 사용자화

음성 선택은 매우 개인적입니다. WIA-AAC 시스템은 의미 있는 선택을 제공해야 합니다:

5.3.1 음성 다양성 요구사항

  • 지원되는 언어당 최소 2개의 남성 및 2개의 여성 음성
  • 가능한 경우 어린이, 성인 및 노인 음성 옵션
  • 지역 억양 변형
  • 다양한 배경을 대표하는 음성

5.3.2 음성 사용자화 매개변수

// 음성 사용자화 옵션
interface VoiceSettings {
    // 핵심 매개변수
    rate: number;       // 0.5 - 2.0, 1.0은 정상 속도
    pitch: number;      // 0.5 - 2.0, 1.0은 정상 피치
    volume: number;     // 0.0 - 1.0

    // 고급 매개변수
    emphasis: number;   // 말하기 스타일 강도
    breathiness: number;
    roughness: number;

    // 발음
    pronunciationDictionary: PronunciationEntry[];

    // 상황별 재정의
    questionIntonation: boolean;
    exclamationEmphasis: boolean;
}

interface PronunciationEntry {
    text: string;
    pronunciation: string;  // IPA 또는 음성 철자
    context?: string;       // 명확화를 위한 선택적 컨텍스트
}

// 예시 사용
const userVoiceSettings: VoiceSettings = {
    rate: 0.9,           // 약간 느리게
    pitch: 1.1,          // 약간 높게
    volume: 0.8,
    emphasis: 1.2,
    breathiness: 0.0,
    roughness: 0.0,
    pronunciationDictionary: [
        { text: 'AAC', pronunciation: '에이 에이 씨' },
        { text: '할게', pronunciation: '할 게' }
    ],
    questionIntonation: true,
    exclamationEmphasis: true
};
개인화 요구사항: 모든 WIA-AAC 호환 시스템은 최소한 말하기 속도, 피치 및 볼륨을 조정할 수 있어야 합니다. 고급 시스템은 이름, 장소 및 전문 어휘에 대한 발음 사전을 지원해야 합니다.

5.4 음성 뱅킹

음성 뱅킹은 AAC에서 나중에 사용할 수 있도록 사람의 자연스러운 목소리를 보존합니다. 이는 말하는 능력을 잃을 수 있는 진행성 질환이 있는 개인에게 특히 중요합니다:

5.4.1 음성 뱅킹 과정

  1. 평가: 현재 음성 품질 및 녹음 용량 평가
  2. 스크립트 선택: 음성 및 언어에 따라 적절한 녹음 스크립트 선택
  3. 녹음: 통제된 환경에서 고품질 음성 샘플 캡처
  4. 처리: 녹음을 합성 가능한 음성 모델로 변환
  5. 통합: AAC 시스템에 개인 음성 배포

5.4.2 음성 뱅킹 API

// 음성 뱅킹 인터페이스
interface VoiceBankingService {
    // 녹음 관리
    startRecordingSession(): Promise<RecordingSession>;

    // 음성 생성

    // 음성 배포
    downloadVoice(voiceId: string): Promise<VoicePackage>;
    installVoice(voicePackage: VoicePackage): Promise<void>;
}

interface RecordingScript {
    id: string;
    language: string;
    sentences: RecordingSentence[];
    estimatedDuration: number;  // 분
    purpose: 'full-voice' | 'voice-repair' | 'accent-adaptation';
}

interface RecordingSentence {
    id: string;
    text: string;
    phonemes: string[];
    targetProsody: ProsodyTarget;
    minAcceptableQuality: number;
}

    targetQuality: 'standard' | 'high' | 'premium';

        pitch: boolean;

}

5.4.3 음성 뱅킹 모범 사례

측면 권장사항
녹음 환경 조용한 방, 최소한의 에코, 일관된 마이크 위치
마이크 USB 콘덴서 마이크, 헤드셋 또는 전문 설정

5.5 메시지 기반 음성 뱅킹

완전한 구문을 녹음하는 전통적인 음성 뱅킹의 대안:

interface MessageBankingService {

    recordMessage(message: string, audioData: AudioData): Promise<MessageRecording>;

    categorizeMessage(messageId: string, category: string): void;

    playBankedMessage(messageId: string): Promise<void>;

    speakWithBanking(text: string): Promise<SpeechResult>;
}

// 뱅킹된 메시지 구조
interface MessageRecording {
    id: string;
    text: string;
    normalizedText: string;  // 매칭용
    audioData: AudioData;
    duration: number;

하이브리드 접근법: 가장 효과적인 구현은 음성 뱅킹(개인화된 소리용)과 메시지 뱅킹(자주 사용하는 구문의 완벽한 녹음용) 및 표준 TTS(무제한 어휘용)를 결합합니다.

5.6 음성 복구 및 적응

질병이나 부상으로 인해 목소리가 변한 사용자용:

5.6.1 음성 복구 기술

  • 향상: 저하된 음성 녹음의 명확성 개선
  • 재구성: 오래된 녹음을 사용하여 합성 음성 생성
  • 적응: 개인 특성에 맞게 표준 음성 수정
  • 음성 기증: 유사한 기증자 음성에서 개인화된 음성 생성

5.6.2 레거시 녹음 처리

5.7 운율과 표현

자연스러운 말에는 단어 이상의 의미를 전달하는 운율적 특징이 포함됩니다:

5.7.1 운율 요소

  • 억양: 질문, 진술, 감정을 나타내는 피치 패턴
  • 강세: 특정 단어나 음절에 대한 강조
  • 리듬: 말의 타이밍과 속도
  • 휴지: 강조나 숨을 위한 전략적 침묵

5.7.2 운율 제어를 위한 SSML

// 표현력 있는 음성을 위한 SSML 생성
class SSMLGenerator {
    static generateExpressive(message: Message): string {
        let ssml = '<speak>';

        for (const segment of message.segments) {
            switch (segment.type) {
                case 'question':
                    ssml += `<prosody pitch="+10%" rate="95%">${segment.text}</prosody>`;
                    break;

                case 'exclamation':
                    ssml += `<prosody volume="loud" rate="110%">${segment.text}</prosody>`;
                    break;

                case 'emphasis':
                    ssml += `<emphasis level="strong">${segment.text}</emphasis>`;
                    break;

                case 'pause':
                    ssml += `<break time="${segment.duration}ms"/>`;
                    break;

                default:
                    ssml += segment.text;
            }
        }

        ssml += '</speak>';
        return ssml;
    }
}

5.8 다국어 음성

음성 출력에서 여러 언어 지원:

// 다국어 음성 구성
interface MultilingualConfig {
    primaryLanguage: string;
    additionalLanguages: string[];

    // 언어별 음성 할당
    voiceMapping: Map<string, string>;  // 언어 -> voiceId

    // 혼합 언어 텍스트 처리
    languageDetection: 'auto' | 'markup' | 'manual';

    // 언어 간 전환
    languageSwitching: {
        insertPause: boolean;
        pauseDuration: number;
        announceSwitch: boolean;
    };
}

// 언어 감지 및 전환
class MultilingualSpeaker {
    async speak(text: string): Promise<void> {
        const segments = await this.detectLanguageSegments(text);

        for (const segment of segments) {
            await this.switchVoice(segment.language);
            await this.tts.speak(segment.text);
        }
    }

    private async detectLanguageSegments(text: string): Promise<LanguageSegment[]> {
        // 언어 감지를 사용하여 언어 경계 식별
        return this.languageDetector.segment(text);
    }
}

5.9 오디오 출력 요구사항

음성 출력 하드웨어 및 소프트웨어에 대한 기술 요구사항:

요구사항 최소 권장
스피커 출력 1미터에서 75 dB 1미터에서 85 dB
주파수 응답 200 Hz - 8 kHz 100 Hz - 16 kHz
오디오 지연 < 500 ms < 200 ms
블루투스 지원 A2DP 프로필 A2DP + aptX
외부 스피커 포트 3.5mm 잭 3.5mm + USB 오디오
음향 환경: AAC 기기는 시끄러운 환경을 포함한 일반적인 의사소통 환경에서 들릴 수 있을 만큼 크고 명확하게 음성을 생성해야 합니다. 시스템은 어려운 음향 상황을 위해 볼륨 부스트 옵션과 외부 스피커 지원을 포함해야 합니다.
