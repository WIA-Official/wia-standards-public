4장: 입력 방법과 접근

4.1 접근의 중요성

접근은 의사소통의 관문입니다. AAC 시스템의 어휘나 언어 기능이 아무리 정교하더라도, 사용자가 상징을 안정적으로 선택하고 메시지를 구성할 수 없다면 의미가 없습니다. WIA-AAC 표준은 다양한 운동 능력을 수용하는 다양한 접근 방법을 지원하는 것을 최우선으로 합니다.

접근 방법은 개인의 운동 능력에 맞춰야 하며, 시스템은 능력이 시간이 지남에 따라 변화할 때 적응할 수 있을 만큼 유연해야 합니다. 아침에 작동하는 것이 오후에 피로해졌을 때는 작동하지 않을 수 있어, 다중 모드 접근이 필수적입니다.

기본권: 모든 개인은 AAC에 접근할 권리가 있습니다. WIA-AAC 표준은 호환 시스템이 운동 제한으로 인해 누구도 배제되지 않도록 여러 접근 방법을 지원하도록 요구합니다.

4.2 직접 선택 방법

직접 선택은 사용자가 대상을 직접 가리켜 선택을 표시할 수 있게 합니다. 운동 능력이 허용하는 경우 일반적으로 가장 빠른 접근 방법입니다:

4.2.1 터치 접근

터치는 충분한 운동 제어가 있는 AAC 사용자에게 가장 일반적인 접근 방법입니다:

// 터치 접근 구성
interface TouchAccessConfig {
    // 활성화 설정
    activation: {
        type: 'on-touch' | 'on-release' | 'dwell';
        dwellTime?: number;  // 밀리초, dwell 활성화인 경우
        requireLift?: boolean;  // 다음 선택 전에 들어올려야 함
    };

    // 터치 조절
    accommodation: {
        holdTime: number;      // 짧은 터치 무시 (ms)
        releaseTime: number;   // 짧은 놓음 무시 (ms)
        touchTolerance: number;  // 픽셀 단위 이동 허용 범위
    };

    // 대상 크기 요구사항
    targetSize: {
        minimum: number;  // mm 단위 최소 터치 대상
        recommended: number;
        spacing: number;  // 대상 간 최소 간격
    };

    // 청각 피드백
    feedback: {
        onTouch: boolean;
        onActivation: boolean;
        soundType: 'click' | 'tone' | 'speech';
    };
}
터치 설정 일반적인 사용자 구성
표준 터치 일반적인 운동 제어를 가진 사용자 터치 시 활성화, 작은 대상 가능
놓음 활성화 선택 전 확인이 필요한 사용자 터치 시 미리보기, 놓을 때 선택
유지 터치 떨림이나 우발적 터치가 있는 사용자 활성화에 지속적인 접촉 필요
터치 드웰 시선 추적으로 전환 중인 사용자 지정된 시간 동안 대상에 유지

4.2.2 시선 추적 접근

시선 추적 기술은 사용자가 어디를 보는지 추적하고 시각적 고정을 통해 선택할 수 있게 합니다:

// 시선 추적 접근 구현
class EyeGazeAccessMethod implements AccessMethod {
    readonly type = 'direct';
    readonly name = '시선 추적';

    private tracker: EyeTracker;
    private config: EyeGazeConfig;
    private calibration: CalibrationProfile;

    async initialize() {
        this.tracker = await EyeTracker.detect();
        if (!this.tracker) {
            throw new AccessError('시선 추적기가 감지되지 않음');
        }

        await this.tracker.connect();
        this.startGazeProcessing();
    }

    async calibrate(): Promise<CalibrationResult> {
        const calibrator = new EyeGazeCalibrator(this.tracker);

        // 표준 9점 또는 5점 보정
        const result = await calibrator.run({
            points: this.config.calibrationPoints,
            animation: 'shrinking-circle',
            duration: 2000,  // 점당 ms
            acceptanceThreshold: 30  // 픽셀
        });

        if (result.quality === 'poor') {
            // 재보정 또는 보조 보정 제공
            return await this.handlePoorCalibration(result);
        }

        this.calibration = result.profile;
        return result;
    }

    private startGazeProcessing() {
        this.tracker.onGazeData((data: GazeData) => {
            // 보정 수정 적용
            const corrected = this.calibration.correct(data);

            // 지터 감소를 위한 스무딩 적용
            const smoothed = this.smoother.process(corrected);

            // 고정(드웰) 감지
            const fixation = this.fixationDetector.update(smoothed);

            if (fixation.complete) {
                this.emitSelection(fixation.target);
            } else if (fixation.inProgress) {
                this.emitDwellProgress(fixation);
            }
        });
    }
}

interface EyeGazeConfig {
    // 드웰 설정
    dwellTime: number;           // 선택을 위한 드웰 시간 (ms)
    dwellTolerance: number;      // 드웰 중 최대 이동 (픽셀)

    // 시각적 피드백
    dwellIndicator: {
        type: 'pie' | 'shrinking' | 'filling' | 'none';
        color: string;
        size: number;
    };

    // 스무딩
    smoothingLevel: 'none' | 'light' | 'medium' | 'heavy';

    // 보정
    calibrationPoints: 5 | 9 | 16;
    autoRecalibrate: boolean;
}
시선 추적 요구사항: WIA-AAC 호환 시선 추적 시스템은 200ms에서 5000ms까지의 드웰 시간을 지원하고, 드웰 중 명확한 시각적 피드백을 제공하며, 품질 평가와 함께 최소 5점 보정을 제공해야 합니다.

4.2.3 머리 추적

머리 추적은 머리 위치의 카메라 기반 감지를 사용하여 커서를 제어합니다:

  • 표준 웹캠 또는 전문 트래커로 작동
  • 사용자가 머리를 움직여 커서 이동
  • 드웰, 스위치 또는 제스처를 통한 선택
  • 다른 접근 방법과 결합 가능

4.2.4 조이스틱과 마우스

전통적인 포인팅 장치는 여전히 가치 있는 접근 방법입니다:

  • 적응된 버튼이 있는 표준 마우스
  • 다양한 크기와 감도의 트랙볼
  • 조이스틱 (비례 또는 디지털)
  • 머리 제어 마우스 대안

4.3 스캐닝 접근 방법

대상을 안정적으로 가리킬 수 없는 사용자의 경우, 스캐닝은 사용자가 스위치를 활성화하여 선택할 때까지 항목을 순차적으로 강조 표시하여 대안을 제공합니다:

4.3.1 스캐닝 패턴

// 스캐닝 패턴 정의
type ScanPattern =
    | LinearScan
    | RowColumnScan
    | GroupScan
    | CustomScan;

interface LinearScan {
    type: 'linear';
    direction: 'left-right' | 'right-left' | 'top-bottom' | 'bottom-top';
    wrap: boolean;  // 마지막 항목 후 처음으로 돌아감
}

interface RowColumnScan {
    type: 'row-column';
    primaryAxis: 'row' | 'column';
    // 첫 번째 스캔은 행을 강조, 두 번째 스캔은 행 내 항목
}

interface GroupScan {
    type: 'group';
    groups: ScanGroup[];
    // 계층적: 그룹 스캔, 그 다음 선택된 그룹 내 항목
}

interface ScanGroup {
    id: string;
    name: string;
    items: GridPosition[];
    subgroups?: ScanGroup[];
}

4.3.2 스캐닝 타이밍

설정 설명 범위
스캔 속도 각 항목/그룹이 강조 표시되는 시간 0.3 - 10초
첫 항목 지연 스위치 누름 후 첫 항목에 대한 추가 시간 0 - 5초
자동 시작 지연 스캐닝이 자동으로 시작되기 전 시간 0 - 10초
가속 초기 패스 후 속도 증가 0% - 50%
루프 중지 전 스캔 횟수 1 - 무제한

4.3.3 스위치 구성

// 스위치 접근 구성
interface SwitchConfig {
    switches: Switch[];
    debounce: number;  // 빠른 활성화 무시 (ms)
    acceptanceTime: number;  // 스위치를 유지해야 하는 시간 (ms)
    releaseTime: number;  // 활성화 사이 필요한 놓음 시간
}

interface Switch {
    id: string;
    name: string;
    inputSource: SwitchInput;
    action: SwitchAction;
}

type SwitchInput =
    | { type: 'keyboard'; key: string }
    | { type: 'gamepad'; button: number }
    | { type: 'usb-switch'; vendorId: number; productId: number }
    | { type: 'bluetooth'; deviceId: string }
    | { type: 'screen-region'; bounds: Rectangle };

type SwitchAction =
    | { type: 'select' }
    | { type: 'next' }  // 단계 스캐닝용
    | { type: 'back' }
    | { type: 'escape' }
    | { type: 'custom'; handler: string };

// 일반적인 구성
const singleSwitchConfig: SwitchConfig = {
    switches: [
        { id: 'main', name: '메인 스위치', inputSource: { type: 'keyboard', key: 'Space' }, action: { type: 'select' } }
    ],
    debounce: 50,
    acceptanceTime: 100,
    releaseTime: 200
};

const twoSwitchConfig: SwitchConfig = {
    switches: [
        { id: 'advance', name: '전진', inputSource: { type: 'keyboard', key: 'Space' }, action: { type: 'next' } },
        { id: 'select', name: '선택', inputSource: { type: 'keyboard', key: 'Enter' }, action: { type: 'select' } }
    ],
    debounce: 50,
    acceptanceTime: 50,
    releaseTime: 100
};

4.3.4 스캐닝 유형

  • 자동 스캐닝: 항목이 설정된 간격으로 자동 진행; 스위치로 강조된 항목 선택
  • 단계 스캐닝: 사용자가 한 스위치로 진행을 제어하고, 다른 스위치로 선택
  • 역 스캐닝: 스위치를 누르는 동안 항목이 스캔됨; 놓아서 현재 항목 선택
  • 방향 스캐닝: 다른 방향을 위한 여러 스위치 (위/아래/왼쪽/오른쪽/선택)
스캐닝 효율성: 잘 설계된 어휘 조직과 스캔 패턴은 의사소통 속도를 크게 향상시킬 수 있습니다. WIA-AAC 시스템은 자주 사용하는 항목을 우선시하는 사용자 정의 가능한 스캔 그룹을 지원해야 합니다.

4.4 신흥 접근 기술

4.4.1 뇌-컴퓨터 인터페이스 (BCI)

BCI는 신경 신호를 감지하여 AAC 시스템의 직접적인 뇌 제어를 가능하게 합니다:

// BCI 접근 인터페이스 (신흥 표준)
interface BCIAccessMethod extends AccessMethod {
    readonly type: 'bci';

    // BCI 특정 메서드
    detectIntent(): Promise<Intent>;
    trainClassifier(samples: TrainingSample[]): Promise<void>;
    getSignalQuality(): SignalQuality;

    // 지원되는 패러다임
    paradigm: BCIParadigm;
}

type BCIParadigm =
    | 'p300'          // P300 스펠러 패러다임
    | 'ssvep'         // 정상 상태 시각 유발 전위
    | 'motor-imagery' // 상상된 움직임
    | 'hybrid';       // 패러다임 조합

// P300 스펠러 구현 스케치
class P300Speller implements BCIAccessMethod {
    async detectIntent(): Promise<Intent> {
        // 항목의 행과 열 깜박임
        await this.runFlashingSequence();

        // P300 반응에 대한 EEG 분석
        const analysis = await this.analyzeERPSignals();

        // 어떤 항목이 P300을 유발했는지 결정
        return this.decodeSelection(analysis);
    }
}

4.4.2 EMG (근육 신호) 접근

근전도 검사는 스위치 활성화를 위해 근육 활동을 감지합니다:

  • 최소한의 근육 움직임으로 작동
  • 제어 가능한 모든 근육 사용 가능
  • 얼굴 근육, 눈 깜박임 또는 잔여 사지 움직임
  • 조절 가능한 민감도 임계값

4.4.3 음성 접근

일부 발성 능력이 있는 사용자용:

  • 소리 기반 스위치 활성화
  • 여러 명령을 위한 모음 구별
  • 허밍 또는 지속된 발성 감지
  • 부분 음성 인식과의 통합

4.5 다중 모드 접근

가장 효과적인 AAC 구현은 여러 접근 방법을 결합합니다:

// 다중 모드 접근 구성
interface MultiModalConfig {
    methods: AccessMethodConfig[];
    switchingRules: SwitchingRule[];
    fallbackOrder: string[];  // 우선순위 순서의 메서드 ID
}

interface SwitchingRule {
    trigger: SwitchTrigger;
    action: SwitchAction;
}

type SwitchTrigger =
    | { type: 'manual'; gesture: string }
    | { type: 'timeout'; duration: number }
    | { type: 'errorRate'; threshold: number }
    | { type: 'fatigue'; indicator: FatigueIndicator }
    | { type: 'schedule'; times: TimeRange[] };

// 예: 피로 시 시선 추적에서 스캐닝으로 전환
const fatigueAdaptation: SwitchingRule = {
    trigger: {
        type: 'fatigue',
        indicator: {
            metric: 'dwellTimeIncrease',
            threshold: 0.5,  // 기준선에서 50% 증가
            window: 300000   // 5분 동안
        }
    },
    action: {
        switchTo: 'scanning',
        notify: true,
        revertAfter: 1800000  // 30분 후 시선 추적 다시 시도
    }
};

4.6 접근 평가 및 보정

적절한 평가는 최적의 접근 구성을 보장합니다:

4.6.1 운동 평가 프로토콜

  1. 운동 범위 및 제어 평가
  2. 피로 패턴 평가
  3. 여러 잠재적 접근 부위 테스트
  4. 다양한 대상 크기에서 정확도 측정
  5. 타이밍 및 지구력 평가

4.6.2 보정 도구

// 내장 보정 및 평가 도구
class AccessAssessment {
    // 데이터 수집과 함께 대상 연습
    async runTargetPractice(config: TargetPracticeConfig): Promise<AssessmentResult> {
        const trials: TrialResult[] = [];

        for (let i = 0; i < config.trialCount; i++) {
            const target = this.generateTarget(config.targetSize);
            const start = performance.now();

            const result = await this.waitForSelection(target, config.timeout);

            trials.push({
                targetPosition: target.position,
                selectionPosition: result.position,
                reactionTime: result.timestamp - start,
                accuracy: this.calculateAccuracy(target, result),
                success: result.success
            });
        }

        return this.analyzeTrials(trials);
    }

    // 최적 대상 크기 결정
    async findOptimalTargetSize(): Promise<TargetSizeRecommendation> {
        const sizes = [40, 50, 60, 80, 100, 120]; // 픽셀
        const results = new Map<number, AssessmentResult>();

        for (const size of sizes) {
            results.set(size, await this.runTargetPractice({ targetSize: size, trialCount: 10 }));
        }

        // 90% 이상 정확도를 가진 가장 작은 크기 찾기
        return this.determineOptimalSize(results);
    }
}
지속적인 적응: 접근 요구는 시간이 지남에 따라 변합니다. WIA-AAC 시스템은 사용 패턴을 모니터링하고 성능이 운동 능력의 변화를 나타낼 때 사전에 조정을 제안해야 합니다.

4.7 접근 방법 표준

모든 WIA-AAC 호환 접근 구현은 다음을 해야 합니다:

  • 표준 AccessMethod 인터페이스 구현
  • 보정 및 평가 도구 제공
  • 지정된 범위 내에서 구성 가능한 타이밍 매개변수 지원
  • 시각, 청각 및 촉각 피드백 옵션 제공
  • 구성 프로필 저장 및 복원
  • 보정 데이터 내보내기/가져오기 지원
  • 하드웨어 연결 해제를 우아하게 처리
  • 실시간 성능 메트릭 제공
