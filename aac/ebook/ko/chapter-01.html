1장: AAC 기초

1.1 보완대체의사소통이란?

보완대체의사소통(AAC)은 구어 또는 문어의 산출이나 이해에 장애가 있는 개인을 위해 말하기와 쓰기를 보충하거나 대체할 수 있는 모든 의사소통 방법을 의미합니다. AAC는 간단한 제스처와 얼굴 표정부터 음성을 생성하는 정교한 전자 기기까지 광범위한 의사소통 방법을 포함합니다.

"보완(augmentative)"이라는 용어는 기존 음성을 보충하는 의사소통 방법을 의미하고, "대체(alternative)"는 음성을 완전히 대체하는 방법을 의미합니다. 많은 AAC 사용자들은 두 가지 접근 방식을 조합하여 사용하며, 주어진 상황에 가장 효과적인 방법을 활용합니다.

핵심 원칙: AAC는 최후의 수단이 아니라 나이, 진단 또는 인지 능력에 관계없이 복잡한 의사소통 요구가 있는 모든 사람의 의사소통을 향상시킬 수 있는 강력한 도구입니다.

1.2 AAC의 역사적 발전

AAC 분야는 지난 수십 년간 극적으로 발전해왔습니다:

1.2.1 초기 시작 (1950년대-1970년대)

AAC의 뿌리는 뇌성마비가 있는 개인들의 의사소통을 돕기 위한 노력에서 시작되었습니다. 초기 접근 방식에는 사용자가 가리킬 수 있는 글자, 단어, 그림이 있는 의사소통 판이 포함되었습니다. Charles Bliss가 개발한 블리스상징은 1971년 온타리오 장애아동센터에서 AAC용으로 사용된 최초의 표준화된 상징 시스템 중 하나가 되었습니다.

1.2.2 전자 혁명 (1980년대-1990년대)

마이크로프로세서의 도입으로 음성 생성 기기(SGD) 개발이 가능해졌습니다. 초기 기기는 크고, 비싸고, 어휘가 제한적이었지만, 독립적인 음성 출력을 제공한다는 점에서 획기적이었습니다. Prentke Romich와 DynaVox 같은 회사들이 전용 AAC 기기 분야의 선두주자로 부상했습니다.

1.2.3 모바일 컴퓨팅 시대 (2000년대-2010년대)

태블릿, 특히 2010년 iPad의 도입은 AAC 애플리케이션을 위한 저렴하고 주류적인 플랫폼을 제공함으로써 AAC에 혁명을 일으켰습니다. Proloquo2Go와 같은 앱은 전용 기기 비용의 일부만으로 상징 기반 의사소통을 훨씬 더 많은 사용자에게 접근 가능하게 만들었습니다.

1.2.4 AI와 클라우드 시대 (2020년대-현재)

현재의 발전에는 예측 텍스트와 어휘 제안을 위한 인공지능, 자연스럽게 들리는 음성을 위한 신경망 텍스트 음성 변환, 기기 간 원활한 동기화를 위한 클라우드 기반 시스템이 포함됩니다. WIA-AAC 표준은 이러한 맥락에서 상호운용성과 품질을 보장하기 위해 등장했습니다.

1.3 AAC 시스템의 유형

AAC 시스템은 일반적으로 기술적 복잡성에 따라 분류됩니다:

범주 설명 예시
무기술 AAC 외부 도구가 필요 없는 방법 제스처, 얼굴 표정, 발성, 수어
저기술 AAC 비전자적 도구 및 디스플레이 의사소통 판, 그림책, 알파벳 차트, PODD 책
중기술 AAC 간단한 전자 기기 단일 메시지 기기, 순차 메시지 기기, 간단한 음성 출력
고기술 AAC 정교한 전자 시스템 전용 SGD, 태블릿 앱, 시선 추적 시스템, 뇌-컴퓨터 인터페이스

1.4 핵심 용어

AAC를 이해하려면 특정 용어에 익숙해야 합니다:

1.4.1 의사소통 파트너

AAC 사용자와 상호작용하는 모든 사람입니다. 효과적인 AAC 구현을 위해서는 의사소통 파트너가 의사소통을 지원하고 촉진하도록 훈련받아야 합니다. 파트너는 응답을 기다리고, AAC 사용을 모델링하고, 의사소통 시도를 인식하는 방법을 배워야 합니다.

1.4.2 상징

개념, 단어 또는 구문을 나타내는 시각적 표현입니다. 상징은 사진, 선 그림, 추상적인 모양 또는 문자일 수 있습니다. WIA-AAC 표준은 PCS(그림 의사소통 상징), SymbolStix, ARASAAC, 블리스상징을 포함한 여러 상징 시스템을 지원합니다.

1.4.3 어휘 조직

상징이 접근을 위해 배열되는 방법입니다. 일반적인 접근 방식은 다음과 같습니다:

  • 그리드 기반: 상징이 행과 열로 배열됨
  • 시각적 장면 디스플레이: 상징이 맥락적 이미지에 포함됨
  • 의미-구문: 의미와 문법적 기능에 따라 조직됨
  • 실용적 조직: 의사소통 기능에 따라 조직됨

1.4.4 접근 방법

사용자가 상징이나 메시지를 선택하는 물리적 수단:

  • 직접 선택: 대상을 직접 가리키거나 터치하거나 시선으로 응시
  • 스캐닝: 항목이 순차적으로 강조 표시되고 사용자가 스위치를 활성화하여 선택
  • 인코딩: 사용자가 미리 저장된 메시지를 검색하기 위해 코드를 입력
// WIA-AAC 접근 방법 구성
const accessConfig = {
    method: 'scanning',
    pattern: 'row-column',
    speed: {
        initial: 2000,  // 밀리초
        accelerated: 1000,
        minimum: 500
    },
    auditoryFeedback: true,
    visualHighlight: {
        color: '#FFD700',
        style: 'border',
        width: 4
    },
    switches: [
        { id: 'primary', action: 'select' },
        { id: 'secondary', action: 'back' }
    ]
};

1.5 AAC의 혜택을 받는 사람들

AAC는 다양한 조건과 상황의 개인들에게 도움이 될 수 있습니다:

1.5.1 발달 조건

  • 자폐 스펙트럼 장애(ASD)
  • 뇌성마비
  • 다운증후군
  • 지적 장애
  • 아동기 말 실행증
  • 발달성 언어 장애

1.5.2 후천적 조건

  • 근위축성 측삭경화증(ALS/MND)
  • 뇌졸중 및 실어증
  • 외상성 뇌손상
  • 다발성 경화증
  • 파킨슨병
  • 치매

1.5.3 일시적 조건

  • 기관삽관 및 기계적 환기
  • 수술 후 회복
  • 일시적인 음성 상실
중요: AAC는 인지 능력에 대한 가정에 따라 거부되어서는 안 됩니다. 연구에 따르면 AAC 사용이 언어 발달을 방해하지 않으며 오히려 촉진하는 경우가 많습니다. "AAC 후보자" 개념은 적절하게 구현되면 누구나 AAC의 혜택을 받을 수 있다는 이해로 대체되었습니다.

1.6 AAC 평가 과정

개인에게 적합한 AAC를 매칭하려면 포괄적인 평가가 필요합니다:

1.6.1 기능 매칭

개인의 능력, 요구 및 상황에 맞는 AAC 기능을 식별하는 과정입니다. 이는 다음을 평가합니다:

  • 운동 능력 및 접근 방법
  • 시각 및 청각 지각
  • 인지 및 언어 기술
  • 의사소통 환경 및 파트너
  • 기술 경험 및 선호도

1.6.2 시험 기간

대부분의 AAC 평가에는 개인이 자연스러운 환경에서 다양한 시스템을 사용할 수 있는 시험 기간이 포함됩니다. 이는 임상 환경을 넘어 실제 효과에 대한 중요한 정보를 제공합니다.

// WIA-AAC 평가 프로필 스키마
interface AACAssessmentProfile {
    userId: string;
    assessmentDate: Date;
    assessors: Assessor[];

    motorProfile: {
        directSelectionCapable: boolean;
        preferredHand: 'left' | 'right' | 'either' | 'neither';
        targetSize: number;  // mm 단위 최소 대상 크기
        reactionTime: number;  // 밀리초
        fatigue: 'low' | 'moderate' | 'high';
        eyeGazeCapable: boolean;
        switchAccessCapable: boolean;
    };

    sensoryProfile: {
        visualAcuity: string;
        colorVision: boolean;
        auditoryProcessing: 'intact' | 'mild' | 'moderate' | 'severe';
        tactileSensitivity: 'typical' | 'hypo' | 'hyper';
    };

    languageProfile: {
        receptiveLevel: 'emerging' | 'basic' | 'functional' | 'advanced';
        expressiveLevel: 'emerging' | 'basic' | 'functional' | 'advanced';
        literacyLevel: 'pre-literate' | 'emerging' | 'functional' | 'literate';
        primaryLanguage: string;
        additionalLanguages: string[];
    };

    recommendations: AACRecommendation[];
}

1.7 핵심 어휘 vs 주변 어휘

어휘 구성을 이해하는 것은 효과적인 AAC 구현에 필수적입니다:

1.7.1 핵심 어휘

일상적인 의사소통에서 사람들이 말하는 것의 80%를 차지하는 약 200-400개의 소규모 단어 세트입니다. 이러한 단어들은 대부분 무한한 아이디어를 표현하기 위해 조합할 수 있는 기능어(대명사, 동사, 전치사)입니다. 예: 나, 너, 그것, 원하다, 가다, 더, 안, 도와줘, 좋아해, 그거.

1.7.2 주변 어휘

개인, 주제 또는 상황에 고유한 내용 특정 단어입니다. 단일 주변 단어는 덜 자주 사용되지만, 이러한 단어들은 개인적인 표현에 필수적입니다. 예: 특정 이름, 장소, 관심사 및 활동 특정 용어.

모범 사례: 효과적인 AAC 시스템은 항상 사용 가능한 견고한 핵심 어휘를 제공하고, 각 사용자의 고유한 요구와 관심사에 맞게 개인화할 수 있는 주변 어휘로 보완합니다.

1.8 신화와 오해

AAC에 대한 몇 가지 지속적인 신화를 다뤄야 합니다:

신화 현실
AAC는 최후의 수단이다 AAC는 언어 발달을 지원하기 위해 일찍 도입되어야 한다
AAC는 말하기를 방해한다 연구에 따르면 AAC는 종종 언어 발달을 촉진한다
사용자는 특정 인지 기술이 있어야 한다 모든 사람은 의사소통한다; AAC는 모든 수준에 적응할 수 있다
하나의 기기가 모두에게 맞는다 다양한 방법을 사용하는 다중 모드 의사소통이 최적이다
전문가만 AAC를 프로그래밍할 수 있다 가족과 사용자는 시스템을 사용자화할 수 있도록 권한을 부여받아야 한다

1.9 WIA-AAC 철학

WIA-AAC 표준은 여러 기본 원칙에 기반합니다:

  • 의사소통은 인권이다: 능력에 관계없이 모든 사람은 의사소통할 권리가 있다
  • 역량 추정: 모든 개인은 배우고 의사소통할 수 있다
  • 사용자 중심 설계: AAC 사용자는 시스템 설계 및 선택에 참여해야 한다
  • 상호운용성: 시스템은 종속을 방지하기 위해 함께 작동해야 한다
  • 지속적인 개선: AAC 시스템은 사용자와 함께 적응하고 성장해야 한다

이 장에서는 AAC의 기초적인 이해를 확립했습니다. 다음 장에서는 이러한 원칙을 WIA-AAC 호환 시스템에서 실현하는 기술 사양을 상세히 설명합니다.
