# 1. 배경 (Background)

## 1.1 AAC란?

**AAC (Augmentative and Alternative Communication)**
- 한국어: 보완대체의사소통
- 말이나 글로 의사소통이 어려운 사람을 위한 모든 방법과 도구

### 대상자

| 질환/장애 | 설명 | 전 세계 규모 |
|----------|------|-------------|
| ALS (루게릭병) | 운동신경 퇴화 | 약 33만 명 |
| 뇌성마비 | 운동 기능 장애 | 약 1,700만 명 |
| 사지마비 | 척수 손상 등 | 약 500만 명 |
| 뇌졸중 후유증 | 언어/운동 장애 | 수천만 명 |
| 중증 자폐 | 언어 표현 장애 | 수백만 명 |

**AAC가 필요한 인구: 전 세계 300만 명 이상**

## 1.2 현재 문제점

### 표준의 부재

```
현재 AAC 시장:

제조사 A: 자체 센서 프로토콜
제조사 B: 다른 센서 프로토콜
제조사 C: 또 다른 프로토콜
...

결과:
- 기기 간 호환 불가
- 소프트웨어 교체 시 처음부터 재설정
- 개발 비용 증가 → 기기 가격 상승
- 사용자 선택권 제한
```

### ALS 환자의 현실

```
병 진행 단계별 센서 변경:

초기: 손가락 → 키보드
중기: 손목 → 조이스틱
후기: 머리 → 머리 센서
말기: 눈 → 시선 추적
최말기: 뺨 → 근육 센서
최후: 뇌파 → BCI

각 단계마다:
- 새 기기 구매
- 새 소프트웨어 학습
- 처음부터 설정
- 비용 + 시간 + 정신적 부담
```

# 2. WIA AAC Standard 목표

## 2.1 핵심 목표

```
"어떤 센서를 사용하든,
 어떤 소프트웨어를 사용하든,
 동일한 방식으로 작동한다."
```

### 구체적 목표

| 목표 | 설명 |
|------|------|
| **센서 표준화** | 모든 AAC 센서의 출력 형식 통일 |
| **API 통일** | 소프트웨어 개발자를 위한 표준 인터페이스 |
| **프로토콜 규격화** | 통신 방식 표준화 (Bluetooth, USB, WiFi) |
| **생태계 연동** | WIA (ISP, Braille, Talk) 자동 연결 |

## 2.2 철학

### 홍익인간 (弘益人間)

```
- 표준은 무료로 공개
- 특허 없음, 영원히 오픈소스
- 누구나 구현 가능
- 인증/지원 서비스로 지속 가능성 확보
```

# 3. 기술 구조

## 3.1 센서 입력 → 텍스트 출력 파이프라인

```
┌─────────────────────────────────────────────────────┐
│                  Physical Layer                      │
│                  (물리적 센서)                       │
├─────────────────────────────────────────────────────┤
│  Eye Tracker │ Switch │ EMG │ EEG/BCI │ Breath     │
└──────┬───────┴────┬───┴──┬──┴────┬────┴─────┬──────┘
       │            │      │       │          │
       ▼            ▼      ▼       ▼          ▼
┌─────────────────────────────────────────────────────┐
│              Signal Layer (신호 계층)                │
│         WIA AAC Signal Format Standard              │
│                                                      │
│   모든 센서 출력 → 표준 JSON/Protocol Buffer         │
└─────────────────────────────────────────────────────┘
                         │
                         ▼
┌─────────────────────────────────────────────────────┐
│               API Layer (API 계층)                   │
│          WIA AAC API Interface Standard             │
│                                                      │
│   표준 함수 호출 → 입력 처리 → 글자 선택            │
└─────────────────────────────────────────────────────┘
                         │
                         ▼
┌─────────────────────────────────────────────────────┐
│            Protocol Layer (프로토콜 계층)            │
│        WIA AAC Communication Protocol               │
│                                                      │
│   Bluetooth │ USB │ WiFi │ Serial                   │
└─────────────────────────────────────────────────────┘
                         │
                         ▼
┌─────────────────────────────────────────────────────┐
│            Output Layer (출력 계층)                  │
│                  Text Output                         │
└─────────────────────────────────────────────────────┘
                         │
         ┌───────────────┼───────────────┐
         ▼               ▼               ▼
    ┌─────────┐    ┌─────────┐    ┌─────────┐
    │   TTS   │    │   ISP   │    │   WIA   │
    │  음성   │    │  수어   │    │ Braille │
    └─────────┘    └─────────┘    └─────────┘
```

## 3.2 계층별 역할

| 계층 | 역할 | Phase |
|------|------|:-----:|
| **Signal Layer** | 센서 신호 → 표준 포맷 | Phase 1 |
| **API Layer** | 표준 함수 인터페이스 | Phase 2 |
| **Protocol Layer** | 통신 프로토콜 규격 | Phase 3 |
| **Output Layer** | WIA 생태계 연동 | Phase 4 |

# 4. 개발 Phase

## Phase 1: Signal Format Standard

**목표**: 모든 AAC 센서의 출력 형식 표준화

```
입력 예시 (시선 추적):
{
  "type": "eye_tracker",
  "timestamp": 1702468800000,
  "data": {
    "gaze_x": 0.45,
    "gaze_y": 0.32,
    "fixation": true,
    "dwell_time_ms": 850
  },
  "confidence": 0.95
}

입력 예시 (스위치):
{
  "type": "switch",
  "timestamp": 1702468800000,
  "data": {
    "switch_id": 1,
    "state": "pressed",
    "duration_ms": 150
  },
  "confidence": 1.0
}
```

## Phase 2: API Interface Standard

**목표**: 소프트웨어 개발자를 위한 표준 API

```javascript
// 예시 API
import { WiaAAC } from 'wia-aac-standard';

const aac = new WiaAAC();

// 센서 연결
await aac.connect({ type: 'eye_tracker', device: 'tobii' });

// 입력 이벤트 수신
aac.on('selection', (char) => {
  console.log('Selected:', char);
});

// 텍스트 출력
aac.on('text', (text) => {
  // TTS, ISP, WIA Braille로 전송 가능
});
```

## Phase 3: Communication Protocol

**목표**: 통신 방식 표준화

| 방식 | 용도 | 스펙 |
|------|------|------|
| Bluetooth LE | 무선 센서 | GATT Profile 정의 |
| USB HID | 유선 센서 | Report Descriptor 정의 |
| WiFi/TCP | 네트워크 센서 | REST/WebSocket API |
| Serial | 레거시 센서 | 패킷 포맷 정의 |

## Phase 4: WIA Ecosystem Integration

**목표**: 텍스트 출력 → WIA 생태계 연동

```
텍스트 "안녕하세요"
    │
    ├─→ TTS 엔진 → 음성 출력
    │
    ├─→ ISP 변환 → 수어 아바타 출력
    │
    └─→ IPA 변환 → WIA Braille → 점자 출력
```

# 5. 기대 효과

## 5.1 사용자 (환자)

```
Before:
- 센서 바꿀 때마다 새 소프트웨어 학습
- 기기마다 다른 설정 방식
- 비싼 전용 기기만 사용 가능

After:
- 센서 바꿔도 소프트웨어 그대로
- 모든 기기 동일한 설정 방식
- 저렴한 범용 기기도 사용 가능
```

## 5.2 제조사

```
Before:
- 자체 프로토콜 개발 비용
- 소프트웨어까지 직접 개발
- 폐쇄적 생태계

After:
- 표준 따르면 개발 비용 절감
- 하드웨어에만 집중 가능
- 개방형 생태계로 시장 확대
```

## 5.3 사회

```
Before:
- 높은 AAC 기기 가격
- 낮은 접근성
- 각 국가별 파편화

After:
- 경쟁으로 가격 하락
- 높은 접근성
- 국제 표준으로 글로벌 호환
```
