4.1 데이터 모델 개요

WIA-AGING 데이터 형식은 모든 노화 관련 건강 데이터를 표현하기 위한 표준화된 구조를 제공합니다. 이 형식은 JSON (JavaScript Object Notation)을 기반으로 하며, 이는 인간이 읽을 수 있고 기계가 파싱할 수 있으며 널리 지원되는 형식입니다. 이 선택은 개발자 친화성과 기존 도구와의 호환성을 보장합니다.

데이터 모델은 여러 핵심 개념으로 구성됩니다:

  • 프로파일: 개인의 기본 정보 및 인구통계
  • 평가: 특정 시점의 바이오마커 측정 컬렉션
  • 바이오마커: 단일 측정 또는 관찰
  • 메타데이터: 측정에 대한 맥락 정보
  • 타임라인: 시간에 따른 평가의 역사적 시퀀스

이러한 개념은 계층적으로 조직되어 있으며, 프로파일이 최상위 수준이고 개별 바이오마커가 가장 세밀한 수준입니다. 이 구조는 데이터를 직관적으로 조직하고 효율적으로 쿼리할 수 있게 합니다.

4.1.1 설계 목표

WIA-AGING 데이터 형식은 다음 목표를 달성하도록 설계되었습니다:

목표 구현 혜택
인간 가독성 JSON 형식, 명확한 필드 이름 쉬운 디버깅, 검토, 이해
기계 파싱 가능성 엄격한 스키마, 타입 정의 신뢰할 수 있는 자동화, 검증
확장 가능성 선택적 필드, 맞춤 확장 미래 요구에 적응
호환성 표준 단위, 코딩 시스템 기존 시스템과 통합
완전성 풍부한 메타데이터, 맥락 정확한 해석, 재현성
효율성 압축, 선택적 필드 저장 및 대역폭 최적화

4.2 JSON 스키마 정의

WIA-AGING 데이터 형식은 JSON Schema (Draft 2020-12)를 사용하여 공식적으로 정의됩니다. 이것은 모든 구현이 데이터를 일관되게 생성하고 검증할 수 있도록 보장합니다. 핵심 스키마를 살펴보겠습니다.

4.2.1 최상위 프로파일 스키마

{
  "$schema": "https://json-schema.org/draft/2020-12/schema",
  "$id": "https://aging.wiastandards.com/schemas/profile/v1.0",
  "title": "WIA-AGING Profile",
  "description": "개인의 노화 관련 건강 프로파일",
  "type": "object",
  "required": ["profileId", "version", "created"],
  "properties": {
    "profileId": {
      "type": "string",
      "description": "프로파일의 고유 식별자 (UUID v4)",
      "pattern": "^[0-9a-f]{8}-[0-9a-f]{4}-4[0-9a-f]{3}-[89ab][0-9a-f]{3}-[0-9a-f]{12}$"
    },
    "version": {
      "type": "string",
      "description": "WIA-AGING 표준 버전",
      "pattern": "^\\d+\\.\\d+\\.\\d+$",
      "examples": ["1.0.0"]
    },
    "created": {
      "type": "string",
      "format": "date-time",
      "description": "프로파일 생성 타임스탬프 (ISO 8601)"
    },
    "updated": {
      "type": "string",
      "format": "date-time",
      "description": "마지막 업데이트 타임스탬프 (ISO 8601)"
    },
    "individual": {
      "$ref": "#/$defs/Individual"
    },
    "assessments": {
      "type": "array",
      "items": {
        "$ref": "#/$defs/Assessment"
      }
    },
    "metadata": {
      "$ref": "#/$defs/Metadata"
    }
  }
}

4.5 검증 규칙

데이터 품질을 보장하기 위해 WIA-AGING은 여러 검증 규칙을 정의합니다. 이러한 규칙은 생성 시점과 수신 시점 모두에서 적용되어야 합니다.

4.5.1 필수 필드 검증

다음 필드는 모든 WIA-AGING 문서에 필수입니다:

  • profileId - 유효한 UUID v4여야 함
  • version - 의미론적 버전 형식 (예: "1.0.0")
  • created - 유효한 ISO 8601 타임스탬프
  • individual.birthYear - 1900과 현재 연도 사이
  • individual.sex - 허용된 값 중 하나

4.5.2 데이터 타입 검증

필드 타입 검증 규칙
birthYear integer 1900 ≤ value ≤ 현재 연도
heightCm number 50 ≤ value ≤ 300
confidence number 0 ≤ value ≤ 1
timestamp string ISO 8601 형식, 미래 날짜 아님
loincCode string 유효한 LOINC 코드 형식

4.5.3 비즈니스 로직 검증

타입 검증 외에도 다음 비즈니스 규칙을 적용해야 합니다:

  • 시간 순서: 평가 날짜는 프로파일 생성 날짜 이후여야 합니다
  • 나이 일관성: 평가 시점의 나이는 출생 연도와 평가 날짜와 일치해야 합니다
  • 참조 범위: 참조 범위의 최소값은 최대값보다 작아야 합니다
  • 단위 일관성: 동일한 바이오마커는 평가 전반에 걸쳐 일관된 단위를 사용해야 합니다
  • 유효한 열거형: 열거 필드는 허용된 값 중 하나만 포함해야 합니다
