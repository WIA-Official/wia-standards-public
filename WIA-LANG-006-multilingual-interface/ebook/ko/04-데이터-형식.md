# 제4장: 데이터 형식

> **弘益人間 (홍익인간)** - 표준화된 데이터 형식은 모든 시스템이 소통할 수 있는 공통 언어입니다.

## 4.1 WIA Translation Format (WTF)

### 4.1.1 개요

WIA-LANG-006 표준의 기본 번역 형식은 **WTF (WIA Translation Format)**입니다. JSON 기반의 구조화된 형식으로, 다양한 언어적 특성을 표현할 수 있습니다.

```typescript
// WTF 스키마 정의
interface WIATranslationFormat {
  // 필수 메타데이터
  $schema: 'https://wia.org/schemas/lang-006/v1.0';
  $version: string;          // 스키마 버전 (semver)
  $locale: string;           // BCP 47 언어 태그
  $namespace?: string;       // 네임스페이스 (선택)

  // 번역 데이터
  translations: TranslationMap;

  // 메타정보
  metadata: Metadata;

  // 설정 (선택)
  config?: Config;
}
```

**언어별 복수형 규칙:**

```javascript
// CLDR 기반 복수형 규칙
const pluralRules = {
  // 한국어: 복수형 구분 없음 (항상 'other')
  'ko': {
    categories: ['other'],
    rule: (n) => 'other'
  },

  // 영어: 1 vs 기타
  'en': {
    categories: ['one', 'other'],
    rule: (n) => n === 1 ? 'one' : 'other'
  },

  // 아랍어: 6가지 형태
  'ar': {
    categories: ['zero', 'one', 'two', 'few', 'many', 'other'],
    rule: (n) => {
      if (n === 0) return 'zero';
      if (n === 1) return 'one';
      if (n === 2) return 'two';
      if (n % 100 >= 3 && n % 100 <= 10) return 'few';
      if (n % 100 >= 11 && n % 100 <= 99) return 'many';
      return 'other';
    }
  },

  // 폴란드어: 3가지 형태
  'pl': {
    categories: ['one', 'few', 'many'],
    rule: (n) => {
      if (n === 1) return 'one';
      if (n % 10 >= 2 && n % 10 <= 4 && (n % 100 < 12 || n % 100 > 14)) {
        return 'few';
      }
      return 'many';
    }
  },

  // 러시아어: 3가지 형태
  'ru': {
    categories: ['one', 'few', 'many'],
    rule: (n) => {
      const mod10 = n % 10;
      const mod100 = n % 100;
      if (mod10 === 1 && mod100 !== 11) return 'one';
      if (mod10 >= 2 && mod10 <= 4 && (mod100 < 12 || mod100 > 14)) {
        return 'few';
      }
      return 'many';
    }
  }
};
```

## 4.3 형식 변환

### 4.3.1 JSON ↔ YAML

```yaml
# YAML 형식 (WTF-YAML)
$schema: 'https://wia.org/schemas/lang-006/v1.0'
$version: '1.0.0'
$locale: 'ko-KR'
$namespace: 'common'

translations:
  greeting: 안녕하세요
  welcome: '환영합니다, {{name}}님'

  items:
    $type: plural
    one: '{{count}}개 항목'
    other: '{{count}}개 항목'

  user:
    profile:
      title: 프로필
      edit: 프로필 수정

metadata:
  created: '2025-01-15T09:00:00Z'
  status: approved
  completeness: 100
```

## 4.7 메타데이터 및 주석

### 4.7.1 번역 메타데이터

```typescript
interface TranslationMetadata {
  // 식별 정보
  id?: string;                    // 고유 ID
  key: string;                    // 번역 키

  // 문맥 정보
  context?: string;               // 사용 문맥
  description?: string;           // 설명
  screenshot?: string;            // 스크린샷 URL
  location?: {                    // 소스 코드 위치
    file: string;
    line: number;
  };

  // 번역 정보
  sourceText?: string;            // 원문
  translatedText: string;         // 번역문
  translator?: string;            // 번역자
  reviewedBy?: string;            // 검토자

  // 상태 및 품질
  status: TranslationStatus;
  quality?: QualityScore;

  // 시간 정보
  createdAt: string;              // 생성 시간
  updatedAt: string;              // 수정 시간
  approvedAt?: string;            // 승인 시간

  // 태그 및 카테고리
  tags?: string[];                // 태그
  category?: string;              // 카테고리

  // 기술 정보
  maxLength?: number;             // 최대 길이 제한
  format?: 'text' | 'html' | 'markdown';
  variables?: string[];           // 변수 목록

  // 참고 정보
  notes?: string;                 // 번역 노트
  glossary?: string[];            // 용어집 참조
  tm?: {                          // 번역 메모리
    source: string;
    similarity: number;           // 유사도 (0-100)
  };
}

enum TranslationStatus {
  DRAFT = 'draft',
  REVIEW = 'review',
  APPROVED = 'approved',
  PUBLISHED = 'published',
  OUTDATED = 'outdated'
}
```

## 요약

WIA-LANG-006 데이터 형식은 다음을 제공합니다:

1. **표준 형식 (WTF)**: JSON 기반의 구조화된 번역 데이터
2. **다양한 번역 타입**: 단순, 복수형, 성별, 복합 번역 지원
3. **형식 변환**: JSON, YAML, XLIFF, PO, CSV 간 자유로운 변환
4. **풍부한 메타데이터**: 번역 품질 및 워크플로우 관리
5. **스키마 검증**: JSON Schema를 통한 데이터 무결성 보장
6. **포맷팅 규칙**: 날짜, 숫자, 통화 등의 지역화

**다음 장**에서는 API 인터페이스 설계를 자세히 살펴보겠습니다.
