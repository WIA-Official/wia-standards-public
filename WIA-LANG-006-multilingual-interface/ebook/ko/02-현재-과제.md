## 2.1 기존 다국어 지원의 문제점

### 2.1.1 임시방편적 접근법

많은 소프트웨어 프로젝트가 다국어 지원을 사후에 추가하면서 다음과 같은 문제에 직면합니다:

```typescript
// ❌ 문제 1: 하드코딩된 문자열
class UserGreeting {
  showWelcome(user: User) {
    if (user.language === 'ko') {
      return "안녕하세요, " + user.name + "님";
    } else if (user.language === 'ja') {
      return "こんにちは、" + user.name + "さん";
    } else if (user.language === 'zh') {
      return "你好，" + user.name;
    } else {
      return "Hello, " + user.name;
    }
  }
}

// 문제점:
// 1. 유지보수 불가능 (새 언어 추가마다 코드 수정)
// 2. 번역가가 코드 접근 필요
// 3. 문법 규칙 무시 (어순, 격 변화 등)
// 4. 배포 없이 번역 업데이트 불가능
```

**실제 통계:**


## 2.2 기술적 도전 과제

### 2.2.1 문자열 길이 변동

동일한 의미의 텍스트도 언어별로 길이가 크게 다릅니다:

```python
# 문자열 길이 비교 예제
translations = {
    'en': 'Save',           # 4 characters
    'fr': 'Enregistrer',    # 12 characters (+200%)
    'de': 'Speichern',      # 9 characters (+125%)
    'es': 'Guardar',        # 7 characters (+75%)
    'ko': '저장',           # 2 characters (-50%)
    'ja': '保存',           # 2 characters (-50%)
    'zh': '保存',           # 2 characters (-50%)
    'ar': 'حفظ',            # 3 characters (-25%)
}

# UI 레이아웃에 미치는 영향
button_width = {
    'en': '60px',   # 기준
    'fr': '140px',  # 버튼이 깨짐
    'de': '110px',  # 텍스트 잘림
```


### 2.2.2 복수형 규칙의 복잡성

영어의 단순한 복수형 규칙(1 vs 기타)과 달리, 많은 언어는 복잡한 규칙을 가집니다:

```javascript
// 언어별 복수형 규칙 (CLDR 기반)
const pluralRules = {
  // 영어: 2가지 형태
  en: {
    rules: ['one', 'other'],
    examples: {
      one: '1 item',
      other: '5 items'
    }
  },

  // 한국어: 1가지 형태 (복수 구분 없음)
  ko: {
    rules: ['other'],
    examples: {
      other: '1개 항목 / 5개 항목'
    }
  },

  // 아랍어: 6가지 형태
  ar: {
    rules: ['zero', 'one', 'two', 'few', 'many', 'other'],
    examples: {
      zero: 'لا عناصر',      // 0개
      one: 'عنصر واحد',       // 1개
      two: 'عنصران',          // 2개
      few: '٣ عناصر',         // 3-10개
      many: '١١ عنصرا',       // 11-99개
      other: '١٠٠ عنصر'      // 100+개
    }
  },

  // 폴란드어: 3가지 형태
  pl: {
    rules: ['one', 'few', 'many'],
    examples: {
      one: '1 element',           // 1
      few: '2 elementy',          // 2-4
      many: '5 elementów'         // 5+
    }
  },

  // 러시아어: 3가지 형태
  ru: {
    rules: ['one', 'few', 'many'],
    examples: {
      one: '1 элемент',           // 1, 21, 31...
      few: '2 элемента',          // 2-4, 22-24...
      many: '5 элементов'         // 5-20, 25-30...
    }
  }
};

// ❌ 나쁜 예: 영어 중심적 접근
function formatItems(count) {
  return count === 1 ? `${count} item` : `${count} items`;
}

// ✅ 좋은 예: CLDR 기반 접근
function formatItems(count, locale) {
  const pr = new Intl.PluralRules(locale);
  const rule = pr.select(count);
  return translations[locale].items[rule].replace('{count}', count);
}
```

```html
</div>
```

### 2.5.2 런타임 성능

```typescript
// 비효율적인 번역 조회
class SlowTranslationService {
  translate(key: string, locale: string): string {
    // ❌ 문제: 매번 파일 읽기
    const translations = JSON.parse(
      fs.readFileSync(`./locales/${locale}.json`, 'utf-8')
    );

    // ❌ 문제: 중첩된 객체 탐색
    const keys = key.split('.');
    let value = translations;
    for (const k of keys) {
      value = value[k];
    }

    return value;
  }
}

// ✅ 개선: 캐싱 및 최적화
class FastTranslationService {
  private cache: Map<string, any> = new Map();
  private flatCache: Map<string, string> = new Map();

  async loadLocale(locale: string): Promise<void> {
    if (!this.cache.has(locale)) {
      const translations = await import(`./locales/${locale}.json`);
      this.cache.set(locale, translations);

      // 평탄화하여 빠른 접근
      this.flattenTranslations(locale, translations);
    }
  }

  private flattenTranslations(locale: string, obj: any, prefix = ''): void {
    for (const [key, value] of Object.entries(obj)) {
      const fullKey = prefix ? `${prefix}.${key}` : key;
      if (typeof value === 'object') {
        this.flattenTranslations(locale, value, fullKey);
      } else {
        this.flatCache.set(`${locale}:${fullKey}`, value);
      }
    }
  }

  translate(key: string, locale: string): string {
    return this.flatCache.get(`${locale}:${key}`) || key;
  }
}

// 성능 비교

    throughput: '20/s'
  },
  fast: {
    time: '0.1ms',     // 번역당 0.1ms
    throughput: '10000/s'
  }
```
