Chapter 06: 접근성 및 포용적 디자인

WCAG 준수, 색각 이상 대응, 스크린 리더 지원으로 모두를 위한 시각화

1. 접근성의 중요성

전 세계 인구의 15% (10억 명)가 어떤 형태로든 장애를 가지고 있습니다 (WHO, 2021). 접근 가능한 데이터 시각화는 도덕적 의무이자 법적 요구사항이며, 더 나은 UX를 제공합니다.

접근성 통계 (글로벌)
• 2억 5천만 명: 시각 장애 (실명 포함)
• 3억 명: 색각 이상 (남성 8%, 여성 0.5%)
• 4억 6천만 명: 청각 장애
• 2억 명: 인지 장애
• 고령화: 2050년까지 60세 이상 20억 명 예상

법적 요구사항
• 미국: ADA (Americans with Disabilities Act)
• 유럽: EN 301 549, EAA (European Accessibility Act)
• 한국: 장애인차별금지법, 웹 접근성 품질인증
• 국제: WCAG 2.1 Level AA 준수

2. 색각 이상 (Color Blindness) 대응

남성의 8%, 여성의 0.5%가 색각 이상을 가지고 있습니다. 가장 흔한 유형은 적록색맹 (Deuteranopia/Protanopia, 6%)입니다.

마우스 사용이 어려운 사용자나 파워 유저를 위해 키보드만으로 모든 기능에 접근할 수 있어야 합니다.

// D3.js 키보드 내비게이션 const dataPoints = svg.selectAll('.data-point') .data(data) .enter() .append('circle') .attr('class', 'data-point') .attr('tabindex', 0) // 포커스 가능 .attr('role', 'button') .attr('aria-label', d => `${d.label}: ${d.value}`) .on('keydown', function(event, d) { const current = d3.select(this); const allPoints = d3.selectAll('.data-point').nodes(); const currentIndex = allPoints.indexOf(this); switch(event.key) { case 'ArrowRight': case 'ArrowDown': event.preventDefault(); // 다음 데이터 포인트로 이동 if (currentIndex < allPoints.length - 1) { allPoints[currentIndex + 1].focus(); } break; case 'ArrowLeft': case 'ArrowUp': event.preventDefault(); // 이전 데이터 포인트로 이동 if (currentIndex > 0) { allPoints[currentIndex - 1].focus(); } break; case 'Home': event.preventDefault(); // 첫 번째 데이터 포인트로 이동 allPoints[0].focus(); break; case 'End': event.preventDefault(); // 마지막 데이터 포인트로 이동 allPoints[allPoints.length - 1].focus(); break; case 'Enter': case ' ': // Space event.preventDefault(); // 선택 이벤트 트리거 selectDataPoint(d); break; case 'Escape': event.preventDefault(); // 선택 해제 clearSelection(); break; } }) .on('focus', function(event, d) { // 포커스 시각화 d3.select(this) .attr('stroke', '#8B5CF6') .attr('stroke-width', 3); // 툴팁 표시 showTooltip(d); }) .on('blur', function() { // 포커스 제거 d3.select(this) .attr('stroke', 'none'); // 툴팁 숨김 hideTooltip(); }); // 포커스 스타일 (CSS) .data-point:focus { outline: 3px solid #8B5CF6; outline-offset: 2px; } // 포커스 표시 숨기기 금지! /* ❌ 절대 하지 마세요: *:focus { outline: none; } */

5.1 WCAG 색상 대비 기준

콘텐츠 유형 WCAG AA WCAG AAA 예시
일반 텍스트 (18pt 미만) 4.5:1 7:1 #000 on #FFF: 21:1 ✅
대형 텍스트 (18pt+, 14pt bold+) 3:1 4.5:1 #8B5CF6 on #FFF: 4.56:1 ✅
UI 구성요소 (버튼, 입력) 3:1 N/A 차트 축, 그리드

// 색상 대비 계산 함수 function getLuminance(r, g, b) { const [rs, gs, bs] = [r, g, b].map(c => { c = c / 255; return c <= 0.03928 ? c / 12.92 : Math.pow((c + 0.055) / 1.055, 2.4); }); return 0.2126 * rs + 0.7152 * gs + 0.0722 * bs; } function getContrastRatio(color1, color2) { const lum1 = getLuminance(...color1); const lum2 = getLuminance(...color2); const lighter = Math.max(lum1, lum2); const darker = Math.min(lum1, lum2); return (lighter + 0.05) / (darker + 0.05); } // 사용 예시 const bgColor = [15, 23, 42]; // #0f172a const textColor = [248, 250, 252]; // #f8fafc const contrast = getContrastRatio(bgColor, textColor); console.log(`Contrast ratio: ${contrast.toFixed(2)}:1`); // Output: 15.68:1 (WCAG AAA ✅) // 자동으로 대비 충족 색상 선택 function getAccessibleTextColor(backgroundColor) { const bgLum = getLuminance(...backgroundColor); const whiteContrast = (1 + 0.05) / (bgLum + 0.05); const blackContrast = (bgLum + 0.05) / (0 + 0.05); return whiteContrast > blackContrast ? '#FFFFFF' : '#000000'; }

5.2 폰트 크기 및 가독성

  • 최소 폰트 크기: 12px (본문은 16px 권장)
  • 행간 (line-height): 1.5 이상 (긴 텍스트는 1.7-2.0)
  • 단락 너비: 45-75자 (이상적: 66자)
  • 자간 (letter-spacing): 대문자 텍스트는 0.05-0.1em
  • 텍스트 확대: 200%까지 확대 시 기능 유지

핵심 요점

  • 15% 인구가 장애: 접근성은 선택이 아닌 필수
  • WCAG 2.1 AA 준수: 법적 요구사항이자 best practice
  • 색각 이상 8%: 색상만으로 정보 전달 금지
  • Paul Tol 팔레트: 과학적으로 검증된 색맹 안전 색상
  • 패턴 + 색상: 줄무늬, 점무늬 등 텍스처 병행
  • ARIA 필수: role, aria-label, aria-describedby
  • 데이터 테이블 제공: 스크린 리더용 숨김 테이블
  • 키보드 내비게이션: 모든 기능을 Tab/Arrow로 접근
  • 색상 대비 4.5:1: WCAG AA 텍스트 기준
  • 자동화 테스트: axe-core CI/CD 통합
