2. D3.js - Data-Driven Documents

D3.js는 데이터 시각화의 사실상 표준 (de facto standard)입니다. SVG, Canvas, HTML을 직접 조작하여 무한한 가능성을 제공하지만 학습 곡선이 가파릅니다.

2.1 D3.js 핵심 개념

  • Data Join: 데이터와 DOM 요소를 바인딩하는 핵심 메커니즘
  • Enter-Update-Exit 패턴: 데이터 변경 시 DOM 업데이트 전략
  • Scales: 데이터 도메인을 시각적 범위로 매핑 (linear, log, time, ordinal)
  • Shapes: 라인, 영역, 호, 심볼 등을 생성하는 생성기
  • Forces: 물리 시뮬레이션 기반 레이아웃 (네트워크 그래프)
  • Transitions: 부드러운 애니메이션 효과
// D3.js 기본 패턴: Enter-Update-Exit const data = [10, 20, 30, 40, 50]; // SVG 요소 선택 const svg = d3.select('svg'); // Data Join: 데이터와 circle 요소 바인딩 const circles = svg.selectAll('circle') .data(data); // ENTER: 새로운 데이터 포인트에 대한 요소 생성 circles.enter() .append('circle') .attr('cx', (d, i) => i * 60 + 30) .attr('cy', 50) .attr('r', 0) // 시작 크기 0 .attr('fill', '#8B5CF6') .transition() // 애니메이션 .duration(800) .attr('r', d => d); // 데이터 값으로 크기 설정 // UPDATE: 기존 요소 업데이트 circles .transition() .duration(800) .attr('r', d => d) .attr('fill', '#3b82f6'); // EXIT: 제거된 데이터 포인트의 요소 삭제 circles.exit() .transition() .duration(800) .attr('r', 0) .remove();

2.2 D3.js Scales (스케일)

스케일은 데이터 값을 픽셀 좌표, 색상, 크기 등으로 변환하는 함수입니다.

// 1. Linear Scale: 연속형 → 연속형 const xScale = d3.scaleLinear() .domain([0, 100]) // 입력 범위 (데이터) .range([0, 800]); // 출력 범위 (픽셀) console.log(xScale(50)); // 400 // 2. Band Scale: 범주형 → 연속형 (Bar Chart에 최적) const yScale = d3.scaleBand() .domain(['A', 'B', 'C', 'D']) .range([0, 400]) .padding(0.1); console.log(yScale('B')); // 100 (대략) console.log(yScale.bandwidth()); // 각 밴드의 너비 // 3. Time Scale: 날짜 → 연속형 const timeScale = d3.scaleTime() .domain([new Date(2024, 0, 1), new Date(2024, 11, 31)]) .range([0, 1000]); // 4. Color Scale: 값 → 색상 const colorScale = d3.scaleSequential(d3.interpolateViridis) .domain([0, 100]); console.log(colorScale(50)); // 중간 값의 색상 // 5. Logarithmic Scale: 로그 스케일 const logScale = d3.scaleLog() .domain([1, 1000000]) .range([0, 500]); console.log(logScale(1000)); // 250 (대략)

2.3 D3.js 실전 예제: 인터랙티브 Bar Chart

// 데이터 const salesData = [ { product: '노트북', sales: 450 }, { product: '모니터', sales: 320 }, { product: '키보드', sales: 180 }, { product: '마우스', sales: 250 }, { product: '헤드셋', sales: 120 } ]; // 설정 const width = 800, height = 400; const margin = { top: 20, right: 30, bottom: 60, left: 80 }; const innerWidth = width - margin.left - margin.right; const innerHeight = height - margin.top - margin.bottom; // SVG 생성 const svg = d3.select('#chart') .append('svg') .attr('width', width) .attr('height', height); const g = svg.append('g') .attr('transform', `translate(${margin.left},${margin.top})`); // Scales const xScale = d3.scaleBand() .domain(salesData.map(d => d.product)) .range([0, innerWidth]) .padding(0.2); const yScale = d3.scaleLinear() .domain([0, d3.max(salesData, d => d.sales)]) .range([innerHeight, 0]) .nice(); // 막대 그리기 g.selectAll('.bar') .data(salesData) .enter() .append('rect') .attr('class', 'bar') .attr('x', d => xScale(d.product)) .attr('y', d => yScale(d.sales)) .attr('width', xScale.bandwidth()) .attr('height', d => innerHeight - yScale(d.sales)) .attr('fill', '#8B5CF6') .on('mouseover', function(event, d) { d3.select(this) .transition() .duration(200) .attr('fill', '#3b82f6'); // 툴팁 표시 tooltip.style('opacity', 1) .html(`${d.product}
판매량: ${d.sales}개`) .style('left', (event.pageX + 10) + 'px') .style('top', (event.pageY - 28) + 'px'); }) .on('mouseout', function() { d3.select(this) .transition() .duration(200) .attr('fill', '#8B5CF6'); tooltip.style('opacity', 0); }); // 축 추가 const xAxis = d3.axisBottom(xScale); const yAxis = d3.axisLeft(yScale); g.append('g') .attr('transform', `translate(0,${innerHeight})`) .call(xAxis) .selectAll('text') .style('font-size', '12px'); g.append('g') .call(yAxis); // 축 레이블 g.append('text') .attr('x', innerWidth / 2) .attr('y', innerHeight + 50) .attr('text-anchor', 'middle') .text('제품'); g.append('text') .attr('transform', 'rotate(-90)') .attr('x', -innerHeight / 2) .attr('y', -60) .attr('text-anchor', 'middle') .text('판매량 (개)'); // 툴팁 const tooltip = d3.select('body') .append('div') .style('position', 'absolute') .style('opacity', 0) .style('background', 'rgba(0,0,0,0.8)') .style('color', 'white') .style('padding', '10px') .style('border-radius', '5px') .style('pointer-events', 'none');
✅ D3.js 장점
• 완벽한 커스터마이징 가능
• 모든 종류의 차트 구현 가능 (네트워크, 지리 등)
• 강력한 데이터 조작 유틸리티
• 대규모 커뮤니티와 풍부한 예제
• SVG, Canvas 모두 지원

⚠️ D3.js 단점
• 가파른 학습 곡선
• 많은 코드 필요 (간단한 차트도)
• 번들 크기 큼 (전체 라이브러리 사용 시)
• 반응형 디자인 수동 구현 필요

3. Chart.js - 간편하고 아름다운

Chart.js는 가장 사용하기 쉬운 시각화 라이브러리입니다. 8가지 기본 차트 유형을 제공하며, 애니메이션과 반응형 디자인이 기본 내장되어 있습니다.

3.1 Chart.js 시작하기

// HTML: Canvas 요소 필요 // // Chart.js 기본 예제 const ctx = document.getElementById('myChart').getContext('2d'); const myChart = new Chart(ctx, { type: 'line', // bar, line, pie, doughnut, radar, polarArea, bubble, scatter data: { labels: ['1월', '2월', '3월', '4월', '5월', '6월'], datasets: [{ label: '월별 매출 (백만원)', data: [12, 19, 3, 5, 2, 3], backgroundColor: 'rgba(139, 92, 246, 0.2)', borderColor: 'rgba(139, 92, 246, 1)', borderWidth: 2, fill: true, tension: 0.4 // 곡선 부드러움 (0-1) }] }, options: { responsive: true, maintainAspectRatio: true, plugins: { title: { display: true, text: '2024 월별 매출 현황', font: { size: 18 } }, legend: { display: true, position: 'top' }, tooltip: { enabled: true, backgroundColor: 'rgba(0,0,0,0.8)', callbacks: { label: function(context) { return context.dataset.label + ': ' + context.parsed.y + '백만원'; } } } }, scales: { y: { beginAtZero: true, ticks: { callback: function(value) { return value + '백만'; } } } }, animation: { duration: 1000, easing: 'easeOutQuart' } } });

✅ Chart.js 장점

• 복잡한 차트 구현 어려움
• 대용량 데이터 성능 저하
• 8가지 차트 유형만 지원

4. Plotly.js - 과학 및 통계 시각화

Plotly.js는 과학, 공학, 금융 분야에서 선호되는 라이브러리입니다. 40+ 차트 유형, 3D 시각화, 통계 차트를 지원합니다.

4.1 Plotly.js 3D Surface Plot

// 3D Surface Plot 예제 const z_data = []; for (let i = 0; i < 50; i++) { z_data[i] = []; for (let j = 0; j < 50; j++) { const x = (i - 25) / 5; const y = (j - 25) / 5; z_data[i][j] = Math.sin(Math.sqrt(x*x + y*y)); } } const data = [{ z: z_data, type: 'surface', colorscale: 'Viridis', contours: { z: { show: true, usecolormap: true, highlightcolor: '#8B5CF6', project: { z: true } } } }]; const layout = { title: '3D 표면 플롯 - sin(√(x² + y²))', scene: { camera: { eye: { x: 1.87, y: 0.88, z: -0.64 } } }, autosize: true, margin: { l: 0, r: 0, b: 0, t: 40 } }; Plotly.newPlot('plotly-div', data, layout);

4.2 Plotly.js 통계 차트

// Box Plot with Multiple Groups const y0 = []; const y1 = []; for (let i = 0; i < 50; i++) { y0.push(Math.random() * 10 + 5); y1.push(Math.random() * 10 + 10); } const trace1 = { y: y0, type: 'box', name: '그룹 A', marker: { color: '#8B5CF6' }, boxmean: 'sd' // 평균과 표준편차 표시 }; const trace2 = { y: y1, type: 'box', name: '그룹 B', marker: { color: '#3b82f6' }, boxmean: 'sd' }; const layout = { title: '그룹별 데이터 분포 비교', yaxis: { title: '값', zeroline: true }, boxmode: 'group' }; Plotly.newPlot('box-plot', [trace1, trace2], layout);

4.3 Plotly.js Contour Plot

// 등고선 플롯 const size = 100; const x = []; const y = []; const z = []; for (let i = 0; i < size; i++) { x[i] = []; y[i] = []; z[i] = []; for (let j = 0; j < size; j++) { const xVal = (i - 50) / 10; const yVal = (j - 50) / 10; x[i][j] = xVal; y[i][j] = yVal; z[i][j] = Math.exp(-(xVal*xVal + yVal*yVal) / 10); } } const data = [{ z: z, x: x[0], y: y.map(row => row[0]), type: 'contour', colorscale: 'Electric', contours: { coloring: 'heatmap' } }]; const layout = { title: '2D 가우시안 분포 등고선', xaxis: { title: 'X' }, yaxis: { title: 'Y' } }; Plotly.newPlot('contour-plot', data, layout);
💡 Plotly.js 특화 기능
• 3D 차트: Surface, Scatter3D, Mesh3D
• 통계 차트: Box Plot, Violin Plot, 2D Histogram
• 과학 차트: Contour, Heatmap, Polar
• 금융 차트: Candlestick, OHLC, Funnel
• 지리 차트: Choropleth, Scattergeo
• 인터랙션: Zoom, Pan, Hover, Export (PNG/SVG)

5. ECharts - 엔터프라이즈급 시각화

Apache ECharts는 Baidu에서 개발한 엔터프라이즈급 시각화 라이브러리입니다. 대용량 데이터 처리, 복잡한 대시보드, 실시간 업데이트에 최적화되어 있습니다.

5.1 ECharts 기본 구조

7. 성능 최적화 전략

7.1 대용량 데이터 처리

  • Sampling: LTTB (Largest-Triangle-Three-Buckets) 알고리즘
  • Virtualization: 화면에 보이는 부분만 렌더링
  • Aggregation: 데이터 집계 후 시각화
  • Progressive Loading: 점진적 로딩
  • WebGL: Canvas 대신 WebGL 사용 (10배 빠름)

7.2 번들 크기 최적화

// D3.js: 필요한 모듈만 임포트 import { select } from 'd3-selection'; import { scaleLinear } from 'd3-scale'; import { axisBottom } from 'd3-axis'; // Chart.js: Tree-shaking 활용 import { Chart, LineController, LineElement, PointElement, LinearScale, CategoryScale } from 'chart.js'; Chart.register( LineController, LineElement, PointElement, LinearScale, CategoryScale ); // ECharts: Custom build import * as echarts from 'echarts/core'; import { LineChart, BarChart } from 'echarts/charts'; import { GridComponent, TooltipComponent } from 'echarts/components'; import { CanvasRenderer } from 'echarts/renderers'; echarts.use([ LineChart, BarChart, GridComponent, TooltipComponent, CanvasRenderer ]);

핵심 요점

  • D3.js는 완벽한 커스터마이징: 복잡한 차트, 네트워크, 지리 시각화에 최적
  • Chart.js는 속도와 단순함: 빠른 프로토타입, 간단한 차트에 최적
  • Plotly.js는 과학과 통계: 3D, Box Plot, 통계 차트에 최적
  • ECharts는 엔터프라이즈: 대시보드, 대용량 데이터, 실시간에 최적
  • 번들 크기 중요: Chart.js (60KB) vs Plotly.js (3MB)
  • 학습 곡선 고려: D3.js (2-3개월) vs Chart.js (1-2일)
  • Tree-shaking 활용: 필요한 모듈만 임포트하여 번들 크기 최소화
  • 성능 최적화: Sampling, Virtualization, WebGL 활용
  • 프레임워크 통합: React는 Recharts, Vue는 Vue-ECharts
  • 라이센스 확인: 상업적 사용 시 라이센스 검토 필수
