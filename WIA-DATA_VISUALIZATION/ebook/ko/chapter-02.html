Chapter 02: 차트 유형 및 선택 가이드

올바른 차트 선택은 데이터 인사이트 발견의 첫걸음입니다

1. 차트 선택의 중요성

동일한 데이터라도 어떤 차트를 선택하느냐에 따라 전달되는 메시지가 완전히 달라집니다. 잘못된 차트 선택은 데이터를 왜곡하거나 핵심 인사이트를 감출 수 있습니다.

차트 선택 황금률
1️⃣ 데이터 유형을 파악하라 (범주형, 연속형, 시계열, 관계형)
2️⃣ 목적을 명확히 하라 (비교, 추세, 분포, 구성, 관계)
3️⃣ 청중을 고려하라 (전문가 vs 일반인, 문화적 배경)
4️⃣ 단순함을 추구하라 (가장 간단한 차트가 최선)

2. 주요 차트 유형 분류

2.1 비교형 차트 (Comparison Charts)

여러 항목 간의 값을 비교하거나 순위를 파악할 때 사용합니다.

차트 유형 최적 사용 사례 데이터 개수 장점 단점

(막대 그래프)		 범주형 데이터 비교

Column Chart
(세로 막대)		 시간별 비교
카테고리 비교		 3-12개 시간 흐름 표현
음수 표현 가능		 레이블 공간 제한
Grouped Bar
(그룹 막대)		 다중 시리즈 비교
서브카테고리 비교		 2-4 시리즈
3-8 카테고리		 직접 비교 용이 시리즈 많으면 혼란

(누적 막대)		 전체 대비 부분 비교

시간에 따른 데이터 변화와 패턴을 분석할 때 사용합니다.

차트 유형 최적 사용 사례 데이터 포인트 핵심 특징

(선 그래프)		 시계열 추세
연속 데이터 변화		 10+ 포인트 패턴 파악 용이
다중 시리즈 비교 가능
Area Chart
(영역 그래프)		 누적 추세
볼륨 강조		 10+ 포인트 변화량 시각적 강조
전체 대비 부분 표현
Sparkline
(미니 차트)		 간단한 추세
대시보드 요약		 10-50 포인트 공간 효율적
여러 지표 동시 표시
Candlestick
(캔들스틱)		 주가 데이터

2.3 분포형 차트 (Distribution Charts)

데이터의 분포 패턴과 이상치를 탐색할 때 사용합니다.

차트 유형 목적 통계 정보 적용 분야
Histogram
(히스토그램)		 빈도 분포 분석 평균, 표준편차
왜도, 첨도		 통계 분석
품질 관리
Box Plot
(상자 그림)		 사분위수 분석
이상치 탐지		 Q1, Q2, Q3
IQR, 이상치		 데이터 비교
품질 검증
Violin Plot
(바이올린 플롯)		 분포 형태 시각화
밀도 추정		 KDE (커널 밀도)
사분위수		 다변량 분석
그룹 비교
Scatter Plot
(산점도)		 변수 간 관계
상관관계 분석		 상관계수
회귀선		 과학 연구
예측 모델링
⚠️ Box Plot 해석 주의사항
Box Plot은 통계적 지식이 필요합니다. 일반 청중에게는 히스토그램이나 간단한 막대 그래프가 더 효과적일 수 있습니다. 상자의 중앙선은 평균이 아닌 중앙값(median)입니다.

전체를 구성하는 부분의 비율과 기여도를 표현할 때 사용합니다.

차트 유형 사용 시기 권장 세그먼트 수 한계점

(원형 차트)		 간단한 비율 표현
(2-5개 항목)		 2-5개

3D 사용 금지
Donut Chart
(도넛 차트)		 중앙에 총합 표시

공간 낭비 가능
Treemap
(트리맵)		 계층적 구성
많은 항목 비교		 10-100개 작은 항목 읽기 어려움
색상 구분 필수
Sunburst
(선버스트)		 다층 계층 구조
드릴다운 탐색		 3-5 레벨 복잡도 높음
인터랙션 필요
🚫 Pie Chart 남용 금지
Pie Chart는 가장 많이 남용되는 차트입니다. 다음 상황에서는 사용하지 마세요:
• 7개 이상의 항목
• 정확한 비교가 필요할 때
• 시간에 따른 변화 표현
• 음수 값 포함
대안: 대부분의 경우 Bar Chart가 더 효과적입니다.

3. 차트 선택 의사결정 트리

데이터와 목적에 따라 최적의 차트를 선택하는 체계적인 접근법입니다.

차트 선택 프로세스

1단계: 데이터 유형 확인
├─ 범주형 (Categorical) → Bar, Column, Pie
├─ 시계열 (Time-series) → Line, Area, Candlestick
├─ 수치형 (Numerical) → Histogram, Box Plot, Scatter
└─ 관계형 (Relational) → Network, Sankey, Chord

2단계: 시각화 목적 확인
├─ 비교 (Comparison) → Bar, Column, Grouped Bar
├─ 구성 (Composition) → Pie, Stacked Bar, Treemap
├─ 분포 (Distribution) → Histogram, Box Plot, Violin
├─ 관계 (Relationship) → Scatter, Heatmap, Network
└─ 추세 (Trend) → Line, Area, Sparkline

3단계: 데이터 포인트 개수 확인
├─ 3-7개 → Pie, Bar
├─ 8-15개 → Bar, Column
├─ 16-50개 → Line, Histogram
└─ 50+ → Scatter, Heatmap, Treemap

4단계: 청중 고려
├─ 일반인 → 단순한 차트 (Bar, Line, Pie)
├─ 전문가 → 복잡한 차트 가능 (Box Plot, Network)
└─ 경영진 → 핵심만 강조 (Sparkline, KPI 카드)

4. 실전 사례 분석

4.1 잘못된 차트 선택 사례

❌ 사례 1: 3D Pie Chart
• 문제: 원근감으로 인한 왜곡, 정확한 비율 파악 불가
• 해결: 2D Bar Chart 사용

❌ 사례 2: 15개 항목 Pie Chart
• 문제: 작은 조각 구분 불가, 범례 읽기 어려움
• 해결: Treemap 또는 Sorted Bar Chart

❌ 사례 3: 이중 Y축 Line Chart
• 문제: 축 조작으로 관계 왜곡 가능
• 해결: 두 개의 분리된 차트 또는 정규화된 단일 축

❌ 사례 4: 누적 막대 + 비율 데이터
• 문제: 100%가 아닌 누적 합계로 혼란
• 해결: 100% Stacked Bar Chart

4.2 탁월한 차트 선택 사례

✅ 사례 1: Slope Chart (경사 차트)
두 시점 간의 순위 변화를 표현할 때 Line Chart보다 효과적

✅ 사례 2: Small Multiples
여러 카테고리를 비교할 때 하나의 복잡한 차트보다 여러 개의 작은 차트

✅ 사례 3: Bullet Chart
목표 대비 실적을 표현할 때 Gauge Chart보다 공간 효율적

✅ 사례 4: Calendar Heatmap
연간 일별 데이터를 표현할 때 GitHub contribution 스타일

5. 고급 차트 선택 기법

5.1 색상 활용 전략

  • 범주형 데이터: 구별 가능한 이산 색상 사용 (최대 10개)
  • 순차형 데이터: 단일 색조의 밝기 그라디언트
  • 발산형 데이터: 양극단 두 색상 + 중립 중간 색상
  • 강조형 데이터: 회색 배경 + 하나의 강조 색상
  • 색맹 대응: ColorBrewer 팔레트 사용, 색상 + 패턴 병행

5.2 축 스케일 선택

스케일 유형 사용 시기 장점 주의사항
Linear
(선형)		 일반적인 수치 데이터 직관적, 이해 쉬움 이상치에 민감
Logarithmic
(로그)		 넓은 범위의 데이터
(1~1,000,000)		 여러 크기 데이터 비교 0 포함 불가
일반인 이해 어려움
Square Root
(제곱근)		 넓은 범위 + 0 포함 0 포함 가능
로그보다 완만		 드물게 사용
Symlog
(대칭 로그)		 양수 + 음수 넓은 범위 0 기준 대칭
양/음 모두 로그		 해석 복잡

5.3 애니메이션 활용

차트 애니메이션은 데이터의 시간적 변화나 전환 과정을 효과적으로 전달합니다.

  • Enter 애니메이션: 데이터가 처음 나타날 때 (500-800ms)
  • Update 애니메이션: 데이터 값이 변경될 때 (300-500ms)
  • Exit 애니메이션: 데이터가 사라질 때 (200-400ms)
  • Transition 애니메이션: 차트 유형 전환 시 (800-1200ms)
⚠️ 애니메이션 과용 주의
• 애니메이션은 첫 로딩 시에만 사용
• 반복 애니메이션 금지 (접근성 문제)
• 사용자가 애니메이션을 건너뛸 수 있게 허용
• 전정기관 장애가 있는 사용자 고려 (vestibular disorders)

6. 차트 조합 및 하이브리드

복잡한 데이터를 표현하기 위해 여러 차트를 조합할 수 있습니다.

조합 사용 사례 구현 예시
Bar + Line 실적 + 목표선
판매량 + 성장률		 차트별 별도 Y축
또는 정규화된 단일 축
Scatter + Regression 데이터 포인트 + 추세선 최소제곱법 회귀선
Heatmap + Dendrogram 클러스터링 + 계층 구조 유전자 발현 분석
Map + Bar 지리적 분포 + 상세 수치 지역별 매출 대시보드

핵심 요점

  • 차트 선택이 메시지를 결정합니다: 동일한 데이터도 차트에 따라 다른 인사이트를 전달
  • 단순함이 최선입니다: 가장 간단한 차트가 가장 효과적인 차트
  • Pie Chart는 신중하게: 7개 이상 항목이면 Bar Chart 사용
  • 3D 차트는 금지: 왜곡과 혼란만 초래
  • 축 스케일에 주의: 로그 스케일은 명시적으로 표기
  • 색상은 의도적으로: 색맹 친화적 팔레트 사용
  • 애니메이션은 절제: 과용 시 접근성 문제
  • 청중을 고려: 일반인과 전문가는 다른 차트 필요
  • 데이터 포인트 수 확인: 3-7개는 Pie, 50+는 Scatter/Heatmap
  • 목적이 차트를 결정: 비교, 추세, 분포, 구성, 관계 중 선택
