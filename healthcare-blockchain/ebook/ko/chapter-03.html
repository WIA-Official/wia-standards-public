제3장: 환자 동의 체인

"진정한 환자 주권은 데이터 소유권에서 시작합니다. 누가, 언제, 어떤 목적으로, 어느 데이터에 접근할 수 있는지를 환자가 완전히 제어할 때, 의료는 진정으로 환자 중심이 됩니다."

— WIA-MED-024 환자 주권 원칙

3.1 현재 동의 시스템의 문제점

전통적인 의료 동의서는 환자 친화적이지 않으며, 실질적인 통제권을 제공하지 못합니다.

⚠️ 주요 문제점

문제 현황 영향
포괄적 동의 "모든 의료 정보 제공 동의" 일괄 체크 세분화된 통제 불가
복잡한 법률 용어 평균 동의서 20페이지, 전문 용어 난무 환자 이해도 30% 미만
철회 어려움 병원 방문, 서류 작성 필요 실질적 철회율 5% 미만
추적 불가 누가 언제 데이터에 접근했는지 모름 무단 접근 감지 불가
제3자 공유 보험사, 연구기관에 자동 공유 환자 인지 없이 데이터 유출

실제 사례: 2018년 영국 NHS 데이터 판매 스캔들에서 환자 동의 없이 160만 명의 의료 기록이 제약회사와 보험사에 판매되어 큰 논란이 일었습니다.

3.2 블록체인 기반 동의 관리

블록체인은 투명하고, 세분화되며, 환자가 완전히 제어할 수 있는 동의 시스템을 가능하게 합니다.

🔑 핵심 원칙

  • 세분화된 권한 (Granular Permissions) — 진단서만, 검사 결과만, 특정 기간만 등 세밀한 제어
  • 시간 제한 (Time-Bound) — "3개월 동안만", "응급 상황 시에만" 등 조건부 접근
  • 투명한 감사 추적 — 모든 접근이 블록체인에 기록되어 환자가 실시간 확인
  • 즉시 철회 (Instant Revocation) — 스마트폰 앱에서 클릭 한 번으로 즉시 권한 취소
  • 목적 기반 접근 (Purpose-Based Access) — "진료 목적", "연구 목적" 등 명확한 사용 목적 명시

3.3 스마트 계약 기반 접근 제어

스마트 계약은 동의 규칙을 자동으로 실행하여 인간 개입 없이 접근 제어를 보장합니다.

🤖 스마트 계약 예시 (Solidity-style Pseudocode)

contract PatientConsent {
  // 동의 구조체
  struct Consent {
    address patient;
    address provider;
    string[] dataCategories;
    uint validUntil;
    bool isActive;
  }

  // 환자 주소 → (제공자 주소 → 동의 내역)
  mapping(address => mapping(address => Consent)) consents;

  // 동의 부여
  function grantConsent(
    address provider,
    string[] dataCategories,
    uint durationDays
  ) public {
    require(msg.sender == patient, "환자만 동의 부여 가능");

    consents[msg.sender][provider] = Consent({
      patient: msg.sender,
      provider: provider,
      dataCategories: dataCategories,
      validUntil: block.timestamp + (durationDays * 1 days),
      isActive: true
    });

    emit ConsentGranted(msg.sender, provider, dataCategories);
  }

  // 접근 권한 확인
  function checkAccess(
    address patient,
    address provider,
    string dataCategory
  ) public view returns (bool) {
    Consent memory consent = consents[patient][provider];

    // 조건 확인
    if (!consent.isActive) return false;
    if (block.timestamp > consent.validUntil) return false;
    if (!contains(consent.dataCategories, dataCategory)) return false;

    return true;
  }

  // 동의 철회
  function revokeConsent(address provider) public {
    require(msg.sender == patient, "환자만 동의 철회 가능");
    consents[msg.sender][provider].isActive = false;
    emit ConsentRevoked(msg.sender, provider);
  }
}

🔄 동의 워크플로우

단계별 프로세스

  1. 환자가 동의 부여: 모바일 앱에서 의사/병원 선택, 데이터 범주 및 기간 설정
  2. 스마트 계약 실행: 블록체인에 동의 기록 저장, 이벤트 발생
  3. 의사가 데이터 요청: API 호출로 환자 데이터 접근 시도
  4. 접근 권한 검증: 스마트 계약이 자동으로 조건 확인 (시간, 데이터 범주, 목적 등)
  5. 접근 허용/거부: 조건 충족 시 데이터 제공, 아니면 거부
  6. 접근 로그 기록: 모든 접근 시도가 블록체인에 기록됨

📊 권한 매트릭스

기본 정보 ✓ 읽기 ✓ 읽기 ✓ 읽기 ✗ 익명화만

환자 통제: 환자는 스마트폰 앱에서 언제든지 이 권한 매트릭스를 수정할 수 있으며, 변경 사항은 즉시 블록체인에 기록되어 실시간 적용됩니다.

3.5 조건부 동의 (Conditional Consent)

스마트 계약은 복잡한 조건을 자동으로 처리할 수 있습니다.

🎯 조건 유형

  • 시간 조건 — "오전 9시~오후 6시만", "주말 제외" 등
  • 위치 조건 — "서울대병원 내에서만" (GPS 기반)
  • 응급 조건 — "응급 상황 시 모든 의사 접근 허용" (자동 판단)
  • 목적 조건 — "진료 목적만", "연구 목적은 별도 승인 필요"
  • 재공유 금지 — "이 데이터를 제3자에게 전달 금지"

💡 실제 사례

사례 1: 응급 상황 자동 권한

환자가 응급실에 의식 불명으로 도착하면, 스마트 계약이 자동으로 "응급 조건"을 감지하여 모든 필수 의료 정보에 대한 접근 권한을 응급의학과 의사들에게 부여합니다. 환자가 회복되면 자동으로 일반 권한 수준으로 복귀합니다.

사례 2: 시한부 연구 동의

환자가 "암 연구 프로젝트"에 익명화된 유전자 데이터 제공을 동의했습니다. 스마트 계약은 자동으로 6개월 후 동의를 만료시키고, 연구기관은 데이터를 삭제해야 합니다. 삭제 증명도 블록체인에 기록됩니다.

3.6 접근 감사 추적

모든 접근 시도가 블록체인에 기록되어 환자는 실시간으로 누가 자신의 데이터를 보았는지 확인할 수 있습니다.

📋 접근 로그 예시

시간 접근자 데이터 목적 결과
2025-01-15 09:30 김철수 의사 진단서 진료 ✓ 허용
2025-01-15 14:20 ABC보험사 정신건강 기록 보험 심사 ✗ 거부
2025-01-16 11:00 이영희 간호사 검사 결과 환자 케어 ✓ 허용
2025-01-17 02:45 응급실 의사 전체 기록 응급 처치 ✓ 응급 권한

📱 환자 대시보드

환자는 모바일 앱에서 다음 정보를 실시간으로 확인할 수 있습니다:

  • 접근 타임라인 — 시간순으로 누가 언제 접근했는지
  • 접근 통계 — 가장 많이 접근한 의사, 데이터 범주별 접근 횟수
  • 거부 알림 — 권한 없는 접근 시도 실시간 알림
  • 동의 현황 — 현재 활성화된 모든 동의 목록

3.7 GDPR 및 법적 준수

블록체인 동의 시스템은 GDPR(유럽 개인정보 보호법) 등 주요 개인정보 보호 규정을 완벽히 준수합니다.

⚖️ GDPR 요구사항 충족

  • 명시적 동의 — 모든 동의는 명확하고 구체적이어야 함 ✓
  • 쉬운 철회 — 동의 부여만큼 쉽게 철회 가능해야 함 ✓
  • 데이터 이동권 — 환자가 자신의 데이터를 다른 시스템으로 이동 가능 ✓
  • 접근 권리 — 환자가 자신의 데이터에 언제든지 접근 가능 ✓
  • 잊힐 권리 — 데이터 삭제 요청 시 준수 (오프체인 데이터 삭제 + 온체인 포인터 무효화) ✓

3.8 다음 장 미리보기

환자 동의 체인으로 데이터 접근 제어는 완벽해졌지만, 의약품의 진위와 안전성도 중요합니다. 다음 장에서는 의약품 공급망 추적을 다룹니다:

  • 제조사부터 환자까지 전체 추적
  • 위조약 방지 메커니즘
  • 온도/습도 등 보관 조건 모니터링
  • 리콜 및 회수 자동화
