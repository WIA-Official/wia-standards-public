제2장: 의료 기록 불변성

"의료 기록의 불변성은 신뢰의 기초입니다. 한 번 기록된 진단, 처방, 검사 결과는 영원히 변조될 수 없어야 하며, 모든 변경 사항은 투명하게 추적되어야 합니다."

— WIA-MED-024 데이터 무결성 원칙

2.1 의료 기록 변조의 위험성

의료 기록 변조는 환자 안전, 법적 책임, 보험 사기 등 심각한 문제를 야기합니다.

⚠️ 실제 사례

사건 내용 결과
의료과실 은폐 수술 중 실수 후 의료 기록 사후 수정 환자 사망, 의사 면허 취소
보험 사기 진단명을 더 심각하게 변경하여 보험금 과다 청구 $수백만 손실, 기관 폐쇄
약물 과다처방 은폐 중독성 약물 처방 기록 삭제 환자 약물 중독, 의사 형사 고발
연구 데이터 조작 임상시험 부작용 데이터 삭제 약물 승인 취소, 수백억 손해배상

🎯 불변성의 핵심 요구사항

  • 타임스탬프 보장 — 정확한 기록 시간 증명
  • 변조 불가 — 기록된 데이터의 사후 수정 방지
  • 완전한 감사 추적 — 모든 변경 사항 영구 기록
  • 법적 증거력 — 법정에서 인정되는 증명력
  • 장기 보존 — 수십 년 후에도 검증 가능

2.2 암호화 해싱 메커니즘

블록체인은 암호화 해시 함수를 사용하여 데이터 무결성을 보장합니다.

🔐 해시 함수란?

해시 함수는 임의 크기의 데이터를 고정된 크기의 고유한 "지문"으로 변환하는 수학 함수입니다:

입력: "환자 홍길동, 혈압 120/80, 2025-01-15 09:30"

SHA-256 해시: 3f8a9b2c...

단 하나의 문자라도 변경되면:

입력: "환자 홍길동, 혈압 130/80, 2025-01-15 09:30"

SHA-256 해시: 9d2f1a6e... (완전히 다름)

✨ 해시 함수의 특성

  • 결정론적(Deterministic) — 같은 입력은 항상 같은 해시 생성
  • 빠른 계산 — 해시 생성은 밀리초 이내
  • 일방향(One-way) — 해시로부터 원본 데이터 복구 불가능
  • 충돌 저항(Collision Resistant) — 서로 다른 입력이 같은 해시를 생성할 확률 거의 0
  • 눈사태 효과(Avalanche Effect) — 미세한 입력 변화로 해시 완전히 변경

🏥 의료 기록 해싱 프로세스

단계별 프로세스

  1. 데이터 생성: 의사가 진료 기록 작성 (진단명, 처방, 검사 결과 등)
  2. 정규화: 데이터를 표준 형식(JSON, FHIR)으로 변환
  3. 해싱: SHA-256 알고리즘으로 해시 생성
  4. 타임스탬핑: 신뢰할 수 있는 타임스탬프 서비스(TSA)로 시간 기록
  5. 블록체인 기록: 해시 + 타임스탬프를 블록체인에 저장
  6. 원본 저장: 실제 의료 데이터는 암호화되어 오프체인(IPFS, 병원 DB) 저장

왜 오프체인 저장? 블록체인에 모든 의료 데이터를 직접 저장하면 비용이 매우 높고 확장성이 떨어집니다. 따라서 블록체인에는 "지문(해시)"만 저장하고, 실제 데이터는 암호화되어 안전한 외부 저장소에 보관합니다.

2.3 블록체인 체인 구조

블록체인은 각 블록이 이전 블록의 해시를 포함하여 연결되는 구조입니다:

🔗 체인 구조 예시

블록 #1 (Genesis Block)

├─ 이전 블록 해시: 0x000000...

├─ 타임스탬프: 2025-01-01 00:00:00

├─ 데이터: [환자 등록 기록 해시]

└─ 블록 해시: 0xabc123...

블록 #2

├─ 이전 블록 해시: 0xabc123...

├─ 타임스탬프: 2025-01-15 09:30:00

├─ 데이터: [진료 기록 해시]

└─ 블록 해시: 0xdef456...

블록 #3

├─ 이전 블록 해시: 0xdef456...

├─ 타임스탬프: 2025-01-20 14:00:00

├─ 데이터: [검사 결과 해시]

└─ 블록 해시: 0xghi789...

🛡️ 변조 방지 메커니즘

악의적인 공격자가 블록 #2의 진료 기록을 변조하려고 시도하면:

⚠️ 변조 시나리오

  1. 공격자가 블록 #2의 데이터 변경 → 블록 #2의 해시가 변경됨
  2. 블록 #3은 블록 #2의 원래 해시를 참조 → 해시 불일치 감지
  3. 공격자가 블록 #3도 수정하려고 시도 → 블록 #4, #5... 이후 모든 블록도 수정해야 함
  4. 네트워크의 다른 노드들이 원본 체인 보유 → 합의 알고리즘이 공격자의 체인 거부

결론: 블록체인에 기록된 의료 데이터를 변조하려면 전체 네트워크의 51% 이상을 장악해야 하며, 이는 사실상 불가능합니다 (특히 퍼블릭 또는 대규모 컨소시엄 블록체인의 경우).

2.4 감사 추적 및 버전 관리

의료 기록은 시간에 따라 업데이트됩니다. 불변성은 "변경 불가"가 아니라 "모든 변경 이력 추적"을 의미합니다.

📋 버전 관리 모델

버전 타임스탬프 변경 내용 블록 해시
v1.0 2025-01-15 09:30 초기 진료 기록 생성 0xabc123...
v1.1 2025-01-15 14:00 검사 결과 추가 0xdef456...
v1.2 2025-01-16 10:00 진단명 수정 (의사 A) 0xghi789...
v1.3 2025-01-17 11:30 처방 추가 (의사 B) 0xjkl012...

각 버전은 독립적인 블록으로 기록되며, 이전 버전을 참조합니다. 따라서:

  • 완전한 이력 보존 — 과거 어느 시점의 상태로도 복원 가능
  • 변경자 추적 — 누가 언제 무엇을 변경했는지 명확
  • 법적 증거 — 의료 분쟁 시 정확한 시점의 기록 제시 가능

┌─ API 게이트웨이 ─────────┐

│ FHIR API, REST API │

│ 인증/권한 검증 │

└──────────┬─────────────┘

│

┌──────┴──────┐

│ │

│- 해시 │ │- 암호화 데이터│

🔧 구현 예시 (Pseudocode)

// 1. 의료 기록 생성
function createMedicalRecord(patientId, data) {
  // 2. 데이터 정규화 (FHIR 형식)
  fhirData = convertToFHIR(data)

  // 3. 암호화
  encryptedData = encrypt(fhirData, patientPublicKey)

  // 4. 오프체인 저장
  ipfsHash = IPFS.store(encryptedData)

  // 5. 데이터 해싱
  recordHash = SHA256(fhirData + timestamp)

  // 6. 블록체인에 기록
  txHash = blockchain.createRecord({
    patientId: patientId,
    recordHash: recordHash,
    ipfsHash: ipfsHash,
    timestamp: now(),
    doctorSignature: sign(recordHash, doctorPrivateKey)
  })

  return txHash
}

// 검증
function verifyMedicalRecord(txHash) {
  // 1. 블록체인에서 해시 가져오기
  record = blockchain.getRecord(txHash)

  // 2. 오프체인에서 실제 데이터 가져오기
  encryptedData = IPFS.get(record.ipfsHash)
  decryptedData = decrypt(encryptedData, patientPrivateKey)

  // 3. 해시 재계산 및 비교
  calculatedHash = SHA256(decryptedData + record.timestamp)

  if (calculatedHash === record.recordHash) {
    return "✓ 데이터 무결성 검증 완료"
  } else {
    throw "✗ 데이터 변조 감지!"
  }
}

2.6 타임스탬프 서비스

정확한 시간 증명은 의료 기록의 법적 효력에 필수적입니다.

⏰ 신뢰할 수 있는 타임스탬프

  • RFC 3161 표준 — 국제 표준 타임스탬프 프로토콜
  • TSA (Time Stamping Authority) — 공인된 제3자 시간 증명 기관
  • 블록체인 타임스탬프 — 블록 생성 시간을 합의 알고리즘으로 검증
  • GPS/NTP 동기화 — 원자시계 기반 정확한 시간

법적 중요성: 의료 분쟁 시 "언제" 진료가 이루어졌고, "언제" 기록이 작성되었는지가 판결의 핵심 증거가 됩니다. 블록체인의 타임스탬프는 변조가 불가능하여 법정에서 강력한 증거력을 가집니다.

2.7 실제 사례: MedRec (MIT)

MedRec은 MIT에서 개발한 블록체인 기반 의료 기록 시스템입니다:

📱 주요 특징

  • Ethereum 기반 — 스마트 계약으로 접근 제어 자동화
  • 환자 중심 설계 — 환자가 모든 의료 기록 접근 권한 관리
  • 포인터 시스템 — 블록체인에는 포인터만, 실제 데이터는 병원 DB에 저장
  • 감사 추적 — 모든 접근이 블록체인에 기록되어 투명성 보장

📊 성과

  • 베스 이스라엘 병원(Beth Israel Deaconess Medical Center)에서 파일럿 테스트 성공
  • 환자 만족도 92% 증가
  • 의료 기록 조회 시간 70% 단축
  • 데이터 무결성 검증 100% 성공

2.8 다음 장 미리보기

의료 기록의 불변성은 확보했지만, 누가 이 기록에 접근할 수 있는지를 결정하는 것도 중요합니다. 다음 장에서는 환자 동의 체인을 다룹니다:

  • 블록체인 기반 동의 관리 시스템
  • 세분화된 권한 부여 (granular permissions)
  • 시간 제한 접근 및 동적 철회
  • 스마트 계약 기반 자동화
