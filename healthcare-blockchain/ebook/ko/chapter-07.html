제7장: 탈중앙화 신원 관리

"신원은 의료의 기반입니다. 잘못된 환자 신원은 생명을 위협하고, 신원 도용은 보험 사기로 이어집니다. 자기주권 신원은 환자가 자신의 신원을 완전히 통제하면서도, 필요할 때 안전하게 증명할 수 있게 합니다."

— WIA-MED-024 신원 주권 원칙

7.1 환자 신원 오인의 위험성

환자 신원 오인은 의료 사고의 주요 원인 중 하나입니다.

⚠️ 충격적인 통계

문제 빈도 영향
환자 오인 사고 연간 수천 건 (미국) 잘못된 치료, 약물 투여
의료 신원 도용 연간 250만 건 (미국) 보험 사기, 의료 기록 오염
중복 기록 환자의 10~20% 불완전한 의료 이력, 중복 검사
경제적 손실 $60억/년 (미국) 사기, 중복 기록 정리 비용

실제 사례: 2015년 미국에서 한 환자가 동명이인과 혼동되어 잘못된 심장 수술을 받았습니다. 수술 도중 오류가 발견되었지만 이미 늦었습니다.

🎯 현재 신원 시스템의 문제점

  • 중앙화된 데이터베이스 — 해킹 시 수백만 명 정보 유출
  • 불일치 — 각 병원마다 다른 환자 ID 사용
  • 취약한 인증 — 이름 + 생년월일만으로 신원 확인
  • 국경 간 문제 — 해외에서 한국 의료 기록 접근 불가

7.2 자기주권 신원 (Self-Sovereign Identity)

SSI (Self-Sovereign Identity)는 개인이 자신의 신원을 완전히 소유하고 통제하는 패러다임입니다.

🔑 SSI의 핵심 원칙

  1. 존재 (Existence): 사용자는 독립적인 존재로서 신원을 가짐
  2. 통제 (Control): 사용자가 자신의 신원을 완전히 통제
  3. 접근 (Access): 사용자는 언제든지 자신의 데이터에 접근
  4. 투명성 (Transparency): 시스템이 어떻게 작동하는지 명확
  5. 영속성 (Persistence): 신원은 오래 지속됨
  6. 이동성 (Portability): 신원 정보를 다른 시스템으로 이동 가능
  7. 상호운용성 (Interoperability): 전 세계 어디서나 사용 가능
  8. 동의 (Consent): 데이터 공유는 명시적 동의 필요
  9. 최소화 (Minimization): 필요한 최소한의 정보만 공개
  10. 보호 (Protection): 사용자의 권리가 보호됨

🏗️ SSI 아키텍처

1. DID (Decentralized Identifier)

did:wia:med:0x9f8e7d6c5b4a3210

│ 환자의 고유한 글로벌 ID

▼

2. DID Document (블록체인에 저장)

│ 공개키, 인증 방법, 서비스 엔드포인트

▼

3. Verifiable Credentials (검증 가능한 자격증명)

│ 병원이 발급한 진단서, 백신 증명서 등

▼

4. Digital Wallet (모바일 앱)

└─ 환자가 개인키와 자격증명 보관

7.3 탈중앙화 식별자 (DID)

DID는 중앙 기관 없이도 검증 가능한 고유한 식별자입니다.

📋 DID 구조

did:wia:med:0x9f8e7d6c5b4a3210

did: 스킴 (항상 "did")
wia: DID 메소드 (WIA 표준)
med: 네트워크 (의료 네트워크)
0x9f... 고유 식별자 (블록체인 주소)

📄 DID Document 예시

{
  "@context": "https://www.w3.org/ns/did/v1",
  "id": "did:wia:med:0x9f8e7d6c5b4a3210",
  "authentication": [{
    "id": "did:wia:med:0x9f8e...#keys-1",
    "type": "EcdsaSecp256k1VerificationKey2019",
    "controller": "did:wia:med:0x9f8e...",
    "publicKeyHex": "0x02b97c3..."
  }],
  "service": [{
    "id": "did:wia:med:0x9f8e...#medicalRecords",
    "type": "HealthDataService",
    "serviceEndpoint": "https://patient-vault.example.com"
  }],
  "created": "2025-01-15T00:00:00Z",
  "updated": "2025-01-15T00:00:00Z"
}

7.4 검증 가능한 자격증명 (Verifiable Credentials)

VC (Verifiable Credentials)는 암호학적으로 검증 가능한 디지털 증명서입니다.

🎓 VC 사용 사례

자격증명 발급자 사용처
진단서 담당 의사 다른 병원, 보험사
백신 증명서 접종 병원 항공사, 국경 검문
혈액형 카드 검사 기관 응급실
처방전 의사 약국
의사 면허 의료 협회 병원, 환자

📋 VC 구조 예시

{
  "@context": ["https://www.w3.org/2018/credentials/v1"],
  "type": ["VerifiableCredential", "VaccinationCredential"],
  "issuer": "did:wia:med:hospital:seoul-central",
  "issuanceDate": "2025-01-15T10:00:00Z",
  "credentialSubject": {
    "id": "did:wia:med:0x9f8e7d6c5b4a3210",
    "givenName": "길동",
    "familyName": "홍",
    "vaccinationType": "COVID-19",
    "vaccineName": "Pfizer-BioNTech",
    "doseNumber": 2,
    "vaccinationDate": "2025-01-15"
  },
  "proof": {
    "type": "EcdsaSecp256k1Signature2019",
    "created": "2025-01-15T10:00:00Z",
    "proofPurpose": "assertionMethod",
    "verificationMethod": "did:wia:med:hospital:seoul-central#keys-1",
    "jws": "eyJhbGciOiJFUzI1NksiLCJiNjQiOmZhbHNlLCJjcml0IjpbImI2NCJdfQ..."
  }
}

🔍 검증 프로세스

  1. 환자가 VC 제시: 백신 증명서를 QR 코드로 제시
  2. 발급자 DID 조회: 블록체인에서 병원의 DID Document 가져오기
  3. 서명 검증: 병원의 공개키로 디지털 서명 확인
  4. 유효성 확인: 만료일, 철회 여부 확인
  5. 결과 반환: ✓ 유효 / ✗ 무효

7.5 생체인증 통합

DID와 생체인증을 결합하여 최고 수준의 보안을 달성합니다.

🔐 생체인증 방법

방법 정확도 사용 사례
지문 99.9% 모바일 앱 로그인
얼굴인식 99.5% 병원 키오스크, 약국
홍채 99.99% 높은 보안 필요 (수술실 접근)
정맥 패턴 99.95% 병원 직원 인증

🎯 블록체인 + 생체인증 워크플로우

  • 등록: 환자가 지문 등록 → 해시 생성 → DID Document에 저장
  • 인증: 지문 스캔 → 해시 계산 → 블록체인의 해시와 비교
  • 개인정보 보호: 실제 생체 데이터는 로컬 저장, 블록체인에는 해시만 기록

보안 강화: 공격자가 환자의 개인키를 훔쳐도, 생체인증 없이는 신원을 도용할 수 없습니다. 이중 인증으로 최고 수준의 보안을 달성합니다.

7.6 국경 간 의료 서비스

DID는 국경을 초월합니다. 환자가 전 세계 어디를 가도 동일한 디지털 신원을 사용할 수 있습니다.

🌍 글로벌 사용 시나리오

시나리오 1: 해외 여행 중 응급 상황

한국인 환자가 미국 여행 중 응급실 방문. DID 제시 → 한국 의료 기록 즉시 조회 → 알레르기, 기저질환 확인 → 안전한 치료

시나리오 2: 의료 관광

태국에서 성형 수술 예정. 한국에서 받은 검사 결과를 VC로 제시 → 태국 병원이 검증 → 중복 검사 불필요 → 비용 및 시간 절감

시나리오 3: 난민 의료 지원

전쟁으로 모든 서류 분실. 하지만 DID는 블록체인에 기록되어 있어 어디서든 신원 증명 가능 → 의료 지원 신속 제공

7.7 개인정보 보호 기술

DID는 투명하지만, 개인정보는 철저히 보호되어야 합니다.

🔐 개인정보 보호 메커니즘

  • 선택적 공개 (Selective Disclosure): VC의 일부 필드만 선택적으로 공개 (예: 나이만 공개, 생년월일은 숨김)
  • 영지식 증명 (Zero-Knowledge Proof): "18세 이상입니다"를 정확한 나이 공개 없이 증명
  • 익명 자격증명 (Anonymous Credentials): 신원 공개 없이 자격 증명 (예: "백신 접종 완료자"임만 증명)
  • Pairwise DIDs: 각 관계마다 다른 DID 사용하여 추적 방지

  • 중복 기록 — 20% → 1% 미만
