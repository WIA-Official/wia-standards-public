Chapter 1: 빅데이터 개요

빅데이터(Big Data)는 21세기 디지털 혁명의 핵심 동력입니다. 전통적인 데이터 처리 도구로는 수집, 저장, 관리, 분석이 어려운 대규모 데이터 집합을 의미하며, 현대 비즈니스와 과학 연구의 필수적인 자원으로 자리잡았습니다. 본 장에서는 빅데이터의 개념, 특성, 그리고 발전사를 상세히 살펴봅니다.

1.1 빅데이터의 정의

빅데이터는 단순히 데이터의 양(Volume)만을 의미하지 않습니다. 다양한 형태의 데이터가 빠른 속도로 생성되고, 이를 효과적으로 분석하여 가치를 창출하는 전체 생태계를 포함합니다. Gartner에 따르면, 빅데이터는 "더 나은 의사결정과 통찰력 발견, 프로세스 최적화를 위해 새로운 형태의 처리를 필요로 하는 대용량, 고속, 다양한 정보 자산"으로 정의됩니다.

1.2 빅데이터의 3V 특성

일부 학자들은 7V, 10V 모델까지 제안합니다. 추가 특성으로는 Visualization(시각화), Viability(실현 가능성), Vulnerability(취약성), Volatility(변동성) 등이 있습니다. 하지만 5V 모델이 가장 널리 채택되고 있습니다.

1.4 빅데이터 발전사

빅데이터의 역사는 데이터 저장 및 처리 기술의 발전과 밀접하게 연결되어 있습니다. 1960년대 데이터베이스의 등장부터 현재의 클라우드 기반 분산 처리까지, 기술 발전의 흐름을 살펴봅니다.

1960년대 - 데이터베이스의 탄생

IBM의 IMS 데이터베이스 시스템 개발. 계층형 및 네트워크형 데이터 모델 등장. 아폴로 우주 프로그램을 위한 대용량 데이터 관리 시작.

1970년대 - 관계형 데이터베이스

Edgar Codd의 관계형 모델 이론 발표. Oracle, IBM DB2 등 RDBMS 탄생. SQL 언어의 표준화 시작.

1980-90년대 - 데이터 웨어하우스

데이터 웨어하우스 개념 정립 (Bill Inmon). OLAP(Online Analytical Processing) 등장. ETL 프로세스 표준화.

2000년대 초반 - 웹 2.0과 데이터 폭발

Google의 MapReduce 논문 발표(2004). Doug Cutting의 Hadoop 프로젝트 시작(2006). NoSQL 데이터베이스 등장.

2010년대 - 빅데이터 시대 본격화

Apache Spark 등장(2014). 실시간 스트리밍 처리 기술 발전. 클라우드 빅데이터 서비스(AWS EMR, Azure HDInsight 등) 확산.

2020년대 - AI와 빅데이터 융합

데이터 레이크하우스 아키텍처 등장. MLOps/DataOps 실천. 실시간 AI 추론과 빅데이터의 통합.

빅데이터 프로젝트 실패 요인

Gartner에 따르면 빅데이터 프로젝트의 약 85%가 실패합니다. 주요 원인으로는 불명확한 비즈니스 목표, 데이터 품질 문제, 적절한 인재 부족, 조직 문화의 저항, 그리고 기술 중심적 접근 방식 등이 있습니다. 성공적인 빅데이터 이니셔티브를 위해서는 기술만큼이나 전략, 조직, 문화적 요소가 중요합니다.
