2장: 데이터 레이크 및 웨어하우스 표준

데이터 레이크 기본 개념

데이터 레이크는 모든 구조화, 반구조화 및 비구조화 데이터를 어떤 규모로든 저장하는 중앙 집중식 저장소입니다. WIA-BIG_DATA 표준은 조직 전체에서 일관성, 검색 가능성 및 거버넌스를 보장하기 위한 데이터 레이크 구현에 대한 특정 요구사항을 정의합니다.

로드하기 전에 데이터를 구조화해야 하는 기존 데이터 웨어하우스와 달리 데이터 레이크는 스키마 온 리드(schema-on-read) 접근 방식을 사용하여 원시 데이터를 네이티브 형식으로 저장하고 분석을 위해 액세스할 때만 변환합니다.

데이터 레이크 영역 아키텍처

표준 영역 레이아웃

  • 랜딩 영역: 수신 데이터를 위한 임시 스테이징 영역
  • 원시 영역: 원본 데이터의 불변 저장소
  • 큐레이션 영역: 정제, 검증 및 보강된 데이터
  • 소비 영역: 분석 준비가 완료된 데이터셋 및 집계
  • 샌드박스 영역: 실험적 및 애드혹 분석 영역

파티셔닝 전략

적절한 파티셔닝은 데이터 레이크의 쿼리 성능에 매우 중요합니다. WIA-BIG_DATA 표준은 데이터 특성 및 쿼리 패턴에 기반한 파티셔닝 모범 사례를 정의합니다.

파티셔닝 가이드라인

  • 시간 기반: 날짜/시간 쿼리가 있는 시계열 데이터에 사용
  • 해시 기반: 균등 분배 및 조인 최적화에 사용
  • 범위 기반: 범위 스캔이 있는 정렬된 데이터에 사용
  • 복합: 복잡한 액세스 패턴을 위한 전략 결합
# PySpark에서 파티셔닝 전략 구현
from pyspark.sql import SparkSession
from pyspark.sql.functions import year, month, day, hour

spark = SparkSession.builder.appName("PartitioningDemo").getOrCreate()

# 원시 이벤트 읽기
events = spark.read.parquet("/data/raw/events")

# 분석을 위한 시간 기반 파티셔닝
events.write \
    .format("delta") \
    .partitionBy("year", "month", "day") \
    .mode("overwrite") \
    .save("/data/curated/events")

# 조인을 위한 해시 기반 파티셔닝
events.repartition(100, "user_id") \
    .write \
    .format("parquet") \
    .bucketBy(100, "user_id") \
    .sortBy("timestamp") \
    .saveAsTable("events_bucketed")

# 파티션 크기 최적화
# 목표: 파티션당 128MB - 1GB
optimal_partition_count = events.count() * avg_row_size_bytes / (256 * 1024 * 1024)

스키마 레지스트리 통합

WIA-BIG_DATA 표준은 중앙 집중식 스키마 레지스트리를 통한 스키마 관리를 요구합니다. 이를 통해 생산자와 소비자 간의 스키마 호환성을 보장합니다.

// 스키마 레지스트리 API 구현
const schemaRegistry = {
  baseUrl: "https://schema-registry.example.com",

  async registerSchema(subject, schema) {
    const response = await fetch(
      `${this.baseUrl}/subjects/${subject}/versions`,
      {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ schema: JSON.stringify(schema) })
      }
    );
    return response.json();
  },

  async getLatestSchema(subject) {
    const response = await fetch(
      `${this.baseUrl}/subjects/${subject}/versions/latest`
    );
    return response.json();
  },

  async checkCompatibility(subject, schema) {
    const response = await fetch(
      `${this.baseUrl}/compatibility/subjects/${subject}/versions/latest`,
      {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ schema: JSON.stringify(schema) })
      }
    );
    return response.json();
  }
};
