Chapter 6: Privacy and Security

Protecting sensitive data while enabling analytics is one of the greatest challenges in big data. This chapter provides comprehensive guidance on implementing privacy-preserving techniques and security measures that meet regulatory requirements without sacrificing data utility.

6.2 Data Encryption

Encryption is fundamental to protecting data at rest and in transit:

Encryption at Rest

All stored data should be encrypted using strong algorithms. Modern cloud storage and big data platforms provide transparent encryption capabilities.

  • AES-256 encryption for data files
  • Key management using HSMs or cloud KMS
  • Automatic key rotation policies
  • Separate keys for different data classifications
Encryption in Transit

All data movement must use encrypted channels to prevent interception:

  • TLS 1.3 for all network communications
  • Certificate management and rotation
  • Mutual TLS for service-to-service communication
  • Encrypted connections to data sources and sinks
Encryption Configuration Example:

# Spark encryption settings

6.3 Privacy-Preserving Techniques

Multiple techniques exist for protecting privacy while maintaining data utility:

Anonymization

Removing or obscuring identifiers to prevent re-identification:

Anonymization Techniques:
┌────────────────────┬────────────────────────────────────────────────┐
│ Technique          │ Description                                    │
├────────────────────┼────────────────────────────────────────────────┤
│ Suppression        │ Remove sensitive fields entirely               │
│ Generalization     │ Replace specific values with ranges/categories │
│ Pseudonymization   │ Replace identifiers with reversible tokens     │
│ Aggregation        │ Combine individual records into groups         │
│ Perturbation       │ Add noise to numerical values                  │
│ Data Swapping      │ Exchange values between records                │
└────────────────────┴────────────────────────────────────────────────┘

K-Anonymity

Ensures each record is indistinguishable from at least k-1 other records on quasi-identifiers:

# K-Anonymity Implementation Example
from dataclasses import dataclass
from typing import List
import pandas as pd

def apply_k_anonymity(df: pd.DataFrame, 
                      quasi_identifiers: List[str],
                      k: int) -> pd.DataFrame:
    """
    Apply k-anonymity by generalizing quasi-identifiers
    """
    result = df.copy()
    
    # Generalize age to ranges
    if 'age' in quasi_identifiers:
        result['age'] = pd.cut(result['age'], 
                               bins=[0, 20, 30, 40, 50, 60, 100],
                               labels=['<20', '20-29', '30-39', 
                                      '40-49', '50-59', '60+'])
    
    # Generalize zip code to prefix
    if 'zip_code' in quasi_identifiers:
        result['zip_code'] = result['zip_code'].str[:3] + '**'
    
    # Verify k-anonymity
    group_sizes = result.groupby(quasi_identifiers).size()
    if group_sizes.min() < k:
        # Further generalization needed
        pass
    
    return result

Differential Privacy

Provides mathematical guarantees about privacy by adding calibrated noise:

Definition: A randomized algorithm M satisfies epsilon-differential privacy if for all datasets D1, D2 differing by one record, and all outputs S:

Pr[M(D1) in S] <= exp(epsilon) * Pr[M(D2) in S]

The privacy budget epsilon controls the trade-off between privacy and accuracy.

from pydp.algorithms.laplacian import BoundedMean, Count

# Differential privacy with Google's DP library
class DifferentiallyPrivateStats:
    def __init__(self, epsilon: float = 1.0):
        self.epsilon = epsilon
    
    def private_count(self, data: list) -> int:
        count_algo = Count(
            epsilon=self.epsilon,
            dtype="int"
        )
        return count_algo.quick_result(data)
    
    def private_mean(self, data: list, 
                     lower_bound: float, 
                     upper_bound: float) -> float:
        mean_algo = BoundedMean(
            epsilon=self.epsilon,
            lower_bound=lower_bound,
            upper_bound=upper_bound,
            dtype="float"
        )
        return mean_algo.quick_result(data)

# Usage
dp_stats = DifferentiallyPrivateStats(epsilon=0.5)
private_avg_salary = dp_stats.private_mean(
    salaries, 
    lower_bound=30000, 
    upper_bound=500000
)

6.7 Chapter Summary

Privacy and security are fundamental requirements for big data systems handling sensitive information. A defense-in-depth approach combines perimeter security, access management, data protection, and monitoring to create comprehensive protection.

Privacy-preserving techniques like anonymization, k-anonymity, and differential privacy enable analytics while protecting individual privacy. Strong encryption, fine-grained access controls, and comprehensive audit logging complete the security picture. Organizations must also prepare for incidents with clear response procedures.
