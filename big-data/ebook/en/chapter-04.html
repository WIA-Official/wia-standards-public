4.2 ML Pipeline Architecture

A robust ML pipeline integrates seamlessly with big data infrastructure, from feature engineering to model deployment:

Stage 1: Data Ingestion

Raw data flows from various sources into the data lake. Streaming and batch ingestion patterns ensure data freshness while maintaining historical records.

Stage 2: Feature Engineering

Transform raw data into features suitable for ML models. This includes aggregations, encodings, normalizations, and derived features stored in a feature store.

Stage 3: Model Training

Train models using distributed computing frameworks. Experiment tracking captures hyperparameters, metrics, and model artifacts.

Stage 4: Model Validation

Validate model performance against holdout data. Automated tests check for data drift, model fairness, and performance degradation.

Stage 5: Model Deployment

Deploy models for batch inference or real-time serving. A/B testing and canary deployments ensure safe rollouts.

Stage 6: Monitoring

Continuous monitoring tracks model performance, data quality, and prediction drift. Automated alerts trigger retraining when needed.

from feast import FeatureStore, Entity, FeatureView, Field
from feast.types import Float32, Int64

# Define entity
customer = Entity(
    name="customer_id",
    description="Customer identifier"
)

# Define feature view
customer_features = FeatureView(
    name="customer_features",
    entities=[customer],
    schema=[
        Field(name="total_purchases_30d", dtype=Float32),
        Field(name="avg_transaction_value", dtype=Float32),
        Field(name="days_since_last_purchase", dtype=Int64),
        Field(name="purchase_frequency", dtype=Float32),
        Field(name="customer_lifetime_value", dtype=Float32),
    ],
    source=BigQuerySource(
        table="project.dataset.customer_features",
        timestamp_field="event_timestamp"
    ),
    online=True,
    ttl=timedelta(days=1)
)

# Apply feature definitions
store = FeatureStore(repo_path=".")
store.apply([customer, customer_features])

# Get training data with point-in-time correctness

4.5 Distributed ML with Spark MLlib

Spark MLlib provides scalable machine learning algorithms that can process data across distributed clusters:

Supported Algorithms

  • Classification: Logistic Regression, Decision Trees, Random Forest, GBT, Naive Bayes
  • Regression: Linear Regression, Decision Tree Regression, GBT Regression
  • Clustering: K-Means, Bisecting K-Means, Gaussian Mixture, LDA
  • Collaborative Filtering: ALS for recommendation systems
  • Feature Engineering: TF-IDF, Word2Vec, PCA, Feature Scaling
from pyspark.ml import Pipeline
from pyspark.ml.feature import (
    StringIndexer, VectorAssembler, StandardScaler
)
from pyspark.ml.classification import GBTClassifier
from pyspark.ml.tuning import CrossValidator, ParamGridBuilder

# Define pipeline stages
category_indexer = StringIndexer(
    inputCols=["category", "region"],
    outputCols=["category_idx", "region_idx"]
)

assembler = VectorAssembler(
    inputCols=["category_idx", "region_idx", "amount", "frequency"],
    outputCol="features_raw"
)

scaler = StandardScaler(
    inputCol="features_raw",
    outputCol="features"
)

gbt = GBTClassifier(
    labelCol="label",
    featuresCol="features",
    maxIter=50
)

pipeline = Pipeline(stages=[
    category_indexer, assembler, scaler, gbt
])

# Hyperparameter tuning
param_grid = ParamGridBuilder() \
    .addGrid(gbt.maxDepth, [5, 10, 15]) \
    .addGrid(gbt.stepSize, [0.01, 0.1]) \
    .build()

crossval = CrossValidator(
    estimator=pipeline,
    estimatorParamMaps=param_grid,
    evaluator=BinaryClassificationEvaluator(),
    numFolds=5,
    parallelism=4
)

# Train with cross-validation
cv_model = crossval.fit(training_data)
best_model = cv_model.bestModel
