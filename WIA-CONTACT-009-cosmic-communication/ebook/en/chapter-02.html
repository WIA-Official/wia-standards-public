Shannon's Information Theory in Cosmic Context

The Channel Capacity Theorem

Claude Shannon's 1948 paper "A Mathematical Theory of Communication" established the fundamental limits of reliable communication. The Shannon-Hartley theorem defines maximum channel capacity:

C = B × log₂(1 + S/N)

Where:

  • C = Channel capacity (bits per second)
  • B = Bandwidth (Hz)
  • S = Signal power
  • N = Noise power
  • S/N = Signal-to-noise ratio

For cosmic communication, this formula reveals harsh realities. As distance increases, signal power decreases by the inverse square law, dramatically reducing S/N ratio and thus capacity.

Practical Example: Earth to Proxima Centauri

Consider transmitting to Proxima Centauri (4.24 light-years away):

// Channel capacity calculation
const DISTANCE = 4.24 * 9.461e15; // meters (light-years to meters)
const TRANSMISSION_POWER = 1e6;   // 1 MW transmitter
const FREQUENCY = 1.42e9;         // 1420 MHz (hydrogen line)
const RECEIVER_AREA = 100;        // 100 m² receiving antenna

// Free space path loss (FSPL)
function calculatePathLoss(distance, frequency) {
  const c = 299792458; // speed of light
  const wavelength = c / frequency;
  const fspl = Math.pow(4 * Math.PI * distance / wavelength, 2);
  return fspl;
}

// Received power
const pathLoss = calculatePathLoss(DISTANCE, FREQUENCY);
const receivedPower = TRANSMISSION_POWER / pathLoss;

// With cosmic background noise at ~3K (2.7K CMB)
const NOISE_TEMPERATURE = 3;      // Kelvin
const BOLTZMANN = 1.38e-23;       // J/K
const BANDWIDTH = 1e6;            // 1 MHz bandwidth

const noisePower = BOLTZMANN * NOISE_TEMPERATURE * BANDWIDTH;
const snr = receivedPower / noisePower;
const capacity = BANDWIDTH * Math.log2(1 + snr);

console.log(`Channel capacity: ${capacity.toFixed(2)} bits/second`);
// Result: Extremely low capacity requiring multi-year transmission times

The "Water Hole" - Optimal Radio Window

The region between the hydrogen line (1420 MHz) and hydroxyl radical lines (1612-1720 MHz) is called the "water hole" because H + OH = H₂O. This frequency range offers several advantages:

  • Low Cosmic Noise: Minimal interference from natural cosmic sources
  • Low Atmospheric Absorption: Signals pass through planetary atmospheres relatively unimpeded
  • Universal Significance: Hydrogen is the most abundant element; any technological civilization would recognize its importance
  • Natural Quiet Zone: International agreements protect this spectrum from terrestrial interference

WIA-CONTACT-009 Recommendation: Primary listening and transmission should occur within the water hole (1420-1720 MHz) for maximum probability of detection and minimal interference.

Implementation: BPSK Modulator

Binary Phase Shift Keying offers excellent noise resistance, making it ideal for cosmic communication:

/**
 * WIA-CONTACT-009 BPSK Modulator
 * Encodes binary data into phase-shifted carrier wave
 */

class BPSKModulator {
  constructor(carrierFrequency, symbolRate) {
    this.carrierFreq = carrierFrequency;
    this.symbolRate = symbolRate;
    this.samplesPerSymbol = 100; // Oversampling ratio
  }

  // Encode binary data to BPSK signal
  modulate(binaryData) {
    const signal = [];
    const symbolDuration = 1 / this.symbolRate;
    const sampleRate = this.samplesPerSymbol * this.symbolRate;

    for (let i = 0; i < binaryData.length; i++) {
      const bit = binaryData[i];
      // 0 → phase 0°, 1 → phase 180°
      const phase = bit === 0 ? 0 : Math.PI;

      // Generate symbol waveform
      for (let j = 0; j < this.samplesPerSymbol; j++) {
        const t = (i * this.samplesPerSymbol + j) / sampleRate;
        const sample = Math.cos(2 * Math.PI * this.carrierFreq * t + phase);
        signal.push(sample);
      }
    }

    return signal;
  }

  // Demodulate BPSK signal to binary data
  demodulate(signal) {
    const binaryData = [];
    const symbolLength = this.samplesPerSymbol;

    for (let i = 0; i < signal.length; i += symbolLength) {
      // Extract one symbol
      const symbolSamples = signal.slice(i, i + symbolLength);

      // Correlate with reference phases
      let correlation0 = 0;  // Phase 0°
      let correlation180 = 0; // Phase 180°

      for (let j = 0; j < symbolLength; j++) {
        const t = (i + j) / (symbolLength * this.symbolRate);
        const ref0 = Math.cos(2 * Math.PI * this.carrierFreq * t);
        const ref180 = Math.cos(2 * Math.PI * this.carrierFreq * t + Math.PI);

        correlation0 += symbolSamples[j] * ref0;
        correlation180 += symbolSamples[j] * ref180;
      }

      // Decide bit based on which correlation is stronger
      binaryData.push(correlation0 > correlation180 ? 0 : 1);
    }

    return binaryData;
  }
}

// Usage example
const modulator = new BPSKModulator(1420.405751e6, 100); // 1420 MHz, 100 symbols/sec
const message = [1,0,1,1,0,0,1,0]; // Binary message
const signal = modulator.modulate(message);
const decoded = modulator.demodulate(signal);

console.log("Original:", message);
console.log("Decoded:", decoded);
console.log("Match:", JSON.stringify(message) === JSON.stringify(decoded));

Error Correction Coding

Forward Error Correction (FEC) adds redundancy allowing receivers to correct errors without retransmission—essential when round-trip times span years:

Reed-Solomon Codes: Correct burst errors; widely used in deep space probes

Turbo Codes: Near Shannon limit performance; ideal for low SNR

LDPC (Low-Density Parity-Check): Excellent performance, flexible code rates

Fountain Codes: Infinite redundancy; receiver collects until message decoded

Channel Coding Strategies

Interleaving for Burst Error Protection

Cosmic interference often creates burst errors—consecutive corrupted bits. Interleaving disperses these bursts:

Original Data Stream:
┌─┬─┬─┬─┬─┬─┬─┬─┬─┬─┬─┬─┐
│A│A│A│B│B│B│C│C│C│D│D│D│
└─┴─┴─┴─┴─┴─┴─┴─┴─┴─┴─┴─┘

After Interleaving:
┌─┬─┬─┬─┬─┬─┬─┬─┬─┬─┬─┬─┐
│A│B│C│D│A│B│C│D│A│B│C│D│
└─┴─┴─┴─┴─┴─┴─┴─┴─┴─┴─┴─┘

Burst Error (affects positions 4-7):
┌─┬─┬─┬X┬X┬X┬X┬─┬─┬─┬─┬─┐
│A│B│C│X│X│X│X│D│A│B│C│D│
└─┴─┴─┴─┴─┴─┴─┴─┴─┴─┴─┴─┘

After De-interleaving:
┌─┬─┬X┬─┬─┬X┬─┬X┬X┬─┬─┬─┐
│A│A│X│B│B│X│C│X│X│D│D│D│
└─┴─┴─┴─┴─┴─┴─┴─┴─┴─┴─┴─┘
Each codeword has only 1-2 errors → Correctable!

Concatenated Codes

Combining multiple error correction schemes provides layered protection:

  • Inner Code: Corrects frequent random errors (e.g., convolutional code)
  • Interleaver: Disperses remaining burst errors
  • Outer Code: Corrects residual errors (e.g., Reed-Solomon)

This approach is used by Voyager, Cassini, and other deep space missions to maintain communication across billions of kilometers.

Bandwidth and Data Rate Optimization

The Bandwidth-Power Tradeoff

Communication systems face a fundamental tradeoff:

Wide Bandwidth: High data rate, but spreads power across frequencies → lower SNR per bit

Narrow Bandwidth: Concentrates power → higher SNR, but lower data rate

For cosmic communication where power is precious, narrow bandwidth with long integration times often proves optimal.

Adaptive Rate Control

WIA-CONTACT-009 recommends adaptive schemes that adjust data rate based on channel conditions:

/**
 * Adaptive Channel Rate Controller
 * Adjusts transmission rate based on measured SNR
 */

class AdaptiveRateController {
  constructor() {
    // SNR thresholds for different modulation schemes
    this.rateTable = [
      { minSNR: 0,  modulation: "OOK",   rate: 1,   efficiency: "Very Low" },
      { minSNR: 6,  modulation: "BPSK",  rate: 2,   efficiency: "Low" },
      { minSNR: 10, modulation: "QPSK",  rate: 4,   efficiency: "Medium" },
      { minSNR: 15, modulation: "8PSK",  rate: 8,   efficiency: "High" },
      { minSNR: 20, modulation: "16QAM", rate: 16,  efficiency: "Very High" }
    ];
  }

  selectOptimalRate(measuredSNR_dB) {
    // Find highest rate that meets SNR requirement
    let selected = this.rateTable[0];

    for (const config of this.rateTable) {
      if (measuredSNR_dB >= config.minSNR) {
        selected = config;
      } else {
        break;
      }
    }

    return {
      modulation: selected.modulation,
      bitsPerSymbol: Math.log2(selected.rate),
      efficiency: selected.efficiency,
      estimatedBER: this.estimateBER(measuredSNR_dB, selected.modulation)
    };
  }

  estimateBER(snr_dB, modulation) {
    // Bit Error Rate estimation (simplified)
    const snr_linear = Math.pow(10, snr_dB / 10);

    switch(modulation) {
      case "BPSK":
        return 0.5 * this.erfc(Math.sqrt(snr_linear));
      case "QPSK":
        return 0.5 * this.erfc(Math.sqrt(snr_linear / 2));
      default:
        return 0.5; // Placeholder
    }
  }

  erfc(x) {
    // Complementary error function approximation
    const t = 1 / (1 + 0.5 * Math.abs(x));
    const tau = t * Math.exp(-x*x - 1.26551223 +
      t * (1.00002368 +
      t * (0.37409196 +
      t * (0.09678418 +
      t * (-0.18628806 +
      t * (0.27886807 +
      t * (-1.13520398 +
      t * (1.48851587 +
      t * (-0.82215223 +
      t * 0.17087277)))))))));
    return x >= 0 ? tau : 2 - tau;
  }
}

Multi-Channel Diversity

Frequency Diversity

Transmitting the same information on multiple frequencies improves reliability when some channels experience fading or interference:

Strategy Description Benefit
Frequency Hopping Rapidly switch between frequencies Avoids narrowband interference
Parallel Transmission Simultaneous multi-frequency broadcast Redundancy, faster detection
Chirped Signals Frequency sweeps over time Doppler tolerance, easy detection

Temporal Diversity

Repeating transmissions at different times combats time-varying interference:

  • Periodic Beacons: Transmit same message every N hours/days/years
  • Randomized Timing: Avoid predictable interference patterns
  • Astronomical Alignment: Transmit when target system has optimal visibility
