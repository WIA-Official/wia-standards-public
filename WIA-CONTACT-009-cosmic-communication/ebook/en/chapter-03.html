Chapter 3: Signal Propagation Physics

Overview

Understanding how electromagnetic signals propagate through the cosmos is fundamental to designing effective communication systems. This chapter explores the physical phenomena that affect signal transmission across interstellar distances: from the inverse square law that governs signal strength, to dispersion effects caused by the interstellar medium, to relativistic effects that arise when transmitter or receiver are in motion.

The laws of physics impose hard constraints on cosmic communication. By understanding these constraints, we can design systems that work with nature rather than against it, maximizing the chance of successful contact.

Electromagnetic Wave Fundamentals

Maxwell's Equations and Wave Propagation

All electromagnetic communication relies on waves described by Maxwell's equations. In vacuum, these waves travel at the speed of light c ≈ 299,792,458 m/s. The relationship between frequency (f), wavelength (λ), and speed is fundamental:

c = f × λ

Example: The hydrogen line at 1420 MHz has wavelength λ = c/f = 21.1 cm

Wave Equation in Free Space

The electric field E of an electromagnetic wave propagating in free space follows:

∇²E - (1/c²)(∂²E/∂t²) = 0

For a plane wave traveling in the z-direction:

E(z,t) = E₀ cos(kz - ωt + φ)

Where: k = 2π/λ (wave number), ω = 2πf (angular frequency), φ = initial phase

Energy and Power in EM Waves

The power flux (energy per unit area per unit time) is given by the Poynting vector:

/**
 * Calculate Poynting vector magnitude for EM wave
 * Returns power flux in W/m²
 */
function poyntingFlux(electricField, magneticField) {
  // S = (E × B) / μ₀
  // For plane wave: |S| = E₀² / (μ₀ c)
  const mu0 = 4 * Math.PI * 1e-7; // Permeability of free space
  const c = 299792458;             // Speed of light

  const E0 = electricField;  // Peak electric field (V/m)
  const flux = (E0 * E0) / (mu0 * c);

  return flux;
}

// Example: 1 MW transmitter with 10m diameter dish antenna
const transmitPower = 1e6;          // 1 MW
const dishArea = Math.PI * 5 * 5;   // 78.5 m²
const flux = transmitPower / dishArea;

console.log(`Power flux at transmitter: ${flux.toFixed(2)} W/m²`);
// Result: ~12,732 W/m²

The Inverse Square Law

Fundamental Distance Attenuation

Perhaps the most significant challenge in cosmic communication is the inverse square law. As a wave expands spherically from its source, the same total power spreads over ever-larger areas:

I(r) = P / (4πr²)

Where: I = intensity (W/m²), P = total radiated power (W), r = distance (m)

Critical insight: Doubling the distance reduces intensity by a factor of 4

Path Loss Calculation

The Free Space Path Loss (FSPL) quantifies signal attenuation over distance:

/**
 * WIA-CONTACT-009 Path Loss Calculator
 * Computes signal attenuation over cosmic distances
 */

class PathLossCalculator {
  constructor() {
    this.c = 299792458; // Speed of light (m/s)
  }

  // Calculate free space path loss in dB
  calculateFSPL_dB(distance_m, frequency_Hz) {
    // FSPL(dB) = 20 log₁₀(d) + 20 log₁₀(f) + 20 log₁₀(4π/c)
    const d = distance_m;
    const f = frequency_Hz;

    const fspl = 20 * Math.log10(d) +
                 20 * Math.log10(f) +
                 20 * Math.log10(4 * Math.PI / this.c);

    return fspl;
  }

  // Calculate received power given transmitted power and path loss
  receivedPower(transmitPower_W, pathLoss_dB, txGain_dBi = 0, rxGain_dBi = 0) {
    // Pr(dBm) = Pt(dBm) + Gt(dBi) + Gr(dBi) - FSPL(dB)
    const Pt_dBm = 10 * Math.log10(transmitPower_W) + 30; // W to dBm
    const Pr_dBm = Pt_dBm + txGain_dBi + rxGain_dBi - pathLoss_dB;
    const Pr_W = Math.pow(10, (Pr_dBm - 30) / 10); // dBm to W

    return Pr_W;
  }

  // Example: Communication with Proxima Centauri
  proximaCentauriExample() {
    const distance = 4.24 * 9.461e15;  // 4.24 light-years in meters
    const frequency = 1.42e9;          // 1420 MHz
    const txPower = 1e6;               // 1 MW transmitter
    const txGain = 60;                 // 100m dish: ~60 dBi gain at 1420 MHz
    const rxGain = 60;                 // 100m receiving dish

    const fspl = this.calculateFSPL_dB(distance, frequency);
    const rxPower = this.receivedPower(txPower, fspl, txGain, rxGain);

    console.log(`=== Proxima Centauri Communication ===`);
    console.log(`Distance: ${(distance/9.461e15).toFixed(2)} light-years`);
    console.log(`Frequency: ${(frequency/1e9).toFixed(2)} GHz`);
    console.log(`Path Loss: ${fspl.toFixed(2)} dB`);
    console.log(`TX Power: ${(10*Math.log10(txPower)+30).toFixed(2)} dBm`);
    console.log(`RX Power: ${(10*Math.log10(rxPower)+30).toFixed(2)} dBm`);
    console.log(`RX Power: ${rxPower.toExponential(2)} W`);

    return { fspl, rxPower };
  }
}

calculator.proximaCentauriExample();

Interstellar Medium Effects

Composition of Interstellar Space

Interstellar space is not empty. The interstellar medium (ISM) contains approximately 1 atom per cubic centimeter, plus dust, magnetic fields, and cosmic rays. These components affect signal propagation:

Component Density Effect on Signals
Neutral hydrogen ~0.5 atoms/cm³ Dispersion, absorption at 1420 MHz
Ionized plasma ~0.03 electrons/cm³ Dispersion, scintillation
Dust grains ~10⁻²⁶ kg/cm³ Scattering, extinction (optical)
Magnetic fields ~5 μG Faraday rotation

Dispersion: Frequency-Dependent Delays

Ionized plasma causes different frequencies to travel at different speeds, spreading pulses over time. The dispersion measure (DM) quantifies this effect:

DM = ∫ nₑ dl

Where: nₑ = electron density (cm⁻³), dl = path element (parsec)

Time delay between two frequencies:

Δt ≈ 4.15 ms × DM × (f₁⁻² - f₂⁻²)

Where frequencies are in GHz

/**
 * Dispersion Measure Calculator
 * Computes pulse spreading due to interstellar plasma
 */

class DispersionCalculator {
  // Calculate time delay between two frequencies
  calculateDelay(DM, freq1_GHz, freq2_GHz) {
    // Δt = 4.15 × DM × (f1^-2 - f2^-2) milliseconds
    const f1 = freq1_GHz;
    const f2 = freq2_GHz;
    const delay_ms = 4.15 * DM * (1/(f1*f1) - 1/(f2*f2));

    return delay_ms;
  }

  // Calculate optimal observing bandwidth to avoid pulse smearing
  calculateOptimalBandwidth(DM, pulseWidth_ms, centerFreq_GHz) {
    // For pulse smearing < pulse width
    // Δf ≈ sqrt(pulse_width / (8.3 × DM × f^-3))
    const f = centerFreq_GHz;
    const pw = pulseWidth_ms;

    const optimalBW = Math.sqrt(pw / (8.3 * DM * Math.pow(f, -3)));

    return optimalBW;
  }

  // Example: Signal from 1000 pc away
  interstellarExample() {
    const distance_pc = 1000;
    const electronDensity = 0.03;  // electrons/cm³ (typical ISM)
    const DM = electronDensity * distance_pc;

    const freq1 = 1.42; // GHz (hydrogen line)
    const freq2 = 1.72; // GHz (OH line)

    const delay = this.calculateDelay(DM, freq1, freq2);

    console.log(`=== Dispersion Analysis ===`);
    console.log(`Distance: ${distance_pc} parsec`);
    console.log(`DM: ${DM.toFixed(1)} pc/cm³`);
    console.log(`Delay between ${freq1} GHz and ${freq2} GHz: ${delay.toFixed(2)} ms`);

    // For 1ms pulse, what bandwidth can we use?
    const maxBW = this.calculateOptimalBandwidth(DM, 1, 1.42);
    console.log(`Max bandwidth for 1ms pulse at 1.42 GHz: ${maxBW.toFixed(2)} GHz`);

    return { DM, delay };
  }
}

dispersion.interstellarExample();

Scattering and Scintillation

Electron density fluctuations cause multi-path propagation, leading to signal scattering and time-varying interference (scintillation):

  • Angular Broadening: Point sources appear larger due to scattering
  • Pulse Broadening: Sharp pulses are smeared in time
  • Scintillation: Random amplitude and phase variations (like stars twinkling)

Scintillation timescale varies from seconds to hours depending on frequency and path. Lower frequencies experience stronger scintillation.

Faraday Rotation

Magnetic fields in the ISM rotate the plane of polarization of linearly polarized waves:

Δφ = RM × λ²

Where: RM = rotation measure (rad/m²), λ = wavelength (m)

Rotation measure: RM = 0.81 ∫ nₑ B∥ dl

B∥ = magnetic field component parallel to line of sight (μG)

This effect is stronger at lower frequencies (scales with λ²), making polarization information unreliable for long-distance low-frequency communication unless corrected.

Doppler Effects

Classical Doppler Shift

Relative motion between transmitter and receiver shifts the observed frequency:

f_observed = f_emitted × (c ± v_observer) / (c ∓ v_source)

For v << c (non-relativistic): Δf/f ≈ v/c

Relativistic Doppler Effect

When relative velocities approach the speed of light, full relativistic treatment is required:

/**
 * Doppler Shift Calculator (Relativistic)
 * Handles velocities approaching speed of light
 */

class DopplerCalculator {
  constructor() {
    this.c = 299792458; // m/s
  }

  // Relativistic Doppler shift
  relativisticDoppler(f_emitted, velocity, approaching = true) {
    const v = velocity;
    const c = this.c;
    const beta = v / c;
    const gamma = 1 / Math.sqrt(1 - beta * beta);

    let f_observed;
    if (approaching) {
      // Source approaching observer
      f_observed = f_emitted * gamma * (1 + beta);
    } else {
      // Source receding from observer
      f_observed = f_emitted * gamma * (1 - beta);
    }

    return f_observed;
  }

  // Calculate frequency shift due to orbital motion
  orbitalDopplerRange(f_emitted, orbitalVelocity) {
    // Maximum blueshift (approaching)
    const f_blue = this.relativisticDoppler(f_emitted, orbitalVelocity, true);

    // Maximum redshift (receding)
    const f_red = this.relativisticDoppler(f_emitted, orbitalVelocity, false);

    return { f_blue, f_red, range: f_blue - f_red };
  }

  // Example: Earth's orbital motion
  earthOrbitalExample() {
    const f_emitted = 1420.405751e6; // Hydrogen line (Hz)
    const earthVelocity = 29780;     // Earth's orbital velocity (m/s)

    const result = this.orbitalDopplerRange(f_emitted, earthVelocity);

    console.log(`=== Earth Orbital Doppler ===`);
    console.log(`Emitted frequency: ${(f_emitted/1e6).toFixed(6)} MHz`);
    console.log(`Earth velocity: ${earthVelocity} m/s (${(earthVelocity/this.c*100).toExponential(2)}% of c)`);
    console.log(`Max blueshift: ${(result.f_blue/1e6).toFixed(6)} MHz`);
    console.log(`Max redshift: ${(result.f_red/1e6).toFixed(6)} MHz`);
    console.log(`Total range: ${(result.range/1e3).toFixed(2)} kHz`);

    return result;
  }
}

doppler.earthOrbitalExample();

// Earth's orbital motion creates ±140 kHz shift at 1420 MHz

Compensating for Doppler in Communication

WIA-CONTACT-009 recommends several strategies for Doppler mitigation:

  • Wide Search Bands: Monitor frequencies across expected Doppler range
  • Chirped Signals: Frequency sweeps are more robust to Doppler uncertainty
  • Ephemeris Correction: Calculate and compensate for known motions
  • Two-Way Ranging: Use round-trip timing to measure and correct velocity

Relativistic Time Dilation

Special Relativity Effects

Clocks moving at different velocities run at different rates. For cosmic communication, this affects message timing and synchronization:

t' = t / √(1 - v²/c²) = γt

Where: γ = Lorentz factor, v = relative velocity

General Relativity: Gravitational Time Dilation

Strong gravitational fields also affect time. Near massive objects, clocks run slower:

t' = t × √(1 - 2GM/rc²)

Where: G = gravitational constant, M = mass, r = distance from center

For communication between civilizations at different gravitational potentials, timestamps must account for these differences.

Atmospheric and Planetary Effects

Earth's Atmosphere

Earth's atmosphere affects signals differently across the spectrum:

Frequency Range Atmospheric Effect Mitigation
<10 MHz Ionospheric reflection (blocks from space) Space-based receivers
10 MHz - 10 GHz Minimal absorption (radio window) Ground-based feasible
20-300 GHz Water vapor, oxygen absorption bands High-altitude or space-based
Optical (infrared) Atmospheric absorption, scattering Space-based or adaptive optics
Optical (visible) Moderate atmospheric effects Mountaintop observatories, space

Ionospheric Scintillation

Earth's ionosphere causes signal fading, particularly at lower frequencies and during solar activity. This affects both transmission and reception.

Multipath Propagation

Gravitational Lensing

Massive objects bend spacetime, creating multiple paths for signals. The Sun can focus signals at distances beyond ~550 AU (the solar gravitational lens focal line):

Gain from Solar Lensing: Up to 10⁸ intensity amplification for perfect alignment

Challenge: Requires precise positioning of receiver along focal line

Application: Future missions could use solar lensing for extreme-distance communication

Interstellar Lensing

Intervening stars or compact objects can also lens signals, though this is rare and unpredictable. Microlensing events can temporarily amplify or obscure signals.

Thermal Noise and Cosmic Background

Cosmic Microwave Background

The CMB provides a fundamental noise floor at ~2.725 K across all frequencies. This sets the minimum achievable system temperature:

/**
 * System Temperature and Sensitivity Calculator
 * Determines minimum detectable signal power
 */

class SensitivityCalculator {
  constructor() {
    this.k = 1.38064852e-23; // Boltzmann constant (J/K)
  }

  // Calculate system noise temperature
  systemTemperature(antennaTemp, receiverTemp, cosmicTemp = 2.725) {
    // T_sys = T_ant + T_rx + T_cosmic
    return antennaTemp + receiverTemp + cosmicTemp;
  }

  // Minimum detectable signal (radiometer equation)
  minimumDetectableSignal(T_sys, bandwidth, integrationTime, SNR_required = 1) {
    // ΔT_min = T_sys / sqrt(B × τ) × SNR
    const deltaT = T_sys / Math.sqrt(bandwidth * integrationTime) * SNR_required;

    // Convert to power
    const P_min = this.k * deltaT * bandwidth;

    return { deltaT, P_min };
  }

  // Example: Optimal SETI receiver
  optimalReceiverExample() {
    const T_ant = 5;        // K (cryogenically cooled, pointed at cold sky)
    const T_rx = 10;        // K (state-of-art cryogenic receiver)
    const T_sys = this.systemTemperature(T_ant, T_rx);

    const bandwidth = 1;    // 1 Hz (narrow filter)
    const integration = 600; // 10 minutes

    const result = this.minimumDetectableSignal(T_sys, bandwidth, integration, 5);

    console.log(`=== Receiver Sensitivity ===`);
    console.log(`System temperature: ${T_sys.toFixed(2)} K`);
    console.log(`Bandwidth: ${bandwidth} Hz`);
    console.log(`Integration time: ${integration} s`);
    console.log(`Min detectable ΔT: ${result.deltaT.toExponential(2)} K`);
    console.log(`Min detectable power: ${result.P_min.toExponential(2)} W`);

    return result;
  }
}

const sensitivity = new SensitivityCalculator();
sensitivity.optimalReceiverExample();

// Result: Can detect signals at 10^-24 W level with long integration

Chapter Summary

Signal propagation physics determines the fundamental limits and challenges of cosmic communication. Understanding these physical phenomena enables engineers to design systems that maximize signal integrity across interstellar distances.

Key Takeaways:

  • The inverse square law causes signal strength to decrease with distance²—the primary challenge
  • Directional antennas provide gain that partially compensates for distance attenuation
  • The interstellar medium causes dispersion, scattering, and polarization rotation
  • Doppler shifts from orbital and galactic motion require wide search bandwidths
  • Relativistic effects (time dilation, Doppler) become significant at high velocities
  • Earth's atmosphere creates "windows" at radio and optical frequencies
  • Gravitational lensing can provide enormous amplification but requires precise alignment
  • Cosmic microwave background sets a fundamental noise floor at 2.7 K

These physical constraints are not obstacles to overcome but rather design parameters to optimize. WIA-CONTACT-009 protocols incorporate these effects into signal design, ensuring maximum robustness against the harsh realities of interstellar propagation.
