장 요약

핵심 내용:

  1. W3C 검증 가능한 자격증명(VC)은 주체에 대한 클레임과 증명 메커니즘을 결합한 암호학적으로 안전하고 변조 방지 디지털 자격증명으로, 검증자가 발급자에게 연락할 필요 없이 신뢰할 수 있는 자격증명 교환을 가능하게 하여 온라인 쿼리에서 오프라인 암호학적 검증으로 자격증명 검증을 근본적으로 변환합니다.
  2. 검증 가능한 자격증명 데이터 모델에는 네 가지 주요 참여자가 포함됩니다: 발급자(자격증명 생성 주체), 보유자(자격증명 통제 주체), 검증자(자격증명 확인 주체), 그리고 검증 가능한 데이터 레지스트리(DID 해석 및 취소 목록을 통해 신뢰를 고정하는 이더리움, 폴리곤, 하이퍼레저 인디와 같은 시스템).
  3. 자격증명 증명은 JSON 웹 서명(JWS), 연결 데이터 서명(LD-Proof), 그리고 선택적 공개를 가능하게 하는 영지식 증명(ZKP) 등 여러 암호화 서명 방식을 지원하며, 보유자가 불필요한 정보를 공개하지 않고 특정 클레임을 증명할 수 있게 하여 GDPR 준수 데이터 최소화에 중요합니다.
  4. 검증 가능한 프레젠테이션은 하나 이상의 VC를 자격증명 소유 및 공유 동의를 입증하는 보유자 생성 증명과 함께 패키징하여, 조직 경계를 넘어 프라이버시 보호 자격증명 공유를 보장하기 위해 유니버설 동의와 통합되는 감사 가능한 권한 추적을 생성합니다.
  5. 자격증명 상태 메커니즘(RevocationList2020, StatusList2021)은 블록체인 고정 또는 분산 상태 목록을 통해 발급 후 자격증명을 취소하거나 정지할 수 있게 하며, 프라이버시 요구사항(보유자가 신원을 노출하지 않고 상태 확인)과 자격증명 생명주기에 대한 발급자 통제 사이의 균형을 맞춥니다.
  6. 자격증명 바인딩(VC를 특정 DID 보유자에 연결), 술어 증명(생년월일을 공개하지 않고 나이 > 18 증명), 연결 불가능한 프레젠테이션(검증자 상관관계를 방지하기 위한 ZKP 사용)을 포함한 프라이버시 강화 기술은 전통적인 무기명 자격증명을 넘어 정교한 프라이버시 보호 신원 검증을 가능하게 합니다.
  7. 자격증명 생태계는 신뢰 레지스트리, 발급자 인증, 스키마 표준화, 책임 할당을 정의하는 거버넌스 프레임워크를 필요로 합니다 - 유니버설 아이덴티티는 기술 인프라를 제공하고 조직 정책은 산업 간 자격증명 수용을 위한 신뢰 경계를 확립합니다.

복습 질문

  1. 발급에서 프레젠테이션 및 검증까지 W3C 검증 가능한 자격증명의 전체 생명주기를 설명하십시오. 암호학적 증명이 검증자가 발급자에게 연락할 필요를 어떻게 제거하며, 프라이버시와 확장성에 대한 영향은 무엇입니까?
  2. 검증 가능한 자격증명을 위한 JSON 웹 서명(JWS) 증명과 연결 데이터 서명(LD-Proof) 형식을 비교하십시오. 개발자 친숙도, JSON-LD 복잡성, 서명 이식성, 선택적 공개 기능 측면에서 어떤 트레이드오프가 있습니까?
  3. 영지식 증명(ZKP)이 검증 가능한 자격증명에서 선택적 공개를 어떻게 가능하게 하는지 설명하십시오. 생년월일을 공개하지 않고 나이 > 21을 증명하는 ZK-SNARK를 사용한 프라이버시 보호 연령 확인 시스템을 설계하고 관련된 암호학적 작업을 설명하십시오.
  4. 검증 가능한 자격증명과 유니버설 동의 간의 관계를 분석하십시오. 자격증명 프레젠테이션이 보유자가 특정 검증자와 특정 클레임 공유를 승인했음을 증명하는 동의 영수증을 어떻게 전달해야 합니까? 동의 인식 VP를 위한 데이터 구조를 설계하십시오.
  5. 다양한 자격증명 취소 전략을 평가하십시오: 온체인 취소 목록(이더리움, 폴리곤), 누산기 기반 취소, 상태 목록 2021. 각 접근 방식의 프라이버시 특성, 비용, 조회 성능, GDPR 준수를 비교하십시오.
  6. 하이퍼레저 인디에서 학위 자격증명을 발급하는 대학을 위한 기업 자격증명 발급 아키텍처를 설계하십시오. 키 관리, 일괄 발급 성능, 자격증명 템플릿, 학생 프라이버시, 조직 변경에도 불구한 장기 자격증명 유효성을 다루십시오.
  7. 자격증명 이식성(보유자가 자격증명 통제)과 발급자 책임(발급자가 자격증명 정확성에 책임) 간의 긴장을 논의하십시오. 자격증명이 의도된 맥락 밖에서 사용될 때 거버넌스 프레임워크가 책임을 어떻게 할당해야 합니까? 유니버설 아이덴티티가 자격증명 스키마와 신뢰 레지스트리를 통해 이를 어떻게 해결합니까?
