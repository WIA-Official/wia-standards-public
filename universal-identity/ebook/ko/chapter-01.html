제 1장: Introduction to Universal Identity

In the rapidly evolving landscape of modern software development, Universal Identity (WIA-CORE-001) provides essential standardization that enables interoperability, security, and compliance across diverse systems and platforms.

This chapter examines the technical foundations, architectural patterns, and implementation strategies that make Universal Identity a critical component of enterprise-grade systems worldwide.

Understanding the principles and practices detailed in this chapter is essential for architects, developers, and technical leaders implementing WIA-CORE-001 compliant solutions.

Through comprehensive examples, detailed specifications, and real-world case studies, this chapter provides the knowledge needed for successful production deployments.

장 요약

핵심 내용:

  1. 유니버설 아이덴티티는 중앙화된 신원 시스템에서 자기주권 신원(SSI)으로의 패러다임 전환을 나타내며, W3C 탈중앙 식별자(DID)와 검증 가능한 자격증명 표준을 통해 개인이 디지털 신원을 직접 통제할 수 있도록 합니다.
  2. WIA-CORE-001 표준은 W3C 명세를 기반으로 블록체인 네트워크(이더리움, 폴리곤, 하이퍼레저 인디)와 전통적 시스템 간 상호운용 가능한 신원 솔루션을 구현하며, 신원 생명주기 전반에 걸쳐 프라이버시 바이 디자인과 GDPR 준수를 보장합니다.
  3. 탈중앙 신원 아키텍처는 단일 장애점을 제거하고 여러 신뢰 앵커에 신원 데이터를 분산하여 데이터 유출 위험을 줄이면서, 공개키 인프라와 영지식 증명을 통해 진위성의 암호학적 증거를 유지합니다.
  4. 유니버설 동의(WIA-CONSENT-001)와의 통합은 신원 공유가 항상 사용자 프라이버시 선호도와 규제 요구사항을 존중하도록 보장하며, 기업 및 소비자 애플리케이션에서 프라이버시 보호 신원 관리를 위한 포괄적 프레임워크를 구축합니다.
  5. 레거시 SSO 시스템에서 유니버설 아이덴티티로의 전환은 DID 해석 인프라, 검증 가능한 자격증명 발급 워크플로우, WebAuthn 통합을 통한 비밀번호 없는 인증 경험 등 신중한 마이그레이션 계획이 필요합니다.
  6. 이 표준은 생체 인식 검증, 하드웨어 보안 키, 블록체인 기반 증명을 포함한 다양한 인증 방법을 지원하여, 다양한 위험 프로필과 규제 환경에 적응하는 유연한 보안 태세를 가능하게 합니다.
  7. 유니버설 아이덴티티는 탈중앙 금융(DeFi) KYC, 국경 간 자격증명 검증, 기본 개인정보를 노출하지 않는 프라이버시 보호 연령 확인 등 신흥 사용 사례의 기반을 마련합니다.

복습 질문

  1. 자기주권 신원(SSI)과 OAuth2/OIDC와 같은 전통적인 연합 신원 시스템의 근본적인 차이점을 설명하십시오. 유니버설 아이덴티티가 상호운용성을 유지하면서 중앙화된 신원 제공자의 한계를 어떻게 해결하는지 설명하십시오.
  2. 유니버설 아이덴티티 아키텍처에서 W3C 탈중앙 식별자(DID)의 역할을 설명하십시오. DID가 중앙화된 레지스트리나 인증 기관 없이 어떻게 검증 가능한 탈중앙 신원을 가능하게 하는지 설명하십시오.
  3. 블록체인 기반 DID 방식(이더리움의 did:ethr, did:polygon, 하이퍼레저 인디의 did:indy)과 웹 기반 방식(did:web, did:key)을 비교하십시오. 비용, 영속성, 프라이버시, 해석 성능 측면에서 어떤 트레이드오프가 있습니까?
  4. 유니버설 아이덴티티가 GDPR 준수를 어떻게 달성하는지 분석하십시오. 특히 삭제권과 데이터 최소화 원칙을 다루십시오. 불변 블록체인 기록이 잊힐 권리와 어떻게 공존할 수 있습니까?
  5. 50,000명의 사용자가 있는 기업 환경에서 전통적인 SSO(SAML, OAuth2)에서 유니버설 아이덴티티로 마이그레이션하는 통합 전략을 설계하십시오. 주요 마이그레이션 단계, 호환성 고려사항, 롤백 전략은 무엇입니까?
  6. 유니버설 아이덴티티와 유니버설 동의 표준 간의 관계를 평가하십시오. 검증 가능한 자격증명이 동의 영수증을 어떻게 전달하며, 이러한 통합이 조직 경계를 넘어 프라이버시 보호 데이터 공유를 어떻게 향상시키는지 설명하십시오.
  7. 다양한 DID 컨트롤러 모델(단일 키, 다중 서명, 소셜 복구)의 보안 영향을 논의하십시오. 이러한 모델이 기업용과 소비자용 사용 사례에서 보안, 사용성, 복구 기능을 어떻게 균형 있게 조정합니까?
