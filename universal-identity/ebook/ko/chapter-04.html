Chapter Summary

Key Takeaways:

  1. DID 방식은 특정 플랫폼에서 식별자의 전체 생명주기 작업(생성, 읽기, 업데이트, 비활성화)을 정의합니다 - 각 방식 명세는 스마트 계약(did:ethr, did:polygon), 분산 원장(did:indy), 웹 인프라(did:web)와 같은 플랫폼별 메커니즘을 사용하여 DID를 인코딩하고 DID 문서를 게시하며 식별자를 해석하는 방법을 설명합니다.
  2. 이더리움 기반 DID 방식(did:ethr)은 비용 효율적인 DID 관리를 위해 ERC-1056 신원 레지스트리를 활용하며, 오프체인 DID 문서 저장을 가능하게 하면서 온체인에 필수 제어 키만 저장하여 탈중앙화와 실용적인 가스 비용 및 해석 성능의 균형을 맞춥니다.
  3. 폴리곤 DID(did:polygon)는 DID 작업을 위한 레이어 2 확장성을 제공하며, 이더리움으로의 주기적 체크포인팅을 통해 보안을 유지하면서 이더리움 메인넷보다 훨씬 낮은 트랜잭션 비용을 제공하여 대용량 소비자 애플리케이션에서 블록체인 기반 신원을 경제적으로 실행 가능하게 만듭니다.
  4. 하이퍼레저 인디(did:indy)는 기업 신원에 최적화된 허가형 원장 아키텍처를 도입하며, DID 해석을 위한 전문 노드, 내장 스키마 및 자격증명 정의 관리, ZKP 기반 선택적 공개 및 상관관계 방지 식별자를 포함한 프라이버시 보호 기능을 특징으로 합니다.
  5. DID 해석은 방식별 해석기를 통해 DID를 DID 문서로 변환하며, 적절한 데이터 소스를 쿼리하여 DID 문서뿐만 아니라 보안에 민감한 검증자에게 필수적인 해석 메타데이터(콘텐츠 유형, 해석 시간)와 문서 메타데이터(생성, 업데이트, 비활성화 상태)도 반환합니다.
  6. 유니버설 해석기는 방식별 해석 논리를 추상화하는 통합 인터페이스를 제공하여, 애플리케이션이 표준 API를 통해 모든 DID 방식을 해석할 수 있게 하며, 해석기 드라이버는 각 방식 명세에서 요구하는 대로 블록체인 쿼리, HTTP 요청 또는 피어투피어 프로토콜을 처리합니다.
  7. 방식 선택은 운영 비용, 프라이버시 특성, 규제 준수, 생태계 상호운용성에 영향을 미칩니다 - 유니버설 아이덴티티는 조직이 일관된 애플리케이션 수준 API를 유지하면서 다양한 사용 사례에 적합한 방식을 선택할 수 있도록 다중 방식 배포를 지원합니다.

Review Questions

  1. 경로, 쿼리 및 프래그먼트 구성 요소를 포함한 DID URL 구문을 설명하십시오. DID URL이 DID 문서 내의 특정 키, 서비스 엔드포인트 또는 리소스에 대한 세밀한 참조를 어떻게 가능하게 하며, 이것이 자격증명 검증 워크플로우에 왜 중요합니까?
  2. 트랜잭션 비용, 최종성 시간, 보안 모델 측면에서 did:ethr과 did:polygon을 비교하십시오. DID를 생성하는 100만 사용자가 있는 소비자 신원 애플리케이션의 경우, 두 방식의 예상 비용을 계산하고 각 방식이 경제적으로 최적인 경우를 설명하십시오.
  3. 쌍별 DID와 DID 회전을 포함한 하이퍼레저 인디의 프라이버시 보호 DID 접근 방식을 설명하십시오. 인디의 아키텍처가 신원의 암호학적 증명을 유지하면서 검증자 간 상관관계를 어떻게 방지합니까? 이더리움의 공개 DID 접근 방식과 비교하십시오.
  4. did:web 방식 명세를 분석하십시오. DID 해석을 위해 DNS와 HTTPS를 사용하는 것의 보안 영향은 무엇입니까? did:web DID가 중앙화된 신원 제공자 종속성을 피하면서 동의 인식 인증을 제공하기 위해 유니버설 동의와 어떻게 통합될 수 있습니까?
  5. 시간당 10,000개의 자격증명을 검증하는 기업 애플리케이션을 위한 DID 해석 캐싱 전략을 설계하십시오. 캐시 무효화, DID 문서 업데이트, 비활성화된 DID 처리, 검증 패턴에 대한 정보를 누출할 수 있는 해석 메타데이터 타이밍 공격 방지를 다루십시오.
  6. 방식별 최적화와 유니버설 해석기 추상화 간의 트레이드오프를 평가하십시오. 애플리케이션이 방식별 해석 논리를 사용해야 하는 경우와 유니버설 해석기를 사용해야 하는 경우는 언제입니까? 유니버설 아이덴티티가 참조 구현에서 이러한 우려를 어떻게 균형 있게 조정합니까?
  7. 한 블록체인에서 발급된 자격증명이 다른 블록체인의 DID를 참조하는 크로스체인 DID 해석 시나리오(예: 이더리움 발급자 DID를 참조하는 폴리곤 자격증명)를 논의하십시오. 보안 고려사항은 무엇이며, 검증자가 다중 체인 검증 워크플로우를 어떻게 처리해야 합니까?
