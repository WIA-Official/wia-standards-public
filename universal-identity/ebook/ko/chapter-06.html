Chapter Summary

Key Takeaways:

  1. 자격증명 검증은 여러 검증 계층을 포함합니다: 구문 검증(JSON 구조), 의미 검증(스키마 일치), 암호학적 검증(서명 검증), 상태 검증(취소 확인), 신뢰 검증(발급자 권한) - 각 계층은 안전한 프로덕션급 검증 워크플로우에 필수적입니다.
  2. 암호학적 증명 검증은 발급자 DID를 해석하여 검증 방법(공개키)을 검색하고, 서명 알고리즘이 자격증명 증명 타입(JWS, LD-Proof, BBS+)과 일치하는지 검증하고, 키 회전 및 다중 서명 자격증명과 같은 엣지 케이스를 처리하면서 적절한 암호화 라이브러리를 사용하여 서명 검증을 수행해야 합니다.
  3. 취소 확인은 프라이버시와 적시성의 균형을 맞춥니다 - 프라이버시 보호 접근 방식(StatusList2021, 누산기 기반 취소)은 발급자가 자격증명 사용을 추적하지 못하도록 방지하며, 검증자는 오프라인 검증 시나리오에서 적절한 캐싱 전략과 오래된 취소 데이터 처리를 결정해야 합니다.
  4. 신뢰 프레임워크 검증은 암호학적 검증을 넘어 정책 시행으로 확장됩니다: 검증자는 발급자 권한(이 DID가 학위 자격증명을 발급할 권한이 있습니까?), 자격증명 신선도(자격증명이 만료되었습니까?), 보유자 바인딩(프레젠테이션 증명이 자격증명 소유를 입증합니까?)을 검증해야 하며, 종종 이더리움 또는 하이퍼레저 인디의 신뢰 레지스트리를 참조합니다.
  5. 영지식 증명 검증은 고유한 과제를 도입합니다 - 검증자는 ZK 증명 정확성을 검증하고, 공개된 속성이 요구사항과 일치하는지 확인하고, 증명 재생 공격을 방지하고, 기본 값을 학습하지 않고 숨겨진 속성이 술어(나이 > 21)를 만족하는지 확인해야 하며, 전문 암호화 라이브러리와 도메인 지식이 필요합니다.
  6. 유니버설 동의와의 통합은 권한 검증 계층을 추가합니다 - 검증자는 자격증명 프레젠테이션에 동반되는 동의 영수증을 검증하여, 데이터 접근이 GDPR 법적 근거, 목적 제한, 보유 정책을 준수하는지 확인하고, 검증을 "이 자격증명을 신뢰할 수 있습니까?"에서 "이 자격증명을 신뢰할 수 있고 받을 권한이 있습니까?"로 변환합니다.
  7. 검증 성능 최적화 기술에는 DID 문서 캐싱, 병렬 취소 확인, 일괄 서명 검증, 점진적 검증(비용이 많이 드는 암호화 작업 전에 기본 확인에서 빠르게 실패)이 포함되며, 초당 수천 개의 자격증명을 처리하는 고처리량 애플리케이션에 중요합니다.

Review Questions

  1. JSON 웹 서명 증명이 있는 W3C 검증 가능한 자격증명의 완전한 검증 워크플로우를 설명하십시오. 특정 검증 단계는 무엇이며, 어떤 순서로 수행되어야 하며, 검증자가 처리해야 하는 오류 조건은 무엇입니까? DID 해석, 서명 검증, 취소 확인을 포함하십시오.
  2. 다양한 취소 방법의 프라이버시 특성을 비교하십시오: 온체인 비트스트링 상태 목록(StatusList2021), 암호화 누산기, 인증서 투명성 로그. 각 접근 방식이 효율적인 검증을 가능하게 하면서 보유자 프라이버시를 어떻게 보호하며, GDPR 준수 영향은 무엇입니까?
  3. 선택적 공개 자격증명을 위한 영지식 증명 검증을 분석하십시오. BBS+ 서명이 다른 클레임을 숨긴 상태로 유지하면서 특정 클레임을 공개하는 것을 어떻게 가능하게 하는지 설명하십시오. 보유자가 이름을 공개하지만 술어 증명을 사용하여 나이 > 18을 증명하는 자격증명에 대한 검증 알고리즘을 설계하십시오.
  4. 자격증명 검증과 유니버설 동의 간의 관계를 평가하십시오. 자격증명 진위성과 특정 클레임을 공유할 보유자 권한을 모두 검증하는 검증 API를 설계하십시오. 동의 영수증이 어떻게 구조화되어야 하며, 검증자가 자격증명 데이터를 수락하기 전에 수행해야 하는 동의 검증 확인은 무엇입니까?
  5. 국경 간 교육 자격증명 검증을 위한 신뢰 프레임워크를 설계하십시오. 발급자 인증(어떤 DID가 대학 학위를 발급할 수 있습니까?), 스키마 검증(국가별 학위 타입을 어떻게 처리합니까?), 검증 정책 시행(검증자가 인증되지 않은 발급자의 자격증명을 언제 거부해야 합니까?)을 다루십시오. 하이퍼레저 인디 신뢰 앵커와 이더리움 기반 신뢰 레지스트리를 고려하십시오.
  6. 시간당 10,000개의 자격증명 검증을 처리하는 시스템을 위한 검증 성능 최적화를 논의하십시오. DID 해석 캐싱 전략, 병렬 취소 확인, 암호화 라이브러리 선택, 점진적 검증 워크플로우를 다루십시오. 검증 속도와 보안 간의 트레이드오프는 무엇입니까?
  7. 검증자가 방어해야 하는 공격 시나리오를 분석하십시오: 서명 제거, 증명 재생, 취소 상태 스푸핑, DID 문서 대체, 만료된 자격증명 수락. 각 공격에 대해 이를 방지하는 검증 확인을 설명하고 검증이 보장하는 보안 속성을 설명하십시오.
