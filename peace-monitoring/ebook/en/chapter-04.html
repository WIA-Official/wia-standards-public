Chapter 4: Phase 1: Data Format Specifications

In this chapter: Deep dive into the data schemas that form the foundation of WIA-UNI-015, including monitoring events, arms inventory, DMZ sensors, troop movements, and verification records.

4.1 Core Data Schema Design

Phase 1 defines six core data types that cover all aspects of peace monitoring. Each schema is designed for clarity, extensibility, and validation.

Design Principles

  • Self-Descriptive: Field names clearly indicate purpose
  • Strongly Typed: Each field has explicit data type
  • Validation Rules: Constraints ensure data quality
  • Extensible: Additional fields can be added without breaking compatibility
  • International: Supports multiple languages, time zones, coordinate systems

4.5 Troop Movement Schema

Declarations of military movements exceeding defined thresholds (typically >1000 personnel or major equipment deployment).

Notification Requirements

  • 72-hour advance notice: Movements >1000 personnel
  • 48-hour notice: Exercises involving live fire
  • 24-hour notice: Movements within 5km of DMZ
  • Immediate notification: Emergency deployments (with justification)

4.7 Verification Record Schema

Documentation of inspection and verification activities, signed by all parties.

Verification Methodology Options

  • ON_SITE_INSPECTION: Physical access to facility
  • SATELLITE_IMAGERY: Remote sensing analysis
  • SENSOR_DATA_ANALYSIS: Automated monitoring review
  • SAMPLE_COLLECTION: Physical specimens for lab analysis
  • DOCUMENTATION_REVIEW: Paper/electronic records examination
  • PERSONNEL_INTERVIEWS: Testimony from facility staff

4.8 Validation Rules and Constraints

Required Fields

All records MUST include: ID, timestamp, type. Missing required fields cause validation failure.

Data Constraints

Field TypeConstraint
Latitude-90 ≤ value ≤ 90
Longitude-180 ≤ value ≤ 180
Confidence0.0 ≤ value ≤ 1.0
Quantitiesvalue ≥ 0 (non-negative integers)
TimestampsISO 8601 with timezone
IDsFormat: PREFIX-timestamp or UUID v4

Business Rules

  • Arms inventory declarations: Quarterly (Jan 1, Apr 1, Jul 1, Oct 1)
  • Troop movements >1000 personnel: 72-hour advance notice required
  • DMZ sensor CRITICAL alerts: Automatic notification to all observers
  • Verification records: Must include digital signature from verifier
  • Geofencing: Coordinates must be within Korean Peninsula bounds (33-43°N, 124-132°E)

Chapter Summary

Key Takeaways:

  • Six core data schemas cover all peace monitoring scenarios
  • JSON format ensures universal compatibility and readability
  • Validation rules maintain data quality and prevent errors
  • Extensible design allows future enhancements without breaking changes
  • Standardized formats eliminate manual data conversion and errors
  • Clear constraints enable automated compliance checking
