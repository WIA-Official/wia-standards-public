Chapter 7

EMR/EHR Integration

Learning Objectives:

Complete guide to integrating health monitoring data with EMR/EHR systems, mastering FHIR R4 resources, HL7 messaging, bidirectional synchronization, and working with major EMR platforms.

7.1 EMR/EHR Integration Overview

Integrating health monitoring data with Electronic Medical Records (EMR) and Electronic Health Records (EHR) systems closes the loop between patient-generated health data and clinical care. This integration enables clinicians to make informed decisions based on comprehensive, real-world data beyond what's captured during office visits.

7.1.1 Benefits of EMR Integration

  • Clinical Decision Support: Providers access longitudinal data to inform treatment decisions
  • Medication Adjustments: Real-world data (e.g., home blood pressure) guides medication titration
  • Care Coordination: Multiple providers access same data source, improving care coordination
  • Reduced Office Visits: Remote monitoring reduces need for in-person visits
  • Quality Measures: Automated reporting for quality metrics and value-based care programs

7.2 HL7 FHIR R4 Integration

7.3 HL7 v2.x Messaging

Despite FHIR's growing adoption, many healthcare systems still rely on HL7 v2.x messages for data exchange. Supporting both standards ensures broad interoperability.

7.3.1 ORU^R01 Observation Result Message

MSH|^~\&|WIA-HEALTH|WIA-ORG|EMR|HOSPITAL|20260112143000||ORU^R01|MSG001|P|2.5 PID|1||12345^^^MRN||DOE^JOHN||19800515|M OBR|1|ORDER001||8867-4^Heart Rate^LN|||20260112143000 OBX|1|NM|8867-4^Heart Rate^LN||72|/min||N|||F|||20260112143000 OBX|2|NM|80404-7^Heart Rate Variability^LN||45.3|ms||N|||F

7.3.2 Message Segments Explained

  • MSH (Message Header): Identifies sender, receiver, message type, and timestamp
  • PID (Patient Identification): Patient demographics and identifiers
  • OBR (Observation Request): Order information for the observations
  • OBX (Observation Result): Actual measurement values and metadata

7.4 Epic Integration

Epic is the largest EMR vendor in the United States, used by many major health systems. Epic provides multiple integration pathways.

7.4.1 Epic FHIR API

Epic supports FHIR R4 for reading and writing clinical data:

// Authentication POST https://fhir.epic.com/interconnect-fhir-oauth/oauth2/token Content-Type: application/x-www-form-urlencoded grant_type=client_credentials &client_id=YOUR_CLIENT_ID &client_secret=YOUR_CLIENT_SECRET // Submit Observation POST https://fhir.epic.com/interconnect-fhir-oauth/api/FHIR/R4/Observation Authorization: Bearer {access_token} Content-Type: application/fhir+json {observation_resource}

7.4.2 Epic MyChart Integration

For patient-facing applications, Epic's MyChart patient portal provides APIs for patients to authorize third-party apps to access their data (following SMART on FHIR authorization).

7.5 Cerner Integration

Cerner (now Oracle Health) is another major EMR vendor with extensive FHIR support.

7.6 Bidirectional Synchronization

True EMR integration requires bidirectional data flow: patient-generated data flows into the EMR, while clinical data (medications, diagnoses, care plans) flows back to the health monitoring system.

7.6.1 Inbound Sync (to EMR)

  • Vital signs from wearables → EMR flowsheet
  • Glucose readings → Diabetes flowsheet
  • Activity data → Physical therapy progress notes
  • Medication adherence → Pharmacy records

7.6.2 Outbound Sync (from EMR)

  • New medication orders → Medication reminder app
  • Lab results → Patient dashboard
  • Care plans → Personalized health coaching
  • Appointment schedules → Calendar integration

7.8 Integration Challenges and Solutions

7.8.1 Common Challenges

Challenge Impact Solution
Patient Matching Data linked to wrong patient Use multiple identifiers (MRN, DOB, name), implement matching confidence scores
Data Volume Overwhelm EMR with thousands of data points Aggregate data, send summaries, allow filtering in EMR
Different Units Glucose in mg/dL vs mmol/L Always include unit in UCUM format, perform conversions
Network Failures Data not synced to EMR Implement retry logic, queue failed messages, monitor sync status
Authorization Expiry Cannot write to EMR Proactively refresh tokens, alert users to re-authorize

7.9 Clinical Workflow Integration

Technical integration alone isn't sufficient; data must integrate into clinical workflows to be useful.

7.9.1 EMR Display Best Practices

  • Trend graphs: Show glucose over time, not just individual readings
  • Summary statistics: Average, time-in-range, % hypoglycemic episodes
  • Actionable alerts: Highlight values requiring clinical attention
  • Context information: Device type, quality scores, user notes
  • Separate from clinical measurements: Clearly distinguish patient-generated from clinician-measured data

Key Takeaways

  • EMR/EHR integration enables clinicians to make informed decisions based on comprehensive patient-generated health data
  • FHIR R4 is the modern standard for healthcare data exchange, with mapping health metrics to FHIR Observation resources
  • HL7 v2.x messaging remains widely used, requiring support for both FHIR and v2.x for broad interoperability
  • Epic and Cerner provide FHIR APIs with OAuth 2.0 / SMART on FHIR authorization for secure data exchange
  • Bidirectional synchronization flows patient data to EMR and clinical information back to monitoring systems
  • Terminology mapping using LOINC, SNOMED CT, RxNorm ensures consistent interpretation across systems
  • Clinical workflow integration requires appropriate data display, summary statistics, and clear distinction from clinical measurements
