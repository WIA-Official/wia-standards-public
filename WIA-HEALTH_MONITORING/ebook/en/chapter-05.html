Chapter 5

Data Analysis & Insights

Learning Objectives:

Learn to analyze health data using statistical methods, detect anomalies, identify trends, and leverage AI/ML for predictive insights and personalized health recommendations.

5.1 Descriptive Statistics

The foundation of health data analysis is descriptive statistics that summarize large datasets into meaningful insights. For continuous monitoring generating thousands of data points daily, appropriate aggregation is essential.

5.1.1 Common Statistical Measures

Measure Description Clinical Relevance Example (Heart Rate)
Mean Average value Overall level of metric Average HR = 72 bpm indicates normal resting rate
Median Middle value Central tendency, robust to outliers Median HR less affected by brief tachycardia episodes
Standard Deviation Variability measure Consistency/stability of metric High SD suggests variable heart rate patterns
Min/Max Extreme values Identify critical events Max HR 185 bpm during exercise
Percentiles Value distribution Understand typical ranges 95th percentile HR shows upper limit

5.1.2 Time-in-Range Analysis

Particularly important for glucose monitoring, time-in-range (TIR) calculates the percentage of time a metric stays within target ranges:

TIR = (readings_in_target_range / total_readings) × 100% For glucose: - Target range: 70-180 mg/dL - TIR > 70% = Good control - TIR > 50% = Fair control - TIR < 50% = Poor control

5.2 Trend Detection

Identifying trends in health data enables early intervention before metrics cross critical thresholds. Trend analysis examines how metrics change over time rather than absolute values.

5.2.1 Moving Averages

Smooth short-term fluctuations to reveal longer-term trends:

  • Simple Moving Average (SMA): Average of last N measurements, equal weighting
  • Exponential Moving Average (EMA): Recent values weighted more heavily, more responsive to changes
  • Weighted Moving Average (WMA): Custom weighting scheme, can emphasize recent or daytime measurements
// 7-day moving average for resting heart rate RHR_7day = (RHR_day1 + RHR_day2 + ... + RHR_day7) / 7 // Detecting trend If (RHR_7day_today > RHR_7day_week_ago + 5 bpm): Alert: "Resting heart rate increasing trend detected"

5.2.2 Regression Analysis

Linear regression quantifies the rate of change over time:

  • Positive slope: Metric increasing over time (e.g., weight gain, deteriorating glucose control)
  • Negative slope: Metric decreasing (e.g., weight loss, improving fitness)
  • Slope magnitude: Rate of change (e.g., 0.5 kg/week weight loss)

5.3 Anomaly Detection

Automated anomaly detection identifies unusual patterns that may indicate health issues requiring attention. Multiple approaches exist, each with different strengths.

5.3.1 Statistical Anomaly Detection

Z-Score Method: Measures how many standard deviations a value is from the mean:

z_score = (value - mean) / standard_deviation If |z_score| > 3: Flag as anomaly (occurs in <0.3% of normal distribution)

Interquartile Range (IQR) Method: More robust to non-normal distributions:

IQR = Q3 - Q1 Lower_bound = Q1 - 1.5 × IQR Upper_bound = Q3 + 1.5 × IQR If value < Lower_bound OR value > Upper_bound: Flag as anomaly

5.3.2 Machine Learning-Based Anomaly Detection

Advanced ML models can detect complex, multivariate anomalies:

Method Approach Best For
Isolation Forest Isolates anomalies by random partitioning High-dimensional data, multiple metrics
Autoencoder Neural network reconstructs normal data poorly for anomalies Complex patterns, time-series data
One-Class SVM Learns boundary around normal data When anomalies are rare
LSTM Networks Learns temporal dependencies, predicts next value Sequential health data, trend analysis

5.4 Predictive Analytics

Predictive models forecast future health metrics, enabling proactive interventions before adverse events occur.

5.4.1 Glucose Prediction

CGM systems can predict glucose levels 30-60 minutes into the future:

  • Features: Recent glucose values, rate of change, trend direction, time of day, recent carbs/insulin (if available)
  • Models: LSTM networks, ARIMA, ensemble methods
  • Use Case: Predict and prevent hypoglycemia before it occurs

5.4.2 Cardiac Event Prediction

Predict atrial fibrillation episodes, heart failure decompensation:

  • Features: HRV trends, resting HR changes, irregular rhythm frequency, activity levels, sleep quality
  • Models: Random forests, gradient boosting, deep learning
  • Use Case: Alert patients and providers before acute cardiac events

5.5 Personalized Insights

Generic insights are less valuable than personalized recommendations based on an individual's unique patterns, responses, and goals.

5.5.1 Baseline Establishment

Systems should learn individual baselines during an initial observation period:

  • Personal normal ranges: Individual's typical heart rate, glucose patterns, activity levels
  • Circadian patterns: How metrics vary throughout the day
  • Response patterns: How quickly glucose returns to baseline after meals
  • Stress indicators: Personal markers of stress (HRV, HR elevation)

5.5.2 Causal Inference

Identify relationships between behaviors and health outcomes:

Analysis: Impact of Exercise on Sleep Quality Data: 90 days of activity and sleep data Finding: Days with >30 min moderate exercise → +12 min deep sleep, +8% sleep efficiency Recommendation: "Exercise at least 30 minutes on weekdays to improve sleep quality"

5.6 Population Health Analytics

Aggregated, de-identified data from many users enables population-level insights:

5.6.1 Cohort Analysis

  • Identify effective interventions by comparing outcomes across cohorts
  • Benchmark individual metrics against age/gender-matched populations
  • Discover new biomarkers and health patterns
  • Support clinical research and drug development
Privacy Requirement: Population analytics must use fully de-identified data, comply with HIPAA Safe Harbor or Expert Determination methods, and obtain appropriate consent for research use.

5.7 Visualization Best Practices

Effective data visualization makes complex health data accessible and actionable:

5.7.1 Chart Types for Health Data

Chart Type Best For Example Use
Line Chart Trends over time Glucose over 24 hours, heart rate during exercise
Bar Chart Categorical comparisons Sleep stages distribution, weekly step counts
Scatter Plot Relationships between variables Correlation between exercise and sleep quality
Heatmap Patterns across time/categories Heart rate by hour and day of week
Box Plot Distribution and outliers Weekly glucose variability

Key Takeaways

  • Descriptive statistics (mean, median, standard deviation) summarize large health datasets into actionable insights
  • Time-in-range analysis is critical for glucose management, with >70% TIR indicating good control
  • Moving averages smooth short-term fluctuations to reveal longer-term trends in health metrics
  • Anomaly detection uses statistical methods and machine learning to identify unusual patterns requiring attention
  • Predictive models can forecast future health metrics, enabling proactive interventions before adverse events
  • Personalized insights based on individual baselines and patterns are more valuable than generic recommendations
  • Effective data visualization makes complex health information accessible and actionable for users and clinicians
