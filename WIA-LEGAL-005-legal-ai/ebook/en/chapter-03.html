Chapter 3: Data Formats for Legal Documents

Standardized data formats are the cornerstone of interoperable legal AI systems. This chapter examines the WIA-LEGAL-005 Phase 1 specification, which defines comprehensive data structures for representing legal documents, case law, statutes, and legal opinions in machine-readable formats that preserve semantic meaning and structural relationships.

3.1 The Need for Standardization

Legal documents exist in countless formats across different jurisdictions, courts, and legal systems. This heterogeneity creates significant challenges for AI systems attempting to process legal information at scale. Without standardized data formats, legal AI implementations become fragmented, with each system requiring custom parsers and converters for different document types.

The WIA-LEGAL-005 standard addresses this challenge by defining a unified JSON-based schema that can represent any legal document type while preserving critical metadata, structural relationships, and semantic annotations. This standardization enables legal AI systems to exchange data seamlessly, reducing integration costs and improving the accuracy of cross-system legal analysis.

3.4 Citation Format Standards

Legal citations follow complex formatting rules that vary by jurisdiction and citation style (Bluebook, ALWD, etc.). The WIA-LEGAL-005 standard provides a normalized citation representation that captures both the canonical citation format and structured citation components.

// WIA-LEGAL-005 Citation Schema

interface LegalCitation {
  id: string;
  type: CitationType;
  canonicalForm: string;
  components: CitationComponents;
  metadata: CitationMetadata;
}

type CitationType =
  | "case"
  | "statute"
  | "regulation"
  | "constitutional-provision"
  | "treaty"
  | "secondary-source"
  | "legislative-history";

interface CaseCitationComponents {
  caseName: string;
  reporter: string;
  volume: number;
  firstPage: number;
  court?: string;
  year: number;
  pinCite?: number;
}

interface StatuteCitationComponents {
  title: number;
  code: string;          // e.g., "U.S.C.", "Cal. Civ. Code"
  section: string;
  subsection?: string;
  year?: number;
}

// Example: Parsing and normalizing citations
class CitationParser {
  parse(citation: string): LegalCitation {
    const type = this.detectCitationType(citation);

    switch (type) {
      case "case":
        return this.parseCaseCitation(citation);
      case "statute":
        return this.parseStatuteCitation(citation);
      case "regulation":
        return this.parseRegulationCitation(citation);
      default:
        throw new Error(`Unsupported citation type: ${type}`);
    }
  }

  private parseCaseCitation(citation: string): LegalCitation {
    // Regex for U.S. case citations: "Name, Volume Reporter Page (Court Year)"
    const regex = /^(.+?),\s*(\d+)\s+([A-Za-z0-9.]+)\s+(\d+)\s*(?:\(([^)]+)\s+(\d{4})\))?/;
    const match = citation.match(regex);

    if (!match) {
      throw new Error(`Invalid case citation: ${citation}`);
    }

    return {
      id: this.generateCitationId(citation),
      type: "case",
      canonicalForm: citation,
      components: {
        caseName: match[1].trim(),
        volume: parseInt(match[2]),
        reporter: match[3],
        firstPage: parseInt(match[4]),
        court: match[5],
        year: parseInt(match[6])
      },
      metadata: {
        parsedAt: new Date().toISOString(),
        confidence: 0.95,
        jurisdiction: this.inferJurisdiction(match[3], match[5])
      }
    };
  }

  private parseStatuteCitation(citation: string): LegalCitation {
    // Regex for federal statute citations: "Title U.S.C. § Section"
    const regex = /^(\d+)\s+(U\.S\.C\.|C\.F\.R\.)\s+§\s*(\d+[A-Za-z]?(?:-\d+)?)/;
    const match = citation.match(regex);

    if (!match) {
      throw new Error(`Invalid statute citation: ${citation}`);
    }

    return {
      id: this.generateCitationId(citation),
      type: "statute",
      canonicalForm: citation,
      components: {
        title: parseInt(match[1]),
        code: match[2],
        section: match[3]
      },
      metadata: {
        parsedAt: new Date().toISOString(),
        confidence: 0.98,
        jurisdiction: match[2] === "U.S.C." ? "federal" : "regulation"
      }
    };
  }
}

// Usage example
const parser = new CitationParser();

const caseCite = parser.parse("Brown v. Board of Education, 347 U.S. 483 (1954)");
console.log(caseCite.components.caseName);  // "Brown v. Board of Education"

const statuteCite = parser.parse("42 U.S.C. § 1983");
console.log(statuteCite.components.section);  // "1983"

3.6 Validation and Quality Assurance

Ensuring data quality requires robust validation mechanisms. The WIA-LEGAL-005 standard includes JSON Schema definitions for all document types, enabling automatic validation of document structure and content.

// WIA-LEGAL-005 Document Validator

import Ajv from 'ajv';
import addFormats from 'ajv-formats';

class WIALegalDocumentValidator {
  private ajv: Ajv;
  private schemas: Map;

  constructor() {
    this.ajv = new Ajv({ allErrors: true, strict: false });
    addFormats(this.ajv);
    this.schemas = new Map();
    this.loadSchemas();
  }

  private loadSchemas(): void {
    // Load base schema
    const baseSchema = {
      $id: "https://schemas.wia.org/legal-005/v1/base",
      type: "object",
      required: ["wiaVersion", "standard", "document"],
      properties: {
        wiaVersion: { type: "string", pattern: "^\\d+\\.\\d+$" },
        standard: { type: "string", const: "WIA-LEGAL-005" },
        document: { $ref: "#/definitions/Document" }
      },
      definitions: {
        Document: {
          type: "object",
          required: ["id", "type", "jurisdiction", "metadata"],
          properties: {
            id: { type: "string", format: "uri" },
            type: {
              type: "string",
              enum: ["court-opinion", "contract", "statute", "regulation", "brief"]
            },
            jurisdiction: { $ref: "#/definitions/Jurisdiction" },
            metadata: { type: "object" },
            content: { type: "object" },
            citations: { type: "array", items: { $ref: "#/definitions/Citation" } },
            entities: { type: "array" },
            annotations: { type: "object" }
          }
        },
        Jurisdiction: {
          type: "object",
          required: ["country"],
          properties: {
            country: { type: "string", pattern: "^[A-Z]{2}$" },
            state: { type: "string", nullable: true },
            court: { type: "string" },
            courtLevel: {
              type: "string",
              enum: ["trial", "appellate", "supreme", "federal-district",
                     "federal-circuit", "federal-supreme"]
            }
          }
        },
        Citation: {
          type: "object",
          required: ["id", "type", "citation"],
          properties: {
            id: { type: "string" },
            type: {
              type: "string",
              enum: ["case", "statute", "regulation", "constitutional-provision",
                     "treaty", "secondary-source"]
            },
            citation: { type: "string" },
            fullCitation: { type: "string" },
            context: { type: "string" },
            treatment: {
              type: "string",
              enum: ["followed", "distinguished", "overruled", "questioned", "cited"]
            }
          }
        }
      }
    };

    this.ajv.addSchema(baseSchema);
    this.schemas.set("base", baseSchema);
  }

  validate(document: any): ValidationResult {
    const schema = this.schemas.get("base");
    const validate = this.ajv.getSchema(schema.$id);

    if (!validate) {
      throw new Error("Schema not found");
    }

    const valid = validate(document);

    return {
      valid: valid as boolean,
      errors: validate.errors || [],
      warnings: this.generateWarnings(document),
      qualityScore: this.calculateQualityScore(document, valid as boolean)
    };
  }

  private generateWarnings(document: any): string[] {
    const warnings: string[] = [];

    // Check for missing optional but recommended fields
    if (!document.document?.metadata?.decidedDate) {
      warnings.push("Missing recommended field: metadata.decidedDate");
    }

    if (document.document?.citations?.length === 0) {
      warnings.push("Document contains no citations (unusual for legal documents)");
    }

    // Check citation quality
    document.document?.citations?.forEach((cite, idx) => {
      if (!cite.fullCitation) {
        warnings.push(`Citation ${idx + 1} missing fullCitation field`);
      }
    });

    return warnings;
  }

  private calculateQualityScore(document: any, valid: boolean): number {
    if (!valid) return 0;

    let score = 70; // Base score for valid document

    // Bonus for complete metadata
    if (document.document?.metadata?.decidedDate) score += 5;
    if (document.document?.metadata?.parties?.length > 0) score += 5;
    if (document.document?.metadata?.judges?.length > 0) score += 5;

    // Bonus for citations
    const citationCount = document.document?.citations?.length || 0;
    if (citationCount > 0) score += Math.min(10, citationCount);

    // Bonus for entities
    const entityCount = document.document?.entities?.length || 0;
    if (entityCount > 0) score += Math.min(5, entityCount / 2);

    return Math.min(100, score);
  }
}

// Usage example
const validator = new WIALegalDocumentValidator();
const result = validator.validate(legalDocument);

if (!result.valid) {
  console.error("Validation errors:", result.errors);
}

console.log(`Document quality score: ${result.qualityScore}/100`);
result.warnings.forEach(warning => console.warn(warning));

Summary

Data Formats for Legal Documents introduces the WIA-LEGAL-005 Phase 1 specification for standardized legal document representation. Key takeaways include:

  • Standardization Benefits: Unified data formats enable interoperability, improve accuracy, and reduce integration costs across legal AI systems
  • Core Schema: Comprehensive JSON-based structure representing legal documents with metadata, content sections, citations, entities, and annotations
  • Document Types: Specialized schema extensions for court opinions, contracts, statutes, regulations, and legal briefs
  • Citation Standards: Normalized citation formats that preserve both canonical forms and structured citation components
  • Metadata Taxonomies: Controlled vocabularies for jurisdictions, practice areas, and document classification
  • Validation Framework: JSON Schema-based validation ensuring data quality and consistency across implementations

These standardized data formats form the foundation for all subsequent phases of the WIA-LEGAL-005 standard, enabling robust API interfaces, protocols, and integrations.
