Chapter 4: API 구현 가이드

4.1 SDK 구현 시작하기

WIA-LEGAL-001 표준은 디지털 법원 시스템과의 통합을 가속화하기 위해 여러 프로그래밍 언어로 공식 소프트웨어 개발 키트(SDK)를 제공합니다. 이러한 SDK는 HTTP 통신, 인증 관리, 오류 처리 및 데이터 직렬화의 복잡성을 추상화하여 개발자가 낮은 수준의 API 세부 사항보다는 애플리케이션 로직에 집중할 수 있도록 합니다.

공식 SDK 생태계에는 JavaScript/TypeScript, Python, Java, C#, Ruby, PHP 및 Go용 라이브러리가 포함되어 있으며, 각각은 플랫폼 간 일관된 기능을 유지하면서 언어별 규칙을 따릅니다. 모든 SDK는 MIT 라이센스 하에 오픈 소스이며 정기적인 업데이트, 보안 패치 및 기능 향상과 함께 WIA 커뮤니티에 의해 적극적으로 유지 관리됩니다.

4.1.1 JavaScript/TypeScript SDK

JavaScript/TypeScript SDK는 Node.js 서버 환경과 브라우저 기반 애플리케이션 모두에 대한 일급 지원을 제공하며, 완전한 TypeScript 타입 정의를 통해 최신 IDE에서 컴파일 타임 타입 검사 및 지능형 코드 완성을 가능하게 합니다.

설치:

# 또는
yarn add @wia/legal-001
# 또는

기본 클라이언트 설정:

import { DigitalCourtClient } from '@wia/legal-001';

// 사용자 이름과 비밀번호로 인증

이 포괄적인 예제는 당사자 등록, 문서 접수, 증거 제출 및 심리 일정 조정을 포함하여 새로운 민사 사건을 접수하기 위한 완전한 워크플로를 보여줍니다:

async function fileNewCivilCase() {
  try {

    const newCase = await client.cases.create({
      caseNumber: '2025-CV-001234',
      caseType: 'civil',
      court: {
        courtId: 'court-seoul-central-001',
        jurisdiction: 'KR-Seoul'
      },
      parties: {
        plaintiffs: [{
          name: '홍길동',
          did: 'did:wia:legal:hong123',
          email: 'hong@example.com',
          contact: {
            phone: '+82-10-1234-5678',
            address: {
              street: '테헤란로 123',
              city: '서울',
              postalCode: '06234',
              country: 'KR'
            }
          }
        }],
        defendants: [{
          name: '주식회사 베타',
          did: 'did:wia:legal:betacorp',
          email: 'legal@betacorp.com',
          type: 'corporation'
        }]
      },
      description: '결함 있는 기계에 대한 제조물 책임 청구',
      claimAmount: {
        value: 500000000,
        currency: 'KRW'
      }
    });

    console.log('사건이 성공적으로 접수되었습니다:', newCase.caseId);
    console.log('사건 번호:', newCase.caseNumber);

    // 단계 2: 소장 제출
    const complaint = await client.documents.file({
      caseId: newCase.caseId,
      type: 'pleading',
      title: '소장',
      file: await fs.readFile('./complaint.pdf'),
      metadata: {
        confidentiality: 'public',
        serveParties: newCase.parties.defendants.map(d => d.partyId)
      }
    });

    console.log('문서 접수됨:', complaint.documentId);

    // 단계 3: 증거 제출
    const evidence = await client.evidence.submit({
      caseId: newCase.caseId,
      type: 'digital',
      subtype: 'email',
      description: '계약 조건에 관한 이메일 서신',
      file: await fs.readFile('./evidence-emails.pdf'),
      chainOfCustody: [{
        custodian: 'did:wia:legal:forensics-001',
        action: 'collected',
        timestamp: new Date().toISOString()
      }]
    });

    console.log('증거 제출됨:', evidence.evidenceId);

    // 단계 4: 예비 심리 일정 조정
    const hearing = await client.hearings.schedule({
      caseId: newCase.caseId,
      type: 'preliminary',
      scheduledDate: '2026-03-15T10:00:00Z',
      mode: 'virtual',
      estimatedDuration: 60
    });

    console.log('심리가 예정되었습니다:', hearing.hearingId);
    console.log('가상 심리 URL:', hearing.virtualRoom.url);

    return {
      caseId: newCase.caseId,
      documentId: complaint.documentId,
      evidenceId: evidence.evidenceId,
      hearingId: hearing.hearingId
    };

  } catch (error) {
    console.error('사건 접수 중 오류 발생:', error.message);
    if (error.code === 'VALIDATION_ERROR') {
      console.error('유효성 검사 오류:', error.details);
    }
    throw error;
  }
}

4.3 웹훅 및 실시간 이벤트

4.3.1 웹훅 구성

웹훅은 사건 업데이트, 문서 접수, 심리 알림과 같은 법원 시스템 이벤트에 대한 실시간 알림을 가능하게 합니다. 시스템은 HTTPS POST 요청을 구성된 엔드포인트로 전송하여 애플리케이션이 자동으로 대응할 수 있도록 합니다.

표 4.1: 지원되는 웹훅 이벤트
이벤트 유형 설명 페이로드 포함 사항 일반적인 사용 사례
case.created 새 사건이 접수됨 사건 세부 정보, 당사자 자동 사건 관리 시스템 업데이트
case.updated 사건 정보가 변경됨 업데이트된 필드, 변경 이력 클라이언트 알림, 로그 업데이트
document.filed 새 문서가 접수됨 문서 메타데이터, 접수자 문서 검토 워크플로 트리거
document.served 문서가 당사자에게 송달됨 송달 확인, 타임스탬프 송달 추적, 법적 준수
hearing.scheduled 심리가 예정됨 심리 세부 정보, 참가자 캘린더 통합, 참가자 알림
hearing.updated 심리 시간/세부사항 변경됨 업데이트된 세부 정보 일정 재조정, 참가자 재알림
evidence.submitted 새 증거가 제출됨 증거 메타데이터, 보관 연쇄 증거 검토 프로세스 시작

// Express.js를 사용한 Node.js 웹훅 핸들러
import express from 'express';
import crypto from 'crypto';

const app = express();
app.use(express.json());

// 웹훅 서명 검증 미들웨어
function verifyWebhookSignature(req, res, next) {
  const signature = req.headers['x-wia-signature'];
  const timestamp = req.headers['x-wia-timestamp'];
  
  // 재생 공격 방지 (타임스탬프가 5분 이상 오래된 경우 거부)
  if (Date.now() - parseInt(timestamp) > 300000) {
    return res.status(401).send('타임스탬프가 너무 오래됨');
  }
  
  // 서명 계산
  const payload = `${timestamp}.${JSON.stringify(req.body)}`;
  const expectedSignature = crypto
    .createHmac('sha256', process.env.WEBHOOK_SECRET)
    .update(payload)
    .digest('hex');
  
  if (signature !== expectedSignature) {
    return res.status(401).send('잘못된 서명');
  }
  
  next();
}

// 웹훅 엔드포인트
app.post('/webhooks/court-events', 
  verifyWebhookSignature, 
  async (req, res) => {
    const { event, data } = req.body;

    try {
      switch (event) {

4.4 일괄 작업 및 대량 처리

4.4.1 일괄 문서 접수

대량의 문서를 접수하는 법률 회사의 경우 일괄 작업 API는 단일 요청으로 여러 문서를 제출할 수 있게 합니다:

async function batchFileDocuments(documents) {
  const batchRequest = {
    operations: documents.map(doc => ({
      operation: 'documents.file',
      data: {
        caseId: doc.caseId,
        type: doc.type,
        title: doc.title,
        file: doc.fileData,
        metadata: doc.metadata
      }
    }))
  };
  
  const result = await client.batch.execute(batchRequest);
  
  // 각 작업의 결과 확인
  result.results.forEach((res, index) => {
    if (res.success) {
      console.log(`문서 ${index + 1} 접수 성공:`, res.data.documentId);
    } else {
      console.error(`문서 ${index + 1} 접수 실패:`, res.error);
    }
  });
  
  return result;
}

4.5 오류 처리 및 재시도 전략

4.5.1 지수 백오프를 사용한 재시도

class RetryableAPIClient {
  constructor(baseClient, maxRetries = 3) {
    this.client = baseClient;
    this.maxRetries = maxRetries;
  }
  
  async executeWithRetry(operation, context = {}) {
    let lastError;
    
    for (let attempt = 0; attempt <= this.maxRetries; attempt++) {
      try {
        return await operation();
      } catch (error) {
        lastError = error;
        
        // 재시도 가능한 오류인지 확인
        if (!this.isRetryable(error) || attempt === this.maxRetries) {
          throw error;
        }
        
        // 지수 백오프 계산 (1초, 2초, 4초, 8초...)
        const delay = Math.min(1000 * Math.pow(2, attempt), 30000);
        
        console.log(
          `시도 ${attempt + 1} 실패, ${delay}ms 후 재시도...`,
          error.message
        );
        
        await this.sleep(delay);
      }
    }
    
    throw lastError;
  }
  
  isRetryable(error) {
    // 일시적인 오류만 재시도
    const retryableCodes = [408, 429, 500, 502, 503, 504];
    return retryableCodes.includes(error.statusCode);
  }
  
  sleep(ms) {
    return new Promise(resolve => setTimeout(resolve, ms));
  }
}

// 사용 예
const retryableClient = new RetryableAPIClient(client);

const result = await retryableClient.executeWithRetry(async () => {
  return await client.cases.create({
    // ... 사건 데이터
  });
});
