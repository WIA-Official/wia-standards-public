Chapter 5: 프로토콜 사양

5.1 통신 프로토콜 개요

디지털 법원 시스템의 핵심은 안전하고 신뢰할 수 있으며 효율적인 통신 프로토콜에 있습니다. WIA-LEGAL-001 3단계는 가상 심리, 안전한 메시지 전송, 문서 교환, 실시간 이벤트 스트리밍을 위한 포괄적인 프로토콜 세트를 정의합니다. 이러한 프로토콜은 법적 절차의 무결성과 기밀성을 유지하면서 최신 웹 표준을 기반으로 구축되었습니다.

프로토콜 계층은 네 가지 주요 통신 유형을 지원하도록 설계되었습니다: (1) 실시간 양방향 비디오/오디오 스트리밍, (2) 종단간 암호화 메시징, (3) 안전한 파일 전송, (4) 이벤트 기반 알림. 각 프로토콜은 특정 사용 사례에 최적화되어 있으며 법원 운영의 엄격한 보안, 프라이버시 및 감사 가능성 요구 사항을 충족합니다.

5.2 가상 심리 프로토콜 (WIA-LEGAL-001-VH)

5.2.1 WebRTC 기반 아키텍처

가상 심리 프로토콜은 WebRTC(Web Real-Time Communication) 기술을 활용하여 브라우저 및 모바일 애플리케이션에서 직접 피어 투 피어 미디어 스트리밍을 가능하게 합니다. 이 접근 방식은 대기 시간을 최소화하고 미디어 품질을 극대화하며 안전한 종단간 암호화를 보장합니다.

5.3 안전한 메시징 프로토콜 (WIA-LEGAL-001-SM)

5.3.1 Signal 프로토콜 기반 E2EE

법원 시스템 내의 모든 메시지는 Signal 프로토콜을 사용하여 종단간 암호화됩니다. 이는 Forward Secrecy를 제공하여 과거 메시지가 미래의 키 손상으로부터 안전하게 유지되도록 하며, Future Secrecy를 통해 하나의 메시지 키가 손상되어도 다른 메시지에 영향을 미치지 않도록 합니다.

5.4 문서 전송 프로토콜 (WIA-LEGAL-001-DT)

5.4.1 안전한 문서 업로드

대용량 법률 문서는 다중 파트 청크 업로드를 사용하여 전송되며, 각 청크는 무결성을 위해 개별적으로 해시되고 서명됩니다. 이를 통해 안정적인 재개 가능 업로드와 전송 중 손상 감지가 가능합니다. 

// 문서 전송 흐름
async function secureDocumentTransfer(document) {
  // 1단계: 문서 준비
  const preparation = {
    // SHA-256 해시 계산
    hash: await crypto.subtle.digest('SHA-256', document),

    // 메타데이터 추출
    metadata: {
      filename: document.name,
      size: document.size,
      mimeType: document.type,
      createdDate: document.lastModified
    },

    // Ed25519 디지털 서명
    signature: await signDocument(document, privateKey)
  };

  // 2단계: 암호화
  const encryption = {
    // AES-256 대칭 키 생성
    symmetricKey: await generateAES256Key(),

    // 문서 암호화
    encryptedDocument: await encryptAES256GCM(
      document,
      encryption.symmetricKey
    ),

    // 수신자의 RSA 공개 키로 대칭 키 암호화
    encryptedKey: await encryptRSAOAEP(
      encryption.symmetricKey,
      recipientPublicKey
    )
  };

  // 3단계: 청크 업로드
  const chunkSize = 5 * 1024 * 1024; // 5MB 청크
  const totalChunks = Math.ceil(
    encryption.encryptedDocument.size / chunkSize
  );

  for (let i = 0; i < totalChunks; i++) {
    const chunk = encryption.encryptedDocument.slice(
      i * chunkSize,
      (i + 1) * chunkSize
    );

    const chunkHash = await crypto.subtle.digest('SHA-256', chunk);

    await uploadChunk({
      documentId: preparation.metadata.documentId,
      chunkNumber: i,
      totalChunks: totalChunks,
      chunkData: chunk,
      chunkHash: chunkHash
    });
  }

  // 4단계: 서버 측 검증
  const verification = await verifyUpload({
    documentId: preparation.metadata.documentId,
    expectedHash: preparation.hash,
    signature: preparation.signature
  });

  // 5단계: 바이러스 스캔
  await scanForMalware(preparation.metadata.documentId);

  // 6단계: 수신자에게 알림
  await notifyRecipient({
    documentId: preparation.metadata.documentId,
    sender: currentUser.did,
    recipient: recipientDid,
    caseId: caseId
  });

  // 7단계: 감사 로그 기록
  await auditLog.record({
    action: 'document.uploaded',
    actor: currentUser.did,
    resource: preparation.metadata.documentId,
    timestamp: new Date().toISOString()
  });

  return {
    documentId: preparation.metadata.documentId,
    status: 'uploaded',
    verification: verification
  };
}

5.5 이벤트 스트리밍 프로토콜

5.5.1 실시간 업데이트

법원 시스템은 Server-Sent Events(SSE) 또는 WebSocket을 통해 실시간 업데이트를 제공하여 클라이언트가 사건 상태 변경, 문서 접수, 심리 알림 등에 즉시 대응할 수 있도록 합니다. 

// SSE 클라이언트 구현
class CourtEventStream {
  constructor(caseId, authToken) {
    this.caseId = caseId;
    this.authToken = authToken;
    this.eventSource = null;
    this.handlers = new Map();
  }

  connect() {
    const url = `https://api.digitalcourt.wia.org/v1/cases/${this.caseId}/events`;

    this.eventSource = new EventSource(url, {
      headers: {
        'Authorization': `Bearer ${this.authToken}`
      }
    });

    // 연결 이벤트
    this.eventSource.onopen = () => {
      console.log('이벤트 스트림 연결됨');
    };

    // 사건 업데이트
    this.eventSource.addEventListener('case.updated', (e) => {
      const data = JSON.parse(e.data);
      this.trigger('case.updated', data);
    });

    // 문서 접수
    this.eventSource.addEventListener('document.filed', (e) => {
      const data = JSON.parse(e.data);
      this.trigger('document.filed', data);
    });

    // 심리 시작
    this.eventSource.addEventListener('hearing.started', (e) => {
      const data = JSON.parse(e.data);
      this.trigger('hearing.started', data);
    });

    // 명령 발행
    this.eventSource.addEventListener('order.issued', (e) => {
      const data = JSON.parse(e.data);
      this.trigger('order.issued', data);
    });

    // 오류 처리
    this.eventSource.onerror = (error) => {
      console.error('이벤트 스트림 오류:', error);
      // 5초 후 재연결 시도
      setTimeout(() => this.reconnect(), 5000);
    };
  }

  on(eventType, handler) {
    if (!this.handlers.has(eventType)) {
      this.handlers.set(eventType, []);
    }
    this.handlers.get(eventType).push(handler);
  }

  trigger(eventType, data) {
    const handlers = this.handlers.get(eventType) || [];
    handlers.forEach(handler => handler(data));
  }

  disconnect() {
    if (this.eventSource) {
      this.eventSource.close();
    }
  }

  reconnect() {
    this.disconnect();
    this.connect();
  }
}

// 사용 예
const eventStream = new CourtEventStream(
  'case-550e8400-...',
  'eyJhbGciOiJIUzI1NiIs...'
);

eventStream.on('document.filed', (data) => {

5.6 블록체인 통합 프로토콜

5.6.1 불변 기록 저장

모든 중요한 법원 기록은 변조 방지를 위해 블록체인에 앵커링됩니다. 실제 문서는 IPFS 및 Filecoin에 저장되고, 해시와 메타데이터는 이더리움 호환 블록체인에 기록됩니다. 

// Solidity 스마트 계약 - 법원 기록
pragma solidity ^0.8.0;

contract CourtRecords {
    struct Record {
        bytes32 recordId;
        bytes32 caseId;
        string recordType; // case|document|hearing|order
        bytes32 contentHash; // SHA-256
        string ipfsHash; // IPFS CID
        address submittedBy;
        uint256 timestamp;
        bytes signature;
    }

    mapping(bytes32 => Record) public records;
    mapping(bytes32 => bytes32[]) public caseRecords;

    event RecordCreated(
        bytes32 indexed recordId,
        bytes32 indexed caseId,
        string recordType,
        address submittedBy
    );

    function createRecord(
        bytes32 _recordId,
        bytes32 _caseId,
        string memory _recordType,
        bytes32 _contentHash,
        string memory _ipfsHash,
        bytes memory _signature
    ) public {
        require(
            records[_recordId].recordId == bytes32(0),
            "Record already exists"
        );

        records[_recordId] = Record({
            recordId: _recordId,
            caseId: _caseId,
            recordType: _recordType,
            contentHash: _contentHash,
            ipfsHash: _ipfsHash,
            submittedBy: msg.sender,
            timestamp: block.timestamp,
            signature: _signature
        });

        caseRecords[_caseId].push(_recordId);

        emit RecordCreated(
            _recordId,
            _caseId,
            _recordType,
            msg.sender
        );
    }

    function getRecord(bytes32 _recordId)
        public
        view
        returns (Record memory)
    {
        return records[_recordId];
    }

    function getCaseRecords(bytes32 _caseId)
        public
        view
        returns (bytes32[] memory)
    {
        return caseRecords[_caseId];
    }
}

// JavaScript 클라이언트 통합
import { ethers } from 'ethers';
import { create } from 'ipfs-http-client';

async function archiveToBlockchain(document, caseId) {
  // IPFS에 문서 업로드
  const ipfs = create({ url: 'https://ipfs.infura.io:5001' });
  const { cid } = await ipfs.add(document);

  // 문서 해시 계산
  const hash = ethers.utils.sha256(document);

  // 서명 생성
  const signature = await signer.signMessage(hash);

  // 블록체인에 기록
  const contract = new ethers.Contract(
    contractAddress,
    contractABI,
    signer
  );

  const tx = await contract.createRecord(
    ethers.utils.id('record-' + Date.now()),
    caseId,
    'document',
    hash,
    cid.toString(),
    signature
  );

  await tx.wait();

  return {
    ipfsHash: cid.toString(),
    blockchainTx: tx.hash,
    contentHash: hash
  };
}

弘益人間(홍익인간) - 보편적 접근성

WIA-LEGAL-001의 프로토콜 설계는 다음을 통해 "널리 인간을 이롭게 하라"는 원칙을 구현합니다:

  • 개방형 표준: WebRTC, WebSocket, HTTPS와 같은 개방형 웹 표준을 사용하여 벤더 종속을 방지합니다.
  • 상호운용성: 모든 프로토콜은 다양한 플랫폼과 장치에서 작동하도록 설계되었습니다.
  • 접근성: 가상 심리는 물리적 출석이 어려운 사람들에게 법원 접근을 제공합니다.
  • 보안과 프라이버시: 강력한 암호화는 모든 사람의 법적 권리와 기밀성을 보호합니다.
  • 투명성: 블록체인 앵커링은 감사 가능하고 변조 방지된 기록을 보장합니다.

챕터 요약

핵심 요점:

  1. WIA-LEGAL-001-VH 프로토콜은 WebRTC와 DTLS-SRTP를 사용하여 안전한 저지연 가상 심리를 제공하며, 1080p 비디오와 고품질 오디오를 지원합니다.
  2. 안전한 메시징 프로토콜은 Signal 프로토콜을 기반으로 하며, X3DH 키 교환과 Double Ratchet 알고리즘을 통해 Forward 및 Future Secrecy를 제공합니다.

  5. 블록체인 통합은 IPFS/Filecoin 스토리지와 이더리움 호환 스마트 계약을 통해 불변하고 감사 가능한 기록 보존을 보장합니다.
  6. 모든 프로토콜은 개방형 표준을 기반으로 하여 弘益人間 철학에 따라 보편적 접근성과 상호운용성을 보장합니다.
