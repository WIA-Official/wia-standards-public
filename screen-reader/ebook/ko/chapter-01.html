[Screen Reader] 제1장: 스크린 리더 접근성 소개

> 弘益人間 (홍익인간) - 널리 인간을 이롭게 하라 ## 개요 스크린 리더는 시각장애인이 디지털 콘텐츠에 접근할 수 있게 해주는 필수적인 보조 기술입니다. WIA 스크린 리더 표준은 모든 플랫폼과 언어에서 일관된 접근성 경험을 제공하는 범용 프레임워크입니다. ## 범용 표준의 필요성 현재 스크린 리더 사용자들은 여러 가지 어려움에 직면해 있습니다: 1. **불일치한 발음**: 다른 스크린 리더가 같은 텍스트를 다르게 발음 2. **제한된 언어 지원**: 많은 스크린 리더가 비라틴 문자에 어려움을 겪음 3. **플랫폼 파편화**: 각 플랫폼마다 고유한 접근성 API 존재 4. **점자 비호환성**: 시스템마다 점자 형식이 다름 ## WIA 스크린 리더 솔루션 WIA 스크린 리더 표준은 다음을 통해 이러한 문제를 해결합니다: - **WIHP (WIA 국제 한글 발음)**: 211개 언어에 대한 일관된 발음 - **WIA 점자**: 모든 디스플레이와 호환되는 범용 점자 형식 - **크로스 플랫폼 API**: 모든 주요 플랫폼을 위한 통합 인터페이스 - **오픈 표준**: MIT 라이선스, 커뮤니티 주도 개발 ## 이 Ebook의 대상 이 Ebook은 다음을 위해 설계되었습니다: - 접근 가능한 애플리케이션을 구축하는 개발자 - 보조 기술 벤더 - 접근성 컨설턴트 - 접근성 표준을 구현하는 조직 - 디지털 접근성에 관심 있는 모든 사람 --- *다음: 제2장 - 보조기술의 현재 과제*

장 간 연결 안내

이 장은 WIA Screen Reader 8장 시리즈에 속합니다. 각 장은 단독으로 읽어도 되지만, 순서대로 읽으면 표준의 4-Phase 사양을 단계별로 이해할 수 있습니다.

  • Phase 1 (데이터 형식) — 발음 힌트, 접근성 트리 노드, 읽기 순서 계획, 점자 매핑, 사용자 프로파일, 텔레메트리 프레임 봉투.
  • Phase 2 (API 인터페이스) — NVDA / VoiceOver / Orca 로컬 IPC, 브라우저 확장 메시지 패싱, 클라우드 릴레이 HTTPS 엔드포인트.
  • Phase 3 (페더레이션 프로토콜) — 발음 힌트 공유, 디바이스 간 프로파일 이동, 텔레메트리 동의 강제, 큐레이터 신뢰 점수.
  • Phase 4 (통합) — NVDA 애드온, VoiceOver 데몬, TalkBack 모듈, Orca 플러그인, Chrome / Firefox / Edge 브라우저 확장, EPUB 3 접근성 통합, WIA 패밀리 합성 (WIHP·WIA Braille·WIA-OMNI-API·WIA-ACCESSIBILITY).

왜 이 표준이 중요한가

스크린 리더 접근성은 그동안 벤더별로 분리되어 있었습니다. NVDA 사용자가 만든 발음 사전을 VoiceOver 사용자가 쓸 수 없었고, 한국어 점자 8점 확장은 영어 Grade 2 점자기와 호환되지 않았습니다. WIA Screen Reader 는 이러한 자원이 이식 가능하도록 설계됐습니다 — 한국 의료 용어 큐레이터가 만든 발음 힌트가 안드로이드 TalkBack 사용자의 스페인어 키보드에서도 작동하는 것이 표준의 핵심 가치입니다.

211개 언어 지원 (WIHP 경유) 이 표준의 대표 기능이지만, 더 본질적인 설계 결정은 Phase 3 §5 의 사용자 프로파일 이식성입니다 — 사용자가 직장 NVDA, 가정 VoiceOver, 모바일 TalkBack 사이를 이동해도 음성 속도, 점자 등급, 힌트 구독을 잃지 않습니다.

구현자 노트

스크린 리더 통합에서 반복적으로 등장하는 세 가지 운영 관심사가 표준에 박혀 있습니다.

  1. 실시간 알림 지연 예산. 라이브 영역 (채팅 메시지, 폼 오류, 검색 결과) 은 사용자 인지 가능 지연 시간 안에 음성으로 도달해야 합니다. 표준은 polite 250 ms, assertive 100 ms, WIA 확장 gentle 500 ms 의 종단간 예산을 권고합니다.
  2. 발음 힌트 캐시 메모리 footprint. 두 명의 큐레이터를 구독한 한국 의료 독자는 5만 개 이상의 힌트를 캐시할 수 있습니다. 표준은 LRU 캐시 100 MB 상한, 사용자 선호 언어에 가중치를 둔 언어별 축출 우선순위를 권고합니다.
  3. 점자 디스플레이 I/O. 새로 고침 가능 점자 디스플레이는 USB / 블루투스를 통해 벤더별 프로토콜로 연결됩니다. 표준은 디스플레이 자체의 와이어 프로토콜을 정의하지 않습니다. 호스트가 발행하는 JSON 봉투만 정의하며, 드라이버 코드는 벤더별로 남습니다.

접근성 우선 설계

이 표준의 모든 결정은 "이것이 시각 장애 사용자에게 더 나은 경험인가, 아니면 단지 구현자에게 더 쉬운가?" 라는 질문을 거칩니다. 두 가지가 충돌할 때는 사용자가 이깁니다. 그래서 표준은 텔레메트리에 대한 명시적 동의를 강제하고, 호스트 측 데이터에 원본 페이지 URL 을 기록하지 않으며, 사용자가 특정 버튼이 왜 그 발음으로 읽히는지 디버깅할 수 있도록 name-source 출처를 노출하도록 요구합니다.
