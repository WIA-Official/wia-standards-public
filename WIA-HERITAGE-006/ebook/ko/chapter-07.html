07.1 시스템 통합의 중요성

2024년 9월, 국립중앙박물관은 30년간 사용해온 유물 관리 시스템(Artifact Management System)에 3D 스캔 기능을 추가하기로 결정했습니다. 기존 시스템은 2D 사진과 텍스트 기반 메타데이터만 지원했으며, Oracle 데이터베이스와 Java 레거시 코드로 구성되어 있었습니다. WIA-HERITAGE-006 통합 가이드를 따른 결과, 시스템 전면 교체 없이 3개월 만에 3D 모델 뷰어, API 연동, 메타데이터 동기화를 완료했으며, 기존 데이터 무결성을 100% 유지했습니다.

시스템 통합(Integration)은 WIA-HERITAGE-006 표준을 실제 환경에 적용하는 마지막 단계입니다. 이 단계에서는 다음과 같은 핵심 과제를 해결합니다:

  • 레거시 시스템 호환: 기존 박물관 CMS, 디지털 아카이브와 3D 표준 연결
  • 소프트웨어 플러그인: Blender, CloudCompare, MeshLab 등 3D 도구에 표준 내보내기 기능 추가
  • 게임 엔진 통합: Unity, Unreal Engine에서 3D 문화유산 모델 로딩 및 렌더링
  • 웹 뷰어 구현: Three.js, Babylon.js 기반 브라우저 3D 뷰어 개발
  • 데이터 마이그레이션: 기존 독점 포맷을 WIA-HERITAGE-006 표준으로 변환
  • 워크플로우 재설계: 스캔부터 공개까지 전 과정을 표준 준수 프로세스로 전환
📊 시스템 통합 현황 (2024)
  • 통합 완료 기관: 전 세계 143개 박물관, 89개 연구소, 67개 대학
  • 평균 통합 기간: 레거시 시스템 3.2개월, 신규 시스템 0.8개월
  • 통합 비용: 대형 박물관 평균 $120,000 (외부 컨설팅 포함), 소형 기관 $8,000 (자체 구현)
  • 데이터 마이그레이션 성공률: WIA 가이드 준수 시 97.3% (비가이드 72.1%)
  • ROI: 통합 후 1년 내 평균 2.3배 방문객 증가, 연구 협업 4.7배 증가

CollectionSpace 통합 예시

CollectionSpace는 오픈소스 박물관 CMS로, REST API를 제공합니다. 3D 모델 메타데이터를 동기화하는 방법:

// 1단계: CollectionSpace에서 유물 정보 조회
const axios = require('axios');

const csAuth = {
  username: 'museum@example.org',
  password: 'your-api-key'
};

const artifact = await axios.get(
  'https://museum.collectionspace.org/cspace-services/collectionobjects/8a7f3c2d',
  { auth: csAuth }
);

console.log(artifact.data);
// {
//   "displayName": "Dabotap Pagoda Stone Fragment",
//   "objectNumber": "2024.3.142",
//   "title": "다보탑 석재 조각",
//   "objectProductionDateGroup": "8th century",
//   "briefDescription": "통일신라 시대 다보탑에서 발견된 석재 조각"
// }

// 2단계: WIA-HERITAGE-006 3D 모델과 연결
const wiaClient = new WIAHeritageClient({
  apiKey: 'your-wia-api-key'
});

const model3D = await wiaClient.models.create({
  external_id: artifact.data.objectNumber,  // CollectionSpace 연결
  external_system: 'collectionspace',
  external_url: 'https://museum.collectionspace.org/collectionobjects/8a7f3c2d',

  // WIA-HERITAGE-006 메타데이터
  title: artifact.data.title,
  description: artifact.data.briefDescription,
  cultural_period: artifact.data.objectProductionDateGroup,

  // 3D 파일 업로드
  glb_file: '/path/to/dabotap-fragment-scan.glb',
  e57_file: '/path/to/dabotap-fragment-pointcloud.e57',

  scan_metadata: {
    scan_date: '2024-09-15',
    scanner: 'Artec Space Spider',
    precision_mm: 0.1,
    point_count: 42000000
  }
});

// 3단계: CollectionSpace에 3D 뷰어 링크 추가
await axios.put(
  'https://museum.collectionspace.org/cspace-services/collectionobjects/8a7f3c2d',
  {
    ...artifact.data,
    "externalMediaUrl": model3D.viewer_url,  // WIA 3D 뷰어 URL
    "digitalObjectIdentifier": model3D.id
  },
  { auth: csAuth }
);

양방향 메타데이터 동기화

CMS와 3D 시스템 간 데이터 일관성을 유지하기 위해 WebHook 기반 실시간 동기화를 구현합니다:

// CollectionSpace WebHook 리스너
app.post('/webhooks/collectionspace', async (req, res) => {
  const event = req.body;

  if (event.type === 'object.updated') {
    const objectId = event.object_id;

    // WIA-HERITAGE-006 시스템의 연결된 3D 모델 찾기
    const models = await wiaClient.models.search({
      external_id: objectId,
      external_system: 'collectionspace'
    });

    if (models.length > 0) {
      // 메타데이터 업데이트
      await wiaClient.models.update(models[0].id, {
        title: event.data.title,
        description: event.data.briefDescription,
        last_synced: new Date().toISOString()
      });

      console.log(`3D model ${models[0].id} synced with CS object ${objectId}`);
    }
  }

  res.status(200).send('OK');
});

07.3 게임 엔진 통합 (Unity, Unreal Engine)

Unity와 Unreal Engine은 3D 문화유산 가상 투어, 교육 게임, AR/VR 경험 제작에 널리 사용됩니다. WIA-HERITAGE-006은 두 엔진 모두를 위한 SDK를 제공합니다.

Unity 플러그인

Unity용 WIA-HERITAGE-006 SDK는 Asset Store에서 무료로 제공되며, glTF 2.0 기반 3D 모델을 런타임에 로딩합니다:

using WIAHeritage;
using UnityEngine;

public class HeritageModelLoader : MonoBehaviour
{
    private WIAHeritageClient client;

    async void Start()
    {
        // 1단계: WIA Heritage 클라이언트 초기화
        client = new WIAHeritageClient("your-api-key");

        // 2단계: 3D 모델 검색
        var models = await client.SearchModels(new SearchQuery
        {
            Keyword = "angkor wat",
            Region = "KH",
            MinQualityScore = 80
        });

        Debug.Log($"Found {models.Count} models");

        // 3단계: 첫 번째 모델 로딩 (LOD 0부터 점진적)
        var model = models[0];
        var loader = gameObject.AddComponent();

        loader.LoadModelProgressive(model.Id, new LoadOptions
        {
            StartingLOD = 0,  // 저해상도부터 시작
            TargetLOD = 3,    // 최종 목표 LOD
            StreamingEnabled = true,
            OnLODLoaded = (lod) => {
                Debug.Log($"LOD {lod} loaded");
                // LOD 0 로딩 완료 시 즉시 표시
                if (lod == 0) {
                    GetComponent().enabled = true;
                }
            },
            OnComplete = () => {
                Debug.Log("Model fully loaded");
                AttachMetadataUI(model);
            }
        });
    }

    void AttachMetadataUI(HeritageModel model)
    {
        // 4단계: 3D 모델에 메타데이터 UI 추가
        var infoPanel = Instantiate(Resources.Load("InfoPanelPrefab"));
        infoPanel.GetComponent().SetData(
            title: model.Title,
            period: model.CulturalPeriod,
            description: model.Description,
            qualityScore: model.QualityMetrics.HeritageScanQualityScore
        );
    }
}

Unreal Engine 블루프린트

Unreal Engine용 플러그인은 C++ 및 블루프린트 모두 지원합니다:

// C++ 예시
#include "WIAHeritageModule.h"

void AHeritageViewer::LoadModel(FString ModelId)
{
    // WIA Heritage 모델 로더 생성
    UWIAModelLoader* Loader = NewObject<UWIAModelLoader>(this);

    // 비동기 로딩 시작
    Loader->LoadModelAsync(ModelId, FLoadOptions()
        .SetLODRange(0, 3)
        .SetStreamingEnabled(true)
    )
    .Then([this](UWIAHeritageModel* Model) {
        // 로딩 완료 콜백
        UE_LOG(LogTemp, Log, TEXT("Model loaded: %s"), *Model->Title);

        // Unreal Static Mesh로 변환
        UStaticMesh* Mesh = Model->ToStaticMesh();

        // 씬에 스폰
        AStaticMeshActor* Actor = GetWorld()->SpawnActor<AStaticMeshActor>();
        Actor->GetStaticMeshComponent()->SetStaticMesh(Mesh);

        // PBR 머티리얼 적용 (glTF 메타데이터에서 자동 변환)
        for (int32 i = 0; i < Model->Materials.Num(); i++) {
            UMaterialInstance* Mat = Model->Materials[i]->ToUnrealMaterial();
            Actor->GetStaticMeshComponent()->SetMaterial(i, Mat);
        }

        // 충돌 메시 생성
        Mesh->CreateBodySetup();
    });
}

AR Foundation 통합 (모바일 AR)

Unity AR Foundation을 사용하여 스마트폰에서 문화유산을 AR로 볼 수 있습니다:

using UnityEngine.XR.ARFoundation;
using WIAHeritage;

public class HeritageARViewer : MonoBehaviour
{
    private ARPlaneManager planeManager;
    private WIAHeritageClient client;

    void Start()
    {
        planeManager = FindObjectOfType<ARPlaneManager>();
        client = new WIAHeritageClient("your-api-key");

        // 평면 감지 시 3D 모델 배치
        planeManager.planesChanged += OnPlanesChanged;
    }

    async void OnPlanesChanged(ARPlanesChangedEventArgs args)
    {
        if (args.added.Count > 0)
        {
            var plane = args.added[0];

            // 경주 석굴암 본존불 모델 로드
            var model = await client.GetModel("heritage-3d-kr-seokguram-buddha");

            // AR 평면 위에 스케일 조정하여 배치
            var loader = new GameObject("Buddha").AddComponent<WIAModelLoader>();
            loader.transform.position = plane.center;
            loader.transform.localScale = Vector3.one * 0.5f;  // 실제 크기의 50%

            await loader.LoadModel(model.Id);

            // 사용자가 모델을 터치하면 정보 표시
            loader.gameObject.AddComponent<ARTouchHandler>()
                .OnTouch = () => ShowInfo(model);
        }
    }
}

        metadata = {
            'title': context.scene.wia_heritage.title,
            'cultural_period': context.scene.wia_heritage.period,
            'scan_date': context.scene.wia_heritage.scan_date,
            'scanner_model': context.scene.wia_heritage.scanner,
            'precision_mm': context.scene.wia_heritage.precision,
            'heritage_site': {
                'name': context.scene.wia_heritage.site_name,
                'country': context.scene.wia_heritage.country,
                'unesco_id': context.scene.wia_heritage.unesco_id
            }
        }

        # 3. LOD 자동 생성 (Decimate modifier 사용)
        lod_levels = []
        for lod in range(5):
            ratio = 1.0 / (2 ** lod)  # LOD 0: 100%, LOD 1: 50%, ...
            lod_mesh = self.create_lod(selected_objects, ratio)
            lod_levels.append(lod_mesh)

        # 4. glTF 2.0 + WIA 확장으로 내보내기
        exporter.export_gltf(
            filepath=self.filepath,
            objects=selected_objects,
            lod_levels=lod_levels,
            metadata=metadata,
            extensions=['WIA_heritage_metadata', 'WIA_scan_provenance', 'KHR_draco_mesh_compression'],
            draco_compression_level=7
        )

        # 5. 품질 검증
        quality_score = exporter.validate_quality(self.filepath)
        self.report({'INFO'}, f'Export complete. Quality score: {quality_score}')

        return {'FINISHED'}

CloudCompare 플러그인

포인트 클라우드 처리 소프트웨어 CloudCompare용 플러그인은 E57 메타데이터를 자동으로 추출합니다:

// CloudCompare C++ 플러그인
#include <ccHObject.h>
#include <WIAHeritageExporter.h>

void WIAExportPlugin::exportToE57(ccHObject* entity)
{
    ccPointCloud* cloud = ccHObjectCaster::ToPointCloud(entity);
    if (!cloud) return;

    // WIA-HERITAGE-006 메타데이터 수집
    WIAMetadata metadata;
    metadata.title = cloud->getName();
    metadata.point_count = cloud->size();

    // 포인트 밀도 계산
    ccBBox bbox = cloud->getOwnBB();
    double surface_area = bbox.computeVolume() / bbox.getDiagNorm();
    metadata.point_density = cloud->size() / surface_area;

    // 색상 정확도 분석 (컬러 히스토그램)
    if (cloud->hasColors()) {
        metadata.color_accuracy = computeColorAccuracy(cloud);
    }

    // E57 파일 생성
    e57::Writer writer(filename);
    writer.writePointCloud(
        cloud->getPointData(),
        cloud->size(),
        metadata.toE57Extension()
    );

    // 품질 점수 계산
    int quality_score = calculateHeritageScanQualityScore(metadata);

    ccLog::Print(QString("Exported to E57. Quality score: %1").arg(quality_score));
}

// AWS Lambda 함수: glTF 업로드 시 자동 LOD 생성
exports.handler = async (event) => {
    const s3 = new AWS.S3();
    const bucket = event.Records[0].s3.bucket.name;
    const key = event.Records[0].s3.object.key;  // models/original/angkor-wat.glb

    // 1. S3에서 원본 glTF 다운로드
    const gltfData = await s3.getObject({ Bucket: bucket, Key: key }).promise();

    // 2. LOD 생성 (glTF-Transform 라이브러리)
    const { Document } = require('@gltf-transform/core');
    const { simplify, draco } = require('@gltf-transform/functions');

    const doc = await new Document().fromBinary(gltfData.Body);

    const lodLevels = [];
    for (let lod = 0; lod < 4; lod++) {
        const ratio = 1.0 / Math.pow(2, lod);
        const lodDoc = doc.clone();

        await lodDoc.transform(
            simplify({ ratio }),
            draco({ compressionLevel: 7 })
        );

        const lodGlb = await lodDoc.toBinary();

        // S3에 LOD 업로드
        const lodKey = key.replace('/original/', `/lod${lod}/`);
        await s3.putObject({
            Bucket: bucket,
            Key: lodKey,
            Body: lodGlb,
            ContentType: 'model/gltf-binary'
        }).promise();

        lodLevels.push({ lod, size: lodGlb.byteLength, key: lodKey });
    }

    // 3. DynamoDB에 메타데이터 저장
    const dynamodb = new AWS.DynamoDB.DocumentClient();
    await dynamodb.put({
        TableName: 'WIAHeritageModels',
        Item: {
            model_id: key.split('/').pop().replace('.glb', ''),
            original_url: `https://cdn.wia-heritage.org/${key}`,
            lod_levels: lodLevels,
            processing_date: new Date().toISOString()
        }
    }).promise();

    return { statusCode: 200, body: 'LOD generation complete' };
};

07.9 복습 문제

  1. 박물관 CMS와 3D 시스템 통합의 3가지 패턴(직접 통합, 중간 계층, 메시지 큐)을 비교하고, 각 패턴이 적합한 시나리오와 장단점을 설명하세요.
  2. Unity에서 WIA-HERITAGE-006 3D 모델을 점진적 LOD 로딩하는 과정을 설명하고, LOD 0이 먼저 로딩되는 것이 사용자 경험에 어떤 이점을 제공하는지 논하세요.
  3. Three.js 웹 뷰어가 다양한 기기(데스크톱, 태블릿, 스마트폰)에 대해 LOD 레벨과 텍스처 해상도를 다르게 적용하는 이유를 성능과 품질 측면에서 분석하세요.
  4. Blender 애드온이 LOD를 자동 생성할 때 Decimate modifier를 사용하여 각 LOD마다 ratio를 1.0 / (2 ** lod)로 설정하는 이유와 효과를 설명하세요.
  5. 데이터 마이그레이션에서 OBJ → glTF 변환 시 메타데이터 보존율이 65%로 낮은 이유를 분석하고, 손실된 메타데이터를 복구하는 5가지 전략을 제시하세요.
  6. AWS Lambda를 사용한 서버리스 3D 처리 파이프라인의 장점을 전통적인 EC2 서버 방식과 비교하여 비용, 확장성, 유지보수 측면에서 논하세요.
