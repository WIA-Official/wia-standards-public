WIA-HERITAGE-006: Heritage 3D Scan Standard

06.1 프로토콜의 역할과 중요성

2024년 10월, 일본 도쿄국립박물관과 한국 국립중앙박물관은 고대 한일 문화교류 전시를 위해 각각 보유한 백제와 아스카 시대 유물 3D 모델을 교환하기로 했습니다. 그러나 두 기관은 서로 다른 3D 스캔 장비, 소프트웨어, 저장 시스템을 사용하고 있었습니다. WIA-HERITAGE-006 프로토콜을 도입한 결과, 단 3일 만에 총 127개 유물의 고정밀 3D 모델을 양방향으로 완벽하게 전송했으며, 메타데이터 손실 없이 각 기관의 시스템에서 즉시 활용할 수 있었습니다.

프로토콜(Protocol)은 서로 다른 시스템이 데이터를 교환할 때 따라야 하는 약속된 규칙입니다. API가 "무엇을" 요청할 수 있는지 정의한다면, 프로토콜은 "어떻게" 전송하고 해석할지를 정의합니다. 문화유산 3D 스캔 분야에서 프로토콜은 다음과 같은 핵심 문제를 해결합니다:

  • 데이터 포맷 통일: glTF, E57, LAZ 등 여러 포맷 간 무손실 변환 규칙
  • 전송 효율성: 수백 GB 데이터를 네트워크로 전송할 때 압축, 분할, 재개 메커니즘
  • 동기화: 여러 기관이 동일한 유물을 스캔할 때 버전 충돌 해결
  • 보안: 전송 중 데이터 암호화, 위변조 방지, 접근 로그 기록
  • 상호운용성: 제조사가 다른 장비와 소프트웨어 간 완벽한 호환성
📊 프로토콜 표준화 효과 (2024)
  • 데이터 교환 성공률: WIA-HERITAGE-006 준수 시스템 98.7% (비표준 시스템 67.3%)
  • 평균 전송 시간: 100GB 3D 모델 기준 42분 (기존 방식 3시간 15분)
  • 메타데이터 손실률: 0.03% (기존 방식 23.7%)
  • 국가 간 협업 프로젝트: 2024년 847건 (2020년 89건에서 852% 증가)
  • 프로토콜 준수 플랫폼: 전 세계 143개 박물관, 89개 연구소, 234개 소프트웨어

WIA-HERITAGE-006 프로토콜 스택

WIA-HERITAGE-006은 OSI 7계층 모델에 영감을 받아 문화유산 3D 데이터에 특화된 5계층 프로토콜 스택을 정의합니다:

06.2 glTF 스트리밍 프로토콜

glTF(GL Transmission Format) 2.0은 3D 모델 전송을 위한 웹 표준입니다. WIA-HERITAGE-006은 glTF를 확장하여 문화유산 특화 기능을 추가했습니다.

glTF 확장 사양

WIA-HERITAGE-006은 다음 glTF 확장을 표준으로 정의합니다:

확장 이름목적주요 필드
WIA_heritage_metadata문화유산 메타데이터cultural_period, unesco_id, significance_level
WIA_scan_provenance스캔 이력 추적scanner_model, scan_date, operator, GPS_coords
WIA_quality_metrics품질 메트릭point_density, precision_mm, coverage_percent
WIA_lod_hierarchyLOD 계층 구조lod_levels[0-5], auto_switch_distance
KHR_mesh_quantization기하 압축position_bits=16, normal_bits=10, texcoord_bits=12
KHR_draco_mesh_compressionDraco 압축compression_level=7 (문화유산용 고품질)
KHR_texture_basisu텍스처 압축Basis Universal, GPU 가속 디코딩

Draco 압축 최적화

Draco는 Google이 개발한 3D 메시 압축 알고리즘으로, 기하 데이터를 90% 이상 압축할 수 있습니다. WIA-HERITAGE-006은 문화유산 특화 Draco 설정을 정의합니다:

{
  "extensions": {
    "KHR_draco_mesh_compression": {
      "bufferView": 0,
      "attributes": {
        "POSITION": 0,
        "NORMAL": 1,
        "TEXCOORD_0": 2
      },
      "WIA_heritage_settings": {
        "quantization_bits": {
          "position": 16,  // 0.015mm 정밀도 유지
          "normal": 10,    // 법선 벡터 1도 이하 오차
          "texcoord": 12   // UV 좌표 1/4096 정밀도
        },
        "compression_level": 7,  // 0-10, 7=고품질
        "preserve_triangle_order": true,  // 원본 토폴로지 유지
        "sequential_encoding": false  // 랜덤 액세스 지원
      }
    }
  }
}
🎯 실전 사례: 경주 석굴암 본존불 압축

국립경주박물관은 석굴암 본존불 3D 모델(원본 47GB)을 Draco 압축으로 3.2GB로 축소했습니다(93% 압축). 압축 후에도 0.08mm RMS 오차를 유지하여 Heritage Scan Quality Score 91점을 달성했습니다. 이를 통해 모바일 기기에서도 실시간 렌더링이 가능해졌습니다.

06.3 E57 포인트 클라우드 교환 표준

포인트 클라우드 압축 알고리즘

WIA-HERITAGE-006은 다음 압축 알고리즘을 표준으로 지원합니다:

  • LAZ (LASzip): 무손실 압축, 평균 압축률 7:1~20:1, 실시간 디코딩 가능
  • Draco Point Cloud: 양자화 기반 손실/무손실 선택, 압축률 30:1 가능
  • JPEG-XL Point Cloud: 색상 정보 압축 특화, 텍스처 압축률 50:1
  • Google Copc (Cloud Optimized Point Cloud): HTTP Range Request 지원, 타일 기반 스트리밍

06.4 WebRTC 실시간 협업 프로토콜

2024년 11월, 프랑스 루브르와 이집트 카이로 박물관의 고고학자들은 로제타 스톤의 3D 스캔 데이터를 함께 분석하면서 실시간으로 화상 회의와 3D 주석을 공유했습니다. 이는 WIA-HERITAGE-006의 WebRTC 기반 협업 프로토콜을 통해 구현되었습니다.

WebRTC 협업 아키텍처

WebRTC(Web Real-Time Communication)는 브라우저 간 P2P 통신을 가능하게 하는 표준입니다. WIA-HERITAGE-006은 이를 확장하여 3D 모델 협업을 지원합니다:

// 1단계: 협업 세션 시작
const session = await WIAHeritage.createCollaborationSession({
  model_id: 'heritage-3d-eg-rosetta-stone',
  participants: [
    'user-louvre-curator@example.com',
    'user-cairo-archaeologist@example.com'
  ],
  permissions: {
    view: true,
    annotate: true,
    edit_metadata: false,
    download: false
  }
});

// 2단계: WebRTC 피어 연결 설정
const peerConnection = new RTCPeerConnection({
  iceServers: [
    {urls: 'stun:stun.wia-heritage.org:3478'},
    {urls: 'turn:turn.wia-heritage.org:3478',
     username: 'heritage', credential: '...'}
  ]
});

// 3단계: 3D 데이터 채널 생성
const dataChannel = peerConnection.createDataChannel('3d-sync', {
  ordered: false,  // 순서 보장 불필요 (최신 상태만 중요)
  maxRetransmits: 0  // 재전송 없음 (실시간성 우선)
});

// 4단계: 카메라 동기화
viewer.camera.on('change', (cameraState) => {
  dataChannel.send(JSON.stringify({
    type: 'camera_update',
    position: cameraState.position,
    target: cameraState.target,
    timestamp: Date.now()
  }));
});

// 5단계: 3D 주석 공유
viewer.on('annotation_created', (annotation) => {
  dataChannel.send(JSON.stringify({
    type: 'annotation',
    position_3d: annotation.position,
    text: annotation.text,
    author: session.currentUser,
    timestamp: Date.now()
  }));
});

스마트 컨트랙트 기반 라이선스 관리

3D 모델의 사용 권한을 스마트 컨트랙트로 자동화합니다:

contract HeritageModelLicense {
  struct License {
    address owner;
    string modelId;
    LicenseType licenseType;
    bool commercialUse;
    bool derivativeWorks;
    bool attribution;
    uint256 expirationDate;
    uint256 downloadLimit;
  }

  enum LicenseType {
    CC0,           // Public Domain
    CC_BY,         // Attribution
    CC_BY_NC,      // Attribution, Non-Commercial
    CC_BY_ND,      // Attribution, No Derivatives
    RESTRICTED     // Custom terms
  }

  mapping(string => License) public licenses;

  function checkAccess(
    string memory modelId,
    address user,
    AccessType accessType
  ) public view returns (bool) {
    License memory license = licenses[modelId];

    if (block.timestamp > license.expirationDate) {
      return false;
    }

    if (accessType == AccessType.COMMERCIAL && !license.commercialUse) {
      return false;
    }

    // 문화적 민감성 체크 (원주민 커뮤니티 동의 필요)
    if (culturallySensitive[modelId]) {
      return hasCommunitApproval(modelId, user);
    }

    return true;
  }
}

06.6 HTTP/3 및 QUIC 기반 고속 전송

HTTP/3는 QUIC(Quick UDP Internet Connections) 프로토콜 위에서 작동하는 차세대 웹 전송 표준입니다. WIA-HERITAGE-006은 대용량 3D 파일 전송에 HTTP/3를 활용합니다.

06.7 CDN 및 P2P 하이브리드 배포

대용량 3D 모델을 전 세계에 배포하려면 CDN(Content Delivery Network)과 P2P(Peer-to-Peer) 네트워크를 결합한 하이브리드 방식이 효율적입니다.

하이브리드 배포 전략

배포 방식적용 시나리오대역폭 비용전송 속도
중앙 서버초기 업로드, 백업높음 ($0.08/GB)느림 (1개 소스)
CDN 엣지인기 모델, 공식 배포중간 ($0.02/GB)빠름 (지리적 근접)
P2P WebTorrent대용량, 많은 다운로드낮음 ($0.001/GB)매우 빠름 (다중 소스)

P2P 3D 모델 공유 프로토콜

// WebTorrent 기반 3D 모델 공유
import WebTorrent from 'webtorrent';

const client = new WebTorrent();

// 1. 시드(제공자)가 3D 모델을 공유
client.seed('/path/to/angkor-wat-full-scan.glb', {
  name: 'Angkor Wat Main Temple - Full Scan',
  comment: 'WIA-HERITAGE-006 certified scan, 2024-06-15',
  wia_heritage_metadata: {
    model_id: 'heritage-3d-kh-angkor-wat-001',
    license: 'CC-BY-NC-4.0',
    quality_score: 94,
    blockchain_hash: 'a7f3c2d8e4b9...'
  }
}, (torrent) => {
  console.log('Magnet URI:', torrent.magnetURI);
  // magnet:?xt=urn:btih:...&dn=Angkor+Wat...
});

// 2. 리처(다운로드)가 3D 모델 받기
client.add('magnet:?xt=urn:btih:...', {
  path: '/downloads/heritage-models'
}, (torrent) => {
  // 점진적 다운로드 + 렌더링
  torrent.files.forEach(file => {
    if (file.name.endsWith('.glb')) {
      const stream = file.createReadStream();
      viewer.loadFromStream(stream);  // 다운로드 중에도 렌더링 시작
    }
  });

  // 다운로드 진행률
  torrent.on('download', (bytes) => {
    console.log('Progress:', (torrent.progress * 100).toFixed(1) + '%');
    console.log('Speed:', (torrent.downloadSpeed / 1024 / 1024).toFixed(2) + ' MB/s');
    console.log('Peers:', torrent.numPeers);
  });
});

IPFS 영구 보존

InterPlanetary File System (IPFS)은 콘텐츠 주소 기반 분산 파일 시스템으로, 문화유산 데이터의 영구 보존에 적합합니다:

// IPFS에 3D 모델 업로드
const ipfs = await IPFS.create();

const file = await ipfs.add({
  path: 'angkor-wat-main-temple.glb',
  content: fs.readFileSync('/path/to/model.glb')
}, {
  wrapWithDirectory: true,
  pin: true,  // 영구 보존
  cidVersion: 1
});

// 고유 CID (Content Identifier)
console.log('IPFS CID:', file.cid.toString());
// bafybeig3c2d7a4e8f9...

// WIA-HERITAGE-006 메타데이터와 함께 저장
await ipfs.files.write('/heritage-models/angkor-wat-001/metadata.json',
  JSON.stringify({
    model_cid: file.cid.toString(),
    wia_standard: 'WIA-HERITAGE-006 v2.0',
    unesco_id: '668',
    blockchain_record: '0xa7f3c2d8e4b9...',
    license: 'CC-BY-NC-4.0'
  }), {
  create: true,
  parents: true
});

// 다른 사용자가 IPFS에서 다운로드
const stream = ipfs.cat('bafybeig3c2d7a4e8f9...');
for await (const chunk of stream) {
  viewer.appendData(chunk);
}

06.9 복습 문제

  1. WIA-HERITAGE-006 프로토콜 스택의 5계층 구조를 설명하고, 각 계층이 3D 문화유산 데이터 전송에서 담당하는 역할을 서술하세요.
  2. glTF 점진적 LOD 스트리밍에서 클라이언트가 카메라를 이동할 때 서버가 "관심 영역(ROI)" 고해상도 데이터만 전송하는 프로토콜을 설명하고, 대역폭 절감 효과를 분석하세요.
  3. E57 포인트 클라우드에서 Octree, kd-tree, 3D R-tree, Morton code 인덱싱 방법의 차이를 비교하고, 각각이 적합한 포인트 수 규모와 이유를 설명하세요.
  4. WebRTC 협업 프로토콜에서 데이터 채널의 ordered: false, maxRetransmits: 0 설정이 왜 필요한지 실시간 3D 협업 특성과 연관지어 논하세요.
  5. 블록체인 기반 출처 추적에서 스마트 컨트랙트가 문화적 민감성을 관리하는 방법을 설명하고, 원주민 커뮤니티 권리 보호에 어떻게 기여하는지 서술하세요.
  6. HTTP/3 (QUIC)이 HTTP/2 (TCP)보다 대용량 3D 파일 전송에 유리한 이유를 Connection Migration, Head-of-line blocking 해결, 0-RTT 연결 측면에서 분석하세요.
