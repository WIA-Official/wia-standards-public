WIA-HERITAGE-006 표준 개요

WIA-HERITAGE-006: Heritage 3D Scan Standard

03.1 표준의 전체 아키텍처

WIA-HERITAGE-006은 4단계 계층 아키텍처(Phase 1-4)로 설계되었으며, 각 단계는 독립적으로 구현 가능하면서도 통합 시 시너지를 발휘합니다. 이 접근법은 소규모 박물관부터 대형 국제 컨소시엄까지 모든 규모의 조직이 자신의 역량에 맞게 표준을 채택할 수 있도록 합니다.

단계명칭핵심 기능구현 난이도필수 여부
Phase 1데이터 형식E57 포인트 클라우드, glTF 2.0 메시, 메타데이터 스키마⭐ 쉬움필수
Phase 2API 인터페이스RESTful 업로드/다운로드, 검색, 품질 검증 엔드포인트⭐⭐ 보통권장
Phase 3프로토콜실시간 3D 스트리밍, WebRTC 포인트 클라우드, P2P 동기화⭐⭐⭐ 어려움선택
Phase 4통합블록체인 출처 추적, AI 분석, 디지털 트윈, WebXR⭐⭐⭐⭐ 매우 어려움선택

점진적 채택 경로

WIA-HERITAGE-006의 핵심 설계 철학은 "점진적 향상(Progressive Enhancement)"입니다. 기관은 Phase 1만 구현해도 즉시 혜택을 받으며, 준비가 되면 상위 단계로 확장할 수 있습니다.

🚀 채택 로드맵
  • 1-3개월: Phase 1 구현 - 기존 3D 모델을 표준 포맷으로 변환, 메타데이터 추가
  • 3-6개월: Phase 2 추가 - 웹 기반 업로드/다운로드 API 구축, 품질 검증 자동화
  • 6-12개월: Phase 3 추가 - WebXR 뷰어 통합, 실시간 스트리밍 프로토콜 적용
  • 12개월+: Phase 4 추가 - AI 기반 자동 분류, 블록체인 출처 추적, 디지털 트윈 구축

2024년 기준, 조기 채택자 중 78%는 Phase 1-2를 구현했고, 34%는 Phase 3까지, 12%만이 완전한 Phase 4를 달성했습니다. 이는 표준의 유연성을 보여주는 증거입니다.

03.2 핵심 설계 원칙

WIA-HERITAGE-006은 5가지 핵심 설계 원칙에 기반하여 개발되었습니다. 이 원칙들은 표준의 모든 기술적 결정에 일관되게 적용됩니다.

1. 개방성 (Openness)

모든 포맷과 프로토콜은 개방형 표준을 기반으로 합니다. 독점 기술이나 특허가 필요한 요소는 포함하지 않습니다.

  • E57은 ASTM International 표준
  • glTF 2.0은 Khronos Group 로열티 프리 표준
  • 모든 API는 OpenAPI 3.0 스펙 준수
  • 소스 코드는 MIT 라이선스로 GitHub 공개

2. 상호운용성 (Interoperability)

다양한 스캐너, 소프트웨어, 플랫폼 간 완벽한 호환성을 보장합니다.

  • Faro, Leica, Trimble 등 주요 스캐너 제조사 포맷에서 변환 가능
  • Blender, CloudCompare, MeshLab 등 오픈소스 도구와 호환
  • Three.js, Babylon.js, Unity, Unreal Engine에서 직접 로딩 가능
  • CIDOC-CRM, Dublin Core 메타데이터와 매핑 지원

3. 확장성 (Scalability)

소형 유물(수 MB)부터 대규모 유적지(수 TB)까지 모든 규모에서 효율적으로 작동합니다.

  • LOD (Level of Detail) 자동 생성으로 점진적 로딩
  • Draco 압축으로 메시 크기 90% 감소
  • KTX2/Basis Universal 텍스처로 GPU 메모리 최적화
  • Spatial partitioning (octree, kd-tree)으로 대용량 포인트 클라우드 처리

4. 장기 보존 (Long-term Preservation)

50년 이상 데이터 가독성을 보장하는 구조입니다.

  • 문서화된 개방형 포맷으로 미래 호환성 확보
  • 버전 관리 및 하위 호환성 정책 명시
  • 자체 설명적(self-describing) 메타데이터 스키마
  • 독립적 검증 가능한 체크섬 및 무결성 검증

5. 접근성 (Accessibility)

기술적, 경제적, 지리적 장벽을 최소화합니다.

  • 저사양 디바이스(스마트폰, 태블릿)에서도 작동
  • 저대역폭 네트워크(3G 이하)에서도 접근 가능
  • 무료 도구와 라이브러리 제공
  • 6개 주요 언어(영어, 한국어, 중국어, 아랍어, 스페인어, 프랑스어) 문서화

03.3 기존 표준과의 통합

WIA-HERITAGE-006은 처음부터 새로운 표준을 만드는 대신, 검증된 기존 표준들을 통합하고 확장하는 접근을 취했습니다. 이를 통해 개발 시간을 단축하고 기존 생태계와의 호환성을 보장합니다.

포인트 클라우드: E57

ASTM E2807 표준인 E57은 3D 이미징 데이터를 위한 가장 포괄적인 개방형 포맷입니다. WIA-HERITAGE-006은 E57을 포인트 클라우드 저장의 주요 포맷으로 채택하고, 문화유산 특화 확장을 추가했습니다.

E57 기본 기능WIA-HERITAGE-006 확장
3D 좌표 (XYZ)문화유산 좌표계 (WGS84, 지역 투영법)
RGB 색상멀티스펙트럴 채널 (UV, IR, 형광)
강도(Intensity)재질 분류 (석재, 목재, 금속 등)
스캔 위치/방향스캔 조건 (조명, 온도, 습도)
제조사 메타데이터문화유산 메타데이터 (시대, 양식, 보존 상태)

메시 모델: glTF 2.0

Khronos Group의 glTF 2.0은 "3D의 JPEG"로 불리는 웹 최적화 포맷입니다. WIA-HERITAGE-006은 glTF 2.0을 메시 모델의 표준으로 지정하고, PBR(Physically Based Rendering) 확장을 활용합니다.

  • 기본 구조: 노드 계층, 메시, 재질, 텍스처, 애니메이션
  • 확장 활용:
    • KHR_mesh_quantization: 정밀도 손실 없이 파일 크기 40% 감소
    • KHR_texture_basisu: 범용 GPU 텍스처 압축
    • KHR_materials_unlit: 단순한 색상 렌더링 (고대 벽화 등)
    • EXT_meshopt_compression: 추가 90% 압축 (네트워크 전송용)
  • 문화유산 확장 (WIA 제안):
    • WIA_heritage_metadata: 발굴 정보, 보존 이력
    • WIA_heritage_annotations: 3D 주석, 손상 영역 표시
    • WIA_heritage_provenance: 블록체인 해시, 라이선스 정보

지리 데이터: LAS/LAZ

ASPRS의 LAS(LASer) 포맷은 지리정보 통합 포인트 클라우드에 특화되어 있습니다. 대규모 야외 유적지는 LAS/LAZ를 병행 지원합니다.

메타데이터: Dublin Core + CIDOC-CRM

문화유산 메타데이터는 Dublin Core (15개 핵심 요소)와 CIDOC-CRM (개념 참조 모델)을 기반으로 합니다. WIA-HERITAGE-006은 JSON-LD 형식으로 인코딩된 메타데이터를 glTF 및 E57 파일에 임베드합니다.

03.4 품질 보증 프레임워크

WIA-HERITAGE-006은 단순한 포맷 표준을 넘어, 3D 스캔 데이터의 품질을 객관적으로 평가하고 인증하는 체계를 제공합니다.

Heritage Scan Quality Score (HSQS)

HSQS는 0-100점 척도로, 6가지 차원의 품질을 종합 평가합니다. 각 차원은 가중치를 가지며, 최종 점수는 가중 평균으로 계산됩니다.

품질 차원가중치측정 메트릭우수 기준
기하학적 정확도30%Ground truth 대비 RMS 오차<0.5mm (유물), <5mm (건축)
완성도20%캡처된 표면 비율>95%
텍스처 품질15%해상도, 색상 정확도2K+ 텍스처, ΔE < 3.0
메타데이터 풍부성15%필수 필드 22개 완성도100% 완성
포맷 준수성10%표준 스펙 위반 여부위반 0개
접근성10%파일 크기, LOD 존재<100MB (웹용), LOD 5단계

HSQS 70점 이상은 "공식 아카이브 등록 가능", 85점 이상은 "연구 등급", 95점 이상은 "박물관 전시 등급"으로 분류됩니다.

자동 검증 도구

WIA-HERITAGE-006 SDK는 다음 검증 도구를 포함합니다:

  • heritage-validator: CLI 도구로 파일 포맷, 메타데이터, 품질 메트릭 일괄 검증
  • web-validator: 브라우저 기반 드래그&드롭 검증 (파일 업로드 없이 클라이언트 측 처리)
  • API 엔드포인트: POST /validate로 원격 검증 및 상세 리포트 생성

03.5 라이선스 및 권리 관리

3D 스캔 데이터의 법적 지위는 복잡합니다. WIA-HERITAGE-006은 명확한 권리 관리 프레임워크를 제공합니다.

3단계 권리 모델

표준은 3가지 레벨의 권리를 구분합니다:

  1. 원본 유물 권리: 물리적 유물의 소유권 또는 문화적 권리
    • 박물관 소유권
    • 원주민 커뮤니티 문화적 권리
    • 국가 문화재 보호법상 권리
  2. 디지털화 권리: 3D 스캔 작업의 저작권
    • 스캔을 수행한 기관/개인의 창작적 노력
    • 단, 단순 기계적 복제는 저작권 주장 불가 (미국 Bridgeman 판례)
  3. 데이터베이스 권리: 3D 모델 컬렉션의 집합적 권리
    • EU 데이터베이스 지침에 따른 15년 보호
    • 큐레이션, 분류, 메타데이터 추가에 대한 투자 보호

권장 라이선스

WIA-HERITAGE-006은 다음 라이선스를 지원하며, 메타데이터에 명시하도록 요구합니다:

라이선스상업적 이용수정공유적용 사례
CC0 (퍼블릭 도메인)✅✅✅공공 박물관 소장품
CC BY 4.0✅✅✅ (저작자 표시)학술 연구 데이터
CC BY-NC 4.0❌✅✅상업 보호 필요 유물
CC BY-SA 4.0✅✅✅ (동일 조건)오픈 협업 프로젝트

디지털 복제권 가이드라인

WIA-HERITAGE-006은 3D 모델을 사용한 물리적 복제에 대한 명확한 가이드라인을 제공합니다:

  • 개인적 사용: 3D 프린팅 허용 (교육, 연구)
  • 박물관 전시: 레플리카 제작 허용 (원본 표시 필수)
  • 상업적 판매: 라이선스 명시적 허가 필요
  • 문화적 민감성: 원주민 커뮤니티 사전 동의 필수

03.6 인증 프로그램

WIA는 3단계 인증 프로그램을 운영하여 표준 준수를 장려합니다.

개인 인증

  • Certified Heritage Scanner: 3D 스캔 기술자 (12시간 온라인 교육 + 실기 시험)
  • Certified Heritage Data Curator: 메타데이터 전문가 (8시간 교육 + 프로젝트)
  • Certified Heritage Developer: API/WebXR 개발자 (16시간 교육 + 코딩 시험)

기관 인증

  • WIA Heritage Compliant: Phase 1 완전 구현 (연 $500, 감사)
  • WIA Heritage Advanced: Phase 1-3 구현 (연 $2,000, 분기 감사)
  • WIA Heritage Excellence: Phase 1-4 완전 구현 (연 $5,000, 월간 감사)

소프트웨어 인증

스캐닝 소프트웨어, 뷰어, 변환 도구는 WIA 호환성 인증을 받을 수 있습니다. 2024년 기준 38개 소프트웨어가 인증받았습니다:

  • Agisoft Metashape Professional (포토그래메트리)
  • Faro Scene (레이저 스캔)
  • CloudCompare (포인트 클라우드 처리)
  • Sketchfab (WebGL 뷰어)
  • Unity Heritage Toolkit (게임 엔진 통합)

채택 성공 사례

프랑스 루브르 박물관: 2023년 10월 WIA-HERITAGE-006을 공식 채택하여 소장품 50,000점을 3년 내 디지털화 목표. 첫 6개월간 12,300점 완료하여 목표를 82% 초과 달성.

한국 국립중앙박물관: 2024년 1월부터 Phase 1-3 구현. 경주 석굴암 본존불을 0.03mm 정밀도로 스캔하여 가상 복원 및 VR 체험관 구축. 일일 방문객 3,200명 기록.

이집트 문화재부: 2024년 3월 피라미드 내부 전체를 WIA 표준으로 스캔. 관광객 접근 제한 구역을 WebXR로 가상 투어 제공. 연간 관광 수입 $2.3M 증가.

03.8 장 요약

✅ 핵심 내용
  1. 4단계 아키텍처: Phase 1(데이터 형식)은 필수, Phase 2-4는 선택적이며 점진적 채택 가능. 78%가 Phase 1-2, 34%가 Phase 3까지 구현.
  2. 5대 설계 원칙: 개방성, 상호운용성, 확장성, 장기 보존, 접근성을 모든 기술 결정에 일관 적용.
  3. 기존 표준 통합: E57(포인트 클라우드), glTF 2.0(메시), LAS/LAZ(지리 데이터), Dublin Core/CIDOC-CRM(메타데이터)을 기반으로 확장.
  4. 품질 보증: Heritage Scan Quality Score(HSQS)로 0-100점 객관적 평가. 70점 이상 아카이브 등록, 85점 이상 연구 등급.
  5. 권리 관리: 원본 유물 권리, 디지털화 권리, 데이터베이스 권리를 3단계로 구분. CC 라이선스 및 WIA-Heritage-Restricted 지원.
  6. 인증 프로그램: 개인(기술자, 큐레이터, 개발자), 기관(Compliant, Advanced, Excellence), 소프트웨어(38개 인증) 3단계 인증.
  7. 글로벌 채택: 454개 기관, 98만 점 디지털화. 루브르, 국립중앙박물관, 이집트 문화재부 등 성공 사례.

03.9 복습 문제

  1. WIA-HERITAGE-006의 4단계 아키텍처를 설명하고, 각 단계의 필수/선택 여부와 구현 난이도를 비교하세요. 점진적 채택이 왜 중요한지 논하세요.
  2. 표준의 5대 설계 원칙(개방성, 상호운용성, 확장성, 장기 보존, 접근성)을 각각 설명하고, 구체적인 구현 사례를 제시하세요.
  3. E57과 glTF 2.0을 WIA-HERITAGE-006이 채택한 이유를 비교 분석하고, 각 포맷에 대한 문화유산 특화 확장을 설명하세요.
  4. Heritage Scan Quality Score(HSQS)의 6가지 품질 차원과 가중치를 나열하고, 70점, 85점, 95점 기준의 의미를 설명하세요.
  5. 3D 스캔 데이터의 3단계 권리 모델(원본 유물 권리, 디지털화 권리, 데이터베이스 권리)을 설명하고, Creative Commons 라이선스 적용 사례를 논하세요.
  6. 글로벌 채택 현황을 지역별로 분석하고, 루브르 박물관, 국립중앙박물관, 이집트 문화재부의 성공 사례에서 얻을 수 있는 교훈을 논하세요.
