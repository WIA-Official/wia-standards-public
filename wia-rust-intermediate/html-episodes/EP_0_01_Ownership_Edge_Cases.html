부분 이동 (Partial Moves) Partial Moves

❌ 잘못된 예제

❌ Wrong Example

struct Person {
name: String,
age: u32,
}
let person = Person {
name: "Alice".to_string(),
age: 30,
};
let name = person.name; // name moved!
println!("{:?}", person); // ❌ Error!

해결책: 참조 또는 Copy Solution: Reference or Copy

✅ 올바른 방법

✅ Correct Way

// Solution 1: Use reference
let name = &person.name;
println!("{:?}", person); // ✅ OK
// Solution 2: Clone
let name = person.name.clone();
println!("{:?}", person); // ✅ OK
// Solution 3: Access Copy fields freely
let age = person.age; // u32 is Copy
println!("{:?}", person); // ✅ OK

임시 값 (Temporary Values) Temporary Values

임시 값의 라이프타임

Lifetime of Temporary Values

// ❌ Temporary value dropped
let s = &String::from("hello").as_str();
// String dropped immediately!
// ✅ Correct: Bind to variable first
let temp = String::from("hello");
let s = temp.as_str();
// ✅ Or use lifetime extension
let s = String::from("hello");
let r = &s;

루프 안에서의 이동 Move in Loop

반복문에서 소유권 주의

Ownership in Loops

let mut vec = vec![1, 2, 3];
// ❌ Moves vec in first iteration
for _ in 0..3 {
let v = vec; // Error on 2nd iteration!
}
// ✅ Use reference
for _ in 0..3 {
let v = &vec; // OK!
}

내부 가변성 패턴 Interior Mutability Pattern

Cell과 RefCell

Cell and RefCell

use std::cell::{Cell, RefCell};
// Cell: Copy types only
let x = Cell::new(42);
x.set(100); // Mutate through shared ref!
// RefCell: Runtime borrow checking
let y = RefCell::new("hello".to_string());
*y.borrow_mut() = "world".to_string();

Drop 순서 Drop Order

값이 drop되는 순서

Order of Dropping Values

struct PrintDrop(&'static str);
impl Drop for PrintDrop {
fn drop(&mut self) {
println!("Dropping: {}", self.0);
}
}
let _a = PrintDrop("a");
let _b = PrintDrop("b");
// Drops in reverse: b, then a

핵심 정리

Key Takeaways

  • 부분 이동 후에는 전체 구조체 사용 불가
  • 임시 값의 라이프타임에 주의
  • 루프에서 소유권 이동 조심
  • 내부 가변성으로 소유권 규칙 우회
  • Drop 순서는 역순 (LIFO)
  • Can't use struct after partial move
  • Watch temporary value lifetimes
  • Be careful with moves in loops
  • Interior mutability bypasses ownership rules
  • Drop order is reverse (LIFO)
