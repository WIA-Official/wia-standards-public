왜 PhantomData? Why PhantomData?

❌ 사용하지 않는 타입 파라미터

❌ Unused Type Parameter

struct Marker<T> {
value: i32,
// ❌ T is not used! Compiler error!
}
// ✅ Solution: PhantomData
struct Marker<T> {
value: i32,
_marker: PhantomData<T>,
}

타입 상태 패턴 Type State Pattern

컴파일 시점 상태 검증

Compile-Time State Verification

struct Locked;
struct Unlocked;
struct Door<State> {
_state: PhantomData<State>,
}
impl Door<Locked> {
fn unlock(self) -> Door<Unlocked> { ... }
}
impl Door<Unlocked> {
fn open(&self) { ... }
}

Indicating Ownership

변성(Variance) 조정

Adjusting Variance

use std::marker::PhantomData;
// Covariant over T
PhantomData<T>
// Invariant over T
PhantomData<fn(T)>
// Contravariant over T
PhantomData<fn() -> T>

실전: Builder Pattern Real: Builder Pattern

타입 안전한 빌더

Type-Safe Builder

struct Unset;
struct Set;
struct Builder<Name, Age> {
name: Option<String>,
age: Option<u32>,
_name: PhantomData<Name>,
_age: PhantomData<Age>,
}
// Only build when both are Set!
impl Builder<Set, Set> {
fn build(self) -> Person { ... }
}

핵심 용도 Key Use Cases

  • ✅ 타입 상태 패턴: 컴파일 시점 상태 체크

  • ✅ Zero-Sized Type: No runtime cost
  • ✅ Safe API: Prevent incorrect usage
