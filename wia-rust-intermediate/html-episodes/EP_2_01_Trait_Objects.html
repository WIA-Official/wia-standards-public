Generics: 정적 디스패치Generics: Static Dispatch

컴파일 시점 단형화

Compile-Time Monomorphization

fn print_all<T: Display>(items: Vec<T>) {
for item in items {
println!("{}", item);
}
}
// Compiler generates separate code for each type
// ✅ Fast (no runtime overhead)
// ❌ Code bloat (larger binary)
// ❌ All items must be same type

dyn Trait: 동적 디스패치dyn Trait: Dynamic Dispatch

런타임 vtable 조회

Runtime vtable Lookup

fn print_all(items: Vec<Box<dyn Display>>) {
for item in items {
println!("{}", item);
}
}
// Runtime vtable lookup
// ✅ Different types in same Vec
// ✅ Smaller binary size
// ❌ Slight runtime overhead

제네릭 T

Generic T

  • 정적 디스패치
  • 최고 성능
  • 코드 크기 증가
  • Static dispatch
  • Best performance
  • Code bloat

dyn Trait

dyn Trait

  • 동적 디스패치
  • 이종 컬렉션
  • 약간 느림
  • Dynamic dispatch
  • Heterogeneous
  • Slightly slower

impl Trait

impl Trait

  • 정적 디스패치
  • 구현 숨기기
  • 단일 타입만
  • Static dispatch
  • Hide impl
  • Single type

Plugin System

trait Plugin {
fn execute(&self);
}
struct PluginManager {
plugins: Vec<Box<dyn Plugin>>,
}
impl PluginManager {
fn run_all(&self) {
for plugin in &self.plugins {
plugin.execute();
}
}

핵심 정리

Key Takeaways

  • 제네릭: 정적 디스패치, 최고 성능
  • dyn Trait: 동적 디스패치, 이종 컬렉션
  • impl Trait: 정적 디스패치, 반환 타입 숨기기
  • Trait Object는 Fat Pointer (data + vtable)
  • 성능 vs 유연성 트레이드오프
  • Generics: Static dispatch, best perf
  • dyn Trait: Dynamic dispatch, heterogeneous
  • impl Trait: Static dispatch, hide return type
  • Trait Object is Fat Pointer (data + vtable)
  • Performance vs flexibility tradeoff
