repr(Rust) - 기본 레이아웃 repr(Rust) - Default Layout

// 기본적으로 Rust는 필드 순서를 재배치할 수 있음
struct Point {
x: f64, // 8 bytes
y: u8, // 1 byte + 7 padding
z: f64, // 8 bytes
}
// Rust는 메모리를 최적화하기 위해
// 필드 순서를 x, z, y로 재배치할 수 있음
✨ 컴파일러가 최적화
✨ 필드 재배치 가능
✨ 메모리 효율적
⚠️ FFI에 사용 불가 ✨ Compiler optimizes
✨ Fields can be reordered
✨ Memory efficient
⚠️ Cannot use with FFI

repr(C) - C 호환 레이아웃 repr(C) - C-Compatible Layout

#[repr(C)]
struct CPoint {
x: f64,
y: u8,
z: f64,
}
// FFI로 C 함수 호출
extern "C" {
fn process_point(point: *const CPoint);
}
🔗 C/C++와 호환
📌 필드 순서 보장
🎯 FFI에 필수
💡 예측 가능한 레이아웃 🔗 C/C++ compatible
📌 Field order guaranteed
🎯 Required for FFI
💡 Predictable layout

repr(packed) - 패딩 제거 repr(packed) - Remove Padding

#[repr(packed)]
struct Packed {
a: u8, // 1 byte
b: u64, // 8 bytes (no padding!)
c: u8, // 1 byte
}
// 총 10 bytes (패딩 없음)
struct Normal {
a: u8, // 1 byte + 7 padding
b: u64, // 8 bytes
c: u8, // 1 byte + 7 padding
}
// 총 24 bytes
💾 메모리 절약
📦 네트워크 프로토콜
⚠️ 정렬 문제 주의
🐢 성능 저하 가능 💾 Save memory
📦 Network protocols
⚠️ Watch alignment
🐢 May slow down

repr(transparent) - 투명한 래퍼 repr(transparent) - Transparent Wrapper

#[repr(transparent)]
struct Wrapper(u64);
// Wrapper는 u64와 동일한 메모리 레이아웃
impl Wrapper {
fn new(value: u64) -> Self {
Self(value)
}
}
// FFI에서 안전하게 u64로 전달 가능
🎭 타입 안전성 추가
🔄 제로 코스트 변환
📦 단일 필드만 가능
🎯 Newtype 패턴에 최적 🎭 Add type safety
🔄 Zero-cost conversion
📦 Single field only
🎯 Perfect for Newtype
