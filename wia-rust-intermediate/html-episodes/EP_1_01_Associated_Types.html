제네릭 매개변수 방식 Generic Parameter Approach

다양한 구현이 가능

Multiple Implementations Possible

trait Container<T> {
fn get(&self) -> &T;
}
// Same type, multiple implementations!
impl Container<i32> for MyType { ... }
impl Container<String> for MyType { ... }
// Must specify which implementation
let x: &dyn Container<i32> = &my_value;

연관 타입 방식 Associated Type Approach

구현당 하나의 타입

One Type Per Implementation

trait Container {
type Item;
fn get(&self) -> &Self::Item;
}
// Only ONE implementation per type
impl Container for MyType {
type Item = i32;
...
}
// No need to specify Item type!
let x: &dyn Container<Item=i32> = &my_value;

언제 무엇을 사용할까? When to Use Which?

🔄 제네릭 매개변수

🔄 Generic Parameters

  • 여러 구현이 필요할 때
  • Add<i32>, Add<f64>
  • 타입이 외부에서 결정
  • Need multiple implementations
  • Add<i32>, Add<f64>
  • Type decided externally

🔗 연관 타입

🔗 Associated Types

  • 구현당 하나의 타입
  • Iterator::Item
  • 타입이 구현에서 결정
  • One type per implementation
  • Iterator::Item
  • Type decided in impl

연관 타입의 장점 Benefits of Associated Types

  • ✅ 명확한 의도: 구현당 하나의 타입만 가능
  • ✅ 깔끔한 시그니처: 타입 파라미터 불필요
  • ✅ 타입 추론 용이: 컴파일러가 Item 추론
  • ✅ 오류 감소: 잘못된 타입 조합 방지
  • ✅ 읽기 쉬운 코드: Iterator::Item처럼 의미 명확
  • ✅ Clear Intent: Only one type per impl
  • ✅ Clean Signatures: No type parameters needed
  • ✅ Easy Inference: Compiler infers Item
  • ✅ Fewer Errors: Prevents wrong type combos
  • ✅ Readable Code: Iterator::Item is clear

핵심 정리

Key Takeaways

  • 연관 타입은 구현당 하나의 타입만 허용
  • 제네릭은 여러 구현이 가능
  • Iterator, Graph 등에서 연관 타입 사용
  • Add<T> 같이 여러 타입 지원 시 제네릭
  • 연관 타입은 코드를 더 명확하고 깔끔하게
  • Associated types allow one type per impl
  • Generics allow multiple implementations
  • Use associated types for Iterator, Graph, etc
  • Use generics for Add<T> multi-type support
  • Associated types make code clearer and cleaner
