HRTB란? What is HRTB?

Higher-Ranked Trait Bounds

// Normal trait bound
fn foo<'a>(x: &impl Fn(&'a str)) {}
// HRTB: Works with ANY lifetime
fn bar(x: &impl for<'a> Fn(&'a str)) {}
// Example: Closure that works with any lifetime
let closure = |s: &str| println!("{}", s);
bar(&closure); // ✅ Works!

HRTB 실전 예제 HRTB Real Example

Fn 트레잇과 HRTB

Fn Trait with HRTB

// Filter function with HRTB
fn apply_to_strings<F>(
items: Vec<String>,
f: F
) where
F: for<'a> Fn(&'a str) -> bool
{
for item in items {
if f(&item) {
println!("{}", item);
}
}
}

라이프타임 생략 규칙 Lifetime Elision Rules

규칙 1

Rule 1

각 참조 매개변수는
자신만의 라이프타임을 갖는다

Each reference parameter
gets its own lifetime

규칙 2

Rule 2

입력 라이프타임이 하나면
그것이 출력 라이프타임

If one input lifetime,
it's the output lifetime

규칙 3

Rule 3

&self나 &mut self가 있으면
self의 라이프타임이 출력

If &self or &mut self,
self's lifetime is output

적용 불가

Can't Apply

위 규칙으로 판단 불가하면
명시적으로 작성 필요

If rules can't determine,
must write explicitly

컴파일러가 추론하는 방식

How Compiler Infers

// Written
fn first_word(s: &str) -> &str
// Compiler infers (Rule 1 + 2)
fn first_word<'a>(s: &'a str) -> &'a str
// Need explicit lifetime
fn longest<'a>(x: &'a str, y: &'a str) -> &'a str
// Rule 1 applies, but not 2 (two inputs)

여러 라이프타임 Multiple Lifetimes

독립적인 라이프타임

Independent Lifetimes

struct Context<'s> {
source: &'s str,
}
impl<'s> Context<'s> {
// 's: context lifetime
// 'a: query lifetime (independent!)
fn search<'a>(&self, query: &'a str) -> Option<&'s str> {
if self.source.contains(query) {
Some(self.source)
} else { None }
}
}

라이프타임 서브타이핑 Lifetime Subtyping

'a: 'b 의미

Meaning of 'a: 'b

// 'a lives at least as long as 'b
struct Parser<'a, 'b> where 'a: 'b {
source: &'a str,
current: &'b str,
}
// 'static: Any lifetime
fn get_static() -> &'static str {
"I live forever!"
}

핵심 정리

Key Takeaways

  • HRTB는 모든 라이프타임에 대해 동작하는 트레잇
  • 생략 규칙 3가지로 대부분 자동 추론
  • 여러 라이프타임은 독립적으로 사용 가능
  • 'a: 'b는 'a가 'b보다 오래 산다는 의미
  • 'static은 프로그램 전체 수명
  • HRTB: Traits that work with any lifetime
  • 3 elision rules auto-infer most cases
  • Multiple lifetimes can be independent
  • 'a: 'b means 'a lives at least as long as 'b
  • 'static lives for entire program
