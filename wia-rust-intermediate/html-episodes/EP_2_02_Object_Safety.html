Object Safe란?What is Object Safe?

dyn Trait 가능 여부

Can Use dyn Trait?

// Object Safe - Can use dyn Trait
trait Draw {
fn draw(&self);
}
let x: Box<dyn Draw>; // ✅ OK
// NOT Object Safe - Cannot use dyn
trait Clone {
fn clone(&self) -> Self;
}
let x: Box<dyn Clone>; // ❌ Error!

규칙 1: Self 반환 금지Rule 1: No Returning Self

반환 타입에 Self 불가

Cannot Return Self

// ❌ NOT Object Safe
trait Builder {
fn new() -> Self;
fn clone(&self) -> Self;
}
// Why? Don't know concrete type at runtime!
let x: Box<dyn Builder> = ...;
let y = x.clone(); // What type is y??

규칙 2: 제네릭 메서드 금지Rule 2: No Generic Methods

메서드는 제네릭 불가

Methods Cannot Be Generic

// ❌ NOT Object Safe
trait Container {
fn insert<T>(&mut self, item: T);
}
// Why? vtable can't hold infinite methods!
// insert<i32>, insert<String>, ...
// ✅ Solution: where Self: Sized
fn insert<T>(&mut self, item: T)
where Self: Sized { ... }

where Self: Sized 트릭where Self: Sized Trick

일부 메서드만 제외

Exclude Some Methods

trait Iterator {
type Item;
// Required - part of object safety
fn next(&mut self) -> Option<Self::Item>;
// Excluded from dyn Iterator
fn collect<B>(self) -> B
where Self: Sized { ... }
}
// Can use dyn Iterator, but not .collect()

실전 해결책Practical Solutions

Clone을 Object Safe하게

Make Clone Object Safe

// dyn-clone crate pattern
trait ClonableBox {
fn clone_box(&self) -> Box<dyn ClonableBox>;
}
impl<T> ClonableBox for T
where
T: Clone + 'static,
{
fn clone_box(&self) -> Box<dyn ClonableBox> {
Box::new(self.clone())
}
}

Key Takeaways

  • Object Safe = dyn Trait 사용 가능
  • Self 반환, 제네릭 메서드 금지
  • where Self: Sized로 메서드 제외
  • Iterator, Debug는 Object Safe
  • Clone, Eq는 Object Safe 아님
  • Object Safe = Can use dyn Trait
  • No returning Self, no generic methods
  • Exclude methods with Self: Sized
  • Iterator, Debug are object safe
  • Clone, Eq are not object safe
