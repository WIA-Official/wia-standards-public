Chapter 5: Emergency Response & Fail-Safe

5.1 The Critical Nature of Emergency Response

When prevention fails, emergency response systems become the last line of defense against harm. The WIA-AI-SAFETY-PHYSICAL standard mandates that all physical AI systems include robust emergency response mechanisms capable of bringing the system to a safe state within defined time limits, regardless of the nature of the emergency.

Critical Requirement: Emergency response systems must function correctly even when other system components have failed. This means emergency circuits must be independent, hardwired where possible, and never rely on software that could be compromised.

5.2 Stop Categories and Response Times

International standards define three categories of stops, each with specific characteristics and use cases:

Category Method Characteristics Use Case
Category 0 Immediate power removal Uncontrolled stop, potential mechanical shock True emergency, imminent danger
Category 1 Controlled stop then power removal Deceleration followed by power cut Standard emergency stop
Category 2 Controlled stop, power maintained Stop with holding torque Protective stop, resume possible

    safetyRatedStop: {

    };
}

// Maximum stopping distance calculation
function calculateStoppingDistance(
    velocity: number,     // m/s
    deceleration: number, // m/s^2
    reactionTime: number  // s
): number {
    const reactionDistance = velocity * reactionTime;
    const brakingDistance = (velocity * velocity) / (2 * deceleration);
    return reactionDistance + brakingDistance;
}

5.3 Emergency Stop System Design

Emergency stop systems require careful design to ensure reliability and accessibility:

E-Stop Button Requirements

  • Color: Red mushroom head on yellow background
  • Operation: Direct-operated (not software-dependent)
  • Latching: Self-latching, requires deliberate reset
  • Accessibility: Within reach from all operator positions
  • Quantity: Multiple buttons covering all access points
  • Wiring: Normally closed (NC) contacts, fail-safe
// Emergency Stop Circuit Design
class EmergencyStopCircuit {
    // Dual-channel safety relay architecture
    private channel1: SafetyRelay;
    private channel2: SafetyRelay;

    // E-stop buttons connected in series (all NC)
    private estopButtons: EStopButton[] = [];

    constructor(config: EStopConfig) {
        // Initialize dual redundant channels
        this.channel1 = new SafetyRelay('CH1', config.relay1);
        this.channel2 = new SafetyRelay('CH2', config.relay2);

        // Register all E-stop buttons
        for (const buttonConfig of config.buttons) {
            const button = new EStopButton(buttonConfig);
            this.estopButtons.push(button);
        }

        // Cross-monitoring between channels
        this.setupCrossMonitoring();
    }

    // E-stop activation sequence
    activateEmergencyStop(source: string): void {
        const activationTime = Date.now();

        // Log activation source
        this.logActivation(source, activationTime);

        // Channel 1: Immediate power cut to motors
        this.channel1.open();

        // Channel 2: Engage mechanical brakes
        this.channel2.triggerBrakes();

        // Verify stop achieved
        setTimeout(() => {
            this.verifyStop(activationTime);
        }, 500); // Verify within 500ms
    }

    // Reset sequence (requires deliberate action)
    reset(): ResetResult {
        // Check all E-stop buttons released
        for (const button of this.estopButtons) {
            if (button.isPressed()) {
                return {
                    success: false,
                    reason: `E-stop button ${button.id} still pressed`
                };
            }
        }

        // Check safe conditions
        if (!this.checkSafeConditions()) {
            return {
                success: false,
                reason: 'Unsafe conditions persist'
            };
        }

        // Require manual reset button
        if (!this.manualResetPressed) {
            return {
                success: false,
                reason: 'Manual reset required'
            };
        }

        // Reset safety relays
        this.channel1.close();
        this.channel2.close();

        return { success: true };
    }

    // Cross-monitoring to detect discrepancies
    setupCrossMonitoring(): void {
        setInterval(() => {
            const ch1State = this.channel1.getState();
            const ch2State = this.channel2.getState();

            if (ch1State !== ch2State) {
                // Channel discrepancy - fault condition
                this.triggerSafeState('Channel discrepancy detected');
            }
        }, 10); // 100Hz monitoring
    }
}

5.4 Fail-Safe Design Principles

Fail-safe design ensures that failures result in safe conditions rather than dangerous ones:

Component Fail-Safe Behavior Implementation
Power Supply Loss triggers brakes Spring-applied, power-release brakes
Communication Loss triggers stop Watchdog timer, heartbeat protocol
Sensors Failure assumes hazard Conservative default, redundancy
Controllers Hang triggers stop Hardware watchdog, dual processors
Actuators Failure holds position Mechanical position locks
Software Exception triggers stop Defensive programming, safe defaults 
// Fail-Safe State Machine
enum SafetyState {
    NORMAL = 'NORMAL',
    WARNING = 'WARNING',
    PROTECTIVE_STOP = 'PROTECTIVE_STOP',
    EMERGENCY_STOP = 'EMERGENCY_STOP',
    FAULT = 'FAULT',
    SAFE = 'SAFE'
}

class FailSafeStateMachine {
    private currentState: SafetyState = SafetyState.SAFE;
    private stateHistory: StateTransition[] = [];

    // Valid state transitions (fail-safe: can always go to safer states)
    private validTransitions: Map<SafetyState, SafetyState[]> = new Map([
        [SafetyState.NORMAL, [SafetyState.WARNING, SafetyState.PROTECTIVE_STOP,
                             SafetyState.EMERGENCY_STOP, SafetyState.FAULT]],
        [SafetyState.WARNING, [SafetyState.NORMAL, SafetyState.PROTECTIVE_STOP,
                              SafetyState.EMERGENCY_STOP, SafetyState.FAULT]],
        [SafetyState.PROTECTIVE_STOP, [SafetyState.NORMAL, SafetyState.WARNING,
                                       SafetyState.EMERGENCY_STOP, SafetyState.FAULT]],
        [SafetyState.EMERGENCY_STOP, [SafetyState.SAFE]], // Can only go to SAFE
        [SafetyState.FAULT, [SafetyState.SAFE]], // Can only go to SAFE
        [SafetyState.SAFE, [SafetyState.NORMAL]] // Requires deliberate restart
    ]);

    transitionTo(newState: SafetyState, reason: string): boolean {
        const validTargets = this.validTransitions.get(this.currentState);

        // Emergency transitions always allowed to safer states
        const safetyOrder = [SafetyState.NORMAL, SafetyState.WARNING,
                            SafetyState.PROTECTIVE_STOP, SafetyState.EMERGENCY_STOP,
                            SafetyState.FAULT, SafetyState.SAFE];

        const currentIndex = safetyOrder.indexOf(this.currentState);
        const newIndex = safetyOrder.indexOf(newState);

        // Always allow transition to safer state (higher index)
        if (newIndex > currentIndex || validTargets?.includes(newState)) {
            this.stateHistory.push({
                from: this.currentState,
                to: newState,
                reason,
                timestamp: Date.now()
            });
            this.currentState = newState;
            this.executeStateActions(newState);
            return true;
        }

        return false;
    }

    executeStateActions(state: SafetyState): void {
        switch (state) {
            case SafetyState.PROTECTIVE_STOP:
                this.executeProtectiveStop();
                break;
            case SafetyState.EMERGENCY_STOP:
                this.executeEmergencyStop();
                break;
            case SafetyState.FAULT:
                this.executeFaultResponse();
                break;
            case SafetyState.SAFE:
                this.verifySafeState();
                break;
        }
    }
}

5.5 Redundancy and Fault Tolerance

Critical safety systems require redundancy to tolerate single faults:

Redundancy Architectures

Architecture Description Fault Tolerance Use Case
1oo1 (Single) Single channel 0 Low-risk applications
1oo2 (Dual) Two channels, either triggers 1 Standard safety
2oo2 (Dual voting) Both must agree 0 (high availability) High availability
2oo3 (TMR) Majority voting 1 High reliability + availability
1oo2D Dual with diagnostics 1 (detected) Common in robotics 
// Triple Modular Redundancy Implementation
class TMRVoter<T> {
    private channels: Channel<T>[];
    private comparator: (a: T, b: T) => boolean;

    constructor(
        channel1: Channel<T>,
        channel2: Channel<T>,
        channel3: Channel<T>,
        comparator: (a: T, b: T) => boolean
    ) {
        this.channels = [channel1, channel2, channel3];
        this.comparator = comparator;
    }

    vote(): VoteResult<T> {
        const values = this.channels.map(ch => ch.getValue());

        // Check for 3-way agreement
        if (this.comparator(values[0], values[1]) &&
            this.comparator(values[1], values[2])) {
            return {
                value: values[0],
                confidence: 1.0,
                faultDetected: false
            };
        }

        // Check for 2-way agreement (one faulty channel)
        if (this.comparator(values[0], values[1])) {
            return {
                value: values[0],
                confidence: 0.67,
                faultDetected: true,
                faultyChannel: 2
            };
        }
        if (this.comparator(values[1], values[2])) {
            return {
                value: values[1],
                confidence: 0.67,
                faultDetected: true,
                faultyChannel: 0
            };
        }
        if (this.comparator(values[0], values[2])) {
            return {
                value: values[0],
                confidence: 0.67,
                faultDetected: true,
                faultyChannel: 1
            };
        }

        // No agreement - all channels different
        return {
            value: null,
            confidence: 0,
            faultDetected: true,
            faultyChannel: 'MULTIPLE'
        };
    }
}

// Dual-Channel Safety Controller
class DualChannelController {
    private primary: SafetyProcessor;
    private secondary: SafetyProcessor;

    async executeSafetyFunction(input: SafetyInput): Promise<SafetyOutput> {
        // Execute on both channels
        const [result1, result2] = await Promise.all([
            this.primary.execute(input),
            this.secondary.execute(input)
        ]);

        // Compare results
        if (!this.compareResults(result1, result2)) {
            // Discrepancy detected - safe state
            return {
                action: SafetyAction.STOP,
                reason: 'Channel discrepancy',
                diagnosticCode: 'DC_MISMATCH'
            };
        }

        return result1;
    }
}

5.6 Graceful Degradation

When non-critical components fail, the system should continue operating safely with reduced capability:

// Graceful Degradation System
class GracefulDegradation {
    private capabilities: Map<string, Capability> = new Map();
    private degradationLevels: DegradationLevel[] = [];

    // Register capability with fallback options
    registerCapability(
        name: string,
        implementation: () => void,
        fallbacks: Fallback[]
    ): void {
        this.capabilities.set(name, {
            name,
            primary: implementation,
            fallbacks,
            currentLevel: 0
        });
    }

    // Handle component failure
    handleFailure(componentId: string): DegradationResult {
        const affectedCapabilities = this.findAffectedCapabilities(componentId);

        for (const capability of affectedCapabilities) {
            const cap = this.capabilities.get(capability);
            if (!cap) continue;

            // Try next fallback level
            if (cap.currentLevel < cap.fallbacks.length) {
                cap.currentLevel++;
                const fallback = cap.fallbacks[cap.currentLevel - 1];

                // Apply degraded mode
                this.applyDegradedMode(cap.name, fallback);

                this.logDegradation(cap.name, fallback);
            } else {
                // No more fallbacks - disable capability
                this.disableCapability(cap.name);
            }
        }

        return this.generateDegradationReport();
    }

    // Example: Vision system degradation
    visionDegradationLevels: Fallback[] = [
        {
            name: 'Reduced Resolution',
            description: 'Process at half resolution',
            speedReduction: 0.8, // 80% speed
            implement: () => this.visionSystem.setResolution('HALF')
        },
        {
            name: 'Single Camera',
            description: 'Use only primary camera',
            speedReduction: 0.5,
            implement: () => this.visionSystem.useSingleCamera()
        },
        {
            name: 'Basic Detection Only',
            description: 'Disable pose estimation, basic detection only',
            speedReduction: 0.3,
            implement: () => this.visionSystem.setMode('BASIC')
        },
        {
            name: 'Proximity Sensors Only',
            description: 'Fall back to non-vision sensors',
            speedReduction: 0.2,
            implement: () => this.disableVisionCompletely()
        }
    ];
}

5.7 Recovery Procedures

Safe recovery from emergency and fault conditions requires careful sequencing:

// Recovery Procedure Manager
class RecoveryManager {
    async executeRecovery(fromState: SafetyState): Promise<RecoveryResult> {
        // Step 1: Verify current state
        const currentState = await this.verifyCurrentState();
        if (currentState !== fromState) {
            return { success: false, reason: 'State changed during recovery' };
        }

        // Step 2: Run pre-recovery diagnostics
        const diagnostics = await this.runDiagnostics();
        if (!diagnostics.allPassed) {
            return {
                success: false,
                reason: 'Diagnostics failed',
                failedTests: diagnostics.failed
            };
        }

        // Step 3: Clear fault conditions
        const faults = await this.getFaultConditions();
        for (const fault of faults) {
            const cleared = await this.attemptFaultClear(fault);
            if (!cleared) {
                return {
                    success: false,
                    reason: `Cannot clear fault: ${fault.description}`
                };
            }
        }

        // Step 4: Verify safe conditions for restart
        const safetyCheck = await this.verifySafeConditions();
        if (!safetyCheck.safe) {
            return {
                success: false,
                reason: 'Unsafe conditions exist',
                conditions: safetyCheck.unsafeConditions
            };
        }

        // Step 5: Execute controlled restart sequence
        const restartSequence = [
            () => this.initializeSafetyControllers(),
            () => this.enableSafetySensors(),
            () => this.verifySensorCoverage(),
            () => this.enableMotionController(),
            () => this.enableBrakes(),
            () => this.releaseBrakes(),
            () => this.verifyMotionCapability()
        ];

        for (let i = 0; i < restartSequence.length; i++) {
            try {
                await restartSequence[i]();
            } catch (error) {
                // Abort restart on any failure
                await this.abortRestart();
                return {
                    success: false,
                    reason: `Restart failed at step ${i}`,
                    error
                };
            }
        }

        // Step 6: Transition to normal operation
        await this.transitionToNormal();

        return { success: true, newState: SafetyState.NORMAL };
    }

    // Manual restart with operator confirmation
    async manualRestart(operatorId: string): Promise<RecoveryResult> {
        // Verify operator authorization
        if (!await this.verifyOperatorAuthorization(operatorId)) {
            return { success: false, reason: 'Unauthorized operator' };
        }

        // Require physical reset button
        console.log('Waiting for physical reset button...');
        const resetPressed = await this.waitForResetButton(30000); // 30s timeout

        if (!resetPressed) {
            return { success: false, reason: 'Reset button timeout' };
        }

        // Execute recovery
        return this.executeRecovery(SafetyState.EMERGENCY_STOP);
    }
}

5.8 Incident Recording and Analysis

Every safety event must be recorded for analysis and continuous improvement:

// Safety Event Recorder
class SafetyEventRecorder {
    private eventLog: SafetyEvent[] = [];
    private blackBox: CircularBuffer<SystemState>;

    constructor() {
        // Keep last 60 seconds of system state at 100Hz
        this.blackBox = new CircularBuffer(6000);
    }

    recordEvent(event: SafetyEvent): void {
        // Capture current system state
        const systemState = this.captureSystemState();

        // Add contextual data
        event.systemState = systemState;
        event.timestamp = Date.now();
        event.blackBoxSnapshot = this.blackBox.getSnapshot();

        // Store event
        this.eventLog.push(event);

        // Persist to non-volatile storage
        this.persistEvent(event);

        // Notify safety management system
        this.notifySafetyManagement(event);
    }

    captureSystemState(): SystemState {
        return {
            robotState: {
                jointPositions: this.robot.getJointPositions(),
                jointVelocities: this.robot.getJointVelocities(),
                jointTorques: this.robot.getJointTorques(),
                tcpPose: this.robot.getTCPPose(),
                tcpVelocity: this.robot.getTCPVelocity()
            },
            safetyState: {
                currentState: this.safetyStateMachine.getCurrentState(),
                activeSafetyFunctions: this.getActiveSafetyFunctions(),
                sensorHealth: this.getSensorHealth()
            },
            environmentState: {
                humanPositions: this.humanDetection.getPositions(),
                obstacles: this.obstacleDetection.getObstacles(),
                zoneOccupancy: this.getZoneOccupancy()
            },
            controllerState: {
                activeProgram: this.controller.getActiveProgram(),
                executionPoint: this.controller.getExecutionPoint(),
                commandedVelocity: this.controller.getCommandedVelocity()
            }
        };
    }

    // Generate incident report
    generateIncidentReport(eventId: string): IncidentReport {
        const event = this.eventLog.find(e => e.id === eventId);
        if (!event) throw new Error('Event not found');

        return {
            eventId,
            timestamp: event.timestamp,
            type: event.type,
            severity: event.severity,
            description: event.description,
            rootCause: this.analyzeRootCause(event),
            timeline: this.reconstructTimeline(event),
            systemState: event.systemState,
            recommendations: this.generateRecommendations(event)
        };
    }
}

Key Takeaways

  • Emergency stop systems must be independent, hardwired, and fail-safe
  • Three stop categories (0, 1, 2) provide different levels of response
  • Response times must be specified and verified for each safety function
  • Fail-safe design ensures failures result in safe conditions
  • Redundancy architectures (1oo2, 2oo3) provide fault tolerance
  • Graceful degradation maintains safe operation with reduced capability
  • Recovery procedures must include verification steps and operator confirmation
  • Incident recording enables analysis and continuous improvement
