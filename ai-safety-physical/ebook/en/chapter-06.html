Chapter 6: Hardware Safety Design

6.1 Inherent Safety Through Hardware

Hardware safety design aims to eliminate or reduce hazards through the physical design of robot systems. Unlike software-based safety measures that can fail due to bugs or malfunctions, hardware safety features provide intrinsic protection that cannot be bypassed by software failures. The WIA-AI-SAFETY-PHYSICAL standard emphasizes hardware safety as the foundation of the defense-in-depth strategy.

Design Philosophy: The safest robot is one that physically cannot cause harm, regardless of what its software commands. Hardware safety measures should make dangerous operation mechanically impossible.

6.2 Power and Force Limiting

Limiting the maximum power and force that a robot can exert is the most fundamental hardware safety measure:

Service Robot 150 W/kg payload 200-500W total Public environment
Industrial Robot No intrinsic limit 1-10kW Guarded workspace
Mobile Platform 50 W/kg 100-500W Pedestrian environment

6.3 Mechanical Stops and Limits

Mechanical stops provide hardware-enforced motion limits that cannot be exceeded regardless of software commands:

Stop Type Purpose Implementation Override
Hard Stops Absolute motion limit Physical blocks on joints Not possible
Soft Stops Normal operating range Software + sensor limits Special mode
Adjustable Stops Application-specific limits Mechanical adjusters + sensors Tool required
Virtual Walls Dynamic workspace limits Safety controller + sensing Authorization 
// Mechanical Limit Enforcement
class MechanicalLimits {
    // Joint angle limits with hardware enforcement
    jointLimits = {
        joint1: { softMin: -170, softMax: 170, hardMin: -175, hardMax: 175 },
        joint2: { softMin: -120, softMax: 120, hardMin: -125, hardMax: 125 },
        joint3: { softMin: -170, softMax: 170, hardMin: -175, hardMax: 175 },
        joint4: { softMin: -120, softMax: 120, hardMin: -125, hardMax: 125 },
        joint5: { softMin: -170, softMax: 170, hardMin: -175, hardMax: 175 },
        joint6: { softMin: -360, softMax: 360, hardMin: -365, hardMax: 365 }
    };

    // Velocity limits per joint
    velocityLimits = {
        joint1: { soft: 180, hard: 200 },  // deg/s
        joint2: { soft: 180, hard: 200 },
        joint3: { soft: 225, hard: 250 },
        joint4: { soft: 225, hard: 250 },
        joint5: { soft: 225, hard: 250 },
        joint6: { soft: 360, hard: 400 }
    };

    // Hardware limit switches
    limitSwitches = {
        type: 'INDUCTIVE_PROXIMITY',
        arrangement: 'DUAL_REDUNDANT',
        failureMode: 'NORMALLY_CLOSED',
        responseTime: 1  // ms
    };

    checkPosition(jointId: string, position: number): LimitStatus {
        const limits = this.jointLimits[jointId];

        if (position <= limits.hardMin || position >= limits.hardMax) {
            return {
                status: 'HARD_LIMIT',
                action: 'EMERGENCY_STOP',
                detail: `Joint ${jointId} at hard limit: ${position}`
            };
        }

        if (position <= limits.softMin || position >= limits.softMax) {
            return {
                status: 'SOFT_LIMIT',
                action: 'CONTROLLED_STOP',
                detail: `Joint ${jointId} at soft limit: ${position}`
            };
        }

        return { status: 'OK', action: 'CONTINUE' };
    }
}

6.4 Compliant and Backdrivable Actuators

Compliant actuators reduce impact forces by allowing controlled deflection:

Actuator Types for Safety

Type Compliance Backdrivability Best For
Series Elastic (SEA) High (spring element) High Cobots, impact absorption
Variable Stiffness (VSA) Adjustable Medium-High Adaptive tasks
Direct Drive Low High Fast, precise motion
Geared Motor Very Low Low High torque (requires guarding)
Cable/Tendon Medium Medium Remote actuation 
// Series Elastic Actuator Modeling
class SeriesElasticActuator {
    private springStiffness: number;  // N*m/rad
    private motorInertia: number;     // kg*m^2
    private linkInertia: number;      // kg*m^2

    constructor(config: SEAConfig) {
        this.springStiffness = config.springStiffness;
        this.motorInertia = config.motorInertia;
        this.linkInertia = config.linkInertia;
    }

    // Calculate impact force with human
    calculateImpactForce(
        approachVelocity: number,  // m/s
        humanStiffness: number,    // N/m (body region dependent)
        effectiveMass: number      // kg
    ): ImpactForce {
        // Combined stiffness (springs in series)
        const combinedStiffness = 1 / (
            1 / this.springStiffness +
            1 / humanStiffness
        );

        // Impact force using energy conservation
        // F_max = v * sqrt(k * m)
        const peakForce = approachVelocity * Math.sqrt(
            combinedStiffness * effectiveMass
        );

        // Duration of impact
        const impactDuration = Math.PI * Math.sqrt(
            effectiveMass / combinedStiffness
        );

        return {
            peakForce,
            impactDuration,
            impulse: peakForce * impactDuration / 2
        };
    }

    // Torque sensing through spring deflection
    measureJointTorque(): number {
        const motorPosition = this.motorEncoder.getPosition();
        const linkPosition = this.linkEncoder.getPosition();
        const springDeflection = motorPosition - linkPosition;

        return springDeflection * this.springStiffness;
    }

    // Backdrivability check
    isBackdrivable(externalTorque: number): boolean {
        const frictionTorque = this.staticFriction;
        return Math.abs(externalTorque) > frictionTorque;
    }
}

// Calculate padding requirements for impact protection
function calculatePaddingRequirement(
    maxVelocity: number,      // m/s
    effectiveMass: number,    // kg
    maxAllowedForce: number   // N
): number {
    // Kinetic energy to absorb
    const energy = 0.5 * effectiveMass * maxVelocity * maxVelocity;

    // Work done by padding: W = F * d
    // Solve for d (padding compression distance)
    const compressionDistance = energy / maxAllowedForce;

    // Padding thickness = 2x compression distance (50% compression max)
    return compressionDistance * 2 * 1000; // Convert to mm
}

6.6 Brake Systems

Reliable braking is essential for stopping and holding position during emergencies:

Brake Types and Characteristics

Brake Type Engagement Response Time Fail-Safe
Spring-Applied Electric-Release Power off = braked 10-50ms Yes
Electric-Applied Power on = braked 20-100ms No
Hydraulic Pressure applied 50-200ms Configurable
Dynamic Braking Motor shorting <10ms Partial 
// Brake System Design
class BrakeSystem {
    private brakes: JointBrake[];

    // Brake sizing calculation
    calculateBrakeRequirements(
        joint: JointConfig,
        maxLoadTorque: number,
        safetyFactor: number = 2.0
    ): BrakeRequirements {
        // Static holding torque
        const staticTorque = maxLoadTorque * safetyFactor;

        // Dynamic braking torque (for stopping)
        const angularMomentum = joint.inertia * joint.maxVelocity;
        const stoppingTime = 0.5; // Target 500ms stop
        const dynamicTorque = angularMomentum / stoppingTime;

        return {
            staticHoldingTorque: staticTorque,
            dynamicBrakingTorque: dynamicTorque,
            minTorque: Math.max(staticTorque, dynamicTorque),
            engagementTime: 50,  // ms max
            releaseTime: 100,    // ms max
            wearIndicator: true,
            temperatureRating: 120  // Celsius
        };
    }

    // Emergency brake sequence
    async engageEmergencyBrakes(): Promise<BrakeResult> {
        const startTime = Date.now();

        // Engage all brakes simultaneously
        await Promise.all(
            this.brakes.map(brake => brake.engage())
        );

        // Verify engagement
        const engagementTime = Date.now() - startTime;

        for (const brake of this.brakes) {
            if (!brake.isEngaged()) {
                return {
                    success: false,
                    error: `Brake ${brake.id} failed to engage`,
                    engagementTime
                };
            }
        }

        return { success: true, engagementTime };
    }

    // Brake wear monitoring
    monitorBrakeHealth(): BrakeHealthReport {
        const reports: JointBrakeHealth[] = [];

        for (const brake of this.brakes) {
            const health = {
                jointId: brake.jointId,
                wearPercentage: brake.getWearLevel(),
                engagementCycles: brake.getCycleCount(),
                lastTestResult: brake.getLastTestResult(),
                estimatedLifeRemaining: brake.estimateRemainingLife()
            };

            reports.push(health);

            // Alert if wear exceeds threshold
            if (health.wearPercentage > 80) {
                this.alertMaintenanceRequired(brake.jointId, 'Brake wear >80%');
            }
        }

        return { joints: reports, timestamp: Date.now() };
    }
}

6.7 Electrical Safety

Electrical safety protects against shock, fire, and interference hazards:

Hazard Protection Standard
Electric Shock Class I grounding, double insulation IEC 60204-1
Overcurrent Fuses, circuit breakers IEC 60204-1
Earth Fault RCD/GFCI, insulation monitoring IEC 60204-1
EMC Interference Shielding, filtering IEC 61000-6
Static Discharge Grounding, ESD protection IEC 61000-4-2
Battery Hazards BMS, thermal management IEC 62133 
// Electrical Safety System
class ElectricalSafety {
    // Ground fault detection
    monitorGroundFault(): GroundFaultStatus {
        const leakageCurrent = this.measureLeakageCurrent();
        const threshold = 30; // mA (personnel protection)

        if (leakageCurrent > threshold) {
            this.tripGroundFaultProtection();
            return {
                fault: true,
                current: leakageCurrent,
                action: 'CIRCUIT_TRIPPED'
            };
        }

        return { fault: false, current: leakageCurrent };
    }

    // Battery management safety
    batteryManagement: BatteryManagementSystem = {
        monitoring: {
            cellVoltageMin: 2.5,    // V
            cellVoltageMax: 4.2,    // V
            temperatureMin: 0,       // C
            temperatureMax: 45,      // C
            maxChargeCurrent: 1.0,   // C-rate
            maxDischargeCurrent: 2.0 // C-rate
        },

        protection: {
            overVoltageProtection: true,
            underVoltageProtection: true,
            overCurrentProtection: true,
            overTemperatureProtection: true,
            shortCircuitProtection: true,
            cellBalancing: true
        },

        thermal: {
            coolingMethod: 'ACTIVE_AIR',
            fanTriggerTemp: 35,      // C
            shutdownTemp: 50         // C
        }
    };

    // Isolation testing
    performIsolationTest(): IsolationTestResult {
        // Test insulation resistance
        const testVoltage = 500; // V DC
        const minResistance = 1; // MOhm

        const measuredResistance = this.measureInsulation(testVoltage);

        return {
            passed: measuredResistance >= minResistance,
            testVoltage,
            measuredResistance,
            minRequired: minResistance
        };
    }
}

Key Takeaways

  • Hardware safety provides intrinsic protection that cannot be bypassed by software
  • Power and force limiting through hardware constraints are fundamental safety measures
  • Mechanical stops provide absolute motion limits regardless of software commands
  • Compliant actuators (SEA, VSA) reduce impact forces through controlled deflection
  • Protective enclosures must prevent pinch points, burns, and entanglement
  • Spring-applied brakes provide fail-safe stopping on power loss
  • Electrical safety requires proper grounding, isolation, and protection circuits
  • Safety sensors need redundancy, fast response, and self-diagnostics
