Chapter 3: Safety Protocols & Standards

3.1 International Safety Standards Landscape

Physical AI systems must comply with a complex web of international safety standards. These standards provide the foundation for safety design, verification, and certification. The WIA-AI-SAFETY-PHYSICAL standard is designed to harmonize with and complement existing standards while addressing AI-specific requirements.

Standards Hierarchy

Type Scope Examples
Type A (Basic) Fundamental safety concepts ISO 12100 (Risk assessment principles)
Type B (Generic) Safety aspects or devices ISO 13849 (Safety-related control), IEC 62061
Type C (Product) Specific machine categories ISO 10218 (Industrial robots), ISO 3691 (AGVs)

3.2 ISO 10218: Industrial Robot Safety

ISO 10218 is the primary international standard for industrial robot safety, consisting of two parts:

ISO 10218-1: Robot Requirements

  • Emergency Stop: Category 0 or 1 stop with hardwired circuit
  • Protective Stop: Category 0, 1, or 2 stop for safeguard activation
  • Speed/Force Limiting: Hardware or safety-rated software limits
  • Pendant: Enabling device, E-stop, mode selection
  • Singularity Protection: Axis limit monitoring

ISO 10218-2: Integration Requirements

  • Risk Assessment: Comprehensive analysis per ISO 12100
  • Safeguarding: Guards, devices, safe design measures
  • Installation: Requirements for cell design
  • Verification: Testing and validation protocols
  • Documentation: User information requirements
// ISO 10218 Compliance Checker
class ISO10218Compliance {
    checkRobotCompliance(robot: RobotSystem): ComplianceReport {
        const checks: ComplianceCheck[] = [];

        // Emergency Stop Requirements
        checks.push(this.checkEmergencyStop(robot));
        checks.push(this.checkProtectiveStop(robot));
        checks.push(this.checkSpeedLimiting(robot));
        checks.push(this.checkForceLimiting(robot));
        checks.push(this.checkAxisLimits(robot));
        checks.push(this.checkPendantFunctions(robot));
        checks.push(this.checkSafetyRatedSoftware(robot));

        return {
            standard: 'ISO 10218-1:2021',
            passed: checks.every(c => c.status === 'PASS'),
            checks,
            recommendations: this.generateRecommendations(checks)
        };
    }

    checkEmergencyStop(robot: RobotSystem): ComplianceCheck {
        const requirements = {
            category: robot.emergencyStop.stopCategory,
            hardwired: robot.emergencyStop.isHardwired,
            responseTime: robot.emergencyStop.responseTimeMs,
            accessibility: robot.emergencyStop.locations.length >= 2
        };

        const passed =
            (requirements.category === 0 || requirements.category === 1) &&
            requirements.hardwired === true &&
            requirements.responseTime <= 500 &&
            requirements.accessibility === true;

        return {
            requirement: '5.5 Emergency Stop',
            status: passed ? 'PASS' : 'FAIL',
            details: requirements,
            findings: passed ? [] : ['Emergency stop does not meet requirements']
        };
    }
}

ISO/TS 15066 provides detailed guidance for collaborative robot (cobot) applications where humans and robots share workspace:

Four Collaborative Modes

Mode Description Key Requirements

Speed and Separation Monitoring Maintains safe distance from humans Real-time monitoring, protective separation

// Power and Force Limiting Implementation
class PowerForceLimiting {
    private readonly bodyRegionLimits: Map<string, ForceLimits> = new Map([
        ['skull', { maxPressure: 13, maxForce: 130 }],
        ['face', { maxPressure: 6.5, maxForce: 65 }],
        ['neck', { maxPressure: 14.5, maxForce: 145 }],
        ['chest', { maxPressure: 14, maxForce: 140 }],
        ['abdomen', { maxPressure: 11, maxForce: 110 }],
        ['hand', { maxPressure: 30, maxForce: 140 }],
        ['leg', { maxPressure: 13, maxForce: 130 }]
    ]);

    // Calculate maximum allowed velocity based on robot mass and target limits
    calculateSafeVelocity(
        effectiveMass: number,  // kg
        targetBodyRegion: string,
        contactArea: number     // cm2
    ): number {
        const limits = this.bodyRegionLimits.get(targetBodyRegion);
        if (!limits) throw new Error(`Unknown body region: ${targetBodyRegion}`);

        // Energy-based calculation
        // E = 0.5 * m * v^2
        // F = sqrt(2 * E * k) where k is contact stiffness
        // Solve for v given max F

        const contactStiffness = 75000; // N/m (typical soft tissue)
        const maxEnergy = (limits.maxForce ** 2) / (2 * contactStiffness);
        const maxVelocity = Math.sqrt((2 * maxEnergy) / effectiveMass);

        return maxVelocity; // m/s
    }

    // Real-time force monitoring
    checkContactForce(
        measuredForce: number,
        contactArea: number,
        bodyRegion: string
    ): SafetyStatus {
        const limits = this.bodyRegionLimits.get(bodyRegion) ||
                       { maxPressure: 6.5, maxForce: 65 }; // Default to face (most restrictive)

        const pressure = measuredForce / contactArea;

        if (measuredForce > limits.maxForce || pressure > limits.maxPressure) {
            return {
                safe: false,
                action: 'IMMEDIATE_STOP',
                reason: `Force ${measuredForce}N or pressure ${pressure}N/cm2 exceeds limit`
            };
        }

        if (measuredForce > limits.maxForce * 0.8) {
            return {
                safe: true,
                action: 'REDUCE_SPEED',
                reason: 'Approaching force limit'
            };
        }

        return { safe: true, action: 'CONTINUE', reason: 'Within safe limits' };
    }
}

3.4 IEC 61508: Functional Safety

IEC 61508 is the umbrella standard for functional safety of electrical/electronic/programmable electronic safety-related systems. It introduces the concept of Safety Integrity Levels (SIL):

SIL Probability of Dangerous Failure (per hour) Risk Reduction Factor Typical Applications
SIL 4 10^-9 to 10^-8 100,000 - 10,000 Nuclear, aviation
SIL 3 10^-8 to 10^-7 10,000 - 1,000 Chemical process, rail
SIL 2 10^-7 to 10^-6 1,000 - 100 General machinery, robots
SIL 1 10^-6 to 10^-5 100 - 10 Basic safety functions

SIL Determination Process

// SIL Determination Based on Risk Parameters
function determineSIL(riskParams: RiskParameters): SIL {
    const { consequence, frequency, probability, avoidance } = riskParams;

    // Risk graph method (simplified)
    const riskScore = consequence + frequency + probability - avoidance;

    if (riskScore >= 8) return 'SIL4';
    if (riskScore >= 6) return 'SIL3';
    if (riskScore >= 4) return 'SIL2';
    if (riskScore >= 2) return 'SIL1';
    return 'NONE'; // No safety function required
}

interface RiskParameters {
    consequence: number;  // 1-4 (minor injury to multiple fatalities)
    frequency: number;    // 1-4 (rare to continuous exposure)
    probability: number;  // 1-4 (improbable to inevitable)
    avoidance: number;   // 1-3 (impossible to possible)
}

// Hardware Fault Tolerance Requirements
const SIL_HFT_REQUIREMENTS = {
    SIL1: { minHFT: 0, safeFailureFraction: 0.60 },
    SIL2: { minHFT: 0, safeFailureFraction: 0.90 },
    SIL3: { minHFT: 1, safeFailureFraction: 0.99 },
    SIL4: { minHFT: 2, safeFailureFraction: 0.999 }
};

// Architectural Constraints for SIL
class SILArchitecture {
    verifySILCompliance(system: SafetySystem, targetSIL: SIL): boolean {
        const requirements = SIL_HFT_REQUIREMENTS[targetSIL];

        // Check Hardware Fault Tolerance
        if (system.hardwareFaultTolerance < requirements.minHFT) {
            return false;
        }

        // Check Safe Failure Fraction
        if (system.safeFailureFraction < requirements.safeFailureFraction) {
            return false;
        }

        // Check diagnostic coverage
        if (!this.verifyDiagnosticCoverage(system, targetSIL)) {
            return false;
        }

        return true;
    }
}

3.5 ISO 13849: Safety-Related Control Systems

ISO 13849 provides an alternative to IEC 61508 for machinery safety, using Performance Levels (PL) instead of SIL:

Performance Level PFHd Range Approximate SIL Equivalent
PL e < 10^-7 SIL 3
PL d 10^-7 to 10^-6 SIL 2
PL c 10^-6 to 3×10^-6 SIL 1
PL b 3×10^-6 to 10^-5 SIL 1
PL a > 10^-5 None

Category Architectures

const CATEGORY_ARCHITECTURES = {
    'B': {
        description: 'Basic safety principles',
        redundancy: false,
        diagnostics: false,

    '1': {

    },
    '2': {
        description: 'Tested safety function',
        redundancy: false,
        diagnostics: true,
        faultTolerance: 0,
        maxPL: 'd'
    },
    '3': {
        description: 'Dual channel',
        redundancy: true,
        diagnostics: true,
        faultTolerance: 1,
        maxPL: 'e'
    },
    '4': {
        description: 'Dual channel with cross-monitoring',
        redundancy: true,
        diagnostics: true,
        faultTolerance: 2,
        maxPL: 'e'
    }
};

// Calculate achievable PL based on architecture
function calculatePL(
    category: string,
    mttfd: number,  // Mean Time To Dangerous Failure (years)
    dcavg: number,  // Average Diagnostic Coverage (0-1)
    ccf: number     // Common Cause Failure score (0-100)
): PerformanceLevel {
    const arch = CATEGORY_ARCHITECTURES[category];

    // Simplified PL calculation
    let plScore = 0;

    // MTTFD contribution
    if (mttfd >= 100) plScore += 3;
    else if (mttfd >= 30) plScore += 2;
    else if (mttfd >= 10) plScore += 1;

    // DC contribution
    if (dcavg >= 0.99) plScore += 2;
    else if (dcavg >= 0.90) plScore += 1;

    // CCF contribution
    if (ccf >= 65) plScore += 1;

    // Category contribution
    if (category === '4') plScore += 2;
    else if (category === '3') plScore += 1;

    // Map to PL
    if (plScore >= 8) return 'e';
    if (plScore >= 6) return 'd';
    if (plScore >= 4) return 'c';
    if (plScore >= 2) return 'b';
    return 'a';
}

3.6 Safety Protocol Implementation

Implementing safety protocols requires careful attention to both hardware and software aspects:

Safety-Rated Software Requirements

// Safety-Rated Software Implementation Guidelines
class SafetyRatedSoftware {
    // Defensive programming techniques
    validateInput(input: any, schema: ValidationSchema): ValidatedInput {
        // Range checking
        if (typeof input === 'number') {
            if (input < schema.min || input > schema.max) {
                throw new SafetyViolation(`Input ${input} out of range [${schema.min}, ${schema.max}]`);
            }
        }

        // Type checking
        if (typeof input !== schema.expectedType) {
            throw new SafetyViolation(`Invalid type: expected ${schema.expectedType}`);
        }

        // Plausibility checking
        if (!this.isPlausible(input, schema)) {
            throw new SafetyViolation('Input fails plausibility check');
        }

        return { value: input, validated: true };
    }

    // Redundant calculation with comparison
    safetyCalculation(input: number): SafetyResult {
        // Primary calculation
        const result1 = this.primaryAlgorithm(input);

        // Diverse secondary calculation
        const result2 = this.diverseSecondaryAlgorithm(input);

        // Compare results
        const tolerance = 0.001; // 0.1% tolerance
        if (Math.abs(result1 - result2) / result1 > tolerance) {
            this.triggerSafeState('Calculation discrepancy detected');
            return { value: null, error: 'CALCULATION_MISMATCH' };
        }

        return { value: result1, error: null };
    }

    // Watchdog monitoring
    startWatchdog(timeoutMs: number): void {
        setInterval(() => {
            if (Date.now() - this.lastHeartbeat > timeoutMs) {
                this.triggerSafeState('Watchdog timeout');
            }
        }, timeoutMs / 4);
    }

    // Program flow monitoring
    monitorProgramFlow(checkpoint: string): void {
        const expectedSequence = this.programFlowMonitor.getExpectedNext();

        if (checkpoint !== expectedSequence) {
            this.triggerSafeState(`Program flow violation: expected ${expectedSequence}, got ${checkpoint}`);
        }

        this.programFlowMonitor.recordCheckpoint(checkpoint);
    }
}

3.7 Safety Communication Protocols

Safety-rated communication requires special protocols that detect and handle errors:

Protocol Standard Safety Level Features
PROFIsafe IEC 61784-3 SIL 3 CRC, sequence numbers, time monitoring
CIP Safety IEC 61784-3 SIL 3 Safety data with EtherNet/IP
FSoE IEC 61784-3 SIL 3 Fail Safe over EtherCAT
openSAFETY IEC 61784-3 SIL 3 Transport-independent, open source 
// Safety Communication Protocol Implementation
class SafetyProtocol {
    private sequenceNumber: number = 0;
    private readonly crcPolynomial: number = 0x04C11DB7;

    createSafetyMessage(payload: SafetyPayload): SafetyMessage {
        const message: SafetyMessage = {
            header: {
                sequenceNumber: this.sequenceNumber++,
                timestamp: Date.now(),
                sourceId: this.nodeId,
                messageType: payload.type
            },
            payload: payload.data,
            safetyLayer: {
                crc1: this.calculateCRC(payload.data),
                crc2: this.calculateCRC(payload.data, true), // Diverse CRC
                virtualAddress: this.virtualAddress,
                watchdogCounter: this.watchdogCounter
            }
        };

        return message;
    }

    validateSafetyMessage(message: SafetyMessage): ValidationResult {
        // Check sequence number
        if (message.header.sequenceNumber !== this.expectedSequence) {
            if (message.header.sequenceNumber < this.expectedSequence) {
                return { valid: false, error: 'REPEATED_MESSAGE' };
            } else {
                return { valid: false, error: 'LOST_MESSAGE' };
            }
        }

        // Check timestamp (message age)
        const age = Date.now() - message.header.timestamp;
        if (age > this.maxMessageAge) {
            return { valid: false, error: 'MESSAGE_TIMEOUT' };
        }

        // Verify CRCs
        const crc1Valid = this.verifyCRC(message.payload, message.safetyLayer.crc1);
        const crc2Valid = this.verifyCRC(message.payload, message.safetyLayer.crc2, true);

        if (!crc1Valid || !crc2Valid) {
            return { valid: false, error: 'CRC_ERROR' };
        }

        this.expectedSequence++;
        return { valid: true, error: null };
    }
}

3.8 Safety Function Implementation

Safety functions must be implemented according to their required SIL/PL:

// Safety Function Implementation
class SafetyFunctionManager {
    private safetyFunctions: Map<string, SafetyFunction> = new Map();

    registerSafetyFunction(config: SafetyFunctionConfig): void {
        const safetyFunction: SafetyFunction = {
            id: config.id,
            name: config.name,
            requiredSIL: config.requiredSIL,
            requiredPL: config.requiredPL,

            // Execution with safety envelope
            execute: async () => {
                // Pre-execution checks
                if (!this.performPreExecutionChecks(config.id)) {
                    return this.safeState(config.id);
                }

                // Execute with timeout
                const result = await Promise.race([
                    config.implementation(),
                    this.timeout(config.maxExecutionTime)
                ]);

                // Post-execution verification
                if (!this.verifyPostConditions(config.id, result)) {
                    return this.safeState(config.id);
                }

                return result;
            },

            // Test interface
            test: () => this.testSafetyFunction(config.id)
        };

        this.safetyFunctions.set(config.id, safetyFunction);
    }

    // Emergency Stop Safety Function
    emergencyStop: SafetyFunctionConfig = {
        id: 'SF-001',
        name: 'Emergency Stop',
        requiredSIL: 'SIL3',
        requiredPL: 'PLe',
        maxExecutionTime: 100, // 100ms max response
        implementation: async () => {
            // Dual-channel implementation
            const channel1 = this.activateEmergencyStopChannel1();
            const channel2 = this.activateEmergencyStopChannel2();

            await Promise.all([channel1, channel2]);

            // Verify stop achieved
            const velocity = await this.measureVelocity();
            if (velocity > 0.001) {
                throw new Error('Emergency stop verification failed');
            }

            return { success: true, stoppedAt: Date.now() };
        }
    };
}

Key Takeaways

  • International standards provide the foundation for physical AI safety compliance
  • ISO 10218 defines requirements for industrial robot safety including emergency stop and speed limiting
  • ISO/TS 15066 specifies four collaborative modes and biomechanical force limits
  • IEC 61508 introduces Safety Integrity Levels (SIL 1-4) for functional safety
  • ISO 13849 uses Performance Levels (PL a-e) and Category architectures
  • Safety-rated software requires defensive programming, redundancy, and monitoring
  • Safety communication protocols provide error detection and handling
  • Comprehensive documentation is mandatory for certification
