3.1 국제 안전 표준 환경

물리적 AI 시스템은 복잡한 국제 안전 표준 체계를 준수해야 합니다. 이러한 표준은 안전 설계, 검증 및 인증의 기초를 제공합니다. WIA-AI-SAFETY-PHYSICAL 표준은 기존 표준과 조화를 이루고 보완하면서 AI 특정 요구사항을 다루도록 설계되었습니다.

표준 계층 구조

유형 범위 예시
A형 (기본) 기본적인 안전 개념 ISO 12100 (위험 평가 원칙)
B형 (일반) 안전 측면 또는 장치 ISO 13849 (안전 관련 제어), IEC 62061
C형 (제품) 특정 기계 카테고리 ISO 10218 (산업용 로봇), ISO 3691 (AGV)

3.2 ISO 10218: 산업용 로봇 안전

ISO 10218은 산업용 로봇 안전을 위한 주요 국제 표준으로, 두 부분으로 구성됩니다:

ISO 10218-1: 로봇 요구사항

  • 비상 정지: 하드와이어 회로가 있는 카테고리 0 또는 1 정지
  • 보호 정지: 안전 장치 활성화를 위한 카테고리 0, 1 또는 2 정지
  • 속도/힘 제한: 하드웨어 또는 안전 등급 소프트웨어 제한
  • 펜던트: 활성화 장치, 비상 정지, 모드 선택
  • 특이점 보호: 축 제한 모니터링

ISO 10218-2: 통합 요구사항

  • 위험 평가: ISO 12100에 따른 포괄적 분석
  • 안전 장치: 가드, 장치, 안전 설계 조치
  • 설치: 셀 설계 요구사항
  • 검증: 테스트 및 검증 프로토콜
  • 문서화: 사용자 정보 요구사항
// ISO 10218 준수 검사기
class ISO10218Compliance {
    checkRobotCompliance(robot: RobotSystem): ComplianceReport {
        const checks: ComplianceCheck[] = [];

        // 비상 정지 요구사항
        checks.push(this.checkEmergencyStop(robot));
        checks.push(this.checkProtectiveStop(robot));
        checks.push(this.checkSpeedLimiting(robot));
        checks.push(this.checkForceLimiting(robot));
        checks.push(this.checkAxisLimits(robot));
        checks.push(this.checkPendantFunctions(robot));
        checks.push(this.checkSafetyRatedSoftware(robot));

        return {
            standard: 'ISO 10218-1:2021',
            passed: checks.every(c => c.status === 'PASS'),
            checks,
            recommendations: this.generateRecommendations(checks)
        };
    }

    checkEmergencyStop(robot: RobotSystem): ComplianceCheck {
        const requirements = {
            category: robot.emergencyStop.stopCategory,
            hardwired: robot.emergencyStop.isHardwired,
            responseTime: robot.emergencyStop.responseTimeMs,
            accessibility: robot.emergencyStop.locations.length >= 2
        };

        const passed =
            (requirements.category === 0 || requirements.category === 1) &&
            requirements.hardwired === true &&
            requirements.responseTime <= 500 &&
            requirements.accessibility === true;

        return {
            requirement: '5.5 비상 정지',
            status: passed ? 'PASS' : 'FAIL',
            details: requirements,
            findings: passed ? [] : ['비상 정지가 요구사항을 충족하지 않음']
        };
    }
}

// 전력 및 힘 제한 구현
class PowerForceLimiting {
    private readonly bodyRegionLimits: Map<string, ForceLimits> = new Map([
        ['skull', { maxPressure: 13, maxForce: 130 }],
        ['face', { maxPressure: 6.5, maxForce: 65 }],
        ['neck', { maxPressure: 14.5, maxForce: 145 }],
        ['chest', { maxPressure: 14, maxForce: 140 }],
        ['abdomen', { maxPressure: 11, maxForce: 110 }],
        ['hand', { maxPressure: 30, maxForce: 140 }],
        ['leg', { maxPressure: 13, maxForce: 130 }]
    ]);

    // 로봇 질량과 목표 한계를 기반으로 최대 허용 속도 계산
    calculateSafeVelocity(
        effectiveMass: number,  // kg
        targetBodyRegion: string,
        contactArea: number     // cm2
    ): number {
        const limits = this.bodyRegionLimits.get(targetBodyRegion);
        if (!limits) throw new Error(`알 수 없는 신체 부위: ${targetBodyRegion}`);

        // 에너지 기반 계산
        // E = 0.5 * m * v^2
        // F = sqrt(2 * E * k) 여기서 k는 접촉 강성
        // 최대 F가 주어졌을 때 v 계산

        const contactStiffness = 75000; // N/m (일반적인 연조직)
        const maxEnergy = (limits.maxForce ** 2) / (2 * contactStiffness);
        const maxVelocity = Math.sqrt((2 * maxEnergy) / effectiveMass);

        return maxVelocity; // m/s
    }

    // 실시간 힘 모니터링
    checkContactForce(
        measuredForce: number,
        contactArea: number,
        bodyRegion: string
    ): SafetyStatus {
        const limits = this.bodyRegionLimits.get(bodyRegion) ||
                       { maxPressure: 6.5, maxForce: 65 }; // 얼굴 기본값 (가장 제한적)

        const pressure = measuredForce / contactArea;

        if (measuredForce > limits.maxForce || pressure > limits.maxPressure) {
            return {
                safe: false,
                action: 'IMMEDIATE_STOP',
                reason: `힘 ${measuredForce}N 또는 압력 ${pressure}N/cm2이 한계 초과`
            };
        }

        if (measuredForce > limits.maxForce * 0.8) {
            return {
                safe: true,
                action: 'REDUCE_SPEED',
                reason: '힘 한계에 근접'
            };
        }

        return { safe: true, action: 'CONTINUE', reason: '안전 한계 내' };
    }
}

3.4 IEC 61508: 기능 안전

IEC 61508은 전기/전자/프로그래머블 전자 안전 관련 시스템의 기능 안전을 위한 상위 표준입니다. 안전 무결성 수준(SIL) 개념을 도입합니다:

SIL 위험 고장 확률 (시간당) 위험 감소 계수 일반적 적용
SIL 4 10^-9 ~ 10^-8 100,000 - 10,000 원자력, 항공
SIL 3 10^-8 ~ 10^-7 10,000 - 1,000 화학 공정, 철도
SIL 2 10^-7 ~ 10^-6 1,000 - 100 일반 기계, 로봇
SIL 1 10^-6 ~ 10^-5 100 - 10 기본 안전 기능

SIL 결정 프로세스

// 위험 매개변수에 기반한 SIL 결정
function determineSIL(riskParams: RiskParameters): SIL {
    const { consequence, frequency, probability, avoidance } = riskParams;

    // 위험 그래프 방법 (단순화)
    const riskScore = consequence + frequency + probability - avoidance;

    if (riskScore >= 8) return 'SIL4';
    if (riskScore >= 6) return 'SIL3';
    if (riskScore >= 4) return 'SIL2';
    if (riskScore >= 2) return 'SIL1';
    return 'NONE'; // 안전 기능 필요 없음
}

interface RiskParameters {
    consequence: number;  // 1-4 (경미한 부상에서 다수 사망까지)
    frequency: number;    // 1-4 (희박에서 지속적 노출까지)
    probability: number;  // 1-4 (불가능에서 불가피까지)
    avoidance: number;   // 1-3 (불가능에서 가능까지)
}

// 하드웨어 고장 허용 요구사항
const SIL_HFT_REQUIREMENTS = {
    SIL1: { minHFT: 0, safeFailureFraction: 0.60 },
    SIL2: { minHFT: 0, safeFailureFraction: 0.90 },
    SIL3: { minHFT: 1, safeFailureFraction: 0.99 },
    SIL4: { minHFT: 2, safeFailureFraction: 0.999 }
};

// SIL을 위한 아키텍처 제약
class SILArchitecture {
    verifySILCompliance(system: SafetySystem, targetSIL: SIL): boolean {
        const requirements = SIL_HFT_REQUIREMENTS[targetSIL];

        // 하드웨어 고장 허용 검사
        if (system.hardwareFaultTolerance < requirements.minHFT) {
            return false;
        }

        // 안전 고장 비율 검사
        if (system.safeFailureFraction < requirements.safeFailureFraction) {
            return false;
        }

        // 진단 커버리지 검사
        if (!this.verifyDiagnosticCoverage(system, targetSIL)) {
            return false;
        }

        return true;
    }
}

3.5 ISO 13849: 안전 관련 제어 시스템

ISO 13849는 기계 안전을 위해 IEC 61508의 대안을 제공하며, SIL 대신 성능 수준(PL)을 사용합니다:

성능 수준 PFHd 범위 대략적인 SIL 등가
PL e < 10^-7 SIL 3
PL d 10^-7 ~ 10^-6 SIL 2
PL c 10^-6 ~ 3x10^-6 SIL 1
PL b 3x10^-6 ~ 10^-5 SIL 1
PL a > 10^-5 없음

카테고리 아키텍처

// ISO 13849 카테고리 아키텍처
const CATEGORY_ARCHITECTURES = {
    'B': {
        description: '기본 안전 원칙',
        redundancy: false,
        diagnostics: false,
        faultTolerance: 0,
        maxPL: 'b'
    },
    '1': {
        description: '검증된 구성 요소',
        redundancy: false,
        diagnostics: false,
        faultTolerance: 0,
        maxPL: 'c'
    },
    '2': {
        description: '테스트된 안전 기능',
        redundancy: false,
        diagnostics: true,
        faultTolerance: 0,
        maxPL: 'd'
    },
    '3': {
        description: '이중 채널',
        redundancy: true,
        diagnostics: true,
        faultTolerance: 1,
        maxPL: 'e'
    },
    '4': {
        description: '교차 모니터링이 있는 이중 채널',
        redundancy: true,
        diagnostics: true,
        faultTolerance: 2,
        maxPL: 'e'
    }
};

// 아키텍처에 기반한 달성 가능 PL 계산
function calculatePL(
    category: string,
    mttfd: number,  // 위험 고장까지의 평균 시간 (년)
    dcavg: number,  // 평균 진단 커버리지 (0-1)
    ccf: number     // 공통 원인 고장 점수 (0-100)
): PerformanceLevel {
    const arch = CATEGORY_ARCHITECTURES[category];

    // 단순화된 PL 계산
    let plScore = 0;

    // MTTFD 기여
    if (mttfd >= 100) plScore += 3;
    else if (mttfd >= 30) plScore += 2;
    else if (mttfd >= 10) plScore += 1;

    // DC 기여
    if (dcavg >= 0.99) plScore += 2;
    else if (dcavg >= 0.90) plScore += 1;

    // CCF 기여
    if (ccf >= 65) plScore += 1;

    // 카테고리 기여
    if (category === '4') plScore += 2;
    else if (category === '3') plScore += 1;

    // PL로 매핑
    if (plScore >= 8) return 'e';
    if (plScore >= 6) return 'd';
    if (plScore >= 4) return 'c';
    if (plScore >= 2) return 'b';
    return 'a';
}

3.6 안전 프로토콜 구현

안전 프로토콜 구현은 하드웨어 및 소프트웨어 측면 모두에 세심한 주의가 필요합니다:

안전 등급 소프트웨어 요구사항

// 안전 등급 소프트웨어 구현 지침
class SafetyRatedSoftware {
    // 방어적 프로그래밍 기법
    validateInput(input: any, schema: ValidationSchema): ValidatedInput {
        // 범위 검사
        if (typeof input === 'number') {
            if (input < schema.min || input > schema.max) {
                throw new SafetyViolation(`입력 ${input}이 범위 [${schema.min}, ${schema.max}]를 벗어남`);
            }
        }

        // 타입 검사
        if (typeof input !== schema.expectedType) {
            throw new SafetyViolation(`잘못된 타입: 예상 ${schema.expectedType}`);
        }

        // 타당성 검사
        if (!this.isPlausible(input, schema)) {
            throw new SafetyViolation('입력이 타당성 검사 실패');
        }

        return { value: input, validated: true };
    }

    // 비교를 포함한 중복 계산
    safetyCalculation(input: number): SafetyResult {
        // 기본 계산
        const result1 = this.primaryAlgorithm(input);

        // 다양한 보조 계산
        const result2 = this.diverseSecondaryAlgorithm(input);

        // 결과 비교
        const tolerance = 0.001; // 0.1% 허용 오차
        if (Math.abs(result1 - result2) / result1 > tolerance) {
            this.triggerSafeState('계산 불일치 감지');
            return { value: null, error: 'CALCULATION_MISMATCH' };
        }

        return { value: result1, error: null };
    }

    // 워치독 모니터링
    startWatchdog(timeoutMs: number): void {
        setInterval(() => {
            if (Date.now() - this.lastHeartbeat > timeoutMs) {
                this.triggerSafeState('워치독 타임아웃');
            }
        }, timeoutMs / 4);
    }

    // 프로그램 흐름 모니터링
    monitorProgramFlow(checkpoint: string): void {
        const expectedSequence = this.programFlowMonitor.getExpectedNext();

        if (checkpoint !== expectedSequence) {
            this.triggerSafeState(`프로그램 흐름 위반: 예상 ${expectedSequence}, 실제 ${checkpoint}`);
        }

        this.programFlowMonitor.recordCheckpoint(checkpoint);
    }
}

안전 등급 통신은 오류를 감지하고 처리하는 특수 프로토콜이 필요합니다:

// 안전 통신 프로토콜 구현
class SafetyProtocol {
    private sequenceNumber: number = 0;
    private readonly crcPolynomial: number = 0x04C11DB7;

    createSafetyMessage(payload: SafetyPayload): SafetyMessage {
        const message: SafetyMessage = {
            header: {
                sequenceNumber: this.sequenceNumber++,
                timestamp: Date.now(),
                sourceId: this.nodeId,
                messageType: payload.type
            },
            payload: payload.data,
            safetyLayer: {
                crc1: this.calculateCRC(payload.data),
                crc2: this.calculateCRC(payload.data, true), // 다양한 CRC
                virtualAddress: this.virtualAddress,
                watchdogCounter: this.watchdogCounter
            }
        };

        return message;
    }

    validateSafetyMessage(message: SafetyMessage): ValidationResult {
        // 시퀀스 번호 확인
        if (message.header.sequenceNumber !== this.expectedSequence) {
            if (message.header.sequenceNumber < this.expectedSequence) {
                return { valid: false, error: 'REPEATED_MESSAGE' };
            } else {
                return { valid: false, error: 'LOST_MESSAGE' };
            }
        }

        // 타임스탬프 확인 (메시지 수명)
        const age = Date.now() - message.header.timestamp;
        if (age > this.maxMessageAge) {
            return { valid: false, error: 'MESSAGE_TIMEOUT' };
        }

        // CRC 검증
        const crc1Valid = this.verifyCRC(message.payload, message.safetyLayer.crc1);
        const crc2Valid = this.verifyCRC(message.payload, message.safetyLayer.crc2, true);

        if (!crc1Valid || !crc2Valid) {
            return { valid: false, error: 'CRC_ERROR' };
        }

        this.expectedSequence++;
        return { valid: true, error: null };
    }
}

3.8 안전 기능 구현

안전 기능은 요구되는 SIL/PL에 따라 구현되어야 합니다:

// 안전 기능 구현
class SafetyFunctionManager {
    private safetyFunctions: Map<string, SafetyFunction> = new Map();

    registerSafetyFunction(config: SafetyFunctionConfig): void {
        const safetyFunction: SafetyFunction = {
            id: config.id,
            name: config.name,
            requiredSIL: config.requiredSIL,
            requiredPL: config.requiredPL,

            // 안전 범위와 함께 실행
            execute: async () => {
                // 실행 전 검사
                if (!this.performPreExecutionChecks(config.id)) {
                    return this.safeState(config.id);
                }

                // 타임아웃과 함께 실행
                const result = await Promise.race([
                    config.implementation(),
                    this.timeout(config.maxExecutionTime)
                ]);

                // 실행 후 검증
                if (!this.verifyPostConditions(config.id, result)) {
                    return this.safeState(config.id);
                }

                return result;
            },

            // 테스트 인터페이스
            test: () => this.testSafetyFunction(config.id)
        };

        this.safetyFunctions.set(config.id, safetyFunction);
    }

    // 비상 정지 안전 기능
    emergencyStop: SafetyFunctionConfig = {
        id: 'SF-001',
        name: '비상 정지',
        requiredSIL: 'SIL3',
        requiredPL: 'PLe',
        maxExecutionTime: 100, // 최대 100ms 응답
        implementation: async () => {
            // 이중 채널 구현
            const channel1 = this.activateEmergencyStopChannel1();
            const channel2 = this.activateEmergencyStopChannel2();

            await Promise.all([channel1, channel2]);

            // 정지 확인
            const velocity = await this.measureVelocity();
            if (velocity > 0.001) {
                throw new Error('비상 정지 검증 실패');
            }

            return { success: true, stoppedAt: Date.now() };
        }
    };
}

  • ISO 10218은 비상 정지 및 속도 제한을 포함한 산업용 로봇 안전 요구사항을 정의합니다
  • ISO/TS 15066은 네 가지 협동 모드와 생체역학적 힘 한계를 지정합니다
  • IEC 61508은 기능 안전을 위한 안전 무결성 수준(SIL 1-4)을 도입합니다
  • ISO 13849는 성능 수준(PL a-e)과 카테고리 아키텍처를 사용합니다
  • 안전 등급 소프트웨어는 방어적 프로그래밍, 중복성 및 모니터링이 필요합니다
  • 안전 통신 프로토콜은 오류 감지 및 처리를 제공합니다
  • 포괄적인 문서화는 인증에 필수적입니다

  4. ISO 13849에서 성능 수준(PL)과 카테고리의 관계를 설명하세요.
  5. 안전 등급 소프트웨어에 어떤 방어적 프로그래밍 기법이 필요합니까?
  6. PROFIsafe와 같은 안전 통신 프로토콜은 오류를 어떻게 감지하고 처리합니까?
  7. ISO/TS 15066은 인간-로봇 접촉에 대해 어떤 생체역학적 한계를 지정합니까?
