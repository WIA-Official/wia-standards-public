6장: 하드웨어 안전 설계

6.1 하드웨어를 통한 본질적 안전

하드웨어 안전 설계는 로봇 시스템의 물리적 설계를 통해 위험을 제거하거나 감소시키는 것을 목표로 합니다. 버그나 오작동으로 인해 실패할 수 있는 소프트웨어 기반 안전 조치와 달리, 하드웨어 안전 기능은 소프트웨어 고장에 의해 우회될 수 없는 본질적인 보호를 제공합니다. WIA-AI-SAFETY-PHYSICAL 표준은 하드웨어 안전을 심층 방어 전략의 기초로 강조합니다.

설계 철학: 가장 안전한 로봇은 소프트웨어가 무엇을 명령하든 물리적으로 피해를 입힐 수 없는 로봇입니다. 하드웨어 안전 조치는 위험한 작동을 기계적으로 불가능하게 만들어야 합니다.

6.2 전력 및 힘 제한

로봇이 발휘할 수 있는 최대 전력과 힘을 제한하는 것은 가장 기본적인 하드웨어 안전 조치입니다:

전력 밀도 한계

응용 최대 전력 밀도 일반적 총 전력 근거
협동 로봇 80 W/kg 페이로드 80-150W 총합 인간 접촉 예상
서비스 로봇 150 W/kg 페이로드 200-500W 총합 공공 환경
산업용 로봇 본질적 한계 없음 1-10kW 보호된 작업 공간
이동 플랫폼 50 W/kg 100-500W 보행자 환경 
// 전력 제한 하드웨어 구성

6.3 기계적 정지 및 한계

기계적 정지는 소프트웨어 명령에 관계없이 초과할 수 없는 하드웨어 강제 동작 한계를 제공합니다:

정지 유형 목적 구현 우회
하드 스톱 절대 동작 한계 조인트의 물리적 블록 불가능
소프트 스톱 정상 작동 범위 소프트웨어 + 센서 한계 특수 모드
조정 가능 스톱 응용 특정 한계 기계적 조정기 + 센서 도구 필요
가상 벽 동적 작업 공간 한계 안전 컨트롤러 + 감지 권한 필요 
// 기계적 한계 시행
class MechanicalLimits {
    // 하드웨어 시행이 있는 조인트 각도 한계
    jointLimits = {
        joint1: { softMin: -170, softMax: 170, hardMin: -175, hardMax: 175 },
        joint2: { softMin: -120, softMax: 120, hardMin: -125, hardMax: 125 },
        joint3: { softMin: -170, softMax: 170, hardMin: -175, hardMax: 175 },
        joint4: { softMin: -120, softMax: 120, hardMin: -125, hardMax: 125 },
        joint5: { softMin: -170, softMax: 170, hardMin: -175, hardMax: 175 },
        joint6: { softMin: -360, softMax: 360, hardMin: -365, hardMax: 365 }
    };

    // 조인트별 속도 한계
    velocityLimits = {
        joint1: { soft: 180, hard: 200 },  // deg/s
        joint2: { soft: 180, hard: 200 },
        joint3: { soft: 225, hard: 250 },
        joint4: { soft: 225, hard: 250 },
        joint5: { soft: 225, hard: 250 },
        joint6: { soft: 360, hard: 400 }
    };

    // 하드웨어 리밋 스위치
    limitSwitches = {
        type: 'INDUCTIVE_PROXIMITY',
        arrangement: 'DUAL_REDUNDANT',
        failureMode: 'NORMALLY_CLOSED',
        responseTime: 1  // ms
    };

    checkPosition(jointId: string, position: number): LimitStatus {
        const limits = this.jointLimits[jointId];

        if (position <= limits.hardMin || position >= limits.hardMax) {
            return {
                status: 'HARD_LIMIT',
                action: 'EMERGENCY_STOP',
                detail: `조인트 ${jointId}이 하드 한계에 도달: ${position}`
            };
        }

        if (position <= limits.softMin || position >= limits.softMax) {
            return {
                status: 'SOFT_LIMIT',
                action: 'CONTROLLED_STOP',
                detail: `조인트 ${jointId}이 소프트 한계에 도달: ${position}`
            };
        }

        return { status: 'OK', action: 'CONTINUE' };
    }
}

6.4 유연 및 역구동 가능 액추에이터

유연 액추에이터는 제어된 변형을 허용하여 충격력을 감소시킵니다:

안전을 위한 액추에이터 유형

유형 유연성 역구동성 최적 용도
직렬 탄성 (SEA) 높음 (스프링 요소) 높음 코봇, 충격 흡수
가변 강성 (VSA) 조정 가능 중간-높음 적응형 작업
직접 구동 낮음 높음 빠르고 정밀한 동작
기어드 모터 매우 낮음 낮음 높은 토크 (보호 필요)
케이블/텐던 중간 중간 원격 구동 
// 직렬 탄성 액추에이터 모델링
class SeriesElasticActuator {
    private springStiffness: number;  // N*m/rad
    private motorInertia: number;     // kg*m^2
    private linkInertia: number;      // kg*m^2

    constructor(config: SEAConfig) {
        this.springStiffness = config.springStiffness;
        this.motorInertia = config.motorInertia;
        this.linkInertia = config.linkInertia;
    }

    // 인간과의 충격력 계산
    calculateImpactForce(
        approachVelocity: number,  // m/s
        humanStiffness: number,    // N/m (신체 부위에 따라 다름)
        effectiveMass: number      // kg
    ): ImpactForce {
        // 결합 강성 (직렬 스프링)
        const combinedStiffness = 1 / (
            1 / this.springStiffness +
            1 / humanStiffness
        );

        // 에너지 보존을 사용한 충격력
        // F_max = v * sqrt(k * m)
        const peakForce = approachVelocity * Math.sqrt(
            combinedStiffness * effectiveMass
        );

        // 충격 지속 시간
        const impactDuration = Math.PI * Math.sqrt(
            effectiveMass / combinedStiffness
        );

        return {
            peakForce,
            impactDuration,
            impulse: peakForce * impactDuration / 2
        };
    }

    // 스프링 변형을 통한 토크 감지
    measureJointTorque(): number {
        const motorPosition = this.motorEncoder.getPosition();
        const linkPosition = this.linkEncoder.getPosition();
        const springDeflection = motorPosition - linkPosition;

        return springDeflection * this.springStiffness;
    }

    // 역구동성 확인
    isBackdrivable(externalTorque: number): boolean {
        const frictionTorque = this.staticFriction;
        return Math.abs(externalTorque) > frictionTorque;
    }
}

6.5 보호 외함 및 커버

물리적 외함은 인간을 위험으로부터 보호하고 로봇을 환경 요인으로부터 보호합니다:

보호 유형 목적 요구사항
끼임 방지 가드 손가락 끼임 방지 4mm 미만 간격 또는 25mm 이상 여유
모터 커버 화상, 얽힘 방지 표면 온도 60C 미만, 매끄러운 표면
케이블 관리 얽힘 방지 내부 배선, 도관
소프트 패딩 충격 심각도 감소 에너지 흡수 폼, 10-50mm
충격 쉘 접촉력 분산 큰 반경, 매끄러운 표면

        };
    };

        };

        };

// 충격 보호를 위한 패딩 요구사항 계산
function calculatePaddingRequirement(
    maxVelocity: number,      // m/s
    effectiveMass: number,    // kg
    maxAllowedForce: number   // N
): number {
    // 흡수해야 할 운동 에너지
    const energy = 0.5 * effectiveMass * maxVelocity * maxVelocity;

    // 패딩이 하는 일: W = F * d
    // d(패딩 압축 거리) 계산
    const compressionDistance = energy / maxAllowedForce;

    // 패딩 두께 = 압축 거리의 2배 (최대 50% 압축)
    return compressionDistance * 2 * 1000; // mm로 변환
}

6.6 브레이크 시스템

신뢰할 수 있는 제동은 비상 시 정지 및 위치 유지에 필수적입니다:

브레이크 유형 및 특성

브레이크 유형 작동 응답 시간 안전 장치
스프링 적용 전기 해제 전원 차단 = 제동 10-50ms 예
전기 적용 전원 인가 = 제동 20-100ms 아니오
유압 압력 인가 50-200ms 구성 가능
동적 제동 모터 단락 <10ms 부분적 
// 브레이크 시스템 설계
class BrakeSystem {
    private brakes: JointBrake[];

    // 브레이크 사이징 계산
    calculateBrakeRequirements(
        joint: JointConfig,
        maxLoadTorque: number,
        safetyFactor: number = 2.0
    ): BrakeRequirements {
        // 정적 유지 토크
        const staticTorque = maxLoadTorque * safetyFactor;

        // 동적 제동 토크 (정지용)
        const angularMomentum = joint.inertia * joint.maxVelocity;
        const stoppingTime = 0.5; // 목표 500ms 정지
        const dynamicTorque = angularMomentum / stoppingTime;

        return {
            staticHoldingTorque: staticTorque,
            dynamicBrakingTorque: dynamicTorque,
            minTorque: Math.max(staticTorque, dynamicTorque),
            engagementTime: 50,  // 최대 ms
            releaseTime: 100,    // 최대 ms
            wearIndicator: true,
            temperatureRating: 120  // 섭씨
        };
    }

    // 비상 브레이크 시퀀스
    async engageEmergencyBrakes(): Promise<BrakeResult> {
        const startTime = Date.now();

        // 모든 브레이크 동시 작동
        await Promise.all(
            this.brakes.map(brake => brake.engage())
        );

        // 작동 확인
        const engagementTime = Date.now() - startTime;

        for (const brake of this.brakes) {
            if (!brake.isEngaged()) {
                return {
                    success: false,
                    error: `브레이크 ${brake.id} 작동 실패`,
                    engagementTime
                };
            }
        }

        return { success: true, engagementTime };
    }

    // 브레이크 마모 모니터링
    monitorBrakeHealth(): BrakeHealthReport {
        const reports: JointBrakeHealth[] = [];

        for (const brake of this.brakes) {
            const health = {
                jointId: brake.jointId,
                wearPercentage: brake.getWearLevel(),
                engagementCycles: brake.getCycleCount(),
                lastTestResult: brake.getLastTestResult(),
                estimatedLifeRemaining: brake.estimateRemainingLife()
            };

            reports.push(health);

            // 마모가 임계값 초과 시 경고
            if (health.wearPercentage > 80) {
                this.alertMaintenanceRequired(brake.jointId, '브레이크 마모 >80%');
            }
        }

        return { joints: reports, timestamp: Date.now() };
    }
}

6.7 전기 안전

전기 안전은 감전, 화재 및 간섭 위험으로부터 보호합니다:

위험 보호 표준
감전 클래스 I 접지, 이중 절연 IEC 60204-1
과전류 퓨즈, 회로 차단기 IEC 60204-1
지락 고장 RCD/GFCI, 절연 모니터링 IEC 60204-1
EMC 간섭 차폐, 필터링 IEC 61000-6
정전기 방전 접지, ESD 보호 IEC 61000-4-2
배터리 위험 BMS, 열 관리 IEC 62133 
// 전기 안전 시스템
class ElectricalSafety {
    // 지락 고장 감지
    monitorGroundFault(): GroundFaultStatus {
        const leakageCurrent = this.measureLeakageCurrent();
        const threshold = 30; // mA (인체 보호)

        if (leakageCurrent > threshold) {
            this.tripGroundFaultProtection();
            return {
                fault: true,
                current: leakageCurrent,
                action: 'CIRCUIT_TRIPPED'
            };
        }

        return { fault: false, current: leakageCurrent };
    }

    // 배터리 관리 안전
    batteryManagement: BatteryManagementSystem = {
        monitoring: {
            cellVoltageMin: 2.5,    // V
            cellVoltageMax: 4.2,    // V
            temperatureMin: 0,       // C
            temperatureMax: 45,      // C
            maxChargeCurrent: 1.0,   // C-레이트
            maxDischargeCurrent: 2.0 // C-레이트
        },

        protection: {
            overVoltageProtection: true,
            underVoltageProtection: true,
            overCurrentProtection: true,
            overTemperatureProtection: true,
            shortCircuitProtection: true,
            cellBalancing: true
        },

        thermal: {
            coolingMethod: 'ACTIVE_AIR',
            fanTriggerTemp: 35,      // C
            shutdownTemp: 50         // C
        }
    };

    // 절연 테스트
    performIsolationTest(): IsolationTestResult {
        // 절연 저항 테스트
        const testVoltage = 500; // V DC
        const minResistance = 1; // MOhm

        const measuredResistance = this.measureInsulation(testVoltage);

        return {
            passed: measuredResistance >= minResistance,
            testVoltage,
            measuredResistance,
            minRequired: minResistance
        };
    }
}

6.8 센서 하드웨어 설계

안전 센서는 신뢰성을 보장하기 위해 신중한 하드웨어 설계가 필요합니다:

// 안전 센서 하드웨어 요구사항
interface SafetySensorHardware {
    // 중복성 요구사항
    redundancy: {
        type: 'DUAL_CHANNEL' | 'TRIPLE_MODULAR',
        diversity: boolean,  // 다른 센서 유형
        crossMonitoring: boolean
    };

    // 응답 시간
    timing: {
        sampleRate: number,      // Hz
        processingTime: number,  // ms
        outputDelay: number,     // ms

    // 환경 등급

    // 진단 기능
    diagnostics: {
        selfTest: boolean,
        crossCheck: boolean,

핵심 요점

  • 하드웨어 안전은 소프트웨어로 우회할 수 없는 본질적 보호를 제공합니다
  • 하드웨어 제약을 통한 전력 및 힘 제한은 기본적인 안전 조치입니다
  • 기계적 정지는 소프트웨어 명령에 관계없이 절대적인 동작 한계를 제공합니다
  • 유연 액추에이터(SEA, VSA)는 제어된 변형을 통해 충격력을 감소시킵니다
  • 보호 외함은 끼임 지점, 화상 및 얽힘을 방지해야 합니다
  • 스프링 적용 브레이크는 전원 손실 시 안전한 정지를 제공합니다
  • 전기 안전은 적절한 접지, 절연 및 보호 회로가 필요합니다
  • 안전 센서는 중복성, 빠른 응답 및 자가 진단이 필요합니다
