7장: 환경 안전 고려사항

7.1 환경별 안전 요구사항

물리적 AI 시스템은 다양한 환경에서 작동하며, 각각 고유한 안전 과제를 제시합니다. WIA-AI-SAFETY-PHYSICAL 표준은 핵심 안전 원칙을 유지하면서 안전 요구사항이 특정 배포 맥락에 맞게 조정되어야 함을 인식합니다. 이 장에서는 산업, 의료, 가정 및 야외 환경 전반에 걸친 안전 고려사항을 검토합니다.

핵심 원칙: 물리적 AI 시스템의 안전 요구사항은 운영 환경의 위험 프로필, 상호작용하는 인간의 취약성, 잠재적 고장의 결과와 일치해야 합니다.

7.2 산업 환경 안전

산업 환경은 훈련된 인력과 확립된 안전 인프라가 있는 고위험 시나리오를 제시합니다:

산업 안전 요구사항

측면 요구사항 표준 참조
인력 훈련된 운영자만, 권한 시스템 ISO 10218-2
작업 공간 정의된 보호 공간, 접근 제한 ISO 10218-2
보호 장치 물리적 장벽 또는 안전 등급 감지 ISO 13855
속도/힘 ISO/TS 15066 한계 또는 완전 보호 ISO/TS 15066
통합 기계 안전 통합 (셀 설계) ISO 10218-2
유지보수 LOTO 절차, 유지보수 모드 ISO 14118 
// 산업 셀 안전 구성
interface IndustrialCellConfig {
    workspace: {

        dimensions: { length: number; width: number; height: number };
        accessPoints: AccessPoint[];
    };

    guarding: {
        physical: {
            fenceHeight: number;  // mm, 일반적으로 1400-2000
            gateType: 'INTERLOCKED' | 'TRAPPED_KEY';
            materialStrength: string;  // 예: 'ISO 14120 준수'
        };
        electronic: {
            lightCurtains: LightCurtainConfig[];
            safetyMats: SafetyMatConfig[];
            areaScanners: AreaScannerConfig[];
        };
    };

    accessControl: {

        keySwitch: boolean;
        biometric: boolean;
        accessLog: boolean;
    };

// ISO 13855에 따른 안전 거리 계산
function calculateSafetyDistance(
    approachSpeed: number,  // mm/s (보행 시 일반적으로 1600-2000)
    systemResponseTime: number,  // 초
    sensorResolution: number,  // mm (감지 능력)
    intrusion: number  // mm (감지면을 지난 신체 침입)
): number {
    // S = K * T + C
    // 여기서 K = 접근 속도, T = 응답 시간, C = 침투 계수
    const C = 8 * (sensorResolution - 14);  // ISO 13855 공식
    const additionalDistance = intrusion;

    return approachSpeed * systemResponseTime + Math.max(C, 0) + additionalDistance;
}

// 예시 계산
const safetyDistance = calculateSafetyDistance(
    1600,   // 1.6 m/s 접근 속도
    0.5,    // 500ms 시스템 응답
    30,     // 30mm 감지 해상도
    100     // 100mm 침입 허용
);
// 결과: 약 1028mm 최소 안전 거리

7.3 의료 환경 안전

의료 환경은 취약한 환자와 멸균 요구사항으로 인해 강화된 안전이 필요합니다:

의료 특정 요구사항

범주 요구사항 근거
환자 접촉 초저 힘 한계 (최대 30-50N) 취약하고 움직일 수 없는 환자
위생 세척 가능한 표면, 밀봉된 조인트 감염 관리
EMC 의료 기기 호환성 모니터와의 간섭 없음
재료 생체적합성, 비독성 환자 안전
신뢰성 중요 기능 99.9%+ 가동 시간 환자 케어 연속성
감독 의료 전문가 감독 임상적 책임 
// 의료 로봇 안전 프로필
class HealthcareRobotSafety {
    // 환자 상호작용 힘 한계 (매우 보수적)
    patientForceLimits = {
        staticContact: 30,      // 최대 N
        transientContact: 50,   // 최대 N
        pressureLimit: 3,       // N/cm^2
        contactArea: 20         // 최소 cm^2
    };

    // 위생 요구사항
    hygieneRequirements = {
        surfaceMaterial: 'MEDICAL_GRADE_POLYMER',
        ipRating: 'IP65',       // 세척 가능
        uvResistant: true,      // UV 멸균용
        chemicalResistant: [
            'ISOPROPYL_ALCOHOL',
            'HYDROGEN_PEROXIDE',
            'QUATERNARY_AMMONIUM'
        ],
        seamless: true,         // 세균 번식 방지
        colorCoding: true       // 시각적 위생 상태
    };

    // 의료 환경 EMC 요구사항
    emcRequirements = {
        emissions: 'IEC_60601_1_2_CLASS_B',
        immunity: 'IEC_60601_1_2',
        wirelessCoexistence: true,
        surgicalCompatible: false  // 수술실 사용 불가
    };

    // 환자 근접 동작 검증
    validatePatientProximity(
        patientState: PatientState,
        robotState: RobotState
    ): SafetyDecision {
        const distance = this.calculateDistance(patientState, robotState);

        // 의료용 더 보수적인 구역
        if (distance < 0.2) {  // < 20cm
            return {
                action: 'MINIMAL_SPEED',
                maxSpeed: 0.05,  // 최대 50mm/s
                maxForce: this.patientForceLimits.staticContact,
                reason: '환자와 근접'
            };
        }

        if (patientState.mobility === 'IMMOBILE' ||
            patientState.awareness === 'UNCONSCIOUS') {
            // 취약한 환자에 대한 추가 주의
            return {
                action: 'ENHANCED_MONITORING',
                maxSpeed: 0.1,
                requireConfirmation: true,
                reason: '취약한 환자'
            };
        }

        return { action: 'NORMAL_HEALTHCARE_MODE' };
    }
}

7.4 가정 환경 안전

가정 환경은 훈련받지 않은 사용자, 어린이 및 애완동물로 인해 고유한 과제를 제시합니다:

가정용 로봇 안전 고려사항

과제 위험 완화
훈련받지 않은 사용자 예측할 수 없는 상호작용 직관적 인터페이스, 보수적 기본값
어린이 호기심, 작은 크기 어린이 감지, 추가 저힘 한계
애완동물 예측할 수 없는 움직임 애완동물 감지, 회피 동작
어수선함 장애물, 넘어짐 위험 강건한 인식, 신중한 내비게이션
다양한 조명 센서 성능 저하 다중 모달 감지, 적응
24/7 운영 야간 운영, 잠자는 사용자 조용한 모드, 야간 활동 감소

7.5 야외 환경 안전

야외 운영은 날씨, 지형 및 대중 상호작용 과제를 도입합니다:

야외 운영 조건

조건 안전에 미치는 영향 완화
비 센서 성능 저하, 미끄러짐 위험 IP67 등급, 속도 감소
눈/얼음 견인력 상실, 센서 막힘 견인 제어, 가열 센서
극한 온도 구성 요소 고장, 배터리 문제 열 관리, 운영 한계
직사광선 카메라 포화, ToF 간섭 ND 필터, 다중 센서 융합
바람 안정성 문제 (드론, 키 큰 로봇) 풍속 한계, 안정성 제어
공공 장소 예측할 수 없는 보행자 보수적 동작, 명확한 신호 
// 야외 로봇 안전 시스템
class OutdoorRobotSafety {
    // 날씨 조건 모니터링
    weatherMonitor = {
        sensors: ['TEMPERATURE', 'HUMIDITY', 'RAIN', 'WIND', 'LIGHT'],

        operatingLimits: {
            temperature: { min: -10, max: 45 },  // 섭씨
            windSpeed: { max: 15 },               // m/s
            rainRate: { max: 10 },                // mm/시간
            visibility: { min: 50 }               // 미터
        },

        adaptations: {
            rain: {
                speedReduction: 0.5,
                increasedFollowDistance: 2.0,
                ultrasonicPreference: true
            },
            lowVisibility: {
                speedReduction: 0.3,
                enhancedLighting: true,
                stopIfCritical: true
            },
            highWind: {
                speedReduction: 0.7,
                avoidTallStructures: true,
                groundRobotOnly: true
            }
        }
    };

    // 공공 장소 동작
    publicSpaceBehavior = {
        pedestrianProtocol: {
            yieldToPedestrians: true,
            predictPedestrianPath: true,
            clearIntentSignaling: true,
            minPedestrianDistance: 1.0  // 미터
        },

        signaling: {
            intentionLights: true,       // 의도된 경로 표시
            audibleWarnings: true,       // 뒤에서 접근 시 경고
            displayScreen: true          // 상태/메시지 표시
        },

        crowdBehavior: {
            maxCrowdDensity: 0.5,        // 명/m^2
            stopInDenseCrowd: true,
            alternateRouting: true
        }
    };

    // 지형 평가
    assessTerrain(terrain: TerrainData): TerrainSafety {
        const hazards: TerrainHazard[] = [];

        // 경사 확인
        if (terrain.slope > 15) {  // 도
            hazards.push({
                type: 'STEEP_SLOPE',
                severity: terrain.slope > 25 ? 'HIGH' : 'MEDIUM',
                mitigation: 'REDUCE_SPEED_OR_AVOID'
            });
        }

        // 표면 유형 확인
        if (['ICE', 'WET_GRASS', 'GRAVEL'].includes(terrain.surface)) {
            hazards.push({
                type: 'LOW_TRACTION',
                severity: 'MEDIUM',
                mitigation: 'REDUCE_SPEED'
            });
        }

        // 장애물 확인
        if (terrain.obstacleHeight > 0.05) {  // 5cm
            hazards.push({
                type: 'OBSTACLE',
                severity: 'HIGH',
                mitigation: 'NAVIGATE_AROUND'
            });
        }

        return {
            passable: hazards.filter(h => h.severity === 'HIGH').length === 0,
            hazards,
            recommendedSpeed: this.calculateSafeSpeed(hazards)
        };
    }
}

7.6 다중 로봇 환경 안전

여러 로봇이 같은 공간에서 작동할 때 조정이 필수적입니다:

// 다중 로봇 조정 안전
class MultiRobotSafety {
    private robots: Map<string, RobotState> = new Map();

    // 로봇 간 충돌 회피
    checkRobotProximity(robotId: string): RobotProximityResult {
        const myRobot = this.robots.get(robotId);
        const nearbyRobots = this.findNearbyRobots(robotId, 5.0);  // 5m 반경

        for (const other of nearbyRobots) {
            const distance = this.calculateDistance(myRobot, other);
            const closingSpeed = this.calculateClosingSpeed(myRobot, other);
            const timeToContact = distance / closingSpeed;

            if (timeToContact < 2.0) {  // 접촉까지 2초 미만
                return {
                    conflict: true,
                    otherRobot: other.id,
                    timeToContact,
                    recommendedAction: this.resolveConflict(myRobot, other)
                };
            }
        }

        return { conflict: false };
    }

    // 교통 관리
    trafficRules = {
        rightOfWay: [
            'EMERGENCY_ROBOT',      // 최고 우선순위
            'PATIENT_TRANSPORT',
            'PAYLOAD_CARRIER',
            'IDLE_ROBOT'            // 최저 우선순위
        ],

        intersectionProtocol: {
            type: 'TOKEN_BASED',
            maxWaitTime: 30,        // 초
            deadlockResolution: 'LOWEST_ID_YIELDS'
        },

        corridorProtocol: {
            passingAllowed: true,
            minPassingWidth: 1.5,   // 미터
            maxSpeedInCorridor: 0.5  // m/s
        }
    };

    resolveConflict(robot1: RobotState, robot2: RobotState): ConflictResolution {
        // 우선순위 확인
        const priority1 = this.trafficRules.rightOfWay.indexOf(robot1.role);
        const priority2 = this.trafficRules.rightOfWay.indexOf(robot2.role);

        if (priority1 < priority2) {
            return {
                action: 'CONTINUE',
                otherAction: 'YIELD',
                message: `${robot2.id}이(가) 양보해야 함`
            };
        } else if (priority2 < priority1) {
            return {
                action: 'YIELD',
                otherAction: 'CONTINUE',
                message: '낮은 우선순위, 양보 중'
            };
        } else {
            // 같은 우선순위 - 결정론적 타이브레이커 사용
            return robot1.id < robot2.id
                ? { action: 'CONTINUE', otherAction: 'YIELD' }
                : { action: 'YIELD', otherAction: 'CONTINUE' };
        }
    }
}

7.7 환경 모니터링 통합

물리적 AI 시스템은 건물 및 환경 모니터링 시스템과 통합되어야 합니다:

// 환경 통합
class EnvironmentalIntegration {
    // 건물 관리 시스템 통합
    bmsIntegration = {
        inputs: [
            'FIRE_ALARM_STATUS',
            'HVAC_STATUS',
            'OCCUPANCY_SENSORS',
            'ACCESS_CONTROL',
            'LIGHTING_LEVELS'
        ],

        responses: {
            FIRE_ALARM: {
                action: 'EMERGENCY_STOP',
                clearPath: true,
                moveToSafeZone: true
            },
            EVACUATION: {
                action: 'ASSIST_EVACUATION',
                wayfinding: true,
                avoidEscapeRoutes: false
            },
            POWER_OUTAGE: {
                action: 'SAFE_SHUTDOWN',
                preserveState: true
            }
        }
    };

    // 공기질 모니터링
    airQualityResponse(reading: AirQualityReading): SafetyResponse {
        if (reading.co2 > 5000) {  // ppm - 위험 수준
            return {
                action: 'ALERT_AND_EVACUATE',
                severity: 'CRITICAL'
            };
        }

        if (reading.particulateMatter > 150) {  // AQI
            return {
                action: 'REDUCE_ACTIVITY',
                reason: '대기질 불량',
                affectsSensors: true
            };
        }

        return { action: 'NORMAL_OPERATION' };
    }

    // 비상 통합
    handleEmergencyBroadcast(emergency: EmergencyBroadcast): void {
        switch (emergency.type) {
            case 'FIRE':
                this.executeFireResponse(emergency);
                break;
            case 'EARTHQUAKE':
                this.executeEarthquakeResponse(emergency);
                break;
            case 'SECURITY':
                this.executeSecurityResponse(emergency);
                break;
            case 'MEDICAL':
                this.executeMedicalResponse(emergency);
                break;
        }
    }
}

핵심 요점

  • 안전 요구사항은 특정 배포 환경에 맞게 조정되어야 합니다
  • 산업 환경은 보호 장치, 훈련된 인력 및 확립된 안전 인프라를 사용합니다
  • 의료 환경은 초저 힘 한계와 위생 고려사항이 필요합니다
  • 가정 환경은 훈련받지 않은 사용자, 어린이 및 애완동물을 고려해야 합니다
  • 야외 환경은 날씨, 지형 및 대중 상호작용 과제를 도입합니다
  • 다중 로봇 환경은 조정 및 교통 관리가 필요합니다
  • 건물 시스템과의 통합은 비상 대응을 가능하게 합니다
  • 환경 조건은 지속적으로 모니터링되고 적응되어야 합니다
