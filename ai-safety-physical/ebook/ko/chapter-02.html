2장: 위험 평가 및 위험 분석

2.1 위험 평가 개요

위험 평가는 위험 요소를 식별하고, 위험을 분석하며, 안전 조치가 적절한지 평가하는 체계적인 프로세스입니다. 물리적 AI 시스템의 경우, 위험 평가는 자율적 의사결정, 학습 시스템, 복잡한 인간-로봇 상호작용이 제기하는 고유한 과제를 고려해야 합니다.

WIA-AI-SAFETY-PHYSICAL 표준은 모든 안전 활동의 기초로서 포괄적인 위험 평가를 요구합니다. 이 평가는 시스템 수명 주기 전반에 걸쳐 반복적으로 수행되어야 하며, 변경 사항이 발생할 때마다 업데이트되어야 합니다.

위험 방정식:
위험 = 심각도 x 확률 x 노출

여기서:
- 심각도: 잠재적 피해 규모(경미한 부상에서 사망까지)
- 확률: 위험한 사건이 발생할 가능성
- 노출: 위험 구역에서 인간의 존재 빈도 및 지속 시간

2.2 위험 요소 식별 방법

체계적인 위험 요소 식별은 중요한 위험이 간과되지 않도록 보장합니다:

구조화된 What-If 기법 (SWIFT)

구조화된 질문을 사용하는 체계적인 브레인스토밍 접근 방식:

질문 범주 예시 질문 발견되는 일반적 위험
시스템 고장 센서가 고장 나면? 통신이 끊기면? 제어 불능 동작, 충돌
인적 오류 운영자가 잘못된 명령을 내리면? 의도하지 않은 작동, 안전 우회
환경적 요인 조명 조건이 변하면? 인식 실패, 내비게이션 오류
AI 동작 AI가 객체를 잘못 식별하면? 잘못된 동작, 예상치 못한 움직임
통합 타이밍이 어긋나면? 경쟁 조건, 조정 실패

위험 및 운영성 연구 (HAZOP)

HAZOP은 가이드 워드를 사용하여 설계 의도에서 벗어나는 경우를 체계적으로 탐색합니다:

// HAZOP 분석 구조
interface HAZOPAnalysis {
    node: string;           // 분석 중인 시스템 구성 요소
    parameter: string;      // 설계 매개변수 (속도, 힘, 위치)
    guideWord: GuideWord;   // NO, MORE, LESS, REVERSE, OTHER 등
    deviation: string;      // 의도에서 벗어난 결과
    causes: string[];       // 이탈의 잠재적 원인
    consequences: string[]; // 잠재적 결과
    safeguards: string[];   // 기존 보호 조치
    recommendations: string[]; // 필요한 추가 조치
}

enum GuideWord {
    NO = "설계 의도의 완전한 부정",
    MORE = "정량적 증가",
    LESS = "정량적 감소",
    REVERSE = "반대 방향/논리",
    PART_OF = "불완전한 이행",
    AS_WELL_AS = "추가적인 의도하지 않은 활동",
    OTHER = "완전한 대체"
}

// HAZOP 항목 예시
const exampleEntry: HAZOPAnalysis = {
    node: "로봇 팔 조인트 3",
    parameter: "각속도",
    guideWord: GuideWord.MORE,
    deviation: "조인트가 명령보다 빠르게 움직임",
    causes: [
        "모터 컨트롤러 고장",
        "인코더 고장으로 더 낮은 속도 보고",
        "소프트웨어 계산 오류"
    ],
    consequences: [
        "충돌 시 충격력 증가",
        "위치 정확도 손실",
        "조인트에 대한 기계적 스트레스"
    ],
    safeguards: [
        "하드웨어 속도 조절기",
        "보조 인코더",
        "워치독 타이머"
    ],
    recommendations: [
        "독립적인 속도 모니터링 추가",
        "속도 범위 검사 구현"
    ]
};

2.3 고장 모드 및 영향 분석 (FMEA)

FMEA는 잠재적 고장 모드와 시스템 안전에 미치는 영향을 체계적으로 분석합니다:

구성요소 고장 모드 영향 심각도 발생도 검출도 RPN
근접 센서 위음성 인간과 충돌 9 3 4 108
모터 드라이버 폭주 제어 불능 동작 10 2 3 60
안전 PLC 프로세서 정지 안전 시스템 비활성화 10 2 2 40
비상 정지 버튼 눌린 상태 고착 재시작 불가 3 2 1 6
AI 비전 오분류 잘못된 동작 7 4 5 140
위험 우선순위 번호 (RPN) = 심각도 x 발생도 x 검출도
  • 각 요소는 1-10으로 평가
  • RPN 범위: 1-1000
  • 높은 RPN 항목은 즉각적인 조치 필요
  • 일반적 임계값: RPN > 100은 완화 조치 필요

2.4 고장 트리 분석 (FTA)

FTA는 원하지 않는 이벤트(정상 이벤트)에서 역으로 작업하여 모든 가능한 원인을 식별합니다:

// 고장 트리 예시: 로봇에 의한 인간 부상
/*
                    [인간 부상]
                          |
                        (OR)
              ____________|____________
             |            |            |
      [충돌]       [압착]       [얽힘]
           |
         (AND)
      ____|____
     |         |
[로봇이    [인간이
 움직임]   작업 공간에]
     |         |
   (OR)      (OR)
    |       __|__
    |      |     |
[정상   [무단    [센서
 작동]  진입]   고장]

*/

interface FaultTreeNode {
    id: string;
    type: 'EVENT' | 'GATE' | 'BASIC';
    gateType?: 'AND' | 'OR' | 'NOT' | 'XOR';
    description: string;
    probability?: number;
    children?: FaultTreeNode[];
}

function calculateTopEventProbability(node: FaultTreeNode): number {
    if (node.type === 'BASIC') {
        return node.probability || 0;
    }

    const childProbs = node.children?.map(calculateTopEventProbability) || [];

    switch (node.gateType) {
        case 'OR':
            // P(A OR B) = P(A) + P(B) - P(A)P(B)
            return childProbs.reduce((acc, p) => acc + p - acc * p, 0);
        case 'AND':
            // P(A AND B) = P(A) * P(B)
            return childProbs.reduce((acc, p) => acc * p, 1);
        default:
            return 0;
    }
}

// 예시 계산
const collisionTree: FaultTreeNode = {
    id: 'TOP',
    type: 'GATE',
    gateType: 'AND',
    description: '인간과의 충돌',
    children: [
        {
            id: 'MOTION',
            type: 'BASIC',
            description: '로봇이 움직임',
            probability: 0.8 // 로봇이 80%의 시간 동안 작동
        },
        {
            id: 'HUMAN',
            type: 'GATE',
            gateType: 'OR',
            description: '인간이 충돌 구역에 있음',
            children: [
                { id: 'AUTH', type: 'BASIC', description: '허가된 진입', probability: 0.1 },
                { id: 'UNAUTH', type: 'BASIC', description: '무단 진입', probability: 0.01 },
                { id: 'SENSOR', type: 'BASIC', description: '센서 고장', probability: 0.001 }
            ]
        }
    ]
};

// 충돌 확률 = 0.8 * (0.1 + 0.01 + 0.001 - ...) ≈ 0.088

2.6 AI 특정 위험 고려사항

물리적 AI 시스템은 특별히 다루어야 할 고유한 위험 요소를 도입합니다:

기계 학습 위험 요소

위험 요소 설명 완화 전략
분포 이동 실제 환경이 훈련 데이터와 다름 분포 외 감지, 보수적 대체
적대적 입력 의도적으로 제작된 기만적 입력 입력 검증, 이상 감지
개념 드리프트 시간이 지남에 따라 환경이 변함 지속적 모니터링, 주기적 재훈련
불확실성 맹목 모델이 자신이 모르는 것을 모름 불확실성 정량화, 신뢰도 임계값
보상 해킹 목적 함수의 허점 발견 다중 안전 제약, 인간 감독 
// AI 위험 평가 프레임워크
class AIRiskAssessor {
    assessModelRisk(model: AIModel, context: DeploymentContext): RiskProfile {
        const risks: Risk[] = [];

        // 분포 이동 위험
        const distributionRisk = this.assessDistributionShift(
            model.trainingData,
            context.expectedInputs
        );
        risks.push(distributionRisk);

        // 적대적 강건성
        const adversarialRisk = this.assessAdversarialRobustness(model);
        risks.push(adversarialRisk);

        // 불확실성 교정
        const uncertaintyRisk = this.assessUncertaintyCalibration(model);
        risks.push(uncertaintyRisk);

        // 고장 모드 분석
        const failureModes = this.identifyAIFailureModes(model);
        risks.push(...failureModes);

        return this.aggregateRisks(risks);
    }

    assessDistributionShift(training: DataProfile, deployment: DataProfile): Risk {
        // 훈련 데이터와 배포 데이터의 통계적 비교
        const divergence = this.calculateKLDivergence(training, deployment);

        if (divergence > 0.5) {
            return {
                type: 'DISTRIBUTION_SHIFT',
                severity: 'HIGH',
                description: '훈련과 배포 간 상당한 불일치',
                mitigation: '배포 대표 데이터로 모델 재훈련'
            };
        }
        // ... 추가 분석
    }

    assessConfidenceThreshold(model: AIModel): number {
        // 결정을 위한 안전한 신뢰도 임계값 결정
        // 이 임계값 미만에서는 시스템이 인간 지원을 요청해야 함
        const calibrationError = this.measureCalibrationError(model);

        // 교정 오류가 높을수록 더 높은 신뢰도 임계값 필요
        return 0.5 + (calibrationError * 0.4); // 범위: 0.5 - 0.9
    }
}

2.7 정량적 위험 평가

정량적 방법은 안전 관련 결정에 필수적인 수치적 위험 추정치를 제공합니다:

확률론적 위험 평가 (PRA)

// 확률론적 위험 평가
interface PRAResult {
    annualizedRisk: number;           // 연간 예상 사고 수
    fatalityRisk: number;             // 연간 사망 확률
    expectedCost: number;             // 연간 예상 손실 (통화)
    confidenceInterval: [number, number]; // 90% 신뢰 구간
}

function calculateAnnualizedRisk(
    hazardRate: number,      // 운영 시간당 위험 수
    operatingHours: number,  // 연간 운영 시간
    exposureFactor: number,  // 인간 노출 시간 비율
    injuryProbability: number // P(부상 | 위험 발생)
): number {
    return hazardRate * operatingHours * exposureFactor * injuryProbability;
}

// 예시: 협동 로봇 응용
const cobot = {
    hazardRate: 1e-6,        // 백만 시간당 한 번의 위험
    operatingHours: 4000,     // 연간 4000시간 운영
    exposureFactor: 0.5,      // 50% 시간 동안 인간 존재
    injuryProbability: 0.01   // 위험 발생 시 1% 부상 확률
};

const annualRisk = calculateAnnualizedRisk(
    cobot.hazardRate,
    cobot.operatingHours,
    cobot.exposureFactor,
    cobot.injuryProbability
);

// 결과: 2e-8 또는 약 5천만 년당 1회 부상
// 대부분의 응용에서 허용 가능한 수준으로 간주됨

2.10 지속적인 위험 모니터링

위험 평가는 일회성 활동이 아니라 운영 전반에 걸쳐 지속적인 모니터링이 필요합니다:

// 실시간 위험 모니터링 시스템
class RuntimeRiskMonitor {
    private riskIndicators: Map<string, RiskIndicator> = new Map();
    private alertThresholds: AlertThresholds;

    registerIndicator(indicator: RiskIndicator): void {
        this.riskIndicators.set(indicator.id, indicator);
    }

    async monitorContinuously(): Promise<void> {
        while (true) {
            const currentRisk = await this.calculateCurrentRisk();

            if (currentRisk.level > this.alertThresholds.warning) {
                this.raiseWarning(currentRisk);
            }

            if (currentRisk.level > this.alertThresholds.critical) {
                this.initiateRiskMitigation(currentRisk);
            }

            this.logRiskMetrics(currentRisk);
            await this.sleep(100); // 10Hz 모니터링
        }
    }

    calculateCurrentRisk(): CurrentRiskProfile {
        let aggregateRisk = 0;
        const contributingFactors: RiskFactor[] = [];

        for (const [id, indicator] of this.riskIndicators) {
            const value = indicator.getCurrentValue();
            const contribution = indicator.weight * value;
            aggregateRisk += contribution;

            if (contribution > 0.1) {
                contributingFactors.push({
                    indicator: id,
                    value,
                    contribution
                });
            }
        }

        return {
            level: aggregateRisk,
            timestamp: Date.now(),
            factors: contributingFactors,
            trend: this.calculateTrend()
        };
    }
}

핵심 요점

  • 위험 평가는 체계적이고 포괄적이며 지속적으로 업데이트되어야 합니다
  • 다양한 위험 식별 방법(SWIFT, HAZOP, FMEA, FTA)이 상호 보완적인 관점을 제공합니다
  • 위험 = 심각도 x 확률 x 노출이 기본 프레임워크를 제공합니다
