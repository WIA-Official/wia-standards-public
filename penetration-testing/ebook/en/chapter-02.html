CHAPTER 02

Reconnaissance and Information Gathering

Mastering OSINT, scanning techniques, and enumeration for comprehensive security assessment

🎯 Overview of Reconnaissance

Reconnaissance, often called "recon," is the first and one of the most critical phases of penetration testing. It involves gathering as much information as possible about the target to identify potential attack vectors and vulnerabilities. The quality of reconnaissance directly impacts the success of subsequent testing phases.

The Reconnaissance Principle

"The more you know about your target, the more effective your attack will be. Information is the most powerful weapon in a penetration tester's arsenal."

Reconnaissance is typically divided into two main categories:

  • Passive Reconnaissance: Gathering information without directly interacting with the target
  • Active Reconnaissance: Directly interacting with the target system to gather information

🕵️ Passive Reconnaissance (OSINT)

Open Source Intelligence (OSINT) involves collecting information from publicly available sources without alerting the target.

Key OSINT Sources

Source Information Type Example Tools/Sites
Search Engines Public documents, cached pages, metadata Google Dorking, Bing, DuckDuckGo
DNS Records Subdomains, mail servers, IP addresses dig, nslookup, host
WHOIS Databases Domain registration, contact info whois, ICANN lookup
Social Media Employee info, tech stack, locations LinkedIn, Twitter, Facebook
Job Postings Technologies used, security gaps LinkedIn Jobs, Indeed, Glassdoor
Public Repositories Source code, credentials, configs GitHub, GitLab, Bitbucket
Certificate Transparency SSL certificates, subdomains crt.sh, Censys

Google Dorking Techniques

Advanced Google search operators for discovering sensitive information:

# Find specific file types
site:example.com filetype:pdf
site:example.com filetype:xlsx

# Discover login pages
site:example.com inurl:admin
site:example.com inurl:login

# Find exposed directories
site:example.com intitle:"index of"

# Search for specific text in pages
site:example.com intext:"confidential"
site:example.com intext:"internal use only"

# Discover subdomains
site:*.example.com

# Find cached versions
cache:example.com

# Search within URL
inurl:admin site:example.com

⚠️ Legal Consideration

While OSINT uses publicly available information, always ensure your reconnaissance activities comply with applicable laws and your rules of engagement. Some jurisdictions may have restrictions on certain types of information gathering.

🌐 DNS Enumeration

Domain Name System (DNS) enumeration reveals crucial information about target infrastructure:

Basic DNS Queries

# Query A records (IPv4 addresses)
dig example.com A

# Query MX records (mail servers)
dig example.com MX

# Query NS records (name servers)
dig example.com NS

# Query TXT records (SPF, DKIM, etc.)
dig example.com TXT

# Attempt zone transfer (rarely works nowadays)
dig @ns1.example.com example.com AXFR

# Query all records
dig example.com ANY

Subdomain Discovery

Finding subdomains is crucial for expanding the attack surface:

# Using common subdomain wordlist
# (Example command - requires appropriate tools)
subdomain-enum --domain example.com \
  --wordlist /usr/share/wordlists/subdomains.txt

# Certificate transparency logs
curl -s "https://crt.sh/?q=%.example.com&output=json" | \
  jq -r '.[].name_value' | sort -u

# Example output:
api.example.com
dev.example.com
staging.example.com
admin.example.com
mail.example.com
Record Type Purpose Security Relevance
A / AAAA IPv4 / IPv6 address Direct target identification
MX Mail exchange servers Email infrastructure, potential phishing targets
NS Name servers DNS infrastructure, potential targets
TXT Text records SPF, DKIM, configuration details
CNAME Canonical name (alias) Service relationships, third-party integrations
SOA Start of authority Zone information, administrative contacts

🔎 Active Reconnaissance

Active reconnaissance involves directly interacting with target systems to gather information:

Port Scanning with Nmap

Nmap is the de facto standard for network discovery and security auditing:

# Basic TCP SYN scan (stealthy, requires root)
sudo nmap -sS target.example.com

# TCP connect scan (no root required)
nmap -sT target.example.com

# UDP scan (slower but important)
sudo nmap -sU target.example.com

# Service version detection
nmap -sV target.example.com

# OS detection
sudo nmap -O target.example.com

# Aggressive scan (version, OS, scripts, traceroute)
nmap -A target.example.com

# Scan specific ports
nmap -p 80,443,8080 target.example.com

# Scan port range
nmap -p 1-1000 target.example.com

# Fast scan (top 100 ports)
nmap -F target.example.com

# Comprehensive scan
nmap -sS -sV -O -p- target.example.com

Nmap Scan Types

Scan Type Flag Use Case Stealth Level
TCP SYN -sS Default, fast, stealthy Medium
TCP Connect -sT Full connection, no root needed Low
UDP Scan -sU Find UDP services Medium
NULL Scan -sN Firewall evasion High
FIN Scan -sF Firewall evasion High
XMAS Scan -sX Firewall evasion High

📊 Service Enumeration

After identifying open ports, enumerate services for vulnerabilities:

HTTP/HTTPS Enumeration

# Banner grabbing
curl -I https://example.com

# Output example:
HTTP/2 200
server: nginx/1.18.0
date: Wed, 27 Dec 2025 10:30:00 GMT
content-type: text/html
x-powered-by: PHP/7.4.3

# Discover hidden directories and files
# (Example - requires dirb or similar tool)
dirb https://example.com /usr/share/wordlists/dirb/common.txt

# Web technology fingerprinting
# Tools like Wappalyzer, whatweb, or builtwith

SMB Enumeration

# List SMB shares
smbclient -L //target.example.com -N

# Connect to a share
smbclient //target.example.com/share -U username

# Enumerate SMB with enum4linux
enum4linux -a target.example.com

SMTP Enumeration

# Connect to SMTP server
telnet target.example.com 25

# Verify email addresses (if VRFY is enabled)
VRFY admin@example.com
VRFY support@example.com

# Note: VRFY is often disabled for security reasons
Service Port(s) Enumeration Focus
HTTP/HTTPS 80, 443, 8080 Web technologies, directories, files, APIs
SSH 22 Version, authentication methods, algorithms
FTP 21 Anonymous access, version, directory listing
SMB 139, 445 Shares, users, groups, policies
SMTP 25, 587 Email addresses, relay configuration
DNS 53 Zone transfers, records, version
MySQL 3306 Version, default credentials

🔐 Metadata Analysis

Documents and images often contain valuable metadata:

Information in Metadata

  • Author names and usernames
  • Software versions (potential vulnerabilities)
  • Internal network paths
  • GPS coordinates (from images)
  • Creation and modification dates
  • Company structure information
# Extract metadata from PDF (example using exiftool)
exiftool document.pdf

# Example output:
Author                : John.Smith
Creator               : Microsoft Word 2019
Create Date          : 2025:01:15 09:30:22
Modify Date          : 2025:01:15 14:22:10
Producer             : Windows 10

# Extract metadata from images
exiftool photo.jpg

# Look for:
# - GPS coordinates
# - Camera model and settings
# - Software used for editing

🌍 Network Mapping

Building a comprehensive map of the target network:

Network Discovery

# Discover live hosts on network
nmap -sn 192.168.1.0/24

# Traceroute to identify network path
traceroute target.example.com

# Identify network blocks
whois 203.0.113.0 | grep -i netrange

# Map network topology
# Using tools like:
# - Zenmap (Nmap GUI)
# - Spiceworks
# - Network Topology Mapper

Information to Gather

Category Details
IP Ranges All IP addresses owned by target
Network Topology How systems are connected, network segments
Hosting Providers Cloud services, CDNs, third-party hosting
Network Devices Routers, firewalls, load balancers
Wireless Networks SSIDs, encryption types, access points

📱 Mobile and IoT Reconnaissance

Modern targets often include mobile apps and IoT devices:

Mobile Application Analysis

  • Download and decompile mobile apps
  • Analyze API endpoints and authentication
  • Extract hardcoded credentials or API keys
  • Review permissions and data storage
  • Identify backend infrastructure

IoT Device Discovery

# Scan for common IoT ports
nmap -p 80,443,8080,8443,1883,8883,5683 target-network

📝 Documentation Best Practices

Proper documentation during reconnaissance is essential:

What to Document

  • IP Addresses and Domains: All discovered infrastructure
  • Services and Versions: Every identified service with version information
  • Employee Information: Names, roles, email addresses (for authorized testing)
  • Technologies in Use: Frameworks, languages, third-party services
  • Interesting Findings: Anything unusual or potentially vulnerable
  • Attack Surface: All potential entry points

Organization Tips

  • Use structured note-taking tools (CherryTree, OneNote, Notion)
  • Save all command outputs and screenshots
  • Maintain a timeline of reconnaissance activities
  • Create network diagrams and visual maps
  • Tag and categorize findings by severity or relevance
