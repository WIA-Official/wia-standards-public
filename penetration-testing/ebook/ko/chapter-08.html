윤리와 법적 고려사항

침투 테스트에서 반드시 준수해야 하는 윤리적 원칙과 법적 요구사항을 다룹니다. 전문가로서의 책임과 행동 규범, 그리고 관련 법규를 이해합니다.

8.1 윤리적 해킹의 원칙

윤리적 해킹(Ethical Hacking)은 조직의 보안을 개선하기 위한 목적으로 허가된 범위 내에서 수행되는 보안 테스트입니다. "윤리적"이라는 수식어가 붙는 이유는 악의적 해킹과 구분되는 명확한 원칙과 경계가 있기 때문입니다.

8.1.1 핵심 윤리 원칙

원칙 설명 실천 방법
허가 명시적 서면 동의 필수 계약서, 업무 범위 문서화
범위 준수 합의된 범위 내에서만 활동 IP 범위, 시스템 목록 확인
기밀 유지 발견 정보의 비밀 보장 NDA 체결, 보고서 보안 전송
무해성 시스템 손상 최소화 안전한 도구 사용, 백업 확인
정직 결과의 정확한 보고 오탐/미탐 구분, 증거 제시
전문성 역량 범위 내 활동 지속적 학습, 자격 유지

⚠️ 윤리적 경계

다음 행위는 어떤 상황에서도 허용되지 않습니다:

  • 허가 없는 시스템 접근
  • 범위 외 시스템 테스트
  • 발견된 취약점을 개인 이익에 활용
  • 민감 데이터 유출 또는 판매
  • 시스템의 의도적 손상
  • 고객 정보를 제3자에게 공개

8.2 법적 프레임워크

8.2.1 대한민국 관련 법규

법률 관련 조항 적용 사항
정보통신망법 제48조, 제71조 정보통신망 침입, 정보 훼손
형법 제316조, 제317조 비밀침해, 업무방해
개인정보보호법 제59조, 제71조 개인정보 수집/이용/제공
신용정보법 제32조, 제50조 신용정보 보호
전자금융거래법 제9조, 제49조 전자금융 접근매체 관리

8.2.2 국제 법규

  • 미국 CFAA: Computer Fraud and Abuse Act - 연방 컴퓨터 범죄법
  • EU GDPR: 개인정보 보호 및 처리에 관한 규정
  • 영국 CMA: Computer Misuse Act - 컴퓨터 오용 방지법
  • 부다페스트 협약: 사이버 범죄에 관한 국제 협약

8.3 계약 및 법적 문서

8.3.1 필수 계약 문서

📝 침투 테스트 계약 체크리스트

  1. 서비스 계약서 (MSA): 전체 서비스 조건
  2. 업무 범위 문서 (SOW): 테스트 범위, 기간, 산출물
  3. 테스트 허가서: 명시적 테스트 승인
  4. 비밀유지계약서 (NDA): 정보 보호 의무
  5. 교전 규칙 (ROE): 테스트 방법, 제한사항
  6. 비상 연락망: 사고 발생 시 연락처

8.3.2 교전 규칙 (Rules of Engagement)

항목 명시 내용
테스트 범위 IP 주소, 도메인, 애플리케이션 목록
제외 대상 테스트하지 않을 시스템/기능
테스트 유형 블랙박스/화이트박스/그레이박스
허용 기법 사회공학, DoS 테스트 등 포함/제외
테스트 시간 업무 시간/비업무 시간
통신 채널 진행 상황 보고 방법
중단 조건 테스트 중지가 필요한 상황
데이터 처리 수집 데이터 보관 및 폐기

8.4 책임 있는 취약점 공개

8.4.1 공개 유형

  • 책임 있는 공개 (Responsible Disclosure): 벤더에게 먼저 알리고 수정 시간 부여
  • 조율된 공개 (Coordinated Disclosure): 벤더, 연구자, 조정 기관 협력
  • 전면 공개 (Full Disclosure): 즉시 공개 (권장하지 않음)
  • 비공개 (Non-Disclosure): 계약상 공개 금지

8.4.2 책임 있는 공개 프로세스

  1. 취약점 발견: 허가된 테스트 중 취약점 식별
  2. 문서화: 상세 기술 보고서 작성
  3. 벤더 연락: 보안 담당자에게 비공개 통보
  4. 수정 기간 부여: 일반적으로 90일
  5. 협력: 패치 개발 지원 (필요시)
  6. 조율된 공개: 패치 출시 후 공개
  7. 크레딧: 발견자 인정 (선택)

8.5 데이터 보호 의무

8.5.1 테스트 중 데이터 처리

🔒 데이터 보호 원칙

  • 최소 수집: 테스트에 필요한 데이터만 수집
  • 안전한 저장: 암호화된 저장소 사용
  • 접근 제한: 테스트 팀만 접근 가능
  • 보고서 보호: 민감 정보 마스킹
  • 안전한 폐기: 테스트 완료 후 데이터 삭제

8.5.2 민감 정보 처리

데이터 유형 처리 방법
개인정보 (PII) 접근 시 즉시 보고, 마스킹 후 기록
금융 정보 접근 증거만 캡처, 상세 내용 기록 않음
의료 정보 (PHI) HIPAA 준수, 최소 접근
자격 증명 해시 저장, 테스트 후 즉시 삭제

8.6 전문가 행동 강령

8.6.1 직업 윤리

  • 고객의 이익을 최우선으로 고려
  • 정직하고 투명한 커뮤니케이션
  • 역량 범위 내에서 활동
  • 지속적인 전문성 개발
  • 동료 전문가 존중
  • 보안 커뮤니티 발전에 기여

8.6.2 이해 충돌 관리

  • 잠재적 이해 충돌 사전 공개
  • 경쟁사 정보 비밀 유지
  • 개인 이익을 위한 정보 사용 금지
  • 객관적이고 공정한 평가
