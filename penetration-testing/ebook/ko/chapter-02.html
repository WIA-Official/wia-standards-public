정찰 및 정보 수집

성공적인 침투 테스트의 첫 번째 단계인 정찰(Reconnaissance)을 다룹니다. 대상에 대한 정보를 체계적으로 수집하는 기법과 도구를 살펴봅니다.

2.1 정찰의 중요성

정찰은 침투 테스트의 가장 중요한 단계 중 하나입니다. 대상 조직, 시스템, 네트워크에 대한 정보를 수집하여 공격 표면을 파악하고 잠재적 진입점을 식별합니다. "정보는 힘이다"라는 말처럼, 철저한 정찰은 성공적인 침투 테스트의 토대가 됩니다.

🎯 정찰의 목표

  • 대상 조직의 인프라 구조 파악
  • 잠재적 공격 벡터 식별
  • 기술 스택 및 사용 시스템 파악
  • 직원 정보 및 조직 구조 이해
  • 외부 노출 자산 발견

2.2 수동 정찰 (Passive Reconnaissance)

수동 정찰은 대상 시스템과 직접 상호작용하지 않고 정보를 수집하는 방법입니다. 대상에 흔적을 남기지 않으므로 탐지될 위험이 없습니다.

2.2.1 OSINT (공개 출처 정보)

출처 수집 가능 정보 도구/방법
검색 엔진 웹사이트, 문서, 파일 Google Dorks, Bing
소셜 미디어 직원 정보, 조직 구조 LinkedIn, Twitter, Facebook
DNS 레코드 서브도메인, 메일 서버 nslookup, dig, dnsenum
WHOIS 도메인 소유자, 연락처 whois, domaintools
웹 아카이브 과거 웹사이트, 삭제된 콘텐츠 Wayback Machine
인증서 투명성 SSL 인증서, 서브도메인 crt.sh, Censys

2.2.2 Google Dorks (고급 검색 연산자)

# 파일 유형별 검색
site:example.com filetype:pdf
site:example.com filetype:xlsx

# 디렉토리 리스팅 발견
site:example.com intitle:"index of"

# 로그인 페이지 찾기
site:example.com inurl:admin
site:example.com inurl:login

# 민감한 파일 검색
site:example.com filetype:sql
site:example.com filetype:env

# 백업 파일 찾기
site:example.com filetype:bak
site:example.com ext:old

2.2.3 DNS 정보 수집

# WHOIS 조회
whois example.com

# DNS 레코드 조회
dig example.com ANY
dig example.com MX
dig example.com TXT

# 네임서버 조회
dig NS example.com

# 영역 전송 시도 (허용된 경우)
dig axfr @ns1.example.com example.com

# 서브도메인 열거
subfinder -d example.com
amass enum -d example.com

2.3 능동 정찰 (Active Reconnaissance)

능동 정찰은 대상 시스템과 직접 상호작용하여 정보를 수집합니다. 탐지될 수 있으므로 허가된 테스트에서만 수행해야 합니다.

2.3.1 네트워크 스캐닝

# Nmap 기본 스캔
nmap -sn 192.168.1.0/24          # 호스트 발견 (Ping 스캔)
nmap -sS 192.168.1.100           # SYN 스캔 (반개방)
nmap -sT 192.168.1.100           # TCP 연결 스캔
nmap -sU 192.168.1.100           # UDP 스캔

# 버전 및 서비스 탐지
nmap -sV 192.168.1.100           # 서비스 버전 탐지
nmap -sV -sC 192.168.1.100       # 기본 스크립트 + 버전

# OS 탐지
nmap -O 192.168.1.100

# 전체 스캔
nmap -A 192.168.1.100            # 종합 스캔 (-sV -sC -O --traceroute)

# 출력 저장
nmap -sV -sC -oA results 192.168.1.100

2.3.2 포트 스캔 결과 분석

포트 서비스 보안 고려사항
21 FTP 익명 로그인, 평문 전송
22 SSH 브루트포스, 취약한 키

3389 RDP 브루트포스, BlueKeep

2.3.3 웹 애플리케이션 정찰

# 웹 서버 핑거프린팅
whatweb https://example.com
curl -I https://example.com

# 기술 스택 식별
wappalyzer https://example.com

# 디렉토리 및 파일 열거
gobuster dir -u https://example.com -w /usr/share/wordlists/dirb/common.txt
ffuf -u https://example.com/FUZZ -w wordlist.txt

# 서브도메인 브루트포스
gobuster dns -d example.com -w subdomains.txt

# 웹 크롤링
gospider -s https://example.com -d 2

# robots.txt, sitemap.xml 확인
curl https://example.com/robots.txt
curl https://example.com/sitemap.xml

2.5 사회공학 정찰

사회공학 정찰은 기술적 방법 외에 인적 요소에 대한 정보를 수집합니다. 직원 정보, 조직 구조, 업무 프로세스 등을 파악합니다.

2.5.1 수집 대상 정보

  • 직원 정보: 이름, 직책, 이메일 주소, 전화번호
  • 조직 구조: 부서, 보고 체계, 의사결정권자
  • 업무 환경: 근무 시간, 위치, 원격 근무 정책
  • 기술 환경: 사용 소프트웨어, 이메일 클라이언트
  • 보안 인식: 보안 교육 수준, 정책 준수

2.5.2 LinkedIn 정찰

# LinkedIn 검색 연산자

# 수집 가능 정보

- 채용 공고 (기술 요구사항)
- 전현직 직원 (조직 변화)
- 연결된 벤더/파트너사

📋 정찰 체크리스트

  1. 도메인 및 IP 범위 식별
  2. 서브도메인 열거
  3. DNS 레코드 수집
  4. WHOIS 정보 조회
  5. 웹사이트 기술 스택 분석
  6. SSL 인증서 분석
  7. 직원 및 조직 정보 수집
  8. 소셜 미디어 분석
  9. 웹 아카이브 확인
  10. 공개 문서/파일 검색

2.6 정찰 결과 문서화

수집된 정보를 체계적으로 문서화하여 다음 단계에서 활용할 수 있도록 합니다. 정찰 보고서에는 다음 내용이 포함되어야 합니다:

섹션 내용
대상 개요 조직명, 산업, 규모, 위치
네트워크 자산 IP 범위, 도메인, 서브도메인, 호스트
기술 스택 웹 서버, 프레임워크, CMS, 데이터베이스
노출된 서비스 열린 포트, 서비스 버전, 잠재적 취약점
인적 정보 핵심 인물, 이메일 형식, 조직도
잠재적 공격 벡터 초기 진입점, 우선순위
