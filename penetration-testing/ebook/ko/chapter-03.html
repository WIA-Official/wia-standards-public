제3장

취약점 분석

정찰 단계에서 수집된 정보를 바탕으로 시스템과 애플리케이션의 취약점을 체계적으로 식별하고 분석하는 방법을 다룹니다. 자동화 도구와 수동 분석 기법을 결합하여 포괄적인 취약점 평가를 수행합니다.

3.1 취약점 분석 개요

취약점 분석(Vulnerability Analysis)은 시스템, 네트워크, 애플리케이션에 존재하는 보안 약점을 식별하고 분류하는 과정입니다. 이 단계는 정찰에서 수집된 정보를 활용하여 악용 가능한 취약점을 찾아내고, 다음 단계인 익스플로잇의 기초를 마련합니다.

3.1.1 취약점 분석 목표

  • 시스템 및 애플리케이션의 보안 취약점 식별
  • 취약점의 심각도와 악용 가능성 평가
  • 잠재적 공격 벡터와 영향 범위 파악
  • 우선순위에 따른 취약점 분류
  • 오탐(False Positive) 제거 및 검증

3.2.2 Nmap 취약점 스크립트

# NSE(Nmap Scripting Engine) 취약점 스캔
nmap --script vuln 192.168.1.100

# 특정 취약점 검사
nmap --script smb-vuln-ms17-010 192.168.1.100    # EternalBlue
nmap --script ssl-heartbleed 192.168.1.100       # Heartbleed
nmap --script http-shellshock 192.168.1.100      # Shellshock

# 인증 관련 취약점
nmap --script ftp-anon 192.168.1.100             # 익명 FTP
nmap --script ssh-brute 192.168.1.100            # SSH 브루트포스
nmap --script mysql-empty-password 192.168.1.100 # MySQL 빈 패스워드

# 모든 취약점 스크립트 실행
nmap --script vuln,exploit 192.168.1.100

# 특정 카테고리 스캔
nmap --script "safe and vuln" 192.168.1.100

3.3.2 웹 취약점 스캐너

# OWASP ZAP 자동 스캔
zap-cli quick-scan --self-contained --start-options '-config api.disablekey=true' https://example.com

# Nikto 웹 서버 스캔
nikto -h https://example.com
nikto -h https://example.com -ssl -Tuning 123bde

# WPScan (WordPress)
wpscan --url https://example.com --enumerate u,p,t
wpscan --url https://example.com --api-token YOUR_TOKEN

# SQLMap (SQL 인젝션)
sqlmap -u "https://example.com/page.php?id=1" --dbs
sqlmap -u "https://example.com/page.php?id=1" --batch --risk=3 --level=5

# XSStrike (XSS)
xsstrike -u "https://example.com/search?q=test"

3.4 수동 취약점 분석

3.4.1 소스 코드 리뷰

화이트박스 테스트에서는 소스 코드를 직접 분석하여 취약점을 발견합니다. 다음은 주요 검토 포인트입니다:

취약점 유형 검색 패턴 (예시) 언어
SQL 인젝션 query(, execute(, cursor. Python, PHP
명령어 인젝션 exec(, system(, popen( Python, PHP
XSS innerHTML, document.write( JavaScript
하드코딩된 비밀 password=, api_key=, secret= 모든 언어
취약한 암호화 MD5(, SHA1(, DES 모든 언어

3.4.2 Burp Suite 수동 테스트

🔧 Burp Suite 테스트 워크플로우

  1. 프록시 설정: 브라우저와 Burp 연결
  2. 사이트 탐색: 모든 기능을 수동으로 탐색
  3. 사이트맵 분석: 발견된 엔드포인트 검토
  4. 요청 수정: Repeater로 매개변수 조작
  5. 취약점 테스트: Intruder로 페이로드 테스트
  6. 결과 분석: 응답 비교 및 취약점 확인

3.5 취약점 분류 및 우선순위

3.5.1 CVSS (공통 취약점 점수 시스템)

CVSS는 취약점의 심각도를 정량화하는 산업 표준 방법입니다. 기본 점수(Base Score), 시간 점수(Temporal Score), 환경 점수(Environmental Score)로 구성됩니다.

점수 범위 심각도 대응 시간
9.0 - 10.0 Critical 즉시 (24시간 이내)
7.0 - 8.9 High 7일 이내
4.0 - 6.9 Medium 30일 이내
0.1 - 3.9 Low 분기 내

3.5.2 취약점 검증

자동화 스캐너의 결과에는 오탐(False Positive)이 포함될 수 있으므로, 발견된 취약점을 수동으로 검증해야 합니다:

  • 재현성 확인: 동일한 조건에서 취약점 재현
  • 영향 범위 파악: 악용 시 접근 가능한 데이터/기능
  • 악용 복잡도: 공격에 필요한 조건과 기술 수준
  • 인증 요구사항: 인증 없이/인증 후 악용 가능 여부
  • 사용자 상호작용: 피해자의 행동 필요 여부

3.6 취약점 분석 보고서

📝 취약점 보고서 구성

  • 취약점 ID: 고유 식별자
  • 제목: 간결한 취약점 설명
  • 영향받는 자산: 시스템, URL, 구성요소
  • 심각도: CVSS 점수 및 등급
  • 설명: 취약점 상세 설명
  • 재현 단계: 취약점 악용 방법
  • 증거: 스크린샷, 로그, PoC 코드
  • 영향: 악용 시 잠재적 피해
  • 권고사항: 개선 조치
  • 참조: CVE, CWE, 관련 문서
