# WIA-SEC-019: Penetration Testing - Glossary

**Version:** 1.0
**Status:** DRAFT
**Last Updated:** 2025-12-25
**Standard ID:** WIA-SEC-019

---

## A

**ABAC (Attribute-Based Access Control)**
Access control model that makes authorization decisions based on attributes of subjects, resources, actions, and environmental context.

**Access Control**
Security mechanism that regulates who or what can view or use resources in a computing environment.

**Active Directory (AD)**
Microsoft's directory service for Windows domain networks, providing authentication and authorization services.

**Advanced Persistent Threat (APT)**
Prolonged and targeted cyberattack in which an intruder gains access to a network and remains undetected for an extended period.

**API (Application Programming Interface)**
Set of protocols and tools for building software applications, defining how components should interact.

**Asset**
Any data, device, or other component of an organization's environment that has value.

**Attack Surface**
Sum of all possible points (attack vectors) where an unauthorized user can attempt to enter data or extract data from an environment.

**Attack Vector**
Path or means by which an attacker can gain unauthorized access to a system or network.

**Authentication**
Process of verifying the identity of a user, system, or application.

**Authorization**
Process of determining whether an authenticated entity has permission to access a specific resource or perform an action.

---

## B

**Backdoor**
Method of bypassing normal authentication or encryption to gain unauthorized access to a system.

**Banner Grabbing**
Technique used to gather information about a remote system by analyzing response banners from network services.

**BAS (Breach and Attack Simulation)**
Security testing methodology that simulates real-world attacks to validate security controls.

**BloodHound**
Active Directory reconnaissance tool for identifying attack paths and privilege escalation opportunities.

**Blue Team**
Defensive security team responsible for protecting systems and detecting/responding to attacks.

**Brute Force Attack**
Trial-and-error method used to obtain credentials by systematically trying all possible combinations.

**Bug Bounty**
Program offering rewards to individuals who identify and report software vulnerabilities.

**Burp Suite**
Integrated platform for web application security testing, including proxy, scanner, and various testing tools.

---

## C

**C2 (Command and Control)**
Server or infrastructure used by attackers to communicate with and control compromised systems.

**CAPEC (Common Attack Pattern Enumeration and Classification)**
Comprehensive dictionary of known attack patterns used by adversaries.

**CEH (Certified Ethical Hacker)**
Professional certification for penetration testers demonstrating knowledge of ethical hacking techniques.

**CREST**
International not-for-profit accreditation body representing the technical information security industry.

**CSRF (Cross-Site Request Forgery)**
Attack that forces an authenticated user to execute unwanted actions on a web application.

**CVE (Common Vulnerabilities and Exposures)**
Dictionary of publicly known information security vulnerabilities and exposures.

**CVSS (Common Vulnerability Scoring System)**
Standardized framework for rating the severity of security vulnerabilities.

**CWE (Common Weakness Enumeration)**
Community-developed list of software and hardware weakness types.

---

## D

**DAST (Dynamic Application Security Testing)**
Security testing methodology that analyzes running applications to find vulnerabilities.

**Defense in Depth**
Information security strategy integrating people, technology, and operations to establish multiple layers of defense.

**Denial of Service (DoS)**
Attack that makes a machine or network resource unavailable to its intended users.

**DID (Decentralized Identifier)**
New type of identifier that enables verifiable, decentralized digital identity.

**DNS (Domain Name System)**
Hierarchical naming system that translates human-readable domain names to IP addresses.

**Domain Fronting**
Technique used to obfuscate the true destination of internet traffic by hiding it behind legitimate domain names.

---

## E

**EDR (Endpoint Detection and Response)**
Security solution that monitors endpoint devices for suspicious activity and provides incident response capabilities.

**Enumeration**
Process of extracting information about a target system, such as user names, network resources, and services.

**Escalation**
Process of increasing privileges or access levels within a system.

**Ethical Hacking**
Authorized practice of bypassing system security to identify potential vulnerabilities.

**Exploit**
Code or technique that takes advantage of a vulnerability to gain unauthorized access or cause unintended behavior.

**Exploitation**
Act of taking advantage of a vulnerability to achieve an attacker's objectives.

**Exposure**
Condition where sensitive information or functionality is accessible to unauthorized parties.

---

## F

**Firewall**
Network security system that monitors and controls incoming and outgoing network traffic based on security rules.

**Footprinting**
Technique used to gather information about a target system or network before launching an attack.

**Fuzzing**
Automated testing technique that provides invalid, unexpected, or random data as input to a system.

---

## G

**GPEN (GIAC Penetration Tester)**
Professional certification validating knowledge of penetration testing methodologies and tools.

**Gray Box Testing**
Security testing approach where the tester has partial knowledge of the system's internals.

---

## H

**Hash**
Fixed-size string of characters produced by a hash function from input data of any size.

**Hardening**
Process of securing a system by reducing its attack surface and eliminating potential vulnerabilities.

**Honeypot**
Decoy system designed to attract and detect attackers.

**HTTP (Hypertext Transfer Protocol)**
Application protocol for distributed, collaborative, hypermedia information systems.

**HTTPS (HTTP Secure)**
Extension of HTTP that uses encryption for secure communication over a network.

---

## I

**IaC (Infrastructure as Code)**
Managing and provisioning computer data centers through machine-readable definition files.

**IAM (Identity and Access Management)**
Framework of policies and technologies ensuring appropriate access to technology resources.

**IDOR (Insecure Direct Object Reference)**
Vulnerability where application exposes reference to internal implementation objects.

**IDS (Intrusion Detection System)**
Device or software that monitors network traffic for suspicious activity.

**Injection**
Security vulnerability that allows an attacker to inject malicious code into an application.

**IPS (Intrusion Prevention System)**
Network security tool that monitors traffic for malicious activity and can take action to prevent it.

**ISO 27001**
International standard for information security management systems.

---

## J

**JWT (JSON Web Token)**
Compact, URL-safe means of representing claims to be transferred between two parties.

---

## K

**Kerberos**
Network authentication protocol designed to provide strong authentication for client/server applications.

**Kerberoasting**
Technique for extracting service account credential hashes from Active Directory.

**Keylogger**
Software or hardware device that records keystrokes made on a computer.

**Kubernetes**
Open-source container orchestration platform for automating deployment, scaling, and management.

---

## L

**Lateral Movement**
Techniques adversaries use to move through a network seeking key assets and data.

**LDAP (Lightweight Directory Access Protocol)**
Protocol for accessing and maintaining distributed directory information services.

**Living off the Land (LOTL)**
Attack technique using legitimate system tools and features to conduct malicious activities.

**LSASS (Local Security Authority Subsystem Service)**
Windows process responsible for enforcing security policy, handling password changes, and creating access tokens.

---

## M

**Malware**
Software designed to disrupt, damage, or gain unauthorized access to a computer system.

**Man-in-the-Middle (MITM)**
Attack where the attacker secretly intercepts and possibly alters communications between two parties.

**Metasploit**
Penetration testing framework used for developing, testing, and executing exploits.

**Mimikatz**
Post-exploitation tool for extracting credentials from Windows systems.

**MITRE ATT&CK**
Globally-accessible knowledge base of adversary tactics and techniques based on real-world observations.

**MFA (Multi-Factor Authentication)**
Authentication method requiring two or more verification factors to gain access.

---

## N

**Network Segmentation**
Practice of dividing a network into smaller segments to improve security and performance.

**NIST (National Institute of Standards and Technology)**
U.S. government agency that develops technology, metrics, and standards.

**Nmap**
Open-source network scanning tool used for network discovery and security auditing.

**NTLM (NT LAN Manager)**
Suite of Microsoft security protocols providing authentication, integrity, and confidentiality.

**NVD (National Vulnerability Database)**
U.S. government repository of standards-based vulnerability management data.

---

## O

**OAuth**
Open standard for access delegation, commonly used for token-based authentication.

**OSCP (Offensive Security Certified Professional)**
Hands-on penetration testing certification requiring demonstration of practical skills.

**OSINT (Open Source Intelligence)**
Collection and analysis of publicly available information.

**OSSTMM (Open Source Security Testing Methodology Manual)**
Peer-reviewed methodology for security testing and analysis.

**OWASP (Open Web Application Security Project)**
Online community producing freely-available articles, methodologies, and tools for web application security.

**OWASP Top 10**
Standard awareness document representing broad consensus about critical web application security risks.

---

## P

**Passthe-Hash (PtH)**
Attack technique allowing authentication to a remote system using password hash instead of plaintext.

**Patch**
Software update designed to fix vulnerabilities or improve functionality.

**PCI DSS (Payment Card Industry Data Security Standard)**
Information security standard for organizations handling credit cards.

**Penetration Testing**
Authorized simulated cyberattack on a system to evaluate its security.

**Persistence**
Techniques adversaries use to maintain access to systems across restarts and credential changes.

**Phishing**
Social engineering attack attempting to trick users into revealing sensitive information.

**PNPT (Practical Network Penetration Tester)**
Certification demonstrating practical penetration testing skills in network environments.

**Post-Exploitation**
Actions taken after successfully compromising a system to achieve broader objectives.

**Privilege Escalation**
Act of exploiting a vulnerability to gain elevated access to resources.

**Proof of Concept (PoC)**
Demonstration that a vulnerability can be exploited.

**PTES (Penetration Testing Execution Standard)**
Standard designed to provide comprehensive framework for penetration testing.

**Purple Team**
Collaborative approach combining red team (offensive) and blue team (defensive) activities.

---

## Q

**QR Code**
Two-dimensional barcode readable by smartphones and QR scanners.

---

## R

**Ransomware**
Malware that encrypts files and demands payment for decryption.

**RBAC (Role-Based Access Control)**
Access control model that assigns permissions to roles rather than individual users.

**Reconnaissance**
Preliminary phase of an attack where information about the target is gathered.

**Red Team**
Offensive security team that simulates real-world attacks to test defenses.

**Remote Code Execution (RCE)**
Vulnerability allowing execution of arbitrary code on a remote system.

**Responsible Disclosure**
Practice of reporting vulnerabilities to vendors before public disclosure.

**Reverse Shell**
Shell session established on a connection initiated from a remote machine.

**Risk Assessment**
Process of identifying, analyzing, and evaluating risk.

**RoE (Rules of Engagement)**
Guidelines defining the scope and constraints of a penetration test.

---

## S

**SAML (Security Assertion Markup Language)**
XML-based standard for exchanging authentication and authorization data.

**SAST (Static Application Security Testing)**
Analysis of source code to find security vulnerabilities without executing the program.

**SCA (Software Composition Analysis)**
Process of identifying open-source components and their known vulnerabilities.

**Scope Creep**
Uncontrolled expansion beyond the originally defined testing boundaries.

**SIEM (Security Information and Event Management)**
Solution providing real-time analysis of security alerts generated by applications and network hardware.

**Social Engineering**
Manipulation technique exploiting human psychology to gain unauthorized access or information.

**SOC (Security Operations Center)**
Centralized unit dealing with security issues on an organizational and technical level.

**SQL Injection (SQLi)**
Code injection technique exploiting vulnerabilities in database layer of an application.

**SSH (Secure Shell)**
Cryptographic network protocol for secure data communication and remote command execution.

**SSL/TLS (Secure Sockets Layer/Transport Layer Security)**
Cryptographic protocols providing secure communications over a network.

---

## T

**Tailgating**
Physical security breach where unauthorized person follows authorized person into a restricted area.

**Threat Actor**
Entity responsible for security incidents.

**Threat Intelligence**
Evidence-based knowledge about existing or emerging threats.

**Threat Modeling**
Process of identifying and evaluating potential threats to a system.

**Token**
Piece of data used for authentication or authorization.

**TTP (Tactics, Techniques, and Procedures)**
Behavior patterns of threat actors.

---

## U

**UAC (User Account Control)**
Windows security feature preventing unauthorized changes to the operating system.

---

## V

**VC (Verifiable Credential)**
Cryptographically secure credential that can be independently verified.

**Vulnerability**
Weakness in a system that can be exploited to cause harm.

**Vulnerability Assessment**
Systematic review of security weaknesses in an information system.

**Vulnerability Scanner**
Automated tool that identifies security vulnerabilities in systems and applications.

---

## W

**WAF (Web Application Firewall)**
Security solution filtering and monitoring HTTP traffic to and from web applications.

**Waterhole Attack**
Security exploit targeting specific group by compromising websites they frequently visit.

**White Box Testing**
Security testing with full knowledge of the system's internals.

**Wireshark**
Network protocol analyzer used to capture and analyze network traffic.

**WMI (Windows Management Instrumentation)**
Infrastructure for management data and operations on Windows systems.

---

## X

**XSS (Cross-Site Scripting)**
Security vulnerability allowing attackers to inject malicious scripts into web pages.

**XXE (XML External Entity)**
Attack against applications parsing XML input.

---

## Y

**YARA**
Tool for identifying and classifying malware samples based on textual or binary patterns.

---

## Z

**Zero-Day**
Vulnerability unknown to those who should be interested in its mitigation.

**Zero Trust**
Security model that assumes no user or system should be trusted by default.

**ZAP (Zed Attack Proxy)**
OWASP's open-source web application security scanner.

**Zombie**
Compromised computer controlled remotely as part of a botnet.
