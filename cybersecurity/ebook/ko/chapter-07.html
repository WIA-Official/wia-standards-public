7.1 SOC 구조 및 역할

보안 운영 센터(SOC)는 조직 및 기술 수준에서 보안 문제를 다루는 중앙 집중식 부서입니다. SOC 팀은 사이버 위협의 모니터링, 탐지, 조사, 대응을 담당합니다.

역할 레벨 책임
SOC 분석가 L1 Tier 1 경보 분류, 초기 조사, 에스컬레이션
SOC 분석가 L2 Tier 2 심층 조사, 사고 처리, 상관관계
SOC 분석가 L3 Tier 3 고급 분석, 위협 헌팅, 악성코드 분석
탐지 엔지니어 엔지니어링 탐지 규칙 개발, 튜닝, 자동화
위협 정보 분석가 인텔리전스 CTI 통합, 행위자 추적, 보고
SOC 매니저 관리 팀 리더십, 지표, 프로세스 개선

7.3 경보 분류 프로세스

단계 활동 목표 시간
1. 수신 큐에서 경보 수신 즉시
2. 초기 검토 경보 세부사항 빠른 평가 < 5분
3. 컨텍스트 수집 추가 데이터 소스로 보강 < 10분
4. 분류 True positive, False positive, 에스컬레이션 필요 < 15분
5. 조치 종료, 에스컬레이션, 또는 사고 개시 < 30분

7.4 사고 대응 워크플로우

NIST IR 수명 주기

사고 대응은 NIST 수명 주기를 따릅니다: 준비, 탐지 및 분석, 봉쇄, 제거 및 복구, 사후 활동.

7.5 SOC 지표 및 KPI

지표 설명 목표
MTTD 평균 탐지 시간 < 1시간
MTTR 평균 대응 시간 < 4시간
MTTC 평균 봉쇄 시간 < 24시간
False Positive Rate FP인 경보의 비율 < 30%
경보 볼륨 분석가당 일일 경보 < 50
커버리지 탐지된 ATT&CK 기법 비율 > 70%

7.6 SOC 성숙도 모델

  • 레벨 1 - 초기: 임시방편 프로세스, 반응적만
  • 레벨 2 - 관리: 정의된 프로세스, 기본 모니터링
  • 레벨 3 - 정의: 문서화된 절차, 일관된 실행
  • 레벨 4 - 정량적 관리: 지표 주도, 지속적 개선
  • 레벨 5 - 최적화: 선제적 헌팅, 자동화, 혁신

7.7 장 요약

핵심 내용

  • SOC는 명확한 책임이 있는 계층화된 분석가 구조 필요
  • SIEM, SOAR, EDR이 핵심 기술 스택 형성
  • 경보 분류는 30분 내 완료되어야 함
  • 사고 심각도가 대응 시간 및 에스컬레이션 결정
  • KPI가 SOC 효과성 측정 및 개선 안내
