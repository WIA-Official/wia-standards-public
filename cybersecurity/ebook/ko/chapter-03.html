3.1 제로 트러스트 아키텍처

3.1.1 제로 트러스트 구현

축 전통적 제로 트러스트
ID 사용자명/비밀번호 강력한 MFA, 지속적 검증
디바이스 회사 관리 장치만 모든 장치 검증, 상태 확인
네트워크 경계 방화벽 마이크로분할, 암호화
애플리케이션 VPN 접근 앱 레벨 인증, 프록시 접근
데이터 네트워크 경계 분류, 암호화, DLP

3.2 심층 방어

심층 방어는 정보 기술 시스템 전체에 여러 계층의 보안 통제를 배치하는 보안 전략입니다. 이 접근법은 한 계층이 실패해도 다른 계층이 시스템을 보호하도록 보장합니다.

계층 통제 목적
경계 방화벽, IDS/IPS, DDoS 완화 경계에서 알려진 위협 필터링
네트워크 분할, 암호화, 모니터링 횡적 이동 제한
엔드포인트 EDR, 백신, 앱 화이트리스팅 워크스테이션 및 서버 보호
애플리케이션 보안 코딩, WAF, 취약점 관리 애플리케이션 익스플로잇 방지
데이터 암호화, DLP, 접근 통제 민감 정보 보호
ID MFA, PAM, 제로 트러스트 모든 접근 요청 검증

3.3 설계에 의한 보안

원칙: 처음부터 보안 구축

보안은 나중에 추가하는 것이 아니라 초기 설계 단계부터 시스템에 통합되어야 합니다. 이 접근법은 보안을 나중에 추가하는 것보다 더 효과적이고 비용이 적게 듭니다.

3.3.1 보안 설계 원칙

  • 공격 표면 최소화: 노출된 구성요소 및 서비스 축소
  • 안전한 기본값: 기본 구성이 안전해야 함
  • 심층 방어: 다중 보호 계층
  • 안전한 실패: 실패가 보안을 손상시키지 않아야 함
  • 직무 분리: 중요 작업에 여러 행위자 필요
  • 보안 단순화: 복잡성은 보안의 적

3.6 장 요약

핵심 내용

  • 제로 트러스트: 신뢰하지 말고, 모든 접근 요청 항상 검증
  • 심층 방어는 다중 보안 계층 제공
  • 설계에 의한 보안은 처음부터 보안 구축
  • 설계에 의한 프라이버시는 프라이버시 보호 내장
  • 핵심 원칙이 모든 보안 결정 안내
