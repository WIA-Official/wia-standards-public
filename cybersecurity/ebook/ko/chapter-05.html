5.1 현대 암호화 표준

암호화는 데이터 보호의 기반을 형성하여 기밀성, 무결성, 인증을 제공합니다. WIA-SEC-015는 현재 보안 요구사항을 충족하는 강력한 암호화 표준을 의무화합니다.

유형 알고리즘 키 크기 사용 사례
대칭 AES-256-GCM 256비트 데이터 암호화
비대칭 RSA 4096비트 키 교환, 서명
비대칭 ECDSA (P-384) 384비트 디지털 서명
해시 SHA-256/512 256/512비트 무결성 검증
해시 BLAKE3 256비트 고성능 해싱
KDF Argon2id 가변 비밀번호 해싱

5.3 키 관리

5.3.1 키 수명 주기

단계 활동 통제
생성 CSPRNG로 키 생성 HSM, 엔트로피 소스
배포 안전한 키 교환 키 래핑, 보안 채널
저장 저장 키 보호 HSM, 키 볼트, 암호화
사용 접근 통제, 감사 역할 분리, 로깅
순환 정기적 키 교체 자동 순환, 버전 관리
파기 안전한 키 삭제 암호화 삭제

5.3.2 키 관리 솔루션

  • HSM(하드웨어 보안 모듈): 물리적 변조 방지 장치
  • 클라우드 키 관리: AWS KMS, Azure Key Vault, GCP Cloud KMS
  • 시크릿 관리: HashiCorp Vault, CyberArk
  • 인증서 관리: PKI, 인증서 수명 주기 자동화

5.5 데이터 손실 방지 (DLP)

DLP 유형 위치 기능
네트워크 DLP 네트워크 경계 이메일, 웹, 파일 전송
엔드포인트 DLP 워크스테이션, 노트북 USB, 인쇄, 클립보드
클라우드 DLP 클라우드 서비스 SaaS, IaaS, 스토리지
발견 DLP 데이터 저장소 분류, 인벤토리

5.6 장 요약

핵심 내용

  • AES-256-GCM이 대칭 암호화 표준
  • 포스트 양자 암호화 전환을 지금 시작해야 함
  • 키 수명 주기 관리가 키 보안 보장
  • TLS 1.3이 전송 중 데이터 보호
  • DLP가 무단 데이터 공개 방지
