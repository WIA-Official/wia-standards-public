6.1 인증 메커니즘

인증은 접근 권한을 부여하기 전에 사용자, 장치, 시스템의 ID를 검증합니다. WIA-SEC-015는 단순한 비밀번호를 넘어서는 강력한 인증을 요구합니다.

요소 유형 설명 예시
지식 알고 있는 것 비밀번호, PIN, 보안 질문
소유 가지고 있는 것 보안 키, 스마트 카드, 휴대폰
고유 당신인 것 지문, 얼굴, 홍채
위치 어디에 있는지 GPS, IP 위치 정보
행동 어떻게 행동하는지 타이핑 패턴, 장치 사용

6.3 SSO (Single Sign-On)

프로토콜 사용 사례 특징
SAML 2.0 엔터프라이즈 웹 앱 XML 기반, 브라우저 리다이렉트
OAuth 2.0 API 권한 부여 토큰 기반, 위임된 접근
OpenID Connect 현대 SSO OAuth 2.0 + ID 계층
Kerberos 온프레미스 티켓 기반, AD 통합

6.3.1 SSO 이점

  • 사용자 경험: 모든 애플리케이션에 단일 로그인
  • 보안: 중앙 집중식 인증, 일관된 정책
  • 관리: 단일 사용자 관리 지점
  • 규정 준수: 통합 감사 추적

6.5 권한 접근 관리 (PAM)

PAM의 중요성

권한 계정은 왕국의 열쇠입니다. 침해된 권한 자격 증명은 보안 침해의 80%에 관련됩니다. PAM은 이러한 중요 계정을 보호하기 위한 통제를 제공합니다.

PAM 기능 설명 구현
자격 증명 볼트 권한 자격 증명의 안전한 저장 암호화 볼트, 순환
세션 관리 권한 세션 제어 및 모니터링 녹화, 실시간 모니터링
Just-In-Time 접근 임시 권한 접근 시간 제한 권한 상승
최소 권한 필요한 최소 접근 역할 분리, 제한

6.6 ID 수명 주기

  • 프로비저닝: 적절한 접근 권한으로 계정 생성
  • 관리: 정확한 ID 속성 유지
  • 접근 검토: 정기적인 접근 권한 인증
  • 디프로비저닝: 퇴사 시 신속한 접근 제거

6.7 장 요약

핵심 내용

  • 강력한 인증은 다중 요소 필요
  • FIDO2/WebAuthn이 피싱 방지 MFA 제공
  • SSO가 보안과 사용자 경험 개선
  • RBAC와 ABAC가 유연한 접근 통제 제공
  • PAM이 중요 권한 계정 보호
