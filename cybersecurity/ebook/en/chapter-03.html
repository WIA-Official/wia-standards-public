3.1 Zero Trust Architecture

Zero Trust is a security concept centered on the belief that organizations should not automatically trust anything inside or outside its perimeters and instead must verify anything and everything trying to connect to its systems before granting access.

Data Network boundary Classified, encrypted, DLP

3.2 Defense in Depth

Defense in Depth is a security strategy that employs multiple layers of security controls throughout an information technology system. This approach ensures that if one layer fails, others remain to protect the system.

Layer Controls Purpose
Perimeter Firewalls, IDS/IPS, DDoS mitigation Filter known threats at boundary
Network Segmentation, encryption, monitoring Limit lateral movement
Endpoint EDR, antivirus, app whitelisting Protect workstations and servers
Application Secure coding, WAF, vulnerability mgmt Prevent application exploits
Data Encryption, DLP, access control Protect sensitive information
Identity MFA, PAM, Zero Trust Verify every access request

3.3 Secure by Design

Principle: Build Security From the Start

Security should be integrated into systems from the initial design phase, not added as an afterthought. This approach is more effective and less costly than retrofitting security.

3.3.1 Secure Design Principles

  • Minimize Attack Surface: Reduce exposed components and services
  • Secure Defaults: Default configurations should be secure
  • Defense in Depth: Multiple protective layers
  • Fail Securely: Failures should not compromise security
  • Separation of Duties: Critical tasks require multiple actors
  • Keep Security Simple: Complexity is the enemy of security
