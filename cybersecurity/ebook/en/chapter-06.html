6.1 Authentication Mechanisms

Authentication verifies the identity of users, devices, and systems before granting access. WIA-SEC-015 requires strong authentication that goes beyond simple passwords.

Factor Type Description Examples
Knowledge Something you know Passwords, PINs, security questions
Possession Something you have Security keys, smart cards, phones
Inherence Something you are Fingerprints, face, iris
Location Where you are GPS, IP geolocation
Behavior How you act Typing patterns, device usage

6.3 Single Sign-On (SSO)

Protocol Use Case Features
SAML 2.0 Enterprise web apps XML-based, browser redirect
OAuth 2.0 API authorization Token-based, delegated access
OpenID Connect Modern SSO OAuth 2.0 + identity layer
Kerberos On-premises Ticket-based, AD integration

6.3.1 SSO Benefits

  • User Experience: Single login for all applications
  • Security: Centralized authentication, consistent policies
  • Administration: Single point of user management
  • Compliance: Unified audit trail

6.5 Privileged Access Management (PAM)

The Importance of PAM

Privileged accounts are the keys to the kingdom. Compromised privileged credentials are involved in 80% of security breaches. PAM provides controls to protect these critical accounts.

PAM Capability Description Implementation
Credential Vault Secure storage of privileged credentials Encrypted vault, rotation
Session Management Control and monitor privileged sessions Recording, live monitoring
Just-In-Time Access Temporary privileged access Time-limited elevation
Least Privilege Minimum required access Role separation, restrictions

6.6 Identity Lifecycle

  • Provisioning: Create accounts with appropriate access
  • Management: Maintain accurate identity attributes
  • Access Reviews: Regular certification of access rights
  • Deprovisioning: Timely removal of access upon termination

6.7 Chapter Summary

Key Takeaways

  • Strong authentication requires multiple factors
  • FIDO2/WebAuthn provides phishing-resistant MFA
  • SSO improves security and user experience
  • RBAC and ABAC provide flexible access control
  • PAM protects critical privileged accounts
