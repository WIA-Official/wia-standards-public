4.2 Behavioral Analysis

Analysis Type Baseline Anomaly Indicators
User Behavior (UEBA) Login times, access patterns Off-hours access, unusual locations
Entity Behavior Device activity, data flows New destinations, data volume spikes
Process Behavior Parent-child relationships Unusual spawning, injection
Network Behavior Traffic patterns, protocols Beaconing, tunneling, exfiltration

4.3 MITRE ATT&CK Framework

Common Language for Threats

MITRE ATT&CK provides a globally-accessible knowledge base of adversary tactics and techniques based on real-world observations. WIA-SEC-015 maps all detection rules to ATT&CK techniques.

Tactic Description Example Techniques
Initial Access Getting into the network Phishing, Valid Accounts
Execution Running malicious code PowerShell, Command Line
Persistence Maintaining access Registry Run Keys, Scheduled Tasks
Privilege Escalation Getting higher permissions Exploitation, Token Manipulation
Defense Evasion Avoiding detection Obfuscation, Log Clearing
Credential Access Stealing credentials Keylogging, Credential Dumping
Lateral Movement Moving through network RDP, SMB, Pass-the-Hash
Exfiltration Stealing data Encrypted Channels, C2

4.5 Threat Hunting

4.5.1 Hunting Methodologies

  • Hypothesis-Driven: Start with threat actor TTPs
  • Intelligence-Driven: Hunt for known IOCs
  • Anomaly-Driven: Investigate statistical outliers
  • Entity-Driven: Deep dive on suspicious entities

4.5.2 Hunting Process

Phase Activities Outputs
Hypothesis Develop hunting hypothesis from intel Hunt plan, queries
Investigation Search data, analyze patterns Findings, artifacts
Validation Confirm or refute hypothesis Validated findings
Response Escalate, contain if needed Incidents, IOCs
Improvement Create detection rules New detections
