2.1 NIST CSF Overview

The NIST Cybersecurity Framework (CSF) provides a policy framework of computer security guidance for organizations to assess and improve their ability to prevent, detect, and respond to cyberattacks. WIA-SEC-015 fully implements all five core functions with practical controls and measurable outcomes.

2.2 Identify Function

ID: Identify

Develop organizational understanding to manage cybersecurity risk to systems, people, assets, data, and capabilities.

Category Description WIA-SEC-015 Implementation
ID.AM Asset Management Automated asset discovery and inventory
ID.BE Business Environment Business context and criticality mapping
ID.GV Governance Policy framework and accountability
ID.RA Risk Assessment Continuous risk scoring and prioritization
ID.RM Risk Management Strategy Risk appetite and tolerance framework
ID.SC Supply Chain Risk Vendor security assessment

2.3 Protect Function

PR: Protect

Develop and implement appropriate safeguards to ensure delivery of critical services.

Category Description WIA-SEC-015 Implementation
PR.AC Access Control Zero Trust, MFA, PAM
PR.AT Awareness Training Security awareness program
PR.DS Data Security Encryption, DLP, classification
PR.IP Protective Processes Secure SDLC, change management
PR.MA Maintenance Patch management, hardening
PR.PT Protective Technology Firewalls, IDS/IPS, EDR

2.5 Respond Function

RS: Respond

Develop and implement appropriate activities to take action regarding detected cybersecurity events.

Category Description WIA-SEC-015 Implementation
RS.RP Response Planning Incident response playbooks
RS.CO Communications Crisis communication plans
RS.AN Analysis Forensic investigation capability
RS.MI Mitigation Automated containment actions
RS.IM Improvements Post-incident reviews

2.6 Recover Function

RC: Recover

Develop and implement appropriate activities to maintain plans for resilience and restore capabilities impaired due to cybersecurity events.

  • RC.RP - Recovery Planning: Business continuity and disaster recovery plans
  • RC.IM - Improvements: Continuous improvement from lessons learned
  • RC.CO - Communications: Stakeholder communication during recovery

2.7 Implementation Tiers

Tier Name Characteristics
Tier 1 Partial Ad hoc, reactive, limited awareness
Tier 2 Risk Informed Risk awareness, approved but not organization-wide
Tier 3 Repeatable Formal policies, consistent implementation
Tier 4 Adaptive Continuous improvement, predictive capabilities

2.8 Chapter Summary

Key Takeaways

  • NIST CSF provides five core functions: Identify, Protect, Detect, Respond, Recover
  • WIA-SEC-015 implements all 23 categories with practical controls
  • Each function has specific categories with measurable outcomes
  • Implementation tiers guide organizational maturity progression
  • Profiles enable customization to organizational context
