5.1 Modern Encryption Standards

Cryptography forms the foundation of data protection, providing confidentiality, integrity, and authentication. WIA-SEC-015 mandates strong encryption standards that meet current security requirements.

Type Algorithm Key Size Use Case
Symmetric AES-256-GCM 256 bits Data encryption
Asymmetric RSA 4096 bits Key exchange, signatures
Asymmetric ECDSA (P-384) 384 bits Digital signatures
Hash SHA-256/512 256/512 bits Integrity verification
Hash BLAKE3 256 bits High-performance hashing
KDF Argon2id Variable Password hashing

5.2 Post-Quantum Cryptography

Quantum Threat

5.3 Key Management

5.3.1 Key Lifecycle

Phase Activities Controls
Generation Create keys with CSPRNG HSM, entropy sources
Distribution Secure key exchange Key wrapping, secure channels
Storage Protect keys at rest HSM, key vault, encryption
Usage Access control, auditing Role separation, logging
Rotation Regular key replacement Automated rotation, versioning
Destruction Secure key deletion Cryptographic erasure

5.3.2 Key Management Solutions

  • Hardware Security Modules (HSM): Physical tamper-resistant devices
  • Cloud Key Management: AWS KMS, Azure Key Vault, GCP Cloud KMS
  • Secrets Management: HashiCorp Vault, CyberArk
  • Certificate Management: PKI, certificate lifecycle automation

5.5 Data Loss Prevention (DLP)

DLP Type Location Capabilities

Endpoint DLP Workstations, laptops USB, printing, clipboard
Cloud DLP Cloud services SaaS, IaaS, storage
Discovery DLP Data repositories Classification, inventory

5.6 Chapter Summary

Key Takeaways

  • AES-256-GCM is the standard for symmetric encryption
  • Post-quantum cryptography transition should begin now
  • Key lifecycle management ensures key security
  • TLS 1.3 protects data in transit
  • DLP prevents unauthorized data disclosure
