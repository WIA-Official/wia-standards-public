챕터 요약

핵심 요점:

  1. 데이터 교환 보안은 전송 암호화(TLS), 인증(OAuth 2.0, JWT), 권한 부여(RBAC, ABAC) 및 데이터 보호(저장 시 암호화, PII 마스킹)를 포함한 심층 방어 전략이 필요합니다.
  2. TLS 1.3은 암호화된 전송을 제공하여 전송 중인 데이터의 기밀성과 무결성을 보장하며, 전방향 보안, 인증서 검증 및 중간자 공격에 대한 보호를 제공합니다.
  3. OAuth 2.0 및 OpenID Connect는 API에 대한 안전한 위임 권한 부여 및 인증을 가능하게 하며, 웹 앱, 모바일 앱 및 기계 간 통신을 포함한 다양한 클라이언트 시나리오를 위한 여러 권한 부여 유형을 지원합니다.
  4. JSON Web Token(JWT)은 클레임과 서명이 포함된 상태 비저장 인증 토큰을 제공하여 분산 세션 관리, API 인증 및 암호화 검증을 통한 정보 교환을 가능하게 합니다.
  5. 속도 제한, API 키 및 스로틀링은 API를 남용, 서비스 거부 공격 및 리소스 소진으로부터 보호하면서 클라이언트 전반에 걸쳐 공정한 사용을 보장하고 시스템 안정성을 유지합니다.
  6. 규정 준수 요구사항(GDPR, HIPAA, SOC 2)은 암호화, 액세스 로깅, 데이터 최소화, 동의 관리 및 침해 알림 절차를 포함한 데이터 보호 제어를 의무화합니다.

복습 질문

  1. 데이터 교환 시스템을 위한 심층 방어 보안 모델을 설명하십시오. 네트워크, 전송, 애플리케이션 및 데이터 계층에서 어떤 보안 제어가 구현되어야 합니까?
  2. OAuth 2.0 권한 부여 유형(Authorization Code, Client Credentials, Implicit, Resource Owner Password)을 비교하십시오. 각 시나리오에 적합한 권한 부여 유형은 무엇입니까: 웹 앱, 모바일 앱, SPA, 서버 간?
  3. JSON Web Token(JWT)이 상태 비저장 인증을 어떻게 가능하게 합니까? JWT 구조(헤더, 페이로드, 서명), 검증 프로세스 및 토큰 만료 및 비밀 관리를 포함한 보안 고려사항을 설명하십시오.
  4. HIPAA를 준수하는 의료 데이터 교환 시스템을 위한 포괄적인 API 보안 아키텍처를 설계하십시오. 인증, 권한 부여, 감사 로깅, 암호화 및 데이터 마스킹 요구사항을 다루십시오.
  5. 전송 중인 다양한 데이터 형식의 보안 영향은 무엇입니까? 인젝션 공격, 스키마 중독 및 리소스 소진과 같은 취약점에 대해 JSON, XML 및 Protocol Buffers를 비교하십시오.
  6. API에 대한 속도 제한 전략을 설명하십시오. 토큰 버킷, 누수 버킷 및 고정 윈도우 알고리즘을 비교하십시오. HTTP 헤더를 통해 클라이언트에게 속도 제한을 어떻게 전달합니까?
  7. 유니버설 데이터 교환(WIA-CORE-003)이 데이터 이식성, 삭제권 및 동의 관리에 대한 GDPR 요구사항을 어떻게 해결합니까? 특정 구현 패턴을 제공하십시오.
