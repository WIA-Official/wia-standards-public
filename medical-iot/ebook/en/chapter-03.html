Chapter 3: Network Architecture

Connectivity Technologies and Infrastructure for Medical IoT

3.1 IoMT Network Topology

Medical IoT networks typically follow a tiered architecture connecting devices to edge gateways, then to cloud platforms, and finally to healthcare applications. This topology balances local processing needs, connectivity constraints, and data flow requirements.

The architecture must address unique healthcare requirements: reliability for critical devices, security for protected health information, low latency for time-sensitive applications, and scalability across potentially thousands of devices.

3.2 Connectivity Technologies

Different connectivity technologies serve different IoMT use cases based on range, power consumption, bandwidth, and reliability requirements.

Short-Range Technologies

Technology Range Use Cases Characteristics
Bluetooth Low Energy (BLE) 10-100m Wearables, home monitors Low power, smartphone pairing
Bluetooth Classic 10-100m Audio devices, higher bandwidth Higher power, continuous streams
ZigBee 10-100m Mesh sensor networks Low power, mesh networking
NFC <10cm CGM scanning, authentication No pairing, tap-to-read
WiFi 50m High-bandwidth devices Higher power, standard infrastructure

Wide-Area Technologies

Technology Range Use Cases Characteristics
4G LTE Cellular RPM devices, mobile health Wide coverage, moderate power
5G Cellular Real-time monitoring, telehealth High bandwidth, low latency
LTE-M Cellular Asset tracking, remote sensors Low power, extended battery
NB-IoT Cellular Low-frequency sensors Very low power, indoor coverage
LoRaWAN 15km+ Campus sensors, environmental Long range, low power, low cost

5G for Healthcare

5G transforms healthcare with three capability types: Enhanced Mobile Broadband (eMBB) enables high-definition telemedicine and AR surgery; Ultra-Reliable Low-Latency Communications (URLLC) supports remote robotic surgery and critical monitoring; Massive Machine-Type Communications (mMTC) connects thousands of sensors per cell. Healthcare-specific network slices can guarantee performance for critical applications.

3.3 Edge Computing Architecture

Edge computing processes data closer to devices, reducing latency, bandwidth usage, and cloud dependency while enabling real-time insights.

Edge Computing Tiers

Tier Location Function
Device Edge On the device itself Basic filtering, compression, local alerts
Gateway Edge Hub, smartphone, local server Data aggregation, protocol translation
Fog Layer Department/floor level Local analytics, real-time dashboards
Enterprise Edge Hospital data center On-premise processing, EHR integration
Cloud Cloud provider Long-term storage, population analytics

Edge vs. Cloud Trade-offs

Edge computing offers lower latency, continued operation during connectivity loss, and reduced bandwidth costs. Cloud computing provides elastic scalability, advanced analytics, and centralized management. Most IoMT architectures use a hybrid approach—real-time processing at the edge, with aggregated data flowing to the cloud for storage, advanced analytics, and cross-patient insights.

3.4 Network Segmentation

Proper network segmentation isolates medical IoT traffic to improve security and performance:

Segmentation Strategies

  • VLANs: Separate IoMT devices onto dedicated virtual LANs
  • Firewalls: Control traffic flow between network segments
  • Micro-segmentation: Fine-grained isolation down to device level
  • DMZ: Place internet-facing components in demilitarized zone
  • Air Gaps: Physical isolation for critical systems (rare)

⚠️ Flat Network Risks

A flat network where IoMT devices share the same segment as clinical workstations creates serious risks. A compromised IoT device becomes a pivot point for lateral movement to access EHR systems or spread ransomware. Network segmentation limits blast radius and enforces access controls between device types.

3.5 Gateway Architecture

Gateways serve as the critical bridge between devices and the broader network, performing multiple functions:

Gateway Functions

  • Protocol Translation: Convert device protocols (BLE, ZigBee) to IP-based
  • Data Aggregation: Combine data from multiple devices
  • Local Processing: Filter, compress, and analyze data locally
  • Security Enforcement: Authenticate devices, encrypt communications
  • Buffering: Store data during connectivity outages
  • Device Management: Firmware updates, configuration changes

Gateway Options

Gateway Type Use Case Advantages
Smartphone Consumer wearables, home monitoring Ubiquitous, cellular connectivity
Dedicated Hub RPM kits, multi-device setups Always-on, cellular, no patient interaction
Clinical Workstation Point-of-care devices Existing infrastructure, EHR integration
Industrial Gateway Hospital infrastructure IoT Ruggedized, high device capacity

3.6 Cloud Integration Patterns

Cloud platforms provide scalable infrastructure for IoMT data storage, processing, and application hosting:

Major Healthcare Cloud Platforms

  • AWS HealthLake: FHIR-native data lake for health information
  • Azure Health Data Services: FHIR, DICOM, MedTech services
  • Google Cloud Healthcare API: FHIR, HL7v2, DICOM support
  • Specialized Platforms: Validic, Redox, Particle Health for device data

✓ Network Architecture Best Practices

Design for resilience with redundant connectivity paths; implement network segmentation from day one; use edge computing for latency-sensitive applications; plan for offline operation where appropriate; size bandwidth for peak usage plus growth; monitor network performance continuously; and document all device connections for security audits.

📌 Key Takeaways

  • IoMT networks follow tiered architecture: devices → gateways → edge → cloud → applications
  • Short-range technologies (BLE, ZigBee, WiFi) connect devices to local gateways; wide-area (cellular, LPWAN) provide broader coverage
  • 5G enables transformative healthcare applications with high bandwidth, low latency, and massive device density
  • Edge computing reduces latency and enables real-time processing; cloud provides scalability and advanced analytics
  • Network segmentation is essential—isolate IoMT devices to limit security breach impact
  • Gateways perform protocol translation, data aggregation, security enforcement, and local processing
