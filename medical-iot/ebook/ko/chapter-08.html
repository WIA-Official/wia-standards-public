제8장: WIA-MEDICAL-IOT 표준

4단계 구현 아키텍처를 포함한 완전한 기술 사양

8.1 표준 개요

WIA-MEDICAL-IOT 표준은 안전하고 상호운용 가능한 의료 사물인터넷 시스템 구현을 위한 포괄적인 프레임워크를 제공합니다. WIA 4단계 아키텍처(데이터, API, 프로토콜, 통합)를 기반으로 이 표준은 의료 기기, 원격 모니터링 플랫폼, 임상 시스템, 분석 애플리케이션 간의 원활한 연결을 가능하게 합니다.

弘益人間(홍익인간) - "널리 인간을 이롭게 하라"의 원칙에 따라, 이 표준은 지리적 또는 경제적 장벽에 관계없이 지속적인 치료와 조기 개입을 가능하게 하여 연결된 건강 기술을 전 세계적으로 접근 가능하게 만드는 것을 목표로 합니다.

8.2 WIA 4단계 아키텍처

WIA-MEDICAL-IOT 표준은 WIA 4단계 아키텍처 패턴을 구현합니다:

아키텍처 철학

4단계 아키텍처는 관심사를 별도의 계층으로 분리합니다: 1단계(데이터)는 FHIR와 호환되는 기기 및 관찰 모델을 정의합니다; 2단계(API)는 기기 관리 및 데이터 접근을 위한 RESTful 엔드포인트를 제공합니다; 3단계(프로토콜)은 BLE, MQTT, 웹 프로토콜을 통한 기기 통신을 가능하게 합니다; 4단계(통합)은 EHR, 분석, 임상 시스템과 연결합니다.

단계 이름 목적 구성 요소
1단계 데이터 계층 정보 모델링 Device, Observation, Patient 모델
2단계 API 계층 프로그램적 접근 FHIR API, 기기 관리, 알림
3단계 프로토콜 계층 기기 통신 BLE PHD, MQTT, IEEE 11073
4단계 통합 계층 외부 연결 EHR, RPM 플랫폼, 분석

8.4 2단계: API 계층 사양

API 계층은 기기 관리 및 데이터 접근을 위한 RESTful 엔드포인트를 제공합니다:

API 엔드포인트 개요

범주 엔드포인트 메서드 설명
기기 /api/v1/devices GET 기기 목록 (필터링)
/api/v1/devices/{id} GET 기기 상세 정보
/api/v1/devices/{id}/assign POST 환자에게 기기 할당
관찰 /fhir/Observation GET 관찰 쿼리
/fhir/Observation POST 관찰 제출
/api/v1/observations/aggregate GET 집계 데이터 조회
알림 /api/v1/alerts GET 활성 알림 쿼리
/api/v1/alerts/{id}/acknowledge POST 알림 확인
/api/v1/alert-rules POST 알림 규칙 구성
프로그램 /api/v1/programs GET RPM 프로그램 목록
/api/v1/programs/{id}/enroll POST 환자 등록

인증

API는 JWT 토큰과 함께 OAuth 2.0을 사용합니다. 스코프에는 devices:read, devices:manage, observations:read, observations:write, alerts:manage, programs:admin이 포함됩니다. SMART on FHIR는 컨텍스트 인식 권한 부여로 EHR 실행 애플리케이션을 가능하게 합니다.

8.5 3단계: 프로토콜 계층 사양

프로토콜 계층은 기기 연결을 위한 통신 표준을 정의합니다:

지원 프로토콜

프로토콜 사용 사례 표준
Bluetooth PHD 개인 건강 기기 IEEE 11073-20601, GATT 프로파일
MQTT IoT 메시징 MQTT 3.1.1/5.0 over TLS
HTTP/REST 게이트웨이에서 클라우드 HTTPS, FHIR REST
WebSocket 실시간 스트리밍 실시간 데이터용 WSS
CoAP 제약 기기 CoAP over DTLS

MQTT 토픽 구조

// 기기 텔레메트리
wia/iomt/{org_id}/devices/{device_id}/telemetry

// 기기 상태
wia/iomt/{org_id}/devices/{device_id}/status

// 알림
wia/iomt/{org_id}/patients/{patient_id}/alerts

// 기기 명령
wia/iomt/{org_id}/devices/{device_id}/commands

// 메시지 페이로드 예시
{
  "device_id": "bp-monitor-123",
  "timestamp": "2025-01-15T10:30:00Z",
  "measurements": [
    { "type": "systolic_bp", "value": 120, "unit": "mmHg" },
    { "type": "diastolic_bp", "value": 80, "unit": "mmHg" },
    { "type": "pulse", "value": 72, "unit": "bpm" }
  ]
}

8.6 4단계: 통합 계층 사양

통합 계층은 IoMT 시스템을 헬스케어 에코시스템과 연결합니다:

통합 패턴

시스템 통합 방법 교환 데이터
EHR FHIR API, SMART on FHIR 관찰, 기기, 케어 플랜
RPM 플랫폼 REST API, 웹훅 기기 데이터, 알림, 프로그램
분석 이벤트 스트리밍, 배치 내보내기 시계열 데이터, 결과
환자 앱 모바일 SDK, 푸시 알림 측정값, 교육, 알림
청구 ADT 이벤트, 청구 캡처 RPM 코드, 전송 일수

✓ 구현 모범 사례

  • 최대 상호운용성을 위해 FHIR에 정렬된 1단계 데이터 모델로 시작
  • TLS 암호화와 함께 실시간 기기 통신을 위한 MQTT 구현
  • 임상 데이터 통합에 FHIR Observation 리소스 사용
  • 알림 피로를 방지하기 위해 임상 입력으로 알림 규칙 설계
  • 안전한 EHR 실행 애플리케이션을 위해 SMART on FHIR 활성화
  • 규정 준수를 위한 포괄적인 감사 로깅 구현

📌 핵심 요점

  • WIA-MEDICAL-IOT는 4단계 아키텍처를 구현합니다: 데이터 계층(FHIR 정렬 모델), API 계층(REST, FHIR), 프로토콜 계층(BLE, MQTT, IEEE 11073), 통합 계층(EHR, 분석)
  • 핵심 데이터 모델에는 완전한 FHIR 호환성을 갖춘 Device, Observation, Alert가 포함됩니다
  • API 계층은 기기 관리, 관찰, 알림, 프로그램 등록을 위한 RESTful 엔드포인트를 제공합니다
  • 프로토콜 계층은 기기 연결을 위해 Bluetooth PHD, MQTT, HTTP/REST, WebSocket, CoAP를 지원합니다
  • 통합 패턴은 IoMT를 EHR(FHIR), RPM 플랫폼, 분석 시스템, 환자 앱과 연결합니다
  • 보안 프레임워크는 기기 인증, 전송 암호화, API 권한 부여, 데이터 보호를 다룹니다

弘益人間 (홍익인간) - 널리 인간을 이롭게 하라

WIA-MEDICAL-IOT 표준은 연결된 건강 기술에 대한 접근을 민주화하여 인류에게 이로움을 주는 원칙을 구현합니다. 표준화되고 상호운용 가능한 시스템을 통해 지속적인 모니터링과 조기 개입을 전 세계 환자에게 확장할 수 있으며—비용과 의료 격차를 줄이면서 결과를 개선하는 선제적 치료를 가능하게 합니다.
