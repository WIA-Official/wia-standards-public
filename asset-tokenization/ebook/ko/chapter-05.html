🏠 2단계: API - 토큰 관리 & 거래

RESTful 및 GraphQL API를 통한 토큰 플랫폼 구축 | 예상 독서 시간: 30분

2단계 개요

2단계는 자산 토큰을 생성, 관리 및 거래하기 위한 표준화된 API를 정의합니다. RESTful 엔드포인트 및 GraphQL 스키마를 제공하여 토큰 발행, 준수 확인, 전송, 투자자 관리 및 배당금 분배를 처리합니다.

🎯 2단계 목표

토큰 생성, 전송, 준수 확인 및 거래를 위한 보편적인 API를 제공합니다. 모든 WIA 호환 플랫폼이 일관된 인터페이스를 통해 상호 작용할 수 있도록 하여 개발자 채택 및 에코시스템 성장을 가능하게 합니다.

토큰 생성 API

새 토큰을 생성하고 발행합니다:

POST /api/v1/tokens { "name": "Austin Office Building Token", "symbol": "AOBT", "totalSupply": 10000000, "assetClass": "real_estate", "issuer": { "name": "Austin Real Estate LLC", "jurisdiction": "US" }, "asset": { "name": "Austin Downtown Office Building", "location": { "address": "123 Congress Avenue", "city": "Austin", "state": "TX" }, "valuation": { "initialValue": 10000000, "currency": "USD" } }, "compliance": { "regulation": "Reg D 506(c)", "accreditedOnly": true, "allowedJurisdictions": ["US"] } } // 응답 { "tokenId": "0x1234...5678", "contractAddress": "0xabcd...ef01", "status": "deployed", "createdAt": "2024-01-15T12:00:00Z" }

준수 확인 API

전송이 규정을 준수하는지 확인합니다:

POST /api/v1/compliance/check-transfer { "tokenId": "0x1234...5678", "from": "0xabc...123", "to": "0xdef...456", "amount": 1000 } // 응답 { "canTransfer": true, "reason": "All compliance checks passed", "checks": { "kycVerified": { "sender": true, "recipient": true }, "jurisdictionAllowed": { "sender": true, "recipient": true }, "lockupPeriod": { "satisfied": true, "remainingDays": 0 }, "ownershipLimit": { "withinLimit": true, "maxOwnership": 10, "recipientOwnershipAfter": 1.5 } } }

전송 API

준수 확인과 함께 토큰을 전송합니다:

POST /api/v1/transfers { "tokenId": "0x1234...5678", "from": "0xabc...123", "to": "0xdef...456", "amount": 1000, "reason": "Secondary market purchase" } // 응답 { "transferId": "tx-123456", "status": "pending", "txHash": "0xtx...789", "estimatedConfirmation": "2024-01-15T12:05:00Z" }

투자자 관리 API

투자자 KYC, 인증 및 포트폴리오를 관리합니다:

POST /api/v1/investors { "email": "investor@example.com", "firstName": "John", "lastName": "Doe", "walletAddress": "0xabc...123", "jurisdiction": "US", "accreditation": { "status": "accredited", "method": "income", "verifiedBy": "VerifyInvestor", "verifiedAt": "2024-01-10" }, "kyc": { "provider": "Onfido", "verificationId": "kyc-123456", "status": "verified", "verifiedAt": "2024-01-10" } } // 응답 { "investorId": "inv-123456", "status": "active", "createdAt": "2024-01-15T10:00:00Z" }

배당금 분배 API

토큰 보유자에게 배당금을 분배합니다:

POST /api/v1/dividends { "tokenId": "0x1234...5678", "amount": 550000, "currency": "USD", "period": "2024-Q1", "paymentMethod": "USDC", "snapshotDate": "2024-03-31", "paymentDate": "2024-04-15" } // 응답 { "dividendId": "div-123456", "status": "scheduled", "recipients": 423, "totalAmount": 550000, "perTokenAmount": 0.0647, "scheduledAt": "2024-04-15T00:00:00Z" }

GraphQL API

복잡한 쿼리 및 실시간 업데이트를 위한 GraphQL 스키마:

query { token(id: "0x1234...5678") { name symbol totalSupply asset { name location { city state } valuation { currentValue currency lastUpdated } } holders { address balance percentage kycVerified } transfers(last: 10) { from to amount timestamp } dividends { amount period paymentDate status } } } // 구독 (실시간 업데이트) subscription { tokenUpdated(tokenId: "0x1234...5678") { event data { ... on Transfer { from to amount } ... on DividendPaid { amount recipients } } } }

🚀 핵심 요점

2단계 API는 자산 토큰화 플랫폼을 위한 보편적인 인터페이스를 제공합니다. 표준화된 엔드포인트를 따르면 개발자는 모든 WIA 호환 시스템과 통합하는 애플리케이션을 구축할 수 있어 에코시스템 성장 및 유동성을 가능하게 합니다.
