프로덕션 시스템과 원활한 통합 | 예상 독서 시간: 35분

4단계 개요

4단계는 자산 토큰화 플랫폼과 실제 금융 인프라 간의 원활한 통합을 제공합니다. 커스터디언(Fireblocks, BitGo), KYC 제공자(Onfido, Jumio), 거래소(tZERO, OpenFinance), 지갑(MetaMask, Ledger) 및 법정화폐 온/오프 램프(Wyre, Ramp)와의 통합을 다룹니다.

🎯 4단계 목표

자산 토큰화 플랫폼이 프로덕션 준비가 되도록 합니다. 기관급 커스터디, 준수 KYC/AML, 규제된 거래소 상장, 지갑 지원 및 법정화폐 액세스를 제공합니다. 투자자를 위한 종단 간 경험을 활성화합니다: 투자 → 보유 → 거래 → 출금.

Fireblocks 통합

import { Fireblocks } from '@fireblocks/sdk'; const fireblocks = new Fireblocks({ apiKey: process.env.FIREBLOCKS_API_KEY, privateKey: process.env.FIREBLOCKS_PRIVATE_KEY }); // 보관소 생성 async function createVault(tokenName: string) { const vault = await fireblocks.createVault({ name: `${tokenName} Vault`, hiddenOnUI: false }); const address = await fireblocks.generateAddress({ vaultAccountId: vault.id, assetId: 'ETH' }); return { vaultId: vault.id, address: address.address }; } // 안전한 전송 async function secureTransfer( from: string, to: string, amount: string, tokenAddress: string ) { const transaction = await fireblocks.createTransaction({ assetId: 'ETH_TEST', source: { type: 'VAULT_ACCOUNT', id: from }, destination: { type: 'EXTERNAL_WALLET', oneTimeAddress: { address: to } }, amount: amount, extraParameters: { contractCallData: encodeTransferCall(to, amount, tokenAddress) } }); return transaction; }

KYC/AML 제공자

타사 KYC/AML 서비스는 규정 준수를 자동화합니다:

Onfido 통합

import { Onfido } from '@onfido/api'; const onfido = new Onfido({ apiToken: process.env.ONFIDO_API_KEY, region: 'US' }); // KYC 워크플로우 async function verifyInvestor( email: string, firstName: string, lastName: string ) { // 1. 신청자 생성 const applicant = await onfido.applicant.create({ email, firstName, lastName }); // 2. 문서 확인 생성 const check = await onfido.check.create({ applicantId: applicant.id, reportNames: [ 'document', 'identity_enhanced', 'watchlist_enhanced' ] }); // 3. 결과 대기 const result = await onfido.check.find(check.id); return { applicantId: applicant.id, checkId: check.id, status: result.status, verified: result.status === 'complete' && result.result === 'clear' }; }

지갑 지원

ERC-1400 토큰은 표준 이더리움 지갑에서 작동하지만 맞춤형 UI가 필요합니다:

지갑 통합

  • MetaMask: 가장 인기 있는 브라우저 지갑, 맞춤형 토큰 표시 필요
  • Ledger: 하드웨어 지갑 지원, 오프라인 서명
  • Trust Wallet: 모바일 우선 지갑, 내장 DApp 브라우저
  • Gnosis Safe: 멀티시그 지갑, DAO 거버넌스

법정화폐 온/오프 램프

투자자가 법정화폐를 토큰으로, 토큰을 법정화폐로 변환할 수 있게 합니다:

Wyre 통합

import { WyreClient } from '@wyre/api'; const wyre = new WyreClient({ apiKey: process.env.WYRE_API_KEY, secretKey: process.env.WYRE_SECRET_KEY }); // 법정화폐에서 암호화폐로 async function buyTokens( amount: number, currency: string, walletAddress: string ) { const order = await wyre.createOrder({ sourceAmount: amount, sourceCurrency: currency, destCurrency: 'USDC', dest: `ethereum:${walletAddress}`, paymentMethod: 'debit-card' }); return order; } // 암호화폐에서 법정화폐로 async function sellTokens( amount: number, bankAccount: string ) { const withdrawal = await wyre.createTransfer({ source: `ethereum:${walletAddress}`, dest: `ach:${bankAccount}`, sourceAmount: amount, sourceCurrency: 'USDC', destCurrency: 'USD' }); return withdrawal; }

🚀 핵심 요점

4단계 통합은 자산 토큰화 플랫폼을 완전한 프로덕션 시스템으로 변환합니다. 커스터디언, KYC 제공자, 거래소 및 법정화폐 온램프와 통합하면 투자자를 위한 원활한 종단 간 경험이 가능해집니다. 이는 주류 채택의 핵심입니다.
