🛠️ Chapter 2: 시작하기

이 장에서는 Rust 개발 환경을 설정하고 첫 번째 프로그램을 작성합니다. Rust 도구 체인, IDE 설정, 그리고 Cargo 빌드 시스템의 기초를 배웁니다.

2.1 Rust 설치

Rust는 rustup이라는 공식 도구를 사용하여 설치하고 관리합니다. rustup은 여러 Rust 버전을 관리하고, 타겟 플랫폼별 컴파일 환경을 설정할 수 있는 강력한 도구입니다.

2.1.1 지원 플랫폼

플랫폼 지원 상태 요구사항
Windows ✅ Tier 1 Windows 7 이상, Visual Studio Build Tools
macOS ✅ Tier 1 macOS 10.12 이상, Xcode Command Line Tools
Linux ✅ Tier 1 glibc 2.17 이상 (대부분의 현대 배포판)
BSD 계열 ✅ Tier 2 FreeBSD, OpenBSD, NetBSD

2.1.2 macOS / Linux 설치

1 터미널을 열고 다음 명령어를 실행하세요: 
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
2 설치 옵션을 선택합니다 (기본값 권장): 
1) Proceed with installation (default)
2) Customize installation
3) Cancel installation

대부분의 경우 1을 선택하면 됩니다.

3 환경 변수를 업데이트합니다: 
source $HOME/.cargo/env

2.1.3 Windows 설치

1 rustup.rs에서 rustup-init.exe를 다운로드하세요
2 Visual Studio Build Tools 설치 (아직 없는 경우):
  • Visual Studio 2022 Community (무료)
  • 또는 Build Tools for Visual Studio 2022
  • "Desktop development with C++" 워크로드 선택
3 rustup-init.exe를 실행하고 기본 옵션으로 설치

2.1.4 설치 확인

✓ 설치가 성공했는지 확인:

rustc --version
cargo --version
rustup --version

출력 예시:

rustc 1.75.0 (82e1608df 2023-12-21)
cargo 1.75.0 (1d8b05cdd 2023-11-20)
rustup 1.26.0 (5af9b9484 2023-04-05)

2.3 Hello, World!

이제 첫 번째 Rust 프로그램을 작성해봅시다. Cargo를 사용하여 새 프로젝트를 생성합니다.

2.3.1 프로젝트 생성

1 터미널에서 새 프로젝트 생성: 
cargo new hello_rust
cd hello_rust

Cargo가 다음과 같은 프로젝트 구조를 생성합니다:

hello_rust/
├── Cargo.toml      # 프로젝트 메타데이터와 의존성
├── .gitignore      # Git 제외 파일
└── src/
    └── main.rs     # 소스 코드

2.3.2 코드 확인

src/main.rs 파일을 열면 다음 코드가 이미 작성되어 있습니다:

fn main() {
    println!("Hello, world!");
}

2.3.3 프로그램 실행

2 프로젝트를 빌드하고 실행: 
cargo run

출력:

   Compiling hello_rust v0.1.0 (/path/to/hello_rust)
    Finished dev [unoptimized + debuginfo] target(s) in 1.23s
     Running `target/debug/hello_rust`
Hello, world!

[i] 참고: cargo run은 자동으로 빌드한 후 실행합니다. 코드가 변경되지 않았다면 빌드를 건너뜁니다.

2.4 코드 이해하기

간단한 "Hello, World!" 프로그램을 분석해봅시다.

2.4.1 main 함수

fn main() {
    // 함수 본문
}
  • fn - 함수 선언 키워드
  • main - 프로그램의 진입점 (entry point)
  • () - 매개변수 없음
  • {} - 함수 본문

모든 Rust 실행 파일은 main 함수에서 시작합니다.

2.4.2 println! 매크로

println!("Hello, world!");
  • println! - 콘솔에 텍스트를 출력하는 매크로
  • ! - 함수가 아닌 매크로를 나타냄
  • "..." - 문자열 리터럴
  • ; - 문장의 끝 (세미콜론 필수)

매크로 vs 함수

매크로는 !로 끝나며, 컴파일 타임에 코드를 생성합니다. println!은 포맷팅과 타입 검사를 컴파일 타임에 수행합니다.

2.4.3 포맷 문자열

println!은 변수 삽입과 포맷팅을 지원합니다:

fn main() {
    let name = "Rust";
    let version = 1.75;

    println!("Hello, {}!", name);
    println!("{} version {}", name, version);
    println!("{name} version {version}");  // Rust 1.58+
}

2.5 Cargo 명령어

Cargo는 Rust의 빌드 시스템이자 패키지 관리자입니다. 가장 자주 사용하는 명령어들을 알아봅시다.

명령어 설명 사용 시점
cargo new <name> 새 프로젝트 생성 프로젝트 시작 시
cargo build 디버그 모드로 빌드 실행 없이 빌드만 할 때
cargo run 빌드 후 실행 개발 중 테스트
cargo check 에러만 확인 (빌드 안 함) 빠른 에러 체크
cargo build --release 최적화된 릴리스 빌드 배포용 바이너리 생성
cargo test 테스트 실행 단위 테스트 및 통합 테스트
cargo fmt 코드 자동 포맷팅 코드 스타일 통일
cargo clippy 린트 검사 코드 품질 개선
cargo doc --open 문서 생성 및 열기 의존성 문서 확인

2.5.1 빌드 모드 비교

특성 Debug 모드 Release 모드
빌드 시간 빠름 (1-2초) 느림 (10초 이상)
실행 속도 느림 빠름 (10-100배)
디버그 정보 포함 제거
최적화 없음 최대
바이너리 크기 큼 작음
사용 시점 개발 중 배포 시

2.6 프로젝트 구조

Cargo 프로젝트의 디렉토리 구조를 이해해봅시다.

my_project/
├── Cargo.toml          # 프로젝트 설정
├── Cargo.lock          # 의존성 버전 고정 (자동 생성)
├── src/
│   ├── main.rs         # 바이너리 진입점
│   ├── lib.rs          # 라이브러리 진입점 (선택)
│   └── bin/            # 추가 바이너리 (선택)
├── tests/              # 통합 테스트
├── examples/           # 예제 코드
├── benches/            # 벤치마크
└── target/             # 빌드 산출물 (Git 제외)
    ├── debug/          # 디버그 빌드
    └── release/        # 릴리스 빌드

2.6.1 Cargo.toml 파일

프로젝트 메타데이터와 의존성을 정의하는 TOML 파일입니다:

[package]
name = "hello_rust"
version = "0.1.0"
edition = "2021"

[dependencies]
# 여기에 의존성 추가
# serde = "1.0"
# tokio = { version = "1.35", features = ["full"] }

2.7 일반적인 오류와 해결

Rust를 시작할 때 자주 발생하는 오류와 해결 방법입니다.

2.7.1 "rustc not found" 오류

증상: rustc: command not found

원인: PATH에 Rust 바이너리가 추가되지 않음

해결:

# macOS/Linux
source $HOME/.cargo/env

# Windows (PowerShell)
$env:Path += ";$env:USERPROFILE\.cargo\bin"

2.7.2 "linker not found" 오류

증상: error: linker 'link.exe' not found

원인: C/C++ 빌드 도구 미설치 (Windows)

해결: Visual Studio Build Tools 설치

2.7.3 "missing semicolon" 오류

error: expected `;`, found `}`
 --> src/main.rs:3:5
  |
2 |     println!("Hello")  // 세미콜론 없음
  |                       ^ help: add `;` here
3 | }
  | - unexpected token

해결: 문장 끝에 세미콜론 추가

2.8 챕터 요약

주요 개념:

  1. rustup: Rust 공식 설치 도구로, 여러 버전과 타겟 관리
  2. Cargo: Rust의 빌드 시스템이자 패키지 관리자
  3. main 함수: 모든 Rust 프로그램의 진입점
  4. println! 매크로: 콘솔 출력을 위한 표준 매크로
  5. 빌드 모드: Debug (개발용)와 Release (배포용) 두 가지
