문자열 생성 Creating Strings

// &str - 문자열 리터럴
let s1: &str = "hello";
// String - 여러 방법
let s2 = String::from("hello");
let s3 = "hello".to_string();
let s4 = "hello".to_owned();
// String → &str
let s5: &str = &s2;

문자열 조작 String Operations

let mut s = String::from("Hello");
// 추가
s.push_str(", World"); // 문자열 추가
s.push('!'); // 문자 추가
// 연결
let s1 = String::from("Hello, ");
let s2 = String::from("World!");
let s3 = s1 + &s2; // s1 소유권 이동!
let s4 = format!("{} {}", "Hello", "World");

문자열 슬라이싱 String Slicing

let s = String::from("hello world");
let hello = &s[0..5]; // "hello"
let world = &s[6..11]; // "world"
let hello = &s[..5]; // 처음부터
let world = &s[6..]; // 끝까지
let all = &s[..]; // 전체
⚠️ 주의: 바이트 단위 슬라이싱! 한글은 3바이트라 &s[0..1]이 깨질 수 있음 ⚠️ Warning: Byte-based slicing! Korean chars are 3 bytes

언제 무엇을 쓸까? When to Use What?

// 함수 매개변수: &str 권장
fn greet(name: &str) {
println!("Hello, {}!", name);
}
// 둘 다 받을 수 있음!
greet("Rust"); // &str
greet(&my_string); // String

String 사용:
• 소유권이 필요할 때
• 수정이 필요할 때
• 구조체 필드로 저장

&str 사용:
• 읽기만 할 때
• 함수 매개변수
• 문자열 리터럴

Use String:
• Need ownership
• Need modification
• Struct fields

Use &str:
• Read-only access
• Function parameters
• String literals
