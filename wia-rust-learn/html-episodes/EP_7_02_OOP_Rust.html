캡슐화

캡슐화 (Encapsulation)

Encapsulation

pub struct BankAccount {
balance: f64, // private! 외부 접근 불가
pub owner: String, // public
}
impl BankAccount {
pub fn new(owner: String) -> Self {
Self { balance: 0.0, owner }
}
pub fn deposit(&mut self, amount: f64) {
if amount > 0.0 { self.balance += amount; }
}
pub fn balance(&self) -> f64 {
self.balance // getter
}
}

인터페이스

Trait = 인터페이스

Trait = Interface

// 인터페이스 정의
trait Drawable {
fn draw(&self);
fn area(&self) -> f64;
}
struct Circle { radius: f64 }
struct Rectangle { width: f64, height: f64 }
impl Drawable for Circle {
fn draw(&self) { println!("○"); }
fn area(&self) -> f64 { 3.14 * self.radius.powi(2) }
}
impl Drawable for Rectangle {
fn draw(&self) { println!("□"); }
fn area(&self) -> f64 { self.width * self.height }
}

다형성

트레이트 객체로 다형성 구현

Polymorphism with Trait Objects

// dyn 트레이트로 동적 디스패치
fn draw_all(shapes: Vec<Box<dyn Drawable>>) {
for shape in shapes {
shape.draw();
println!("넓이: {}", shape.area());
}
}
fn main() {
let shapes: Vec<Box<dyn Drawable>> = vec![
Box::new(Circle { radius: 5.0 }),
Box::new(Rectangle { width: 3.0, height: 4.0 }),
];
draw_all(shapes); // 다형성!
}

구성

구성 (Composition) - 상속 대신!

Composition - Instead of Inheritance!

// 상속 대신 구성 사용
struct Engine { horsepower: u32 }
struct Wheels { count: u32 }
// Car는 Engine과 Wheels를 "가지고 있다" (has-a)
struct Car {
engine: Engine,
wheels: Wheels,
brand: String,
}
impl Car {
fn total_power(&self) -> u32 {
self.engine.horsepower // 위임
}
}

기본 구현

기본 구현 메서드

Default Method Implementations

trait Greet {
fn name(&self) -> &str; // 필수 구현
// 기본 구현 제공!
fn greet(&self) {
println!("안녕, {}!", self.name());
}
fn formal_greet(&self) {
println!("안녕하세요, {} 님!", self.name());
}
}
struct Person { name: String }
impl Greet for Person {
fn name(&self) -> &str { &self.name }
// greet(), formal_greet()은 자동으로 사용 가능!
}

EP 7-02 Summary

Rust OOP 핵심

  • 캡슐화: pub/private
  • Encapsulation: pub/private
  • 인터페이스: trait
  • Interface: trait
  • 다형성: dyn Trait
  • Polymorphism: dyn Trait
  • 구성 > 상속
  • Composition > Inheritance

핵심 철학

  • 상속 없이도 코드 재사용
  • Code reuse without inheritance
  • 명시적 > 암시적
  • Explicit > Implicit
  • 제로 코스트 추상화
  • Zero-cost abstractions
