기본 테스트

첫 번째 테스트 작성

Writing Your First Test

// src/lib.rs
pub fn add(a: i32, b: i32) -> i32 {
a + b
}
#[cfg(test)]
mod tests {
use super::*;
#[test]
fn test_add() {
assert_eq!(add(2, 3), 5);
}
}

💡 #[test] 속성으로 테스트 함수를 표시하고, cargo test로 실행! 💡 Mark test functions with #[test], run with cargo test!

Assert 매크로

다양한 Assert 매크로

Various Assert Macros

#[test]
fn test_asserts() {
// 조건이 참인지
assert!(2 + 2 == 4);
// 두 값이 같은지
assert_eq!(4, 2 + 2);
// 두 값이 다른지
assert_ne!(4, 5);
// 커스텀 에러 메시지
assert!(1 > 0, "1은 0보다 커야 합니다!");
}
✓ assert! - 조건 검사
✓ assert_eq! - 같음 검사
✓ assert_ne! - 다름 검사

Panic 테스트

패닉이 발생해야 하는 테스트

Testing for Panics

pub fn divide(a: i32, b: i32) -> i32 {
if b == 0 {
panic!("0으로 나눌 수 없습니다!");
}
a / b
}
#[test]
#[should_panic]
fn test_divide_by_zero() {
divide(10, 0); // 패닉 발생해야 통과!
}
#[test]
#[should_panic(expected = "0으로 나눌")]
fn test_panic_message() {
divide(10, 0); // 특정 메시지 확인
}

Result 테스트

Result를 반환하는 테스트

Tests That Return Result

#[test]
fn test_with_result() -> Result<(), String> {
let value = 42;
if value == 42 {
Ok(()) // 테스트 통과
} else {
Err(String::from("값이 42가 아닙니다"))
}
}
// ? 연산자 사용 가능!
#[test]
fn test_with_question_mark() -> Result<(), Box<dyn std::error::Error>> {
let parsed: i32 = "42".parse()?;
assert_eq!(parsed, 42);
Ok(())
}

테스트 구조

단위 테스트 vs 통합 테스트

Unit Tests vs Integration Tests

단위 테스트

Unit Tests

// src/lib.rs 내부
#[cfg(test)]
mod tests {
use super::*;
// private 함수도 테스트 가능
}
  • src 폴더 내에 위치
  • Located in src folder
  • private 함수 테스트 가능
  • Can test private functions

통합 테스트

Integration Tests

// tests/integration_test.rs
use my_crate;
#[test]
fn test_public_api() {
assert!(my_crate::add(1,2) == 3);
}
  • tests/ 폴더에 위치
  • Located in tests/ folder
  • public API만 테스트
  • Tests only public API

명령어

유용한 테스트 명령어

Useful Test Commands

# 모든 테스트 실행
$ cargo test
# 특정 테스트만 실행
$ cargo test test_add
# 무시된 테스트 포함
$ cargo test -- --ignored
# 테스트 출력 표시 (println! 포함)
$ cargo test -- --nocapture
# 단일 스레드로 실행 (순차적)
$ cargo test -- --test-threads=1
