NRT: Once transferred, cannot repeat!

Copy (복사) Copy

let x = 5;
let y = x; // Copy!
println!("x={}, y={}", x, y); // ✅ OK!
// 스택에 저장된 간단한 타입은 자동 복사
x
5
→
y
5

Copy 타입: i32, f64, bool, char, 튜플 등

Copy types: i32, f64, bool, char, tuples etc

Clone (복제) Clone

let s1 = String::from("hello");
let s2 = s1.clone(); // 명시적 복제!
println!("s1={}, s2={}", s1, s2); // ✅ OK!
// 힙 데이터까지 전부 복사
s1
"hello"
clone()
s2
"hello"
⚠️ Clone은 비용이 큼! 필요할 때만 사용 ⚠️ Clone is expensive! Use only when needed

비교 Comparison

Move

소유권 이전
원본 무효화
비용: 거의 없음

Ownership transfer
Original invalidated
Cost: Nearly zero

Copy

비트 복사 (자동)
스택 타입만
비용: 작음

Bitwise copy (auto)
Stack types only
Cost: Small

Clone

전체 복제 (명시적)
힙 데이터 포함
비용: 큼

Deep copy (explicit)
Includes heap data
Cost: Large

커스텀 타입 Custom Types

// Copy + Clone 구현
#[derive(Copy, Clone)]
struct Point {

y: i32,
}

#[derive(Clone)]

}

정리 Summary

✅ Move: 소유권 이전 (기본 동작)
✅ Copy: 스택 타입 자동 복사
✅ Clone: 명시적 전체 복제
✅ Copy가 가능하려면 모든 필드가 Copy여야 함

✅ Move: Ownership transfer (default)
✅ Copy: Auto-copy for stack types
✅ Clone: Explicit deep copy
✅ All fields must be Copy for Copy to work
