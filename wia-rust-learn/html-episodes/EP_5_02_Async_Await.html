🦀 Rust NRT Academy
async fn

async 함수 기초

async Function Basics

// async 함수 정의
async fn fetch_data() -> String {
// 비동기 작업 시뮬레이션
String::from("데이터 로드 완료!")
}
// async 함수는 Future를 반환!
// 호출만으로는 실행되지 않음
let future = fetch_data(); // 아직 실행 안됨!
// await로 실행 및 결과 대기
let result = fetch_data().await;
println!("{}", result);

💡 async fn은 즉시 실행되지 않습니다. .await가 있어야 실행! 💡 async fn doesn't run immediately. It needs .await to execute!

Tokio

Tokio - 비동기 런타임

Tokio - Async Runtime

// Cargo.toml
// [dependencies]
// tokio = { version = "1", features = ["full"] }
use tokio;
#[tokio::main]
async fn main() {
println!("Hello, async world!");
// 비동기 sleep
tokio::time::sleep(
std::time::Duration::from_secs(1)
).await;
println!("1초 후!");
}
🦀 Rust NRT Academy
동시 실행

여러 작업 동시 실행

Running Multiple Tasks Concurrently

use tokio::time::{sleep, Duration};
async fn task_a() -> i32 {
sleep(Duration::from_secs(2)).await;
1
}
async fn task_b() -> i32 {
sleep(Duration::from_secs(1)).await;
2
}
#[tokio::main]
async fn main() {
// join!으로 동시 실행 - 총 2초만 걸림!
let (a, b) = tokio::join!(task_a(), task_b());
println!("결과: {} + {} = {}", a, b, a + b);
}

spawn

tokio::spawn - 백그라운드 태스크

tokio::spawn - Background Tasks

#[tokio::main]
async fn main() {
// 백그라운드 태스크 시작
let handle = tokio::spawn(async {
// 이 작업은 별도로 실행됨
42
});
// 다른 작업 수행...
println!("메인 작업 중...");
// 결과 대기
let result = handle.await.unwrap();
println!("결과: {}", result);
}

💡 spawn은 'static 수명이 필요합니다. 참조 대신 소유권을 넘기세요! 💡 spawn requires 'static lifetime. Transfer ownership instead of references!

실전 예제

실전: HTTP 요청 (reqwest)

Real Example: HTTP Requests (reqwest)

// Cargo.toml: reqwest = { version = "0.11", features = ["json"] }
use reqwest;
use serde::{Deserialize};
#[derive(Deserialize, Debug)]
struct User { login: String, id: u64 }
#[tokio::main]
async fn main() -> Result<(), Box<dyn std::error::Error>> {
let user: User = reqwest::get(
"https://api.github.com/users/rust-lang"
).await?.json().await?;
println!("{:?}", user);
Ok(())
}

스레드 vs async/await

Threads vs async/await

스레드 Threads async/await async/await
메모리 Memory 수 MB/스레드 Several MB/thread 수 KB/태스크 Several KB/task
생성 비용 Creation Cost 높음 High 매우 낮음 Very low
동시 실행 수 Concurrent Tasks 수천 개 Thousands 수백만 개 Millions
적합한 용도 Best For CPU 집약 작업 CPU-intensive work I/O 대기 작업 I/O-bound work

💡 웹 서버, API 호출 등 I/O 작업에는 async가 최적! 💡 async is optimal for I/O operations like web servers and API calls!

EP 5-02 Summary

async/await 핵심

  • async fn → Future 반환
  • async fn → Returns Future
  • .await → 실제 실행
  • .await → Actually executes
  • 런타임 필요 (tokio)
  • Needs runtime (tokio)
  • 논블로킹 I/O
  • Non-blocking I/O

주요 도구

  • join! - 동시 실행
  • join! - Concurrent execution
  • spawn - 백그라운드 태스크
  • spawn - Background tasks
  • select! - 먼저 완료된 것
  • select! - First to complete
  • reqwest - HTTP 클라이언트
  • reqwest - HTTP client
