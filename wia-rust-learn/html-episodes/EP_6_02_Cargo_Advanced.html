Cargo 워크스페이스 - 모노레포

Cargo Workspaces - Monorepo

# Cargo.toml (루트)
[workspace]
members = [
"app",
"core",
"utils",
]
# 공유 의존성 (중복 방지)
[workspace.dependencies]
serde = { version = "1.0", features = ["derive"] }
tokio = { version = "1", features = ["full"] }

💡 워크스페이스는 여러 크레이트를 하나의 저장소에서 관리합니다. 의존성 공유와 빌드 최적화! 💡 Workspaces manage multiple crates in one repo. Share dependencies and optimize builds!

Features

조건부 컴파일 - Features

Conditional Compilation - Features

# Cargo.toml
[features]
default = ["json"] # 기본 활성화
json = ["serde_json"]
yaml = ["serde_yaml"]
full = ["json", "yaml"]
[dependencies]
serde_json = { version = "1.0", optional = true }
serde_yaml = { version = "0.9", optional = true }
# 사용법
$ cargo build --features yaml
$ cargo build --features "json yaml"
$ cargo build --no-default-features --features yaml

프로파일

빌드 프로파일 - 최적화 설정

Build Profiles - Optimization Settings

# Cargo.toml
# dev 프로파일 (cargo build)
[profile.dev]
opt-level = 0 # 최적화 없음 (빠른 컴파일)
debug = true # 디버그 정보 포함
# release 프로파일 (cargo build --release)
[profile.release]
opt-level = 3 # 최대 최적화
lto = true # Link-Time Optimization
strip = true # 심볼 제거 (작은 바이너리)
panic = "abort" # 패닉시 즉시 종료

배포

crates.io에 배포하기

Publishing to crates.io

# Cargo.toml - 필수 메타데이터
[package]
name = "my-awesome-crate"
version = "0.1.0"
edition = "2021"
description = "An awesome Rust crate"
license = "MIT"
repository = "https://github.com/user/repo"
keywords = ["rust", "awesome"]
categories = ["development-tools"]
# 배포 명령어
$ cargo login [API_TOKEN]
$ cargo publish --dry-run # 테스트
$ cargo publish # 실제 배포!
