Chapter 5: Data Formats and Protocols

5.1 Standard Data Format Overview

Interoperability depends on standardized data formats that enable seamless exchange between devices, apps, and services. The WIA-PET-007 standard defines JSON-based structures for all data types, ensuring compatibility across platforms and vendors.

5.1.1 Design Principles

  • Human-Readable: JSON format is easily inspected and debugged
  • Self-Describing: Field names clearly indicate content
  • Extensible: New fields can be added without breaking compatibility
  • Versioned: Schema version included for forward/backward compatibility
  • Compact: Optimized structure to minimize data transmission size

5.1.2 Core Data Model

{ "schemaVersion": "1.0.0", "standard": "WIA-PET-007", "deviceId": "PW-DOG-12345", "timestamp": "2025-12-25T10:30:00.000Z", "petProfile": { "petId": "PET-ABC-789", "name": "Max", "species": "dog", "breed": "Golden Retriever", "birthdate": "2020-03-15", "gender": "male", "weight": 30.5, "weightUnit": "kg" }, "metadata": { "firmwareVersion": "2.3.1", "batteryLevel": 87, "signalStrength": -65 } }

5.7 Data Synchronization

5.7.1 Sync Strategy

Efficient synchronization minimizes data usage and latency:

  • Delta Sync: Only transmit changes since last sync
  • Compression: GZIP compression for bulk data transfers
  • Conflict Resolution: Device data always takes precedence for sensor measurements
  • Offline Queue: Data cached locally when connectivity unavailable

  • Direct Transfer: Import data when switching to different device brand

5.9 Versioning and Backward Compatibility

5.9.1 Semantic Versioning

The WIA-PET-007 standard uses semantic versioning (MAJOR.MINOR.PATCH):

  • MAJOR: Incompatible changes, breaking backward compatibility
  • MINOR: New features added in backward-compatible manner
  • PATCH: Backward-compatible bug fixes and clarifications

5.9.2 Compatibility Requirements

Chapter Summary: Key Takeaways

  1. Standardized JSON Format: WIA-PET-007 defines comprehensive JSON schemas for health, activity, and location data, ensuring cross-platform interoperability and future extensibility.
  2. Multi-Protocol Support: BLE for device-to-phone communication, RESTful APIs for cloud services, with well-defined characteristic UUIDs and endpoints.
  3. Efficient Synchronization: Delta sync, compression, and intelligent triggering (scheduled, event-driven, manual) minimize data usage while maintaining data freshness.
  4. Data Portability: Multiple export formats (JSON, CSV, PDF, GPX, FHIR) ensure users retain full ownership and can migrate between platforms.
  5. Robust Error Handling: Standardized HTTP status codes, detailed error messages, and rate limiting protect API stability while supporting developer needs.
