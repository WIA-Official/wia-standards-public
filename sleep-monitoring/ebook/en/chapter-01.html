Chapter 01: Introduction to Sleep Monitoring

1.1 Why this book exists

WIA-MED-021 is the open standard for sleep monitoring data exchange. This book is the practitioner introduction: it walks through the four-phase architecture (Data Format, API Interface, Federation Protocol, Integration) with the design rationale and worked examples that an implementer needs to build a conformant component without reading the specification cold.

1.2 The state of sleep medicine data

Sleep medicine sits at an awkward intersection. Clinical sleep labs run vendor PSG (polysomnography) hardware speaking proprietary protocols; consumer wearables track sleep via heart rate and accelerometer with very different accuracy profiles; home sleep apnoea testing devices sit in their own category. A patient with a clinical PSG report from one hospital and an Apple Watch sleep history cannot easily reconcile the two, and the clinician cannot confidently combine them.

1.3 What WIA-MED-021 changes

The standard collapses the three vendor surfaces into one open wire format with explicit clinical-grade flagging. A consumer wearable tagging itself aasm_compliance_level: consumer_estimate is welcome on the same wire as a clinical PSG tagging itself full — but the receiver can filter on the compliance level and make decisions on appropriate evidence.

1.4 The 4-Phase architecture

Phase 1 (Data Format) defines the JSON envelopes for sessions, sleep stages, respiratory events, arousals, and scoring metadata. Phase 2 (API Interface) specifies the HTTP surface for ingest, query, and re-scoring. Phase 3 (Federation Protocol) handles cross-host data flow with patient consent envelopes as the gating control. Phase 4 (Integration) bridges to AASM scoring, EDF / EDF+ raw recordings, HL7 FHIR R5, vendor wearable platforms, and the WIA family.

1.5 AASM is the clinical reference

The American Academy of Sleep Medicine (AASM) Manual for the Scoring of Sleep and Associated Events is the authoritative clinical reference for sleep stage scoring. WIA-MED-021 does not re-publish AASM rules; it provides the data shape that AASM-compliant scorers consume and emit. The standard preserves the algorithm version explicitly so that an evaluation done under the 2018 Manual can be distinguished from one done under the 2024 Manual without ambiguity.

1.6 EDF / EDF+ stays sovereign

European Data Format (EDF and EDF+) is the de-facto raw biosignal format for sleep medicine. WIA-MED-021 references EDF files by URL plus integrity hash rather than re-encoding the raw waveform. This composition pattern keeps EDF as the source of truth for raw data while letting WIA-MED-021 carry the structured scoring and federation metadata.

Patient and Clinician Considerations

Sleep data is biometric and high-stakes. Phase 1 carries a clinical_grade flag and per-channel signal quality metrics so consumer wearable estimates do not get conflated with clinical PSG measurements. Phase 3 enforces patient consent before any cross-host data flow, with explicit per-audience scopes (clinical_dashboard, ehr_export, research_de_identified). Phase 4 maps to the AASM compliance level so downstream consumers can filter by appropriateness for their clinical context.

The standard's default audience controls are intentionally restrictive: even the patient's own family / caregiver receives only stage histograms by default, not the raw event stream. Operators MAY relax these defaults per local policy, but the standard's baseline assumes minimum disclosure unless explicitly elevated by consent.
