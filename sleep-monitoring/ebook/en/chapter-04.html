Chapter 04: Phase 1 — Data Format

4.1 Phase 1 in one paragraph

Phase 1 specifies the JSON envelopes that every WIA-MED-021 conformant device, sleep lab system, EHR bridge, or research aggregator must read and emit. The envelopes are signed with Ed25519 (IETF RFC 8032) and round-trip byte-identically through encode and decode.

4.2 Session record

The session record is the registration envelope. It declares the session kind (PSG, HSAT, consumer wearable, nap), the start and end time, the recording devices, the clinical-grade flag, and the EDF reference for clinical-grade sessions.

{
  "wia_sleep_monitoring_version": "1.0.0",
  "type": "session",
  "session_id": "ses_01HXY",
  "patient_id": "did:wia:patient:01HXY",
  "session_kind": "polysomnography",
  "started_at": "2026-04-26T22:30:00Z",
  "ended_at":   "2026-04-27T07:00:00Z",
  "device_ids": ["did:wia:device:psg-vendor-A"],
  "clinical_grade": true,
  "edf_url": "https://recordings.example/sessions/ses_01HXY.edf",
  "edf_sha256": "7d8f2c..."
}

4.3 Sleep stage record

Each epoch is one envelope. Required fields: session_id, epoch_index, epoch_start, epoch_duration_seconds, stage, scoring_algorithm. Stages are AASM tokens (wake, n1, n2, n3, rem, unscored).

4.4 Respiratory event record

Apnoea / hypopnoea / RERA / oxygen desaturation events. Each carries the start time, duration, associated desaturation percentage, associated arousal flag, and the scoring algorithm. Downstream AHI / RDI computation reads the event stream.

4.5 Arousal record

EEG or movement-based arousals. Minimum 3-second duration per AASM. The standard refuses sub-3-second arousals at the API layer.

4.6 Scoring metadata record

Ties a session to its scoring algorithm and scorer (technician). Carries the summary statistics: total_sleep_minutes, REM minutes, N3 minutes, AHI, minimum SpO2. Downstream consumers MUST trust the published summary rather than recomputing from individual events (because consumer wearables sometimes derive AHI from non-AASM-equivalent inputs).

Patient and Clinician Considerations

Sleep data is biometric and high-stakes. Phase 1 carries a clinical_grade flag and per-channel signal quality metrics so consumer wearable estimates do not get conflated with clinical PSG measurements. Phase 3 enforces patient consent before any cross-host data flow, with explicit per-audience scopes (clinical_dashboard, ehr_export, research_de_identified). Phase 4 maps to the AASM compliance level so downstream consumers can filter by appropriateness for their clinical context.

The standard's default audience controls are intentionally restrictive: even the patient's own family / caregiver receives only stage histograms by default, not the raw event stream. Operators MAY relax these defaults per local policy, but the standard's baseline assumes minimum disclosure unless explicitly elevated by consent.
