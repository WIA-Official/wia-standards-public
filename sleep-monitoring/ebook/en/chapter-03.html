Chapter 03: WIA-MED-021 Standard Overview

3.1 Standard at a glance

WIA-MED-021 is a four-phase open standard for sleep monitoring data. Phase 1 fixes JSON envelopes. Phase 2 fixes the HTTP surface. Phase 3 fixes federation and consent semantics. Phase 4 bridges to AASM scoring, EDF / EDF+ raw recordings, HL7 FHIR R5, and vendor wearable platforms.

3.2 Object families

  • Session — one contiguous sleep recording (one night, or one nap)
  • Sleep stage — one epoch (typically 30 seconds) classified into AASM stages
  • Respiratory event — apnoea, hypopnoea, RERA, oxygen desaturation
  • Arousal — EEG-based or movement-based arousal event
  • Scoring metadata — algorithm + scorer responsible for the session classification

3.3 Roles in the federation

Phase 3 defines five roles: recording host (holds raw session data), scoring service (performs AASM-compliant scoring), clinical host (hospital sleep clinic platform), research aggregator (de-identified research datasets), and custodian (holds the patient consent envelope; for paediatric patients, the legal guardian).

3.5 Where it sits in the WIA family

WIA-MED-021 sits beside WIA-MED-020 (Wearable Health) and composes with WIA Vital Sign Streaming for high-rate continuous biosignals (the EEG / EMG / ECG channels in a clinical PSG). Patient identity and clinician credentials live in WIA-OMNI-API. Patient and clinician UI accommodations come from WIA-ACCESSIBILITY.

Where to Look Next

  • Phase 1 specification at spec/PHASE-1-DATA-FORMAT.md — the JSON envelope shapes for sessions, sleep stages, respiratory events, arousals, and scoring metadata.
  • Phase 2 specification at spec/PHASE-2-API-INTERFACE.md — the HTTP surface for ingest, query, and re-scoring.
  • Phase 3 specification at spec/PHASE-3-PROTOCOL.md — federation handshake, replay defence, patient consent, audience controls.
  • Phase 4 specification at spec/PHASE-4-INTEGRATION.md — AASM, EDF/EDF+, HL7 FHIR R5, vendor wearable bridges, WIA family composition.
  • AASM Manual for the Scoring of Sleep and Associated Events — the authoritative clinical reference for sleep stage scoring rules. WIA-MED-021 carries data shapes; AASM defines the scoring rules themselves.
  • European Data Format (EDF / EDF+) — the de-facto raw biosignal format that WIA-MED-021 references rather than re-encoding.
  • HL7 FHIR R5 Procedure / Observation resources — the EHR integration target for sleep-clinic results.

Patient and Clinician Considerations

Sleep data is biometric and high-stakes. Phase 1 carries a clinical_grade flag and per-channel signal quality metrics so consumer wearable estimates do not get conflated with clinical PSG measurements. Phase 3 enforces patient consent before any cross-host data flow, with explicit per-audience scopes (clinical_dashboard, ehr_export, research_de_identified). Phase 4 maps to the AASM compliance level so downstream consumers can filter by appropriateness for their clinical context.

The standard's default audience controls are intentionally restrictive: even the patient's own family / caregiver receives only stage histograms by default, not the raw event stream. Operators MAY relax these defaults per local policy, but the standard's baseline assumes minimum disclosure unless explicitly elevated by consent.
