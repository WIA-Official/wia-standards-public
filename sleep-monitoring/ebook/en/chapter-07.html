Chapter 07: Phase 4 — Integration

7.1 Phase 4 in one paragraph

Phase 4 specifies how WIA-MED-021 integrates with three classes of existing system: clinical sleep medicine standards (AASM, EDF/EDF+, HL7 FHIR R5), vendor sleep platforms (Apple HealthKit, Google Fit, Samsung Health, Fitbit, Oura, Withings), and the WIA family standards.

7.2 AASM scoring integration

AASM is the clinical reference standard for sleep stage scoring. The standard\x27s scoring_algorithm field carries an opaque identifier registered at https://wiastandards.com/sleep-monitoring/scoring-algorithms/. Reference algorithms include AASM-2024-default, AASM-2018-default (preserved for cohort comparability), AASM-2024-paediatric, and reference vendor-specific algorithms tagged at aasm_compliance_level: consumer_estimate.

7.3 EDF / EDF+ bridge

European Data Format and EDF+ are the de-facto raw biosignal formats for sleep medicine. The bridge reads an EDF file from object storage, translates per-channel metadata into the WIA session envelope, preserves the EDF URL plus SHA-256 integrity hash, and writes scoring results back to a parallel WIA scoring_metadata envelope without modifying the original EDF file.

7.4 HL7 FHIR R5 bridge

Each WIA session maps to an HL7 FHIR R5 Procedure resource (representing the sleep study) plus one or more Observation resources (representing scoring summary metrics: AHI, RDI, total sleep time, stage minutes). Default LOINC mappings cover the standard summary fields.

7.5 Vendor platform bridges

HealthKit, Google Fit, Samsung Health, Fitbit, Oura, Withings each have a vendor-specific bridge profile under spec/profiles/. Each profile documents OAuth scopes, field-by-field mapping, polling cadence, and credential management.

7.6 WIA family integration

WIA-OMNI-API holds patient identity, vendor credentials, clinician licences, paediatric custodian relationships. WIA-MED-020 and WIA-MED-021 are sister standards: consumer wearable sleep data typically arrives as WIA-MED-020 measurement streams and is aggregated into WIA-MED-021 sessions. WIA Vital Sign Streaming carries the high-rate biosignal channels for clinical PSG. WIA-AIR-SHIELD provides transport hardening. WIA-ACCESSIBILITY drives patient and clinician UI accommodations.

7.7 Compliance mappings

HIPAA Security Rule controls apply to every PHI-bearing host. The patient consent envelope is the GDPR Article 7 consent record. For clinical-grade PSG hardware, the device record carries the regulatory-clearance reference (FDA 510(k), EU CE-MDR, KFDA approval). Sleep clinics seeking AASM accreditation can use the standard\x27s audit log as evidence of conformant scoring practice.

Patient and Clinician Considerations

Sleep data is biometric and high-stakes. Phase 1 carries a clinical_grade flag and per-channel signal quality metrics so consumer wearable estimates do not get conflated with clinical PSG measurements. Phase 3 enforces patient consent before any cross-host data flow, with explicit per-audience scopes (clinical_dashboard, ehr_export, research_de_identified). Phase 4 maps to the AASM compliance level so downstream consumers can filter by appropriateness for their clinical context.

The standard's default audience controls are intentionally restrictive: even the patient's own family / caregiver receives only stage histograms by default, not the raw event stream. Operators MAY relax these defaults per local policy, but the standard's baseline assumes minimum disclosure unless explicitly elevated by consent.
