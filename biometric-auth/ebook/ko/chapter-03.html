3.1.1 각 단계 개요

단계 목적 핵심 구성요소
1단계: 데이터 형식 상호운용성을 위한 데이터 표준화 템플릿 형식, 이미지 형식, 품질 점수
2단계: API 표준화된 작업 인터페이스 등록, 검증, 식별 API
3단계: 프로토콜 보안 및 검증 절차 PAD, 보안 매칭, 감사
4단계: 통합 외부 시스템 연결 IdP, 결제, 출입 통제

3.2 보안 보증 레벨 (SAL)

WIA-BIOMETRIC-AUTH는 다양한 위험 수준에 맞는 4개의 보안 보증 레벨을 정의합니다.

레벨 용도 FAR 요구사항 PAD 레벨
SAL-1 (기본) 저위험 접근 ≤ 1% 레벨 1
SAL-2 (표준) 일반 인증 ≤ 0.1% 레벨 2
SAL-3 (고) 금융/의료 ≤ 0.01% 레벨 3
SAL-4 (최고) 국가 안보 ≤ 0.001% 레벨 3+

// SAL 선택 매트릭스
{
  "riskAssessment": {
    "SAL-1": {

    },

      "impact": "보통",
      "examples": ["기업 로그인", "소비자 금융", "헬스케어 포털"]

    "SAL-3": {

}

3.4 국제 표준과의 관계

국제 표준 범위 WIA 관계
ISO/IEC 19794 생체 데이터 형식 1단계에서 채택 및 확장
ISO/IEC 24745 템플릿 보호 3단계에서 통합
ISO/IEC 30107 PAD (위조 탐지) 3단계 PAD 요구사항 기반
FIDO2/WebAuthn 온라인 인증 4단계에서 연결
ICAO Doc 9303 여행 문서 국경 간 인식 호환

3.6 장 요약

핵심 요점

  • 4단계 아키텍처가 데이터부터 통합까지 포괄적 프레임워크 제공
  • 4개 보안 보증 레벨(SAL)이 다양한 위험 프로필에 대응
  • 핵심 구성요소는 캡처 장치, 처리 엔진, 저장소 포함
  • 국제 표준(ISO, FIDO, ICAO)과 정렬되어 상호운용성 보장
  • 인증은 제3자 테스트와 연간 갱신 필요
