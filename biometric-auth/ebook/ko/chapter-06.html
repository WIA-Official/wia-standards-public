Chapter 06

다중 모달 인증

여러 생체 양식을 결합한 멀티모달 생체인증의 융합 전략과 정확도 향상 방법을 학습합니다.

다중 모달 생체인증 개요

다중 모달 생체인증(Multimodal Biometrics)은 두 개 이상의 생체 양식을 결합하여 인증 정확도와 보안성을 향상시키는 기술입니다. 단일 생체 양식의 한계를 극복하고, 스푸핑 공격에 대한 저항성을 높이며, 더 넓은 사용자 적용 범위를 확보할 수 있습니다.

예를 들어, 지문 인식의 경우 피부 상태나 상처로 인해 일부 사용자에게는 적용이 어려울 수 있습니다. 이때 얼굴 인식을 보조 수단으로 결합하면 모든 사용자에게 안정적인 인증 서비스를 제공할 수 있습니다. 또한, 두 가지 생체 양식을 모두 위조해야 하므로 보안성이 크게 향상됩니다.

단일 vs 다중 모달 생체인증 
단일 모달 시스템의 한계
──────────────────────
1. 노이즈 민감성
   - 환경 조건에 따른 성능 저하
   - 센서 품질 의존성

2. 사용자 적용 범위 제한
   - FTE (등록 실패) 문제
   - 특정 사용자 그룹 배제

3. 스푸핑 취약성
   - 단일 공격 벡터
   - 위조 시 전체 시스템 무력화

4. 정확도 천장
   - 단일 특징의 고유성 한계
   - 동일 오류율(EER) 개선 한계

다중 모달 시스템의 장점
──────────────────────
✓ 정확도 향상: 여러 특징의 상호보완
✓ 보안성 강화: 다중 스푸핑 방어
✓ 보편성 확대: 대체 양식 제공
✓ 강건성 증가: 부분 실패 허용
✓ 유연성: 상황별 양식 선택

다중 모달 시스템 유형

다중 모달 시스템은 어떤 요소를 다중화하느냐에 따라 여러 유형으로 분류됩니다.

유형 설명 예시 장점
다중 생체 양식 서로 다른 생체 특성 결합 지문 + 얼굴 높은 보안성, 다양한 환경 대응
다중 인스턴스 동일 양식의 여러 인스턴스 양손 지문, 양쪽 홍채 구현 용이, 단일 센서
다중 센서 동일 양식을 다른 센서로 광학 + 정전용량 지문 센서 노이즈 보완
다중 알고리즘 동일 데이터에 여러 알고리즘 지문의 특징점 + 텍스처 알고리즘 편향 감소
다중 샘플 동일 양식의 여러 샘플 여러 번의 지문 스캔 템포럴 변동 완화

융합 전략

다중 모달 시스템에서 여러 생체 정보를 결합하는 방식을 융합(Fusion)이라 합니다. 융합은 처리 단계에 따라 센서 레벨, 특징 레벨, 점수 레벨, 결정 레벨로 구분됩니다.

센서 레벨 융합

원시 센서 데이터를 직접 결합하는 방식입니다. 2D 얼굴 이미지와 3D 깊이 맵을 픽셀 단위로 결합하는 것이 대표적입니다. 정보 손실이 가장 적지만, 센서 간 동기화와 정합이 필요합니다.

특징 레벨 융합

각 양식에서 추출한 특징 벡터를 연결(concatenation)하거나 결합하여 단일 특징 벡터를 생성합니다. 풍부한 정보를 활용할 수 있지만, 차원의 저주 문제와 특징 호환성 문제가 발생할 수 있습니다.

점수 레벨 융합 기법 
1. 단순 규칙 기반 융합
─────────────────────
Sum Rule:    S = s₁ + s₂ + ... + sₙ
Product Rule: S = s₁ × s₂ × ... × sₙ
Max Rule:    S = max(s₁, s₂, ..., sₙ)
Min Rule:    S = min(s₁, s₂, ..., sₙ)

- 계산 효율적
- 해석 용이
- 학습 불필요

2. 가중 합 융합
────────────
S = w₁s₁ + w₂s₂ + ... + wₙsₙ
where Σwᵢ = 1

- 양식별 신뢰도 반영
- 가중치 최적화 필요

3. 학습 기반 융합
────────────────
- SVM, Random Forest 등 분류기 사용
- 점수 벡터를 입력으로 학습
- 비선형 결합 가능
- 최적 융합 함수 학습

4. 밀도 추정 융합
────────────────
- Likelihood Ratio 기반
- GMM으로 분포 모델링
- 확률적 프레임워크

다중 모달 시스템 구현

import numpy as np
from typing import List, Tuple, Dict
from sklearn.preprocessing import MinMaxScaler
from sklearn.svm import SVC

class MultimodalBiometricSystem:
    """다중 모달 생체인증 시스템"""
    
    def __init__(self, modalities: List[str], fusion_strategy: str = 'score'):
        """
        Args:
            modalities: 사용할 생체 양식 리스트 ['fingerprint', 'face', 'voice']
            fusion_strategy: 융합 전략 ('feature', 'score', 'decision')
        """
        self.modalities = modalities
        self.fusion_strategy = fusion_strategy
        self.unimodal_systems = {}
        self.fusion_weights = {}
        self.fusion_model = None
        self.scaler = MinMaxScaler()
        
    def initialize_modalities(self, systems: Dict):
        """각 양식별 인식 시스템 초기화"""
        for modality in self.modalities:
            if modality in systems:
                self.unimodal_systems[modality] = systems[modality]
                self.fusion_weights[modality] = 1.0 / len(self.modalities)
    
    def enroll(self, user_id: str, biometric_data: Dict) -> bool:
        """다중 모달 등록"""
        success_count = 0
        for modality in self.modalities:
            if modality in biometric_data:
                system = self.unimodal_systems[modality]
                if system.enroll(user_id, biometric_data[modality]):
                    success_count += 1
        
        # 최소 하나 이상의 양식 등록 필요
        return success_count >= 1
    
    def verify_score_fusion(self, user_id: str, 
                            biometric_data: Dict) -> Tuple[bool, float]:
        """점수 레벨 융합 검증"""
        scores = []
        weights = []
        
        for modality in self.modalities:
            if modality in biometric_data and modality in self.unimodal_systems:
                system = self.unimodal_systems[modality]
                is_match, score = system.verify(user_id, biometric_data[modality])
                scores.append(score)
                weights.append(self.fusion_weights[modality])
        
        if not scores:
            return False, 0.0
        
        # 점수 정규화 (0-1 범위)
        scores = np.array(scores)
        weights = np.array(weights)
        weights = weights / weights.sum()  # 정규화
        
        # 가중 합 융합
        fused_score = np.sum(scores * weights)
        
        # 결정 임계값
        threshold = 0.5
        return fused_score >= threshold, fused_score
    
    def verify_feature_fusion(self, user_id: str, 
                              biometric_data: Dict) -> Tuple[bool, float]:
        """특징 레벨 융합 검증"""
        features = []
        
        for modality in self.modalities:
            if modality in biometric_data:
                system = self.unimodal_systems[modality]
                feature = system.extract_features(biometric_data[modality])
                features.append(feature)
        
        # 특징 연결 (Concatenation)
        fused_feature = np.concatenate(features)
        
        # 융합된 특징으로 매칭
        # (실제로는 등록된 융합 템플릿과 비교)
        score = self._match_fused_features(user_id, fused_feature)
        
        return score >= 0.5, score
    
    def verify_decision_fusion(self, user_id: str, 
                               biometric_data: Dict) -> Tuple[bool, float]:
        """결정 레벨 융합 검증"""
        decisions = []
        confidences = []
        
        for modality in self.modalities:
            if modality in biometric_data:
                system = self.unimodal_systems[modality]
                is_match, score = system.verify(user_id, biometric_data[modality])
                decisions.append(is_match)
                confidences.append(score)
        
        # 다수결 투표 (Majority Voting)
        accept_votes = sum(decisions)
        total_votes = len(decisions)
        
        # AND 규칙: 모든 양식 통과 필요
        # final_decision = all(decisions)
        
        # OR 규칙: 하나라도 통과
        # final_decision = any(decisions)
        
        # 과반수 규칙
        final_decision = accept_votes > total_votes / 2
        
        # 평균 신뢰도
        avg_confidence = np.mean(confidences) if confidences else 0.0
        
        return final_decision, avg_confidence
    
    def train_fusion_model(self, training_data: List[Dict], labels: List[int]):
        """학습 기반 점수 융합 모델 훈련"""
        # 각 양식의 점수를 수집
        score_vectors = []
        for sample in training_data:
            scores = []
            for modality in self.modalities:
                if modality in sample:
                    scores.append(sample[modality]['score'])
            score_vectors.append(scores)
        
        # 점수 정규화
        score_vectors = np.array(score_vectors)
        score_vectors = self.scaler.fit_transform(score_vectors)
        
        # SVM 분류기 훈련
        self.fusion_model = SVC(probability=True, kernel='rbf')
        self.fusion_model.fit(score_vectors, labels)
    
    def verify(self, user_id: str, biometric_data: Dict) -> Tuple[bool, float]:
        """융합 전략에 따른 검증"""
        if self.fusion_strategy == 'score':
            return self.verify_score_fusion(user_id, biometric_data)
        elif self.fusion_strategy == 'feature':
            return self.verify_feature_fusion(user_id, biometric_data)
        elif self.fusion_strategy == 'decision':
            return self.verify_decision_fusion(user_id, biometric_data)
        else:
            raise ValueError(f"Unknown fusion strategy: {self.fusion_strategy}")

성능 향상 분석

다중 모달 융합을 통한 성능 향상은 수학적으로 분석할 수 있습니다. 독립적인 두 생체 양식 A, B의 오류율이 각각 FAR_A, FAR_B일 때, AND 규칙으로 결합하면 전체 FAR은 FAR_A x FAR_B로 크게 감소합니다.

융합 규칙별 오류율 분석 
독립 양식 가정: 지문 (FAR=0.01%, FRR=0.5%), 얼굴 (FAR=0.1%, FRR=1%)

AND 규칙 (직렬 결합)
────────────────────
FAR_fused = FAR_A × FAR_B = 0.0001 × 0.001 = 0.0000001 (0.00001%)
FRR_fused = 1 - (1-FRR_A)(1-FRR_B) = 1 - 0.995 × 0.99 = 1.495%

→ FAR 대폭 감소, FRR 소폭 증가
→ 고보안 환경에 적합

OR 규칙 (병렬 결합)
────────────────────
FAR_fused = 1 - (1-FAR_A)(1-FAR_B) = 1 - 0.9999 × 0.999 = 0.10999%
FRR_fused = FRR_A × FRR_B = 0.005 × 0.01 = 0.00005 (0.005%)

→ FRR 대폭 감소, FAR 소폭 증가
→ 사용자 편의 우선 환경에 적합

가중 합 규칙 (최적화)
─────────────────────
점수 분포 분석 후 최적 가중치 결정
EER 최소화 목표 함수로 최적화
일반적으로 AND/OR 사이의 균형점
