5장: Phase 2 — API 인터페이스

장 개요: REST + OpenAPI 3.1 façade와 GS1 EPCIS 2.0 capture/query façade를 같은 상태에서 연결하고, EUDR DDS·MAFRA 어댑터·CloudEvents 1.0 웹훅·Iceberg 스냅샷 헤더까지 호출 단위로 보여준다.

5.1 두 façade, 같은 진실

구현은 다음 두 인터페이스를 동시에 노출해야 한다.

  1. REST + OpenAPI 3.1(/v1/...) — 동기 CRUD. 매장 운영자 앱·도매 백오피스에 적합.
  2. EPCIS 2.0 REST(/epcis/v2/...) — 이벤트 소싱. 농가↔도매↔소매 cross‑org 이력 교환에 표준.

https://conformance.wia-supply-chain.org에서 동일 시나리오를 두 façade로 동시 실행, 결과 해시 일치 여부로 인증한다.

5.3 출하 생성 + 멱등성

curl -X POST https://api.wia-supply-chain.org/v1/shipments \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -H "Idempotency-Key: 4f1d3c0e-7b2c-4a98-9c7e-2c0e9b8a4a01" \
  -d @shipment.json

Idempotency-Key(draft‑ietf‑httpapi‑idempotency‑key)는 산간 지역 통신 단절 후 자동 재시도 시에도 중복 출하 생성을 막는다. 서버는 24시간 동안 응답을 캐시한다.

5.4 동시성 제어 — ETag/If‑Match

PATCH /v1/shipments/SHIP-2026-0421
Content-Type: application/merge-patch+json
If-Match: "f3a2e1c"

{ "status": "DELIVERED", "deliveredAt": "2026-04-23T17:42:11+09:00" }

RFC 7396 merge‑patch는 wire 트래픽을 줄이고, If-Match는 lost‑update 사고를 차단한다.

5.7 콜드체인 ingest는 Iceberg 스냅샷을 만든다

POST /v1/cold-chain/data
→ HTTP/1.1 201 Created
   X-Snapshot-Id: 7841029384712340987
   ETag: "snap-7841029384712340987"

다운스트림 분석 잡(Trino, DuckDB, Spark)은 X-Snapshot-Id로 스냅샷‑격리 스캔을 한다 — 롤링 ingest 도중에도 더티 리드 없이 일관된 결과.

장 요약

핵심 요점

  1. REST와 EPCIS 두 façade는 동치성을 가져야 하며, 인증 시 결과 해시까지 비교된다.
  2. Idempotency‑Key + ETag/If‑Match는 산간·해상 환경의 재시도와 동시 갱신을 안전하게 만든다.
  3. 오류는 RFC 9457 problem+json으로 일관, type URI는 클라이언트 처리의 안정 키다.
  4. 콜드체인 ingest 응답마다 X-Snapshot-Id가 따라온다 — Iceberg 스냅샷 격리가 “옵션”이 아니다.
  5. EUDR DDS·MAFRA 어댑터는 표준의 1급 엔드포인트로, 규제 시스템이 그대로 소비한다.
