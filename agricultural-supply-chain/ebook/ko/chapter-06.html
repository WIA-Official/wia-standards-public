6장: Phase 3 — 프로토콜

장 개요: 데이터 평면을 정의한다. MQTT 5(OASIS, 2019)로 운송 차량 텔레메트리, CoAP + OSCORE로 배터리 제약 BLE 로거, LoRaWAN 1.1로 산간·도서 농가, Hyperledger Fabric / WIA Chain PoA로 변조 불가능 앵커링을 묶는다.

6.1 MQTT 5 — 운송 차량 텔레메트리 베이스라인

한국 도로 환경(터널·페리·산간 음영)이 MQTT 5의 세 가지 신규 속성을 사실상 필수화한다.

  • Session Expiry Interval — 차량이 음영 구간을 벗어나도 브로커 세션이 살아 있다.
  • Payload Format Indicator — CBOR 인코딩된 센서 프레임과 JSON 대시보드 트래픽을 라우팅 단계에서 분기.
  • AUTH 패킷 + Reason Codes — 게이트웨이 로그에 인증 실패 사유가 직접 남는다(RFC 7635 SCRAM 브리지).

토픽 구조

wia/agri/{tenant}/shipment/{shipmentId}/temperature
wia/agri/{tenant}/shipment/{shipmentId}/humidity
wia/agri/{tenant}/shipment/{shipmentId}/gps
wia/agri/{tenant}/shipment/{shipmentId}/shock
wia/agri/{tenant}/shipment/{shipmentId}/alerts
wia/agri/{tenant}/sensor/{deviceId}/state

센서 프레임(CBOR로 인코딩되는 JSON 모양)

{
  "deviceId":"BLE-LOG-77AA",
  "ts":"2026-04-22T14:31:08+09:00",
  "data":{"temperatureC":4.2,"humidityPct":74,"shockG":0.18},
  "meta":{"battMv":3015,"sigQual":-78,"firmware":"1.4.2"}
}

6.2 CoAP + OSCORE — 배터리 제약 로거

BLE 코인셀(CR2032) 로거는 TLS 핸드셰이크 비용을 감당하지 못한다. CoAP(RFC 7252) over UDP + OSCORE(RFC 8613)는 200 byte 미만 오버헤드로 인증·암호화 요청을 보낸다.

CON POST /shipment/SHIP-2026-0421/temperature
Content-Format: application/cbor
OSCORE: kid=0x01,iv=0x...
Payload (CBOR): {"t":4.2,"u":"C","ts":1714032668}

Block‑wise transfer(RFC 7959)로 1KB 초과 GPS 트레이스를 IP 단편화 없이 옮기고, Observe(RFC 7641)로 운영자 포털이 단일 센서를 폴링 없이 구독한다.

6.3 LoRaWAN 1.1 — 산간·도서 농가

제주 한라봉 농가, 강원 옥수수 산지, 전남 도서지역은 셀룰러 음영이 잦다. LoRaWAN 1.1은 KR920 대역에서 수 km 거리 업링크를 1mA 미만 평균 전류로 처리한다.

DevEUI:    0018B20000000001
JoinEUI:   70B3D57ED0000000
NwkSEncKey: 세션마다 회전 (LoRaWAN 1.1)
DataRate:  SF7BW125 (도시) | SF12BW125 (산간 음영)
Frequency: 920 900 000 Hz (KR920) | 868 100 000 Hz (EU868)

네트워크 서버(ChirpStack, KT MTN, SK 텔레콤 LoRa)는 디코드된 프레임을 §6.1과 같은 MQTT 토픽으로 재발행한다.

6.4 변조 불가능한 앵커링

Hyperledger Fabric chaincode

async function recordShipment(ctx, shipmentJson) {
  const s = JSON.parse(shipmentJson);
  const canon = require('canonicalize')(s);                      // RFC 8785
  const key = ctx.stub.createCompositeKey('SHIP', [s.shipmentId]);
  await ctx.stub.putState(key, Buffer.from(canon));
  await ctx.stub.setEvent('AGRI_SHIPMENT', Buffer.from(s.shipmentId));
  return s.shipmentId;
}

endorsement 정책 AND('Producers.member','Carriers.member','Retailers.member') — 어느 한 조직도 단독으로 이력을 다시 쓸 수 없다.

WIA Chain PoA + BSC 앵커링

WIA Chain은 30초 단위로 모든 ingest의 머클 루트를 BSC 메인넷에 앵커링한다. 외부 감사자는 비공개 데이터에 접근하지 않고도 시점 무결성을 검증한다.

6.5 실시간 스트리밍

const ws = new WebSocket('wss://stream.wia-supply-chain.org/v1?shipment=SHIP-2026-0421');
ws.onmessage = (e) => {

6.7 프로토콜 계층이 다뤄야 할 실패 모드

  • 브로커 다운 — 게이트웨이는 24시간 로컬 버퍼링, 원본 ts로 리플레이.
  • 시계 드리프트 — 6시간마다 NTP 동기, |now − ts| > 300 s 표본 거절.
  • 리플레이 — OSCORE 시퀀스 윈도, JWT JTI 캐시.
  • 운송 중 센서 교체 — EPCIS 2.0 AssociationEvent로 명시 — 감사가 왜‑언제를 추적 가능.

장 요약

핵심 요점

  1. MQTT 5의 Session Expiry/Payload Format/AUTH 세 속성이 한국 도로 환경의 음영·터널 문제를 푼다.
  2. CoAP + OSCORE는 200 byte 미만으로 인증·암호화된 센서 프레임을 운반한다.
  3. LoRaWAN 1.1 KR920은 산간·도서 농가의 음영을 메우며, 같은 MQTT 토픽 트리에 흘려 보낸다.
  4. Fabric의 다중 조직 endorsement + WIA Chain PoA의 BSC 앵커링이 두 단위에서 무결성을 보장한다.
  5. TLS 1.3·OSCORE·Ed25519·HSM JWKS — 보안 베이스라인은 “옵션 X”가 없다.

  3. 한국에서 LoRaWAN 적용 시 권장 주파수 대역과 두 데이터레이트 선택은?

  5. 시계 드리프트로 인한 리플레이를 어떻게 거르는가?
