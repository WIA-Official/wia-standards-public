4장: Phase 1 — 데이터 형식

장 개요: GS1 식별자 부여 규칙, JSON Schema 2020‑12 본 레코드, HL7 FHIR R5 Provenance/ProductShelfLife 프로파일, EUDR용 GeoJSON 폴리곤, Apache Iceberg 파티션 전략을 한 자리에 정렬한다.

4.3 콜드체인 텔레메트리

{
  "shipmentId":"SHIP-2026-0421",
  "deviceId":"BLE-LOG-77AA",
  "deviceModel":"Sensitech TempTale Ultra",
  "timestamp":"2026-04-22T14:31:08+09:00",
  "temperatureC":4.2,"humidityPct":74,
  "shockG":0.18,"tiltDeg":3.2,
  "gps":{"lat":36.34,"lon":127.40,"hdop":1.2},
  "battery":{"voltageMv":3015,"percent":78}
}

이 레코드는 Apache Iceberg v2 테이블 agri.cold_chain.readings에 append‑only로 적재된다. 파티션 변환은 hours(timestamp) + bucket(16, shipmentId) 조합. 단일 화물 시계열 조회 시 매니페스트 16개 이내만 스캔하므로, 12개월 누적 후에도 P95 응답 시간 200 ms 이내가 유지된다.

4.4 프로비넌스 (HL7 FHIR R5 정렬)

수확·세척·등급·포장·출하·입고·회수 등 법적 책임이 발생하는 모든 상태 변화는 FHIR Provenance 리소스(https://wia-supply-chain.org/StructureDefinition/AgriProvenance 프로파일)로 기록된다.

  • occurredDateTime — 시각, ISO 8601 + KST(+09:00).
  • activity — https://wia-supply-chain.org/cs/agri-stage 코드(harvest|wash|grade|pack|ship|receive|recall).
  • agent.who — GLN 기반 운영자 식별자.
  • signature — RFC 8785(JCS) 정규화된 JSON에 대한 Ed25519 detached JOSE 서명.

{
  "type":"Feature",
  "geometry":{"type":"Polygon","coordinates":[[
    [126.5310,33.4990],[126.5325,33.4990],
    [126.5325,33.5005],[126.5310,33.5005],[126.5310,33.4990]

  "properties":{
    "plotId":"PLT-JJ-001","areaHa":1.4,

EU EUDR(2023/1115)은 통관 시 폴리곤이 없으면 거절한다. 한국 산림청 위성 자료(GIS국토포털)와 ESA Sentinel‑2 GLAD‑S2 alert 오버레이로 위험도를 평가해 폴리곤 properties에 동봉한다.

4.8 검증 규칙(Validation)

  • 모든 시각은 ISO 8601‑1 또는 ‑2, 시간대 오프셋 명시.
  • 좌표는 WGS‑84 십진도, 소수점 7자리 이하.
  • 온도는 항상 단위 동반(C 또는 F), 한 출하 내 단위 혼용 금지.
  • GTIN 모듈러10 체크섬, SSCC 18자리 체크섬 검증.
  • 서명 대상은 RFC 8785(JCS) 정규화된 JSON만 인정.

장 요약

핵심 요점

  1. GS1 GTIN/SSCC/GLN/Lot/GIAI는 wire에 그대로 등장하며, 자체 식별 체계는 매핑 부속물로만 허용된다.
  2. JSON Schema 2020‑12 + GeoJSON RFC 7946 + FHIR R5 프로파일이 데이터 어휘를 구성한다.
  3. 콜드체인 시계열은 Iceberg v2 테이블의 시간·해시 파티션 위에 산다.
  4. 프로비넌스 서명은 RFC 8785 정규화 JSON에 대한 Ed25519 JOSE detached 서명이다.
  5. EUDR용 폴리곤은 본 표준의 1급 시민이며, 별도 파이프라인을 두지 않는다.
