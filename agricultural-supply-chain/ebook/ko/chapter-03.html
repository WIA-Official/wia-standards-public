3장: WIA‑AGRI‑014 전체 구조도

장 개요: 5계층 아키텍처, 4단계(Phase) 구현 흐름, 3가지 핵심 원칙, 그리고 WIA 생태계와의 결합 — 한 장 안에서 전체 그림을 보여준다.

3.1 5계층 아키텍처

┌─────────────────────────────────────┐
│  소비자 앱 (QR, 매대 라벨, 검증 페이지)│   ← Phase 4
├─────────────────────────────────────┤
│  업무 로직 (REST + EPCIS, 분석, ESG) │   ← Phase 2/4
├─────────────────────────────────────┤
│  프로토콜 (MQTT 5, CoAP, LoRaWAN)    │   ← Phase 3
├─────────────────────────────────────┤
│  데이터 (JSON Schema, Iceberg, 블록체인)│ ← Phase 1
├─────────────────────────────────────┤
│  디바이스 (BLE 로거, RFID, 냉장차 게이트웨이)│
└─────────────────────────────────────┘

각 계층은 직상위 계층에 단일 책임 인터페이스만 노출한다. 디바이스 계층 교체(예: BLE 로거 → eSIM 셀룰러 로거)는 프로토콜 계층의 어댑터만 손보면 끝나야 한다.

3.2 세 가지 핵심 원칙

3.2.1 상호운용성을 우선으로 (Interoperability First)

모든 식별자는 GS1, 모든 의미 코드는 CBV 1.2.2 / LOINC, 모든 메시지는 OpenAPI 3.1 / EPCIS 2.0. 자체 식별 체계는 표준 식별자에 매핑되는 한에서만 허용된다.

3.2.2 변조 불가능성 (Tamper Evidence)

핵심 이벤트(수확·가공·출하)는 Hyperledger Fabric 다중 조직 endorsement에 기록되고, 그 머클 루트가 WIA Chain PoA를 통해 BSC 메인넷에 30초 단위로 앵커링된다.

3.2.3 설계로서의 프라이버시 (Privacy by Design)

가격·계약 같은 영업 비밀은 사적(private) 채널, 원산지·인증 같은 공개 정보는 공적(public) 채널로 분리한다. zk‑SNARK 기반 selective disclosure로 “유기 인증된 한라봉이지만 공급가는 비공개” 같은 표현을 wire에서 그대로 다룬다.

3.4 핵심 컴포넌트 한눈에

  • 출하 추적 — SHIP‑id를 키로 한 상태 기계, EPCIS 이벤트 시계열로 동치성 검증.
  • 콜드체인 모니터링 — Iceberg v2 테이블에 5분 partition, P95 200ms 응답.
  • 블록체인 프로비넌스 — Hyperledger Fabric + WIA Chain PoA 이중 앵커링.
  • 품질·검사 기록 — LOINC 코드 부여된 실험실 결과를 FHIR Observation으로 직접 매핑.
  • 소비자 검증 — verify URL + WIA‑VC 서명 검증 + 회수 시 즉시 차단 화면.

3.6 WIA 생태계 결합

  • WIA‑BLOCKCHAIN — 프로비넌스 기록과 검증의 베이스 레이어.
  • WIA‑VC — 친환경·GAP·할랄·코셔 인증을 W3C VC 2.0으로 발급.
  • WIA‑IoT — 디바이스 등록·키 회전·OTA 업데이트.
  • WIA‑CARBON — 콜드체인 운송 ISO 14083:2023 Scope 3 자동 계산.
  • WIA‑IDENTITY — 농가·검수원·운송기사 신원 관리(DID).

3.7 “시스템”이 “표준”이 되는 순간

한 회사의 시스템은 그 회사의 데이터로 끝난다. 본 표준은 EPCIS 2.0 capture 메시지가 식약처 회수 시스템으로, MAFRA 이력번호로, EU TRACES NT로, 미국 FDA 트레이서빌러티 로그로 같은 의미를 그대로 옮겨가는 동치성을 보장한다. 외부 시스템이 비트 단위로 같은 의미를 받아들이게 되는 그 지점이 표준의 시작점이다.

장 요약

핵심 요점

  1. 5계층(디바이스·데이터·프로토콜·업무·소비자)이 단일 책임 인터페이스로 수직 분리된다.
  2. 상호운용성·변조 불가능성·프라이버시 — 세 원칙이 모든 설계 결정의 기준이다.
  3. 4단계(Phase) 로드맵은 직전 단계 산출물을 입력으로 받는다 — 건너뛰지 말 것.
  4. WIA 생태계(블록체인·VC·IoT·CARBON·IDENTITY)와 결합해 단일 시스템보다 훨씬 큰 가치를 만든다.
  5. 외부 시스템이 같은 비트를 같은 의미로 받아들일 때 비로소 “표준”이다.
