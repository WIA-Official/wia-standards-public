Chapter 6: Phase 3 — Protocol

Overview: Phase 3 wires the data plane: MQTT 5 (OASIS, 2019) for reefer trucks and farm gateways, CoAP (RFC 7252) + OSCORE (RFC 8613) for battery-bound loggers, LoRaWAN 1.1 for cross-border last-mile, and Hyperledger Fabric / WIA Chain PoA for tamper-evident anchoring.

6.1 MQTT 5 for fleet telemetry

MQTT 5 is the baseline for reefer telemetry because it adds three properties a cold chain depends on:

  • Session Expiry Interval — carriers crossing dead zones (mountain tunnels, ferries) keep their session alive on the broker.
  • Payload Format Indicator — brokers route binary CBOR-encoded sensor frames separately from JSON dashboards.
  • Reason Codes & AUTH packet — surface enhanced authentication failures to the gateway logs (RFC 7635 SCRAM bridges).

Topic structure

6.2 CoAP + OSCORE for battery-bound loggers

BLE temperature loggers powered by a CR2032 cell cannot afford TLS handshakes. CoAP (RFC 7252) over UDP with OSCORE (RFC 8613) provides authenticated, encrypted requests at <200 bytes overhead:

CON POST /shipment/SHIP-2026-0421/temperature
Content-Format: application/cbor
OSCORE: kid=0x01,iv=0x...
Payload (CBOR): {"t":4.2,"u":"C","ts":1714032668}

Block-wise transfers (RFC 7959) carry GPS traces > 1 KB without IP fragmentation. Observe (RFC 7641) lets the operator portal subscribe to a single sensor without polling.

6.3 LoRaWAN 1.1 for last-mile and remote farms

Smallholder farms on Jeju, the Andes coffee belt, and the Mekong delta cannot rely on cellular. LoRaWAN 1.1 (LoRa Alliance, 2017 + 2020 errata) provides 868/915 MHz long-range uplinks:

DevEUI:    0018B20000000001
JoinEUI:   70B3D57ED0000000
NwkSEncKey:rotated per session (LoRaWAN 1.1)
DataRate:  SF7BW125 (urban) | SF12BW125 (mountain shadow)
Frequency: 868 100 000 Hz (EU868) | 920 900 000 Hz (KR920)

Network servers (ChirpStack, The Things Network) forward decoded frames into the WIA broker as MQTT 5 messages, preserving the QoS choice from §6.1.

6.4 Tamper-evident anchoring

Hyperledger Fabric chaincode

async function recordShipment(ctx, shipmentJson) {
  const s = JSON.parse(shipmentJson);
  const canon = require('canonicalize')(s);                      // RFC 8785
  const hash = ctx.stub.createCompositeKey('SHIP', [s.shipmentId]);
  await ctx.stub.putState(hash, Buffer.from(canon));
  await ctx.stub.setEvent('AGRI_SHIPMENT', Buffer.from(s.shipmentId));
  return s.shipmentId;
}

Endorsement policy: AND('Producers.member','Carriers.member','Retailers.member') — no single party can rewrite history.

WIA Chain PoA anchoring

WIA Chain (Phase-3 reference) anchors a Merkle root of every 30-second ingest window onto BSC mainnet, giving public auditors a one-click attestation without exposing private payloads.

6.5 Real-time streaming to the operator portal

WebSocket (RFC 6455) gateway re-publishes broker-side messages with backpressure-aware fan-out. The portal subscribes per-shipment:

const ws = new WebSocket('wss://stream.wia-supply-chain.org/v1?shipment=SHIP-2026-0421');
ws.onmessage = (e) => {
  const evt = JSON.parse(e.data);
  if (evt.type === 'cold_chain.violation') alarm.fire(evt);
};

6.6 Security baseline

LayerRequired
Transport (gateway ↔ broker)TLS 1.3 with TLS_AES_256_GCM_SHA384 or TLS_CHACHA20_POLY1305_SHA256
End-device (logger ↔ gateway)OSCORE (RFC 8613) on CoAP, AES-128-CCM on LoRaWAN
Application signingEd25519 (RFC 8032) over RFC 8785 canonical JSON
Token formatJWT with EdDSA, audience-restricted, 30 min lifetime
Key managementHSM-backed, RFC 7517 JWKS rotation, 90-day max

6.7 Failure modes the protocol layer must handle

  • Broker outage — gateways MUST buffer up to 24 h locally; replay with original timestamps.
  • Clock drift — NTP every 6 h; server rejects readings with |now - ts| > 300 s.
  • Replay attacks — OSCORE sequence number window + JTI cache for JWTs.
  • Sensor swap mid-shipment — emit AssociationEvent (EPCIS 2.0) so audit can reconcile.

Chapter Summary

Key Takeaways

  1. MQTT 5 brings session expiry, payload-format indicator, and AUTH packet — all needed by reefer fleets.
  2. CoAP + OSCORE secures < 200 byte sensor frames without a TLS handshake.
  3. LoRaWAN 1.1 carries telemetry from rural farms; gateways adapt frames into the same MQTT topic tree.
  4. Hyperledger Fabric multi-org endorsement + WIA Chain PoA anchoring give tamper evidence at two scales.
  5. TLS 1.3, OSCORE, Ed25519, and HSM-backed JWKS define the baseline; everything else is an exception.
