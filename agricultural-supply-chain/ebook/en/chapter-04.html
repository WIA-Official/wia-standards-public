Chapter 4: Phase 1 — Data Format

Overview: The Phase-1 data format binds JSON Schema 2020-12 records to GS1 EPCIS 2.0 identifiers, FHIR R5 ProductShelfLife profiles, GeoJSON-encoded EUDR plots, and Apache Iceberg storage so that a single record carries authority-grade meaning across producers, carriers, retailers, and regulators.

Every traceable thing in WIA-AGRI-014 has at least one GS1 identifier:

4.2 Shipment record

A shipment is the smallest unit of accountability. Schema id https://schemas.wia-supply-chain.org/Shipment.json, JSON Schema draft 2020-12:

{
  "$schema": "https://json-schema.org/draft/2020-12/schema",
  "shipmentId":"SHIP-2026-0421",
  "sscc":"356123450000000017",
  "productInfo":{
    "gtin":"08801234567896","productName":"Jeju Organic Hallabong",
    "quantity":480,"unit":"kg","harvestDate":"2026-04-22",
    "lotNumber":"JJ-HBG-2026-W17","organic":true,
    "certifications":["EU_ORGANIC","KOREA_FRIENDLY"]
  },
  "origin":{"gln":"8801234500000","farmId":"FARM-KR-JJ-001",
            "latitude":33.4996,"longitude":126.5312},
  "destination":{"gln":"8801234500113","facilityId":"RETAIL-SEL-021",
                 "latitude":37.5665,"longitude":126.9780},
  "coldChainRequired":true,
  "targetTemperature":{"min":2,"max":8,"unit":"C"},
  "status":"IN_TRANSIT"
}

Status values follow CBV 1.2.2 dispositions: PENDING, AT_ORIGIN, IN_TRANSIT, AT_HUB, OUT_FOR_DELIVERY, DELIVERED, REJECTED, RECALLED.

4.3 Cold-chain telemetry record

Each sensor reading is an immutable row in the Iceberg v2 table agri.cold_chain.readings. The wire schema:

{
  "shipmentId":"SHIP-2026-0421",
  "deviceId":"BLE-LOG-77AA",
  "deviceModel":"Sensitech TempTale Ultra",
  "timestamp":"2026-04-22T14:31:08Z",
  "temperatureC":4.2,"humidityPct":74,
  "shockG":0.18,"tiltDeg":3.2,
  "gps":{"lat":36.34,"lon":127.40,"hdop":1.2},
  "battery":{"voltageMv":3015,"percent":78}
}

The Iceberg table partitions by hours(timestamp) with an additional bucket transform on shipmentId (16 buckets) so that range scans for a single shipment never touch more than 16 manifest files even after years of accumulation.

4.4 Provenance record (FHIR R5 aligned)

Every state change that creates legal accountability — harvest, sort, pack, ship, receive, recall — is captured as an FHIR R5 Provenance resource with the WIA AgriProvenance profile (https://wia-supply-chain.org/StructureDefinition/AgriProvenance). Required slices:

  • occurredDateTime — when it happened, ISO 8601 with timezone offset.
  • activity — coded with https://wia-supply-chain.org/cs/agri-stage (harvest | wash | grade | pack | ship | receive | recall).
  • agent.who — the GLN-rooted identifier of the responsible operator.
  • signature — JOSE-format Ed25519 detached signature over canonical JSON (RFC 8785).

4.5 Shelf-life and product definition

FHIR R5 ProductShelfLife attached to a SubstanceDefinition communicates open-date, after-pack, and after-thaw durations:

{
  "resourceType":"ProductShelfLife",
  "type":{"coding":[{"system":"http://hl7.org/fhir/CodeSystem/product-shelf-life",
                     "code":"shelfLifeAfterPacking"}]},
  "periodDuration":{"value":18,"unit":"d","system":"http://unitsofmeasure.org","code":"d"},
  "specialPrecautionsForStorage":[{"text":"Keep 2-8°C, 75-85% RH"}]
}

4.6 Geolocation: GeoJSON for EUDR plots

EU 2023/1115 (EUDR) requires geolocation of every plot of land where a regulated commodity originated. WIA-AGRI-014 carries plot polygons as GeoJSON Features (RFC 7946):

{
  "type":"Feature",
  "geometry":{"type":"Polygon","coordinates":[[
    [-79.5101,-1.2301],[-79.5050,-1.2301],
    [-79.5050,-1.2250],[-79.5101,-1.2250],[-79.5101,-1.2301]
  ]]},
  "properties":{"plotId":"PLT-001","areaHa":3.2,
                "deforestationCheck":{"basis":"GLAD-S2","risk":"NEGLIGIBLE"}}
}

4.8 Validation rules

  • All timestamps MUST be ISO 8601-1 (2026-04-22T14:31:08Z) or 8601-2 with offset.
  • Geographic coordinates use WGS-84 decimal degrees; precision ≤ 7 decimals.
  • Temperature values MUST carry the unit field; mixing C and F in the same shipment is forbidden.
  • GTIN MUST validate the GS1 mod-10 check digit; SSCC MUST be 18 digits including the check digit.
  • Signed payloads MUST canonicalize per RFC 8785 (JCS) before signature verification.

Chapter Summary

Key Takeaways

  1. GS1 identifiers (GTIN, SSCC, GLN) are mandatory on the wire — internal surrogates do not satisfy conformance.
  2. JSON Schema 2020-12 + GeoJSON RFC 7946 + FHIR R5 profiles compose the full record vocabulary.
  3. Cold-chain readings land in an Apache Iceberg v2 table partitioned for shipment-scoped fast scans.
  4. Provenance signatures are detached JOSE Ed25519 over RFC 8785 canonical JSON.
  5. EUDR-grade plot polygons travel inside the same record set, eliminating a parallel data pipeline.
