Chapter 5: Phase 2 — API Interface

Overview: Phase 2 binds the Phase-1 records to two equally normative HTTP façades — REST/OpenAPI 3.1 and GS1 EPCIS 2.0 — and connects them to authority systems (FDA FSMA, EU EUDR, Korea MAFRA) without duplicating data.

5.1 Two façades, one truth

An implementation MUST surface the same shipment state through both:

  1. REST + OpenAPI 3.1 at /v1/... — synchronous CRUD over JSON resources. Suitable for retailer dashboards and operator apps.
  2. EPCIS 2.0 REST at /epcis/v2/... — event-sourced wire protocol keyed by SGTIN/SSCC/GLN. Required for cross-organization interoperability.

Conformance suite (https://conformance.wia-supply-chain.org) executes the same trace scenarios against both façades and compares result hashes.

5.3 REST endpoints — the must-know set

Create shipment

curl -X POST https://api.wia-supply-chain.org/v1/shipments \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -H "Idempotency-Key: 4f1d3c0e-7b2c-4a98-9c7e-2c0e9b8a4a01" \
  -d @shipment.json

The Idempotency-Key (draft-ietf-httpapi-idempotency-key) lets carriers retry POSTs after network blips. The server caches responses for 24 h. Successful responses include Location and a strong ETag.

Patch with optimistic concurrency

PATCH /v1/shipments/SHIP-2026-0421
Content-Type: application/merge-patch+json
If-Match: "f3a2e1c"

{ "status": "DELIVERED", "deliveredAt": "2026-04-23T17:42:11Z" }

RFC 7396 merge-patch keeps the wire small; If-Match guards against lost-update.

Conditional read

GET /v1/shipments/SHIP-2026-0421
If-None-Match: "W/\"f3a2e1c\""
→ HTTP/1.1 304 Not Modified

5.6 Cold-chain ingest writes Iceberg snapshots

POST /v1/cold-chain/data
→ HTTP/1.1 201 Created
   X-Snapshot-Id: 7841029384712340987
   ETag: "snap-7841029384712340987"

Downstream analytics jobs (Trino, Spark, DuckDB) can read the table at X-Snapshot-Id for snapshot-isolated scans, eliminating dirty reads on rolling ingests.

Chapter Summary

Key Takeaways

  1. REST and EPCIS 2.0 are equally normative; both must surface identical state.
  2. Idempotency-Key + ETag/If-Match make retries and concurrent updates safe by design.
  3. Errors travel as RFC 9457 problem+json with a stable type URI for client handling.
  4. Cold-chain ingest commits to an Iceberg v2 table; the snapshot id is on every write response.
  5. EUDR DDS and MAFRA receipt endpoints turn the standard into a regulator-ready system out of the box.
