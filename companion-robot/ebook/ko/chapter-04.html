제4장: 1단계 - 데이터 형식

1단계는 반려 로봇 시스템의 기초를 정의합니다. 표준화된 데이터 형식을 통해 플랫폼 간 상호운용성과 데이터 이동성을 보장합니다.

4.1 반려 로봇 프로필 형식

4.1.1 기본 프로필 구조

interface CompanionProfile {
  // 식별 정보
  id: string;                    // UUID v7 (시간 정렬)
  version: string;               // 시맨틱 버전 (예: "1.0.0")
  createdAt: ISO8601;
  lastUpdated: ISO8601;

  // 정체성
  name: string;
  avatar?: string;               // Base64 또는 URI
  description: string;

  // 성격 (OCEAN+ 모델)
  personality: PersonalityTraits;

  // 구성
  primaryPurpose: CompanionPurpose;
  targetAgeRange: AgeRange;
  communicationStyle: CommunicationStyle;

  // 언어
  languagePrimary: string;       // ISO 639-1 (예: "ko", "en")
  languagesSupported: string[];

  // 기능
  capabilities: Capability[];
  limitations: string[];
}

4.1.2 OCEAN+ 성격 모델

OCEAN 모델 (Big Five)에 반려 로봇을 위한 확장 특성을 추가합니다.

interface PersonalityTraits {
  // 빅 파이브 (Big Five)
  openness: number;              // 개방성 (0-100)
  conscientiousness: number;     // 성실성 (0-100)
  extraversion: number;          // 외향성 (0-100)
  agreeableness: number;         // 친화성 (0-100)
  neuroticism: number;           // 신경성 (0-100)

  // 확장 특성
  playfulness: number;           // 장난기 (0-100)
  formality: number;             // 격식 (0-100)
  proactivity: number;           // 능동성 (0-100)
}
특성 낮은 점수 (0-30) 중간 점수 (40-60) 높은 점수 (70-100)
개방성 전통적, 실용적 균형적 창의적, 모험적
성실성 자발적, 유연함 적응적 조직적, 계획적
외향성 내성적, 조용함 중립적 사교적, 활동적
친화성 분석적, 객관적 균형적 공감적, 협조적
신경성 침착함, 안정적 균형적 민감함, 정서적
장난기 진지함 상황적 유머러스
격식 편안함, 캐주얼 상황적 정중함, 격식
능동성 반응적 균형적 선제적, 제안적

4.2 상호작용 메시지 형식

4.2.1 메시지 구조

interface InteractionMessage {
  // 메타데이터
  id: string;                    // UUID v7
  timestamp: ISO8601;
  sender: "user" | "companion";

  // 콘텐츠
  content: MessageContent;

  // 문맥
  context: MessageContext;

  // 메타데이터
  metadata: MessageMetadata;
}

interface MessageContent {
  text: string;
  language: string;              // ISO 639-1

  emotionalTone?: EmotionalTone;

  // 선택적 멀티모달 콘텐츠
  voice?: VoiceData;
  image?: ImageData;
}

interface MessageContext {
  conversationId: string;
  sessionId: string;
  threadId?: string;

  // 사용자 상태
  userEmotion?: EmotionState;
  userActivity?: string;
  userLocation?: string;

  // 시간적 문맥
  timeOfDay: "morning" | "afternoon" | "evening" | "night";
  dayOfWeek: string;
  specialOccasion?: string;
}

4.2.2 감정 톤 표현

VAD (Valence-Arousal-Dominance) 모델을 사용합니다.

interface EmotionalTone {
  valence: number;               // -1.0 (부정) ~ +1.0 (긍정)
  arousal: number;               // 0.0 (평온) ~ 1.0 (흥분)
  dominance: number;             // -1.0 (순종) ~ +1.0 (지배)
  intensity: number;             // 0.0 ~ 1.0
}
감정 Valence Arousal Dominance
기쁨 +0.8 0.7 +0.5
슬픔 -0.7 0.2 -0.5
분노 -0.6 0.9 +0.7
두려움 -0.8 0.8 -0.8
평온 +0.3 0.1 0.0

4.3 감정 인식 형식

4.3.1 감정 상태 표현

이산형(discrete)과 차원형(dimensional) 모델을 모두 지원합니다.

interface EmotionState {
  timestamp: ISO8601;

  // 이산형 감정 모델
  primaryEmotion: PrimaryEmotion;
  secondaryEmotions: EmotionProbability[];

  // 차원형 모델
  dimensions: EmotionalDimensions;

  // 신뢰도 및 출처
  overallConfidence: number;     // 0.0 ~ 1.0
  sources: EmotionSources;
}

enum PrimaryEmotion {
  JOY = "joy",               // 기쁨
  SADNESS = "sadness",       // 슬픔
  ANGER = "anger",           // 분노
  FEAR = "fear",             // 두려움
  SURPRISE = "surprise",     // 놀람
  DISGUST = "disgust",       // 혐오
  CONTEMPT = "contempt"      // 경멸
}

interface EmotionProbability {
  emotion: string;
  probability: number;           // 0.0 ~ 1.0
}

4.3.2 멀티모달 감정 인식

interface EmotionSources {
  text?: {
    emotion: PrimaryEmotion;
    confidence: number;
  };
  voice?: {
    emotion: PrimaryEmotion;
    confidence: number;
  };
  facial?: {
    emotion: PrimaryEmotion;
    confidence: number;
  };
}

융합 전략: 멀티모달 감정 인식은 다음 가중치를 사용합니다:

  • 음성: 40% (음조, 속도, 강도)
  • 얼굴: 35% (표정, 미세 표정)
  • 텍스트: 25% (단어 선택, 구문)

최종 신뢰도 = (음성 신뢰도 × 0.4) + (얼굴 신뢰도 × 0.35) + (텍스트 신뢰도 × 0.25)

4.6 데이터 검증 규칙

4.6.1 필수 검증 항목

필드 형식 검증 규칙
UUID string 유효한 UUID v7, 타임스탬프 인코딩
Timestamp ISO8601 유효한 ISO 8601, 타임존 포함 (UTC 권장)
성격 점수 number 0-100 범위, 정수
언어 코드 string 유효한 ISO 639-1 코드
감정 차원 number valence/dominance: -1.0~+1.0, arousal: 0.0~1.0
신뢰도 number 0.0~1.0 범위, 소수점 2자리
텍스트 string 최대 10,000자
Enum string 정의된 값만 허용

4.6.2 검증 코드 예시

// TypeScript 검증 함수 예시
function validatePersonalityTraits(traits: PersonalityTraits): boolean {
  const requiredTraits = [
    'openness', 'conscientiousness', 'extraversion',
    'agreeableness', 'neuroticism', 'playfulness',
    'formality', 'proactivity'
  ];

  for (const trait of requiredTraits) {
    const value = traits[trait];
    if (typeof value !== 'number' || value < 0 || value > 100) {
      return false;
    }
  }

  return true;
}

function validateEmotionalTone(tone: EmotionalTone): boolean {
  if (tone.valence < -1.0 || tone.valence > 1.0) return false;
  if (tone.arousal < 0.0 || tone.arousal > 1.0) return false;
  if (tone.dominance < -1.0 || tone.dominance > 1.0) return false;
  if (tone.intensity < 0.0 || tone.intensity > 1.0) return false;

  return true;
}

4.8 프라이버시 및 보안 요구사항

4.8.1 데이터 보호

요구사항 설명 표준
저장 시 암호화 모든 개인 데이터 암호화 AES-256
전송 시 암호화 모든 통신 암호화 TLS 1.3+
접근 제어 역할 기반 접근 제어 RBAC

4.8.2 민감 데이터 처리

특별 보호가 필요한 데이터:

  • 정신 건강 정보
  • 위기 상호작용 기록
  • 개인 관계 정보
  • 재정 정보
  • 건강 상태
  • 위치 데이터

추가 조치: 익명화, 접근 로깅, 정기 감사, 자동 삭제 정책

핵심 내용 요약

  • CompanionProfile: UUID v7, OCEAN+ 성격 모델 (8개 특성), 목적 및 대상 연령 정의
  • InteractionMessage: 멀티모달 콘텐츠, VAD 감정 톤, 문맥 정보 포함
  • EmotionState: 이산형(7개 기본 감정) + 차원형(VAD) 모델, 멀티모달 융합 (음성 40%, 얼굴 35%, 텍스트 25%)
  • LongTermMemory: 사용자 프로필, 5가지 메모리 유형, 중요도 기반 보존
  • CrisisDetection: 4단계 심각도, 6가지 위기 유형, 자원 제공 및 후속 조치
  • 데이터 검증: UUID v7, ISO 8601 타임스탬프, 0-100 성격 점수, ISO 639-1 언어 코드
  • 데이터 이동성: JSON (필수), CSV (권장), PDF (선택)
  • 보안: AES-256 암호화, TLS 1.3+, RBAC, GDPR 준수, 30일 이내 삭제
