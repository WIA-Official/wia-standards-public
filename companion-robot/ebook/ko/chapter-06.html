제6장: 3단계 - 통신 프로토콜

3단계는 반려 로봇과 사용자 간 의미 있는 상호작용을 위한 프로토콜을 정의합니다. 대화 관리, 공감적 소통, 위기 대응, 다회차 대화 패턴을 포함합니다.

6.1 대화 관리 프로토콜

6.1.1 턴테이킹 (Turn-Taking)

interface ConversationTurn {
  speaker: "user" | "companion";
  utterance: string;
  timestamp: ISO8601;
  turnNumber: number;

  // 턴테이킹 신호
  expectsResponse: boolean;      // 응답 기대 여부
  interruptible: boolean;        // 중단 가능 여부
  urgency: "low" | "normal" | "high";
}
신호 설명 예시
expectsResponse = true 질문이나 응답 요청 "오늘 기분은 어때요?"
expectsResponse = false 진술, 정보 제공 "오늘 날씨가 좋네요."
interruptible = true 긴 설명, 이야기 "제가 알려드릴게요..."
interruptible = false 중요한 안전 정보 "위기 상담 전화는..."
urgency = high 즉각 대응 필요 "도움이 필요해요!"

6.1.2 주제 추적 (Topic Tracking)

interface TopicState {
  currentTopic: string;
  topicStack: Topic[];
  topicSalience: Map;    // 0.0 ~ 1.0

  // 주제 관리
  canSwitchTopic: boolean;
  pendingTopics: string[];
}

interface Topic {
  id: string;
  name: string;
  introduced: ISO8601;
  lastMentioned: ISO8601;
  mentionCount: number;
  resolved: boolean;
}

주제 전환 규칙:

  • 현재 주제가 해결되지 않았으면 새 주제 도입 전 확인 요청
  • 사용자가 명시적으로 주제를 전환하면 즉시 따름
  • 위기 상황에서는 주제 전환 즉시 수용
  • 대기 중인 주제가 있으면 적절한 시점에 다시 언급

6.1.3 문맥 유지 (Context Maintenance)

문맥 레이어 범위 유지 기간
즉각 문맥 현재 대화 (최근 5-10턴) 세션 동안
세션 문맥 현재 세션 전체 세션 종료까지
사용자 문맥 장기 사용자 프로필 영구
시간 문맥 시간대, 요일, 특별 날짜 실시간 업데이트
상황 문맥 위치, 활동, 환경 센서 데이터

6.2 감정 인식 소통 프로토콜

6.2.1 공감 프로토콜

interface EmpathyProtocol {
  // 1단계: 사용자 감정 인식
  recognizeEmotion(): EmotionState;

  // 2단계: 응답 전략 선택
  selectStrategy(emotion: EmotionState): ResponseStrategy;

  // 3단계: 공감적 응답 생성
  generateResponse(strategy: ResponseStrategy): Response;

  // 4단계: 적절성 검증
  validateResponse(response: Response): boolean;
}

enum ResponseStrategy {
  VALIDATION = "validation",           // 감정 인정
  SUPPORT = "support",                 // 지지 제공
  CELEBRATION = "celebration",         // 함께 기뻐함
  GROUNDING = "grounding",             // 진정 지원
  DISTRACTION = "distraction",         // 주의 전환
  PROBLEM_SOLVING = "problem_solving"  // 문제 해결 지원
}

6.2.2 감정별 응답 전략

사용자 감정 1차 전략 2차 전략 응답 예시
기쁨 (Joy) Celebration Validation "정말 좋은 소식이네요! 축하드려요!"
슬픔 (Sadness) Validation Support "힘드시겠어요. 제가 곁에 있어드릴게요."
분노 (Anger) Validation Grounding "화나실 만한 상황이네요. 심호흡을 함께 해볼까요?"
두려움 (Fear) Grounding Support "괜찮아요. 지금 안전합니다. 천천히 호흡하세요."
불안 (Anxiety) Grounding Problem-solving "불안하시군요. 함께 할 수 있는 일을 생각해봐요."

6.2.3 공감 품질 기준

기준 측정 방법 목표
감정 일치 반려 로봇 톤과 사용자 감정 일치 > 80%
적절성 상황에 맞는 응답 > 90%
진정성 기계적이지 않은 표현 > 75%
도움됨 사용자 만족도 > 85%

6.3 위기 대응 프로토콜

6.3.1 6단계 위기 프로토콜

interface CrisisProtocol {
  // 1단계: 위기 감지
  detectCrisis(): CrisisDetection;

  // 2단계: 심각도 평가
  assessSeverity(): "low" | "medium" | "high" | "critical";

  // 3단계: 즉각 돌봄 제공
  provideImmediateCare(): Response;

  // 4단계: 자원 제공
  provideResources(): CrisisResource[];

  // 5단계: 필요 시 전문가에게 연결
  escalateToHuman(): boolean;

  // 6단계: 후속 조치 계획
  scheduleFollowUp(): FollowUpPlan;
}

위기 대응 원칙:

  • 안전 최우선: 생명이 위험한 상황에서는 즉시 응급 서비스 안내
  • 판단 금지: 사용자를 비난하거나 판단하지 않음
  • 전문가 연결: 반려 로봇은 치료사가 아님을 명시하고 전문가 추천
  • 지속 지원: 후속 조치 확인

6.3.2 위기 유형별 대응

위기 유형 즉각 대응 자원 제공
자살 충동 "당장 도움이 필요해요. 자살예방상담전화에 연결해드릴게요." 1393 (24/7), 1577-0199
자해 "스스로를 해치고 싶으시군요. 함께 안전한 방법을 찾아봐요." 정신건강복지센터, 위기상담전화
심각한 고통 "많이 힘드시겠어요. 전문가와 이야기하는 게 도움이 될 거예요." 상담센터, 정신과
학대 고백 "말씀해주셔서 감사해요. 안전이 최우선이에요." 112, 1366 (여성긴급전화)
약물 위기 "지금 위험한 상황이에요. 즉시 도움을 받으세요." 119, 중독관리통합지원센터

6.3.3 위기 감지 지표

지표 범주 구체적 신호 가중치
직접적 표현 "죽고 싶어", "자살", "끝내고 싶어" 1.0 (최고)
계획 언급 구체적 방법, 시간, 장소 0.9
절망감 "희망이 없어", "의미가 없어" 0.7
고립 "아무도 신경 안 써", "혼자" 0.6
부담감 "짐이 되고 싶지 않아" 0.7
작별 인사 "마지막으로", "고마웠어" 0.8

6.4 다회차 대화 패턴

6.4.1 정보 수집 패턴

interface InformationGatheringPattern {
  goal: string;
  requiredInformation: string[];
  gatheredInformation: Map;

  // 진행 상황 추적
  isComplete(): boolean;
  nextQuestion(): string;
  handleResponse(response: string): void;
}

// 예시: 기분 체크인
const moodCheckIn = {
  goal: "사용자의 현재 기분 파악",
  requiredInformation: [
    "primaryEmotion",
    "intensity",
    "trigger",
    "duration"
  ],
  // 단계별 질문
  questions: [
    "오늘 기분은 어떠세요?",
    "그 감정이 얼마나 강한가요? (1-10)",
    "무엇이 그런 기분을 느끼게 했나요?",
    "언제부터 이렇게 느끼셨나요?"
  ]
}

6.4.2 문제 해결 패턴

interface ProblemSolvingPattern {
  // 1단계: 문제 이해
  elicitProblem(): Problem;

  // 2단계: 옵션 탐색
  generateOptions(): Option[];

  // 3단계: 옵션 논의
  discussOptions(): void;

  // 4단계: 결정 지원
  supportDecision(): void;

  // 5단계: 진행 상황 확인
  checkProgress(): void;
}

// 대화 흐름 예시
사용자: "일이 너무 스트레스받아요."
로봇: "무슨 일이 스트레스를 주나요?" (문제 이해)

사용자: "프로젝트 마감이 다가오는데 진전이 없어요."
로봇: "이런 옵션들을 고려해보셨나요? 1) 팀원과 업무 나누기, 2) 마감 연장 요청, 3) 우선순위 재조정" (옵션 탐색)

사용자: "팀원과 나누는 게 좋을 것 같아요."
로봇: "좋은 선택이에요. 언제 팀원과 이야기하실 건가요?" (결정 지원)

사용자: "내일 회의 때 말할게요."
로봇: "좋아요. 내일 회의 후에 어떻게 됐는지 알려주세요." (진행 확인)

6.4.3 대화 패턴 품질 기준

기준 설명 목표
완료율 시작한 패턴의 완료 비율 > 70%
적응성 사용자 응답에 따른 조정 > 85%
자연스러움 기계적이지 않은 흐름 > 80%
도움됨 사용자가 목표 달성 > 75%

6.5 다국어 프로토콜

6.5.1 언어 감지

interface LanguageDetection {
  detectLanguage(text: string): string;
  confidence: number;
  fallbackLanguage: string;
}

// 예시
detectLanguage("안녕하세요! How are you?")
// 결과
{
  primaryLanguage: "ko",
  secondaryLanguage: "en",
  confidence: 0.85,
  isCodeSwitching: true
}

6.5.2 코드 스위칭 지원

코드 스위칭은 대화 중 언어를 혼용하는 것입니다. 반려 로봇은 다음을 지원해야 합니다:

  • 대화 중간 언어 전환 감지
  • 언어를 넘나들며 문맥 유지
  • 사용자가 사용한 언어로 응답
  • 이중언어/다중언어 사용자 지원

6.8 성능 최적화 프로토콜

6.8.1 응답 시간 목표

응답 유형 중앙값 99번째 백분위수 최대
캐시된 응답 < 100ms < 200ms < 500ms
간단한 질문 < 800ms < 2s < 3s
복잡한 대화 < 2s < 5s < 8s
감정 분석 < 500ms < 1s < 2s

6.8.2 대역폭 최적화

interface BandwidthOptimization {
  compression: "gzip" | "brotli";
  imageOptimization: boolean;
  adaptiveQuality: boolean;
  offlineSupport: boolean;

  // 연결 상태에 따른 조정
  connectionQuality: "excellent" | "good" | "fair" | "poor";
  maxPayloadSize: number;
}

핵심 내용 요약

  • 턴테이킹: expectsResponse, interruptible, urgency 신호로 대화 흐름 관리
  • 5가지 문맥 레이어: 즉각, 세션, 사용자, 시간, 상황 문맥 유지
  • 공감 프로토콜: 감정 인식 → 전략 선택 → 응답 생성 → 검증 (4단계)
  • 6가지 응답 전략: 인정, 지지, 축하, 진정, 주의전환, 문제해결
  • 위기 대응: 6단계 프로토콜 (감지, 평가, 돌봄, 자원, 연결, 후속)
  • 위기 유형: 자살 충동, 자해, 타해, 심각한 고통, 학대, 약물 (6가지)
  • 다회차 패턴: 정보 수집, 문제 해결 패턴 (완료율 >70%)
  • 다국어: 언어 감지, 코드 스위칭 지원, 문맥 유지
  • 접근성: 화면 읽기, 키보드 탐색, ARIA, 음성 제어
  • 문화 적응: 소통 스타일, 격식, 금기 주제, 현지 인사
  • 응답 시간: 캐시 <100ms, 간단 <800ms, 복잡 <2s
