5.1 API Architecture Overview

Phase 2 of WIA-ROB-013 defines comprehensive RESTful APIs that enable developers to build companion robot applications, integrate third-party services, and access robot capabilities in a standardized manner. The API architecture emphasizes security, developer experience, and extensibility.

5.1.1 API Design Principles

  • RESTful Architecture: Resource-oriented design with standard HTTP methods
  • Versioning: Explicit version management for backward compatibility
  • Authentication: OAuth 2.0 with role-based access control
  • Rate Limiting: Protective throttling to prevent abuse
  • Idempotency: Safe retry semantics for critical operations
  • Comprehensive Documentation: OpenAPI 3.0 specifications

5.1.2 Base API Endpoint Structure

https://api.companion-robot.wia.org/v1/{resource}

Example endpoints:
  GET    /v1/emotions
  POST   /v1/interactions
  GET    /v1/users/{userId}/profile
  PUT    /v1/behaviors/control
  DELETE /v1/sessions/{sessionId}

5.4 Behavior Control API

The Behavior API enables control of robot physical movements, expressions, and behavioral responses.

5.4.1 Execute Behavior

POST /v1/behaviors/execute

Request Body:
  {
    "behaviorType": "gesture|expression|movement|complex_behavior",
    "behaviorName": "empathetic_lean",
    "parameters": {
      "intensity": 0.7,
      "duration": 2.5,
      "smoothness": 0.8
    },
    "robotId": "robot-serial-number"
  }

Response (200 OK):
  {
    "executionId": "execution-uuid",
    "status": "executing",
    "estimatedDuration": 2.5,
    "startTime": "2025-01-15T14:30:00Z"
  }

5.4.2 Available Behaviors Catalog

Behavior Category Example Behaviors Parameters
Facial Expressions smile, frown, surprise, concern, neutral intensity, duration
Body Gestures nod, wave, lean_forward, clap, shrug intensity, speed, repetitions
Movements approach, retreat, turn, follow speed, distance, direction
Gaze Control look_at_user, look_away, track_face duration, smoothness
Complex Behaviors greet_enthusiastically, comfort_user, celebrate emotional_tone, personalization

5.6 Interaction History API

APIs for logging, querying, and analyzing interaction history between users and companion robots.

5.6.1 Log Interaction

POST /v1/interactions

Request Body:
  {
    "sessionId": "session-uuid",
    "userId": "user-uuid",
    "type": "conversation",
    "conversationData": {
      "userUtterance": {...},
      "robotResponse": {...}
    },
    "detectedEmotions": ["emotion-uuid-1"]
  }

Response (201 Created):
  {
    "interactionId": "interaction-uuid",
    "timestamp": "2025-01-15T14:30:00Z",
    "status": "logged"
  }

5.6.2 Query Interaction History

GET /v1/interactions?userId={userId}&startDate={start}&endDate={end}

Query Parameters:
  - userId: User identifier
  - startDate: ISO-8601 date (optional)
  - endDate: ISO-8601 date (optional)
  - type: Interaction type filter (optional)
  - limit: Results per page (default 50, max 200)
  - offset: Pagination offset

Response (200 OK):
  {
    "interactions": [...],
    "totalCount": 523,
    "limit": 50,
    "offset": 0,
    "hasMore": true
  }

5.9 Authentication and Authorization

WIA-ROB-013 uses OAuth 2.0 for secure API authentication with role-based access control.

5.9.1 OAuth 2.0 Flow

1. Authorization Request:
   GET /oauth/authorize?
       client_id={client_id}&
       redirect_uri={redirect_uri}&
       response_type=code&
       scope=emotions.read interactions.write

2. Token Exchange:
   POST /oauth/token
   Content-Type: application/x-www-form-urlencoded

   grant_type=authorization_code&
   code={authorization_code}&
   client_id={client_id}&
   client_secret={client_secret}

3. Response:
   {
     "access_token": "eyJhbGciOiJSUzI1NiIs...",
     "token_type": "Bearer",
     "expires_in": 3600,
     "refresh_token": "tGzv3JOkF0XG5Qx2TlKW",
     "scope": "emotions.read interactions.write"
   }

5.9.2 API Scopes and Permissions

Scope Permissions Typical Use Case
emotions.read Read emotion detection results Analytics dashboards, research applications
interactions.write Create interaction logs Companion apps, conversation interfaces
profile.manage Read/write user profiles Settings management, personalization apps
behaviors.control Execute robot behaviors Remote control apps, automation systems
health.read Access health data Healthcare integrations, monitoring dashboards

5.10 Rate Limiting and Quotas

To ensure fair usage and system stability, all API endpoints implement rate limiting.

5.10.1 Rate Limit Headers

Response Headers:
  X-RateLimit-Limit: 1000
  X-RateLimit-Remaining: 847
  X-RateLimit-Reset: 1642262400

When exceeded (429 Too Many Requests):
  {
    "error": "rate_limit_exceeded",
    "message": "Rate limit exceeded. Retry after 60 seconds.",
    "retryAfter": 60
  }

5.10.2 Rate Limit Tiers

Tier Requests/Hour Concurrent Connections
Free 1,000 5
Standard 10,000 25
Premium 100,000 100
Enterprise Custom Custom

Key Takeaways

  • Phase 2 APIs provide RESTful interfaces following industry best practices with OpenAPI specifications
  • Emotion Recognition API supports multi-modal emotion detection (facial, vocal, physiological)
  • Speech API enables natural language interaction with transcription, synthesis, and understanding
  • Behavior Control API allows programmatic control of robot movements, expressions, and gestures
  • Profile Management API enables deep personalization through preference configuration
  • Interaction History API provides comprehensive logging and querying of user-robot exchanges
  • Session Management API groups related interactions with outcome tracking
  • Health Monitoring API enables vital sign tracking with alert generation
  • OAuth 2.0 authentication with role-based access control ensures secure API usage
  • Rate limiting prevents abuse while ensuring fair resource allocation across users
