4.1 Data Format Foundation

Phase 1 of WIA-ROB-013 establishes standardized data structures that enable interoperability, data portability, and consistent privacy management across all companion robot implementations. These formats form the foundation upon which APIs, protocols, and integrations are built.

The data format specifications support three primary goals:

  • Interoperability: Different robot brands can understand and exchange data
  • Portability: Users can migrate their data between systems
  • Privacy: Consistent metadata enables automated privacy compliance

4.2 Emotion Data Schema

Emotion data represents the core of companion robot interactions, capturing detected emotional states, confidence levels, and contextual information.

4.2.2 Emotion Categories and Dimensions

Emotion Category Valence Range Arousal Range Typical Indicators
Joy 0.5 to 1.0 0.4 to 0.9 Smiling, laughter, upbeat tone
Sadness -1.0 to -0.3 -0.6 to 0.2 Frowning, slow speech, low energy
Anger -1.0 to -0.4 0.5 to 1.0 Furrowed brow, raised voice, tension
Fear -0.8 to -0.2 0.6 to 1.0 Wide eyes, elevated pitch, trembling
Surprise -0.3 to 0.7 0.7 to 1.0 Raised eyebrows, gasping, sudden movement
Disgust -1.0 to -0.5 0.2 to 0.6 Nose wrinkle, avoidance, rejection expressions
Neutral -0.2 to 0.2 -0.3 to 0.3 Relaxed features, calm demeanor

4.3.2 Interaction Types

Interaction Type Description Data Retention Default

4.9 Data Export and Portability Format

Users must be able to export their complete companion robot data in standardized formats for portability between systems.

4.9.1 Export Package Structure

Complete Data Export Contents

  • Profile Data: User profile, preferences, personalization settings
  • Interaction History: Complete conversation logs and activity records
  • Emotion Data: All detected emotions with metadata
  • Session Summaries: Aggregated session information
  • Health Records: Vital signs and activity tracking data (if applicable)
  • Media Assets: Encrypted audio/video recordings (if enabled)
  • Metadata: Privacy settings, consent records, audit trails

Key Takeaways

  • Phase 1 data formats establish interoperability through standardized JSON schemas
  • Emotion data uses dimensional model with discrete categories, confidence scores, and multi-modal sources
  • Interaction logs capture complete conversation history with privacy metadata
  • User profiles enable deep personalization through preferences, routines, and interests
  • Privacy metadata on every data object enables automated regulatory compliance
  • Data classification levels (public, personal, sensitive, anonymized) determine protection requirements
  • Session management groups interactions into coherent units for context awareness
  • Health data formats enable integration with healthcare ecosystems (HIPAA compliant)
  • Standardized event schemas ensure consistent emergency response and notifications
  • Data export packages enable full portability between companion robot systems
