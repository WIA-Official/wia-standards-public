2장: 데이터 스키마

2.1 EMG 데이터 프레임 구조

WIA 근전 표준은 다양한 센서 하드웨어, 처리 소프트웨어 및 의수 장치 간의 상호 운용성을 보장하는 포괄적인 EMG 신호 데이터 형식을 정의합니다.

2.1.1 최상위 구조

{
  "version": "1.0.0",
  "metadata": { ... },
  "channels": [ ... ],
  "startTime": 1705312800000,
  "duration": 1000
}
필드 타입 설명
version 문자열 WIA 표준 버전 (시맨틱 버저닝)
metadata 객체 기록 구성 및 장치 정보
channels 배열 샘플이 포함된 EMG 채널 객체 배열
startTime 정수 밀리초 단위 Unix 타임스탬프 (UTC)
duration 정수 밀리초 단위 기록 기간

2.2 메타데이터 객체

메타데이터 객체는 기록 구성에 대한 중요한 정보를 포함합니다:

필드 타입 필수 설명
sampleRate 정수 예 Hz 단위 샘플링 주파수 (1000-2000 권장)
channelCount 정수 예 EMG 채널 수 (일반적으로 2-8)
resolution 정수 예 비트 단위 ADC 해상도 (12-24)
gain 숫자 예 증폭 이득 계수 (1000-10000)
referenceType 문자열 예 전극 구성: monopolar, bipolar, differential
deviceId 문자열 아니오 센서 장치의 고유 식별자
firmwareVersion 문자열 아니오 장치 펌웨어 버전

2.2.1 참조 타입

타입 한국어 설명 사용 사례
monopolar 단극 원격 참조가 있는 단일 전극 범용, 노이즈에 취약
bipolar 이극 동일한 근육 위의 두 전극 표준 임상 사용 (권장)
differential 차동 공통 모드 제거가 있는 두 전극 고노이즈 환경

2.3 채널 객체

각 EMG 채널은 특정 근육 부위의 신호를 나타냅니다:

{
  "id": 0,
  "name": "flexor_carpi_radialis",
  "placement": { ... },
  "samples": [0.012, -0.008, 0.015, ...],
  "unit": "mV",
  "calibration": { ... }
}
필드 타입 필수 설명
id 정수 예 0부터 시작하는 채널 식별자
name 문자열 예 근육 이름 (snake_case 형식)
placement 객체 예 전극 배치 정보
samples 배열 예 전압 값의 숫자 배열
unit 문자열 예 측정 단위 (mV, μV)
calibration 객체 아니오 채널별 보정 데이터

2.4 전극 배치

배치 객체는 전극 위치를 문서화합니다:

2.4.1 지원되는 근육 이름

근육 이름 한국어 위치 기능
flexor_carpi_radialis 요측수근굴근 전완 손바닥쪽, 요측 손목 굴곡, 손 닫기
extensor_carpi_radialis 요측수근신근 전완 등쪽, 요측 손목 신전, 손 열기
flexor_digitorum 지굴근 전완 중앙 손바닥쪽 손가락 굴곡, 정밀 제어
extensor_digitorum 지신근 전완 중앙 등쪽 손가락 신전, 펴기
biceps_brachii 상완이두근 상완 전면 팔꿈치 굴곡, 회외
triceps_brachii 상완삼두근 상완 후면 팔꿈치 신전

2.5 보정 데이터

보정 객체는 신호 정규화를 위한 MVC(최대 수의 수축) 값을 저장합니다:

{
  "mvcValue": 2.5,
  "calibrationDate": 1705312800000,
  "expiryDate": 1705916400000,
  "method": "manual_mvc",
  "trials": 3
}
필드 타입 설명
mvcValue 숫자 mV 단위 최대 기록 전압
calibrationDate 정수 보정 수행 시 타임스탬프
expiryDate 정수 재보정 권장 시점
method 문자열 보정 방법 (manual_mvc, automated)
trials 정수 평균화된 수축 횟수

2.6 바이너리 데이터 형식

대용량 EMG 기록의 경우, 표준은 효율적인 저장을 위해 바이너리 형식을 지원합니다:

바이너리 인코딩

  • 헤더: JSON 메타데이터 (처음 1024바이트)
  • 샘플: 리틀 엔디언 float32 또는 int16 배열
  • 압축: zlib 또는 gzip 압축 지원
  • 체크섬: 무결성 검증을 위한 CRC32

2.7 데이터 품질 요구사항

매개변수 최소값 권장값 이유
샘플링 속도 1000 Hz 2000 Hz EMG 주파수 대역 (20-450 Hz) 캡처
ADC 해상도 12비트 16-24비트 신호 충실도 및 동적 범위
증폭 이득 1000× 5000× 미약한 EMG 신호 (0.1-5 mV) 증폭
신호 대 잡음비 30 dB 40 dB 패턴 인식 정확도
공통 모드 제거비 80 dB 100 dB 전력선 간섭 억제

2.8 실제 예제

완전한 EMG 데이터 프레임

{
  "version": "1.0.0",
  "metadata": {
    "sampleRate": 2000,
    "channelCount": 4,
    "resolution": 16,
    "gain": 5000,
    "referenceType": "bipolar",
    "deviceId": "EMG-SENSOR-2024-001",
    "firmwareVersion": "2.1.3"
  },
  "channels": [
    {
      "id": 0,
      "name": "flexor_carpi_radialis",
      "placement": {
        "anatomicalLocation": "volar_forearm_radial",
        "distance_from_elbow_mm": 75
      },
      "samples": [0.012, -0.008, 0.015, 0.022],
      "unit": "mV",
      "calibration": {
        "mvcValue": 2.5,
        "calibrationDate": 1705312800000,
        "method": "manual_mvc",
        "trials": 3
      }
    }
  ],
  "startTime": 1705312800000,
  "duration": 2
}
