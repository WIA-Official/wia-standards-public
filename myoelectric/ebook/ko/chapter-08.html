제8장: 규제 및 표준 준수

8.1 ISO 22523 — 외부 의지 국제 표준

ISO 22523:2006 "외부 지체 의지 및 외부 보조기 — 요구사항 및 시험 방법"은 근전 의지를 포괄하는 핵심 국제 표준입니다. 주요 요구사항:

  • 분류 코드: P0~P3, E1~E3, K1~K5. WIA-MYO 세션 메타데이터 iso_22523_class 필드에 필수 기재.
  • 정적 강도: 파악 자세(동력 그립)에서 500 N 정하중에 영구 변형 없이 견뎌야 함.
  • 동적 피로: 정격 파지력으로 10만 회 개폐 반복 후 기구 고장 없음.
  • 생체적합성: 소켓 소재는 ISO 10993-1 (생체적합성 평가) 준수.
  • 파지력 재현성: 정격 최대 파지력 10~90% 범위에서 명령값의 ±15% 이내 재현.

8.2 IEC 60601-1 — 의료용 전기기기

임상 환경에서 사용되는 sEMG 증폭기 내장 근전 의지 시스템은 IEC 60601-1:2005+AMD2:2020 (의료용 전기기기 — 제1부: 기본 안전 및 필수 성능 일반 요구사항)을 준수해야 합니다.

요구사항사양
적용 부위 분류BF형 (Body Floating) — 피부 표면 전극
환자 누설 전류정상 상태 ≤ 10 μA, 단일 고장 ≤ 50 μA
EMC 내성IEC 60601-1-2 — 병원 전자기 환경 (2 V/m @ 26~1000 MHz)
필수 성능EMC 시험 중 sEMG 신호 수집이 사양 범위 내 유지

이식형 전극(IMES — 이식형 근전 센서) 인터페이스 시스템에는 IEC 60601-2-77이 추가 적용됩니다. WIA-MYO는 현재 표면 전극 시스템만 커버합니다.

8.4 건강보험 급여 등재 (HIRA)

국내 근전 의지의 건강보험 급여 등재는 건강보험심사평가원(HIRA) 보조기기 등재 절차를 거칩니다:

  1. 식약처 허가 취득: 2등급 기술문서 심사 또는 임상 데이터 심사.
  2. HIRA 신청: 임상적 유효성·안전성·경제성 평가 자료 제출.
  3. 선별급여 결정: 일반적으로 본인부담 30~50%로 선별급여 적용. 2022년 이후 상지 절단 장애인 의지 구입비 90% 지원(장애인복지법 §65).
  4. 청구 코드 발급: HIRA 보조기기 청구 코드 발급 후 의료기관에서 청구 가능.

큐렉소 수술 로봇 시스템이 2022년 선별급여 등재된 선례처럼, 혁신적 근전 의지 제품도 임상 근거 확보 시 급여 등재 가속화 가능합니다.

8.6 EU MDR 2017/745

EU MDR 하에서 근전 의지의 분류: 상지 근전 의지 Class IIa (Rule 9: 비침습 능동 치료 기기), 이식형 전극 기반 시스템 Class III (Rule 7: 이식형). CE 인증은 Notified Body(DEKRA, TÜV SÜD, BSI) 적합성 평가, ISO 14971 위험 관리, IEC 62304 소프트웨어 수명주기 필수 조화 표준 적용 필요.

8.7 WIA-MYO 자체 인증 점검 목록

ros2 run wia_myo_compliance check_node \
  --ros-args -p report:=/tmp/wia_myo_compliance.json

# 점검 항목:
# [1] 모든 EmgFrame에 $schema URI 포함 (JSON 스키마 v2)
# [2] 세션 메타데이터에 SENIAM 배치 코드 기재
# [3] 200 ms 윈도우 · ≥ 1초 기록으로 Hudgins TD4 특징 계산
# [4] 세션 메타데이터에 ISO 22523 분류 코드 기재
# [5] 샘플레이트 ≥ 2 kHz (EmgFrame.device.sample_rate_hz 확인)
# [6] 모든 프레임에 SNR 품질 지표 포함
# [7] 분류기 지연 < 30 ms (EmgClassification.classifier_latency_ms)
# [8] 세션 메타데이터의 환자 ID 익명화 (개인정보 미포함)
# [9] 분류기 설정에 거부 임계값 문서화
# [10] IEC 60601-1 환자 누설 전류 시험 결과 문서화

8.8 WIA 표준 등록소

WIA-MYO를 구현한 제조사는 WIA 근전 기기 등록소(https://standards.wia.global/myoelectric/registry)에 디바이스 프로파일을 제출합니다. 등록소는 WIA-MYO-{제조사}-{모델}-{연도} 형식의 wia_device_id를 부여하며, 이는 FHIR Device 리소스 및 WIA REST API 호출의 표준 식별자로 사용됩니다.
