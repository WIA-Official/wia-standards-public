3장: API 엔드포인트

3.1 API 개요

WIA 근전 표준은 EMG 장치 제어, 보정 세션 관리 및 제스처 인식을 위한 RESTful API를 정의합니다. 모든 엔드포인트는 요청/응답 본문에 JSON을 사용하며 HTTP 규약을 따릅니다.

기본 URL 구조

https://api.myoelectric.device/v1/{resource}

모든 API 요청에는 API 키 또는 OAuth 2.0 토큰을 통한 인증이 필요합니다.

3.2 장치 관리 엔드포인트

3.2.1 장치 목록 조회

GET /devices

연결된 모든 EMG 장치와 현재 상태를 반환합니다.

응답:

{
  "devices": [
    {
      "id": "EMG-SENSOR-001",
      "name": "근전 센서 어레이",
      "status": "connected",
      "firmwareVersion": "2.3.1",
      "batteryLevel": 85,
      "channelCount": 8,
      "sampleRate": 1000
    }
  ]
}

3.2.2 장치 세부 정보 조회

GET /devices/{deviceId}

구성 및 기능을 포함한 특정 장치의 세부 정보를 반환합니다.

3.2.3 장치 구성

POST /devices/{deviceId}/configure

요청 본문:

{
  "sampleRate": 1000,
  "resolution": 16,
  "gain": 1000,
  "referenceType": "bipolar",
  "channels": [
    {
      "id": 0,
      "enabled": true,
      "muscle": "flexor_carpi_radialis"
    }
  ]
}

3.3 보정 엔드포인트

3.3.1 MVC 보정 시작

POST /calibration/mvc/start

최대 수의 수축(MVC) 보정 세션을 시작합니다.

요청 본문:

{
  "deviceId": "EMG-SENSOR-001",
  "channels": [0, 1, 2, 3],
  "contractionDuration": 5000,
  "restDuration": 30000,
  "repetitions": 3
}

응답:

{
  "sessionId": "cal-session-12345",
  "status": "in_progress",
  "currentPhase": "rest_baseline",
  "instructions": "5초 동안 모든 근육을 이완하세요"
}

3.3.2 보정 상태 조회

GET /calibration/mvc/{sessionId}

실시간 피드백과 함께 보정 세션의 현재 상태를 반환합니다.

3.3.3 보정 완료

POST /calibration/mvc/{sessionId}/complete

응답:

{
  "sessionId": "cal-session-12345",
  "status": "completed",
  "results": [
    {
      "channel": 0,
      "muscle": "flexor_carpi_radialis",
      "mvcValue": 0.85,
      "restBaseline": 0.015,
      "confidence": 0.95,
      "quality": "excellent"
    }
  ]
}

3.4 신호 처리 엔드포인트

3.4.1 원시 EMG 데이터 스트리밍

WebSocket: wss://api.myoelectric.device/v1/stream/raw

원시 EMG 샘플 스트리밍을 위한 실시간 WebSocket 연결을 설정합니다.

메시지 형식:

{
  "timestamp": 1705312800123,
  "deviceId": "EMG-SENSOR-001",
  "samples": {
    "ch0": [0.012, -0.008, 0.015],
    "ch1": [-0.005, 0.003, -0.002],
    "ch2": [0.018, 0.021, 0.017],
    "ch3": [-0.012, -0.009, -0.015]
  }
}

3.4.2 특징 추출

POST /processing/features

EMG 데이터에서 시간 영역 및 주파수 영역 특징을 계산합니다.

요청 본문:

{
  "deviceId": "EMG-SENSOR-001",
  "windowSize": 256,
  "features": ["rms", "mav", "wl", "zc", "ssc", "mdf"]
}

3.5 제스처 인식 엔드포인트

3.5.1 제스처 인식 시작

POST /recognition/start

실시간 제스처 분류를 시작합니다.

요청 본문:

{
  "deviceId": "EMG-SENSOR-001",
  "modelId": "gesture-model-v2",
  "gestures": ["rest", "hand_open", "hand_close", "wrist_flexion",
               "wrist_extension", "pronation", "supination", "pinch"]
}

응답 (WebSocket 스트림):

{
  "timestamp": 1705312800456,
  "gesture": "hand_close",
  "confidence": 0.97,
  "intensity": 0.65
}

3.5.2 모델 학습

POST /recognition/train

사용자별 제스처 인식 모델을 학습합니다.

요청 본문:

{
  "userId": "patient-12345",
  "trainingData": [
    {
      "gesture": "hand_close",
      "samples": [ ... ],
      "repetitions": 5
    }
  ],
  "algorithm": "LDA",
  "crossValidation": true
}

3.6 의수 제어 엔드포인트

3.6.1 의수 명령 전송

POST /prosthetic/command

요청 본문:

{
  "prostheticId": "prosthetic-001",
  "action": "grip",
  "gripType": "power",
  "force": 0.6,
  "speed": 0.8
}
매개변수 타입 설명
action 문자열 동작 타입: grip, release, rotate
gripType 문자열 파지 유형: power, precision, lateral, tripod
force 숫자 힘 강도 (0.0-1.0)
speed 숫자 동작 속도 (0.0-1.0)

3.7 API 인증

인증 방법

  • API 키: Authorization: ApiKey YOUR_API_KEY
  • OAuth 2.0: Authorization: Bearer YOUR_TOKEN
  • 상호 TLS: 클라이언트 인증서 기반 인증

3.8 오류 처리

HTTP 코드 오류 타입 설명
400 잘못된 요청 요청 형식 또는 매개변수 오류
401 인증 실패 유효하지 않거나 만료된 자격 증명
404 찾을 수 없음 장치 또는 리소스가 존재하지 않음
429 속도 제한 초과 너무 많은 요청
500 서버 오류 내부 처리 오류
