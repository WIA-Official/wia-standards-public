8.1 의료 기기 규제 개요

근전 의수 시스템은 전 세계적으로 의료 기기로 분류되며 엄격한 규제 요구사항을 준수해야 합니다.

지역 규제 기관 주요 규정
미국 FDA 21 CFR Part 890 (물리 의학 기기)
유럽 Notified Bodies EU MDR 2017/745
캐나다 Health Canada Medical Devices Regulations (SOR/98-282)
일본 PMDA Pharmaceutical and Medical Device Act
한국 식약처 (MFDS) 의료기기법

8.2 FDA 인증 (미국)

8.2.1 장치 분류

근전 의수는 일반적으로 Class II 의료 기기로 분류됩니다:

  • 제품 코드: KDY (전기 동력 상지 의수)
  • 규제 번호: 21 CFR 890.3420
  • 승인 경로: 510(k) 사전 통지

8.2.2 510(k) 제출 요구사항

필수 문서

  • 장치 설명 및 의도된 사용
  • 실질적 동등성 (SE) 비교
  • 성능 사양 및 테스트 데이터
  • 생체 적합성 테스트 (ISO 10993)
  • 전기 안전 테스트 (IEC 60601-1)
  • EMC 테스트 (IEC 60601-1-2)
  • 소프트웨어 검증 (IEC 62304)
  • 사용성 엔지니어링 (IEC 62366)
  • 위험 관리 파일 (ISO 14971)

8.3 EU MDR 준수 (유럽)

8.3.1 장치 분류

EU MDR에 따른 의수 분류:

  • 규칙 5: 능동 치료 장치 - Class IIa
  • 규칙 11: 소프트웨어 - Class IIa (진단/치료 결정에 사용)

8.3.2 적합성 평가

요구사항 설명
기술 문서 장치 설명, 위험 분석, 임상 평가
품질 관리 시스템 ISO 13485 인증
Notified Body 검토 Class IIa 장치의 경우 NB 평가 필요
CE 마크 규정 준수를 나타내는 CE 마크 부착
UDI 시스템 고유 장치 식별자 할당

8.4 위험 관리 (ISO 14971)

8.4.1 위험 분석 프로세스

위험 관리 단계:

1. 위험 분석
   - 의도된 사용 및 합리적으로 예측 가능한 오용 식별
   - 위험 관련 특성 식별
   - 위해 식별

2. 위험 평가
   - 각 위해에 대한 위험 추정
   - 위험 평가 (허용 가능한지 판단)

3. 위험 제어
   - 위험 제어 조치 구현
     a) 설계에 의한 고유 안전성
     b) 보호 조치
     c) 정보를 위한 안전 정보
   - 잔여 위험 평가

4. 생산 및 사후 생산
   - 생산 정보 수집
   - 필요시 위험 관리 파일 업데이트

8.4.2 근전 시스템 위험 예시

위해 위험 완화 조치
부정확한 제스처 인식 의도하지 않은 의수 동작, 낙하 95% 분류 정확도 요구, 다중 센서 검증
전극 접촉 불량 신호 품질 저하, 제어 손실 임피던스 모니터링, 사용자 경고
배터리 소진 갑작스러운 기능 손실 배터리 경고, 안전 모드 (부드러운 열림)
소프트웨어 오류 예측 불가능한 동작 워치독 타이머, 안전 제한, 철저한 테스트

8.5 임상 증거

8.5.1 임상 평가 보고서

규제 제출에는 다음을 입증하는 임상 증거가 필요합니다:

  • 안전성: 허용할 수 없는 위험이 없음
  • 성능: 의도된 목적대로 작동
  • 이점-위험 비율: 이점이 위험보다 큼

8.5.2 성능 메트릭

지표 목표 측정 방법
제스처 인식 정확도 > 95% 실험실 및 현장 테스트
제어 지연 < 200 ms EMG에서 의수 동작까지의 시간
일일 사용 시간 > 8 시간 사용자 로그, 설문조사
사용자 만족도 > 7/10 검증된 설문지 (예: QUEST)
기능적 이득 통계적으로 유의함 SHAP, DASH, Box and Blocks Test

8.6 품질 관리 시스템 (ISO 13485)

8.6.1 핵심 프로세스

  • 설계 제어: 설계 입력, 출력, 검증, 유효성 확인
  • 문서 제어: 버전 관리, 승인, 배포
  • 구매 제어: 공급업체 평가 및 모니터링
  • 생산 제어: 프로세스 유효성 확인, 추적성
  • 시정 및 예방 조치 (CAPA): 부적합 처리
  • 내부 감사: 정기적인 QMS 검토

8.7 WIA 인증 프로그램

WIA는 근전 표준 준수를 위한 자발적 인증 프로그램을 제공합니다.

WIA 인증 혜택

  • 시장에서의 차별화 및 신뢰 구축
  • 상호 운용성 및 표준 준수 보장
  • 규제 제출 간소화 (표준 참조 가능)
  • WIA 인증 로고 및 마케팅 사용
  • 기술 지원 및 업데이트 접근

8.7.1 인증 프로세스

WIA 인증 단계:

1. 사전 평가
   - 표준 요구사항 검토
   - 격차 분석 수행

2. 문서 제출
   - 기술 사양
   - 테스트 보고서
   - 품질 관리 시스템 증거

3. 기술 검토
   - WIA 심사관의 문서 평가
   - 필요시 명확화 요청

4. 적합성 테스트
   - 독립 실험실에서 검증 테스트
   - 데이터 형식, API, 성능 검증

5. 인증 결정
   - 모든 요구사항 충족 시 인증 부여
   - 3년 동안 유효, 갱신 가능

6. 사후 시장 감시
   - 연간 자체 선언
   - 정기적인 재평가

8.8 표준 준수 체크리스트

구현 검증

  • ☐ EMG 데이터 형식이 WIA 스키마를 따름
  • ☐ API 엔드포인트가 명세와 일치
  • ☐ 실시간 스트리밍이 성능 요구사항 충족
  • ☐ 보안 제어가 구현됨 (암호화, 인증)
  • ☐ 개인정보보호 규정 준수 (GDPR, HIPAA)
  • ☐ 의료 기기 규제 요구사항 충족 (해당되는 경우)
  • ☐ 품질 관리 시스템 수립 (ISO 13485)
  • ☐ 위험 관리 프로세스 완료 (ISO 14971)
  • ☐ 임상 증거 문서화
  • ☐ 사용자 설명서 및 라벨링 준비
