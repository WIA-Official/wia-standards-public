3.1 The Hudgins Time-Domain Feature Set

The foundational contribution of Hudgins, Parker, and Scott (1993) "A new strategy for multifunction myoelectric control" established four time-domain features — collectively the Hudgins TD4 set — computed over a 200–256 ms analysis window. This work demonstrated for the first time that reliable multi-class classification of hand gestures (up to 4 classes) was achievable from forearm sEMG in real-time. 

import numpy as np

def hudgins_td4(segment: np.ndarray) -> dict:
    """Hudgins et al. 1993 four time-domain features."""
    N = len(segment)
    # MAV: mean absolute value
    mav = np.mean(np.abs(segment))
    # ZC: zero crossings (with threshold to suppress noise)
    threshold = 0.01 * np.max(np.abs(segment))
    zc = np.sum((np.sign(segment[:-1]) != np.sign(segment[1:])) &
                (np.abs(segment[:-1] - segment[1:]) >= threshold))
    # SSC: slope sign changes
    diff = np.diff(segment)
    ssc = np.sum((np.sign(diff[:-1]) != np.sign(diff[1:])) &
                 ((np.abs(diff[:-1]) + np.abs(diff[1:])) >= threshold))
    # WL: waveform length
    wl = np.sum(np.abs(np.diff(segment)))
    return {"mav": float(mav), "zc": int(zc), "ssc": int(ssc), "wl": float(wl)}

def ar4_burg(segment: np.ndarray) -> list:
    """AR(4) coefficients via Burg's method — fast, numerically stable."""
    # Uses spectrum.burg or manual implementation
    from scipy.signal import lfilter
    N = len(segment)
    ar = np.zeros(4)
    # Levinson-Durbin recursion (simplified)
    r = np.correlate(segment, segment, mode='full')[N-1:N+4]
    r = r / r[0]
    # Yule-Walker equations (simplified)
    R = np.array([[r[abs(i-j)] for j in range(4)] for i in range(4)])
    ar = np.linalg.solve(R, -r[1:5])
    return ar.tolist()

3.3 Analysis Window Parameters

The choice of analysis window duration and overlap significantly affects classifier performance and control responsiveness:

ParameterTypical ValueTrade-off
Window size200–256 msLonger = better features; shorter = faster response. ≥150 ms required for stable Hudgins TD4.
Window overlap50% (100 ms overlap)50% overlap at 200 ms → 10 Hz classification output. Higher overlap reduces delay.
Incremental processing25–50 ms increment25 ms increment at 200 ms window → 40 Hz output, ~25 ms additional delay.

System control loop delay = window_size/2 (inherent) + processing_time. For 200 ms window: minimum achievable delay ≈ 125–150 ms. Physiologically imperceptible delay threshold: ~300 ms (Parker & Scott 1986).

3.4 Linear Discriminant Analysis (LDA)

LDA is the clinical standard classifier for myoelectric pattern recognition. Despite its simplicity, LDA consistently achieves 90–98% offline accuracy for 4–8 gesture classes with Hudgins TD4 features, while meeting the real-time constraint (classifer latency < 5 ms on embedded hardware). 

from sklearn.discriminant_analysis import LinearDiscriminantAnalysis
import numpy as np

# Training phase (in prosthetist's office)
clf = LinearDiscriminantAnalysis(solver='svd', shrinkage=None)
clf.fit(X_train, y_train)  # X_train: (N_samples, 32) TD4 × 8 channels

# Real-time classification

3.5 LSTM Neural Networks for Sequential Gestures

Long Short-Term Memory (LSTM) networks capture temporal dependencies between successive EMG frames, enabling smoother, more natural transitions between gestures. WIA-MYO supports LSTM classifiers that process a sequence of feature vectors rather than a single window: 

import torch
import torch.nn as nn

class MyoLSTM(nn.Module):
    def __init__(self, input_size=32, hidden_size=128,
                 num_layers=2, num_classes=8):
        super().__init__()
        self.lstm = nn.LSTM(input_size, hidden_size, num_layers,
                            batch_first=True, dropout=0.2)
        self.fc = nn.Linear(hidden_size, num_classes)

    def forward(self, x):           # x: (batch, seq_len, 32)
        out, _ = self.lstm(x)
        return self.fc(out[:, -1, :])  # use last time step

# Inference: feed 10-frame sequence (200 ms each = 2 s context)
model = MyoLSTM()
model.eval()
with torch.no_grad():
    logits = model(feature_sequence.unsqueeze(0))  # (1, 10, 32)
    proba = torch.softmax(logits, dim=1).numpy()[0]
