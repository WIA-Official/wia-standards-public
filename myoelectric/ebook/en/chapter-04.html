Chapter 4: Proportional Control and Coapt Integration

4.1 Proportional Myoelectric Control

Traditional proportional control (also called amplitude-based or threshold control) maps the rectified-and-smoothed sEMG amplitude directly to prosthesis velocity or grip force. It requires only two electrode sites (flexion and extension channels) and is the most widely deployed control strategy globally due to simplicity and robustness.

Dual-site proportional control algorithm:

def proportional_command(flex_rms_uv: float, ext_rms_uv: float) -> dict:
    """Dual-site proportional controller per WIA-MYO-DATA-001 §5."""
    DEAD_BAND = 8.0       # uV — noise floor
    ACTIVATION_LOW = 15.0  # uV — activation threshold
    ACTIVATION_HIGH = 25.0 # uV — full-speed threshold (two-speed mode)
    SATURATION = 120.0     # uV — output saturates at this amplitude

    def channel_output(rms: float) -> float:
        if rms < DEAD_BAND:
            return 0.0
        raw = (rms - DEAD_BAND) / (SATURATION - DEAD_BAND)
        return min(1.0, max(0.0, raw))

    flex_out = channel_output(flex_rms_uv)
    ext_out = channel_output(ext_rms_uv)

    # Co-contraction rejection: if both > activation_high, enter mode switch
    if flex_rms_uv > ACTIVATION_HIGH and ext_rms_uv > ACTIVATION_HIGH:
        return {"command": 0.0, "direction": "mode_switch"}

    if flex_out > ext_out:
        return {"command": flex_out, "direction": "flexion"}
    else:
        return {"command": ext_out, "direction": "extension"}

4.2 Two-Speed and Mode Switching

Most commercial prosthetic hands extend proportional control with:

  • Two-speed control: below activation_high → slow speed; above → fast speed. Enables both precision grip (slow) and rapid repositioning (fast).
  • Co-contraction mode switch: simultaneous contraction of both muscles (above activation_high) triggers grip pattern selection. The WIA schema encodes this as "direction": "mode_switch".
  • Rate limiting / ramp: velocity command ramps at 0.02 per ms (WIA default) to prevent jerky motion onset.

4.3 Coapt COMPLETE CONTROL

The Coapt COMPLETE CONTROL system (Coapt LLC, Chicago) is the most widely deployed commercial pattern recognition controller for upper limb prosthetics. It combines 8-channel sEMG with an onboard LDA classifier, proprietary continuous re-calibration ("Adapt" mode), and a USB interface to commercially available prosthetic hands (BeBionic, Ottobock Michelangelo, Össur i-Limb). 

# Coapt RE-NET Python SDK (unofficial wrapper)
from coapt_sdk import CoaptReNet

device = CoaptReNet.connect(port="/dev/ttyUSB0", baud=115200)
device.set_mode(CoaptReNet.MODE_PATTERN_RECOGNITION)
device.set_gesture_set([
    "rest", "open_hand", "close_hand", "pronate",
    "supinate", "wrist_flex", "wrist_ext", "tripod_pinch"
])

# Start streaming classifications (10 Hz)
for frame in device.stream_classifications():
    print(f"Gesture: {frame.gesture} ({frame.confidence:.2f})"
          f" latency={frame.latency_ms:.1f}ms")

4.4 Ottobock OrthoConnect SDK

Ottobock's OrthoConnect platform provides Bluetooth connectivity to the MyoHand, Michelangelo, and bebionic v4 prosthetic hands, enabling direct grip selection, force adjustment, and fitting data logging from a mobile app or custom clinical software. 

# Ottobock OrthoConnect BLE pairing (Python bleak library)
import asyncio
from bleak import BleakClient

OTTOBOCK_SERVICE_UUID = "0000xxxx-0000-1000-8000-00805f9b34fb"  # product-specific

async def send_grip_command(mac: str, grip_id: int):
    async with BleakClient(mac) as client:
        await client.write_gatt_char(
            OTTOBOCK_SERVICE_UUID,
            bytes([0x01, grip_id, 0x00])  # command format per OrthoConnect spec
        )
