6.1 Protocol Overview

WIA-MYO supports three primary communication protocols for myoelectric prosthetic devices:

ProtocolUse CaseLatencyRange
BLE 5.4 GATTWireless electrode streaming (2 kHz compressed) → mobile app / hub7.5–15 ms connection interval10–50 m
USB HID / CDCWired research-grade systems (Delsys, full-rate 2 kHz raw)< 1 msWired (5 m)
CAN bus (ISO 11898)Internal prosthetic hand actuator control (grip force, speed)0.1–1 msInternal

6.2 BLE GATT Profile (WIA-MYO Profile)

The WIA-MYO BLE GATT profile defines standardised service and characteristic UUIDs for wireless sEMG electrode modules, compatible with BLE 5.4 (2 Mbps PHY, LE Audio CIS for time-synchronised multi-sensor arrays). 

# WIA-MYO GATT Profile (BLE 5.4)
Service UUID: 181D-WIA-Myoelectric-Service
  Characteristic: 2A56-WIA-EmgRawData
    Properties: NOTIFY, READ
    Format: Uint8 channel_id + Int16[] samples (little-endian)
    Notify interval: 2.5 ms (8 samples at 2 kHz = 4 ms per packet)

  Characteristic: 2A57-WIA-EmgFeatures
    Properties: NOTIFY, READ
    Format: Float32 mav + Float32 rms + Uint8 zc + Uint8 ssc + Float32 wl (per channel)
    Notify interval: 25 ms (10 Hz feature output)

  Characteristic: 2A58-WIA-Classification
    Properties: NOTIFY, READ
    Format: Uint8 class_index + Float32 confidence + Uint8 rejected
    Notify interval: 100 ms (10 Hz output)

  Characteristic: 2A59-WIA-DeviceConfig
    Properties: READ, WRITE
    Format: Uint8 gain + Uint16 sample_rate_hz + Uint8 num_channels

6.3 Python BLE Client (bleak)

import asyncio
import struct
from bleak import BleakClient

WIA_EMG_FEATURES_UUID = "0000-2A57-0000-1000-8000-00805f9b34fb"

async def stream_features(mac_address: str):
    def feature_callback(handle, data: bytearray):
        # Unpack 5 × float32 per channel (8 channels = 40 floats)
        floats = struct.unpack(f"<{len(data)//4}f", data)
        mav_ch0 = floats[0]
        print(f"MAV ch0 = {mav_ch0:.2f} uV")

    async with BleakClient(mac_address) as client:
        await client.start_notify(WIA_EMG_FEATURES_UUID, feature_callback)
        print(f"Streaming features from {mac_address}")
        await asyncio.sleep(60.0)
        await client.stop_notify(WIA_EMG_FEATURES_UUID)

asyncio.run(stream_features("AA:BB:CC:DD:EE:FF"))

6.4 USB HID Protocol (Wired Systems)

Wired sEMG systems (Delsys Trigno, OTB Quattrocento) use USB HID or CDC-ACM (virtual serial port). WIA-MYO defines a standard USB descriptor for compliant devices: 

# WIA-MYO USB HID Report Descriptor (simplified)
USAGE_PAGE Generic Desktop
USAGE Sensor
COLLECTION Application
  USAGE 0xE1  # WIA sEMG raw data
  REPORT_SIZE 16        # 16-bit ADC samples
  REPORT_COUNT 256      # 16 channels × 16 samples per frame (8 ms at 2 kHz)
  INPUT Data|Var|Abs
END_COLLECTION

# Python HID read (hidapi)
import hid
device = hid.device()
device.open(0x1234, 0xABCD)  # VID, PID from WIA-MYO registry
data = device.read(512, timeout_ms=10)
samples = [int.from_bytes(data[i:i+2], 'little', signed=True)
           for i in range(0, 512, 2)]

6.6 Latency Measurement

End-to-end latency from muscle contraction to prosthetic motion onset is critical for intuitive control. WIA-MYO defines a standardised latency test protocol: 

# Latency test: LED flash at muscle contraction → camera detects prosthetic motion
# Reference: WIA-MYO-LATENCY-TEST-001

ros2 run wia_myo_diagnostics latency_test_node \
  --ros-args \
  -p emg_topic:=/emg/raw \
  -p command_topic:=/prosthetic/command \
  -p camera_topic:=/camera/image_raw \
  -p num_trials:=50 \
  -p output_csv:=/tmp/wia_myo_latency.csv

# Expected output:
# Mean latency: 127 ms (EMG onset → motion onset)
# P95 latency: 185 ms
# WIA target: P95 ≤ 300 ms
