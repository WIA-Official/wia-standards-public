2.1 EMG Data Frame Structure

The WIA Myoelectric Standard defines a comprehensive data format for EMG signals that ensures interoperability between different sensor hardware, processing software, and prosthetic devices.

2.1.1 Top-Level Structure

{
  "version": "1.0.0",
  "metadata": { ... },
  "channels": [ ... ],
  "startTime": 1705312800000,
  "duration": 1000
}
Field Type Description
version string WIA standard version (semantic versioning)
metadata object Recording configuration and device information
channels array Array of EMG channel objects with samples
startTime integer Unix timestamp in milliseconds (UTC)
duration integer Recording duration in milliseconds

2.2 Metadata Object

The metadata object contains critical information about the recording configuration:

Field Type Required Description
sampleRate integer Yes Sampling frequency in Hz (1000-2000 recommended)
channelCount integer Yes Number of EMG channels (2-8 typical)
resolution integer Yes ADC resolution in bits (12-24)
gain number Yes Amplification gain factor (1000-10000)
referenceType string Yes Electrode configuration: monopolar, bipolar, differential
deviceId string No Unique identifier for sensor device
firmwareVersion string No Device firmware version

2.2.1 Reference Types

Type Korean Description Use Case
monopolar 단극 Single electrode with remote reference General purpose, susceptible to noise
bipolar 이극 Two electrodes over same muscle Standard clinical use (recommended)
differential 차동 Two electrodes with common-mode rejection High noise environments

2.3 Channel Object

Each EMG channel represents signals from a specific muscle site:

{
  "id": 0,
  "name": "flexor_carpi_radialis",
  "placement": { ... },
  "samples": [0.012, -0.008, 0.015, ...],
  "unit": "mV",
  "calibration": { ... }
}
Field Type Required Description
id integer Yes Zero-indexed channel identifier
name string Yes Muscle name (snake_case format)
placement object Yes Electrode placement information
samples array Yes Raw EMG samples as 32-bit floats
unit string Yes Measurement unit (typically "mV")
calibration object No Channel calibration parameters

2.4 Electrode Placement

Precise electrode placement is critical for reliable EMG control. The placement object specifies exact anatomical positioning:

{
  "muscle": "flexor_carpi_radialis",
  "location": {
    "x": 12.5,
    "y": 0.0,
    "circumference": 45.0
  },
  "anatomicalLandmark": "medial_epicondyle",
  "orientation": "parallel"
}

Coordinate System

Origin: Lateral epicondyle of humerus (elbow)

  • X-axis: Distance from elbow toward wrist (cm)
  • Y-axis: Offset from muscle center (cm, positive = radial)
  • Circumference: Angular position around forearm (degrees, 0° = posterior)

2.6 Binary Data Format

For real-time applications requiring high efficiency, a compact binary format is supported:

Header Structure (32 bytes)

Bytes Field Type Description
0-3 Magic char[4] "WIA1" signature
4-5 Version uint16 Format version number
6-7 Channel count uint16 Number of EMG channels
8-11 Sample rate uint32 Sampling frequency (Hz)
12-15 Sample count uint32 Samples per channel
16-23 Start time uint64 Unix timestamp (ms)
24-27 Flags uint32 Configuration flags
28-31 Reserved uint32 Reserved for future use

Channel Headers (16 bytes each)

struct ChannelHeader {
    uint32_t channel_id;           // Channel identifier
    uint32_t muscle_code;          // Muscle enumeration
    float calibration_factor;      // Scaling factor
    float offset;                  // DC offset
};

Sample Data

Interleaved float32 samples: [ch0_s0, ch1_s0, ..., chN_s0, ch0_s1, ...]

2.7 Quality Requirements

Metric Acceptable Range Description
SNR (Signal-to-Noise Ratio) > 20 dB Signal quality relative to noise floor
Baseline Drift < 0.1 mV/s DC offset change rate
50/60 Hz Noise < -40 dB Power line interference suppression
Cross-talk < -20 dB Inter-channel interference
Electrode Impedance < 10 kΩ Skin-electrode contact quality

Data Integrity Requirements

  • All samples must be timestamped with microsecond precision
  • Missing samples must be indicated with NaN values
  • Maximum packet loss: 0.1% for real-time control
  • Checksums required for all binary transmissions

2.8 Example: Complete EMG Data Frame

{
  "version": "1.0.0",
  "metadata": {
    "sampleRate": 1000,
    "channelCount": 4,
    "resolution": 16,
    "gain": 1000,
    "referenceType": "bipolar",
    "deviceId": "EMG-SENSOR-001",
    "firmwareVersion": "2.3.1"
  },
  "channels": [
    {
      "id": 0,
      "name": "flexor_carpi_radialis",
      "placement": {
        "muscle": "flexor_carpi_radialis",
        "location": { "x": 10.0, "y": 0.0, "circumference": 145.0 },
        "anatomicalLandmark": "medial_epicondyle",
        "orientation": "parallel"
      },
      "samples": [0.012, -0.008, 0.015, 0.022, ...],
      "unit": "mV",
      "calibration": {
        "mvcValue": 0.85,
        "calibrationDate": 1705312800000
      }
    },
    {
      "id": 1,
      "name": "extensor_carpi_radialis_longus",
      "placement": {
        "muscle": "extensor_carpi_radialis_longus",
        "location": { "x": 8.0, "y": 0.0, "circumference": 50.0 },
        "anatomicalLandmark": "lateral_epicondyle",
        "orientation": "parallel"
      },
      "samples": [-0.005, 0.003, -0.002, 0.008, ...],
      "unit": "mV"
    }
  ],
  "startTime": 1705312800000,
  "duration": 1000
}
