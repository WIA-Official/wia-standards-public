Chapter 3: API Endpoints

3.1 API Overview

The WIA Myoelectric Standard defines a RESTful API for controlling EMG devices, managing calibration sessions, and performing gesture recognition. All endpoints use JSON for request/response bodies and follow HTTP conventions.

Base URL Structure

https://api.myoelectric.device/v1/{resource}

All API requests require authentication via API key or OAuth 2.0 token.

3.2 Device Management Endpoints

3.2.1 List Devices

GET /devices

Returns all connected EMG devices and their current status.

Response:

{
  "devices": [
    {
      "id": "EMG-SENSOR-001",
      "name": "Myoelectric Sensor Array",
      "status": "connected",
      "firmwareVersion": "2.3.1",
      "batteryLevel": 85,
      "channelCount": 8,
      "sampleRate": 1000
    }
  ]
}

3.2.2 Get Device Details

GET /devices/{deviceId}

Returns detailed information about a specific device including configuration and capabilities.

3.2.3 Configure Device

POST /devices/{deviceId}/configure

Request Body:

{
  "sampleRate": 1000,
  "resolution": 16,
  "gain": 1000,
  "referenceType": "bipolar",
  "channels": [
    {
      "id": 0,
      "enabled": true,
      "muscle": "flexor_carpi_radialis"
    }
  ]
}

3.3 Calibration Endpoints

3.3.1 Start MVC Calibration

POST /calibration/mvc/start

Initiates a Maximum Voluntary Contraction (MVC) calibration session.

Request Body:

{
  "deviceId": "EMG-SENSOR-001",
  "channels": [0, 1, 2, 3],
  "contractionDuration": 5000,
  "restDuration": 30000,
  "repetitions": 3
}

Response:

{
  "sessionId": "cal-session-12345",
  "status": "in_progress",
  "currentPhase": "rest_baseline",
  "instructions": "Relax all muscles for 5 seconds"
}

3.3.2 Get Calibration Status

GET /calibration/mvc/{sessionId}

Returns current status of calibration session with real-time feedback.

3.3.3 Complete Calibration

POST /calibration/mvc/{sessionId}/complete

Response:

{
  "sessionId": "cal-session-12345",
  "status": "completed",
  "results": [
    {
      "channel": 0,
      "muscle": "flexor_carpi_radialis",
      "mvcValue": 0.85,
      "restBaseline": 0.015,
      "confidence": 0.95,
      "quality": "excellent"
    }
  ]
}

3.4 Signal Processing Endpoints

3.4.1 Stream Raw EMG Data

WebSocket: wss://api.myoelectric.device/v1/stream/raw

Establishes real-time WebSocket connection for streaming raw EMG samples.

Message Format:

{
  "timestamp": 1705312800123,
  "deviceId": "EMG-SENSOR-001",
  "samples": {
    "ch0": [0.012, -0.008, 0.015],
    "ch1": [-0.005, 0.003, -0.002],
    "ch2": [0.018, 0.021, 0.017],
    "ch3": [-0.012, -0.009, -0.015]
  }
}

3.4.2 Extract Features

POST /processing/features

Computes time-domain and frequency-domain features from EMG window.

Request Body:

{
  "channels": [
    {
      "id": 0,
      "samples": [0.012, -0.008, 0.015, ...]
    }
  ],
  "sampleRate": 1000,
  "features": ["mav", "rms", "wl", "zc", "ssc", "mnf", "mdf"]
}

Response:

{
  "featureVector": [0.123, 0.156, 45.2, 18, 22, 87.5, 95.2, ...],
  "featureNames": ["ch0_mav", "ch0_rms", "ch0_wl", "ch0_zc", ...],
  "timestamp": 1705312800000
}

3.5 Gesture Recognition Endpoints

3.5.1 Train Classifier

POST /recognition/train

Trains a gesture classifier using collected EMG patterns.

Request Body:

{
  "trainingData": [
    {
      "gesture": "hand_open",
      "features": [[0.12, 0.15, 42.1, ...], [0.11, 0.14, 43.5, ...]]
    },
    {
      "gesture": "hand_close",
      "features": [[0.45, 0.52, 89.3, ...], [0.43, 0.51, 87.1, ...]]
    }
  ],
  "algorithm": "lda",
  "crossValidation": true
}

Response:

{
  "modelId": "classifier-78901",
  "accuracy": 0.96,
  "confusionMatrix": [[95, 5], [3, 97]],
  "classes": ["hand_open", "hand_close"],
  "trainingTime": 245
}

3.5.2 Recognize Gesture (Real-time)

POST /recognition/predict

Classifies current EMG pattern into gesture class.

Request Body:

{
  "modelId": "classifier-78901",
  "features": [0.123, 0.156, 45.2, 18, 22, 87.5, 95.2, ...]
}

Response:

{
  "gesture": "hand_close",
  "confidence": 0.94,
  "probabilities": {
    "rest": 0.01,
    "hand_open": 0.03,
    "hand_close": 0.94,
    "wrist_flexion": 0.02
  },
  "latency": 12
}

3.5.3 Stream Gesture Recognition

WebSocket: wss://api.myoelectric.device/v1/stream/gestures

Continuous gesture recognition stream with sub-200ms latency.

Message Format:

{
  "timestamp": 1705312800456,
  "gesture": "precision_pinch",
  "confidence": 0.92,
  "intensity": 0.67,
  "holdDuration": 1250
}

3.6 Prosthetic Control Endpoints

3.6.1 Send Motor Command

POST /prosthetic/command

Sends control command to prosthetic actuators based on recognized gesture.

Request Body:

{
  "deviceId": "PROSTHETIC-HAND-001",
  "command": {
    "type": "grip",
    "gesture": "power_grasp",
    "force": 0.65,
    "speed": 0.80
  }
}

3.6.2 Get Prosthetic Status

GET /prosthetic/{deviceId}/status

Response:

{
  "deviceId": "PROSTHETIC-HAND-001",
  "currentGrip": "power_grasp",
  "position": [0.0, 45.0, 90.0, 120.0, 150.0],
  "force": 0.65,
  "batteryLevel": 72,
  "status": "active"
}

3.7 Session Management

3.7.1 Create Training Session

POST /sessions/training

Creates a new training session for gesture data collection.

Request Body:

{
  "patientId": "patient-001",
  "gestures": ["rest", "hand_open", "hand_close", "wrist_flexion", "wrist_extension"],
  "repetitionsPerGesture": 10,
  "contractionDuration": 3000,
  "restDuration": 2000
}

3.7.2 Log Session Data

POST /sessions/{sessionId}/data

Uploads EMG data collected during session for analysis and storage.

3.8 Error Responses

Status Code Error Type Description
400 Bad Request Invalid request parameters or malformed JSON
401 Unauthorized Missing or invalid authentication credentials
404 Not Found Requested resource does not exist
409 Conflict Resource state conflict (e.g., device already in use)
422 Unprocessable Entity Valid request but semantic errors (e.g., poor signal quality)
429 Rate Limit Exceeded Too many requests in time window
500 Internal Server Error Server-side processing failure

Example Error Response:

{
  "error": {
    "code": "SIGNAL_QUALITY_LOW",
    "message": "EMG signal quality below acceptable threshold",
    "details": {
      "channels": [0, 2],
      "snr": [12.3, 15.8],
      "threshold": 20.0
    }
  }
}
