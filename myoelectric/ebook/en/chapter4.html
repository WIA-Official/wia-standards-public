4.1 EMG Preprocessing Pipeline

The WIA standard defines a comprehensive signal processing pipeline that transforms raw EMG signals into clean, normalized data suitable for gesture recognition and prosthetic control.

Pipeline Architecture

Raw EMG → Notch Filter → Bandpass Filter → Rectification → Envelope → Normalization → Output

Total latency budget: < 50 ms for real-time control

4.2 Stage 1: Notch Filter

Removes 50 Hz (Europe/Asia) or 60 Hz (Americas) power line interference and harmonics.

Algorithm: Second-Order IIR Notch Filter

// Transfer function
H(z) = (b0 + b1*z^-1 + b2*z^-2) / (1 + a1*z^-1 + a2*z^-2)

// Coefficients calculation
w0 = 2 * PI * f0 / fs
alpha = sin(w0) / (2 * Q)

b0 = 1
b1 = -2 * cos(w0)
b2 = 1
a1 = -2 * cos(w0)
a2 = 1 - alpha

// Normalize by (1 + alpha)
Parameter Value Notes
Center frequency (f0) 50 or 60 Hz Based on regional power system
Quality factor (Q) 30.0 Higher = narrower notch
Attenuation > 40 dB At center frequency
Bandwidth (-3dB) < 2 Hz Narrow to preserve EMG

4.3 Stage 2: Bandpass Filter

Removes motion artifacts (low frequency) and high-frequency noise while preserving EMG signal content.

Algorithm: 4th-Order Butterworth Bandpass

Application Low Cutoff High Cutoff Notes
General purpose 20 Hz 450 Hz Standard EMG range (recommended)
Fine control 20 Hz 500 Hz Higher bandwidth for detailed signals
Motion rejection 30 Hz 450 Hz More aggressive low-frequency cut
Limited bandwidth 20 Hz 350 Hz For lower sample rates

Requirements

  • Passband ripple: < 0.5 dB
  • Stopband attenuation: > 40 dB
  • Phase response: Linear phase preferred
  • Group delay: < 10 ms

4.4 Stage 3: Rectification

Converts bipolar EMG signal to unipolar signal representing muscle activation intensity.

Methods

Method Formula Use Case
Full-Wave (Recommended) y = |x| Preserves all signal energy, standard approach
Half-Wave y = max(x, 0) Simple but loses half the information
Squaring y = x² Emphasizes large amplitudes

4.5 Stage 4: Envelope Extraction

Extracts smooth envelope representing muscle activation level over time.

Moving Average (MAV)

MAV = (1/N) * Σ|xi|

Window size: 100-300 ms
Overlap: 50-90%
Output rate: 20-50 Hz

Root Mean Square (RMS) - Recommended

RMS = √((1/N) * Σxi²)

Window size: 150 ms (default)
Overlap: 75% (default)
Output rate: 27 Hz

Low-Pass Filter Alternative

Cutoff frequency: 2-6 Hz
Filter type: 4th-order Butterworth
Minimal phase distortion
Parameter Typical Value Trade-off
Window size 100-300 ms Shorter = faster response, more noise
Overlap 50-90% Higher = smoother output, more computation
Update rate 20-50 Hz Higher = more responsive, higher CPU load

4.6 Stage 5: Normalization

Scales EMG values to standard range for consistent interpretation across users and sessions.

MVC Normalization (Recommended)

normalized_value = (current_value / mvc_value) * 100

Output: Percentage of Maximum Voluntary Contraction (%MVC)
Range: 0-150% (clamped)
Requires: Prior MVC calibration session

MVC Calibration Protocol

  1. Rest baseline (5 seconds): Record resting EMG level
  2. Submaximal contraction (3 reps): 50% effort warm-up, 10 sec rest
  3. Maximum contraction (3 reps): 3-5 seconds each, 30 second rest between
  4. Calculate MVC: Use peak value or mean of top 3 peaks
  5. Validation: Verify consistency across repetitions (CV < 10%)

4.7 Feature Extraction

Computes numerical descriptors from EMG windows for pattern recognition.

Time-Domain Features

Feature Formula Interpretation
MAV (1/N) * Σ|xi| Average signal amplitude
RMS √((1/N) * Σxi²) Signal power
WL Σ|xi - xi-1| Signal complexity
ZC Count(sign changes) Frequency estimate
SSC Count(slope changes) Motor unit complexity

Frequency-Domain Features

Feature Description Use Case
Mean Frequency (MNF) Center of mass of power spectrum Fatigue indicator, shifts lower with fatigue
Median Frequency (MDF) Frequency dividing spectrum in half Robust fatigue measure
Peak Frequency Frequency with maximum power Dominant motor unit firing rate
Total Power Sum of all spectral power Overall muscle activation

Standard Feature Set (4 channels)

Time domain: MAV, RMS, WL, ZC, SSC (5 features × 4 channels = 20)
Frequency domain: MNF, MDF (2 features × 4 channels = 8)
Total: 28 features

4.8 Real-Time Processing Requirements

Stage Maximum Latency Throughput
Notch + Bandpass Filtering 5 ms 1000-2000 samples/sec
Rectification 0.1 ms Trivial computation
Envelope Extraction 10 ms Window-dependent
Feature Extraction 5 ms Per window (~200 ms)
Classification 10 ms Linear algebra operations
Total Pipeline < 50 ms Target for responsive control

Quality Metrics

Metric Good Acceptable Poor

4.10 Streaming Protocol

WebSocket Packet Structure

{
  "type": "emg_data",
  "sequenceNumber": 12345,
  "timestamp": 1705312800123,
  "deviceId": "EMG-SENSOR-001",
  "data": {
    "raw": {
      "ch0": [0.012, -0.008, 0.015, ...],
      "ch1": [-0.005, 0.003, -0.002, ...],
      "ch2": [0.018, 0.021, 0.017, ...],
      "ch3": [-0.012, -0.009, -0.015, ...]
    },
    "processed": {
      "ch0_rms": 0.156,
      "ch1_rms": 0.089,
      "ch2_rms": 0.234,
      "ch3_rms": 0.112
    }
  },
  "quality": {
    "snr": [28.3, 26.1, 31.2, 24.8],
    "impedance": [4.2, 5.1, 3.8, 6.3]
  }
}

Binary Packet Format (Efficient)

Header (8 bytes):
  [0-1]   Sequence number: uint16
  [2-5]   Timestamp delta (μs): uint32
  [6-7]   Sample count: uint16

Data:
  [8-N]   Interleaved samples: float32[channels × samples]
