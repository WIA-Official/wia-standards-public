Card Networks & ISO 8583 Protocol

4.1 ISO 8583 Message Standard

ISO 8583 is the international standard for financial transaction card originated messages. It defines the message format for authorization, clearing, and settlement between acquirers and issuers.

Message Structure

ISO 8583 Message Components:

[MTI][Bitmap][Data Elements]

Example Authorization Request (ASCII):
0100 7220000000C00000 4532123456789010=25121015432101234567

Breakdown:
0100                  - Message Type Indicator
7220000000C00000      - Bitmap (indicates which fields are present)
4532123456789010...   - Data elements (PAN, expiry, amount, etc.)

4.2 Card Network Routing

Transaction routing through card networks follows specific protocols and message flows.

Visa Network Protocol

Visa Transaction Flow:

1. Terminal → Acquirer
   - ISO 8583 0100 Authorization Request
   - Contains: PAN, amount, merchant ID, terminal data

2. Acquirer → VisaNet
   - Formatted to Visa specifications
   - BIN routing determines destination issuer

3. VisaNet → Issuer
   - Routes based on BIN table
   - Applies Visa risk rules
   - Logs for settlement

4. Issuer Processing
   - Account validation
   - Fraud checks
   - Authorization decision

5. Issuer → VisaNet → Acquirer → Terminal
   - ISO 8583 0110 Response
   - Response code (00=approved, etc.)
   - Auth code if approved

Timing: Typically 1-3 seconds end-to-end

Mastercard Network Protocol

Mastercard Transaction Flow:

Similar to Visa with differences:
- Banknet network infrastructure
- Different message format specifications
- Unique response codes
- CVC2 vs CVV2 naming
- Authorization Characteristics (Field 61)

Key Differences from Visa:
- Field 61: Point of Service Data
- Field 123: Verification Data
- Different BIN ranges
- Network assessment fee structure

4.3 EMV Transaction Flow

Chip Card Transaction Steps

1. Card Insertion/Contact
   - Terminal powers chip
   - Application selection (SELECT command)
   - Card returns Application Interchange Profile (AIP)

2. Initialize Transaction
   - GET PROCESSING OPTIONS command
   - Terminal sends amount, currency, transaction type
   - Card returns Application File Locator (AFL)

3. Read Application Data
   - READ RECORD commands
   - Retrieve card data (PAN, expiry, etc.)
   - Retrieve issuer parameters

4. Cardholder Verification
   - CVM List processing
   - PIN entry if required
   - Signature capture if needed
   - No CVM for contactless low value

5. Terminal Risk Management
   - Floor limit checking
   - Random transaction selection
   - Velocity checking

6. Card Risk Management
   - Card checks issuer risk parameters
   - Decides online/offline auth
   - Generates ARQC (online) or TC (offline)

7. Online Authorization (if required)
   - GENERATE AC command with ARQC
   - ISO 8583 message to issuer
   - Issuer validates ARQC
   - Returns ARPC in response

8. Second GENERATE AC
   - Terminal sends ARPC to card
   - Card validates issuer response
   - Generates TC (transaction certificate)

9. Transaction Completion
   - Receipt printing
   - Transaction logging
   - Card removal

Protocol Best Practices

  • Implement proper timeout handling (5-30 seconds)
  • Support reversal messages for failed transactions
  • Log all messages for reconciliation and dispute resolution
  • Use proper field validation before transmission
  • Implement retry logic with exponential backoff
  • Secure PIN handling with HSM encryption
  • Validate message authenticity with MAC (Message Authentication Code)
