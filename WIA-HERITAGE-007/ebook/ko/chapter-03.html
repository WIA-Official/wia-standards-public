WIA-HERITAGE-007 표준 개요

WIA-HERITAGE-007: Museum Digital Standard

03.1 표준의 전체 아키텍처

WIA-HERITAGE-007은 박물관 디지털 전환을 위한 포괄적인 생태계를 정의합니다. 이 표준은 단일 소프트웨어나 플랫폼이 아니라, 박물관이 자체적으로 구현하거나 기존 시스템을 통합할 수 있는 개방형 사양 세트입니다.

5계층 아키텍처

WIA-HERITAGE-007은 상호 독립적이면서도 함께 작동하는 5개의 계층으로 구성됩니다:

계층역할주요 기술예시
1. 데이터 계층소장품 정보의 구조화 저장LIDO, JSON-LD, SQL/NoSQL소장품 데이터베이스
2. 메타데이터 계층데이터의 의미론적 표현Dublin Core, VRA Core, CDWA메타데이터 스키마
3. API 계층프로그램 간 상호운용REST, GraphQL, IIIF검색/조회 API
4. 경험 계층관람객과의 인터랙션웹 앱, 모바일 앱, AR/VR디지털 전시
5. 분석 계층데이터 기반 의사결정BI 도구, ML 모델관람객 분석
🎯 설계 철학: 레고 블록 접근법

WIA-HERITAGE-007은 "레고 블록" 철학을 따릅니다. 각 구성 요소는 독립적으로 작동하면서도 서로 조립될 수 있습니다. 소규모 박물관은 필요한 블록만 사용하고, 대형 박물관은 모든 블록을 조합하여 복잡한 시스템을 구축할 수 있습니다.

예: 박물관은 자체 데이터베이스를 유지하면서도 표준 API만 추가하여 글로벌 플랫폼과 연동할 수 있습니다.

03.2 4단계 구현 접근법의 상세

WIA-HERITAGE-007의 핵심은 점진적 채택이 가능한 4단계(Phase) 구조입니다. 각 단계는 이전 단계의 성과를 기반으로 하며, 박물관은 자신의 준비 상태에 맞춰 진행할 수 있습니다.

Phase 1: 데이터 형식 표준화

항목내용
목표소장품 데이터를 국제 표준 형식으로 저장
핵심 기술LIDO XML/JSON, SPECTRUM 5.0 준수
구현 난이도⭐⭐ (중간)
예상 기간3-6개월
필요 인력큐레이터 1명 + IT 담당자 0.5명
예상 비용$15,000-50,000
즉시 혜택데이터 백업 및 복구 용이, 향후 마이그레이션 대비

단계별 실행 계획:

  1. Week 1-2: 현재 데이터 포맷 감사, 샘플 100개 선정
  2. Week 3-4: LIDO 메타데이터 스키마 설계, 매핑 테이블 작성
  3. Week 5-8: 변환 스크립트 개발, 샘플 100개 변환 및 검증
  4. Week 9-16: 전체 컬렉션 변환 (배치 프로세스)
  5. Week 17-20: 품질 검사, 누락 데이터 보완
  6. Week 21-24: 직원 교육, 신규 데이터 입력 프로세스 수립

Phase 2: API 인터페이스 구축

항목내용
목표외부 시스템이 프로그램적으로 컬렉션에 접근 가능
핵심 기술RESTful API, IIIF Image API, OAuth 2.0
구현 난이도⭐⭐⭐ (높음)
예상 기간6-12개월
필요 인력백엔드 개발자 1명 + DevOps 0.5명
예상 비용$50,000-150,000
즉시 혜택Google Arts & Culture 자동 연동, 연구자 API 접근 제공

핵심 엔드포인트:

  • GET /api/v1/objects: 소장품 목록 조회 (페이지네이션, 필터링)
  • GET /api/v1/objects/{id}: 개별 소장품 상세 정보
  • GET /api/v1/search?q={query}: 전문 검색
  • GET /api/v1/images/{id}/iiif/info.json: IIIF 이미지 정보
  • POST /api/v1/analytics/event: 관람객 행동 데이터 수집

예상 비용$100,000-300,000
즉시 혜택4K 가상 투어, 실시간 관람객 흐름 분석, 환경 모니터링

03.3 기존 표준과의 관계

WIA-HERITAGE-007은 박물관 분야의 기존 표준들을 대체하는 것이 아니라 통합하고 확장합니다. 각 기존 표준의 강점을 활용하면서 현대 기술 요구사항을 추가합니다.

기존 표준제정 기관WIA-HERITAGE-007과의 관계통합 방식
LIDO (Lightweight Information Describing Objects)ICOM CIDOCPhase 1의 핵심 메타데이터 형식으로 채택직접 사용
SPECTRUM 5.0Collections Trust (UK)소장품 관리 절차 표준으로 참조절차 지침

Dublin CoreDCMI기본 메타데이터 요소 세트하위 호환

03.4 레거시 시스템과의 공존 전략

WIA-HERITAGE-007의 독특한 강점은 기존 시스템을 완전히 대체하지 않고도 혜택을 누릴 수 있다는 점입니다. "Strangler Fig Pattern"을 사용하여 점진적으로 이전합니다.

3가지 공존 모드

🔄 모드 1: 읽기 전용 어댑터

시나리오: TMS, EMu 등 상업용 CMS를 계속 사용하면서 표준 API만 추가

구현: 기존 데이터베이스를 읽기 전용으로 접근하는 중간 API 서버 구축

장점: 기존 워크플로 무중단, 최소 비용 ($10K-30K)

단점: 양방향 동기화 불가, 레거시 시스템 의존 지속

적합 대상: 예산이 극히 제한적이지만 온라인 공개가 시급한 박물관

🔄 모드 2: 하이브리드 운영

시나리오: 신규 데이터는 WIA-HERITAGE-007로, 기존 데이터는 레거시 시스템에 유지

구현: 통합 검색 레이어로 두 시스템을 투명하게 연결

장점: 점진적 전환, 위험 분산

단점: 중복 관리 부담, 데이터 일관성 유지 복잡

적합 대상: 대형 박물관 (소장품 10만+ 점), 3-5년 전환 계획

🔄 모드 3: 완전 마이그레이션

시나리오: 모든 데이터를 WIA-HERITAGE-007 호환 시스템으로 이전

구현: ETL (Extract-Transform-Load) 파이프라인 구축하여 일괄 변환

장점: 단일 진실의 원천, 장기 유지보수 비용 최소화

단점: 높은 초기 비용과 위험, 6-18개월 작업

적합 대상: 신규 박물관 또는 레거시 시스템이 지원 종료된 경우

마이그레이션 위험 관리

위험발생 확률영향도완화 전략
데이터 손실중간 (35%)치명적3단계 백업 (원본+변환+검증), 샘플 테스트 먼저 수행
다운타임높음 (70%)높음블루-그린 배포, 롤백 계획 수립
직원 저항매우 높음 (85%)중간조기 교육, 챔피언 프로그램, 점진적 전환
예산 초과높음 (65%)높음25% 예비비 확보, 단계별 검증 후 다음 단계 승인
기술 호환성중간 (40%)중간POC (Proof of Concept) 먼저 실행, 전문가 자문

03.5 오픈소스 구현체: WIA-Museum-Kit

WIA 재단은 WIA-HERITAGE-007 표준의 참조 구현으로 "WIA-Museum-Kit"이라는 오픈소스 소프트웨어 스택을 제공합니다. 이는 박물관이 즉시 사용하거나 커스터마이징할 수 있는 완전한 솔루션입니다.

구성 요소

컴포넌트기술 스택역할라이선스
Backend APINode.js + Express + PostgreSQLRESTful API 서버MIT
Frontend WebReact + Next.js + Tailwind공개 웹사이트 및 관리 포털MIT
Mobile AppReact Native + ExpoiOS/Android 가이드 앱MIT
Image ServerPython + Pillow + IIIF고해상도 이미지 서빙MIT
Search EngineElasticsearch전문 검색 및 패싯Apache 2.0
AnalyticsMatomo (self-hosted)프라이버시 중심 분석GPL v3
Admin PanelReact Admin큐레이터 작업 도구MIT

설치 요구사항

# 최소 서버 사양 (소규모 박물관, 소장품 <10,000점)
- CPU: 4 코어
- RAM: 16GB
- 스토리지: 500GB SSD
- 대역폭: 100Mbps

# 권장 서버 사양 (중대형 박물관, 소장품 >10,000점)
- CPU: 8 코어 이상
- RAM: 32GB 이상
- 스토리지: 2TB SSD + 10TB HDD (백업)
- 대역폭: 1Gbps

빠른 시작 (Docker Compose)

# GitHub에서 클론
git clone https://github.com/WIA-Official/wia-museum-kit.git
cd wia-museum-kit

# 환경 변수 설정
cp .env.example .env
nano .env  # 데이터베이스 비밀번호 등 설정

# Docker Compose로 전체 스택 실행
docker-compose up -d

# 샘플 데이터 임포트
docker-compose exec api npm run seed

# 웹브라우저에서 http://localhost:3000 접속

10분 내에 완전히 작동하는 디지털 박물관이 로컬에서 실행됩니다!

03.6 인증 및 컴플라이언스 프로그램

WIA 재단은 표준 준수를 검증하는 공식 인증 프로그램을 운영합니다. 인증을 받은 박물관은 WIA 네트워크의 신뢰받는 구성원이 되며, 글로벌 디지털 박물관 연합에 참여할 수 있습니다.

3단계 인증 레벨

레벨요구사항혜택연간 비용

🥈 SilverPhase 1+2 완료, 표준 API 운영API 키 발급, 글로벌 검색 노출$500
🥇 GoldPhase 1+2+3 완료, 보안 감사 통과WIA 컨퍼런스 무료, 기술 지원 우선$2,000
💎 PlatinumPhase 1-4 완료, 사례 연구 공유표준 개발 참여, 컨설팅 할인$5,000

인증 프로세스

  1. 자가 평가: 온라인 체크리스트 작성 (무료)
  2. 기술 검토: WIA 팀이 API 및 데이터 품질 검증 (2주)
  3. 보안 감사: 제3자 보안 전문가의 침투 테스트 (Gold 이상만 해당)
  4. 인증서 발급: 디지털 인증서 및 물리적 플라크 발송
  5. 연례 갱신: 매년 간단한 업데이트 검토로 갱신

2024년 12월 기준, 전 세계 234개 박물관이 WIA-HERITAGE-007 인증을 받았으며, 이 중 87개가 Gold 이상입니다.

의사결정 프로세스

WIA-HERITAGE-007은 "대략적인 합의와 실행 코드(Rough Consensus and Running Code)" 원칙을 따릅니다:

  1. 제안서 제출: 누구나 GitHub Issues에 개선 제안 가능
  2. 커뮤니티 토론: 최소 2주간 공개 토론
  3. 프로토타입 구현: 제안자가 작동하는 프로토타입 제시
  4. 파일럿 테스트: 최소 3개 박물관에서 현장 테스트
  5. 투표: 기술 작업반 2/3 이상 찬성 시 채택
  6. 표준 통합: 다음 버전 릴리스에 포함

이 과정은 평균 6-9개월 소요되며, 급하게 도입되는 미성숙한 기능을 방지합니다.

03.8 장 요약

✅ 핵심 내용
