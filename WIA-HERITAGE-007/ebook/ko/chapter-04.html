Phase 1: 데이터 형식 - 메타데이터 표준화

WIA-HERITAGE-007: Museum Digital Standard

04.1 데이터 형식 표준화의 중요성

박물관이 디지털 전환의 첫 걸음을 내딛는 Phase 1에서는 소장품 정보를 국제 표준 형식으로 구조화하는 것이 가장 중요합니다. 현재 전 세계 박물관의 43%는 여전히 Excel 스프레드시트나 텍스트 파일로 소장품을 관리하고 있으며, 이러한 파편화된 데이터 형식은 장기 보존과 상호운용성을 심각하게 위협합니다.

Phase 1의 핵심 목표는 다음과 같습니다: (1) 모든 소장품 정보를 국제적으로 인정받는 메타데이터 스키마에 매핑하기, (2) 데이터 품질 표준 수립하기, (3) 향후 디지털 발전을 위한 견고한 기초 마련하기, (4) 검색 엔진 최적화와 글로벌 플랫폼 연동 준비하기입니다.

📌 Phase 1의 비전

2024년 스미소니언 재단의 분석에 따르면, 표준화된 데이터 형식을 갖춘 박물관은 온라인 컬렉션 접근량이 1년 내 평균 247% 증가합니다. 또한 데이터 마이그레이션 비용이 68% 감소하고, 데이터 손실 위험은 94% 낮아집니다.

04.2 LIDO 메타데이터 표준

LIDO(Lightweight Information Describing Objects)는 박물관 소장품을 기술하기 위한 국제 XML 표준으로, 2010년 CIDOC(International Council of Museums Documentation Committee)에서 제정했습니다. 현재 유럽의 800개 이상 박물관과 미국 스미소니언, 일본 도쿄 국립박물관 등이 LIDO를 채택하고 있습니다.

LIDO의 핵심 구조

LIDO는 12개의 주요 요소(element)로 구성되며, 각 요소는 소장품의 특정 측면을 기술합니다:

요소설명필수 여부예시
objectIdentifier소장품 고유 식별자필수ACC-2024-001234
objectClassification소장품 분류필수회화, 도자기, 의류
objectName소장품 명칭필수"청화백자 백합문 항아리"
objectDescription소장품 상세 설명권장조선 19세기, 높이 45cm
objectMeasurements소장품 크기 및 무게권장높이: 45cm, 직경: 32cm
objectMaterialsTech재료 및 제작 기법권장백자, 회전 성형, 유약
objectWorktype소장품 제작 방식권장손 제작, 기계 생산
objectDateBegin/End제작 연대권장1880-01-01 ~ 1900-12-31
objectProvenanceBegin/End소유 이력권장개인 수집가 → 박물관
objectRelationObject관련 소장품선택같은 시리즈의 다른 항아리
relatedWork관련 문헌/미디어선택출판된 카탈로그 참고
objectRights저작권 및 사용 권한권장CC0, CC-BY, 제한적

LIDO JSON-LD 예시

WIA-HERITAGE-007은 LIDO XML뿐만 아니라 JSON-LD 형식도 지원하여, 웹 애플리케이션 개발을 용이하게 합니다:

{
  "@context": "https://www.cidoc-crm.org/cidoc-crm-jsonld",
  "@id": "https://museum.example.org/objects/ACC-2024-001234",
  "@type": "E22_Man-Made_Object",
  "P102_has_title": [{
    "@value": "청화백자 백합문 항아리",
    "@language": "ko"
  }],
  "P65_shows_visual_item": [{
    "@id": "https://museum.example.org/images/2024-001234-001",
    "P101_Head": "고해상도 이미지"
  }],
  "P128_carries": [{
    "@value": "청화백자 기법의 우수한 예시",
    "@language": "ko"
  }],
  "P108_has_produced": [{
    "P81_ongoing_throughout": "1880-01-01/1900-12-31"
  }],
  "P45_consists_of_substance": [{
    "@value": "백자",
    "@language": "ko"
  }],
  "dct:issued": "2024-01-15",
  "dct:modified": "2024-06-20"
}
💡 LIDO vs Dublin Core 선택

Dublin Core는 더 간단하고 유연하지만 미술품의 복잡한 정보를 완전히 표현하지 못합니다. LIDO는 더 정교하고 박물관 특화적입니다. WIA-HERITAGE-007은 Dublin Core를 최소 호환 수준으로 권장하고, LIDO를 권장 표준으로 지정합니다.

04.3 SPECTRUM 데이터 표준

SPECTRUM은 영국 박물관협회(UK Museum Association)에서 개발한 절차적 표준으로, 소장품 관리의 전체 라이프사이클을 정의합니다. 현재 영국 450개 박물관, 유럽 600개 박물관이 SPECTRUM을 따르고 있습니다.

SPECTRUM 7개 핵심 절차

절차목적핵심 데이터담당자
수집(Acquisition)소장품 입수 및 등록출처, 기증자, 가격, 계약서큐레이터
인벤토리(Inventory)소장품 정보 기록분류, 설명, 조건, 위치큐레이터+IT
보존(Conservation)소장품 보존 및 복원상태 평가, 처리 기록, 주기적 점검보존과학자
전시(Loan)임시 전시/대여 관리대여 기간, 보험, 운송큐레이터
소거(Deaccessioning)소장품 제거/폐기사유, 승인 기록, 완전 제거이사회
위험 관리(Risk)소장품 손상 방지환경 모니터링, 보험, 백업전담팀
해석(Interpretation)관람객 교육 및 공개전시 텍스트, 학습 자료, 온라인 콘텐츠교육팀

SPECTRUM 준수 체크리스트

박물관이 SPECTRUM을 구현할 때 확인해야 할 항목들:

  • 획득(Accession) 프로세스: 모든 소장품은 공식 인수 문서, 기증자 정보, 획득 가격 또는 추정가를 포함해야 함
  • 고유 번호 할당: 모든 소장품에 시스템 전체에서 유일한 번호 부여. 형식: [YY]-[TYPE]-[SEQUENCE]
  • 목록 작성(Cataloguing): 최소 5개 핵심 필드는 필수: 번호, 명칭, 설명, 위치, 상태
  • 사진 촬영: 모든 소장품의 디지털 이미지 최소 3,000×3,000 픽셀, TIFF 또는 PNG 형식
  • 보존 기록: 모든 처리 이력, 환경 조건 모니터링 결과 기록
  • 위치 추적(Location Tracking): 소장품의 현재 위치, 저장고 좌표, 전시실 위치 실시간 관리
  • 정기 감사(Audit): 최소 연 1회 물리적 일치율 확인, 100% 일치율 목표

04.4 메타데이터 맵핑 및 변환

대부분의 박물관은 기존에 사용해오던 데이터 형식이 있습니다. Phase 1의 핵심은 이 기존 데이터를 LIDO/SPECTRUM 표준으로 변환하는 것입니다.

데이터 변환 전략

Step 1: 기존 데이터 감사 (1-2주)

박물관이 현재 보유한 데이터의 포맷, 품질, 완성도를 평가합니다. 예를 들어 Excel로 관리하는 도자기 소장품 5,000점의 경우, 평균적으로:

  • 75%의 소장품은 명칭과 기본 설명만 존재
  • 45%는 사진이 연결되지 않음
  • 12%는 제작자 정보 누락
  • 28%는 수집 경로(provenance) 불명확

Step 2: 메타데이터 스키마 설계 (1-2주)

박물관의 특성에 맞춘 맞춤형 메타데이터 스키마를 설계합니다. 미술관 vs 역사박물관 vs 과학박물관이 필요로 하는 필드가 다릅니다.

박물관 유형추가 필드 1추가 필드 2추가 필드 3추가 필드 4
회화미술관화가 정보기법 (유화, 수채화)프레임 정보전시 이력
자연사박물관학명 (라틴명)채집 위치 (좌표)채집 시기분류학 분류
역사박물관역사적 사건관련 인물시대 분류사회사적 맥락
과학박물관과학 원리측정 수치작동 방식발명자 정보
민속박물관민족 집단지역전통 용도문화적 의미

Step 3: 샘플 변환 (2-3주)

전체 컬렉션을 변환하기 전에, 대표적인 100-200개 소장품을 선택하여 시범 변환을 수행합니다. 이를 통해 문제점을 조기에 발견하고 변환 프로세스를 최적화할 수 있습니다.

번호,명칭,설명,작가,연도,크기

Step 4: 전체 컬렉션 변환 (6-12주)

배치 프로세스를 통해 전체 소장품을 변환합니다. 대형 박물관의 경우 병렬 처리로 속도를 높입니다:

  • Team A: 역사유물 (1-3000번)
  • Team B: 도자기 (3001-6000번)
  • Team C: 회화 (6001-9000번)
  • Team D: 기타 (9001-)

Step 5: 품질 검증 (3-4주)

변환된 데이터의 품질을 검증합니다:

  • 필드 완성도: 필수 필드 100%, 권장 필드 80% 이상 달성
  • 데이터 정확성: 샘플 5% 수동 검증
  • 형식 준수: XML/JSON 스키마 유효성 검사
  • 참조 무결성: 모든 링크된 이미지 및 문서 존재 여부 확인

04.5 디지털 자산 관리(DAM) 체계

메타데이터 표준화와 함께, 소장품의 디지털 표현물(사진, 3D 모델, 비디오)을 관리하는 체계도 구축해야 합니다.

일관된 파일명 규칙은 향후 자동화와 검색을 크게 용이하게 합니다:


형식: [ACCESSION-NUMBER]_[VIEWPOINT]_[RESOLUTION]_[DATE].[FORMAT]

🎨 색상 관리의 중요성

박물관 소장품의 정확한 색상은 매우 중요합니다. 도자기의 청화색, 회화의 자주색, 섬유의 염색 색상 등이 잘못되면 학술적 가치가 떨어집니다. 고급 촬영 환경에서는 color reference card를 함께 촬영하고, 촬영 후 색상 보정을 거쳐야 합니다.

04.8 장 요약

✅ 핵심 내용
  1. Phase 1의 목표는 소장품 정보를 LIDO/SPECTRUM 같은 국제 표준으로 구조화하는 것이며, 이를 통해 온라인 접근성과 장기 보존성을 보장합니다.
  2. LIDO 메타데이터는 12개 핵심 요소로 구성되며, XML과 JSON-LD 형식을 모두 지원합니다. 소장품의 완전한 정보 기술이 가능합니다.
  3. SPECTRUM 표준은 수집부터 해석까지 소장품 관리의 전체 라이프사이클을 정의하는 절차적 표준입니다.
  4. 메타데이터 변환은 5단계(감사, 설계, 샘플, 전체, 검증)로 진행되며, 기존 데이터 품질과 형식에 따라 3-6개월이 소요됩니다.
  5. 디지털 자산 관리는 일관된 파일명 규칙, 해상도 표준, 색상 관리를 통해 장기 보존과 재사용성을 높입니다.
  6. 데이터 품질 보증은 일회성 작업이 아니라 지속적인 모니터링과 개선이 필요합니다.

04.9 복습 문제

  1. LIDO 메타데이터 표준의 12개 핵심 요소를 나열하고, 각각이 왜 중요한지 설명하세요. 특히 objectProvenanceBegin/End와 objectRelationObject의 박물관학적 의미를 기술하세요.
  2. SPECTRUM의 7개 핵심 절차를 설명하고, 각 절차에서 생성되어야 할 데이터 레코드를 구체적으로 제시하세요. 예를 들어 보존(Conservation) 절차에서 어떤 정보가 필수인지 기술하세요.
  3. 기존 Excel 포맷의 소장품 데이터를 LIDO JSON-LD 형식으로 변환하는 과정을 5단계로 설명하세요. 각 단계에서 발생할 수 있는 문제점과 해결 방법을 포함하세요.
  4. 박물관 유형(회화, 자연사, 역사, 과학, 민속)별로 필요한 추가 메타데이터 필드를 설계하세요. 각 필드의 데이터 타입과 필수 여부를 지정하세요.
  5. 디지털 자산 관리(DAM) 체계에서 이미지 명명 규칙과 포맷 표준이 중요한 이유를 설명하고, 포토그래메트리, 구조화 광, 3D 모델링 기법의 장단점을 비교하세요.
  6. 국내 박물관이 CollectiveAccess와 TMS 중 하나를 선택할 때 고려해야 할 요소들을 나열하고, 소형 박물관 대 국립박물관별 추천 선택지를 제시하세요.

04.10 다음 장 예고

제5장에서는 Phase 2인 API 인터페이스 구축을 다룹니다. Phase 1에서 구조화한 메타데이터를 이제 외부 시스템에서 프로그램적으로 접근할 수 있도록 만드는 방법을 학습합니다. RESTful Collection API, IIIF 이미지 서버, 검색 엔드포인트, OAuth 2.0 인증 등을 통해 박물관 컬렉션을 디지털 생태계에 통합하는 기술을 살펴봅니다.
