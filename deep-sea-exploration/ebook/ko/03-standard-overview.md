# 제3장: WIA 표준 개요

## WIA 심해 탐사 표준의 아키텍처와 설계 원칙

---

## 3.1 표준 범위와 목표

### 경계 정의

WIA 심해 탐사 표준은 수중 차량과 해양 관측 시스템에서 데이터의 수집, 형식화, 전송, 보관을 위한 포괄적인 사양을 제공합니다. 이 표준은 센서에서 아카이브까지 완전한 데이터 수명주기를 다루어 상호운용성과 장기적 가치를 보장합니다.

**범위 내**:

| 범주 | 포함 요소 |
|------|----------|
| 데이터 형식 | 원격측정, 해양학, 수심측량, 생물학, 지질학, 비디오 메타데이터 |
| API | 차량 제어, 데이터 접근, 미션 관리, 스트리밍 |
| 프로토콜 | 음향 통신, 파일 전송, 실시간 스트리밍 |
| 통합 | 센서 프레임워크, 클라우드 연결, 시각화 |
| 인증 | 준수 수준, 테스트 절차, 인증 프로세스 |

**범위 외**:

| 범주 | 이유 | 참조 표준 |
|------|------|----------|
| 차량 기계 설계 | 응용별로 다양 | 선급 협회 |
| 압력 용기 인증 | 안전 중요, 규제됨 | ASME, DNV |
| 전기 안전 | 기존 표준으로 커버됨 | IEC 60335 |
| 생명 유지 (HOV) | 전문 분야 | 선급 협회 |
| 음향 모뎀 하드웨어 | 제조업체별 | ITU-T |

### 전략적 목표

WIA 심해 탐사 표준의 목표:

1. **상호운용성**: 전 세계 기관, 차량, 시스템 간의 원활한 데이터 교환 가능
2. **효율성**: 현재 데이터 변환 및 재포맷에 사용되는 연구 시간의 20-30% 감소
3. **품질**: 데이터에 과학적 유효성과 장기 사용성에 필요한 메타데이터 포함 보장
4. **혁신**: 잠금 없이 기술 개발을 장려하는 안정적인 기반 제공
5. **접근성**: 기관 자원에 관계없이 연구자들이 심해 데이터에 접근 가능
6. **지속 가능성**: 해양 데이터의 장기 보관 및 발견 가능

## 3.2 4단계 구현

WIA 심해 탐사 표준은 각각 이전 단계를 기반으로 하는 4단계로 구성됩니다:

### Phase 1: 데이터 형식 사양

**목적**: 모든 데이터 유형의 구조와 내용 정의

**주요 결과물**:
- 검증 스키마가 있는 JSON 기반 메시지 형식
- 대역폭 제한 시나리오를 위한 이진 형식
- 파일 구성 및 보관 구조
- 모든 데이터 유형에 대한 메타데이터 요구사항

**핵심 데이터 유형**:

| 데이터 유형 | 주요 용도 | 업데이트 속도 |
|------------|----------|-------------|
| 차량 원격측정 | 실시간 모니터링 | 1-10 Hz |
| 해양학 데이터 | 환경 조건 | 0.1-1 Hz |
| 수심측량 데이터 | 해저 매핑 | 조사 기반 |
| 샘플 메타데이터 | 생물/지질 샘플 | 이벤트 기반 |
| 항법 데이터 | 위치 및 방향 | 1-10 Hz |
| 비디오 메타데이터 | 미디어 인덱싱 | 프레임 기반 |
| 미션 로그 | 이벤트 문서화 | 이벤트 기반 |

### Phase 2: API 인터페이스 사양

**목적**: 데이터 접근 및 차량 제어를 위한 프로그래매틱 인터페이스 정의

**주요 결과물**:
- RESTful API 사양 (OpenAPI 3.0)
- WebSocket 스트리밍 프로토콜
- GraphQL 쿼리 인터페이스 (선택)
- 인증 및 권한 부여 프레임워크
- SDK 참조 구현

**API 범주**:

| 범주 | 엔드포인트 | 메서드 |
|------|----------|--------|
| 원격측정 | /telemetry | GET, WebSocket |
| 데이터 접근 | /data, /samples, /bathymetry | GET, POST |
| 미션 | /missions, /waypoints | GET, POST, PUT, DELETE |
| 차량 제어 | /commands, /configuration | POST, GET |
| 시스템 | /health, /version, /calibration | GET |

### Phase 3: 통신 프로토콜

**목적**: 수중 환경에서 신뢰할 수 있는 데이터 전송을 위한 프로토콜 지정

**주요 결과물**:
- 음향 통신 패킷 형식
- 오류 수정 및 재시도 메커니즘
- 우선순위 기반 데이터 전송
- 다중 차량 네트워킹
- 저장 후 전달 프로토콜

**프로토콜 스택**:

```
┌─────────────────────────────────────┐
│     애플리케이션 (WIA 데이터)       │
├─────────────────────────────────────┤
│   전송 (신뢰성/최선 노력)           │
├─────────────────────────────────────┤
│   네트워크 (음향 주소 지정)         │
├─────────────────────────────────────┤
│  물리 (음향/광학/광섬유)            │
└─────────────────────────────────────┘
```

### Phase 4: 시스템 통합

**목적**: 종단간 시스템 통합을 위한 지침 제공

**주요 결과물**:
- 참조 아키텍처
- 센서 통합 프레임워크
- 클라우드 연결 패턴
- 시각화 요구사항
- 테스트 및 검증 절차

## 3.3 준수 수준

WIA 표준은 세 가지 준수 수준을 정의하여 조직이 능력과 요구에 따라 구현할 수 있습니다:

### Level 1: 기본 준수

**요구사항**:
- 핵심 메시지 형식 구현 (기본 헤더 필드)
- 최소 세 가지 데이터 유형 지원
- WIA JSON 스키마에 대한 메시지 검증
- 기본 메타데이터 제공 (소스, 타임스탬프, 체크섬)

**적합 대상**:
- 소규모 연구선
- 교육 기관
- 프로토타입 시스템
- 레거시 시스템 통합

**인증 프로세스**:
- 테스트 스위트로 자체 인증
- 온라인 검증 도구
- 공식 감사 불필요

### Level 2: 표준 준수

**요구사항**:
- 모든 Level 1 요구사항
- 모든 Phase 1 데이터 유형 구현
- 핵심 Phase 2 API 구현
- 사양에 따른 전체 메타데이터
- 저대역폭 시나리오를 위한 이진 형식 지원
- 오류 처리 및 재시도 메커니즘

**적합 대상**:
- 대학 연구선
- 상업 조사 운영
- 국가 연구 기관
- 산업 표준 장비 제조업체

**인증 프로세스**:
- 공식 테스트 스위트 실행
- 샘플 데이터 제출
- 문서 검토
- WIA에서 인증서 발급

### Level 3: 고급 준수

**요구사항**:
- 모든 Level 2 요구사항
- 전체 Phase 2 API 구현
- Phase 3 프로토콜 구현
- 참조 아키텍처와 Phase 4 통합
- 보장된 전달이 있는 실시간 스트리밍
- 클라우드 통합 기능
- 확장된 메타데이터 및 출처 추적

**적합 대상**:
- 주요 연구 기관 (WHOI, MBARI, JAMSTEC, KIOST)
- 심해 채광 운영
- 정부 기관
- 플랫폼 제조업체

**인증 프로세스**:
- WIA 인증 기관의 전체 감사
- 참조 구현과의 상호운용성 테스트
- 연간 재인증
- WIA 인증 제품 레지스트리에 등재

### 준수 매트릭스

| 요구사항 | Level 1 | Level 2 | Level 3 |
|---------|---------|---------|---------|
| 기본 메시지 형식 | ✓ | ✓ | ✓ |
| JSON 스키마 검증 | ✓ | ✓ | ✓ |
| 세 가지 데이터 유형 | ✓ | ✓ | ✓ |
| 모든 데이터 유형 | - | ✓ | ✓ |
| 핵심 API | - | ✓ | ✓ |
| 전체 API | - | - | ✓ |
| 이진 형식 | - | ✓ | ✓ |
| 프로토콜 구현 | - | - | ✓ |
| 클라우드 통합 | - | - | ✓ |
| 자체 인증 | ✓ | - | - |
| 공식 인증 | - | ✓ | ✓ |
| 연간 감사 | - | - | ✓ |

## 3.4 기존 표준과의 관계

### 조화 접근법

WIA 표준은 기존 해양학 표준을 대체하려는 것이 아니라 조화시키고 확장하는 통합 프레임워크를 제공합니다:

**설계 원칙**:
1. **채택**: 기존 표준의 입증된 요소 통합
2. **확장**: 기존 표준에서 다루지 않는 기능 추가
3. **매핑**: 명확한 변환 경로 제공
4. **공존**: 전환 중 병렬 사용 지원

### 관련 표준

| 표준/조직 | 초점 영역 | WIA 관계 |
|----------|----------|----------|
| NOAA 데이터 표준 | 미국 해양 데이터 | 호환, 확장됨 |
| OOI 데이터 제품 | 관측소 데이터 | 호환 형식 |
| MBARI VARS | 비디오 주석 | 메타데이터 정렬 |
| ISO 19115 | 지리 메타데이터 | 프로파일 통합 |
| CF 규약 | NetCDF 속성 | 매핑 제공 |
| OGC SensorML | 센서 설명 | 호환 |

### NOAA 표준 정렬

WIA 표준은 NOAA의 해양학 데이터 표준과 정렬됩니다:

**수심측량 데이터**:
- BAG (Bathymetric Attributed Grid) 형식과 호환
- NOAA의 NCEI 보관 요구사항 지원
- 다중빔 관련 메타데이터 포함

**수주 데이터**:
- NOAA의 WCD 형식 요구사항과 정렬
- CTD 데이터가 SeaDataNet 형식과 호환
- NOAA의 ERDDAP 데이터 서버 지원

**생물 관찰**:
- Darwin Core 표준에 매핑
- OBIS (Ocean Biodiversity Information System) 호환
- GBIF 준비 내보내기 옵션

## 3.5 인증 프로세스

### 개요

WIA 인증은 시스템이 표준을 준수하고 다른 인증 시스템과 상호운용할 수 있음을 증명합니다. 인증 프로세스는 준수 수준에 따라 다릅니다.

### Level 1 자체 인증

**프로세스**:
1. WIA 검증 툴킷 다운로드
2. 시스템에 대해 자동화된 테스트 실행
3. 통과 테스트 보고서를 WIA 레지스트리에 제출
4. Level 1 인증 배지 수령

**타임라인**: 1-2일
**비용**: 무료

### Level 2 공식 인증

**프로세스**:

1. **신청** (1주차)
   - 신청서 제출
   - 시스템 문서 제공
   - 인증 수수료 지불

2. **문서 검토** (2-3주차)
   - WIA 검토자가 문서 평가
   - 공백 또는 문제 식별
   - 수정을 위한 피드백 제공

3. **테스트 실행** (4-5주차)
   - 전체 테스트 스위트 실행
   - 테스트 결과 및 샘플 데이터 제출
   - WIA가 결과 검증

4. **인증 결정** (6주차)
   - 검토 위원회가 준수 평가
   - 인증서 발급 또는 시정 목록
   - 인증 제품 레지스트리에 게시

**타임라인**: 6-8주
**비용**: $2,500 USD

### Level 3 고급 인증

**프로세스**:

1. **사전 평가** (1개월)
   - 초기 시스템 평가
   - 통합 요구사항 식별
   - 상호운용성 테스트 계획

2. **구현 검토** (2개월)
   - 현장 또는 원격 감사
   - 코드 및 아키텍처 검토
   - 문서 평가

3. **상호운용성 테스트** (3개월)
   - WIA 참조 구현에 대해 테스트
   - 다중 시스템 데이터 교환 테스트
   - 실제 시나리오 검증

4. **인증** (4개월)
   - 최종 보고서 및 권장사항
   - 인증서 발급
   - 연간 재인증 일정

**타임라인**: 3-4개월
**비용**: $10,000-25,000 USD (시스템 복잡성에 따라)

## 3.6 버전 관리 및 업데이트

### 버전 체계

WIA 표준은 시맨틱 버저닝(SemVer)을 사용합니다:

```
MAJOR.MINOR.PATCH
  │      │     │
  │      │     └── 버그 수정, 명확화
  │      └──────── 새 기능, 하위 호환
  └─────────────── 주요 변경사항
```

**현재 버전**: 1.0.0

**버전 히스토리**:

| 버전 | 날짜 | 변경사항 |
|------|------|---------|
| 1.0.0 | 2025-01 | 초기 릴리스 |
| 0.9.0 | 2024-09 | 릴리스 후보 |
| 0.8.0 | 2024-06 | 베타, Phase 4 완료 |
| 0.5.0 | 2024-03 | 알파, Phase 2 완료 |

### 업데이트 프로세스

**연간 검토 주기**:
1. 1-3월: 피드백 및 변경 요청 수집
2. 4-6월: 워킹 그룹 검토 및 초안 작성
3. 7-8월: 공개 의견 기간
4. 9월: 최종 수정
5. 10-11월: 상호운용성 테스트
6. 12월: 버전 릴리스

### 하위 호환성

WIA 표준은 하위 호환성을 우선시합니다:

**보장됨**:
- Level 2 시스템이 Level 1 데이터를 읽음
- Level 3 시스템이 Level 1 및 Level 2 데이터를 읽음
- 마이너 버전 업데이트는 하위 호환
- 마이너 버전에서 필수 필드 제거 안 됨

**전환 기간**:
- 주요 변경사항 12개월 전 공지
- 전환 중 이전 형식과 새 형식 모두 지원
- 데이터 마이그레이션 도구 제공
- 인증 시스템에 재인증 연장 부여

## 장 요약

WIA 심해 탐사 표준은 4단계 접근법을 통해 해양학 데이터 상호운용성을 위한 포괄적인 프레임워크를 제공합니다: 데이터 형식, API 인터페이스, 통신 프로토콜, 시스템 통합. 세 가지 준수 수준(기본, 표준, 고급)은 조직이 능력에 따라 구현할 수 있도록 하며, 각 수준에 대한 명확한 인증 경로가 있습니다.

표준은 기존 해양학 표준(NOAA, OOI, CF 규약)과 조화되면서 수중 차량 데이터, 실시간 스트리밍, 시스템 통합의 공백을 해결하기 위해 기능을 확장합니다.

WIA 표준을 구현하는 조직은 데이터 변환 노력 감소, 파트너와의 협업 개선, 미래 대비 데이터 보관의 혜택을 받습니다.

---

## 핵심 요점

1. **4단계가 점진적으로 구축됨**: 데이터 형식 → API → 프로토콜 → 통합
2. **세 가지 준수 수준이 다양한 요구 수용**: 기본(자체인증), 표준(공식), 고급(감사)
3. **표준은 기존 표준과 조화**되며 대체하지 않음
4. **인증은 상호운용성을 입증**하며 Level 2/3에 필수
5. **시맨틱 버저닝은 안정성을 보장**하며 명확한 하위 호환성 보장
6. **확장 메커니즘은 벤더별 데이터를** 표준 단편화 없이 허용
