2. 네이버 (NAVER)

2.1 네이버 정책 개요

네이버는 2024년 8월부터 "디지털 유산 관리 서비스"를 정식 출시했습니다. 이는 국내 포털 중 최초로 체계적인 사망자 계정 처리 시스템을 갖춘 사례입니다.

서비스 처리 방법 필요 서류 처리 기간
네이버 계정 삭제 또는 추모 전환 사망진단서, 가족관계증명서 7-14일
네이버 블로그 비공개, 추모, 삭제 선택 법원 결정문 또는 유언장 14-21일
네이버 페이 잔액 상속인 이체 법원 결정문 + 상속인 동의 21-30일
네이버 클라우드 데이터 다운로드 후 삭제 집행자 인증서 즉시-7일
애드포스트 수익 정산 후 지급 법원 결정문 + 계좌정보 30-60일

2.2 네이버 디지털 유산 신청 절차

/**
 * 네이버 디지털 유산 처리 API
 * @platform NAVER
 * @endpoint https://api.naver.com/v1/digital-estate
 */

interface NaverEstateRequest {
  // 인증 정보
  authentication: {
    executorId: string;           // 집행자 ID
    courtOrderNumber: string;     // 법원 결정문 번호
    digitalCertificate: string;   // 공동인증서
  };

  // 고인 정보
  deceased: {
    naverId: string;              // 네이버 아이디
    name: string;
    dateOfDeath: Date;
    deathCertificate: string;     // 사망진단서 파일
  };

  // 처리 요청
  requestType: {
    action: 'DELETE' | 'MEMORIAL' | 'TRANSFER' | 'DOWNLOAD';
    services: [
      'NAVER_ACCOUNT',
      'NAVER_BLOG',
      'NAVER_CAFE',
      'NAVER_PAY',
      'NAVER_CLOUD',
      'ADPOST'
    ];
  };

  // 추모 계정 설정 (선택)
  memorialSettings?: {
    displayName: string;
    memorialMessage: string;
    allowComments: boolean;
    visibleTo: 'PUBLIC' | 'FRIENDS' | 'FAMILY';
  };

  // 데이터 다운로드 옵션
  downloadOptions?: {
    includePhotos: boolean;
    includeVideos: boolean;
    includePosts: boolean;
    includeComments: boolean;
    format: 'ZIP' | 'JSON' | 'HTML';
  };
}

// 네이버 응답
interface NaverEstateResponse {
  requestId: string;
  status: 'PENDING' | 'IN_PROGRESS' | 'COMPLETED' | 'REJECTED';
  estimatedCompletion: Date;

  processedServices: Array<{
    service: string;
    action: string;
    completedAt: Date;
    downloadUrl?: string;          // 데이터 다운로드 링크
  }>;

  financialSettlement?: {
    naverPay Balance: number;
    adpostRevenue: number;
    totalAmount: number;
    transferredTo: string;
    transferDate: Date;
  };
}

3. 카카오 (Kakao)

3.1 카카오 정책 개요

카카오는 2025년 1월부터 "카카오 가족 인증 시스템"을 도입하여 유족의 신원 확인 절차를 강화했습니다.

서비스 자동 휴면 기간 유족 처리 방법 필요 인증
카카오톡 12개월 계정 삭제 또는 대화 백업 Level 3
카카오페이 6개월 잔액 이체 (법원 승인) Level 4
카카오스토리 24개월 추모 전환 또는 삭제 Level 2
카카오톡 채널 없음 운영자 변경 또는 폐쇄 Level 3
다음 블로그 12개월 데이터 다운로드 가능 Level 2

3.2 카카오톡 대화 백업 프로세스

⚠️ 카카오톡 대화 백업 제한사항

개인정보보호법 준수: 카카오톡 대화는 제3자의 개인정보가 포함되어 있어 전체 백업이 제한됩니다.

  • 허용: 1:1 대화 (고인과 신청인 간)
  • 제한: 그룹 대화 (다른 참여자 동의 필요)
  • 금지: 타인 간 대화 (고인이 참여하지 않은 대화)

백업 기간: 사망일로부터 6개월 이내 신청 (이후 자동 삭제)

백업 형식: PDF 또는 텍스트 파일 (이미지, 동영상 별도)

4. 구글 (Google)

4.1 비활성 계정 관리자 (Inactive Account Manager)

구글은 2013년부터 "비활성 계정 관리자" 기능을 제공하여, 사용자가 생전에 사망 후 계정 처리 방법을 미리 설정할 수 있습니다.

✅ 구글 비활성 계정 관리자 설정 권장사항

  1. 비활성 기간 설정: 3개월, 6개월, 12개월, 18개월 중 선택
  2. 신뢰할 수 있는 연락처 지정: 최대 10명까지 가능
  3. 알림 설정: 비활성 기간 만료 1개월 전 SMS/이메일 통지
  4. 데이터 공유 범위: Gmail, Drive, Photos 등 서비스별 선택
  5. 계정 삭제 옵션: 데이터 공유 후 자동 삭제 여부 결정

4.2 구글 서비스별 처리 방법

서비스 사전 설정 시 사후 신청 시 데이터 보존 기간
Gmail 즉시 접근 가능 법원 명령 필요 무기한
Google Drive 다운로드 가능 소유자 이전 요청 무기한

5. 메타 (Meta - Facebook, Instagram)

5.1 추모 계정 (Memorialized Account) 정책

메타는 2015년부터 추모 계정 기능을 제공하며, 2019년에는 "유산 담당자(Legacy Contact)" 제도를 도입했습니다.

/**
 * 메타 추모 계정 설정
 * @platform Meta (Facebook, Instagram)
 */

interface MetaMemorialAccount {
  // 유산 담당자 (생전 지정)
  legacyContact?: {
    userId: string;               // 메타 사용자 ID
    permissions: {
      manageTribute: boolean;     // 추모글 관리
      updateProfile: boolean;     // 프로필 사진 변경
      respondRequests: boolean;   // 친구 요청 응답
      pinPost: boolean;           // 고정 게시물 설정
      downloadData: boolean;      // 데이터 다운로드
    };
    // 유산 담당자가 할 수 없는 것
    restrictions: [
      'LOGIN_AS_USER',            // 계정 로그인
      'READ_MESSAGES',            // 메시지 읽기
      'DELETE_POSTS',             // 기존 게시물 삭제
      'FRIEND_UNFRIEND'           // 친구 삭제
    ];
  };

  // 추모 계정 설정
  memorialSettings: {
    displayBanner: boolean;       // "추모" 배너 표시
    allowTributes: boolean;       // 추모글 허용
    visibilityLevel: 'PUBLIC' | 'FRIENDS' | 'FRIENDS_OF_FRIENDS';
    pinnedPost?: string;          // 상단 고정 게시물
    disableLogin: boolean;        // 로그인 영구 비활성화
  };

  // 계정 삭제 옵션 (대안)
  deletionOption?: {
    deleteInstead: boolean;       // 추모 대신 삭제
    deletionDate?: Date;          // 예약 삭제일
    dataDownload: boolean;        // 삭제 전 데이터 다운로드
  };
}

6. 애플 (Apple)

6.1 디지털 유산 프로그램 (Digital Legacy)

애플은 2021년 iOS 15.2부터 "디지털 유산 담당자" 기능을 도입했습니다. 최대 5명까지 지정 가능하며, 각 담당자는 고유한 액세스 키를 받습니다.

🍎 애플 디지털 유산 설정 방법

  1. 설정 > Apple ID > 암호 및 보안
  2. 디지털 유산 담당자 선택
  3. 담당자 추가 (연락처에서 선택 또는 수동 입력)
  4. 담당자에게 액세스 키 전송 (메시지 또는 인쇄)
  5. 담당자가 Apple ID로 키 수락

액세스 범위:

  • ✅ iCloud 사진, 동영상, 메모, 파일
  • ✅ 메시지 백업 (암호화 해제 가능)
  • ✅ Safari 북마크, 리딩 리스트
  • ❌ 키체인 (비밀번호, 신용카드 정보)
  • ❌ 라이선스가 있는 미디어 (음악, 영화, 책)
  • ❌ Apple Pay 정보

7.2 암호화폐 거래소

암호화폐 거래소는 중앙화 거래소(CEX)와 탈중앙화 거래소(DEX)로 나뉘며, 처리 방식이 크게 다릅니다.

/**
 * 암호화폐 상속 처리
 * @type Centralized Exchange (중앙화 거래소)
 */

interface CryptoInheritance {
  // 업비트, 빗썸 등 중앙화 거래소
  centralizedExchange: {
    verification: {
      courtOrder: true,           // 법원 결정문 필수
      deathCertificate: true,
      kycDocuments: true,         // 본인 인증
      walletProof: false          // 지갑 소유권 증명 불필요
    };

    process: {
      step1: '거래소 고객센터 문의',
      step2: '서류 제출 (온라인 또는 방문)',
      step3: '본인 인증 (집행자)',
      step4: '출금 계좌 등록',
      step5: '자산 이체 (14-30일)',
      step6: '출금 수수료 0.05%'
    };

    assets: {
      canTransfer: ['KRW', 'BTC', 'ETH', 'USDT', '모든 상장 코인'],
      limitations: {
        dailyLimit: 50000000,     // 일일 출금 한도 5천만원
        requiresApproval: true    // 거래소 승인 필요
      }
    }
  };

  // 개인 지갑 (비수탁형)
  nonCustodialWallet: {
    requirement: {
      privateKey: true,           // 개인키 필수
      seedPhrase: true,           // 시드 문구 필수
      password: false             // 비밀번호는 부차적
    };

    challenge: {
      lostPrivateKey: '복구 불가능',
      legalAccess: '법원도 강제 불가',
      technicalBarrier: '암호학적 보안'
    };

    solution: {
      preventive: '생전에 안전한 방법으로 개인키 공유',
      methods: [
        '하드웨어 월렛 + 금고 보관',
        '시드 문구 분할 보관 (Shamir Secret Sharing)',
        '다중서명 지갑 (Multisig)',
        '스마트 계약 기반 상속 (예: Safe{Wallet} Inheritance)'
      ]
    }
  };
}

/**

interface UnifiedDashboard {

  connectedPlatforms: Array<{
    platform: string;
    status: 'CONNECTED' | 'PENDING' | 'FAILED';

  assetSummary: {
    total Platforms: number;
    totalAccounts: number;

      currency: 'KRW';
      financial: number;

    message: string;
    actionRequired: boolean;
    deadline?: Date;
  }>;

9. 실무 팁과 주의사항

9.1 플랫폼별 처리 우선순위

✅ 권장 처리 순서

  1. 1단계 (긴급): 금융 자산
    • 은행 계좌 (동결 방지)
    • 암호화폐 거래소 (시세 변동 리스크)
    • 네이버페이, 카카오페이 (소액이라도 우선 처리)
  2. 2단계 (중요): 수익 발생 계정
    • 유튜브 (수익화 채널)
    • 네이버 블로그 (애드포스트)
    • 인스타그램 (광고 수익)
  3. 3단계 (일반): 데이터 보존
    • Google Drive, iCloud (자동 삭제 전)
    • 네이버 클라우드
    • 카카오톡 대화 백업
  4. 4단계 (선택): 소셜 미디어
    • 페이스북, 인스타그램 (추모 전환)
    • 트위터(X), 링크드인

9.2 자주 발생하는 문제와 해결

문제 원인 해결방법
2단계 인증 문제 고인의 휴대폰 접근 불가 플랫폼에 법원 결정문 제출하여 우회
이메일 주소 불명 계정 정보 미파악 통신사에 이메일 사용 내역 조회 요청
플랫폼 응답 지연 서류 미비, 담당자 부재 WIA 표준 통합 신청서 활용
개인키 분실 암호화폐 지갑 접근 불가 복구 불가 (사전 대비 필요)
