1. 디지털 유산 법제화의 시작

2024년 12월, 법무부는 "디지털 유산의 상속 및 관리에 관한 민법 일부개정법률안"을 발표했습니다. 이는 1958년 민법 제정 이래 가장 중요한 상속법 개정으로 평가받고 있습니다.

📌 학습 목표

  • 민법 개정안의 핵심 조항 이해
  • 개인정보보호법과의 조화 방안
  • 금융실명법 특례 적용 범위
  • 전자서명법의 디지털 유산 인증
  • 법률 간 충돌 해결 메커니즘

2. 민법 개정안의 주요 내용

2.1 신설 조항: 제1008조의2 (디지털 유산의 상속)

📜 민법 제1008조의2 (신설)

제1항: 디지털 유산은 상속재산에 포함된다. 디지털 유산이란 전자적 방법으로 생성·저장·관리되는 다음 각 호의 자산을 말한다.

  1. 전자금융거래법에 따른 전자금융자산
  2. 정보통신망법에 따른 정보통신서비스 계정 및 그에 저장된 정보
  3. 전자상거래법에 따른 디지털 콘텐츠 및 가상자산
  4. 저작권법에 따른 디지털 저작물
  5. 그 밖에 재산적 가치가 있는 전자적 형태의 자산

제2항: 피상속인이 생전에 디지털 유산의 상속을 명시적으로 금지한 경우에는 제1항을 적용하지 아니한다. 다만, 재산적 가치가 있는 경우에는 그러하지 아니하다.

제3항: 디지털 서비스 제공자는 법원이 지정한 디지털 유산 집행자의 정당한 요청에 협조하여야 한다.

2.2 개정 조항: 제1091조 (유언집행자의 지정)

구분 현행 (2025년 1월 이전) 개정안 (2026년 시행 예정)
대상 재산 동산, 부동산, 금융자산 + 디지털 유산 명시
집행자 자격 성년, 상속인 + 디지털 유산 전문인력
권한 범위 재산 목록 작성, 분할, 이전 + 계정 접근, 데이터 이전, 삭제
보고 의무 법원에 분기별 보고 + 플랫폼별 처리 현황 상세

2.3 신설 조항: 제1103조의2 (디지털 계정의 처리)

/**
 * 민법 제1103조의2 시행을 위한 WIA 표준 매핑
 * @legal 민법 제1103조의2
 * @standard WIA-DE-002
 */

enum AccountDisposition {
  DELETE = "DELETE",              // 완전 삭제
  MEMORIAL = "MEMORIAL",          // 추모 계정 전환
  TRANSFER = "TRANSFER",          // 상속인에게 이전
  ARCHIVE = "ARCHIVE",            // 아카이브 (읽기 전용)
  MONETIZE = "MONETIZE"           // 수익화 계속
}

interface AccountProcessingRequest {
  accountId: string;
  platform: string;
  disposition: AccountDisposition;

  // 법적 근거
  legalBasis: {
    willProvision?: string;        // 유언장 조항
    courtOrder?: string;           // 법원 결정문
    heirConsensus?: {              // 상속인 합의
      signatories: string[];
      agreementDate: Date;
    }
  };

  // 처리 기한
  deadline: Date;                  // 민법 제1103조의2 제4항: 6개월 이내

  // 특별 조건
  conditions?: {
    privacyLevel: 'PUBLIC' | 'FAMILY_ONLY' | 'PRIVATE';
    downloadData: boolean;
    notifyContacts: boolean;
  };
}

3. 개인정보보호법과의 조화

3.1 사망자 정보의 법적 지위

개인정보보호법은 2023년 개정을 통해 사망자의 디지털 정보에 대한 특별 규정을 신설했습니다.

조항 내용 디지털 유산에의 적용
제2조의2 사망자 정보의 정의 사망 후 24개월까지 개인정보로 보호
제15조 제4항 정보주체 동의 간주 유언장 명시 시 사전 동의로 인정
제39조의4 유족의 열람·제공 청구권 직계 가족은 정보 접근 가능
제39조의5 (신설) 디지털 유산 집행자의 특례 법원 지정 시 전체 접근 권한

3.2 민감정보 처리의 특례

⚠️ 주의: 민감정보 처리 제한

다음 정보는 디지털 유산 집행자도 접근이 제한됩니다:

  • 건강정보: 의료기록, 처방 이력, 건강검진 결과
  • 성생활 정보: 데이팅 앱, 성인 콘텐츠 구독
  • 정치적 신념: 정당 가입, 투표 이력, 정치 기부
  • 노동조합 정보: 가입 여부, 활동 내역

예외: 법원의 특별 결정이 있거나, 고인이 유언장에 명시적으로 공개를 허용한 경우

4. 금융실명법 특례 조항

4.1 금융거래정보의 제공

금융실명거래 및 비밀보장에 관한 법률(금융실명법) 제4조의2가 2025년 신설되어 디지털 금융자산의 상속 절차를 명확히 했습니다.

/**
 * 금융감독원 디지털 자산 조회 API
 * @legal 금융실명법 제4조의2
 * @agency 금융감독원
 */

interface FinancialAssetInquiry {
  // 조회 요청자 (디지털 유산 집행자)
  executor: {
    id: string;
    name: string;
    courtOrderNumber: string;      // 필수: 법원 결정문 번호
  };

  // 고인 정보
  deceased: {
    name: string;
    ssn: string;                   // 주민등록번호 (암호화)
    dateOfDeath: Date;
  };

  // 조회 범위
  scope: {
    institutions: string[];        // 은행, 증권사, 암호화폐 거래소
    assetTypes: FinancialAssetType[];
    periodFrom: Date;              // 조회 시작일
    periodTo: Date;                // 조회 종료일
  };
}

type FinancialAssetType =
  | 'BANK_ACCOUNT'                 // 은행 계좌
  | 'SECURITIES'                   // 증권 (주식, 채권 등)
  | 'CRYPTOCURRENCY'               // 암호화폐
  | 'INSURANCE'                    // 보험
  | 'PENSION'                      // 연금
  | 'FUND';                        // 펀드

// 금융감독원 통합 조회 결과
interface FinancialAssetReport {
  reportId: string;
  generatedAt: Date;

  assets: Array<{
    institution: string;
    accountNumber: string;         // 마스킹된 계좌번호
    type: FinancialAssetType;
    balance: number;               // KRW 환산 가치
    lastActivity: Date;
    status: 'ACTIVE' | 'DORMANT' | 'CLOSED';
  }>;

  totalValue: number;              // 총 자산 가치
  asOfDate: Date;
}

4.2 금융기관별 처리 프로세스

기관 유형 필요 서류 처리 기간 수수료
은행 법원 결정문 + 가족관계증명서 3-7일 무료
증권사 법원 결정문 + 상속인 전원 동의서 7-14일 거래 금액의 0.1%
암호화폐 거래소 법원 결정문 + 본인 인증 14-30일 출금액의 0.05%

5. 전자서명법의 적용

5.1 디지털 유산 집행자의 전자 인증

전자서명법 제2조의2에 따라, 디지털 유산 집행자는 반드시 공인전자서명 또는 공동인증서를 사용해야 합니다.

🔐 전자서명 요구사항

  • Level 1 (기본): 이메일, 소셜 미디어 계정
    • 간편 인증 (카카오, 네이버 인증)
    • SMS/이메일 2단계 인증
  • Level 2 (표준): 클라우드 저장소, 일반 자산
    • 공동인증서 (구 공인인증서)
    • 민간 인증서 (PASS, 페이코)
  • Level 3 (강화): 금융 자산, 암호화폐
    • 금융인증서 (금융결제원 발급)
    • 범용 공동인증서
  • Level 4 (최고): 고액 자산, 기업 자산
    • 생체인증 연동 공동인증서
    • 법원 지정 특수 인증서

6. 법률 간 충돌 해결

6.1 주요 법률 간 우선순위

순위 법률 적용 우선 사항 근거
1순위 헌법 재산권 보장, 상속권 제23조, 제36조
2순위 민법 상속 및 유언 집행 제5편 상속
3순위 금융실명법 금융자산 조회·이전 제4조의2
4순위 개인정보보호법 개인정보 처리·삭제 제39조의4, 39조의5
5순위 정보통신망법 계정 및 서비스 관리 제29조의2

6.2 충돌 사례와 해결

⚖️ 사례 1: 민법 vs 개인정보보호법

상황: 상속인이 고인의 이메일 전체 내역을 요청했으나, 플랫폼이 개인정보보호법을 이유로 거부

쟁점: 상속권(민법)과 사생활 보호(개인정보보호법)의 충돌

해결:

  1. 법원이 디지털 유산 집행자 지정
  2. 집행자가 필요한 정보만 선별 조회 (최소 정보 원칙)
  3. 민감정보는 법원의 추가 승인 필요
  4. 조회 내역 전체를 법원에 보고

법적 근거: 민법 제1008조의2 제3항 + 개인정보보호법 제39조의5

⚖️ 사례 2: 금융실명법 vs 상속세법

상황: 은행이 상속세 납부 확인서 없이는 계좌 해지 불가 주장

쟁점: 금융실명법의 비밀보장과 상속세 징수의 충돌

해결:

  1. 디지털 유산 집행자가 먼저 전체 자산 목록 작성
  2. 국세청에 상속세 신고 (6개월 이내)
  3. 상속세 납부 또는 분납 계획 수립
  4. 납부 확인 후 금융자산 이전

특례: 상속세 납부 전이라도 생활비 등 긴급 자금은 법원 승인으로 인출 가능

8. 국제 법률 비교

8.1 RUFADAA (미국)

Revised Uniform Fiduciary Access to Digital Assets Act는 2015년 미국 통일주법위원회가 제정한 모범법으로, 현재 48개 주에서 채택했습니다.

// RUFADAA 핵심 원칙의 WIA 표준 반영
interface RUFADAACompliance {
  // 원칙 1: 사용자 의사 우선
  userIntent: {
    priority: 'HIGHEST',
    source: 'WILL' | 'ONLINE_TOOL' | 'TOS',  // 유언 > 온라인 도구 > 약관
    enforceable: boolean
  };

  // 원칙 2: 수탁자 접근권
  fiduciaryAccess: {
    catalogAccess: boolean;        // 계정 목록 조회
    contentAccess: boolean;        // 콘텐츠 접근
    controlAccess: boolean;        // 계정 제어
  };

  // 원칙 3: 플랫폼 면책
  platformImmunity: {
    goodFaith: boolean;            // 선의의 협력 시 면책
    complianceCost: 'ZERO_FEE'     // 무료 제공 의무
  };

  // 원칙 4: 프라이버시 보호
  privacyProtection: {
    communicationPrivilege: boolean;  // 통신 내용 보호
    thirdPartyConsent: boolean;       // 제3자 동의 필요
  };
}

8.2 한국 vs 미국 vs 독일 vs 일본

구분 한국 (2026) 미국 (RUFADAA) 독일 (BGB) 일본
법적 근거 민법 개정 모범법 (주법) 민법 제1922조 가이드라인
상속성 재산적 가치 있으면 상속 계정별 차별 전면 상속 플랫폼 재량
집행자 지정 법원 또는 유언 유언 우선 자동 승계 없음
플랫폼 의무 협조 의무 (법률) 면책 조건부 전면 협조 자율 규제
처리 기간 6개월 규정 없음 3개월 플랫폼별 상이

  • 지방자치 조례: 서울시 최초 제정(2024.7), 상담센터·법률지원·교육 사업 시행
  • 국제 비교: 한국은 법률 강제, 미국은 모범법, 독일은 전면 상속, 일본은 자율규제
