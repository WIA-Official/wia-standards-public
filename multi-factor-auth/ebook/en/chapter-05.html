5.1 MFA Enrollment Flow Design

The enrollment experience sets the tone for users' perception of your MFA system. A well-designed enrollment flow balances security, usability, and accessibility while minimizing drop-off rates.

Progressive Enrollment Strategy

1
Account Creation

Basic password-only authentication to reduce initial friction

2
Soft MFA Prompt

Explain benefits, allow users to skip with "Remind me later"

3
Primary Factor Enrollment

Guide user through TOTP app or security key setup with QR code

4
Verification Test

Immediate authentication to confirm setup works correctly

5
Backup Methods

Offer backup codes and optional secondary authenticator

6
Recovery Contact

Collect verified email/phone for account recovery

When to Require Step-Up Authentication

  • Account Settings Changes: Email, password, phone number, security questions
  • Security Modifications: Adding/removing MFA factors, changing recovery options
  • Financial Transactions: Wire transfers, ACH payments, large purchases
  • Data Access: Exporting personal data, accessing sensitive records
  • Privilege Escalation: Granting admin access, assuming different user role
  • Destructive Actions: Account deletion, bulk data deletion

5.3 Account Recovery Mechanisms

Robust account recovery is critical to MFA adoption. Users will lose devices, forget passwords, and need access when traditional methods fail. Recovery mechanisms must balance accessibility and security.

Backup Code Implementation

Backup Code Generation and Usage 
// Generate backup codes during enrollment
function generateBackupCodes(userId, count = 10) {
  const codes = [];
  const crypto = require('crypto');

  for (let i = 0; i < count; i++) {
    // Generate cryptographically secure random code
    const code = crypto.randomBytes(6).toString('hex').toUpperCase();

    // Format: XXXX-XXXX-XXXX
    const formatted = code.match(/.{1,4}/g).join('-');

    // Hash for storage (never store plain text)
    const hash = crypto.createHash('sha256')
      .update(formatted + process.env.BACKUP_CODE_PEPPER)
      .digest('hex');

    codes.push({
      code: formatted,
      hash: hash,
      created_at: new Date(),
      used_at: null,
      ip_address: null
    });
  }

  // Store hashed codes in database
  database.backupCodes.insertMany({
    user_id: userId,
    codes: codes.map(c => ({
      hash: c.hash,
      created_at: c.created_at,
      used: false
    }))
  });

  // Return plain codes to user (display once, never shown again)
  return codes.map(c => c.code);
}

// Verify backup code during authentication
async function verifyBackupCode(userId, submittedCode) {
  const pepper = process.env.BACKUP_CODE_PEPPER;
  const hash = crypto.createHash('sha256')
    .update(submittedCode + pepper)
    .digest('hex');

  const codeRecord = await database.backupCodes.findOne({
    user_id: userId,
    'codes.hash': hash,
    'codes.used': false
  });

  if (!codeRecord) {
    return {
      success: false,
      error: 'invalid_or_used',
      message: 'Backup code is invalid or already used'
    };
  }

  // Mark code as used (one-time use only)
  await database.backupCodes.updateOne(
    { user_id: userId, 'codes.hash': hash },
    {
      $set: {
        'codes.$.used': true,
        'codes.$.used_at': new Date(),
        'codes.$.ip_address': getRequestIP()
      }
    }
  );

  // Check remaining codes
  const remaining = await database.backupCodes.countDocuments({
    user_id: userId,
    'codes.used': false
  });

  return {
    success: true,
    remaining_codes: remaining,
    warning: remaining < 3 ? 'Low backup code count. Generate new codes.' : null
  };
}

Recovery Flow Best Practices

1
Recovery Initiation

User clicks "Lost access to authenticator" during login

2
Identity Verification

Verify email/phone, ask security questions, or check device fingerprint

3
Recovery Method Selection

Offer backup codes, backup device, or email recovery link

4
Temporary Access Grant

Provide limited session to set up new MFA factors

5
MFA Re-enrollment

Guide user through enrolling new primary and backup factors

6
Audit and Notification

Log recovery event, notify all recovery contacts, revoke old factors

⚠️ Recovery Security Considerations
  • Rate Limiting: Limit recovery attempts to 3 per 24 hours to prevent brute force
  • Cooling Period: Delay MFA removal by 24-72 hours with email notification
  • Session Restrictions: Recovery sessions should have limited permissions (can't transfer money, change email, etc.)
  • Full Re-enrollment: Require complete MFA setup, not just restoration of old factors
  • Audit Trail: Log all recovery events with IP, device, and timestamp
  • Multi-channel Notification: Alert user via email, SMS, and push notification

5.4 Adaptive Authentication Policies

Adaptive (or risk-based) authentication dynamically adjusts security requirements based on contextual risk factors, providing strong security without unnecessary user friction.

Risk Scoring Factors

Risk Assessment Algorithm 
class RiskScorer {
  async calculateRiskScore(authenticationContext) {
    let riskScore = 0; // 0 (lowest risk) to 100 (highest risk)
    const factors = [];

    // 1. Device Recognition (Weight: 20)
    const deviceRisk = await this.assessDeviceRisk(authenticationContext.device);
    riskScore += deviceRisk.score * 0.20;
    factors.push({ name: 'device', score: deviceRisk.score, details: deviceRisk.details });

    // 2. Location Analysis (Weight: 15)
    const locationRisk = await this.assessLocationRisk(authenticationContext.location);
    riskScore += locationRisk.score * 0.15;
    factors.push({ name: 'location', score: locationRisk.score, details: locationRisk.details });

    // 3. Network Analysis (Weight: 10)
    const networkRisk = await this.assessNetworkRisk(authenticationContext.network);
    riskScore += networkRisk.score * 0.10;
    factors.push({ name: 'network', score: networkRisk.score, details: networkRisk.details });

    // 4. Time Pattern Analysis (Weight: 10)
    const timeRisk = await this.assessTimePattern(authenticationContext.timestamp);
    riskScore += timeRisk.score * 0.10;
    factors.push({ name: 'time_pattern', score: timeRisk.score, details: timeRisk.details });

    // 5. Velocity Checks (Weight: 15)
    const velocityRisk = await this.assessVelocity(authenticationContext.userId);
    riskScore += velocityRisk.score * 0.15;
    factors.push({ name: 'velocity', score: velocityRisk.score, details: velocityRisk.details });

    // 6. Historical Behavior (Weight: 20)
    const behaviorRisk = await this.assessBehavior(authenticationContext.userId);
    riskScore += behaviorRisk.score * 0.20;
    factors.push({ name: 'behavior', score: behaviorRisk.score, details: behaviorRisk.details });

    // 7. Threat Intelligence (Weight: 10)
    const threatRisk = await this.checkThreatIntelligence(authenticationContext.ipAddress);
    riskScore += threatRisk.score * 0.10;
    factors.push({ name: 'threat_intel', score: threatRisk.score, details: threatRisk.details });

    return {
      totalScore: Math.round(riskScore),
      level: this.getRiskLevel(riskScore),
      factors: factors,
      recommendedAction: this.getRecommendedAction(riskScore)
    };
  }

  async assessDeviceRisk(device) {
    let score = 0;
    const details = [];

    // Unknown device
    if (!device.recognized) {
      score += 40;
      details.push('Device not recognized');
    }

    // Device fingerprint changed
    if (device.fingerprintChanged) {
      score += 25;
      details.push('Device fingerprint mismatch');
    }

    // Suspicious browser/OS combination
    if (this.isSuspiciousCombination(device.userAgent)) {
      score += 15;
      details.push('Unusual browser/OS combination');
    }

    // Automation detected (headless browser, selenium, etc.)
    if (device.automationDetected) {
      score += 50;
      details.push('Automation tools detected');
    }

    return { score, details };
  }

  async assessLocationRisk(location) {
    let score = 0;
    const details = [];

    const userHistory = await this.getUserLocationHistory(location.userId);

    // New country
    if (!userHistory.countries.includes(location.country)) {
      score += 30;
      details.push(`First login from ${location.country}`);
    }

    // Impossible travel (e.g., Tokyo to New York in 2 hours)
    if (this.isImpossibleTravel(userHistory.lastLocation, location)) {
      score += 60;
      details.push('Impossible travel detected');
    }

    // High-risk country
    if (this.isHighRiskCountry(location.country)) {
      score += 20;
      details.push('Login from high-risk country');
    }

    // VPN/Tor detected
    if (location.vpnDetected || location.torDetected) {
      score += 25;
      details.push('VPN or Tor network detected');
    }

    return { score, details };
  }

  getRiskLevel(score) {
    if (score < 20) return 'low';
    if (score < 40) return 'medium';
    if (score < 70) return 'high';
    return 'critical';
  }

  getRecommendedAction(score) {
    if (score < 20) {
      return { action: 'allow', mfaRequired: false };
    } else if (score < 40) {
      return { action: 'allow', mfaRequired: true, factorTypes: ['totp', 'push'] };
    } else if (score < 70) {
      return { action: 'challenge', mfaRequired: true, factorTypes: ['webauthn', 'totp'], additionalVerification: true };
    } else {
      return { action: 'block', mfaRequired: true, manualReview: true };
    }
  }
}
