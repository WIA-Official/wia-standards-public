Chapter 4: Battery Optimization & Power Management

In this chapter: Master power consumption analysis, implement adaptive tracking intervals, optimize cellular connectivity, design ultra-low-power modes, and achieve 30-day battery life targets.

4.1 Power Consumption Analysis

Pet tracker battery life is the #1 user complaint and feature request. Understanding power consumption patterns is essential for designing long-lasting devices.

Component Power Draw

Typical power consumption for pet tracker components:

  • GPS Active: 30-50mA (highest consumer)
  • Cellular Transmit: 200-500mA peak, 20-40mA average
  • Cellular Idle: 2-5mA (connected to network)
  • Accelerometer: 0.05-0.2mA (always-on motion detection)
  • CPU Active: 10-30mA
  • CPU Sleep: 0.1-1mA
  • LED Indicators: 2-20mA per LED

Battery Capacity and Runtime

Common battery sizes for pet trackers:

  • Small Collar (cats, small dogs): 200-400mAh
  • Medium Collar (medium dogs): 500-800mAh
  • Large Collar (large dogs): 1000-1500mAh

Example calculation: 500mAh battery with average 5mA consumption = 100 hours (4.2 days) runtime. Target: Reduce average to 0.7mA for 30-day battery life.

4.2 Adaptive Tracking Strategies

The key to long battery life: GPS and cellular radios spend most time OFF, only activating when needed.

Motion-Based Wake-Up

// Accelerometer monitors movement at 0.1mA
// GPS/Cellular stay OFF until movement detected

const STATIONARY_THRESHOLD = 0.05; // g-force
const MOTION_TIMEOUT = 300; // seconds

function onAccelerometerReading(acceleration) {
  const magnitude = Math.sqrt(
    acceleration.x² + acceleration.y² + acceleration.z²
  );
  
  if (magnitude > STATIONARY_THRESHOLD) {
    wakeGPS();
    lastMotionTime = Date.now();
  } else if (Date.now() - lastMotionTime > MOTION_TIMEOUT * 1000) {
    enterSleepMode(); // No movement for 5 minutes
  }
}

Context-Aware Tracking Intervals

  • At Home (geofence): 5-10 minute GPS updates
  • Walking (moderate motion): 30-60 second updates
  • Running (high motion): 10-20 second updates
  • Vehicle (very high speed): Continuous GPS, may indicate theft
  • Stationary: 10-30 minute updates or motion-triggered only
  • Low Battery (<20%): 10-30 minute updates regardless of activity

4.3 Cellular Connectivity Optimization

Cellular transmission is extremely power-hungry. Minimize connections while maintaining responsiveness.

Data Batching

Instead of transmitting every GPS fix individually (expensive), batch multiple fixes and send together:

  • Collect 5-10 location points locally
  • Connect to cellular network once
  • Upload all points in single transmission
  • Disconnect immediately
  • Result: 5-10x reduction in cellular power consumption

Intelligent Connection Management

class CellularOptimizer {
  constructor() {
    this.buffer = [];
    this.maxBufferSize = 10;
    this.forceUploadInterval = 300000; // 5 minutes max delay
  }

  addLocation(location) {
    this.buffer.push(location);

    // Immediate upload for critical events
    if (location.isGeofenceAlert || location.isLowBattery) {
      return this.uploadNow();
    }

    // Batch upload when buffer full or timeout
    if (this.buffer.length >= this.maxBufferSize || 
        Date.now() - this.lastUpload > this.forceUploadInterval) {
      return this.uploadBatch();
    }
  }

  async uploadBatch() {
    if (this.buffer.length === 0) return;

    await connectCellular();
    await uploadData(this.buffer);
    await disconnectCellular();

    this.buffer = [];
    this.lastUpload = Date.now();
  }
}

4.4 Ultra-Low-Power Sleep Modes

When pet is stationary at home overnight, device should enter deep sleep consuming <0.5mA.

Sleep Mode Configuration

  • GPS: Completely OFF
  • Cellular: Detached from network (saves 2-5mA vs. idle)
  • CPU: Deep sleep, wake on accelerometer interrupt
  • Accelerometer: Ultra-low-power mode, sampling at 1Hz
  • RTC: Keep-alive for scheduled wake-ups

Smart Wake Conditions

  • Motion detected (accelerometer interrupt)
  • Scheduled GPS check (e.g., every 4 hours)
  • Remote wake command from server
  • Low battery threshold for final location report

4.5 Battery State Monitoring

Accurate battery monitoring allows intelligent power management decisions.

Battery Level Estimation

function estimateBatteryLife(currentLevel, averagePower) {
  const batteryCapacity = 500; // mAh
  const remainingCapacity = batteryCapacity * (currentLevel / 100);
  const hoursRemaining = remainingCapacity / averagePower;
  const daysRemaining = hoursRemaining / 24;
  
  return {
    percentage: currentLevel,
    hoursRemaining: Math.round(hoursRemaining),
    daysRemaining: Math.round(daysRemaining * 10) / 10,
    recommendation: daysRemaining < 2 ? 'charge_soon' : 'ok'
  };
}

Progressive Power Saving

  • 100-50%: Normal operation, all features enabled
  • 50-30%: Reduce GPS accuracy, increase update intervals by 50%
  • 30-20%: Disable non-essential features, double update intervals
  • 20-10%: Critical mode - geofence monitoring only, 5-minute updates
  • 10-5%: Emergency mode - position updates every 15 minutes
  • <5%: Final position broadcast, then shutdown to preserve battery

💡 Key Takeaway: Achieving 30-day battery life requires aggressive power management: motion-based GPS activation, data batching, deep sleep modes, and context-aware tracking. Every component must be optimized. The difference between 3-day and 30-day battery life is not hardware - it's intelligent software power management following WIA-PET-008 best practices.
