Chapter 3: Geofencing & Alert Systems

In this chapter: Learn geofence algorithms, detect zone entry/exit, implement smart alerts, prevent notification fatigue, and build community-based lost pet recovery systems.

3.1 Geofencing Fundamentals

Geofencing creates virtual boundaries around geographic areas. When a pet enters or exits these zones, the system triggers alerts. This is the most critical safety feature for pet tracking, preventing escapes and enabling rapid response when pets wander beyond safe areas.

Geofence Types

  • Circular Geofences: Defined by center point and radius. Simplest to implement and most computationally efficient. Example: 100-meter radius around home.
  • Polygon Geofences: Defined by series of lat/lng points forming irregular shapes. Useful for property boundaries, parks, or neighborhoods.
  • Corridor Geofences: Path-based zones with width tolerance. Useful for walking routes or allowing pets along specific streets.
  • Dynamic Geofences: Follow moving entities like owner's phone. Maintains proximity alerts when traveling with pets.

Circular Geofence Detection Algorithm

function isInsideCircularGeofence(petLocation, geofence) {
  const R = 6371000; // Earth radius in meters
  
  const φ1 = petLocation.lat * Math.PI / 180;
  const φ2 = geofence.center.lat * Math.PI / 180;
  const Δφ = (geofence.center.lat - petLocation.lat) * Math.PI / 180;
  const Δλ = (geofence.center.lng - petLocation.lng) * Math.PI / 180;

  const a = Math.sin(Δφ/2) * Math.sin(Δφ/2) +
            Math.cos(φ1) * Math.cos(φ2) *
            Math.sin(Δλ/2) * Math.sin(Δλ/2);
  const c = 2 * Math.atan2(Math.sqrt(a), Math.sqrt(1-a));
  const distance = R * c;

  return distance <= geofence.radius;
}

// Example usage
const petLocation = { lat: 37.7749, lng: -122.4194 };
const homeGeofence = {
  center: { lat: 37.7750, lng: -122.4195 },
  radius: 100  // meters
};

const isHome = isInsideCircularGeofence(petLocation, homeGeofence);
console.log("Pet is home:", isHome);

Polygon Geofence Detection (Point-in-Polygon)

function isInsidePolygonGeofence(point, polygon) {
  let inside = false;
  for (let i = 0, j = polygon.length - 1; i < polygon.length; j = i++) {
    const xi = polygon[i].lng, yi = polygon[i].lat;
    const xj = polygon[j].lng, yj = polygon[j].lat;
    
    const intersect = ((yi > point.lat) !== (yj > point.lat))
        && (point.lng < (xj - xi) * (point.lat - yi) / (yj - yi) + xi);
    
    if (intersect) inside = !inside;
  }
  return inside;
}

// Example: Park boundary
const parkBoundary = [
  { lat: 37.7750, lng: -122.4200 },
  { lat: 37.7755, lng: -122.4195 },
  { lat: 37.7753, lng: -122.4188 },
  { lat: 37.7748, lng: -122.4193 }
];

const isInPark = isInsidePolygonGeofence(petLocation, parkBoundary);

3.2 Smart Alert Management

Raw geofence alerts can be noisy and cause alert fatigue. Smart alert systems use hysteresis, dwell time, and confidence scoring to ensure notifications are meaningful.

Hysteresis Buffering

Prevents rapid toggling near boundaries. If geofence radius is 100m, create an inner zone (90m) and outer zone (110m). Only trigger exit when pet moves beyond 110m, re-entry when returning inside 90m. This 20m buffer prevents alerts from GPS jitter near boundaries.

Dwell Time Requirements

Require pet to be outside geofence for minimum duration before alerting. Example: Only alert if pet has been outside safe zone for 60 seconds continuously. Prevents false alarms from brief GPS errors or pets momentarily crossing boundaries. 

class SmartGeofenceMonitor {
  constructor(geofence, options = {}) {
    this.geofence = geofence;
    this.dwellTimeMs = options.dwellTime || 60000; // 60 seconds
    this.hysteresisMeters = options.hysteresis || 10;
    this.state = 'inside';
    this.transitionStart = null;
  }

  checkLocation(location, timestamp) {
    const distance = calculateDistance(location, this.geofence.center);
    const innerRadius = this.geofence.radius - this.hysteresisMeters;
    const outerRadius = this.geofence.radius + this.hysteresisMeters;

    let newState;
    if (distance < innerRadius) {
      newState = 'inside';
    } else if (distance > outerRadius) {
      newState = 'outside';
    } else {
      newState = 'boundary'; // In hysteresis zone
    }

    // Check for state transition
    if (newState !== this.state && newState !== 'boundary') {
      if (this.transitionStart === null) {
        this.transitionStart = timestamp;
      } else if (timestamp - this.transitionStart > this.dwellTimeMs) {
        // Dwell time exceeded - confirm transition
        const alert = {
          type: newState === 'inside' ? 'geofence_enter' : 'geofence_exit',
          geofenceId: this.geofence.id,
          location: location,
          timestamp: timestamp
        };
        this.state = newState;
        this.transitionStart = null;
        return alert;
      }
    } else if (newState === this.state) {
      // Back to stable state - reset transition
      this.transitionStart = null;
    }

    return null; // No alert
  }
}

3.3 Multi-Channel Alert Delivery

Critical alerts should use multiple notification channels to ensure owners are reached quickly, especially during escape scenarios.

Alert Priority Levels

  • Critical: Geofence exit, device offline, battery critical. Use push + SMS + email + phone call.
  • High: Low battery warning, unusual activity patterns. Use push + SMS.
  • Medium: Geofence entry, daily activity summary. Use push notification.
  • Low: Device status updates, achievements. Use in-app notification only.

Intelligent Notification Timing

Respect quiet hours and user preferences while ensuring critical alerts always get through. Example: Non-critical alerts during 10pm-7am are queued until morning, but geofence exits always alert immediately regardless of time.

3.4 Lost Pet Recovery Features

When a pet goes missing, every minute counts. Advanced alert systems include community-based features to maximize recovery chances.

Lost Mode Activation

One-tap activation that:

  • Increases GPS update frequency to every 10 seconds
  • Enables maximum transmission power
  • Activates audio beacon on collar (if equipped)
  • Broadcasts lost pet alert to nearby WIA-PET-008 users
  • Shares location with local shelters and veterinary clinics
  • Creates social media posts with last known location and description

Community Alert Network

Pet owners within radius of lost pet receive notifications. If they spot the pet, they can report sightings that are displayed on owner's map. Privacy-preserving: only approximate location shared with community, precise location only visible to owner and authorized helpers.

3.5 Activity-Based Alerts

Beyond geofencing, monitor activity patterns and alert on anomalies that may indicate health issues or unusual behavior.

Anomaly Detection Scenarios

  • Sudden Activity Change: Pet normally walks 5km/day but hasn't moved in 24 hours → Potential health issue
  • Excessive Activity: Restless pacing or running could indicate anxiety, pain, or distress
  • Unusual Hours: Active movement at 3am when normally sleeping → Possible disturbance or illness
  • Rapid Movement: Sudden high-speed movement could indicate pet in vehicle (stolen, ran into road)
  • Stationary Away from Home: Pet hasn't moved for 2 hours at unknown location → May be trapped or injured

💡 Key Takeaway

Effective geofencing requires careful algorithm design with hysteresis and dwell time to prevent false alarms. Smart alert systems use multiple channels, priority levels, and intelligent timing. Lost pet recovery features leverage community networks and rapid response protocols. Activity-based alerts can detect health issues early, potentially saving lives beyond just preventing escapes.
