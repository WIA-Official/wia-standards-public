In this chapter: We explore Global Navigation Satellite Systems (GNSS), understand how GPS works for pet tracking, implement multi-constellation support, optimize accuracy, and handle edge cases like indoor positioning and urban canyons.

2.1 Understanding GNSS Technology

Global Navigation Satellite Systems form the backbone of modern pet tracking. While commonly called "GPS," modern trackers actually use multiple satellite constellations to improve accuracy, availability, and reliability. Understanding how these systems work is essential for building effective tracking solutions.

How Satellite Positioning Works

GNSS positioning relies on a principle called trilateration. Each satellite broadcasts its precise location and the current time. A receiver on Earth (like a pet tracker) measures the time it takes for signals to arrive from multiple satellites. Since radio signals travel at the speed of light (299,792,458 meters per second), the receiver can calculate the distance to each satellite. With distances from at least four satellites, the receiver can determine its exact 3D position (latitude, longitude, altitude) and the current time.

Mathematical Foundation: Trilateration

Given distances to satellites with known positions:

// Distance equation from satellite to receiver
d1 = √[(x - x1)² + (y - y1)² + (z - z1)²]
d2 = √[(x - x2)² + (y - y2)² + (z - z2)²]
d3 = √[(x - x3)² + (y - y3)² + (z - z3)²]
d4 = √[(x - x4)² + (y - y4)² + (z - z4)²]

Where:
- (x, y, z) = unknown receiver position
- (xi, yi, zi) = known satellite positions
- di = measured distance (pseudo-range)

Solving these four equations yields position + clock offset

2.2 Multi-Constellation GNSS for Pet Tracking

Modern pet trackers should support multiple GNSS constellations simultaneously. This multi-GNSS approach provides several critical advantages for pet tracking applications.

Benefits of Multi-GNSS

  • Improved Accuracy: More satellites visible means better geometric diversity (GDOP - Geometric Dilution of Precision), resulting in more accurate position fixes. Typical improvement: 30-50% better accuracy in challenging environments.
  • Faster Time-to-First-Fix (TTFF): With 100+ satellites available across all systems vs. 31 for GPS alone, receivers can acquire signals and calculate position much faster. Critical when a pet escapes and every second counts.
  • Better Urban Performance: In urban canyons where tall buildings block sky view, having more satellite options significantly improves availability. Instead of seeing 4-6 GPS satellites, a multi-GNSS receiver might see 15-20 total satellites.
  • Global Coverage Optimization: Different constellations have different orbital characteristics. BeiDou has better coverage in Asia-Pacific, GLONASS excels at high latitudes, while GPS provides consistent global coverage.
  • Redundancy and Reliability: If one constellation experiences technical issues or intentional signal degradation, others provide backup positioning capability.

2.3 Assisted GPS (A-GPS) for Faster Fixes

Traditional GPS requires downloading almanac and ephemeris data from satellites, which can take 30-60 seconds for a cold start. For pet tracking, this delay is unacceptable when a pet escapes. Assisted GPS solves this by providing satellite data through cellular or Wi-Fi connections.

A-GPS Components

  1. Reference Network: Ground stations with known positions continuously track all visible satellites
  2. Assistance Server: Collects data from reference network and predicts satellite positions
  3. Data Delivery: Sends assistance data to devices via cellular data or Wi-Fi
  4. Fast Calculation: Device uses assistance data to quickly calculate position without waiting for satellite downloads

A-GPS Performance Benefits

Scenario Traditional GPS A-GPS Improvement
Cold Start (no prior data) 30-60 seconds 5-10 seconds 6-10x faster
Warm Start (recent data) 15-30 seconds 2-5 seconds 5-10x faster
Hot Start (fresh data) 5-10 seconds 1-2 seconds 3-5x faster
Indoor/Weak Signal 120+ seconds or fail 10-30 seconds 4-10x faster

💡 Implementation Note

WIA-PET-008 requires A-GPS support for all certified devices. Assistance data should be cached locally with validity periods of 4 hours for ephemeris and 7 days for almanac data. Devices should automatically refresh assistance data when connected to cellular networks to ensure optimal performance.

2.4 Signal Quality and Accuracy Optimization

Raw GPS coordinates are rarely perfect. Various factors introduce errors that must be filtered and corrected for accurate pet tracking.

Error Sources in GNSS

Error Source Magnitude Mitigation Strategy
Atmospheric Delay 1-5 meters Ionospheric models, dual-frequency receivers
Multipath Effects 0.5-30 meters Signal filtering, constellation diversity
Satellite Clock 0-10 meters Clock corrections in navigation message
Orbital Errors 1-3 meters Precise ephemeris data via A-GPS
Receiver Noise 0.1-1 meter Quality hardware, Kalman filtering
Geometric Dilution Multiplier effect Multi-GNSS, satellite selection

Kalman Filtering for Position Smoothing

Kalman filters are optimal estimators that combine noisy sensor measurements with a mathematical model of expected behavior to produce smooth, accurate position estimates. For pet tracking, this means filtering out GPS jitter while still tracking real movement. 

// Simplified Kalman Filter for Pet Tracking
class LocationKalmanFilter {
  constructor() {
    this.state = {
      lat: 0, lng: 0,           // Position estimate
      vLat: 0, vLng: 0,         // Velocity estimate
      uncertainty: 100          // Position uncertainty (meters)
    };
    this.processNoise = 0.5;    // Expected process noise
    this.measurementNoise = 5.0; // GPS measurement noise
  }

  predict(dt) {
    // Predict next position based on velocity
    this.state.lat += this.state.vLat * dt;
    this.state.lng += this.state.vLng * dt;

    // Increase uncertainty due to process noise
    this.state.uncertainty += this.processNoise * dt;
  }

  update(measurement) {
    // Calculate Kalman gain
    const K = this.state.uncertainty /
              (this.state.uncertainty + this.measurementNoise);

    // Update estimate
    this.state.lat += K * (measurement.lat - this.state.lat);
    this.state.lng += K * (measurement.lng - this.state.lng);

    // Update velocity estimate
    const dt = (measurement.timestamp - this.lastTimestamp) / 1000;
    this.state.vLat = (measurement.lat - this.state.lat) / dt;
    this.state.vLng = (measurement.lng - this.state.lng) / dt;

    // Reduce uncertainty
    this.state.uncertainty *= (1 - K);

    this.lastTimestamp = measurement.timestamp;
    return this.state;
  }
}

2.5 Indoor and Challenging Environment Positioning

GPS signals are significantly degraded or completely blocked indoors. Since pets spend considerable time inside homes, vehicles, and buildings, alternative positioning methods are essential.

Hybrid Positioning Techniques

1. Wi-Fi Positioning

Uses signal strength from nearby Wi-Fi access points to estimate position. Accuracy: 5-20 meters in urban areas, requires database of Wi-Fi locations. 

// Wi-Fi Positioning Request
{
  "wifiAccessPoints": [
    {
      "macAddress": "00:11:22:33:44:55",
      "signalStrength": -45,
      "channel": 6
    },
    {
      "macAddress": "AA:BB:CC:DD:EE:FF",
      "signalStrength": -62,
      "channel": 11
    }
  ],
  "fallbackToGPS": true
}

2. Cellular Positioning (Cell-ID + Timing)

Uses cellular tower triangulation and timing advances. Accuracy varies widely: 50-2000 meters depending on cell density. Always available when device has cellular connectivity.

3. Bluetooth Beacons

Pre-installed BLE beacons in homes can provide room-level accuracy (1-5 meters). Requires initial setup but excellent for determining which room a pet is in.

4. Sensor Fusion and Motion Detection

Accelerometers and gyroscopes detect movement patterns. When GPS unavailable, dead reckoning can estimate position based on last known location plus detected movement.

WIA-PET-008 Indoor Positioning Standard

Devices should implement the following cascade:

  1. Primary: Multi-GNSS with minimum 4 satellites, HDOP < 5
  2. Secondary: Wi-Fi positioning when < 4 satellites available
  3. Tertiary: Cellular positioning (Cell-ID + TA) when Wi-Fi unavailable
  4. Fallback: Last known position + motion detection

Position reports must include confidence metric and positioning method used.

2.6 Location Data Quality Metrics

Not all position fixes are equal. Reporting confidence and accuracy metrics allows applications to make intelligent decisions about how to use location data.

Key Quality Indicators

Horizontal Dilution of Precision (HDOP)

Measures the geometric quality of satellite configuration. Lower is better:

  • < 1: Ideal - Excellent geometry
  • 1-2: Excellent - Very good for navigation
  • 2-5: Good - Acceptable for most applications
  • 5-10: Moderate - Use with caution
  • > 10: Poor - Do not use for critical applications

Estimated Position Error (EPE)

Statistical estimate of position uncertainty in meters. Modern GNSS receivers calculate this in real-time based on signal quality, satellite geometry, and known error sources.

2.7 Power-Efficient GPS Usage

GPS receivers are among the most power-hungry components in pet trackers, consuming 20-50mA during active operation. Efficient GPS usage is critical for battery life.

Power Optimization Strategies

1. Adaptive Update Intervals

Adjust GPS sampling frequency based on movement and context:

  • Stationary: 60-300 second intervals (pet at rest)
  • Walking: 30 second intervals (normal activity)
  • Running: 10 second intervals (high activity/escape)
  • Geofence Alert: 5 second intervals (exited safe zone)
  • Power Save Mode: 300-600 second intervals (low battery)

2. Duty Cycling

GPS on for brief periods, off between fixes. Example: Turn on GPS for 10 seconds every 60 seconds. Reduces average power by 80% while maintaining reasonable tracking.

3. Motion Detection Wake-Up

Use low-power accelerometer to detect movement. GPS stays off when pet is stationary, instantly activates when movement detected. Can reduce GPS power consumption by 60-90% for sedentary pets. 

// Battery-Aware GPS Scheduling
function calculateGPSInterval(batteryLevel, movementSpeed, mode) {
  const baseIntervals = {
    'power_save': 300,  // 5 minutes
    'normal': 60,       // 1 minute
    'active': 10        // 10 seconds
  };

  let interval = baseIntervals[mode] || 60;

  // Adjust for battery level
  if (batteryLevel < 20) {
    interval *= 2;  // Double interval when battery low
  } else if (batteryLevel < 10) {
    interval *= 5;  // 5x interval when critically low
  }

  // Adjust for movement
  if (movementSpeed > 2.0) {  // Running/vehicle
    interval = Math.min(interval, 10);
  } else if (movementSpeed < 0.1) {  // Stationary
    interval = Math.max(interval, 120);
  }

  return interval;
}

// Example power consumption calculation
const gpsPowerDraw = 45;  // mA when active
const cpuIdlePower = 2;   // mA when idle
const interval = 60;      // seconds between fixes
const fixDuration = 10;   // seconds GPS on per fix

const avgPower = (gpsPowerDraw * fixDuration + cpuIdlePower * (interval - fixDuration)) / interval;
// Result: ~9.5mA average vs. 45mA continuous
