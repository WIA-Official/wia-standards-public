CHAPTER 4

Feedback and Learning Loops

← Contents ← Previous Next →

The most powerful aspect of AI-human collaboration is continuous mutual improvement through feedback loops. Humans provide labeled examples and corrections that improve AI models, while AI insights and pattern recognition enhance human decision-making. This chapter explores how to design, implement, and optimize bidirectional learning systems guided by 弘益人間 principles of collective growth.

Bidirectional Learning Architecture

Traditional machine learning is unidirectional - humans label data, AI learns from it. True collaboration requires bidirectional learning where both parties improve through interaction. This creates a virtuous cycle: better AI helps humans make better decisions, and better human decisions create better training data for AI.

Human-to-AI Learning

Every human decision in a collaborative system is a potential training signal. Capturing, curating, and incorporating this feedback enables continuous AI improvement without dedicated labeling projects. 

interface HumanFeedback {
  feedback_id: string;
  timestamp: Date;
  human_id: string;
  context: {
    task_id: string;
    ai_prediction: any;
    ai_confidence: number;
    features: Map;
  };
  human_decision: {
    action: 'accept' | 'reject' | 'modify';
    final_output: any;
    confidence?: number;
    explanation?: string;
    time_spent: number;
  };
  feedback_quality: {
    confidence: number;
    expertise_match: number;
    consistency_with_policy: number;
  };
}

class FeedbackCollector {
  async captureFeedback(
    ai_output: any,
    human_decision: any,
    context: any
  ): Promise {
    const feedback: HumanFeedback = {
      feedback_id: generateId(),
      timestamp: new Date(),
      human_id: human_decision.reviewer_id,
      context: {
        task_id: context.task_id,
        ai_prediction: ai_output.prediction,
        ai_confidence: ai_output.confidence,
        features: context.features
      },
      human_decision: {
        action: determineAction(ai_output, human_decision),
        final_output: human_decision.output,
        confidence: human_decision.confidence,
        explanation: human_decision.explanation,
        time_spent: human_decision.time_spent
      },
      feedback_quality: await assessFeedbackQuality(
        human_decision,
        context
      )
    };

    await this.storeFeedback(feedback);
    return feedback;
  }

  async incorporateFeedback(
    feedback: HumanFeedback[],
    model: any
  ): Promise {
    // Filter for high-quality feedback
    const quality_feedback = feedback.filter(f =>
      f.feedback_quality.confidence > 0.7 &&
      f.feedback_quality.expertise_match > 0.8
    );

    // Convert to training examples
    const training_data = quality_feedback.map(f => ({
      features: f.context.features,
      label: f.human_decision.final_output,
      weight: f.feedback_quality.confidence
    }));

    // Incrementally update model
    return await model.update(training_data);
  }
}

AI-to-Human Learning

AI systems can enhance human decision-making by surfacing patterns, suggesting alternatives, and providing decision support. Over time, humans internalize these insights, improving their independent performance. 

interface AIInsight {
  insight_type: 'pattern' | 'anomaly' | 'prediction' | 'recommendation';
  content: string;
  supporting_evidence: Array<{
    source: string;
    confidence: number;
  }>;
  actionable: boolean;
  novelty_score: number;
}

class AITeacher {
  async generateInsights(
    human_decisions: HumanFeedback[],
    ai_model: any
  ): Promise {
    const insights: AIInsight[] = [];

    // Identify patterns in human decisions
    const patterns = await this.findPatterns(human_decisions);
    insights.push(...patterns.map(p => ({
      insight_type: 'pattern' as const,
      content: p.description,
      supporting_evidence: p.evidence,
      actionable: p.actionable,
      novelty_score: p.novelty
    })));

    // Highlight anomalies human might miss
    const anomalies = await ai_model.detectAnomalies(
      human_decisions.map(d => d.context)
    );
    insights.push(...anomalies.map(a => ({
      insight_type: 'anomaly' as const,
      content: `Unusual pattern detected: ${a.description}`,
      supporting_evidence: a.evidence,
      actionable: true,
      novelty_score: a.severity
    })));

    return insights;
  }

  async trackHumanLearning(
    human_id: string,
    insights_shown: AIInsight[],
    subsequent_decisions: HumanFeedback[]
  ): Promise<{
    insights_adopted: number;
    performance_change: number;
  }> {
    // Measure whether human incorporated AI insights
    const adoption_rate = this.measureInsightAdoption(
      insights_shown,
      subsequent_decisions
    );

    // Measure performance change
    const before_accuracy = await this.getAccuracy(
      human_id,
      new Date(Date.now() - 7 * 24 * 60 * 60 * 1000)
    );
    const after_accuracy = await this.getAccuracy(
      human_id,
      new Date()
    );

    return {
      insights_adopted: adoption_rate,
      performance_change: after_accuracy - before_accuracy
    };
  }
}

Active Learning Strategies

Not all feedback is equally valuable. Active learning selects which cases to present to humans to maximize learning efficiency - getting the most model improvement per human review.

Uncertainty Sampling

Prioritize cases where the AI model is most uncertain, as these labels provide the most information: 

interface UncertaintySampling {
  calculateUncertainty(prediction: any): number;
  selectForLabeling(
    candidates: any[],
    budget: number
  ): any[];
}

class EntropyBasedSampling implements UncertaintySampling {
  calculateUncertainty(prediction: {
    class_probabilities: Map
  }): number {
    // Calculate Shannon entropy
    let entropy = 0;
    for (const prob of prediction.class_probabilities.values()) {
      if (prob > 0) {
        entropy -= prob * Math.log2(prob);
      }
    }
    return entropy;
  }

  selectForLabeling(
    candidates: Array<{
      id: string;
      prediction: any;
    }>,
    budget: number
  ): any[] {
    // Score by uncertainty
    const scored = candidates.map(c => ({
      id: c.id,
      uncertainty: this.calculateUncertainty(c.prediction)
    }));

    // Select highest uncertainty cases
    scored.sort((a, b) => b.uncertainty - a.uncertainty);
    return scored.slice(0, budget);
  }
}

Query-by-Committee

Train multiple models and prioritize cases where they disagree: 

class QueryByCommittee {
  async selectControversialCases(
    models: any[],
    candidates: any[],
    budget: number
  ): Promise {
    const disagreements = [];

    for (const candidate of candidates) {
      // Get predictions from all models
      const predictions = await Promise.all(
        models.map(m => m.predict(candidate.features))
      );

      // Measure disagreement
      const disagreement = this.measureDisagreement(predictions);

      disagreements.push({
        id: candidate.id,
        disagreement_score: disagreement,
        predictions
      });
    }

    // Select most controversial
    disagreements.sort((a, b) =>
      b.disagreement_score - a.disagreement_score
    );

    return disagreements.slice(0, budget);
  }

  measureDisagreement(predictions: any[]): number {
    // For classification: fraction of predictions that differ
    const unique_predictions = new Set(
      predictions.map(p => JSON.stringify(p))
    );
    return unique_predictions.size / predictions.length;
  }
}

Expected Model Change

Estimate how much labeling each case would change the model, prioritizing high-impact labels: 

class ExpectedModelChange {
  async estimateImpact(
    candidate: any,
    current_model: any,
    possible_labels: any[]
  ): Promise {
    let expected_change = 0;

    for (const label of possible_labels) {
      // Estimate probability of this label
      const label_prob = await current_model.predict(
        candidate.features
      ).getProbability(label);

      // Estimate model change if we got this label
      const hypothetical_model = await this.simulateUpdate(
        current_model,
        candidate.features,
        label
      );

      const model_diff = this.modelDistance(
        current_model,
        hypothetical_model
      );

      expected_change += label_prob * model_diff;
    }

    return expected_change;
  }
}

Feedback Quality Assessment

Not all human feedback is equally reliable. Assess feedback quality to weight training examples appropriately and identify when humans need additional guidance.

Consistency Checking

Compare decisions across similar cases and reviewers: 

class ConsistencyChecker {
  async assessConsistency(
    feedback: HumanFeedback,
    historical_feedback: HumanFeedback[]
  ): Promise<{
    intra_reviewer_consistency: number;
    inter_reviewer_consistency: number;
    policy_consistency: number;
  }> {
    // Find similar past cases from same reviewer
    const similar_same_reviewer = historical_feedback.filter(f =>
      f.human_id === feedback.human_id &&
      this.areSimilar(f.context, feedback.context)
    );

    const intra_consistency = this.measureAgreement(
      feedback,
      similar_same_reviewer
    );

    // Find similar past cases from other reviewers
    const similar_other_reviewers = historical_feedback.filter(f =>
      f.human_id !== feedback.human_id &&
      this.areSimilar(f.context, feedback.context)
    );

    const inter_consistency = this.measureAgreement(
      feedback,
      similar_other_reviewers
    );

    // Check against documented policy
    const policy_consistency = await this.checkAgainstPolicy(
      feedback
    );

    return {
      intra_reviewer_consistency: intra_consistency,
      inter_reviewer_consistency: inter_consistency,
      policy_consistency
    };
  }
}

Confidence Calibration

Track whether reviewers' stated confidence matches actual accuracy: 

class ConfidenceCalibration {
  async calibrateReviewer(
    reviewer_id: string,
    feedback_history: HumanFeedback[],
    ground_truth: Map
  ): Promise<{
    calibration_curve: Array<{confidence: number, accuracy: number}>;
    calibration_score: number;
  }> {
    // Group feedback by confidence buckets
    const buckets = new Map();

    for (const f of feedback_history) {
      if (!f.human_decision.confidence) continue;

      const bucket = Math.floor(f.human_decision.confidence * 10) / 10;
      if (!buckets.has(bucket)) {
        buckets.set(bucket, {correct: 0, total: 0});
      }

      const truth = ground_truth.get(f.feedback_id);
      const stats = buckets.get(bucket)!;
      stats.total++;

      if (truth && this.isCorrect(f.human_decision.final_output, truth)) {
        stats.correct++;
      }
    }

    // Build calibration curve
    const curve = Array.from(buckets.entries()).map(([conf, stats]) => ({
      confidence: conf,
      accuracy: stats.correct / stats.total
    }));

    // Calculate calibration error (lower is better)
    const calibration_error = curve.reduce((sum, point) =>
      sum + Math.abs(point.confidence - point.accuracy),
      0
    ) / curve.length;

    return {
      calibration_curve: curve,
      calibration_score: 1 - calibration_error
    };
  }
}

Handling Feedback Conflicts

Different reviewers may provide conflicting feedback on similar cases. Resolving these conflicts is essential for maintaining model quality.

Conflict Resolution Strategies

  • Majority voting: Accept the most common label
  • Expertise weighting: Weight by reviewer expertise
  • Confidence weighting: Weight by stated confidence
  • Recency weighting: Prefer recent feedback (reflects current policy)
  • Expert arbitration: Escalate to senior reviewer
  • Policy clarification: Update guidelines when conflicts reveal ambiguity
class ConflictResolver {
  async resolveConflict(
    conflicting_feedback: HumanFeedback[]
  ): Promise<{
    resolved_label: any;
    resolution_method: string;
    confidence: number;
  }> {
    // Check if we can resolve by expertise
    const expertise_resolution = this.resolveByExpertise(
      conflicting_feedback
    );
    if (expertise_resolution.confidence > 0.8) {
      return expertise_resolution;
    }

    // Try confidence weighting
    const confidence_resolution = this.resolveByConfidence(
      conflicting_feedback
    );
    if (confidence_resolution.confidence > 0.7) {
      return confidence_resolution;
    }

    // Fall back to majority voting
    const majority_resolution = this.resolveByMajority(
      conflicting_feedback
    );
    if (majority_resolution.confidence > 0.6) {
      return majority_resolution;
    }

    // Escalate to expert arbitration
    return {
      resolved_label: null,
      resolution_method: 'requires_escalation',
      confidence: 0
    };
  }

  resolveByExpertise(
    feedback: HumanFeedback[]
  ): {resolved_label: any, resolution_method: string, confidence: number} {
    // Weight by reviewer expertise
    const weighted_votes = new Map();

    for (const f of feedback) {
      const expertise = f.feedback_quality.expertise_match;
      const label = JSON.stringify(f.human_decision.final_output);

      weighted_votes.set(
        label,
        (weighted_votes.get(label) || 0) + expertise
      );
    }

    const sorted = Array.from(weighted_votes.entries())
      .sort((a, b) => b[1] - a[1]);

    const total_weight = Array.from(weighted_votes.values())
      .reduce((a, b) => a + b, 0);

    return {
      resolved_label: JSON.parse(sorted[0][0]),
      resolution_method: 'expertise_weighted',
      confidence: sorted[0][1] / total_weight
    };
  }
}

Continuous Model Retraining

Incorporate feedback into models through continuous or periodic retraining. The strategy depends on data volume, model complexity, and operational requirements.

Incremental Learning

Update models continuously as new feedback arrives: 

class IncrementalLearner {
  async updateModel(
    current_model: any,
    new_feedback: HumanFeedback[]
  ): Promise {
    // Convert feedback to training examples
    const examples = new_feedback.map(f => ({
      features: f.context.features,
      label: f.human_decision.final_output,
      weight: f.feedback_quality.confidence
    }));

    // Incremental update
    return await current_model.partialFit(examples);
  }
}

Batch Retraining

Periodically retrain from scratch with accumulated feedback: 

class BatchRetrainer {
  async shouldRetrain(
    last_retrain: Date,
    feedback_since_retrain: number,
    performance_metrics: any
  ): boolean {
    // Retrain if:
    // 1. Been > 30 days since last retrain
    const days_since = (Date.now() - last_retrain.getTime()) /
                       (24 * 60 * 60 * 1000);
    if (days_since > 30) return true;

    // 2. Accumulated significant new feedback
    if (feedback_since_retrain > 10000) return true;

    // 3. Performance degraded
    if (performance_metrics.accuracy < 0.85) return true;

    return false;
  }

  async retrain(
    all_feedback: HumanFeedback[],
    model_architecture: any
  ): Promise {
    // Full retraining with all historical feedback
    const training_data = this.prepareTrainingData(all_feedback);
    return await model_architecture.train(training_data);
  }
}

Measuring Learning Effectiveness

Track metrics to ensure feedback loops are improving both AI and human performance:

AI Learning Metrics

  • Model accuracy over time: Is the AI improving?
  • Confidence calibration: Does AI know what it knows?
  • Feedback efficiency: Model improvement per feedback example
  • Generalization: Performance on novel cases

Human Learning Metrics

  • Decision accuracy over time: Are humans improving?
  • Review speed: Are humans getting faster?
  • Insight adoption rate: Do humans incorporate AI suggestions?
  • Consistency: Are decisions becoming more consistent?
弘益人間 in Learning Loops

Feedback loops should foster growth for both AI and humans. Avoid designs that extract labor from humans purely to train AI. Instead, ensure humans learn and develop through the feedback process. Provide AI insights that enhance human expertise. The goal is mutual flourishing, not optimization at human expense.

Chapter Summary

Effective feedback loops enable continuous improvement of both AI systems and human decision-makers. Key concepts include:

  • Bidirectional learning where humans improve AI through feedback and AI enhances human decision-making through insights
  • Active learning strategies (uncertainty sampling, query-by-committee) maximize learning efficiency
  • Feedback quality assessment through consistency checking and confidence calibration
  • Conflict resolution strategies when reviewers disagree on similar cases
  • Incremental and batch retraining approaches for incorporating feedback into models
  • Comprehensive metrics to track both AI and human learning effectiveness
  • Design principles that foster mutual growth rather than exploiting human labor for AI training
