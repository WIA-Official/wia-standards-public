# 6장: 보안 및 개인 정보 보호

## 6.1 생체 인식 데이터 보호

### VR의 고유한 개인 정보 보호 문제

가상 현실 및 증강 현실 장치는 전례 없는 규모와 세분성으로 생체 인식 데이터를 수집합니다. 지문이나 얼굴 스캔을 수집할 수 있는 스마트폰과 달리 VR 헤드셋은 사용자에 대한 친밀한 세부 정보를 드러내는 행동 생체 인식을 지속적으로 수집합니다.

**수집되는 생체 인식 데이터 유형**:

**눈 추적 데이터**:
- 시선 패턴 (무엇을 얼마나 오래 보는지)
- 동공 팽창 (감정적 각성, 인지 부하)
- 깜박임 속도 및 패턴 (주의, 스트레스, 속임수)
- 단속 운동 패턴 (읽기 능력, 신경학적 마커)
- **민감도**: 의학적 상태, 성적 지향, 감정 상태, 속임수 추론 가능

**얼굴 표정 캡처**:
- 내부를 향한 카메라가 얼굴 움직임 캡처
- 혀 움직임, 뺨 팽창, 눈 찡그림
- 아바타를 위한 얼굴 표정 재구성
- **민감도**: 사용자가 숨기고 싶어할 수 있는 진정한 감정 반응 공개

**음성 지문 보호**:
- 통신 및 명령을 위한 음성 녹음
- 고유한 음성 특성 (음성 지문)
- 말 패턴, 억양, 언어 능력
- **민감도**: 영구적인 식별자, 사칭에 사용될 수 있음

**보행 분석**:
- 신체 추적에서 걷기 패턴
- 보폭 길이, 케이던스, 비대칭
- 균형 및 자세
- **민감도**: 고유하게 식별 (사람은 걸음걸이로 식별 가능), 부상이나 상태 공개

**손 움직임 패턴**:
- 미세 운동 제어 특성
- 떨림 패턴
- 손잡이 및 손재주
- **민감도**: 파킨슨병, 기타 신경학적 상태 표시 가능

### 법적 및 규제 프레임워크

**GDPR (일반 데이터 보호 규정 - EU)**:
- 생체 인식 데이터 = "특별 범주 데이터" (9조)
- 명시적 동의 필요 (약관 수락만으로는 부족)
- 삭제 권리 (사용자가 삭제 요구 가능)
- 데이터 최소화 (필요한 것만 수집)
- 목적 제한 (동의 없이 데이터 재사용 불가)
- **처벌**: 최대 €2천만 또는 글로벌 수익의 4%

**CCPA/CPRA (캘리포니아 소비자 개인 정보 보호법)**:
- 생체 정보 = "민감한 개인 정보"
- 데이터 판매에 대한 옵트아웃 권리
- 수집되는 데이터를 알 권리
- 삭제 권리
- **처벌**: 위반당 $2,500-7,500

**BIPA (일리노이 생체 정보 개인 정보 보호법)**:
- 개인 소송권 (사용자가 직접 소송 가능)
- 정보에 입각한 서면 동의 필요
- 보유 제한 및 파기 일정 의무
- **처벌**: 위반당 $1,000-5,000
- **주목할 만한**: 여러 VR/AR 회사가 BIPA 소송 직면 (2023-2025년)

**중국 PIPL (개인 정보 보호법)**:
- 생체 인식 데이터 = "민감한 개인 정보"
- 개인 동의 필요
- 국경 간 이전 제한
- 현지화 요구 사항

**신흥 규정**:
- **미국 연방**: 여러 생체 인식 개인 정보 보호 법안 제안 (2024-2026년)
- **주 수준**: 15개 이상의 미국 주에서 생체 인식 개인 정보 보호 법 고려 중
- **산업별**: 의료 VR은 HIPAA 적용, 어린이 VR은 COPPA 적용

### WIA-VR 생체 인식 데이터 보호 표준

**원칙 1: 데이터 최소화**:
- 기능에 필수적인 생체 인식 데이터만 수집
- 중심 와 렌더링을 위한 눈 추적: 필요
- 광고 프로파일링을 위한 눈 추적: 명시적 옵트인 없이 금지

**원칙 2: 온디바이스 처리**:
- 원시 생체 인식 데이터는 동의 없이 장치를 떠나서는 안 됨
- 로컬에서 처리, 파생/익명화된 데이터만 전송
- 예: 눈 시선 → 중심 와 렌더링 명령 (OK)
- 예: 원시 눈 비디오 스트림 → 클라우드 (금지)

**원칙 3: 명시적 동의**:
- 세분화된 동의 (TOS 수락과 번들되지 않음)
- 애플리케이션별 동의 (플랫폼 전체 일괄 승인 아님)
- 언제든지 취소 가능
- 어떤 데이터, 왜, 어떻게 사용되는지 명확한 설명

**원칙 4: 투명한 수집**:
- 생체 인식 센서가 활성화될 때 시각적 표시기
- 카메라 라이트 (노트북 웹캠 라이트처럼)
- 시야에 눈 추적 아이콘
- 녹음 중일 때 마이크 아이콘

**원칙 5: 안전한 저장**:
- 저장 시 암호화 (최소 AES-256)
- 전송 중 암호화 (TLS 1.3+)
- 액세스 제어 (모든 앱이 액세스할 수 없음)
- 키를 위한 하드웨어 보안 모듈

**원칙 6: 제한된 보유**:
- 세션 전용 데이터는 세션 후 즉시 삭제
- 임시 데이터는 30일 이내에 삭제
- 명시적 동의 및 정당화가 있는 경우에만 장기 데이터
- 사용자가 시작한 삭제는 48시간 이내에 처리

**원칙 7: 놀라움 없음**:
- 공개되지 않은 생체 인식 수집 없음
- 새로운 동의 없이 재사용 불가
- 명시적 옵트인 없이 판매 또는 공유 불가
- 데이터 사용을 확장하는 알고리즘 변경은 새로운 동의 필요

### 익명화 및 비식별화

**과제**: 생체 인식 데이터의 진정한 익명화는 극도로 어렵습니다
- 눈 추적 패턴은 지문만큼 고유
- "익명화된" 데이터로도 재식별 가능

**접근 방식**:

**차등 개인 정보 보호**:
- 데이터에 통계적 노이즈 추가
- 인구 추세를 보존하면서 개인 식별 방지
- 많은 분석에 대한 Apple의 접근 방식

**집계**:
- 그룹 통계만 보고 (개인 절대 안 됨)
- 최소 그룹 크기 (k-익명성: k≥100)

**데이터 세분화**:
- 식별 데이터와 행동 데이터 분리
- 사용자 ID 대신 가명/토큰 사용
- 가명을 개인에게 다시 연결할 방법이 없는지 확인

**WIA-VR 요구 사항**:
- 익명화 기술이 문서화되어야 함
- 익명화 효과의 타사 감사
- 재식별 위험이 >1%인 경우 사용자 알림

## 6.2 공간 데이터 보안

### 방 스캔의 개인 정보 보호 의미

VR/AR 장치는 물리적 공간의 상세한 3D 맵을 만듭니다. 이 공간 데이터는 다음을 공개합니다:
- 집 레이아웃 (평면도, 방 크기)
- 가구 및 소유물 (부의 지표)
- 사진, 예술 작품, 개인 물품 (정치적 견해, 종교, 관계)
- 존재하는 사람들 (패스스루에서 얼굴 인식)
- 위치 (GPS 또는 Wi-Fi 위치 결정과 결합)

### 공간 데이터 유형

**방 기하학**:
- 표면의 3D 메시 (벽, 바닥, 가구)
- 객체 인식 (소파, 테이블, TV)
- 공간 앵커 (AR 콘텐츠를 위한 영구 위치)

**환경 스캔**:
- 패스스루 카메라 피드 (주변의 컬러 비디오)
- 깊이 맵 (LiDAR 또는 스테레오 깊이)
- 의미론적 이해 (이것은 부엌, 저것은 침실)

**위치 컨텍스트**:
- GPS 좌표 (실외 AR)
- Wi-Fi SSID 매핑 (실내 위치 결정)
- 시각적 랜드마크 (특정 장소 인식)

### 보안 위험

**물리적 보안**:
- 도둑이 침입 계획을 세우기 위해 방 스캔 사용 (레이아웃 알기)
- 집의 귀중한 물품 식별
- 집이 비어 있을 때 결정

**개인 정보 침해**:
- 민감한 물품 공개 (약물, 정치 포스터, 종교 상징)
- 무단 감시 (스파이 카메라로서의 패스스루 카메라)
- 환경에 있는 사람들이 동의 없이 캡처됨

**기업 스파이 활동**:
- 기업 AR이 경쟁업체 시설 스캔
- 작업장 스캔에서 보이는 영업 비밀
- 패스스루를 통해 사진 촬영된 독점 디자인

### WIA-VR 공간 데이터 보호 표준

**원칙 1: 로컬 우선**:
- 기본적으로 로컬에 저장된 공간 맵
- 명시적 사용자 선택으로만 클라우드 동기화
- 클라우드 저장 맵에 대한 엔드 투 엔드 암호화

**원칙 2: 목적 제한**:
- 장치 기능만을 위한 공간 데이터 (추적, MR 배치)
- 명시적 옵트인 없이 광고 사용 금지
- 공간 데이터 판매 또는 공유 금지

**원칙 3: 일시성**:
- 임시 공간 데이터 (현재 세션 추적용)는 종료 시 삭제
- 사용자 생성/승인이 있는 경우에만 영구 앵커
- 90일 동안 사용하지 않은 오래된 공간 데이터 자동 삭제

**원칙 4: 패스스루 개인 정보 보호**:
- 기본적으로 패스스루 비디오 녹화 안 됨
- 녹화된 경우 (화면 캡처, 스크린샷), 명확한 표시기
- 패스스루 데이터는 명시적 작업 없이 장치를 떠나지 않음
- 방관자의 얼굴 흐림/익명화

**원칙 5: 객체 인식 제한**:
- 의미론적 이해 (방 유형, 가구) OK
- 특정 객체 식별 (브랜드, 모델)은 동의 필요
- 텍스트 인식 (OCR) 옵트인 전용
- 얼굴당 명시적 동의 없이 얼굴 인식 금지

**기술 구현**:

**공간 데이터 암호화**:
- 암호화된 저장소 (파일 시스템 수준 + 애플리케이션 수준)
- 민감한 공간 토큰을 위한 보안 엔클레이브
- 제로 지식 아키텍처 (클라우드 서비스는 사용자 맵을 해독할 수 없음)

**차등 매핑**:
- 추적에 필요한 기하학적 기능만 저장
- 가능한 경우 색상/텍스처 데이터 삭제
- 사실적인 것보다 추상적 표현 사용

**일시적 매핑**:
- 영구 저장 없이 실시간 SLAM
- 각 세션마다 맵 재구성 (느리지만 개인 정보 보호)
- 일시적과 영구 사이의 사용자 선택

**공간 데이터 방화벽**:
- 앱이 공간 맵에 액세스하기 위한 권한 요청
- 세분화된 권한 (방 기하학 OK, 객체 레이블 NO)
- 사용자가 볼 수 있는 권한 프롬프트

## 6.3 인증 및 액세스 제어

### 생체 인식 로그인 시스템

VR 헤드셋은 새로운 생체 인식 인증 방법을 제공합니다:

**눈 기반 인증 (Optic ID)**:
- 홍채 패턴 인식 (Apple Vision Pro)
- 지문이나 얼굴보다 더 안전 (속이기 어려움)
- 손이 필요하지 않음 (헤드셋 착용 시 유용)
- 개인 정보 보호: 홍채 템플릿은 보안 엔클레이브에 저장, 전송되지 않음

**얼굴 기반 (VR의 Face ID)**:
- 내부를 향한 카메라가 얼굴 캡처
- 하이브리드 접근 방식: 부분 얼굴 (눈 주변 영역, 코다리)
- 지속적인 인증 (사용자가 여전히 헤드셋을 착용하고 있는지 확인)

**시선 패턴 인증**:
- 사용자가 눈으로 움직이는 대상을 따라감
- 고유한 눈 움직임 패턴이 신원 확인
- 연구 단계 (아직 상용화되지 않음)

**음성 인증**:
- 음성 지문 인식
- 챌린지-응답 (특정 구문 말하기)
- 지속적인 배경 확인

**WIA-VR 인증 표준**:

**최소 (기본 계층)**:
- PIN 또는 패턴 잠금 해제
- 단일 사용자 장치 가정

**표준 계층**:
- 빠른 전환이 가능한 다중 사용자 지원
- 생체 인식 잠금 해제 (최소 한 가지 방법)
- 대체 인증 (생체 인식 실패 시)

**프리미엄 계층**:
- 다중 요소 인증
- 지속적인 인증 (사용자가 헤드셋을 교환하는지 감지)
- 안전한 결제 승인

**기업 계층**:
- SSO (Single Sign-On) 통합
- SAML, OAuth 2.0 지원
- 하드웨어 보안 키 지원
- 조건부 액세스 (위치, 장치 상태 확인)

### 다중 사용자 장치 관리

**과제**: VR 헤드셋은 개인 장치이지만 (핏, IPD, 선호도) 공유될 수 있습니다 (가족, 기업).

**사용자 프로필 시스템**:
- 빠른 사용자 전환 (헤드셋 착용 시 사용자 인식)
- 사용자별 설정: IPD, 키, 컨트롤러 선호도
- 별도의 콘텐츠 라이브러리, 저장된 게임
- 개인 정보 보호 분리 (한 사용자가 다른 사용자의 데이터를 볼 수 없음)

**자동 사용자 인식**:
- 헤드셋 착용 시 생체 인식 식별
- 시선 패턴, IPD 또는 부분 얼굴 인식
- 수동적 (명시적 로그인 없음)
- 불확실한 경우 수동 선택으로 대체

**게스트 모드**:
- 방문자를 위한 임시 프로필
- 제한된 액세스 (앱 설치, 개인 데이터 액세스 불가)
- 게스트 모드 종료 시 자동 데이터 삭제

**부모 통제**:
- 연령 제한 콘텐츠 필터링
- 사용 시간 제한
- 활동 모니터링 (어떤 앱을 얼마나 사용했는지)
- 구매 제어 (부모 승인 필요)

**기업 장치 관리**:
- 모바일 장치 관리 (MDM) 통합
- 원격 프로비저닝
- 앱 화이트리스트/블랙리스트
- 사용 분석
- 원격 와이프 기능

## 6.4 콘텐츠 보안

### VR 콘텐츠용 DRM

**과제**:
- 고가치 콘텐츠 (영화, AAA 게임)는 보호 필요
- VR 콘텐츠가 2D보다 더 크고 복잡
- 여러 배포 플랫폼 (상호 운용 가능한 DRM 필요)

**DRM 시스템**:

**Widevine (Google)**:
- VR의 비디오 콘텐츠에 사용 (YouTube VR, 스트리밍 앱)
- 하드웨어 기반 보안 (신뢰 실행 환경)
- 레벨: L1 (하드웨어), L2 (소프트웨어), L3 (최저)
- WIA-VR 요구 사항: 프리미엄 계층을 위한 Widevine L1

**FairPlay (Apple)**:
- Apple 생태계 DRM (Vision Pro 콘텐츠)
- Apple ID 및 구매와 통합
- 하드웨어 암호화

**플랫폼별 DRM**:
- **Quest**: Oculus DRM (계정 + 장치에 콘텐츠 연결)
- **PlayStation**: PlayStation DRM
- **SteamVR**: Steam DRM (덜 제한적, 종종 크랙됨)

**WIA-VR DRM 원칙**:
- **사용자 권리**: 구매한 콘텐츠는 계속 액세스 가능
- **이식성**: 사용자가 새 장치에서 재다운로드 가능
- **오프라인**: DRM이 오프라인에서 작동 (초기 활성화 후)
- **투명성**: DRM 제한이 명확하게 공개됨
- **루트킷 없음**: DRM이 장치 보안을 손상시키지 않아야 함

### 불법 복제 방지 조치

**기술**:

**소프트웨어 난독화**:
- 역공학을 방지하기 위한 코드 난독화
- 변조 방지 (수정된 앱 감지, 실행 거부)

**서버 측 검증**:
- 합법적인 라이선스에 대한 온라인 확인
- 우아한 저하 (싱글 플레이어는 오프라인에서 작동, 멀티플레이어는 온라인 필요)

**워터마킹**:
- 콘텐츠의 보이지 않는 마커가 소스 계정으로 누출 추적
- 스크린캡 불법 복제 억제

**하드웨어 바인딩**:
- 특정 장치 일련 번호에 연결된 라이선스
- 전송 허용 (제한된 횟수)

**WIA-VR 불법 복제 방지 균형**:
- 불법 복제 보호 합법적
- 합법적인 사용자를 처벌하지 않아야 함 (과도한 온라인 확인, 성능 영향)
- 사용자 권리 보존 (접근성, 백업)

### 안전한 결제 처리

**VR 내 구매**:
- 콘텐츠, 인앱 구매, 가상 상품
- 편의성 (VR에 있는 동안 구매)
- 보안 위험 (VR UI가 스푸핑될 수 있음, 사용자가 가짜 결제 화면을 알아차리지 못할 수 있음)

**WIA-VR 결제 보안 요구 사항**:

**신뢰할 수 있는 UI**:
- 결제 화면은 플랫폼 제어여야 함 (앱 렌더링 아님)
- 명확한 시각적 표시기 (앱으로 위조될 수 없음)
- 예: Apple Vision Pro의 시스템 결제 시트

**생체 인식 승인**:
- 임계값 이상의 구매에 대해 생체 인식 필요 (눈 스캔, 음성)
- $0-5: 인증 없음 (마찰 없음)
- $5-50: 선택적 생체 인식
- $50+: 필수 생체 인식

**거래 제한**:
- 일일/주간 지출 한도 (사용자 구성 가능)
- 비정상적인 구매에 대한 사기 경고

**결제 방법**:
- 신용/직불 카드 (PCI-DSS 준수)
- 디지털 지갑 (Apple Pay, Google Pay)
- 암호화폐 (지원되는 경우, 변동성 경고 포함)
- 통신사 청구 (모바일 결제)

**환불 권리**:
- 14일 환불 기간 (EU 법, 전 세계 모범 사례)
- 플레이 시간 제한 (예: <2시간 플레이)
- VR 내 명확한 환불 정책

**어린이 구매**:
- 부모 승인 필요 (명시적, 비밀번호만으로는 부족)
- 조작적 전술 금지 (어린이를 대상으로 하는 루트 박스)
- COPPA 준수 (미국, 13세 미만)

## 6.5 네트워크 보안

### 암호화된 통신

**WIA-VR 네트워크 암호화 표준**:

**전송 보안**:
- **TLS 1.3** 최소 (더 이상 사용되지 않는 TLS 1.2 및 이전)
- 중요한 연결에 대한 인증서 피닝 (MITM 방지)
- 완전 순방향 비밀성 (세션 키 손상이 과거 세션을 노출하지 않음)

**애플리케이션 수준 암호화**:
- 통신 앱을 위한 엔드 투 엔드 암호화 (음성/비디오 채팅)
- 제로 지식 아키텍처 (서비스 제공자가 해독할 수 없음)

**VPN 지원**:
- 네이티브 VPN 클라이언트 또는 WireGuard/OpenVPN 지원
- 기업: 기업 앱에 대한 필수 VPN
- 소비자: 개인 정보 보호 의식이 있는 사용자를 위한 옵션

### 중간자 공격 방지

**공격 벡터**:
- 공용 Wi-Fi 가로채기 (커피숍, 공항)
- 손상된 홈 라우터
- 이블 트윈 액세스 포인트
- DNS 스푸핑

**완화**:

**인증서 검증**:
- 엄격한 인증서 확인 (잘못된 인증서에 대한 사용자 재정의 없음)
- 인증서 투명성 시행

**DNS 보안**:
- DNS-over-HTTPS (DoH) 또는 DNS-over-TLS (DoT)
- DNSSEC 검증

**Wi-Fi 보안**:
- WPA3 필수 (WPA2 더 이상 사용 안 함)
- SAE (Simultaneous Authentication of Equals)가 사전 공격 방지
- 개방 네트워크를 위한 OWE (Opportunistic Wireless Encryption)

**네트워크 신뢰 표시기**:
- 신뢰할 수 없는 네트워크에 대한 시각적 경고
- 공용 Wi-Fi에서 VPN 권장

### 안전한 클라우드 렌더링

**과제**:
- 클라우드 렌더링은 입력 데이터를 원격 서버로 전송
- 서버가 렌더링하고 비디오를 다시 스트리밍
- 모든 상호 작용이 클라우드를 통과 (개인 정보 보호 위험)

**클라우드 VR에 대한 보안 요구 사항**:

**데이터 보유**:
- 입력 데이터 (머리 위치, 컨트롤러 상태)는 사용 후 즉시 삭제
- 사용자 작업 로깅 금지
- 렌더 서버는 상태 비저장이어야 함

**격리**:
- 격리된 컨테이너의 각 사용자 세션
- 세션 간 데이터 누출 없음

**암호화**:
- 엔드 투 엔드 암호화된 비디오 스트림
- 인증된 연결만

**준수**:
- GDPR 준수 데이터 처리
- 데이터 거주 (EU 사용자 데이터는 EU에 유지)

**타사 감사**:
- 클라우드 인프라의 연간 보안 감사
- 침투 테스트
- SOC 2 Type II 인증

## 6.6 규제 준수

### 건강 데이터 (의료 VR용 HIPAA)

의료 및 치료 VR 애플리케이션은 의료 규정을 준수해야 합니다:

**HIPAA (건강 보험 이식성 및 책임에 관한 법 - 미국)**:

**적용 대상**:
- 환자 치료를 위한 VR 앱 (수술 교육, 치료, 진단)
- 수집된 건강 데이터 (증상, 치료 반응)

**요구 사항**:
- **암호화**: ePHI (전자 보호 건강 정보)는 저장 및 전송 시 암호화
- **액세스 제어**: 역할 기반, 감사 로그
- **비즈니스 제휴 계약**: ePHI를 처리하는 모든 공급업체와 BAA
- **침해 통지**: 데이터가 손상된 경우 60일 통지
- **환자 권리**: 액세스, 수정, 공개 회계

**기술적 안전 조치**:
- 고유한 사용자 식별
- 비활성 후 자동 로그오프
- 감사 제어 (누가 무엇을, 언제 액세스했는지)
- 데이터 무결성 제어 (무단 변경 감지)

**WIA-VR 의료 인증**:
- HIPAA 준수 문서
- 위험 분석 및 완화 계획
- 의료 IT 전문가의 보안 감사
- 지속적인 준수 모니터링

### 교육 기록 (FERPA)

**FERPA (가족 교육 권리 및 개인 정보 보호법 - 미국)**:

교육의 VR에 적용:
- 학생 성과 데이터
- 학습 분석
- 교육 VR의 행동 데이터

**요구 사항**:
- 부모 동의 (18세 미만 학생의 경우)
- 교육 목적 제한
- 데이터 최소화
- 학생 데이터 판매 금지

### 금융 규정

**VR 뱅킹/금융 앱**:
- **PCI-DSS**: 신용 카드 처리 시
- **SOX**: 상장 기업 재무 데이터인 경우
- **지역**: EU PSD2 등

**가상 상품 및 통화**:
- **자금 세탁 방지 (AML)**: 가상 통화에 대한 고객 알기
- **세금 준수**: 가상 상품 판매는 판매세 적용
- **증권법**: 가상 상품이 증권으로 자격이 있는 경우 (드물음)

### 수출 통제

**군사/국방 VR**:
- **ITAR (국제 무기 거래 규정)**
- 국방 애플리케이션을 위한 VR 교육은 수출 통제될 수 있음
- 기술 이전 제한

**암호화 수출**:
- 강력한 암호화 (>56비트)는 역사적으로 수출 통제됨
- 대부분의 소비자 제품에 대해 완화되었지만 문서 필요

### 지역 요구 사항

**중국**:
- **사이버 보안법**: 데이터 현지화 (중국 사용자 데이터는 중국에)
- **콘텐츠 검열**: 금지된 콘텐츠 (정치, 폭력, 도박)
- **실명 등록**: 사용자 신원 확인
- **정부 백도어**: 법 집행 액세스를 위한 잠재적 요구 사항

**EU**:
- **GDPR**: 이미 다룸 (6.1절)
- **디지털 서비스법 (DSA)**: 콘텐츠 조정 요구 사항
- **AI 법**: 생체 인식 분류 제한

**러시아**:
- **데이터 현지화**: 사용자 데이터는 러시아에 저장되어야 함
- **SORM**: 법 집행 액세스 요구 사항

**WIA-VR 접근 방식**:
- 각 주요 시장에 대한 준수 지침
- 지역 인증 트랙
- 준수 상태의 명확한 라벨링
