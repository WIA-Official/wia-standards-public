IoT 센서 네트워크

센서 융합, 엣지 컴퓨팅, 무선 프로토콜, 스마트 빌딩 등 연결된 센서 생태계를 탐구합니다.

⏱️ 읽기 시간 35분 📊 중급 수준 📅 2025년 1월 업데이트

🌐 IoT 센서 개요

IoT(Internet of Things) 센서 네트워크는 물리적 세계의 데이터를 수집하여 인터넷을 통해 전송하고 분석하는 시스템입니다. 스마트 홈, 스마트 시티, 산업 IoT(IIoT), 농업, 헬스케어 등 다양한 분야에서 활용됩니다.

📊 IoT 시장 전망

2025년까지 전 세계적으로 750억 개 이상의 IoT 장치가 연결될 것으로 예상됩니다. IoT 센서 시장은 연평균 25% 이상 성장하고 있습니다.

IoT 아키텍처 레이어

📡
인지 레이어 (Perception Layer)
센서, 액추에이터, RFID 태그 등 물리적 데이터 수집 장치
🔗
네트워크 레이어 (Network Layer)
WiFi, BLE, LoRa, NB-IoT 등 데이터 전송 인프라
⚙️
처리 레이어 (Processing Layer)
엣지 게이트웨이, 클라우드 서버에서 데이터 처리 및 분석
📱
응용 레이어 (Application Layer)
대시보드, 모바일 앱, 자동화 시스템 등 사용자 인터페이스

📡 무선 통신 프로토콜

IoT 센서 네트워크에서는 다양한 무선 통신 프로토콜이 사용됩니다. 각 프로토콜은 전력 소비, 데이터 속도, 통신 거리, 네트워크 토폴로지 등에서 서로 다른 특성을 가지고 있습니다.

주요 무선 프로토콜 비교

프로토콜 범위 데이터 속도 전력 용도
WiFi 50-100m ~1 Gbps 높음 고속 데이터, 카메라
BLE 5.0 100-400m 2 Mbps 매우 낮음 웨어러블, 비콘
Zigbee 10-100m 250 kbps 낮음 홈 자동화, 조명
Z-Wave 30-100m 100 kbps 낮음 스마트 홈
LoRa 2-15 km 0.3-50 kbps 매우 낮음 스마트 시티, 농업
NB-IoT 셀룰러 200 kbps 낮음 자산 추적, 미터링
Thread 30m 250 kbps 낮음 스마트 홈 (Matter)

LPWAN (Low Power Wide Area Network)

📶
LoRaWAN

비면허 대역, 자가 네트워크 구축 가능. 스마트 시티, 농업, 환경 모니터링에 적합.

📱
NB-IoT

LTE 네트워크 활용, 통신사 서비스. 깊은 실내 침투, 대량 연결에 적합.

🌐
LTE-M

이동성 지원, VoLTE 가능. 자산 추적, 웨어러블에 적합.

📡
Sigfox

글로벌 네트워크, 초저전력. 단방향 통신 주로, 단순 센서에 적합.

🧠 엣지 컴퓨팅

엣지 컴퓨팅은 데이터 생성 위치(센서)에 가까운 곳에서 데이터를 처리하는 분산 컴퓨팅 패러다임입니다. 지연 시간 감소, 대역폭 절약, 프라이버시 향상 등의 이점이 있습니다.

엣지 컴퓨팅 이점

⚡
저지연

로컬 처리로 밀리초 단위 응답. 자율주행, 산업 제어에 필수.

📉
대역폭 절약

원시 데이터 대신 결과만 전송. 네트워크 비용 절감.

🔒
프라이버시

민감한 데이터를 로컬에서 처리. 개인정보 보호 강화.

🔌
오프라인 작동

네트워크 단절 시에도 기능 유지. 안정성 향상.

엣지 AI

엣지 AI는 머신러닝 모델을 센서 장치 또는 게이트웨이에서 직접 실행합니다. TensorFlow Lite, ONNX Runtime, NVIDIA TensorRT 등의 프레임워크가 사용됩니다.

// 엣지 센서 노드 구조
interface EdgeSensorNode {
    id: string;
    sensors: Sensor[];
    connectivity: {
        primary: 'WiFi' | 'BLE' | 'LoRa' | 'NB-IoT';
        backup?: string;
    };
    processing: {
        mcu: string;           // ESP32, STM32, nRF52
        memory: number;        // KB
        aiCapable: boolean;
    };
    power: {
        source: 'battery' | 'solar' | 'mains';
        capacity?: number;     // mAh
        consumption: number;   // mW
    };
}

async function processLocally(data: SensorData): Promise<ProcessedResult> {
    // 로컬에서 AI 추론 실행
    const prediction = await tflite.run(model, data);
    
    // 이상 탐지 시에만 클라우드로 전송
    if (prediction.anomaly) {
        await mqtt.publish('alerts', prediction);
    }
    
    return prediction;
}

🏢 스마트 빌딩

스마트 빌딩은 IoT 센서와 자동화 시스템을 통해 에너지 효율성, 편의성, 안전성을 최적화합니다. BMS(Building Management System)가 모든 시스템을 통합 관리합니다.

스마트 빌딩 센서

🌡️
환경 센서

온도, 습도, CO2, VOC 측정. HVAC 최적화 및 실내 공기질 관리.

👥
점유 센서

PIR, ToF, 레이더로 재실 감지. 조명/공조 자동 제어.

💡
조도 센서

자연광 측정, 디머블 조명 제어. 에너지 절약 및 편안함.

⚡
에너지 미터

전력, 가스, 물 소비 모니터링. 비용 분석 및 최적화.

🚪
접근 제어

카드, 생체인식, 모바일 키. 출입 기록 및 보안.

🔥
화재 감지

연기, 열, 가스 감지. 생명 안전 및 법규 준수.

빌딩 자동화 프로토콜

프로토콜 유형 용도
BACnet 유/무선 HVAC, 빌딩 자동화 표준
Modbus 유선 산업용 제어, 레거시 시스템
KNX 유/무선 유럽 표준, 조명/블라인드
DALI 유선 디지털 조명 제어
EnOcean 무선 에너지 하베스팅 스위치

🏭 산업용 IoT (IIoT)

IIoT는 제조업, 에너지, 물류 등 산업 환경에서의 IoT 적용입니다. 예측 유지보수, 품질 관리, 자산 추적, 작업자 안전 등에 활용됩니다.

IIoT 응용

🔧
예측 유지보수

진동, 온도, 전류 모니터링으로 고장 예측. 다운타임 최소화.

📊
OEE 모니터링

설비 가동률, 성능, 품질 실시간 측정. 생산성 향상.

📦
자산 추적

RFID, BLE 태그로 재고, 공구, 컨테이너 위치 파악.

👷
작업자 안전

웨어러블로 낙상 감지, 위험 구역 경보, 가스 노출 모니터링.

⚠️ IIoT 보안 고려사항

산업 시스템은 사이버 공격 시 물리적 피해가 발생할 수 있습니다. OT(Operational Technology) 네트워크 분리, 접근 제어, 암호화, 정기적 패치가 필수입니다.

🔐 IoT 보안

IoT 장치는 제한된 리소스, 물리적 접근 가능성, 대규모 배포 등으로 인해 고유한 보안 과제를 가지고 있습니다. 엔드투엔드 보안 전략이 필요합니다.

IoT 보안 영역

🔑
디바이스 인증

X.509 인증서, 보안 요소(SE), TPM으로 장치 신원 확인.

🔒
데이터 암호화

TLS/DTLS로 전송 중 암호화. 저장 데이터도 암호화.

🔄
펌웨어 업데이트

OTA(Over-The-Air) 보안 업데이트. 서명 검증 필수.

🛡️
네트워크 분리

IoT 전용 VLAN, 방화벽 규칙. 제로 트러스트 아키텍처.
