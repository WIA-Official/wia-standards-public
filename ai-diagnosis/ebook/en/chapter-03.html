Introduction to Radiology AI

Radiology represents one of the most mature applications of artificial intelligence in medicine. The visual nature of radiological interpretation makes it well-suited for deep learning approaches that excel at image pattern recognition. AI systems in radiology aim to assist radiologists by automating routine tasks, highlighting potential abnormalities, providing quantitative measurements, and prioritizing critical cases.

The WIA-AI_DIAGNOSIS standard establishes requirements for radiology AI systems that ensure clinical reliability, workflow integration, and regulatory compliance. This chapter explores the technical foundations of these systems and provides implementation guidance for organizations deploying radiology AI.

Convolutional Neural Networks (CNNs)

    
CNNs form the backbone of most medical image analysis systems. Their hierarchical feature learning capabilities enable automatic extraction of diagnostically relevant patterns from raw pixel data:


    
// Medical Imaging CNN Architecture Configuration
class RadiologyModelArchitecture {
    constructor(config) {
        this.inputShape = config.inputShape;     // e.g., [512, 512, 1] for grayscale
        this.numClasses = config.numClasses;
        this.backboneType = config.backbone;
    }

    buildModel() {
        // Use transfer learning from ImageNet-pretrained backbone
        const backbone = this.loadBackbone(this.backboneType);

        // Adapt for medical imaging
        const model = new Sequential([
            // Input preprocessing for medical images
            new PreprocessingLayer({
                normalization: 'per-image-standardization',
                intensityWindowing: { center: 40, width: 400 } // CT soft tissue
            }),

            // Pretrained backbone with modifications
            backbone.withModifications({
                inputChannels: this.inputShape[2],
                freezeLayers: 'first-50%',
                dropoutRate: 0.3
            }),

            // Custom classification head for medical diagnosis
            new GlobalAveragePooling2D(),
            new Dense(512, { activation: 'relu' }),
            new BatchNormalization(),
            new Dropout(0.5),
            new Dense(256, { activation: 'relu' }),
            new Dense(this.numClasses, { activation: 'sigmoid' }) // Multi-label
        ]);

        return model;
    }

    loadBackbone(backboneType) {
        const backbones = {
            'densenet121': DenseNet121,
            'resnet50': ResNet50,
            'efficientnet-b4': EfficientNetB4,
            'convnext': ConvNeXt
        };

        return new backbones[backboneType]({
            weights: 'imagenet',
            includeTop: false,
            inputShape: this.inputShape
        });
    }
}

Computer-Aided Detection (CADe)

    
CADe systems automatically identify regions of interest in medical images that warrant physician attention. These systems serve as a "second reader" to help radiologists avoid overlooking subtle findings:


    
// Computer-Aided Detection Pipeline
class CADeSystem {
    constructor(config) {
        this.detectionModel = this.loadDetectionModel(config.modelPath);
        this.sensitivityThreshold = config.sensitivityThreshold || 0.3;
        this.nmsThreshold = config.nmsThreshold || 0.4;
        this.maxDetections = config.maxDetections || 100;
    }

    async detectFindings(medicalImage) {
        // Preprocess image for detection
        const preprocessed = await this.preprocess(medicalImage);

        // Run detection model (e.g., YOLO, Faster R-CNN adapted for medical imaging)
        const rawDetections = await this.detectionModel.predict(preprocessed);

        // Apply non-maximum suppression to remove duplicate detections
        const filteredDetections = this.applyNMS(rawDetections, this.nmsThreshold);

        // Filter by confidence threshold
        const significantFindings = filteredDetections.filter(
            det => det.confidence >= this.sensitivityThreshold
        );

        // Enrich detections with additional analysis
        const enrichedFindings = await Promise.all(
            significantFindings.map(async (finding) => ({
                ...finding,
                anatomicalLocation: await this.locateAnatomically(finding, medicalImage),
                measurements: await this.computeMeasurements(finding, medicalImage),
                characteristics: await this.analyzeCharacteristics(finding, medicalImage)
            }))
        );

        // Sort by clinical priority
        return this.prioritizeFindings(enrichedFindings);
    }

    prioritizeFindings(findings) {
        // Priority scoring based on finding type, size, and confidence
        return findings.sort((a, b) => {
            const priorityA = this.computePriority(a);
            const priorityB = this.computePriority(b);
            return priorityB - priorityA;
        });
    }

    computePriority(finding) {
        let priority = finding.confidence * 100;

        // Increase priority for potentially critical findings
        const criticalTypes = ['mass', 'nodule', 'hemorrhage', 'pneumothorax'];
        if (criticalTypes.includes(finding.type)) {
            priority += 50;
        }

        // Size-based adjustment
        if (finding.measurements && finding.measurements.maxDiameter > 30) {
            priority += 25;
        }

        return priority;
    }
}
