================================================================================
제8장: 재보존 프로토콜
================================================================================

[목차]

--------------------------------------------------------------------------------
8.1 실패 인식
--------------------------------------------------------------------------------

소생을 성공적으로 완료할 수 없을 때, 향후 발전된 기술로
소생을 다시 시도할 수 있도록 재보존을 고려할 수 있습니다.

[ID] 소생 실패 지표:

    FailureIndicators {
      cardiac: [
        "최대 중재에도 불구하고 불응성 심정지",
        "4시간 이상 후에도 안정적 리듬 달성 불가",
        "ECMO에 반응하지 않는 심인성 쇼크",
        "심장 조직의 비가역적 손상"
      ]

      neurological: [
        "72시간 이상 뇌 전기 활동 없음",
        "48시간 이상 뇌간 반사 없음",
        "탈출을 동반한 미만성 뇌부종",
        "영상에서 대량의 뇌조직 손실"
      ]

      multiOrgan: [
        "3개 이상 장기 시스템 부전",
        "불응성 대사성 산증 (pH < 7.0)",
        "어떤 혈역학적 지원도 유지 불가"
      ]
    }

[ID] 실패 vs 지연 회복:

    +------------------------------------------------------------------+
    | 소견                      | 회복 가능       | 실패 가능성        |
    +------------------------------------------------------------------+
    | 평탄 EEG                  | < 48시간        | > 72시간           |
    | 동공 반응 없음            | < 24시간        | > 48시간           |
    | 심장 리듬 없음            | < 4시간         | > 6시간            |
    | 불응성 산증               | 개선 중         | 악화 중            |
    | 다장기 부전               | 2개 장기        | 3개 이상 장기      |
    +------------------------------------------------------------------+

[!] 중요: 실패를 조기에 선언하지 마세요. 일부 대상자는
    지연 회복이 있을 수 있습니다. 재보존 고려를 시작하기 전에
    전체 팀과 상담하세요.


--------------------------------------------------------------------------------
8.2 결정 기준
--------------------------------------------------------------------------------

재보존 결정은 엄격한 기준과 팀 합의를 따릅니다.

[ID] 재보존 트리거:

    RePreservationTriggers {
      IRREVERSIBLE_CARDIAC_FAILURE: {
        criteria: [
          "다수의 심율동전환/제세동 실패",
          "최대 승압제 및 강심제 지원",
          "ECMO 실패 또는 불가능",
          "비가역성에 대한 심장학 합의"
        ]
      }

      IRREVERSIBLE_BRAIN_DAMAGE: {
        criteria: [
          "72시간 초과 EEG 활동 없음",
          "48시간 초과 뇌간 반사 없음",
          "영상에서 광범위한 손상 확인",
          "비가역성에 대한 신경과 합의"
        ]
      }

      MULTIPLE_ORGAN_FAILURE: {
        criteria: [
          "3개 이상 장기 시스템 부전",
          "모든 시스템에 대한 최대 지원",
          "24시간 동안 개선 경향 없음"
        ]
      }

      SUBJECT_REQUEST: {
        criteria: [
          "대상자가 능력을 회복함",
          "대상자가 소생 계속을 명시적으로 거부",
          "증인과 함께 거부 문서화"
        ]
      }

      GUARDIAN_REQUEST: {
        criteria: [
          "CRYO-CONSENT 문서 조항에 따라",
          "투표 규칙에 따른 후견인 합의",
          "의료팀과의 정보에 입각한 상담 후"
        ]
      }
    }

[ID] 표준 기준 평가:

    CriteriaEvaluation {
      CARDIAC_IRREVERSIBILITY: {
        weight:       0.25
        description:  "심장 기능을 복원할 수 없음"
        assessment:   "다수의 시도 실패, 지원에 불응"
      }

      NEUROLOGICAL_IRREVERSIBILITY: {
        weight:       0.30
        description:  "비가역적 뇌 손상의 증거"
        assessment:   "반사 없음, 평탄 EEG, 영상 소견"
      }

      MULTI_ORGAN_FAILURE: {
        weight:       0.20
        description:  "3개 이상 장기 부전"
        assessment:   "최대 지원에도 신장, 간, 호흡기 부전"
      }

      TIME_THRESHOLD: {
        weight:       0.10
        description:  "최대 소생 시도 기간 초과"
        assessment:   "개선 없이 72시간 초과 소생 시도"
      }

      TISSUE_VIABILITY: {
        weight:       0.15
        description:  "재보존을 위해 조직이 생존 가능한 상태로 유지"
        assessment:   "관류 유지, 제한된 허혈성 손상"
      }
    }

[ID] 점수 산정:

    기준 점수 = 모든 기준에 대한 (가중치 x 충족) 합계

    점수 >= 0.70 AND 팀 투표 > 50% 진행 -> 재보존
    점수 <  0.30 OR 팀 투표 < 30% 진행  -> 계속
    그 외                               -> 검토 (에스컬레이션)


--------------------------------------------------------------------------------
8.3 팀 투표
--------------------------------------------------------------------------------

재보존에는 공식적인 팀 투표가 필요합니다.

[ID] 투표 참가자:

    VotingTeam {
      required: [
        "소생 책임자",
        "냉동보존 전문의",
        "심혈관 외과의",
        "신경과 전문의"
      ]

      optional: [
        "마취과 전문의",
        "윤리 자문",
        "후견인 대표"
      ]

      minimumVoters: 3
    }

[ID] 투표 과정:

    VotingProcess {
      step1: "책임자가 사례 요약 발표"
      step2: "모든 기준 평가 검토"
      step3: "공개 토론 (제한 시간: 30분)"
      step4: "각 투표자가 근거와 함께 투표"
      step5: "투표 집계 및 문서화"
      step6: "결정 발표 및 기록"
    }

[ID] 투표 옵션:

    PROCEED (진행):  "재보존이 이루어져야 한다는 데 동의"
    OPPOSE (반대):   "소생을 계속해야 한다고 믿음"
    ABSTAIN (기권):  "투표하지 않음 (백분율에서 제외)"

[ID] 투표 기록 예시:

    VoteRecord {
      votes: [
        {
          member:    "박정호 박사 (소생 책임자)"
          role:      "REVIVAL_DIRECTOR"
          vote:      "PROCEED"
          rationale: "80시간 후에도 신경학적 회복 없음"
        },
        {
          member:    "김냉동 박사 (냉동보존 전문의)"
          role:      "CRYONICS_SPECIALIST"
          vote:      "PROCEED"
          rationale: "보존을 위해 조직이 여전히 생존 가능"
        },
        {
          member:    "이신경 박사 (신경과)"
          role:      "NEUROLOGIST"
          vote:      "PROCEED"
          rationale: "EEG 평탄, 뇌간 반사 없음"
        },
        {
          member:    "최심장 박사 (심혈관)"
          role:      "CARDIOVASCULAR_SURGEON"
          vote:      "OPPOSE"
          rationale: "심장 기능이 약간 개선 중"
        }
      ]

      summary: {
        PROCEED: 3
        OPPOSE:  1
        ABSTAIN: 0
      }

      decision: "재보존 (75% 진행)"
    }


--------------------------------------------------------------------------------
8.4 재보존 실행
--------------------------------------------------------------------------------

재보존이 결정되면, 미래 생존 가능성을 극대화하기 위해
신중하게 실행합니다.

[ID] 후견인 통지:

    GuardianNotification {
      timing:      "실행 시작 전"
      method:      "직접 전화 + 보안 메시지"

      content: {
        situation:    "소생 실패"
        decision:     "재보존 권장"
        rationale:    "[의학적 소견 요약]"
        nextSteps:    "재보존 및 저장소로 복귀"
        futurePlans:  "미래 시도를 위한 기술 발전 대기"
      }

      guardianResponse: {
        options:    ["수락", "시간 요청", "거부"]
        timeLimit:  "응답까지 2시간"
        default:    "응답 없으면 동의 문서에 따라 진행"
      }
    }

[ID] 재보존 프로토콜:

    RePreservationExecution {
      preparation: {
        team:           "동일 팀 또는 전담 보존 팀"
        equipment:      "냉각, 관류, 동결보호제 준비"
        documentation:  "전체 절차 기록"
      }

      cooling: {
        method:         "제어된 냉각 관류"
        rate:           "0C까지 분당 1-2 C, 이후 더 느리게"
        targetTemp:     "동결보호제 로딩을 위해 -10C"
      }

      cryoprotectantLoading: {
        solution:       "표준 유리화 용액"
        concentration:  "60%까지 점진적 증가"
        duration:       "4-6시간"
      }

      finalCooling: {
        method:         "제어된 속도 냉각"
        rate:           "분당 0.5 C"
        targetTemp:     "-196C (액체질소)"
      }

      storage: {
        facility:       "원래 또는 지정된 시설로 복귀"
        container:      "장기 저장 듀어"
        monitoring:     "표준 보존 모니터링"
      }
    }

[ID] 문서화:

    RePreservationRecord {
      procedureId:       "REVIVAL-2075-001"
      originalRevival:   "2075-06-15"
      rePreservation:    "2075-06-18"

      revivalDuration:   "72시간"
      failureReason:     "NEUROLOGICAL_IRREVERSIBILITY"

      criteriaScore:     0.75
      teamVote:          "3-1 진행"

      rePreservation: {
        coolingStarted:  "2075-06-18T14:00:00Z"
        coolingComplete: "2075-06-18T22:00:00Z"
        cryoprotectant:  "VS-55 유도체"
        finalTemp:       "-196C"
        storageLocation: "WIA 서울 냉동보존 센터"
      }

      notes: "대상자가 성공적으로 재보존됨. 두 번째 소생 시도를 위한
              미래 기술 발전 대기."
    }

|                                                                            |
|  "홍익인간" - 널리 인간을 이롭게 하라                                      |
|                                                                            |
|  재보존은 실패가 아닌 희망을 나타냅니다. 현재 기술이 소생을 완료할 수      |
|  없을 때, 보존은 대상자에게 미래에 또 다른 기회를 제공합니다. 이 표준은    |
|  결정이 적절한 동의와 함께 신중하게 이루어지고, 재보존이 미래 성공         |
|  가능성을 극대화하도록 실행되도록 보장합니다.                              |

--------------------------------------------------------------------------------
장 요약
--------------------------------------------------------------------------------

핵심 개념:
  [OK] 실패 지표는 심장, 신경학적, 다장기 기준 포함
  [OK] 재보존 트리거는 특정 문서화된 소견 필요
  [OK] 결정을 위해 최소 3명의 투표자로 팀 투표 필요
  [OK] 기준 점수 >= 0.70 AND 다수 투표로 재보존 시작
  [OK] 실행 전 후견인 통지
  [OK] 재보존 프로토콜은 미래 소생을 위해 조직 최적화
