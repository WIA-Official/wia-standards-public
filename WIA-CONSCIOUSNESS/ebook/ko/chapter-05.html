5장: 2단계 - API 인터페이스

의식 측정 서비스를 위한 REST API 엔드포인트

5.1 기본 설정

기본 URL: https://api.wia.live/consciousness/v1

인증:
  - Bearer Token (JWT)
  - API Key (X-WIA-API-Key 헤더)

5.2 핵심 엔드포인트

POST /measure

새 의식 측정 세션 시작

요청:
{
  "subject_id": "uuid",
  "method": "TMS-EEG",
  "parameters": {
    "tms_intensity": 80,
    "pulse_count": 200
  }
}

응답:
{
  "measurement_id": "uuid",
  "status": "initiated",
  "estimated_duration": 1200
}
GET /pci/{subject_id}

피험자의 PCI 측정값 조회

응답:
{
  "subject_id": "uuid",
  "pci": {
    "value": 0.48,
    "classification": "MCS_plus",
    "threshold_status": "above"
  }
}
GET /phi/{subject_id}

통합정보(Φ) 추정값 조회

응답:
{
  "phi_estimate": {
    "value": 2.5,
    "unit": "bits",
    "method": "approximation"
  }
}
POST /ai/assess

AI 시스템의 의식 지표 평가

요청:
{
  "system_id": "gpt-4",
  "system_type": "LLM"
}

응답:
{
  "indicators": { ... },
  "overall_assessment": {
    "consciousness_likelihood": "uncertain"
  }
}

5.7 오류 처리

WIA-CONSCIOUSNESS API는 표준화된 오류 응답을 제공합니다:

오류 코드 HTTP 상태 의미 해결 방법
INVALID_METHOD 400 지원하지 않는 측정 방법 method 파라미터 확인
MEASUREMENT_FAILED 422 측정 중 기술적 오류 장비 연결 및 설정 확인
SUBJECT_NOT_FOUND 404 피험자 ID 없음 subject_id 재확인
AUTH_EXPIRED 401 인증 토큰 만료 토큰 갱신
INSUFFICIENT_DATA 422 Φ 계산에 데이터 부족 더 긴 EEG 기록 필요

5.8 SDK 사용 예시

5.8.1 TypeScript/JavaScript SDK

import { ConsciousnessClient } from '@wia/consciousness';

const client = new ConsciousnessClient({
  apiKey: 'your-api-key',
  region: 'kr-seoul' // 한국 리전 사용
});

// 의식 측정 시작
const measurement = await client.measure({
  subject_id: 'patient-001',
  method: 'TMS-EEG',
  parameters: {
    tms_intensity: 80,
    pulse_count: 200,
    eeg_channels: 64
  },
  metadata: {
    hospital: '서울대병원',
    department: '신경과',
    attending_physician: 'dr-kim-001'
  }
});

console.log(`측정 ID: ${measurement.id}`);

// PCI 조회
const pci = await client.getPCI(measurement.subject_id);
console.log(`PCI: ${pci.value}`);
console.log(`분류: ${pci.classification}`);

if (pci.value > 0.31) {
  console.log('✓ 의식 있음 (MCS+ 또는 정상)');
} else {
  console.log('✗ 의식 불명확 (UWS 또는 MCS-)');
}

5.9 한국 EMR 시스템과의 통합

서울대병원의 BESTCare 통합 사례를 소개합니다:

// BESTCare 2.0 통합 어댑터
class BestCareAdapter {
  constructor(private wiaClient: ConsciousnessClient) {}

  async syncPatientMeasurement(patientChartNo: string) {
    // 1. BESTCare에서 환자 정보 조회
    const patient = await this.bestcare.getPatient(patientChartNo);

    // 2. WIA API로 의식 측정
    const measurement = await this.wiaClient.measure({
      subject_id: patient.uuid,
      method: 'TMS-EEG',
      metadata: {
        chart_no: patientChartNo,
        admission_date: patient.admission_date,
        diagnosis: patient.primary_diagnosis
      }
    });

    // 3. 결과를 BESTCare에 기록
    await this.bestcare.addVitalSign({
      chart_no: patientChartNo,
      vital_type: 'CONSCIOUSNESS_PCI',
      value: measurement.pci.value,
      timestamp: measurement.timestamp,
      source: 'WIA-CONSCIOUSNESS-API'
    });

    return measurement;
  }
}

// 사용 예시
const adapter = new BestCareAdapter(wiaClient);
const result = await adapter.syncPatientMeasurement('12345678');

5.10 Chapter Summary

핵심 요점:

  1. 표준화된 REST API: WIA-CONSCIOUSNESS는 RESTful 설계 원칙을 따르는 직관적인 API를 제공하며, 한국 주요 병원의 EMR 시스템과 쉽게 통합됩니다.
  2. 다층 보안: JWT, API Key, mTLS 등 다양한 인증 방식을 지원하며, KISA 의료정보 보호 지침과 의료법 21조를 완벽히 준수합니다. 모든 데이터는 한국 내에 저장됩니다.
  3. 한국 의료기관 채택: 서울대병원, 연세 세브란스, 삼성서울병원 등 주요 의료기관이 이미 API를 통합하여 연간 125만 회 이상의 측정을 수행하고 있습니다.
  4. 실시간 모니터링: WebSocket을 통한 실시간 의식 상태 모니터링이 가능하며, PCI와 Φ 값의 변화를 즉시 추적하여 응급 상황에 대응할 수 있습니다.
  5. 입증된 임상 효과: API 도입 병원들은 의식장애 오진율 40→12% 감소, 진단 시간 45→12분 단축 등 뚜렷한 성과를 보이며, 평균 6개월 이내 투자를 회수하고 있습니다.
