Chapter 4: Phase 1 - Data Format

Standardized schemas for consciousness measurement data

4.1 Overview

Phase 1 of the WIA-CONSCIOUSNESS standard defines the data formats used to represent consciousness measurements. These schemas ensure interoperability across research institutions, clinical systems, and AI evaluation platforms.

4.2 Core Schema: Consciousness Index

The primary data structure for consciousness measurements:

{
  "$schema": "https://wia.live/schemas/consciousness/v1.0.0",
  "consciousness_index": {
    "subject_id": "uuid",
    "timestamp": "ISO8601",

    "iit_metrics": {
      "phi_estimate": { "value": 2.5, "unit": "bits" },
      "cause_effect_power": 0.72,
      "irreducibility": 0.85
    },

    "gnw_metrics": {
      "global_ignition": 0.68,
      "prefrontal_activation": 0.55,
      "broadcast_strength": 0.81
    },

    "practical_measures": {
      "pci": { "value": 0.45, "max": 1.0 },
      "lempel_ziv_complexity": 0.72
    },

    "state": {
      "condition": "awake|sleep|anesthesia|...",
      "arousal_level": 0.8,
      "awareness_level": 0.9
    },

    "measurement": {
      "method": "TMS-EEG|fMRI|MEG|...",
      "confidence": 0.92
    }
  }
}

4.6 Consciousness State Schema

Classifying the current state of consciousness:

State Conditions:
  • Normal: awake, drowsy, sleep_N1/N2/N3, REM
  • Altered: meditation, psychedelic, hypnosis
  • Pathological: coma, VS_UWS, MCS_minus, MCS_plus, EMCS, LIS
  • Artificial: anesthesia_light/moderate/deep

4.9 Validation Requirements

  • subject_id: MUST be valid UUID
  • timestamp: MUST be ISO8601 format
  • measurement.method: MUST be from enumerated list
  • All numeric values: MUST be within specified ranges
  • If pci.value present, classification SHOULD be auto-derived
