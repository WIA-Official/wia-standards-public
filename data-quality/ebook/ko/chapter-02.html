데이터 품질 차원 이해하기

데이터 품질은 단일 지표로 측정할 수 없는 다면적 개념입니다. 업계는 데이터 품질에 대한 포괄적인 관점을 제공하는 6가지 기본 차원에 수렴했습니다. 각 차원은 데이터를 의도된 목적에 적합하게 만드는 특정 측면을 다룹니다.

"측정할 수 없으면 관리할 수 없습니다. 데이터 품질의 6가지 차원은 데이터 자산의 품질을 측정하고 관리하기 위한 프레임워크를 제공합니다."
🎯

1. 정확성 (Accuracy)

정의: 데이터가 설명하는 실제 엔터티나 이벤트를 올바르게 나타내는 정도

왜 중요한가

부정확한 데이터는 잘못된 결론과 결함 있는 의사결정으로 이어집니다. 고객 주소가 잘못되면 배송이 잘못된 장소로 갑니다. 재무 수치가 부정확하면 예산과 예측이 결함이 있습니다.

측정 방법

  • 직접 검증: 권위 있는 출처와 비교 (예: 우편 데이터베이스와 주소 확인)
  • 통계 분석: 부정확한 값을 시사하는 이상치와 이상 징후 식별
  • 비즈니스 규칙: 데이터가 알려진 비즈니스 제약 조건을 준수하는지 확인
  • 수동 샘플링: 레코드를 무작위로 샘플링하고 사람의 검토를 통해 정확성 확인

일반적인 정확성 문제

  • 오타 및 데이터 입력 오류
  • 한때 정확했지만 구식이 된 정보
  • 센서나 기기의 측정 오류
  • 파생 필드의 계산 오류
  • 데이터 통합 중 잘못된 매핑
✔️

2. 완전성 (Completeness)

정의: 필요한 모든 데이터가 존재하는 정도

왜 중요한가

누락된 데이터는 의도된 목적으로 레코드를 사용할 수 없게 만들 수 있습니다. 불완전한 고객 프로필은 마케팅 캠페인 발송을 방해할 수 있습니다. 누락된 재무 데이터는 보고 및 규정 준수 활동을 차단할 수 있습니다.

완전성 유형

  • 컬럼 완전성: 컬럼의 null이 아닌 값 비율
  • 행 완전성: 모든 필수 필드가 채워진 행 비율
  • 데이터셋 완전성: 예상되는 모든 레코드가 존재하는지 여부
  • 시간적 완전성: 데이터가 모든 필요한 기간을 포함하는지 여부

측정 방법

  • 각 필드의 null/누락 값 비율 계산
  • 예상 볼륨과 레코드 수 비교
  • 시계열 데이터의 간격 확인
  • 외래 키 관계의 완전성 확인
🔄

3. 일관성 (Consistency)

정의: 데이터 값이 다른 시스템 간 및 시간에 걸쳐 균일한 정도

왜 중요한가

일관성 없는 데이터는 혼란을 초래하고 단일 진실 공급원을 확립하는 것을 불가능하게 합니다. 동일한 고객이 다른 시스템에서 다른 주소를 가질 때, 어느 것이 올바른가요?

일관성 유형

  • 교차 시스템 일관성: 동일한 엔터티가 시스템 전반에 동일하게 표현됨
  • 시간적 일관성: 과거 데이터가 현재 데이터와 모순되지 않음
  • 형식 일관성: 동일한 데이터 유형이 일관된 형식을 사용
  • 의미적 일관성: 동일한 비즈니스 용어가 일관된 의미를 가짐
⏱️

4. 적시성 (Timeliness)

정의: 데이터가 필요할 때 사용 가능한지와 데이터가 얼마나 최신인지를 측정

왜 중요한가

구식 데이터는 부정확한 데이터만큼 문제가 될 수 있습니다. 실시간 비즈니스 의사결정은 현재 데이터가 필요합니다. 오늘의 이행 결정에 지난달 재고 데이터를 사용하면 오류가 발생합니다.

적시성 측면

  • 통용성: 데이터가 얼마나 최근인가?
  • 지연 시간: 이벤트 발생과 데이터 가용성 사이의 시간은?
  • 신선도: 데이터가 사용 사례에 충분히 자주 업데이트되는가?
  • 나이: 데이터의 유효 기간과 비교한 나이는?
✅

5. 유효성 (Validity)

정의: 데이터가 정의된 형식, 유형 및 비즈니스 규칙을 준수하는 정도

왜 중요한가

유효하지 않은 데이터는 시스템과 프로세스를 중단시킬 수 있습니다. 잘못된 자릿수의 신용카드 번호, @가 없는 이메일 또는 불가능한 형식의 날짜는 모두 유효성 문제를 나타냅니다.

유효성 규칙 유형

  • 형식 유효성: 데이터가 예상 패턴과 일치 (이메일, 전화, 우편번호)
  • 유형 유효성: 데이터가 올바른 데이터 유형 (숫자 vs 문자열)
  • 범위 유효성: 숫자 값이 허용 범위 내에 있음
  • 도메인 유효성: 값이 허용된 옵션 세트에서 나옴
  • 비즈니스 규칙 유효성: 데이터가 비즈니스 논리 제약 조건을 만족
🔑

6. 고유성 (Uniqueness)

정의: 각 실제 엔터티가 데이터셋에서 한 번만 표현되도록 보장

왜 중요한가

중복 레코드는 부풀려진 카운트, 낭비된 마케팅 지출(동일한 캠페인을 여러 번 발송), 운영 비효율성 및 고객 불만(동일한 사람에 대한 여러 계정)으로 이어집니다.

중복 유형

  • 정확한 중복: 모든 필드가 정확히 일치
  • 근사 중복: 사소한 변형 (오타, 형식)
  • 의미적 중복: 동일한 엔터티의 다른 표현

중복 탐지

  • 고유 식별자의 정확한 일치 (이메일, 계정 번호)
  • 이름과 주소의 퍼지 매칭
  • 음성 매칭 (Soundex, Metaphone)
  • 머신러닝 기반 엔터티 해결

차원 간 균형 맞추기

이러한 차원들은 상호 연관되어 있으며 때로는 절충을 수반합니다:

  • 적시성 vs 정확성: 실시간 데이터는 검증이 포함된 배치 처리 데이터보다 덜 정확할 수 있음
  • 완전성 vs 유효성: 모든 필드를 요구하는 것은 비현실적일 수 있음; 중요한 필드 우선순위 지정
  • 정확성 vs 비용: 더 높은 정확성은 종종 더 비싼 검증 및 확인 프로세스를 요구

핵심은 모든 차원에서 완벽을 추구하는 것이 아니라 각 사용 사례와 차원에 적절한 품질 임계값을 설정하는 것입니다.

핵심 요점

  • 데이터 품질은 다차원적—단일 지표로 전체 품질을 포착할 수 없음
  • 각 차원은 특정 측면 다룸: 정확성, 완전성, 일관성, 적시성, 유효성, 고유성
  • 차원은 상호 연관되어 있으며 절충을 수반할 수 있음
  • 품질 임계값은 비즈니스 요구사항과 사용 사례에 기반해야 함
  • 정기적인 측정과 스코어카드가 시간에 따른 품질 추적과 개선에 도움
  • 다른 데이터셋은 목적에 따라 다른 차원을 우선시할 수 있음
