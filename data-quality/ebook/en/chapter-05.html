What is Data Cleansing?

Data cleansing (also called data cleaning or scrubbing) is the process of detecting and correcting inaccurate, incomplete, or irrelevant data. While validation prevents bad data from entering, cleansing fixes data that's already problematic.

Common Data Quality Issues

1. Missing Values

Handling nulls and empty fields:

  • Remove: Delete records with critical missing data
  • Impute: Fill with mean, median, mode, or predicted values
  • Flag: Keep record but mark field as incomplete
  • Default: Use default value (e.g., "Unknown", 0)
# Python example - Missing value handling
import pandas as pd

df = pd.read_csv('customers.csv')

# Remove rows with missing critical fields
df_clean = df.dropna(subset=['customer_id', 'email'])

# Impute missing age with median
df['age'].fillna(df['age'].median(), inplace=True)

# Fill missing category with 'Unknown'
df['category'].fillna('Unknown', inplace=True)

2. Duplicate Records

Strategies for deduplication:

# Exact duplicates
df_deduped = df.drop_duplicates(subset=['email'], keep='first')

# Fuzzy matching for near duplicates
from fuzzywuzzy import fuzz

def find_similar_names(name1, name2, threshold=85):
    return fuzz.ratio(name1, name2) >= threshold

3. Formatting Issues

Standardizing data formats:

# Standardize case
df['email'] = df['email'].str.lower()
df['name'] = df['name'].str.title()

# Remove whitespace
df['phone'] = df['phone'].str.strip().str.replace(' ', '')

# Standardize dates
df['date'] = pd.to_datetime(df['date'], format='%Y-%m-%d')

4. Outliers and Anomalies

Detecting and handling outliers:

# Z-score method
from scipy import stats
import numpy as np

z_scores = np.abs(stats.zscore(df['price']))
df_no_outliers = df[z_scores < 3]

# IQR method
Q1 = df['price'].quantile(0.25)
Q3 = df['price'].quantile(0.75)
IQR = Q3 - Q1
df_filtered = df[(df['price'] >= Q1 - 1.5*IQR) & (df['price'] <= Q3 + 1.5*IQR)]

Data Cleansing Workflow

  1. Audit: Profile data to identify issues
  2. Define Rules: Establish cleansing standards
  3. Backup: Create snapshot before cleansing
  4. Execute: Run automated cleansing scripts
  5. Review: Sample and verify results
  6. Validate: Run quality checks
  7. Document: Record changes made
  8. Monitor: Track ongoing quality

Best Practices

  • Always backup before cleansing
  • Document all transformations
  • Test on sample before full dataset
  • Track cleansing metrics and results
  • Implement cleansing in pipelines, not ad-hoc
  • Prefer prevention over correction

Key Takeaways

  • Cleansing fixes existing data quality issues
  • Common issues: missing values, duplicates, format inconsistencies, outliers
  • Balance automated cleansing with manual review
  • Follow systematic workflow: audit, define, execute, validate, monitor
  • Prevention is better than cleansing—fix root causes
