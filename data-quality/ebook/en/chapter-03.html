Data Profiling and Discovery

What is Data Profiling?

Data profiling is the process of examining, analyzing, and summarizing datasets to understand their structure, content, quality, and relationships. It's like taking an X-ray of your data—revealing patterns, anomalies, and issues that aren't immediately visible.

Data profiling is typically the first step in any data quality initiative. You can't improve what you don't understand, and profiling provides the insights needed to make informed decisions about data quality improvements.

Why Profile Data?

Data profiling serves multiple critical purposes:

  • Discovery: Understand what data you actually have (vs what you think you have)
  • Quality Assessment: Identify completeness, accuracy, and consistency issues
  • Anomaly Detection: Find outliers, unexpected patterns, and data corruption
  • Metadata Creation: Document data structures, formats, and business rules
  • Impact Analysis: Understand dependencies before making changes
  • Compliance: Identify sensitive data requiring protection (PII, PHI, PCI)

Types of Data Profiling

1. Column Profiling

Column profiling analyzes individual columns to understand their characteristics:

Key Metrics:

  • Data Type: String, integer, decimal, date, boolean
  • Null Count/Percentage: How many values are missing
  • Distinct Values: Number of unique values (cardinality)
  • Min/Max Values: Range of values for numeric/date fields
  • Average/Median: Central tendency for numeric fields
  • Standard Deviation: Measure of dispersion
  • Pattern Analysis: Common formats and structures
  • Value Distribution: Frequency of each value
Example: Email Column Profile
Column: email
Data Type: VARCHAR(255)
Total Rows: 50,000
Non-Null: 47,500 (95%)
Null: 2,500 (5%)
Distinct Values: 46,800
Duplicate Values: 700 (1.4%)

Top Patterns:
  *@gmail.com: 15,000 (31.6%)
  *@yahoo.com: 8,500 (17.9%)
  *@outlook.com: 5,200 (10.9%)

Invalid Formats: 250 (0.5%)
  - Missing @: 120
  - Missing domain: 80
  - Invalid characters: 50

2. Row Profiling

Row profiling examines records holistically to understand completeness and quality at the record level:

  • Count of populated fields per row
  • Identification of fully empty or duplicate rows
  • Cross-field validation (e.g., if country="USA", state must be valid US state)
  • Record-level quality scores

3. Relationship Profiling

Relationship profiling discovers and validates connections between tables and columns:

  • Foreign Key Detection: Discover referential relationships
  • Orphan Detection: Find child records without parents
  • Dependency Analysis: Identify functional dependencies
  • Correlation Analysis: Find statistically correlated columns
Example: Foreign Key Validation
Table: orders
Foreign Key: customer_id → customers.id

Total Orders: 100,000
Valid References: 98,500 (98.5%)
Orphan Records: 1,500 (1.5%)

Analysis:
- 1,500 orders reference customer_ids that don't exist
- Likely caused by customer deletions without cascading
- Action: Implement referential integrity constraints

The Profiling Process

Step 1: Scope Definition

Define what you're profiling and why:

  • Which databases, schemas, and tables?
  • What are your quality objectives?
  • Who are the stakeholders and what do they need to know?
  • What is the timeframe and resource budget?

Step 2: Automated Discovery

Run automated profiling tools to gather statistics:

# Example using Python pandas
import pandas as pd

df = pd.read_csv('customers.csv')

# Basic profiling
profile = {
    'shape': df.shape,
    'columns': df.dtypes.to_dict(),
    'null_counts': df.isnull().sum().to_dict(),
    'null_percentages': (df.isnull().sum() / len(df) * 100).to_dict(),
    'unique_counts': df.nunique().to_dict(),
}

# Numeric column statistics
numeric_stats = df.describe().to_dict()

# Value distributions
for col in df.columns:
    value_counts = df[col].value_counts()
    print(f"\n{col} - Top 10 values:")
    print(value_counts.head(10))

Step 3: Pattern Recognition

Identify common patterns, formats, and structures:

  • Email formats, phone number formats
  • Naming conventions (camelCase, snake_case)
  • Code structures (customer IDs, product codes)
  • Date/time formats

Step 4: Anomaly Detection

Look for outliers and unexpected values:

  • Values outside expected ranges
  • Unusual patterns or formats
  • Statistically rare values
  • Suspicious duplicates

Step 5: Business Rule Validation

Check data against known business rules:

  • Age should be 0-120
  • Order date should be <= delivery date
  • Profit = Revenue - Cost
  • Status codes must be from valid list

Step 6: Documentation and Reporting

Create comprehensive reports and metadata documentation:

  • Data catalog with column descriptions
  • Quality scorecards
  • Issue tracking and prioritization
  • Recommendations for improvement

Advanced Profiling Techniques

Statistical Profiling

Apply statistical methods to detect anomalies:

  • Z-score Analysis: Identify values beyond 3 standard deviations
  • Interquartile Range (IQR): Detect outliers using quartiles
  • Benford's Law: Detect fraud in financial data
  • Chi-square Tests: Test independence of categorical variables

Machine Learning for Profiling

Modern tools use ML to enhance profiling:

  • Clustering: Group similar records to find anomalous groups
  • Classification: Automatically classify data types and patterns
  • Anomaly Detection: Isolate unusual records automatically
  • Semantic Analysis: Understand column meaning from content

Continuous Profiling

Rather than one-time profiling, implement continuous monitoring:

  • Schedule automated profiling jobs (daily, weekly)
  • Track metrics over time to spot trends
  • Alert when metrics exceed thresholds
  • Build historical baselines for comparison

Common Pitfalls to Avoid

  • Profiling Production During Peak Hours: Can impact performance; use replicas or off-peak times
  • Ignoring Sample Size: Small samples may miss rare issues; balance performance vs completeness
  • Not Involving Business Users: Technical profiling alone misses business context
  • Profiling Once and Forgetting: Data quality degrades over time; profile regularly
  • Analysis Paralysis: Don't get stuck in endless profiling; act on findings

Key Takeaways

  • Data profiling is essential for understanding data structure, content, and quality
  • Three main types: column profiling, row profiling, and relationship profiling
  • Automated tools accelerate profiling but require business context for interpretation
  • Profiling reveals not just problems but also opportunities for data enhancement
  • Continuous profiling is more valuable than one-time exercises
  • Document findings and track metrics over time
  • Use profiling results to prioritize quality improvement efforts
