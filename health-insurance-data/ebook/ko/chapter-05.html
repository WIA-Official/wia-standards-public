제5장: 자격 검증 및 사전 승인

"진료 전 자격 검증은 환자와 제공자 모두를 보호합니다. 청구 거부를 예방하고, 환자의 예상 비용을 미리 알려주며, 제공자의 현금 흐름을 안정화합니다."

— 자격 검증의 중요성

5.1 실시간 자격 검증

자격 검증(Eligibility Verification)은 환자의 보험 상태와 혜택을 진료 전에 확인하는 프로세스입니다.

🎯 자격 검증의 필요성

문제 영향 해결책
비활성 보험 청구 거부, 환자 책임 진료 전 270/271 확인
보장 제외 서비스 지불 분쟁, 소송 혜택 상세 조회
사전 승인 미확인 청구 거부 278 사전 승인
잘못된 환자 정보 청구 지연 실시간 인구통계 업데이트

⚡ 270/271 트랜잭션 흐름

1. 제공자 → 270 조회 전송
   ↓
2. 보험사 실시간 처리 (< 5초)
   ↓
3. 271 응답 수신
   ↓
4. 자동 분석 및 표시
   - ✅ Active Coverage: 보험 활성
   - 💰 Copay: $25
   - 📊 Deductible: $500 남음 (총 $1,500)
   - 🏥 Prior Auth: MRI, CT 필요
   - 📅 Coverage Dates: 2025-01-01 ~ 2025-12-31

📋 확인해야 할 주요 정보

  • 보험 활성 상태
    • Active / Inactive / Pending
    • Effective Date (개시일)
    • Term Date (종료일)
  • 혜택 정보
    • Plan Type (HMO, PPO, EPO, POS)
    • In-Network vs Out-of-Network
    • Service-specific Coverage
  • 비용 분담
    • Copayment (본인부담금)
    • Coinsurance (공동보험)
    • Deductible (공제액)
    • Out-of-Pocket Max (본인부담 한도)
  • 제한사항
    • Prior Authorization Requirements
    • Referral Needed
    • Visit Limits (방문 횟수 제한)

💡 271 응답 예시 분석

EB*1*FAM**30~  → Active Coverage (1), Family (FAM)
DTP*307*RD8*20250101-20251231~  → 보장 기간

EB*B*FAM**30*$25~  → Copayment $25
EB*A*FAM**30*$1500~  → Deductible $1,500
EB*G*FAM**30*$500*MN*1~  → $500 남은 공제액, 올해 1개월 경과

EB*C*FAM**98*80-20~  → 외래 진료 공동보험 80-20
EB*1**27**30~  → Prior Auth 필요 (Service Code 27: MRI)

해석:
- 보험 활성 (2025년 12월 31일까지)
- 외래 본인부담금: $25
- 연간 공제액: $1,500 (아직 $500 남음)
- 외래 공동보험: 보험 80%, 환자 20%
- MRI는 사전 승인 필요

5.2 사전 승인 (Prior Authorization)

사전 승인은 특정 고가 또는 전문 서비스를 제공하기 전에 보험사의 승인을 받는 프로세스입니다.

🏥 사전 승인이 필요한 서비스

카테고리 서비스 예시 평균 승인 시간
영상 검사 MRI, CT, PET 스캔 1-3 영업일
수술 선택적 수술, 척추 수술 3-5 영업일
전문 약물 생물학적 제제, 항암제 2-7 영업일
재활 물리치료, 작업치료 1-2 영업일
의료 기기 휠체어, 인공호흡기, DME 2-5 영업일
입원 계획된 입원, 장기 입원 즉시 ~ 3일

📝 사전 승인 요청 프로세스

1단계: 의료 필요성 확립
   - 진단 코드 (ICD-10)
   - 증상 및 징후
   - 기존 치료 시도 및 실패
   - 임상 지침 준수

2단계: 278 요청 제출
   - 환자 정보
   - 제공자 정보
   - 서비스 상세 (CPT 코드)
   - 임상 정보
   - 지원 문서 첨부

3단계: 보험사 심사
   - 자동 승인 (규칙 기반)
   - 임상 심사 (간호사)
   - 의료 디렉터 심사 (의사)

4단계: 278 응답 수신
   - A1: Certified (승인)
   - A2: Modified (조건부)
   - A3: Denied (거부)
   - A4: Pended (보류)

5단계: 후속 조치
   - 승인 번호 기록
   - 환자 통지
   - 서비스 예약

🤖 자동화된 사전 승인 시스템

최신 보험사는 AI 기반 자동 승인 시스템을 구축하고 있습니다:

  • 규칙 엔진
    • 임상 지침 자동 매칭
    • Medical Necessity 기준 자동 평가
    • Step Therapy 준수 확인
  • 즉시 승인
    • 저위험 서비스 자동 승인 (< 1분)
    • 표준 프로토콜 일치 시 즉시 처리
  • 예측 분석
    • 승인 가능성 사전 예측
    • 누락 문서 자동 식별
    • 대안 치료 제안

💡 Medical Necessity 판단 기준

보험사는 다음 기준으로 의료 필요성을 평가합니다:

기준 설명
Appropriate 진단 및 치료에 적절한 서비스
Evidence-based 임상 증거 및 지침 기반
Not Experimental FDA 승인 또는 표준 치료
Cost-effective 동등 효과의 저렴한 대안 없음
Provider Qualified 자격 있는 제공자에 의한 서비스

5.3 환자 책임 추정

Patient Responsibility Estimation은 환자가 진료 전에 본인 부담 비용을 알 수 있도록 합니다.

💰 비용 추정 계산

예시: 외래 진료 + X-ray

서비스 비용:
1. 외래 진찰 (99213): $150
2. X-ray 2 View (73030): $80
총 청구액: $230

환자 보험 혜택:
- Deductible: $1,500 (이미 $1,200 충족, $300 남음)
- Copay: 외래 $25
- Coinsurance: 80-20 (보험 80%, 환자 20%)

계산:
1. Copay 적용: $25 (외래)
2. 남은 금액: $230 - $25 = $205
3. Deductible 적용: $205 (전액 공제액에 적용, $300 > $205)
4. 공제액 충족 후 남은 금액: $0
5. Coinsurance 적용: $0 × 20% = $0

총 환자 부담: $25 (copay) + $205 (deductible) = $230

만약 공제액이 이미 충족되었다면:
- Copay: $25
- Coinsurance: ($230 - $25) × 20% = $41
- 총 환자 부담: $25 + $41 = $66

📊 실시간 비용 투명성 도구

환자가 진료 전에 비용을 알 수 있도록 하는 시스템:

  • Price Transparency Tool
    • 서비스별 예상 비용
    • 제공자 간 가격 비교
    • Quality Ratings 통합
  • Out-of-Pocket Calculator
    • 현재 공제액 진행 상황
    • 연간 본인부담 한도 추적
    • 서비스 시나리오 비교
  • Good Faith Estimate
    • No Surprises Act (2022) 준수
    • 무보험/Out-of-Network 환자 보호
    • 진료 전 서면 추정 제공

5.4 Referral 관리

Referral (의뢰)은 HMO 및 일부 PPO 플랜에서 전문의 진료를 위해 필요합니다.

🔄 Referral 워크플로우

1. 환자가 Primary Care Physician (PCP) 방문
2. PCP가 전문의 진료 필요성 판단
3. Referral 생성:
   - Specialist Type (전문과)
   - Number of Visits (방문 횟수)
   - Valid Dates (유효 기간)
   - Diagnosis Code (진단 코드)
4. 전자 의뢰서 전송 (EDI 또는 Portal)
5. 보험사 승인 (자동 또는 수동)
6. 환자 및 전문의에게 통지
7. 전문의 진료
8. Referral 번호를 청구에 포함

📋 전자 Referral 표준

  • HL7 FHIR ServiceRequest
    • Referral 정보 표준 형식
    • 상호운용성 보장
    • EHR 통합 용이
  • EDI 278 통합
    • Referral = Simplified Prior Auth
    • 자동 승인 가능

자격 검증과 사전 승인의 자동화는 의료 시스템의 효율성을 극적으로 향상시킵니다. WIA-MED-028 표준은 이러한 프로세스를 간소화하고 표준화하여, 환자와 제공자 모두의 경험을 개선합니다.
