제3장: EDI X12 표준

"EDI X12는 의료 산업의 공통 언어입니다. 이를 통해 수천 개의 보험사, 병원, 의사가 하나의 표준화된 방식으로 소통하여, 청구 처리 시간을 몇 주에서 며칠로 단축했습니다."

— HIPAA EDI 표준의 영향

3.1 EDI X12란 무엇인가?

EDI (Electronic Data Interchange)는 컴퓨터 간 표준화된 형식으로 비즈니스 문서를 교환하는 방법입니다. X12는 미국 국가 표준 협회(ANSI)가 승인한 EDI 표준입니다.

🎯 HIPAA 필수 트랜잭션

1996년 HIPAA(Health Insurance Portability and Accountability Act)는 다음 트랜잭션을 표준화했습니다:

트랜잭션 이름 용도
837P/I/D Health Care Claim 청구서 제출
835 Payment/Remittance 지급 및 설명
270 Eligibility Inquiry 자격 조회
271 Eligibility Response 자격 응답
276 Claim Status Inquiry 청구 상태 조회
277 Claim Status Response 청구 상태 응답
278 Authorization 사전 승인 요청/응답
834 Enrollment 보험 가입/변경
820 Premium Payment 보험료 납부

📐 X12 구문 구조

X12 메시지는 계층적으로 구성됩니다:

Interchange (ISA/IEA)
  └─ Functional Group (GS/GE)
       └─ Transaction Set (ST/SE)
            └─ Loop
                 └─ Segment
                      └─ Element
                           └─ Sub-element

구분자:
- Segment Terminator: ~ (물결표)
- Element Separator: * (별표)
- Sub-element Separator: : (콜론)
- Repetition Separator: ^ (캐럿)

💡 EDI 세그먼트 예시

NM1*IL*1*SMITH*JOHN*A***MI*123456789~

구조 분석:
- NM1: 세그먼트 ID (Individual or Organizational Name)
- IL: Entity Identifier (Insured/Subscriber)
- 1: Entity Type Qualifier (Person)
- SMITH: Last Name
- JOHN: First Name
- A: Middle Name
- (공백): Name Prefix
- (공백): Name Suffix
- MI: ID Code Qualifier (Member ID)
- 123456789: Identification Code

3.2 837 청구 트랜잭션

📋 837P (Professional Claim)

의사, 클리닉, 외래 서비스 청구에 사용됩니다.

핵심 Loop 구조

  • 2000A Loop: Billing Provider
    • NM1: 청구 제공자 이름
    • N3: 주소
    • N4: 도시, 주, 우편번호
    • REF: 제공자 식별자 (NPI, Tax ID)
    • PER: 연락처 정보
  • 2000B Loop: Subscriber
    • SBR: Subscriber Information
    • NM1: 가입자 이름
    • N3, N4: 주소
    • DMG: 인구통계 (생년월일, 성별)
    • 2010BA: Subscriber Name 상세
    • 2010BB: Payer Name (보험사)
  • 2300 Loop: Claim Information
    • CLM: 청구 정보 (ID, 금액)
    • DTP: 서비스 날짜
    • CL1: 입원 정보 (해당 시)
    • HI: 진단 코드 (ICD-10)
    • 2310A: Attending Provider
    • 2310B: Operating Physician
  • 2400 Loop: Service Line
    • LX: Line Number
    • SV1: Professional Service (CPT, 금액)
    • DTP: Service Date
    • REF: Line Item Control Number

🏥 837I (Institutional Claim)

병원, 요양시설 청구에 사용되며, 837P보다 복잡합니다.

추가 Loop 및 세그먼트

  • 2300 Loop 추가 요소
    • CLM05-3: Admit Type Code (입원 유형)
    • CLM05-4: Admit Source Code (입원 경로)
    • CLM05-5: Patient Status Code (퇴원 상태)
    • HI02: Admitting Diagnosis
    • HI03: Patient Reason for Visit
    • HI04-HI12: Other Diagnoses
  • 2400 Loop: Service Line
    • SV2: Institutional Service Line
    • SV201: Revenue Code (0250, 0450 등)
    • SV202: Unit/Basis for Measurement (Days, Units)
    • SV203: Service Unit Count
    • SV204: Unit Rate

3.3 835 지급/송금 조언 (ERA)

835 트랜잭션은 보험사가 제공자에게 지급 정보를 전송할 때 사용합니다.

💰 835 핵심 정보

  • Payer Information — 보험사 이름, 주소, 연락처
  • Payee Information — 제공자 정보
  • Payment Summary — 총 지급액, 공제, 조정
  • Claim-level Details — 각 청구의 처리 결과
  • Service-level Details — 각 서비스 라인의 지급/거부
  • Adjustment Codes — 조정 이유 (CO, PR, OA 등)

📊 조정 코드 (CARC/RARC)

Group Code 의미 책임 주체
CO Contractual Obligation 보험사 계약 조정
PR Patient Responsibility 환자 부담
OA Other Adjustment 기타
PI Payer Initiated 보험사 주도

  • Service Type Codes
    • 30: Health Benefit Plan Coverage
    • 98: Professional (Physician)
    • AL: Vision
    • F6: Maternity

3.5 278 사전 승인

278 트랜잭션은 고가 시술, 입원, 전문 치료의 사전 승인을 요청하고 응답합니다.

📝 278 Request 핵심 정보

  • Service Information
    • UM*HS: Health Services Review (승인 요청 유형)
    • SV1/SV2: Requested Service (요청 서비스)
    • HI: Diagnosis Codes (진단 코드)
    • DTP: Service Dates (서비스 날짜)
  • Clinical Information
    • CR1: Ambulance Transport
    • CR2: Spinal Manipulation
    • PWK: Paperwork (지원 문서)

✅ 278 Response 결정 코드

코드 의미 다음 단계
A1 Certified (승인) 서비스 진행 가능
A2 Modified (조건부 승인) 수정 사항 확인
A3 Denied (거부) 이의 제기 또는 대안
A4 Pended (보류) 추가 정보 제공

EDI 트랜잭션은 의료 비즈니스의 신경계입니다. WIA-MED-028 표준은 이러한 트랜잭션을 더욱 견고하고 확장 가능하게 만들어, 글로벌 의료 데이터 교환의 기반을 마련합니다.
