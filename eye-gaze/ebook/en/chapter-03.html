Chapter 3: API Endpoints

3.1 Overview

The WIA Eye Gaze API defines a standardized set of endpoints for controlling eye tracking devices, managing calibration, streaming gaze data, and handling interaction zones. This chapter covers the complete API surface.

3.2 Device Discovery & Connection

List Available Devices

GET /api/v1/devices

Response:

{
  "devices": [
    {
      "deviceId": "tobii-pro-fusion-001",
      "vendor": "Tobii",
      "model": "Pro Fusion",
      "status": "connected",
      "capabilities": {
        "samplingRate": [60, 120, 250],
        "binocular": true,
        "accuracy": 0.4
      }
    }
  ]
}

Get Device Capabilities

GET /api/v1/devices/{deviceId}/capabilities

Response:

{
  "device": {
    "deviceId": "tobii-pro-fusion-001",
    "vendor": "Tobii",
    "model": "Pro Fusion",
    "firmwareVersion": "2.1.0",

    "binocular": true,

    "GAZE_POINT",

Connect to Device

POST /api/v1/devices/{deviceId}/connect

Request Body:

{
  "clientId": "aac-app-001",
  "clientName": "My AAC Application",
  "requestedCapabilities": ["GAZE_POINT", "CALIBRATION"]
}

Response:

{
  "sessionId": "session_2025_001",
  "status": "connected",
  "grantedCapabilities": ["GAZE_POINT", "CALIBRATION"],
  "streamUrl": "ws://localhost:8080/stream"
}

3.3 Calibration API

Start Calibration

POST /api/v1/devices/{deviceId}/calibration/start

Request Body:

{
  "type": "accessibility",  // standard | quick | accessibility
  "pointCount": 5,          // Number of calibration points
  "screenInfo": {
    "width": 1920,
    "height": 1080,
    "dpi": 96
  }
}

Response:

{
  "calibrationId": "cal_2025_001",
  "pointCount": 5,
  "points": [
    { "x": 0.5, "y": 0.5 },   // Center
    { "x": 0.1, "y": 0.1 },   // Top-left
    { "x": 0.9, "y": 0.1 },   // Top-right
    { "x": 0.1, "y": 0.9 },   // Bottom-left
    { "x": 0.9, "y": 0.9 }    // Bottom-right
  ],
  "status": "ready"
}

Collect Calibration Point

POST /api/v1/devices/{deviceId}/calibration/{calibrationId}/collect

Request Body:

{
  "pointIndex": 0,
  "position": { "x": 0.5, "y": 0.5 }
}

Response:

{
  "pointIndex": 0,
  "status": "collected",
  "quality": "good",
  "samplesCollected": 120
}

Finalize Calibration

POST /api/v1/devices/{deviceId}/calibration/{calibrationId}/compute

Response:

{
  "calibrationId": "cal_2025_001",
  "result": {
    "success": true,
    "averageAccuracy": 0.45,
    "averagePrecision": 0.12,
    "pointResults": [
      {
        "pointIndex": 0,
        "position": { "x": 0.5, "y": 0.5 },
        "accuracy": 0.38,
        "precision": 0.10,
        "valid": true
      }
    ]
  }
}

Calibration Types

  • Standard: 9-point, best accuracy, 45-60 seconds
  • Quick: 5-point, good accuracy, 20-30 seconds
  • Accessibility: Adaptive timing, larger targets, extra validation

3.4 Data Streaming API

Start Gaze Stream

POST /api/v1/devices/{deviceId}/stream/start

Request Body:

{
  "frequency": 60,           // Sampling rate in Hz
  "includeEyeData": true,    // Include individual eye data
  "includeEvents": true,     // Include fixation/saccade events
  "format": "json"           // json | binary | msgpack
}

Response:

{
  "streamId": "stream_001",
  "status": "streaming",
  "actualFrequency": 60,
  "websocketUrl": "ws://localhost:8080/stream/stream_001"
}

WebSocket Stream Message

Once connected to the WebSocket, clients receive continuous gaze data:

{
  "type": "gaze_point",
  "data": {
    "timestamp": 1704067200000,
    "x": 0.5234,
    "y": 0.3891,
    "confidence": 0.95,
    "valid": true,
    "leftEye": {
      "gaze": { "x": 0.5198, "y": 0.3876 },
      "valid": true,
      "pupilDiameter": 4.2
    },
    "rightEye": {
      "gaze": { "x": 0.5270, "y": 0.3906 },
      "valid": true,
      "pupilDiameter": 4.1
    }
  }
}

Stop Gaze Stream

POST /api/v1/devices/{deviceId}/stream/stop

3.5 Event Subscription

Subscribe to Events

POST /api/v1/devices/{deviceId}/events/subscribe

Request Body:

{
  "eventTypes": [
    "fixation_start",
    "fixation_end",
    "blink",
    "dwell_complete"
  ]
}

Event Message Format

{
  "type": "event",
  "event": {
    "type": "fixation_start",
    "timestamp": 1704067200000,
    "eventId": "evt_001",
    "fixationId": "fix_001",
    "centroid": { "x": 0.52, "y": 0.39 }
  }
}

3.6 Interaction Zones API

Register Interaction Zone

Define screen regions for gaze interaction (buttons, keyboard keys, etc.):

POST /api/v1/devices/{deviceId}/zones/register

Request Body:

{
  "zoneId": "button_send",
  "boundingBox": {
    "x": 0.7,
    "y": 0.8,
    "width": 0.2,
    "height": 0.1
  },
  "semanticType": "button",
  "label": "Send Message",
  "dwellTime": 800,          // ms for selection
  "enableDwell": true
}

Response:

{
  "zoneId": "button_send",
  "status": "registered"
}

Zone Event Notifications

When gaze enters/leaves or dwells in a registered zone:

{
  "type": "zone_event",
  "event": {
    "type": "dwell_complete",
    "timestamp": 1704067200800,
    "zoneId": "button_send",
    "dwellTime": 850,
    "position": { "x": 0.78, "y": 0.85 }
  }
}

Update Zone

PATCH /api/v1/devices/{deviceId}/zones/{zoneId}

Request Body:

{
  "dwellTime": 1000,  // Update dwell time
  "enabled": true
}

Unregister Zone

DELETE /api/v1/devices/{deviceId}/zones/{zoneId}

3.9 Error Responses

Standard Error Format

{
  "error": {
    "code": "DEVICE_NOT_FOUND",
    "message": "Device tobii-pro-fusion-002 not found",
    "details": {
      "availableDevices": ["tobii-pro-fusion-001"]
    }
  }
}

Common Error Codes

Code HTTP Status Description
DEVICE_NOT_FOUND 404 Requested device doesn't exist
DEVICE_BUSY 409 Device in use by another client
CALIBRATION_REQUIRED 428 Device needs calibration
INVALID_PARAMETER 400 Invalid request parameter
TRACKING_LOST 503 Eyes not detected
UNAUTHORIZED 401 Authentication required

3.10 Rate Limiting & Performance

API Rate Limits

  • Control Endpoints: 60 requests/minute
  • Configuration Updates: 10 requests/minute
  • Data Streaming: No limit (controlled by sampling rate)
  • Calibration: 5 starts/hour (prevent abuse)

Performance Best Practices

  • Batch Operations: Register multiple zones in a single request when possible
  • WebSocket Preferred: Use WebSocket streaming instead of polling for real-time data
  • Filter Events: Subscribe only to events you need
  • Binary Format: Use binary format for high-frequency data (120Hz+)

3.11 Multi-Monitor Support

Register Screen Layout

POST /api/v1/devices/{deviceId}/screens

Request Body:

{
  "screens": [
    {
      "screenId": "primary",
      "bounds": { "x": 0, "y": 0, "width": 1920, "height": 1080 },
      "primary": true
    },
    {
      "screenId": "secondary",
      "bounds": { "x": 1920, "y": 0, "width": 1920, "height": 1080 },
      "primary": false
    }
  ]
}

Gaze coordinates will be normalized per-screen or across the entire virtual desktop depending on configuration.
