Chapter 8: Certification & Compliance

8.1 Overview

WIA certification ensures that eye tracking devices and applications meet interoperability, performance, and accessibility standards. This chapter covers the certification process, testing procedures, and compliance requirements.

8.2 Certification Levels

Level Requirements Testing Use Cases
WIA Basic Core data format compliance, Basic API support Format validation, Basic functionality General applications, Gaming
WIA AAC Full event support, Dwell timing accuracy, Accessibility features AAC-specific tests, Latency verification AAC systems, Accessibility apps
WIA Research High accuracy, Timestamp precision, Data export Accuracy testing, Timing validation Research applications
WIA Medical AAC + Research requirements, HIPAA/GDPR compliance, Security audit Complete test suite, Security review Clinical AAC, Medical research

8.3 Device Certification

Hardware Requirements

Minimum Hardware Specifications

Parameter Basic AAC Research
Sampling Rate 30Hz 60Hz 120Hz
Accuracy < 2.0° < 1.5° < 0.5°
Precision < 0.5° < 0.3° < 0.1°
Latency < 100ms < 50ms < 10ms
Data Loss < 10% < 5% < 1%

Software Requirements

  • Data Format Compliance - Must output WIA-standard GazePoint format
  • API Support - Implement required API endpoints for certification level
  • Protocol Support - WebSocket minimum, IPC for AAC+ levels
  • Calibration - Standard calibration procedure with quality assessment
  • Documentation - Complete API documentation and user guide

Certification Process for Devices

1. Application Submission
   └─ Manufacturer submits device specifications
   └─ Documentation review
   └─ Preliminary evaluation

2. Technical Testing
   └─ Data format validation
   └─ API endpoint testing
   └─ Performance benchmarking
   └─ Accuracy measurement
   └─ Latency profiling

3. Accessibility Testing (AAC+ levels)
   └─ Dwell timing accuracy
   └─ Event detection reliability
   └─ User testing with AAC users

4. Security Audit (Medical level)
   └─ Encryption verification
   └─ Access control testing
   └─ Privacy compliance review

5. Certification Award
   └─ Certification badge issued
   └─ Listed in WIA certified devices registry
   └─ Marketing materials approved

8.4 Application Certification

Application Requirements

WIA Basic (General Applications)

  • ✓ Correctly parse WIA GazePoint format
  • ✓ Handle invalid/missing data gracefully
  • ✓ Support at least one WIA-certified device
  • ✓ Provide user documentation

WIA AAC (AAC Applications)

  • ✓ All Basic requirements
  • ✓ Implement dwell selection with configurable timing
  • ✓ Provide visual feedback for dwell progress
  • ✓ Support multiple selection methods (dwell, blink, switch)
  • ✓ Accessibility features (high contrast, customizable UI)
  • ✓ Error recovery and undo functionality
  • ✓ User testing with AAC users (minimum 5 participants)

WIA Research (Research Applications)

  • ✓ All Basic requirements
  • ✓ Export data in standard formats (CSV, JSON, HDF5)
  • ✓ Preserve full timestamp precision
  • ✓ Support data synchronization with external systems
  • ✓ Provide analysis tools or export for standard tools

Testing Checklist for Applications

□ Data Handling
  □ Correctly parse all required GazePoint fields
  □ Handle optional fields appropriately
  □ Validate data quality (confidence, validity)
  □ Implement timeout/reconnection logic

□ User Interface
  □ Minimum target size: 80x80 pixels (AAC)
  □ Adequate spacing between interactive elements
  □ High contrast ratio (WCAG AAA: 7:1)
  □ Clear visual feedback for gaze interactions

□ Performance
  □ Smooth rendering at application frame rate
  □ No dropped gaze updates
  □ Responsive to user actions
  □ Low CPU/memory usage

□ Accessibility
  □ Works with screen readers (when applicable)
  □ Keyboard navigation alternative
  □ Customizable settings (font size, contrast, dwell time)
  □ Multi-language support

□ Error Handling
  □ Graceful degradation when tracking quality degrades
  □ Clear error messages
  □ Automatic recalibration prompts
  □ Data loss prevention

□ Privacy & Security
  □ Obtain informed consent
  □ Encrypt sensitive data
  □ Implement access controls
  □ Provide data deletion option

□ Documentation
  □ Installation guide
  □ User manual
  □ Troubleshooting guide
  □ API documentation (if extensible)

8.5 Interoperability Testing

Device-Application Compatibility Matrix

Certified applications must be tested with multiple WIA-certified devices:

Device Category Minimum Testing Recommended
Screen-based trackers 2 different models 3+ models from different manufacturers
Wearable trackers 1 model 2 models
Webcam-based 1 solution 2 solutions

Cross-Platform Testing

Applications should be tested on multiple platforms:

  • Operating Systems: Windows 10/11, macOS 12+, Linux (Ubuntu 20.04+)
  • Browsers (web apps): Chrome, Firefox, Safari, Edge (latest versions)
  • Screen Resolutions: 1920x1080, 2560x1440, 3840x2160, and unusual aspect ratios

8.6 Accessibility Standards Compliance

WCAG 2.1 Compliance

AAC applications must meet Web Content Accessibility Guidelines:

WCAG Success Criteria for Eye Gaze Apps

  • 1.4.3 Contrast (Minimum) - AAA level: 7:1 contrast ratio
  • 1.4.11 Non-text Contrast - UI components have 3:1 contrast
  • 2.1.1 Keyboard - Alternative keyboard access available
  • 2.5.5 Target Size - Minimum 44x44 CSS pixels (prefer larger for gaze)
  • 3.2.2 On Input - No unexpected changes on dwell selection

Section 508 Compliance (US)

For US government/education use:

  • Alternative input methods available
  • Compatible with assistive technologies
  • Closed captions for audio/video content
  • Accessible documentation

EN 301 549 Compliance (EU)

European accessibility standard for ICT products:

  • Supports alternative input methods
  • Operable without precise hand movements
  • Works with screen magnification
  • Configurable timeouts and timing

8.7 Performance Benchmarking

Benchmark Suite

Standard tests for WIA certification:

1. Accuracy Test

Test: 9-Point Accuracy Measurement
Setup:
  - Display 9 fixed points on screen
  - User fixates each point for 2 seconds
  - Record gaze coordinates

Measurement:
  - Angular deviation from target point
  - Average across all points
  - Separate results for each eye (binocular)

Pass Criteria:
  - Basic: < 2.0° average
  - AAC: < 1.5° average
  - Research: < 0.5° average

2. Precision Test

Test: Fixation Stability
Setup:
  - User fixates single point for 10 seconds
  - Record all gaze samples

Measurement:
  - Standard deviation of gaze points
  - RMS error from fixation centroid

Pass Criteria:
  - Basic: < 0.5° RMS
  - AAC: < 0.3° RMS
  - Research: < 0.1° RMS

3. Latency Test

Test: End-to-End Latency
Setup:
  - High-speed camera records screen and eye
  - User makes rapid saccade on cue
  - Measure time from eye movement to cursor update

Measurement:
  - Time difference between physical movement and update
  - Average across 20 saccades

Pass Criteria:
  - Basic: < 100ms
  - AAC: < 50ms
  - Research: < 10ms

4. Data Loss Test

Test: Sample Completeness
Setup:
  - Stream gaze data for 60 seconds
  - Expected samples = frequency × 60

Measurement:
  - Received samples / Expected samples
  - Percentage of valid samples

Pass Criteria:
  - Basic: > 90% received, > 85% valid
  - AAC: > 95% received, > 90% valid
  - Research: > 99% received, > 95% valid

8.8 Certification Maintenance

Recertification Requirements

  • Annual Review - Confirm continued compliance
  • Major Updates - Recertify after significant changes
  • Security Patches - Document and test security updates
  • User Feedback - Address reported issues

Version Compatibility

Certified products must maintain backward compatibility:

Update Type Compatibility Required Recertification
Bug fixes Full backward compatibility Not required
Minor features Additive only Recommended
Major version Support previous version Required
Breaking changes Migration path provided Required

8.9 Certification Badge & Marketing

WIA Certified Badge Usage

Badge Display Guidelines

Certified products may display the WIA badge:

  • Badge must include certification level (Basic/AAC/Research/Medical)
  • Version number must be specified
  • No modifications to badge design
  • Link to WIA certification verification page

Marketing Claims

Approved claims for certified products:

  • ✓ "WIA Eye Gaze Certified"
  • ✓ "Compatible with all WIA-certified eye trackers"
  • ✓ "Meets WIA AAC accessibility standards"
  • ✗ "Best eye tracking solution" (not verifiable)
  • ✗ Claims about non-tested features

8.10 Getting Certified

Step-by-Step Process

1. Prepare Your Product
   └─ Review requirements for desired certification level
   └─ Implement required features
   └─ Self-test against checklist
   └─ Prepare documentation

2. Apply for Certification
   └─ Submit application at wia.org/certify
   └─ Pay certification fee
   └─ Provide product access for testing

3. Technical Review
   └─ WIA team performs testing
   └─ Results shared with applicant
   └─ Address any issues found

4. Final Approval
   └─ Certification committee review
   └─ Certificate issued
   └─ Product listed in registry

5. Maintain Certification
   └─ Annual compliance check
   └─ Update certification for major releases
