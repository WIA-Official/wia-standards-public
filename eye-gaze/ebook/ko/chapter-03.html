제3장: API 엔드포인트

3.1 개요

WIA 시선 추적 API는 시선 추적 장치를 제어하고, 보정을 관리하고, 시선 데이터를 스트리밍하고, 상호작용 영역을 처리하기 위한 표준화된 엔드포인트 세트를 정의합니다. 이 장에서는 완전한 API 인터페이스를 다룹니다.

3.2 장치 탐색 및 연결

사용 가능한 장치 목록

GET /api/v1/devices

응답:

{
  "devices": [
    {
      "deviceId": "tobii-pro-fusion-001",
      "vendor": "Tobii",
      "model": "Pro Fusion",
      "status": "connected",
      "capabilities": {
        "samplingRate": [60, 120, 250],
        "binocular": true,
        "accuracy": 0.4
      }
    }
  ]
}

장치에 연결

POST /api/v1/devices/{deviceId}/connect

3.3 보정 API

보정 시작

POST /api/v1/devices/{deviceId}/calibration/start

요청 본문:

{
  "type": "accessibility",
  "pointCount": 5,
  "screenInfo": {
    "width": 1920,
    "height": 1080,
    "dpi": 96
  }
}

보정 유형

  • 표준: 9포인트, 최고 정확도, 45-60초
  • 빠른: 5포인트, 양호한 정확도, 20-30초
  • 접근성: 적응형 타이밍, 더 큰 대상, 추가 검증

3.4 데이터 스트리밍 API

시선 스트림 시작

POST /api/v1/devices/{deviceId}/stream/start

WebSocket 스트림 메시지

WebSocket에 연결되면 클라이언트는 연속적인 시선 데이터를 수신합니다:

{
  "type": "gaze_point",
  "data": {
    "timestamp": 1704067200000,
    "x": 0.5234,
    "y": 0.3891,
    "confidence": 0.95,
    "valid": true
  }
}

3.5 상호작용 영역 API

상호작용 영역 등록

시선 상호작용을 위한 화면 영역(버튼, 키보드 키 등) 정의:

POST /api/v1/devices/{deviceId}/zones/register

요청 본문:

{
  "zoneId": "button_send",
  "boundingBox": {
    "x": 0.7,
    "y": 0.8,
    "width": 0.2,
    "height": 0.1
  },
  "label": "메시지 보내기",
  "dwellTime": 800,
  "enableDwell": true
}

3.6 장치 구성

현재 구성 가져오기

GET /api/v1/devices/{deviceId}/config

응답:

{
  "samplingRate": 120,
  "fixationDetection": {
    "minDuration": 100,
    "maxDispersion": 1.0
  },
  "dwellSelection": {
    "defaultThreshold": 800,
    "visualFeedback": true
  }
}

3.7 오류 응답

일반 오류 코드

코드 HTTP 상태 설명
DEVICE_NOT_FOUND 404 요청한 장치가 존재하지 않음
CALIBRATION_REQUIRED 428 장치에 보정이 필요함
TRACKING_LOST 503 눈을 감지할 수 없음
