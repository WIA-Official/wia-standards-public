5.1 AAC 시스템 통합

시선 추적은 말할 수 없는 개인에게 중요한 의사소통 방법을 제공합니다. 이 섹션에서는 AAC 응용 프로그램에 WIA 시선 추적을 통합하는 방법을 다룹니다.

응시 선택 구현

응시 선택은 사용자가 대상을 일정 기간(일반적으로 800ms) 동안 보고 선택하는 것을 허용합니다.

// 응시 클릭 구현
class DwellSelector {
  constructor(threshold = 800) {
    this.threshold = threshold;
    this.dwellStart = null;
    this.currentTarget = null;
  }

  onGazePoint(point) {
    const target = this.getTargetAt(point.x, point.y);

    if (target !== this.currentTarget) {
      this.cancelDwell();
      this.currentTarget = target;
      if (target) {
        this.dwellStart = Date.now();
      }
    } else if (target && this.dwellStart) {
      const elapsed = Date.now() - this.dwellStart;
      if (elapsed >= this.threshold) {
        this.select(target);
        this.cancelDwell();
      }
    }
  }
}

AAC 키보드 레이아웃

키 크기 응시 시간 사용 사례
큰 (100x100px) 600-800ms 초보 사용자, 낮은 정확도
중간 (60x60px) 800-1000ms 일반 사용자
작은 (40x40px) 1000-1200ms 고급 사용자, 고정확도

5.2 컴퓨터 제어 통합

시선 추적을 사용하여 전체 컴퓨터를 제어할 수 있습니다.

마우스 에뮬레이션

// 시선을 마우스 커서로 변환
function gazeToMouse(gazePoint) {
  const screenX = gazePoint.x * screen.width;
  const screenY = gazePoint.y * screen.height;

  // 평활화 적용
  const smoothed = smoothGaze(screenX, screenY);

  // 마우스 커서 이동
  moveCursor(smoothed.x, smoothed.y);
}

5.3 게임 통합

시선 추적은 게임을 더욱 접근 가능하고 몰입감 있게 만듭니다.

시선 기반 게임 메커니즘

  • 시선 조준: 보는 곳을 겨냥
  • 카메라 제어: 시선으로 게임 세계 둘러보기
  • 메뉴 탐색: 시선으로 옵션 선택
  • 주의 기반 게임플레이: 바라보는 곳에 반응하는 게임 요소

5.4 연구 응용 프로그램

시선 추적은 주의, 인지 및 사용자 경험에 대한 통찰력을 제공합니다.

데이터 수집

// 연구 데이터 로깅
class GazeDataLogger {
  constructor(participantId, studyId) {
    this.participantId = participantId;
    this.studyId = studyId;
    this.data = [];
  }

  logGazePoint(point) {
    this.data.push({
      participant: this.participantId,
      study: this.studyId,
      timestamp: point.timestamp,
      x: point.x,
      y: point.y,
      confidence: point.confidence
    });
  }

  exportData(format = 'csv') {
    return this.convertToFormat(this.data, format);
  }
}

5.5 모범 사례

사용자 경험

  • 명확한 피드백: 응시 진행률을 시각적으로 표시
  • 실수 방지: 중요한 작업에 확인 요구
  • 사용자 정의: 응시 시간 및 대상 크기를 조정 가능하게 함
  • 교육: 튜토리얼 및 연습 모드 제공

접근성 고려사항

  • 높은 대비: 명확한 시각적 대상 사용
  • 적응형 타이밍: 사용자 능력에 맞게 조정
  • 오류 복구: 쉬운 실행 취소 메커니즘
  • 휴식 알림: 눈의 피로 방지

통합 체크리스트

  • 장치 기능 확인
  • 적절한 보정 수행
  • 품질 모니터링 구현
  • 사용자 피드백 제공
  • 오류 복구 처리
  • 실제 사용자로 테스트
