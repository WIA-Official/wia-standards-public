제2장: 데이터 스키마

버전 1.0.0

제2장: 데이터 스키마

2.1 개요

WIA 시선 추적 데이터 스키마는 시선 추적 데이터를 표현하기 위한 표준화된 구조를 정의하여 모든 장치와 응용 프로그램 간의 상호운용성을 보장합니다. 이 장에서는 핵심 데이터 유형, 좌표 시스템 및 이벤트 구조를 다룹니다.

2.2 기본 타입

Vector2D

정규화된 또는 픽셀 공간의 2D 좌표를 나타냅니다.

interface Vector2D {
  x: number;  // X 좌표
  y: number;  // Y 좌표
}

Vector3D

공간 시선 추적을 위한 3D 좌표를 나타냅니다.

interface Vector3D {
  x: number;  // X 좌표 (mm)
  y: number;  // Y 좌표 (mm)
  z: number;  // Z 좌표 (mm, 장치로부터의 거리)
}

BoundingBox

화면의 직사각형 영역을 정의합니다.

interface BoundingBox {
  x: number;       // 왼쪽 상단 X (정규화됨)
  y: number;       // 왼쪽 상단 Y (정규화됨)
  width: number;   // 너비 (정규화됨)
  height: number;  // 높이 (정규화됨)
}

2.3 GazePoint - 핵심 데이터 구조

GazePoint는 시선 추적 데이터의 기본 단위로, 특정 순간에 사용자가 어디를 보고 있는지를 나타냅니다.

완전한 GazePoint 구조

interface GazePoint {
  // === 필수 필드 ===

  /** 밀리초 단위의 Unix 타임스탬프 */
  timestamp: number;

  /** 화면 X 좌표 (0.0 = 왼쪽, 1.0 = 오른쪽) */
  x: number;

  /** 화면 Y 좌표 (0.0 = 위, 1.0 = 아래) */
  y: number;

  /** 데이터 신뢰도 (0.0 - 1.0) */
  confidence: number;

  /** 데이터 유효성 플래그 */
  valid: boolean;

  // === 선택적 필드 ===

  /** 왼쪽 눈 데이터 */
  leftEye?: EyeData;

  /** 오른쪽 눈 데이터 */
  rightEye?: EyeData;

  /** 현재 고정 중인지 여부 */
  fixation?: boolean;

  /** 현재 단속적 안구 운동 중인지 여부 */
  saccade?: boolean;

  /** 고정 ID (고정 포인트 그룹화) */
  fixationId?: string;

  /** 장치별 타임스탬프 */
  deviceTimestamp?: number;

  /** 추가 메타데이터 */
  metadata?: Record<string, unknown>;
}

2.4 EyeData - 개별 눈 정보

왼쪽 및 오른쪽 눈에 대한 개별 상세 데이터입니다.

interface EyeData {
  // === 기본 시선 데이터 ===

  /** 이 눈의 시선 좌표 */
  gaze: Vector2D;

  /** 데이터 유효성 */
  valid: boolean;

  // === 동공 데이터 ===

  /** 밀리미터 단위의 동공 지름 */
  pupilDiameter?: number;

  /** 카메라 이미지에서의 동공 중심 (정규화됨) */
  pupilCenter?: Vector2D;

  // === 3D 데이터 ===

  /** 3D 공간에서의 눈 위치 (추적기로부터 mm) */
  gazeOrigin?: Vector3D;

  /** 단위 벡터로 표현된 시선 방향 */
  gazeDirection?: Vector3D;

  // === 눈 상태 ===

  /** 눈의 열림 정도 (0.0 = 닫힘, 1.0 = 완전히 열림) */
  eyeOpenness?: number;

  /** 밀리미터 단위의 눈 열림 */
  eyeOpennessMm?: number;
}

동공 지름의 응용

동공 지름은 다음과 같은 용도로 유용합니다:

  • 인지 부하 - 더 큰 동공은 더 높은 정신적 노력을 나타냄
  • 감정 반응 - 동공 변화는 각성 및 관심과 상관관계
  • 조명 적응 - 밝기 변화에 대한 동공 반응 추적
  • 피로 감지 - 피로와 함께 동공 동역학이 변화

2.5 좌표 시스템

화면 좌표 (정규화됨)

모든 시선 좌표는 0.0에서 1.0까지의 정규화된 값을 사용합니다:

화면 레이아웃:

(0.0, 0.0) ──────────────────────► X (1.0, 0.0)
     │
     │     ┌─────────────────────┐
     │     │                     │
     │     │       화면          │
     │     │                     │
     │     │   (0.5, 0.5)        │
     │     │      중심           │
     │     │                     │
     │     └─────────────────────┘
     │
     ▼
Y (0.0, 1.0)                    (1.0, 1.0)

좌표 변환

정규화된 좌표와 픽셀 좌표 간 변환:

// 정규화됨에서 픽셀로
function toPixelCoordinates(
  normalizedPoint: Vector2D,
  screenWidth: number,
  screenHeight: number
): { px: number; py: number } {
  return {
    px: Math.round(normalizedPoint.x * screenWidth),
    py: Math.round(normalizedPoint.y * screenHeight)
  };
}

// 예: 1920x1080 화면
// (0.5, 0.5) → (960, 540)

// 픽셀에서 정규화됨으로
function toNormalizedCoordinates(
  px: number,
  py: number,
  screenWidth: number,
  screenHeight: number
): Vector2D {
  return {
    x: px / screenWidth,
    y: py / screenHeight
  };
}

2.6 데이터 품질 지표

신뢰도 값

범위 품질 권장 조치
0.9 - 1.0 우수 정상 사용
0.7 - 0.9 양호 정상 사용
0.5 - 0.7 보통 품질 모니터링
0.3 - 0.5 불량 재보정 고려
0.0 - 0.3 매우 불량 데이터가 신뢰할 수 없을 수 있음

2.7 이벤트 시스템

이벤트 카테고리

카테고리 이벤트 사용 사례
눈 움직임 고정, 단속적 안구 운동, 부드러운 추적 읽기 분석, 주의 추적
눈 상태 깜빡임, 윙크, 이중 깜빡임 대체 입력, 피로 감지

고정 이벤트

고정은 눈이 상대적으로 정지해 있을 때 발생하며 시각적 주의를 나타냅니다.

interface FixationEvent {
  type: 'fixation_start' | 'fixation_update' | 'fixation_end';
  timestamp: number;
  eventId: string;
  fixationId: string;

  /** 고정 중심점 */
  centroid: Vector2D;

  /** 고정 반경 (정규화됨) */
  radius?: number;

  /** 각도 분산 (도) */
  dispersion?: number;

  /** 지속 시간 (ms, fixation_end 시) */
  duration?: number;
}

단속적 안구 운동 이벤트

고정 사이의 빠른 눈 움직임입니다.

interface SaccadeEvent {
  type: 'saccade_start' | 'saccade_end';
  timestamp: number;
  eventId: string;
  saccadeId: string;

  /** 시작 위치 */
  startPosition: Vector2D;

  /** 종료 위치 (saccade_end 시) */
  endPosition?: Vector2D;

  /** 이동 거리 (도) */
  amplitude?: number;

  /** 최대 속도 (도/초) */
  peakVelocity?: number;

  /** 지속 시간 (ms) */
  duration?: number;
}

깜빡임 이벤트

입력 및 피로 모니터링을 위한 눈 닫힘 감지입니다.

interface BlinkEvent {
  type: 'blink_start' | 'blink_end' | 'blink' | 'double_blink';
  timestamp: number;
  eventId: string;
  blinkId: string;

  /** 어느 눈이 깜빡였는지 */
  eye: 'left' | 'right' | 'both';

  /** 깜빡임 지속 시간 (ms) */
  duration?: number;

  /** 이중 깜빡임 간격 (ms) */
  interval?: number;
}

의도적 vs. 자연스러운 깜빡임

  • 자연스러운 깜빡임: 100-400ms 지속 시간
  • 의도적 깜빡임: 400ms+ 지속 시간 (구성 가능한 임계값)

AAC 응용 프로그램은 의도적인 깜빡임을 선택 메커니즘으로 사용할 수 있습니다.

응시 이벤트

AAC에 중요 - 대상을 보면서 시선 기반 선택.

interface DwellEvent {
  type: 'dwell_start' | 'dwell_progress' | 'dwell_complete' | 'dwell_cancel';
  timestamp: number;
  eventId: string;
  dwellId: string;

  /** 대상 요소 */
  target: GazeTarget;

  /** 구성된 응시 시간 (ms) */
  dwellThreshold: number;

  /** 경과 시간 (ms) */
  elapsed: number;

  /** 진행률 (0.0 - 1.0) */
  progress: number;

  /** 취소 이유 (취소된 경우) */
  cancelReason?: 'gaze_left' | 'timeout' | 'user_cancel';
}
