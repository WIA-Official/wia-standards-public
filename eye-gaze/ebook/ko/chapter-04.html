4.1 개요

WIA 시선 추적 표준은 다양한 사용 사례에 최적화된 여러 통신 프로토콜을 지원합니다: 네트워크 통신을 위한 WebSocket, 로컬 초저지연을 위한 IPC, 고주파수 데이터 전송을 위한 바이너리 형식.

4.2 WebSocket 프로토콜

연결 설정

// 클라이언트가 WebSocket 엔드포인트에 연결
const ws = new WebSocket('ws://localhost:8080/stream');

ws.onopen = () => {
  ws.send(JSON.stringify({
    type: 'handshake',
    version: '1.0.0',
    clientId: 'aac-app-001',
    capabilities: ['GAZE_POINT', 'EVENTS']
  }));
};

메시지 유형

유형 방향 설명
handshake 클라이언트 → 서버 초기 연결 핸드셰이크
gaze_point 서버 → 클라이언트 단일 시선 데이터 포인트
event 서버 → 클라이언트 시선 이벤트 알림
heartbeat 양방향 연결 유지 핑

연결 시간 초과

서버는 연속 3회 하트비트를 놓친 클라이언트를 연결 해제합니다 (15초).

4.3 프로세스 간 통신 (IPC)

초저지연(<1ms)이 필요한 로컬 응용 프로그램의 경우 IPC는 직접 공유 메모리 액세스를 제공합니다.

플랫폼별 전송

플랫폼 전송 경로/이름
Linux Unix 도메인 소켓 /tmp/wia-eye-gaze.sock
macOS Unix 도메인 소켓 /tmp/wia-eye-gaze.sock
Windows 명명된 파이프 \\.\pipe\wia-eye-gaze

성능 비교

전송 지연 시간 (p50) 지연 시간 (p99) 처리량
공유 메모리 0.01ms 0.05ms 1M points/s
Unix 소켓 0.3ms 0.8ms 100K msg/s
WebSocket (로컬) 2.0ms 5.0ms 20K msg/s

4.4 바이너리 형식 사양

바이너리 형식 사용 시기

  • 높은 샘플링 속도: 120Hz 이상
  • 대역폭 제한: 모바일 또는 원격 연결
  • 저장: 장기 데이터 기록
  • 성능 중요: 실시간 게임 또는 VR

바이너리 vs JSON 크기

형식 포인트당 크기 120Hz/초 1시간
JSON (정돈) ~450 bytes 54 KB 194 MB
JSON (압축) ~180 bytes 22 KB 78 MB
WIA 바이너리 21 bytes 2.5 KB 9 MB

4.5 지연 시간 요구사항

AAC의 지연 시간 분석

눈 → 추적기 처리:        3-10ms
추적기 → 컴퓨터 전송:    1-3ms
API 처리:               1-2ms
네트워크/IPC:            0.1-5ms
응용 프로그램 처리:      5-10ms
화면 렌더링:            16ms (60Hz)
─────────────────────────────────────
총 AAC 지연 시간:       26-46ms (허용 가능)
게임 목표:              <20ms 총

4.6 동기화 및 타임스탬프

타임스탬프 유형

  • 장치 타임스탬프: 하드웨어 클록 (마이크로초 정밀도)
  • 시스템 타임스탬프: OS 클록 (밀리초 정밀도)
  • 서버 타임스탬프: API 서버 클록
  • 클라이언트 타임스탬프: 응용 프로그램 수신 시간

4.7 보안 고려사항

WebSocket용 TLS/SSL

// 보안 WebSocket 연결
const ws = new WebSocket('wss://eye-tracker.local:8443/stream');

IPC 보안

  • 파일 권한: 소유자 및 그룹으로 제한된 소켓/파이프 (0660)
  • 프로세스 검증: 연결 프로세스 신원 확인
  • 메시지 인증: 민감한 명령에 대한 HMAC 서명
