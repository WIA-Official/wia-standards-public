제6장: 보안 및 프라이버시

6.1 눈 데이터 보호

눈 추적 데이터는 민감한 생체 인식 정보로 간주되며 강력한 보호가 필요합니다.

데이터 분류

데이터 유형 민감도 보호 수준
시선 위치 높음 암호화 필수
동공 크기 매우 높음 암호화 + 접근 제어
개인 식별자 매우 높음 암호화 + 감사
집계 통계 낮음 익명화

6.2 동의 관리

사용자는 시선 데이터 수집 및 사용에 대한 명확한 통제권을 가져야 합니다.

동의 요구사항

  • 명시적 동의: 데이터 수집 전에 명확한 승인
  • 목적 제한: 명시된 목적으로만 사용
  • 철회 가능: 언제든지 동의 철회 가능
  • 투명성: 데이터 사용 방법 명확히 설명

동의 기록 예제

{
  "consentId": "consent_2025_001",
  "userId": "user_12345",
  "timestamp": 1704067200000,
  "purposes": [
    "AAC_COMMUNICATION",
    "SYSTEM_CALIBRATION"
  ],
  "dataTypes": ["GAZE_POINT", "EVENTS"],
  "retentionPeriod": "90_DAYS",
  "grantedPermissions": [
    "COLLECT",
    "PROCESS",
    "STORE"
  ],
  "deniedPermissions": ["SHARE", "RESEARCH"]
}

6.3 데이터 최소화

필요한 최소한의 데이터만 수집하고 저장합니다.

최소화 원칙

  • 목적 제한: 명시된 목적에 필요한 데이터만
  • 시간 제한: 필요한 기간 동안만 보관
  • 범위 제한: 관련 데이터 필드만 수집

6.4 암호화

전송 중 암호화

// TLS 1.3 WebSocket
const ws = new WebSocket('wss://tracker.example.com:8443');

// 인증서 고정
const expectedFingerprint = '...';
ws.addEventListener('open', () => {
  verifyServerCertificate(expectedFingerprint);
});

저장 시 암호화

  • AES-256: 저장된 데이터 파일
  • 키 관리: 안전한 키 저장 및 교체
  • 필드 수준 암호화: 민감한 필드 개별 암호화

6.5 접근 제어

역할 기반 접근

역할 권한 사용 사례
사용자 자신의 데이터 읽기/삭제 개인 AAC 사용
응용 프로그램 실시간 데이터 읽기 AAC/게임 앱
연구자 익명화된 데이터 읽기 연구 연구
관리자 시스템 구성 장치 관리

6.6 프라이버시 위험

잠재적 위험

  • 신원 노출: 시선 패턴이 개인을 식별할 수 있음
  • 관심사 추론: 보는 내용이 관심사를 드러냄
  • 건강 정보: 동공 반응이 상태를 나타낼 수 있음
  • 인지 상태: 시선 패턴이 정신 상태를 드러낼 수 있음

완화 전략

  • 익명화: 개인 식별자 제거
  • 집계: 개별 패턴 숨기기 위해 데이터 집계
  • 차등 프라이버시: 통계적 노이즈 추가
  • 목적 제한: 승인된 사용으로만 제한

6.7 규정 준수

관련 규정

  • GDPR (EU): 생체 인식 데이터 보호
  • CCPA (캘리포니아): 소비자 프라이버시 권리
  • HIPAA (미국 의료): 건강 정보 보호
  • PIPEDA (캐나다): 개인정보 보호

법적 요구사항

시선 추적 데이터는 많은 관할권에서 생체 인식 데이터로 간주됩니다. 법률 자문을 받아 모든 관련 규정을 준수하는지 확인하십시오.
