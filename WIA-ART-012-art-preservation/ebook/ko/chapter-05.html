5.1 API 기반 보존의 필요성

Phase 1에서 구축한 메타데이터는 훌륭한 시작점이지만, 대규모 컬렉션을 관리하려면 자동화가 필수입니다. Phase 2 API Interface는 보존 시스템과 프로그래밍 방식으로 상호작용할 수 있게 하여, 수동 작업을 최소화하고 오류를 방지합니다.

🎯 Phase 2의 목표
  • 자동화: 메타데이터 생성, 검증, 업데이트를 스크립트로 처리
  • 통합: 기존 갤러리, 박물관, 플랫폼 시스템과 연동
  • 분산 보존: LOCKSS 원칙으로 여러 기관이 협력
  • 버전 관리: Git처럼 메타데이터와 작품의 변경 이력 추적
  • 접근 제어: OAuth 2.0으로 안전한 인증 및 권한 관리

MoMA(Museum of Modern Art)는 2019년 디지털 아트 컬렉션 3,000점을 재목록화할 때, API를 통한 자동화로 예상 시간을 18개월에서 4개월로 단축했습니다. 수동으로 메타데이터를 입력하면 평균 45분/작품이 소요되었지만, API 통합으로 평균 3분/작품으로 감소했습니다.

5.2 RESTful API 설계

WIA-ART-012 API는 REST(Representational State Transfer) 원칙을 따라 설계되었습니다. 모든 작품은 고유한 URL을 가지며, 표준 HTTP 메서드(GET, POST, PUT, DELETE)로 조작합니다.

5.2.1 API 엔드포인트 구조

HTTP Method Endpoint 설명 인증 필요
GET /api/v1/artworks 작품 목록 조회 (검색, 필터, 페이지네이션) 선택
GET /api/v1/artworks/{id} 특정 작품의 전체 메타데이터 조회 선택

5.3 LOCKSS: Lots of Copies Keep Stuff Safe

LOCKSS는 스탠포드 대학 도서관에서 개발한 분산 디지털 보존 시스템입니다. 핵심 원칙은 단순합니다: "많은 복사본이 있으면 안전하다." 여러 기관이 독립적으로 동일한 콘텐츠를 보존하면, 한 기관이 실패해도 다른 기관에서 복구할 수 있습니다.

📚 LOCKSS의 역사

2000년대 초반, 전자 저널 출판사들이 폐업하거나 인수합병되면서 많은 학술 논문이 사라졌습니다. LOCKSS는 이 문제를 해결하기 위해 도서관들이 전자 저널의 사본을 로컬에 보관하고, 서로 검증하는 시스템을 구축했습니다. 2024년 현재 전 세계 500개 이상의 기관이 LOCKSS 네트워크에 참여하고 있습니다.

5.3.1 LOCKSS 핵심 개념

개념 설명 WIA-ART-012 적용
AU (Archival Unit) 보존의 최소 단위 (예: 1년치 저널) 개별 예술 작품 또는 컬렉션
Polling 참여 노드들이 서로 체크섬을 비교하여 무결성 검증 주간/월간 자동 검증
Voting 체크섬 불일치 시 다수결로 올바른 버전 결정 3개 이상 노드 참여 시 적용
Repair 손상된 콘텐츠를 정상 노드에서 복구 자동 복구 및 로그 기록
Audit 정기적 보존 상태 감사 및 보고 월간 보존 건강도 리포트

5.6 실시간 알림 및 웹훅 (확장)

웹훅은 이벤트 기반 아키텍처의 핵심입니다. 시스템이 중요한 이벤트를 감지하면 즉시 등록된 URL로 HTTP POST 요청을 보냅니다.

5.6.1 웹훅 이벤트 유형

이벤트 발생 시점 페이로드 예시

integrity.failed 체크섬 불일치 감지 시 { id, expected, actual }

migration.required 형식 노후화 감지 시 { id, format, risk_level }

5.6.2 Slack 통합 예시

// 웹훅 수신 서버 (Express.js)
import express from 'express';
import crypto from 'crypto';
import axios from 'axios';

const app = express();
app.use(express.json());

const SLACK_WEBHOOK = process.env.SLACK_WEBHOOK_URL;
const WIA_WEBHOOK_SECRET = process.env.WIA_WEBHOOK_SECRET;

app.post('/webhooks/wia', async (req, res) => {
  // 1. 서명 검증
  const signature = req.headers['x-wia-signature'];
  const expectedSignature = crypto
    .createHmac('sha256', WIA_WEBHOOK_SECRET)
    .update(JSON.stringify(req.body))
    .digest('hex');

  if (`sha256=${expectedSignature}` !== signature) {
    return res.status(401).send('Invalid signature');
  }

  // 2. 이벤트 처리
  const { event, artwork_id, details } = req.body;

  let slackMessage;
  if (event === 'integrity.failed') {
    slackMessage = {
      text: `🚨 무결성 검증 실패: ${artwork_id}`,
      attachments: [{
        color: 'danger',
        fields: [
          { title: 'Artwork ID', value: artwork_id, short: true },
          { title: 'Expected Checksum', value: details.expected_checksum.substring(0, 16) + '...', short: true },
          { title: 'Actual Checksum', value: details.actual_checksum.substring(0, 16) + '...', short: true },
          { title: 'Action Required', value: details.action_required }
        ]
      }]
    };

    // Slack 전송
    await axios.post(SLACK_WEBHOOK, slackMessage);

    // 자동 복구 트리거
    await triggerAutoRecovery(artwork_id);
  }

  res.status(200).send('OK');
});

app.listen(3000, () => console.log('Webhook server running on port 3000'));

5.7.2 Node.js 배치 작업 (node-cron)

import cron from 'node-cron';
import { WIAClient } from '@wia/art-preservation-sdk';

const client = new WIAClient({ apiKey: process.env.WIA_API_KEY });

// 매일 새벽 2시: 무결성 검증
cron.schedule('0 2 * * *', async () => {
  console.log('Starting daily integrity check...');

  const artworks = await client.artworks.list({ limit: 10000 });
  let passed = 0, failed = 0;

  for (const artwork of artworks) {
    try {
      const result = await client.artworks.verify(artwork.id);
      if (result.isValid) {
        passed++;
      } else {
        failed++;
        console.error(`Failed: ${artwork.id}`);
        await client.events.log({
          event_type: 'integrity_check_failed',
          artwork_id: artwork.id
        });
      }
    } catch (error) {
      console.error(`Error verifying ${artwork.id}:`, error.message);
    }
  }

  console.log(`Integrity check complete: ${passed} passed, ${failed} failed`);

  // 요약 이메일 전송
  await sendDailySummary({ passed, failed, total: artworks.length });
});

// 매주 일요일: 백업
cron.schedule('0 3 * * 0', async () => {
  console.log('Starting weekly backup...');

  const artworks = await client.artworks.list({ limit: 10000 });

  for (const artwork of artworks) {
    await backupToS3(artwork.id);
    await backupToIPFS(artwork.id);
  }

  console.log('Weekly backup complete');
});
