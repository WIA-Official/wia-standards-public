6.1 탈중앙화 보존의 필요성

Phase 3 Protocol은 중앙 서버에 대한 의존성을 제거하고, P2P(Peer-to-Peer) 네트워크를 통한 탈중앙화 보존을 가능하게 합니다. 단일 기관이나 회사가 파산하거나 서비스를 종료해도 작품이 네트워크에 계속 존재합니다.

🎯 Phase 3의 목표
  • 영구성: IPFS/Arweave로 콘텐츠 주소 지정 및 영구 저장
  • 탈중앙화: 단일 장애점(Single Point of Failure) 제거
  • 실시간 동기화: WebSocket으로 여러 노드 간 즉시 동기화
  • 고성능: gRPC를 통한 빠른 데이터 전송
  • 검열 저항: 중앙 권한 없이 콘텐츠 보존

6.2 IPFS (InterPlanetary File System)

IPFS는 콘텐츠 주소 지정(Content Addressing)을 사용하는 분산 파일 시스템입니다. 파일 위치(URL)가 아닌 파일 내용(해시)으로 식별합니다.

개념 설명 예시
CID (Content Identifier) 파일 내용의 암호화 해시 QmT5NvU...
Pinning 파일을 로컬에 유지 ipfs pin add QmT5NvU...
DHT (Distributed Hash Table) 파일 위치 검색 누가 이 파일을 가지고 있나?
Gateway HTTP로 IPFS 접근 https://ipfs.io/ipfs/QmT5NvU...

6.2.1 IPFS로 작품 저장하기

// JavaScript (ipfs-http-client)
import { create } from 'ipfs-http-client';

const ipfs = create({ url: 'https://ipfs.infura.io:5001/api/v0' });

// 작품 업로드
const artwork = {
  path: 'artwork/index.html',
  content: await fs.readFile('./dist/index.html')
};

const result = await ipfs.add(artwork);
console.log(`IPFS CID: ${result.cid}`);
// QmT5NvUtoM5pQ8Y28v8VgEz6GEJLj3RbJZ9M2F5zNX9aB3

// WIA 메타데이터 업데이트
const metadata = {
  artwork: { id: 'art-2024-001' },
  preservation: {
    primary_storage: 'ipfs',
    ipfs_cid: result.cid.toString(),
    ipfs_gateways: [
      'https://ipfs.io/ipfs/' + result.cid,
      'https://gateway.pinata.cloud/ipfs/' + result.cid
    ]
  }
};

// 영구 핀 서비스 (Pinata, NFT.Storage)
await pinataSDK.pinByHash(result.cid);

6.2.2 IPFS 조회 및 검증

// IPFS에서 파일 읽기
const chunks = [];
for await (const chunk of ipfs.cat('QmT5NvUtoM5...')) {
  chunks.push(chunk);
}
const fileContent = Buffer.concat(chunks);

// 체크섬 검증
import crypto from 'crypto';
const calculatedHash = crypto.createHash('sha256').update(fileContent).digest('hex');

if (calculatedHash === metadata.preservation.checksum_sha256) {
  console.log('✅ IPFS 파일 무결성 검증 완료');
} else {
  console.error('❌ 체크섬 불일치!');
}

6.3 Arweave: 영구 저장소

Arweave는 "영원히 저장"을 목표로 하는 블록체인 기반 스토리지입니다. 한 번 지불하면 최소 200년간 저장을 보장합니다.

특징 IPFS Arweave 전통적 서버
비용 모델 핀 서비스 월 구독 1회 지불 월간 요금
영구성 핀 유지 필요 ✅ 200년+ 보장 서비스 종료 시 손실
변조 불가능 ✅ CID로 보장 ✅ 블록체인 기록 ❌ 변조 가능
검색 속도 중간-빠름 중간 빠름
1GB 저장 비용 ~$2/월 ~$5 (1회) ~$0.10/월

6.3.1 Arweave에 작품 업로드

// Arweave SDK
import Arweave from 'arweave';
import fs from 'fs';

const arweave = Arweave.init({
  host: 'arweave.net',
  port: 443,
  protocol: 'https'
});

// 지갑 로드 (AR 토큰 필요)
const wallet = JSON.parse(await fs.readFile('wallet.json', 'utf8'));

// 트랜잭션 생성
const data = await fs.readFile('./artwork.html');
const transaction = await arweave.createTransaction({ data }, wallet);

// WIA 메타데이터를 Arweave 태그로 추가
transaction.addTag('Content-Type', 'text/html');
transaction.addTag('WIA-Standard', '1.0');
transaction.addTag('WIA-Artwork-ID', 'art-2024-001');
transaction.addTag('Artist', 'Jane Doe');
transaction.addTag('Creation-Date', '2024-01-15');

// 서명 및 업로드
await arweave.transactions.sign(transaction, wallet);
await arweave.transactions.post(transaction);

console.log(`Arweave TX: ${transaction.id}`);
console.log(`URL: https://arweave.net/${transaction.id}`);

6.4 Bit Rot 감지 및 자동 복구

Bit rot은 저장 매체의 물리적 노화로 데이터가 손상되는 현상입니다. WIA-ART-012는 정기적 스캔과 자동 복구를 지원합니다. 

// 월간 Bit Rot 스캔 스크립트
async function bitRotScan() {
  const artworks = await client.artworks.list({ limit: 1000 });

  for (const artwork of artworks) {
    const localPath = `/archive/${artwork.id}`;

    // 로컬 파일 체크섬 계산
    const localChecksum = await calculateSHA256(localPath);

    if (localChecksum !== artwork.preservation.checksum_sha256) {
      console.error(`❌ Bit rot detected: ${artwork.id}`);

      // IPFS에서 복구 시도
      if (artwork.preservation.ipfs_cid) {
        await restoreFromIPFS(artwork.preservation.ipfs_cid, localPath);
        console.log(`✅ Restored from IPFS`);
      }

      // 복구 로그
      await client.events.log({
        event_type: 'bit_rot_detected_and_recovered',
        artwork_id: artwork.id,
        timestamp: new Date().toISOString()
      });
    }
  }
}

6.6 gRPC: 고성능 데이터 전송

gRPC는 Google에서 개발한 고성능 RPC 프레임워크입니다. Protocol Buffers를 사용하여 REST API보다 최대 7배 빠른 데이터 전송을 제공합니다.

6.6.1 REST vs gRPC 성능 비교

측정 항목 REST (JSON) gRPC (Protobuf) 개선율
요청 크기 500 bytes 150 bytes 70% 감소
응답 크기 2,500 bytes 800 bytes 68% 감소
지연 시간 50ms 15ms 70% 감소
처리량 1,000 req/s 7,000 req/s 7배 증가

6.7 P2P 네트워크 및 LibP2P

LibP2P는 IPFS의 기반이 되는 모듈식 P2P 네트워킹 라이브러리입니다. WIA-ART-012는 LibP2P를 사용하여 분산 보존 네트워크를 구축할 수 있습니다.

6.7.1 분산 검증 프로토콜

여러 피어가 협력하여 작품 무결성을 검증합니다:

  • 1단계: 노드 A가 검증 요청 브로드캐스트
  • 2단계: 노드 B, C, D가 각자 체크섬 계산
  • 3단계: 다수결로 정상 체크섬 결정
  • 4단계: 불일치 노드는 자동 복구

6.9 콘텐츠 주소 지정의 철학

IPFS와 Arweave가 사용하는 콘텐츠 주소 지정은 디지털 보존의 패러다임 전환입니다.

측면 위치 주소 지정 (HTTP) 콘텐츠 주소 지정 (IPFS)
식별자 URL (위치) CID (내용 해시)
변조 감지 불가능 자동 (해시 불일치)
중복 제거 수동 자동 (같은 내용 = 같은 CID)
중앙화 서버 의존 분산 네트워크
404 오류 서버 종료 시 발생 1개 노드라도 있으면 접근 가능
🌐 콘텐츠 주소 지정의 이점
  • 영구성: 내용이 바뀌지 않으면 주소도 바뀌지 않음
  • 검증 가능: 받은 데이터가 요청한 것과 일치하는지 자동 검증
  • 효율성: 중복 저장 없이 동일 콘텐츠 재사용
  • 분산화: 단일 실패 지점 제거

6.9.1 실제 사례: NFT.Storage

NFT.Storage는 Protocol Labs에서 제공하는 무료 IPFS 핀 서비스입니다. NFT 메타데이터와 이미지를 영구히 저장합니다.

  • 무료 저장: 최대 31GB 무료 (NFT 용도)
  • 자동 핀: Filecoin 네트워크에 자동 백업
  • 고가용성: 여러 IPFS 게이트웨이 제공
  • API 접근: 간단한 REST API

6.11 Filecoin: 경제적 인센티브 기반 저장

Filecoin은 IPFS의 인센티브 레이어로, 스토리지 제공자에게 FIL 토큰으로 보상합니다. 이는 장기 보존을 경제적으로 지속 가능하게 만듭니다.

특징 IPFS (무료 핀) Filecoin
비용 핀 서비스 월 구독 스토리지 거래 (FIL)
보장 서비스 지속성에 의존 스마트 컨트랙트 보장
검증 수동 자동 (Proof-of-Replication)
인센티브 없음 FIL 보상

6.11.1 Filecoin 거래 생성

Filecoin 네트워크에 데이터를 저장하는 과정:

  1. 데이터 준비: CAR (Content Addressable aRchive) 파일 생성
  2. 거래 제안: 스토리지 제공자에게 거래 제안
  3. 거래 체결: 스마트 컨트랙트로 조건 합의
  4. 데이터 전송: 제공자에게 데이터 전송
  5. 증명 제출: 정기적으로 저장 증명 제출
💰 비용 예시 (2024년 기준)
  • 1GB, 1년 저장: 약 0.001 FIL (~$0.005)
  • 100GB, 3년 저장: 약 0.3 FIL (~$1.50)
  • 1TB, 10년 저장: 약 10 FIL (~$50)

참고: AWS Glacier Deep Archive와 비슷한 가격대, 하지만 탈중앙화
