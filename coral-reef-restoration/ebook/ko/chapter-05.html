5.1 API 아키텍처

WIA-REEF API는 산호초 모니터링 데이터 접근, 평가 제출, 복원 노력 조정을 위한 종합 인터페이스를 제공합니다. API는 JSON 페이로드와 표준 HTTP 메서드를 사용하는 RESTful 원칙을 따릅니다.

// API 구성
{
  "base_url": "https://api.wia-reef.org/v1",
  "authentication": {
    "methods": ["api_key", "oauth2"],
    "header": "Authorization: Bearer {token}"
  },
  "rate_limits": {
    "standard": "1000 요청/시간",
    "premium": "10000 요청/시간",
    "streaming": "구독 내 무제한"
  },
  "versioning": "URL 경로 (v1, v2)",
  "formats": ["application/json", "application/geo+json"]
}

5.2 핵심 엔드포인트

5.2.1 산호초 사이트

GET /sites - 필터링으로 모든 산호초 사이트 목록
GET /sites/{site_id} - 특정 사이트 상세 정보
POST /sites - 새 산호초 사이트 등록

5.2.2 건강 평가

5.4 백화 예측

엔드포인트 설명 응답
/predict/bleaching/{site_id} 7일 백화 예보 위험 수준, DHW 예측
/predict/dhw/{site_id} 누적 열 스트레스 주(DHW) 현재 DHW, 12주 이력
/alerts/bleaching 활성 백화 경보 영향받는 사이트, 심각도
/predict/recovery/{site_id} 회복 잠재력 추정 타임라인, 성공 확률

5.5 복원 API

GET /restoration/projects - 복원 프로젝트 목록
POST /restoration/projects - 새 프로젝트 생성
POST /restoration/projects/{id}/fragments - 파편 배치 기록
PUT /restoration/fragments/{id}/monitoring - 파편 상태 업데이트

5.7 장 요약

핵심 내용

  • WIA-REEF API는 JSON 페이로드를 사용하는 RESTful 설계 따름
  • 핵심 엔드포인트는 사이트, 평가, 복원 커버
  • 실시간 모니터링은 라이브 센서 데이터 및 경보 제공
  • 백화 예측은 DHW 및 온도 예보 사용
  • OAuth 2.0 및 API 키가 유연한 인증 제공
