6.1 스트리밍 아키텍처

WIA-REEF 프로토콜 계층은 수중 센서, 부이, 모니터링 스테이션에서 중앙 시스템 및 구독자 애플리케이션으로의 실시간 데이터 스트리밍을 가능하게 합니다. 이 아키텍처는 지속적인 모니터링과 이벤트 기반 경보를 모두 지원합니다.

// 스트리밍 아키텍처
{
  "protocols": {
    "mqtt": {
      "broker": "mqtt.wia-reef.org",
      "port": 8883,
      "tls": true,
      "qos_levels": [0, 1, 2],
      "use_case": "IoT 센서, 저대역폭"
    },
    "websocket": {
      "endpoint": "wss://stream.wia-reef.org",
      "use_case": "웹 대시보드, 실시간 시각화"
    },
    "sse": {
      "endpoint": "https://api.wia-reef.org/v1/events",
      "use_case": "서버 전송 이벤트, 경보"
    }
  },
  "message_format": "스키마 검증을 포함한 JSON",
  "compression": "대용량 페이로드용 gzip"
}

6.2 MQTT 토픽 구조

토픽 패턴 설명 QoS
wia/reef/{site_id}/sensors/+ 사이트의 모든 센서 0
wia/reef/{site_id}/sensors/temperature 온도 측정값 0
wia/reef/{site_id}/alerts/# 사이트의 모든 경보 1
wia/reef/global/alerts/critical 글로벌 긴급 경보 2
wia/reef/{site_id}/restoration/+ 복원 업데이트 1 
// MQTT 센서 메시지
{
  "topic": "wia/reef/reef-aus-gbr-001/sensors/temperature",
  "payload": {
    "sensor_id": "temp-001",
    "site_id": "reef-aus-gbr-001",
    "timestamp": "2024-03-15T14:30:00.000Z",
    "value": 28.4,
    "unit": "celsius",
    "depth_m": 5,
    "quality": "good",
    "battery_pct": 85
  },
  "qos": 0,
  "retain": false
}

6.5 오프라인 운영

현장 운영

많은 산호초 모니터링 활동이 신뢰할 수 있는 연결 없이 원격 지역에서 수행됩니다. WIA-REEF 프로토콜은 연결이 복원되면 자동 동기화와 함께 오프라인 데이터 수집을 지원합니다.

// 오프라인 동기화 프로토콜
{
  "offline_mode": {
    "local_storage": "IndexedDB / SQLite",
    "queue_limit": "10000 레코드",
    "conflict_resolution": "server_wins_with_merge",
    "sync_on_connect": true
  },
  "sync_process": {
    "1_connect": "보안 연결 수립",
    "2_auth": "필요시 토큰 갱신",
    "3_pull": "보류 중인 업데이트 다운로드",
    "4_merge": "로컬에서 충돌 해결",
    "5_push": "대기 중인 레코드 업로드",
