4.1 산호초 사이트 스키마

산호초 사이트 스키마는 모든 WIA-REEF 데이터의 기초를 형성합니다. 모든 모니터링 활동, 건강 평가, 복원 프로젝트는 이 표준화된 형식을 사용하여 정의된 특정 산호초 사이트에 연결됩니다.

// 완전한 산호초 사이트 스키마
{
  "$schema": "https://wia.org/schemas/reef-site-v1.json",
  "type": "object",
  "required": ["id", "name", "location", "reef_type"],
  "properties": {
    "id": {
      "type": "string",
      "format": "uuid",
      "description": "산호초 사이트의 고유 식별자"
    },
    "name": {
      "type": "string",
      "maxLength": 200
    },
    "location": {
      "type": "object",
      "properties": {
        "type": { "const": "Point" },
        "coordinates": {
          "type": "array",
          "items": { "type": "number" },
          "minItems": 2,
          "maxItems": 3
        }
      }
    },
    "reef_type": {
      "enum": ["fringing", "barrier", "atoll", "patch", "platform"]
    },
    "area_km2": { "type": "number", "minimum": 0 },
    "depth_range": {
      "type": "object",
      "properties": {
        "min_m": { "type": "number" },
        "max_m": { "type": "number" }
      }
    },
    "protection_status": {
      "enum": ["mpa", "reserve", "sanctuary", "unprotected", "other"]
    },
    "country_code": { "type": "string", "pattern": "^[A-Z]{2}$" },
    "ecoregion": { "type": "string" }
  }
}

4.5 복원 추적

데이터 유형 필수 필드 업데이트 빈도
프로젝트 정의 id, site, method, start_date, targets 프로젝트 시작
파편 배치 fragment_id, species, source, deployment_date 배치 시마다
생존 모니터링 status, size, condition, photo_id 월간
성장 추적 measurements, growth_rate, tissue_mortality 분기별
번식 이벤트 spawning_observed, recruits_counted 관찰 시

4.6 데이터 검증 규칙

품질 보증

모든 WIA-REEF 데이터는 제출 전에 검증 규칙을 통과해야 합니다. 이는 글로벌 네트워크 전체에서 데이터 품질과 상호운용성을 보장합니다.

  • 범위 검사: 온도 0-40°C, pH 7.0-9.0, 염도 0-45 PSU
  • 일관성: 백분율 합계는 100%여야 함 (±0.5%)
  • 시간: 타임스탬프는 유효한 ISO 8601, 미래 날짜 불가
  • 공간: 좌표는 유효하고 산호초 가능 구역 내
  • 참조: 모든 site_id는 기존 산호초 사이트 참조 필요

4.7 장 요약

핵심 내용

  • 산호초 사이트 스키마는 모든 WIA-REEF 데이터의 기초
  • 건강 평가는 산호 피복률, 백화, 기질 추적
  • 생물다양성 데이터는 산호, 어류, 무척추동물, 지수 포함
  • 수질 지표는 물리적, 영양염, 탄산염 매개변수 커버
  • 복원 추적은 배치부터 번식까지 파편 추적
