4.1 Reef Site Schema

The reef site schema forms the foundation of all WIA-REEF data. Every monitoring activity, health assessment, and restoration project is linked to a specific reef site defined using this standardized format.

// Complete Reef Site Schema
{
  "$schema": "https://wia.org/schemas/reef-site-v1.json",
  "type": "object",
  "required": ["id", "name", "location", "reef_type"],
  "properties": {
    "id": {
      "type": "string",
      "format": "uuid",
      "description": "Unique identifier for the reef site"
    },
    "name": {
      "type": "string",
      "maxLength": 200
    },
    "location": {
      "type": "object",
      "properties": {
        "type": { "const": "Point" },
        "coordinates": {
          "type": "array",
          "items": { "type": "number" },
          "minItems": 2,
          "maxItems": 3
        }
      }
    },
    "reef_type": {
      "enum": ["fringing", "barrier", "atoll", "patch", "platform"]
    },
    "area_km2": { "type": "number", "minimum": 0 },
    "depth_range": {
      "type": "object",
      "properties": {
        "min_m": { "type": "number" },
        "max_m": { "type": "number" }
      }
    },
    "protection_status": {
      "enum": ["mpa", "reserve", "sanctuary", "unprotected", "other"]
    },
    "country_code": { "type": "string", "pattern": "^[A-Z]{2}$" },
    "ecoregion": { "type": "string" }
  }
}

4.6 Data Validation Rules

Quality Assurance

All WIA-REEF data must pass validation rules before submission. This ensures data quality and interoperability across the global network.

  • Range Checks: Temperature 0-40°C, pH 7.0-9.0, salinity 0-45 PSU
  • Consistency: Percentages must sum to 100% (±0.5%)
  • Temporal: Timestamps must be valid ISO 8601, not future-dated
  • Spatial: Coordinates must be valid and within reef-probable zones
  • Referential: All site_ids must reference existing reef sites
