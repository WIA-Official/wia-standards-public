6.1 Streaming Architecture

The WIA-REEF protocol layer enables real-time data streaming from underwater sensors, buoys, and monitoring stations to central systems and subscriber applications. This architecture supports both continuous monitoring and event-driven alerts.

// Streaming Architecture
{
  "protocols": {
    "mqtt": {
      "broker": "mqtt.wia-reef.org",
      "port": 8883,
      "tls": true,
      "qos_levels": [0, 1, 2],
      "use_case": "IoT sensors, low bandwidth"
    },
    "websocket": {
      "endpoint": "wss://stream.wia-reef.org",
      "use_case": "Web dashboards, real-time viz"
    },
    "sse": {
      "endpoint": "https://api.wia-reef.org/v1/events",
      "use_case": "Server-sent events, alerts"
    }
  },
  "message_format": "JSON with schema validation",
  "compression": "gzip for large payloads"
}

6.2 MQTT Topic Structure

Topic Pattern Description QoS
wia/reef/{site_id}/sensors/+ All sensors for a site 0
wia/reef/{site_id}/sensors/temperature Temperature readings 0
wia/reef/{site_id}/alerts/# All alerts for a site 1
wia/reef/global/alerts/critical Global critical alerts 2
wia/reef/{site_id}/restoration/+ Restoration updates 1

6.5 Offline Operation

Field Operations

Many reef monitoring activities occur in remote locations without reliable connectivity. The WIA-REEF protocol supports offline data collection with automatic synchronization when connectivity is restored.

// Offline Sync Protocol
{
  "offline_mode": {
    "local_storage": "IndexedDB / SQLite",
    "queue_limit": "10000 records",
    "conflict_resolution": "server_wins_with_merge",
    "sync_on_connect": true
  },
  "sync_process": {
    "1_connect": "Establish secure connection",
    "2_auth": "Refresh tokens if needed",
    "3_pull": "Download pending updates",
    "4_merge": "Resolve conflicts locally",
    "5_push": "Upload queued records",
    "6_confirm": "Acknowledge successful sync"
  },
  "conflict_types": {
    "create_create": "Merge with UUIDs",
    "update_update": "Timestamp comparison",
    "delete_update": "Flag for review"
  }
}
