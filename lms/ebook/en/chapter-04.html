WIA-EDU-009 | Chapter 4

Assignments & Assessments

Measuring Learning and Providing Feedback

Assignment Types

WIA-EDU-009 supports diverse assignment formats to accommodate various learning objectives and assessment strategies.

File Upload Assignments

  • Single File: Essays, reports, presentations
  • Multiple Files: Projects with supporting materials
  • Accepted Formats: PDF, DOCX, PPTX, images, videos, code files
  • File Size Limits: Configurable per assignment (default: 100MB)
  • Virus Scanning: Automatic malware detection

Online Text Submissions

  • Rich text editor with formatting tools
  • Inline images and multimedia embeds
  • Mathematical equation support (LaTeX, MathJax)
  • Version history and draft saving
  • Plagiarism detection integration (Turnitin, SafeAssign)

External Tool Assignments

  • LTI-integrated applications (Google Docs, Padlet, H5P)
  • Automatic grade passback
  • Embedded submission interface
  • Support for specialized tools (coding environments, simulations)

Group Assignments

  • Self-Selection: Students choose team members
  • Instructor-Assigned: Manual or algorithmic grouping
  • Collaborative Workspaces: Shared documents and resources
  • Individual Grading Options: Same grade for all or differentiated
  • Peer Evaluation: Team member contribution ratings

Quiz & Test Builder

Question Types

  • Multiple Choice: Single or multiple correct answers
  • True/False: Binary response questions
  • Fill-in-the-Blank: Short answer with auto-grading
  • Matching: Pair items from two lists
  • Essay: Long-form written responses
  • Numerical: Math problems with formula-based grading
  • Hotspot: Click correct area on an image
  • File Upload: Submit supporting documents

Quiz Settings

  • Time Limits: Enforced duration for test completion
  • Attempts: Number of tries allowed (1 to unlimited)
  • Question Randomization: Different order for each student
  • Answer Randomization: Shuffle multiple choice options
  • Question Banks: Random selection from pools
  • Access Restrictions: IP restrictions, browser lockdown

Feedback Options

  • Immediate feedback after each question
  • Deferred feedback after quiz submission
  • Show correct answers (configurable timing)
  • Detailed explanations for each answer
  • Partial credit for multiple-answer questions

Submission Management

Student Submission Workflow

  1. Access Assignment: Click from course page or calendar
  2. Review Instructions: Read requirements, rubric, deadline
  3. Upload/Enter Work: Submit file(s) or online text
  4. Confirm Submission: Receive confirmation email/notification
  5. Track Status: See submission time and status

Late Submission Policies

  • Acceptance Window: Allow late submissions for X days
  • Automatic Deductions: Percentage or point penalties per day/hour
  • Hard Cutoffs: No submissions accepted after date
  • Grace Periods: Short buffer time without penalty
  • Instructor Overrides: Manual extensions for individual students

Resubmission Options

  • Allow students to replace submissions before deadline
  • Version tracking (keep all submissions)
  • Instructor can unlock for revision after initial grading
  • Resubmission deadline separate from original

Peer Review System

Benefits of Peer Review:
  • Develops critical thinking and evaluation skills
  • Exposes students to diverse perspectives and approaches
  • Reduces instructor grading workload
  • Promotes reflective learning through giving feedback

Peer Review Workflow

  1. Submission Phase: Students submit original work
  2. Assignment Phase: System assigns peers (anonymous or named)
  3. Review Phase: Students evaluate assigned submissions
  4. Grading Phase: Instructor grades both work and reviews
  5. Release Phase: Feedback shared with original authors

Peer Review Configuration

  • Number of Reviews: Each student reviews 2-5 peers
  • Anonymity: Anonymous, partially anonymous, or named
  • Review Criteria: Rubrics or open-ended feedback
  • Instructor Moderation: Approve reviews before release
  • Self-Assessment: Students also review their own work

Plagiarism Detection

Integration with Detection Services

  • Turnitin: Comprehensive plagiarism and AI detection
  • SafeAssign: Blackboard's originality checking
  • Unicheck: Real-time similarity reports
  • Grammarly: Writing assistance with plagiarism checking

Similarity Reports

  • Percentage match with sources
  • Highlighted matching text
  • Source identification and links
  • Exclusion options (quotes, bibliography, small matches)
  • Student access to reports (optional)

Automated Grading

Auto-Graded Question Types

  • Multiple Choice/True-False: Instant scoring
  • Fill-in-the-Blank: Exact match or pattern matching
  • Numerical: Answer range or formula-based
  • Matching: Correct pairings
  • Code Submissions: Unit test execution (CS courses)

Partial Credit Logic

  • Multiple-answer questions (e.g., 3 out of 4 correct = 75%)
  • Numerical ranges (closer = more points)
  • Configurable partial credit percentages
  • Negative scoring for incorrect multiple-choice selections

Manual Grading Interface

SpeedGrader Features

  • Inline Annotation: Comments directly on PDF/document submissions
  • Audio/Video Feedback: Record personalized feedback
  • Rubric Integration: Click-to-grade with criteria
  • Keyboard Shortcuts: Rapid navigation between students
  • Blind Grading: Hide student names to reduce bias

Grading Workflow Optimization

  • Grade assignments by question (all Q1s, then Q2s, etc.)
  • Saved comments bank for common feedback
  • Batch actions (same score for multiple students)
  • Filter by section, submission status, or score range
  • Progress tracking (45 of 120 graded)

Assignment Analytics

Submission Statistics

  • On-time vs. late submission rates
  • Average time to complete
  • Resubmission frequency
  • Missing assignment patterns

Performance Insights

  • Class average and distribution
  • Question-level difficulty analysis
  • Most commonly missed questions
  • Correlation with overall course performance

Best Practices for Effective Assignments

Assignment Design Principles:
  1. Align assignments with clear learning objectives
  2. Provide detailed instructions and examples
  3. Use rubrics to communicate expectations
  4. Offer multiple submission opportunities when appropriate
  5. Balance formative (low-stakes) and summative (high-stakes) assessments
  6. Give timely, actionable feedback
  7. Vary assignment types to assess different skills
  8. Build in checkpoints for large projects
  9. Consider academic integrity in design
  10. Regularly review and update assignments based on student performance
