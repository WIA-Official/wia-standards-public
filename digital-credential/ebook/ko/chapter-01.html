Chapter 1: 디지털 자격증 소개

디지털 자격증이란?

디지털 자격증은 암호학적으로 검증 가능한 성과, 자격 및 인증을 전자적으로 표현한 것입니다. 전통적인 종이 증명서와 달리 디지털 자격증은 변조 방지, 즉시 검증 가능하며 국경과 플랫폼을 넘어 안전하게 공유할 수 있습니다.

WIA-EDU-011 표준은 W3C 검증 가능한 자격증과 블록체인 기술을 기반으로 교육 분야에서 디지털 자격증을 발급, 저장, 검증 및 공유하기 위한 포괄적인 프레임워크를 정의합니다.

교육 자격증의 진화

전통적인 종이 증명서 (1800년대-2000년대): 특수 용지에 인쇄된 물리적 문서로, 전화나 우편을 통한 수동 검증이 필요하며, 분실, 손상 및 위조에 취약합니다.

PDF 증명서 (2000년대-2010년대): 이메일로 전송되거나 다운로드되는 디지털 파일로, 배포는 쉽지만 위조도 쉽고, 검증 메커니즘이 내장되어 있지 않습니다.

블록체인 기반 검증 가능한 자격증 (2020년대-현재): 암호학적으로 서명되고 변조 방지되며, 누구나 어디서나 즉시 검증 가능하고, 블록체인에 고정되어 불변 기록을 제공하며, W3C 검증 가능한 자격증 표준을 기반으로 하고, 선택적 공개로 프라이버시를 보호합니다.

주요 구성 요소

🎓 자격증 주체

자격증이 대상으로 하는 사람 또는 단체입니다. 개인 정보를 노출하지 않고 소유자를 고유하게 식별하는 분산 식별자(DID)를 포함합니다.

🏛️ 발급자

자격증을 생성하고 서명하는 기관 또는 조직입니다. 대학, 인증 기관 및 교육 제공자가 모두 발급자가 될 수 있습니다.

✅ 검증자

자격증의 진위를 확인해야 하는 모든 당사자입니다. 고용주, 다른 교육 기관, 이민 당국 또는 면허 위원회가 될 수 있습니다.

디지털 자격증의 이점

학생과 졸업생을 위해: 디지털 지갑에서 자격증을 소유하고 제어하며, 누구에게나 즉시 공유하고, 필요한 정보만 선택적으로 공개하며, 영구적으로 보관할 수 있습니다.

교육 기관을 위해: 자격증 발급 비용을 80-90% 절감하고, 학위 사기를 제거하며, 전 세계적으로 인정받고, 자동화된 발급 및 검증을 제공합니다.

고용주를 위해: 몇 주가 아닌 몇 초 만에 자격증을 검증하고, 암호학적 진위성 증명을 받으며, 제3자 검증 서비스를 제거하여 비용을 절감합니다.

철학: 弘益人間

WIA-EDU-011 표준은 교육 성과를 보편적으로 접근 가능하고 검증 가능하며 이동 가능하게 만들어 弘益人間(널리 인간을 이롭게 하라)의 철학을 구현합니다.

참조 표준 및 적용 지침 (Chapter 01)

이 표준은 다음 ALLOW 출처를 단일 진실원으로 삼아 데이터 교환·서명· 감사 증적 인터페이스를 제공합니다.

  • OpenAPI Specification 3.1 — REST API 정의
  • JSON Schema 2020-12 — 스키마 검증
  • ISO/IEC 27001:2022 — 정보보안 경영시스템
  • ISO/IEC 17065:2012 — 제품 인증 기관
  • CycloneDX 1.5 / SPDX 2.3 — 소프트웨어 BOM
  • Sigstore (DSSE 봉투, Rekor 투명성 로그) — 서명 앵커링
  • in-toto Attestation Framework 1.0 — 빌드 출처 검증

실무 적용 체크리스트

  • 매니페스트 무결성: 모든 발급은 SHA-256 또는 Sigstore 서명을 부착해 변조를 차단합니다. 발급 후 변경은 새 매니페스트로 처리 되며 이전 매니페스트는 supersede 마킹과 함께 보존됩니다.
  • 보관 기간: ISO/IEC 17065:2012 감사 기대치를 충족하기 위해 7년 이상의 보관 정책을 명시합니다. 관할 규정이 더 길면 그것을 따릅니다.
  • 버전 관리: 정정·확장은 시맨틱 버저닝(SemVer 2.0.0)을 따릅니다. 필수 요건의 추가는 마이너, 호환 단절은 메이저 버전 갱신을 트리거합니다. 메이저 변경은 최소 90일 병행 호환 기간을 거치며 그 기간 동안 신·구 두 버전 모두 WIA 등록부에 Stable로 표기 됩니다.
  • 부정결과 보고: 누락된 지표는 사유와 함께 매니페스트에 기록 되어야 합니다. 사유가 없는 누락은 감사 시 실패로 간주됩니다.

거버넌스 및 변경 관리

모든 변경 제안과 정정 요청은 GitHub Issues 트래커에 digital-credential 레이블로 접수됩니다. WIA 표준 워킹그룹은 매 마이너 릴리스 사이클의 시작 시점에 열린 이슈를 검토하고, 결정 로그를 릴리스 노트와 함께 공개합니다.

弘益人間 — 기술이 인간 안전과 편의를 동시에 충족하도록 설계되었 습니다.

실무 시나리오 및 참조 토폴로지

구현체의 운용 토폴로지는 다음 4가지 중 하나로 선언됩니다. 각 토폴로지는 참조 테스트 벡터를 포함하며 매니페스트의 profile 필드로 명시됩니다.

  1. 단일 테넌트 엣지: 조직당 1개 런타임. 공유 상태 없음. 초기 도입자에게 권장.
  2. 멀티 테넌트 게이트웨이: 1개 런타임이 헤더 기반 격리로 다중 테넌트 서비스. OAuth 2.1 (RFC 9700) 신원 공급자 필수.
  3. 연합 메시: 다중 런타임이 디렉토리 서비스에 매니페스트를 발행하고 상호 피어링. 디렉토리는 집계 인덱스를 별도 키로 서명.
  4. 에어갭 배치: 디렉토리 서비스 라이브 연결 없음. 서명된 증적 패키지를 out-of-band 채널로 전송. 규제 환경 배포에 사용.

매니페스트 구조

매니페스트는 표준 슬러그, 버전, 구현체 빌드 다이제스트, SBOM URL, 프로파일 이름, 발급 시각, Sigstore DSSE 서명을 포함합니다. 매니페스트는 Sigstore Rekor에 anchoring되며 매 분기마다 재발급되어 환경 드리프트를 감지할 수 있게 합니다.

마이그레이션 가이드

이전 메이저 버전에서 새 메이저 버전으로의 마이그레이션은 최소 90일 병행 호환 기간을 거칩니다. 그 기간 동안 신·구 두 버전 모두 WIA 등록부에 Stable로 표기되며, 새 버전의 매니페스트 키는 이전 키 이력에 등록되어 과거 매니페스트 검증을 보장합니다. 마이그레이션 노트는 GitHub Releases와 WIA 등록부 양쪽에 동시 공시됩니다.
