Chapter 8: Implementation Guide

Getting Started with WIA-EDU-011

This chapter provides step-by-step guidance for institutions, developers, and verifiers to implement WIA-EDU-011 digital credentials.

For Educational Institutions

Phase 1: Assessment and Planning (2-4 weeks)

Week 1-2: Stakeholder Assessment
  • Identify key stakeholders (registrar, IT, legal, communications)
  • Review current credential issuance processes
  • Estimate annual credential volume
  • Calculate ROI and cost savings
Week 3-4: Technical Planning
  • Evaluate integration points with SIS
  • Choose blockchain network (Polygon recommended)
  • Select implementation partner or DIY
  • Plan pilot program scope

Phase 2: Setup and Integration (4-8 weeks)

// Install WIA SDK
npm install @wia/credential-issuer

// Initialize issuer
import { CredentialIssuer } from '@wia/credential-issuer';

const issuer = new CredentialIssuer({
  did: 'did:wia:edu:university-name',
  privateKey: process.env.ISSUER_PRIVATE_KEY,
  blockchain: 'polygon',
  apiKey: process.env.WIA_API_KEY
});

// Issue a credential
const credential = await issuer.issueCredential({
  recipientDID: 'did:wia:student:alice',
  credentialType: 'BachelorDegree',
  degree: {
    type: 'Bachelor of Science',
    field: 'Computer Science',
    gpa: '3.85',
    graduationDate: '2024-06-15'
  }
});

Phase 3: Pilot Program (2-3 months)

  • Issue credentials to graduating class
  • Provide wallet app and training
  • Gather feedback from students and employers
  • Refine processes based on learnings
  • Measure success metrics

Phase 4: Full Deployment (Ongoing)

  • Roll out to all programs and departments
  • Offer credentials to alumni upon request
  • Integrate with transcript services
  • Market digital credentials to prospective students

For Developers

Issuer Implementation

// Complete issuer setup
import { CredentialIssuer, DIDManager } from '@wia/credential-issuer';

// 1. Create institutional DID
const didManager = new DIDManager();
const { did, privateKey } = await didManager.createDID({
  method: 'wia',
  type: 'education',
  organization: 'University Name'
});

// 2. Register DID in trust registry
await didManager.registerDID(did, {
  name: 'University Name',
  country: 'US',
  accreditation: ['CHEA', 'WASC'],
  website: 'https://university.edu'
});

// 3. Issue credentials
const issuer = new CredentialIssuer({
  did: did,
  privateKey: privateKey,
  blockchain: 'polygon',
  apiKey: process.env.WIA_API_KEY
});

const credential = await issuer.issueCredential({
  recipientEmail: 'student@email.com',
  credentialType: 'BachelorDegree',
  metadata: {
    degree: 'Bachelor of Science',
    field: 'Computer Science',
    gpa: '3.85'
  }
});

// 4. Deliver to student
await issuer.deliverCredential(credential, {
  email: true,
  wallet: true,
  downloadLink: true
});

For Employers and Verifiers

Setup Process

  1. Create account at verify.wiastandards.com
  2. Download WIA Verifier app (iOS/Android/Web)
  3. Request credentials from candidates via email
  4. Scan QR codes or verify links
  5. Review verification results

Bulk Verification

// Verify multiple credentials
const verifier = new CredentialVerifier({
  apiKey: process.env.WIA_API_KEY
});

const credentials = [
  /* array of credentials */
];

const results = await verifier.verifyBatch(credentials);

results.forEach(result => {
  console.log(`${result.holder}: ${result.verified ? '✅' : '❌'}`);
});

Best Practices

Security

  • Store private keys in hardware security modules (HSM)
  • Use key rotation every 12-24 months
  • Implement multi-signature for high-value credentials
  • Regular security audits
  • Incident response plan for key compromise

Privacy

  • Minimize data in credentials (only necessary fields)
  • Support selective disclosure
  • Don't store personal data on blockchain
  • Provide right-to-revoke mechanisms
  • GDPR/CCPA compliance by design

User Experience

  • Provide clear instructions for wallet setup
  • Offer multiple delivery methods (email, QR, download)
  • Support both digital and printable credentials
  • 24/7 help desk for credential issues
  • Multilingual support
