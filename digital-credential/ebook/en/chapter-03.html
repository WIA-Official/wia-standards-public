Chapter 3: W3C Verifiable Credentials

Understanding the W3C Standard

The World Wide Web Consortium (W3C) Verifiable Credentials Data Model is an open standard that provides a secure, privacy-preserving way to express credentials on the web. Published in 2019 and updated in 2022, it's now the global standard for digital credentials.

Core Components

1. Credentials

A set of one or more claims made by an issuer about a subject. In education, this could be a degree, certificate, or transcript.

2. Verifiable Credentials

Credentials that include cryptographic proofs that make them tamper-evident and cryptographically verifiable.

3. Decentralized Identifiers (DIDs)

Unique identifiers that don't require a centralized registry. Each student, institution, and verifier has their own DID.

Anatomy of a Verifiable Credential

{
  "@context": [
    "https://www.w3.org/2018/credentials/v1",
    "https://wia.org/credentials/education/v1"
  ],
  "id": "did:wia:edu:mit:degree:12345",
  "type": ["VerifiableCredential", "BachelorDegree"],
  "issuer": {
    "id": "did:wia:edu:mit",
    "name": "Massachusetts Institute of Technology"
  },
  "issuanceDate": "2024-06-15T00:00:00Z",
  "credentialSubject": {
    "id": "did:wia:student:alice",
    "name": "Alice Johnson",
    "degree": {
      "type": "Bachelor of Science",
      "field": "Computer Science",
      "graduationDate": "2024-06-15",
      "gpa": "3.85"
    }
  },
  "proof": {
    "type": "Ed25519Signature2020",
    "created": "2024-06-15T12:00:00Z",
    "verificationMethod": "did:wia:edu:mit#key-1",
    "proofPurpose": "assertionMethod",
    "proofValue": "z58DAdFfa9SkqZMVPxAQp...Wn1AKYYzBTFJM"
  }
}

Key Properties

Tamper-Evident

Any modification to the credential invalidates the cryptographic signature. Even changing a single character makes the credential unverifiable.

Privacy-Preserving

Credentials support selective disclosure - sharing only the information needed. For example, proving you have a degree without revealing your GPA.

Machine-Verifiable

Software can automatically verify credentials without human intervention, enabling instant automated verification at scale.

Portable

Credentials are JSON files that can be stored anywhere - in digital wallets, email, cloud storage, or printed as QR codes.

The Trust Triangle

Holder

The student or graduate who possesses the credential

Issuer

The institution that creates and signs the credential

Verifier

The employer, school, or authority that checks the credential

Verification Process

  1. Holder presents credential to Verifier
  2. Verifier checks the cryptographic signature
  3. Verifier resolves the Issuer's DID to get public key
  4. Verifier confirms signature matches Issuer's key
  5. Verifier checks credential hasn't been revoked
  6. Verification complete - typically under 3 seconds

Privacy Features

Selective Disclosure

Share only specific claims from a credential. Prove you graduated without revealing your GPA, graduation date, or other details.

Zero-Knowledge Proofs

Prove properties about credentials without revealing the credentials themselves. For example, prove GPA > 3.5 without revealing the exact GPA.

Minimal Disclosure

The standard encourages revealing only what's necessary for each interaction, protecting privacy by default.

WIA-EDU-011 Extensions

The WIA-EDU-011 standard extends W3C VCs with education-specific schemas:

  • Degree and diploma types
  • Accreditation information
  • Course completion records
  • Skills and competencies
  • Professional certifications
  • Continuing education credits
