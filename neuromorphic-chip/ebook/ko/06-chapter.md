## 스파이킹 신경망 아키텍처

### 피드포워드 SNN

피드포워드 SNN은 입력에서 출력으로 한 방향으로 정보를 처리하는 가장 간단한 아키텍처입니다.

**아키텍처:**
```
입력 레이어 → 은닉 레이어 → 출력 레이어
    |           |              |
  인코딩    LIF 뉴런      디코딩/
  (스파이크)  + STDP         결정
```

**구현 예제:**
```python
class FeedforwardSNN:
    def __init__(self, layer_sizes, dt=0.1):
        self.layer_sizes = layer_sizes
        self.dt = dt
        self.n_layers = len(layer_sizes)
        
        # 각 레이어에 대한 뉴런 초기화
        self.neurons = []
        for size in layer_sizes:
            layer = [LIFNeuron() for _ in range(size)]
            self.neurons.append(layer)
        
        # 시냅스 가중치 초기화
        self.weights = []
        for i in range(self.n_layers - 1):
            w = np.random.randn(layer_sizes[i], layer_sizes[i+1]) * 0.1
            self.weights.append(w)
```

### 순환 SNN

순환 연결은 시간적 처리와 메모리를 가능하게 하며, 순차 데이터 처리에 중요합니다.

**리저버 컴퓨팅 (Liquid State Machine):**
- 리저버: 무작위로 연결된 큰 순환 네트워크
- 에코 상태 속성: 네트워크 상태가 입력 이력 포함
- 읽기: 출력 가중치만 학습 (단순, 빠름)

### 컨볼루션 SNN

CNN을 스파이킹 도메인으로 확장하여 효율적인 시각 처리를 수행합니다.

**핵심 적응:**
- 스파이크 기반 컨볼루션
- 시간 코딩
- 이벤트 기반 처리

## 멤리스터 하드웨어

### 멤리스터 기본

멤리스터(메모리 + 저항)는 2008년 HP Labs에서 처음 물리적으로 구현된 네 번째 기본 회로 소자입니다.

**핵심 속성:**
- 비휘발성: 전원 없이 상태 유지
- 아날로그 상태: 연속 저항 값
- 양방향 프로그래밍: 저항 증가/감소 가능
- 나노스케일: 10-100nm 스케일로 제작 가능
- 저에너지: 펨토줄 스케일 연산

### 크로스바 어레이 아키텍처

크로스바 어레이는 각 교차점에 멤리스터가 있는 와이어 그리드입니다:

```
         열 라인 (입력 전압)
              V₁  V₂  V₃  V₄
               |   |   |   |
    행 1  ----M---M---M---M---- → I₁ = Σ(V_j / R₁ⱼ)
               |   |   |   |
    행 2  ----M---M---M---M---- → I₂ = Σ(V_j / R₂ⱼ)
```

**수학적 연산:**
```
I_row = Σ(V_col / R) = Σ(V_col × G)
여기서 G = 1/R은 전도도 (시냅스 가중치)
이는 정확히: I = W × V (행렬-벡터 곱)
```

## 스파이크 인코딩 프로토콜

### 1. 속도 코딩

**원리:** 발화 빈도로 정보 인코딩

**공식:**
```
firing_rate = k × input_value
spike_probability = input_value × dt / ISI
```

**장점:**
- 구현 간단
- 노이즈 강인
- 생물학적 타당성

**단점:**
- 높은 지연 시간
- 에너지 비효율
- 낮은 정보 밀도

### 2. 시간 (지연) 코딩

**원리:** 정확한 스파이크 타이밍으로 정보 인코딩

**공식:**
```
spike_time = (1 - value) × time_window
```

**장점:**
- 매우 낮은 지연 (1-10ms)
- 높은 정보 용량
- 에너지 효율적

### 3. 집단 코딩

**원리:** 여러 뉴런에 걸친 분산 표현

**가우시안 튜닝 곡선:**
- 각 뉴런은 선호하는 값을 가짐
- 가우시안 응답 곡선
- 노이즈에 매우 강인

### 4. 위상 코딩

**원리:** 진동 기준에 상대적인 스파이크 타이밍

**특징:**
- 높은 시간 정밀도
- 자연스러운 진동 시스템
- 한 주기에 여러 값 다중화 가능

## 벤치마크 및 성능 메트릭

### 주요 성능 메트릭

**1. 에너지 효율:**
```
SOPS/W = (총 시냅스 연산) / (소비 에너지)

예시:
- 100만 뉴런 × 1K 시냅스 × 50 Hz × 1초 = 500억 연산
- 에너지: 5 mW × 1초 = 5 mJ
- 효율: 500억 / 0.005 J = 10 TOPS/W
```

**2. 지연 시간:**
```
총 지연 = 인코딩 시간 + 처리 시간 + 디코딩 시간
```

**3. 정확도 vs 효율 트레이드오프**

### 표준 벤치마크

**N-MNIST (뉴로모픽 MNIST):**
- 데이터셋: DVS 카메라로 녹화된 70,000개 손글씨 숫자
- 일반적 결과: SNN 99.1%, 에너지 0.1 mJ

**DVS-Gesture:**
- 데이터셋: 11개 손 제스처, 1342개 샘플

- 결과: 정확도 92.4%, 에너지 0.5 μJ

## 부록

### 부록 A: 빠른 참조

**뉴런 모델:**
- LIF: τ dV/dt = -(V - V_rest) + RI
- Izhikevich: dv/dt = 0.04v² + 5v + 140 - u + I
- Hodgkin-Huxley: C dV/dt = I - I_Na - I_K - I_L

**인코딩 방법:**
- 속도: f = k × value
- 지연: t = (1 - value) × window
- 집단: 가우시안 튜닝 곡선
- 위상: 진동 대비 스파이크 타이밍

**하드웨어 플랫폼:**
- Intel Loihi 2: 100만 뉴런, 1억 2천만 시냅스, <100mW

- BrainChip Akida: 상업용 엣지 AI, <1mW

**소프트웨어 도구:**
- Brian2: 유연한 시뮬레이션
- SpikingJelly: PyTorch 통합
- Lava: Loihi 배포

**주요 메트릭:**
- 에너지: SOPS/W (GPU보다 10,000배 우수)
