# 📝 기존 수어 표기법 분석

## 왜 이 분석이 필요한가?
WIA Talk이 기존 체계와 무엇이 다른지, 왜 새로운 접근이 필요한지 명확히 하기 위함.

---

## 1. HamNoSys (Hamburg Notation System)

### 개요
- **개발**: 1985년, 함부르크 대학교
- **목적**: 학술 연구용 정밀 표기
- **특징**: 유니코드 기반 기호 체계

### 예시
```
ASL "MOTHER" → 󰁦󰁢󰀁󰂢
(실제로는 특수 폰트 필요)
```

### 장점
- ✅ 매우 정밀한 표기 가능
- ✅ 학술적으로 인정받음
- ✅ 모든 수어에 적용 가능

### 단점
- ❌ 극도로 복잡 (200개+ 기호)
- ❌ 특수 폰트 필요
- ❌ 일반인 학습 불가능
- ❌ 컴퓨터 처리 어려움
- ❌ 농인 커뮤니티에서 사용 안 함

### WIA Talk 비교
| HamNoSys | WIA Talk |
|----------|----------|
| 200개+ 기호 | 5개 컴포넌트 |
| 특수 폰트 | 일반 텍스트 |
| 학술용 | 실용 + 학술 |

## 2. SignWriting

### 개요
- **개발**: 1974년, Valerie Sutton
- **목적**: 수어의 시각적 표기
- **특징**: 그림 기반 표기

### 예시
```
ASL "HELLO" → [손 모양 그림 + 위치 + 움직임 화살표]
(실제로는 그래픽 이미지)
```

### 장점
- ✅ 시각적으로 직관적
- ✅ 농인 커뮤니티 일부 채택
- ✅ 교육용으로 활용

### 단점
- ❌ 배우기 매우 어려움
- ❌ 컴퓨터 입력 어려움
- ❌ 검색/데이터베이스 불가
- ❌ 텍스트 기반 시스템과 호환 안 됨
- ❌ 일관성 부족 (작성자마다 다름)

### WIA Talk 비교
| SignWriting | WIA Talk |
|-------------|----------|
| 그래픽 기반 | 코드 기반 |
| 검색 불가 | 데이터베이스 가능 |
| 비표준화 | 표준화된 코드 |

## 3. Stokoe Notation

### 개요
- **개발**: 1960년, William Stokoe
- **목적**: ASL 언어학 연구
- **의의**: 수어가 진정한 언어임을 증명한 최초 체계

### 예시
```
ASL "MOTHER" → 5̯x
(위치, 손모양, 움직임의 조합)
```

### 장점
- ✅ 최초의 과학적 수어 표기
- ✅ 언어학적으로 중요
- ✅ 간결함

### 단점
- ❌ ASL 전용 (다른 수어 적용 어려움)
- ❌ 표정, 비수지 신호 표기 부족
- ❌ 현대적 확장성 부족

### WIA Talk 비교
| Stokoe | WIA Talk |
|--------|----------|
| ASL 전용 | 300개 수어 통합 |
| 3요소 | 5요소 (표정 포함) |
| 확장 어려움 | 모듈식 확장 |

## 4. Gloss Notation

### 개요
- **방식**: 영어 단어로 수어 표현을 기술
- **용도**: 교육, 비공식 표기

### 예시
```
ASL "What is your name?" → WHAT YOUR NAME?
(대문자 = 수어 단어)
```

### 장점
- ✅ 배우기 쉬움
- ✅ 타이핑 쉬움
- ✅ 비농인에게 이해 쉬움

### 단점
- ❌ 손 모양, 위치 등 정보 없음
- ❌ 실제 수어 재현 불가
- ❌ 언어마다 Gloss 방식 다름

### WIA Talk 비교
| Gloss | WIA Talk |
|-------|----------|
| 의미만 기록 | 동작 상세 기록 |
| 재현 불가 | 정확한 재현 가능 |
| 비표준 | 표준 코드 |

## WIA Talk의 핵심 차별점

### 1. 5대 컴포넌트 단순화
```
HS (손모양) + LC (위치) + MV (움직임) + OR (방향) + FE (표정)
= 완전한 수어 표현
```

### 2. 코드 기반 = 컴퓨터 친화적
```
ASL "HELLO" = HS02-LC02-MV03-OR01-FE02
→ 데이터베이스 저장
→ 검색 가능
→ AI 학습 가능
→ 번역 시스템 연동
```

### 3. 점자와 직접 연동
```
WIA Talk ←→ WIA Braille
수어     ←→ 점자
(시각장애+청각장애 동시 지원)
```

### 4. 300개 수어 통합
```
ASL "LOVE" = HS-XX-LC-XX-MV-XX...
KSL "사랑" = HS-XX-LC-XX-MV-XX...
→ 같은 코드 체계로 비교 가능
→ 수어 간 번역 기반
```

### 5. 모듈식 확장
```
기본: HS + LC + MV + OR + FE
확장: + MM (입움직임) + NMF (비수지신호) + ...
→ 필요에 따라 세부 코드 추가 가능
```
