================================================================================--------------------------------------------------------------------------------
5.1 보관 연속성의 법적 및 윤리적 중요성
--------------------------------------------------------------------------------

보관 연속성(Chain of Custody, CoC)은 대상이 한 인증된 관리자에서 다음 관리자로
이송되는 방법을 문서화하는 법적 및 윤리적 요구사항입니다. 이는 무결성, 진정성,
책임성을 보장합니다.

[CRYO] 보관 연속성이 중요한 이유:

    법적 이유:
      [1] 소유권 증명
          - 누가 언제 대상을 제어했는지 입증
          - 분쟁 시 법적 방어
          - 보험 청구 지원

      [2] 규제 준수
          - FDA (미국), EMA (EU), PMDA (일본) 요구사항
          - 국제 운송 규정
          - 생물학적 재료 추적

      [3] 책임성
          - 각 단계에서 누가 책임이 있는지 식별
          - 오류 또는 사건 발생 시 책임 귀속
          - 전문적 인증 검증

    윤리적 이유:
      [1] 존중 및 존엄성
          - 보존된 개인에 대한 존중 표시
          - 가족 신뢰 유지
          - 홍익인간 원칙 - 모든 인간 이익

      [2] 투명성
          - 모든 행동이 기록됨
          - 이해관계자 접근
          - 비밀 없음

      [3] 신뢰
          - 냉동학 커뮤니티 신뢰성
          - 공공 인식 개선
          - 미래 클라이언트 확보

[!] 보관 연속성 핵심 원칙:

    +-----------------------------------------------------------------------+
    | 원칙 1: 문서화 완전성                                                 |
    |   모든 이송, 모든 핸들러, 모든 위치, 모든 시간 기록                   |
    +-----------------------------------------------------------------------+
    | 원칙 2: 변조 방지                                                     |
    |   디지털 서명, 블록체인 타임스탬프, 변경 불가 로그                    |
    +-----------------------------------------------------------------------+
    | 원칙 3: 연속성 중단 없음                                              |
    |   항상 명확한 관리자, 핸드오프 순간 명시적 문서화                     |
    +-----------------------------------------------------------------------+
    | 원칙 4: 검증 가능성                                                   |
    |   독립적 감사자가 전체 체인 검증 가능                                 |
    +-----------------------------------------------------------------------+
    | 원칙 5: 장기 보존                                                     |
    |   최소 50년 또는 대상 수명, 여러 백업                                 |
    +-----------------------------------------------------------------------+

5.2 보관 연속성 문서 구조
--------------------------------------------------------------------------------

보관 연속성 문서는 이송 생애주기의 모든 단계를 포괄하는 구조화되고 포괄적인
기록입니다.

[CRYO] CoC 문서 구성 요소:

    섹션 1: 대상 식별
      ├─ 대상 고유 ID (영구, 변경 불가)
      ├─ 이름 및 날짜 (출생, 냉동보존)
      ├─ 대상 유형 (전신, 신경, 장기)
      ├─ 보존 방법 (유리화, 냉동)
      └─ 바이오마커 (DNA 샘플, 지문)

    섹션 2: 출발지 시설
      ├─ 시설 이름 및 인증 번호
      ├─ 주소 및 GPS 좌표
      ├─ 담당자 (이름, 인증, 서명)
      ├─ 출발 전 검사 결과
      └─ 출발 타임스탬프 (UTC)

    섹션 3: 이송 세부사항
      ├─ 이송 ID (고유)
      ├─ 이송 팀 (모든 구성원, 인증)
      ├─ 차량 ID 및 듀어 일련번호
      ├─ 계획된 경로 및 체크포인트
      ├─ 초기 온도 및 LN2 레벨
      └─ 보험 정책 번호

    섹션 4: 이송 중 이벤트
      ├─ 모든 체크포인트 (시간, 위치, 서명)
      ├─ LN2 보충 (시간, 양, 공급원)
      ├─ 경로 우회 (이유, 승인)
      ├─ 사건 또는 알림 (열, 충격 등)
      └─ 통신 로그 (주요 업데이트)

    섹션 5: 도착 시설
      ├─ 시설 이름 및 인증 번호
      ├─ 주소 및 GPS 좌표
      ├─ 수신 담당자 (이름, 인증, 서명)
      ├─ 도착 검사 결과
      ├─ 도착 타임스탬프 (UTC)
      └─ 수용 확인 (서명)

    섹션 6: 이송 후 인증
      ├─ 품질 검증 (온도 로그, TQS)
      ├─ 규정 준수 인증
      ├─ 최종 승인 서명
      └─ 아카이브 위치

[i] CoC 문서 JSON 예시 (축약됨):

{

5.3 디지털 서명 및 블록체인 인증
--------------------------------------------------------------------------------

현대 보관 연속성 문서화는 변조 방지 및 장기 검증을 보장하기 위해 암호화 기술에
의존합니다.

[CRYO] 디지털 서명 프로토콜:

    서명 유형:
      [1] 개인 서명 (Individual Signature)
          - 각 관리자의 개인 키
          - PKI (공개 키 인프라) 기반
          - 생체 인식 연결 (지문/얼굴)
          - 만료 및 갱신

      [2] 시설 서명 (Facility Signature)
          - 조직 수준 키
          - HSM (하드웨어 보안 모듈) 저장
          - 다중 서명 (2명 중 2명)
          - 감사 로그

      [3] 타임스탬프 서명 (Timestamp Signature)
          - 신뢰할 수 있는 타임스탬프 기관 (TSA)
          - RFC 3161 표준
          - 재생 공격 방지
          - 장기 유효성 (50년 이상)

    서명 프로세스:

      단계 1: 문서 해시 생성
        SHA-256 해시 계산 (문서 내용)

      단계 2: 개인 키로 서명
        서명 = RSA_Sign(해시, 개인_키)
        또는 ECDSA_Sign(해시, 개인_키)

      단계 3: 인증서 첨부
        서명 + X.509 인증서 (공개 키 포함)

      단계 4: 타임스탬프 추가
        TSA에 해시 제출 → 타임스탬프 토큰 수신

      단계 5: 문서에 첨부
        원본 문서 + 서명 + 인증서 + 타임스탬프

블록체인 타임스탬핑:

블록체인은 장기 변조 방지 타임스탬핑을 위한 추가 계층을 제공합니다.

    지원되는 블록체인:
      - Bitcoin (가장 안전, 느림)
      - Ethereum (스마트 계약, 중간)
      - Polygon (빠름, 저렴, 덜 안전)

    블록체인 앵커링 프로세스:

      단계 1: 문서 해시 생성
        doc_hash = SHA-256(CoC_document)

      단계 2: 블록체인 트랜잭션
        tx = blockchain.submit_transaction(doc_hash)

      단계 3: 블록 확인 대기
        Bitcoin: ~60분 (6블록)
        Ethereum: ~5분 (25블록)

      단계 4: 트랜잭션 해시 저장
        tx_hash, block_number, timestamp 문서에 첨부

    검증 (50년 후에도):

      단계 1: 원본 문서 해시 재계산
        doc_hash_now = SHA-256(CoC_document)

      단계 2: 블록체인에서 조회
        blockchain.get_transaction(tx_hash)

      단계 3: 해시 비교
        doc_hash_now == doc_hash_blockchain?

      단계 4: 타임스탬프 확인
        블록 타임스탬프가 주장된 시간과 일치?

[!] 블록체인 장점:

    ✓ 영구 불변성 (블록 확인 후)
    ✓ 탈중앙화 (단일 실패 지점 없음)
    ✓ 공개 검증 가능 (누구나 확인)
    ✓ 장기 신뢰성 (50년 이상)
    ✓ 감사 흔적 (모든 변경 기록)

5.4 보관 연속성 감사
--------------------------------------------------------------------------------

정기적인 보관 연속성 감사는 표준 준수를 보장하고 시스템 약점을 식별합니다.

[CRYO] 감사 유형:

    내부 감사 (분기별):
      범위:
        - 모든 최근 이송 검토 (지난 3개월)
        - 문서 완전성 확인
        - 서명 검증
        - 타임라인 일관성

      프로세스:
        [1] 무작위 샘플 선택 (20% 이송)
        [2] CoC 문서 검토
        [3] 센서 데이터 교차 검증
        [4] 인터뷰 관련 인력
        [5] 발견 사항 문서화
        [6] 시정 조치 권장

      출력:
        - 감사 보고서
        - 규정 준수 점수 (0-100)
        - 시정 조치 계획

    외부 감사 (연간):
      감사자:
        - 독립적 제3자
        - WIA 인증 감사자
        - 경험 있는 냉동학 전문가

      범위:
        - 모든 이송 (전년도)
        - 시설 절차
        - 인력 교육 기록
        - 장비 인증
        - 보안 및 보관

      출력:
        - 공식 인증 보고서
        - 인증 갱신 (또는 거부)
        - 개선 권장사항

[i] 감사 체크리스트:

    문서 완전성:
      [ ] 모든 필수 필드 완료됨
      [ ] 대상 ID 정확하고 일치함
      [ ] 출발/도착 시설 인증 유효함
      [ ] 모든 관리자 서명 존재함
      [ ] 타임스탬프 순차적이고 논리적임
      [ ] 이벤트 완전히 문서화됨

    기술 검증:
      [ ] 온도 로그 연속적임 (간격 없음)
      [ ] GPS 추적 계획된 경로와 일치함
      [ ] LN2 레벨 일관적임 (증발률)
      [ ] 센서 보정 최신임
      [ ] 디지털 서명 유효함
      [ ] 블록체인 해시 일치함

5.5 장기 보관 및 검색
--------------------------------------------------------------------------------

보관 연속성 문서는 대상이 보존되는 한 (잠재적으로 수세기) 접근 가능하고 판독
가능해야 합니다.

[CRYO] 장기 보관 전략:

    저장 매체 (다중 계층):
      레벨 1: 온라인 데이터베이스 (즉시 액세스)
        - 클라우드 저장소 (AWS, Azure, GCP)
        - 지역 간 복제
        - 자동 백업 (일일)
        - 99.999% 가동 시간

      레벨 2: 아카이브 저장소 (장기)
        - 광학 미디어 (M-DISC, 1000년 수명)
        - 자기 테이프 (LTO-9, 30년)
        - 온도 제어 볼트 (15-20°C, 30-50% RH)
        - 연간 무결성 검증

      레벨 3: 오프사이트 백업 (재해 복구)
        - 지리적으로 분산 (대륙 간)
        - 안전 금고 (은행, 전문 시설)
        - 암호화 (AES-256)
        - 분기별 무결성 확인

      레벨 4: 블록체인 앵커 (불변 증명)
        - 문서 해시 (영구 기록)
        - 분산 검증
        - 미래 증명

    데이터 형식 (미래 증명):
      +----------------------+------------------+---------------------------+
      | 형식                 | 수명             | 장점                      |
      +----------------------+------------------+---------------------------+
      | JSON                 | 50년 이상         | 인간 판독 가능, 널리 지원 |
      | XML                  | 50년 이상         | 구조화, 스키마 검증       |
      | PDF/A-3              | 100년 이상        | ISO 표준, 내장 파일       |
      | CSV                  | 무제한           | 단순, 보편적              |
      | Plaintext            | 무제한           | 가장 단순, 영구적         |
      +----------------------+------------------+---------------------------+

    검색 시스템:

      인덱싱:
        - 대상 ID (고유, 기본 키)
        - 이송 ID
        - 날짜 범위
        - 시설 (출발/도착)
        - 인증 번호

      검색 API:

        GET /api/coc/specimen/{specid}
          → 대상의 전체 CoC 기록 반환

        GET /api/coc/transport/{transportId}
          → 특정 이송 세부사항

        GET /api/coc/search?facility=NYC&date=2025-12
          → 쿼리 매개변수로 검색

        GET /api/coc/verify/{documentHash}
          → 블록체인 검증


--------------------------------------------------------------------------------
장 요약
--------------------------------------------------------------------------------

이 장에서는 극저온 이송에서 보관 연속성 문서화의 법적, 윤리적, 기술적 측면을
다루었습니다. 포괄적이고 변조 방지되며 장기 보존된 CoC 기록은 신뢰, 책임성 및
규정 준수의 기초입니다.

주요 요점:

  [1] 보관 연속성은 소유권 증명, 규제 준수, 책임성을 위한 법적 요구사항이며,
      존중, 투명성, 신뢰를 위한 윤리적 의무로서 5가지 핵심 원칙(완전성, 변조 방지,
      연속성, 검증 가능성, 장기 보존)을 따릅니다.

  [2] CoC 문서는 대상 식별, 출발지 시설, 이송 세부사항, 이송 중 이벤트, 도착 시설,
      이송 후 인증의 6개 섹션으로 구성되며, JSON 스키마로 구조화됩니다.

  [3] 디지털 서명(RSA/ECDSA)과 블록체인 타임스탬핑(Bitcoin, Ethereum)은 변조 방지
      및 장기 검증(50년 이상)을 보장하며, 암호화 인증을 제공합니다.

  [4] 정기적인 감사(내부 분기별, 외부 연간)는 표준 준수를 검증하고, 문서 완전성,
      기술 검증을 위한 체크리스트를 사용합니다.

  [5] 장기 보관은 4계층 전략(온라인, 아카이브, 오프사이트, 블록체인)을 사용하며,
      미래 증명 형식으로 수세기 후에도 접근성을 보장합니다.

복습 질문
--------------------------------------------------------------------------------

Q1: 보관 연속성이 법적(소유권, 규제, 책임성)과 윤리적(존중, 투명성, 신뢰) 측면에서
    중요한 이유는 무엇이며, 5가지 핵심 원칙은 무엇입니까?

Q2: CoC 문서의 6개 섹션을 설명하고, 각 섹션에 포함되어야 하는 주요 정보는
    무엇입니까?
