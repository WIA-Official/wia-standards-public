통신 및 항법

GPS가 작동하지 않는 세계에서: 음향 통신, 관성 항법, 그리고 수중 위치 결정 시스템.

수중 통신의 도전

전자기파는 해수에서 급격히 감쇠되어 라디오와 GPS가 수중에서 사용 불가능합니다. 심해 탐사선은 음향 통신과 대체 항법 시스템에 의존해야 합니다. WIA-OCEAN-001 표준은 수중 통신 및 위치 결정을 위한 포괄적인 프로토콜을 정의합니다.

학습 목표

  • 수중 위치 결정: LBL, USBL, SBL 시스템 탐구

  • 한국의 수중 통신 기술 발전 인식

음향 통신

음파는 수중에서 전자기파보다 훨씬 잘 전파됩니다. 음향 모뎀은 수 킬로미터 거리에서 데이터를 전송할 수 있지만, 대역폭이 극도로 제한됩니다(보통 수십 kbps).

음향 통신 제약

  • 낮은 대역폭: 10-40 kbps (WiFi는 수백 Mbps)
  • 높은 지연: 음파 속도 ~1,500 m/s (빛은 300,000 km/s)
  • 다중 경로: 해저 및 수면 반사로 인한 간섭
  • 도플러 효과: 이동하는 플랫폼에서 주파수 변이

관성 항법 시스템 (INS)

INS는 가속도계와 자이로스코프를 사용하여 초기 위치로부터 이동을 추적합니다. GPS 없이 작동하지만, 시간이 지남에 따라 드리프트가 발생합니다.

도플러 속도 로그 (DVL)

DVL은 해저로부터 반사된 음파의 도플러 변이를 측정하여 정확한 속도를 계산합니다. INS와 결합하면 드리프트를 크게 줄일 수 있습니다.

수중 위치 결정 시스템

심해 탐사선은 정확한 위치 결정을 위해 음향 트랜스폰더 네트워크를 사용합니다.

LBL (Long Baseline)

해저에 배치된 3개 이상의 트랜스폰더가 삼각측량을 통해 정확한 위치를 제공합니다. 정확도가 가장 높지만 설치가 필요합니다.

USBL (Ultra-Short Baseline)

수면 선박의 단일 트랜스듀서 배열이 잠수정의 방향과 거리를 측정합니다. 설치 불필요, 정확도는 LBL보다 낮습니다.

TypeScript 구현: 음향 통신 프로토콜

interface AcousticMessage {
  id: string;
  priority: 'critical' | 'high' | 'normal' | 'low';
  payload: Uint8Array;
  timestamp: Date;
  checksum: number;
}

class AcousticModem {
  private txQueue: AcousticMessage[] = [];
  private maxBandwidth: number = 20000; // bps

  sendMessage(msg: AcousticMessage): void {
    // 우선순위에 따라 큐에 삽입
    this.txQueue.push(msg);
    this.txQueue.sort((a, b) => {
      const priorities = {
        critical: 0, high: 1, normal: 2, low: 3
      };
      return priorities[a.priority] - priorities[b.priority];
    });

    this.transmit();
  }

  transmit(): void {
    if (this.txQueue.length === 0) return;

    const msg = this.txQueue[0];
    const bits = msg.payload.length * 8;
    const timeMs = (bits / this.maxBandwidth) * 1000;

    console.log(`전송: ${msg.id}, ${timeMs}ms 소요`);

    // 에러 정정 코드 적용
    const encoded = this.applyFEC(msg.payload);

    // 실제 음향 전송 시뮬레이션
    setTimeout(() => {
      this.txQueue.shift();
      if (this.txQueue.length > 0) this.transmit();
    }, timeMs);
  }

  applyFEC(data: Uint8Array): Uint8Array {
    // Forward Error Correction 적용
    // 간단화: 데이터를 2배로 확장 (실제로는 Reed-Solomon 등 사용)
    const encoded = new Uint8Array(data.length * 2);
    for (let i = 0; i < data.length; i++) {
      encoded[i * 2] = data[i];
      encoded[i * 2 + 1] = data[i]; // 중복
    }
    return encoded;
  }

  compress(data: string): Uint8Array {
    // 간단한 데이터 압축 (실제로는 LZ77 등 사용)
    const encoder = new TextEncoder();
    return encoder.encode(data);
  }
}

한국의 수중 통신 기술

KIOST는 국산 음향 모뎀과 USBL 시스템을 개발했습니다. 이사부호는 첨단 USBL 시스템을 탑재하여 해미래 ROV를 1미터 정확도로 추적합니다. 현대중공업과 삼성중공업은 해양 시추 플랫폼용 수중 위치 결정 시스템을 생산합니다.

통신 단절 대비

음향 통신은 해류, 온도층, 그리고 생물학적 소음에 의해 방해받을 수 있습니다. WIA-OCEAN-001은 모든 탐사선이 통신 단절 시 자동으로 수면으로 귀환하는 프로토콜을 요구합니다.

요약: 핵심 사항

  • 음향 통신: 수중 유일한 장거리 통신 수단
  • 낮은 대역폭: 10-40 kbps, 데이터 압축 필수
  • INS+DVL: GPS 없이 정확한 항법
  • LBL/USBL: 음향 위치 결정 시스템
  • 한국 기술: KIOST 음향 모뎀, USBL 시스템
