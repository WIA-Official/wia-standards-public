Chapter 2: Virtual Attendance Systems

Standards for Remote Participation in Funeral Services

2.1 Introduction to Virtual Attendance

Virtual attendance systems enable remote participants to fully engage in funeral and memorial services from any location worldwide. This chapter defines comprehensive standards for implementing virtual attendance platforms that provide meaningful participation experiences while maintaining the dignity and sanctity of funeral ceremonies.

The WIA Virtual Attendance standard addresses video conferencing integration, virtual seating arrangements, interactive features, presence verification, and emotional connection tools. These systems must seamlessly blend technology with tradition, ensuring remote attendees feel genuinely present and connected.

Core Requirements

  • Presence Verification: Confirm remote attendee identity and registration
  • Real-time Interaction: Enable chat, reactions, and virtual tributes during services
  • Quality Assurance: HD video/audio with minimal latency (under 500ms)
  • Accessibility: Closed captioning, sign language interpretation, audio descriptions
  • Privacy Protection: Secure access controls and encrypted communications
  • Device Compatibility: Support desktop, mobile, tablet, smart TV platforms

2.5 Implementation Examples

Example 1: Virtual Attendance Registration (JavaScript)

class VirtualAttendanceSystem {
  constructor(apiKey, serviceId) {
    this.apiKey = apiKey;
    this.serviceId = serviceId;
    this.baseUrl = 'https://api.funeral-services.org';
  }

  async registerAttendee(attendeeData) {
    const registration = {
      service_id: this.serviceId,
      name: attendeeData.name,
      email: attendeeData.email,
      phone: attendeeData.phone,
      relationship_to_deceased: attendeeData.relationship,
      accessibility_needs: {
        closed_captions: attendeeData.needsCaptions || false,
        sign_language: attendeeData.needsSignLanguage || false,
        language_preference: attendeeData.language || 'en'
      },
      privacy_settings: {
        visible_to_others: attendeeData.visibleToOthers !== false,
        allow_messages: attendeeData.allowMessages !== false
      }
    };

    try {
      const response = await fetch(
        `${this.baseUrl}/api/v1/virtual-attendance/register`,
        {
          method: 'POST',
          headers: {
            'Content-Type': 'application/json',
            'Authorization': `Bearer ${this.apiKey}`
          },
          body: JSON.stringify(registration)
        }
      );

      if (!response.ok) {
        throw new Error(`Registration failed: ${response.statusText}`);
      }

      const result = await response.json();

      // Send confirmation email
      await this.sendConfirmationEmail(result.attendee_id, attendeeData.email);

      return result;
    } catch (error) {
      console.error('Virtual attendance registration error:', error);
      throw error;
    }
  }

  async joinService(attendeeId, deviceType) {
    const joinData = {
      attendee_id: attendeeId,
      device_type: deviceType,
      timestamp: new Date().toISOString()
    };

    const response = await fetch(
      `${this.baseUrl}/api/v1/virtual-attendance/${this.serviceId}/join`,
      {
        method: 'POST',
        headers: {
          'Content-Type': 'application/json',
          'Authorization': `Bearer ${this.apiKey}`
        },
        body: JSON.stringify(joinData)
      }
    );

    const result = await response.json();

    // Initialize real-time presence updates
    this.initializePresenceTracking(attendeeId);

    // Get stream URL
    const streamUrl = await this.getStreamUrl();

    return {
      ...result,
      stream_url: streamUrl
    };
  }

  async sendReaction(attendeeId, reactionType) {
    const reaction = {
      attendee_id: attendeeId,
      reaction_type: reactionType, // 'candle', 'flower', 'prayer', 'heart'
      timestamp: new Date().toISOString()
    };

    await fetch(
      `${this.baseUrl}/api/v1/virtual-attendance/${this.serviceId}/reaction`,
      {
        method: 'POST',
        headers: {
          'Content-Type': 'application/json',
          'Authorization': `Bearer ${this.apiKey}`
        },
        body: JSON.stringify(reaction)
      }
    );
  }

  initializePresenceTracking(attendeeId) {
    // Send heartbeat every 30 seconds
    setInterval(async () => {
      await fetch(
        `${this.baseUrl}/api/v1/virtual-attendance/${this.serviceId}/presence`,
        {
          method: 'PUT',
          headers: {
            'Content-Type': 'application/json',
            'Authorization': `Bearer ${this.apiKey}`
          },
          body: JSON.stringify({
            attendee_id: attendeeId,
            last_activity: new Date().toISOString(),
            is_online: true
          })
        }
      );
    }, 30000);
  }
}

// Usage
const attendanceSystem = new VirtualAttendanceSystem('api-key', 'service-123');

const attendee = await attendanceSystem.registerAttendee({
  name: 'Jane Smith',
  email: 'jane@example.com',
  phone: '+1-555-0123',
  relationship: 'friend',
  needsCaptions: true,
  language: 'en'
});

Example 2: Virtual Seating Management (Python)

import requests
from typing import List, Dict, Optional
from dataclasses import dataclass

@dataclass
class VirtualSeat:
    section: str
    row: int
    seat_number: int
    attendee_id: Optional[str] = None
    camera_view: str = "main"

class VirtualSeatingManager:
    def __init__(self, api_key: str, service_id: str):
        self.api_key = api_key
        self.service_id = service_id
        self.base_url = 'https://api.funeral-services.org'
        self.headers = {
            'Authorization': f'Bearer {api_key}',
            'Content-Type': 'application/json'
        }

    def create_seating_chart(self, total_capacity: int) -> List[VirtualSeat]:
        """Generate virtual seating chart"""
        seats = []
        sections = ['Family', 'Friends', 'Colleagues', 'General']
        seats_per_section = total_capacity // len(sections)

        for section in sections:
            rows = 10
            seats_per_row = seats_per_section // rows

            for row in range(1, rows + 1):
                for seat_num in range(1, seats_per_row + 1):
                    seats.append(VirtualSeat(
                        section=section,
                        row=row,
                        seat_number=seat_num,
                        camera_view=self._get_camera_view(section)
                    ))

        return seats

    def assign_seat(self, attendee_id: str, preference: str = 'auto') -> VirtualSeat:
        """Assign seat to virtual attendee"""
        seating_data = {
            'attendee_id': attendee_id,
            'preference': preference
        }

        response = requests.post(
            f'{self.base_url}/api/v1/virtual-attendance/{self.service_id}/seating/assign',
            headers=self.headers,
            json=seating_data
        )

        response.raise_for_status()
        seat_info = response.json()

        return VirtualSeat(**seat_info)

    def get_seating_arrangement(self) -> Dict:
        """Retrieve current seating arrangement"""
        response = requests.get(
            f'{self.base_url}/api/v1/virtual-attendance/{self.service_id}/seating',
            headers=self.headers
        )

        return response.json()

    def _get_camera_view(self, section: str) -> str:
        """Determine camera view based on section"""
        camera_views = {
            'Family': 'altar',
            'Friends': 'main',
            'Colleagues': 'main',
            'General': 'wide'
        }
        return camera_views.get(section, 'main')

# Usage
seating_mgr = VirtualSeatingManager('api-key', 'service-123')
seats = seating_mgr.create_seating_chart(500)
assigned_seat = seating_mgr.assign_seat('attendee-456', preference='family')

Example 3: Real-time Chat System (WebSocket)

// WebSocket-based chat for virtual attendees
class VirtualServiceChat {
  constructor(serviceId, attendeeId, apiKey) {
    this.serviceId = serviceId;
    this.attendeeId = attendeeId;
    this.apiKey = apiKey;
    this.ws = null;
    this.messageHandlers = [];
  }

  connect() {
    const wsUrl = `wss://chat.funeral-services.org/ws/${this.serviceId}`;

    this.ws = new WebSocket(wsUrl);

    this.ws.onopen = () => {
      console.log('Connected to virtual service chat');

      // Authenticate
      this.ws.send(JSON.stringify({
        type: 'authenticate',
        attendee_id: this.attendeeId,
        api_key: this.apiKey
      }));
    };

    this.ws.onmessage = (event) => {
      const message = JSON.parse(event.data);
      this.handleMessage(message);
    };

    this.ws.onerror = (error) => {
      console.error('WebSocket error:', error);
    };

    this.ws.onclose = () => {
      console.log('Disconnected from chat');
      // Attempt reconnection after 5 seconds
      setTimeout(() => this.connect(), 5000);
    };
  }

  sendMessage(text, isPrivate = false) {
    if (this.ws && this.ws.readyState === WebSocket.OPEN) {
      const message = {
        type: 'chat_message',
        attendee_id: this.attendeeId,
        text: text,
        is_private: isPrivate,
        timestamp: new Date().toISOString()
      };

      this.ws.send(JSON.stringify(message));
    }
  }

  sendReaction(reactionType) {
    if (this.ws && this.ws.readyState === WebSocket.OPEN) {
      const reaction = {
        type: 'reaction',
        attendee_id: this.attendeeId,
        reaction_type: reactionType,
        timestamp: new Date().toISOString()
      };

      this.ws.send(JSON.stringify(reaction));
    }
  }

  handleMessage(message) {
    switch(message.type) {
      case 'chat_message':
        this.messageHandlers.forEach(handler => handler(message));
        break;
      case 'reaction':
        this.handleReaction(message);
        break;
      case 'system_announcement':
        this.handleAnnouncement(message);
        break;
    }
  }

  onMessage(handler) {
    this.messageHandlers.push(handler);
  }

  handleReaction(reaction) {
    // Display reaction animation
    console.log(`${reaction.attendee_name} sent ${reaction.reaction_type}`);
  }

  handleAnnouncement(announcement) {
    // Display system announcement
    console.log('Announcement:', announcement.text);
  }

  disconnect() {
    if (this.ws) {
      this.ws.close();
    }
  }
}

// Usage
const chat = new VirtualServiceChat('service-123', 'attendee-456', 'api-key');
chat.connect();

chat.onMessage((message) => {
  console.log(`${message.author_name}: ${message.text}`);
  // Update UI with new message
});

// Send condolence message
chat.sendMessage('My deepest condolences to the family.');

// Send virtual candle
chat.sendReaction('candle');

2.7 Best Practices

  • Send invitations 48 hours before service with clear joining instructions
  • Provide technical rehearsal option for family members unfamiliar with platform
  • Assign moderators to manage chat and handle technical issues during service
  • Display attendee count prominently to show family how many are participating
  • Enable "gallery view" to show virtual attendees during appropriate moments
  • Mute all attendees by default to prevent audio disruptions
  • Provide downloadable service program/order of service PDF
  • Record service only with explicit family permission and secure storage
