--------------------------------------------------------------------------------
6.1 금고 아키텍처
--------------------------------------------------------------------------------

신원 금고는 암호화, 이중화, 무결성 검증을 통해
모든 신원 자격증명의 안전한 장기 저장을 제공합니다.

[ID] 금고 구조:

    IdentityVault
    +------------------------------------------------------------------+
    |  VAULT-2025-001-a1b2c3                                           |
    +------------------------------------------------------------------+
    |  구성                                                            |
    |  - 암호화: AES-256-GCM                                           |
    |  - 키 관리: GUARDIAN                                             |
    |  - 이중화: 5개 복사본                                            |
    +------------------------------------------------------------------+
    |  내용물                                                          |
    |  +-- coreIdentity         [암호화됨]                             |
    |  +-- legalDocuments[]     [암호화됨]                             |
    |  +-- personalRecords[]    [암호화됨]                             |
    |  +-- digitalCredentials[] [암호화됨]                             |
    |  +-- biometricData[]      [암호화됨]                             |
    |  +-- memoryArchives[]     [암호화됨]                             |
    +------------------------------------------------------------------+


--------------------------------------------------------------------------------
6.2 암호화 표준
--------------------------------------------------------------------------------

[ID] 지원 암호화:

    AES-256-GCM        -> 표준 대칭 암호화
    CHACHA20-POLY1305  -> 현대 스트림 암호

[ID] 키 관리 옵션:

    THRESHOLD  -> 샤미르 비밀 분산 (k-of-n)
    MULTI_SIG  -> 다중 키 보유자 공동 서명
    GUARDIAN   -> 후견인 네트워크 접근 제어

[ID] 암호화된 페이로드:

    EncryptedPayload {
      payloadId:       "PAYLOAD-001"
      contentType:     "application/json"
      encryptedData:   "[BASE64 암호화 데이터]"
      encryptionKeyId: "KEY-VAULT-001"
      nonce:           "a1b2c3d4e5f6..."
      createdAt:       "2025-03-01T10:00:00Z"
      lastModified:    "2025-03-01T10:00:00Z"
      integrityHash:   "sha256:f6e5d4c3b2a1..."
    }

[!] 암호화 요구사항:
    - 최소 256비트 키 길이
    - 인증된 암호화 (GCM, Poly1305)
    - 정기적 키 순환
    - 양자내성 알고리즘 마이그레이션 계획


--------------------------------------------------------------------------------
6.3 저장소 이중화
--------------------------------------------------------------------------------

[ID] 저장 위치:

    StorageLocation {
      locationId:   "LOC-PRIMARY-001"
      locationType: "PRIMARY"       // PRIMARY | BACKUP | COLD_STORAGE
      provider:     "WIA 금고 클라우드"
      region:       "asia-east"
      jurisdiction: "대한민국"
      status:       "ACTIVE"
      lastSync:     "2025-03-01T10:00:00Z"
      integrityVerified: "2025-03-01T10:00:00Z"
    }

[ID] 이중화 수준:

    Level 3:  기본 + 2개 백업 (최소)
    Level 5:  기본 + 4개 백업 (권장)
    Level 7:  기본 + 6개 백업 (최대 보안)

[ID] 지리적 분산:

    - 최소 2개의 서로 다른 지리적 지역
    - 최소 2개의 서로 다른 법적 관할권
    - 에어갭 콜드 스토리지 권장

[ID] 저장 위치 예시:

    +---------------------------------------------------+
    | 위치             | 유형         | 관할권          |
    +---------------------------------------------------+
    | 서울 데이터센터  | PRIMARY      | 대한민국        |
    | 부산 데이터센터  | BACKUP       | 대한민국        |
    | 싱가포르 DC      | BACKUP       | 싱가포르        |
    | 취리히 금고      | COLD_STORAGE | 스위스          |
    | 런던 DC          | BACKUP       | 영국            |
    +---------------------------------------------------+


--------------------------------------------------------------------------------
6.4 접근 제어
--------------------------------------------------------------------------------

[ID] 접근 정책:

    AccessPolicy {
      defaultAccess: "NONE"

      authorizedAccessors: [
        {
          accessorId:    "GUARD-001"
          accessorType:  "GUARDIAN"
          accessLevel:   "FULL"
          permissions:   ["READ", "UPDATE", "RECOVER"]
          validFrom:     "2025-01-01"
          conditions:    ["2인 후견인 승인 필요"]
        },
        {
          accessorId:    "FAC-001"
          accessorType:  "FACILITY"
          accessLevel:   "METADATA"
          permissions:   ["READ_METADATA", "VERIFY"]
        }
      ]

      emergencyAccess: {
        enabled:            true
        threshold:          3
        waitingPeriod:      72  // 시간
        notificationRequired: true
      }
    }

[ID] 접근 수준:

    NONE      -> 접근 불가
    METADATA  -> 메타데이터만 읽기
    READ      -> 콘텐츠 읽기
    UPDATE    -> 수정 가능
    FULL      -> 전체 권한 (복구 포함)

6.5 무결성 검증
--------------------------------------------------------------------------------

[ID] 무결성 증명:

    IntegrityProof {
      proofId:    "PROOF-2025-001"
      vaultId:    "VAULT-2025-001"
      timestamp:  "2025-03-01T10:00:00Z"

      hashes: {
        coreIdentity:      "sha256:a1b2c3..."
        legalDocuments:    ["sha256:d4e5f6...", "sha256:g7h8i9..."]
        biometricData:     ["sha256:j0k1l2...", "sha256:m3n4o5..."]
      }

      rootHash:   "sha256:z9y8x7w6v5u4..."
    }

[ID] 검증 일정:

    매일:    저장소 가용성 확인
    매주:    해시 무결성 검증
    매월:    전체 콘텐츠 검증
    매년:    교차 위치 조정

[ID] 머클 트리 구조:

                    [루트 해시]
                   /           \
            [해시 A-B]       [해시 C-D]
            /      \         /      \
      [해시A]  [해시B]  [해시C]  [해시D]
         |        |        |        |
      법적문서  개인기록  디지털ID  생체데이터

6.6 금고 접근 로그
--------------------------------------------------------------------------------

[ID] 접근 로그 항목:

    VaultAccessLog {
      vaultId:         "VAULT-2025-001"
      accessorId:      "GUARD-001"
      accessorType:    "GUARDIAN"
      accessType:      "READ"
      sectionAccessed: "biometricData"
      accessGranted:   true
      accessedAt:      "2025-03-01T10:00:00Z"
    }

[i] 감사 요구사항:
    - 모든 접근 시도 로깅
    - 로그는 금고와 별도 저장
    - 불변 감사 추적 (블록체인 앵커링)
    - 보존 기간 동안 유지

[ID] 로그 보존:

    AccessLogRetention {
      minimumRetention:  "보존 기간 전체"
      storageFormat:     "JSON + 블록체인 해시"
      redundancy:        3  // 복사본
      immutability:      true
      queryable:         true
    }
