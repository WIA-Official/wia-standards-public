4.1 탈중앙화 식별자 (DIDs)
--------------------------------------------------------------------------------

DID는 중앙 기관 의존 없이 시간과 시스템을 넘어 지속되는
암호학적 신원 기반을 제공합니다.

[ID] DID 구조:

    Primary DID:   did:wia:cryo:cryoid2025001001
                   |    |   |   |
                   |    |   |   +-- 신원 고유 식별자
                   |    |   +------ 냉동보존 네임스페이스
                   |    +---------- WIA 메서드
                   +--------------- DID 스킴

[ID] DID 구성:

    DIDs {
      primary:   "did:wia:cryo:cryoid2025001001"
      secondary: [
        "did:key:z6MkhaXgBZDvotDkL5257faiztiGiC2QtKLGpbnnEGta2doK"
      ]
      recovery: [
        "did:wia:guardian:guardian001",
        "did:wia:guardian:guardian002",
        "did:wia:guardian:guardian003"
      ]
    }

[ID] DID 문서:

    DID 문서는 블록체인에 기록되어 변조가 불가능합니다.

    DIDDocument {
      "@context": ["https://www.w3.org/ns/did/v1"]
      id:         "did:wia:cryo:cryoid2025001001"
      controller: "did:wia:cryo:cryoid2025001001"

      verificationMethod: [
        {
          id:           "did:wia:cryo:cryoid2025001001#key-1"
          type:         "Ed25519VerificationKey2020"
          controller:   "did:wia:cryo:cryoid2025001001"
          publicKeyMultibase: "z6MkhaXgBZDvotDkL5257..."
        }
      ]

      authentication: ["#key-1"]
      assertionMethod: ["#key-1"]
    }

4.2 자격증명 지갑
--------------------------------------------------------------------------------

자격증명 지갑은 검증 가능한 자격증명을 저장하고 관리합니다.

[ID] 지갑 유형:

    HARDWARE   -> 물리적 보안 모듈 (예: Ledger, Trezor)
    SOFTWARE   -> 소프트웨어 기반 지갑 (암호화 저장)
    CUSTODIAL  -> 제3자 관리형 (시설 보관)

[ID] 지갑 구성:

    CredentialWallet {
      walletId:           "WALLET-2025-001-a1b2c3"
      walletType:         "HARDWARE"
      provider:           "WIA 보안 금고"
      publicKey:          "0x04a1b2c3..."
      recoveryMechanism:  "GUARDIAN"
      guardians: [
        { guardianId: "G001", threshold: 2 },
        { guardianId: "G002", threshold: 2 },
        { guardianId: "G003", threshold: 2 }
      ]
    }

[ID] 복구 메커니즘:

    SEED_PHRASE     -> 12/24단어 복구 문구 (금고 보관)
    SOCIAL_RECOVERY -> 후견인 네트워크 승인
    GUARDIAN        -> 지정 후견인 접근
    MULTI_SIG       -> 다중 키 보유자 필요

4.3 온라인 계정
--------------------------------------------------------------------------------

온라인 계정은 부활 후 신원 복원을 위해 보존되어야 합니다.

[ID] 계정 카테고리:

    SOCIAL         -> 소셜 미디어 플랫폼
    FINANCIAL      -> 은행, 투자 계정
    PROFESSIONAL   -> LinkedIn, 업무 계정
    ENTERTAINMENT  -> 스트리밍, 게임 계정
    UTILITY        -> 이메일, 클라우드 저장소

[ID] 계정 기록:

    OnlineAccount {
      accountId:          "ACCT-001"
      platform:           "Google"
      username:           "gildong.hong@gmail.com"
      accountType:        "UTILITY"
      status:             "ACTIVE"
      createdDate:        "2005-03-15"
      lastAccess:         "2025-03-01"
      recoveryOptions:    ["전화", "복구이메일", "보안키"]
      credentialVaultRef: "vault://credentials/google"
    }

[ID] 주요 플랫폼 목록:

    +---------------------------------------------------+
    | 플랫폼          | 카테고리      | 복구 난이도    |
    +---------------------------------------------------+
    | Google          | UTILITY       | 중간           |
    | Apple           | UTILITY       | 어려움         |
    | 네이버          | UTILITY       | 중간           |
    | 카카오          | SOCIAL        | 중간           |
    | Facebook        | SOCIAL        | 어려움         |
    | 국민은행        | FINANCIAL     | 매우 어려움    |
    | 삼성증권        | FINANCIAL     | 매우 어려움    |
    +---------------------------------------------------+


--------------------------------------------------------------------------------
4.4 디지털 자산
--------------------------------------------------------------------------------

디지털 자산은 장기 보존을 위해 주의 깊은 문서화가 필요합니다.

[ID] 자산 유형:

    CRYPTOCURRENCY    -> 비트코인, 이더리움 등
    NFT               -> 대체 불가능 토큰
    DOMAIN            -> 도메인 소유권
    VIRTUAL_PROPERTY  -> 메타버스 토지, 게임 아이템
    DIGITAL_CONTENT   -> 음악, 미술, 출판물
    SOFTWARE_LICENSE  -> 소프트웨어 라이선스

[ID] 디지털 자산 기록:

    DigitalAsset {
      assetId:         "ASSET-BTC-001"
      assetType:       "CRYPTOCURRENCY"
      name:            "비트코인 보유"
      platform:        "비트코인 네트워크"
      walletAddress:   "bc1q..."
      quantity:        "2.5"
      estimatedValue:  {
        amount:   300000000
        currency: "KRW"
        asOfDate: "2025-03-01"
      }
      accessMethod:    "하드웨어 지갑 + 시드 문구 (금고 보관)"
      legalOwnership:  "SOLE"
    }

[!] 암호화폐 주의사항:
    - 시드 문구는 반드시 금고에 안전하게 보관
    - 다중 서명 지갑 권장
    - 정기적 잔액 확인 메커니즘 필요


--------------------------------------------------------------------------------
4.5 암호화 키
--------------------------------------------------------------------------------

장기 키 관리는 신원 복원을 보장합니다.

[ID] 키 기록:

    CryptographicKey {
      keyId:              "KEY-SIGN-001"
      keyType:            "ED25519"
      purpose:            "SIGNING"
      publicKey:          "ed25519:abc123..."
      privateKeyLocation: "VAULT"  // VAULT | HARDWARE | ESCROW
      rotationPolicy:     "5년마다 또는 알고리즘 업데이트시"
    }

[!] 알고리즘 민첩성:
    암호화 알고리즘이 발전함에 따라 키도 마이그레이션되어야 합니다.
    장기 저장을 위해 양자내성 알고리즘을 고려해야 합니다.

[ID] 양자내성 고려:

    현재 권장:
    - Ed25519 (서명용)
    - X25519 (키 교환용)
    - AES-256-GCM (암호화용)

    미래 대비:
    - CRYSTALS-Dilithium (양자내성 서명)
    - CRYSTALS-Kyber (양자내성 키 캡슐화)
