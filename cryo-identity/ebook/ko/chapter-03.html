--------------------------------------------------------------------------------
3.1 개인 기록 구조
--------------------------------------------------------------------------------

개인 신원은 공식 문서로 정의되지 않는 "나"의 측면입니다.
관계, 경험, 기억이 모여 진정한 정체성을 형성합니다.

[ID] 개인 신원 기록:

    PersonalIdentityRecord {
      recordId:        "PERSONAL-2025-001"
      identityId:      "CRYOID-2025-001"
      type:            "PERSONAL"

      biography: {
        nickname:      "길동이"
        birthplace:    "서울특별시 종로구"
        hometown:      "서울"
        languages:     ["한국어", "영어", "일본어"]
        religion:      "불교"
        hobbies:       ["독서", "등산", "바둑"]
      }

      personalStatement: "vault://personal/statement.txt"
      lifeTimeline:      "vault://personal/timeline.json"
    }


--------------------------------------------------------------------------------
3.2 관계 네트워크
--------------------------------------------------------------------------------

[ID] 관계 기록:

    인간관계는 신원의 핵심입니다. 부활 후 재연결을 위해
    중요한 관계들을 문서화합니다.

    Relationship {
      relationshipId:  "REL-001"
      relatedPerson: {
        name:          "홍길순"
        relationship:  "배우자"
        since:         "2000-06-15"
      }
      contactInfo:     "[암호화됨]"
      significance:    "PRIMARY"      // PRIMARY | FAMILY | FRIEND | PROFESSIONAL
      notes:           "대학 동창으로 만남, 25년간 결혼 생활"
      verificationStatus: "VERIFIED"
    }

[ID] 관계 네트워크 시각화:

                          [본인]
                            |
            +-------+-------+-------+-------+
            |       |       |       |       |
         [배우자] [자녀]  [부모]  [형제]  [친구]
            |       |       |       |       |
           ...     ...     ...     ...     ...

[ID] 관계 카테고리:

    PRIMARY:      배우자, 자녀
    FAMILY:       부모, 형제, 친척
    FRIEND:       친한 친구, 동창
    PROFESSIONAL: 직장 동료, 사업 파트너
    COMMUNITY:    종교, 동호회, 커뮤니티

[ID] 미디어 큐레이션:

    부활 후 리마인드를 위한 핵심 미디어 선별:

    CuratedMemories {
      essentialPhotos: [
        { path: "vault://media/essential/family_2020.jpg",
          caption: "2020년 가족 사진",
          people: ["배우자", "자녀"] }
      ]
      essentialVideos: [
        { path: "vault://media/essential/message_to_future.mp4",
          duration: "10분",
          description: "미래의 나에게 보내는 메시지" }
      ]
    }

[i] 저장 권장사항:
    - 무손실 형식 선호 (JPEG보다 PNG/TIFF)
    - 메타데이터 보존 (날짜, 위치, 태그)
    - 인물 태깅으로 관계 네트워크 연결
    - 정기적 백업 및 무결성 검증
