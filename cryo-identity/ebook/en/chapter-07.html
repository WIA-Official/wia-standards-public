7.1 GUARDIAN NETWORK OVERVIEW
--------------------------------------------------------------------------------

Guardians are trusted entities authorized to participate in identity
recovery decisions. A well-configured guardian network ensures identity
can be restored even after decades of preservation.

[ID] GUARDIAN TYPES:

    PERSON           -> Individual trusted by the subject
    ORGANIZATION     -> Legal entity (law firm, trust company)
    SMART_CONTRACT   -> Autonomous on-chain guardian

[ID] GUARDIAN RECORD:

    Guardian {
      guardianId:          "GUARD-2025-001"
      guardianType:        "PERSON"
      name:                "Jane Smith"
      contactInfo:         "[ENCRYPTED]"
      threshold:           1       // Weight in recovery
      relationship:        "Sister"
      status:              "ACTIVE"
      createdAt:           "2025-01-01"
      lastVerified:        "2025-03-01"
    }


--------------------------------------------------------------------------------
7.2 THRESHOLD CONFIGURATION
--------------------------------------------------------------------------------

Threshold recovery requires a minimum number of guardian approvals.

[ID] THRESHOLD MODELS:

    2-of-3:   Two out of three guardians required
    3-of-5:   Three out of five guardians required
    4-of-7:   Four out of seven guardians required

[ID] CONFIGURATION EXAMPLE:

    RecoveryConfig {
      identityId:       "CRYOID-2025-001"
      threshold:        3
      totalGuardians:   5
      waitingPeriod:    72      // hours before execution
      emergencyBypass:  false   // No bypass allowed
    }


--------------------------------------------------------------------------------
7.3 RECOVERY INITIATION
--------------------------------------------------------------------------------

[ID] RECOVERY REQUEST:

    RecoveryRequest {
      requestId:    "RECOVERY-2025-001"
      identityId:   "CRYOID-2025-001"
      requester:    "John Smith Jr."
      reason:       "Identity verification for revival preparation"
      status:       "PENDING"
      threshold:    3
      approvals:    []
      rejections:   []
      initiatedAt:  "2025-06-01T10:00:00Z"
      expiresAt:    "2025-06-08T10:00:00Z"  // 7 days
    }

[ID] INITIATION WORKFLOW:

    1. Requester submits recovery request
    2. System verifies requester identity
    3. All guardians notified via registered channels
    4. Waiting period begins (optional)
    5. Guardians submit decisions


--------------------------------------------------------------------------------
7.4 GUARDIAN DECISION PROCESS
--------------------------------------------------------------------------------

[ID] GUARDIAN DECISION:

    GuardianDecision {
      requestId:         "RECOVERY-2025-001"
      guardianId:        "GUARD-2025-001"
      decision:          "APPROVE"  // APPROVE | REJECT
      timestamp:         "2025-06-02T14:30:00Z"
      verificationProof: "0x7f8e9a..."
      notes:             "Verified requester identity via video call"
    }

[ID] DECISION REQUIREMENTS:

    - Multi-factor authentication of guardian
    - Cryptographic signature on decision
    - Optional: Verification proof (video, in-person)
    - Time-stamped and logged immutably


--------------------------------------------------------------------------------
7.5 RECOVERY EXECUTION
--------------------------------------------------------------------------------

[ID] RECOVERY STATUS TRANSITIONS:

    PENDING   --> Guardian notifications sent
        |
        v
    AWAITING  --> Collecting guardian decisions
        |
        +----> APPROVED (threshold reached)
        |         |
        |         v
        |      EXECUTING --> Credentials issued
        |         |
        |         v
        |      COMPLETED --> Access granted
        |
        +----> REJECTED (threshold impossible)
        |
        +----> EXPIRED (time limit exceeded)

[ID] RECOVERY CREDENTIALS:

    RecoveryCredentials {
      requestId:    "RECOVERY-2025-001"
      identityId:   "CRYOID-2025-001"
      issuedTo:     "John Smith Jr."
      issuedAt:     "2025-06-03T10:00:00Z"
      validFor:     "24h"
      permissions:  ["IDENTITY_ACCESS", "CREDENTIAL_RECOVERY"]
      auditLog:     [/* guardian decisions */]
    }


--------------------------------------------------------------------------------
7.6 GUARDIAN MAINTENANCE
--------------------------------------------------------------------------------

[ID] MAINTENANCE TASKS:

    - Annual guardian verification
    - Contact information updates
    - Guardian replacement procedures
    - Threshold adjustment (with multi-guardian approval)

[ID] GUARDIAN REPLACEMENT:

    To replace a guardian:
    1. Existing guardians vote on replacement
    2. Threshold approval required
    3. New guardian onboarded
    4. Old guardian removed
    5. All changes logged

[!] GUARDIAN FAILURE SCENARIOS:

    - Guardian deceased: Replacement procedure initiated
    - Guardian unreachable: Timeout and proceed with remaining
    - Guardian compromised: Emergency revocation by threshold
