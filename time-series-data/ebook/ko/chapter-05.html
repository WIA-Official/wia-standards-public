5.1 시간 범위 쿼리.2 상세 설명

이 섹션에서는 구체적인 구현 방법과 실제 사용 시나리오를 다룹니다. 다음 표는 주요 옵션과 특징을 비교합니다:

5.1 시간 범위 쿼리.3 코드 예시

# 5.1 시간 범위 쿼리 구현 예시
-- 설정 및 초기화
CREATE TABLE time_series_data (
    time TIMESTAMPTZ NOT NULL,
    metric_name TEXT NOT NULL,
    value DOUBLE PRECISION,
    tags JSONB
);

-- 샘플 데이터 삽입
INSERT INTO time_series_data VALUES
  (NOW(), 'temperature', 23.5, '{"location": "office"}'),
  (NOW(), 'humidity', 65.2, '{"location": "office"}'),
  (NOW(), 'pressure', 1013.25, '{"location": "office"}');

-- 집계 쿼리
SELECT 
    time_bucket('5 minutes', time) AS bucket,
    metric_name,
    AVG(value) as avg_value,
    MIN(value) as min_value,
    MAX(value) as max_value
FROM time_series_data
WHERE time > NOW() - INTERVAL '1 hour'
GROUP BY bucket, metric_name
ORDER BY bucket DESC;

5.1 시간 범위 쿼리.4 모범 사례

  • ✅ 권장사항 1: 프로덕션 환경에서 검증된 패턴 사용
  • ✅ 권장사항 2: 성능 모니터링 및 지속적 최적화
  • ✅ 권장사항 3: 문서화 및 팀 간 지식 공유
  • ⚠️ 주의사항: 안티패턴 회피 및 잠재적 문제 사전 파악

-- 설정 및 초기화
CREATE TABLE time_series_data (
    time TIMESTAMPTZ NOT NULL,
    metric_name TEXT NOT NULL,
    value DOUBLE PRECISION,
    tags JSONB
);

-- 샘플 데이터 삽입
INSERT INTO time_series_data VALUES
  (NOW(), 'temperature', 23.5, '{"location": "office"}'),
  (NOW(), 'humidity', 65.2, '{"location": "office"}'),
  (NOW(), 'pressure', 1013.25, '{"location": "office"}');

-- 집계 쿼리
SELECT 
    time_bucket('5 minutes', time) AS bucket,
    metric_name,
    AVG(value) as avg_value,
    MIN(value) as min_value,
    MAX(value) as max_value
FROM time_series_data
WHERE time > NOW() - INTERVAL '1 hour'
GROUP BY bucket, metric_name
ORDER BY bucket DESC;
