제3장: 데이터 표준 및 프로토콜

상호 운용성은 효과적인 배터리 여권의 초석입니다. 이 장에서는 배터리 가치 사슬의 모든 참여자 간의 원활한 데이터 교환을 가능하게 하는 데이터 형식, 통신 프로토콜 및 API 사양을 정의합니다. 이러한 표준을 준수하면 관련된 시스템이나 조직에 관계없이 배터리 여권 데이터가 접근 가능하고 의미 있게 유지됩니다.

데이터 형식 사양

WIA-BATTERY_PASSPORT 표준은 JSON을 주요 데이터 교환 형식으로 사용합니다. 모든 데이터 교환은 정의된 JSON 스키마를 준수해야 하며, 스키마는 필수 필드, 데이터 유형, 유효성 검사 규칙 및 의미론적 의미를 지정합니다. 스키마는 버전이 지정되어 하위 호환성을 유지하면서 발전할 수 있습니다.

핵심 스키마 구조

{
    "$schema": "https://json-schema.org/draft/2020-12/schema",
    "$id": "https://wia.org/schemas/battery-passport/v1.0",
    "title": "WIA 배터리 여권 스키마",
    "type": "object",
    "required": ["passportId", "version", "identity", "technical"],
    "properties": {
        "passportId": {
            "type": "string",
            "pattern": "^WIA-BP-[0-9]{4}-[0-9]{6}$",
            "description": "배터리 여권의 고유 식별자"
        },
        "version": {
            "type": "string",
            "enum": ["1.0"],
            "description": "스키마 버전"
        },
        "identity": {
            "$ref": "#/$defs/IdentityBlock"
        },
        "technical": {
            "$ref": "#/$defs/TechnicalBlock"
        },
        "materials": {
            "$ref": "#/$defs/MaterialsBlock"
        },
        "carbonFootprint": {
            "$ref": "#/$defs/CarbonFootprintBlock"
        },
        "lifecycle": {
            "$ref": "#/$defs/LifecycleBlock"
        }
    }
}

통신 프로토콜

표준은 다양한 사용 사례와 통합 시나리오를 수용하기 위해 여러 통신 프로토콜을 정의합니다. 모든 프로토콜은 동기 및 비동기 작업을 모두 지원합니다.

REST API 프로토콜

대부분의 통합을 위한 주요 프로토콜은 RESTful HTTP API입니다. 모든 엔드포인트는 최소 TLS 1.3으로 HTTPS를 사용합니다. 인증은 JWT 토큰과 함께 OAuth 2.0을 사용하여 수행됩니다. 속도 제한 및 스로틀링은 남용으로부터 보호하면서 모든 참여자에게 공정한 접근을 보장합니다.

// REST API 엔드포인트
기본 URL: https://api.wia-passport.org/v1

GET    /passports/{id}           - 여권 조회
POST   /passports                - 새 여권 생성
PUT    /passports/{id}           - 여권 업데이트
DELETE /passports/{id}           - 여권 비활성화

GET    /passports/{id}/events    - 수명주기 이벤트 목록
POST   /passports/{id}/events    - 수명주기 이벤트 추가

GET    /passports/{id}/soh       - 현재 SoH 조회
GET    /passports/{id}/history   - SoH 이력 조회

POST   /verify                   - 여권 진위 검증

GraphQL 프로토콜

유연한 데이터 선택이 필요한 복잡한 쿼리의 경우 GraphQL 엔드포인트가 제공됩니다. 이는 여러 여권에서 데이터를 집계하거나 정보의 특정 하위 집합을 검색해야 하는 대시보드 응용 프로그램에 특히 유용합니다.

# GraphQL 쿼리 예제
query GetBatteryDetails($id: ID!) {
    passport(id: $id) {
        passportId
        identity {
            manufacturer {
                name
                country
            }
            productionDate
        }
        technical {
            chemistry
            nominalCapacity
            energyDensity
        }
        currentSoH
        lifecycle {
            events(last: 10) {
                eventType
                timestamp
                location {
                    facility
                    country
                }
            }
        }
    }
}

이벤트 스트리밍 프로토콜

실시간 모니터링 응용 프로그램의 경우, 표준은 WebSocket 연결 및 Apache Kafka 토픽을 통한 이벤트 스트리밍을 지원합니다. 이를 통해 수명주기 이벤트 및 지속적인 SoH 업데이트의 즉각적인 알림이 가능합니다.

// WebSocket 연결 예제
const ws = new WebSocket('wss://stream.wia-passport.org/v1');

ws.onopen = () => {
    ws.send(JSON.stringify({
        action: 'subscribe',
        passportIds: ['WIA-BP-2025-001234'],
        eventTypes: ['soh_update', 'incident_reported']
    }));
};

ws.onmessage = (event) => {
    const data = JSON.parse(event.data);
    console.log('이벤트 수신:', data);
};

식별자 시스템

고유 식별은 여권 시스템의 기본입니다. 표준은 인간 가독성을 유지하면서 글로벌 고유성을 보장하는 계층적 식별자 체계를 정의합니다.

여권 ID 형식

형식: WIA-BP-YYYY-NNNNNN

설명:
- WIA    = World Interoperability Alliance 접두사
- BP     = 배터리 여권 표시자
- YYYY   = 여권 생성 연도
- NNNNNN = 순차 번호 (000001-999999)

예시:
- WIA-BP-2025-000001 (2025년 첫 번째 여권)
- WIA-BP-2025-123456 (2025년 123,456번째 여권)

데이터 유효성 검사 규칙

여권 시스템에 제출되는 모든 데이터는 수락 전에 유효성 검사를 통과해야 합니다. 유효성 검사 프레임워크에는 스키마 유효성 검사, 비즈니스 규칙 유효성 검사, 교차 참조 유효성 검사가 포함됩니다.

  • 스키마 유효성 검사: JSON 구조가 정의된 스키마를 준수해야 함
  • 유형 유효성 검사: 값이 지정된 데이터 유형과 일치해야 함
  • 범위 유효성 검사: 숫자 값이 유효한 범위 내에 있어야 함
  • 형식 유효성 검사: 문자열이 필수 패턴과 일치해야 함(날짜, ID 등)
  • 참조 유효성 검사: 외래 키가 기존 엔티티를 참조해야 함
  • 시간적 유효성 검사: 이벤트가 논리적인 시간순을 가져야 함

API 인증 및 권한 부여

보안은 배터리 여권 시스템에서 가장 중요합니다. 인증 및 권한 부여 프레임워크는 검증된 참여자만 데이터에 접근할 수 있도록 보장하고, 역할 및 권한에 따라 접근이 적절하게 범위가 지정되도록 합니다.

// OAuth 2.0 토큰 요청
POST /oauth/token HTTP/1.1
Host: auth.wia-passport.org
Content-Type: application/x-www-form-urlencoded

grant_type=client_credentials
&client_id=org_123456
&client_secret=secret_value
&scope=passport:read passport:write events:write

// 응답
{
    "access_token": "eyJhbGciOiJSUzI1NiIs...",
    "token_type": "Bearer",
    "expires_in": 3600,
    "scope": "passport:read passport:write events:write"
}

// 토큰 사용
GET /v1/passports/WIA-BP-2025-001234 HTTP/1.1
Host: api.wia-passport.org
Authorization: Bearer eyJhbGciOiJSUzI1NiIs...

데이터 프라이버시 및 접근 제어

배터리 여권에는 공개 정보와 기밀 정보가 모두 포함됩니다. 접근 제어 시스템은 민감한 비즈니스 데이터를 보호하면서 다양한 이해관계자 범주에 적절한 가시성을 보장합니다.

  • 공개 접근: 기본 신원, 화학 유형, 용량, 인증
  • 소비자 접근: 환경 데이터, 안전 기록, 유지보수 이력
  • 운영자 접근: 상세 기술 데이터, 운영 이력
  • 제조업체 접근: 전체 생산 데이터, 공급망 세부 정보
  • 규제 기관 접근: 기밀 정보를 포함한 완전한 데이터 세트
