Chapter 7: Blockchain and Digital Identity

Blockchain technology provides the trust layer for battery passports, ensuring data immutability, transparent verification, and decentralized governance. This chapter explores how distributed ledger technology enables secure, tamper-proof battery passport systems that all stakeholders can trust.

Why Blockchain for Battery Passports

The battery passport ecosystem involves numerous stakeholders with varying interests: manufacturers want to protect proprietary information, consumers want transparency, regulators need verification, and recyclers need reliable data. Blockchain technology addresses these needs through its fundamental properties.

  • Immutability: Once recorded, data cannot be altered or deleted without detection
  • Transparency: All authorized parties can view the same verified data
  • Decentralization: No single entity controls the system
  • Auditability: Complete history of all changes is preserved
  • Trust: Cryptographic verification eliminates need for trusted intermediaries

WIA-CHAIN Architecture

The WIA-BATTERY_PASSPORT standard specifies integration with WIA-CHAIN, a purpose-built blockchain network optimized for product passport applications. WIA-CHAIN uses a hybrid architecture combining the security of public blockchains with the privacy features needed for commercial applications.

Smart Contract Interface

Battery passports are registered and managed through smart contracts on WIA-CHAIN. The core contracts provide functions for passport creation, event logging, ownership transfer, and verification.

// PassportRegistry Smart Contract (Solidity-like pseudocode)
contract PassportRegistry {
    
    struct Passport {
        bytes32 passportId;
        address manufacturer;
        address currentOwner;
        bytes32 dataHash;
        uint256 createdAt;
        uint256 updatedAt;
        bool isActive;
    }
    
    mapping(bytes32 => Passport) public passports;
    mapping(bytes32 => bytes32[]) public eventHashes;
    
    event PassportCreated(bytes32 indexed passportId, address manufacturer);
    event PassportUpdated(bytes32 indexed passportId, bytes32 dataHash);
    event OwnershipTransferred(bytes32 indexed passportId, address from, address to);
    event EventLogged(bytes32 indexed passportId, bytes32 eventHash);
    
    function createPassport(
        bytes32 passportId,
        bytes32 initialDataHash
    ) external onlyManufacturer {
        require(passports[passportId].createdAt == 0, "Passport exists");
        
        passports[passportId] = Passport({
            passportId: passportId,
            manufacturer: msg.sender,
            currentOwner: msg.sender,
            dataHash: initialDataHash,
            createdAt: block.timestamp,
            updatedAt: block.timestamp,
            isActive: true
        });
        
        emit PassportCreated(passportId, msg.sender);
    }
    
    function logEvent(
        bytes32 passportId,
        bytes32 eventHash
    ) external onlyAuthorized(passportId) {
        eventHashes[passportId].push(eventHash);
        passports[passportId].updatedAt = block.timestamp;
        emit EventLogged(passportId, eventHash);
    }
    
    function verifyPassport(
        bytes32 passportId,
        bytes32 expectedDataHash
    ) external view returns (bool) {
        return passports[passportId].dataHash == expectedDataHash;
    }
}

Data Privacy on Blockchain

While blockchain provides transparency, some passport data is commercially sensitive. The WIA-BATTERY_PASSPORT standard addresses this through a hybrid on-chain/off-chain architecture.

Data Storage Architecture

  • On-Chain (Public): Passport ID, data hashes, ownership records, event timestamps
  • Off-Chain (Encrypted): Detailed technical data, supplier information, pricing
  • Access Control: Role-based permissions enforced by smart contracts

Verification Process

Anyone can verify the authenticity and integrity of a battery passport using the blockchain. The verification process confirms that the passport data matches the on-chain hash and that all events have been properly recorded.

// Passport Verification Flow
async function verifyPassport(passportId: string): Promise<VerificationResult> {
    // 1. Fetch on-chain record
    const onChainRecord = await passportRegistry.getPassport(passportId);
    
    // 2. Fetch off-chain data
    const offChainData = await passportStorage.getPassport(passportId);
    
    // 3. Calculate hash of off-chain data
    const calculatedHash = keccak256(JSON.stringify(offChainData));
    
    // 4. Compare hashes
    const dataIntegrity = calculatedHash === onChainRecord.dataHash;
    
    // 5. Verify event chain
    const events = await passportRegistry.getEvents(passportId);
    const eventsValid = await verifyEventChain(events, offChainData.events);
    
    // 6. Verify signatures
    const signaturesValid = await verifyAllSignatures(offChainData);
    
    return {
        passportId,
        valid: dataIntegrity && eventsValid && signaturesValid,
        checks: {
            dataIntegrity,
            eventsValid,
            signaturesValid
        },
        timestamp: new Date().toISOString()
    };
}
