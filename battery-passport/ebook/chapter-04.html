Chapter 4: API Protocol Design

REST Endpoints, QR Code Encoding, and Authentication

4.1 API Overview

[API] Base URL

https://api.wia.org/v1/battery-passport

4.2 Battery Passport Endpoints

POST /passports

Create a new battery passport. Only authorized manufacturers can issue passports.

{
  "batteryId": "BAT-2024-001234",
  "category": "ev_battery",
  "model": "94Ah NMC811",
  "chemistry": "NMC811",
  "capacity": { "value": 75, "unit": "kWh" },
  "carbonFootprint": { "value": 61, "unit": "kg CO2eq/kWh" },
  "recycledContent": { "cobalt": 16, "lithium": 6, "nickel": 6 }
}
GET /passports/{id}

Retrieve a battery passport by DID or battery ID.

GET /passports/{id}/compliance

Check EU Regulation 2023/1542 compliance status.

// Response
{
  "compliant": true,
  "checks": {
    "carbonFootprint": { "status": "pass", "value": 61, "class": "B" },
    "recycledCobalt": { "status": "pass", "value": 16, "required": 16 },
    "recycledLithium": { "status": "pass", "value": 6, "required": 6 },
    "dueDiligence": { "status": "pass" },
    "qrCode": { "status": "pass" }
  }
}

4.3 QR Code Encoding

Each battery must have a QR code providing access to the digital passport:

QR Code Content
// Encoded JWT in QR Code
{
  "iss": "did:wia:manufacturer:samsung-sdi",
  "sub": "did:wia:battery:BAT-2024-001234",
  "bat": "BAT-2024-001234",
  "cat": "ev",
  "exp": 1893456000,
  "url": "https://dpp.wia.org/b/abc123"
}

// Scan result redirects to:
https://dpp.wia.org/b/abc123
// Shows full passport data with EU compliance status

4.4 Authentication

[KEY] Permission Scopes

  • passport:read - Public read access
  • passport:write - Manufacturers only
  • soh:write - Update State of Health
  • compliance:verify - EU authorities
