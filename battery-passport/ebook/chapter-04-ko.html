챕터 4

건강 상태(SoH) 모니터링

챕터 개요: 이 챕터에서는 배터리의 건강 상태(SoH) 모니터링 방법론과 구현을 다룹니다. 정확한 SoH 평가는 안전성, 보증 관리, 2차 수명 적용 및 수명 종료 결정에 중요합니다.

건강 상태 이해하기

건강 상태(SoH)는 배터리의 원래 사양 대비 상태를 측정하는 지표입니다. 배터리가 사용과 시간을 통해 노화됨에 따라 용량이 감소하고 내부 저항이 증가합니다. SoH는 이 열화를 정량화하며, 일반적으로 100%가 새 배터리를 나타내고 낮은 값이 점진적 노화를 나타내는 백분율로 표현됩니다.

EU 배터리 규정은 배터리 관리 시스템이 SoH 값을 결정하고 보고할 수 있어야 한다고 의무화합니다. 이 데이터는 배터리 여권 시스템을 통해 접근 가능해야 하며 배터리의 운영 수명 전체에 걸쳐 업데이트되어야 합니다.

주요 건강 지표

포괄적인 SoH 평가에는 배터리 노화 및 열화의 다양한 측면을 나타내는 여러 매개변수를 모니터링해야 합니다.

건강 상태 - 용량 (SoH-C)

초기 정격 용량 대비 남은 용량을 측정합니다. 대부분의 응용에서 주요 SoH 지표입니다.

SoH-C = (현재 용량 / 정격 용량) x 100%

건강 상태 - 저항 (SoH-R)

출력 능력과 효율성에 영향을 미치는 내부 저항의 증가를 추적합니다.

SoH-R = (초기 저항 / 현재 저항) x 100%

건강 상태 - 출력 (SoH-P)

고성능 응용에 중요한 남은 출력 공급 능력을 나타냅니다.

SoH-P = (현재 최대 출력 / 정격 최대 출력) x 100%

SoH 추정 방법

정확한 SoH 추정에는 사용 가능한 센서 데이터로 작동하고 실제 배터리 작동의 복잡성을 처리할 수 있는 정교한 알고리즘이 필요합니다.

실시간 모니터링 시스템

WIA-BATTERY_PASSPORT 표준은 SoH 모니터링을 배터리 여권 시스템과 통합하기 위한 인터페이스를 정의합니다. 실시간 데이터는 배터리 관리 시스템에서 클라우드 기반 분석으로 흐르고 궁극적으로 여권 기록에 기록됩니다.

// TypeScript: SoH 모니터링 서비스
interface SoHMonitoringService {
  readonly batteryId: string;
  readonly bmsConnector: BMSDataConnector;
  readonly estimationEngine: SoHEstimationEngine;
  readonly passportClient: PassportAPIClient;

  startMonitoring(): Promise<void>;
  getCurrentSoH(): SoHSnapshot;
  getSoHHistory(timeRange: TimeRange): SoHTimeSeries;
  predictRemainingLife(): RULPrediction;
}

class WIASoHMonitor implements SoHMonitoringService {
  private lastUpdate: Date;
  private currentSnapshot: SoHSnapshot;

  async startMonitoring(): Promise<void> {
    // BMS 데이터 스트림 구독
    this.bmsConnector.subscribe({
      parameters: ['voltage', 'current', 'temperature', 'soc'],
      interval: 1000, // 밀리초
      callback: this.onBMSData.bind(this)
    });

    // 주기적 SoH 추정 시작
    setInterval(() => this.performSoHEstimation(), 3600000); // 매시간
  }

  private async onBMSData(data: BMSDataPacket): void {
    // 추정 알고리즘을 위한 데이터 버퍼링
    this.dataBuffer.push({
      timestamp: Date.now(),
      voltage: data.packVoltage,
      current: data.packCurrent,
      temperature: data.averageTemperature,
      cellVoltages: data.cellVoltages
    });

    // 이상 확인
    const anomaly = this.anomalyDetector.analyze(data);
    if (anomaly.detected) {
      await this.handleAnomaly(anomaly);
    }
  }

  private async performSoHEstimation(): Promise<void> {
    // 추정 알고리즘 실행
    const estimate = await this.estimationEngine.estimate(this.dataBuffer);

    // 추정 품질 검증
    if (estimate.confidence > 0.9) {
      this.currentSnapshot = {
        sohCapacity: estimate.sohC,
        sohResistance: estimate.sohR,
        sohPower: estimate.sohP,
        timestamp: new Date(),
        confidence: estimate.confidence,
        method: estimate.method
      };

      // 여권 기록 업데이트
      await this.updatePassportRecord(this.currentSnapshot);
    }
  }
}

이상 감지 및 경고

일상적인 SoH 추적 외에도 모니터링 시스템은 안전 문제 또는 가속화된 열화를 나타낼 수 있는 이상을 감지해야 합니다.

이상 유형 감지 방법 대응 조치
급격한 용량 저하 통계적 편차 추적 서비스 알림, 진단 요청
셀 불균형 전압 편차 모니터링 밸런싱 증가, 조사
열 이상 온도 기울기 분석 출력 제한, 안전 경고
저항 급증 임피던스 분광법 연결 점검, 셀 분석
안전 우선: SoH 모니터링 시스템은 다른 모든 고려 사항보다 안전을 우선시합니다. 안전 위험을 나타낼 수 있는 모든 이상은 즉각적인 경고를 트리거하고 배터리 관리 시스템을 통해 보호 조치를 시작할 수 있습니다.
