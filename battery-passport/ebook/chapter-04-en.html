Chapter 4

State of Health (SoH) Monitoring

Chapter Overview: This chapter covers the methodologies and implementation of State of Health (SoH) monitoring for batteries. Accurate SoH assessment is critical for safety, warranty management, second-life applications, and end-of-life decisions.

Understanding State of Health

State of Health (SoH) is a measure of a battery's condition relative to its original specifications. As batteries age through use and calendar time, their capacity decreases and internal resistance increases. SoH quantifies this degradation, typically expressed as a percentage where 100% represents a new battery and lower values indicate progressive aging.

The EU Battery Regulation mandates that battery management systems must be capable of determining and reporting SoH values. This data must be accessible through the battery passport system and updated throughout the battery's operational life.

Key Health Metrics

Comprehensive SoH assessment requires monitoring multiple parameters that indicate different aspects of battery aging and degradation.

State of Health - Capacity (SoH-C)

Measures remaining capacity relative to initial rated capacity. This is the primary SoH metric for most applications.

SoH-C = (Current Capacity / Rated Capacity) x 100%

State of Health - Resistance (SoH-R)

Tracks increase in internal resistance, which affects power capability and efficiency.

SoH-R = (Initial Resistance / Current Resistance) x 100%

State of Health - Power (SoH-P)

Indicates remaining power delivery capability, critical for high-performance applications.

SoH-P = (Current Max Power / Rated Max Power) x 100%

SoH Estimation Methods

Accurate SoH estimation requires sophisticated algorithms that can work with available sensor data and handle the complexities of real-world battery operation.

Real-Time Monitoring System

The WIA-BATTERY_PASSPORT standard defines interfaces for integrating SoH monitoring with the battery passport system. Real-time data flows from the battery management system to cloud-based analytics and ultimately to the passport record.

// TypeScript: SoH Monitoring Service
interface SoHMonitoringService {
  readonly batteryId: string;
  readonly bmsConnector: BMSDataConnector;
  readonly estimationEngine: SoHEstimationEngine;
  readonly passportClient: PassportAPIClient;

  startMonitoring(): Promise<void>;
  getCurrentSoH(): SoHSnapshot;
  getSoHHistory(timeRange: TimeRange): SoHTimeSeries;
  predictRemainingLife(): RULPrediction;
}

class WIASoHMonitor implements SoHMonitoringService {
  private lastUpdate: Date;
  private currentSnapshot: SoHSnapshot;

  async startMonitoring(): Promise<void> {
    // Subscribe to BMS data stream
    this.bmsConnector.subscribe({
      parameters: ['voltage', 'current', 'temperature', 'soc'],
      interval: 1000, // milliseconds
      callback: this.onBMSData.bind(this)
    });

    // Start periodic SoH estimation
    setInterval(() => this.performSoHEstimation(), 3600000); // hourly
  }

  private async onBMSData(data: BMSDataPacket): void {
    // Buffer data for estimation algorithms
    this.dataBuffer.push({
      timestamp: Date.now(),
      voltage: data.packVoltage,
      current: data.packCurrent,
      temperature: data.averageTemperature,
      cellVoltages: data.cellVoltages
    });

    // Check for anomalies
    const anomaly = this.anomalyDetector.analyze(data);
    if (anomaly.detected) {
      await this.handleAnomaly(anomaly);
    }
  }

  private async performSoHEstimation(): Promise<void> {
    // Run estimation algorithm
    const estimate = await this.estimationEngine.estimate(this.dataBuffer);

    // Validate estimate quality
    if (estimate.confidence > 0.9) {
      this.currentSnapshot = {
        sohCapacity: estimate.sohC,
        sohResistance: estimate.sohR,
        sohPower: estimate.sohP,
        timestamp: new Date(),
        confidence: estimate.confidence,
        method: estimate.method
      };

      // Update passport record
      await this.updatePassportRecord(this.currentSnapshot);
    }
  }

  async updatePassportRecord(snapshot: SoHSnapshot): Promise<void> {
    await this.passportClient.updateSoH({
      batteryId: this.batteryId,
      sohData: {
        capacity: snapshot.sohCapacity,
        resistance: snapshot.sohResistance,
        power: snapshot.sohPower,
        timestamp: snapshot.timestamp.toISOString(),
        estimationMethod: snapshot.method,
        confidence: snapshot.confidence
      }
    });
  }
}

Anomaly Detection and Alerts

Beyond routine SoH tracking, the monitoring system must detect anomalies that could indicate safety issues or accelerated degradation.

Anomaly Type Detection Method Response Action
Sudden capacity drop Statistical deviation tracking Service alert, diagnostic request
Cell imbalance Voltage deviation monitoring Increased balancing, investigation
Thermal anomaly Temperature gradient analysis Power limiting, safety alert
Resistance spike Impedance spectroscopy Connection inspection, cell analysis
Safety Priority: The SoH monitoring system prioritizes safety above all other considerations. Any anomaly that could indicate a safety risk triggers immediate alerts and may initiate protective actions through the battery management system.
