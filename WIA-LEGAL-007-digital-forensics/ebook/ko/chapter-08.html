Chapter 8: 디지털 증거 법정 제출 절차

디지털 포렌식의 궁극적인 목적은 법정에서 증거 능력을 인정받는 것입니다. 아무리 정교한 기술적 분석이라도 법적 요건을 충족하지 못하면 증거로 채택되지 않습니다. 이 장에서는 디지털 증거의 수집부터 법정 제출까지의 전 과정을 다루며, 증거의 무결성 보장, 체인 오브 커스터디 유지, 전문가 증언 방법, 그리고 각국의 법적 프레임워크를 학습합니다.

섹션 1: 디지털 증거의 법적 요건

디지털 증거가 법정에서 인정받기 위해서는 적법성(Legality), 신뢰성(Reliability), 무결성(Integrity), 관련성(Relevance)의 네 가지 요건을 충족해야 합니다. 적법성은 증거 수집 과정이 법적 절차를 준수했는지를 의미하며, 영장 없는 수색이나 사생활 침해는 증거 배제 사유가 됩니다. 신뢰성은 분석 방법과 도구의 과학적 타당성을 의미하며, 표준화된 방법론과 검증된 도구 사용이 필수입니다. 무결성은 증거가 수집 이후 변조되지 않았음을 입증하는 것으로, 해시값과 체인 오브 커스터디가 핵심입니다.

증거 능력과 증명력

증거 능력(Admissibility)은 증거가 법정에서 사용될 자격이 있는지를 판단하는 것이고, 증명력(Probative Value)은 그 증거가 사실 인정에 얼마나 기여하는지를 의미합니다. 디지털 증거는 복제가 용이하고 변조의 위험이 있어 전통적 증거보다 엄격한 기준이 적용됩니다. 원본성(Authenticity) 입증, 최량 증거 원칙(Best Evidence Rule) 준수, 전문 증거 예외(Hearsay Exception) 적용 여부가 주요 쟁점이 됩니다.

표 8.1: 디지털 증거 법적 요건
요건 정의 입증 방법 관련 법규
적법성 수집 절차의 합법성 영장, 동의서 형사소송법, 통신비밀보호법
신뢰성 방법론의 과학적 타당성 표준 절차, 인증 도구 Daubert 기준
무결성 변조 없음 입증 해시값, CoC 전자서명법
관련성 사건과의 연관성 맥락 분석 증거법

섹션 2: 체인 오브 커스터디

체인 오브 커스터디(Chain of Custody, CoC)는 증거의 수집부터 법정 제출까지 전 과정에서 증거가 누구의 관리 하에 있었는지를 문서화한 기록입니다. 증거가 언제, 어디서, 누구에 의해 수집되었고, 이후 어떻게 보관, 분석, 전달되었는지를 빠짐없이 기록해야 합니다. CoC의 단절은 증거 오염 또는 조작의 가능성을 제기하여 증거 능력을 부정하는 사유가 될 수 있습니다.

// 체인 오브 커스터디 관리 시스템
interface ChainOfCustodyEntry {
  id: string;
  evidenceId: string;
  timestamp: Date;
  action: CoCAction;
  performer: PersonInfo;
  location: string;
  description: string;
  previousHash: string;
  currentHash: string;
  digitalSignature: string;
  witnesses?: PersonInfo[];
  attachments?: Attachment[];
}

type CoCAction =
  | 'collection'
  | 'transport'
  | 'storage'
  | 'retrieval'
  | 'analysis'
  | 'copy'
  | 'transfer'
  | 'court_submission'
  | 'return'
  | 'destruction';

interface PersonInfo {
  name: string;
  title: string;
  organization: string;
  badgeNumber?: string;
  contactInfo: string;
  signature?: string;
}

interface EvidenceItem {
  id: string;
  caseNumber: string;
  description: string;
  type: 'physical' | 'digital' | 'hybrid';
  originalLocation: string;
  collectionDate: Date;
  collector: PersonInfo;
  initialHashes: HashSet;
  currentStatus: EvidenceStatus;
  chainOfCustody: ChainOfCustodyEntry[];
  forensicImages?: ForensicImage[];
}

interface HashSet {
  md5: string;
  sha1: string;
  sha256: string;
}

type EvidenceStatus =
  | 'collected'
  | 'in_transit'
  | 'in_storage'
  | 'under_analysis'
  | 'submitted_to_court'
  | 'returned'
  | 'destroyed';

class ChainOfCustodyManager {
  private evidence: Map<string, EvidenceItem>;
  private privateKey: Buffer;

  constructor(privateKey: Buffer) {
    this.evidence = new Map();
    this.privateKey = privateKey;
  }

  async collectEvidence(
    params: CollectionParams
  ): Promise<EvidenceItem> {
    // 증거 ID 생성
    const evidenceId = this.generateEvidenceId(params.caseNumber);

    // 초기 해시 계산
    const initialHashes = await this.calculateHashes(params.data);

    // 증거 항목 생성
    const evidence: EvidenceItem = {
      id: evidenceId,
      caseNumber: params.caseNumber,
      description: params.description,
      type: params.type,
      originalLocation: params.location,
      collectionDate: new Date(),
      collector: params.collector,
      initialHashes,
      currentStatus: 'collected',
      chainOfCustody: []
    };

    // 첫 번째 CoC 엔트리 추가
    const cocEntry = await this.createCoCEntry({
      evidenceId,
      action: 'collection',
      performer: params.collector,
      location: params.location,
      description: `증거 최초 수집: ${params.description}`,
      previousHash: '',
      currentHash: initialHashes.sha256,
      witnesses: params.witnesses
    });

    evidence.chainOfCustody.push(cocEntry);

    this.evidence.set(evidenceId, evidence);

    return evidence;
  }

  async transferEvidence(
    evidenceId: string,
    from: PersonInfo,
    to: PersonInfo,
    location: string,
    reason: string
  ): Promise<ChainOfCustodyEntry> {
    const evidence = this.evidence.get(evidenceId);
    if (!evidence) {
      throw new Error(`Evidence not found: ${evidenceId}`);
    }

    // 현재 해시 검증
    const currentHashes = await this.verifyIntegrity(evidenceId);

    // 이전 CoC 엔트리의 해시
    const lastEntry = evidence.chainOfCustody[evidence.chainOfCustody.length - 1];

    // 새 CoC 엔트리 생성
    const cocEntry = await this.createCoCEntry({
      evidenceId,
      action: 'transfer',
      performer: from,
      location,
      description: `증거 인계: ${from.name} → ${to.name}. 사유: ${reason}`,
      previousHash: lastEntry.currentHash,
      currentHash: currentHashes.sha256,
      witnesses: [to]
    });

    evidence.chainOfCustody.push(cocEntry);

    return cocEntry;
  }

  async createForensicImage(
    evidenceId: string,
    analyst: PersonInfo,
    imagePath: string
  ): Promise<ForensicImage> {
    const evidence = this.evidence.get(evidenceId);
    if (!evidence) {
      throw new Error(`Evidence not found: ${evidenceId}`);
    }

    // 포렌식 이미지 해시 계산
    const imageHashes = await this.calculateHashes(
      await fs.readFile(imagePath)
    );

    const forensicImage: ForensicImage = {
      id: `IMG-${evidenceId}-${Date.now()}`,
      originalEvidenceId: evidenceId,
      path: imagePath,
      hashes: imageHashes,
      creationDate: new Date(),
      creator: analyst,
      tool: 'FTK Imager v4.7',
      toolVersion: '4.7.0.0',
      verificationStatus: 'verified'
    };

    // CoC 엔트리 추가
    const cocEntry = await this.createCoCEntry({
      evidenceId,
      action: 'copy',
      performer: analyst,
      location: analyst.organization,
      description: `포렌식 이미지 생성: ${imagePath}`,
      previousHash: evidence.chainOfCustody[evidence.chainOfCustody.length - 1].currentHash,
      currentHash: imageHashes.sha256
    });

    evidence.chainOfCustody.push(cocEntry);
    evidence.forensicImages = evidence.forensicImages || [];
    evidence.forensicImages.push(forensicImage);

    return forensicImage;
  }

  async verifyIntegrity(evidenceId: string): Promise<HashSet> {
    const evidence = this.evidence.get(evidenceId);
    if (!evidence) {
      throw new Error(`Evidence not found: ${evidenceId}`);
    }

    // 현재 해시 계산 (실제 구현에서는 파일 읽기)
    // const currentHashes = await this.calculateHashes(evidenceData);

    // 해시 비교
    // if (currentHashes.sha256 !== evidence.initialHashes.sha256) {
    //   throw new IntegrityError('Evidence has been modified');
    // }

    return evidence.initialHashes;
  }

  private async createCoCEntry(
    params: Partial<ChainOfCustodyEntry>
  ): Promise<ChainOfCustodyEntry> {
    const entry: ChainOfCustodyEntry = {
      id: `COC-${Date.now()}-${Math.random().toString(36).substr(2, 9)}`,
      evidenceId: params.evidenceId!,
      timestamp: new Date(),
      action: params.action!,
      performer: params.performer!,
      location: params.location!,
      description: params.description!,
      previousHash: params.previousHash!,
      currentHash: params.currentHash!,
      digitalSignature: '',
      witnesses: params.witnesses
    };

    // 디지털 서명 생성
    const signatureData = JSON.stringify({
      id: entry.id,
      timestamp: entry.timestamp,
      action: entry.action,
      currentHash: entry.currentHash
    });

    entry.digitalSignature = this.sign(signatureData);

    return entry;
  }

  private sign(data: string): string {
    const crypto = require('crypto');
    const sign = crypto.createSign('SHA256');
    sign.update(data);
    return sign.sign(this.privateKey, 'hex');
  }

  async generateReport(evidenceId: string): Promise<CoCReport> {
    const evidence = this.evidence.get(evidenceId);
    if (!evidence) {
      throw new Error(`Evidence not found: ${evidenceId}`);
    }

    return {
      evidenceId: evidence.id,
      caseNumber: evidence.caseNumber,
      description: evidence.description,
      collectionInfo: {
        date: evidence.collectionDate,
        location: evidence.originalLocation,
        collector: evidence.collector
      },
      initialHashes: evidence.initialHashes,
      chainOfCustody: evidence.chainOfCustody.map(entry => ({
        timestamp: entry.timestamp,
        action: entry.action,
        performer: entry.performer.name,
        organization: entry.performer.organization,
        location: entry.location,
        description: entry.description,
        verified: this.verifySignature(entry)
      })),
      forensicImages: evidence.forensicImages,
      integrityStatus: 'verified',
      reportGeneratedAt: new Date(),
      reportGeneratedBy: 'ChainOfCustodyManager'
    };
  }
}

섹션 3: 포렌식 보고서 작성

포렌식 보고서는 기술적 분석 결과를 법률 전문가와 법관이 이해할 수 있는 형태로 전달하는 문서입니다. 보고서는 명확하고 객관적이어야 하며, 결론을 뒷받침하는 증거와 분석 과정을 상세히 기술해야 합니다. 기술 용어는 정의와 함께 사용하고, 시각적 자료(스크린샷, 다이어그램)를 활용하여 이해를 돕습니다. 보고서는 법정에서 전문가 증언의 기초가 되므로 사실과 의견을 명확히 구분해야 합니다.

표 8.2: 포렌식 보고서 구성 요소

부록 기술 상세, 로그 검증 자료 
// 포렌식 보고서 생성 시스템
interface ForensicReport {
  reportId: string;
  caseInfo: CaseInfo;
  executiveSummary: string;
  requestInfo: RequestInfo;
  evidenceList: EvidenceDescription[];
  methodology: MethodologySection;
  findings: Finding[];
  conclusions: Conclusion[];
  expertInfo: ExpertInfo;
  appendices: Appendix[];
  generatedAt: Date;
  version: string;
}

interface CaseInfo {
  caseNumber: string;
  caseName: string;
  investigatingAgency: string;
  investigator: string;
  dateReceived: Date;
  dateCompleted: Date;
}

interface Finding {
  id: string;
  category: FindingCategory;
  title: string;
  description: string;
  evidence: EvidenceReference[];
  significance: 'high' | 'medium' | 'low';
  confidence: number;
  technicalDetails?: string;
  screenshots?: Screenshot[];
  timeline?: TimelineEntry[];
}

type FindingCategory =
  | 'user_activity'
  | 'file_access'
  | 'communication'
  | 'internet_history'
  | 'malware'
  | 'data_exfiltration'
  | 'timeline_reconstruction'
  | 'deleted_data_recovery';

interface Conclusion {
  question: string;
  answer: string;
  supportingFindings: string[];
  confidence: 'certain' | 'probable' | 'possible' | 'inconclusive';
  limitations?: string[];
}

class ForensicReportGenerator {
  private analysisData: AnalysisData;
  private caseInfo: CaseInfo;
  private expert: ExpertInfo;

  constructor(
    analysisData: AnalysisData,
    caseInfo: CaseInfo,
    expert: ExpertInfo
  ) {
    this.analysisData = analysisData;
    this.caseInfo = caseInfo;
    this.expert = expert;
  }

  async generateReport(): Promise<ForensicReport> {
    const reportId = this.generateReportId();

    // 발견 사항 컴파일
    const findings = this.compileFindings();

    // 타임라인 구성
    const timeline = this.constructTimeline();

    // 결론 도출
    const conclusions = this.drawConclusions(findings);

    // 요약 작성
    const executiveSummary = this.writeExecutiveSummary(findings, conclusions);

    return {
      reportId,
      caseInfo: this.caseInfo,
      executiveSummary,
      requestInfo: this.analysisData.request,
      evidenceList: this.formatEvidenceList(),
      methodology: this.documentMethodology(),
      findings,
      conclusions,
      expertInfo: this.expert,
      appendices: this.compileAppendices(),
      generatedAt: new Date(),
      version: '1.0'
    };
  }

  private compileFindings(): Finding[] {
    const findings: Finding[] = [];

    // 사용자 활동 분석 결과
    if (this.analysisData.userActivity) {
      for (const activity of this.analysisData.userActivity) {
        findings.push({
          id: `F-${findings.length + 1}`,
          category: 'user_activity',
          title: `사용자 활동: ${activity.type}`,
          description: this.formatActivityDescription(activity),
          evidence: activity.evidenceRefs,
          significance: this.assessSignificance(activity),
          confidence: activity.confidence,
          timeline: activity.timeline
        });
      }
    }

    // 파일 접근 분석 결과
    if (this.analysisData.fileAccess) {
      for (const access of this.analysisData.fileAccess) {
        findings.push({
          id: `F-${findings.length + 1}`,
          category: 'file_access',
          title: `파일 접근: ${access.filePath}`,
          description: this.formatFileAccessDescription(access),
          evidence: access.evidenceRefs,
          significance: this.assessSignificance(access),
          confidence: access.confidence,
          screenshots: access.screenshots
        });
      }
    }

    // 통신 분석 결과
    if (this.analysisData.communications) {
      findings.push({
        id: `F-${findings.length + 1}`,
        category: 'communication',
        title: '통신 기록 분석',
        description: this.formatCommunicationSummary(),
        evidence: this.analysisData.communications.evidenceRefs,
        significance: 'high',
        confidence: 95
      });
    }

    // 삭제된 데이터 복구
    if (this.analysisData.recoveredData) {
      for (const recovered of this.analysisData.recoveredData) {
        findings.push({
          id: `F-${findings.length + 1}`,
          category: 'deleted_data_recovery',
          title: `복구된 데이터: ${recovered.type}`,
          description: this.formatRecoveryDescription(recovered),
          evidence: recovered.evidenceRefs,
          significance: 'high',
          confidence: recovered.confidence,
          technicalDetails: recovered.recoveryMethod
        });
      }
    }

    return findings;
  }

  private drawConclusions(findings: Finding[]): Conclusion[] {
    const conclusions: Conclusion[] = [];

    // 의뢰 사항별 결론 도출
    for (const question of this.analysisData.request.questions) {
      const relevantFindings = findings.filter(f =>
        this.isRelevantToQuestion(f, question)
      );

      const answer = this.formAnswer(question, relevantFindings);
      const confidence = this.assessConfidence(relevantFindings);

      conclusions.push({
        question: question.text,
        answer,
        supportingFindings: relevantFindings.map(f => f.id),
        confidence,
        limitations: this.identifyLimitations(question, relevantFindings)
      });
    }

    return conclusions;
  }

                 분석은 포렌식 복사본에서만 수행되어 원본 증거의 무결성이 보존되었습니다.`,

  private writeExecutiveSummary(
    findings: Finding[],
    conclusions: Conclusion[]
  ): string {
    const highSigFindings = findings.filter(f => f.significance === 'high');

    let summary = `## 요약\n\n`;
    summary += `본 보고서는 사건번호 ${this.caseInfo.caseNumber}과 관련하여 `;

    summary += `디지털 포렌식 분석 결과입니다.\n\n`;

    summary += `### 분석 범위\n`;
    summary += `총 ${this.analysisData.evidence.length}개의 증거물에 대해 분석이 수행되었습니다.\n\n`;

    summary += `### 주요 발견 사항\n`;
    for (const finding of highSigFindings.slice(0, 5)) {
      summary += `- ${finding.title}: ${finding.description.substring(0, 100)}...\n`;
    }
    summary += `\n`;

    summary += `### 결론\n`;
    for (const conclusion of conclusions) {
      summary += `**${conclusion.question}**\n`;
      summary += `${conclusion.answer}\n`;
      summary += `(신뢰도: ${this.translateConfidence(conclusion.confidence)})\n\n`;
    }

    return summary;
  }

  private translateConfidence(
    confidence: 'certain' | 'probable' | 'possible' | 'inconclusive'
  ): string {
    const translations: Record<string, string> = {
      'certain': '확실함',
      'probable': '높음',
      'possible': '가능성 있음',
      'inconclusive': '결론 불가'
    };
    return translations[confidence];
  }

섹션 5: 국제 법적 프레임워크

디지털 증거에 관한 법적 프레임워크는 국가마다 다르며, 국경을 넘는 사이버 범죄 수사에서는 복잡한 관할권 문제가 발생합니다. 부다페스트 협약(Convention on Cybercrime)은 사이버 범죄에 대한 국제 협력의 기초를 제공합니다. MLAT(상호법률지원조약)는 국가 간 증거 공유의 법적 근거가 됩니다. EU의 GDPR은 개인 데이터 처리에 관한 엄격한 규정을 두고 있어 포렌식 조사 시 고려해야 합니다.

// 법적 준수 검사 시스템
interface JurisdictionInfo {

interface LegalComplianceCheck {
  jurisdiction: string;
  collectionMethod: string;
  complianceStatus: 'compliant' | 'non_compliant' | 'review_required';
  issues: ComplianceIssue[];
  recommendations: string[];
}

class LegalComplianceChecker {
  private jurisdictions: Map<string, JurisdictionInfo>;

  checkCompliance(
    collection: EvidenceCollection,
    jurisdiction: string
  ): LegalComplianceCheck {
    const jurisdictionInfo = this.jurisdictions.get(jurisdiction);
    if (!jurisdictionInfo) {
      throw new Error(`Unknown jurisdiction: ${jurisdiction}`);
    }

    const issues: ComplianceIssue[] = [];
    const recommendations: string[] = [];

    // 영장/법적 근거 확인
    const warrantReq = jurisdictionInfo.evidenceRequirements.find(r =>
      r.type === 'warrant' || r.type === 'legal_basis'
    );

    if (warrantReq?.required && !collection.legalBasis) {
      issues.push({
        severity: 'critical',
        requirement: warrantReq.description,
        issue: '법적 근거 미확보',
        recommendation: `${warrantReq.description} 확보 필요`
      });
    }

    // 무결성 검증 확인
    const hashReq = jurisdictionInfo.evidenceRequirements.find(r =>
      r.type === 'hash_verification' || r.type === 'authentication'
    );

    if (hashReq?.required && !collection.hashVerified) {
      issues.push({
        severity: 'high',
        requirement: hashReq.description,
        issue: '해시 검증 미수행',
        recommendation: 'MD5/SHA-256 해시값 계산 및 문서화 필요'
      });
    }

    // 체인 오브 커스터디 확인
    if (!collection.chainOfCustody || collection.chainOfCustody.length === 0) {
      issues.push({
        severity: 'critical',
        requirement: 'Chain of Custody',
        issue: '체인 오브 커스터디 미기록',
        recommendation: '증거 취급 전 과정 문서화 필요'
      });
    }

    // 데이터 보존 기간 확인
    if (collection.retentionPeriod > jurisdictionInfo.dataRetentionPeriod * 365) {
      recommendations.push(
        `데이터 보존 기간이 ${jurisdictionInfo.dataRetentionPeriod}년을 초과합니다. ` +
        `법적 요건 확인 필요.`
      );
    }

    // 국외 이전 확인
    if (collection.crossBorderTransfer) {
      const rule = jurisdictionInfo.crossBorderTransferRules.find(r =>
        r.destination === collection.transferDestination
      );

      if (rule) {
        recommendations.push(
          `국외 이전 시 ${rule.requirement} 준수 필요 (${rule.legalBasis})`
        );
      }
    }

    const complianceStatus = issues.length === 0 ? 'compliant' :
      issues.some(i => i.severity === 'critical') ? 'non_compliant' : 'review_required';

    return {
      jurisdiction,
      collectionMethod: collection.method,
      complianceStatus,
      issues,
      recommendations
    };
  }

  generateComplianceReport(checks: LegalComplianceCheck[]): ComplianceReport {
    const criticalIssues = checks.flatMap(c =>
      c.issues.filter(i => i.severity === 'critical')
    );

    const overallStatus = criticalIssues.length > 0 ? 'non_compliant' :
      checks.some(c => c.complianceStatus === 'review_required') ?
      'review_required' : 'compliant';

    return {
      reportDate: new Date(),
      overallStatus,
      jurisdictionsChecked: checks.map(c => c.jurisdiction),
      totalIssues: checks.reduce((sum, c) => sum + c.issues.length, 0),
      criticalIssues: criticalIssues.length,
      checks,
      summary: this.generateComplianceSummary(checks, overallStatus),
      recommendations: this.prioritizeRecommendations(checks)
    };
  }
}

증거 보전의 중요성

디지털 증거의 휘발성과 변조 가능성으로 인해 초기 대응 단계에서의 증거 보전이 매우 중요합니다. 시스템을 끄거나 재부팅하면 메모리의 휘발성 데이터가 손실됩니다. 네트워크 연결을 유지하면 원격 삭제나 데이터 변조의 위험이 있습니다. 압수수색 현장에서는 증거 보전을 위한 신속하고 체계적인 대응이 필요하며, 이를 위한 표준 운영 절차(SOP)를 사전에 수립해야 합니다.

Benefit All Humanity

디지털 포렌식과 법정 증거 절차는 디지털 시대의 정의 실현을 위한 기반입니다. 기술적 정확성과 법적 엄격성의 균형을 통해 공정한 사법 절차에 기여합니다.

장 요약

핵심 요점:

  1. 디지털 증거는 적법성, 신뢰성, 무결성, 관련성의 네 가지 법적 요건을 충족해야 법정에서 인정됩니다.
  2. 체인 오브 커스터디는 증거의 수집부터 법정 제출까지 모든 취급 과정을 문서화하여 무결성을 입증합니다.
  3. 포렌식 보고서는 기술적 분석 결과를 법률 전문가가 이해할 수 있도록 명확하게 작성해야 합니다.
  4. 전문가 증언은 분석 방법의 타당성과 결론의 신뢰성을 법정에서 설명하는 중요한 역할을 합니다.
  5. 국제 사건에서는 부다페스트 협약, MLAT, GDPR 등 다양한 법적 프레임워크를 이해하고 준수해야 합니다.
  6. 증거 수집 시 각 국가의 영장 요건, 개인정보보호법, 국외 이전 규정을 확인해야 합니다.
  7. 초기 대응 단계에서의 신속한 증거 보전이 이후 분석과 법정 활용의 성패를 좌우합니다.
