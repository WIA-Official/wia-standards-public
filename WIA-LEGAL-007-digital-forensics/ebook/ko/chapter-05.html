Chapter 5: 모바일 디바이스 포렌식

모바일 디바이스 포렌식은 스마트폰과 태블릿에서 디지털 증거를 수집, 분석, 보존하는 전문 분야입니다. 현대인의 일상이 모바일 기기에 기록되는 시대에서 통화 기록, 문자 메시지, 위치 정보, 앱 데이터는 수사에서 핵심 증거로 활용됩니다. 이 장에서는 iOS와 Android 플랫폼의 포렌식 기법, 물리적/논리적 추출 방법, 그리고 증거의 무결성을 보장하는 체계적인 프로세스를 학습합니다.

섹션 1: 모바일 포렌식 개요

모바일 포렌식은 전통적인 컴퓨터 포렌식과 구별되는 고유한 특성을 가집니다. 모바일 기기는 항상 연결된 상태를 유지하며, 원격 삭제 명령이나 자동 동기화로 인해 증거가 변조될 위험이 상존합니다. 또한 제조사별로 다른 운영체제와 보안 메커니즘은 표준화된 추출 방법의 적용을 어렵게 합니다. 그럼에도 모바일 기기는 사용자의 행동 패턴, 커뮤니케이션 이력, 위치 추적 데이터 등 풍부한 증거를 담고 있어 현대 수사에서 필수적인 증거원으로 자리잡았습니다.

모바일 증거의 유형

모바일 기기에서 추출할 수 있는 증거는 크게 세 가지 카테고리로 분류됩니다. 첫째, 통신 데이터로 통화 기록, SMS/MMS, 이메일, 인스턴트 메시지가 포함됩니다. 둘째, 미디어 데이터로 사진, 비디오, 음성 녹음과 함께 EXIF 메타데이터가 중요한 증거가 됩니다. 셋째, 앱 데이터로 소셜 미디어 활동, 브라우저 기록, 클라우드 동기화 정보, 금융 거래 내역 등이 있습니다. 특히 위치 정보(GPS, 셀 타워, Wi-Fi)는 피의자의 동선을 입증하는 결정적 증거로 활용됩니다.

표 5.1: 모바일 포렌식 추출 레벨

섹션 2: iOS 포렌식

Apple iOS는 강력한 보안 아키텍처로 포렌식 분석을 어렵게 만듭니다. Secure Enclave, Data Protection API, Hardware Security Module(HSM)이 통합되어 기기 암호 없이는 데이터 접근이 사실상 불가능합니다. iOS 8 이후 모든 사용자 데이터는 기본적으로 AES-256으로 암호화되며, 암호화 키는 사용자 패스코드와 하드웨어 고유 키(UID)의 조합으로 파생됩니다. 이러한 보안 체계는 법 집행 기관에게 큰 도전이지만, 적법한 절차에 따른 여러 우회 기법이 존재합니다.

iOS 백업 분석

iTunes 또는 Finder를 통한 로컬 백업은 iOS 포렌식의 핵심 데이터 소스입니다. 암호화되지 않은 백업에서는 연락처, 메시지, 통화 기록, 사진, 앱 데이터를 추출할 수 있습니다. 암호화된 백업은 키체인 데이터와 Health 앱 데이터까지 포함하여 더 풍부한 증거를 제공하지만, 백업 암호를 알아야 합니다. 백업 파일은 SQLite 데이터베이스와 plist 파일로 구성되며, 각 파일은 SHA-1 해시를 파일명으로 사용합니다.

// iOS 백업 분석 시스템
interface iOSBackupManifest {
  version: string;
  date: Date;
  deviceName: string;
  productType: string;
  serialNumber: string;
  uniqueDeviceID: string;
  encrypted: boolean;
}

interface BackupFile {
  fileID: string;           // SHA-1 해시
  domain: string;           // 앱 번들 ID
  relativePath: string;     // 원본 경로
  flags: number;
  file: {
    lastModified: Date;
    lastStatusChange: Date;
    birth: Date;
    size: number;
    protection: number;     // Data Protection 클래스
  };
}

class iOSBackupAnalyzer {
  private backupPath: string;
  private manifest: Map<string, BackupFile>;
  private keychain: Map<string, Buffer>;

  constructor(backupPath: string) {
    this.backupPath = backupPath;
    this.manifest = new Map();
    this.keychain = new Map();
  }

  async parseManifestDB(): Promise<iOSBackupManifest> {
    const manifestPath = `${this.backupPath}/Manifest.db`;
    const db = await this.openSQLite(manifestPath);

    // Manifest.db의 Files 테이블 파싱
    const files = await db.query(`
      SELECT fileID, domain, relativePath, flags, file
      FROM Files
      WHERE flags != 2
    `);

    for (const row of files) {
      const fileData = this.parsePlist(row.file);
      this.manifest.set(row.fileID, {
        fileID: row.fileID,
        domain: row.domain,
        relativePath: row.relativePath,
        flags: row.flags,
        file: {
          lastModified: new Date(fileData.$objects[1].LastModified * 1000),
          lastStatusChange: new Date(fileData.$objects[1].LastStatusChange * 1000),
          birth: new Date(fileData.$objects[1].Birth * 1000),
          size: fileData.$objects[1].Size,
          protection: fileData.$objects[1].ProtectionClass
        }
      });
    }

    // Info.plist에서 기기 정보 추출
    const infoPlist = await this.readPlist(`${this.backupPath}/Info.plist`);

    return {
      version: infoPlist['Product Version'],
      date: new Date(infoPlist['Last Backup Date']),
      deviceName: infoPlist['Device Name'],
      productType: infoPlist['Product Type'],
      serialNumber: infoPlist['Serial Number'],
      uniqueDeviceID: infoPlist['Unique Identifier'],
      encrypted: infoPlist['Is Encrypted'] ?? false
    };
  }

  async extractMessages(): Promise<iMessage[]> {
    // SMS/iMessage 데이터베이스 위치
    const smsDBHash = this.findFile(
      'HomeDomain',
      'Library/SMS/sms.db'
    );

    if (!smsDBHash) {
      throw new Error('SMS database not found in backup');
    }

    const dbPath = `${this.backupPath}/${smsDBHash.substring(0,2)}/${smsDBHash}`;
    const db = await this.openSQLite(dbPath);

    const messages = await db.query(`
      SELECT
        m.ROWID,
        m.guid,
        m.text,
        m.date / 1000000000 + 978307200 as timestamp,
        m.is_from_me,
        m.is_read,
        m.service,
        h.id as handle,
        c.chat_identifier
      FROM message m
      LEFT JOIN handle h ON m.handle_id = h.ROWID
      LEFT JOIN chat_message_join cmj ON m.ROWID = cmj.message_id
      LEFT JOIN chat c ON cmj.chat_id = c.ROWID
      ORDER BY m.date DESC
    `);

    return messages.map(msg => ({
      id: msg.ROWID,
      guid: msg.guid,
      text: msg.text,
      timestamp: new Date(msg.timestamp * 1000),
      isFromMe: msg.is_from_me === 1,
      isRead: msg.is_read === 1,
      service: msg.service, // 'iMessage' or 'SMS'
      handle: msg.handle,
      chatIdentifier: msg.chat_identifier
    }));
  }

  async extractLocationHistory(): Promise<LocationRecord[]> {
    // Significant Locations 데이터베이스
    const locDBHash = this.findFile(
      'RootDomain',
      'Library/Caches/locationd/consolidated.db'
    );

    if (!locDBHash) {
      console.warn('Location database not found - may require file system extraction');
      return [];
    }

    const dbPath = `${this.backupPath}/${locDBHash.substring(0,2)}/${locDBHash}`;
    const db = await this.openSQLite(dbPath);

    // Cell 타워 기반 위치
    const cellLocations = await db.query(`
      SELECT
        Timestamp + 978307200 as timestamp,
        Latitude,
        Longitude,
        HorizontalAccuracy,
        Altitude,
        MCC,
        MNC,
        LAC,
        CI as CellID
      FROM CellLocation
      WHERE Latitude != 0 AND Longitude != 0
      ORDER BY Timestamp DESC
    `);

    return cellLocations.map(loc => ({
      timestamp: new Date(loc.timestamp * 1000),
      latitude: loc.Latitude,
      longitude: loc.Longitude,
      accuracy: loc.HorizontalAccuracy,
      altitude: loc.Altitude,
      source: 'cell_tower',
      metadata: {
        mcc: loc.MCC,
        mnc: loc.MNC,
        lac: loc.LAC,
        cellId: loc.CellID
      }
    }));
  }

  async extractKeychain(password?: string): Promise<KeychainItem[]> {
    if (!password) {
      throw new Error('Keychain extraction requires backup password for encrypted backups');
    }

    // Keychain 백업 데이터베이스
    const keychainPath = `${this.backupPath}/Manifest.db`;

    // PBKDF2로 키 파생
    const key = await this.deriveKeychainKey(password);

    // keychain-backup.plist 복호화
    const keychainData = await this.decryptKeychain(key);

    return keychainData.map(item => ({
      class: item.class,          // genp, inet, cert, keys
      service: item.svce,
      account: item.acct,
      data: item.data,
      accessGroup: item.agrp,
      creationDate: item.cdat,
      modificationDate: item.mdat,
      protection: item.pdmn       // Data Protection 클래스
    }));
  }

  private findFile(domain: string, path: string): string | undefined {
    for (const [hash, file] of this.manifest) {
      if (file.domain === domain && file.relativePath === path) {
        return hash;
      }
    }
    return undefined;
  }

  private async deriveKeychainKey(password: string): Promise<Buffer> {
    const iterations = 10000;  // iOS 10.2 이상
    const salt = await this.getBackupSalt();

    return crypto.pbkdf2Sync(
      password,
      salt,
      iterations,
      32,  // 256 bits
      'sha256'
    );
  }
}

// 사용 예시
async function analyzeiOSBackup(backupPath: string) {
  const analyzer = new iOSBackupAnalyzer(backupPath);

  const manifest = await analyzer.parseManifestDB();
  console.log(`Device: ${manifest.deviceName} (${manifest.productType})`);
  console.log(`iOS Version: ${manifest.version}`);
  console.log(`Backup Date: ${manifest.date.toISOString()}`);
  console.log(`Encrypted: ${manifest.encrypted}`);

  const messages = await analyzer.extractMessages();
  console.log(`Found ${messages.length} messages`);

  const locations = await analyzer.extractLocationHistory();
  console.log(`Found ${locations.length} location records`);

  return {
    manifest,
    messages,
    locations
  };
}

섹션 3: Android 포렌식

Android는 오픈 소스 특성과 제조사별 커스터마이징으로 인해 iOS보다 다양한 포렌식 접근법이 가능합니다. 그러나 Android 7.0부터 도입된 파일 기반 암호화(FBE)와 하드웨어 백업 키 보호는 보안 수준을 크게 향상시켰습니다. Android 포렌식의 핵심은 ADB(Android Debug Bridge)를 통한 접근, 루팅을 통한 권한 상승, 그리고 칩 레벨 추출 기법입니다. 각 방법은 증거의 범위와 무결성 유지 측면에서 트레이드오프가 있습니다.

Android 데이터 추출

Android 기기에서 데이터를 추출하는 방법은 기기 상태와 보안 설정에 따라 달라집니다. USB 디버깅이 활성화된 기기에서는 ADB를 통한 논리적 추출이 가능합니다. 루팅된 기기에서는 /data 파티션 전체에 대한 파일 시스템 추출이 가능합니다. 잠긴 기기나 암호화된 기기에서는 JTAG, ISP(In-System Programming), 또는 칩오프(Chip-off) 기법이 필요할 수 있습니다.

// Android 포렌식 추출 시스템
interface AndroidDeviceInfo {
  manufacturer: string;
  model: string;
  androidVersion: string;
  buildNumber: string;
  serialNumber: string;
  securityPatchLevel: string;
  encryptionStatus: 'none' | 'fde' | 'fbe';
  rootStatus: 'not_rooted' | 'rooted' | 'unknown';
}

interface AndroidArtifact {
  path: string;
  type: 'database' | 'file' | 'directory';
  description: string;
  evidenceValue: 'high' | 'medium' | 'low';
}

class AndroidForensicExtractor {
  private adbPath: string;
  private deviceSerial: string;
  private outputDir: string;

  // 주요 포렌식 아티팩트 경로
  private readonly CRITICAL_ARTIFACTS: AndroidArtifact[] = [
    {
      path: '/data/data/com.android.providers.contacts/databases/contacts2.db',
      type: 'database',
      description: '연락처 데이터베이스',
      evidenceValue: 'high'
    },
    {
      path: '/data/data/com.android.providers.telephony/databases/mmssms.db',
      type: 'database',
      description: 'SMS/MMS 메시지',
      evidenceValue: 'high'
    },
    {
      path: '/data/data/com.android.providers.calendar/databases/calendar.db',
      type: 'database',
      description: '캘린더 이벤트',
      evidenceValue: 'medium'
    },
    {
      path: '/data/data/com.google.android.gms/databases/',
      type: 'directory',
      description: 'Google 서비스 데이터',
      evidenceValue: 'high'
    },
    {
      path: '/data/system/locksettings.db',
      type: 'database',
      description: '잠금 설정 정보',
      evidenceValue: 'medium'
    },
    {
      path: '/data/data/com.whatsapp/databases/',
      type: 'directory',
      description: 'WhatsApp 메시지',
      evidenceValue: 'high'
    }
  ];

  constructor(adbPath: string, deviceSerial: string, outputDir: string) {
    this.adbPath = adbPath;
    this.deviceSerial = deviceSerial;
    this.outputDir = outputDir;
  }

  async getDeviceInfo(): Promise<AndroidDeviceInfo> {
    const props = await this.execAdb('shell getprop');
    const propMap = this.parseProperties(props);

    // 암호화 상태 확인
    const cryptoState = await this.execAdb('shell getprop ro.crypto.state');
    const cryptoType = await this.execAdb('shell getprop ro.crypto.type');

    let encryptionStatus: 'none' | 'fde' | 'fbe' = 'none';
    if (cryptoState.trim() === 'encrypted') {
      encryptionStatus = cryptoType.includes('file') ? 'fbe' : 'fde';
    }

    // 루트 상태 확인
    let rootStatus: 'not_rooted' | 'rooted' | 'unknown' = 'unknown';
    try {
      const whoami = await this.execAdb('shell su -c whoami');
      rootStatus = whoami.includes('root') ? 'rooted' : 'not_rooted';
    } catch {
      rootStatus = 'not_rooted';
    }

    return {
      manufacturer: propMap['ro.product.manufacturer'] || 'Unknown',
      model: propMap['ro.product.model'] || 'Unknown',
      androidVersion: propMap['ro.build.version.release'] || 'Unknown',
      buildNumber: propMap['ro.build.display.id'] || 'Unknown',
      serialNumber: propMap['ro.serialno'] || this.deviceSerial,
      securityPatchLevel: propMap['ro.build.version.security_patch'] || 'Unknown',
      encryptionStatus,
      rootStatus
    };
  }

  async extractCallLogs(): Promise<CallLogEntry[]> {
    // Content Provider를 통한 추출 (루트 불필요)
    const callLogUri = 'content://call_log/calls';
    const projection = '_id,number,date,duration,type,name,cached_name';

    const result = await this.execAdb(
      `shell content query --uri ${callLogUri} --projection ${projection}`
    );

    const calls: CallLogEntry[] = [];
    const rows = result.split('Row:').filter(r => r.trim());

    for (const row of rows) {
      const fields = this.parseContentRow(row);
      calls.push({
        id: parseInt(fields._id),
        number: fields.number,
        date: new Date(parseInt(fields.date)),
        duration: parseInt(fields.duration),
        type: this.parseCallType(parseInt(fields.type)),
        contactName: fields.cached_name || fields.name || null
      });
    }

    return calls;
  }

  async extractSMSMessages(): Promise<SMSMessage[]> {
    const smsUri = 'content://sms';
    const projection = '_id,address,date,body,type,read,seen';

    const result = await this.execAdb(
      `shell content query --uri ${smsUri} --projection ${projection}`
    );

    const messages: SMSMessage[] = [];
    const rows = result.split('Row:').filter(r => r.trim());

    for (const row of rows) {
      const fields = this.parseContentRow(row);
      messages.push({
        id: parseInt(fields._id),
        address: fields.address,
        date: new Date(parseInt(fields.date)),
        body: fields.body,
        type: fields.type === '1' ? 'received' : 'sent',
        read: fields.read === '1',
        seen: fields.seen === '1'
      });
    }

    return messages;
  }

  async extractWithRoot(): Promise<ForensicImage> {
    const deviceInfo = await this.getDeviceInfo();

    if (deviceInfo.rootStatus !== 'rooted') {
      throw new Error('Root access required for full extraction');
    }

    // /data 파티션의 dd 이미지 생성
    const timestamp = Date.now();
    const imagePath = `${this.outputDir}/data_${timestamp}.img`;

    console.log('Creating forensic image of /data partition...');

    // 블록 디바이스 경로 확인
    const fstab = await this.execAdb('shell su -c cat /etc/fstab');
    const dataPartition = this.findDataPartition(fstab);

    // dd로 이미지 생성
    await this.execAdb(
      `shell su -c "dd if=${dataPartition} bs=4096" > ${imagePath}`,
      { timeout: 3600000 } // 1시간 타임아웃
    );

    // 해시 계산
    const md5 = await this.calculateHash(imagePath, 'md5');
    const sha256 = await this.calculateHash(imagePath, 'sha256');

    // 아티팩트 추출
    const artifacts: ExtractedArtifact[] = [];
    for (const artifact of this.CRITICAL_ARTIFACTS) {
      try {
        const extracted = await this.pullArtifact(artifact);
        artifacts.push(extracted);
      } catch (err) {
        console.warn(`Failed to extract ${artifact.path}: ${err.message}`);
      }
    }

    return {
      deviceInfo,
      imagePath,
      imageSize: await this.getFileSize(imagePath),
      hashes: { md5, sha256 },
      artifacts,
      extractionTime: new Date(),
      chainOfCustody: this.generateChainOfCustody()
    };
  }

  async extractLocationHistory(): Promise<LocationRecord[]> {
    const locations: LocationRecord[] = [];

    // Google 위치 기록
    try {
      const googleLocPath = '/data/data/com.google.android.gms/databases/herrevad';
      await this.pullFile(googleLocPath, `${this.outputDir}/google_location.db`);

      const db = await this.openSQLite(`${this.outputDir}/google_location.db`);
      const records = await db.query(`
        SELECT
          _id,
          timestamp,
          latitude / 1e7 as lat,
          longitude / 1e7 as lng,
          accuracy,
          source
        FROM location_history
        ORDER BY timestamp DESC
      `);

      locations.push(...records.map(r => ({
        id: r._id,
        timestamp: new Date(r.timestamp),
        latitude: r.lat,
        longitude: r.lng,
        accuracy: r.accuracy,
        source: r.source
      })));
    } catch (err) {
      console.warn('Google location history not accessible');
    }

    // Wi-Fi 연결 기록에서 위치 추론
    try {
      const wifiPath = '/data/misc/wifi/WifiConfigStore.xml';
      const wifiConfig = await this.pullFile(wifiPath, `${this.outputDir}/wifi_config.xml`);
      // Wi-Fi SSID/BSSID에서 위치 추론 가능
    } catch (err) {
      console.warn('Wi-Fi config not accessible');
    }

    return locations;
  }

  private parseCallType(type: number): string {
    const types: Record<number, string> = {
      1: 'incoming',
      2: 'outgoing',
      3: 'missed',
      4: 'voicemail',
      5: 'rejected',
      6: 'blocked'
    };
    return types[type] || 'unknown';
  }

  private async execAdb(command: string, options?: { timeout?: number }): Promise<string> {
    return new Promise((resolve, reject) => {
      const fullCommand = `${this.adbPath} -s ${this.deviceSerial} ${command}`;
      exec(fullCommand, {
        timeout: options?.timeout || 30000,
        maxBuffer: 50 * 1024 * 1024 // 50MB
      }, (error, stdout, stderr) => {
        if (error) reject(error);
        else resolve(stdout);
      });
    });
  }

  private generateChainOfCustody(): ChainOfCustodyEntry[] {
    return [{
      timestamp: new Date(),
      action: 'forensic_extraction',
      actor: process.env.USER || 'unknown',
      description: 'Initial forensic extraction performed',
      hash: null // Will be filled after extraction
    }];
  }
}

섹션 4: 물리적 추출 기법

논리적 추출이 불가능한 경우, 물리적 추출 기법이 최후의 수단으로 사용됩니다. JTAG(Joint Test Action Group)은 프로세서의 테스트 포트를 통해 메모리에 직접 접근하는 방법으로, 기기가 부팅되지 않는 경우에도 데이터를 추출할 수 있습니다. Chip-off는 메모리 칩을 기판에서 물리적으로 분리하여 읽는 방법으로, 기기가 완전히 손상된 경우에도 적용 가능하지만 비파괴적이지 않으며 고도의 전문성이 필요합니다. ISP(In-System Programming)는 칩을 분리하지 않고 테스트 포인트를 통해 데이터를 읽는 중간 단계의 기법입니다.

표 5.3: 물리적 추출 기법 비교
기법 장점 단점 적용 상황
JTAG 비파괴적, 암호화된 기기에서도 가능 JTAG 포트 필요, 시간 소요 부팅 불가 기기
ISP 칩 분리 불필요, 빠른 추출 테스트 포인트 필요 손상 최소화 필요시
Chip-off 완전 손상 기기에서도 가능 파괴적, 고가 장비 필요 모든 방법 실패시
Micro Read 극한 상황에서도 가능 전자 현미경 필요, 매우 고가 국가 안보 사건

섹션 5: 앱 데이터 분석

현대 모바일 포렌식에서 앱 데이터 분석은 핵심 영역입니다. 메시징 앱(WhatsApp, Telegram, Signal), 소셜 미디어(Facebook, Instagram), 금융 앱의 데이터는 중요한 증거가 됩니다. 각 앱은 고유한 데이터 구조와 암호화 방식을 사용하므로, 앱별 분석 기법이 필요합니다. 특히 종단간 암호화를 사용하는 앱에서는 키 추출이 중요한 과제가 됩니다.

// 앱 데이터 분석기
interface AppArtifact {
  appId: string;
  appName: string;
  version: string;
  databases: DatabaseInfo[];
  sharedPrefs: Map<string, any>;
  files: FileInfo[];
  cache: CacheInfo[];
}

interface WhatsAppMessage {
  id: string;
  chatId: string;
  sender: string;
  content: string;
  timestamp: Date;
  type: 'text' | 'image' | 'video' | 'voice' | 'document' | 'location';
  status: 'sent' | 'delivered' | 'read';
  mediaPath?: string;
  location?: { lat: number; lng: number };
}

class AppForensicAnalyzer {
  private dataPath: string;
  private platform: 'ios' | 'android';

  constructor(dataPath: string, platform: 'ios' | 'android') {
    this.dataPath = dataPath;
    this.platform = platform;
  }

  async analyzeWhatsApp(): Promise<WhatsAppAnalysisResult> {
    const dbPath = this.platform === 'android'
      ? `${this.dataPath}/com.whatsapp/databases/msgstore.db`
      : `${this.dataPath}/ChatStorage.sqlite`;

    const db = await this.openSQLite(dbPath);

    // 메시지 추출
    const messages = await this.extractWhatsAppMessages(db);

    // 연락처 추출
    const contacts = await this.extractWhatsAppContacts(db);

    // 그룹 정보 추출
    const groups = await this.extractWhatsAppGroups(db);

    // 미디어 파일 매핑
    const media = await this.mapWhatsAppMedia();

    // 삭제된 메시지 복구 시도
    const deletedMessages = await this.recoverDeletedMessages(dbPath);

    return {
      messages,
      contacts,
      groups,
      media,
      deletedMessages,
      analysisTime: new Date()
    };
  }

  private async extractWhatsAppMessages(db: SQLiteDB): Promise<WhatsAppMessage[]> {
    const query = this.platform === 'android'
      ? `
        SELECT
          messages._id,
          messages.key_remote_jid,
          messages.key_from_me,
          messages.data,
          messages.timestamp,
          messages.media_wa_type,
          messages.media_name,
          messages.latitude,
          messages.longitude,
          messages.status,
          jid.user as sender_number
        FROM messages
        LEFT JOIN jid ON messages.key_remote_jid = jid._id
        ORDER BY messages.timestamp DESC
      `
      : `
        SELECT
          ZWAMESSAGE.Z_PK,
          ZWAMESSAGE.ZCHATSESSION,
          ZWAMESSAGE.ZFROMJID,
          ZWAMESSAGE.ZTEXT,
          ZWAMESSAGE.ZMESSAGEDATE + 978307200 as timestamp,
          ZWAMESSAGE.ZMEDIAITEM,
          ZWAMESSAGE.ZMESSAGESTATUS
        FROM ZWAMESSAGE
        ORDER BY ZWAMESSAGE.ZMESSAGEDATE DESC
      `;

    const rows = await db.query(query);

    return rows.map(row => this.parseWhatsAppRow(row));
  }

  async analyzeTelegram(): Promise<TelegramAnalysisResult> {
    // Telegram은 MTProto 프로토콜 사용
    // 로컬 데이터베이스는 부분적으로 암호화됨

    const dbPath = this.platform === 'android'
      ? `${this.dataPath}/org.telegram.messenger/files/cache4.db`
      : `${this.dataPath}/telegram-data/cache.db`;

    const db = await this.openSQLite(dbPath);

    // 메시지 테이블
    const messages = await db.query(`
      SELECT
        mid,
        uid,
        data,
        date
      FROM messages
      ORDER BY date DESC
    `);

    // data 컬럼은 TL 직렬화 형식
    const parsedMessages = messages.map(msg => {
      const tlData = this.parseTLObject(msg.data);
      return {
        id: msg.mid,
        peerId: msg.uid,
        content: tlData.message || '',
        timestamp: new Date(msg.date * 1000),
        media: tlData.media || null
      };
    });

    return {
      messages: parsedMessages,
      chats: await this.extractTelegramChats(db),
      contacts: await this.extractTelegramContacts(db)
    };
  }

  async analyzeSignal(): Promise<SignalAnalysisResult> {
    // Signal은 강력한 종단간 암호화 사용
    // 로컬 키가 있어야 복호화 가능

    const dbPath = this.platform === 'android'
      ? `${this.dataPath}/org.thoughtcrime.securesms/databases/signal.db`
      : `${this.dataPath}/signal.sqlite`;

    // 키 추출 시도
    const keyPath = this.platform === 'android'
      ? `${this.dataPath}/org.thoughtcrime.securesms/shared_prefs/org.thoughtcrime.securesms_preferences.xml`
      : null;

    let key: Buffer | null = null;
    if (keyPath) {
      key = await this.extractSignalKey(keyPath);
    }

    if (!key) {
      console.warn('Signal database key not found - limited analysis available');
      return {
        messages: [],
        note: 'Database encrypted, key extraction failed'
      };
    }

    // SQLCipher로 열기
    const db = await this.openSQLCipher(dbPath, key);

    const messages = await db.query(`
      SELECT
        _id,
        thread_id,
        body,
        date,
        type,
        address
      FROM sms
      UNION ALL
      SELECT
        _id,
        thread_id,
        body,
        date,
        msg_box as type,
        address
      FROM mms
      ORDER BY date DESC
    `);

    return {
      messages: messages.map(msg => ({
        id: msg._id,
        threadId: msg.thread_id,
        content: msg.body,
        timestamp: new Date(msg.date),
        type: msg.type,
        address: msg.address
      }))
    };
  }

  async recoverDeletedMessages(dbPath: string): Promise<RecoveredMessage[]> {
    // SQLite 프리리스트 및 WAL 파일 분석
    const recovered: RecoveredMessage[] = [];

    // WAL 파일 분석
    const walPath = `${dbPath}-wal`;
    if (await this.fileExists(walPath)) {
      const walRecords = await this.parseWALFile(walPath);
      recovered.push(...walRecords);
    }

    // 프리리스트 페이지 분석
    const freelistRecords = await this.analyzeFreelistPages(dbPath);
    recovered.push(...freelistRecords);

    // 슬랙 스페이스 분석
    const slackRecords = await this.analyzeSlackSpace(dbPath);
    recovered.push(...slackRecords);

    return recovered;
  }

  private async parseWALFile(walPath: string): Promise<RecoveredMessage[]> {
    const buffer = await fs.readFile(walPath);
    const records: RecoveredMessage[] = [];

    // WAL 헤더 파싱 (32 bytes)
    const magic = buffer.readUInt32BE(0);
    if (magic !== 0x377f0682 && magic !== 0x377f0683) {
      return records; // 유효하지 않은 WAL 파일
    }

    const pageSize = buffer.readUInt32BE(8);
    const checkpoint = buffer.readUInt32BE(12);

    // WAL 프레임 순회
    let offset = 32; // 헤더 이후
    while (offset < buffer.length) {
      const frameHeader = buffer.slice(offset, offset + 24);
      const pageNumber = frameHeader.readUInt32BE(0);
      const dbSize = frameHeader.readUInt32BE(4);

      if (pageNumber === 0) break;

      const pageData = buffer.slice(offset + 24, offset + 24 + pageSize);

      // B-tree 페이지에서 삭제된 레코드 찾기
      const deletedRecords = this.findDeletedRecords(pageData);
      records.push(...deletedRecords);

      offset += 24 + pageSize;
    }

    return records;
  }
}

실무 고려사항

모바일 포렌식에서 가장 중요한 것은 증거의 무결성 유지입니다. 기기 확보 즉시 비행기 모드로 전환하거나 패러데이 백에 보관하여 원격 삭제를 방지해야 합니다. 모든 추출 과정은 상세히 문서화하고, 해시값을 기록하여 법정에서 증거 능력을 입증할 수 있어야 합니다. 또한 개인정보보호법과 통신비밀보호법을 준수하여 적법한 영장 없이는 데이터를 열람하지 않아야 합니다.

Benefit All Humanity

모바일 포렌식 기술은 범죄 수사와 사법 정의 실현에 기여하되, 개인의 프라이버시와 인권을 존중하는 균형 잡힌 접근이 필요합니다.

장 요약

핵심 요점:

  1. 모바일 포렌식은 논리적 추출, 파일 시스템 추출, 물리적 추출의 단계로 구분되며, 각 단계별로 추출 가능한 데이터 범위와 필요한 권한이 다릅니다.
  2. iOS는 Secure Enclave와 Data Protection API로 강력한 보안을 제공하며, iTunes 백업 분석이 주요 포렌식 방법입니다.
  3. Android는 ADB를 통한 접근과 루팅을 통한 권한 상승이 가능하며, 제조사별 커스터마이징으로 인해 다양한 접근법이 필요합니다.
  4. JTAG, ISP, Chip-off 등 물리적 추출 기법은 논리적 추출이 불가능한 경우의 최후 수단으로, 고도의 전문성이 필요합니다.
  5. 메시징 앱 분석에서 종단간 암호화는 주요 과제이며, 앱별 데이터 구조와 암호화 방식을 이해해야 합니다.
  6. SQLite WAL 파일과 프리리스트 분석을 통해 삭제된 메시지 복구가 가능합니다.
  7. 모든 포렌식 과정은 증거의 무결성을 유지하고 법적 요건을 충족하도록 문서화되어야 합니다.

복습 질문

  2. iOS의 Data Protection 클래스 4가지를 설명하고, 포렌식 관점에서 각 클래스가 갖는 의미를 분석하세요.
  3. Android 기기에서 ADB를 통한 데이터 추출 방법과 Content Provider를 활용한 데이터 접근 방법을 비교하세요.
  4. JTAG와 Chip-off 기법의 장단점을 설명하고, 각 기법이 적합한 상황을 제시하세요.
  5. WhatsApp과 Signal의 암호화 방식 차이가 포렌식 분석에 미치는 영향을 설명하세요.
  6. SQLite WAL 파일에서 삭제된 데이터를 복구하는 원리를 설명하세요.
  7. 모바일 기기 확보 시 원격 삭제를 방지하기 위한 절차를 설명하세요.
  8. iOS 백업 분석에서 암호화된 백업이 비암호화 백업보다 더 많은 데이터를 제공하는 이유를 설명하세요.
