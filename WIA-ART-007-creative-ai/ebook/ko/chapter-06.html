제6장: Phase 3 실시간 프로토콜

弘益人間 · 실시간으로 함께 창작하라

개요: 실시간 협업의 시대

2024년 12월, 서울, 뉴욕, 런던의 크리에이티브 팀이 동일한 광고 캠페인을 실시간으로 함께 작업했습니다. 서울의 디자이너가 Figma에서 레이아웃을 수정하면, 그 변경사항이 즉시 뉴욕의 Premiere Pro와 런던의 After Effects에 반영되었습니다. 3개 대륙, 15명의 팀원, 5개의 서로 다른 도구가 마치 하나의 통합된 시스템처럼 완벽하게 동기화되었습니다.

이것이 바로 Phase 3: 실시간 프로토콜의 힘입니다. WebSocket 기반의 양방향 통신으로 모든 변경사항이 즉시 전파되고, 충돌이 자동으로 해결되며, 협업자의 활동이 실시간으로 공유됩니다.

WebSocket 기반 통신

1. WebSocket vs HTTP의 차이

Phase 2의 REST API는 요청-응답 모델을 사용하지만, 실시간 협업에는 한계가 있습니다. WebSocket은 클라이언트와 서버 간 지속적인 연결을 유지하여 실시간 양방향 통신을 가능하게 합니다.

특성 HTTP / REST API WebSocket
통신 방식 요청-응답 (Request-Response) 양방향 (Bidirectional)
연결 매 요청마다 새 연결 지속적인 연결 유지
서버 푸시 불가능 (클라이언트가 폴링 필요) 가능 (서버가 즉시 푸시)
지연 시간 높음 (매번 핸드셰이크) 낮음 (연결 유지)
용도 일반 API 호출, 데이터 조회 실시간 협업, 채팅, 라이브 업데이트

2. WebSocket 연결 수립

WIA-ART-007 WebSocket 엔드포인트에 연결하는 방법입니다. 

// JavaScript WebSocket 연결
const ws = new WebSocket('wss://realtime.wia.org/art-007/v1');

// 연결 성공
ws.onopen = (event) => {
  console.log('Connected to WIA-ART-007 realtime server');

  // 인증
  ws.send(JSON.stringify({
    type: 'auth',
    token: 'YOUR_ACCESS_TOKEN'
  }));
};

// 메시지 수신
ws.onmessage = (event) => {
  const message = JSON.parse(event.data);
  console.log('Received:', message);

  switch(message.type) {
    case 'asset.created':
      handleAssetCreated(message.data);
      break;
    case 'asset.modified':
      handleAssetModified(message.data);
      break;
    case 'collaboration.cursor':
      updateCursorPosition(message.data);
      break;
  }
};

// 연결 종료
ws.onclose = (event) => {
  console.log('Disconnected:', event.reason);
  // 재연결 로직
  setTimeout(() => reconnect(), 5000);
};

// 에러 처리
ws.onerror = (error) => {
  console.error('WebSocket error:', error);
};

충돌 해결 전략

1. Operational Transformation (OT)

여러 사용자가 동시에 같은 에셋을 수정할 때 충돌을 자동으로 해결합니다.

충돌 시나리오 해결 방법 예시
동일 속성 수정 Last-Write-Wins (마지막 변경 우선) A가 밝기 1.2로 변경, B가 1.3으로 변경 → 1.3 적용
다른 속성 수정 병합 (Merge) A가 밝기 변경, B가 대비 변경 → 둘 다 적용
에셋 삭제 vs 수정 삭제 우선 A가 삭제, B가 수정 → 삭제 적용하고 B에게 알림
동시 추가 둘 다 추가 A와 B가 각각 에셋 추가 → 둘 다 추가됨

2. 충돌 알림 및 해결

사용자에게 충돌을 알리고 수동 해결을 요청합니다. 

// collaboration.conflict - 충돌 발생 알림

실시간 SDK 사용

1. JavaScript 실시간 클라이언트

공식 SDK를 사용한 실시간 협업 구현 예제입니다. 

import { WIARealtimeClient } from '@wia/art-007-sdk/realtime';

const realtime = new WIARealtimeClient({
  token: 'YOUR_ACCESS_TOKEN',
  projectId: 'proj-abc123'
});

// 연결
await realtime.connect();

// 에셋 생성 이벤트 구독
realtime.on('asset.created', (data) => {
  console.log('New asset:', data.asset_id);
  updateUIWithNewAsset(data);
});

// 협업자 커서 추적
realtime.on('collaboration.cursor', (data) => {
  updateCursor(data.user_id, data.cursor_position, data.user_color);
});

// 내 커서 위치 전송
function handleMouseMove(event) {
  realtime.emit('cursor.update', {
    position: { x: event.clientX, y: event.clientY },
    active_asset: getCurrentAsset()
  });
}

// 충돌 처리
realtime.on('collaboration.conflict', (data) => {
  if (data.resolution_needed) {
    showConflictDialog(data);
  }
});

// 연결 해제
realtime.disconnect();

성능 최적화

1. 이벤트 쓰로틀링

커서 이동 같은 빈번한 이벤트는 쓰로틀링하여 네트워크 부하를 줄입니다. 

// 쓰로틀링: 100ms마다 최대 1회만 전송
let lastEmit = 0;
const THROTTLE_MS = 100;

function handleMouseMove(event) {
  const now = Date.now();
  if (now - lastEmit < THROTTLE_MS) return;

  lastEmit = now;
  realtime.emit('cursor.update', {
    position: { x: event.clientX, y: event.clientY }
  });
}

2. 지능형 재연결

네트워크 끊김 시 자동으로 재연결하고 누락된 이벤트를 복구합니다.

재연결 전략 설명 적용 시점
Exponential Backoff 재시도 간격을 점진적으로 증가 1초 → 2초 → 4초 → 8초...
이벤트 재생 마지막 수신 이벤트 이후 누락분 요청 재연결 즉시
상태 동기화 전체 프로젝트 상태를 다시 가져옴 10초 이상 끊겼을 때

이번 장에서는 Phase 3 실시간 프로토콜의 구조와 구현 방법을 살펴보았습니다. 제7장에서는 Phase 4인 플랫폼 통합을 다룹니다. 클라우드 스토리지, 블록체인, AI 마켓플레이스 등 외부 플랫폼과의 통합을 통해 창의 AI 생태계를 확장하는 방법을 배우게 됩니다.

핵심 요점

  • WebSocket 통신: HTTP와 달리 지속적인 양방향 연결로 실시간 데이터 교환이 가능합니다.
  • 실시간 이벤트: 에셋 변경, 워크플로우 진행, 협업자 활동 등 모든 이벤트가 즉시 전파됩니다.
  • 충돌 해결: Operational Transformation과 Last-Write-Wins 전략으로 동시 편집 충돌을 자동 해결합니다.
  • 협업 기능: Figma처럼 커서 위치, 선택 영역, 실시간 코멘트를 공유하여 팀워크를 강화합니다.
  • 성능 최적화: 이벤트 쓰로틀링과 지능형 재연결로 네트워크 효율성을 극대화합니다.
  • 공식 SDK: React Hook 등 프레임워크 통합으로 실시간 협업 구현이 간단합니다.

복습 질문

  1. WebSocket과 HTTP/REST API의 차이점을 5가지 측면(통신 방식, 연결, 서버 푸시, 지연 시간, 용도)에서 비교하고, 각각이 적합한 사용 사례를 논의하십시오.
  2. 실시간 이벤트 3가지 카테고리(에셋, 워크플로우, 협업)를 설명하고, 각 카테고리의 주요 이벤트 타입과 데이터 구조를 분석하십시오.
  3. 동시 편집 충돌 해결 전략 4가지를 설명하고, Operational Transformation이 Last-Write-Wins보다 나은 상황을 논의하십시오.
  4. 커서 위치 업데이트 같은 빈번한 이벤트에 쓰로틀링이 필요한 이유를 설명하고, 100ms 쓰로틀링의 장단점을 분석하십시오.
  5. 네트워크 끊김 시 지능형 재연결 전략 3가지(Exponential Backoff, 이벤트 재생, 상태 동기화)를 설명하고, 각 전략이 적용되는 시점을 논의하십시오.
  6. React Hook을 사용한 실시간 협업 구현 예제를 분석하고, useWIARealtime이 제공하는 기능과 이점을 설명하십시오.
