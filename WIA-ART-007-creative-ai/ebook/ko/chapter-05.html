제5장: Phase 2 API 인터페이스

弘益人間 · API로 세상을 연결하라

개요: 프로그래밍 가능한 창의성

2024년 가을, 한 글로벌 마케팅 에이전시가 30개 국가에서 동시에 진행되는 캠페인을 준비했습니다. 각 국가의 크리에이티브 팀은 서로 다른 AI 도구를 사용했고, 모든 에셋을 중앙에서 관리해야 했습니다. 기존 방식대로라면 각 팀이 파일을 수동으로 업로드하고, 메타데이터를 수작업으로 입력해야 했습니다.

하지만 이 에이전시는 WIA-ART-007 Phase 2 API를 활용했습니다. 모든 AI 도구가 동일한 API를 통해 자동으로 프로젝트를 생성하고, 에셋을 업로드하며, 메타데이터를 동기화했습니다. 30개 팀의 작업이 실시간으로 통합되었고, 캠페인 론칭 시간은 기존 6주에서 2주로 단축되었습니다.

이것이 바로 Phase 2: API 인터페이스의 힘입니다. Phase 2는 창의 AI 도구들이 프로그래밍 방식으로 통신할 수 있게 하여, 수작업을 자동화하고, 워크플로우를 간소화하며, 새로운 가능성을 열어줍니다.

REST API 설계 원칙

1. RESTful 아키텍처

WIA-ART-007 API는 REST(Representational State Transfer) 원칙을 따릅니다. REST는 웹의 기본 프로토콜인 HTTP를 사용하여 간단하고 확장 가능한 API를 설계하는 방법론입니다.

REST 원칙 설명 WIA-ART-007 구현
Stateless 서버는 클라이언트 상태를 저장하지 않음 모든 요청에 인증 토큰 포함
Resource-based 리소스를 URL로 표현 /projects/{id}, /assets/{id}
HTTP Methods GET, POST, PUT, DELETE 사용 표준 HTTP 메서드로 CRUD 구현
JSON Format 데이터 교환 형식 모든 요청/응답은 JSON
HATEOAS 링크로 탐색 가능 응답에 관련 리소스 링크 포함

2. API 엔드포인트 구조

WIA-ART-007은 다음의 표준 엔드포인트 구조를 정의합니다: 

Base URL: https://api.wia.org/art-007/v1

# 프로젝트 관리
GET    /projects                    # 프로젝트 목록 조회
POST   /projects                    # 새 프로젝트 생성
GET    /projects/{id}               # 프로젝트 상세 조회
PUT    /projects/{id}               # 프로젝트 수정
DELETE /projects/{id}               # 프로젝트 삭제

# 에셋 관리
GET    /projects/{id}/assets        # 프로젝트의 에셋 목록
POST   /projects/{id}/assets        # 에셋 추가
GET    /assets/{id}                 # 에셋 상세 조회
PUT    /assets/{id}                 # 에셋 수정
DELETE /assets/{id}                 # 에셋 삭제

# AI 생성
POST   /generate/image              # 이미지 생성 요청
POST   /generate/video              # 비디오 생성 요청
POST   /generate/audio              # 오디오 생성 요청
GET    /generate/{job_id}/status    # 생성 작업 상태 확인

# 워크플로우
GET    /projects/{id}/workflow      # 워크플로우 히스토리
POST   /projects/{id}/workflow      # 워크플로우 단계 추가

# 인증 및 검증
POST   /auth/token                  # 인증 토큰 발급
POST   /certify                     # WIA 인증 요청
GET    /verify/{id}                 # 출처 검증

인증 및 보안

1. OAuth 2.0 인증 플로우

WIA-ART-007은 OAuth 2.0을 사용한 안전한 인증을 지원합니다.

단계 설명 엔드포인트
1. 클라이언트 등록 WIA에 앱 등록 및 Client ID 획득 https://developer.wia.org/register
2. 인증 코드 요청 사용자 로그인 및 권한 승인 GET /oauth/authorize
3. 액세스 토큰 교환 인증 코드를 토큰으로 교환 POST /oauth/token
4. API 호출 토큰을 사용하여 API 요청 Authorization: Bearer {token}
5. 토큰 갱신 만료된 토큰 갱신 POST /oauth/refresh 
# 액세스 토큰 발급
POST /oauth/token
Content-Type: application/x-www-form-urlencoded

grant_type=authorization_code&
code=AUTH_CODE_HERE&
client_id=YOUR_CLIENT_ID&
client_secret=YOUR_CLIENT_SECRET&
redirect_uri=https://yourapp.com/callback

# 응답
{
  "access_token": "eyJhbGciOiJSUzI1NiIsInR5cCI6IkpXVCJ9...",
  "token_type": "Bearer",
  "expires_in": 3600,
  "refresh_token": "tGzv3JOkF0XG5Qx2TlKWIA...",
  "scope": "read write"
}

SDK 구현

1. JavaScript/TypeScript SDK

공식 JavaScript SDK를 사용한 간단한 통합 예제입니다. 

// NPM 설치
npm install @wia/art-007-sdk

// SDK 초기화
import { WIAArtClient } from '@wia/art-007-sdk';

const client = new WIAArtClient({
  apiKey: 'YOUR_API_KEY',
  apiSecret: 'YOUR_API_SECRET'
});

// 프로젝트 생성
const project = await client.projects.create({
  title: 'My Creative Project',
  type: 'video',
  tags: ['demo', 'test']
});

console.log('Created project:', project.id);

// 에셋 업로드
const asset = await client.assets.upload(project.id, {
  file: './path/to/image.png',
  metadata: {
    type: 'background-image',
    ai_generated: true,
    source_tool: 'midjourney'
  }
});

console.log('Uploaded asset:', asset.cdn_url);

// AI 이미지 생성
const job = await client.generate.image({
  project_id: project.id,
  prompt: {
    positive: 'Beautiful landscape at sunset',
    quality: 'high'
  },
  num_outputs: 2
});

// 생성 완료 대기
const result = await client.generate.waitForCompletion(job.job_id);
console.log('Generated assets:', result.outputs);

2. Python SDK

Python 환경에서의 사용 예제입니다. 

# pip 설치
pip install wia-art-007

# SDK 사용
from wia_art_007 import WIAArtClient

client = WIAArtClient(
    api_key='YOUR_API_KEY',
    api_secret='YOUR_API_SECRET'
)

# 프로젝트 생성
project = client.projects.create(
    title='Python Creative Project',
    type='image',
    tags=['python', 'ai-art']
)

# 이미지 생성
job = client.generate.image(
    project_id=project.id,
    prompt={
        'positive': 'Abstract digital art, vibrant colors',
        'aspect_ratio': '1:1',
        'quality': 'ultra'
    },
    model='wia-image-gen-v2'
)

# 완료 대기 및 결과 조회
result = client.generate.poll_until_complete(job.job_id, timeout=300)

for output in result.outputs:
    print(f"Generated: {output.url}")
    print(f"Confidence: {output.metadata.confidence}")

다음 장 미리보기

이번 장에서는 Phase 2 API 인터페이스의 구조와 사용법을 상세히 살펴보았습니다. 제6장에서는 Phase 3인 실시간 프로토콜을 다룹니다. WebSocket 통신, 이벤트 스트리밍, 실시간 협업 기능을 통해 여러 창작자가 서로 다른 도구를 사용하면서도 실시간으로 협업하는 방법을 배우게 됩니다.

핵심 요점

  • RESTful API: HTTP 표준을 따르는 간단하고 확장 가능한 API 설계로 모든 프로그래밍 언어에서 사용 가능합니다.
  • 리소스 기반 구조: 프로젝트, 에셋, AI 생성, 워크플로우 등 명확한 리소스 구조로 직관적인 API 제공합니다.
  • OAuth 2.0 인증: 표준 OAuth 2.0 프로토콜을 사용하여 안전하고 확장 가능한 인증을 구현합니다.
  • 공식 SDK: JavaScript, Python 등 주요 언어용 공식 SDK로 개발 시간을 크게 단축합니다.
  • 비동기 작업: AI 생성 같은 시간이 오래 걸리는 작업은 비동기로 처리하고 상태를 폴링할 수 있습니다.
  • HATEOAS 링크: 응답에 관련 리소스 링크를 포함하여 API를 탐색 가능하게 만듭니다.
