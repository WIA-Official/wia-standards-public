제4장: Phase 1 데이터 형식 표준

弘益人間 · 데이터 구조로 지식을 보존하라

개요: 데이터가 모든 것의 기반이다

2024년 초, 한 유명 아트 갤러리가 10년 전에 제작된 디지털 아트 작품을 재전시하려 했습니다. 파일은 완벽하게 보존되어 있었지만, 문제가 발생했습니다. 어떤 소프트웨어로 만들어졌는지, 어떤 버전이었는지, 어떤 플러그인이 사용되었는지에 대한 정보가 전혀 없었습니다. 작품을 만든 아티스트는 연락이 닿지 않았고, 원래 사용된 소프트웨어는 이미 단종되었습니다.

결국 갤러리는 작품을 재현하는 데 3개월과 $50,000를 투입했습니다. 리버스 엔지니어링을 통해 파일 구조를 분석하고, 유사한 결과를 내는 현대적 도구를 찾아야 했습니다. 적절한 메타데이터가 있었다면 단 하루면 해결될 문제였습니다.

이것이 바로 Phase 1: 데이터 형식 표준이 중요한 이유입니다. Phase 1은 창의 AI 작업의 모든 정보를 표준화된 JSON 형식으로 저장하여, 10년, 50년, 100년 후에도 작품을 정확히 재현할 수 있게 보장합니다.

JSON 스키마 기본 구조

1. 왜 JSON인가?

WIA-ART-007은 메타데이터 형식으로 JSON(JavaScript Object Notation)을 채택했습니다. JSON이 선택된 이유는 다음과 같습니다:

장점 설명 예시
보편성 모든 프로그래밍 언어에서 지원 JavaScript, Python, Java, C++, Rust 등
가독성 사람이 읽고 이해하기 쉬움 XML보다 간결하고 명확
확장성 새로운 필드를 쉽게 추가 가능 하위 호환성 유지하면서 확장
검증 가능 JSON Schema로 유효성 검사 자동화된 검증 및 오류 감지
경량성 파일 크기가 작고 파싱 속도 빠름 네트워크 전송 및 저장 효율적

2. 루트 레벨 구조

모든 WIA-ART-007 메타데이터 파일은 다음의 루트 레벨 구조를 따릅니다:

  // 2. 프로젝트 정보 (필수)

  // 3. 창작자 정보 (필수)

  "assets": [ /* ... */ ],

  // 5. 워크플로우 히스토리 (선택)
  "workflow": [ /* ... */ ],

  // 6. AI 모델 정보 (AI 사용 시 필수)
  "ai_models": [ /* ... */ ],

  // 7. 라이선스 정보 (필수)
  "license": { /* ... */ },

  "extensions": { /* ... */ }

    // 프로젝트 제목 (필수)

2. 기여도 계산 방법

기여도는 다음 요소들을 종합적으로 고려하여 계산합니다:

  • 창의적 결정: 콘셉트, 스타일, 방향성 결정 (가중치 40%)
  • 작업 시간: 실제 소요된 시간 (가중치 30%)
  • 기술적 기여: 전문 기술 및 도구 활용 (가중치 20%)
  • 최종 결과물 영향: 창작물에 미친 영향도 (가중치 10%)

워크플로우 히스토리 (Workflow History)

1. 워크플로우 추적의 중요성

워크플로우 히스토리는 창작 과정의 모든 단계를 기록합니다. 이를 통해 작품을 재현하고, 문제를 디버깅하며, 창작 과정을 학습할 수 있습니다. 

{
  "workflow": [
    {
      // 단계 번호 (필수)
      "step": 1,

      // 단계 이름 (필수)
      "name": "background-generation",

      // 사용 도구 (필수)
      "tool": "adobe-firefly",
      "tool_version": "2.3.0",

      // 수행 작업 (필수)
      "action": "generate-image",

      // 입력 (선택)
      "input": {
        "prompt": "Modern office with natural lighting",
        "style": "photorealistic",
        "aspect_ratio": "16:9"
      },

      // 출력 (필수)
      "output": {
        "asset_id": "asset-bg-001",
        "file_path": "./assets/backgrounds/summer-office-01.png"
      },

      // AI 관련 정보 (AI 사용 시 필수)
      "ai_info": {
        "model": "Firefly Image 2",
        "confidence": 0.94,
        "alternatives_generated": 4,
        "selected_variant": 2
      },

      // 수행자 (필수)
      "performer": {
        "type": "human",
        "name": "김예술"
      },

      // 소요 시간 (초, 권장)
      "duration_seconds": 45,

      // 타임스탬프 (필수)
      "timestamp": "2024-06-16T09:15:00Z",

      // 비고 (선택)
      "notes": "첫 번째 시도에서 만족스러운 결과 얻음"
    }
  ]
}

다음 장 미리보기

이번 장에서는 Phase 1 데이터 형식 표준의 모든 구성 요소를 상세히 살펴보았습니다. 제5장에서는 Phase 2인 API 인터페이스를 다룹니다. REST API 설계, SDK 구현, 인증 및 보안, 그리고 실제 API 호출 예제를 통해 프로그래밍 방식으로 창의 AI 도구들을 통합하는 방법을 배우게 됩니다.

핵심 요점

  • JSON 스키마: 모든 메타데이터는 표준화된 JSON 형식으로 저장되어 상호운용성과 가독성을 보장합니다.
  • 10개 핵심 섹션: 표준 식별자, 프로젝트, 창작자, 에셋, 워크플로우, AI 모델, 라이선스, 출처, 품질, 확장으로 구성됩니다.
  • 투명한 기여도: 인간 창작자와 AI 도구의 기여를 명확히 구분하고 퍼센트로 기록합니다.
  • 워크플로우 추적: 창작 과정의 모든 단계를 시간순으로 기록하여 재현 가능성을 보장합니다.
  • AI 투명성: 사용된 AI 모델, 학습 데이터, 편향 테스트 결과를 모두 공개합니다.
  • 블록체인 검증: Ethereum과 IPFS를 활용하여 창작물의 출처와 소유권을 불변하게 기록합니다.

복습 질문

  1. WIA-ART-007이 메타데이터 형식으로 JSON을 선택한 5가지 이유를 설명하고, 각각이 창의 AI 워크플로우에서 어떤 이점을 제공하는지 논의하십시오.
  2. 프로젝트 메타데이터의 필수 필드와 선택 필드를 구분하고, 각 필드가 어떤 목적으로 사용되는지 설명하십시오. 프로젝트 타입에 따라 어떤 추가 필드가 필요한가요?
  3. 창작자 정보에서 인간 창작자와 AI 도구를 구분하는 방법을 설명하고, 기여도를 계산하는 4가지 요소(창의적 결정, 작업 시간, 기술적 기여, 최종 영향)의 가중치를 논의하십시오.
  4. 에셋 메타데이터 구조를 분석하고, AI 생성 에셋의 경우 반드시 포함되어야 하는 정보(프롬프트, 모델 정보, 라이선스 등)를 나열하십시오. 파일 무결성을 보장하기 위한 SHA-256 해시의 역할은 무엇입니까?
  5. 워크플로우 히스토리가 창작 과정의 재현성과 디버깅에 어떻게 기여하는지 설명하십시오. 제공된 워크플로우 예제를 분석하고 각 단계의 주요 정보를 논의하십시오.
  6. 블록체인 기반 출처 검증의 장점을 설명하고, Ethereum 트랜잭션 해시, IPFS 해시, NFT 토큰이 창작물의 소유권과 진위성을 어떻게 보장하는지 논의하십시오.
