8.3 Database Schema

CORE TABLES
===========

-- Pet emotion profiles
CREATE TABLE emotion_profiles (
    profile_id UUID PRIMARY KEY,
    pet_id UUID NOT NULL,
    owner_id UUID NOT NULL,
    species VARCHAR(50) NOT NULL,
    breed VARCHAR(100),
    name VARCHAR(100),
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- Emotion readings (time-series)
CREATE TABLE emotion_readings (
    reading_id UUID PRIMARY KEY,
    profile_id UUID REFERENCES emotion_profiles,
    timestamp TIMESTAMPTZ NOT NULL,
    primary_emotion VARCHAR(20) NOT NULL,
    intensity SMALLINT CHECK (intensity BETWEEN 1 AND 5),
    confidence DECIMAL(5,2),
    valence DECIMAL(3,2),
    arousal DECIMAL(3,2),
    data_sources JSONB,
    created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Create hypertable for time-series
SELECT create_hypertable('emotion_readings', 'timestamp');

-- Alerts
CREATE TABLE alerts (
    alert_id UUID PRIMARY KEY,
    profile_id UUID REFERENCES emotion_profiles,
    alert_type VARCHAR(50) NOT NULL,
    priority VARCHAR(20) NOT NULL,
    message TEXT,
    emotion_data JSONB,
    acknowledged BOOLEAN DEFAULT FALSE,
    resolved_at TIMESTAMPTZ,
    created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Baselines
CREATE TABLE emotion_baselines (
    baseline_id UUID PRIMARY KEY,
    profile_id UUID REFERENCES emotion_profiles,
    resting_heart_rate DECIMAL(5,2),
    resting_hrv DECIMAL(5,2),
    normal_activity DECIMAL(5,2),
    calibration_date TIMESTAMPTZ,
    calibration_quality DECIMAL(5,2)
);

8.4 API Implementation

Core Endpoints

API ROUTES
==========

Profiles:
  POST   /api/v1/profiles          Create profile
  GET    /api/v1/profiles          List profiles
  GET    /api/v1/profiles/{id}     Get profile
  PUT    /api/v1/profiles/{id}     Update profile
  POST   /api/v1/profiles/{id}/baseline  Calibrate

Readings:
  POST   /api/v1/readings          Submit reading
  GET    /api/v1/readings/{profileId}/current   Current state
  GET    /api/v1/readings/{profileId}/history   History
  WS     /ws/emotions/{profileId}  Real-time stream

Analysis:
  POST   /api/v1/analysis/detect   One-shot detection
  GET    /api/v1/analysis/{id}/summary    Summary
  GET    /api/v1/analysis/{id}/trends     Trends
  GET    /api/v1/analysis/{id}/wellbeing  Wellbeing

Alerts:
  GET    /api/v1/alerts/{profileId}       Get alerts
  POST   /api/v1/alerts/{id}/acknowledge  Acknowledge
  PUT    /api/v1/alerts/settings/{id}     Settings

8.6 Edge Device Integration

Device SDK

EDGE SDK INTERFACE
==================

class WiaEmotionEdgeSDK {
  // Initialize with credentials
  constructor(config: EdgeConfig)

  // Device registration
  async registerDevice(deviceInfo: DeviceInfo): Promise<DeviceId>

  // Start emotion processing
  async startProcessing(profileId: string): Promise<void>

  // Stop processing
  async stopProcessing(): Promise<void>

  // Event handlers
  onEmotionDetected(callback: (emotion: EmotionReading) => void)
  onAlert(callback: (alert: Alert) => void)
  onConnectionChange(callback: (status: ConnectionStatus) => void)

  // Data input
  submitVisualFrame(frame: ImageData): void
  submitAudioChunk(audio: AudioBuffer): void
  submitPhysiological(data: PhysioData): void

  // Local processing status
  getProcessingStatus(): ProcessingStatus
  getLocalBuffer(): BufferedReading[]
}

Validation Dataset Requirements

DATASET SPECIFICATIONS
======================

Minimum Size:
  - 10,000 labeled emotion samples
  - At least 500 per emotion category
  - Multiple species represented

Labeling Requirements:
  - Expert veterinary behaviorist validation
  - Multiple annotator agreement (>80%)
  - Clear context documentation

Distribution:
  - 70% training
  - 15% validation
  - 15% test (held out)

Quality Metrics:
  - Inter-annotator agreement: Kappa > 0.75
  - Clear emotion expression in samples
  - Balanced representation across intensities

8.9 Security Considerations

SECURITY REQUIREMENTS
=====================

1. Authentication
   - OAuth 2.0 / OpenID Connect
   - API key for device authentication
   - JWT tokens with short expiry

2. Authorization
   - Role-based access control (RBAC)
   - Owner can only access their pets
   - Explicit sharing for caregivers

3. Data Protection
   - TLS 1.3 for all communications
   - AES-256 encryption at rest
   - Edge processing for privacy

4. Privacy Compliance
   - GDPR compliance
   - Data minimization
   - Right to deletion
   - Consent management

5. Audit Logging
   - All data access logged
   - Tamper-proof audit trail
   - Retention policy enforcement
