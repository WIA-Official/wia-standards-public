5.2 WebSocket Protocol

Connection Establishment

WEBSOCKET HANDSHAKE
===================

1. Client connects
   ws://api.wia-emotion.org/ws/emotions/{profileId}

2. Authentication
   Client sends:
   {
     "action": "authenticate",
     "token": "Bearer xxx"
   }

3. Server confirms
   {
     "type": "authenticated",
     "profileId": "pet-123",
     "subscriptions": []
   }

4. Client subscribes
   {
     "action": "subscribe",
     "dataTypes": ["emotions", "alerts", "vitals"]
   }

5. Server confirms
   {
     "type": "subscribed",
     "dataTypes": ["emotions", "alerts", "vitals"],
     "updateInterval": 1000
   }

Message Types

Type Direction Description
emotion_update Server to Client New emotion reading
alert Server to Client Alert triggered
vitals_update Server to Client Physiological data
subscribe Client to Server Subscribe to data types
unsubscribe Client to Server Unsubscribe from data
ping/pong Bidirectional Connection keepalive

5.5 Data Buffering

BUFFERING STRATEGY
==================

Purpose: Handle network interruptions

1. Local Buffer
   - Capacity: 24 hours of readings
   - Storage: SQLite on edge device
   - Sync: Automatic when connected

2. Priority Queue
   Level 1 (Immediate): Alerts
   Level 2 (High): Emotion changes
   Level 3 (Normal): Regular updates
   Level 4 (Low): Historical sync

3. Offline Mode
   - Continue local processing
   - Store all readings locally
   - Queue alerts for transmission
   - Sync when connection restored

4. Bandwidth Management
   Normal mode: 1 update/second
   Low bandwidth: 1 update/5 seconds
   Offline: Local storage only

5.6 Update Frequency

Scenario Update Rate Rationale
Stable emotion 1 per 5 seconds Reduce bandwidth
Emotion transition 1 per second Capture change
Alert condition Immediate Urgent response
High arousal state 2 per second Close monitoring
Sleep/rest 1 per 30 seconds Minimal data needed

5.7 Connection Management

WEBSOCKET LIFECYCLE
===================

Connection States:
  CONNECTING --> AUTHENTICATED --> SUBSCRIBED
       |              |               |
       v              v               v
  DISCONNECTED   AUTH_FAILED    ACTIVE_STREAMING
       |
       v
  RECONNECTING (exponential backoff)

Keepalive Protocol:
  Client sends PING every 30 seconds
  Server responds with PONG
  No PONG in 60 seconds = reconnect

Reconnection Strategy:
  Attempt 1: Immediate
  Attempt 2: 1 second delay
  Attempt 3: 2 second delay
  Attempt 4: 4 second delay
  ...
  Max delay: 30 seconds
  Max attempts: Unlimited

5.8 Client Implementation

JavaScript Client Example

CLIENT IMPLEMENTATION
=====================

class EmotionStreamClient {
  constructor(profileId, token) {
    this.profileId = profileId;
    this.token = token;
    this.ws = null;
    this.handlers = {};
  }

  connect() {
    const url = `wss://api.wia-emotion.org/ws/emotions/${this.profileId}`;
    this.ws = new WebSocket(url);

    this.ws.onopen = () => {
      this.authenticate();
    };

    this.ws.onmessage = (event) => {
      const message = JSON.parse(event.data);
      this.handleMessage(message);
    };

    this.ws.onclose = () => {
      this.scheduleReconnect();
    };
  }

  authenticate() {
    this.send({
      action: "authenticate",
      token: `Bearer ${this.token}`
    });
  }

  subscribe(dataTypes) {
    this.send({
      action: "subscribe",
      dataTypes: dataTypes
    });
  }

  onEmotion(callback) {
    this.handlers["emotion_update"] = callback;
  }

  onAlert(callback) {
    this.handlers["alert"] = callback;
  }
}
