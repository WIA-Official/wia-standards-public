4.2 Visual Analysis Algorithm

Body Posture Classification

POSTURE ANALYSIS ALGORITHM
==========================

Input: Keypoint coordinates (17 points)

1. Calculate spine curvature
   curvature = angle(shoulder, mid_spine, hip)

2. Measure tension indicators
   head_height = normalize(head.y - shoulder.y)
   tail_position = normalize(tail_base.y - spine.y)
   leg_spread = distance(front_legs) + distance(back_legs)

3. Classify posture
   if tail_tucked AND head_low:
       posture = "fearful_crouch"
   elif spine_curved_up AND legs_spread:
       posture = "aggressive_stance"
   elif lying AND relaxed_muscles:
       posture = "resting_comfortable"
   elif standing AND tail_high:
       posture = "alert_confident"

4. Calculate tension score (0-1)
   tension = weighted_average(
       muscle_stiffness,
       movement_jerkiness,
       posture_rigidity
   )

Output: PostureResult(posture, tension, confidence)

4.3 Facial Expression Algorithm

Expression Feature Extraction

FACIAL ANALYSIS STEPS
=====================

1. Face Detection
   - Locate face bounding box
   - Extract face region

2. Landmark Detection
   - Eyes (4 points each)
   - Ears (3 points each)
   - Nose (1 point)
   - Mouth corners (2 points)

3. Feature Computation
   eye_openness = eye_height / eye_width
   ear_angle = angle(ear_tip, ear_base, head_center)
   brow_position = brow.y - eye_center.y

4. Action Unit Detection
   AU_scores = {}
   for each AU in [101, 102, 201, 202, 301, 401]:
       AU_scores[AU] = classify_AU(landmarks, AU)

5. Expression Classification
   expression = map_AU_to_expression(AU_scores)

4.4 Audio Analysis Algorithm

Vocalization Processing

AUDIO ANALYSIS PIPELINE
=======================

1. Audio Preprocessing
   - Sample rate: 44.1kHz
   - Noise gate: -40dB threshold
   - Band-pass filter: 50Hz - 8kHz

2. Vocalization Detection
   energy = compute_energy(frame)
   if energy > threshold:
       mark_vocalization_start()
   elif energy < release_threshold:
       mark_vocalization_end()

3. Feature Extraction
   For each vocalization segment:
   - Fundamental frequency (F0)
   - Formants (F1, F2, F3)
   - MFCC coefficients (13)
   - Duration and energy contour

4. Classification
   features = extract_features(segment)
   vocalization_type = classifier.predict(features)

   Types: bark, whine, growl, howl, purr, meow, hiss

5. Emotion Inference
   emotion_scores = vocalization_to_emotion(
       type, pitch, duration, energy
   )

Vocalization-Emotion Mapping

VOCALIZATION EMOTION RULES
==========================

BARK Analysis:
  High pitch + Rapid = Excited or Fearful
  Low pitch + Single = Alert
  Mid pitch + Rhythmic = Playful
  Harsh + Continuous = Aggressive

WHINE Analysis:
  Sustained + High = Anxious
  Intermittent + Rising = Attention-seeking
  Sharp onset = Pain indicator

GROWL Analysis:
  Sustained + Low = Warning (Aggressive)
  Varying pitch + Playful context = Play growl
  With body stiffness = Genuine threat

PURR Analysis (cats):
  Steady + Relaxed body = Content
  Louder + Seeking contact = Affectionate
  With illness signs = Self-soothing

4.5 Physiological Analysis Algorithm

Heart Rate Analysis

HEART RATE EMOTION ALGORITHM
============================

Input: HR time series, baseline values

1. Calculate current metrics
   current_hr = average(last_30_seconds)
   current_hrv = rmssd(rr_intervals)

2. Compare to baseline
   hr_deviation = (current_hr - baseline_hr) / baseline_hr
   hrv_ratio = current_hrv / baseline_hrv

3. Classify arousal state
   if hr_deviation > 0.3:
       arousal = HIGH
   elif hr_deviation > 0.1:
       arousal = MODERATE
   elif hr_deviation < -0.1:
       arousal = LOW
   else:
       arousal = NEUTRAL

4. Assess stress
   if hrv_ratio < 0.7 AND hr_deviation > 0.2:
       stress_level = HIGH
   elif hrv_ratio < 0.85:
       stress_level = MODERATE
   else:
       stress_level = LOW

Output: PhysioState(arousal, stress_level, confidence)

4.6 Multi-Modal Fusion Algorithm

Weighted Fusion

EMOTION FUSION ALGORITHM
========================

Input:
  visual_scores: Dict[EmotionType, float]
  audio_scores: Dict[EmotionType, float]
  physio_scores: Dict[EmotionType, float]
  confidences: Dict[source, float]

1. Normalize confidence weights
   total_conf = sum(confidences.values())
   weights = {src: conf/total_conf for src, conf in confidences}

2. Apply base modality weights
   modality_weights = {
       'visual': 0.35,
       'audio': 0.25,
       'physio': 0.30,
       'environment': 0.10
   }

3. Compute combined weights
   for source in sources:
       combined_weight[source] = (
           modality_weights[source] *
           weights[source]
       )
   # Renormalize
   combined_weight = normalize(combined_weight)

4. Fuse emotion scores
   fused_scores = {}
   for emotion in EmotionType:
       fused_scores[emotion] = (
           combined_weight['visual'] * visual_scores[emotion] +
           combined_weight['audio'] * audio_scores[emotion] +
           combined_weight['physio'] * physio_scores[emotion]
       )

5. Determine primary emotion
   primary = max(fused_scores, key=fused_scores.get)
   confidence = fused_scores[primary]

Output: EmotionResult(primary, intensity, confidence)

4.7 Temporal Integration

Emotion Smoothing

TEMPORAL SMOOTHING ALGORITHM
============================

Purpose: Reduce noise, ensure stability

Parameters:
  window_size = 5 seconds
  min_duration = 2 seconds
  change_threshold = 0.3

1. Maintain emotion history buffer
   history = RingBuffer(capacity=window_size * fps)

2. For each new reading
   history.append(current_emotion)

3. Calculate stability
   emotion_counts = Counter(history)
   dominant_emotion = emotion_counts.most_common(1)[0]
   stability = dominant_emotion[1] / len(history)

4. Apply smoothing
   if stability > 0.6:
       output = dominant_emotion[0]
   elif current != previous AND current.confidence > change_threshold:
       output = current  # Allow transition
   else:
       output = previous  # Maintain stability

5. Track state duration
   if output == previous_output:
       duration += delta_time
   else:
       duration = 0
       trigger_state_change_event()

Output: SmoothedEmotion(emotion, duration, stability)

4.8 Confidence Calculation

CONFIDENCE SCORING ALGORITHM
============================

Factors contributing to confidence:

1. Data Quality Score (0-25 points)
   - Sensor readings within normal range
   - No missing data
   - Signal-to-noise ratio acceptable

2. Source Agreement (0-25 points)
   - All sources suggest same emotion: +25
   - Majority agreement: +15
   - Split decision: +5
   - Contradiction: +0

3. Historical Consistency (0-25 points)
   - Matches recent pattern: +25
   - Reasonable transition: +15
   - Unexpected change: +5

4. Baseline Deviation (0-25 points)
   - Clear deviation from baseline: +25
   - Moderate deviation: +15
   - Within normal variation: +10

Final Confidence = sum(scores)

Confidence Level:
  85-100: HIGH
  65-84:  MODERATE
  50-64:  LOW
  0-49:   INSUFFICIENT
