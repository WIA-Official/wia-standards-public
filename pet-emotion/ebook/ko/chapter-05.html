5.2 WebSocket 프로토콜

연결 수립

WEBSOCKET 핸드셰이크
===================

1. 클라이언트 연결
   ws://api.wia-emotion.org/ws/emotions/{profileId}

2. 인증
   클라이언트 전송:
   {
     "action": "authenticate",
     "token": "Bearer xxx"
   }

3. 서버 확인
   {
     "type": "authenticated",
     "profileId": "pet-123",
     "subscriptions": []
   }

4. 클라이언트 구독
   {
     "action": "subscribe",
     "dataTypes": ["emotions", "alerts", "vitals"]
   }

5. 서버 확인
   {
     "type": "subscribed",
     "dataTypes": ["emotions", "alerts", "vitals"],
     "updateInterval": 1000
   }

메시지 유형

유형 방향 설명
emotion_update 서버 -> 클라이언트 새 감정 읽기
alert 서버 -> 클라이언트 경보 발생
vitals_update 서버 -> 클라이언트 생리 데이터
subscribe 클라이언트 -> 서버 데이터 유형 구독
unsubscribe 클라이언트 -> 서버 데이터 구독 해제
ping/pong 양방향 연결 유지

5.5 데이터 버퍼링

버퍼링 전략
==========

목적: 네트워크 중단 처리

1. 로컬 버퍼
   - 용량: 24시간 읽기
   - 저장소: 엣지 장치의 SQLite
   - 동기화: 연결 시 자동

2. 우선순위 큐
   레벨 1 (즉시): 경보
   레벨 2 (높음): 감정 변화
   레벨 3 (보통): 일반 업데이트
   레벨 4 (낮음): 과거 동기화

3. 오프라인 모드
   - 로컬 처리 계속
   - 모든 읽기 로컬 저장
   - 전송 대기 경보 큐잉
   - 연결 복원 시 동기화

4. 대역폭 관리
   일반 모드: 1 업데이트/초
   낮은 대역폭: 1 업데이트/5초
   오프라인: 로컬 저장만

5.6 업데이트 빈도

시나리오 업데이트 속도 근거
안정적 감정 5초당 1회 대역폭 감소
감정 전환 초당 1회 변화 캡처
경보 조건 즉시 긴급 대응
높은 각성 상태 초당 2회 면밀한 모니터링
수면/휴식 30초당 1회 최소 데이터 필요

5.7 연결 관리

WEBSOCKET 수명주기
=================

연결 상태:
  CONNECTING --> AUTHENTICATED --> SUBSCRIBED
       |              |               |
       v              v               v
  DISCONNECTED   AUTH_FAILED    ACTIVE_STREAMING
       |
       v
  RECONNECTING (지수적 백오프)

킵얼라이브 프로토콜:
  클라이언트 30초마다 PING 전송
  서버 PONG으로 응답
  60초 내 PONG 없음 = 재연결

재연결 전략:
  시도 1: 즉시
  시도 2: 1초 지연
  시도 3: 2초 지연
  시도 4: 4초 지연
  ...
  최대 지연: 30초
  최대 시도: 무제한

5.8 클라이언트 구현

JavaScript 클라이언트 예제

클라이언트 구현
==============

class EmotionStreamClient {
  constructor(profileId, token) {
    this.profileId = profileId;
    this.token = token;
    this.ws = null;
    this.handlers = {};
  }

  connect() {
    const url = `wss://api.wia-emotion.org/ws/emotions/${this.profileId}`;
    this.ws = new WebSocket(url);

    this.ws.onopen = () => {
      this.authenticate();
    };

    this.ws.onmessage = (event) => {
      const message = JSON.parse(event.data);
      this.handleMessage(message);
    };

    this.ws.onclose = () => {
      this.scheduleReconnect();
    };
  }

  authenticate() {
    this.send({
      action: "authenticate",
      token: `Bearer ${this.token}`
    });
  }

  subscribe(dataTypes) {
    this.send({
      action: "subscribe",
      dataTypes: dataTypes
    });
  }

  onEmotion(callback) {
    this.handlers["emotion_update"] = callback;
  }

  onAlert(callback) {
    this.handlers["alert"] = callback;
  }
}
