제7장: 경보 및 대응 시스템

"홍익인간: 고통에 대한 신속한 대응은 우리가 돌보는 이들에 대한 헌신의 깊이를 반영합니다."

7.1 경보 시스템 개요

경보 시스템은 감정 감지를 실행 가능한 알림으로 변환하여, 반려동물이 고통이나 우려되는 감정 상태를 경험할 때 시의적절한 개입을 가능하게 합니다.

경보 아키텍처

경보 파이프라인
==============

                            v
                     [경보 생성]
                            |

              |             |             |

7.2 경보 우선순위 수준

우선순위 응답 시간 트리거 조건 전달 방법
URGENT 즉시 극심한 두려움/통증, 응급 푸시 + SMS + 전화
HIGH 15분 이내 지속적 고통, 공격성 푸시 + SMS
MEDIUM 1시간 이내 장기 불안, 기분 하락 푸시 알림
LOW 하루 끝 사소한 우려, 패턴 앱 배지, 일일 요약

7.3 경보 트리거 규칙

경보 규칙 정의
=============

규칙: SUSTAINED_DISTRESS
  조건:
    (ANXIOUS OR FEARFUL OR SAD)
    AND 강도 >= 3
    AND 지속시간 > 30분
  우선순위: HIGH
  메시지: "{이름}이 30분 넘게 고통받고 있습니다"

규칙: ACUTE_FEAR
  조건:
    FEARFUL
    AND 강도 >= 4
    AND 급격한_시작 = true
  우선순위: URGENT
  메시지: "{이름}이 급성 두려움을 경험하고 있습니다"

규칙: AGGRESSIVE_BEHAVIOR
  조건:
    AGGRESSIVE
    AND 강도 >= 3
    AND 지속시간 > 5분
  우선순위: HIGH
  메시지: "{이름}이 공격적 행동을 보입니다"

규칙: SEPARATION_ANXIETY
  조건:
    ANXIOUS
    AND 보호자_부재 = true
    AND 지속시간 > 20분
  우선순위: MEDIUM
  메시지: "{이름}이 분리 불안을 경험할 수 있습니다"

규칙: WELLBEING_DECLINE
  조건:
    웰빙_추세 = DECLINING
    AND 하락률 > 5점/주
  우선순위: MEDIUM
  메시지: "{이름}의 전반적 웰빙이 하락하고 있습니다"

규칙: UNUSUAL_PATTERN
  조건:
    현재_감정 != 시간대_예상
    AND 편차_유의미 = true
  우선순위: LOW
  메시지: "{이름}의 감정 패턴이 평소와 다릅니다"

7.5 경보 설정

사용자 구성 가능 설정
===================

AlertSettings {
  profileId: string

  // 카테고리 활성화/비활성화
  enabledAlertTypes: [
    "distress",
    "anxiety",
    "aggression",
    "wellbeing",
    "patterns"
  ]

  // 민감도 조정
  sensitivityLevel: "low" | "medium" | "high"
  // low = 적은 경보, 주요 문제만
  // high = 많은 경보, 사소한 우려 포함

  // 시간 기반 설정
  quietHours: {
    enabled: true,
    start: "22:00",
    end: "07:00",
    allowUrgent: true
  }

  // 전달 선호도
  deliveryMethods: {
    push: true,
    sms: true,
    email: false,
    phone: false
  }

  // 긴급 연락망
  emergencyContacts: [
    {name: "보호자", phone: "+82...", priority: 1},
    {name: "펫시터", phone: "+82...", priority: 2}
  ]
}

7.6 경보 수명주기

경보 상태 머신
=============

     [TRIGGERED]
          |
          v
     [DELIVERED]
          |
     +----+----+
     |         |
     v         v
[VIEWED]   [EXPIRED]
     |
     v
[ACKNOWLEDGED]
     |
     +----+----+
     |         |
     v         v
[RESOLVED]  [ESCALATED]

상태 정의

상태 설명 다음 상태
TRIGGERED 경보 조건 충족 DELIVERED
DELIVERED 사용자에게 전송됨 VIEWED, EXPIRED
VIEWED 사용자가 경보 열람 ACKNOWLEDGED
ACKNOWLEDGED 사용자가 수신 확인 RESOLVED, ESCALATED
RESOLVED 문제 해결됨 (종료)
ESCALATED 다음 연락처로 상향 RESOLVED
EXPIRED 조건이 더 이상 활성 아님 (종료)

7.7 대응 조치

자동 대응

자동 대응 옵션
=============

1. 환경 조정
   - 진정 음악/소리 활성화
   - 조명 조정 (스마트 홈 경우)
   - 간식 디스펜서 활성화
   - 인터랙티브 장난감 시작

2. 커뮤니케이션
   - 보호자 음성 녹음 재생
   - 양방향 오디오 활성화
   - 보호자 영상 표시 (화면 있으면)

3. 에스컬레이션
   - 백업 케어기버 알림
   - 수의사 알림 (건강 문제 시)
   - 펫시터 서비스 연락

4. 문서화
   - 영상 클립 저장
   - 생리 데이터 기록
   - 수의사 상담용 로그

수동 대응 옵션

조치 설명
실시간 보기 카메라/센서 피드 열기
양방향 오디오 원격으로 반려동물에게 말하기
간식 주기 연결된 급식기 활성화
시터 연락 펫시터 빠른 전화
귀가하기 집으로 길 안내
수의사용 로그 상담용 사건 저장

7.8 경보 피로 방지

피로 방지 전략
=============

1. 지능형 그룹화
   - 관련 경보 결합
   - 진행 상황 요약
   - 지속 상태에 단일 경보

2. 적응형 임계값
   - 정상 패턴 학습
   - 개인에게 트리거 조정
   - 시간에 따라 오탐 감소

3. 쿨다운 기간
   - 유사 경보 간 최소 시간
   - 에스컬레이션 예외
   - 사용자 구성 가능

4. 스마트 타이밍
   - 회의 중 비긴급 지연
   - 방해 금지 시간 준수
   - 낮은 우선순위 요약으로 일괄

5. 관련성 점수
   - 진정 실행 가능한 경보 우선
   - 조치 불가 시 억제
   - 사용자 맥락 고려

7.9 경보 분석

지표 목적 목표
일일 경보 수 볼륨 모니터링 반려동물당 <5
응답 시간 사용자 참여 HIGH의 경우 <15분
오탐률 정확도 <10%
확인률 경보 관련성 >80%
해결률 실행 가능성 >70%
에스컬레이션률 심각도 정확성 <5%

7.10 케어 제공자 통합

수의사 통합
==========

1. 경보 공유
   보호자가 수의사와 경보 히스토리 공유 가능
   포함: 감정 데이터, 맥락, 빈도

2. 전문가 경보
   고심각도 경보가 수의사에게 직접 알림 가능
   명시적 보호자 동의 필요
   의료 응급상황에 사용

3. 행동 상담
   집계 패턴을 행동 전문가와 공유
   진단과 치료 계획 지원

4. 약물 모니터링
   치료에 대한 감정 반응 추적
   부작용 감지 시 경보

펫시터 통합
==========

1. 실시간 가시성
   시터가 현재 감정 상태 확인
   주의 우선순위 지정에 도움

2. 경보 전달
   보호자가 시터에게 경보 라우팅 가능
   시터가 확인/응답 가능

3. 교대 보고서
   자동 감정 요약
   케어 연속성 지원

장 요약

  • [OK] URGENT부터 LOW까지 4단계 우선순위
  • [OK] 유연한 조건의 구성 가능한 경보 규칙
  • [OK] 트리거에서 해결까지 전체 경보 수명주기
  • [OK] 자동 및 수동 대응 옵션 사용 가능
  • [OK] 스마트 그룹화와 타이밍으로 피로 방지
  • [OK] 수의사 및 펫케어 제공자와 통합
