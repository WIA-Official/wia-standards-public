8.1 구현 개요

구현 단계
=========

1단계: 기반 (1-4주)
  - 인프라 설정
  - 핵심 서비스 배포
  - 기본 감정 감지

2단계: 통합 (5-8주)
  - 장치 통합
  - 다중 모달 융합
  - 실시간 스트리밍

3단계: 인텔리전스 (9-12주)
  - 웰빙 분석
  - 경보 시스템
  - 인사이트 생성

4단계: 최적화 (지속적)
  - 모델 개선
  - 성능 튜닝
  - 기능 확장

8.3 데이터베이스 스키마

핵심 테이블
==========

-- 반려동물 감정 프로필
CREATE TABLE emotion_profiles (
    profile_id UUID PRIMARY KEY,
    pet_id UUID NOT NULL,
    owner_id UUID NOT NULL,
    species VARCHAR(50) NOT NULL,
    breed VARCHAR(100),
    name VARCHAR(100),
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- 감정 읽기 (시계열)
CREATE TABLE emotion_readings (
    reading_id UUID PRIMARY KEY,
    profile_id UUID REFERENCES emotion_profiles,
    timestamp TIMESTAMPTZ NOT NULL,
    primary_emotion VARCHAR(20) NOT NULL,
    intensity SMALLINT CHECK (intensity BETWEEN 1 AND 5),
    confidence DECIMAL(5,2),
    valence DECIMAL(3,2),
    arousal DECIMAL(3,2),
    data_sources JSONB,
    created_at TIMESTAMPTZ DEFAULT NOW()
);

-- 하이퍼테이블 생성 (시계열용)
SELECT create_hypertable('emotion_readings', 'timestamp');

-- 경보
CREATE TABLE alerts (
    alert_id UUID PRIMARY KEY,
    profile_id UUID REFERENCES emotion_profiles,
    alert_type VARCHAR(50) NOT NULL,
    priority VARCHAR(20) NOT NULL,
    message TEXT,
    emotion_data JSONB,
    acknowledged BOOLEAN DEFAULT FALSE,
    resolved_at TIMESTAMPTZ,
    created_at TIMESTAMPTZ DEFAULT NOW()
);

-- 기준선
CREATE TABLE emotion_baselines (
    baseline_id UUID PRIMARY KEY,
    profile_id UUID REFERENCES emotion_profiles,
    resting_heart_rate DECIMAL(5,2),
    resting_hrv DECIMAL(5,2),
    normal_activity DECIMAL(5,2),
    calibration_date TIMESTAMPTZ,
    calibration_quality DECIMAL(5,2)
);

8.4 API 구현

핵심 엔드포인트

API 라우트
==========

프로필:
  POST   /api/v1/profiles          프로필 생성
  GET    /api/v1/profiles          프로필 목록
  GET    /api/v1/profiles/{id}     프로필 조회
  PUT    /api/v1/profiles/{id}     프로필 업데이트
  POST   /api/v1/profiles/{id}/baseline  교정

읽기:
  POST   /api/v1/readings          읽기 제출
  GET    /api/v1/readings/{profileId}/current   현재 상태
  GET    /api/v1/readings/{profileId}/history   히스토리
  WS     /ws/emotions/{profileId}  실시간 스트림

분석:
  POST   /api/v1/analysis/detect   원샷 감지
  GET    /api/v1/analysis/{id}/summary    요약
  GET    /api/v1/analysis/{id}/trends     추세
  GET    /api/v1/analysis/{id}/wellbeing  웰빙

경보:
  GET    /api/v1/alerts/{profileId}       경보 조회
  POST   /api/v1/alerts/{id}/acknowledge  확인
  PUT    /api/v1/alerts/settings/{id}     설정

검증 데이터셋 요구사항

데이터셋 사양
============

최소 크기:
  - 10,000개 레이블된 감정 샘플
  - 감정 카테고리당 최소 500개
  - 여러 종 대표

레이블링 요구사항:
  - 전문 수의 행동 전문가 검증
  - 다중 주석자 동의 (>80%)
  - 명확한 맥락 문서화

분포:
  - 70% 훈련
  - 15% 검증
  - 15% 테스트 (홀드아웃)

품질 지표:
  - 주석자 간 동의: Kappa > 0.75
  - 샘플에서 명확한 감정 표현
  - 강도 전반에 균형 잡힌 대표

8.9 보안 고려사항

보안 요구사항
============

1. 인증
   - OAuth 2.0 / OpenID Connect
   - 장치 인증용 API 키
   - 짧은 만료의 JWT 토큰

2. 인가
   - 역할 기반 접근 제어 (RBAC)
   - 보호자는 자신의 반려동물만 접근
   - 케어기버를 위한 명시적 공유

3. 데이터 보호
   - 모든 통신에 TLS 1.3
   - 저장 시 AES-256 암호화
   - 개인정보용 엣지 처리

4. 개인정보 규정 준수
   - GDPR 준수
   - 데이터 최소화
   - 삭제권
   - 동의 관리

5. 감사 로깅
   - 모든 데이터 접근 로깅
   - 변조 방지 감사 추적
   - 보존 정책 시행
