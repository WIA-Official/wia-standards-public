CHAPTER 6

Phase 3 - Protocol

Phase 3 of WIA-BEMS elevates building energy management from simple data access and control to sophisticated, coordinated workflows. While Phases 1 and 2 established what data looks like and how to access it, Phase 3 defines how systems communicate and coordinate to achieve complex energy management objectives.

This chapter explores the protocol specifications that enable real-time monitoring, automated control sequences, predictive maintenance, fault detection and diagnostics (FDD), and energy optimization algorithms. We'll see how these standardized protocols transform buildings from passive structures into intelligent, responsive systems.

Understanding Protocol vs. API

Before diving into specific protocols, it's important to understand how Phase 3 builds upon Phase 2.

APIs: Individual Transactions

Phase 2 APIs enable individual transactions: "Get energy consumption for last hour," "Set temperature to 22°C," "Retrieve equipment status." Each API call is independent, stateless, and focused on a specific resource or action.

Protocols: Coordinated Workflows

Phase 3 protocols define sequences of interactions that accomplish complex objectives: "Monitor building conditions, predict occupancy, adjust HVAC 30 minutes before arrival, verify comfort levels, log performance." Protocols orchestrate multiple API calls, maintain state across interactions, and implement business logic that spans devices and systems.

Analogy: If Phase 2 APIs are individual words in a language, Phase 3 protocols are the grammar and syntax that combine those words into meaningful sentences and paragraphs. APIs provide capabilities; protocols coordinate those capabilities toward objectives.

Real-Time Monitoring Protocol

The real-time monitoring protocol enables continuous observation of building energy systems with automated alerting and response.

Stream Subscription Pattern

Instead of polling APIs repeatedly, monitoring systems subscribe to data streams and receive updates as they occur.

Automated Control Sequences

Control sequences coordinate multiple devices to achieve specific energy management objectives. Phase 3 defines standard patterns for common scenarios.

Occupancy-Based HVAC Control

Occupancy-Based Control Workflow

  1. Monitor Occupancy: Subscribe to occupancy sensors in target zones
  2. Detect State Change: Occupied → Vacant or Vacant → Occupied
  3. Implement Delay: Wait configured period to avoid false triggers
  4. Adjust Setpoints: Change temperature, airflow, and lighting based on state
  5. Verify Execution: Confirm equipment responded to commands
  6. Monitor Performance: Track energy savings and comfort impact
  7. Log Results: Record for analysis and optimization

Demand Response Control

When grid demand response events occur, buildings must quickly reduce load while maintaining acceptable comfort levels.

Demand Response Workflow

  1. Receive DR Event: OpenADR signal indicates event start and duration
  2. Calculate Reduction Target: Determine kW to shed based on enrollment
  3. Prioritize Actions: Order load reduction actions by impact/comfort cost
  4. Execute Sequence: Implement reductions progressively until target met
  5. Monitor Compliance: Track actual vs. target load reduction
  6. Adjust if Needed: Implement additional measures if target not met
  7. Restore After Event: Gradually restore normal operation
  8. Report Performance: Submit compliance data to utility

Predictive Maintenance Protocol

Predictive maintenance protocols monitor equipment performance, detect degradation, and schedule maintenance before failures occur.

Condition Monitoring

Maintenance Workflow

Predictive Maintenance Workflow

  1. Continuous Monitoring: Collect performance data from all equipment
  2. Baseline Comparison: Compare current vs. expected performance
  3. Anomaly Detection: Identify deviations from normal operation
  4. Trend Analysis: Project when maintenance will be needed
  5. Generate Work Order: Create maintenance task with details
  6. Schedule Execution: Coordinate with operations and procurement
  7. Track Completion: Verify maintenance performed correctly
  8. Validate Resolution: Confirm performance returns to normal
  9. Update Models: Refine prediction algorithms based on outcomes

Energy Optimization Protocols

Optimization protocols coordinate building systems to minimize energy consumption while maintaining comfort and operational requirements.

Model Predictive Control (MPC)

MPC uses building models and weather forecasts to proactively optimize energy use.

Model Predictive Control Workflow

  1. Gather Inputs: Weather forecast, occupancy schedule, energy prices
  2. Run Building Model: Simulate building behavior over planning horizon
  3. Optimize Control Strategy: Find minimum-cost control sequence
  4. Validate Constraints: Ensure comfort and operational limits met
  5. Execute First Step: Implement first interval of optimal sequence
  6. Monitor Performance: Compare actual vs. predicted performance
  7. Update Model: Refine predictions based on observations
  8. Repeat: Recalculate optimization every interval

Load Shifting Strategies

Load shifting moves energy consumption from high-cost to low-cost periods.

Strategy Description Typical Savings Requirements
Thermal Precooling Cool building before peak periods 15-30% peak reduction Good building thermal mass
Ice/Chilled Water Storage Make ice off-peak, use for cooling on-peak 40-60% peak reduction Thermal storage tanks
Battery Arbitrage Charge off-peak, discharge on-peak 30-50% peak reduction Battery storage system
Equipment Scheduling Run non-critical loads off-peak 10-20% cost reduction Flexible operations

Protocol Integration and Coordination

Multiple protocols often operate simultaneously. Phase 3 defines how they coordinate without conflicting.

Priority Hierarchy

  1. Life Safety: Fire, security, critical systems (highest priority)
  2. Equipment Protection: Prevent damage to expensive equipment
  3. Occupant Comfort: Maintain acceptable environmental conditions
  4. Demand Response: Grid events and emergency load reduction
  5. Energy Optimization: Cost and consumption minimization
  6. Routine Operations: Standard scheduling and control

Implementation Best Practices

Successfully implementing Phase 3 protocols requires careful design and testing.

Testing and Validation

  • Simulation first: Test protocols in digital twins before deployment
  • Progressive rollout: Start with non-critical systems and zones
  • Monitor closely: Track all protocol actions and impacts during initial deployment
  • Maintain overrides: Always provide manual override capability
  • Log comprehensively: Record all protocol decisions for analysis
  • Performance tracking: Measure actual vs. predicted savings

Reliability and Resilience

  • Graceful degradation: Protocols should fail safely if data unavailable
  • Timeout handling: Commands must timeout if not acknowledged
  • Retry logic: Implement exponential backoff for transient failures
  • Health monitoring: Protocols self-monitor and report status
  • Fallback modes: Revert to safe defaults if protocol fails

Chapter Summary

This chapter explored Phase 3 of WIA-BEMS, examining the protocols that enable sophisticated, coordinated building energy management. We've seen how standardized protocols orchestrate multiple systems and devices to achieve objectives that simple API calls cannot accomplish alone.

Key Takeaways

  1. Protocols enable coordination: While Phase 2 APIs handle individual transactions, Phase 3 protocols orchestrate sequences of actions across multiple systems to achieve complex energy management objectives.
  2. Real-time monitoring enables responsiveness: Stream-based monitoring protocols enable buildings to respond immediately to changing conditions rather than relying on periodic polling and delayed reactions.
  3. Automated sequences reduce manual work: Standardized control sequences for occupancy response, demand management, and optimization eliminate the need for manual intervention and ensure consistent, optimal performance.
  4. Predictive approaches prevent failures: Predictive maintenance and fault detection protocols identify problems before they cause failures, reducing downtime, repair costs, and energy waste.
  5. Optimization protocols maximize savings: Advanced techniques like model predictive control and load shifting deliver significantly greater savings than simple rule-based controls by considering forecasts and optimizing across time horizons.
