⠿ Chapter 5: Phase 2 - API Interface Standard

Part III: Technical Specification

Introduction

The API Interface Standard (Phase 2) defines how software applications interact with Braille displays at the programmatic level. This chapter covers the complete WIA Braille Display API, providing developers with everything needed to integrate Braille display support into applications, screen readers, and operating systems.

Building upon Phase 1's data format foundation, this API layer abstracts the complexity of device communication while maintaining fine-grained control when needed. The design philosophy emphasizes simplicity for common use cases while providing advanced features for specialized applications.

Design Principle: The API should be simple enough for a beginner to send "Hello World" to a Braille display in five lines of code, yet powerful enough to support professional screen reader implementations.

5.1 Core API Design Philosophy

5.1.1 Design Goals

The WIA Braille Display API is built around five core principles:

Principle Description Example
Simplicity First Common operations require minimal code 3 lines to display text
Type Safety Leverage TypeScript for compile-time checking No magic strings or numbers
Async by Default All I/O operations return Promises Modern async/await patterns
Event-Driven Subscribe to device events (keys, routing) Standard EventEmitter pattern
Graceful Degradation Works with partial device features Detect and adapt to capabilities

5.1.2 API Layers

The API is structured in three layers, each building upon the previous:

  1. High-Level API - Simple methods for common tasks (90% of use cases)
  2. Mid-Level API - Direct cell and cursor control
  3. Low-Level API - Raw protocol access for advanced implementations

Most developers will only need the high-level API. Screen reader developers may use mid-level. Hardware manufacturers may need low-level access.

5.1.3 Language Bindings

The reference implementation is TypeScript/JavaScript, but the API design supports bindings in:

  • TypeScript/JavaScript (primary)
  • Python (via PyPI)
  • Rust (via crates.io)
  • C/C++ (native library)
  • Java/Kotlin (Android)
  • Swift (iOS/macOS)

This chapter uses TypeScript for examples, but concepts translate directly to other languages.

5.2 WiaBrailleDisplay Class

5.2.1 Class Overview

The WiaBrailleDisplay class is the primary entry point for all API interactions. It represents a single connected Braille display device.

import { WiaBrailleDisplay } from '@wia/braille-display';

// Create instance (does not auto-connect)
const display = new WiaBrailleDisplay();

// Connect to first available device
await display.connect();

// Display text
await display.write('Hello World');

// Disconnect when done
await display.disconnect();

5.2.2 Constructor Options

The constructor accepts optional configuration:

interface WiaBrailleDisplayOptions {
    // Auto-connect to first available device
    autoConnect?: boolean;

    // Preferred connection type
    connectionType?: 'usb' | 'bluetooth' | 'serial' | 'any';

    // Specific device identifier
    deviceId?: string;

    // Enable debug logging
    debug?: boolean;

    // Custom event handlers
    onConnect?: () => void;
    onDisconnect?: () => void;
    onError?: (error: Error) => void;
}

// Example with options
const display = new WiaBrailleDisplay({
    autoConnect: true,
    connectionType: 'bluetooth',
    debug: true,
    onConnect: () => console.log('Display connected!'),
    onError: (err) => console.error('Display error:', err)
});

5.2.3 Device Information

Once connected, access device capabilities:

// Get device info
const info = display.getDeviceInfo();

console.log(info);

5.2.4 Capability Detection

Always check device capabilities before using features:

// Check for specific capability
if (display.hasCapability('routingKeys')) {
    // Enable cursor routing
    display.on('routing', (cellIndex) => {
        console.log(`Routing key pressed at cell ${cellIndex}`);
    });
}

// Check cell count
const cellCount = display.getCellCount();
if (cellCount >= 40) {
    // Display full line
    await display.write(text);
} else {
    // Display with scrolling
    await display.write(text, { scroll: true });
}

5.3 Device Discovery and Enumeration

5.3.1 Discovering Devices

The WiaBrailleDisplay.discover() static method finds all available Braille displays:

import { WiaBrailleDisplay } from '@wia/braille-display';

// Discover all available devices
const devices = await WiaBrailleDisplay.discover();

console.log(`Found ${devices.length} Braille display(s)`);

devices.forEach((device, index) => {
    console.log(`${index + 1}. ${device.manufacturer} ${device.model}`);
    console.log(`   Connection: ${device.connectionType}`);
    console.log(`   Cells: ${device.cellCount}`);
});

5.3.2 Filtering Devices

Filter discovery by connection type or manufacturer:

// Find only USB devices
const usbDevices = await WiaBrailleDisplay.discover({
    connectionType: 'usb'
});

// Find specific manufacturer
const hwDevices = await WiaBrailleDisplay.discover({
    manufacturer: 'HumanWare'
});

// Multiple filters
const bluetoothHumanware = await WiaBrailleDisplay.discover({
    connectionType: 'bluetooth',
    manufacturer: 'HumanWare',
    minCells: 40
});

5.3.3 Connecting to Specific Device

Connect to a specific device by ID or index:

// Discover devices
const devices = await WiaBrailleDisplay.discover();

// Connect to first device
const display = new WiaBrailleDisplay();
await display.connectToDevice(devices[0]);

// Or use device ID directly
await display.connect({ deviceId: 'hw-brailliant-bi40x-12345' });

5.3.4 Device Events

Listen for device connection/disconnection events:

// Monitor for new devices
WiaBrailleDisplay.on('deviceConnected', (device) => {
    console.log(`New device: ${device.model}`);
});

WiaBrailleDisplay.on('deviceDisconnected', (device) => {
    console.log(`Device removed: ${device.model}`);
});

// Start monitoring
WiaBrailleDisplay.startMonitoring();

// Stop when done
WiaBrailleDisplay.stopMonitoring();

5.4 Cell Management

5.4.1 Writing to Cells

The primary operation is writing content to Braille cells:

// Simple text write
await display.write('Hello World');

// Write at specific position
await display.write('World', { startCell: 6 });

// Write with options
await display.write(text, {
    startCell: 0,           // Start position
    clearBefore: true,      // Clear display first
    scroll: false,          // Enable auto-scroll
    scrollDelay: 500,       // MS between scrolls
    showCursor: true,       // Show cursor position
    cursorCell: 0          // Cursor cell index
});

5.4.2 Raw Cell Data

For advanced control, write raw cell patterns:

// Write raw dot patterns (8-bit values)
const patterns = [
    0b00100000,  // Dot 6 (capital indicator)
    0b00010000,  // Dot 5
    0b00001000,  // Dot 4
    0b00000100,  // Dot 3
    0b00000010,  // Dot 2
    0b00000001   // Dot 1
];

await display.writeCells(patterns);

// Or use cell objects
const cells = [
    { dots: [1, 2, 4], cursor: false },
    { dots: [1, 3, 4, 5], cursor: true },
    { dots: [1, 2, 3, 4, 5, 6], cursor: false }
];

await display.writeCellObjects(cells);

5.4.3 Clearing Display

Clear all or specific cells:

// Clear entire display
await display.clear();

// Clear specific range
await display.clearRange(10, 20);

// Clear specific cells
await display.clearCells([5, 10, 15, 20]);

5.4.4 Reading Current State

Read what's currently displayed:

// Get current cell patterns
const currentPatterns = display.getCurrentCells();

// Get as text
const currentText = display.getCurrentText();

// Get specific cell
const cellPattern = display.getCell(10);

console.log(`Cell 10: ${cellPattern.toString(2).padStart(8, '0')}`);

5.4.5 Scrolling

Implement panning for long content:

// Auto-scroll long text
await display.write(longText, {
    scroll: true,
    scrollDelay: 3000  // 3 seconds per page
});

// Manual scrolling
await display.scrollTo(40);  // Scroll to character 40

// Scroll by pages
await display.scrollLeft();   // Previous page
await display.scrollRight();  // Next page

// Get scroll state
const scrollState = display.getScrollState();
console.log(`Position: ${scrollState.position}/${scrollState.totalLength}`);
console.log(`Page: ${scrollState.currentPage}/${scrollState.totalPages}`);

5.5 Cursor Routing

5.5.1 Cursor Routing Overview

Cursor routing keys (also called routing buttons) are physical buttons above each Braille cell. Pressing a routing key typically moves the cursor to that position or activates an element.

5.5.2 Handling Routing Events

// Basic routing event
display.on('routing', (event) => {
    console.log(`Routing key pressed at cell ${event.cellIndex}`);
    console.log(`Physical cell: ${event.physicalCell}`);
    console.log(`Logical position: ${event.logicalPosition}`);
});

// With modifier keys
display.on('routing', (event) => {
    if (event.modifiers.includes('space')) {
        // Space + routing = select
        console.log(`Select at position ${event.logicalPosition}`);
    } else if (event.modifiers.includes('dot7')) {
        // Dot 7 + routing = context menu
        console.log(`Context menu at position ${event.logicalPosition}`);
    } else {
        // Normal routing = move cursor
        console.log(`Move cursor to ${event.logicalPosition}`);
    }
});

5.5.3 Routing Event Object

Property Type Description
cellIndex number Physical cell index (0-based)
logicalPosition number Position in scrolled content
modifiers string[] Pressed modifier keys
timestamp number Event timestamp (ms)
device object Device information

5.5.4 Cursor Display

Show cursor at specific position:

// Show cursor at cell 10
await display.setCursor(10, true);

// Hide cursor
await display.setCursor(10, false);

// Move cursor
await display.moveCursor(15);

// Get cursor position
const cursorPos = display.getCursorPosition();
console.log(`Cursor at cell ${cursorPos}`);

5.5.5 Advanced Routing Patterns

// Double-tap detection
let lastTap = { cell: -1, time: 0 };

display.on('routing', (event) => {
    const now = Date.now();
    const isDoubleTap =
        event.cellIndex === lastTap.cell &&
        now - lastTap.time < 500;  // 500ms threshold

    if (isDoubleTap) {
        console.log(`Double-tap at cell ${event.cellIndex}`);
        // Activate element
    }

    lastTap = { cell: event.cellIndex, time: now };
});

// Long press detection
display.on('routingPress', (event) => {
    console.log(`Long press at cell ${event.cellIndex}`);
    // Show context menu
});

// Swipe detection (sequential routing keys)
let swipeStart = null;

display.on('routing', (event) => {
    if (!swipeStart) {
        swipeStart = { cell: event.cellIndex, time: Date.now() };
        return;
    }

    const duration = Date.now() - swipeStart.time;
    const distance = event.cellIndex - swipeStart.cell;

    if (duration < 300 && Math.abs(distance) >= 3) {
        console.log(`Swipe ${distance > 0 ? 'right' : 'left'}`);
        swipeStart = null;
    }
});

5.6 Key Event Handling

5.6.1 Key Types

Braille displays have several types of keys:

Key Type Description Example Keys
Navigation Keys Arrow/panning buttons Left, Right, Up, Down
Routing Keys One per cell Routing 0-39
Perkin Keys Braille input dots Dot 1-8, Space, Enter
Function Keys Device-specific F1-F4, Escape, Tab
Modifier Keys Change other keys Shift, Control, Alt

5.6.2 Listening to Key Events

// All key events
display.on('key', (event) => {
    console.log(`Key: ${event.key}`);
    console.log(`Type: ${event.type}`);  // 'down' | 'up' | 'press'
    console.log(`Modifiers: ${event.modifiers.join(', ')}`);
});

// Specific key types
display.on('navigationKey', (event) => {
    if (event.key === 'left') {
        // Previous page
    } else if (event.key === 'right') {
        // Next page
    }
});

display.on('functionKey', (event) => {
    if (event.key === 'f1') {
        // Help menu
    }
});

5.6.3 Perkin Key Input

Perkin keys allow Braille input:

// Enable Braille input mode
display.enableBrailleInput();

// Listen for Braille character input
display.on('brailleInput', (event) => {
    console.log(`Dots: ${event.dots.join('')}`);
    console.log(`Pattern: ${event.pattern.toString(2)}`);
    console.log(`Character: ${event.character}`);

    // Add to input buffer
    inputBuffer += event.character;
});

// Chord detection (multiple dots pressed together)
display.on('chord', (event) => {
    if (event.dots.includes(1) && event.dots.includes(2) &&
        event.dots.includes(3)) {
        console.log('Command chord detected');
    }
});

// Disable Braille input
display.disableBrailleInput();

5.6.4 Key Combinations

// Key combination handler
display.on('key', (event) => {
    const combo = event.modifiers.concat([event.key]).join('+');

    switch (combo) {
        case 'space+dots1+2':
            console.log('Select all');
            break;
        case 'space+dots2+3':
            console.log('Copy');
            break;
        case 'space+dots1+3':
            console.log('Paste');
            break;
    }
});

5.6.5 Preventing Default Actions

// Prevent default behavior
display.on('key', (event) => {
    if (event.key === 'f1') {
        event.preventDefault();  // Don't trigger default help
        // Custom help implementation
        showCustomHelp();
    }
});

5.7 Error Handling

5.7.1 Error Types

The API defines specific error types:

enum WiaBrailleErrorCode {
    DEVICE_NOT_FOUND = 'DEVICE_NOT_FOUND',
    CONNECTION_FAILED = 'CONNECTION_FAILED',
    DEVICE_DISCONNECTED = 'DEVICE_DISCONNECTED',
    INVALID_CELL_INDEX = 'INVALID_CELL_INDEX',
    INVALID_DOT_PATTERN = 'INVALID_DOT_PATTERN',
    UNSUPPORTED_FEATURE = 'UNSUPPORTED_FEATURE',
    TIMEOUT = 'TIMEOUT',
    PERMISSION_DENIED = 'PERMISSION_DENIED',
    FIRMWARE_ERROR = 'FIRMWARE_ERROR'
}

class WiaBrailleError extends Error {
    constructor(
        public code: WiaBrailleErrorCode,
        message: string,
        public details?: any
    ) {
        super(message);
        this.name = 'WiaBrailleError';
    }
}

5.7.2 Try-Catch Pattern

try {
    await display.connect();
    await display.write('Hello World');
} catch (error) {
    if (error instanceof WiaBrailleError) {
        switch (error.code) {
            case WiaBrailleErrorCode.DEVICE_NOT_FOUND:
                console.error('No Braille display found');
                // Prompt user to connect device
                break;
            case WiaBrailleErrorCode.CONNECTION_FAILED:
                console.error('Failed to connect:', error.message);
                // Retry connection
                break;
            case WiaBrailleErrorCode.PERMISSION_DENIED:
                console.error('Permission denied. Check browser settings.');
                break;
            default:
                console.error('Braille error:', error.message);
        }
    } else {
        console.error('Unexpected error:', error);
    }
}

5.7.3 Error Events

// Listen for errors
display.on('error', (error) => {
    console.error('Display error:', error);

    if (error.code === WiaBrailleErrorCode.DEVICE_DISCONNECTED) {
        // Attempt reconnection
        setTimeout(() => {
            display.connect().catch(console.error);
        }, 1000);
    }
});

5.7.4 Validation

The API validates all inputs:

// Cell index validation
try {
    await display.write('Hello', { startCell: 50 });
} catch (error) {
    // WiaBrailleError: INVALID_CELL_INDEX
    // Cell index 50 exceeds device capacity (40 cells)
}

// Dot pattern validation
try {
    await display.writeCells([0b100000000]);  // 9 bits (invalid)
} catch (error) {
    // WiaBrailleError: INVALID_DOT_PATTERN
    // Pattern exceeds 8-dot format
}

// Feature support validation
try {
    await display.setVibration(true);
} catch (error) {
    // WiaBrailleError: UNSUPPORTED_FEATURE
    // Device does not support vibration
}

5.7.5 Timeout Handling

// Set operation timeout
display.setTimeout(5000);  // 5 seconds

try {
    await display.write(text);
} catch (error) {
    if (error.code === WiaBrailleErrorCode.TIMEOUT) {
        console.error('Operation timed out');
        // Check device connection
    }
}

// Or per-operation timeout
await display.write(text, { timeout: 3000 });

5.8 Code Examples

5.8.1 Simple Text Display

import { WiaBrailleDisplay } from '@wia/braille-display';

async function displayText(text: string) {
    const display = new WiaBrailleDisplay();

    try {
        await display.connect();
        await display.write(text);

        // Keep displayed for 5 seconds
        await new Promise(resolve => setTimeout(resolve, 5000));

    } finally {
        await display.disconnect();
    }
}

5.8.2 Interactive Reader

class BrailleReader {
    private display: WiaBrailleDisplay;
    private content: string;
    private position: number = 0;

    async init(content: string) {
        this.content = content;
        this.display = new WiaBrailleDisplay();

        await this.display.connect();

        // Navigation handlers
        this.display.on('navigationKey', (event) => {
            if (event.key === 'left') this.previousPage();
            if (event.key === 'right') this.nextPage();
        });

        // Routing for cursor positioning
        this.display.on('routing', (event) => {
            const globalPos = this.position + event.cellIndex;
            console.log(`Cursor at character ${globalPos}`);
        });

        await this.displayCurrentPage();
    }

    async displayCurrentPage() {
        const cellCount = this.display.getCellCount();
        const page = this.content.substr(this.position, cellCount);
        await this.display.write(page);
    }

    async nextPage() {
        const cellCount = this.display.getCellCount();
        if (this.position + cellCount < this.content.length) {
            this.position += cellCount;
            await this.displayCurrentPage();
        }
    }

    async previousPage() {
        const cellCount = this.display.getCellCount();
        if (this.position > 0) {
            this.position = Math.max(0, this.position - cellCount);
            await this.displayCurrentPage();
        }
    }

    async close() {
        await this.display.disconnect();
    }
}

// Usage
const reader = new BrailleReader();
await reader.init('Your long document content here...');

5.8.3 Form Input with Braille

class BrailleFormInput {
    private display: WiaBrailleDisplay;
    private inputBuffer: string = '';

    async init(prompt: string) {
        this.display = new WiaBrailleDisplay();
        await this.display.connect();

        // Show prompt
        await this.display.write(prompt);

        // Enable Braille input
        this.display.enableBrailleInput();

        // Handle character input
        this.display.on('brailleInput', async (event) => {
            this.inputBuffer += event.character;
            await this.updateDisplay();
        });

        // Handle backspace (chord: dots 7+8)
        this.display.on('chord', async (event) => {
            if (event.dots.includes(7) && event.dots.includes(8)) {
                this.inputBuffer = this.inputBuffer.slice(0, -1);
                await this.updateDisplay();
            }
        });

        // Handle submit (chord: dots 2+3+6)
        this.display.on('chord', (event) => {
            const dots = event.dots.sort().join('');
            if (dots === '236') {
                this.submit();
            }
        });
    }

    async updateDisplay() {
        const prompt = 'Input: ';
        await this.display.write(prompt + this.inputBuffer, {
            cursorCell: prompt.length + this.inputBuffer.length,
            showCursor: true
        });
    }

    async submit() {
        console.log('Submitted:', this.inputBuffer);
        await this.display.write('Submitted!');
        await new Promise(resolve => setTimeout(resolve, 2000));
        await this.display.disconnect();
    }
}

// Usage
const form = new BrailleFormInput();
await form.init('Enter name:');

5.8.4 Multi-Device Support

class BrailleDeviceManager {
    private devices: Map = new Map();

    async discoverAndConnect() {
        const found = await WiaBrailleDisplay.discover();

        for (const deviceInfo of found) {
            const display = new WiaBrailleDisplay();
            await display.connectToDevice(deviceInfo);

            this.devices.set(deviceInfo.serialNumber, display);
            console.log(`Connected: ${deviceInfo.model}`);
        }

        return this.devices.size;
    }

    async broadcastMessage(message: string) {
        const promises = Array.from(this.devices.values()).map(
            display => display.write(message)
        );
        await Promise.all(promises);
    }

    async disconnectAll() {
        const promises = Array.from(this.devices.values()).map(
            display => display.disconnect()
        );
        await Promise.all(promises);
        this.devices.clear();
    }
}

// Usage
const manager = new BrailleDeviceManager();
await manager.discoverAndConnect();
await manager.broadcastMessage('Hello all displays!');

5.8.5 Real-Time Terminal Display

class BrailleTerminal {
    private display: WiaBrailleDisplay;
    private lines: string[] = [];
    private topLine: number = 0;

    async init() {
        this.display = new WiaBrailleDisplay();
        await this.display.connect();

        // Check for multi-line support
        const info = this.display.getDeviceInfo();
        if (info.capabilities.multiLine) {
            console.log('Multi-line display detected');
        }

        // Navigation
        this.display.on('navigationKey', (event) => {
            if (event.key === 'up') this.scrollUp();
            if (event.key === 'down') this.scrollDown();
        });
    }

    async println(line: string) {
        this.lines.push(line);

        // Auto-scroll to bottom
        const maxVisible = this.getVisibleLineCount();
        if (this.lines.length > maxVisible) {
            this.topLine = this.lines.length - maxVisible;
        }

        await this.refresh();
    }

    getVisibleLineCount(): number {
        const info = this.display.getDeviceInfo();
        return info.capabilities.multiLine ? info.lineCount || 1 : 1;
    }

    async refresh() {
        const visibleLines = this.getVisibleLineCount();
        const cellCount = this.display.getCellCount();

        const displayLines = this.lines
            .slice(this.topLine, this.topLine + visibleLines)
            .map(line => line.padEnd(cellCount).substr(0, cellCount));

        await this.display.writeMultiLine(displayLines);
    }

    async scrollUp() {
        if (this.topLine > 0) {
            this.topLine--;
            await this.refresh();
        }
    }

    async scrollDown() {
        const maxTop = this.lines.length - this.getVisibleLineCount();
        if (this.topLine < maxTop) {
            this.topLine++;
            await this.refresh();
        }
    }
}

// Usage
const terminal = new BrailleTerminal();
await terminal.init();
await terminal.println('Welcome to Braille Terminal');
await terminal.println('Line 2');
await terminal.println('Line 3');

Chapter Summary

Phase 2 of the WIA Braille Display Standard defines a comprehensive API for interacting with Braille displays programmatically. The key takeaways from this chapter are:

  • Simple by Design - The API prioritizes ease of use while maintaining power for advanced scenarios
  • WiaBrailleDisplay Class - Central entry point for all operations with clear, intuitive methods
  • Device Discovery - Robust enumeration and filtering of available devices
  • Cell Management - Comprehensive control over Braille cell content and display state
  • Cursor Routing - Event-driven handling of routing keys with support for gestures and modifiers
  • Key Events - Unified handling of navigation, Perkin, function, and routing keys
  • Error Handling - Typed errors with clear codes and graceful degradation
  • Code Examples - Practical patterns for common use cases from simple text to complex applications

The API Layer provides the foundation for Phase 3 (Communication Protocol) and Phase 4 (Ecosystem Integration), enabling developers to build accessible applications without needing deep knowledge of hardware protocols.
