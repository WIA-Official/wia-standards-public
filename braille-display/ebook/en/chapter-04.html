Chapter 4: Phase 1 - Data Format

"Data is the foundation. Get the data format right, and everything else—protocols, translation, user experience—follows naturally. Get it wrong, and you build castles on sand."

4.1 Braille Cell Data Model

4.1.1 The Fundamental Unit: A Single Braille Cell

At the heart of the WIA standard is a simple, elegant data model for representing a single Braille cell. Every Braille display, regardless of manufacturer or configuration, presents information as a series of tactile cells. The WIA standard defines how to represent these cells in digital form.

A Braille cell consists of:

  • Position: The cell's location on the display (0-indexed from left to right)
  • Dots: Which dots are raised (numbered 1-8)
  • Attributes: Optional metadata (cursor position, emphasis, etc.)

4.1.2 Dot Numbering Convention

The WIA standard uses the international dot numbering convention:

6-dot cell:          8-dot cell:
    1 • • 4             1 • • 4
    2 • • 5             2 • • 5
    3 • • 6             3 • • 6
                        7 • • 8

Dots are numbered 1 through 6 for traditional Braille, with dots 7 and 8 added for computer Braille and attributes.

4.1.3 Cell Representation in JSON

A single cell is represented as a JSON object:

{
  "position": 0,
  "dots": [1, 2, 5],
  "cursor": false,
  "attributes": {
    "bold": false,
    "blink": false,
    "emphasis": 0
  }
}

Field Descriptions:

Field Type Required Description
position integer Yes 0-indexed position from left. Position 0 is leftmost cell.
dots array of integers Yes Which dots are raised. Array contains 1-8. Empty array = blank cell. Must be sorted ascending.
cursor boolean No (default: false) True if this cell should display the cursor indicator
attributes object No Optional display attributes for emphasis, formatting, etc.

4.1.4 Examples of Common Characters

Character Dots Raised JSON Representation Unicode Braille
a 1 {"position": 0, "dots": [1]} U+2801 (⠁)
b 1, 2 {"position": 0, "dots": [1, 2]} U+2803 (⠃)
c 1, 4 {"position": 0, "dots": [1, 4]} U+2809 (⠉)
1 (numeric) 2, 3, 4, 5, 6 then 1 Two cells needed ⠼⠁ (number sign + 'a')
A (capital) 6 then 1 Two cells or dot 7 ⠠⠁ or ⡁ (8-dot)
(space) (none) {"position": 0, "dots": []} U+2800 (⠀)
period 2, 5, 6 {"position": 0, "dots": [2, 5, 6]} U+2826 (⠦)

4.1.5 Cell Array Representation

An entire display line is represented as an array of cell objects:

{
  "command": "SET_BRAILLE",
  "cells": [
    {"position": 0, "dots": [1, 2, 3, 5]},      // 'h'
    {"position": 1, "dots": [1, 5]},            // 'e'
    {"position": 2, "dots": [1, 2, 4, 5]},      // 'l'
    {"position": 3, "dots": [1, 2, 4, 5]},      // 'l'
    {"position": 4, "dots": [1, 3, 5]},         // 'o'
    {"position": 5, "dots": []}                 // (space)
  ]
}

This represents the word "hello " displayed in Grade 1 English Braille.

4.3 Unicode Braille Mapping (U+2800 to U+28FF)

4.3.1 Unicode Braille Patterns Block

Unicode allocates 256 code points (U+2800 through U+28FF) for Braille patterns, providing a one-to-one mapping for every possible 8-dot Braille cell combination.

Encoding Logic:

The Unicode value is calculated as: U+2800 + bit_pattern

Where bit_pattern is an 8-bit value with bits representing dots:

Bit 0 (LSB) = Dot 1
Bit 1       = Dot 2
Bit 2       = Dot 3
Bit 3       = Dot 4
Bit 4       = Dot 5
Bit 5       = Dot 6
Bit 6       = Dot 7
Bit 7 (MSB) = Dot 8

4.3.2 Conversion Examples

Example 1: Letter 'a' (dot 1 only)

Dots: [1]
Bit pattern: 00000001 (binary) = 1 (decimal) = 0x01 (hex)
Unicode: U+2800 + 0x01 = U+2801
Character: ⠁

Example 2: Letter 'w' (dots 2, 4, 5, 6)

Dots: [2, 4, 5, 6]
Bits: 0 1 0 1 1 1 0 0 (reading right-to-left in dot order)
Binary: 00111010 = 58 (decimal) = 0x3A (hex)
Unicode: U+2800 + 0x3A = U+283A
Character: ⠺

Example 3: Full cell (all dots 1-8)

Dots: [1, 2, 3, 4, 5, 6, 7, 8]
Bit pattern: 11111111 (binary) = 255 (decimal) = 0xFF (hex)
Unicode: U+2800 + 0xFF = U+28FF
Character: ⣿

4.3.3 Conversion Algorithm (Pseudocode)

function dotsToUnicode(dots):
    bit_value = 0
    for dot in dots:
        bit_value = bit_value OR (1 << (dot - 1))
    return 0x2800 + bit_value

function unicodeToDots(unicode_char):
    bit_value = unicode_char - 0x2800
    dots = []
    for i in range(1, 9):
        if bit_value AND (1 << (i - 1)):
            dots.append(i)
    return dots

U+2803 2803 [1, 2] ⠃ b

4.3.5 Using Unicode Braille in WIA Protocol

The WIA standard allows three equivalent representations for efficiency:

Method 1: Dot Array (Most Explicit)

{"position": 0, "dots": [1, 2, 5]}

Method 2: Unicode Character (Most Compact)

{"position": 0, "unicode": "⠓"}  // U+2813

Method 3: Unicode Hex Code (Most Compatible)

{"position": 0, "unicode": "U+2813"}

All three are valid and semantically identical. Implementations must support all three for maximum compatibility.

4.4 6-Dot and 8-Dot Formats

4.4.1 6-Dot Cell Format

For traditional 6-dot displays, only dots 1-6 are used:

{
  "cell_type": "6-dot",
  "cell": {
    "position": 0,
    "dots": [1, 2, 5],  // Only values 1-6 allowed
    "unicode": "⠓"      // U+2813 (dots 1, 2, 5)
  }
}

Validation Rule: If cell_type is "6-dot", the dots array must not contain values 7 or 8. If present, the implementation should return an error.

4.4.2 8-Dot Cell Format

For computer Braille displays supporting 8 dots:

{
  "cell_type": "8-dot",
  "cell": {
    "position": 0,
    "dots": [1, 2, 5, 7],  // Values 1-8 allowed
    "unicode": "⡓"         // U+2853 (dots 1, 2, 5, 7)
  }
}

Dot 7 and 8 Usage:

  • Dot 7: Often indicates capitalization or bold in 8-dot Braille
  • Dot 8: Often indicates italics or alternative character set
  • Dots 7+8: Together can indicate special modes or underline

4.4.3 Mixed Cell Displays

Some displays have different cell types in different positions (e.g., 6-dot for main text, 8-dot for status):

{
  "device_config": {
    "cell_layout": [
      {"range": [0, 1], "type": "8-dot", "purpose": "status"},
      {"range": [2, 41], "type": "6-dot", "purpose": "text"}
    ]
  }
}

The screen reader can query this configuration and send appropriate data to each cell type.

4.4.4 Automatic Conversion

When an 8-dot pattern must be displayed on a 6-dot cell:

Input: {dots: [1, 2, 5, 7]}  // Capital 'h' in 8-dot
6-dot display conversion:
  Cell 1: {dots: [6]}        // Capital indicator
  Cell 2: {dots: [1, 2, 5]}  // 'h'

The WIA standard defines conversion rules for all common 8-dot to 6-dot mappings.

4.5.3 Multi-Language Text Handling

When text contains multiple languages, the WIA protocol supports language tagging:

{
  "command": "SET_BRAILLE",
  "text": "Hello mundo 你好",
  "language_segments": [
    {"start": 0, "end": 5, "language": "en", "table": "en-US-g2"},
    {"start": 6, "end": 11, "language": "es", "table": "es-g2"},
    {"start": 12, "end": 14, "language": "zh", "table": "zh-g1"}
  ]
}

The display or screen reader can then apply the correct Braille table for each segment.

4.5.4 Table Selection Priority

When multiple tables could apply, the WIA standard defines selection priority:

  1. Explicit user override: User has manually selected a table (highest priority)
  2. Document metadata: Document declares its language (e.g., HTML lang attribute)
  3. Language detection: Automatic detection based on content
  4. System locale: Operating system language setting
  5. Fallback default: en-US-g1 (lowest priority)

4.6 Validation Rules

4.6.1 Cell Data Validation

All cell data must pass these validation checks:

Rule Check Error if Violated
Position Range 0 ≤ position < cell_count INVALID_POSITION
Dots Range All values in dots array must be 1-6 (6-dot) or 1-8 (8-dot) INVALID_DOT_NUMBER
Dots Unique No duplicate values in dots array DUPLICATE_DOTS
Dots Sorted Dots array must be in ascending order UNSORTED_DOTS
Unicode Match If both dots and unicode provided, they must match UNICODE_MISMATCH
One Cursor Only one cell per display can have cursor=true MULTIPLE_CURSORS

4.6.2 Protocol Message Validation

All protocol messages must be valid JSON and conform to their schema:

// Invalid: Missing required field
{
  "command": "SET_BRAILLE"
  // ERROR: Missing "cells" field
}

// Invalid: Wrong type
{
  "command": "SET_BRAILLE",
  "cells": "hello"  // ERROR: "cells" must be array, not string
}

// Invalid: Unknown command
{
  "command": "FANCY_ANIMATION",
  "cells": []
  // ERROR: Unknown command
}

// Valid
{
  "command": "SET_BRAILLE",
  "cells": [
    {"position": 0, "dots": [1, 2]}
  ]
}

When validation fails, the device returns a structured error:

{

4.7 Code Examples

4.7.1 Python Example: Sending Braille to Display

import json
import serial

class WIABrailleDisplay:
    def __init__(self, port='/dev/ttyUSB0'):
        self.serial = serial.Serial(port, 9600)
        self.capabilities = self.handshake()

    def handshake(self):
        """Perform initial handshake and get capabilities"""
        # Display sends HANDSHAKE automatically on connect
        response = json.loads(self.serial.readline())
        if response['command'] == 'HANDSHAKE':
            # Send ACKNOWLEDGE
            ack = {
                "command": "ACKNOWLEDGE",
                "screen_reader": "ExampleReader",
                "version": "1.0.0"
            }
            self.serial.write(json.dumps(ack).encode() + b'\n')
            return response['capabilities']
        else:
            raise Exception("Handshake failed")

    def set_braille(self, text):
        """Convert text to Braille and send to display"""
        cells = []
        for i, char in enumerate(text):
            dots = self.char_to_dots(char)
            cells.append({
                "position": i,
                "dots": dots
            })

        command = {
            "command": "SET_BRAILLE",
            "cells": cells
        }

        self.serial.write(json.dumps(command).encode() + b'\n')
        response = json.loads(self.serial.readline())
        return response

    def char_to_dots(self, char):
        """Simple Grade 1 English Braille mapping"""
        braille_map = {
            'a': [1], 'b': [1,2], 'c': [1,4], 'd': [1,4,5],
            'e': [1,5], 'f': [1,2,4], 'g': [1,2,4,5],
            'h': [1,2,5], 'i': [2,4], 'j': [2,4,5],
            # ... (rest of alphabet)
            ' ': []
        }
        return braille_map.get(char.lower(), [])

# Usage
display = WIABrailleDisplay('/dev/ttyUSB0')
print(f"Connected to: {display.capabilities['manufacturer']} {display.capabilities['model']}")
print(f"Cell count: {display.capabilities['cell_count']}")

display.set_braille("hello")

4.7.2 JavaScript Example: Web-Based Braille Display Simulator

class BrailleCell {
    constructor(position) {
        this.position = position;
        this.dots = [];
    }

    setDots(dots) {
        // Validate
        if (!Array.isArray(dots)) {
            throw new Error('Dots must be array');
        }
        if (dots.some(d => d < 1 || d > 8)) {
            throw new Error('Dot values must be 1-8');
        }
        if (dots.length !== new Set(dots).size) {
            throw new Error('Duplicate dots not allowed');
        }

        this.dots = [...dots].sort((a, b) => a - b);
        this.render();
    }

    toUnicode() {
        let bitValue = 0;
        for (let dot of this.dots) {
            bitValue |= (1 << (dot - 1));
        }
        return String.fromCodePoint(0x2800 + bitValue);
    }

    render() {
        // Render to HTML canvas or DOM
        const unicode = this.toUnicode();
        document.getElementById(`cell-${this.position}`).textContent = unicode;
    }
}

class WIABrailleSimulator {
    constructor(cellCount) {
        this.cells = [];
        for (let i = 0; i < cellCount; i++) {
            this.cells.push(new BrailleCell(i));
        }
    }

    handleCommand(command) {
        switch(command.command) {
            case 'SET_BRAILLE':
                this.setBraille(command.cells);
                return {status: 'OK'};

            case 'CLEAR_BRAILLE':
                this.clear();
                return {status: 'OK'};

            default:
                return {
                    status: 'ERROR',
                    error_code: 'UNKNOWN_COMMAND',
                    message: `Command ${command.command} not recognized`
                };
        }
    }

    setBraille(cellData) {
        for (let data of cellData) {
            if (data.position >= this.cells.length) {
                throw new Error(`Position ${data.position} out of range`);
            }
            this.cells[data.position].setDots(data.dots);
        }
    }

    clear() {
        for (let cell of this.cells) {
            cell.setDots([]);
        }
    }
}

// Usage

        {position: 2, dots: [1, 2, 4, 5]},   // 'l'
        {position: 3, dots: [1, 2, 4, 5]},   // 'l'
        {position: 4, dots: [1, 3, 5]}       // 'o'

4.7.3 C Example: Embedded Device Implementation

#include <stdio.h>
#include <stdint.h>
#include <stdbool.h>

#define MAX_CELLS 80
#define MAX_DOTS 8

typedef struct {
    uint8_t position;
    uint8_t dots[MAX_DOTS];
    uint8_t dot_count;
    bool cursor;
} BrailleCell;

typedef struct {
    BrailleCell cells[MAX_CELLS];
    uint8_t cell_count;
    uint8_t cell_type; // 6 or 8
} BrailleDisplay;

// Convert dots array to Unicode Braille pattern
uint16_t dots_to_unicode(uint8_t *dots, uint8_t dot_count) {
    uint8_t bit_value = 0;
    for (int i = 0; i < dot_count; i++) {
        bit_value |= (1 << (dots[i] - 1));
    }
    return 0x2800 + bit_value;
}

// Set a cell's dots
int set_cell_dots(BrailleDisplay *display, uint8_t position,
                  uint8_t *dots, uint8_t dot_count) {
    if (position >= display->cell_count) {
        return -1; // Invalid position
    }

    // Validate dots
    for (int i = 0; i < dot_count; i++) {
        if (dots[i] < 1 || dots[i] > display->cell_type) {
            return -2; // Invalid dot number
        }
    }

    // Set the dots
    BrailleCell *cell = &display->cells[position];
    cell->position = position;
    cell->dot_count = dot_count;
    for (int i = 0; i < dot_count; i++) {
        cell->dots[i] = dots[i];
    }

    // Update hardware (piezoelectric actuators)
    update_hardware_cell(position, dots, dot_count);

    return 0; // Success
}

// Clear all cells
void clear_display(BrailleDisplay *display) {
    for (int i = 0; i < display->cell_count; i++) {
        display->cells[i].dot_count = 0;
        update_hardware_cell(i, NULL, 0);
    }
}

// Initialize display
void init_display(BrailleDisplay *display, uint8_t cell_count, uint8_t cell_type) {
    display->cell_count = cell_count;
    display->cell_type = cell_type;
    clear_display(display);
}

// Example usage
int main() {
    BrailleDisplay display;
    init_display(&display, 40, 8);

    // Display "hi"
    uint8_t h_dots[] = {1, 2, 5};
    uint8_t i_dots[] = {2, 4};

    set_cell_dots(&display, 0, h_dots, 3);
    set_cell_dots(&display, 1, i_dots, 2);

    printf("Unicode for 'h': U+%04X\n", dots_to_unicode(h_dots, 3));
    printf("Unicode for 'i': U+%04X\n", dots_to_unicode(i_dots, 2));

    return 0;
}

Chapter Summary

This chapter provided a comprehensive technical exploration of Phase 1 of the WIA Braille Display Standard—the Data Format and Communication layer. Key takeaways include:

  1. The Braille cell data model uses a simple, elegant JSON representation with position, dots array, and optional attributes. This human-readable format is self-documenting and extensible while remaining efficient for machine processing.
  2. JSON schemas provide formal definitions for all protocol messages, enabling automatic validation, code generation, and consistent implementation across platforms and programming languages. The schema-based approach ensures interoperability.
  3. Unicode Braille Patterns (U+2800 to U+28FF) provide a standardized character encoding for all 256 possible 8-dot Braille patterns. The WIA standard leverages Unicode as the canonical representation, ensuring compatibility with modern text processing systems.
  4. Support for both 6-dot and 8-dot formats ensures the standard works with traditional literary Braille displays and modern computer Braille displays. Automatic conversion rules enable seamless operation across device types.
  5. Multi-language Braille tables cover 100+ languages with Grade 1, Grade 2, and Computer Braille variants. The JSON-based table format is human-readable, version-controlled, and based on the proven liblouis library while remaining implementation-agnostic.
  6. Comprehensive validation rules catch errors at the data level before they reach hardware, preventing undefined behavior and providing clear, actionable error messages that accelerate debugging.
  7. Working code examples in Python, JavaScript, and C demonstrate that the standard is practical and implementable across all major platforms—from embedded devices to web applications.

Phase 1 establishes the foundational data layer that makes all higher-level features possible. By standardizing how Braille cells are represented, transmitted, and validated, the WIA standard creates a common language that manufacturers, screen reader developers, and operating system vendors can all speak. This foundation is essential for achieving the mission of universal Braille accessibility.
