⠿ Chapter 7: Phase 4 - Ecosystem Integration

Part III: Technical Specification

Introduction

Phase 4 represents the culmination of the WIA Braille Display Standard: seamless integration with the broader accessibility ecosystem. While Phases 1-3 defined data formats, APIs, and protocols, Phase 4 ensures that WIA-compliant displays work harmoniously with screen readers, operating systems, and web applications.

This chapter covers integration patterns for major platforms including Windows (NVDA, JAWS), macOS/iOS (VoiceOver), Android (TalkBack), and Linux (Orca). We'll explore driver development, API bridging, and best practices for providing consistent Braille output across the accessibility landscape.

Integration Goal: A WIA-compliant Braille display should provide a consistent, high-quality experience regardless of the screen reader or platform—write once, work everywhere.

7.1.3 Integration Patterns

Three primary integration patterns:

  1. Direct Integration - Screen reader uses WIA SDK directly
  2. Driver Wrapper - WIA SDK wrapped as native driver format
  3. Bridge Service - Standalone service exposes WIA displays to multiple clients
// Pattern 1: Direct Integration
import { WiaBrailleDisplay } from '@wia/braille-display';

class ScreenReader {
    async initBraille() {
        this.braille = new WiaBrailleDisplay();
        await this.braille.connect();
        this.braille.on('routing', this.handleRouting);
    }

    async updateBrailleDisplay(text) {
        await this.braille.write(text);
    }
}

// Pattern 2: Driver Wrapper (e.g., NVDA)
class WiaNvdaDriver extends nvdaDisplayDriver {
    constructor() {
        this.wiaDisplay = new WiaBrailleDisplay();
    }

    async initialize() {
        await this.wiaDisplay.connect();
        return {
            numCells: this.wiaDisplay.getCellCount(),
            hasRoutingKeys: this.wiaDisplay.hasCapability('routingKeys')
        };
    }

    display(cells) {
        return this.wiaDisplay.writeCells(cells);
    }
}

// Pattern 3: Bridge Service
class WiaBrailleService {
    constructor() {
        this.displays = new Map();
        this.clients = new Map();
    }

    async registerClient(clientId) {
        const display = new WiaBrailleDisplay();
        await display.connect();
        this.clients.set(clientId, display);
    }

    async writeForClient(clientId, text) {
        const display = this.clients.get(clientId);
        await display.write(text);
    }
}

7.2.2 NVDA Driver Structure

NVDA Braille drivers are Python modules implementing a specific interface:

# brailleDisplayDrivers/wia.py

import braille
import inputCore
from logHandler import log
from wia_braille import WiaBrailleDisplay, WiaBrailleError

class BrailleDisplayDriver(braille.BrailleDisplayDriver):
    """WIA Braille Display driver for NVDA"""

    name = "wia"
    description = "WIA Braille Display (Universal)"
    isThreadSafe = True

    @classmethod
    def check(cls):
        """Check if WIA displays are available"""
        try:
            devices = WiaBrailleDisplay.discover()
            return len(devices) > 0
        except Exception:
            return False

    def __init__(self):
        super().__init__()
        self._display = WiaBrailleDisplay()

    def _connect(self):
        """Establish connection to display"""
        try:
            self._display.connect()

            # Get device capabilities
            info = self._display.get_device_info()
            self.numCells = info['cellCount']

            # Set up event handlers
            self._display.on('routing', self._handleRouting)
            self._display.on('key', self._handleKey)
            self._display.on('error', self._handleError)

            log.info(f"Connected to {info['manufacturer']} {info['model']}")

        except WiaBrailleError as e:
            log.error(f"Failed to connect: {e}")
            raise

    def terminate(self):
        """Clean up on driver unload"""
        try:
            self._display.disconnect()
        except Exception:
            pass

    def display(self, cells):
        """Update Braille display with cell patterns"""
        try:
            self._display.write_cells(cells)
        except WiaBrailleError as e:
            log.error(f"Display update failed: {e}")

    def _handleRouting(self, event):
        """Handle routing key press"""
        try:
            # Map to NVDA gesture
            inputCore.manager.executeGesture(
                InputGesture(event['cellIndex'])
            )
        except Exception as e:
            log.error(f"Routing error: {e}")

    def _handleKey(self, event):
        """Handle navigation/function keys"""
        gesture = self._mapKeyToGesture(event)
        if gesture:
            inputCore.manager.executeGesture(gesture)

    def _mapKeyToGesture(self, event):
        """Map WIA key event to NVDA gesture"""
        keyMap = {
            'left': 'braille(wia):panLeft',
            'right': 'braille(wia):panRight',
            'up': 'braille(wia):previousLine',
            'down': 'braille(wia):nextLine',
        }
        return keyMap.get(event['key'])

class InputGesture(braille.BrailleDisplayGesture):
    """Represents a WIA Braille display gesture"""

    source = BrailleDisplayDriver.name

    def __init__(self, routingIndex=None, key=None):
        super().__init__()
        if routingIndex is not None:
            self.routingIndex = routingIndex
            self.id = f"routing:{routingIndex}"
        elif key:
            self.id = f"key:{key}"

7.3.2 JAWS Driver Development

JAWS drivers are native DLLs implementing the Braille Display Driver API:

// WiaBraille.dll - JAWS driver (C++)

#include "BrailleDriverAPI.h"
#include "wia_braille_sdk.h"

static WiaBrailleDisplay* g_display = nullptr;

// Driver initialization
BOOL APIENTRY DllMain(HMODULE hModule, DWORD reason, LPVOID reserved) {
    if (reason == DLL_PROCESS_ATTACH) {
        g_display = wia_braille_display_new();
    }
    else if (reason == DLL_PROCESS_DETACH) {
        if (g_display) {
            wia_braille_display_free(g_display);
        }
    }
    return TRUE;
}

// Get driver information
extern "C" __declspec(dllexport)
BOOL GetBrailleDisplayInfo(BrailleDisplayInfo* info) {
    if (!info) return FALSE;

    wcscpy_s(info->wszName, L"WIA Braille Display");
    wcscpy_s(info->wszManufacturer, L"WIA Standards");
    info->dwVersion = 0x00010000;  // Version 1.0
    info->dwCapabilities =
        BRAILLE_CAP_ROUTING |
        BRAILLE_CAP_CURSOR |
        BRAILLE_CAP_SCROLL;

    return TRUE;
}

// Open connection
extern "C" __declspec(dllexport)
BOOL OpenBrailleDisplay() {
    if (!g_display) return FALSE;

    try {
        wia_braille_display_connect(g_display);

        // Get device info
        WiaDeviceInfo info;
        wia_braille_display_get_info(g_display, &info);

        // Register callbacks
        wia_braille_display_on_routing(g_display, OnRoutingKey, nullptr);
        wia_braille_display_on_key(g_display, OnNavigationKey, nullptr);

        return TRUE;
    }
    catch (...) {
        return FALSE;
    }
}

// Close connection
extern "C" __declspec(dllexport)
BOOL CloseBrailleDisplay() {
    if (!g_display) return FALSE;

    wia_braille_display_disconnect(g_display);
    return TRUE;
}

// Write to display
extern "C" __declspec(dllexport)
BOOL WriteBrailleCells(BYTE* cells, DWORD cellCount, DWORD startPos) {
    if (!g_display || !cells) return FALSE;

    try {
        wia_braille_display_write_cells_partial(
            g_display,
            cells,
            cellCount,
            startPos
        );
        return TRUE;
    }
    catch (...) {
        return FALSE;
    }
}

// Get cell count
extern "C" __declspec(dllexport)
DWORD GetCellCount() {
    if (!g_display) return 0;

    WiaDeviceInfo info;
    wia_braille_display_get_info(g_display, &info);
    return info.cellCount;
}

// Routing key callback
void CALLBACK OnRoutingKey(int cellIndex, void* userData) {
    // Notify JAWS of routing key press
    SendJAWSBrailleKey(BRAILLE_ROUTING_KEY, cellIndex);
}

// Navigation key callback
void CALLBACK OnNavigationKey(const char* key, void* userData) {
    // Map WIA key to JAWS key code
    int jawsKey = MapWiaKeyToJaws(key);
    if (jawsKey != 0) {
        SendJAWSBrailleKey(jawsKey, 0);
    }
}

// Key mapping
int MapWiaKeyToJaws(const char* wiaKey) {
    static const struct {
        const char* wiaKey;
        int jawsKey;
    } keyMap[] = {
        { "left", BRAILLE_KEY_LEFT },
        { "right", BRAILLE_KEY_RIGHT },
        { "up", BRAILLE_KEY_UP },
        { "down", BRAILLE_KEY_DOWN },
        { "f1", BRAILLE_KEY_F1 },
        // ... more mappings
    };

    for (const auto& mapping : keyMap) {
        if (strcmp(wiaKey, mapping.wiaKey) == 0) {
            return mapping.jawsKey;
        }
    }

    return 0;
}

7.4.2 macOS Driver Development

// WiaBrailleDriver.swift
// macOS VoiceOver Braille driver

import Foundation
import Accessibility
import WiaBrailleSDK

@objc(WiaBrailleDriver)
class WiaBrailleDriver: BRLTDriver {

    private var display: WiaBrailleDisplay?
    private var modelName: String = "WIA Braille Display"
    private var cellCount: Int = 0

    // Initialize driver
    override func initialize() -> Bool {
        display = WiaBrailleDisplay()

        do {
            try display?.connect()

            // Get device info
            if let info = display?.getDeviceInfo() {
                modelName = "\(info.manufacturer) \(info.model)"
                cellCount = Int(info.cellCount)

                // Set up event handlers
                display?.onRouting { [weak self] event in
                    self?.handleRoutingEvent(event)
                }

                display?.onKey { [weak self] event in
                    self?.handleKeyEvent(event)
                }

                NSLog("VoiceOver: Connected to WIA display - \(modelName)")
                return true
            }
        }
        catch {
            NSLog("VoiceOver: WIA connection failed - \(error)")
            return false
        }

        return false
    }

    // Clean up
    override func terminate() {
        do {
            try display?.disconnect()
        }
        catch {
            NSLog("VoiceOver: Disconnect error - \(error)")
        }
        display = nil
    }

    // Get display size
    override func getDisplaySize() -> BRLTDisplaySize {
        return BRLTDisplaySize(
            columns: UInt32(cellCount),
            rows: 1
        )
    }

    // Write to display
    override func write(_ cells: UnsafePointer,
                       length: Int,
                       at position: Int) -> Bool {
        guard let display = display else { return false }

        do {
            let cellArray = Array(UnsafeBufferPointer(
                start: cells,
                count: length
            ))

            try display.writeCells(cellArray, startCell: position)
            return true
        }
        catch {
            NSLog("VoiceOver: Write error - \(error)")
            return false
        }
    }

    // Handle routing key
    private func handleRoutingEvent(_ event: WiaRoutingEvent) {
        // Create VoiceOver routing gesture
        let gesture = BRLTGesture(
            type: .routing,
            cellIndex: UInt32(event.cellIndex)
        )

        // Send to VoiceOver
        self.postGesture(gesture)
    }

    // Handle navigation keys
    private func handleKeyEvent(_ event: WiaKeyEvent) {
        var gestureType: BRLTGestureType

        switch event.key {
        case "left":
            gestureType = .panLeft
        case "right":
            gestureType = .panRight
        case "up":
            gestureType = .scrollUp
        case "down":
            gestureType = .scrollDown
        default:
            return  // Unsupported key
        }

        let gesture = BRLTGesture(type: gestureType)
        self.postGesture(gesture)
    }

    // Get driver name
    override func getName() -> String {
        return modelName
    }

    // Check for device availability
    override class func isAvailable() -> Bool {
        do {
            let devices = try WiaBrailleDisplay.discover()
            return !devices.isEmpty
        }
        catch {
            return false
        }
    }
}

// iOS VoiceOver integration via MFi (Made for iPhone) protocol

import ExternalAccessory
import WiaBrailleSDK

class iOSWiaBrailleDriver: NSObject, EAAccessoryDelegate {

    private var session: EASession?
    private var display: WiaBrailleDisplay?

    // Detect accessory connection
    func accessoryDidConnect(_ notification: Notification) {
        guard let accessory = notification.userInfo?[
            EAAccessoryKey
        ] as? EAAccessory else {
            return
        }

        // Check for WIA protocol
        if accessory.protocolStrings.contains("com.wia.braille") {
            connectToAccessory(accessory)
        }
    }

    // Establish connection
    private func connectToAccessory(_ accessory: EAAccessory) {
        session = EASession(
            accessory: accessory,
            forProtocol: "com.wia.braille"
        )

        guard let session = session else { return }

        // Initialize WIA display over MFi connection
        display = WiaBrailleDisplay(transport: .mfi(session))

        do {
            try display?.connect()
            notifyVoiceOverDisplayReady()
        }
        catch {
            print("iOS WIA connection failed: \(error)")
        }
    }

    // VoiceOver display protocol
    func updateDisplay(text: String) {
        do {
            try display?.write(text)
        }
        catch {
            print("Display update failed: \(error)")
        }
    }
}

// WiaBrailleService.kt
// Android TalkBack Braille service

package com.wia.braille.talkback

import android.accessibilityservice.AccessibilityService
import android.view.accessibility.AccessibilityEvent
import com.wia.braille.sdk.WiaBrailleDisplay
import com.wia.braille.sdk.WiaDeviceInfo

class WiaBrailleService : AccessibilityService() {

    private var display: WiaBrailleDisplay? = null
    private var deviceInfo: WiaDeviceInfo? = null

    override fun onServiceConnected() {
        super.onServiceConnected()

        // Initialize WIA Braille display
        initializeBrailleDisplay()
    }

    private fun initializeBrailleDisplay() {
        try {
            display = WiaBrailleDisplay()

            // Discover and connect
            display?.connect(
                connectionType = WiaConnectionType.BLUETOOTH,
                autoConnect = true
            )

            // Get device info
            deviceInfo = display?.getDeviceInfo()

            // Set up event handlers
            display?.setRoutingListener { event ->
                handleRoutingKey(event)
            }

            display?.setKeyListener { event ->
                handleNavigationKey(event)
            }

            Log.i(TAG, "TalkBack: WIA Braille connected - " +
                  "${deviceInfo?.manufacturer} ${deviceInfo?.model}")

        } catch (e: Exception) {
            Log.e(TAG, "Braille initialization failed", e)
        }
    }

    // Handle accessibility events from TalkBack
    override fun onAccessibilityEvent(event: AccessibilityEvent) {
        when (event.eventType) {
            AccessibilityEvent.TYPE_VIEW_FOCUSED -> {
                // Update Braille with focused element
                updateBrailleDisplay(event)
            }
            AccessibilityEvent.TYPE_VIEW_TEXT_CHANGED -> {
                // Update Braille with new text
                updateBrailleDisplay(event)
            }
        }
    }

    private fun updateBrailleDisplay(event: AccessibilityEvent) {
        try {
            // Get text from event
            val text = event.text.joinToString(" ")

            // Convert to Braille and display
            display?.write(text)

            // Show cursor if editable
            if (event.source?.isEditable == true) {
                val cursorPosition = event.source?.textSelectionStart ?: 0
                display?.setCursor(cursorPosition, visible = true)
            }

        } catch (e: Exception) {
            Log.e(TAG, "Display update failed", e)
        }
    }

    private fun handleRoutingKey(event: WiaRoutingEvent) {
        // Simulate touch at routing position
        val node = rootInActiveWindow ?: return

        // Find node at Braille position
        val textNode = findNodeAtBraillePosition(
            node,
            event.logicalPosition
        )

        // Perform click
        textNode?.performAction(AccessibilityNodeInfo.ACTION_CLICK)
    }

    private fun handleNavigationKey(event: WiaKeyEvent) {
        when (event.key) {
            "left" -> performGlobalAction(GLOBAL_ACTION_BACK)
            "right" -> {
                // Navigate to next element
                rootInActiveWindow?.performAction(
                    AccessibilityNodeInfo.ACTION_NEXT_AT_MOVEMENT_GRANULARITY
                )
            }
            "up" -> {
                // Scroll up
                performGlobalAction(GLOBAL_ACTION_SCROLL_UP)
            }
            "down" -> {
                // Scroll down
                performGlobalAction(GLOBAL_ACTION_SCROLL_DOWN)
            }
        }
    }

    override fun onInterrupt() {
        // TalkBack interrupted
    }

    override fun onDestroy() {
        super.onDestroy()

        // Clean up
        try {
            display?.disconnect()
        } catch (e: Exception) {
            Log.e(TAG, "Disconnect failed", e)
        }
    }

    companion object {
        private const val TAG = "WiaBrailleService"
    }
}

7.5.4 Braille Keyboard Input

// Braille keyboard for text input
class WiaBrailleKeyboard {

    private val display: WiaBrailleDisplay
    private var inputBuffer = StringBuilder()

    init {
        display.enableBrailleInput()

        display.setBrailleInputListener { event ->
            handleBrailleInput(event)
        }
    }

    private fun handleBrailleInput(event: WiaBrailleInputEvent) {
        // Convert Braille pattern to character
        val char = brailleToText(event.pattern, grade = 2)

        // Check for special commands
        when {
            event.isBackspace() -> {
                if (inputBuffer.isNotEmpty()) {
                    inputBuffer.deleteCharAt(inputBuffer.length - 1)
                    updateDisplay()
                }
            }
            event.isEnter() -> {
                submitInput()
            }
            else -> {
                inputBuffer.append(char)
                updateDisplay()
            }
        }
    }

    private fun updateDisplay() {
        display.write("Input: $inputBuffer")
    }

    private fun submitInput() {
        // Send input to focused text field
        val inputConnection = currentInputConnection
        inputConnection?.commitText(inputBuffer.toString(), 1)
        inputBuffer.clear()
    }
}

# orca/braille/wia_braille.py
# Direct WIA integration for Orca

from orca import braille
from wia_braille import WiaBrailleDisplay, WiaError
import logging

logger = logging.getLogger(__name__)

class WiaBraille(braille.BrailleDevice):
    """WIA Braille Display support for Orca"""

    def __init__(self):
        super().__init__()
        self._display = None
        self._numCells = 0

    def open(self):
        """Open connection to WIA Braille display"""
        try:
            self._display = WiaBrailleDisplay()
            self._display.connect()

            info = self._display.get_device_info()
            self._numCells = info['cellCount']

            # Event handlers
            self._display.on('routing', self._onRouting)
            self._display.on('key', self._onKey)

            logger.info(f"Orca: Connected to WIA display "
                       f"({info['manufacturer']} {info['model']})")

            return True

        except WiaError as e:
            logger.error(f"Orca: WIA connection failed - {e}")
            return False

    def close(self):
        """Close display connection"""
        if self._display:
            try:
                self._display.disconnect()
            except WiaError as e:
                logger.error(f"Disconnect error: {e}")
            finally:
                self._display = None

    def getNumCells(self):
        """Get number of Braille cells"""
        return self._numCells

    def write(self, cells):
        """Write to Braille display"""
        if not self._display:
            return False

        try:
            self._display.write_cells(cells)
            return True
        except WiaError as e:
            logger.error(f"Write error: {e}")
            return False

    def _onRouting(self, event):
        """Handle routing key press"""
        # Trigger Orca routing action
        braille.processRoutingKey(event['cellIndex'])

    def _onKey(self, event):
        """Handle navigation keys"""
        keyMap = {
            'left': 'panLeft',
            'right': 'panRight',
            'up': 'lineUp',
            'down': 'lineDown',
        }

        orcaAction = keyMap.get(event['key'])
        if orcaAction:
            braille.processKey(orcaAction)

# Register driver
braille.registerDevice("wia", WiaBraille)

7.8 弘益人間 (Hongik Ingan) - Philosophy Reflection

弘益人間 (Hongik Ingan)
"Benefit All Humanity"

The WIA Braille Display Standard embodies the ancient Korean philosophy of 弘益人間 (Hongik Ingan) — "Benefit All Humanity." This isn't merely a tagline; it's the foundational principle guiding every technical decision in the standard.

Breaking Down Barriers

For decades, the Braille display ecosystem has been fragmented. A person who is blind purchasing a $3,000 Braille display faced uncertainty: Will it work with my screen reader? What about the next operating system update? Can I switch to a different screen reader if needed? These shouldn't be questions anyone has to ask.

WIA's universal approach means freedom of choice. A student using NVDA on Windows can collaborate with a colleague using VoiceOver on macOS, both with compatible Braille displays. A developer can switch from JAWS to Orca without replacing expensive hardware. This is 弘益人間 in action — removing barriers that unfairly burden a community.

Economic Justice

Braille displays are prohibitively expensive, partly due to hardware complexity, but also due to ecosystem fragmentation requiring device manufacturers to develop and maintain dozens of proprietary drivers. A single universal standard:

  • Reduces manufacturer development costs → Lower device prices
  • Increases competition → Better products
  • Enables innovation → New form factors and features
  • Encourages adoption → More users benefit

When we lower costs and increase availability, we honor 弘益人間 — making assistive technology accessible to all, not just the wealthy.

Global Inclusion

The WIA standard supports all Braille codes across all languages. English Grade 2, Unified English Braille (UEB), Korean Braille, Japanese Braille, Arabic Braille, Nemeth Code for mathematics — all are first-class citizens.

This multilingual commitment reflects 弘익人間 on a global scale. Accessibility isn't a Western concern; it's a human right worldwide. A blind student in Seoul deserves the same quality tools as one in San Francisco. A researcher in Cairo should have the same access to mathematical notation in Braille as one in Cambridge.

Open and Free Forever

The WIA Braille Display Standard is released under the MIT License — free forever, for anyone, anywhere. No licensing fees. No proprietary restrictions. No gatekeepers.

This openness is deliberate. If the goal is to "benefit all humanity," the standard must belong to humanity, not to a corporation or consortium. Open source enables:

  • Transparency — Anyone can audit the standard for accessibility and security
  • Contribution — Experts worldwide can improve and extend the specification
  • Innovation — Startups and researchers can build without permission
  • Longevity — The standard outlives any single organization

Developer Empowerment

By providing clean, well-documented APIs and comprehensive SDKs in multiple languages, WIA empowers developers to build accessible applications without becoming Braille experts. A web developer who's never used a Braille display can now integrate support in minutes:

import { WiaBrailleDisplay } from '@wia/braille-display';

const display = new WiaBrailleDisplay();
await display.connect();
await display.write('Accessible content!');

This simplicity is intentional. When accessibility is easy, it becomes ubiquitous. When it's ubiquitous, everyone benefits. This is 弘益人間.

A Standard for the Future

Technology evolves. Screen readers change. Operating systems update. Braille displays become smaller, cheaper, more advanced. But the WIA standard is designed to evolve with technology, not against it.

The four-phase architecture (Data, API, Protocol, Integration) provides stability at each layer while allowing innovation above. New protocols can be added (Phase 3) without breaking APIs (Phase 2). New devices can emerge without rewriting screen reader drivers (Phase 4).

This forward-thinking design ensures that WIA will benefit not just today's users, but future generations. Our children's children, should they need Braille, will inherit a robust, mature, universal ecosystem. That's the long-term vision of 弘益人間.

Chapter Summary

Phase 4 of the WIA Braille Display Standard addresses ecosystem integration, ensuring WIA-compliant devices work seamlessly across platforms and screen readers:

  • Integration Patterns - Direct integration, driver wrappers, and bridge services provide flexibility for different architectures
  • NVDA (Windows) - Python-based driver with gesture mapping and NVDA-specific event handling
  • JAWS (Windows) - Native C++ driver implementing the Braille Display Driver API
  • VoiceOver (macOS/iOS) - Swift/Objective-C driver using Apple's accessibility frameworks and MFi protocol for iOS
  • TalkBack (Android) - Kotlin-based accessibility service with Braille keyboard input support
  • Orca (Linux) - Python driver with BRLTTY integration and AT-SPI compatibility
  • Web Accessibility - ARIA integration and proposed Web Braille API for direct browser support
  • 弘益人間 Philosophy - The standard's commitment to universal accessibility, economic justice, and open collaboration

With comprehensive platform support, WIA enables true "write once, work everywhere" Braille accessibility.
