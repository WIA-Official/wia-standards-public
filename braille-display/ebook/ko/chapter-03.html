챕터 3: WIA 표준 개요

보편적 점자 접근성을 향한 통합 표준

학습 목표

이 챕터를 마친 후, 독자는 다음을 이해할 수 있습니다:

  1. WIA 점자 디스플레이 표준의 미션과 철학적 기반
  2. 4단계 계층 아키텍처의 구조와 각 계층의 역할
  3. 표준의 핵심 설계 원칙: 상호운용성, 확장성, 개방성
  4. 지원하는 점자 코드 유형과 다국어 처리 방식
  5. 6점 및 8점 점자 지원 전략
  6. 기존 표준(USB HID, Bluetooth, Unicode)과의 호환성
  7. 단계별 구현 및 채택 로드맵

3.1 미션: 보편적 점자 접근성

3.1.1 弘益人間 철학의 구현

弘益人間 (홍익인간)

"널리 인간을 이롭게 하라"

Technology should empower everyone, everywhere, without barriers.

기술은 장벽 없이 모든 사람, 모든 곳에서 사람들에게 힘을 실어주어야 합니다.

WIA 점자 디스플레이 표준은 이 고대 한국 철학을 현대 기술로 실현합니다. 챕터 2에서 본 마리아, 제임스, 사라, 압둘의 문제들은 기술적 해결책이 존재함에도 표준화 부재로 인한 것입니다.

3.1.2 핵심 미션

WIA 점자 디스플레이 표준의 미션:

  • 보편적 상호운용성: 모든 점자 디스플레이가 모든 소프트웨어, 모든 플랫폼에서 작동
  • 경제적 접근성: 표준화로 제조 비용 절감, 저가 혁신 촉진
  • 개발자 부담 경감: 단일 API로 모든 디스플레이 지원
  • 사용자 자유: 공급업체 종속 없이 자유로운 선택
  • 혁신 가속화: 표준 기반 위에서 새로운 기능 추가
  • 글로벌 포용: 모든 언어, 모든 국가, 모든 경제 수준 지원

3.1.3 문제 해결 비전

챕터 2의 사례들이 WIA 표준으로 어떻게 해결되는지:

사용자 챕터 2 문제 WIA 표준 해결책
개발자 마리아 VarioUltra 80을 Linux에서 설정하는 데 1주 소요, 일부 기능 작동 안 함 WIA 호환 디스플레이는 즉시 작동. OS 기본 드라이버로 플러그 앤 플레이. 모든 기능 보장.
학생 제임스 BrailleSense를 도서관 PC에 연결 불가. 온라인 시험 접근 불가. WIA 표준 HID/Bluetooth 프로토콜로 관리자 권한 없이 즉시 연결. 모든 플랫폼 호환.
IT 관리자 사라 3개 제조업체 디스플레이로 3배 비용 및 IT 시간 100시간/년 단일 WIA 드라이버로 모든 디스플레이 관리. IT 시간 20시간/년 이하. 비용 절감.
나이지리아 압둘 $599 디스플레이가 연소득 20%. 현지 지원 없음. 현지어 테이블 없음. 표준화로 저가 디스플레이 $100-200 목표. 오픈소스로 현지 개발자 기여 가능. 원격 지원 용이.

3.1.4 성공 지표

WIA 표준이 성공적으로 구현되면:

지표 현재 상태 WIA 목표 (5년)
제조업체 채택 독자 프로토콜 100% WIA 준수 70% 이상
평균 디스플레이 가격 (40셀) $3,000 $1,500 이하
OS 기본 지원 제한적 (특정 모델만) WIA 디스플레이 100% 즉시 작동
개발자 통합 시간 100-200시간 (모든 모델) 10-20시간 (단일 WIA API)
글로벌 점자 문해율 10-20% (선진국) 30-40% (접근성 향상으로)

3.2 4단계 아키텍처 개요

WIA 점자 디스플레이 표준은 OSI 모델처럼 계층화된 아키텍처를 채택합니다. 각 계층은 독립적으로 발전하면서도 상호 호환됩니다.

3.2.2 Phase 1: 데이터 형식 (Data Format Layer)

목적: 점자 데이터를 표현하는 표준 형식 정의

주요 구성 요소:

  • 점자 셀 데이터 모델: 각 셀의 핀 상태를 비트 또는 유니코드로 표현
  • JSON 스키마: 구조화된 점자 문서 형식
  • 유니코드 매핑: U+2800~U+28FF (6점), 확장 (8점)
  • 점자 테이블: 1급, 2급, 컴퓨터 점자, Nemeth 수학 등
  • 메타데이터: 언어, 등급, 속성 정보

예시:

{
  "version": "1.0",
  "type": "braille-display-data",
  "braille_grade": 2,
  "language": "en-US",
  "cells": [
    {"pattern": "⠓", "unicode": "U+2813", "dots": [1,2,5]},      // H
    {"pattern": "⠑", "unicode": "U+2811", "dots": [1,5]},        // e
    {"pattern": "⠇", "unicode": "U+2807", "dots": [1,2,3]},      // l
    {"pattern": "⠇", "unicode": "U+2807", "dots": [1,2,3]},      // l
    {"pattern": "⠕", "unicode": "U+2815", "dots": [1,3,5]}       // o
  ],
  "metadata": {
    "cursor_position": 0,
    "selection_start": null,
    "selection_end": null
  }
}

이점:

  • 제조업체 독립적 데이터 교환
  • 점자 문서 보관 및 전송 표준
  • 상위 계층의 견고한 기반

3.2.3 Phase 2: 프로토콜 (Protocol Layer)

목적: 디바이스와 호스트 간 통신 방법 표준화

지원 전송 계층:

전송 방식 기반 기술 용도
USB HID Braille (Usage Page 0x41) 데스크톱/노트북 유선 연결
Bluetooth HID over GATT (HOGP) 무선 모바일/PC 연결
WiFi WebSocket over HTTP 네트워크 기반, 다중 호스트
Serial RS-232 / UART 레거시 시스템, 임베디드

명령 세트 구조:

// WIA Braille Protocol 명령 예시
명령 종류:
- DISPLAY_TEXT: 점자 셀에 텍스트 표시
- GET_STATUS: 디스플레이 상태 조회
- SET_CURSOR: 커서 위치 설정
- KEY_EVENT: 사용자 키 입력 보고
- FIRMWARE_INFO: 펌웨어 버전 조회
- UPDATE_FIRMWARE: 펌웨어 업데이트

명령 형식 (USB HID):
Byte 0: 명령 ID
Byte 1: 데이터 길이
Byte 2-N: 페이로드
Byte N+1: 체크섬 (CRC8)

이점:

  • 통일된 통신 방식으로 드라이버 단순화
  • 플러그 앤 플레이 자동 인식
  • 안정적인 오류 처리 및 재연결

3.2.4 Phase 3: 드라이버 인터페이스 (Driver Interface Layer)

목적: 운영체제와 통합되는 통일된 드라이버 제공

운영체제별 구현:

운영체제 드라이버 형태 통합 방식
Windows UMDF (User-Mode Driver Framework) Accessibility API 통합
macOS IOKit Driver + VoiceOver 플러그인 NSAccessibility
Linux BRLTTY 드라이버 확장 AT-SPI (Assistive Technology SPI)
iOS VoiceOver 내장 UIAccessibility
Android AccessibilityService 확장 TalkBack 통합

핵심 기능:

  • 장치 자동 감지: WIA 준수 디스플레이를 즉시 인식
  • 능력 협상: 디스플레이 셀 수, 점자 등급, 키 배열 자동 감지
  • 설정 동기화: 사용자 설정을 디바이스와 OS 간 동기화
  • 전원 관리: 배터리 상태 모니터링, 절전 모드

이점:

  • 제임스의 문제 해결: 관리자 권한 없이 즉시 작동
  • 마리아의 문제 해결: Linux에서 복잡한 설정 없이 플러그 앤 플레이
  • 사라의 문제 해결: 단일 드라이버로 모든 WIA 디스플레이 관리

3.2.5 Phase 4: 애플리케이션 API (Application API Layer)

목적: 개발자를 위한 고수준 SDK 제공

지원 언어:

  • TypeScript / JavaScript (웹 애플리케이션, Electron)
  • Python (데이터 과학, 교육 도구)
  • C++ (고성능 네이티브 앱)
  • Java (Android, 엔터프라이즈)
  • Rust (시스템 프로그래밍, 성능 중시)
  • Go (서버 사이드, 마이크로서비스)

API 예시 (TypeScript):

import { WIABrailleDisplay } from 'wia-braille-sdk';

// 디스플레이 연결
const display = await WIABrailleDisplay.connect({
  autoDetect: true,
  brailleGrade: 2,
  language: 'en-US'
});

// 텍스트 표시
await display.setText("Hello, World!");

// 커서 위치 설정
await display.setCursor(7);

// 키 이벤트 처리
display.on('keyPress', (event) => {
  console.log(`Key pressed: ${event.key}`);
  if (event.key === 'ROUTING_BUTTON') {
    console.log(`Routing to cell: ${event.cellIndex}`);
  }
});

// 디스플레이 정보 조회
const info = await display.getInfo();
console.log(`Display: ${info.manufacturer} ${info.model}`);
console.log(`Cells: ${info.cellCount}, Type: ${info.cellType}`);

// 연결 해제
await display.disconnect();

이점:

  • 개발 시간 90% 단축 (100-200시간 → 10-20시간)
  • 일관된 API로 학습 곡선 낮음
  • 크로스 플랫폼 개발 용이

3.2.6 계층 간 상호작용

실제 사용 시나리오: 사용자가 "Python" 단어를 점자로 읽는 경우

1. 애플리케이션 계층 (Phase 4):
   - 스크린 리더가 WIA SDK 호출: setText("Python")

2. API 계층이 요청 처리:
   - 언어(en-US), 등급(2급) 확인
   - 텍스트를 점자로 변환: "Python" → "⠏⠽⠹⠕⠝"

3. 드라이버 계층 (Phase 3)에 전달:
   - OS 드라이버가 연결된 디스플레이 식별
   - 디스플레이 능력 확인 (40셀, 6점)

4. 프로토콜 계층 (Phase 2)으로 변환:
   - DISPLAY_TEXT 명령 생성
   - Bluetooth HID 패킷 구성

5. 데이터 형식 계층 (Phase 1):
   - 각 문자를 점자 패턴으로 인코딩
   - ⠏(U+280F), ⠽(U+283D), ⠹(U+2839), ⠕(U+2815), ⠝(U+281D)

6. 하드웨어로 전송:
   - Bluetooth를 통해 디스플레이에 패킷 전송
   - 디스플레이 펌웨어가 피에조 액추에이터 제어
   - 핀들이 움직여 점자 패턴 형성

7. 사용자 상호작용:
   - 사용자가 손가락으로 점자 읽음
   - 라우팅 버튼 누름 (예: 3번째 셀)

8. 이벤트 역방향 전파:
   - 하드웨어 → 프로토콜: KEY_EVENT 명령
   - 프로토콜 → 드라이버: 키 이벤트 해석
   - 드라이버 → API: onKeyPress 콜백
   - API → 애플리케이션: 커서를 "t" 위치로 이동

이 계층화된 접근 방식은 각 계층이 독립적으로 발전하면서도 전체 시스템이 조화롭게 작동하도록 합니다.

3.3 핵심 설계 원칙

3.3.1 상호운용성 (Interoperability)

원칙: "어떤 디스플레이든, 어떤 소프트웨어든, 어떤 플랫폼이든 함께 작동해야 한다."

구현 전략:

  • 제조업체 중립성: WIA 표준은 특정 제조업체를 선호하지 않음
  • 하위 호환성: 신규 버전이 구버전 디바이스와 호환
  • 능력 감지: 디바이스 능력을 자동 감지하고 적응
  • 폴백 메커니즘: 고급 기능 미지원 시 기본 기능으로 대체

테스트 기준:

테스트 기준
크로스 플랫폼 Windows, macOS, Linux, iOS, Android에서 동일 기능
크로스 제조업체 5개 이상 제조업체 디스플레이에서 테스트
스크린 리더 호환 JAWS, NVDA, VoiceOver, TalkBack, Orca 모두 지원
레거시 지원 10년 이상 된 디바이스도 기본 기능 작동

3.3.2 확장성 (Scalability)

원칙: "표준은 현재뿐 아니라 미래 기술도 수용해야 한다."

확장 메커니즘:

  • 버전 관리: 시맨틱 버전닝 (SemVer) 사용
  • 선택적 기능: 핵심 기능과 확장 기능 분리
  • 확장 포인트: 제조업체별 커스텀 기능 허용
  • 미래 대비: 그래픽 점자, 촉각 피드백 등 미래 기술 고려

확장 로드맵:

버전 핵심 기능 확장 기능
1.0 (현재) - 기본 텍스트 표시
- 6/8점 점자
- USB/Bluetooth		 - 다국어 테이블
- 점자 속성
1.5 (2026) - WiFi 지원
- 다중 호스트		 - 그래픽 점자 (프리뷰)
- 햅틱 피드백
2.0 (2027) - 그래픽 점자 (정식)
- AI 통합 API		 - 3D 촉각
- 신경 인터페이스 (연구)

3.3.3 개방성 (Openness)

원칙: "표준은 모두에게 자유롭고 투명하게 공개되어야 한다."

오픈소스 전략:

  • 라이선스: Creative Commons BY-SA 4.0 (문서), MIT/Apache 2.0 (코드)
  • 참여: GitHub 기반 협업, 누구나 기여 가능

3.4.3 점자 변환 엔진

WIA 표준은 liblouis 오픈소스 점자 변환 라이브러리를 권장합니다:

  • 300개 이상 언어 테이블 지원
  • 1급, 2급, 컴퓨터, Nemeth 점자 변환
  • 역방향 변환 (점자 → 텍스트)
  • 커뮤니티 주도, 지속적 업데이트

예시 (Python):

import louis

# 영어 2급 점자로 변환
text = "Hello, world!"
braille = louis.translateString(["en-us-g2.ctb"], text)
print(braille)  # ⠠⠓⠑⠇⠇⠕⠠ ⠺⠕⠗⠇⠙⠲

# 한국어 점자로 변환
korean_text = "안녕하세요"
korean_braille = louis.translateString(["ko-g1.ctb"], korean_text)
print(korean_braille)  # ⠁⠝⠝⠕⠝⠓⠁⠎⠑⠽⠕

# Nemeth 수학 점자
math_text = "x^2 + y^2 = r^2"
math_braille = louis.translateString(["nemeth.ctb"], math_text)
print(math_braille)  # ⠭⠘⠆⠐⠬⠽⠘⠆⠀⠐⠶⠀⠗⠘⠆

3.5 6점 및 8점 점자 지원

3.5.1 6점 점자 (Standard Braille)

구조: 2×3 그리드, 6개 핀

1 ● ○ 4
2 ● ○ 5
3 ● ○ 6

예: 문자 'A' = ⠁ = 점 1만
    문자 'B' = ⠃ = 점 1,2
    문자 'Z' = ⠵ = 점 1,3,5,6

유니코드 범위: U+2800 ~ U+283F (64개 조합)

용도:

  • 일반 문학 텍스트 (소설, 에세이)
  • 교육 자료 (대부분의 교과서)
  • 웹 콘텐츠 (뉴스, 블로그)

3.5.2 8점 점자 (Computer Braille)

구조: 2×4 그리드, 8개 핀

1 ● ○ 4
2 ● ○ 5
3 ● ○ 6
7 ● ○ 8

예: 대문자 'A' = ⡁ = 점 1, 7
    '{' = ⢸ = 점 4,6,7,8
    '[' = ⢷ = 점 1,2,4,6,7,8

유니코드 범위: U+2800 ~ U+28FF (256개 조합)

용도:

  • 프로그래밍 코드 (Python, Java, C++, etc.)
  • 시스템 관리 (터미널, 셸 스크립트)
  • 특수 문자 (괄호, 제어 문자)
  • Nemeth 수학 (확장 기호)

3.5.3 WIA 표준의 통합 지원

WIA 표준은 6점과 8점을 동시에 지원하며, 자동 전환 기능을 제공합니다:

컨텍스트 사용 점자 예시
일반 텍스트 6점 (1급 또는 2급) 이메일 본문, 문서
코드 블록 8점 (컴퓨터 점자) <code>...</code>
터미널 출력 8점 bash, cmd, PowerShell
수학 수식 8점 (Nemeth) <math>...</math>
파일 경로 8점 /usr/bin/python

자동 모드 전환 예시:

사용자가 다음 텍스트를 읽는 경우:

"To calculate the area, use this formula:

    area = pi * r^2

This Python code implements it:

    def area(radius):
        return 3.14159 * radius ** 2"

WIA 스마트 변환:
1. "To calculate..." → 6점 영어 2급 점자
2. "area = pi * r^2" → 8점 Nemeth 점자 (수학)
3. "This Python..." → 6점 영어 2급
4. "def area(radius)..." → 8점 컴퓨터 점자 (코드)
5. 사용자는 자연스럽게 전환되며, 모드 표시자로 변경 알림

3.5.4 하드웨어 호환성

디스플레이 유형 6점 지원 8점 지원 WIA 전략
6점 전용 디스플레이 ✓ ✗ 8점 콘텐츠를 6점으로 근사 변환 (손실 가능)
8점 디스플레이 ✓ ✓ 완전 지원, 동적 전환

6점 디스플레이의 8점 폴백:

// 8점 문자 '{' (⢸, 점 4,6,7,8)를 6점 디스플레이에 표시
옵션 1: 점자 표시자 사용
- "⠸" (점 4,5,6) + "{" 텍스트 설명

옵션 2: 가장 가까운 6점 패턴
- "⠨" (점 4,6) 사용 (불완전하지만 구별 가능)

옵션 3: 텍스트 설명
- "left brace" 또는 "여는 중괄호"

WIA 표준은 제조업체/사용자가 선호하는 폴백 전략을 설정할 수 있도록 합니다.

3.6 기존 표준과의 호환성

3.6.1 USB HID Braille (Usage Page 0x41)

WIA 표준은 USB-IF (USB Implementers Forum)의 HID Braille 표준을 기반으로 확장합니다.

항목 USB HID Braille WIA 확장
리포트 형식 기본 입출력 리포트 메타데이터, 능력 협상 추가
점자 데이터 바이트 배열 유니코드 점자 + 속성
키 이벤트 단순 키 코드 제스처, 터치, 라우팅 강화

하위 호환성: WIA 준수 디스플레이는 기본 HID Braille 모드로 폴백하여 레거시 시스템과도 작동합니다.

3.6.2 Bluetooth HID over GATT (HOGP)

Bluetooth 무선 연결을 위해 WIA는 HOGP (HID over GATT Profile)를 사용합니다.

  • 장점: iOS, Android, Windows, macOS 모두 기본 지원
  • 전력 효율: Bluetooth Low Energy (BLE) 사용
  • 보안: 페어링 암호화 (AES-128)

WIA 확장:

  • 커스텀 GATT 서비스: 00001812-0000-1000-8000-00805f9b34fb (HID)
  • 추가 특성 (Characteristics): 펌웨어 버전, 배터리 상태, 능력 정보

3.6.3 Unicode Braille Patterns (U+2800 ~ U+28FF)

WIA 표준은 유니코드 점자 패턴을 기본 인코딩으로 사용합니다.

범위 점 수 조합 수 용도
U+2800 ~ U+283F 6점 64 일반 점자
U+2840 ~ U+28FF 8점 (점 7,8 포함) 192 (추가) 컴퓨터 점자

예시:

문자 'A':
- 1급 점자: ⠁ (U+2801, 점 1)
- 컴퓨터 점자 (대문자): ⡁ (U+2840+0x01, 점 1,7)

문자 '{':
- 컴퓨터 점자: ⢸ (U+28B8, 점 4,6,7,8)

3.8 챕터 요약: 5가지 핵심 포인트

  1. WIA는 弘益人間 철학을 기술로 구현합니다

    보편적 점자 접근성을 미션으로, 챕터 2의 마리아, 제임스, 사라, 압둘의 문제를 해결합니다. 모든 디스플레이가 모든 플랫폼에서 작동하고, 비용은 절감되며, 공급업체 종속이 없습니다.

  2. 4단계 계층 아키텍처는 확장 가능하고 미래 지향적입니다

    데이터 형식 → 프로토콜 → 드라이버 → API로 구성된 레이어드 접근은 각 계층이 독립적으로 발전하면서도 전체가 조화롭게 작동합니다. OSI 모델처럼 명확한 책임 분리가 있습니다.

  3. 상호운용성, 확장성, 개방성이 핵심 설계 원칙입니다

    어떤 디스플레이든 어떤 소프트웨어든 작동(상호운용성), 미래 기술 수용(확장성), 오픈소스로 모두에게 공개(개방성). Creative Commons 문서, MIT/Apache 코드 라이선스로 진입 장벽을 제거합니다.

  4. 1급, 2급, 컴퓨터, Nemeth 점자와 10개 이상 언어를 지원합니다

    영어, 한국어, 중국어, 아랍어 등 다국어 점자 테이블을 기본 제공하고, 커뮤니티가 새 언어 추가 가능. 6점과 8점 모두 지원하며, 컨텍스트에 따라 자동 전환(코드 블록은 8점, 일반 텍스트는 6점).

  5. USB HID, Bluetooth, 유니코드 기존 표준과 완전 호환됩니다

    WIA는 바퀴를 재발명하지 않고, USB-IF HID Braille, Bluetooth HOGP, Unicode U+2800-28FF 위에 구축됩니다. 하위 호환성으로 레거시 시스템과도 작동하며, 2026년부터 단계별 채택 목표입니다.

    4단계: (1) Phase 1 데이터 형식 - 점자 셀 표현 (JSON, 유니코드), (2) Phase 2 프로토콜 - USB/Bluetooth 통신 명령, (3) Phase 3 드라이버 - OS 통합 및 장치 추상화, (4) Phase 4 API - 고수준 SDK (TypeScript, Python 등). 시나리오: "Python" 표시 시 → (4) SDK setText() 호출 → (3) 드라이버가 디스플레이 식별 → (2) DISPLAY_TEXT 명령 생성 → (1) 각 문자를 점자 패턴 인코딩 → Bluetooth 전송 → 하드웨어 핀 제어. 역방향: 라우팅 버튼 누름 → (1) KEY_EVENT → (2) 프로토콜 해석 → (3) 드라이버 이벤트 → (4) onKeyPress 콜백.

  2. WIA 표준이 챕터 2의 4명 사용자(마리아, 제임스, 사라, 압둘) 문제를 각각 어떻게 해결하는지 구체적으로 설명하세요.
    답안 보기

    마리아(개발자): Linux에서 VarioUltra 80 설정 1주 소요 → WIA 표준 드라이버로 플러그 앤 플레이, 즉시 작동. 제임스(학생): 도서관 PC에 BrailleSense 연결 불가(관리자 권한) → WIA HID/Bluetooth로 권한 없이 즉시 연결, 온라인 시험 접근 가능. 사라(IT 관리자): 3개 제조업체 디스플레이로 IT 시간 100시간/년 → 단일 WIA 드라이버로 20시간 이하, 비용 3분의 1. 압둘(나이지리아): $599가 연소득 20% → 표준화로 저가 디스플레이 $100-200 목표, 오픈소스로 Hausa 점자 테이블 현지 개발 가능, 원격 지원 용이.

  3. WIA 표준의 3가지 핵심 설계 원칙(상호운용성, 확장성, 개방성)을 설명하고, 각 원칙이 왜 중요한지 실제 예와 함께 논하세요.
    답안 보기

    상호운용성: 어떤 디스플레이든 어떤 소프트웨어/플랫폼에서든 작동. 중요성: 사용자 선택 자유, 공급업체 종속 제거. 예: 제임스가 Focus 40에서 Brailliant로 변경해도 모든 앱 그대로 작동. 확장성: 미래 기술(그래픽 점자, AI) 수용. 중요성: 표준이 오래 유지됨. 예: 2027년 그래픽 점자가 나와도 WIA 2.0 확장으로 대응, 기존 장치도 펌웨어 업데이트로 지원. 개방성: 오픈소스, 무료 인증, 커뮤니티 거버넌스. 중요성: 진입 장벽 제거, 혁신 촉진. 예: 압둘 같은 신흥 시장 현지 개발자가 점자 테이블 기여 가능, NGO가 저가 디스플레이 제조 가능.

  4. 6점 점자와 8점 점자의 차이를 설명하고, WIA 표준이 두 유형을 어떻게 통합 지원하는지 자동 모드 전환 예시와 함께 서술하세요.
    답안 보기

    6점: 2×3 그리드, 64 조합, 일반 문학 텍스트. 8점: 2×4 그리드, 256 조합, 컴퓨터 코드/수학. WIA 통합: 컨텍스트 자동 감지로 전환. 예: "Python 코드: def area(r): return 3.14 * r**2" 문장 → "Python 코드:" (6점 영어 2급) → def area... (8점 컴퓨터 점자, <code> 태그 감지) → 수식 "r**2" (8점 Nemeth). 6점 전용 디스플레이: 8점 문자 '{' (⢸)를 폴백 - 옵션 1: 점자 표시자 + 설명, 옵션 2: 가장 가까운 6점 패턴, 옵션 3: "left brace" 텍스트. 사용자가 선호 전략 설정 가능.

  5. WIA 표준이 지원하는 점자 등급(1급, 2급, 컴퓨터, Nemeth)과 언어 테이블(최소 5개)을 나열하고, 각각의 용도를 설명하세요.
    답안 보기

    등급: (1) 1급 - 문자 대 문자, 초보자/기술 문서, (2) 2급 - 축약형, 일반 읽기 (UEB), (3) 컴퓨터 - 8점, 프로그래밍/ASCII, (4) Nemeth - 수학/과학, STEM 교육. 언어: (1) en-US/GB - 영어 UEB 1/2급, (2) ko-KR - 한글 점자 + 약자, (3) zh-CN - 중국어 간체, 성조 표시, (4) ar-SA - 아랍어, 오른쪽→왼쪽, (5) ja-JP - 일본어, 가나/한자 혼용, (6) de-DE - 독일어 움라우트, (7) fr-FR - 프랑스어 악센트. 확장: 커뮤니티가 GitHub로 새 언어 테이블 기여 가능(예: 압둘의 Hausa).

  6. WIA 표준과 기존 표준(USB HID Braille, Bluetooth HOGP, Unicode)의 관계를 설명하고, 하위 호환성이 어떻게 보장되는지 논하세요.
    답안 보기

    WIA는 기존 표준 위에 구축되어 재발명 방지. USB HID Braille: Usage Page 0x41 기반, WIA가 메타데이터/능력 협상/유니코드 점자 추가. Bluetooth HOGP: HID over GATT 사용, BLE 전력 효율, AES-128 암호화. WIA는 커스텀 GATT 특성(펌웨어 버전, 배터리) 추가. Unicode U+2800-28FF: 6점(U+2800-283F)/8점(U+2840-28FF) 기본 인코딩. 하위 호환: WIA 디스플레이는 "레거시 모드"로 폴백하여 기본 HID Braille만 지원하는 구 시스템과도 작동. 예: 2015년 Windows 7 PC에서도 기본 텍스트 표시 가능 (고급 기능은 제외).
