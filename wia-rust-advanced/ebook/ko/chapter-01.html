🎯 Unsafe란 무엇인가?

Rust의 가장 큰 특징은 메모리 안전성입니다. 하지만 때로는 컴파일러의 안전장치를 벗어나야 하는 경우가 있습니다. 바로 이때 unsafe 키워드를 사용합니다.

💡 핵심 개념
unsafe는 "위험하다"는 의미가 아니라, "컴파일러가 안전성을 보장할 수 없으니 프로그래머가 책임진다"는 의미입니다. 올바르게 사용하면 안전하고 성능이 뛰어난 코드를 작성할 수 있습니다.

Unsafe가 필요한 경우

  • 저수준 시스템 프로그래밍: 운영체제, 디바이스 드라이버 개발
  • FFI (Foreign Function Interface): C/C++ 라이브러리와 상호작용
  • 성능 최적화: 컴파일러 검사를 우회하여 성능 향상
  • 하드웨어 직접 접근: 메모리 맵 I/O, 임베디드 시스템

🔍 Unsafe의 다섯 가지 능력

Unsafe 블록 내에서는 다음 다섯 가지를 추가로 할 수 있습니다:

1. 원시 포인터 역참조

*const T와 *mut T 타입의 포인터를 역참조할 수 있습니다.

2. Unsafe 함수 호출

unsafe로 표시된 함수나 메서드를 호출할 수 있습니다.

3. 정적 변수 접근/수정

가변 정적 변수(static mut)에 접근하거나 수정할 수 있습니다.

4. Unsafe 트레잇 구현

unsafe로 표시된 트레잇을 구현할 수 있습니다.

5. Union 필드 접근

union 타입의 필드에 접근할 수 있습니다.

📝 원시 포인터 (Raw Pointers)

원시 포인터는 Rust의 참조와 비슷하지만, 컴파일러의 차용 검사를 받지 않습니다. 두 가지 타입이 있습니다:

  • *const T: 불변 원시 포인터
  • *mut T: 가변 원시 포인터

원시 포인터 생성 및 사용

fn raw_pointer_demo() {
    let mut num = 42;

    // 안전한 컨텍스트에서 원시 포인터 생성 가능
    let r1 = &num as *const i32;
    let r2 = &mut num as *mut i32;

    // 역참조는 unsafe 블록 필요
    unsafe {
        println!("r1: {}", *r1);
        println!("r2: {}", *r2);

        // 가변 포인터로 값 수정
        *r2 = 100;
    }

    println!("num: {}", num); // 100
}

임의의 메모리 주소 접근

fn arbitrary_address() {
    let address = 0x012345usize;
    let r = address as *const i32;

    // ⚠️ 위험: 이 주소가 유효한지 알 수 없음!
    unsafe {
        // println!("{}", *r); // 충돌 가능성 있음!
    }
}
⚠️ 주의사항
원시 포인터는 다음을 보장하지 않습니다:
  • 항상 유효한 메모리를 가리킨다
  • null이 아니다
  • 데이터 경쟁이 없다
  • 수명(lifetime)이 유효하다
프로그래머가 직접 이를 보장해야 합니다!

🔧 Unsafe 함수와 메서드

함수 전체가 unsafe한 작업을 수행한다면, 함수 자체를 unsafe로 표시할 수 있습니다. 

unsafe fn dangerous() {
    // unsafe 코드
}

fn main() {
    unsafe {
        dangerous();
    }
}

실전 예제: 슬라이스 분할

슬라이스를 두 개의 가변 슬라이스로 분할하는 함수를 만들어봅시다. 일반적인 Rust로는 불가능하지만, unsafe를 사용하면 가능합니다. 

use std::slice;

fn split_at_mut(values: &mut [i32], mid: usize) -> (&mut [i32], &mut [i32]) {
    let len = values.len();
    let ptr = values.as_mut_ptr();

    assert!(mid <= len);

    unsafe {
        (
            slice::from_raw_parts_mut(ptr, mid),
            slice::from_raw_parts_mut(ptr.add(mid), len - mid),
        )
    }
}

fn main() {
    let mut v = vec![1, 2, 3, 4, 5, 6];

    let (left, right) = split_at_mut(&mut v, 3);

    left[0] = 100;
    right[0] = 200;

    println!("{:?}", v); // [100, 2, 3, 200, 5, 6]
}
💡 왜 안전한가?
위 코드는 다음을 보장합니다:
  • 두 슬라이스가 겹치지 않음 (mid 인덱스로 분할)
  • 원본 슬라이스의 범위를 벗어나지 않음 (assert!로 검증)
  • 포인터 연산이 올바름 (ptr.add(mid))

🌐 정적 변수 (Static Variables)

정적 변수는 프로그램 전체 수명 동안 존재하는 전역 변수입니다.

불변 정적 변수

static HELLO_WORLD: &str = "Hello, world!";

fn main() {
    println!("name is: {}", HELLO_WORLD);
}

가변 정적 변수

static mut COUNTER: u32 = 0;

fn add_to_count(inc: u32) {
    unsafe {
        COUNTER += inc;
    }
}

fn main() {
    add_to_count(3);

    unsafe {
        println!("COUNTER: {}", COUNTER);
    }
}
⚠️ 데이터 경쟁 주의
가변 정적 변수는 여러 스레드에서 동시에 접근할 수 있어 데이터 경쟁이 발생할 수 있습니다. 멀티스레드 환경에서는 Mutex나 Atomic 타입을 사용하세요!

🎭 Unsafe 트레잇

트레잇 자체가 unsafe한 경우, 구현도 unsafe로 표시해야 합니다. 

unsafe trait Foo {
    // 메서드들
}

unsafe impl Foo for i32 {
    // 구현
}

실전 예제: Send와 Sync

Rust의 대표적인 unsafe 트레잇인 Send와 Sync: 

// Send: 다른 스레드로 소유권 이전 가능
unsafe impl Send for MyType {}

// Sync: 여러 스레드에서 불변 참조 공유 가능
unsafe impl Sync for MyType {}

struct MyType {
    // 필드들
}

✅ Unsafe 코드 작성 가이드라인

✅ 모범 사례
  1. 최소화: unsafe 블록을 가능한 한 작게 유지
  2. 문서화: 왜 안전한지 주석으로 설명
  3. 추상화: unsafe 코드를 안전한 API로 감싸기
  4. 테스트: 철저한 단위 테스트 작성
  5. Miri: Rust의 중간 표현 인터프리터로 검증

안전한 추상화 예제

pub struct Buffer {
    ptr: *mut u8,
    len: usize,
}

impl Buffer {
    // unsafe 생성자: 호출자가 안전성 보장
    pub unsafe fn from_raw_parts(ptr: *mut u8, len: usize) -> Self {
        Buffer { ptr, len }
    }

    // 안전한 메서드: 내부에서 검증
    pub fn get(&self, index: usize) -> Option<u8> {
        if index < self.len {
            unsafe {
                Some(*self.ptr.add(index))
            }
        } else {
            None
        }
    }
}
