🎯 매크로란 무엇인가?

매크로(Macro)는 코드를 생성하는 코드입니다. 컴파일 타임에 실행되어 반복적인 코드를 자동화하고, DSL(Domain Specific Language)을 만들 수 있게 해줍니다.

💡 매크로 vs 함수
  • 실행 시점: 매크로는 컴파일 타임, 함수는 런타임
  • 타입 제약: 매크로는 다양한 타입 받을 수 있음
  • 가변 인자: 매크로는 가변 개수 인자 가능
  • 메타프로그래밍: 매크로는 코드 구조 자체를 조작

📝 선언적 매크로 (macro_rules!)

가장 일반적인 형태의 매크로입니다. macro_rules!로 정의하며, 패턴 매칭을 통해 코드를 생성합니다.

기본 문법

macro_rules! say_hello {
    () => {
        println!("Hello!");
    };
}

fn main() {
    say_hello!(); // Hello!
}

매개변수가 있는 매크로

macro_rules! create_function {
    // 패턴: $func_name:ident 형태
    ($func_name:ident) => {
        fn $func_name() {
            println!("You called {:?}()", stringify!($func_name));
        }
    };
}

// 함수 생성
create_function!(foo);
create_function!(bar);

fn main() {
    foo(); // You called "foo"()
    bar(); // You called "bar"()
}

매크로 지정자(Designators)

expr

표현식 (1 + 2, func())

ident

식별자 (변수명, 함수명)

ty

타입 (i32, String)

pat

패턴 (match 패턴)

stmt

구문 (let x = 5;)

block

블록 ({ ... })

item

아이템 (fn, struct)

tt

토큰 트리 (모든 토큰)

반복 패턴

macro_rules! vec_of_strings {
    // $( ... )* 는 0회 이상 반복
    ($($element:expr),*) => {
        {
            let mut v = Vec::new();
            $(
                v.push($element.to_string());
            )*
            v
        }
    };
}

fn main() {
    let v = vec_of_strings!("hello", "world", "rust");
    println!("{:?}", v); // ["hello", "world", "rust"]
}

실전 예제: HashMap 생성 매크로

macro_rules! hashmap {
    ($($key:expr => $val:expr),* $(,)?) => {
        {
            let mut map = HashMap::new();
            $(
                map.insert($key, $val);
            )*
            map
        }
    };
}

use std::collections::HashMap;

fn main() {
    let map = hashmap! {
        "name" => "Alice",
        "age" => "30",
        "city" => "Seoul",
    };

    println!("{:?}", map);
}

🎨 절차적 매크로 (Procedural Macros)

절차적 매크로는 Rust 코드를 입력으로 받아 새로운 Rust 코드를 생성하는 함수입니다. 세 가지 종류가 있습니다:

  1. Custom Derive Macros: #[derive(MyTrait)]
  2. Attribute Macros: #[my_attribute]
  3. Function-like Macros: my_macro!(...)

프로젝트 설정

# Cargo.toml
[lib]
proc-macro = true

[dependencies]
syn = "2.0"
quote = "1.0"
proc-macro2 = "1.0"

1. 커스텀 Derive 매크로

구조체에 대해 자동으로 트레잇을 구현합니다. 

// my_macro/src/lib.rs
use proc_macro::TokenStream;
use quote::quote;
use syn;

#[proc_macro_derive(HelloMacro)]
pub fn hello_macro_derive(input: TokenStream) -> TokenStream {
    // 입력을 구문 트리로 파싱
    let ast = syn::parse(input).unwrap();

    // 트레잇 구현 빌드
    impl_hello_macro(&ast)
}

fn impl_hello_macro(ast: &syn::DeriveInput) -> TokenStream {
    let name = &ast.ident;
    let gen = quote! {
        impl HelloMacro for #name {
            fn hello_macro() {
                println!("Hello, Macro! My name is {}!", stringify!(#name));
            }
        }
    };
    gen.into()
}

// 사용 예제
use my_macro::HelloMacro;

trait HelloMacro {
    fn hello_macro();
}

#[derive(HelloMacro)]
struct Pancakes;

fn main() {
    Pancakes::hello_macro(); // Hello, Macro! My name is Pancakes!
}

2. 속성 매크로 (Attribute Macros)

함수나 구조체에 커스텀 속성을 추가합니다. 

#[proc_macro_attribute]
pub fn route(attr: TokenStream, item: TokenStream) -> TokenStream {
    let input = syn::parse_macro_input!(item as syn::ItemFn);
    let attrs = syn::parse_macro_input!(attr as syn::AttributeArgs);

    let fn_name = &input.sig.ident;

    let gen = quote! {
        pub fn #fn_name() {
            println!("Route registered!");
            #input
        }
    };

    gen.into()
}

// 사용 예제
#[route(GET, "/")]
fn index() -> String {
    "Hello, World!".to_string()
}

3. 함수형 매크로 (Function-like Macros)

함수처럼 보이지만 컴파일 타임에 실행되는 매크로입니다. 

#[proc_macro]
pub fn sql(input: TokenStream) -> TokenStream {
    let input_str = input.to_string();

    // SQL 파싱 및 검증 로직
    let gen = quote! {
        // SQL 쿼리 실행 코드 생성
    };

    gen.into()
}

// 사용 예제
let query = sql!(SELECT * FROM users WHERE id = ?);

🔍 고급 매크로 패턴

TT Muncher 패턴

복잡한 입력을 단계적으로 처리하는 패턴입니다. 

macro_rules! tt_muncher {
    // 베이스 케이스: 빈 입력
    () => {
        println!("Done!");
    };

    // 재귀 케이스: 첫 토큰 처리 후 나머지 반복
    ($tt:tt $($rest:tt)*) => {
        println!("Processing: {}", stringify!($tt));
        tt_muncher!($($rest)*);
    };
}

fn main() {
    tt_muncher!(hello world 123);
}

Push-down Accumulation

결과를 누적하면서 매크로를 확장합니다. 

macro_rules! reverse {
    // 베이스 케이스
    (@rev [] [$($reversed:tt)*]) => {
        [$($reversed),*]
    };

    // 재귀 케이스
    (@rev [$first:tt $($rest:tt)*] [$($reversed:tt)*]) => {
        reverse!(@rev [$($rest)*] [$first $($reversed)*])
    };

    // 진입점
    ($($tokens:tt)*) => {
        reverse!(@rev [$($tokens)*] [])
    };
}

fn main() {
    let arr = reverse!(1 2 3 4 5);
    println!("{:?}", arr); // [5, 4, 3, 2, 1]
}

🛠️ 실전 예제: Builder 패턴 매크로

use proc_macro::TokenStream;
use quote::quote;
use syn::{parse_macro_input, DeriveInput, Data, Fields};

#[proc_macro_derive(Builder)]
pub fn derive_builder(input: TokenStream) -> TokenStream {
    let input = parse_macro_input!(input as DeriveInput);
    let name = input.ident;
    let builder_name = syn::Ident::new(
        &format!("{}Builder", name),
        name.span()
    );

    let fields = match input.data {
        Data::Struct(ref data) => match data.fields {
            Fields::Named(ref fields) => &fields.named,
            _ => panic!("Builder only works with named fields"),
        },
        _ => panic!("Builder only works with structs"),
    };

    let builder_fields = fields.iter().map(|f| {
        let name = &f.ident;
        let ty = &f.ty;
        quote! {
            #name: Option<#ty>
        }
    });

    let methods = fields.iter().map(|f| {
        let name = &f.ident;
        let ty = &f.ty;
        quote! {
            pub fn #name(mut self, #name: #ty) -> Self {
                self.#name = Some(#name);
                self
            }
        }
    });

    let build_fields = fields.iter().map(|f| {
        let name = &f.ident;
        quote! {
            #name: self.#name.take().ok_or("Missing field")?
        }
    });

    let expanded = quote! {
        impl #name {
            pub fn builder() -> #builder_name {
                #builder_name::default()
            }
        }

        #[derive(Default)]
        pub struct #builder_name {
            #(#builder_fields),*
        }

        impl #builder_name {
            #(#methods)*

            pub fn build(mut self) -> Result<#name, &'static str> {
                Ok(#name {
                    #(#build_fields),*
                })
            }
        }
    };

    TokenStream::from(expanded)
}

// 사용 예제
#[derive(Builder)]
struct User {
    name: String,
    email: String,
    age: u32,
}

fn main() {
    let user = User::builder()
        .name("Alice".to_string())
        .email("alice@example.com".to_string())
        .age(30)
        .build()
        .unwrap();

    println!("{} ({})", user.name, user.email);
}

}

    };
}

✅ 매크로 모범 사례

✅ 모범 사례
  1. 명확한 이름: 매크로 이름은 명확하고 직관적으로
  2. 문서화: 예제와 함께 상세히 문서화
  3. 위생(Hygiene): 의도치 않은 변수 충돌 방지
  4. 에러 메시지: 유용한 컴파일 에러 제공
  5. 최소화: 꼭 필요한 경우에만 사용
  6. 테스트: 다양한 입력에 대해 철저히 테스트
⚠️ 주의사항
  • 매크로는 디버깅이 어려움 - 단순함 유지
  • 과도한 매크로 사용은 코드 가독성 저하
  • 컴파일 시간 증가 가능
  • IDE 지원이 제한적일 수 있음
