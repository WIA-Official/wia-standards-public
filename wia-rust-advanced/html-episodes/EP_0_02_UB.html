UB란 무엇인가? What is UB?

⚠️ Undefined Behavior ⚠️ Undefined Behavior

컴파일러가 "이 코드는 절대 실행되지 않을 것"이라고 가정하는 상황.
실제로 실행되면 예측 불가능한 결과 발생.

💥 프로그램 크래시
💥 데이터 손상
💥 보안 취약점
💥 "정상 작동"처럼 보이다가 갑자기 실패

Situations where compiler assumes "this code will never execute."
If actually executed, unpredictable results occur.

💥 Program crash
💥 Data corruption
💥 Security vulnerabilities
💥 "Works fine" then suddenly fails

❌ UB 예제: 잘못된 코드 ❌ UB Example: Bad Code

unsafe {
// ❌ Null pointer dereference
let ptr = std::ptr::null::<i32>();
let val = *ptr; // UB!
// ❌ Use after free
let ptr = Box::into_raw(Box::new(42));
drop(Box::from_raw(ptr));
let val = *ptr; // UB!
// ❌ Uninitialized memory
let x: i32;
println!("{}", x); // UB!
}

⚠️ 이 코드들은 절대 작성하지 마세요! ⚠️ Never write code like this!

컴파일러가 이 코드를 완전히 최적화로 제거하거나, 예상치 못한 동작을 할 수 있습니다.

Compiler may optimize this code away completely or cause unexpected behavior.

✅ 안전한 코드 ✅ Safe Code

unsafe {
// ✅ Check before dereferencing
let ptr: *const i32 = get_ptr();
if !ptr.is_null() {
let val = *ptr; // Safe!
}
// ✅ Use safe abstractions
let boxed = Box::new(42);
let val = *boxed; // Safe!
// ✅ Always initialize
let x: i32 = 0;
println!("{}", x); // Safe!
}

✅ 안전 규칙 ✅ Safety Rules

1️⃣ 포인터는 사용 전 항상 null 체크
2️⃣ 가능하면 안전한 추상화 사용 (Box, Rc, Arc)
3️⃣ 변수는 사용 전 반드시 초기화

1️⃣ Always check pointers for null before use
2️⃣ Use safe abstractions when possible (Box, Rc, Arc)
3️⃣ Always initialize variables before use

컴파일러 최적화와 UB Compiler Optimizations and UB

❌ UB가 있는 코드 ❌ Code with UB

fn sum(ptr: *const i32) {
unsafe {
// Assumes non-null
*ptr + 1
}
}

컴파일러: "ptr은 null이 아니다"
→ null 체크 제거
→ 크래시 발생

Compiler: "ptr is not null"
→ Removes null checks
→ Crash occurs

✅ 안전한 코드 ✅ Safe Code

fn sum(val: &i32) {
// Reference guaranteed valid
*val + 1
}

컴파일러: "참조는 항상 유효"
→ 적극적으로 최적화
→ 안전하고 빠른 코드

Compiler: "Reference always valid"
→ Aggressive optimization
→ Safe and fast code

UB 방지 전략 Preventing UB

1️⃣ 안전한 Rust 우선 사용 1️⃣ Prefer Safe Rust

unsafe는 정말 필요할 때만 사용. 안전한 추상화로 감싸기.

Use unsafe only when absolutely necessary. Wrap in safe abstractions.

2️⃣ 포인터보다 참조 사용 2️⃣ Use References Over Pointers

*const T, *mut T 대신 &T, &mut T 사용. 컴파일러가 유효성 보장.

Use &T, &mut T instead of *const T, *mut T. Compiler guarantees validity.

3️⃣ Miri로 테스트 3️⃣ Test with Miri

unsafe 코드는 반드시 Miri로 검증. UB를 런타임에 탐지.

Always validate unsafe code with Miri. Detects UB at runtime.

4️⃣ 문서화 및 불변조건 명시 4️⃣ Document and State Invariants

unsafe 함수의 안전 조건을 주석으로 명확히 기술.

Clearly document safety conditions for unsafe functions.
