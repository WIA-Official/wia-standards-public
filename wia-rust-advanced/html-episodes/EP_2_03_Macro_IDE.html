스팬 보존 (Span Preservation) Span Preservation

// ❌ 나쁜 방법: 새 식별자 생성
let field_name = format_ident!("field");
// ✅ 좋은 방법: 원본 스팬 사용
let field_name = &field.ident;
let setter = format_ident!(
"set_{}",
field_name,
span = field_name.span() // 스팬 보존!
);

⚡ 핵심 개념 ⚡ Key Concept

스팬은 소스 코드의 위치 정보
에러 발생 시 정확한 위치 표시
IDE가 심볼을 추적할 수 있게 함

Span is source code location info
Shows exact error location
Enables IDE symbol tracking

타입 힌트 제공 Providing Type Hints

// ❌ 타입 정보 없음
quote! {
pub fn get(&self) { self.#field }
}
// ✅ 명시적 타입 지정
let field_ty = &field.ty;
quote! {
pub fn get(&self) -> &#field_ty {
&self.#field
}
}

💡 장점 💡 Benefits

IDE가 반환 타입 추론 가능
자동완성 및 타입 체크 지원
리팩토링 도구 동작

IDE can infer return types
Autocomplete and type checking
Refactoring tools work

명확한 에러 메시지 Clear Error Messages

let fields = match &input.data {
Data::Struct(DataStruct { fields, .. }) => fields,
_ => {
return Error::new(
input.span(), // 정확한 위치
"Builder can only be derived for structs"
).to_compile_error().into();
}
};
if let Fields::Unit = fields {
return Error::new(
input.span(),
"Cannot derive Builder for unit structs"
).to_compile_error().into();
}
