분기 예측 최적화 Branch Prediction

// 나쁜 예: 예측 불가능한 분기
if random_condition() { // 50% true/false
do_something();
}
// 좋은 예: likely/unlikely 힌트
#[cold]
fn unlikely_path() { /* ... */ }
if std::intrinsics::unlikely(error_condition()) {
unlikely_path();
}
💡 분기 예측 실패(branch misprediction)는 10-20 사이클 손실! 💡 Branch misprediction costs 10-20 cycles!

캐시 최적화 Cache Optimization

// 나쁜 예: 캐시 미스 많음 (row-major 순회)
for j in 0..cols {
for i in 0..rows {
matrix[i][j] += 1; // 비연속적 접근
}
}
// 좋은 예: 캐시 친화적 (column-major)
for i in 0..rows {
for j in 0..cols {
matrix[i][j] += 1; // 연속적 접근
}
}

인라인 최적화 Inlining Optimization

a + b
}
// 절대 인라인 금지
#[inline(never)]

// 큰 함수는 인라인하면 코드 크기 증가
}
// 기본 동작 (크로스 크레이트)
#[inline]
fn pub_helper() { }

루프 최적화 Loop Optimization

// 루프 언롤링
for i in (0..data.len()).step_by(4) {
result[i] = data[i] * 2;
result[i+1] = data[i+1] * 2;
result[i+2] = data[i+2] * 2;
result[i+3] = data[i+3] * 2;
}
// 프리페칭 힌트
unsafe {
std::intrinsics::prefetch_read_data(data.as_ptr().add(i + 64), 3);
}
💡 LLVM이 자동으로 언롤링하므로 수동 언롤링은 프로파일링 후에만! 💡 LLVM auto-unrolls, so manual unrolling only after profiling!

LTO & PGO LTO & PGO

# Cargo.toml
[profile.release]
lto = "fat" # Link-Time Optimization
codegen-units = 1 # 병렬 컴파일 비활성화
opt-level = 3 # 최대 최적화
# PGO (Profile-Guided Optimization)
$ RUSTFLAGS="-Cprofile-generate=/tmp/pgo" cargo build --release
$ ./target/release/myapp # 프로파일 수집
$ RUSTFLAGS="-Cprofile-use=/tmp/pgo" cargo build --release
