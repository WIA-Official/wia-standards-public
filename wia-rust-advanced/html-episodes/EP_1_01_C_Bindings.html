C 함수 선언 Declaring C Functions

C 코드

// math.h
double sqrt(double x);
// stdio.h
int printf(
const char *fmt,
...
);

Rust 바인딩

extern "C" {
fn sqrt(x: f64) -> f64;
fn printf(
fmt: *const i8,
...
) -> i32;
}

EP 1-01

타입 매핑 Type Mapping

// C types → Rust types
char → i8 // signed byte
int → c_int // platform-specific
size_t → usize // pointer-sized
void* → *mut c_void
const char* → *const i8
float → f32
double → f64

⚠️ 주의사항 ⚠️ Important

std::os::raw 모듈의 타입 사용 (c_int, c_long 등)
플랫폼마다 크기가 다를 수 있음

Use types from std::os::raw (c_int, c_long, etc.)
Sizes may vary across platforms

EP 1-01

실전 예제: C 함수 호출 Practical Example: Calling C Functions

use std::os::raw::c_int;
extern "C" {
fn abs(x: c_int) -> c_int;
}
fn main() {
let result = unsafe {
abs(-42)
};
println!("abs(-42) = {}", result);
}

💡 핵심 포인트 💡 Key Points

• extern "C" 블록에서 함수 선언
• unsafe 블록에서 호출 (안전성 보장 불가)
• 빌드 시스템이 C 라이브러리 자동 링크

• Declare functions in extern "C" block
• Call in unsafe block (no safety guarantees)
• Build system auto-links C library

링킹 설정 Linking Configuration

// build.rs
fn main() {
println!("cargo:rustc-link-lib=m"); // libm
println!("cargo:rustc-link-search=/usr/lib");
}
// 또는 #[link] 속성 사용
#[link(name = "m")]
extern "C" {
fn sqrt(x: f64) -> f64;
}

안전한 래퍼 작성 Writing Safe Wrappers

extern "C" {
fn sqrt(x: f64) -> f64;
}
// Safe wrapper
pub fn safe_sqrt(x: f64) -> Option<f64> {
if x < 0.0 {
return None;
}
Some(unsafe { sqrt(x) })
}

✅ 베스트 프랙티스 ✅ Best Practice

unsafe FFI 호출을 안전한 Rust 함수로 감싸기
입력 검증 및 에러 처리 추가

Wrap unsafe FFI calls in safe Rust functions
Add input validation and error handling
