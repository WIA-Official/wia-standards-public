크기와 정렬 Size and Alignment

use std::mem;
struct Data {
a: u8, // 1 byte
b: u32, // 4 bytes
c: u16, // 2 bytes
}
// Size: 12 bytes (padding added!)
// Alignment: 4 bytes (largest field)

Compiler adds padding between fields for performance.
Use #[repr(C)] to force C-compatible layout

소유권 메모리 모델 Ownership Memory Model

📦 Owned (T) 📦 Owned (T)

  • 유일한 소유자
  • Single owner
  • 스코프 끝에서 drop
  • Dropped at scope end
  • Move semantics

  • 읽기 전용 참조
  • Read-only reference
  • 여러 개 동시 가능
  • Multiple allowed
  • 라이프타임 검증
  • Lifetime verified

Drop과 RAII Drop and RAII

struct Resource {
name: String,
}
impl Drop for Resource {
fn drop(&mut self) {
println!("Dropping {}", self.name);
}
}
// 스코프 끝에서 자동 호출
// Automatically called at scope end

RAII (Resource Acquisition Is Initialization)

리소스 획득과 해제를 생성자/소멸자와 결합
메모리 누수 방지 및 리소스 안전성 보장

Ties resource acquisition/release to constructor/destructor
Prevents memory leaks and ensures resource safety

내부 가변성 패턴 Interior Mutability Pattern

Cell<T>

  • Copy 타입 전용
  • Copy types only
  • 런타임 체크 없음
  • No runtime checks
  • 단일 스레드
  • Single-threaded

RefCell<T>

  • 모든 타입 지원
  • All types supported
  • 런타임 빌림 검사
  • Runtime borrow checking
  • 단일 스레드
  • Single-threaded

Mutex<T>

  • 멀티 스레드
  • Multi-threaded
  • 락으로 동기화
  • Lock-based sync
  • 성능 오버헤드
  • Performance overhead

RwLock<T>

  • 읽기/쓰기 분리
  • Read/write separation
  • 동시 읽기 가능
  • Concurrent reads
  • 멀티 스레드
  • Multi-threaded

메모리 순서 보장 Memory Ordering Guarantees

Relaxed
순서 보장 없음 (가장 빠름) No ordering (Fastest)
Acquire
이후 연산이 앞으로 이동 금지 Prevents later ops from moving before
Release
이전 연산이 뒤로 이동 금지 Prevents earlier ops from moving after
AcqRel
Acquire + Release Acquire + Release
SeqCst
순차 일관성 (가장 안전, 느림) Sequential consistency (Safest, Slowest)

⚠️ 중요 ⚠️ Important

잘못된 메모리 순서 사용은 데이터 레이스 발생 가능
확실하지 않으면 SeqCst 사용 권장

Wrong memory ordering can cause data races
Use SeqCst if unsure
