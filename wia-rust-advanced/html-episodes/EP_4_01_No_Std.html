no_std란? What is no_std?

📚 std (표준) 📚 std (Standard)

  • OS 의존성
  • OS-dependent
  • 파일, 네트워크
  • Files, networking
  • 동적 할당 (heap)
  • Dynamic allocation
  • 스레딩
  • Threading

🔌 no_std 🔌 no_std

  • OS 없이 실행
  • Runs without OS
  • core만 사용
  • Only core lib
  • 임베디드 시스템
  • Embedded systems
  • 최소 바이너리
  • Minimal binary

no_std 설정 Setting Up no_std

// lib.rs 또는 main.rs 상단
#![no_std]
#![no_main] // main 함수 사용 안함
use core::panic!PanicInfo;
// 패닉 핸들러 필수
#[panic_handler]
fn panic(_info: &PanicInfo) -> ! {
loop {}
}

⚡ 핵심 개념 ⚡ Key Concept

no_std는 std 대신 core 사용
패닉 핸들러 직접 구현 필요
OS 없이 bare-metal 실행

no_std uses core instead of std
Custom panic handler required
Runs bare-metal without OS

core vs alloc vs std core vs alloc vs std

// core: 항상 사용 가능 (의존성 없음)
use core::{ptr, mem, slice};
use core::sync::atomic::AtomicU32;
// alloc: 힙 할당 필요 (글로벌 allocator 필요)
extern crate alloc;
use alloc::{vec, vec::Vec, string::String};
// std: OS 필요 (no_std에서 불가)
// use std::fs::File; // ❌
// use std::net::TcpStream; // ❌

📦 라이브러리 계층 📦 Library Hierarchy

core ⊂ alloc ⊂ std
core: 기본 타입, 트레이트
alloc: Box, Vec, String

core ⊂ alloc ⊂ std
core: Basic types, traits
alloc: Box, Vec, String

주요 no_std 크레이트 Common no_std Crates

heapless

  • 정적 크기 컬렉션
  • Static collections
  • Vec, String, Map
  • 힙 없이 사용
  • No heap needed

embedded-hal

  • 하드웨어 추상화
  • Hardware abstraction
  • GPIO, SPI, I2C
  • 크로스 플랫폼
  • Cross-platform

cortex-m

  • ARM Cortex-M
  • 인터럽트 관리
  • Interrupt mgmt
  • 어셈블리 매크로
  • Assembly macros

defmt

  • 임베디드 로깅
  • Embedded logging
  • 효율적 포맷팅
  • Efficient formatting
  • 최소 오버헤드
  • Minimal overhead

no_std 라이브러리 작성 Writing no_std Library

#![no_std]
// std와 no_std 모두 지원
#[cfg(not(feature = "std"))]
extern crate alloc;
#[cfg(feature = "std")]
use std::vec::Vec;
#[cfg(not(feature = "std"))]
use alloc::vec::Vec;
pub fn my_function() -> Vec<u8> {
vec![1, 2, 3]
}
