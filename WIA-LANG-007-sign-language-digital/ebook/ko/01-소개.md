# 1장: 수어 디지털화 표준 소개

## 수어의 디지털 혁명

전 세계 7천만 명 이상의 청각장애인이 300개 이상의 서로 다른 수어를 사용합니다. 각 수어는 고유한 문법, 어휘, 문화적 뉘앙스를 가진 완전한 자연 언어입니다. 그러나 디지털 시대에도 수어는 여전히 기술 접근성, 교육 자원, 의사소통 도구에서 심각하게 소외되어 있습니다.

### 수어란 무엇인가?

수어는 단순히 음성 언어를 손으로 번역한 것이 아닙니다. 수어는 다음을 통해 의미를 전달하는 완전한 시각-공간 언어입니다:

**손모양 (Handshape):**
- 손가락과 엄지의 구성
- 50개 이상의 고유한 기본 손모양
- 예시: 평평한 손, 주먹, 가리키는 손가락, "C" 모양

**손위치 (Location):**
- 신체 또는 수어 공간에서의 손의 위치
- 예시: 이마, 턱, 가슴, 중립 공간

**손동작 (Movement):**
- 손의 경로와 방향
- 예시: 직선, 원형, 반복, 진동
- 속도와 긴장도가 의미를 변경

**손바닥 방향 (Palm Orientation):**
- 손바닥이 향하는 방향
- 예시: 위, 아래, 수어자를 향해, 밖으로

**비수지 신호 (Non-Manual Signals):**
- 얼굴 표정: 문법 및 감정 의미
- 머리 움직임: 질문, 확인, 부정
- 몸통 이동: 참조 전환, 강조
- 눈 응시: 대명사 참조, 주제 마커

### 전 세계 수어의 규모

**전 세계 청각장애인 (2025년):**
- **7천만 명** 이상의 심각한 청각장애인
- **3억 6천만 명**의 중등도 이상 청각장애인 (세계 인구의 5%)
- 청각장애인의 **70%**가 수어를 주요 언어로 사용
- **300개 이상**의 다른 수어
- 청각장애인의 **90%**는 청각 부모를 둠 (접근성이 중요한 이유)

**주요 수어:**
- ASL (미국 수어): 50만-200만 명 사용자
- BSL (영국 수어): 15만 명 사용자
- JSL (일본 수어): 10만 명 사용자
- KSL (한국 수어): 25만 명 사용자
- CSL (중국 수어): 2천만 명 사용자
- ISL (국제 수어): 회의 및 국제 행사용

**한국의 청각장애인 커뮤니티:**
- **25만 명** 이상의 청각장애인
- **5만 명**의 KSL(한국수어) 능통 사용자
- 장애인 인권 법안 (2016): 한국수어를 공식 언어로 인정
- 국립국어원 산하 한국수어연구원 설립
- 한국농아인협회: 전국 16개 시도지부 운영
- 수어 통역사: 약 **1,500명** (수요 대비 부족)

### 디지털 시대의 수어 과제

수어 사용자는 디지털 세상에서 심각한 장벽에 직면합니다:

#### 교육 자원 부족

**문제:**
- 수어 교육 비디오: 표준화되지 않은 형식
- 검색 불가능: 수어 콘텐츠를 검색할 방법 없음
- 품질 일관성 없음: 조명, 배경, 카메라 각도
- 플랫폼 간 호환성 없음

**영향:**
- 청각장애 학생들은 제한된 교육 자료로 어려움
- 부모들은 자녀에게 수어를 가르칠 자원을 찾기 어려움
- 수어 학습자들은 일관성 없는 품질 경험
- 교사들은 매번 자료를 처음부터 만들어야 함

#### 통신 기술 접근성

**비디오 통화 제한사항:**
- 낮은 프레임률: 수어는 60fps 필요, 대부분의 앱은 30fps
- 압축 아티팩트: 손가락 세부사항 손실
- 조명 부족: 손모양 인식 어려움
- 지연: 실시간 대화 방해
- 모바일 데이터 비용: 고품질 비디오는 비쌈

**자막의 한계:**
- 자막은 수어를 완전히 포착하지 못함
- 청각장애인의 많은 수가 읽기 수준 낮음 (수어가 제1언어)
- 감정 및 뉘앙스 손실
- 문화적 맥락 누락

#### 자동화된 통역 부족

**현재 상황:**
- 음성-텍스트 ASR: 우수 (95%+ 정확도)
- 텍스트-음성 TTS: 우수 (자연스러운 소리)
- **수어 인식: 열악 (60-70% 정확도, 제한된 어휘)**
- **수어 생성: 매우 열악 (로봇적, 부정확)**

**결과:**
- 수어 사용자는 인간 통역사에 의존 (부족하고 비쌈)
- 비디오 콘텐츠 대부분 접근 불가능
- 고객 서비스 접근성 제한
- 의료, 법률 상황에서 오해 위험

#### 데이터 및 표준 부족

**문제:**
- 수어 데이터셋: 작고 균형 잡히지 않음
- 주석 형식: 표준화되지 않음
- 어휘 데이터베이스: 불완전하고 단편화됨
- 비디오 품질: 일관성 없고 문서화 부족

**영향:**
- AI 연구자들은 시스템을 훈련할 데이터 부족
- 서로 다른 프로젝트 간 호환성 없음
- 연구 진전 느림
- 상업적 응용 제한적

#### 4. 제스처 인식 AI

**입력 모달리티:**
- **RGB 비디오:** 표준 카메라
- **깊이 센서:** Kinect, RealSense, LiDAR
- **웨어러블:** 장갑, 팔 밴드 (IMU)
- **모바일:** 스마트폰 카메라
- **VR/AR:** 핸드 트래킹

**AI 아키텍처:**
```python
class SignLanguageRecognizer:
    def __init__(self):
        self.pose_estimator = MediaPipe()      # 손/몸 포즈
        self.hand_shape_classifier = CNN()     # 손모양 분류
        self.movement_analyzer = LSTM()        # 시간적 패턴
        self.facial_recognizer = FaceNet()     # 얼굴 표정
        self.sequence_model = Transformer()    # 전체 문장 이해

    def recognize(self, video_frames):
        # 1. 포즈 추출
        poses = [self.pose_estimator(frame) for frame in video_frames]

        # 2. 손모양 분류
        handshapes = [self.hand_shape_classifier(pose.hands)
                      for pose in poses]

        # 3. 움직임 분석
        movements = self.movement_analyzer(poses)

        # 4. 얼굴 표정
        expressions = [self.facial_recognizer(frame) for frame in video_frames]

        # 5. 시퀀스 이해
        result = self.sequence_model.decode(
            handshapes, movements, expressions
        )

        return result  # "안녕하세요, 만나서 반갑습니다"
```

**성능 목표:**
- **어휘 크기:** 10,000+ 수어
- **정확도:** 95%+ (고립된 수어)
- **정확도:** 85%+ (연속 수어)
- **지연:** <100ms (실시간 통역)
- **언어:** 다국어 지원 (KSL, ASL, BSL 등)

### 한국수어 (KSL) 특징

**어휘 특징:**
- 한국 문화 특정 수어 (한복, 김치, 절)

- 두 손 사용: ASL보다 양손 수어 많음

**평등한 접근:** 모든 청각장애인이 디지털 콘텐츠, 교육, 의사소통에 완전히 접근할 수 있도록 합니다.

**문화 보존:** 수어는 살아있는 언어이자 문화적 유산입니다. 디지털화는 미래 세대를 위해 보존합니다.

**인간 연결:** 기술은 청각과 청각장애 커뮤니티 간 격차를 해소하고, 이해와 포용을 촉진합니다.

**경제적 기회:** 접근 가능한 기술은 청각장애인에게 교육, 고용, 기업가 정신의 문을 엽니다.

**글로벌 영향:** 7천만 명의 청각장애인은 모든 국가, 모든 커뮤니티에 있습니다. 이 표준은 전 세계적으로 이익을 줍니다.

수어는 언어입니다. 수어 사용자는 시민입니다. 기술은 포용적이어야 합니다.
