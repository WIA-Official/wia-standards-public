🤚 Chapter 3: WIA Haptic Standard Overview

弘益人間 (Hongik Ingan)

The WIA Haptic Standard creates a universal tactile vocabulary that enables consistent, learnable communication for users with visual impairments across all devices and platforms.

3.1 Mission and Goals

3.1.1 Mission Statement

To establish an open, interoperable haptic feedback framework that provides a consistent tactile language for assistive technology, enabling users with visual impairments to navigate, communicate, and interact with digital devices through standardized touch-based feedback.

3.1.2 Key Goals

  1. Universality: Common haptic vocabulary across all platforms and devices
  2. Learnability: Patterns designed for quick learning and long-term retention
  3. Accessibility: Optimized for users who rely on tactile perception
  4. Interoperability: Works seamlessly across iOS, Android, Web, and wearables
  5. Efficiency: Battery-conscious patterns for mobile and wearable devices

3.3 Design Principles

3.3.1 Accessibility First

  • All patterns distinguishable without visual reference
  • Frequency ranges optimized for human tactile perception
  • Patterns designed for quick learning and retention
  • Support for varying sensitivity levels

3.3.2 Hardware Agnostic

  • Primitives abstract over device capabilities
  • Graceful degradation for limited hardware
  • Consistent experience across device types
  • Capability detection and adaptation

3.3.3 Energy Efficient

  • Optimized for mobile/wearable battery constraints
  • Short, effective patterns over long continuous vibrations
  • Power-aware pattern selection

3.5 Development Phases

Phase 1: Haptic Primitives

Fundamental building blocks: waveforms, ADSR envelopes, frequency bands, and intensity levels.

Phase 2: Pattern Library

Standardized patterns for navigation, alerts, confirmations, and Braille encoding.

Phase 3: Device Protocol

Cross-platform communication protocol mapping to iOS, Android, Web, and wearable APIs.

Phase 4: WIA Integration

Integration with WIA AAC, BCI, Eye Gaze, and navigation standards.

  2. Architecture: Four-layer design (Primitives → Patterns → Protocol → Integration)
