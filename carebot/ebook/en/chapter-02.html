2.1 System Architecture Overview

A WIA-CAREBOT compliant care robot consists of several interconnected subsystems that work together to perceive the environment, make decisions, and take actions. The architecture must be modular, allowing for customization and upgrades while maintaining system reliability and safety.

2.2 Mechanical Platform Design

The mechanical platform forms the physical foundation of the care robot. Design choices must balance mobility, stability, payload capacity, and safety in the challenging environment of human living spaces.

Mobile Platform Options

Platform Type Advantages Challenges Best For
Differential Drive Simple, reliable, zero-turn radius Limited speed stability Indoor navigation
Omnidirectional Holonomic motion, precise positioning Complex mechanics, higher cost Tight spaces, companion robots
Ackermann (Car-like) Stable at speed, natural motion Large turning radius Outdoor mobility assistance
Legged Handles stairs, uneven terrain Complex control, high cost Multi-floor operation

2.3 Sensor Systems

Care robots require comprehensive sensing capabilities to perceive their environment, detect humans, monitor user health, and ensure safe operation. Sensor selection must consider accuracy, reliability, environmental robustness, and cost.

Perception Sensors

  • RGB-D Cameras: Provide color images with depth information for person detection, gesture recognition, and spatial mapping
  • LiDAR: Accurate distance measurements for navigation, obstacle detection, and environment mapping
  • Microphone Arrays: Enable voice command recognition, speaker localization, and acoustic event detection
  • Thermal Cameras: Detect human presence and assess body temperature without contact
// Sensor Fusion Pipeline for Person Detection
class PersonDetectionPipeline {
    private rgbdCamera: RGBDCamera;
    private thermalCamera: ThermalCamera;
    private lidar: LiDAR;
    private fusionAlgorithm: SensorFusion;

    async detectPersons(): Promise {
        // Parallel sensor acquisition
        const [rgbdData, thermalData, lidarData] = await Promise.all([
            this.rgbdCamera.capture(),
            this.thermalCamera.capture(),
            this.lidar.scan()
        ]);

        // Individual detection pipelines
        const rgbdDetections = this.processRGBD(rgbdData);
        const thermalDetections = this.processThermal(thermalData);
        const lidarDetections = this.processLidar(lidarData);

        // Multi-sensor fusion for robust detection
        const fusedDetections = this.fusionAlgorithm.fuse({
            rgbd: rgbdDetections,
            thermal: thermalDetections,
            lidar: lidarDetections
        }, {
            confidenceThreshold: 0.85,
            spatialToleranceMeters: 0.3,
            requireMultipleSources: true  // Higher reliability
        });

        return fusedDetections;
    }

    private processRGBD(data: RGBDFrame): Detection[] {
        // Deep learning based person detection
        const detections = this.personDetector.detect(data.rgb);

        // Add depth information
        return detections.map(det => ({
            ...det,
            distance: this.calculateDistance(det.boundingBox, data.depth),
            pose3D: this.estimatePose3D(det, data.depth)
        }));
    }
}

2.4 Actuator Systems

Actuators convert electronic commands into physical motion. For care robots, actuator selection emphasizes safety, precision, and appropriate force limits to prevent injury during human-robot interaction.

Series Elastic Actuators

Series Elastic Actuators (SEAs) incorporate a compliant element between the motor and output, providing inherent safety through force limiting and enabling accurate force control. This makes them ideal for care robot arms and manipulation systems.

// Series Elastic Actuator Controller
class SEAController {
    private motor: BrushlessMotor;
    private encoder: AbsoluteEncoder;
    private springConstant: number;  // N/m
    private maxForce: number;        // Safety limit

    constructor(config: SEAConfig) {
        this.motor = new BrushlessMotor(config.motorId);
        this.encoder = new AbsoluteEncoder(config.encoderId);
        this.springConstant = config.springConstant;
        this.maxForce = config.safetyForceLimit;
    }

    // Impedance control for safe human interaction
    async setImpedance(
        desiredPosition: number,
        stiffness: number,
        damping: number
    ): Promise {
        const currentPosition = await this.encoder.read();
        const currentVelocity = await this.encoder.readVelocity();

        // Calculate desired force with impedance model
        const positionError = desiredPosition - currentPosition;
        const desiredForce = stiffness * positionError - damping * currentVelocity;

        // Apply safety limit
        const safeForce = Math.min(
            Math.max(desiredForce, -this.maxForce),
            this.maxForce
        );

        // Convert force to motor command through spring model
        const springDeflection = safeForce / this.springConstant;
        const motorPosition = desiredPosition + springDeflection;

        await this.motor.setPosition(motorPosition);
    }

    // Read actual interaction force
    async readForce(): Promise {
        const motorPos = await this.motor.getPosition();
        const outputPos = await this.encoder.read();
        const springDeflection = motorPos - outputPos;
        return springDeflection * this.springConstant;
    }
}

Path Planning and Obstacle Avoidance

Care robots must navigate smoothly and predictably, avoiding obstacles while respecting social norms like maintaining appropriate distances from humans and not blocking doorways.

// Social-Aware Path Planner for Care Robots
class SocialPathPlanner {
    private costmap: Costmap2D;
    private humanTracker: HumanTracker;

    async planPath(
        start: Pose2D,
        goal: Pose2D,
        socialConstraints: SocialConstraints
    ): Promise {
        // Update costmap with dynamic obstacles
        const humans = await this.humanTracker.getTrackedHumans();

        // Add social cost layers
        for (const human of humans) {
            // Personal space bubble (higher cost)
            this.costmap.addGaussianCost(
                human.position,
                socialConstraints.personalSpaceRadius,  // ~1.2m
                socialConstraints.personalSpaceCost
            );

            // Movement prediction zone
            if (human.velocity.magnitude > 0.1) {
                const predictedPath = this.predictHumanPath(human, 3.0);
                this.costmap.addPathCost(
                    predictedPath,
                    socialConstraints.crossingCost
                );
            }
        }

        // Add furniture and doorway costs
        this.addEnvironmentalSocialCosts();

        // Plan with social-aware A* or RRT*
        const path = this.hybridAStarPlanner.plan(
            start,
            goal,
            this.costmap,
            {
                allowReverse: true,
                smoothness: 0.8,
                maxCurvature: 2.0  // Comfortable for following humans
            }
        );

        return this.smoothPath(path);
    }
}

2.6 Computing Architecture

Care robots require substantial computational resources for perception, planning, and AI inference while maintaining low latency for safety-critical operations. A distributed computing architecture typically provides the best balance of performance and reliability.

Computing Requirements:
- Real-time control: ARM/x86 MCU with RTOS, sub-millisecond latency
- Perception processing: GPU (NVIDIA Jetson), 30+ FPS object detection
- AI inference: NPU/TPU for voice recognition and LLM responses
- Cloud connectivity: For updates, analytics, and remote monitoring
