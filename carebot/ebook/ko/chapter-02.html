2.1 시스템 아키텍처 개요

WIA-CAREBOT 준수 케어 로봇은 환경을 인식하고, 결정을 내리고, 행동을 취하기 위해 함께 작동하는 여러 상호 연결된 서브시스템으로 구성됩니다. 아키텍처는 시스템 신뢰성과 안전을 유지하면서 사용자 정의 및 업그레이드를 허용하는 모듈식이어야 합니다.

// WIA-CAREBOT 시스템 아키텍처 계층
interface CareRobotArchitecture {
    hardwareLayer: {
        mechanicalPlatform: MobilePlatform | StationaryPlatform;
        powerSystem: BatterySystem;
        computeUnits: ProcessingHardware[];
        peripherals: Peripheral[];
    };

    sensorLayer: {
        perceptionSensors: PerceptionSensor[];
        proximitySensors: SafetySensor[];
        internalSensors: DiagnosticSensor[];
    };

    controlLayer: {
        lowLevelControl: MotorController[];
        trajectoryPlanning: MotionPlanner;
        safetyMonitor: SafetySystem;
    };

    cognitionLayer: {
        perception: PerceptionPipeline;
        planning: TaskPlanner;
        decision: DecisionEngine;
        learning: AdaptationModule;
    };

    interactionLayer: {
        speechInterface: SpeechSystem;
        displayInterface: VisualDisplay;
        gestureRecognition: GestureSystem;
        hapticFeedback: HapticSystem;
    };
}

기계적 플랫폼은 케어 로봇의 물리적 기반을 형성합니다. 설계 선택은 인간 생활 공간의 도전적인 환경에서 이동성, 안정성, 적재 용량 및 안전성의 균형을 맞추어야 합니다.

플랫폼 유형 장점 과제 최적 용도

2.3 센서 시스템

케어 로봇은 환경을 인식하고, 인간을 감지하고, 사용자 건강을 모니터링하고, 안전한 운영을 보장하기 위한 포괄적인 감지 기능이 필요합니다. 센서 선택 시 정확도, 신뢰성, 환경 견고성 및 비용을 고려해야 합니다.

인식 센서

  • RGB-D 카메라: 사람 감지, 제스처 인식 및 공간 매핑을 위한 깊이 정보가 포함된 컬러 이미지 제공
  • 라이다: 내비게이션, 장애물 감지 및 환경 매핑을 위한 정확한 거리 측정
  • 마이크 어레이: 음성 명령 인식, 화자 위치 파악 및 음향 이벤트 감지 가능
  • 열화상 카메라: 비접촉 방식으로 인간 존재 감지 및 체온 평가
// 사람 감지를 위한 센서 융합 파이프라인
class PersonDetectionPipeline {
    private rgbdCamera: RGBDCamera;
    private thermalCamera: ThermalCamera;
    private lidar: LiDAR;
    private fusionAlgorithm: SensorFusion;

    async detectPersons(): Promise<DetectedPerson[]> {
        // 병렬 센서 획득
        const [rgbdData, thermalData, lidarData] = await Promise.all([
            this.rgbdCamera.capture(),
            this.thermalCamera.capture(),
            this.lidar.scan()
        ]);

        // 개별 감지 파이프라인
        const rgbdDetections = this.processRGBD(rgbdData);
        const thermalDetections = this.processThermal(thermalData);
        const lidarDetections = this.processLidar(lidarData);

        // 강건한 감지를 위한 다중 센서 융합
        const fusedDetections = this.fusionAlgorithm.fuse({
            rgbd: rgbdDetections,
            thermal: thermalDetections,
            lidar: lidarDetections
        }, {
            confidenceThreshold: 0.85,
            spatialToleranceMeters: 0.3,
            requireMultipleSources: true  // 더 높은 신뢰성
        });

        return fusedDetections;
    }
}

2.4 액추에이터 시스템

액추에이터는 전자 명령을 물리적 움직임으로 변환합니다. 케어 로봇의 경우 액추에이터 선택은 인간-로봇 상호작용 중 부상을 방지하기 위한 안전, 정밀도 및 적절한 힘 제한을 강조합니다.

직렬 탄성 액추에이터

직렬 탄성 액추에이터(SEA)는 모터와 출력 사이에 탄성 요소를 통합하여 힘 제한을 통한 고유한 안전성과 정확한 힘 제어를 가능하게 합니다. 이는 케어 로봇 팔과 조작 시스템에 이상적입니다.

// 직렬 탄성 액추에이터 컨트롤러
class SEAController {
    private motor: BrushlessMotor;
    private encoder: AbsoluteEncoder;
    private springConstant: number;  // N/m
    private maxForce: number;        // 안전 한계

    // 안전한 인간 상호작용을 위한 임피던스 제어
    async setImpedance(
        desiredPosition: number,
        stiffness: number,
        damping: number
    ): Promise<void> {
        const currentPosition = await this.encoder.read();
        const currentVelocity = await this.encoder.readVelocity();

        // 임피던스 모델로 원하는 힘 계산
        const positionError = desiredPosition - currentPosition;
        const desiredForce = stiffness * positionError - damping * currentVelocity;

        // 안전 한계 적용
        const safeForce = Math.min(
            Math.max(desiredForce, -this.maxForce),
            this.maxForce
        );

        // 스프링 모델을 통해 힘을 모터 명령으로 변환
        const springDeflection = safeForce / this.springConstant;
        const motorPosition = desiredPosition + springDeflection;

        await this.motor.setPosition(motorPosition);
    }
}

2.6 컴퓨팅 아키텍처

케어 로봇은 안전에 중요한 작업에 대한 낮은 지연 시간을 유지하면서 인식, 계획 및 AI 추론을 위한 상당한 계산 리소스가 필요합니다. 분산 컴퓨팅 아키텍처는 일반적으로 성능과 신뢰성의 최상의 균형을 제공합니다.

컴퓨팅 요구사항:
- 실시간 제어: RTOS가 있는 ARM/x86 MCU, 밀리초 미만 지연
- 인식 처리: GPU(NVIDIA Jetson), 30+ FPS 객체 감지
- AI 추론: 음성 인식 및 LLM 응답을 위한 NPU/TPU
- 클라우드 연결: 업데이트, 분석 및 원격 모니터링용

2.7 전원 시스템 설계

신뢰할 수 있는 전원은 지속적으로 작동하거나 필요 시 사용 가능해야 하는 케어 로봇에 중요합니다. 전원 시스템은 용량, 충전 편의성 및 안전의 균형을 맞추어야 합니다.

요구사항 사양 근거
작동 시간 8시간 이상 충전 없이 하루 종일 운영
충전 시간 4시간 미만 야간 재충전 가능
자동 도킹 필수 사용자 개입 없이 자율 재충전
저전력 경고 30분 충전 스테이션 도달 시간
비상 예비 10분 중요 작업 완료 및 안전 종료
챕터 요약: 케어 로봇 아키텍처는 가정 환경에 최적화된 기계적 플랫폼, 인식 및 안전을 위한 포괄적인 센서 시스템, 안전한 인간 상호작용을 위해 설계된 액추에이터, 지능형 내비게이션, 분산 컴퓨팅 및 신뢰성 있는 전원 시스템을 포함합니다. 이러한 구성요소는 안전하고 유능하며 사용자 친화적인 케어 로봇을 만들기 위해 신중하게 통합되어야 합니다. 다음 챕터에서는 이러한 시스템이 특정 케어 기능을 제공하기 위해 어떻게 결합되는지 탐구합니다.
