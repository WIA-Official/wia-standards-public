1.1 서비스 로보틱스의 발전

서비스 로보틱스는 20세기 후반 산업 자동화에서 출발했지만, 케어 로보틱스는 근본적으로 다른 패러다임을 나타냅니다. 산업용 로봇이 예측 가능한 조건의 통제된 환경에서 작동하는 반면, 케어 로봇은 안전, 편안함 및 사회적 수용이 가장 중요한 관심사인 역동적인 인간 공간에서 기능해야 합니다.

공장 현장에서 거실로의 여정에는 감지, 의사결정 및 물리적 안전 시스템의 혁명적인 발전이 필요했습니다. RI-MAN(2006)과 RIBA(2009)와 같은 초기 케어 로봇은 로봇이 환자 이송과 같은 물리적 작업을 위해 인간과 안전하게 상호작용할 수 있음을 보여주며 오늘날의 정교한 케어 시스템의 기반을 마련했습니다.

1.2 인간-로봇 상호작용 원칙

효과적인 케어 로봇은 인간 심리학 및 사회적 상호작용의 복잡한 환경을 탐색해야 합니다. 인간-로봇 상호작용(HRI) 분야는 케어 로봇 설계와 행동을 안내하는 기본 원칙을 제공합니다.

케어 환경에서의 불쾌한 골짜기

불쾌한 골짜기 현상은 거의 인간처럼 보이지만 완전히 인간 같지 않은 로봇이 불편함을 유발하는 것으로, 케어 환경에서 특히 중요합니다. 연구에 따르면 노인 사용자는 종종 휴머노이드 로봇보다 동물 같은 디자인이나 추상적인 기하학적 형태와 같이 명확히 비인간적이지만 친근한 외관을 가진 로봇을 선호합니다.

interface RobotAppearanceGuidelines {
    // 불쾌한 골짜기 회피
    humanoidFeatures: "최소" | "양식화" | "없음";

    // 케어 로봇에 선호되는 특성
    design: {
        faceType: "단순_표현적" | "만화" | "추상적";
        eyeSize: "큰";  // 친근함 증가
        colorPalette: "따뜻한" | "부드러운";  // 임상적 흰색 회피
        surfaceTexture: "부드러운" | "따뜻한_촉감";
    };

    // 크기 고려사항
    physicalPresence: {
        heightWhenSeated: "사용자_눈높이_이하";
        bulkiness: "위협적이지_않은";
        footprint: "가정_환경에서_관리_가능";
    };
}

신뢰 형성 및 유지

신뢰는 성공적인 케어 로봇 배포의 기반입니다. 사용자는 로봇의 역량(작업 수행 능력), 신뢰성(성능의 일관성), 선의(로봇이 자신의 이익을 위해 행동한다는 것)를 신뢰해야 합니다. 신뢰는 긍정적인 상호작용을 통해 발전하며 실패로 손상될 수 있어 오류 복구가 중요한 설계 고려사항입니다.

신뢰 요소 구축 메커니즘 손상 방지
역량 성공적인 작업 완료, 명확한 기능 전달 과대 약속 회피, 우아한 실패 처리
신뢰성 일관된 행동, 예측 가능한 응답 정기 유지보수, 투명한 상태 업데이트
선의 개인화된 케어, 공감적 응답 프라이버시 보호, 사용자 자율성 보존
투명성 설명 가능한 행동, 명확한 결정 근거 감사 로깅, 보호자 가시성

1.3 케어 로봇 카테고리

WIA-CAREBOT은 각각 고유한 기능 요구사항과 설계 고려사항을 가진 5가지 주요 케어 로봇 카테고리를 정의합니다:

카테고리 A: 신체 보조 로봇

이 로봇들은 이동성, 이송 및 일상 활동에 대한 직접적인 신체 지원을 제공합니다. 인체를 안전하게 지지하거나 이동시킬 수 있는 견고한 기계 시스템과 부상을 방지하기 위한 정교한 힘 감지가 필요합니다.

// ROS 2 신체 보조 서비스 정의
# mobility_assist.srv
---
# 요청
string assistance_type  # "walking", "transfer", "standing"
geometry_msgs/Pose user_position
float32 user_weight_estimate
float32 confidence_level

# 응답
bool can_assist
string[] required_preparations
float32 estimated_duration
string safety_advisory

카테고리 B: 동반자 로봇

동반자 로봇은 사용자의 정서적, 사회적 요구를 해결하며 대화, 오락 및 인지 자극을 제공합니다. 물리적 조작보다 표현 능력, 자연어 이해 및 감성 지능을 강조합니다.

설계 원칙: 동반자 로봇은 사회적 연결을 대체하기보다 촉진해야 합니다. 목표는 고립을 줄이고 인간의 동반자가 없는 시간에 지원을 제공하면서 사용자가 인간 관계를 유지하도록 장려하는 것입니다.

카테고리 C: 건강 모니터링 로봇

건강 모니터링 로봇은 지속적으로 또는 주기적으로 건강 데이터를 수집하고, 생체 신호를 추적하며, 건강 문제를 나타낼 수 있는 이상을 감지합니다. 의료 시스템과 통합하여 개입이 필요할 때 보호자에게 알립니다.

카테고리 D: 재활 로봇

재활 로봇은 치료 운동과 부상이나 질병으로부터의 회복을 지원합니다. 정밀하고 반복 가능한 움직임을 제공하고, 진행 상황을 측정하며, 개별 환자 능력에 적응합니다.

카테고리 E: 서비스 로봇

서비스 로봇은 식사 배달, 청소, 물건 가져오기 및 환경 제어를 포함하여 독립 생활을 지원하는 가사 작업을 수행합니다. 내비게이션, 조작 및 작업 계획 기능을 강조합니다.

1.4 심리적 고려사항

케어 로봇 사용자의 심리적 요구와 우려를 이해하는 것은 성공적인 배포에 필수적입니다. 주요 고려사항은 다음과 같습니다:

자율성 보존

사용자는 로봇이 자신의 삶을 통제하기보다 지원한다고 느껴야 합니다. 로봇은 필요할 때 도움을 제공하면서 사용자가 도움을 거부하거나 원할 때 독립적으로 작업을 수행할 권리를 존중해야 합니다.

// 자율성 보존 상호작용 패턴
class AutonomyPreservingAssistant {
    async offerAssistance(task: string, userContext: UserContext): Promise<void> {
        // 사용자가 일반적으로 이 작업에 대한 도움을 수락하는지 확인
        const assistanceHistory = await this.getAssistanceHistory(task);

        if (assistanceHistory.declineRate > 0.7) {
            // 사용자가 보통 거부 - 수동적 제안
            this.subtleReminder(task);
        } else {
            // 적극적이지만 비침습적 제안
            const offer = await this.generateOffer(task, userContext);
            const response = await this.presentChoice(offer, {
                timeout: 30000,
                defaultAction: "no_action",  // 절대 강제하지 않음
                respectDecline: true
            });

            if (response.accepted) {
                await this.provideAssistance(task, response.preferences);
            } else {
                this.acknowledgeDecision("알겠습니다. 마음이 바뀌시면 알려주세요.");
                this.recordPreference(task, "declined", userContext);
            }
        }
    }
}

1.5 수용성 요인

연구에서 노인 사용자와 그 가족의 케어 로봇 수용을 예측하는 주요 요인을 확인했습니다:

요인 영향 설계 의미
인지된 유용성 높음 실제 일상 과제 해결에 집중
사용 용이성 높음 단순한 인터페이스, 음성 제어, 최소 학습 곡선
사회적 영향 중간 도입 시 가족과 보호자 포함
불안 부정적 점진적 도입, 명확한 안전 보장
사전 경험 긍정적 스마트폰 및 음성 비서에 대한 친숙함 활용
