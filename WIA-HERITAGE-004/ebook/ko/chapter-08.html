08.1 구현 개요와 접근 전략

WIA-HERITAGE-004 표준을 실제 조직에 도입하는 것은 기술적 역량뿐 아니라 문화적 민감성, 공동체 협력, 장기적 지속가능성을 요구하는 복합적 과제입니다. 이 장에서는 소규모 지역 박물관부터 UNESCO 지정 유산 관리 기관까지, 다양한 규모와 역량을 가진 조직이 단계적으로 표준을 도입할 수 있는 실무 가이드를 제공합니다.

구현 전략의 핵심은 "Phase by Phase, Community First" 원칙입니다. Phase 1(데이터 형식)부터 시작해 조직의 역량과 공동체 준비도에 따라 점진적으로 확장하며, 모든 단계에서 문화 보유자 및 공동체 구성원의 동의와 참여를 최우선으로 합니다.

  • 공동체 협약 (Community Agreement): 문화 보유자와 사전 동의 체결
  • 단계별 검증 (Phase Validation): 각 Phase 완료 후 파일럿 테스트
  • 역량 구축 (Capacity Building): 내부 인력 교육 및 외부 전문가 협력
  • 예산 확보 (Budget Allocation): 초기 투자 + 3년 운영비 사전 계획

구현 로드맵 타임라인

표준적인 중형 기관(예: 광역시 문화재단)의 Phase 1-3 구현 타임라인:

  • 0-3개월: 현황 진단, 공동체 협의, 파일럿 프로젝트 설계
  • 3-9개월: Phase 1 구현 (데이터 형식 표준화, 메타데이터 스키마 적용)
  • 9-15개월: Phase 2 구현 (RESTful API 개발, 업로드 포털 구축)
  • 15-24개월: Phase 3 구현 (실시간 스트리밍, 분산 아카이빙)
  • 24-30개월: 통합 테스트, WIA 사전인증 신청
  • 30-36개월: WIA 공식 인증 획득, 운영 전환

08.2 Phase별 구현 상세 가이드

Phase 1: 데이터 형식 표준화 구현

목표: 기존 아카이브를 WIA-HERITAGE-004 데이터 형식 사양에 맞춰 재구성하고, 향후 수집 자산의 표준화 파이프라인 구축.

핵심 작업:

  1. 현황 조사: 기존 디지털 자산 목록 작성 (포맷, 해상도, 메타데이터 현황)
  2. 변환 파이프라인: FFmpeg/HandBrake 기반 비디오 트랜스코딩 (H.264 → ProRes 4:2:2 또는 H.265 10bit)
  3. 오디오 리마스터링: 기존 MP3/AAC를 WAV 96kHz/24bit 또는 FLAC로 업샘플링
  4. 메타데이터 마이그레이션: Dublin Core/MARC21에서 WIA JSON-LD 스키마로 변환
  5. 스토리지 재설계: 파일 명명 규칙, 폴더 구조 표준화 (예: /pansori/chunhyang/2024/04/15/master.mov)

필수 기술 스택:

  • 비디오: FFmpeg 6.0+, HandBrake CLI, DaVinci Resolve (선택)
  • 오디오: Audacity, Adobe Audition (또는 오픈소스 Ardour)
  • 메타데이터: Python + jsonld 라이브러리, Apache Jena (RDF 변환)
  • 스토리지: NAS (QNAP/Synology) 또는 클라우드 (AWS S3 + Glacier)

Phase 2: API 인터페이스 개발

목표: 외부 연구자, 교육 기관, 공동체 구성원이 아카이브에 접근할 수 있는 RESTful API 및 GraphQL 엔드포인트 구축.

핵심 작업:

  1. API 설계: OpenAPI 3.0 스펙 작성, 엔드포인트 정의 (GET /performances, POST /upload 등)
  2. 인증 시스템: OAuth 2.0 + JWT 구현, 5단계 접근 제어 (Public/Researcher/Community/Master/Keeper)
  3. 스트리밍 서버: HLS/DASH 적응형 비트레이트 스트리밍 구성
  4. 업로드 포털: 공동체용 100GB 대용량 파일 업로드 인터페이스 (TUS 프로토콜)
  5. WebHook 알림: 신규 업로드 시 Slack/Discord 알림 연동

필수 기술 스택:

  • 백엔드: Node.js (Express) 또는 Python (FastAPI/Django)
  • API 게이트웨이: Kong, AWS API Gateway
  • 인증: Auth0, Keycloak (오픈소스)
  • 스트리밍: Wowza Streaming Engine, Nginx-RTMP
  • 데이터베이스: PostgreSQL (메타데이터) + Elasticsearch (검색)

08.4 공동체 협의 및 동의 프로세스

WIA-HERITAGE-004의 핵심 가치는 문화 보유자의 주권 존중입니다. 모든 구현 단계에서 공동체의 자유로운 사전 동의(FPIC, Free Prior and Informed Consent)를 확보해야 합니다.

FPIC 실행 단계

  1. 정보 제공 (Information): 표준 도입 목적, 방법, 기대 효과를 공동체 언어로 설명 (브로슈어, 동영상)
  2. 협의 (Consultation): 3회 이상 공동체 회의 개최, 우려사항 청취 및 반영
  3. 동의 확보 (Consent): 공동체 대표자와 문서 동의서 체결 (최소 70% 찬성)
  4. 지속적 참여 (Ongoing Engagement): 분기별 보고회, 연 1회 공동체 감사

파주 두레농악 보존회는 2023년 WIA-HERITAGE-004 Phase 1-2 도입 시, 3개월간 5회의 마을 회의를 거쳐 다음 합의를 도출했습니다:

  • 상쇠·부쇠·장구잡이의 개인 기량 영상: Community 등급 (회원만 접근)
  • 마을 축제 단체 공연 영상: Public 등급 (전체 공개, CC BY-NC-SA 라이선스)
  • 제의용 농악 녹음: Keeper 등급 (보존회장 승인 필수)

Phase 2 품질 기준

  • 인증 보안: OWASP Top 10 취약점 테스트 통과, JWT 만료 시간 ≤ 1시간
  • 스트리밍 품질: HLS 세그먼트 길이 6초, 3단계 이상 ABR (480p/720p/1080p)
  • 동시 접속: 최소 500명 동시 스트리밍 시 버퍼링 발생률 < 5%
  • 업로드 안정성: 100GB 파일 업로드 성공률 > 95% (재시도 포함)

Phase 3 품질 기준

  • WebRTC 레이턴시: 송출자-시청자 간 지연 < 500ms (국내), < 1500ms (해외)
  • 타임코드 정확도: PTP 동기화 오차 ≤ ±1 microsecond
  • 블록체인 무결성: 100개 샘플 파일의 IPFS 해시 일치율 100%
  • QoS 보장: 네트워크 혼잡 시에도 비디오 스트림 패킷 손실률 < 0.1%

자동화 테스트 스크립트 예시

#!/bin/bash
# WIA-HERITAGE-004 Phase 1 품질 검증 스크립트

echo "비디오 코덱 검증 중..."
for file in /archive/master/*.mov; do
  codec=$(mediainfo --Inform="Video;%Format%" "$file")
  if [[ "$codec" != "ProRes" ]]; then
    echo "❌ FAIL: $file - 코덱 $codec (ProRes 필요)"
    exit 1
  fi
done

echo "JSON-LD 스키마 검증 중..."
for meta in /archive/metadata/*.jsonld; do
  jsonld validate "$meta" --context=https://wia-standards.org/heritage/v1/context.jsonld
  if [[ $? -ne 0 ]]; then
    echo "❌ FAIL: $meta - 스키마 오류"
    exit 1
  fi
done

echo "✅ 모든 Phase 1 품질 기준 통과"

08.6 WIA 공식 인증 프로세스

인증 절차

08.9 복습 문제

  1. 구현 성숙도 모델의 5단계를 순서대로 나열하고, Level 3 "정의형(Defined)"에 해당하는 조직의 특징을 설명하세요.

    힌트: Level 3은 Phase 1+2를 구현한 상태이며, API를 제공하기 시작합니다.

  2. 중형 기관(연 예산 ₩100M-1B)이 Phase 1-2를 3년간 구현할 때 예상되는 총 비용 범위를 제시하고, 이 중 Phase 1과 Phase 2의 예산 비율을 계산하세요.

    힌트: Phase 1 약 ₩30M, Phase 2 약 ₩95M, 총 ₩125M 수준.

  3. FPIC(Free Prior and Informed Consent)의 4단계를 설명하고, 파주시 전통농악 공동체 사례에서 "제의용 농악 녹음"에 적용된 접근 제어 등급과 그 이유를 논하세요.

    힌트: 정보 제공 → 협의 → 동의 확보 → 지속적 참여. 제의용 녹음은 Keeper 등급으로 보존회장 승인 필수.

  4. Phase 2 품질 기준 중 "API 응답 시간"의 p95/p99 지표를 설명하고, 각각의 합격 기준을 제시하세요. 또한 HLS 스트리밍의 ABR(Adaptive Bitrate) 최소 단계 수를 밝히세요.

    힌트: p95 ≤ 200ms, p99 ≤ 500ms. ABR 최소 3단계 (480p/720p/1080p).

  5. WIA-HERITAGE-004 인증의 4가지 등급(Bronze/Silver/Gold/Platinum)을 구현 범위에 따라 정의하고, 국립국악원이 획득한 Gold 등급의 심사 과정에서 발견된 문제점과 해결 방법을 설명하세요.

    힌트: Bronze(Phase 1), Silver(Phase 1-2), Gold(Phase 1-3), Platinum(Phase 1-4). 국악원은 JSON-LD 국악 용어 번역 오류를 2주 내 수정.

  6. Phase 1-2 구현 기관의 연간 운영 비용(약 ₩106M)을 5가지 지속가능성 확보 전략으로 어떻게 충당할 수 있는지 구체적인 금액과 함께 제시하세요. 또한 예산 30% 삭감 시나리오에서 최우선 절감 항목을 선택하고 이유를 설명하세요.

    힌트: 정부 지원금 ₩50M + API 유료화 ₩20M + 교육 ₩10M + 기부금 ₩60M 등. 위기 시 Phase 3 중단, 인력 축소 우선.
