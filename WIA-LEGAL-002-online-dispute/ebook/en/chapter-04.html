Chapter 4: API Implementation

4.1 Introduction to Implementation

Implementing WIA-LEGAL-002 APIs requires careful attention to authentication, security, error handling, and client SDK development. This chapter provides practical guidance for building production-ready integrations that leverage the standard's full capabilities while maintaining security and reliability.

Whether developing a new ODR platform from scratch, integrating an existing dispute resolution system with the standard, or building client applications that consume ODR APIs, understanding implementation patterns, best practices, and common pitfalls ensures successful deployment.

4.2 Authentication and Authorization

4.2.1 OAuth 2.0 Implementation

The WIA-LEGAL-002 standard mandates OAuth 2.0 for API authentication, supporting multiple grant types appropriate for different integration scenarios. OAuth provides secure, token-based authentication without requiring clients to handle user credentials directly.

4.4 Error Handling Strategies

Robust error handling differentiates production-ready integrations from prototype code. Clients must gracefully handle network failures, API errors, rate limiting, and transient issues.

4.4.1 Retry Logic with Exponential Backoff

class ODRClient {
  private async retryWithBackoff(
    operation: () => Promise,
    maxRetries: number = 3,
    baseDelay: number = 1000
  ): Promise {
    for (let attempt = 0; attempt <= maxRetries; attempt++) {
      try {
        return await operation();
      } catch (error) {
        // Don't retry on client errors (4xx)
        if (error.status >= 400 && error.status < 500 && error.status !== 429) {
          throw error;
        }

        // Last attempt - throw error
        if (attempt === maxRetries) {
          throw error;
        }

        // Calculate delay with exponential backoff and jitter
        const delay = baseDelay * Math.pow(2, attempt) +
                     Math.random() * 1000;

        console.log(`Retry ${attempt + 1}/${maxRetries} after ${delay}ms`);
        await this.sleep(delay);
      }
    }
  }

  private sleep(ms: number): Promise {
    return new Promise(resolve => setTimeout(resolve, ms));
  }

  async getDispute(disputeId: string): Promise {
    return this.retryWithBackoff(() =>
      this.http.get(`/api/v1/disputes/${disputeId}`)
    );
  }
}

4.4.2 Rate Limit Handling

class RateLimitHandler {
  private async handleRateLimit(response: Response): Promise {
    const retryAfter = response.headers.get('Retry-After');

    if (retryAfter) {
      const delay = parseInt(retryAfter) * 1000; // Convert to milliseconds
      console.log(`Rate limited. Waiting ${delay}ms before retry`);
      await this.sleep(delay);
    } else {
      // Default to 60 seconds if Retry-After not provided
      await this.sleep(60000);
    }
  }

  async makeRequest(url: string, options: RequestOptions): Promise {
    let response = await fetch(url, options);

    // Handle 429 Too Many Requests
    if (response.status === 429) {
      await this.handleRateLimit(response);
      // Retry the request
      response = await fetch(url, options);
    }

    return response;
  }
}

4.6 Performance Optimization

Optimizing API integrations improves user experience and reduces infrastructure costs through efficient resource usage.

4.6.1 Caching Strategies

class ODRCache {
  private cache: Map = new Map();
  private ttl: number = 300000; // 5 minutes

  async get(key: string): Promise {
    const entry = this.cache.get(key);

    if (!entry) return null;

    // Check if expired
    if (Date.now() > entry.expiresAt) {
      this.cache.delete(key);
      return null;
    }

    return entry.value;
  }

  set(key: string, value: any, ttl?: number): void {
    this.cache.set(key, {
      value,
      expiresAt: Date.now() + (ttl || this.ttl)
    });
  }

  // Use cache for dispute retrieval
  async getDisputeWithCache(disputeId: string): Promise {
    const cacheKey = `dispute:${disputeId}`;

    // Try cache first
    let dispute = await this.cache.get(cacheKey);

    if (!dispute) {
      // Cache miss - fetch from API
      dispute = await client.disputes.get(disputeId);
      this.cache.set(cacheKey, dispute);
    }

    return dispute;
  }
}

4.6.2 Batch Operations

// Batch multiple requests to reduce round trips
class BatchProcessor {
  async getMultipleDisputes(disputeIds: string[]): Promise {
    // Use batch endpoint if available
    const response = await fetch('/api/v1/disputes/batch', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ ids: disputeIds })
    });

    return response.json();
  }

  // Alternative: Use Promise.all for concurrent requests
  async getMultipleDisputesConcurrent(ids: string[]): Promise {
    const promises = ids.map(id => client.disputes.get(id));
    return Promise.all(promises);
  }
}

Production Checklist

Before deploying ODR integrations to production:

  • Implement comprehensive error handling with retry logic
  • Configure appropriate timeout values (recommended: 30s for API calls)
  • Enable logging at appropriate levels (INFO for production, DEBUG for development)
  • Set up monitoring and alerting for critical failures
  • Implement rate limiting on the client side to avoid 429 errors
  • Use connection pooling for HTTP clients to improve performance
  • Validate SSL certificates in production (disable only for testing)
  • Rotate API credentials regularly and store securely
  • Test failover scenarios and disaster recovery procedures
