Chapter 2: Data Format Deep Dive

2.1 Introduction to ODR Data Structures

Phase 1 of the WIA-LEGAL-002 standard establishes the foundational data formats that enable interoperability across online dispute resolution platforms. Just as HTML standardized web content exchange and JSON standardized API communication, WIA-LEGAL-002 provides standardized schemas for representing disputes, parties, communications, evidence, and resolutions in machine-readable formats.

The importance of standardized data formats cannot be overstated. Without common data structures, each ODR platform operates as an isolated silo, unable to exchange case information with other systems, transfer cases between providers, or integrate with court systems and e-commerce platforms. Disputants become locked into specific platforms, neutrals cannot work across multiple systems efficiently, and the entire ODR ecosystem remains fragmented.

The WIA-LEGAL-002 data format specifications address these challenges through comprehensive JSON schemas that capture all aspects of the dispute resolution lifecycle while maintaining flexibility for different dispute types, jurisdictional requirements, and procedural variations.

2.2 Core Data Models

2.2.1 Dispute Record Structure

The dispute record serves as the central data entity in any ODR system, containing all information about a particular conflict from initial filing through final resolution. The WIA-LEGAL-002 dispute record balances completeness with simplicity, capturing essential information while avoiding unnecessary complexity.

  "disputeType": "CONSUMER_ECOMMERCE",
  "claimAmount": {
    "value": 2500.00,

2.5 Status and Workflow States

Dispute lifecycle management requires clear status definitions that all systems interpret consistently. WIA-LEGAL-002 defines a comprehensive state machine for dispute progression.

Table 2.3: Dispute Status States and Transitions
Status Description Allowed Transitions Typical Duration
FILED Dispute submitted, awaiting review ACCEPTED, REJECTED 1-3 days
ACCEPTED Dispute accepted, awaiting response NEGOTIATION, MEDIATION, ARBITRATION 5-14 days
NEGOTIATION Parties negotiating directly SETTLED, MEDIATION, ARBITRATION, WITHDRAWN 7-30 days
MEDIATION_IN_PROGRESS Mediation sessions underway SETTLED, ARBITRATION, IMPASSE 14-45 days
ARBITRATION_IN_PROGRESS Arbitration hearings underway AWARD_ISSUED, SETTLED 30-90 days
SETTLED Agreement reached ENFORCEMENT, COMPLIANCE_VERIFIED Variable
AWARD_ISSUED Arbitrator decision rendered ENFORCEMENT, APPEAL 30 days
COMPLIANCE_VERIFIED Settlement terms fulfilled CLOSED N/A
CLOSED Dispute fully resolved None (terminal state) N/A

Character Encoding Best Practices

All WIA-LEGAL-002 implementations must use UTF-8 encoding to properly support international characters, symbols, and multi-language content. Systems should validate encoding on input and normalize text representations to prevent encoding-related data corruption.

Chapter Summary

Key Takeaways:

  1. WIA-LEGAL-002 Phase 1 establishes comprehensive JSON schemas for dispute records, party information, communications, evidence, and settlements, enabling interoperability across ODR platforms.
  2. The dispute record serves as the central data entity, containing all information from filing through resolution, with standardized fields for parties, claims, timelines, and status tracking.
  3. Evidence and document management schemas support multiple file types, authenticity verification, metadata capture, and chain-of-custody tracking essential for credible dispute resolution.
  4. Settlement agreements use structured data formats to capture terms, obligations, signatures, and enforceability information, generating both machine-readable and human-readable documentation.
  5. Extension mechanisms allow customization for different dispute types and jurisdictions while maintaining core interoperability, with support for internationalization, privacy controls, and version tracking.
  6. Data format standardization enables dispute portability between platforms, integration with external systems, and comprehensive audit trails for compliance and transparency.
