제7장: HealthKit/Google Fit - 플랫폼 통합과 데이터 상호운용성

"건강 데이터의 진정한 가치는 통합에서 나옵니다. 한 곳에 모여 있을 때 비로소 전체 그림을 볼 수 있습니다."

— 팀 쿡, Apple CEO

7.1 건강 데이터 플랫폼의 필요성

현대인은 평균 3-5개의 건강 앱과 2-3개의 웨어러블 기기를 사용합니다. 이들이 만들어내는 데이터는 각각 고립되어 있습니다.

🔍 문제점

  • 데이터 사일로: Fitbit, Apple Watch, Strava, MyFitnessPal 각각 저장
  • 중복 입력: 같은 운동을 여러 앱에 수동 기록
  • 불완전한 그림: 의사에게 일부 데이터만 공유
  • 전환 비용: 기기 바꾸면 모든 히스토리 손실

✅ 해결책: 건강 데이터 허브

중앙 집중식 플랫폼:

  • 모든 건강 앱/기기가 한 곳에 데이터 저장
  • 사용자가 데이터 소유 및 제어
  • 의사, 가족과 선택적 공유
  • 앱 간 자동 동기화

7.2 Apple HealthKit

2014년 iOS 8과 함께 출시된 HealthKit은 Apple 생태계의 건강 데이터 중앙 저장소입니다.

📦 아키텍처


┌─────────────────────────────────────────────┐
│            사용자 제어                       │
│         (건강 앱 대시보드)                   │
└─────────────────────────────────────────────┘
                    ▲
                    │
┌─────────────────────────────────────────────┐
│          HealthKit Store                     │
│     (암호화된 on-device 저장소)              │
└─────────────────────────────────────────────┘
        ▲           ▲           ▲
        │           │           │
   ┌────┴───┐  ┌───┴───┐  ┌───┴────┐
   │ Watch  │  │ Fitbit│  │ Strava │
   │  App   │  │  App  │  │  App   │
   └────────┘  └───────┘  └────────┘

🗂️ 데이터 타입

카테고리별 분류:

카테고리 데이터 타입 예시 개수
Activity 걸음 수, 칼로리, 거리, 계단 15+
Body Measurements 체중, 체지방률, BMI, 키 14+
Heart 심박수, HRV, ECG, 혈압 10+
Nutrition 칼로리, 단백질, 비타민, 수분 50+
Sleep 수면 시간, 단계, 호흡률 8+
Vital Signs 체온, SpO2, 호흡률 8+
Reproductive Health 생리, 배란, 성관계 12+

총 150+ 데이터 타입 (iOS 17 기준)

🔐 보안 및 프라이버시

핵심 원칙:

  • 사용자 제어: 각 앱마다 읽기/쓰기 권한 세밀 설정
  • On-Device: 데이터는 기기에만 저장 (iCloud 선택적)
  • 암호화: AES-256, 암호 잠금 시 접근 불가
  • 투명성: 어떤 앱이 어떤 데이터를 읽었는지 로그

앱 권한 예시:

  • MyFitnessPal → ✅ 칼로리 읽기, ❌ 심박수 읽기
  • Strava → ✅ 운동 데이터 읽기/쓰기, ❌ 체중 읽기
  • 의료 앱 → ✅ 모든 데이터 읽기

💻 개발자 API

주요 클래스:

  • HKHealthStore: 데이터 읽기/쓰기 중앙 인터페이스
  • HKQuantityType: 측정 가능 데이터 (걸음 수, 심박수 등)
  • HKCategoryType: 이산 데이터 (수면, 성관계 등)
  • HKWorkout: 운동 세션 전체 정보
  • HKQuery: 데이터 조회 (통계, 시계열 등)

코드 예시 (Swift):

// 걸음 수 읽기 권한 요청
let stepType = HKQuantityType.quantityType(
    forIdentifier: .stepCount
)!

healthStore.requestAuthorization(
    toShare: [],
    read: [stepType]
) { success, error in
    if success {
        // 오늘 걸음 수 조회
        queryStepsToday()
    }
}

7.3 Google Fit

2014년 출시된 Google Fit은 Android 생태계의 건강 데이터 플랫폼입니다.

🏗️ 아키텍처

클라우드 기반:

  • 데이터는 Google 클라우드에 저장 (HealthKit은 on-device)
  • 모든 기기에서 동기화 (Android, iOS, 웹)
  • Google 계정으로 관리

📊 데이터 모델

핵심 개념:

1. Data Types

  • Activity: 걸음 수, 이동 거리, 칼로리
  • Location: GPS 경로
  • Nutrition: 칼로리, 영양소
  • Body: 체중, 체지방률
  • Heart: 심박수, BPM

2. Data Sources

  • 각 앱/기기는 고유 Data Source ID
  • 동일 데이터 타입의 중복 값 처리
  • 우선순위 설정 (예: Garmin > Mi Band)

3. Sessions

  • 운동 세션 (시작/종료 시간)
  • Activity Type: 달리기, 사이클, 수영 등 120+ 종류

♥️ Heart Points (심장 점수)

Google Fit의 독특한 활동 지표:

  • 1 Heart Point: 중강도 활동 1분
  • 2 Heart Points: 고강도 활동 1분
  • 권장 목표: 주당 150 Heart Points (WHO 기준)
  • 예시: 빠르게 걷기 30분 = 30 Points

🔌 REST API

주요 엔드포인트:

API 기능
DataSources 데이터 소스 등록/조회
Datasets 시계열 데이터 읽기/쓰기
Sessions 운동 세션 관리
Aggregate 통계 데이터 (일/주/월)

7.4 HealthKit vs Google Fit 비교

특성 Apple HealthKit Google Fit
플랫폼 iOS, watchOS만 Android, iOS, 웹
저장 위치 On-device (암호화) Google 클라우드

의료 데이터 처방전, 알레르기, 혈액형 제한적

7.5 기타 건강 데이터 플랫폼

Samsung Health

  • 플랫폼: Android, Tizen (Galaxy Watch)
  • 강점: 체성분 분석 (BIA), 수면 코칭
  • 연동: Google Fit과 양방향 동기화
  • 독점 데이터: BIA는 Samsung Health에만

Garmin Connect

  • 대상: 운동선수, 아웃도어
  • 강점: 상세한 운동 분석, Training Load
  • 데이터 수출: FIT, TCX, GPX 형식
  • 연동: Strava, TrainingPeaks, MyFitnessPal

Fitbit (Google)

  • 상태: Google 인수 후 Google Fit 통합 중
  • 강점: 수면 분석, 커뮤니티
  • Premium: 월 $9.99 (고급 인사이트)

Strava

  • 전문: 러닝, 사이클링
  • 소셜: 운동 기록 공유, 세그먼트 경쟁
  • 연동: 100+ 앱/기기 (Garmin, Wahoo, Zwift 등)

7.6 데이터 상호운용성 표준

🔗 FHIR (Fast Healthcare Interoperability Resources)

HL7 표준:

  • 목적: 의료 데이터 교환 표준
  • 형식: JSON, XML
  • 리소스: Patient, Observation, Condition 등

Apple Health Records:

  • 미국 800+ 병원 연동
  • FHIR로 의료 기록 다운로드
  • 처방전, 알레르기, 검사 결과 통합

FHIR Observation 예시:

{
  "resourceType": "Observation",
  "status": "final",
  "code": {
    "coding": [{
      "system": "http://loinc.org",
      "code": "8867-4",
      "display": "Heart rate"
    }]
  },
  "valueQuantity": {
    "value": 72,
    "unit": "beats/min"
  }
}

📁 FIT (Flexible and Interoperable Data Transfer)

Garmin 개발:

  • 피트니스 데이터 교환 표준
  • 바이너리 형식 (효율적)
  • GPS, 심박수, 파워, 케이던스 등
  • Strava, TrainingPeaks 등 지원

🗺️ GPX/TCX

  • GPX: GPS 경로 (XML)
  • TCX: Training Center XML (심박수 포함)
  • 범용성: 거의 모든 앱 지원
