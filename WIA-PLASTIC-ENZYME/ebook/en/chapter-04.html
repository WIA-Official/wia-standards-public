Chapter 4

Phase 1: Data Format

Standardized schemas for enzyme profiles, degradation processes, and quality metrics

4.1 Overview of Phase 1

Phase 1 of the WIA-PLASTIC-ENZYME standard defines the data formats used throughout the enzymatic plastic degradation ecosystem. These standardized schemas enable interoperability between research laboratories, production facilities, regulatory bodies, and supply chain partners.

The data format specification addresses three core entity types:

  • Enzyme Profiles: Comprehensive characterization of plastic-degrading enzymes
  • Degradation Processes: Recording of industrial degradation operations
  • Quality Metrics: Standardized measurement of outputs and performance

4.2 Enzyme Profile Schema

The enzyme profile schema captures all information needed to characterize, compare, and deploy plastic-degrading enzymes. Each profile follows JSON-LD format for semantic interoperability.

{ "$schema": "https://wia.live/schemas/plastic-enzyme/v1.0.0", "enzyme_profile": { "enzyme_id": "urn:wia:enzyme:turbopetase-v2", "name": "TurboPETase-v2", "classification": "PETase", "source": { "organism": "Ideonella sakaiensis (engineered)", "domain": "bacteria", "environment": "laboratory", "engineered": true, "parent_enzyme": "IsPETase", "mutations": ["S121E", "D186H", "R280A", "S238F"] }, "target_plastics": ["PET", "PBAT"], "kinetics": { "km": { "value": 0.23, "unit": "mM" }, "kcat": { "value": 15.8, "unit": "s-1" }, "kcat_km": { "value": 68700, "unit": "M-1s-1" } }, "optimal_conditions": { "temperature_c": { "optimal": 55, "range": [45, 65] }, "ph": { "optimal": 8.0, "range": [6.5, 9.5] }, "substrate_loading_percent": 15 }, "stability": { "half_life_hours": 72, "thermostability_tm_c": 68.5, "storage_conditions": "-20°C in 50% glycerol" }, "products": { "primary": ["TPA", "MHET", "EG"], "toxicity_class": "non-toxic" } } }

Schema Field Definitions

Field Type Description
enzyme_id URN Globally unique identifier in URN format
classification Enum PETase, MHETase, Cutinase, Lipase, BHETase
kinetics.km Measurement Michaelis constant with unit
kinetics.kcat Measurement Turnover number with unit
stability.half_life_hours Number Time to 50% activity loss at optimal temp

4.3 Degradation Process Schema

The degradation process schema records all parameters and outcomes of enzymatic plastic degradation operations. This enables traceability, quality control, and process optimization.

{ "degradation_process": { "process_id": "urn:wia:process:2025-001-abc", "facility_id": "urn:wia:facility:carbios-lyon", "timestamp": "2025-01-15T08:30:00Z", "plastic_input": { "type": "PET", "form": "flakes", "weight_kg": 500, "crystallinity_percent": 28, "contamination_level": "low", "source": "post-consumer bottles" }, "enzyme_cocktail": [ { "enzyme_id": "urn:wia:enzyme:turbopetase-v2", "concentration_mg_g": 2.5 }, { "enzyme_id": "urn:wia:enzyme:mhetase-std", "concentration_mg_g": 1.2 } ], "conditions": { "temperature_c": 55, "ph": 8.0, "duration_hours": 48, "agitation_rpm": 150, "volume_liters": 5000 }, "output": { "degradation_percent": 98.5, "tpa_yield_kg": 425, "tpa_purity_percent": 99.2, "eg_yield_kg": 145, "eg_purity_percent": 98.7, "residue_kg": 7.5 }, "quality_grade": "food-contact", "certification_status": "WIA-certified" } }

Process Validation Rules

The WIA standard defines validation rules to ensure data integrity:

  • Degradation percentage must be 0-100
  • Output yields must not exceed input mass (with tolerance for measurement error)
  • Temperature and pH must be within enzyme's optimal range
  • Enzyme concentrations must be positive values
  • Timestamps must be in ISO 8601 format

4.4 Quality Metrics Schema

The quality metrics schema defines how to measure and report the outputs of enzymatic degradation:

{ "quality_metrics": { "batch_id": "urn:wia:batch:2025-001-abc", "measurement_date": "2025-01-16T14:00:00Z", "monomer_analysis": { "tpa": { "concentration_mM": 285.5, "purity_percent": 99.2, "contaminants": { "mhet_ppm": 120, "bhet_ppm": 45, "oligomers_ppm": 200 }, "color_hazen": 15, "method": "HPLC-UV" }, "eg": { "concentration_mM": 142.0, "purity_percent": 98.7, "method": "GC-FID" } }, "safety_assessment": { "heavy_metals_compliant": true, "microbial_count_cfu_g": 10, "endotoxin_eu_ml": 0.5, "residual_enzyme_activity": "undetectable" }, "grade_qualification": { "food_contact_eligible": true, "bottle_grade_eligible": true, "textile_grade_eligible": true, "certification_ids": ["WIA-PET-FC-2025-001"] } } }

4.5 Plastic Type Codes

The standard defines codes for all plastic types covered by enzymatic degradation:

Code Name Description
PET Polyethylene Terephthalate Bottles, containers, textiles
PBAT Polybutylene Adipate Terephthalate Biodegradable packaging
PLA Polylactic Acid Bioplastic from renewable sources
PCL Polycaprolactone Medical and packaging applications
PHA Polyhydroxyalkanoates Bacterial bioplastics

4.6 Data Exchange Formats

The WIA standard supports multiple data exchange formats:

  • JSON: Primary format for API exchanges and data storage
  • JSON-LD: For semantic web integration and linked data
  • CSV: For bulk data import/export and spreadsheet compatibility
  • Protocol Buffers: For high-performance binary transmission

Interoperability Principle

All WIA-certified systems must support JSON format at minimum. JSON-LD support is recommended for research institutions and regulatory bodies to enable semantic queries and data federation.

4.7 Version Control and Evolution

The data format specification includes versioning to support evolution while maintaining backward compatibility:

  • Major version: Breaking changes requiring data migration
  • Minor version: New optional fields, backward compatible
  • Patch version: Bug fixes and clarifications only

The current version is 1.0.0. All compliant systems must include schema version in their data exports.

4.8 Implementation Examples

Creating an Enzyme Profile

// TypeScript SDK Example import { WiaPlasticEnzyme } from '@wia/plastic-enzyme-sdk'; const client = new WiaPlasticEnzyme({ apiKey: 'your-key' }); const enzyme = await client.enzymes.create({ name: 'MyPETase-v1', classification: 'PETase', source: { organism: 'Engineered E. coli', engineered: true }, kinetics: { km: { value: 0.35, unit: 'mM' }, kcat: { value: 12.5, unit: 's-1' } }, optimal_conditions: { temperature_c: { optimal: 50, range: [40, 60] }, ph: { optimal: 7.5, range: [6.5, 8.5] } } });

Recording a Degradation Process

// Recording a degradation batch const process = await client.processes.create({ plastic_input: { type: 'PET', weight_kg: 100, crystallinity_percent: 30 }, enzyme_cocktail: [ { enzyme_id: 'urn:wia:enzyme:turbopetase-v2', concentration_mg_g: 2.0 } ], conditions: { temperature_c: 55, ph: 8.0, duration_hours: 48 } }); // Record output after completion await client.processes.recordOutput(process.id, { degradation_percent: 97.5, tpa_yield_kg: 82, eg_yield_kg: 28 });

4.9 Chapter Summary

Phase 1 of the WIA-PLASTIC-ENZYME standard establishes the foundational data formats that enable global interoperability in enzymatic plastic recycling. Key components include:

  • Enzyme Profile Schema: Complete characterization of plastic-degrading enzymes
  • Degradation Process Schema: Recording of industrial operations and outcomes
  • Quality Metrics Schema: Standardized measurement and certification
  • Validation Rules: Ensuring data integrity and consistency
  • Exchange Formats: Multiple formats for different use cases

In the next chapter, we will explore Phase 2: the API interfaces that allow systems to exchange data and coordinate operations using these standard formats.
