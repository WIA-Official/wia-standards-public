제2장: 탐지 방법론

2.1 탐지 방법 개요

IDS/IPS는 다양한 탐지 방법을 사용하여 네트워크나 시스템에서 발생하는 악의적인 활동을 식별합니다. 각 탐지 방법은 고유한 강점과 약점을 가지고 있으며, 현대의 IDS/IPS 시스템은 일반적으로 여러 방법을 조합하여 사용합니다. 주요 탐지 방법으로는 시그니처 기반 탐지, 이상 탐지, 프로토콜 분석, 그리고 하이브리드 접근 방식이 있습니다.

탐지 방법 원리 장점 단점
시그니처 기반 알려진 공격 패턴 매칭 높은 정확도, 낮은 오탐 신종 공격 탐지 불가
이상 탐지 정상 행위 프로파일링 제로데이 탐지 가능 높은 오탐율
프로토콜 분석 프로토콜 표준 위반 탐지 정교한 우회 탐지 높은 리소스 소비
하이브리드 여러 방법 조합 포괄적 탐지 복잡한 설정 및 관리

2.2 시그니처 기반 탐지 (Signature-Based Detection)

시그니처 기반 탐지는 가장 오래되고 널리 사용되는 탐지 방법입니다. 이 방식은 안티바이러스 소프트웨어가 악성코드 시그니처를 사용하는 것과 유사하게, 알려진 공격의 특징적인 패턴을 데이터베이스에 저장하고 네트워크 트래픽이나 시스템 활동과 비교합니다.

2.2.1 시그니처의 구성 요소

효과적인 시그니처는 다음과 같은 요소들로 구성됩니다:

  • 패턴(Pattern): 공격의 고유한 바이트 시퀀스나 문자열
  • 메타데이터: 프로토콜, 포트, IP 주소 등의 컨텍스트 정보
  • 심각도(Severity): 위협의 중요도 수준
  • 분류(Classification): 공격 유형 카테고리
  • 참조(Reference): CVE, OSVDB 등 외부 참조 정보

2.2.2 Snort 규칙 예시

Snort는 가장 널리 사용되는 오픈소스 IDS로, 다음과 같은 형식의 규칙을 사용합니다: 

# SQL Injection 공격 탐지
alert tcp any any -> $HOME_NET $HTTP_PORTS (
    msg:"SQL Injection 시도 탐지";
    flow:to_server,established;
    content:"UNION"; nocase;
    content:"SELECT"; nocase; distance:0;
    pcre:"/(\%27)|(\')|(\-\-)|(\%23)|(#)/i";
    classtype:web-application-attack;
    sid:1000001; rev:1;
)

# SSH Brute Force 공격 탐지
alert tcp any any -> $HOME_NET 22 (
    msg:"SSH Brute Force 공격 의심";
    flow:to_server,established;
    content:"SSH-"; depth:4;
    detection_filter:track by_src, count 5, seconds 60;
    classtype:attempted-admin;
    sid:1000002; rev:1;
)

# Buffer Overflow 시도 탐지
alert tcp any any -> $HOME_NET any (
    msg:"Buffer Overflow 시도";
    content:"|90 90 90 90|"; depth:500;
    content:"/bin/sh"; distance:0;
    classtype:shellcode-detect;
    priority:1;
    sid:1000003; rev:1;
)

# 악성 파일 다운로드 탐지
alert tcp $HOME_NET any -> $EXTERNAL_NET $HTTP_PORTS (
    msg:"의심스러운 실행 파일 다운로드";
    flow:to_server,established;
    content:"GET"; http_method;
    content:".exe"; http_uri; nocase;
    content:"User-Agent|3a| curl"; http_header;
    classtype:policy-violation;
    sid:1000004; rev:1;
)

2.2.3 시그니처 매칭 알고리즘

시그니처 매칭의 성능은 사용되는 알고리즘에 크게 의존합니다. 주요 알고리즘은 다음과 같습니다:

알고리즘 복잡도 특징 사용 사례
Boyer-Moore O(n/m) 역방향 검색, 효율적 단일 패턴 검색
Aho-Corasick O(n+m+z) 다중 패턴 동시 검색 대규모 시그니처 세트
Wu-Manber O(n/m) 다중 패턴, 해시 활용 Snort 기본 엔진
정규표현식 (PCRE) O(n²) 유연한 패턴 매칭 복잡한 패턴
하이퍼스캔 O(n) SIMD 가속, 고성능 Intel 플랫폼

2.2.4 시그니처 기반 탐지의 한계

주요 한계점:
  • 제로데이 공격: 시그니처가 아직 생성되지 않은 신종 공격은 탐지 불가
  • 변종 공격: 약간만 변형된 공격도 탐지하지 못할 수 있음
  • 시그니처 관리: 수천 개의 시그니처를 지속적으로 업데이트해야 함
  • 성능 저하: 시그니처 수가 증가하면 처리 속도 감소
  • 우회 기술: 인코딩, 난독화, 분할 등으로 우회 가능

2.3 이상 탐지 (Anomaly-Based Detection)

이상 탐지는 정상적인 네트워크 트래픽이나 시스템 행위의 베이스라인을 학습한 후, 이 베이스라인에서 벗어나는 활동을 탐지하는 방법입니다. 이 접근 방식은 제로데이 공격이나 아직 알려지지 않은 위협을 탐지할 수 있다는 장점이 있습니다.

2.3.1 베이스라인 학습 과정

  1. 데이터 수집 단계 (1-4주)
    • 정상 운영 기간 동안 네트워크 트래픽 수집
    • 다양한 시간대와 요일의 패턴 포함
    • 피크 시간과 오프 시간 모두 고려
  2. 프로파일링 단계
    • 통계적 분석: 평균, 표준편차, 분포 계산
    • 행위 패턴 식별: 일반적인 프로토콜 사용, 접속 패턴
    • 임계값 설정: 정상 범위의 상한과 하한 정의
  3. 검증 단계
    • 학습된 프로파일의 정확성 검증
    • 오탐율 측정 및 조정
    • 계절성 및 주기성 고려

2.3.2 이상 탐지 기법

기법 방법 장점 적용 분야
통계 기반 평균, 표준편차로 이상치 탐지 간단하고 빠름 트래픽 볼륨 분석
임계값 기반 사전 정의된 한계 초과 탐지 구현 용이 연결 수, 대역폭
프로파일 기반 사용자/호스트별 행위 학습 정교한 탐지 내부자 위협
프로토콜 기반 정상 프로토콜 흐름 학습 프로토콜 오용 탐지 DDoS, 스캔
머신러닝 패턴 자동 학습 적응적 탐지 복잡한 공격

2.3.3 이상 탐지 구현 예시

# Python을 사용한 간단한 통계 기반 이상 탐지
import numpy as np
from collections import defaultdict

class AnomalyDetector:
    def __init__(self, threshold=3):
        self.baseline = defaultdict(list)
        self.threshold = threshold  # 표준편차 배수

    def train(self, traffic_data):
        """정상 트래픽으로 베이스라인 학습"""
        for record in traffic_data:
            src_ip = record['src_ip']
            bytes_sent = record['bytes']
            self.baseline[src_ip].append(bytes_sent)

        # 각 IP별 통계 계산
        self.stats = {}
        for ip, data in self.baseline.items():
            self.stats[ip] = {
                'mean': np.mean(data),
                'std': np.std(data)
            }

    def detect(self, record):
        """새로운 트래픽이 이상인지 판단"""
        src_ip = record['src_ip']
        bytes_sent = record['bytes']

        if src_ip not in self.stats:
            return False, "신규 IP - 학습 필요"

        mean = self.stats[src_ip]['mean']
        std = self.stats[src_ip]['std']

        # Z-score 계산
        z_score = abs((bytes_sent - mean) / std) if std > 0 else 0

        if z_score > self.threshold:
            return True, f"이상 트래픽 탐지 (Z-score: {z_score:.2f})"
        return False, "정상"

# 사용 예시
detector = AnomalyDetector(threshold=3)

# 학습 데이터 (정상 트래픽)
training_data = [
    {'src_ip': '192.168.1.10', 'bytes': 1500},
    {'src_ip': '192.168.1.10', 'bytes': 1600},
    {'src_ip': '192.168.1.10', 'bytes': 1450},
    # ... 더 많은 데이터
]

detector.train(training_data)

# 새로운 트래픽 검사
test_record = {'src_ip': '192.168.1.10', 'bytes': 50000}
is_anomaly, message = detector.detect(test_record)
print(f"탐지 결과: {message}")

2.3.4 이상 탐지의 도전과제

  • 높은 오탐율: 정상적이지만 드문 활동을 위협으로 오인
  • 베이스라인 드리프트: 네트워크 환경 변화로 인한 베이스라인 노후화
  • 학습 데이터 오염: 학습 기간 중 공격이 포함될 경우 잘못된 베이스라인 생성
  • 점진적 공격: 천천히 진행되는 공격은 베이스라인에 흡수될 수 있음
  • 리소스 집약적: 지속적인 학습과 분석에 많은 컴퓨팅 파워 필요

2.4 프로토콜 분석 (Protocol Analysis)

프로토콜 분석은 네트워크 프로토콜의 표준 동작을 이해하고, 이 표준에서 벗어나는 행위를 탐지하는 방법입니다. 이 기법은 RFC(Request for Comments) 표준과 프로토콜 사양을 기반으로 합니다.

2.4.1 프로토콜 계층별 분석

OSI 계층 프로토콜 분석 항목 탐지 가능 공격
2계층 (데이터링크) Ethernet, ARP MAC 주소, ARP 테이블 ARP 스푸핑, MAC 플러딩
3계층 (네트워크) IP, ICMP IP 헤더, 단편화 IP 스푸핑, Ping of Death
4계층 (전송) TCP, UDP 포트, 시퀀스 번호, 플래그 SYN 플러드, 포트 스캔
7계층 (응용) HTTP, DNS, SMTP 요청/응답 패턴, 페이로드 SQL Injection, XSS, DNS 터널링

2.4.2 상태 기반 프로토콜 분석

상태 기반 분석은 프로토콜의 정상적인 상태 전이를 추적하고, 비정상적인 전이를 탐지합니다. 

# TCP 3-Way Handshake 상태 검증 예시
class TCPStateAnalyzer:
    def __init__(self):
        self.connections = {}  # {(src_ip, src_port, dst_ip, dst_port): state}

    def analyze_packet(self, packet):
        key = (packet.src_ip, packet.src_port,
               packet.dst_ip, packet.dst_port)

        current_state = self.connections.get(key, 'CLOSED')

        # TCP 플래그에 따른 상태 전이 검증
        if packet.flags['SYN'] and not packet.flags['ACK']:
            # SYN 패킷
            if current_state == 'CLOSED':
                self.connections[key] = 'SYN_SENT'
                return True, "정상 연결 시작"
            else:
                return False, "비정상 SYN (잘못된 상태)"

        elif packet.flags['SYN'] and packet.flags['ACK']:
            # SYN-ACK 패킷
            reverse_key = (packet.dst_ip, packet.dst_port,
                          packet.src_ip, packet.src_port)
            if self.connections.get(reverse_key) == 'SYN_SENT':
                self.connections[reverse_key] = 'SYN_RECEIVED'
                return True, "정상 SYN-ACK"
            else:
                return False, "비정상 SYN-ACK (SYN 없이 발생)"

        elif packet.flags['ACK'] and not packet.flags['SYN']:
            # ACK 패킷
            if current_state == 'SYN_RECEIVED':
                self.connections[key] = 'ESTABLISHED'
                return True, "연결 확립"
            elif current_state == 'ESTABLISHED':
                return True, "데이터 전송"
            else:
                return False, "비정상 ACK (연결되지 않음)"

        elif packet.flags['FIN']:
            # FIN 패킷
            if current_state == 'ESTABLISHED':
                self.connections[key] = 'FIN_WAIT'
                return True, "정상 연결 종료 시작"
            else:
                return False, "비정상 FIN"

        return False, "알 수 없는 패킷 조합"

2.5 행위 기반 탐지 (Behavioral Detection)

행위 기반 탐지는 사용자, 호스트, 또는 애플리케이션의 전형적인 행위 패턴을 학습하고, 이상 행위를 식별합니다. 이는 내부자 위협이나 계정 탈취 공격을 탐지하는 데 특히 효과적입니다.

2.5.1 사용자 행위 분석 (UEBA)

분석 영역 정상 행위 이상 행위 가능한 위협
접속 시간 근무 시간 내 접속 심야 시간 접속 계정 탈취
접속 위치 사무실 IP 범위 해외 IP 접속 크리덴셜 도용
데이터 접근 업무 관련 파일만 접근 대량 파일 다운로드 데이터 유출
권한 사용 일반 사용자 권한 갑작스런 관리자 권한 사용 권한 상승 공격
네트워크 활동 내부 통신만 외부로 대용량 전송 데이터 유출

2.5.2 엔티티 행위 분석

# 사용자 행위 프로파일링 예시
class UserBehaviorProfile:
    def __init__(self, user_id):
        self.user_id = user_id
        self.login_times = []
        self.login_locations = []
        self.accessed_resources = set()
        self.data_transfer_sizes = []

    def update_profile(self, event):
        """사용자 활동으로 프로파일 업데이트"""
        if event.type == 'login':
            self.login_times.append(event.timestamp.hour)
            self.login_locations.append(event.ip_address)
        elif event.type == 'resource_access':
            self.accessed_resources.add(event.resource)
        elif event.type == 'data_transfer':
            self.data_transfer_sizes.append(event.size)

    def is_anomalous(self, event):
        """이상 행위 탐지"""
        anomaly_score = 0
        reasons = []

        # 1. 비정상 시간대 체크
        if event.type == 'login':
            avg_hour = np.mean(self.login_times)
            if abs(event.timestamp.hour - avg_hour) > 8:
                anomaly_score += 30
                reasons.append("비정상 시간대 접속")

        # 2. 새로운 위치 체크
        if event.type == 'login':
            if event.ip_address not in self.login_locations:
                anomaly_score += 25
                reasons.append("신규 위치에서 접속")

        # 3. 비정상적인 리소스 접근
        if event.type == 'resource_access':
            if event.resource not in self.accessed_resources:
                if event.resource.sensitivity == 'high':
                    anomaly_score += 40
                    reasons.append("민감한 신규 리소스 접근")

        # 4. 대량 데이터 전송
        if event.type == 'data_transfer':
            avg_size = np.mean(self.data_transfer_sizes) if self.data_transfer_sizes else 0
            std_size = np.std(self.data_transfer_sizes) if self.data_transfer_sizes else 0
            if event.size > avg_size + 3 * std_size:
                anomaly_score += 35
                reasons.append("비정상적으로 큰 데이터 전송")

        return anomaly_score >= 50, anomaly_score, reasons

2.6 하이브리드 탐지 접근법

현대의 고급 IDS/IPS 시스템은 여러 탐지 방법을 결합한 하이브리드 접근법을 사용합니다. 이를 통해 각 방법의 장점을 활용하고 단점을 보완할 수 있습니다.

2.7 탐지 성능 최적화

IDS/IPS의 탐지 성능을 최적화하기 위해서는 정확성과 처리 속도 사이의 균형을 찾아야 합니다.

2.7.1 성능 최적화 기법

  • 규칙 정렬: 가장 자주 매칭되는 규칙을 먼저 배치
  • 조기 종료: 첫 번째 매칭에서 검사 중단 (단일 매칭 모드)
  • 프리필터링: 빠른 1차 필터로 대부분의 정상 트래픽 통과
  • 병렬 처리: 멀티코어 CPU 활용으로 동시 처리
  • 하드웨어 가속: FPGA, GPU, ASIC 활용
  • 패킷 샘플링: 고부하 상황에서 일부 패킷만 검사
  • 스트림 재구성 제한: 세션 크기 제한으로 메모리 보호

2.8 오탐 및 미탐 관리

IDS/IPS 운영에서 가장 큰 도전 과제는 오탐(False Positive)과 미탐(False Negative)의 균형을 맞추는 것입니다.

2.8.1 오탐 감소 전략

  1. 베이스라인 튜닝: 조직의 정상 트래픽 패턴 학습 후 규칙 조정
  2. 화이트리스트 활용: 알려진 정상 활동을 명시적으로 허용
  3. 컨텍스트 인식: 네트워크 위치, 시간, 사용자 등 컨텍스트 고려
  4. 임계값 조정: 탐지 민감도를 환경에 맞게 조정
  5. 상관 분석: 여러 이벤트를 조합하여 신뢰도 향상
  6. 피드백 루프: 분석가의 판단을 시스템에 반영

2.8.2 탐지 효과성 측정

# 탐지 성능 메트릭 계산
class DetectionMetrics:
    def __init__(self):
        self.true_positives = 0   # 정확히 탐지한 공격
        self.false_positives = 0  # 정상을 공격으로 오인
        self.true_negatives = 0   # 정확히 정상으로 판단
        self.false_negatives = 0  # 공격을 놓침

    def calculate_metrics(self):
        total = (self.true_positives + self.false_positives +
                self.true_negatives + self.false_negatives)

        # 정확도 (Accuracy)
        accuracy = (self.true_positives + self.true_negatives) / total

        # 정밀도 (Precision) - 경보 중 실제 공격 비율
        precision = (self.true_positives /
                    (self.true_positives + self.false_positives))

        # 재현율 (Recall) - 실제 공격 중 탐지 비율
        recall = (self.true_positives /
                 (self.true_positives + self.false_negatives))

        # F1 점수 - 정밀도와 재현율의 조화 평균
        f1_score = 2 * (precision * recall) / (precision + recall)

        # 오탐률 (False Positive Rate)
        fpr = (self.false_positives /
              (self.false_positives + self.true_negatives))

        return {
            'accuracy': accuracy,
            'precision': precision,
            'recall': recall,
            'f1_score': f1_score,
            'false_positive_rate': fpr
        }

# 이상적인 목표 값
# Precision: > 90% (오탐 최소화)
# Recall: > 95% (미탐 최소화)
# F1 Score: > 92% (균형)

요약

이 장에서는 IDS/IPS의 핵심 탐지 방법론을 상세히 살펴보았습니다. 시그니처 기반 탐지는 알려진 공격 패턴을 데이터베이스에 저장하고 매칭하는 방식으로, 높은 정확도를 제공하지만 신종 공격은 탐지하지 못합니다. 이상 탐지는 정상 행위의 베이스라인을 학습하여 이탈을 탐지하므로 제로데이 공격도 발견할 수 있지만, 오탐률이 높다는 단점이 있습니다.

프로토콜 분석은 RFC 표준에 기반하여 프로토콜 위반을 탐지하며, 정교한 우회 공격을 식별할 수 있습니다. 행위 기반 탐지는 사용자와 엔티티의 정상 행위 패턴을 학습하여 내부자 위협이나 계정 탈취를 탐지합니다. 현대의 IDS/IPS는 이러한 방법들을 조합한 하이브리드 접근법을 사용하여 각 방법의 장점을 극대화하고 단점을 보완합니다.

효과적인 탐지를 위해서는 성능 최적화가 필수적입니다. 규칙 정렬, 병렬 처리, 하드웨어 가속 등의 기법을 통해 높은 처리량과 낮은 지연시간을 달성할 수 있습니다. 또한 오탐과 미탐의 균형을 맞추기 위해 지속적인 튜닝, 화이트리스트 활용, 컨텍스트 인식 탐지가 필요하며, 정밀도, 재현율, F1 점수 등의 메트릭을 통해 탐지 성능을 정량적으로 평가해야 합니다.
