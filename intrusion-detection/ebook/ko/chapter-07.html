제7장: 사고 대응 (Incident Response)

7.1 사고 대응 개요

IDS/IPS가 위협을 탐지한 후 효과적으로 대응하지 못하면 탐지의 가치는 반감됩니다. 사고 대응(Incident Response, IR)은 보안 사고를 체계적으로 처리하여 피해를 최소화하고, 신속하게 정상 운영으로 복구하며, 향후 유사한 사고를 예방하는 프로세스입니다.

7.1.1 사고 대응 생명주기 (NIST)

1. 준비 (Preparation)
   ↓
2. 탐지 및 분석 (Detection & Analysis)
   ↓
3. 격리, 제거, 복구 (Containment, Eradication, Recovery)
   ↓
4. 사후 활동 (Post-Incident Activity)
   ↓
   (교훈 반영하여 1단계로 순환)

7.1.2 사고 대응 팀 구성

역할 책임 필요 기술
IR 매니저 팀 조율, 의사결정, 경영진 보고 리더십, 의사소통
보안 분석가 IDS/SIEM 모니터링, 경보 분류 보안 지식, 로그 분석
포렌식 전문가 증거 수집, 근본 원인 분석 디지털 포렌식, 악성코드 분석
네트워크 엔지니어 네트워크 격리, 트래픽 분석 네트워크 인프라
시스템 관리자 호스트 격리, 시스템 복구 OS, 서버 관리
법무/PR 법적 대응, 대외 커뮤니케이션 법률, 홍보

7.2 사고 탐지 및 분류

IDS 경보가 발생했을 때 실제 사고인지, 오탐인지, 그리고 심각도가 어느 정도인지 신속하게 판단해야 합니다.

7.2.2 심각도 평가 매트릭스

우선순위 정의 예시 대응 SLA
P1 (Critical) 중요 시스템 침해, 데이터 유출 DB 서버 랜섬웨어, 신용카드 유출 즉시 (15분 이내)
P2 (High) 활성 공격 진행 중, 일부 시스템 영향 웹 서버 침해, 내부 확산 시도 1시간 이내
P3 (Medium) 정책 위반, 잠재적 위협 업무 외 시간 접근, 의심 파일 4시간 이내
P4 (Low) 정보성 경보, 최소 영향 외부 포트 스캔, 알려진 오탐 24시간 이내

7.2.3 경보 분류 스크립트

#!/usr/bin/env python3
from datetime import datetime

class AlertTriage:
    def __init__(self):
        self.critical_assets = ['192.168.10.50', '192.168.10.51']  # DB 서버
        self.business_hours = (9, 18)  # 09:00 - 18:00
        self.known_false_positives = set()

    def classify_alert(self, alert):
        """경보 분류 및 우선순위 결정"""
        # 1. 오탐 체크
        if self.is_false_positive(alert):
            return {
                'priority': 'P4',
                'action': 'ignore',
                'reason': '알려진 오탐 패턴'
            }

        # 2. 자산 중요도
        asset_criticality = self.assess_asset(alert['dst_ip'])

        # 3. 공격 심각도
        attack_severity = self.assess_attack(alert)

        # 4. 컨텍스트 (시간, 사용자 등)
        context_risk = self.assess_context(alert)

        # 5. 종합 점수 계산
        total_score = (
            asset_criticality * 0.4 +
            attack_severity * 0.4 +
            context_risk * 0.2
        )

        # 6. 우선순위 결정
        priority = self.determine_priority(total_score)

        return {
            'priority': priority,
            'score': total_score,
            'asset_criticality': asset_criticality,
            'attack_severity': attack_severity,
            'context_risk': context_risk,
            'recommended_action': self.get_recommended_action(priority),
            'sla': self.get_sla(priority)
        }

    def is_false_positive(self, alert):
        """오탐 여부 확인"""
        # 알려진 오탐 패턴 체크
        signature = f"{alert['src_ip']}:{alert['signature_id']}"
        return signature in self.known_false_positives

    def assess_asset(self, ip):
        """자산 중요도 평가 (0-10)"""
        if ip in self.critical_assets:
            return 10  # 중요 자산
        elif ip.startswith('192.168.10.'):
            return 7   # 서버 세그먼트
        elif ip.startswith('192.168.20.'):
            return 5   # 사용자 세그먼트
        else:
            return 3   # 기타

    def assess_attack(self, alert):
        """공격 심각도 평가 (0-10)"""
        severity_map = {
            'data_exfiltration': 10,
            'ransomware': 10,
            'privilege_escalation': 9,
            'remote_code_execution': 9,
            'sql_injection': 8,
            'xss': 6,
            'port_scan': 3,
            'dos': 7
        }

        attack_type = alert.get('attack_type', 'unknown')
        return severity_map.get(attack_type, 5)

    def assess_context(self, alert):
        """컨텍스트 위험도 평가 (0-10)"""
        risk = 0

        # 업무 외 시간
        hour = datetime.fromisoformat(alert['timestamp']).hour
        if hour < self.business_hours[0] or hour > self.business_hours[1]:
            risk += 3

        # 외부 IP
        if not alert['src_ip'].startswith('192.168.'):
            risk += 2

        # 반복 공격
        if alert.get('repeat_count', 1) > 5:
            risk += 3

        # 다중 표적
        if len(alert.get('targets', [])) > 3:
            risk += 2

        return min(risk, 10)

    def determine_priority(self, score):
        """점수를 우선순위로 변환"""
        if score >= 8:
            return 'P1'
        elif score >= 6:
            return 'P2'
        elif score >= 4:
            return 'P3'
        else:
            return 'P4'

    def get_recommended_action(self, priority):
        """우선순위별 권장 조치"""
        actions = {
            'P1': ['즉시 IR 팀 소집', '영향받은 시스템 격리', '경영진 통보'],
            'P2': ['담당 분석가 할당', '상세 조사 시작', '모니터링 강화'],
            'P3': ['큐에 추가', '정기 검토 시 조사'],
            'P4': ['로그 기록', '필요 시 규칙 조정']
        }
        return actions.get(priority, [])

    def get_sla(self, priority):
        """우선순위별 SLA (분)"""
        sla_map = {'P1': 15, 'P2': 60, 'P3': 240, 'P4': 1440}
        return sla_map.get(priority, 1440)

# 사용 예시
triage = AlertTriage()

alert = {
    'timestamp': '2024-12-26T22:30:00',
    'src_ip': '185.220.101.50',
    'dst_ip': '192.168.10.50',
    'signature_id': 100001,
    'attack_type': 'sql_injection',
    'repeat_count': 3
}

result = triage.classify_alert(alert)
print(f"우선순위: {result['priority']}")
print(f"점수: {result['score']:.2f}")
print(f"SLA: {result['sla']}분")
print(f"권장 조치:")
for action in result['recommended_action']:
    print(f"  - {action}")

7.3 격리 (Containment)

사고를 확인한 후 첫 번째 목표는 추가 피해를 방지하는 것입니다. 격리는 공격자의 접근을 차단하고 악성코드 확산을 막습니다.

7.3.1 격리 전략

격리 방법 구현 장점 단점
네트워크 차단 방화벽 규칙, ACL 빠름, 원격 실행 이미 내부 침투 시 제한적
VLAN 격리 스위치 포트 이동 네트워크 분리, 모니터링 가능 물리적 접근 가능
호스트 격리 네트워크 인터페이스 비활성화 완전 격리 원격 관리 불가
계정 비활성화 AD, IAM에서 계정 잠금 크리덴셜 도용 대응 공격자가 다른 계정 사용 가능
서비스 중단 취약 서비스 셧다운 익스플로잇 차단 서비스 중단

7.3.2 자동 격리 스크립트

#!/usr/bin/env python3
import subprocess
import requests
import json

class ContainmentActions:
    def __init__(self, firewall_api, ad_api):
        self.firewall_api = firewall_api
        self.ad_api = ad_api

    def contain_host(self, ip_address, method='network_block'):
        """호스트 격리"""
        actions_taken = []

        if method == 'network_block':
            # 방화벽에서 IP 차단
            result = self.block_at_firewall(ip_address)
            actions_taken.append(result)

        elif method == 'vlan_isolation':
            # 격리 VLAN으로 이동
            result = self.move_to_quarantine_vlan(ip_address)
            actions_taken.append(result)

        elif method == 'full_isolation':
            # 완전 격리 (네트워크 차단 + 호스트 셧다운)
            actions_taken.append(self.block_at_firewall(ip_address))
            actions_taken.append(self.disable_host_network(ip_address))

        # 로그 기록
        self.log_containment(ip_address, method, actions_taken)

        return actions_taken

    def block_at_firewall(self, ip_address):
        """방화벽에서 IP 차단"""
        try:
            rule = {
                'action': 'deny',
                'source': ip_address,
                'destination': 'any',
                'log': True,
                'description': f'IR - Compromised host {ip_address}'
            }

            response = requests.post(
                f"{self.firewall_api}/rules",
                json=rule,
                headers={'Authorization': 'Bearer token'},
                timeout=10
            )

            return {
                'action': 'firewall_block',
                'success': response.ok,
                'details': f"Blocked {ip_address} at firewall"
            }
        except Exception as e:
            return {'action': 'firewall_block', 'success': False, 'error': str(e)}

    def move_to_quarantine_vlan(self, ip_address):
        """격리 VLAN으로 이동"""
        # 1. IP로 MAC 주소 찾기
        mac = self.get_mac_from_ip(ip_address)

        # 2. MAC으로 스위치 포트 찾기
        switch, port = self.find_switch_port(mac)

        # 3. 포트를 격리 VLAN으로 이동
        try:
            # 스위치 CLI 명령 (Cisco 예시)
            commands = [
                f"interface {port}",
                "switchport access vlan 999",  # 격리 VLAN
                "description QUARANTINE - IR"
            ]

            # SSH로 스위치 접속 및 명령 실행
            self.execute_switch_commands(switch, commands)

            return {
                'action': 'vlan_isolation',
                'success': True,
                'details': f"Moved {ip_address} to VLAN 999"
            }
        except Exception as e:
            return {'action': 'vlan_isolation', 'success': False, 'error': str(e)}

    def disable_host_network(self, ip_address):
        """호스트의 네트워크 인터페이스 비활성화"""
        # WMI, SSH, 또는 에이전트를 통해 원격 실행
        try:
            # Windows 예시 (PSExec 또는 PowerShell Remoting)
            command = f"Disable-NetAdapter -Name 'Ethernet' -Confirm:$false"

            result = subprocess.run(
                ['powershell', '-Command', command],
                capture_output=True,
                text=True,
                timeout=30
            )

            return {
                'action': 'disable_network',
                'success': result.returncode == 0,
                'output': result.stdout
            }
        except Exception as e:
            return {'action': 'disable_network', 'success': False, 'error': str(e)}

    def disable_user_account(self, username):
        """사용자 계정 비활성화"""
        try:
            # Active Directory API 호출
            response = requests.post(
                f"{self.ad_api}/users/{username}/disable",
                headers={'Authorization': 'Bearer token'},
                timeout=10
            )

            if response.ok:
                # 모든 활성 세션 종료
                self.terminate_user_sessions(username)

            return {
                'action': 'disable_account',
                'success': response.ok,
                'user': username
            }
        except Exception as e:
            return {'action': 'disable_account', 'success': False, 'error': str(e)}

    def terminate_user_sessions(self, username):
        """사용자의 모든 활성 세션 종료"""
        # VPN, RDP, SSH 세션 강제 종료
        pass

    def log_containment(self, target, method, actions):
        """격리 조치 로깅"""
        log_entry = {
            'timestamp': datetime.now().isoformat(),
            'target': target,
            'method': method,
            'actions': actions,
            'operator': 'ir_automation'
        }

        with open('/var/log/ir_containment.log', 'a') as f:
            f.write(json.dumps(log_entry) + '\n')

# 사용
containment = ContainmentActions(
    firewall_api="https://firewall.internal/api",
    ad_api="https://ad.internal/api"
)

# 침해된 호스트 격리
actions = containment.contain_host('192.168.10.100', method='full_isolation')

# 탈취된 계정 비활성화
containment.disable_user_account('john.doe')

7.4 제거 (Eradication)

격리 후에는 공격자의 접근 경로와 악성코드를 완전히 제거해야 합니다.

7.4.1 제거 체크리스트

시스템 정화:
  • 악성코드 제거 (안티바이러스, 수동 제거)
  • 백도어 탐색 및 제거
  • 루트킷 검사 및 제거
  • 지속성 메커니즘 제거 (레지스트리, cron job)
  • 웹셸 제거
계정 및 크리덴셜:
  • 모든 비밀번호 변경 (특히 특권 계정)
  • 불법 계정 삭제
  • 세션 토큰 무효화
  • SSH 키, API 키 교체
취약점 패치:
  • 익스플로잇된 취약점 패치
  • 보안 설정 강화
  • 불필요한 서비스 비활성화

7.4.2 악성코드 제거 스크립트

#!/bin/bash
# 리눅스 시스템 악성코드 제거 스크립트

LOG="/var/log/eradication.log"

log() {
    echo "[$(date '+%Y-%m-%d %H:%M:%S')] $1" | tee -a $LOG
}

log "=== 악성코드 제거 시작 ==="

# 1. 의심스러운 프로세스 종료
log "의심스러운 프로세스 검사..."
suspicious_procs=$(ps aux | grep -E '(nc|ncat|socat).*-l' | grep -v grep | awk '{print $2}')
for pid in $suspicious_procs; do
    log "프로세스 종료: PID $pid"
    kill -9 $pid
done

# 2. 백도어 포트 확인 및 차단
log "백도어 포트 검사..."
netstat -tulpn | grep -E ':(4444|5555|6666|8888|31337)' | while read line; do
    log "의심 포트 발견: $line"
    pid=$(echo $line | awk '{print $NF}' | cut -d'/' -f1)
    if [ ! -z "$pid" ]; then
        kill -9 $pid
        log "프로세스 종료: PID $pid"
    fi
done

# 3. 웹셸 검색 및 제거
log "웹셸 검사..."
find /var/www -name "*.php" -type f -exec grep -l "eval.*base64_decode\|system.*\$_" {} \; | while read file; do
    log "웹셸 발견: $file"
    rm -f "$file"
    log "제거 완료: $file"
done

# 4. Cron job 검사
log "크론 작업 검사..."
crontab -l | grep -E '(curl|wget).*bash' | while read job; do
    log "의심 크론 작업: $job"
    # 크론탭 백업 후 제거
    crontab -l | grep -v "$job" | crontab -
done

# 5. SSH 키 검사
log "SSH authorized_keys 검사..."
for user_home in /home/*; do
    auth_keys="$user_home/.ssh/authorized_keys"
    if [ -f "$auth_keys" ]; then
        # 알 수 없는 키 확인
        while read key; do
            # 정상 키 화이트리스트와 비교
            if ! grep -q "$key" /etc/ssh/known_good_keys; then
                log "의심 SSH 키 발견: $user_home"
                # 백업 후 제거
                cp "$auth_keys" "${auth_keys}.backup"
                grep -v "$key" "$auth_keys" > "${auth_keys}.tmp"
                mv "${auth_keys}.tmp" "$auth_keys"
            fi
        done < "$auth_keys"
    fi
done

# 6. 루트킷 검사
log "루트킷 검사 (rkhunter)..."
rkhunter --update
rkhunter --check --sk | tee -a $LOG

# 7. 파일 무결성 검증
log "파일 무결성 검사..."
tripwire --check | tee -a $LOG

# 8. 임시 디렉토리 청소
log "임시 디렉토리 청소..."
find /tmp /var/tmp -type f -executable -ctime -1 -exec rm -f {} \;

log "=== 악성코드 제거 완료 ==="

7.5 복구 (Recovery)

시스템을 정상 운영 상태로 복원하는 단계입니다. 데이터와 기능을 복구하고, 재발 방지 조치를 구현합니다.

7.5.1 복구 절차

  1. 백업 복원
    • 감염 이전의 깨끗한 백업 식별
    • 백업 무결성 검증
    • 단계적 복원 (중요 시스템 우선)
  2. 시스템 재구축
    • 심각한 침해 시 OS 재설치
    • 보안 설정 적용
    • 최신 패치 적용
  3. 모니터링 강화

요약

사고 대응은 IDS/IPS가 탐지한 위협을 효과적으로 처리하는 체계적인 프로세스입니다. NIST 사고 대응 생명주기는 준비, 탐지 및 분석, 격리/제거/복구, 사후 활동의 4단계로 구성됩니다. 효과적인 대응을 위해서는 IR 매니저, 보안 분석가, 포렌식 전문가, 네트워크 엔지니어 등으로 구성된 전담 팀이 필요합니다.

IDS 경보 발생 시 신속한 분류(Triage)가 중요합니다. 자산 중요도, 공격 심각도, 컨텍스트를 종합하여 P1(Critical)부터 P4(Low)까지 우선순위를 결정하고, 각 우선순위에 맞는 SLA를 적용합니다. 격리 단계에서는 네트워크 차단, VLAN 격리, 계정 비활성화 등을 통해 추가 피해를 방지합니다. 제거 단계에서는 악성코드, 백도어, 웹셸을 완전히 제거하고 모든 크리덴셜을 변경합니다.

복구 과정에서는 깨끗한 백업 복원 또는 시스템 재구축을 수행하며, 철저한 검증을 통해 악성코드가 완전히 제거되었음을 확인합니다. 사후 검토 회의에서는 사고 타임라인을 재구성하고, 잘된 점과 개선점을 분석하여 향후 대응 능력을 향상시킵니다. 법적 요구사항 준수를 위해 증거를 적절히 보존하고, GDPR, 개인정보보호법 등의 신고 의무를 이행해야 합니다. 정기적인 훈련을 통해 팀의 대응 능력을 유지하고 개선합니다.
