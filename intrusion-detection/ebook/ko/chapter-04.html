제4장: 호스트 기반 IDS (HIDS)

4.1 HIDS 개요

호스트 기반 침입 탐지 시스템(Host-based Intrusion Detection System, HIDS)은 개별 컴퓨터나 서버에서 실행되며, 해당 호스트의 활동을 모니터링하여 악의적인 행위를 탐지합니다. NIDS가 네트워크 트래픽을 분석한다면, HIDS는 운영체제 수준의 이벤트, 파일 시스템 변경, 시스템 호출, 로그 파일 등을 분석합니다.

4.1.1 HIDS의 주요 특징

특징 설명 장점
깊은 가시성 OS 수준의 모든 활동 모니터링 암호화된 데이터도 분석 가능
컨텍스트 인식 사용자, 프로세스, 파일 정보 접근 정확한 위협 판단
로컬 탐지 네트워크에 나타나지 않는 공격 탐지 내부자 위협, 권한 상승 탐지
무결성 검증 중요 시스템 파일 변조 탐지 루트킷, 백도어 발견

4.2 파일 무결성 모니터링 (FIM)

파일 무결성 모니터링은 HIDS의 핵심 기능으로, 중요한 시스템 파일과 설정 파일의 무단 변경을 탐지합니다. 이는 악성코드 설치, 시스템 설정 변조, 백도어 삽입 등을 발견하는 데 효과적입니다.

4.2.1 FIM 동작 원리

1. 베이스라인 생성
   ↓
   중요 파일의 해시값(MD5, SHA-256) 계산
   파일 속성 기록 (권한, 소유자, 크기, 수정시간)

2. 정기 검사
   ↓
   현재 파일의 해시값 계산
   베이스라인과 비교

3. 변경 탐지
   ↓
   해시값 불일치 시 경보 발생
   변경 내용 로깅

4. 베이스라인 업데이트
   ↓
   정당한 변경은 베이스라인 갱신

4.2.2 Tripwire 구현 예시

4.2.3 실시간 FIM 구현

#!/usr/bin/env python3
import hashlib
import os
import time
import json
from watchdog.observers import Observer
from watchdog.events import FileSystemEventHandler

class FileIntegrityMonitor(FileSystemEventHandler):
    def __init__(self, baseline_file):
        self.baseline = self.load_baseline(baseline_file)
        self.baseline_file = baseline_file

    def load_baseline(self, filename):
        """베이스라인 로드"""
        if os.path.exists(filename):
            with open(filename, 'r') as f:
                return json.load(f)
        return {}

    def save_baseline(self):
        """베이스라인 저장"""
        with open(self.baseline_file, 'w') as f:
            json.dump(self.baseline, f, indent=2)

    def calculate_hash(self, filepath):
        """파일 해시 계산"""
        sha256 = hashlib.sha256()
        try:
            with open(filepath, 'rb') as f:
                for chunk in iter(lambda: f.read(4096), b''):
                    sha256.update(chunk)
            return sha256.hexdigest()
        except:
            return None

    def get_file_info(self, filepath):
        """파일 메타데이터 수집"""
        stat = os.stat(filepath)
        return {
            'hash': self.calculate_hash(filepath),
            'size': stat.st_size,
            'mtime': stat.st_mtime,
            'mode': oct(stat.st_mode),
            'uid': stat.st_uid,
            'gid': stat.st_gid
        }

    def on_modified(self, event):
        """파일 수정 이벤트 처리"""
        if event.is_directory:
            return

        filepath = event.src_path
        current_info = self.get_file_info(filepath)

        if filepath in self.baseline:
            baseline_info = self.baseline[filepath]

            # 해시 비교
            if current_info['hash'] != baseline_info['hash']:
                self.alert('MODIFIED', filepath, baseline_info, current_info)

            # 권한 변경 확인
            if current_info['mode'] != baseline_info['mode']:
                self.alert('PERMISSION_CHANGE', filepath, baseline_info, current_info)
        else:
            self.alert('NEW_FILE', filepath, None, current_info)

    def on_deleted(self, event):
        """파일 삭제 이벤트 처리"""
        if not event.is_directory:
            filepath = event.src_path
            if filepath in self.baseline:
                self.alert('DELETED', filepath, self.baseline[filepath], None)

    def alert(self, alert_type, filepath, baseline, current):
        """경보 생성"""
        timestamp = time.strftime('%Y-%m-%d %H:%M:%S')
        print(f"[{timestamp}] 경보: {alert_type}")
        print(f"  파일: {filepath}")

        if alert_type == 'MODIFIED':
            print(f"  이전 해시: {baseline['hash']}")
            print(f"  현재 해시: {current['hash']}")
        elif alert_type == 'PERMISSION_CHANGE':
            print(f"  이전 권한: {baseline['mode']}")
            print(f"  현재 권한: {current['mode']}")

        # SIEM으로 전송하거나 로그에 기록
        self.log_alert(alert_type, filepath, baseline, current)

    def log_alert(self, alert_type, filepath, baseline, current):
        """경보 로깅"""
        with open('/var/log/fim_alerts.log', 'a') as f:
            log_entry = {
                'timestamp': time.time(),
                'type': alert_type,
                'file': filepath,
                'baseline': baseline,
                'current': current
            }
            f.write(json.dumps(log_entry) + '\n')

# 사용 예시
if __name__ == '__main__':
    monitor_paths = ['/etc', '/bin', '/sbin', '/usr/bin']

    monitor = FileIntegrityMonitor('baseline.json')
    observer = Observer()

    for path in monitor_paths:
        observer.schedule(monitor, path, recursive=True)

    observer.start()
    print("파일 무결성 모니터링 시작...")

    try:
        while True:
            time.sleep(1)
    except KeyboardInterrupt:
        observer.stop()

    observer.join()

4.3 로그 분석

시스템 로그는 호스트에서 발생하는 모든 활동의 기록을 포함하고 있어, 침입 탐지의 귀중한 정보원입니다. 효과적인 로그 분석을 통해 공격의 징후를 조기에 발견할 수 있습니다.

4.3.1 주요 로그 소스

로그 유형 위치 (Linux) 주요 정보 탐지 가능 위협
인증 로그 /var/log/auth.log 로그인 시도, sudo 사용 무차별 대입 공격, 권한 상승
시스템 로그 /var/log/syslog 시스템 이벤트, 서비스 상태 서비스 중단, 시스템 오류
커널 로그 /var/log/kern.log 커널 메시지, 하드웨어 이벤트 커널 익스플로잇, 드라이버 문제
웹 서버 /var/log/apache2/ HTTP 요청, 에러 웹 공격, SQL Injection
방화벽 /var/log/ufw.log 차단된 연결 포트 스캔, 외부 공격
응용프로그램 /var/log/application.log 앱 이벤트, 에러 앱 취약점 악용

4.3.2 로그 분석 패턴

#!/usr/bin/env python3
import re
from collections import defaultdict
from datetime import datetime, timedelta

class LogAnalyzer:
    def __init__(self):
        self.failed_logins = defaultdict(list)
        self.sudo_usage = defaultdict(list)
        self.alerts = []

    def parse_auth_log(self, log_file):
        """인증 로그 파싱"""
        with open(log_file, 'r') as f:
            for line in f:
                self.analyze_auth_line(line)

    def analyze_auth_line(self, line):
        """인증 로그 라인 분석"""

        # SSH 로그인 실패 탐지
        failed_ssh = re.search(
            r'Failed password for (\w+) from ([\d.]+) port (\d+)',
            line
        )
        if failed_ssh:
            user, ip, port = failed_ssh.groups()
            timestamp = self.extract_timestamp(line)
            self.failed_logins[ip].append({
                'user': user,
                'time': timestamp
            })
            self.check_brute_force(ip)

        # Sudo 사용 탐지
        sudo_use = re.search(
            r'(\w+) : TTY=(\S+) ; PWD=(\S+) ; USER=(\w+) ; COMMAND=(.+)',
            line
        )
        if sudo_use:
            user, tty, pwd, target_user, command = sudo_use.groups()
            timestamp = self.extract_timestamp(line)
            self.sudo_usage[user].append({
                'time': timestamp,
                'command': command,
                'target_user': target_user
            })
            self.check_suspicious_sudo(user, command)

        # 권한 상승 실패
        if 'authentication failure' in line:
            self.create_alert('AUTH_FAILURE', line)

    def check_brute_force(self, ip):
        """무차별 대입 공격 탐지"""
        recent_failures = [
            f for f in self.failed_logins[ip]
            if (datetime.now() - f['time']).seconds < 300  # 5분 이내
        ]

        if len(recent_failures) >= 5:
            self.create_alert(
                'BRUTE_FORCE',
                f"5분 동안 {ip}에서 {len(recent_failures)}회 로그인 실패"
            )

    def check_suspicious_sudo(self, user, command):
        """의심스러운 sudo 사용 탐지"""
        suspicious_commands = [
            '/bin/bash', '/bin/sh', 'passwd', 'useradd',
            'usermod', 'chmod 777', 'iptables -F'
        ]

        for sus_cmd in suspicious_commands:
            if sus_cmd in command:
                self.create_alert(
                    'SUSPICIOUS_SUDO',
                    f"사용자 {user}가 의심스러운 명령 실행: {command}"
                )

    def extract_timestamp(self, line):
        """로그에서 타임스탬프 추출"""
        # 예: Jan 15 10:30:45
        match = re.search(
            r'(\w+)\s+(\d+)\s+(\d+):(\d+):(\d+)',
            line
        )
        if match:
            # 간단한 파싱 (실제로는 더 정교한 처리 필요)
            return datetime.now()
        return datetime.now()

    def create_alert(self, alert_type, message):
        """경보 생성"""
        alert = {
            'timestamp': datetime.now(),
            'type': alert_type,
            'message': message
        }
        self.alerts.append(alert)
        print(f"[경보] {alert_type}: {message}")

# 사용 예시
analyzer = LogAnalyzer()
analyzer.parse_auth_log('/var/log/auth.log')

4.4 프로세스 모니터링

프로세스 모니터링은 시스템에서 실행 중인 프로세스를 추적하여 악성 프로세스나 비정상적인 행위를 탐지합니다.

4.4.1 모니터링 대상

  • 신규 프로세스 생성: 알 수 없는 실행 파일 시작
  • 프로세스 계층: 비정상적인 부모-자식 관계
  • 리소스 사용: 과도한 CPU, 메모리, 네트워크 사용
  • 프로세스 인젝션: 다른 프로세스의 메모리 조작
  • 권한 수준: 예상치 못한 높은 권한으로 실행
  • 네트워크 연결: 프로세스의 외부 통신

4.4.2 Linux 프로세스 모니터링

#!/usr/bin/env python3
import psutil
import os
import time

class ProcessMonitor:
    def __init__(self):
        self.known_processes = set()
        self.whitelist = {'/usr/bin/', '/bin/', '/sbin/'}

    def monitor(self):
        """프로세스 모니터링 시작"""
        while True:
            current_processes = set()

            for proc in psutil.process_iter(['pid', 'name', 'exe', 'cmdline']):
                try:
                    pinfo = proc.info
                    pid = pinfo['pid']
                    current_processes.add(pid)

                    # 새로운 프로세스 탐지
                    if pid not in self.known_processes:
                        self.analyze_new_process(proc, pinfo)

                    # 기존 프로세스 분석
                    self.analyze_process_behavior(proc, pinfo)

                except (psutil.NoSuchProcess, psutil.AccessDenied):
                    continue

            # 종료된 프로세스 정리
            self.known_processes = current_processes
            time.sleep(5)

    def analyze_new_process(self, proc, pinfo):
        """새 프로세스 분석"""
        exe_path = pinfo.get('exe', '')
        cmdline = ' '.join(pinfo.get('cmdline', []))

        # 화이트리스트에 없는 경로에서 실행
        if exe_path and not any(exe_path.startswith(wl) for wl in self.whitelist):
            self.alert('SUSPICIOUS_LOCATION',
                      f"의심스러운 위치에서 실행: {exe_path}")

        # 의심스러운 명령어
        suspicious_patterns = [
            'nc -l',           # 백도어 리스너
            'bash -i',         # 인터랙티브 셸
            '/dev/shm/',       # 임시 디렉토리 실행
            'wget http',       # 다운로드
            'curl http',
            'python -c',       # 인라인 파이썬 실행
        ]

        for pattern in suspicious_patterns:
            if pattern in cmdline:
                self.alert('SUSPICIOUS_COMMAND',
                          f"의심스러운 명령: {cmdline}")

    def analyze_process_behavior(self, proc, pinfo):
        """프로세스 행위 분석"""
        try:
            # CPU 사용률 체크 (암호화폐 채굴 의심)
            cpu_percent = proc.cpu_percent(interval=1)
            if cpu_percent > 80:
                self.alert('HIGH_CPU',
                          f"{pinfo['name']} 프로세스 CPU 사용률 {cpu_percent}%")

            # 네트워크 연결 확인
            connections = proc.connections()
            for conn in connections:
                if conn.status == 'ESTABLISHED':
                    # 비정상적인 포트로 연결
                    if conn.raddr and conn.raddr.port in [4444, 5555, 6666, 8888]:
                        self.alert('SUSPICIOUS_CONNECTION',
                                  f"{pinfo['name']}가 {conn.raddr.ip}:{conn.raddr.port}에 연결")

            # 열린 파일 확인
            open_files = proc.open_files()
            for file in open_files:
                # 민감한 파일 접근
                if '/etc/shadow' in file.path or '/etc/passwd' in file.path:
                    if pinfo['name'] not in ['passwd', 'chpasswd', 'usermod']:
                        self.alert('SENSITIVE_FILE_ACCESS',
                                  f"{pinfo['name']}가 {file.path} 접근")

        except (psutil.NoSuchProcess, psutil.AccessDenied):
            pass

    def alert(self, alert_type, message):
        """경보 생성"""
        timestamp = time.strftime('%Y-%m-%d %H:%M:%S')
        print(f"[{timestamp}] {alert_type}: {message}")

# 사용
monitor = ProcessMonitor()
monitor.monitor()

4.5 시스템 호출 모니터링

시스템 호출(System Call) 모니터링은 프로세스가 커널에 요청하는 작업을 추적하여 악의적인 행위를 탐지합니다. 이는 가장 낮은 수준의 모니터링으로, 루트킷이나 고급 악성코드도 탐지할 수 있습니다.

4.5.1 주요 시스템 호출

카테고리 시스템 호출 정상 사용 악의적 사용
파일 접근 open, read, write 일반 파일 I/O 민감한 파일 무단 접근
프로세스 fork, exec, clone 새 프로세스 생성 악성 프로세스 실행
네트워크 socket, connect, bind 네트워크 통신 C&C 서버 연결
권한 setuid, setgid 권한 변경 권한 상승 공격
메모리 ptrace, mmap 디버깅, 메모리 할당 프로세스 인젝션

4.5.2 auditd를 이용한 시스템 호출 감사

# auditd 규칙 설정 (/etc/audit/rules.d/ids.rules)

4.6 레지스트리 모니터링 (Windows)

Windows 환경에서는 레지스트리 변경 모니터링이 중요합니다. 많은 악성코드가 지속성 확보나 설정 변경을 위해 레지스트리를 수정합니다.

4.6.1 주요 모니터링 레지스트리 키

레지스트리 경로 용도 위협
HKLM\Software\Microsoft\Windows\CurrentVersion\Run 시스템 시작 시 실행 지속성 확보
HKCU\Software\Microsoft\Windows\CurrentVersion\Run 사용자 로그인 시 실행 자동 실행 악성코드
HKLM\System\CurrentControlSet\Services 서비스 등록 악성 서비스 설치
HKLM\Software\Microsoft\Windows NT\CurrentVersion\Winlogon 로그온 프로세스 크리덴셜 탈취
HKLM\System\CurrentControlSet\Control\SecurityProviders 보안 제공자 인증 우회

4.6.2 PowerShell을 이용한 레지스트리 모니터링

# PowerShell 레지스트리 모니터링 스크립트

# 모니터링할 레지스트리 키
$monitorPaths = @(
    "HKLM:\Software\Microsoft\Windows\CurrentVersion\Run",
    "HKCU:\Software\Microsoft\Windows\CurrentVersion\Run",
    "HKLM:\System\CurrentControlSet\Services"
)

# 베이스라인 생성
$baseline = @{}
foreach ($path in $monitorPaths) {
    if (Test-Path $path) {
        $baseline[$path] = Get-ItemProperty -Path $path
    }
}

# 변경 모니터링
while ($true) {
    Start-Sleep -Seconds 60

    foreach ($path in $monitorPaths) {
        if (Test-Path $path) {
            $current = Get-ItemProperty -Path $path
            $baselineData = $baseline[$path]

            # 새로운 값 탐지
            foreach ($prop in $current.PSObject.Properties) {
                if ($prop.Name -notin @("PSPath", "PSParentPath", "PSChildName", "PSDrive", "PSProvider")) {
                    if (-not $baselineData.PSObject.Properties[$prop.Name]) {
                        Write-Warning "새로운 레지스트리 항목: $path\$($prop.Name) = $($prop.Value)"
                        # SIEM으로 전송하거나 로그 기록
                    }
                }
            }

            # 삭제된 값 탐지
            foreach ($prop in $baselineData.PSObject.Properties) {
                if ($prop.Name -notin @("PSPath", "PSParentPath", "PSChildName", "PSDrive", "PSProvider")) {
                    if (-not $current.PSObject.Properties[$prop.Name]) {
                        Write-Warning "레지스트리 항목 삭제: $path\$($prop.Name)"
                    }
                }
            }

            # 베이스라인 업데이트
            $baseline[$path] = $current
        }
    }
}

요약

호스트 기반 IDS(HIDS)는 개별 시스템 수준에서 심층적인 보안 모니터링을 제공합니다. NIDS가 네트워크 트래픽을 분석하는 반면, HIDS는 운영체제 수준의 이벤트, 파일 시스템 변경, 프로세스 활동, 시스템 호출을 모니터링하여 암호화된 통신이나 내부자 위협도 탐지할 수 있습니다.

파일 무결성 모니터링(FIM)은 HIDS의 핵심 기능으로, Tripwire나 OSSEC와 같은 도구를 사용하여 중요 시스템 파일의 무단 변경을 탐지합니다. 로그 분석을 통해 인증 시도, sudo 사용, 시스템 이벤트를 추적하여 무차별 대입 공격이나 권한 상승 시도를 발견할 수 있습니다. 프로세스 모니터링은 의심스러운 프로세스 생성, 비정상적인 리소스 사용, 악의적인 네트워크 연결을 식별합니다.

시스템 호출 모니터링은 가장 낮은 수준의 감시로, auditd를 사용하여 커널 수준의 작업을 추적합니다. Windows 환경에서는 레지스트리 모니터링과 Sysmon을 통한 상세한 시스템 활동 로깅이 중요합니다. OSSEC와 같은 통합 HIDS 솔루션은 FIM, 로그 분석, 루트킷 탐지, 능동적 대응을 하나의 플랫폼에서 제공합니다. 대규모 환경에서는 도메인 컨트롤러, 데이터베이스 서버 등 중요 시스템부터 우선적으로 HIDS를 배포해야 합니다.
