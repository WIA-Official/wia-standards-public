Reference Standards and Implementation Notes

This chapter follows the global health data interoperability framework codified in the underlying ALLOW sources. Implementations SHOULD treat the following as the single source of truth and use the WIA-LEG-008 data formats as the cross-vendor interoperability layer.

  • HL7 FHIR R5 (release 5) — clinical data resources
  • SNOMED CT International — clinical terminology
  • LOINC 2.76 — laboratory and clinical observations
  • WHO ICD-11 — disease classification
  • ISO/IEC 27001:2022 — information security management
  • ISO/IEC 27018:2019 — PII protection in public clouds
  • OpenAPI Specification 3.1, JSON Schema 2020-12
  • Sigstore (DSSE envelope, Rekor transparency log)

Practical Implementation Checklist

  • Record integrity — every report is signed via Sigstore DSSE. Tampering is detectable without trusting an intermediary.
  • Retention — keep signed evidence for at least seven years to satisfy ISO/IEC 17065:2012 audit expectations.
  • Versioning — follow Semantic Versioning so consumers can plan migration windows when normative requirements change.
  • Negative-result reporting — a skipped indicator without justification is treated as a failed indicator at audit.
  • De-identification — apply differential privacy or k-anonymity (k≥5) per ISO/IEC 27559:2022 before publishing aggregations.

Federation patterns

Cross-border health data exchange follows one of three patterns: national hub aggregating provincial agencies; cross-border corridor between two or more national operators; multi-agency cluster federated through a cluster-coordination manifest. Each pattern ships with reference test vectors so that implementations can validate their deployment matches the canonical pattern.
