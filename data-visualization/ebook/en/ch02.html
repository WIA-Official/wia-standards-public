Chapter 2: Chart Types & Selection Guide

Comprehensive Guide to Choosing the Right Visualization for Your Data

🎯 Chapter Overview

This chapter provides a comprehensive taxonomy of chart types and practical guidance for selecting the most appropriate visualization for different data types and communication objectives. We explore basic charts, statistical visualizations, hierarchical displays, and specialized diagrams.

2.1 The Chart Selection Framework

Choosing the right chart type is crucial for effective data communication. The selection should be driven by three primary factors: the type of data you have, the relationship you want to show, and the message you want to convey.

Data Types and Their Implications

Data Type Description Examples Suitable Charts
Categorical Discrete groups or categories Product types, regions, departments Bar, pie, tree map
Numerical Continuous quantitative values Sales figures, temperature, revenue Line, area, scatter, histogram
Temporal Time-ordered data Daily metrics, quarterly trends Line, area, stream graph
Spatial Geographic or coordinate data Locations, regions, coordinates Maps, choropleth, bubble map
Hierarchical Nested or tree-structured data Org charts, file systems, taxonomies Tree map, sunburst, dendrogram
Network Nodes and connections Social networks, dependencies Network graph, chord diagram

Analytical Questions Guide Chart Selection

The question you're trying to answer should drive your chart selection. Here's a framework for matching questions to chart types:

Question Type Purpose Recommended Charts
What is the value? Show specific values Table, KPI card, metric display
How does it compare? Compare values across categories Bar chart, grouped bar, dot plot
How does it change? Show trends over time Line chart, area chart, slope graph
What is the distribution? Show spread and frequency Histogram, box plot, violin plot
What is the composition? Show parts of a whole Pie chart, stacked bar, tree map
What is the relationship? Show correlation or connection Scatter plot, bubble chart, heatmap

2.2 Basic Chart Types

Basic charts form the foundation of data visualization. These tried-and-true chart types are familiar to most audiences and effective for common analytical tasks.

Bar Charts: The Workhorse of Comparison

Bar charts use length to encode quantitative values, making them highly effective for comparing values across categories. They are one of the most versatile and widely used chart types.

When to Use Bar Charts

  • Comparing categories: Sales by product, performance by department
  • Showing rankings: Top 10 customers, bottom performers
  • Displaying discrete time periods: Quarterly results, yearly comparisons
  • Part-to-whole comparisons: Stacked bars for composition over categories

Bar Chart Variations

Variation Use Case Best Practice
Vertical Bar Standard comparison, time series Use for temporal data (time flows left to right)
Horizontal Bar Long category labels, rankings Better for many categories or long labels
Grouped Bar Multiple series comparison Limit to 2-3 groups for clarity
Stacked Bar Part-to-whole with categories Use for composition, not precise value comparison
100% Stacked Proportional composition Emphasizes percentage rather than absolute values

💡 Best Practice: Bar Chart Axes

Always start bar chart axes at zero. Starting at a non-zero value distorts the visual comparison of bar lengths and can mislead viewers about the magnitude of differences. If the range is large, consider using a break in the axis (clearly indicated) or log scale instead.

Line Charts: Revealing Trends and Patterns

Line charts connect data points with lines, making them ideal for showing trends, patterns, and changes over continuous intervals, especially time.

When to Use Line Charts

  • Time series data: Stock prices, temperature over time, website traffic
  • Continuous data: Any data where intermediate values are meaningful
  • Multiple series comparison: Comparing trends across groups
  • Showing volatility: Fluctuations and variability over time

Line Chart Design Principles

  1. Limit number of lines: Maximum 5-7 lines for readability
  2. Use distinct colors: Ensure lines are easily distinguishable
  3. Direct labeling: Label lines directly instead of using a legend when possible
  4. Highlight key series: Use brighter colors or thicker lines for primary data
  5. Show context: Include reference lines, benchmarks, or target values

Pie Charts: Showing Parts of a Whole

Pie charts use angles and area to represent proportions of a whole. While popular, they should be used judiciously due to limitations in human perception of angles and areas.

When Pie Charts Work Well

  • Few categories: 2-5 slices maximum for clarity
  • Clear proportions: When exact values matter less than relative size
  • Simple composition: One dominant category with smaller others
  • Familiar context: Audience expects percentages that sum to 100%

⚠️ When to Avoid Pie Charts

  • More than 5-6 categories (use bar chart instead)
  • Precise value comparison needed (angles are hard to compare)
  • Similar-sized slices (differences become invisible)
  • Comparing multiple pies (extremely difficult to compare angles across charts)

Pie Chart Alternatives

  • Donut chart: Removes center, reduces area comparison difficulty
  • Bar chart: More accurate for value comparison
  • Tree map: Better for many categories
  • Waffle chart: Shows proportions using grid of squares

Scatter Plots: Exploring Relationships

Scatter plots use position to encode two variables, revealing correlations, clusters, and outliers. They are essential for exploratory data analysis and relationship discovery.

When to Use Scatter Plots

  • Correlation analysis: Understanding relationship between two variables
  • Outlier detection: Finding unusual data points
  • Cluster identification: Discovering natural groupings
  • Distribution examination: Seeing the spread of bivariate data

Enhancing Scatter Plots

Enhancement Purpose Implementation
Third variable via size Show magnitude (bubble chart) Scale point size to represent value
Third variable via color Show category or continuous value Color points by group or gradient
Trend line Show overall pattern Add linear, polynomial, or LOESS fit
Reference lines Show thresholds or targets Add horizontal/vertical lines
Density indication Handle overplotting Use transparency or 2D density estimation

2.3 Statistical Visualizations

Statistical charts are designed to reveal distribution properties, variability, and statistical relationships in data. These are essential tools for data analysis and scientific communication.

Histograms: Understanding Distributions

Histograms bin continuous data into intervals and display frequency counts, revealing the shape, central tendency, and spread of distributions.

Key Considerations for Histograms

  • Bin width selection: Critical parameter affecting interpretation
    • Too wide: Hides important details and variations
    • Too narrow: Creates noise and obscures overall pattern
    • Rule of thumb: Start with √n bins for n data points
  • Bin boundaries: Ensure bins don't create artificial patterns
  • Context: Add mean, median, or quartile lines for reference
  • Overlays: Can add density curve for smooth distribution shape

Box Plots: Compact Statistical Summary

Box plots (box-and-whisker plots) display five key statistics: minimum, first quartile, median, third quartile, and maximum. They efficiently summarize distributions and facilitate comparisons.

Box Plot Components

Component Represents Statistical Meaning
Box bottom First quartile (Q1) 25th percentile
Box middle line Median (Q2) 50th percentile
Box top Third quartile (Q3) 75th percentile
Box height Interquartile range (IQR) Spread of middle 50% of data
Whiskers Data range (within 1.5×IQR) Expected range without outliers
Points beyond whiskers Outliers Values > 1.5×IQR from quartiles

When to Use Box Plots

  • Comparing distributions: Multiple groups side-by-side
  • Identifying outliers: Automatically flags unusual values
  • Compact summary: When space is limited
  • Statistical audience: Viewers familiar with quartiles and IQR

Heatmaps: Visualizing Matrices

Heatmaps use color to encode values in a matrix format, making them ideal for revealing patterns in large datasets, correlations, and temporal patterns.

Heatmap Applications

  • Correlation matrices: Showing relationships between many variables
  • Temporal patterns: Hour-of-day by day-of-week activity
  • Genomic data: Gene expression across samples
  • Confusion matrices: Classification algorithm performance

Heatmap Design Best Practices

  1. Choose appropriate color scale:
    • Sequential (one color): For values from low to high
    • Diverging (two colors): For values above/below a midpoint
    • Categorical: For discrete categories (use sparingly)
  2. Ensure sufficient contrast: Light to dark should be visually distinct
  3. Add cell annotations: Include actual values when appropriate
  4. Cluster if meaningful: Reorder rows/columns to reveal patterns
  5. Include scale legend: Always show color-to-value mapping

2.4 Hierarchical and Network Visualizations

Some data has inherent structure - hierarchies, trees, or networks. Specialized visualizations are designed to reveal these structural relationships effectively.

Tree Maps: Hierarchical Proportions

Tree maps use nested rectangles to show hierarchical data, with rectangle size representing quantitative values. They efficiently use space to show both hierarchy and magnitude.

Tree Map Strengths and Limitations

Strengths Limitations
Space-efficient for many items Difficult to see hierarchical structure
Shows both hierarchy and size Hard to compare similar-sized rectangles
Can encode additional dimension via color Aspect ratio varies (some rectangles very elongated)
Interactive drill-down works well Ordering within level can be arbitrary

Network Graphs: Visualizing Connections

Network graphs (or node-link diagrams) show entities as nodes and relationships as connecting edges. They are essential for understanding complex systems of interactions.

Network Layout Algorithms

Algorithm Description Best For
Force-directed Simulates physical forces between nodes General networks, community detection
Hierarchical Arranges nodes in levels Tree structures, organizational charts
Circular Places nodes around a circle Chord diagrams, showing all connections
Radial Emanates from central node Ego networks, centered hierarchies

2.5 Specialized Chart Types

Beyond basic charts, specialized visualizations address specific analytical needs and data types. Understanding when to use these can significantly enhance your analytical capabilities.

Sankey Diagrams: Flow and Transfer

Sankey diagrams show flows between nodes, with flow width representing magnitude. They excel at visualizing transfers, conversions, and multi-stage processes.

Sankey Diagram Use Cases

  • Energy flow: Production to consumption pathways
  • Material flow: Supply chain visualization
  • User journeys: Website navigation paths
  • Budget allocation: How funds flow through departments
  • Conversion funnels: Drop-off at each stage

Parallel Coordinates: Multivariate Exploration

Parallel coordinates plot multiple variables as vertical axes with lines connecting values for each observation. They enable exploration of multivariate data and identification of patterns across many dimensions.

Parallel Coordinates Applications

  • Multivariate filtering: Interactive brushing across dimensions
  • Pattern detection: Finding groups with similar profiles
  • Optimization visualization: Pareto frontier exploration
  • High-dimensional data: More than 3-4 variables

Small Multiples: Comparing Many Charts

Small multiples (also called trellis charts or faceting) create series of similar charts with consistent scales, enabling comparison across categories or time periods.

Small Multiples Design Principles

  1. Consistent scales: Use same axes across all panels
  2. Logical ordering: Arrange by time, magnitude, or category
  3. Minimal decoration: Reduce clutter in each panel
  4. Clear labels: Each panel must be identifiable
  5. Appropriate size: Large enough to see patterns, small enough to compare

2.6 Chart Selection Decision Tree

To help navigate the many chart options, use this decision tree framework:

1. Start with your data structure:

  • One variable → Histogram, density plot, or single-value metric
  • Two variables → Scatter, line, or bar chart
  • Three+ variables → Consider multivariate techniques or multiple charts
  • Hierarchical → Tree map, sunburst, or dendrogram
  • Network → Network graph or chord diagram

2. Consider your analytical goal:

  • Compare → Bar chart (categorical) or box plot (distributions)
  • Show trend → Line or area chart
  • Show composition → Stacked bar, pie, or tree map
  • Show relationship → Scatter plot or heatmap
  • Show distribution → Histogram, box plot, or violin plot

3. Account for your audience:

  • General public → Stick to basic, familiar charts
  • Business executives → Focus on KPIs, trends, and comparisons
  • Technical analysts → Can use more sophisticated statistical charts
  • Domain experts → Leverage domain-specific visualization conventions

Summary

🎯 Key Takeaways

  • Chart selection depends on data type, analytical question, and audience
  • Bar charts excel at comparisons; line charts show trends; scatter plots reveal relationships
  • Use statistical charts (box plots, histograms) for distribution analysis
  • Hierarchical data requires specialized visualizations like tree maps or sunbursts
  • Network visualizations show connections and flows between entities
  • When in doubt, start with simple, familiar charts and add complexity only if needed
  • Always prioritize clarity and accuracy over visual novelty
