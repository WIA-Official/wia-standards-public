5.2 D3.js: Data-Driven Documents

D3.js is the foundational library for web-based data visualization. It provides powerful primitives for binding data to DOM elements and applying data-driven transformations.

D3.js Core Concepts

  • Selections: jQuery-like API for selecting and manipulating DOM elements
  • Data binding: Join data arrays to DOM elements with enter/update/exit pattern
  • Scales: Map data values to visual properties (position, color, size)
  • Axes: Generate tick marks and labels automatically
  • Shapes: SVG path generators for lines, areas, arcs, symbols
  • Transitions: Animate changes smoothly over time
  • Modules: Composable sub-libraries for specific tasks

Example: D3.js Bar Chart

import * as d3 from 'd3';

const data = [12, 19, 28, 15, 21, 34];
const width = 800, height = 400;
const margin = {top: 20, right: 20, bottom: 30, left: 40};

const svg = d3.select('#chart')
  .append('svg')
  .attr('width', width)
  .attr('height', height);

const x = d3.scaleBand()
  .domain(d3.range(data.length))
  .range([margin.left, width - margin.right])
  .padding(0.1);

const y = d3.scaleLinear()
  .domain([0, d3.max(data)])
  .range([height - margin.bottom, margin.top]);

svg.selectAll('rect')
  .data(data)
  .join('rect')
  .attr('x', (d, i) => x(i))
  .attr('y', d => y(d))
  .attr('width', x.bandwidth())
  .attr('height', d => y(0) - y(d))
  .attr('fill', '#10B981');

5.3 Chart.js: Simple Yet Flexible

Chart.js provides a high-level API for creating common chart types with Canvas rendering. It's known for simplicity, good performance, and extensive customization options.

Chart.js Strengths

  • Simple API: Minimal configuration for standard charts
  • Canvas rendering: Better performance for many data points than SVG
  • Responsive: Built-in responsiveness and mobile support
  • Animation: Smooth default animations for all interactions
  • Plugins: Extensible architecture for custom functionality
  • Zero dependencies: Lightweight with no external requirements

Example: Chart.js Line Chart

import Chart from 'chart.js/auto';

const ctx = document.getElementById('myChart').getContext('2d');
const chart = new Chart(ctx, {
  type: 'line',
  data: {
    labels: ['Jan', 'Feb', 'Mar', 'Apr', 'May', 'Jun'],
    datasets: [{
      label: 'Sales',
      data: [12, 19, 3, 5, 2, 3],
      borderColor: '#10B981',
      backgroundColor: 'rgba(16, 185, 129, 0.1)',
      tension: 0.4
    }]
  },
  options: {
    responsive: true,
    plugins: {
      legend: { display: true },
      tooltip: { enabled: true }
    },
    scales: {
      y: { beginAtZero: true }
    }
  }
});

5.4 Plotly: Interactive Scientific Visualization

Plotly excels at scientific and statistical visualizations with built-in interactivity. Available in JavaScript, Python, and R with consistent APIs across platforms.

Plotly Advantages

  • 3D visualization: Native support for 3D scatter, surface, and mesh plots
  • Statistical charts: Box plots, violin plots, contours out-of-the-box
  • Built-in interactivity: Zoom, pan, hover without additional code
  • Multi-language: Python, R, MATLAB, Julia bindings
  • Dash framework: Build dashboards with Python backend

5.7 Integration with WIA-DATA-011

WIA-DATA-011 provides standard adapters for popular libraries, ensuring consistent behavior and API compatibility across different visualization tools.

WIA Adapter Pattern

import { WIAAdapter } from '@wia/visualization';
import * as d3 from 'd3';

// Create WIA-compliant D3 visualizations
const wiaD3 = new WIAAdapter('d3', d3);

const chart = wiaD3.createChart({
  type: 'bar',
  container: '#chart',
  data: salesData,
  config: {
    // WIA-standard configuration
    title: 'Monthly Sales',
    colors: WIAAdapter.standardColors,
    accessibility: {
      ariaLabel: 'Bar chart showing monthly sales',
      keyboardNav: true
    }
  }
});

// Works consistently across different libraries
const chartJs = new WIAAdapter('chartjs', Chart);
const plotly = new WIAAdapter('plotly', Plotly);

5.8 Choosing the Right Library

Selection criteria should consider technical requirements, team expertise, and long-term maintenance.

Decision Framework

  1. Project requirements: Standard charts vs custom visualizations?
  2. Data volume: How many data points? Need for optimization?
  3. Interactivity needs: Static, basic interaction, or complex exploration?
  4. Team skills: JavaScript expertise level? Learning curve acceptable?
  5. Performance requirements: Real-time updates? Mobile devices?
  6. Maintenance: Active development? Community support? Documentation quality?
