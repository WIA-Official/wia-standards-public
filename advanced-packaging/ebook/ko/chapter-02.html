2장: 패키지 아키텍처

2.1 2.5D 패키징

2.5D 패키징은 실리콘 인터포저를 사용하여 여러 다이를 수평으로 연결하는 기술입니다. 다이들은 동일 평면에 위치하며 인터포저를 통해 고밀도 배선으로 연결됩니다.

2.5.1 실리콘 인터포저

// 2.5D 인터포저 사양
interface SiliconInterposer {
    // 물리적 특성
    thickness: number;     // µm (일반적으로 50-100)
    size: { x: number; y: number };  // mm

    // TSV 특성
    tsv: {
        diameter: number;   // µm (5-10)
        pitch: number;      // µm (40-50)
        density: number;    // TSV/mm²
        depth: number;      // µm
    };

    // RDL 층
    rdl: {
        layers: number;     // 4-6
        lineWidth: number;  // µm (0.4-2)
        spacing: number;    // µm (0.4-2)
        material: 'Cu';
    };

    // 다이 배치
    dieSlots: DieSlot[];
}

// 대표적 2.5D 기술
const technologies2_5D = {
    CoWoS: {
        vendor: 'TSMC',
        interposerSize: '2500mm²',
        maxDies: 6,
        rdlLayers: 6
    },
    EMIB: {
        vendor: 'Intel',
        type: 'embedded bridge',
        interposerSize: 'localized',
        maxDies: 4
    }
};
CoWoS의 장점: TSMC의 CoWoS(Chip-on-Wafer-on-Substrate)는 HBM과 GPU/CPU를 연결하는 대표적인 2.5D 플랫폼으로, TB/s 단위의 메모리 대역폭을 제공합니다.

2.2 3D 패키징

3D 패키징은 다이를 수직으로 적층하여 최단 거리 연결과 최소 풋프린트를 달성합니다:

기술 본딩 방식 피치 적용
마이크로범프 솔더 40-55µm HBM, 로직
하이브리드 본딩 Cu-Cu 직접 9-10µm 이미지센서, SoIC
TCB 열압착 20-40µm HBM 
// 3D 패키지 스택 정의
interface Stack3D {
    layers: StackLayer[];
    totalHeight: number;  // µm
    interconnect: {
        type: 'microbump' | 'hybrid_bonding' | 'tcb';
        pitch: number;
        density: number;  // connections/mm²
    };
    thermalPath: ThermalConfig;
}

interface StackLayer {
    position: number;  // bottom = 0
    die: Die;
    thickness: number;  // µm
    tsvPresent: boolean;
    orientation: 'face-up' | 'face-down';
}

// HBM 스택 예시
const hbmStack: Stack3D = {
    layers: [
        { position: 0, die: baseDie, thickness: 720, tsvPresent: true },
        { position: 1, die: dramDie, thickness: 40, tsvPresent: true },
        { position: 2, die: dramDie, thickness: 40, tsvPresent: true },
        { position: 3, die: dramDie, thickness: 40, tsvPresent: true },
        { position: 4, die: dramDie, thickness: 40, tsvPresent: false }
    ],
    totalHeight: 880,
    interconnect: { type: 'microbump', pitch: 55, density: 330 }
};

2.3 Fan-Out 패키징

Fan-Out 패키징은 다이 크기보다 넓은 영역에 I/O를 분포시켜 더 많은 연결과 더 나은 전기적 특성을 제공합니다:

2.3.1 FOWLP (Fan-Out Wafer Level Packaging)

  • 웨이퍼 레벨에서 재성형(reconstitution)
  • 에폭시 몰드 컴파운드로 다이 확장
  • RDL로 I/O 재배치
  • WLP 수준의 얇은 프로파일

2.3.2 FOPLP (Fan-Out Panel Level Packaging)

  • 대형 패널(600x600mm)에서 처리
  • 더 낮은 단위 비용
  • 대량 생산에 유리

2.4 칩렛 아키텍처

// 칩렛 시스템 정의
interface ChipletSystem {
    chiplets: Chiplet[];
    interposer?: Interposer;
    substrate: Substrate;
    dieTodie: D2DInterface;
}

interface Chiplet {
    id: string;
    function: 'compute' | 'io' | 'memory' | 'analog';
    processNode: string;  // e.g., '5nm', '14nm'
    size: { x: number; y: number };  // mm
    tdp: number;  // W
    interfaces: D2DPort[];
}

interface D2DPort {
    type: 'UCIe' | 'BoW' | 'AIB';
    bandwidth: number;  // GB/s
    lanes: number;
    direction: 'bidirectional' | 'tx' | 'rx';
}

// 예: AMD EPYC 칩렛 구성
const epycChiplets = {
    ccd: { count: 8, node: '7nm', function: 'compute' },
    iod: { count: 1, node: '14nm', function: 'io' },
    d2dInterface: 'Infinity Fabric',
    totalCores: 64
};
칩렛의 경제성: 단일 모놀리식 다이 대신 작은 칩렛으로 분할하면 수율이 향상되고, 각 칩렛에 최적의 공정을 적용할 수 있어 총 비용이 절감됩니다.

2.5 SiP (System in Package)

SiP는 다양한 부품(능동/수동)을 단일 패키지에 통합합니다:

  • AP + 메모리 + PMIC 통합
  • RF 모듈 (안테나 포함)
  • 센서 허브
  • IoT 솔루션

2.6 아키텍처 선택 가이드

요구사항 권장 아키텍처
최대 메모리 대역폭 2.5D + HBM
최소 폼팩터 3D 적층, FOWLP
이종 통합 칩렛 + 인터포저
저비용 대량 생산 FOPLP
RF/아날로그 통합 SiP, AiP

2.7 아키텍처 설계 고려사항

첨단 패키지 아키텍처를 선택할 때는 다양한 기술적, 경제적 요소를 종합적으로 고려해야 합니다. 성능 요구사항, 비용 목표, 생산 수량, 열 관리 능력, 그리고 공급망 준비 상태가 모두 중요한 판단 기준이 됩니다.

설계 초기 결정의 중요성: 패키지 아키텍처 선택은 제품 개발 초기에 이루어져야 합니다. 이 결정은 다이 설계, 인터페이스 사양, 테스트 전략, 그리고 전체 시스템 아키텍처에 광범위한 영향을 미칩니다. 설계 변경 비용은 프로젝트가 진행될수록 기하급수적으로 증가하므로, 충분한 사전 검토와 시뮬레이션을 통해 최적의 아키텍처를 선택하는 것이 중요합니다.

2.7.1 성능 대 비용 트레이드오프

  • 2.5D 인터포저: 높은 대역폭, 중간 비용, 검증된 기술
  • 3D 적층: 최고 밀도, 높은 비용, 열 관리 도전
  • 칩렛: 유연한 구성, 수율 이점, 에코시스템 의존
  • Fan-Out: 얇은 프로파일, 경쟁력 있는 비용, 제한된 I/O
