5.1 RESTful API 설계 원칙

WIA-DIGITAL_FUNERAL 표준의 PHASE 2는 디지털 장례 서비스 간의 상호운용성을 보장하기 위한 API 인터페이스를 정의합니다. RESTful 아키텍처를 기반으로 하며, 모든 구현체는 이 인터페이스를 준수해야 합니다. REST(Representational State Transfer)는 웹 서비스 설계의 아키텍처 스타일로, 단순성, 확장성, 그리고 독립성을 특징으로 합니다.

디지털 장례 서비스 API는 추모관 생성, 조의록 관리, 미디어 업로드, 라이브스트리밍 제어 등 다양한 기능을 제공합니다. 각 기능은 명확하게 정의된 엔드포인트를 통해 접근되며, HTTP 메서드(GET, POST, PUT, DELETE)를 적절히 활용하여 의미론적으로 명확한 인터페이스를 제공합니다. 또한 JSON 형식을 주요 데이터 교환 포맷으로 사용하여 다양한 프로그래밍 언어와의 호환성을 보장합니다.

5.1.1 REST 설계 원칙

REST의 6가지 제약 조건

  • 클라이언트-서버 분리: 사용자 인터페이스와 데이터 저장소의 관심사를 분리하여 이식성과 확장성을 향상시킵니다.
  • 무상태성(Stateless): 각 요청은 독립적이며 필요한 모든 정보를 포함합니다. 서버는 클라이언트의 상태를 저장하지 않습니다.
  • 캐시 가능성: 응답은 캐시 가능 여부를 명시하여 성능과 확장성을 개선합니다.
  • 계층화 시스템: 클라이언트는 중간 계층(로드 밸런서, 캐시, 게이트웨이)의 존재를 알 수 없어야 합니다.
  • 균일한 인터페이스: 일관된 방식으로 자원을 식별하고 조작합니다.
  • 주문형 코드(선택사항): 서버가 클라이언트에 실행 가능한 코드를 전송할 수 있습니다.

5.1.2 API 기본 구조

// WIA-DIGITAL_FUNERAL API 기본 구조

// 베이스 URL
const BASE_URL = 'https://api.wia.org/v1/wia-digital-funeral';

// API 클라이언트 초기화
class DigitalFuneralClient {
    constructor(apiKey) {
        this.apiKey = apiKey;
        this.baseUrl = BASE_URL;
    }

    // 공통 요청 헤더
    getHeaders() {
        return {
            'Authorization': `Bearer ${this.apiKey}`,
            'Content-Type': 'application/json',
            'X-WIA-Standard': 'WIA-DIGITAL_FUNERAL',
            'X-WIA-Version': '1.0',
            'Accept': 'application/json'
        };
    }

    // 추모관 생성
    async createMemorial(data) {
        const response = await fetch(`${this.baseUrl}/memorials`, {
            method: 'POST',
            headers: this.getHeaders(),
            body: JSON.stringify({
                deceased: {
                    name: data.name,
                    birthDate: data.birthDate,
                    deathDate: data.deathDate,
                    biography: data.biography
                },
                settings: {
                    visibility: data.visibility || 'public',
                    theme: data.theme || 'serene',
                    language: data.language || 'ko'
                },
                features: {
                    tributeWall: true,
                    condolenceBook: true,
                    photoGallery: true,
                    livestreaming: data.enableLivestreaming || false
                }
            })
        });

        if (!response.ok) {
            throw new Error(`API Error: ${response.status}`);
        }

        return await response.json();
    }

    // 추모관 조회
    async getMemorial(id) {
        const response = await fetch(`${this.baseUrl}/memorials/${id}`, {
            method: 'GET',
            headers: this.getHeaders()
        });

        if (!response.ok) {
            throw new Error(`Memorial not found: ${id}`);
        }

        return await response.json();
    }

    // 조의록 작성
    async addCondolence(memorialId, message) {
        const response = await fetch(
            `${this.baseUrl}/memorials/${memorialId}/condolences`,
            {
                method: 'POST',
                headers: this.getHeaders(),
                body: JSON.stringify({
                    author: message.author,
                    email: message.email,
                    content: message.content,
                    relationship: message.relationship,
                    isAnonymous: message.isAnonymous || false
                })
            }
        );

        return await response.json();
    }
}

5.3 인증 및 권한 부여

디지털 장례 서비스는 매우 민감한 개인 정보를 다루기 때문에 강력한 인증 및 권한 관리 시스템이 필수적입니다. WIA-DIGITAL_FUNERAL 표준은 다층 보안 모델을 채택하여 API 키 인증, OAuth 2.0, JWT(JSON Web Token) 등 다양한 인증 메커니즘을 지원합니다. 또한 역할 기반 접근 제어(RBAC)를 통해 세밀한 권한 관리를 제공합니다.

5.3.2 권한 레벨 및 범위

역할 권한 범위 설명 일반적 사용 사례
Guest memorials:read_public 공개 추모관 조회만 가능 일반 방문자
User memorials:read, condolences:write 조회 및 조의 메시지 작성 조문객
Memorial Creator memorials:create, memorials:read 추모관 생성 및 자신의 추모관 관리 유족, 지인
Memorial Admin memorials:*, condolences:moderate 특정 추모관의 모든 권한 추모관 관리자
Service Admin *:* 시스템의 모든 기능에 대한 권한 서비스 운영자 
// 역할 기반 접근 제어 구현 예시

class AuthorizationMiddleware {
    constructor() {
        this.permissions = {
            'memorial_admin': [
                'memorials:read',
                'memorials:write',
                'memorials:delete',
                'condolences:read',
                'condolences:moderate',
                'media:manage'
            ],
            'user': [
                'memorials:read_public',
                'condolences:write',
                'condolences:read'
            ],
            'guest': [
                'memorials:read_public'
            ]
        };
    }

    // 권한 확인
    hasPermission(userRole, requiredPermission) {
        const userPermissions = this.permissions[userRole] || [];

        // 와일드카드 권한 확인
        if (userPermissions.includes('*:*')) {
            return true;
        }

        // 정확한 권한 매칭
        if (userPermissions.includes(requiredPermission)) {
            return true;
        }

        // 부분 매칭 (예: memorials:* 는 memorials:read, memorials:write 모두 포함)
        const [resource, action] = requiredPermission.split(':');
        if (userPermissions.includes(`${resource}:*`)) {
            return true;
        }

        return false;
    }

    // Express 미들웨어
    requirePermission(permission) {
        return (req, res, next) => {
            const user = req.user; // JWT에서 추출된 사용자 정보

            if (!user) {
                return res.status(401).json({
                    error: 'Unauthorized',
                    message: '인증이 필요합니다.'
                });
            }

            if (!this.hasPermission(user.role, permission)) {
                return res.status(403).json({
                    error: 'Forbidden',
                    message: '이 작업을 수행할 권한이 없습니다.',
                    required: permission,
                    userRole: user.role
                });
            }

            next();
        };
    }
}

// 사용 예시
const auth = new AuthorizationMiddleware();

// 추모관 삭제는 memorial_admin 역할만 가능
app.delete('/memorials/:id',
    auth.requirePermission('memorials:delete'),
    async (req, res) => {
        // 삭제 로직...
    }
);

5.4 웹훅 (Webhook) 시스템

웹훅은 특정 이벤트가 발생했을 때 서버에서 클라이언트로 실시간 알림을 전송하는 메커니즘입니다. 디지털 장례 서비스에서는 새로운 조의 메시지 작성, 추모관 방문, 라이브스트리밍 시작 등의 이벤트를 실시간으로 통지하는 데 웹훅을 활용합니다. 이를 통해 유족은 즉각적으로 알림을 받을 수 있고, 다른 시스템과의 연동도 원활하게 이루어집니다.

5.4.1 웹훅 이벤트 타입

이벤트 타입 설명 페이로드 주요 필드 우선순위
memorial.created 새 추모관 생성됨 memorialId, deceased, creator Normal
memorial.updated 추모관 정보 수정됨 memorialId, changes, updatedBy Low
memorial.deleted 추모관 삭제됨 memorialId, deletedBy, reason High
condolence.posted 새 조의 메시지 작성됨 memorialId, condolenceId, author High
media.uploaded 사진/동영상 업로드됨 memorialId, mediaId, type Normal
livestream.started 라이브스트리밍 시작됨 memorialId, streamUrl, scheduledTime Critical
visitor.milestone 방문자 이정표 도달 (100명, 1000명 등) memorialId, visitorCount, milestone Normal

5.4.2 웹훅 설정 및 구현

// 웹훅 등록
async function registerWebhook() {
    const webhook = await client.post('/webhooks', {
        url: 'https://yourapp.com/webhooks/wia-funeral',
        events: [
            'condolence.posted',
            'livestream.started',
            'memorial.created'
        ],
        secret: 'whsec_your_webhook_secret_key',
        description: '유족 알림 시스템',
        active: true
    });

    console.log('웹훅 등록됨:', webhook.id);
    return webhook;
}

// 웹훅 수신 서버 구현 (Node.js/Express)
const express = require('express');
const crypto = require('crypto');
const app = express();

app.post('/webhooks/wia-funeral', express.json(), async (req, res) => {
    // 1. 서명 검증 (보안)
    const signature = req.headers['x-wia-signature'];
    const timestamp = req.headers['x-wia-timestamp'];
    const payload = JSON.stringify(req.body);

    const expectedSignature = crypto
        .createHmac('sha256', process.env.WEBHOOK_SECRET)
        .update(`${timestamp}.${payload}`)
        .digest('hex');

    if (signature !== `sha256=${expectedSignature}`) {
        console.error('웹훅 서명 검증 실패');
        return res.status(401).send('Invalid signature');
    }

    // 2. 타임스탬프 검증 (재생 공격 방지)
    const currentTime = Math.floor(Date.now() / 1000);
    if (Math.abs(currentTime - parseInt(timestamp)) > 300) { // 5분 이내
        console.error('웹훅 타임스탬프 만료');
        return res.status(401).send('Timestamp expired');
    }

    // 3. 이벤트 처리
    const event = req.body;

    try {
        switch (event.type) {
            case 'condolence.posted':
                await handleNewCondolence(event.data);
                break;

            case 'livestream.started':
                await notifySubscribers(event.data);
                break;

            case 'memorial.created':
                await setupMemorialNotifications(event.data);
                break;

            default:
                console.log('알 수 없는 이벤트 타입:', event.type);
        }

        // 4. 성공 응답 (중요: 200 반환하지 않으면 재전송됨)
        res.status(200).json({ received: true });

    } catch (error) {
        console.error('웹훅 처리 오류:', error);
        // 5xx 에러는 재시도됨, 4xx는 재시도 안 됨
        res.status(500).json({ error: 'Processing failed' });
    }
});

// 조의 메시지 처리 핸들러
async function handleNewCondolence(data) {
    const { memorialId, condolence } = data;

    // 유족에게 이메일/푸시 알림
    await sendNotification({
        to: data.memorial.familyEmail,
        subject: '새로운 조의 메시지가 도착했습니다',
        template: 'new_condolence',
        data: {
            deceasedName: data.memorial.deceased.name,
            authorName: condolence.author.name,
            message: condolence.content
        }
    });

    // 통계 업데이트
    await updateMemorialStats(memorialId, {
        condolenceCount: { $inc: 1 }
    });
}

async function notifySubscribers(data) {
    const { memorialId, streamUrl } = data;

    // 구독자 목록 조회
    const subscribers = await getMemorialSubscribers(memorialId);

    // 대량 알림 전송

            body: `${data.memorial.deceased.name}님의 장례식이 시작되었습니다.`,
            data: { streamUrl, memorialId }
        })
    ));
}

5.5 오류 처리 및 응답 형식

일관된 오류 처리는 API의 사용성과 디버깅 용이성을 크게 향상시킵니다. WIA-DIGITAL_FUNERAL 표준은 표준화된 오류 응답 형식을 정의하며, HTTP 상태 코드를 의미론적으로 올바르게 사용합니다. 모든 오류 응답은 기계가 읽을 수 있는 오류 코드와 사람이 이해할 수 있는 메시지를 함께 제공합니다.

class APIError extends Error {
    constructor(code, message, statusCode = 500, details = null) {
        super(message);
        this.code = code;
        this.statusCode = statusCode;
        this.details = details;

                message: this.message,
                details: this.details,
                timestamp: this.timestamp
            }
        };
    }
}

function validateMemorialData(data) {

            400,
            errors
        );
