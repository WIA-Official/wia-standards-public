Chapter 2: Capability Declaration (WIA-LLM-CAPABILITY)

"Before AI models can collaborate, they must first understand each other's strengths."

2.1 What is Capability Declaration?

The WIA-LLM-CAPABILITY standard defines how AI models describe their abilities. Just as a professional has a resume, each AI model has a Capability Document that formally declares:

  • What domains it excels in
  • What languages it supports
  • What capability level it has
  • What tools it can use
  • What constraints it operates under

2.2 The Capability Document Structure

A complete Capability Document contains these sections:

CapabilityDocument
├── @context           # JSON-LD context
├── id                 # Document identifier
├── type               # "LLMCapabilityDocument"
├── version            # Protocol version
├── identity           # Model identification
├── capability         # Abilities declaration
├── constraints        # Operating limits
├── metadata           # Additional information
└── signature          # Cryptographic proof

2.3 Model Identity

Every AI model must have a unique identity using the DID format:

identity: {
    did: "did:wia:llm:anthropic:claude-3-opus",
    name: "Claude 3 Opus",
    provider: "Anthropic",
    version: "2024-01",
    description: "Advanced reasoning and analysis model"
}

DID Format Breakdown

Component Description Example
did: DID scheme prefix did:
wia: WIA method identifier wia:
llm: LLM namespace llm:
provider Model provider anthropic, openai, local
model-id Specific model claude-3-opus, gpt-4

2.4 Capability Levels

The standard defines four capability levels:

Level Name Description Examples
1 Basic Simple tasks, pattern matching, basic generation FAQ bots, simple assistants
2 Reasoning Complex analysis, multi-step reasoning, synthesis General-purpose LLMs
3 Specialist Domain expertise, specialized knowledge Code Llama, legal models
4 Advanced Research-level, novel problem solving, creativity GPT-4, Claude 3 Opus

2.6 Language Support

Declare supported languages with proficiency scores:

languages: [
    { code: "en", proficiency: 0.98, native: true },
    { code: "ko", proficiency: 0.92 },
    { code: "ja", proficiency: 0.88 },
    { code: "zh", proficiency: 0.85 }
]

Language codes follow ISO 639-1 standard (en, ko, ja, zh, es, fr, de, etc.)

2.7 Modality Support

Specify what input/output modalities the model supports:

ModalityInputOutputDescription
text[OK][OK]Text processing
image[OK]variesImage understanding
audio[OK]variesAudio processing
video[OK]-Video understanding
code[OK][OK]Code execution
modalities: {
    input: ["text", "image", "code"],
    output: ["text", "code"]
}

2.8 Tool Capabilities

Declare tools the AI model can use:

Standard Tool Types

ToolDescription
web_searchSearch the internet
code_interpreterExecute code
file_analysisRead and analyze files
image_generationCreate images
database_queryQuery databases
api_callCall external APIs
tools: [
    {
        type: "STANDARD",
        name: "web_search",
        description: "Search the web for information"
    },
    {
        type: "STANDARD",
        name: "code_interpreter",
        description: "Execute Python code"
    }
]

2.9 Constraints

Declare operating constraints and limits:

constraints: {
    contextWindow: 200000,       // Max input tokens
    maxOutputTokens: 4096,       // Max output tokens
    rateLimits: {
        requestsPerMinute: 60,
        tokensPerMinute: 100000
    },
    contentPolicy: {
        allowsNSFW: false,
        allowsMedicalAdvice: false,
        allowsLegalAdvice: false
    },
    availability: {
        regions: ["global"],
        uptime: 0.999
    }
}

2.11 Capability Registration

To register capabilities with a WIA network:

REST API

POST /api/v1/capability
Content-Type: application/json
Authorization: Bearer {did-auth-token}

{CapabilityDocument}

Response: 201 Created
Location: /api/v1/capability/{capability-id}

WebSocket

{
    "type": "CAPABILITY_REGISTER",
    "payload": {CapabilityDocument}
}
