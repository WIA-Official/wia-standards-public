제5장: 알고리즘과 매칭

"적합한 작업에 적합한 AI - 지능적 매칭이 효과적인 협력의 핵심입니다."

5.1 핵심 알고리즘 개요

WIA LLM 상호운용 표준은 여러 알고리즘을 정의합니다:

  • 능력 매칭 - 작업에 가장 적합한 AI 찾기
  • 메시지 라우팅 - 효율적인 메시지 전달
  • 작업 분해 - 복잡한 작업을 하위 작업으로 분할
  • 합의 구축 - AI 간 합의 도달
  • 신뢰 점수 - AI 신뢰성 평가
  • 부하 분산 - 작업을 고르게 분배

5.2 능력 매칭 알고리즘

요구 사항을 기반으로 주어진 작업에 가장 적합한 AI 모델을 찾습니다.

매칭 가중치

요소가중치설명
도메인 매칭40%필요한 전문 분야
수준 매칭25%최소 능력 수준
언어 매칭20%필요한 언어 지원
도구 매칭15%필요한 도구 능력

알고리즘: CalculateCapabilityMatch

function CalculateCapabilityMatch(requirements, capability):

    # 도메인 점수 (40% 가중치)
    domain_score = 0
    for required_domain in requirements.domains:
        for cap_domain in capability.domains:
            if cap_domain.code == required_domain:
                domain_score += cap_domain.proficiency
    domain_score = domain_score / len(requirements.domains)

    # 수준 점수 (25% 가중치)
    if capability.level >= requirements.min_level:
        level_score = capability.level / 4.0
    else:
        level_score = 0  # 최소 요건 미충족

    # 언어 점수 (20% 가중치)
    language_score = 0
    for lang in capability.languages:
        if lang.code == requirements.language:
            language_score = lang.proficiency
            break

    # 도구 점수 (15% 가중치)
    tool_score = 0
    if requirements.tools:
        matches = 0
        for tool in requirements.tools:
            if tool in capability.tools:
                matches += 1
        tool_score = matches / len(requirements.tools)
    else:
        tool_score = 1.0  # 도구 필요 없음

    # 가중 총점 계산
    total_score = (
        domain_score * 0.40 +
        level_score * 0.25 +
        language_score * 0.20 +
        tool_score * 0.15
    )

    return total_score

계산 예시

요구 사항:
- 도메인: [CODE, ANALYSIS]
- 최소 수준: 3
- 언어: ko
- 도구: [code_interpreter]

후보: Claude 3 Opus
- 도메인: CODE(0.94), ANALYSIS(0.96)
- 수준: 4
- 언어: ko(0.90)
- 도구: [code_interpreter, file_analysis]

계산:
- 도메인: (0.94 + 0.96) / 2 = 0.95
- 수준: 4 / 4 = 1.0
- 언어: 0.90
- 도구: 1/1 = 1.0

총점 = 0.95*0.40 + 1.0*0.25 + 0.90*0.20 + 1.0*0.15
     = 0.38 + 0.25 + 0.18 + 0.15
     = 0.96 (96% 매칭)

5.3 능력 핸드셰이크 프로토콜

협력 전에 AI 모델들이 상호 능력을 검증합니다:

단계 1: 개시자가 능력 쿼리 전송
AI-A ---> AI-B: CAPABILITY_REQUEST

단계 2: 응답자가 능력 문서 공유
AI-B ---> AI-A: CAPABILITY_RESPONSE

단계 3: 개시자가 호환성 평가
AI-A: CalculateCapabilityMatch()

단계 4: 호환되면 협력 진행
AI-A ---> AI-B: HANDSHAKE_ACCEPT

단계 5: 세션 수립
AI-A <---> AI-B: 메시지 준비 완료

5.4 메시지 라우팅 알고리즘

직접 라우팅

알려진 수신자에게 직접 전송합니다.

function DirectRoute(message, recipient_did):
    endpoint = ResolveEndpoint(recipient_did)
    return SendToEndpoint(message, endpoint)

능력 기반 라우팅

메시지 요구 사항을 기반으로 최적의 수신자를 찾습니다.

function CapabilityRoute(message, requirements):
    # 능력 레지스트리 쿼리
    candidates = QueryCapabilities(requirements)

    # 매칭 점수로 정렬
    sorted_candidates = SortByMatchScore(candidates)

    # 성공할 때까지 각 후보 시도
    for candidate in sorted_candidates:
        result = DirectRoute(message, candidate.did)
        if result.success:
            return result

    return Error("적합한 수신자를 찾을 수 없음")

부하 분산 라우팅

사용 가능한 모델 간에 메시지를 분배합니다.

function LoadBalancedRoute(message, requirements):
    candidates = QueryCapabilities(requirements)

    # 최소 매칭 점수로 필터링
    qualified = [c for c in candidates if c.score >= 0.7]

    # 가용성과 부하로 가중치 부여
    for candidate in qualified:
        candidate.weight = (
            candidate.score * 0.5 +
            candidate.availability * 0.3 +
            (1 - candidate.load) * 0.2
        )

    # 가중 랜덤 선택
    selected = WeightedRandomSelect(qualified)
    return DirectRoute(message, selected.did)

브로드캐스트 라우팅

모든 연합 멤버에게 전송합니다.

function BroadcastRoute(message, federation):
    results = []
    for member in federation.members:
        result = DirectRoute(message, member.did)
        results.append(result)
    return AggregateResults(results)

5.5 작업 분해 알고리즘

복잡한 작업을 관리 가능한 하위 작업으로 분할합니다:

function DecomposeTask(task, federation):
    # 작업 복잡도 분석
    complexity = AnalyzeComplexity(task)

    if complexity == "LOW":
        return [task]  # 분해 불필요

    # 필요한 능력 식별
    required_caps = ExtractRequirements(task)

    # 능력을 기반으로 하위 작업 생성
    subtasks = []

    for capability in required_caps:
        # 이 능력에 가장 적합한 멤버 찾기
        best_member = FindBestMatch(federation.members, capability)

        subtask = {
            "id": GenerateId(),
            "description": GenerateSubtaskDescription(task, capability),
            "assignee": best_member.did,
            "capability": capability,
            "dependencies": []
        }
        subtasks.append(subtask)

    # 의존성 수립
    subtasks = EstablishDependencies(subtasks, task.decomposition_strategy)

    return subtasks

function EstablishDependencies(subtasks, strategy):
    if strategy == "PARALLEL":
        # 의존성 없음
        return subtasks

    elif strategy == "SEQUENTIAL":
        # 각각이 이전에 의존
        for i in range(1, len(subtasks)):
            subtasks[i].dependencies = [subtasks[i-1].id]
        return subtasks

    elif strategy == "HIERARCHICAL":
        # 의존성 트리 구축
        return BuildDependencyTree(subtasks)

    elif strategy == "ADAPTIVE":
        # 런타임에 의존성 결정
        return subtasks

5.6 합의 알고리즘

다수결 합의

function MajorityConsensus(votes, threshold=0.5):
    yes_count = count(v for v in votes if v.vote == YES)
    no_count = count(v for v in votes if v.vote == NO)
    total = yes_count + no_count

    if total == 0:
        return UNDECIDED

    yes_ratio = yes_count / total

    if yes_ratio > threshold:
        return YES
    elif yes_ratio < (1 - threshold):
        return NO
    else:
        return UNDECIDED

가중 합의

function WeightedConsensus(votes, threshold=0.6):
    yes_weight = 0
    no_weight = 0
    total_weight = 0

    for vote in votes:
        if vote.vote == ABSTAIN:
            continue

        weight = vote.confidence * vote.trust_score
        total_weight += weight

        if vote.vote == YES:
            yes_weight += weight
        else:
            no_weight += weight

    if total_weight == 0:
        return UNDECIDED

    yes_ratio = yes_weight / total_weight

    if yes_ratio >= threshold:
        return YES
    elif (1 - yes_ratio) >= threshold:
        return NO
    else:
        return UNDECIDED

토론 합의

function DebateConsensus(question, members, max_rounds=5):
    positions = {}

    # 라운드 1: 초기 입장
    for member in members:
        positions[member.did] = GetInitialPosition(member, question)

    for round in range(2, max_rounds + 1):
        # 입장과 반박 공유
        for member in members:
            other_positions = {k:v for k,v in positions.items()
                            if k != member.did}

            # 멤버가 다른 입장 고려
            refined = RefinePosition(member, positions[member.did],
                                    other_positions)
            positions[member.did] = refined

        # 수렴 확인
        convergence = CalculateConvergence(positions)
        if convergence >= 0.8:
            return DetermineConsensus(positions)

    # 최대 라운드 도달
    return DetermineConsensus(positions)

5.7 신뢰 점수 알고리즘

function CalculateTrustScore(model):
    # 구성 요소 가중치
    ACCURACY_WEIGHT = 0.40
    RESPONSE_TIME_WEIGHT = 0.15
    CONSISTENCY_WEIGHT = 0.25
    PEER_REVIEW_WEIGHT = 0.20

    # 응답 시간 정규화 (낮을수록 좋음)
    max_acceptable_time = 30000  # 30초
    time_score = max(0, 1 - (model.avg_response_time / max_acceptable_time))

    # 가중 신뢰 점수 계산
    trust_score = (
        model.historical_accuracy * ACCURACY_WEIGHT +
        time_score * RESPONSE_TIME_WEIGHT +
        model.consistency_score * CONSISTENCY_WEIGHT +
        model.peer_review_avg * PEER_REVIEW_WEIGHT
    )

    return trust_score

function UpdateTrustScore(model, interaction_result):
    # 오래된 데이터에 대한 감쇠 계수
    DECAY = 0.95

    # 새 상호작용의 가중치
    NEW_WEIGHT = 0.1

    new_score = (
        model.trust_score * DECAY * (1 - NEW_WEIGHT) +
        interaction_result.score * NEW_WEIGHT
    )

    model.trust_score = clamp(new_score, 0, 1)
    return model.trust_score

5.8 악성 AI 탐지

function DetectMaliciousAI(model, recent_interactions):
    THRESHOLD_ACCURACY = 0.3
    THRESHOLD_CONSISTENCY = 0.2
    THRESHOLD_CONFLICT = 0.5

    # 지속적으로 틀린 답변 확인
    recent_accuracy = CalculateRecentAccuracy(recent_interactions)
    if recent_accuracy < THRESHOLD_ACCURACY:
        return SUSPICIOUS, "낮은 정확도"

    # 일관성 없는 응답 확인
    consistency = CalculateConsistency(recent_interactions)
    if consistency < THRESHOLD_CONSISTENCY:
        return SUSPICIOUS, "일관성 없는 응답"

    # 충돌 유발 행동 확인
    conflict_rate = CalculateConflictRate(recent_interactions)
    if conflict_rate > THRESHOLD_CONFLICT:
        return SUSPICIOUS, "높은 충돌률"

    return TRUSTED, "문제 없음"

5.9 서킷 브레이커 패턴

class CircuitBreaker:
    CLOSED = "CLOSED"       # 정상 운영
    OPEN = "OPEN"           # 요청 차단
    HALF_OPEN = "HALF_OPEN" # 복구 테스트

    def __init__(self, threshold=5, timeout=30000):
        self.state = CLOSED
        self.failure_count = 0
        self.threshold = threshold
        self.timeout = timeout
        self.last_failure_time = None

    def execute(self, operation):
        if self.state == OPEN:
            if time_since(self.last_failure_time) > self.timeout:
                self.state = HALF_OPEN
            else:
                raise CircuitOpenError()

        try:
            result = operation()
            self.on_success()
            return result
        except Exception as e:
            self.on_failure()
            raise e

    def on_success(self):
        self.failure_count = 0
        self.state = CLOSED

    def on_failure(self):
        self.failure_count += 1
        self.last_failure_time = now()

        if self.failure_count >= self.threshold:
            self.state = OPEN

5.10 부하 분산 알고리즘

function BalanceLoad(task, candidates):
    # 각 후보에 대한 유효 용량 계산
    for candidate in candidates:
        candidate.effective_capacity = (
            candidate.capacity * (1 - candidate.current_load) *
            candidate.trust_score
        )

    # 유효 용량으로 정렬 (높은 순)
    sorted_candidates = sort(candidates,
                            key=lambda c: c.effective_capacity,
                            reverse=True)

    # 가장 여유 있는 후보 선택
    selected = sorted_candidates[0]

    # 부하 추적 업데이트
    selected.current_load += task.estimated_load

    return selected
