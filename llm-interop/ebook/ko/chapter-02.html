제2장: 능력 선언 (WIA-LLM-CAPABILITY)

"AI 모델들이 협력하기 전에, 먼저 서로의 강점을 이해해야 합니다."

2.1 능력 선언이란?

WIA-LLM-CAPABILITY 표준은 AI 모델이 자신의 능력을 설명하는 방법을 정의합니다. 전문가가 이력서를 가지고 있듯이, 각 AI 모델은 공식적으로 선언하는 능력 문서를 가집니다:

  • 어떤 분야에서 뛰어난지
  • 어떤 언어를 지원하는지
  • 어떤 능력 수준인지
  • 어떤 도구를 사용할 수 있는지
  • 어떤 제약 조건이 있는지

2.2 능력 문서 구조

완전한 능력 문서는 다음 섹션을 포함합니다:

CapabilityDocument
├── @context           # JSON-LD 컨텍스트
├── id                 # 문서 식별자
├── type               # "LLMCapabilityDocument"
├── version            # 프로토콜 버전
├── identity           # 모델 식별
├── capability         # 능력 선언
├── constraints        # 운영 제한
├── metadata           # 추가 정보
└── signature          # 암호화 증명

2.3 모델 신원

모든 AI 모델은 DID 형식을 사용한 고유 신원이 있어야 합니다:

identity: {
    did: "did:wia:llm:anthropic:claude-3-opus",
    name: "Claude 3 Opus",
    provider: "Anthropic",
    version: "2024-01",
    description: "고급 추론 및 분석 모델"
}

DID 형식 분해

구성 요소 설명 예시
did: DID 스킴 접두사 did:
wia: WIA 메서드 식별자 wia:
llm: LLM 네임스페이스 llm:
provider 모델 제공자 anthropic, openai, local
model-id 특정 모델 claude-3-opus, gpt-4

2.4 능력 수준

표준은 네 가지 능력 수준을 정의합니다:

수준 이름 설명 예시
1 기본 단순 작업, 패턴 매칭, 기본 생성 FAQ 봇, 단순 어시스턴트
2 추론 복잡한 분석, 다단계 추론, 종합 범용 LLM
3 전문가 도메인 전문성, 특화된 지식 Code Llama, 법률 모델
4 고급 연구 수준, 새로운 문제 해결, 창의성 GPT-4, Claude 3 Opus

2.5 도메인 전문성

AI 모델은 표준화된 도메인 코드를 사용하여 전문 분야를 선언합니다:

도메인 코드 참조

코드도메인설명
CODE프로그래밍소프트웨어 개발, 디버깅
MATH수학계산, 증명, 통계
MEDICAL의료의료 지식, 진단 지원
LEGAL법률법률 분석, 규정 준수
FINANCIAL금융금융 분석, 거래
SCIENTIFIC과학연구, 과학적 방법
CREATIVE창작글쓰기, 예술, 디자인
ANALYSIS분석데이터 분석, 연구
EDUCATION교육교육, 튜터링
LANGUAGE언어번역, 언어학

도메인 선언 예시

domains: [
    { code: "CODE", proficiency: 0.95 },
    { code: "ANALYSIS", proficiency: 0.90 },
    { code: "MATH", proficiency: 0.85 }
]

2.6 언어 지원

숙련도 점수와 함께 지원 언어를 선언합니다:

languages: [
    { code: "en", proficiency: 0.98, native: true },
    { code: "ko", proficiency: 0.92 },
    { code: "ja", proficiency: 0.88 },
    { code: "zh", proficiency: 0.85 }
]

언어 코드는 ISO 639-1 표준을 따릅니다 (en, ko, ja, zh, es, fr, de 등)

2.7 모달리티 지원

모델이 지원하는 입력/출력 모달리티를 지정합니다:

모달리티입력출력설명
text[OK][OK]텍스트 처리
image[OK]다양이미지 이해
audio[OK]다양오디오 처리
video[OK]-비디오 이해
code[OK][OK]코드 실행
modalities: {
    input: ["text", "image", "code"],
    output: ["text", "code"]
}

2.8 도구 능력

AI 모델이 사용할 수 있는 도구를 선언합니다:

표준 도구 유형

도구설명
web_search인터넷 검색
code_interpreter코드 실행
file_analysis파일 읽기 및 분석
image_generation이미지 생성
database_query데이터베이스 쿼리
api_call외부 API 호출
tools: [
    {
        type: "STANDARD",
        name: "web_search",
        description: "웹에서 정보 검색"
    },
    {
        type: "STANDARD",
        name: "code_interpreter",
        description: "Python 코드 실행"
    }
]

2.9 제약 조건

운영 제약 조건과 제한을 선언합니다:

constraints: {
    contextWindow: 200000,       // 최대 입력 토큰
    maxOutputTokens: 4096,       // 최대 출력 토큰
    rateLimits: {
        requestsPerMinute: 60,
        tokensPerMinute: 100000
    },
    contentPolicy: {
        allowsNSFW: false,
        allowsMedicalAdvice: false,
        allowsLegalAdvice: false
    },
    availability: {
        regions: ["global"],
        uptime: 0.999
    }
}

2.11 능력 등록

WIA 네트워크에 능력을 등록하려면:

REST API

POST /api/v1/capability
Content-Type: application/json
Authorization: Bearer {did-auth-token}

{CapabilityDocument}

응답: 201 Created
Location: /api/v1/capability/{capability-id}

WebSocket

{
    "type": "CAPABILITY_REGISTER",
    "payload": {CapabilityDocument}
}

2.12 장 요약

  • [OK] 능력 문서는 AI 모델의 능력을 설명
  • [OK] DID 기반 신원은 고유 식별 보장
  • [OK] 기본부터 고급까지 네 가지 능력 수준
  • [OK] 도메인 코드가 전문 분야 표준화
  • [OK] 제약 조건이 운영 제한 정의
  • [OK] 등록을 통해 다른 모델이 발견 가능
