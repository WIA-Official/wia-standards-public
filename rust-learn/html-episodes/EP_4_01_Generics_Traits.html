✅ 제네릭 사용

✅ With Generics

fn largest<T: PartialOrd>(
list: &[T]
) -> &T {
let mut largest = &list[0];
for item in list {
if item > largest { largest = item; }
}
largest
}
// 모든 타입에 동작! ✨

Generic 문법

제네릭 사용 위치

Where to Use Generics

// 1. 함수에서
fn swap<T>(a: T, b: T) -> (T, T) { (b, a) }
// 2. 구조체에서
struct Point<T> {
x: T,
y: T,
}
// 3. Enum에서
enum Option<T> { Some(T), None }
enum Result<T, E> { Ok(T), Err(E) }
// 4. impl 블록에서
impl<T> Point<T> {
fn x(&self) -> &T { &self.x }
}

트레이트란? - 공유된 행동 정의

What is a Trait? - Defining Shared Behavior

Trait은 다른 언어의 Interface와 비슷합니다.
"이 타입이 할 수 있는 것"을 정의합니다. Trait is similar to Interface in other languages.
It defines "what this type can do".

// 트레이트 정의
trait Summary {
fn summarize(&self) -> String;
}
// 트레이트 구현
struct Article { title: String, author: String }
impl Summary for Article {
fn summarize(&self) -> String {
format!("{} by {}", self.title, self.author)
}
}

Derive

자동 구현: #[derive(...)]

Automatic Implementation: #[derive(...)]

// 컴파일러가 자동으로 트레이트 구현 생성!
#[derive(Debug, Clone, PartialEq, Default)]
struct User {
name: String,
age: u32,
active: bool,
}
fn main() {
let user = User::default(); // Default
println!("{:?}", user); // Debug
let copy = user.clone(); // Clone
assert!(user == copy); // PartialEq
}

💡 derive는 보일러플레이트를 없애줍니다. 하지만 모든 필드가 해당 트레이트를 구현해야 합니다! 💡 derive eliminates boilerplate. But all fields must implement the trait!
