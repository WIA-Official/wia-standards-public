스레드 생성

스레드 생성하기

Spawning Threads

use std::thread;
use std::time::Duration;
fn main() {
// 새 스레드 생성
let handle = thread::spawn(|| {
for i in 1..5 {
println!("스레드: {}", i);
thread::sleep(Duration::from_millis(100));
}
});
// 메인 스레드
for i in 1..3 {
println!("메인: {}", i);
}
handle.join().unwrap(); // 스레드 완료 대기
}

move 클로저

move 클로저로 소유권 이동

Moving Ownership with move Closure

use std::thread;
fn main() {
let data = vec![1, 2, 3];
// ❌ 이건 안됨! data의 수명이 불확실
// let handle = thread::spawn(|| println!("{:?}", data));
// ✅ move로 소유권 이동!
let handle = thread::spawn(move || {
println!("{:?}", data);
});
// println!("{:?}", data); // ❌ data는 이동됨!
handle.join().unwrap();
}

💡 스레드는 언제 끝날지 모르므로, move로 데이터 소유권을 명확히 해야 합니다. 💡 Since threads may outlive the main function, use move to clarify data ownership.

채널

채널로 스레드 간 통신

Thread Communication with Channels

use std::sync::mpsc; // Multi-Producer, Single-Consumer
use std::thread;
fn main() {
// 채널 생성: (송신자, 수신자)
let (tx, rx) = mpsc::channel();
thread::spawn(move || {
let msg = String::from("안녕!");
tx.send(msg).unwrap(); // 전송!
// println!("{}", msg); // ❌ msg 소유권 이동됨!
});
let received = rx.recv().unwrap(); // 수신 (블로킹)
println!("받음: {}", received);
}

다중 생산자

여러 스레드에서 메시지 보내기

Sending Messages from Multiple Threads

use std::sync::mpsc;
use std::thread;
let (tx, rx) = mpsc::channel();
let tx2 = tx.clone(); // 송신자 복제!
thread::spawn(move || {
tx.send("Thread 1").unwrap();
});
thread::spawn(move || {
tx2.send("Thread 2").unwrap();
});
// 모든 메시지 수신
for msg in rx {
println!("받음: {}", msg);
}

Arc + Mutex - 안전한 공유 상태

Arc + Mutex - Safe Shared State

use std::sync::{Arc, Mutex};
use std::thread;
let counter = Arc::new(Mutex::new(0));
let mut handles = vec![];
for _ in 0..10 {
let counter = Arc::clone(&counter);
let handle = thread::spawn(move || {
let mut num = counter.lock().unwrap();
*num += 1; // 안전하게 증가!
});
handles.push(handle);
}
for h in handles { h.join().unwrap(); }
println!("결과: {}", *counter.lock().unwrap()); // 10

Send & Sync

Send와 Sync 트레이트

Send and Sync Traits

Send

스레드 간 소유권 이동이 가능

대부분의 Rust 타입은 Send
예외: Rc<T> (스레드 안전 X) Ownership transfer between threads

Most Rust types are Send
Exception: Rc<T> (not thread-safe)

Sync

여러 스레드에서 참조 공유가 가능

T가 Sync면 &T는 Send
예외: RefCell<T> (런타임 검사) Reference sharing across threads

If T is Sync, &T is Send
Exception: RefCell<T> (runtime check)

💡 컴파일러가 자동으로 검사! 스레드 안전하지 않은 코드는 컴파일 에러 💡 The compiler checks automatically! Non-thread-safe code causes compile errors
