Chapter 02

차등 프라이버시 (Differential Privacy) 기초

Mathematical Foundations of Differential Privacy

2.1 차등 프라이버시의 개념

차등 프라이버시(Differential Privacy, DP)는 2006년 Cynthia Dwork에 의해 제안된 프라이버시 보존 기술로, 데이터 분석 결과에서 개인의 참여 여부를 수학적으로 숨기는 것을 보장합니다. 이 기술의 핵심 아이디어는 "당신이 데이터셋에 포함되든 포함되지 않든, 분석 결과는 거의 동일하다"는 것입니다.

차등 프라이버시는 단순히 데이터를 익명화하거나 집계하는 것과는 근본적으로 다릅니다. 익명화된 데이터도 외부 정보와 결합하면 재식별될 수 있지만, 차등 프라이버시는 이러한 공격에도 수학적으로 엄격한 보호를 제공합니다.

차등 프라이버시의 핵심 직관

차등 프라이버시는 다음 질문에 "예"라고 답할 수 있도록 설계되었습니다:

"내 데이터가 분석에 사용되더라도, 나에 대해 배울 수 있는 것은 내 데이터 없이 배울 수 있는 것과 거의 같은가?"

2.1.1 왜 차등 프라이버시인가?

전통적인 익명화 기법(k-익명성, l-다양성 등)은 다양한 공격에 취약합니다. 차등 프라이버시는 이러한 한계를 극복하기 위해 다음과 같은 강력한 특성을 제공합니다.

특성 설명 장점
수학적 보장 프라이버시 손실의 정량적 측정 명확한 보호 수준 설정 가능
공격 독립성 미래의 공격에도 견고함 배경 지식에 무관한 보호
구성 가능성 여러 분석의 프라이버시 손실 누적 계산 프라이버시 예산 관리 가능
사후 처리 면역 DP 출력에 추가 처리해도 프라이버시 유지 분석 파이프라인 유연성

2.2 수학적 정의

차등 프라이버시의 수학적 정의는 두 가지 "인접한" 데이터셋에서 알고리즘의 출력 분포가 얼마나 유사한지를 측정합니다. 두 데이터셋이 "인접"하다는 것은 정확히 한 개인의 데이터만 다르다는 것을 의미합니다.

ε-차등 프라이버시 정의
Pr[M(D₁) ∈ S] ≤ e^ε × Pr[M(D₂) ∈ S]
여기서 M은 무작위 알고리즘, D₁과 D₂는 인접 데이터셋, S는 임의의 출력 집합, ε(엡실론)는 프라이버시 파라미터입니다.

2.2.1 ε(엡실론)의 의미

ε는 프라이버시 예산 또는 프라이버시 손실 파라미터라고 불립니다. ε 값이 작을수록 더 강한 프라이버시 보호를 의미하고, ε 값이 클수록 더 약한 보호를 의미합니다.

ε = 0.1
매우 강한 보호
높은 노이즈
ε = 1.0
표준 보호
중간 노이즈
ε = 3.0
약한 보호
낮은 노이즈
ε = 10
최소 보호
거의 원본

ε 값 선택 시 주의사항

ε 값의 선택은 중요한 결정입니다. 일반적으로 ε ≤ 1은 강한 프라이버시를, 1 < ε ≤ 3은 중간 수준을, ε > 3은 약한 프라이버시를 제공합니다. 실제 구현에서는 데이터의 민감도와 분석 정확도 요구사항에 따라 결정해야 합니다.

  • 의료 데이터: ε ≤ 0.5 권장
  • 금융 데이터: ε ≤ 1.0 권장
  • 일반 통계: ε ≤ 3.0 권장

2.2.2 (ε, δ)-차등 프라이버시

순수 ε-차등 프라이버시(pure DP)는 때로 너무 엄격하여 실용적인 정확도를 달성하기 어렵습니다. (ε, δ)-차등 프라이버시(approximate DP)는 약간의 완화를 허용합니다.

(ε, δ)-차등 프라이버시 정의
Pr[M(D₁) ∈ S] ≤ e^ε × Pr[M(D₂) ∈ S] + δ
δ는 프라이버시가 완전히 깨질 수 있는 작은 확률을 나타냅니다. 일반적으로 δ < 1/n² (n은 데이터셋 크기)로 설정합니다.

2.3 노이즈 메커니즘

차등 프라이버시를 달성하는 가장 일반적인 방법은 쿼리 결과에 적절한 노이즈를 추가하는 것입니다. 노이즈의 양은 쿼리의 민감도(sensitivity)에 의해 결정됩니다.

2.3.1 민감도 (Sensitivity)

민감도는 한 개인의 데이터가 쿼리 결과에 미칠 수 있는 최대 영향을 측정합니다. 두 가지 주요 유형의 민감도가 있습니다.

전역 민감도 (Global Sensitivity)
Δf = max |f(D₁) - f(D₂)|
인접한 모든 데이터셋 쌍에 대해 쿼리 결과의 최대 차이
쿼리 유형 전역 민감도 예시
카운트 1 암 환자 수 집계
합계 (제한된 범위) 범위 최대값 0-100 범위 점수 합계 → Δ=100
평균 범위/(n+1) 데이터셋 크기에 의존
히스토그램 2 한 사람이 버킷 이동 가능

2.3.2 라플라스 메커니즘

라플라스 메커니즘은 순수 ε-차등 프라이버시를 달성하는 가장 기본적인 방법입니다. 쿼리 결과에 라플라스 분포에서 추출한 노이즈를 추가합니다.

라플라스 메커니즘
M(D) = f(D) + Lap(Δf/ε)
Lap(b)는 스케일 파라미터 b를 가진 라플라스 분포를 의미합니다.
PDF: p(x) = (1/2b) × exp(-|x|/b) 
# Python 구현 예시
import numpy as np

def laplace_mechanism(true_value, sensitivity, epsilon):
    """
    라플라스 메커니즘으로 차등 프라이버시 노이즈 추가

    Parameters:
    - true_value: 실제 쿼리 결과
    - sensitivity: 쿼리의 전역 민감도
    - epsilon: 프라이버시 파라미터

    Returns:
    - 노이즈가 추가된 결과
    """
    scale = sensitivity / epsilon
    noise = np.random.laplace(0, scale)
    return true_value + noise

# 예시: 암 환자 수 카운트 쿼리
true_count = 150
epsilon = 1.0
sensitivity = 1  # 카운트 쿼리의 민감도

noisy_count = laplace_mechanism(true_count, sensitivity, epsilon)
print(f"실제 환자 수: {true_count}")
print(f"노이즈 추가 결과: {noisy_count:.1f}")

2.3.3 가우시안 메커니즘

가우시안 메커니즘은 (ε, δ)-차등 프라이버시를 달성하며, 고차원 데이터에서 더 나은 정확도를 제공할 수 있습니다.

가우시안 메커니즘
M(D) = f(D) + N(0, σ²)
σ ≥ Δf × √(2 ln(1.25/δ)) / ε
N(0, σ²)는 평균 0, 분산 σ²인 가우시안 분포 
# Python 구현 예시
import numpy as np

def gaussian_mechanism(true_value, sensitivity, epsilon, delta):
    """
    가우시안 메커니즘으로 (ε, δ)-차등 프라이버시 노이즈 추가

    Parameters:
    - true_value: 실제 쿼리 결과
    - sensitivity: L2 민감도
    - epsilon: 프라이버시 파라미터
    - delta: 실패 확률

    Returns:
    - 노이즈가 추가된 결과
    """
    sigma = sensitivity * np.sqrt(2 * np.log(1.25 / delta)) / epsilon
    noise = np.random.normal(0, sigma)
    return true_value + noise

# 예시: 평균 소득 쿼리
true_average = 45000
epsilon = 1.0
delta = 1e-5
sensitivity = 100000 / 10000  # 범위/n

noisy_average = gaussian_mechanism(true_average, sensitivity, epsilon, delta)
print(f"실제 평균: {true_average}")
print(f"노이즈 추가 결과: {noisy_average:.2f}")

2.3.4 지수 메커니즘

지수 메커니즘은 숫자가 아닌 출력(예: 카테고리, 항목 선택)에 대해 차등 프라이버시를 달성하는 방법입니다.

지수 메커니즘
Pr[output = o] ∝ exp(ε × u(D, o) / (2Δu))
u(D, o)는 출력 o의 품질(유용성) 점수,
Δu는 품질 함수의 민감도

2.4 구성 정리 (Composition Theorems)

실제 시스템에서는 동일한 데이터에 여러 쿼리를 수행해야 할 때가 많습니다. 구성 정리는 여러 차등 프라이버시 메커니즘의 조합이 어떤 프라이버시 보장을 제공하는지 알려줍니다.

2.4.1 기본 구성 (Basic Composition)

기본 순차 구성
k개의 ε-DP 메커니즘 조합 → kε-DP
프라이버시 손실이 선형적으로 누적됩니다.

2.4.2 고급 구성 (Advanced Composition)

고급 구성 정리
k개의 ε-DP 메커니즘 → (ε√(2k ln(1/δ)) + kε(e^ε - 1), δ)-DP
프라이버시 손실이 √k에 비례하여 증가합니다.

구성 정리의 실용적 의미

기본 구성에서 100개의 쿼리를 각각 ε=0.1로 수행하면 총 ε=10이 됩니다. 그러나 고급 구성을 사용하면 약 ε≈1.5 정도로 줄일 수 있습니다. 이는 프라이버시 예산을 더 효율적으로 사용할 수 있게 해줍니다.

2.4.3 병렬 구성 (Parallel Composition)

데이터셋이 겹치지 않는 부분집합으로 분할되어 각각 독립적으로 분석되는 경우, 프라이버시 손실은 누적되지 않고 최대값만 취합니다.

병렬 구성
분리된 부분집합에 대한 ε₁, ε₂, ..., εₖ-DP → max(ε₁, ε₂, ..., εₖ)-DP
히스토그램과 같이 각 개인이 하나의 버킷에만 기여하는 경우 적용됩니다.

2.5 로컬 vs 중앙 차등 프라이버시

차등 프라이버시는 노이즈가 추가되는 위치에 따라 두 가지 모델로 구분됩니다.

🏢

중앙 DP (Central DP)

신뢰할 수 있는 데이터 수집자가 원본 데이터를 수집하고, 쿼리 결과를 공개하기 전에 노이즈를 추가합니다.

  • 더 높은 정확도
  • 신뢰된 수집자 필요
  • 예: 정부 통계, 기업 내부 분석
📱

로컬 DP (Local DP)

각 사용자가 데이터를 전송하기 전에 자신의 장치에서 노이즈를 추가합니다.

  • 더 강한 신뢰 모델
  • 낮은 정확도 (더 많은 노이즈)
  • 예: Apple, Google 키보드 분석
특성 중앙 DP 로컬 DP
신뢰 모델 수집자를 신뢰 아무도 신뢰 안 함
정확도 높음 (√n에 비례) 낮음 (1에 가까움)
필요 샘플 크기 적음 많음
구현 위치 서버 클라이언트
적용 사례 인구조사, 의료연구 모바일 앱 분석

2.6.1 Apple의 차등 프라이버시

Apple은 2016년부터 iOS에서 로컬 차등 프라이버시를 적용하여 이모지 사용 빈도, QuickType 제안, 건강 앱 데이터 등을 수집합니다.

Apple의 구현 특징

  • Randomized Response: 각 비트를 확률적으로 뒤집어 전송
  • Count Mean Sketch: 해시 기반 빈도 추정
  • Hadamard Count Mean Sketch: 더 효율적인 변형
  • 엡실론 값: 일일 ε=1-4, 총 ε=8-16

2.6.2 Google의 RAPPOR

Google의 RAPPOR(Randomized Aggregatable Privacy-Preserving Ordinal Response)는 Chrome 브라우저에서 사용자 통계를 수집하는 데 사용됩니다. 

# RAPPOR의 간단한 구현 개념
import numpy as np
import hashlib

def rappor_encode(value, num_bits, f, p, q):
    """
    RAPPOR 인코딩: 영구적 무작위 응답 + 즉석 무작위 응답

    Parameters:
    - value: 인코딩할 값
    - num_bits: Bloom filter 크기
    - f: 영구적 뒤집기 확률
    - p: 진실 응답 확률 (1비트)
    - q: 거짓 응답 확률 (0비트)
    """
    # 1. Bloom filter로 값을 비트 배열로 인코딩
    bloom = np.zeros(num_bits, dtype=int)
    for i in range(2):  # 2개의 해시 함수 사용
        h = int(hashlib.sha256(f"{value}_{i}".encode()).hexdigest(), 16)
        bloom[h % num_bits] = 1

    # 2. 영구적 무작위 응답 (PRR) - 사용자별 고정
    prr = np.array([
        np.random.choice([0, 1, b], p=[f/2, f/2, 1-f])
        for b in bloom
    ])

    # 3. 즉석 무작위 응답 (IRR) - 매번 변경
    irr = np.array([
        np.random.choice([1, 0], p=[q if b==0 else p, 1-q if b==0 else 1-p])
        for b in prr
    ])

    return irr

2.6.3 미국 인구조사국

미국 인구조사국은 2020년 인구조사부터 차등 프라이버시를 공식 적용하여 인구통계 데이터를 보호합니다. 이는 차등 프라이버시가 국가 수준의 대규모 데이터에 적용된 가장 중요한 사례입니다.

2020 미국 인구조사 DP 적용

  • 적용 대상: 모든 인구조사 데이터 공개
  • 알고리즘: TopDown Algorithm
  • Privacy Loss Budget: 전체 ε=19.61
  • 주요 과제: 정확도와 프라이버시 간 균형, 소수 집단 보호

2.9 구현 시 고려사항

차등 프라이버시 구현의 함정

  • 부동소수점 문제: 실수 연산의 오차가 프라이버시를 깨뜨릴 수 있음
  • 사이드 채널: 실행 시간, 메모리 접근 패턴에서 정보 유출 가능
  • 민감도 계산 오류: 과소 추정된 민감도는 불충분한 보호 제공
  • 예산 관리 실패: 프라이버시 예산 초과 사용
  • 상관관계 무시: 상관된 데이터에서 그룹 프라이버시 미고려

2.9.1 모범 구현 가이드라인

권장 사항

  1. 검증된 라이브러리 사용: Google DP, OpenDP, IBM Diffprivlib 등
  2. 프라이버시 예산 사전 설정: 분석 전 총 예산과 할당 계획 수립
  3. 민감도 상한 설정: 데이터 클리핑으로 unbounded 민감도 방지
  4. 테스트 및 감사: 독립적인 프라이버시 감사 수행
  5. 투명한 문서화: ε, δ, 민감도 등 모든 파라미터 공개

2.10 다음 장 미리보기

다음 장에서는 동형 암호화(Homomorphic Encryption)를 다룹니다. 차등 프라이버시가 통계적 노이즈로 프라이버시를 보호하는 반면, 동형 암호화는 데이터를 암호화된 상태에서 연산할 수 있게 하여 데이터 자체를 완전히 보호합니다. 완전 동형 암호화(FHE)의 원리, 주요 스킴(BFV, CKKS, TFHE), 그리고 실용적 구현 방법을 학습합니다.
