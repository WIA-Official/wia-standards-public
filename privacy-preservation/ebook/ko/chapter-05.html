연합 학습 (Federated Learning)

Distributed Machine Learning with Data Privacy

5.1 연합 학습의 개념

연합 학습(Federated Learning, FL)은 2016년 Google에서 발표한 분산 머신러닝 패러다임으로, 데이터를 중앙 서버로 모으지 않고 데이터가 있는 곳에서 모델을 훈련하는 방식입니다. 각 클라이언트(디바이스 또는 조직)는 로컬 데이터로 모델을 훈련하고, 모델 업데이트(그래디언트 또는 가중치)만 중앙 서버에 공유합니다.

연합 학습의 핵심 원칙

"데이터를 모델에게 가져가지 말고, 모델을 데이터에게 가져가라"

원시 데이터는 절대 로컬 디바이스를 떠나지 않습니다. 서버가 보는 것은 집계된 모델 업데이트뿐입니다.

연합 학습 라운드
  1. 클라이언트 선택: 서버가 참여할 클라이언트 K개 선택
  2. 모델 배포: 현재 글로벌 모델을 선택된 클라이언트에 전송
  3. 로컬 훈련: 각 클라이언트가 로컬 데이터로 모델 훈련
  4. 업데이트 전송: 클라이언트가 모델 업데이트를 서버에 전송
  5. 집계: 서버가 업데이트들을 집계하여 글로벌 모델 업데이트
  6. 반복: 수렴할 때까지 1-5단계 반복

5.2 FedAvg 알고리즘

FedAvg(Federated Averaging)는 가장 기본적이고 널리 사용되는 연합 학습 알고리즘입니다. 각 클라이언트에서 여러 에포크의 SGD를 수행한 후, 가중치를 평균하여 집계합니다.

FedAvg 집계
w_{t+1} = Σ (n_k / n) × w_k^{t+1}
w_{t+1}: 업데이트된 글로벌 모델 가중치
n_k: 클라이언트 k의 데이터 수
n: 전체 데이터 수
w_k^{t+1}: 클라이언트 k의 로컬 훈련 후 가중치 
# FedAvg 구현 예시
import numpy as np
from typing import List, Dict, Tuple

class FedAvgServer:
    """FedAvg 서버 구현"""

    def __init__(self, model_fn, initial_weights):
        self.model_fn = model_fn
        self.global_weights = initial_weights
        self.round = 0

    def select_clients(self, available_clients: List, fraction: float) -> List:
        """라운드에 참여할 클라이언트 선택"""
        num_clients = max(1, int(len(available_clients) * fraction))
        return np.random.choice(available_clients, num_clients, replace=False)

    def aggregate_weights(self, client_weights: List[Tuple[np.ndarray, int]]) -> np.ndarray:
        """
        클라이언트 가중치 집계 (가중 평균)

        Parameters:
        - client_weights: [(weights, num_samples), ...] 리스트

        Returns:
        - 집계된 글로벌 가중치
        """
        total_samples = sum(num_samples for _, num_samples in client_weights)

        # 가중 평균 계산
        weighted_sum = None
        for weights, num_samples in client_weights:
            weight_factor = num_samples / total_samples
            if weighted_sum is None:
                weighted_sum = [w * weight_factor for w in weights]
            else:
                weighted_sum = [ws + w * weight_factor
                               for ws, w in zip(weighted_sum, weights)]

        return weighted_sum

    def run_round(self, clients: List['FedAvgClient']) -> Dict:
        """
        연합 학습 라운드 실행
        """
        self.round += 1

        # 1. 클라이언트 선택
        selected_clients = self.select_clients(clients, fraction=0.1)

        # 2. 클라이언트에 현재 모델 전송 및 로컬 훈련
        client_updates = []
        for client in selected_clients:
            # 클라이언트에서 로컬 훈련
            updated_weights, num_samples, metrics = client.train(
                self.global_weights,
                epochs=5,
                batch_size=32
            )
            client_updates.append((updated_weights, num_samples))

        # 3. 집계
        self.global_weights = self.aggregate_weights(client_updates)

        return {
            'round': self.round,
            'num_clients': len(selected_clients),
            'total_samples': sum(ns for _, ns in client_updates)
        }


class FedAvgClient:
    """FedAvg 클라이언트 구현"""

    def __init__(self, client_id: str, local_data, local_labels):
        self.client_id = client_id
        self.data = local_data
        self.labels = local_labels

    def train(self, global_weights, epochs: int, batch_size: int):
        """
        로컬 데이터로 모델 훈련

        Parameters:
        - global_weights: 서버로부터 받은 글로벌 가중치
        - epochs: 로컬 훈련 에포크 수
        - batch_size: 배치 크기

        Returns:
        - (updated_weights, num_samples, metrics)
        """
        # 글로벌 가중치로 로컬 모델 초기화
        local_model = self._create_model()
        local_model.set_weights(global_weights)

        # 로컬 훈련
        history = local_model.fit(
            self.data, self.labels,
            epochs=epochs,
            batch_size=batch_size,
            verbose=0
        )

        # 업데이트된 가중치 반환
        return (
            local_model.get_weights(),
            len(self.data),
            {'loss': history.history['loss'][-1]}
        )

    def _create_model(self):

5.3 비IID 데이터 문제

연합 학습의 가장 큰 도전 중 하나는 비IID(Non-Independent and Identically Distributed) 데이터입니다. 각 클라이언트의 로컬 데이터 분포가 서로 다르면 모델 수렴이 어려워지고 성능이 저하됩니다.

비IID 유형 설명 예시
Label Skew 클라이언트마다 레이블 분포가 다름 사용자마다 선호하는 이모지가 다름
Feature Skew 동일 레이블의 특성 분포가 다름 지역마다 손글씨 스타일이 다름
Quantity Skew 클라이언트마다 데이터 양이 다름 활발한 사용자 vs 비활성 사용자
Temporal Skew 시간에 따라 데이터 분포 변화 계절별 쇼핑 패턴 변화

5.3.1 비IID 대응 기법

🔄

FedProx

로컬 모델이 글로벌 모델에서 너무 멀어지지 않도록 근접 항(proximal term)을 추가합니다. 

L_local += (μ/2)||w - w_global||²

🎯

FedNova

각 클라이언트의 로컬 스텝 수를 정규화하여 불균형한 업데이트를 보정합니다.

🧠

5.4 프라이버시 강화 기법

연합 학습 자체가 원시 데이터를 공유하지 않지만, 모델 업데이트에서도 정보가 유출될 수 있습니다. 이를 방지하기 위한 추가적인 프라이버시 강화 기법이 필요합니다.

연합 학습의 프라이버시 위협

  • 그래디언트 역추론 공격: 그래디언트에서 훈련 데이터 복원
  • 멤버십 추론 공격: 특정 데이터가 훈련에 사용되었는지 판별
  • 모델 역전 공격: 모델에서 훈련 데이터의 특성 추출
  • 악의적 서버: 서버가 클라이언트 업데이트를 분석

5.4.1 연합 학습 + 차등 프라이버시

DP-FedAvg
w_k ← w_k + N(0, σ²I)
각 클라이언트 업데이트에 가우시안 노이즈 추가
또는 서버에서 집계 전 노이즈 추가 
# DP-FedAvg 구현 예시
import numpy as np

class DPFedAvgClient:
    """차등 프라이버시가 적용된 FedAvg 클라이언트"""

    def __init__(self, epsilon: float, delta: float, clip_norm: float):
        self.epsilon = epsilon
        self.delta = delta
        self.clip_norm = clip_norm

    def compute_dp_update(self, local_weights, global_weights):
        """
        DP 노이즈가 추가된 모델 업데이트 계산

        1. 업데이트(델타) 계산
        2. 그래디언트 클리핑
        3. 노이즈 추가
        """
        # 1. 업데이트 계산
        update = [lw - gw for lw, gw in zip(local_weights, global_weights)]

        # 2. 그래디언트 클리핑 (L2 norm 제한)
        update_flat = np.concatenate([u.flatten() for u in update])
        norm = np.linalg.norm(update_flat)

        if norm > self.clip_norm:
            scale = self.clip_norm / norm
            update = [u * scale for u in update]

        # 3. 가우시안 노이즈 추가
        sigma = self._compute_noise_scale()
        noisy_update = []
        for u in update:
            noise = np.random.normal(0, sigma * self.clip_norm, u.shape)
            noisy_update.append(u + noise)

        return noisy_update

    def _compute_noise_scale(self):
        """
        (ε, δ)-DP를 위한 노이즈 스케일 계산
        """
        # 가우시안 메커니즘의 노이즈 스케일
        return np.sqrt(2 * np.log(1.25 / self.delta)) / self.epsilon


class DPFedAvgServer:
    """중앙에서 DP를 적용하는 서버"""

    def aggregate_with_dp(self, client_updates, epsilon, delta, num_clients):
        """
        서버 측 DP 집계 (Central DP)
        """
        # 먼저 일반 집계
        aggregated = self._average_updates(client_updates)

        # 집계 결과에 노이즈 추가
        sensitivity = 2.0 / num_clients  # 집계의 민감도
        sigma = sensitivity * np.sqrt(2 * np.log(1.25 / delta)) / epsilon

        noisy_aggregated = []
        for layer in aggregated:
            noise = np.random.normal(0, sigma, layer.shape)
            noisy_aggregated.append(layer + noise)

        return noisy_aggregated

5.4.2 연합 학습 + 안전한 집계

Secure Aggregation은 암호학적 기법을 사용하여 서버가 개별 클라이언트 업데이트를 볼 수 없게 하면서 집계 결과만 얻을 수 있게 합니다.

Secure Aggregation의 작동 원리

  1. 마스크 생성: 각 클라이언트 쌍이 공유 시드로 마스크 생성
  2. 마스킹: 업데이트에 마스크 추가/감산
  3. 집계: 마스크가 상쇄되어 합만 노출

5.5 통신 효율화

연합 학습에서 통신은 주요 병목입니다. 특히 Cross-Device FL에서는 대역폭이 제한되고 연결이 불안정합니다.

기법 설명 압축률
양자화 (Quantization) 32비트 → 8비트 또는 더 적은 비트 4-32배
희소화 (Sparsification) 상위 k% 그래디언트만 전송 10-100배
압축 (Compression) 무손실/손실 압축 적용 2-10배
지식 증류 작은 모델로 지식 전달 모델 크기 의존 
# 그래디언트 압축 예시
import numpy as np

def top_k_sparsify(gradient, k_ratio=0.01):
    """
    상위 k% 그래디언트만 유지 (희소화)
    """
    flat = gradient.flatten()
    k = max(1, int(len(flat) * k_ratio))

    # 절대값 기준 상위 k개 인덱스
    top_k_indices = np.argsort(np.abs(flat))[-k:]

    # 희소 그래디언트 생성
    sparse_grad = np.zeros_like(flat)
    sparse_grad[top_k_indices] = flat[top_k_indices]

    return sparse_grad.reshape(gradient.shape), top_k_indices

def quantize_gradient(gradient, num_bits=8):
    """
    그래디언트 양자화 (8비트)
    """
    min_val = gradient.min()
    max_val = gradient.max()

    # 스케일링
    scale = (max_val - min_val) / (2**num_bits - 1)
    quantized = np.round((gradient - min_val) / scale).astype(np.uint8)

    # 복원에 필요한 메타데이터
    metadata = {'min': min_val, 'scale': scale, 'shape': gradient.shape}

    return quantized, metadata

def dequantize_gradient(quantized, metadata):
    """
    양자화된 그래디언트 복원
    """
    return quantized.astype(np.float32) * metadata['scale'] + metadata['min']

  • 악의적 참여자: Byzantine 공격, 데이터 포이즈닝
  • 공정성: 모든 클라이언트에 대한 공평한 모델 성능
  • 개인화: 글로벌 모델과 개인화된 성능 간 균형

5.10 다음 장 미리보기

다음 장에서는 영지식 증명(Zero-Knowledge Proofs)을 다룹니다. 연합 학습이 데이터를 이동시키지 않는 아키텍처적 접근이라면, 영지식 증명은 "알지 못하면서 증명하는" 암호학적 기술입니다. 블록체인, 인증, 규제 준수 등에서 활용되는 zk-SNARKs, zk-STARKs 등의 실용적 구현을 학습합니다.
