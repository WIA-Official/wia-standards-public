Chapter 03

동형 암호화 (Homomorphic Encryption)

Computing on Encrypted Data Without Decryption

3.1 동형 암호화의 개념

동형 암호화(Homomorphic Encryption, HE)는 암호화된 데이터에 대해 복호화 없이 연산을 수행할 수 있게 해주는 혁신적인 암호 기술입니다. 연산 결과를 복호화하면 원본 데이터에 동일한 연산을 수행한 결과와 같습니다. 이는 "암호화된 클라우드 컴퓨팅"의 핵심 기술로, 데이터 소유자가 민감한 데이터를 제3자에게 맡기면서도 완전한 기밀성을 유지할 수 있게 합니다.

동형 암호화의 핵심 속성

어떤 암호화 스킴 E가 동형성을 가진다는 것은 다음을 의미합니다:

E(m₁) ⊕ E(m₂) = E(m₁ + m₂)

즉, 암호문들에 대한 연산(⊕)이 평문들의 연산(+)과 대응됩니다.

3.1.1 동형 암호화의 직관적 이해

동형 암호화를 쉽게 이해하기 위해 "장갑 낀 손으로 상자 안 작업하기" 비유를 사용할 수 있습니다. 잠긴 상자 안에 귀중품이 있고, 상자에 달린 장갑을 통해서만 내부를 조작할 수 있습니다. 장갑을 낀 손으로는 상자 안의 물건을 만지고 조작할 수 있지만, 상자 자체를 열거나 내용물을 직접 볼 수는 없습니다.

1. 암호화
평문 m → 암호문 c
→
2. 암호문 연산
f(c₁, c₂, ...)
→
3. 복호화
결과 = f(m₁, m₂, ...)

3.1.2 동형 암호화의 유형

동형 암호화는 지원하는 연산의 종류와 횟수에 따라 세 가지 유형으로 분류됩니다.

유형 지원 연산 연산 횟수 대표 스킴
부분 동형 (PHE) 덧셈 또는 곱셈 중 하나 무제한 RSA, Paillier, ElGamal
준동형 (SHE) 덧셈과 곱셈 모두 제한적 BGN, 초기 LWE 기반
완전 동형 (FHE) 덧셈과 곱셈 모두 무제한 BGV, BFV, CKKS, TFHE

FHE의 중요성

완전 동형 암호화(FHE)는 임의의 함수를 암호화된 데이터에 적용할 수 있게 해줍니다. 이는 곧 임의의 프로그램을 암호화된 입력에 대해 실행할 수 있다는 것을 의미합니다. 2009년 Craig Gentry의 FHE 구현은 암호학 역사상 가장 중요한 발견 중 하나로 평가됩니다.

3.2 수학적 기초

3.2.1 격자 암호학 (Lattice Cryptography)

현대의 대부분의 동형 암호화 스킴은 격자(lattice) 문제의 어려움에 기반합니다. 격자는 n차원 공간에서 선형적으로 독립인 벡터들의 정수 선형 결합으로 이루어진 점들의 집합입니다.

격자의 정의
L(B) = {Bx : x ∈ Z^n}
B는 기저 행렬, x는 정수 벡터입니다.

3.2.2 Learning With Errors (LWE) 문제

LWE 문제는 대부분의 현대 FHE 스킴의 보안 기반입니다. 이 문제는 작은 오류가 추가된 선형 방정식 시스템에서 비밀 벡터를 찾는 것입니다.

LWE 문제
b = As + e (mod q)
A는 공개 행렬, s는 비밀 벡터, e는 작은 오류 벡터,
주어진 (A, b)에서 s를 찾는 것이 어려움

양자 안전성

LWE 기반 암호는 현재까지 알려진 양자 알고리즘으로도 효율적으로 풀 수 없습니다. 이는 양자 컴퓨터 시대에도 안전한 암호화를 가능하게 합니다. NIST 포스트양자암호 표준화 과정에서 격자 기반 스킴들이 선정되었습니다.

3.2.3 Ring-LWE

Ring-LWE는 LWE의 효율적인 변형으로, 다항식 환(polynomial ring)에서 정의됩니다. 이는 키 크기와 연산 효율성을 크게 개선합니다.

다항식 환
R_q = Z_q[X] / (X^n + 1)
n은 2의 거듭제곱, q는 모듈러스

3.3.1 BFV 스킴 상세

BFV(Brakerski/Fan-Vercauteren) 스킴은 정확한 정수 연산이 필요한 경우에 적합합니다. 투표 집계, 통계 쿼리, 데이터베이스 검색 등에 사용됩니다.

BFV 암호화
c = (c₀, c₁) = (pk₀·u + e₁ + Δm, pk₁·u + e₂)
pk는 공개키, u는 작은 다항식, e는 오류, Δ = q/t (스케일링 인자) 
# Python (SEAL 라이브러리 사용) - BFV 예시
from seal import *

# 파라미터 설정
parms = EncryptionParameters(scheme_type.bfv)
poly_modulus_degree = 4096
parms.set_poly_modulus_degree(poly_modulus_degree)
parms.set_coeff_modulus(CoeffModulus.BFVDefault(poly_modulus_degree))
parms.set_plain_modulus(PlainModulus.Batching(poly_modulus_degree, 20))

context = SEALContext(parms)

# 키 생성
keygen = KeyGenerator(context)
public_key = keygen.create_public_key()
secret_key = keygen.secret_key()
relin_keys = keygen.create_relin_keys()

# 암호화/복호화 객체
encryptor = Encryptor(context, public_key)
decryptor = Decryptor(context, secret_key)
evaluator = Evaluator(context)

# 배칭 인코더
encoder = BatchEncoder(context)

# 데이터 암호화
data1 = [1, 2, 3, 4] * (poly_modulus_degree // 4)
data2 = [5, 6, 7, 8] * (poly_modulus_degree // 4)

plain1 = encoder.encode(data1)
plain2 = encoder.encode(data2)

cipher1 = encryptor.encrypt(plain1)
cipher2 = encryptor.encrypt(plain2)

# 암호문 덧셈
cipher_sum = evaluator.add(cipher1, cipher2)

# 암호문 곱셈
cipher_prod = evaluator.multiply(cipher1, cipher2)
evaluator.relinearize_inplace(cipher_prod, relin_keys)

# 복호화 및 디코딩
plain_result = decryptor.decrypt(cipher_sum)
result = encoder.decode(plain_result)
print(f"덧셈 결과: {result[:8]}")  # [6, 8, 10, 12, 6, 8, 10, 12]

3.3.2 CKKS 스킴 상세

CKKS(Cheon-Kim-Kim-Song) 스킴은 실수와 복소수 연산을 지원하여 머신러닝 추론, 통계 분석 등에 적합합니다. 근사 계산을 사용하므로 작은 오차가 발생할 수 있습니다.

CKKS의 특징

  • 인코딩: 실수 벡터를 다항식 계수로 인코딩
  • 스케일 관리: 곱셈 후 스케일 조정 필요
  • SIMD: 단일 암호문에 여러 값을 배칭
  • 정밀도: 스케일 비트로 정밀도 제어
# CKKS를 이용한 암호화된 ML 추론 예시
import numpy as np

class EncryptedLogisticRegression:
    """CKKS를 사용한 암호화된 로지스틱 회귀"""

    def __init__(self, context, encoder, encryptor, evaluator):
        self.context = context
        self.encoder = encoder
        self.encryptor = encryptor
        self.evaluator = evaluator

    def encrypt_weights(self, weights):
        """가중치 암호화"""
        plain_weights = self.encoder.encode(weights)
        return self.encryptor.encrypt(plain_weights)

    def encrypted_dot_product(self, cipher_x, cipher_w):
        """암호화된 내적 계산"""
        cipher_result = self.evaluator.multiply(cipher_x, cipher_w)
        # 모든 슬롯 합산 (rotate-and-sum)
        rotated = cipher_result
        for i in range(int(np.log2(len(cipher_x)))):
            rotated = self.evaluator.rotate_vector(rotated, 2**i)
            cipher_result = self.evaluator.add(cipher_result, rotated)
        return cipher_result

    def encrypted_sigmoid_approx(self, cipher_x):
        """암호화된 시그모이드 근사 (다항식 근사)"""
        # σ(x) ≈ 0.5 + 0.197x - 0.004x³ (테일러 급수 근사)
        x_squared = self.evaluator.square(cipher_x)
        x_cubed = self.evaluator.multiply(x_squared, cipher_x)

        # 0.5 + 0.197x - 0.004x³
        result = self.evaluator.multiply_plain(cipher_x, 0.197)
        result = self.evaluator.add_plain(result, 0.5)
        temp = self.evaluator.multiply_plain(x_cubed, 0.004)
        result = self.evaluator.sub(result, temp)

        return result

3.3.3 TFHE 스킴

TFHE(Torus Fully Homomorphic Encryption)는 빠른 부트스트래핑을 특징으로 하며, 부울 게이트 연산에 최적화되어 있습니다. 비교 연산, 조건문 등을 효율적으로 수행합니다.

연산 TFHE 게이트 부트스트래핑 후 시간
AND HomAND ~13ms
OR HomOR ~13ms
XOR HomXOR ~13ms
NOT HomNOT 무시 가능
MUX HomMUX ~26ms

3.4 부트스트래핑

FHE에서 연산을 수행할 때마다 노이즈가 누적됩니다. 특히 곱셈은 노이즈를 급격히 증가시킵니다. 노이즈가 임계값을 초과하면 복호화가 불가능해집니다. 부트스트래핑은 이 노이즈를 제거하여 무제한 연산을 가능하게 합니다.

부트스트래핑의 핵심
Refresh(c) → c' where noise(c') ≪ noise(c)
암호화된 상태에서 복호화 회로를 동형적으로 평가하여 노이즈 감소

부트스트래핑의 비용

부트스트래핑은 계산 집약적인 연산입니다. 스킴에 따라 다르지만:

  • TFHE: 약 13ms (가장 빠름)
  • CKKS/BGV: 수 초 ~ 수십 초
  • 최적화된 구현: GPU 가속으로 10배 이상 개선 가능

3.5 성능 최적화 기법

3.5.1 배칭 (Batching/SIMD)

배칭은 단일 암호문에 여러 개의 평문 값을 패킹하여 SIMD(Single Instruction, Multiple Data) 연산을 수행합니다. 예를 들어, 4096개의 정수를 하나의 암호문에 패킹하면 병렬 처리가 가능합니다.

배칭의 효과

배칭 없이 1000개의 데이터를 처리하면 1000번의 암호화와 연산이 필요합니다. 배칭을 사용하면 하나의 암호문으로 처리 가능하여 처리량을 수천 배 향상시킬 수 있습니다.

3.5.2 모듈러 스위칭 (Modulus Switching)

모듈러 스위칭은 암호문의 모듈러스를 줄여 노이즈를 관리하는 기법입니다. 부트스트래핑보다 훨씬 저렴하며, 정해진 깊이의 회로에서 효과적입니다.

3.5.3 키 스위칭 (Key Switching)

키 스위칭은 한 키로 암호화된 암호문을 다른 키에 대한 암호문으로 변환합니다. 재선형화(relinearization)에 사용되어 곱셈 후 암호문 크기를 관리합니다.

🧬

🏥

암호화된 진단

의료 영상을 암호화하여 AI 모델로 진단을 수행합니다. 환자 데이터가 병원 외부에 평문으로 노출되지 않습니다.

💊

신약 개발

제약사 간 데이터를 암호화된 상태로 공유하여 공동 연구를 수행합니다. 각자의 영업 비밀을 보호합니다.

3.6.1 암호화된 게놈 분석 예시

# 암호화된 SNP 기반 질병 위험도 계산
class EncryptedGWAS:
    """동형 암호화를 이용한 유전체 분석"""

    def __init__(self, he_context):
        self.context = he_context

    def encrypt_genotype(self, snp_data):
        """SNP 데이터 암호화 (0, 1, 2 인코딩)"""
        return self.context.encrypt(snp_data)

    def compute_polygenic_risk_score(self, encrypted_snps, effect_sizes):
        """
        암호화된 다유전자 위험 점수 계산

        PRS = Σ (effect_size_i × snp_i)
        """
        encrypted_prs = self.context.encrypt(0)

        for encrypted_snp, effect in zip(encrypted_snps, effect_sizes):
            # 평문 가중치와 암호문 곱셈
            weighted = self.context.multiply_plain(encrypted_snp, effect)
            encrypted_prs = self.context.add(encrypted_prs, weighted)

        return encrypted_prs

    def encrypted_chi_square_test(self, case_genotypes, control_genotypes):
        """암호화된 카이제곱 검정 (간략화)"""
        # 암호화된 상태에서 빈도 집계
        case_counts = self.aggregate_counts(case_genotypes)
        control_counts = self.aggregate_counts(control_genotypes)

        # 암호화된 카이제곱 통계량 계산
        chi_sq = self.compute_chi_square_stat(case_counts, control_counts)

        return chi_sq

3.10 다음 장 미리보기

다음 장에서는 안전한 다자간 연산(Secure Multi-Party Computation, MPC)을 다룹니다. 동형 암호화가 단일 데이터 소유자의 데이터를 보호하는 데 초점을 맞춘다면, MPC는 여러 참여자가 각자의 데이터를 공개하지 않으면서 공동으로 함수를 계산하는 방법을 제공합니다. 비밀 분산, Garbled Circuits, Oblivious Transfer 등의 핵심 기법과 실제 응용 사례를 학습합니다.
