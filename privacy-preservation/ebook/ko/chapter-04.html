Chapter 04

안전한 다자간 연산 (Secure MPC)

Secure Multi-Party Computation for Collaborative Privacy

4.1 안전한 다자간 연산의 개념

안전한 다자간 연산(Secure Multi-Party Computation, MPC 또는 SMPC)은 여러 참여자가 각자의 비밀 입력을 공개하지 않으면서 공동으로 함수를 계산하는 암호학적 프로토콜입니다. 1982년 Andrew Yao가 제시한 "백만장자 문제"에서 시작되어, 오늘날 금융, 의료, 선거 등 다양한 분야에서 활용됩니다.

백만장자 문제 (Millionaires' Problem)

두 백만장자가 자신의 재산을 공개하지 않고 누가 더 부자인지 알고 싶어합니다. 신뢰할 수 있는 제3자 없이, 각자의 재산액을 비밀로 유지하면서 비교 결과만 알아낼 수 있을까요?

MPC는 이 문제를 해결합니다. 두 사람 모두 결과(누가 더 부자인지)만 알게 되고, 상대방의 정확한 재산액은 알 수 없습니다.

4.1.1 MPC의 보안 속성

안전한 MPC 프로토콜은 다음과 같은 보안 속성을 보장합니다.

속성 설명 의미
기밀성 (Privacy) 입력 데이터가 비밀 유지 출력과 자신의 입력 외에는 아무것도 배울 수 없음
정확성 (Correctness) 출력이 올바른 결과 정직한 참여자는 정확한 함수 결과를 받음
입력 독립성 입력이 다른 입력에 의존하지 않음 악의적 참여자가 다른 입력을 보고 자신의 입력 변경 불가
공정성 (Fairness) 모두가 출력을 받거나 아무도 못 받음 일부만 결과를 얻고 다른 참여자 배제 불가
보장된 출력 정직한 참여자는 반드시 출력 획득 악의적 참여자가 프로토콜 중단 불가

4.1.2 이상적 기능성 (Ideal Functionality)

MPC의 보안은 "이상적 세계"와의 비교로 정의됩니다. 이상적 세계에서는 신뢰할 수 있는 제3자(TTP)가 모든 입력을 받아 계산을 수행하고 결과만 반환합니다. 현실 세계의 MPC 프로토콜이 이상적 세계와 구분 불가능하면 안전하다고 합니다.

Alice
입력: x
Bob
입력: y
Carol
입력: z
↓ MPC 프로토콜 ↓
f(x, y, z) = 결과 (모든 참여자에게 전달)

4.2 적대자 모델

MPC 프로토콜은 적대자의 능력에 따라 다른 보안 수준을 제공합니다.

👀

Semi-Honest (정직하지만 호기심 있는)

프로토콜을 정확히 따르지만, 수집한 모든 정보에서 추가적인 지식을 추론하려고 시도합니다. 대부분의 실용적 시스템이 이 모델을 가정합니다.

  • 더 효율적인 프로토콜
  • 낮은 통신 비용
  • 신뢰할 수 있는 환경에 적합
😈

Malicious (악의적)

프로토콜을 임의로 위반하여 정보를 얻거나 결과를 조작하려고 시도합니다. 가장 강력한 보안을 제공합니다.

  • 영지식 증명으로 검증
  • 더 높은 계산/통신 비용
  • 적대적 환경에 필수

4.2.1 손상 임계값

n명의 참여자 중 최대 t명의 적대자를 허용하는 프로토콜을 t-of-n 프로토콜이라고 합니다.

임계값 의미 적합한 상황
t < n/2 정직한 다수 (Honest Majority) 반수 이상이 정직한 경우, 더 효율적
t < n 부정직한 다수 허용 모든 참여자가 악의적일 수 있는 경우
t = n-1 한 명만 정직해도 안전 최대 보안, 높은 비용

4.3 비밀 분산 (Secret Sharing)

비밀 분산은 MPC의 핵심 구성 요소로, 비밀 값을 여러 조각(share)으로 나누어 개별 조각만으로는 비밀을 복원할 수 없게 합니다.

4.3.1 Shamir 비밀 분산

Shamir의 비밀 분산 스킴은 (t, n)-임계값 스킴으로, n개의 조각 중 최소 t개가 있어야 비밀을 복원할 수 있습니다. 다항식 보간법을 기반으로 합니다.

Shamir 비밀 분산
f(x) = s + a₁x + a₂x² + ... + a_{t-1}x^{t-1}
s는 비밀, a₁...a_{t-1}은 무작위 계수
각 참여자 i는 조각 (i, f(i))를 받음
t개 이상의 조각으로 라그랑주 보간을 통해 s = f(0) 복원 
# Python 구현 - Shamir 비밀 분산
import random
from functools import reduce

def create_shares(secret, threshold, num_shares, prime):
    """
    비밀을 여러 조각으로 분산

    Parameters:
    - secret: 분산할 비밀 값
    - threshold: 복원에 필요한 최소 조각 수 (t)
    - num_shares: 총 조각 수 (n)
    - prime: 유한체 크기

    Returns:
    - [(x1, y1), (x2, y2), ...] 형태의 조각들
    """
    # 무작위 다항식 계수 생성 (상수항 = 비밀)
    coefficients = [secret] + [random.randint(0, prime - 1)
                               for _ in range(threshold - 1)]

    def evaluate_polynomial(x):
        """다항식 f(x) 계산"""
        result = 0
        for i, coef in enumerate(coefficients):
            result = (result + coef * pow(x, i, prime)) % prime
        return result

    # 각 참여자에게 조각 생성
    shares = [(i, evaluate_polynomial(i)) for i in range(1, num_shares + 1)]
    return shares

def reconstruct_secret(shares, prime):
    """
    조각들로부터 비밀 복원 (라그랑주 보간)
    """
    def lagrange_interpolation(x, shares):
        total = 0
        for i, (xi, yi) in enumerate(shares):
            numerator = denominator = 1
            for j, (xj, _) in enumerate(shares):
                if i != j:
                    numerator = (numerator * (x - xj)) % prime
                    denominator = (denominator * (xi - xj)) % prime
            # 모듈러 역원 계산
            lagrange_coef = (numerator * pow(denominator, prime - 2, prime)) % prime
            total = (total + yi * lagrange_coef) % prime
        return total

    return lagrange_interpolation(0, shares)

# 예시 사용
prime = 2147483647  # 큰 소수
secret = 12345

# (3, 5) 스킴: 5개 조각 생성, 3개로 복원 가능
shares = create_shares(secret, threshold=3, num_shares=5, prime=prime)
print(f"원본 비밀: {secret}")
print(f"생성된 조각들: {shares}")

# 3개 조각으로 복원
reconstructed = reconstruct_secret(shares[:3], prime)
print(f"복원된 비밀: {reconstructed}")

4.3.2 Additive Secret Sharing

덧셈적 비밀 분산은 더 간단한 스킴으로, 비밀을 n개의 무작위 조각으로 나누고 모든 조각의 합이 비밀이 되도록 합니다.

덧셈적 비밀 분산
s = s₁ + s₂ + ... + sₙ (mod p)
s₁...sₙ₋₁은 무작위, sₙ = s - (s₁ + ... + sₙ₋₁) mod p

덧셈적 분산의 장점

  • 덧셈 연산: 로컬에서 조각끼리 더하면 됨 (통신 불필요)
  • 상수 곱셈: 로컬에서 조각에 상수 곱하면 됨
  • 간단한 구현: Shamir보다 연산이 단순

4.4 Garbled Circuits

Garbled Circuits(혼란 회로)는 Andrew Yao가 제안한 2자간 MPC 프로토콜로, 부울 회로를 "혼란시켜" 입력을 알지 못한 채 회로를 평가할 수 있게 합니다.

4.4.1 Garbled Circuits의 작동 원리

  1. 회로 표현: 계산할 함수를 부울 회로(AND, OR, XOR 게이트)로 변환
  2. Garbling: 각 와이어에 두 개의 무작위 레이블(0과 1에 대응) 할당
  3. 게이트 테이블: 각 게이트에 대해 암호화된 진리표 생성
  4. 평가: 입력 레이블로 회로를 통해 출력 레이블 계산
  5. 디코딩: 출력 레이블을 실제 비트로 변환
Garbler (Alice)
- 회로 혼란
- 자신의 입력 레이블 선택
- 혼란 회로 전송
Evaluator (Bob)
- OT로 입력 레이블 획득
- 혼란 회로 평가
- 출력 계산 
# Garbled AND 게이트 구현 (개념적)
import os
from cryptography.hazmat.primitives.ciphers import Cipher, algorithms, modes

class GarbledGate:
    """간단한 Garbled AND 게이트"""

    def __init__(self):
        # 각 와이어에 대한 레이블 (0과 1에 대응)
        self.labels = {}

    def generate_wire_labels(self, wire_id):
        """와이어에 대한 두 레이블 생성"""
        label_0 = os.urandom(16)  # 0에 대응하는 레이블
        label_1 = os.urandom(16)  # 1에 대응하는 레이블
        self.labels[wire_id] = (label_0, label_1)
        return label_0, label_1

    def garble_and_gate(self, wire_a, wire_b, wire_out):
        """AND 게이트 혼란"""
        la0, la1 = self.labels[wire_a]
        lb0, lb1 = self.labels[wire_b]
        lo0, lo1 = self.generate_wire_labels(wire_out)

        # 진리표: AND(a, b)
        # a=0, b=0 -> 0
        # a=0, b=1 -> 0
        # a=1, b=0 -> 0
        # a=1, b=1 -> 1

        garbled_table = []
        for a_bit in [0, 1]:
            for b_bit in [0, 1]:
                la = self.labels[wire_a][a_bit]
                lb = self.labels[wire_b][b_bit]
                out_bit = a_bit & b_bit  # AND 연산
                lo = self.labels[wire_out][out_bit]

                # 두 입력 레이블로 출력 레이블 암호화
                key = self._derive_key(la, lb)
                encrypted = self._encrypt(lo, key)
                garbled_table.append(encrypted)

        # 테이블 무작위 셔플 (어떤 행이 어떤 입력에 대응하는지 숨김)
        import random
        random.shuffle(garbled_table)

        return garbled_table

    def _derive_key(self, label_a, label_b):
        """두 레이블에서 암호화 키 유도"""
        from cryptography.hazmat.primitives import hashes
        from cryptography.hazmat.primitives.kdf.hkdf import HKDF
        hkdf = HKDF(algorithm=hashes.SHA256(), length=16,
                    salt=None, info=b'garbled_gate')
        return hkdf.derive(label_a + label_b)

    def _encrypt(self, plaintext, key):
        """AES 암호화"""
        iv = os.urandom(16)
        cipher = Cipher(algorithms.AES(key), modes.CTR(iv))
        encryptor = cipher.encryptor()
        return iv + encryptor.update(plaintext) + encryptor.finalize()

4.5 Oblivious Transfer (OT)

Oblivious Transfer(망각 전송)는 MPC의 핵심 프리미티브로, 발신자가 여러 메시지 중 수신자가 선택한 하나만 전송하되, 발신자는 어떤 것이 선택되었는지 모르고, 수신자는 선택하지 않은 메시지를 알 수 없습니다.

1-out-of-2 OT

발신자 Alice는 두 메시지 m₀, m₁을 가지고 있습니다.
수신자 Bob은 선택 비트 b를 가지고 있습니다.
프로토콜 후: Bob은 m_b만 알고, Alice는 b를 모릅니다.

4.5.1 OT Extension

기본 OT는 공개키 암호 연산이 필요하여 비용이 높습니다. OT Extension은 소수의 기본 OT로 다수의 OT를 효율적으로 수행합니다.

OT Extension 효율성
κ개의 기본 OT → n개의 OT (n ≫ κ)
κ는 보안 파라미터 (예: 128), n은 필요한 OT 수
추가 OT는 대칭키 연산만으로 수행

4.6.1 SPDZ 프로토콜

SPDZ(발음: "Speedz")는 악의적 적대자에 대해 안전한 다자간 MPC 프로토콜입니다. 사전 처리 단계에서 Beaver Triple을 생성하여 온라인 단계를 효율적으로 만듭니다.

Beaver Triple
(a, b, c) where c = a × b
무작위 a, b와 그 곱 c를 미리 분산
온라인 곱셈: x×y = (x-a)(y-b) + (x-a)b + a(y-b) + c 
# SPDZ 스타일 곱셈 (개념적)
class SPDZMultiplication:
    """Beaver Triple을 이용한 안전한 곱셈"""

    def __init__(self, beaver_triple, my_share_a, my_share_b, my_share_c):
        # 사전 처리된 Beaver Triple의 내 조각
        self.share_a = my_share_a
        self.share_b = my_share_b
        self.share_c = my_share_c

    def multiply(self, share_x, share_y):
        """
        두 비밀 분산된 값의 곱 계산

        [x] × [y] = [z]

        여기서 [v]는 v의 비밀 분산을 의미

        # 1단계: d = x - a, e = y - b 계산 (비밀 분산 상태)
        share_d = share_x - self.share_a
        share_e = share_y - self.share_b

        d = self.reveal(share_d)  # 통신 필요

        share_z = d * e * self.is_party_one()

        share_z += d * self.share_b
        share_z += self.share_c

        return share_z

        내 고객 ID가 다른 은행의 블랙리스트에 있는지 확인
        - 내 고객 전체 목록은 공개되지 않음
        - 다른 은행의 블랙리스트도 공개되지 않음
        - 교집합의 크기(매칭 수)만 알 수 있음
        """
        # 1. 내 고객 ID를 해시하여 비밀 분산
        my_hashes = [self.hash_id(cid) for cid in my_customer_ids]
        my_shares = [self.mpc.share(h) for h in my_hashes]

        # 2. PSI 프로토콜 실행
        intersection_size = self.mpc.private_set_intersection(
            my_shares,
            other_bank_blacklist_hashes
        )

        # 3. 결과: 매칭 수만 공개
        return intersection_size

    def hash_id(self, customer_id):
        """고객 ID 해시 (키 포함)"""
        import hashlib
        return hashlib.sha256(customer_id.encode()).hexdigest()

4.9 공공 분야 응용

전자 투표 시스템

MPC를 활용한 전자 투표는 다음을 보장합니다:

  • 투표 비밀: 개별 투표 내용이 공개되지 않음
  • 검증 가능성: 유권자가 자신의 투표가 집계되었는지 확인 가능
  • 정확성: 집계 결과가 정확함을 검증 가능
  • 강건성: 일부 참여자 이탈에도 시스템 동작
