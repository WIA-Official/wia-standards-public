4.1 Introduction to Secure Multi-Party Computation

Secure Multi-Party Computation (MPC) enables multiple parties to jointly compute a function over their inputs while keeping those inputs private. Each party learns only the output of the computation—nothing about other parties' inputs beyond what can be inferred from the output itself.

Consider the classic "Millionaires' Problem" posed by Andrew Yao in 1982: Two millionaires want to determine who is richer without revealing their actual wealth to each other. MPC provides a solution where they can learn who has more money without either learning the other's net worth.

Secure Multi-Party Computation
An MPC protocol allows n parties P₁, ..., Pₙ with private inputs x₁, ..., xₙ to jointly compute y = f(x₁, ..., xₙ) such that:

Privacy: Each party learns nothing beyond its input and the output y
Correctness: The output is correct even if some parties try to cheat
Independence: No party can influence other parties' inputs

4.2 Fundamental Concepts

4.2.1 The Ideal vs. Real Paradigm

MPC security is defined by comparing real protocol execution to an ideal world with a trusted third party. A protocol is secure if anything an adversary can achieve in the real world could also be achieved in the ideal world.

4.2.2 Adversary Models

MPC protocols are designed against specific adversary models that define what corrupted parties can do:

Model Behavior Security Guarantee
Semi-Honest (Passive) Follows protocol, tries to learn from transcript Privacy against curious parties
Malicious (Active) Can deviate arbitrarily from protocol Privacy and correctness against cheaters
Covert Cheats if unlikely to be caught Deterrence through detection probability

4.2.3 Corruption Thresholds

Protocols vary in how many corrupted parties they can tolerate:

  • Honest Majority: Tolerates t < n/2 corruptions (more efficient)
  • Dishonest Majority: Tolerates t < n corruptions (more robust)
  • Two-Party: Special case where one party may be corrupted

4.3 Secret Sharing

Secret sharing is a fundamental building block for MPC. It splits a secret into shares distributed among parties such that a threshold number of shares is needed to reconstruct the secret.

4.3.1 Shamir's Secret Sharing

Shamir's scheme uses polynomial interpolation. To share secret s among n parties with threshold t+1:

  1. Construct polynomial p(x) = s + a₁x + a₂x² + ... + aₜxᵗ where coefficients are random
  2. Give party i the share sᵢ = p(i)
  3. Any t+1 shares can reconstruct p(x) via Lagrange interpolation, recovering s = p(0)
import numpy as np
from functools import reduce

class ShamirSecretSharing:
    def __init__(self, prime=2**61 - 1):
        """Initialize with a large prime for the finite field."""
        self.prime = prime

    def share(self, secret, n_shares, threshold):
        """
        Split a secret into n shares with given threshold.

        Args:
            secret: The secret to share (integer)
            n_shares: Number of shares to create
            threshold: Minimum shares needed to reconstruct

        Returns:
            List of (x, y) share tuples
        """
        # Generate random polynomial coefficients
        # p(x) = secret + a1*x + a2*x^2 + ... + a_{t-1}*x^{t-1}
        coefficients = [secret] + [
            np.random.randint(0, self.prime)
            for _ in range(threshold - 1)
        ]

        # Evaluate polynomial at points 1, 2, ..., n
        shares = []
        for x in range(1, n_shares + 1):
            y = self._evaluate_polynomial(coefficients, x)
            shares.append((x, y))

        return shares

    def reconstruct(self, shares):
        """
        Reconstruct secret from shares using Lagrange interpolation.

        Args:
            shares: List of (x, y) share tuples

        Returns:
            The reconstructed secret
        """
        secret = 0

        for i, (xi, yi) in enumerate(shares):
            # Compute Lagrange basis polynomial at x=0
            numerator = 1
            denominator = 1

            for j, (xj, _) in enumerate(shares):
                if i != j:
                    numerator = (numerator * (-xj)) % self.prime
                    denominator = (denominator * (xi - xj)) % self.prime

            # Modular inverse of denominator
            lagrange = (yi * numerator * pow(denominator, -1, self.prime)) % self.prime
            secret = (secret + lagrange) % self.prime

        return secret

    def _evaluate_polynomial(self, coefficients, x):
        """Evaluate polynomial at point x."""
        result = 0
        x_power = 1
        for coeff in coefficients:
            result = (result + coeff * x_power) % self.prime
            x_power = (x_power * x) % self.prime
        return result

# Example usage
sss = ShamirSecretSharing()

# Share secret 12345 among 5 parties with threshold 3
secret = 12345
shares = sss.share(secret, n_shares=5, threshold=3)
print("Shares:", shares)

# Reconstruct from any 3 shares
reconstructed = sss.reconstruct(shares[:3])
print(f"Reconstructed: {reconstructed}")
assert reconstructed == secret

4.3.2 Additive Secret Sharing

Additive sharing is simpler but requires all shares for reconstruction. It's efficient for two-party computation and forms the basis for many MPC protocols.

import numpy as np

class AdditiveSecretSharing:
    def __init__(self, prime=2**61 - 1):
        self.prime = prime

    def share(self, secret, n_parties):
        """Split secret into n additive shares."""
        # Generate n-1 random shares
        shares = [np.random.randint(0, self.prime) for _ in range(n_parties - 1)]

        # Last share makes sum equal to secret
        last_share = (secret - sum(shares)) % self.prime
        shares.append(last_share)

        return shares

    def reconstruct(self, shares):
        """Reconstruct by adding all shares."""
        return sum(shares) % self.prime

    def add_shares(self, shares1, shares2):
        """Add two shared values (local operation)."""
        return [(s1 + s2) % self.prime for s1, s2 in zip(shares1, shares2)]

    def multiply_by_constant(self, shares, constant):
        """Multiply shared value by constant (local operation)."""
        return [(s * constant) % self.prime for s in shares]

# Example: Private addition
ass = AdditiveSecretSharing()

# Alice has value 100, Bob has value 50
alice_value = 100
bob_value = 50

# Each creates shares of their value
alice_shares = ass.share(alice_value, 2)  # [share_to_alice, share_to_bob]
bob_shares = ass.share(bob_value, 2)       # [share_to_alice, share_to_bob]

# Local addition of shares
sum_shares = ass.add_shares(alice_shares, bob_shares)

# Reconstruct (in practice, shares combined at output phase)
result = ass.reconstruct(sum_shares)
print(f"Private sum: {result}")  # 150

4.4 Garbled Circuits

Garbled circuits, introduced by Andrew Yao, enable two-party computation of any Boolean circuit. One party "garbles" the circuit, the other evaluates it on encrypted inputs.

4.4.1 How Garbling Works

GARBLED CIRCUIT PROTOCOL 1. GARBLER (Party 1): - Takes circuit C and input x₁ - For each wire: generate two random labels (one for 0, one for 1) - For each gate: create encrypted truth table (garbled table) - Send garbled circuit + labels for x₁ to evaluator 2. OBLIVIOUS TRANSFER: - Evaluator has input x₂ - For each input bit, evaluator receives corresponding label - Without garbler learning x₂ 3. EVALUATOR (Party 2): - Has labels for inputs x₁ (from garbler) and x₂ (from OT) - Evaluates circuit gate-by-gate - Each gate: use input labels to decrypt one entry of garbled table - Output: label for output wire 4. OUTPUT: - Evaluator sends output label to garbler - Garbler reveals which value (0 or 1) it corresponds to
Figure 4.2: Garbled circuit protocol overview
import os
import hashlib
from cryptography.fernet import Fernet
import base64

class SimpleGarbledGate:
    """Simplified garbled AND gate for educational purposes."""

    def __init__(self):
        self.key_length = 32  # 256 bits

    def generate_wire_labels(self):
        """Generate two random labels for a wire (one for 0, one for 1)."""
        return {
            0: os.urandom(self.key_length),
            1: os.urandom(self.key_length)
        }

    def hash_labels(self, label_a, label_b, gate_id):
        """Derive encryption key from two input labels."""
        combined = label_a + label_b + gate_id.to_bytes(4, 'big')
        return hashlib.sha256(combined).digest()

    def garble_and_gate(self, input_a_labels, input_b_labels, output_labels, gate_id):
        """
        Create garbled table for AND gate.

        Returns:
            Garbled table (encrypted truth table entries, shuffled)
        """
        garbled_table = []

        # For each possible input combination
        for a_val in [0, 1]:
            for b_val in [0, 1]:
                # Compute output value
                output_val = a_val & b_val

                # Get labels
                label_a = input_a_labels[a_val]
                label_b = input_b_labels[b_val]
                label_out = output_labels[output_val]

                # Derive encryption key from input labels
                key = self.hash_labels(label_a, label_b, gate_id)

                # Encrypt output label
                encrypted = self._encrypt(key, label_out)
                garbled_table.append(encrypted)

        # Shuffle table to hide which entry corresponds to which input
        import random
        random.shuffle(garbled_table)

        return garbled_table

    def evaluate_garbled_gate(self, garbled_table, label_a, label_b, gate_id):
        """
        Evaluate garbled gate given input labels.

        Returns:
            Output wire label
        """
        # Derive key from input labels
        key = self.hash_labels(label_a, label_b, gate_id)

        # Try to decrypt each table entry
        for encrypted in garbled_table:
            try:
                decrypted = self._decrypt(key, encrypted)
                return decrypted  # Only one will decrypt successfully
            except:
                continue

        raise ValueError("Could not evaluate gate")

    def _encrypt(self, key, plaintext):
        """Simple symmetric encryption."""
        # Use key to derive Fernet key
        fernet_key = base64.urlsafe_b64encode(key[:32])
        f = Fernet(fernet_key)
        return f.encrypt(plaintext)

    def _decrypt(self, key, ciphertext):
        """Simple symmetric decryption."""
        fernet_key = base64.urlsafe_b64encode(key[:32])
        f = Fernet(fernet_key)
        return f.decrypt(ciphertext)

# Example: Garbled AND gate
garbler = SimpleGarbledGate()

# Generate wire labels
wire_a = garbler.generate_wire_labels()  # First input
wire_b = garbler.generate_wire_labels()  # Second input
wire_out = garbler.generate_wire_labels()  # Output

# Garble the gate
garbled_table = garbler.garble_and_gate(wire_a, wire_b, wire_out, gate_id=0)

# Evaluate with inputs a=1, b=1 (should give 1)
result_label = garbler.evaluate_garbled_gate(
    garbled_table,
    wire_a[1],  # Label for a=1
    wire_b[1],  # Label for b=1
    gate_id=0
)

# Check result
assert result_label == wire_out[1]  # Should be label for output=1
print("Garbled gate evaluated successfully!")

4.4.2 Oblivious Transfer

Oblivious Transfer (OT) is crucial for garbled circuits. It allows a receiver to obtain one of multiple messages from a sender without revealing which message was selected.

# Conceptual 1-out-of-2 OT (simplified)
class ObliviousTransfer:
    """
    1-out-of-2 Oblivious Transfer
    Sender has messages (m0, m1)
    Receiver has choice bit b
    Receiver gets m_b, Sender learns nothing about b
    """

    def __init__(self, prime):
        self.prime = prime
        self.generator = 2  # Generator of group

    def sender_round1(self):
        """Sender generates public parameters."""
        self.sender_secret = np.random.randint(1, self.prime - 1)
        public = pow(self.generator, self.sender_secret, self.prime)
        return public

    def receiver_round1(self, sender_public, choice_bit):
        """Receiver generates response based on choice."""
        self.receiver_secret = np.random.randint(1, self.prime - 1)

        if choice_bit == 0:
            # pk0 = g^r, pk1 = A / g^r
            pk = pow(self.generator, self.receiver_secret, self.prime)
        else:
            # pk1 = g^r, pk0 = A / g^r (swapped)
            pk = pow(self.generator, self.receiver_secret, self.prime)

        return pk, choice_bit

    def sender_round2(self, receiver_pk, m0, m1):
        """Sender encrypts both messages."""
        # In real OT, use proper encryption
        # This is a simplified illustration
        enc0 = (m0 + hash(str(pow(receiver_pk, self.sender_secret, self.prime)))) % self.prime
        # ... similar for m1
        return enc0, None  # Simplified

    def receiver_decrypt(self, encrypted, sender_public):
        """Receiver decrypts chosen message."""
        shared = pow(sender_public, self.receiver_secret, self.prime)
        # Decrypt using shared secret
        return encrypted  # Simplified

4.5 Modern MPC Protocols

4.5.1 GMW Protocol

The GMW protocol uses secret sharing and Boolean circuits. It requires interaction for each AND gate but supports multiple parties.

4.5.2 BGW Protocol

BGW uses Shamir secret sharing for arithmetic circuits and achieves information-theoretic security with honest majority.

4.5.3 SPDZ Protocol

SPDZ (pronounced "Speedz") is a practical protocol for dishonest majority using preprocessing. It achieves malicious security efficiently through MACs.

# Conceptual SPDZ-style multiplication triple
class SPDZTriple:
    """
    Beaver multiplication triples for SPDZ.
    Preprocessing: Generate random a, b, c where c = a * b
    Online: Use triple to compute x * y securely
    """

    def __init__(self, prime):
        self.prime = prime

    def generate_triple(self, n_parties):
        """Generate random multiplication triple (offline phase)."""
        a = np.random.randint(0, self.prime)
        b = np.random.randint(0, self.prime)
        c = (a * b) % self.prime

        # Secret share a, b, c among parties
        a_shares = self._share(a, n_parties)
        b_shares = self._share(b, n_parties)
        c_shares = self._share(c, n_parties)

        return a_shares, b_shares, c_shares

    def multiply_with_triple(self, x_shares, y_shares, triple, n_parties):
        """
        Compute [x * y] using multiplication triple.

        Online phase:
        1. Compute [d] = [x] - [a] and [e] = [y] - [b]
        2. Open d and e (reveal these values)
        3. Compute [xy] = [c] + e*[a] + d*[b] + d*e
        """
        a_shares, b_shares, c_shares = triple

        # Step 1: Compute masked values
        d_shares = [(x - a) % self.prime for x, a in zip(x_shares, a_shares)]
        e_shares = [(y - b) % self.prime for y, b in zip(y_shares, b_shares)]

        # Step 2: Open d and e (in practice, parties exchange shares)
        d = sum(d_shares) % self.prime
        e = sum(e_shares) % self.prime

        # Step 3: Compute result shares
        # [xy] = [c] + e*[a] + d*[b] + d*e (party 0 adds d*e, others don't)
        result_shares = []
        for i in range(n_parties):
            share = (c_shares[i] + e * a_shares[i] + d * b_shares[i]) % self.prime
            if i == 0:
                share = (share + d * e) % self.prime
            result_shares.append(share)

        return result_shares

    def _share(self, secret, n_parties):
        """Additive secret sharing."""
        shares = [np.random.randint(0, self.prime) for _ in range(n_parties - 1)]
        shares.append((secret - sum(shares)) % self.prime)
        return shares

# Example: Private multiplication
spdz = SPDZTriple(prime=2**61 - 1)

# Generate preprocessing material
triple = spdz.generate_triple(n_parties=3)

# Parties have secret-shared inputs
x = 7  # Known only to Party 0
y = 8  # Known only to Party 1

x_shares = spdz._share(x, 3)
y_shares = spdz._share(y, 3)

# Multiply using triple
result_shares = spdz.multiply_with_triple(x_shares, y_shares, triple, 3)

# Reconstruct result
result = sum(result_shares) % spdz.prime
print(f"Private multiplication: {x} * {y} = {result}")

4.8 Performance Considerations

MPC Performance Factors

  • Communication: Network latency and bandwidth are major bottlenecks
  • Computation: Cryptographic operations dominate CPU time
  • Circuit Size: Larger circuits require more resources
  • Number of Parties: Communication scales with party count
  • Security Model: Malicious security adds 2-10x overhead

4.9 Summary

Secure Multi-Party Computation enables collaborative computation while protecting each party's private inputs. From secret sharing to garbled circuits to modern SPDZ protocols, MPC provides powerful tools for privacy-preserving collaboration across organizations.

Key Takeaways

  • MPC allows joint computation without revealing private inputs
  • Security defined by comparison to ideal trusted third party
  • Semi-honest vs. malicious security have different guarantees
  • Secret sharing enables efficient arithmetic operations
  • Garbled circuits enable any Boolean computation in constant rounds
  • Modern protocols (SPDZ) achieve practical performance
  • Real deployments exist in finance, healthcare, and government
