2.1 Introduction to Differential Privacy

Differential privacy, introduced by Cynthia Dwork in 2006, provides a rigorous mathematical framework for quantifying and limiting privacy loss when releasing statistical information about datasets. Unlike traditional anonymization techniques that have repeatedly failed to prevent re-identification attacks, differential privacy offers provable guarantees that hold regardless of what auxiliary information an adversary might possess.

The key insight of differential privacy is to shift focus from protecting specific data to protecting the computation process. Rather than trying to anonymize data (which has proven inadequate), differential privacy ensures that the output of a computation reveals almost nothing about any individual's participation in the dataset.

The Differential Privacy Guarantee
A randomized algorithm M satisfies (ε, δ)-differential privacy if for any two adjacent datasets D and D' (differing in at most one individual's data), and for any set of outputs S:

Pr[M(D) ∈ S] ≤ e^ε × Pr[M(D') ∈ S] + δ

This means the probability of any output changes by at most a factor of e^ε (plus δ) whether or not any individual's data is included.

The parameter ε (epsilon) is called the privacy budget or privacy loss parameter. Smaller epsilon values provide stronger privacy guarantees but typically require more noise, reducing utility. The parameter δ represents a small probability of complete privacy failure and is typically set to a negligible value like 1/n² where n is the dataset size.

2.2 The Mathematics of Differential Privacy

2.2.1 Adjacent Datasets

Two datasets D and D' are adjacent (denoted D ~ D') if they differ in exactly one record. This can mean either:

  • Bounded differential privacy: One record is different between D and D' (same size)
  • Unbounded differential privacy: D' is D with one record added or removed
Dataset D: Dataset D' (adjacent): ┌────────────────┐ ┌────────────────┐ │ Alice: $50,000 │ │ Alice: $50,000 │ │ Bob: $60,000 │ │ Bob: $60,000 │ │ Carol: $70,000 │ │ Carol: $70,000 │ │ Dave: $80,000 │ │ │ ← Dave's record removed └────────────────┘ └────────────────┘ For any query Q, differential privacy ensures: Pr[Q(D) = output] ≈ Pr[Q(D') = output]

2.2.2 Sensitivity

The sensitivity of a function measures how much the function's output can change when one individual's data changes. This is crucial for determining how much noise to add.

Global Sensitivity (L1)
Δf = maxD~D' |f(D) - f(D')|

Maximum change in output over all pairs of adjacent datasets

L2 Sensitivity
Δ₂f = maxD~D' ||f(D) - f(D')||₂

Euclidean distance for vector-valued functions

Examples of Sensitivity

Function Sensitivity Explanation
Count 1 Adding/removing one record changes count by 1
Sum (bounded values in [0,B]) B One person can contribute at most B
Mean (n records, bounded) B/n One person affects average by at most B/n
Histogram (k bins) 2 One person affects at most 2 bins by 1 each
Median Unbounded Requires different approach (exponential mechanism)

2.3 Noise Mechanisms

Differential privacy is achieved by adding carefully calibrated random noise to query results. The amount of noise depends on the sensitivity of the query and the desired privacy level (epsilon).

2.3.1 Laplace Mechanism

The Laplace mechanism is the most common approach for achieving differential privacy on numeric queries. It adds noise drawn from the Laplace distribution centered at zero.

Laplace Mechanism
M(D) = f(D) + Lap(Δf/ε)

Add Laplace noise scaled to sensitivity/epsilon

import numpy as np

def laplace_mechanism(true_value, sensitivity, epsilon):
    """
    Apply Laplace mechanism for differential privacy.

    Args:
        true_value: The true query result
        sensitivity: The L1 sensitivity of the query
        epsilon: The privacy parameter

    Returns:
        Differentially private result
    """
    scale = sensitivity / epsilon
    noise = np.random.laplace(0, scale)
    return true_value + noise

# Example: Private count query
def private_count(dataset, condition, epsilon=1.0):
    """Count records matching condition with ε-differential privacy."""
    true_count = sum(1 for record in dataset if condition(record))
    # Count has sensitivity 1
    return laplace_mechanism(true_count, sensitivity=1, epsilon=epsilon)

# Example: Private sum query
def private_sum(dataset, attribute, max_value, epsilon=1.0):
    """Sum attribute values with ε-differential privacy."""
    true_sum = sum(record[attribute] for record in dataset)
    # Sum sensitivity equals max possible value
    return laplace_mechanism(true_sum, sensitivity=max_value, epsilon=epsilon)

# Example usage
dataset = [
    {"age": 25, "income": 50000},
    {"age": 35, "income": 75000},
    {"age": 45, "income": 100000},
    {"age": 30, "income": 60000}
]

# Private count of people over 30
count = private_count(dataset, lambda r: r["age"] > 30, epsilon=0.5)
print(f"Private count (age > 30): {count:.2f}")

# Private sum of incomes (assuming max income is 200000)
total = private_sum(dataset, "income", max_value=200000, epsilon=0.5)
print(f"Private total income: ${total:.2f}")

2.3.2 Gaussian Mechanism

The Gaussian mechanism adds noise from a normal distribution. It provides (ε, δ)-differential privacy and is often preferred when multiple queries are composed due to tighter bounds.

Gaussian Mechanism
M(D) = f(D) + N(0, σ²) where σ ≥ Δ₂f · √(2ln(1.25/δ)) / ε
import numpy as np
from scipy import stats

def gaussian_mechanism(true_value, sensitivity, epsilon, delta):
    """
    Apply Gaussian mechanism for (ε, δ)-differential privacy.

    Args:
        true_value: The true query result
        sensitivity: The L2 sensitivity of the query
        epsilon: The privacy parameter
        delta: The failure probability parameter

    Returns:
        Differentially private result
    """
    sigma = sensitivity * np.sqrt(2 * np.log(1.25 / delta)) / epsilon
    noise = np.random.normal(0, sigma)
    return true_value + noise

# Example: Private mean with Gaussian noise
def private_mean(values, min_val, max_val, epsilon=1.0, delta=1e-5):
    """Compute mean with (ε, δ)-differential privacy."""
    n = len(values)
    true_mean = np.mean(values)

    # Clip values to bounded range
    clipped = np.clip(values, min_val, max_val)

    # Sensitivity of mean for bounded data
    sensitivity = (max_val - min_val) / n

    return gaussian_mechanism(true_mean, sensitivity, epsilon, delta)

# Example
incomes = [50000, 75000, 100000, 60000, 85000]
private_avg = private_mean(incomes, 0, 200000, epsilon=0.5, delta=1e-5)
print(f"Private mean income: ${private_avg:.2f}")

2.3.3 Exponential Mechanism

The exponential mechanism is used for non-numeric queries, such as selecting the best option from a set. It assigns higher probability to better options while providing differential privacy.

Exponential Mechanism
Pr[M(D) = r] ∝ exp(ε · u(D, r) / 2Δu)

Probability of output r proportional to exponential of utility score

import numpy as np

def exponential_mechanism(dataset, candidates, utility_fn, sensitivity, epsilon):
    """
    Select from candidates using the exponential mechanism.

    Args:
        dataset: The input dataset
        candidates: List of possible outputs
        utility_fn: Function(dataset, candidate) -> utility score
        sensitivity: Maximum change in utility from one record
        epsilon: Privacy parameter

    Returns:
        Selected candidate with differential privacy
    """
    # Compute utility scores
    utilities = [utility_fn(dataset, c) for c in candidates]

    # Compute probabilities (exponential of scaled utility)
    scores = np.array(utilities) * epsilon / (2 * sensitivity)

    # Normalize to prevent overflow
    scores = scores - np.max(scores)
    probabilities = np.exp(scores)
    probabilities = probabilities / np.sum(probabilities)

    # Sample according to probabilities
    idx = np.random.choice(len(candidates), p=probabilities)
    return candidates[idx]

# Example: Private mode (most common value)
def private_mode(dataset, attribute, possible_values, epsilon=1.0):
    """Find the most common value with differential privacy."""

    def count_utility(data, value):
        return sum(1 for record in data if record[attribute] == value)

    # Sensitivity of count is 1
    return exponential_mechanism(
        dataset, possible_values, count_utility,
        sensitivity=1, epsilon=epsilon
    )

# Example usage
preferences = [
    {"color": "blue"}, {"color": "red"}, {"color": "blue"},
    {"color": "green"}, {"color": "blue"}, {"color": "red"}
]
colors = ["red", "blue", "green", "yellow"]

mode_color = private_mode(preferences, "color", colors, epsilon=1.0)
print(f"Private mode color: {mode_color}")

2.4 Composition Theorems

Real-world applications often require multiple queries on the same dataset. Composition theorems quantify how privacy degrades when multiple differentially private computations are performed.

2.4.1 Sequential Composition

When multiple queries are performed on the same dataset, privacy losses accumulate.

Basic Composition
If M₁ is ε₁-DP and M₂ is ε₂-DP, then (M₁, M₂) is (ε₁ + ε₂)-DP

Advanced Composition Theorem

For k queries each with ε-DP, the total privacy loss is bounded by:

  • Basic: kε (linear growth)
  • Advanced: √(2k ln(1/δ)) · ε + kε(e^ε - 1) for (·, δ)-DP
  • Optimal: Using concentrated DP or Rényi DP for tighter bounds
class PrivacyBudgetAccountant:
    """Track and manage privacy budget across multiple queries."""

    def __init__(self, total_epsilon, total_delta=0):
        self.total_epsilon = total_epsilon
        self.total_delta = total_delta
        self.spent_epsilon = 0
        self.spent_delta = 0
        self.query_log = []

    def can_query(self, epsilon, delta=0):
        """Check if we have enough budget for a query."""
        return (self.spent_epsilon + epsilon <= self.total_epsilon and
                self.spent_delta + delta <= self.total_delta)

    def spend(self, epsilon, delta=0, description=""):
        """Spend privacy budget on a query."""
        if not self.can_query(epsilon, delta):
            raise ValueError("Insufficient privacy budget")

        self.spent_epsilon += epsilon
        self.spent_delta += delta
        self.query_log.append({
            "epsilon": epsilon,
            "delta": delta,
            "description": description,
            "remaining_epsilon": self.total_epsilon - self.spent_epsilon
        })

        return self.remaining_budget()

    def remaining_budget(self):
        """Get remaining privacy budget."""
        return {
            "epsilon": self.total_epsilon - self.spent_epsilon,
            "delta": self.total_delta - self.spent_delta
        }

    def get_report(self):
        """Generate a privacy budget report."""
        return {
            "total_budget": {"epsilon": self.total_epsilon, "delta": self.total_delta},
            "spent": {"epsilon": self.spent_epsilon, "delta": self.spent_delta},
            "remaining": self.remaining_budget(),
            "num_queries": len(self.query_log),
            "queries": self.query_log
        }

# Example usage
accountant = PrivacyBudgetAccountant(total_epsilon=3.0, total_delta=1e-5)

# Perform several queries
accountant.spend(0.5, description="Count query 1")
accountant.spend(1.0, description="Sum query")
accountant.spend(0.5, description="Count query 2")

print(f"Remaining budget: {accountant.remaining_budget()}")
print(f"Total queries: {len(accountant.query_log)}")

2.4.2 Parallel Composition

When queries are performed on disjoint subsets of data, the privacy guarantee is determined by the maximum epsilon, not the sum.

Parallel Composition
If M is ε-DP and applied to disjoint subsets D₁, D₂, ..., Dₖ, the combined output is ε-DP (not kε)
def parallel_histogram(dataset, attribute, bins, epsilon):
    """
    Compute a histogram with parallel composition.
    Each bin uses the full epsilon budget since bins are disjoint.

    Args:
        dataset: Input records
        attribute: Attribute to bin
        bins: List of (min, max) tuples defining bins
        epsilon: Privacy parameter (applied to each bin)

    Returns:
        Differentially private histogram counts
    """
    histogram = []

    for bin_min, bin_max in bins:
        # Count records in this bin
        true_count = sum(
            1 for record in dataset
            if bin_min <= record[attribute] < bin_max
        )

        # Each bin uses full epsilon (parallel composition)
        # Sensitivity of count is 1
        noisy_count = laplace_mechanism(true_count, sensitivity=1, epsilon=epsilon)
        histogram.append(max(0, round(noisy_count)))  # Ensure non-negative

    return histogram

# Example
ages = [{"age": a} for a in [22, 25, 31, 35, 42, 45, 28, 55, 38, 29]]
bins = [(20, 30), (30, 40), (40, 50), (50, 60)]

hist = parallel_histogram(ages, "age", bins, epsilon=0.5)
print("Age distribution histogram:", hist)

2.5 Local vs. Global Differential Privacy

Differential privacy can be applied in two different trust models, each with distinct advantages and trade-offs.

2.5.1 Global (Central) Differential Privacy

In the global model, a trusted curator collects raw data and adds noise to query results. This provides better utility but requires trusting the data collector.

┌──────────────────────────────────────────────────────────┐ │ GLOBAL MODEL │ ├──────────────────────────────────────────────────────────┤ │ │ │ User 1 ────┐ │ │ User 2 ────┤ ┌───────────────┐ ┌───────────┐ │ │ User 3 ────┼───>│ TRUSTED │───>│ + Noise │───> Output │ User 4 ────┤ │ CURATOR │ │ │ │ │ User 5 ────┘ │ (raw data) │ └───────────┘ │ │ └───────────────┘ │ │ │ │ Trust: Users must trust the curator with raw data │ │ Utility: High (noise added to aggregate only) │ └──────────────────────────────────────────────────────────┘
Figure 2.2: Global differential privacy model

2.5.2 Local Differential Privacy

In the local model, each user adds noise to their own data before sending it. This provides privacy guarantees even against the data collector but requires more noise for the same utility.

┌──────────────────────────────────────────────────────────┐ │ LOCAL MODEL │ ├──────────────────────────────────────────────────────────┤ │ │ │ User 1 ─[+noise]─┐ │ │ User 2 ─[+noise]─┤ ┌───────────────┐ │ │ User 3 ─[+noise]─┼──>│ UNTRUSTED │───> Aggregate │ │ User 4 ─[+noise]─┤ │ COLLECTOR │ Output │ │ User 5 ─[+noise]─┘ │ (noisy data) │ │ │ └───────────────┘ │ │ │ │ Trust: No trust required (user adds noise locally) │ │ Utility: Lower (noise in each response) │ └──────────────────────────────────────────────────────────┘
Figure 2.3: Local differential privacy model

Randomized Response

Randomized response is a classic local differential privacy mechanism, originally developed for surveys on sensitive topics.

import numpy as np

def randomized_response(true_answer, epsilon):
    """
    Apply randomized response for local differential privacy.

    Args:
        true_answer: Boolean - the user's true response
        epsilon: Privacy parameter

    Returns:
        Randomized response (may or may not match true answer)
    """
    # Probability of telling the truth
    p = np.exp(epsilon) / (1 + np.exp(epsilon))

    if np.random.random() < p:
        return true_answer  # Tell the truth
    else:
        return not true_answer  # Lie (flip the answer)

def estimate_from_randomized_responses(responses, epsilon):
    """
    Estimate the true proportion from randomized responses.

    Args:
        responses: List of boolean randomized responses
        epsilon: Privacy parameter used in collection

    Returns:
        Estimated true proportion of "yes" answers
    """
    p = np.exp(epsilon) / (1 + np.exp(epsilon))
    observed_proportion = sum(responses) / len(responses)

    # Unbiased estimator
    estimated_true = (observed_proportion - (1 - p)) / (2 * p - 1)
    return np.clip(estimated_true, 0, 1)

# Example: Survey about sensitive behavior
np.random.seed(42)
true_positives = 300  # 30% actual rate
total_users = 1000
epsilon = 2.0  # Privacy parameter

# Simulate true answers
true_answers = [True] * true_positives + [False] * (total_users - true_positives)

# Collect randomized responses
responses = [randomized_response(ans, epsilon) for ans in true_answers]

# Estimate true proportion
estimated_rate = estimate_from_randomized_responses(responses, epsilon)
print(f"True rate: {true_positives/total_users:.1%}")
print(f"Estimated rate: {estimated_rate:.1%}")

2.6 Differential Privacy in Practice

2.6.1 Choosing Epsilon

Selecting an appropriate epsilon value is one of the most challenging aspects of deploying differential privacy. There is no universal "right" answer—it depends on the sensitivity of the data, the stakes involved, and the utility requirements.

Epsilon Range Privacy Level Typical Use Cases
0.01 - 0.1 Very strong Highly sensitive data, strong adversaries
0.1 - 1.0 Strong Medical data, financial records
1.0 - 3.0 Moderate General analytics, many real deployments
3.0 - 10.0 Weak Less sensitive data, research settings
> 10 Minimal Often considered insufficient protection

Epsilon Selection Guidelines

  • Apple uses ε = 1-4 for local DP in iOS
  • Google uses ε = 0.5-2.25 for Chrome RAPPOR
  • US Census Bureau used ε ≈ 17-19 for 2020 Census (controversial)
  • Academic recommendations often suggest ε ≤ 1 for strong privacy
  • Consider total budget across all queries, not just per-query epsilon

  • Google: Chrome RAPPOR, mobility reports during COVID-19

from opendp.measurements import make_base_laplace
from opendp.transformations import make_count

# Example: Private count with OpenDP
def opendp_count(data, epsilon):
    """Count with differential privacy using OpenDP."""
    count_transformation = make_count(TIA=str)
    laplace_measurement = make_base_laplace(scale=1.0/epsilon)
    dp_count = count_transformation >> laplace_measurement
    return dp_count(data)

    return clf

2.8 Summary

Differential privacy provides a rigorous mathematical framework for protecting individual privacy in data analysis. Through careful application of noise mechanisms—Laplace, Gaussian, or exponential—and thoughtful management of privacy budgets through composition, organizations can release valuable statistics while providing provable privacy guarantees.

Key Takeaways

  • Differential privacy guarantees that output distributions are nearly identical whether any individual's data is included or not
  • Epsilon (ε) controls the privacy-utility trade-off; smaller values mean stronger privacy
  • Sensitivity determines how much noise is needed for a given query
  • Laplace mechanism for numeric queries, exponential mechanism for selection queries
  • Composition theorems govern privacy loss across multiple queries
  • Local DP provides stronger trust assumptions but requires more noise
  • Multiple production-ready libraries are available for implementation
