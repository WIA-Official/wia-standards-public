Executive Summary

World Certification Industry Association (WIA) has released the WIA Perception Clarity Standard v1.0.0 — a cross-cutting international standard that defines how physical AI agents (vehicles, robots, drones, AMRs) measure, state, and report how clearly their optical and sensing systems see — under the MIT License for free. The standard introduces the Perception Clarity Index (PCI), a 0–100 scale, four clarity states, a contamination taxonomy, and a VDA5050-style reporting protocol.

🚨 Current Situation: Why This Standard is Needed

Physical AI agents see the world through cameras, LiDAR, radar, and ultrasonic sensors. But the moment a transparent window — a lens, cover, or radome — is fouled by rain, mud, dust, salt spray, insects, frost, condensation, or glare, perception degrades sharply. In the worst case, a sensor enters the most dangerous state of all: blind, yet convinced it can still see. The deeper problem is not the physics of contamination — it is a standardization gap. Today there is no open language for a machine to express how clearly it is seeing right now.

💡 WIA's Solution: The Perception Clarity Standard

The WIA Perception Clarity Standard standardizes four things, and four things only: the PCI index, a clarity-state vocabulary, a contamination taxonomy, and a reporting message schema. It deliberately follows the VDA5050 philosophy — that standard never dictated how to build an AGV, only how fleet and vehicle exchange messages, and it became a de facto industry standard precisely because of that restraint.

Perception Clarity Index (PCI) — 0 to 100

PCI is a 0–100 integer computed per sensor (100 = perfect, 0 = total blindness). It is the weighted sum of three measurable axes — occlusion ratio, detection-distance degradation, and contrast / MTF reduction — with weights tuned per sensor class. A radar radome, for instance, weights signal attenuation far above optical resolution.

🚦 Four Clarity States

State PCI Range Required Behavior
clear 90–100 Normal operation
degraded 60–89 Monitor, trigger cleaning
obstructed 30–59 Safe action — slow down, reroute, clean
blind 0–29 Disable sensor, enter safe-state

The PCI bands are defaults that may be tuned per sensor class and operating environment, but the meaning and ordering of the states — clear > degraded > obstructed > blind — are invariant.

📊 What Is — and Is Not — Standardized

Standardized ✅ Not Standardized ❌
PCI index (0–100) Cleaning hardware, nozzles
Clarity states & contamination taxonomy Cleaning fluids, air curtains
Measurement / contamination test protocol Vendor detection algorithms
VDA5050-style reporting protocol Hydrophobic coatings, heaters

Sensor cleaning hardware is already a dense patent thicket — spray nozzles, compressed-air curtains, ultrasonic vibration, heated defogging, hydrophobic coatings. Standardizing that domain would collide with existing patents and block adoption. So the standard leaves how you clean entirely to the vendor, and standardizes only how clearly you see and how you report it.

📡 The Reporting Protocol

Following the VDA5050 message model, each agent emits a JSON Sensor Clarity Report — a header plus a sensors[] array. Every sensor reports its sensorId, sensorClass, pci, state, observed contaminants[], lastCleanedAt, dwellSeconds (how long it has remained non-clear), and a self-diagnostic confidence. This gives a control center, an auditor, or a peer agent a single shared vocabulary for perception health.

🏆 Conformance Levels

Level Requirement
L-A Computes PCI and exposes the clarity-state enum
L-B L-A + emits the standard reporting message
L-C L-B + dwell-time SLA + safe-action trigger integration

🔗 A Cross-Cutting Standard

Perception Clarity does not stand alone. It interlocks horizontally with the wider WIA family — robot, auto, drone, lidar-sensor, vision-ai, and ai-sensor-fusion standards — so that any agent type can declare its clarity in the same terms. The agentType enum covers vehicle, robot, drone, amr, and other, and the standard references VDA5050, SAE J3016, ISO 21448 (SOTIF), and ISO 12233 / MTF.

🏢 About WIA

World Certification Industry Association (WIA) is an international standardization organization whose mission is to provide equal opportunities to all humanity through technology and certification.

WIA develops over 600 international standards across accessibility, security, AI, environment, healthcare, and physical-AI perception, and releases all standards as free open source.
