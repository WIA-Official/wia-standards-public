제7장: 적합성 레벨과 인증

WIA 인식 명료도 표준은 점진적 채택을 위해 세 가지 적합성 레벨 — L-A, L-B, L-C — 을 정의한다. 각 레벨은 측정·보고·안전 동작 연동의 단계적 확장이다. 이 장은 레벨 판정 기준, 적합성 선언 스키마, 인증 절차를 다룬다.

7.1 세 적합성 레벨

ConformanceLevel enum은 누적적이다. 상위 레벨은 하위 레벨의 모든 요건을 포함한다.

enum ConformanceLevel {
  L_A = "L-A"   // PCI 산출 + 상태 enum 노출
  L_B = "L-B"   // A + 표준 보고 메시지 송출
  L_C = "L-C"   // B + dwell-time SLA + 안전 동작 트리거 연동
}
레벨요건달성 내용
L-A측정PCI 0–100 산출 + ClarityState enum 노출
L-B측정 + 보고L-A + SensorClarityReport JSON 표준 송출
L-C측정 + 보고 + 동작L-B + dwell-time SLA + 안전 동작 트리거 연동

7.2 L-A — 측정의 기반

L-A는 가장 낮은 진입 레벨로, 센서 단위 PCI를 산출하고 그 결과를 ClarityState enum으로 노출하는 것을 요건으로 한다. L-A 적합 제품은 내부적으로 "지금 얼마나 잘 보이는가"를 0–100과 네 가지 상태로 표현할 수 있어야 한다. 다만 이를 외부에 표준 메시지로 송출할 의무는 없다.

L-A의 가치는 측정의 재현성에 있다. PCI 산출에 사용한 세 축(가림·거리저하·MTF)과 클래스별 가중치가 표준 정의(제3장)와 일치해야 한다. 이로써 제조사가 달라도 동일한 오염 조건에서 비교 가능한 명료도 값이 나온다.

7.3 L-B — 보고의 상호운용

L-B는 L-A에 더해 SensorClarityReport JSON 메시지를 표준 스키마(제5장)로 송출하는 능력을 요건으로 한다. 이 레벨에서 비로소 차량 ↔ 관제, 에이전트 ↔ 에이전트 간 명료도 정보의 상호운용이 가능해진다.

L-B 적합 검증은 메시지의 스키마 적합성과 값 제약(제5장 검증 규칙)을 모두 만족하는지를 확인한다. 필수 필드 존재, 값 범위, 상태-PCI 일관성, 가중치 합 등이 검사 대상이다.

7.4 L-C — 안전 동작의 폐루프

L-C는 최고 레벨로, L-B에 더해 dwell-time SLA를 선언하고 안전 동작 트리거(제6장)를 실제로 연동하는 것을 요건으로 한다. L-C 적합 제품은 측정→보고→동작의 폐루프(closed loop)를 완성한다. 즉 오염을 측정하고, 보고하고, 정의된 SLA를 넘으면 감속·우회·세척·safe-state·센서배제 같은 동작을 자동으로 실행한다.

L-C가 안전 표준과 맞닿는 지점

L-C는 ISO 21448 SOTIF와 SAE J3016의 fallback 개념과 직접 연동된다. PCI·dwell-time이 ODD의 인식 요구를 만족하지 못함을 판정하면, 시스템은 정의된 최소위험상태(MRC)로 전환한다. L-C 적합은 인식 성능 한계를 인지하고 안전하게 대응하는 능력을 입증하는 것이다.

7.5 적합성 선언 스키마

제조사는 제품의 적합성을 ConformanceDeclaration으로 선언한다. 이 선언은 인증·감사의 기준 문서가 된다.

{
  "declaration_id": "uuid",
  "version": "1.0.0",
  "vendor": "string",
  "product": "string",
  "agentType": "vehicle | robot | drone | amr | other",
  "conformanceLevel": "L-A | L-B | L-C",
  "sensors": [
    {
      "sensorClass": "SensorClass",
      "count": "number",
      "pciSupported": "boolean",
      "reportingSupported": "boolean",
      "dwellSlaSeconds": "number | null",
      "safeActionTriggers": ["slow_down | reroute | trigger_cleaning | safe_state | disable_sensor"]
    }
  ]
}

필수 필드는 declaration_id·version·vendor·conformanceLevel·sensors[]다. 선언 레벨과 실제 지원 능력은 일관되어야 한다. 예컨대 conformanceLevel이 L-C인데 어떤 센서의 dwellSlaSeconds가 null이면 모순이다.

7.6 레벨별 선언 일관성 검사

선언 레벨필수 일관성
L-A모든 센서 pciSupported = true
L-BL-A 조건 + reportingSupported = true
L-CL-B 조건 + dwellSlaSeconds ≠ null + safeActionTriggers 비어있지 않음

7.7 인증 절차

적합성 인증은 3단계 평가로 이루어진다. 이는 자율 시스템 인증의 일반적 모델을 인식 명료도 영역에 적용한 것이다.

인증 3단계
============
1. 문서 검토 (Documentation Review)
   - ConformanceDeclaration 스키마 적합성
   - 선언 레벨과 지원 능력의 일관성
   - PCI 가중치·임계 정의 명시 여부

2. 실험실 시험 (Laboratory Test)
   - 오염 재현 프로토콜(제4장)로 PCI 산출 재현성 검증
   - 보고 메시지 스키마·값 제약 적합성 (L-B 이상)
   - dwell-time SLA·안전 동작 트리거 동작 (L-C)

3. 실 운영 검증 (Field Validation)
   - 실제 운용 환경에서 명료도 보고의 정합성
   - 안전 동작의 적시성·예측 가능성 (L-C)

인증은 정기 감사와 함께 일정 기간 유효하다. 표준 버전이 Major로 갱신되면 재인증이 필요하다. Minor·Patch 갱신은 하위호환을 유지하므로 기존 인증의 연속성을 보장한다.

7.8 부분 적합과 혼합 차량대

실제 차량대는 여러 적합성 레벨의 에이전트가 섞여 운용될 수 있다. L-A 에이전트는 내부 측정만, L-B는 보고까지, L-C는 안전 동작까지 수행한다. 관제는 각 에이전트의 선언 레벨을 알고 그에 맞게 처리한다. 예컨대 L-B 에이전트는 명료도를 보고하지만 자체 안전 동작은 보장하지 않으므로, 관제가 보고를 받아 능동적으로 개입할 수 있다.

점진적 채택의 의의

세 레벨 구조는 기존 제품이 한 번에 모든 요건을 만족하지 않아도 표준 생태계에 진입할 수 있게 한다. 먼저 L-A로 측정을 표준화하고, L-B로 보고를 추가하고, L-C로 안전 동작을 완성하는 단계적 경로는 채택 장벽을 낮춘다. 이것이 VDA5050이 산업 표준이 된 방식이기도 하다.

다음 장에서는 인식 명료도 표준이 다른 WIA 피지컬 AI 표준들과 어떻게 횡단 연동되는지를 다룬다.
