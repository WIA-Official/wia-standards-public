WIA Perception Clarity

WIA Perception Clarity Standard v1.0.0

A cross-cutting international standard for how physical AI agents — vehicles, robots, drones, and AMRs — measure, state, and report how clearly their optical and sensing surfaces currently see. It standardizes the Perception Clarity Index (PCI), the clarity-state vocabulary, the contamination taxonomy, and a VDA5050-style reporting message. It deliberately does NOT standardize cleaning hardware or algorithms.

Executive Overview

A physical AI agent is only as trustworthy as the windows it sees through. A camera lens streaked with rain, a LiDAR cover caked in mud, or a radar radome glazed with ice can leave an otherwise sound system acting confidently on a corrupted view of the world — the "sees but is blind" failure mode. Until now there has been no common, reproducible way to express how clearly a sensor sees. The WIA Perception Clarity Standard fills exactly that gap.

Following the philosophy that made VDA5050 a de facto industry baseline, this standard fixes the interface rather than the mechanism. It defines how an agent measures clarity (the 0-100 PCI from three physical axes), states it (clear / degraded / obstructed / blind), and reports it (a JSON message that feeds sensor fusion and fleet control). How a vendor keeps a surface clean remains entirely a vendor choice.

What This Standard Covers

  • Measure — the Perception Clarity Index (PCI), a per-sensor 0-100 integer from occlusion, distance-degradation, and MTF/contrast axes with class-specific weights.
  • State — a four-value clarity enum mapping PCI bands to required behavior.
  • Report — a VDA5050-style SensorClarityReport JSON for agent-to-fleet and agent-to-agent communication.
  • Act — dwell-time SLAs and five standardized safe-action triggers.
  • Certify — three cumulative conformance levels, L-A / L-B / L-C.

Standard Architecture

WIA Perception Clarity Standard
===============================================================
  MEASURE   PCI 0-100  =  100 - 100 * ( w_occ*OCC + w_dist*DIST + w_mtf*MTF )
            axes: occlusion / distance-degradation / MTF (ISO 12233)
            class weights per: rgb_camera ir_thermal lidar_window
                               radar_radome ultrasonic
---------------------------------------------------------------
  STATE     clear 90-100 | degraded 60-89 | obstructed 30-59 | blind 0-29
---------------------------------------------------------------
  REPORT    SensorClarityReport JSON  (header + sensors[])
            contaminants[] · dwellSeconds · confidence
---------------------------------------------------------------
  ACT       dwell-time SLA  ->  slow_down | reroute | trigger_cleaning
                                | safe_state | disable_sensor
---------------------------------------------------------------
  CERTIFY   L-A measure  ->  L-B report  ->  L-C safe-action
===============================================================

Out of Scope (by design)

This standard does not specify cleaning hardware, nozzles, fluids, or vendor cleaning algorithms — an area already covered by a dense patent landscape. It standardizes only the measurement, state, and reporting of perception clarity, leaving how a surface is cleaned to vendor choice.
