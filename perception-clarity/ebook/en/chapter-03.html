Chapter 3: The Perception Clarity Index (PCI)

The Perception Clarity Index is the quantitative heart of the standard: a single 0-100 integer, computed per sensor, that says how clearly that sensor currently sees. PCI is not a vendor opinion or a black-box confidence score — it is a reproducible weighted combination of three physically grounded damage axes, with class-specific weights. This chapter defines the axes, the weights, the computation, and works through complete numerical examples for each sensor class.

3.1 Definition and Orientation

PCI is computed independently for every sensor and reported as an integer in the closed interval [0, 100]. Higher is clearer: 100 means a pristine, fully transparent aperture; 0 means total perceptual blindness. This orientation is deliberate so that PCI reads like a "health percentage" — intuitive to operators and monotonic with safety.

The index is derived from three damage axes, each expressed as a fraction in [0.0, 1.0] where 0.0 is no damage and 1.0 is total damage:

AxisSymbolDefinitionRange
Occlusion ratioOCCFraction of the effective field of view that is blocked0.0 - 1.0
Detection-distance degradationDIST(reference range − current effective range) / reference range0.0 - 1.0
Contrast / MTF reductionMTF1 − (current MTF50 / reference MTF50)0.0 - 1.0

3.2 The Computation

Each axis is a damage fraction; PCI subtracts the weighted total damage (scaled to 0-100) from a perfect 100:

PCI = round( 100 - 100 * ( w_occ * OCC + w_dist * DIST + w_mtf * MTF ) )

where  w_occ + w_dist + w_mtf = 1.00   (per sensor class)
       OCC, DIST, MTF  in [0.0, 1.0]
       PCI clamped to [0, 100]

Because the weights sum to 1.0 and each axis is bounded by 1.0, the weighted damage term is itself bounded by 1.0, so PCI before clamping naturally lands in [0, 100]; the clamp guards only against floating-point edge cases and out-of-spec axis inputs. The result is rounded to the nearest integer.

3.3 Class Weights

The three axes do not matter equally for every sensor — a fact established physically in Chapter 2. The standard fixes default weight sets per class. Implementations may tune thresholds for their operating environment but must expose the weights actually used (via the optional pciWeights field) for auditability.

Sensor Classw_occw_distw_mtf
rgb_camera0.400.250.35
ir_thermal0.400.350.25
lidar_window0.450.450.10
radar_radome0.300.650.05
ultrasonic0.500.500.00

Each row sums to exactly 1.00. Note how radar concentrates 0.65 on distance degradation — capturing RF attenuation as range collapse — and assigns only 0.05 to MTF, which is nearly meaningless for radio frequencies. Ultrasonic zeroes MTF entirely and splits its weight evenly between occlusion and distance. These weight sets encode the failure physics directly into the index.

3.4 PCI to State Mapping

The integer PCI maps to the four-value clarity state through fixed default bands. The bands are tunable per §6.2 of the data-format specification, but their order and meaning are invariant: CLEAR > DEGRADED > OBSTRUCTED > BLIND.

PCI RangeStateOperational meaning
90 - 100clearNominal; full trust
60 - 89degradedMonitor; consider cleaning
30 - 59obstructedSafe action: slow / reroute / clean
0 - 29blindStop using sensor; safe-state

3.5 Worked Example A — RGB Camera in Rain

A front camera in moderate rain shows a thin film over part of the frame. Measured axes: occlusion 0.18 (edge spray blocks a strip), distance degradation 0.22 (effective range shortened by low contrast), MTF reduction 0.40 (the film visibly softens fine detail).

Class: rgb_camera   weights = (0.40, 0.25, 0.35)
weighted damage = 0.40*0.18 + 0.25*0.22 + 0.35*0.40
               = 0.072 + 0.055 + 0.140
               = 0.267
PCI = round(100 - 100*0.267) = round(73.3) = 73   ->  state = degraded

PCI 73 places the camera in DEGRADED: usable but worth monitoring and a candidate for cleaning. (The specification's reference example reports the same value, 73, for this profile.)

3.6 Worked Example B — LiDAR Caked in Mud

A navigation LiDAR takes a mud splatter over a large angular sector. Measured axes: occlusion 0.81, distance degradation 0.88, MTF reduction 0.30 (some scatter, but resolution matters little for LiDAR).

Class: lidar_window   weights = (0.45, 0.45, 0.10)
weighted damage = 0.45*0.81 + 0.45*0.88 + 0.10*0.30
               = 0.3645 + 0.3960 + 0.0300
               = 0.7905
PCI = round(100 - 100*0.7905) = round(20.95) = 21  ->  state = blind

PCI 21 is BLIND: the LiDAR must be removed from the trusted set and a safe-state response taken. Note that even the relatively high MTF damage of 0.30 contributes only 3 points, because LiDAR weights contrast at just 0.10 — the index correctly refuses to be dominated by an axis that barely matters for this class.

3.7 Worked Example C — Radar Radome with Invisible Ice

A radome glazes with ice. Optically it looks merely wet, so a camera-style occlusion estimate is low, but RF range has collapsed. Measured axes: occlusion 0.15, distance degradation 0.70, MTF reduction 0.05.

Class: radar_radome   weights = (0.30, 0.65, 0.05)
weighted damage = 0.30*0.15 + 0.65*0.70 + 0.05*0.05
               = 0.045 + 0.455 + 0.0025
               = 0.5025
PCI = round(100 - 100*0.5025) = round(49.75) = 50  ->  state = obstructed

PCI 50 is OBSTRUCTED. This is the critical payoff of class-specific weighting: a naive equal-weight formula would have read the low optical occlusion and rated the radar healthy, perpetuating exactly the "sees but is blind" deception. Because radar weights distance degradation at 0.65, the RF range collapse drives PCI down honestly.

3.8 Worked Example D — Clear Sensor Baseline

A freshly cleaned roof LiDAR: occlusion 0.02, distance degradation 0.03, MTF reduction 0.05.

Class: lidar_window   weights = (0.45, 0.45, 0.10)
weighted damage = 0.45*0.02 + 0.45*0.03 + 0.10*0.05 = 0.009 + 0.0135 + 0.005 = 0.0275
PCI = round(100 - 100*0.0275) = round(97.25) = 97  ->  state = clear

PCI 97 is CLEAR — full trust, no action.

3.9 Reproducibility and Auditability

Because PCI is a deterministic function of measurable axes and published weights, two independent implementations measuring the same physical condition should produce closely matching values. This reproducibility is what lets PCI serve as evidence in safety audits and as a shared currency between agents and fleets — properties impossible for a private confidence heuristic. The axes and weights may be reported alongside the PCI (the optional axes and pciWeights fields) so that any consumer can recompute and verify the index.

Chapter Summary

PCI = round(100 − 100 × (w_occ·OCC + w_dist·DIST + w_mtf·MTF)), clamped to [0,100], with per-class weights summing to 1.00. The three axes — occlusion, distance degradation, and MTF reduction — are physically defined and bounded in [0,1]. Class weights encode failure physics: radar leans on distance (0.65), cameras balance occlusion and MTF, ultrasonic ignores MTF entirely. Worked examples show the index correctly flagging a rainy camera as degraded, a muddy LiDAR as blind, and — most importantly — an optically-clean but RF-iced radar as obstructed, defeating the "sees but is blind" trap.
