Shield Configuration & Threat Detection

2.1 Shield Configuration Schema

The shield configuration defines how WIA-AIR-SHIELD protects your wireless communications. This schema allows you to customize protection levels and behaviors.

2.1.1 Basic Configuration

{
  "shield_id": "shield-uuid-v4",
  "version": "1.0.0",
  "mode": "balanced",
  "enabled": true,
  "auto_detect": true,
  "created_at": "2025-01-15T10:30:00Z",
  "updated_at": "2025-01-15T10:30:00Z"
}
Field Type Description
shield_id String (UUID) Unique identifier for this shield instance
version String WIA-AIR-SHIELD version (semantic versioning)
mode Enum Protection mode: stealth, fortress, ghost, paranoid, balanced, performance
enabled Boolean Whether shield is active
auto_detect Boolean Enable automatic threat detection

2.3 Zero-Knowledge Proof Schema

2.3.1 Proof Structure

{
  "proof_id": "zkproof-uuid",
  "proof_type": "identity",
  "algorithm": "zk-SNARK",
  "created_at": "2025-01-15T15:00:00Z",
  "proof_data": {
    "commitment": "base64-encoded-commitment",
    "challenge": "base64-encoded-challenge",
    "response": "base64-encoded-response"
  },
  "public_inputs": {
    "predicate": "age_over_18",
    "threshold": 18
  },
  "verification_key": "base64-encoded-vk",
  "metadata": {
    "version": "1.0",
    "circuit_hash": "sha256-hash"
  }
}

2.3.2 Proof Types

Proof Type Purpose Example Use Case
identity Prove identity without revealing it Anonymous authentication
attribute Prove attribute without revealing value Age verification without birthdate
range Prove value in range without revealing it Sufficient balance without amount
membership Prove membership in set Authorized user without user ID
location Prove location in area without exact coordinates Proximity verification

2.4 Activity Log Schema

2.4.1 Activity Entry

{
  "entry_id": "activity-uuid",
  "timestamp": "2025-01-15T16:45:30Z",
  "event_type": "shield_activated",
  "category": "protection",
  "severity": "info",
  "details": {
    "mode": "fortress",
    "trigger": "user_manual",
    "environment": "public_wifi"
  },
  "actions_taken": [
    "Enabled all shields",
    "Started monitoring",
    "Verified access point"
  ],
  "metadata": {
    "device_id": "device-uuid",
    "network_ssid": "Cafe_WiFi",
    "location_type": "public"
  }
}

2.4.2 Event Types

  • shield_activated - Shield was enabled
  • shield_deactivated - Shield was disabled
  • threat_detected - A threat was identified
  • threat_blocked - A threat was successfully blocked
  • connection_verified - Connection passed verification
  • connection_rejected - Connection failed verification
  • mode_changed - Protection mode was changed
  • configuration_updated - Settings were modified
  • scan_completed - Environment scan finished
  • alert_raised - User was alerted to a threat

Schema Best Practices

  • Validation: Always validate incoming data against these schemas
  • Versioning: Include version numbers to support schema evolution
  • Timestamps: Use ISO 8601 format (UTC) for all timestamps
  • UUIDs: Use UUID v4 for unique identifiers
  • Enums: Use lowercase with underscores for enum values
  • Privacy: Never include personally identifiable information in logs
