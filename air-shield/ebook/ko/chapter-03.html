3.1 핵심 방패 API

핵심 방패 API는 WIA-AIR-SHIELD 보호를 활성화하고 제어하기 위한 기본 인터페이스를 제공합니다.

3.1.1 초기화

import { AirShield } from '@anthropic/wia-air-shield';

// 방패 인스턴스 생성
const shield = new AirShield({
  mode: 'balanced',
  autoDetect: true,
  alertCallback: (threat) => {
    console.warn('위협 감지:', threat);
  }
});

// 방패 활성화
await shield.activate();

// 상태 확인
const status = shield.getStatus();
console.log(`보호 활성: ${status.enabled}`);
console.log(`위협 수준: ${status.threatLevel}`);

3.1.2 방패 제어 메서드

메서드 매개변수 반환값 설명
activate() 없음 Promise<void> 방패 활성화
deactivate() 없음 Promise<void> 방패 비활성화
setMode(mode) ProtectionMode Promise<void> 보호 모드 변경
scan() 없음 Promise<ThreatReport> 위협 스캔

3.2 보호 계층 API

3.2.2 교란 계층

// 디코이 트래픽 시작

// 노이즈 강도 설정 (0-100)
shield.noise.setIntensity(75);

// 디코이 패턴 설정
shield.noise.setPattern('mimicry'); // 또는 'constant', 'bursty'

// 디코이 트래픽 중지

// 무선 송신기 검증

shield.verify.setStrictness('paranoid'); // relaxed, normal, strict, paranoid

3.3 영지식 API

3.3.1 증명자 API

// 신원 증명 생성
const identityProof = await shield.zk.prover.createIdentityProof(myIdentity);

// 속성 증명 생성 (예: 18세 이상)

  myBalance,
  [requiredAmount, Infinity]
);

// 증명 검증

if (isValid) {
  console.log('증명 검증 성공');
} else {
  console.error('증명 검증 실패');

3.4 WIA-AIR-POWER 통합

이모(AIR-SHIELD)는 삼촌(AIR-POWER)과 긴밀히 협력하여 무선 충전이 안전하도록 보장합니다:

import { AirShield } from '@anthropic/wia-air-shield';
import { AirPower } from '@anthropic/wia-air-power';

const shield = new AirShield();
const power = new AirPower();

// 충전 전 송신기 검증
power.onChargingStart(async (transmitter) => {
  // 이모가 삼촌의 친구인지 확인
  const verified = await shield.verify.verifyTransmitter(transmitter);

  if (!verified) {
    console.warn('🛡️ 이모: 이 충전기 수상해!');
    power.rejectTransmitter(transmitter);
    return;
  }

  // 충전 중 사이드채널 보호 활성화
  shield.enableSideChannelProtection();
  console.log('🛡️ 이모: 보호된 충전 활성화');
});

// 충전 중 위협 모니터링
power.onCharging((status) => {
  const threats = shield.scanForThreats();

  if (threats.length > 0) {
    power.pauseCharging();
    shield.alert.raise('threats_during_charging', threats);
  }
});

3.5 모니터링 및 진단 API

3.5.1 실시간 모니터링

// 실시간 모니터링 시작
shield.monitor.start({
  interval: 5000, // 5초마다 확인
  deepScan: false
});

// 모니터링 이벤트 청취
shield.monitor.onUpdate((status) => {
  console.log(`위협 수준: ${status.threatLevel}`);
  console.log(`활성 위협: ${status.activeThreats}`);
  console.log(`보호된 데이터: ${status.dataProtectedMB} MB`);
});

// 모니터링 중지
shield.monitor.stop();

API 모범 사례

  • 항상 await: 대부분의 방패 작업은 비동기입니다
  • 오류 처리: 네트워크 문제로 인해 검증이 실패할 수 있습니다
  • 적극적으로 모니터링: 신뢰할 수 없는 환경에서 실시간 모니터링 활성화
  • 먼저 검증: 무선 소스에 연결하기 전에 항상 검증
  • 정리: 민감한 세션에서 연결 해제하기 전에 cleanTraces() 사용
