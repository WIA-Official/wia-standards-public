Chapter 4: Phase 1 - Data Format Specification

The Data Format phase establishes the foundation for all WIA-ART-001 interoperability. This chapter provides detailed technical specifications for file formats, metadata schemas, color management, and quality tiers that ensure digital artwork can be reliably created, exchanged, and preserved across any platform or tool that implements the standard.

Understanding these specifications is essential for developers building WIA-compliant applications. The format requirements balance comprehensive functionality with practical implementability, ensuring that both simple tools and sophisticated platforms can participate in the standardized ecosystem.

4.1 Supported File Formats

WIA-ART-001 supports a carefully selected set of file formats that together address the needs of different digital art workflows. Each format has specific requirements for compliance, ensuring consistent interpretation across implementations.

4.1.1 Raster Graphics

Table 4.1: Raster Format Requirements
FormatUse CaseBit DepthRequirements

4.1.2 PNG Requirements

PNG (Portable Network Graphics) serves as the primary format for lossless digital artwork. All WIA-compliant implementations MUST support PNG with the following specifications:

{
  "format": "PNG",
  "version": "1.2",
  "compression": "deflate",
  "bitDepth": [8, 16],
  "colorType": ["truecolor", "truecolor-alpha"],
  "requiredChunks": {
    "iCCP": "ICC color profile (REQUIRED)",
    "iTXt": "UTF-8 metadata (REQUIRED)",
    "tEXt": "ASCII metadata (RECOMMENDED)",
    "tIME": "Modification timestamp (RECOMMENDED)"
  }
}
  • MUST include embedded ICC color profile (iCCP chunk)
  • MUST include UTF-8 metadata in iTXt chunks
  • SHOULD use 16-bit depth for professional/archival work
  • MUST NOT use any lossy preprocessing

4.1.3 JPEG Requirements

JPEG remains essential for photographic content and web distribution. While lossy compression is inherent to the format, quality parameters ensure acceptable fidelity:

  • Quality setting MUST be ≥85 for distribution
  • MUST use 4:4:4 chroma sampling (no subsampling)
  • MUST include EXIF 2.3+ metadata
  • SHOULD use progressive encoding for web

4.1.4 Vector Graphics

For vector artwork, SVG (Scalable Vector Graphics) is the primary format:

<?xml version="1.0" encoding="UTF-8"?>
<svg xmlns="http://www.w3.org/2000/svg"
     xmlns:wia="https://wia.org/standards/ART-001/v1"
     version="1.1"
     viewBox="0 0 1000 1000">
  <metadata>
    <rdf:RDF xmlns:rdf="http://www.w3.org/1999/02/22-rdf-syntax-ns#"
             xmlns:dc="http://purl.org/dc/elements/1.1/">
      <rdf:Description rdf:about="">
        <dc:title>Artwork Title</dc:title>
        <dc:creator>Artist Name</dc:creator>
        <dc:rights>© 2025 Artist Name</dc:rights>
      </rdf:Description>
    </rdf:RDF>
  </metadata>
  <!-- Artwork content -->
</svg>

4.2.3 Medium Enumeration

Valid values for the technical.medium field:

  • digital-painting - Created with digital painting tools
  • 3d-modeling - Three-dimensional computer graphics
  • vector-art - Vector-based illustration
  • pixel-art - Pixel-based art
  • generative - Algorithmically generated
  • photo-manipulation - Edited photography
  • mixed-media - Combination of techniques
  • ai-assisted - Created with AI tools

4.3 Color Space Requirements

Consistent color reproduction across devices requires proper color management. All WIA-compliant artwork MUST specify color space and embed appropriate ICC profiles.

Table 4.3: Standard Color Spaces
Color SpaceUsageGamutProfile
sRGBWeb, general useStandardIEC 61966-2-1
Adobe RGBProfessional printWideAdobe RGB (1998)

ProPhoto RGBArchival, editingUltra-wideROMM RGB
Color Management Rules:
  • MUST embed ICC color profile in all files
  • MUST specify color space in metadata
  • SHOULD use perceptual rendering intent for conversions
  • SHOULD provide sRGB version for web distribution

4.4 Quality Tiers

Different use cases require different quality levels. The standard defines three tiers:

4.4.1 Archival Master

  • Format: TIFF or PNG (lossless)
  • Bit Depth: 16-bit minimum
  • Color Space: ProPhoto RGB or Adobe RGB
  • Resolution: Maximum available
  • Metadata: Complete

4.4.2 Professional Distribution

  • Format: PNG or high-quality JPEG
  • Bit Depth: 8-bit or 16-bit
  • Color Space: Adobe RGB or sRGB
  • DPI: 300 minimum

4.4.3 Web Display

  • Format: WebP, AVIF, or JPEG
  • Bit Depth: 8-bit
  • Color Space: sRGB
  • File Size: Optimized

4.5 File Naming Convention

{artist-slug}_{artwork-slug}_{version}_{variant}.{ext}

Examples:
jane-doe_sunset-mountains_v1_master.tiff
jane-doe_sunset-mountains_v1_web.jpg
jane-doe_sunset-mountains_v1_thumb.webp

4.6 Validation Requirements

All WIA-compliant files MUST pass these validation checks:

  1. Valid file format structure
  2. Embedded ICC color profile
  3. Core metadata fields present
  4. Checksum matches content
  5. File size within acceptable limits
  6. Proper UTF-8 encoding for text
  7. Valid URI/URL references

4.7 Chapter Summary

Key Takeaways

  1. Format support includes PNG, JPEG, TIFF for raster; SVG for vector; glTF for 3D.
  2. Metadata schema provides comprehensive structure for technical, descriptive, and rights information.
  3. Color management requires embedded ICC profiles and specified color spaces.
  4. Quality tiers (Archival, Professional, Web) address different use cases.
  5. Validation requirements ensure interoperability across implementations.
