WIA-ART-001 - 파일 형식과 메타데이터 명세

4.1 데이터 형식 개요

Phase 1은 WIA-ART-001 표준의 기초입니다. 이 단계에서는 디지털 아트 파일의 구조, 지원 형식, 메타데이터 스키마, 그리고 색상 관리 규칙을 정의합니다.

🎯 Phase 1의 목표
  • 표준화된 파일 형식 지원
  • 일관된 메타데이터 스키마
  • 정확한 색상 재현
  • 장기 보존을 위한 아카이브 형식

PNG 요구사항:

│   ├── 아카이브: 16비트 필수
│   ├── 전문가: 16비트 권장, 8비트 허용
│   └── 웹: 8비트
├── 압축
│   ├── 레벨 6-9 권장
│   └── 무손실 압축만 사용
├── 색 공간
│   ├── sRGB, Adobe RGB, ProPhoto RGB, Display P3
│   └── ICC 프로파일 포함 필수 (아카이브)
└── 메타데이터
    ├── tEXt 청크: 기본 정보
    ├── iTXt 청크: 국제 텍스트
    └── eXIf 청크: EXIF 데이터

4.2.3 JPEG 상세 명세

JPEG 요구사항:
├── 품질
│   ├── 전문가: 90-100%
│   ├── 웹: 75-85%
│   └── 미리보기: 60-70%
├── 샘플링
│   ├── 4:4:4 (전문가)
│   └── 4:2:0 (웹)
├── 프로파일
│   ├── 표준 JPEG (베이스라인)
│   └── 프로그레시브 권장 (웹)
└── 메타데이터
    ├── EXIF: 촬영/생성 정보
    ├── IPTC: 저작권, 설명
    └── XMP: 확장 메타데이터

4.4.3 색 공간 변환

// 색 공간 변환 예시 (Node.js + Sharp)
import sharp from 'sharp';

async function convertToSRGB(inputBuffer) {
  return await sharp(inputBuffer)
    .toColorspace('srgb')
    .withMetadata({
      icc: 'srgb'  // sRGB 프로파일 포함
    })
    .toBuffer();
}

// 원본 프로파일 보존 변환
async function convertWithProfile(input, targetProfile) {
  const image = sharp(input);
  const metadata = await image.metadata();
  
  // 원본 ICC 프로파일 추출
  const sourceProfile = metadata.icc;
  
  return image
    .withMetadata({
      icc: targetProfile,
      exif: metadata.exif  // EXIF 보존
    })
    .toBuffer();
}

// TypeScript 예제: 청크 업로드로 대용량 파일 처리
async function uploadLargeArtwork(file: File) {
  const CHUNK_SIZE = 5 * 1024 * 1024; // 5MB chunks
  const totalChunks = Math.ceil(file.size / CHUNK_SIZE);
  
  for (let i = 0; i < totalChunks; i++) {
    const start = i * CHUNK_SIZE;
    const end = Math.min(start + CHUNK_SIZE, file.size);
    const chunk = file.slice(start, end);
    
    await uploadChunk(chunk, i, totalChunks);
  }
  
  // 모든 청크 업로드 후 병합 요청
  await finalizeUpload(file.name);
}

// 안전한 파일 업로드 검증
function validateUpload(file: File, metadata: ArtworkMetadata): ValidationResult {
  const checks = [
    // 1. MIME 타입 검증
    validateMimeType(file.type, metadata.format),
    
    // 2. 파일 크기 제한
    validateFileSize(file.size, MAX_FILE_SIZE),
    
    // 3. 파일 내용 검사 (매직 넘버)
    validateFileContent(file),
    
    // 4. 바이러스 스캔
    await scanForVirus(file),
    
    // 5. 메타데이터 스키마 검증
    validateSchema(metadata, WIA_ART_SCHEMA)
  ];
  
  return {
    valid: checks.every(c => c.passed),
    errors: checks.filter(c => !c.passed).map(c => c.error)
  };
}
