Chapter 4: Phase 1 - Data Format & Structure

4.1 EPUB3 Foundation

Phase 1 of WIA-EDU-008 establishes EPUB3 as the mandatory format for digital textbooks. EPUB3 is an open standard developed by the World Wide Web Consortium (W3C) and maintained by the EPUB 3 Community Group.

Why EPUB3?

  • Open Standard: No proprietary ownership or licensing fees
  • Reflowable Content: Automatically adapts to any screen size
  • Rich Media Support: Native support for audio, video, SVG, and MathML
  • Accessibility Built-In: Semantic HTML5 and ARIA support
  • Scripting Support: JavaScript for interactive elements
  • Global Adoption: Supported by hundreds of reading systems worldwide

4.2 Required File Structure

Every WIA-compliant digital textbook must follow this structure:

textbook.epub
├── META-INF/
│   ├── container.xml          # Points to OPF file
│   └── encryption.xml         # Optional DRM info
├── OEBPS/
│   ├── content.opf            # Package document (metadata, manifest, spine)
│   ├── toc.ncx                # Navigation (EPUB2 compatibility)
│   ├── nav.xhtml              # Navigation document (EPUB3)
│   ├── chapters/
│   │   ├── chapter01.xhtml
│   │   ├── chapter02.xhtml
│   │   └── ...
│   ├── images/
│   │   ├── cover.jpg
│   │   ├── figure01.svg
│   │   └── ...
│   ├── audio/
│   │   └── pronunciation.mp3
│   ├── video/
│   │   └── experiment.mp4
│   ├── css/
│   │   └── styles.css
│   └── js/
│       └── interactive.js
└── mimetype                   # Must be first file, uncompressed

4.3 Metadata Schema

Comprehensive metadata is crucial for discoverability, accessibility, and proper handling. The standard requires Dublin Core metadata extended with educational and accessibility properties.

Required Metadata Fields

Field Type Description Example
dc:title Text Primary title "Introduction to Biology"
dc:creator Text Author(s) "Smith, Jane; Doe, John"
dc:identifier Text ISBN or DOI "978-3-16-148410-0"
dc:language ISO 639 Primary language "en-US"
dc:publisher Text Publishing entity "Academic Press"
dc:date ISO 8601 Publication date "2025-01-15"
dc:subject Text Subject keywords "Biology, Life Sciences"
wia:educationLevel Text Target grade/level "Grade 9-10"

4.4 Accessibility Requirements

Phase 1 mandates WCAG 2.1 Level AA compliance at minimum. All content must be accessible to users with disabilities.

Image Accessibility

<img src="images/cell-structure.svg"
     alt="Diagram of a eukaryotic cell showing nucleus, mitochondria,
          endoplasmic reticulum, and Golgi apparatus"
     role="img"
     aria-describedby="cell-desc" />

<div id="cell-desc" class="image-description">
  A detailed cross-section of a eukaryotic cell. The large central
  nucleus contains chromatin. Surrounding it are various organelles
  including rod-shaped mitochondria for energy production...
</div>

Mathematical Notation

Mathematics must be encoded in MathML with fallback text:

<math xmlns="http://www.w3.org/1998/Math/MathML" alttext="E equals m c squared">
  <mi>E</mi>
  <mo>=</mo>
  <mi>m</mi>
  <msup>
    <mi>c</mi>
    <mn>2</mn>
  </msup>
</math>

4.5 Interactive Elements

Digital textbooks can include interactive exercises, simulations, and assessments. These must be accessible and work offline.

Interactive Quiz Example

<section class="quiz" aria-label="Chapter 1 Quiz">
  <h2>Self-Assessment</h2>

  <div class="question" role="group" aria-labelledby="q1">
    <p id="q1">What is the powerhouse of the cell?</p>
    <div role="radiogroup" aria-labelledby="q1">
      <label>
        <input type="radio" name="q1" value="a"> Nucleus
      </label>
      <label>
        <input type="radio" name="q1" value="b"> Mitochondria
      </label>
      <label>
        <input type="radio" name="q1" value="c"> Ribosome
      </label>
    </div>
  </div>

  <button onclick="checkAnswers()" aria-label="Check your answers">
    Submit
  </button>
</section>

4.6 Multimedia Integration

Audio and video must include transcripts and captions:

<video controls aria-label="Photosynthesis process animation">
  <source src="video/photosynthesis.mp4" type="video/mp4">
  <track kind="captions" src="captions/photosynthesis-en.vtt"
         srclang="en" label="English">
  <track kind="descriptions" src="descriptions/photosynthesis-en.vtt"
         srclang="en" label="Audio Description">
</video>

<details>
  <summary>Video Transcript</summary>
  <p>Photosynthesis begins when light energy is absorbed by chlorophyll...</p>
</details>

4.7 Annotation and Bookmarking Support

Content must support user annotations, highlights, and bookmarks. These are stored separately from the content and synchronized across devices (Phase 3).

Annotation Data Format

{
  "@context": "http://www.w3.org/ns/anno.jsonld",
  "type": "Annotation",
  "id": "urn:uuid:12345678-1234-1234-1234-123456789abc",
  "created": "2025-12-25T10:30:00Z",
  "creator": "user@example.com",
  "motivation": "highlighting",
  "target": {
    "source": "urn:isbn:978-3-16-148410-0",
    "selector": {
      "type": "FragmentSelector",
      "conformsTo": "http://www.w3.org/TR/media-frags/",
      "value": "xpointer(/html/body/section[2]/p[3])"
    }
  },
  "body": {
    "type": "TextualBody",
    "value": "Important concept for exam",
    "format": "text/plain",
    "language": "en",
    "color": "#FFFF00"
  }
}

4.8 Content Versioning

The standard requires version information in metadata to support content updates while preserving user annotations:

<meta property="wia:contentVersion">1.2.0</meta>
<meta property="wia:previousVersion">1.1.0</meta>
<meta property="wia:versionDate">2025-12-25</meta>
<meta property="wia:changelog">
  Updated Chapter 5 with new research findings.
  Corrected errors in Chapter 7 equations.
</meta>

📚 Chapter Summary

Key Takeaways:

  • EPUB3 is the mandatory format, providing openness, accessibility, and rich media support
  • Structured file organization ensures consistency and tool compatibility
  • Comprehensive metadata enables discoverability and proper content handling
  • WCAG 2.1 AA accessibility is mandatory, not optional
  • Interactive elements must be accessible and work offline
  • Multimedia requires captions, transcripts, and audio descriptions
  • Annotation format supports cross-platform synchronization
  • Version control preserves user work across content updates
