Chapter 3: Standard Overview & Architecture

3.1 Introduction to WIA-EDU-008

The WIA Digital Textbook Standard (WIA-EDU-008) is a comprehensive framework designed to address all the challenges identified in Chapter 2. It provides a complete specification for creating, distributing, and consuming digital educational content in a way that is accessible, interoperable, and student-centered.

Design Principles

The standard is built on five core principles:

  1. Universal Accessibility: Every feature must work for every user, regardless of ability or assistive technology
  2. Open Interoperability: Content must work across all platforms, devices, and ecosystems
  3. Privacy by Design: Student data protection is built into every layer, not added as an afterthought
  4. Student Ownership: Learners truly own their content and data
  5. Vendor Neutrality: No dependency on proprietary platforms or technologies

Standard Scope

WIA-EDU-008 covers:

  • File formats and data structures for educational content
  • Metadata schemas for discoverability and classification
  • API specifications for content delivery and interaction
  • Synchronization protocols for multi-device access
  • Accessibility requirements and testing procedures
  • Privacy and security standards
  • Analytics and assessment data formats
  • Integration patterns with learning management systems
  • Licensing and digital rights frameworks
  • Certification requirements for compliance

3.2 The 4-Phase Architecture

WIA-EDU-008 is organized into four progressive phases, each building upon the previous one. This modular approach allows for incremental implementation while ensuring that even Phase 1 compliance provides significant benefits.

Phase Focus Key Deliverables Complexity
Phase 1 Data Format EPUB3, JSON schemas, accessibility markup Low
Phase 2 API Interface REST APIs, authentication, content delivery Medium
Phase 3 Protocol Synchronization, offline support, versioning High
Phase 4 WIA Integration Ecosystem connectivity, certification, registry Medium

Phase Interdependencies

Each phase builds on the previous one:

  • Phase 1 is the foundation - all implementations must start here
  • Phase 2 requires Phase 1 compliance for data exchange
  • Phase 3 uses Phase 2 APIs for synchronization
  • Phase 4 integrates all previous phases into the broader WIA ecosystem

3.3 Phase 1: Data Format & Structure

📄 Phase 1 Overview

Phase 1 establishes the foundational data formats and structures for digital textbooks. It ensures that content is accessible, portable, and future-proof.

Core Format: EPUB3

WIA-EDU-008 mandates EPUB3 as the primary content format because:

  • It's an open, internationally recognized standard (ISO/IEC TS 30135)
  • Supports reflowable content that adapts to any screen size
  • Native support for multimedia (audio, video, interactive elements)
  • Built-in accessibility features (semantic markup, navigation)
  • Wide support across reading systems and platforms
  • Supports scripting for interactive exercises and simulations

Accessibility Markup

Phase 1 requires semantic HTML5 and ARIA attributes throughout:

  • All images must have alternative text (alt attributes)
  • Headings must follow proper hierarchy (h1, h2, h3, etc.)
  • Tables must have proper headers and captions
  • Interactive elements must have ARIA roles and labels
  • Language changes must be marked with lang attributes
  • Navigation landmarks must be properly defined

3.4 Phase 2: API Interface & Integration

🔌 Phase 2 Overview

Phase 2 defines standard APIs for accessing, delivering, and interacting with digital textbook content. This enables seamless integration with learning management systems and third-party tools.

RESTful API Specification

The standard defines a complete REST API with the following endpoints:

Endpoint Method Purpose
/api/v1/textbooks GET List available textbooks
/api/v1/textbooks/{id} GET Get textbook metadata
/api/v1/textbooks/{id}/content GET Download textbook content
/api/v1/textbooks/{id}/chapters/{chapter} GET Get specific chapter
/api/v1/annotations GET/POST/PUT/DELETE Manage user annotations
/api/v1/progress GET/POST Track reading progress
/api/v1/analytics GET Retrieve learning analytics

Authentication and Authorization

Phase 2 requires OAuth 2.0 or OpenID Connect for user authentication, with support for:

  • Single Sign-On (SSO) via SAML or OAuth
  • LTI (Learning Tools Interoperability) integration
  • API keys for server-to-server communication
  • Granular permissions (read, annotate, export, share)

3.5 Phase 3: Protocol & Synchronization

🔄 Phase 3 Overview

Phase 3 addresses the challenges of multi-device access, offline functionality, and content synchronization. It ensures students can seamlessly transition between devices without losing their work.

Synchronization Protocol

The standard defines a conflict-free synchronization protocol based on:

  • Vector Clocks: For tracking changes across devices
  • Operational Transformation: For resolving annotation conflicts
  • Delta Sync: Only changed data is transmitted
  • Offline Queue: Changes made offline are queued and synced when connected

What Gets Synchronized?

Data Type Sync Strategy Conflict Resolution
Reading Position Real-time Last-write-wins (most recent position)
Annotations Delta sync Operational transformation (merge all)
Bookmarks Delta sync Union (keep all bookmarks)
Settings/Preferences On-change Last-write-wins per setting
Assessment Responses Immediate First-write-wins (no overwrites)

Offline Support Requirements

Phase 3 compliance requires:

  • Full content available offline after initial download
  • All reading and annotation features work without connectivity
  • Changes tracked locally with timestamps
  • Automatic synchronization when connectivity is restored
  • Conflict resolution that preserves all user work
  • Clear UI indicators of sync status

3.6 Phase 4: WIA Ecosystem Integration

🌐 Phase 4 Overview

Phase 4 connects digital textbooks to the broader WIA ecosystem, enabling interoperability with other WIA standards, certification programs, and global educational infrastructure.

Cross-Standard Integration

WIA-EDU-008 integrates with other WIA standards:

  • WIA-EDU-001 (Digital Credentials): Issue verifiable credentials for course completion
  • WIA-EDU-005 (Learning Records): Store learning analytics in standardized format
  • WIA-ACCESS-001 (Accessibility): Ensure assistive technology compatibility
  • WIA-DATA-001 (Privacy): Student data protection and consent management

WIA Registry Integration

All compliant textbooks are registered in the global WIA registry:

  • Unique WIA identifiers for discoverability
  • Metadata indexing for search and recommendation
  • Version tracking and update notifications
  • Certification status and compliance verification
  • Publisher reputation and quality metrics

Certification Requirements

To receive WIA-EDU-008 certification, textbooks must:

  1. Pass automated conformance testing (format validation)
  2. Pass accessibility testing (WCAG 2.1 AA minimum)
  3. Pass interoperability testing (cross-platform compatibility)
  4. Undergo manual review by certified evaluators
  5. Demonstrate privacy compliance (GDPR, FERPA, COPPA)
  6. Complete security audit (data encryption, secure APIs)

3.7 Implementation Pathways

Different stakeholders can adopt the standard in ways that fit their needs:

For Publishers

  1. Phase 1 First: Convert existing content to EPUB3 with proper metadata
  2. Add APIs: Implement Phase 2 APIs for content delivery
  3. Enable Sync: Add Phase 3 synchronization capabilities
  4. Join Ecosystem: Register in WIA registry and obtain certification

For Reading System Developers

  1. Support EPUB3: Ensure proper rendering of Phase 1 content
  2. Implement APIs: Integrate with Phase 2 APIs for content access
  3. Add Sync: Implement Phase 3 synchronization protocol
  4. Connect to Registry: Enable discovery of WIA-certified content

For Educational Institutions

  1. Set Requirements: Require WIA-EDU-008 compliance in procurement
  2. Integrate LMS: Connect learning management systems via Phase 2 APIs
  3. Support Students: Provide devices and training for multi-device sync
  4. Monitor Compliance: Verify ongoing certification of adopted textbooks
