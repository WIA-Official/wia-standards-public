Chapter 6: Phase 3 - Protocol & Synchronization

6.1 Multi-Device Synchronization Challenge

Modern students access their textbooks from multiple devices: tablets in class, laptops at home, smartphones on the bus. Phase 3 ensures that reading progress, annotations, and preferences seamlessly sync across all devices, even when offline.

Synchronization Requirements

  • Real-time Sync: Changes propagate within seconds when online
  • Offline-First: Full functionality without internet connection
  • Conflict Resolution: Intelligent handling of simultaneous edits
  • Bandwidth Efficiency: Only sync changed data (delta sync)
  • Privacy Preservation: End-to-end encryption for user data
  • Cross-Platform: Works across iOS, Android, Windows, macOS, Linux

6.2 Synchronization Architecture

The standard uses a hybrid synchronization model combining real-time WebSocket updates with periodic REST API polling as fallback.

6.3 Conflict Resolution with Operational Transformation

When two devices make simultaneous annotations, Operational Transformation (OT) ensures both edits are preserved without data loss.

OT Example

Scenario: User highlights the same paragraph on both tablet and phone while offline.

Tablet: Highlights characters 100-150 in yellow

Phone: Highlights characters 120-180 in green

Resolution: System creates two separate, overlapping highlights preserving both user intents.

}

6.4 Offline-First Architecture

The application works fully offline with a local database, queueing sync operations for when connectivity returns.

Local Storage Strategy

  • Content: Full EPUB stored locally using IndexedDB
  • User Data: Annotations, bookmarks, progress in local SQLite
  • Sync Queue: Pending operations tracked with retry logic
  • Cache Strategy: Content cached indefinitely, media on-demand

6.5 Version Control and Content Updates

When textbook content is updated, the system preserves user annotations while applying the new content.

Update Process

  1. Detect new version available via metadata comparison
  2. Download delta patch (only changed content)
  3. Map existing annotations to new content structure
  4. Apply update while preserving user data
  5. Notify user of changes with changelog
  6. Offer option to view previous version for reference

6.6 Bandwidth Optimization

Delta sync ensures only changed data is transmitted, critical for students on limited data plans.

Data Compression Techniques

  • Binary Diff: For content updates, send only byte-level differences
  • JSON Patch: For metadata/annotations, use RFC 6902 JSON Patch
  • Deduplication: Identify and eliminate redundant sync operations
  • Batching: Combine multiple small operations into single request
  • Compression: Gzip/Brotli compression on all transfers

6.7 Security and Encryption

All synchronized data is encrypted end-to-end to protect student privacy.

Encryption Standards

  • Transport: TLS 1.3 for all network communication
  • Storage: AES-256 encryption for local data
  • Keys: User-specific keys derived from credentials
  • E2E: Optional end-to-end encryption for annotations
