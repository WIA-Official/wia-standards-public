5.1 Market Value Assessment
--------------------------------------------------------------------------------

Market value represents the price at which an asset would trade between
willing buyers and sellers in an open market.

[ID] Market Value Definition:

    MarketValueAssessment {
      definition: {
        concept:      "Fair Market Value (FMV)"
        description:  "Price between willing buyer and seller,
                      neither under compulsion, both with
                      reasonable knowledge of relevant facts"
        standard:     "Most commonly used valuation standard"
      }

      applicability: {
        ideal: [
          "Publicly traded securities",
          "Commodities",
          "Currency",
          "Assets with active markets"
        ]
        challenging: [
          "Private businesses",
          "Unique real estate",
          "Collectibles",
          "Intellectual property"
        ]
      }
    }

[ID] Market-Based Valuation:

    MarketBasedValuation {
      publicSecurities: {
        method:       "Last traded price or average"
        source:       "Major exchanges, authorized feeds"
        frequency:    "Real-time or daily close"
        adjustments:  "Block discounts for large positions"
      }

      realEstate: {
        method:       "Comparable sales analysis"
        process: [
          "Identify recent comparable sales",
          "Adjust for differences",
          "Weight adjustments",
          "Derive indicated value"
        ]
        comparables:  "Similar properties, same area, recent"
      }

      privateEquity: {
        method:       "Comparable transaction analysis"
        process: [
          "Identify similar company transactions",
          "Calculate transaction multiples",
          "Apply to subject company",
          "Apply discounts as appropriate"
        ]
        discounts:    "Lack of marketability, minority interest"
      }
    }

5.3 Discounted Cash Flow Analysis
--------------------------------------------------------------------------------

Discounted Cash Flow (DCF) analysis values assets based on the present
value of expected future cash flows.

[ID] DCF Framework:

    DCFFramework {
      concept: {
        principle:    "Value equals present value of future cash flows"
        formula:      "PV = Sum of (CF_t / (1 + r)^t)"
        components: [
          "Projected cash flows",
          "Discount rate",
          "Terminal value"
        ]
      }

      applicability: {
        ideal: [
          "Operating businesses",
          "Income-producing real estate",
          "Intellectual property with royalties",
          "Financial instruments"
        ]
        challenging: [
          "Early-stage companies",
          "Distressed situations",
          "Non-cash-producing assets"
        ]
      }
    }

[ID] DCF Calculation:

    DCFCalculation {
      projectionPeriod: {
        typical:      "5-10 years explicit projection"
        considerations: [
          "Business stability",
          "Predictability of cash flows",
          "Industry characteristics"
        ]
      }

      cashFlowProjection: {
        businessAsset: {
          startingPoint: "Historical free cash flow"
          adjustments: [
            "Revenue growth assumptions",
            "Margin trends",
            "Capital expenditure needs",
            "Working capital changes"
          ]
        }
        realEstate: {
          startingPoint: "Net operating income"
          adjustments: [
            "Rent growth",
            "Vacancy rates",
            "Operating expense changes",
            "Capital reserves"
          ]
        }
      }

      discountRate: {
        businessAsset: {
          method:     "Weighted Average Cost of Capital (WACC)"
          components: [
            "Cost of equity (CAPM or build-up)",
            "Cost of debt",
            "Capital structure weights"
          ]
          adjustments: "Size premium, company-specific risk"
        }
        realEstate: {
          method:     "Market extraction or build-up"
          components: [
            "Risk-free rate",
            "Property risk premium",
            "Market conditions"
          ]
        }
      }

      terminalValue: {
        methods: [
          "Gordon Growth Model: CF * (1+g) / (r-g)",
          "Exit Multiple: Final Year Metric * Multiple"
        ]
        growthRate:   "Typically 2-3% (inflation-linked)"
        sanityCheck:  "Should be reasonable % of total value"
      }
    }

[ID] DCF Example:

    DCFExample {
      asset:          "Small manufacturing business"
      projectedCashFlows: [
        { year: 1, cashFlow: 500000 },
        { year: 2, cashFlow: 550000 },
        { year: 3, cashFlow: 605000 },
        { year: 4, cashFlow: 660000 },
        { year: 5, cashFlow: 720000 }
      ]
      discountRate:   15.0
      terminalGrowthRate: 2.5
      terminalValue:  5904000    // 720000 * 1.025 / (0.15 - 0.025)

      calculation: {
        pvCashFlows:  2094000    // Sum of discounted yearly CFs
        pvTerminal:   2935000    // 5904000 / (1.15)^5
        totalValue:   5029000
      }

      conclusion: {
        indicatedValue: 5000000  // Rounded
        confidence:     "MEDIUM"
        sensitivities: [
          "1% change in discount rate = $400k value change",
          "1% change in growth rate = $350k value change"
        ]
      }
    }

5.4 Algorithmic Valuation
--------------------------------------------------------------------------------

Algorithmic valuation uses automated models for real-time or frequent
asset valuations.

[ID] Algorithmic Valuation Framework:

    AlgorithmicValuation {
      concept: {
        definition:   "Automated valuation using data models"
        advantages: [
          "Speed and efficiency",
          "Consistency",
          "Frequent updates",
          "Lower cost for routine valuations"
        ]
        limitations: [
          "May miss unique characteristics",
          "Dependent on data quality",
          "Less reliable for unusual assets"
        ]
      }

      applicability: {
        wellSuited: [
          "Residential real estate (AVMs)",
          "Public securities portfolios",
          "Cryptocurrency",
          "Standard financial instruments"
        ]
        poorlysuited: [
          "Unique commercial properties",
          "Private businesses",
          "Art and collectibles",
          "Complex derivatives"
        ]
      }
    }

[ID] Automated Valuation Models (AVMs):

    AVMFramework {
      realEstate: {
        dataInputs: [
          "Property characteristics",
          "Recent sales data",
          "Tax assessments",
          "Market indices"
        ]
        methodology: [
          "Hedonic regression models",
          "Repeat sales indices",
          "Neural networks"
        ]
        accuracy:     "Within 5-10% for typical properties"
        limitations:  "Less accurate for unique properties"
      }

      securities: {
        dataInputs: [
          "Market prices",
          "Trading volume",
          "Corporate actions"
        ]
        methodology:  "Direct price lookup, adjusted"
        accuracy:     "High for liquid securities"
        limitations:  "Block/liquidity discounts needed"
      }

      cryptocurrency: {
        dataInputs: [
          "Exchange prices",
          "Trading volume",
          "On-chain metrics"
        ]
        methodology:  "Volume-weighted average price"
        accuracy:     "Good for major coins"
        limitations:  "Volatile, exchange differences"
      }
    }

[ID] Model Governance:

    ModelGovernance {
      validation: {
        frequency:    "Annual backtesting"
        benchmarks:   "Compare to professional appraisals"
        thresholds:   "Investigation if variance > 15%"
      }

      oversight: {
        responsibility: "Trustee or investment committee"
        documentation:  "Model methodology documentation"
        approval:       "Required for model changes"
      }

      limitations: {
        disclosure:   "Beneficiaries informed of AVM use"
        restrictions: "Not used for contested valuations"
        escalation:   "Professional appraisal if disputed"
      }
    }

5.5 Multi-Method Reconciliation
--------------------------------------------------------------------------------

When multiple valuation methods are applied, reconciliation determines
the final value conclusion.

[ID] Reconciliation Process:

    ReconciliationProcess {
      step1_collectValues: {
        action:       "Gather all applicable valuations"
        sources: [
          "Market-based value",
          "Income approach (DCF)",
          "Cost approach",
          "Comparable transaction"
        ]
      }

      step2_evaluateQuality: {
        action:       "Assess reliability of each method"
        factors: [
          "Quantity and quality of data",
          "Appropriateness for asset type",
          "Adjustments required",
          "Verifiability of inputs"
        ]
      }

      step3_assignWeights: {
        action:       "Weight methods by reliability"
        typical: {
          operatingBusiness: {
            incomeApproach:   50
            marketApproach:   30
            costApproach:     20
          }
          realProperty: {
            salesComparison:  50
            incomeApproach:   30
            costApproach:     20
          }
        }
        justification: "Document reasoning for weights"
      }

      step4_calculate: {
        action:       "Compute weighted average"
        formula:      "Sum of (Value * Weight) / Sum of Weights"
        rounding:     "To appropriate precision"
      }

      step5_conclude: {
        action:       "State final value conclusion"
        format:       "Point value or range"
        confidence:   "State confidence level"
        documentation: "Full reconciliation report"
      }
    }

[ID] Reconciliation Example:

    ReconciliationExample {
      asset:          "Office building with tenants"

      valuations: {
        incomeApproach: {
          value:      4200000
          weight:     50
          notes:      "Strong data, stabilized income"
        }
        salesComparison: {
          value:      4500000
          weight:     35
          notes:      "Good comparables, some adjustments"
        }
        costApproach: {
          value:      3800000
          weight:     15
          notes:      "Functional obsolescence difficult"
        }
      }

      calculation: {
        weighted: "(4200000*50 + 4500000*35 + 3800000*15) / 100"
        result:   4245000
      }

      conclusion: {
        finalValue:   4250000
        range:        "4,100,000 - 4,400,000"
        confidence:   "HIGH"
        effectiveDate: "2075-12-01"
      }
    }

[!] Important: Reconciliation is judgment-based. The appraiser or trustee
    must document the reasoning for weights assigned and the final
    value selected.
