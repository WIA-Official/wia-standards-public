6.1 Risk Profile Management
--------------------------------------------------------------------------------

Investment strategy begins with understanding and managing risk appropriately
for the unique circumstances of preservation.

[ID] Risk Profile Framework:

    RiskProfileFramework {
      profiles: {
        CONSERVATIVE: {
          description:    "Capital preservation focus"
          targetReturn:   "Inflation + 1-2%"
          volatility:     "Low"
          drawdownTolerance: 10
          suitability: [
            "Short expected preservation period",
            "Older subject at preservation",
            "Limited assets relative to needs",
            "Subject preferences indicated"
          ]
        }

        MODERATE: {
          description:    "Balanced growth and preservation"
          targetReturn:   "Inflation + 3-4%"
          volatility:     "Medium"
          drawdownTolerance: 20
          suitability: [
            "Default for most situations",
            "Adequate assets for preservation",
            "Unknown preservation duration"
          ]
        }

        AGGRESSIVE: {
          description:    "Growth-oriented"
          targetReturn:   "Inflation + 5-6%"
          volatility:     "Higher"

          suitability: [
            "Young subject at preservation",
            "Very long expected preservation",
            "Substantial assets with margin",
            "Subject indicated growth preference"
          ]
        }
      }

      determination: {
        factors: [
          "Subject's documented preferences",

          "Asset level relative to needs",
          "Income requirements",
          "Contingent beneficiary interests"

        documentation: "Investment policy statement"

      }

      longevityRisk: {

        calculation:  "Model for 100+ year scenarios"

        mitigation:   "Maintain liquid reserves"

6.3 Rebalancing Strategies
--------------------------------------------------------------------------------

Rebalancing maintains target allocation as market values change.

[ID] Rebalancing Framework:

    RebalancingFramework {
      purposes: [
        "Maintain risk profile",
        "Enforce buy-low/sell-high discipline",
        "Prevent drift from policy allocation"
      ]

      triggers: {
        calendar: {
          description:  "Fixed schedule"
          options:      ["Monthly", "Quarterly", "Annually"]
          advantage:    "Simple, predictable"
          disadvantage: "May miss opportunities"
        }

        threshold: {
          description:  "When allocation drifts beyond tolerance"
          typicalBand:  "5% absolute deviation"
          advantage:    "Responds to market moves"
          disadvantage: "More complex to monitor"
        }

        hybrid: {
          description:  "Calendar review with threshold action"
          implementation: "Quarterly review, rebalance if >5% drift"
          advantage:    "Balance of simplicity and responsiveness"
          recommended:  true
        }
      }
    }

[ID] Rebalancing Process:

    RebalancingProcess {
      step1_review: {
        action:       "Calculate current allocation"
        data:         "Current market values by asset class"
        output:       "Allocation percentages"
      }

      step2_compare: {
        action:       "Compare to target allocation"
        calculation:  "Current % - Target % for each class"
        output:       "Deviation percentages"
      }

      step3_decision: {
        action:       "Determine if rebalancing needed"
        criteria:     "Any deviation exceeds threshold"
        documentation: "Record decision and rationale"
      }

      step4_execute: {
        action:       "Execute trades to rebalance"
        considerations: [
          "Tax efficiency (harvest losses)",
          "Transaction costs",
          "Liquidity constraints",
          "Market conditions"
        ]
      }

      step5_document: {
        action:       "Record rebalancing activity"
        contents: [
          "Pre-rebalance allocation",
          "Trades executed",
          "Post-rebalance allocation",
          "Rationale and considerations"
        ]
      }
    }

[ID] Tax-Efficient Rebalancing:

    TaxEfficientRebalancing {
      strategies: {
        directingCashFlows: {
          description:  "Use new money to buy underweight assets"
          advantage:    "No selling, no taxes"
          application:  "When cash flow available"
        }

        assetLocation: {
          description:  "Place tax-inefficient assets in tax-advantaged accounts"
          example:      "Bonds in IRA, stocks in taxable"
          benefit:      "Reduce tax drag on portfolio"
        }

        lossHarvesting: {
          description:  "Sell losers to realize losses"
          application:  "Rebalance while generating tax benefits"
          washSaleRule: "Avoid repurchase within 30 days"
        }

        specificLotSales: {
          description:  "Sell highest-basis lots first"
          implementation: "Specify lots when selling"
          benefit:      "Minimize capital gains"
        }
      }
    }

6.4 Sustainable Withdrawal Rates
--------------------------------------------------------------------------------

Sustainable withdrawal rates ensure assets last throughout preservation.

[ID] Withdrawal Rate Framework:

    WithdrawalRateFramework {
      concept: {
        definition:   "Percentage of assets withdrawn annually"
        goal:         "Sustain withdrawals without depleting assets"
        traditionalRule: "4% rule based on 30-year retirement"
        preservationChallenge: "Unknown, potentially much longer"
      }

      factors: {
        timehorizon: {
          shorter:    "Higher sustainable rate possible"
          longer:     "Lower rate required"
          unknown:    "Plan for very long duration"
        }

        assetAllocation: {
          moreEquity: "Higher rate may be sustainable"
          lesEquity:  "Lower rate required"
          tradeoff:   "Higher equity = higher volatility"
        }

        inflationAssumption: {
          higher:     "Lower real withdrawal sustainable"
          lower:      "Higher real withdrawal sustainable"
          planning:   "Assume moderate inflation"
        }

        failureTolerance: {
          zeroTolerance: "Very conservative rate needed"
          someTolerance: "Moderate rate possible"
          preservation:  "Zero tolerance required"
        }
      }
    }

6.5 Long-Duration Considerations
--------------------------------------------------------------------------------

Preservation investment strategies must account for potentially very long
time horizons with unique challenges.

[ID] Long-Duration Challenges:

    LongDurationChallenges {
      economicChange: {
        challenge:    "Economic systems may change dramatically"
        example:      "Currency changes, market structure"
        mitigation:   "Global diversification, real assets"
      }

      technologicalObsolescence: {
        challenge:    "Companies/sectors may become obsolete"
        example:      "Kodak, Blockbuster scenarios"
        mitigation:   "Index funds, periodic modernization"
      }

      regulatoryChange: {
        challenge:    "Tax and regulatory environment will change"
        example:      "Trust law modifications, tax changes"
        mitigation:   "Flexible trust structures, tax planning"
      }

      institutionalRisk: {
        challenge:    "Financial institutions may fail"
        example:      "Bank failures, custodian risks"
        mitigation:   "Multiple custodians, insurance limits"
      }

      currencyRisk: {
        challenge:    "Currency may lose value or disappear"
        example:      "Historical currency failures"
        mitigation:   "Multi-currency, real assets, crypto"
      }
    }

[ID] Long-Duration Strategies:

    LongDurationStrategies {
      globalDiversification: {
        rationale:    "No single country/currency dependence"
        implementation: [
          "Geographic diversification in equities",
          "Multi-currency bond holdings",
          "International real estate exposure"
        ]
      }

      realAssetEmphasis: {
        rationale:    "Tangible assets survive currency changes"
        implementation: [
          "Higher allocation to real estate",
          "Precious metals allocation",
          "Productive land if appropriate"

        rationale:    "Plan for multi-generational timeframe"

          "Dynasty trust structures",

          "Trust protector provisions"
        ]
      }

      adaptabilityProvisions: {
        rationale:    "Allow strategy evolution"
        implementation: [
          "Broad investment authority",

          "Decanting provisions"

      }
