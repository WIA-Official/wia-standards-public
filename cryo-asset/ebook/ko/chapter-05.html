5.1 시장가치 평가
--------------------------------------------------------------------------------

시장가치는 공개 시장에서 자발적인 매수자와 매도자 사이에서
자산이 거래될 가격을 나타냅니다.

[ID] 시장가치 정의:

    MarketValueAssessment {
      definition: {

                      양측 모두 관련 사실에 대해 합리적인
                      지식을 가진 상태에서의 가격"
        standard:     "가장 일반적으로 사용되는 가치평가 기준"
      }

      applicability: {
        ideal: [
          "공개 거래 증권",
          "상품",
          "통화",
          "활발한 시장이 있는 자산"

        challenging: [
          "비공개 사업체",

5.3 현금흐름할인 분석
--------------------------------------------------------------------------------

현금흐름할인(DCF) 분석은 예상 미래 현금흐름의 현재가치를 기반으로
자산을 가치평가합니다.

[ID] DCF 프레임워크:

    DCFFramework {
      concept: {
        principle:    "가치는 미래 현금흐름의 현재가치와 같음"
        formula:      "PV = Sum of (CF_t / (1 + r)^t)"
        components: [
          "예상 현금흐름",
          "할인율",
          "잔존가치"
        ]
      }

      applicability: {
        ideal: [
          "운영 사업체",
          "수익 창출 부동산",
          "로열티가 있는 지적재산권",
          "금융 상품"
        ]
        challenging: [
          "초기 단계 회사",
          "위기 상황",
          "현금 창출이 없는 자산"
        ]
      }
    }

[ID] DCF 계산:

    DCFCalculation {
      projectionPeriod: {
        typical:      "5-10년 명시적 예상"
        considerations: [
          "사업 안정성",
          "현금흐름 예측 가능성",
          "산업 특성"
        ]
      }

      cashFlowProjection: {
        businessAsset: {
          startingPoint: "과거 잉여현금흐름"
          adjustments: [
            "매출 성장 가정",
            "마진 추세",
            "자본 지출 필요",
            "운전자본 변화"
          ]
        }
        realEstate: {
          startingPoint: "순영업수익"
          adjustments: [
            "임대료 상승",
            "공실률",
            "운영 비용 변화",
            "자본 적립금"
          ]
        }
      }

      discountRate: {
        businessAsset: {
          method:     "가중평균자본비용 (WACC)"
          components: [
            "자기자본비용 (CAPM 또는 적층법)",
            "타인자본비용",
            "자본 구조 비중"
          ]
          adjustments: "규모 프리미엄, 회사 특수 위험"
        }
        realEstate: {
          method:     "시장 추출 또는 적층법"
          components: [
            "무위험이자율",
            "부동산 위험 프리미엄",
            "시장 상황"
          ]
        }
      }

      terminalValue: {
        methods: [
          "영구성장모델: CF * (1+g) / (r-g)",
          "출구 배수: 최종 연도 지표 * 배수"
        ]
        growthRate:   "일반적으로 2-3% (인플레이션 연동)"
        sanityCheck:  "총 가치의 합리적인 %여야 함"
      }
    }

[ID] DCF 예시:

    DCFExample {
      asset:          "소규모 제조 사업"
      projectedCashFlows: [
        { year: 1, cashFlow: 600000000 },
        { year: 2, cashFlow: 660000000 },
        { year: 3, cashFlow: 726000000 },
        { year: 4, cashFlow: 792000000 },
        { year: 5, cashFlow: 864000000 }
      ]
      discountRate:   15.0
      terminalGrowthRate: 2.5
      terminalValue:  7084800000    // 864000000 * 1.025 / (0.15 - 0.025)

      calculation: {
        pvCashFlows:  2512800000    // 할인된 연간 CF의 합계
        pvTerminal:   3522000000    // 7084800000 / (1.15)^5
        totalValue:   6034800000
      }

      conclusion: {
        indicatedValue: 6000000000  // 반올림
        confidence:     "MEDIUM"
        sensitivities: [
          "할인율 1% 변화 = 4.8억 가치 변화",
          "성장률 1% 변화 = 4.2억 가치 변화"
        ]
      }
    }

5.4 알고리즘 평가
--------------------------------------------------------------------------------

알고리즘 가치평가는 실시간 또는 빈번한 자산 가치평가를 위해
자동화된 모델을 사용합니다.

[ID] 알고리즘 가치평가 프레임워크:

    AlgorithmicValuation {
      concept: {
        definition:   "데이터 모델을 사용한 자동화된 가치평가"
        advantages: [
          "속도 및 효율성",
          "일관성",
          "빈번한 업데이트",
          "일상적인 가치평가에 대한 낮은 비용"
        ]
        limitations: [
          "독특한 특성을 놓칠 수 있음",
          "데이터 품질에 의존",
          "비정상적인 자산에 대해 덜 신뢰할 수 있음"
        ]
      }

      applicability: {
        wellSuited: [
          "주거용 부동산 (AVM)",
          "공개 증권 포트폴리오",
          "암호화폐",
          "표준 금융 상품"
        ]
        poorlysuited: [
          "독특한 상업용 부동산",
          "비공개 사업체",
          "예술품 및 수집품",
          "복잡한 파생상품"
        ]
      }
    }

    AVMFramework {

      }

      securities: {
        dataInputs: [

          "거래량",
          "기업 조치"
        ]

        accuracy:     "유동성 있는 증권에 대해 높음"

      }

      cryptocurrency: {
        dataInputs: [
          "거래소 가격",

5.5 다중 방법 조정
--------------------------------------------------------------------------------

여러 가치평가 방법이 적용될 때, 조정은 최종 가치 결론을
결정합니다.

[ID] 조정 절차:

    ReconciliationProcess {
      step1_collectValues: {
        action:       "모든 적용 가능한 가치평가 수집"
        sources: [
          "시장 기반 가치",
          "수익 접근법 (DCF)",
          "비용 접근법",
          "비교 거래"
        ]
      }

      step2_evaluateQuality: {
        action:       "각 방법의 신뢰성 평가"
        factors: [
          "데이터의 양과 품질",
          "자산 유형에 대한 적합성",
          "필요한 조정",
          "입력의 검증 가능성"
        ]
      }

      step3_assignWeights: {
        action:       "신뢰성에 따라 방법에 가중치 부여"
        typical: {
          operatingBusiness: {
            incomeApproach:   50
            marketApproach:   30
            costApproach:     20
          }
          realProperty: {
            salesComparison:  50
            incomeApproach:   30
            costApproach:     20
          }
        }
        justification: "가중치에 대한 근거 문서화"
      }

      step4_calculate: {
        action:       "가중 평균 계산"
        formula:      "Sum of (가치 * 가중치) / Sum of 가중치"
        rounding:     "적절한 정밀도로"
      }

      step5_conclude: {
        action:       "최종 가치 결론 진술"
        format:       "단일 가치 또는 범위"
        confidence:   "신뢰도 수준 진술"
        documentation: "전체 조정 보고서"
      }
    }

[ID] 조정 예시:

    ReconciliationExample {
      asset:          "임차인이 있는 사무실 건물"

      valuations: {
        incomeApproach: {
          value:      5040000000
          weight:     50
          notes:      "강한 데이터, 안정화된 수입"
        }
        salesComparison: {
          value:      5400000000
          weight:     35
          notes:      "좋은 비교 대상, 일부 조정"
        }
        costApproach: {
          value:      4560000000
          weight:     15
          notes:      "기능적 노후화 어려움"
        }
      }

      calculation: {
        weighted: "(5040000000*50 + 5400000000*35 + 4560000000*15) / 100"
        result:   5094000000
      }

      conclusion: {
        finalValue:   5100000000
        range:        "4,920,000,000 - 5,280,000,000"
        confidence:   "HIGH"
        effectiveDate: "2075-12-01"
      }
    }

[!] 중요: 조정은 판단 기반입니다. 감정인이나 수탁자는 부여된
    가중치와 선택한 최종 가치에 대한 근거를 문서화해야 합니다.
