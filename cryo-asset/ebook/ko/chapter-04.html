--------------------------------------------------------------------------------

4.2 수탁자 책임
--------------------------------------------------------------------------------

신임 책임을 집니다.

[ID] 신임 의무:

      dutyOfPrudence: {

          "위험 및 수익 목표",

4.3 수익자 관리
--------------------------------------------------------------------------------

신탁은 보존된 개인과 우발적 수익자를 포함한 여러 수익자 이익을
관리해야 합니다.

[ID] 수익자 범주:

    BeneficiaryCategories {
      primaryBeneficiary: {
        identity:       "보존된 개인"
        interest:       "회복 시 100%"
        rights: [
          "자산 회복 권리",
          "회계 정보에 대한 권리",
          "보존을 위한 신탁 분배 권리"
        ]
        representation: "후견인이 이익 대표"
      }

      contingentBeneficiaries: {
        identity:       "신탁 또는 법에 의해 지정된 상속인"
        interest:       "영구 사망 시 활성화"
        rights: [
          "회계에 대한 권리 (제한적)",
          "주요 조치 통지 권리",
          "부적절한 조치에 이의 제기 권리"
        ]
        condition:      "이익이 우발적, 확정되지 않을 수 있음"
      }

      remainderBeneficiaries: {
        identity:       "신탁 종료 후 수령자"
        interest:       "신탁 자산에 대한 미래 이익"
        rights:         "우발적 수익자와 유사"
      }

      charities: {
        identity:       "지정된 경우 자선 단체"
        interest:       "신탁 조건에 따라"
        rights:         "검찰을 통한 집행"
      }
    }

[ID] 수익자 권리:

    BeneficiaryRights {
      informationRights: {
        primaryBeneficiary: {

4.4 분배 및 적립금
--------------------------------------------------------------------------------

신탁 분배는 현재 필요와 장기 보존 목표 사이에서
균형을 유지해야 합니다.

[ID] 분배 프레임워크:

    DistributionFramework {
      preservationExpenses: {
        priority:       "최고"
        description:    "보존에 필요한 비용"
        examples: [
          "냉동보존 시설 비용",
          "의료 모니터링",
          "보험료",
          "법적 준수 비용"
        ]
        authorization:  "신탁 조건에 따라 자동"
      }

      trustAdministration: {
        priority:       "높음"
        description:    "신탁 관리 비용"
        examples: [
          "수탁자 보수",
          "전문가 비용 (법률, 회계)",
          "세금 신고",
          "자산 관리 비용"
        ]
        authorization:  "신탁 조건 또는 합리적"
      }

      assetMaintenance: {
        priority:       "중간"
        description:    "자산 가치 유지 비용"
        examples: [
          "재산세 및 보험",
          "부동산 관리",
          "사업 운영 비용",
          "IP 유지 비용"
        ]
        authorization:  "수탁자 재량"
      }

      discretionaryDistributions: {
        priority:       "낮음"

          "가족 긴급 상황"
        ]
        authorization:  "신탁 조건에 따른 수탁자 재량"

    법원 승인 없이 금지됩니다.
