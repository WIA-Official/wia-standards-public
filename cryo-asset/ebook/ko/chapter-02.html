--------------------------------------------------------------------------------
2.1 등록부 구조
--------------------------------------------------------------------------------

자산 등록부는 보존된 개인과 관련된 모든 자산을 추적하기 위한
중앙 저장소입니다.

[ID] 등록부 기록 구조:

    AssetRegistryRecord {
      registryId:       "REG-2075-KR-000012345"
      subjectId:        "CID-2075-KR-001234"
      subjectName:      "김보존"
      jurisdiction:     "KR"

      status:           "TRUST_MANAGED"
      statusHistory: [
        { status: "ACTIVE",        date: "2074-01-15", actor: "시스템" },
        { status: "FROZEN",        date: "2075-06-15", actor: "CRYO-LEGAL" },
        { status: "TRUST_MANAGED", date: "2075-07-01", actor: "법원" }
      ]

      trustInfo: {
        trustId:        "TR-2075-KR-000012345"
        trustName:      "김보존 보존 신탁"
        trustType:      "PRESERVATION_TRUST"
        establishedDate: "2075-07-01"
        trusteeName:    "글로벌 신탁 서비스 주식회사"
      }

      assetSummary: {
        totalAssets:    47
        totalValue:     4250000000.00
        currency:       "KRW"
        lastValuation:  "2075-12-01"

      createdAt:        "2075-06-15T10:30:00Z"
      updatedAt:        "2075-12-01T14:22:00Z"
    }

    | 구성요소          | 설명                                         |
    +------------------------------------------------------------------+

엄격한 규칙을 따릅니다.

    StatusTransitions {
      ACTIVE: {

        documentation:    "보존 선언 참조"

      FROZEN: {

        documentation:    "신탁 설정 또는 취소 문서"

      TRUST_MANAGED: {

      RESTORED: {
        validTransitions: []

      ESTATE_DISTRIBUTED: {

--------------------------------------------------------------------------------
2.3 등록부 작업
--------------------------------------------------------------------------------

등록부는 자산 관리를 위한 포괄적인 작업을 지원합니다.

[ID] 핵심 작업:

    RegistryOperations {
      createRegistry: {
        description:  "새 자산 등록부 초기화"
        trigger:      "보존 동의 완료"
        requiredData: [
          "대상자 신원 참조",
          "관할권",
          "초기 자산 목록"
        ]
        output:       "ACTIVE 상태의 새 등록부"
      }

      addAsset: {
        description:  "등록부에 새 자산 등록"
        authorization: "수탁자 또는 승인된 대리인"
        requiredData: [
          "자산 유형 및 세부사항",
          "가치평가 정보",
          "소유권 문서"
        ]
        validation:   "자산이 다른 곳에 이미 등록되지 않음"
      }

      updateAsset: {
        description:  "자산 정보 수정"
        authorization: "수탁자 또는 승인된 대리인"
        scope: [
          "가치평가 업데이트",
          "상태 변경",
          "위치 업데이트",
          "문서 업데이트"
        ]
        auditTrail:   "전체 변경 이력 유지"
      }

      removeAsset: {
        description:  "등록부에서 자산 제거"
        authorization: "수탁자 (필요시 법원 승인 포함)"
        validReasons: [
          "자산 매각/청산",
          "자산이 다른 기관으로 이전",
          "자산 파손/노후화",
          "중복 항목 정정"
        ]
        documentation: "제거 사유 및 지원 문서"
      }

      transferAsset: {
        description:  "등록부 간 자산 이동"
        authorization: "양측 수탁자 및 법원 승인"
        useCase:      "신탁 구조조정, 유산 분배"
        validation:   "소유권 체인 유지"
      }
    }

[ID] 조회 작업:

    QueryOperations {
      getRegistry: {
        parameters:   ["registryId"]
        returns:      "모든 자산이 포함된 전체 등록부 기록"
        authorization: "수탁자, 후견인, 승인된 당사자"
      }

      getAssetsByType: {
        parameters:   ["registryId", "assetType"]
        returns:      "지정된 유형의 모든 자산"
        useCase:      "범주별 분석"
      }

      getAssetsByStatus: {
        parameters:   ["registryId", "assetStatus"]
        returns:      "지정된 상태의 모든 자산"
        useCase:      "관리 워크플로우 추적"
      }

      getTotalValue: {
        parameters:   ["registryId", "currency", "valuationDate"]
        returns:      "집계된 자산 가치"
        useCase:      "포트폴리오 요약, 보고"
      }

      getStatusHistory: {
        parameters:   ["registryId"]
        returns:      "완전한 상태 전환 이력"
        useCase:      "감사 추적, 준수 확인"
      }

      searchAssets: {
        parameters:   ["검색 조건 객체"]
        returns:      "등록부 전체에서 일치하는 자산"
        authorization: "상승된 권한 필요"
        useCase:      "시스템 전체 자산 위치"
      }
    }

감사 기능을 유지합니다.

    AuditFramework {
      auditTrail: {

        contents: [
          "수행된 작업",
          "행위자 신원",
          "타임스탬프",

          "새 값",
          "승인 참조"
        ]
        immutability: "추가 전용, 암호화 봉인"
      }

      periodicAudits: {
        annual: {
          scope:      "전체 등록부 조정"
          auditor:    "독립 공인 감사인"

          findings:   "불일치는 설명되어야 함"

        quarterly: {
          scope:      "자산 존재 및 가치평가"

          findings:   "중요한 문제는 상위 보고"

        asNeeded: {
          scope:      "특정 우려사항 또는 사건"

    이어질 수 있습니다.
