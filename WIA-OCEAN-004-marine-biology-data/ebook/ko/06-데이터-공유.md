# 6장: 데이터 공유 및 상호운용성

## 모두가 해양 생물학 데이터에 접근할 수 있도록 만들기

해양 생물학 데이터는 공개적으로 공유되고 출처 간에 통합될 때만 완전한 가치를 달성합니다. 이 장에서는 전 세계 연구자들이 해양 생물다양성 및 환경 데이터를 발견, 액세스 및 결합할 수 있게 하는 저장소, API, 데이터 표준 및 모범 사례를 탐구합니다.

### 글로벌 해양 데이터 인프라

해양 생물학은 분산된 데이터 저장소 및 서비스 네트워크에 의존합니다:

```typescript
interface MarineDataRepository {
  name: string;
  organization: string;
  scope: "biodiversity" | "environmental" | "genomic" | "imagery" | "multi_domain";
  coverage: "global" | "regional" | "national";

  dataTypes: string[];
  recordCount: number;
  updateFrequency: "real_time" | "daily" | "monthly" | "irregular";

  access: {
    webPortal: string;           // URL
    api: APISpecification;
    bulkDownload: boolean;
    dataFormats: string[];       // ["CSV", "NetCDF", "JSON", "Darwin Core Archive"]
  };

  standards: {
    metadata: string[];          // ["ISO 19115", "Dublin Core", "FGDC"]
    data: string[];              // ["Darwin Core", "CF-NetCDF", "OBIS schema"]
    vocabularies: string[];      // ["WoRMS", "NERC P01", "BODC"]
  };

  licensing: {
    defaultLicense: string;      // "CC0", "CC-BY", "CC-BY-NC"
    commercialUseAllowed: boolean;
    attributionRequired: boolean;
  };
}
```

#### OBIS API 클라이언트 구현

```typescript
class OBISClient {
  private baseURL = "https://api.obis.org/v3";

  // 종 발생 검색
  async searchOccurrences(params: {
    scientificname?: string;
    aphiaID?: number;
    geometry?: string;           // WKT 폴리곤
    startdate?: string;          // ISO 8601
    enddate?: string;
    mindepth?: number;
    maxdepth?: number;
    size?: number;               // 페이지당 레코드 (최대 10000)
    after?: string;              // 페이지네이션 커서
  }): Promise<{
    total: number;
    results: OBISOccurrence[];
    after?: string;              // 다음 페이지 커서
  }> {
    const queryParams = new URLSearchParams();

    Object.entries(params).forEach(([key, value]) => {
      if (value !== undefined) {
        queryParams.append(key, String(value));
      }
    });

    const response = await fetch(`${this.baseURL}/occurrence?${queryParams}`);
    return response.json();
  }

  // 지역별 종 체크리스트 가져오기
  async getChecklist(params: {
    geometry: string;            // WKT 폴리곤 또는 POINT
    taxonid?: number;            // 상위 분류군으로 필터링
    mindepth?: number;
    maxdepth?: number;
  }): Promise<{
    total: number;
    results: {
      taxonID: number;
      scientificName: string;
      taxonRank: string;
      records: number;
    }[];
  }> {
    const queryParams = new URLSearchParams();
    Object.entries(params).forEach(([key, value]) => {
      if (value !== undefined) {
        queryParams.append(key, String(value));
      }
    });

    const response = await fetch(`${this.baseURL}/checklist?${queryParams}`);
    return response.json();
  }
}

interface OBISOccurrence {
  occurrenceID: string;
  scientificName: string;
  scientificNameID: string;      // WoRMS LSID
  taxonRank: string;
  kingdom: string;
  phylum: string;
  class: string;
  order: string;
  family: string;
  genus: string;

  decimalLatitude: number;
  decimalLongitude: number;
  coordinateUncertaintyInMeters?: number;
  minimumDepthInMeters?: number;
  maximumDepthInMeters?: number;

  eventDate: string;             // ISO 8601
  year: number;
  month?: number;
  day?: number;

  basisOfRecord: string;
  individualCount?: number;

  institutionCode: string;
  catalogNumber?: string;

  datasetID: string;
  datasetName: string;

  flags: string[];               // 품질 플래그
  absence: boolean;              // 부재 기록의 경우 true
}

// 사용 예
const obis = new OBISClient();

// 제주도 주변의 모든 산호 관측 찾기
const jejuCorals = await obis.searchOccurrences({
  scientificname: "Scleractinia",  // 경산호목
  geometry: "POLYGON((126 33, 127 33, 127 34, 126 34, 126 33))",
  size: 5000
});

console.log(`${jejuCorals.total}건의 기록 발견`);

// 동해 특정 지역의 종 체크리스트 가져오기
const checklist = await obis.getChecklist({
  geometry: "POINT(130.9 37.5)",
  mindepth: 0,
  maxdepth: 50
});

console.log(`${checklist.results.length}종이 지역에서 발견됨`);
```

### 데이터 통합 워크플로

```typescript
class MarineDataIntegrator {
  private obis: OBISClient;
  private kiost: KIOSTClient;

  constructor() {
    this.obis = new OBISClient();
    this.kiost = new KIOSTClient();
  }

  // 종 발생과 환경 데이터 통합
  async integrateOccurrencesEnvironment(params: {
    species: string;
    region: string;              // WKT 폴리곤
    startDate: Date;
    endDate: Date;
  }): Promise<{
    occurrenceID: string;
    scientificName: string;
    latitude: number;
    longitude: number;
    date: Date;
    temperature?: number;
    salinity?: number;
    chlorophyll?: number;
  }[]> {
    // 1. OBIS에서 종 발생 가져오기
    const occurrences = await this.obis.searchOccurrences({
      scientificname: params.species,
      geometry: params.region,
      startdate: params.startDate.toISOString(),
      enddate: params.endDate.toISOString(),
      size: 10000
    });

    // 2. 각 발생에 대해 환경 데이터 가져오기
    const integrated = await Promise.all(
      occurrences.results.map(async (occ) => {
        // 발생 위치/시간에서 환경 데이터 쿼리
        const env = await this.getEnvironmentalData({
          latitude: occ.decimalLatitude,
          longitude: occ.decimalLongitude,
          date: new Date(occ.eventDate)
        });

        return {
          occurrenceID: occ.occurrenceID,
          scientificName: occ.scientificName,
          latitude: occ.decimalLatitude,
          longitude: occ.decimalLongitude,
          date: new Date(occ.eventDate),
          temperature: env?.temperature,
          salinity: env?.salinity,
          chlorophyll: env?.chlorophyll
        };
      })
    );

    return integrated;
  }

  private async getEnvironmentalData(params: {
    latitude: number;
    longitude: number;
    date: Date;
  }): Promise<{
    temperature?: number;
    salinity?: number;
    chlorophyll?: number;
  } | null> {
    // KIOST 데이터 쿼리 (단순화됨)
    return await this.kiost.queryEnvironmentalData(params);
  }
}

class KIOSTClient {
  async queryEnvironmentalData(params: any) {
    // 구현 생략
    return {
      temperature: 18.5,
      salinity: 34.2,
      chlorophyll: 1.2
    };
  }
}
```

### 데이터 공유 모범 사례

**1. 영구 식별자 사용**
영구 인용을 위해 데이터셋에 DOI를 할당하세요.

**2. 포괄적인 메타데이터**
누가, 무엇을, 어디서, 언제, 왜, 어떻게를 포함하세요.

**3. 개방형 라이선스**
최대 재사용성을 위해 CC0 또는 CC-BY를 사용하세요.

**4. 표준 형식**
생물다양성에는 Darwin Core, 격자 데이터에는 CF-NetCDF.

**5. 품질 문서화**
QC 절차, 알려진 문제, 제한 사항을 문서화하세요.

**6. 버전 관리**
데이터셋 버전을 추적하세요; 이전 버전을 삭제하지 마세요.

**7. 프로그래밍 방식 액세스를 위한 API**
자동화된 데이터 발견 및 검색을 가능하게 하세요.

**8. 상호운용성**
권위 있는 분류학 (WoRMS), 어휘 (NERC)에 링크하세요.

### 한국-글로벌 데이터 통합 예시

```typescript
// 한국 해역 데이터를 글로벌 시스템과 통합
async function integrateKoreanMarineData() {
  const obis = new OBISClient();
  const kobic = new KOBICClient();

  // 1. KOBIC에서 한국 고유종 가져오기
  const koreanEndemics = await kobic.getEndemicSpecies();

  // 2. 각 종에 대해 글로벌 분포 확인
  for (const species of koreanEndemics) {
    const globalOccurrences = await obis.searchOccurrences({
      aphiaID: species.aphiaID,
      size: 10000
    });

    console.log(`${species.scientificName}:`);
    console.log(`  한국: ${species.recordCount}건`);
    console.log(`  전 세계: ${globalOccurrences.total}건`);

    // 3. 한국 기록을 OBIS에 기여
    if (species.notInOBIS) {
      await obis.contributeData(species.records);
      console.log(`  ✓ OBIS에 ${species.recordCount}건 기여`);
    }
  }
}

class KOBICClient {
  async getEndemicSpecies() {
    // KOBIC API 쿼리 (단순화됨)
    return [
      {
        scientificName: "Sebastes koreanus",
        aphiaID: 999999,
        recordCount: 234,
        notInOBIS: true,
        records: [/* 상세 기록 */]
      }
      // ... 더 많은 종
    ];
  }
}
```

### 철학: 공공재로서의 데이터

해양 생물학 데이터는 弘益人間을 따라 자유롭게 공유되는 글로벌 공유지를 나타냅니다. OBIS 데이터베이스만으로도 전 세계 수천 명의 연구자가 기여한 1억 건 이상의 관측을 포함하며, 모두 제한 없이 자유롭게 접근 가능합니다.

데이터를 공개적으로 공유할 때:
- 상상하지 못한 발견을 가능하게 합니다
- 연구 투자에 대한 수익을 극대화합니다
- 전 세계적으로 과학 진보를 가속화합니다
- 보전 및 지속 가능한 관리를 지원합니다
- 모든 기여자의 공헌을 존중합니다

데이터 공유는 단순히 모범 사례가 아니라 인류와 해양에 대한 우리의 책임입니다.
