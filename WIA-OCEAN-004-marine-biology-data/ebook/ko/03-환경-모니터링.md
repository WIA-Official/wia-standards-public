# 3장: 환경 모니터링 데이터

## 해양의 물리적 및 화학적 매개변수 측정

해양 생물은 온도, 염분, 해류, 영양염류 및 수많은 다른 요인에 의해 형성되는 복잡한 환경에 살고 있습니다. 종 분포, 생태계 건강 및 기후 변화 영향을 이해하려면 전 지구 해양에 배치된 센서에서 수집된 포괄적인 환경 모니터링 데이터가 필요합니다.

### 해양 관측 시스템

**Global Ocean Observing System (GOOS)**은 분산된 플랫폼 네트워크를 통해 해양 상태를 모니터링하는 국제적 노력을 조정합니다:

**Argo 플로트 어레이:**
- 해류를 따라 표류하는 **3,800개 이상의 자율 플로트**
- 10일마다 **2,000미터 깊이**까지 잠수
- 온도, 염분, 압력 측정
- 2000년 이래 **200만 개 이상의 프로파일** 수집
- 위성을 통한 실시간 데이터 전송

**계류 부이:**
- **1,200개 이상의 고정 위치** 기상 및 해양 부이
- 표면 조건의 연속 측정
- 특정 위치에서의 시계열 데이터
- 기후 모니터링 및 모델 검증에 중요

**연구선:**
- **100척 이상의 활성** 해양 연구선
- CTD 캐스트 (전도도-온도-깊이)
- 화학 및 생물학적 분석을 위한 물 샘플링
- 다중 빔 소나 수심 측량

### 한국의 해양 관측 인프라

한국은 자체 해역을 모니터링하기 위한 강력한 해양 관측 시스템을 운영합니다:

**국가해양관측망 (Korean Ocean Observing and Forecasting System, KOOFS)**
- 해안 및 항만 모니터링
- 실시간 조석 및 해류 데이터
- 수온 및 염분 관측
- 파고 및 주기 측정

**이어도 해양과학기지**
- 한국 최남단 해양 관측소
- 동중국해 환경 모니터링
- 기상, 해양, 지진 관측
- 태풍 경로 예측에 중요

**동해 해양 모니터링**
- 울릉분지 관측
- 대마난류 추적
- 동해 중층수 연구
- 기후변화 영향 평가

**서해 (황해) 관측**
- 조석 및 조류 모니터링
- 퇴적물 이동 연구
- 어장 환경 조사
- 적조 조기 경보 시스템

### 핵심 환경 매개변수

해양 생물학자는 종 관측을 위한 포괄적인 환경 맥락이 필요합니다:

```typescript
interface OceanEnvironmentalData {
  metadata: {
    platformType: "argo_float" | "moored_buoy" | "ship_ctd" | "glider" | "satellite";
    platformID: string;
    cruiseID?: string;            // 선박 기반 데이터의 경우
    program: string;              // 예: "GOOS", "KOOFS"
    dataProvider: string;
  };

  spatiotemporal: {
    timestamp: Date;              // UTC
    latitude: number;             // 십진 도
    longitude: number;
    depth: number;                // 미터 (양수 아래)
    bottomDepth?: number;         // 총 수심
  };

  physical: {
    temperature: {
      value: number;              // 섭씨
      unit: "degC";
      precision: number;
      sensor: string;
      calibrationDate: Date;
      qualityFlag: QualityFlag;
    };

    salinity: {
      value: number;              // PSU (실용 염분 단위)
      unit: "PSU";
      precision: number;
      calculationMethod: "measured" | "calculated_from_conductivity";
      qualityFlag: QualityFlag;
    };

    pressure: {
      value: number;              // 데시바
      unit: "dbar";
      precision: number;
      qualityFlag: QualityFlag;
    };

    density: {
      value: number;              // kg/m³
      unit: "kg/m3";
      type: "sigma_t" | "sigma_theta" | "potential_density";
      qualityFlag: QualityFlag;
    };

    current: {
      velocity: number;           // m/s
      direction: number;          // 진북 기준 각도 (0-360)
      uComponent: number;         // 동서 (m/s)
      vComponent: number;         // 남북 (m/s)
      method: "ADCP" | "surface_drifter" | "model";
      qualityFlag: QualityFlag;
    };
  };

  chemical: {
    dissolvedOxygen: {
      value: number;              // μmol/kg 또는 mg/L
      unit: "umol/kg" | "mg/L";
      saturation: number;         // 백분율
      method: "Winkler" | "optical_sensor";
      qualityFlag: QualityFlag;
    };

    pH: {
      value: number;              // pH 단위
      scale: "total" | "seawater" | "free";
      temperature: number;        // pH 측정 시 온도
      qualityFlag: QualityFlag;
    };

    nutrients: {
      nitrate: Measurement;       // μmol/L
      nitrite: Measurement;
      phosphate: Measurement;
      silicate: Measurement;
      ammonia?: Measurement;
      method: "colorimetric" | "chemiluminescence";
    };

    chlorophyll: {
      value: number;              // mg/m³
      method: "fluorometry" | "HPLC" | "satellite";
      qualityFlag: QualityFlag;
    };
  };

  biological_samples?: {
    planktonNet?: {
      meshSize: number;           // 마이크로미터
      volumeFiltered: number;     // 입방미터
      abundance: number;          // 입방미터당 생물
    };

    waterSample?: {
      volume: number;             // 리터
      preservation: string;       // "frozen", "formalin", "alcohol"
      analyses: string[];         // "eDNA", "phytoplankton", "nutrients"
    };
  };
}

type QualityFlag =
  | 0  // 품질 관리 없음
  | 1  // 양호한 데이터
  | 2  // 아마도 양호한 데이터
  | 3  // 잠재적으로 수정 가능한 불량 데이터
  | 4  // 불량 데이터
  | 5  // 값 변경됨
  | 6  // 사용되지 않음
  | 7  // 명목상 값
  | 8  // 보간된 값
  | 9  // 누락된 값;
```

### CTD 프로파일링: 골드 스탠다드

**CTD (전도도-온도-깊이)** 기기는 해양학의 주력으로 해양 특성의 수직 프로파일을 제공합니다:

```typescript
interface CTDCast {
  castID: string;
  ship: string;
  cruise: string;
  station: string;
  castDate: Date;

  location: {
    latitude: number;
    longitude: number;
    bottomDepth: number;          // 미터
  };

  instrument: {
    manufacturer: "Sea-Bird" | "RBR" | "other";
    model: string;
    serialNumber: string;
    lastCalibration: Date;
    sensors: {
      temperature: SensorSpec;
      conductivity: SensorSpec;
      pressure: SensorSpec;
      oxygen?: SensorSpec;
      fluorometer?: SensorSpec;
      transmissometer?: SensorSpec;
    };
  };

  profile: {
    maxDepth: number;             // 미터
    downcastRate: number;         // m/s
    upcastRate: number;
    sampleInterval: number;       // 초
    binSize: number;              // 평균 데이터를 위한 미터 간격
  };

  data: CTDDataPoint[];           // 각 깊이에서의 측정 배열

  processing: {
    software: string;             // "SBE Data Processing", "TEOS-10"
    version: string;
    filters: string[];            // 적용된 필터 (예: "7-point median")
    derivedVariables: string[];   // 계산된 매개변수
  };

  qualityControl: {
    performedBy: string;
    date: Date;
    flags: Record<string, QualityFlag>;
    notes: string;
  };
}
```

### 데이터 품질 관리

환경 데이터 품질은 해양 생물학 연구에 중요합니다:

```typescript
class OceanDataQualityControl {

  // 범위 확인: 물리적으로 현실적인 범위 내의 값
  rangeCheck(data: CTDDataPoint): QualityFlag {
    const ranges = {
      temperature: { min: -2, max: 40 },      // 섭씨
      salinity: { min: 0, max: 42 },          // PSU
      depth: { min: 0, max: 11000 }           // 미터
    };

    if (data.temperature < ranges.temperature.min ||
        data.temperature > ranges.temperature.max) {
      return 4;  // 불량 데이터
    }
    if (data.salinity < ranges.salinity.min ||
        data.salinity > ranges.salinity.max) {
      return 4;
    }

    return 1;  // 양호한 데이터
  }

  // 스파이크 확인: 비정상적인 단일 포인트 감지
  spikeCheck(data: CTDDataPoint[], index: number): QualityFlag {
    if (index === 0 || index === data.length - 1) return 1;

    const prev = data[index - 1];
    const curr = data[index];
    const next = data[index + 1];

    const tempGradient1 = Math.abs(curr.temperature - prev.temperature);
    const tempGradient2 = Math.abs(next.temperature - curr.temperature);

    // 온도 스파이크 임계값: 1미터에서 2°C 변화
    if (tempGradient1 > 2 || tempGradient2 > 2) {
      return 3;  // 잠재적으로 수정 가능
    }

    return 1;
  }

  // 안정성 확인: 센서 고착 감지
  stabilityCheck(data: CTDDataPoint[], windowSize: number = 10): QualityFlag {
    const window = data.slice(0, windowSize);
    const temps = window.map(d => d.temperature);

    const variance = this.calculateVariance(temps);

    // 분산이 너무 낮으면 센서 고착 의미
    if (variance < 0.0001) {
      return 4;  // 불량 데이터
    }

    return 1;
  }

  private calculateVariance(values: number[]): number {
    const mean = values.reduce((a, b) => a + b) / values.length;
    return values.reduce((sum, val) => sum + Math.pow(val - mean, 2), 0) / values.length;
  }
}
```

### 환경 데이터 모범 사례

**1. 항상 메타데이터 포함**
기기 유형, 보정 날짜 및 측정 방법을 기록하세요.

**2. 표준 단위 사용**
- 온도: 섭씨 (ITS-90)
- 염분: 실용 염분 단위 (PSU)
- 압력: 데시바 (dbar)
- 산소: 킬로그램당 마이크로몰 (μmol/kg)
- 영양염류: 리터당 마이크로몰 (μmol/L)

**3. 품질 플래그 구현**
모든 매개변수에 대해 표준화된 품질 플래그 (IODE/IOC 체계) 사용.

**4. 보정 문서화**
배치 전후 보정을 날짜 및 방법과 함께 문서화.

**5. 원시 데이터 보존**
처리된 버전과 함께 미처리 데이터를 유지.

**6. 표준 형식 사용**
격자 데이터에는 NetCDF-CF, 표 형식 데이터에는 메타데이터가 있는 CSV 사용.

### 철학: 모두를 위한 해양 측정

환경 모니터링 데이터는 글로벌 공공재입니다. Argo 프로그램만으로도 5억 달러의 국제 투자를 나타내며, 모든 데이터는 24시간 이내에 제한 없이 자유롭게 이용 가능합니다. 이것은 弘益人間 - 모든 인류에게 이익을 주는 것을 구현합니다.

CTD 프로파일이나 위성 이미지를 다운로드할 때, 센서를 배치하고, 플랫폼을 유지 관리하며, 데이터 품질을 보장한 수천 명의 과학자, 엔지니어 및 선원들의 작업에 접근하는 것입니다. 현명하게 사용하고, 자유롭게 공유하며, 가능할 때 기여하세요.
