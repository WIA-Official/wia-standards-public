# 5장: 생태계 모델링

## 해양 생태계 이해를 위한 데이터 통합

해양 생태계는 상호 작용하는 종과 환경 과정의 복잡한 네트워크입니다. 생태계 모델은 생물학적 관측, 환경 데이터 및 유전체 정보를 통합하여 먹이 그물을 이해하고, 어업 수확량을 예측하며, 기후 변화 영향을 평가하고, 보전 결정을 안내합니다.

### 해양 생태계 모델의 유형

다른 모델링 접근 방식은 다른 목적에 사용됩니다:

```typescript
interface EcosystemModel {
  modelType: "food_web" | "population" | "biogeochemical" | "end_to_end" | "spatial";
  purpose: string;
  spatialExtent: "local" | "regional" | "global";
  temporalExtent: {
    start: Date;
    end: Date;
    timestep: string;          // "hourly", "daily", "monthly"
  };

  components: ModelComponent[];
  forcingData: ForcingData[];
  parameters: ModelParameter[];
  calibration: CalibrationData;
  validation: ValidationData;
}
```

#### 먹이 그물 모델

영양 관계 및 에너지 흐름을 설명합니다:

```typescript
interface FoodWebModel {
  ecosystem: string;             // "coral_reef", "kelp_forest", "open_ocean"
  location: string;

  species: SpeciesNode[];
  interactions: TrophicLink[];

  structure: {
    trophicLevels: number;       // 일반적으로 4-6
    connectance: number;         // 실현된 링크의 비율 (0-1)
    complexity: number;          // 종 × 링크
  };

  energetics: {
    primaryProduction: number;   // gC/m²/년
    totalBiomass: number;        // gC/m²
    transferEfficiency: number;  // 영양 단계 간 (일반적으로 10%)
  };
}

interface SpeciesNode {
  aphiaID: number;
  scientificName: string;
  functionalGroup: string;       // "phytoplankton", "herbivorous_fish", "apex_predator"

  trophicLevel: number;          // 1 (생산자)에서 5+ (최상위 포식자)
  biomass: number;               // gC/m² 또는 톤/km²
  production: number;            // gC/m²/년
  consumption: number;           // gC/m²/년
}
```

### 모델 검증 및 보정

```typescript
class EcosystemModelValidator {
  validateModel(
    observations: number[],
    predictions: number[]
  ): {
    RMSE: number;
    bias: number;
    correlation: number;
    skillScore: number;
  } {
    const n = observations.length;

    // 평균 제곱근 오차
    const squaredErrors = observations.map((obs, i) =>
      Math.pow(obs - predictions[i], 2)
    );
    const RMSE = Math.sqrt(squaredErrors.reduce((a, b) => a + b) / n);

    // 편향 (평균 오차)
    const errors = observations.map((obs, i) => predictions[i] - obs);
    const bias = errors.reduce((a, b) => a + b) / n;

    // 상관관계
    const correlation = this.calculateCorrelation(observations, predictions);

    // Murphy 기술 점수 (1 = 완벽, 0 = 기술 없음, <0 = 기후학보다 나쁨)
    const obsVariance = this.calculateVariance(observations);
    const skillScore = 1 - (RMSE * RMSE) / obsVariance;

    return { RMSE, bias, correlation, skillScore };
  }

  private calculateCorrelation(x: number[], y: number[]): number {
    const n = x.length;
    const meanX = x.reduce((a, b) => a + b) / n;
    const meanY = y.reduce((a, b) => a + b) / n;

    let numerator = 0;
    let denomX = 0;
    let denomY = 0;

    for (let i = 0; i < n; i++) {
      const dx = x[i] - meanX;
      const dy = y[i] - meanY;
      numerator += dx * dy;
      denomX += dx * dx;
      denomY += dy * dy;
    }

    return numerator / Math.sqrt(denomX * denomY);
  }

  private calculateVariance(values: number[]): number {
    const mean = values.reduce((a, b) => a + b) / values.length;
    return values.reduce((sum, val) => sum + Math.pow(val - mean, 2), 0) / values.length;
  }
}
```

### 생태계 모델링 모범 사례

**1. 모델 선택**
질문에 적절한 복잡성을 선택하세요. 이해를 위한 단순 모델, 예측을 위한 복잡 모델.

**2. 매개변수 불확실성**
항상 매개변수 불확실성을 정량화하고 모델 출력을 통해 전파하세요.

**3. 여러 데이터 소스**
다양한 관측으로 보정: 위성, 현장, 조사, 실험.

**4. 검증**
보정에 사용되지 않은 독립적인 데이터로 테스트.

**5. 앙상블 접근**
구조적 불확실성을 포착하기 위해 여러 모델 실행.

**6. 투명성**
모든 가정, 방정식, 매개변수 및 데이터 소스를 문서화.

**7. 오픈 소스**
재현성 및 개선을 가능하게 하기 위해 모델 코드 및 데이터 공유.

### 철학: 공유된 이해로서의 모델
