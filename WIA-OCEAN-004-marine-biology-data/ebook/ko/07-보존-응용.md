# 7장: 보전 응용

## 데이터를 사용하여 해양 생물다양성 보호

해양 생물학 데이터는 전 세계 보전 결정에 직접 정보를 제공합니다 - 해양 보호 구역 지정부터 멸종 위기 종 상태 평가, 불법 어업 추적까지. 이 장에서는 표준화된 데이터가 弘益人間 (모든 인류에게 이익)의 원칙에 따라 효과적인 해양 보전을 가능하게 하는 방법을 탐구합니다.

### IUCN 적색 목록 평가

**국제자연보전연맹 (IUCN) 적색 목록**은 표준화된 기준과 데이터를 기반으로 전 세계 종의 멸종 위험을 평가합니다:

```typescript
interface IUCNRedListAssessment {
  assessmentID: string;
  aphiaID: number;
  scientificName: string;
  commonName: string;

  category: "EX" | "EW" | "CR" | "EN" | "VU" | "NT" | "LC" | "DD" | "NE";
  /*
    EX = 멸종 (Extinct)
    EW = 야생 멸종 (Extinct in the Wild)
    CR = 위급 (Critically Endangered)
    EN = 위기 (Endangered)
    VU = 취약 (Vulnerable)
    NT = 준위협 (Near Threatened)
    LC = 관심대상 (Least Concern)
    DD = 정보부족 (Data Deficient)
    NE = 미평가 (Not Evaluated)
  */

  criteria: string[];            // IUCN 기준 코드

  rationale: {
    populationTrend: "increasing" | "stable" | "decreasing" | "unknown";
    populationDecline: number;   // 기간 동안의 백분율
    timeFrame: number;           // 년 또는 세대

    rangeSize: {
      EOO: number;               // 분포 범위 (km²)
      AOO: number;               // 점유 면적 (km²)
      fragmentation: boolean;
      decline: boolean;
      fluctuations: boolean;
    };

    populationSize: {
      matureIndividuals: number;
      uncertainty: [number, number];  // 최소, 최대 추정
      numberOfSubpopulations: number;
      largestSubpopulation: number;
    };

    threats: Threat[];
  };

  assessment: {
    assessor: string[];
    reviewer: string[];
    assessmentDate: Date;
    previousCategory?: string;
    previousAssessmentDate?: Date;
    reasonForChange?: "genuine" | "new_information" | "taxonomy" | "criteria_revision";
  };
}

interface Threat {
  code: string;                  // IUCN 위협 분류 체계
  timing: "ongoing" | "past" | "future" | "unknown";
  scope: "whole" | "majority" | "minority" | "unknown";
  severity: "very_rapid" | "rapid" | "slow" | "negligible" | "unknown";
  score: "high" | "medium" | "low" | "unknown";
  stresses: string[];
}
```

### 불법 어업 탐지

데이터를 사용하여 불법, 비보고, 비규제 (IUU) 어업에 대응:

```typescript
interface VesselTrackingData {
  vesselID: string;              // IMO 또는 MMSI 번호
  vesselName: string;
  flag: string;                  // 등록 국가
  vesselType: string;

  ais: {                         // 자동 식별 시스템
    timestamp: Date;
    latitude: number;
    longitude: number;
    speed: number;               // 노트
    course: number;              // 도
    status: string;
  }[];

  violations: {
    type: "MPA_incursion" | "unlicensed_fishing" | "AIS_gap" | "transshipment";
    timestamp: Date;
    location: { latitude: number; longitude: number };
    evidence: string;
    severity: "high" | "medium" | "low";
  }[];
}

class IUUFishingDetector {
  // AIS 속도 패턴에서 어업 활동 감지
  detectFishingFromAIS(ais: VesselTrackingData['ais']): {
    timestamp: Date;
    location: { latitude: number; longitude: number };
    confidence: number;
  }[] {
    const fishingEvents: any[] = [];

    for (let i = 1; i < ais.length - 1; i++) {
      const prev = ais[i - 1];
      const curr = ais[i];
      const next = ais[i + 1];

      // 어선은 일반적으로 잦은 방향 변경과 함께 천천히 이동
      const avgSpeed = (prev.speed + curr.speed + next.speed) / 3;
      const courseChange = Math.abs(curr.course - prev.course);

      if (avgSpeed < 3 && courseChange > 30) {
        fishingEvents.push({
          timestamp: curr.timestamp,
          location: {
            latitude: curr.latitude,
            longitude: curr.longitude
          },
          confidence: 0.75
        });
      }
    }

    return fishingEvents;
  }

  // MPA 위반 감지
  detectMPAViolations(
    vessel: VesselTrackingData,
    mpas: MarineProtectedArea[]
  ): VesselTrackingData['violations'] {
    const violations: VesselTrackingData['violations'] = [];

    vessel.ais.forEach(point => {
      mpas.forEach(mpa => {
        if (this.pointInMPA(point.latitude, point.longitude, mpa)) {
          // 어업이 금지되어 있는지 확인
          const fishingProhibited = mpa.protection.regulations.some(
            reg => reg.activity === 'commercial_fishing' && reg.status === 'prohibited'
          );

          if (fishingProhibited) {
            violations.push({
              type: "MPA_incursion",
              timestamp: point.timestamp,
              location: {
                latitude: point.latitude,
                longitude: point.longitude
              },
              evidence: `${mpa.name}에서 어업이 금지된 선박`,
              severity: "high"
            });
          }
        }
      });
    });

    return violations;
  }

  private pointInMPA(lat: number, lon: number, mpa: MarineProtectedArea): boolean {
    // 간단한 점-폴리곤 확인
    // 실제 구현은 강력한 지리공간 라이브러리를 사용할 것
    return false;  // 플레이스홀더
  }
}
```

### 보전 데이터 모범 사례

**1. 장기 모니터링**
보호 전에 기준선을 확립하고 수십 년 동안 변화를 모니터링하세요.

**2. 표준화된 방법**
시간적 비교 가능성을 위해 일관된 조사 프로토콜을 사용하세요.

**3. 전후-대조-영향 (BACI) 설계**
보호 구역을 비보호 대조군과 비교하세요.

**4. 여러 지표**
생물다양성, 생물량, 서식지 및 사회경제적 결과를 평가하세요.

**5. 적응적 관리**
모니터링 데이터를 사용하여 관리 전략을 조정하세요.

**6. 이해관계자 참여**
지역 사회를 모니터링 및 의사 결정에 포함하세요.

**7. 개방형 데이터 공유**
보전 결과를 공유하여 글로벌 모범 사례를 알리세요.

### 철학: 지식을 통한 보전

보전 데이터는 弘益人間을 구현합니다 - 현재와 미래 세대를 위해 해양을 보호함으로써 모든 인류에게 이익을 줍니다. 보전에서 무엇이 효과적인지에 대한 데이터를 공유할 때, 우리는 다음을 가능하게 합니다:

- 증거 기반 정책 결정
- 글로벌 학습 및 개선
- 해양 회복에 대한 희망
