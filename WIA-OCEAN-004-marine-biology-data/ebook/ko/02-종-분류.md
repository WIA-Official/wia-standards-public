# 2장: 종 분류 및 분류학

## 해양 생물다양성의 조직화

해양은 미세한 박테리아부터 역사상 가장 큰 동물인 대왕고래까지 약 220만 종을 포함하고 있습니다. 이 놀라운 다양성을 조직하려면 전 세계 과학자들이 해양 생물에 대해 정확하게 소통할 수 있게 하는 정교한 분류 시스템이 필요합니다.

### 해양 분류학의 과제

해양 종은 육상 생물과 달리 독특한 분류학적 과제를 제시합니다:

**잠재종 (Cryptic Species):** 유전적으로 구별되지만 동일하게 보이는 생물. 유럽 해변게(*Carcinus maenas*)는 실제로 여러 잠재종으로 구성됩니다.

**형태학적 가소성:** 환경 조건이 외모에 극적인 영향을 미칩니다. 다른 깊이의 심해 물고기는 같은 종임에도 완전히 다르게 보일 수 있습니다.

**유생 단계:** 많은 해양 생물은 성체와 전혀 다른 모습의 플랑크톤 유생을 가지고 있어 식별이 어렵습니다.

**미발견 다양성:** 심해는 95%가 탐사되지 않은 채로 남아 있습니다. 우리는 **연간 2,000종 이상의 새로운 해양 종**을 발견하고 있습니다.

**역사적 복잡성:** 1700-1800년대에 기술된 종은 종종 불완전하거나 부정확한 설명을 가지고 있어 현대적 수정이 필요합니다.

### 린네 분류학

현대 분류학은 오늘날에도 여전히 사용되는 2부 학명 체계인 이명법을 확립한 Carl Linnaeus (1758)로 거슬러 올라갑니다.

#### 분류학적 계층구조

```typescript
interface TaxonomicClassification {
  domain: string;      // Eukarya, Bacteria, Archaea
  kingdom: string;     // Animalia, Plantae, Chromista 등
  phylum: string;      // Chordata, Arthropoda, Mollusca 등
  class: string;       // Mammalia, Actinopterygii, Gastropoda
  order: string;       // Cetacea, Perciformes, Nudibranchia
  family: string;      // Balaenopteridae, Labridae, Chromodorididae
  genus: string;       // Balaenoptera, Thalassoma, Chromodoris
  species: string;     // musculus, bifasciatum, quadricolor
  subspecies?: string; // 선택적 아종 지정
  scientificName: string;  // 전체 이명
  authority: string;       // 저자 및 연도 (예: "Linnaeus, 1758")
  taxonomicStatus: "accepted" | "synonym" | "unaccepted";
}

// 예: 대왕고래
const blueWhale: TaxonomicClassification = {
  domain: "Eukarya",
  kingdom: "Animalia",
  phylum: "Chordata",
  class: "Mammalia",
  order: "Cetacea",
  family: "Balaenopteridae",
  genus: "Balaenoptera",
  species: "musculus",
  scientificName: "Balaenoptera musculus",
  authority: "(Linnaeus, 1758)",
  taxonomicStatus: "accepted"
};
```

### 세계 해양 종 등록부 (WoRMS)

WoRMS는 전 세계 300명 이상의 분류학 편집자가 유지 관리하는 **해양 생물 이름의 권위 있는 글로벌 레지스트리** 역할을 합니다.

#### WoRMS 아키텍처

**데이터베이스 규모:**
- **240,000개 이상의 인정된 종** 이름
- **480,000개 이상의 동의어** 및 역사적 이름
- **56개 문** 대표
- **740,000개 이상의 총 분류학적 이름**

**업데이트 빈도:**
- 전문가 편집자의 **매일 업데이트**
- 연간 **2,000-3,000개의 새로운 종** 추가
- 기존 기록의 **지속적인 수정**

#### AphiaID 시스템

WoRMS의 모든 종은 고유한 **AphiaID**를 받습니다 - 학명이 수정되더라도 절대 변경되지 않는 영구 식별자입니다.

```typescript
interface WoRMSRecord {
  AphiaID: number;              // 고유 식별자
  scientificname: string;       // 전체 학명
  authority: string;            // 저자 및 연도
  status: "accepted" | "unaccepted";
  rank: string;                 // Kingdom, Phylum, Class 등
  valid_AphiaID: number;        // 동의어인 경우 인정된 이름을 가리킴
  valid_name: string;           // 인정된 학명
  kingdom: string;
  phylum: string;
  class: string;
  order: string;
  family: string;
  genus: string;
  citation: string;             // 이 기록을 인용하는 방법
  lsid: string;                 // 생명 과학 식별자
  isMarine: 0 | 1;
  isBrackish: 0 | 1;
  isFreshwater: 0 | 1;
  isTerrestrial: 0 | 1;
  isExtinct: 0 | 1;
  match_type: string;
  modified: string;             // ISO 8601 타임스탬프
}
```

#### WoRMS REST API

WoRMS는 자동화된 분류 체계 검증을 위한 프로그래밍 방식 액세스를 제공합니다:

```typescript
class WoRMSClient {
  private baseURL = "https://www.marinespecies.org/rest";

  // AphiaID로 종 기록 가져오기
  async getRecordByAphiaID(aphiaID: number): Promise<WoRMSRecord> {
    const response = await fetch(`${this.baseURL}/AphiaRecordByAphiaID/${aphiaID}`);
    return response.json();
  }

  // 이름으로 종 검색
  async searchByName(name: string): Promise<WoRMSRecord[]> {
    const response = await fetch(
      `${this.baseURL}/AphiaRecordsByName/${encodeURIComponent(name)}?like=false&marine_only=true`
    );
    return response.json();
  }

  // 전체 분류학적 분류 가져오기
  async getClassification(aphiaID: number): Promise<TaxonomicClassification[]> {
    const response = await fetch(`${this.baseURL}/AphiaClassificationByAphiaID/${aphiaID}`);
    return response.json();
  }

  // 여러 이름 매칭 (퍼지 매칭)
  async matchNames(names: string[]): Promise<WoRMSRecord[]> {
    const response = await fetch(`${this.baseURL}/AphiaRecordsByMatchNames`, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ scientificnames: names, marine_only: true })
    });
    return response.json();
  }

  // 일반명 가져오기
  async getVernaculars(aphiaID: number): Promise<VernacularName[]> {
    const response = await fetch(`${this.baseURL}/AphiaVernacularsByAphiaID/${aphiaID}`);
    return response.json();
  }
}

interface VernacularName {
  vernacular: string;     // 일반명
  language_code: string;  // ISO 639 언어 코드
  language: string;       // 언어 이름
}

// 사용 예
const worms = new WoRMSClient();

// 종 이름 검증
const records = await worms.searchByName("Tursiops truncatus");
console.log(`큰돌고래 AphiaID: ${records[0].AphiaID}`);

// 전체 분류 가져오기
const classification = await worms.getClassification(137111);
classification.forEach(taxon => {
  console.log(`${taxon.rank}: ${taxon.scientificname}`);
});

// 일반명 가져오기
const vernaculars = await worms.getVernaculars(137111);
// 반환: "큰돌고래" (한국어), "Common bottlenose dolphin" (영어), "Gran delfín" (스페인어) 등
```

### DNA 바코딩을 통한 종 식별

**Barcode of Life** 이니셔티브는 종 식별을 위한 표준화된 유전자 마커를 확립했습니다:

```typescript
interface DNABarcodeRecord {
  // 식별자
  processID: string;           // BOLD 프로세스 ID
  genBankID?: string;          // GenBank 수탁 번호
  aphiaID?: number;            // WoRMS 분류학 ID

  // 분류학
  scientificName: string;
  kingdom: string;
  phylum: string;
  class: string;
  order: string;
  family: string;
  genus: string;
  species: string;

  // 표본 데이터
  specimen: {
    catalogNumber: string;
    institution: string;       // 예: "국립해양생물자원관"
    collectionCode: string;
    fieldNumber?: string;
    lifeStage: "adult" | "juvenile" | "larva" | "egg" | "unknown";
    sex?: "male" | "female" | "hermaphrodite" | "unknown";
    tissueType: string;        // "muscle", "fin_clip", "whole_organism"
    preservation: string;      // "ethanol", "frozen", "dried"
  };

  // 채집 이벤트
  collection: {
    eventID: string;
    collectors: string[];
    collectionDate: Date;
    location: {
      locality: string;        // 자유 텍스트 설명
      latitude: number;
      longitude: number;
      coordinateUncertainty: number;
      depth?: number;          // 미터
      habitat: string;         // "coral_reef", "rocky_shore", "deep_sea"
    };
  };

  // 서열 데이터
  sequence: {
    markerCode: string;        // "COI-5P", "16S", "18S"
    nucleotides: string;       // FASTA 서열 (A,T,G,C,N)
    sequenceLength: number;
    ambiguousBases: number;    // 'N' (알 수 없는 염기) 개수
    GC_content: number;        // 백분율

    primers: {
      forward: string;         // 프라이머 이름/서열
      reverse: string;
    };

    sequencingFacility: string;
    sequencingMethod: "Sanger" | "Illumina" | "PacBio" | "Nanopore";
    sequencingDate: Date;
  };
}
```

### 한국 해양 종 데이터베이스

한국은 자체 해양 종 데이터베이스를 구축하고 있으며, 이는 WoRMS 및 기타 글로벌 시스템과 통합됩니다:

**한국해양생물목록 (Korean Register of Marine Species)**
- KIOST 및 MABIK에서 유지 관리
- WoRMS와 연계
- 한국 해역의 고유종 포함
- 한국어 및 영어 이름

**주요 한국 해양 종:**
- 참돔 (*Pagrus major*) - 중요한 어업 종
- 대게 (*Chionoecetes opilio*) - 동해의 주요 수산 자원
- 명태 (*Gadus chalcogrammus*) - 한국 문화에서 중요
- 미역 (*Undaria pinnatifida*) - 주요 양식 해조류

### 분류학 데이터 모범 사례

**1. 항상 AphiaID 사용**
학명과 함께 WoRMS AphiaID를 저장하세요. 이름은 변경되지만 ID는 변경되지 않습니다.

**2. 분류 체계 버전 관리**
WoRMS를 쿼리한 시점을 기록하세요. 분류 체계는 진화하므로 사용한 버전을 추적하세요.

**3. 전체 분류 저장**
필터링 및 분석을 위해 완전한 분류학적 계층구조 (계에서 종까지)를 유지하세요.

**4. 동의어 우아하게 처리**
사용자가 이전 이름을 검색할 때 설명과 함께 인정된 이름으로 리디렉션하세요.

**5. 입력 시 검증**
몇 달 후가 아니라 데이터가 입력될 때 WoRMS에 대해 종 이름을 확인하세요.

**6. 권위 있는 출처에 링크**
검증을 위해 WoRMS, GBIF, IUCN 기록에 대한 URL을 제공하세요.

**7. 불확실한 식별 문서화**
"cf." (비교), "aff." (유사), "sp." (종 미상)와 같은 한정어를 사용하세요.

### 철학: 해양의 이름 짓기

모든 종 이름은 수세기에 걸친 탐험, 연구, 헌신을 나타냅니다. WoRMS의 240,000종은 전 세계 분류학자들의 수백만 시간의 작업이며, 모든 인류의 이익을 위해 (弘益人間) 자유롭게 공유됩니다.

종을 올바르게 식별하면 관측을 다음과 연결합니다:
- 전 세계의 해당 종의 모든 다른 관측
- 진화적 관계
- 생태학적 역할
- 보전 상태
- 수세기에 걸친 과학적 지식

분류학은 단순히 이름 짓기가 아닙니다 - 생명 자체에 대한 우리의 이해를 조직하는 것입니다.
