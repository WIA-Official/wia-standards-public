CHAPTER 6

📊 Cultural Metadata and Documentation

Metadata is the information that makes cultural heritage discoverable, understandable, and usable. Without proper documentation, even the highest-quality 3D scan is merely a digital file with little research or educational value.

What is Cultural Heritage Metadata?

Metadata is "data about data"—structured information describing cultural objects, their context, creation, significance, and digital representation. It answers:

  • What: Object identification and description
  • Who: Creator, culture, current custodian
  • When: Creation date, historical period
  • Where: Origin, current location, excavation site
  • Why: Function, significance, cultural context
  • How: Materials, techniques, condition

Dublin Core Metadata Standard

Dublin Core provides 15 core elements for describing any resource, widely adopted across libraries, museums, and archives:

Core Elements

  1. Title: Name of the object
  2. Creator: Artist, maker, culture
  3. Subject: Topics, themes, keywords
  4. Description: Detailed account
  5. Publisher: Institution making it available
  6. Contributor: Others involved in creation/documentation
  7. Date: Creation date or period
  8. Type: Genre or format (sculpture, painting, etc.)
  9. Format: Physical medium and digital format
  10. Identifier: Unique reference (accession number, URI)
  11. Source: Related resources it derives from
  12. Language: Language of content/inscriptions
  13. Relation: Related resources
  14. Coverage: Spatial and temporal context
  15. Rights: Copyright, licensing, access restrictions

CIDOC Conceptual Reference Model (CRM)

CIDOC-CRM (ISO 21127) is the international standard ontology for cultural heritage information, providing rich semantic relationships.

Key Concepts

Entities: Physical objects, actors, places, time-spans, concepts, events

Relationships: How entities connect (was created by, consists of, took place at)

Events: Production, modification, acquisition, exhibition, conservation

Example CIDOC-CRM Structure

E22 Man-Made Object: "Tang Dynasty Vase"
  - P45 consists of: "Glazed ceramic"
  - P43 has dimension: "30cm height"
  - P53 has former or current location: "Xi'an Museum"
  - P108i was produced by: "Production Event"
      - P4 has time-span: "618-907 CE"
      - P7 took place at: "Xi'an, China"
      - P9 consists of: "Ceramic Firing Event"

Why CIDOC-CRM Matters

Unlike simple flat metadata, CIDOC-CRM captures complex relationships and provenance. It enables sophisticated queries like "Show me all objects made in the same workshop that traveled to Europe during the 19th century."

Technical Metadata

Documents the digital capture process and file characteristics:

Digitization Metadata

  • Scanning equipment: Make, model, calibration date
  • Scanning parameters: Resolution, accuracy, point density
  • Scanning date: When digitization occurred
  • Operator: Who performed the scan
  • Processing software: Tools used for processing
  • Processing steps: Cleaning, meshing, texturing

File Metadata

  • Format: File type and version (glTF 2.0, E57, etc.)
  • File size: Storage requirements
  • Checksums: MD5, SHA-256 for integrity verification
  • Compression: Algorithm if compressed
  • Color space: sRGB, Adobe RGB, etc.

Rights and Licensing Metadata

Copyright Information

  • Copyright holder (if applicable)
  • Copyright expiration date
  • Rights jurisdiction

Creative Commons Licensing

Common licenses for cultural heritage:

  • CC0: Public domain dedication
  • CC BY: Attribution required
  • CC BY-SA: Attribution + Share-Alike
  • CC BY-NC: Attribution + Non-Commercial

Cultural Rights

  • Indigenous cultural rights and protocols
  • Sacred or sensitive materials restrictions
  • Repatriation claims or requests

Linked Open Data

Publishing cultural heritage as Linked Data connects datasets globally:

Benefits

  • Discoverability: Cross-collection searches
  • Context: Link to external authority files (Getty, Wikidata)
  • Integration: Combine data from multiple institutions
  • Standardization: Consistent terminology and identifiers

Authority Files and Vocabularies

  • Getty AAT: Art & Architecture Thesaurus
  • Getty TGN: Thesaurus of Geographic Names
  • Getty ULAN: Union List of Artist Names
  • Wikidata: Collaborative knowledge base
  • GeoNames: Geographic database

Multilingual Metadata

Cultural heritage has global significance—metadata should reflect this:

  • Provide translations of core descriptive fields
  • Use Unicode (UTF-8) for all text fields
  • Include original language names alongside translations
  • Link to multilingual authority files
  • Respect cultural naming conventions

Metadata Management Systems

Collections Management Systems

  • CollectiveAccess: Open-source, flexible
  • ResourceSpace: Digital asset management
  • Omeka: Web publishing platform
  • Axiell Collections: Enterprise solution

Repository Systems

  • DSpace: Institutional repositories
  • Fedora: Flexible architecture
  • Archivematica: Digital preservation

WIA-EDU-023 Metadata Requirements

Minimum Metadata

  • ✅ Dublin Core 15 core elements
  • ✅ Basic CIDOC-CRM structure
  • ✅ Technical digitization metadata
  • ✅ Rights and licensing clear statement
  • ✅ Persistent identifier (DOI, ARK, Handle)
  • ✅ At minimum bilingual (English + source culture language)

Recommended Enhancements

  • Full CIDOC-CRM semantic modeling
  • Linked Data URIs to authority files
  • Provenance chain documentation
  • Conservation history
  • Exhibition history
  • Related publications bibliography

Metadata is Forever

While technology changes, metadata provides enduring context. Invest time in comprehensive documentation—future researchers will thank you. A well-documented artifact teaches; an undocumented one is merely a curiosity.

Comprehensive metadata transforms digital cultural heritage from isolated files into a rich, interconnected global knowledge network that serves research, education, and cultural understanding for generations to come.
