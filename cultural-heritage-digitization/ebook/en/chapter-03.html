CHAPTER 3

💾 Digital Preservation Standards

Creating a digital artifact is only the beginning. True preservation requires ensuring that digital files remain accessible, authentic, and usable for decades or centuries to come. This chapter explores the standards and strategies for long-term digital preservation.

The Digital Preservation Challenge

Unlike physical objects that can last thousands of years, digital data faces unique threats:

  • Media Degradation: Hard drives fail, optical discs degrade
  • Format Obsolescence: Software to read files becomes unavailable
  • Technology Evolution: Hardware and platforms change rapidly
  • Bit Rot: Slow corruption of data over time
  • Loss of Context: Metadata and documentation get separated from files

OAIS Reference Model

The Open Archival Information System (OAIS) provides the international standard (ISO 14721) for digital preservation. It defines:

Information Packages

  • Submission Information Package (SIP): What institutions submit for preservation
  • Archival Information Package (AIP): How data is stored long-term
  • Dissemination Information Package (DIP): What users receive when accessing content

File Format Standards

Preferred Formats (WIA-EDU-023)

3D Models:

  • glTF 2.0 (GL Transmission Format) - Web delivery and interoperability
  • X3D - Long-term archival, ISO standard
  • OBJ + MTL - Interchange format, widely supported
  • E57 - Point cloud archival standard

Images and Textures:

  • TIFF uncompressed - Master preservation copies
  • PNG - Lossless web delivery
  • JPEG 2000 - High-quality compressed archives

Documentation:

  • PDF/A (ISO 19005) - Long-term document preservation
  • XML - Structured metadata
  • Plain text (UTF-8) - Maximum longevity

Format Selection Criteria

Choose formats that are: 1) Non-proprietary and openly documented, 2) Widely adopted with multiple software implementations, 3) Uncompressed or losslessly compressed, 4) Self-documenting with embedded metadata, 5) Platform-independent.

Metadata Requirements

Essential Metadata Layers

Descriptive Metadata: What the object is

  • Title, creator, date, culture, materials
  • Historical context and significance
  • Physical location and provenance

Technical Metadata: How it was created

  • Scanning equipment and settings
  • File formats and specifications
  • Processing software and versions
  • Resolution and accuracy measurements

Administrative Metadata: Rights and management

  • Copyright and licensing
  • Access restrictions
  • Preservation events and migrations

Structural Metadata: How components relate

  • File relationships and dependencies
  • Multi-part object assemblies
  • Alternative representations

Storage Architecture

Redundancy Strategy (3-2-1 Rule)

  • 3 copies of every file
  • 2 different storage media types
  • 1 copy off-site for disaster recovery

Storage Technologies

  • Primary storage: RAID arrays, enterprise NAS
  • Nearline storage: Tape libraries for cost-effective backup
  • Cloud storage: Geographic distribution, managed redundancy

Data Integrity

Fixity Checking

Regular verification that files haven't changed using cryptographic checksums:

  • MD5, SHA-256 hashes generated at ingest
  • Automated verification on schedule (monthly/quarterly)
  • Alerts if corruption detected
  • Automatic restoration from backup copies

Migration and Emulation

Format Migration

Periodically converting files to current formats while preserving content and metadata. Balance between maintaining original formats versus ensuring future readability.

Emulation

Preserving original software environments to run obsolete formats. Useful for interactive digital experiences and complex file types.

Access Management

Persistent Identifiers

Assign permanent identifiers (DOIs, ARKs, Handles) so objects can be cited and located even if URLs change.

Access Tiers

  • Public access: Low-resolution models and general metadata
  • Research access: High-resolution models for qualified users
  • Preservation masters: Highest quality, restricted access

WIA-EDU-023 Preservation Checklist

  1. ✅ Files in approved archival formats
  2. ✅ Complete metadata package (Dublin Core + CIDOC-CRM)
  3. ✅ Technical documentation of creation process
  4. ✅ Cryptographic checksums for all files
  5. ✅ Three copies on different media
  6. ✅ One geographic backup location
  7. ✅ Persistent identifier assigned
  8. ✅ Rights and licensing clearly documented
  9. ✅ Scheduled integrity verification
  10. ✅ Migration plan for format obsolescence

Preservation Is an Ongoing Process

Digital preservation isn't a one-time event but a continuous commitment. Regular monitoring, periodic format migration, technology updates, and organizational sustainability are all essential to ensuring cultural heritage remains accessible for future generations.

By following these standards and best practices, institutions can ensure their digital cultural heritage collections remain authentic, accessible, and usable far into the future.
