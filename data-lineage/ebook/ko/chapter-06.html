6장

계보 시각화

그림 하나가 천 개의 메타데이터 레코드의 가치가 있습니다. 계보 시각화는 추상적인 그래프 데이터를 직관적이고 상호작용적인 다이어그램으로 변환하여 팀이 복잡한 데이터 관계를 한눈에 이해할 수 있도록 돕습니다.

시각화가 중요한 이유

텍스트 계보는 강력하지만 압도적일 수 있습니다. 1,000개 이상의 테이블이 있는 일반적인 데이터 웨어하우스를 생각해보세요. 원시 계보 메타데이터는 연결을 보여줄 수 있지만, 시각화는 패턴을 드러냅니다:

  • 병목 현상: 너무 많은 의존성이 있는 테이블
  • 고아: 리소스를 소비하는 사용되지 않는 데이터셋
  • 복잡성 핫스팟: 리팩토링이 필요한 영역
  • 데이터 흐름 패턴: 시스템을 통한 데이터 이동 방식
핵심 원칙: 좋은 시각화는 복잡한 계보를 비기술 이해관계자에게 접근 가능하게 만듭니다. CEO는 SQL 지식 없이도 데이터 흐름을 이해할 수 있어야 합니다.

시각화 유형

1. 방향성 비순환 그래프 (DAG)

클래식 계보 시각화:

  • 노드: 데이터 자산 (테이블, 컬럼, 보고서)
  • 엣지: 변환 및 의존성
  • 방향: 소스에서 목적지로의 데이터 흐름
  • 레이아웃: 위에서 아래 또는 왼쪽에서 오른쪽

2. 상키 다이어그램

변환을 통해 흐르는 데이터 볼륨을 보여줍니다:

  • 연결의 너비는 데이터 볼륨을 나타냅니다
  • 데이터 손실 또는 증폭을 식별하는 데 탁월합니다
  • 비효율적인 변환을 찾는 데 도움이 됩니다

3. 트리 뷰

단일 자산에 초점을 맞춘 계층적 표현:

  • 드릴다운을 위한 확장 가능한 노드
  • 업스트림 트리: 소스 및 의존성
  • 다운스트림 트리: 소비자 및 파생물

4. 매트릭스 뷰

테이블 간 의존성 매트릭스:

  • 행: 소스 테이블
  • 열: 목적지 테이블
  • 셀: 변환 유형 또는 개수
  • 결합 패턴을 빠르게 식별

5. 컬럼 레벨 흐름

컬럼 계보의 세밀한 시각화:

  • 개별 컬럼 변환 표시
  • PII 또는 민감한 데이터 흐름 추적
  • 더 복잡하지만 매우 상세함

상호작용 기능

필수 상호작용

  1. 클릭하여 포커스: 노드를 클릭하여 계보 경로 강조 표시
  2. 확대 및 이동: 큰 그래프 탐색
  3. 유형별 필터링: 테이블만 또는 보고서만 표시
  4. 검색: 특정 자산을 빠르게 찾기
  5. 확장/축소: 세부 수준 제어
  6. 메타데이터 툴팁: 호버로 상세 정보 표시

고급 상호작용

  • 시간 여행: 다른 시점의 계보 보기
  • 차이 보기: 버전 간 계보 비교
  • 영향 강조 표시: 변경의 영향 범위 표시
  • 중요 경로: 가장 중요한 의존성 강조 표시
  • 협업 주석: 팀원이 메모 추가 가능

6. 필터링 및 검색 기능

대규모 계보 그래프에서는 효과적인 필터링과 검색이 필수적입니다.

7. 그래프 레이아웃 알고리즘

계보 그래프를 시각적으로 명확하게 배치하는 것은 가독성에 중요합니다.

레이아웃 알고리즘 비교

알고리즘 장점 최적 사용
계층적 (Sugiyama) 명확한 방향성, 층별 구성 DAG, 파이프라인 흐름
Force-Directed 클러스터 자동 발견, 유기적 복잡한 네트워크, 탐색
Radial/Circular 중심 노드 강조, 대칭 단일 자산 중심 뷰
Tree (Reingold-Tilford) 계층 구조 명확, 공간 효율 업스트림/다운스트림 트리

시각화 라이브러리

D3.js 계보 시각화 예시
// D3.js로 상호작용 계보 그래프 생성
import * as d3 from 'd3';

class LineageViz {
    constructor(containerId, data) {
        this.container = d3.select(`#${containerId}`);
        this.data = data;
        this.width = 1200;
        this.height = 800;
    }

    render() {
        // SVG 캔버스 생성
        const svg = this.container.append('svg')
            .attr('width', this.width)
            .attr('height', this.height);

        // 줌 및 팬 활성화
        const g = svg.append('g');
        svg.call(d3.zoom()
            .scaleExtent([0.1, 4])
            .on('zoom', (event) => {
                g.attr('transform', event.transform);
            }));

        // Force 시뮬레이션 설정
        const simulation = d3.forceSimulation(this.data.nodes)
            .force('link', d3.forceLink(this.data.edges)
                .id(d => d.id)
                .distance(100))
            .force('charge', d3.forceManyBody().strength(-300))
            .force('center', d3.forceCenter(this.width / 2, this.height / 2));

        // 엣지 그리기
        const link = g.append('g')
            .selectAll('line')
            .data(this.data.edges)
            .enter().append('line')
            .attr('stroke', '#10B981')
            .attr('stroke-width', 2)
            .attr('marker-end', 'url(#arrowhead)');

        // 노드 그리기
        const node = g.append('g')
            .selectAll('circle')
            .data(this.data.nodes)
            .enter().append('circle')
            .attr('r', d => d.critical ? 12 : 8)
            .attr('fill', d => this.getNodeColor(d.type))
            .call(this.drag(simulation))
            .on('click', (event, d) => this.onNodeClick(d))
            .on('mouseover', (event, d) => this.showTooltip(event, d))
            .on('mouseout', () => this.hideTooltip());

        // 레이블 추가
        const label = g.append('g')
            .selectAll('text')
            .data(this.data.nodes)
            .enter().append('text')
            .text(d => d.name)
            .attr('font-size', 12)
            .attr('dx', 15)
            .attr('dy', 4);

        // 시뮬레이션 틱
        simulation.on('tick', () => {
            link
                .attr('x1', d => d.source.x)
                .attr('y1', d => d.source.y)
                .attr('x2', d => d.target.x)
                .attr('y2', d => d.target.y);

            node
                .attr('cx', d => d.x)
                .attr('cy', d => d.y);

            label
                .attr('x', d => d.x)
                .attr('y', d => d.y);
        });
    }

    getNodeColor(type) {
        const colors = {
            'table': '#3B82F6',
            'view': '#8B5CF6',
            'report': '#F59E0B',
            'dashboard': '#EF4444',
            'api': '#10B981'
        };
        return colors[type] || '#6B7280';
    }

    showTooltip(event, d) {
        const tooltip = this.container.append('div')
            .attr('class', 'lineage-tooltip')
            .style('position', 'absolute')
            .style('left', `${event.pageX + 10}px`)
            .style('top', `${event.pageY - 10}px`)
            .html(`
                ${d.name}

                Type: ${d.type}

                Owner: ${d.owner}

                Last Updated: ${d.lastUpdated}

                Upstream: ${d.upstreamCount}

                Downstream: ${d.downstreamCount}
            `);
    }

    drag(simulation) {
        function dragstarted(event) {
            if (!event.active) simulation.alphaTarget(0.3).restart();
            event.subject.fx = event.subject.x;
            event.subject.fy = event.subject.y;
        }

        function dragged(event) {
            event.subject.fx = event.x;
            event.subject.fy = event.y;
        }

        function dragended(event) {
            if (!event.active) simulation.alphaTarget(0);
            event.subject.fx = null;
            event.subject.fy = null;
        }

        return d3.drag()
            .on('start', dragstarted)
            .on('drag', dragged)
            .on('end', dragended);
    }
}

8. 성능 최적화

대규모 계보 그래프의 시각화 성능을 최적화하는 방법:

성능 전략

  1. 가상화 (Virtualization): 보이는 영역의 노드만 렌더링
  2. 레벨 오브 디테일 (LOD): 줌 레벨에 따라 세부 정보 조정
  3. 점진적 로딩: 필요에 따라 계보 확장
  4. 캐싱: 자주 접근하는 계보 경로 캐시
  5. 웹 워커: 백그라운드에서 레이아웃 계산
  6. Canvas vs SVG: 대규모 그래프에는 Canvas 사용
점진적 계보 로딩
class ProgressiveLineageLoader {
    constructor(apiClient) {
        this.api = apiClient;
        this.loadedNodes = new Set();
        this.graph = { nodes: [], edges: [] };
    }

    async loadInitial(rootDataset, initialDepth = 2) {
        // 초기 계보 로드 (얕은 깊이)
        const lineage = await this.api.getLineage(rootDataset, {
            depth: initialDepth,
            include_metadata: false  // 빠른 로딩
        });

        this.graph = lineage;
        this.loadedNodes = new Set(lineage.nodes.map(n => n.id));

        // 시각화 렌더링
        this.renderGraph();
    }

    async expandNode(nodeId) {
        // 노드 클릭 시 추가 계보 로드
        if (this.loadedNodes.has(nodeId)) {
            // 이미 로드됨
            return;
        }

        // 해당 노드의 추가 업스트림/다운스트림 가져오기
        const additionalLineage = await this.api.getLineage(nodeId, {
            depth: 1,
            direction: 'both'
        });

        // 그래프에 병합
        this.mergeLineage(additionalLineage);
        this.loadedNodes.add(nodeId);

        // 증분 업데이트
        this.updateGraph(additionalLineage);
    }

    mergeLineage(newLineage) {
        // 중복 제거하면서 병합
        const existingIds = new Set(this.graph.nodes.map(n => n.id));

        newLineage.nodes.forEach(node => {
            if (!existingIds.has(node.id)) {
                this.graph.nodes.push(node);
            }
        });

        newLineage.edges.forEach(edge => {
            this.graph.edges.push(edge);
        });
    }

    updateGraph(newData) {
        // 애니메이션과 함께 새 노드 추가
        // 기존 시각화 유지하면서 업데이트
    }
}

9. 실시간 계보 업데이트

데이터 파이프라인이 실행될 때 계보 시각화를 실시간으로 업데이트합니다.

WebSocket 기반 실시간 업데이트
class RealtimeLineageViz {
    constructor(containerId, apiEndpoint) {
        this.viz = new LineageViz(containerId, { nodes: [], edges: [] });
        this.ws = new WebSocket(`ws://${apiEndpoint}/lineage/live`);

        this.setupWebSocket();
    }

    setupWebSocket() {
        this.ws.onmessage = (event) => {
            const update = JSON.parse(event.data);

            switch (update.type) {
                case 'JOB_STARTED':
                    this.highlightActiveJob(update.jobId);
                    break;

                case 'JOB_COMPLETED':
                    this.markJobComplete(update.jobId);
                    this.updateDatasetTimestamp(update.outputDataset);
                    break;

                case 'JOB_FAILED':
                    this.markJobFailed(update.jobId, update.error);
                    break;

                case 'SCHEMA_CHANGED':
                    this.showSchemaChangeAlert(update.dataset, update.changes);
                    this.refreshLineage(update.dataset);
                    break;

                case 'NEW_DEPENDENCY':
                    this.addEdge(update.source, update.target);
                    break;
            }
        };

        this.ws.onerror = (error) => {
            console.error('WebSocket error:', error);
            this.showReconnectNotification();
        };
    }

    highlightActiveJob(jobId) {
        // 실행 중인 작업을 시각적으로 강조
        d3.select(`#job-${jobId}`)
            .transition()
            .duration(300)
            .attr('stroke', '#F59E0B')
            .attr('stroke-width', 4)
            .style('opacity', 1);

        // 펄스 애니메이션 추가
        this.addPulseAnimation(jobId);
    }

    addPulseAnimation(nodeId) {
        // CSS 애니메이션 또는 D3 transition 사용
        const node = d3.select(`#${nodeId}`);

        function pulse() {
            node.transition()
                .duration(1000)
                .attr('r', 15)
                .transition()
                .duration(1000)
                .attr('r', 10)
                .on('end', pulse);
        }

        pulse();
    }
}

11. 모범 사례

  1. 컨텍스트로 시작: 전체 그래프에서 사용자의 위치 표시

  8. 점진적 공개: 기본은 단순하게, 필요시 세부 정보 표시
  9. 성능 모니터링: 렌더링 시간 추적 및 최적화
