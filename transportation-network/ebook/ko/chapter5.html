챕터 5

2단계 - API 인터페이스

5.1 RESTful API 설계

WIA-UNI-008 API는 표준 HTTP 메서드를 사용한 RESTful 원칙을 따릅니다:

  • GET: 노선, 예약, 추적 데이터 검색
  • POST: 새 예약, 화물 배송, 차량 생성
  • PUT: 기존 리소스 업데이트
  • DELETE: 예약 취소, 차량 등록 취소

5.2 인증 및 보안

세분화된 액세스 제어를 위한 범위가 있는 OAuth 2.0 토큰 기반 인증:

  • routes:read / routes:write
  • bookings:read / bookings:write
  • tracking:read
  • cargo:read / cargo:write

5.3 주요 API 엔드포인트

노선 API

  • GET /v1/routes - 필터링을 사용한 노선 목록
  • GET /v1/routes/{id} - 노선 세부 정보 가져오기
  • POST /v1/routes - 새 노선 생성

예약 API

  • GET /v1/bookings/availability - 가용성 검색
  • POST /v1/bookings - 예약 생성
  • GET /v1/bookings/{id} - 예약 세부 정보 가져오기
  • DELETE /v1/bookings/{id} - 예약 취소

추적 API

  • GET /v1/tracking/vehicles/{id} - 실시간 차량 추적
  • GET /v1/tracking/bookings/{id} - 승객 여정 추적
  • WebSocket /v1/tracking/stream - 실시간 업데이트 구독

5.4 웹훅

예약 확인, 취소, 지연, 화물 상태 변경에 대한 이벤트 기반 알림. 웹훅은 보안을 위해 HMAC-SHA256 서명을 사용합니다.

5.5 속도 제한

공정 사용 정책: 시간당 1,000개 요청(표준), 시간당 10,000개 요청(프리미엄), 기업용 맞춤형.
