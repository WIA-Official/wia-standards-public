WIA-UNI-008 표준 개요

3.1 4단계 아키텍처

WIA-UNI-008은 4개의 뚜렷한 단계로 점진적 구현 모델을 따릅니다:

1단계: 데이터 형식

의미적 상호운용성을 위해 JSON-LD를 사용하여 공통 데이터 스키마를 설정하는 기초 계층. 노선, 일정, 예약, 화물 추적, 차량 등록을 다룹니다.

2단계: API 인터페이스

교통 시스템에 대한 프로그래밍 방식 액세스를 가능하게 하는 RESTful API 및 SDK. 예약 시스템, 실시간 추적, 물류 관리 및 웹훅 알림을 포함합니다.

3단계: 프로토콜

시스템 통합을 위한 통신 프로토콜 및 기술 표준. 철도 신호(ERTMS/ETCS), 교통 제어(ITS), 항공(ICAO), 해운(AIS), IoT 센서 네트워크를 다룹니다.

4단계: 통합

시베리아 횡단철도, 지역 협력 프레임워크, 세관 시스템, WIA 생태계 표준을 포함한 국제 네트워크에 연결하는 생태계 통합.

3.2 설계 원칙

  • 상호운용성: 서로 다른 시스템과 관할권 간의 원활한 운영
  • 확장성: 초기 국경 간 노선에서 완전한 대륙 연결까지 성장 지원
  • 보안: TLS 1.3 암호화, 인증 및 데이터 보호
  • 개방성: 모든 구현자가 액세스할 수 있는 개방형 표준
  • 호환성: 국제 표준(ISO, UIC, ICAO, IMO)과의 정렬

3.3 기술 스택

  • 데이터 형식: 연결 데이터 및 시맨틱 웹 호환성을 위한 JSON-LD
  • API: OAuth 2.0 인증을 사용한 RESTful 설계
  • 실시간: 실시간 추적 업데이트를 위한 WebSocket
  • 지리공간: WGS84 좌표 시스템
  • 프로토콜: IoT용 MQTT, CoAP; 철도용 ERTMS; 해운용 AIS
