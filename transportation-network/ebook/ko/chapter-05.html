05장: Phase 2 — API 인터페이스

이 장의 한국어 안내

이 장은 WIA-UNI-008 교통 네트워크 표준 v1.0.0의 한국어 학습용 자료입니다. 표준 본문은 4개 페이즈로 구성되며 영문판이 정본 (canonical) 입니다. 번역상 모호한 부분이 있으면 항상 영문판 (../en/chapter-05.html "Phase 2 — API Interface")을 우선합니다.

WIA-UNI-008이 다루는 핵심

WIA-UNI-008은 다중 모드 교통 네트워크 데이터 교환을 위한 개방형 표준입니다 — 버스·철도·페리·마이크로모빌리티·라이드 셰어·자율주행 셔틀. 4-페이즈 아키텍처: Phase 1(데이터 형식)은 노선·시간표·차량 위치·운행 업데이트·서비스 경보·요금 상품·공유 차량·라이더 여행 봉투를 정의합니다. Phase 2(API 인터페이스)는 발행·조회·스트림·페더레이션을 위한 HTTP 표면. Phase 3(페더레이션 프로토콜)은 교통 기관·운영자·라이더 앱 간 데이터 흐름 처리. Phase 4(통합)는 GTFS·GTFS-Realtime·NeTEx·SIRI·GBFS·MDS·WIA 패밀리와 연결.

한국 환경에서의 적용 노트

한국 대중교통은 서울교통공사·부산교통공사·코레일·각 지자체 버스 회사·따릉이·SOCAR·KAKAO T 등으로 분단되어 있습니다. 환승 정보는 한국 국토교통부의 ODsay·카카오맵·네이버 지도 등 통합 플랫폼이 각 사업자 데이터를 개별 통합하지만, 표준화된 와이어 형식이 없어 신규 모빌리티 진입 장벽이 높습니다. WIA-UNI-008은 한국 환경에 다음과 같이 적용됩니다: 한국 표준 GTFS-KR(국토교통부 발간)과 호환되는 Phase 4 GTFS 브리지, 따릉이/스마트모빌리티 데이터를 위한 GBFS 브리지, 자율주행 셔틀 시범 운영(KT·현대모비스·라이드플럭스 등) 데이터를 위한 vehicle_position 스트림.

핵심 객체 가족

  • route — 서비스 정의 (버스 노선·철도 노선·페리 항로·스쿠터 구역)
  • schedule — 서비스 일자별 계획 운행
  • vehicle_position — 실시간 GPS·점유율·방향
  • trip_update — 예측 도착·지연·취소
  • service_alert — 운행 중단 공지 (지연·우회·취소)
  • fare_product — 요금 구조 (단일 운행·일일 패스·월 정기권·거리 기반)
  • shared_vehicle — 마이크로모빌리티 가용성 (자전거·스쿠터·전기 모페드)
  • rider_trip — 동의 봉투가 있는 완료된 여행 기록 (익명화)

라이더 프라이버시 우선 설계

표준은 라이더 여행 데이터 프라이버시를 Phase 3 핵심 원리로 다룹니다. 실용적 결과: 교통 기관은 라이더의 명시적 서명 동의 없이 마케팅 파트너·라이드 셰어 플랫폼·연구 집계자와 라이더 여행 데이터를 공유할 수 없습니다. k-익명성 ≥ 5인 집계 통계는 공개되지만 여행별 세부는 여행별 동의가 필요합니다.

다중 모드 여행 계획 패턴

표준의 핵심 사용 사례는 다중 모드 여행 계획입니다 — 여행자가 A 지점에서 B 지점으로 이동하려 하고 플래너는 버스·철도·마이크로모빌리티·도보를 결합해야 합니다. 표준은 벤더별 통합 없이 이를 가능하게 합니다 — 여행자 앱은 도시 내 모든 참여 제공자의 피드를 집계하는 하나의 WIA-UNI-008 호스트를 구독합니다. 여행 검색은 총 시간·총 비용·탄소 발자국으로 순위 매겨진 다중 모드 옵션을 반환합니다. 여행자가 옵션을 선택하면 앱은 여행 기간만에 대한 명시적 동의로 rider_trip 봉투를 시작합니다.

구현자 노트

교통 네트워크 인프라에서 반복되는 세 가지 운영 관심사가 표준에 박혀 있습니다. 첫째, vehicle_position 갱신 주기: 대부분의 미국 교통 기관은 15-30초마다 발행합니다. 한국 광역버스는 1분, 도시철도는 30초가 일반적입니다. 둘째, 라이더 여행 동의 취소: 라이더가 동의를 취소할 때 데이터는 분 단위로 흐름이 멈춰야 합니다. 셋째, GTFS 피드 갱신: 기관은 일반적으로 주간 GTFS 갱신을 발행합니다. 표준의 schedule 봉투 버저닝은 다운스트림 컨슈머가 폴링 없이 새 피드를 감지할 수 있게 합니다.
