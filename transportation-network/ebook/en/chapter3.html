CHAPTER 3

WIA-UNI-008 Standard Overview

3.1 Four-Phase Architecture

WIA-UNI-008 follows a progressive implementation model with four distinct phases:

Phase 1: Data Format

Foundation layer establishing common data schemas using JSON-LD for semantic interoperability. Covers routes, schedules, bookings, cargo tracking, and vehicle registration.

Phase 2: API Interface

RESTful APIs and SDKs enabling programmatic access to transportation systems. Includes booking systems, real-time tracking, logistics management, and webhook notifications.

Phase 3: Protocol

Communication protocols and technical standards for system integration. Covers railway signaling (ERTMS/ETCS), traffic control (ITS), aviation (ICAO), maritime (AIS), and IoT sensor networks.

Phase 4: Integration

Ecosystem integration connecting to international networks including Trans-Siberian Railway, regional cooperation frameworks, customs systems, and WIA ecosystem standards.

3.2 Design Principles

  • Interoperability: Seamless operation across different systems and jurisdictions
  • Scalability: Support for growth from initial cross-border routes to full continental connectivity
  • Security: TLS 1.3 encryption, authentication, and data protection
  • Openness: Open standard accessible to all implementers
  • Compatibility: Alignment with international standards (ISO, UIC, ICAO, IMO)

3.3 Technology Stack

  • Data Format: JSON-LD for linked data and semantic web compatibility
  • APIs: RESTful design with OAuth 2.0 authentication
  • Real-time: WebSocket for live tracking updates
  • Geospatial: WGS84 coordinate system
  • Protocols: MQTT, CoAP for IoT; ERTMS for railway; AIS for maritime
