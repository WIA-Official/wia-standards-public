CHAPTER 4

Phase 1 - Data Format Specification

4.1 Transportation Route Data

The route data format captures all metadata for transportation corridors including railway lines, highways, air routes, and maritime lanes.

Key Fields

  • routeType: high-speed-rail, conventional-rail, highway, air-route, maritime
  • origin and destination: Station/terminal information with geospatial coordinates
  • waypoints: Intermediate stops with dwell times
  • schedule: Departure times, frequency, validity periods
  • specifications: Technical details (speed, capacity, gauge, electrification)

4.2 Booking and Ticketing Data

Standardized format for passenger and cargo bookings enabling cross-operator ticketing:

  • Passenger information (name, passport, nationality, contact)
  • Journey details (route, date, time, class, seat)
  • Pricing breakdown (base price, taxes, fees, total)
  • Payment information and transaction tracking
  • Verifiable Credential integration for digital tickets

4.3 Cargo Tracking Format

Multi-modal freight tracking from origin to destination:

  • Shipment Details: Tracking number, status, shipper, consignee
  • Cargo Information: Type, weight, volume, container number, HS code
  • Route Segments: Mode, from/to locations, vehicle ID, estimated times
  • Customs: Declaration number, clearance status, estimated processing
  • Conditions: Temperature, humidity monitoring for sensitive cargo

4.4 Vehicle Registration

Standardized registration format for trains, buses, trucks, aircraft, and vessels with specifications, certifications, maintenance records, and IoT device integration.

Validation: All data must conform to JSON-LD schema and resolve against WIA-UNI-008 context at https://wiastandards.com/contexts/uni-008/v1
