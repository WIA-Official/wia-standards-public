Acoustic Communication Systems

Sound as the workhorse of underwater communication: exploring transducers, modulation schemes, and signal processing techniques that overcome the hostile acoustic channel.

Fundamentals of Underwater Acoustics

Acoustic communication dominates underwater networks because sound propagates efficiently through water where electromagnetic waves cannot. The speed of sound in seawater—approximately 1,500 m/s—varies with temperature, salinity, and pressure, creating complex propagation paths that both challenge and enable long-range communication.

Unlike radio waves that travel at the speed of light, acoustic signals are slow. This creates significant latency: a 10 km acoustic link introduces a 6.7 second propagation delay, fundamentally changing network protocol design. The WIA-OCEAN-003 standard addresses these unique characteristics through specialized protocols and modulation techniques.

Sound Speed and Propagation

The speed of sound in seawater depends on three factors: temperature (T), salinity (S), and depth (D). The Mackenzie equation provides an accurate empirical formula:

c = 1448.96 + 4.591T - 0.05304T² + 0.0002374T³ + 0.0160D + (1.340 - 0.0010T)(S - 35) + 1.675×10⁻⁷D²

where c is in m/s, T in °C, S in PSU, and D in meters. Typical open ocean values yield c ≈ 1,500 m/s, but variations create sound channels and shadow zones that dramatically affect communication range.

Sound Speed Profile and Multipath

Temperature decreases with depth in the upper ocean, reducing sound speed, then remains nearly constant in deep water. This creates a sound channel—the SOFAR channel—where acoustic energy can propagate for thousands of kilometers. However, this channel also creates severe multipath propagation, where signals arrive via multiple paths with different delays, causing inter-symbol interference (ISI).

Transducer Parameters

Key specifications defined by WIA-OCEAN-003 include:

  • Transmitting Voltage Response (TVR): dB re 1 μPa/V @ 1m—efficiency of electrical to acoustic conversion
  • Receiving Sensitivity: dB re 1 V/μPa—acoustic to electrical conversion efficiency
  • Beam Pattern: Directivity in azimuth and elevation angles
  • Source Level (SL): Maximum acoustic output in dB re 1 μPa @ 1m
  • Bandwidth: -3 dB frequency range for transmission/reception

Modulation Schemes

Modulation encodes digital data onto acoustic carrier waves. The harsh underwater channel—with multipath, Doppler, and frequency-selective fading—requires robust modulation schemes.

Frequency Shift Keying (FSK)

FSK encodes bits by shifting between discrete frequencies. Simple and robust, FSK works well in noisy environments but achieves low spectral efficiency (typically 0.5-1 bit/s/Hz). Binary FSK (BFSK) uses two frequencies; M-ary FSK increases data rate by using multiple frequencies.

Phase Shift Keying (PSK)

PSK modulates carrier phase to encode data. BPSK uses two phases (0°, 180°) for 1 bit per symbol; QPSK uses four phases (0°, 90°, 180°, 270°) for 2 bits per symbol. PSK achieves better spectral efficiency than FSK but requires accurate phase tracking, challenging with Doppler spread.

Orthogonal Frequency-Division Multiplexing (OFDM)

OFDM divides the available bandwidth into many narrow subcarriers, each modulated with PSK or QAM. By making symbol duration long relative to multipath delay spread, OFDM converts the frequency-selective fading channel into multiple flat-fading subchannels. Guard intervals (cyclic prefixes) prevent ISI. OFDM has become the dominant technique for high-rate acoustic communication, achieving 10-50 kbps in practice.

TypeScript Implementation: OFDM Acoustic Modem

interface OFDMParams {
  numSubcarriers: number;    // Total OFDM subcarriers
  cpLength: number;          // Cyclic prefix samples
  symbolDuration: number;    // Symbol time (ms)
  centerFrequency: number;   // Carrier frequency (Hz)
  bandwidth: number;         // Total bandwidth (Hz)
  modulation: 'BPSK' | 'QPSK' | 'QAM16';
}

class AcousticOFDMModem {
  private params: OFDMParams;
  private sampleRate: number;

  constructor(params: OFDMParams) {
    this.params = params;
    this.sampleRate = params.bandwidth * 2; // Nyquist
  }

  modulateOFDM(dataBits: number[]): Float32Array {
    const bitsPerSymbol = this.getBitsPerSymbol();
    const dataSymbols = this.mapToConstellation(dataBits, bitsPerSymbol);

    // Allocate symbols to subcarriers (skip DC and Nyquist)
    const ofdmSymbol = new Array(this.params.numSubcarriers).fill(0);
    for (let i = 0; i < dataSymbols.length; i++) {
      ofdmSymbol[i + 1] = dataSymbols[i]; // Start at subcarrier 1
    }

    // IFFT to convert to time domain
    const timeDomain = this.ifft(ofdmSymbol);

    // Add cyclic prefix
    const withCP = this.addCyclicPrefix(timeDomain);

    // Upconvert to carrier frequency
    return this.upconvert(withCP, this.params.centerFrequency);
  }

  demodulateOFDM(receivedSignal: Float32Array): number[] {
    // Downconvert from carrier
    const baseband = this.downconvert(receivedSignal, this.params.centerFrequency);

    // Remove cyclic prefix
    const ofdmSymbol = this.removeCyclicPrefix(baseband);

    // FFT to frequency domain
    const frequencyDomain = this.fft(ofdmSymbol);

    // Channel equalization
    const equalized = this.equalize(frequencyDomain);

    // Demap constellation to bits
    return this.demapFromConstellation(equalized);
  }

  private getBitsPerSymbol(): number {
    switch (this.params.modulation) {
      case 'BPSK': return 1;
      case 'QPSK': return 2;
      case 'QAM16': return 4;
    }
  }

  private mapToConstellation(bits: number[], bitsPerSymbol: number): Complex[] {
    const symbols: Complex[] = [];
    for (let i = 0; i < bits.length; i += bitsPerSymbol) {
      const symbolBits = bits.slice(i, i + bitsPerSymbol);
      symbols.push(this.mapSymbol(symbolBits));
    }
    return symbols;
  }

  private mapSymbol(bits: number[]): Complex {
    // QPSK constellation mapping example
    if (this.params.modulation === 'QPSK') {
      const value = bits[0] * 2 + bits[1];
      const phase = (value * Math.PI / 2) + Math.PI / 4;
      return { real: Math.cos(phase), imag: Math.sin(phase) };
    }
    // ... other modulations
    return { real: 1, imag: 0 };
  }

  private equalize(received: Complex[]): Complex[] {
    // Simple zero-forcing equalization
    // In practice, use channel estimation from pilot symbols
    return received.map(symbol => ({
      real: symbol.real / this.estimateChannelGain(),
      imag: symbol.imag / this.estimateChannelGain()
    }));
  }

    return 0.8;

    const output = new Float32Array(N);

    return output;
  }

    return [];

  private addCyclicPrefix(signal: Float32Array): Float32Array {
    const cpLength = this.params.cpLength;
    const result = new Float32Array(signal.length + cpLength);
    // Copy last cpLength samples to beginning
    result.set(signal.slice(-cpLength), 0);
    result.set(signal, cpLength);
    return result;
  }

  private removeCyclicPrefix(signal: Float32Array): Float32Array {
    return signal.slice(this.params.cpLength);
  }

  private upconvert(baseband: Float32Array, fc: number): Float32Array {
    const result = new Float32Array(baseband.length);
    for (let i = 0; i < baseband.length; i++) {
      const t = i / this.sampleRate;
      result[i] = baseband[i] * Math.cos(2 * Math.PI * fc * t);
    }
    return result;
  }

  private downconvert(signal: Float32Array, fc: number): Float32Array {
    const result = new Float32Array(signal.length);
    for (let i = 0; i < signal.length; i++) {
      const t = i / this.sampleRate;
      result[i] = signal[i] * Math.cos(2 * Math.PI * fc * t);
    }
    return result;
  }

  private demapFromConstellation(symbols: Complex[]): number[] {
    const bits: number[] = [];
    const bitsPerSymbol = this.getBitsPerSymbol();
    for (const symbol of symbols) {
      const symbolBits = this.demapSymbol(symbol);
      bits.push(...symbolBits);
    }
    return bits;
  }

  private demapSymbol(symbol: Complex): number[] {
    // QPSK demapping example
    if (this.params.modulation === 'QPSK') {
      const phase = Math.atan2(symbol.imag, symbol.real);
      const index = Math.round((phase - Math.PI/4) / (Math.PI/2)) % 4;
      return [(index >> 1) & 1, index & 1];
    }
    return [0];
  }
}

Equalization and Doppler Compensation

Multipath propagation creates ISI that must be removed through equalization. Adaptive equalizers—particularly decision-feedback equalizers (DFE) and turbo equalizers—track the time-varying channel and remove ISI.

Motion between transmitter and receiver creates Doppler shift: f_received = f_transmitted × (1 + v/c), where v is the relative velocity. At 1 m/s relative motion and 25 kHz carrier, Doppler shift is 17 Hz. OFDM subcarriers spaced at 100 Hz can handle this, but fast-moving AUVs require Doppler compensation through resampling or frequency tracking.

Marine Mammal Protection Requirements

WIA-OCEAN-003 mandates maximum source levels to protect marine mammals:
• Below 10 kHz: 185 dB re 1 μPa @ 1m (overlaps whale hearing)
• 10-100 kHz: 195 dB re 1 μPa @ 1m (dolphin/porpoise range)
• Above 100 kHz: 205 dB re 1 μPa @ 1m (less biological sensitivity)
Ramping protocols and marine mammal observers required for high-power deployments.

KIOST Acoustic Modem Development

KIOST has developed the K-MAC (Korea Maritime Acoustic Communication) series of modems optimized for the East Sea and Yellow Sea environments. These modems feature adaptive OFDM with real-time Doppler compensation, achieving 15 kbps at 5 km range in challenging shallow-water conditions.

The K-MAC modem incorporates machine learning for channel prediction, training neural networks on historical channel measurements to predict future channel states and pre-compensate for fading. This approach has demonstrated 3-5 dB SNR improvement over traditional methods.
