Chapter 6

Data Encoding and Error Correction

Fighting the noise: advanced coding techniques that detect and correct errors from multipath, fading, and interference to ensure reliable data delivery through the hostile underwater channel.

The Need for Error Correction

Underwater channels exhibit bit error rates (BER) of 10⁻³ to 10⁻² without coding—1000 to 10,000 times worse than fiber optics. Multipath creates inter-symbol interference, ambient noise corrupts weak signals, and Doppler spread introduces frequency errors. Forward error correction (FEC) codes add redundancy, enabling receivers to detect and correct errors without retransmission.

WIA-OCEAN-003 mandates minimum coding gains and specifies approved FEC schemes. Target BER after decoding: <10⁻⁶ for data, <10⁻⁹ for critical command/control.

Block Codes

Block codes encode k data bits into n code bits (n > k), adding n-k redundancy bits. The code rate R = k/n indicates efficiency (R=0.5 means 50% overhead).

Hamming Codes

Classic block codes that correct single-bit errors using parity checks. Hamming(7,4) encodes 4 data bits into 7 coded bits, achieving single error correction. Simple to implement but limited error correction capability—unsuitable for harsh underwater channels.

Reed-Solomon (RS) Codes

RS codes operate on symbols (groups of bits) rather than individual bits, making them highly effective against burst errors. RS(255,223) encodes 223 bytes into 255 bytes, correcting up to 16 byte errors. Widely used in underwater acoustic modems due to excellent burst error performance.

Convolutional Codes

Convolutional codes encode data continuously using shift registers and modulo-2 adders. The Viterbi algorithm performs maximum likelihood decoding, finding the most probable transmitted sequence. Constraint length K determines memory and complexity—typical underwater systems use K=7 to K=9.

Punctured Convolutional Codes

Puncturing periodically deletes coded bits to increase code rate while retaining most coding gain. Rate-1/2 code punctured to rate-2/3 or 3/4 provides flexible trade-off between throughput and robustness. WIA-OCEAN-003 specifies puncture patterns for common rates.

Turbo Codes and LDPC

Modern iterative codes approach the Shannon limit—the theoretical maximum channel capacity. Turbo codes and low-density parity-check (LDPC) codes achieve <1 dB from Shannon limit with reasonable complexity.

Turbo Codes

Two parallel convolutional encoders with an interleaver between them create a powerful code. Iterative decoding exchanges probability information between decoders, refining estimates with each iteration. Typical systems use 4-8 iterations for BER <10⁻⁶. Excellent for low SNR but high latency due to iteration.

LDPC Codes

LDPC codes use sparse parity-check matrices, enabling efficient decoding via belief propagation. Parallelizable decoder suits FPGA/ASIC implementation. LDPC outperforms Turbo codes slightly and offers lower decoding latency—ideal for real-time acoustic OFDM systems.

TypeScript Implementation: Reed-Solomon Encoder

// Simplified RS(7,3) encoder over GF(8) for demonstration
class ReedSolomonEncoder {
  private n: number = 7;  // Codeword length
  private k: number = 3;  // Data symbols
  private t: number = 2;  // Error correction capability = (n-k)/2

  // Galois Field GF(8) primitive polynomial: x^3 + x + 1
  private gfExp: number[] = [1, 2, 4, 3, 6, 7, 5, 1];  // Powers of alpha
  private gfLog: number[] = [0, 0, 1, 3, 2, 6, 4, 5];  // Logarithm table

  encode(dataSymbols: number[]): number[] {
    if (dataSymbols.length !== this.k) {
      throw new Error(\`Expected \${this.k} data symbols\`);
    }

    // Generator polynomial for 2-error correction
    const generator = this.getGenerator();

    // Multiply data by x^(n-k) (shift left by n-k positions)
    const message = [...dataSymbols, 0, 0, 0, 0];

    // Divide by generator polynomial, get remainder
    const remainder = this.polyDivide(message, generator);

    // Codeword = data || remainder
    return [...dataSymbols, ...remainder.slice(0, this.n - this.k)];
  }

  private getGenerator(): number[] {
    // Generator g(x) = (x - alpha^1)(x - alpha^2)(x - alpha^3)(x - alpha^4)
    // For RS(7,3), we need 4 roots
    let g = [1];  // Start with 1

    for (let i = 1; i <= 2 * this.t; i++) {
      // Multiply by (x - alpha^i)
      g = this.polyMultiply(g, [1, this.gfExp[i % 7]]);
    }

    return g;
  }

  private polyMultiply(p1: number[], p2: number[]): number[] {
    const result = new Array(p1.length + p2.length - 1).fill(0);

    for (let i = 0; i < p1.length; i++) {
      for (let j = 0; j < p2.length; j++) {
        result[i + j] ^= this.gfMultiply(p1[i], p2[j]);
      }
    }

    return result;
  }

  private polyDivide(dividend: number[], divisor: number[]): number[] {
    const result = [...dividend];

    for (let i = 0; i < dividend.length - divisor.length + 1; i++) {
      if (result[i] !== 0) {
        const coef = result[i];
        for (let j = 0; j < divisor.length; j++) {
          result[i + j] ^= this.gfMultiply(coef, divisor[j]);
        }
      }
    }

    return result.slice(-(this.n - this.k));
  }

  private gfMultiply(a: number, b: number): number {
    if (a === 0 || b === 0) return 0;
    return this.gfExp[(this.gfLog[a] + this.gfLog[b]) % 7];
  }

  // Error correction (simplified syndrome decoding)
  decode(received: number[]): { corrected: number[], errors: number } {
    // Calculate syndromes
    const syndromes = this.calculateSyndromes(received);

    // If all syndromes zero, no errors
    if (syndromes.every(s => s === 0)) {
      return { corrected: received.slice(0, this.k), errors: 0 };
    }

    // Error locator polynomial (Berlekamp-Massey algorithm)
    // Simplified: assume single error for demonstration
    const errorLocation = this.findErrorLocation(syndromes);
    const errorValue = syndromes[0];

    const corrected = [...received];
    if (errorLocation >= 0) {
      corrected[errorLocation] ^= errorValue;
    }

    return {
      corrected: corrected.slice(0, this.k),
      errors: errorLocation >= 0 ? 1 : -1
    };
  }

  private calculateSyndromes(received: number[]): number[] {
    const syndromes: number[] = [];

    for (let i = 1; i <= 2 * this.t; i++) {
      let syndrome = 0;
      for (let j = 0; j < received.length; j++) {
        syndrome ^= this.gfMultiply(
          received[j],
          this.gfExp[(i * j) % 7]
        );
      }
      syndromes.push(syndrome);
    }

    return syndromes;
  }

  private findErrorLocation(syndromes: number[]): number {
    // Simplified: search for single error location
    for (let i = 0; i < this.n; i++) {
      let match = true;
      for (let j = 0; j < syndromes.length; j++) {
        const expected = this.gfMultiply(
          syndromes[0],
          this.gfExp[(j * i) % 7]
        );
        if (syndromes[j] !== expected) {
          match = false;
          break;
        }
      }
      if (match) return i;
    }
    return -1; // Uncorrectable
  }
}

Interleaving

Multipath fading creates burst errors—consecutive symbols corrupted by a deep fade. Interleaving spreads codeword symbols across time/frequency, converting burst errors into isolated errors that FEC codes handle better. Block interleavers rearrange symbols in a matrix; convolutional interleavers use shift registers.

Depth-Time Interleaving

For OFDM systems, interleaving across both OFDM symbols (time) and subcarriers (frequency) provides diversity against time-selective and frequency-selective fading. WIA-OCEAN-003 specifies minimum interleaver depths based on channel coherence time and bandwidth.

Automatic Repeat Request (ARQ)

ARQ combines error detection (CRC, checksum) with retransmission. Three basic schemes:

  • Stop-and-Wait: Send one packet, wait for ACK. Simple but inefficient with long delays.
  • Go-Back-N: Send N packets without waiting, retransmit from error point. Better throughput.
  • Selective Repeat: Retransmit only erroneous packets. Best throughput but complex buffering.

Hybrid ARQ

Combines FEC with ARQ. Type-I HARQ: FEC detects/corrects; if fail, ARQ requests retransmission. Type-II/III HARQ: Initial transmission uses low-redundancy code; retransmissions send additional parity for incremental redundancy.

Summary: Key Takeaways

  • High Error Rates: Underwater BER 10⁻³–10⁻² requires strong FEC for reliable communication
  • Block Codes: RS codes excel at burst errors from fading; simple implementation
  • Convolutional Codes: Viterbi decoding provides good performance/complexity trade-off
  • Advanced Codes: Turbo and LDPC approach Shannon limit; essential for low SNR
  • Interleaving: Converts burst errors to random errors for better FEC performance
  • Hybrid ARQ: Combines FEC and retransmission for guaranteed reliability
  • WIA-OCEAN-003: Specifies codes and target BER for different applications
