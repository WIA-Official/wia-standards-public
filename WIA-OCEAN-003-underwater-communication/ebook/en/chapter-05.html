Chapter 5

Network Protocols for Subsea

Building underwater networks that overcome seconds of latency, collision domains measured in kilometers, and time-varying channels that challenge traditional protocol assumptions.

Underwater Network Challenges

Traditional network protocols—designed for terrestrial RF or fiber—fail underwater. TCP assumes millisecond latencies; acoustic links incur seconds. Ethernet CSMA/CD breaks when propagation delay exceeds packet duration. Even basic assumptions like bidirectional links and stable topologies no longer hold.

WIA-OCEAN-003 defines protocol stacks adapted to the unique underwater environment: delay-tolerant routing, specialized MAC protocols, and cross-layer optimization that shares channel state information between physical, MAC, and network layers.

MAC Layer Protocols

Medium access control determines when nodes can transmit. The long propagation delay underwater—up to 6.7 seconds for a 10 km link—creates enormous collision domains where traditional CSMA/CD fails.

TDMA: Time Division Multiple Access

TDMA assigns each node specific time slots for transmission. Requires precise time synchronization but eliminates collisions. Guard intervals between slots must accommodate propagation delay plus clock drift. For a 5 km network, guard intervals exceed 3 seconds—significantly reducing efficiency.

CDMA: Code Division Multiple Access

CDMA assigns each node a unique spreading code, allowing simultaneous transmissions. The receiver correlates received signals with expected codes to extract individual messages. CDMA tolerates multipath and near-far problems but requires accurate power control.

Aloha and CSMA Variants

Pure Aloha: transmit whenever data is ready, retransmit on collision. Simple but inefficient (18% maximum throughput). Slotted Aloha improves to 37% by synchronizing transmission starts. Carrier Sense Multiple Access (CSMA) listens before transmitting but long propagation delays reduce effectiveness.

Routing Protocols

Underwater sensor networks require routing protocols that handle mobility (AUVs, drifting buoys), variable topology, and limited energy budgets.

Vector-Based Forwarding (VBF)

VBF routes packets along a "virtual pipe" from source to sink. Nodes within the pipe radius forward packets; others discard. Robust to node mobility and topology changes without explicit route discovery.

Depth-Based Routing (DBR)

Exploits depth stratification in underwater networks. Each node knows its depth; packets forward only to shallower neighbors (toward surface sink). Simple, energy-efficient, and naturally load-balanced.

Delay-Tolerant Networking (DTN)

DTN stores messages at intermediate nodes until connectivity becomes available. Essential for sparse networks with intermittent connectivity. Bundle Protocol provides store-and-forward with custody transfer, ensuring reliable delivery despite disconnections.

TypeScript Implementation: DBR Routing

interface UnderwaterNode {
  id: string;
  depth: number;           // meters below surface
  position: { x: number; y: number; z: number };
  neighbors: UnderwaterNode[];
  energyRemaining: number; // Joules
}

interface Packet {
  source: string;
  destination: string;
  payload: Uint8Array;
  hopCount: number;
  timestamp: number;
}

class DepthBasedRouter {
  private node: UnderwaterNode;
  private maxHops: number = 10;

  constructor(node: UnderwaterNode) {
    this.node = node;
  }

  routePacket(packet: Packet): UnderwaterNode | null {
    // Depth-based routing: forward to shallowest neighbor
    if (packet.hopCount >= this.maxHops) {
      console.log('Packet exceeded max hops, dropping');
      return null;
    }

    // Find neighbors shallower than us
    const shallowerNeighbors = this.node.neighbors.filter(
      n => n.depth < this.node.depth
    );

    if (shallowerNeighbors.length === 0) {
      // No shallower neighbor - we're at local minimum, use greedy
      return this.greedyForward(packet);
    }

    // Select shallowest neighbor (closest to surface)
    const nextHop = shallowerNeighbors.reduce((shallowest, current) =>
      current.depth < shallowest.depth ? current : shallowest
    );

    // Energy-aware: skip if neighbor energy too low
    if (nextHop.energyRemaining < 100) {
      console.log('Next hop low energy, finding alternative');
      const alternatives = shallowerNeighbors.filter(
        n => n.id !== nextHop.id && n.energyRemaining >= 100
      );
      return alternatives.length > 0 ? alternatives[0] : null;
    }

    return nextHop;
  }

  private greedyForward(packet: Packet): UnderwaterNode | null {
    // Fallback: forward to neighbor with highest energy
    if (this.node.neighbors.length === 0) return null;

    return this.node.neighbors.reduce((best, current) =>
      current.energyRemaining > best.energyRemaining ? current : best
    );
  }

  forwardPacket(packet: Packet): void {
    const nextHop = this.routePacket(packet);

    if (!nextHop) {
      console.log(\`Node \${this.node.id}: No route to destination, buffering\`);
      this.bufferPacket(packet);
      return;
    }

    packet.hopCount++;
    console.log(\`Node \${this.node.id}: Forwarding to \${nextHop.id} at depth \${nextHop.depth}m\`);

    // Simulate transmission
    this.transmit(packet, nextHop);
  }

  private bufferPacket(packet: Packet): void {
    // DTN-style buffering for later forwarding
  }

  private transmit(packet: Packet, nextHop: UnderwaterNode): void {
    // Actual transmission via acoustic/optical modem
    const transmitEnergy = this.calculateTransmitEnergy(nextHop);
    this.node.energyRemaining -= transmitEnergy;
  }

  private calculateTransmitEnergy(nextHop: UnderwaterNode): number {
    const distance = this.distance(this.node.position, nextHop.position);
    // Energy ~ distance^2 for acoustic
    return 0.1 * distance * distance; // Joules
  }

  private distance(p1: any, p2: any): number {
    const dx = p1.x - p2.x;
    const dy = p1.y - p2.y;
    const dz = p1.z - p2.z;
    return Math.sqrt(dx*dx + dy*dy + dz*dz);
  }
}

Transport Layer Adaptations

TCP's additive-increase/multiplicative-decrease congestion control interprets long acoustic delays as congestion, throttling throughput unnecessarily. Underwater-adapted transport protocols modify TCP or replace it entirely.

MUTP: Multi-path Underwater Transport Protocol

MUTP exploits multiple paths to improve reliability without retransmission delays. Splits data across diverse routes and reassembles at destination. Tolerates link failures and reduces latency compared to single-path retransmission.

RTTCP: Reliable Time-based Transport

RTTCP acknowledges packets based on time windows rather than sequence numbers. Accounts for variable propagation delays and avoids premature retransmissions. Improves throughput by 2-5× over vanilla TCP in deep-water scenarios.

WIA-OCEAN-003 Protocol Standards

The standard defines a layered protocol stack with cross-layer optimization:

  • Physical Layer: Acoustic OFDM, optical OOK, VLF EM as per Chapters 2-4
  • MAC Layer: Slotted Aloha (low traffic), TDMA (high traffic), CDMA (many nodes)
  • Network Layer: DBR (sensor networks), VBF (mobile nodes), DTN (sparse topology)
  • Transport Layer: MUTP (multi-path capable), RTTCP (delay-tolerant)
  • Application Layer: Sensor data aggregation, command/control, file transfer

KIOST Network Research

KIOST developed UW-HARMONI (Underwater Hybrid Adaptive Routing for Mobile Oceanographic Networks Initiative), combining DBR with machine learning to predict AUV mobility patterns and pre-position routes. Field trials in the East Sea demonstrated 40% reduction in retransmissions and 25% energy savings.

Summary: Key Takeaways

  • Traditional Protocols Fail: Terrestrial assumptions (low latency, stable links) don't hold underwater
  • MAC Challenges: Long propagation creates massive collision domains; TDMA/CDMA outperform CSMA
  • Geographic Routing: VBF and DBR exploit physical position, avoid route discovery overhead
  • DTN Essential: Store-and-forward handles sparse, mobile topologies
  • Transport Adaptations: Modified TCP or specialized protocols (MUTP, RTTCP) handle delay and loss
  • Cross-Layer Design: Sharing state between layers improves efficiency
  • Korean Innovation: KIOST UW-HARMONI uses ML for predictive routing
