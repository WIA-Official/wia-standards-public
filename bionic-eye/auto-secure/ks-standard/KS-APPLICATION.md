## 2. 표준 제안 배경

### 2.1 현황 및 문제점

#### 2.1.1 HTTPS 보급률 현황
| 구분 | 비율 | 출처 |
|------|------|------|
| 전 세계 웹사이트 HTTPS 적용률 | 약 85% | W3Techs (2025) |
| 한국 중소기업 웹사이트 HTTPS | 약 60% | 추정치 |
| 한국 소상공인 웹사이트 HTTPS | 약 40% | 추정치 |

#### 2.1.2 문제점
1. **기술적 장벽**: SSL/TLS 인증서 설치에 전문 지식 필요
2. **비용 부담**: 유료 인증서 연간 10만원~100만원
3. **유지보수 어려움**: 90일~1년마다 수동 갱신 필요
4. **표준 부재**: 자동화 도구의 표준화된 인터페이스 없음

#### 2.1.3 파급 효과
- 암호화되지 않은 웹사이트는 개인정보 유출 위험
- 구글 검색 순위 불이익 (2014년부터 HTTPS 가산점)
- 최신 브라우저에서 "안전하지 않음" 경고 표시
- 소상공인/중소기업의 디지털 신뢰도 저하

### 2.2 표준 제안 필요성

1. **접근성 향상**: 비전문가도 쉽게 HTTPS 적용 가능
2. **비용 절감**: 무료 인증서 + 자동화로 유지비용 0원
3. **보안 강화**: 전체 웹 생태계 암호화 촉진
4. **산업 표준화**: 도구 간 상호운용성 확보

## 3. 표준 내용

### 3.1 적용 범위

본 표준은 다음에 적용된다:
- 웹 서버 보안 인증서(SSL/TLS) 자동 설치 도구
- 인증서 자동 갱신 시스템
- 웹 서버 보안 설정 자동화 도구

### 3.2 용어 정의

| 용어 | 정의 |
|------|------|
| SSL/TLS | 전송 계층 보안 프로토콜 |
| HTTPS | HTTP over TLS, 암호화된 웹 통신 |
| 인증서 | 웹사이트 신원을 증명하는 디지털 문서 |
| ACME | Automatic Certificate Management Environment, 인증서 자동화 프로토콜 |
| CA | Certificate Authority, 인증서 발급 기관 |

### 3.3 기능 요구사항

#### 3.3.1 필수 요구사항 (Mandatory)

| ID | 요구사항 | 설명 |
|----|---------|------|
| M-01 | 운영체제 자동 감지 | Ubuntu, Debian, CentOS, RHEL, Amazon Linux 등 |
| M-02 | 웹 서버 자동 감지 | nginx, Apache 자동 감지 |
| M-03 | 인증서 자동 발급 | ACME 프로토콜 지원 CA 연동 |
| M-04 | 웹 서버 자동 설정 | HTTPS 리다이렉트, TLS 버전 설정 |
| M-05 | 자동 갱신 설정 | cron 또는 systemd timer 등록 |
| M-06 | 비대화형 모드 | 스크립트 자동 실행 지원 |

#### 3.3.2 선택 요구사항 (Optional)

| ID | 요구사항 | 설명 |
|----|---------|------|
| O-01 | 와일드카드 인증서 | *.example.com 지원 |
| O-02 | 다중 도메인 | SAN 인증서 지원 |
| O-03 | DNS API 연동 | Cloudflare, Route53 등 |
| O-04 | 롤백 기능 | 설치 실패 시 원복 |
| O-05 | 로그 기록 | 설치 과정 로깅 |

### 3.4 보안 요구사항

| ID | 요구사항 | 설명 |
|----|---------|------|
| S-01 | TLS 1.2 이상 | TLS 1.0, 1.1 비활성화 |
| S-02 | 강력한 암호 스위트 | AEAD 암호만 허용 |
| S-03 | HSTS 설정 | HTTP Strict Transport Security |
| S-04 | 개인키 보호 | 600 퍼미션 설정 |
| S-05 | 입력 검증 | 도메인, 이메일 형식 검증 |

### 3.5 인터페이스 규격

#### 3.5.1 CLI 인터페이스

```
사용법: wia-secure-install [옵션]

필수 옵션:
  --domain DOMAIN     대상 도메인
  --email EMAIL       연락처 이메일

선택 옵션:
  --wildcard          와일드카드 인증서 발급
  --no-www            www 서브도메인 제외
  --webroot PATH      웹루트 경로 지정
  --dry-run           실제 실행 없이 테스트
  --help              도움말 표시
  --version           버전 정보 표시
```

#### 3.5.2 종료 코드

| 코드 | 의미 |
|------|------|
| 0 | 성공 |
| 1 | 일반 오류 |
| 2 | 입력 검증 오류 |
| 3 | 권한 오류 |
| 4 | 네트워크 오류 |
| 5 | 인증서 발급 오류 |
| 6 | 웹 서버 설정 오류 |

### 3.6 적합성 시험

#### 3.6.1 시험 항목

| 시험 ID | 시험 항목 | 판정 기준 |
|---------|---------|----------|
| T-01 | 인증서 발급 | 유효한 인증서 발급됨 |
| T-02 | HTTPS 접속 | https://도메인 접속 성공 |
| T-03 | HTTP 리다이렉트 | http→https 자동 전환 |
| T-04 | 자동 갱신 | 갱신 테스트 통과 |
| T-05 | TLS 버전 | TLS 1.2 이상만 허용 |
| T-06 | 보안 등급 | SSL Labs A 등급 이상 |

## 4. 기대 효과

### 4.1 경제적 효과

| 항목 | 현재 | 표준 적용 후 | 절감액 |
|------|------|-------------|--------|
| 인증서 비용 | 10만원/년 | 0원 | 10만원/년 |

| 갱신 인건비 | 5만원/회 | 0원 | 5만원/년 |

3. **디지털 신뢰**: 국내 웹 생태계 전반의 보안 수준 향상
4. **국제 경쟁력**: 글로벌 보안 표준 선도
