Chapter 3: API Endpoints

Device Control, Image Streaming, and Calibration

3.1 API Overview

The WIA Bionic Eye API provides RESTful endpoints for controlling bionic eye devices, streaming visual data, and managing system configuration. The API uses JSON for data exchange and supports both HTTP and WebSocket connections.

Base URL

https://api.bionic-eye.wia.org/v1

Authentication

All API requests require authentication using Bearer tokens:

Authorization: Bearer {access_token}
Security: API tokens must be kept secure. Use HTTPS for all communications and implement token rotation every 24 hours.

3.4 Image Streaming

WebSocket: /stream/video

Real-time video streaming from camera to client using WebSocket connection.

Connection

ws://api.bionic-eye.wia.org/v1/stream/video?deviceId=BIO-EYE-001&token={access_token}

Message Format

{
  "type": "frame",
  "frameId": "uuid",
  "timestamp": 1703001234567,
  "sequenceNumber": 12345,
  "encoding": "jpeg",
  "data": "base64_encoded_image"
}

Control Messages

// Client to Server
{
  "type": "control",
  "action": "pause" | "resume" | "adjust_quality",
  "parameters": {
    "quality": 80
  }
}

// Server to Client
{
  "type": "status",
  "state": "streaming",
  "fps": 18.5,
  "droppedFrames": 2
}

GET /stream/status

Get current streaming status.

Response

{
  "isStreaming": true,
  "clients": 2,
  "currentFps": 19.8,
  "bandwidth": "15.2 Mbps",
  "latency": 45,
  "droppedFrames": 5
}

3.5 Calibration

POST /calibration/start

Start a calibration session for electrode mapping.

Request Body

{
  "deviceId": "BIO-EYE-001",
  "type": "phosphene_mapping" | "threshold_detection" | "impedance_check",
  "electrodes": [0, 1, 2, 3, 4, 5],  // null for all electrodes
  "parameters": {
    "startCurrent": 50,    // μA
    "maxCurrent": 800,
    "stepSize": 25
  }
}

Response

{
  "sessionId": "cal-20251220-001",
  "status": "started",
  "estimatedDuration": 1800,  // seconds
  "nextElectrode": 0
}

POST /calibration/record

Record patient response during calibration.

Request Body

{
  "sessionId": "cal-20251220-001",
  "electrodeIndex": 0,
  "currentLevel": 150,  // μA
  "response": {
    "perceived": true,
    "brightness": 5,    // 0-10 scale
    "location": {
      "x": 0.3,         // normalized -1 to 1
      "y": -0.2
    },
    "shape": "spot",    // spot, elongated, irregular
    "size": "small"     // small, medium, large
  }
}

Response

{
  "status": "recorded",
  "nextElectrode": 1,
  "progress": 1.67  // percentage
}

GET /calibration/results/{sessionId}

Get calibration results.

Response

{
  "sessionId": "cal-20251220-001",
  "completedAt": "2025-12-20T12:00:00Z",
  "electrodeData": [
    {
      "index": 0,
      "threshold": 125,        // μA
      "optimalCurrent": 200,
      "maxSafe": 650,
      "phosphene": {
        "location": { "x": 0.3, "y": -0.2 },
        "reliableSize": "small"
      }
    }
  ],
  "recommendations": {
    "disableElectrodes": [15, 42],
    "reduceMaxCurrent": [7, 23]
  }
}

3.6 Stimulation Control

PUT /stimulation/parameters

Update stimulation parameters for neural encoding.

Request Body

{
  "deviceId": "BIO-EYE-001",
  "mode": "navigation" | "reading" | "face_detection",
  "globalParameters": {
    "pulseWidth": 450,          // μs
    "frequency": 20,            // Hz
    "maxCurrent": 800           // μA
  },
  "electrodeOverrides": [
    {
      "index": 5,
      "maxCurrent": 650,
      "disabled": false
    }
  ]
}

Response

{
  "status": "updated",
  "appliedAt": "2025-12-20T10:45:00Z",
  "warnings": [
    "Electrode 5 current reduced due to impedance increase"
  ]
}

POST /stimulation/test

Send test stimulation pattern.

Request Body

{
  "pattern": "checkerboard" | "horizontal_lines" | "vertical_lines" | "single_electrode",
  "duration": 5000,  // ms
  "electrodes": [0, 2, 4, 6, 8]
}

Request Body

{
  "deviceId": "BIO-EYE-001",
  "reason": "Patient discomfort",
  "triggeredBy": "patient" | "caregiver" | "automatic"

3.10 Error Handling

Error Response Format

{
  "error": {
    "code": "DEVICE_NOT_FOUND",
    "message": "Device BIO-EYE-999 not found",
    "status": 404,
    "timestamp": "2025-12-20T10:55:00Z",
    "requestId": "req-abc123"
  }
}

Common Error Codes

Code Status Description
UNAUTHORIZED 401 Invalid or expired authentication token
FORBIDDEN 403 Insufficient permissions for operation
DEVICE_NOT_FOUND 404 Specified device does not exist
DEVICE_OFFLINE 503 Device is not currently connected
SAFETY_LIMIT_EXCEEDED 400 Requested parameters exceed safety limits
CALIBRATION_IN_PROGRESS 409 Cannot modify settings during calibration
Best Practice: Always check error responses and implement appropriate error handling and user feedback in client applications.
