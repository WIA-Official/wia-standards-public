제3장: API 엔드포인트

장치 제어, 이미지 스트리밍, 보정

3.1 API 개요

WIA 바이오닉 아이 API는 바이오닉 아이 장치 제어, 시각 데이터 스트리밍 및 시스템 구성 관리를 위한 RESTful 엔드포인트를 제공합니다. API는 데이터 교환을 위해 JSON을 사용하며 HTTP 및 WebSocket 연결을 모두 지원합니다.

기본 URL

https://api.bionic-eye.wia.org/v1

인증

모든 API 요청에는 Bearer 토큰을 사용한 인증이 필요합니다:

Authorization: Bearer {access_token}
보안: API 토큰은 안전하게 보관해야 합니다. 모든 통신에 HTTPS를 사용하고 24시간마다 토큰 교체를 구현하십시오.

3.2 장치 관리

GET /devices

인증된 사용자에 대해 등록된 모든 바이오닉 아이 장치를 나열합니다.

요청

GET /devices
Authorization: Bearer {token}

응답

{
  "devices": [
    {
      "deviceId": "BIO-EYE-001",
      "name": "왼쪽 눈 Argus II",
      "type": "epiretinal",
      "manufacturer": "Second Sight",
      "model": "Argus II",
      "status": "active",
      "implantDate": "2024-03-15",
      "electrodes": {
        "total": 60,
        "active": 58,
        "failed": 2
      },
      "lastSync": "2025-12-20T10:30:00Z"
    }
  ]
}

POST /camera/capture

테스트 또는 보정을 위해 단일 프레임을 캡처합니다.

요청 본문

{
  "saveToStorage": true,
  "includeMetadata": true
}

3.3 이미지 스트리밍

WebSocket: /stream/video

WebSocket 연결을 사용하여 카메라에서 클라이언트로 실시간 비디오 스트리밍합니다.

연결

ws://api.bionic-eye.wia.org/v1/stream/video?deviceId=BIO-EYE-001&token={access_token}

메시지 형식

{
  "type": "frame",
  "frameId": "uuid",
  "timestamp": 1703001234567,
  "sequenceNumber": 12345,
  "encoding": "jpeg",
  "data": "base64_encoded_image"
}

3.4 보정

POST /calibration/start

전극 매핑을 위한 보정 세션을 시작합니다.

요청 본문

{
  "deviceId": "BIO-EYE-001",
  "type": "phosphene_mapping" | "threshold_detection" | "impedance_check",
  "electrodes": [0, 1, 2, 3, 4, 5],
  "parameters": {
    "startCurrent": 50,    // μA
    "maxCurrent": 800,
    "stepSize": 25
  }
}

응답

{
  "sessionId": "cal-20251220-001",
  "status": "started",
  "estimatedDuration": 1800,  // 초
  "nextElectrode": 0
}

3.5 자극 제어

PUT /stimulation/parameters

신경 인코딩을 위한 자극 매개변수를 업데이트합니다.

요청 본문

{
  "deviceId": "BIO-EYE-001",
  "mode": "navigation" | "reading" | "face_detection",
  "globalParameters": {
    "pulseWidth": 450,          // μs
    "frequency": 20,            // Hz
    "maxCurrent": 800           // μA
  }
}

3.6 안전 및 비상

POST /emergency/stop

모든 자극을 즉시 중지합니다.

요청 본문

{
  "deviceId": "BIO-EYE-001",
  "reason": "환자 불편감",
  "triggeredBy": "patient" | "caregiver" | "automatic"
}

응답

{
  "status": "stopped",
  "stoppedAt": "2025-12-20T10:50:00Z",
  "responseTime": 2.3,  // ms
  "allElectrodesDisabled": true
}

3.7 오류 처리

오류 응답 형식

{
  "error": {
    "code": "DEVICE_NOT_FOUND",
    "message": "장치 BIO-EYE-999를 찾을 수 없음",
    "status": 404,
    "timestamp": "2025-12-20T10:55:00Z",
    "requestId": "req-abc123"
  }
}

일반 오류 코드

코드 상태 설명
UNAUTHORIZED 401 잘못되거나 만료된 인증 토큰
FORBIDDEN 403 작업에 대한 권한 부족
DEVICE_NOT_FOUND 404 지정된 장치가 존재하지 않음
DEVICE_OFFLINE 503 장치가 현재 연결되지 않음
SAFETY_LIMIT_EXCEEDED 400 요청된 매개변수가 안전 제한을 초과함
모범 사례: 항상 오류 응답을 확인하고 클라이언트 응용 프로그램에서 적절한 오류 처리 및 사용자 피드백을 구현하십시오.
