5장: 2단계 - API 인터페이스 정의

5.1 API 설계 원칙

WIA-AI-ROBOT 표준의 2단계는 AI 시스템과 로봇을 연결하는 포괄적인 API 인터페이스를 정의합니다:

원칙 설명 구현
RESTful 설계 표준 HTTP 메서드 및 상태 코드 GET, POST, PUT, DELETE 작업
무상태성 각 요청에 모든 필요한 정보 포함 JWT 토큰, 요청의 세션
리소스 지향 URL이 리소스를 나타냄 /robots/{id}/commands, /sensors/{id}
버전 관리 API가 여러 버전 지원 URL 경로에 /v1/, /v2/
잘 문서화됨 OpenAPI 사양 Swagger/OpenAPI 3.0

5.2 AI 추론 엔드포인트

LLM 및 비전 AI 통합을 위한 표준 API:

LLM 추론 API

POST /v1/ai/language/infer
Content-Type: application/json

요청:
{
  "model": "gpt-4",
  "messages": [
    {"role": "system", "content": "로봇 작업 계획자..."},
    {"role": "user", "content": "빨간 상자를 집어 선반에 놓으세요"}
  ],
  "context": {
    "robot_state": {...},
    "environment": {...}
  }
}

응답:
{
  "commands": [
    {"type": "scan_area", "target": "red_box"},
    {"type": "navigate_to", "object": "red_box"},
    {"type": "grasp", "object": "red_box"}
  ],
  "confidence": 0.92
}

비전 AI API

POST /v1/ai/vision/detect
Content-Type: multipart/form-data

요청:
- image: [바이너리 이미지 데이터]
- task: "object_detection"
- classes: ["person", "box", "shelf"]

응답:
{
  "detections": [
    {
      "class": "box",
      "confidence": 0.95,
      "bounding_box": {"x": 120, "y": 80},
      "color": "red"
    }
  ]
}

5.3 로봇 제어 API

다양한 로봇 플랫폼을 제어하기 위한 통합 인터페이스:

명령 실행 API

POST /v1/robots/{robot_id}/commands
Content-Type: application/json

요청:
{
  "type": "navigate_to",
  "parameters": {
    "target": {"x": 5.0, "y": 3.2, "z": 0.0}
  }
}

응답: 202 Accepted
{
  "command_id": "cmd_123",
  "status": "accepted",
  "estimated_duration": 30000
}

상태 조회 API

GET /v1/robots/{robot_id}/status

응답:
{
  "robot_id": "rb_001",
  "state": "executing",
  "battery": 85,
  "position": {"x": 4.2, "y": 3.0},
  "health": "good"
}

5.4 원격 측정 스트리밍 API

실시간 센서 데이터 게시:

WebSocket 원격 측정 스트림

WS /v1/robots/{robot_id}/telemetry/stream

구독 메시지:
{
  "action": "subscribe",
  "channels": ["camera", "position", "battery"],
  "rate_hz": {
    "camera": 10,
    "position": 50,
    "battery": 1
  }
}

스트림 메시지:
{
  "channel": "position",
  "data": {"x": 4.2, "y": 3.0}
}

5.5 구성 관리 API

로봇 기능 및 매개변수 관리를 위한 API:

기능 발견

GET /v1/robots/{robot_id}/capabilities

응답:
{
  "capabilities": {
    "manipulation": {
      "available": true,
      "dof": 6,
      "reach": 1.3
    },
    "navigation": {
      "available": true,
      "max_speed": 1.5
    }
  },
  "supported_commands": [
    "navigate_to", "grasp", "release"
  ]
}

5.7 인증 및 권한 부여

API 액세스를 위한 보안 메커니즘:

방법 사용 사례 구현
API 키 서비스 간 X-API-Key 헤더
JWT 토큰 사용자 인증 Bearer 토큰
OAuth 2.0 타사 액세스 권한 부여 흐름
mTLS 로봇-서버 인증서 기반

5.8 오류 처리

표준 HTTP 상태 코드 및 오류 형식:

상태 코드 의미 예
200 OK 성공적인 요청 GET 상태 성공
202 Accepted 처리를 위해 요청 수락됨 명령 대기 중
400 Bad Request 잘못된 요청 형식 필수 필드 누락
401 Unauthorized 인증 필요 토큰 누락
404 Not Found 리소스가 존재하지 않음 로봇 ID를 찾을 수 없음
422 Unprocessable 형식은 유효하지만 내용 무효 명령이 안전 규칙 위반

오류 응답 형식

HTTP 422 Unprocessable Entity
{
  "error": {
    "code": "SAFETY_VIOLATION",
    "message": "명령이 작업 공간 경계 위반",
    "details": {
      "command_id": "cmd_123",
      "violation": "목표 위치가 허용 작업 공간 밖"
    }
  }
}

5.9 속도 제한 및 조절

과부하로부터 서비스 보호:

HTTP/1.1 200 OK
X-RateLimit-Limit: 1000
X-RateLimit-Remaining: 950
X-RateLimit-Reset: 1642251600

// 제한 초과 시:

{
  "error": {
    "code": "RATE_LIMIT_EXCEEDED",
    "retry_after": 60
  }
}

5.10 API 문서 모범 사례

OpenAPI를 사용한 포괄적인 API 문서:

openapi: 3.0.0
info:
  title: WIA-AI-ROBOT API
  version: 1.0.0
paths:
  /robots/{robot_id}/commands:
    post:
      summary: 로봇 명령 실행
      parameters:
        - name: robot_id
          in: path
          required: true
      responses:
        '202':
          description: 명령 수락됨

핵심 내용

  • WIA-AI-ROBOT API는 실시간 지원과 함께 RESTful 원칙을 따릅니다
  • AI 추론 엔드포인트는 LLM 및 비전 AI에 대한 표준화된 액세스를 제공합니다
  • 로봇 제어 API는 다양한 로봇 플랫폼을 위한 통합 인터페이스를 제공합니다
  • 원격 측정 스트리밍은 실시간 데이터 전달을 위해 WebSocket 및 SSE를 사용합니다
  • 구성 API는 동적 기능 발견 및 매개변수 관리를 가능하게 합니다
  • 플릿 관리 API는 여러 로봇을 효율적으로 조정합니다
  • 여러 인증 방법이 다양한 사용 사례를 지원합니다 (API 키, JWT, OAuth, mTLS)
  • 표준 HTTP 상태 코드 및 오류 형식은 일관된 오류 처리를 보장합니다
  • 속도 제한은 과부하로부터 서비스를 보호합니다
  • OpenAPI 사양은 포괄적인 API 문서를 제공합니다
