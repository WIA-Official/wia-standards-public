4장: 1단계 - 데이터 형식 표준화

4.1 데이터 형식 표준의 중요성

데이터 형식 표준화는 WIA-AI-ROBOT 표준의 기초를 형성합니다. 일관된 데이터 구조 없이는 AI 모델과 로봇이 효과적으로 통신할 수 없습니다.

4.2 AI 프롬프트 템플릿

구조화된 프롬프트 템플릿은 LLM이 로봇 컨텍스트를 이해하고 적절한 명령을 생성할 수 있게 합니다:

{
  "role": "system",
  "content": {
    "identity": "로봇 작업 계획자 및 명령자",
    "robot_type": "{{로봇_유형}}",
    "capabilities": [{{기능_목록}}],
    "constraints": {
      "workspace": {{작업_공간_경계}},
      "payload": "{{최대_페이로드}}",
      "speed_limits": {{속도_제한}},
      "safety_zones": [{{안전_구역}}]
    },
    "output_format": {
      "type": "structured_commands",
      "schema": "wia-airi-v1",
      "validation_required": true
    }
  }
}

4.3 로봇 명령 스키마

모든 로봇 명령은 표준화된 JSON 스키마를 따릅니다:

기본 명령 구조

{
  "command_id": "cmd_{{UUID}}",
  "timestamp": "{{ISO8601_타임스탬프}}",
  "version": "1.0",
  "type": "{{명령_유형}}",
  "priority": "normal|high|critical",
  "parameters": {
    // 명령별 매개변수
  },
  "safety": {
    "require_validation": true,
    "allow_override": false,
    "timeout_ms": 30000,
    "collision_check": true
  }
}

표준 명령 유형

명령 유형 목적 주요 매개변수
navigate_to 목표 위치로 이동 target, waypoints, speed
grasp 객체 집기 object_id, approach_vector, force_limit
release 잡은 객체 놓기 location, placement_pose
scan_area 영역 시각적 조사 region, resolution, modality
manipulate 복잡한 조작 작업 tool, operation, target
wait 실행 일시 중지 duration_ms, condition
abort 현재 동작 취소 reason, safe_stop

4.4 센서 데이터 형식

AI 시스템이 로봇 인식을 이해할 수 있도록 센서 데이터를 위한 표준화된 형식:

카메라 이미지 데이터

{
  "sensor_type": "camera",
  "sensor_id": "cam_front",
  "timestamp": "2025-01-15T10:30:45.500Z",
  "image": {
    "encoding": "jpeg",
    "width": 1920,
    "height": 1080,
    "data": "{{BASE64_인코딩_이미지}}"
  },
  "camera_info": {
    "fov_horizontal": 90,
    "fov_vertical": 60
  }
}

4.5 응답 사양

로봇은 AI 시스템에 구조화된 피드백을 제공합니다:

명령 확인

{
  "response_type": "acknowledgment",
  "command_id": "cmd_a7b3c9d1",
  "timestamp": "2025-01-15T10:30:45.200Z",
  "status": "accepted|rejected|queued",
  "validation_result": {
    "passed": true,
    "checks": {
      "syntax": "pass",
      "safety": "pass",
      "feasibility": "pass"
    }
  },
  "estimated_duration": 30000
}

실행 상태

{
  "response_type": "status",
  "command_id": "cmd_a7b3c9d1",
  "state": "executing|completed|failed|aborted",
  "progress": 0.75,
  "current_action": "목표에 접근 중",
  "telemetry": {
    "position": {"x": 4.2, "y": 3.0, "z": 0.0},
    "battery": 85
  }
}

4.7 기능 설명 형식

로봇은 기계 판독 가능한 형식으로 기능을 광고합니다:

{
  "robot_id": "rb_industrial_001",
  "model": "UR10e",
  "capabilities": {
    "manipulation": {
      "available": true,
      "dof": 6,
      "reach": 1.3,
      "payload": 12.5
    },
    "navigation": {
      "available": true,
      "max_speed": 1.5
    }
  },
  "supported_commands": [
    "navigate_to", "grasp", "release", "scan_area"
  ]
}

4.8 오류 및 예외 형식

표준화된 오류 보고:

{
  "error_type": "execution_error",
  "command_id": "cmd_a7b3c9d1",
  "severity": "critical|error|warning|info",
  "code": "E_COLLISION_DETECTED",
  "message": "내비게이션 중 충돌 감지",
  "recovery_actions": [
    "back_up",
    "replan_path",
    "request_assistance"
  ]
}

4.9 데이터 형식 버전 관리

모든 데이터 형식에는 진화를 위한 버전 정보가 포함됩니다:

버전 필드 위치 목적
schema_version 모든 메시지의 루트 전체 형식 버전
api_version HTTP 헤더 API 호환성
protocol_version 프로토콜 핸드셰이크 통신 프로토콜

4.10 데이터 형식 모범 사례

  • 사람이 읽을 수 있는 데이터에 JSON 사용: 구성, 명령, 응답
  • 대량 데이터에 바이너리 사용: 포인트 클라우드, 이미지, 원격 측정
  • 타임스탬프 포함: 모든 메시지에 ISO 8601 타임스탬프 필요
  • 모든 것 검증: JSON 스키마 검증 사용
  • 명시적이어야 함: 단위 포함 (미터, 라디안, 밀리초)
  • 확장 지원: 호환성을 깨지 않고 사용자 정의 필드 허용

핵심 내용

  • 데이터 형식 표준화는 모든 AI 모델과 로봇 간의 상호운용성을 가능하게 합니다
  • AI 프롬프트 템플릿은 로봇 컨텍스트와 제약으로 LLM 상호작용을 구조화합니다
  • 로봇 명령은 안전 매개변수가 있는 통합 JSON 스키마를 따릅니다
  • 표준 명령 유형에는 navigate_to, grasp, release, scan_area, manipulate가 포함됩니다
  • 센서 데이터 형식은 카메라, LiDAR, IMU 등 인식 시스템을 다룹니다
  • 응답 사양에는 확인, 상태 업데이트, 완료 보고서가 포함됩니다
  • 비전 AI 결과는 객체 감지 및 장면 이해를 위한 표준화된 형식을 가집니다
  • 기능 설명은 로봇 기능의 동적 발견을 가능하게 합니다
  • 버전 협상은 구현 전반에 걸쳐 호환성을 보장합니다
