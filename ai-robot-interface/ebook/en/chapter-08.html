Quick Start Example

# Install WIA client library
pip install wia-airi

# Create a simple robot client
from wia_airi import RobotClient, Command

# Initialize client
client = RobotClient(
    robot_id="my_robot_001",
    api_endpoint="https://api.wia-robot.org/v1",
    auth_token="your_token_here"
)

# Connect to WIA server
await client.connect()

# Define command handler
@client.on_command
async def handle_command(command: Command):
    if command.type == "navigate_to":
        target = command.parameters["target"]
        print(f"Navigating to {target}")
        # Execute navigation
        await robot.navigate_to(target["x"], target["y"])
        return {"status": "success"}

# Stream telemetry
@client.telemetry_stream(rate_hz=10)
async def publish_position():
    return {
        "position": robot.get_position(),
        "battery": robot.get_battery()
    }

# Run client
await client.run()

8.4 Testing Requirements

Comprehensive testing ensures reliable AI-robot integration:

Test Categories

Test Type Coverage Pass Criteria
Unit Tests Individual functions, message parsing 100% critical paths
Integration Tests Component interactions, API contracts All interfaces tested
System Tests End-to-end workflows 95% success rate
Safety Tests Emergency stops, collision avoidance 100% safety scenarios
Performance Tests Latency, throughput, reliability Meet SLA requirements
Security Tests Authentication, authorization, encryption Zero critical vulnerabilities

Safety Test Scenarios

1. Emergency Stop Response
   - Trigger: Emergency button pressed
   - Expected: Robot stops within 100ms
   - Verify: All motion ceases, hold position

2. Obstacle Detection
   - Trigger: Obstacle in path
   - Expected: Stop or avoid within safe distance
   - Verify: No collision occurs

3. Communication Loss
   - Trigger: Network disconnection
   - Expected: Enter safe mode within 5s
   - Verify: Robot stops safely

4. AI Hallucination Detection
   - Trigger: Invalid command from AI
   - Expected: Command rejected
   - Verify: Robot does not execute unsafe action

5. Boundary Violation
   - Trigger: Command exceeds workspace
   - Expected: Command rejected
   - Verify: Robot remains in allowed area

WIA-AI-ROBOT offers four certification levels:

Level Requirements Testing Validity

Standard Phase 1-3 compliance + safety Third-party review + lab tests 2 years
Advanced Full compliance + rigorous testing On-site audit + extended testing 3 years

   - Functional tests

8.7 Common Implementation Challenges

Solutions to frequently encountered problems:

Challenge Symptom Solution
High Latency Commands take >5s to execute Use edge AI, optimize network, reduce payload size
Connection Drops Frequent disconnections Implement reconnection logic, use WebSocket keepalive
AI Errors Invalid commands from LLM Improve prompt engineering, add validation layers
Message Format Errors Validation failures Use JSON schema validation, check examples
Authentication Issues 401 Unauthorized errors Verify token not expired, check API key
