Chapter 4: Phase 1 - Data Format Standardization

4.1 The Importance of Data Format Standards

Data format standardization forms the foundation of the WIA-AI-ROBOT standard. Without consistent data structures, AI models and robots cannot communicate effectively. Phase 1 establishes universal formats for all data exchanged between AI systems and robots.

Key objectives of data format standardization:

  • Interoperability: Any AI model can understand any robot's data
  • Clarity: Unambiguous representation of commands and sensor data
  • Efficiency: Compact formats for low-latency transmission
  • Extensibility: Easy to add new fields without breaking compatibility

4.3 Robot Command Schema

All robot commands follow a standardized JSON schema:

Base Command Structure

{
  "command_id": "cmd_{{UUID}}",
  "timestamp": "{{ISO8601_TIMESTAMP}}",
  "version": "1.0",
  "type": "{{COMMAND_TYPE}}",
  "priority": "normal|high|critical",
  "parameters": {
    // Command-specific parameters
  },
  "safety": {
    "require_validation": true,
    "allow_override": false,
    "timeout_ms": 30000,
    "collision_check": true
  },
  "metadata": {
    "source": "ai|human|system",
    "session_id": "{{SESSION_ID}}",
    "user_id": "{{USER_ID}}"
  }
}

Standard Command Types

Command Type Purpose Key Parameters
navigate_to Move to target location target, waypoints, speed, avoid_obstacles
grasp Pick up object object_id, approach_vector, force_limit
release Release held object location, placement_pose, gentle
scan_area Visually survey region region, resolution, modality
manipulate Complex manipulation task tool, operation, target
wait Pause execution duration_ms, condition
abort Cancel current action reason, safe_stop

4.9 Data Format Versioning

All data formats include version information for evolution:

Version Field Location Purpose
schema_version Root of all messages Overall format version
api_version HTTP headers API compatibility
protocol_version Protocol handshake Communication protocol

Version Negotiation

Client: "I support WIA-AIRI v1.0, v1.1"
Server: "I support WIA-AIRI v1.1, v1.2"
Result: Both use v1.1 (highest common version)

4.10 Best Practices for Data Formats

  • Use JSON for Human-Readable Data: Configuration, commands, responses
  • Use Binary for High-Volume Data: Point clouds, images, telemetry
  • Include Timestamps: All messages must have ISO 8601 timestamps
  • Validate Everything: Use JSON Schema validation
  • Be Explicit: Include units (meters, radians, milliseconds)
  • Support Extensions: Allow custom fields without breaking compatibility
  • Document Examples: Provide real-world examples for each format

Key Takeaways

  • Data format standardization enables interoperability between any AI model and any robot
  • AI prompt templates structure LLM interactions with robot context and constraints
  • Robot commands follow a unified JSON schema with safety parameters
  • Standard command types include navigate_to, grasp, release, scan_area, manipulate, wait, abort
  • Sensor data formats cover cameras, LiDAR, IMU, and other perception systems
  • Response specifications include acknowledgment, status updates, and completion reports
  • Vision AI results have standardized formats for object detection and scene understanding
  • Capability descriptions enable dynamic discovery of robot features
  • Error formats provide structured exception reporting with recovery actions
  • Version negotiation ensures compatibility across implementations
