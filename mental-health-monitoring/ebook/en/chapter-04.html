Chapter 4: CBT App Integration

"Cognitive-behavioral therapy works when it is practised, not when it is promised. A platform's job is to make the right practice the easy practice — at the right moment, in the right frame of mind."

— WIA-MED-020, Therapy-App Interoperability Charter

4.1 What CBT looks like in software

CBT is an evidence-based, structured psychotherapy developed by Aaron T. Beck and refined over five decades. Software adaptations — guided self-help apps, clinician-assisted programmes, and chatbot companions — share a common core: psychoeducation, thought records, behavioural activation, exposure hierarchies, sleep and worry scheduling, relapse prevention, and between-session homework. WIA-MED-020 does not prescribe a specific therapeutic school but defines the interfaces through which any reputable CBT implementation can plug into the monitoring platform.

4.2 Supported CBT modules

  • Thought records (ABC, columns). Structured capture of activating event, automatic thought, cognitive distortion, alternative thought, and outcome; each record is an FHIR QuestionnaireResponse linked to the thought-record Questionnaire canonical.
  • Behavioural activation. Pleasure/mastery ratings, activity scheduling, and follow-through tracking.
  • Exposure hierarchies. SUDS-rated ladders for anxiety and OCD; the module MUST ensure no one-click escalation — every step requires affirmative user confirmation.
  • Sleep restriction and worry time. Combined with the Chapter 2 sleep metrics to allow closed-loop feedback.
  • Relapse prevention. Warning-sign inventory, coping card library, safety plan (Chapter 5) co-editable with clinician.

4.3 Interoperability contract

Any compliant third-party CBT app exposes:

  1. OAuth 2.1 + PKCE authorisation with WIA as the resource server.
  2. FHIR R5 Task + QuestionnaireResponse endpoints for homework assignments and completions.
  3. A SMART-on-FHIR launch context so the clinician dashboard can jump straight into the relevant exercise.
  4. A deep-link URI scheme (wia-mhm://cbt/module/{id}?exercise={eid}) so notifications route users into the right place without lengthy navigation.

All communication happens over TLS 1.3. CBT apps never see raw digital-phenotype data; they receive coarse-grained, user-approved summaries such as {"mood_7d": "declining", "sleep_7d": "fragmented"} so that the exercises chosen are contextually relevant without privacy leakage.

4.4 Evidence quality

Digital CBT efficacy is supported by an extensive randomised-controlled-trial literature, with notable meta-analyses (Carlbring 2018; Andersson 2019) showing effect sizes comparable to face-to-face CBT for mild-to-moderate depression and anxiety. However, the evidence base is heterogeneous: some apps in the consumer market have never been tested. WIA-MED-020 therefore defines an evidence badge — displayed next to each integrated app — that classifies it by the strength of public evidence (RCT, open trial, uncontrolled, not yet tested). Clinicians can filter recommendations to only show apps meeting a configurable evidence minimum.

4.5 Worked example: thought-record sync

POST /fhir/r5/QuestionnaireResponse
Authorization: Bearer ...
Content-Type: application/fhir+json

{
  "resourceType": "QuestionnaireResponse",
  "status": "completed",
  "questionnaire": "https://wia-mhm.org/fhir/Questionnaire/thought-record",
  "subject": { "reference": "Patient/pat-0042" },
  "authored": "2026-04-25T14:30:00Z",
  "item": [
    { "linkId": "1.situation",     "answer": [{"valueString":"Meeting with manager"}] },
    { "linkId": "2.automatic",     "answer": [{"valueString":"I'm going to be fired"}] },
    { "linkId": "3.distortion",    "answer": [{"valueCoding":{"code":"catastrophizing"}}] },
    { "linkId": "4.evidence-for",  "answer": [{"valueString":"Tone was curt"}] },
    { "linkId": "5.evidence-against","answer": [{"valueString":"Agenda mentioned roadmap, not review"}] },
    { "linkId": "6.alternative",   "answer": [{"valueString":"This may be a roadmap discussion"}] },
    { "linkId": "7.outcome-mood",  "answer": [{"valueInteger": 58}] }
  ]
}

The response is stored on the user's own device first; upload to a clinician or research endpoint is always preceded by an explicit toggle and a summary of what fields will leave the device.

4.6 Cultural and linguistic adaptation

CBT was developed in a specific cultural context and does not translate uncritically. WIA-MED-020 requires integrators to declare which locales and cultural adaptations a module supports. Modules validated specifically for non-Western contexts (e.g., Korean, Japanese, Arabic cognitive restructuring adaptations) are preferentially offered when the user's locale matches. The goal is not to maximise engagement but to maximise effective engagement.
