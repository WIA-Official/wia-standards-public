제8장: 개인정보 보호 및 윤리

"정신건강 데이터는 가장 민감한 개인정보입니다. 신뢰 없이는 치료도 없습니다. 개인정보 보호는 선택이 아니라 필수입니다."

— WIA-MED-020 핵심 원칙

8.1 정신건강 데이터의 특수성

🔐 왜 더 민감한가?

위험 결과
고용 차별 우울증 이력으로 채용 거부
보험 거부 정신질환 기록으로 보험 가입 불가 또는 보험료 급증
사회적 낙인 관계, 평판 손상
법적 불이익 양육권 소송 시 불리, 총기 소지 제한
신뢰 파괴 한 번 유출되면 치료 시스템 전체 불신

→ 정신건강 앱은 사용자의 가장 취약한 순간을 다룹니다. 절대적 보호가 필요합니다.

8.2 법적 규제 준수

🌍 주요 규제

미국: HIPAA (Health Insurance Portability and Accountability Act)

  • 적용 대상: 의료 제공자, 건강보험사, 이들의 협력업체
  • 핵심 요구사항:
    • 환자 동의 없이 정보 공유 금지
    • AES-256 암호화
    • 감사 로그 (audit trail) 유지
    • 위반 시 최대 $50,000/건 + 형사 처벌

유럽: GDPR (General Data Protection Regulation)

  • 특별 보호 데이터: 건강 정보는 "특별 카테고리" (Article 9)
  • 사용자 권리:
    • 열람권 (Right to Access)
    • 삭제권 (Right to Erasure, "잊힐 권리")
    • 이동권 (Data Portability)
    • 자동 결정 거부권 (AI 진단 거부 가능)
  • 위반 벌금: 최대 €20M 또는 연간 매출 4%

한국: 개인정보보호법 + 의료법

  • 민감정보: 건강, 유전, 범죄 기록 등 (별도 동의 필요)
  • 의료 데이터: 비의료기관은 수집·보관 제한
  • 국외 이전: 별도 고지 및 동의 필수

8.3 기술적 보안 조치

🔒 1. 암호화

단계 방법 표준
전송 중 (In Transit) 앱 ↔ 서버 통신 TLS 1.3
저장 중 (At Rest) 서버 DB, 기기 로컬 AES-256
엔드투엔드 (E2E) 환자 ↔ 치료사 메시지 Signal Protocol

🗝️ 2. 접근 제어

  • 역할 기반 접근 제어 (RBAC) — 치료사는 자기 환자만, 관리자는 통계만
  • 다단계 인증 (MFA) — 비밀번호 + OTP 또는 생체인식
  • 세션 타임아웃 — 15분 비활성 시 자동 로그아웃
  • 감사 로그 — 누가 언제 어떤 데이터에 접근했는지 기록

🛡️ 3. 데이터 최소화

수집하는 데이터의 양을 최소화:

원칙

  • 목적 제한: 치료에 필요한 것만 수집
  • 로컬 처리 우선: 가능하면 기기에서 처리, 서버 전송 최소화
  • 익명화: 연구 목적 데이터는 개인 식별 정보 제거
  • 자동 삭제: 치료 종료 후 90일 경과 시 데이터 삭제 (법적 요구사항 제외)

🔬 4. 차등 개인정보 보호 (Differential Privacy)

통계 분석 시 개인 프라이버시 보호:

예시: "우울증 환자의 평균 수면 시간은?"

일반적 방법:
→ 모든 환자 데이터 취합 → 평균 계산
→ 위험: 데이터 역추적으로 개인 식별 가능

차등 개인정보 보호:
→ 각 데이터에 수학적 노이즈 추가
→ 평균 계산 (여전히 통계적으로 유효)
→ 개인 데이터 역추적 불가능

8.4 윤리적 고려사항

🤝 1. 진정한 동의 (Informed Consent)

정신질환 환자의 동의는 특별히 주의:

문제점

  • 인지 기능 저하: 우울증 시 집중력·판단력 저하로 약관 이해 어려움
  • 절박함: 위기 상황에서 급히 동의 → 충분한 고려 없음
  • 약관 피로: 긴 개인정보 처리방침을 읽지 않음

해결책

  • 단계적 동의: 처음 앱 사용 시 핵심만, 추가 기능 사용 시 재동의
  • 명확한 언어: 법률 용어 대신 "당신의 기분 데이터를..."
  • 시각적 설명: 인포그래픽, 비디오
  • 재확인: 3개월마다 "계속 데이터를 공유하시겠습니까?" 재동의

⚖️ 2. 알고리즘 공정성

AI 편향으로 인한 불평등 방지:

편향 유형 예시 완화 방법
인종 편향 흑인 환자 우울증 과소 진단 다양한 인종 데이터 균형
성별 편향 남성 우울증 저평가 성별별 정확도 모니터링
사회경제적 편향 고가 스마트폰 사용자만 대상 저사양 기기 지원

🧠 3. 자율성 vs 보호주의

딜레마: 언제까지 사용자 선택을 존중하고, 언제 개입해야 하는가?

시나리오

환자가 앱 사용을 중단:

  • 자율성 존중: "사용자가 원하지 않으면 내버려둔다"
  • 보호주의: "중단은 악화 신호일 수 있다 → 치료사에게 알린다"

WIA-MED-020 권장: 중간 경로 — 3일 미사용 시 부드러운 리마인더, 7일 미사용 시 치료사 알림 (환자에게 사전 고지)

  • ☐ 24/7 핫라인 연결
  • ☐ 정기적 보안 감사
  • ☐ 침해 사고 대응 계획
  • ☐ 데이터 백업 및 복구

8.6 미래 도전과제

🔮 새로운 위협

  • 양자 컴퓨팅: 현재 암호화 방식 무력화 위험
