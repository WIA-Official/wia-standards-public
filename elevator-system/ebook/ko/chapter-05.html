제5장

🔌 2단계 - API 인터페이스 설계

RESTful API, WebSocket, GraphQL을 통한 엘리베이터 시스템과의 상호작용을 위한 표준 인터페이스를 정의합니다.

5.1 RESTful HTTP API 아키텍처

WIA 표준은 엘리베이터 제어에 REST 원칙을 사용합니다. HTTP 동사(GET, POST, PUT, DELETE)가 엘리베이터 작업에 매핑됩니다.

WIA 표준의 이 영역에서 취한 접근 방식은 전통적인 독점 방법에 비해 상당한 발전을 나타냅니다. 개방형 프로토콜과 명확한 사양을 확립함으로써 호환성을 유지하면서 혁신을 가능하게 합니다. 이는 제조업체, 건물 소유주, 서비스 제공업체, 그리고 궁극적으로 안전하고 신뢰할 수 있는 수직 이동에 의존하는 최종 사용자에게 이익이 됩니다.

기술 사양

RESTful HTTP API 아키텍처를 구현하기 위한 기술 요구사항은 엄격하면서도 유연하게 설계되었습니다. 신규 건설부터 개조 현대화, 단일 엘리베이터부터 대규모 다차량 그룹까지 다양한 배포 시나리오를 고려합니다. 사양에는 데이터 형식, 통신 프로토콜, 보안 요구사항, 성능 벤치마크가 포함됩니다.

구성 요소 요구사항 검증 방법 준수 수준
데이터 형식 JSON 스키마 v2021-12 자동화된 스키마 검증 필수
통신 TLS 1.3+ 암호화 보안 감사 필수
응답 시간 중요 작업 < 100ms 성능 테스트 필수
가용성 99.9% 가동 시간 모니터링 로그 권장

이러한 사양의 구현에는 신중한 계획과 테스트가 필요합니다. 조직은 개발, 검증, 인증에 충분한 리소스를 할당해야 합니다. WIA 재단은 규정 준수를 단순화하기 위해 참조 구현, 테스트 도구, 인증 서비스를 제공합니다.

구현 참고사항: RESTful HTTP API 아키텍처를 구현할 때 기존 시스템과의 하위 호환성에 특히 주의하십시오. WIA 표준은 점진적 전환을 용이하게 하는 마이그레이션 어댑터와 번역 레이어를 제공합니다. 완전한 시스템 교체는 거의 필요하지 않습니다. 증분 업그레이드가 권장되는 접근 방식입니다.

5.7 속도 제한 및 조절

WIA 표준의 이 영역에서 취한 접근 방식은 전통적인 독점 방법에 비해 상당한 발전을 나타냅니다. 개방형 프로토콜과 명확한 사양을 확립함으로써 호환성을 유지하면서 혁신을 가능하게 합니다. 이는 제조업체, 건물 소유주, 서비스 제공업체, 그리고 궁극적으로 안전하고 신뢰할 수 있는 수직 이동에 의존하는 최종 사용자에게 이익이 됩니다.

기술 사양

속도 제한 및 조절를 구현하기 위한 기술 요구사항은 엄격하면서도 유연하게 설계되었습니다. 신규 건설부터 개조 현대화, 단일 엘리베이터부터 대규모 다차량 그룹까지 다양한 배포 시나리오를 고려합니다. 사양에는 데이터 형식, 통신 프로토콜, 보안 요구사항, 성능 벤치마크가 포함됩니다.
