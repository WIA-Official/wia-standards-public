7.1 수정 절차
--------------------------------------------------------------------------------

동의 문서는 활성화 후에도 승인을 받아 수정할 수 있습니다.

[ID] 수정 기록:

    Amendment {
      amendmentId:      "AMD-CONSENT-2025-001-001"
      amendmentNumber:  1

      effectiveDate:    "2025-06-15"
      createdAt:        "2025-06-01T10:00:00Z"

      amendmentType:    "REVIVAL_CONDITIONS"
      description:      "최소 보존 기간을 75년으로 연장"

      changes: [
        {
          section:       "revival_conditions"
          field:         "minimum_duration"
          previousValue: "P50Y"
          newValue:      "P75Y"
          justification: "기술 전망에 따르면 더 긴 대기가 최적"
        }
      ]

      initiator:        "홍명동"  // 후견인
      reason:           "업데이트된 의료 기술 예측"

      approvals:        []
      requiredApprovals: 3

      status:           "PENDING"
    }

[ID] 수정 워크플로:

    +------------------+
    | 시작             |
    | 후견인/대상자가  |
    | 변경 제안        |
    +--------+---------+
             |
             v
    +------------------+
    | 통지             |
    | 모든 후견인에게  |
    | 알림             |
    +--------+---------+
             |
             v
    +------------------+

    | 후견인들이       |

    +--------+---------+

    +--------v---------+
    | 임계값 도달?     |
    +--------+---------+
        |         |
       예        아니오
        |         |
        v         v
    [승인됨]    [거부됨]


--------------------------------------------------------------------------------
7.2 중요 vs 비중요 변경
--------------------------------------------------------------------------------

[ID] 중요 변경 (2/3 과반수 필요):

    - 부활 조건 수정
    - 철회 승인
    - 후견인 지정 변경
    - 연구 동의 변경
    - 대안 지시 변경
    - 재정 배치 변경

[ID] 비중요 변경 (단순 과반수):

    - 연락처 정보 업데이트
    - 사소한 명확화
    - 시설 선호도
    - 행정 세부사항

[ID] 승인 임계값:

    +----------------------------------------------------------+
    | 변경 유형      | 임계값    | 대기 기간    | 비고         |
    +----------------------------------------------------------+
    | 중요           | 2/3       | 30일         | 법률 검토    |
    | 비중요         | 1/2       | 7일          | 표준         |
    | 긴급           | 2/3       | 없음         | 긴급         |
    | 완전 철회      | 만장일치  | 30일         | 모든 후견인  |
    +----------------------------------------------------------+


--------------------------------------------------------------------------------
7.3 철회 유형
--------------------------------------------------------------------------------

[ID] 보존 전 철회:

    보존이 시작되기 전에 대상자는 자유롭게 철회할 수 있습니다.

    PrePreservationWithdrawal {
      withdrawalType: "PRE_PRESERVATION"
      initiator:      "홍길동"  // 대상자
      initiatorType:  "SUBJECT"

      requirements: {
        subjectSignature:     true
        guardianApproval:     false  // 필요 없음
        waitingPeriod:        "없음"
        notarization:         false
      }

      consequences: {
        documentStatus:       "WITHDRAWN"
        refundAvailable:      true
        recordsDestroyed:     true  // 선택
      }
    }

[ID] 보존 중 철회:

    보존 후에는 후견인만 철회를 시작할 수 있습니다.

    DuringPreservationWithdrawal {
      withdrawalType: "DURING_PRESERVATION"
      initiator:      "홍길순"
      initiatorType:  "GUARDIAN"

      requirements: {
        guardianApproval:     true
        approvalThreshold:    "2/3 과반수"
        waitingPeriod:        "30일"
        legalReview:          true
        notarization:         true
      }

      consequences: {
        documentStatus:       "WITHDRAWN"
        preservationTerminated: true
        remainsDisposition:   "대안 지시에 따라"
      }
    }

[ID] 부활 거부:

    대상자는 조건이 충족된 후에도 부활을 거부할 권리가 있습니다.

    RevivalRefusal {
      withdrawalType: "REVIVAL_REFUSAL"
      initiator:      "홍길동"  // 부활된 대상자
      initiatorType:  "SUBJECT"

      requirements: {
        subjectConsent:       true
        capacityVerification: true  // 정신적 능력 있어야 함
        counselingCompleted:  true
        waitingPeriod:        "48시간"
      }
    }


--------------------------------------------------------------------------------
7.4 환불 정책
--------------------------------------------------------------------------------

[ID] 환불 정책 구조:

    RefundPolicy {
      prePreservationWithdrawal: {
        fullRefund:          false
        refundPercentage:    80
        deductions: [
          "행정 수수료: 250만원",
          "의료 검사 비용",
          "법적 문서 준비"
        ]
        processingTime:      "30일"
      }

      duringPreservation: {
        refundAvailable:     true
        conditions: [
          "보존 시간에 따라 비례 배분",
          "발생한 시설 비용 공제",
          "최소 12개월 보존 비용"
        ]
      }

      revivalRefusal: {
        refundAvailable:     true
        beneficiaries: [
          { name: "홍길순",   share: 50 },
          { name: "홍명동",   share: 25 },
          { name: "홍명희",   share: 25 }
        ]
      }
    }

[ID] 보존 중 환불 공식:

    기본 금액:          총 선불 금액
    차감: 소비 시간     (보존 연수 x 연간 요율)
    차감: 행정비        500만원 정액
    차감: 법적 비용     실제 발생 비용
    차감: 해지 수수료   잔액의 10%
    --------------------------------------------------
    결과: 환불 금액

[!] 환불 지연:
    보존 중 환불은 법적 및 행정적 요구사항으로 인해
    90-180일이 소요될 수 있습니다.

장 요약

핵심 개념:
  [OK] 수정은 중요도에 따라 후견인 승인 필요

  [OK] 세 가지 철회 유형: 보존 전, 보존 중, 부활 거부
