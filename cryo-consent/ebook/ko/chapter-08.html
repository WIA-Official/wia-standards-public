--------------------------------------------------------------------------------
8.1 조건 평가
--------------------------------------------------------------------------------

정기적인 평가가 부활 조건이 충족되었는지 결정합니다.

[ID] 평가 일정:

    자동 평가:
    - 매년: 기본 상태 확인
    - 주요 이벤트: 의료 돌파구, 기술 이정표
    - 후견인 요청: 모든 후견인이 평가 요청 가능

[ID] 평가 프로세스:

    +------------------+
    | 증거 수집        |
    | - 의료 상태      |
    | - 기술 진행      |

    | - 사회적 요인    |
    +--------+---------+
             |
             v
    +------------------+
    | 각 조건          |
    | 평가             |
    +--------+---------+

--------------------------------------------------------------------------------
8.2 부활 결정
--------------------------------------------------------------------------------

[ID] 부활 결정 기록:

    RevivalDecision {
      documentId:   "CONSENT-2025-001"
      decision:     "APPROVE"

      conditionsMet: [
        "COND-MED-001",   // 암 완치 가능
        "COND-TECH-001",  // 부활 기술
        "COND-TIME-001"   // 50년 경과
      ]

      conditionsNotMet:   []
      conditionsPending:  []

      overallScore:   1.0
      recommendation: "모든 조건 충족. 부활 승인됨."

      evaluatedAt:    "2075-06-15T10:30:00Z"
    }

[ID] 결정 결과:

    APPROVE (승인):
    - 모든 필수 조건 충족
    - 부활 준비 진행
    - 모든 후견인과 가족에게 통지
    - 시설 준비 시작

    DENY (거부):
    - 필수 조건 미충족
    - 보존 계속
    - 다음 평가 일정
    - 이유 문서화

    PENDING (보류):
    - 일부 조건 평가 중
    - 추가 증거 대기
    - 지정된 기간 내 재평가

8.3 대상자 통지
--------------------------------------------------------------------------------

[ID] 부활 전 통지:

    부활 결정 승인 시, 다음에게 통지됩니다:

    NotificationSequence {
      step1: {
        recipients:   ["모든 후견인"]
        method:       "보안 메시지 + 전화"
        content:      "부활 조건 충족, 승인 대기 중"
        responseTime: "72시간"
      }

      step2: {
        recipients:   ["직계 가족"]
        method:       "보안 메시지"
        content:      "부활 계획됨, 가족 상담 예정"
        responseTime: "7일"
      }

      step3: {
        recipients:   ["확장 네트워크"]
        method:       "이메일"
        content:      "동의서에 명시된 대로 부활 통지"
        responseTime: "30일"
      }
    }

[ID] 부활 준비 통지:

--------------------------------------------------------------------------------
8.4 부활 거부권
--------------------------------------------------------------------------------

[!] 대상자의 궁극적 권리:

    부활 조건이 충족되고 과정이 시작된 후에도,
    대상자는 부활을 거부할 권리를 유지합니다.

[ID] 거부 기간:

    RefusalWindow {
      windowStart:    "의식 회복 시"
      windowEnd:      "능력 확인 후 48시간"

      requirements: {
        mentalCapacity: "능력있음으로 평가되어야 함"
        counseling:     "필수 상담 세션"
        waitingPeriod:  "상담 후 24시간"
        witnessedDecision: true
      }
    }

[ID] 거부 프로세스:

    1. 능력 평가
       - 정신과 의사가 대상자 평가
       - 상황 이해 확인
       - 의사결정 능력 문서화

    2. 상담 세션
       - 현재 날짜와 상황 설명
       - 무엇이 변했는지 논의
       - 사용 가능한 옵션 검토
       - 질문과 우려 해결

    3. 대기 기간
       - 최소 24시간
       - 대상자가 마음을 바꿀 수 있음
       - 추가 상담 가능

    4. 증인된 결정
       - 대상자가 결정을 명확히 진술
       - 두 명의 증인 참석
       - 결정 기록됨

    5. 실행
       - 거부 시: 대안 지시에 따라 처리
       - 수락 시: 부활 프로세스 계속

8.5 부활 후 지원
--------------------------------------------------------------------------------

[ID] 재통합 지원:

    부활이 진행되면, 동의 문서는 지원 배치를 명시해야 합니다:

    ReintegrationSupport {
      medicalCare: {
        duration:     "최소 6개월"
        provider:     "WIA 의료 파트너"
        coverage:     "재정 배치에 따라"
      }

      psychologicalSupport: {
        counseling:   "1년간 매주"
        adjustment:   "특화된 시간 이동 치료"
      }

      socialIntegration: {
        familyReunification: true
        communityOrientation: true
        educationSupport:     "기술 및 사회 업데이트"
      }

      identityRestoration: {
        legalStatus:    "CRYO-LEGAL 표준에 따라"
        documentReissuance: true
        digitalIdentity: "CRYO-IDENTITY 표준에 따라"
      }
    }
