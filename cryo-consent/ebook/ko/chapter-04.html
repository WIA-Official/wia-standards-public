제4장: 부활 조건

[목차]

--------------------------------------------------------------------------------
4.1 조건 유형
--------------------------------------------------------------------------------

부활 조건은 보존이 종료되고 부활이 시작되어야 할 때를 지정합니다.

[ID] 조건 카테고리:

    MEDICAL      -> 특정 질병이 치료 가능/관리 가능해짐
    TECHNOLOGY   -> 필요한 기술적 능력이 존재
    TIME         -> 최소/최대 보존 기간
    SOCIAL       -> 가족 또는 후견인 승인 필요
    LEGAL        -> 법적 지위 요구사항
    FINANCIAL    -> 자금 가용성 확인
    CUSTOM       -> 사용자 정의 조건

[ID] 조건 기록:

    RevivalCondition {
      conditionId:    "COND-MED-001"
      conditionType:  "MEDICAL"
      description:    "암이 95% 이상의 성공률로 완치 가능해야 함"

      required:       true
      weight:         0.30  // 평가에서 30% 가중치

      parameters: {
        targetCondition:   "암"
        requiredOutcome:   "CURABLE"
        evidenceRequired:  ["동료 검토 연구", "FDA 승인"]
        verifyingAuthority: "세계보건기구"
      }

      status:         "PENDING"
    }


--------------------------------------------------------------------------------
4.2 의료 트리거
--------------------------------------------------------------------------------

[ID] 의료 조건 매개변수:

    MedicalConditionParams {
      targetCondition:   "췌장암"
      requiredOutcome:   "CURABLE"  // CURABLE | TREATABLE | MANAGEABLE

      evidenceRequired: [
        "WHO 인증",
        "5년 생존율 90% 이상",
        "출판된 임상 시험"
      ]

      verifyingAuthority: "WIA 의료 위원회"
    }

[ID] 결과 정의:

    CURABLE (완치 가능):
    - 질병의 완전한 제거 가능
    - 재발 예상되지 않음
    - 완전한 건강 회복

    TREATABLE (치료 가능):
    - 질병을 통제할 수 있음
    - 기대 수명 크게 연장
    - 삶의 질 유지 가능

    MANAGEABLE (관리 가능):
    - 증상을 통제할 수 있음
    - 만성적이지만 살 수 있음
    - 지속적인 치료 필요

[ID] 예시 조건:

    +----------------------------------------------------------+
    | 상태                   | 트리거 수준    | 우선순위       |
    +----------------------------------------------------------+
    | 원래 말기 진단         | CURABLE        | 주요           |
    | 노화/노쇠              | REVERSIBLE     | 부차           |
    | 심장병                 | TREATABLE      | 부차           |
    | 알츠하이머             | MANAGEABLE     | 3차            |
    +----------------------------------------------------------+


--------------------------------------------------------------------------------
4.3 기술 요구사항
--------------------------------------------------------------------------------

[ID] 기술 조건 매개변수:

    TechnologyConditionParams {
      requiredCapability: "SUCCESSFUL_HUMAN_REVIVAL"
      minimumSuccessRate: 90  // 퍼센트
      minimumCases:       100 // 성공적인 부활 수

      additionalRequirements: [
        "완전한 인지 기능 회복",
        "유의미한 기억 상실 없음",
        "신체 이동성 회복"
      ]

      verifyingAuthority: "WIA 기술 표준 위원회"
    }

[ID] 기술 이정표:

    [  ] 성공적인 포유류 부활 (쥐, 토끼)
    [  ] 성공적인 대형 포유류 부활 (돼지, 영장류)
    [  ] 최초 인간 부활 시도
    [  ] 성공적인 인간 부활 (n=1)
    [  ] 재현 가능한 인간 부활 (n=10)
    [  ] 일상적인 인간 부활 (n=100+)

[i] 대상자는 어떤 이정표가 부활을 트리거하는지 지정할 수 있습니다.


--------------------------------------------------------------------------------
4.4 시간 및 사회적 조건
--------------------------------------------------------------------------------

[ID] 시간 조건:

    TimeConditionParams {
      minimumDuration: "P50Y"   // ISO 8601: 최소 50년
      maximumDuration: "P500Y"  // 최대 500년

      referencePoint: "PRESERVATION_START"
      // 옵션: PRESERVATION_START | LEGAL_DEATH | FACILITY_REGISTRATION
    }

[ID] 시간 조건 근거:

    최소 기간:
    - 기술이 발전할 시간 보장
    - 조기 부활 시도 방지
    - 상당한 의학 발전 허용

    최대 기간:
    - 무기한 보존 제한
    - 결정 시점 강제
    - 사회적/법적 의미 고려

[ID] 사회적 조건:

    SocialConditionParams {
      requiredApprovals: [
        { guardianId: "GUARD-001", weight: 2 },
        { guardianId: "GUARD-002", weight: 1 },
        { guardianId: "GUARD-003", weight: 1 }
      ]

      familyNotification:   true
      publicAnnouncement:   false
      supportSystemVerified: true
    }


--------------------------------------------------------------------------------
4.5 조건 로직
--------------------------------------------------------------------------------

[ID] 로직 옵션:

    ALL:    모든 조건이 충족되어야 함
    ANY:    어떤 단일 조건이 부활을 트리거
    CUSTOM: 불린 표현식 평가

[ID] 사용자 정의 로직 예시:

    단순:
    "(MEDICAL_CANCER AND TECHNOLOGY_REVIVAL)"

    복잡:
    "((MEDICAL_CANCER OR MEDICAL_AGING) AND TECHNOLOGY_REVIVAL AND TIME_MIN)"

[ID] 평가 프로세스:

    +------------------+
    | 증거 수집        |
    | - 의료 상태      |
    | - 기술 진행      |
    | - 경과 시간      |
    +--------+---------+
             |
             v
    +------------------+
    | 각 조건          |
    | 평가             |
    +--------+---------+
             |
             v
    +------------------+
    | 로직 적용        |
    | (ALL/ANY/CUSTOM) |
    +--------+---------+
             |
        +----+----+
        |         |
      통과       실패
        |         |
        v         v
    [부활승인] [보존계속]


--------------------------------------------------------------------------------
장 요약
--------------------------------------------------------------------------------

핵심 개념:
  [OK] 여섯 가지 조건 유형: 의료, 기술, 시간, 사회, 법적, 재정
  [OK] 의료 트리거는 질병과 필요한 결과 수준 지정
  [OK] 기술 조건은 검증된 부활 능력 요구
  [OK] 시간 조건은 최소/최대 보존 기간 설정
  [OK] 로직 옵션: ALL, ANY, 또는 CUSTOM 불린 표현식
