# 6장: 프로토콜 사양

## 학습 목표

이 장을 마치면 다음을 수행할 수 있습니다:

- 표준 고객 실사(CDD) 프로토콜의 6단계 프로세스 이해
- 강화된 실사(EDD)가 필요한 시기 결정
- 제재, PEP, 부정적 미디어 스크리닝 수행
- 다차원 위험 평가 프레임워크 적용
- 거래 모니터링 규칙 및 시나리오 구현
- 정기적인 고객 검토 프로토콜 실행
- 의심스러운 활동 보고서(SAR) 준비 및 제출

---

## 개요

이 장에서는 고객 실사(CDD), 강화된 실사(EDD), 스크리닝 워크플로 및 거래 모니터링 규칙을 포함하여 KYC/AML 규정 준수 절차를 구현하기 위한 운영 프로토콜을 자세히 설명합니다.

---

## 고객 실사 (CDD) 프로토콜

### 표준 CDD 워크플로

#### 적용 대상

- 저위험에서 중간 위험 고객
- 표준 금융 제품
- 일반적인 거래 패턴

#### 프로세스 단계

```
1단계: 고객 정보 수집
    ↓
2단계: 신원 확인
    ↓
3단계: 스크리닝 (제재, PEP)
    ↓
4단계: 위험 평가
    ↓
5단계: 승인 결정
    ↓
6단계: 지속적 모니터링
```

---

### 1단계: 고객 정보 수집

#### 개인 고객 - 필수 정보

| 범주 | 필수 필드 | 선택 필드 |
|------|-----------|-----------|
| **개인 정보** | 이름, 성, 생년월일, 국적 | 중간 이름, 이전 이름, 출생지 |
| **신원 문서** | 문서 유형, 문서 번호, 발행/만료 날짜, 발행 국가 | 발행지 |
| **연락처** | 이메일, 전화, 거주 주소 | 우편 주소 (다를 경우) |
| **재무** | 자금 출처, 예상 계정 활동 | 부의 출처, 순자산 추정 |
| **고용** | 직업 | 고용주 이름, 산업 |

#### 법인 고객 (KYB) - 필수 정보

| 범주 | 필수 필드 | 선택 필드 |
|------|-----------|-----------|
| **회사 정보** | 법적 이름, 등록 번호, 등록 국가, 설립 날짜 | 상호명, 이전 이름 |
| **사업 활동** | 산업, 사업 설명 | 연간 수익, 직원 수 |
| **소유권** | 실소유자 (>25% 소유권), 최종 실소유자 | 중간 실체 |
| **통제** | 이사, 승인된 서명자 | 이사회 구성원 |
| **문서** | 설립 증명서, 등록 주소 증명 | 정관, 양호한 상태 증명서 |

#### 데이터 품질 표준

- ✅ 모든 필수 필드가 완전해야 함
- ✅ 데이터 형식은 WIA 스키마를 준수해야 함
- ✅ 필드 간 일관성 (예: 나이가 생년월일과 일치)
- ✅ 자리 표시자 또는 더미 값 없음
- ✅ 주소는 완전해야 함 (도시/국가만이 아님)

### 2단계: 신원 확인

#### 개인 고객

**수준 1: 기본 확인 (저위험)**

```
필수:
- 이메일 확인 (OTP)
- 전화 확인 (SMS OTP)
- 데이터베이스 교차 확인 (신용 조사 기관 또는 동등)

승인 기준:
- 이메일 전달 및 OTP 검증
- 전화 활성 및 OTP 검증
- 데이터베이스에서 3개 중 최소 2개 일치: 이름, 생년월일, 주소
```

**수준 2: 표준 확인 (중간 위험)**

```
필수:
- 수준 1 요구 사항 +
- 정부 발급 ID 문서
  * 문서 진위 확인
  * 데이터 추출 및 검증
  * 만료 확인 (>3개월 남음)
- 주소 확인
  * 공과금 청구서, 은행 명세서 또는 정부 문서
  * 최근 3개월 이내 발행
  * 이름 및 주소 일치

승인 기준:
- 문서가 진위 확인 통과 (신뢰도 >85%)
- 추출된 데이터가 제공된 정보와 일치 (>95% 일치)
- 주소 문서 확인됨
```

**수준 3: 강화된 확인 (고위험)**

```
필수:
- 수준 2 요구 사항 +
- 생체 인식 확인
  * 활성 감지 (능동 또는 수동)
  * 문서와 라이브 셀카 간 얼굴 매칭
  * 일치 신뢰도 >85%
- 비디오 확인 또는 대면 확인 (최고 위험의 경우)

승인 기준:
- 활성 확인 통과
- 얼굴 일치 신뢰도 >85%
- 필요한 경우 비디오/대면 확인 완료
```

#### 법인 고객 (KYB)

```
회사 확인:
1. 사업자 등록 확인
   - 회사가 존재하고 활동 중임을 확인
   - 등록 세부 정보 확인
   - 해산 또는 폐업 여부 확인

2. 문서 확인
   - 설립 증명서 진위
   - 정관 제공
   - 양호한 상태 증명서 (가능한 경우)

3. 주소 확인
   - 등록 주소 확인
   - 운영 주소 확인

4. 소유권 확인
   - 실소유자 식별 (>25% 임계값)
   - 각 실소유자의 신원 확인 (개인 CDD 사용)
   - 소유권 구조 문서화

5. 통제 인물 확인
   - 이사 및 승인된 서명자 식별
   - 각 사람의 신원 확인
   - 권한 및 권력 확인
```

### 3단계: 스크리닝

#### 제재 스크리닝

```
프로세스:
1. 스크리닝 데이터 추출:
   - 전체 이름
   - 생년월일
   - 국적
   - 신원 문서 번호
   - 주소

2. 목록에 대해 스크리닝:
   - OFAC SDN (미국)
   - OFAC Non-SDN
   - UN 안전보장이사회 통합 목록
   - EU 제재 목록
   - UK HM Treasury 금융 제재
   - 관할권별 목록

3. 퍼지 매칭:
   - 이름 변형 (80%+ 일치 임계값)
   - 음역
   - 별칭 및 닉네임
   - 생년월일 허용 오차 (검토용 ±1년)

4. 결과 평가:
   일치 점수      조치
   100%          하드 스톱 - 수동 검토 필요
   85-99%        소프트 스톱 - 수동 검토 필요
   70-84%        검토 대기열
   <70%          클리어 - 진행

5. 거짓 양성 검토:
   - 생년월일 비교 (정확한 일치 = 참 일치 가능성 높음)
   - 국적 비교
   - 주소 비교
   - 별칭 확인
   - 추가 식별자 검토
```

#### PEP 스크리닝

```
프로세스:
1. PEP 데이터베이스에 대해 스크리닝
   - 정부 관리 (모든 수준)
   - 국영 기업 임원
   - 정당 관계자
   - 가족 구성원 및 측근

2. PEP 등급 분류:
   등급 1: 국가 원수, 고위 정치인, 고위 정부 관료
   등급 2: 중급 정부 관료, 군 장교
   등급 3: 가족 구성원, 측근 사업 관계자

3. 일치 검토:
   - 정확한/높은 신뢰도 일치 → PEP로 분류
   - 중간 신뢰도 → 수동 검토
   - 낮은 신뢰도 → 클리어

4. PEP 확인되면:
   - 강화된 실사 (EDD)로 승격
   - PEP 상태 및 등급 문서화
   - 온보딩을 위한 선임 승인 획득
```

#### 부정적 미디어 스크리닝

```
프로세스:
1. 부정적 뉴스 소스 검색 (지난 5년)
   범주:
   - 금융 범죄 (자금 세탁, 사기)
   - 부패
   - 테러리즘
   - 조직 범죄
   - 규제 집행 조치
   - 형사 유죄 판결

2. 관련성 평가:
   - 신원 일치 확인 (동명이인이 아님)
   - 심각성 및 최신성 평가
   - 위험 등급에 미치는 영향 결정

3. 발견 사항에 따른 조치:
   - 주요 부정적 (예: 유죄 판결): 거부 또는 EDD + 선임 승인
   - 경미한 부정적 (예: 입증되지 않은 혐의): 문서화 및 CDD 계속
   - 부정적 없음: 진행
```

### 4단계: 위험 평가

#### 위험 점수 프로세스

```
1. 지리적 위험 평가:
   입력:
   - 고객 거주 국가
   - 시민권/국적
   - 사업 운영 국가
   - 예상 거래 국가

   점수:
   - 저위험 관할권 (FATF 준수): 0-20
   - 중간 위험: 21-50
   - 고위험 (FATF 고위험 목록): 51-80
   - 금지 (제재 국가): 81-100

2. 제품/서비스 위험 평가:
   입력:
   - 요청된 계정 유형
   - 활성화된 서비스 (국제 송금, 현금 등)
   - 예상 거래량

   점수:
   - 기본 계정 (국제 없음): 0-20
   - 국제 송금이 있는 표준: 21-40
   - 고가치 또는 복잡한 제품: 41-70
   - 고위험 서비스 (예: 코레스 뱅킹): 71-100

3. 고객 유형 위험 평가:
   입력:
   - 개인 대 사업
   - 직업 / 산업
   - 공개 노출 (PEP 상태)

   점수:
   - 개인, 고용된 전문가, PEP 아님: 0-20
   - 개인, 자영업 또는 사업: 21-40
   - 중소기업, 합법적 산업: 21-40
   - 고위험 산업 (MSB, 암호화폐, 게임): 41-70
   - PEP: +20점
   - 쉘 회사 지표: 71-100

4. 행동 위험 평가:
   입력:
   - 자금 출처 명확성
   - 거래 목적 근거
   - 프로필과의 일관성

   점수:
   - 명확하고 잘 문서화된 출처: 0-20
   - 부분적으로 문서화됨: 21-40
   - 모호하거나 일관성 없는 설명: 41-70
   - 설명하거나 확인할 수 없음: 71-100

5. 관계 위험 평가:
   입력:
   - 알려진 관계자
   - 관련 실체
   - 스크리닝 결과

   점수:
   - 부정적 발견 없음: 0-20
   - 경미한 부정적 미디어: 21-40
   - 고위험 관계: 41-70
   - 제재 일치 또는 주요 범죄 이력: 71-100

6. 전체 위험 점수 계산:
   가중 평균:
   전체 점수 = (지리적 × 0.25) +
                (제품 × 0.20) +
                (고객 유형 × 0.20) +
                (행동 × 0.20) +
                (관계 × 0.15)

7. 위험 범주 할당:
   점수 0-39:    저위험
   점수 40-69:   중간 위험
   점수 70-89:   고위험
   점수 90-100:  금지
```

### 5단계: 승인 결정

#### 결정 매트릭스

| 위험 범주 | 스크리닝 결과 | 신원 확인 | 결정 |
|----------|--------------|----------|------|
| **낮음** | 클리어 | 통과 | 자동 승인 |
| **낮음** | PEP/부정적 | 통과 | 수동 검토 → 승인/거부 |
| **중간** | 클리어 | 통과 | 자동 승인 또는 수동 검토* |
| **중간** | PEP/부정적 | 통과 | 수동 검토 → 승인/거부 |
| **높음** | 클리어 | 통과 | 선임 승인 필요 |
| **높음** | PEP | 통과 | 선임 승인 + EDD 필요 |
| **높음** | 제재 일치 | 모두 | 거부 |
| **금지** | 모두 | 모두 | 자동 거부 |
| 모두 | 모두 | 실패 | 거부 또는 재확인 요청 |

*기관이 구성 가능한 수동 검토 임계값

#### 승인 권한

| 위험 범주 | 권한 | 필요한 문서 |
|----------|------|-------------|
| **낮음** | 시스템 자동 승인 | 표준 CDD 체크리스트 |
| **중간** | 규정 준수 분석가 | CDD 체크리스트 + 위험 평가 |
| **높음** | 규정 준수 관리자 | EDD 문서 + 선임 승인 |

#### 모니터링 요구 사항

| **높음** | 강화된 규칙 + 수동 감독 | 분기별 | 일일 (자동) |

## 강화된 실사 (EDD) 프로토콜

### EDD가 필요한 경우

#### 필수 EDD 트리거

1. ✅ 고객이 PEP (모든 등급)
2. ✅ 고위험 국가 관여
3. ✅ 고위험 산업 (MSB, 암호화폐 거래소, 카지노 등)
4. ✅ 복잡한 소유 구조
5. ✅ 높은 거래량 (기관이 정의한 >임계값)
6. ✅ 부정적 미디어 발견 (심각한 혐의)
7. ✅ 비정상적이거나 의심스러운 활동 감지
8. ✅ 선임 관리 또는 규제 지시

### EDD 추가 요구 사항

#### 표준 CDD 이상

```
1. 부의 출처 (자금 출처만이 아님)
   필수:
   - 부 축적에 대한 자세한 설명
   - 지원 문서 (세금 신고서, 사업 기록, 상속 문서)
   - 주요 부의 출처 확인

2. 계정/관계 목적
   필수:
   - 자세한 사업 계획 또는 계정 사용 설명
   - 특정 제품/서비스에 대한 근거
   - 관련된 관할권에 대한 설명

3. 강화된 신원 확인
   필수:
   - 비디오 확인 또는 대면 미팅
   - 추가 문서 교차 확인
   - 고용/사업의 독립적 확인

4. 실소유자 심층 조사 (실체의 경우)
   필수:
   - 자연인에 대한 전체 소유 체인
   - 명의인 구조 조사
   - 각 계층 확인
   - 통제 메커니즘 문서화

5. 독립적 정보 출처
   필수:
   - 공공 기록 검색
   - 인터넷/미디어 검색
   - 법인 등록부 확인
   - 제3자 참조 확인

6. 선임 관리 승인
   필수:
   - 규정 준수 관리자 검토
   - 위험 위원회 승인 (최고 위험의 경우)
   - 승인 근거 문서화

7. 강화된 모니터링
   필수:
   - 낮은 거래 모니터링 임계값
   - 대규모/비정상 거래의 수동 검토
   - 분기별 계정 검토
   - 일일 자동 스크리닝 업데이트
```

### EDD 문서 체크리스트

#### PEP 고객의 경우

```
☐ PEP 상태 확인 및 문서화
☐ 등급 분류 할당
☐ 부의 출처 문서화 및 확인
☐ 현재 직위 및 책임 문서화
☐ 관계 목적 명확하게 명시
☐ 선임 승인 획득
☐ 강화된 모니터링 규칙 구성
☐ 분기별 검토 일정 설정
```

#### 고위험 관할권 고객의 경우

```
☐ 관할권 관여 이유 문서화
☐ 추가 신원 확인 완료
☐ 고위험 국가로부터의 자금 출처 설명 및 확인
☐ 고위험 관할권과의 거래 목적 문서화
☐ 제재 준수 확인
☐ 강화된 거래 모니터링 활성화
```

#### 고가치/복잡한 실체의 경우

```
☐ 완전한 소유 구조 매핑
☐ 모든 실소유자 (>25%) 식별 및 확인
☐ 통제 인물 식별 및 확인
☐ 사업 모델 및 수익원 확인
☐ 주요 고객/공급업체 식별
☐ 재무제표 검토
☐ 세금 준수 확인
☐ 라이선스 및 등록 확인
```

## 지속적 모니터링 프로토콜

### 거래 모니터링 규칙

#### 구조화 / 스머핑 탐지

```
규칙: 보고 임계값 바로 아래의 여러 거래

구성:
- 임계값: $10,000 (CTR 임계값) 또는 기관 정의
- 조회 기간: 30일
- 트리거: 임계값의 10% 이내 3개 이상 거래

경보 논리:
IF (SUM(예금, 30일) > 임계값 × 3) AND
   (각 예금 BETWEEN 임계값 × 0.90 AND 임계값 × 0.99) AND
   (COUNT(예금) >= 3)
THEN generate_alert(severity: HIGH)

예제:
예금: $9,500, $9,800, $9,900 (30일 동안)
합계: $29,200 (>$30,000)
각: $10,000의 90-99%
개수: 3
결과: 경보
```

#### 빠른 자금 이동

```
규칙: 큰 예금 후 빠른 인출

구성:
- 최소 금액: $5,000 또는 구성 가능
- 시간 창: 24-48시간
- 인출 임계값: 예금의 90%

경보 논리:
IF (credit_amount > minimum) AND
   (debit_within_hours < time_window) AND
   (debit_amount > credit_amount × 0.90)
THEN generate_alert(severity: HIGH)

예제:
크레딧: $50,000 (1일 오전 10시)
데빗: $48,000 (1일 오후 2시)
시간: 4시간 (<24)
백분율: 96% (>90%)
결과: 경보
```

#### 지리적 위험 신호

```
규칙: 고위험 국가와 관련된 거래

구성:
- 고위험 국가 목록: FATF 고위험 관할권 + 기관 정의
- 최소 금액: $1,000 또는 구성 가능
- PEP 고객에 대한 위험 배율

경보 논리:
IF (transaction_country IN high_risk_list) AND
   (amount > minimum)
THEN generate_alert(severity: MEDIUM to HIGH based on country)

고객이 PEP인 경우:
   severity = HIGH

예제:
국가 X로 송금 (FATF 고위험 목록에 있음)
금액: $5,000
고객: PEP 아님
결과: 경보 (severity: MEDIUM)
```

#### 거래량 이상 탐지

```
규칙: 프로필과 비교하여 비정상적인 거래량

구성:
- 기준선: 지난 6개월 동안의 평균 월별 활동
- 임계값: 기준선의 200%
- 경보 최소: $10,000 절대 증가

경보 논리:
IF (current_month_volume > baseline × 2.0) AND
   (current_month_volume - baseline > 10000)
THEN generate_alert(severity: MEDIUM)

예제:
기준선: 월 평균 $15,000
현재 월: $35,000 (월 중순 기준)
예상: $70,000
증가: $55,000 (>$10,000)
배수: 4.7배 (>2배)
결과: 경보
```

#### 라운드 트리핑 탐지

```
규칙: 자금의 순환 이동

구성:
- 조회 기간: 90일
- 최소 금액: $5,000
- 일치 허용 오차: 원래 금액의 95%

경보 논리:
거래 체인 식별:
A → B → C → ... → A
WHERE (final_amount > original_amount × 0.95) AND
      (original_amount > minimum)
THEN generate_alert(severity: HIGH)

예제:
계정 A가 계정 B로 $10,000 송금
계정 B가 계정 C로 $9,800 송금
계정 C가 계정 A로 $9,600 송금
결과: 경보 (라운드 트립 탐지)
```

### 정기 검토 프로토콜

#### 저위험 고객 (2년마다 검토)

```
검토 체크리스트:
☐ 고객 연락처 정보 최신 확인
☐ 제재, PEP 목록에 대해 재스크리닝
☐ 프로필과 일치하는 거래 활동 검토
☐ 위험 점수 업데이트
☐ 부정적 미디어 없음 확인
☐ 검토 완료 문서화

조치:
- 변경 없으면: 저위험 분류 계속
- 위험 지표 변경되면: 위험 범주 재평가
```

#### 중간 위험 고객 (연간 검토)

```
검토 체크리스트:
☐ 저위험 검토의 모든 항목 +
☐ 자금/부의 출처 검토 (변경 사항?)
☐ 고용/사업 상태 확인
☐ 거래 패턴 세부 검토
☐ 비정상적인 활동 평가 (경보 임계값 이하라도)
☐ 고객 프로필 업데이트
☐ 위험 점수 재평가
☐ 검토 및 발견 사항 문서화

조치:
- 프로필 변경 없고 활동 일관성 있으면: 중간 위험 계속
- 위험 증가하면: 고위험으로 승격, EDD 고려
- 위험 감소하면: 저위험으로 다운그레이드 고려
```

#### 고위험 고객 (분기별 검토)

```
검토 체크리스트:
☐ 중간 위험 검토의 모든 항목 +
☐ 대규모 거래 심층 분석 (경보되지 않아도)
☐ 모든 모니터링 경보 검토 (종료 및 열림)
☐ 해당되는 경우 PEP 상태 확인
☐ 새로운 부정적 미디어 확인
☐ 부의 출처 문서 검토
☐ 계속되는 사업 근거 평가
☐ EDD 문서 업데이트
☐ 관계에 대한 선임 관리 검토
☐ 결정: 계속, 모니터링 강화 또는 종료

조치:
- 근거와 함께 결정 문서화
- 종료하는 경우: 위험 감소 프로토콜 따름
```

## 의심스러운 활동 보고 (SAR) 프로토콜

### SAR 제출 시기

#### 필수 제출 트리거

1. **알려진 또는 의심되는 범죄 활동과 관련된 임계값 이상의 거래**
   - 미국: 거래 >$5,000
   - 관할권에 따라 다름

2. **BSA 요구 사항을 회피하도록 설계된 거래**
   - 구조화
   - 허위 진술
   - 명의인 계정 사용

3. **명백한 합법적 목적이 없는 거래**
   - 경제적 근거 없음
   - 비정상적인 복잡성
   - 고객이 설명할 수 없거나 하지 않음

4. **컴퓨터 침입 또는 도난**
   - 무단 접근
   - 계정 탈취

5. **자금 세탁을 나타내는 패턴**
   - 계층화 체계
   - 불법 자금 통합

### SAR 제출 프로세스

```
1단계: 경보/의심스러운 활동 식별
    ↓
2단계: 예비 조사 (24-48시간)
    ↓
3단계: 전체 조사 (5-10 영업일)
    ↓
4단계: SAR 결정
    ├─ SAR 없음: 결정 문서화, 사례 종료
    └─ SAR 제출: 5단계로 진행
        ↓
5단계: SAR 준비
    ↓
6단계: 감독자 검토
    ↓
7단계: SAR 제출 (탐지 후 30일 이내)
    ↓
8단계: 법 집행 기관 통지 (긴급한 경우)
    ↓
9단계: 계속 모니터링 (고객 통지 없음)
```

### SAR 서술 요구 사항

#### 포함해야 함

1. **누구**: 완전한 식별 정보가 있는 SAR의 대상
2. **무엇**: 의심스러운 활동에 대한 설명
3. **언제**: 활동의 날짜 및 타임라인
4. **어디**: 관련된 위치
5. **왜**: 의심의 이유 (무엇이 비정상적/의심스러운지)
6. **어떻게**: 활동을 수행하는 데 사용된 방법

#### 서술 구조

```
서론:
- 의심스러운 활동의 간단한 요약
- 대상 식별
- 관련된 총액

배경:
- 고객 관계 이력
- 정상적인 계정 활동 패턴
- 의심스러운 활동이 시작된 시기

의심스러운 활동 세부 정보:
- 거래/행동의 시간순 설명
- 관찰된 특정 위험 신호
- 예상 활동과의 편차

조사:
- 조사를 위해 취한 단계
- 고객 접촉 (있는 경우) 및 응답
- 수집된 외부 정보

분석:
- 활동이 의심스러운 것으로 간주되는 이유
- 일치하는 유형론 또는 패턴
- 의심되는 가능한 범죄 활동

결론:
- SAR 제출 근거 요약
- 지속적인 모니터링 계획
```

### SAR 제출 타임라인

```
0일: 의심스러운 활동 탐지
    ↓
1-2일: 예비 검토
    ↓
3-10일: 전체 조사
    ↓
11-15일: SAR 준비 및 검토
    ↓
30일: 제출 마감일 (최대)
```

**참고**: 대상의 신원을 알 수 없는 경우 60일로 연장하되, 연장 이유를 문서화해야 함.
