## 주요 용어

### 핵심 개념

| 용어 | 정의 |
|------|------|
| **KYC** | Know Your Customer (고객 확인) - 고객 신원을 확인하고 위험을 평가하는 프로세스 |
| **AML** | Anti-Money Laundering (자금 세탁 방지) - 범죄 수익이 합법적인 자금으로 위장되는 것을 방지하기 위한 조치 |
| **CDD** | Customer Due Diligence (고객 실사) - 표준 수준의 확인 및 위험 평가 |
| **EDD** | Enhanced Due Diligence (강화된 실사) - 고위험 고객에 대한 더 높은 수준의 조사 |
| **PEP** | Politically Exposed Person (정치적 주요 인물) - 주요 공직에 있는 개인 (더 높은 위험) |
| **SAR** | Suspicious Activity Report (의심스러운 활동 보고) - 의심스러운 거래가 감지될 때 제출되는 보고서 |
| **CTR** | Currency Transaction Report (통화 거래 보고) - 대규모 현금 거래에 대한 보고 (일반적으로 >$10,000) |
| **FATF** | Financial Action Task Force (자금세탁방지 국제기구) - AML 표준을 설정하는 국제 기구 |

### 신원 확인

| 용어 | 정의 |
|------|------|
| **IDV** | Identity Verification (신원 확인) - 개인의 주장된 신원을 확인하는 프로세스 |
| **생체 인증** | 식별에 사용되는 생물학적 특성 (지문, 얼굴, 홍채) |
| **활성 감지** | 생체 샘플이 사진/비디오가 아닌 살아있는 사람으로부터 나온 것임을 확인하는 기술 |
| **문서 확인** | 정부 발급 신분증 문서의 인증 |
| **KBA** | Knowledge-Based Authentication (지식 기반 인증) - 개인 정보 질문을 사용한 확인 |

### 위험 평가

| 용어 | 정의 |
|------|------|
| **위험 점수** | 고객의 위험 수준을 나타내는 숫자 값 (일반적으로 0-100) |
| **위험 범주** | 분류: 낮음, 중간, 높음 또는 금지 |
| **위험 요소** | 평가 시 고려되는 요소: 지리, 직업, 거래 패턴 |
| **위험 선호도** | 조직이 특정 위험 수준을 받아들이려는 의지 |
| **위험 완화** | 식별된 위험을 줄이기 위해 취한 조치 |

### 거래 모니터링

| 용어 | 정의 |
|------|------|
| **TM** | Transaction Monitoring (거래 모니터링) - 고객 거래의 지속적인 감시 |
| **임계값** | 초과 시 경보를 트리거하는 미리 정의된 한계 |
| **경보** | 의심스러운 활동이 감지될 때 생성되는 알림 |
| **거짓 양성** | 합법적인 활동에 대해 트리거된 경보 (이를 줄이는 것이 핵심 효율성 목표) |
| **사례 관리** | 경보를 조사하고 해결하는 프로세스 |
| **유형론** | 자금 세탁에 일반적으로 사용되는 패턴 또는 방법 |

### 스크리닝

| 용어 | 정의 |
|------|------|
| **제재 목록** | 금지된 개인/단체의 정부 목록 (OFAC, UN, EU 등) |
| **감시 목록** | 추가 조사가 필요한 개인/단체의 데이터베이스 |
| **부정적 미디어** | 잠재적 위험을 나타내는 부정적인 뉴스 보도 |
| **퍼지 매칭** | 변형을 잡기 위해 대략적인 이름 일치를 허용하는 알고리즘 |
| **히트** | 스크리닝 프로세스 중 발견된 일치 항목 |

### 규제 준수

| 용어 | 정의 |
|------|------|
| **RegTech** | Regulatory Technology (규제 기술) - 규정 준수를 위해 기술을 사용하는 솔루션 |
| **BSA** | Bank Secrecy Act (은행 비밀법, 미국) - 금융 기관이 자금 세탁 탐지를 지원하도록 요구 |
| **FinCEN** | Financial Crimes Enforcement Network (미국 규제 기관) |
| **GDPR** | General Data Protection Regulation (EU) - 고객 데이터 처리 방식에 영향 |
| **KYB** | Know Your Business (사업체 확인) - 기업 고객에 대한 실사 |
| **UBO** | Ultimate Beneficial Owner (최종 실소유자) - 궁극적으로 법인을 소유/통제하는 자연인 |

## 표준 구성 요소

WIA KYC/AML 표준은 다음으로 구성됩니다:

1. **데이터 스키마** - 고객 데이터, 문서 및 위험 평가를 위한 표준화된 JSON 형식
2. **API 사양** - 신원 확인, 스크리닝 및 모니터링을 위한 RESTful 인터페이스
3. **프로토콜 지침** - CDD, EDD 및 지속적 모니터링을 위한 단계별 절차
4. **통합 프레임워크** - 은행 시스템 및 제3자 서비스와 연결하기 위한 패턴
5. **참조 구현** - TypeScript SDK 및 샘플 애플리케이션
