수학적 기초

HE 뒤의 수학 이해하기

동형 암호는 마법처럼 보일 수 있지만 엄격한 수학에 기반을 두고 있습니다. 이러한 기초를 이해하는 것은 HE 시스템을 적절하게 구현, 최적화 및 보안하는 데 필수적입니다. 이 장에서는 기본 대수학에서 고급 격자 이론까지 동형 암호를 가능하게 하는 수학적 구조를 탐구합니다.

수학자가 아니어도 걱정하지 마십시오—개념을 점진적으로 구축하고 공식 정의와 함께 직관을 제공할 것입니다. 끝까지 마치면 HE가 왜 안전한지, 연산이 어떻게 작동하는지, 매개변수가 무엇을 의미하는지 이해하게 될 것입니다.

대수 구조: 기초

군, 환, 체

동형 암호는 숫자가 상호 작용하는 방식을 정의하는 대수 구조 내에서 작동합니다:

군(Group)

다음을 만족하는 하나의 연산(덧셈 등)이 있는 집합:

  • 닫힘성: a + b는 군에 속함
  • 결합법칙: (a + b) + c = a + (b + c)
  • 항등원: a + 0 = a인 0이 존재
  • 역원: 각 a에 대해 a + (-a) = 0인 -a가 존재

예: 덧셈 하의 정수 (ℤ, +)

환(Ring)

다음과 같은 두 개의 연산(덧셈과 곱셈)이 있는 집합:

  • 덧셈 하에서 군을 형성
  • 곱셈은 결합법칙과 덧셈에 대한 분배법칙을 만족
  • 곱셈 항등원(1)을 가짐

예: 정수 (ℤ, +, ×)

체(Field)

모든 0이 아닌 원소가 곱셈 역원을 가지는 환.

예: 유리수 (ℚ), 실수 (ℝ)

모듈러 산술

HE 방식은 모듈러 산술—"순환이 있는" 산술을 사용하여 유한 구조에서 작동합니다:

a ≡ b (mod n)은 n이 (a - b)를 나눈다는 것을 의미

mod 7을 사용한 예:

8 ≡ 1 (mod 7)     왜냐하면 8 - 1 = 7
15 ≡ 1 (mod 7)    왜냐하면 15 - 1 = 14 = 2 × 7
3 + 5 ≡ 1 (mod 7) 왜냐하면 8 ≡ 1 (mod 7)

모듈러 산술은 ℤn = {0, 1, 2, ..., n-1}과 같은 유한환을 mod n 덧셈 및 곱셈으로 만듭니다.

다항식 환: 핵심 구조

다항식 환이란?

현대 HE 방식은 단일 숫자를 암호화하지 않습니다—다항식을 암호화합니다. 이는 SIMD 스타일 배칭을 통해 엄청난 효율성 향상을 제공합니다.

R = ℤ[x] / (f(x))

이 표기법은 정수 계수를 가진 다항식을 의미하며, f(x) = 0입니다. HE의 경우 일반적으로 다음을 사용합니다:

R = ℤ[x] / (xn + 1)

여기서 n은 2의 거듭제곱입니다(예: 4096, 8192, 16384).

다항식 연산

n = 4인 이 환에서 다항식이 어떻게 작동하는지 살펴봅시다:

R = ℤ[x] / (x⁴ + 1)

R의 다항식:
a(x) = 5 + 3x + 7x² + 2x³
b(x) = 1 + 4x + 6x² + 8x³

덧셈 (계수별):
a(x) + b(x) = 6 + 7x + 13x² + 10x³

곱셈 (x⁴ + 1에 의한 축약과 함께):
1단계: 정상적으로 곱하기
a(x) × b(x) = 5 + 23x + 61x² + 96x³ + 74x⁴ + 86x⁵ + 68x⁶ + 16x⁷

2단계: x⁴ = -1을 사용하여 축약:
x⁴ → -1
x⁵ → -x
x⁶ → -x²
x⁷ → -x³

결과: (5 - 74) + (23 - 86)x + (61 - 68)x² + (96 - 16)x³
      = -69 + (-63)x + (-7)x² + 80x³

왜 이 구조인가?

다항식 환 R = ℤ[x]/(xn + 1)은 특별한 속성을 가지고 있습니다:

  • 배칭: 단일 다항식이 중국인의 나머지 정리를 사용하여 n/2개의 값을 인코딩할 수 있음
  • FFT 친화적: 다항식 곱셈이 고속 푸리에 변환을 사용하여 O(n²) 대신 O(n log n)이 됨
  • 보안: 구조가 어려운 격자 문제(RLWE)를 지원
  • 동형성: 다항식에 대한 연산이 인코딩된 값에 대한 연산으로 변환됨

Ring Learning With Errors (RLWE)

보안 기반

현대 HE 방식은 Ring Learning With Errors 문제—양자 컴퓨터에게도 어려운 문제에 보안을 기반으로 합니다:

RLWE 문제

주어진: 쌍 (ai, bi) 여기서:

  • ai ∈ Rq는 균일하게 무작위
  • bi = ai · s + ei (mod q)
  • s ∈ Rq는 비밀 다항식
  • ei는 가우시안 분포에서 나온 작은 오류

찾기: 비밀 다항식 s

이 문제는 최악의 경우 격자 문제만큼 어렵다고 믿어지며 강력한 보안 보장을 제공합니다.

오류 분포

"오류" e는 표준 편차 σ를 가진 이산 가우시안 분포 χ에서 샘플링됩니다:

P(e = k) ∝ exp(-k²/2σ²)

일반적인 값: σ = 3.2, 대부분 [-10, 10] 범위의 오류를 제공합니다.

표준 편차 (σ)일반적인 오류 범위보안 영향
3.2[-10, 10]표준 보안
6.4[-20, 20]더 높은 보안
1.6[-5, 5]더 낮은 보안 (더 빠름)

BFV 방식: 상세한 수학

매개변수 설정

BFV (Brakerski-Fan-Vercauteren) 방식은 다음 매개변수로 작동합니다:

  • n: 다항식 차수 (2의 거듭제곱, 예: 8192)
  • q: 암호문 모듈러스 (큰 정수, 예: 2²¹⁸)
  • t: 평문 모듈러스 (더 작은 정수, 예: 1024)
  • χ: 오류 분포 (이산 가우시안, σ = 3.2)
  • Δ = ⌊q/t⌋: 스케일링 계수

키 생성

1. 비밀 키 샘플링:
   s ← χ (작은 계수를 가진 다항식)

2. 무작위 다항식 샘플링:
   a ← R_q (균일하게 무작위)

3. 오류 샘플링:
   e ← χ

4. 공개 키 원소 계산:
   b = -a·s - e (mod q)

5. 공개 키: pk = (b, a)
   비밀 키: sk = s

암호화

메시지 m ∈ Rt를 암호화하려면:

1. 작은 무작위 다항식 샘플링:
   r ← χ

2. 작은 오류 샘플링:
   e₀, e₁ ← χ

3. 암호문 구성 요소 계산:
   c₀ = b·r + e₀ + Δ·m (mod q)
   c₁ = a·r + e₁ (mod q)

4. 출력 암호문: ct = (c₀, c₁)

복호화

1. 비밀 키로 계산:
   noisy = c₀ + c₁·s (mod q)

2. 무슨 일이 일어나는지 관찰:
   noisy = (b·r + e₀ + Δ·m) + (a·r + e₁)·s
         = (-a·s - e)·r + e₀ + Δ·m + a·r·s + e₁·s
         = -a·r·s - e·r + e₀ + Δ·m + a·r·s + e₁·s
         = Δ·m + (e₀ + e₁·s - e·r)
         = Δ·m + 노이즈

3. 스케일 다운 및 반올림:
   scaled = ⌊(t/q)·noisy⌉ (mod t)
         ≈ ⌊t·(Δ·m)/q⌉
         ≈ ⌊m + small_error⌉
         = m (노이즈가 충분히 작으면)

노이즈 상쇄의 마법

a·r·s 항이 어떻게 상쇄되는지 주목하세요! 이것은 복호화를 작동하게 하는 아름다운 대수 구조입니다. 남은 노이즈 (e₀ + e₁·s - e·r)는 모든 오류 항이 작기 때문에 작습니다.

동형 연산

덧셈

두 암호문을 더하는 것은 간단한 구성 요소별 덧셈입니다:

ct₁ = (c₀⁽¹⁾, c₁⁽¹⁾)은 m₁을 암호화
ct₂ = (c₀⁽²⁾, c₁⁽²⁾)은 m₂를 암호화

ct_sum = (c₀⁽¹⁾ + c₀⁽²⁾, c₁⁽¹⁾ + c₁⁽²⁾)

왜 작동하는가:
(c₀⁽¹⁾ + c₀⁽²⁾) + (c₁⁽¹⁾ + c₁⁽²⁾)·s
= (c₀⁽¹⁾ + c₁⁽¹⁾·s) + (c₀⁽²⁾ + c₁⁽²⁾·s)
= Δ·m₁ + noise₁ + Δ·m₂ + noise₂
= Δ·(m₁ + m₂) + (noise₁ + noise₂)

노이즈는 가산적으로 증가합니다—천천히!

곱셈

곱셈은 더 복잡합니다:

ct₁ = (c₀⁽¹⁾, c₁⁽¹⁾)은 m₁을 암호화
ct₂ = (c₀⁽²⁾, c₁⁽²⁾)은 m₂를 암호화

계산:
d₀ = c₀⁽¹⁾·c₀⁽²⁾
d₁ = c₀⁽¹⁾·c₁⁽²⁾ + c₁⁽¹⁾·c₀⁽²⁾
d₂ = c₁⁽¹⁾·c₁⁽²⁾

결과: ct_mult = (d₀, d₁, d₂)는 3개의 구성 요소를 가집니다!

복호화하려면: d₀ + d₁·s + d₂·s² ≈ Δ²·m₁·m₂/q

이것은 두 가지 문제를 만듭니다: 3개의 구성 요소(2 대신) 및 잘못된 스케일링(Δ 대신 Δ²).

재선형화

재선형화는 평가 키를 사용하여 3구성 요소 암호문을 다시 2구성 요소로 변환합니다:

평가 키: evk는 s 하에서 s²를 암호화

재선형화 적용:
ct' = (d₀, d₁) + Decrypt(evk, d₂)

결과: m₁·m₂를 암호화하는 2구성 요소 암호문

재스케일링

곱셈 후 스케일링은 Δ 대신 Δ²입니다. 재스케일링이 이를 수정합니다:

ct_rescaled = ⌊(t/q)·ct_mult⌉ (mod q')

여기서 q' = q/p (일부 소수 p에 대해)

노이즈 버짓 분석

노이즈 증가 추적

모든 암호문은 "노이즈 버짓" B를 가지고 있습니다—견딜 수 있는 노이즈의 양:

연산노이즈 증가버짓 영향
새로운 암호화σ√n초기: ~log₂(q/t) 비트
덧셈max(noise₁, noise₂) + small일반적으로 -1 비트
곱셈noise₁ × noise₂버짓 반감
재선형화+작은 노이즈-몇 비트

곱셈 깊이

최대 곱셈 깊이는 노이즈 버짓에 의해 제한됩니다:

초기 버짓: B₀ = log₂(q/t) 비트

k번의 곱셈 후:
B_k ≈ B₀ - k·log₂(B₀) - k·log₂(n) - 오버헤드

B₀ = 150 비트, n = 8192인 예:
깊이 1: ~137 비트
깊이 2: ~124 비트
깊이 3: ~111 비트
깊이 4: ~98 비트
깊이 5: ~85 비트 (위험해짐)
깊이 6: ~72 비트 (실패할 수 있음)

장 요약

  • HE는 모듈러 산술을 가진 다항식 환 R = ℤ[x]/(xn + 1)에서 작동합니다
  • 보안은 Ring-LWE 문제에 기반: 노이즈가 있는 선형 조합에서 비밀 찾기
  • BFV 암호화는 RLWE 인스턴스에 노이즈 마스킹된 메시지 Δ·m을 추가합니다
  • 복호화는 대수 항을 상쇄하고 Δ·m + 작은 노이즈를 남깁니다
  • 덧셈은 노이즈를 가산적으로(천천히) 증가시키고, 곱셈은 승산적으로(빠르게) 증가시킵니다
  • 재선형화는 곱셈 후 암호문 크기를 줄입니다
  • 노이즈 버짓은 복호화가 실패하기 전의 곱셈 깊이를 제한합니다
  • 매개변수 선택은 보안, 성능 및 연산 깊이의 균형을 맞춥니다
  • 중국인의 나머지 정리는 n/2 값의 SIMD 배칭을 가능하게 합니다
  • 일반적인 매개변수: 128비트 보안을 위한 n = 8192, log₂(q) = 218, t = 1024
