Healthcare: Privacy-Preserving Medical Research

Genomic Privacy

Genomic data is highly sensitive—it reveals not just an individual's health risks but also information about their relatives. HE enables genomic analysis while protecting privacy:

Case Study: iDASH Competition

The annual iDASH (integrating Data for Analysis, Anonymization, and SHaring) competition challenges teams to develop privacy-preserving genomic analysis tools:

  • Task: Compute genomic similarity between patients without revealing sequences
  • Data: 10,000 genetic variants per patient
  • Solution: CKKS scheme with batching (pack 10,000 variants into single ciphertext)
  • Performance: ~1 second per comparison (acceptable for research)
  • Privacy: Patients control their data, researchers get aggregate results

class EncryptedCreditScoring:

        self.weights = model_weights  // Proprietary scoring formula
    
    def compute_score(self, ct_user_data):
        // User data encrypted: income, debts, payment history, etc.
        ct_score = 0
        
        // Income factor
        ct_score += multiply_plain(ct_user_data['income'], self.weights['w_income'])

        ct_score += multiply_plain(ct_payment_score, self.weights['w_payment'])

Government and Defense

Secure Voting Systems

HE enables verifiable elections with complete privacy:

E-Voting Protocol

  1. Voter: Encrypts vote with HE (e.g., Paillier for additive)
  2. Voting system: Homomorphically adds all encrypted votes
  3. Tallying: Decrypts only the final sum (not individual votes)
  4. Verification: Each voter can verify their vote was counted
  5. Privacy: Individual votes never revealed, even to administrators
// Simplified voting
function tally_votes(encrypted_votes):
    // Each vote: Enc(1) for YES, Enc(0) for NO
    total = Enc(0)
    for vote in encrypted_votes:
        total = add(total, vote)  // Homomorphic addition
    
    // Threshold decryption (requires multiple authorities)
    final_count = threshold_decrypt(total)
    
    return final_count

// Example: 1,000,000 votes
// Encryption: 1 second per vote (parallel)
// Tallying: 1 second total (just additions!)
// Decryption: 1 second (one operation)

Lessons from Deployments

Success Factors

  • Start simple: Begin with linear models, add complexity incrementally
  • Optimize early: Performance matters—use batching, GPU acceleration from day one
  • Hybrid approaches: Combine HE with other privacy tech (MPC, DP, TEEs)
  • Domain expertise: Work with cryptographers AND domain experts
  • User experience: Hide complexity—users shouldn't need to understand HE

Common Pitfalls

Mistakes to Avoid

  • Over-encryption: Don't encrypt everything—only sensitive data
  • Wrong scheme: Match scheme to use case (BFV for exact, CKKS for approximate)
  • Ignoring noise: Track noise budget carefully to avoid decryption failures
  • Parameter mistakes: Too small (insecure), too large (slow)
  • No benchmarking: Test performance before production
  • Poor key management: Secure key storage and rotation critical
