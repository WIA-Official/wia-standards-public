CHAPTER 6

Privacy-Preserving Machine Learning

The ML Privacy Challenge

Machine learning thrives on data—the more, the better. Yet privacy regulations, competitive concerns, and ethical obligations often prevent data sharing. Homomorphic encryption offers a revolutionary solution: train and deploy ML models on encrypted data, preserving privacy throughout the entire pipeline from training to inference.

This chapter explores how HE enables privacy-preserving machine learning (PPML), from simple linear models to deep neural networks, with practical implementations and real-world case studies.

Privacy-Preserving Inference

The Basic Problem

Consider a medical diagnosis model. A hospital wants to use a pharmaceutical company's AI model, but neither party wants to expose their data:

  • Hospital: Has patient data (sensitive, regulated)
  • Pharma: Has valuable model (trade secret, competitive advantage)
  • Traditional solution: Hospital sends plaintext data to Pharma → Privacy violation!
  • HE solution: Hospital encrypts data, Pharma computes on encrypted data, returns encrypted prediction

Linear Regression with HE

The simplest ML model translates naturally to HE:

// Model: y = w₀ + w₁x₁ + w₂x₂ + ... + wₙxₙ

// Client (has data x)
encrypt_vector(x) → ct_x

// Server (has weights w)
ct_result = w₀  // Plaintext constant
for i in 1..n:
    ct_result += multiply_plain(ct_x[i], w[i])

// Client
y = decrypt(ct_result)

// Performance: ~10ms for n=100 features (CKKS)

Logistic Regression

Requires approximating the sigmoid function:

// Sigmoid: σ(x) = 1 / (1 + e^(-x))
// Polynomial approximation (degree 3):
// σ(x) ≈ 0.5 + 0.125x - 0.0078125x³

def encrypted_sigmoid_approx(ct_x):
    ct_x_squared = multiply(ct_x, ct_x)
    ct_x_cubed = multiply(ct_x_squared, ct_x)
    
    result = 0.5  // Plaintext constant
    result += multiply_plain(ct_x, 0.125)
    result -= multiply_plain(ct_x_cubed, 0.0078125)
    
    return result

// Accuracy: ~98% match to true sigmoid in range [-4, 4]
// Depth: 2 multiplications

Neural Networks on Encrypted Data

Forward Propagation

Neural network inference consists of matrix operations and activation functions:

Layer Structure

// Dense layer: y = activation(Wx + b)

function encrypted_dense_layer(ct_x, W, b, activation):
    // Matrix multiplication (W is plaintext)
    ct_y = matrix_multiply_plain(W, ct_x)
    
    // Add bias
    ct_y = add_plain(ct_y, b)
    
    // Apply activation
    ct_activated = encrypted_activation(ct_y)
    
    return ct_activated

Complete Neural Network Example

// 2-layer network: 784 → 128 → 10 (MNIST digit classification)

class EncryptedNeuralNetwork:
    def __init__(self, W1, b1, W2, b2):
        self.W1 = W1  // 784 × 128
        self.W2 = W2  // 128 × 10
        self.b1 = b1
        self.b2 = b2
    
    def predict(self, ct_input):
        // Layer 1: 784 → 128
        ct_hidden = matrix_mult_plain(self.W1, ct_input)
        ct_hidden = add_plain(ct_hidden, self.b1)
        ct_hidden = encrypted_relu(ct_hidden)  // Polynomial approx
        
        // Layer 2: 128 → 10
        ct_output = matrix_mult_plain(self.W2, ct_hidden)
        ct_output = add_plain(ct_output, self.b2)
        
        // No activation on output (softmax done by client)
        return ct_output
    
    def encrypted_relu(self, ct_x):
        // ReLU approximation: max(0, x) ≈ polynomial
        ct_squared = multiply(ct_x, ct_x)
        ct_cubed = multiply(ct_squared, ct_x)
        
        result = multiply_plain(ct_x, 0.5)
        result = add_plain(result, multiply_plain(ct_squared, 0.25))
        result = subtract_plain(result, multiply_plain(ct_cubed, 0.0625))
        
        return result

// Performance: ~3 seconds per image on GPU (CKKS)
// Accuracy: 97.2% (vs 98.1% plaintext)

Privacy-Preserving Training

Encrypted Gradient Descent

Training models on encrypted data is more complex but possible:

// Linear regression training: minimize ||Xw - y||²

function encrypted_gradient_descent(ct_X, ct_y, epochs, learning_rate):
    w = initialize_weights()  // Plaintext weights
    
    for epoch in 1..epochs:
        // Compute encrypted predictions
        ct_predictions = matrix_mult_plain(w, ct_X)
        
        // Compute encrypted error
        ct_error = subtract(ct_predictions, ct_y)
        
        // Compute gradient: ∇w = X^T · error / n
        ct_gradient = matrix_mult_plain(X_transpose, ct_error)
        
        // Decrypt gradient (only gradients, not data!)
        gradient = decrypt(ct_gradient)
        
        // Update weights (plaintext operation)
        w -= learning_rate * gradient / n
    
    return w

// Hybrid approach: data encrypted, weights plaintext
// Privacy: Training data never revealed
// Performance: 10-100x slower than plaintext training

# Privacy guarantees:

Convolutional Neural Networks

Encrypted Convolution

Convolution is matrix multiplication in disguise, perfect for HE:

// 2D convolution with HE
function encrypted_conv2d(ct_image, kernel, stride):
    // Image: h × w encrypted in batched ciphertext
    // Kernel: k × k plaintext filter
    
    ct_output = []
    for i in range(0, h-k+1, stride):
        for j in range(0, w-k+1, stride):
            // Extract patch via rotations
            ct_patch = extract_patch(ct_image, i, j, k)
            
            // Multiply patch by kernel (plaintext)
            ct_conv_result = multiply_plain(ct_patch, kernel)
            
            // Sum to get single output value
            ct_sum = sum_slots(ct_conv_result)
            
            ct_output.append(ct_sum)
    
    return ct_output

// Optimization: Use im2col transformation
// Converts convolution to matrix multiplication
// Enables batching across multiple images

Pooling Operations

Max pooling requires comparisons (use TFHE) or approximations (use CKKS):

// Approximate max pooling (CKKS)
// max(a, b) ≈ (a + b + |a - b|) / 2

function encrypted_max_pool(ct_values):
    n = len(ct_values)
    
    if n == 1:
        return ct_values[0]
    
    // Pairwise max
    ct_pairs = []
    for i in range(0, n, 2):
        if i+1 < n:
            ct_max = approximate_max(ct_values[i], ct_values[i+1])
        else:
            ct_max = ct_values[i]
        ct_pairs.append(ct_max)
    
    // Recursive max
    return encrypted_max_pool(ct_pairs)

function approximate_max(ct_a, ct_b):
    ct_sum = add(ct_a, ct_b)
    ct_diff = subtract(ct_a, ct_b)
    ct_abs_diff = approximate_abs(ct_diff)
    ct_result = add(ct_sum, ct_abs_diff)
    ct_result = multiply_plain(ct_result, 0.5)
    return ct_result

Medical Diagnosis

  • Data: Patient genomic sequences (30GB per patient)
  • Model: Deep neural network (proprietary)
  • HE Scheme: CKKS (approximate computation)
  • Performance: 5 minutes per patient (acceptable for diagnosis)
  • Privacy: Complete—neither genomic data nor model weights revealed

Users query cloud ML models without revealing their queries:

  • Cloud: Runs ML model on encrypted input

  • Benefit: Cloud provider never sees user data

Optimization Techniques for PPML

Model Compression

  • Pruning: Remove less important weights (reduces depth)
  • Quantization: Use lower precision (faster HE operations)
  • Knowledge distillation: Train smaller model that mimics large one

Hybrid Execution

// Mix plaintext and encrypted computation strategically

// Cheap operations in plaintext
preprocessing = plaintext_normalize(raw_data)
features = plaintext_feature_extraction(preprocessing)

// Expensive/sensitive operations encrypted
ct_features = encrypt(features)
ct_prediction = encrypted_model(ct_features)

// Cheap post-processing in plaintext
prediction = decrypt(ct_prediction)
formatted_result = plaintext_formatting(prediction)

// Trade-off: Some information leakage, much better performance

Transfer Learning

Train base model on plaintext public data, fine-tune on encrypted private data:

// Pre-train on ImageNet (public data, plaintext)
base_model = train_on_imagenet()

// Fine-tune last layer on hospital data (encrypted)
ct_hospital_data = encrypt(sensitive_patient_images)
fine_tuned_model = encrypted_fine_tune(base_model, ct_hospital_data)

// Result: Better accuracy, less encrypted training time

Challenges and Future Directions

Current Limitations

  • Performance: Still 10-1000x slower than plaintext
  • Activation functions: Polynomial approximations lose accuracy
  • Deep networks: Many layers consume noise budget quickly
  • Batch normalization: Requires division (expensive in HE)

Emerging Solutions

  • Custom activation functions: Polynomials designed for HE
  • Architecture search: Auto-design networks optimized for HE
  • Hardware acceleration: Specialized chips for FHE operations
  • Hybrid protocols: Combine HE with MPC and differential privacy
