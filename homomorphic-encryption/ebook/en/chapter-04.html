CHAPTER 4

FHE Libraries and Tools

The FHE Ecosystem

Implementing homomorphic encryption from scratch is a monumental task requiring deep expertise in cryptography, number theory, and high-performance computing. Fortunately, mature open-source libraries make HE accessible to developers. This chapter surveys the major FHE libraries, their capabilities, and how to get started with each one.

Microsoft SEAL

Overview

Microsoft SEAL (Simple Encrypted Arithmetic Library) is the most popular FHE library, developed by Microsoft Research's Cryptography and Privacy Research group.

Feature Details
Language C++17, with .NET, Python, and Rust wrappers
Schemes BFV and CKKS
License MIT (permissive)
Performance Excellent (highly optimized)
Documentation Comprehensive with examples
Maturity Production-ready, widely deployed

Installation and Setup

# Install on Linux/macOS
git clone https://github.com/Microsoft/SEAL.git
cd SEAL
cmake -S . -B build
cmake --build build
sudo cmake --install build

# Python wrapper
pip install seal

# .NET wrapper
dotnet add package Microsoft.Research.SEALNet

Basic SEAL Example (C++)

#include "seal/seal.h"
using namespace seal;

int main() {
    // 1. Set encryption parameters
    EncryptionParameters params(scheme_type::bfv);
    size_t poly_modulus_degree = 8192;
    params.set_poly_modulus_degree(poly_modulus_degree);
    params.set_coeff_modulus(
        CoeffModulus::BFVDefault(poly_modulus_degree)
    );
    params.set_plain_modulus(1024);

    // 2. Create SEALContext
    SEALContext context(params);

    // 3. Generate keys
    KeyGenerator keygen(context);
    SecretKey secret_key = keygen.secret_key();
    PublicKey public_key;
    keygen.create_public_key(public_key);

    // 4. Create encryptor, evaluator, decryptor
    Encryptor encryptor(context, public_key);
    Evaluator evaluator(context);
    Decryptor decryptor(context, secret_key);

    // 5. Encrypt values
    Plaintext plain1("42");
    Plaintext plain2("17");
    Ciphertext cipher1, cipher2;
    encryptor.encrypt(plain1, cipher1);
    encryptor.encrypt(plain2, cipher2);

    // 6. Homomorphic addition
    Ciphertext cipher_sum;
    evaluator.add(cipher1, cipher2, cipher_sum);

    // 7. Decrypt result
    Plaintext plain_result;
    decryptor.decrypt(cipher_sum, plain_result);
    std::cout << "42 + 17 = " << plain_result.to_string()
              << std::endl; // Output: 59

    return 0;
}

SEAL Python Example

from seal import *

# Setup parameters
parms = EncryptionParameters(scheme_type.bfv)
parms.set_poly_modulus_degree(8192)
parms.set_coeff_modulus(CoeffModulus.BFVDefault(8192))
parms.set_plain_modulus(1024)

context = SEALContext(parms)

# Generate keys
keygen = KeyGenerator(context)
public_key = keygen.create_public_key()
secret_key = keygen.secret_key()

# Setup tools
encryptor = Encryptor(context, public_key)
evaluator = Evaluator(context)
decryptor = Decryptor(context, secret_key)
encoder = IntegerEncoder(context)

# Encrypt and compute
plain1 = encoder.encode(42)
plain2 = encoder.encode(17)
cipher1 = encryptor.encrypt(plain1)
cipher2 = encryptor.encrypt(plain2)

result = evaluator.add(cipher1, cipher2)
decrypted = decryptor.decrypt(result)
print(f"Result: {encoder.decode_int32(decrypted)}")  # 59

When to Use SEAL

  • Production applications: Battle-tested and optimized
  • BFV or CKKS schemes: Best implementation available
  • Cross-platform needs: Windows, Linux, macOS support
  • Multiple languages: C++, Python, C#, Rust bindings
  • Microsoft ecosystem: Integrates with Azure

PALISADE (OpenFHE)

Overview

PALISADE was a comprehensive lattice cryptography library that has evolved into OpenFHE, supported by a consortium of government and industry partners.

Feature Details
Language C++, Python bindings
Schemes BFV, BGV, CKKS, FHEW, TFHE
License BSD 2-Clause
Features Most comprehensive scheme support
Threshold FHE Yes (unique feature)
Documentation Extensive with research focus

OpenFHE Installation

# Install OpenFHE
git clone https://github.com/openfheorg/openfhe-development.git
cd openfhe-development
mkdir build && cd build
cmake ..
make -j
sudo make install

# Python wrapper
pip install openfhe

OpenFHE Example

#include "openfhe.h"

using namespace lbcrypto;

int main() {
    // Parameters for CKKS
    CCParams parameters;
    parameters.SetMultiplicativeDepth(5);
    parameters.SetScalingModSize(50);
    parameters.SetBatchSize(8);

    CryptoContext cc =
        GenCryptoContext(parameters);
    cc->Enable(PKE);
    cc->Enable(KEYSWITCH);
    cc->Enable(LEVELEDSHE);

    // Key generation
    auto keys = cc->KeyGen();
    cc->EvalMultKeyGen(keys.secretKey);

    // Encode vectors
    std::vector x = {1.0, 2.0, 3.0, 4.0};
    Plaintext ptxt = cc->MakeCKKSPackedPlaintext(x);

    // Encrypt
    auto ctxt = cc->Encrypt(keys.publicKey, ptxt);

    // Homomorphic operations
    auto ctxt_square = cc->EvalMult(ctxt, ctxt);

    // Decrypt
    Plaintext result;
    cc->Decrypt(keys.secretKey, ctxt_square, &result);
    result->SetLength(4);
    std::cout << result << std::endl;
    // Output: [1.0, 4.0, 9.0, 16.0]

    return 0;
}

When to Use OpenFHE

  • Research projects: Cutting-edge features and schemes
  • Multiple schemes: Need TFHE alongside BFV/CKKS
  • Threshold cryptography: Unique support for threshold FHE
  • Government/defense: DARPA-funded, trusted in sensitive applications
  • Flexibility: More configuration options than SEAL

TFHE-rs

Overview

TFHE-rs is a pure Rust implementation of the TFHE scheme, offering memory safety and excellent performance for boolean circuits.

Feature Details
Language Rust
Schemes TFHE (boolean and shortint)
License BSD-3-Clause-Clear
Performance Excellent for gates and comparisons
GPU Support Yes (CUDA backend)
Safety Memory-safe Rust implementation

TFHE-rs Installation

# Add to Cargo.toml
[dependencies]
tfhe = { version = "0.5", features = ["boolean", "shortint", "x86_64-unix"] }

# With GPU support
tfhe = { version = "0.5", features = ["boolean", "shortint", "gpu"] }

TFHE-rs Boolean Example

use tfhe::boolean::prelude::*;

fn main() {
    // Generate keys
    let (client_key, server_key) = gen_keys();

    // Client encrypts
    let ct_1 = client_key.encrypt(true);
    let ct_2 = client_key.encrypt(false);

    // Server computes (without seeing values)
    let ct_and = server_key.and(&ct_1, &ct_2);
    let ct_or = server_key.or(&ct_1, &ct_2);
    let ct_xor = server_key.xor(&ct_1, &ct_2);

    // Client decrypts
    let and_result = client_key.decrypt(&ct_and);
    let or_result = client_key.decrypt(&ct_or);
    let xor_result = client_key.decrypt(&ct_xor);

    println!("AND: {}", and_result);  // false
    println!("OR: {}", or_result);    // true
    println!("XOR: {}", xor_result);  // true
}

TFHE-rs Integer Operations

use tfhe::shortint::prelude::*;

fn main() {
    // Setup parameters (2-bit integers)
    let (client_key, server_key) = gen_keys(
        Parameters::new(2, 2)
    );

    // Encrypt 8-bit integers
    let ct1 = client_key.encrypt(42u64);
    let ct2 = client_key.encrypt(17u64);

    // Homomorphic comparison
    let ct_greater = server_key.gt(&ct1, &ct2);
    let result = client_key.decrypt(&ct_greater);
    println!("42 > 17: {}", result);  // true

    // Homomorphic max
    let ct_max = server_key.max(&ct1, &ct2);
    let max_val = client_key.decrypt(&ct_max);
    println!("max(42, 17): {}", max_val);  // 42
}

When to Use TFHE-rs

  • Rust ecosystem: Memory safety and performance
  • Boolean circuits: Decision trees, comparisons
  • Fast bootstrapping: Deep circuits with frequent refresh
  • GPU acceleration: CUDA support for massive speedup
  • Integer comparisons: Encrypted sorting, min/max operations

HElib

Overview

HElib, developed by IBM Research, was one of the first production-quality FHE libraries and continues to be actively developed.

Feature Details
Language C++
Schemes BGV, CKKS
License Apache 2.0
Bootstrapping Advanced BGV bootstrapping
Maturity Very mature, research-proven
Documentation Academic focus, learning curve

HElib Example

#include 

using namespace helib;

int main() {
    // Setup parameters
    long m = 16384;  // cyclotomic polynomial
    long p = 127;    // plaintext prime
    long r = 1;      // lifting
    long L = 5;      // levels
    long c = 2;      // columns in key-switching matrix
    long w = 64;     // Hamming weight of secret key
    long d = 1;      // degree of field extension

    Context context(m, p, r);
    buildModChain(context, L, c);

    // Generate keys
    SecKey secretKey(context);
    secretKey.GenSecKey();
    addSome1DMatrices(secretKey);

    const PubKey& publicKey = secretKey;

    // Encrypt
    std::vector ptxt1 = {1, 2, 3, 4};
    std::vector ptxt2 = {5, 6, 7, 8};

    Ctxt ctxt1(publicKey);
    Ctxt ctxt2(publicKey);
    publicKey.Encrypt(ctxt1, to_ZZX(ptxt1));
    publicKey.Encrypt(ctxt2, to_ZZX(ptxt2));

    // Homomorphic operations
    ctxt1 += ctxt2;  // Element-wise addition

    // Decrypt
    std::vector result;
    secretKey.Decrypt(result, ctxt1);
    // result: [6, 8, 10, 12]

    return 0;
}

When to Use HElib

  • BGV-specific needs: Best BGV implementation
  • Academic research: Well-documented in literature
  • Deep bootstrapping: Advanced techniques
  • IBM ecosystem: Integration with IBM tools

Concrete

Overview

Concrete is Zama's FHE framework with a focus on usability and production deployments.

Feature Details
Language Python, Rust core
Schemes TFHE-based
License BSD-3-Clause-Clear
Compiler Automatic circuit compilation
ML Focus Concrete-ML for encrypted ML
Ease of Use High-level Python API

Concrete Example

from concrete import fhe

# Define function to encrypt
@fhe.compiler({"x": "encrypted"})
def add_one(x):
    return x + 1

# Compile for specific input range
inputset = range(100)
circuit = add_one.compile(inputset)

# Encrypt
encrypted_x = circuit.encrypt(42)

# Execute on encrypted data
encrypted_result = circuit.run(encrypted_x)

# Decrypt
result = circuit.decrypt(encrypted_result)
print(f"Result: {result}")  # 43

Concrete-ML for Machine Learning

from concrete.ml.sklearn import LinearRegression
import numpy as np

# Train model
X = np.array([[1], [2], [3], [4]])
y = np.array([2, 4, 6, 8])

model = LinearRegression()
model.fit(X, y)

# Compile to FHE
model.compile(X)

# Predict on encrypted data
X_test = np.array([[5]])
encrypted_X = model.quantize_input(X_test)
encrypted_prediction = model.predict(encrypted_X, fhe="execute")

print(f"Prediction: {encrypted_prediction}")  # ~10

When to Use Concrete

  • Python developers: Pythonic API, easy learning curve
  • Rapid prototyping: Quick experimentation
  • ML applications: Concrete-ML integration
  • Automatic optimization: Compiler handles parameters
  • Modern workflow: Best developer experience

Best Practices for Library Selection

Decision Framework

  1. Choose by scheme:
    • BFV/CKKS → SEAL or OpenFHE
    • TFHE → TFHE-rs or Concrete
    • BGV → HElib or OpenFHE
  2. Choose by language:
    • Python → Concrete or SEAL Python bindings
    • C++ → SEAL, OpenFHE, or HElib
    • Rust → TFHE-rs
    • .NET → SEAL .NET
  3. Choose by use case:
    • Production deployment → SEAL
    • Research → OpenFHE
    • Machine learning → Concrete-ML or SEAL
    • Boolean logic → TFHE-rs
  4. Choose by ecosystem:
    • Microsoft/Azure → SEAL
    • Defense/Gov → OpenFHE
    • Startup/Modern → Concrete

Chapter Summary

  • Microsoft SEAL: Production-ready BFV/CKKS, best performance, wide language support
  • OpenFHE: Most comprehensive scheme support, research-focused, threshold crypto
  • TFHE-rs: Best for boolean circuits, memory-safe Rust, GPU acceleration
  • HElib: Mature BGV implementation, IBM-backed, academic focus
  • Concrete: Python-first, easiest to learn, excellent for ML with Concrete-ML
  • Choose library based on scheme needs, programming language, and use case
  • All major libraries are production-ready and actively maintained
  • Performance varies by use case—benchmark for your specific application
  • Most libraries support parameter selection and optimization tools
  • Modern development increasingly focuses on usability and automation
