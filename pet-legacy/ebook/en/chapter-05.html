5.2 Tribute Submission

TRIBUTE SUBMISSION PROCESS
==========================

1. Visitor Access
   - Public: No login required
   - Shared: Access via invite link
   - Family: Requires authentication

2. Author Information
   - Name (required)
   - Email (optional, for notification)
   - Relationship to pet/family

3. Content Creation
   - Select tribute type
   - Write message
   - Attach media (optional)
   - Add virtual offering

4. Submission
   - Preview tribute
   - Submit for review
   - Receive confirmation

5. Moderation (if enabled)
   - Appears in queue
   - Family reviews
   - Approve or reject
   - Notification sent

5.4 Moderation System

TRIBUTE MODERATION WORKFLOW
===========================

Status Flow:
  PENDING --> APPROVED --> Published
         \--> REJECTED --> Hidden
         \--> FLAGGED  --> Review

Moderation Actions:
  [Approve] - Publish to memorial
  [Reject]  - Hide, optionally notify
  [Flag]    - Mark for further review
  [Edit]    - Minor corrections (owner only)
  [Delete]  - Permanent removal

Auto-Moderation:
  - Profanity filter (configurable)
  - Spam detection
  - Link checking
  - Length limits

Moderation Queue

Tribute Author Status Action

5.5 Notification System

TRIBUTE NOTIFICATIONS
=====================

For Family:
  - New tribute submitted
  - Tribute approved (if auto-moderation)
  - Moderation action needed
  - Offering received

For Tribute Authors:
  - Tribute approved
  - Tribute rejected (optional message)
  - Response from family (optional)

Notification Channels:
  - Email (primary)
  - Push notification (if app)
  - In-app notification
  - SMS (for urgent, optional)

5.6 Guest Tributes

GUEST TRIBUTE HANDLING
======================

Guest tributes (no account):
  - Require name
  - Optional email
  - Subject to moderation
  - Cannot be edited after submission
  - Author cannot track status

Registered user tributes:
  - Linked to account
  - Can edit before approval
  - Receive notifications
  - Can see tribute status
  - Can manage their tributes
