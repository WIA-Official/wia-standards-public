2.1 Legacy Profile

LEGACY PROFILE STRUCTURE
========================

LegacyProfile {
  legacyId: UUID           // Unique identifier
  petId: UUID              // Reference to pet record
  ownerId: UUID            // Primary owner

  petInfo: PetBiography    // Complete biography

  lifeTimeline: LifeEvent[]    // Life events
  memorialConfig: MemorialConfiguration
  accessControl: AccessSettings

  status: LegacyStatus
  createdAt: DateTime
  activatedAt: DateTime | null
  updatedAt: DateTime
}

Status Values:
  PLANNING  - Being prepared
  DORMANT   - Ready, not active
  ACTIVE    - Memorial live
  ARCHIVED  - Long-term storage

2.4 Life Events

Event Types

Type Category Description
BIRTH Beginning Pet's birthday
ADOPTION Beginning Joined the family
FIRST_HOME Beginning Arrived at home
MILESTONE Growth Major achievement
HEALTH_EVENT Health Medical milestone
TRAVEL Adventure Trip or vacation
ACHIEVEMENT Growth Training or award
CELEBRATION Celebration Birthday, holiday
MEMORABLE_MOMENT Daily Life Special memory
PASSING Farewell End of life

Event Structure

LifeEvent {
  eventId: UUID
  legacyId: UUID

  eventType: LifeEventType
  category: EventCategory

  title: String
  description: Text
  date: Date
  location: String | null

  media: MediaAttachment[]
  participants: Person[]

  significance: 1-5
  featured: Boolean

  createdAt: DateTime
  createdBy: UUID
}

2.5 Media Attachments

MEDIA STRUCTURE
===============

MediaAttachment {
  mediaId: UUID
  legacyId: UUID
  eventId: UUID | null

  // Type
  mediaType: PHOTO | VIDEO | AUDIO | DOCUMENT
  mimeType: String

  // Storage
  originalUrl: URL
  thumbnailUrl: URL
  archiveUrl: URL

  // Metadata
  filename: String
  fileSize: Number
  dimensions: { width, height } | null
  duration: Number | null  // For video/audio

  // Description
  caption: String
  altText: String
  tags: String[]

  // Dates
  captureDate: Date | null
  uploadDate: DateTime

  // AI Analysis
  aiTags: String[]
  petDetected: Boolean
  emotionDetected: String | null

  // Access
  visibility: PRIVATE | FAMILY | MEMORIAL | PUBLIC
}

Visibility Levels

Level Who Can View Use Case
PRIVATE Owner only Personal memories
FAMILY Family members Shared family photos
MEMORIAL Memorial visitors Public tribute gallery
PUBLIC Anyone Open sharing

2.6 Tribute Messages

TRIBUTE STRUCTURE
=================

Tribute {
  tributeId: UUID
  legacyId: UUID

  // Author
  authorId: UUID | null
  authorName: String
  authorEmail: String | null
  relationship: String

  // Content
  message: Text
  media: MediaAttachment[]

  // Type
  tributeType: MESSAGE | MEMORY | POEM | SONG | ARTWORK

  // Status
  status: PENDING | APPROVED | REJECTED

  // Offerings
  offerings: VirtualOffering[]

  submittedAt: DateTime
  publishedAt: DateTime | null
}

VirtualOffering {
  offeringType: CANDLE | FLOWER | HEART | PAW_PRINT | RAINBOW
  message: String | null
  offeredAt: DateTime
  offeredBy: String
}

2.7 Access Control

ACCESS CONTROL STRUCTURE
========================

LegacyAccessControl {
  legacyId: UUID
  primaryOwner: UUID

  familyMembers: [
    {
      userId: UUID
      relationship: String
      role: PRIMARY_OWNER | CO_OWNER | FAMILY_MEMBER | CARETAKER
      accessLevel: FULL | EDIT | CONTRIBUTE | VIEW
    }
  ]

  accessGrants: [
    {
      grantId: UUID
      grantType: user | email | link
      grantee: String
      accessLevel: AccessLevel
      expiresAt: DateTime | null
    }
  ]

  inheritanceSettings: {
    successors: Successor[]
    inactivityPeriod: Number  // days
    onInheritance: Action[]
  }
}
