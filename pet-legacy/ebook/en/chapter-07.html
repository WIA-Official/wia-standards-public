7.2 Storage Tiers

ARCHIVE STORAGE TIERS
=====================

TIER 1: HOT STORAGE
  Technology: SSD-backed cloud
  Access Time: Instant
  Cost: Highest
  Use: Active memorials
  Retention: While active

TIER 2: WARM STORAGE
  Technology: Standard cloud
  Access Time: Minutes
  Cost: Medium
  Use: Dormant legacies
  Retention: 5+ years

TIER 3: COLD STORAGE
  Technology: Glacier Deep Archive
  Access Time: 12-48 hours
  Cost: Lowest
  Use: Long-term preservation
  Retention: 100+ years

TIER 4: PHYSICAL (Optional)
  Technology: M-DISC optical media
  Access Time: Manual retrieval
  Cost: One-time
  Use: Ultimate preservation
  Retention: 1000+ years

7.3 Data Integrity

INTEGRITY VERIFICATION
======================

At Archive Creation:
  1. Calculate SHA-256 for each file
  2. Calculate SHA-256 for archive bundle
  3. Store checksums in manifest
  4. Encrypt with AES-256
  5. Verify after upload

Ongoing Verification:
  - Monthly spot checks (random 10%)
  - Annual full verification
  - Compare all checksums
  - Flag and repair issues

Redundancy:
  - Minimum 3 copies
  - Geographic distribution
  - Different providers
  - Different media types

7.5 Inheritance System

INHERITANCE CONFIGURATION
=========================

Successors:
  List of people who inherit access
  Ordered by priority
  Must accept to inherit

  Example:
  1. spouse@email.com (Primary)
  2. child@email.com (Secondary)
  3. friend@email.com (Tertiary)

Inactivity Period:
  Days without owner login before
  succession process begins
  Default: 365 days (1 year)

On Inheritance Actions:
  TRANSFER_OWNERSHIP - New owner
  MAINTAIN_PUBLIC    - Keep memorial live
  ARCHIVE            - Move to cold storage
  DELETE             - Permanent removal

Inheritance Process

INHERITANCE WORKFLOW
====================

Step 1: Detect Inactivity
  Owner inactive for [configured days]
  System flags for succession

Step 2: Attempt Contact
  - Email to owner
  - SMS if configured
  - Wait 30 days

Step 3: If No Response
  - Notify primary successor
  - Provide acceptance link
  - Wait 30 days

Step 4: Successor Accepts
  - Verify identity
  - Transfer access
  - Apply configured actions

Step 5: If Declined/No Response
  - Move to next successor
  - Repeat until exhausted
  - Apply fallback action

7.7 Access Recovery

ACCESS RECOVERY OPTIONS
=======================

1. Password Reset
   - Standard email recovery
   - Security questions
   - Two-factor verification

2. Trusted Contact Recovery
   - Pre-designated recovery contact
   - Identity verification required
   - 7-day waiting period

3. Legal Access Request
   - Death certificate of owner
   - Proof of relationship
   - Legal documentation
   - Manual review process

4. Emergency Access
   - Court order
   - Immediate family
   - Expedited review
