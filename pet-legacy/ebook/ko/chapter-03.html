3.1 사진 큐레이션

AI 기반 사진 선택

사진 점수 알고리즘
=================

점수 구성요소 (0-100 총점):

1. 기술 품질 (0-25점)
   - 선명도: 0-10
   - 노출: 0-10
   - 구도: 0-5

2. 반려동물 존재 (0-30점)
   - 반려동물 감지: 0-15
   - 얼굴 보임: 0-10
   - 눈맞춤: 0-5

3. 감정 콘텐츠 (0-25점)
   - 긍정적 감정 감지: 0-20
   - 따뜻함/기쁨 보임: 0-5

4. 맥락 가치 (0-20점)
   - 캡션 있음: 0-5
   - 이벤트 연결: 0-5
   - 태그됨: 0-10

다양성 기준

기준 제한 목적
같은 달 최대 3장 시간 다양성
같은 장소 최대 2장 장소 다양성
비슷한 포즈 최대 2장 시각 다양성

3.4 타임라인 생성

타임라인 알고리즘
================

1. 사건 수집
   - 전기 이정표 (출생, 입양)
   - 사용자 생성 사건
   - 미디어 클러스터에서 자동 감지
   - 해당 시 떠남 사건

2. 시간순 정렬
   events.sort(by: date)

3. 미디어에서 사건 추론
   각 3장 이상 사진이 있는 날짜:
     - 공통 테마 분석
     - "[날짜]의 추억" 생성
     - 대표 사진 첨부

4. 생애 기간으로 그룹화
   - 시작 (첫해)
   - 성장 (발달 단계)
   - 전성기
   - 노년 (해당 시)
   - 작별

5. 타임라인 객체 생성

3.5 보관 표준

보존 티어

저장소 티어
==========

티어 1: 핫 스토리지
  목적: 활성 접근
  기술: SSD 기반 클라우드
  접근 시간: 즉시
  용도: 활성 추모

티어 2: 웜 스토리지
  목적: 가끔 접근
  기술: 표준 클라우드
  접근 시간: 분 단위
  용도: 대기 유산

티어 3: 콜드 스토리지
  목적: 장기 보관
  기술: Glacier Deep Archive
  접근 시간: 시간 단위
  용도: 영구 보존

티어 4: 물리 (선택)
  목적: 궁극적 보존
  기술: M-DISC 광학 미디어
  접근 시간: 수동 검색
  용도: 1000년 이상 보관

3.6 데이터 무결성

무결성 검증
==========

업로드 시:
  1. SHA-256 체크섬 계산
  2. 메타데이터와 저장
  3. 전송 후 검증

보관 시:
  1. 체크섬 재계산
  2. 원본과 비교
  3. 불일치 플래그

연간 검증:
  1. 보관소 검색
  2. 모든 체크섬 검증
  3. 무결성 상태 보고
  4. 필요 시 복구

중복성:
  - 최소 3개 복사본
  - 지리적 분산
  - 다른 제공자
