제7장: 보관 및 상속

"홍익인간: 오늘 보존하는 것이 내일의 소중한 유산이 됩니다."

7.1 보관 유형

유형 내용 크기 용도
FULL 모든 것 1-10 GB 완전 보존
MEDIA_ONLY 사진, 동영상 500MB-5GB 시각적 기억
ESSENTIAL 핵심 데이터만 50-200 MB 빠른 내보내기
EXPORT 사용자 선택 가변 맞춤 필요

7.2 저장소 티어

보관 저장소 티어
===============

티어 1: 핫 스토리지
  기술: SSD 기반 클라우드
  접근 시간: 즉시
  비용: 최고
  용도: 활성 추모

티어 2: 웜 스토리지
  기술: 표준 클라우드
  접근 시간: 분
  비용: 중간
  용도: 대기 유산

티어 3: 콜드 스토리지
  기술: Glacier Deep Archive
  접근 시간: 12-48시간
  비용: 최저
  용도: 장기 보존

티어 4: 물리 (선택)
  기술: M-DISC 광학 미디어
  접근 시간: 수동 검색
  비용: 일회성
  용도: 1000년 이상 보관

7.3 데이터 무결성

무결성 검증
==========

보관 생성 시:
  1. 각 파일 SHA-256 계산
  2. 보관 번들 SHA-256 계산
  3. 매니페스트에 체크섬 저장
  4. AES-256 암호화
  5. 업로드 후 검증

지속적 검증:
  - 월별 무작위 검사 (10%)
  - 연간 전체 검증
  - 모든 체크섬 비교
  - 문제 플래그 및 복구

중복성:
  - 최소 3개 복사본
  - 지리적 분산
  - 다른 제공자
  - 다른 미디어 유형

7.4 보존 정책

정책 기간 적합
표준 50년 기본 선택
확장 100년 다세대
영구 무기한 영원한 보존
가족 상속인 존재 동안 상속 가능 유산

7.5 상속 시스템

상속 설정
=========

후계자:
  접근 권한을 상속받을 사람 목록
  우선순위 순서
  상속하려면 수락 필요

  예시:
  1. spouse@email.com (주)
  2. child@email.com (부)
  3. friend@email.com (3순위)

비활성 기간:
  소유자 로그인 없이 승계 프로세스
  시작 전 일수
  기본: 365일 (1년)

상속 시 행동:
  TRANSFER_OWNERSHIP - 새 소유자
  MAINTAIN_PUBLIC    - 추모 유지
  ARCHIVE            - 콜드 스토리지로 이동
  DELETE             - 영구 제거

상속 프로세스

상속 워크플로
============

1단계: 비활성 감지
  소유자 [설정 일수] 비활성
  시스템이 승계 플래그

2단계: 연락 시도
  - 소유자에게 이메일
  - 설정된 경우 SMS
  - 30일 대기

3단계: 응답 없으면
  - 주 후계자에게 알림
  - 수락 링크 제공
  - 30일 대기

4단계: 후계자 수락
  - 신원 확인
  - 접근 권한 이전
  - 설정된 행동 적용

5단계: 거절/응답 없으면
  - 다음 후계자로 이동
  - 모두 소진될 때까지 반복
  - 폴백 행동 적용

7.6 내보내기 옵션

형식 내용 용도
JSON 데이터만 마이그레이션, 백업
ZIP 데이터 + 미디어 완전 백업
PDF 인쇄용 추모 실물 기념품
HTML 독립형 웹사이트 자체 호스팅 추모

7.7 접근 복구

접근 복구 옵션
=============

1. 비밀번호 재설정
   - 표준 이메일 복구
   - 보안 질문
   - 2단계 인증

2. 신뢰할 수 있는 연락처 복구
   - 사전 지정된 복구 연락처
   - 신원 확인 필요
   - 7일 대기 기간

3. 법적 접근 요청
   - 소유자 사망 증명서
   - 관계 증명
   - 법적 문서
   - 수동 검토 프로세스

4. 긴급 접근
   - 법원 명령
   - 직계 가족
   - 신속 검토

7.8 장기 보존

100년 보존 계획
==============

기술 마이그레이션:
  - 포맷 구식화 모니터링
  - 새 표준으로 변환
  - 하위 호환성 유지
  - 연간 복원 테스트

인프라:
  - 지리적 중복성
  - 제공자 다양성
  - 정기 감사
  - 재해 복구

자금 모델:
  - 선불 보존
  - 기금
  - 연간 확인
  - 승계 계획

품질 보증:
  - 연간 무결성 확인
  - 접근성 테스트
  - 가족 알림
  - 보고서 생성

장 요약

  • [OK] 다양한 필요를 위한 4가지 보관 유형
  • [OK] 다중 티어 저장소가 접근과 비용 균형
  • [OK] 데이터 무결성 지속적 검증
  • [OK] 상속이 세대 간 접근 보장
  • [OK] 다중 내보내기 형식 사용 가능
  • [OK] 100년 보존 계획
