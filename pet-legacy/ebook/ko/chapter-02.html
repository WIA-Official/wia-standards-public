제2장: 유산 데이터 구조

"홍익인간: 구조화된 데이터는 기억이 시간과 기술 변화를 통해 지속되도록 보장합니다."

2.1 유산 프로필

유산 프로필 구조
===============

LegacyProfile {
  legacyId: UUID           // 고유 식별자
  petId: UUID              // 반려동물 기록 참조
  ownerId: UUID            // 주 보호자

  petInfo: PetBiography    // 완전한 전기
  lifeTimeline: LifeEvent[]    // 생애 사건
  memorialConfig: MemorialConfiguration
  accessControl: AccessSettings

  status: LegacyStatus
  createdAt: DateTime
  activatedAt: DateTime | null
}

상태 값:
  PLANNING  - 준비 중
  DORMANT   - 준비됨, 비활성
  ACTIVE    - 추모 활성화
  ARCHIVED  - 장기 저장

2.2 반려동물 전기

필드 타입 설명
name String 반려동물 이름
nickname String[] 애칭들
species String 개, 고양이 등
breed String 품종 또는 믹스
birthDate Date 생년월일
adoptionDate Date 가족이 된 날
passingDate Date 떠난 날
personalityTraits String[] 성격 특성
biography Text 삶의 이야기
epitaph String 추모 문구

2.4 생애 사건

사건 유형

유형 범주 설명
BIRTH 시작 생일
ADOPTION 시작 가족이 됨
FIRST_HOME 시작 집에 도착
MILESTONE 성장 주요 성취
TRAVEL 모험 여행
ACHIEVEMENT 성장 훈련 또는 수상
CELEBRATION 축하 생일, 기념일
MEMORABLE_MOMENT 일상 특별한 추억
PASSING 작별 삶의 마지막

2.5 미디어 첨부

미디어 구조
==========

MediaAttachment {
  mediaId: UUID
  legacyId: UUID
  eventId: UUID | null

  // 유형
  mediaType: PHOTO | VIDEO | AUDIO | DOCUMENT
  mimeType: String

  // 저장소
  originalUrl: URL
  thumbnailUrl: URL
  archiveUrl: URL

  // 메타데이터
  filename: String
  fileSize: Number
  dimensions: { width, height } | null

  // 설명
  caption: String
  tags: String[]

  // 날짜
  captureDate: Date | null
  uploadDate: DateTime

  // AI 분석
  aiTags: String[]
  petDetected: Boolean

  // 접근
  visibility: PRIVATE | FAMILY | MEMORIAL | PUBLIC
}

2.6 헌사 메시지

헌사 구조
=========

Tribute {
  tributeId: UUID
  legacyId: UUID

  // 작성자
  authorName: String
  authorEmail: String | null
  relationship: String

  // 내용
  message: Text
  media: MediaAttachment[]

  // 유형
  tributeType: MESSAGE | MEMORY | POEM | ARTWORK

  // 상태
  status: PENDING | APPROVED | REJECTED

  // 추모 선물
  offerings: VirtualOffering[]

  submittedAt: DateTime
  publishedAt: DateTime | null
}

VirtualOffering {
  offeringType: CANDLE | FLOWER | HEART | PAW_PRINT | RAINBOW
  message: String | null
  offeredAt: DateTime
}

2.7 접근 제어

접근 제어 구조
=============

LegacyAccessControl {
  legacyId: UUID
  primaryOwner: UUID

  familyMembers: [
    {
      userId: UUID
      relationship: String
      role: PRIMARY_OWNER | CO_OWNER | FAMILY_MEMBER
      accessLevel: FULL | EDIT | VIEW
    }
  ]

  inheritanceSettings: {
    successors: Successor[]      // 후계자 목록
    inactivityPeriod: Number     // 비활성 일수
    onInheritance: Action[]      // 상속 시 행동
  }
}

장 요약

  • [OK] 유산 프로필은 모든 반려동물 기억 데이터 포함

  • [OK] 생애 사건은 타임라인 이정표 기록
  • [OK] 미디어는 사진, 동영상, 문서 포함
  • [OK] 헌사는 사랑하는 이들의 메시지 보존
  • [OK] 접근 제어가 개인정보 보호
