Chapter 5: MoveIt 2 Motion Planning

5.1 MoveIt 2 Architecture

MoveIt 2 is the primary motion planning framework in the ROS 2 ecosystem. It provides a unified interface to multiple planning algorithms, inverse kinematics (IK) solvers, collision checking, and trajectory execution for manipulators and robot arms. WIA-ROBOT integrates MoveIt 2 for all robots with articulated joints requiring goal-based trajectory planning.

Key MoveIt 2 components used by WIA-ROBOT:

ComponentRole
MoveGroupInterfacePython/C++ client API for planning and execution
PlanningSceneMonitorReal-time collision world from /collision_object topic
OMPL Planning PipelineSampling-based planners (RRTConnect, PRM, BiTRRT)
STOMPStochastic gradient optimiser for smooth trajectories
PILZ Industrial MotionDeterministic LIN/PTP/CIRC profiles for industrial arms
KDL IK SolverDefault numerical IK; replaced by TRAC-IK or bio_ik for redundant robots
FCL Collision CheckerFlexible Collision Library for planning-time collision avoidance

5.2 Planning Group Configuration

MoveIt 2 requires a SRDF (Semantic Robot Description Format) file defining planning groups:

<robot name="wia_exoskeleton">
  <group name="lower_body">
    <chain base_link="pelvis" tip_link="right_foot"/>
  </group>
  <group name="left_leg">
    <joint name="hip_l"/>
    <joint name="knee_l"/>
    <joint name="ankle_l"/>
  </group>
  <group name="right_leg">
    <joint name="hip_r"/>
    <joint name="knee_r"/>
    <joint name="ankle_r"/>
  </group>
  <disable_collisions link1="thigh_l" link2="pelvis" reason="Adjacent"/>
  <disable_collisions link1="thigh_r" link2="pelvis" reason="Adjacent"/>
</robot>

5.3 OMPL Planners

The Open Motion Planning Library (OMPL) provides the default sampling-based planners in MoveIt 2. For WIA assistive robots, the recommended planners are:

PlannerUse CaseNotes
RRTConnectGeneral joint-space planningFast bidirectional search; default for exoskeletons
BiTRRTHigh-DOF redundant armsTemperature-based transition; more uniform exploration
LBKPIECENarrow-passage problemsLazy bidirectional KPIECE; good for confined rehab workspaces
PRM*Repeated queries in static environmentsBuild roadmap once; reuse for fast planning
# Python MoveIt 2 planning with RRTConnect
from moveit.core.robot_model import RobotModel
from moveit.planning import MoveItPy

moveit = MoveItPy(node_name="wia_planner")
robot = moveit.get_robot()
planning_component = moveit.get_planning_component("left_leg")

# Set goal state
planning_component.set_goal_state(configuration_name="stand_upright")

# Plan with OMPL RRTConnect
plan_params = moveit.get_default_plan_solution_parameters()
plan_params.planner_id = "RRTConnect"
plan_params.planning_time = 5.0
plan_params.num_planning_attempts = 3

plan_result = planning_component.plan(plan_params)
if plan_result:
    moveit.execute(plan_result.trajectory, controllers=["joint_trajectory_controller"])

5.4 STOMP — Stochastic Trajectory Optimisation

STOMP (Stochastic Trajectory Optimization for Motion Planning) generates smooth trajectories by iteratively improving a trajectory with stochastic gradient descent. It is preferred for rehabilitation robots where smooth, human-like motion is clinically important. 

# moveit_config/config/ompl_planning.yaml
planning_plugins:
  - moveit_planners_stomp/StompPlannerManager

stomp:
  group_name: left_leg
  num_timesteps: 60
  num_iterations: 200
  num_rollouts: 30
  delta_t: 0.05              # 50 ms time step
  noise_stddev: [0.05, 0.08, 0.03]  # per joint (hip, knee, ankle)
  cost_function:
    obstacle_cost_weight: 1.0
    smoothness_cost_weight: 0.1
    constraint_cost_weight: 5.0

5.5 PILZ Industrial Motion Profiles

PILZ Industrial Motion Planner provides deterministic, time-optimal trajectories conforming to industrial robot motion profiles — appropriate for surgical robots and precise assembly tasks:

  • PTP (Point-to-Point) — all joints reach goal simultaneously; minimum time.
  • LIN (Linear) — end-effector moves in a straight Cartesian line.
  • CIRC (Circular) — end-effector follows a circular arc; requires auxiliary point.
# PILZ LIN Cartesian move for surgical robot
from moveit_msgs.msg import MotionPlanRequest

req = MotionPlanRequest()
req.planner_id = "LIN"
req.group_name = "right_arm"
req.max_velocity_scaling_factor = 0.3   # 30% of rated velocity
req.max_acceleration_scaling_factor = 0.2

5.6 Collision Checking with FCL

The Flexible Collision Library (FCL) performs real-time collision checking against a 3D planning scene. WIA-ROBOT populates the planning scene with:

  • Static objects — room geometry loaded from a 3D map or SLAM-generated point cloud.
  • Dynamic objects — published as moveit_msgs/CollisionObject on /collision_object topic.
  • Attached objects — objects held by the robot (e.g. rehabilitation handle) as /attached_collision_object.
  • Octomap — 3D occupancy from depth camera, published as octomap_msgs/Octomap.

5.7 IK Solvers

The default KDL numerical IK solver struggles with redundant robots (DOF > 6). WIA-ROBOT recommends:

SolverStrengthsInstallation
TRAC-IKFast, reliable numerical IK with bounds; 90%+ success rateros-jazzy-trac-ik
bio_ikGoal functions beyond end-effector pose (e.g. minimal joint displacement, balance)bio_ik ROS 2 package
ikfastAnalytic IK, near-100% success; requires robot-specific code generationMoveIt IKFast plugin
