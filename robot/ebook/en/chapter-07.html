Chapter 7: NVIDIA Isaac Sim Integration

7.1 Isaac Sim Overview

NVIDIA Isaac Sim (based on Omniverse Kit and the USD Composer framework) provides a physically accurate robot simulation environment for WIA-ROBOT development, testing, and synthetic data generation. Isaac Sim integrates natively with ROS 2 via the isaacsim.ros2_bridge extension, exposing the same topics, services, and actions as a physical robot — enabling seamless sim-to-real transfer.

Key capabilities relevant to WIA assistive robot development:

  • Physics simulation — NVIDIA PhysX 5.3 rigid body dynamics; articulation drives for motorised joints.
  • Sensor simulation — synthetic LiDAR (RTX ray-traced), RGB-D cameras, IMU, contact sensors.
  • Synthetic data generation — domain randomisation for training object detection / gait analysis models.
  • Occupancy map generation — export Nav2-compatible maps from 3D hospital environment USD scenes.

7.2 ROS 2 Bridge Setup

The Isaac Sim ROS 2 bridge publishes and subscribes to ROS 2 topics within the same DDS domain as the physical robot test bench. After launching Isaac Sim, enable the bridge via the Extension Manager or programmatically: 

# Isaac Sim Python scripting (Omniverse Kit)
import omni.kit.app
from isaacsim.ros2_bridge import ROS2Bridge

bridge = ROS2Bridge()
bridge.start(
    ros_domain_id=200,       # Isaac Sim virtual robot domain
    robot_namespace="/wia_robot_sim",
    publish_joint_states=True,
    joint_state_hz=50,
    publish_clock=True       # /clock topic for use_sim_time=true
)
print("ROS 2 bridge active on domain 200")

7.3 URDF Import and Articulation

Import the WIA exoskeleton URDF into Isaac Sim using the URDF importer: 

from isaacsim.robot.import_utils import URDFImporter

importer = URDFImporter()
result = importer.import_robot(
    urdf_path="/opt/wia_robot/urdf/wia_exoskeleton.urdf.xacro",
    dest_path="/World/WIAExoskeleton",
    import_inertia_tensor=True,
    import_materials=True,
    convex_decomp=True,          # convex hull collision shapes
    fix_base=False,              # exoskeleton is mobile
    merge_fixed_joints=False
)
# Creates USD prim at /World/WIAExoskeleton with ArticulationRoot

7.4 Synthetic Training Data Generation

WIA-ROBOT uses Isaac Sim's domain randomisation pipeline (omni.replicator) to generate synthetic RGB-D datasets for training gait analysis and obstacle detection models: 

import omni.replicator.core as rep

with rep.new_layer():
    # Randomise robot pose and lighting
    robot = rep.get.prims(path_pattern="/World/WIAExoskeleton")
    lights = rep.create.light(light_type="Sphere", count=3)

    with rep.trigger.on_frame(num_frames=5000):
        rep.modify.pose(robot,
                        position=rep.distribution.uniform((-2,0,-2),(2,0,2)),
                        rotation=rep.distribution.uniform((0,-10,0),(0,10,360)))
        rep.modify.attribute(lights, "intensity",
                             rep.distribution.uniform(500, 3000))

rep.orchestrator.run()
# Output: 5000 frames with ground-truth joint angles and depth images

7.6 Isaac Sim Hospital Environment USD

WIA provides a pre-built hospital ward USD scene for testing mobility-aid robot navigation: 

# Load WIA hospital ward USD scene
import omni.usd

stage = omni.usd.get_context().get_stage()
stage.GetRootLayer().subLayerPaths.append(
    "omniverse://hospital-assets.wia.global/WIA/HospitalWard_v2.usd"
)

# Generate Nav2-compatible 2D occupancy map from the 3D scene
from isaacsim.nav2_utils import OccupancyMapGenerator

gen = OccupancyMapGenerator(resolution=0.05)
gen.generate_from_scene(
    output_yaml="/opt/wia_robot/maps/isaac_ward.yaml",
    output_pgm="/opt/wia_robot/maps/isaac_ward.pgm",
    height=1.0,    # slice plane at 1 m above floor
    free_thresh=0.25,
    occupied_thresh=0.65
)
