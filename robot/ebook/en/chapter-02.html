Chapter 2: Data Formats and Robot Models

2.1 URDF — Unified Robot Description Format

The URDF (Unified Robot Description Format) is the canonical XML schema for robot kinematic and dynamic models in ROS 2. A URDF file defines links (rigid bodies) and joints (kinematic constraints between links), enabling collision checking, visualisation in RViz, and kinematic planning in MoveIt 2. 

<robot name="wia_exoskeleton">
  <link name="pelvis">
    <inertial>
      <mass value="5.0"/>
      <inertia ixx="0.08" ixy="0" ixz="0" iyy="0.06" iyz="0" izz="0.04"/>
    </inertial>
    <visual>
      <geometry><box size="0.35 0.25 0.1"/></geometry>
      <material name="wia_orange"><color rgba="0.976 0.451 0.086 1"/></material>
    </visual>
    <collision><geometry><box size="0.35 0.25 0.1"/></geometry></collision>
  </link>

  <joint name="hip_l" type="revolute">
    <parent link="pelvis"/>
    <child link="thigh_l"/>
    <origin xyz="-0.1 -0.12 -0.05" rpy="0 0 0"/>
    <axis xyz="0 1 0"/>
    <limit lower="-0.35" upper="1.0" effort="120" velocity="1.5"/>
    <dynamics damping="2.0" friction="0.5"/>
  </joint>
</robot>

2.2 XACRO — XML Macros for Robot Models

XACRO (XML Macros) extends URDF with parametric macros, enabling reuse of leg/arm templates across left/right sides and robot variants. WIA-ROBOT mandates XACRO for all production robot descriptions. 

<xacro:macro name="leg" params="side reflect">
  <link name="thigh_${side}">...</link>
  <link name="shank_${side}">...</link>
  <joint name="knee_${side}" type="revolute">
    <parent link="thigh_${side}"/>
    <child link="shank_${side}"/>
    <origin xyz="0 0 -0.42" rpy="0 0 0"/>
    <axis xyz="0 1 0"/>
    <limit lower="-1.57" upper="0.1" effort="100" velocity="2.0"/>
  </joint>
</xacro:macro>

<xacro:leg side="l" reflect="1"/>
<xacro:leg side="r" reflect="-1"/>

2.3 Joint State Schema (sensor_msgs/JointState)

The primary sensor data type for robot joint feedback is sensor_msgs/msg/JointState. WIA-ROBOT publishes joint states at up to 50 Hz on the /joint_states topic with BEST_EFFORT QoS. 

# sensor_msgs/msg/JointState
std_msgs/Header header
string[]  name        # joint names matching URDF
float64[] position    # radians (revolute) or meters (prismatic)
float64[] velocity    # rad/s or m/s
float64[] effort      # N·m or N

The WIA Phase 1 JSON serialisation wraps this in a typed envelope with: robot_id (UUID), seq (monotonic counter for packet-loss detection), and a safety object embedding E-Stop and watchdog state.

2.4 IMU Data (sensor_msgs/Imu)

Inertial measurement unit data, essential for balance control in exoskeletons, is published as sensor_msgs/msg/Imu. A covariance value of -1 in position [0] indicates the field is not populated. Kalman-filtered orientation (from imu_filter_madgwick ROS 2 package) feeds directly into the Centre-of-Mass controller. 

# sensor_msgs/msg/Imu
std_msgs/Header header
geometry_msgs/Quaternion orientation
float64[9] orientation_covariance       # row-major 3x3
geometry_msgs/Vector3 angular_velocity
float64[9] angular_velocity_covariance
geometry_msgs/Vector3 linear_acceleration
float64[9] linear_acceleration_covariance

2.5 Force-Torque Data (ISO 15066)

ISO 15066:2016 collaborative robot safety requires real-time force monitoring. WIA-ROBOT publishes geometry_msgs/msg/WrenchStamped at 1 kHz from six-axis F/T sensors. Contact force thresholds per ISO 15066 Table A.1: body region 3 (chest/back) — maximum transient contact force 140 N; quasi-static force 65 N. The WIA safety zone controller triggers Category 0 stop if these limits are exceeded. 

# geometry_msgs/msg/WrenchStamped
std_msgs/Header header
geometry_msgs/Wrench wrench
  geometry_msgs/Vector3 force   # Fx, Fy, Fz (Newtons)
  geometry_msgs/Vector3 torque  # Tx, Ty, Tz (N·m)

2.6 EMG / Prosthetic Control Signals

For prosthetic and exoskeleton control driven by surface electromyography (sEMG), WIA-ROBOT defines a custom message type wia_robot_msgs/msg/EmgArray. The standard mandates a 2 kHz minimum sampling rate to support pattern recognition classifiers using the Hudgins TD4 feature set (MAV, ZC, SSC, WL). 

std_msgs/Header header
uint8 num_channels           # 8 (Thalmic Myo) or 16 (Delsys Trigno)
float32[] raw_uv             # raw EMG in microvolts
float32[] rms_uv             # 200 ms RMS window
float32[] mav                # mean absolute value window
string[]  electrode_labels   # ["FCR","ECR","BB","TB","FDS"...]

2.7 FHIR R5 Health Data Integration

Clinical outcomes from robot-assisted rehabilitation are represented using HL7 FHIR R5 resources. Key resources used by WIA-ROBOT:

FHIR ResourceUsage
ObservationGait metrics (cadence, stride length, symmetry index), force measurements, EMG quality
ProcedureRehabilitation session records linked to robot device
DeviceRobot registration with FDA UDI, model, manufacturer, software version
DeviceMetricReal-time joint angle and torque observables
