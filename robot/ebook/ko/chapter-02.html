제2장: 데이터 형식과 로봇 모델

2.1 URDF — 통합 로봇 기술 형식

URDF(Unified Robot Description Format)는 ROS 2에서 로봇의 기구학·동역학 모델을 기술하는 표준 XML 스키마입니다. 링크(강체)와 관절(운동학적 구속 조건)로 구성되며, MoveIt 2 충돌 검사, RViz 시각화, 동작 계획에 사용됩니다. 

<robot name="wia_exoskeleton">
  <link name="pelvis">
    <inertial>
      <mass value="5.0"/>
      <inertia ixx="0.08" ixy="0" ixz="0" iyy="0.06" iyz="0" izz="0.04"/>
    </inertial>
    <visual>
      <geometry><box size="0.35 0.25 0.1"/></geometry>
      <material name="wia_orange"><color rgba="0.976 0.451 0.086 1"/></material>
    </visual>
  </link>

  <joint name="hip_l" type="revolute">
    <parent link="pelvis"/>
    <child link="thigh_l"/>
    <axis xyz="0 1 0"/>
    <limit lower="-0.35" upper="1.0" effort="120" velocity="1.5"/>
    <dynamics damping="2.0" friction="0.5"/>
  </joint>
</robot>

2.2 XACRO — XML 매크로 확장

XACRO는 매개변수화된 매크로로 URDF를 확장해 좌우 다리/팔 템플릿 재사용을 가능하게 합니다. 국내 외골격 제조사(큐렉소, 현대로보틱스)의 모델은 XACRO 기반으로 WIA 표준 저장소에 기여될 수 있습니다. 

<xacro:macro name="leg" params="side reflect">
  <link name="thigh_${side}">...</link>
  <link name="shank_${side}">...</link>
  <joint name="knee_${side}" type="revolute">
    <parent link="thigh_${side}"/>
    <child link="shank_${side}"/>
    <axis xyz="0 1 0"/>
    <limit lower="-1.57" upper="0.1" effort="100" velocity="2.0"/>
  </joint>
</xacro:macro>

<xacro:leg side="l" reflect="1"/>
<xacro:leg side="r" reflect="-1"/>

2.3 관절 상태 스키마 (sensor_msgs/JointState)

로봇 관절 피드백의 기본 ROS 2 메시지 타입은 sensor_msgs/msg/JointState입니다. WIA-ROBOT은 BEST_EFFORT QoS로 최대 50 Hz에서 /joint_states 토픽에 발행합니다. 

std_msgs/Header header
string[]  name        # URDF와 일치하는 관절 이름
float64[] position    # 라디안 (회전 관절) / 미터 (직선 관절)
float64[] velocity    # rad/s 또는 m/s
float64[] effort      # N·m 또는 N

WIA Phase 1 JSON 직렬화는 이를 robot_id(UUID), seq(단조 증가 카운터), safety(비상 정지·워치독 상태) 객체로 감싸는 타입 있는 봉투 형식으로 제공합니다.

2.4 IMU 데이터 (sensor_msgs/Imu)

외골격의 균형 제어에 필수적인 관성 측정 장치(IMU) 데이터는 sensor_msgs/msg/Imu로 발행됩니다. 국내 외골격 제품에서는 Xsens MTi-3, InvenSense ICM-42688-P 등이 주로 사용되며, imu_filter_madgwick ROS 2 패키지로 쿼터니언 방향을 필터링합니다. 

std_msgs/Header header
geometry_msgs/Quaternion orientation
float64[9] orientation_covariance   # 위치[0]=-1이면 미제공
geometry_msgs/Vector3 angular_velocity
geometry_msgs/Vector3 linear_acceleration

2.5 힘/토크 데이터 (ISO 15066)

협동 로봇 안전(ISO 15066)을 위해 WIA-ROBOT은 6축 힘/토크 센서에서 1 kHz로 geometry_msgs/msg/WrenchStamped를 발행합니다. ISO 15066 표 A.1 접촉 한계: 흉부·등 — 과도 접촉력 140 N, 준정적 65 N. 이 한계 초과 시 Category 0 정지 트리거. 

geometry_msgs/Wrench wrench
  geometry_msgs/Vector3 force   # Fx, Fy, Fz (뉴턴)
  geometry_msgs/Vector3 torque  # Tx, Ty, Tz (N·m)

2.6 근전도(sEMG) / 의수 제어 신호

표면 근전도(sEMG) 기반 의수·외골격 제어를 위해 WIA-ROBOT은 커스텀 메시지 wia_robot_msgs/msg/EmgArray를 정의합니다. 국내 연구기관(서울대 바이오로보틱스 연구실, KAIST 로봇학과)이 사용하는 Delsys Trigno IM 시스템(16채널, 2 kHz)과 호환됩니다. Hudgins(1993) 특징 집합(MAV, ZC, SSC, WL) 기반 패턴 인식 분류기를 위해 최소 2 kHz 샘플링 레이트를 권장합니다. 

uint8 num_channels           # 8 (Thalmic) 또는 16 (Delsys Trigno)
float32[] raw_uv             # 원시 근전도 (마이크로볼트)
float32[] rms_uv             # 200 ms RMS 윈도우
float32[] mav                # 평균 절댓값 윈도우
string[]  electrode_labels   # ["FCR","ECR","BB","TB","FDS"...]

2.7 FHIR R5 임상 데이터 연동

로봇 보조 재활의 임상 결과는 HL7 FHIR R5 리소스로 표현됩니다. 국내 보건의료 디지털 정책의 큰 흐름은 표준 기반 EMR 상호운용성 확보이며, FHIR(Fast Healthcare Interoperability Resources)는 그 핵심 후보 표준으로 자리잡고 있습니다. 대형 병원의 EMR도 점차 FHIR API를 지원하는 방향으로 발전하고 있습니다. 

{
  "resourceType": "Observation",
  "status": "final",
  "code": {"coding": [{"system": "http://loinc.org",
                        "code": "80316-3",
                        "display": "보행 속도 [거리/시간]"}]},
  "subject": {"reference": "Patient/환자-001"},
  "device": {"reference": "Device/wia-exoskeleton-042"},
  "valueQuantity": {"value": 0.72, "unit": "m/s"}
}
