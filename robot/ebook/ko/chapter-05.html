제5장: MoveIt 2 동작 계획

5.1 MoveIt 2 아키텍처

MoveIt 2는 ROS 2 생태계의 기본 동작 계획 프레임워크입니다. 다양한 계획 알고리즘, 역기구학(IK) 솔버, 충돌 검사, 관절형 로봇의 궤적 실행을 위한 통합 인터페이스를 제공합니다. WIA-ROBOT은 목표 기반 궤적 계획이 필요한 모든 관절형 로봇에 MoveIt 2를 통합합니다.

WIA-ROBOT이 사용하는 MoveIt 2 핵심 컴포넌트:

컴포넌트역할
MoveGroupInterfacePython/C++ 계획·실행 클라이언트 API
PlanningSceneMonitor/collision_object 토픽에서 실시간 충돌 세계 구성
OMPL 계획 파이프라인샘플링 기반 계획기 (RRTConnect, PRM, BiTRRT)
STOMP부드러운 궤적을 위한 확률적 기울기 최적화기
PILZ Industrial Motion산업용 팔을 위한 결정론적 LIN/PTP/CIRC 프로파일
KDL IK 솔버기본 수치 IK; 과잉 구속 로봇에는 TRAC-IK로 교체
FCL 충돌 검사기Flexible Collision Library 계획 시간 충돌 회피

5.2 계획 그룹 설정

MoveIt 2는 SRDF(Semantic Robot Description Format) 파일에서 계획 그룹을 정의해야 합니다:

<robot name="wia_exoskeleton">
  <group name="lower_body">
    <chain base_link="pelvis" tip_link="right_foot"/>
  </group>
  <group name="left_leg">
    <joint name="hip_l"/>
    <joint name="knee_l"/>
    <joint name="ankle_l"/>
  </group>
  <disable_collisions link1="thigh_l" link2="pelvis" reason="Adjacent"/>
</robot>

5.3 OMPL 계획기

OMPL(Open Motion Planning Library)은 MoveIt 2의 기본 샘플링 기반 계획기를 제공합니다. WIA 보조 로봇에 권장되는 계획기:

계획기사용 사례비고
RRTConnect일반 관절 공간 계획빠른 양방향 탐색; 외골격 기본값
BiTRRT고자유도 과잉 구속 팔온도 기반 전이; 더 균일한 탐색
LBKPIECE좁은 통로 문제지연 양방향 KPIECE; 재활 작업 공간
PRM*정적 환경에서 반복 쿼리로드맵 한 번 구축 후 재사용
# Python MoveIt 2 계획 예시
from moveit.planning import MoveItPy

moveit = MoveItPy(node_name="wia_planner")
planning_component = moveit.get_planning_component("left_leg")

planning_component.set_goal_state(configuration_name="stand_upright")

plan_params = moveit.get_default_plan_solution_parameters()
plan_params.planner_id = "RRTConnect"
plan_params.planning_time = 5.0

plan_result = planning_component.plan(plan_params)
if plan_result:
    moveit.execute(plan_result.trajectory,
                   controllers=["joint_trajectory_controller"])

5.4 STOMP — 확률적 궤적 최적화

STOMP(Stochastic Trajectory Optimization for Motion Planning)는 확률적 경사 하강법으로 궤적을 반복 개선하여 부드러운 궤적을 생성합니다. 임상적으로 자연스러운 동작이 중요한 재활 로봇에 적합합니다. 다수 비교 연구에서 STOMP 기반 궤적은 RRTConnect 대비 관절 가속도 진동(jerk) 측면에서 유의하게 부드러운 결과를 보이는 것으로 보고됩니다(Kalakrishnan et al. 2011 *ICRA*). 

# STOMP 설정 (moveit_config/config/ompl_planning.yaml)
planning_plugins:
  - moveit_planners_stomp/StompPlannerManager
stomp:
  group_name: left_leg
  num_timesteps: 60
  num_iterations: 200
  num_rollouts: 30
  delta_t: 0.05         # 50 ms 시간 단계
  noise_stddev: [0.05, 0.08, 0.03]  # 관절별 (고관절, 무릎, 발목)

5.5 PILZ 산업 동작 프로파일

PILZ Industrial Motion Planner는 수술 로봇과 정밀 조립 작업에 적합한 결정론적·시간 최적 궤적을 제공합니다:

  • PTP(Point-to-Point) — 모든 관절이 동시에 목표 도달; 최소 시간.
  • LIN(직선) — 엔드 이펙터가 데카르트 직선 이동.
  • CIRC(원호) — 엔드 이펙터가 원호 경로 이동; 보조점 필요.

7-DOF 이상의 여유 자유도를 가진 재활 로봇팔에서는 TRAC-IK가 KDL의 단순 Newton-Raphson 대비 IK 성공률을 큰 폭으로 향상시키는 것으로 비교 연구에서 보고됩니다(Beeson & Ames 2015 *IEEE-RAS Humanoids*).
