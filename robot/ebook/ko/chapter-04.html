제4장: DDS 통신 프로토콜

4.1 DDS/RTPS 개요

ROS 2는 OMG DDS(Data Distribution Service) 표준과 유선 프로토콜 RTPS(Real-Time Publish-Subscribe, RTPS 2.5)를 기본 미들웨어로 사용합니다. DDS는 데이터 중심 발행/구독(DCPS) 모델에 선언적 QoS 정책을 제공하여 안전 임계 로봇 시스템의 전달 신뢰성, 지연 시간, 리소스 사용을 세밀하게 제어할 수 있습니다.

WIA-ROBOT 지원 RMW(ROS 미들웨어) 구현체:

RMWDDS 구현체용도
rmw_fastrtps_cppeProsima FastDDS 3.x기본값; Linux에서 최고 성능
rmw_cyclonedds_cppEclipse Cyclone DDS 0.10저지연; 임베디드 시스템 적합
rmw_connextddsRTI Connext DDS 7.x엔터프라이즈; DO-178C 항공 인증 지원

4.2 QoS 프로파일

WIA-ROBOT은 세 가지 QoS 프로파일 계층을 정의합니다:

Tier 1 — 안전 임계 명령

적용: /emergency_stop, /wia_robot/execute_trajectory 액션 목표, /robot_description

qos_commands:
  reliability: RELIABLE           # 보장 전달, 손실 시 재전송
  durability: TRANSIENT_LOCAL     # 지연 참여자에게 최근 N개 메시지 전달
  history: KEEP_LAST
  depth: 10
  deadline: {nsec: 100000000}     # 100 ms — ISO 10218-1 응답 예산
  liveliness_lease_duration: {nsec: 500000000}  # 500 ms 워치독

Tier 2 — 고속 텔레메트리

적용: /joint_states(50 Hz), /imu/data_raw, /wrench(1 kHz)

qos_telemetry_fast:
  reliability: BEST_EFFORT        # 오래된 프레임 폐기; 지연 최소화
  durability: VOLATILE
  history: KEEP_LAST
  depth: 1
  lifespan: {nsec: 25000000}      # 25 ms 후 폐기 (50 Hz에서 2프레임)

Tier 3 — 지도/정적 상태

적용: /map(SLAM 점유 격자), /tf_static

qos_map_static:
  reliability: RELIABLE
  durability: TRANSIENT_LOCAL
  history: KEEP_LAST
  depth: 1

4.4 FastDDS 3.x XML 설정

<!-- FASTRTPS_DEFAULT_PROFILES_FILE=/opt/wia_robot/dds/wia_profiles.xml -->
<profiles>
  <participant profile_name="wia_safety" is_default_participant="true">
    <rtps>
      <builtin>
        <discovery_config>
          <leaseDuration><sec>DURATION_INFINITY</sec></leaseDuration>
          <initial_announcements>
            <count>5</count>
            <period><nanosec>100000000</nanosec></period>
          </initial_announcements>
        </discovery_config>
      </builtin>
      <sendBufferSize>1048576</sendBufferSize>
      <receiveBufferSize>4194304</receiveBufferSize>
    </rtps>
  </participant>
</profiles>

4.5 MQTT 5 병원 시스템 브리지

병원 EMR 및 간호사 호출 시스템은 전체 ROS 2 스택을 실행하기 어렵습니다. WIA-ROBOT MQTT 5 브리지(wia_robot_mqtt_bridge)는 중요 이벤트를 MQTT 5 브로커로 재발행하여 표준 병원 네트워크 인프라로 임상 대시보드를 연결합니다. 

bridge_config:
  mqtt_broker: mqtts://hospital-iot.internal:8883
  subscriptions:
    - ros2_topic: /emergency_stop
      mqtt_topic: wia/robot/{robot_id}/estop
      qos: 2        # MQTT QoS 2 = 정확히 한 번 (RELIABLE DDS와 매핑)
    - ros2_topic: /joint_states
      mqtt_topic: wia/robot/{robot_id}/joints
      qos: 0        # MQTT QoS 0 = 최대 한 번 (BEST_EFFORT DDS와 매핑)
      rate_limit_hz: 5   # 임상 대시보드용 50 Hz → 5 Hz 다운샘플

4.6 DDS 발견 문제 해결 (병원 네트워크)

병원 네트워크는 보안 정책상 UDP 멀티캐스트를 차단하는 경우가 많습니다. 이 경우 유니캐스트 전용 FastDDS 프로파일을 사용합니다: 

# 멀티캐스트 차단 환경에서 유니캐스트 강제
export RMW_FASTRTPS_USE_QOS_FROM_XML=1
export FASTRTPS_DEFAULT_PROFILES_FILE=/opt/wia_robot/dds/unicast_profiles.xml

# 진단 명령
ros2 topic hz /joint_states      # 예상: ~50 Hz
ros2 topic info /joint_states --verbose  # QoS 확인
ros2 daemon status               # ROS 2 데몬 상태 확인
