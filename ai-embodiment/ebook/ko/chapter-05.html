5.1 REST API Endpoints

The WIA AI-EMBODIMENT API provides comprehensive control over embodied AI systems:

Base URL

https://api.wiastandards.com/ai-embodiment/v1

Core Endpoints

Method Endpoint Description
GET /status Get system status
GET /state Get complete state
POST /command Send motor command
POST /emergency-stop Immediate halt
GET /sensors/{id} Get sensor data
GET /motors List all motors
PUT /motors/{id} Update motor config
GET /safety Get safety status
PUT /safety/config Update safety config
GET /ethics/check Ethics validation

5.2 WebSocket Real-time Streams

Connection


WebSocket URL: wss://api.wiastandards.com/ai-embodiment/v1/stream

// JavaScript Example
const ws = new WebSocket('wss://api.wiastandards.com/ai-embodiment/v1/stream');

ws.onopen = () => {
  // Subscribe to channels
  ws.send(JSON.stringify({
    action: 'subscribe',
    channels: ['state', 'sensors', 'safety', 'commands']
  }));
};

ws.onmessage = (event) => {
  const data = JSON.parse(event.data);
  console.log('Channel:', data.channel, 'Data:', data.payload);
};

Stream Channels

Channel Update Rate Description
state 10 Hz Complete system state
sensors 30-100 Hz Raw sensor data
safety 100 Hz Safety status updates
commands On event Command status updates
events On event System events, alerts

5.3 Authentication & Authorization

JWT Token Structure


{
  "header": {
    "alg": "RS256",
    "typ": "JWT"
  },
  "payload": {
    "sub": "user-uuid",
    "iss": "wiastandards.com",
    "aud": "ai-embodiment",
    "exp": 1734567890,
    "iat": 1734564290,
    "permissions": [
      "read:state",
      "write:commands",
      "read:sensors",
      "write:safety:config"
    ],
    "embodiment_ids": ["emb-001", "emb-002"],
    "safety_level": "operator"
  }
}

Permission Levels

Level Permissions Use Case
viewer read:state, read:sensors Monitoring only
operator + write:commands Normal operation
supervisor + write:safety:config Safety management
admin Full access System administration

5.5 Error Handling

Error Response Format


{
  "error": {
    "code": "SAFETY_CONSTRAINT_VIOLATED",
    "message": "Command rejected: human detected in path",
    "details": {
      "constraint": "human_proximity",
      "required_distance": 1.0,
      "actual_distance": 0.5,
      "human_position": { "x": 3.2, "y": 1.1, "z": 0 }
    },
    "timestamp": "2025-12-19T15:30:00.000Z",
    "request_id": "req-uuid-12345",
    "remediation": "Wait for human to move or reduce target distance"
  }
}

Error Codes

Code HTTP Description
SAFETY_CONSTRAINT_VIOLATED 403 Safety check failed
ETHICS_CHECK_FAILED 403 Ethics validation failed
MOTOR_LIMIT_EXCEEDED 400 Command exceeds limits
CONNECTION_LOST 503 Body connection lost
UNAUTHORIZED 401 Invalid/expired token
