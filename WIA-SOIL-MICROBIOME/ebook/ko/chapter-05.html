🌐 RESTful API 설계

REST API를 선택한 이유

WIA-SOIL-MICROBIOME API는 REST (Representational State Transfer) 아키텍처를 따릅니다. REST는 웹의 표준 프로토콜인 HTTP를 사용하며, 다음과 같은 장점이 있습니다:

  • 단순성: HTTP 메서드(GET, POST, PUT, DELETE)를 직관적으로 사용
  • 무상태성: 각 요청이 독립적이어서 확장성이 뛰어남
  • 캐싱 가능: HTTP 캐싱 메커니즘 활용 가능
  • 도구 지원: 모든 프로그래밍 언어와 플랫폼에서 지원
  • 방화벽 친화적: 표준 HTTP 포트(80, 443) 사용

API 기본 구조

모든 API 엔드포인트는 다음 기본 URL 구조를 따릅니다: 

https://api.wia-soil-microbiome.org/v1/{resource}
  • HTTPS: 모든 통신은 암호화됩니다
  • api 서브도메인: API 전용 도메인으로 분리
  • 버전 (/v1): API 버전을 URL에 포함하여 하위 호환성 유지
  • 리소스: samples, datasets, projects 등

HTTP 메서드 사용

메서드 용도 예시
GET 리소스 조회 GET /v1/samples/{id}
POST 리소스 생성 POST /v1/samples
PUT 리소스 전체 업데이트 PUT /v1/samples/{id}
PATCH 리소스 부분 업데이트 PATCH /v1/samples/{id}
DELETE 리소스 삭제 DELETE /v1/samples/{id}

📡 핵심 API 엔드포인트

1. 샘플 관리 API

POST /v1/samples

설명: 새로운 토양 샘플 데이터를 등록합니다.

요청 예시:

POST /v1/samples
Content-Type: application/json
Authorization: Bearer {access_token}

{
  "id": "a1b2c3d4-e5f6-4a5b-8c7d-9e0f1a2b3c4d",
  "version": "1.0",
  "timestamp": "2025-06-15T09:30:00+09:00",
  "collector": {
    "name": "김철수",
    "organization": "녹색농장"
  },
  "sample_type": "agricultural_soil",
  ...
}

응답 예시:

HTTP/1.1 201 Created
Location: /v1/samples/a1b2c3d4-e5f6-4a5b-8c7d-9e0f1a2b3c4d

{
  "id": "a1b2c3d4-e5f6-4a5b-8c7d-9e0f1a2b3c4d",
  "status": "created",
  "url": "https://api.wia-soil-microbiome.org/v1/samples/a1b2c3d4...",
  "created_at": "2025-06-15T09:30:15Z"
}
GET /v1/samples/{id}

설명: 특정 샘플의 전체 데이터를 조회합니다.

요청 예시:

GET /v1/samples/a1b2c3d4-e5f6-4a5b-8c7d-9e0f1a2b3c4d
Accept: application/json

응답: 전체 샘플 JSON 데이터 반환

GET /v1/samples

설명: 샘플 목록을 검색하고 필터링합니다.

쿼리 파라미터:

  • location: 위치 필터 (bbox 또는 point+radius)
  • date_from, date_to: 날짜 범위
  • land_use: 토지 이용 유형
  • min_quality_level: 최소 데이터 품질 수준
  • limit: 결과 개수 (기본 20, 최대 100)
  • offset: 페이지네이션 오프셋

요청 예시:

GET /v1/samples?location=bbox:126.5,37.0,128.0,38.0&date_from=2025-01-01&limit=50

응답 예시:

{
  "total": 1250,
  "limit": 50,
  "offset": 0,
  "samples": [
    {
      "id": "...",
      "timestamp": "2025-06-15T09:30:00Z",
      "location": {...},
      "quality_level": 3,
      "url": "https://api.wia-soil-microbiome.org/v1/samples/..."
    },
    ...
  ],
  "links": {
    "self": "/v1/samples?...",
    "next": "/v1/samples?...&offset=50",
    "prev": null
  }
}

🔐 인증과 권한 관리

OAuth 2.0 인증

WIA-SOIL-MICROBIOME API는 OAuth 2.0 프로토콜을 사용하여 안전한 인증을 제공합니다.

1. 액세스 토큰 발급

POST /v1/auth/token
Content-Type: application/x-www-form-urlencoded

grant_type=client_credentials&
client_id={your_client_id}&
client_secret={your_client_secret}

응답:

{
  "access_token": "eyJhbGciOiJSUzI1NiIsInR5cCI6IkpXVCJ9...",
  "token_type": "Bearer",
  "expires_in": 3600,
  "scope": "read write"
}

2. API 요청 시 토큰 사용

GET /v1/samples
Authorization: Bearer eyJhbGciOiJSUzI1NiIsInR5cCI6IkpXVCJ9...

권한 수준

권한 설명 허용 작업
public 공개 데이터 읽기 GET (공개 샘플), 통계 조회
read 모든 데이터 읽기 GET (모든 샘플), 검색, 다운로드
write 데이터 쓰기 POST, PUT (자신의 데이터)
admin 관리자 모든 작업, 타인 데이터 수정/삭제

데이터 접근 제어

샘플 데이터는 다음 접근 수준을 가질 수 있습니다:

  • public: 누구나 접근 가능
  • registered: 등록된 사용자만 접근
  • restricted: 특정 그룹만 접근
  • private: 소유자만 접근

데이터 공유 권장사항:

WIA-SOIL-MICROBIOME의 목표는 글로벌 협력이므로, 가능한 한 데이터를 공개(public) 또는 등록 사용자 공개(registered)로 설정하는 것을 권장합니다. 민감한 정보는 위치 정밀도 조절로 보호할 수 있습니다.

📊 페이지네이션과 필터링

커서 기반 페이지네이션

대규모 데이터셋의 효율적인 탐색을 위해 커서 기반 페이지네이션을 지원합니다: 

GET /v1/samples?limit=100&cursor=eyJpZCI6Imxhc3RfaWQiLCJ0aW1lc3RhbXAiOiIyMDI1LTA2LTE1In0=

{
  "samples": [...],
  "pagination": {
    "next_cursor": "eyJpZCI6Im5leHRfaWQiLCJ0aW1lc3RhbXAiOiIyMDI1LTA2LTE2In0=",
    "has_more": true
  }
}

고급 필터링

지리적 필터링:

# Bounding box
?location=bbox:126.5,37.0,128.0,38.0

# 반경 검색
?location=point:127.0,37.5,radius:50km

# 국가
?country=KR

# 여러 국가
?country=KR,JP,CN

시간 필터링:

# 날짜 범위
?date_from=2025-01-01&date_to=2025-06-30

# 최근 데이터
?recent=30days

# 특정 연도
?year=2025

환경 필터링:

# pH 범위
?ph_min=6.0&ph_max=7.5

# 토지 이용
?land_use=organic_farming,conventional_farming

# 관리 실천
?management=no_till,cover_crops

품질 필터링:

# 최소 품질 수준
?min_quality_level=3

# 특정 데이터 포함
?has_taxonomy=true&has_function=true

정렬

# 날짜 순 (기본: 최신순)
?sort=timestamp:desc

# 여러 필드
?sort=timestamp:desc,quality_level:desc

# 다양성 지수
?sort=diversity.shannon:desc

⚡ 성능 최적화

필드 선택 (Sparse Fieldsets)

필요한 필드만 요청하여 전송 데이터 크기를 줄입니다: 

GET /v1/samples/{id}?fields=id,timestamp,location,environment.soil.ph

# 응답에 요청한 필드만 포함됩니다

데이터 압축

gzip 압축을 지원하여 네트워크 전송량을 70-90% 감소시킵니다: 

GET /v1/samples
Accept-Encoding: gzip

# 서버가 압축된 응답 반환
Content-Encoding: gzip

캐싱

HTTP 캐싱 헤더를 사용하여 반복 요청을 최적화합니다: 

# 서버 응답
Cache-Control: public, max-age=3600
ETag: "a1b2c3d4e5f6"

# 클라이언트 재요청 시
If-None-Match: "a1b2c3d4e5f6"

# 변경 없으면
HTTP/1.1 304 Not Modified

비동기 처리

시간이 오래 걸리는 작업(대규모 분석, 데이터 다운로드)은 비동기로 처리합니다: 

POST /v1/export/bulk
{
  "query": {...},
  "format": "csv"
}

# 즉시 응답
HTTP/1.1 202 Accepted
{
  "job_id": "export-12345",
  "status_url": "/v1/jobs/export-12345"
}

# 나중에 상태 확인
GET /v1/jobs/export-12345
{
  "status": "completed",
  "download_url": "https://downloads.wia.../export-12345.csv.gz",
  "expires_at": "2025-06-16T10:00:00Z"
}

🛡️ 에러 처리

표준 HTTP 상태 코드

코드 의미 사용 예
200 OK 성공 GET, PUT 성공
201 Created 생성 완료 POST로 새 리소스 생성
204 No Content 성공, 응답 없음 DELETE 성공
400 Bad Request 잘못된 요청 유효하지 않은 JSON
401 Unauthorized 인증 필요 토큰 없음 또는 만료
403 Forbidden 권한 없음 접근 권한 부족
404 Not Found 리소스 없음 존재하지 않는 ID
429 Too Many Requests 요청 과다 Rate limit 초과
500 Internal Server Error 서버 오류 예상치 못한 오류

Rate Limiting

API 남용을 방지하기 위해 요청 횟수 제한을 적용합니다: 

# 응답 헤더
X-RateLimit-Limit: 1000
X-RateLimit-Remaining: 987
X-RateLimit-Reset: 1718452800

# 제한 초과 시
HTTP/1.1 429 Too Many Requests
Retry-After: 3600

{
  "error": {
    "code": "RATE_LIMIT_EXCEEDED",
    "message": "시간당 요청 한도를 초과했습니다. 1시간 후 재시도하세요."
  }
}

Rate Limit 기준:

  • 익명 (public): 100 요청/시간
  • 등록 사용자: 1,000 요청/시간
  • 프리미엄: 10,000 요청/시간
  • 대량 분석: 별도 협의

📚 SDK와 클라이언트 라이브러리

공식 SDK

WIA-SOIL-MICROBIOME은 주요 프로그래밍 언어용 SDK를 제공합니다:

TypeScript/JavaScript SDK:

import { WIASoilClient } from '@wia/soil-microbiome';

const client = new WIASoilClient({
  apiKey: process.env.WIA_API_KEY
});

// 샘플 업로드
const sample = await client.samples.create({
  id: '...',
  timestamp: new Date(),
  collector: {...},
  ...
});

// 샘플 검색
const results = await client.samples.search({
  location: { bbox: [126.5, 37.0, 128.0, 38.0] },
  dateFrom: '2025-01-01',
  limit: 50
});

// 다양성 분석
const job = await client.analysis.diversity({
  sampleIds: ['id1', 'id2', 'id3'],
  method: 'bray_curtis'
});

const result = await job.wait();

🎯 주요 내용

  • WIA-SOIL-MICROBIOME API는 REST 아키텍처를 따르며, HTTPS를 통해 안전하게 통신하고 버전 관리(/v1)로 하위 호환성을 유지합니다. HTTP 메서드(GET, POST, PUT, DELETE)를 직관적으로 사용합니다.
  • 핵심 API는 샘플 관리, 데이터셋 관리, 분석 요청, 통계 조회로 구성되며, 각 엔드포인트는 명확한 역할과 입출력 형식을 가집니다. 비동기 처리로 대규모 작업을 효율적으로 처리합니다.
  • OAuth 2.0 기반 인증으로 안전한 접근을 제공하며, public/read/write/admin 4단계 권한 수준으로 세밀한 접근 제어를 합니다. 데이터는 public/registered/restricted/private 접근 수준을 가질 수 있습니다.
  • 커서 기반 페이지네이션, 지리적/시간적/환경적 고급 필터링, 필드 선택, gzip 압축, HTTP 캐싱으로 성능을 최적화합니다. Rate limiting으로 API 남용을 방지합니다.
  • TypeScript, Python, R 등 주요 언어용 공식 SDK를 제공하여 API를 쉽게 사용할 수 있습니다. 표준 HTTP 상태 코드와 일관된 에러 응답 형식으로 디버깅을 지원합니다.
