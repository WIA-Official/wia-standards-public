Phase 2: API Interface

5.1 Introduction to Surgical Robot APIs

Phase 2 of WIA-ROB-006 defines standardized Application Programming Interfaces (APIs) enabling programmatic control, monitoring, and integration of surgical robots. RESTful APIs, WebSocket connections, and standardized protocols create an open ecosystem where third-party developers can build compatible applications, research tools, training simulators, and OR integration systems.

5.3 Real-Time Video Streaming

WebRTC Protocol

WIA-ROB-006 specifies WebRTC (Web Real-Time Communication) for low-latency video streaming:

  • Codec: VP9 or H.265 with hardware acceleration
  • Resolution: Adaptive 720p-4K based on bandwidth
  • Latency: End-to-end < 100ms for real-time control
  • Bandwidth: 5-50 Mbps depending on quality
  • Encryption: DTLS-SRTP mandatory for all streams

5.6 Authentication and Security

OAuth 2.0 Implementation

All API access requires OAuth 2.0 authentication with role-based access control:

  • Surgeon Role: Full control authority, all APIs accessible
  • Assistant Role: Camera control, limited instrument access
  • Observer Role: Read-only access, video streams, telemetry
  • Technician Role: System diagnostics, maintenance, calibration
  • Administrator Role: System configuration, user management

TLS 1.3 Encryption

  • All HTTP/HTTPS traffic encrypted with TLS 1.3
  • Perfect forward secrecy (PFS) required
  • Certificate pinning for robot-to-console connections
  • Mutual authentication (client and server certificates)

5.10 SDK Examples and Integration Patterns

Python SDK - Instrument Control

from wia_surgical_robot import SurgicalRobotAPI, InstrumentController

# Initialize API client with OAuth 2.0
api = SurgicalRobotAPI(
    base_url="https://robot.hospital.org/api/v1",
    client_id="surgical-app-001",
    client_secret="secret-key",
    scopes=["robot:read", "robot:control"]
)

# Authenticate
api.authenticate()

# Get instrument controller
arm1 = InstrumentController(api, instrument_id="ARM-1")

# Move to target position
arm1.move_to(
    position=(125.5, 68.3, -24.1),
    orientation=(15.0, -9.0, 46.0),
    velocity=10.0,
    collision_avoidance=True
)

# Monitor movement progress
while not arm1.is_at_target():
    status = arm1.get_status()
    print(f"Progress: {status.progress_percent}% - Distance: {status.distance_to_target}mm")
    time.sleep(0.1)

print("Movement complete")

JavaScript/TypeScript SDK - Real-Time Telemetry

import { SurgicalRobotClient, TelemetryStream } from '@wia/surgical-robot-sdk';

// Initialize client
const client = new SurgicalRobotClient({
  baseUrl: 'https://robot.hospital.org/api/v1',
  apiKey: process.env.ROBOT_API_KEY
});

// Subscribe to real-time telemetry
const stream = new TelemetryStream(client, {
  robotId: 'davinci-si-001',
  streams: ['pose', 'force', 'video'],
  sampleRate: 100 // Hz
});

stream.on('pose', (data) => {
  console.log(`Position: (${data.position.x}, ${data.position.y}, ${data.position.z})`);
  updateVisualization(data);
});

stream.on('force', (data) => {
  if (data.magnitude > 5.0) {
    console.warn(`High force detected: ${data.magnitude}N`);
  }
});

stream.connect();

5.11 API Versioning and Deprecation Policy

WIA-ROB-006 APIs evolve over time. Version management ensures backward compatibility while enabling innovation:

  • URL-Based Versioning: Major versions in path (/api/v1/, /api/v2/)
  • Backward Compatibility: Minor updates (v1.1, v1.2) maintain compatibility within major version
  • Deprecation Notice: Minimum 12 months notice before endpoint removal
  • Migration Support: Detailed migration guides, automated conversion tools, parallel operation period
  • Change Documentation: Comprehensive changelog, API diff tools, breaking change highlights

Key Takeaways

  • RESTful APIs provide standardized endpoints for robot control, instrument manipulation, video streaming, and system monitoring.
  • WebRTC enables real-time video streaming with <100ms latency, supporting multiple concurrent streams at resolutions up to 4K 60fps.
  • Haptic feedback API standardizes force/torque data transmission at 100-1000 Hz for future force-reflecting master manipulators.
  • OAuth 2.0 with role-based access control ensures only authorized users can control surgical robots based on clinical roles.
  • TLS 1.3 encryption with mutual authentication protects all API communications from eavesdropping and tampering.
  • WebSocket connections enable low-latency bi-directional communication for telemetry streaming and real-time control.
  • Safety constraints (velocity limits, force limits, collision avoidance) are enforced at both hardware and software levels.
