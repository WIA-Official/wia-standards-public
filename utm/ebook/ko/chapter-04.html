원격 ID: 드론을 위한 디지털 번호판

2021년 4월 FAA는 미국 드론 규제에서 가장 중요한 규칙 중 하나인 원격 ID 규칙을 발표했습니다. 2023년 9월 16일부터 미국에서 운영되는 거의 모든 드론은 원격 ID 정보를 브로드캐스트해야 합니다. 원격 ID를 드론의 "디지털 번호판"으로 생각하십시오—드론을 고유하게 식별하고 실시간 위치를 제공하며 운영자 정보를 사용 가능하게 합니다.

원격 ID는 여러 목적을 제공합니다:

  • 안전: 당국과 다른 공역 사용자가 드론을 식별하고 추적할 수 있도록 합니다
  • 보안: 무단 드론 활동을 감지하고 대응할 수 있도록 합니다
  • 책임성: 운영자가 자신의 행동에 책임을 질 수 있도록 합니다
  • UTM 통합: USS가 드론을 실시간으로 추적하고 준수를 확인할 수 있도록 합니다
  • 공공 신뢰: 투명성을 통해 드론 기술에 대한 대중의 수용을 향상시킵니다
弘益人間 (홍익인간) - 널리 인간을 이롭게 하라

원격 ID는 단순히 규제 부담이 아니라 드론 기술에 대한 대중의 신뢰를 구축하는 데 필수적입니다. 드론이 식별 가능하고 추적 가능하며 책임을 질 수 있도록 함으로써 원격 ID는 커뮤니티가 드론 작업을 수용할 수 있도록 하여 모든 이에게 혜택을 주는 혁신과 서비스를 가능하게 합니다. 이 투명성은 무단 활동으로부터 보호하면서 합법적인 사용을 촉진합니다.

원격 ID 요구 사항

FAA 원격 ID 규칙은 세 가지 주요 범주의 드론을 정의합니다:

  • 표준 원격 ID가 있는 드론: 제조업체가 내장한 원격 ID를 가진 드론 (2023년 9월 16일 이후 제조된 모든 드론)
  • 원격 ID 브로드캐스트 모듈이 있는 드론: 원격 ID 기능을 추가하기 위해 애프터마켓 모듈이 장착된 기존 드론
  • 원격 ID 없는 드론: FAA 인정 식별 영역(FRIA) 내에서만 작동할 수 있음

원격 ID 메시지 형식

원격 ID는 ASTM F3411 표준에 따라 특정 데이터 필드를 브로드캐스트합니다. 세 가지 주요 메시지 유형이 있습니다:

1. 기본 ID 메시지

드론 및 운영자를 식별합니다:

  • UAS ID: 드론의 고유 일련 번호 또는 세션 ID
  • ID 유형: 일련 번호, 세션 ID 또는 UTM 할당 UUID
  • UA 유형: 비행기, 헬리콥터, 멀티로터, 하이브리드 등

2. 위치/벡터 메시지

드론의 현재 상태를 제공합니다:

  • 위도/경도: WGS-84 데이텀의 현재 위치
  • 고도: 지오이드 위 압력 고도 및 지오메트릭 고도
  • 속도: 지상 속도 및 수직 속도
  • 방향: 진북 기준 트랙 방향
  • 타임스탬프: UTC 시간 (GPS 동기화)
  • 비상 상태: 정상, 비상, 원격 ID 시스템 고장

3. 시스템 메시지 (네트워크 원격 ID만)

운영자 위치 및 시스템 정보를 제공합니다:

  • 운영자 위치: 조종사 또는 지상 제어 스테이션의 위도/경도
  • 운영자 고도: 이륙점 고도
  • 다중 UA 수: 이 운영자가 제어하는 드론 수
  • 영역 반경: 작업이 포함된 원의 반경

브로드캐스트 vs. 네트워크 원격 ID

원격 ID는 두 가지 방법으로 전송될 수 있습니다: 직접 무선 브로드캐스트 또는 인터넷 네트워크를 통해. 각각 장점과 사용 사례가 있습니다.

브로드캐스트 원격 ID

브로드캐스트 원격 ID는 드론에서 직접 무선 신호를 전송합니다. 주변의 누구나 적절한 수신기로 신호를 수신할 수 있습니다:

  • 기술: Wi-Fi Aware (IEEE 802.11) 또는 Bluetooth 5.0 Extended Advertising
  • 범위: 일반적으로 400미터 (이상적인 조건)
  • 주파수: 1초마다 메시지 전송
  • 인프라: 셀룰러 또는 인터넷 연결 불필요
  • 수신자: 스마트폰 앱, 전용 수신기, 법 집행 장치

브로드캐스트 원격 ID의 장점:

  • 셀룰러 범위 또는 인터넷 연결이 필요하지 않음
  • 더 나은 개인 정보 보호—위치 데이터가 중앙 서버로 전송되지 않음
  • 더 낮은 대기 시간—직접 브로드캐스트, 서버를 통하지 않음
  • 더 간단한 기술—대부분의 스마트폰이 이미 지원함

단점:

  • 제한된 범위—수신자가 근처에 있어야 함
  • 장애물로 인한 차단—건물, 언덕이 신호를 방해함
  • 중앙 추적 없음—FIMS/USS가 직접 수신하지 않음
  • 스푸핑에 취약—암호화 또는 인증 없음

네트워크 원격 ID

네트워크 원격 ID는 인터넷(일반적으로 셀룰러)을 통해 원격 ID 서비스 공급자(USS 또는 전용 서비스)에 위치 데이터를 전송합니다:

  • 기술: 4G/5G 셀룰러, Wi-Fi, 위성
  • 범위: 셀룰러 범위 또는 인터넷 연결이 있는 곳이면 어디든지
  • 주파수: 1초마다 또는 USS 요구 사항에 따라
  • 인프라: USS/FIMS와의 인터넷 연결 필요
  • 수신자: 승인된 당사자(USS, FIMS, 법 집행)

네트워크 원격 ID의 장점:

  • 무제한 범위—셀룰러 범위가 있는 곳이면 어디든지 작동
  • 중앙 추적—USS/FIMS가 모든 드론을 실시간으로 볼 수 있음
  • 더 나은 보안—암호화되고 인증된 연결
  • UTM 통합—USS가 직접 데이터를 받음
  • 추가 메타데이터—비행 계획, 운영자 연락처

단점:

  • 셀룰러 범위 필요—원격 지역에서 작동하지 않을 수 있음
  • 데이터 비용—셀룰러 데이터 요금제 또는 USS 구독
  • 개인 정보 보호 우려—중앙 서버에 위치 기록
  • 단일 장애 지점—네트워크 중단이 준수에 영향을 미침

하이브리드 접근 방식

많은 현대 드론은 브로드캐스트와 네트워크 원격 ID를 모두 구현합니다:

  • 로컬 투명성을 위한 지속적인 브로드캐스트
  • UTM 통합을 위한 네트워크 업로드
  • 네트워크가 사용 불가능한 경우 브로드캐스트로 대체
  • 작업 유형에 따라 운영자가 선택 가능(개인 정보 보호 대 기능)

LAANC: 저고도 인증 및 알림 기능

LAANC는 UTM의 가장 성공적인 초기 구현 중 하나입니다. 2017년에 시작되어 2020년까지 전국적으로 확대된 LAANC는 통제 공역 근처에서 드론 작업에 대한 자동 승인을 제공합니다. 승인 시간을 몇 주에서 몇 초로 단축했습니다.

LAANC 이전 상황

LAANC 이전에 공항 근처에서 드론을 비행하려는 운영자는 다음을 수행해야 했습니다:

  • FAA에 종이 양식 제출 (또는 이메일)
  • ATC 시설로 수동으로 전달
  • 인간 관제사가 검토하고 승인하기를 기다립니다
  • 며칠에서 몇 주 기다립니다
  • 종종 예측할 수 없는 결과를 받습니다

이 프로세스는 느리고 불투명하며 확장 가능하지 않았습니다. 상업용 드론 산업이 성장함에 따라 FAA는 하루에 수만 건의 요청을 받을 것입니다—인간 처리로는 불가능합니다.

LAANC 작동 방식

LAANC는 프로세스를 자동화합니다:

  1. 운영자가 USS 앱을 통해 요청 제출: 위치, 고도, 시간 지정
  2. USS가 USFM 한도 확인: 요청된 고도가 USFM 최대값 미만인지 확인
  3. FIMS에 요청 전송: USS가 FAA 승인을 요청
  4. FIMS가 자동으로 평가: TFR, 특별 사용 공역, 기타 제약 확인
  5. 즉시 승인 (또는 거부): 일반적으로 몇 초 이내
  6. ATC 시설에 알림: 관제사가 승인된 드론 작업을 볼 수 있음

전체 프로세스는 종종 10초 미만이 걸립니다. LAANC는 비행 계획 전에 제출하거나 몇 분 또는 몇 시간 전에 제출할 수 있습니다.

LAANC 승인 기준

LAANC는 다음과 같은 경우 자동으로 승인합니다:

  • 요청된 고도가 USFM 한도 미만입니다
  • 위치에 활성 TFR이 없습니다
  • 작업이 Part 107 규칙(VLOS, 사람들 위 등)을 준수합니다
  • 운영자의 Part 107 인증이 현재 유효합니다
  • 특별 보안 요구 사항이 없습니다

요청이 기준을 충족하지 않으면 LAANC는 다음을 수행합니다:

  • 거부하고 이유를 설명합니다 (예: "USFM 한도 초과")
  • 수동 검토를 위해 ATC에 전달합니다 (더 긴 처리)
  • 대안을 제안합니다 (예: "100피트에서 승인됨, 200피트는 아님")

LAANC USS 제공업체

여러 승인된 USS 제공업체가 LAANC 서비스를 제공합니다:

  • AirMap: 최초의 LAANC 제공업체 중 하나, 광범위한 채택
  • Skyward (Verizon): 기업 중심, 플릿 관리
  • Kittyhawk: DJI와 통합, 사용자 친화적
  • UASidekick: 간단한 인터페이스, 신속한 승인
  • Aloft: 날씨 통합, 자동 확인

운영자는 USS를 자유롭게 선택할 수 있습니다. 모두 동일한 FAA LAANC 시스템에 연결되므로 승인은 동등합니다. 차이점은 사용자 경험, 기능 및 가격입니다.

LAANC의 영향

LAANC는 상업용 드론 작업을 변화시켰습니다:

  • 130만+ 승인: 2020년 이후 LAANC를 통해 처리됨
  • 몇 주에서 몇 초로: 평균 승인 시간이 90% 이상 감소
  • 600+ 공항: 전국적으로 LAANC 서비스 제공
  • 예측 가능성: 운영자는 승인을 받을 수 있는지 즉시 알 수 있음
  • ATC 부담 감소: 인간 관제사는 복잡한 경우에만 참여

비행 승인 워크플로

LAANC는 공항 근처 작업을 처리하지만 다른 시나리오에는 다른 승인 경로가 필요합니다.

Part 107 표준 작업

클래스 G 공역, 공항에서 멀리 떨어진, 표준 Part 107 제한 내(400' AGL, VLOS, 낮 등)에서 작동하는 드론은 사전 승인이 필요하지 않습니다. 그러나 UTM에서는 여전히 다음을 수행해야 합니다:

  • USS와 비행 계획 제출 (권장, 곧 필수)
  • 다른 드론 작업과의 충돌 확인
  • TFR 및 특별 사용 공역 확인
  • 원격 ID 브로드캐스트 (2023년 9월 이후)
  • 날씨 및 지상 위험 확인

Part 107 면제 작업

표준 Part 107 한도를 초과하는 작업(BVLOS, 야간, 사람들 위, 고고도)에는 FAA 면제가 필요합니다:

  • 야간 작업: 시민 황혼 후 30분 이후 (이제 Part 107에서 허용, 교육 필요)
  • 사람들 위로 비행: 범주 1-4 요구 사항 또는 면제
  • BVLOS: 시각선 너머, 감지 및 회피 필요
  • 고고도: 400' AGL 이상
  • 다중 드론: 한 조종사가 여러 UAS 동시 제어

면제는 일반적으로 수개월이 걸리지만 일단 승인되면 계속되는 작업에 적용됩니다. 일부 면제는 특정 위치에 묶여 있고 다른 면제는 전국적으로 적용됩니다.

공공 안전 및 정부 작업

공공 안전 기관(경찰, 소방, EMS) 및 정부 기관은 Part 107에 따라 작동하지 않습니다. 대신 공공 항공기 작업(PAO) 또는 인증서(COA)를 받습니다:

  • 더 빠른 승인: 긴급 작업의 경우 종종 24시간 이내
  • 더 큰 유연성: BVLOS, 사람들 위, 야간이 허용될 수 있음
  • 관할 구역 제한: 일반적으로 특정 영역 또는 작업 유형으로 제한됨
  • 안전 계획 필요: 위험 완화를 설명하는 자세한 계획

자동화된 승인 프로세스

미래의 UTM은 대부분의 비행 계획을 자동으로 승인할 것입니다:

  1. 운영자가 USS에 제출: 4D 볼륨, UAS 세부 정보, 운영자 자격
  2. USS가 확인: 운영자 인증, UAS 등록, 보험
  3. USS가 FIMS에 쿼리: 공역 제약, TFR, 특별 사용 공역
  4. USS가 충돌 확인: 다른 USS의 작업과 비교
  5. USS가 날씨 평가: SDSP에서 조건 확인
  6. USS가 위험 평가: 지상 위험, 인구 밀도
  7. 자동 승인 (또는 거부): 몇 초 이내
  8. 실시간 모니터링: 비행 중 준수 추적

이 프로세스는 완전히 자동화되어 인간 개입 없이 초당 수백 건의 요청을 처리할 수 있습니다.

개인 정보 보호 및 보안 고려 사항

원격 ID와 UTM 추적은 중요한 개인 정보 보호 및 보안 질문을 제기합니다.

개인 정보 보호 문제

비판자들은 원격 ID가 다음을 허용한다고 주장합니다:

  • 위치 감시: 정부 또는 기업이 드론 운영자 추적
  • 속성 침입: 사람들이 누가 자신의 속성 위를 비행하는지 파악
  • 경쟁 정보: 경쟁자가 상업 활동 모니터링
  • 대중 괴롭힘: 반드론 활동가가 운영자 식별 및 대립
  • 데이터 남용: 원격 ID 데이터의 무단 사용 또는 판매

개인 정보 보호 완화 조치

FAA와 ASTM은 개인 정보 보호를 보호하기 위한 여러 조치를 구현했습니다:

  • 운영자 개인 정보 제한: 브로드캐스트 원격 ID는 운영자 이름, 주소 또는 전화번호를 포함하지 않습니다—UAS ID만
  • 세션 ID 옵션: 운영자는 일련 번호 대신 임시 세션 ID를 브로드캐스트할 수 있습니다
  • 법 집행 전용 데이터: 일부 원격 ID 데이터는 승인된 당국만 액세스할 수 있습니다
  • 암호화: 네트워크 원격 ID는 암호화되고 인증됩니다
  • 데이터 보존 제한: USS는 작업 후 일정 기간 이후 데이터를 삭제해야 할 수 있습니다

보안 위협

원격 ID 및 UTM 시스템은 사이버 보안 위협에 직면합니다:

  • 스푸핑: 가짜 원격 ID 신호를 브로드캐스트하여 당국을 오도
  • 재밍: 원격 ID 신호를 차단하여 추적 불가능하게 만듭니다
  • 하이재킹: 드론 또는 지상 제어 스테이션에 대한 제어 획득
  • 서비스 거부: USS 또는 FIMS 시스템을 압도하여 중단 유발
  • 데이터 침해: USS/FIMS 데이터베이스에서 민감한 정보 도용

보안 대책

UTM 시스템은 강력한 보안을 구현합니다:

  • 암호화: 모든 USS-FIMS 통신에 대한 TLS/SSL
  • 인증: 암호화 키로 운영자 및 UAS 확인
  • 무결성 확인: 원격 ID 메시지의 변조 감지
  • 중복: 여러 USS 및 통신 경로가 단일 장애 지점 방지
  • 이상 감지: 의심스러운 패턴을 식별하는 AI/ML (예: 스푸핑 시도)

핵심 요점

  • 원격 ID는 2023년 9월부터 거의 모든 미국 드론에 필수이며 드론을 위한 "디지털 번호판" 역할을 하여 식별, 추적 및 책임을 가능하게 합니다
  • 브로드캐스트 원격 ID는 로컬 수신기에 직접 무선으로 전송하는 반면 네트워크 원격 ID는 인터넷을 통해 USS/FIMS에 데이터를 보내며 각각 장단점이 있습니다
  • LAANC는 공항 근처 드론 작업에 대한 자동 승인을 제공하여 승인 시간을 몇 주에서 몇 초로 단축하고 130만 건 이상의 요청을 처리했습니다
  • ASTM F3411 표준은 기본 ID, 위치/벡터 및 시스템 메시지를 포함한 원격 ID 메시지 형식을 정의하여 모든 시스템 간의 상호 운용성을 보장합니다
  • 弘益人間 철학은 원격 ID의 투명성에 구현되어 있으며, 이는 드론 기술에 대한 대중의 신뢰를 구축하고 모든 이에게 혜택을 주는 혁신을 가능하게 합니다
  • 개인 정보 보호 보호(세션 ID, 제한된 운영자 데이터) 및 보안 조치(암호화, 인증)는 원격 ID 시스템의 남용 및 사이버 위협으로부터 보호하는 데 필수적입니다
  • 미래의 UTM은 대부분의 비행 계획을 자동으로 승인하여 공역 제약, 다른 작업, 날씨 및 위험을 몇 초 만에 확인하여 인간 개입을 최소화합니다

복습 문제

  1. 원격 ID의 목적을 설명하고 안전, 보안, 책임성 및 UTM 통합에 어떻게 기여하는지 설명하십시오.
  2. 브로드캐스트 원격 ID와 네트워크 원격 ID를 비교 및 대조하십시오. 각 접근 방식의 장점과 단점은 무엇입니까?
  3. ASTM F3411에 정의된 세 가지 주요 원격 ID 메시지 유형(기본 ID, 위치/벡터, 시스템)과 각각이 포함하는 정보를 설명하십시오.
  4. LAANC가 작동하는 방식과 공항 근처 드론 작업에 대한 승인 프로세스를 어떻게 변화시켰는지 설명하십시오.
  5. Part 107 표준 작업, Part 107 면제 작업 및 공공 안전 작업에 대한 다양한 비행 승인 경로를 설명하십시오.
  6. 원격 ID 및 UTM 추적과 관련된 개인 정보 보호 문제와 이러한 문제를 완화하기 위해 구현된 조치를 논의하십시오.
  7. 弘益人間 원칙이 원격 ID 개발에 어떻게 적용되는지, 그리고 투명성이 어떻게 대중의 신뢰를 구축하고 모든 이에게 혜택을 주는 드론 응용 프로그램을 가능하게 하는지 설명하십시오.

투명성과 책임성을 통해 원격 ID는 드론 기술에 대한 대중의 신뢰를 구축하여 커뮤니티를 보호하고 권한을 부여하는 혁신적인 서비스를 가능하게 합니다.
