Chapter 3 · WIA Ransomware Protection v1.0.0

대응 액션과 롤백 토큰 원칙

response_action 봉투는 한 개 이상의 detection_event에 대응한 SOAR 액션을 기술합니다. 호스트 격리 행위 자체가 환경 상태를 바꾸므로, 대응은 1급 서명 관측으로 다뤄야 합니다.

3.1 봉투 스키마

{
  "action_id": "01HXR9C2KQ3V7N4M8B6P0Z9F1A",
  "incident_id": "01HXR9C2KQ3V7N4M8B6P0Z9F1B",
  "kind": "isolate_host",
  "executed_at": "2026-04-27T14:23:42.991Z",
  "executed_by": "did:wia:soar:acme-kr",
  "rollback_token": "opaque",
  "signature": { "alg": "Ed25519", "value": "..." }
}

3.2 액션 클래스

표준은 6개 기본 액션 클래스를 인정합니다: isolate_host, revoke_session, snapshot_volume, block_egress_cidr, rotate_credential, quarantine_file. 벤더가 독자 클래스를 추가할 수 있으나 위 6개는 모든 적합 SOAR에서 상호운용됩니다.

3.3 롤백 토큰 원칙

모든 response_action은 반드시 rollback_token을 포함해야 합니다. 토큰은 불투명(opaque)하지만 존재 자체가 의무입니다. 자동화된 격리는 결국 잘못된 호스트를 격리할 것이며, 그때 IR 리드가 SOAR 내부 플레이북 상태를 다시 읽지 않고 액션을 되돌릴 수 있어야 하기 때문입니다.

3.4 권한 액션 증명

rotate_credential과 block_egress_cidr 클래스는 사고 범위를 넘는 영향이 있습니다. 표준은 권한 액션 증명자(보통 SOC 관리자 또는 자동 리스크 룰 엔진)의 추가 서명을 요구합니다.

3.5 한국 적용 노트

국내 금융권은 금융보안원의 침해사고 대응 가이드라인에 따라 격리·차단·차단 해제 행위를 모두 시계열로 보고해야 합니다. 본 봉투의 executed_at + rollback_token 패턴이 그대로 충족 증거가 됩니다.

3.6 시간 원칙

executed_at은 SOAR 측 벽시계 시각입니다. 탐지 시각과의 차이 — 흔히 "평균 봉쇄 시간(MTTC)" — 가 핵심 SOC 지표이며, 다운스트림은 원본 탐지의 observed_at과 액션의 executed_at의 차로 계산합니다.

3.7 실패 모드

SOAR가 액션을 시도했으나 하부 강제 평면이 거절(네트워크 타임아웃, 자격증명 만료, 대상 호스트 미상)하면, kind: "failed"와 구조화된 failure_reason으로 봉투가 여전히 발행됩니다. 침묵 실패는 최악의 결과이며 표준은 구조적으로 이를 불가능하게 만듭니다.

워크 예제

중규모 병원 네트워크를 가정합니다 — 18대 EHR 서버, 240대 영상 워크스테이션, 42대 IoT 의료 기기, 별도 생체의학 엔지니어링 서브넷. SOC는 정직원 분석가 1명, 계약 분석가 2명, MSSP 관계 1건. 사이버 보험사는 연간 태세 증거를 요구하고, 국가 보건 규제기관은 24시간 침해 통지를 요구합니다.

표준 이전 세계에서, SOC는 모든 관측을 EDR 포털·SIEM·MSSP 티켓 시스템·보험 설문·규제 신고서에 다시 입력합니다. 실제 사고 1건당 총 작업량이 첫 72시간 동안 분석가 인력의 3–5배를 초과하며, 그것은 기술적 봉쇄가 끝난 후에 하는 작업입니다.

표준 이후 세계에서, 동일 관측이 한 번 서명되고, 적절한 TLP·연합 규칙 하에 테넌트를 구독한 모든 소비자에게 자동 전파되며, 다운스트림 어댑터가 각 소비자의 선호 형식으로 렌더링합니다. SOC 분석가는 첫 72시간을 데이터 입력이 아니라 분석과 의사결정에 씁니다. 그것이 봉투 서명의 전체 회수입니다.

다음 단계

다음 장에서 다음 봉투 클래스를 다룹니다. 8개 장 전체는 약 2시간이면 읽을 수 있도록 의도됐습니다 — 일하는 SOC 분석가가 사고 사이에 책을 끝낼 수 있을 만큼 짧습니다.

편집자 노트

본 표준의 한국어 챕터는 영문 챕터의 직역이 아니라 한국 규제 환경(K-ISA, 금융보안원, 개인정보보호위원회, 의료ISAC)을 염두에 둔 응용 노트와 함께 재작성됐습니다. 직역이 필요한 경우 영문판을 우선 인용해 주십시오. 봉투 서명·검증 코드 예제와 한국 ISAC 연결 예제는 별도 부속서에 수록될 예정입니다.
