머클 루트 기반 불변 백업 증명

backup_snapshot 봉투는 랜섬웨어 운영자가 실존적 질문으로 만든 문제에 대한 표준의 답입니다 — 암호화가 끝났을 때, 조직이 미암호화 사본의 존재를 증명할 수 있는가, 그리고 침해를 다시 들이지 않고 그것을 복원할 수 있는가.

4.1 봉투 스키마

{
  "snapshot_id": "01HXRA9C2KQ3V7N4M8B6P0Z9F1",
  "tenant_id": "did:wia:tenant:acme-kr",
  "captured_at": "2026-04-27T02:00:00.000Z",
  "asset_class": "volume",
  "size_bytes": 1099511627776,
  "encryption": { "algorithm": "AES-256-GCM" },
  "immutability": {
    "retention_until": "2026-07-27T02:00:00.000Z",
    "lock_kind": "compliance"
  },
  "merkle_root": "sha384-...",
  "signature": { "alg": "Ed25519", "value": "..." }
}

4.2 머클 루트인 이유

1 TB 스냅샷 위에 단일 평면 해시는 전체 무결성은 증명하지만 부분 집합에 대해서는 아무것도 증명하지 못합니다. 객체별 해시 위에 머클 루트를 두면 둘 다 증명됩니다 — 루트는 전체를, 루트 + 로그 깊이의 증명 경로는 임의의 단일 객체를 증명합니다.

4.3 불변성 의미론

immutability 블록은 세 가지를 명명합니다: 언제 다시 변경 가능한가(retention_until), 어떤 잠금인가(compliance는 테넌트 관리자도 제거 불가; governance는 감사 하에 해제 가능; object-lock-s3는 S3 호환), 그리고 잠금을 보유하는 브로커.

4.4 한국 적용 노트

전자금융감독규정 제12조의2 백업 의무, 개인정보보호법 시행령 제48조의2 백업 보관 의무는 모두 본 봉투의 retention_until + lock_kind: compliance 조합으로 만족합니다. 감사관은 봉투 서명만 검증하면 됩니다.

4.5 법의 보존

스냅샷의 일부는 forensic_preservation: true로 표시됩니다. 부모 사고가 incident_closed_at을 기록하기 전까지 일반 보존 만료에서 제외됩니다. 표준은 어느 관할의 증거 처리 규칙이 적용되는지 강제하지 않지만, 보관 사슬 필드(custodian, transfer_log, tamper_seal)는 필수입니다.

4.6 다중 리전 증명

표준은 권고 — 강제는 아님 — 으로 임계 자산이 최소 두 개의 지리적으로 분리된 리전에 증명된 스냅샷을 보유할 것을 권고합니다. 활성 사고 중 리전 장애는 오프사이트 복제본이 96시간 동안 갱신되지 않았음을 발견할 최악의 시점입니다.

워크 예제

중규모 병원 네트워크를 가정합니다 — 18대 EHR 서버, 240대 영상 워크스테이션, 42대 IoT 의료 기기, 별도 생체의학 엔지니어링 서브넷. SOC는 정직원 분석가 1명, 계약 분석가 2명, MSSP 관계 1건. 사이버 보험사는 연간 태세 증거를 요구하고, 국가 보건 규제기관은 24시간 침해 통지를 요구합니다.

표준 이전 세계에서, SOC는 모든 관측을 EDR 포털·SIEM·MSSP 티켓 시스템·보험 설문·규제 신고서에 다시 입력합니다. 실제 사고 1건당 총 작업량이 첫 72시간 동안 분석가 인력의 3–5배를 초과하며, 그것은 기술적 봉쇄가 끝난 후에 하는 작업입니다.

표준 이후 세계에서, 동일 관측이 한 번 서명되고, 적절한 TLP·연합 규칙 하에 테넌트를 구독한 모든 소비자에게 자동 전파되며, 다운스트림 어댑터가 각 소비자의 선호 형식으로 렌더링합니다. SOC 분석가는 첫 72시간을 데이터 입력이 아니라 분석과 의사결정에 씁니다. 그것이 봉투 서명의 전체 회수입니다.

다음 단계

편집자 노트

본 표준의 한국어 챕터는 영문 챕터의 직역이 아니라 한국 규제 환경(K-ISA, 금융보안원, 개인정보보호위원회, 의료ISAC)을 염두에 둔 응용 노트와 함께 재작성됐습니다. 직역이 필요한 경우 영문판을 우선 인용해 주십시오. 봉투 서명·검증 코드 예제와 한국 ISAC 연결 예제는 별도 부속서에 수록될 예정입니다.
