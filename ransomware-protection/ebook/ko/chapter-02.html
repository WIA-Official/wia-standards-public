탐지 봉투와 MITRE ATT&CK 매핑

detection_event 봉투는 본 표준의 가장 작은 공유 단위입니다. EDR이 발화하거나 SIEM이 상관 분석을 마치거나 카나리가 트립될 때, 결과 레코드가 다운스트림 모든 소비자에게 모호함 없이 전달되어야 합니다.

2.1 봉투 스키마

{
  "event_id": "01HXR8E2ZQ9V8K3M5W7P0T2N4F",
  "tenant_id": "did:wia:tenant:acme-kr",
  "observed_at": "2026-04-27T14:22:18.412Z",
  "mitre_attack": [
    { "tactic": "TA0040", "technique": "T1486" }
  ],
  "confidence": 0.94,
  "severity": "CRITICAL",
  "signature": { "alg": "Ed25519", "value": "..." }
}

2.2 MITRE 정렬이 필수인 이유

모든 탐지기는 결국 자체 명명 체계를 만듭니다. 한 EDR은 "Mass File Encryption", 다른 EDR은 "Crypto Activity", 또 다른 곳은 "Ransomware Behaviour Pattern 7"이라 부릅니다. 공통 참조 프레임이 없으면 다운스트림 상관 분석이 문자열 매칭으로 무너지고, 문자열 매칭은 노이즈로 무너집니다.

2.3 신뢰도는 보정값

confidence 필드는 보정된 사후 확률입니다. 0.94를 발행하는 탐지기는 "과거 동일 점수로 발행한 관측 모집단에서 실제 참 양성 비율이 약 94 %였다"라고 주장하는 것입니다. 보정 값을 만들 수 없는 벤더는 휴리스틱 대신 null을 발행해야 합니다.

2.4 탐지기 클래스

표준은 4개 탐지기 클래스를 인정합니다: 행위 카나리, 암호화 이상, 신원 유출, 측면 이동. 각 클래스는 운영적 함의가 다릅니다 — 카나리는 좁지만 고신뢰, 암호화 이상은 광범위하지만 백업/배포 윈도우에 오탐 위험.

2.5 한국 적용 노트

국내 SIEM 벤더(이글루코퍼레이션, 안랩, 윈스)는 자체 분류 체계를 사용합니다. 본 표준의 봉투를 추가 발행하면 ISAC(KISA, 금융보안원, 의료ISAC) 공유 시 별도 변환 없이 곧바로 흘려보낼 수 있습니다.

2.6 리플레이 방어

탐지 봉투는 96-bit nonce를 포함하고 ±300초 스큐 허용 윈도우 안에서만 수용됩니다. 600초 캐시 안의 리플레이는 거부됩니다. 협상 대상이 아닙니다 — 리플레이된 탐지는 공격자가 알려진-노이즈 사건으로 SOC를 익사시켜 재진입을 숨기는 방법입니다.

워크 예제

중규모 병원 네트워크를 가정합니다 — 18대 EHR 서버, 240대 영상 워크스테이션, 42대 IoT 의료 기기, 별도 생체의학 엔지니어링 서브넷. SOC는 정직원 분석가 1명, 계약 분석가 2명, MSSP 관계 1건. 사이버 보험사는 연간 태세 증거를 요구하고, 국가 보건 규제기관은 24시간 침해 통지를 요구합니다.

표준 이전 세계에서, SOC는 모든 관측을 EDR 포털·SIEM·MSSP 티켓 시스템·보험 설문·규제 신고서에 다시 입력합니다. 실제 사고 1건당 총 작업량이 첫 72시간 동안 분석가 인력의 3–5배를 초과하며, 그것은 기술적 봉쇄가 끝난 후에 하는 작업입니다.

표준 이후 세계에서, 동일 관측이 한 번 서명되고, 적절한 TLP·연합 규칙 하에 테넌트를 구독한 모든 소비자에게 자동 전파되며, 다운스트림 어댑터가 각 소비자의 선호 형식으로 렌더링합니다. SOC 분석가는 첫 72시간을 데이터 입력이 아니라 분석과 의사결정에 씁니다. 그것이 봉투 서명의 전체 회수입니다.

다음 단계

다음 장에서 다음 봉투 클래스를 다룹니다. 8개 장 전체는 약 2시간이면 읽을 수 있도록 의도됐습니다 — 일하는 SOC 분석가가 사고 사이에 책을 끝낼 수 있을 만큼 짧습니다.

편집자 노트

본 표준의 한국어 챕터는 영문 챕터의 직역이 아니라 한국 규제 환경(K-ISA, 금융보안원, 개인정보보호위원회, 의료ISAC)을 염두에 둔 응용 노트와 함께 재작성됐습니다. 직역이 필요한 경우 영문판을 우선 인용해 주십시오. 봉투 서명·검증 코드 예제와 한국 ISAC 연결 예제는 별도 부속서에 수록될 예정입니다.
