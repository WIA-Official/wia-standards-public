08.7 In Depth — chapter focus

The WIA Space standard treats a spaceflight operation as a layered data flow: vehicle preparation → mission profile → manifest reservation → wire envelope (telemetry, conjunction, incident) → server-side mission control and federation → response envelope → ground display and SAR coordination. Each step has a published shape; the goal of this chapter is to make that shape and its conformance gate explicit so an integrator can build a WIA-Space-conformant component without reading the entire specification first.

The relevant Phase artifacts are: a JSON envelope (Phase 1), a signed HTTP endpoint (Phase 2), a federation handshake when crossing organisational boundaries between operators / regulators / SAR (Phase 3), and a bridge profile for ICAO Doc 4444, CCSDS Space Packet Protocol, and range-safety stacks (Phase 4). Implementations are tested by the open conformance suite at https://github.com/WIA-Official/wia-space-conformance against a mock host.

Two recurring design choices show up across every chapter and are worth flagging once here. First, every space envelope carries an explicit operator and mission identifier; multi-operator missions (commercial launch + national agency payload + university experiment) are common, so the standard treats per-operator audience controls as a first-class concern. Second, every wire envelope is signed with Ed25519 (IETF RFC 8032) and the federation handshake reuses the same shape that WIA-SOCIAL Phase 3 §5 fixes; this keeps the cross-standard mental model uniform.

08.8 Reference Implementation Walk-Through

The reference container wia/space-host:1.0.0 implements every Phase 2 endpoint and ships with the open conformance suite preconfigured. A typical local exploration looks like:

docker run -p 8080:8080 wia/space-host:1.0.0
curl -X POST http://localhost:8080/space/mission \
     -H "Content-Type: application/json" \
     -d '{ "wia_space_version":"1.0.0", "type":"mission", "...":"..." }'

The container ships with mock telemetry and a small library of mission profiles so an integrator can exercise every endpoint without an actual launch vehicle. Once the integration is working against the mock, the operator swaps in their real vehicle telemetry stream and ground-segment integration.

08.9 Operational Considerations

Space infrastructure has three operational considerations that integrators consistently underestimate. First, telemetry retention scales aggressively with mission duration; the standard makes retention a per-mission decision so an operator can size storage against the actual scope. Second, conjunction warnings have hard real-time deadlines (collision avoidance manoeuvres typically need minutes of notice); the host MUST treat conjunction warnings as the highest-priority traffic and MUST page on-call when warning latency exceeds a baseline threshold. Third, range-safety integration requires HSM-backed credentials that must rotate at every mission boundary.
