제6장: 응급분류 시스템 (Triage) - 우선순위 결정

"응급분류는 의학이자 예술입니다. 제한된 자원으로 최대한 많은 생명을 구하는 것이 목표입니다."

6.1 응급분류 (Triage)란?

Triage는 프랑스어로 '선별'을 의미하며, 환자의 중증도를 신속하게 평가하여 치료 우선순위를 결정하는 과정입니다.

🎯 KTAS (한국형 응급환자 분류도구)

레벨 분류 대응 시간 예시
1단계 (적색) 소생 (Resuscitation) 즉시 심정지, 대량 출혈
2단계 (주황) 응급 (Emergency) 15분 이내 심근경색, 뇌졸중
3단계 (황색) 긴급 (Urgent) 30분 이내 골절, 복통
4단계 (녹색) 준응급 (Less Urgent) 1시간 이내 염좌, 열상
5단계 (백색) 비응급 (Non-Urgent) 2시간 이내 감기, 경미한 통증

6.2 현장 응급분류 (Field Triage)

대규모 재난이나 다수 사상자 발생 시 현장에서의 신속한 분류가 핵심입니다.

🚨 START 분류법 (Simple Triage And Rapid Treatment)

START 분류 알고리즘 (30초 이내)

1. 걸을 수 있는가?
   YES → 🟢 GREEN (경증)
   NO → 다음 단계

2. 호흡이 있는가?
   NO → 기도 개방 시도
      여전히 없음 → ⚫ BLACK (사망)
      회복됨 → 🔴 RED (즉시)
   YES → 다음 단계

3. 호흡수는?
   > 30회/분 → 🔴 RED (즉시)
   < 30회/분 → 다음 단계

4. 순환 상태는?
   맥박 없음 또는 모세혈관 재충전 > 2초 → 🔴 RED
   정상 → 다음 단계

5. 의식 상태는?
   명령 따를 수 없음 → 🔴 RED (즉시)
   명령 따름 → 🟡 YELLOW (지연)

🏷️ 분류 태그 시스템

각 환자에게 색상 태그를 부착하여 우선순위를 표시합니다.

  • 🔴 RED (즉시) — 즉각적인 치료 없이는 생존 불가
  • 🟡 YELLOW (지연) — 치료 지연 가능하나 모니터링 필요
  • 🟢 GREEN (경증) — 경미한 부상, 걸어다닐 수 있음
  • ⚫ BLACK (사망/기대) — 사망 또는 생존 가능성 극히 낮음

📱 스마트 응급분류 앱

환자나 보호자가 스스로 예비 분류를 할 수 있는 모바일 앱입니다.

  • 증상 체크리스트 — 간단한 질문으로 중증도 예측
  • 사진 분석 — 외상, 발진 등 시각적 평가
  • 음성 인식 — 환자 상태 음성으로 기록
  • 병원 추천 — 증상에 적합한 병원 안내

6.4 재분류 (Re-triage)

환자의 상태는 변할 수 있으므로 주기적인 재평가가 필수입니다.

🔄 재분류 프로토콜

초기 레벨 재평가 주기 상태 변화 시 조치
1단계 (소생) 연속 모니터링 즉시 대응
2단계 (응급) 15분마다 악화 시 1단계로 상향
3단계 (긴급) 30분마다 증상 진행 시 2단계로
4-5단계 1시간마다 필요시 상향 조정

6.5 특수 상황 응급분류

👶 소아 응급분류 (Pediatric Triage)

소아는 성인과 다른 기준을 적용해야 합니다.

  • 연령별 생체징후 — 나이에 따른 정상 범위 차이
  • 탈수 평가 — 피부 긴장도, 눈물, 소변량
  • 보호자 관찰 — "평소와 다름" 중요한 단서
  • 통증 척도 — FLACC, Wong-Baker 얼굴 척도

🧓 노인 응급분류

  • 비특이적 증상 — 명확하지 않은 증상도 심각할 수 있음
  • 낙상 위험 — 낙상 이력 및 기립성 저혈압 평가
  • 인지 평가 — 기저 치매 vs 급성 섬망 구분
  • 다약제 — 약물 상호작용 및 부작용 고려

6.6 대량 사상자 사건 (Mass Casualty Incident)

재난 상황에서는 "최대 다수의 최대 생존"을 목표로 합니다.

🌊 MCI 응급분류 원칙

평상시 원칙: 가장 중한 환자를 먼저 치료

MCI 원칙: 가장 많은 사람을 살리기 위해, 살릴 가능성이 높은 환자를 우선 치료

이는 윤리적 딜레마를 수반하지만, 제한된 자원으로 최대 생명 구조를 위한 불가피한 선택입니다.
