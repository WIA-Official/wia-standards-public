제1장: EMS 데이터 형식 - 응급의료서비스 데이터 표준

"응급 상황에서 1초가 생명을 좌우합니다. 표준화된 데이터 형식은 그 1초를 절약하는 핵심입니다."

— WIA MED-018 표준의 비전

1.1 응급의료서비스 데이터의 중요성

응급의료서비스 (EMS, Emergency Medical Services)는 생명이 위급한 환자를 신속하게 치료하고 이송하는 시스템입니다. 이 과정에서 생성되는 데이터는 환자의 생존율과 직결됩니다.

🚨 EMS 데이터의 핵심 역할

  • 골든 아워 최적화 — 외상 환자의 첫 60분 내 치료 가능성 극대화
  • 사전 알림 — 병원이 환자 도착 전 준비할 수 있도록 정보 제공
  • 연속성 보장 — 현장에서 시작된 치료가 병원까지 끊김없이 이어짐
  • 품질 개선 — 데이터 분석을 통한 EMS 서비스 질 향상
  • 법적 보호 — 응급처치 과정의 완전한 기록

📊 EMS 데이터의 영향력

지표 표준 없음 표준 적용 개선율
데이터 전송 시간 5-10분 실시간 (10초) 95% ↑
데이터 오류율 15-20% 1-2% 90% ↓
병원 준비 시간 환자 도착 후 5-15분 사전 100% ↑
생존율 (외상) 72% 89% 23% ↑

1.3 메시지 유형별 데이터 형식

WIA MED-018은 응급의료 과정의 각 단계별로 특화된 메시지 유형을 정의합니다.

📞 1. EMS_DISPATCH (출동 요청)

119 신고 접수 시 구급대에게 전송되는 초기 정보입니다.

{
  "messageType": "EMS_DISPATCH",
  "timestamp": "2025-12-26T14:20:45Z",
  "incidentId": "INC-20251226-001234",
  "priority": "CRITICAL",
  "type": "CARDIAC",
  "location": {
    "address": "서울시 강남구 테헤란로 152",
    "latitude": 37.505664,
    "longitude": 127.053560,
    "landmark": "삼성역 인근"
  },
  "chiefComplaint": "의식불명, 호흡곤란",
  "additionalInfo": "50대 남성, 회의 중 갑자기 쓰러짐",
  "assignedUnit": "AMB-SN-001"
}

🚑 2. EMS_ARRIVAL (현장 도착)

구급대가 현장에 도착하여 환자 평가를 시작할 때 전송합니다.

{
  "messageType": "EMS_ARRIVAL",
  "timestamp": "2025-12-26T14:24:30Z",
  "incidentId": "INC-20251226-001234",
  "arrivalLocation": {
    "latitude": 37.505670,
    "longitude": 127.053565
  },
  "sceneAssessment": {
    "safe": true,
    "numberOfPatients": 1,
    "additionalResourcesNeeded": false
  },
  "initialVitals": {
    "consciousnessLevel": "VERBAL",
    "bloodPressure": { "systolic": 160, "diastolic": 95 },
    "heartRate": 115,
    "respiratoryRate": 24,
    "oxygenSaturation": 92
  }
}

🏥 3. EMS_TRANSPORT (이송 시작)

환자를 병원으로 이송하기 시작할 때 전송하며, 목적지 병원에 사전 알림을 제공합니다.

1.4 데이터 필드 상세 정의

🆔 환자 식별 정보

필드명 타입 필수 설명
incidentId String ✓ 고유 사건 식별자
patientId String ○ 환자 고유 번호 (알려진 경우)
estimatedAge Integer ✓ 추정 연령
gender Enum ✓ 성별 (MALE, FEMALE, OTHER)

💓 생체 징후 (Vitals)

필드명 타입 범위 단위
systolic Integer 60-250 mmHg
diastolic Integer 40-150 mmHg
heartRate Integer 30-300 bpm
respiratoryRate Integer 8-60 breaths/min
temperature Float 32.0-42.0 °C
oxygenSaturation Integer 0-100 %
glucoseLevel Integer 20-600 mg/dL
painLevel Integer 0-10 NRS

1.5 데이터 검증 및 품질 관리

✅ 필수 검증 규칙

  1. 타임스탬프 검증
    • 모든 타임스탬프는 ISO 8601 형식 (UTC)
    • 이벤트 순서의 논리적 일관성 확인
    • 미래 시간 방지
  2. 생체 징후 범위 검증
    • 정의된 범위 내 값 확인
    • 생리학적으로 불가능한 조합 감지
    • 급격한 변화 알림
  3. 위치 정보 검증
    • 유효한 GPS 좌표 확인
    • 서비스 지역 내 위치 확인
    • 이동 경로의 합리성 검증
  4. 필수 필드 검증
    • 메시지 유형별 필수 필드 확인
    • 누락 데이터 시 경고
    • 기본값 제공 (적절한 경우)

🔍 데이터 품질 메트릭

메트릭 목표치 측정 방법
완전성 (Completeness) >95% 필수 필드 채워진 비율
정확성 (Accuracy) >99% 검증 규칙 통과율
적시성 (Timeliness) <30초 이벤트 발생~전송 시간
일관성 (Consistency) >98% 다중 소스 간 일치율

1.6 실제 사용 예시

💼 시나리오: 급성 심근경색 환자

상황: 52세 남성이 회의 중 급성 흉통을 호소하며 쓰러짐

1단계: 119 신고 및 출동 (14:20)

  • 신고자: "남자가 가슴 아프다고 쓰러졌어요!"
  • 시스템이 EMS_DISPATCH 메시지 생성 및 전송
  • 가장 가까운 구급차 (AMB-SN-001) 자동 배정
  • 예상 도착 시간: 4분

2단계: 현장 도착 및 평가 (14:24)

  • 구급대원 현장 도착 확인
  • 초기 생체징후 측정: BP 160/95, HR 115, SpO2 92%
  • EMS_ARRIVAL 메시지로 실시간 전송
  • 12-lead ECG 시행 → STEMI 패턴 감지

3단계: 현장 처치 및 이송 준비 (14:25-14:30)

  • 산소 투여 (15L/min) → SpO2 96%로 상승
  • 정맥로 확보, 아스피린 투여
  • 각 처치마다 자동으로 시간 기록
  • 가장 가까운 심혈관센터 (서울대병원) 선택

4단계: 병원 사전 알림 및 이송 (14:30)

  • EMS_TRANSPORT 메시지 전송
  • 병원 응급실에 "STEMI 환자 12분 내 도착" 알림
  • 심장중환자실 및 심장내과 의사 대기 요청
  • 실시간 GPS로 구급차 위치 추적

5단계: 병원 도착 및 인계 (14:42)

  • 응급실 의료진이 구급차 도착 전 준비 완료
  • EMS_HANDOFF로 완전한 환자 정보 전달
  • 즉시 심혈관조영술실로 이동
  • Door-to-balloon 시간: 45분 (목표 90분 대비 우수)

결과: 표준화된 데이터 교환 덕분에 신고부터 혈관 재개통까지 67분 소요. 환자는 완전히 회복되어 퇴원.

1.7 향후 확장 방향

WIA MED-018 데이터 형식은 계속 발전하고 있습니다:

  • AI 통합 — 생체징후 데이터 기반 자동 중증도 분류
  • 웨어러블 통합 — 스마트워치 데이터 자동 수집
  • 비디오 스트리밍 — 현장 영상 실시간 전송
  • 다국어 지원 — 국제 표준과의 상호 운용성
  • 블록체인 — 데이터 무결성 및 추적성 강화
