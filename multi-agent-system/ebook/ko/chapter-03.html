조정 메커니즘

弘益人間 (홍익인간) · 널리 인간을 이롭게 하라

조정은 집단 목표를 향해 개별 행동을 조화시키는 예술입니다. 각 음악가가 완벽한 동기화로 자신의 파트를 연주하는 오케스트라처럼, 다중 에이전트 시스템은 에이전트가 모두의 이익을 위해 효과적으로 함께 작업하도록 보장하는 정교한 조정 메커니즘을 통해 놀라운 결과를 달성합니다.

조정의 필요성

다중 에이전트 시스템에서 조정은 에이전트의 활동 간 상호 의존성을 관리하는 문제를 다룹니다. 조정이 없으면 에이전트는 상반된 목적으로 작업하거나, 노력을 중복하거나, 시스템이 목표를 달성하지 못하게 하는 충돌을 만들 수 있습니다.

정의: 조정은 일관된 시스템 행동과 목표 달성을 보장하기 위해 다양한 에이전트가 수행하는 활동 간의 종속성을 관리하는 프로세스입니다.

다음과 같은 경우 조정이 필요합니다:

  • 에이전트가 할당되어야 하는 공유 리소스를 공유
  • 작업에 순차적 또는 병렬 실행이 필요한 종속성이 있음
  • 에이전트에 조정되어야 하는 충돌하는 목표가 있음
  • 시스템에 집단 행동이 필요한 전역 목표가 있음
  • 환경 제약이 가능한 행동을 제한함

중앙 집중식 vs. 분산형 조정

중앙 집중식 조정

중앙 조정자 에이전트가 모든 조정 결정을 관리합니다. 이 접근 방식은 전역 최적화를 제공하지만 단일 장애 지점과 확장성 병목 현상을 만듭니다.

class CentralCoordinator {
    constructor() {
        this.agents = new Map();
        this.tasks = [];
        this.schedule = [];
    }

    registerAgent(agent) {
        this.agents.set(agent.id, {
            agent: agent,
            capabilities: agent.getCapabilities(),
            currentLoad: 0,
            status: 'available'
        });
    }

    allocateTasks(tasks) {
        // 중앙 조정자가 모든 할당 결정을 내림
        const allocation = new Map();

        for (const task of tasks) {
            const bestAgent = this.findBestAgent(task);
            if (bestAgent) {
                allocation.set(task.id, bestAgent.id);
                bestAgent.currentLoad += task.estimatedEffort;
            }
        }

        return this.executeAllocation(allocation);
    }

    findBestAgent(task) {
        let best = null;
        let bestScore = -Infinity;

        for (const [id, agentInfo] of this.agents) {
            if (!agentInfo.agent.canPerform(task)) continue;

            const score = this.scoreAgent(agentInfo, task);
            if (score > bestScore) {
                bestScore = score;
                best = agentInfo;
            }
        }

        return best;
    }

    scoreAgent(agentInfo, task) {
        // 능력 매치, 현재 부하, 가용성을 기반으로 점수 계산
        const capabilityScore = this.matchCapabilities(
            agentInfo.capabilities,
            task.requirements
        );
        const loadScore = 100 - agentInfo.currentLoad;

        return capabilityScore * 0.7 + loadScore * 0.3;
    }
}

분산형 조정

에이전트는 중앙 권한 없이 P2P 상호작용을 통해 조정합니다. 이는 더 나은 확장성과 강건성을 제공하지만 전역 최적화를 달성하지 못할 수 있습니다.

협상 메커니즘

협상을 통해 에이전트는 반복적인 제안과 반대 제안 교환을 통해 합의에 도달할 수 있습니다.

단조 양보 프로토콜

에이전트는 합의에 도달하거나 협상이 실패할 때까지 연속적인 양보를 합니다.

class NegotiatingAgent {
    constructor(id, initialPosition, minAcceptable) {
        this.id = id;
        this.position = initialPosition;
        this.minAcceptable = minAcceptable;
        this.concessionRate = 0.1;
    }

    async negotiate(otherAgent, issue) {
        let round = 0;
        const maxRounds = 10;

        while (round < maxRounds) {
            // 제안 생성
            const myProposal = this.makeProposal(issue);
            const theirProposal = await otherAgent.receiveProposal(myProposal);

            // 제안이 겹치는지 확인 (합의 영역)
            if (this.isAcceptable(theirProposal)) {
                return this.formAgreement(myProposal, theirProposal);
            }

            // 양보
            this.makeConcession(theirProposal);
            round++;
        }

        return null; // 협상 실패
    }

    makeProposal(issue) {
        return {
            agent: this.id,
            issue: issue,
            value: this.position,
            round: this.currentRound
        };
    }

    isAcceptable(proposal) {
        return proposal.value >= this.minAcceptable;
    }

    makeConcession(theirProposal) {
        // 상대방 입장으로 이동
        const gap = theirProposal.value - this.position;
        this.position += gap * this.concessionRate;

        // 최소값 이하로 내려가지 않음
        this.position = Math.max(this.position, this.minAcceptable);
    }

    formAgreement(myProposal, theirProposal) {
        // 차이를 절반으로 나눔
        return {
            agents: [this.id, theirProposal.agent],
            agreedValue: (myProposal.value + theirProposal.value) / 2,
            timestamp: Date.now()
        };
    }
}

투표 및 합의

에이전트가 집단 결정을 내려야 할 때, 투표 메커니즘은 개별 선호도를 그룹 결정으로 집계합니다.

투표 방법

방법 설명 장단점
다수결 가장 많은 표를 받은 옵션이 승리 간단하지만 소수 선호도를 선출할 수 있음
과반수 50% 이상의 표를 받은 옵션이 승리 광범위한 지원을 보장하지만 결선 투표가 필요할 수 있음
만장일치 모든 에이전트가 동의해야 함 강력한 합의, 그러나 느리고 차단될 수 있음
가중치 투표에 다른 가중치가 있음 전문성/지분을 반영하지만 덜 민주적
순위 선택 에이전트가 선호도를 순위화 미묘한 선호도를 포착하지만 복잡함
보르다 카운트 순위에 기반한 점수 합의 선택에 보상

합의 알고리즘

분산 합의 알고리즘은 실패와 네트워크 문제에도 불구하고 에이전트가 시스템 상태에 동의하도록 보장합니다.

Raft 합의

Raft는 이해하기 쉽도록 설계된 합의 알고리즘입니다. 에이전트 클러스터가 일련의 값(로그 항목)에 동의하도록 보장합니다.

시장 기반 조정

시장 메커니즘은 공급, 수요 및 가격 책정을 통해 에이전트 행동을 조정하기 위해 경제 원칙을 사용합니다.

경매 메커니즘

class Auctioneer {
    constructor() {
        this.auctions = new Map();
    }

    // 영국식 경매 (상승 가격)
    async englishAuction(item, startPrice, minIncrement) {
        let currentBid = startPrice;
        let currentWinner = null;
        let bidders = this.getBidders(item);
        let noBidRounds = 0;

        while (noBidRounds < 3) {
            const bids = [];

            for (const bidder of bidders) {
                const bid = await bidder.considerBid(item, currentBid);
                if (bid > currentBid + minIncrement) {
                    bids.push({ bidder: bidder.id, amount: bid });
                }
            }

            if (bids.length === 0) {
                noBidRounds++;
            } else {
                noBidRounds = 0;
                const highestBid = Math.max(...bids.map(b => b.amount));
                currentBid = highestBid;
                currentWinner = bids.find(b => b.amount === highestBid).bidder;
            }
        }

        return { winner: currentWinner, price: currentBid };
    }

    // 밀봉 입찰 (1등 가격)
    async sealedBidAuction(item) {
        const bidders = this.getBidders(item);
        const bids = await Promise.all(
            bidders.map(b => b.submitSealedBid(item))
        );

        const highestBid = Math.max(...bids.map(b => b.amount));
        const winner = bids.find(b => b.amount === highestBid);

        return { winner: winner.bidder, price: winner.amount };
    }

    // 비크리 경매 (2등 가격 밀봉 입찰)
    async vickreyAuction(item) {
        const bidders = this.getBidders(item);
        const bids = await Promise.all(
            bidders.map(b => b.submitSealedBid(item))
        );

        bids.sort((a, b) => b.amount - a.amount);

        return {
            winner: bids[0].bidder,
            price: bids[1].amount // 두 번째로 높은 가격 지불
        };
    }
}

계획 병합 및 조정

에이전트가 개별 계획을 만들 때, 이러한 계획은 충돌을 피하고 시너지를 활용하기 위해 조정되어야 합니다.

군집 조정

군집 조정을 통해 많은 수의 간단한 에이전트가 로컬 상호작용을 통해 복잡한 집단 행동을 달성할 수 있습니다.

class SwarmAgent {
    constructor(id, position) {
        this.id = id;
        this.position = position;
        this.velocity = { x: 0, y: 0 };
        this.neighbors = [];
    }

    // 무리 지기를 위한 Reynolds의 Boids 알고리즘
    updateVelocity(allAgents) {
        const neighbors = this.findNeighbors(allAgents, 50);

        const separation = this.separate(neighbors);
        const alignment = this.align(neighbors);
        const cohesion = this.cohere(neighbors);

        // 가중치를 사용하여 행동 결합
        this.velocity.x += separation.x * 1.5 + alignment.x * 1.0 + cohesion.x * 1.0;
        this.velocity.y += separation.y * 1.5 + alignment.y * 1.0 + cohesion.y * 1.0;

        // 속도 제한
        const speed = Math.sqrt(this.velocity.x ** 2 + this.velocity.y ** 2);
        const maxSpeed = 5;
        if (speed > maxSpeed) {
            this.velocity.x = (this.velocity.x / speed) * maxSpeed;
            this.velocity.y = (this.velocity.y / speed) * maxSpeed;
        }
    }

    // 이웃과의 혼잡 방지
    separate(neighbors) {
        const steer = { x: 0, y: 0 };
        for (const neighbor of neighbors) {
            const dx = this.position.x - neighbor.position.x;
            const dy = this.position.y - neighbor.position.y;
            const dist = Math.sqrt(dx * dx + dy * dy);

            if (dist > 0 && dist < 25) {
                steer.x += dx / dist;
                steer.y += dy / dist;
            }
        }
        return steer;
    }

    // 이웃의 평균 속도와 정렬
    align(neighbors) {
        if (neighbors.length === 0) return { x: 0, y: 0 };

        const avgVel = neighbors.reduce(
            (sum, n) => ({
                x: sum.x + n.velocity.x,
                y: sum.y + n.velocity.y
            }),
            { x: 0, y: 0 }
        );

        return {
            x: avgVel.x / neighbors.length - this.velocity.x,
            y: avgVel.y / neighbors.length - this.velocity.y
        };
    }

    // 이웃의 평균 위치로 이동
    cohere(neighbors) {
        if (neighbors.length === 0) return { x: 0, y: 0 };

        const avgPos = neighbors.reduce(
            (sum, n) => ({
                x: sum.x + n.position.x,
                y: sum.y + n.position.y
            }),
            { x: 0, y: 0 }
        );

        return {
            x: avgPos.x / neighbors.length - this.position.x,
            y: avgPos.y / neighbors.length - this.position.y
        };
    }
}

요약

  • 조정은 일관된 시스템 행동을 위해 에이전트 활동 간 종속성을 관리
  • 중앙 집중식 조정은 전역 최적화를 제공하지만 확장성이 제한됨
  • 분산형 조정은 P2P 상호작용을 통해 더 나은 확장성을 제공
  • 협상을 통해 에이전트가 반복적인 제안을 통해 합의에 도달
  • 투표 및 합의 메커니즘이 집단 결정을 위한 선호도를 집계
  • Raft와 같은 합의 알고리즘이 실패에도 불구하고 합의를 보장
  • 시장 기반 조정이 자원 할당을 위해 경매와 가격 책정을 사용
  • 계획 조정이 충돌을 해결하면서 개별 계획을 병합
  • 군집 조정이 간단한 로컬 규칙을 통해 복잡한 행동을 달성
  • 효과적인 조정은 집단 이익을 위해 개별 행동을 조화시켜 弘益人間을 구현
