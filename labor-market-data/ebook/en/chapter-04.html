Chapter 4: Phase 1 - Data Format Specification

4.1 JSON Schema Foundation

WIA-SOC-020 Phase 1 defines the fundamental data structures for labor market information exchange using JSON Schema. This ensures data consistency, validates structure, and enables automated processing across platforms.

{
  "$schema": "https://json-schema.org/draft/2020-12/schema",
  "title": "WIA-SOC-020 Worker Profile",
  "type": "object",
  "properties": {
    "workerId": {"type": "string", "pattern": "^WKR-[0-9]{4}-[0-9]{6}$"},
    "personalInfo": {
      "type": "object",
      "properties": {
        "name": {"type": "string"},
        "dateOfBirth": {"type": "string", "format": "date"},
        "location": {"$ref": "#/definitions/Location"}
      },
      "required": ["name"]
    },
    "skills": {
      "type": "array",
      "items": {"$ref": "#/definitions/Skill"}
    },
    "experience": {
      "type": "array",
      "items": {"$ref": "#/definitions/Employment"}
    }
  },
  "required": ["workerId", "personalInfo"]
}

4.7 Field Definitions

Comprehensive field definitions ensure semantic consistency across implementations:

  • workerId: Unique identifier following pattern WKR-YYYY-NNNNNN
  • employmentType: Enum [full-time, part-time, contract, freelance, temporary, seasonal]
  • workArrangement: Enum [on-site, remote, hybrid, distributed]
  • occupationCode: SOC 2018 code with ISCO-08 cross-reference
  • industry: NAICS 2022 code with ISIC Rev. 4 mapping
  • skillProficiency: Integer 1-5 scale standardized across all skills
  • currency: ISO 4217 three-letter currency code
  • region: ISO 3166-2 subdivision code or city-level geocode

4.8 Validation Rules

Business logic validation ensures data quality beyond structural correctness:

  • Start dates must precede end dates in employment records
  • Salary ranges: min ≤ max, both within reasonable bounds for occupation
  • Skill proficiency must have supporting evidence (certification, experience)
  • Employment gaps >6 months should include reason code
  • Industry codes must be active (not deprecated) in current NAICS version
  • Location codes validated against authoritative geographic databases
