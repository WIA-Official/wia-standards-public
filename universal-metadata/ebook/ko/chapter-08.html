챕터 요약

핵심 요점:

  1. REST API는 콘텐츠 협상(application/ld+json, application/rdf+xml, text/turtle)을 통해 HTTP 엔드포인트를 통해 메타데이터를 노출하여, 클라이언트가 선호하는 직렬화 형식으로 Dublin Core 및 Schema.org 레코드를 검색할 수 있게 합니다.
  2. OAI-PMH(Open Archives Initiative Protocol for Metadata Harvesting)는 저장소 간의 대량 메타데이터 동기화를 가능하게 하며, 연합 발견 시스템을 위한 날짜 범위, 세트 및 메타데이터 형식별 선택적 수집을 지원합니다.
  3. SWORD(Simple Web-service Offering Repository Deposit)는 표준화된 메타데이터 입금 워크플로를 제공하여, 연관된 디지털 객체와 함께 Dublin Core 레코드를 기관 저장소 및 보존 시스템에 자동으로 수집할 수 있게 합니다.
  4. 웹훅 알림은 유니버설 타임스탬프와 통합되어 구독된 시스템에 실시간 메타데이터 변경 이벤트를 푸시하며, 다운스트림 애플리케이션이 메타데이터 업데이트에 즉시 반응하는 이벤트 기반 아키텍처를 가능하게 합니다.
  5. ETL 파이프라인은 레거시 메타데이터를 WIA-CORE-008 준수 JSON-LD로 변환하며, 독점 스키마를 Dublin Core 및 Schema.org 어휘에 매핑하면서 JSON Schema 및 SHACL 제약 조건에 대해 검증합니다.
  6. GraphQL 연합은 여러 서비스가 스키마 조각을 기여하는 분산 메타데이터 아키텍처를 가능하게 하며, 데이터 저장소를 중앙 집중화하거나 스키마 복제를 요구하지 않고 통합 쿼리 인터페이스를 제공합니다.
  7. SPARQL 연합(SERVICE 키워드)은 분산 RDF 엔드포인트를 쿼리하여, 의미론적 관계를 보존하고 복잡한 크로스 저장소 조인을 지원하면서 이질적인 트리플 스토어에서 메타데이터를 집계합니다.

복습 문제

  1. 콘텐츠 협상을 사용한 유니버설 메타데이터용 REST API를 설계하십시오. Accept 헤더(application/ld+json, text/turtle, application/rdf+xml)가 응답 형식을 어떻게 결정합니까? 여러 직렬화로 Dublin Core 레코드를 반환하는 엔드포인트를 구현하십시오.
  2. OAI-PMH 수집 프로토콜을 설명하십시오. ListRecords, GetRecord 및 ListIdentifiers 동사가 메타데이터 동기화를 어떻게 가능하게 합니까? 수백만 개의 레코드가 있는 대규모 Dublin Core 저장소에 대해 resumptionToken을 사용한 수집 전략을 설계하십시오.
  3. SWORD 메타데이터 입금 워크플로를 구현하십시오. 클라이언트가 디지털 객체와 함께 Dublin Core 메타데이터를 어떻게 패키징합니까? 제출된 메타데이터가 저장소 시스템에 수집되기 전에 Schema.org 요구 사항을 충족하도록 보장하는 검증 규칙은 무엇입니까?
  4. 유니버설 타임스탬프를 사용한 웹훅 기반 메타데이터 알림을 설계하십시오. 어떤 이벤트가 웹훅을 트리거합니까(생성, 업데이트, 삭제)? 시간적 서명이 웹훅 페이로드 진위성을 보장하고 재생 공격을 방지하는 방법은 무엇입니까?
  5. 레거시 메타데이터를 WIA-CORE-008로 마이그레이션하는 ETL 파이프라인을 생성하십시오. 독점 XML 스키마를 Dublin Core 요소에 어떻게 매핑합니까? 변환된 JSON-LD가 SHACL 제약 조건 및 Schema.org 어휘를 충족하도록 보장하는 검증 검사는 무엇입니까?
  6. 분산 메타데이터를 위한 GraphQL 연합을 설명하십시오. 여러 서비스가 스키마 타입(CMS의 Article, LDAP의 Person, 저장소의 Dataset)을 어떻게 기여합니까? 서비스 경계를 넘어 메타데이터를 조인하는 연합 쿼리를 구현하십시오.
  7. SERVICE 키워드를 사용하여 SPARQL 연합 쿼리를 작성하십시오. 원격 엔드포인트에서 Dublin Core 메타데이터를 어떻게 조인합니까? 분산 트리플 스토어 쿼리와 중앙 집중식 저장소 쿼리 시 어떤 성능 고려 사항이 있습니까?
