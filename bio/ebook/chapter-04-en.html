Chapter 4: Clinical Trial Data Management

Standards for Clinical Data Capture, Quality, and Regulatory Submission

4.1 Introduction to Clinical Data Standards

Clinical trials are the critical pathway from laboratory discovery to patient benefit. The WIA-BIO clinical trial data management standards establish comprehensive requirements for data capture, quality assurance, adverse event reporting, and regulatory submission that align with global regulatory expectations while leveraging modern biotechnology capabilities.

Effective clinical data management is essential for demonstrating the safety and efficacy of biotechnology products, from gene therapies to personalized medicine approaches. WIA-BIO standards integrate with established frameworks including CDISC, ICH E6 (GCP), and HL7 FHIR while providing specific guidance for biotechnology applications.

4.2 Clinical Trial Phases and Data Requirements

Each phase of clinical development has distinct data requirements that must be addressed through appropriate data management strategies:

Phase Primary Objectives Participants Key Data Types
Phase I Safety, dosing, PK/PD 20-100 healthy volunteers Safety, PK, biomarkers
Phase II Efficacy, dose-response 100-500 patients Efficacy endpoints, safety
Phase III Confirm efficacy, monitor AEs 1,000-5,000 patients Primary/secondary endpoints
Phase IV Post-market surveillance Thousands of patients Long-term safety, RWE

4.3 CDISC Standards Implementation

WIA-BIO mandates compliance with Clinical Data Interchange Standards Consortium (CDISC) standards for regulatory submissions: 

// CDISC SDTM Domain Structure
interface SDTMDomain {
    domainCode: string;          // e.g., "DM", "AE", "LB"
    datasetName: string;
    description: string;
    class: 'General Observations' | 'Events' | 'Findings' | 'Special Purpose';
    structure: 'One record per subject' | 'One record per event' | 'One record per result';
}

// Demographics Domain (DM)
interface DMDomain {
    STUDYID: string;             // Study Identifier
    DOMAIN: 'DM';                // Domain Abbreviation
    USUBJID: string;             // Unique Subject Identifier
    SUBJID: string;              // Subject Identifier for the Study
    RFSTDTC: string;             // Subject Reference Start Date/Time
    RFENDTC: string;             // Subject Reference End Date/Time
    SITEID: string;              // Study Site Identifier
    BRTHDTC?: string;            // Date/Time of Birth
    AGE: number;                 // Age
    AGEU: 'YEARS' | 'MONTHS';    // Age Units
    SEX: 'M' | 'F' | 'U';        // Sex
    RACE: string;                // Race
    ETHNIC: string;              // Ethnicity
    ARMCD: string;               // Planned Arm Code
    ARM: string;                 // Description of Planned Arm
    COUNTRY: string;             // Country
}

// Adverse Events Domain (AE)
interface AEDomain {
    STUDYID: string;
    DOMAIN: 'AE';
    USUBJID: string;
    AESEQ: number;               // Sequence Number
    AETERM: string;              // Reported Term for AE
    AEDECOD: string;             // Dictionary-Derived Term
    AEBODSYS: string;            // Body System or Organ Class
    AESEV: 'MILD' | 'MODERATE' | 'SEVERE';  // Severity
    AESER: 'Y' | 'N';            // Serious Event
    AEACN: string;               // Action Taken with Study Treatment
    AEREL: string;               // Causality
    AEOUT: string;               // Outcome of Adverse Event
    AESTDTC: string;             // Start Date/Time of AE
    AEENDTC?: string;            // End Date/Time of AE
}

4.4 Electronic Data Capture (EDC) Requirements

WIA-BIO specifies requirements for electronic data capture systems used in clinical trials:

EDC System Requirements

  • 21 CFR Part 11 Compliance: Electronic signatures, audit trails, access controls
  • Data Validation: Real-time edit checks, range validation, consistency checks
  • Audit Trail: Complete history of all data changes with user, timestamp, reason
  • Security: Role-based access, encryption at rest and in transit
  • Integration: HL7 FHIR, CDISC ODM support for data exchange
  • Backup: Automated backups with validated recovery procedures
// EDC System Configuration Interface
interface EDCConfiguration {
    studyId: string;
    protocolVersion: string;

    // Form definitions
    forms: CRFForm[];

    // Validation rules
    editChecks: EditCheck[];

    // User management
    roles: UserRole[];

    // Integration endpoints
    integrations: {
        labSystems: LabIntegration[];
        eProSystems: eProIntegration[];
        safetyDatabases: SafetyIntegration[];
    };

    // Compliance settings
    compliance: {
        cfr21Part11: boolean;
        gdpr: boolean;
        hipaa: boolean;
    };
}

interface EditCheck {
    checkId: string;
    description: string;
    formId: string;
    fieldId: string;
    checkType: 'RANGE' | 'CONSISTENCY' | 'CONDITIONAL' | 'CROSS_FORM';
    expression: string;
    severity: 'ERROR' | 'WARNING' | 'INFO';
    message: string;
}

4.5 Adverse Event Reporting

Timely and accurate adverse event reporting is critical for patient safety. WIA-BIO establishes stringent requirements for AE capture and reporting:

Event Type Reporting Timeline Report Destination Format
Fatal/Life-threatening SAE 7 calendar days Regulatory authority, IRB CIOMS-I, MedWatch
Other Serious AE 15 calendar days Regulatory authority, IRB CIOMS-I, MedWatch
Non-serious AE Per protocol schedule Safety database EDC entry
SUSAR (Expected) 15 calendar days IND Safety Report Narrative + CIOMS 
// Adverse Event Processing Workflow
interface AEProcessingWorkflow {
    stages: {
        intake: {
            source: 'EDC' | 'SITE_CALL' | 'PATIENT_REPORT';
            initialAssessment: boolean;

        medicalReview: {
            reviewerQualification: 'MD' | 'PharmD';

            causalityAssessment: boolean;
            expectednessAssessment: boolean;
        };

        regulatory: {
            reportabilityDetermination: boolean;
            narrativeGeneration: boolean;
            regulatorySubmission: boolean;
            trackingNumber: string;
        };

        followUp: {
            resolutionTracking: boolean;
            sequelaeDocumentation: boolean;
            outcomeRecording: boolean;
        };
    };
}

4.6 Data Quality Management

Ensuring data quality is essential for regulatory acceptance and scientific validity. WIA-BIO mandates comprehensive quality management processes: 

// Data Quality Metrics Interface
interface DataQualityMetrics {
    studyId: string;
    reportDate: Date;

    // Completeness metrics
    completeness: {
        overallCompletionRate: number;    // Target: > 95%
        criticalFieldCompletion: number;  // Target: > 99%
        missingDataRate: number;          // Target: < 2%
    };

    // Accuracy metrics
    accuracy: {
        queryRate: number;                // Target: < 5%
        errorRate: number;                // Target: < 1%
        discrepancyRate: number;          // Target: < 3%
    };

    // Timeliness metrics
    timeliness: {
        dataEntryLag: number;             // Days from visit
        queryResolutionTime: number;      // Days average
        lockToDBLockTime: number;         // Days
    };

    // Consistency metrics
    consistency: {
        protocolDeviationRate: number;
        editCheckFailureRate: number;
        crossFormConsistency: number;     // Target: > 98%
    };
}

// Query Management System
interface QueryManagement {
    queryId: string;
    siteId: string;
    subjectId: string;
    formName: string;
    fieldName: string;
    queryType: 'AUTO' | 'MANUAL';
    status: 'OPEN' | 'ANSWERED' | 'CLOSED' | 'CANCELLED';
    priority: 'HIGH' | 'MEDIUM' | 'LOW';
    queryText: string;
    response?: string;
    openedDate: Date;
    closedDate?: Date;
    daysOpen: number;
}

4.7 Biotechnology-Specific Data Elements

Biotechnology clinical trials require specialized data elements not covered by standard CDISC domains. WIA-BIO defines extended data structures: 

// Gene Therapy Trial Data Extensions
interface GeneTherapyData {
    // Vector information
    vector: {
        type: 'AAV' | 'Lentivirus' | 'Adenovirus' | 'mRNA-LNP';
        serotype?: string;
        titer: number;
        tiferUnit: string;
        lotNumber: string;
        manufacturingDate: Date;
    };

    // Dosing information
    dosing: {
        route: 'IV' | 'IT' | 'IM' | 'SC' | 'Intravitreal' | 'Other';
        doseLevel: number;
        doseUnit: string;           // e.g., "vg/kg"
        administrationDate: Date;
        premedication: string[];
    };

    // Biomarker tracking
    biomarkers: {
        transductionEfficiency?: number;
        transgeneExpression: {
            tissue: string;
            level: number;
            unit: string;
            timepoint: string;
        }[];
        immunogenicity: {
            aabPresence: boolean;
            neutralizingAb: boolean;
            tCellResponse: boolean;
        };
    };

    // Long-term follow-up
    ltfu: {
        followUpYears: number;      // Typically 5-15 years
        malignancyScreening: boolean;
        insertionalMutagenesis: boolean;
        durabilityAssessment: boolean;
    };
}

4.8 Regulatory Submission Standards

WIA-BIO specifies data formats and structures required for regulatory submissions across major authorities:

Key Takeaways

  • CDISC Compliance: SDTM for raw data, ADaM for analysis datasets mandatory for FDA submissions
  • EDC Requirements: 21 CFR Part 11 compliant systems with comprehensive audit trails
  • AE Reporting: Strict timelines for serious adverse event reporting to regulatory authorities
  • Data Quality: Continuous monitoring with defined metrics and thresholds
  • Biotech Extensions: Specialized data structures for gene therapy, cell therapy, and other biotechnology modalities
  • Global Alignment: Standards designed to meet FDA, EMA, PMDA, and other regulatory requirements
