Chapter 5: Bioinformatics Pipeline Standards

Workflow Standards for Reproducible Computational Biology

5.1 Introduction to Pipeline Standards

Bioinformatics pipelines are the computational backbone of modern biotechnology, transforming raw data into actionable biological insights. The WIA-BIO bioinformatics pipeline standards establish comprehensive requirements for workflow definition, containerization, reproducibility, and validation that ensure computational analyses are reliable, portable, and auditable.

As biotechnology increasingly relies on computational analysis for discovery and clinical decision-making, standardized pipelines become essential for regulatory acceptance, scientific reproducibility, and collaborative research. WIA-BIO standards support multiple workflow definition languages while establishing common requirements for all implementations.

5.2 Workflow Definition Languages

WIA-BIO supports three primary workflow definition languages, each optimized for different use cases:

Language Strengths Best For Execution Engines
CWL (Common Workflow Language) Portability, standards-based Multi-platform deployment cwltool, Toil, Arvados
Nextflow Flexibility, active community Complex pipelines, HPC Nextflow engine
WDL (Workflow Description Language) Cloud-native, readable Cloud genomics Cromwell, miniwdl
Snakemake Python integration Python-centric teams Snakemake engine

5.3 CWL Implementation Standards

Common Workflow Language (CWL) provides the most portable workflow definitions. WIA-BIO specifies CWL v1.2 as the minimum supported version: 

# WIA-BIO Compliant CWL Workflow Example
cwlVersion: v1.2
class: Workflow
label: WIA-BIO Variant Calling Pipeline
doc: |
  Standard variant calling pipeline for WGS data
  following WIA-BIO specifications v1.0

requirements:
  SubworkflowFeatureRequirement: {}
  ScatterFeatureRequirement: {}
  MultipleInputFeatureRequirement: {}

inputs:
  sample_id:
    type: string
    doc: Unique sample identifier
  fastq_r1:
    type: File
    doc: Forward reads (FASTQ)
  fastq_r2:
    type: File
    doc: Reverse reads (FASTQ)
  reference_genome:
    type: File
    secondaryFiles:
      - .fai
      - .dict
      - ^.dict
    doc: Reference genome with indices

outputs:
  aligned_bam:
    type: File
    outputSource: alignment/bam_output
  variants_vcf:
    type: File
    outputSource: variant_calling/vcf_output
  qc_report:
    type: File
    outputSource: quality_control/report

steps:
  quality_control:
    run: tools/fastqc.cwl
    in:
      fastq_r1: fastq_r1
      fastq_r2: fastq_r2
    out: [report]

  alignment:
    run: tools/bwa_mem2.cwl
    in:
      sample_id: sample_id
      fastq_r1: fastq_r1
      fastq_r2: fastq_r2
      reference: reference_genome
    out: [bam_output]

  variant_calling:
    run: tools/deepvariant.cwl
    in:
      bam_input: alignment/bam_output
      reference: reference_genome
    out: [vcf_output]

5.4 Nextflow Pipeline Standards

Nextflow provides excellent flexibility for complex pipelines. WIA-BIO specifies DSL2 syntax requirements: 

#!/usr/bin/env nextflow
nextflow.enable.dsl=2

/*
 * WIA-BIO Compliant Nextflow Pipeline
 * RNA-Seq Analysis Pipeline v1.0
 */

// Pipeline parameters with WIA-BIO defaults
params.reads = null
params.reference = null
params.gtf = null
params.outdir = './results'
params.wia_bio_version = '1.0'

// Validate required parameters
if (!params.reads) { exit 1, "Error: --reads parameter required" }
if (!params.reference) { exit 1, "Error: --reference parameter required" }

// Define channels
reads_ch = Channel.fromFilePairs(params.reads)
reference_ch = Channel.fromPath(params.reference)

// Quality Control Process
process FASTQC {
    tag "$sample_id"
    publishDir "${params.outdir}/qc", mode: 'copy'

    container 'quay.io/biocontainers/fastqc:0.11.9--0'

    input:
    tuple val(sample_id), path(reads)

    output:
    path("*_fastqc.{html,zip}"), emit: reports

    script:
    """
    fastqc -t ${task.cpus} ${reads}
    """
}

// Alignment Process
process STAR_ALIGN {
    tag "$sample_id"
    publishDir "${params.outdir}/alignment", mode: 'copy'

    container 'quay.io/biocontainers/star:2.7.10a--h9ee0642_0'

    input:
    tuple val(sample_id), path(reads)
    path reference

    output:
    tuple val(sample_id), path("${sample_id}.Aligned.sortedByCoord.out.bam"), emit: bam
    path "${sample_id}.Log.final.out", emit: log

    script:
    """
    STAR --runThreadN ${task.cpus} \
         --genomeDir ${reference} \
         --readFilesIn ${reads[0]} ${reads[1]} \
         --outFileNamePrefix ${sample_id}. \
         --outSAMtype BAM SortedByCoordinate
    """
}

// Quantification Process
process FEATURECOUNTS {
    tag "$sample_id"
    publishDir "${params.outdir}/counts", mode: 'copy'

    container 'quay.io/biocontainers/subread:2.0.1--h7132678_2'

    input:
    tuple val(sample_id), path(bam)
    path gtf

    output:
    path "${sample_id}.counts.txt", emit: counts

    script:
    """
    featureCounts -T ${task.cpus} \
                  -a ${gtf} \
                  -o ${sample_id}.counts.txt \
                  ${bam}
    """
}

// Main workflow
workflow {
    FASTQC(reads_ch)
    STAR_ALIGN(reads_ch, reference_ch)
    FEATURECOUNTS(STAR_ALIGN.out.bam, params.gtf)
}

5.5 Containerization Requirements

WIA-BIO mandates containerization for all production pipelines to ensure reproducibility:

Container Standards

  • Docker/OCI Compliance: All containers must be OCI-compatible
  • Immutable Tags: Use SHA256 digests or semantic versioning, never :latest
  • BioContainers Preference: Use community-maintained containers when available
  • Singularity Support: Containers must be Singularity-compatible for HPC environments
  • Size Optimization: Multi-stage builds to minimize image size
  • Security Scanning: All containers must pass vulnerability scanning
# WIA-BIO Compliant Dockerfile Template
FROM ubuntu:22.04 AS builder

LABEL maintainer="WIA-BIO Standards <standards@wia-bio.org>"
LABEL org.opencontainers.image.source="https://github.com/wia-bio/pipelines"
LABEL org.opencontainers.image.version="1.0.0"
LABEL org.wia-bio.pipeline.version="1.0"

# Install build dependencies
RUN apt-get update && apt-get install -y \
    build-essential \
    wget \
    && rm -rf /var/lib/apt/lists/*

# Build bioinformatics tool
WORKDIR /build
RUN wget https://example.com/tool-1.0.tar.gz \
    && tar xzf tool-1.0.tar.gz \
    && cd tool-1.0 \
    && make

# Production image
FROM ubuntu:22.04

# Copy only necessary files
COPY --from=builder /build/tool-1.0/bin/tool /usr/local/bin/

# Create non-root user
RUN useradd -m -s /bin/bash wia-bio
USER wia-bio

WORKDIR /data
ENTRYPOINT ["tool"]

5.6 Reproducibility Requirements

Reproducibility is essential for scientific validity and regulatory compliance. WIA-BIO mandates comprehensive provenance tracking: 

// WIA-BIO Pipeline Provenance Schema
interface PipelineProvenance {
    // Pipeline identification
    pipelineId: string;
    pipelineName: string;
    pipelineVersion: string;
    wiaBioCompliance: string;      // e.g., "1.0"

    // Execution context
    execution: {
        executionId: string;
        startTime: Date;
        endTime: Date;
        status: 'SUCCESS' | 'FAILED' | 'RUNNING';
        executor: string;           // e.g., "Nextflow 23.04.0"
        platform: string;           // e.g., "AWS Batch"
    };

    // Input provenance
    inputs: {
        name: string;
        path: string;
        sha256: string;
        size: number;
        source?: string;            // URL or repository
    }[];

    // Tool versions
    tools: {
        name: string;
        version: string;
        container: string;
        containerDigest: string;    // SHA256 of container
    }[];

    // Reference data
    references: {
        name: string;
        version: string;
        source: string;
        sha256: string;
    }[];

    // Output provenance
    outputs: {
        name: string;
        path: string;
        sha256: string;
        size: number;
    }[];

    // Resource usage
    resources: {
        totalCpuHours: number;
        peakMemoryGb: number;
        totalStorageGb: number;
        estimatedCost?: number;
    };
}

5.7 Validation and Testing

All WIA-BIO compliant pipelines must include comprehensive validation:

Test Type Purpose Requirement
Unit Tests Individual tool validation Mandatory for each process
Integration Tests Workflow connectivity Mandatory
Regression Tests Output consistency Mandatory with reference data
Performance Tests Resource benchmarking Recommended
Security Tests Vulnerability scanning Mandatory for clinical 
// Pipeline Validation Configuration
interface PipelineValidation {
    testSuite: {
        unitTests: {
            enabled: boolean;
            minCoverage: number;      // Minimum: 80%
            framework: string;        // e.g., "pytest-workflow"
        };

        integrationTests: {
            enabled: boolean;
            testDataPath: string;
            expectedOutputPath: string;
            tolerances: {
                numericTolerance: number;
                allowedMissingFields: string[];
            };
        };

        regressionTests: {
            enabled: boolean;
            referenceDataVersion: string;
            checksumValidation: boolean;
        };
    };

    continuousIntegration: {
        platform: 'GitHub Actions' | 'GitLab CI' | 'Jenkins';
        triggers: ('push' | 'pull_request' | 'schedule')[];
        testEnvironments: string[];  // e.g., ["ubuntu-22.04", "centos-8"]
    };
}

5.8 Clinical Pipeline Requirements

Pipelines used for clinical decision-making have additional requirements:

Key Takeaways

  • Workflow Languages: CWL, Nextflow, and WDL supported with specific version requirements
  • Containerization: All tools must be containerized with immutable version tags
  • Provenance: Complete tracking of inputs, tools, references, and outputs
  • Validation: Comprehensive testing including unit, integration, and regression tests
  • Clinical Grade: Additional validation and documentation for clinical applications
  • Reproducibility: Any execution must be reproducible given same inputs and container versions
