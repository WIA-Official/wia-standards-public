제5장: 생물정보학 파이프라인 표준

재현 가능한 계산 생물학을 위한 워크플로우 표준

5.1 파이프라인 표준 소개

생물정보학 파이프라인은 현대 생명공학의 계산적 중추로서, 원시 데이터를 실행 가능한 생물학적 통찰력으로 변환합니다. WIA-BIO 생물정보학 파이프라인 표준은 계산 분석이 신뢰할 수 있고, 이식 가능하며, 감사 가능하도록 보장하는 워크플로우 정의, 컨테이너화, 재현성 및 검증에 대한 포괄적인 요구사항을 수립합니다.

생명공학이 발견 및 임상 의사결정을 위해 계산 분석에 점점 더 의존함에 따라, 표준화된 파이프라인은 규제 승인, 과학적 재현성 및 협력 연구에 필수적이 됩니다. WIA-BIO 표준은 모든 구현에 대한 공통 요구사항을 수립하면서 여러 워크플로우 정의 언어를 지원합니다.

5.2 워크플로우 정의 언어

WIA-BIO는 각각 다른 사용 사례에 최적화된 세 가지 주요 워크플로우 정의 언어를 지원합니다:

언어 강점 최적 용도 실행 엔진
CWL 이식성, 표준 기반 다중 플랫폼 배포 cwltool, Toil, Arvados
Nextflow 유연성, 활발한 커뮤니티 복잡한 파이프라인, HPC Nextflow 엔진
WDL 클라우드 네이티브, 가독성 클라우드 유전체학 Cromwell, miniwdl
Snakemake Python 통합 Python 중심 팀 Snakemake 엔진

5.3 CWL 구현 표준

Common Workflow Language (CWL)는 가장 이식성 있는 워크플로우 정의를 제공합니다. WIA-BIO는 CWL v1.2를 최소 지원 버전으로 지정합니다: 

# WIA-BIO 준수 CWL 워크플로우 예시
cwlVersion: v1.2
class: Workflow
label: WIA-BIO 변이 호출 파이프라인
doc: |
  WIA-BIO 사양 v1.0을 따르는
  WGS 데이터용 표준 변이 호출 파이프라인

requirements:
  SubworkflowFeatureRequirement: {}
  ScatterFeatureRequirement: {}
  MultipleInputFeatureRequirement: {}

inputs:
  sample_id:
    type: string
    doc: 고유 샘플 식별자
  fastq_r1:
    type: File
    doc: 정방향 리드 (FASTQ)
  fastq_r2:
    type: File
    doc: 역방향 리드 (FASTQ)
  reference_genome:
    type: File
    secondaryFiles:
      - .fai
      - .dict
      - ^.dict
    doc: 인덱스가 있는 참조 유전체

outputs:
  aligned_bam:
    type: File
    outputSource: alignment/bam_output
  variants_vcf:
    type: File
    outputSource: variant_calling/vcf_output
  qc_report:
    type: File
    outputSource: quality_control/report

steps:
  quality_control:
    run: tools/fastqc.cwl
    in:
      fastq_r1: fastq_r1
      fastq_r2: fastq_r2
    out: [report]

  alignment:
    run: tools/bwa_mem2.cwl
    in:
      sample_id: sample_id
      fastq_r1: fastq_r1
      fastq_r2: fastq_r2
      reference: reference_genome
    out: [bam_output]

  variant_calling:
    run: tools/deepvariant.cwl
    in:
      bam_input: alignment/bam_output
      reference: reference_genome
    out: [vcf_output]

5.4 Nextflow 파이프라인 표준

Nextflow는 복잡한 파이프라인을 위한 탁월한 유연성을 제공합니다. WIA-BIO는 DSL2 문법 요구사항을 지정합니다: 

#!/usr/bin/env nextflow
nextflow.enable.dsl=2

/*
 * WIA-BIO 준수 Nextflow 파이프라인
 * RNA-Seq 분석 파이프라인 v1.0
 */

// WIA-BIO 기본값이 있는 파이프라인 매개변수
params.reads = null
params.reference = null
params.gtf = null
params.outdir = './results'
params.wia_bio_version = '1.0'

// 필수 매개변수 검증
if (!params.reads) { exit 1, "오류: --reads 매개변수 필요" }
if (!params.reference) { exit 1, "오류: --reference 매개변수 필요" }

// 채널 정의
reads_ch = Channel.fromFilePairs(params.reads)
reference_ch = Channel.fromPath(params.reference)

// 품질 관리 프로세스
process FASTQC {
    tag "$sample_id"
    publishDir "${params.outdir}/qc", mode: 'copy'

    container 'quay.io/biocontainers/fastqc:0.11.9--0'

    input:
    tuple val(sample_id), path(reads)

    output:
    path("*_fastqc.{html,zip}"), emit: reports

    script:
    """
    fastqc -t ${task.cpus} ${reads}
    """
}

// 정렬 프로세스
process STAR_ALIGN {
    tag "$sample_id"
    publishDir "${params.outdir}/alignment", mode: 'copy'

    container 'quay.io/biocontainers/star:2.7.10a--h9ee0642_0'

    input:
    tuple val(sample_id), path(reads)
    path reference

    output:
    tuple val(sample_id), path("${sample_id}.Aligned.sortedByCoord.out.bam"), emit: bam
    path "${sample_id}.Log.final.out", emit: log

    script:
    """
    STAR --runThreadN ${task.cpus} \\
         --genomeDir ${reference} \\
         --readFilesIn ${reads[0]} ${reads[1]} \\
         --outFileNamePrefix ${sample_id}. \\
         --outSAMtype BAM SortedByCoordinate
    """
}

// 주 워크플로우
workflow {
    FASTQC(reads_ch)
    STAR_ALIGN(reads_ch, reference_ch)
}

5.5 컨테이너화 요구사항

WIA-BIO는 재현성을 보장하기 위해 모든 프로덕션 파이프라인에 대한 컨테이너화를 의무화합니다:

컨테이너 표준

  • Docker/OCI 준수: 모든 컨테이너는 OCI 호환이어야 함
  • 불변 태그: SHA256 다이제스트 또는 시맨틱 버전 사용, :latest 절대 사용 금지
  • BioContainers 선호: 가능한 경우 커뮤니티 유지 컨테이너 사용
  • Singularity 지원: HPC 환경을 위해 Singularity 호환 필수
  • 크기 최적화: 이미지 크기 최소화를 위한 멀티스테이지 빌드
  • 보안 스캔: 모든 컨테이너는 취약점 스캔 통과 필수
# WIA-BIO 준수 Dockerfile 템플릿
FROM ubuntu:22.04 AS builder

LABEL maintainer="WIA-BIO Standards <standards@wia-bio.org>"
LABEL org.opencontainers.image.source="https://github.com/wia-bio/pipelines"
LABEL org.opencontainers.image.version="1.0.0"
LABEL org.wia-bio.pipeline.version="1.0"

# 빌드 의존성 설치
RUN apt-get update && apt-get install -y \
    build-essential \
    wget \
    && rm -rf /var/lib/apt/lists/*

# 생물정보학 도구 빌드
WORKDIR /build
RUN wget https://example.com/tool-1.0.tar.gz \
    && tar xzf tool-1.0.tar.gz \
    && cd tool-1.0 \
    && make

# 프로덕션 이미지
FROM ubuntu:22.04

# 필요한 파일만 복사
COPY --from=builder /build/tool-1.0/bin/tool /usr/local/bin/

# 비루트 사용자 생성
RUN useradd -m -s /bin/bash wia-bio
USER wia-bio

WORKDIR /data
ENTRYPOINT ["tool"]

5.6 재현성 요구사항

재현성은 과학적 타당성과 규제 준수에 필수적입니다. WIA-BIO는 포괄적인 출처 추적을 의무화합니다: 

// WIA-BIO 파이프라인 출처 스키마
interface PipelineProvenance {
    // 파이프라인 식별
    pipelineId: string;          // 파이프라인 ID
    pipelineName: string;        // 파이프라인 이름
    pipelineVersion: string;     // 파이프라인 버전
    wiaBioCompliance: string;    // 예: "1.0"

    // 실행 컨텍스트
    execution: {
        executionId: string;     // 실행 ID
        startTime: Date;         // 시작 시간
        endTime: Date;           // 종료 시간
        status: 'SUCCESS' | 'FAILED' | 'RUNNING';
        executor: string;        // 예: "Nextflow 23.04.0"
        platform: string;        // 예: "AWS Batch"
    };

    // 입력 출처
    inputs: {
        name: string;            // 이름
        path: string;            // 경로
        sha256: string;          // SHA256 해시
        size: number;            // 크기
        source?: string;         // URL 또는 저장소
    }[];

    // 도구 버전
    tools: {
        name: string;            // 도구명
        version: string;         // 버전
        container: string;       // 컨테이너
        containerDigest: string; // 컨테이너 SHA256
    }[];

    // 참조 데이터
    references: {
        name: string;            // 이름
        version: string;         // 버전
        source: string;          // 소스
        sha256: string;          // SHA256 해시
    }[];

    // 출력 출처
    outputs: {
        name: string;            // 이름
        path: string;            // 경로
        sha256: string;          // SHA256 해시
        size: number;            // 크기
    }[];

    // 리소스 사용량
    resources: {
        totalCpuHours: number;   // 총 CPU 시간
        peakMemoryGb: number;    // 최대 메모리 (GB)
        totalStorageGb: number;  // 총 스토리지 (GB)
        estimatedCost?: number;  // 예상 비용
    };
}

5.7 검증 및 테스트

모든 WIA-BIO 준수 파이프라인은 포괄적인 검증을 포함해야 합니다:

테스트 유형 목적 요구사항
단위 테스트 개별 도구 검증 각 프로세스에 필수
통합 테스트 워크플로우 연결성 필수
회귀 테스트 출력 일관성 참조 데이터와 함께 필수
성능 테스트 리소스 벤치마킹 권장
보안 테스트 취약점 스캔 임상용 필수 
// 파이프라인 검증 구성
interface PipelineValidation {
    testSuite: {
        unitTests: {
            enabled: boolean;
            minCoverage: number;      // 최소: 80%
            framework: string;        // 예: "pytest-workflow"
        };

        integrationTests: {
            enabled: boolean;
            testDataPath: string;     // 테스트 데이터 경로
            expectedOutputPath: string; // 예상 출력 경로
            tolerances: {
                numericTolerance: number; // 수치 허용오차
                allowedMissingFields: string[]; // 허용 누락 필드
            };
        };

        regressionTests: {
            enabled: boolean;
            referenceDataVersion: string; // 참조 데이터 버전
            checksumValidation: boolean;  // 체크섬 검증
        };
    };

    continuousIntegration: {
        platform: 'GitHub Actions' | 'GitLab CI' | 'Jenkins';
        triggers: ('push' | 'pull_request' | 'schedule')[];
        testEnvironments: string[];  // 예: ["ubuntu-22.04", "centos-8"]
    };
}

5.8 임상 파이프라인 요구사항

임상 의사결정에 사용되는 파이프라인에는 추가 요구사항이 있습니다:

핵심 요약

  • 워크플로우 언어: 특정 버전 요구사항과 함께 CWL, Nextflow, WDL 지원
  • 컨테이너화: 모든 도구는 불변 버전 태그로 컨테이너화 필수
  • 출처: 입력, 도구, 참조 및 출력의 완전한 추적
  • 검증: 단위, 통합 및 회귀 테스트를 포함한 포괄적인 테스트
  • 임상 등급: 임상 응용을 위한 추가 검증 및 문서화
  • 재현성: 동일한 입력과 컨테이너 버전이 주어지면 모든 실행이 재현 가능해야 함
