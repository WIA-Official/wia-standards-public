5.1 REST API Design Principles

WIA AI-City APIs follow RESTful conventions with additional constraints to ensure consistency across all compliant implementations.

Base URL Structure

https://api.{city-domain}/wia/v1/{resource}

Examples:

  • https://api.seoul.ai-city.kr/wia/v1/traffic
  • https://api.barcelona.smart-city.es/wia/v1/environment
  • https://api.singapore.gov.sg/wia/v1/citizens

Standard Response Format

{
  "@context": "https://schema.wia.org/ai-city/v1",
  "success": true,
  "data": { ... },
  "meta": {
    "requestId": "REQ-2024-ABC123",
    "timestamp": "2024-12-24T10:30:00Z",
    "pagination": {
      "page": 1,
      "pageSize": 20,
      "totalItems": 156,
      "totalPages": 8
    }
  }
}

Error Response Format

{
  "@context": "https://schema.wia.org/ai-city/v1",
  "success": false,
  "error": {
    "code": "CONSENT_REQUIRED",
    "message": "Access to location data requires citizen consent",
    "details": {
      "dataType": "L2",
      "requiredScope": "location:read"
    },
    "helpUrl": "https://docs.wia.org/errors/CONSENT_REQUIRED"
  },
  "meta": {
    "requestId": "REQ-2024-XYZ789",
    "timestamp": "2024-12-24T10:30:00Z"
  }
}

5.2 Core API Endpoints

Traffic Domain

GET/traffic/sensors

List all traffic sensors with optional filtering

Query Parameters:

  • zone - Filter by zone ID
  • status - Filter by status (active, maintenance, offline)
  • type - Filter by sensor type

GET/traffic/sensors/{sensorId}/readings

Get historical readings from a specific sensor

GET/traffic/congestion

Real-time city-wide congestion status

Environment Domain

GET/environment/air-quality

Current air quality index across zones

GET/environment/weather

Current weather conditions and forecast

Sharing Engine Domain

GET/sharing/stats

Aggregate contribution statistics

GET/sharing/allocations

Fund allocation details by category

5.3 Citizen Portal APIs

The citizen portal APIs provide the technical foundation for data sovereignty rights. These endpoints enable citizens to view, manage, and control their personal data.

Consent Management

GET/citizens/me/consents

List all active consent grants

POST/citizens/me/consents

Grant new consent for data usage

{
  "dataType": "L2",
  "purpose": "traffic_optimization",
  "scope": ["location", "travel_patterns"],
  "expiresAt": "2025-12-24T00:00:00Z"
}

DELETE/citizens/me/consents/{consentId}

Revoke an existing consent

Data Access

GET/citizens/me/data

View all data the city holds about you

GET/citizens/me/access-log

See who accessed your data and when

POST/citizens/me/data/export

Request data export (portability right)

DELETE/citizens/me/data

Request complete data deletion (erasure right)

5.4 GraphQL Interface

For complex queries and real-time subscriptions, WIA AI-City provides a GraphQL interface alongside REST.

Schema Overview

type Query {
  traffic(zone: ID, limit: Int): TrafficData
  environment(type: EnvironmentType): EnvironmentData
  sharing: SharingStats
  citizen: CitizenProfile
}

type Mutation {
  grantConsent(input: ConsentInput!): Consent
  revokeConsent(consentId: ID!): Boolean
  requestDataExport: ExportRequest
  requestDataDeletion: DeletionRequest
}

type Subscription {
  trafficUpdates(zone: ID): TrafficReading
  alertsForZone(zone: ID!): Alert
  congestionChanges: CongestionEvent
}

Example Queries

# Get traffic data with nested sensor information
query {
  traffic(zone: "gangnam") {
    sensors {
      id
      status
      lastReading {
        vehicleCount
        averageSpeed
        timestamp
      }
    }
    congestionLevel
  }
}
# Subscribe to real-time traffic updates
subscription {
  trafficUpdates(zone: "gangnam") {
    sensorId
    vehicleCount
    averageSpeed
    congestionLevel
    timestamp
  }
}

5.5 Authentication & Authorization

WIA AI-City uses OAuth 2.0 with OpenID Connect for authentication, supporting multiple grant types for different use cases.

Supported Grant Types

Grant Type Use Case Token Lifetime
Authorization Code + PKCE Citizen-facing applications 1 hour (refresh: 30 days)
Client Credentials Machine-to-machine (city systems) 1 hour
Device Authorization IoT devices, kiosks 15 minutes (refresh: 90 days)

Scope Hierarchy

# Data type scopes (classification levels)
L1:*         # All L1 data (never granted externally)
L2:read      # Read location data
L2:write     # Update location preferences
L3:read      # Read behavioral data
L4:read      # Read public data (default)

# Domain scopes
traffic:read
environment:read
sharing:read
citizen:profile:read
citizen:consents:manage
citizen:data:export
citizen:data:delete

Token Request Example

POST /oauth/token
Content-Type: application/x-www-form-urlencoded

grant_type=authorization_code
&code=AUTH_CODE_HERE
&redirect_uri=https://myapp.example.com/callback
&client_id=APP_CLIENT_ID
&code_verifier=PKCE_VERIFIER

5.7 SDK Implementation

TypeScript SDK

npm install @wia/ai-city-sdk
import { AICityClient, TrafficAPI, CitizenAPI } from '@wia/ai-city-sdk';

// Initialize client
const client = new AICityClient({
  baseUrl: 'https://api.seoul.ai-city.kr/wia/v1',
  apiKey: process.env.WIA_API_KEY,
  cityId: 'AIC-2024-SEOUL1'
});

// Traffic data
const traffic = await client.traffic.getCongestion({ zone: 'gangnam' });
console.log(`Congestion level: ${traffic.congestionLevel}`);

// Citizen consent management
const consents = await client.citizen.listConsents();
await client.citizen.revokeConsent('CST-ABC123');

Python SDK

pip install wia-ai-city
from wia_ai_city import AICityClient

client = AICityClient(
    base_url='https://api.seoul.ai-city.kr/wia/v1',
    api_key=os.environ['WIA_API_KEY'],
    city_id='AIC-2024-SEOUL1'
)

# Get air quality
air_quality = client.environment.get_air_quality(zone='gangnam')
print(f"PM2.5: {air_quality.pm25}")

# Export citizen data
export_request = client.citizen.request_data_export(format='json-ld')
print(f"Export ID: {export_request.id}")

Rust SDK

// Cargo.toml
[dependencies]
wia-ai-city = "1.0"
use wia_ai_city::{Client, Result};

#[tokio::main]
async fn main() -> Result<()> {
    let client = Client::from_env().await?;

    // Get traffic sensors
    let sensors = client.traffic().sensors()
        .zone("gangnam")
        .status("active")
        .fetch().await?;

    for sensor in sensors {
        println!("{}: {} vehicles", sensor.id, sensor.reading.vehicle_count);
    }

    Ok(())
}

5.9 Chapter Summary

Key Takeaways

  • REST Design: APIs follow RESTful conventions with JSON-LD responses, standard error formats, and consistent pagination.
  • Core Endpoints: 85+ standard endpoints across traffic, environment, sharing engine, and citizen domains.
  • Citizen Portal: APIs for consent management, data access, export, and deletion implement the Data Sovereignty pillar technically.
  • GraphQL: Flexible querying and real-time subscriptions complement REST for complex use cases.
  • OAuth 2.0: Authentication uses OAuth 2.0 with OpenID Connect, supporting multiple grant types for different clients.
  • Scopes: Fine-grained scopes map to data classification levels and domain permissions.
  • Rate Limiting: Tiered rate limits protect the system while enabling legitimate high-volume access.
  • SDKs: Official SDKs for TypeScript, Python, and Rust reduce integration friction.
  • Webhooks: Event-driven integration enables real-time notifications for critical city events.
