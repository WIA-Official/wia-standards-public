Chapter 4: Phase 1 - Data Format Specification

4.1 JSON-LD Foundation

WIA-EDU-007 uses JSON-LD (JSON for Linked Data) as its base format, enabling semantic interoperability across platforms.

Why JSON-LD?

  • Human-readable: Easy for developers to understand and debug
  • Machine-processable: Enables automated validation and semantic querying
  • Extensible: New fields can be added without breaking compatibility
  • W3C standard: Widely adopted across web technologies

4.2 Robot Profile Schema

Every educational robot must publish a standardized profile declaring its capabilities:

{
  "@context": "https://wiastandards.com/contexts/educational-robot/v1",
  "@type": "EducationalRobotProfile",
  "robotId": "EDU-ROBOT-2025-001",
  "robotType": "stem-education",
  "specifications": {
    "ageGroup": "7-12",
    "subjectAreas": ["mathematics", "science", "programming"],
    "interactionModes": ["voice", "touch", "visual"],
    "languages": ["en", "es", "zh", "ko"]
  },
  "capabilities": {
    "teaching": ["adaptive-instruction", "personalized-feedback"],
    "interaction": ["natural-language-processing", "emotion-detection"],
    "assessment": ["real-time-quizzing", "progress-tracking"]
  },
  "safetyCompliance": {
    "certifications": ["COPPA", "GDPR", "ISO-13482"],
    "childSafe": true,
    "supervisionRequired": false
  }
}

4.3 Learning Session Data

Each teaching session generates structured data capturing engagement, assessment, and adaptation:

Field Category Key Metrics
Interaction Engagement score, emotional state, responsiveness
Assessment Pre-test score, post-test score, improvement rate
Adaptation Difficulty adjustments, pace modifications, modality shifts

4.4 Student Progress Tracking

Cumulative progress records enable long-term learning analytics:

  • Metrics: Sessions completed, total learning time, average engagement
  • Skills: Mastered skills, challenge areas, recommended practice
  • Achievements: Credentials earned, mastery levels, learning paths

4.5 Safety & Compliance Metadata

Every robot must declare compliance with child safety and privacy regulations:

  • COPPA (Children's Online Privacy Protection Act - USA)
  • GDPR (General Data Protection Regulation - EU)
  • FERPA (Family Educational Rights and Privacy Act - USA)
  • PIPL (Personal Information Protection Law - China)
💡 Validation: All data must pass JSON Schema validation before acceptance. Invalid data is rejected with clear error messages, preventing data corruption and ensuring interoperability.

Chapter Summary

Key Takeaways:
  • JSON-LD provides semantic, extensible data format
  • Robot profiles declare capabilities and safety compliance
  • Learning sessions capture engagement, assessment, and adaptation
  • Progress tracking enables long-term learning analytics
  • Safety metadata ensures regulatory compliance
