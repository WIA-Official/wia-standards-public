Chapter 6: Phase 3 - Protocol & Real-Time Communication

6.1 WebSocket Protocol

Real-time, bidirectional communication enables responsive teaching interactions.

Connection Flow

  1. Client initiates WebSocket handshake with JWT token
  2. Server validates authentication and establishes connection
  3. Bidirectional message streaming begins
  4. Heartbeat messages maintain connection health
  5. Graceful disconnection with session persistence

6.2 Message Types

Type Direction Purpose
instruction Robot → Student Teaching content delivery
response Student → Robot Student answer/input
emotion Student → Robot Emotional state update
feedback Robot → Student Performance feedback

6.3 Multimodal Interaction

Students and robots communicate through multiple modalities:

  • Voice: Speech-to-text input, text-to-speech output (<500ms latency)
  • Touch: Touchscreen interaction (<100ms latency)
  • Gesture: Hand tracking, body movement (<200ms latency)
  • Visual: Eye tracking, facial expression (<50ms latency)
  • Emotion: Sentiment analysis, affective response (<300ms latency)

6.4 Emotion-Aware Adaptation

Robots detect student emotions and adapt in real-time:

Detected Emotion Robot Response
Frustration Reduce difficulty, provide encouragement
Boredom Increase difficulty, add gamification
Confusion Re-explain concept, add visual aids
Excitement Maintain pace, introduce challenges

6.5 Safety Protocols

Real-time safety monitoring protects students:

  • Content Filtering: Block inappropriate language/topics
  • Emergency Stop: Immediate session halt mechanism
  • Teacher Alerts: Notify supervisors of concerning behavior
  • Privacy Protection: Anonymize all real-time data
Latency Requirements: Voice responses must occur within 500ms, touch feedback within 100ms, and visual updates within 50ms to maintain natural, engaging interaction.

Chapter Summary

Key Takeaways:
  • WebSocket enables real-time bidirectional communication
  • Multiple message types support various interaction patterns
  • Multimodal input/output with strict latency requirements
  • Emotion-aware adaptation personalizes learning
  • Safety protocols protect students in real-time
